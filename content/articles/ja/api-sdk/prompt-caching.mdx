## プロンプトキャッシングとは

Claude API の **プロンプトキャッシング** は、API リクエスト間で同じコンテキストを再利用することで、入力トークンのコストを大幅に削減する機能です。同一のシステムプロンプトやドキュメントを含むリクエストを繰り返し送信する場合、2回目以降は大幅な割引料金が適用されます。

## いつ使うべきか

以下のパターンで特に効果を発揮します。

- 長いシステムプロンプトを全リクエストに含める場合
- 大きなドキュメントを参照しながら複数の質問をする場合
- 同じ Few-Shot 例を繰り返し使う場合
- RAG（検索拡張生成）で頻繁に同じコンテキストを送信する場合

## 基本的な使い方

キャッシュしたい部分に `cache_control` を指定します。

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "あなたは法律アシスタントです。以下の契約書を参照して質問に答えてください。\n\n[3万文字の契約書テキスト]",
            "cache_control": {"type": "ephemeral"}
        }
    ],
    messages=[
        {"role": "user", "content": "この契約の解約条件を要約してください"}
    ]
)
```

### cache_control の種類

- **ephemeral**: 最低5分間キャッシュが保持されます（使用されるたびにTTLがリセット）

## コスト構造

キャッシュの状態によって3種類の料金が発生します。

1. **キャッシュ書き込み**: 通常の入力トークン料金の 25% 増し（初回のみ）
2. **キャッシュ読み取り**: 通常の入力トークン料金の **90% オフ**
3. **通常入力**: キャッシュ対象外の部分は通常料金

つまり、2回目以降の同じコンテキストは 1/10 のコストで処理されます。

## レスポンスでの確認

レスポンスの `usage` フィールドでキャッシュの効果を確認できます。

```json
{
  "usage": {
    "input_tokens": 50,
    "output_tokens": 200,
    "cache_creation_input_tokens": 15000,
    "cache_read_input_tokens": 0
  }
}
```

2回目のリクエスト:

```json
{
  "usage": {
    "input_tokens": 50,
    "output_tokens": 180,
    "cache_creation_input_tokens": 0,
    "cache_read_input_tokens": 15000
  }
}
```

## 設計のベストプラクティス

### キャッシュ可能な部分を前に配置

メッセージ配列の先頭に変化しない部分を配置し、可変部分は後ろに配置します。

### 十分な長さを確保

キャッシュが有効になるには、対象部分が最低 1,024 トークン（Claude Haiku は 2,048 トークン）必要です。短すぎるとキャッシュが作成されません。

### キャッシュブレークポイントの設計

`cache_control` は最大4箇所まで指定できます。システムプロンプト、ドキュメント、Few-Shot 例など、再利用される単位で設定しましょう。

## 実践例: チャットボット

```python
# 長いシステムプロンプト + ドキュメントをキャッシュ
system_message = {
    "type": "text",
    "text": system_prompt + "\n\n" + product_documentation,
    "cache_control": {"type": "ephemeral"}
}

# ユーザーとの会話（変化する部分）
messages = conversation_history + [
    {"role": "user", "content": user_question}
]
```

この設計なら、数万トークンのシステムプロンプトとドキュメントは2回目以降 1/10 のコストで処理されます。

## まとめ

プロンプトキャッシングは、特に長いコンテキストを持つアプリケーションで大きなコスト削減効果をもたらします。システムプロンプトの設計を工夫するだけで導入できるため、API を使う開発者は必ず検討すべき機能です。
