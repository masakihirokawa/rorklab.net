## バッチ処理とは

Claude API の **Message Batches** は、大量のリクエストを一括で送信し、非同期で処理する機能です。通常の API 呼び出しと比較して最大 50% のコスト削減が可能で、24 時間以内に結果が返されます。

## ユースケース

### データ分析

何千件もの顧客レビューの感情分析、テキスト分類、要約など。

### コンテンツ生成

大量の商品説明文、メタデータ、翻訳テキストの一括生成。

### データ変換

構造化されていないテキストデータを JSON やスプレッドシート形式に一括変換。

## 基本的な使い方

### バッチの作成

Python SDK を使った例です。

```python
import anthropic

client = anthropic.Anthropic()

# バッチリクエストの作成
batch = client.messages.batches.create(
    requests=[
        {
            "custom_id": "review-001",
            "params": {
                "model": "claude-sonnet-4-6",
                "max_tokens": 1024,
                "messages": [
                    {"role": "user", "content": "このレビューの感情を分析してください: 素晴らしい商品です！"}
                ]
            }
        },
        {
            "custom_id": "review-002",
            "params": {
                "model": "claude-sonnet-4-6",
                "max_tokens": 1024,
                "messages": [
                    {"role": "user", "content": "このレビューの感情を分析してください: 期待外れでした。"}
                ]
            }
        }
    ]
)

print(f"Batch ID: {batch.id}")
```

### バッチの状態確認

```python
# バッチの処理状態を確認
batch_status = client.messages.batches.retrieve(batch.id)
print(f"Status: {batch_status.processing_status}")
print(f"完了: {batch_status.request_counts.succeeded}")
print(f"失敗: {batch_status.request_counts.errored}")
```

### 結果の取得

```python
# 処理完了後、結果を取得
for result in client.messages.batches.results(batch.id):
    print(f"ID: {result.custom_id}")
    if result.result.type == "succeeded":
        print(f"Response: {result.result.message.content[0].text}")
    else:
        print(f"Error: {result.result.error}")
```

## 設計のポイント

### custom_id の活用

各リクエストに一意の `custom_id` を設定することで、結果を元のデータと紐づけやすくなります。ファイル名、データベース ID、行番号などを使うのが効果的です。

### エラーハンドリング

バッチ内の一部のリクエストが失敗する可能性があります。結果を処理する際は、各リクエストの成否を必ず確認しましょう。

### バッチサイズの最適化

1 つのバッチには最大 10,000 リクエストまで含められます。大量のデータがある場合は複数のバッチに分割します。

## コスト削減のコツ

1. **適切なモデル選択**: 単純な分類タスクには Haiku を使用
2. **max_tokens の設定**: 必要最小限に設定してコストを抑制
3. **バッチの活用**: リアルタイム性が不要なら必ずバッチを使用
4. **プロンプトの効率化**: 簡潔なプロンプトで入力トークンを削減

## まとめ

Message Batches は大規模なデータ処理を行う際の必須機能です。コスト効率が良く、レート制限も緩和されるため、定期的なデータ分析やコンテンツ生成パイプラインに最適です。
