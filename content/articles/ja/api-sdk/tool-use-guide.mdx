## Tool Use とは

Tool Use (関数呼び出しとしても知られている) を使うと、Claude は外部ツールと関数の実行をリクエストできます。Claude がテキストを生成するのではなく、Claude は API を直接呼び出し、関数を実行し、外部システムと対話できます。

<Callout type="info">
Tool Use により Claude は以下をできます:
- データベースと API をクエリ
- ファイルを読み書き
- 計算を実行
- カスタム ビジネス ロジックを実行
- 外部サービスと統合
</Callout>

## 基本的なツール定義

ツールは API リクエストの `tools` パラメータを使用して定義されます。各ツールには名前、説明、入力スキーマがあります。

### ツール スキーマの構造

```json
{
  "name": "get_weather",
  "description": "Get the current weather for a location",
  "input_schema": {
    "type": "object",
    "properties": {
      "location": {
        "type": "string",
        "description": "The city and state/country"
      },
      "unit": {
        "type": "string",
        "enum": ["celsius", "fahrenheit"],
        "description": "Temperature unit"
      }
    },
    "required": ["location"]
  }
}
```

### ツール定義のベストプラクティス

- **明確な名前**: わかりやすいツール名を使う (例: `query_customer_db` ではなく `func1`)
- **詳細な説明**: ツールが何をするか、いつ使うかを説明
- **完全なスキーマ**: すべてのパラメータとその制約を定義
- **必須フィールドを指定**: 提供する必要があるパラメータをマーク
- **例を追加**: 説明に enum 値または制約を含める

## Python での実装

### ツールを定義する

```python
import anthropic
import json

client = anthropic.Anthropic()

tools = [
    {
        "name": "get_user_info",
        "description": "Retrieve user information from the database",
        "input_schema": {
            "type": "object",
            "properties": {
                "user_id": {
                    "type": "string",
                    "description": "The unique user identifier"
                }
            },
            "required": ["user_id"]
        }
    },
    {
        "name": "send_email",
        "description": "Send an email to a user",
        "input_schema": {
            "type": "object",
            "properties": {
                "to": {
                    "type": "string",
                    "description": "Recipient email address"
                },
                "subject": {
                    "type": "string",
                    "description": "Email subject"
                },
                "body": {
                    "type": "string",
                    "description": "Email body content"
                }
            },
            "required": ["to", "subject", "body"]
        }
    }
]
```

### ツール付きリクエストを作成する

```python
messages = [
    {
        "role": "user",
        "content": "Get the user info for user123 and send them a welcome email"
    }
]

response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

print(response.content)
```

## ツール結果を処理する

Claude がツールの使用を決定すると、ツール名と入力パラメータを含む `tool_use` コンテンツ ブロックを返します。その後、ツールを実行して結果を返す必要があります。

### ツール使用レスポンスを処理する

```python
# リクエストを実行する
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

# Claude がツールを使用したいかどうかを確認
if response.stop_reason == "tool_use":
    # ツール使用ブロックを検索
    tool_use_block = next(
        block for block in response.content
        if block.type == "tool_use"
    )

    tool_name = tool_use_block.name
    tool_input = tool_use_block.input

    print(f"Claude wants to use: {tool_name}")
    print(f"With input: {tool_input}")
```

### ツール ハンドラーを実装する

```python
def execute_tool(tool_name: str, tool_input: dict):
    """Execute a tool and return the result"""

    if tool_name == "get_user_info":
        return get_user_info(tool_input["user_id"])
    elif tool_name == "send_email":
        return send_email(
            tool_input["to"],
            tool_input["subject"],
            tool_input["body"]
        )
    else:
        return {"error": f"Unknown tool: {tool_name}"}

def get_user_info(user_id: str):
    """Retrieve user information from database"""
    # データベース呼び出しをシミュレート
    return {
        "user_id": user_id,
        "name": "John Doe",
        "email": "john@example.com"
    }

def send_email(to: str, subject: str, body: str):
    """Send an email"""
    # メール送信をシミュレート
    return {
        "success": True,
        "message": f"Email sent to {to}"
    }
```

## ツール使用を伴うマルチターン会話

ツール使用は通常複数のターンが必要です: Claude がツールをリクエストし、それを実行し、Claude が結果に基づいて続行します。

### 完全なマルチターンの例

```python
def process_tool_use(user_message: str):
    messages = [
        {"role": "user", "content": user_message}
    ]

    while True:
        # Claude からレスポンスを取得
        response = client.messages.create(
            model="claude-opus-4-6",
            max_tokens=1024,
            tools=tools,
            messages=messages
        )

        # Claude がテキストを生成した場合、それを返す
        if response.stop_reason == "end_turn":
            text_blocks = [
                block.text for block in response.content
                if hasattr(block, "text")
            ]
            return "\n".join(text_blocks)

        # Claude がツールを使用したい場合
        if response.stop_reason == "tool_use":
            # ツール使用ブロックを検索
            tool_use_block = next(
                block for block in response.content
                if block.type == "tool_use"
            )

            # ツールを実行
            tool_result = execute_tool(
                tool_use_block.name,
                tool_use_block.input
            )

            # Claude のレスポンスとツール結果をメッセージに追加
            messages.append({"role": "assistant", "content": response.content})
            messages.append({
                "role": "user",
                "content": [
                    {
                        "type": "tool_result",
                        "tool_use_id": tool_use_block.id,
                        "content": json.dumps(tool_result)
                    }
                ]
            })

            # ループを続行
            continue

# 使用してください
result = process_tool_use("Who is user123 and send them a welcome email?")
print(result)
```

## tool_choice パラメータ

`tool_choice` パラメータは、Claude がいつツールを使用できるかを制御します。

### ツール選択オプション

```python
# Claude はツールを使用するか、しないかを選択できます
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice="auto",  # デフォルト: Claude が決定
    messages=messages
)

# Claude はこのリクエストでツールを使用する必要があります
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice="required",  # ツール使用が必須
    messages=messages
)

# Claude は特定のツールを使用する必要があります
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "tool", "name": "get_user_info"},
    messages=messages
)

# Claude はツール (テキストのみ) を使用できません
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    # ツール パラメータなし
    messages=messages
)
```

## TypeScript での実装

### ツールを定義して使用する

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

const tools: Anthropic.Tool[] = [
  {
    name: "get_user_info",
    description: "Retrieve user information from the database",
    input_schema: {
      type: "object",
      properties: {
        user_id: {
          type: "string",
          description: "The unique user identifier"
        }
      },
      required: ["user_id"]
    }
  }
];

async function processToolUse(userMessage: string): Promise<string> {
  const messages: Anthropic.MessageParam[] = [
    { role: "user", content: userMessage }
  ];

  while (true) {
    const response = await client.messages.create({
      model: "claude-opus-4-6",
      max_tokens: 1024,
      tools: tools,
      messages: messages
    });

    if (response.stop_reason === "end_turn") {
      return response.content
        .filter((block): block is Anthropic.TextBlock => block.type === "text")
        .map(block => block.text)
        .join("\n");
    }

    if (response.stop_reason === "tool_use") {
      const toolUseBlock = response.content.find(
        (block): block is Anthropic.ToolUseBlock => block.type === "tool_use"
      );

      if (!toolUseBlock) break;

      const toolResult = executeToolFunction(
        toolUseBlock.name,
        toolUseBlock.input as Record<string, unknown>
      );

      messages.push({ role: "assistant", content: response.content });
      messages.push({
        role: "user",
        content: [
          {
            type: "tool_result",
            tool_use_id: toolUseBlock.id,
            content: JSON.stringify(toolResult)
          }
        ]
      });
    }
  }

  return "Error: Unexpected response";
}

function executeToolFunction(
  name: string,
  input: Record<string, unknown>
): unknown {
  switch (name) {
    case "get_user_info":
      return { user_id: input.user_id, name: "John Doe" };
    default:
      return { error: "Unknown tool" };
  }
}
```

## エラー処理

常にツール実行エラーを適切に処理してください:

```python
def execute_tool_safely(tool_name: str, tool_input: dict):
    """Execute a tool with error handling"""
    try:
        if tool_name == "get_user_info":
            user_id = tool_input.get("user_id")
            if not user_id:
                return {"error": "user_id is required"}
            return get_user_info(user_id)
        else:
            return {"error": f"Unknown tool: {tool_name}"}
    except Exception as e:
        return {"error": f"Tool execution failed: {str(e)}"}
```

エラーを Claude に返す場合、常にツール結果に含めてください:

```python
messages.append({
    "role": "user",
    "content": [
        {
            "type": "tool_result",
            "tool_use_id": tool_use_block.id,
            "content": json.dumps(tool_result),
            "is_error": "error" in tool_result  # 必要に応じてエラーをマーク
        }
    ]
})
```

## ベストプラクティス

1. **明確なツール境界を設計**: 各ツールは 1 つのことを上手に実行する
2. **入力を検証**: 実行前にパラメータをチェック
3. **構造化結果を返す**: 成功/エラー フィールドが明確な JSON を返す
4. **コンテキストを提供**: Claude が使用する明確なエラー メッセージを与える
5. **ツール数を制限**: 3～5 個の設計の良いツールは 20 個の平凡なツールより良い
6. **徹底的にテストする**: 本番前にすべてのツール パスをテスト
7. **パラメータを文書化**: ツール説明を包括的にする
8. **タイムアウトを処理**: ツールに合理的な実行タイムアウトを設定

