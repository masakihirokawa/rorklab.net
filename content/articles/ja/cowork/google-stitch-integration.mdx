## Google Stitch とは

Google Stitch は、Google Labs が2025年5月の Google I/O で発表した、AI駆動のUIデザイン・ウェブ開発ツールです。旧名の Project Antigravity や Project Mariner として知られていたプロジェクトが、ついに一般公開されました。

Stitch は、自然言語プロンプトまたはアップロードされた画像から、本番環境対応のフロントエンドコードと完全なUIデザインを生成します。Google の最新AIモデル Gemini 2.5 Pro のマルチモーダル機能を活用することで、デザイン構想から開発実装へのシームレスな流れを実現しています。

<Callout type="info">
Stitch は Google Labs の実験的プロジェクトとして無料で利用でき、2026年現在でも追加課金なしでアクセスできます。アクセスは [stitch.withgoogle.com](https://stitch.withgoogle.com/) から。
</Callout>

## Google Stitch の主な特徴

### 1. テキストプロンプトからのUI生成
英語で自然言語でアプリケーションを説明するだけで、Stitch が視覚的に完成度の高いUIを生成します。色パレット、レイアウト、ユーザーエクスペリエンスの詳細をプロンプトに含めることで、より正確な結果が得られます。

### 2. 画像入力による高速プロトタイピング
スケッチ、既存アプリのスクリーンショット、ラフなワイヤーフレームなどをアップロードすると、Stitch がそれを認識して、対応するデジタルUIに変換します。手書きのアイデアから数分でプロトタイプが完成します。

### 3. 本番環境対応のコード生成
Stitch が生成する UI は単なるビジュアルデザインではなく、クリーンで機能的なフロントエンドコード（HTML/CSS/React）を含みます。基本的なインタラクティブ要素も自動生成されるため、すぐに開発を進められます。

### 4. Figma へのシームレスなエクスポート
生成されたデザインは Figma に直接貼り付けでき、Auto Layout と編集可能なレイヤーとして利用できます。デザインチームとの協力やデザインシステムへの統合がスムーズです。

### 5. Google Cloud / Firebase との統合
Stitch で生成されたプロジェクトは、Google Cloud や Firebase と統合しやすい構造になっており、バックエンド接続やデータベース連携の準備が整っています。

## Claude Code / Claude Cowork との使い分け

Google Stitch と Claude は、異なる強みを持つ相補的なツールです。併用することで、開発プロセスをより効率的にできます。

| 項目 | Google Stitch | Claude Code |
|------|----------------|-------------|
| **得意な領域** | ビジュアルデザイン、素早いプロトタイピング、UIの自動生成 | ロジック開発、テスト、リファクタリング、大規模プロジェクト |
| **スピード** | 非常に高速（数分でUIが完成） | コンテキストや要件の理解に時間をかけ、品質を重視 |
| **コード品質** | 基本的なUIコード生成。さらなる最適化が必要な場合も | 高品質で保守性の高いコード |
| **複雑なロジック** | 限定的（主にUIインタラクション） | 豊富なサポート |
| **設計パターン** | テンプレートベースの標準的な設計 | 要件に応じた柔軟な設計 |
| **テスト・デバッグ** | 基本的なUIテストのみ | ユニットテスト、エンドツーエンドテスト対応 |

<Callout type="tip">
Stitch はビジュアルファーストの思考に最適です。一方、Claude は**問題解決ファースト**です。両者を組み合わせることで、見た目も内側も優れたアプリケーションが実現できます。
</Callout>

## ワークフロー例1: Stitch でビジュアル設計 → Claude Code でロジック統合

これは**デザインファースト**のアプローチです。

### ステップ1: Stitch でUIプロトタイプを作成
1. アプリケーションのコンセプトを自然言語で説明
2. 「カラフルなタスク管理アプリ、ダークモード対応、リアルタイム同期機能」などと指定
3. 数分以内に複数のデザイン案が生成される
4. 気に入ったものを選択し、HTML/CSS/React コードをエクスポート

### ステップ2: Claude Code で機能を拡張
1. Stitch から出力されたコードを Claude Code にアップロード
2. 以下のようなリクエストを実行：
   - バックエンド API との連携ロジック追加
   - 状態管理（Redux, Zustand など）の実装
   - エラーハンドリングと入力バリデーション
   - ユニットテストの作成
   - パフォーマンス最適化

### ステップ3: 必要に応じて Stitch に戻って調整
Claude Code での機能追加後、UIの細かい調整が必要になれば、再び Stitch に戻ってビジュアルを修正。生成されたコードを Claude Code に統合し直します。

**このワークフローに適したプロジェクト例：**
- MVP（最小実行可能製品）の素早い立ち上げ
- スタートアップのプロトタイプ開発
- B2Cアプリケーション（ビジュアル重視）

## ワークフロー例2: Claude Code で構築 → Stitch でビジュアル調整

これは**ロジックファースト**のアプローチです。主に既存プロジェクトや複雑なシステムに適しています。

### ステップ1: Claude Code でバックエンド・ロジック構築
1. データベーススキーマ設計
2. API エンドポイント実装
3. 認証・認可ロジック
4. ビジネスロジックの実装
5. テストスイート作成

### ステップ2: Stitch でフロントエンドを生成
1. Claude Code で定義した API 仕様をもとに、Stitch へのプロンプトを作成
2. エンドポイントの仕様、必要なフォーム、表示データを説明
3. Stitch がそれを受け取り、適切な React コンポーネントを生成
4. 生成されたコードが Claude Code のバックエンドと連携するよう、簡単な調整のみが必要

### ステップ3: 統合テストとUI調整
Claude Code で統合テストを実施し、フロントエンドとバックエンドが正常に連携することを確認。細かいUIの改善が必要なら、再度 Stitch を使用。

**このワークフローに適したプロジェクト例：**
- Enterprise 向けアプリケーション（ロジック複雑）
- マイクロサービスアーキテクチャ
- API主導の開発
- 既存バックエンドへのフロントエンド追加

## MCP 連携の可能性

Claude の MCP（Model Context Protocol）サーバーを通じて、Stitch プロジェクトをより統合されたワークフローで操作することが可能になる見込みです。

今後の展開として期待される機能：
- Stitch で生成されたプロジェクトを自動的に Claude Code にインポート
- Claude Code 内から Stitch の生成機能を直接呼び出し
- プロジェクトファイルの自動同期
- 変更履歴の統合管理

<Callout type="info">
MCP 連携はロードマップの段階で、2026年内の実装を予定しています。進捗は Claude Lab の公式ブログで発表されます。
</Callout>

## 他のAIコーディングツールとの比較

Google Stitch は市場内で独特なポジションを占めています。

| ツール | Stitch | Cursor | Replit | Bolt.new |
|--------|--------|--------|--------|----------|
| **主な特徴** | UIビジュアル生成・ビジュアル駆動 | コード エディタ + AI | フルスタック IDE | フルスタックアプリ生成 |
| **得意な領域** | フロントエンド、プロトタイピング | コード編集・補完 | 小～中規模プロジェクト | 完全なアプリケーション生成 |
| **ラーニングカーブ** | 非常に低い | 中程度 | 低～中程度 | 低い |
| **本番環境対応** | 初期段階のコード出力 | 高い | 中程度 | 中程度 |
| **チーム向け** | △（デザイナー向け） | ○ | ○ | △ |
| **価格** | 無料（Google Labs） | 有料 | 無料～有料 | 無料～有料 |

Stitch は**デザイナーと開発者の橋渡し**となるツールで、特に **UI生成の速度と品質** で他を圧倒しています。

## ベストプラクティス：効率的な併用のための Tips

### 1. 目的に応じて正しいツールを選択する
| 質問 | 答え | 使うツール |
|------|------|-----------|
| 素早くビジュアルをプロトタイプしたい？ | はい | Stitch |
| 複雑なビジネスロジックが必要？ | はい | Claude Code |
| UIの微調整が中心？ | はい | Stitch |
| 既存コードの改善・リファクタリング？ | はい | Claude Code |

### 2. プロンプトの詳細さが重要
**Stitch の場合：**
```
悪い例：「タスク管理アプリを作って」

良い例：「Trello風のカンバンボード。カラフルなカード型タスク、
ダークモード対応、リアルタイム同期機能。
ヘッダーは紺色、ボタンはグラデーション付きの青。」
```

**Claude Code の場合：**
```
悪い例：「ログイン機能を作って」

良い例：「JWT トークンを使用した OAuth 2.0 対応の認証。
ユーザー登録時にメール確認。セッションタイムアウトは30分。
デバイス紛失時のセキュアなログアウト機構。」
```

### 3. バージョン管理とイテレーション
- Stitch で生成したコードは必ず Git で管理
- Claude Code でのカスタマイズ後も各バージョンを保存
- 「Stitch 生成 → Claude 拡張 → Stitch 調整」のサイクルを繰り返す際は、各段階の成果物を明確に記録

### 4. チームワークの工夫
```
デザイナー → Stitch でUIプロトタイプ作成
         ↓（Figma エクスポート）
        チーム間で承認
         ↓
開発者 → Claude Code で機能化
         ↓（デプロイ）
テスター → 統合テスト実施
         ↓（必要に応じて）
デザイナー → Stitch で微調整
```

### 5. 出力コードの品質チェック
Stitch の出力は基本的に良質ですが、以下の点をチェック：
- **アクセシビリティ**：ARIA ラベルが適切か？
- **パフォーマンス**：不要な再レンダリングはないか？
- **セキュリティ**：XSS 脆弱性の可能性？
- **レスポンシブ設計**：モバイル・タブレット対応？

Claude Code でこれらを一括チェックし、改善することをお勧めします。

### 6. ドメイン知識の活用
- **単純な CRUD アプリ**：Stitch → Claude Code → 完成
- **規制業界アプリ**：Claude Code → Stitch（必要に応じて）
- **マルチテナント SaaS**：Claude Code がメイン、Stitch はプロトタイプのみ
- **ゲーム・3D 描画**：Stitch は不向き、Claude Code 推奨

## Stitch を使うときの注意点

<Callout type="warning">
Google Stitch は**ビジュアル生成に特化**しており、以下の領域では制限があります：
- 複雑なステート管理ロジック
- バックエンド API 統合（初期段階）
- 高度なセキュリティ要件
- パフォーマンス最適化の詳細
- レガシーコード互換性

これらが必要な場合は、必ず Claude Code で追加実装してください。
</Callout>

## まとめ

Google Stitch と Claude Code は、互いに補完し合うツールです：

- **Stitch は「速さ」と「ビジュアル」に最適** — プロトタイプ、UI生成、デザイン実装
- **Claude Code は「質」と「複雑性への対応」に最適** — ロジック、テスト、スケーラビリティ

両者を組み合わせることで、**スタートアップから Enterprise まで、あらゆる規模のプロジェクト** に対応できる、強力で柔軟な開発パイプラインが実現できます。

2026年は、AI コーディングツールの多様化と専門化が進む年になります。Stitch のようなビジュアル特化ツール、Claude Code のようなロジック特化ツール、そして用途別の多くの選択肢が存在する中で、**自分のプロジェクトに最適なツールの組み合わせ**を見つけることが、開発効率の最大化につながるのです。
