## Cowork とは

Cowork は Anthropic のデスクトップ アプリケーションで、Claude の機能をファイル システムとコンピューターに直接もたらします。ワークスペースを離れることなく、ファイルを管理し、バッチ操作を実行し、形式を変換し、繰り返しのデスクトップ タスクを自動化できます。

<Callout type="info">
Cowork はコンピューターでローカルに実行され、ファイル システムにアクセスできます。Claude がアクセスできるディレクトリと実行できる操作を完全に制御できます。
</Callout>

## Cowork のインストール

### システム要件

- macOS 11 以降、Windows 10 以降、または Linux (Ubuntu 20.04 以降)
- RAM 最小 4GB
- ディスク空き容量 500MB
- Anthropic アカウント (API キー付き)

### インストール手順

1. **ダウンロード** [anthropic.com](https://anthropic.com) から Cowork アプリケーションをダウンロードします
2. **インストール** プラットフォーム固有のインストーラーを使用してインストール
3. **起動** アプリケーション フォルダから Cowork を起動
4. **認証** Anthropic アカウントで認証
5. **フォルダ アクセスを許可** 使用するディレクトリへのアクセスを許可

## フォルダ アクセスについて

Cowork の主要なセキュリティ機能の 1 つは、きめ細かいフォルダ アクセス制御です。Claude がアクセスできるディレクトリを明示的に選択します。

### フォルダ アクセスをリクエスト

Claude にファイルを操作するよう求める場合:

1. Claude は特定のフォルダへのアクセスをリクエスト
2. 確認を求めるダイアログが表示される
3. フォルダを参照して選択する
4. Claude はそのフォルダとそのコンテンツへのアクセスを取得
5. いつでも設定でアクセスを取り消すことができます

### フォルダ アクセスのベストプラクティス

- **最小限のアクセスを許可**: Claude が操作する必要があるフォルダへのアクセスのみを許可
- **ファイルを整理する**: 関連ファイルを同じフォルダに保つ
- **センシティブ フォルダを避ける**: 認証情報または設定ディレクトリへのアクセスを許可しない
- **定期的に権限を確認**: Claude がアクセスできるフォルダを定期的に確認

## 一般的なファイル操作

### ファイルを読む

Claude にファイルを読んで分析するよう依頼する:

```
CSV ファイル「sales_data.csv」を読み、主要メトリクスを要約してください。
```

Claude は以下を読む場合があります:
- テキスト ファイル (.txt、.md)
- コード ファイル (.py、.js、.java など)
- 構造化データ (.csv、.json)
- ドキュメント (.pdf、.docx)

### ファイルを作成して書き込む

Claude は認可されたフォルダに新しいファイルを作成できます:

```
「analysis.py」という Python スクリプトを作成し、データを処理します。
```

Claude は以下を実行します:
- 指定されたコンテンツでファイルを作成
- 適切なフォーマットと構文を使用
- ファイルを正しい場所に配置
- 保存前にコンテンツを表示

### ファイルをコピーして移動する

Claude にファイルを再編成するよう依頼して、ファイルを整理します:

```
Downloads フォルダからすべての PNG ファイルを新しい「Images」サブフォルダーにコピーします。
```

### ファイルの一括名前変更

Claude は複数のファイルを一度に名前変更できます:

```
このフォルダのすべての PDF ファイルに今日の日付をファイル名に含めるように名前変更してください。
「document.pdf」を「document_2026-03-09.pdf」に変更
```

## ファイル形式変換

Cowork の最も強力な機能の 1 つは、バッチ ファイル形式変換です。Claude は多くの形式を自動的に変換できます。

### サポートされている変換

| From | To | 一般的な用途 |
|------|----|----|
| CSV | JSON、Excel、PDF | データ移行、API 統合 |
| Markdown | HTML、PDF、DOCX | 公開、ドキュメント |
| 画像 | PNG、JPG、WEBP | 圧縮、形式の標準化 |
| JSON | CSV、Excel | データ分析、スプレッドシート インポート |
| オーディオ | MP3、WAV、OGG | 圧縮、互換性 |

### 例: CSV から JSON へ

```
「sales_data.csv」を JSON 形式に変換します。
構造をフラットに保ち、わかりやすいフィールド名を使用してください。
「sales_data.json」として保存します。
```

Claude は、データ整合性を保ちながら変換を自動的に処理します。

### 例: Markdown から PDF へ

```
このフォルダのすべての .md ファイルを PDF に変換します。
標準的なフォントでプロフェッショナルな形式を適用します。
PDF を同じファイル名で保存します。
```

### バッチ形式変換

複数のファイルを一度に変換します:

```
20 個の TIFF 画像を JPG 形式に変換する必要があります。
それらを 80% の品質に圧縮し、同じフォルダに保つ。
各ファイル名に「_compressed」を追加します。
```

## バッチ処理操作

### ファイルの検索とフィルター

Claude は条件に基づいて特定のファイルを見つけることができます:

```
過去 7 日間に変更されたすべての Excel ファイルを検索します。
サイズと作成日を一覧表示します。
```

### 種類別に整理する

ファイルをカテゴリー化されたフォルダに自動的に並べ替えます:

```
自分の Downloads フォルダを整理します。
サブフォルダを作成: ドキュメント、画像、ビデオ、アーカイブ、その他。
ファイルを種類に応じて適切なフォルダに移動します。
```

### 重複を整理する

重複ファイルを検索して管理します:

```
My Documents フォルダの重複ファイルを検索 (同じサイズとコンテンツ)。
一覧表示して、どれを保持するかを表示します。
```

### ファイルからデータを抽出する

複数のファイルを処理し、情報を抽出します:

```
このフォルダのすべての PDF 請求書をスキャンします。
請求書番号、日付、合計金額を抽出します。
結果を含む CSV サマリーを作成します。
```

休暇からの 50 枚の写真があります。
1. すべてを 1920x1080 にリサイズ
2. 「© 2026」透かしを追加
3. 90% の品質で JPG に変換

```

Claude はこれらの複数ステップ操作を効率的に処理できます。

### ドキュメント処理

```
Word ドキュメントのフォルダがあり、以下が必要です:
1. markdown に変換
2. 見出しを # 構文を使用するように再フォーマット
3. コード ブロックを ``` でマークする
4. .md ファイルとして保存
```

### データ抽出

```
このフォルダの PDF からすべてのメール アドレスを抽出します。
また、電話番号を (XXX) XXX-XXXX 形式で抽出します。
列を含む CSV を作成: source_file、email、phone。
```

## ワークスペース設定と構成

### プロジェクト ワークスペースを作成する

プロジェクト用の専用ワークスペースを設定します:

```
「Website Redesign」プロジェクト用に新しいフォルダ構造を作成します:
- /designs (モックアップとスクリーンショット用)
- /assets (画像、アイコン、フォント用)
- /content (テキストと markdown 用)
- /exports (最終成果物用)
```

### 日付別に整理する

時系列の整理を維持します:

```
アーカイブを年月別に並べ替えます。
フォルダを作成: 2024/01、2024/02、...2026/03
作成日に基づいて、ファイルを適切な日付フォルダに移動します。
```

### ドキュメント テンプレート

繰り返されるタスク用の再利用可能なテンプレートを作成します:

```
次を含むプロジェクト テンプレート フォルダを作成します:
- README.md (プロジェクトの説明)
- CHANGELOG.md (バージョン履歴)
- TODO.md (タスク リスト)
- .gitignore (標準 git ignores)
```

## セキュリティに関する注意事項

### センシティブ ファイルを保護する

- **フォルダへのアクセスを許可しない** パスワードまたは認証情報を含むフォルダ
- **暗号化を使用** 処理前にセンシティブなドキュメント
- **Claude が処理するファイルを確認** アクセス共有前
- **スコープを制限**: 必要なフォルダへのアクセスのみを付与

### ベストプラクティス

1. **認証情報を分離**: API キーとパスワードを Cowork のアクセス外に保存
2. **操作を監査**: 重大なファイル操作を実行前に確認
3. **重要なファイルをバックアップ**: バッチ操作前にバックアップを保管
4. **アクセスを取り消す**: 不要になったらフォルダ権限を削除
5. **活動を監視**: 定期的に操作ログを確認

<Callout type="warning">
Cowork は実際のファイル操作を実行します。削除または大規模な変更を伴うバッチ操作を確認する前に、Claude が何をするかを常に確認してください。
</Callout>

## 一般的な問題のトラブルシューティング

### 権限拒否エラー

**問題**: フォルダにアクセスするときに「権限がありません」

**解決策**:
- Cowork 設定でフォルダ アクセスを許可
- システム上のファイル/フォルダ権限を確認
- 読み取り/書き込み権限があることを確認

### ファイル エンコーディングの問題

**問題**: テキスト ファイルが正しく表示されない

**解決策**:
- ファイルが UTF-8 エンコードされていることを確認
- 処理前に UTF-8 に変換してみてください
- ファイル拡張子が実際の形式と一致していることを確認

### 変換の失敗

**問題**: ファイル変換が完了しない

**解決策**:
- ファイル整合性を確認 (ファイルが破損していない)
- 十分なディスク容量を確認
- 多くのファイルを処理する場合は、より小さなバッチを試してください
- ファイル形式の互換性を確認

## パフォーマンス ヒント

1. **合理的なバッチ サイズで作業**: 一度に数千ではなく 50～100 個のファイルを処理
2. **不要なアプリケーションを閉じる**: 変換のためのシステム リソースを解放
3. **可能な限り SSD を使用**: 高速ストレージはパフォーマンスを向上させる
4. **システム リソースを監視**: 大規模操作中に CPU とディスク使用率を確認
5. **大規模なプロジェクトを分割**: より小さく管理可能なタスクに分割
