## スキルとは何か

Claude Code や Cowork は、標準で多くの機能を備えていますが、「スキル」を追加することで、さらに専門的なタスクへ対応させることができます。スキルとは、SKILL.md というマークダウンファイルに書かれた指示セットのことで、Claude が特定の場面でどのように振る舞うべきかを定義します。

たとえば「コードレビュー時にはセキュリティと可読性を重視する」「デプロイ時にはテストを実行してから本番にプッシュする」といった、チームや個人のワークフローに合わせたカスタムな指示を、スキルとして保存・再利用できます。

スキルは [Agent Skills](https://agentskills.io) というオープンスタンダードに準拠しており、Claude Code 以外の AI ツールとも互換性があります。

## スキルの基本構造

スキルは「ディレクトリ + SKILL.md」というシンプルな構成です。

```
my-skill/
├── SKILL.md           # メインの指示ファイル（必須）
├── reference.md       # 詳細なリファレンス（オプション）
├── examples/
│   └── sample.md      # 出力サンプル（オプション）
└── scripts/
    └── helper.sh      # 実行スクリプト（オプション）
```

SKILL.md の中身は、YAML フロントマターとマークダウンの指示文で構成されます。

```yaml
---
name: code-review
description: コードの品質とセキュリティをレビューする。PRレビュー時に使用。
---

コードレビュー時に以下を確認してください：

1. **コード構造**: 関数やモジュールの責務が明確か
2. **エラーハンドリング**: 例外処理が適切か
3. **セキュリティ**: SQLインジェクションやXSSの脆弱性がないか
4. **テストカバレッジ**: 変更に対するテストが書かれているか
```

## フロントマターの主なフィールド

SKILL.md のフロントマターでは、スキルの挙動を細かく制御できます。

`name` はスキルの表示名で、`/name` というスラッシュコマンドになります。`description` は Claude がスキルを自動で呼び出すかどうかを判断するための説明文です。`disable-model-invocation` を `true` に設定すると、ユーザーが明示的に `/name` で呼び出したときだけ実行されるようになります。デプロイやコミットのように副作用があるワークフローに適しています。

`allowed-tools` にはスキル実行中に許可なしで使えるツールを指定でき、`context` を `fork` にするとサブエージェントとして分離実行されます。

## スキルの保存場所

スキルを置く場所によって、利用範囲が変わります。

個人用スキルは `~/.claude/skills/<skill-name>/SKILL.md` に置くと、すべてのプロジェクトで使えます。プロジェクト用スキルは `.claude/skills/<skill-name>/SKILL.md` に置くと、そのプロジェクト内でのみ有効です。プラグインに含まれるスキルは `<plugin>/skills/<skill-name>/SKILL.md` に配置し、プラグインが有効な場所で使えます。

Enterprise 設定で配布すれば、組織全体のユーザーが利用可能です。

## スキルの呼び出し方

スキルを呼び出す方法は 2 つあります。

手動で呼び出す場合は、スラッシュコマンドを使います。

```
/code-review src/auth/login.ts
```

自動呼び出しの場合は、Claude が会話の文脈から適切なスキルを判断し、自動的に読み込みます。たとえば「このコードをレビューして」と尋ねると、`description` に「コードレビュー」「PRレビュー」と書かれたスキルが自動的にアクティブになります。

引数を渡す場合は `$ARGUMENTS` プレースホルダーが使えます。

```yaml
---
name: fix-issue
description: GitHub Issue を修正する
disable-model-invocation: true
---

GitHub Issue $ARGUMENTS を以下の手順で修正してください：

1. Issue の内容を確認する
2. 修正を実装する
3. テストを書く
4. コミットを作成する
```

`/fix-issue 123` と実行すると、`$ARGUMENTS` が `123` に置き換わります。

## 実践例：よく使うスキルを作る

### 例 1：コミットメッセージの規約を守るスキル

```yaml
---
name: commit
description: コミットメッセージを Conventional Commits 形式で作成する
disable-model-invocation: true
---

変更内容を分析し、Conventional Commits 形式でコミットしてください。

形式: <type>(<scope>): <description>

type は以下から選択：
- feat: 新機能
- fix: バグ修正
- docs: ドキュメントの変更
- refactor: リファクタリング
- test: テストの追加・修正
- chore: ビルドやツールの変更
```

### 例 2：動的コンテキストを使ったPRレビュースキル

```yaml
---
name: pr-summary
description: PR の変更内容を要約する
context: fork
agent: Explore
---

## PRの情報
- 差分: !`gh pr diff`
- コメント: !`gh pr view --comments`
- 変更ファイル: !`gh pr diff --name-only`

## タスク
この PR の変更内容を簡潔に要約してください。
```

`!`command`` 構文は、スキルが読み込まれる前にシェルコマンドを実行し、その結果をスキルの中に挿入します。

## プラグインとは

プラグインは、スキル・エージェント・フック・MCPサーバーなどを一つのパッケージにまとめた拡張機能です。個人で使うだけなら `.claude/skills/` にスキルを置けば十分ですが、チームで共有したり、コミュニティに配布したい場合はプラグインとしてパッケージ化します。

プラグインの基本構造は以下の通りです。

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json       # プラグインメタデータ
├── skills/
│   └── review/
│       └── SKILL.md
├── agents/
│   └── reviewer.md
├── hooks/
│   └── hooks.json
├── .mcp.json             # MCPサーバー設定
└── .lsp.json             # LSPサーバー設定
```

`plugin.json` にはプラグイン名、説明、バージョンなどのメタデータを記述します。

```json
{
  "name": "my-team-tools",
  "description": "チーム開発用のスキルとフックのセット",
  "version": "1.0.0",
  "author": {
    "name": "Your Team"
  }
}
```

## プラグインのインストールと管理

プラグインのインストールはCLIから行います。

```bash
claude plugin install <plugin-name>@<marketplace>
```

スコープを指定することで、インストール範囲を制御できます。`--scope user` は全プロジェクトで利用可能（デフォルト）、`--scope project` はチーム共有用（`.claude/settings.json` に記録）、`--scope local` はプロジェクトローカル（gitignore 対象）です。

プラグインの無効化・有効化やアンインストールも CLI で簡単に行えます。

```bash
claude plugin disable <plugin-name>
claude plugin enable <plugin-name>
claude plugin uninstall <plugin-name>
```

## プラグインマーケットプレイスと skills.sh

プラグインの配布には「マーケットプレイス」という仕組みが使われます。マーケットプレイスは、プラグインのカタログを管理する Git リポジトリや URL エンドポイントです。

公式マーケットプレイスは Anthropic が運営しており、Claude Code の `/plugin` メニューの Discover タブからブラウズできます。自分のプラグインを公式マーケットプレイスに登録するには、[claude.ai/settings/plugins/submit](https://claude.ai/settings/plugins/submit) から申請できます。

### skills.sh — コミュニティのスキルハブ

[skills.sh](https://skills.sh) は、Claude Code 向けのスキルやプラグインが公開されているコミュニティハブです。ユーザーが作成したスキルを閲覧・検索でき、GitHub リポジトリと連携して簡単にインストールできます。

たとえば、エンジニアリング、マーケティング、コンプライアンスなど、さまざまな分野のプロダクションレディなスキルが 100 以上公開されています。気に入ったスキルやプラグインは、マーケットプレイスとして Claude Code に登録してインストールできます。

### チーム独自のマーケットプレイス

チーム内で独自のプラグインを共有する場合は、GitHub リポジトリにマーケットプレイスを作成することもできます。`marketplace.json` にプラグインのカタログを記述し、チームメンバーがリポジトリ URL を登録するだけで、`/plugin` メニューからインストール可能になります。

## MCP（Model Context Protocol）連携

MCP は AI ツールを外部のデータソースやサービスに接続するためのオープンプロトコルです。Claude Code は MCP を通じて、Google Drive、Slack、Jira、GitHub、データベースなど、数百のサービスと連携できます。

### MCP サーバーの設定

MCP サーバーの追加は `claude mcp add` コマンドで行います。

```bash
# GitHub MCP サーバーを追加
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Figma MCP サーバーを追加
claude mcp add figma -- npx -y figma-developer-mcp --figma-api-key=YOUR_KEY

# PostgreSQL データベースに接続
claude mcp add postgres -- npx -y @modelcontextprotocol/server-postgres postgresql://localhost/mydb
```

プロジェクト共有用の設定ファイル `.mcp.json` を使うこともできます。

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

`${GITHUB_TOKEN}` のように環境変数を使えるため、API キーをファイルに直接書かずにチームで設定を共有できます。

### MCP の活用例

GitHub MCP を設定すると、「Issue #42 を修正して PR を出して」のように自然言語で指示するだけで、Claude が Issue の内容を読み取り、修正を実装し、プルリクエストを作成するところまで自動化できます。

Figma MCP を使えば、デザインファイルから直接コンポーネント情報やスタイル情報を取得し、それに基づいたコード生成が可能です。デザインと実装の橋渡しが格段にスムーズになります。

Slack MCP では、特定のチャンネルのメッセージを分析したり、Claude の作業結果を自動で通知したりできます。

Google の [Stitch](https://stitch.ai) MCP を使えば、テキストプロンプトから UI デザインを生成し、そのまま Claude Code でコーディングに活用するワークフローが構築できます。デザインのブラウズ、スクリーンの確認、デザイントークンの抽出を、チャット画面の中で直接行えます。

モバイルアプリ開発では、[Rork](https://rork.com) との連携も注目されています。Rork Max は Claude Code と Opus 4.6 をバックエンドに使い、アプリの説明文だけでネイティブ Swift アプリを自動生成します。UIからナビゲーション、データ永続化、Apple API 連携まで、一貫して AI が処理します。

### プラグインと MCP の組み合わせ

プラグインには `.mcp.json` を含めることができるため、スキルと MCP サーバーをセットで配布できます。たとえば「Figmaデザインレビュー」プラグインを作成し、Figma MCP サーバーの設定とレビュー用スキルをまとめてチームに配布する、といった使い方が可能です。

## Cowork でのスキル活用

Cowork 環境でも、スキルは重要な役割を果たしています。Cowork には PDF 処理、Excel 作成、PowerPoint 生成などの組み込みスキルが用意されており、これらは Claude が適切なタイミングで自動的に読み込みます。

たとえば「Excelファイルを作って」と依頼すると、xlsx スキルが自動的にアクティブになり、適切なライブラリのインストールやフォーマット設定を含む高品質なスプレッドシートが生成されます。

独自のスキルを Cowork に追加することもできます。作業フォルダに `.claude/skills/` ディレクトリを作成し、SKILL.md を配置すれば、Cowork のセッションでもスキルが利用可能になります。

## バンドルスキル

Claude Code にはプリインストールされたバンドルスキルがいくつか含まれています。

`/simplify` は最近変更したファイルをコード再利用・品質・効率の観点からレビューし、改善を適用します。3つのレビューエージェントを並列実行して結果を集約するため、手動で行うよりもはるかに効率的です。

`/batch` はコードベース全体に対する大規模な変更を並列で調整します。変更の説明を提供すると、コードベースを分析してタスクを分解し、各タスクを独立した git worktree で並列実行します。たとえば `/batch migrate src/ from Solid to React` のように使います。

`/debug` は現在のClaude Codeセッションのデバッグログを分析し、問題のトラブルシューティングを支援します。

## まとめ

Claude Code のスキルとプラグインシステムは、AI の機能を自分のニーズに合わせて拡張するための強力なツールです。SKILL.md ファイルを1つ書くだけで、Claude の振る舞いをカスタマイズできます。

まずは個人用スキルから始めて、便利だと感じたらチームで共有し、さらに広く役立つものであればプラグインとして公開するという段階的なアプローチがお勧めです。日常の開発ワークフローで「いつも同じことを指示している」と感じる部分があれば、それはスキル化の良い候補です。
