Rork を使ったことがある方なら、チャット欄に「ログインボタンを青くして」と入力するだけで UI が変わる体験をしたことがあると思います。これが Rork Companion と呼ばれる機能です。

単なる命令実行ツールではなく、文脈を理解してアプリの改善提案をしてくれる AI アシスタントとして機能します。この記事では、Rork Companion を最大限活用するためのコツを実例を交えて解説します。

## Rork Companion の基本的な動作

Rork Companion は Rork のプレビュー画面の横に表示されるチャットパネルです。自然言語でアプリへの指示を入力すると、AI がコードを生成・変更してリアルタイムでプレビューに反映されます。

できることの範囲は広く、以下のような操作が可能です。

- **UI の変更**: 色、フォント、レイアウト、コンポーネントの追加・削除
- **機能の追加**: 「お気に入り機能を追加して」「検索バーを上部に配置して」
- **データの操作**: 「サンプルデータを5件追加して」「一覧の並び順を逆にして」
- **バグの修正**: 「ボタンをタップしても何も起きない問題を直して」

## 効果的なプロンプトの書き方

Rork Companion は高性能ですが、曖昧な指示だと意図と異なる変更をすることがあります。以下のポイントを意識すると精度が上がります。

**具体的に場所を指定する**:
```
❌「ボタンをきれいにして」
✅「ホーム画面の一番下にある『投稿する』ボタンのテキストを白にして、背景色を #4F46E5 にして」
```

**変更前の状態も伝える**:
```
❌「色を変えて」
✅「現在グレーになっているカードの背景色を白に変えて、影（シャドウ）も追加して」
```

**一度に複数の変更を頼まない**:
複数の変更を同時に頼むと、どれかが意図と異なる可能性が高まります。1 つずつ確認しながら進めるほうが安全です。

## 実践的な活用シナリオ

**シナリオ 1: プロトタイプの高速作成**

```
Companion: 「ユーザー登録フォームを作って。メールアドレスとパスワードの入力欄と、
           利用規約への同意チェックボックスと、登録ボタンを含めて」

→ フォーム画面が生成される

Companion: 「メールアドレスの入力欄の下に、
           リアルタイムでメール形式を検証する小さなエラーメッセージを追加して」

→ バリデーション付きフォームに更新される
```

**シナリオ 2: デザインの微調整**

```
Companion: 「一覧の各アイテムの左に、カテゴリによって色が変わるドットを追加して。
           カテゴリ『仕事』は青、『プライベート』は緑、『その他』はグレー」
```

**シナリオ 3: バグの報告と修正**

```
Companion: 「プロフィール画面に戻るボタンがなくて、
           ナビゲーションで戻れなくなっています。戻るボタンを追加してください」
```

## Companion を使うときの注意点

**変更履歴を確認する習慣をつける**

Rork Companion が変更したコードは、Rork の「履歴」パネルで確認できます。意図しない変更があった場合は、すぐに「元に戻す」で戻してください。

**大きな変更は分割して頼む**

「アプリ全体のデザインをモダンにして」のような大きな指示は、どこが変わるか予測が難しくなります。画面ごと、セクションごとに分けて進めるほうが結果をコントロールしやすいです。

**コンポーネントの名前を使う**

Rork で命名したコンポーネント名を Companion への指示に使うと、より正確に変更箇所を指定できます。

## まとめ

Rork Companion は、コードを書かなくても AI との対話でアプリを作り上げていける機能です。プロンプトの精度を上げることで、理想の UI に素早く近づけることができます。
