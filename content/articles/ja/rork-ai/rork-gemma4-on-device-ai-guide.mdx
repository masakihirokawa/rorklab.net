## Rork × Gemma 4 — なぜ今この組み合わせが注目されるのか

2026年4月2日、Google が「**Gemma 4**」をリリースしましました。Apache 2.0 ライセンスで提供されるこのオープンモデルは、E2B（約1.2GB）という超軽量バリアントから始まり、スマートフォン上でもオンデバイス推論が可能な新しい世代のモデルです。そして Rork（ならびに Rork Max）とこのモデルを組み合わせることで、これまでとは全く異なるレベルの AI モバイルアプリ開発が実現できるようになりましました。

なぜこの組み合わせが強力なのでしょうか。Rork はすでに Claude Opus 4.6 や Gemini API など複数の AI エンジンと連携したアプリ生成を得意としています。しかし、これらはすべて「クラウド AI」です。**ユーザーのデータがサーバーに送られる**、**ネットワーク接続が必須**、**リクエストごとに API 費用が発生する** — これらの課題を Gemma 4 のオンデバイス推論は一気に解決します。

Rork で構築したモバイルアプリの中で、Gemma 4 をデバイス上で動かせば：

- 個人情報（日記・健康データ・財務記録）をクラウドに送らずに AI 処理できる
- 飛行機内・地下・山岳地帯など通信不能な環境でも AI 機能が動作する
- 月額 API 費用を大幅に削減し、無料プランのユーザーにも AI 機能を提供できる

## Rork × Gemma 4 の技術的な仕組み

### アーキテクチャの概要

Rork で作成したアプリ（React Native / Expo ベース）に Gemma 4 を組み込む方法は主に2つあります。

**方法1: Expo + ONNX Runtime Mobile（推奨）**
Gemma 4 の ONNX 形式モデルを使い、React Native から ONNX Runtime Mobile 経由で推論を実行します。Android・iOS 両方に対応しており、比較的セットアップが簡単です。

**方法2: Rork Max + ネイティブモジュール（iOS: Core ML、Android: LiteRT）**
Rork Max で生成した Swift（iOS）または Kotlin（Android）コードに、Core ML（Apple）または LiteRT（Google）ネイティブ API を使った推論モジュールを組み込みます。より高いパフォーマンスが得られますが、プラットフォーム別の実装が必要です。

### 方法1: Expo + ONNX Runtime Mobile の実装

```typescript
// Rork で生成した React Native アプリへの Gemma 4 組み込み
// 1. パッケージインストール: npx expo install onnxruntime-react-native
// 2. モデルファイル: Gemma 4 E2B ONNX 形式を assets/ に配置

import { InferenceSession, Tensor } from 'onnxruntime-react-native';
import * as FileSystem from 'expo-file-system';
import { Asset } from 'expo-asset';

interface Gemma4Config {
  maxNewTokens: number;
  temperature: number;
  topP: number;
}

class Gemma4Engine {
  private session: InferenceSession | null = null;
  private tokenizer: any = null;  // Gemma トークナイザー
  private isInitialized = false;

  async initialize(config: Gemma4Config = {
    maxNewTokens: 256,
    temperature: 0.7,
    topP: 0.9
  }): Promise<void> {
    if (this.isInitialized) return;

    console.log('Gemma 4 エンジンを初期化中...');

    // アセットからモデルファイルのパスを取得
    const modelAsset = Asset.fromModule(require('../assets/gemma4-e2b.onnx'));
    await modelAsset.downloadAsync();

    // ONNX Runtime セッションを作成
    this.session = await InferenceSession.create(
      modelAsset.localUri!,
      {
        executionProviders: ['CoreML', 'Nnapi', 'CPU'],  // 優先順位: GPU → NPU → CPU
        graphOptimizationLevel: 'all',
        enableCpuMemArena: true
      }
    );

    console.log('✅ Gemma 4 エンジン初期化完了');
    this.isInitialized = true;
  }

  async generateText(
    prompt: string,
    systemPrompt: string = 'あなたは親切なAIアシスタントです。'
  ): Promise<string> {
    if (!this.session) {
      throw new Error('Gemma 4 エンジンが初期化されていません');
    }

    // プロンプトのトークン化
    // 注意: 実際の実装では Gemma 専用トークナイザーが必要です
    // ここでは概念的な実装を示しています
    const formattedPrompt = `<bos><start_of_turn>system\n${systemPrompt}<end_of_turn>\n<start_of_turn>user\n${prompt}<end_of_turn>\n<start_of_turn>model\n`;

    // 入力テンソルの準備（実際のトークン ID に変換が必要）
    const inputIds = new Tensor('int64', [/* トークン ID */], [1, formattedPrompt.length]);

    // 推論実行
    const startTime = Date.now();
    const outputs = await this.session.run({ input_ids: inputIds });
    const inferenceTime = Date.now() - startTime;

    console.log(`推論時間: ${inferenceTime}ms`);

    // 出力トークンをデコード
    const outputTokens = outputs.logits.data as Float32Array;
    // デコード処理（実際の実装ではトークナイザーのデコードが必要）
    return this.decodeTokens(outputTokens);
  }

  private decodeTokens(tokens: Float32Array): string {
    // 実際の実装では Gemma トークナイザーを使ってデコードします
    // このサンプルは概念的な実装です
    return "（デコードされたテキスト）";
  }

  async dispose(): Promise<void> {
    if (this.session) {
      await this.session.release();
      this.session = null;
      this.isInitialized = false;
    }
  }
}

// React Native コンポーネントでの使用例
import React, { useState, useEffect, useCallback } from 'react';
import { View, Text, TextInput, TouchableOpacity, StyleSheet, ActivityIndicator } from 'react-native';

const gemma4 = new Gemma4Engine();

export default function AIAssistantScreen() {
  const [isReady, setIsReady] = useState(false);
  const [isLoading, setIsLoading] = useState(false);
  const [userInput, setUserInput] = useState('');
  const [response, setResponse] = useState('');
  const [error, setError] = useState('');

  useEffect(() => {
    // アプリ起動時に Gemma 4 を初期化
    (async () => {
      try {
        await gemma4.initialize();
        setIsReady(true);
      } catch (err) {
        setError('AIエンジンの初期化に失敗しました');
        console.error(err);
      }
    })();

    return () => {
      gemma4.dispose();
    };
  }, []);

  const handleSubmit = useCallback(async () => {
    if (!userInput.trim() || isLoading) return;

    setIsLoading(true);
    setError('');

    try {
      const result = await gemma4.generateText(
        userInput,
        'あなたは役に立つアシスタントです。簡潔で正確な回答を日本語で提供してください。'
      );
      setResponse(result);
    } catch (err) {
      setError('テキスト生成中にエラーが発生しました');
      console.error(err);
    } finally {
      setIsLoading(false);
    }
  }, [userInput, isLoading]);

  if (!isReady) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#4285F4" />
        <Text style={styles.loadingText}>Gemma 4 を読み込み中...</Text>
        <Text style={styles.subText}>初回起動時は少し時間がかかります</Text>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <Text style={styles.badge}>🔒 オンデバイスAI（データ送信なし）</Text>

      <TextInput
        style={styles.input}
        value={userInput}
        onChangeText={setUserInput}
        placeholder="何でも聞いてください..."
        multiline
        placeholderTextColor="#999"
      />

      <TouchableOpacity
        style={[styles.button, isLoading && styles.buttonDisabled]}
        onPress={handleSubmit}
        disabled={isLoading}
      >
        {isLoading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>送信 →</Text>
        )}
      </TouchableOpacity>

      {error ? (
        <Text style={styles.error}>{error}</Text>
      ) : response ? (
        <View style={styles.responseContainer}>
          <Text style={styles.responseLabel}>Gemma 4 の回答（デバイス内処理）</Text>
          <Text style={styles.responseText}>{response}</Text>
        </View>
      ) : null}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 16, backgroundColor: '#f5f5f5' },
  center: { flex: 1, justifyContent: 'center', alignItems: 'center', gap: 12 },
```

ユーザーの感情や個人的な思いを記録・分析するアプリ。クラウドに送らないからこそユーザーが安心して本音を書ける。Gemma 4 が日記の内容を分析し、気分のトレンドを可視化したり、励ましのメッセージを生成したりします。

## クラウド AI との使い分け戦略

Gemma 4 をオンデバイスで使う場合と、クラウド AI（Gemini API、Claude API）を使う場合は適切に使い分けることが重要です。

**Gemma 4 オンデバイスを使うべき場面**:
個人情報・健康データ・財務データを含む処理、オフライン環境での動作が必要な場面、大量のリクエストが発生しコスト削減が重要な場面

**クラウド AI（Gemini API 等）を使うべき場面**:
最新情報の検索・参照が必要な場面、非常に長いコンテキストや複雑な推論が必要な場面、高品質な画像生成・音声合成が必要な場面

ベストプラクティスは**ハイブリッドアーキテクチャ**です。基本的な AI 処理は Gemma 4 でオンデバイス処理し、高度な処理が必要な場合のみクラウド AI にフォールバックすることで、コスト・プライバシー・パフォーマンスのバランスを最適化できます。

## よくあるエラーと対処法

**「モデルの読み込みに失敗する」**
モデルファイル（約 1.2〜2.5GB）のダウンロードが完了していない、またはストレージ容量不足が原因であることが多いです。ダウンロード進捗を表示する UI を実装し、ユーザーに状況を知らせましょう。

**「推論が遅すぎる（5秒以上かかる）」**
古いデバイスや RAM 不足が考えられます。E2B（2B）モデルへのダウングレードを検討するか、バックグラウンドスレッドで推論を実行してUIをブロックしないよう実装してください。

**「日本語の出力品質が低い」**
プロンプトに「日本語で回答してください」と明示するか、日本語データでファインチューニングされた Gemma 4 派生モデル（Hugging Face で入手可能）を使うことで改善できます。

## FAQ

**Q1: Rork のどのプランで Gemma 4 を使えますか？**

Gemma 4 の組み込みは React Native / Expo ベースの実装となるため、Rork の基本プランでも対応したコードを生成できます。ただし、iOS の Core ML 統合や Android の LiteRT ネイティブ実装には Rork Max が必要です。

**Q2: モデルのダウンロードはアプリの初回起動時に行われますか？**

はい、通常は初回起動時にモデルをデバイスにダウンロードします（E2B の場合 約 1.2GB）。ユーザー体験を考慮し、Wi-Fi 接続時のみダウンロードするオプションや、事前ダウンロードの進捗を表示する UI を実装することを強く推奨します。

**Q3: App Store の審査に通りますか？**

Gemma 4 は Apache 2.0 ライセンスのオープンモデルであり、アプリへの組み込みは Apple の開発者規約上問題ありません。ただし、AI 生成コンテンツに関する適切な表示（AI が生成したコンテンツである旨の明示）を行うことを推奨します。

**Q4: Google Play の審査で問題になることはありますか？**

Gemma は Google 自身が開発したモデルであるため、Google Play での公開において特別な審査上の問題は生じないはずです。通常の AI 機能アプリと同様の審査プロセスが適用されます。
