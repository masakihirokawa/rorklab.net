## AI生成UIの「あるある問題」— なぜ見た目がバラバラになるのか

Rorkでアプリを生成すると、UIは自動的に構築されますが、画面ごとにデザインの雰囲気が微妙に異なるという経験はないでしょうか。ホーム画面は落ち着いた配色なのに、設定画面は急にビビッドになる。ボタンの角丸が画面によって違う。影の強さが統一されていない。

これは「デザインシステムが不在の状態で、AIが画面ごとに最適解を探している」ために起きる現象です。人間のデザイナーが関わる場合、無意識のうちに一貫性を保ちますが、AIは明示的なルールがなければ画面単位で独立した判断をします。

坪田朋さんが開発した「Melta UI」というClaude Code向けデザインシステムの考え方は、この課題への実践的な解決策です。本記事では、その知見をRorkアプリ開発に応用し、**AIが生成するUIの品質を安定させるデザインシステム**の作り方を具体的に解説します。

## デザインシステムが解決する3つの問題

### 1. 色のブレ — AIは「安全な選択」に収束しがち

AIにUIを任せると、紫系のグラデーション、青系のプライマリカラー、白背景という「AIっぽい」配色に収束する傾向があります。これを防ぐには、アプリ固有のカラーパレットを事前に定義しておく必要があります。

### 2. タイポグラフィの不統一 — フォントサイズがバラバラに

見出しのフォントサイズ、本文のサイズ、行間が画面ごとに異なると、ユーザーは無意識に「雑なアプリだ」と感じます。タイポグラフィスケールを定義しておけば、AIは毎回同じサイズ体系を使います。

### 3. コンポーネントの一貫性 — ボタンのスタイルが3種類ある

同じ「確認」ボタンでも、画面によって角丸・影・パディングが違うと、アプリ全体の品質感が下がります。コンポーネントの仕様を定義することで、AIは毎回同じスタイルのボタンを生成します。

## Rorkアプリ向けデザイントークンの設計

### デザイントークンとは

デザイントークンは、色・タイポグラフィ・スペーシング・影などの視覚的な値を、変数として定義したものです。これをRorkのプロンプトに含めることで、AIの出力を制御します。

### 実践：デザイントークンを定義する

以下は、読書管理アプリを例にしたデザイントークンです。

```json
{
  "colors": {
    "primary": "#2563EB",
    "primaryLight": "#DBEAFE",
    "secondary": "#059669",
    "background": "#FAFAFA",
    "surface": "#FFFFFF",
    "text": {
      "primary": "#1F2937",
      "secondary": "#6B7280",
      "accent": "#2563EB"
    },
    "border": "#E5E7EB",
    "error": "#DC2626",
    "success": "#059669"
  },
  "typography": {
    "heading1": { "size": 28, "weight": "bold", "lineHeight": 1.3 },
    "heading2": { "size": 22, "weight": "semibold", "lineHeight": 1.35 },
    "heading3": { "size": 18, "weight": "semibold", "lineHeight": 1.4 },
    "body": { "size": 16, "weight": "regular", "lineHeight": 1.6 },
    "caption": { "size": 13, "weight": "regular", "lineHeight": 1.5 }
  },
  "spacing": {
    "xs": 4,
    "sm": 8,
    "md": 16,
    "lg": 24,
    "xl": 32,
    "xxl": 48
  },
  "borderRadius": {
    "sm": 6,
    "md": 12,
    "lg": 16,
    "full": 9999
  },
  "shadows": {
    "sm": "0 1px 2px rgba(0,0,0,0.05)",
    "md": "0 4px 6px rgba(0,0,0,0.07)",
    "lg": "0 10px 15px rgba(0,0,0,0.1)"
  }
}
```

## Rorkプロンプトにデザインシステムを組み込む方法

### 方法1: プロンプトに直接記述

最もシンプルな方法は、アプリ生成時のプロンプトにデザイントークンを含めることです。

```
読書管理アプリを作って。以下のデザインルールに従ってください:

【カラー】
- プライマリ: #2563EB（青）
- 背景: #FAFAFA
- テキスト: #1F2937
- ボーダー: #E5E7EB
- 影は控えめに（rgba(0,0,0,0.05)程度）

【タイポグラフィ】
- 見出し: 22px / semibold
- 本文: 16px / regular / 行間1.6
- キャプション: 13px / regular

【コンポーネント】
- ボタン: 角丸12px、パディング12px 24px
- カード: 角丸16px、影sm、ボーダーなし
- 入力欄: 角丸6px、ボーダー1px #E5E7EB
```

### 方法2: Rork Maxのカスタムテーマ設定

Rork Maxを使っている場合は、`theme.ts`ファイルでデザイントークンを定義し、アプリ全体に適用できます。

```typescript
// theme.ts — Rork Max用のカスタムテーマ定義
export const appTheme = {
  colors: {
    primary: "#2563EB",
    primaryLight: "#DBEAFE",
    secondary: "#059669",
    background: "#FAFAFA",
    surface: "#FFFFFF",
    textPrimary: "#1F2937",
    textSecondary: "#6B7280",
    border: "#E5E7EB",
    error: "#DC2626",
    success: "#059669",
  },
  typography: {
    h1: { fontSize: 28, fontWeight: "700" as const, lineHeight: 36 },
    h2: { fontSize: 22, fontWeight: "600" as const, lineHeight: 30 },
    h3: { fontSize: 18, fontWeight: "600" as const, lineHeight: 25 },
    body: { fontSize: 16, fontWeight: "400" as const, lineHeight: 26 },
    caption: { fontSize: 13, fontWeight: "400" as const, lineHeight: 20 },
  },
  spacing: (factor: number) => factor * 8,
  borderRadius: { sm: 6, md: 12, lg: 16, full: 9999 },
} as const;

// 使用例:
// <View style={{ padding: appTheme.spacing(2), borderRadius: appTheme.borderRadius.md }}>
//   <Text style={appTheme.typography.h2}>本のリスト</Text>
// </View>
```

### 方法3: デザインレビュープロンプトの活用

アプリ生成後に、デザインシステム準拠をチェックするプロンプトを実行する方法もあります。

```
生成したアプリのUIを以下の基準でレビューして:
1. カラーパレットが統一されているか（#2563EB, #FAFAFA, #1F2937以外の色が使われていないか）
2. フォントサイズが定義済みのスケール（13, 16, 18, 22, 28px）に収まっているか
3. ボタンの角丸が全て12pxで統一されているか
4. 影の強さが控えめ（opacity 0.05〜0.1）に収まっているか

違反があれば修正して。
```

## デザイントークンの「なぜ」— 選定の考え方

### 配色選定のポイント

デザイントークンを「なんとなく」決めると、結局AIが生成するのと同じ結果になります。以下の観点で意図的に選びましょう。

- **プライマリカラー**: アプリのジャンルに合わせる（金融→青、健康→緑、エンタメ→オレンジ）
- **背景色**: 純白（#FFFFFF）ではなくオフホワイト（#FAFAFA, #F9FAFB）にすると目に優しい
- **影の強さ**: AIは影を強くしすぎる傾向がある。`opacity: 0.05〜0.07`程度に制限する
- **アクセントカラー**: プライマリの補色を1色だけ用意し、重要なアクション（CTA）に限定使用

### タイポグラフィスケールの設計

フォントサイズの比率を決める際は、**1.25倍スケール**（Major Third）がモバイルアプリに適しています。

```
13px (caption) × 1.25 = 16px (body)
16px (body)    × 1.125 = 18px (h3)
18px (h3)      × 1.22  = 22px (h2)
22px (h2)      × 1.27  = 28px (h1)
```

## よくある質問（FAQ）

### Q: Rorkの無料プランでもデザインシステムを適用できますか？

はい。プロンプトにデザインルールを記述する方法（方法1）は、無料プランでも利用できます。Rork Maxの`theme.ts`を使う方法（方法2）はMax限定ですが、プロンプトベースでも十分な効果があります。

### Q: デザイントークンの数が多すぎるとAIの出力に影響しますか？

はい。トークンが多すぎるとAIが混乱し、逆に品質が下がることがあります。坪田さんのMelta UIの知見によると、基本的なデザイン要素（カラー・タイポグラフィ・スペーシング・角丸・影）を400トークン程度に収めるのが最適です。

### Q: 既にアプリを作った後からデザインシステムを適用できますか？

可能です。デザインレビュープロンプト（方法3）を使って既存のUIを監査し、違反箇所を修正してもらう方法が効率的です。ただし、大幅な変更が必要な場合は、デザインシステムを定義した上で新しくアプリを生成し直す方が早い場合もあります。

### Q: iOS / Android間でデザインの一貫性を保つには？

Rorkで生成するアプリはReact Native / Expoベースのため、基本的にiOSとAndroidで同じコードが動作します。プラットフォーム固有の調整（iOSのSafe Area、AndroidのMaterial Design等）が必要な場合は、プロンプトに「各プラットフォームのガイドラインに従って調整して」と追加してください。

## まとめ — デザインシステムは「AIへの設計図」

AI時代のアプリ開発では、デザインシステムの役割が変わりました。人間のデザイナーのための「共有言語」から、**AIへの設計指示書**へと進化しています。

デザイントークンを定義し、Rorkのプロンプトに組み込むだけで、AIが生成するUIの品質は劇的に安定します。配色のブレ、タイポグラフィの不統一、コンポーネントの不一致 — これらの「AIっぽさ」は、適切なデザインシステムで解消できる問題です。

まずはカラーパレットとタイポグラフィスケールの2つだけ定義してみてください。それだけで生成されるUIの印象が大きく変わるはずです。
