## はじめに — 「冷蔵庫を撮るだけ」で献立が決まるアプリ

「冷蔵庫を開けたけど、何を作ればいいかわからない」。誰もが日常的に経験するこの悩みを、AIの力で解決するアプリを Rork で作ってみましょう。

このチュートリアルでは、Gemini Vision API の画像認識機能を活用し、カメラで撮影した食材を自動で識別してレシピを提案するアプリを構築します。プログラミングの知識がなくても、Rork の AI アプリ生成機能を使えば、本格的な AI アプリを短時間で形にできます。

この記事で学べること:

- Expo Image Picker を使ったカメラ撮影・画像選択の実装方法
- Gemini Vision API で食材を認識する仕組みと API 連携
- 認識した食材からレシピを自動生成するプロンプト設計
- AsyncStorage を使ったお気に入りレシピの保存機能
- アプリ全体の画面遷移とUI設計のポイント

対象読者は、Rork で1〜2本のアプリを作った経験がある中級者を想定しています。カメラ機能や外部 API 連携が初めての方でも、ステップバイステップで進められる構成にしています。

## アプリの全体設計 — 3つの画面で構成するシンプルな UX

まず、アプリの全体像を整理します。ユーザー体験をシンプルに保つため、画面は大きく3つに分けます。

**ホーム画面（カメラ撮影）**: アプリを開くとすぐにカメラ撮影ボタンが表示されます。ユーザーは冷蔵庫の中身を撮影するか、既存の写真をギャラリーから選択します。撮影後、AI が食材を認識する処理が走ります。

**レシピ一覧画面**: 認識された食材のリストと、それらを使ったレシピ候補が3〜5件表示されます。各レシピには調理時間の目安、難易度、必要な追加食材が記載されています。

**レシピ詳細画面**: レシピをタップすると、材料リスト・手順・コツが表示されます。お気に入りボタンで保存でき、後から見返すことも可能です。

Rork でこの3画面構成を実現するプロンプトの例を見てみましょう:

```
AI食材認識レシピアプリを作ってください。

画面構成:
1. ホーム画面 - カメラ撮影ボタンとギャラリー選択ボタン、最近のスキャン履歴
2. レシピ一覧画面 - 認識した食材タグと、おすすめレシピカード一覧
3. レシピ詳細画面 - 材料・手順・調理時間・お気に入りボタン

デザイン: 温かみのあるオレンジ×クリーム配色、
角丸カード、食材タグはチップ形式で表示
```

このプロンプトを Rork に入力するだけで、基本的な画面構成とナビゲーションが自動生成されます。ここからカメラ機能と AI 連携を追加していきましょう。

## Expo Image Picker でカメラ撮影機能を実装する

食材認識の第一歩は、ユーザーが食材の写真を撮影できるようにすることです。Rork では Expo Image Picker がプリインストールされているため、追加インストールなしでカメラ機能を利用できます。

```typescript
// カメラ撮影とギャラリー選択の実装例
import * as ImagePicker from 'expo-image-picker';

// カメラで撮影する関数
const takePhoto = async () => {
  // カメラのパーミッションをリクエスト
  const { status } = await ImagePicker.requestCameraPermissionsAsync();
  if (status !== 'granted') {
    alert('カメラへのアクセスを許可してください');
    return;
  }

  // カメラを起動して撮影
  const result = await ImagePicker.launchCameraAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    quality: 0.8,        // 画質（0〜1）: 0.8で十分な品質とサイズのバランス
    base64: true,         // Base64エンコードを有効化（API送信用）
    allowsEditing: true,  // 撮影後にトリミング可能
    aspect: [4, 3],       // トリミングのアスペクト比
  });

  if (!result.canceled && result.assets[0]) {
    // 撮影成功 → 食材認識APIへ送信
    const base64Image = result.assets[0].base64;
    await recognizeIngredients(base64Image);
  }
};

// ギャラリーから選択する関数
const pickFromGallery = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    quality: 0.8,
    base64: true,
    allowsEditing: true,
  });

  if (!result.canceled && result.assets[0]) {
    const base64Image = result.assets[0].base64;
    await recognizeIngredients(base64Image);
  }
};
```

ポイントは `base64: true` を設定することです。これにより、撮影した画像が Base64 文字列として取得でき、そのまま Gemini Vision API に送信できます。`quality: 0.8` は画質とファイルサイズのバランスが最も良い設定で、API のレスポンス速度にも影響するため重要です。

なお、カメラ機能の基本的な実装パターンについては [Rork Max カメラ・ギャラリー機能ガイド](/articles/rork-dev/rork-max-camera-gallery) でも詳しく解説しています。

## Gemini Vision API で食材を自動認識する

撮影した画像から食材を識別するために、Google の Gemini Vision API を利用します。Gemini は画像の内容を高精度で理解し、テキストとして出力できるマルチモーダル AI です。

まず、API との通信を行うユーティリティ関数を作成します:

```typescript
// Gemini Vision API で食材を認識する関数
const GEMINI_API_URL = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';

interface RecognizedIngredient {
  name: string;       // 食材名
  confidence: number; // 認識の確信度（0〜1）
  category: string;   // 分類（野菜・肉・魚・乳製品・調味料など）
}

const recognizeIngredients = async (base64Image: string): Promise<RecognizedIngredient[]> => {
  try {
    const response = await fetch(`${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        contents: [{
          parts: [
            {
              text: `この画像に写っている食材を全て識別してください。
              以下のJSON形式で回答してください:
              [
                {
                  "name": "食材名（日本語）",
                  "confidence": 0.95,
                  "category": "野菜|肉|魚|乳製品|穀物|調味料|果物|その他"
                }
              ]
              注意:
              - 食材以外のもの（皿、棚、ラップなど）は含めない
              - 確信度が0.5未満の食材は含めない
              - JSON配列のみを返し、他のテキストは含めない`
            },
            {
              inlineData: {
                mimeType: 'image/jpeg',
                data: base64Image,
              },
            },
          ],
        }],
        generationConfig: {
          temperature: 0.1,  // 低めの温度で安定した認識結果を得る
          maxOutputTokens: 1024,
        },
      }),
    });

    const data = await response.json();
    const text = data.candidates[0].content.parts[0].text;

    // JSONをパースして返す
    const jsonMatch = text.match(/\[[\s\S]*\]/);
    if (jsonMatch) {
      return JSON.parse(jsonMatch[0]);
    }
    return [];
  } catch (error) {
    console.error('食材認識エラー:', error);
    return [];
  }
};
// 期待する出力例:
// [
//   { "name": "トマト", "confidence": 0.97, "category": "野菜" },
//   { "name": "鶏むね肉", "confidence": 0.92, "category": "肉" },
//   { "name": "玉ねぎ", "confidence": 0.89, "category": "野菜" },
//   { "name": "にんにく", "confidence": 0.85, "category": "野菜" }
// ]
```

このコードの重要なポイントをいくつか説明します。`temperature: 0.1` に設定しているのは、食材認識では創造性よりも正確性が重要だからです。プロンプトで「JSON配列のみを返す」と明示することで、パースしやすい出力を得られます。

Gemini API の基本的な連携方法については [Rork × Gemini 2.0 Flash 完全ガイド](/articles/rork-ai/rork-gemini-integration) でも詳しく解説していますので、API キーの取得方法や基本設定がわからない方はそちらも参考にしてください。

## 認識した食材からレシピを自動生成する

食材の認識ができたら、次はそれらの食材を使ったレシピを AI に生成させます。ここでも Gemini API を活用しますが、今度はテキスト生成モデルとして使います。

```typescript
// 食材リストからレシピを生成する関数
interface Recipe {
  id: string;
  title: string;
  cookingTime: number;   // 分
  difficulty: 'easy' | 'medium' | 'hard';
  servings: number;
  ingredients: { name: string; amount: string; isAvailable: boolean }[];
  steps: string[];
  tips: string;
}

const generateRecipes = async (
  ingredients: RecognizedIngredient[]
): Promise<Recipe[]> => {
  const ingredientNames = ingredients.map(i => i.name).join('、');

  const response = await fetch(`${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{
        parts: [{
          text: `以下の食材を使ったレシピを3〜5個提案してください。

手持ちの食材: ${ingredientNames}

以下のJSON形式で回答してください:
[
  {
    "id": "ユニークID",
    "title": "レシピ名",
    "cookingTime": 20,
    "difficulty": "easy",
    "servings": 2,
    "ingredients": [
      { "name": "食材名", "amount": "分量", "isAvailable": true }
    ],
    "steps": ["手順1", "手順2", "手順3"],
    "tips": "調理のコツやアレンジ方法"
  }
]

ルール:
- 手持ちの食材をメインに使うレシピを提案する
- 追加で必要な食材は isAvailable: false にする
- 追加食材は一般的な調味料（塩、醤油、油など）に限定する
- 難易度は easy（15分以内）、medium（30分以内）、hard（30分超）
- 家庭で作りやすい日本の家庭料理を優先する
- JSON配列のみを返す`
        }],
      }],
      generationConfig: {
        temperature: 0.7,  // レシピ提案では適度な多様性を持たせる
        maxOutputTokens: 4096,
      },
    }),
  });

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;
  const jsonMatch = text.match(/\[[\s\S]*\]/);

  if (jsonMatch) {
    return JSON.parse(jsonMatch[0]);
  }
  return [];
};
// 期待する出力例（トマト・鶏むね肉・玉ねぎ・にんにくの場合）:
// [
//   {
//     "id": "chicken-tomato-1",
//     "title": "チキンのトマト煮込み",
//     "cookingTime": 25,
//     "difficulty": "easy",
//     "servings": 2,
//     "ingredients": [
//       { "name": "鶏むね肉", "amount": "200g", "isAvailable": true },
//       { "name": "トマト", "amount": "2個", "isAvailable": true },
//       ...
//     ],
//     "steps": ["鶏むね肉を一口大に切る", "玉ねぎとにんにくをみじん切りにする", ...],
//     "tips": "鶏肉に片栗粉をまぶすとジューシーに仕上がります"
//   },
//   ...
// ]
```

食材認識の `temperature` は `0.1` に設定しましたが、レシピ生成では `0.7` にしている点に注目してください。レシピ提案では毎回同じ結果ではなく、適度なバリエーションがあったほうがユーザー体験として優れています。

## お気に入りレシピの保存機能を実装する

気に入ったレシピを後から見返せるよう、AsyncStorage を使った保存機能を追加します。Rork アプリでは AsyncStorage がデフォルトで利用可能です。

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

const FAVORITES_KEY = '@recipe_favorites';

// お気に入りに保存する
const saveFavorite = async (recipe: Recipe) => {
  try {
    const existing = await AsyncStorage.getItem(FAVORITES_KEY);
    const favorites: Recipe[] = existing ? JSON.parse(existing) : [];

    // 重複チェック
    if (favorites.some(f => f.id === recipe.id)) {
      return; // 既に保存済み
    }

    favorites.unshift(recipe); // 先頭に追加
    await AsyncStorage.setItem(FAVORITES_KEY, JSON.stringify(favorites));
  } catch (error) {
    console.error('保存エラー:', error);
  }
};

// お気に入り一覧を取得する
const getFavorites = async (): Promise<Recipe[]> => {
  try {
    const data = await AsyncStorage.getItem(FAVORITES_KEY);
    return data ? JSON.parse(data) : [];
  } catch (error) {
    console.error('取得エラー:', error);
    return [];
  }
};

// お気に入りから削除する
const removeFavorite = async (recipeId: string) => {
  try {
    const existing = await AsyncStorage.getItem(FAVORITES_KEY);
    const favorites: Recipe[] = existing ? JSON.parse(existing) : [];
    const updated = favorites.filter(f => f.id !== recipeId);
    await AsyncStorage.setItem(FAVORITES_KEY, JSON.stringify(updated));
  } catch (error) {
    console.error('削除エラー:', error);
  }
};
```

このシンプルな実装で十分実用的ですが、保存するレシピの数が100件を超えるような場合は、MMKV や SQLite への移行を検討してください。AsyncStorage は小〜中規模のデータ保存に最適化されています。

レシピアプリの印象を大きく左右するのが、レシピカードのデザインです。Rork で美しいカードコンポーネントを生成するためのプロンプトテクニックを紹介します。

カードに含める要素:
- レシピ名（太字、最大2行）
- 調理時間アイコン + "20分"
- 難易度バッジ（簡単=緑、普通=黄、難しい=赤）
- 人数表示
- 手持ちの食材カバー率（"食材80%カバー"のように表示）

デザイン仕様:

- カード幅は画面幅の90%

食材カバー率の計算は以下のように実装します:

```typescript
// 食材カバー率を計算するユーティリティ
const calculateCoverage = (recipe: Recipe): number => {
  const total = recipe.ingredients.length;
  const available = recipe.ingredients.filter(i => i.isAvailable).length;
  return Math.round((available / total) * 100);
};

// 表示例: "食材 80% カバー" → 追加で買う食材が少ないレシピほど高スコア
```

食材カバー率が高い順にレシピを並べることで、「今ある食材だけで作れるレシピ」を優先的に表示でき、ユーザーにとって実用的な体験を提供できます。

## エラーハンドリングとオフライン対応

AI アプリで見落としがちなのがエラーハンドリングです。ネットワーク接続が不安定な場合や、API がタイムアウトした場合にも、ユーザーが困らないようにしましょう。

```typescript
import NetInfo from '@react-native-community/netinfo';

// ネットワーク状態を確認してから API を呼び出す
const safeRecognize = async (base64Image: string) => {
  // ネットワーク接続確認
  const netState = await NetInfo.fetch();
  if (!netState.isConnected) {
    alert('インターネット接続を確認してください。食材認識にはネット接続が必要です。');
    return null;
  }

  // タイムアウト付きの API 呼び出し
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), 15000); // 15秒でタイムアウト

  try {
    const result = await recognizeIngredients(base64Image);
    clearTimeout(timeoutId);

    if (result.length === 0) {
      // 食材が認識できなかった場合
      alert('食材を認識できませんでした。明るい場所で食材が見えるように撮影してください。');
      return null;
    }

    return result;
  } catch (error) {
    clearTimeout(timeoutId);
    if (error instanceof Error && error.name === 'AbortError') {
      alert('通信がタイムアウトしました。もう一度お試しください。');
    } else {
      alert('エラーが発生しました。しばらく待ってから再度お試しください。');
    }
    return null;
  }
};
```

また、過去の認識結果をキャッシュしておくことで、同じ食材の組み合わせに対しては API を呼ばずにレシピを表示できます。これにより、API コスト（Gemini API は従量課金）の削減にもつながります。

さらに高度な AI API のコスト管理手法については [Rork × AI API コスト最適化ガイド](/articles/rork-dev/rork-ai-api-cost-optimization-mobile-guide) で詳しく解説しています。

## よくある質問（FAQ）

### Q1: Gemini API の利用料金はどのくらいかかりますか？

Gemini 2.5 Flash は無料枠が用意されており、1日あたり一定回数のリクエストが無料で利用できます。個人開発レベル（1日数十回程度の利用）であれば、無料枠内で収まることがほとんどです。画像認識（Vision）のリクエストはテキストのみの場合より若干コストが高くなりますが、1リクエストあたり数円程度です。本番環境で多くのユーザーに提供する場合は、Google AI Studio でAPI使用量を監視し、必要に応じて予算アラートを設定しておきましょう。

### Q2: 食材の認識精度はどのくらいですか？

Gemini Vision API の食材認識精度は、撮影条件によって大きく変わります。明るい場所で食材が整理された状態であれば90%以上の精度が期待できます。一方、暗い場所や食材が重なっている場合は精度が落ちることがあります。精度を向上させるコツとして、プロンプトに「日本の一般的な食材を優先して認識する」と追加したり、認識結果をユーザーが手動で修正できる UI を用意しておくと実用性が大幅に向上します。

### Q3: カメラ機能は iOS と Android の両方で動作しますか？

はい、Expo Image Picker は iOS と Android の両プラットフォームに対応しています。Rork で生成したアプリはクロスプラットフォーム対応なので、追加の設定なしで両 OS で動作します。ただし、カメラのパーミッション（権限）リクエストの挙動は OS ごとに若干異なります。iOS では `Info.plist` に使用目的の説明が必要ですが、Expo が自動で設定してくれるため、通常は意識する必要はありません。

### Q4: 認識結果をユーザーが編集できるようにすべきですか？

強くおすすめします。AI の認識結果は100%正確ではないため、ユーザーが食材の追加・削除・修正ができる UI を用意しておくと、より実用的なアプリになります。具体的には、認識された食材をタグ形式で表示し、タップで削除、テキスト入力で手動追加できるインターフェースが効果的です。

## まとめ — AI × カメラで「日常の困りごと」を解決するアプリを作ろう

この記事では、Rork と Gemini Vision API を組み合わせて、カメラで食材を撮影するだけでレシピを提案する AI アプリの作り方を解説しました。

ポイントを振り返ると、Expo Image Picker でカメラ撮影と Base64 エンコードを処理し、Gemini Vision API の構造化出力プロンプトで食材を高精度に認識し、認識結果からレシピを自動生成して AsyncStorage でお気に入り保存する、という流れでした。

AI × カメラの組み合わせは、食材認識以外にも植物の種類判定、ファッションコーディネート提案、部屋のインテリア診断など、さまざまなアプリに応用できます。まずはこの記事のレシピアプリを完成させて、そこから自分だけのアイデアに発展させてみてください。
