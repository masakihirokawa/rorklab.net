## はじめに — Rork AIが変えるアプリ開発の常識

「アプリを作りたい」という思いを持ちながら、開発のハードルの高さに諦めてしまった方は少なくないでしょう。Swift や Kotlin を習得する時間も、デザインのスキルも、バックエンドの知識も——従来のアプリ開発には多くの専門知識が必要でした。

Rork AI はその常識を変えています。自然言語で「こういうアプリを作りたい」と伝えるだけで、実際に動くコードを生成し、デザインを整え、機能を追加していくことができます。本記事では、Rork AI を使って本格的なアプリを0から作る全工程を、実例を交えながらお伝えします。

今回作成するのは **「習慣トラッカーアプリ」** です。毎日の習慣を記録し、継続率を可視化する、シンプルだけど実用的なアプリです。

### Rork AIとの最初の会話

アプリ開発を始める前に、作るものを明確にすることが大切です。Rork AI との最初のやりとりでは、以下のような形でアプリのビジョンを伝えるのが効果的です。

```
「毎日の習慣を管理するアプリを作りたい。

- 習慣の追加・削除
- 毎日のチェックイン
- 継続日数のカウント
- 週間・月間の達成率グラフ

対象ユーザー: 自己成長に関心がある20〜30代
デザイン方針: シンプルでモチベーションが上がるUI
```

この初期プロンプトに対して、Rork AI は基本的なアプリ構造の提案と初期コードを生成してくれます。

### 要件を具体化するイテレーション

Rork AI の提案を見ながら、さらに要件を絞り込んでいきます。

「習慣カードのデザインをもっとカラフルにしてほしい。各習慣にカテゴリカラーを設定できるようにして」といった具体的なフィードバックを繰り返すことで、アプリが形になっていきます。

## フェーズ3: データ管理の実装

### ローカルデータストレージ

習慣データのローカル保存には、AsyncStorage または Expo SQLite を使います。Rork AI に以下のように依頼すると、適切な実装を提案してくれます。

```
「習慣データを AsyncStorage で管理する hooks を実装してください。
- useHabits: 習慣のCRUD操作
- useCompletions: 今日のチェックイン管理
- useStreaks: 継続日数の計算
TypeScript の型定義も含めてください」
```

Rork AI が生成する hooks の例:

```typescript
import { useState, useEffect, useCallback } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { Habit, Completion } from '../types';

const HABITS_KEY = '@habits';
const COMPLETIONS_KEY = '@completions';

export const useHabits = () => {
  const [habits, setHabits] = useState<Habit[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    loadHabits();
  }, []);

  const loadHabits = async () => {
    try {
      const stored = await AsyncStorage.getItem(HABITS_KEY);
      if (stored) {
        setHabits(JSON.parse(stored));
      }
    } catch (error) {
      console.error('習慣の読み込みに失敗:', error);
    } finally {
      setLoading(false);
    }
  };

  const addHabit = useCallback(async (habit: Omit<Habit, 'id' | 'createdAt'>) => {
    const newHabit: Habit = {
      ...habit,
      id: Date.now().toString(),
      createdAt: new Date().toISOString(),
    };
    const updated = [...habits, newHabit];
    setHabits(updated);
    await AsyncStorage.setItem(HABITS_KEY, JSON.stringify(updated));
    return newHabit;
  }, [habits]);

  const deleteHabit = useCallback(async (habitId: string) => {
    const updated = habits.filter(h => h.id !== habitId);
    setHabits(updated);
    await AsyncStorage.setItem(HABITS_KEY, JSON.stringify(updated));
  }, [habits]);

  return { habits, loading, addHabit, deleteHabit };
};
```

### 継続日数（ストリーク）の計算ロジック

習慣アプリの核心的な機能であるストリーク計算を、Rork AI に実装させましょう。

```typescript
export const calculateStreak = (completions: string[]): number => {
  if (completions.length === 0) return 0;

  const today = new Date();
  today.setHours(0, 0, 0, 0);
  
  let streak = 0;
  let checkDate = new Date(today);
  
  // ソートして最新日順に並べる
  const sortedCompletions = [...completions]
    .map(d => new Date(d))
    .sort((a, b) => b.getTime() - a.getTime());
  
  for (const date of sortedCompletions) {
    const completionDay = new Date(date);
    completionDay.setHours(0, 0, 0, 0);
    
    const diffDays = Math.round(
      (checkDate.getTime() - completionDay.getTime()) / (1000 * 60 * 60 * 24)
    );
    
    if (diffDays === 0 || diffDays === 1) {
      streak++;
      checkDate = completionDay;
    } else {
      break;
    }
  }
  
  return streak;
};
```

## Rork AIで開発する際の実践的なヒント

### ヒント1: 一度に大きな変更を頼まない

Rork AI に指示する際は、一度に多くの変更を依頼するよりも、小さなステップに分けて進める方が品質が安定します。「全画面のデザインを変えて」よりも「ホーム画面のカードデザインをこう変えて」の方が、意図した結果を得やすいです。

### ヒント2: 生成されたコードを理解する

Rork AI が生成したコードを盲目的に使わず、何をしているかを理解しながら進めましょう。コードへの理解が深まるほど、より良い指示を出せるようになり、開発の質が向上します。

### ヒント3: バージョン管理を忘れずに

Rork AI との開発でも、Git によるバージョン管理は必須です。大きな変更を加える前にコミットしておくことで、うまくいかない変更を安全に戻すことができます。
