## はじめに — AI が旅行計画を変える

「どこに行こう？」「何泊がベスト？」「予算内で楽しめるプランは？」——旅行計画の悩みは尽きません。しかし、AI の力を借りれば、目的地・予算・日数を入力するだけで最適な旅行プランを瞬時に生成できます。

この記事では、Rork と Google の Gemini API を組み合わせて、**AI 旅行プランナーアプリ**を一から構築する方法を解説します。Rork の直感的な UI 生成機能と Gemini の高度な自然言語処理を活用することで、実用的なアプリを短時間で完成させることができます。

**この記事で学べること:**

- Rork プロジェクトから Gemini API を呼び出す方法
- AI に適切な旅行プランを生成させるプロンプト設計
- ユーザー入力を受け取り、結果を見やすく表示する UI の実装
- エラーハンドリングとローディング状態の管理

**対象読者:** Rork で基本的なアプリを作った経験があり、外部 API との連携に挑戦したい方

## 前提知識と環境準備

このチュートリアルを進めるには、以下の準備が必要です。

**必要なもの:**

- Rork アカウント（Free プランでも可）
- Google AI Studio のアカウントと Gemini API キー
- 基本的な JavaScript / React Native の知識

まだ Rork を触ったことがない方は、[はじめてのアプリを30分で作るガイド](/articles/rork-basics/rork-first-app-30-minutes-guide)を先にお読みください。

### Gemini API キーの取得

Google AI Studio（[aistudio.google.com](https://aistudio.google.com)）にアクセスし、API キーを発行します。無料枠でも十分な回数の API コールが可能なので、開発段階ではコストを気にする必要はありません。

```bash
# API キーの動作確認（ターミナルで実行）
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"contents":[{"parts":[{"text":"Hello"}]}]}'
# 期待する出力: JSON レスポンスに "candidates" 配列が含まれていれば成功
```

## アプリの全体設計

今回作成するアプリの構成は以下のとおりです。

**画面構成:**

1. **入力画面（HomeScreen）** — 目的地、旅行日数、予算、旅のスタイルを入力
2. **結果画面（PlanScreen）** — AI が生成した旅行プランを日程ごとに表示
3. **詳細画面（DetailScreen）** — 各スポットの詳細情報とマップリンク

**データフロー:**

```
ユーザー入力 → プロンプト組み立て → Gemini API 呼び出し → JSON パース → UI 表示
```

## Rork プロジェクトの作成とプロンプト設計

### プロジェクトの初期化

Rork のダッシュボードで新規プロジェクトを作成します。プロンプトには以下のように入力しましょう。

```
旅行プランナーアプリを作ってください。

画面構成:
1. ホーム画面: 目的地（テキスト入力）、旅行日数（1〜14日のピッカー）、
   予算（低・中・高の3段階）、旅のスタイル（観光・グルメ・アクティビティ・のんびり）
   を選択できるフォーム。「プランを作成」ボタン。
2. プラン結果画面: 日程ごとにカードで表示。各日のタイトル、
   訪問スポット名、推定費用、一言メモを含む。
3. スポット詳細画面: スポット名、説明文、おすすめ時間帯、
   Google Maps へのリンクボタン。

デザイン: 明るくモダンな旅行アプリらしい配色。
アイコンは飛行機とマップピンを使用。
```

Rork がベースの UI を自動生成してくれます。ここから Gemini API との連携部分を追加していきます。

### Gemini API 呼び出しの実装

API 連携のコアとなる関数を作成します。ポイントは、Gemini に**構造化された JSON で回答させる**ことです。

```javascript
// services/geminiService.js
const GEMINI_API_KEY = 'YOUR_API_KEY'; // 本番では環境変数を使用
const API_URL = `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=${GEMINI_API_KEY}`;

export async function generateTravelPlan({ destination, days, budget, style }) {
  const budgetMap = { low: '節約重視（1日5,000円以下）', medium: '標準（1日10,000〜20,000円）', high: 'リッチ（金額を気にしない）' };
  const styleMap = { sightseeing: '観光名所めぐり', food: 'グルメ・食べ歩き', activity: 'アクティビティ・体験', relaxing: 'のんびり・リラックス' };

  const prompt = `
あなたは旅行プランナーです。以下の条件で旅行プランを JSON 形式で作成してください。

条件:
- 目的地: ${destination}
- 日数: ${days}日間
- 予算: ${budgetMap[budget]}
- スタイル: ${styleMap[style]}

以下の JSON 形式で回答してください（JSON のみ、説明文不要）:
{
  "title": "プランのタイトル",
  "summary": "プラン全体の概要（2文程度）",
  "days": [
    {
      "day": 1,
      "theme": "その日のテーマ",
      "spots": [
        {
          "name": "スポット名",
          "description": "説明（50文字程度）",
          "duration": "滞在目安（例: 2時間）",
          "cost": "推定費用（例: 1,500円）",
          "tip": "ワンポイントアドバイス"
        }
      ]
    }
  ],
  "totalBudget": "合計推定費用",
  "packingTips": ["持ち物のアドバイス1", "持ち物のアドバイス2"]
}`;

  const response = await fetch(API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: {
        temperature: 0.8,       // 創造性と正確性のバランス
        maxOutputTokens: 4096,  // 長い旅行プランにも対応
        responseMimeType: 'application/json', // JSON モードを強制
      },
    }),
  });

  const data = await response.json();
  // Gemini のレスポンスからテキスト部分を抽出
  const text = data.candidates[0].content.parts[0].text;
  return JSON.parse(text);
}
// 期待する出力: 構造化された旅行プラン JSON オブジェクト
```

### プロンプト設計のポイント

AI に良質な旅行プランを生成させるために、プロンプト設計で意識すべき点があります。

**1. 出力形式を明示する** — JSON スキーマをプロンプト内に直接記述することで、パースしやすい構造化データが得られます。`responseMimeType: 'application/json'` を指定すれば、Gemini が JSON 以外を返すことを防げます。

**2. 具体的な制約を与える** — 「予算」を抽象的な「低・中・高」ではなく具体的な金額レンジに変換してからプロンプトに渡すと、より実用的な結果になります。

**3. temperature を調整する** — `0.8` は創造性と正確性のバランスが良い値です。毎回同じ結果にならず、かつ突飛すぎないプランが生成されます。

## UI の実装 — 入力フォームと結果表示

### 入力画面のコンポーネント

ユーザーが旅行条件を入力するフォームを実装します。

```javascript
// screens/HomeScreen.jsx
import React, { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, ScrollView, ActivityIndicator } from 'react-native';
import { Picker } from '@react-native-picker/picker';
import { generateTravelPlan } from '../services/geminiService';

export default function HomeScreen({ navigation }) {
  const [destination, setDestination] = useState('');
  const [days, setDays] = useState(3);
  const [budget, setBudget] = useState('medium');
  const [style, setStyle] = useState('sightseeing');
  const [loading, setLoading] = useState(false);

  const handleGenerate = async () => {
    if (!destination.trim()) {
      alert('目的地を入力してください');
      return;
    }
    setLoading(true);
    try {
      const plan = await generateTravelPlan({ destination, days, budget, style });
      navigation.navigate('Plan', { plan });
    } catch (error) {
      alert('プランの生成に失敗しました。もう一度お試しください。');
      console.error('Gemini API error:', error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <ScrollView style={{ flex: 1, padding: 20, backgroundColor: '#F8FAFB' }}>
      <Text style={{ fontSize: 28, fontWeight: 'bold', marginBottom: 8 }}>
        AI 旅行プランナー ✈️
      </Text>
      <Text style={{ fontSize: 16, color: '#666', marginBottom: 24 }}>
        条件を入力するだけで、あなただけの旅行プランを AI が作成します
      </Text>

      <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 6 }}>目的地</Text>
      <TextInput
        value={destination}
        onChangeText={setDestination}
        placeholder="例: 京都、バンコク、バルセロナ"
        style={{ borderWidth: 1, borderColor: '#DDD', borderRadius: 12, padding: 14, marginBottom: 16, fontSize: 16, backgroundColor: '#FFF' }}
      />

      <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 6 }}>旅行日数</Text>
      <Picker selectedValue={days} onValueChange={setDays} style={{ marginBottom: 16 }}>
        {[...Array(14)].map((_, i) => (
          <Picker.Item key={i + 1} label={`${i + 1}日間`} value={i + 1} />
        ))}
      </Picker>

      <TouchableOpacity
        onPress={handleGenerate}
        disabled={loading}
        style={{ backgroundColor: loading ? '#AAA' : '#4A90D9', borderRadius: 12, padding: 16, alignItems: 'center', marginTop: 16 }}
      >
        {loading ? (
          <ActivityIndicator color="#FFF" />
        ) : (
          <Text style={{ color: '#FFF', fontSize: 18, fontWeight: 'bold' }}>プランを作成</Text>
        )}
      </TouchableOpacity>
    </ScrollView>
  );
}
// 期待する動作: フォーム入力後「プランを作成」タップで Gemini API を呼び出し、結果画面に遷移
```

### 結果画面 — 日程カードの表示

Gemini が返した JSON を日程ごとのカードとして表示します。

```javascript
// screens/PlanScreen.jsx
import React from 'react';
import { View, Text, ScrollView, TouchableOpacity } from 'react-native';

export default function PlanScreen({ route, navigation }) {
  const { plan } = route.params;

  return (
    <ScrollView style={{ flex: 1, backgroundColor: '#F8FAFB', padding: 16 }}>
      <Text style={{ fontSize: 24, fontWeight: 'bold', marginBottom: 4 }}>{plan.title}</Text>
      <Text style={{ fontSize: 14, color: '#666', marginBottom: 20 }}>{plan.summary}</Text>

      {plan.days.map((day) => (
        <View key={day.day} style={{ backgroundColor: '#FFF', borderRadius: 16, padding: 16, marginBottom: 12, shadowColor: '#000', shadowOpacity: 0.05, shadowRadius: 8 }}>
          <Text style={{ fontSize: 18, fontWeight: '700', color: '#4A90D9', marginBottom: 8 }}>
            Day {day.day}: {day.theme}
          </Text>
          {day.spots.map((spot, idx) => (
            <TouchableOpacity
              key={idx}
              onPress={() => navigation.navigate('Detail', { spot })}
              style={{ paddingVertical: 8, borderBottomWidth: idx < day.spots.length - 1 ? 1 : 0, borderBottomColor: '#EEE' }}
            >
              <Text style={{ fontSize: 16, fontWeight: '600' }}>{spot.name}</Text>
              <Text style={{ fontSize: 13, color: '#888', marginTop: 2 }}>
                {spot.duration} · {spot.cost}
              </Text>
            </TouchableOpacity>
          ))}
        </View>
      ))}

      <View style={{ backgroundColor: '#E8F4FD', borderRadius: 12, padding: 16, marginTop: 8, marginBottom: 32 }}>
        <Text style={{ fontSize: 16, fontWeight: '700', marginBottom: 8 }}>合計推定費用: {plan.totalBudget}</Text>
        <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 4 }}>持ち物アドバイス:</Text>
        {plan.packingTips.map((tip, i) => (
          <Text key={i} style={{ fontSize: 14, color: '#555' }}>• {tip}</Text>
        ))}
      </View>
    </ScrollView>
  );
}
// 期待する動作: 日程ごとのカードが縦に並び、各スポットをタップすると詳細画面へ遷移
```

## エラーハンドリングとUXの改善

実際のアプリでは、ネットワークエラーや API レート制限への対応が不可欠です。

### リトライ機能の実装

```javascript
// utils/retry.js
export async function withRetry(fn, maxRetries = 3, delay = 1000) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;
      // 指数バックオフ: 1秒 → 2秒 → 4秒
      await new Promise(resolve => setTimeout(resolve, delay * Math.pow(2, attempt - 1)));
    }
  }
}

// 使用例
const plan = await withRetry(() =>
  generateTravelPlan({ destination, days, budget, style })
);
// 期待する動作: API エラー時に最大3回リトライ。すべて失敗した場合のみエラーをスロー
```

### API キーの安全な管理

開発中はハードコードでも構いませんが、本番環境では必ず環境変数や Supabase Edge Functions などのサーバーサイドで API キーを管理してください。クライアントアプリに API キーを埋め込むと、リバースエンジニアリングで漏洩するリスクがあります。

```javascript
// Supabase Edge Function で API キーを隠蔽する例
// supabase/functions/generate-plan/index.ts
const GEMINI_KEY = Deno.env.get('GEMINI_API_KEY');

Deno.serve(async (req) => {
  const { destination, days, budget, style } = await req.json();
  // ここで Gemini API を呼び出し、結果を返す
  // クライアントは Supabase Function URL を叩くだけ
});
// 期待する動作: API キーがサーバー側のみに保持され、クライアントには露出しない
```

## よくある質問（FAQ）

### Q1: Gemini API の無料枠はどのくらいですか？

Gemini 2.0 Flash の無料枠は1分あたり15リクエスト、1日あたり1,500リクエストです（2026年3月時点）。個人開発のアプリであれば十分な量です。本格的にユーザー数が増えた場合は、従量課金プランへの移行を検討しましょう。

### Q2: Gemini 以外の AI モデルも使えますか？

はい。`geminiService.js` の API エンドポイントとリクエスト形式を変更するだけで、OpenAI の GPT や Anthropic の Claude API に切り替えられます。プロンプトの構造はほぼそのまま流用可能です。Rork と各 AI モデルとの連携については、[Rork × Gemini 連携ガイド](/articles/rork-ai/rork-gemini-integration)も参考になります。

### Q3: 生成されたプランの精度を上げるには？

プロンプトに「地元の人がおすすめする穴場スポットも含めて」「移動時間も考慮して」といった制約を追加すると、より実用的なプランになります。また、`temperature` を `0.6` 程度に下げると、より定番スポット中心の安定したプランが生成されます。

### Q4: オフラインでも使えるようにするには？

一度生成したプランを AsyncStorage に保存すれば、オフラインでも閲覧できます。生成自体には API 通信が必要ですが、保存済みプランの表示はネットワーク不要です。

### Q5: 地図表示を追加するには？

各スポットの座標を Gemini のプロンプトに含めるよう指示すれば、`react-native-maps` で地図上にピンを表示できます。地図連携の詳しい実装は[MapView とスポット共有アプリのチュートリアル](/articles/rork-dev/rork-mapview-supabase-spot-sharing-app)をご覧ください。
