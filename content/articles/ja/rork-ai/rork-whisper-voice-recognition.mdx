## OpenAI Whisper API とは？

**Whisper** は OpenAI が開発した音声認識（Speech-to-Text）モデルです。2022年に公開され、2025年以降は `whisper-1` モデルが API として広く利用されています。

| 特徴 | 内容 |
|------|------|
| 対応言語 | 99言語以上（日本語含む） |
| 精度 | 人間レベルに近い高精度 |
| 価格 | $0.006 / 分（2026年3月時点） |
| 最大ファイルサイズ | 25MB |
| 対応フォーマット | mp3, mp4, mpeg, mpga, m4a, wav, webm |

**なぜ Whisper を選ぶのか？** Apple の Speech Framework は iOS 専用ですが、Whisper API はクロスプラットフォームで動作し、iOS・Android 両対応の Rork アプリに最適です。

## 概念解説：音声認識の仕組み

Rork アプリで音声認識を実現する流れは以下のとおりです：

```
[ユーザー] → マイクに話す
     ↓
[Expo Audio] → 音声を録音し .m4a ファイルとして保存
     ↓
[FormData] → ファイルを Whisper API に送信
     ↓
[OpenAI Whisper API] → 音声を文字起こし
     ↓
[アプリ] → テキストを画面に表示
```

ポイントは、**Expo Audio**（React Native / Expo の音声録音ライブラリ）と **Whisper API** の橋渡しをするコードを Rork に生成してもらうことです。

## ステップバイステップ チュートリアル

### Step 1：Rork でプロジェクトを作成

Rork を開き、以下のプロンプトを入力します：

```
音声録音と文字起こし機能を持つアプリを作ってください。

機能要件：
- 録音開始・停止ボタン
- 録音中は赤いインジケーターを表示
- 録音データを OpenAI Whisper API に送信して文字起こし
- 文字起こし結果をスクロール可能なテキストエリアに表示
- コピーボタンで結果をクリップボードにコピー
- 複数の文字起こし履歴を保持（最新10件）
- ダークモード対応

技術スタック：
- Expo Audio（expo-av）で録音
- axios または fetch で Whisper API（https://api.openai.com/v1/audio/transcriptions）に POST
- API キーは環境変数 EXPO_PUBLIC_OPENAI_API_KEY から読み取る
```

Rork が UI とコードの骨格を自動生成します。

### Step 2：Rork が生成するコアコード

Rork が生成する音声録音 + Whisper 送信のコードは以下のような構造になります：

```typescript
// hooks/useVoiceRecorder.ts
import { useState, useRef } from 'react';
import { Audio } from 'expo-av';
import * as FileSystem from 'expo-file-system';

const WHISPER_API_URL = 'https://api.openai.com/v1/audio/transcriptions';
const OPENAI_API_KEY = process.env.EXPO_PUBLIC_OPENAI_API_KEY ?? '';

export function useVoiceRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const [isTranscribing, setIsTranscribing] = useState(false);
  const [transcription, setTranscription] = useState('');
  const [error, setError] = useState<string | null>(null);
  const recordingRef = useRef<Audio.Recording | null>(null);

  // マイク録音の開始
  const startRecording = async () => {
    try {
      // マイクのパーミッションをリクエスト
      const { granted } = await Audio.requestPermissionsAsync();
      if (!granted) throw new Error('マイクの使用が許可されていません');

      await Audio.setAudioModeAsync({
        allowsRecordingIOS: true,
        playsInSilentModeIOS: true,
      });

      const { recording } = await Audio.Recording.createAsync(
        Audio.RecordingOptionsPresets.HIGH_QUALITY
      );
      recordingRef.current = recording;
      setIsRecording(true);
      setError(null);
    } catch (err) {
      setError(`録音エラー: ${err}`);
    }
  };

  // 録音停止 & Whisper API に送信
  const stopRecordingAndTranscribe = async () => {
    if (!recordingRef.current) return;
    setIsRecording(false);
    setIsTranscribing(true);

    try {
      await recordingRef.current.stopAndUnloadAsync();
      const uri = recordingRef.current.getURI();
      if (!uri) throw new Error('録音ファイルが見つかりません');

      // Whisper API へ送信
      const result = await transcribeAudio(uri);
      setTranscription(result);
    } catch (err) {
      setError(`文字起こしエラー: ${err}`);
    } finally {
      setIsTranscribing(false);
      recordingRef.current = null;
    }
  };

  return { isRecording, isTranscribing, transcription, error,
           startRecording, stopRecordingAndTranscribe };
}

// Whisper API に音声ファイルを送信する関数
async function transcribeAudio(audioUri: string): Promise<string> {
  const formData = new FormData();

  // React Native の FormData は URI 形式で添付
  formData.append('file', {
    uri: audioUri,
    type: 'audio/m4a',
    name: 'recording.m4a',
  } as any);
  formData.append('model', 'whisper-1');
  formData.append('language', 'ja'); // 日本語を指定（省略すると自動検出）

  const response = await fetch(WHISPER_API_URL, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${OPENAI_API_KEY}`,
      // Content-Type は FormData が自動設定するので指定不要
    },
    body: formData,
  });

  if (!response.ok) {
    const errorData = await response.json();
    throw new Error(errorData.error?.message ?? 'API エラー');
  }

  const data = await response.json();
  return data.text; // 文字起こし結果
  // 期待する出力例: "今日はいい天気ですね。明日の会議の準備をしましょう。"
}
```

### Step 3：環境変数の設定

`app.config.js` または `app.json` に API キーを追加します：

```javascript
// app.config.js
export default {
  expo: {
    name: 'VoiceNote',
    slug: 'voice-note',
    extra: {
      openaiApiKey: process.env.EXPO_PUBLIC_OPENAI_API_KEY,
    },
    plugins: [
      [
        'expo-av',
        {
          microphonePermission:
            '音声メモを録音するためにマイクへのアクセスが必要です。',
        },
      ],
    ],
  },
};
```

**.env ファイル**（リポジトリには含めないこと）：

```bash
EXPO_PUBLIC_OPENAI_API_KEY=sk-proj-xxxxxxxxxxxxxxxxxxxx
```

> ⚠️ **セキュリティ注意**: `EXPO_PUBLIC_` プレフィックスは Expo でクライアントサイドに公開されます。本番環境では Cloudflare Workers や Supabase Edge Functions などのバックエンドを経由して API キーを隠すことを推奨します。詳しくは [Rork + Cloudflare AI バックエンドガイド](/articles/rork-dev/rork-cloudflare-ai-backend) を参照。

## よくあるエラーと対処法

### エラー 1：`Audio.Recording.createAsync is not a function`

**原因**: `expo-av` が正しくインストールされていない
**解決策**: Rork に「expo-av パッケージを追加してください」と伝えてインストールしてもらう

### エラー 2：`Invalid file format`（Whisper API）

**原因**: 送信した音声ファイルのフォーマットが非対応
**解決策**: `Audio.RecordingOptionsPresets.HIGH_QUALITY` を使用（m4a フォーマットが保証される）

```typescript
// ✅ 推奨
const { recording } = await Audio.Recording.createAsync(
  Audio.RecordingOptionsPresets.HIGH_QUALITY
);

// ❌ カスタム設定でフォーマットを間違えるとエラーになる
```

### エラー 3：`Request too large`

**原因**: 録音ファイルが 25MB を超えている（約4時間以上の録音）
**解決策**: 録音時間を制限するか、ファイルを分割して送信する

```typescript
// 録音時間の上限（例：3分 = 180,000ms）
const MAX_RECORDING_MS = 180_000;

setTimeout(() => {
  if (isRecording) stopRecordingAndTranscribe();
}, MAX_RECORDING_MS);
```

### エラー 4：マイクパーミッションが取得できない

**原因**: iOS Info.plist または Android Manifest に権限設定がない
**解決策**: `app.config.js` の `plugins` 設定を確認（Step 3 参照）

### アイデア 1：リアルタイム議事録アプリ

## FAQ

**Q1. Whisper API の料金はどのくらいかかりますか？**

A. $0.006 / 分です（2026年3月時点）。1分の音声を100回送信しても $0.60 程度です。月1,000分の利用で約 $6、非常にコスト効率が良いです。ただし OpenAI の価格は変動することがあるため、[公式ページ](https://openai.com/pricing) で最新情報を確認してください。

**Q2. オフライン（インターネット接続なし）でも使えますか？**

A. Whisper API はクラウド API なのでオフラインでは使えません。オフライン対応が必要な場合は、Apple の Speech Framework（iOS 専用）や、Rork Max で Core ML を使ったオンデバイス推論を検討してください。

**Q3. 録音した音声データはどこに保存されますか？**

A. `expo-av` で録音したファイルはデバイスの一時ストレージに保存されます。Whisper API に送信後はアプリが削除します。OpenAI は API で送信した音声を30日間保存する場合があるため、プライバシーポリシーで利用者に明示しましょう。

**Q4. Rork（無料プラン）でも使えますか？**

A. Rork の無料プランでもプロジェクト作成は可能ですが、生成できるコードの複雑さに制限があります。API 連携コードの完全生成には Pro プラン以上を推奨します。

**Q5. Android でもマイクは動作しますか？**

A. はい。`expo-av` は iOS・Android 両対応です。Android では `AndroidManifest.xml` に `RECORD_AUDIO` パーミッションが必要ですが、`app.config.js` の設定で Expo が自動的に追加します。
