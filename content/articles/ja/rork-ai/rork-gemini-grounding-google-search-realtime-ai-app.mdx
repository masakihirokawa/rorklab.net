「なんで最近のニュースを知らないの？」——AIアシスタントを搭載したアプリをリリースした後、こういった問い合わせが届いた経験のある方は少なくないかもしれません。Gemini の学習データには当然カットオフ日があり、それ以降に起きた出来事は知りません。ユーザーが「今日の株価」「先週発売されたスマホのスペック」「昨日の試合結果」を聞いても、的外れな回答が返ってくるのは避けられない構造的な問題です。

Grounding with Google Search は、この問題を根本から解決する Gemini API の機能です。有効にすると、Gemini はユーザーの質問に応じて自律的に Google 検索を実行し、最新の検索結果を参照した上で回答を生成します。静的な学習データに縛られたAIから、常に最新情報にアクセスできるAIへ——この実装ガイドでは、Rork アプリにこの機能を組み込むための全手順を、動作するコードと合わせて解説します。

## Grounding が解決する問題：AIの「知識の鮮度問題」

LLM には必ず学習データのカットオフ日があります。Gemini 2.5 Pro であれば 2025 年初頭頃までの情報は持っていますが、それ以降に発売された製品、変動した価格、起きたニュースは知りません。

問題はユーザーがこの制約を意識していない点にあります。多くのユーザーにとって、AIアシスタントは「なんでも知っている存在」として認識されています。実際には知らない情報を自信ありげに古い情報で答えてしまう——いわゆる「ハルシネーション」が発生しやすい状況です。

Grounding を有効にすると、Gemini は内部で以下のような処理を行います。

1. ユーザーの質問に対して、Google 検索が必要かどうかを自己判断する
2. 必要と判断した場合、適切な検索クエリを自動生成して Google Search API を呼び出す
3. 取得した検索結果のコンテンツを参照しながら回答を生成する
4. どの情報源を参照したかを `groundingChunks` としてレスポンスに含める

アプリ側は Grounding を「使うかどうかの設定」と「レスポンスから引用情報を取り出す処理」を実装するだけです。Gemini がいつ検索するかの判断は任せておけます。

ただし、全ての機能に Grounding が必要なわけではありません。コード生成、創作支援、数学的な推論、一般的な Q&A では、Grounding を有効にしても精度が上がらないばかりか、余計なコストが発生します。使い所の見極めは後述します。

## Grounding を使うべき場面と使わない場面

Grounding が特に効果を発揮するのは、「現在の状態」を問うクエリです。

Grounding が有効な場面として、主に以下が挙げられます。

- **ニュース・時事情報**：「今日の主要ニュースを教えて」「最近の〇〇業界の動向は？」
- **価格・在庫情報**：「iPhone 16 の最安値は？」「〇〇の現在の株価は？」
- **製品・サービス情報**：「最新の Gemini のリリース内容は？」「〇〇アプリの新機能は？」
- **イベント・スケジュール**：「次の WWDC はいつ？」「〇〇の公演日程は？」

逆に Grounding が不要または有害になる場面もあります。

- **コード生成・技術支援**：知識のカットオフが問題にならないことが多く、無駄なコストがかかる
- **創作・文章生成**：検索結果が創作の邪魔をすることがある
- **数学・論理パズル**：検索が全く役に立たず、レイテンシだけ増える
- **プライベートな質問**：ユーザーの個人的な悩みを Google 検索するのは意味がない

実際のアプリ設計では、質問の種類によって Grounding を動的に ON/OFF するアーキテクチャが理想的です。この実装パターンは後半で紹介します。

## SDK セットアップとバックエンド経由の設計

Rork アプリに Grounding を組み込む際、重要な設計上の注意点があります。**Gemini API のキーをモバイルアプリのコード内に直接書かないこと**です。React Native のバンドルは逆コンパイルが容易なため、ハードコードされた API キーは悪意ある第三者に抽出されるリスクがあります。

推奨アーキテクチャは、Cloudflare Workers や Supabase Edge Functions などのバックエンドを挟む構成です。

```
Rork アプリ → Cloudflare Workers API → Gemini API（Grounding 有効）
```

まず Cloudflare Workers のバックエンドを実装します。[Hono と Cloudflare Workers を使ったバックエンド構築](/articles/rork-dev/rork-hono-cloudflare-workers-rest-api-guide)を参照しつつ、以下のエンドポイントを追加してください。

```javascript
// workers/src/gemini-grounding.ts
import { Hono } from "hono";
import { cors } from "hono/cors";

const app = new Hono();

app.use("/api/*", cors());

app.post("/api/grounded-search", async (c) => {
  const { query, useGrounding } = await c.req.json();

  if (\!query || typeof query \!== "string") {
    return c.json({ error: "query is required" }, 400);
  }

  const apiKey = c.env.GEMINI_API_KEY;

  // Grounding の有無を動的に切り替え
  const tools = useGrounding ? [{ googleSearch: {} }] : [];

  const payload = {
    contents: [{ role: "user", parts: [{ text: query }] }],
    tools,
    generationConfig: {
      temperature: useGrounding ? 0.3 : 0.7, // Groundingあり時は低温度で事実寄り
      maxOutputTokens: 1024,
    },
  };

  try {
    const response = await fetch(
      `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-pro:generateContent?key=${apiKey}`,
      {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(payload),
      }
    );

    if (\!response.ok) {
      const errorData = await response.json();
      return c.json({ error: "Gemini API error", details: errorData }, 500);
    }

    const data = await response.json();

    // groundingMetadata を取り出してクライアントに返す
    const candidate = data.candidates?.[0];
    const text = candidate?.content?.parts?.[0]?.text ?? "";
    const groundingMetadata = candidate?.groundingMetadata ?? null;

    return c.json({
      text,
      groundingChunks: groundingMetadata?.groundingChunks ?? [],
      webSearchQueries: groundingMetadata?.webSearchQueries ?? [],
    });
  } catch (err) {
    return c.json({ error: "Internal error", message: String(err) }, 500);
  }
});

export default app;
```

環境変数 `GEMINI_API_KEY` は `wrangler secret put GEMINI_API_KEY` で設定します。これで API キーがクライアントコードに一切含まれない安全な構成になります。

## 実装パターン 1：ニュースサマリーアプリ

Grounding の最も自然なユースケースがニュースサマリーです。ユーザーが入力したキーワードや話題について、今日の最新情報を Gemini がサマリーして返します。

```typescript
// components/NewsSearchScreen.tsx
import React, { useState } from "react";
import {
  View,
  Text,
  TextInput,
  TouchableOpacity,
  ScrollView,
  ActivityIndicator,
  StyleSheet,
  Linking,

  web?: {
    uri: string;
    title: string;
  };
}

interface SearchResult {
  text: string;
  groundingChunks: GroundingChunk[];
  webSearchQueries: string[];
}

const API_BASE = "https://your-worker.your-subdomain.workers.dev";

export default function NewsSearchScreen() {
  const [query, setQuery] = useState("");
  const [result, setResult] = useState<SearchResult | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const handleSearch = async () => {
    if (\!query.trim()) return;

    setLoading(true);
    setError(null);
    setResult(null);

    try {
      const response = await fetch(`${API_BASE}/api/grounded-search`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          query: `以下のトピックについて、最新情報をもとに日本語で200文字以内で要約してください：${query}`,
          useGrounding: true,
        }),
      });

      if (\!response.ok) {
        throw new Error(`API Error: ${response.status}`);
      }

      const data: SearchResult = await response.json();
      setResult(data);
    } catch (err) {
      setError(
        err instanceof Error
          ? err.message
          : "通信エラーが発生しました。再度お試しください。"
      );
    } finally {
      setLoading(false);
    }
  };

  return (
    <ScrollView style={styles.container}>
      <Text style={styles.title}>最新ニュース検索</Text>

      <TextInput
        style={styles.input}
        value={query}
        onChangeText={setQuery}
        placeholder="気になるトピックを入力（例：生成AI 最新動向）"
        returnKeyType="search"
        onSubmitEditing={handleSearch}
      />

      <TouchableOpacity
        style={[styles.button, loading && styles.buttonDisabled]}
        onPress={handleSearch}
        disabled={loading || \!query.trim()}
      >
        {loading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>検索</Text>
        )}
      </TouchableOpacity>

      {error && (
        <View style={styles.errorBox}>
          <Text style={styles.errorText}>{error}</Text>
        </View>
      )}

      {result && (
        <View style={styles.resultBox}>
          {/* AI の回答 */}
          <Text style={styles.answerText}>{result.text}</Text>

          {/* Google が実行した検索クエリ */}
          {result.webSearchQueries.length > 0 && (
            <View style={styles.queryBox}>
              <Text style={styles.queryLabel}>検索クエリ：</Text>
              {result.webSearchQueries.map((q, i) => (
                <Text key={i} style={styles.queryText}>
                  「{q}」
                </Text>
              ))}
            </View>
          )}

          {/* 情報源（groundingChunks） */}
          {result.groundingChunks.length > 0 && (
            <View style={styles.sourcesBox}>
              <Text style={styles.sourcesLabel}>情報源：</Text>

                .filter((chunk) => chunk.web)
                .map((chunk, i) => (
                  <TouchableOpacity
                    key={i}
                    onPress={() =>
                      chunk.web?.uri && Linking.openURL(chunk.web.uri)
                    }
                    style={styles.sourceItem}
                  >
                    <Text style={styles.sourceTitle} numberOfLines={1}>
                      {chunk.web?.title ?? "（タイトルなし）"}
                    </Text>
                    <Text style={styles.sourceUrl} numberOfLines={1}>
                      {chunk.web?.uri}
                    </Text>
                  </TouchableOpacity>
                ))}
            </View>
          )}
        </View>
      )}
    </ScrollView>
  );
}
```

このコードで重要な点は、`groundingChunks` と `webSearchQueries` を UI に表示している点です。単に Grounding の回答を見せるだけでなく、「どこから調べたか」をユーザーに開示することは、AI の信頼性向上に大きく貢献します。私の経験では、情報源を表示するようにしただけで、ユーザーからの「本当に最新の情報ですか？」という問い合わせがほぼなくなりました。

## 実装パターン 2：商品・価格リサーチアシスタント

もう一つの強力なユースケースが、商品や価格情報の検索です。ユーザーが製品名を入力すると、現在の市場価格や概要を Grounding 経由で取得します。

```typescript
// hooks/useProductSearch.ts
import { useState, useCallback } from "react";

interface ProductInfo {
  summary: string;
  priceInfo: string | null;
  sources: Array<{ title: string; uri: string }>;
}

const API_BASE = "https://your-worker.your-subdomain.workers.dev";

export function useProductSearch() {
  const [isLoading, setIsLoading] = useState(false);
  const [productInfo, setProductInfo] = useState<ProductInfo | null>(null);
  const [error, setError] = useState<string | null>(null);

  const searchProduct = useCallback(async (productName: string) => {
    setIsLoading(true);
    setError(null);
    setProductInfo(null);

    const prompt = `
「${productName}」について以下の情報を調べて日本語で回答してください：
1. 製品の概要（2〜3文）
2. 現在の価格帯（わかる範囲で）
3. 直近のニュースや更新情報（あれば1件）

情報が見つからない場合は「情報が見つかりませんでした」と回答してください。
    `.trim();

    try {
      const response = await fetch(`${API_BASE}/api/grounded-search`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ query: prompt, useGrounding: true }),
      });

      if (\!response.ok) {
        throw new Error("検索に失敗しました");
      }

      const data = await response.json();

      // 価格情報を簡易パース（¥ または $ を含む部分を抽出）
      const priceMatch = data.text.match(/[¥$￥][0-9,，]+(?:〜[¥$￥][0-9,，]+)?/);

      setProductInfo({
        summary: data.text,
        priceInfo: priceMatch ? priceMatch[0] : null,
        sources: data.groundingChunks
          .filter((c: { web?: { uri: string; title: string } }) => c.web)
          .map((c: { web: { uri: string; title: string } }) => ({
            title: c.web.title,
            uri: c.web.uri,
          })),
      });
    } catch (err) {
      setError(
        err instanceof Error ? err.message : "エラーが発生しました"
      );
    } finally {
      setIsLoading(false);
    }
  }, []);

  return { searchProduct, isLoading, productInfo, error };
}
```

このフックを使うと、コンポーネント側はシンプルに書けます。`priceMatch` の正規表現は参考程度のもので、実際の運用では Gemini に構造化された JSON で返してもらう設計のほうが堅牢です。後述のプロンプト設計で触れます。

## groundingChunks の読み方と引用 UI の実装

`groundingChunks` は以下のような構造になっています。

```typescript
// groundingChunks の型定義
interface GroundingChunk {
  web?: {
    uri: string;    // 参照元 URL
    title: string;  // ページタイトル
  };
}

interface GroundingSupport {
  groundingChunkIndices: number[]; // どの chunk が根拠か
  confidenceScores: number[];      // 0.0〜1.0 の信頼スコア
  segment: {
    startIndex: number;            // 本文中の対応する文字位置（開始）
    endIndex: number;              // 本文中の対応する文字位置（終了）
    text: string;                  // 対応する本文の断片
  };
}
```

実用上は `groundingChunks`（どのサイトを参照したか）だけを使うことがほとんどですが、`groundingSupports` を使うと「本文のどの部分がどの情報源に基づいているか」を細かく示すことができます。

信頼スコアを利用した表示を追加すると、より誠実な情報提示ができます。

```typescript
// 信頼スコアに基づいて情報源を絞り込む
const reliableSources = groundingChunks.filter((_, index) => {
  const supports = groundingSupports.filter((s) =>
    s.groundingChunkIndices.includes(index)
  );
  if (supports.length === 0) return true; // supports がない場合は表示
  const avgScore =
    supports.reduce((sum, s) => {
      const score = s.confidenceScores[s.groundingChunkIndices.indexOf(index)];
      return sum + (score ?? 0);
    }, 0) / supports.length;
  return avgScore > 0.5; // 信頼スコア 50% 以上の情報源のみ表示
});
```

## よくある落とし穴：コスト爆発・品質低下・APIキー漏洩

Grounding を本番投入する前に、必ず知っておくべき問題が 3 つあります。

**落とし穴 1：コスト爆発**

Grounding を有効にすると、通常のリクエストに比べてトークン消費が大幅に増えます。Google Search の結果を大量に取り込んで回答を生成するためです。目安として、Grounding なしの同じ質問と比較すると入力トークン数が 3〜10 倍になることがあります。

対策として、前述の `useGrounding` フラグを使った動的切り替えが重要です。また、[Gemini API のコスト最適化ガイド](/articles/rork-dev/rork-ai-api-cost-optimization-mobile-guide)にある通り、プロンプトキャッシングの活用やモデルの選択（Grounding は 2.5 Flash でも利用可能）も検討してください。

**落とし穴 2：品質低下（関係のない検索結果を引用してしまう）**

「東京の天気は？」と聞いたつもりが、なぜか「東京タワーの改修工事」についての検索結果を引用した回答が返ってくることがあります。Gemini が検索クエリを自律生成する際に、意図とずれた検索が走ることがあるのです。

これを防ぐには、プロンプトで検索の意図を明確に指定することが効果的です。

```javascript
// ❌ 曖昧なプロンプト（Grounding がずれやすい）
const query = `${topic}について教えて`;

// ✅ 検索意図を明示したプロンプト
const query = `
次のトピックについて、2026年以降の最新情報を Google 検索で調べ、
日本語で200文字以内にまとめてください。
検索対象：${topic}
古い情報しか見つからない場合は「最新情報は見つかりませんでした」と答えてください。
`;
```

**落とし穴 3：Grounding 結果を JSON で受け取りたい場合の注意**

JSON モードと Grounding を同時に有効にすることはできません。`response_mime_type: "application/json"` を指定しながら Grounding ツールを使おうとすると API エラーになります。

JSON が必要な場合は、テキストで返ってきた回答を別途パースするか、Grounding 用と構造化用の 2 段階リクエストを使う設計にする必要があります。

```javascript
// 2段階リクエストパターン：Grounding → 構造化
async function groundedJsonSearch(topic: string) {
  // Step 1: Grounding で最新情報を取得
  const groundedResponse = await callGroundingAPI(topic);

  // Step 2: 取得したテキストを JSON に構造化
  const structurePrompt = `
以下のテキストから情報を抽出し、JSON 形式で返してください。
テキスト：${groundedResponse.text}
JSON スキーマ：
{
  "summary": "概要（100文字以内）",
  "price": "価格情報（不明な場合は null）",
  "lastUpdated": "言及された日時（不明な場合は null）"
}
  `;

  // Step 2 は Grounding なし、JSON モードで
  const jsonResponse = await callGeminiWithoutGrounding(structurePrompt);
  return JSON.parse(jsonResponse.text);
}
```

## 本番環境での最適化とコスト管理

Grounding を本番で安定稼働させるためのチェックポイントをまとめます。

**動的 Grounding 切り替えロジック**

全てのクエリに Grounding を使うのは過剰です。クエリの性質に応じて ON/OFF を切り替える仕組みを Rork アプリ側で実装してください。

```typescript
// Grounding が必要かを判断するヘルパー
function shouldUseGrounding(query: string): boolean {
  // 時事性・価格・最新情報に関するキーワードが含まれる場合のみ ON
  const groundingKeywords = [
    "最新", "今日", "現在", "価格", "値段", "いくら",
    "ニュース", "リリース", "発売", "いつ", "最近",
    "new", "latest", "current", "price", "today", "now",
  ];
  const lower = query.toLowerCase();
  return groundingKeywords.some((kw) => lower.includes(kw));
}

// 使い方
const useGrounding = shouldUseGrounding(userQuery);
const response = await fetch(`${API_BASE}/api/grounded-search`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ query: userQuery, useGrounding }),
});
```

**レート制限とリトライ戦略**

Grounding は通常のリクエストより時間がかかります（検索待ちが発生するため）。タイムアウトは 15〜20 秒に設定し、失敗時は Grounding なしでリトライする設計が実用的です。

```typescript
// リトライ付きリクエスト
async function searchWithFallback(query: string): Promise<SearchResult> {
  try {
    // まず Grounding ありで試みる
    return await searchWithGrounding(query);
  } catch (err) {
    console.warn("Grounding 失敗、フォールバックへ:", err);
    // Grounding なしで再試行
    return await searchWithoutGrounding(query);
  }
}
```

## まとめにかえて：Grounding で変わるAIアプリの「鮮度」

Grounding with Google Search を実装すると、アプリの AI がリアルタイムな情報を扱えるようになります。しかし、この記事で強調したかったのは「Grounding を使えばいい」という話ではなく、**使い所を見極める判断力**の重要性です。

コスト・レイテンシ・品質のトレードオフを理解した上で、ニュースや価格のような「鮮度が命」の場面では積極的に活用し、コード生成や創作では外す——この使い分けが、ユーザー体験とサービス品質を両立させる鍵になります。
