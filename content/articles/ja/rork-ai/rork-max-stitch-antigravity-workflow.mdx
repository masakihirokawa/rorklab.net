## はじめに：3つのAIツールが変えるアプリ開発の常識

2026年のモバイルアプリ開発において、最もホットな組み合わせが **Rork Max × Google Stitch × Antigravity** のトリプルAIワークフローです。

それぞれのツールが得意とする領域を組み合わせることで、**デザインの初稿からApp Store申請まで、最短1日**で完結できる開発体制を構築できます。

- **Google Stitch**：自然言語から美しいUIデザイン（HTML/CSS）を生成。デザイナーなしでも高品質なビジュアルを作れる

- **Rork Max**：生成されたコードをベースにSwiftUIネイティブアプリを構築し、クラウドMacでコンパイル・App Store申請

この記事では、ヘルスケアアプリを題材に、この3ツールを使った**エンドツーエンドの実践チュートリアル**を詳しく解説します。プレミアム限定で、実際のプロンプト集や設計パターン、よくある落とし穴の回避方法まで網羅します。

### Google StitchのMCPサーバーをAntigravityに追加する

StitchはAntigravityのMCPストアに正式対応しています。

1. [ai.google.dev/antigravity](https://ai.google.dev) にアクセスしてAntigravityを開く
2. 左サイドバーの「MCP Servers」→「+ Add Server」をクリック
3. 検索フォームに「Stitch」と入力して公式MCPを選択
4. Stitch側でAPIキーを発行：Stitch アプリ → **Exports** パネル → 「Get API Key」
5. Antigravity の設定フォームにAPIキーを貼り付けて「Connect」

```json
// Antigravity MCPの設定例（settings.json）
{
  "mcpServers": {
    "stitch": {
      "name": "Google Stitch",
      "apiKey": "YOUR_STITCH_API_KEY",
      "tools": [
        "generate_screen",
        "edit_screen",
        "export_html_css",
        "create_variant"
      ]
    }
  }
}
```

接続後、Antigravityのプロンプトから `@stitch` でStitchの機能を直接呼び出せます。

## 概念と設計思想：なぜこの3ツール組み合わせが最強なのか

### 従来の開発フローとの比較

従来のノーコード開発では、**デザイン → コード変換 → ネイティブ化**という3段階がそれぞれ分断されていました。Figmaでデザインを作り、FlutterFlowやAdaloにインポートし、ネイティブ機能は別途実装する...という作業で数日かかっていた工程が、このトリプルAIワークフローでは大幅に短縮されます。

```
【従来フロー】
Figma（デザイン）→ 手動コード変換 → FlutterFlow/React Native → ネイティブ化
所要時間: 3〜7日

【新ワークフロー】
Google Stitch（AIデザイン）→ Antigravity（コード変換・補完）→ Rork Max（ネイティブSwiftUI）
所要時間: 4〜8時間
```

### 各ツールの役割分担

**Google Stitch**は「ビジュアルデザインのAIエンジン」として機能します。テキストプロンプトから、色彩設計・タイポグラフィ・コンポーネント配置まで整ったUIを生成します。特にカラースキームやスペーシングの一貫性が高く、デザインシステムを持たない個人開発者にとって強力な味方です。

**Antigravity**は「AIコードエージェント」として、StitchのデザインをリアルタイムでReact Native/SwiftUIコードに変換します。MCPサーバー統合により、Stitch内のデザインを直接参照しながらコード生成できるため、デザインと実装の乖離が生じにくいのが最大の特徴です。

**Rork Max**はこのパイプラインの最終ステージで、「ネイティブSwiftUIコンパイル環境」として動作します。AntigravityからエクスポートしたコードをRork Maxのプロジェクトにマージし、クラウドMacでビルド・シミュレーション・App Store申請までワンクリックで完結します。

詳細な比較はRork vs FlutterFlow完全比較や[Rork Max AI機能ガイド](/articles/rork-ai/rork-max-ai-features)もあわせてご覧ください。

### Step 2：Antigravity でデザインをコードに変換する

StitchのデザインがMCPで連携されたAntigravityで、以下のプロンプトを実行します。

```
Using the @stitch "health_app_home" design, generate SwiftUI code for:
1. HomeView with CircularProgressView components for steps, calories, water
2. Use @Observable macro for state management (iOS 17+)
3. Implement smooth animations using withAnimation(.spring(duration: 0.5))
4. Add HapticFeedback on ring completion
5. Data model: HealthMetric struct with id, type, value, goal, date
```

Antigravityが生成するSwiftUIコードの例：

```swift
// AntigravityがStitchデザインから生成したCircularProgressViewの例
import SwiftUI

struct CircularProgressView: View {
    let value: Double      // 現在値（例: 6500歩）
    let goal: Double       // 目標値（例: 10000歩）
    let color: Color       // リング色
    let label: String      // ラベル（"Steps"など）

    private var progress: Double {
        min(value / goal, 1.0)
    }

    var body: some View {
        ZStack {
            // 背景リング
            Circle()
                .stroke(color.opacity(0.15), lineWidth: 12)

            // 進捗リング
            Circle()
                .trim(from: 0, to: progress)
                .stroke(
                    color,
                    style: StrokeStyle(lineWidth: 12, lineCap: .round)
                )
                .rotationEffect(.degrees(-90))
                .animation(.spring(duration: 0.8), value: progress)

            // 中央テキスト
            VStack(spacing: 2) {
                Text("\(Int(value))")
                    .font(.system(size: 22, weight: .bold, design: .rounded))
                    .foregroundStyle(.primary)
                Text(label)
                    .font(.caption2)
                    .foregroundStyle(.secondary)
            }
        }
        .frame(width: 100, height: 100)
    }
}

// 期待する出力: 滑らかなアニメーションで進捗が表示されるリング
// 値が変化するたびにspringアニメーションで更新される
```

### Step 4：サブスクリプション課金の実装（StoreKit 2）

Rork Maxが生成するStoreKit 2の実装コード：

```swift
// Rork MaxのAIが生成するSubscriptionManager
import StoreKit

@Observable
class SubscriptionManager {
    var isPremium: Bool = false
    var products: [Product] = []

    // App Store Connect で設定したProduct ID
    let productIDs = [
        "com.yourapp.health.monthly",   // $4.99/月
        "com.yourapp.health.yearly"     // $39.99/年（約33%割引）
    ]

    init() {
        Task {
            await loadProducts()
            await updateSubscriptionStatus()
        }
    }

    func loadProducts() async {
        do {
            products = try await Product.products(for: productIDs)
        } catch {
            print("Failed to load products: \(error)")
        }
    }

    func purchase(_ product: Product) async throws {
        let result = try await product.purchase()
        switch result {
        case .success(let verification):
            let transaction = try checkVerified(verification)
            await transaction.finish()
            await updateSubscriptionStatus()
        case .userCancelled, .pending:
            break
        @unknown default:
            break
        }
    }

    func updateSubscriptionStatus() async {
        for await result in Transaction.currentEntitlements {
            guard case .verified(let transaction) = result else { continue }
            if productIDs.contains(transaction.productID) {
                isPremium = transaction.revocationDate == nil
            }
        }
    }

    private func checkVerified<T>(_ result: VerificationResult<T>) throws -> T {
        switch result {
        case .unverified:
            throw StoreError.failedVerification
        case .verified(let safe):
            return safe
        }
    }
}

enum StoreError: Error {
    case failedVerification
}

// 使い方：
// @Environment(SubscriptionManager.self) var subscriptionManager
// if subscriptionManager.isPremium { /* プレミアム機能を表示 */ }
```

### パターン3：Rork Max + Cloudflare Workers バックエンド

フロントエンドをRork Maxで構築しつつ、バックエンドAPIをCloudflare Workersで実装する組み合わせも強力です。

```javascript
// Cloudflare Worker: HealthData APIエンドポイント例
export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    if (url.pathname === '/api/health/sync' && request.method === 'POST') {
      const { userId, metrics } = await request.json();

      // Cloudflare KVに保存（高速read）
      await env.HEALTH_KV.put(
        `user:${userId}:metrics`,
        JSON.stringify({ metrics, updatedAt: Date.now() }),
        { expirationTtl: 86400 * 30 } // 30日間保存
      );

      return Response.json({ success: true });
    }

    return new Response('Not Found', { status: 404 });
  }
};
```

## トラブルシューティング：よくある問題と解決法

### 問題1：StitchのデザインがAntigravityで正しく参照されない

**原因**: MCPサーバーのAPIキー期限切れ、またはStitch画面のIDが変更されている

**解決法**:
1. AntigravityのMCP設定でStitchのAPIキーを再発行・更新
2. `@stitch list_screens` でスクリーンID一覧を確認してからコード生成

### 問題2：Rork MaxへのSwiftUIコードインポートで型エラーが発生する

**原因**: AntigravityがiOS 16以前の古いAPIを使用するコードを生成することがある

**解決法**:
```
// Antigravityへの追加指示
"Ensure all generated code targets iOS 17+ minimum.
Use @Observable instead of ObservableObject+@Published.
Use SwiftData instead of Core Data where possible."
```

### 問題3：HealthKitデータが実機でnilになる

**原因**: HealthKitのUsage Descriptionがplistに正しく記載されていない

**解決法**: Rork MaxのInfo.plistエディタで`NSHealthShareUsageDescription`の値を確認。App Store審査でも引っかかる項目なので必ず設定する。

### 問題4：StoreKit 2の購読状態がアプリ再起動後にリセットされる

**原因**: `Transaction.currentEntitlements`の監視処理が`App`の`init()`で呼ばれていない

**解決法**:
```swift
@main
struct HealthApp: App {
    @State private var subscriptionManager = SubscriptionManager()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environment(subscriptionManager)
                // アプリ起動時に購読状態を自動確認
                .task {
                    await subscriptionManager.updateSubscriptionStatus()
                }
        }
    }
}
```

## FAQ

**Q1. Google StitchとAntigravityは無料で使えますか？**

A. 2026年3月時点では両ツールともGoogleの無料枠内で利用可能です。ただし、Stitch APIには月間リクエスト数の上限があります。Antigravityは現在ベータ版として無料提供されており、将来的に有料プランが導入される可能性があります。最新の料金情報は[Google AI公式サイト](https://ai.google.dev)でご確認ください。

**Q2. このワークフローで作ったアプリはReact NativeとSwiftUIのどちらですか？**

A. 最終的にRork Maxでビルドする場合は**SwiftUI（ネイティブSwift）**です。Antigravityが最初にReact Nativeコードを生成した場合でも、Rork Maxのプロンプトで「Convert to SwiftUI」と指示することでネイティブコードに変換できます。Android対応が必要な場合は、従来のRork（React Native版）を使うことを検討してください。

**Q3. HealthKitはRork Maxで標準サポートされていますか？**

A. はい。Rork MaxはiOSのネイティブAPIに完全アクセスできるため、HealthKit・Core Data・StoreKit 2・UserNotificationsなどはすべてRork Maxのチャットプロンプトで実装を指示できます。[Rork Max HealthKitガイド](/articles/rork-dev/rork-max-healthkit-guide)も参照してください。

**Q4. Antigravityが生成するコードの品質はどのくらいですか？**

A. StitchのMCPを介したコード生成は、Stitch単独よりも精度が高く、実際のデザインファイルを参照するため色・スペーシング・フォントの再現度が高いです。ただし、ビジネスロジック（データ保存・API連携）については生成後に人間がレビューすることを推奨します。Rork MaxのFix Now自動デバッグ機能と組み合わせることで、エラー修正も自動化できます。

**Q5. このワークフローはAndroidアプリにも対応できますか？**

A. Google StitchとAntigravityはプラットフォーム非依存のため、両対応のコード生成が可能です。ただしRork MaxはApple専用（SwiftUI）のため、Androidネイティブ対応が必要な場合は、AntigravityでReact Native/Expoコードを生成し、通常のRorkプロジェクトにインポートするフローを採用してください。

**Q6. 個人開発者でも1日でApp Storeリリースまで完走できますか？**

A. UIのデザインが固まっていれば、このワークフローで1日以内にTestFlight配布まで到達することは十分可能です。App Store審査（通常24〜48時間）があるため、実際のリリースは翌日以降になりますが、開発作業自体は1日で完結します。審査短縮のコツは[TestFlightガイド](/articles/rork-dev/rork-max-testflight-guide)をご参照ください。
