クラウドに依存しない **エッジ AI**（オンデバイス AI）が、モバイルアプリ開発の新しいスタンダードになりつつあります。2026年現在、Gartner は新規アプリの75%がローコード・ノーコードプラットフォームで構築されると予測しており、その中でもAI機能の組み込みが差別化の鍵を握っています。

## TensorFlow Lite とは何か — モバイル向け機械学習の基盤

TensorFlow Lite（TFLite）は、TensorFlow のモバイル・組み込みデバイス向け軽量ランタイムです。通常の TensorFlow モデルを `.tflite` 形式に変換することで、スマートフォン上で直接推論を実行できます。

### クラウド AI とエッジ AI の違い

| 項目 | クラウド AI | エッジ AI（TFLite） |
|------|-----------|-------------------|
| ネットワーク | 必須 | 不要（オフライン動作） |
| レイテンシ | 100〜500ms | 10〜50ms |
| プライバシー | データが外部に送信される | デバイス内で完結 |
| コスト | API 利用料が発生 | 無料（モデルをアプリに同梱） |
| モデルサイズ | 制限なし | 数MB〜数十MBに最適化 |

特にヘルスケア、金融、教育といったプライバシーが重要な領域では、ユーザーのデータをデバイスから外に出さずに AI 処理できるエッジ AI の価値は非常に大きいです。

### なぜ Rork × TensorFlow Lite なのか

Rork は AI を活用したアプリビルダーとして、React Native / Expo ベースのアプリを生成します。React Native のエコシステムには `@tensorflow/tfjs-react-native` や `expo-camera` など、TensorFlow Lite と組み合わせて使えるライブラリが充実しており、Rork で生成したコードにこれらを統合することで、高度な AI アプリを効率よく開発できます。

```typescript
// app/_layout.tsx または App.tsx
import * as tf from '@tensorflow/tfjs';
import { bundleResourceIO } from '@tensorflow/tfjs-react-native';
import { useEffect, useState } from 'react';

export default function RootLayout() {
  const [isTfReady, setIsTfReady] = useState(false);

  useEffect(() => {
    const initTf = async () => {
      // TensorFlow.js バックエンドを初期化
      await tf.ready();

    return <LoadingScreen message="AI エンジンを準備中..." />;
  }

  return <MainApp />;
}
// 期待する出力: "TensorFlow.js ready, backend: rn-webgl"
```

## 画像分類モデルの統合 — MobileNet をアプリに組み込む

### ステップ1: 事前学習済みモデルの準備

TensorFlow Hub から MobileNet V2（軽量な画像分類モデル）をダウンロードし、TFLite 形式に変換します。すでに変換済みのモデルも公開されています。

```typescript
// hooks/useImageClassifier.ts
import * as tf from '@tensorflow/tfjs';
import { bundleResourceIO } from '@tensorflow/tfjs-react-native';

// モデルとラベルのアセットを import
const modelJSON = require('../assets/model/mobilenet_v2/model.json');
const modelWeights = require('../assets/model/mobilenet_v2/weights.bin');

export function useImageClassifier() {
  const [model, setModel] = useState<tf.GraphModel | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    const loadModel = async () => {
      try {
        // バンドルされたモデルを読み込み
        const loadedModel = await tf.loadGraphModel(
          bundleResourceIO(modelJSON, modelWeights)
        );
        setModel(loadedModel);
        console.log('MobileNet V2 モデル読み込み完了');
      } catch (error) {
        console.error('モデル読み込みエラー:', error);
      } finally {
        setIsLoading(false);
      }
    };
    loadModel();
  }, []);

  return { model, isLoading };
}
// 期待する出力: "MobileNet V2 モデル読み込み完了"
```

### ステップ2: 画像の前処理と推論

カメラやギャラリーから取得した画像を、モデルが期待する入力テンソルに変換して推論を実行します。

```typescript
// utils/imageClassification.ts
import * as tf from '@tensorflow/tfjs';
import { decodeJpeg } from '@tensorflow/tfjs-react-native';
import * as FileSystem from 'expo-file-system';
import { IMAGENET_LABELS } from '../constants/imagenetLabels';

interface ClassificationResult {
  label: string;
  confidence: number;
}

export async function classifyImage(
  model: tf.GraphModel,
  imageUri: string
): Promise<ClassificationResult[]> {
  // 画像をバイナリデータとして読み込み
  const imageData = await FileSystem.readAsStringAsync(imageUri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // Base64 → Uint8Array → テンソルに変換
  const rawBytes = tf.util.encodeString(imageData, 'base64');
  const imageTensor = decodeJpeg(new Uint8Array(rawBytes.buffer));

  // MobileNet V2 の入力サイズ（224x224）にリサイズ
  const resized = tf.image.resizeBilinear(imageTensor, [224, 224]);

  // 正規化（0〜255 → -1〜1）
  const normalized = resized.div(127.5).sub(1.0);

  // バッチ次元を追加 [224,224,3] → [1,224,224,3]
  const batched = normalized.expandDims(0);

  // 推論を実行
  const predictions = model.predict(batched) as tf.Tensor;
  const probabilities = await predictions.data();

  // 上位5件の結果を取得
  const topK = Array.from(probabilities)
    .map((prob, index) => ({ label: IMAGENET_LABELS[index], confidence: prob }))
    .sort((a, b) => b.confidence - a.confidence)
    .slice(0, 5);

  // メモリ解放（重要: メモリリークを防止）
  tf.dispose([imageTensor, resized, normalized, batched, predictions]);

  return topK;
}
// 使用例の期待する出力:
// [
//   { label: "golden retriever", confidence: 0.89 },
//   { label: "Labrador retriever", confidence: 0.05 },
//   { label: "cocker spaniel", confidence: 0.02 },
//   ...
// ]
```

## リアルタイムカメラ推論の実装

ギャラリーからの画像認識だけでなく、カメラのライブプレビューからリアルタイムに推論を行うパターンも実装してみましょう。

```typescript
// screens/RealtimeClassifier.tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useRef, useState, useCallback } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useImageClassifier } from '../hooks/useImageClassifier';
import { classifyImage } from '../utils/imageClassification';
import { ClassificationResultView } from '../components/ClassificationResult';

export default function RealtimeClassifier() {
  const cameraRef = useRef<CameraView>(null);
  const [permission, requestPermission] = useCameraPermissions();
  const { model, isLoading } = useImageClassifier();
  const [results, setResults] = useState<{ label: string; confidence: number }[]>([]);
  const [isProcessing, setIsProcessing] = useState(false);

  const handleCapture = useCallback(async () => {
    if (!cameraRef.current || !model || isProcessing) return;

    setIsProcessing(true);
    try {
      // カメラからフレームをキャプチャ
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.5,  // 推論には低解像度で十分
        skipProcessing: true,
      });

      if (photo?.uri) {
        const classificationResults = await classifyImage(model, photo.uri);
        setResults(classificationResults);
      }
    } catch (error) {
      console.error('キャプチャエラー:', error);
    } finally {
      setIsProcessing(false);
    }
  }, [model, isProcessing]);

  if (!permission?.granted) {
    return (
      <View style={styles.center}>
        <Text>カメラへのアクセス許可が必要です</Text>
        <TouchableOpacity style={styles.button} onPress={requestPermission}>
          <Text style={styles.buttonText}>許可する</Text>
        </TouchableOpacity>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <CameraView ref={cameraRef} style={styles.camera} facing="back" />

      <TouchableOpacity
        style={[styles.captureButton, isProcessing && styles.disabled]}
        onPress={handleCapture}
        disabled={isLoading || isProcessing}
      >
        <Text style={styles.captureText}>
          {isProcessing ? '認識中...' : '撮影して認識'}
        </Text>
      </TouchableOpacity>

      {results.length > 0 && <ClassificationResultView results={results} />}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
  camera: { flex: 1 },
  center: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  button: { marginTop: 16, paddingHorizontal: 24, paddingVertical: 12, backgroundColor: '#007AFF', borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
  captureButton: { position: 'absolute', bottom: 180, alignSelf: 'center', paddingHorizontal: 32, paddingVertical: 16, backgroundColor: '#007AFF', borderRadius: 24 },
  disabled: { opacity: 0.5 },
  captureText: { color: '#fff', fontSize: 16, fontWeight: '700' },
});
```

## モデルの最適化 — アプリサイズと推論速度を改善する

### 量子化によるモデル圧縮

TensorFlow Lite のモデルは量子化（Quantization）によって大幅に軽量化できます。

```python
# scripts/quantize_model.py
# Python 環境で実行（モデル変換用）
import tensorflow as tf

# SavedModel を読み込み
converter = tf.lite.TFLiteConverter.from_saved_model('saved_model_dir')

# 動的範囲量子化を適用（最も簡単な方法）
converter.optimizations = [tf.lite.Optimize.DEFAULT]

# 変換を実行
tflite_model = converter.convert()

# 量子化済みモデルを保存
with open('mobilenet_v2_quantized.tflite', 'wb') as f:
    f.write(tflite_model)

print(f'元のモデル: ~14MB → 量子化後: ~{len(tflite_model) / 1024 / 1024:.1f}MB')
# 期待する出力: "元のモデル: ~14MB → 量子化後: ~3.5MB"
```

### 最適化のベストプラクティス

| 手法 | 効果 | トレードオフ |
|------|------|------------|
| 動的範囲量子化 | サイズ 75% 削減 | 精度低下 1〜2% |
| float16 量子化 | サイズ 50% 削減 | GPU 推論で最適 |
| 整数量子化（INT8） | サイズ 75% 削減 + 推論高速化 | キャリブレーションデータが必要 |
| モデル蒸留 | 推論速度 2〜5倍 | 教師モデルの準備が必要 |

アプリのユースケースに応じて、精度と速度のバランスを取ることが重要です。例えばリアルタイムのカメラ推論では INT8 量子化が効果的ですし、高精度が求められる医療系アプリでは float16 を選択するケースが多いです。

## オフライン動作の設計パターン

エッジ AI の最大のメリットは、ネットワーク接続なしで動作することです。以下のパターンを組み合わせることで、堅牢なオフライン AI アプリを構築できます。

```typescript
// hooks/useOfflineAI.ts
import NetInfo from '@react-native-community/netinfo';
import { useState, useEffect } from 'react';

type AIMode = 'edge' | 'cloud' | 'hybrid';

export function useOfflineAI() {
  const [isConnected, setIsConnected] = useState(true);
  const [aiMode, setAiMode] = useState<AIMode>('hybrid');

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      setIsConnected(state.isConnected ?? false);

      // ネットワーク状況に応じて AI モードを自動切替
      if (!state.isConnected) {
        setAiMode('edge');     // オフライン → エッジ AI のみ
      } else if (state.type === 'wifi') {
        setAiMode('cloud');    // Wi-Fi → クラウド AI（高精度）
      } else {
        setAiMode('hybrid');   // モバイル回線 → ハイブリッド
      }
    });

    return () => unsubscribe();
  }, []);

  return { isConnected, aiMode };
}
// 使用例:
// const { aiMode } = useOfflineAI();
// if (aiMode === 'edge') → TFLite で推論
// if (aiMode === 'cloud') → API 経由で推論
// if (aiMode === 'hybrid') → エッジで高速推論 + クラウドで補正
```

このハイブリッドアプローチは、NVIDIA NeMo をサーバーサイドで使うアーキテクチャとも組み合わせることができます。サーバー側の高精度モデルとデバイス側の軽量モデルを連携させることで、ユーザー体験と精度の両方を高められます。より高度なエッジ AI 推論アーキテクチャについては、[NVIDIA NeMo × Rork — エッジAI推論を活用した次世代モバイルアプリ開発](/articles/rork-dev/nvidia-nemo-edge-ai-mobile-app-development) で詳しく解説しています。

## よくある質問（FAQ）

### Q1: TensorFlow Lite のモデルはアプリのサイズにどのくらい影響しますか？

MobileNet V2 の量子化済みモデルは約 3.5MB です。これはアプリ全体のサイズに比べれば軽微な増加ですが、複数のモデルを同梱する場合は App Bundle の初回ダウンロードと On-Demand Resources（iOS）や Play Feature Delivery（Android）を活用し、必要なモデルだけを後からダウンロードする設計が推奨されます。

### Q2: Core ML と TensorFlow Lite はどちらを選ぶべきですか？

iOS のみをターゲットにする場合は Core ML が Apple のハードウェア最適化（Neural Engine）を最大限に活用でき、推論速度が優れています。一方、iOS と Android の両方に対応する必要がある場合は TensorFlow Lite がクロスプラットフォーム対応として適しています。Rork で開発する場合、React Native ベースなので TensorFlow Lite の方が統合しやすいケースが多いです。Core ML の詳細については [Rork Max × Core ML：オンデバイスAIアプリを作る完全ガイド](/articles/rork-dev/rork-max-core-ml-guide) も参考にしてください。

### Q3: リアルタイム推論のフレームレートはどのくらい出ますか？

MobileNet V2 の量子化モデルを使った場合、iPhone 14 以降では約 30 FPS、Android のミッドレンジ端末でも 15〜20 FPS のリアルタイム推論が可能です。GPU デリゲートを有効にすることでさらに高速化できます。ただし、React Native のブリッジオーバーヘッドがあるため、ネイティブ実装と比べると 20〜30% 程度遅くなる点は留意してください。

### Q4: カスタムモデル（独自の分類モデル）を使えますか？

はい、TensorFlow / Keras で学習した任意のモデルを TFLite 形式に変換して使用できます。Transfer Learning を使えば、MobileNet V2 をベースに少量のデータ（数百枚程度）でカスタム分類器を作成でき、食品認識、植物図鑑、商品識別などの専用アプリに応用できます。

### Q5: バッテリー消費は問題になりませんか？

推論ごとのバッテリー消費は非常に少なく、1回の画像分類で約 0.001% 程度です。ただし、カメラのリアルタイム推論を長時間続ける場合は、推論間隔を調整する（例: 毎フレームではなく 500ms ごと）、画面がバックグラウンドに入ったら推論を停止するなどの工夫が必要です。
