Rork に「ToDoアプリを作って」と入力してみたら、想像と違うものが出てきた——そういう経験がある方も多いのではないでしょうか。

Rork は自然言語でアプリを作れる強力なツールですが、同じような指示でも書き方の違いで出来上がりが大きく変わります。「Rork が読み取りやすい情報」を意識してプロンプトを書くと、修正の回数が減って完成に近いものが最初から出てきます。

## コツ1: 「誰が使うか」を最初に書く

目的と対象ユーザーを明確にすると、UIの設計方向が決まりやすくなります。

```
❌ 曖昧な例:
「家計管理アプリを作って」

✅ 改善例:
「30代の共働き夫婦が毎日のレシートを撮影して出費を記録できる
 家計管理アプリを作って。シンプルで直感的に使えることを重視してください」
```

「誰が」「何のために」「どんな状況で」使うかが入ると、Rork が作るUIのデザイントーンや操作フローが変わってきます。

## コツ2: 主要な画面を列挙する

「どんな画面があるか」を最初に書くと、構成の意図が伝わります。

```
「以下の画面を含むタスク管理アプリを作ってください:
 1. タスク一覧（完了・未完了でフィルター可能）
 2. タスク追加画面（タイトル・期限・優先度を入力）
 3. タスク詳細画面（編集・削除・完了マークが可能）
 4. 設定画面（通知のオン・オフ）」
```

画面の数が多い場合は、「最小構成でいいので動くものを先に作って」と付け加えると、シンプルなベースから始められます。

## コツ3: デザインの方向性をキーワードで伝える

見た目の好みを一言で伝えると、かなり印象が変わります。

```
デザインキーワードの例:
- 「ミニマルでモノトーン」
- 「明るく親しみやすい。アクセントカラーは緑」
- 「ビジネス向けのプロフェッショナルな印象。ダークモード対応」
- 「子ども向けのカラフルで大きな文字」
- 「Instagram のような写真中心のレイアウト」
```

参考にしたいアプリの名前を挙げると、さらにイメージが伝わりやすくなります。「〇〇（アプリ名）のような雰囲気で」という指定は、Rork が理解できる場合に有効です。

## コツ4: データの構造を具体的に書く

「どんなデータを扱うか」を書くと、データモデルとUIの連携が自然になります。

```
「ユーザーが記録できる内容:
 - 食事名（テキスト）
 - カロリー（数値）
 - 食事の時間（日時）
 - 写真（任意）
 - メモ（任意、テキスト）
 
 一覧表示: 日付別にグループ化して表示する
 集計: 1日の合計カロリーを表示する」
```

この程度の詳細を最初から書いておくと、後から「カロリーも記録できるようにして」「日付でまとめて表示して」という修正指示をしなくて済みます。

## コツ5: 「しないこと」も書く

入れてほしくない機能・デザイン要素も書くと、意図しない方向への暴走を防げます。

```
「注意してほしいこと:
 - ソーシャル機能（シェア・フォロー等）は不要
 - ログイン・会員登録機能は最初は不要
 - 広告表示エリアは含めない
 - 画面遷移は最小限にしてください」
```

「入れてほしいもの」と「入れないでほしいもの」の両方を明示すると、Rork の判断ミスが減ります。

## コツ6: 修正指示は具体的な箇所を指定する

出力されたアプリに修正を加えるとき、「もっとおしゃれに」より「〇〇画面の△△を▲▲に変更してほしい」の方が期待通りになります。

```
❌ 曖昧な修正指示:
「デザインをよくして」

✅ 具体的な修正指示:
「タスク一覧画面のタスクカードの高さを少し大きくして、
 タスク名のフォントサイズを16pxに、テキストカラーを #333333 にしてください」
```

修正指示のたびに全体が変わってしまうことを防ぐためにも、「この部分だけ」という範囲指定が重要です。

## コツ7: 段階的に作る

最初から完全なアプリを作ろうとすると、複雑すぎて意図通りにならないことがあります。

```
第1フェーズ:
「シンプルなメモアプリを作って。
 メモの追加・削除・一覧表示だけで OK」

動作確認後→

第2フェーズ:
「今のメモアプリに、メモに色をつけるラベル機能を追加して。
 ラベルの色は赤・黄・青・緑の4色から選べるようにして」

確認後→

第3フェーズ:
「ラベルでフィルタリングできる検索機能を追加して」
```

機能を少しずつ積み上げていくことで、それぞれの変更が確認しやすくなり、全体として意図通りのアプリに近づいていきます。

## 全部まとめたプロンプトの例

上記のポイントを組み合わせたプロンプト例です。

```
【アプリの目的】
毎日の睡眠時間を記録して可視化するアプリ。
ユーザー像: 睡眠の質を改善したいと思っている20〜40代の社会人。

【主要な画面】
1. ホーム（今日の睡眠記録 + 直近7日間のグラフ）
2. 記録追加（就寝時刻・起床時刻・睡眠の質評価・メモ）
3. 履歴（月別に過去の記録を確認できる）

【デザイン方向性】
落ち着いた夜のイメージ。ダークブルー〜ネイビーをベースにした
ダークテーマで。文字は白・薄いグレーを使用。

【データ構造】
各記録: 就寝時刻・起床時刻・睡眠時間（自動計算）・
       睡眠の質（1〜5のスコア）・メモ（テキスト）

【入れないもの】
ソーシャル機能・ログイン機能・広告枠
```

最初から全部書かなくて大丈夫です。「まず動くものを見てから詳細を決めたい」という場合は、短く書いて出力を確認してから修正指示を出す方が効率的な場合もあります。自分のスタイルを見つけながら試してみてください。
