AI チャットボットはアプリの差別化機能として最も注目されています。しかし「作れる」と「ユーザーが喜ぶ体験を作れる」は別物です。レスポンスを待たせる UX は離脱率を高め、アプリの評価を下げます。

本記事では **Rork Max** と **Gemini API のストリーミング機能**を組み合わせ、トークンが生成されるたびにリアルタイムで表示されるチャットボットアプリを構築します。ChatGPT アプリのような滑らかな UX を、個人開発でも実現できます。

## 開発環境と前提条件

- Rork Max アカウント（Claude Opus 4.6 搭載）
- Google AI Studio の Gemini API キー（無料で取得可能: https://aistudio.google.com）
- Expo SDK 53 以上の知識（基礎レベルで OK）

### プロジェクトの初期設定

Rork Max のプロンプトエリアに以下を入力してプロジェクトを生成します。

```
Gemini API ストリーミング対応のチャットボットアプリを作成してください。
要件:
- メッセージバブル（ユーザー/AI で色分け）
- タイピングインジケーター（...アニメーション）
- マークダウン対応
- 会話履歴（最大 20 件保持）
- エラートースト表示
```

Rork Max が生成したベースに、以下の詳細実装を追加していきます。

## Gemini API ストリーミング統合の核心部分

### API クライアントの実装

```typescript
// services/gemini.ts
import { GoogleGenerativeAI, HarmCategory, HarmBlockThreshold } from "@google/generative-ai";

const API_KEY = process.env.EXPO_PUBLIC_GEMINI_API_KEY!;

// 安全設定: 本番アプリ向けに適切なブロックレベルを設定
const SAFETY_SETTINGS = [
  { category: HarmCategory.HARM_CATEGORY_HARASSMENT, threshold: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE },
  { category: HarmCategory.HARM_CATEGORY_HATE_SPEECH, threshold: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE },
  { category: HarmCategory.HARM_CATEGORY_SEXUALLY_EXPLICIT, threshold: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE },
  { category: HarmCategory.HARM_CATEGORY_DANGEROUS_CONTENT, threshold: HarmBlockThreshold.BLOCK_MEDIUM_AND_ABOVE },
];

export interface ChatMessage {
  id: string;
  role: "user" | "model";
  content: string;
  timestamp: Date;
  isStreaming?: boolean;  // ストリーミング中フラグ
}

/**
 * Gemini API とのチャットセッションを管理するクライアント
 */
export class GeminiChatClient {
  private genAI: GoogleGenerativeAI;
  private model: ReturnType<GoogleGenerativeAI["getGenerativeModel"]>;

  constructor(systemInstruction?: string) {
    this.genAI = new GoogleGenerativeAI(API_KEY);
    this.model = this.genAI.getGenerativeModel({
      model: "gemini-2.5-pro",
      systemInstruction: systemInstruction ?? "あなたは親切で知識豊富な AI アシスタントです。",
      safetySettings: SAFETY_SETTINGS,
      generationConfig: {
        temperature: 0.7,      // 創造性と一貫性のバランス
        topP: 0.95,
        maxOutputTokens: 2048,
      },
    });
  }

  /**
   * 会話履歴を Gemini API フォーマットに変換する
   */
  private toGeminiHistory(messages: ChatMessage[]) {
    return messages
      .filter((m) => !m.isStreaming)  // 未完了のストリーミングメッセージは除外
      .map((m) => ({
        role: m.role,
        parts: [{ text: m.content }],
      }));
  }

  /**
   * ストリーミングでメッセージを送信し、チャンクごとにコールバックを呼び出す
   *
   * @param userMessage ユーザーの入力テキスト
   * @param history 過去の会話履歴（最新 20 件）
   * @param onChunk トークン受信時のコールバック
   * @param onComplete 生成完了時のコールバック
   * @param onError エラー発生時のコールバック
   */
  async sendMessageStreaming(
    userMessage: string,
    history: ChatMessage[],
    onChunk: (text: string) => void,
    onComplete: (fullText: string) => void,
    onError: (error: Error) => void,
  ): Promise<void> {
    try {
      const chat = this.model.startChat({
        history: this.toGeminiHistory(history),
      });

      const result = await chat.sendMessageStream(userMessage);
      let fullText = "";

      for await (const chunk of result.stream) {
        const chunkText = chunk.text();
        fullText += chunkText;
        onChunk(chunkText);  // UI にリアルタイム反映
      }

      onComplete(fullText);
    } catch (error) {
      onError(error instanceof Error ? error : new Error(String(error)));
    }
  }
}
```

### チャット状態管理 Hook

```typescript
// hooks/useChat.ts
import { useState, useCallback, useRef } from "react";
import { GeminiChatClient, ChatMessage } from "@/services/gemini";
import { nanoid } from "nanoid";

const MAX_HISTORY = 20;  // 保持する会話件数の上限

export function useChat(systemInstruction?: string) {
  const [messages, setMessages] = useState<ChatMessage[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const clientRef = useRef(new GeminiChatClient(systemInstruction));

  const sendMessage = useCallback(async (text: string) => {
    if (!text.trim() || isLoading) return;

    // ユーザーメッセージを即座に表示
    const userMessage: ChatMessage = {
      id: nanoid(),
      role: "user",
      content: text.trim(),
      timestamp: new Date(),
    };

    // AI の応答プレースホルダー（ストリーミング中）
    const aiMessageId = nanoid();
    const aiPlaceholder: ChatMessage = {
      id: aiMessageId,
      role: "model",
      content: "",
      timestamp: new Date(),
      isStreaming: true,
    };

    setMessages((prev) => {
      const updated = [...prev, userMessage, aiPlaceholder];
      // 上限を超えた場合は古いものを削除（ペア単位で削除）
      return updated.length > MAX_HISTORY * 2
        ? updated.slice(updated.length - MAX_HISTORY * 2)
        : updated;
    });

    setIsLoading(true);

    // 直前の会話履歴を渡す（ユーザーの新しいメッセージと AI プレースホルダーは除く）
    const historyForApi = messages.slice(-MAX_HISTORY * 2);

    await clientRef.current.sendMessageStreaming(
      text.trim(),
      historyForApi,
      // onChunk: 受信したテキストを既存のプレースホルダーに追記
      (chunk) => {
        setMessages((prev) =>
          prev.map((m) =>
            m.id === aiMessageId ? { ...m, content: m.content + chunk } : m
          )
        );
      },
      // onComplete: ストリーミング完了フラグを解除
      (_fullText) => {
        setMessages((prev) =>
          prev.map((m) =>
            m.id === aiMessageId ? { ...m, isStreaming: false } : m
          )
        );
        setIsLoading(false);
      },
      // onError: エラー処理
      (error) => {
        setMessages((prev) =>
          prev.map((m) =>
            m.id === aiMessageId
              ? { ...m, content: "⚠️ エラーが発生しました。もう一度お試しください。", isStreaming: false }
              : m
          )
        );
        setIsLoading(false);
        console.error("Gemini API error:", error);
      }
    );
  }, [messages, isLoading]);

  const clearHistory = useCallback(() => {
    setMessages([]);
    clientRef.current = new GeminiChatClient(systemInstruction);
  }, [systemInstruction]);

  return { messages, isLoading, sendMessage, clearHistory };
}
```

## チャット UI の実装

### メインチャット画面

```typescript
// screens/ChatScreen.tsx
import React, { useRef, useCallback } from "react";
import {
  View, FlatList, TextInput, TouchableOpacity,
  KeyboardAvoidingView, Platform, StyleSheet, ActivityIndicator,
} from "react-native";
import { ChatBubble } from "@/components/ChatBubble";
import { TypingIndicator } from "@/components/TypingIndicator";
import { useChat } from "@/hooks/useChat";

export default function ChatScreen() {
  const { messages, isLoading, sendMessage, clearHistory } = useChat(
    "あなたは Rork Max の専門家です。モバイルアプリ開発の質問に詳しく答えてください。"
  );
  const [inputText, setInputText] = React.useState("");
  const flatListRef = useRef<FlatList>(null);

  const handleSend = useCallback(() => {
    if (!inputText.trim()) return;
    sendMessage(inputText);
    setInputText("");
    // 送信後に最下部へスクロール
    setTimeout(() => flatListRef.current?.scrollToEnd({ animated: true }), 100);
  }, [inputText, sendMessage]);

  return (
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === "ios" ? "padding" : "height"}
      keyboardVerticalOffset={90}
    >
      <FlatList
        ref={flatListRef}
        data={messages}
        keyExtractor={(item) => item.id}
        renderItem={({ item }) => <ChatBubble message={item} />}
        ListFooterComponent={isLoading && messages[messages.length - 1]?.isStreaming === false
          ? <TypingIndicator /> : null}
        onContentSizeChange={() => flatListRef.current?.scrollToEnd({ animated: true })}
        contentContainerStyle={styles.messageList}
      />

      <View style={styles.inputContainer}>
        <TextInput
          style={styles.input}
          value={inputText}
          onChangeText={setInputText}
          placeholder="メッセージを入力..."
          multiline
          maxLength={2000}
          onSubmitEditing={handleSend}
        />
        <TouchableOpacity
          style={[styles.sendButton, (!inputText.trim() || isLoading) && styles.sendButtonDisabled]}
          onPress={handleSend}
          disabled={!inputText.trim() || isLoading}
        >
          {isLoading
            ? <ActivityIndicator color="#fff" size="small" />
            : <Text style={styles.sendIcon}>↑</Text>}
        </TouchableOpacity>
      </View>
    </KeyboardAvoidingView>
  );
}
```

## パフォーマンス最適化とベストプラクティス

### ストリーミング中のレンダリング最適化

ストリーミング中は `setMessages` が頻繁に呼ばれるため、不要な再レンダリングを防ぐことが重要です。

```typescript
// components/ChatBubble.tsx — React.memo で不要な再描画を防止
import React, { memo } from "react";
import { View, StyleSheet } from "react-native";
import Markdown from "react-native-markdown-display";

interface Props {
  message: ChatMessage;
}

export const ChatBubble = memo(({ message }: Props) => {
  const isUser = message.role === "user";

  return (
    <View style={[styles.bubble, isUser ? styles.userBubble : styles.aiBubble]}>
      {/* マークダウン対応でコードブロック・リストも綺麗に表示 */}
      <Markdown style={isUser ? markdownUserStyle : markdownAiStyle}>
        {message.content || (message.isStreaming ? "▊" : "")}
        {/* ストリーミング中はカーソルを表示 */}
      </Markdown>
    </View>
  );
}, (prev, next) => {
  // カスタム比較: content と isStreaming が変わった時だけ再描画
  return prev.message.content === next.message.content &&
         prev.message.isStreaming === next.message.isStreaming;
});
```

## よくある質問（FAQ）

**Q1. Gemini API のレート制限に引っかかった場合の対処法は？**

Gemini API の無料枠は 1 分あたり 15 リクエスト（gemini-2.5-pro の場合）です。本番アプリでは以下の対策を組み合わせてください。①ユーザーごとの送信間隔を最低 2 秒に制限する（送信ボタンをデバウンス）、②API キーを有料プランにアップグレードする、③エラーコード 429 を検知したら「しばらくしてから再試行してください」とトーストを表示します。

**Q2. 会話履歴が長くなるとコストが増える？**

はい、Gemini API はリクエストごとに会話履歴全体をトークンとして送信するため、履歴が長いほどコストが増加します。本記事のコードでは `MAX_HISTORY = 20` を設定していますが、コスト優先なら 10〜15 件に減らすのが現実的です。また `gemini-2.5-flash` を使用するとコストを 1/5 程度に抑えられます。

**Q3. ストリーミング中にユーザーが操作を中断できるようにしたいのですが？**

`AbortController` を `sendMessageStream` に渡すことで中断できます。`useChat` hook に `abortRef = useRef<AbortController>()` を追加し、送信時に `abortRef.current = new AbortController()` を生成、中断ボタンを押した際に `abortRef.current.abort()` を呼び出します。Gemini SDK v0.5 以降でサポートされています。

## まとめ

Rork Max × Gemini API のストリーミング統合で実現できることをまとめます。

- **リアルタイム UX**：トークン生成と同時に文字が表示され、ユーザーの離脱を防ぐ
- **会話継続性**：最大 20 件の履歴を管理し文脈を保った対話を実現
- **パフォーマンス**：`React.memo` + カスタム比較で頻繁な更新時の不要描画を排除
- **堅牢性**：エラーハンドリングと安全設定でアプリとユーザーを保護
