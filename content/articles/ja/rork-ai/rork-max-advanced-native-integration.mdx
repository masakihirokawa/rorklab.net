## CoreData：永続化とオフラインファースト設計

**基本的な利用パターン：**

```swift

@Entity

    @NSManaged var id: UUID
    @NSManaged var title: String
    @NSManaged var description: String
    @NSManaged var isCompleted: Bool

    @NSManaged var dueDate: Date?

// SwiftUI View で使用
@Environment(\.managedObjectContext) var viewContext

var body: some View {
    NavigationStack {
        List {
            ForEach(tasks, id: \.id) { task in
                TaskRow(task: task)
            }
            .onDelete(perform: deleteTask)
```

## Vision API：画像認識・機械学習の統合

### Vision フレームワークの可能性

Vision APIは、オンデバイスで実行される機械学習モデルを活用し、画像認識・テキスト認出・顔検出など多くの機能を提供します。

**代表的なVision機能：**

| 機能 | 用途 | 実用例 |
|---|---|---|
| 顔認識 | 顔領域の検出 | フェイスフィルター、認証 |
| テキスト認識 | OCR処理 | 名刺スキャン、書類デジタル化 |
| バーコード認識 | QRコード・バーコード読み込み | 在庫管理、チケット確認 |
| 被写体認識 | 画像内の物体分類 | 商品認識、自動タグ付け |
| 顔のランドマーク | 顔のパーツ位置検出 | 美顔フィルター、表情認識 |

### Rork Max での Vision API 統合

**ユースケース：名刺読み込みアプリ**

```swift
// Vision + SwiftUI の統合
import Vision

struct BusinessCardScannerView: View {
    @State private var capturedImage: UIImage?
    @State private var recognizedText: String = ""

    var body: some View {
        VStack {
            if let image = capturedImage {
                Image(uiImage: image)
                    .resizable()
                    .scaledToFit()
                Text(recognizedText)
            }

            Button("Scan Business Card") {
                performOCR()
            }
        }
    }

    func performOCR() {
        guard let image = capturedImage,
              let cgImage = image.cgImage else { return }

        let request = VNRecognizeTextRequest { request, error in
            let observations = request.results as? [VNRecognizedTextObservation] ?? []
            self.recognizedText = observations
                .compactMap { $0.topCandidates(1).first?.string }
                .joined(separator: "\n")
        }

        let handler = VNImageRequestHandler(cgImage: cgImage)
        try? handler.perform([request])
    }
}
```

### SwiftUI + ARKit 統合パターン

```swift
import ARKit
import RealityKit

struct ARViewContainer: UIViewControllerRepresentable {
    func makeUIViewController(context: Context) -> ARViewController {
        return ARViewController()
    }

    func updateUIViewController(
        _ uiViewController: ARViewController,
        context: Context
    ) {}
}

class ARViewController: UIViewController, ARViewDelegate {
    var arView: ARView!

    override func viewDidLoad() {
        super.viewDidLoad()

        // AR セッション初期化
        arView = ARView(frame: view.bounds)
        view.addSubview(arView)

        // 3Dモデル読み込みと配置
        loadAndPlaceModel()
    }

    func loadAndPlaceModel() {
        // RealityKit でモデル読み込み
        guard let modelAnchor = try? ModelEntity
            .loadModel(named: "furniture_model") else { return }

        var transform = modelAnchor.transform
        transform.translation.z = -0.5
        modelAnchor.move(to: transform, relativeTo: arView.scene)
    }
}
```

## よくある質問と回答

**Q: CoreData とiCloud Syncの関係は？**

A: `NSPersistentCloudKitContainer` を使用すれば、CoreDataのデータを自動的にiCloud経由で複数デバイスに同期できます。Rork Maxでこの設定を指示すれば、自動生成されます。

**Q: Vision APIはオフラインで動作するのか？**

A: はい。Vision フレームワークはオンデバイス処理のため、インターネット接続不要です。これはサーバー依存のAIサービスとの大きな違いです。

**Q: ARKit はiPadでも動作するのか？**

A: はい。ARKit は、iPad Pro（A12Bionic以降）でも フル機能で動作します。iPhone との同じコードベースで対応可能です。

**Q: メモリ使用量が多いのでは？**

A: ネイティブAPIの使用により、正確に最適化された処理が実行されるため、実は思ったほどメモリを消費しません。Rork Maxで生成されるコードは、Apple公式のベストプラクティスに従っています。

**Q: Rork Max で生成したコード後、手作業のカスタマイズはどのくらい必要？**

A: 基本機能は完全に自動生成されます。その後の調整（色・フォント・レイアウト微調整、業務ロジック）は通常、生成後の工数の 20-30% 程度で済みます。

**Q: CoreDataの複雑な関連性（リレーション）はRork Maxで対応できる？**

A: シンプルな1対多、多対多の関係は自動生成可能。複雑な相互参照や制約が必要な場合は、生成後の手作業カスタマイズで対応します。
