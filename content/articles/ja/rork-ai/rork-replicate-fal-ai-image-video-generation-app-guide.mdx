## Replicate vs fal.ai — 選択基準と特徴比較

AI 生成 API の代表的な選択肢として Replicate と fal.ai があります。どちらを選ぶかはユースケースによって異なります。

### Replicate の特徴

Replicate はモデルの豊富さが強みです。Flux Dev、Flux Schnell、SDXL、Stable Video Diffusion など数千のモデルが揃い、マイナーなニッチモデルも利用できます。課金はモデルの実行時間（秒・GPU 種別）に応じた従量課金で、安価なモデルなら 1 枚あたり $0.001〜$0.005 程度です。

- **強み**: モデル数が多い、カスタムモデルのデプロイが可能、ドキュメントが充実
- **弱み**: レイテンシがやや高い（コールドスタートで数十秒かかる場合がある）、WebSocket 非対応

### fal.ai の特徴

fal.ai はリアルタイム生成に特化した設計が特徴です。WebSocket / Server-Sent Events（SSE）によるストリーミングに対応しており、生成プログレスをリアルタイムに UI へ反映できます。Flux Schnell の場合、ウォームサーバーなら 1〜2 秒で結果が返ります。

- **強み**: 低レイテンシ、リアルタイムストリーミング対応、React Native SDK 提供
- **弱み**: Replicate と比べてモデル種類は少ない、一部のマイナーモデルは非対応

### 選択の指針

- **リアルタイム性が重要**（プロンプト入力中にプレビューを表示したい）→ fal.ai
- **多様なモデルを使い分けたい・カスタムモデルを使いたい** → Replicate
- **両方の長所を活かしたい** → 画像生成は fal.ai、特殊モデルは Replicate と使い分けるハイブリッド構成

本記事では **fal.ai をメイン、Replicate をサブ**とするハイブリッド構成を実装例として解説します。

## 画像生成機能の実装（Flux Schnell）

### Supabase Edge Function の作成

生成リクエストを受け取り、fal.ai または Replicate に転送するバックエンド関数を作成します。

```typescript
// supabase/functions/generate-image/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const FAL_API_KEY = Deno.env.get("FAL_API_KEY")!;
const SUPABASE_URL = Deno.env.get("SUPABASE_URL")!;
const SUPABASE_SERVICE_KEY = Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!;

serve(async (req) => {
  const { prompt, negativePrompt, style, aspectRatio, userId } = await req.json();

  // クレジット確認（後述の課金実装を参照）
  const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_KEY);
  const { data: user } = await supabase
    .from("users")
    .select("credits")
    .eq("id", userId)
    .single();

  if (!user || user.credits < 1) {
    return new Response(
      JSON.stringify({ error: "クレジットが不足しています" }),
      { status: 402 }
    );
  }

  // スタイルプリセットをプロンプトに統合
  const styleMap: Record<string, string> = {
    anime: "anime style, highly detailed, vibrant colors, Studio Ghibli inspired",
    photo: "photorealistic, professional photography, 8k, sharp focus",
    oil: "oil painting, impressionist style, textured brush strokes",
    pixel: "pixel art, 16-bit style, retro game aesthetic",
  };

  const enhancedPrompt = `${prompt}, ${styleMap[style] || ""}`.trim();

  // fal.ai へリクエスト（Flux Schnell モデル）
  const falResponse = await fetch(
    "https://fal.run/fal-ai/flux/schnell",
    {
      method: "POST",
      headers: {
        "Authorization": `Key ${FAL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        prompt: enhancedPrompt,
        negative_prompt: negativePrompt || "blur, low quality, watermark",
        image_size: aspectRatioToSize(aspectRatio),
        num_inference_steps: 4,
        seed: Math.floor(Math.random() * 1000000),
      }),
    }
  );

  const result = await falResponse.json();

  if (!result.images?.[0]?.url) {
    return new Response(
      JSON.stringify({ error: "生成に失敗しました" }),
      { status: 500 }
    );
  }

  // クレジット消費（1クレジット = 1枚）
  await supabase
    .from("users")
    .update({ credits: user.credits - 1 })
    .eq("id", userId);

  // 生成履歴を保存
  const imageUrl = result.images[0].url;
  await supabase.from("generations").insert({
    user_id: userId,
    prompt: enhancedPrompt,
    image_url: imageUrl,
    model: "flux-schnell",
    aspect_ratio: aspectRatio,
    credits_used: 1,
  });

  return new Response(
    JSON.stringify({ imageUrl, creditsRemaining: user.credits - 1 }),
    { headers: { "Content-Type": "application/json" } }
  );
});

function aspectRatioToSize(ratio: string): string {
  const sizeMap: Record<string, string> = {
    "1:1": "square",
    "16:9": "landscape_16_9",
    "9:16": "portrait_16_9",
  };
  return sizeMap[ratio] || "square";
}
```

### React Native クライアント側の実装

```typescript
// src/hooks/useImageGeneration.ts
import { useState, useRef } from "react";
import { supabase } from "../lib/supabase";
import { useAuthStore } from "../stores/authStore";
import { useCreditsStore } from "../stores/creditsStore";

export interface GenerationParams {
  prompt: string;
  negativePrompt?: string;
  style: "anime" | "photo" | "oil" | "pixel";
  aspectRatio: "1:1" | "16:9" | "9:16";
}

export function useImageGeneration() {
  const [isGenerating, setIsGenerating] = useState(false);
  const [progress, setProgress] = useState(0);
  const [generatedImageUrl, setGeneratedImageUrl] = useState<string | null>(null);
  const [error, setError] = useState<string | null>(null);
  const abortRef = useRef(false);

  const { user } = useAuthStore();
  const { credits, updateCredits } = useCreditsStore();

  const generate = async (params: GenerationParams) => {
    if (!user) return;
    if (credits < 1) {
      setError("クレジットが不足しています。チャージしてください。");
      return;
    }

    setIsGenerating(true);
    setProgress(10);
    setError(null);
    abortRef.current = false;

    // プログレスアニメーション
    const progressInterval = setInterval(() => {
      setProgress((prev) => {
        if (abortRef.current || prev >= 90) {
          clearInterval(progressInterval);
          return prev;
        }
        return prev + Math.random() * 15;
      });
    }, 400);

    try {
      const { data, error: fnError } = await supabase.functions.invoke(
        "generate-image",
        {
          body: {
            prompt: params.prompt,
            negativePrompt: params.negativePrompt,
            style: params.style,
            aspectRatio: params.aspectRatio,
            userId: user.id,
          },
        }
      );

      if (abortRef.current) return;

      if (fnError || data?.error) {
        throw new Error(data?.error || "生成に失敗しました");
      }

      setProgress(100);
      setGeneratedImageUrl(data.imageUrl);
      updateCredits(data.creditsRemaining);
    } catch (err) {
      if (!abortRef.current) {
        setError(err instanceof Error ? err.message : "予期しないエラーが発生しました");
      }
    } finally {
      clearInterval(progressInterval);
      setIsGenerating(false);
    }
  };

  const cancel = () => {
    abortRef.current = true;
    setIsGenerating(false);
    setProgress(0);
  };

  return { generate, cancel, isGenerating, progress, generatedImageUrl, error };
}
```

## 動画生成機能の実装（Wan 2.1）

動画生成は処理時間が長い（30〜120 秒）ため、非同期ジョブ方式で実装します。fal.ai のキュー API を使うと、結果をポーリングまたは Webhook で受け取れます。

### 動画生成 Edge Function

```typescript
// supabase/functions/generate-video/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const FAL_API_KEY = Deno.env.get("FAL_API_KEY")!;

serve(async (req) => {
  const { prompt, imageUrl, userId } = await req.json();
  const supabase = createClient(
    Deno.env.get("SUPABASE_URL")!,
    Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
  );

  // 動画生成は5クレジット消費
  const { data: user } = await supabase
    .from("users")
    .select("credits")
    .eq("id", userId)
    .single();

  if (!user || user.credits < 5) {
    return new Response(
      JSON.stringify({ error: "動画生成には5クレジット必要です" }),
      { status: 402 }
    );
  }

  // fal.ai キューに投入（Wan 2.1 モデル）
  const queueResponse = await fetch(
    "https://queue.fal.run/fal-ai/wan-t2v",
    {
      method: "POST",
      headers: {
        "Authorization": `Key ${FAL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        prompt,
        num_frames: 81,       // 約3秒の動画
        frame_rate: 24,
        resolution: "480p",
        num_inference_steps: 30,
      }),
    }
  );

  const { request_id } = await queueResponse.json();

  // ジョブ ID を DB に保存（クライアントがポーリングするために使用）
  await supabase.from("video_jobs").insert({
    user_id: userId,
    request_id,
    prompt,
    status: "pending",
    credits_reserved: 5,
  });

  // クレジットを先に引き落とし（失敗時は補填処理が別途必要）
  await supabase
    .from("users")
    .update({ credits: user.credits - 5 })
    .eq("id", userId);

  return new Response(
    JSON.stringify({ requestId: request_id }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### ポーリングによる進捗確認

```typescript
// src/hooks/useVideoGeneration.ts
export function useVideoGeneration() {
  const [jobStatus, setJobStatus] = useState<
    "idle" | "pending" | "processing" | "completed" | "failed"
  >("idle");
  const [videoUrl, setVideoUrl] = useState<string | null>(null);
  const [progressPercent, setProgressPercent] = useState(0);
  const pollingRef = useRef<ReturnType<typeof setInterval> | null>(null);

  const pollJobStatus = async (requestId: string) => {
    pollingRef.current = setInterval(async () => {
      const { data } = await supabase.functions.invoke("check-video-job", {
        body: { requestId },
      });

      if (data?.status === "completed") {
        clearInterval(pollingRef.current!);
        setVideoUrl(data.videoUrl);
        setJobStatus("completed");
        setProgressPercent(100);
      } else if (data?.status === "failed") {
        clearInterval(pollingRef.current!);
        setJobStatus("failed");
      } else {
        // ログから進捗を推定（0〜90%）
        setProgressPercent((prev) => Math.min(prev + 8, 90));
      }
    }, 5000); // 5秒ごとにポーリング
  };

  return { generateVideo, jobStatus, videoUrl, progressPercent, pollJobStatus };
}
```

## 生成コンテンツの管理・保存・共有

### Supabase Storage への自動保存

fal.ai が返す画像 URL は一時的なもの（数時間で失効する場合がある）です。本番環境では、生成直後に自分の Supabase Storage へ転送してURLを永続化します。

```typescript
// src/lib/saveGeneratedImage.ts
import * as FileSystem from "expo-file-system";
import { decode } from "base64-arraybuffer";
import { supabase } from "./supabase";

export async function saveToSupabaseStorage(
  temporaryUrl: string,
  userId: string
): Promise<string> {
  // 一時 URL から画像をダウンロード
  const downloadResult = await FileSystem.downloadAsync(
    temporaryUrl,
    FileSystem.cacheDirectory + "temp_image.jpg"
  );

  // Base64 に変換
  const base64 = await FileSystem.readAsStringAsync(
    downloadResult.uri,
    { encoding: FileSystem.EncodingType.Base64 }
  );

  const fileName = `${userId}/${Date.now()}.jpg`;
  const { data, error } = await supabase.storage
    .from("generations")
    .upload(fileName, decode(base64), {
      contentType: "image/jpeg",
      upsert: false,
    });

  if (error) throw error;

  // 公開URLを取得
  const { data: { publicUrl } } = supabase.storage
    .from("generations")
    .getPublicUrl(fileName);

  return publicUrl;
}
```

### 端末カメラロールへの保存

```typescript
import * as MediaLibrary from "expo-media-library";
import * as FileSystem from "expo-file-system";

export async function saveToDeviceLibrary(imageUrl: string): Promise<void> {
  const { status } = await MediaLibrary.requestPermissionsAsync();
  if (status !== "granted") {
    throw new Error("カメラロールへのアクセス権限が必要です");
  }

  const localUri = FileSystem.cacheDirectory + "download.jpg";
  await FileSystem.downloadAsync(imageUrl, localUri);

  const asset = await MediaLibrary.createAssetAsync(localUri);
  await MediaLibrary.createAlbumAsync("AI Generated", asset, false);
}
```

## クレジット制課金モデルの設計と実装

### クレジットパックの設計

AI 生成アプリでは、サブスクリプションよりもクレジット制（Pay-as-you-go）のほうがユーザーに受け入れられやすい傾向があります。

推奨するクレジットパック構成例:

- **スターターパック（無料）**: 10 クレジット（新規登録ボーナス）
- **ライトパック**: 50 クレジット — ¥120（1クレジット = ¥2.4）
- **スタンダードパック**: 200 クレジット — ¥380（1クレジット = ¥1.9）
- **プロパック**: 500 クレジット — ¥780（1クレジット = ¥1.56）

クレジット消費レート例:
- 画像生成（Flux Schnell）: 1 クレジット
- 画像生成（SDXL 高解像度）: 2 クレジット
- 動画生成（Wan 2.1 短編）: 5 クレジット
- 動画生成（Wan 2.1 長編）: 10 クレジット

### Stripe Checkout による課金実装

```typescript
// supabase/functions/create-credit-checkout/index.ts
import Stripe from "https://esm.sh/stripe@14";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!, {
  apiVersion: "2023-10-16",
});

const CREDIT_PACKS = {
  lite: { credits: 50, price_jpy: 120 },
  standard: { credits: 200, price_jpy: 380 },
  pro: { credits: 500, price_jpy: 780 },
```

### Webhook でクレジット付与

```typescript
// supabase/functions/stripe-webhook/index.ts
serve(async (req) => {
  const signature = req.headers.get("stripe-signature")!;
  const body = await req.text();

  const event = stripe.webhooks.constructEvent(
    body,
    signature,
    Deno.env.get("STRIPE_WEBHOOK_SECRET")!
  );

  if (event.type === "checkout.session.completed") {
    const session = event.data.object as Stripe.Checkout.Session;
    const { user_id, credits } = session.metadata!;

    // クレジット付与
    const supabase = createClient(
      Deno.env.get("SUPABASE_URL")!,
      Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
    );

    const { data: user } = await supabase
      .from("users")
      .select("credits")
      .eq("id", user_id)
      .single();

    await supabase
      .from("users")
      .update({ credits: (user?.credits ?? 0) + parseInt(credits) })
      .eq("id", user_id);

    // 購入履歴を記録
    await supabase.from("credit_purchases").insert({
      user_id,
      credits_added: parseInt(credits),
      stripe_session_id: session.id,
      amount_jpy: session.amount_total,
    });
  }

  return new Response(JSON.stringify({ received: true }));
});
```

## App Store 審査対策とコンテンツモデレーション

### Apple のガイドライン要件

AI 生成コンテンツに関して Apple は明確なガイドラインを設けています。2026 年現在、特に以下の点に注意が必要です。

- **成人向けコンテンツ（4+〜17+）**: 生成画像がセンシティブなコンテンツを含む可能性がある場合、App Store Connect の「コンテンツレーティング」を適切に設定する
- **AI 開示義務**: 生成 AI を使用しているアプリは「このコンテンツは AI により生成されます」という明示が必要
- **リアルな人物の生成禁止**: 実在する人物を複製するような用途は審査拒否の原因になる

### 実装すべきモデレーション層

```typescript
// src/lib/contentModeration.ts
const BANNED_KEYWORDS = [

export function sanitizePrompt(prompt: string): {
```

## パフォーマンス最適化とコスト管理

### 画像のキャッシュ戦略

同一プロンプト・同一スタイルの組み合わせは Seed を固定してキャッシュを活用することでコストを削減できます。

```typescript
// src/lib/generationCache.ts
import AsyncStorage from "@react-native-async-storage/async-storage";

const CACHE_PREFIX = "gen_cache_";
const CACHE_TTL_MS = 24 * 60 * 60 * 1000; // 24時間

export async function getCachedGeneration(
  prompt: string,
  style: string,
  aspectRatio: string
): Promise<string | null> {
  const key = `${CACHE_PREFIX}${btoa(prompt + style + aspectRatio)}`;
  const cached = await AsyncStorage.getItem(key);

  if (!cached) return null;

  const { url, timestamp } = JSON.parse(cached);
  if (Date.now() - timestamp > CACHE_TTL_MS) {
    await AsyncStorage.removeItem(key);
    return null;
  }

  return url;
}

export async function setCachedGeneration(
  prompt: string, style: string, aspectRatio: string, url: string
): Promise<void> {
  const key = `${CACHE_PREFIX}${btoa(prompt + style + aspectRatio)}`;
  await AsyncStorage.setItem(key, JSON.stringify({ url, timestamp: Date.now() }));
}
```

### モデル選択によるコスト最適化

- **プレビュー・素案確認**: Flux Schnell — 約 $0.001/枚
- **高品質な最終出力**: Flux Dev — 約 $0.025/枚
- **アーティスティック表現**: SDXL Turbo — 約 $0.003/枚
- **短編動画（3秒）**: Wan 2.1 T2V — 約 $0.05/本

ユーザーの生成フローを「プレビュー（低コスト）→ 本番生成（高品質）」の 2 段階にすることで、品質を維持しながらコストを削減できます。

## FAQ

**Q: fal.ai の API 利用に月額固定費はかかりますか？**

A: いいえ、fal.ai は完全な従量課金制です。生成リクエスト数に応じてのみ課金されます。無料枠として月$5 相当のクレジットが付与されるため（2026 年 4 月時点）、開発・テスト段階ではほぼ無料で利用できます。本番環境では使用量に応じてクレジットをチャージします。

**Q: Rork のプロジェクトに Supabase Edge Functions を追加する際、どのように指示すればよいですか？**

A: Rork に「Supabase Edge Functions を使って、画像生成リクエストを受け取り fal.ai API に転送するサーバーサイド関数を作ってください」と指示します。Rork Max はネイティブコード生成に対応しているため、Supabase CLI のセットアップ手順まで含めた実装コードを出力してくれます。Edge Function のコードを手動で修正したい場合は、本記事のコードサンプルをそのまま貼り付けて「この内容で Edge Function を作成してください」と伝える方法も効果的です。

**Q: App Store 審査で生成 AI アプリが拒否されやすいパターンを教えてください。**

A: 主な拒否理由は以下の 3 つです。①モデレーション不足：不適切なコンテンツが生成できる状態でリリースしてしまう（対策: Safety Checker + 禁止ワードフィルター）。②AI 開示の不足：アプリ内でコンテンツが AI 生成であることを明示していない（対策: 生成画像に「AI Generated」バッジを表示）。③プライバシーポリシーの不備：ユーザーのプロンプトデータの扱いが不明確（対策: プロンプトデータのサーバー保存有無、学習利用の有無を明記する）。これら 3 点を事前に対処しておくと審査通過率が大幅に向上します。

**Q: クレジット不正消費（API の直接呼び出し）を防ぐ方法は？**

A: クライアントから直接 fal.ai / Replicate を呼び出さず、必ず自社の Supabase Edge Function を経由させることが基本です。Edge Function 側でユーザーの JWT を検証し（`supabase.auth.getUser(token)` で確認）、認証済みユーザーのみリクエストを処理します。さらにレート制限（1 ユーザーあたり 1 分間 3 リクエストまで等）を実装することで、クレジットの大量不正消費を防げます。
