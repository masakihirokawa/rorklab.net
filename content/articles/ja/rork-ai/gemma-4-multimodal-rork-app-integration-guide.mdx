## Gemma 4 マルチモーダル API がモバイルアプリ開発を変える

2026年3月31日、Google DeepMind は Gemma 4 を正式リリースしました。Apache 2.0 ライセンスの完全オープンソースモデルでありながら、Arena AI で世界3位のオープンモデルにランクインするほどの性能を誇ります。

Gemma 4 の最大の特徴は**マルチモーダル対応**です。テキストだけでなく画像の理解、エッジモデルでは音声処理にも対応しています。さらに **256K トークン**のロングコンテキストにより、書籍一冊分に相当するテキストを一度に処理できます。

Rork でアプリを開発しているなら、この強力なモデルをクラウド API 経由で統合することで、以下のような高度な AI 機能をアプリに追加できます。

- ユーザーがアップロードした画像を AI が分析して説明を返す
- 長文ドキュメントを一括で要約・翻訳する
- 音声入力をリアルタイムでテキスト化し、AI が応答する

この記事では、Gemma 4 の API を Rork アプリに統合する具体的な手順を、Vertex AI を利用したアプローチで解説します。

## Gemma 4 の4モデルを理解する — どれを選ぶべきか

Gemma 4 には用途に応じた4つのバリアントが用意されています。

- **Gemma 4 E2B**: 約1.2Bパラメータの超軽量モデル。スマートフォン上での推論に最適化されており、テキスト+画像+音声のマルチモーダル入力に対応
- **Gemma 4 E4B**: 約4Bパラメータのエッジモデル。E2Bより高い精度を維持しつつ、タブレットやハイエンドスマートフォンで動作。音声対応
- **Gemma 4 26B MoE**: 26Bパラメータの Mixture-of-Experts モデル。推論時に一部のエキスパートのみがアクティブになるため、パラメータ数の割に高速。クラウドデプロイ向け
- **Gemma 4 31B Dense**: 31Bパラメータのフルモデル。Arena AI で世界3位の実力を持ち、Gemini 3 の技術をベースに構築。最高精度が必要な場面で使用

Rork アプリでクラウド API 経由で利用する場合は、**26B MoE** または **31B Dense** が推奨です。API コスト効率を重視するなら MoE、精度最優先なら Dense を選びましょう。

## Vertex AI で Gemma 4 API をセットアップする

Gemma 4 をクラウド API として利用する最も簡単な方法は、Google Cloud の Vertex AI を使うことです。以下の手順でセットアップします。

### Google Cloud プロジェクトの準備

```bash
# Google Cloud CLI でプロジェクトを作成・選択
gcloud projects create my-rork-gemma4-app --name="Rork Gemma 4 App"
gcloud config set project my-rork-gemma4-app

# Vertex AI API を有効化
gcloud services enable aiplatform.googleapis.com

# サービスアカウントキーを生成
gcloud iam service-accounts create gemma4-api \
  --display-name="Gemma 4 API Service Account"
gcloud iam service-accounts keys create ./gemma4-key.json \
  --iam-account=gemma4-api@my-rork-gemma4-app.iam.gserviceaccount.com
```

### Rork アプリからの API 呼び出し

Rork で生成したアプリのバックエンドに、Gemma 4 API を呼び出す関数を追加します。

```typescript
// services/gemma4.ts — Gemma 4 マルチモーダル API クライアント
import { GoogleAuth } from 'google-auth-library';

const PROJECT_ID = 'my-rork-gemma4-app';
const LOCATION = 'us-central1';
const MODEL_ID = 'gemma-4-31b-dense';

// Vertex AI エンドポイント
const ENDPOINT = `https://${LOCATION}-aiplatform.googleapis.com/v1/projects/${PROJECT_ID}/locations/${LOCATION}/publishers/google/models/${MODEL_ID}:predict`;

interface GemmaRequest {
  text: string;
  imageBase64?: string;  // オプション: 画像入力
  maxTokens?: number;
}

interface GemmaResponse {
  text: string;
  tokenCount: number;
}

export async function callGemma4(request: GemmaRequest): Promise<GemmaResponse> {
  const auth = new GoogleAuth({
    scopes: ['https://www.googleapis.com/auth/cloud-platform'],
  });
  const client = await auth.getClient();
  const accessToken = await client.getAccessToken();

  // マルチモーダル入力の構築
  const parts: any[] = [{ text: request.text }];
  if (request.imageBase64) {
    parts.push({
      inlineData: {
        mimeType: 'image/jpeg',
        data: request.imageBase64,
      },
    });
  }

  const response = await fetch(ENDPOINT, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${accessToken.token}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      instances: [{ content: { parts } }],
      parameters: {
        maxOutputTokens: request.maxTokens || 2048,
        temperature: 0.7,
      },
    }),
  });

  const data = await response.json();
  // レスポンスのパース
  // 期待する出力: { text: "AIの応答テキスト", tokenCount: 150 }
  return {
    text: data.predictions[0].content.parts[0].text,
    tokenCount: data.metadata.tokenMetadata.outputTokenCount,
  };
}
```

この関数をRorkアプリの画面から呼び出すことで、テキストと画像の両方を入力として Gemma 4 に処理を依頼できます。

## 画像認識機能を Rork アプリに実装する

Gemma 4 のマルチモーダル機能を活かした実践的な実装として、ユーザーがカメラで撮影した画像を AI が分析する機能を作ってみましょう。

```typescript
// screens/ImageAnalyzer.tsx — 画像分析画面
import React, { useState } from 'react';
import { View, Image, Text, TouchableOpacity, ActivityIndicator } from 'react-native';
import * as ImagePicker from 'expo-image-picker';
import * as FileSystem from 'expo-file-system';
import { callGemma4 } from '../services/gemma4';

export default function ImageAnalyzer() {
  const [image, setImage] = useState<string | null>(null);
  const [analysis, setAnalysis] = useState<string>('');
  const [loading, setLoading] = useState(false);

  const pickAndAnalyze = async () => {
    // カメラまたはギャラリーから画像を選択
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      quality: 0.8,
    });

    if (!result.canceled && result.assets[0]) {
      const uri = result.assets[0].uri;
      setImage(uri);
      setLoading(true);

      try {
        // 画像を Base64 に変換
        const base64 = await FileSystem.readAsStringAsync(uri, {
          encoding: FileSystem.EncodingType.Base64,
        });

        // Gemma 4 で画像を分析
        const response = await callGemma4({
          text: 'この画像の内容を詳しく説明してください。物体、色、構図、雰囲気を含めて分析してください。',
          imageBase64: base64,
          maxTokens: 1024,
        });

        setAnalysis(response.text);
      } catch (error) {
        setAnalysis('画像の分析中にエラーが発生しました。');
      } finally {
        setLoading(false);
      }
    }
  };

  return (
    <View style={{ flex: 1, padding: 16 }}>
      <TouchableOpacity onPress={pickAndAnalyze} style={styles.button}>
        <Text style={styles.buttonText}>画像を選択して分析</Text>
      </TouchableOpacity>
      {image && <Image source={{ uri: image }} style={styles.preview} />}
      {loading && <ActivityIndicator size="large" />}
      {analysis && <Text style={styles.analysis}>{analysis}</Text>}
    </View>
  );
}
```

Gemma 4 の 31B Dense モデルは画像理解において非常に高い精度を発揮します。料理の写真からレシピを提案する、商品画像から詳細情報を抽出する、風景写真のキャプションを自動生成するなど、さまざまなユースケースに応用できます。

## 256K コンテキストを活かした長文処理アプリ

Gemma 4 の **256K トークン**ロングコンテキストは、従来のモデルでは不可能だった長文処理を実現します。たとえば、PDFドキュメント全体の要約や、長い会話履歴を保持した AI チャットボットを構築できます。

```typescript
// services/longContext.ts — 長文処理サービス
export async function summarizeLongDocument(
  documentText: string,
  language: 'ja' | 'en' = 'ja'
): Promise<string> {
  const prompt = language === 'ja'
    ? `以下のドキュメントを読み、重要なポイントを箇条書きで要約してください。
       その後、全体の要約を3段落で記述してください。\n\n${documentText}`
    : `Read the following document and summarize the key points as bullet points.
       Then write a 3-paragraph overall summary.\n\n${documentText}`;

  // 256K トークンの範囲内であれば、ドキュメント全文を一度に送信可能
  const response = await callGemma4({
    text: prompt,
    maxTokens: 4096,
  });

  return response.text;
}

// 使用例: Rork アプリで PDF の内容を要約
// const pdfText = await extractTextFromPDF(pdfUri);
// const summary = await summarizeLongDocument(pdfText, 'ja');
// 期待する出力:
// "■ 主要ポイント:
//  - ポイント1: ...
//  - ポイント2: ...
//  ■ 全体要約:
//  第1段落: ..."
```

256K トークンは日本語で約12万〜15万文字に相当します。一般的なビジネス文書やレポートであれば、分割処理なしに全文を一度に渡せるため、コンテキストの欠落による誤解がなくなります。

## API コスト管理とレート制限の実装

クラウド API を利用する場合、コスト管理は重要な課題です。Rork アプリに適切なレート制限とキャッシュを実装しましょう。

```typescript
// middleware/rateLimiter.ts — API コスト管理
import AsyncStorage from '@react-native-async-storage/async-storage';

const DAILY_LIMIT = 100;       // 1日あたりのリクエスト上限
const CACHE_TTL = 3600 * 1000; // キャッシュ有効期間: 1時間

// シンプルなリクエストキャッシュ
const cache = new Map<string, { data: string; timestamp: number }>();

export async function rateLimitedGemma4(request: GemmaRequest): Promise<GemmaResponse> {
  // キャッシュチェック（同じクエリには既存の結果を返す）
  const cacheKey = JSON.stringify({ text: request.text, hasImage: !!request.imageBase64 });
  const cached = cache.get(cacheKey);
  if (cached && Date.now() - cached.timestamp < CACHE_TTL) {
    return { text: cached.data, tokenCount: 0 };
  }

  // 日次リクエスト数のチェック
  const today = new Date().toISOString().split('T')[0];
  const countKey = `gemma4_count_${today}`;
  const currentCount = parseInt(await AsyncStorage.getItem(countKey) || '0', 10);

  if (currentCount >= DAILY_LIMIT) {
    throw new Error('本日のAI分析回数の上限に達しました。明日再度お試しください。');
  }

  // API 呼び出し
  const response = await callGemma4(request);

  // カウンターとキャッシュを更新
  await AsyncStorage.setItem(countKey, String(currentCount + 1));
  cache.set(cacheKey, { data: response.text, timestamp: Date.now() });

  return response;
}
```

この方法で、ユーザーの利用頻度が予想を超えた場合でも API コストが爆発的に増加することを防げます。フリーミアムモデルのアプリでは、無料ユーザーに日次制限を設けつつ、サブスクリプション会員には上限を緩和するのが一般的な設計パターンです。

## FAQ

**Q: Gemma 4 の API 利用にはどのくらいの費用がかかりますか？**

A: Vertex AI での Gemma 4 利用料金は、入力トークンと出力トークンに対して課金されます。31B Dense モデルの場合、1,000トークンあたり約 $0.0005〜$0.001 程度です。画像入力は追加トークンとして換算されます。月間1万リクエスト程度であれば、月額 $10〜$30 の範囲に収まることが多いです。

**Q: Gemma 4 と Gemini API はどう使い分けるべきですか？**

A: Gemini API は Google の最高性能モデルを利用でき、Google サービスとの統合も強力です。一方 Gemma 4 は Apache 2.0 のオープンソースモデルなので、自社サーバーへのデプロイや Vertex AI でのカスタマイズが可能です。商用利用で柔軟性を重視するなら Gemma 4、手軽さと最高性能を求めるなら Gemini を選ぶと良いでしょう。

**Q: Rork で生成したアプリに後から Gemma 4 API を追加できますか？**

A: はい、可能です。Rork で生成されたアプリは標準的な React Native / Expo プロジェクトです。この記事で紹介した `services/gemma4.ts` のようなモジュールを追加し、既存の画面コンポーネントから呼び出すだけで統合できます。Rork Max を利用している場合は、AI 機能の追加がさらにスムーズです。

**Q: 画像認識の精度はどの程度ですか？**

A: Gemma 4 31B Dense モデルの画像認識精度は、一般的な物体検出・シーン理解において GPT-4V と同等以上の結果が報告されています。特に日本語での画像説明生成では、オープンモデルとしてトップクラスの精度を発揮します。ただし、医療画像や専門的な技術図面の解析には、ドメイン特化のファインチューニングが必要です。
