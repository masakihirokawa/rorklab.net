## はじめに — AI で植物の健康を見守る時代

「この葉っぱの黄色い斑点、大丈夫かな？」「水やりのタイミングがわからない…」。植物を育てていると、こうした悩みは尽きません。しかし、AI 画像認識の進化により、スマートフォンのカメラで植物を撮影するだけで健康状態を診断できるアプリが現実のものになりました。

この記事では、Rork を使って **AI 搭載の植物ケアアプリ** を構築する方法を、プロンプト設計から UI 実装まで段階的に解説します。完成するアプリでは、ユーザーがカメラで植物を撮影すると、AI が葉の色・形状・斑点などを分析し、考えられる症状と対処法を提示します。

**この記事で学べること:**

- Rork でカメラ機能を実装し、撮影した画像を AI に送信する方法
- マルチモーダル AI（画像＋テキスト）を活用したプロンプト設計のコツ
- 診断結果を見やすく表示する UI コンポーネントの構築
- 診断履歴を保存して植物の成長を記録する仕組み

**対象読者:** Rork の基本操作を理解しており、AI 連携アプリに挑戦したい中級者。

## カメラ機能の実装 — expo-camera と expo-image-picker

植物診断アプリの基盤となるのがカメラ機能です。Rork では、React Native / Expo のエコシステムを活用して、比較的少ないコードでカメラアクセスを実現できます。

Rork のプロンプトで以下のように指示すると、カメラ機能付きの画面を生成してくれます。

```
植物を撮影するためのカメラ画面を作成してください。
以下の機能を含めてください:
- カメラプレビュー画面（リアルタイムで被写体を確認）
- 撮影ボタン（丸型、画面下部中央に配置）
- ギャラリーから画像を選択するボタン
- 撮影後のプレビュー画面（「診断する」「撮り直す」ボタン付き）
```

生成されるコードでは、`expo-camera` でリアルタイムプレビューを表示し、`expo-image-picker` でギャラリーからの画像選択にも対応します。ポイントは、撮影した画像を base64 エンコードして AI API に渡せる形式にしておくことです。

```typescript
// カメラで撮影した画像を base64 に変換する例
import * as FileSystem from 'expo-file-system';

const captureAndEncode = async (cameraRef: CameraView) => {
  // 撮影して画像URIを取得
  const photo = await cameraRef.takePictureAsync({
    quality: 0.8,  // 画質（0〜1）— API転送を考慮して0.8程度に
    base64: true,   // base64エンコードを同時に取得
  });

  // photo.base64 に画像データが格納される
  return {
    uri: photo.uri,
    base64: photo.base64,
  };
};
// 期待する出力: { uri: "file:///...", base64: "/9j/4AAQSkZ..." }
```

ここで重要なのは `quality` パラメータです。値を `1.0` にすると画像サイズが大きくなり API 通信に時間がかかります。植物の診断には `0.7〜0.8` 程度で十分な精度が得られます。

## AI プロンプト設計 — 植物診断の精度を左右するポイント

カメラで撮影した画像を AI に送るだけでは、有用な診断結果は得られません。**プロンプト設計**が診断精度を大きく左右します。

以下は、植物診断に最適化したシステムプロンプトの例です。

```typescript
// 植物診断用のシステムプロンプト
const PLANT_DIAGNOSIS_PROMPT = `
あなたは植物の健康診断を専門とするAIアシスタントです。
ユーザーが送信した植物の画像を分析し、以下の形式で回答してください。

## 回答フォーマット
1. **植物の種類**: 画像から推定される植物名（不明な場合は「特定困難」と記載）
2. **健康状態**: 「良好」「注意」「要対処」の3段階で評価
3. **観察される症状**: 葉の色・形状・斑点・萎れなどの具体的な所見
4. **考えられる原因**: 水不足、日照不足、病気、害虫など
5. **推奨される対処法**: 具体的なケア方法を3つ以内で提案
6. **緊急度**: 「低」「中」「高」で評価

## 注意事項
- 専門的すぎる用語は避け、初心者にもわかりやすく説明する
- 断定的な診断は避け、「〜の可能性があります」という表現を使う
- 画像が不鮮明な場合は、再撮影を促す
`;
// 期待する出力: 構造化されたJSON形式の診断結果
```

このプロンプトのポイントは、**出力フォーマットを明確に指定**していることです。AI の応答を構造化データとして受け取ることで、アプリ側でパースして UI に反映しやすくなります。

実際に Rork で実装する場合は、以下のようなプロンプトを Rork に伝えます。

```
植物診断のAI機能を実装してください。
- カメラで撮影した画像をbase64形式でAI APIに送信
- レスポンスをJSON形式でパースして構造化データとして扱う
- 診断結果を画面に表示（植物名・健康状態・症状・対処法）
- ローディング中はスケルトンUIを表示
- エラー時はリトライボタンを表示
```

## 診断結果 UI の構築 — ユーザーにわかりやすく伝える

AI から返された診断結果を、ユーザーにとってわかりやすい形で表示することが重要です。単にテキストを並べるだけでは、せっかくの AI 診断の価値が伝わりません。

以下のような UI 構成を Rork で生成します。

```
診断結果の表示画面を作成してください:

1. ヘッダー部分:
   - 撮影した植物の画像（角丸でシャドウ付き）
   - 健康状態バッジ（良好=緑、注意=黄、要対処=赤）

2. 診断カード:
   - 植物名と種類アイコン
   - 症状の詳細（アコーディオン形式で展開可能）
   - 原因の一覧（アイコン付きリスト）

3. 対処法セクション:
   - 各対処法をカード形式で表示
   - 優先度順に並べる
   - 「詳しく見る」タップで追加説明を表示

4. アクションボタン:
   - 「履歴に保存」ボタン
   - 「もう一度撮影」ボタン
   - 「SNSでシェア」ボタン

カラーパレットはグリーン系を基調にしてください。
```

健康状態に応じて色分けすることで、ユーザーは一目で植物の状態を把握できます。

```typescript
// 健康状態に応じた色とアイコンの定義
type HealthStatus = 'good' | 'caution' | 'critical';

const STATUS_CONFIG: Record<HealthStatus, { color: string; icon: string; label: string }> = {
  good: {
    color: '#22C55E',    // 緑 — 健康
    icon: '🌿',
    label: '良好',
  },
  caution: {
    color: '#F59E0B',    // 黄 — 注意
    icon: '⚠️',
    label: '注意',

    color: '#EF4444',    // 赤 — 要対処
    icon: '🚨',
    label: '要対処',
  },

// 期待する出力: STATUS_CONFIG['good'] → { color: '#22C55E', icon: '🌿', label: '良好' }
```

## 診断履歴の保存と成長記録 — AsyncStorage で永続化

一回きりの診断では、植物の経過観察ができません。診断履歴を端末に保存し、植物ごとの成長記録として活用できる機能を追加しましょう。

```typescript
// 診断履歴の保存・読み込み（AsyncStorage）
import AsyncStorage from '@react-native-async-storage/async-storage';

interface DiagnosisRecord {
  id: string;
  plantName: string;
  imageUri: string;
  status: HealthStatus;
  symptoms: string[];
  recommendations: string[];
  diagnosedAt: string;  // ISO 8601 形式
}

// 診断結果を保存
const saveDiagnosis = async (record: DiagnosisRecord) => {
  const existing = await AsyncStorage.getItem('diagnosis_history');
  const history: DiagnosisRecord[] = existing ? JSON.parse(existing) : [];

  history.unshift(record);  // 新しい記録を先頭に追加

  // 最大100件まで保持（ストレージ節約）
  const trimmed = history.slice(0, 100);
  await AsyncStorage.setItem('diagnosis_history', JSON.stringify(trimmed));
};

// 履歴を取得
const getDiagnosisHistory = async (): Promise<DiagnosisRecord[]> => {
  const data = await AsyncStorage.getItem('diagnosis_history');
  return data ? JSON.parse(data) : [];
};
// 期待する出力: [{ id: "abc123", plantName: "モンステラ", status: "good", ... }]
```

Rork で履歴画面を生成するには、以下のようなプロンプトが効果的です。

```
診断履歴一覧画面を作成してください:
- FlatListで履歴をカード形式で表示
- 各カードに植物の写真サムネイル、植物名、健康状態バッジ、診断日時を表示
- カードタップで詳細画面に遷移
- 上部に検索バーとフィルター（健康状態で絞り込み）
- 履歴がない場合は「まだ診断履歴がありません」と撮影ボタンを表示
- スワイプで削除可能
```

同じ植物を定期的に撮影することで、時系列での健康状態の推移を確認できるようになります。これは、園芸初心者にとって非常に価値のある機能です。

## 応用機能 — 水やりリマインダーと植物図鑑

基本の診断機能ができたら、ユーザーの継続利用を促す応用機能を追加しましょう。

**水やりリマインダー**は、診断結果をもとに AI が最適な水やり頻度を提案し、ローカル通知でリマインドする機能です。

```typescript
// 水やりリマインダーのスケジュール設定
import * as Notifications from 'expo-notifications';

const scheduleWateringReminder = async (
  plantName: string,
  intervalDays: number  // AI が推奨した水やり間隔（日数）
) => {
  await Notifications.scheduleNotificationAsync({
    content: {
      title: '水やりの時間です 💧',
      body: `${plantName} に水をあげましょう。前回の診断結果に基づく推奨スケジュールです。`,
      sound: 'default',
    },
    trigger: {
      type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL,
      seconds: intervalDays * 24 * 60 * 60,
      repeats: true,
    },
  });
};
// 期待する出力: 指定した間隔でプッシュ通知が繰り返し送信される
```

**植物図鑑機能**は、過去に診断した植物の情報をまとめて、ユーザー独自の図鑑として閲覧できるようにする機能です。AI が植物を識別した情報（学名、原産地、適切な日照条件など）を蓄積することで、育て方のナレッジベースが自然に構築されます。

さらに発展的なアプローチとして、カメラだけでなく音声入力やテキストも組み合わせた **マルチモーダルなインタラクション** を検討する場合は、[Rork × マルチモーダルAI：カメラ・音声・テキストを統合したインテリジェントアプリ設計パターン](/articles/rork-ai/rork-multimodal-ai-app-design-patterns) で詳しい設計パターンを解説しています。

## よくある質問（FAQ）

### Q1: AI の植物診断はどの程度正確ですか？

AI による植物診断は、一般的な症状（水不足、日照不足、よくある病気）については高い精度で判定できます。ただし、専門的な病理診断の代わりにはなりません。深刻な症状が疑われる場合は、園芸店や植物病院への相談を促すメッセージをアプリに組み込むことをおすすめします。

### Q2: API の利用コストはどのくらいかかりますか？

画像を含むマルチモーダル API のコストは、テキストのみのリクエストより高くなります。1回の診断あたり概ね数円〜十数円程度が目安です。コスト最適化のためには、画像の解像度を適切に圧縮する（前述の `quality: 0.8` 設定）、キャッシュを活用して同じ画像の再診断を避ける、といった工夫が有効です。

### Q3: オフラインでも使えるようにできますか？

完全なオフライン診断には、TensorFlow Lite や Core ML などのオンデバイス AI モデルが必要です。Rork の標準的な構成ではクラウド API を利用しますが、キャッシュ済みの診断結果の閲覧や、水やりリマインダーの通知はオフラインでも動作します。オフライン時には「接続を確認してください」のメッセージを表示し、再接続後に自動で診断を実行する仕組みにするのがスムーズです。

### Q4: 収益化するにはどうすればよいですか？

基本的な診断機能を無料で提供し、以下の機能をプレミアム（有料）にする **フリーミアムモデル** が効果的です。月あたりの診断回数上限（無料: 5回/月、有料: 無制限）、詳細な成長分析レポート、AI による個別ケアプランの生成、広告の非表示などが差別化ポイントになります。Rork で Stripe サブスクリプションを組み込む方法については、既存の解説記事も参考にしてみてください。
