## はじめに — なぜ AI 議事録アプリなのか

会議のたびにメモを取り、終了後に要約を作成し、タスクを抽出する。この作業に毎回30分以上費やしているビジネスパーソンは少なくありません。AI 議事録アプリは、この一連の作業を「録音ボタンを押すだけ」に変えてくれます。

本記事では、Rork を使って以下の機能を持つ AI 議事録アプリをゼロから構築します。

- **音声録音**: デバイスのマイクで会議音声を録音
- **文字起こし**: OpenAI Whisper API で音声をテキストに変換
- **AI 要約**: Claude API でテキストを自動要約し、アクションアイテムを抽出
- **履歴管理**: 過去の議事録を一覧表示・検索

Rork のコード生成機能と、Whisper・Claude という2つの強力な AI API を組み合わせることで、プロダクションレベルの議事録アプリを効率的に開発できます。

この記事は中級者向けですが、Rork の基本操作と React Native の基礎知識があれば問題なく進められます。

## 必要な環境と事前準備

アプリ開発を始める前に、以下のサービスとツールを準備してください。

**必須アカウント:**

- Rork アカウント（無料プランでも開発可能）
- OpenAI アカウント（Whisper API 用）
- Anthropic アカウント（Claude API 用）

**API キーの取得:**

OpenAI のダッシュボード（`platform.openai.com`）から API キーを発行し、安全な場所に保管します。同様に、Anthropic のコンソール（`console.anthropic.com`）からも API キーを取得してください。

**Rork プロジェクトの作成:**

Rork のダッシュボードで新しいプロジェクトを作成し、以下のようなプロンプトで基本構造を生成します。

```
会議の音声を録音して、AIで文字起こしと要約を行う議事録アプリを作ってください。
タブナビゲーションで「録音」「履歴」「設定」の3画面構成にしてください。
```

Rork が生成した初期コードをベースに、以降のステップで AI 機能を追加していきます。

## 音声録音機能の実装

React Native で音声を録音するには `expo-av` ライブラリを使用します。Rork のプロジェクトでは、プロンプトで指示するだけで依存関係のインストールまで自動で処理されます。

以下は録音のコア機能を実装するカスタムフックの例です。

```typescript
// hooks/useAudioRecorder.ts
import { Audio } from 'expo-av';
import { useState, useRef } from 'react';

export function useAudioRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const [recordingUri, setRecordingUri] = useState<string | null>(null);
  const recordingRef = useRef<Audio.Recording | null>(null);

  const startRecording = async () => {
    // マイク権限をリクエスト
    const permission = await Audio.requestPermissionsAsync();
    if (!permission.granted) {
      throw new Error('マイクの使用許可が必要です');
    }

    // 録音設定（Whisper API に最適な設定）
    await Audio.setAudioModeAsync({
      allowsRecordingIOS: true,
      playsInSilentModeIOS: true,
    });

    const recording = new Audio.Recording();
    await recording.prepareToRecordAsync({
      android: {
        extension: '.m4a',
        outputFormat: Audio.AndroidOutputFormat.MPEG_4,
        audioEncoder: Audio.AndroidAudioEncoder.AAC,
        sampleRate: 16000,  // Whisper推奨サンプルレート
        numberOfChannels: 1,
        bitRate: 128000,
      },
      ios: {
        extension: '.m4a',
        audioQuality: Audio.IOSAudioQuality.HIGH,
        sampleRate: 16000,
        numberOfChannels: 1,
        bitRate: 128000,
        outputFormat: Audio.IOSOutputFormat.MPEG4AAC,
      },
      web: { mimeType: 'audio/webm', bitsPerSecond: 128000 },
    });

    await recording.startAsync();
    recordingRef.current = recording;
    setIsRecording(true);
  };

  const stopRecording = async () => {
    if (!recordingRef.current) return null;

    await recordingRef.current.stopAndUnloadAsync();
    const uri = recordingRef.current.getURI();
    recordingRef.current = null;
    setIsRecording(false);
    setRecordingUri(uri);
    return uri;
  };

  return { isRecording, recordingUri, startRecording, stopRecording };
}
```

**ポイント**: Whisper API は 16kHz のサンプルレートで最も精度が高くなります。また、モノラル（1チャンネル）で録音することでファイルサイズを抑え、API へのアップロード時間を短縮できます。

音声ファイルのサイズ制限として、Whisper API は最大 25MB のファイルを受け付けます。一般的な会議（1時間程度）であれば、上記の設定で 10MB 前後に収まるため問題ありません。

## Whisper API で音声をテキストに変換する

録音した音声ファイルを OpenAI の Whisper API に送信して、テキストに変換します。

```typescript
// services/transcription.ts
export async function transcribeAudio(audioUri: string): Promise<string> {
  const formData = new FormData();

  // 音声ファイルをFormDataに追加
  formData.append('file', {
    uri: audioUri,
    type: 'audio/m4a',
    name: 'meeting-audio.m4a',
  } as any);

  formData.append('model', 'whisper-1');
  formData.append('language', 'ja');        // 日本語を指定
  formData.append('response_format', 'verbose_json');
  formData.append('timestamp_granularities[]', 'segment');

  const response = await fetch(
    'https://api.openai.com/v1/audio/transcriptions',
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${YOUR_OPENAI_API_KEY}`,
      },
      body: formData,
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`文字起こし失敗: ${error.error?.message}`);
  }

  const result = await response.json();

  // セグメントごとのタイムスタンプ付きテキストを生成
  const transcriptWithTimestamps = result.segments
    .map((seg: any) => {
      const minutes = Math.floor(seg.start / 60);
      const seconds = Math.floor(seg.start % 60);
      const timestamp = `[${minutes}:${seconds.toString().padStart(2, '0')}]`;
      return `${timestamp} ${seg.text}`;
    })
    .join('\n');

  return transcriptWithTimestamps;
}

// 期待する出力例:
// [0:00] 本日の定例ミーティングを始めます。
// [0:05] まずは先週のタスクの進捗確認からお願いします。
// [0:12] はい、デザインの修正は完了しました。レビュー待ちの状態です。
```

**`verbose_json` を使う理由**: 単純なテキスト出力ではなく `verbose_json` を指定することで、セグメントごとのタイムスタンプが取得できます。これにより、要約から元の発言箇所にジャンプする機能が実装可能になります。

**コスト目安**: Whisper API の料金は $0.006/分です。1時間の会議で約 $0.36（約55円）と非常にリーズナブルです。

## Claude API で議事録を自動要約する

文字起こしが完了したら、Claude API を使ってインテリジェントな要約を生成します。Claude の長いコンテキストウィンドウは、長時間の会議テキストを一度に処理するのに最適です。

```typescript
// services/summarize.ts
interface MeetingSummary {
  title: string;
  overview: string;
  keyPoints: string[];
  actionItems: { task: string; assignee: string; deadline: string }[];
  decisions: string[];
  nextSteps: string;
}

export async function summarizeMeeting(
  transcript: string
): Promise<MeetingSummary> {
  const systemPrompt = `あなたは優秀なビジネスアシスタントです。
会議の文字起こしテキストを受け取り、以下の形式でJSON要約を生成してください。

{
  "title": "会議のタイトル（内容から推測）",
  "overview": "会議全体の概要（3〜5文）",
  "keyPoints": ["重要なポイント1", "重要なポイント2", ...],
  "actionItems": [
    {"task": "タスク内容", "assignee": "担当者名", "deadline": "期限"}
  ],
  "decisions": ["決定事項1", "決定事項2", ...],
  "nextSteps": "次回までにやるべきことの概要"
}

担当者名が不明な場合は「未定」、期限が不明な場合は「要確認」と記載してください。
JSONのみを出力し、それ以外のテキストは含めないでください。`;

  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': YOUR_ANTHROPIC_API_KEY,
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify({
      model: 'claude-sonnet-4-6-20250329',
      max_tokens: 2048,
      system: systemPrompt,
      messages: [
        {
          role: 'user',
          content: `以下の会議の文字起こしを要約してください:\n\n${transcript}`,
        },
      ],
    }),
  });

  if (!response.ok) {
    throw new Error('要約の生成に失敗しました');
  }

  const data = await response.json();
  const content = data.content[0].text;
  return JSON.parse(content) as MeetingSummary;
}
```

**プロンプト設計のコツ**: 出力フォーマットを JSON で厳密に指定することで、パース可能な構造化データを安定して取得できます。また、「担当者名が不明な場合は未定」のようにフォールバック指示を入れることで、情報が不足している場合でもエラーなく処理が進みます。

音声認識から要約生成までの一連のパイプラインについて、より高度なマルチモーダル処理を学びたい方は [Rork × マルチモーダルAI：カメラ・音声・テキストを統合したインテリジェントアプリ設計パターン](/articles/rork-ai/rork-multimodal-ai-app-design-patterns) もあわせてご覧ください。

## 録音から要約までの統合パイプライン

個別の機能を組み合わせて、ワンタップで録音→文字起こし→要約までを実行するパイプラインを構築します。

```typescript
// hooks/useMeetingPipeline.ts
import { useState } from 'react';
import { useAudioRecorder } from './useAudioRecorder';
import { transcribeAudio } from '../services/transcription';
import { summarizeMeeting } from '../services/summarize';

type PipelineStatus =
  | 'idle'
  | 'recording'
  | 'transcribing'
  | 'summarizing'
  | 'complete'
  | 'error';

export function useMeetingPipeline() {
  const { isRecording, startRecording, stopRecording } = useAudioRecorder();
  const [status, setStatus] = useState<PipelineStatus>('idle');
  const [transcript, setTranscript] = useState('');
  const [summary, setSummary] = useState<MeetingSummary | null>(null);
  const [error, setError] = useState<string | null>(null);

  const start = async () => {
    setError(null);
    setStatus('recording');
    await startRecording();
  };

  const stop = async () => {
    try {
      // Step 1: 録音停止
      const uri = await stopRecording();
      if (!uri) throw new Error('録音ファイルが見つかりません');

      // Step 2: 文字起こし
      setStatus('transcribing');
      const text = await transcribeAudio(uri);
      setTranscript(text);

      // Step 3: AI 要約
      setStatus('summarizing');
      const result = await summarizeMeeting(text);
      setSummary(result);

      setStatus('complete');
    } catch (err) {
      setError(err instanceof Error ? err.message : '処理中にエラーが発生しました');
      setStatus('error');
    }
  };

  return { status, transcript, summary, error, start, stop, isRecording };
}
```

**UX のポイント**: 各ステップのステータスを管理することで、「文字起こし中...」「要約生成中...」のように進捗をユーザーに伝えられます。1時間の会議の場合、文字起こしに約30秒、要約に約10秒が目安です。

## 議事録一覧画面と検索機能

過去の議事録を管理するために、AsyncStorage を使ったローカル保存と検索機能を実装します。

```typescript
// services/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const MEETINGS_KEY = 'meeting_notes';

export interface SavedMeeting {
  id: string;
  date: string;
  title: string;
  overview: string;
  transcript: string;
  summary: MeetingSummary;
}

export async function saveMeeting(meeting: SavedMeeting) {
  const existing = await getMeetings();
  existing.unshift(meeting); // 新しい順に並べる
  await AsyncStorage.setItem(MEETINGS_KEY, JSON.stringify(existing));
}

export async function getMeetings(): Promise<SavedMeeting[]> {
  const data = await AsyncStorage.getItem(MEETINGS_KEY);
  return data ? JSON.parse(data) : [];
}

// キーワード検索（タイトル・概要・文字起こしを横断検索）
export async function searchMeetings(query: string): Promise<SavedMeeting[]> {
  const meetings = await getMeetings();
  const lowerQuery = query.toLowerCase();
  return meetings.filter(
    (m) =>
      m.title.toLowerCase().includes(lowerQuery) ||
      m.overview.toLowerCase().includes(lowerQuery) ||
      m.transcript.toLowerCase().includes(lowerQuery)
  );
}
```

Rork でこの保存機能を組み込む際には、「議事録を保存して一覧表示するリスト画面を作って。各アイテムにはタイトル、日付、概要のプレビューを表示して」とプロンプトで指示すれば、FlatList を使った一覧画面が自動生成されます。

音声メモの基本的な録音・再生パターンについては [Rork で作るボイスメモアプリ チュートリアル](/articles/rork-basics/rork-voice-memo-app-tutorial) で詳しく解説していますので、初めて音声機能を扱う方はそちらもご参考ください。

## セキュリティとコスト管理のベストプラクティス

AI API を使ったアプリを本番運用する際には、セキュリティとコストの管理が重要です。

**API キーの安全な管理:**

クライアントサイドに API キーを直接埋め込むのは危険です。本番環境では必ずバックエンドサーバーを経由させてください。

```typescript
// ❌ 危険: クライアントに直接APIキーを埋め込む
const response = await fetch('https://api.openai.com/v1/audio/transcriptions', {
  headers: { Authorization: `Bearer ${YOUR_API_KEY}` },
});

// ✅ 安全: 自前のバックエンドを経由する
const response = await fetch('https://your-backend.com/api/transcribe', {
  method: 'POST',
  headers: { Authorization: `Bearer ${userToken}` },
  body: formData,
});
```

Supabase Edge Functions や Cloudflare Workers を使えば、サーバーレスで API プロキシを構築できます。Rork のプロンプトに「Supabase Edge Function で Whisper API へのプロキシを作成して」と指示すれば、対応するコードが生成されます。

**コスト管理の仕組み:**

- OpenAI・Anthropic の両方で月額上限（Usage Limits）を設定する
- ユーザーごとの利用回数を制限する（例: 無料プランは月5回まで）
- 長時間の録音は分割して処理することで、1回あたりのコストを予測可能にする

**コスト試算の目安:**

- 30分の会議: Whisper $0.18 + Claude 約$0.02 = **約$0.20（約30円）**
- 1時間の会議: Whisper $0.36 + Claude 約$0.04 = **約$0.40（約60円）**
- 月20回（1時間）利用した場合: **約$8（約1,200円）**

さらに高度な AI アプリのアーキテクチャ設計を学びたい方には、[Rork × RAG パターン完全ガイド — モバイルAIチャットにナレッジベースを統合する設計と実装](/articles/rork-ai/rork-rag-knowledge-base-ai-chat-app-guide) がおすすめです。ナレッジベースと組み合わせることで、議事録の内容を蓄積し、過去の決定事項をAIに質問できるような発展形も実現できます。

## よくある質問（FAQ）

### Q1: Whisper API は日本語の議事録でも精度は十分ですか？

はい、Whisper は多言語対応モデルで、日本語の認識精度も非常に高いです。`language: 'ja'` を明示的に指定することで、日本語に最適化された認識が行われます。ただし、専門用語や社内用語は誤認識されることがあるため、重要な会議では文字起こし結果の確認をおすすめします。

### Q2: 録音中にアプリがバックグラウンドに移行した場合はどうなりますか？

iOS ではバックグラウンドでの音声録音を許可する設定が必要です。`Info.plist` に `UIBackgroundModes` の `audio` を追加するか、Rork のプロンプトで「バックグラウンド録音を有効にして」と指示してください。Android では、フォアグラウンドサービスとして録音を継続する方法が推奨されます。

### Q3: オフラインで使用することは可能ですか？

録音機能自体はオフラインで動作しますが、文字起こし（Whisper API）と要約（Claude API）はインターネット接続が必要です。オフライン時は録音のみ行い、接続が復旧した段階で自動的に API 処理を実行するキュー機能を実装するとよいでしょう。

### Q4: 複数人の話者を区別することはできますか？

Whisper API 単体では話者分離（Speaker Diarization）に対応していません。話者を識別したい場合は、AssemblyAI や Deepgram などの話者分離対応 API と組み合わせるか、Whisper の文字起こし結果に対して Claude に「発言者を推測して分類してください」とプロンプトで指示する方法もあります。
