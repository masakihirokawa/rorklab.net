技術書を読んでいて、気になるコードに出会ったとき、あなたはどうしますか？

コピーしてAIチャットに貼り付ける方法もありますが、物理的な本のコードや、他人の画面に映ったコード、ホワイトボードに書かれた設計図は、テキストとして取り出せません。「カメラで撮影して、そのままAIに解説してもらえたら」——そのアイデアから生まれたのが、このガイドで作るアプリです。

Rork Max と Gemini 2.5 Flash を組み合わせると、このアプリは1日で形になります。ただし、実装してみると3つの大きな落とし穴があります。ネット上の記事ではほとんど触れられていない、マルチモーダルAPIならではの問題です。この記事では、その落とし穴の回避策も含めた完全な実装を公開します。

## このガイドで実装するアプリの全体像

完成するアプリの機能は次の通りです。

カメラで任意のコードを撮影すると、Gemini 2.5 Flash がそのコードを認識し、「初心者向け」「中級者向け」「上級者向け」の3段階で解説を生成します。解析結果は履歴として保存され、後から見返せます。無料プランは1日3回まで、プレミアムプランは無制限という課金設計にします。

ターゲットユーザーは「プログラミングを学習中で、気になるコードに出会ったとき即座に解説が欲しい人」です。技術書籍・Udemy動画・YouTubeのライブコーディング・職場でのペアプロなど、コードに触れる場面は多く、需要は確実にあります。

### アーキテクチャ選定：なぜ Gemini 2.5 Flash なのか

カメラ→AI解析のパイプラインには、主に3つの実装方式があります。

**方式A: クライアント直接呼び出し**（今回採用）
クライアントから直接 Gemini API を呼び出す、最もシンプルな構成です。APIキーの管理に注意が必要ですが、インフラ不要で個人開発・プロトタイプに最適です。

**方式B: バックエンド経由**
Supabase Edge Functions や Cloudflare Workers を経由します。APIキーが安全で、レートリミット管理も容易ですが、インフラ準備のコストがあります。本番リリース後に移行することをおすすめします。

**方式C: On-Device AI**
Core MLやGemma 3などのオンデバイスモデルを使います。オフライン動作が可能ですが、コード認識精度と推論速度にまだ課題があります。

Gemini 2.5 Flash を選ぶ理由は2つです。**応答速度**と**コードテキスト認識精度**です。カメラアプリでは撮影から結果表示までの待ち時間がユーザー体験を左右します。GPT-4oと比較してFlashは平均応答時間が約40%速く、コストも低いです。また、手書きや印刷物のコードに対する文字認識精度がマルチモーダルモデルの中で高い評価を受けています。

## Step 2: expo-camera の権限処理を完全にする

Rork Max が生成するカメラコードには、権限処理が簡略化されていることがあります。App Store審査でリジェクトされる最も多い理由の一つが、カメラ権限の説明文が不適切なことです。以下が権限処理の完全な実装です。

```tsx
// app/camera.tsx — 権限処理の完全版
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useState, useRef } from 'react';
import { StyleSheet, Text, TouchableOpacity, View, Alert } from 'react-native';
import { router } from 'expo-router';

export default function CameraScreen() {
  const [permission, requestPermission] = useCameraPermissions();
  const [isCapturing, setIsCapturing] = useState(false);
  const cameraRef = useRef<CameraView>(null);

  // 権限状態の読み込み中
  if (\!permission) {
    return <View style={styles.container} />;
  }

  // 権限未許可の場合は説明画面を表示
  if (\!permission.granted) {
    return (
      <View style={styles.permissionContainer}>
        <Text style={styles.title}>カメラのアクセスが必要です</Text>
        <Text style={styles.message}>
          コードを撮影してAI解析するために、カメラへのアクセスが必要です。{'\n\n'}
          撮影した画像はGemini AI（Google）に送信されて解析されます。
          画像はサーバーに保存されることはありません。
        </Text>
        <TouchableOpacity style={styles.grantButton} onPress={requestPermission}>
          <Text style={styles.grantButtonText}>カメラを許可する</Text>
        </TouchableOpacity>
      </View>
    );
  }

  const handleCapture = async () => {
    if (\!cameraRef.current || isCapturing) return;

    setIsCapturing(true);
    try {
      // quality: 0.7 で圧縮（重要：理由は後述）
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.7,
        base64: true,
        exif: false, // 位置情報を含むExifを除去（プライバシー対策）
      });

      if (\!photo?.base64) {
        throw new Error('画像データの取得に失敗しました');
      }

      // 解析画面へ遷移
      router.push({
        pathname: '/result',
        params: { imageBase64: photo.base64 },
      });
    } catch (error) {
      Alert.alert(
        '撮影エラー',
        '撮影に失敗しました。もう一度お試しください。',
        [{ text: 'OK' }]
      );
      console.error('Camera capture error:', error);
    } finally {
      setIsCapturing(false);
    }
  };

  return (
    <View style={styles.container}>
      <CameraView ref={cameraRef} style={styles.camera} facing="back">
        <View style={styles.overlay}>
          {/* スキャンフレーム：コードの撮影範囲をガイド */}
          <View style={styles.scanFrame} />
          <Text style={styles.hint}>
            コードが枠内に収まるよう、明るい場所で撮影してください
          </Text>
        </View>
      </CameraView>

      <TouchableOpacity
        style={[styles.captureButton, isCapturing && styles.capturing]}
        onPress={handleCapture}
        disabled={isCapturing}
      >
        <Text style={styles.captureText}>
          {isCapturing ? '処理中...' : '撮影して解析'}
        </Text>
      </TouchableOpacity>
    </View>
  );
}
```

`quality: 0.7` の指定がなぜ重要かを説明します。デフォルト（quality: 1.0）のままにすると、最新のiPhoneやPixelでは1枚の写真が8〜12MBになります。Gemini APIはリクエストサイズに制限があり、大きすぎる画像はタイムアウトするか、エラーを返します。0.7に設定すると平均2〜3MBになり、コードの文字認識精度にはほぼ影響がありません。

## Step 3: Gemini 2.5 Flash マルチモーダルAPIの完全実装

Rork Max が生成する Gemini API 呼び出しコードにはよくある問題があります。JSONパースエラーです。Gemini は指示なしにJSONをMarkdownのコードブロックで囲んで返すことがあります。以下は、それを防ぐ完全な実装です。

```typescript
// utils/gemini.ts
const GEMINI_API_URL =
  'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';

export type ExplanationLevel = 'beginner' | 'intermediate' | 'advanced';

export interface CodeAnalysisResult {
  language: string;       // 検出されたプログラミング言語
  explanation: string;    // AI生成の解説テキスト
  keyPoints: string[];    // 重要ポイントのリスト
  potentialIssues?: string[]; // 注意点・改善点（上級向けのみ）
}

// レベル別のシステムプロンプト
const LEVEL_PROMPTS: Record<ExplanationLevel, string> = {
  beginner: `
プログラミング初心者向けのコード解説アシスタントとして動作してください。
画像内のコードを分析し、以下のJSON形式で出力してください（コードブロックで囲まないこと）：
{
  "language": "プログラミング言語名（例: JavaScript, Python, Swift）",
  "explanation": "このコードが何をするか、中学生でも分かる言葉で200文字以内で説明",
  "keyPoints": [
    "最も重要な処理の説明（30文字以内）",
    "2番目に重要な点（30文字以内）",
    "初心者が覚えておくべきこと（30文字以内）"
  ]
}
専門用語には必ず括弧で補足説明を入れてください（例: 「関数（処理をまとめたもの）」）。
  `.trim(),

  intermediate: `
中級プログラマー向けのコードレビュアーとして動作してください。
画像内のコードを分析し、以下のJSON形式で出力してください（コードブロックで囲まないこと）：
{
  "language": "プログラミング言語名",
  "explanation": "処理内容・設計パターン・使用技術を300文字以内で説明",
  "keyPoints": [
    "この実装で使われている設計パターンや技術的判断",
    "コードの構造やデータフローの特徴",
    "実務で応用できるポイント"
  ],
  "potentialIssues": [
    "改善できる点や注意点（ある場合のみ）"
  ]
}
  `.trim(),

  advanced: `
シニアエンジニアとして、アーキテクチャ・パフォーマンス・セキュリティの視点でコードをレビューしてください。
画像内のコードを分析し、以下のJSON形式で出力してください（コードブロックで囲まないこと）：
{
  "language": "プログラミング言語名",
  "explanation": "アーキテクチャ的判断・パフォーマンス特性・スケーラビリティを含む400文字以内の詳細分析",
  "keyPoints": [
    "採用されているアーキテクチャパターンとその理由",
    "パフォーマンスに影響する設計上の選択",
    "本番環境でのスケーラビリティへの影響"
  ],
  "potentialIssues": [
    "セキュリティリスクまたは改善点",
    "パフォーマンスボトルネックの可能性",
    "長期的なメンテナンス性の課題"
  ]
}
  `.trim(),
};

export async function analyzeCodeImage(
  imageBase64: string,
  level: ExplanationLevel,
  apiKey: string
): Promise<CodeAnalysisResult> {
  const requestBody = {
    contents: [
      {
        parts: [
          {
            inline_data: {
              mime_type: 'image/jpeg',
              data: imageBase64,
            },
          },
          {
            text: LEVEL_PROMPTS[level],
          },
        ],
      },
    ],
    generationConfig: {
      temperature: 0.1,      // 低温度で安定したJSON出力を確保
      topP: 0.8,
      maxOutputTokens: 1024,
      responseMimeType: 'application/json', // JSONを強制することでパースエラーを防ぐ
    },
  };

  const response = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestBody),
  });

  if (\!response.ok) {
    const errorText = await response.text();
    // エラー種別を判別してユーザーに分かりやすいメッセージを返す
    if (response.status === 400) {
      throw new Error('画像の形式が正しくありません。再撮影してください。');
    }
    if (response.status === 403) {
      throw new Error('APIキーの認証に失敗しました。設定を確認してください。');
    }
    if (response.status === 429) {
      throw new Error('API利用制限に達しました。しばらく待ってから再試行してください。');
    }
    throw new Error(`解析に失敗しました（エラー: ${response.status}）`);
  }

  const data = await response.json();
  const content = data.candidates?.[0]?.content?.parts?.[0]?.text;

  if (\!content) {
    throw new Error('AIからの応答が空でした。画像を変えて再試行してください。');
  }

  try {
    return JSON.parse(content) as CodeAnalysisResult;
  } catch {
    // JSONパースに失敗した場合（responseMimeType指定でも稀に起きる）のフォールバック
    return {
      language: '解析済み',
      explanation: content,
      keyPoints: [],
    };
  }
}
```

`responseMimeType: 'application/json'` の指定がこの実装で最も重要なポイントです。この指定がないと、Geminiは「```json ... ```」のようなMarkdownコードブロックでJSONを囲んで返すことがあります。そのまま `JSON.parse()` に渡すとエラーになります。この1行があるかどうかで、本番環境の安定性が大きく変わります。

## Step 4: 解析結果画面とレベル切り替えのキャッシュ設計

```tsx
// app/result.tsx
import { useLocalSearchParams, router } from 'expo-router';
import { useState, useEffect } from 'react';
import { ScrollView, Text, View, TouchableOpacity, ActivityIndicator, Alert } from 'react-native';
import { analyzeCodeImage, type ExplanationLevel, type CodeAnalysisResult } from '../utils/gemini';
import { saveToHistory } from '../utils/storage';

const LEVEL_LABELS: Record<ExplanationLevel, string> = {
  beginner: '初心者向け',
  intermediate: '中級者向け',
  advanced: '上級者向け',
};

export default function ResultScreen() {
  const { imageBase64 } = useLocalSearchParams<{ imageBase64: string }>();
  const [level, setLevel] = useState<ExplanationLevel>('beginner');
  const [result, setResult] = useState<CodeAnalysisResult | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  // キャッシュ：同じ画像への再APIコールを防いでコストを削減する
  const [cache, setCache] = useState<Partial<Record<ExplanationLevel, CodeAnalysisResult>>>({});

  useEffect(() => {
    loadAnalysis(level);
  }, [level]);

  const loadAnalysis = async (targetLevel: ExplanationLevel) => {
    // キャッシュがあれば再APIコールしない
    if (cache[targetLevel]) {
      setResult(cache[targetLevel]\!);
      return;
    }

    setIsLoading(true);
    setError(null);

    try {
      const analysisResult = await analyzeCodeImage(
        imageBase64,
        targetLevel,
        process.env.EXPO_PUBLIC_GEMINI_API_KEY\!
      );
      setResult(analysisResult);
      setCache((prev) => ({ ...prev, [targetLevel]: analysisResult }));
    } catch (err) {
      const message = err instanceof Error ? err.message : '解析に失敗しました';
      setError(message);
    } finally {
      setIsLoading(false);
    }
  };

  const handleSave = async () => {
    if (\!result) return;
    await saveToHistory({ result, level, timestamp: Date.now() });
    Alert.alert('保存しました', '解析結果を履歴に追加しました。');
  };

  return (
    <ScrollView style={styles.container} contentContainerStyle={styles.content}>
      {/* 解説レベル切り替えタブ */}
      <View style={styles.levelTabs}>
        {(Object.keys(LEVEL_LABELS) as ExplanationLevel[]).map((l) => (
          <TouchableOpacity
            key={l}
            style={[styles.tab, level === l && styles.activeTab]}
            onPress={() => setLevel(l)}
            disabled={isLoading}
          >
            <Text style={[styles.tabText, level === l && styles.activeTabText]}>
              {LEVEL_LABELS[l]}
            </Text>
          </TouchableOpacity>
        ))}
      </View>

      {isLoading ? (
        <View style={styles.loadingContainer}>
          <ActivityIndicator size="large" color="#4CAF50" />
          <Text style={styles.loadingText}>AIが解析中です...</Text>
          <Text style={styles.loadingSubtext}>通常3〜5秒で完了します</Text>
        </View>
      ) : error ? (
        <View style={styles.errorContainer}>
          <Text style={styles.errorText}>{error}</Text>
          <TouchableOpacity style={styles.retryButton} onPress={() => loadAnalysis(level)}>
            <Text style={styles.retryText}>もう一度試す</Text>
          </TouchableOpacity>
          <TouchableOpacity style={styles.backButton} onPress={() => router.back()}>
            <Text style={styles.backText}>撮り直す</Text>
          </TouchableOpacity>
        </View>
      ) : result ? (
        <View style={styles.resultContainer}>
          <View style={styles.languageBadge}>
            <Text style={styles.languageText}>{result.language}</Text>
          </View>
          <Text style={styles.explanation}>{result.explanation}</Text>

          {result.keyPoints.length > 0 && (
            <View style={styles.section}>
              <Text style={styles.sectionTitle}>ポイント</Text>
              {result.keyPoints.map((point, i) => (
                <Text key={i} style={styles.keyPoint}>• {point}</Text>
              ))}
            </View>
          )}

          {result.potentialIssues && result.potentialIssues.length > 0 && (
            <View style={styles.issuesSection}>
              <Text style={styles.sectionTitle}>注意点・改善点</Text>
              {result.potentialIssues.map((issue, i) => (
                <Text key={i} style={styles.issue}>⚠ {issue}</Text>
              ))}
            </View>
          )}

          <TouchableOpacity style={styles.saveButton} onPress={handleSave}>
            <Text style={styles.saveButtonText}>履歴に保存する</Text>
          </TouchableOpacity>
        </View>
      ) : null}
    </ScrollView>
  );
}
```

レベル切り替え時のキャッシュ処理が設計のポイントです。同じ画像に対して3レベル全てのAPIを呼び出すと、Gemini APIのコストが3倍になります。このキャッシュ設計により、一度解析した画像はAPI再コールなしで別レベルの解説に切り替えられます。プレミアムユーザーが3レベルを行き来しても、コストは1回分で済みます。

## よくある落とし穴3選と実際の回避方法

実際に実装・テストすると、必ずといっていいほど詰まるポイントがあります。公式ドキュメントには書かれていない落とし穴です。

### 落とし穴1: 暗い場所や低コントラストで認識率が下がる

Gemini Flash はGPT-4oと比べて画像品質への感度が高いです。白地に黒文字のコードは認識率が90%以上ですが、ダークテーマのIDEのスクリーンショット（暗い背景に明るい文字）では70%を下回る場合があります。

実用的な対策として、撮影画面にガイダンスを表示するだけで効果があります。「白い紙や明るい画面のコードを撮影すると認識精度が上がります」というヒントを、カメラ画面に常時表示してください。ユーザーが自然に最適な条件で撮影するよう誘導できます。

### 落とし穴2: 長いコードが途中で途切れる

`maxOutputTokens: 1024` の設定では、非常に長いコードは解説が途中で切れます。かといって増やすと応答時間とコストが増加します。

推奨する設計は「部分選択UI」を実装することです。スクロールして「ここからここまでを解析」と範囲指定できるUIを追加すると、長いコードファイルから重要な部分だけを解析できます。これはUX改善と同時に、トークン使用量の削減にもなります。実装は `react-native-gesture-handler` の `LongPressGestureHandler` を使うと実現できます。

### 落とし穴3: 環境変数の扱いによるAPIキー漏洩リスク

`process.env.EXPO_PUBLIC_GEMINI_API_KEY` はビルド時にクライアントバンドルに埋め込まれます。Expo アプリは逆コンパイルすると環境変数の値を取り出せるため、セキュリティリスクがあります。

プロトタイプ段階では問題ありませんが、本番リリース後に1000人以上のユーザーがいる場合は、Cloudflare Workers や Supabase Edge Functions をAPIプロキシとして実装することをおすすめします。クライアントからは自分のサーバーに画像を送り、サーバーがGemini APIを呼び出す構成にすることで、APIキーがクライアントに渡らなくなります。このガイドの段階では方式Aで進め、スケールしてから移行するのが現実的です。

## RevenueCat でサブスクリプションを実装する

無料プラン（1日3回）とプレミアムプラン（無制限）の課金設計を実装します。

まず RevenueCat のダッシュボードで以下を設定します。

- Offering: `default`
- Package: `$rc_monthly`（月額プラン）
- Product ID（iOS）: `monthly_premium`
- Product ID（Android）: `monthly_premium`
- Price: 日本 ¥580、米国 $4.99

次に、使用回数管理のユーティリティを実装します。

```typescript
// utils/usageManager.ts — 無料プランの使用回数管理
import AsyncStorage from '@react-native-async-storage/async-storage';

const USAGE_KEY = 'daily_usage';
const FREE_DAILY_LIMIT = 3;

interface DailyUsage {
  count: number;
  date: string; // YYYY-MM-DD 形式
}

export async function canAnalyze(): Promise<{
  allowed: boolean;
  remaining: number;
  isNewDay: boolean;
}> {
  const today = new Date().toISOString().split('T')[0]; // "2026-04-18"
  const stored = await AsyncStorage.getItem(USAGE_KEY);

  let usage: DailyUsage = stored
    ? JSON.parse(stored)
    : { count: 0, date: today };

  // 日付が変わっていればリセット
  const isNewDay = usage.date \!== today;
  if (isNewDay) {
    usage = { count: 0, date: today };
  }

  const allowed = usage.count < FREE_DAILY_LIMIT;
  return { allowed, remaining: FREE_DAILY_LIMIT - usage.count, isNewDay };
}

export async function incrementUsage(): Promise<void> {
  const today = new Date().toISOString().split('T')[0];
  const stored = await AsyncStorage.getItem(USAGE_KEY);
  let usage: DailyUsage = stored
    ? JSON.parse(stored)
    : { count: 0, date: today };

  if (usage.date \!== today) {
    usage = { count: 0, date: today };
  }
  usage.count += 1;
  await AsyncStorage.setItem(USAGE_KEY, JSON.stringify(usage));
}
```

RevenueCat の購入確認と解析実行を連携させる核となるロジックです。

```typescript
// utils/subscription.ts
import Purchases, { PURCHASES_ERROR_CODE } from 'react-native-purchases';
import { canAnalyze, incrementUsage } from './usageManager';

export async function isPremiumUser(): Promise<boolean> {
  try {
    const customerInfo = await Purchases.getCustomerInfo();
    return customerInfo.entitlements.active['premium'] \!== undefined;
  } catch {
    // RevenueCat接続失敗時は無料プランとして扱う
    return false;
  }
}

// 解析前に呼び出す：プレミアムか、無料残回数があるかを確認
export async function checkAnalysisPermission(): Promise<{
  canProceed: boolean;
  reason: 'premium' | 'free_allowed' | 'limit_reached';
}> {
  const isPremium = await isPremiumUser();
  if (isPremium) {
    return { canProceed: true, reason: 'premium' };
  }

  const { allowed } = await canAnalyze();
  if (allowed) {
    await incrementUsage();
    return { canProceed: true, reason: 'free_allowed' };
  }

  return { canProceed: false, reason: 'limit_reached' };
}

export async function purchasePremium(): Promise<boolean> {
  try {
    const offerings = await Purchases.getOfferings();
    const monthlyPackage = offerings.current?.monthly;

    if (\!monthlyPackage) {
      throw new Error('プランが見つかりませんでした');
    }

    const { customerInfo } = await Purchases.purchasePackage(monthlyPackage);
    return customerInfo.entitlements.active['premium'] \!== undefined;
  } catch (error: any) {
    // ユーザーが自分でキャンセルした場合はエラー扱いにしない
    if (error.code === PURCHASES_ERROR_CODE.PURCHASE_CANCELLED_ERROR) {
      return false;
    }
    throw error;
  }
}
```

この実装で重要なのは `PURCHASE_CANCELLED_ERROR` の処理です。ユーザーが購入画面をキャンセルした場合、エラーとして扱わずに静かに処理を終了します。これをしないと、キャンセルのたびにエラーアラートが表示されてしまいます。

## App Store 審査でスムーズに通過するための3ポイント

カメラとサブスクリプションの両方を使うアプリは、審査でチェックされやすい類型です。

**ポイント1: カメラ使用目的の具体的な説明**

`app.json` の `infoPlist` に以下を設定します。

```json
{
  "expo": {
    "ios": {
      "infoPlist": {
        "NSCameraUsageDescription": "コードを撮影してAI解析するために使用します。撮影した画像はGemini AI（Google）に送信されて解析されます。画像はサーバーに保存されることはありません。"
      }
    }
  }
}
```

「カメラを使用します」という説明では審査を通過しない場合があります。「何のために使い」「どこに送られ」「どこには送られないか」を明示することが重要です。

**ポイント2: プライバシーポリシーへのGemini API記載**

AIサービスへの画像送信を行うアプリは、プライバシーポリシーに「第三者サービスへのデータ送信」の記載が必須です。審査チームはプライバシーポリシーのURLを確認し、Gemini（Google）へのデータ送信が記載されているかを確認します。

**ポイント3: 購入復元ボタンの必須設置**

サブスクリプションアプリで最も多いリジェクト理由の一つが「購入復元ボタンの欠如」です。RevenueCat の `Purchases.restorePurchases()` を呼び出すボタンを、ペイウォール画面に必ず設置してください。「以前購入した方はこちら」というラベルでも問題ありません。

このガイドで、カメラ撮影→マルチモーダルAI解析→段階別解説→課金という一連のフローを持つアプリの実装パターンを習得できました。

次にやることは一つです。まず最小構成（カメラ画面 + Gemini API統合 + 結果表示）を Rork Max で生成し、自分のiPhoneで動作確認してください。RevenueCatの設定はTestFlightに出せる状態になってから追加するのが効率的です。
