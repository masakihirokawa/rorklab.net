## Rork Max を個人開発者目線で評価してみました

2026年初頭に公開された Rork Max は、個人アプリ開発者にとって「ついにここまで来たか」と感じる進化でした。これまでの Rork が React Native ベースだったのに対し、Rork Max は **SwiftUI のネイティブコードを生成**します。iPhone だけでなく iPad や Vision Pro にも対応し、Xcode を開かずに App Store Connect への提出までできるため、Mac を持たない個人開発者でも iOS アプリを公開できる環境が整いました。

私自身、過去12年間で40本以上の iPhone アプリを個人で開発・公開してきました。その経験から言えば、Rork Max は「Xcode の代わりになる」わけではありませんが、**「Xcode を開かなくて良い時間を大幅に増やしてくれる」**ツールとしては本物です。この記事では、実際に Rork Max で SwiftUI アプリを作って App Store に公開するまでの流れと、向き不向き、組み込みで詰まるポイントをお伝えします。

## Rork Max と従来の Rork の決定的な違い

まず技術スタックの違いを整理します。

**従来の Rork**：React Native + Expo でアプリを生成します。iOS と Android を同一コードベースでカバーできる反面、ネイティブ API へのアクセスは Expo の対応状況に左右されます。Vision Pro 対応は限定的で、ネイティブに近いパフォーマンスを要求される処理は苦手です。

**Rork Max**：SwiftUI のネイティブコードを生成します。iOS ネイティブの全 API にアクセスでき、Vision Pro（visionOS 3）にも一級市民として対応します。ただし Android 対応はありません。生成されるのは Swift パッケージとして完全に動作する Xcode プロジェクトで、必要ならローカルの Xcode で開いて編集することもできます。

個人開発者としての判断基準はシンプルです。

- **iOS のみで、高品質・ネイティブ機能フル活用**を目指すなら Rork Max
- **iOS と Android を同時リリース**したいなら従来の Rork
- **既に Xcode で開発を始めているが加速したい**なら Rork Max（Swift コードが出力されるので既存プロジェクトに取り込みやすい）

私の個人開発の主戦場は「iOS に特化した高品質な単機能アプリ」なので、Rork Max が出てからは新規プロジェクトはすべて Rork Max に切り替えました。

## プロジェクト生成の実際の流れ

Rork Max を開くと、通常の Rork とほぼ同じチャット UI が表示されます。プロジェクトタイプの選択で「Rork Max (SwiftUI Native)」を選ぶだけで、以降は SwiftUI 前提のコード生成モードに切り替わります。

最初に送るプロンプトは、できるだけ **具体的な画面仕様** を含めるのがコツです。例えば「タスク管理アプリを作って」ではなく、以下のように書きます。

> iOS 17+ 対応のタスク管理アプリを作成してください。
> - タブ1: 今日のタスク一覧（チェックボックス + タイトル + 優先度ラベル）
> - タブ2: 全タスク一覧（カテゴリ別にセクション分け）
> - タブ3: 設定（通知時刻、テーマ切替）
> - データはSwiftData で永続化
> - ダークモード完全対応
> - iPad では NavigationSplitView レイアウト

このレベルの具体性で指示すると、1回目の生成でほぼ動作する SwiftUI コードが出てきます。iPhone と iPad の両方に対応したレイアウトが、`NavigationSplitView` と `NavigationStack` を使い分ける形で自動生成されるのは気持ちが良いです。

## Rork Companion アプリでプレビューする

Rork Max の強みは **Rork Companion アプリ**との連携です。iPhone や iPad に Companion アプリをインストールしておくと、Rork Max で生成中のアプリをリアルタイムでプレビューできます。

Companion でのプレビューの良いところは、以下の3点です。

- **実機のタッチ操作感を確認できる**：シミュレーターと違って、スクロールの慣性やハプティックフィードバックが本物です
- **Touch ID / Face ID・通知・位置情報などの実機限定機能が動く**：権限ダイアログも本物が出ます
- **複数デバイスで同時プレビュー**：iPhone と iPad で同じ変更が即座に反映されます

以前は「シミュレーターで動いたのに実機で挙動が違う」というバグに悩まされていましたが、Companion 経由だと最初から実機で動かしているので、そのギャップがほぼ消えます。

## 2クリック App Store 公開フロー

Rork Max で本当に驚くのが、**App Store への提出が Rork の画面内で完結する**点です。

公開フローは以下の通りです。

1. プロジェクト設定画面で App Store Connect アカウントを連携
2. アプリのメタデータ（タイトル、説明、スクリーンショット）を入力
3. 「Submit to App Store」ボタンを押す

これだけで、バックグラウンドで Rork のクラウド Mac がコンパイル・署名・アップロードまでを行い、App Store Connect に提出されます。スクリーンショットすら Rork Max が自動生成してくれます（もちろん手動アップロードも可能です）。

ただし、実際に使ってみていくつか注意点があります。

**Apple Developer Program の年間費用（$99）は当然必要です**。Rork Max はコンパイルと提出の自動化までは肩代わりしますが、Apple との契約関係は開発者本人のものです。

**審査リジェクトは自分で対応する必要があります**。Apple のレビューチームから Metadata Rejection や Guideline Violation の指摘が来た場合、Rork Max のチャットで「〇〇という指摘が来たので修正して」と伝えれば修正コードは生成されますが、**なぜその指摘が来たのかを理解するのは開発者側の責任**です。

**複雑なエンタイトルメント（プッシュ通知・HealthKit・アプリグループなど）は Rork 経由では設定できない場合があります**。このときは生成された Xcode プロジェクトをローカルで開き、Xcode 上で Capabilities タブから設定する必要があります。

## ネイティブ機能を組み込む実務手順

Rork Max の真価は、iOS ネイティブ機能を自然言語だけで組み込めることです。実務でよく使う機能の組み込み指示例を共有します。

**Vision Framework（OCR・画像認識）**：
> カメラで撮影した画像からテキストを抽出する機能を追加してください。Vision Framework の VNRecognizeTextRequest を使い、日本語と英語の両方を認識対象にしてください。結果はクリップボードにもコピーできるようにしてください。

この指示で、カメラ起動・撮影・OCR 処理・結果表示までが実装されたコードが生成されます。

**Core ML（機械学習モデルの組み込み）**：
> Core ML モデルを使って画像分類する機能を追加してください。モデルは Apple の公式モデル（MobileNetV2）を使ってください。結果はトップ3のカテゴリとその信頼度を表示してください。

Rork Max は Apple が公開しているモデルの組み込み手順を理解しているので、ダウンロードから推論までの流れをワンショットで生成できます。

**StoreKit 2（アプリ内課金）**：
> StoreKit 2 を使って月額サブスクリプションを実装してください。プロダクトID は "com.example.app.premium.monthly" です。購入状態は UserDefaults ではなく StoreKit の Transaction API で常に確認する形にしてください。

StoreKit 2 は以前の StoreKit より遥かに扱いやすくなりましたが、それでも正しく実装するには知識が要ります。Rork Max は最新のベストプラクティス（Transaction.currentEntitlements の使い方、Server-to-server 通知との連携など）をコードに反映してくれます。

## Rork Max が向かないケース

正直にお伝えすると、Rork Max は万能ではありません。以下のケースでは従来の Xcode 開発の方が適しています。

**高度にパフォーマンスチューニングが必要なゲーム**：Metal や SpriteKit を本格的に使う場合、細かい調整は人間の手が必要です。Rork Max で生成されたコードは「動く」レベルには達しますが、「60FPS を安定して出す」ためのプロファイリングは開発者が行う必要があります。

**サードパーティ SDK を多用するアプリ**：Firebase・RevenueCat・Mixpanel などの主要 SDK は対応していますが、マイナーな SDK や、Objective-C ベースの古いライブラリは Rork Max が認識しないことがあります。

**既存の大規模 Xcode プロジェクトへの追加開発**：Rork Max は新規プロジェクト生成が得意で、既存プロジェクトへの機能追加は期待通りに動かないことがあります。

**規制の厳しい業務アプリ（金融・医療）**：コードのトレーサビリティや監査要件が厳しい業界では、生成 AI のブラックボックス生成は受け入れられにくいです。

## これから始める個人開発者へ

個人開発者にとっての Rork Max の価値は、**「アプリを出すまでの心理的ハードルが劇的に下がる」**ことです。

以前の私なら、新しいアイデアを思いついても、「Xcode 立ち上げて、プロジェクト作って、レイアウト組んで…」という準備工数を考えると、試作品に到達する前に興味が冷めることがありました。Rork Max なら、アイデアを思いついた 30 分後にはプロトタイプが動き、2時間後には Companion で実機プレビューができ、その日のうちに App Store 申請までできてしまいます。

もちろん、App Store で成功するアプリを作るには、デザイン・UX・プロモーション・ユーザーフィードバックへの対応など、ツールだけでは解決しないことがたくさんあります。でも「アイデアを形にするまでの時間」が短縮されれば、その後の成長のためのサイクルも早く回せるようになります。

焦らず、まずは1本、小さなアプリを Rork Max で最後まで作って App Store に出してみることをおすすめします。そこで得られる経験は、他のどんなチュートリアルよりも価値があります。
