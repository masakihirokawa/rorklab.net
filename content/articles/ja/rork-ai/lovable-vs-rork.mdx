## Lovable と Rork — 何が違うのか

2026年の AI アプリビルダー市場において、**Lovable** と **Rork** は双璧をなす存在です。どちらも「AIを使ってノーコード/ローコードでアプリを作る」というコンセプトですが、対象プラットフォームと哲学が異なります。

## 基本比較

| 項目 | Lovable | Rork |
|------|---------|------|
| 主なプラットフォーム | **Webアプリ** | **モバイルアプリ (iOS/Android)** |
| 技術スタック | React + Supabase | React Native (Expo) |
| ターゲットユーザー | Web開発者・スタートアップ | モバイルアプリ志望者 |
| 月額料金 | $20〜 | $25〜 |
| 無料プラン | あり（制限あり） | なし |
| App Store 提出 | 非対応 | 対応 |

## Lovable の強み

### Web ファーストのUI構築

Lovable は Web アプリの構築に特化しており、React コンポーネントを素早く生成する能力に優れています。Supabase との統合がネイティブで、データベース設計からフロントエンドまで一貫して作れます。

### GitHub 連携

生成したコードを GitHub に同期できるため、開発者が後からカスタマイズしやすい設計です。

### デプロイの簡単さ

Lovable で作ったWebアプリは、ワンクリックで公開できます。独自ドメインの設定も簡単です。

## Rork の強み

### ネイティブモバイル対応

iOS と Android 向けのネイティブアプリを作れるのは Rork 独自の強みです。Lovable では不可能な、実際の App Store / Google Play への提出が可能です。

### Rork Max のデバイス API 活用

Rork Max（上位プラン）では、カメラ、GPS、プッシュ通知、HealthKit など、モバイル特有の機能を活用したアプリを作れます。

### Fix Now 機能

Rork 独自の「Fix Now」機能は、バグを自動で検出して修正提案を行います。モバイルアプリ特有のランタイムエラーに強いです。

## どちらを選ぶべきか

**Lovable を選ぶべき人**:
- Webアプリ・SaaS プロダクトを作りたい
- コードを後から自分でカスタマイズしたい
- Supabase ベースのフルスタックアプリを作りたい
- 無料で始めたい

**Rork を選ぶべき人**:
- iOS/Android ネイティブアプリを作りたい
- App Store / Google Play にリリースしたい
- モバイル特有の機能（カメラ・GPS・通知）を使いたい
- モバイルアプリの開発経験がない

## 両者の共通点

- AI によるコード生成
- リアルタイムプレビュー
- 非エンジニアでも使えるインターフェース
- 継続的なアップデートと機能追加

## まとめ

Lovable と Rork は競合というより、**補完関係**にあります。Web アプリを作るなら Lovable、モバイルアプリを作るなら Rork が適切な選択です。「どちらが優れているか」ではなく、「何を作りたいか」で選びましょう。
