## はじめに（なぜ今、自律運用が必要か）

個人でRorkアプリを複数運営していると、毎日の運用作業が積み重なり、開発の時間が圧迫されます。クラッシュ報告の確認・ユーザーレビューへの返信・プッシュ通知の設定・コンテンツの更新…。これらをAIエージェントに委任できたら、あなたはアイデアと設計に集中できます。

この記事では、Rorkアプリの自律運用を実現する4つのAIエージェントの設計と実装を詳しく解説します。

Rork MaxのAI基本機能については[Rork Max AI機能ガイド](/articles/rork-ai/rork-max-ai-features)をご覧ください。

## 自律運用システムのアーキテクチャ概要

自律運用システムは4つのエージェントで構成されます。

- **クラッシュ検知エージェント**: 毎時Sentryを監視 → 重大クラッシュを検知 → Claude APIで原因分析 → GitHub Issueを自動作成
- **コンテンツ更新エージェント**: 毎日スケジュール実行 → ユーザー行動データを分析 → AIで新コンテンツ生成 → Supabaseに自動投入
- **ユーザー行動分析エージェント**: 週次でApp Store Connectメトリクスを集計 → AIで改善提案レポートを生成
- **通知パーソナライズエージェント**: ユーザーセグメントを判定 → AIで最適な通知文を生成 → 最適時刻に配信

全体アーキテクチャのデータフロー:

```
Rorkアプリ → Supabase（データ蓄積） → 各AIエージェント（分析・生成）
                                      ↓
                          GitHub Issues / プッシュ通知 / コンテンツDB
```

## エージェント1: クラッシュ検知と自動デバッグ

### Sentry連携とAI自動分析

```javascript
// クラッシュ検知エージェント: Sentry API → Claude APIで自動分析
const Sentry = require('@sentry/node');
const Anthropic = require('@anthropic-ai/sdk');

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY
});

async function analyzeCrash(crashEvent) {
  // クラッシュのスタックトレースを取得
  const stackTrace = crashEvent.exception?.values?.[0]?.stacktrace?.frames
    ?.map(f => `${f.filename}:${f.lineno} - ${f.function}`)
    ?.join('\n') || 'スタックトレース不明';

  // Claude APIで原因を分析
  const analysis = await anthropic.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 500,
    messages: [{
      role: 'user',
      content: `以下のReact NativeアプリのクラッシュレポートをJSON形式で分析してください。

スタックトレース:
${stackTrace}

エラーメッセージ: ${crashEvent.exception?.values?.[0]?.value || '不明'}

JSON形式で回答: {"root_cause": "原因", "fix_suggestion": "修正方法", "severity": "critical|high|medium"}`
    }]
  });

  return JSON.parse(analysis.content[0].text);
}

async function createGitHubIssue(crash, analysis) {
  // GitHub APIで自動Issue作成
  const response = await fetch(
    'https://api.github.com/repos/YOUR_USERNAME/YOUR_REPO/issues',
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.GITHUB_TOKEN}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        title: `[AI検知] ${crash.exception?.values?.[0]?.type}: ${analysis.severity}`,
        body: `**根本原因**: ${analysis.root_cause}\n\n**修正提案**: ${analysis.fix_suggestion}`,
        labels: ['bug', 'ai-detected', analysis.severity]
      })
    }
  );

  const issue = await response.json();
  console.log(`Issue作成: #${issue.number}`);
  // 期待する出力: Issue作成: #42
  return issue;
}
```

## エージェント2: コンテンツの自動更新

Supabaseと連携して、AIがアプリ内コンテンツを自動更新するシステムを構築します。

```javascript
// コンテンツ更新エージェント
const { createClient } = require('@supabase/supabase-js');
const Anthropic = require('@anthropic-ai/sdk');

const supabase = createClient(
  process.env.SUPABASE_URL,
  process.env.SUPABASE_SERVICE_KEY
);
const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

async function generateDailyContent(category) {
  // 過去7日間の人気コンテンツを分析
  const { data: popularContent } = await supabase
    .from('content_views')
    .select('content_id, view_count, category')
    .eq('category', category)
    .order('view_count', { ascending: false })
    .limit(5);

  const topTopics = popularContent.map(c => c.content_id).join(', ');

  // AIで新コンテンツを生成
  const generated = await anthropic.messages.create({
    model: 'claude-haiku-4-5-20251001', // コスト最適化のためHaikuを使用
    max_tokens: 800,
    messages: [{
      role: 'user',
      content: `カテゴリ「${category}」の新しいコンテンツを生成してください。

人気コンテンツID（参考）: ${topTopics}

JSON形式で: {"title": "タイトル", "body": "本文（200文字以内）", "tags": ["タグ1", "タグ2"]}`
    }]
  });

  const content = JSON.parse(generated.content[0].text);

  // Supabaseに保存
  const { data, error } = await supabase
    .from('contents')
    .insert({
      ...content,
      category,
      created_by: 'ai-agent',
      status: 'pending_review' // 人間がレビュー後に公開
    });

  if (error) throw error;
  console.log(`コンテンツ生成・保存完了: ${content.title}`);
  // 期待する出力: コンテンツ生成・保存完了: AIが選ぶ今日のヘルスチップ

  return data;
}
```

## エージェント3: ユーザー行動分析と改善提案

```javascript
// 週次レポート生成エージェント
async function generateWeeklyReport(appId) {
  // App Store ConnectからメトリクスをAPIで取得
  const metrics = await fetchAppStoreMetrics(appId);

  // 前週比の計算
  const retentionChange = metrics.d7Retention - metrics.prevD7Retention;
  const revenueChange = metrics.weeklyRevenue - metrics.prevWeeklyRevenue;

  // AIで改善提案を生成
  const report = await anthropic.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 1000,
    messages: [{
      role: 'user',
      content: `以下のアプリメトリクスを分析して、改善提案を3つ提供してください。

DAU: ${metrics.dau} (前週比: ${metrics.dauChange > 0 ? '+' : ''}${metrics.dauChange}%)
D7リテンション: ${metrics.d7Retention}% (変化: ${retentionChange > 0 ? '+' : ''}${retentionChange}%)
週次収益: ¥${metrics.weeklyRevenue.toLocaleString()} (変化: ${revenueChange > 0 ? '+' : ''}${revenueChange}%)
クラッシュ率: ${metrics.crashRate}%

JSON形式で: {"improvements": [{"area": "エリア", "action": "具体的アクション", "priority": "high|medium|low"}]}`
    }]
  });

  const suggestions = JSON.parse(report.content[0].text);
  console.log(`改善提案 ${suggestions.improvements.length}件を生成しました`);
  // 期待する出力: 改善提案 3件を生成しました
  return suggestions;
}
```

## エージェント4: プッシュ通知のパーソナライズ

```javascript
// 通知パーソナライズエージェント
async function sendPersonalizedNotification(userId, appData) {
  // ユーザーの行動パターンを取得
  const { data: userProfile } = await supabase
    .from('user_behaviors')
    .select('*')
    .eq('user_id', userId)
    .single();

  // ユーザーセグメントを判定
  const segment = classifyUser(userProfile);

  // AIで最適な通知文を生成
  const notification = await anthropic.messages.create({
    model: 'claude-haiku-4-5-20251001',
    max_tokens: 100,
    messages: [{
      role: 'user',
      content: `ユーザーセグメント「${segment}」向けのプッシュ通知を生成。

アプリ: ${appData.name}
ユーザー最終ログイン: ${userProfile.last_login_days_ago}日前

JSON形式で: {"title": "通知タイトル（20字以内）", "body": "本文（50字以内）"}`
    }]
  });

  const { title, body } = JSON.parse(notification.content[0].text);

  // Expo Push Notifications APIで送信
  await sendPushNotification(userProfile.push_token, title, body);

  console.log(`通知送信完了: ${title}`);
  // 期待する出力: 通知送信完了: 久しぶりです！新機能が追加されました
}

function classifyUser(profile) {
  if (profile.last_login_days_ago > 30) return 'churned_risk';
  if (profile.daily_sessions > 3) return 'power_user';
  if (profile.is_paid_user) return 'premium_user';
  return 'regular_user';
}
```

## コスト管理と安全策

AIエージェントが暴走しないための設計パターン:

- **人間によるレビューゲート**: 自動生成コンテンツは`status: 'pending_review'`で保存。公開は人間が確認後に手動実行
- **レート制限**: 1時間あたりのAPI呼び出し回数を上限設定（例: max 100 calls/hour）
- **ドライランモード**: 本番実行前に`DRY_RUN=true`で出力のみ確認
- **アラート閾値**: API費用が日次予算の80%に達したらSlack通知

```javascript
// 安全なエージェント実行ラッパー
async function safeAgentRun(agentFn, options = {}) {
  const { dryRun = false, maxRetries = 3 } = options;

  if (dryRun) {
    console.log('[DRY RUN] エージェント実行をシミュレート中...');
    return null;
  }

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const result = await agentFn();
      console.log(`✅ エージェント実行成功 (試行: ${attempt}/${maxRetries})`);
      return result;
    } catch (error) {
      console.error(`❌ 試行 ${attempt}/${maxRetries} 失敗: ${error.message}`);
      if (attempt === maxRetries) throw error;
      await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
    }
  }
}
```

## 段階的な導入ロードマップ

個人開発者向けの段階的導入計画（週1〜2時間の作業時間を想定）:

- **Week 1**: クラッシュ検知エージェントのみ導入。Sentryの無料プランで十分
- **Week 2-3**: 週次レポート生成エージェントを追加。週1回だけ実行
- **Week 4-5**: コンテンツ更新エージェントの試験導入。週3件から開始
- **Week 6以降**: 通知パーソナライズエージェントの導入とチューニング

## 運用チェックリスト

実装を完了したら、以下の項目を確認してください。

- **エージェント実行ログの整備**: 全実行を監視可能なダッシュボードで表示
- **ロールバック手順の文書化**: AI生成コンテンツ削除・修正の方法を明記
- **定期的な品質監査**: 月1回、生成されたコンテンツをサンプルで人間が審査
- **コスト上限の設定**: Anthropicダッシュボードで月額上限を設定
- **エラーアラート設定**: エージェント実行失敗時にSlack通知

## よくある質問

**Q: 自律エージェントが予期しない変更を加えてしまった場合は？**

A: 全ての自動変更は`pending_review`ステータスで保存され、人間の確認なしには公開されません。また、Supabaseの`created_by: 'ai-agent'`フラグで自動生成コンテンツを識別し、一括削除・ロールバックも容易です。

**Q: このシステムの月額運用コストはどのくらいかかりますか？**

A: Sentry無料プラン + Supabase無料プラン + Claude Haiku APIで、1日あたり50〜100回のエージェント実行をしても月額$5〜15程度が目安です。コスト最適化の手法を適用すればさらに削減できます。

**Q: RorkアプリにSupabaseは必須ですか？**

A: 必須ではありませんが、Supabaseはリアルタイム機能とEdge Functionsを無料プランで使えるため、エージェントのバックエンドとして最適です。FirebaseやPlanetScaleでも同様のシステムを構築できます。

**Q: App Store ConnectのAPIアクセスにはApple開発者メンバーシップが必要ですか？**

A: はい、App Store Connect APIを使うにはApple Developer Program（年額$99）のメンバーシップが必要です。

**Q: AIエージェントのミスを検知する方法は？**

A: 全てのエージェント実行をSupabaseの`agent_logs`テーブルに記録し、エラー率・生成品質スコアを定期的に確認します。品質スコアが閾値を下回ったらSlackアラートを送る仕組みを組み込むことを推奨します。

**Q: クラッシュ検知エージェントをSentryの有料プランに変更する時期は？**

A: 月1万件以上のイベント（クラッシュ）が記録されたら、有料プラン（月$99～）への移行を検討してください。無料プランではイベント保持期間が短くなります。
