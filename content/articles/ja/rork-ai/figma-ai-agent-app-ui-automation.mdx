## はじめに

2026年3月24日、Figma が重大な発表をしました。**AI エージェントが Figma キャンバスを直接操作できるようになった**のです。

これまで、AI は「デザイン案をテキストで説明する」「CSS コードを提案する」という間接的な支援しかできていませんでした。しかし新機能により、AI エージェントが「フレームを作成する」「コンポーネントを配置する」「変数を設定する」という**デザイン操作をネイティブに実行**できるようになりました。

本記事では、Figma AIエージェント機能の全貌と、Rork でのアプリ開発への活用方法を解説します。

---

## Figma AIエージェント機能とは

### 発表内容

Figma が発表した新機能は以下の通りです：

1. **use_figma ツール**：AI エージェント向けの MCP（Model Context Protocol）サーバー
2. **Skills 機能**：マークダウン形式でデザイン操作をガイドする指示セット
3. **Code Connect 拡張**：デザインシステムとソースコードの紐付け強化

これらを組み合わせることで、開発チームと AI が同じキャンバス上で協働できるようになります。

### なぜ革新的なのか

従来のデザインツール連携：
```
開発者の要望
    ↓
デザイナーが手動操作
    ↓
デザイン完成
    ↓
開発者が実装
```

新しいワークフロー：
```
AI エージェント
    ↓
use_figma で直接操作
    ↓
デザイン自動生成
    ↓
Code Connect で即座にコード化
```

時間短縮は明らかです。しかも**デザインの一貫性も AI が保証**します。

## use_figma ツール詳細

### コア操作

use_figma ツールで利用可能な主要操作：

| 操作 | 説明 | 用途 |
|---|---|---|
| `createFrame()` | フレームを作成 | 画面レイアウトの定義 |
| `createComponent()` | コンポーネント定義 | ボタン、カード等の再利用部品 |
| `createVariant()` | コンポーネントバリアント | 状態（hover、disabled等）の管理 |
| `placeAsset()` | アセット（画像）を配置 | イラスト、アイコンの挿入 |
| `setVariable()` | 変数を設定 | 色、サイズ、字体の一元管理 |
| `applyToken()` | デザイントークンを適用 | ブランドガイドラインに準拠 |
| `createLayout()` | Auto Layout を設定 | レスポンシブデザイン実現 |

### 基本的な使用例

```typescript
// AI エージェント向けの Figma MCP 実装例
import { useFigma } from "@figma/ai-sdk";

async function generateLoginScreen() {
  const figma = useFigma({
    token: process.env.FIGMA_TOKEN,
    fileKey: "YOUR_FILE_KEY",
  });

  // フレームを作成
  const frame = await figma.createFrame({
    name: "Login Screen",
    width: 375,
    height: 667,
    x: 0,
    y: 0,
  });

  // ロゴを追加
  const logo = await figma.placeAsset({
    parentId: frame.id,
    assetUrl: "https://brand.com/logo.png",
    width: 120,
    height: 120,
    x: 127.5, // 中央配置
    y: 50,
  });

  // テキストフィールドコンポーネントを配置
  const emailInput = await figma.createComponent({
    parentId: frame.id,
    name: "Email Input",
    type: "text_input",
    x: 20,
    y: 200,
    width: 335,
    height: 48,
    placeholder: "Enter your email",
  });

  // パスワードフィールド
  const passwordInput = await figma.createComponent({
    parentId: frame.id,
    name: "Password Input",
    type: "text_input",
    x: 20,
    y: 270,
    width: 335,
    height: 48,
    placeholder: "Enter your password",
    isSecure: true,
  });

  // ログインボタン
  const loginButton = await figma.createComponent({
    parentId: frame.id,
    name: "Login Button",
    type: "button",
    x: 20,
    y: 340,
    width: 335,
    height: 48,
    label: "Log In",
    backgroundColor: "#007AFF",
    textColor: "#FFFFFF",
  });

  // Auto Layout を有効化（レスポンシブ対応）
  await figma.createLayout({
    targetId: frame.id,
    mode: "vertical",
    spacing: 20,
    padding: 20,
  });

  console.log("✓ ログイン画面を自動生成しました");
  return frame;
}
```

## Skills 機能の活用法

### Skills とは

Figma の **Skills** は、AI エージェント向けの指示セット。マークダウン形式で記述し、AI エージェントの操作をガイドします。

### 実装例：ボタンコンポーネント群の自動生成

```markdown
# Button Component System

## Objective
Create a comprehensive button component system with variants for all interaction states.

## Requirements
- Component family: "Button"
- Variants: Primary, Secondary, Danger
- States: Default, Hover, Active, Disabled
- Sizes: Small (32px), Medium (48px), Large (56px)

## Step 1: Create Main Component
1. Create a Frame named "Button/Primary/Medium"
2. Set dimensions: Width 200, Height 48
3. Fill color: #007AFF (Primary Blue)
4. Border radius: 8px
5. Add text layer "Label" with font size 16, weight Bold

## Step 2: Create Variants
For each combination:
- Button/Primary/Small (32px height)
- Button/Primary/Medium (48px height)
- Button/Primary/Large (56px height)
- Button/Secondary/* (same heights, fill #E5E5EA)
- Button/Danger/* (same heights, fill #FF3B30)

## Step 3: Create States
For each variant, create these component states:
- Default: opacity 100%
- Hover: background darker by 10%
- Active: background darker by 20%
- Disabled: opacity 50%

## Step 4: Set Variables
Create design tokens:
- color-primary: #007AFF
- color-secondary: #E5E5EA
- color-danger: #FF3B30
- spacing-small: 8px
- spacing-medium: 16px

## Constraints
- Do NOT delete existing components
- Always maintain 8px grid alignment
- Ensure minimum contrast ratio WCAG AA
```

AI エージェントはこの Skills を読み込んで、ステップ by ステップで操作を実行します。

## Code Connect との連携

### デザイン ↔ コード の自動紐付け

Figma の **Code Connect** 機能により、デザインコンポーネントとソースコードを紐付けられます：

```typescript
// figma.config.ts
import { figma } from "@figma/code-connect";

figma.connect(
  "Button/Primary/Medium",
  {
    component: "./components/Button.tsx",
    props: {
      variant: figma.enum("Variant"),
      size: figma.enum("Size"),
      disabled: figma.boolean("Disabled"),
      label: figma.string("Label"),
    },
  },
);
```

これにより、AI エージェントが Figma でコンポーネントを作成すると、自動的にコードが生成される可能性が広がります。

## Rork でのアプリ開発との連携

### 想定される開発フロー

1. **要件定義**：「ユーザー認証画面を作成したい」
2. **AI エージェント実行**：Figma MCP を通じて UI を自動生成
3. **Code Connect 実行**：デザインからコンポーネント実装を抽出
4. **Rork 統合**：生成されたコンポーネントを Rork アプリに組み込む
5. **テスト＆修正**：AI エージェントによる自動テスト

### 実装例

```bash
# Rork プロジェクトで Figma MCP を有効化
rork init --with-figma-mcp

# AI エージェントを実行
rork agent run --task "Create a login screen in Figma and integrate into app"

# 生成されたデザイン＆コンポーネントを確認
rork figma review

# アプリにコンポーネントを統合
rork integrate figma:LoginScreen --target src/screens/
```

## FAQ

### Q1: Figma AIエージェント機能は有料か？

**A:** Figma Pro 以上のプランで利用可能です。2026年3月時点では、タブ数制限なしで use_figma ツールにアクセスできます。

### Q2:既存の Figma ファイルに対して AI エージェントを使える？

**A:** はい。既存ファイルに新しいページを追加して、AI エージェントにそのページ内で操作させることができます。既存デザインは保護されます。

### Q3: Code Connect で自動生成されたコンポーネント実装の品質は大丈夫か？

**A:** 基本構造は自動生成されますが、細かいビジネスロジックは手修正が必要です。テストカバレッジは AI エージェント（ハーネスエンジニアリング）が担当する想定です。
