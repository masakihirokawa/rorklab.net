## はじめに

Rork Max が提供する SwiftUI ネイティブ機能により、AI が生成するアプリの品質が劇的に向上しました。これまでのノーコードツールは主に React Native や クロスプラットフォーム向けのコードを生成していましたが、Rork Max は **ネイティブ iOS/macOS 開発向けの SwiftUI コードを直接生成できるようになったのです**。

**3. デザインシステムとの統合**

### コード品質のベンチマーク

```swift
// Rork Max が生成するフォームの例
import SwiftUI

@MainActor
final class FormViewModel: ObservableObject {
    @Published var email: String = ""
    @Published var password: String = ""
    @Published var isLoading: Bool = false
    @Published var errorMessage: String?

    // バリデーション
    var isFormValid: Bool {
        !email.isEmpty &&
        password.count >= 8 &&
        email.contains("@")
    }

    @MainActor
    func submit() async {
        isLoading = true
        defer { isLoading = false }

        do {
            // API呼び出し
            _ = try await AuthService.login(email: email, password: password)
        } catch {
            errorMessage = error.localizedDescription
        }
    }
}

struct FormView: View {
    @StateObject private var viewModel = FormViewModel()
    @Environment(\.dismiss) var dismiss

    var body: some View {
        NavigationStack {
            Form {
                Section("ログイン") {
                    TextField("メールアドレス", text: $viewModel.email)
                        .textInputAutocapitalization(.none)
                        .keyboardType(.emailAddress)

                    SecureField("パスワード", text: $viewModel.password)
                }

                if let errorMessage = viewModel.errorMessage {
                    Section {
                        Text(errorMessage)
                            .foregroundStyle(.red)
                    }
                }

                Section {
                    Button(action: {
                        Task {
                            await viewModel.submit()
                        }
                    }) {
                        if viewModel.isLoading {
                            ProgressView()
                        } else {
                            Text("ログイン")
                        }
                    }
                    .disabled(!viewModel.isFormValid || viewModel.isLoading)
                }
            }
            .navigationTitle("認証")
            .toolbar {
                ToolbarItem(placement: .cancellationAction) {
                    Button("キャンセル") { dismiss() }
                }
            }
        }
    }
}

#Preview {
    FormView()
}
```

このコード例は以下の特徴を持っています。

- `@MainActor` による UI スレッド安全性の確保
- async/await による非同期処理の適切な実装
- フォームバリデーション
- エラーハンドリング
- Previews による開発効率化

## カスタムUI とアニメーション

### Canvas と SwiftUI Modifiers の活用

Rork Max では、複雑なカスタム UI も生成可能です。

```swift
import SwiftUI

struct GlassmorphismCard: View {
    var title: String
    var subtitle: String

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text(title)
                .font(.headline)
                .foregroundStyle(.white)

            Text(subtitle)
                .font(.caption)
                .foregroundStyle(.white.opacity(0.7))
        }
        .frame(maxWidth: .infinity, alignment: .leading)
        .padding()
        .background(
            .ultraThinMaterial,
            in: RoundedRectangle(cornerRadius: 12)
        )
        .overlay(
            RoundedRectangle(cornerRadius: 12)
                .stroke(
                    .white.opacity(0.2),
                    lineWidth: 1
                )
        )
    }
}

struct AnimatedProgressView: View {
    @State private var progress: Double = 0

    var body: some View {
        VStack(spacing: 20) {
            CircularProgressView(progress: progress)
                .frame(height: 200)

            Button("Start") {
                withAnimation(.easeInOut(duration: 2)) {
                    progress = 1.0
                }
            }
        }
        .padding()
    }
}

struct CircularProgressView: View {
    let progress: Double

    var body: some View {
        ZStack {
            Circle()
                .stroke(.gray.opacity(0.3), lineWidth: 12)

            Circle()
                .trim(from: 0, to: progress)
                .stroke(
                    LinearGradient(
                        gradient: Gradient(colors: [.blue, .purple]),
                        startPoint: .topLeading,
                        endPoint: .bottomTrailing
                    ),
                    style: StrokeStyle(lineWidth: 12, lineCap: .round)
                )
                .rotationEffect(.degrees(-90))
                .animation(.easeInOut(duration: 0.5), value: progress)

            Text("\(Int(progress * 100))%")
                .font(.title2.bold())
        }
    }
}
```

### ジェスチャー操作とアニメーション遷移

```swift
struct DraggableCard: View {
    @State private var offset = CGSize.zero
    @State private var scale: CGFloat = 1.0

    var body: some View {
        RoundedRectangle(cornerRadius: 12)
            .fill(.blue.gradient)
            .frame(height: 300)
            .offset(offset)
            .scaleEffect(scale)
            .gesture(
                DragGesture()
                    .onChanged { value in
                        offset = value.translation
                    }
                    .onEnded { value in
                        withAnimation(.spring(response: 0.3, dampingFraction: 0.7)) {
                            if abs(value.translation.width) > 100 {
                                offset = CGSize(width: value.translation.width > 0 ? 500 : -500, height: 0)
                                scale = 0.8
                            } else {
                                offset = .zero
                                scale = 1.0
                            }
                        }
                    }
            )
    }
}
```

## Apple Watch と iPad への対応

### Apple Watch アプリ生成

Rork Max で指定する際に「Watch OS」オプションを選択すると、自動的に WatchKit フレームワーク対応のコードが生成されます。

```swift
// Apple Watch (watchOS 10+) 向けコード例
import SwiftUI

@main
struct RorkWatchApp: App {
    var body: some Scene {
        WindowGroup {
            NavigationStack {
                List {
                    NavigationLink("カウンター", value: "counter")
                    NavigationLink("タイマー", value: "timer")
                    NavigationLink("天気", value: "weather")
                }
                .navigationDestination(for: String.self) { destination in
                    Group {
                        switch destination {
                        case "counter":
                            CounterView()
                        case "timer":
                            TimerView()
                        case "weather":
                            WeatherView()
                        default:
                            Text("不明")
                        }
                    }
                }
            }
        }
    }
}

// コンパクトで効率的なカウンター
struct CounterView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("\(count)")
                .font(.title)
                .monospacedDigit()

            HStack {
                Button(action: { count -= 1 }) {
                    Image(systemName: "minus.circle.fill")
                }

                Button(action: { count += 1 }) {
                    Image(systemName: "plus.circle.fill")
                }
            }
        }
    }
}
```

### iPad と macOS への自動対応

Rork Max は、同じ SwiftUI コードから iPad / macOS 版を自動生成します。`@Environment(\.horizontalSizeClass)` や `@Environment(\.verticalSizeClass)` を活用します。

```swift
struct ResponsiveLayout: View {
    @Environment(\.horizontalSizeClass) var horizontalSizeClass

    var body: some View {
        Group {
            if horizontalSizeClass == .compact {
                // iPhone レイアウト
                VStack {
                    headerView()
                    contentView()
                }
            } else {
                // iPad / macOS レイアウト
                HStack {
                    sidebarView()
                    contentView()
                }
            }
        }
    }
}
```

## React Native との比較

| 項目 | Rork Max SwiftUI | React Native |
|---|---|---|
| **ネイティブ性能** | ネイティブ iOS API を直接使用。高速 | JavaScript ブリッジを経由するため若干遅延 |
| **プラットフォーム対応** | iOS/macOS/watchOS/tvOS | iOS/Android（コード記述が異なる） |
| **UI 品質** | Human Interface Guidelines 準拠 | Material Design 中心（カスタマイズ必要） |
| **バッテリー消費** | 最適化された Native 実装 | JavaScript エンジン起動により消費高め |
| **App Store 審査** | Native コードのため通常より審査が厳格 | ハイブリッド扱い（緩和される傾向） |
| **カスタマイズ** | Swift + Objective-C との融合が容易 | Bridge を経由する複雑さ |
| **学習曲線** | SwiftUI 知識が有利 | JavaScript/React 知識で対応可能 |

## よくある質問（FAQ）

### Q1: Rork Max で生成した SwiftUI コードを直接 Xcode で編集できますか？

**A:** はい、完全に可能です。むしろ Rork Max で生成したコードを基盤として、カスタム機能を追加する開発フローが想定されています。SwiftUI の標準構文を用いているため、チーム開発での統合も容易です。

### Q2: App Store 審査で「AI生成コード」であることを明記する必要がありますか？

**A:** いいえ。App Store ガイドラインでは、最終的なアプリの品質が基準を満たしていれば、開発方法は問われません。ただし、知的財産や規制に関わる機能は手動検証を推奨します。

### Q3: 既存の Swift Package（CocoaPods、SPM）との統合は？

**A:** Rork Max は SPM（Swift Package Manager）に対応しており、Package.swift への依存関係自動追加も可能です。CocoaPods は部分対応です。

### Q4: リアルタイム通信（WebSocket）への対応状況は？

**A:** URLSession の WebSocket API 対応により、リアルタイムチャット、ライブ配信通知、マルチプレイゲームなど実装可能です。ただし複雑な XMPP や Firebase Realtime は外部ライブラリとの統合が必要な場合があります。

### Q5: App Store 価格設定とサブスクリプション機能は？

**A:** StoreKit 2 対応により、アプリ内購入・サブスクリプション・ファミリー共有の実装が自動化されます。RevenueCat や Adapty との連携も可能です。
