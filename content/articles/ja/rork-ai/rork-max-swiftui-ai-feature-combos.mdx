Rork Max の AI 機能と SwiftUI 生成は、それぞれ単体での解説記事は数多く存在します。ただ、私が個人開発で実際に「これは生産性に効いた」と感じるのは、両者を組み合わせた実装パターンです。1 つ 1 つの機能の使い方ではなく、「同時に何と何を使うと、開発の摩擦が減るか」という視点でまとめます。

紹介する 7 つのコンボは、すべて自分のアプリ開発で効果を確認したものです。プロトタイプではなく、TestFlight や App Store に実際に出した範囲で評価しているため、再現性は比較的高いと思います。

## コンボ 1: 仕様書 AI 整理 × ホーム画面生成

仕様書がモヤッとしているとき、Rork Max の AI に「この仕様書から SwiftUI のホーム画面に必要な状態を 5 個列挙して」と依頼します。返ってきた状態リストをそのまま、ホーム画面生成のプロンプトに添えます。

このコンボの効用は、仕様書の曖昧さを「画面の状態」レベルまで具体化してから生成に進めることです。仕様書のまま生成に投げると、AI が勝手に状態を補完して、後から自分の意図とずれていることに気付くケースが多発します。

## コンボ 2: SwiftUI プレビュー生成 × エラー診断 AI

`#Preview` ブロックをまとめて生成させたあと、プレビューが落ちる場合に Rork Max の AI に「このエラーログから原因を特定して、修正パッチを SwiftUI コード内で示して」と依頼します。プレビューエラーは原因が型の不整合か環境変数の不足のどちらかが多く、AI に診断させると修正までが速いです。

```swift
// プレビューエラーが出やすいパターン
#Preview {
  ContentView()
    .environmentObject(AppState())   // ← これを忘れると即落ち
    .environment(\.locale, .init(identifier: "ja"))
}
```

## コンボ 3: フォーム自動生成 × バリデーション AI

入力フォーム画面を生成させると、見た目はキレイにできるのですが、バリデーションが浅いことがあります。生成後に AI に「このフォームに対するバリデーションを、ユーザー体験を損なわない順序で 5 個提案して」と聞きます。

「メールアドレスの形式チェック → 入力直後ではなくフォーカスアウト時 → エラー表示は赤テキストでなくフィールド下にグレーで」という具合に、UX 配慮を含んだ提案が返ってくることがあります。これを SwiftUI コードに反映する流れが、私の中で定着しつつあります。

## コンボ 4: アニメーション生成 × アクセシビリティ AI

凝ったアニメーションを生成させたあと、必ず AI に「このアニメーションが VoiceOver や Reduce Motion 設定でどう見えるかを評価して、必要なら修正案を出して」と問います。Apple のアクセシビリティ要件は App Store 審査でも見られるため、ここを後回しにすると申請直前に慌てます。

```swift
// Reduce Motion 対応の最小パターン
@Environment(\.accessibilityReduceMotion) var reduceMotion

var body: some View {
  Text("Hello")
    .scaleEffect(isExpanded ? 1.2 : 1.0)
    .animation(reduceMotion ? nil : .spring(), value: isExpanded)
}
```

## コンボ 5: ネットワーク層生成 × エラーハンドリング AI

API 通信のクライアントコードを生成させたあと、AI に「ネットワーク失敗時のユーザー体験を改善するための具体策を 3 つ」聞きます。リトライ戦略、オフラインキャッシュ、ユーザー向けエラーメッセージの 3 点セットが返ってくることが多く、これをそのまま SwiftUI 側のエラー表示に組み込みます。

私のアプリで実際に役立ったのは、「バックオフ付きリトライを 2 回 → それでも失敗ならキャッシュからフォールバック → キャッシュもなければユーザーに『電波の良い場所で再度お試しください』と表示」というフローです。

## コンボ 6: ローカライズ自動生成 × 文化的レビュー AI

`Localizable.xcstrings` を多言語化するとき、Rork Max に翻訳を依頼すると速いのですが、そのまま使うのは危険です。生成後に「この訳語が日本語ネイティブ視点でどう聞こえるか」「英語ネイティブ視点でどう聞こえるか」を AI に評価させます。

私が直近で気づいたのは、「Get Started」を「始める」と訳すか「はじめる」とひらがなにするかで、ユーザーの初動が変わるという話です。AI は両方の選択肢の印象を整理して提示してくれるので、最終判断の材料が増えます。

## コンボ 7: ストア説明文生成 × ASO キーワード AI

App Store 申請の段階で、Rork Max に「このアプリの説明文を 4000 文字以内で書いて」と依頼するのと同時に、別タブで「このアプリで狙うべき ASO キーワードを、競合の少ない順で 10 個」と聞きます。説明文と ASO キーワードを別々に作るのではなく、互いに参照しながら詰めるのが効率的です。

```
[ASO キーワード抽出プロンプト例]
このアプリの主要機能は X、Y、Z です。
日本語のApp Storeで、月間検索量がそこそこあり、かつ競合の少ない
キーワードを 10 個提案してください。
1. キーワード
2. 検索意図の推定
3. 競合度の所感（低/中/高）
の形式で出力してください。
```

## コンボを習慣化するための小さな工夫

これらのコンボは、単発で試しても効果は出にくいです。私は Notion の「Rork Max 運用ノート」というページに、コンボごとのテンプレプロンプトを貯めています。新しいプロジェクトを始めるときに、上から順に流していくだけで品質が一定以上に揃うようになりました。

プロンプトテンプレを資産として育てていく感覚は、Rork Max を使い込むうえで地味に重要です。最初は時間がかかりますが、3〜4 アプリ作る頃には、自分専用のレシピ集ができあがります。
