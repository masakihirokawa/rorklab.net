## はじめに — なぜ AI 名刺スキャナーを作るのか

ビジネスシーンで名刺を受け取った後、手動で連絡先を入力するのは面倒で、入力ミスも起きがちです。AI を活用した名刺スキャナーアプリがあれば、カメラで名刺を撮影するだけで、名前・会社名・メールアドレス・電話番号などを自動的に読み取って連絡先に登録できます。

この記事では、Rork を使って AI 名刺スキャナーアプリをゼロから構築する方法を解説します。具体的には、カメラによる撮影機能、OCR（光学文字認識）によるテキスト抽出、AI による構造化データ変換、そしてスマートフォンの連絡先への自動登録まで、すべての実装ステップを順を追って紹介します。

**この記事で学べること:**

- Expo Camera を使った名刺撮影機能の実装
- Google Cloud Vision API による OCR テキスト認識
- Claude API / Gemini API を使ったテキストの構造化
- Expo Contacts による連絡先への自動登録
- スキャン履歴の管理とUI設計のポイント

**対象読者:** Rork でアプリ開発の基本を理解しており、AI API を組み合わせた実用的なアプリを作りたい中級者向けです。

## カメラ撮影機能の実装

名刺スキャナーの入り口となるカメラ機能を実装します。`expo-camera` を使えば、Rork 上でもネイティブカメラにアクセスできます。

```typescript
// CameraScreen.tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useState, useRef } from 'react';
import { View, TouchableOpacity, Text, StyleSheet } from 'react-native';

export default function CameraScreen({ onCapture }: { onCapture: (uri: string) => void }) {
  const [permission, requestPermission] = useCameraPermissions();
  const cameraRef = useRef<CameraView>(null);

  // パーミッションの確認と要求
  if (\!permission?.granted) {
    return (
      <View style={styles.container}>
        <Text style={styles.message}>カメラへのアクセスを許可してください</Text>
        <TouchableOpacity style={styles.button} onPress={requestPermission}>
          <Text style={styles.buttonText}>許可する</Text>
        </TouchableOpacity>
      </View>
    );
  }

  const handleCapture = async () => {
    if (cameraRef.current) {
      // 高品質で撮影（OCR精度向上のため）
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.9,
        base64: true,
      });
      if (photo?.uri) {
        onCapture(photo.uri);
      }
    }
  };

  return (
    <View style={styles.container}>
      <CameraView ref={cameraRef} style={styles.camera} facing="back">
        {/* 名刺の位置ガイド */}
        <View style={styles.overlay}>
          <View style={styles.cardGuide} />
          <Text style={styles.guideText}>名刺を枠内に合わせてください</Text>
        </View>
      </CameraView>
      <TouchableOpacity style={styles.captureButton} onPress={handleCapture}>
        <Text style={styles.captureText}>撮影</Text>
      </TouchableOpacity>
    </View>
  );
}
// 期待する動作: カメラプレビューが表示され、
// 撮影ボタンを押すと画像URIがコールバックで返される
```

ポイントは、`quality: 0.9` を指定して高解像度で撮影することです。OCR の精度は画像品質に大きく依存するため、圧縮しすぎないように注意しましょう。また、名刺の位置を示すガイド枠を表示すると、ユーザーが正確に名刺を撮影しやすくなります。

さらに、`expo-image-picker` を使えば、既に撮影済みの名刺写真をギャラリーから選択する機能も追加できます。

```typescript
import * as ImagePicker from 'expo-image-picker';

const pickFromGallery = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ['images'],
    quality: 0.9,
    base64: true,
  });
  if (\!result.canceled && result.assets[0]) {
    onCapture(result.assets[0].uri);
  }
};
// 期待する動作: ギャラリーが開き、選択した画像のURIが返される
```

カメラ機能の詳しい実装パターンについては、[Rork Max カメラ＆ギャラリー実装ガイド](/articles/rork-dev/rork-max-camera-gallery) も参考にしてください。

## OCR テキスト認識の実装

撮影した名刺画像からテキストを抽出するために、Google Cloud Vision API の OCR 機能を利用します。Cloud Vision API は日本語と英語の混在テキストにも対応しており、名刺のスキャンに適しています。

```typescript
// services/ocrService.ts
import * as FileSystem from 'expo-file-system';

const VISION_API_ENDPOINT = 'https://vision.googleapis.com/v1/images:annotate';

export async function extractTextFromImage(imageUri: string): Promise<string> {
  // 画像をBase64に変換
  const base64Image = await FileSystem.readAsStringAsync(imageUri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // Cloud Vision API にリクエスト
  const response = await fetch(
    `${VISION_API_ENDPOINT}?key=YOUR_CLOUD_VISION_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        requests: [
          {
            image: { content: base64Image },
            features: [
              { type: 'TEXT_DETECTION', maxResults: 1 },
            ],
            imageContext: {
              languageHints: ['ja', 'en'],
            },
          },
        ],
      }),
    }
  );

  const data = await response.json();
  const fullText = data.responses?.[0]?.fullTextAnnotation?.text || '';
  return fullText;
}
// 期待する出力例:
// "株式会社テクノロジー\n田中 太郎\nプロダクトマネージャー\n
//  tanaka@example.com\n03-1234-5678\nwww.example.com"
```

**OCR 精度を上げるコツ:**

- 画像の解像度は 1000px 以上を確保する
- `languageHints` に `['ja', 'en']` を指定して日英混在に対応
- 撮影時にフラッシュを使わず、均一な照明で撮影するようユーザーに促す
- 画像の傾き補正（deskew）を前処理として追加すると精度が向上する

Cloud Vision API の代わりに、オンデバイスの ML Kit Text Recognition を使えば、ネットワーク接続なしでも OCR を実行できます。ただし、日本語の認識精度は Cloud Vision API のほうが優れているため、ビジネスユースでは API 方式をおすすめします。

## AI によるテキストの構造化データ変換

OCR で取得した生テキストは、そのままでは「名前はどれか」「メールアドレスはどれか」の判別ができません。ここで AI の力を借りて、非構造化テキストを構造化データに変換します。

Claude API を使った実装例を示します。

```typescript
// services/aiParserService.ts

interface BusinessCardData {
  name: string;
  nameReading?: string;      // ふりがな
  company: string;
  department?: string;
  title?: string;            // 役職
  email?: string;
  phone?: string;
  mobile?: string;
  fax?: string;
  website?: string;
  address?: string;
  postalCode?: string;
}

export async function parseBusinessCard(ocrText: string): Promise<BusinessCardData> {
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify({
      model: 'claude-sonnet-4-5-20250514',
      max_tokens: 1024,
      messages: [
        {
          role: 'user',
          content: `以下は名刺をOCRで読み取ったテキストです。
このテキストから連絡先情報を抽出し、JSON形式で返してください。

OCRテキスト:
${ocrText}

以下のJSON形式で返してください（該当しないフィールドはnull）:
{
  "name": "氏名",
  "nameReading": "ふりがな（推測できる場合）",
  "company": "会社名",
  "department": "部署名",
  "title": "役職",
  "email": "メールアドレス",
  "phone": "固定電話",
  "mobile": "携帯電話",
  "fax": "FAX番号",
  "website": "Webサイト",
  "address": "住所",
  "postalCode": "郵便番号"
}

JSONのみを返してください。説明文は不要です。`,
        },
      ],
    }),
  });

  const data = await response.json();
  const jsonText = data.content[0].text;
  return JSON.parse(jsonText) as BusinessCardData;
}
// 期待する出力:
// {
//   "name": "田中 太郎",
//   "company": "株式会社テクノロジー",
//   "title": "プロダクトマネージャー",
//   "email": "tanaka@example.com",
//   "phone": "03-1234-5678",
//   "website": "www.example.com",
//   ...
// }
```

AI を使う最大のメリットは、テキストの位置やフォーマットが名刺ごとに異なっても、文脈を理解して正しくフィールドを割り当てられる点です。正規表現だけでは対応しきれないケースでも、AI なら柔軟に処理できます。

**コスト最適化のポイント:** 名刺の解析は比較的短いテキストなので、`claude-haiku-4-5-20251001` を使えばコストを大幅に抑えられます。精度が十分であれば、軽量モデルへの切り替えを検討しましょう。AI のコスト管理について詳しく知りたい方は、[Rork AI API コスト最適化ガイド](/articles/rork-dev/rork-ai-api-cost-optimization-mobile-guide) も参考になります。

## 連絡先への自動登録

構造化データが取得できたら、`expo-contacts` を使ってスマートフォンの連絡先に自動登録します。

```typescript
// services/contactService.ts
import * as Contacts from 'expo-contacts';

export async function saveToContacts(card: BusinessCardData): Promise<string> {
  // 連絡先へのアクセス許可を確認
  const { status } = await Contacts.requestPermissionsAsync();
  if (status \!== 'granted') {
    throw new Error('連絡先へのアクセスが許可されていません');
  }

  // 連絡先データの構築
  const contact: Contacts.Contact = {
    contactType: Contacts.ContactTypes.Person,
    name: card.name,
    firstName: card.name.split(' ')[1] || card.name,
    lastName: card.name.split(' ')[0] || '',
    company: card.company,
    jobTitle: card.title || '',
    department: card.department || '',
    emails: card.email
      ? [{ email: card.email, label: 'work' }]
      : [],
    phoneNumbers: [
      ...(card.phone
        ? [{ number: card.phone, label: 'work' }]
        : []),
      ...(card.mobile
        ? [{ number: card.mobile, label: 'mobile' }]
        : []),
    ],
    urlAddresses: card.website
      ? [{ url: card.website, label: 'work' }]
      : [],
    addresses: card.address
      ? [{
          street: card.address,
          postalCode: card.postalCode || '',
          label: 'work',
        }]
      : [],
    note: `名刺スキャン: ${new Date().toLocaleDateString('ja-JP')}`,
  };

  // 連絡先を保存
  const contactId = await Contacts.addContactAsync(contact);
  return contactId;
}
// 期待する動作: 端末の連絡先に新しいエントリが追加される
```

連絡先登録の前に、同名の連絡先がすでに存在するかどうかをチェックする処理も追加すると、重複登録を防止できます。

```typescript
// 重複チェック
export async function checkDuplicate(name: string): Promise<boolean> {
  const { data } = await Contacts.getContactsAsync({
    fields: [Contacts.Fields.Name],
    name: name,
  });
  return data.length > 0;
}
```

## スキャン結果画面と履歴管理

スキャン結果を表示し、ユーザーが内容を確認・編集できる画面を作りましょう。AI の解析結果は完璧ではないため、ユーザーが修正できることが重要です。

```typescript
// ScanResultScreen.tsx
import { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, ScrollView } from 'react-native';

export default function ScanResultScreen({
  cardData,
  onSave,
}: {
  cardData: BusinessCardData;
  onSave: (data: BusinessCardData) => void;
}) {
  const [editData, setEditData] = useState(cardData);

  const fields = [
    { key: 'name', label: '氏名', icon: '👤' },
    { key: 'company', label: '会社名', icon: '🏢' },
    { key: 'title', label: '役職', icon: '💼' },
    { key: 'email', label: 'メール', icon: '✉️' },
    { key: 'phone', label: '電話', icon: '📞' },
    { key: 'mobile', label: '携帯', icon: '📱' },
    { key: 'website', label: 'Web', icon: '🌐' },
    { key: 'address', label: '住所', icon: '📍' },
  ];

  return (
    <ScrollView style={{ flex: 1, padding: 16 }}>
      <Text style={{ fontSize: 20, fontWeight: 'bold', marginBottom: 16 }}>
        スキャン結果
      </Text>
      {fields.map(({ key, label, icon }) => (
        <View key={key} style={{ marginBottom: 12 }}>
          <Text style={{ fontSize: 14, color: '#666', marginBottom: 4 }}>
            {icon} {label}
          </Text>
          <TextInput
            value={editData[key as keyof BusinessCardData] || ''}
            onChangeText={(text) =>
              setEditData((prev) => ({ ...prev, [key]: text }))
            }
            style={{
              borderWidth: 1,
              borderColor: '#ddd',
              borderRadius: 8,
              padding: 12,
              fontSize: 16,
            }}
            placeholder={`${label}を入力`}
          />
        </View>
      ))}
      <TouchableOpacity
        style={{
          backgroundColor: '#007AFF',
          padding: 16,
          borderRadius: 12,
          alignItems: 'center',
          marginTop: 16,
        }}
        onPress={() => onSave(editData)}
      >
        <Text style={{ color: '#fff', fontSize: 16, fontWeight: '600' }}>
          連絡先に保存
        </Text>
      </TouchableOpacity>
    </ScrollView>
  );
}
```

スキャン履歴は `AsyncStorage` や `expo-sqlite` を使ってローカルに保存すると、過去にスキャンした名刺を後から検索・再利用できます。

```typescript
// services/historyService.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const HISTORY_KEY = 'scan_history';

export async function addToHistory(
  card: BusinessCardData,
  imageUri: string
): Promise<void> {
  const history = await getHistory();
  history.unshift({
    id: Date.now().toString(),
    card,
    imageUri,
    scannedAt: new Date().toISOString(),
  });
  // 最新100件を保持
  await AsyncStorage.setItem(
    HISTORY_KEY,
    JSON.stringify(history.slice(0, 100))
  );
}

export async function getHistory() {
  const json = await AsyncStorage.getItem(HISTORY_KEY);
  return json ? JSON.parse(json) : [];
}
// 期待する動作: スキャン履歴がローカルに保存・取得できる
```

## セキュリティ対策とプライバシー配慮

名刺には個人情報が含まれるため、アプリのセキュリティ対策は特に重要です。

**API キーの管理:** API キーはアプリのソースコードに直接記述せず、環境変数やサーバーサイドプロキシを経由してリクエストを送信しましょう。Rork のプロジェクト設定で環境変数を設定するか、Supabase Edge Functions や Cloudflare Workers をプロキシとして利用する方法が安全です。

```typescript
// サーバーサイドプロキシ経由のリクエスト例
const response = await fetch('https://your-api-proxy.workers.dev/ocr', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${userToken}`,
  },
  body: JSON.stringify({ image: base64Image }),
});
```

**画像データの扱い:** 名刺画像をクラウドに送信する際は、処理後すぐにサーバー側で画像を削除するポリシーを明示しましょう。端末内でのみ画像を保持し、サーバーには送信しないオプションを提供するのも良い方法です。

**連絡先へのアクセス:** `expo-contacts` のパーミッション要求時には、なぜ連絡先へのアクセスが必要なのかをユーザーに説明するダイアログを表示しましょう。Apple のガイドラインでは、用途を明確に説明することが求められています。

セキュリティの実装パターンをさらに詳しく知りたい方は、[Rork アプリのセキュリティ完全実装ガイド](/articles/rork-dev/rork-app-security-complete-implementation-guide) で体系的に解説しています。

## よくある質問（FAQ）

### Q1: OCR の認識精度はどのくらいですか？

Google Cloud Vision API を使った場合、一般的な名刺で 90〜95% 程度の精度が期待できます。ただし、デザイン性の高い名刺（特殊フォント、背景画像の上にテキスト配置など）では精度が下がることがあります。AI による構造化と組み合わせることで、フィールドの割り当て精度を補完できます。

### Q2: オフラインでも使えますか？

完全なオフライン対応には ML Kit Text Recognition（オンデバイス）の利用が必要です。Google ML Kit は日本語の認識にも対応していますが、Cloud Vision API と比較すると精度が若干低くなります。AI による構造化処理はオンライン環境が必要です。ハイブリッド方式として、OCR はオフラインで実行し、AI 構造化はオンライン復帰時にバッチ処理する設計も検討できます。

### Q3: API の利用コストはどのくらいかかりますか？

Google Cloud Vision API は月 1,000 リクエストまで無料です。個人利用であれば無料枠内で十分収まるでしょう。Claude API は Haiku モデルを使えば 1 回の解析で約 0.01 円程度です。月に 500 枚の名刺をスキャンしても、API コストは数十円程度に収まります。

### Q4: App Store の審査で注意すべき点はありますか？

カメラと連絡先のパーミッション使用理由を `Info.plist` に明記する必要があります。また、個人情報を扱うアプリなので、プライバシーポリシーの掲載が必須です。Apple は特にカメラと連絡先へのアクセスについて厳しく審査するため、用途説明を具体的に記載してください。

### Q5: 両面名刺にも対応できますか？

表裏それぞれを撮影し、2 つの OCR 結果を AI に渡して統合する設計にすれば対応できます。プロンプトに「以下は名刺の表面と裏面の OCR テキストです。両方を統合して連絡先情報を抽出してください」と指示すれば、AI が適切にマージしてくれます。
