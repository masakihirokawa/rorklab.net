# Rork Max × Vision フレームワーク完全ガイド

Apple の Vision フレームワークは、iOS デバイス上で高精度な画像解析を実現するネイティブ API です。Rork Max を使えば、AI プロンプトだけでこの強力な機能をアプリに組み込むことができます。

このガイドでは、Vision フレームワークの3つの主要機能 — **テキスト認識（OCR）**、**物体検出**、**バーコード読み取り** — の実装方法を、Rork Max のプロンプト例と Swift コードを交えて解説します。

## Vision フレームワークとは

Vision は Apple が提供する画像解析フレームワークで、iOS 11 から利用可能です。機械学習モデルがデバイスに組み込まれているため、ネットワーク接続なしで動作し、ユーザーの画像データがデバイス外に送信されることもありません。

主な特徴は以下のとおりです。

- **オンデバイス処理**: クラウドへのデータ送信が不要で、プライバシーを保護しながら高速に動作します
- **Neural Engine 活用**: iPhone の専用 AI チップを使って推論を高速化します
- **豊富なリクエスト種別**: テキスト認識、顔検出、物体追跡、バーコード読み取りなど20種類以上の解析機能を提供します
- **Core ML との連携**: カスタムモデルを Vision パイプラインに組み込むことも可能です

## テキスト認識（OCR）を実装する

名刺読み取り、レシートスキャン、ドキュメントデジタル化など、テキスト認識は幅広いアプリで活用されています。

### Rork Max へのプロンプト例

Rork Max でテキスト認識機能を実装するには、以下のようなプロンプトを入力します。

```
カメラで撮影した画像からテキストを読み取るアプリを作ってください。
- カメラビューをフルスクリーンで表示
- シャッターボタンで撮影
- 撮影後に Vision の VNRecognizeTextRequest でテキストを抽出
- 認識結果をリスト表示し、タップでクリップボードにコピー
- 日本語と英語の両方を認識対象にする
```

### 実装のポイント

Rork Max が生成するコードの中核は `VNRecognizeTextRequest` です。iOS 16 以降では `automaticallyDetectsLanguage` プロパティが追加され、言語を明示的に指定しなくても日本語を含む多言語テキストを自動認識します。

```swift
let request = VNRecognizeTextRequest { request, error in
    guard let observations = request.results as? [VNRecognizedTextObservation] else { return }
    let recognizedText = observations.compactMap { observation in
        observation.topCandidates(1).first?.string
    }
    DispatchQueue.main.async {
        self.detectedTexts = recognizedText
    }
}
request.recognitionLevel = .accurate
request.automaticallyDetectsLanguage = true
```

`recognitionLevel` には `.fast` と `.accurate` の2つがあります。リアルタイムプレビューには `.fast` を、撮影後の確定処理には `.accurate` を使い分けるのが実用的な設計です。

### 認識精度を高めるテクニック

テキスト認識の精度を向上させるには、いくつかのテクニックがあります。`recognitionLanguages` に優先言語を指定すると、対象言語のテキスト認識精度が向上します。また `minimumTextHeight` を設定することで、小さすぎるノイズテキストを除外できます。

画像の前処理も重要です。コントラストの低い画像やぼけた画像は認識精度が下がるため、`CIFilter` を使った画像補正を撮影後に適用すると結果が改善します。

## 物体検出を実装する

Vision フレームワークの物体検出機能を使うと、画像内のオブジェクトの位置と種類を特定できます。

### Rork Max へのプロンプト例

```
カメラのリアルタイムプレビュー上に物体検出の結果をオーバーレイ表示するアプリを作ってください。
- AVCaptureSession でカメラフィードを取得
- VNDetectRectanglesRequest で矩形を検出
- 検出した矩形にバウンディングボックスを描画
- ドキュメントスキャンモードも追加（検出した矩形で画像をクロップ・補正）
```

### 矩形検出の応用

矩形検出はドキュメントスキャナーの基盤技術です。`VNDetectRectanglesRequest` は書類やカードの輪郭を検出し、射影変換で正面から見た画像に補正します。

```swift
let rectangleRequest = VNDetectRectanglesRequest { request, error in
    guard let results = request.results as? [VNRectangleObservation],
          let rectangle = results.first else { return }

    let topLeft = rectangle.topLeft
    let topRight = rectangle.topRight
    let bottomLeft = rectangle.bottomLeft
    let bottomRight = rectangle.bottomRight

    // CIFilter でパースペクティブ補正を適用
    self.applyPerspectiveCorrection(
        topLeft: topLeft, topRight: topRight,
        bottomLeft: bottomLeft, bottomRight: bottomRight
    )
}
rectangleRequest.minimumConfidence = 0.8
rectangleRequest.maximumObservations = 1
```

`minimumConfidence` を高めに設定することで、誤検出を減らせます。書類スキャンアプリであれば 0.8 以上が推奨値です。

### カスタム物体検出モデルの統合

より高度な物体検出には、Create ML で学習したカスタムモデルを Vision パイプラインに統合できます。Rork Max では、以下のようなプロンプトでカスタムモデルの統合を指示できます。

```
Core ML の物体検出モデル（.mlmodel）をプロジェクトに追加して、
カメラフィードでリアルタイム推論を行うアプリを作ってください。
VNCoreMLRequest を使って Vision パイプラインに統合してください。
```

## バーコード・QR コード読み取り

Vision フレームワークはバーコードと QR コードの読み取りにも対応しています。商品管理、チケット確認、URL 共有など、実用性の高い機能です。

### Rork Max へのプロンプト例

```
QRコードとバーコードを読み取るスキャナーアプリを作ってください。
- カメラプレビューの中央にスキャン枠を表示
- VNDetectBarcodesRequest で QR / EAN-13 / Code 128 に対応
- 読み取り結果をカード形式で表示
- URL が含まれる場合は Safari で開くボタンを追加
- 読み取り時にハプティックフィードバックを鳴らす
```

### 実装のポイント

`VNDetectBarcodesRequest` は、対応するシンボル体系を `symbologies` プロパティで指定します。

```swift
let barcodeRequest = VNDetectBarcodesRequest { request, error in
    guard let results = request.results as? [VNBarcodeObservation] else { return }
    for barcode in results {
        if let payload = barcode.payloadStringValue {
            DispatchQueue.main.async {
                self.scannedCodes.append(ScannedCode(
                    value: payload,
                    symbology: barcode.symbology,
                    timestamp: Date()
                ))
            }
        }
    }
}
barcodeRequest.symbologies = [.qr, .ean13, .code128, .pdf417]
```

連続スキャンを実装する場合は、同じコードの重複検出を防ぐロジックが必要です。直近の検出結果をキャッシュし、一定時間（たとえば3秒間）は同一の値をスキップする実装が一般的です。

## パフォーマンス最適化

Vision フレームワークをリアルタイム処理で使用する際は、パフォーマンスの最適化が重要です。

**リクエストの再利用**: `VNImageRequestHandler` はフレームごとに新規作成しますが、`VNRequest` オブジェクトは再利用可能です。毎フレーム新規作成するとメモリ消費が増加します。

**処理キューの分離**: カメラフレームの取得（メインスレッド）と Vision の推論処理（バックグラウンドスレッド）を分離し、UI のフレームレートが低下しないようにします。

```swift
private let visionQueue = DispatchQueue(
    label: "com.app.vision",
    qos: .userInitiated
)
```

**フレームスキップ**: 30fps のカメラフィードをすべて処理する必要はありません。2〜3フレームに1回の処理で、十分なユーザー体験を提供できる場合がほとんどです。

**Region of Interest（ROI）の設定**: 画像全体ではなく、スキャン枠内の領域のみを処理対象にすることで、認識速度と精度の両方が向上します。

```swift
request.regionOfInterest = CGRect(x: 0.25, y: 0.25, width: 0.5, height: 0.5)
```

## Rork Max で開発する際のベストプラクティス

Vision フレームワークを使ったアプリを Rork Max で効率的に開発するためのポイントをまとめます。

**プロンプトを段階的に進める**: 最初のプロンプトではカメラプレビューの表示だけを指示し、動作確認後に Vision の処理を追加していくアプローチが安定した結果を得やすいです。

**エラーハンドリングを明示的に指示する**: カメラのアクセス許可が拒否された場合や、Vision リクエストが失敗した場合のフォールバック UI を、プロンプトで明示的に指示してください。

**テスト用画像を用意する**: 実機テスト前に、シミュレータで静止画像を使ったテストができるようプロンプトで「テストモード」の実装を指示しておくと、開発サイクルが短縮されます。

**Info.plist のカメラ権限文言を指定する**: `NSCameraUsageDescription` に設定する文言をプロンプトで指定しておくと、App Store 審査で差し戻されるリスクを減らせます。

## まとめ

Vision フレームワークは、テキスト認識、物体検出、バーコード読み取りといった画像解析機能を、クラウドに依存せずデバイス上で実現する強力な API です。Rork Max のプロンプト駆動開発と組み合わせることで、これらの機能を短時間でアプリに統合できます。

プライバシーを重視するユーザーが増えている現在、オンデバイスで完結する AI 機能は大きなアドバンテージです。まずはテキスト認識から試して、Vision フレームワークの可能性を体験してみてください。
