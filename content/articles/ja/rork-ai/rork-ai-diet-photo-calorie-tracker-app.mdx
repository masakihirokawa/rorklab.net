## はじめに — 写真を撮るだけでカロリーがわかるアプリ

ダイエットや健康管理で最も面倒な作業の一つが、毎回の食事のカロリーを記録することです。食材を一つひとつ検索して入力する従来のアプリでは、続けられずに挫折してしまう方も多いのではないでしょうか。

この記事では、Rork と Gemini Vision API を組み合わせて、食事の写真を撮影するだけでカロリーと主要栄養素（タンパク質・脂質・炭水化物）を自動推定する AI ダイエット記録アプリを構築します。AI が画像から料理名と分量を認識し、栄養データを返してくれるので、ユーザーは写真を撮るだけで日々の食事を簡単に記録できます。

この記事で学べること:

- Rork で Gemini Vision API を呼び出し、食事画像を解析する方法
- カメラ撮影 → AI 分析 → データ保存の一連のフローの実装
- 日別・週別の栄養サマリーを表示するダッシュボード UI
- AsyncStorage を使ったローカルデータの永続化

対象読者は、Rork の基本操作を理解している中級者の方です。初めての方は [Rork でアプリを作り始めるための完全ガイド](/articles/rork-basics/rork-zero-to-app-complete-guide) を先にご覧ください。

## アプリの全体設計

まず、今回作成するアプリの画面構成と処理フローを整理しましょう。

**画面構成（4画面）:**

- **ホーム画面**: 今日の食事記録一覧と、合計カロリー・栄養素のサマリー
- **カメラ撮影画面**: カメラを起動して食事の写真を撮影（またはライブラリから選択）
- **AI 分析結果画面**: 推定された料理名・カロリー・栄養素の確認と編集
- **週間レポート画面**: 過去7日間のカロリー推移をグラフで表示

**処理フロー:**

1. ユーザーがカメラで食事を撮影
2. 画像を Base64 エンコードして Gemini Vision API に送信
3. AI が料理名・推定カロリー・栄養素を JSON で返却
4. ユーザーが内容を確認（必要に応じて手動修正）
5. AsyncStorage にデータを保存
6. ホーム画面と週間レポートに反映

## Gemini Vision API で食事画像を解析する

このアプリの核心部分は、Gemini の画像認識機能です。適切なプロンプトを設計することで、料理の種類と栄養情報を高精度で取得できます。

まず、Rork のプロンプトで API 呼び出しを行うサービスモジュールを作成します。

```typescript
// services/foodAnalyzer.ts
// Gemini Vision API で食事画像を解析するサービス

const GEMINI_API_URL = "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent";

interface NutritionResult {
  dishName: string;
  calories: number;
  protein: number;   // g
  fat: number;       // g
  carbs: number;     // g
  confidence: "high" | "medium" | "low";
}

export async function analyzeFoodImage(
  base64Image: string,
  apiKey: string
): Promise<NutritionResult[]> {
  const prompt = `
    この食事の写真を分析してください。
    写真に写っている各料理について、以下の情報をJSON配列で返してください。
    - dishName: 料理名（日本語）
    - calories: 推定カロリー（kcal, 整数）
    - protein: タンパク質（g, 小数点1桁）
    - fat: 脂質（g, 小数点1桁）
    - carbs: 炭水化物（g, 小数点1桁）
    - confidence: 推定の信頼度（"high", "medium", "low"）
    
    一般的な一人前の分量を基準に推定してください。
    JSONのみを返し、他のテキストは含めないでください。
  `;

  const response = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      contents: [{
        parts: [
          { text: prompt },
          {
            inlineData: {
              mimeType: "image/jpeg",
              data: base64Image,
            },
          },
        ],
      }],
    }),
  });

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;

  // JSON部分を抽出してパース
  const jsonMatch = text.match(/\[[\s\S]*\]/);
  if (!jsonMatch) throw new Error("AI の応答をパースできませんでした");

  return JSON.parse(jsonMatch[0]) as NutritionResult[];
}
```

ポイントは、プロンプトで「JSON のみを返す」ことを明確に指示している点です。これにより、レスポンスのパース処理がシンプルになります。

## カメラ撮影と画像取得の実装

Rork では `expo-image-picker` を利用して、カメラ撮影やライブラリからの画像選択を実装します。

```typescript
// hooks/useFoodCamera.ts
// カメラ撮影と画像の Base64 変換を行うカスタムフック

import * as ImagePicker from "expo-image-picker";
import { useState } from "react";
import { analyzeFoodImage } from "../services/foodAnalyzer";

export function useFoodCamera(apiKey: string) {
  const [isAnalyzing, setIsAnalyzing] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const captureAndAnalyze = async () => {
    setError(null);
    
    // カメラの権限を要求
    const permission = await ImagePicker.requestCameraPermissionsAsync();
    if (!permission.granted) {
      setError("カメラのアクセス許可が必要です");
      return null;
    }

    // カメラを起動して撮影
    const result = await ImagePicker.launchCameraAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      quality: 0.7,       // 画質を少し落として転送量を削減
      base64: true,        // Base64エンコードを有効化
      allowsEditing: true, // トリミングを許可
      aspect: [4, 3],
    });

    if (result.canceled || !result.assets[0].base64) return null;

    // AI で食事を分析
    setIsAnalyzing(true);
    try {
      const nutrition = await analyzeFoodImage(
        result.assets[0].base64,
        apiKey
      );
      return {
        imageUri: result.assets[0].uri,
        nutrition,
        timestamp: new Date().toISOString(),
      };
    } catch (err) {
      setError("食事の分析に失敗しました。もう一度撮影してください");
      return null;
    } finally {
      setIsAnalyzing(false);
    }
  };

  return { captureAndAnalyze, isAnalyzing, error };
}

// 使用例:
// const { captureAndAnalyze, isAnalyzing } = useFoodCamera("YOUR_GEMINI_API_KEY");
// const meal = await captureAndAnalyze();
// → { imageUri: "file://...", nutrition: [{dishName: "親子丼", calories: 580, ...}], timestamp: "..." }
```

`quality: 0.7` を指定してファイルサイズを抑えているのがポイントです。Gemini API にはリクエストサイズの上限があるため、高解像度のままだと送信に失敗する場合があります。

## データの保存とカロリー集計

食事記録を端末に保存し、日別の合計カロリーを計算するロジックを実装します。

```typescript
// services/mealStorage.ts
// AsyncStorage を使った食事記録の永続化

import AsyncStorage from "@react-native-async-storage/async-storage";

interface MealRecord {
  id: string;
  imageUri: string;
  dishes: {
    dishName: string;
    calories: number;
    protein: number;
    fat: number;
    carbs: number;
  }[];
  totalCalories: number;
  timestamp: string;
}

const STORAGE_KEY = "meal_records";

// 食事記録を保存
export async function saveMealRecord(meal: MealRecord): Promise<void> {
  const existing = await getMealRecords();
  existing.push(meal);
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(existing));
}

// 全記録を取得
export async function getMealRecords(): Promise<MealRecord[]> {
  const data = await AsyncStorage.getItem(STORAGE_KEY);
  return data ? JSON.parse(data) : [];
}

// 指定日の合計栄養素を計算
export async function getDailySummary(date: string) {
  const records = await getMealRecords();
  const dayRecords = records.filter(
    (r) => r.timestamp.startsWith(date)
  );

  return dayRecords.reduce(
    (sum, record) => ({
      totalCalories: sum.totalCalories + record.totalCalories,
      totalProtein: sum.totalProtein +
        record.dishes.reduce((s, d) => s + d.protein, 0),
      totalFat: sum.totalFat +
        record.dishes.reduce((s, d) => s + d.fat, 0),
      totalCarbs: sum.totalCarbs +
        record.dishes.reduce((s, d) => s + d.carbs, 0),
      mealCount: sum.mealCount + 1,
    }),
    { totalCalories: 0, totalProtein: 0, totalFat: 0, totalCarbs: 0, mealCount: 0 }
  );
}

// 過去7日間の日別カロリーを取得（グラフ表示用）
export async function getWeeklyCalories(): Promise<{ date: string; calories: number }[]> {
  const result: { date: string; calories: number }[] = [];
  
  for (let i = 6; i >= 0; i--) {
    const d = new Date();
    d.setDate(d.getDate() - i);
    const dateStr = d.toISOString().split("T")[0];
    const summary = await getDailySummary(dateStr);
    result.push({ date: dateStr, calories: summary.totalCalories });
  }
  
  return result;
}
```

`getDailySummary` では、タイムスタンプの先頭を日付文字列と照合して、その日に記録された食事のカロリーを集計しています。シンプルですが、日常的な使用には十分な実装です。

ホーム画面では、今日の食事一覧と栄養素サマリーを表示します。Rork のプロンプトで以下のような画面を指示しましょう。

```
ホーム画面を作成してください。
- 上部に今日の合計カロリーを大きく円形プログレスバーで表示
  （目標カロリー2000kcalに対する達成率）
- その下にタンパク質・脂質・炭水化物の3つのミニバー
- 食事記録はカード形式で表示（写真サムネイル + 料理名 + カロリー）

- カラースキーム: グリーン系（ヘルスケアアプリらしく）
```

円形プログレスバーのコンポーネント例:

```tsx
// components/CalorieRing.tsx
// 目標カロリーに対する達成率を円形で表示

import React from "react";
import { View, Text, StyleSheet } from "react-native";
import Svg, { Circle } from "react-native-svg";

interface Props {
  current: number;
  target: number;
}

export function CalorieRing({ current, target }: Props) {
  const progress = Math.min(current / target, 1);
  const radius = 80;
  const strokeWidth = 12;
  const circumference = 2 * Math.PI * radius;
  const strokeDashoffset = circumference * (1 - progress);

  return (
    <View style={styles.container}>
      <Svg width={200} height={200}>
        {/* 背景の円 */}
        <Circle
          cx={100} cy={100} r={radius}
          stroke="#E8F5E9"
          strokeWidth={strokeWidth}
          fill="none"
        />
        {/* 進捗の円 */}
        <Circle
          cx={100} cy={100} r={radius}
          stroke="#4CAF50"
          strokeWidth={strokeWidth}
          fill="none"
          strokeDasharray={circumference}
          strokeDashoffset={strokeDashoffset}
          strokeLinecap="round"
          rotation={-90}
          origin="100,100"
        />
      </Svg>
      <View style={styles.textContainer}>
        <Text style={styles.calories}>{current}</Text>
        <Text style={styles.unit}>/ {target} kcal</Text>
      </View>
    </View>
  );
}

// 表示例: <CalorieRing current={1250} target={2000} />
// → 円形プログレスバーが62.5%まで緑色で表示される

const styles = StyleSheet.create({
  container: { alignItems: "center", justifyContent: "center" },
  textContainer: { position: "absolute", alignItems: "center" },
  calories: { fontSize: 36, fontWeight: "bold", color: "#2E7D32" },
  unit: { fontSize: 14, color: "#666" },
});
```

## AI 分析精度を高めるプロンプトの工夫

食事写真の AI 分析では、プロンプトの設計が精度を大きく左右します。以下のテクニックを活用することで、より正確な結果を得られます。

**1. 複数料理への対応**: 定食やプレートに複数の料理が載っている場合、各料理を個別に認識させます。先ほどのプロンプトで「各料理について」と指定しているのはこのためです。

**2. 分量の推定精度の向上**: プロンプトに「お皿のサイズを手がかりに分量を推定してください」と追記すると、AI がお皿や箸などのスケール参照物を活用して、より正確な分量を推定します。

**3. 和食への最適化**: 日本語ユーザー向けアプリの場合、プロンプトに「日本食の栄養データベース（文部科学省 食品成分表）を参考にしてください」と指示を加えると、和食の推定精度が改善されます。

```typescript
// プロンプトの改良版（抜粋）
const enhancedPrompt = `
  この食事の写真を分析してください。
  お皿のサイズや周囲のオブジェクトを手がかりに、各料理の分量を推定してください。
  日本食の場合は文部科学省の食品成分表を参考にしてください。
  飲み物がある場合はそれも含めてください。
  
  各料理について以下のJSON配列で返してください:
  [{ "dishName": "料理名", "calories": 数値, "protein": 数値, 
     "fat": 数値, "carbs": 数値, "confidence": "high|medium|low" }]
`;
```

## よくあるエラーと対処法

アプリ開発中に遭遇しやすい問題と解決策をまとめます。

**カメラの権限が取得できない**: iOS の場合、`app.json` の `ios.infoPlist` に `NSCameraUsageDescription` を追加する必要があります。Rork のプロンプトに「カメラ権限の説明文を Info.plist に設定してください」と指示しましょう。

**Gemini API のレスポンスが遅い**: 画像サイズが大きいとレスポンスに時間がかかります。`quality: 0.5`〜`0.7` で十分な認識精度が得られるので、圧縮してから送信するのが効果的です。

**JSON のパースエラー**: AI がまれに JSON 以外のテキストを含むことがあります。正規表現で JSON 部分のみを抽出する処理（上記コードの `text.match(/\[[\s\S]*\]/)` の部分）を必ず入れておきましょう。

**栄養素の推定値が大きくずれる**: AI の推定はあくまで目安です。ユーザーが手動で修正できる編集画面を用意し、「AI 推定値」であることを明示する UI にすることが重要です。

## FAQ

### Q. AI のカロリー推定はどれくらい正確ですか？

Gemini Vision API による推定値は、一般的に実際のカロリーの ±20〜30% 程度の誤差があります。単品の料理（ラーメン、カレーライスなど）は比較的正確ですが、複数の食材が混ざった料理やソースの量が見た目でわかりにくい料理は誤差が大きくなる傾向があります。あくまで目安として利用し、厳密なカロリー管理が必要な場合は手動での修正を推奨する UI にするのがベストです。

### Q. オフラインでも使えるようにできますか？

食事の撮影と AI 分析にはインターネット接続が必要です。ただし、過去の記録の閲覧やグラフ表示は AsyncStorage に保存されたデータで動作するため、オフラインでも可能です。オフライン時に撮影した写真をキューに保存し、オンライン復帰後に一括で AI 分析を行う仕組みを追加すれば、より実用的になります。[Rork アプリのオフライン対応ガイド](/articles/rork-dev/rork-offline-support) も参考にしてください。

### Q. Gemini API の無料枠で運用できますか？

Gemini API には無料利用枠があり、個人利用やプロトタイプ開発には十分です。ただし、ユーザー数が増えた場合はコスト管理が重要になります。1回の画像分析あたり約 0.001〜0.003 ドル程度のコストが目安です。詳しい API の設定方法は [Rork × Gemini 連携ガイド](/articles/rork-ai/rork-gemini-integration) で解説しています。

### Q. 写真以外の入力方法にも対応できますか？

テキストでの料理名入力や、バーコードスキャンによる食品パッケージの読み取りなど、補助的な入力手段を追加することも可能です。テキスト入力の場合は Gemini のテキスト API に「この料理名のカロリーと栄養素を推定してください」と送信するだけで実装できます。
