## テキスト読み上げ（TTS）はなぜアプリに必要なのか

スマートフォンアプリにおける音声体験の重要性が、年々高まっています。手が離せない場面でのナビゲーション案内、視覚障害を持つユーザーへのアクセシビリティ対応、英語学習アプリでの発音確認など、テキスト読み上げ（Text-to-Speech、以下TTS）が活躍する場面は多岐にわたります。

Rorkで開発するReact Nativeアプリにも、TTSを組み込むことは難しくありません。本記事では、大きく2つのアプローチを紹介します。

- **expo-speech**：Expoが提供するネイティブTTSライブラリ。追加の費用や外部APIなしで、iOSとAndroidのシステム音声エンジンを呼び出せます。
- **ElevenLabs API**：AI音声合成の最前線にあるサービス。自然でリアルな人間に近い音声を生成できます。

この2つを使い分けることで、コスト・品質・ユースケースのバランスに応じた最適な実装が可能です。

## expo-speech で即時に TTS を実装する

まず最もシンプルな方法から始めましょう。`expo-speech`はExpoが標準で提供するライブラリで、追加インストール不要でRorkプロジェクトから利用できます（Expo SDKに同梱）。

### 基本的な使い方

```tsx
import * as Speech from 'expo-speech';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export default function TTSBasicScreen() {
  const text = 'こんにちは！これはRorkアプリのテキスト読み上げデモです。';

  const speak = () => {
    // 読み上げを開始する
    Speech.speak(text, {
      language: 'ja-JP',   // 言語コード（日本語）
      pitch: 1.0,          // 音の高さ（0.5〜2.0）
      rate: 0.9,           // 読み上げ速度（0.1〜2.0）
      onStart: () => console.log('読み上げ開始'),
      onDone: () => console.log('読み上げ完了'),
      onError: (err) => console.error('エラー:', err),
    });
  };

  const stop = () => {
    // 読み上げを途中で停止する
    Speech.stop();
  };

  return (
    <View style={styles.container}>
      <Text style={styles.label}>{text}</Text>
      <TouchableOpacity style={styles.button} onPress={speak}>
        <Text style={styles.buttonText}>▶ 読み上げる</Text>
      </TouchableOpacity>
      <TouchableOpacity style={styles.stopButton} onPress={stop}>
        <Text style={styles.buttonText}>■ 停止</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 24 },
  label: { fontSize: 16, marginBottom: 24, textAlign: 'center', lineHeight: 26 },
  button: { backgroundColor: '#4F46E5', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8, marginBottom: 12 },
  stopButton: { backgroundColor: '#6B7280', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});

// 期待する動作:
// ▶ 読み上げるボタンを押すと、日本語音声で文章が読み上げられる
// ■ 停止ボタンを押すと、途中でも読み上げが止まる
```

### 対応言語と音声の選択

`expo-speech`では、端末にインストールされている音声エンジンが使用されます。日本語（`ja-JP`）はiOS・Androidともに標準で利用可能です。利用可能な音声の一覧を取得するには `Speech.getAvailableVoicesAsync()` を使います。

```tsx
import * as Speech from 'expo-speech';
import { useEffect, useState } from 'react';

export function useAvailableVoices(language: string) {
  const [voices, setVoices] = useState<Speech.Voice[]>([]);

  useEffect(() => {
    Speech.getAvailableVoicesAsync().then((all) => {
      // 指定言語に対応した音声だけをフィルタリング
      const filtered = all.filter((v) => v.language.startsWith(language));
      setVoices(filtered);
    });
  }, [language]);

  return voices;
}
```

このカスタムフックを使えば、ユーザーが好みの音声を選べるセレクターUIも簡単に作れます。

## ElevenLabs API でリアルな AI 音声を生成する

`expo-speech`はシステム音声のため、どうしても機械的な印象が残ります。語学学習アプリや読み聞かせ機能など、より自然な音声が求められる場合は **ElevenLabs API** が有力な選択肢です。

### 事前準備

1. [ElevenLabs 公式サイト](https://elevenlabs.io/) でアカウントを作成
2. APIキーを取得（無料枠で月10,000文字まで利用可能）
3. `expo-av`をインストール（音声ファイルの再生に使用）

```bash
npx expo install expo-av
```

### ElevenLabs API を呼び出す実装

```tsx
import { Audio } from 'expo-av';
import { useState } from 'react';
import { View, Text, TouchableOpacity, ActivityIndicator, StyleSheet } from 'react-native';

// ⚠️ APIキーは環境変数で管理し、コードに直接書かないこと
const ELEVENLABS_API_KEY = process.env.EXPO_PUBLIC_ELEVENLABS_API_KEY ?? '';
// ElevenLabsで選択した音声ID（ダッシュボードで確認できる）
const VOICE_ID = 'YOUR_VOICE_ID';

export default function ElevenLabsTTSScreen() {
  const [isLoading, setIsLoading] = useState(false);
  const [sound, setSound] = useState<Audio.Sound | null>(null);

  const speak = async (text: string) => {
    setIsLoading(true);
    try {
      // ElevenLabs TTS APIを呼び出して音声データ（MP3）を取得
      const response = await fetch(
        `https://api.elevenlabs.io/v1/text-to-speech/${VOICE_ID}`,
        {
          method: 'POST',
          headers: {
            'xi-api-key': ELEVENLABS_API_KEY,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({
            text,
            model_id: 'eleven_multilingual_v2', // 多言語対応モデル
            voice_settings: {
              stability: 0.5,       // 音声の安定性（0〜1）
              similarity_boost: 0.8, // 音声の再現性（0〜1）
            },
          }),
        }
      );

      if (!response.ok) throw new Error(`API Error: ${response.status}`);

      // レスポンスをBase64エンコードしてData URIに変換
      const audioBlob = await response.blob();
      const reader = new FileReader();
      reader.onload = async () => {
        const dataUri = reader.result as string;
        // expo-avで音声を再生
        const { sound: newSound } = await Audio.Sound.createAsync({ uri: dataUri });
        setSound(newSound);
        await newSound.playAsync();
      };
      reader.readAsDataURL(audioBlob);
    } catch (error) {
      console.error('TTS エラー:', error);
    } finally {
      setIsLoading(false);
    }
  };

  const stopSound = async () => {
    if (sound) {
      await sound.stopAsync();
      await sound.unloadAsync();
      setSound(null);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>AI音声読み上げデモ</Text>
      {isLoading ? (
        <ActivityIndicator size="large" color="#4F46E5" />
      ) : (
        <TouchableOpacity
          style={styles.button}
          onPress={() => speak('こんにちは！ElevenLabsのAI音声です。とても自然な声ですよね。')}
        >
          <Text style={styles.buttonText}>▶ AI音声で読み上げる</Text>
        </TouchableOpacity>
      )}
      <TouchableOpacity style={styles.stopButton} onPress={stopSound}>
        <Text style={styles.buttonText}>■ 停止</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 24 },
  title: { fontSize: 20, fontWeight: '700', marginBottom: 32 },
  button: { backgroundColor: '#4F46E5', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8, marginBottom: 12 },
  stopButton: { backgroundColor: '#6B7280', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});

// 期待する動作:
// ▶ AI音声で読み上げるボタンを押すと、ローディング後にElevenLabsの自然な音声が再生される
// ■ 停止ボタンで再生を中断できる
```

> **コスト管理のヒント**: ElevenLabs APIは文字数課金です。1リクエストあたりの文字数を制限したり、同じテキストの音声をキャッシュして再リクエストを避けるなど、コスト最適化を検討しましょう。

## よくあるエラーと対処法

TTS実装でよく遭遇するエラーとその解決策をまとめました。

### expo-speech が iOS シミュレーターで無音になる

iOSシミュレーターでは音声出力が正常に動作しないケースがあります。**実機でのテスト**を基本にしてください。Xcodeシミュレーターの設定で「Audio and MIDI」を有効にするとシミュレーターでも動くことがあります。

### ElevenLabs API で 401 エラーが出る

APIキーが正しく設定されていない場合に発生します。`process.env.EXPO_PUBLIC_ELEVENLABS_API_KEY`のように`EXPO_PUBLIC_`プレフィックスをつけた環境変数をアプリから参照していることを確認してください。`.env.local`ファイルに記述し、`.gitignore`に追加してリポジトリに含めないよう注意しましょう。

### 音声が途切れる・遅延が大きい

ElevenLabs APIはネットワーク通信を伴うため、音声データのダウンロードに時間がかかる場合があります。以下の対策が有効です。

- テキストを短い文単位に分割して送信し、先頭部分から順次再生する（ストリーミング再生）
- 固定テキストはビルド時や初回起動時に音声を生成してローカルキャッシュに保存する

## TTS の応用シナリオ

TTSを活用できる実用的なユースケースを挙げます。

**語学学習アプリ**: 単語帳の例文をネイティブ音声で読み上げる。ElevenLabsの多言語モデルを使えば英語・スペイン語・フランス語など多言語に対応できます。

**ニュースリーダー**: 記事本文をバックグラウンドで再生し、通勤中に"ながら聴き"できるアプリ。[Rorkで作るAI議事録アプリ](/articles/rork-ai/rork-ai-meeting-notes-whisper-claude-tutorial)の逆パターンとして、音声→テキスト（Whisper）と組み合わせると、テキスト→音声の双方向音声インターフェースが実現します。

**アクセシビリティ対応**: 視覚障害のあるユーザー向けに、画面内のテキスト要素を音声で伝える補助機能。`expo-speech`の`speak()`をアクセシビリティイベントと連動させることで実装できます。

**読み聞かせアプリ**: 子ども向けの絵本アプリで、ページのテキストを自動的に読み上げる。ElevenLabsの子ども向け音声を使うと親しみやすい体験になります。

さらに高品質な音声AIアシスタントを作りたい場合は、[Rork × OpenAI Realtime API：リアルタイム音声AIアシスタントアプリ完全実装ガイド](/articles/rork-ai/rork-openai-realtime-api-voice-assistant-app)もぜひ参考にしてみてください。より対話的な音声体験の実装方法を詳しく解説しています。

## よくある質問（FAQ）

**Q1. expo-speech は無料で使えますか？**

はい、`expo-speech`はExpo SDKに含まれており追加費用は不要です。デバイスのシステム音声エンジン（iOSはAVSpeechSynthesizer、AndroidはAndroid TTS）を使うため、オフラインでも動作します。ただし音声品質はデバイスの標準音声に依存します。

**Q2. 日本語以外の言語も読み上げられますか？**

`expo-speech`では`language`オプションに`en-US`（英語）、`zh-CN`（中国語）など標準の言語コードを指定することで多言語対応が可能です。ElevenLabsの`eleven_multilingual_v2`モデルは28言語以上に対応しており、テキストの言語を自動判定して読み上げます。

**Q3. 読み上げ中にアプリをバックグラウンドに移動させても続きますか？**

`expo-speech`によるデバイス標準TTSはフォアグラウンドが必要な場合がほとんどです。バックグラウンド再生には`expo-av`と`Audio.setAudioModeAsync({ staysActiveInBackground: true })`の設定が必要です。ElevenLabsで生成した音声ファイルを`expo-av`で再生する場合は、この設定でバックグラウンド再生が可能です。

**Q4. TTS とSTT（音声認識）を組み合わせることはできますか？**

はい、可能です。OpenAI Whisper APIで音声をテキストに変換し、AIで処理した後にTTSで読み上げる、という音声インターフェースの構築が可能です。[Rork + OpenAI Whisper API で音声認識アプリを作る完全ガイド](/articles/rork-ai/rork-whisper-voice-recognition)と本記事を組み合わせれば、双方向の音声チャットアプリが実現します。
