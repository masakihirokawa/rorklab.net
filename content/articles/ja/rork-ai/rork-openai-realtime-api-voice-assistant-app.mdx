## はじめに：なぜ Realtime API が音声アプリの常識を変えるのか

これまでモバイルアプリに音声対話機能を実装しようとすると、「録音 → Whisper で文字起こし → GPT-4o で回答生成 → TTS で読み上げ」という複数ステップのパイプラインが必要でした。各ステップに数百ミリ秒〜数秒のレイテンシが積み重なり、ユーザーは「考えている間の不自然な沈黙」を体験せざるを得ませんでした。

OpenAI が2024年10月にリリースした **Realtime API** はこの問題を根本から解決します。音声入力・LLM 推論・音声出力が単一の WebSocket セッションの中で並列処理され、**エンドツーエンドのレイテンシを100ms以下**に抑えることが可能です。さらに、音声から音声への直接処理により感情のトーンや会話のリズムも維持されます。

この記事では Rork Max を使ってこの Realtime API を React Native アプリに統合し、プロダクション品質の音声 AI アシスタントアプリを完成させる手順を、実際に動くコードと設計パターンを交えながら徹底的に解説します。

対象読者は Rork でのアプリ開発経験があり、API 統合の基礎知識を持つ方です。[OpenAI の GPT-4o を使った基本的なチャットアプリの実装](/articles/rork-ai/gpt-4o-rork-openai-app-development)はすでに理解していることを前提とします。

## OpenAI Realtime API の仕組みを理解する

### WebSocket セッションモデル

Realtime API は REST ではなく **永続的な WebSocket 接続**を使います。クライアントとサーバーはイベントを双方向でストリーミングし、音声データはバイナリチャンクとして送受信されます。

主要なイベントの流れは以下の通りです：

- `session.created` — 接続確立直後に受け取るセッション情報
- `input_audio_buffer.append` — クライアントが音声チャンクを送信
- `input_audio_buffer.commit` — 発話終了を通知（手動モード時）
- `response.audio.delta` — サーバーから音声チャンクが届く
- `response.text.delta` — 同時にテキストトランスクリプトも届く
- `response.done` — レスポンス生成完了

### Voice Activity Detection（VAD）モード

Realtime API には2つの音声検出モードがあります。

**server_vad モード（推奨）**: サーバー側が発話の開始・終了を自動検出します。ユーザーが話し終えたら自動的にレスポンスが生成されるため、会話の流れが自然です。大半のユースケースではこちらを使います。

**none モード（手動制御）**: アプリ側がボタン操作などで `input_audio_buffer.commit` を送信して発話終了を明示します。ノイズの多い環境や、ユーザーが意図的にプッシュ・トゥ・トークをしたい場合に適しています。

### 対応音声フォーマット

入出力ともに **PCM16（16-bit signed PCM、24kHz、モノラル）** が基本フォーマットです。React Native の `expo-av` や `react-native-audio-record` はデフォルトでこの形式に対応していないため、サンプリングレート変換が必要な場合があります（後述）。

## WebSocket 接続レイヤーの実装

### RealtimeClient クラス

接続管理・イベント送受信の責務を持つクライアントクラスを作成します：

```typescript
// src/services/RealtimeClient.ts
import { EventEmitter } from 'eventemitter3';
import { OPENAI_API_KEY } from '../config/openai';

const REALTIME_URL =
  'wss://api.openai.com/v1/realtime?model=gpt-4o-realtime-preview-2024-12-17';

export type RealtimeEvent =
  | { type: 'session.created'; session: SessionConfig }
  | { type: 'response.audio.delta'; delta: string }  // base64 encoded PCM16
  | { type: 'response.text.delta'; delta: string }
  | { type: 'response.done' }
  | { type: 'input_audio_buffer.speech_started' }
  | { type: 'input_audio_buffer.speech_stopped' }
  | { type: 'error'; error: { message: string; code: string } };

export interface SessionConfig {
  modalities: ('audio' | 'text')[];
  voice: 'alloy' | 'echo' | 'fable' | 'onyx' | 'nova' | 'shimmer';
  turn_detection: { type: 'server_vad' } | { type: 'none' } | null;
  instructions?: string;
  temperature?: number;
  max_response_output_tokens?: number | 'inf';
}

export class RealtimeClient extends EventEmitter {
  private ws: WebSocket | null = null;
  private isConnected = false;

  async connect(sessionConfig: SessionConfig): Promise<void> {
    return new Promise((resolve, reject) => {
      this.ws = new WebSocket(REALTIME_URL, [], {
        headers: {
          Authorization: `Bearer ${OPENAI_API_KEY}`,
          'OpenAI-Beta': 'realtime=v1',
        },
      });

      this.ws.onopen = () => {
        this.isConnected = true;
        // セッション設定を送信
        this.send({
          type: 'session.update',
          session: sessionConfig,
        });
      };

      this.ws.onmessage = (event) => {
        try {
          const data: RealtimeEvent = JSON.parse(event.data);
          this.emit(data.type, data);

          if (data.type === 'session.created') {
            resolve();
          }
        } catch (e) {
          console.error('[RealtimeClient] JSON parse error:', e);
        }
      };

      this.ws.onerror = (error) => {
        this.isConnected = false;
        reject(new Error(`WebSocket error: ${JSON.stringify(error)}`));
      };

      this.ws.onclose = (event) => {
        this.isConnected = false;
        this.emit('disconnected', { code: event.code, reason: event.reason });
      };

      // 接続タイムアウト（10秒）
      setTimeout(() => {
        if (!this.isConnected) {
          reject(new Error('Connection timeout'));
        }
      }, 10000);
    });
  }

  sendAudioChunk(base64Audio: string): void {
    if (!this.isConnected) return;
    this.send({
      type: 'input_audio_buffer.append',
      audio: base64Audio,
    });
  }

  commitAudio(): void {
    if (!this.isConnected) return;
    this.send({ type: 'input_audio_buffer.commit' });
    this.send({ type: 'response.create' });
  }

  interrupt(): void {
    if (!this.isConnected) return;
    this.send({ type: 'response.cancel' });
    this.send({ type: 'input_audio_buffer.clear' });
  }

  disconnect(): void {
    this.ws?.close();
    this.ws = null;
    this.isConnected = false;
  }

  private send(event: object): void {
    if (this.ws?.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(event));
    }
  }
}
```

## 音声録音・再生パイプラインの構築

### サンプリングレート変換ユーティリティ

React Native のデフォルト録音フォーマット（44.1kHz）を Realtime API が要求する 24kHz に変換する必要があります：

```typescript
// src/utils/audioConverter.ts

/**
 * 44100Hz Float32Array を 24000Hz Int16Array に変換する
 * ダウンサンプリング + PCM16 エンコード
 */
export function downsampleAudio(
  input: Float32Array,
  inputSampleRate: number,
  outputSampleRate: number = 24000
): Int16Array {
  const ratio = inputSampleRate / outputSampleRate;
  const outputLength = Math.floor(input.length / ratio);
  const output = new Int16Array(outputLength);

  for (let i = 0; i < outputLength; i++) {
    const srcIndex = Math.floor(i * ratio);
    // Float32 (-1.0 〜 1.0) を Int16 (-32768 〜 32767) に変換
    const sample = Math.max(-1, Math.min(1, input[srcIndex]));
    output[i] = sample < 0 ? sample * 0x8000 : sample * 0x7fff;
  }

  return output;
}

/**
 * ArrayBuffer を base64 文字列に変換
 */
export function arrayBufferToBase64(buffer: ArrayBuffer): string {
  const bytes = new Uint8Array(buffer);
  let binary = '';
  for (let i = 0; i < bytes.byteLength; i++) {
    binary += String.fromCharCode(bytes[i]);
  }
  return btoa(binary);
}

/**
 * base64 PCM16 文字列を再生可能な WAV ArrayBuffer に変換
 */
export function pcm16ToWav(
  base64Pcm: string,
  sampleRate: number = 24000
): ArrayBuffer {
  const pcmData = Uint8Array.from(atob(base64Pcm), (c) => c.charCodeAt(0));
  const wavHeader = createWavHeader(pcmData.length, sampleRate);
  const wav = new Uint8Array(wavHeader.byteLength + pcmData.byteLength);
  wav.set(new Uint8Array(wavHeader), 0);
  wav.set(pcmData, wavHeader.byteLength);
  return wav.buffer;
}

function createWavHeader(dataLength: number, sampleRate: number): ArrayBuffer {
  const buffer = new ArrayBuffer(44);
  const view = new DataView(buffer);
  // RIFF チャンク
  view.setUint32(0, 0x52494646, false); // "RIFF"
  view.setUint32(4, 36 + dataLength, true);
  view.setUint32(8, 0x57415645, false); // "WAVE"
  // fmt チャンク
  view.setUint32(12, 0x666d7420, false); // "fmt "
  view.setUint32(16, 16, true);          // Subchunk size
  view.setUint16(20, 1, true);           // PCM format
  view.setUint16(22, 1, true);           // Mono
  view.setUint32(24, sampleRate, true);
  view.setUint32(28, sampleRate * 2, true); // Byte rate
  view.setUint16(32, 2, true);           // Block align
  view.setUint16(34, 16, true);          // Bits per sample
  // data チャンク
  view.setUint32(36, 0x64617461, false); // "data"
  view.setUint32(40, dataLength, true);
  return buffer;
}
```

### useVoiceRecorder カスタムフック

録音の開始・停止とチャンクの送信をカプセル化したカスタムフックを作ります：

```typescript
// src/hooks/useVoiceRecorder.ts
import { useState, useRef, useCallback } from 'react';
import { Audio } from 'expo-av';
import AudioRecord from 'react-native-audio-record';
import { downsampleAudio, arrayBufferToBase64 } from '../utils/audioConverter';
import type { RealtimeClient } from '../services/RealtimeClient';

export function useVoiceRecorder(client: RealtimeClient | null) {
  const [isRecording, setIsRecording] = useState(false);
  const isRecordingRef = useRef(false);

  const startRecording = useCallback(async () => {
    // マイク権限チェック
    const { status } = await Audio.requestPermissionsAsync();
    if (status !== 'granted') {
      throw new Error('マイクの使用許可が必要です');
    }

    // expo-av のオーディオモード設定
    await Audio.setAudioModeAsync({
      allowsRecordingIOS: true,
      playsInSilentModeIOS: true,
      staysActiveInBackground: false,
    });

    // AudioRecord の設定（PCM16・24kHz・モノラル）
    AudioRecord.init({
      sampleRate: 24000,
      channels: 1,
      bitsPerSample: 16,
      audioSource: 6, // VOICE_RECOGNITION
      wavFile: 'temp.wav',
    });

    // チャンク受信コールバック
    AudioRecord.on('data', (data: string) => {
      if (!isRecordingRef.current || !client) return;
      // data は既に base64 エンコード済み PCM16
      client.sendAudioChunk(data);
    });

    AudioRecord.start();
    isRecordingRef.current = true;
    setIsRecording(true);
  }, [client]);

  const stopRecording = useCallback(() => {
    AudioRecord.stop();
    isRecordingRef.current = false;
    setIsRecording(false);
    // 手動モード（none VAD）の場合は commit を送信
    client?.commitAudio();
  }, [client]);

  return { isRecording, startRecording, stopRecording };
}
```

## 音声再生と UI 状態管理

### useAudioPlayer カスタムフック

サーバーからストリーミングされてくる音声チャンクをバッファリングして逐次再生します：

```typescript
// src/hooks/useAudioPlayer.ts
import { useRef, useCallback } from 'react';
import { Audio } from 'expo-av';
import { pcm16ToWav } from '../utils/audioConverter';

export function useAudioPlayer() {
  const audioQueue = useRef<string[]>([]);
  const isPlaying = useRef(false);
  const currentSound = useRef<Audio.Sound | null>(null);

  const enqueueChunk = useCallback((base64Chunk: string) => {
    audioQueue.current.push(base64Chunk);
    if (!isPlaying.current) {
      playNext();
    }
  }, []);

  const playNext = useCallback(async () => {
    if (audioQueue.current.length === 0) {
      isPlaying.current = false;
      return;
    }

    isPlaying.current = true;
    const chunk = audioQueue.current.shift()!;

    try {
      const wavBuffer = pcm16ToWav(chunk);
      const base64Wav = btoa(
        String.fromCharCode(...new Uint8Array(wavBuffer))
      );
      const uri = `data:audio/wav;base64,${base64Wav}`;

      const { sound } = await Audio.Sound.createAsync({ uri }, { shouldPlay: true });
      currentSound.current = sound;

      sound.setOnPlaybackStatusUpdate((status) => {
        if (status.isLoaded && status.didJustFinish) {
          sound.unloadAsync();
          playNext();
        }
      });
    } catch (e) {
      console.error('[AudioPlayer] playback error:', e);
      playNext(); // エラーをスキップして次へ
    }
  }, []);

  const stopAll = useCallback(() => {
    audioQueue.current = [];
    isPlaying.current = false;
    currentSound.current?.stopAsync().catch(() => {});
    currentSound.current?.unloadAsync().catch(() => {});
  }, []);

  return { enqueueChunk, stopAll };
}
```

interface Message {
  id: string;
  role: 'user' | 'assistant';
  text: string;
  timestamp: Date;

const SESSION_CONFIG: SessionConfig = {
  modalities: ['audio', 'text'],
  voice: 'shimmer',
  turn_detection: { type: 'server_vad' },
  instructions:
    'あなたは親切で知識豊富なアシスタントです。日本語で丁寧に、簡潔に回答してください。',
  temperature: 0.8,
  max_response_output_tokens: 1024,
};

export default function VoiceAssistantScreen() {
  const clientRef = useRef<RealtimeClient | null>(null);
  const [isConnected, setIsConnected] = useState(false);
  const [isAISpeaking, setIsAISpeaking] = useState(false);
  const [messages, setMessages] = useState<Message[]>([]);
  const [currentTranscript, setCurrentTranscript] = useState('');
  const scrollViewRef = useRef<ScrollView>(null);

  const { enqueueChunk, stopAll } = useAudioPlayer();
  const { isRecording, startRecording, stopRecording } = useVoiceRecorder(
    clientRef.current
  );

  // WebSocket 接続の確立
  useEffect(() => {
    const client = new RealtimeClient();
    clientRef.current = client;

    client.connect(SESSION_CONFIG)
      .then(() => setIsConnected(true))
      .catch((e) => console.error('接続失敗:', e));

    // イベントリスナー登録
    client.on('response.audio.delta', ({ delta }) => {
      setIsAISpeaking(true);
      enqueueChunk(delta);
    });

    client.on('response.text.delta', ({ delta }) => {
      setCurrentTranscript((prev) => prev + delta);
    });

    client.on('response.done', () => {
      setIsAISpeaking(false);
      setCurrentTranscript((prev) => {
        if (prev.trim()) {
          setMessages((msgs) => [
            ...msgs,
            {
              id: Date.now().toString(),
              role: 'assistant',
              text: prev.trim(),
              timestamp: new Date(),
            },
          ]);
        }
        return '';
      });
    });

    client.on('input_audio_buffer.speech_started', () => {
      // ユーザーが話し始めたら AI の音声を中断
      stopAll();
      client.interrupt();
    });

    return () => {
      client.disconnect();
    };
  }, []);

  const handleMicPress = useCallback(async () => {
    if (isRecording) {
      stopRecording();
      setMessages((msgs) => [
        ...msgs,
        { id: Date.now().toString(), role: 'user', text: '（音声入力中）', timestamp: new Date() },
      ]);
    } else {
      await startRecording();
    }
  }, [isRecording, startRecording, stopRecording]);

  return (
    <View style={styles.container}>
      {/* 会話履歴 */}
      <ScrollView
        ref={scrollViewRef}
        style={styles.messageList}
        onContentSizeChange={() => scrollViewRef.current?.scrollToEnd()}
      >
        {messages.map((msg) => (
          <View
            key={msg.id}
            style={[
              styles.bubble,
              msg.role === 'user' ? styles.userBubble : styles.aiBubble,
            ]}
          >
            <Text style={styles.bubbleText}>{msg.text}</Text>
          </View>
        ))}
        {currentTranscript ? (
          <View style={[styles.bubble, styles.aiBubble, styles.streamingBubble]}>
            <Text style={styles.bubbleText}>{currentTranscript}</Text>
            <ActivityIndicator size="small" color="#888" style={{ marginTop: 4 }} />
          </View>
        ) : null}
      </ScrollView>

      {/* ステータスバー */}
      <Text style={styles.status}>
        {!isConnected
          ? '接続中...'
          : isRecording
          ? '🎙️ 録音中...'
          : isAISpeaking
          ? '🤖 応答中...'
          : '準備完了'}
      </Text>

      {/* マイクボタン */}
      <Pressable
        style={[styles.micButton, isRecording && styles.micButtonActive]}

        onPressOut={isRecording ? stopRecording : undefined}
        disabled={!isConnected}

        <Text style={styles.micIcon}>{isRecording ? '⏹' : '🎙️'}</Text>
      </Pressable>
    </View>
  );
}

## 本番品質のエラーハンドリングと再接続ロジック

### 指数バックオフによる自動再接続

```typescript
// src/hooks/useRealtimeConnection.ts
import { useCallback, useEffect, useRef, useState } from 'react';
import { RealtimeClient, SessionConfig } from '../services/RealtimeClient';

const MAX_RETRIES = 5;
const BASE_DELAY_MS = 1000;

export function useRealtimeConnection(sessionConfig: SessionConfig) {
  const [client, setClient] = useState<RealtimeClient | null>(null);
  const [connectionState, setConnectionState] = useState<
    'disconnected' | 'connecting' | 'connected' | 'error'
  >('disconnected');
  const retryCount = useRef(0);
  const retryTimer = useRef<ReturnType<typeof setTimeout>>();

  const connect = useCallback(async () => {
    setConnectionState('connecting');
    const newClient = new RealtimeClient();

    try {
      await newClient.connect(sessionConfig);
      retryCount.current = 0;
      setClient(newClient);
      setConnectionState('connected');

      newClient.on('disconnected', () => {
        setConnectionState('disconnected');
        scheduleRetry();
      });
    } catch (e) {
      newClient.disconnect();
      setConnectionState('error');
      scheduleRetry();
    }
  }, [sessionConfig]);

  const scheduleRetry = useCallback(() => {
    if (retryCount.current >= MAX_RETRIES) {
      console.error('[Connection] 最大リトライ回数に達しました');
      setConnectionState('error');
      return;
    }

    const delay = BASE_DELAY_MS * Math.pow(2, retryCount.current);
    retryCount.current += 1;
    console.log(`[Connection] ${delay}ms 後に再接続... (${retryCount.current}/${MAX_RETRIES})`);
    retryTimer.current = setTimeout(connect, delay);
  }, [connect]);

  useEffect(() => {
    connect();
    return () => {
      clearTimeout(retryTimer.current);
      client?.disconnect();
    };
  }, []);

  return { client, connectionState };
}
```

### コスト管理：トークン使用量のモニタリング

Realtime API は音声入出力とテキストトークンの両方で課金されます。`response.done` イベントには使用量情報が含まれます：

```typescript
client.on('response.done', (event) => {
  const usage = event.response?.usage;
  if (usage) {
    console.log('[Cost]', {
      inputAudioTokens: usage.input_token_details?.audio_tokens,
      outputAudioTokens: usage.output_token_details?.audio_tokens,
      inputTextTokens: usage.input_token_details?.text_tokens,
      totalTokens: usage.total_tokens,
    });
    // 費用概算: audio input ~$0.06/1K tokens, audio output ~$0.24/1K tokens
  }
});
```

コスト最適化の実践的なヒント：

- `max_response_output_tokens` を適切に制限する（デフォルト `inf`）
- 不必要なシステムプロンプトの長文化を避ける
- 使用しているときだけ接続し、バックグラウンドでは即座に切断する
- 開発時は [AIの API コスト最適化ガイド](/articles/rork-dev/rork-ai-api-cost-optimization-mobile-guide)も参照

### 割り込み処理（AI の発話中にユーザーが話した場合）

高品質な会話体験の要は「ユーザーが AI の発話を自然に遮れること」です。server_vad モードでは `input_audio_buffer.speech_started` イベントが発火した瞬間に、クライアント側で音声再生を停止し `response.cancel` を送信します（前出のコードに実装済み）。

### 波形アニメーションの実装

録音中・再生中の視覚的フィードバックは UX に大きく寄与します：

```typescript
// src/components/WaveformIndicator.tsx
import React, { useEffect, useRef } from 'react';
import { View, StyleSheet, Animated } from 'react-native';

interface Props {
  isActive: boolean;
  color?: string;
  barCount?: number;
}

export function WaveformIndicator({ isActive, color = '#1a73e8', barCount = 5 }: Props) {
  const animations = useRef(
    Array.from({ length: barCount }, () => new Animated.Value(0.3))
  ).current;

  useEffect(() => {
    if (!isActive) {
      animations.forEach((anim) => anim.setValue(0.3));
      return;
    }

    const loops = animations.map((anim, i) =>
      Animated.loop(
        Animated.sequence([
          Animated.delay(i * 80),
          Animated.timing(anim, {
            toValue: 1,
            duration: 300,
            useNativeDriver: true,
          }),
          Animated.timing(anim, {
            toValue: 0.3,
            duration: 300,
            useNativeDriver: true,
          }),
        ])
      )
    );

    loops.forEach((loop) => loop.start());
    return () => loops.forEach((loop) => loop.stop());
  }, [isActive]);

  return (
    <View style={styles.container}>
      {animations.map((anim, i) => (
        <Animated.View
          key={i}
          style={[
            styles.bar,
            { backgroundColor: color, transform: [{ scaleY: anim }] },
          ]}
        />
      ))}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flexDirection: 'row', alignItems: 'center', gap: 4, height: 40 },
  bar: { width: 4, height: 30, borderRadius: 2 },
});
```

## セキュリティ設計：API キーの扱いと本番環境

### クライアントサイドでの API キー利用の問題

現在の実装ではクライアントアプリから直接 OpenAI Realtime API に接続しています。この設計の**最大のリスクはAPI キーがアプリバンドルに含まれること**です。デコンパイルにより抜き取られる危険があります。

本番アプリでは必ず**サーバー側中継レイヤー**を挟む設計にしてください：

```
ユーザーデバイス
    ↕ WebSocket（独自認証トークン）
バックエンドサーバー（Cloudflare Workers / Node.js）
    ↕ WebSocket
OpenAI Realtime API（OpenAI API キー）
```

Cloudflare Workers での中継サーバーの実装については、[Rork Max の Cloudflare AI バックエンド構築ガイド](/articles/rork-dev/rork-cloudflare-ai-backend) も参考にしてください。

### 中継サーバーの最小実装例（Node.js / WebSocket）

```typescript
// server/realtimeProxy.ts （Node.js + ws ライブラリ）
import { WebSocketServer, WebSocket } from 'ws';
import { verifyAppToken } from './auth';  // アプリ独自認証

const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', async (clientWs, req) => {
  // アプリのJWTトークンを検証
  const token = req.headers['x-app-token'] as string;
  const user = await verifyAppToken(token);
  if (!user) {
    clientWs.close(4001, 'Unauthorized');
    return;
  }

  // OpenAI への接続（API キーはサーバー環境変数から）
  const openaiWs = new WebSocket(
    'wss://api.openai.com/v1/realtime?model=gpt-4o-realtime-preview-2024-12-17',
    {
      headers: {
        Authorization: `Bearer ${process.env.OPENAI_API_KEY}`,
        'OpenAI-Beta': 'realtime=v1',
      },
    }
  );

  // 双方向プロキシ
  clientWs.on('message', (data) => openaiWs.send(data));
  openaiWs.on('message', (data) => clientWs.send(data));

  clientWs.on('close', () => openaiWs.close());
  openaiWs.on('close', () => clientWs.close());
});
```

## よくある問題と対処法

**Q: WebSocket 接続が確立できず、エラーが返ってくる**

A: Realtime API はまだベータ機能であるため、アクセスには有効な OpenAI API キーが必要です。ダッシュボードでキーの `Realtime API` 機能が有効になっているか確認してください。また、ネットワーク環境によっては WebSocket の持続接続がプロキシやファイアウォールによってブロックされる場合があります。

**Q: 音声が再生されるものの、大きなノイズやプチプチ音が入る**

A: PCM16 のサンプリングレートが 24kHz になっていない可能性が高いです。`AudioRecord.init` の `sampleRate` を `24000` に設定してください。また、チャンクのサイズが小さすぎると再生バッファが不安定になります。`react-native-audio-record` のバッファサイズ設定を調整してみてください。

**Q: server_vad モードで AI が常に発話を途中で切ってしまう**

A: VAD の感度が高すぎる場合は、セッション設定で `turn_detection` に追加パラメータを設定します：

```json
"turn_detection": {
  "type": "server_vad",
  "threshold": 0.5,
  "prefix_padding_ms": 300,
  "silence_duration_ms": 600
}
```

`silence_duration_ms` を大きくするとユーザーの発話終了をより長く待ちます。

**Q: API コストが想定より高くなってしまう**

A: Realtime API の音声処理はテキストチャットより高コストです。以下を確認してください：接続しっぱなしになっていないか（アプリのバックグラウンド移行時に切断する）、`max_response_output_tokens` が適切に設定されているか、不要なシステムプロンプトを削っているか。

**Q: iOS でバックグラウンド中に音声再生が止まる**

A: `Audio.setAudioModeAsync` で `staysActiveInBackground: true` を設定し、必要に応じて `playsInSilentModeIOS: true` も加えてください。また Info.plist に `UIBackgroundModes` の `audio` 追加が必要です。

## まとめ

OpenAI Realtime API を Rork Max アプリに統合することで、従来の多段パイプライン方式では実現が難しかった**超低レイテンシの自然な音声対話**を、比較的シンプルなコードで実現できます。

本記事で実装した主要なコンポーネントを振り返ります：

- **RealtimeClient**: WebSocket 接続・イベント管理の責務を分離したクラス
- **useVoiceRecorder**: 録音・チャンク送信をカプセル化したフック
- **useAudioPlayer**: バッファキュー方式による滑らかな音声再生フック
- **useRealtimeConnection**: 指数バックオフによる堅牢な再接続ロジック
- **WaveformIndicator**: 会話状態を視覚的に伝えるアニメーションコンポーネント

次のステップとして、[マルチAIオーケストレーションガイド](/articles/rork-ai/rork-multi-ai-orchestration-guide) と組み合わせることで、Realtime API を会話のインターフェースとして、バックエンドの複数 AI エージェントを音声で操作するシステムを構築することもできます。

音声 AI の進化は目覚ましく、Realtime API はその最前線にあります。ぜひ手を動かして、ユーザーに驚きを与えるアプリを作り上げてください。
