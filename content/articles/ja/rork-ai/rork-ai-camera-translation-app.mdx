## はじめに — カメラ翻訳アプリで「言語の壁」をなくす

海外旅行中にメニューが読めなかった、街中の看板の意味が分からなかった——そんな経験は誰にでもあるのではないでしょうか。スマートフォンのカメラで撮影するだけでテキストを認識・翻訳してくれるアプリは、まさに現代の「翻訳機」として多くの人に求められています。

この記事では、AIノーコード開発ツール **Rork** と **Google Cloud Vision API（OCR）**・**Cloud Translation API** を組み合わせて、カメラ翻訳アプリをゼロから構築する方法を解説します。

この記事で学べること：

- Rork を使ったカメラ連携とリアルタイム画像キャプチャの実装方法
- Google Cloud Vision API で画像内のテキストを認識（OCR）する仕組み
- Cloud Translation API で認識したテキストを任意の言語に翻訳する手順
- ユーザーが使いやすい翻訳 UI の設計パターン

**対象読者**: Rork の基本操作ができ、API 連携に挑戦したい中級者の方

---

## 前提知識・環境準備

### 必要なもの

アプリ開発を始める前に、以下をご用意ください。

- **Rork アカウント**（Free または Max プラン）
- **Google Cloud Platform（GCP）アカウント**（無料枠あり）
- スマートフォン実機（カメラ機能テスト用）

### Google Cloud API の有効化

GCP コンソール（[console.cloud.google.com](https://console.cloud.google.com)）にログインし、以下の手順で API を有効にします。

1. 新しいプロジェクトを作成（例：`camera-translate-app`）
2. 「API とサービス」→「ライブラリ」を開く
3. **Cloud Vision API** を検索して有効化
4. **Cloud Translation API** を検索して有効化
5. 「認証情報」→「API キーを作成」でキーを取得

取得した API キーは後ほど Rork のプロンプトで使用します。コード例では `YOUR_GOOGLE_API_KEY` というプレースホルダーを使用しますので、実際のキーに置き換えてください。

## カメラ連携と画像キャプチャの実装

### expo-camera を活用する

Rork（React Native / Expo ベース）では、`expo-camera` を使ってカメラ機能を実装します。以下のプロンプトで Rork にコードを生成させましょう。

```
expo-cameraを使って以下の機能を実装してください：
- カメラのアクセス許可リクエスト
- フロント/リアカメラの切り替えボタン
- 撮影ボタンで静止画をキャプチャ
- キャプチャした画像をBase64形式で取得
```

生成されるコードのポイント部分は以下のようになります。

```tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useRef, useState } from 'react';

export default function CameraScreen() {
  const [permission, requestPermission] = useCameraPermissions();
  const [facing, setFacing] = useState<'back' | 'front'>('back');
  const cameraRef = useRef<CameraView>(null);

  // 写真を撮影してBase64データを取得
  const takePicture = async () => {
    if (!cameraRef.current) return;
    const photo = await cameraRef.current.takePictureAsync({
      base64: true,      // Base64形式で取得（Vision APIに送信するため）
      quality: 0.8,      // 画質（0〜1）
    });
    // photo.base64 をAPIに渡す
    return photo.base64;
  };

  return (
    <CameraView ref={cameraRef} facing={facing} style={{ flex: 1 }}>
      {/* カメラUI */}
    </CameraView>
  );
}
```

**ポイント**: `base64: true` を指定することで、画像を直接 Google Cloud Vision API に送信できる形式で取得できます。

## OCR 機能の実装 — Google Cloud Vision API

### Vision API でテキストを認識する

撮影した画像を Google Cloud Vision API に送信して、画像内のテキストを抽出します。

```typescript
// utils/visionApi.ts

const VISION_API_KEY = 'YOUR_GOOGLE_API_KEY'; // 実際のキーに置き換える
const VISION_API_URL = `https://vision.googleapis.com/v1/images:annotate?key=${VISION_API_KEY}`;

export async function detectTextFromImage(base64Image: string): Promise<string> {
  const requestBody = {
    requests: [
      {
        image: {
          content: base64Image, // Base64エンコードされた画像
        },
        features: [
          {
            type: 'TEXT_DETECTION', // テキスト検出モード
            maxResults: 1,
          },
        ],
      },
    ],
  };

  const response = await fetch(VISION_API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestBody),
  });

  const data = await response.json();

  // テキストブロック全体を取得
  const fullText = data.responses?.[0]?.fullTextAnnotation?.text;
  if (!fullText) {
    throw new Error('テキストを検出できませんでした');
  }

  // 期待する出力例: "Hello World\nThis is a test."
  return fullText;
}
```

**`TEXT_DETECTION` vs `DOCUMENT_TEXT_DETECTION`**: 街中の看板など短いテキストには `TEXT_DETECTION`、書類やパンフレットなど構造化されたドキュメントには `DOCUMENT_TEXT_DETECTION` が適しています。用途に合わせて切り替えましょう。

## 翻訳機能の実装 — Cloud Translation API

### テキストを翻訳する

OCR で認識したテキストを Cloud Translation API に渡して翻訳します。

```typescript
// utils/translationApi.ts

const TRANSLATION_API_KEY = 'YOUR_GOOGLE_API_KEY'; // 実際のキーに置き換える
const TRANSLATION_API_URL = `https://translation.googleapis.com/language/translate/v2?key=${TRANSLATION_API_KEY}`;

// 言語コードの対応表
export const LANGUAGES: Record<string, string> = {
  'ja': '日本語',
  'en': '英語',
  'zh': '中国語（簡体字）',
  'ko': '韓国語',
  'fr': 'フランス語',
  'es': 'スペイン語',
  'de': 'ドイツ語',
};

export async function translateText(
  text: string,
  targetLanguage: string
): Promise<{ translatedText: string; detectedSourceLanguage: string }> {
  const requestBody = {
    q: text,          // 翻訳するテキスト
    target: targetLanguage, // 翻訳先言語コード（例: 'ja'）
    format: 'text',
  };

  const response = await fetch(TRANSLATION_API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestBody),
  });

  const data = await response.json();
  const translation = data.data?.translations?.[0];

  if (!translation) {
    throw new Error('翻訳に失敗しました');
  }

  return {
    translatedText: translation.translatedText,
    detectedSourceLanguage: translation.detectedSourceLanguage ?? 'unknown',
    // 期待する出力例: { translatedText: "こんにちは世界", detectedSourceLanguage: "en" }
  };
}
```

**自動言語検出**: `source` パラメータを省略すると、Translation API が元のテキストの言語を自動的に検出します。これにより、ユーザーが翻訳元言語を意識することなく自然に使えるアプリになります。

## OCR と翻訳を組み合わせる

### メイン処理フロー

撮影 → OCR → 翻訳 → 表示、の一連のフローを実装します。

```tsx
// screens/ResultScreen.tsx

import { useState } from 'react';
import { detectTextFromImage } from '../utils/visionApi';
import { translateText } from '../utils/translationApi';

export default function ResultScreen({ capturedBase64 }: { capturedBase64: string }) {
  const [detectedText, setDetectedText] = useState('');
  const [translatedText, setTranslatedText] = useState('');
  const [targetLang, setTargetLang] = useState('ja'); // デフォルト: 日本語
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  const handleTranslate = async () => {
    setIsLoading(true);
    setError('');
    try {
      // Step 1: OCR でテキスト認識
      const text = await detectTextFromImage(capturedBase64);
      setDetectedText(text);

      // Step 2: 翻訳
      const result = await translateText(text, targetLang);
      setTranslatedText(result.translatedText);
    } catch (err) {
      setError('処理中にエラーが発生しました。もう一度お試しください。');
    } finally {
      setIsLoading(false);
    }
  };

  // ... UI の描画
}
```

Rork にこのロジックを追加するプロンプト例：

```
上記の detectTextFromImage と translateText 関数を ResultScreen に統合してください。
「翻訳する」ボタンを押したら、ローディングスピナーを表示しながら
OCR → 翻訳の順に処理し、結果をカード形式で表示してください。
エラー時はアラートダイアログで「もう一度お試しください」と表示してください。
```

## よくあるエラーと対処法

### CORS エラーや API キーエラー

**問題**: `403 Forbidden` または `API key not valid` エラーが表示される

**対処法**: GCP コンソールで API キーの制限を確認してください。開発中は制限なし、本番では Android/iOS アプリの制限を設定するのがおすすめです。また、Vision API と Translation API の両方が有効になっているかも確認しましょう。

### テキストが認識されない

**問題**: 画像を撮影しても `テキストを検出できませんでした` エラーが返る

**対処法**: 以下の点を確認してください。

- 画像が鮮明か（手ぶれ・ピンぼけがないか）
- テキストが十分な大きさで撮影されているか（小さすぎると認識精度が下がる）
- 照明が十分か（暗い環境では認識率が低下する）
- `DOCUMENT_TEXT_DETECTION` への切り替えを試みる

### 翻訳結果が不自然

**問題**: 翻訳結果の日本語が不自然または意味が通じない

**対処法**: OCR の結果に改行や記号が混入している場合があります。翻訳前に `text.replace(/\n/g, ' ').trim()` などで前処理を加えると翻訳精度が向上します。

## 発展的な活用法

### リアルタイム翻訳モード（ライブ翻訳）

一定間隔でカメラフレームをキャプチャし、自動的に OCR → 翻訳を繰り返すことで、カメラをかざすだけで翻訳されるライブモードを実装できます。ただし、API コールが増えるため **コスト管理**が重要です。Vision API の無料枠（月 1,000 回）を超えないよう、フレームレートは 2〜3 秒間隔にするのがバランスのよい設定です。

### 翻訳履歴の保存

`AsyncStorage` や Supabase を使って翻訳履歴をローカル/クラウドに保存すると、旅行中に後で見返せる便利な機能になります。

## FAQ

**Q1. Google Cloud Vision API は無料で使えますか？**

A. はい、月 1,000 ユニットまで無料で使用できます（2026年4月時点）。超過分は 1,000 ユニットあたり $1.50（TEXT_DETECTION の場合）です。個人開発レベルであれば、無料枠内に収まることがほとんどです。GCP コンソールでアラートを設定しておくと安心です。

**Q2. Rork Free プランでもカメラアプリは作れますか？**

A. Rork Free プランでも開発・プレビューは可能ですが、実機でのカメラテストや App Store 公開には Rork Max プランが必要です。まずは Free プランで UI の組み立てを行い、カメラのテストが必要になった段階でアップグレードするという進め方がおすすめです。

**Q3. 翻訳の精度を上げるにはどうすればよいですか？**

A. いくつかの方法があります。①撮影時の画像品質を上げる（明るい場所・安定した手・適切な距離）、②OCR 結果に前処理を加えてノイズを除去する、③特定の専門分野（医療・法律・技術文書）には、より高精度な DeepL API や専門翻訳モデルを検討する、といった方法が効果的です。

**Q4. iOS と Android 両方に対応できますか？**

A. Rork は React Native / Expo ベースのため、一度の開発で iOS・Android 両方に対応できます。カメラのアクセス権限リクエストの文言は `app.json` の `expo.ios.infoPlist` と `expo.android.permissions` で個別に設定してください。

**Q5. API キーをソースコードに直書きするのは危険ですか？**

A. 直書きはセキュリティ上好ましくありません。本番運用では API キーをサーバーサイド（Supabase Edge Functions や Cloudflare Workers）に隠蔽し、アプリはバックエンド経由で API を呼び出す設計を推奨します。開発・テスト段階では GCP コンソールで API キーに IP アドレス制限をかけるだけでも一定のリスク軽減になります。
