音声AIアシスタントといえば OpenAI の Realtime API、というイメージが定着してきた感があります。でも実際に使ってみると、1分あたりのコストがそれなりにかかることに気づきます。個人開発のアプリで毎日使われる会話機能として組み込むには、少し腰が引ける価格帯です。

そんなときに検討してほしいのが **Gemini Live API** です。Google の Gemini モデルをリアルタイム音声会話に特化させたこの API は、テキスト・音声の双方向ストリーミングに対応しており、割り込みや感情的なトーンの変化にも反応できます。コストも OpenAI と比べて競争力があります。

以下では、WebSocket 接続の初期化から、マイク入力のストリーミング、AIレスポンスの再生、そして多くの人が詰まる割り込み処理まで、実際に動くコードとともに順番に説明していきます。

## Gemini Live API の特徴と OpenAI Realtime API との違い

実装に入る前に、Gemini Live API がどんな API なのかを整理しておきます。これを理解しておくと、実装中に「なぜこう書くのか」が腑に落ちます。

Gemini Live API は Google AI Studio（または Vertex AI）経由で利用できる WebSocket ベースのリアルタイム API です。クライアントからサーバーへ音声を送ると、サーバーが音声または テキストで応答を返します。OpenAI の Realtime API と構造は似ていますが、いくつか違いがあります。

**入出力モデルの違い**として、Gemini Live API は音声入力に加えてテキスト入力も同一セッションで混在させられます。「音声で質問して、テキストでフォローアップする」という自然な会話フローが実現しやすいです。

**コスト面**では、2026年4月時点では Gemini 2.0 Flash Live モデルが最もコスト効率よく使えます。大量の会話ターンが発生するユースケースでは、OpenAI Realtime API より安くなるケースがあります（ただし常にそうとは限らないので、自分のユースケースで試算することをおすすめします）。

**割り込みの扱い**では、ユーザーが話している最中に AI も応答しているという状態になったとき、Gemini は自動でセッションを調整してくれます。OpenAI の場合は明示的に `response.cancel` を送る必要がありますが、Gemini の場合は音声入力が来た時点で自動的に処理が切り替わります。

## アーキテクチャ設計

Rork アプリに音声 AI アシスタントを組み込む場合、クライアント（アプリ）とサーバー（Gemini API）の間に何を置くかで構成が変わります。

**直接接続方式**は、アプリから Gemini Live API の WebSocket に直接接続する方法です。実装がシンプルで開発初期に向いていますが、API キーをクライアントに埋め込む必要があるため、本番環境には向きません。

**バックエンドプロキシ方式**は、Cloudflare Workers や Supabase Edge Functions などのバックエンドが Gemini の API キーを保持し、アプリはバックエンドの WebSocket エンドポイントに接続する方法です。API キーが漏洩するリスクを防げます。本番環境ではこちらを選んでください。

まずは理解しやすい直接接続方式で動かす方法を示し、その後バックエンドプロキシへの移行ポイントを説明します。

## WebSocket 接続の実装

Gemini Live API との WebSocket 接続を確立するコアロジックです。接続 URL の形式が独特なので、ここを正確に実装することが最初の壁です。

```typescript
// hooks/useGeminiLive.ts
import { useState, useEffect, useRef, useCallback } from 'react';

const GEMINI_API_KEY = 'YOUR_GEMINI_API_KEY'; // 本番ではバックエンドに移動
const MODEL = 'gemini-2.0-flash-live-001';
const GEMINI_WS_URL = `wss://generativelanguage.googleapis.com/ws/google.ai.generativelanguage.v1beta.GenerativeService.BidiGenerateContent?key=${GEMINI_API_KEY}`;

type SessionState = 'idle' | 'connecting' | 'connected' | 'error';

interface Message {
  role: 'user' | 'model';
  text: string;
}

export const useGeminiLive = () => {
  const ws = useRef<WebSocket | null>(null);
  const [sessionState, setSessionState] = useState<SessionState>('idle');
  const [messages, setMessages] = useState<Message[]>([]);
  const [isModelSpeaking, setIsModelSpeaking] = useState(false);

  const connect = useCallback(async () => {
    if (ws.current?.readyState === WebSocket.OPEN) return;

    setSessionState('connecting');
    ws.current = new WebSocket(GEMINI_WS_URL);

    ws.current.onopen = () => {
      // セッション開始メッセージを送信
      const setupMessage = {
        setup: {
          model: `models/${MODEL}`,
          generation_config: {
            response_modalities: ['AUDIO'], // 音声で応答
            speech_config: {
              voice_config: {
                prebuilt_voice_config: {
                  voice_name: 'Puck', // 利用可能なボイス: Puck, Charon, Kore, Fenrir, Aoede
                },
              },
            },
          },
          system_instruction: {
            parts: [
              {
                text: 'あなたは親切で丁寧な日本語アシスタントです。簡潔に、でも温かみのある口調で話してください。',
              },
            ],
          },
        },
      };

      ws.current?.send(JSON.stringify(setupMessage));
      setSessionState('connected');
    };

    ws.current.onmessage = (event) => {
      handleServerMessage(event.data);
    };

    ws.current.onerror = (error) => {
      console.error('WebSocket error:', error);
      setSessionState('error');
    };

    ws.current.onclose = (event) => {
      console.log('WebSocket closed:', event.code, event.reason);
      setSessionState('idle');
    };
  }, []);

  const disconnect = useCallback(() => {
    ws.current?.close();
    ws.current = null;
    setSessionState('idle');
  }, []);

  const handleServerMessage = (data: string) => {
    try {
      const message = JSON.parse(data);

      // セットアップ完了確認
      if (message.setupComplete) {
        console.log('Gemini Live session ready');
        return;
      }

      // モデルの音声/テキストレスポンス処理
      if (message.serverContent) {
        const { modelTurn, turnComplete } = message.serverContent;

        if (modelTurn?.parts) {
          for (const part of modelTurn.parts) {
            if (part.text) {
              // テキストレスポンスを会話履歴に追加
              setMessages((prev) => [
                ...prev,
                { role: 'model', text: part.text },
              ]);
            }

            if (part.inlineData?.mimeType?.startsWith('audio/')) {
              // 音声レスポンスの処理（後述）
              playAudioResponse(part.inlineData.data);
              setIsModelSpeaking(true);
            }
          }
        }

        if (turnComplete) {
          setIsModelSpeaking(false);
        }
      }
    } catch (error) {
      console.error('Error parsing server message:', error);
    }
  };

  return {
    connect,
    disconnect,
    sessionState,
    messages,
    isModelSpeaking,
    ws,
    setMessages,
  };
};
```

## マイク入力とオーディオストリーミングの実装

ユーザーの声をリアルタイムでキャプチャして Gemini に送るパートです。ここが最も実装で詰まりやすい部分です。

Gemini Live API が期待する音声フォーマットは **PCM 16bit、16kHz、モノラル** です。expo-av でデフォルト録音すると異なるフォーマットになることがあるため、明示的に指定する必要があります。

```typescript
// hooks/useAudioCapture.ts
import { Audio } from 'expo-av';
import { useRef, useCallback } from 'react';

// Gemini Live API が要求する音声フォーマット
const RECORDING_OPTIONS: Audio.RecordingOptions = {
  android: {
    extension: '.wav',
    outputFormat: Audio.AndroidOutputFormat.PCM_16BIT,
    audioEncoder: Audio.AndroidAudioEncoder.DEFAULT,
    sampleRate: 16000,
    numberOfChannels: 1,
    bitRate: 256000,
  },
  ios: {
    extension: '.wav',
    outputFormat: Audio.IOSOutputFormat.LINEARPCM,
    audioQuality: Audio.IOSAudioQuality.HIGH,
    sampleRate: 16000,
    numberOfChannels: 1,
    bitRate: 128000,
    linearPCMBitDepth: 16,
    linearPCMIsBigEndian: false,
    linearPCMIsFloat: false,
  },
  web: {
    mimeType: 'audio/webm',
    blobEvent: {
      timeslice: 250, // 250ms ごとにチャンクで送信
    },
  },
};

export const useAudioCapture = (
  onAudioChunk: (base64Audio: string) => void
) => {
  const recording = useRef<Audio.Recording | null>(null);
  const isCapturing = useRef(false);

  const startCapture = useCallback(async () => {
    try {
      // マイク権限の確認
      const permission = await Audio.requestPermissionsAsync();
      if (\!permission.granted) {
        throw new Error('マイクの権限が必要です');
      }

      await Audio.setAudioModeAsync({
        allowsRecordingIOS: true,
        playsInSilentModeIOS: true,
      });

      const { recording: newRecording } = await Audio.Recording.createAsync(
        RECORDING_OPTIONS
      );

      recording.current = newRecording;
      isCapturing.current = true;

      // 500ms ごとに音声データを取得して送信
      const interval = setInterval(async () => {
        if (\!isCapturing.current || \!recording.current) {
          clearInterval(interval);
          return;
        }

        try {
          const status = await recording.current.getStatusAsync();
          if (status.isRecording) {
            // 録音データをBase64で取得して送信
            // 注: これは概念的な実装。実際には録音ファイルを読み取る処理が必要
            const uri = recording.current.getURI();
            if (uri) {
              const base64 = await fileUriToBase64(uri);
              onAudioChunk(base64);
            }
          }
        } catch (e) {
          console.error('Audio chunk error:', e);
        }
      }, 500);

    } catch (error) {
      console.error('Failed to start audio capture:', error);
      throw error;
    }
  }, [onAudioChunk]);

  const stopCapture = useCallback(async () => {
    isCapturing.current = false;
    if (recording.current) {
      await recording.current.stopAndUnloadAsync();
      recording.current = null;
    }
  }, []);

  return { startCapture, stopCapture };
};

// Base64変換ヘルパー
const fileUriToBase64 = async (uri: string): Promise<string> => {
  const response = await fetch(uri);
  const blob = await response.blob();
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onloadend = () => {
      const base64 = (reader.result as string).split(',')[1];
      resolve(base64);
    };
    reader.onerror = reject;
    reader.readAsDataURL(blob);
  });
};
```

### 音声データを Gemini に送信する

取得した Base64 音声データを WebSocket 経由で送信します。

```typescript
// WebSocketへ音声を送信する関数（useGeminiLive に追加）
const sendAudioChunk = useCallback((base64Audio: string) => {
  if (ws.current?.readyState \!== WebSocket.OPEN) return;

  const audioMessage = {
    realtimeInput: {
      mediaChunks: [
        {
          mimeType: 'audio/pcm;rate=16000',
          data: base64Audio,
        },
      ],
    },
  };

  ws.current.send(JSON.stringify(audioMessage));
}, []);

// テキストを送信する関数（音声と混在させられる）
const sendText = useCallback((text: string) => {
  if (ws.current?.readyState \!== WebSocket.OPEN) return;

  const textMessage = {
    clientContent: {
      turns: [
        {
          role: 'user',
          parts: [{ text }],
        },
      ],
      turnComplete: true,
    },
  };

  ws.current.send(JSON.stringify(textMessage));

  // ユーザーメッセージを履歴に追加
  setMessages((prev) => [...prev, { role: 'user', text }]);
}, []);
```

## AIレスポンス音声の再生

Gemini から受け取った音声データを再生する処理です。`expo-av` の `Sound` を使います。

```typescript
// utils/audioPlayer.ts
import { Audio } from 'expo-av';

let currentSound: Audio.Sound | null = null;

export const playAudioResponse = async (base64Audio: string) => {
  try {
    // 前の音声が再生中なら停止
    if (currentSound) {
      await currentSound.stopAsync();
      await currentSound.unloadAsync();
      currentSound = null;
    }

    await Audio.setAudioModeAsync({
      allowsRecordingIOS: false,
      playsInSilentModeIOS: true,
      staysActiveInBackground: false,
    });

    // Base64 から音声を読み込んで再生
    const { sound } = await Audio.Sound.createAsync(
      { uri: `data:audio/pcm;base64,${base64Audio}` },
      { shouldPlay: true }
    );

    currentSound = sound;

    sound.setOnPlaybackStatusUpdate((status) => {
      if (status.isLoaded && status.didJustFinish) {
        sound.unloadAsync();
        currentSound = null;
      }
    });
  } catch (error) {
    console.error('Audio playback error:', error);
    // エラーが発生しても会話は継続できるよう、ここでは投げ直さない
  }
};

export const stopAudioPlayback = async () => {
  if (currentSound) {
    await currentSound.stopAsync();
    await currentSound.unloadAsync();
    currentSound = null;
  }
};
```

## 割り込み処理の実装

音声AIアシスタントとして自然な会話を実現するには、「ユーザーが話し始めたときに AI の応答を止める」割り込み処理が欠かせません。Gemini Live API ではこれが比較的シンプルに実装できます。

```typescript
// VoiceAssistantScreen.tsx の核心部分
import React, { useState, useCallback } from 'react';
import { View, TouchableOpacity, Text, StyleSheet } from 'react-native';
import { useGeminiLive } from '../hooks/useGeminiLive';
import { useAudioCapture } from '../hooks/useAudioCapture';
import { stopAudioPlayback } from '../utils/audioPlayer';

export const VoiceAssistantScreen = () => {
  const [isListening, setIsListening] = useState(false);
  const {
    connect,
    disconnect,
    sessionState,
    messages,
    isModelSpeaking,
    sendAudioChunk,
    sendText,
  } = useGeminiLive();

  const handleAudioChunk = useCallback((base64Audio: string) => {
    sendAudioChunk(base64Audio);
  }, [sendAudioChunk]);

  const { startCapture, stopCapture } = useAudioCapture(handleAudioChunk);

  const handleMicPress = useCallback(async () => {
    if (\!isListening) {
      // 録音開始 — AI が話している場合は割り込む
      if (isModelSpeaking) {
        await stopAudioPlayback(); // AI の音声を止める
        // Gemini 側への割り込みシグナル（音声入力の開始が自動的に割り込みとして認識される）
      }

      await startCapture();
      setIsListening(true);
    } else {
      // 録音停止
      await stopCapture();
      setIsListening(false);
    }
  }, [isListening, isModelSpeaking, startCapture, stopCapture]);

  return (
    <View style={styles.container}>
      {/* 会話履歴の表示 */}
      <View style={styles.messagesContainer}>
        {messages.map((msg, index) => (
          <View
            key={index}
            style={[
              styles.messageBubble,
              msg.role === 'user' ? styles.userBubble : styles.modelBubble,
            ]}
          >
            <Text style={styles.messageText}>{msg.text}</Text>
          </View>
        ))}
      </View>

      {/* ステータス表示 */}
      {isModelSpeaking && (
        <Text style={styles.statusText}>AI が話しています...</Text>
      )}

      {/* マイクボタン */}
      <TouchableOpacity
        style={[styles.micButton, isListening && styles.micButtonActive]}
        onPress={handleMicPress}
        onLongPress={() => {}} // 長押しトリガーも設定可能
      >
        <Text style={styles.micButtonText}>
          {isListening ? '⏹ 停止' : '🎤 話す'}
        </Text>
      </TouchableOpacity>

      {/* 接続ボタン */}
      <TouchableOpacity
        style={styles.connectButton}
        onPress={sessionState === 'connected' ? disconnect : connect}
      >
        <Text>
          {sessionState === 'connected' ? '切断' : '接続'}
        </Text>
      </TouchableOpacity>
    </View>
  );
};
```

## よくある落とし穴と対処法

実際に実装してみると、多くの人が同じ箇所で詰まります。ここにまとめておきます。

**「setupComplete が来ない」問題**: WebSocket が接続されると `onopen` が発火しますが、Gemini のセッションはその後に `setup` メッセージを送って初めて開始します。`setupComplete` のレスポンスが来る前に音声を送ると無視されます。`setupComplete` の受信をフラグで管理して、それ以降だけ送信するようにしてください。

**音声フォーマットのミスマッチ**: Gemini Live API に送る音声は `audio/pcm;rate=16000` が基本です。これ以外のフォーマットを送ると無言のまま応答が来なくなります（エラーが返ってこないので気づきにくい）。録音設定で `sampleRate: 16000` と `numberOfChannels: 1` を明示することが重要です。

**iOS のオーディオセッション競合**: 録音中と再生中でオーディオモードが異なるため、`Audio.setAudioModeAsync` の設定を切り替えるタイミングを誤るとクラッシュしたり無音になったりします。録音開始前は `allowsRecordingIOS: true`、再生前は `allowsRecordingIOS: false` に切り替えることを忘れないでください。

**長い沈黙でのセッション切断**: Gemini Live API はアクティビティのないセッションを自動で切断します。通常の会話アプリなら問題ありませんが、ユーザーが長時間操作しない場合に備えて再接続ロジックを用意しておくとよいです。

```typescript
// 再接続ロジックの例
const handleConnectionError = useCallback(async () => {
  let retries = 0;
  const maxRetries = 3;

  while (retries < maxRetries) {
    try {
      await new Promise(resolve => setTimeout(resolve, 1000 * (retries + 1)));
      await connect();
      console.log('再接続成功');
      return;
    } catch (e) {
      retries++;
      console.log(`再接続試行 ${retries}/${maxRetries}`);
    }
  }

  console.error('再接続に失敗しました。ユーザーに通知してください。');
}, [connect]);
```

## バックエンドプロキシへの移行

本番環境では API キーをクライアントに持たせることは避けてください。Cloudflare Workers でシンプルなプロキシを作る例を示します。

```javascript
// Cloudflare Worker (ws-proxy.js)
// wrangler.toml の [vars] に GEMINI_API_KEY を設定する

export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    if (request.headers.get('Upgrade') \!== 'websocket') {
      return new Response('WebSocket only', { status: 400 });
    }

    // クライアントとの WebSocket を開く
    const [client, server] = Object.values(new WebSocketPair());
    server.accept();

    // Gemini との WebSocket を開く
    const geminiUrl = `wss://generativelanguage.googleapis.com/ws/google.ai.generativelanguage.v1beta.GenerativeService.BidiGenerateContent?key=${env.GEMINI_API_KEY}`;
    const geminiWs = new WebSocket(geminiUrl);

    // クライアント → Gemini への中継
    server.addEventListener('message', (event) => {
      if (geminiWs.readyState === WebSocket.OPEN) {
        geminiWs.send(event.data);
      }
    });

    // Gemini → クライアントへの中継
    geminiWs.addEventListener('message', (event) => {
      server.send(event.data);
    });

    // 接続クローズのハンドリング
    server.addEventListener('close', () => geminiWs.close());
    geminiWs.addEventListener('close', () => server.close());

    return new Response(null, { status: 101, webSocket: client });
  },
};
```

アプリ側の接続先をプロキシの URL に変更するだけで、他のコードはほぼそのまま使えます。

## まとめ

Gemini Live API を使った音声 AI アシスタントは、実装の複雑さは確かにありますが、それを超える価値があります。まずは開発者モードで API キーを直接使ってローカルで動かしてみてください。音声が返ってきた瞬間の感覚は、テキストのやり取りとはまったく違う体験です。

次のステップとしては、Cloudflare Workers のプロキシを立てて、実際のデバイスで友人や家族に試してもらうことをおすすめします。使い勝手へのフィードバックを得てから、ボイスの種類やシステムプロンプトを調整すると、完成度が一気に上がります。

音声 AI は、まだ多くのアプリが本格的に取り入れていない領域です。今のうちに実装経験を積んでおくと、次のアプリアイデアで大きなアドバンテージになるかもしれません。
