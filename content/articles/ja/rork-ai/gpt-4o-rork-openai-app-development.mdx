Rork は Gemini 2.0 Flash をデフォルトの AI エンジンとして使っていますが、**完成したアプリの中に外部 AI サービスを組み込む**ことも可能です。中でも OpenAI の **GPT-4o** は、テキスト生成・画像理解・音声処理を1つのモデルで扱えるマルチモーダル AI として、モバイルアプリとの相性が抜群です。

この記事では、Rork で構築したアプリに GPT-4o を統合する方法を、API キーの取得から具体的な実装パターンまで順を追って解説します。

## GPT-4o とは何か

GPT-4o（"o" は "omni" の略）は、OpenAI が提供するフラッグシップモデルです。テキスト・画像・音声を単一のモデルで処理できる点が最大の特徴で、従来の GPT-4 Turbo と比較して応答速度が大幅に改善されています。

Rork アプリに GPT-4o を組み込むことで、次のような機能を実現できます。

**テキストベースの AI 機能**: チャットボット、文章要約、翻訳、コンテンツ生成など。ユーザーが自然言語で質問すると、GPT-4o が即座に回答を生成します。

**画像認識機能**: ユーザーがカメラで撮影した写真をGPT-4o に送信し、画像の内容を分析させることができます。料理の写真からレシピを提案する、商品写真から情報を抽出するといった活用が可能です。

**構造化データ出力**: JSON モードを使えば、GPT-4o の出力を構造化されたデータとして受け取れます。アプリの UI に直接バインドしやすい形式でレスポンスを取得できるため、パース処理のコードが大幅に簡素化されます。

## OpenAI API キーの取得と管理

GPT-4o を使うには、まず OpenAI の API キーが必要です。

OpenAI のプラットフォーム（platform.openai.com）にアクセスし、アカウントを作成します。ダッシュボードの「API Keys」セクションから新しいキーを生成してください。

API キーの管理には注意が必要です。**キーをアプリのソースコードに直接ハードコードしてはいけません**。React Native アプリの場合、JavaScript バンドルはデバイス上で展開されるため、コード内のキーは比較的簡単に抽出されてしまいます。

推奨されるアプローチは、**バックエンドプロキシ**を介して API を呼び出す方法です。Supabase Edge Functions や Cloudflare Workers を中継サーバーとして設置し、API キーはサーバー側の環境変数に保管します。アプリからはプロキシサーバーにリクエストを送り、サーバーが OpenAI API を呼び出してレスポンスを返す構成です。

## Rork でのプロンプト設計：GPT-4o チャット機能

Rork に「GPT-4o を使ったチャットボットアプリを作って」と指示する際、プロンプトの書き方が成果物の品質を左右します。

効果的なプロンプトの例を紹介します。

```
AI チャットアプリを作ってください。
- ホーム画面にチャットインターフェースを配置
- ユーザーのメッセージと AI の応答をバブル形式で表示
- 入力欄とSendボタンをフッターに固定
- OpenAI の GPT-4o API を呼び出す関数を用意
  - エンドポイント: https://api.openai.com/v1/chat/completions
  - モデル: gpt-4o
  - リクエスト形式: messages 配列に会話履歴を含める
- API キーは定数ファイルで管理（後でバックエンドに移行）
- ローディング中はタイピングインジケーターを表示
- エラー時は「接続に失敗しました」のトーストを表示
```

このように、UI のレイアウト、API の呼び出し仕様、エラーハンドリングまで具体的に指示すると、Rork は実用的なチャットアプリを一度の生成で作り上げます。

## API 呼び出しの基本構造

Rork が生成するコードの中核となる API 呼び出し部分は、次のような構造になります。

```typescript
const callGPT4o = async (messages: Message[]) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: messages.map(m => ({
        role: m.role,
        content: m.content,
      })),
      temperature: 0.7,
      max_tokens: 1024,
    }),
  });

  const data = await response.json();
  return data.choices[0].message.content;
};
```

`temperature` パラメータは出力のランダム性を制御します。チャットボットなら 0.7 程度、事実に基づく回答が必要なら 0.2〜0.3 に設定するとよいでしょう。`max_tokens` はレスポンスの最大長です。モバイルアプリでは長すぎる応答は UX を損なうため、用途に応じて調整してください。

## 画像認識機能の実装

GPT-4o のマルチモーダル機能を使えば、ユーザーがカメラで撮影した画像を AI に分析させることができます。

Rork のプロンプトで次のように指示します。

```
カメラで撮影した料理の写真を分析して、
カロリーと栄養素の推定値を表示するアプリを作ってください。
- expo-camera でカメラプレビューを表示
- 撮影ボタンで写真を撮影
- 撮影した画像を base64 エンコードして GPT-4o に送信
- GPT-4o の vision 機能で画像を分析
- 結果をカード形式で表示（料理名、推定カロリー、主要栄養素）
```

画像を GPT-4o に送信する際は、`content` フィールドに画像データを含めた形式でリクエストを構築します。

```typescript
const analyzeImage = async (base64Image: string) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: [
        {
          role: 'user',
          content: [
            {
              type: 'text',
              text: 'この料理の名前、推定カロリー、主要栄養素を教えてください。JSON形式で回答してください。',
            },
            {
              type: 'image_url',
              image_url: {
                url: `data:image/jpeg;base64,${base64Image}`,
                detail: 'low',
              },
            },
          ],
        },
      ],
      response_format: { type: 'json_object' },
    }),
  });

  const data = await response.json();
  return JSON.parse(data.choices[0].message.content);
};
```

`detail` パラメータを `low` に設定すると、画像の解像度を下げてトークン消費を抑えることができます。モバイルアプリではネットワーク帯域の制約もあるため、用途に応じて `low` と `high` を使い分けましょう。

## ストリーミングレスポンスの実装

長い応答を待つ間、ユーザーに何も表示しないのは UX として問題があります。GPT-4o はストリーミングレスポンスをサポートしており、生成されたテキストをリアルタイムで表示できます。

```typescript
const streamGPT4o = async (
  messages: Message[],
  onChunk: (text: string) => void
) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: messages.map(m => ({ role: m.role, content: m.content })),
      stream: true,
    }),
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  while (reader) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n').filter(line => line.startsWith('data: '));

    for (const line of lines) {
      const json = line.replace('data: ', '');
      if (json === '[DONE]') return;

      const parsed = JSON.parse(json);
      const content = parsed.choices[0]?.delta?.content;
      if (content) onChunk(content);
    }
  }
};
```

Rork のプロンプトでストリーミング対応を指示する際は、「応答をリアルタイムで1文字ずつ表示する」「ChatGPT のようにタイピングアニメーションを実装する」と伝えると、適切なコードが生成されます。

## コスト管理のベストプラクティス

GPT-4o の API は従量課金制です。モバイルアプリで大量のユーザーが利用する場合、コストが急速に膨らむ可能性があります。

**リクエスト制限**: ユーザーごとに1日あたりのリクエスト数を制限します。AsyncStorage や Supabase でカウントを管理し、上限に達したらメッセージを表示しましょう。

**会話履歴の圧縮**: チャットの文脈を維持するために全履歴を送信すると、トークン消費が増大します。直近 10 メッセージだけを送信する、または要約を活用するなど、コンテキストウィンドウの管理が重要です。

**モデルの使い分け**: すべてのリクエストに GPT-4o を使う必要はありません。簡単な質問には GPT-4o-mini を、複雑な分析には GPT-4o を使うといった動的なモデル選択を実装すると、コストを大幅に削減できます。

**キャッシュの活用**: 同じ質問に対する回答はローカルにキャッシュしておくことで、不要な API 呼び出しを削減できます。

## GPT-4o と Gemini の使い分け

Rork のデフォルト AI エンジンである Gemini 2.0 Flash と、アプリ内に組み込む GPT-4o は、役割が異なります。

Gemini 2.0 Flash は **Rork がアプリを生成する段階**で使用される AI です。プロンプトからコードを生成し、アプリの構造を設計する役割を担っています。

一方、GPT-4o は**完成したアプリのユーザーが利用する AI 機能**として組み込まれます。チャットボット、画像分析、コンテンツ生成など、エンドユーザー向けの機能を提供します。

つまり、Rork + Gemini でアプリを「作り」、GPT-4o でアプリに「知性を与える」という二重構造になるわけです。もちろん、アプリ内 AI として Gemini API を使うことも可能ですし、Claude API を使うこともできます。プロジェクトの要件に応じて最適なモデルを選択してください。

## 実践的なアプリアイデア

GPT-4o を Rork アプリに組み込む具体的なアイデアをいくつか紹介します。

**AI 語学学習アプリ**: ユーザーの入力を GPT-4o が添削し、文法の間違いを指摘しながら自然な表現を提案します。音声入力と組み合わせれば、スピーキング練習アプリにもなります。

**スマートレシピアシスタント**: 冷蔵庫の中身を写真で撮影し、GPT-4o の画像認識で食材を特定。手持ちの食材で作れるレシピを提案します。

**AI 旅行プランナー**: 旅行先の希望条件を入力すると、GPT-4o が最適な旅程を生成。地図アプリとの連携で、ルート案内まで一気通貫で提供します。

**ドキュメントスキャナー**: カメラで書類を撮影し、GPT-4o で OCR＋要約を実行。契約書のポイント抽出やレシートの家計簿自動入力に活用できます。

## まとめ

Rork と GPT-4o の組み合わせは、AI を活用したモバイルアプリ開発の可能性を大きく広げます。Rork がアプリの骨格を自動生成し、GPT-4o がその中に知的な機能を提供する。この二層構造を理解すれば、コーディング経験が少なくても高度な AI アプリを構築できます。

まずはシンプルなチャットボットから始めて、画像認識やストリーミングといった高度な機能を段階的に追加していくのがおすすめです。OpenAI の API ドキュメントと Rork のプロンプトガイドを併用しながら、自分だけの AI アプリを作ってみましょう。
