## はじめに

毎月の家計管理、手入力が面倒で続かなかった経験はありませんか。レシートをスマホのカメラで撮影するだけで、金額や店舗名を自動認識し、食費・交通費・日用品などのカテゴリに自動分類してくれるアプリがあれば、家計管理のハードルは大きく下がります。

この記事では、Rork を使って AI レシート読み取り機能を搭載した家計簿アプリを構築する方法を解説します。カメラ撮影から OCR（光学文字認識）によるテキスト抽出、AI を活用した支出カテゴリの自動分類、そして月次レポートの可視化まで、一連の実装をステップバイステップで進めていきます。

**この記事で学べること:**

- Rork アプリでカメラ撮影と画像取り込みを実装する方法
- OCR API を使ったレシートからのテキスト抽出
- AI（Gemini API）による支出カテゴリの自動分類
- 月次支出レポートのグラフ表示

**対象読者:** Rork の基本操作を理解しており、API 連携を含むアプリを作りたい中級者の方

## 前提知識と環境準備

このチュートリアルを進めるにあたり、以下の準備が必要です。

- **Rork アカウント**: 無料プランでも基本的な開発は可能ですが、カメラ API のフル活用には Rork Max が推奨されます
- **Google Cloud アカウント**: Cloud Vision API（OCR）を利用するために必要です
- **Gemini API キー**: カテゴリ自動分類に使用します

Rork でカメラ機能を使う基礎については [Rork Max カメラ・ギャラリー活用ガイド](/articles/rork-dev/rork-max-camera-gallery) で詳しく解説していますので、初めての方はそちらも参考にしてください。

## アプリの全体設計

まずアプリの構成を整理しましょう。主に4つの画面で構成します。

```
ReceiptBudgetApp/
├── HomeScreen          # 今月の支出サマリー・最近の記録一覧
├── CaptureScreen       # カメラ撮影 / ギャラリー選択
├── ResultScreen        # OCR結果の確認・カテゴリ編集
└── ReportScreen        # 月次レポート（カテゴリ別グラフ）
```

データの流れは次のとおりです。

1. ユーザーがレシートを撮影またはギャラリーから選択
2. 画像を Cloud Vision API に送信し、テキストを抽出
3. 抽出テキストを Gemini API に送信し、金額・店舗名・カテゴリを構造化
4. 結果をローカルストレージに保存
5. 月次レポートとしてグラフに可視化

## カメラ撮影とギャラリー選択の実装

Rork では `expo-camera` と `expo-image-picker` を使ってカメラ機能を実装します。以下は CaptureScreen のコア部分です。

```typescript
// CaptureScreen.tsx — レシート撮影画面
import * as ImagePicker from 'expo-image-picker';
import { CameraView, useCameraPermissions } from 'expo-camera';

export default function CaptureScreen({ navigation }) {
  const [permission, requestPermission] = useCameraPermissions();
  const cameraRef = useRef(null);

  // カメラで撮影
  const handleCapture = async () => {
    if (cameraRef.current) {
      const photo = await cameraRef.current.takePictureAsync({
        base64: true,  // OCR API に送信するため base64 で取得
        quality: 0.8,  // ファイルサイズと画質のバランス
      });
      navigation.navigate('Result', { imageBase64: photo.base64 });
    }
  };

  // ギャラリーから選択
  const handlePickImage = async () => {
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      base64: true,
      quality: 0.8,
    });

    if (!result.canceled && result.assets[0].base64) {
      navigation.navigate('Result', {
        imageBase64: result.assets[0].base64,
      });
    }
  };

  return (
    // カメラプレビュー + 撮影ボタン + ギャラリーボタンのUI
    // ...
  );
}
```

ポイントは `base64: true` オプションです。次のステップで OCR API に画像データを直接送信するため、Base64 エンコード済みの画像データを取得しておきます。

## Cloud Vision API で OCR テキスト抽出

撮影した画像から文字情報を読み取るには、Google Cloud Vision API の `TEXT_DETECTION` 機能を使います。

```typescript
// services/ocrService.ts — OCR テキスト抽出サービス
const VISION_API_URL =
  'https://vision.googleapis.com/v1/images:annotate';

export async function extractTextFromReceipt(
  imageBase64: string
): Promise<string> {
  const requestBody = {
    requests: [
      {
        image: { content: imageBase64 },
        features: [{ type: 'TEXT_DETECTION', maxResults: 1 }],
      },
    ],
  };

  const response = await fetch(
    `${VISION_API_URL}?key=YOUR_VISION_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(requestBody),
    }
  );

  const data = await response.json();
  // fullTextAnnotation に連結済みのテキストが格納される
  const fullText =
    data.responses?.[0]?.fullTextAnnotation?.text ?? '';

  return fullText;
}
// 期待する出力例:
// "マルエツ 渋谷店\n2026/03/27\n牛乳 ¥198\nパン ¥248\n合計 ¥446"
```

Cloud Vision API は日本語のレシートにも高い精度で対応しています。`fullTextAnnotation.text` にはレシート全体のテキストが改行区切りで格納されるため、次のステップで AI にそのまま渡せます。

## Gemini API によるカテゴリ自動分類

OCR で取得したテキストを Gemini API に渡し、金額・店舗名・カテゴリを構造化データとして抽出します。プロンプトエンジニアリングがこのステップの鍵です。

```typescript
// services/categorizationService.ts — AI カテゴリ分類
const GEMINI_API_URL =
  'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';

interface ReceiptData {
  storeName: string;
  date: string;
  totalAmount: number;
  category: string;
  items: { name: string; price: number }[];
}

export async function categorizeReceipt(
  ocrText: string
): Promise<ReceiptData> {
  const prompt = `
以下はレシートのOCRテキストです。このテキストから以下の情報をJSON形式で抽出してください。

【抽出項目】
- storeName: 店舗名
- date: 日付（YYYY-MM-DD形式）
- totalAmount: 合計金額（数値のみ）
- category: 支出カテゴリ（以下から1つ選択）
  食費, 日用品, 交通費, 外食, 医療費, 衣服, 娯楽, 教育, 通信費, その他
- items: 商品リスト（name と price の配列）

【OCRテキスト】
${ocrText}

【出力形式】
JSONのみを出力してください。説明文は不要です。
`;

  const response = await fetch(
    `${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        contents: [{ parts: [{ text: prompt }] }],
        generationConfig: {
          responseMimeType: 'application/json',
        },
      }),
    }
  );

  const data = await response.json();
  const jsonText =
    data.candidates?.[0]?.content?.parts?.[0]?.text ?? '{}';

  return JSON.parse(jsonText) as ReceiptData;
}
// 期待する出力:
// {
//   "storeName": "マルエツ 渋谷店",
//   "date": "2026-03-27",
//   "totalAmount": 446,
//   "category": "食費",
//   "items": [
//     { "name": "牛乳", "price": 198 },
//     { "name": "パン", "price": 248 }
//   ]
// }
```

`responseMimeType: 'application/json'` を指定することで、Gemini が JSON 形式のみを返すよう制御できます。これにより、レスポンスのパース処理が安定します。

### カテゴリ分類の精度を高めるコツ

AI によるカテゴリ分類の精度を向上させるために、いくつかの工夫が有効です。

- **カテゴリの定義を具体的にする**: 「食費」を「スーパー・コンビニでの食料品購入」のように補足説明を加える
- **過去の分類結果をプロンプトに含める**: 同じ店舗の過去の分類結果を few-shot 例として渡す
- **ユーザーの修正をフィードバックする**: ユーザーがカテゴリを手動修正した場合、その情報を蓄積して精度を改善する

より高度な AI 統合パターンについては、[Rork × マルチAIオーケストレーション完全ガイド](/articles/rork-ai/rork-multi-ai-orchestration-guide) で詳しく解説しています。

## 結果画面と手動修正機能

OCR と AI 分類の結果をユーザーに表示し、必要に応じて手動で修正できる画面を作ります。

```typescript
// ResultScreen.tsx — OCR結果の確認・編集画面
export default function ResultScreen({ route, navigation }) {
  const { imageBase64 } = route.params;
  const [receiptData, setReceiptData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    processReceipt();
  }, []);

  const processReceipt = async () => {
    try {
      // Step 1: OCR でテキスト抽出
      const ocrText = await extractTextFromReceipt(imageBase64);
      // Step 2: AI でカテゴリ分類
      const categorized = await categorizeReceipt(ocrText);
      setReceiptData(categorized);
    } catch (error) {
      Alert.alert('エラー', 'レシートの読み取りに失敗しました');
    } finally {
      setLoading(false);
    }
  };

  const handleSave = async () => {
    // AsyncStorage にレシートデータを保存
    const existing = await AsyncStorage.getItem('receipts');
    const receipts = existing ? JSON.parse(existing) : [];
    receipts.push({
      ...receiptData,
      id: Date.now().toString(),
      createdAt: new Date().toISOString(),
    });
    await AsyncStorage.setItem('receipts', JSON.stringify(receipts));
    navigation.navigate('Home');
  };

  // カテゴリをドロップダウンで変更可能にする
  // 金額の手動修正も対応
  // ...
}
```

ユーザーが AI の分類結果を修正した場合、その修正データを蓄積しておくと、将来的にプロンプトの改善材料として活用できます。

## 月次レポートの可視化

蓄積したレシートデータから月次レポートを生成します。カテゴリ別の支出割合を円グラフで、日別の推移を棒グラフで表示しましょう。

```typescript
// ReportScreen.tsx — 月次支出レポート
import { PieChart } from 'react-native-chart-kit';

export default function ReportScreen() {
  const [monthlyData, setMonthlyData] = useState([]);

  useEffect(() => {
    loadMonthlyReport();
  }, []);

  const loadMonthlyReport = async () => {
    const stored = await AsyncStorage.getItem('receipts');
    const receipts = stored ? JSON.parse(stored) : [];

    // 今月のデータをカテゴリ別に集計
    const now = new Date();
    const thisMonth = receipts.filter((r) => {
      const d = new Date(r.date);
      return (
        d.getMonth() === now.getMonth() &&
        d.getFullYear() === now.getFullYear()
      );
    });

    const categoryTotals = {};
    thisMonth.forEach((r) => {
      const cat = r.category || 'その他';
      categoryTotals[cat] =
        (categoryTotals[cat] || 0) + r.totalAmount;
    });

    // PieChart 用のデータ形式に変換
    const COLORS = [
      '#FF6384', '#36A2EB', '#FFCE56', '#4BC0C0',
      '#9966FF', '#FF9F40', '#C9CBCF', '#7BC67E',
    ];

    const chartData = Object.entries(categoryTotals).map(
      ([name, amount], index) => ({
        name,
        amount,
        color: COLORS[index % COLORS.length],
        legendFontColor: '#333',
        legendFontSize: 14,
      })
    );

    setMonthlyData(chartData);
  };

  return (
    <View style={{ padding: 16 }}>
      <Text style={{ fontSize: 20, fontWeight: 'bold' }}>
        今月の支出レポート
      </Text>
      {monthlyData.length > 0 && (
        <PieChart
          data={monthlyData}
          width={350}
          height={220}
          chartConfig={{
            color: (opacity = 1) =>
              `rgba(0, 0, 0, ${opacity})`,
          }}
          accessor="amount"
          backgroundColor="transparent"
          paddingLeft="15"
        />
      )}
    </View>
  );
}
```

データの可視化をさらに充実させたい方は、[Rork で作る家計簿アプリのデータ可視化ガイド](/articles/rork-dev/rork-budget-tracker-data-visualization) も参考になります。

## よくあるエラーと対処法

### カメラのパーミッションが拒否される

`expo-camera` の `useCameraPermissions` でパーミッションをリクエストした際に拒否された場合、端末の設定画面へ誘導するフォールバック処理を入れておきましょう。`Linking.openSettings()` でアプリの設定画面を直接開けます。

### OCR の精度が低い

レシートの撮影環境によっては OCR の精度が下がることがあります。撮影時のガイドラインとして、明るい場所で撮影すること、レシートをなるべく平らに置くこと、影が入らないように注意することをユーザーに伝える UI を追加すると効果的です。画像の前処理として、コントラスト調整やグレースケール変換を加えるとさらに精度が向上します。

### Gemini API がタイムアウトする

レシートのテキスト量が多い場合や API の負荷が高い場合にタイムアウトが発生することがあります。`AbortController` でタイムアウト処理を実装し、リトライロジックを加えておくと安定します。

## FAQ

**Q: Cloud Vision API の無料枠はどれくらいですか？**

A: Google Cloud Vision API は毎月1,000リクエストまで無料です。個人の家計管理であれば、1日30枚以上のレシートを処理しない限り無料枠内で十分にまかなえます。無料枠を超える場合は1,000リクエストあたり $1.50 の従量課金となります。

**Q: オフラインでも使えますか？**

A: 撮影とデータの閲覧はオフラインでも可能です。ただし、OCR テキスト抽出と AI カテゴリ分類にはインターネット接続が必要です。オフライン時は画像を一時保存しておき、接続が復旧した際にまとめて処理するキューイング機能を実装すると利便性が向上します。

**Q: レシート以外の書類（領収書や請求書）にも対応できますか？**

A: はい、Gemini API のプロンプトを調整することで、領収書や請求書、クレジットカードの明細書など、さまざまな形式の書類に対応できます。プロンプト内の抽出項目を書類の種類に応じてカスタマイズしてください。

**Q: データはどこに保存されますか？**

A: この記事の実装では AsyncStorage（端末内のローカルストレージ）にデータを保存しています。クラウド同期や複数端末での利用が必要な場合は、Supabase などのバックエンド連携を検討してください。バックエンド連携の基本は [Rork × Supabase 認証・リアルタイム連携ガイド](/articles/rork-dev/rork-supabase-auth-realtime-guide) で解説しています。
