## Rork Companionとは何か？

Rork Companionは、Rorkが提供する**AIペアプログラミングパートナー**です。開発中にリアルタイムでコードレビュー、提案、最適化を行い、あなたのアプリ開発を加速させます。

従来のAIコードジェネレータが「ゼロから作成」を得意とするなら、Rork Companionは**既存コードの改善・最適化・デバッグ**を専門とします。

### Rork Companionの主な特徴

- **リアルタイムコード分析**: あなたが書いたコードをその場で解析
- **文脈を保持した提案**: プロジェクト全体の構造を理解した上での改善案
- **UIコンポーネント最適化**: Figmaデザインとの自動連携
- **パフォーマンス診断**: 隠れたボトルネックを自動検出

## Rork Companionを始める3ステップ

### Step 1: Rorkプロジェクトにアクセス

1. [Rork](https://rorkapp.com/) にログイン
2. 既存プロジェクトを開くか、新規作成
3. プロジェクト設定から「Companion機能を有効化」をオン

### Step 2: Companion UIを立ち上げる

Rork Maxエディタの右サイドバーに「Companion」パネルが表示されます。これがあなたの開発アシスタントです。

```
┌─────────────────────────────┐
│ Rork Max エディタ            │
├──────────────┬──────────────┤
│              │ Companion    │
│              │ Panel        │
│  Code Area   ├──────────────┤
│              │ • Chat       │
│              │ • Refactor   │
│              │ • Test Gen   │
└──────────────┴──────────────┘
```

### Step 3: 最初のアシスタント指示を出す

Companionに対して以下のような指示を出してみてください：

- 「このコンポーネントをTypeScript化して」
- 「このロジックはもっと効率的に書けますか？」
- 「このUIは中ピクセルで崩れています。修正パターンを提案してください」

## Rork Companion使用例

### 例1: レガシーコードをモダン化

古い関数をCompanionに見せると：

```javascript
// Before: 古い非同期処理
function fetchUserData(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const user = { id: id, name: "User" };
      resolve(user);
    }, 1000);
  });
}
```

Companionが提案する改善案：

```typescript
// After: async/await + TypeScript
async function fetchUserData(id: string): Promise<User> {
  await delay(1000);
  return { id, name: "User" };
}
```

### 例2: UIバグの検出と修正

「スマートフォンで表示がおかしい」と言うだけで、Companionは：

1. レスポンシブブレークポイント違反を検出
2. Flexboxの問題箇所を指摘
3. 修正コードを自動生成

### 例3: テストケース自動生成

「このロジックのテスト書いて」と指示すると、主要なエッジケースをカバーしたテストが自動生成されます。

## Companion活用のベストプラクティス

### 1. 明確で具体的な質問を心がける

**悪い例**: 「これ改善して」
**良い例**: 「このリスト描画が遅いです。大量データでのパフォーマンス最適化を提案してください」

### 2. コンテキストを与える

- 使用している技術スタック（React、SwiftUI等）を明記
- パフォーマンス目標（読み込み時間◎秒以内など）を共有
- ターゲットデバイス（iPhone SE 第2世代以上、Android 8+など）を指定

### 3. 段階的な改善を指示

一度に全部をCompanionに投げるのではなく、段階的に：

1. 構造的な改善
2. パフォーマンス最適化
3. デザイン品質向上

この順序で進めると、AIの提案も品質が上がります。

## Rork Companionが得意な6つのタスク

### 1. コード品質向上
- 変数命名規則の統一
- 関数の分割と再利用性向上
- コメント・ドキュメンテーション自動生成

### 2. パフォーマンス最適化
- 不要なレンダリングの検出
- バンドルサイズ削減の提案
- メモリリークの予防

### 3. セキュリティ改善
- 認証・認可ロジックのレビュー
- APIキーの流出リスク検出
- SQL インジェクション対策の提案

### 4. テスト自動化
- ユニットテストの骨組み生成
- インテグレーションテストのシナリオ作成
- エッジケース検出

### 5. デザイン連携
- Figmaコンポーネントと実装の乖離検出
- レスポンシブ対応状況の診断
- アクセシビリティ改善提案

### 6. ドキュメンテーション
- JSDoc形式のコメント自動生成
- README更新の提案
- 設定ファイルの説明追加

## Rork Companionの注意点

### AIが誤る可能性もある

Companionの提案は99%正確ですが、複雑なビジネスロジックについては必ず人間がレビューしてください。

### プライベートデータは送信しない

APIキー、顧客情報、機密コードはCompanionに見せないようにしましょう。ローカル処理またはスクラブデータで検証してください。

### 過信は禁物

「Companionがこう言ったから」ではなく、提案理由を理解した上で採用を判断してください。

## よくある質問

### Q: Companionはオフラインで動きますか？

A: いいえ。クラウドベースのため、インターネット接続が必須です。

### Q: 複数プロジェクトで同時に使えますか？

A: はい。アカウント内の全プロジェクトでCompanionが機能します。

### Q: Rork Proプラン以上が必要ですか？

A: Rork Proプラン以上。フリープランではCompanion機能は利用できません。

### Q: Companionのアウトプットはプロジェクトに自動反映されますか？

A: いいえ。全て承認型です。あなたが「適用」ボタンを押さない限り、コードは変わりません。
