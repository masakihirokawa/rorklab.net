## Rork Companion とは？

**Rork Companion** は、Rorkプラットフォームに組み込まれたAIペアプログラマー機能です。単にコードを自動生成するだけでなく、アプリの改善・デバッグ・機能追加について **会話形式でリアルタイムに相談・実装** できるのが大きな特徴です。

従来のAIコーディングツールは「指示を出して待つ」一方通行でしたが、Rork Companion は開発者と対話しながらアプリを進化させていく **ペア開発体験** を提供します。

---

## Rork Companion でできること

### 1. 会話型アプリ改善

「このボタンをもっと目立たせて」「ユーザーがログイン後にダッシュボードにリダイレクトされるようにして」といった自然言語の指示を出すだけで、Companion がコードの該当箇所を特定し、修正案を実装します。

### 2. バグの相談・デバッグ支援

エラーメッセージを貼り付けて「これはなぜ起きているの？」と聞くと、原因を特定して修正コードを提案してくれます。スタックトレースの読み方がわからない初心者でも、Companion が翻訳・解説してくれます。

### 3. 機能の追加・拡張

「プッシュ通知を追加したい」「ダークモードに対応させたい」「SQLiteでオフラインデータを保存したい」など、機能追加の要望を伝えると、実装方法の選択肢を提示し、選んだ方針で実装してくれます。

### 4. コードの説明

「このコードは何をしているの？」と聞くと、コンポーネントやロジックを平易な言葉で解説してくれます。既存のプロジェクトを引き継いだ時や、自動生成されたコードを理解したい時に便利です。

## 効果的な指示の出し方

Rork Companion から最良の結果を得るためには、指示の出し方が重要です。

### 良い指示の例

```
「ユーザーが初回起動時だけオンボーディング画面を表示したい。
AsyncStorageで表示済みフラグを保存して、次回以降はスキップして欲しい。」

「ホーム画面のカード一覧がスクロールできない。
FlatListで実装しているが、親のViewにflex:1が設定されていないかもしれない。確認して修正して。」

「APIからのデータ取得中にローディングスピナーを表示したい。
成功時はデータを表示し、エラー時はリトライボタンを出したい。」
```

### 改善が必要な指示の例と対処法

- 「アプリをよくして」→ 「ホーム画面のローディング速度を改善して」と具体化する
- 「エラーが出た」→ エラーメッセージ全文を貼り付けて「この原因と修正方法を教えて」と伝える
- 「もっとかっこよく」→ 「デザインをマテリアルデザインに合わせて、ボタンにrippleエフェクトを追加して」と基準を示す

### 指示を改善する3つのポイント

**目的を明確に**: 「何を達成したいか」を最初に伝えます。「ユーザーがより簡単にXできるようにしたい」という形式が効果的です。

**現状を伝える**: 「今はこうなっている」→「これをこう変えたい」という構造で伝えると、Companion が変更の意図を正確に理解できます。

**制約・前提を含める**: 「既存のスタイルを崩さずに」「外部ライブラリを追加しないで」などの制約を明示すると、意図しない変更を防げます。

## よくある活用パターン

### パターン1：スクリーンの追加

```
「設定画面を追加したい。以下の項目を設定できるようにして欲しい：
- 通知のオン/オフ
- テーマ（ライト/ダーク）
- フォントサイズ（小/中/大）

設定値はAsyncStorageに保存し、アプリ再起動後も反映されるようにして。」
```

### パターン2：APIとの連携

```
「JSONPlaceholder の /posts エンドポイントからデータを取得して表示したい。

- fetchを使ってデータを取得
- ローディング中はActivityIndicatorを表示
- エラー時はアラートを表示して
- FlatListで一覧表示（タイトルと本文のプレビューを表示）

既存のホーム画面の中に追加してください。」
```

### パターン3：パフォーマンス改善

```
「スクロール時にリストが重くなっている。以下を試して改善して欲しい：
1. FlatListのkeyExtractorが正しく設定されているか確認
2. 不要なre-renderを防ぐためmemo/useCallbackを適用
3. getItemLayoutで固定高さを指定

変更前と変更後で何が変わったか説明もお願い。」
```

## Rork Companion を使う際の注意点

### 変更前にプレビューを確認する

Companion が実装した変更は、適用前にプレビューで確認できます。特に複数のファイルにまたがる変更は、全体の影響を把握してから適用しましょう。

### 段階的に指示を出す

一度に大きな変更を依頼するより、小さなステップに分けて指示を出す方が、意図通りの結果になりやすく、問題が起きた時の切り分けも楽になります。

### スナップショットを活用する

大きな機能追加の前に手動でスナップショットを取っておくと、いつでも元の状態に戻れる安心感があります。

## よくある質問（FAQ）

**Q. Rork Companion は無料で使えますか？**

Rork Companion の基本機能は Rork のプランに応じて利用できます。詳細な利用制限は Rork の公式サイトの料金ページをご確認ください。より高度な機能や無制限の利用には Rork Max プランが必要です。

**Q. 日本語で指示を出せますか？**

はい。Rork Companion は日本語での指示に対応しています。ただし、技術的な用語（コンポーネント名・ライブラリ名・メソッド名など）は英語で記述した方が、より正確に認識される場合があります。

**Q. Rork Companion はどのフレームワーク・言語に対応していますか？**

Rork は React Native ベースのアプリ開発に特化しており、Companion も React Native / JavaScript / TypeScript を主な対象としています。React Navigation・Expo・Reanimated等の主要ライブラリへの対応も充実しています。

**Q. Companion が間違ったコードを生成した場合はどうすればいいですか？**

変更を適用する前に「なぜこのコードが正しいか説明して」と聞くことで、Companion の意図を確認できます。「この部分はこうすべきではないですか？」と具体的に指摘すると修正してくれます。「元に戻して」と伝えることで変更前の状態に戻すことも可能です。

## まとめ

Rork Companion は、コーディングの知識が浅い方でも自然言語でアプリ開発を進められる強力なAIペアプログラマーです。

- **会話形式**でアプリの改善・デバッグ・機能追加ができる
- **具体的な指示**を出すほど、期待通りの結果が得やすい
- **段階的なアプローチ**で複雑な機能も確実に実装できる
- **コードの説明機能**でAI生成コードへの理解も深まる

Rork をさらに深く活用したい方は、[Claude Opus 4.6 が Rork Max を支える仕組み](/articles/rork-ai/claude-opus-powers-rork-max)もぜひご覧ください。AI エンジンの詳細を知ることで、Companion をより効果的に使いこなせます。
