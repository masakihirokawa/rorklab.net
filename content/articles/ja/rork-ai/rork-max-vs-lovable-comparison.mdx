## はじめに

本記事では、機能、料金、対応プラットフォーム、デプロイ方法、ユースケール別の選び方をまとめました。

## ネイティブ vs Web：プラットフォーム対応

### Rork Max：ネイティブアプリ開発に特化

```swift
// iOS での実装例（Rork Max）
import SwiftUI

struct CounterView: View {
    @State private var count = 0

    var body: some View {
        VStack {
            Text("カウント: \(count)")
                .font(.title)

            Button(action: { count += 1 }) {
                Image(systemName: "plus.circle.fill")
                    .font(.title2)
            }
        }
    }
}
```

**利点:**
- App Store / Google Play での配布
- オフラインで完全動作
- Push 通知（ネイティブ）
- Apple Watch / iPad 対応
- パフォーマンス最適化

**欠点:**
- Web ブラウザアクセス不可
- デプロイに審査期間が必要（1〜3日）

### Lovable：Web-First アプローチ

```jsx
// Web での実装例（Lovable）
export default function Counter() {
    const [count, setCount] = useState(0);

    return (
        <div className="flex flex-col items-center gap-4">
            <p className="text-2xl font-bold">
                カウント: {count}
            </p>
            <button
                onClick={() => setCount(count + 1)}
                className="px-4 py-2 bg-blue-500 text-white rounded"
            >
                +1
            </button>
        </div>
    );
}
```

**利点:**
- URL で即座にアクセス可能
- クロスプラットフォーム（デスクトップ、モバイル、タブレット）
- デプロイが数秒で完了
- PWA で疑似ネイティブ化も可能

**欠点:**
- オフラインサポートが限定的（Service Worker のみ）
- Push 通知はウェブプッシュのみ
- パフォーマンスは JavaScript の制約を受ける

## デプロイメント方法

### Rork Max のデプロイメント

1. **iOS / iPadOS**
   - Rork Max の「Export」機能で `.xcodeproj` を生成
   - Xcode でビルド＆テスト
   - App Store Connect にアップロード
   - Apple の審査（1〜3 日）

2. **Android**
   - Google Play Console への直接アップロード
   - Rork Max が APK / AAB 自動生成
   - Google の審査（数時間〜1 日）

3. **Web / PWA**
   - Rork Max が React コードも生成可能
   - Vercel / Netlify へのワンクリックデプロイ

### Lovable のデプロイメント

1. **Web**
   - Vercel / Netlify への自動デプロイ
   - カスタムドメイン設定（別途 DNS 設定）
   - HTTPS 自動化（Let's Encrypt）

2. **PWA（疑似アプリ化）**
   - PWA マニフェストの自動生成
   - iOS ホーム画面への追加対応
   - オフラインページのサービスワーカー対応

## 対応プラットフォーム別の適性

### iOS / Android ネイティブが必要な場合

**Rork Max を選択すべき理由:**
- SwiftUI / Jetpack Compose の完全サポート
- App Store / Google Play での配布が前提
- パフォーマンスが最優先
- Push 通知などネイティブ機能が必須

**使用例:**
- SNS アプリ（チャット、画像処理）

- IoT コントローラー（Bluetooth 連携）
- ヘルスケアアプリ（HealthKit 連携）

### Web / PWA が要件の場合

**Lovable を選択すべき理由:**
- ブラウザアクセスで即座に利用可能
- デプロイが数秒で完了
- URL でシェア可能
- ユーザー登録不要でアクセス

**使用例:**
- 業務管理ツール（SaaS）
- プロジェクト管理アプリ（Notion ライク）
- オンラインエディタ（Figma ライク）
- データ可視化ダッシュボード

### クロスプラットフォーム展開が目標の場合

**ハイブリッド戦略:**
1. **MVP（最小限製品）**: Lovable で Web 版をリリース → ユーザー検証
2. **スケーリング**: Rork Max で iOS / Android ネイティブ版を並行開発

## よくある質問（FAQ）

### Q1: Rork Max で Web 版は作れますか？

**A:** はい。Rork Max は、iOS/Android と同時に Web（React）コードも生成できます。ただし最適化は iOS/Android に傾いているため、Web-First が必要な場合は Lovable の方が効率的です。

### Q2: Lovable で iOS アプリは作れますか？

**A:** 直接的には不可。ただし PWA としてホーム画面に追加できるため、疑似ネイティブアプリとして動作させることは可能です。完全なネイティブ機能（Push、HealthKit）が必要な場合は対応外です。

### Q3: どちらが学習曲線が緩い？

**A:** Lovable の方が始めやすいです。なぜなら JavaScript/React の知識があれば、生成コードのカスタマイズが容易だからです。Rork Max は Swift / Kotlin の知識があると強いですが、ゼロベースなら Lovable から始めるのをお勧めします。

### Q4: 料金は？どちらがお得？

**A:** 用途次第です。Lovable は $29/月で無制限デプロイが可能。Rork Max は $99/月ですが、App Store/Google Play デプロイが無制限です。1 年でのモネタイズを想定なら Rork Max、MVP リリースなら Lovable が経済的です。

### Q5: AI 生成コードの品質は？手動編集は必要？

**A:** 両ツールとも業界水準の品質です。ただし本番運用では必ず手動レビュー・テストが必要です。特に認証、決済、個人情報はコード監査の対象になります。

## 最終的な選択基準

| シナリオ | 推奨ツール | 理由 |
|---|---|---|
| **スタートアップの MVP** | Lovable → Rork Max | 素早く Web リリース後、マネタイズ時に iOS/Android 対応 |
| **エンタープライズ SaaS** | Lovable | Web ベースの業務ツールは Lovable が適 |
| **ゲーム・娯楽** | Rork Max | パフォーマンス・ネイティブ機能が必須 |
| **個人開発・副業** | Lovable | 初期投資を最小化、URL でシェア可能 |
| **AI スタートアップ** | Rork Max + Lovable | ネイティブアプリとしてリッチなエコシステム構築 |

## まとめ

- **Rork Max**: ネイティブアプリ開発に最適化。App Store/Google Play デプロイが前提。
- **Lovable**: Web アプリ開発に最適化。即座のリリースと URL シェアが強み。
