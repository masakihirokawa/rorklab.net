# Rork × Claude API で作る AI アシスタントアプリ

AI アシスタント機能をアプリに組み込む需要は急速に高まっています。中でも Anthropic が提供する Claude API は、自然で安全性の高い応答品質が特長で、モバイルアプリとの相性も抜群です。

本記事では、Rork を使って Claude API を統合した AI アシスタントアプリを構築する方法をステップバイステップで解説します。API キーの管理からストリーミング応答の実装、会話コンテキストの保持まで、プロダクションレベルの実装パターンを網羅します。

## 前提知識と環境準備

このチュートリアルを始める前に、以下の準備が必要です。

Rork の基本操作に慣れていることが前提となります。まだ Rork を使ったことがない方は、[Rork 入門ガイド](/articles/rork-basics/getting-started)から始めることをおすすめします。

必要なもの:

- Rork アカウント（無料プランでも開発可能）
- Anthropic API キー（[console.anthropic.com](https://console.anthropic.com) で取得）
- Node.js 18 以上がインストールされた環境
- 基本的な TypeScript / React Native の知識

Anthropic API は従量課金制で、Claude Sonnet 4.6 モデルの場合、入力 100 万トークンあたり $3、出力 100 万トークンあたり $15 です（2026 年 3 月時点）。個人開発であれば月 $5〜10 程度のコストで十分な利用が可能です。

## Anthropic SDK のセットアップ

Rork プロジェクトに Anthropic の公式 TypeScript SDK をインストールします。

```bash
# Rork プロジェクトのルートで実行
npm install @anthropic-ai/sdk
```

API キーは環境変数で管理するのが鉄則です。`.env` ファイルに以下を追加してください。

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxxxxxxxxxx
```

次に、API クライアントを初期化するユーティリティを作成します。

```typescript
// src/lib/claude.ts
import Anthropic from "@anthropic-ai/sdk";

// APIクライアントのシングルトンインスタンス
const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

export default anthropic;
```

**重要**: API キーはクライアントサイド（React Native アプリ内）に直接埋め込まないでください。必ずバックエンド API を経由してリクエストを送信する構成にします。Rork ではサーバーサイドの API ルートを活用できるため、セキュリティを確保しやすい設計になっています。

## バックエンド API ルートの構築

Claude API へのリクエストはサーバーサイドで処理します。以下は基本的な API ルートの実装例です。

```typescript
// src/app/api/chat/route.ts
import anthropic from "@/lib/claude";
import { NextRequest, NextResponse } from "next/server";

// メッセージの型定義
interface ChatMessage {
  role: "user" | "assistant";
  content: string;
}

export async function POST(request: NextRequest) {
  try {
    const { messages }: { messages: ChatMessage[] } = await request.json();

    // Claude API にリクエストを送信
    const response = await anthropic.messages.create({
      model: "claude-sonnet-4-6",  // 最新のSonnetモデル
      max_tokens: 1024,
      system: "あなたは親切で知識豊富なAIアシスタントです。日本語で丁寧に回答してください。",
      messages: messages,
    });

    // テキストレスポンスを抽出
    const assistantMessage =
      response.content[0].type === "text"
        ? response.content[0].text
        : "";

    return NextResponse.json({
      message: assistantMessage,
      usage: response.usage,  // トークン使用量も返す
    });
  } catch (error) {
    console.error("Claude API Error:", error);
    return NextResponse.json(
      { error: "AIアシスタントからの応答に失敗しました" },
      { status: 500 }
    );
  }
}
```

このコードでは、クライアントから送信されたメッセージ履歴をそのまま Claude API に渡しています。`system` パラメータでアシスタントの振る舞いを定義しているのがポイントです。

## ストリーミング応答の実装

ユーザー体験を大幅に向上させるのがストリーミング応答です。Claude の応答をリアルタイムで1文字ずつ表示することで、待ち時間を感じさせない UX を実現できます。

```typescript
// src/app/api/chat/stream/route.ts
import anthropic from "@/lib/claude";
import { NextRequest } from "next/server";

export async function POST(request: NextRequest) {
  const { messages } = await request.json();

  // ストリーミングレスポンスを作成
  const stream = await anthropic.messages.stream({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    system: "あなたは親切で知識豊富なAIアシスタントです。日本語で丁寧に回答してください。",
    messages: messages,
  });

  // ReadableStreamとしてクライアントに返す
  const encoder = new TextEncoder();
  const readableStream = new ReadableStream({
    async start(controller) {
      for await (const event of stream) {
        if (
          event.type === "content_block_delta" &&
          event.delta.type === "text_delta"
        ) {
          // テキストの差分をSSE形式で送信
          const data = JSON.stringify({ text: event.delta.text });
          controller.enqueue(
            encoder.encode(`data: ${data}\n\n`)
          );
        }
      }
      controller.enqueue(encoder.encode("data: [DONE]\n\n"));
      controller.close();
    },
  });

  return new Response(readableStream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      Connection: "keep-alive",
    },
  });
}
```

## フロントエンドのチャット UI 実装

React Native 側のチャット画面を実装します。ストリーミング応答に対応したリアルタイム更新を含む実装です。

```typescript
// src/components/ChatScreen.tsx
import React, { useState, useRef, useCallback } from "react";
import {
  View,
  Text,
  TextInput,
  FlatList,
  TouchableOpacity,
  StyleSheet,
  ActivityIndicator,
  KeyboardAvoidingView,
  Platform,
} from "react-native";

interface Message {
  id: string;
  role: "user" | "assistant";
  content: string;
}

export default function ChatScreen() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState("");
  const [isLoading, setIsLoading] = useState(false);
  const flatListRef = useRef<FlatList>(null);

  const sendMessage = useCallback(async () => {
    if (!input.trim() || isLoading) return;

    const userMessage: Message = {
      id: Date.now().toString(),
      role: "user",
      content: input.trim(),
    };

    // ユーザーメッセージを追加
    const updatedMessages = [...messages, userMessage];
    setMessages(updatedMessages);
    setInput("");
    setIsLoading(true);

    // アシスタントのプレースホルダーを追加
    const assistantId = (Date.now() + 1).toString();
    setMessages((prev) => [
      ...prev,
      { id: assistantId, role: "assistant", content: "" },
    ]);

    try {
      // ストリーミングAPIを呼び出す
      const response = await fetch("/api/chat/stream", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          messages: updatedMessages.map(({ role, content }) => ({
            role,
            content,
          })),
        }),
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();

      if (!reader) throw new Error("ストリームの読み取りに失敗");

      // ストリーミングデータを逐次読み取る
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const chunk = decoder.decode(value);
        const lines = chunk.split("\n");

        for (const line of lines) {
          if (line.startsWith("data: ") && line !== "data: [DONE]") {
            const { text } = JSON.parse(line.slice(6));
            // アシスタントメッセージを更新
            setMessages((prev) =>
              prev.map((msg) =>
                msg.id === assistantId
                  ? { ...msg, content: msg.content + text }
                  : msg
              )
            );
          }
        }
      }
    } catch (error) {
      console.error("通信エラー:", error);
      setMessages((prev) =>
        prev.map((msg) =>
          msg.id === assistantId
            ? { ...msg, content: "申し訳ございません。エラーが発生しました。" }
            : msg
        )
      );
    } finally {
      setIsLoading(false);
    }
  }, [input, messages, isLoading]);

  return (
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === "ios" ? "padding" : "height"}
    >
      <FlatList
        ref={flatListRef}
        data={messages}
        keyExtractor={(item) => item.id}
        onContentSizeChange={() =>
          flatListRef.current?.scrollToEnd({ animated: true })
        }
        renderItem={({ item }) => (
          <View
            style={[
              styles.bubble,
              item.role === "user" ? styles.userBubble : styles.assistantBubble,
            ]}
          >
            <Text style={styles.messageText}>{item.content}</Text>
          </View>
        )}
      />
      <View style={styles.inputRow}>
        <TextInput
          style={styles.input}
          value={input}
          onChangeText={setInput}
          placeholder="メッセージを入力..."
          multiline
        />
        <TouchableOpacity
          style={styles.sendButton}
          onPress={sendMessage}
          disabled={isLoading}
        >
          {isLoading ? (
            <ActivityIndicator color="#fff" />
          ) : (
            <Text style={styles.sendText}>送信</Text>
          )}
        </TouchableOpacity>
      </View>
    </KeyboardAvoidingView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  bubble: { padding: 12, borderRadius: 16, marginVertical: 4, marginHorizontal: 12, maxWidth: "80%" },
  userBubble: { backgroundColor: "#007AFF", alignSelf: "flex-end" },
  assistantBubble: { backgroundColor: "#fff", alignSelf: "flex-start" },
  messageText: { fontSize: 16, lineHeight: 22 },
  inputRow: { flexDirection: "row", padding: 8, backgroundColor: "#fff" },
  input: { flex: 1, borderWidth: 1, borderColor: "#ddd", borderRadius: 20, paddingHorizontal: 16, paddingVertical: 8, fontSize: 16 },
  sendButton: { marginLeft: 8, backgroundColor: "#007AFF", borderRadius: 20, paddingHorizontal: 20, justifyContent: "center" },
  sendText: { color: "#fff", fontWeight: "bold" },
});
```

## 会話コンテキストの管理とトークン最適化

Claude API は会話の全履歴をリクエストごとに送信する仕組みです。会話が長くなるとトークン消費量が増大するため、適切な管理が重要になります。

```typescript
// src/lib/conversation.ts

interface ConversationMessage {
  role: "user" | "assistant";
  content: string;
  timestamp: number;
}

// 直近N件のメッセージだけを送信するスライディングウィンドウ
export function getRecentMessages(
  messages: ConversationMessage[],
  maxMessages: number = 20
): ConversationMessage[] {
  if (messages.length <= maxMessages) return messages;

  // 最初のメッセージ（コンテキスト設定）と直近のメッセージを保持
  const first = messages[0];
  const recent = messages.slice(-maxMessages + 1);
  return [first, ...recent];
}

// トークン数の概算（日本語は1文字≒1.5トークン）
export function estimateTokens(text: string): number {
  const japaneseChars = (text.match(/[\u3000-\u9fff]/g) || []).length;
  const otherChars = text.length - japaneseChars;
  return Math.ceil(japaneseChars * 1.5 + otherChars * 0.25);
}

// 会話全体のトークン数を概算
export function estimateConversationTokens(
  messages: ConversationMessage[]
): number {
  return messages.reduce(
    (total, msg) => total + estimateTokens(msg.content),
    0
  );
}
```

長い会話でもコストを抑えるコツは、スライディングウィンドウ方式で直近のメッセージのみを送信することです。`system` プロンプトに会話の要約を含めることで、古いメッセージを切り捨ててもコンテキストを維持できます。

## エラーハンドリングとレートリミット対策

本番運用では、API のエラーやレートリミットへの対応が不可欠です。

```typescript
// src/lib/claude-with-retry.ts
import anthropic from "./claude";

const MAX_RETRIES = 3;
const BASE_DELAY = 1000; // 1秒

export async function sendWithRetry(
  messages: Array<{ role: "user" | "assistant"; content: string }>,
  systemPrompt: string,
  retries = MAX_RETRIES
): Promise<string> {
  for (let attempt = 0; attempt <= retries; attempt++) {
    try {
      const response = await anthropic.messages.create({
        model: "claude-sonnet-4-6",
        max_tokens: 1024,
        system: systemPrompt,
        messages,
      });

      return response.content[0].type === "text"
        ? response.content[0].text
        : "";
    } catch (error: any) {
      // レートリミット（429）はリトライ
      if (error.status === 429 && attempt < retries) {
        const delay = BASE_DELAY * Math.pow(2, attempt);
        console.warn(
          `レートリミット到達。${delay}ms 後にリトライ (${attempt + 1}/${retries})`
        );
        await new Promise((resolve) => setTimeout(resolve, delay));
        continue;
      }

      // 認証エラー（401）はリトライ不要
      if (error.status === 401) {
        throw new Error("APIキーが無効です。設定を確認してください。");
      }

      // サーバーエラー（500番台）はリトライ
      if (error.status >= 500 && attempt < retries) {
        const delay = BASE_DELAY * Math.pow(2, attempt);
        await new Promise((resolve) => setTimeout(resolve, delay));
        continue;
      }

      throw error;
    }
  }

  throw new Error("最大リトライ回数に達しました");
}
```

エクスポネンシャルバックオフ（指数的待機）を使うことで、レートリミットに引っかかった場合でも自動的にリトライし、ユーザーにエラーを見せずに処理を続行できます。

## よくある質問（FAQ）

### Q1: Claude API の料金はどのくらいかかりますか？

Claude Sonnet 4.6 の場合、入力 100 万トークンあたり $3、出力 100 万トークンあたり $15 です。一般的なチャットアプリであれば、1 回の会話（10 往復程度）で約 $0.01〜0.03 です。月間 1,000 回の会話でも $10〜30 程度に収まるため、個人開発でも十分に現実的なコストです。

### Q2: OpenAI の GPT-4o と Claude API はどちらを選ぶべきですか？

用途によって異なります。Claude API は長文コンテキスト（最大 200K トークン）の処理に優れ、安全性のガードレールが標準で組み込まれている点が強みです。一方、GPT-4o は画像・音声のマルチモーダル対応が先行しています。日本語の自然さを重視する場合は Claude が高い評価を得ています。Rork では [GPT-4o との連携ガイド](/articles/rork-ai/gpt-4o-rork-openai-app-development)もあるので、比較検討の参考にしてください。

### Q3: ストリーミング応答を使わない場合のデメリットは？

ストリーミングを使わない場合、Claude がすべてのテキストを生成し終わるまでユーザーは空白の画面を見続けることになります。長い応答では 5〜15 秒の待ち時間が発生し、アプリを閉じてしまう原因になります。ストリーミングを使えばリアルタイムで文字が流れるため、体感の待ち時間はほぼゼロになります。

### Q4: API キーが漏洩した場合はどうなりますか？

API キーが第三者に利用されると、意図しない課金が発生する可能性があります。漏洩に気づいたら、Anthropic Console で即座にキーを無効化し、新しいキーを発行してください。予防策として、月ごとの使用上限を設定することを強くおすすめします。
