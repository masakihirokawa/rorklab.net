動画を「見せる」ことで AI が理解してくれる——これが Gemini 2.5 Pro の Video Understanding（動画理解）機能です。「スイングフォームを撮影したら改善点を指摘してくれる」「料理している様子を送ったらレシピを書き起こしてくれる」——そんなアプリを実現するための API が、今や個人開発者でも手の届く場所にあります。

Rork でこの機能を実装しようとしたとき、最初に戸惑うのが「Files API を使うべきか、それともバイト列を直接送るべきか」という判断です。公式ドキュメントを読んでも、どちらも動くように書かれているため、どちらを選べばいいか迷うことが多いのです。

このガイドでは、その判断基準から始めて、Rork アプリで Video Understanding を実際に動かすまでの全工程を、動作するコードとともに解説していきます。

## Gemini 2.5 Pro の Video Understanding とは何ができるのか

Gemini 2.5 Pro は、テキストだけでなく動画を入力として受け取り、内容を理解してテキストで回答する能力を持ちます。具体的には以下のことができます。

動画の中で何が起きているかを説明する、特定のシーンの時刻を特定する、複数のフレームにわたる動きやパターンを解析する、音声が含まれていれば内容もあわせて解釈する——といったことが可能です。

動画を「フレームの連続」として扱うのではなく、「動き・コンテキスト・意図を持つひとつの情報」として理解できる点が、従来の画像認識と根本的に異なります。

サポートされる動画フォーマットは MP4、MOV、AVI、MKV など主要なものをカバーしています。また、1 リクエストで処理できる動画の長さは Files API 経由で最大 1 時間、inline データでも短い動画なら対応できます。

## Files API と inline base64：どちらを使うべきか

Gemini の動画入力には 2 つの方法があります。

**Files API（ファイルアップロード方式）**は、動画ファイルを先に Gemini のサーバーにアップロードし、取得した `file_uri` を使って API を呼び出す方式です。アップロードしたファイルは 48 時間保持されます。

**inline base64 方式**は、動画をバイナリデータとして base64 エンコードし、API リクエストのボディに直接含める方式です。

どちらを使うべきかの基準はシンプルです。

動画ファイルサイズが 20MB 以上の場合、または同じ動画を繰り返し解析する場合は **Files API を使います**。20MB 以下で 1 回限りの解析なら **inline base64 が手軽です**。

Rork アプリでは、ユーザーがカメラロールから動画を選択する場合がほとんどです。実際のスマートフォンで撮影した動画は数十 MB になることも多く、Files API を主軸に実装することをおすすめしています。

## バックエンドの準備：Cloudflare Workers で動画処理 API を作る

Rork アプリから直接 Gemini API を呼ぶ方法もありますが、API キーをアプリに埋め込むのはセキュリティリスクがあります。Cloudflare Workers を使ったバックエンドを経由することで、キーを安全に管理しつつ、処理の柔軟性も高まります。

```typescript
// workers/src/video-analysis.ts
// Cloudflare Workers でGemini Video Understanding APIを呼び出すエンドポイント

import { GoogleGenAI } from '@google/genai';

interface Env {
  GEMINI_API_KEY: string;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    if (request.method \!== 'POST') {
      return new Response('Method Not Allowed', { status: 405 });
    }

    const url = new URL(request.url);
    const path = url.pathname;

    // --- Step 1: 動画ファイルのアップロード ---
    if (path === '/api/upload-video') {
      return handleVideoUpload(request, env);
    }

    // --- Step 2: 動画の解析 ---
    if (path === '/api/analyze-video') {
      return handleVideoAnalysis(request, env);
    }

    return new Response('Not Found', { status: 404 });
  }
};

async function handleVideoUpload(request: Request, env: Env): Promise<Response> {
  try {
    const formData = await request.formData();
    const videoFile = formData.get('video') as File;

    if (\!videoFile) {
      return Response.json({ error: 'No video file provided' }, { status: 400 });
    }

    // Gemini Files API にアップロード
    const uploadUrl = 'https://generativelanguage.googleapis.com/upload/v1beta/files';
    const arrayBuffer = await videoFile.arrayBuffer();

    // 初期化リクエスト（resumable upload）
    const initResponse = await fetch(`${uploadUrl}?key=${env.GEMINI_API_KEY}`, {
      method: 'POST',
      headers: {
        'X-Goog-Upload-Protocol': 'resumable',
        'X-Goog-Upload-Command': 'start',
        'X-Goog-Upload-Header-Content-Length': arrayBuffer.byteLength.toString(),
        'X-Goog-Upload-Header-Content-Type': videoFile.type,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        file: { display_name: videoFile.name }
      })
    });

    if (\!initResponse.ok) {
      throw new Error(`Upload init failed: ${initResponse.status}`);
    }

    const uploadUri = initResponse.headers.get('X-Goog-Upload-URL');
    if (\!uploadUri) throw new Error('No upload URI received');

    // ファイル本体をアップロード
    const uploadResponse = await fetch(uploadUri, {
      method: 'POST',
      headers: {
        'Content-Length': arrayBuffer.byteLength.toString(),
        'X-Goog-Upload-Offset': '0',
        'X-Goog-Upload-Command': 'upload, finalize',
      },
      body: arrayBuffer
    });

    if (\!uploadResponse.ok) {
      const errorText = await uploadResponse.text();
      throw new Error(`Upload failed: ${errorText}`);
    }

    const fileData = await uploadResponse.json() as { file: { uri: string; name: string; state: string } };

    return Response.json({
      fileUri: fileData.file.uri,
      fileName: fileData.file.name,
      state: fileData.file.state
    });

  } catch (error) {
    console.error('Upload error:', error);
    return Response.json(
      { error: error instanceof Error ? error.message : 'Upload failed' },
      { status: 500 }
    );
  }
}

async function handleVideoAnalysis(request: Request, env: Env): Promise<Response> {
  try {
    const { fileUri, prompt, mimeType } = await request.json() as {
      fileUri: string;
      prompt: string;
      mimeType: string;
    };

    if (\!fileUri || \!prompt) {
      return Response.json({ error: 'fileUri and prompt are required' }, { status: 400 });
    }

    const genAI = new GoogleGenAI({ apiKey: env.GEMINI_API_KEY });

    // ファイルの状態を確認（処理完了まで待機）
    let fileState = 'PROCESSING';
    let attempts = 0;
    const maxAttempts = 10;

    while (fileState === 'PROCESSING' && attempts < maxAttempts) {
      await new Promise(resolve => setTimeout(resolve, 2000));
      const fileNameMatch = fileUri.match(/files\/([^/]+)/);
      if (\!fileNameMatch) break;

      const statusResponse = await fetch(
        `https://generativelanguage.googleapis.com/v1beta/files/${fileNameMatch[1]}?key=${env.GEMINI_API_KEY}`
      );
      const statusData = await statusResponse.json() as { state: string };
      fileState = statusData.state;
      attempts++;
    }

    if (fileState \!== 'ACTIVE') {
      return Response.json({ error: 'File is not ready for analysis' }, { status: 422 });
    }

    // Gemini 2.5 Pro で解析
    const result = await genAI.models.generateContent({
      model: 'gemini-2.5-pro-preview-0325',
      contents: [
        {
          parts: [
            {
              fileData: {
                fileUri: fileUri,
                mimeType: mimeType || 'video/mp4'
              }
            },
            { text: prompt }
          ]
        }
      ],
      config: {
        temperature: 0.2,  // 解析タスクは低めの温度設定で正確性を優先
        maxOutputTokens: 2048,
      }
    });

    const analysisText = result.text;
    return Response.json({ analysis: analysisText });

  } catch (error) {
    console.error('Analysis error:', error);
    return Response.json(
      { error: error instanceof Error ? error.message : 'Analysis failed' },
      { status: 500 }
    );
  }
}
```

このコードで注目してほしいのは、ファイルのアップロード後に `state` が `ACTIVE` になるまで待機している点です。アップロードが完了しても、Gemini 側での処理が終わるまで解析リクエストを送ると `PROCESSING` エラーが返ってきます。この待機ループを省略すると、断続的に失敗するバグになりやすいです。

## Rork アプリ側の実装：動画選択から結果表示まで

次に、Rork アプリ側の実装です。expo-image-picker を使って動画を選択し、バックエンドに送信して解析結果を表示するフローを実装します。

```typescript
// app/video-analyzer.tsx
// Rork (React Native / Expo) での動画解析画面

import { useState } from 'react';
import {
  View,
  Text,
  TouchableOpacity,
  ScrollView,
  ActivityIndicator,
  StyleSheet,
  Alert
} from 'react-native';
import * as ImagePicker from 'expo-image-picker';
import * as FileSystem from 'expo-file-system';
import { Video, ResizeMode } from 'expo-av';

const API_BASE_URL = 'https://your-worker.your-account.workers.dev';

// 解析のプリセットプロンプト（ユースケース別）
const ANALYSIS_PRESETS = {
  sports_form: `このスポーツ動画を解析してください。
    以下の観点で詳細にフィードバックを提供してください：
    1. フォームの良い点（具体的に）
    2. 改善が必要な点（優先度順に）
    3. 怪我のリスクがある動作があれば指摘してください
    4. 次の練習で意識すべき1つのポイント
    日本語で回答してください。`,

  recipe_recognition: `この料理動画を見て、以下を答えてください：
    1. 作られている料理名（推定）
    2. 使用されている主な食材（見えているもの）
    3. 調理の手順（動画から読み取れる範囲で）
    4. 調理時間の目安
    日本語で、料理レシピ風にまとめてください。`,

  lecture_summary: `この動画の内容を要約してください：
    1. メインテーマ（1行で）
    2. 主要なポイント（箇条書き、3〜5点）
    3. 重要なキーワード
    4. 視聴者に向けた学習アドバイス
    日本語で、わかりやすく構造化してまとめてください。`
} as const;

type PresetKey = keyof typeof ANALYSIS_PRESETS;

interface AnalysisResult {
  analysis: string;
  processingTime: number;
}

export default function VideoAnalyzerScreen() {
  const [videoUri, setVideoUri] = useState<string | null>(null);
  const [isUploading, setIsUploading] = useState(false);
  const [isAnalyzing, setIsAnalyzing] = useState(false);
  const [result, setResult] = useState<AnalysisResult | null>(null);
  const [selectedPreset, setSelectedPreset] = useState<PresetKey>('sports_form');
  const [fileUri, setFileUri] = useState<string | null>(null);

  // 動画を選択してアップロード
  const pickAndUploadVideo = async () => {
    try {
      const permission = await ImagePicker.requestMediaLibraryPermissionsAsync();
      if (\!permission.granted) {
        Alert.alert('許可が必要です', 'ビデオライブラリへのアクセスを許可してください');
        return;
      }

      const result = await ImagePicker.launchImageLibraryAsync({
        mediaTypes: ImagePicker.MediaTypeOptions.Videos,
        allowsEditing: false,
        quality: 1,
      });

      if (result.canceled || \!result.assets[0]) return;

      const asset = result.assets[0];
      setVideoUri(asset.uri);
      setResult(null);

      // ファイルサイズチェック（20MB以上はFiles API必須）
      const fileInfo = await FileSystem.getInfoAsync(asset.uri);
      const fileSizeMB = fileInfo.exists ? (fileInfo.size ?? 0) / (1024 * 1024) : 0;
      console.log(`📹 動画サイズ: ${fileSizeMB.toFixed(1)}MB`);

      setIsUploading(true);

      // バックエンド経由でGemini Files APIにアップロード
      const formData = new FormData();
      formData.append('video', {
        uri: asset.uri,
        type: asset.mimeType || 'video/mp4',
        name: `video_${Date.now()}.mp4`,
      } as unknown as Blob);

      const uploadResponse = await fetch(`${API_BASE_URL}/api/upload-video`, {
        method: 'POST',
        body: formData,
        headers: {
          'Content-Type': 'multipart/form-data',
        },
      });

      if (\!uploadResponse.ok) {
        throw new Error(`アップロード失敗: ${uploadResponse.status}`);
      }

      const uploadData = await uploadResponse.json() as { fileUri: string };
      setFileUri(uploadData.fileUri);

    } catch (error) {
      Alert.alert('エラー', error instanceof Error ? error.message : 'アップロードに失敗しました');
    } finally {
      setIsUploading(false);
    }
  };

  // 解析の実行
  const analyzeVideo = async () => {
    if (\!fileUri) {
      Alert.alert('動画を選択してください');
      return;
    }

    setIsAnalyzing(true);
    const startTime = Date.now();

    try {
      const response = await fetch(`${API_BASE_URL}/api/analyze-video`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          fileUri: fileUri,
          prompt: ANALYSIS_PRESETS[selectedPreset],
          mimeType: 'video/mp4',
        }),
      });

      if (\!response.ok) {
        const errorData = await response.json() as { error: string };
        throw new Error(errorData.error || '解析に失敗しました');
      }

      const data = await response.json() as { analysis: string };
      setResult({
        analysis: data.analysis,
        processingTime: Date.now() - startTime,
      });

    } catch (error) {
      Alert.alert('解析エラー', error instanceof Error ? error.message : '解析に失敗しました');
    } finally {
      setIsAnalyzing(false);
    }
  };

  return (
    <ScrollView style={styles.container}>
      <Text style={styles.title}>AI 動画解析</Text>

      {/* 解析タイプの選択 */}
      <View style={styles.presetContainer}>
        {(Object.keys(ANALYSIS_PRESETS) as PresetKey[]).map((key) => (
          <TouchableOpacity
            key={key}
            style={[styles.presetButton, selectedPreset === key && styles.presetButtonActive]}
            onPress={() => setSelectedPreset(key)}
          >
            <Text style={[styles.presetText, selectedPreset === key && styles.presetTextActive]}>
              {key === 'sports_form' ? 'スポーツフォーム' :
               key === 'recipe_recognition' ? '料理レシピ' : '動画要約'}
            </Text>
          </TouchableOpacity>
        ))}
      </View>

      {/* 動画選択ボタン */}
      <TouchableOpacity
        style={[styles.button, (isUploading || isAnalyzing) && styles.buttonDisabled]}
        onPress={pickAndUploadVideo}
        disabled={isUploading || isAnalyzing}
      >
        {isUploading ? (
          <View style={styles.loadingRow}>
            <ActivityIndicator color="#fff" size="small" />
            <Text style={styles.buttonText}>  アップロード中...</Text>
          </View>
        ) : (
          <Text style={styles.buttonText}>
            {videoUri ? '別の動画を選択' : '動画を選択してアップロード'}
          </Text>
        )}
      </TouchableOpacity>

      {/* 動画プレビュー */}
      {videoUri && (
        <Video
          source={{ uri: videoUri }}
          style={styles.video}
          useNativeControls
          resizeMode={ResizeMode.CONTAIN}
        />
      )}

      {/* 解析ボタン */}
      {fileUri && \!isUploading && (
        <TouchableOpacity
          style={[styles.analyzeButton, isAnalyzing && styles.buttonDisabled]}
          onPress={analyzeVideo}
          disabled={isAnalyzing}
        >
          {isAnalyzing ? (
            <View style={styles.loadingRow}>
              <ActivityIndicator color="#fff" size="small" />
              <Text style={styles.buttonText}>  Gemini が解析中...</Text>
            </View>
          ) : (
            <Text style={styles.buttonText}>AI で解析する</Text>
          )}
        </TouchableOpacity>
      )}

      {/* 解析結果 */}
      {result && (
        <View style={styles.resultContainer}>
          <Text style={styles.resultLabel}>
            解析結果（{(result.processingTime / 1000).toFixed(1)}秒）
          </Text>
          <Text style={styles.resultText}>{result.analysis}</Text>
        </View>
      )}
    </ScrollView>
  );
}

const styles = StyleSheet.create({

  presetButton: {
```

このコードの中で、特に重要なのが `formData.append` の部分です。React Native では `Blob` 型の代わりに URI・type・name を持つオブジェクトを使います。ここを通常の `FormData` の使い方（ブラウザ向け）のまま書いてしまうと、動画が正しく送信されません。

## コスト最適化：フレームサンプリングとモデル選択

Gemini 2.5 Pro は高精度ですが、長い動画や頻繁な解析にはコストが積み上がります。実用的なアプリを作るには、コスト最適化の戦略が欠かせません。

### 解析内容によってモデルを使い分ける

すべての解析に 2.5 Pro を使う必要はありません。簡単な内容確認や要約であれば Gemini 2.5 Flash で十分で、コストを大幅に下げられます。

```typescript
// workers/src/model-selector.ts
// 解析の複雑さに応じてモデルを自動選択する

type AnalysisType = 'simple_summary' | 'detailed_analysis' | 'expert_coaching';

interface ModelConfig {
  model: string;
  maxOutputTokens: number;
  temperature: number;
}

function selectModel(analysisType: AnalysisType): ModelConfig {
  const configs: Record<AnalysisType, ModelConfig> = {
    // 動画の要約・ざっくりした内容確認
    simple_summary: {
      model: 'gemini-2.5-flash-preview-04-17',
      maxOutputTokens: 512,
      temperature: 0.3,
    },
    // 詳細分析・複数の観点からの評価
    detailed_analysis: {
      model: 'gemini-2.5-pro-preview-0325',
      maxOutputTokens: 1024,
      temperature: 0.2,
    },
    // 専門的なコーチング・精密な動作解析
    expert_coaching: {
      model: 'gemini-2.5-pro-preview-0325',
      maxOutputTokens: 2048,
      temperature: 0.1,
    }
  };
  return configs[analysisType];
}

// 動画の長さに応じた処理方針を決定する
function getVideoStrategy(durationSeconds: number): {
  shouldSample: boolean;
  samplingIntervalSec: number;
  model: AnalysisType;
} {
  if (durationSeconds <= 30) {
    // 30秒以内：そのまま送れる。Pro で精密解析
    return { shouldSample: false, samplingIntervalSec: 0, model: 'detailed_analysis' };
  } else if (durationSeconds <= 180) {
    // 30秒〜3分：Flash で十分な場合が多い
    return { shouldSample: false, samplingIntervalSec: 0, model: 'simple_summary' };
  } else {
    // 3分超：重要フレームのみをサンプリングして送る
    return { shouldSample: true, samplingIntervalSec: 10, model: 'simple_summary' };
  }
}

export { selectModel, getVideoStrategy };
```

### 短い動画は inline base64 で高速化

動画が 5MB 以下の短い動画（30秒程度）であれば、Files API のアップロード→待機→解析という手順を省いて、直接 base64 で送るほうが処理時間を大幅に短縮できます。

```typescript
// workers/src/inline-video-analysis.ts
// 小さな動画はbase64で直接送信する高速版

import { GoogleGenAI } from '@google/genai';

interface Env {
  GEMINI_API_KEY: string;
}

export async function analyzeVideoInline(
  videoBuffer: ArrayBuffer,
  mimeType: string,
  prompt: string,
  env: Env
): Promise<string> {
  // サイズチェック：20MB以上はFiles APIを使うべき
  const sizeInMB = videoBuffer.byteLength / (1024 * 1024);
  if (sizeInMB > 20) {
    throw new Error('動画が大きすぎます。Files API を使用してください（20MB以下）');
  }

  const genAI = new GoogleGenAI({ apiKey: env.GEMINI_API_KEY });

  // ArrayBuffer を base64 に変換
  const uint8Array = new Uint8Array(videoBuffer);
  const base64Video = btoa(
    Array.from(uint8Array).map(byte => String.fromCharCode(byte)).join('')
  );

  const result = await genAI.models.generateContent({
    model: 'gemini-2.5-flash-preview-04-17',  // 小さな動画にはFlashで十分
    contents: [
      {
        parts: [
          {
            inlineData: {
              mimeType: mimeType,
              data: base64Video
            }
          },
          { text: prompt }
        ]
      }
    ],
    config: {
      temperature: 0.2,
      maxOutputTokens: 1024,
    }
  });

  return result.text ?? '';
}
```

## よくある実装ミスと解決方法

Video Understanding を実装していると、いくつかのパターンでハマることがあります。実際に経験した主なものを挙げておきます。

**① ファイルが PROCESSING 状態のまま解析しようとする**

アップロード直後にすぐ解析リクエストを送ると、`File is not in ACTIVE state` というエラーが返ります。前述のコード例のように、`state` が `ACTIVE` になるまでポーリングで待機する実装が必要です。通常 1〜3 秒で完了しますが、ネットワーク状態によっては 10 秒以上かかることもあります。

**② Files API のファイル URI が 48 時間で失効する**

アップロードしたファイルは 48 時間後に自動削除されます。「先週解析したのと同じ動画を再解析したい」という場合、保存しておいた URI は使えません。同じ動画を繰り返し解析するユースケースでは、必要なタイミングで再アップロードするか、解析結果をキャッシュする設計にしましょう。

**③ 動画の向きが正しく解釈されない**

スマートフォンで縦向きに撮影した動画は、EXIF メタデータに向き情報が含まれています。Gemini は多くの場合これを正しく解釈しますが、まれに横向きに解釈することがあります。プロンプトに「この動画は縦向き（ポートレート）で撮影されています」と明示することで回避できます。

**④ 長い動画でトークンが不足する**

Gemini 2.5 Pro の動画入力は、1 フレームあたり 258 トークン（デフォルト）を消費します。24fps の 5 分間動画をそのまま送ると膨大なトークンになります。長い動画は必ずサンプリングするか、解析が必要な区間のみをトリミングして送るよう設計しましょう。

## 実用ユースケース：スポーツフォーム解析アプリの設計パターン

ここまでの実装をベースに、スポーツフォーム解析アプリとして仕上げるための設計パターンを紹介します。

ユーザー体験の流れは次のようになります。ユーザーが練習中の動画（10〜30秒）を選択します。アプリがバックエンドにアップロードし、Gemini が解析を実行します。フォームの良い点・改善点・次の練習へのアドバイスが返ってきます。解析結果を履歴として保存し、成長の記録として活用できます。

この流れで月額課金アプリとして成立させるには、無料プランでは「月3回まで解析可能」、有料プランでは「無制限解析 + 解析履歴保存 + コーチからのフィードバック機能」という構成が現実的です。

Gemini 2.5 Pro の API コストは、30秒の動画 1 本あたりおよそ 0.05〜0.10 ドル程度です。月額 500 円（約 3.5 ドル）のプランであれば、ユーザーあたり月 30〜40 回まで採算が合う計算になります。

## まとめ：動画解析を武器にするために

Gemini 2.5 Pro の Video Understanding は、「カメラで撮ったものをそのまま AI に見せる」というシンプルな体験を実現してくれます。実装の要点を整理すると以下になります。

Files API は 20MB 超の動画や再利用するファイルに。inline base64 は小さく 1 回限りの動画に使います。アップロード後は `ACTIVE` 状態を確認してから解析を開始します。長い動画はモデルとサンプリング戦略を組み合わせてコストを管理します。エラーハンドリングには「状態待機」と「リトライ」の 2 つを必ず組み込みましょう。

まず 30 秒以内の動画で基本的なフローを動かしてみることをおすすめします。「アップロード → 解析 → 結果表示」が端末で動いたとき、AI アプリの可能性の広さを実感できるはずです。

アプリのコンセプトが固まったら、[Rork × Gemini のストリーミングチャットUI設計](/articles/rork-ai/rork-max-ai-chatbot-gemini-streaming)の記事も参考にしながら、解析結果をリアルタイムに表示するUXを組み合わせてみてください。
