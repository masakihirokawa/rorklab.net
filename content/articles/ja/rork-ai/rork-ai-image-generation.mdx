## Rork Max × AI 画像生成ガイド

AI の急速な進化により、テキストから高品質な画像を自動生成する技術が現実となりましました。これをモバイルアプリに統合することで、デザインアシスタント、クリエイティブツール、パーソナライズされたコンテンツ生成など、革新的なユーザー体験が実現できます。

本ガイドでは、Rork Max アプリケーションに DALL-E や Imagen などの AI 画像生成 API を組み込む方法を詳細に解説します。

## AI 画像生成の基本概念

### プロンプトエンジニアリング

高品質な画像を生成するには、適切に構造化されたプロンプトが必須です。プロンプトエンジニアリングは、AI モデルから最良の結果を引き出すスキルです。

効果的なプロンプトの要素：
- **主体（Subject）**: 何を生成するか
- **スタイル（Style）**: アート風、写真風など
- **品質修飾子（Quality）**: 高解像度、詳細な など
- **背景・環境（Setting）**: どこの場所か

例: 「A serene Japanese garden with cherry blossoms, watercolor painting style, highly detailed, 8k resolution」

## DALL-E API との統合

### OpenAI DALL-E の実装

DALL-E は OpenAI が提供する高性能な画像生成 API です。

```jsx
import { useState } from 'react';
import axios from 'axios';

export function ImageGenerationScreen() {
  const [prompt, setPrompt] = useState('');
  const [generatedImage, setGeneratedImage] = useState(null);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState(null);

  const generateImage = async () => {
    setIsLoading(true);
    setError(null);

    try {
      const response = await axios.post(
        'https://api.openai.com/v1/images/generations',
        {
          prompt: prompt,
          n: 1,
          size: '1024x1024',
          quality: 'hd',
        },
        {
          headers: {
            'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
            'Content-Type': 'application/json',
          },
        }
      );

      const imageUrl = response.data.data[0].url;
      setGeneratedImage(imageUrl);
      console.log(`画像生成成功: ${imageUrl}`);
      // 期待出力: 画像生成成功: https://oaidalleapiprodscus...
    } catch (err) {
      console.error('画像生成エラー:', err);
      setError('画像生成に失敗しました');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      <textarea
        value={prompt}
        onChange={(e) => setPrompt(e.target.value)}
        placeholder="画像生成のプロンプトを入力..."
        style={{ width: '100%', height: '100px', marginBottom: '10px' }}
      />
      <button
        onClick={generateImage}
        disabled={isLoading || !prompt}
        style={{ marginBottom: '10px' }}
      >
        {isLoading ? '生成中...' : '画像を生成'}
      </button>

      {error && <p style={{ color: 'red' }}>{error}</p>}
      {generatedImage && (
        <img
          src={generatedImage}
          alt="Generated"
          style={{ maxWidth: '100%', maxHeight: '500px' }}
        />
      )}
    </div>
  );
}
```

DALL-E API は高品質な結果を提供しますが、リクエストごとにコストが発生します。キャッシング戦略を導入することが重要です。

## Google Imagen との連携

### Vertex AI Imagen の活用

Google Cloud の Imagen API は DALL-E に匹敵する品質を提供し、エンタープライズ向けの機能が充実しています。

```jsx
import { VertexAI } from '@google-cloud/vertexai';

export async function generateImageWithImagen(prompt) {
  const project = process.env.GCP_PROJECT_ID;
  const location = 'us-central1';

  const vertexAI = new VertexAI({
    project: project,
    location: location,
  });

  const imageGenerationModel = vertexAI.getGenerativeModel({
    model: 'imagegeneration@006',
  });

  try {
    const result = await imageGenerationModel.generateImages({
      prompt: prompt,
      sampleCount: 1,
      generationConfig: {
        mimeType: 'image/jpeg',
        width: 1024,
        height: 1024,
      },
    });

    console.log(`Imagen 画像生成完了: ${result.results[0].bytesBase64Encoded.slice(0, 50)}...`);
    // 期待出力: Imagen 画像生成完了: /9j/4AAQSkZJRgABAQEAYABgA...

    return result.results[0];
  } catch (error) {
    console.error('Imagen エラー:', error);
    throw error;
  }
}

export function GoogleImagenScreen() {
  const [prompt, setPrompt] = useState('');
  const [generatedImage, setGeneratedImage] = useState(null);

  const handleGenerate = async () => {
    try {
      const result = await generateImageWithImagen(prompt);
      // Base64 をデータ URI に変換
      const dataUri = `data:image/jpeg;base64,${result.bytesBase64Encoded}`;
      setGeneratedImage(dataUri);
    } catch (error) {
      console.error('生成失敗:', error);
    }
  };

  return (
    <div>
      <input
        type="text"
        value={prompt}
        onChange={(e) => setPrompt(e.target.value)}
        placeholder="プロンプト入力..."
      />
      <button onClick={handleGenerate}>Imagen で生成</button>
      {generatedImage && <img src={generatedImage} alt="Generated" />}
    </div>
  );
}
```

## プロンプト最適化戦略

### 効果的なプロンプト設計

AI モデルから最高品質の結果を得るには、プロンプトの質が最重要です。

```jsx
// プロンプトテンプレートの例
const promptTemplates = {
  portrait: (subject, style) =>
    `A stunning portrait of ${subject}, ${style} style, professional photography, studio lighting, sharp focus, highly detailed, 8k`,

  landscape: (location, mood) =>

  product: (item, background) =>
    `Professional product photography of ${item}, ${background}, studio lighting, clean composition, high resolution, commercial quality`,

export function PromptOptimizationExample() {
  const portraitPrompt = promptTemplates.portrait(
    'a Japanese businesswoman',
    'professional'

  console.log(portraitPrompt);
  // 期待出力: A stunning portrait of a Japanese businesswoman, professional style, ...
}
```

プロンプトテンプレートを使用することで、一貫性を保ちながら効率的に画像生成が可能です。

## キャッシングと最適化

### 生成済み画像のキャッシング

同じプロンプトで複数回生成を避け、コスト削減と応答速度向上を実現します。

```jsx
import { useRef } from 'react';

export function CachedImageGenerator() {
  const cacheRef = useRef(new Map());

  const generateImageWithCache = async (prompt) => {
    // キャッシュをチェック
    if (cacheRef.current.has(prompt)) {
      console.log('キャッシュからロード:', prompt);
      // 期待出力: キャッシュからロード: A serene garden...
      return cacheRef.current.get(prompt);
    }

    // キャッシュにない場合は新規生成
    const imageUrl = await generateImage(prompt);

    // キャッシュに保存（最大100個まで）
    if (cacheRef.current.size >= 100) {
      const firstKey = cacheRef.current.keys().next().value;
      cacheRef.current.delete(firstKey);
    }
    cacheRef.current.set(prompt, imageUrl);

    return imageUrl;
  };

  return (
    <div>
      {/* UI コンポーネント */}
    </div>
  );
}
```

キャッシュサイズは LRU（Least Recently Used）アルゴリズムで管理し、メモリ効率を保ちます。

## エラーハンドリングとレート制限

### API 呼び出しの安全な管理

AI API はレート制限があります。適切なエラーハンドリングと再試行ロジックが必須です。

```jsx
export async function generateImageWithRetry(
  prompt,
  maxRetries = 3,
  delayMs = 1000
) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetch('https://api.openai.com/v1/images/generations', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
        },
        body: JSON.stringify({
          prompt: prompt,
          n: 1,
          size: '1024x1024',
        }),
      });

      if (response.status === 429) {
        // レート制限: 待機して再試行
        console.log(`レート制限: ${delayMs}ms 待機します`);
        await new Promise(resolve => setTimeout(resolve, delayMs));
        continue;
      }

      if (!response.ok) {
        throw new Error(`API エラー: ${response.status}`);
      }

      const data = await response.json();
      console.log('画像生成成功');
      // 期待出力: 画像生成成功
      return data.data[0].url;
    } catch (error) {
      console.error(`試行 ${attempt + 1} 失敗:`, error);
      if (attempt === maxRetries - 1) throw error;
      await new Promise(resolve => setTimeout(resolve, delayMs));
    }
  }
}
```

## FAQ

### Q1: AI 画像生成の品質を最大化するにはどうする？

**A:** プロンプトに具体的な詳細情報を含めることが重要です。「風景」ではなく「Kyoto temple at golden hour with cherry blossoms, oil painting style, 8k resolution」といった詳細なプロンプトが効果的です。また、生成後の画像品質オプション（DALL-E では「hd」）を選択しましょう。

### Q2: 生成にかかる時間を短縮できますか？

**A:** API は通常 30 秒～2 分で完了します。キャッシングを積極的に使用し、同じプロンプトの再生成を避けることが最良の方法です。また、非同期処理でユーザーが待機中に他の操作ができるようにしましょう。

### Q3: 著作権や利用規約について注意することは？

**A:** DALL-E や Imagen で生成された画像は、利用契約に従う必要があります。商用利用の場合は各サービスの規約を確認してください。また、ユーザーが生成した画像の権利についても、アプリの利用規約で明確にしておくことが重要です。
