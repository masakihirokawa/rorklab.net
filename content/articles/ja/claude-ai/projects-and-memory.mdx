## プロジェクトとは

Claude.ai のプロジェクトは、会話、指示、ファイルを 1 か所で管理できる組織化されたワークスペースです。すべての会話を新規開始する代わりに、プロジェクトはコンテキスト、指示、ファイルを保持し、ワークフローをより効率的にし、会話をより個人化します。

<Callout type="info">
プロジェクトはフリープランを含む全ての Claude.ai プランで利用可能です。各ユーザーはプロジェクトの作成と管理にアクセスできます。
</Callout>

## なぜプロジェクトを使うのか?

### 従来のアプローチ (プロジェクトなし)
- すべての会話は最初から開始される
- 以前のやり取りのメモリがない
- 頻繁に指示とコンテキストを繰り返す
- ファイルが異なる会話に散在している

### プロジェクトベースのアプローチ
- 会話全体で持続するコンテキスト
- カスタム指示が保存され再利用される
- 一元化されたファイル管理
- より良い組織化とチーム協力

## 最初のプロジェクトを作成する

### ステップバイステップセットアップ

1. **Claude.ai に移動** してアカウントにログインします
2. **「新しいプロジェクト」** をサイドバーからクリックします
3. **プロジェクト詳細を入力します:**
   - **プロジェクト名**: 説明的な名前を選択 (例: 「ブログ執筆」、「研究 2026 年 Q1」)
   - **説明**: オプションですが、プロジェクトの目的を覚えておくのに役立ちます
4. **プロジェクト内で最初の会話を開始します**

作成されると、このプロジェクト内のすべての会話が整理され、プロジェクト全体に適用されるカスタム指示を追加できます。

## プロジェクト指示を追加する

プロジェクト指示はそのプロジェクト内のすべての会話に適用されるカスタムガイドラインです。執筆スタイル、技術的な好み、または役割固有の動作を定義するのに最適です。

### 例: 執筆プロジェクト指示

```
# 執筆プロジェクト指示

## トーンとスタイル
- 明確で簡潔な言語を使う
- 必要でない限り専門用語を避ける
- 一般的なオーディエンス向けに書く
- 要点を説明するために例を含める

## 構造
- フック質問で始める
- わかりやすい見出しを使う
- 実行可能なポイントで終わる
- パラグラフを 3～4 文に制限する

## フォーマット
- markdown フォーマットを使う
- 関連する場合はリンクを含める
- リストに箇条書きを追加する
```

<Callout type="tip">
プロジェクト指示は立ったままの指示のようなものです。Claude はそれぞれの会話の始めにそれらを読むため、すべてのメッセージで繰り返す必要がありません。
</Callout>

### 例: 開発プロジェクト指示

```
# 開発プロジェクト指示

## テック スタック
- App Router 付き Next.js 16
- TypeScript (厳密モード)
- Tailwind CSS v4
- データベース用 PostgreSQL

## コーディング規約
- hooks を使った関数型コンポーネントを使う
- すべての props に対して TypeScript インターフェイスを書く
- エクスポートされた関数に JSDoc コメントを含める
- スペースと命名については Airbnb スタイル ガイドに従う

## テスト要件
- Vitest でユニットテストを書く
- 80% 以上のコード カバレッジを目指す
- 成功パスとエッジ ケースの両方をテストする
```

## ファイルの管理

プロジェクトを使用すると、Claude がそのプロジェクト内のすべての会話全体で参照できるドキュメント、コード ファイル、データセット、その他のリソースを添付できます。

### ファイルのアップロード方法

1. **プロジェクトを開き** て会話を開始します
2. **チャット ボックスの添付アイコン** (クリップ) をクリックします
3. **コンピューターからファイルを選択** します
4. **サポートされている形式:**
   - ドキュメント: PDF、DOCX、TXT、MD
   - コード: PY、JS、TS、JAVA、CPP、C#など
   - データ: CSV、JSON、XLSX
   - 画像: PNG、JPG、GIF
   - アーカイブ: ZIP、TAR.GZ

アップロードされたら、Claude はプロジェクト内のすべての会話でこれらのファイルを参照できます。

### ファイル管理のベストプラクティス

- **ファイルを整理する:** わかりやすいファイル名を使う
- **古いファイルを削除する:** 不要になったファイルをアーカイブまたは削除する
- **重要なファイルをバージョン管理する:** 日付を含めることを検討する (例: 「requirements_2026-03-01.md」)
- **ファイルサイズ制限を尊重する:** 大きなファイルはパフォーマンスに影響を与える可能性があります

<Callout type="warning">
一部のファイル タイプには処理制限があります。ファイルが非常に大きい場合 (>10MB)、アップロード前にそれを分割または圧縮することを検討してください。
</Callout>

## メモリ機能

メモリ機能はあなた、あなたの好み、プロジェクトの目標に関する重要な詳細を自動的に覚えています。明示的にアップロードしているプロジェクト ファイルとは異なり、メモリはやり取りに基づいて学習し、進化します。

### メモリが検出するもの

- コミュニケーション スタイルの好み
- 使用する技術的な好み規約
- 繰り返される目標優先度
- 頻繁に議論する人々とプロジェクト
- 使用する命名規約と用語

### メモリの動作方法

1. **Claude はパターンを観察する** あなたの会話全体から
2. **メモリは自動的に更新される** あなたのやり取りに基づいて
3. **今後の会話に適用される** 情報を繰り返す必要なく
4. **手動でメモリ エントリを確認および編集できる** あなたはいつでも

### メモリの管理

いつでもメモリを確認および編集できます:

1. **プロジェクトを開く**
2. **「メモリ」** または **「プロジェクト設定」** をクリック
3. **保存されている情報を確認する** あなたの好みについて
4. **エントリを編集または削除する** 必要に応じて

<Callout type="info">
メモリはプロジェクト固有です。各プロジェクトは独自のメモリを維持し、異なるプロジェクトに対して異なる好みを持つことができます。
</Callout>

### メモリの例

**執筆プロジェクト メモリ:**
- イギリス英語の綴りを好む
- 箇条書きよりも番号付きリストを好む
- ターゲット オーディエンス: ビジネス プロフェッショナル
- 執筆スタイル: 会話的だがプロフェッショナル

**開発プロジェクト メモリ:**
- JavaScript 変数に camelCase を使う
- JavaScript より TypeScript を好む
- テスティング フレームワーク: Vitest
- データベース: PostgreSQL

## チームとプロジェクトを共有する

プロジェクトはチーム メンバーと共有でき、シームレスなコラボレーションが可能になります。

### プロジェクトを共有する方法

1. **プロジェクトを開く**
2. **「共有」** または **「プロジェクト設定」** をクリック
3. **メール アドレスでチーム メンバーを追加** します
4. **権限を設定します:**
   - **表示のみ**: 会話とファイルを読むことができます
   - **編集**: 会話を修正し、ファイルを追加できます
   - **管理者**: 共有や削除を含む完全なコントロール

### チーム コラボレーションのメリット

- **共有コンテキスト:** 全員がプロジェクトバックグラウンドを理解
- **統一された会話:** チーム メンバーがお互いの議論を続けられる
- **一貫した指示:** プロジェクト指示がすべてのチーム メンバーに適用される
- **ファイル アクセス:** すべてのチーム メンバーが共有ファイルを参照できます

## 複数プロジェクトの整理

さらに多くのプロジェクトを作成するにつれて、組織化が重要になります。

### プロジェクト組織戦略

1. **タイプ別:** 執筆、開発、研究などの個別プロジェクト
2. **クライアント/チーム別:** 各クライアントまたはチーム専用のプロジェクト
3. **時間帯別:** 四半期または年次プロジェクト (例: 「2026 年 Q1 マーケティング」)
4. **主題別:** 主題別にグループ化されたプロジェクト

### 命名規約のヒント

- 明確でわかりやすい名前を使う
- 時間に敏感な場合は日付を含める (例: 「Book Draft 2026-Q1」)
- 「Project 1」や「Misc」のような一般的な名前を避ける
- 名前を簡潔にする (最大 3～5 単語)

## プロジェクト設定とカスタマイズ

各プロジェクトには、ニーズに応じてカスタマイズするオプションがあります。

### 利用可能な設定

- **プロジェクト名と説明** — いつでも更新可能
- **カスタム指示** — 動作と好みを定義する
- **ファイル添付** — リソースをアップロードおよび管理する
- **共有設定** — アクセス可能なユーザーをコントロール
- **アーカイブ/削除** — 非アクティブなプロジェクトを管理

## プロジェクト管理のベストプラクティス

1. **明確な目的から始める** — 作成する前にプロジェクトが何であるかを知る
2. **詳細な指示を書く** — より具体的であるほど、Claude の回答は向上する
3. **ファイルを最新に保つ** — 定期的に古いファイルまたは関連のないファイルを削除する
4. **メモリを定期的に確認する** — 保存されている好みが正確であることを確認する
5. **戦略的に共有する** — アクセスが必要なユーザーとのみ共有する
6. **一貫性のある名前を付ける** — 明確でわかりやすいプロジェクト名を使う


## 一般的な問題のトラブルシューティング

### Claude が以前の会話を覚えていない
- **解決策:** 同じプロジェクトにいることを確認します。異なるプロジェクトはメモリを共有しません。
- プロジェクト指示が正しく保存されていることを確認します。

### ファイルがアップロードされない
- **解決策:** ファイル サイズと形式の互換性を確認します。
- テストするために小さいファイルを試すか、大きいファイルを分割します。

### メモリが更新されない
- **解決策:** メモリは自動的に更新されますが、数回の会話が必要な場合があります。
- プロジェクト設定でメモリを手動で編集できます。
