## Extended Thinking とは

**Extended Thinking（拡張思考）** は、Claude に回答前の「思考プロセス」を許可する機能です。通常の応答では内部で瞬時に処理される推論を、明示的なステップとして展開させることで、複雑な問題でより正確な回答を導き出します。

## なぜ重要なのか

人間が難しい数学の問題を解くとき、いきなり答えを書くのではなく、紙に途中計算を書きながら考えます。Extended Thinking は Claude にも同じ「紙に書いて考える」プロセスを与えるものです。

### 効果が高いタスク

- 多段階の数学的証明
- 複雑なコードのデバッグ・設計
- 法律文書や契約書の分析
- 科学的な仮説の検証
- 長文の論理的な構成

### 効果が限定的なタスク

- 簡単な Q&A（「日本の首都は？」など）
- 定型的な文書作成
- 短い翻訳

## Claude.ai での使い方

Claude.ai では特別な設定は不要です。Claude は質問の複雑さに応じて自動的に思考の深さを調整します。ただし、明示的に深い思考を求めることもできます。

```
「この問題をステップバイステップで慎重に考えて回答してください」
```

## API での使い方

API では `thinking` パラメータを設定して Extended Thinking を有効にします。

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000
    },
    messages=[{
        "role": "user",
        "content": "素数が無限に存在することを証明してください"
    }]
)

for block in response.content:
    if block.type == "thinking":
        print(f"思考過程: {block.thinking}")
    elif block.type == "text":
        print(f"回答: {block.text}")
```

### パラメータの説明

- **type**: `"enabled"` で手動有効化。Claude Opus 4.6 では `"adaptive"` も利用可能
- **budget_tokens**: 思考に使える最大トークン数。`max_tokens` より小さい値を設定

## Adaptive Thinking

Claude Opus 4.6 では **Adaptive Thinking** という新しいモードが利用できます。

```python
thinking={"type": "adaptive"}
```

`budget_tokens` を指定する必要がなく、問題の複雑さに応じて Claude が自動的に思考の深さを調整します。

## コストに関する注意

Extended Thinking のトークンは課金対象です。思考過程のトークン数は出力トークンとしてカウントされるため、`budget_tokens` の設定はコスト管理の観点からも重要です。

簡単な質問に大きな思考バジェットを設定すると無駄なコストが発生するため、タスクの複雑さに応じて適切な値を設定しましょう。

## 実践的な活用例

### コードレビュー

```
thinking={"type": "enabled", "budget_tokens": 8000}

「このコードにセキュリティ上の脆弱性がないか、攻撃パターンごとに検証してください」
```

### データ分析

```
thinking={"type": "enabled", "budget_tokens": 12000}

「この売上データのトレンドを分析し、来期の予測を根拠とともに提示してください」
```

## まとめ

Extended Thinking は、Claude を「即答マシン」から「深く考えるパートナー」に変える機能です。複雑な推論が必要な場面で積極的に活用し、簡単なタスクでは通常モードを使うことで、品質とコストのバランスを最適化できます。
