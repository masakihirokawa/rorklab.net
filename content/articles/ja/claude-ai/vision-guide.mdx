## はじめに

Claude の Vision 機能は、テキストと共に画像を処理する強力なマルチモーダル機能です。画像分析、図表の解析、スクリーンショットの理解など、様々な用途で活用できます。本ガイドでは、Claude の画像認識機能を最大限に活用するための方法とベストプラクティスを詳しく説明します。

## 対応フォーマットと制限事項

### サポートされているファイル形式

Claude Vision は以下の画像フォーマットに対応しています：

- **JPEG** - 最も一般的な形式。高い互換性があります
- **PNG** - ロスレス圧縮。透明度情報をサポート
- **GIF** - アニメーション画像もサポート（最初のフレームを処理）
- **WebP** - 最新の圧縮形式。より効率的なサイズ

### 複数画像の処理

- 1つのAPI リクエストで**最大100枚の画像**を処理可能
- 複数の画像を組み合わせた複雑な分析が可能
- メモリと処理時間の効率化が実現できます

> **ヒント**: 100枚の制限に達しそうな場合は、関連する画像をグループ化して複数のリクエストに分割することをお勧めします。

## 画像入力の3つの方法

### 方法1：Base64 エンコーディング

画像をBase64形式でエンコードしてリクエストに含める方法です。小さな画像やテスト環境に適しています。

```python
import anthropic
import base64
from pathlib import Path

client = anthropic.Anthropic()

# 画像をBase64にエンコード
image_path = "path/to/image.jpg"
with open(image_path, "rb") as image_file:
    image_data = base64.standard_b64encode(image_file.read()).decode("utf-8")

# メディアタイプを決定
media_type = "image/jpeg"  # JPEG, PNG, GIF, WebPに対応

message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": media_type,
                        "data": image_data,
                    },
                },
                {
                    "type": "text",
                    "text": "この画像の内容を詳しく説明してください。"
                }
            ],
        }
    ],
)

print(message.content[0].text)
```

### 方法2：URLによる参照

インターネット上で公開されている画像をURLで直接参照できます。ファイルアップロードが不要で、スピーディです。

```python
import anthropic

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "url",
                        "url": "https://example.com/image.jpg",
                    },
                },
                {
                    "type": "text",
                    "text": "このスクリーンショットのUI要素を説明してください。"
                }
            ],
        }
    ],
)

print(message.content[0].text)
```

> **注意**: URLで参照する場合、画像は公開アクセス可能である必要があります。プライベート画像の場合はBase64または Files API を使用してください。

### 方法3：Files API

大規模な画像ファイルや長期保存が必要な場合、Files API を使用します。ファイルをアップロードして一意のID を取得します。

```python
import anthropic

client = anthropic.Anthropic()

# ファイルをアップロード
with open("path/to/image.png", "rb") as image_file:
    response = client.beta.files.upload(
        file=("image.png", image_file, "image/png"),
    )
    file_id = response.id

# アップロードされたファイルを使用してメッセージを作成
message = client.beta.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "file",
                        "file_id": file_id,
                    },
                },
                {
                    "type": "text",
                    "text": "この図表が示しているデータの傾向は何ですか？"
                }
            ],
        }
    ],
    biliing_metadata={"usage_metadata": True},
)

print(message.content[0].text)
```

## トークン計算

Claude の Vision 機能では、画像はトークンとして計算されます。テキストの詳細度によってトークン数が変わります。

### トークン計算式

基本的なトークン計算式は以下の通りです：

```
トークン数 = (画像の幅 × 画像の高さ) ÷ 750
```

### 具体例

以下に一般的な画像サイズのトークン計算例を示します：

| 画像サイズ | 計算式 | トークン数（概算） |
|-----------|--------|------------------|
| 200×200px | (200 × 200) ÷ 750 | ~53 トークン |
| 400×400px | (400 × 400) ÷ 750 | ~213 トークン |
| 600×600px | (600 × 600) ÷ 750 | ~480 トークン |
| 800×800px | (800 × 800) ÷ 750 | ~853 トークン |
| 1000×1000px | (1000 × 1000) ÷ 750 | ~1,333 トークン |
| 1092×1092px | (1092 × 1092) ÷ 750 | ~1,588 トークン |

> **最適化のコツ**: 画像を 1568px 以下にリサイズすることで、処理効率を向上させ、トークン消費を削減できます。

## 価格設定

画像処理のコストは、画像のサイズと数に基づいて計算されます。

### 200×200px 画像の場合

- トークン数：約 53 トークン
- コスト：**約 $0.00016**

### 600×600px 画像の場合

- トークン数：約 480 トークン
- コスト：**約 $0.00145**

### 1092×1092px 画像の場合

- トークン数：約 1,588 トークン
- コスト：**約 $0.0048**

```typescript
// TypeScript での価格計算例
function calculateImageTokens(width: number, height: number): number {
  return Math.ceil((width * height) / 750);
}

function estimateCost(tokenCount: number, costPerToken: number = 0.000004): number {
  return tokenCount * costPerToken;
}

// 使用例
const width = 800;
const height = 600;
const tokens = calculateImageTokens(width, height);
const estimatedCost = estimateCost(tokens);

console.log(`画像サイズ: ${width}×${height}px`);
console.log(`トークン数: ${tokens}`);
console.log(`推定コスト: $${estimatedCost.toFixed(6)}`);
```

## ベストプラクティス

### 1. 画像の配置

テキスト質問の**前に画像を配置**することが重要です。これにより、モデルは画像コンテンツをコンテキストとして理解してから質問に回答します。

```python
# 良い例
message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",  # テキストの前に画像
                    "source": {
                        "type": "url",
                        "url": "https://example.com/chart.png",
                    },
                },
                {
                    "type": "text",
                    "text": "この売上チャートの主な傾向を説明してください。"
                }
            ],
        }
    ],
)
```

### 2. 画像のリサイズ

パフォーマンスとコスト最適化のため、画像を最大 1568px にリサイズしてください。

```python
from PIL import Image

def resize_image(image_path: str, max_dimension: int = 1568) -> str:
    """
    画像を指定された最大寸法にリサイズします
    """
    img = Image.open(image_path)

    # アスペクト比を保持しながらリサイズ
    img.thumbnail((max_dimension, max_dimension), Image.Resampling.LANCZOS)

    # リサイズされた画像を保存
    output_path = image_path.replace(".jpg", "_resized.jpg")
    img.save(output_path, quality=95)

    return output_path

# 使用例
resized_path = resize_image("original_image.jpg")
```

### 3. 複数画像の効果的な使用

複数の画像を分析する場合は、関連性のあるものをグループ化します。

```python
import anthropic

client = anthropic.Anthropic()

# 複数の画像を処理
images_urls = [
    "https://example.com/chart1.png",
    "https://example.com/chart2.png",
    "https://example.com/chart3.png",
]

content = []

# すべての画像を追加
for url in images_urls:
    content.append({
        "type": "image",
        "source": {
            "type": "url",
            "url": url,
        },
    })

# テキスト質問を最後に追加
content.append({
    "type": "text",
    "text": "これら3つのグラフを比較して、各四半期の傾向を説明してください。"
})

message = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=2048,
    messages=[
        {
            "role": "user",
            "content": content,
        }
    ],
)

print(message.content[0].text)
```

## 制限事項と注意点

Claude Vision 機能にはいくつかの重要な制限があります。適切に理解して使用することが大切です。

### 1. 人物識別不可

**セキュリティとプライバシーの理由により、Claude は画像内の人物を特定または識別することはできません。**

```python
# これはできません
prompt = "この写真の人物は誰ですか？"  # 失敗
prompt = "この人物の顔の特徴を説明してください"  # 失敗

# これはできます
prompt = "この写真に写っている人物の数は何人ですか？"  # 成功
prompt = "人物が着ている服装について説明してください"  # 成功
```

### 2. 空間推論の制限

Claude の空間推論能力には制限があります。複雑な3D配置や精密な位置関係の理解は得意ではありません。

```python
# 空間推論の例
prompt = "物体Aと物体Bの間の距離を計算してください"  # 不正確
prompt = "部屋の3D レイアウトを説明してください"  # 近似的

# より正確な結果が期待できる
prompt = "この部屋には主にどのような家具がありますか？"  # より正確
prompt = "物体Aはどの物体の近くにありますか？"  # より信頼性が高い
```

### 3. 数値の概算のみ

画像内のオブジェクトの正確なカウント、特に密集した集合は、概算値のみが返されます。

```python
# これは概算です
prompt = "群衆の中に何人の人物がいますか？"  # 概算値のみ

# これはより正確
prompt = "この写真に犬は何匹写っていますか？"  # 小さい数であればより正確
prompt = "テーブルの上のコップはいくつありますか？"  # より正確
```

> **警告**: 金銭的決定や医学的診断など、高精度が必要な場合は、画像の自動カウント機能に依存しないでください。常に人的確認を行ってください。

## トラブルシューティング

### 画像が処理されない場合

1. **ファイル形式の確認**: JPEG、PNG、GIF、WebP のいずれかであることを確認
2. **ファイルサイズ**: 極端に大きなファイルは事前にリサイズ
3. **Base64エンコーディング**: 正しくエンコードされていることを確認

### 精度が低い場合

1. **画像の品質**: より高品質な画像を使用
2. **詳細な指示**: プロンプトをより具体的に
3. **文脈提供**: 背景情報を含める

### API エラーが発生する場合

- Rate limit に達していないか確認
- トークン数が超過していないか確認
- 画像URLがアクセス可能か確認
