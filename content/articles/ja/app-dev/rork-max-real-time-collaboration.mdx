## リアルタイム機能がアプリを変える

Notion、Figma、Google Docs——現代の生産性アプリには必ずといっていいほど「リアルタイムコラボレーション」機能が備わっています。複数人が同じドキュメントやキャンバスを同時に編集でき、相手のカーソルや変更がリアルタイムで見える。この体験がユーザーエンゲージメントを大幅に高めます。

Rork Max を使えば、このようなリアルタイム機能をモバイルアプリに比較的少ないコードで実装できます。この記事では、Supabase Realtime をバックエンドに使ったリアルタイムコラボレーション機能の実装を、3つの具体的なユースケースで解説します。

---

## アーキテクチャ概要

今回実装するシステムの全体像は以下の通りです：

```
[ユーザーA の Rork アプリ]
        ↕ WebSocket
[Supabase Realtime サーバー]
        ↕ WebSocket
[ユーザーB の Rork アプリ]
```

Supabase Realtime は PostgreSQL の変更をリアルタイムでクライアントに配信する仕組み（Broadcast、Presence、PostgreSQL Changes）を提供します。これを Rork Max のプロンプトで適切に指示することで、複雑な WebSocket 実装を自前で書かずに済みます。

## セットアップ：Supabase プロジェクトの準備

まず Supabase でプロジェクトを作成し、以下のテーブルを作成します：

```sql
-- ドキュメントテーブル
CREATE TABLE documents (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  title text NOT NULL,
  content text DEFAULT '',
  updated_at timestamptz DEFAULT now(),
  updated_by text
);

-- リアルタイム有効化
ALTER PUBLICATION supabase_realtime ADD TABLE documents;

-- RLS（行レベルセキュリティ）を適切に設定
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;
CREATE POLICY "全ユーザーが読み書き可能" ON documents
  FOR ALL USING (true) WITH CHECK (true);
```

次に Rork Max で新規プロジェクトを作成し、以下を設定します：

```
Supabase URL: [your-project].supabase.co
Supabase Anon Key: [your-anon-key]
```

## ユースケース 1: リアルタイムチャット

最もシンプルなリアルタイム機能から始めましょう。Rork Max へのプロンプト：

```
Supabase Realtime の Broadcast チャンネルを使って、
ルームIDベースのリアルタイムチャット画面を作ってください。

要件:
- メッセージは Supabase の messages テーブルに保存
- 新規メッセージは Broadcast で即時配信
- 自分のメッセージは右、相手は左に表示
- ユーザー名とタイムスタンプを表示
- 送信中インジケーター（typing indicator）を表示
```

Rork Max が生成するコアロジック：

```typescript
import { createClient } from '@supabase/supabase-js';
import { useEffect, useState, useRef } from 'react';

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

interface Message {
  id: string;
  userId: string;
  userName: string;
  text: string;
  timestamp: string;
}

export function useRealTimeChat(roomId: string, userId: string) {
  const [messages, setMessages] = useState<Message[]>([]);
  const [typingUsers, setTypingUsers] = useState<string[]>([]);
  const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null);

  useEffect(() => {
    // Broadcast チャンネルに接続
    const channel = supabase.channel(`room:${roomId}`, {
      config: { broadcast: { self: false } },
    });

    channelRef.current = channel;

    // メッセージ受信
    channel.on('broadcast', { event: 'message' }, ({ payload }) => {
      setMessages(prev => [...prev, payload as Message]);
    });

    // タイピングインジケーター
    channel.on('broadcast', { event: 'typing' }, ({ payload }) => {
      const { userName, isTyping } = payload;
      setTypingUsers(prev =>
        isTyping
          ? [...prev.filter(u => u !== userName), userName]
          : prev.filter(u => u !== userName)
      );
    });

    channel.subscribe();

    // 既存メッセージを取得
    supabase
      .from('messages')
      .select('*')
      .eq('room_id', roomId)
      .order('timestamp', { ascending: true })
      .then(({ data }) => {
        if (data) setMessages(data);
      });

    return () => {
      channel.unsubscribe();
    };
  }, [roomId]);

  const sendMessage = async (text: string, userName: string) => {
    const message: Message = {
      id: crypto.randomUUID(),
      userId,
      userName,
      text,
      timestamp: new Date().toISOString(),
    };

    // DBに保存 + ブロードキャスト
    await supabase.from('messages').insert({ ...message, room_id: roomId });
    await channelRef.current?.send({
      type: 'broadcast',
      event: 'message',
      payload: message,
    });
  };

  const sendTyping = async (userName: string, isTyping: boolean) => {
    await channelRef.current?.send({
      type: 'broadcast',
      event: 'typing',
      payload: { userName, isTyping },
    });
  };

  return { messages, typingUsers, sendMessage, sendTyping };
}
```

## ユースケース 2: プレゼンス（オンライン状態の表示）

「今誰が画面を見ているか」をリアルタイムで表示する Presence 機能です：

```
Supabase Realtime の Presence を使って、
同じドキュメントを開いているユーザーのアバターリストをヘッダーに表示してください。
ユーザーがアプリを離れると自動的にリストから消える仕様にしてください。
```

```typescript
export function usePresence(documentId: string, currentUser: User) {
  const [onlineUsers, setOnlineUsers] = useState<User[]>([]);

  useEffect(() => {
    const channel = supabase.channel(`presence:${documentId}`);

    channel
      .on('presence', { event: 'sync' }, () => {
        const state = channel.presenceState<User>();
        const users = Object.values(state).flat();
        setOnlineUsers(users);
      })
      .on('presence', { event: 'join' }, ({ newPresences }) => {
        console.log('joined:', newPresences);
      })
      .on('presence', { event: 'leave' }, ({ leftPresences }) => {
        console.log('left:', leftPresences);
      })
      .subscribe(async (status) => {
        if (status === 'SUBSCRIBED') {
          // 自分の存在を送信
          await channel.track({
            id: currentUser.id,
            name: currentUser.name,
            avatar: currentUser.avatar,
            joinedAt: new Date().toISOString(),
          });
        }
      });

    return () => {
      channel.unsubscribe();
    };
  }, [documentId, currentUser.id]);

  return { onlineUsers };
}
```

## ユースケース 3: リアルタイムドキュメント同期

PostgreSQL Changes を使って、データベースの変更を他のクライアントにリアルタイム配信します：

```
documents テーブルの変更（UPDATE）を Supabase Realtime で
他のクライアントに即時配信するドキュメントエディタを作ってください。
編集中はデバウンスをかけて過剰なDB書き込みを防いでください。
```

```typescript
export function useRealtimeDocument(documentId: string) {
  const [content, setContent] = useState('');
  const [lastEditor, setLastEditor] = useState<string | null>(null);
  const debounceRef = useRef<NodeJS.Timeout>();

  useEffect(() => {
    // 初期データ取得
    supabase
      .from('documents')
      .select('content, updated_by')
      .eq('id', documentId)
      .single()
      .then(({ data }) => {
        if (data) {
          setContent(data.content);
          setLastEditor(data.updated_by);
        }
      });

    // リアルタイム変更を購読
    const channel = supabase
      .channel(`document:${documentId}`)
      .on(
        'postgres_changes',
        {
          event: 'UPDATE',
          schema: 'public',
          table: 'documents',
          filter: `id=eq.${documentId}`,
        },
        (payload) => {
          setContent(payload.new.content);
          setLastEditor(payload.new.updated_by);
        }
      )
      .subscribe();

    return () => {
      channel.unsubscribe();
    };
  }, [documentId]);

  const updateContent = useCallback((newContent: string, userId: string) => {
    setContent(newContent); // ローカルを即時更新

    // デバウンス: 500ms後にDBへ書き込み
    clearTimeout(debounceRef.current);
    debounceRef.current = setTimeout(async () => {
      await supabase
        .from('documents')
        .update({
          content: newContent,
          updated_by: userId,
          updated_at: new Date().toISOString(),
        })
        .eq('id', documentId);
    }, 500);
  }, [documentId]);

  return { content, lastEditor, updateContent };
}
```

## 競合解決：楽観的更新とコンフリクト防止

複数ユーザーが同時に編集する場合、変更の衝突が起きる可能性があります。Rork Max に以下のプロンプトで対策を組み込みます：

```
ドキュメントの同時編集でコンフリクトが起きた場合、
「最後に書き込んだ人が優先（Last Write Wins）」方式で処理してください。
また、編集中のフィールドにはロックインジケーターを表示して、
他のユーザーが編集中であることを視覚的に示してください。
```

Rork Max はこれに対してロック管理ロジックと UI フィードバックを自動生成します。複雑な OT（操作変換）や CRDT が不要なシンプルなユースケースでは、LWW（Last Write Wins）で十分対応できます。

## パフォーマンス最適化のヒント

リアルタイム機能はネットワーク負荷が高くなりがちです。以下の点を Rork Max に指示して対策しましょう：

**1. メッセージのバッチ処理**
頻繁なブロードキャストは 100ms のデバウンスでまとめる

**2. チャンネルの適切な管理**
画面遷移時に必ずチャンネルを `unsubscribe()` する

**3. フィルタリングの活用**
`postgres_changes` の `filter` パラメータで不要な変更を購読しない

**4. 接続数の管理**
Supabase の無料プランでは同時接続数に制限があるため、必要最小限のチャンネル数に抑える
