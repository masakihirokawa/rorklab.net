## 1位：Rorkで作ったアプリが実機でクラッシュする——原因別の対処法

Rork のプレビューでは問題なく動いていたのに、TestFlight や実機ビルドに入れた瞬間にアプリが起動せずクラッシュする——この壁にぶつかった方は多いと思います。記事では、JS バンドル、ネイティブ依存、権限周りのマニフェスト、リリースビルド特有の最適化差分など、原因別に切り分けの手順をまとめました。

私自身、自分の壁紙アプリで「プレビューでは完璧、実機ではスプラッシュ直後に落ちる」状態に半日はまったことがあります。その経験ベースで、最初にどこから潰していくべきかの順序を書いたつもりです。

[Rorkで作ったアプリが実機でクラッシュする——原因別の対処法](/articles/rork-dev/rork-app-crashes-on-device-fix)

## 2位：Rorkアプリが iOS で動くのに Android だけおかしい：プラットフォーム差異を修正する実践ガイド

Rork（および React Native）で作ったアプリは、基本的にクロスプラットフォームで動きますが、細部に入ると iOS / Android の挙動差がどうしても出ます。この記事では、ステータスバー、キーボード挙動、フォント、ファイルパス、権限ダイアログ——プラットフォーム差異で詰まりやすいトピックを、具体的な修正例とセットで整理しました。

個人的に一番「やっかいだな」と感じるのは、Android だけでボタンのタップが2回必要になる現象です。原因が複数ある中でも、ジェスチャーハンドラの設定ミスか ripple の設定が影響しているケースが多いので、その切り分け方も本記事に含めています。

[Rorkアプリが iOS で動くのに Android だけおかしい](/articles/rork-dev/rork-ios-android-behavior-difference-fix)

## 3位：Rork Max × AWS Amplify Gen 2 完全実装ガイド — Cognito認証・AppSync GraphQL・S3ストレージを一気通貫で構築する

Rork Max で作ったアプリに、AWS Amplify Gen 2 をバックエンドとして統合する本格派のプレミアム記事です。Cognito での認証、AppSync による GraphQL API、S3 でのメディアストレージ——個別に組んだら週単位で沈むトピックを、一気通貫で実装する手順を整理しました。

Firebase 一択だった個人開発者の方にも、「ここだけは AWS が向いている」というユースケースは確実にあります。本記事では、そのユースケース判断も含めて、Amplify を選ぶ意義から書いています。「大手企業向けの B2B2C アプリを個人で作る」という読者の方から質問をいただくことが多かったテーマでもあります。

[Rork Max × AWS Amplify Gen 2 完全実装ガイド](/articles/rork-dev/rork-max-aws-amplify-gen2-complete-guide)

## 4位：Rork × Fastlane × EAS リリース自動化完全ガイド — スクリーンショット生成・メタデータ同期・TestFlight/Play Console 配信をワンコマンド化する実装パターン

リリース作業のたびに手元で何時間も拘束される、その時間を全部取り戻すためのプレミアム記事です。Fastlane と EAS を組み合わせて、スクリーンショット生成、メタデータ同期、TestFlight / Play Console への配信を、ワンコマンドで完結させる実装パターンを丁寧に書きました。

私自身、アプリの更新作業で「プレビュー録画 → スクショ撮り直し → テキスト更新 → ビルド → 配信」を手動でやっていた時期があり、毎週末がそれで潰れていました。記事で紹介している一本化のスクリプトを導入してから、1本あたり30分まで圧縮できた実感があります。

[Rork × Fastlane × EAS リリース自動化完全ガイド](/articles/rork-dev/rork-fastlane-eas-release-automation-complete-guide)

## 5位：Rork × Notion API で CMS 駆動アプリを作る実装ガイド — 認証・ブロック描画・レート制限・画像URLの運用まで

Notion を CMS として使い、Rork アプリのコンテンツを非エンジニアでも更新できるようにするプレミアム記事です。OAuth 認証、ブロック単位のレンダリング、Notion API の厳しいレート制限、そして「1時間で期限切れになる画像 URL」の実務運用——これらを一つずつ解決する実装を書きました。

個人開発者にとって、Notion CMS は「自分で管理画面を作らずに済む」点が最大の魅力です。ただし、画像 URL の期限問題は本番運用で必ずぶつかる壁なので、そこをどうキャッシュ＆再発行するかのパターンを、コード例つきで解説しました。
