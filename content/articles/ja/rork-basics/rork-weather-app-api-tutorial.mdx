## はじめに — 天気予報アプリで学ぶ API 連携の基本

天気予報アプリは、モバイルアプリ開発の基礎を学ぶうえで最適なプロジェクトのひとつです。外部 API からデータを取得し、リアルタイムで画面に反映するという一連の流れは、多くのアプリ開発に応用できる重要なスキルです。

**Rork** を使えば、こうした API 連携アプリをプロンプト入力だけで驚くほど短時間に構築できます。本記事では、OpenWeatherMap API と連携して現在の天気情報と週間予報を表示するアプリを、ゼロから作成する方法をご紹介します。

この記事で学べること:

- Rork で外部 API を呼び出すアプリの作り方
- OpenWeatherMap API の無料プランでできること
- 位置情報を使った現在地の天気取得
- 天気に応じた動的 UI デザインの実装方法

対象読者は、Rork を初めて使う方や、API 連携アプリを作ったことがない方です。[Rork の基本的な使い方](/articles/rork-basics/getting-started)を一通り理解していれば、スムーズに進められます。

## 前提知識と環境準備

### 必要なもの

天気予報アプリを作成するにあたり、以下を準備してください。

1. **Rork アカウント** — 無料プランでも作成可能です
2. **OpenWeatherMap API キー** — [openweathermap.org](https://openweathermap.org/api) で無料アカウントを作成し、API キーを取得します
3. **基本的なアプリ開発の知識** — プロンプトの書き方がわかれば十分です

### OpenWeatherMap API キーの取得手順

OpenWeatherMap は、天気データを提供する最も広く使われている API の一つです。無料プランでは、1分あたり60回のリクエストが可能で、個人アプリには十分な量です。

1. [openweathermap.org](https://openweathermap.org/) にアクセスし、アカウントを作成
2. ダッシュボードの「API keys」タブに移動
3. デフォルトのキーをコピー、または新しいキーを生成

API キーが有効になるまで最大2時間かかる場合がありますので、先にこの手順を済ませておくことをおすすめします。

## API 連携の実装 — OpenWeatherMap との通信

### API リクエストの基本構造

天気データを取得する関数は、以下のような形になります。Rork が自動生成するコードの中核部分です。

```typescript
// services/weatherApi.ts
const API_KEY = 'YOUR_API_KEY';
const BASE_URL = 'https://api.openweathermap.org/data/2.5';

// 都市名で現在の天気を取得
export const fetchWeatherByCity = async (city: string) => {
  const response = await fetch(
    `${BASE_URL}/weather?q=${city}&appid=${API_KEY}&units=metric&lang=ja`
  );
  if (!response.ok) throw new Error('天気データの取得に失敗しました');
  return response.json();
};

// 座標で5日間予報を取得
export const fetchForecast = async (lat: number, lon: number) => {
  const response = await fetch(
    `${BASE_URL}/forecast?lat=${lat}&lon=${lon}&appid=${API_KEY}&units=metric&lang=ja`
  );
  if (!response.ok) throw new Error('予報データの取得に失敗しました');
  return response.json();
};
```

このコードでは、`units=metric` で摂氏表示、`lang=ja` で日本語の天気説明を取得しています。

## 位置情報の取得 — expo-location の活用

### 現在地から天気を自動取得する

ユーザーの現在地を取得して天気を表示する機能は、天気アプリの要です。Rork は Expo を使ったアプリを生成するため、`expo-location` が利用できます。

```typescript
// hooks/useLocation.ts
import * as Location from 'expo-location';
import { useState, useEffect } from 'react';

export const useLocation = () => {
  const [location, setLocation] = useState<Location.LocationObject | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    (async () => {
      // 位置情報の使用許可を要求
      const { status } = await Location.requestForegroundPermissionsAsync();
      if (status !== 'granted') {
        setError('位置情報の使用が許可されていません');
        return;
      }
      // 現在地を取得
      const currentLocation = await Location.getCurrentPositionAsync({});
      setLocation(currentLocation);
    })();
  }, []);

  return { location, error };
};
```

### パーミッション処理のベストプラクティス

位置情報のパーミッションが拒否された場合にも、アプリが正常に動作するよう設計することが重要です。都市名での手動検索をフォールバックとして用意しておくと、ユーザー体験が向上します。

## 天気に応じた動的 UI の構築

### 背景色の動的変更

天気アプリでは、天気の状態に応じて UI を変化させることで、直感的なユーザー体験を実現できます。

```typescript
// utils/weatherTheme.ts
export const getWeatherTheme = (weatherMain: string) => {
  const themes: Record<string, { background: string[]; textColor: string }> = {
    Clear: {
      background: ['#4DA0B0', '#D39D38'], // 青空グラデーション
      textColor: '#FFFFFF',
    },
    Clouds: {
      background: ['#757F9A', '#D7DDE8'], // グレーグラデーション
      textColor: '#333333',
    },
    Rain: {
      background: ['#373B44', '#4286f4'], // 暗めの青
      textColor: '#FFFFFF',
    },
    Snow: {
      background: ['#E6DADA', '#274046'], // 雪色
      textColor: '#333333',
    },
  };
  return themes[weatherMain] || themes['Clear'];
};
```

Rork にプロンプトで「天気に応じて背景色を変更してください」と指示すれば、このような実装を自動生成してくれます。

## よくあるエラーと対処法

### API キーが認識されない

OpenWeatherMap の API キーは、発行後に有効化されるまで最大2時間かかることがあります。「401 Unauthorized」エラーが出た場合は、しばらく待ってから再試行してください。

### 位置情報が取得できない

シミュレーター環境では位置情報がデフォルト値（サンフランシスコ付近）になる場合があります。実機テストか、シミュレーターの位置設定を変更して確認しましょう。

### データの型エラー

API レスポンスのデータ構造が想定と異なる場合があります。TypeScript の型定義を適切に行い、オプショナルチェイニング（`?.`）を活用してエラーを防ぎましょう。

## FAQ — よくある質問

### Q. OpenWeatherMap API は無料で使えますか？

はい、無料プラン（Free tier）で1分あたり60回のリクエストが可能です。個人アプリであればこの制限で十分に運用できます。より多くのリクエストが必要な場合は有料プランへのアップグレードも検討できます。

### Q. Rork の無料プランでも天気アプリは作れますか？

作成可能です。Rork Free では週5回までプロジェクトの生成が可能なので、基本的な天気アプリは無料枠内で十分に開発できます。API 連携やデザインの微調整を繰り返す場合は、Rork Max プランの方が効率的です。

### Q. 天気データを日本語で表示するにはどうすればいいですか？

OpenWeatherMap API のリクエストパラメータに `lang=ja` を追加することで、天気の説明文が日本語で返されます。都市名は英語で送信し、表示用に日本語マッピングテーブルを別途用意するとさらに自然な表示になります。

### Q. このアプリを App Store に公開できますか？

もちろんです。Rork Max を利用していれば TestFlight でのベータテストから App Store 申請まで、Rork の管理画面から直接行えます。天気アプリは審査でリジェクトされにくいカテゴリなので、初めての公開にも向いています。
