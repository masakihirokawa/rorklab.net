## 音楽プレーヤーは作れたが、動画になると途端にハマる

Rork で音楽プレーヤーアプリを作ったことがある人は多いだろう。`expo-av` の `Audio.Sound` を使えば、再生・一時停止・シークくらいはすぐに動く。ところが「動画もやってみよう」と `Video` コンポーネントに切り替えた瞬間、予想外の壁にぶつかる。

全画面表示のときコントロールが消えない、プレイリストの次の動画に自動遷移しない、バックグラウンドに回すと再生が止まる——こういった問題は `expo-av` のドキュメントを読んだだけでは解決しにくい。なぜなら、動画プレーヤーの UX は「再生する」ことよりも「操作体験を設計する」ことに本質があるからです。

この記事では Rork を使って、カスタムコントロール付きの動画プレーヤーアプリをゼロから構築する。プレイリスト管理、自動連続再生、そして iOS のピクチャインピクチャ（PiP）対応まで、実際に動くコードで一つずつ実装していく。

## expo-av の Video コンポーネントを理解する

`expo-av` には `Audio.Sound`（音声）と `Video`（動画）の2つの再生 API があります。動画プレーヤーを作る場合、`Video` コンポーネントを使うが、ここで最初に理解すべき重要な違いがあります。

`Audio.Sound` はバックグラウンドで動作するように設計されているが、`Video` コンポーネントは画面上に表示されていることが前提です。つまり、動画の再生状態は画面のライフサイクルと密接に結びついています。

まずは基本の動画プレーヤーコンポーネントを作ろう。

```tsx
// components/VideoPlayer.tsx
import React, { useRef, useState, useCallback } from 'react';
import { View, StyleSheet, Dimensions, TouchableOpacity, Text } from 'react-native';
import { Video, ResizeMode, AVPlaybackStatus } from 'expo-av';

const { width: SCREEN_WIDTH } = Dimensions.get('window');
const VIDEO_HEIGHT = SCREEN_WIDTH * (9 / 16); // 16:9 アスペクト比

interface VideoPlayerProps {
  uri: string;
  title: string;
  onPlaybackFinish?: () => void;
}

export default function VideoPlayer({ uri, title, onPlaybackFinish }: VideoPlayerProps) {
  const videoRef = useRef<Video>(null);
  const [status, setStatus] = useState<AVPlaybackStatus | null>(null);

  const isPlaying = status?.isLoaded ? status.isPlaying : false;
  const position = status?.isLoaded ? status.positionMillis : 0;
  const duration = status?.isLoaded ? status.durationMillis ?? 0 : 0;

  const handlePlaybackStatusUpdate = useCallback((newStatus: AVPlaybackStatus) => {
    setStatus(newStatus);
    // 再生完了を検知して次の動画へ遷移するためのコールバック
    if (newStatus.isLoaded && newStatus.didJustFinish) {
      onPlaybackFinish?.();
    }
  }, [onPlaybackFinish]);

  const togglePlayPause = async () => {
    if (\!videoRef.current) return;
    if (isPlaying) {
      await videoRef.current.pauseAsync();
    } else {
      await videoRef.current.playAsync();
    }
  };

  const formatTime = (millis: number) => {
    const totalSeconds = Math.floor(millis / 1000);
    const minutes = Math.floor(totalSeconds / 60);
    const seconds = totalSeconds % 60;
    return `${minutes}:${seconds.toString().padStart(2, '0')}`;
  };

  return (
    <View style={styles.container}>
      <Video
        ref={videoRef}
        source={{ uri }}
        style={styles.video}
        resizeMode={ResizeMode.CONTAIN}
        onPlaybackStatusUpdate={handlePlaybackStatusUpdate}
        useNativeControls={false} // カスタムコントロールを使うため無効化
      />
      <TouchableOpacity style={styles.overlay} onPress={togglePlayPause}>
        <Text style={styles.playButton}>{isPlaying ? '⏸' : '▶'}</Text>
      </TouchableOpacity>
      <View style={styles.timeBar}>
        <Text style={styles.timeText}>{formatTime(position)}</Text>
        <View style={styles.progressTrack}>
          <View
            style={[
              styles.progressFill,
              { width: duration > 0 ? `${(position / duration) * 100}%` : '0%' },
            ]}
          />
        </View>
        <Text style={styles.timeText}>{formatTime(duration)}</Text>
      </View>
      <Text style={styles.title}>{title}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { width: SCREEN_WIDTH, backgroundColor: '#000' },
  video: { width: SCREEN_WIDTH, height: VIDEO_HEIGHT },
  overlay: {
    ...StyleSheet.absoluteFillObject,
    height: VIDEO_HEIGHT,
    justifyContent: 'center',
    alignItems: 'center',
  },
  playButton: { fontSize: 48, color: 'rgba(255,255,255,0.8)' },
  timeBar: {
    flexDirection: 'row',
    alignItems: 'center',
    paddingHorizontal: 12,
    paddingVertical: 8,
  },
  timeText: { color: '#aaa', fontSize: 12, width: 45 },
  progressTrack: {
    flex: 1,
    height: 3,
    backgroundColor: '#333',
    borderRadius: 1.5,
    marginHorizontal: 8,
  },
  progressFill: { height: 3, backgroundColor: '#e63946', borderRadius: 1.5 },
  title: { color: '#fff', fontSize: 14, paddingHorizontal: 12, paddingBottom: 8 },
});
```

ここでのポイントは `useNativeControls={false}` です。OS 標準のコントロールを使うと手軽だが、プレイリスト連携や PiP ボタンの配置など、後で自由がきかなくなります。最初からカスタムコントロールで作っておくと、あとの拡張がスムーズになります。

## シークバーをドラッグ可能にする

先ほどのプログレスバーは表示するだけで、ドラッグして任意の位置にジャンプできません。動画プレーヤーとしては致命的な欠点です。`PanResponder` を使ってドラッグ操作を実装しよう。

```tsx
// components/SeekBar.tsx
import React, { useRef } from 'react';
import { View, StyleSheet, PanResponder, GestureResponderEvent } from 'react-native';

interface SeekBarProps {
  progress: number; // 0〜1
  onSeek: (progress: number) => void;
  onSeekStart?: () => void;
  onSeekEnd?: () => void;
}

export default function SeekBar({ progress, onSeek, onSeekStart, onSeekEnd }: SeekBarProps) {
  const barRef = useRef<View>(null);

  const getProgressFromEvent = (evt: GestureResponderEvent) => {
    // タッチ位置からバー上の割合を計算
    return new Promise<number>((resolve) => {
      barRef.current?.measure((_x, _y, width, _height, pageX) => {
        const touchX = evt.nativeEvent.pageX - pageX;
        resolve(Math.max(0, Math.min(1, touchX / width)));
      });
    });
  };

  const panResponder = useRef(
    PanResponder.create({
      onStartShouldSetPanResponder: () => true,
      onPanResponderGrant: async (evt) => {
        onSeekStart?.();
        const p = await getProgressFromEvent(evt);
        onSeek(p);
      },
      onPanResponderMove: async (evt) => {
        const p = await getProgressFromEvent(evt);
        onSeek(p);
      },
      onPanResponderRelease: () => {
        onSeekEnd?.();
      },
    })
  ).current;

  return (
    <View ref={barRef} style={styles.track} {...panResponder.panHandlers}>
      <View style={[styles.fill, { width: `${progress * 100}%` }]} />
      <View style={[styles.thumb, { left: `${progress * 100}%` }]} />
    </View>
  );
}

const styles = StyleSheet.create({
  track: {
    flex: 1,
    height: 20, // タッチ領域を広めにとる（見た目のバーは3px）
    justifyContent: 'center',
    marginHorizontal: 8,
  },
  fill: {
    position: 'absolute',
    height: 3,
    backgroundColor: '#e63946',
    borderRadius: 1.5,
  },
  thumb: {
    position: 'absolute',
    width: 14,
    height: 14,
    borderRadius: 7,
    backgroundColor: '#e63946',
    marginLeft: -7,
    top: 3,
  },
});
```

実装上の注意点がひとつある。タッチ領域（`height: 20`）と見た目のバー（`height: 3`）を分けていることです。3px のバーだけだと指で正確にタッチするのが難しく、ユーザーが「反応しない」と感じる原因になります。YouTube アプリも同じ手法で、見た目は細いが実際のタッチ領域は広い。

## プレイリスト管理と自動連続再生

動画プレーヤーの価値は「1本再生できる」ことではなく「連続して快適に視聴できる」ことにある。プレイリストを管理するカスタム Hook を作ろう。

```tsx
// hooks/usePlaylist.ts
import { useState, useCallback } from 'react';

export interface VideoItem {
  id: string;
  uri: string;
  title: string;
  duration?: number; // 秒
  thumbnail?: string;
}

export function usePlaylist(initialItems: VideoItem[]) {
  const [items, setItems] = useState<VideoItem[]>(initialItems);
  const [currentIndex, setCurrentIndex] = useState(0);
  const [isShuffled, setIsShuffled] = useState(false);
  const [repeatMode, setRepeatMode] = useState<'none' | 'all' | 'one'>('none');

  const currentItem = items[currentIndex] ?? null;

  const playNext = useCallback(() => {
    setCurrentIndex((prev) => {
      if (repeatMode === 'one') return prev; // 同じ動画をリピート
      const next = prev + 1;
      if (next >= items.length) {
        return repeatMode === 'all' ? 0 : prev; // 全体リピートか停止
      }
      return next;
    });
  }, [items.length, repeatMode]);

  const playPrevious = useCallback(() => {
    setCurrentIndex((prev) => {
      const prevIdx = prev - 1;
      if (prevIdx < 0) {
        return repeatMode === 'all' ? items.length - 1 : 0;
      }
      return prevIdx;
    });
  }, [items.length, repeatMode]);

  const jumpTo = useCallback((index: number) => {
    if (index >= 0 && index < items.length) {
      setCurrentIndex(index);
    }
  }, [items.length]);

  const toggleShuffle = useCallback(() => {
    setIsShuffled((prev) => {
      if (\!prev) {
        // シャッフル有効化：現在の曲は固定し、残りをランダム並び替え
        const current = items[currentIndex];
        const rest = items.filter((_, i) => i \!== currentIndex);
        const shuffled = rest.sort(() => Math.random() - 0.5);
        setItems([current, ...shuffled]);
        setCurrentIndex(0);
      }
      return \!prev;
    });
  }, [items, currentIndex]);

  const cycleRepeatMode = useCallback(() => {
    setRepeatMode((prev) => {
      const modes: Array<'none' | 'all' | 'one'> = ['none', 'all', 'one'];
      const nextIdx = (modes.indexOf(prev) + 1) % modes.length;
      return modes[nextIdx];
    });
  }, []);

  return {
    items,
    currentItem,
    currentIndex,
    isShuffled,
    repeatMode,
    playNext,
    playPrevious,
    jumpTo,
    toggleShuffle,
    cycleRepeatMode,
  };
}
```

`playNext` のロジックで重要なのは、`repeatMode` に応じた挙動の分岐です。「なし」「全体リピート」「1曲リピート」の3モードはユーザーが直感的に期待するパターンで、これを最初から組み込んでおくと後から追加するより遥かにシンプルになります。

シャッフルの実装では、現在再生中の動画を先頭に固定してから残りを並び替えています。こうしないと、シャッフルボタンを押した瞬間に別の動画にジャンプしてしまい、ユーザーが混乱する。Spotify でも同じアプローチが使われています。

## プレイリスト画面の UI を組み立てる

プレイリストの一覧表示は `FlatList` で効率よく描画する。動画のサムネイル、タイトル、再生時間を表示し、タップで即座に再生を切り替える。

```tsx
// components/PlaylistView.tsx
import React from 'react';
import { View, Text, FlatList, TouchableOpacity, Image, StyleSheet } from 'react-native';
import { VideoItem } from '../hooks/usePlaylist';

interface PlaylistViewProps {
  items: VideoItem[];
  currentIndex: number;
  onSelect: (index: number) => void;
}

export default function PlaylistView({ items, currentIndex, onSelect }: PlaylistViewProps) {
  const formatDuration = (seconds?: number) => {
    if (\!seconds) return '--:--';
    const m = Math.floor(seconds / 60);
    const s = seconds % 60;
    return `${m}:${s.toString().padStart(2, '0')}`;
  };

  const renderItem = ({ item, index }: { item: VideoItem; index: number }) => {
    const isActive = index === currentIndex;
    return (
      <TouchableOpacity
        style={[styles.item, isActive && styles.activeItem]}
        onPress={() => onSelect(index)}
      >
        <View style={styles.thumbnailContainer}>
          {item.thumbnail ? (
            <Image source={{ uri: item.thumbnail }} style={styles.thumbnail} />
          ) : (
            <View style={[styles.thumbnail, styles.placeholderThumb]}>
              <Text style={styles.placeholderIcon}>🎬</Text>
            </View>
          )}
          <Text style={styles.durationBadge}>{formatDuration(item.duration)}</Text>
        </View>
        <View style={styles.info}>
          <Text style={[styles.title, isActive && styles.activeTitle]} numberOfLines={2}>
            {item.title}
          </Text>
          {isActive && <Text style={styles.nowPlaying}>再生中</Text>}
        </View>
      </TouchableOpacity>
    );
  };

  return (
    <FlatList
      data={items}
      keyExtractor={(item) => item.id}
      renderItem={renderItem}
      showsVerticalScrollIndicator={false}
    />
  );
}

const styles = StyleSheet.create({
  item: {
    flexDirection: 'row',
    padding: 12,
    borderBottomWidth: StyleSheet.hairlineWidth,
    borderBottomColor: '#222',
  },
  activeItem: { backgroundColor: '#1a1a2e' },
  thumbnailContainer: { position: 'relative' },
  thumbnail: { width: 120, height: 68, borderRadius: 6, backgroundColor: '#333' },
  placeholderThumb: { justifyContent: 'center', alignItems: 'center' },
  placeholderIcon: { fontSize: 24 },
  durationBadge: {
    position: 'absolute',
    bottom: 4,
    right: 4,
    backgroundColor: 'rgba(0,0,0,0.75)',
    color: '#fff',
    fontSize: 10,
    paddingHorizontal: 4,
    paddingVertical: 1,
    borderRadius: 2,
    overflow: 'hidden',
  },
  info: { flex: 1, marginLeft: 12, justifyContent: 'center' },
  title: { color: '#eee', fontSize: 14, lineHeight: 20 },
  activeTitle: { color: '#e63946', fontWeight: '600' },
  nowPlaying: { color: '#e63946', fontSize: 11, marginTop: 4 },
});
```

`FlatList` を使う理由は、プレイリストが数十〜数百件になってもスクロールパフォーマンスが劣化しないからです。`ScrollView` + `map` でレンダリングすると、50件を超えたあたりからフレーム落ちが目立ち始める。

## ピクチャインピクチャ（PiP）に対応する

iOS ではピクチャインピクチャ（PiP）を使うと、アプリを離れても動画を小さなウィンドウで視聴し続けられます。この機能は `expo-av` の `Video` コンポーネントのプロパティとして標準サポートされています。

ただし、PiP を有効にするにはいくつかの前提条件があります。

```tsx
// PiP 対応の Video コンポーネント設定
import { Video, ResizeMode, Audio } from 'expo-av';

// アプリ起動時にオーディオセッションを設定する（App.tsx等で1回だけ実行）
async function configureAudioSession() {
  await Audio.setAudioModeAsync({
    allowsRecordingIOS: false,
    staysActiveInBackground: true, // バックグラウンド再生を許可
    playsInSilentModeIOS: true,    // サイレントモードでも音声を出す
    shouldDuckAndroid: true,
    playThroughEarpieceAndroid: false,
  });
}

// Video コンポーネントに PiP 関連のプロパティを追加
<Video
  ref={videoRef}
  source={{ uri: currentItem.uri }}
  style={styles.video}
  resizeMode={ResizeMode.CONTAIN}
  onPlaybackStatusUpdate={handlePlaybackStatusUpdate}
  useNativeControls={false}
  // PiP対応の設定
  usePoster={true}
  posterSource={{ uri: currentItem.thumbnail }}
  posterStyle={{ resizeMode: 'cover' }}
/>
```

`staysActiveInBackground: true` は必須です。これがないと、ホーム画面に戻った瞬間にオーディオトラックも止まり、PiP ウィンドウが数秒で消えてしまう。

実際の PiP 起動には、Expo の managed workflow では制約があります。`expo-av` 単体で完全な PiP 制御を行うには `expo-dev-client` でカスタムネイティブモジュールを追加するか、Rork Max を使って SwiftUI の `AVPlayerViewController` を直接操作する方法があります。

managed workflow でできる範囲では、ネイティブコントロールを使う場合（`useNativeControls={true}`）に PiP ボタンが自動表示されます。カスタムコントロールを使いつつ PiP も利用したい場合は、以下のように切り替えボタンを作ります。

```tsx
// PiP モード切替の実装例
const [useNative, setUseNative] = useState(false);

const enterPiPMode = () => {
  // ネイティブコントロールに一時的に切り替えてPiPボタンを露出
  setUseNative(true);
  // iOSの場合、ネイティブコントロール表示後にPiPアイコンが左上に出る
};

// ボタン
<TouchableOpacity onPress={enterPiPMode} style={styles.pipButton}>
  <Text style={styles.pipIcon}>📌</Text>
  <Text style={styles.pipLabel}>PiP</Text>
</TouchableOpacity>
```

この方法は完璧ではないが、managed workflow の範囲内で最も実用的なアプローチです。本格的な PiP 実装が必要なプロダクションアプリでは、[Rork で音楽プレイヤーアプリを作る](/articles/rork-dev/rork-audio-player-app-tutorial) で紹介したバックグラウンド再生の仕組みと組み合わせることで、より安定した動作が期待できます。

## 全画面表示の切り替え

動画アプリで全画面表示は必須機能です。ステータスバーの非表示、画面回転のロック解除、そして戻るときの復元——これらを一括で管理する Hook を作ります。

```tsx
// hooks/useFullscreen.ts
import { useState, useCallback } from 'react';
import { StatusBar, Dimensions } from 'react-native';
import * as ScreenOrientation from 'expo-screen-orientation';

export function useFullscreen() {
  const [isFullscreen, setIsFullscreen] = useState(false);

  const enterFullscreen = useCallback(async () => {
    StatusBar.setHidden(true, 'fade');
    await ScreenOrientation.unlockAsync(); // 画面回転を許可
    setIsFullscreen(true);
  }, []);

  const exitFullscreen = useCallback(async () => {
    StatusBar.setHidden(false, 'fade');
    await ScreenOrientation.lockAsync(
      ScreenOrientation.OrientationLock.PORTRAIT_UP
    );
    setIsFullscreen(false);
  }, []);

  const toggleFullscreen = useCallback(async () => {
    if (isFullscreen) {
      await exitFullscreen();
    } else {
      await enterFullscreen();
    }
  }, [isFullscreen, enterFullscreen, exitFullscreen]);

  const dimensions = isFullscreen
    ? {
        width: Dimensions.get('screen').width,
        height: Dimensions.get('screen').height,
      }
    : {
        width: Dimensions.get('window').width,
        height: Dimensions.get('window').width * (9 / 16),
      };

  return { isFullscreen, toggleFullscreen, enterFullscreen, exitFullscreen, dimensions };
}
```

`Dimensions.get('screen')` と `Dimensions.get('window')` の違いに注意してほしい。`screen` はステータスバーを含む画面全体のサイズ、`window` はアプリが使える領域のサイズを返す。全画面時には `screen` を使わないと、ステータスバー分の隙間が残ってしまう。

`expo-screen-orientation` を使う場合は `npx expo install expo-screen-orientation` で追加する。Rork なら依存関係の追加はプロンプトで「expo-screen-orientation を追加して」と指示するだけでいい。

## つまずきやすいポイント3選

ここまでの実装で、動画プレーヤーアプリとしてはかなり機能が揃った。ただし、実際に動かすと以下の問題に遭遇しやすいので、事前に知っておくと時間を節約できます。

**1. Android で動画が真っ黒になる**

Android の一部端末で `resizeMode={ResizeMode.CONTAIN}` を使うと、動画は再生されているのに画面が真っ黒のまま音声だけ聞こえることがあります。これは `expo-av` のハードウェアアクセラレーション周りの問題で、以下のワークアラウンドが有効です。

```tsx
// Android で黒画面になる場合の対処
<Video
  // ...他のプロパティ
  style={[styles.video, { opacity: 0.99 }]} // レンダリングレイヤーの再構築を強制
/>
```

`opacity: 0.99` は意味がないように見えるが、React Native のレンダリングエンジンが別のコードパスを通るため、ハードウェアアクセラレーションの問題を回避できます。

**2. 画面遷移時に音声が残る**

React Navigation で画面を離れたのに動画の音声が鳴り続ける問題は、`useEffect` のクリーンアップで解決します。

```tsx
import { useIsFocused } from '@react-navigation/native';

const isFocused = useIsFocused();

useEffect(() => {
  if (\!isFocused && videoRef.current) {
    videoRef.current.pauseAsync();
  }
}, [isFocused]);
```

**3. 長い動画でメモリ不足になる**

30分以上の動画を再生するとメモリ使用量が膨らむ。`Video` コンポーネントの `progressUpdateIntervalMillis` を 1000（1秒ごと）に設定してステータス更新頻度を下げると、パフォーマンスが改善します。

```tsx
<Video
  progressUpdateIntervalMillis={1000} // デフォルトの500msから変更
  // ...
/>
```

## 次のステップ — ストリーミング対応とオフライン再生

ここまでで、カスタムコントロール・プレイリスト・PiP・全画面対応の動画プレーヤーが完成しました。ローカルファイルや直接URLの動画なら十分に実用的です。

実際のプロダクションアプリに発展させるには、HLS（HTTP Live Streaming）への対応が次の課題になります。`expo-av` は HLS を標準サポートしており、`source={{ uri: 'https://example.com/video.m3u8' }}` とするだけで再生できます。ただし、画質の自動切り替え（Adaptive Bitrate）の制御にはネイティブコードが必要になるので、そこが Rork Max の出番です。

オフライン再生を実装するなら、`expo-file-system` でダウンロードした動画ファイルを [ローカルストレージ管理](/articles/rork-dev/rork-local-storage-asyncstorage-mmkv-sqlite-guide) と組み合わせて管理する設計が王道です。ダウンロード進捗の表示、ストレージ容量のチェック、期限切れコンテンツの自動削除まで考えると、1本の記事では収まらないテーマなので、別の機会に深掘りします。

動画プレーヤーは一見シンプルだが、作り込むほど細かな UX の積み重ねが求められるアプリです。まずはこの記事のコードをベースに動くものを作り、そこから「自分のアプリに必要な機能」を選んで追加していくのが最短ルートだと私は考える。
