## 今週のコードスニペット

収益化ガイドで取り上げたAdMobバナー広告のRork実装例です。Expo経由でAdMobを組み込む最小構成をご紹介します。

```typescript
import { BannerAd, BannerAdSize, TestIds } from "react-native-google-mobile-ads";

const adUnitId = __DEV__
  ? TestIds.ADAPTIVE_BANNER
  : "ca-app-pub-YOUR_ADMOB_ID/YOUR_AD_UNIT_ID";

export function AdBanner() {
  return (
    <BannerAd
      unitId={adUnitId}
      size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
      requestOptions={{
        requestNonPersonalizedAdsOnly: true,
      }}
    />
  );
}
```

開発中は `TestIds.ADAPTIVE_BANNER` を使うことで、本番のAdMobアカウントに影響を与えずに表示確認できます。`requestNonPersonalizedAdsOnly: true` はATT（App Tracking Transparency）の許可を得ていないユーザーへの安全な配信設定です。

## よくあるご質問

**Q: Rorkで作ったアプリにAdMobを組み込むのは難しいですか？**

A: Expo Go環境ではAdMobは動作しないため、開発ビルド（expo-dev-client）の準備が必要です。一度セットアップすれば、実装自体はコンポーネントを配置するだけと非常にシンプルです。

**Q: Rork MaxとRork（無料版）の違いは何ですか？**

A: Rork Maxは複雑なUI・ネイティブモジュール連携・高度なデバッグ機能など、商用レベルのアプリ開発に必要な機能を拡張したプランです。個人開発者が収益化を目指す段階では、Rork Maxへの移行を検討する価値があります。
