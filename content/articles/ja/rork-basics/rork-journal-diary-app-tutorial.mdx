## はじめに：なぜ今、ジャーナリングアプリなのか

「毎日の振り返りを記録したい」「気分の変化を可視化したい」——ジャーナリング（日記・記録習慣）への関心は年々高まっています。Day One、Journey、Notionといったアプリが数百万のユーザーに使われていることからも分かるように、日記アプリは根強い需要を持つジャンルです。

しかし、既存のアプリには「機能が多すぎる」「自分の使い方に合わない」「サブスクリプションが高い」といった不満を感じている人も少なくありません。だからこそ、**自分だけのジャーナリングアプリ**を作ることには大きな価値があります。

このチュートリアルでは、**Rork**を使ってAIとの対話だけで日記アプリをゼロから作る方法を解説します。プログラミング経験がなくても、読み終える頃にはiOS・Android両対応のジャーナリングアプリが完成しているはずです。

このチュートリアルで学べること：

- Rorkでジャーナリングアプリの基本構造を設計する方法
- テキスト入力・気分トラッキング機能の実装
- カレンダービューで過去の記録を振り返る仕組み
- AsyncStorageを使ったローカルデータ保存
- AIを活用した振り返りプロンプトの自動生成

## 前提知識・環境準備

### 必要なもの

- Rorkアカウント（無料プランでOK）
- スマートフォン（iOSまたはAndroid）またはExpo Goアプリ
- 日記アプリの大まかなイメージ（どんな情報を記録したいか）

### Rorkのプランについて

基本的な日記アプリの作成は**無料プラン**（週5回のAI生成）で十分です。画像添付やクラウド同期などの高度な機能を追加したい場合は、**Rork Max**（$200/月）を検討してください。Rork Maxでは**ネイティブSwift**によるiOS専用アプリの開発も可能です。

### Rorkの基本操作

Rorkに初めて触れる方は、まず[Rork入門ガイド](/articles/rork-basics/getting-started)をお読みください。プロジェクトの作成方法やAIとの対話方法が分かります。

## アプリの全体設計

ジャーナリングアプリを作る前に、どんな画面が必要かを整理しましょう。今回作るアプリの構成は以下の通りです。

### 画面構成

1. **ホーム画面（Today）** — 今日の日記を書く画面。テキスト入力と気分選択
2. **カレンダー画面** — 月単位で過去のエントリを一覧表示
3. **エントリ詳細画面** — 過去の日記を読み返す
4. **統計画面** — 気分の推移グラフと継続日数の表示

### データモデル

各日記エントリには以下の情報を持たせます。

```typescript
// 日記エントリの型定義
interface JournalEntry {
  id: string;          // ユニークID（日付ベース）
  date: string;        // ISO 8601形式の日付
  content: string;     // 日記本文
  mood: number;        // 気分スコア（1〜5）
  tags: string[];      // カスタムタグ
  createdAt: string;   // 作成日時
  updatedAt: string;   // 更新日時
}
// 期待する出力例:
// { id: "2026-03-22", date: "2026-03-22", content: "今日は...", mood: 4, tags: ["仕事", "運動"], ... }
```

## ステップ1：プロジェクトの作成とプロンプト設計

Rorkで新しいプロジェクトを作成し、以下のようなプロンプトを入力します。

### 初期プロンプトの例

```
日記アプリを作ってください。以下の機能を含めます：

1. ホーム画面：今日の日記を書くエリア（テキスト入力）と気分を5段階で選べるアイコン
2. カレンダー画面：月表示のカレンダーで、日記を書いた日にドットマーカーを表示
3. エントリ詳細画面：タップで過去の日記を全文表示
4. 統計画面：過去30日間の気分推移を折れ線グラフで表示

デザインは柔らかく温かみのある配色（クリーム系の背景＋ブラウン系のアクセント）で、
毎日開きたくなるような優しい雰囲気にしてください。
データはAsyncStorageでローカルに保存してください。
```

このプロンプトだけで、Rorkは4画面構成のアプリを自動生成します。生成結果を確認しながら、細かい調整を進めていきましょう。

### プロンプト設計のコツ

Rorkに伝えるプロンプトは「具体的であるほど良い」のが鉄則です。色の指定、画面数、使用するライブラリ名を含めることで、AIの出力精度が格段に上がります。プロンプト設計について詳しくは[Rorkプロンプトエンジニアリングのコツ](/articles/rork-basics/rork-prompt-engineering-app-design-tips)を参考にしてください。

## ステップ2：気分トラッキング機能の実装

日記アプリの差別化ポイントとなるのが、気分トラッキング機能です。ユーザーがその日の気分を5段階で記録できるUIを作りましょう。

### 気分選択コンポーネント

Rorkに以下のようなプロンプトを追加します。

```
気分選択UIを改善してください：
- 5つの絵文字アイコン（😢 😕 😐 😊 😄）を横並びに表示
- 選択中のアイコンは拡大＋背景色付き
- 選択するとハプティックフィードバック（振動）を発生させる
- 気分を選ばないと日記を保存できないバリデーションを追加
```

生成されるコードの構造は以下のようになります。

```typescript
// 気分選択コンポーネントの概要
const moodOptions = [
  { value: 1, emoji: '😢', label: 'とても悪い' },
  { value: 2, emoji: '😕', label: '悪い' },
  { value: 3, emoji: '😐', label: 'ふつう' },
  { value: 4, emoji: '😊', label: '良い' },
  { value: 5, emoji: '😄', label: 'とても良い' },
];

// タップ時の処理
const handleMoodSelect = (mood: number) => {
  setSelectedMood(mood);
  Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
};
// 期待する動作: アイコンタップで気分値がセットされ、端末が軽く振動する
```

## ステップ3：カレンダービューの作成

過去のエントリを振り返るために、カレンダービューを作りましょう。React Nativeでは`react-native-calendars`ライブラリが定番です。

### カレンダーのプロンプト例

```
カレンダー画面を追加してください：
- react-native-calendarsを使用
- 日記を書いた日には小さなドットマーカー（色は気分スコアに応じて変化）を表示
- 日付をタップするとその日のエントリ詳細画面に遷移
- 気分スコア1-2は青、3は黄、4-5はオレンジのドットにしてください
```

カレンダーにドットマーカーを表示するためのデータ変換は以下のような仕組みです。

```typescript
// エントリデータからカレンダーのマーカーを生成
const generateMarkedDates = (entries: JournalEntry[]) => {
  const marked: Record<string, any> = {};
  entries.forEach(entry => {
    const color = entry.mood <= 2 ? '#6B9BD2'
                : entry.mood === 3 ? '#E8C547'
                : '#E8945A';
    marked[entry.date] = {
      marked: true,
      dotColor: color,
    };
  });
  return marked;
};
// 期待する出力: { "2026-03-20": { marked: true, dotColor: "#E8945A" }, ... }
```

## ステップ4：データの永続化（AsyncStorage）

日記データはデバイス内に安全に保存する必要があります。React NativeではAsyncStorageが最もシンプルな選択肢です。

### データ保存・読み込みの実装

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

// 日記エントリを保存する関数
const saveEntry = async (entry: JournalEntry) => {
  const key = `journal_${entry.date}`;
  await AsyncStorage.setItem(key, JSON.stringify(entry));

  // インデックスも更新（全エントリの日付リスト）
  const indexRaw = await AsyncStorage.getItem('journal_index');
  const index: string[] = indexRaw ? JSON.parse(indexRaw) : [];
  if (!index.includes(entry.date)) {
    index.push(entry.date);
    await AsyncStorage.setItem('journal_index', JSON.stringify(index));
  }
};

// 特定の日のエントリを取得する関数
const getEntry = async (date: string): Promise<JournalEntry | null> => {
  const key = `journal_${date}`;
  const raw = await AsyncStorage.getItem(key);
  return raw ? JSON.parse(raw) : null;
};
// 期待する動作: saveEntry()で保存後、getEntry("2026-03-22")で同じエントリが返る
```

AsyncStorageはキーバリュー形式のため、エントリごとに`journal_2026-03-22`のようなキーで保存し、別途インデックスキーで日付リストを管理するのがポイントです。

より高度なデータ管理が必要な場合は、[Rorkオフラインサポートガイド](/articles/rork-dev/rork-offline-support)も参考にしてください。

## ステップ5：統計画面で気分の推移を可視化

記録を続けるモチベーションを維持するために、統計画面を追加しましょう。

### 統計表示のプロンプト例

```
統計画面を追加してください：
- 過去30日間の気分スコアを折れ線グラフで表示（react-native-chart-kitを使用）
- 平均気分スコアを大きな数字で表示
- 連続記録日数（ストリーク）をカウントして表示
- 最もよく使ったタグをTop3で表示
```

連続記録日数の計算ロジックは以下のようになります。

```typescript
// 連続記録日数（ストリーク）を計算する関数
const calculateStreak = (entries: JournalEntry[]): number => {
  if (entries.length === 0) return 0;

  const sortedDates = entries
    .map(e => e.date)
    .sort()
    .reverse(); // 新しい日付順

  let streak = 0;
  const today = new Date();

  for (let i = 0; i < sortedDates.length; i++) {
    const expectedDate = new Date(today);
    expectedDate.setDate(today.getDate() - i);
    const expected = expectedDate.toISOString().split('T')[0];

    if (sortedDates[i] === expected) {
      streak++;
    } else {
      break;
    }
  }
  return streak;
};
// 期待する出力例: 3日連続で記録していれば 3 が返る
```

## よくあるエラーと対処法

### AsyncStorageの容量制限

AsyncStorageにはプラットフォームごとに容量制限があります（Androidではデフォルト6MB）。毎日500文字の日記を書いた場合、約10年分は保存できる計算ですが、画像を保存する場合はファイルシステム（expo-file-system）の利用を検討してください。

### カレンダーライブラリのタイムゾーン問題

`react-native-calendars`で日付がずれる場合は、日付の生成時にタイムゾーンオフセットを考慮する必要があります。`date.toISOString().split('T')[0]`ではなく、ローカルタイムで日付文字列を生成しましょう。

```typescript
// タイムゾーンを考慮した日付文字列の生成
const getLocalDateString = (date: Date): string => {
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');
  const day = String(date.getDate()).padStart(2, '0');
  return `${year}-${month}-${day}`;
};
// 期待する出力: "2026-03-22"（ローカルタイムゾーン基準）
```

### Rorkの生成結果が意図と異なる場合

AIが期待通りのUIを生成しない場合は、プロンプトをより具体的にしましょう。「カレンダーを表示」ではなく「react-native-calendarsのCalendarコンポーネントを使い、markedDatesプロパティでドットを表示」のように、ライブラリ名とプロパティ名まで指定すると精度が上がります。

### AIで振り返りプロンプトを自動生成する

毎日「何を書けばいいか分からない」という問題を解決するために、AIが書き出しのヒントを提案する機能を追加できます。

```
以下の機能を追加してください：
```

複数デバイスで日記を同期したい場合は、Supabaseをバックエンドに使うのがおすすめです。認証・データベース・リアルタイム同期がすべてそろっています。詳しくは[Rork × Supabaseチャットアプリ構築ガイド](/articles/rork-dev/rork-supabase-chat-app)のバックエンド設定を参考にしてください。

## FAQ

### Q: プログラミング経験がなくても日記アプリを作れますか？

はい、Rorkを使えばコードを書く必要はありません。AIにアプリの要件を自然言語で伝えるだけで、React Nativeのコードが自動生成されます。このチュートリアルで紹介したコード例は仕組みの理解を助けるためのもので、実際にはRorkのAIが自動的に記述します。

### Q: 無料プランで日記アプリは完成できますか？

基本的な機能（テキスト入力・気分トラッキング・カレンダー表示・ローカル保存）は無料プラン内で作成できます。ただし、AIの生成回数に制限があるため、細かい修正を繰り返す場合はJuniorプラン以上が快適です。

### Q: 日記データのプライバシーは大丈夫ですか？

AsyncStorageを使う場合、データはデバイス内にのみ保存されるため、サーバーに送信されることはありません。ただしデバイスを紛失した場合のリスクを考慮して、将来的にはパスコードロック機能の追加を検討しましょう。

### Q: App StoreやGoogle Playに公開できますか？

はい、Rorkで作ったアプリはそのまま各ストアに提出できます。Rork Maxなら2クリックでApp Storeへの公開が可能です。公開手順の詳細は[App Store公開ガイド](/articles/rork-dev/app-store-publishing)をご覧ください。

### Q: 他の日記アプリ（Day Oneなど）と差別化するには？

特定のニッチに絞ることが効果的です。例えば「感謝日記」「育児日記」「ダイエット記録」「読書ログ」など、テーマを限定することで明確なターゲットユーザーが生まれます。Rorkなら追加機能の実装も対話だけで完了するので、アイデアをすぐに形にできます。

## まとめ

Rorkを使えば、プログラミング経験がなくても本格的なジャーナリングアプリを作ることができます。今回のチュートリアルでは、テキスト入力・気分トラッキング・カレンダービュー・統計表示という4つの主要機能を持つアプリの作り方を解説しましました。

日記アプリは「毎日使ってもらえる」という点で、個人開発のテーマとして非常に優れています。継続率の高いアプリはストアでの評価も上がりやすく、サブスクリプションモデルとの相性も良いジャンルです。

まずはRorkで基本機能を作り、実際に自分で使いながら改善を重ねていくのがおすすめです。自分が欲しい機能を追加していくうちに、他のユーザーにも刺さるアプリが出来上がっていくはずです。
