# Rorkで初めてのTodoアプリを作る：ステップバイステップチュートリアル

モバイルアプリ開発は難しいと思っていませんか？**Rork**を使えば、コーディング経験がなくても数分で完全に機能するTodoアプリが作成できます。このチュートリアルでは、プロセスの各ステップを詳しく説明します。

## このチュートリアルで作るもの

このチュートリアルを完了すると、以下の機能を持つTodoアプリケーションが完成します：
- タスクの新規作成
- タスク完了のマーク
- タスク削除
- セッション間でのデータ保持
- モバイルプラットフォームへのデプロイ

## 前提条件

はじめる前に、以下を準備してください：
- Rork Maxのインストール
- テキストエディタまたはIDE
- モバイルアプリの基本的な理解
- 30～60分の時間

## ステップ1：Rorkプロジェクトの初期化

まず、新しいRorkプロジェクトを作成しましょう。ターミナルを開いて実行してください：

```bash
rork create todo-app
cd todo-app
```

Rorkはすべての必要なファイルを含む新しいプロジェクトを自動生成します。プロジェクト構造は以下のようになります：
- `/src` - アプリケーションコードの場所
- `rork.config.js` - プロジェクト設定
- `package.json` - 依存関係

次に、必要な依存関係をインストールします：

```bash
npm install
```

React Native、Rorkコンポーネント、その他のライブラリがインストールされます。

## ステップ2：Rorkプロンプトを使ってUIを設計する

Rorkの最大の特徴は、自然言語プロンプトでUIコンポーネントを生成できることです。Rorkに望むものを正確に伝えることで、美しいTodoアプリインターフェースを作成しましょう。

Rork UIビルダーを開き、このプロンプトを入力してください：

```
以下の特徴を持つ洗練されたTodoアプリインターフェースを作成してください：
- 「マイタスク」と表示するヘッダー
- 新しいタスクを追加するための入力フィールド
- 「タスクを追加」というラベルの送信ボタン
- すべてのタスクを表示するリスト
- 各タスクに完了チェックボックス
- 各タスクに削除ボタン（ゴミ箱アイコン）
- 水色のカラースキーム、角丸で統一
- 現代的な見た目のため、適切なパディングと間隔を設定
```

Rorkが自動的に美しく、レスポンシブなUIコンポーネントを生成します。すぐにプレビューでき、フォローアッププロンプトで調整することも可能です：

```
タスク項目に細かい影を追加して、タスクが追加されるときなめらかなアニメーションをつけてください
```

生成されたコンポーネントは`/src/components`ディレクトリに配置されます。

## ステップ3：CRUD機能を実装する

それでは、アプリを機能させましょう。4つの基本的な操作を実装します：

### Create（タスク追加）
まず、タスク管理サービスを作成するようRorkに指示してください：

```
以下の機能を持つタスク管理サービスを作成してください：
- ユニークなIDを持つ配列にタスクを保存する
- 各タスクは以下を含む：ID、タイトル、完了ステータス、タイムスタンプ
- タスク追加、取得、更新、削除の関数を実装する
- AsyncStorageでデータを永続化する
- 以下の関数をエクスポート：addTask、getAllTasks、updateTask、deleteTask
```

### Read（タスク表示）
Rorkがタスクを表示するコンポーネントを生成します。このプロンプトを使用してください：

```
以下の機能を持つタスクリストコンポーネントを作成してください：
- マウント時にすべてのタスクを取得する
- タスクを順序付けして表示する（未完了が先、その後完了したタスク）
- 残りのタスク数を表示する
- タスクが更新されたときにリフレッシュする
- 空状態のときはフレンドリーなメッセージを表示する
```

### Update（完了マーク）
タスク完了機能を実装してください：

```
以下の機能を持つ関数を作成してください：
- タスクの完了ステータスを切り替える
- 視覚的に表現する（完了したテキストに打ち消し線）
- AsyncStorageのデータを更新する
- トランジションをなめらかにアニメーションさせる
```

### Delete（タスク削除）
最後に、タスク削除機能を追加してください：

```
以下の機能を持つ削除関数を作成してください：
- IDでタスクを削除する
- 削除前に確認ダイアログを表示する
- タスクリストをすぐに更新する
- AsyncStorageからタスクを削除する
```

## ステップ4：アプリケーションにスタイルを適用する

Todoアプリを視覚的に魅力的にしましょう。Rorkのスタイリングプロンプトを使用します：

```
Todoアプリに専門的なスタイルを適用してください：
- 薄い青から白へのグラデーション背景
- 完了したタスクは薄い灰色で表現する
- タスク追加用のフローティングアクションボタンを追加
- 明確なタイポグラフィの階層構造を使用
- ボタンはホバーエフェクトとスムーズなトランジション効果を持つ
- すべてがレスポンシブで、異なる画面サイズに対応する
```

フィードバックを提供して色とスタイルを調整できます：

```
プライマリカラーの青をティール色に変更して、タスクボックスを少し大きくしてください
```

## ステップ5：アプリケーションをテストする

デプロイ前に、すべてが正しく動作することを確認しましょう。以下を実行してください：

```bash
npm run dev
```

開発サーバーが起動します。各機能をテストしてください：

1. **タスク追加** - 何か入力して「タスクを追加」を押す
2. **タスク完了** - タスクの横にあるチェックボックスをクリック
3. **タスク削除** - 削除ボタンをクリック
4. **アプリをリロード** - タスクが保持されているか確認（AsyncStorageのおかげ）
5. **空状態を試す** - すべてのタスクを削除してUIを確認

Rorkのテスト機能を使用して：

```
Todoアプリの単体テストを作成してください。以下を検証してください：
- タスク追加がタスク数を増やす
- タスク完了がそのステータスを更新する
- タスク削除がリストから削除する
- アプリ再起動後もタスクが保持される
```

## ステップ6：アプリをデプロイする

Todoアプリに満足したら、デプロイしましょう。Rorkは複数のプラットフォームに対応しています。

### iOSの場合：
```bash
rork build ios
rork deploy ios
```

### Androidの場合：
```bash
rork build android
rork deploy android
```

対象プラットフォームについてRorkに相談してください：

```
このTodoアプリをiOS TestFlightにデプロイして、ベータテスターと共有するプロセスを説明してください
```

Rorkがプラットフォーム固有の指示を提供し、複雑なビルド設定を自動的に処理します。

## 成功のコツ

- **シンプルに始める** - 一度にすべての機能を追加するのではなく、段階的に追加する
- **詳細なプロンプトを使う** - Rorkに提供する情報が詳しいほど、結果がよくなります
- **頻繁にテストする** - 最後までテストを待たずに、こまめにテストしましょう
- **質問する** - Rorkはバグのデバッグとパフォーマンス最適化を手伝えます
- **反復する** - フィードバックを使ってアプリの設計と機能を改善してください
