## はじめに — 語学学習アプリを自分で作る時代

語学学習アプリは、世界中で数十億ドル規模の市場を形成しています。Duolingo や Anki のようなアプリを使ったことがある方も多いのではないでしょうか。しかし、既存のアプリでは「自分だけの単語帳が作りたい」「特定の分野に特化した学習がしたい」といったニーズに応えきれないことがあります。

Rork を使えば、AIにプロンプトを渡すだけで、フラッシュカード・クイズ・進捗管理といった本格的な機能を備えた語学学習アプリを開発できます。プログラミングの深い知識がなくても、この記事の手順に沿って進めれば、実用的なアプリが完成します。

この記事では以下の内容を学べます。

- Rork でのプロジェクト作成とプロンプト設計
- フラッシュカード機能の実装方法
- クイズモードの設計と実装
- 学習進捗を追跡するダッシュボード
- AsyncStorage を使ったデータの永続化

対象読者は、Rork の基本操作を理解していて、実践的なアプリ開発に挑戦したい方です。Rork の基本については「[Rorkとは？AIアプリ開発プラットフォームの全体像](/articles/rork-basics/what-is-rork)」を参考にしてください。

## プロジェクトの構成を設計する

まず、アプリの全体構成を整理しましょう。語学学習アプリに必要な画面と機能は次のとおりです。

**画面構成:**
- ホーム画面 — 学習デッキの一覧と進捗サマリー
- フラッシュカード画面 — カードをめくって学習する画面
- クイズ画面 — 4択クイズで理解度をテストする画面
- 進捗画面 — 学習履歴と正答率を可視化するダッシュボード
- デッキ編集画面 — 単語の追加・編集・削除

**データモデル:**

```typescript
// デッキ（単語帳）の型定義
interface Deck {
  id: string;
  title: string;          // デッキ名（例: "英語 - ビジネス用語"）
  language: string;       // 学習言語
  cards: Card[];          // フラッシュカードの配列
  createdAt: string;
  updatedAt: string;
}

// フラッシュカードの型定義
interface Card {
  id: string;
  front: string;          // 表面（問題）
  back: string;           // 裏面（答え）
  difficulty: number;     // 難易度（1〜5、間隔反復法で使用）
  nextReviewDate: string; // 次の復習予定日
  reviewCount: number;    // 復習回数
  correctCount: number;   // 正解回数
}

// 期待する出力: 上記の型定義に基づいて、各画面でデータを管理する
```

この構造を把握した上で、Rork にプロンプトを渡します。

## フラッシュカード機能を実装する

フラッシュカードの核となるのは「間隔反復法（Spaced Repetition）」のアルゴリズムです。正解したカードの復習間隔を徐々に広げ、不正解のカードは短い間隔で繰り返し出題することで、効率的に記憶を定着させます。

Rork の追加プロンプトで、以下のロジックを実装するよう指示します。

```typescript
// 間隔反復法のアルゴリズム（SM-2の簡易版）
const calculateNextReview = (
  card: Card,
  isCorrect: boolean
): Card => {
  const now = new Date();
  let { difficulty, reviewCount, correctCount } = card;

  if (isCorrect) {
    correctCount += 1;
    // 正解するたびに難易度を下げ、間隔を広げる
    difficulty = Math.max(1, difficulty - 1);
    const intervalDays = Math.pow(2, correctCount - 1); // 1, 2, 4, 8, 16日...
    const nextDate = new Date(now.getTime() + intervalDays * 86400000);

    return {
      ...card,
      difficulty,
      reviewCount: reviewCount + 1,
      correctCount,
      nextReviewDate: nextDate.toISOString(),
    };
  } else {
    // 不正解の場合は難易度を上げ、翌日に再出題
    difficulty = Math.min(5, difficulty + 1);
    const nextDate = new Date(now.getTime() + 86400000);

    return {
      ...card,
      difficulty,
      reviewCount: reviewCount + 1,
      correctCount: 0, // 連続正解をリセット
      nextReviewDate: nextDate.toISOString(),
    };
  }
};

// 期待する出力:
// 正解時 → difficulty: 2→1, nextReviewDate: 2日後
// 不正解時 → difficulty: 2→3, nextReviewDate: 翌日, correctCount: 0にリセット
```

カードのフリップアニメーションは、Rork に次のようなプロンプトで指示します。


## クイズモードを構築する

クイズモードは、フラッシュカードのデータを活用して4択問題を自動生成します。学習の理解度を客観的にテストする重要な機能です。

```typescript
// クイズ問題の自動生成ロジック
const generateQuizQuestions = (
  deck: Deck,
  questionCount: number = 10
): QuizQuestion[] => {
  const cards = [...deck.cards];
  // 難易度が高いカードを優先的に出題
  cards.sort((a, b) => b.difficulty - a.difficulty);
  const selectedCards = cards.slice(0, questionCount);

  return selectedCards.map((card) => {
    // 正解以外のカードからランダムに3つ選んで選択肢を作成
    const otherCards = deck.cards.filter((c) => c.id !== card.id);
    const shuffled = otherCards.sort(() => Math.random() - 0.5);
    const wrongAnswers = shuffled.slice(0, 3).map((c) => c.back);

    // 正解を含む4つの選択肢をシャッフル
    const options = [...wrongAnswers, card.back].sort(
      () => Math.random() - 0.5
    );

    return {
      question: card.front,
      correctAnswer: card.back,
      options,
      cardId: card.id,
    };
  });
};

// 期待する出力:
// [
//   {
//     question: "sustainable",
//     correctAnswer: "持続可能な",
//     options: ["持続可能な", "一時的な", "経済的な", "革新的な"],
//     cardId: "card-001"
//   },
//   ...
// ]
```

Rork への追加プロンプトで、クイズ画面の UI とフィードバック機能を指示します。

```

- 不正解時: 選んだボタンが赤色に変化 + 正解のボタンが緑色に変化

- クイズ終了後にスコアサマリーを表示（正答率、所要時間）
```

## 学習進捗ダッシュボードを作る

学習のモチベーション維持には、自分の成長が可視化されることが欠かせません。進捗ダッシュボードでは、日別の学習量と正答率の推移をグラフで表示します。

```typescript
// 学習ログの型定義と集計ロジック
interface StudyLog {
  date: string;        // YYYY-MM-DD
  cardsStudied: number;
  correctAnswers: number;
  totalAnswers: number;
  studyTimeMinutes: number;
}

// 週間サマリーを計算する関数
const getWeeklySummary = (logs: StudyLog[]): WeeklySummary => {
  const now = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 86400000);

  const weekLogs = logs.filter(
    (log) => new Date(log.date) >= weekAgo
  );

  const totalCards = weekLogs.reduce(
    (sum, log) => sum + log.cardsStudied, 0
  );
  const totalCorrect = weekLogs.reduce(
    (sum, log) => sum + log.correctAnswers, 0
  );
  const totalAnswers = weekLogs.reduce(
    (sum, log) => sum + log.totalAnswers, 0
  );
  const totalMinutes = weekLogs.reduce(
    (sum, log) => sum + log.studyTimeMinutes, 0
  );

  return {
    totalCards,
    accuracy: totalAnswers > 0
      ? Math.round((totalCorrect / totalAnswers) * 100)
      : 0,
    totalMinutes,
    streak: calculateStreak(logs), // 連続学習日数
  };
};

// 期待する出力:
// { totalCards: 84, accuracy: 78, totalMinutes: 45, streak: 5 }
```


## データの永続化と同期

アプリのデータを AsyncStorage で永続化する処理も、Rork に明示的に指示することで安定した実装が得られます。

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

// デッキデータの保存と読み込み
const STORAGE_KEY = '@language_app_decks';

const saveDecks = async (decks: Deck[]): Promise<void> => {
  try {
    const jsonValue = JSON.stringify(decks);
    await AsyncStorage.setItem(STORAGE_KEY, jsonValue);
  } catch (error) {
    console.error('Failed to save decks:', error);
  }
};

const loadDecks = async (): Promise<Deck[]> => {
  try {
    const jsonValue = await AsyncStorage.getItem(STORAGE_KEY);
    return jsonValue != null ? JSON.parse(jsonValue) : [];
  } catch (error) {
    console.error('Failed to load decks:', error);
    return [];
  }
};

// 期待する出力: アプリを閉じて再起動してもデッキデータが保持される
```

AsyncStorage の詳細な活用方法は「[Rorkオフラインファースト — AsyncStorageとNetInfoで安定したアプリを作る](/articles/rork-basics/rork-offline-ready-asyncstorage-netinfo)」で解説しています。

## よくある質問（FAQ）

### Q1: デッキに登録できるカード数に上限はありますか？

AsyncStorage の容量制限（Android で約6MB、iOS で制限なし）が実質的な上限です。1枚あたりのデータサイズは数百バイト程度なので、数千枚のカードを登録しても問題ありません。大規模なデータを扱う場合は、Supabase などの外部データベースとの連携を検討してください。

### Q2: 音声読み上げ機能を追加できますか？

はい。Expo の `expo-speech` ライブラリを使えば、テキスト読み上げ機能を追加できます。Rork に「カードの表面/裏面をタップしたときに音声で読み上げる機能を追加してください。expo-speech を使用」と指示すれば、対応する言語で発音してくれる機能が実装されます。

### Q3: 他のユーザーとデッキを共有する方法はありますか？

Supabase をバックエンドに連携すれば、デッキの共有機能を実装できます。JSON 形式でデッキをエクスポート/インポートする機能をまず実装し、その後 Supabase のリアルタイムデータベースを使った共有機能に拡張するのがおすすめです。

### Q4: アプリを収益化するにはどうすればいいですか？

語学学習アプリの収益化には、プレミアムデッキの販売（Stripe 課金）、広告表示（無料版）、サブスクリプション（毎月新しいデッキを配信）などの方法があります。Rork Max を使えば、App Store への公開と課金機能の実装がスムーズに行えます。
