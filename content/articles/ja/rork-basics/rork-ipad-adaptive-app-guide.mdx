## なぜ iPad 対応が重要なのか

iPhone 向けに作ったアプリをそのまま iPad で動かすと、巨大な画面に小さな UI が中央にポツンと表示されるか、単純に引き伸ばされただけの不格好な見た目になります。App Store の審査でも iPad 対応の品質が重視されており、レビュー評価にも直結します。

iPad はいまや教育現場からビジネスの第一線まで幅広く使われています。2026年時点の iPad ラインナップ（iPad mini、iPad Air、iPad Pro 11"・13"）はそれぞれ異なる画面サイズを持ち、さらに Split View やSlide Over といったマルチタスク機能にも対応しなければなりません。

Rork を使えば、プロンプトで iPad のアダプティブ UI を指示するだけで、大部分のレイアウト調整を AI が自動生成してくれます。

## Rork で iPad レイアウトを指定する方法

### プロンプトの書き方

Rork に iPad 対応を指示するときは、具体的な画面構成をプロンプトに含めるのがポイントです。

**基本的なプロンプト例：**

```
メモアプリを作って。iPad では左側にメモ一覧、右側にメモの詳細を表示する
2カラムレイアウトにして。iPhone では一覧と詳細を別画面にして。
```

このように「iPad では〇〇、iPhone では〇〇」と明示すると、Rork は `useWindowDimensions` や Flexbox ベースのレスポンシブレイアウトを自動生成します。

### 画面サイズの検出

Rork が生成するコードでは、React Native の `useWindowDimensions` フックを使って画面幅を取得し、条件分岐でレイアウトを切り替えるパターンが一般的です。

```typescript
const { width } = useWindowDimensions();
const isTablet = width >= 768;
```

この判定によって、タブレットでは 2 カラム、スマートフォンでは 1 カラムといった切り替えが実現します。

## マルチカラムナビゲーションの実装

iPad アプリで最も効果的なパターンの一つが、マルチカラムナビゲーションです。Apple 純正アプリ（メール、メモ、設定）でおなじみのレイアウトで、左にリスト・右に詳細というマスター／ディテール構成になります。

### プロンプトの具体例

```
タスク管理アプリを作って。
iPad の横画面では以下の3カラム構成にして：
- 左サイドバー（200px）：プロジェクト一覧
- 中央カラム（300px）：タスク一覧
- 右メインエリア：タスク詳細と編集
iPhone では通常のスタックナビゲーションで。
```

Rork はこのプロンプトから、`Dimensions` API を使ったレスポンシブなレイアウトコンポーネントを生成します。カラム幅の比率や折りたたみ動作もプロンプトで調整できます。

### タブバーとサイドバーの切り替え

iPhone ではタブバーが画面下部に表示されますが、iPad の広い画面では左側のサイドバーに切り替えるのが定番です。

```
iPad ではタブバーの代わりに左サイドバーにナビゲーションを表示して。
アイコンとラベル付きで、選択中のタブはハイライトして。
```

## Split View と Slide Over への対応

iPad ユーザーの多くは、Split View を使って 2 つのアプリを並べて作業します。あなたのアプリがSplit View に対応していないと、画面が崩れたり操作できなくなったりする問題が起きます。

### Rork での対応方法

Rork で作るアプリは React Native（Expo）ベースのため、iOS のマルチタスク機能は基本的にサポートされています。ただし、レイアウトが固定幅でハードコードされている場合、Split View で表示幅が変わったときに崩れます。

**プロンプトのコツ：**

```
すべてのレイアウトは固定幅ではなくパーセンテージで指定して。
Split View で画面幅が変わっても正しく表示されるようにして。
最小幅 320px でも操作できるレイアウトにして。
```

Rork はこの指示を受けて、`flex` プロパティとパーセンテージベースのスタイルを生成します。

## Rork Max でのネイティブ iPad 機能

Rork Max を使えば、Swift ネイティブコードで iPad 固有の機能をフル活用できます。

### Apple Pencil 対応

Rork Max では PencilKit フレームワークを使って、Apple Pencil による手書き入力やスケッチ機能を実装できます。

```
お絵描きアプリを作って。Apple Pencil で描画できるキャンバスを画面いっぱいに表示して。
筆圧で線の太さが変わるようにして。消しゴムツールとカラーピッカーも付けて。
```

Rork Max は `PKCanvasView` を使ったネイティブ描画ビューを生成し、筆圧感知やパームリジェクション（手のひら誤入力防止）も自動で有効にします。

### ドラッグ＆ドロップ

iPad ではアプリ間でのドラッグ＆ドロップがサポートされています。写真アプリから画像をドラッグして自分のアプリにドロップする、といった操作を実装できます。

```
写真整理アプリを作って。他のアプリから画像をドラッグ＆ドロップで追加できるようにして。
アプリ内でも写真の並べ替えをドラッグで行えるようにして。
```

### キーボードショートカット

iPad に外付けキーボードを接続して使うユーザーも増えています。Rork Max なら `UIKeyCommand` を使ったキーボードショートカットを簡単に追加できます。

```
テキストエディタアプリで、Cmd+S で保存、Cmd+B で太字、Cmd+Z でアンドゥの
キーボードショートカットを追加して。
```

## レスポンシブ画像とフォントサイズ

### 画像の最適化

iPad の高解像度ディスプレイでは、iPhone 用の画像がぼやけて見えることがあります。

**プロンプト例：**

```
画像は端末の画面幅に応じてリサイズして表示して。
iPad では高解像度の画像を使って、iPhone ではデータ量を抑えた画像にして。
```

### Dynamic Type への対応

ユーザーがシステム設定でフォントサイズを変更した場合にも対応すべきです。Rork で Dynamic Type を使うよう指示できます。

```
テキストのフォントサイズはシステムの Dynamic Type 設定に従って。
iPad の大画面ではベースサイズを少し大きくして読みやすくして。
```

## iPad アプリの App Store 審査対策

iPad 対応アプリを App Store に提出する際は、以下のスクリーンショット要件を満たす必要があります。

**必要なスクリーンショットサイズ：**

- iPad Pro 13インチ（2048 × 2732 px）
- iPad Pro 11インチ（1668 × 2388 px）

Rork Max の Companion App を使えば、実機の iPad でアプリを動作させ、スクリーンショットを撮影できます。シミュレータでの撮影も可能です。

### 審査でリジェクトされやすいポイント

- iPhone のレイアウトをそのまま拡大しただけの UI
- 横向き（ランドスケープ）表示で崩れるレイアウト
- Split View 使用時にクラッシュする
- iPad でタッチターゲットが小さすぎる（最低 44×44pt を確保）

これらのポイントはプロンプトで明示的に指示しておくと、Rork が対応コードを生成してくれます。

## 実践チュートリアル：iPad 対応メモアプリ

ここでは、実際に Rork で iPad 対応のメモアプリを作る手順を紹介します。

### ステップ 1：基本プロンプト

```
メモアプリを作って。以下の要件で：

【共通機能】
- メモの作成、編集、削除
- Markdown 記法をサポート
- 検索機能付き

【iPad レイアウト】
- 2カラム：左にメモ一覧、右にエディタ
- サイドバーの幅は画面の30%
- 横画面・縦画面の両方に対応

【iPhone レイアウト】
- 一覧画面とエディタ画面を別々に
- 戻るボタンでナビゲーション
```

### ステップ 2：プレビューと調整

Rork のプレビューで iPhone と iPad の両方を確認します。レイアウトが思い通りでなければ、追加のプロンプトで調整します。

```
iPad のサイドバーを折りたたみ可能にして。
折りたたんだときはハンバーガーメニューで開けるようにして。
```

### ステップ 3：仕上げ

```
iPad でキーボードショートカットを追加して：
- Cmd+N で新規メモ
- Cmd+F で検索
- Cmd+Delete でメモ削除（確認ダイアログ付き）
```

## よくある質問（FAQ）

### Q: iPhone 用に作ったアプリを後から iPad 対応にできますか？

はい、Rork なら既存のプロジェクトに iPad 対応を後から追加できます。チャットで「このアプリを iPad にも対応させて」と指示するだけで、Rork がレスポンシブレイアウトへの変更を提案してくれます。

### Q: Rork（無料版）でも iPad アプリは作れますか？

React Native ベースの基本的な iPad 対応は無料版でも可能です。ただし、Apple Pencil やドラッグ＆ドロップなどのネイティブ機能を活用する場合は Rork Max が必要です。

### Q: iPad と iPhone でまったく別の画面デザインにできますか？

できます。プロンプトで iPad 専用のレイアウトと iPhone 専用のレイアウトをそれぞれ指定すれば、画面サイズに応じて完全に異なる UI を表示するコードを Rork が生成します。

### Q: iPad の横画面（ランドスケープ）にも対応すべきですか？

強く推奨します。iPad ユーザーの多くは横画面で使用しており、App Store 審査でも横画面表示が確認されます。プロンプトに「横画面と縦画面の両方に対応して」と含めておきましょう。

### Q: Split View 対応は必須ですか？

必須ではありませんが、対応していないと App Store のレビュー評価が下がる傾向にあります。Rork でパーセンテージベースのレイアウトを使えば、特別な対応なしでもSplit View でほぼ正常に動作します。
