## はじめに — なぜ割り勘アプリを作るのか

友人との食事、旅行の宿泊費、サークルの会費精算。日常生活の中で「割り勘」が必要になる場面は想像以上に多いものです。市販のアプリも便利ですが、自分だけのカスタマイズされた割り勘アプリを作れたら、もっと使いやすくなると思いませんか？

Rork を使えば、コードを書かずに実用的な割り勘アプリを作ることができます。この記事では、以下の機能を持つアプリを一緒に作っていきます。

- **グループ作成**: 旅行や食事会ごとにグループを管理
- **メンバー登録**: 参加者の追加・削除
- **支出入力**: 誰がいくら払ったかを記録
- **自動計算**: 各メンバーの負担額と精算額を自動算出
- **履歴管理**: 過去のグループ・精算履歴を一覧表示

Rork の AI アプリビルダーを活用すれば、これらの機能を備えたアプリを短時間で形にできます。

## 前提知識・環境準備

このチュートリアルを始める前に、以下を準備してください。

- **Rork アカウント**: [rork.com](https://rork.com/) で無料アカウントを作成
- **基本操作の理解**: Rork のプロジェクト作成とプロンプト入力の基本（[Rork 超入門ガイド](/articles/rork-basics/rork-first-app-30min)を参考にしてください）
- **スマートフォンまたはエミュレータ**: プレビュー確認用

プログラミングの経験は一切不要です。Rork の AI がコードを自動生成してくれるので、アプリのロジックや設計に集中できます。

## ステップ 1 — プロジェクトの作成とプロンプト設計

まず、Rork で新しいプロジェクトを作成します。プロジェクト名は「Warikan」や「SplitEasy」など、わかりやすい名前を付けましょう。

AI へのプロンプトは、具体的であるほど良い結果が得られます。以下のようなプロンプトを入力してみてください。

```text
割り勘・グループ支出管理アプリを作ってください。

主な機能:
1. グループ作成（名前、メンバー追加）
2. 支出の記録（支払った人、金額、説明）
3. 自動割り勘計算（均等割り）
4. 各メンバーの精算額表示（誰が誰にいくら払うか）
5. グループ一覧と履歴表示

デザイン:
- シンプルでモダンなUI
- タブナビゲーション（グループ一覧、新規作成、履歴）
- 金額は日本円（¥）表示
- ダークモード対応
```

このプロンプトを送信すると、Rork の AI が画面構成、ナビゲーション、データ構造を自動で設計してくれます。

### プロンプトのコツ

プロンプトを書く際に意識すると良いポイントがあります。

- **機能を箇条書きにする**: AI が構造を理解しやすくなる
- **デザインの方向性を伝える**: 「シンプル」「モダン」などのキーワードが効果的
- **データ形式を指定する**: 通貨の表示形式やソート順など

## ステップ 2 — データ構造を理解する

Rork が生成するアプリでは、主に以下のデータ構造が使われます。割り勘アプリの設計を理解するために、簡単に確認しておきましょう。

```typescript
// グループの型定義
type Group = {
  id: string;
  name: string;           // グループ名（例: "沖縄旅行"）
  members: string[];      // メンバー名の配列
  createdAt: Date;        // 作成日時
};

// 支出の型定義
type Expense = {
  id: string;
  groupId: string;        // 所属グループのID
  paidBy: string;         // 支払った人の名前
  amount: number;         // 金額
  description: string;    // 説明（例: "ランチ代"）
  createdAt: Date;        // 記録日時
};

// 精算結果の型定義
type Settlement = {
  from: string;           // 支払う人
  to: string;             // 受け取る人
  amount: number;         // 精算額
};
```

このようなデータ構造を AI が自動で作成し、画面と連携させてくれます。

## ステップ 3 — 割り勘計算ロジックの実装

割り勘アプリの核心は、各メンバーの精算額を正しく計算するロジックです。Rork が生成するコードの中核部分を見てみましょう。

```typescript
// 割り勘計算関数
// Calculate how much each member owes or is owed
function calculateSettlements(
  expenses: Expense[],
  members: string[]
): Settlement[] {
  // 各メンバーの支払い合計を計算
  const paid: Record<string, number> = {};
  members.forEach(m => { paid[m] = 0; });
  expenses.forEach(e => { paid[e.paidBy] += e.amount; });

  // 1人あたりの負担額
  const total = expenses.reduce((sum, e) => sum + e.amount, 0);
  const perPerson = total / members.length;

  // 差額を計算（プラス=もらう人、マイナス=払う人）
  const balances = members.map(m => ({
    name: m,
    balance: paid[m] - perPerson,
  }));

  // 精算ペアを生成
  const settlements: Settlement[] = [];
  const debtors = balances.filter(b => b.balance < 0)
    .sort((a, b) => a.balance - b.balance);
  const creditors = balances.filter(b => b.balance > 0)
    .sort((a, b) => b.balance - a.balance);

  let i = 0, j = 0;
  while (i < debtors.length && j < creditors.length) {
    const amount = Math.min(-debtors[i].balance, creditors[j].balance);
    if (amount > 0) {
      settlements.push({
        from: debtors[i].name,
        to: creditors[j].name,
        amount: Math.round(amount),  // 端数を丸める
      });
    }
    debtors[i].balance += amount;
    creditors[j].balance -= amount;
    if (debtors[i].balance === 0) i++;
    if (creditors[j].balance === 0) j++;
  }

  return settlements;
}

// 使用例:
// 3人で旅行。太郎が15,000円、花子が9,000円、健が6,000円を支払った場合
// → 1人あたり10,000円
// → 健は太郎に4,000円、花子は太郎に1,000円を支払う
```

この計算ロジックにより、最小限の送金回数で精算が完了する仕組みになっています。

## ステップ 4 — UI のカスタマイズと改善

Rork が生成した初期画面をベースに、さらに使いやすく改善していきましょう。以下のようなプロンプトを追加で送ることで、UI を洗練させることができます。

```text
以下の改善をお願いします:
1. グループ一覧画面にメンバー数と合計金額のサマリーを表示
2. 支出入力時に金額のテンキーパッドを表示
3. 精算結果をカード形式で見やすく表示（矢印アイコン付き）
4. グループの色分け（旅行=青、食事=緑、その他=グレー）
```

### デザインのポイント

割り勘アプリで特に重要なのは、金額の視認性です。大きなフォントサイズで金額を表示し、「誰が」「誰に」「いくら」払うかが一目でわかるようにしましょう。

Rork では、生成後のプレビュー画面で各要素をタップして微調整することも可能です。色やフォントサイズ、余白などを直感的に変更できます。

## ステップ 5 — データの永続化と履歴管理

アプリを閉じてもデータが消えないよう、Rork が提供するローカルストレージ機能を活用します。追加プロンプトで以下を指示してみましょう。

```text
AsyncStorageを使って、グループと支出データをローカルに保存してください。
アプリ起動時に自動で読み込み、新しいデータ追加時に自動保存する仕組みにしてください。
また、グループを長押しで削除できる機能も追加してください。
```

これにより、データがデバイス内に保存され、アプリを再起動しても以前の記録が復元されます。

より高度なデータ管理が必要な場合は、Supabase や Firebase などのバックエンドサービスとの連携も検討できます。複数デバイス間でのデータ同期が可能になります。

## よくあるエラーと対処法

割り勘アプリを作る過程で遭遇しやすい問題と、その解決方法をまとめました。

**金額の端数処理がおかしい**: 割り切れない金額の場合、1円単位の誤差が生じることがあります。`Math.round()` で四捨五入するか、幹事が端数を負担するルールをアプリに組み込むとスムーズです。

**メンバー名が重複する**: 同名のメンバーがいる場合に備えて、内部的には一意の ID で管理するようプロンプトで指示しましょう。表示名とは別に管理 ID を持つ設計が安全です。

**支出の編集・削除ができない**: 初回生成時に含まれないことがあります。「支出を左スワイプで削除、タップで編集できるようにしてください」と追加プロンプトを送りましょう。

## 応用 — さらに便利な機能を追加する

基本の割り勘アプリが完成したら、以下のような発展的な機能も追加してみましょう。

**不均等割り**: 「大人と子供で負担額を変えたい」というケースに対応するため、メンバーごとの負担比率を設定できる機能です。プロンプトで「メンバーごとに割合（1.0, 0.5 など）を設定できるようにしてください」と指示できます。

**レシート画像の添付**: 支出ごとに写真を添付できるようにすると、後からの確認に便利です。Expo の ImagePicker を活用した画像添付機能を追加できます。

**通知リマインダー**: 未精算のメンバーにプッシュ通知でリマインドする機能も実装可能です。

**CSV エクスポート**: グループの支出データを CSV ファイルとして書き出し、Excel や Google Sheets で管理できるようにする機能も便利です。

## FAQ

### Q: プログラミング経験がなくても本当に作れますか？

はい、Rork の AI がコードを自動生成するため、プログラミングの知識がなくても割り勘アプリを作れます。この記事のプロンプト例をそのままコピーして使うところから始めてみてください。操作に慣れたら、少しずつ自分なりのカスタマイズを加えていくと良いでしょう。

### Q: 作ったアプリを友達に配布することはできますか？

Rork で作成したアプリは、Expo Go を通じてリンク共有でプレビューできます。正式に配布したい場合は、App Store や Google Play にリリースする方法もあります。詳しくは [Rork はじめてガイド](/articles/rork-basics/getting-started) で公開手順を確認できます。

### Q: オフラインでも使えますか？

AsyncStorage を使ったローカル保存を実装すれば、インターネット接続なしでも基本的な割り勘計算と記録は可能です。ただし、クラウド同期機能を利用する場合はインターネット接続が必要になります。

### Q: 外貨（ドルやユーロ）に対応させるには？

追加プロンプトで「通貨選択機能を追加してください。日本円、USドル、ユーロに対応し、グループごとに通貨を設定できるようにしてください」と指示すれば対応可能です。為替レート変換まで組み込むと、海外旅行にも使えるアプリになります。
