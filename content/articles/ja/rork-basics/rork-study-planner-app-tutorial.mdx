## はじめに — なぜスタディプランナーアプリなのか

「今日は何時間勉強したっけ？」「先週と比べて進歩しているのかな？」——学習を継続する上で、自分の努力を可視化することはモチベーション維持に非常に効果的です。

この記事では、Rork を使ってスタディプランナーアプリをゼロから構築します。学習時間の記録、科目別カテゴリの管理、そして進捗をグラフで表示する機能を実装しながら、React Native の基本的なデータ管理やコンポーネント設計を学べます。

プログラミング経験がなくても大丈夫です。Rork の AI アシスタントがコードの大部分を生成してくれるので、アイデアを形にすることに集中できます。

- `useState` / `useEffect` による状態管理の基礎
- AsyncStorage を使ったローカルデータの永続化
- タイマー機能の実装方法
- Victory Native でデータをグラフ表示する方法

### 対象読者

- Rork を初めて触る方、または基本操作を覚えたばかりの方

## 環境準備 — Rork プロジェクトを作成する

まず、Rork で新しいプロジェクトを作成します。ブラウザで [rork.com](https://rork.com/) にアクセスし、ログイン後に新規プロジェクトを開始してください。

Rork の AI チャットに以下のようなプロンプトを入力します：

```
学習時間を記録できるスタディプランナーアプリを作ってください。
以下の画面が必要です:
1. ホーム画面: 今日の学習サマリーとクイックスタートタイマー
2. タイマー画面: 科目を選んで学習時間を計測
3. 履歴画面: 過去の学習記録をリスト表示
4. 統計画面: 週間・月間の学習時間をグラフ表示
```

Rork がベースのコードを生成したら、ここから各機能を強化していきましょう。

## 科目カテゴリの定義 — データ構造の設計

スタディプランナーの核となるのは「どの科目を何分勉強したか」というデータです。まず、科目カテゴリとセッションデータの型を定義します。

```typescript
// types/study.ts
export interface Subject {
  id: string;
  name: string;
  color: string;       // グラフ表示用のカラーコード
  icon: string;        // 絵文字アイコン
}

export interface StudySession {
  id: string;
  subjectId: string;
  startTime: string;   // ISO 8601 形式
  duration: number;     // 秒単位
  memo: string;         // 任意のメモ
}

// デフォルトの科目リスト
export const DEFAULT_SUBJECTS: Subject[] = [
  { id: 'math',    name: '数学',     color: '#4A90D9', icon: '📐' },
  { id: 'english', name: '英語',     color: '#E8734A', icon: '📖' },
  { id: 'science', name: '理科',     color: '#50C878', icon: '🔬' },
  { id: 'history', name: '社会',     color: '#F5A623', icon: '🌍' },
  { id: 'coding',  name: 'プログラミング', color: '#9B59B6', icon: '💻' },
];
```

このように型を定義しておくことで、アプリ全体で一貫したデータ構造を保てます。`color` プロパティは後ほどグラフの配色に使用します。

## タイマー機能の実装 — 学習時間を正確に計測する

スタディプランナーの主要機能はタイマーです。ユーザーが科目を選んでスタートボタンを押すと、学習時間の計測が始まります。

```typescript
// hooks/useStudyTimer.ts
import { useState, useEffect, useRef, useCallback } from 'react';

export function useStudyTimer() {
  const [isRunning, setIsRunning] = useState(false);
  const [elapsed, setElapsed] = useState(0); // 経過秒数
  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);

  // タイマー開始
  const start = useCallback(() => {
    setIsRunning(true);
  }, []);

  // タイマー停止（経過時間を返す）
  const stop = useCallback(() => {
    setIsRunning(false);
    const finalElapsed = elapsed;
    setElapsed(0);
    return finalElapsed;
  }, [elapsed]);

  // 1秒ごとにカウントアップ
  useEffect(() => {
    if (isRunning) {
      intervalRef.current = setInterval(() => {
        setElapsed((prev) => prev + 1);
      }, 1000);
    }
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
    };
  }, [isRunning]);

  // 表示用フォーマット: "01:23:45"
  const formatTime = (seconds: number): string => {
    const h = Math.floor(seconds / 3600);
    const m = Math.floor((seconds % 3600) / 60);
    const s = seconds % 60;
    return `${String(h).padStart(2, '0')}:${String(m).padStart(2, '0')}:${String(s).padStart(2, '0')}`;
  };

  return {
    isRunning,
    elapsed,
    displayTime: formatTime(elapsed),
    start,
    stop,
  };
}
// 期待する出力:
// isRunning: true/false — タイマーの動作状態
// elapsed: 0, 1, 2, ... — 経過秒数が1秒ごとに増加
// displayTime: "00:00:00" → "00:01:30" → "01:00:00"
```

カスタムフック `useStudyTimer` にロジックを閉じ込めることで、どの画面からでも同じタイマー機能を呼び出せます。`useRef` でインターバルIDを管理し、コンポーネントのアンマウント時にクリーンアップするのが React Native のお作法です。

## データの永続化 — AsyncStorage で学習記録を保存する

タイマーで計測した学習セッションを端末に保存します。AsyncStorage はキー・バリュー形式のシンプルなローカルストレージです。

```typescript
// utils/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import { StudySession } from '../types/study';

const SESSIONS_KEY = '@study_sessions';

// セッションを保存
export async function saveSession(session: StudySession): Promise<void> {
  const existing = await getSessions();
  existing.push(session);
  await AsyncStorage.setItem(SESSIONS_KEY, JSON.stringify(existing));
}

// 全セッションを取得
export async function getSessions(): Promise<StudySession[]> {
  const data = await AsyncStorage.getItem(SESSIONS_KEY);
  return data ? JSON.parse(data) : [];
}

// 特定期間のセッションを取得
export async function getSessionsByDateRange(
  startDate: Date,
  endDate: Date
): Promise<StudySession[]> {
  const sessions = await getSessions();
  return sessions.filter((s) => {
    const sessionDate = new Date(s.startTime);
    return sessionDate >= startDate && sessionDate <= endDate;
  });
}
// 期待する出力:
// getSessions() → [{ id: "abc123", subjectId: "math", startTime: "2026-03-26T10:00:00", duration: 1800, memo: "微積分の練習" }]
```

ポイントは、`getSessionsByDateRange` で日付範囲のフィルタリングに対応していることです。後述する統計画面で「今週の学習時間」「今月の学習時間」を算出する際に活用します。

## タイマー画面の UI — 科目選択とカウントダウン表示

ここまでのロジックを画面に組み立てます。Rork の AI に追加の指示を出して UI を生成し、必要に応じて手動で調整しましょう。

```tsx
// screens/TimerScreen.tsx（簡略版）
import React, { useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useStudyTimer } from '../hooks/useStudyTimer';
import { saveSession } from '../utils/storage';
import { DEFAULT_SUBJECTS, Subject } from '../types/study';

export default function TimerScreen() {
  const [selectedSubject, setSelectedSubject] = useState<Subject | null>(null);
  const { isRunning, displayTime, start, stop } = useStudyTimer();

  const handleStop = async () => {
    if (!selectedSubject) return;
    const duration = stop();
    // 30秒未満の記録は保存しない
    if (duration < 30) return;

    await saveSession({
      id: Date.now().toString(),
      subjectId: selectedSubject.id,
      startTime: new Date(Date.now() - duration * 1000).toISOString(),
      duration,
      memo: '',
    });
  };

  return (
    <View style={styles.container}>
      {/* 科目選択ボタン */}
      <View style={styles.subjects}>
        {DEFAULT_SUBJECTS.map((subject) => (
          <TouchableOpacity
            key={subject.id}
            style={[
              styles.subjectButton,
              selectedSubject?.id === subject.id && { backgroundColor: subject.color },
            ]}
            onPress={() => setSelectedSubject(subject)}
          >
            <Text style={styles.subjectIcon}>{subject.icon}</Text>
            <Text style={styles.subjectName}>{subject.name}</Text>
          </TouchableOpacity>
        ))}
      </View>

      {/* タイマー表示 */}
      <Text style={styles.timer}>{displayTime}</Text>

      {/* 開始 / 停止ボタン */}
      <TouchableOpacity
        style={[styles.actionButton, isRunning ? styles.stopButton : styles.startButton]}
        onPress={isRunning ? handleStop : start}
        disabled={!selectedSubject}
      >
        <Text style={styles.actionText}>
          {isRunning ? 'ストップ' : 'スタート'}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center', padding: 20 },
  subjects: { flexDirection: 'row', flexWrap: 'wrap', justifyContent: 'center', gap: 12, marginBottom: 40 },
  subjectButton: { alignItems: 'center', padding: 12, borderRadius: 12, backgroundColor: '#F0F0F0', width: 80 },
  subjectIcon: { fontSize: 28 },
  subjectName: { fontSize: 12, marginTop: 4, color: '#333' },
  timer: { fontSize: 64, fontWeight: '200', fontVariant: ['tabular-nums'], marginBottom: 40 },
  actionButton: { paddingHorizontal: 48, paddingVertical: 16, borderRadius: 32 },
  startButton: { backgroundColor: '#4A90D9' },
  stopButton: { backgroundColor: '#E74C3C' },
  actionText: { color: '#FFF', fontSize: 20, fontWeight: '600' },
});
```

科目を選んでからスタートする流れにすることで、記録ミスを防ぎます。30秒未満の記録は保存しないガードを入れているのも、実用上のポイントです。

## 統計画面 — Victory Native で学習データをグラフ化する

蓄積した学習データを視覚的に確認できる統計画面を作ります。[Victory Native](https://github.com/FormidableLabs/victory-native) を使って、科目別の円グラフと日別の棒グラフを表示しましょう。

```typescript
// utils/statistics.ts
import { StudySession, DEFAULT_SUBJECTS } from '../types/study';

// 科目別の合計学習時間を算出
export function calcSubjectTotals(sessions: StudySession[]) {
  const totals: Record<string, number> = {};

  sessions.forEach((s) => {
    totals[s.subjectId] = (totals[s.subjectId] || 0) + s.duration;
  });

  return DEFAULT_SUBJECTS
    .filter((subj) => totals[subj.id] > 0)
    .map((subj) => ({
      x: subj.name,
      y: Math.round(totals[subj.id] / 60), // 分に変換
      color: subj.color,
    }));
}

// 過去7日間の日別学習時間を算出
export function calcDailyTotals(sessions: StudySession[]) {
  const days: Record<string, number> = {};
  const now = new Date();

  for (let i = 6; i >= 0; i--) {
    const d = new Date(now);
    d.setDate(d.getDate() - i);
    const key = d.toLocaleDateString('ja-JP', { month: 'short', day: 'numeric' });
    days[key] = 0;
  }

  sessions.forEach((s) => {
    const d = new Date(s.startTime);
    const key = d.toLocaleDateString('ja-JP', { month: 'short', day: 'numeric' });
    if (key in days) {
      days[key] += s.duration;
    }
  });

  return Object.entries(days).map(([label, seconds]) => ({
    x: label,
    y: Math.round(seconds / 60), // 分に変換
  }));
}
// 期待する出力:
// calcSubjectTotals → [{ x: "数学", y: 90, color: "#4A90D9" }, { x: "英語", y: 45, color: "#E8734A" }]
// calcDailyTotals → [{ x: "3月20日", y: 120 }, { x: "3月21日", y: 45 }, ...]
```

この統計ロジックを Victory Native のチャートコンポーネントに渡せば、美しいグラフが描画されます。Rork のプロジェクトで `victory-native` をインストールし、`VictoryPie` と `VictoryBar` を配置するだけです。

棒グラフで日別の推移を見ることで「昨日より長く勉強できた」という達成感が生まれ、円グラフで科目バランスを確認することで「英語が少ないから今日は英語を重点的にやろう」という計画を立てやすくなります。

## 週間レポート通知 — 学習習慣をサポートする仕組み

毎週決まったタイミングで学習レポートを表示する機能を追加すると、ユーザーの学習習慣の定着に役立ちます。

```typescript
// utils/weeklyReport.ts
import { getSessionsByDateRange } from './storage';
import { DEFAULT_SUBJECTS } from '../types/study';

export async function generateWeeklyReport() {
  const now = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 24 * 60 * 60 * 1000);
  const sessions = await getSessionsByDateRange(weekAgo, now);

  const totalMinutes = Math.round(
    sessions.reduce((sum, s) => sum + s.duration, 0) / 60
  );

  const activeDays = new Set(
    sessions.map((s) => new Date(s.startTime).toDateString())
  ).size;

  const topSubject = sessions.reduce((acc, s) => {
    acc[s.subjectId] = (acc[s.subjectId] || 0) + s.duration;
    return acc;
  }, {} as Record<string, number>);

  const topId = Object.entries(topSubject).sort(([, a], [, b]) => b - a)[0]?.[0];
  const topName = DEFAULT_SUBJECTS.find((s) => s.id === topId)?.name || '—';

  return {
    totalMinutes,          // 週間合計（分）
    activeDays,            // 学習した日数
    sessionCount: sessions.length,
    topSubject: topName,   // 最も勉強した科目
  };
}
// 期待する出力:
// { totalMinutes: 420, activeDays: 5, sessionCount: 12, topSubject: "数学" }
```

レポートデータをホーム画面のカード UI に表示すれば、アプリを開くたびに先週の成果を振り返ることができます。

## よくある質問（FAQ）

### Q1: 学習記録のデータはクラウドに同期できますか？

AsyncStorage はローカル保存のため、端末を替えるとデータが引き継がれません。クラウド同期を追加したい場合は、[Rork でユーザー認証を実装する — Firebase・Supabase 連携ガイド](/articles/rork-dev/rork-auth-firebase-supabase)を参考に Supabase や Firebase との連携を検討してください。テーブル設計は `StudySession` 型をそのまま活用できます。

### Q2: iOS と Android の両方で動作しますか？

はい。Rork は React Native（Expo）ベースなので、今回のコードは iOS・Android の両プラットフォームで動作します。タイマーの精度やバックグラウンド動作については OS による違いがありますが、フォアグラウンドでの計測は両 OS で安定して動きます。

### Q3: 科目カテゴリをユーザーが自由に追加・編集できるようにするには？

`DEFAULT_SUBJECTS` を固定配列ではなく AsyncStorage に保存する設計に変更すれば、ユーザーが自由にカテゴリを追加・編集・削除できるようになります。設定画面を追加して、科目名・カラー・アイコンの入力フォームを作ればOKです。

### Q4: Rork 以外のツールと比べて、この種のアプリを作る利点は？

Rork はプロンプトから UI を生成し、React Native のコードをそのままカスタマイズできる点が強みです。ノーコードツールでは難しい「カスタムフック」や「独自のデータ集計ロジック」もコードレベルで自由に書けます。Rork の使い方をまだ確認していない方は、[Rork を使い始めるためのガイド](/articles/rork-basics/getting-started)もあわせてご覧ください。
