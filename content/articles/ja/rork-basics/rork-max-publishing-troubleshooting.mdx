Rork Max でアプリを生成した後、「いざ公開しよう」というとき、様々な問題にぶつかることがあります。本記事では、実際の公開フロー全体を整理し、症状別に原因と解決策をお伝えします。

## Rork Max とは・公開フローの全体像

Rork Max は、AI がテキスト指示から SwiftUI のネイティブ iOS アプリを自動生成するサービスです。生成されたアプリは、ローカルで実行・テストしたあと、App Store へ公開することが最終目標となります。

公開までの流れは、大きく 3 つのステージに分かれます。

1. **Rork Max での準備フェーズ**
   - アプリを生成
   - Apple Developer Account との連携（認証情報の設定）
   - アプリの基本情報入力（アプリ名、説明、スクリーンショット等）
   - Rork Max 上での「公開」ボタンをクリック

2. **App Store Connect での登録・設定フェーズ**
   - App Store Connect にログイン
   - 新規アプリを作成 or 既存アプリに紐付け
   - メタデータ（カテゴリ、ターゲット OS、プライバシーポリシー URL など）を設定
   - ビルドを提出

3. **Apple の審査フェーズ**
   - Apple による自動・手動審査
   - 承認 or 却下のレビュー結果
   - 却下時は修正して再提出

これら 3 つのいずれかの段階で問題が発生することが多いです。次章から、症状別の原因と対処法をご説明します。

## 公開できない原因・症状別チェックリスト

### a. Rork Max 上での「公開」操作自体でエラーが出る場合

**症状**: Rork Max のアプリ詳細ページで「公開」ボタンをクリックしても、エラーメッセージが表示される、または何も起こらない。

**考えられる原因**:

- **Apple Developer Account が未連携**
  - Rork Max のアカウント設定で Apple ID が登録されていない
  - または Apple ID のパスワードが間違っている

- **証明書（Signing Certificate）が無い**
  - Apple Developer Program に登録した後、Xcode や Web から証明書を発行していない
  - 証明書有効期限が切れている

- **Provisioning Profile が無い or 無効**
  - 開発用・配布用の Provisioning Profile が存在しない
  - プロファイルの有効期限が切れている

**確認・対処方法**:

1. **Apple ID が正しく登録されているか確認**
   - Rork Max アプリの設定画面（左上のメニュー → 設定）を開く
   - 「Apple Developer Account」セクションで Apple ID が表示されているか確認
   - 表示されていない場合は、Apple ID でログイン後、パスワード入力を求められます。2FA (2要素認証) コードも入力してください

2. **証明書を確認・発行**
   - [Apple Developer](https://developer.apple.com/) にログイン
   - 「Certificates, Identifiers & Profiles」→「Certificates」をクリック
   - iOS Development / iOS Distribution の証明書が存在するか確認
   - 無ければ「+」から新規作成（以降 Xcode でも同じ証明書を共有できます）

3. **Provisioning Profile を確認・更新**
   - Apple Developer サイト → 「Provisioning Profiles」をクリック
   - 「Development」と「Distribution」両方のプロファイルが存在するか確認
   - 無ければ「+」から新規作成
   - 有効期限が切れている場合は削除して再作成

上記をクリアしても問題が続く場合は、Rork Max のアプリ設定画面で「Apple Account を再接続」オプションをお試しください。

### b. App Store Connect への送信で詰まる場合

**症状**: Rork Max から「公開」後、App Store Connect に自動でビルドが送信されたはずだが、ビルドが到着していない / 手動で送信しようとしてもエラーが出る。

**考えられる原因**:

- **Bundle ID が不正 or 重複している**
  - iOS アプリは「com.example.appname」の形式で Bundle ID が決まります
  - 既に他のアプリで同じ Bundle ID が使用されていないか確認

- **App ID が App Store Connect に存在しない**
  - Bundle ID に対応する App ID (Identifier) が Apple Developer で登録されていない

- **ビルドの署名情報が無効**
  - 使用する証明書・Provisioning Profile と Bundle ID が不一致

**確認・対処方法**:

1. **Bundle ID を確認・変更**
   - Rork Max アプリの「設定」→「App Info」で Bundle ID を確認
   - 以前テストで別の Bundle ID を使った場合は、そちらに統一するか新規の値に変更

2. **App Store Connect で Identifier を登録**
   - App Store Connect にログイン → 「Certificates, Identifiers & Profiles」
   - 「Identifiers」から「+」をクリック
   - Bundle ID に Rork Max で設定した値を入力
   - Capabilities に「Push Notifications」等が必要なら追加

3. **ビルドの署名を確認**
   - Rork Max でビルドを再生成 → 「公開」
   - または Xcode から手動で提出する場合、Product → Archive → Distribute App で署名情報を確認

### c. Apple のレビューで却下される場合

**症状**: ビルドが App Store Connect に到着したが、数時間〜数日後に「Rejected」と却下されてしまった。

**考えられる原因**:

Apple が却下する理由の多くは「ガイドライン違反」です。一般的な理由：

- **プライバシーポリシーが無い or 要件を満たしていない**
  - ユーザーデータを扱うアプリには必須
  - URL が正しく機能するか Apple が実際にアクセスして確認

- **説明文・スクリーンショットが不十分**
  - アプリの機能が正確に説明されていない
  - スクリーンショットが古い / 実装と異なる

- **デザイン or 機能が「不完全」と判断**
  - UI が極端に粗い
  - ボタンが反応しない
  - ランダムクラッシュが発生する

- **Rork Max 生成特有の問題**
  - 予期しないレイアウト崩れ
  - テキスト入力フィールドが機能しない
  - ネットワーク処理が失敗して復帰しない

**確認・対処方法**:

1. **App Store Connect の Review Information を確認**
   - 「Rejection」メッセージを読む
   - Apple から示されたガイドライン番号（例：1.2 Spam など）を確認

2. **プライバシーポリシーを用意**
   - ユーザーデータ（ユーザー名、メールアドレス、位置情報等）を扱う場合は必須
   - HTTPS でアクセス可能な URL でホストする
   - 日本語版と英語版の両方が あると尚良い

3. **スクリーンショットと説明を実装と整える**
   - Rork Max で再度アプリを実行して、最新の見た目でスクリーンショットを取り直す
   - 説明文も実装に合わせて更新

4. **アプリをローカルでテスト**
   - Rork Max 生成のアプリを iPhone 実機 or シミュレータで試す
   - 全機能が正常に動作するか確認
   - 予期しないクラッシュが無いか確認

5. **却下理由に応じて修正＋再提出**
   - Rork Max でアプリを改善 → 新しいビルドを生成 → 再度提出
   - または Xcode で直接コードを修正

### d. Google Play での審査落ち

**症状**: Android 版を Google Play に公開しようとして却下された。

**考えられる原因**:

- **プライバシーポリシーが無い**
  - iOS 同様、Google Play でも必須

- **説明・スクリーンショットの不足**
  - iOS より審査が簡潔だが、基本は同じ

- **APK の署名情報不正**
  - 誤ったキーストアで署名されている

**対処方法**:

Rork Max の Android 版がまだ完全リリースされていない場合もあります。その場合は Rork Max で生成した Swift コードから Flutter / React Native などでクロスプラットフォーム対応することを検討してください。あるいは Rork Companion で Android 版を別途生成することもできます。

### e. アプリが承認されたのに App Store で見つからない

**症状**: 「Approved」のメールが届いたはずだが、App Store で検索してもアプリが出ない。

**考えられる原因**:

- **App Store のインデックスに反映されるまで時間がかかっている**
  - 承認後、数時間〜24 時間程度を要することがある

- **App Store に公開予約が設定されている**
  - 「Release on Approval」ではなく「Manual Release」が選ばれている

- **リージョン設定で日本が除外されている**
  - App Store Connect の「Pricing and Availability」で「Available in Japan」が OFF になっている

**確認・対処方法**:

1. **App Store Connect で Release オプションを確認**
   - 対象アプリの「App Information」→「Release」をクリック
   - 「Release on Approval」が選ばれているか確認
   - 「Manual Release」の場合は「Release Version」をクリックして手動で公開

2. **地域設定を確認**
   - 「Pricing and Availability」から「Japan」が「Available」に設定されているか確認

3. **24 時間待つ**
   - それでも見つからない場合は、Apple Support に問い合わせ

## Rork Max で生成されたアプリが App Store 審査を通るためのポイント

### プライバシーポリシーの必須設定

ユーザーデータを扱う場合は、以下を準備してください。

- **プライバシーポリシーのテキスト**
  - ユーザー データの収集・利用・保護方法を明記
  - GDPR（EU）・CCPA（米国）等への対応も表記

- **ホスティング**
  - Wix・Notion・Squarespace など無料サイト作成サービスでも OK
  - または自社 Web サイトのページとして追加

- **アプリ内からアクセス可能**
  - App Store Connect メタデータで「Privacy Policy URL」に入力
  - アプリ内の「設定」等からも URL リンクを配置推奨

### スクリーンショット要件

- **最大 5 枚（iOS）・8 枚（Android）**
  - 最初の 1〜2 枚が最重要。アプリの主機能を一目で理解できるものを選ぶ
  - テキストオーバーレイで「キャッチコピー」を入れると効果的

- **見た目**
  - デバイスフレーム（iPhone 枠）が自動で付与される場合が多い
  - 余白・テキストサイズは大きめが読みやすい

- **更新タイミング**
  - アプリを大幅アップデートしたら必ず新しいスクリーンショットに変更

### メタデータの書き方

**App Name** (アプリ名)
- 30 文字以内
- Rork Max で生成した名前をそのまま使うか、より短くしたバージョンを検討

**Subtitle** (字幕・説明用副題)
- iOS 1.1.1 以降で利用可
- 30 文字以内
- 主な機能を 1 行で表現（例：「AI が画像を識別する」）

**Description** (詳細説明)
- 4,000 文字まで
- 最初の 170 文字が重要（App Store で省略表示される）
- 改行して段落を付けると読みやすい

**Keywords** (検索キーワード)
- 最大 100 文字（全キーワードを合わせた文字数）
- Rork Max のトピック名・機能名を複数選定
- 例：「AI, 画像認識, 写真」

**Support URL**
- サポートサイト・メールアドレス等
- ユーザーの問い合わせ対応先

**Privacy Policy URL**
- 必須（データを扱う場合）
- HTTPS で始まる有効な URL

## よくある却下理由 TOP 5 と対処法

### 1. 「Guideline 4.3 - Design: Spam」

**意味**: スパム的・コンテンツが薄いと判断された。

**対処法**:

- アプリの説明文を詳しくする
- スクリーンショットで機能を明確に示す
- Rork Max で生成したアプリの機能に独自性があるか見直す（同じようなアプリが既にたくさんあるなら、差別化ポイントを追加）

### 2. 「Guideline 2.1 - Information Needed」

**意味**: Apple がメタデータの説明が不足していると判断。

**対処法**:

- メタデータの全項目を埋める（Subtitle・Description・Support URL 等）
- Description に具体的な使用例を 2〜3 個書く
- 曖昧な表現（「素晴らしい」「便利」等）は避け、「何ができるのか」を客観的に書く

### 3. 「Guideline 1.2 - Safety: User-Generated Content」

**意味**: アプリがユーザーコンテンツを扱う場合、モデレーション体制が不充分。

**対処法**:

- Rork Max でユーザーコンテンツ機能が自動生成された場合、悪質投稿のフィルタリング・ブロック機能を明示
- プライバシーポリシーにコンテンツ削除手順を記載
- Support URL（連絡先）を明確にする

### 4. 「Guideline 5.1 - Legal: Intellectual Property」

**意味**: 著作権・商標侵害の疑い。

**対処法**:

- Rork Max で生成した際、既存の有名キャラ・曲等を勝手に使っていないか確認
- アプリの説明文・スクリーンショットに他社ロゴ・製品名が含まれていないか確認
- 著作権帰属表示が必要な場合は明記

### 5. 「Guideline 1.4.4 - Health and Fitness」

**意味**: 医療・健康情報を扱うアプリが、根拠のない主張をしている。

**対処法**:

- Rork Max で健康系アプリを生成した場合、「医学的根拠がない」旨を明記
- 医学アドバイスではなく「参考情報」として提示
- 医学的な正確性が求められる場合は医師監修を検討

## スムーズに公開するための事前準備リスト

公開前に以下をチェックしておくと、審査が通りやすくなります。

### Apple Developer Program への登録

- [ ] Apple ID を持っている
- [ ] Apple Developer Program に年 99 ドルで登録済み
- [ ] Developer Profile を完成させている（苗字・会社名等）

### 証明書・Provisioning Profile の準備

- [ ] iOS Development Certificate を発行済み
- [ ] iOS Distribution Certificate を発行済み
- [ ] Development Provisioning Profile が存在
- [ ] Distribution Provisioning Profile が存在

### App Store Connect の初期設定

- [ ] App Store Connect にログイン可能
- [ ] 新しいアプリを作成 or 既存アプリを確認
- [ ] App Name・Bundle ID を決定
- [ ] プライバシーポリシー URL を準備（ホスティング先決定）

### メタデータの準備

- [ ] アプリアイコン（1024×1024 PNG）を用意
- [ ] スクリーンショット（3 枚以上）を撮影
- [ ] Subtitle を 30 文字以内で作成
- [ ] Description を 170 文字以上で作成
- [ ] Keywords を複数選定
- [ ] Support URL を決定

### Rork Max での最終確認

- [ ] アプリをローカルで実行して全機能をテスト
- [ ] テキスト入力・ボタン押下・画面遷移に問題がないか確認
- [ ] 予期しないクラッシュがないか確認
- [ ] iOS 17 以降で動作確認

### 公開直前のチェック

- [ ] Rork Max で Bundle ID・App Name を最終確認
- [ ] Apple Developer Account が Rork Max で正しく連携されているか確認
- [ ] App Store Connect で 「Version Release」が「Release on Approval」に設定されているか確認
- [ ] 「Pricing and Availability」で日本が「Available」に設定されているか確認

全てを準備してから「公開」ボタンをクリックすると、審査に進むまでのエラーはほぼ回避できます。

Rork Max で生成したアプリをスムーズに公開できることを願っています。困ったときは、本記事のチェックリストに立ち戻ってご確認ください。
