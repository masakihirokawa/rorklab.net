Rork Maxの「2クリック公開」は非常に強力ですが、実際には様々な問題が起こることがあります。本記事では、Rork Maxでアプリを公開する際に直面する一般的な問題と、その解決方法を体系的に解説します。

## Apple Developer Accountの設定問題

アプリ公開の失敗は、Apple Developer Accountの設定に問題があることが多いです。

### プロビジョニングプロファイルが見つからない

**症状**: 「Provisioning Profile not found」というエラーが表示される

**原因**: Rork Maxは自動的にプロビジョニングプロファイルを生成しますが、以下の場合に失敗します：
- Apple Developer Programに登録していない
- Team IDが正しく設定されていない
- Xcodeに証明書がキャッシュされていない

**解決方法**:
1. Rork Max設定画面で「Apple Developer Account」を再接続
2. Two-Factor Authentication（2FA）を有効にしている場合、App Password を生成してください
3. Rork Maxでチーム選択を明示的に行い、保存する

### ビルド署名エラー（Signing失敗）

**症状**: 「Failed to sign the build」または「Code signing is required」

**原因**: 開発用の証明書が無効化、または失効している

**解決方法**:
1. Apple Developer Website（https://developer.apple.com）にログイン
2. 「Certificates, Identifiers & Profiles」 → 「Certificates」を開く
3. 有効な証明書があるか確認し、必要に応じて新規作成
4. Xcodeを再起動して証明書をキャッシュリセット: `rm -rf ~/Library/MobileDevice/Provisioning\ Profiles/*`

### Bundle IDが既に使用されている

**症状**: 「Bundle ID is already in use by another team」

**原因**: 同じBundle IDで複数のアプリを公開しようとしている、または誤った古いBundle IDを使用している

**解決方法**:
1. Rork Maxで別のBundle IDを入力（例: `com.mycompany.myapp2`）
2. Apple Developer Websiteで既存のBundle IDが登録されていないか確認
3. 必要に応じてBundle IDの命名規則を見直す: `com.[company].[appname]`

## App Storeレビュー拒否（Rejection）への対応

ビルドは成功しても、App Storeのレビュー段階で拒否される場合があります。以下は頻出の理由と対策です。

### Guideline 2.1 - アプリのクラッシュやバグ

**症状**: 「The app crashed on launch」

**原因**: アプリに重大なバグがあり、起動時にクラッシュしている

**対策**:
- Rork Maxで生成されたコードを確認
- TestFlightでベータテストを実施し、問題を再現
- クラッシュログを確認して根本原因を特定
- 修正後、新しいビルド番号でレビュー再申請

### Guideline 2.1 - 不完全な機能説明

**症状**: 「The feature is not fully implemented or behaves unexpectedly」

**原因**: アプリの説明と実装のギャップ、または実装されていない機能を説明している

**対策**:
- App Store Connect のメタデータを確認
- スクリーンショットで実装された機能のみ説明
- キーワードとアプリの機能が一致していることを確認
- 「Coming Soon」機能は説明や画像に含めない

### Guideline 5.1.1 - プライバシーポリシーの欠如

**症状**: 「This app requests user-sensitive data but does not have a privacy policy」

**原因**: 位置情報、カメラ、連絡先などのセンシティブ情報にアクセスするのに、プライバシーポリシーが設定されていない

**対策**:
1. プライバシーポリシーの URL を App Store Connect に設定
2. 以下の項目を明記すること：
   - どのデータを収集するか
   - どのような目的で使用するか
   - データの第三者への共有
   - ユーザーが自分のデータを削除する方法

### ガイドライン 4.3 - スパムのような外観

**症状**: 「Your app looks like spam or is functionally similar to an existing app」

**原因**: アプリが既存アプリと過度に似ている、または低品質に見える

**対策**:
- 独自価値を持つ機能を追加
- 高品質なスクリーンショット（最新デバイスでの実際の表示）
- プレビュー動画を提供
- アプリの説明を詳細に、読みやすく

## メタデータ・スクリーンショットの問題

### スクリーンショット解像度エラー

**症状**: 「The screenshot dimensions are incorrect」

**原因**: スクリーンショットが要求される解像度（デバイスごと）に合致していない

**解決方法**:
- iPhone 15 Pro Max: 1290×2796 px
- iPhone 15: 1179×2556 px
- iPad Pro（12.9）: 2048×2732 px

各デバイスの正確なサイズで撮影・編集してください。

### アプリアイコンが不適切

**症状**: 「The app icon does not meet our requirements」

**原因**: アイコンが要求される仕様に合致していない

**要件**:
- サイズ: 1024×1024 px
- 形式: PNG、JPEG、GIF
- 角丸なし（App Storeが自動的に角丸化）
- 透明度不可（完全な背景色必須）

### アプリ説明文が不十分

**症状**: Approveされるが、ユーザーの評価が低い

**原因**: メタデータの品質が低く、ユーザーの期待値との乖離

**改善策**:
- 最初の2行で主要機能を明記
- 箇条書きで利点をまとめる
- キーワード詰め込みではなく、自然な文体で
- 各言語版でネイティブな表現を使う

## ビルドエラーの診断

### 「Build Failed」が出たが詳細がわからない

**対策**:
1. Rork Maxのビルドログを確認：設定 → ビルド履歴
2. Xcodeを直接開く場合は、以下のコマンドで詳細ログを確認：
```bash
xcodebuild -showBuildSettings
```
3. よくあるエラー：
   - `Module not found`: 必要なライブラリのインストール漏れ
   - `Syntax error in Swift`: 生成コードにバグがある（Rork Maxサポートに報告）
   - `Unable to boot simulator`: シミュレーターの再起動が必要

### メモリ不足でビルド失敗

**症状**: ビルド途中でプロセスが終了する

**対策**:
- 使用中のアプリを閉じてメモリ解放
- Xcodeキャッシュをクリア: `rm -rf ~/Library/Developer/Xcode/DerivedData/*`
- Macを再起動
- 最後の手段: Rork Maxをアンインストール・再インストール

## TestFlightでのベータテストフロー

App Storeへの直接公開の前に、TestFlightを必ず活用してください。

### TestFlightへのビルド登録

1. Rork Maxでビルド完了後、「TestFlightに送信」ボタンを押す
2. App Store Connect で TestFlight タブを開く
3. 内部テスター（自分やチーム）を追加
4. ビルドのテスト期間を待つ（通常1～3時間）
5. iPhoneにTestFlightアプリをインストールし、テスト版をインストール

### ベータテストのチェックリスト

- [ ] アプリが起動する
- [ ] 主要機能が動作する
- [ ] クラッシュが発生しない
- [ ] 画面表示がiPhone / iPadで正しい
- [ ] 位置情報・カメラなどのパーミッション要求が正常か
- [ ] バックグラウンド処理が正常か

テストで問題を発見した場合、修正後に新しいビルドを作成し、再度TestFlightに登録します。

## よくある404・空白画面の原因

### 「アプリが見つかりません」エラー

**原因**: App Store Connectのビルド選択が間違っている

**対策**:
1. App Store Connect → ビルド タブ
2. 正しいビルド番号が「Submitted」状態か確認
3. 必要に応じて別のビルドを選択

### アプリが起動したが空白画面

**原因**: UI生成に失敗、またはアセットが読み込まれていない

**対策**:
- Rork Maxの「プレビュー」で動作確認
- アセット（画像等）が正しく含まれているか確認
- ネットワーク接続が必要な場合、TestFlightでネット接続を確認
- ローカルで Xcode でシミュレータ実行し、コンソールエラーを確認

## 公開後の運用

### バージョンアップの流れ

1. Rork Maxでアプリを改変
2. App Store Connect でビルド番号をインクリメント（例: 1.0.1）
3. リリースノート（What's New）を記入
4. 審査に提出

### よくある拒否理由（アップデート時）

- 新機能を説明していない（What's Newが不十分）
- 古い機能の削除が説明されていない
- プライバシーポリシーが変更されていない

これらはリリースノートと設定の見直しで解決します。

## まとめ

Rork Maxでのアプリ公開は簡単ですが、App Storeのガイドラインをしっかり理解し、事前にTestFlightで充分なテストを実施することが成功の鍵です。本記事のチェックリストを参考に、スムーズな公開を目指してください。

不明な点がある場合は、Rork Labのドキュメントやコミュニティを活用してサポートを受けることをお勧めします。
