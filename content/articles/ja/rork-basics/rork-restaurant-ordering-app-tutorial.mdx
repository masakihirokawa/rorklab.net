飲食業界でのデジタル化は急速に進んでいます。QRコードメニューからオンライン予約まで、顧客体験を向上させるアプリへの需要は高まる一方です。Rork Maxを使えば、こうした本格的な飲食店向けアプリをコードなしで構築できます。

## 飲食店アプリの要件定義

まず、どんな機能が必要かを整理しましょう。一般的な飲食店アプリには次のような機能が求められます。

- **メニュー表示**: カテゴリ別のメニュー一覧と詳細ページ
- **テーブル予約**: 日時・人数・席種の選択と確認
- **注文管理**: 席からのモバイル注文と厨房への通知
- **決済**: Stripeを使ったカード決済
- **スタッフ用ダッシュボード**: 予約・注文の一元管理

この記事では、上記の全機能を持つアプリをRork Maxで作る流れを順を追って説明します。

## Supabaseのスキーマ設計

バックエンドにはSupabaseを使います。以下のテーブル構成が基本になります。

**menu_items テーブル**

```
id            uuid        (主キー)
name          text        (商品名)
description   text        (説明文)
price         integer     (価格、円)
category      text        (カテゴリ: appetizer/main/drink/dessert)
image_url     text        (画像URL)
is_available  boolean     (提供可否)
created_at    timestamp
```

**reservations テーブル**

```
id            uuid        (主キー)
customer_name text        (予約者名)
phone         text        (電話番号)
party_size    integer     (人数)
reserved_at   timestamp   (予約日時)
table_type    text        (テーブル種別: counter/table/private)
status        text        (pending/confirmed/cancelled)
notes         text        (備考)
created_at    timestamp
```

**orders テーブル**

```
id            uuid        (主キー)
table_number  text        (テーブル番号)
items         jsonb       (注文アイテム配列)
total_amount  integer     (合計金額)
status        text        (pending/cooking/ready/served/paid)
stripe_session_id text   (決済ID)
created_at    timestamp
```

Supabaseのダッシュボードで上記テーブルを作成したら、Row Level Security (RLS) の設定も行います。スタッフ用の操作には認証が必要になるよう設定しましょう。

## Rork Maxでのプロジェクト設定

Rork Maxを開いて新しいプロジェクトを作成します。プロンプトは具体的であるほど良い結果が得られます。

**最初のプロンプト例:**

```
飲食店向けのモバイルアプリを作ってください。

機能:
1. メニュー一覧画面（カテゴリフィルター付き）
2. メニュー詳細画面（画像・価格・説明）
3. テーブル予約フォーム（日時・人数・席種選択）
4. 注文画面（カートに追加・数量変更）
5. 注文確認・決済画面
6. スタッフ用管理画面（予約・注文の一覧と状態更新）

データベース: Supabase（接続情報を後で設定）
決済: Stripe
デザイン: 温かみのある和食レストランのイメージ、落ち着いたブラウン系
```

このプロンプトを入力するとRorkがアプリの骨格を生成します。生成後、Supabaseの接続設定を行います。

## Supabase接続の設定

Rorkの設定画面でSupabaseのプロジェクトURLとAPIキー（anon key）を入力します。入力後、「テスト接続」ボタンで疎通確認を行いましょう。

接続が確認できたら、Rorkのプロンプトでデータ取得の実装を依頼します。

```
Supabaseのmenu_itemsテーブルからデータを取得して、メニュー一覧画面に表示してください。
is_available=trueのアイテムのみ表示し、categoryでフィルタリングできるようにしてください。
```

## 予約フォームの実装

予約機能は飲食店アプリの核心です。次のようにプロンプトを追加します。

```
予約フォームを実装してください。
- 日付選択（カレンダーUI、過去日は選択不可）
- 時間選択（17:00〜22:00、30分刻み）
- 人数選択（1〜20名）
- 席種選択（カウンター/テーブル/個室）
- 予約者名と電話番号の入力
送信後はSupabaseのreservationsテーブルに保存し、確認画面を表示してください。
```

予約の重複チェックも重要です。同じ時間帯・同じ席種で既に予約が入っている場合はエラーを表示するよう追加で指示します。

## Stripe決済の組み込み

決済はStripeを使います。Rorkの設定画面でStripeのパブリッシャブルキーを設定した後、以下のプロンプトで実装します。

```
注文確認画面にStripe決済を組み込んでください。
合計金額を計算してStripe Checkoutを起動し、決済完了後にordersテーブルのstatusを'paid'に更新してください。
決済IDはstripe_session_idカラムに保存してください。
```

テストモードでカード番号 `4242 4242 4242 4242`（有効期限は任意の将来の日付、CVCは任意の3桁）を使って動作確認ができます。

## スタッフ用ダッシュボードの追加

管理者向け機能も追加しましょう。

```
スタッフ用のダッシュボード画面を追加してください。
- 本日の予約一覧（時間順）
- 現在進行中の注文一覧（テーブル番号・ステータス・経過時間）
- 注文ステータスの更新ボタン（cooking/ready/served）
- 認証: メールアドレスとパスワードでのログイン（Supabase Auth使用）
```

スタッフがログインしていない状態ではダッシュボードにアクセスできないよう、RLSポリシーと合わせて設定します。

## テストとデバッグのコツ

アプリが完成したら、以下の手順でテストします。

まずメニューデータをSupabaseに直接入力し、アプリに正しく表示されることを確認します。次に予約フォームを操作し、Supabaseのダッシュボードでデータが保存されているか確認します。

注文から決済までの流れはStripeのテストモードで何度も試しましょう。特に決済失敗時のエラーハンドリングが正しく動作するか確認が重要です。

## まとめ

Rork Maxを使えば、飲食店向けの本格的なアプリを短時間で構築できます。Supabaseによるリアルタイムデータ管理とStripeの決済機能を組み合わせることで、実際の営業で使えるレベルのアプリが完成します。

次のステップとして、プッシュ通知（新しい予約・注文が入った際のスタッフへの通知）やQRコードによるテーブル識別機能の追加もご検討ください。Rork Maxのプロンプトに機能を追加するだけで、アプリはさらに進化していきます。
