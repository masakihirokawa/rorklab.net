## Rork AI とは — コード不要でアプリ開発ができる時代へ

2026年、アプリ開発の風景は大きく変わりましました。プログラミング経験がなくても、AI の力を借りて本格的なモバイルアプリを作成できるようになったのです。その中心にあるのが **Rork** です。

Rork は、自然言語で指示を与えるだけで、React Native による iOS・Android 対応アプリを自動生成する革新的なプラットフォーム。デザイナーであっても、ビジネスパーソンであっても、あなたのアイデアを数時間でアプリにできます。

本記事では、Rork 初心者が知るべきすべてを、セットアップから App Store・Google Play への公開まで、実践的に解説します。

### この記事で学べること

- Rork の基本的な仕組みと選べるプラン
- アカウント作成からプロジェクト作成までのステップ
- AI を効果的に活用するプロンプトのコツ
- UI 生成とデザインカスタマイズの方法
- テストとデバッグの基本
- App Store・Google Play への公開手順

対象は、Rork を初めて使う方、アプリ開発経験がない方です。難しい用語は説明を加えているので、安心して読み進めてください。

## Rork の基本 — どんなアプリが作れるのか

### Rork でできること

Rork は、以下のような多様なアプリ開発に対応しています。

- **実用ツール系** — 天気アプリ、計算機、TODOリスト、メモアプリ
- **SNS・コミュニティ** — チャットアプリ、掲示板、マッチングアプリ
- **ビジネスアプリ** — 顧客管理、在庫管理、請求書作成ツール
- **ゲーム** — パズルゲーム、クイズゲーム、簡易的な RPG
- **位置情報連携** — 地図ベースのアプリ、配達管理アプリ
- **IoT 連携** — スマートホーム操作、センサーデータ表示

重要なのは、これらが「プロンプト入力だけで」実現できるということです。JavaScript や Swift の知識は不要です。

### Rork と他のノーコード平台の違い

市場には様々なノーコードプラットフォームがありますが、Rork が際立つ理由は以下の通りです。

| 特徴 | Rork | FlutterFlow | Bubble | Adalo |
|------|------|------------|--------|-------|
| ネイティブアプリ対応 | iOS・Android ネイティブ | iOS・Android | Web のみ | iOS・Android |
| AI による自動生成 | ✓ 強力 | × | × | × |
| App Store 直接公開 | ✓ 2クリック | 手動 | 不可 | 手動 |
| 料金 | 月額 $39～ | 月額 $25～ | 月額 $12～ | 月額 $99～ |
| 学習曲線 | 非常に優しい | やや急 | やや急 | 中程度 |

**つまり、Rork は「AI の自動生成力」と「ネイティブアプリの品質」を両立させた、現時点で最もアクセスしやすいプラットフォームなのです。**

## Rork のプランと選び方

### 3つのプランの比較

```
【Rork Free】
- 月額 0 円
- 生成回数 週 5 回まで
- App Store / Google Play への公開 不可
- 最大ファイルサイズ 50MB
- 用途：試験的な学習、シンプルなアプリ開発

【Rork Pro】
- 月額 $39（約 5,900 円）
- 生成回数 無制限
- App Store / Google Play への公開 ✓
- 最大ファイルサイズ 200MB
- 高度な API 連携
- 用途：本格的なアプリ開発、個人・小規模チーム向け

【Rork Max】
- 月額 $99（約 15,000 円）
- Pro のすべての機能 + 優先サポート
- ワンクリック App Store / Google Play 申請
- カスタムドメイン対応
- API レート無制限
- 用途：複数アプリ展開、ビジネス利用、成長中の企業
```

**初心者向けアドバイス：** まずは Free プランで Rork の感覚を掴み、アプリを公開したくなったら Pro にアップグレードするのが無駄がありません。Pro なら個人開発者が複数アプリを公開して収益化するのに十分な機能があります。

## セットアップから最初のアプリ生成まで

### ステップ 1：Rork に登録する

1. **Rork 公式サイト**（https://rork.ai）にアクセス
2. 「Get Started」をクリック
3. メールアドレスまたは Google/GitHub アカウントで登録
4. メール確認を完了

登録完了後、ダッシュボードが表示されます。ここがあなたのアプリ開発の拠点になります。

### ステップ 2：新規プロジェクトを作成する

ダッシュボードの「+ New Project」ボタンをクリックします。以下が表示されます。

```
プロジェクト名（例：MyWeatherApp）
説明（オプション）
テンプレート選択
  - Blank（白紙から開始）
  - React Native 基本テンプレート
  - ナビゲーションテンプレート
```

初心者には「Blank」がおすすめです。自由度が高く、AI が最大限に活躍します。

### ステップ 3：効果的なプロンプトを書く

Rork の品質を左右するのがプロンプト（指示文）です。**具体的かつ詳細であるほど、AI がより正確に仕上げてくれます。**

**非効果的なプロンプト：**
```
天気アプリを作成してください。
```

**効果的なプロンプト：**
```
Rork で天気予報アプリを作成してください。

【基本機能】
- ユーザーの現在地を自動取得して天気を表示
- 都市名で検索可能
- 現在の気温、体感温度、湿度、風速を表示
- 5日間の週間予報を表示
- 気温は摂氏・華氏を切り替え可能

【デザイン】
- 天気の状態（晴れ・曇り・雨・雪）に応じて背景色が自動変化
- 大きな天気アイコンを中央に配置
- カード形式で詳細情報を整理
- 下部タブナビゲーション（現在の天気 / 週間予報 / 設定）
- シンプルで直感的なデザイン

【技術要件】
- OpenWeatherMap API を使用
- AsyncStorage でお気に入り都市を保存
- エラーハンドリング完備
```

このプロンプトを入力すれば、Rork がアプリの基本構造、UI、ロジックを自動生成してくれます。

### ステップ 4：生成されたアプリを確認する

生成が完了すると、以下のように画面に表示されます。

```
プロジェクトツリー（左側）
├── screens/
│   ├── HomeScreen.tsx
│   ├── ForecastScreen.tsx
│   └── SettingsScreen.tsx
├── components/
│   ├── WeatherCard.tsx
│   └── TemperatureToggle.tsx
├── services/
│   └── weatherApi.ts
└── App.tsx

ライブプレビュー（中央・右側）
  → iOS / Android シミュレーターで動作確認
```

ここで動作を確認しながら、さらに改善を指示できます。

## UI のカスタマイズと改善

### AI による反復改善

生成されたアプリが 100% 完璧でなくても心配しません。Rork では「チャット形式の指示」でどんどん改善できます。

**例：デザインの改善**
```
天気アイコンをもっと大きくしてください。
背景グラデーションをもっと濃い青にしてください。
フォントを少し大きくして読みやすくしてください。
```

指示を出すと、AI が該当コンポーネントを検出して修正を加えます。新しい画像の出力を見て「イメージ通り」でなければ、さらに指示を重ねます。

**例：機能追加**
```
「最後に確認した時刻」をホーム画面に追加してください。
プッシュ通知で気象警報が出たときにアラートを表示してください。
```

このような反復プロセスを 2～3 回繰り返せば、かなり完成度の高いアプリになります。

### コード直接編集も可能

Rork の視覚的エディターで十分ですが、必要に応じてコードにアクセスすることも可能です。「Code」タブをクリックすると、生成された React Native/TypeScript コードが表示され、細かな調整ができます。

```typescript
// services/weatherApi.ts の一例
const API_KEY = 'YOUR_API_KEY';
const BASE_URL = 'https://api.openweathermap.org/data/2.5';

export const fetchWeatherByCity = async (city: string) => {
  const response = await fetch(
    `${BASE_URL}/weather?q=${city}&appid=${API_KEY}&units=metric&lang=ja`
  );
  if (!response.ok) throw new Error('天気データの取得に失敗しました');
  return response.json();
};
```

ただし、ほとんどのユーザーはビジュアルエディターだけで十分に開発を進められます。

## テストとデバッグ

### シミュレーターでのテスト

Rork は iOS・Android 両方のシミュレーター環境を内蔵しています。

1. **iOS シミュレーター** — Mac ユーザー向け、Safari の DevTools 連携可能
2. **Android エミュレーター** — Windows・Mac・Linux 対応

右側の「Preview」パネルで、リアルタイムにアプリの動作を確認できます。スクロールやタップの動きもシミュレートできるため、UI/UX の検証に有効です。

### よくある初期エラーと対処法

| エラー | 原因 | 対処法 |
|------|------|--------|
| API が 401（認証エラー） | API キーが間違っている、または有効化されていない | API キーを再確認。発行後 2 時間待機 |
| 位置情報が取得できない | シミュレーターのロケーション設定が未設定 | シミュレーター設定で位置情報を東京など具体的に設定 |
| 画像が表示されない | 外部画像 URL が無効 | URL を確認、または画像ファイルをアップロード |
| 白い画面が表示される | コンポーネントにエラーがある | 「Console」タブでエラーメッセージを確認 |

### 実機でのテスト（重要！）

シミュレーターでの動作確認後、**必ず実機でテストしてください。** タッチの反応速度、スクロール、ネットワーク接続など、シミュレーターでは再現されない問題が浮かぶことがあります。

Rork は Expo ベースのため、以下の方法で実機テストができます。

**方法 1：Expo Go アプリ（最も簡単）**
- iOS・Android スマートフォンに「Expo Go」アプリをインストール
- Rork ダッシュボードから QR コード生成
- スマートフォンでスキャン → アプリがインストールされる

**方法 2：TestFlight（iOS）/ Play Console（Android）**
- ビルド → TestFlight・Beta 版として配布
- テスター数人に事前に試してもらう

## App Store / Google Play への公開

### 準備：Apple Developer / Google Play Developer アカウント

| プラットフォーム | 登録料 | 有効期限 | 必要なもの |
|---|---|---|---|
| Apple Developer Program | $99/年 | 1年間 | Apple ID（日本国籍のみ）、クレジットカード |
| Google Play Developer | $25（一度きり） | 無期限 | Google アカウント、クレジットカード |

**Apple の手続きは複雑です。** 身分確認や Apple Business Register への登録が必要な場合があります。計画的に 2～3 週間前から準備を開始しましょう。

### Rork Pro・Max の場合：2クリック公開

Rork Pro 以上なら、ダッシュボードの「Publish」ボタンで以下のフローが始まります。

```
1. ストア掲載情報の入力
   - アプリ名（最大 30 文字）
   - 説明（最大 4,000 文字）
   - スクリーンショット 2～8 枚（1170 × 2532 px）
   - プライバシーポリシー URL
   - サポートメール

2. ビルド生成
   - Rork が自動的に iOS（.ipa）・Android（.aab）をビルド
   - 約 5～10 分

3. ストアへの申請
   - Rork Max なら自動申請
   - Pro なら手動で Apple・Google のコンソールにアップロード
```

### 審査通過のコツ

Apple・Google の審査に落ちないための注意点：

1. **プライバシーポリシーは必須** — 個人情報やアクセス許可（位置情報など）について明記
2. **不適切なコンテンツを避ける** — 暴力・差別・児童虐待関連は絶対禁止
3. **著作権・商標に注意** — 有名企業のロゴ無断使用は避ける
4. **テスター情報が正確に** — Apple の場合、テスト用のダミーアカウントを用意

**平均的な審査期間** — Apple は 1～3 日、Google は 2～4 時間で結果が出ます。

## よくある質問

### Q. Rork で作ったアプリは、本当に App Store で承認されますか？

はい。Rork が生成するコードは、Apple・Google の公式ガイドラインに準拠しています。多くのユーザーがすでに Rork で作成したアプリを公開・収益化しています。ただし、アプリのロジックやコンテンツが違反していないかどうかは、あなたの責任で確認する必要があります。

### Q. Rork で開発中のアプリを途中で別プラットフォーム（Flutter など）に乗り換えられますか？

可能です。Rork から「Export」機能でコード一式をダウンロードできるため、ローカル環境で Continue することもできます。ただし、その後の保守は自分で行う必要があります。Rork のメリット（AI による快速更新）が失われるため、あまりおすすめはしません。

### Q. 複数のアプリを開発・公開したい場合、どのプランが最適ですか？

Rork Pro で複数アプリの開発・公開は十分可能です。ただし、複数アプリの同時保守、優先サポートが欲しい場合は Rork Max が便利です。Max は実質「チーム向け」と位置づけられており、月額の追加でチームメンバーを招待できます。

### Q. Rork のアプリって、処理速度は大丈夫ですか？

Rork は React Native で生成するため、ネイティブアプリ（Swift・Kotlin）と比べるとやや遅い傾向です。ただし、ユーザーが気付く程度の遅延（100ms 以上）が生じるのはごくまれです。ほとんどのアプリは「気になる遅さ」を感じさせません。

### Q. Rork で作ったアプリの所有権は誰にありますか？

**あなたです。** Rork はツール・プラットフォームにすぎず、生成されたコード・アプリ・収益はすべてあなたのものです。Rork が突然サービス停止しても、エクスポートしたコードでビジネスを続けられます。

本記事では、Rork を使ったアプリ開発の全体像をご説明しましました。
