## はじめに：バイブコーディングとの出会い

「アプリ開発なんて、自分にはできない」

多くの非エンジニアはそう思っていますよね。私も最初はそうでした。50代の主婦で、プログラミング知識ゼロ。スマートフォンのアプリなんて使うだけで、作るなんて考えたこともありませんでした。

でも、あることがきっかけで、その常識が変わります。

それが **バイブコーディング（Vibe Coding）** という考え方との出会いです。

Rorkが提唱する「ユーザーの気持ち（ビブ）に寄り添う開発」という概念は、エンジニアの専門知識ではなく、**日常の工夫や想像力で成り立つ**もの。実はこれ、非エンジニアこそが得意な領域なんです。

この記事では、私のアプリ開発初体験を通じて、バイブコーディングの本質とRorkでの実装方法を、実践的にお伝えします。

---

## バイブコーディングとは？ Rorkが最適な理由

### バイブコーディングの定義

バイブコーディングは、「ユーザーの気持ちを感じ取り、そのビブ（雰囲気・感覚）に合わせた開発」を意味します。

- **従来のコーディング**: 仕様書 → 実装 → テスト（左脳駆動）
- **バイブコーディング**: ユーザーの心理 → 体験設計 → 実装（直感駆動）

つまり、「プログラミングの構文知識」よりも、「ユーザーの気持ちをどう汲み取るか」が重要なのです。

### なぜRorkが非エンジニアに最適か

Rorkがバイブコーディングに最適な理由は、3つあります：

**1. 自然言語でアプリが作れる**
プログラミング言語を学ぶ必要がありません。日本語で「タスク管理アプリが欲しい」と説明するだけで、AIがコードを生成してくれます。

**2. ビジュアルフィードバックがリアルタイム**
「このボタンもっと大きくして」「色を変えてほしい」という感覚的な要望をすぐに反映できます。左脳ではなく、右脳で設計できるんです。

**3. 試行錯誤が簡単**
修正や改善が迅速。だからこそ、ユーザー心理を何度も検証しながら、本当に欲しいアプリへと進化させられます。

これが、バイブコーディングとRorkの親和性です。

## 実践：タスク管理アプリを作る手順

では、実際に私が作ったタスク管理アプリ「MyTask」の開発過程を、ステップごとにご紹介します。

### Step 1: アイデアを感覚で言葉にする

まず大切なのは、「自分が何を欲しいのか」を言葉にすること。

私の場合：
> 「毎日のやることを、朝にさっと入力して、終わったらタップで消せるアプリが欲しい。色もかわいく、一目で今日やることが見える感じで」

これだけです。専門用語は一切ありません。これが **バイブコーディングの第一歩** です。

### Step 2: Rorkでプロンプトを書く

Rorkのプロンプト例：

```
タスク管理アプリ「MyTask」を作ってください。

【必須機能】
- タスクを入力して追加できる
- 終わったタスクをスワイプまたはタップで削除
- 追加したタスクが上から順に表示される

【デザイン】
- メインカラーはソフトなピンク（#FFB6C1）
- フォントは丸ゴシック系で親しみやすく
- ボタンは大きめで、指でタップしやすいサイズ

【感覚的な要望】
- 見た感じ「優しい」「親切」という雰囲気を大切に
- 入力欄は常に見えていて、すぐにタスクを追加できるUI
- 完了時のアニメーションは、「ふわり」と消える感じで
```

このプロンプトのコツは、**技術用語を避け、自分の気持ちを率直に書くこと**。AIは「ふわり」という表現から、スプリングアニメーションを自動で提案してくれます。

### Step 3: AIの提案を見て、感覚で調整する

Rorkがコードを生成したら、以下を確認します：

- **見た目**: 本当に「優しい」感じがするか？
- **操作性**: ストレスなくタスクを入力・削除できるか？
- **速度**: もたつかないか？

感じることができれば、「もっと〇〇に」という修正リクエストを出すだけ。エンジニア的な説明は不要です。

```
「タスクを入力する欄の背景色を、もっと白に近い方が見やすいです」
```

こういった感覚的なフィードバックで、どんどん理想形に近づきます。

### Step 4: 機能を足していく

基本ができたら、次の機能を追加します：

**追加機能の例：**
- 優先度の色分け（赤=緊急、黄=普通、緑=気楽に）
- 完了したタスク数を画面上部に表示
- 夜間モード（黒背景で目が疲れにくい）

これらも、プロンプトで素朴に説明するだけ：

```
完了したタスク数を「5個完了した！」と大きく表示してください。
完了ボタンを押したときに、花火のようなアニメーションが出たら嬉しいです。
```

感覚と言葉で十分です。

## プロンプトの書き方：非エンジニアが意識すべきこと

バイブコーディングを成功させるには、プロンプトの質が重要です。以下のポイントを押さえましょう。

### 1. 「感覚」を言葉にする癖をつける

- ❌ 「レスポンシブデザインを実装してください」
- ✅ 「スマホでもタブレットでも見やすく、どちらでも快適に使える感じで」

専門用語を避け、自分なら何と言うかで表現します。

### 2. ユーザーの気持ちを起点に考える

- ❌ 「非同期処理を実装」
- ✅ 「ボタンをタップしてからアプリが反応するまで、待ち時間を感じさせないようにしたい」

結果ではなく、ユーザー体験を説明すること。AIはそこから最適な実装方法を判断します。

### 3. 矛盾を恐れない

「この部分は優雅に、あの部分は機能的に」といった一見矛盾した要望も大丈夫。それがバイブコーディングです。

```
トップのヘッダーは、まるで空を見上げるような壮大さで。
でも、タスク一覧はシンプルで、余計な装飾なしで。
```

こうした「感覚の対比」こそが、奥行きのあるUXを生みます。

## 実例：プロンプト→調整→完成の流れ

以下は、実際の対話ログです。

**1回目のプロンプト：**
```
シンプルなタスク管理アプリを作ってください。
- 入力欄、追加ボタン、タスク一覧
- ピンク系のかわいいデザイン
- タスクは上から順に表示
```

**AIからの提案を見た感想：**
「悪くないけど、なんか冷たい感じ。もっと温かみが欲しい」

**2回目のリクエスト：**
```
全体的に「温かい」感じにしてほしいです。
- 背景は白ではなく、薄いベージュに
- ボタンのコーナーを丸くする
- フォントも丸ゴシックで、やさしい雰囲気で
```

**改良後の感想：**
「いいですね！でも、タスク数を数えるのが大変です」

**3回目のリクエスト：**
```
完了したタスクの数を自動で数えて、画面上部に表示してください。
「あと3個！」みたいな励ましの言葉も一緒に出たら、やる気が出そう。
```

このように、何度も小さく修正しながら、理想のアプリへと仕上げていきます。これが **バイブコーディングの本質** なのです。

## App Store公開まで：非エンジニアが進めるステップ

アプリが完成したら、世界に公開できます。これも意外とシンプルです。

### 1. Rorkでのビルド

Rork内で「ビルド」ボタンを押すだけ。複雑な設定は不要です。

### 2. App Store Connect への登録

Apple IDがあれば、App Store Connectに登録できます。
- アプリ名
- スクリーンショット（Rorkで作った画面をそのまま撮影）
- 説明文（「タスク管理が簡単になるアプリです」程度でOK）
- プライバシーポリシー（テンプレートが用意されているものを使用）

### 3. 審査に出す

Appleの審査ガイドラインに従っているか確認して、提出。

Rorkで作ったアプリは基本的にガイドラインを満たしているため、大きな問題は発生しにくいです。

### 4. 承認待ち＆リリース

通常1〜3日で審査結果が来ます。承認されたら「リリース」ボタンで公開。

これだけです。50代の主婦でも、全て自分でできました。

## よくある質問：非エンジニアが感じるモヤモヤ

### Q1: 「プロンプトが上手くいかないときは？」

**A**: 完璧を目指さないこと。むしろ、「ダメ出し」の方が簡単です。

- 「ここ違う」と言う
- 「もっと〇〇に」と言う
- 何度も試す

この繰り返しで、AIとユーザーが共に創作するプロセスが始まります。

### Q2: 「ちゃんとしたアプリになるの？」

**A**: はい、なります。Rorkで作ったアプリは、世界中のユーザーが使用しているスマートフォンで、プロのアプリと遜色なく動作します。

感覚で設計しても、背景にはしっかりとした工学があります。

### Q3: 「セキュリティとか個人情報とか、大丈夫？」

**A**: Rorkが基本的なセキュリティ実装と業界標準を満たしているため、非エンジニアでも安全なアプリが作れます。

ただし、「ユーザーの信用を保つ」という感覚は大事。プライバシーポリシーは正確に、ユーザーの質問には丁寧に対応すること。これは、プログラミングではなく、人としての誠実さです。

### Q4: 「何か法律的な問題とかある？」

**A**: アプリの内容に寄ります。例えば：

- **金融系のアプリ**: 金融ライセンスが必要な場合がある
- **医療・健康アプリ**: 医学的な正確さが求められる
- **位置情報を使うアプリ**: プライバシー法への対応が必要

ただ、シンプルなタスク管理アプリなら、ほぼ問題ありません。不安な場合は、弁護士やアプリ業界の専門家に相談することをお勧めします。

### Q5: 「アップデートや修正はどうする？」

**A**: Rorkで修正して、再度ビルド＆App Storeに「新バージョン」として提出するだけ。

ユーザーは自動で最新版にアップデートされます。

### Q6: 「儲かるの？」

**A**: その人の工夫次第です。

私のタスク管理アプリは、基本無料で公開していますが、「Pro版」として有料アップグレードも提供しています（月額150円程度）。

感覚的に「このアプリ、良いな」と思ったユーザーの一部が課金してくれます。それが収入になります。

## バイブコーディングが変えたもの

アプリ開発に挑戦してから、私の人生観が変わりました。

**Before**: 「デジタルは難しい、エンジニア向け」
**After**: 「自分の気持ちを言葉にすれば、どんなアプリでも作れる」

さらに嬉しいことは、これが **年齢や学歴と関係ない** ということです。

50代の主婦、70代の定年後の方、子育て中のお母さん。皆さんが、自分の「欲しい」を形にできる時代が来ました。

それを可能にするのが、バイブコーディングとRorkという道具です。

---

## まとめ：今日から始めるアプリ開発

非エンジニアがアプリを開発することは、もう夢ではありません。

1. **感覚を大切にする**：完璧な説明書より、「こんな感じ」という気持ちが優先
2. **AIとの対話を楽しむ**：修正と提案の繰り返しで、作品が成熟する
3. **自分のアイデアを信じる**：プログラミング知識がなくても、あなたの工夫と想像力は何物にも代え難い

Rorkとバイブコーディングは、その第一歩を確かなものにしてくれます。

あなたも、今日からアプリ開発者です。
