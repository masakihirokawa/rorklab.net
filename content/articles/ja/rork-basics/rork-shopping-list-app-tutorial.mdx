## はじめに — 買い物リストアプリで学ぶアプリ開発の基本

日常で最もよく使われるアプリのひとつが「買い物リスト」です。シンプルに見えるこのアプリには、データの追加・編集・削除・表示（CRUD操作）というアプリ開発の基礎がすべて詰まっています。

この記事では、Rorkを使って実用的な買い物リストアプリを一から作る方法をご紹介します。プログラミング経験がなくても大丈夫。Rorkなら自然言語でアプリの仕様を伝えるだけで、動くアプリが手に入ります。

この記事で学べること:

- Rorkでリスト表示アプリを構築する流れ
- アイテムの追加・編集・削除・チェック機能の実装
- カテゴリごとにアイテムを分類する方法
- AsyncStorageを使ったローカルデータ保存
- UIを整えてストア公開に近づけるコツ

対象読者は、Rorkをこれから触り始める方や、初めてのアプリ開発に挑戦したい方です。すでにRorkの基本操作に慣れている方は、[30分でアプリ完成！Rork超入門ガイド](/articles/rork-basics/rork-first-app-30min)もあわせてご覧ください。

## 前提知識と環境準備

### 必要なもの

買い物リストアプリを作り始める前に、以下を準備しましょう。

- **Rorkアカウント**: [rork.com](https://rork.com/) で無料登録できます。無料プランでも週5回までアプリを生成できます
- **ブラウザ**: Chrome、Safari、Edgeなどモダンブラウザがあれば十分です
- **スマートフォン（任意）**: Rork Companionアプリを入れれば実機で動作確認できます

### Rorkの基本的な使い方

Rorkでのアプリ開発は、チャット形式でAIにアプリの仕様を伝えるだけです。「こんなアプリが欲しい」と書くと、AIがReact Nativeベースのコードを自動生成してくれます。

まだRorkを使ったことがない方は、先に[IT知識ゼロからアプリ開発](/articles/rork-basics/rork-zero-to-app-complete-guide)の記事で基本操作を確認しておくと、この先の手順がスムーズに進みます。

## ステップ1: プロジェクトを作成する

Rorkにログインしたら、新しいプロジェクトを作成します。プロンプト入力欄に以下のような指示を入力してみましょう。

```text
買い物リストアプリを作ってください。

機能:
- アイテムを追加できる（名前、数量、カテゴリを入力）
- アイテムをタップしてチェック済みにできる
- スワイプでアイテムを削除できる
- カテゴリ（野菜、肉・魚、乳製品、飲み物、日用品、その他）でフィルタリングできる
- データはローカルに保存される

デザイン:
- 背景は白基調のクリーンなデザイン
- カテゴリごとに色分けされたバッジ
- チェック済みアイテムは取り消し線で表示
- 画面下部にフローティングの追加ボタン
```

このプロンプトだけで、Rorkは画面構成からデータ管理まで含めた完成度の高いアプリを生成してくれます。

## ステップ2: アイテム追加機能を改善する

Rorkが生成した初期バージョンを確認したら、追加機能をさらに便利にしていきましょう。チャットで以下のように指示します。

```text
アイテム追加のモーダルを改善してください。

- 名前の入力欄にオートコンプリート機能を追加
  （過去に入力したアイテム名を候補として表示）
- 数量はプラス/マイナスボタンで変更できるようにする
- カテゴリはアイコン付きのボタンで選択できるようにする
- 追加ボタンを押したら入力欄をクリアしてすぐ次のアイテムを追加できるようにする
```

このように、段階的に機能を追加していくのがRorkでの開発のコツです。一度に完璧なアプリを作ろうとするよりも、まず基本形を作ってから少しずつ改善するほうが、期待通りの結果が得られやすくなります。

### オートコンプリートの仕組み

Rorkが生成するオートコンプリート機能は、過去の入力履歴をAsyncStorageに保存し、テキスト入力時にフィルタリングして候補を表示する仕組みです。

```typescript
// 入力履歴の保存と取得（Rorkが自動生成する例）
import AsyncStorage from '@react-native-async-storage/async-storage';

const HISTORY_KEY = '@shopping_history';

// 履歴に新しいアイテム名を追加
const addToHistory = async (itemName: string) => {
  const history = await getHistory();
  // 重複を除いて先頭に追加（最大50件）
  const updated = [itemName, ...history.filter(h => h !== itemName)].slice(0, 50);
  await AsyncStorage.setItem(HISTORY_KEY, JSON.stringify(updated));
};

// 履歴から候補を検索
const searchHistory = async (query: string): Promise<string[]> => {
  const history = await getHistory();
  return history.filter(item =>
    item.toLowerCase().includes(query.toLowerCase())
  );
};
```

```text
画面上部に横スクロールのカテゴリフィルターを追加してください。

- 「すべて」タブと各カテゴリのタブを表示
- 選択中のカテゴリはハイライト表示
```

## ステップ4: データの永続化を確認する

買い物リストアプリで最も大切なのは、アプリを閉じてもデータが消えないことです。RorkはデフォルトでAsyncStorageを使ったローカル保存を実装してくれますが、念のため動作を確認しておきましょう。

```text
データの保存について確認・改善してください。

- アプリを閉じて再起動してもリストが残ることを確認
- データの保存は自動で行う（手動保存ボタンは不要）
- エラーが起きた場合は「保存に失敗しました」とトースト表示する
```

### AsyncStorageの基本パターン

Rorkが生成するデータ永続化のコードは、以下のようなパターンが一般的です。

```typescript
// データの保存（Rorkが自動生成する例）
const STORAGE_KEY = '@shopping_list';

const saveItems = async (items: ShoppingItem[]) => {
  try {
    await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(items));
  } catch (error) {
    console.error('Failed to save items:', error);
    // トースト通知を表示
    Alert.alert('エラー', '保存に失敗しました');
  }
};

// データの読み込み（アプリ起動時）
const loadItems = async (): Promise<ShoppingItem[]> => {
  try {
    const json = await AsyncStorage.getItem(STORAGE_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('Failed to load items:', error);
    return [];
  }
};
```

## よくある質問

### Q: 買い物リストのデータをクラウドに保存して複数端末で共有できますか？

Rorkで生成した初期バージョンではAsyncStorage（端末ローカル）にデータが保存されます。複数端末で共有したい場合は、Supabaseなどのバックエンドを追加する指示をRorkに出すことで実現できます。「Supabaseと連携して、ログインしたユーザーのリストをクラウドに同期してください」と追加指示するのがおすすめです。

### Q: アプリをApp StoreやGoogle Playに公開できますか？

はい、Rork Maxプランを使えばApp StoreやGoogle Playへの申請が可能です。無料プランでもRork Companionアプリを通じて自分のiPhoneでテストできます。ストア公開の手順については、Rorkの公式ドキュメントも参考にしてください。

### Q: カテゴリを自分で追加・編集できるようにするにはどうすればいいですか？

Rorkに「カテゴリを設定画面で追加・編集・削除できるようにしてください。カスタムカテゴリには好きな色とアイコンを設定できるようにしてください」と指示するだけで、設定画面付きのカテゴリ管理機能を追加できます。

## まとめ — 小さなアプリから始めて、開発スキルを積み上げよう

買い物リストアプリの開発を通じて、以下のスキルが身についたはずです。

- **CRUD操作**: データの作成・読み取り・更新・削除の基本パターン
- **状態管理**: Reactの`useState`を使ったリストデータの管理
- **ローカル保存**: AsyncStorageによるデータの永続化
- **UIパターン**: リスト表示、モーダル、フィルタリング、スワイプ操作

こうした基本パターンは、ToDoアプリや[メモアプリ](/articles/rork-basics/build-todo-app)など他のアプリ開発にも応用できます。まずは今回のアプリを自分好みにカスタマイズしてみてください。カテゴリの追加、テーマカラーの変更、共有機能の追加など、アイデア次第でどんどん発展させていけます。

Rorkでのアプリ開発は、小さなプロジェクトの積み重ねが大きな成長につながります。ぜひ次のアプリにも挑戦してみてください。
