# Rorkで家計簿アプリを作る完全ガイド：支出管理アプリの開発チュートリアル

毎月の支出を把握したいけれど、既存の家計簿アプリは機能が多すぎて使いこなせない——そんな経験はありませんか？**Rork**を使えば、自分好みの家計簿アプリをAIプロンプトだけで開発できます。コーディング経験がなくても、カテゴリ分類やグラフ表示を備えた本格的な支出管理アプリが30分ほどで完成します。

この記事では、Rorkを使って家計簿アプリをゼロから構築する手順を、実際のプロンプト例やコード例とともに詳しく解説します。

## この記事で学べること

このチュートリアルを完了すると、以下の機能を持つ家計簿アプリが完成します。

- 支出・収入の記録と編集
- カテゴリ別の自動分類（食費、交通費、娯楽費など）
- 月ごとの支出グラフ（円グラフ・棒グラフ）
- 月次サマリーレポート
- ローカルストレージによるデータ永続化

対象読者は、Rorkの基本操作を一度でも触ったことがある方です。まだRorkを使ったことがない方は、先に[Rork入門ガイド](/articles/rork-basics/getting-started)を読むことをおすすめします。

## 前提知識・環境準備

チュートリアルを始める前に、以下を準備してください。

- **Rorkアカウント**: 無料プランでも基本的な開発は可能です。プランの詳細はRork料金プランの徹底解説を参照してください。
- **ブラウザ**: Chrome、Safari、EdgeなどモダンブラウザがあればOKです。
- **アプリの完成イメージ**: どんな支出カテゴリが必要か、大まかに考えておくとプロンプト作成がスムーズになります。

## Step 1: プロジェクトの作成とベースUI

Rorkにログインしたら、新しいプロジェクトを作成します。最初のプロンプトでアプリ全体の骨組みを指示しましょう。

### 初回プロンプト例

```text
家計簿アプリを作ってください。
以下の画面構成にしてください：

1. ホーム画面：今月の支出合計と直近の取引リスト
2. 入力画面：金額、カテゴリ、メモ、日付を入力するフォーム
```

このプロンプトだけで、Rorkは4つの画面を持つアプリの基本構造を生成します。タブナビゲーション、各画面のレイアウト、基本的なコンポーネントが自動的に作られるため、最初から整ったUIが手に入ります。

## Step 2: データモデルとストアの設計

家計簿アプリの核となるのはデータの管理です。Rorkにデータ構造を具体的に指示することで、堅牢なストアが生成されます。

### データ構造の指示プロンプト

```text
支出データを以下の構造で管理してください：

- id: ユニークID（UUID）
- amount: 金額（数値）
- category: カテゴリ（食費、交通費、日用品、娯楽、医療、教育、その他）
- note: メモ（任意）
- date: 日付（ISO形式）
- type: "income" または "expense"

Zustandでストアを作り、AsyncStorageでデータを永続化してください。
月ごとのフィルタリングとカテゴリ別集計の関数も用意してください。
```

### 生成されるストアのコード例

Rorkが生成するストアは次のような構造になります。コメントで各関数の役割を説明しています。

```typescript
// stores/expenseStore.ts
import { create } from 'zustand';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface Transaction {
  id: string;
  amount: number;
  category: string;
  note: string;
  date: string;
  type: 'income' | 'expense';
}

interface ExpenseStore {
  transactions: Transaction[];
  addTransaction: (tx: Omit<Transaction, 'id'>) => void;
  getMonthlyTotal: (year: number, month: number) => number;
  getCategoryBreakdown: (year: number, month: number) => Record<string, number>;
}

// Zustandストアの作成
export const useExpenseStore = create<ExpenseStore>((set, get) => ({
  transactions: [],

  // 新しい取引を追加しAsyncStorageに保存
  addTransaction: (tx) => {
    const newTx = { ...tx, id: crypto.randomUUID() };
    set((state) => {
      const updated = [...state.transactions, newTx];
      AsyncStorage.setItem('transactions', JSON.stringify(updated));
      return { transactions: updated };
    });
  },

  // 指定月の支出合計を計算
  getMonthlyTotal: (year, month) => {
    return get()
      .transactions.filter(
        (tx) =>
          tx.type === 'expense' &&
          new Date(tx.date).getFullYear() === year &&
          new Date(tx.date).getMonth() === month
      )
      .reduce((sum, tx) => sum + tx.amount, 0);
  },

  // カテゴリ別の支出内訳を取得
  getCategoryBreakdown: (year, month) => {
    const filtered = get().transactions.filter(
      (tx) =>
        tx.type === 'expense' &&
        new Date(tx.date).getFullYear() === year &&
        new Date(tx.date).getMonth() === month
    );
    return filtered.reduce(
      (acc, tx) => {
        acc[tx.category] = (acc[tx.category] || 0) + tx.amount;
        return acc;
      },
      {} as Record<string, number>
    );
  },
}));

// 期待する出力例:
// getMonthlyTotal(2026, 2) → 85000（3月の支出合計）
// getCategoryBreakdown(2026, 2) → { "食費": 35000, "交通費": 12000, "娯楽": 18000, ... }
```

## Step 3: 入力フォームのUI改善

自動生成されたフォームは基本的な構造を持っていますが、ユーザー体験を向上させるために追加のプロンプトで調整しましょう。

### UIブラッシュアップのプロンプト

```text
入力画面を改善してください：

- 金額入力は数字キーボードを表示し、3桁ごとにカンマ区切り表示
- カテゴリ選択はアイコン付きのグリッドボタンにする（食費🍽️、交通費🚃、日用品🏠、娯楽🎮、医療💊、教育📚、その他📌）
```

ポイントは、UIの細かな挙動（キーボードの種類、デフォルト値、遷移の動作）まで具体的に指示することです。Rorkはこうした細部の指定にも対応し、期待どおりのインタラクションを実装します。

### 生成されるグラフコンポーネントの例

```tsx
// components/MonthlyPieChart.tsx
import { PieChart } from 'react-native-chart-kit';
import { Dimensions } from 'react-native';

const screenWidth = Dimensions.get('window').width;

// カテゴリごとの配色定義
const CATEGORY_COLORS: Record<string, string> = {
  食費: '#FF9800',
  交通費: '#2196F3',
  日用品: '#4CAF50',
  娯楽: '#9C27B0',
  医療: '#F44336',
  教育: '#00BCD4',
  その他: '#607D8B',
};

interface Props {
  breakdown: Record<string, number>;
}

export function MonthlyPieChart({ breakdown }: Props) {
  const data = Object.entries(breakdown).map(([name, amount]) => ({
    name,
    amount,
    color: CATEGORY_COLORS[name] || '#999',
    legendFontColor: '#333',
    legendFontSize: 12,
  }));

  return (
    <PieChart
      data={data}
      width={screenWidth - 32}
      height={220}
      chartConfig={{
        color: (opacity = 1) => `rgba(0, 0, 0, ${opacity})`,
      }}
      accessor="amount"
      backgroundColor="transparent"
      paddingLeft="15"
    />
  );
}

// 期待する出力: カテゴリ別の色分け円グラフが描画される
// 食費が最も大きい場合、オレンジのセグメントが最大になる
```

## Step 5: データの永続化と復元

アプリを閉じてもデータが消えないように、AsyncStorageとの連携を確認しましょう。

### 永続化の確認プロンプト

```text
アプリ起動時にAsyncStorageからデータを読み込む処理を追加してください。
- アプリ起動時に "transactions" キーからデータを復元
- データが存在しない場合はサンプルデータ3件をセット
- 読み込み中はスプラッシュスクリーンを表示
- エラー時はトースト通知で「データの読み込みに失敗しました」と表示
```

これにより、Rorkはアプリのライフサイクルに合わせたデータ管理コードを生成します。初回起動時にはサンプルデータが表示されるため、ユーザーはすぐにアプリの使い方を理解できます。

## よくあるエラーと対処法

家計簿アプリの開発中に起こりやすい問題と、その解決策をまとめます。

**グラフが表示されない場合**: `react-native-chart-kit` がインストールされていない可能性があります。Rorkのプロンプトで「react-native-chart-kit をインストールしてグラフを表示して」と追加指示を出してください。

**金額が正しくフォーマットされない場合**: `toLocaleString('ja-JP')` を使った通貨フォーマットが環境によって動作しないことがあります。独自のフォーマット関数を用意するよう指示しましょう。

**AsyncStorageのデータが消える場合**: Expoの開発モードではキャッシュクリア時にデータが消えることがあります。本番ビルドでは問題ありません。長期的にはSupabaseなどのクラウドデータベースへの移行を検討してください。詳しくはSupabaseを使ったチャットアプリの構築の記事が参考になります。

## 応用：収益化とクラウド対応

基本的な家計簿アプリが完成したら、次のステップとして収益化やクラウド対応を検討しましょう。

**サブスクリプションモデルの導入**: プレミアム機能（CSV/PDFエクスポート、予算設定アラート、複数口座管理）を有料プランとして提供できます。Rork MaxとStripeを組み合わせた課金の実装方法は[Rork Max + Stripeでサブスクリプション課金を実装する方法](/articles/rork-dev/rork-max-stripe-subscription)で解説しています。

**Supabaseによるクラウド同期**: AsyncStorageだけではデバイス間でデータを同期できません。Supabaseを導入すれば、複数デバイスからの支出入力やリアルタイム同期が可能になります。

**App Storeへの公開**: 完成したアプリはRork Maxの2クリックパブリッシュ機能で簡単にApp Storeに公開できます。公開手順の詳細は[App Store公開ガイド](/articles/rork-dev/app-store-publishing)を参照してください。

## FAQ

### Q: Rorkの無料プランで家計簿アプリは作れますか？

はい、無料プランでも基本的な家計簿アプリは作成できます。ただし、無料プランではプロンプト回数に制限があるため、複雑な機能の追加には有料プランへのアップグレードが必要になる場合があります。

### Q: グラフ表示以外のライブラリを使うことはできますか？

Rorkは `react-native-chart-kit` のほかに、`victory-native` や `react-native-svg-charts` などにも対応しています。プロンプトで使いたいライブラリ名を指定すれば、そのライブラリを使ったコードが生成されます。

### Q: 作った家計簿アプリにログイン機能を追加するにはどうすればいいですか？

「Firebase Authを使ったメールアドレス認証を追加してください」のようにプロンプトで指示するだけで、Rorkが認証フローを自動生成します。Google・Appleのソーシャルログインも同様に対応可能です。認証の詳しい実装方法は[Rork認証ガイド：Firebase & Supabase連携](/articles/rork-dev/rork-auth-firebase-supabase)を参照してください。

### Q: iPhoneとAndroidの両方に対応できますか？

Rorkが生成するアプリはReact Native（Expo）ベースなので、iOSとAndroidの両方で動作します。Rork Maxを使えばネイティブSwiftアプリとしてiOS向けに特化した開発も可能です。
