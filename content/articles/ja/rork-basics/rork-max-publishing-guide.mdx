Rork Max を使ってアプリを作成したら、次のステップは **本番環境への公開** です。このガイドでは、開発したアプリを実際のユーザーに届けるまでの全工程をわかりやすく解説します。

## Rork Max での公開とは

Rork Max（旧 Rork AI）は、ノーコードで Web アプリケーションを構築できるプラットフォームです。完成したアプリを公開（パブリッシュ）することで、誰でもアクセスできる本物の Web サービスになります。

**Rork Max で公開できるアプリの例:**
- 顧客管理システム（CRM）
- 在庫管理ツール
- データ分析ダッシュボード
- コンテンツ配信プラットフォーム
- 個人ポートフォリオサイト

## Rork Max の3つの公開方法

### 1. クイック公開（推奨：初心者向け）

もっとも簡単な方法です。

**手順:**
1. Rork Max エディタを開く
2. 画面右上の「**公開**」ボタンをクリック
3. 確認ダイアログで「**公開する**」を選択
4. 自動生成された URL が表示される
5. その URL をシェアすれば、ユーザーがアクセス可能に

**メリット:**
- セットアップが不要
- 1クリックで公開完了
- Rork のサーバーで自動ホスティング

**デメリット:**
- カスタムドメイン不可（`rorkapp.io` というサブドメイン）
- SEO対策が限定的

### 2. カスタムドメイン公開（推奨：本格運用向け）

独自ドメイン（例：`myapp.com`）でアプリを公開する方法です。

**前提条件:**
- 独自ドメインを取得済み（GoDaddy, ムームードメイン等）
- Rork Max Pro 以上のプラン

**手順:**

```markdown
**Step 1: ドメインをRork Maxに追加**
1. Rork Max ダッシュボード → 「設定」→「カスタムドメイン」
2. 「ドメインを追加」をクリック
3. ドメインを入力（例：myapp.com）

**Step 2: DNS設定を確認**
- Rork Max が表示する CNAME レコード（例: `cname.rorkapp.io`）をコピー
- ドメイン提供事業者（GoDaddy等）のDNS管理画面を開く
- 新しいレコードを追加:
  - タイプ: CNAME
  - 名前: @ または www
  - 値: cname.rorkapp.io
- 設定が反映されるまで数時間待つ

**Step 3: SSL証明書を発行**
- Rork Max が自動的に Let's Encrypt で SSL証明書を発行
- 数分～数時間で有効化
```

**メリット:**
- プロフェッショナルな見た目
- SEO効果あり
- ブランド信頼性が向上

**デメリット:**
- ドメイン取得費用（年千円～数千円）
- DNS設定に技術的知識が必要
- 反映に時間がかかることがある

### 3. セルフホスティング（上級者向け）

自分のサーバー（AWS, Heroku, Vercel等）でアプリをホストする方法です。

**対応サービス:**
- Vercel（推奨：Next.js ベース）
- Netlify
- AWS Amplify
- Heroku

**メリット:**
- 完全な制御が可能
- スケーラビリティが高い
- カスタマイズが自由

**デメリット:**
- 技術的難易度が高い
- サーバー維持費がかかる
- 管理の手間が大きい

## 公開前のチェックリスト

公開する前に、以下を確認しましょう。

```markdown
- [ ] アプリが正常に動作するか（全ページ・全機能を確認）
- [ ] モバイル（スマホ）でも見やすいか
- [ ] エラーメッセージが表示されていないか
- [ ] リンクが正しく機能するか
- [ ] ログイン機能がある場合、テストアカウントで確認
- [ ] 読み込み速度が極度に遅くないか
- [ ] 個人情報や秘密情報が含まれていないか
```

## よくあるトラブルと対処法

### トラブル 1: カスタムドメイン設定後もアクセスできない

**原因:** DNS設定が反映されていない

**対処法:**
- 数時間〜24時間待つ
- ブラウザキャッシュをクリア（Ctrl+Shift+Delete）
- 別のブラウザで試す
- `nslookup myapp.com` で DNS設定を確認

### トラブル 2: 「403 Forbidden」エラーが表示される

**原因:** アクセス権限設定が不適切

**対処法:**
1. Rork Max 設定で「公開範囲」を確認
2. 「全員に公開」に変更
3. キャッシュをクリアして再度アクセス

### トラブル 3: 画像が表示されない

**原因:** 画像 URL が相対パスになっている

**対処法:**
- Rork Max でアップロード済みの画像を使用
- 外部 URL の画像は HTTPS に対応させる

## 公開後のベストプラクティス

### 1. パフォーマンス監視

```markdown
定期的に以下を確認:
- ページ読み込み時間（Google PageSpeed Insights）
- エラーログ（Rork Max ダッシュボード）
- ユーザーアクセス数（Google Analytics 連携）
```

### 2. セキュリティ対策

```markdown
- ユーザー認証を有効化
- 機密データは環境変数に保存（ハードコード禁止）
- 定期的にバックアップを取得
- SSL/HTTPS が有効になっているか確認
```

### 3. アナリティクス設定

Google Analytics や Rork Max 標準アナリティクスで以下を追跡:
- ユーザー数とアクティブ状況
- どのページが人気か
- ユーザーがどこで離脱するか
- コンバージョン率（会員登録、購入等）

### 4. 定期的なアップデート

ユーザーフィードバックをもとに:
- UI/UX の改善
- バグの修正
- 新機能の追加

## FAQ

**Q1: 公開したアプリを非公開に戻せますか？**

A: はい。Rork Max ダッシュボード → 設定で「非公開」に変更すれば、アクセス不可になります。

**Q2: 無料プランでも公開できますか？**

A: はい。ただし、クイック公開のみで、カスタムドメインは Pro プラン以上が必要です。

**Q3: 公開後にアプリを編集しても大丈夫ですか？**

A: 大丈夫です。編集して「更新」をクリックすれば、公開中のアプリもリアルタイムで更新されます。

**Q4: アクセス数に制限はありますか？**

A: Rork Max のプランによって異なります。詳細は[料金ページ](https://rork.com/pricing)を参照。

**Q5: 複数のドメインでアプリを公開できますか？**

A: はい。Pro プラン以上なら複数のカスタムドメインを設定可能です。
