## 「Fix Now」とは

Rorkでアプリを開発していると、ビルドエラーやランタイムエラーに遭遇することがあります。「Fix Now」は、これらのエラーをAIがワンクリックで自動修正してくれる機能です。

コーディング経験がなくても、エラーの原因を理解していなくても、ボタン一つで問題を解決できるのがFix Nowの最大の魅力です。

## Fix Now が対応するエラーの種類

### ビルドエラー

アプリのビルド時に発生するエラーを自動検出・修正します。

- TypeScript / JavaScript の構文エラー
- インポートパスの間違い
- 型の不一致
- 依存パッケージのバージョン衝突

### ランタイムエラー

アプリの実行中に発生するエラーにも対応します。

- undefined / null 参照エラー
- APIレスポンスの形式不一致
- ナビゲーションの参照エラー
- レンダリングの無限ループ

### デザイン崩れ

見た目の問題にも対応する場合があります。

- レイアウトの崩れ
- スクロールの問題
- 画面サイズへの未対応

## 使い方

### 基本的な流れ

1. エラーが発生すると、画面上部に赤いバナーが表示される
2. バナー内の「Fix Now」ボタンをクリック
3. AIがエラーの原因を分析し、修正コードを生成
4. 修正が適用され、自動的に再ビルドが実行される

### チャットでの詳細指示

Fix Nowだけでは解決しないケースでは、チャットでより詳細な情報を伝えることで修正精度が上がります。

```
Fix Nowでタイマーのエラーが直りませんでした。
タイマーが0になった時にアプリがクラッシュします。
カウントダウンが0に達した時に停止する処理が
抜けているかもしれません。
```

### 連続Fix Now

1回のFix Nowで解決しない場合、複数回実行することもできます。AIは前回の修正結果を踏まえて別のアプローチを試みるため、2〜3回目で解決することも多いです。

## Fix Now の仕組み

Fix Nowは以下のプロセスで動作します。

1. **エラーログの解析**: スタックトレースやエラーメッセージを解析
2. **関連コードの特定**: エラーに関連するファイルとコード箇所を特定
3. **修正案の生成**: AIが修正コードを生成
4. **自動適用とテスト**: コードを修正し、再ビルドして動作確認

## 効果的に使うコツ

### エラーが出たらすぐFix Now

エラーを放置して開発を続けると、問題が複雑化します。エラーが出たらすぐにFix Nowを試すのが効果的です。

### 修正内容を確認する

Fix Nowは自動修正ですが、何が変更されたか確認する習慣をつけましょう。コードエディタで差分を見ることで、同様のエラーを今後防げるようになります。

### チャットとの併用

Fix Nowとチャットを組み合わせるのが最も効果的です。Fix Nowで大まかな修正を行い、細かい調整はチャットで指示するフローがおすすめです。

## Fix Now で解決できないケース

以下のようなケースでは、Fix Nowだけでは解決が難しい場合があります。

- アプリの設計レベルの問題（データフローの根本的な見直しが必要）
- 外部APIの仕様変更
- プラットフォーム固有の制限に起因する問題

これらの場合は、チャットで問題を詳しく説明し、AIに設計の見直しを含めた対応を依頼しましょう。

## まとめ

Fix Nowは、Rorkでのアプリ開発をスムーズに進めるための心強い味方です。エラーに遭遇してもパニックにならず、まずはFix Nowを押してみてください。多くの場合、AIが数秒で問題を解決してくれます。
