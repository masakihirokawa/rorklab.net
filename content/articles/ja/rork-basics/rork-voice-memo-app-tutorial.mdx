## はじめに — 音声メモアプリを Rork で作ってみよう

スマートフォンで最も手軽に情報を記録できる方法の一つが、音声メモです。アイデアが浮かんだとき、会議の内容を記録したいとき、買い物リストを声で残したいとき — テキスト入力よりもはるかに素早く情報を保存できます。

この記事では、**Rork を使って音声メモアプリをゼロから作る方法** を初心者向けに解説します。単なる録音アプリにとどまらず、**録音した音声を自動で文字に起こす機能** まで実装していきます。

### この記事で学べること

- Rork のプロンプトだけで音声メモアプリの基本構造を構築する方法
- Expo AV を使った録音・再生機能の仕組み
- OpenAI Whisper API を活用した音声の文字起こし（トランスクリプション）
- AsyncStorage を使ったメモの保存と管理
- アプリの UI デザインとユーザー体験の工夫

### 対象読者

- Rork を使い始めたばかりの初心者の方
- 音声アプリの開発に興味がある方
- 実用的なアプリのチュートリアルを探している方

Rork の最大の魅力は、自然言語のプロンプトだけでアプリの基本構造を生成できることです。以下のようなプロンプトを入力してみましょう。

```text
音声メモアプリを作ってください。

主な機能:
- 録音ボタンを押すと音声を録音開始、もう一度押すと停止
- 録音した音声をリスト表示
- 各メモをタップすると再生
- メモにタイトルを付けられる
- 録音日時を自動記録
```

このプロンプトを送信すると、Rork が React Native + Expo をベースにしたアプリのコードを自動生成します。録音ボタン、メモ一覧画面、再生コントロールといった基本的な UI が数分で完成します。

## Step 2: 録音機能の仕組みを理解する

Rork が生成するコードの中核となるのが、`expo-av` ライブラリを使った録音機能です。仕組みを理解しておくと、カスタマイズする際に役立ちます。

### 録音の基本フロー

```typescript
import { Audio } from 'expo-av';

// 録音の権限をリクエスト
const { granted } = await Audio.requestPermissionsAsync();

// 録音モードを設定
await Audio.setAudioModeAsync({
  allowsRecordingIOS: true,
  playsInSilentModeIOS: true,
});

// 録音を開始
const { recording } = await Audio.Recording.createAsync(
  Audio.RecordingOptionsPresets.HIGH_QUALITY // 高品質録音
);

// 録音を停止して URI を取得
await recording.stopAndUnloadAsync();
const uri = recording.getURI(); // ファイルパス
```

このコードのポイントは、`Audio.RecordingOptionsPresets.HIGH_QUALITY` を使って高品質な録音を行うことです。用途に応じて `LOW_QUALITY` に切り替えることで、ファイルサイズを抑えることもできます。

### カスタム録音フックの実装例

Rork が生成するカスタムフックをベースに、録音時間の計測機能を追加した例です。

```typescript
// hooks/useAudioRecorder.ts
import { useState, useRef, useCallback } from 'react';
import { Audio } from 'expo-av';

export function useAudioRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const [duration, setDuration] = useState(0);
  const recordingRef = useRef<Audio.Recording | null>(null);
  const timerRef = useRef<NodeJS.Timeout | null>(null);

  // 録音開始
  const startRecording = useCallback(async () => {
    const { granted } = await Audio.requestPermissionsAsync();
    if (!granted) return;

    await Audio.setAudioModeAsync({
      allowsRecordingIOS: true,
      playsInSilentModeIOS: true,
    });

    const { recording } = await Audio.Recording.createAsync(
      Audio.RecordingOptionsPresets.HIGH_QUALITY
    );

    recordingRef.current = recording;
    setIsRecording(true);
    setDuration(0);

    // 録音時間をカウント
    timerRef.current = setInterval(() => {
      setDuration(prev => prev + 1);
    }, 1000);
  }, []);

  // 録音停止
  const stopRecording = useCallback(async () => {
    if (!recordingRef.current) return null;

    if (timerRef.current) clearInterval(timerRef.current);

    await recordingRef.current.stopAndUnloadAsync();
    const uri = recordingRef.current.getURI();

    recordingRef.current = null;
    setIsRecording(false);

    return { uri, duration };
  }, [duration]);

  return { isRecording, duration, startRecording, stopRecording };
}
```

## Step 3: 再生機能を実装する

録音したメモを再生する機能も `expo-av` の `Sound` オブジェクトを使って実装します。

```typescript
// hooks/useAudioPlayer.ts
import { useState, useRef, useCallback } from 'react';
import { Audio } from 'expo-av';

export function useAudioPlayer() {
  const [isPlaying, setIsPlaying] = useState(false);
  const [position, setPosition] = useState(0);
  const [totalDuration, setTotalDuration] = useState(0);
  const soundRef = useRef<Audio.Sound | null>(null);

  // 音声を読み込んで再生
  const play = useCallback(async (uri: string) => {
    // 既存の再生を停止
    if (soundRef.current) {
      await soundRef.current.unloadAsync();
    }

    const { sound } = await Audio.Sound.createAsync(
      { uri },
      { shouldPlay: true },
      (status) => {
        if (status.isLoaded) {
          setPosition(status.positionMillis);
          setTotalDuration(status.durationMillis ?? 0);
          setIsPlaying(status.isPlaying);

          // 再生完了時
          if (status.didJustFinish) {
            setIsPlaying(false);
            setPosition(0);
          }
        }
      }
    );

    soundRef.current = sound;
  }, []);

  // 一時停止 / 再開
  const togglePause = useCallback(async () => {
    if (!soundRef.current) return;
    if (isPlaying) {
      await soundRef.current.pauseAsync();
    } else {
      await soundRef.current.playAsync();
    }
  }, [isPlaying]);

  return { isPlaying, position, totalDuration, play, togglePause };
}
```

### 再生の進捗バーを UI に反映する

再生中の進捗状況をスライダーで表示すると、ユーザー体験が格段に向上します。

```typescript
// 進捗率を計算（0〜1）
const progress = totalDuration > 0 ? position / totalDuration : 0;

// 表示用の時間フォーマット
const formatTime = (ms: number) => {
  const seconds = Math.floor(ms / 1000);
  const min = Math.floor(seconds / 60);
  const sec = seconds % 60;
  return `${min}:${sec.toString().padStart(2, '0')}`;
};
// 期待する出力: "1:05", "0:30" のような形式
```

## Step 4: 文字起こし機能を追加する（Whisper API 連携）

ここからが、この音声メモアプリの最大の差別化ポイントです。録音した音声を **自動的にテキストに変換** する機能を、OpenAI の Whisper API を使って実装します。

### Whisper API の仕組み

Whisper は OpenAI が提供する音声認識モデルで、多言語対応かつ高精度な文字起こしが可能です。音声ファイルを API に送信すると、テキストが返ってきます。

```typescript
// utils/transcribe.ts

const OPENAI_API_KEY = 'your-api-key-here'; // 環境変数で管理推奨

export async function transcribeAudio(uri: string): Promise<string> {
  // 音声ファイルを FormData に変換
  const formData = new FormData();
  formData.append('file', {
    uri,
    type: 'audio/m4a',
    name: 'recording.m4a',
  } as any);
  formData.append('model', 'whisper-1');
  formData.append('language', 'ja'); // 日本語を指定

  // OpenAI API にリクエスト
  const response = await fetch(
    'https://api.openai.com/v1/audio/transcriptions',
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${OPENAI_API_KEY}`,
      },
      body: formData,
    }
  );

  const data = await response.json();
  return data.text; // 文字起こし結果
}

// 使用例:
// const text = await transcribeAudio(recordingUri);
// 期待する出力: "今日の会議では新しいプロジェクトの進捗について話し合いました"
```

### 録音停止時に自動で文字起こしを実行する

録音が完了したタイミングで自動的に文字起こしを行い、メモに紐づけて保存する処理を追加します。

```typescript
const handleStopRecording = async () => {
  const result = await stopRecording();
  if (!result) return;

  // 文字起こしを実行（バックグラウンドで処理）
  const transcription = await transcribeAudio(result.uri);

  // メモとして保存
  const newMemo = {
    id: Date.now().toString(),
    title: transcription.slice(0, 30) + '...', // 先頭30文字をタイトルに
    uri: result.uri,
    duration: result.duration,
    transcription,
    createdAt: new Date().toISOString(),
  };

  await saveMemo(newMemo);
};
```

## Step 5: データの永続化（AsyncStorage）

録音したメモのメタデータは `AsyncStorage` に保存します。音声ファイル自体はデバイスのファイルシステムに保存され、そのパス（URI）をメタデータとして管理します。

```typescript
// utils/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const STORAGE_KEY = 'voice_memos';

// メモ型定義
interface VoiceMemo {
  id: string;
  title: string;
  uri: string;
  duration: number;
  transcription: string;
  createdAt: string;
}

// 全メモを取得
export async function getMemos(): Promise<VoiceMemo[]> {
  const data = await AsyncStorage.getItem(STORAGE_KEY);
  return data ? JSON.parse(data) : [];
}

// メモを保存
export async function saveMemo(memo: VoiceMemo): Promise<void> {
  const memos = await getMemos();
  memos.unshift(memo); // 新しいものを先頭に
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(memos));
}

// メモを削除
export async function deleteMemo(id: string): Promise<void> {
  const memos = await getMemos();
  const filtered = memos.filter(m => m.id !== id);
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(filtered));
}
```

データの永続化についてさらに詳しく知りたい方は、[Rork でメモ・ノートアプリを作る完全チュートリアル](/articles/rork-basics/rork-note-taking-app-tutorial) もあわせてご覧ください。

### 録音ボタンのアニメーション

パルスアニメーションは、ユーザーに「現在録音中」であることを視覚的に伝える重要な UI 要素です。

```typescript
import { useEffect, useRef } from 'react';
import { Animated, Pressable, StyleSheet } from 'react-native';

function RecordButton({ isRecording, onPress }: {
  isRecording: boolean;
  onPress: () => void;
}) {
  const scale = useRef(new Animated.Value(1)).current;

  useEffect(() => {
    if (isRecording) {
      // パルスアニメーション開始
      Animated.loop(
        Animated.sequence([
          Animated.timing(scale, {
            toValue: 1.2,
            duration: 500,
            useNativeDriver: true,
          }),
          Animated.timing(scale, {
            toValue: 1.0,
            duration: 500,
            useNativeDriver: true,
          }),
        ])
      ).start();
    } else {
      scale.setValue(1);
    }
  }, [isRecording]);

  return (
    <Pressable onPress={onPress}>
      <Animated.View
        style={[
          styles.button,
          isRecording && styles.recording,
          { transform: [{ scale }] },
        ]}
      />
    </Pressable>
  );
}

const styles = StyleSheet.create({
  button: {
    width: 80,
    height: 80,
    borderRadius: 40,
    backgroundColor: '#FF3B30',
  },
  recording: {
    backgroundColor: '#FF6B6B',
  },
});
```

## よくあるエラーと対処法

### 録音の権限が拒否される

iOS と Android では、マイクの使用に明示的な権限が必要です。Rork が生成するコードには通常、権限リクエストが含まれていますが、ユーザーが拒否した場合のエラーハンドリングも忘れずに追加しましょう。

```typescript
const { granted } = await Audio.requestPermissionsAsync();
if (!granted) {
  Alert.alert(
    'マイクの権限が必要です',
    '設定アプリからマイクの使用を許可してください。',
    [{ text: 'OK' }]
  );
  return;
}
```

### シミュレータで録音がうまくいかない

iOS シミュレータではマイク入力がサポートされていないことがあります。実機でのテストをおすすめします。Rork Max をお使いの方は、[Rork Companion Mac App](/articles/rork-dev/rork-companion-mac-app) を使うと手軽に実機テストができます。

### 音声ファイルが大きすぎる

長時間の録音はファイルサイズが大きくなります。`Audio.RecordingOptionsPresets.LOW_QUALITY` を使うか、録音時間に上限（例：5分）を設けることを検討してください。

## FAQ

### Q1: Whisper API を使わずに文字起こしはできますか？

はい、可能です。React Native には `expo-speech` を使った音声合成（テキスト→音声）はありますが、音声認識（音声→テキスト）にはネイティブモジュールの `@react-native-voice/voice` を利用できます。ただし、精度と多言語対応の面では Whisper API が優れているため、品質を重視する場合は API の利用をおすすめします。

### Q2: 録音データはクラウドに保存できますか？

はい、Supabase Storage や Firebase Cloud Storage と連携することで、録音データをクラウドに保存できます。バックエンド連携の詳しい手順は [Rorkでバックエンド連携：SupabaseとFirebaseの統合ガイド](/articles/rork-dev/supabase-firebase-backend) をご覧ください。

### Q3: オフライン環境でも使えますか？

録音と再生はオフラインでも動作します。ただし、Whisper API による文字起こしにはインターネット接続が必要です。オフラインでの文字起こしを実現したい場合は、オンデバイスの音声認識モデルを検討してください。

### Q4: Rork の無料プランでもこのアプリは作れますか？

基本的な録音・再生機能は無料プランで作成できます。ただし、ネイティブ API へのアクセスやカスタムコードの編集には Rork Max が必要になる場合があります。料金プランの詳細は [Rork料金プラン完全ガイド](/articles/rork-business/rork-pricing-plans) をご確認ください。
