# Rorkの画面遷移・ナビゲーションエラーを解決する完全ガイド

Rorkで生成したアプリを動かしていると、「ボタンを押しても画面が変わらない」「`Screen not found`というエラーが出る」「タブバーが表示されない」といったナビゲーション関連のトラブルに遭遇することがあります。

Rorkが採用しているExpo Routerは、ファイル構造に基づいてルーティングを自動生成する強力な仕組みですが、生成されたコードが想定外の構造になっていたり、わずかな設定ミスでナビゲーションが壊れてしまうことがあります。

この記事では、Rorkユーザーが最もよく経験するナビゲーションエラーを6つのパターンに分類し、それぞれの原因と確実な解決策を丁寧に解説します。

## よくあるナビゲーションエラーの種類

まず、代表的なエラーのパターンを整理しておきましょう。

- **`Couldn't find a navigation object`** — ナビゲーションオブジェクトが取得できない
- **`Screen not found`** — 遷移先の画面が見つからない
- **タブバーが表示されない** — 下部タブが消える・切り替えられない
- **戻るボタンが効かない** — スタックナビゲーションで戻れない
- **ディープリンクが動作しない** — 外部URLからの遷移が失敗する
- **画面遷移後にスクロール位置がリセットされない** — 状態が引き継がれてしまう

それぞれ順番に見ていきましょう。

## エラー1: `Couldn't find a navigation object`

### 症状

画面内のボタンをタップしたとき、以下のエラーがコンソールに表示されます。

```
Error: Couldn't find a navigation object. Is your component inside a screen in a Navigator?
```

### 原因

このエラーは、`useNavigation()` フックや `useRouter()` フックを、ナビゲーターのスコープ外で呼び出したときに発生します。Rorkが生成するコードでは、コンポーネントをモーダルやポップオーバーとして呼び出す際にこのパターンが起きやすくなっています。

```tsx
// ❌ 問題のあるパターン
// Navigatorの外側で定義されたコンポーネントがuseRouterを使おうとしている
export function StandaloneButton() {
  const router = useRouter(); // ← ここでエラー発生
  return <Button onPress={() => router.push('/detail')} title="詳細へ" />;
}

// ✅ 解決策: 必ずNavigatorのスコープ内で呼び出す
// app/_layout.tsx のStack.Screen内で描画されるコンポーネントから呼ぶ
```

### 解決手順

1. エラーが出ているコンポーネントが `app/` フォルダ内のファイルから直接呼び出されているか確認する
2. `app/_layout.tsx` の構造を見直し、対象コンポーネントがナビゲーターツリーの内側に配置されているかチェックする
3. Rorkに「このコンポーネントをNavigatorの内側に移動して」と指示して再生成を試みる

## エラー2: `Screen not found` — 遷移先の画面が見つからない

### 症状

`router.push('/detail/123')` や `router.navigate('profile')` を実行すると以下が表示されます。

```
Error: No route matching path "/detail/123" was found.
```

### 原因

Expo Routerはファイルパスがそのままルートになります。`app/detail/[id].tsx` というファイルがなければ `/detail/123` へは遷移できません。Rorkが生成したコードで遷移先のファイルが抜けていたり、ファイル名のスペルミスが原因のことがほとんどです。

```
# ✅ 正しいファイル構成の例
app/
├── _layout.tsx
├── index.tsx          → /
├── profile.tsx        → /profile
├── detail/
│   └── [id].tsx       → /detail/123, /detail/456 など
└── (tabs)/
    ├── _layout.tsx
    ├── home.tsx       → /(tabs)/home
    └── settings.tsx   → /(tabs)/settings
```

### 解決手順

1. 遷移先のパスを確認する（例: `router.push('/detail/123')` なら `/detail/[id].tsx` が必要）
2. `app/` ディレクトリに対応するファイルが存在するか確認する
3. 動的ルート（`[id].tsx` など）の場合は `[]` の中のパラメータ名が一致しているか確認する
4. Rorkに「`app/detail/[id].tsx` の画面を追加して」と指示して不足ファイルを生成してもらう

## エラー3: タブバーが表示されない・切り替えられない

### 症状

アプリを起動しても下部のタブバーが表示されない、または特定のタブに切り替えができません。

### 原因

Expo RouterのタブナビゲーションはRorkが自動生成することが多いですが、`(tabs)` フォルダの構成や `_layout.tsx` の設定に問題が生じることがあります。よくある原因は以下の通りです。

- `app/(tabs)/_layout.tsx` が存在しない
- `Tabs.Screen` の `name` プロップがファイル名と一致していない
- `tabBarIcon` の設定が未定義になっている

```tsx
// ✅ 正しいタブ設定の例 (app/(tabs)/_layout.tsx)
import { Tabs } from 'expo-router';
import { Ionicons } from '@expo/vector-icons';

export default function TabLayout() {
  return (
    <Tabs
      screenOptions={{
        tabBarActiveTintColor: '#007AFF',
      }}
    >
      {/* nameはファイル名（拡張子なし）と一致させる */}
      <Tabs.Screen
        name="index"         // ← app/(tabs)/index.tsx に対応
        options={{
          title: 'ホーム',
          tabBarIcon: ({ color }) => (
            <Ionicons name="home" size={24} color={color} />
          ),
        }}
      />
      <Tabs.Screen
        name="settings"      // ← app/(tabs)/settings.tsx に対応
        options={{
          title: '設定',
          tabBarIcon: ({ color }) => (
            <Ionicons name="settings" size={24} color={color} />
          ),
        }}
      />
    </Tabs>
  );
}
```

### 解決手順

1. `app/(tabs)/` フォルダと `_layout.tsx` が存在するか確認する
2. `Tabs.Screen` の `name` とファイル名が完全一致しているか確認する（`index.tsx` → `name="index"`）
3. アイコンライブラリ（`@expo/vector-icons` など）がインストールされているか確認する
4. Rorkに「タブナビゲーションの設定を修正して」と伝え、`_layout.tsx` を再生成してもらう

## エラー4: 戻るボタンが機能しない

### 症状

スタックナビゲーションで画面を積み上げたあと、ヘッダーの戻るボタンや `router.back()` が効かない、または意図しない画面に戻る。

### 原因

Expo Routerのスタックナビゲーションでは、遷移の履歴（スタック）が自動管理されます。`router.replace()` を使うとスタックに履歴が残らないため「戻る」ができなくなることがよくあります。

```tsx
// ❌ 戻るボタンが消えるパターン
router.replace('/home'); // historyが消える

// ✅ 戻れるようにするにはpushを使う
router.push('/home');    // historyが積まれる

// ✅ ログイン後など「戻らせたくない」画面にはreplaceを意図的に使う
router.replace('/(tabs)/home');
```

### 解決手順

1. 遷移に使っているメソッドを確認する（`push` vs `replace` vs `navigate`）
2. 「戻れるべき」画面への遷移は `router.push()` を使う
3. 「戻らせたくない」画面（ログイン後のホームなど）への遷移は `router.replace()` を使う
4. 実機やシミュレーターで Android のシステムバックボタンが効くかも確認する

## エラー5: ディープリンクが動作しない

### 症状

通知やSNSのリンクからアプリを開こうとすると、ホーム画面に飛ぶか、アプリが起動してもディープリンク先の画面に遷移しません。

### 原因

ディープリンクを機能させるには、`app.json`（または `app.config.js`）にスキームの設定が必要です。Rorkが生成したプロジェクトでは、スキームが未設定のままになっていることがあります。

```json
{
  "expo": {
    "name": "MyApp",
    "slug": "my-app",
    "scheme": "myapp",
    "ios": {
      "bundleIdentifier": "com.example.myapp"
    },
    "android": {
      "package": "com.example.myapp",
      "intentFilters": [
        {
          "action": "VIEW",
          "data": [
            {
              "scheme": "myapp",
              "host": "open"
            }
          ],
          "category": ["BROWSABLE", "DEFAULT"]
        }
      ]
    }
  }
}
```

### 解決手順

1. `app.json` に `scheme` が設定されているか確認する
2. Expo Routerを使う場合は `expo-linking` の設定が正しいか確認する
3. 実機テストでは `npx uri-scheme open myapp://profile --ios` などでディープリンクをシミュレートできる
4. Rorkに「ディープリンクの設定を `app.json` に追加して」と指示する

React Nativeのディープリンク設定について基礎から学びたい方には、[React Native実践開発ガイド（古市竜太 著）](https://www.amazon.co.jp/dp/4798169889/?tag=pinocchio-22)が参考になります。

## エラー6: 画面遷移後にスクロール位置や状態がリセットされない

### 症状

一覧画面からスクロールして詳細を見て戻ったとき、スクロール位置が先頭に戻っていなかったり、入力フォームの内容が消えない（または消える）。

### 原因

Expo Routerでは、スタックに積まれた画面はメモリ上に保持されることが多く、戻ったときの状態が残ります。これが「意図した挙動」か「意図しない挙動」かで対応が変わります。

```tsx
// スクロール位置を戻ったときにリセットしたい場合
import { useFocusEffect } from 'expo-router';
import { useCallback, useRef } from 'react';
import { ScrollView } from 'react-native';

export default function ListScreen() {
  const scrollRef = useRef<ScrollView>(null);

  useFocusEffect(
    useCallback(() => {
      scrollRef.current?.scrollTo({ x: 0, y: 0, animated: false });
    }, [])
  );

  return (
    <ScrollView ref={scrollRef}>
      {/* リスト内容 */}
    </ScrollView>
  );
}
```

### 解決手順

1. スクロール位置をリセットしたい場合は `useFocusEffect` + `scrollTo` を使う
2. フォームの入力をクリアしたい場合は `useFocusEffect` 内で状態をリセットする
3. Rorkに「画面にフォーカスが当たったときに○○をリセットするコードを追加して」と指示する

## FAQ

**Q. Rorkで再生成すれば自動的に直りますか？**

A. 多くの場合、同じプロンプトで再生成しても同じ構成のコードが出力されます。「`app/(tabs)/_layout.tsx` にタブを3つ追加して」「ナビゲーションの戻るボタンが機能するよう修正して」のように、問題を具体的に伝えることで修正コードを得やすくなります。

**Q. Expo RouterとReact Navigationの違いは何ですか？**

A. Expo RouterはファイルベースのルーティングでWebのNext.jsに近い感覚で使えます。React NavigationはコードでルートとStack/Tab/Drawerを明示的に定義します。RorkはExpo Routerを標準で採用しているため、`app/` フォルダ内のファイル構成がルーティングに直結しています。Expo Routerの詳細は[Expo Routerナビゲーション完全ガイド](/articles/rork-dev/expo-router-navigation-guide)も参考にしてください。

**Q. ナビゲーションエラーが複数種類重なっているときはどこから直せばいいですか？**

A. まず `app/_layout.tsx` と `app/(tabs)/_layout.tsx`（タブを使っている場合）の構造を確認するところから始めてください。ルートのレイアウト設定が崩れると、下位のナビゲーションが全て機能しなくなります。次に、エラーメッセージをそのままRorkのプロンプトに貼り付けて修正を依頼するのが効率的です。Rorkのエラー自動修正機能については[Fix Now 自動デバッグの使い方](/articles/rork-basics/fix-now-auto-debug)もご覧ください。

## まとめ

Rorkで生成したアプリのナビゲーションエラーは、大きく分けると「ファイル構成のズレ」「遷移メソッドの使い分けミス」「レイアウト設定の不足」の3パターンが原因です。

エラーメッセージが出たときは、まず `app/` フォルダのファイル構造を確認し、遷移先のパスに対応するファイルが存在するかチェックする習慣をつけておくと、多くのトラブルを素早く解消できます。それでも解決しない場合は、エラーログをそのままRorkのチャットに貼り付けて修正を依頼するのが最も効率的な方法です。

Rorkの生成コードとExpo Routerについてさらに詳しく学びたい方には、[Reactネイティブ+Expo実践マスター（樋口理 著）](https://www.amazon.co.jp/dp/4297140918/?tag=pinocchio-22)もおすすめです。ランタイムエラー全般のトラブルシューティングについては[Expo ランタイムエラー解決ガイド](/articles/rork-dev/rork-react-native-expo-runtime-errors-fix)もあわせてご確認ください。
