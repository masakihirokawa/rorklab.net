## はじめに — カウントダウンアプリで学ぶ Rork の基本

「あのイベントまであと何日だろう？」と気になった経験は誰にでもあるはずです。誕生日、旅行、資格試験、プロジェクトの締め切り——大切な日までの残り日数が一目で分かるアプリがあれば、毎日のモチベーションにもつながります。

この記事では、**Rork** を使ってイベントカウントダウンアプリをゼロから作る方法をステップバイステップで解説します。プログラミングの経験がなくても、AIプロンプトを活用して美しいカウントダウンアプリを完成させることができます。

この記事で学べることは以下のとおりです。

- Rork でのプロジェクト作成からプレビューまでの流れ
- 日付の入力・計算・表示のロジック
- 複数のイベントを管理するリスト画面の設計
- カスタマイズ可能なデザインの実装方法

対象読者は、Rork を初めて使う方やアプリ開発に興味がある初心者の方です。Rork の基本的なセットアップがまだの方は、[Rork をはじめよう — 初心者向けスタートガイド](/articles/rork-basics/getting-started)をご覧ください。

## プロジェクトの作成とプロンプト設計

### Rork で新規プロジェクトを作成する

まず [rork.com](https://rork.com/) にアクセスして、新しいプロジェクトを作成します。プロジェクト名は「EventCountdown」など、分かりやすい名前にしましょう。

Rork では AI にプロンプトを渡すだけでアプリの骨格が生成されます。以下のようなプロンプトを入力してみてください。

```text
イベントカウントダウンアプリを作ってください。

機能:
- イベント名と日付を登録できる
- 各イベントまでの残り日数をリアルタイムで表示する
- イベントはリスト形式で一覧表示する
- イベントの追加・編集・削除ができる
- カテゴリ別に色分けできる（誕生日、旅行、仕事、その他）

デザイン:
- モダンなカード型UI
- ダークモード対応
- 残り日数が大きく表示されるデザイン
- カテゴリごとにアイコンと色が変わる
```

このプロンプトを送信すると、Rork が自動的に画面構成やコンポーネントを生成してくれます。

### プロンプト設計のポイント

Rork で良い結果を得るためのコツをいくつか紹介します。

**具体的な機能を箇条書きにする** — 「カウントダウンアプリを作って」だけでは曖昧すぎます。どんな機能が必要かを明確に伝えましょう。

**デザインの方向性を指定する** — 「モダン」「ミニマル」「カラフル」といったキーワードを添えると、AIがデザインの方向性を理解しやすくなります。

**ユーザー操作の流れを説明する** — 「イベントを追加するボタンを押すとフォームが表示される」のように、操作フローを伝えるとより正確な画面構成が得られます。

## 日付計算のロジックを実装する

カウントダウンアプリの核心は日付計算です。Rork が生成したコードの中で、以下のような計算ロジックが使われます。

```typescript
// イベントまでの残り日数を計算する関数
const calculateDaysRemaining = (targetDate: string): number => {
  const now = new Date();
  const target = new Date(targetDate);

  // 時刻部分をリセットして日付のみで比較する
  now.setHours(0, 0, 0, 0);
  target.setHours(0, 0, 0, 0);

  const diffTime = target.getTime() - now.getTime();
  const diffDays = Math.ceil(diffTime / (1000 * 60 * 60 * 24));

  return diffDays;
};

// 使用例
const daysLeft = calculateDaysRemaining("2026-12-25");
console.log(`クリスマスまであと ${daysLeft} 日`);
// 出力例: クリスマスまであと 276 日
```

このコードのポイントは、`setHours(0, 0, 0, 0)` で時刻部分をリセットしている点です。これにより、「今日の23時に計算しても残り1日」といった直感に反する結果を防ぐことができます。

### 残り時間の詳細表示

日数だけでなく、時間・分・秒まで表示したい場合は、リアルタイム更新を追加します。

```typescript
// 残り時間を詳細に計算する関数
const calculateTimeRemaining = (targetDate: string) => {
  const now = new Date();
  const target = new Date(targetDate);
  const diff = target.getTime() - now.getTime();

  if (diff <= 0) {
    return { days: 0, hours: 0, minutes: 0, seconds: 0, isPast: true };
  }

  return {
    days: Math.floor(diff / (1000 * 60 * 60 * 24)),
    hours: Math.floor((diff % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60)),
    minutes: Math.floor((diff % (1000 * 60 * 60)) / (1000 * 60)),
    seconds: Math.floor((diff % (1000 * 60)) / 1000),
    isPast: false,
  };
};

// 期待する出力:
// { days: 276, hours: 14, minutes: 32, seconds: 8, isPast: false }
```

Rork で生成されたコードにこのロジックを追加するには、チャット欄で「残り時間を日・時・分・秒で詳細に表示してほしい」と伝えるだけで対応してくれます。

## イベント管理画面の設計

### イベントの追加フォーム

ユーザーがイベントを登録するためのフォーム画面を設計します。以下のフィールドを含めましょう。

- **イベント名** — テキスト入力（例: 「夏休みの旅行」）
- **日付** — 日付ピッカーで選択
- **カテゴリ** — ドロップダウンで選択（誕生日、旅行、仕事、学校、その他）
- **メモ** — 任意のテキスト入力

Rork にフォーム画面を追加するプロンプト例はこちらです。

```text
イベント追加画面を改善してください。

- イベント名の入力欄（必須、最大30文字）
- 日付ピッカー（今日以降の日付のみ選択可能）
- カテゴリ選択（誕生日🎂、旅行✈️、仕事💼、学校📚、その他⭐）
- メモ欄（任意、最大100文字）
- 「保存」ボタンと「キャンセル」ボタン
```

### イベントリストの表示

登録したイベントはカード形式のリストで表示します。残り日数が少ないイベントが上に来るようにソートすると使いやすくなります。

```typescript
// イベントを残り日数でソートする
const sortedEvents = events
  .filter((event) => calculateDaysRemaining(event.date) >= 0)
  .sort((a, b) => {
    const daysA = calculateDaysRemaining(a.date);
    const daysB = calculateDaysRemaining(b.date);
    return daysA - daysB;
  });

// 期待する出力例（3月24日時点）:
// [
//   { name: "プレゼン発表", date: "2026-03-28", daysLeft: 4 },
//   { name: "友達の誕生日", date: "2026-04-15", daysLeft: 22 },
//   { name: "夏休みの旅行", date: "2026-08-01", daysLeft: 130 }
// ]
```

## データの永続化と状態管理

アプリを閉じてもイベントデータが消えないように、ローカルストレージにデータを保存する必要があります。React Native + Expo 環境では `AsyncStorage` を使うのが一般的です。

```typescript
import AsyncStorage from "@react-native-async-storage/async-storage";

// イベントデータの型定義
interface CountdownEvent {
  id: string;
  name: string;
  date: string;
  category: "birthday" | "travel" | "work" | "school" | "other";
  memo?: string;
  createdAt: string;
}

// データを保存する
const saveEvents = async (events: CountdownEvent[]) => {
  try {
    await AsyncStorage.setItem("countdown_events", JSON.stringify(events));
  } catch (error) {
    console.error("保存に失敗しました:", error);
  }
};

// データを読み込む
const loadEvents = async (): Promise<CountdownEvent[]> => {
  try {
    const data = await AsyncStorage.getItem("countdown_events");
    return data ? JSON.parse(data) : [];
  } catch (error) {
    console.error("読み込みに失敗しました:", error);
    return [];
  }
};
```

Rork が生成するコードでは、状態管理が自動的に組み込まれていることが多いですが、永続化が含まれていない場合は「AsyncStorage を使ってイベントデータを永続化してください」とプロンプトで指示しましょう。データの永続化についてさらに詳しく知りたい方は、[Rork アプリのオフライン対応 — AsyncStorage と NetInfo 活用ガイド](/articles/rork-dev/rork-offline-ready-asyncstorage-netinfo)も参考になります。

### 残り日数に応じた強調表示

残り日数が少なくなったイベントを視覚的に目立たせるのも効果的です。例えば、残り7日以内のイベントには赤いバッジを表示する、残り30日以内はオレンジに変えるといった工夫ができます。

Rork のチャットで「残り日数が7日以内のカードは赤色のハイライト、30日以内はオレンジのハイライトにしてほしい」と伝えれば、条件付きスタイリングを自動で実装してくれます。

## よくあるエラーと対処法

### 日付が1日ズレる

タイムゾーンの処理が原因で日付がズレることがあります。`new Date("2026-12-25")` のように文字列から Date オブジェクトを作成すると UTC として解釈される場合があるため、`setHours(0, 0, 0, 0)` でローカルタイムの午前0時にリセットするのが確実です。

### 過去のイベントが表示される

イベント日が過ぎたものを非表示にするには、フィルター処理を追加します。「過去のイベントを自動的に非表示にして」と Rork に伝えるか、`filter` で `daysRemaining >= 0` のイベントのみ表示するようにしましょう。

### アプリを再起動するとデータが消える

AsyncStorage によるデータ永続化が実装されていない可能性があります。Rork のチャットで「アプリを閉じてもデータが残るようにしてほしい」と伝えてください。

## FAQ

### Q: プログラミング未経験でもカウントダウンアプリは作れますか？

はい、作れます。Rork は AI がコードを自動生成してくれるため、プログラミングの経験がなくても自然言語のプロンプトだけでアプリを作ることができます。日本語でやりたいことを説明するだけで、画面構成やロジックが自動的に組み立てられます。

### Q: カウントダウンの精度はどの程度ですか？

日単位のカウントダウンであれば、JavaScript の Date オブジェクトによる計算で十分な精度が得られます。秒単位のリアルタイム表示を行う場合も、`setInterval` で1秒ごとに更新すれば正確なカウントダウンが実現できます。

### Q: イベントの数に上限はありますか？

AsyncStorage の容量制限（通常6MB程度）の範囲内であれば、数百件のイベントを問題なく保存できます。一般的な利用であれば上限を気にする必要はありません。

### Q: 作ったアプリを App Store に公開できますか？

Rork で作成したアプリは、Rork の公開機能を使って App Store や Google Play に提出できます。Rork Max プランであれば、2クリックで App Store への申請が可能です。アプリ公開の詳細な手順については、ストア申請ガイドをご参照ください。
