## はじめに — なぜポートフォリオ「アプリ」なのか

フリーランスのデザイナーや個人開発者にとって、ポートフォリオは名刺代わりの存在です。Webサイトで作品を公開している方は多いですが、**モバイルアプリ**としてポートフォリオを持つことには、独自のメリットがあります。

クライアントとの打ち合わせ中にサッとアプリを開いて作品を見せたり、プッシュ通知で新作の公開を知らせたり。App Store や Google Play に自分のアプリがあること自体が、技術力の証明にもなります。

この記事では、[Rork](https://rork.com/) を使って、プロフェッショナルなポートフォリオアプリをコード不要で構築する方法をステップバイステップで解説します。

## この記事で学べること

この記事を読み終えると、以下のことができるようになります。

- Rork でポートフォリオアプリのプロジェクトを作成する
- 作品一覧画面・詳細画面・プロフィール画面を構築する
- 画像ギャラリーとフィルタリング機能を実装する
- アニメーションやダークモードで見た目を洗練させる
- TestFlight / Google Play で実機テスト・公開する

**対象読者**: Rork 初心者〜中級者、デザイナー、フリーランス開発者

## 前提知識・環境準備

ポートフォリオアプリを作り始める前に、以下を準備しておきましょう。

- **Rork アカウント**: [rork.com](https://rork.com/) で無料登録（Rork Max プランを推奨）
- **作品素材**: 表示したい画像やプロジェクト情報（5〜10点程度）
- **プロフィール情報**: 名前、自己紹介、SNSリンク、連絡先

Rork の基本的な使い方がわからない方は、先に「[Rork とは？AIアプリ開発の始め方](/articles/rork-basics/what-is-rork)」を読んでおくとスムーズです。

## ステップ1: プロジェクトの作成とプロンプト設計

Rork でアプリを作る際に最も重要なのは、**最初のプロンプト**です。具体的で構造化されたプロンプトを書くことで、生成されるアプリの品質が大きく変わります。

以下のようなプロンプトを入力しましょう。

```text
ポートフォリオアプリを作成してください。

【画面構成】
1. ホーム画面: 作品のグリッド表示（2列）、カテゴリフィルター付き
2. 作品詳細画面: 大きな画像、プロジェクト名、説明文、使用技術タグ
3. プロフィール画面: アバター、自己紹介、スキルバー、SNSリンク
4. お問い合わせ画面: 名前・メール・メッセージの送信フォーム

【デザイン要件】
- ミニマルでモダンなデザイン
- ダークモード対応
- スムーズなページ遷移アニメーション
- 画像はタップで拡大表示（ズーム可能）

【カテゴリ】
- UI/UXデザイン
- モバイルアプリ
- Webサイト
- イラスト
```

このプロンプトのポイントは、**画面構成を明確に番号付き**で指定していることです。Rork の AI は構造化された指示に対して特に高い精度でアプリを生成します。

## ステップ2: 生成されたアプリの調整

Rork が初期アプリを生成したら、プレビューで確認しながら微調整を加えていきます。

### グリッドレイアウトの最適化

作品一覧画面では、グリッドレイアウトが見た目の印象を大きく左右します。もし生成されたレイアウトに改善が必要な場合は、追加プロンプトで指示します。

```text
作品一覧のグリッドを以下のように変更してください:
- カード間のスペースを12pxに
- カードに影（shadow）を追加
- 画像のアスペクト比を4:3に統一
- カードのホバー時に少し浮き上がるアニメーション
```

### 作品データの構造

ポートフォリオに表示するデータは、以下のような構造で管理するのがベストです。

```typescript
// 作品データの型定義
interface PortfolioItem {
  id: string;
  title: string;           // プロジェクト名
  category: string;        // カテゴリ（UIデザイン、アプリ等）
  thumbnail: string;       // サムネイル画像URL
  images: string[];        // 詳細画像URL配列
  description: string;     // プロジェクト概要
  technologies: string[];  // 使用技術
  url?: string;            // 外部リンク（任意）
  date: string;            // 制作日
}

// 作品データの例
const portfolioItems: PortfolioItem[] = [
  {
    id: "1",
    title: "フィットネスアプリ UI",
    category: "UI/UXデザイン",
    thumbnail: "https://example.com/thumb1.jpg",
    images: [
      "https://example.com/detail1-1.jpg",
      "https://example.com/detail1-2.jpg",
    ],
    description: "ヘルスケア企業向けに設計したフィットネスアプリのUI",
    technologies: ["Figma", "React Native", "Rork"],
    url: "https://example.com/project1",
    date: "2026-02",
  },
];
```

## ステップ3: カテゴリフィルター機能の実装

ポートフォリオアプリに欠かせないのが、カテゴリ別のフィルター機能です。訪問者が目的の作品をすぐに見つけられるようにしましょう。

Rork に次のプロンプトを入力してフィルターを追加します。

```text
作品一覧画面の上部にカテゴリフィルターを追加してください:
- 水平スクロールのタブバー形式
- 「すべて」「UI/UXデザイン」「モバイルアプリ」「Webサイト」「イラスト」
- 選択中のタブはアクセントカラーでハイライト
- フィルター切替時にフェードアニメーション
```

このような段階的な指示で、Rork は既存の画面を壊さずに機能を追加してくれます。一度に多くの変更を依頼するよりも、**小さな単位で確認しながら進める**のがコツです。詳しくは「[Rork のプロンプトエンジニアリング](/articles/rork-basics/rork-prompt-engineering-app-design-tips)」もご参照ください。

## ステップ4: プロフィール画面の作成

ポートフォリオアプリのプロフィール画面は、あなた自身をアピールする場所です。スキルバーや SNS リンクを効果的に配置しましょう。

```text
プロフィール画面を以下のように更新してください:
- 上部にアバター画像（円形、120px）
- 名前と肩書き（フリーランスデザイナー等）
- 自己紹介文（3〜4行）
- スキルセクション: プログレスバーで各スキルの習熟度を表示
  - Figma: 95%
  - React Native: 85%
  - UI Design: 90%
  - Illustration: 80%
- SNSリンク: アイコンボタンで横並び（Instagram, X, GitHub, LinkedIn）
- 「お仕事のご依頼」ボタン（お問い合わせ画面へ遷移）
```

## ステップ5: ダークモード対応とアニメーション

見た目の洗練さを高めるために、ダークモードとアニメーションを追加します。

```text
以下のデザイン改善を行ってください:
1. ダークモードの自動切替（端末設定に連動）
2. 作品カードの表示時にフェードイン + 下からスライドアニメーション
3. 画面遷移時にスムーズなトランジション
4. プロフィール画面のスキルバーにアニメーション（表示時にゼロから伸びる）
```

ダークモードは、デザイナーのポートフォリオでは特に作品が映えるため重要です。Rork Max を使えば、[ダークモード対応](/articles/rork-dev/rork-max-dark-mode-theming)もプロンプト一つで実装できます。

## ステップ6: お問い合わせフォームの設置

クライアントからの問い合わせを受け付けるフォームを作成します。Rork では [Supabase](/articles/rork-dev/rork-auth-firebase-supabase) と連携してフォームデータを保存できます。

```text
お問い合わせ画面を作成してください:
- 入力欄: 名前（必須）、メールアドレス（必須）、メッセージ（必須、複数行）
- バリデーション付き（空欄チェック、メール形式チェック）
- 送信ボタンにローディングスピナー
- 送信成功時にサンキューモーダル表示
- Supabase にデータを保存
```

## 実機テストと公開

アプリが完成したら、実機でテストして公開しましょう。

Rork Max を使っている場合は、EAS Build で iOS / Android 両方のバイナリを生成できます。TestFlight でベータテスターにフィードバックをもらい、品質を確認した上で App Store / Google Play に申請します。

公開手順の詳細は「[App Store 公開ガイド](/articles/rork-basics/app-store-publishing)」と「[Google Play 公開ガイド](/articles/rork-basics/google-play-publishing)」をご確認ください。

## よくある質問（FAQ）

### Q1: Rork 無料プランでもポートフォリオアプリは作れますか？

はい、基本的な機能は無料プランでも作成できます。ただし、ダークモードの自動切替、高度なアニメーション、EAS Build による実機ビルドなどは Rork Max プランが必要です。まずは無料プランで試してみて、必要に応じてアップグレードするのがおすすめです。

### Q2: 作品画像はどこにホスティングすればよいですか？

いくつかの選択肢があります。Supabase Storage を使えばアプリ内から直接アップロードできます。他にも Cloudflare R2 や AWS S3 が利用可能です。画像は表示速度を考慮して、サムネイルは 400×300px 程度、詳細画像は 1200px 幅程度に最適化しておくとよいでしょう。

### Q3: ポートフォリオアプリを App Store に公開するのにいくらかかりますか？

Apple Developer Program の年会費が 12,800円（$99 USD）、Google Play Developer の登録料が $25（一回のみ）かかります。Rork 自体の利用料は無料プランなら $0、Rork Max は月額プランから利用できます。詳しくは「[Rork の料金プラン](/articles/rork-business/rork-pricing-plans)」をご覧ください。

### Q4: 公開後に作品を追加・更新するにはどうすればよいですか？

Rork のプロジェクトに戻って作品データを追加し、再ビルド・再デプロイするのが基本的な方法です。より頻繁に更新したい場合は、Supabase をバックエンドとして使い、管理画面から作品を追加できる CMS 的な仕組みを作ることも可能です。

## まとめ

Rork を使えば、プロフェッショナルなポートフォリオアプリを短時間で構築できます。重要なポイントをおさらいしましょう。

- **プロンプト設計が品質を決める**: 画面構成やデザイン要件を具体的に指示する
- **段階的に構築する**: 一度に全てを作るのではなく、画面ごとに確認しながら進める
- **作品が映えるデザイン**: ダークモードやアニメーションで見た目を洗練させる
- **実用的な機能を忘れずに**: お問い合わせフォームや SNS リンクを必ず設置する

Webサイトだけでなく、モバイルアプリとしてポートフォリオを持つことは、デザイナーや開発者としてのブランド力を一段引き上げてくれます。ぜひ Rork で自分だけのポートフォリオアプリに挑戦してみてください。
