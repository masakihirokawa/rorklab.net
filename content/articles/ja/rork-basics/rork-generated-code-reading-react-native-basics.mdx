Rork でアプリを作って App Store に出したあと、細かい修正を加えたくなってコードを開いたら、何が書いてあるのかさっぱり分からず、そっと閉じた ― こんな経験をされた方は、きっと少なくないのではないでしょうか。私自身も最初の2〜3本は、生成されたコードに手を加えることを完全に諦めていました。

ですが、実は Rork が吐き出すコードの8割は、ごく決まったパターンの繰り返しです。読み方の勘所さえ掴めれば、プロの React Native エンジニアになる必要は全くありません。この記事では、「全部を理解しようとしない」方針で、Rork ユーザーが最低限知っておきたい読み方だけを整理します。

## なぜ Rork ユーザーでも『読み方』を覚える価値があるのか

完全に AI に任せきる選択も、もちろん悪くありません。ですが実際に運用していると、次のような場面で必ず「コードを読む力」が効いてきます。

- AI が同じバグを何度直してもうまく修正できず、手動でワンポイント直したほうが速い場面
- レビュー機能や課金フローなど、わずかな文言を変えたい場面
- 申請リジェクト対応で「この画面のボタン色を変更してください」と Apple から指摘された場面

これらは、**1ファイル・数行の修正で済むのに、AI にお願いすると関係ないところまで変わってしまう**、というよくあるパターンです。10 分の自力修正が1時間の生成ガチャに化けると、心も折れます。

大事なのは、全部を書けるようになることではなく、**「どのファイルの、どのあたりに、何が書いてあるか」の見取り図**を持つことです。

## まず開くべきファイルは、たった3つだけ

Rork が生成したプロジェクトを開くと、ファイルがたくさんあって気が遠くなりますが、最初に見るべきはこの3つで十分です。

- `app/_layout.tsx` ― アプリ全体のレイアウトと、どの画面がどのタブに属するかを決めているファイル
- `app/(tabs)/index.tsx` ― ホームタブを開いたときに最初に表示される画面
- `app.json` ― アプリ名、アイコン、スプラッシュ画面、バージョン番号などを設定するファイル

乱暴に言えば、画面の「文言や配置を変えたい」なら `(tabs)` 配下のどれか、アプリそのものの「名前やアイコンを変えたい」なら `app.json` を見ればほぼ片がつきます。逆に `node_modules/` や `ios/` `android/` ディレクトリは、最初のうちは触らない・開かないで構いません。

関連して[Rork を始めるためのはじめの一歩](/articles/rork-basics/getting-started)では、プロジェクト構造の全体像もごく簡単に触れています。あわせて読むと、ファイルの位置関係が頭に入りやすいはずです。

## JSX は「入れ子の家」だと思えば、もう読める

画面ファイルを開くと、HTML に似ていて似ていない、独特の記法が出てきます。これが JSX です。たとえば Rork が作ったシンプルな画面は、こんな形をしています。

```tsx
// app/(tabs)/index.tsx
// 役割: ホーム画面 — タイトルとボタンを並べて表示する
import { View, Text, Pressable, StyleSheet } from 'react-native';

export default function HomeScreen() {
  const handlePress = () => {
    console.log('ボタンが押されました');
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>ようこそ</Text>
      <Pressable style={styles.button} onPress={handlePress}>
        <Text style={styles.buttonText}>はじめる</Text>
      </Pressable>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 20 },
  button: { backgroundColor: '#0A84FF', paddingHorizontal: 24, paddingVertical: 12, borderRadius: 8 },
  buttonText: { color: 'white', fontSize: 16 },
});
```

読み方のコツは、`<View>` を「箱」、`<Text>` を「文字」、`<Pressable>` を「押せるボタン」と、物理的なパーツとして想像することです。上のコードは要するに「中央寄せの箱の中に、見出しと青いボタンが縦に並んでいる」と言っています。文字を「ようこそ」から「おかえりなさい」に変えたい時は、`<Text>` の中身を書き換えるだけで済みます。

慣れてくると、生成されたコードを開いた瞬間、**「これはカードを縦に並べたリスト画面だな」**といった形の情報が、ぱっと入ってくるようになります。

## 状態（useState）が分かると、アプリの半分は読める

静的な画面の次に理解したいのが、「状態」という概念です。たとえば「カウンターを押すたびに数字が増える」「入力欄に文字を打つとリアルタイムで反映される」といった、**画面の中で変化する値**のことを、React Native では state（ステート）と呼びます。

```tsx
// 役割: タップするたびにカウンターが 1 増えるボタン
import { useState } from 'react';
import { View, Text, Pressable } from 'react-native';

export default function Counter() {
  // count: 現在の数値（初期値は 0）
  // setCount: count を更新するための専用関数
  const [count, setCount] = useState(0);

  return (
    <View>
      <Text>現在: {count}</Text>
      <Pressable onPress={() => setCount(count + 1)}>
        <Text>増やす</Text>
      </Pressable>
    </View>
  );
}
```

ポイントは `useState(0)` の一行です。これは「0 から始まる値を、この画面の中で記憶しておいてね」と Rork 製アプリに頼んでいる宣言です。そして、数値を変えたいときは必ず `setCount(...)` を経由する、という約束事があります。

Rork が生成したコードの中で `useState(...)` を探してみてください。ログイン情報、入力文字、表示中のタブ、ローディングのオンオフなど、**ユーザーの動きで変わる値はたいていここに集まっています**。裏を返せば、この行の近くが「アプリの挙動を司る中枢」だと当たりをつけられます。

## Rork 生成コードによく出てくる5つのパターン

読み方に慣れるために、Rork が生成するコードで頻出するパターンを5つだけ先に知っておくと近道です。

1. **画面遷移は `router.push('/path')` の形で書かれる** ― Expo Router を使っているので、ファイル名がそのままルートになります。`app/settings.tsx` があれば `router.push('/settings')` で開けるわけです
2. **API 通信は `fetch` を `async/await` で囲む** ― 生成されたコードで `await fetch(...)` と書かれていたら、そこが外部サーバーとの通信箇所です
3. **色やフォントは `StyleSheet.create` にまとまっている** ― デザインの変更は、そのファイル最下部のスタイル定義だけ触ればほぼ完結します
4. **画像や音は `require('./assets/...')` で読み込む** ― 画像の差し替えは `assets/` フォルダのファイル入れ替え＋ここのパス変更で済みます
5. **環境変数は `process.env.EXPO_PUBLIC_XXX` で参照する** ― API キーなどは `.env` ファイルに書き、コード側ではこの形で読み出します

この5パターンを頭に入れておくだけで、Rork 出力のほとんどは「だいたい何をしているコードか」が読めるようになります。さらに一歩踏み込みたい方は、[React Native 再利用可能コンポーネントの設計パターン](/articles/rork-dev/rork-react-native-reusable-component-design-patterns)も参考になるはずです。

## 読めるようになった次は「最小の修正」から始める

読めるようになったら、最初の一歩は「テキストの文言変更」と「色の変更」で十分です。画面の中の `<Text>` の中身を書き換える、`StyleSheet` の中の `backgroundColor` を別の色に変える ― これだけでもコードに手を入れる心理的ハードルは一気に下がります。

なお、プロンプトの書き方そのものを磨きたい場合は、[Rork の AI プロンプト精度を上げるマスターガイド](/articles/rork-ai/rork-ai-prompt-engineering-mastery-guide-2026)もセットで読んでおくと、「AI に任せる部分」と「自分で手を入れる部分」を切り分けやすくなります。

もう少し体系立てて React Native を学びたくなった方には、[React Native & Expo アプリ開発入門（掌田津耶乃 著）](https://www.amazon.co.jp/dp/4798069914/?tag=pinocchio-22)が、平易な言葉で全体像をつかむのに向いています。すべてを読破する必要はなく、「目次から今必要な章だけつまみ読み」する使い方でも十分に元が取れるはずです。

まずは今日、生成されたコードの `index.tsx` を一つだけ開いて、`<Text>` の中の文字を1箇所書き換えて、端末で反映を確認してみてください。その 10 分が、これから Rork を長く使っていくための、いちばん大きな一歩になります。
