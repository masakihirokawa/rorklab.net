## Rorkへようこそ

モバイルアプリケーションの構築はこれまで以上に簡単になりました。次の5分で、コードを書かずに完全に機能するモバイルアプリケーションを作成します。このガイドは各ステップをご説明します。

## ステップ1：アカウントを作成する

**時間：1分**

1. Rorkのウェブサイトにアクセスします
2. 「サインアップ」または「始める」をクリックします
3. サインアップ方法を選択します：
   - メールアドレスとパスワード
   - Googleアカウント
   - Apple ID
   - その他のソーシャル認証オプション
4. メールアドレスを確認します
5. これで完了です！

これでRorkプラットフォームにアクセスできます。デフォルトでは、アプリを生成するためのプロンプト数が制限されているフリーティアになります。

## ステップ2：プランを選択する

**時間：1分**

### フリーティア

フリーティアは実験に最適です：

- 週に約5回のプロンプトでアプリを生成する機能
- 完全なプレビューとテスト機能
- 生成されたコードをダウンロードする機能
- すべての基本機能へのアクセス
- 学習と考えをためす人に最適

**最適な対象**：学生、実験者、試してみている人

### 有料プラン

Rorkは真摯なアプリ開発のための有料サブスクリプションプランを提供しています：

- **Rork Pro**：機能に応じて月額 25～50 ドル
- **Rork Max**：ネイティブ Swift iOS開発用に月額 200 ドル
- **無制限のプロンプト**：好きなだけアプリを生成できます
- **優先サポート**：質問への回答が速くなります
- **商用利用**：Rorkで構築したアプリを構築・販売します

<Callout type="info">
このガイドを完了するにはフリーティアに留まることができます。より多くのアプリを構築する準備ができたら、後でアップグレードしてください。
</Callout>

## ステップ3：最初のプロジェクトを開始する

**時間：1分**

1. ダッシュボードから「新しいアプリを作成」または「生成を開始」をクリックします
2. テキスト入力フィールドまたはプロンプトインターフェースが表示されます
3. これはアプリのアイデアを説明する場所です

## ステップ4：効果的なプロンプトを書く

**時間：1～2分**

ここはあなたの創造性の出番です。プロンプトは以下を説明する必要があります：

- **アプリのタイプ**：（「天気アプリ」、「タスク管理」、「レシピ検索ツール」）
- **主な機能**：（「現在の天気と7日間の予報を表示」、「ユーザーはタスクを追加して完了としてマーク」）
- **ユーザー**：（「忙しいプロフェッショナル向け」、「ホームシェフ向け」）
- **表示する重要な情報**：（「気温、湿度、風速」、「タスクタイトル、期限、優先度」）

### プロンプトの例

**シンプルなプロンプト**：
> 「ユーザーが大きなボタンで数値をインクリメントおよびデクリメントできるシンプルなカウンターアプリを作成してください。」

**詳細なプロンプト**：
> 「ユーザーが毎日の習慣をログして、ストリークのカレンダービューを表示し、習慣を完了したときに励ましをもらえる習慣追跡アプリを構築してください。」

**高度なプロンプト**：
> 「カテゴリ、検索機能、ダークモード、テキストファイルとしてメモをエクスポートする機能を備えたメモ取りアプリを作成してください。ユーザーはメモを作成、編集、削除、および整理できるようにしてください。」

### より良いプロンプトのコツ

**具体的に**：「天気アプリを作成」ではなく、「現在の気温、湿度、風速、およびアイコン付きの7日間の予報を表示する天気アプリを作成」と言いましょう。

**ユーザーエクスペリエンスを説明する**：ユーザーがアプリと関わる方法について考えてください。「ユーザーはボタンをタップして画面を移動する」または「ユーザーはカードベースのインターフェースをスワイプする」

**ビジュアルスタイルを含める**：「モダンでミニマル」、「カラフルで楽しい」、または「専門的で企業的」などのデザイン設定を提案してください。

**データを含める**：アプリがデータを保存する必要があるかどうかを説明してください：「ユーザーはタスクを作成でき、それらはローカルで保存されます」、または「アプリは天気API からデータを取得します。」

**強力なプロンプト例**：
> 「ユーザーに毎時間水を飲むよう促す通知を送信する水分補給リマインダーアプリを構築してください。アプリは今日飲んだ水の量を視覚的なプログレスバーで表示する必要があります。ユーザーはリマインダー頻度をカスタマイズできます。青と白のカラースキームにモダンで整理されたデザインを使用してください。」

<Callout type="tip">
最初のアプリはシンプルに保ちます。Rorkに慣れてきたら、より複雑な機能で実験してください。
</Callout>

## ステップ5：アプリを生成する

**時間：1～2分**

1. テキストフィールドにプロンプトを入力またはペーストします
2. 「生成」、「アプリを作成」、または類似のボタンをクリックします
3. Rorkの AI が起動します：
   - 説明を分析
   - コードを生成
   - アプリを構築
   - プレビューを準備
4. これは通常 30～90 秒かかります
5. AI が機能している間、読み込みインジケーターが表示されます

## ステップ6：アプリをプレビューする

### Rork（React Native）の場合

**Expo Go を使用**（テストに推奨）：

1. プレビューが画面に表示されます
2. QR コードが生成されます
3. 携帯電話で Expo Go アプリを開きます（App Store または Google Play から無料でダウンロード）
4. 携帯電話のカメラで QR コードをスキャンします
5. アプリがデバイスで即座に起動します
6. すべての機能、ボタン、相互作用をテストします

**ブラウザプレビュー**：
- シミュレータビューはアプリの見た目を示します
- デスクトップで相互作用できますが、体験は限定的です
- 電話プレビューを使用してより良いテストを行ってください

### Rork Max（ネイティブSwift）の場合

1. ストリーミングシミュレータープレビューがブラウザに読み込まれます
2. 実行中の iOS アプリのライブプレビューを表示します
3. ダウンロードの必要はありません。直接ストリーミングします
4. リアルタイムでシミュレーターと相互作用します
5. プロンプトを変更するときに変更を即座に確認します

<Callout type="info">
Rork Max のストリーミングシミュレータにより、Mac を所有したり Xcode を扱ったりする必要がなくなります。
</Callout>

## ステップ7：テストと反復

**時間：1～3分**

ここは楽しい部分です。作成した内容をテストしましょう：

**デバイス上で**：
- ボタンをタップしてインターフェースと相互作用します
- すべての機能をテストします
- 問題や予期しない動作を探します
- アプリがあなたの入力に応答する方法を確認します

**フィードバックを提供**：
何かが正しくない場合は、洗練できます：

1. プロンプトに戻ります
2. 具体的なフィードバックで変更します
3. 例：
   - 「右上に保存ボタンを追加」
   - 「背景色をライトブルーに変更」
   - 「アイテムを検索する機能を追加」
   - 「ボタンが小さすぎます。大きくしてください」
4. 「再生成」または「洗練」をクリックします
5. Rork はフィードバックに基づいて更新バージョンを作成します

反復的なプロセスは Rork の最大の力の1つです。アプリがあなたのビジョンと一致するまで改善し続けます。

## ステップ8：ダウンロードまたはデプロイ

**時間：1～5分（プラットフォームによって異なります）**

### プロジェクトをダウンロード

アプリに満足したら：

**Rork の場合**：
1. 「コードをダウンロード」または「プロジェクトをエクスポート」をクリックします
2. アプリのソースコードは zip ファイルとしてダウンロードされます
3. 完全な React Native プロジェクトを取得します
4. 次のことができます：
   - React Native ツールでローカルで構築し続ける
   - App Store と Google Play に自分でデプロイする
   - さらにカスタマイズするために開発者とコードを共有する

**Rork Max の場合**：
1. 「App Store に公開」または同様をクリックします
2. Rork Max はクラウドでコンパイルを処理します
3. App Store に送信（2クリックプロセス）
4. Xcode インストール不要

### App Store のデプロイメント

**アプリを公開する**（より長いプロセス、しかし Rork は支援します）：

1. Apple Developer および/または Google Play Developer アカウントを作成します（Apple は年間 99 ドル、Google は 25 ドルの 1 回限りの費用）
2. App Store リストを準備します：
   - 説明
   - スクリーンショット
   - アイコン
   - プライバシーポリシー
   - キーワード/カテゴリ
3. Rork の場合：標準ツールを使用してアプリを送信します
4. Rork Max：2クリック送信を使用（さらにシンプル）
5. アプリレビューを待機します（通常 Google では 24 時間、Apple では 1～3 日）
6. アプリはアプリストアに表示されます

## ステップ9：共有してお祝い

アプリが作成されたら、次のことができます：

- **QR コードを共有**して、友人がテストするよう促します
- **App Store にデプロイ**して、数百万のユーザーに到達します
- **コードをダウンロード**して開発を継続します
- **別のアプリを作成**して同じプロセスを使用します

## 初心者向けの一般的な質問

### 「プロンプトで間違いを犯したらどうなりますか？」

問題ありません。次のことができます：
- プロンプトを編集して再生成します
- フォローアップのプロンプトでアプリを洗練します
- 削除して最初からやり直す
- 古いバージョンを保持して新しいバージョンを試す

### 「アプリを自分のバックエンドと組み合わせることはできますか？」

もちろんです。Rork はサポートしています：
- Supabase
- Firebase
- Airtable
- カスタム API
- 独自のバックエンド

プロンプトで特定のバックエンドとの統合を希望することを説明してください。

### 「生成されたコードは本当に使用可能ですか？」

はい。Rork は本番環境対応のコードを生成します。次のことができます：
- App Store に直接デプロイ
- 従来の開発でさらにカスタマイズする
- 改善のために開発者と共有する
- より大きなものの出発点として使用する

### 「Rork はどの言語をサポートしていますか？」

Rork がサポートしています：
- **Rork（オリジナル）**：React Native は JavaScript/TypeScript コードを生成します
- **Rork Max**：ネイティブ Swift コード

どちらも業界標準のコードをベストプラクティスで生成します。

### 「起動後にアプリを更新するにはどうすればよいですか？」

シンプル：
1. Rork に戻ります
2. 新機能でプロンプトを変更します
3. アプリを再生成します
4. 新しいバージョンをダウンロードします
5. App Store にアップデートをデプロイします

## 成功のコツ

<Callout type="tip">
**小さく始める**：最初のアプリはシンプルにしてください。カウンター、やることリスト、または単純なクイズです。基礎を習得してから複雑なアプリを構築してください。
</Callout>

**明確な説明が重要です**：アプリをよく説明するほど、結果は良くなります。色、ボタンの配置、ユーザーフローについての詳細を含めてください。

**徹底的にテスト**：App Store にデプロイする前に、実際のデバイスですべての機能をテストしてください。ボタンが機能することを確認し、データが保存され、体験がスムーズであることを確認します。

**オープンに反復する**：アプリを何度も再生成することを恐れないでください。Rork は高速な反復のために設計されています。

**結果から学ぶ**：最初のアプリを生成したら、コードを確認してください（ダウンロードした場合）。モダンなアプリがどのように機能するかの理解が始まります。

**コミュニティとリソース**：例とインスピレーションについては、Rork のドキュメントとコミュニティフォーラムを確認してください。
