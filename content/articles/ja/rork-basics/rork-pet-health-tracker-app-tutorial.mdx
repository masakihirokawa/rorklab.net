## はじめに — ペット健康管理アプリを Rork で作ろう

ペットを飼っている方なら、体重の変化や食事の内容、ワクチン接種の記録などを一元管理したいと思ったことがあるのではないでしょうか。Rork を使えば、こうした実用的なアプリをプログラミングの知識がなくても短時間で形にすることができます。

この記事では、Rork AI を使ってペットの健康データを記録・閲覧できるモバイルアプリを一から作る手順を解説します。フォーム入力、データの保存（AsyncStorage）、リスト表示といったアプリ開発の基本パターンを実践的に学べる内容です。

**この記事で学べること:**

- Rork のプロンプトでペット管理アプリの画面を生成する方法
- フォーム入力とバリデーションの基本
- AsyncStorage を使ったローカルデータ保存
- FlatList によるデータ一覧表示
- 日付ピッカーやカテゴリ選択の実装パターン

**対象読者:** Rork をこれから始める方、または基本操作を覚えたばかりの初心者の方

## 前提知識と環境準備

このチュートリアルを進めるには、以下の準備が必要です。

- **Rork アカウント**: 無料プラン（Free）でも基本的な機能は利用可能です。週5回までアプリを生成できます
- **ブラウザ**: Chrome または Safari の最新版を推奨
- **基本的な Rork の操作**: プロジェクト作成とプロンプト入力の経験があると理解が深まります

まだ Rork を使ったことがない方は、[Rork をはじめよう — 初心者のための完全入門ガイド](/articles/rork-basics/getting-started)を先にお読みいただくと、スムーズに進められます。

## ステップ1: プロジェクトを作成しよう

Rork にログインしたら、ダッシュボードの「New Project」ボタンをクリックして新しいプロジェクトを作成します。

プロジェクト名は「PetCare Tracker」など、わかりやすい名前を付けましょう。次に、以下のプロンプトを入力してアプリの骨格を生成します。

```text
ペットの健康管理アプリを作ってください。

必要な画面:
1. ホーム画面 — 登録済みペットの一覧をカードで表示
2. ペット登録画面 — 名前、種類（犬/猫/その他）、生年月日、体重を入力するフォーム
3. 健康記録画面 — 体重測定、食事内容、通院記録を日付付きで登録
4. 記録一覧画面 — 登録した記録を時系列で表示

デザインは柔らかいパステルカラーで、ペットのアイコンを使った親しみやすいUIにしてください。
```

このプロンプトを送信すると、Rork AI が4つの画面を含むアプリの初期構成を自動で生成してくれます。

## ステップ2: データモデルを設計する

Rork が生成した初期コードを確認したら、データの構造を整理しましょう。ペット健康管理アプリでは、大きく分けて2種類のデータを扱います。

```typescript
// ペット情報の型定義
// Type definition for pet information
interface Pet {
  id: string;
  name: string;           // ペットの名前
  species: 'dog' | 'cat' | 'other';  // 種類
  birthday: string;       // 生年月日（ISO形式）
  weight: number;         // 体重（kg）
  imageUri?: string;      // プロフィール写真（任意）
}

// 健康記録の型定義
// Type definition for health records
interface HealthRecord {
  id: string;
  petId: string;          // 紐づくペットのID
  type: 'weight' | 'meal' | 'vet';   // 記録種別
  date: string;           // 記録日
  value: string;          // 記録内容
  notes?: string;         // メモ（任意）
}
```

Rork のチャットで「上記のデータ構造に合わせてコードを修正してください」と指示すれば、自動的にコンポーネントが更新されます。

## ステップ3: フォーム入力画面を作り込む

ペット登録画面では、ユーザーが入力した情報を正しく受け取るためにバリデーション（入力チェック）を追加しましょう。

Rork に以下のプロンプトを送って、入力チェック付きのフォームを生成します。

```text
ペット登録フォームに以下のバリデーションを追加してください:
- 名前: 必須、1〜20文字
- 種類: 必須、選択式（犬/猫/その他）
- 生年月日: 必須、日付ピッカーで選択（未来の日付は不可）
- 体重: 必須、0.1〜200kgの範囲、小数点第1位まで

バリデーションエラーは入力欄の下に赤字で表示してください。
```

このように具体的な条件を指示することで、Rork AI は実用的なバリデーションロジックを含むフォームコンポーネントを生成します。

## ステップ4: AsyncStorage でデータを永続化する

アプリを閉じてもデータが消えないように、AsyncStorage を使ってローカルにデータを保存しましょう。React Native の AsyncStorage は、ブラウザの localStorage に似たシンプルなキーバリューストアです。

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

// ペット一覧を保存する関数
// Function to save pet list
const savePets = async (pets: Pet[]): Promise<void> => {
  try {
    const jsonValue = JSON.stringify(pets);
    await AsyncStorage.setItem('@petcare_pets', jsonValue);
  } catch (e) {
    console.error('Failed to save pets:', e);
  }
};

// ペット一覧を読み込む関数
// Function to load pet list
const loadPets = async (): Promise<Pet[]> => {
  try {
    const jsonValue = await AsyncStorage.getItem('@petcare_pets');
    return jsonValue != null ? JSON.parse(jsonValue) : [];
  } catch (e) {
    console.error('Failed to load pets:', e);
    return [];
  }
};

// 使用例: アプリ起動時にデータを読み込む
// Usage: Load data on app launch
const [pets, setPets] = useState<Pet[]>([]);

useEffect(() => {
  loadPets().then(setPets);
}, []);
```

**期待する動作**: アプリを起動すると `loadPets` が実行され、以前登録したペットのデータがメモリに読み込まれます。新しいペットを登録すると `savePets` が呼ばれ、データがストレージに永続化されます。

## ステップ5: 記録一覧をリスト表示する

健康記録を時系列で見やすく表示するには、React Native の FlatList コンポーネントを使います。Rork に以下のようなプロンプトを送って実装しましょう。

```text
健康記録の一覧画面を改善してください:
- FlatList で記録を新しい順に表示
- 記録タイプ（体重/食事/通院）ごとにアイコンと色を分ける
- 体重: 青、スケールアイコン
- 食事: 緑、フォークアイコン
- 通院: オレンジ、病院アイコン
- 各カードに日付と内容を表示
- プルダウンリフレッシュに対応
```

FlatList は大量のデータでもスムーズにスクロールできるため、記録が増えてもパフォーマンスが落ちにくいというメリットがあります。

## よくあるエラーと対処法

### AsyncStorage のデータが消える

開発中に Expo のキャッシュをクリアすると AsyncStorage のデータも消えます。テスト用のダミーデータを簡単に投入できる開発者メニューを用意しておくと便利です。

### 日付ピッカーが表示されない

プラットフォーム（iOS / Android）によって日付ピッカーの挙動が異なります。`@react-native-community/datetimepicker` を使う場合は、Rork に「iOS と Android の両方で動作する日付ピッカーを使ってください」と明示しましょう。

### 画像が表示されない

ペットのプロフィール写真を扱う場合は、`expo-image-picker` を使います。Rork のプロンプトに「カメラロールから画像を選択できるようにしてください」と指示すれば、必要なパーミッション設定も含めて実装してくれます。

## 応用 — さらに便利にするアイデア

基本機能が完成したら、以下の機能を追加してアプリの完成度を高められます。

- **リマインダー通知**: ワクチン接種日や定期検診の日程を通知する機能。[プッシュ通知の実装方法](/articles/rork-dev/rork-push-notifications)も参考になります
- **体重グラフ**: 記録した体重データをグラフで可視化。`react-native-chart-kit` を使えば折れ線グラフを簡単に追加できます
- **複数ペット対応**: タブやドロワーナビゲーションでペットを切り替えられるようにする
- **データエクスポート**: 通院時に獣医に見せるための PDF 出力機能

## FAQ

### Q: Rork の無料プランでもこのアプリは作れますか？

はい、無料プラン（Free）でも十分に作成可能です。週5回までの生成制限がありますが、プロンプトを工夫してまとめて指示すれば、数回のやり取りで基本機能を完成させられます。

### Q: データはクラウドに保存できますか？

このチュートリアルでは端末内（AsyncStorage）にデータを保存していますが、Supabase や Firebase を連携すればクラウド保存にも対応できます。Rork は Supabase との連携機能を標準で備えているため、比較的簡単にクラウド対応へ移行できます。

### Q: iOS と Android の両方で動きますか？

はい、Rork は React Native / Expo ベースでアプリを生成するため、1つのコードから iOS と Android の両プラットフォーム向けにビルドできます。ただし、カメラや通知など一部の機能はプラットフォームごとの挙動の違いに注意が必要です。

### Q: アプリを App Store に公開するにはどうすればいいですか？

Rork で作成したアプリは、Expo の EAS Build を使って実機用のバイナリを作成し、App Store や Google Play に提出できます。公開手順の詳細は [30分で初めての Rork アプリを作って公開する方法](/articles/rork-basics/rork-first-app-30-minutes-guide) でも解説しています。
