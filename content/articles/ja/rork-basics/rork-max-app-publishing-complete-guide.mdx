Rork Max でアプリが完成したとき、次の壁は「どうやって App Store に出すか」です。Apple の審査プロセスは初めてだとかなり複雑に見えます。しかし手順を整理すると、つまずきやすいポイントはいくつかに絞られます。

この記事では、Rork Max で作ったアプリを App Store に公開するまでの流れを、実際に引っかかりやすいポイントを中心に解説します。

## 前提：必要なもの

まず以下を準備してください。

- **Apple Developer Program への登録**（年間 $99、個人または法人）
  - [Apple Developer](https://developer.apple.com) からサインアップ
  - 審査に 24〜48 時間かかることがあります
- **Mac（Xcode が必要）**
  - Rork Max から直接ビルドを書き出す場合でも、最終的には Xcode から App Store Connect へアップロードします
- **App Store Connect のアカウント**
  - Developer Program に参加すると自動的にアクセスできます

## Step 1: Rork Max でのエクスポート設定

Rork Max の「Export」メニューから iOS 向けビルドを書き出します。

```
Rork Max → File → Export → iOS App
```

エクスポート時に以下を設定します。

- **Bundle Identifier**: `com.yourname.appname`（一度決めると変更が難しいので慎重に）
- **Display Name**: App Store に表示されるアプリ名
- **Version**: `1.0.0`（最初のリリース）
- **Build Number**: `1`（リリースするたびに増やす）

## Step 2: Xcode での設定

Rork Max からエクスポートされたプロジェクトを Xcode で開きます。

**確認すべき項目**:

```
Project Settings → Signing & Capabilities
- Team: 自分の Apple Developer チームを選択
- Bundle Identifier: Rork Max で設定したものと一致しているか確認
- Automatically manage signing: チェックを入れる
```

**Icons の設定**:
App Store 審査では、アイコンに透明度（Alpha チャンネル）が含まれているとリジェクトされます。Rork Max のアイコン出力を確認し、透明ピクセルがないかチェックしてください。

## Step 3: App Store Connect でのアプリ作成

[App Store Connect](https://appstoreconnect.apple.com) にログインし、「マイ App」→「+」から新規アプリを作成します。

**必要な情報**:
- アプリ名（30 文字以内）
- Bundle ID（Step 1 で設定したもの）
- SKU（内部管理用。英数字で自由に設定）
- プライマリ言語

## Step 4: アプリ情報の入力

App Store ページの表示内容を入力します。審査通過率に影響する重要な項目です。

**説明文のポイント**:
最初の 3 行（プレビュー）が最も重要です。ユーザーが「続きを読む」を押さなくても価値が伝わるように書きます。

**スクリーンショットの要件**:
- iPhone 6.5 インチ（必須）
- iPhone 5.5 インチ（必須）
- iPad は提出する場合のみ
- シミュレータのスクリーンショットでも審査は通ります

**プライバシーポリシー**:
個人データを収集しないアプリでも、App Store 申請にはプライバシーポリシーの URL が必要です。GitHub Pages や Notion 等で簡単なページを作成してください。

## Step 5: ビルドのアップロード

Xcode で Archive → Distribute App の順に進みます。

```
Xcode → Product → Archive
→ Distribute App
→ App Store Connect
→ Upload
```

アップロードが完了すると、App Store Connect の「TestFlight & ビルド」にビルドが表示されます（反映まで最大 30 分かかることがあります）。

## 審査でよく引っかかるポイント

**1. クラッシュレポート**
審査チームが実際にアプリを動かして、クラッシュが発生するとリジェクトされます。TestFlight で事前に動作確認を徹底してください。

**2. 誤解を招く説明**
アプリの機能を誇張した説明や、実装されていない機能の記載はリジェクト原因になります。

**3. サインインの強制**
コア機能を使うために会員登録を強制するアプリはリジェクトされることがあります。「ゲストで試す」オプションを提供するか、登録が必要な理由を明確にしてください。

**4. アイコンの透明度**
前述のとおり、アイコンに Alpha チャンネルがあるとリジェクトされます。

## 審査期間の目安

初回申請では通常 24〜48 時間で審査結果が出ます。リジェクトされた場合、理由を修正して再申請すれば多くのケースで通過できます。

App Store Connect でリジェクト理由を確認し、対応内容を「審査に関するメモ」に記載して再申請するのが効果的です。

## まとめ

Rork Max でのアプリ公開は、慣れてしまえばそれほど難しくありません。最初の申請は手順を覚えるのに時間がかかりますが、2 本目以降は大幅に短縮できます。
