## プラグインシステムの概要

Claude Code のプラグインは、Claude Code の機能を拡張するためのパッケージです。以下の5つのコンポーネントを含めることができます：

- **Skills** — プロンプトテンプレートやワークフロー定義
- **Agents** — 特定タスクに特化した自律エージェント
- **MCP servers** — 外部サービスとの連携（Model Context Protocol）
- **Hooks** — イベント駆動の自動化（ファイル変更時の自動処理など）
- **LSP servers** — 言語サーバープロトコルによるエディタ統合

## プラグインの構造

プラグインは以下のディレクトリ構造で構成されます：

```
my-plugin/
├── plugin.json        # マニフェストファイル（必須）
├── skills/
│   └── my-skill/
│       └── SKILL.md   # スキル定義
├── agents/
│   └── my-agent.md    # エージェント定義
├── mcp/
│   └── server.js      # MCP サーバー実装
└── hooks/
    └── on-save.sh     # フック処理
```

## plugin.json マニフェスト

プラグインの心臓部である `plugin.json` の基本構造：

```json
{
  "name": "my-awesome-plugin",
  "version": "1.0.0",
  "description": "プロジェクトのコード品質を自動チェックするプラグイン",
  "author": "your-name",
  "components": {
    "skills": ["skills/code-review"],
    "mcpServers": {
      "quality-check": {
        "command": "node",
        "args": ["mcp/server.js"]
      }
    }
  }
}
```

<Callout type="info">
プラグインマニフェストの全フィールド仕様は、公式ドキュメントの Plugins Reference で確認できます。
</Callout>

## インストールスコープ

プラグインは4つのスコープでインストールできます：

| スコープ | 説明 | 使用場面 |
|---------|------|---------|
| `user` | ユーザーグローバル | 個人の共通ツール |
| `project` | プロジェクト固有 | チーム共有の設定 |
| `local` | ローカルのみ | 実験的なプラグイン |
| `managed` | 組織管理 | エンタープライズ向け |

## CLI でのプラグイン管理

```bash
# インストール
claude plugin install @example/my-plugin

# アンインストール
claude plugin uninstall @example/my-plugin

# 有効化 / 無効化
claude plugin enable my-plugin
claude plugin disable my-plugin

# アップデート
claude plugin update my-plugin

# 一覧表示
claude plugin list
```

## 実践：シンプルなスキルプラグインを作る

最も簡単なプラグインの例として、コードレビュー用のスキルを作ってみましょう。

### 1. ディレクトリ作成

```bash
mkdir -p my-review-plugin/skills/review
```

### 2. plugin.json を作成

```json
{
  "name": "code-review-ja",
  "version": "0.1.0",
  "description": "日本語でコードレビューを行うスキル",
  "components": {
    "skills": ["skills/review"]
  }
}
```

### 3. スキルファイルを作成

`skills/review/SKILL.md`:

```markdown
---
description: "コードの品質をレビューし、日本語で改善提案を行う"
---

# コードレビュースキル

以下の観点でコードをレビューしてください：

1. **バグの可能性** — 潜在的なバグやエッジケース
2. **パフォーマンス** — 非効率な処理やN+1問題
3. **セキュリティ** — 脆弱性やインジェクションリスク
4. **可読性** — 命名規則、コメント、構造の明確さ
5. **テスト** — テストカバレッジの不足

各指摘には「重要度（高/中/低）」と「修正例」を付けてください。
```

### 4. ローカルインストール

```bash
claude plugin install ./my-review-plugin --scope local
```

<Callout type="tip">
開発中は `--scope local` でインストールし、完成したら npm に公開して他のユーザーと共有できます。
</Callout>

## MCP サーバーの統合

プラグインに MCP サーバーを含めると、Claude Code から外部 API やデータベースにアクセスできます。

```json
{
  "components": {
    "mcpServers": {
      "my-db": {
        "command": "npx",
        "args": ["-y", "@example/db-mcp-server"],
        "env": {
          "DATABASE_URL": "${DATABASE_URL}"
        }
      }
    }
  }
}
```

<Callout type="warning">
MCP サーバーはローカル環境で実行されます。信頼できるソースからのみインストールし、環境変数で機密情報を管理してください。
</Callout>
