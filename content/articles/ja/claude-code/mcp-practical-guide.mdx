## MCP が変える Claude の開発体験

MCP（Model Context Protocol）は 2024 年 11 月に Anthropic が発表したオープン標準プロトコルです。AI モデルと外部ツール・データソースを統一的に接続するインターフェースとして設計され、わずか1年で月間 9,700 万以上の SDK ダウンロードと 10,000 以上のアクティブサーバーを持つエコシステムへと成長しました。

Claude Code、Claude Desktop、さらには Cursor や VS Code など多くのクライアントが MCP をネイティブサポートしており、MCP サーバーを追加するだけで Claude の能力を無限に拡張できます。

<Callout type="info">
**MCP のコア概念：**
- **ツール（Tools）** — Claude が実行できるアクション（API 呼び出し、DB クエリなど）
- **リソース（Resources）** — Claude が読み取れるデータソース（ファイル、DB レコードなど）
- **プロンプト（Prompts）** — 事前定義されたテンプレートやワークフロー
</Callout>

## おすすめ MCP サーバー 7 選

### 1. GitHub MCP Server

GitHub 公式の MCP サーバーは、Claude Code ユーザーの 9 割以上が最初に導入するサーバーです。リポジトリ操作、PR の作成・レビュー、Issue 管理、コード検索、CI/CD 状況の確認まで、GitHub のほぼ全機能に Claude からアクセスできます。

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    }
  }
}
```

**活用シナリオ：** 「この PR のレビューコメントを確認して修正して」「Issue #42 の内容に基づいてブランチを切ってコードを書いて」といった指示が自然言語で通ります。

### 2. Filesystem MCP Server

ローカルファイルシステムへのアクセスを提供します。明示的な許可ゲートがあるため安全に使え、ファイルの読み書き、検索、整理が Claude から直接行えます。

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
```

**活用シナリオ：** プロジェクトフォルダ全体を Claude に渡して「このディレクトリのファイル構造を整理して」「古いログファイルを見つけて削除して」などのファイル管理タスクに最適です。

### 3. Figma MCP Server（Dev Mode）

Figma の公式 Dev Mode MCP サーバーは、デザインデータを直接 Claude に渡してコードを生成できます。スクリーンショットではなく実際のデザイン構造（レイヤー、コンポーネント、Auto Layout、デザイントークン）を理解するため、高精度なコード変換が可能です。

**活用シナリオ：** 「この Figma のフレームを React コンポーネントにして」「デザイントークンに合わせてスタイルを更新して」

### 4. Sentry MCP Server

リアルタイムのエラートラッキングと連携するサーバーです。Sentry の UI を開かずに、Claude がエラーの全コンテキスト（スタックトレース、ユーザー影響範囲、発生頻度）を取得し、修正案を提示できます。

**活用シナリオ：** 「今日発生したエラーの一覧を見せて」「この TypeError の原因を特定して修正コードを書いて」

### 5. Brave Search MCP Server

Web 検索機能を Claude に追加します。開発中に最新のドキュメントやライブラリ情報を確認したい場合に、チャットの中で直接検索結果を取得できます。

### 6. PostgreSQL / SQLite MCP Server

データベースに直接クエリを実行できるサーバーです。開発中のデータ確認、テストデータの投入、スキーマの分析などを Claude 経由で行えます。

### 7. Cloudflare MCP Server

Cloudflare Workers、KV、D1、R2 などのリソース管理を Claude から行えます。デプロイ状況の確認やワーカーの設定変更に便利です。

## リモート MCP サーバーと認証

MCP は当初ローカル実行（stdio）が中心でしたが、現在はリモートサーバーもサポートしています。

### Streamable HTTP トランスポート

現在の MCP 標準トランスポートは **Streamable HTTP** です。従来の SSE（Server-Sent Events）方式は段階的に非推奨となっています。

### OAuth 2.1 認証

リモート MCP サーバーは OAuth 2.1 プロトコルによる認証をサポートしています。Claude の OAuth コールバック URL は `https://claude.ai/api/mcp/auth_callback` で、トークンの有効期限とリフレッシュにも対応しています。

<Callout type="warning">
**セキュリティのベストプラクティス：**
- 各 MCP サーバーをマイクロサービスとして扱い、影響範囲を限定する
- TLS を必須にし、クライアント・サーバー間の認証を要求する
- API キーのスコープを最小限にし、高影響サーバーは読み取り専用で開始する
- `GITHUB_PERSONAL_ACCESS_TOKEN` などの認証情報は環境変数で管理する
</Callout>

## 実践シナリオ：MCP で開発フローを自動化

### シナリオ 1：Issue → コード → PR の完全自動化

GitHub MCP + Filesystem MCP を組み合わせることで、Issue の内容を読み取り、ブランチを作成、コードを書いて PR を出すまでを Claude に任せられます。

```
あなた: Issue #15 の機能要件を読んで実装して、PR を作成して
Claude: Issue #15 を確認します...「ダークモード対応」ですね。
        feature/dark-mode ブランチを作成して実装を始めます。
        ...（コーディング）...
        PR #23 を作成しました。レビューをお願いします。
```

### シナリオ 2：エラー検知 → 原因特定 → 修正

Sentry MCP + GitHub MCP の連携で、本番エラーの検知から修正 PR の作成まで一気通貫で対応できます。

### シナリオ 3：デザイン → コード → デプロイ

Figma MCP → Filesystem MCP → GitHub MCP → Cloudflare MCP を連携させれば、デザインからデプロイまでの全工程を Claude が支援します。

## MCP サーバーの設定方法

### Claude Code での設定

```bash
# プロジェクトスコープ（.mcp.json）
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxx -- npx -y @modelcontextprotocol/server-github

# グローバルスコープ
claude mcp add --scope user filesystem -- npx -y @modelcontextprotocol/server-filesystem ~/projects
```

### Claude Desktop での設定

`claude_desktop_config.json` に MCP サーバーの設定を追加します。

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/projects"]
    }
  }
}
```

## まとめ

MCP は Claude の能力を外部ツールと接続する標準プロトコルとして急速に普及しています。GitHub、Figma、Sentry などの主要サーバーを組み合わせることで、Issue 管理からデザイン変換、エラー対応、デプロイまでの開発ワークフロー全体を Claude が支援できるようになります。

まずは GitHub MCP サーバーから導入し、プロジェクトの必要に応じてサーバーを追加していくのが実践的なアプローチです。
