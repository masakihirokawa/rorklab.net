## skills.sh とは

skills.sh は、Claude Code、Cowork、その他の AI エージェントの機能を拡張するオープンソースのエージェントスキルマーケットプレイスです。[agentskills.io](https://agentskills.io/specification) で定義されたオープンスタンダードに基づいており、開発者が新しい機能を発見、インストール、公開できる統一されたエコシステムを提供します。

2026年3月現在、スキルマーケットプレイスは数千個の検証済みスキルをホストしており、創造的なアプリケーション（アート、音楽、デザイン）、技術的なタスク（テスト、コードレビュー、Git自動化）、エンタープライズワークフロー（コミュニケーション、ブランディング、データ処理）にまたがっています。

<Callout type="info">
スキルとスラッシュコマンドは根本的に異なります。スキルはモデル呼び出し型です—Claude がコンテキストに基づいて自動的に使用するかどうかを判断します。一方、スラッシュコマンドはユーザー呼び出し型です—ユーザーがコマンドを明示的に入力してトリガーします。つまり、スキルはユーザーの明示的な操作を必要としない、より知的でコンテキスト対応な自動化を実現します。
</Callout>

## スキルのアーキテクチャを理解する

スキルは以下を含むモジュール機能です：

- **SKILL.md ファイル**: メタデータと指示を含む YAML フロントマター付きマークダウンファイル
- **メタデータ**: スキルの目的、バージョン、作成者、トリガーに関する情報
- **指示**: このスキルをどのように使用すべきかについての詳細なガイダンス
- **オプションリソース**: スクリプト、参照資料、アセットなどのサポートファイル

スキルは2つの可能な場所に保存されます：
- **グローバルインストール**: すべてのプロジェクトで利用可能な `~/.claude/skills/`
- **プロジェクト固有のインストール**: プロジェクトディレクトリ内の `./.claude/skills/`

モジュール設計により、エージェントが必要に応じて自動的に関連スキルを読み込み、シームレスで文脈に対応したエクスペリエンスを実現します。

## Skills CLI のインストール

Skills CLI はスキルを管理するためのコマンドラインツールです。npm を使用してインストールします：

```bash
npm install -g skills
```

またはインストールせずにコマンドを実行するために npx を使用します：

```bash
npx skills --help
```

<Callout type="tip">
Node.js を使用している場合、おそらく npm がすでにインストールされています。`npm --version` と `node --version` で設定を確認してから進めてください。
</Callout>

## スキルの検索と導入

### スキルを検索する

ニーズに合致するスキルをマーケットプレイスで検索します：

```bash
npx skills find [クエリ]
```

**検索例：**

```bash
# React と Next.js 開発スキルを検索
npx skills find react

# デプロイ関連のスキルを検索
npx skills find deploy

# テストフレームワークとツールを検索
npx skills find testing

# Git 自動化スキルを検索
npx skills find git

# コードレビューとリント スキルを検索
npx skills find review
```

各検索結果には、スキルの名前、説明、バージョン、作成者情報が表示されます。これにより、スキルが特定の要件と一致しているかを評価できます。

### スキルをインストールする

スキルを特定したら、以下のコマンドでインストールします：

```bash
npx skills add owner/repo@skill-name
```

**インストール例：**

```bash
# 特定のスキルをプロジェクトにインストール
npx skills add vercel/skills@react-best-practices

# グローバルインストール（すべてのプロジェクトで利用可能）
npx skills add vercel/skills@react-best-practices -g

# 確認プロンプトなしでサイレントインストール
npx skills add vercel/skills@react-best-practices -y

# グローバルかつサイレントにインストール
npx skills add vercel/skills@react-best-practices -g -y
```

グローバルインストール（`-g` フラグ）はすべてのプロジェクトでスキルを利用可能にし、プロジェクトレベルのインストールはスキルを現在のプロジェクトの `.claude/skills/` ディレクトリにのみ保存します。

<Callout type="warning">
インストール前に常にスキルのソースと説明を確認してください。マーケットプレイスはコミュニティが検証したスキルを含みますが、作成者を信頼し、スキルがワークフローで何をするかを理解していることを確認してください。
</Callout>

## スキルの管理

### 更新をチェックする

インストール済みのすべてのスキルを表示し、利用可能な更新をチェックします：

```bash
npx skills check
```

このコマンドは、より新しいバージョンが利用可能なスキルを表示し、更新に関するガイダンスを提供します。

### スキルを更新する

最新の改善とバグ修正で、スキルを最新の状態に保ちます：

```bash
npx skills update
```

これはすべてのインストール済みスキルを最新バージョンに更新します。スキル名を指定して個別のスキルを更新することもできます。

### インストール済みスキルをリストアップする

現在のプロジェクトにインストールされているすべてのスキルを表示します：

```bash
npx skills list
```

### スキルを削除する

使用しなくなったスキルを削除します：

```bash
npx skills remove [skill-name]
```

または対話形式の削除ツールを使用します：

```bash
npx skills remove
```

これにより、アンインストールするスキルを選択するメニューが表示されます。

## SKILL.md フォーマットを理解する

すべてのスキルは YAML フロントマターとマークダウンコンテンツを含む `SKILL.md` ファイルで定義されます。

### フロントマター構造

```yaml
---
name: "React ベストプラクティス"
description: "React コンポーネントパターン、フック のベストプラクティス、パフォーマンス最適化について Claude をガイドします"
version: "1.0.0"
author: "Vercel Labs"
license: "MIT"
triggers: ["react", "component", "hooks"]
metadata:
  category: "development"
  difficulty: "intermediate"
  keywords: ["react", "javascript", "frontend"]
---
```

**フロントマターフィールド：**

| フィールド | 説明 | 必須 |
|-----------|------|------|
| `name` | スキルの表示名 | はい |
| `description` | スキルの目的の簡潔な説明 | はい |
| `version` | セマンティックバージョン番号 | はい |
| `author` | スキルの作成者/メンテナ | はい |
| `license` | オープンソースライセンスの種類 | はい |
| `triggers` | このスキルをアクティベートするキーワード | いいえ |
| `metadata` | 追加の分類とタギング | いいえ |

### マークダウンコンテンツ

フロントマターの後、エージェントがこのスキルを使用する方法について明確な指示を記述します：

```markdown
# React ベストプラクティス

React 開発でユーザーをサポートする際に、以下のパターンを適用してください：

## コンポーネント構造
- フック付きの関数型コンポーネントを使用
- 複雑なロジックをカスタムフックに抽出
- コンポーネントを焦点を絞った単一責任にする

## パフォーマンス最適化
- 高コストなコンポーネントに React.memo() を使用
- 高コストな計算に useMemo() を実装
- React DevTools Profiler でプロファイリング

## テストベストプラクティス
- ユーザー動作に焦点を当てたテストを作成
- React Testing Library を使用
- 実装詳細のテストは避ける
```

<Callout type="tip">
SKILL.md の指示は簡潔に保ち、5,000 トークン未満にしてください。Claude がワークフロー内でより良い判断を下すのに役立つ本質的なガイダンスに焦点を当ててください。
</Callout>

## カスタムスキルの作成

### 新しいスキルを初期化する

新しいスキルプロジェクト構造を作成します：

```bash
npx skills init my-custom-skill
```

これにより以下が生成されます：
- テンプレートフロントマターとコンテンツを持つ `SKILL.md`
- スクリプト、参照資料、アセット用のオプションディレクトリ
- 基本的なプロジェクト構造

### スキル開発ワークフロー

1. **目的を定義する**: スキルが対応するタスクまたはドメインを明確に述べる
2. **フロントマターを記述する**: すべての必須メタデータフィールドを完成させる
3. **指示をドキュメント化する**: AI エージェントのための明確で実用的なガイダンスを提供
4. **リソースを追加する**（オプション）: サポートディレクトリにスクリプトまたは参照ファイルを含める
5. **ローカルでテストする**: 公開前に Claude Code でスキルが機能することを確認
6. **GitHub に公開する**: スキルを共有するためにリポジトリをプッシュ

### カスタムスキルの例

自動テストなどの特定のワークフロー用のスキルを作成します：

```yaml
---
name: "自動テスト生成"
description: "Jest を使用して JavaScript 関数の包括的なテストスイートを生成"
version: "1.0.0"
author: "あなたの名前"
license: "MIT"
triggers: ["test", "jest", "testing"]
---

# 自動テスト生成

JavaScript 関数のテストを作成するよう要求されたときは：

## テスト構造
- テストファイルを `[function].test.js` という名前で作成
- テストを論理的な describe() ブロックに整理
- テスト名は明確で説明的にする

## カバレッジ要件
- ハッピーパスシナリオをテスト
- エッジケースとエラー条件をテスト
- 最小80%のコードカバレッジを達成
- 複雑な関数の統合テストを含める

## ベストプラクティス
- テストセットアップに beforeEach() を使用
- 外部依存関係をモック化
- アサーションは焦点を絞り具体的にする
```

### スキルのテスト

公開前に Claude Code でスキルをテストします：

```bash
# スキルをローカルスキルディレクトリにコピー
cp -r my-custom-skill ~/.claude/skills/

# Claude Code でテスト
npx claude code "test my new skill"
```

## Cowork でのスキル活用

デスクトップアプリケーション Cowork は、互換性のあるスキルを自動的に読み込み、生産性を向上させます。

### ビルトインスキルバンドル

Cowork には専門的なスキルコレクションが含まれています：

- **PDF スキル**: PDF 文書の抽出、注釈、変換
- **Excel スキル**: データ分析、ピボットテーブル作成、数式構築
- **PowerPoint スキル**: プレゼンテーション設計、ビジュアルコンテンツ作成
- **ドキュメント スキル**: 文書の編集、フォーマット、構造化

### Cowork でスキルをアクティベートする

スキルはスキルのトリガーと一致するドキュメントタイプを開くと、自動的に検出されて読み込まれます。Cowork の設定パネルでスキルを手動で有効/無効にすることもできます。

<Callout type="info">
Cowork はグローバルな `~/.claude/skills/` ディレクトリとプロジェクト固有の `.claude/skills/` フォルダからスキルを同期します。スキルは Cowork のすべてのドキュメントで利用可能です。
</Callout>

## スキルの公開

### リポジトリの準備

1. スキル用の GitHub リポジトリを作成
2. リポジトリを構造化：
   ```
   my-skill/
   ├── SKILL.md
   ├── scripts/
   ├── references/
   ├── assets/
   └── README.md
   ```

3. スキルの目的と使用法を説明する包括的な README を記述
4. リポジトリに `agent-skill` と `skills` ラベルをタグ付け

### skills.sh に登録する

1. スキルリポジトリを GitHub にプッシュ
2. skills.sh レジストリにアクセスしてスキルを送信
3. 以下を提供：
   - リポジトリ URL
   - スキル名と説明
   - カテゴリとキーワード
   - ライセンスタイプ

4. コミュニティがスキルをレビューして検証
5. 承認後、マーケットプレイス検索に表示

### ドキュメント作成のベストプラクティス

- README に使用例を含める
- スキルが何をするかについて明確な説明を提供
- 外部依存関係をドキュメント化
- よくある問題のトラブルシューティングセクションを追加
- 更新を記録した CHANGELOG を保持

## トラブルシューティング

### 検索でスキルが見つからない

**問題**: `npx skills find [クエリ]` で結果が返されない

**解決策：**
- より広い検索用語を試す（「react-hooks」ではなく「react」）
- GitHub でスキル名を確認し、作成者で検索
- マーケットプレイスがアクセス可能か確認: `npx skills check-connection`

### パーミッションエラーでインストール失敗

**問題**: パーミッション拒否でインストール失敗

**解決策：**
```bash
# グローバルインストールに sudo を使用
sudo npx skills add owner/repo@skill-name -g

# または npm パーミッションを修正
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH
```

### Claude がスキルを使用していない

**問題**: インストール済みスキルが関連タスクに役立たない

**解決策：**
- インストール確認: `npx skills list`
- スキルトリガー確認: `npx skills info owner/repo@skill-name`
- SKILL.md の指示の明確さをレビュー
- 明示的な言及でテスト: 「このタスクにカスタムスキルを使用してください」

### スキルの競合

**問題**: 複数のスキルが予期しない動作を引き起こす

**解決策：**
- 競合スキルを削除: `npx skills remove conflicting-skill`
- プロジェクト固有のインストールを使用してスキルを隔離
- プロジェクト README にスキル互換性をドキュメント化

### 更新失敗

**問題**: `npx skills update` が失敗またはロールバック

**解決策：**
```bash
# 特定のスキルを強制更新
npx skills add owner/repo@skill-name --force

# スキルバージョン互換性をチェック
npx skills check

# 必要に応じて前のバージョンにリバート
npx skills add owner/repo@skill-name@0.9.0
```

## 高度な設定

### プロジェクトレベル設定

`.claude/skills.config.json` を作成してスキル動作をカスタマイズ：

```json
{
  "autoLoad": true,
  "maxTokens": 5000,
  "enabledSkills": [
    "vercel/skills@react-best-practices",
    "composio/skills@git-automation"
  ],
  "disabledSkills": [
    "old-skill@outdated"
  ]
}
```

### 環境変数

環境変数によるスキル設定：

```bash
# 特定のスキルを無効化
export DISABLED_SKILLS="old-skill,experimental-skill"

# 最大トークン使用量を設定
export SKILL_MAX_TOKENS=8000

# カスタムスキルレジストリを使用
export SKILLS_REGISTRY="https://custom-registry.example.com"
```

## まとめ

skills.sh マーケットプレイスは、Claude の機能を特殊化されたドメイン固有の専門知識で拡張する力をあなたに与えます。開発ワークフローの構築、ドキュメント編集の強化、またはビジネスプロセスの自動化など、スキルは Claude Code と Cowork を特定のニーズに合わせてカスタマイズするための強力な方法を提供します。

マーケットプレイスを探索することから始め、ワークフローに合致するスキルをインストールし、他の開発者がより良い AI 駆動型アプリケーションを構築するのに役立つ独自のスキルの貢献を検討してください。
