## CLAUDE.md とは

**CLAUDE.md** は、Claude Code がセッション開始時に自動的に読み込むマークダウンファイルです。プロジェクトのルートディレクトリに配置することで、コーディング規約、アーキテクチャの決定事項、使用ライブラリ、レビューチェックリストなどを Claude に伝えることができます。

## なぜ必要なのか

Claude Code は毎回新しいセッションを開始するため、前回のやり取りの文脈を持っていません。CLAUDE.md があれば、毎回同じ説明をする必要がなくなり、プロジェクト固有のルールを一貫して適用できます。

## CLAUDE.md の書き方

プロジェクトのルートに `CLAUDE.md` を作成します。

```markdown
# プロジェクト概要
ECサイトのバックエンドAPI（Node.js + Express + TypeScript）

# コーディング規約
- 関数は必ず TypeScript の型注釈をつける
- エラーハンドリングには Result 型パターンを使用
- テストは Jest を使用し、各関数に最低1つのテストを書く

# アーキテクチャ
- src/routes/ — API ルート定義
- src/services/ — ビジネスロジック
- src/models/ — データモデル（Prisma）
- src/middleware/ — 認証・バリデーションミドルウェア

# ビルド・テスト
- ビルド: npm run build
- テスト: npm test
- 開発サーバー: npm run dev

# 注意事項
- データベースは PostgreSQL を使用
- 環境変数は .env.local に定義（コミットしない）
- API レスポンスは必ず { success, data, error } 形式
```

## 配置場所と優先順位

CLAUDE.md は複数の場所に配置でき、それぞれ異なるスコープを持ちます。

1. **プロジェクトルート** (`./CLAUDE.md`): そのプロジェクト全体に適用
2. **サブディレクトリ** (`./src/CLAUDE.md`): 特定ディレクトリに適用
3. **ホームディレクトリ** (`~/.claude/CLAUDE.md`): すべてのプロジェクトに共通適用

複数のファイルがある場合、すべてが読み込まれ、より具体的なスコープが優先されます。

## Auto Memory

Claude Code には **Auto Memory** 機能があり、作業中に学習した内容を自動的に記憶します。

### 何が記憶されるか

- プロジェクトのビルドコマンドとその結果
- デバッグ時に発見した問題と解決策
- よく使うファイルパスやパターン
- ユーザーの好みやワークフロー

### メリット

Auto Memory を使えば、CLAUDE.md を手動で更新しなくても、Claude Code が使い込むほどプロジェクトへの理解が深まります。次回のセッションでも前回学んだことが活かされます。

## チームでの活用

CLAUDE.md をリポジトリにコミットすれば、チームメンバー全員が同じ指示を Claude Code に共有できます。

```
# レビューチェックリスト
PR を作成する前に以下を確認:
- [ ] TypeScript の型エラーがないこと（tsc --noEmit）
- [ ] テストがすべてパスすること（npm test）
- [ ] ESLint の警告がないこと（npm run lint）
- [ ] API の変更がある場合は OpenAPI スペックを更新
```

## 効果的な書き方のコツ

1. **簡潔に書く**: Claude が毎回読むので、不要な情報は省く
2. **構造化する**: ヘッダーやリストを使って整理する
3. **更新を続ける**: プロジェクトの変化に合わせて CLAUDE.md も更新する
4. **具体的に書く**: 「良いコードを書いて」ではなく「関数は30行以内にする」のように具体的に

## まとめ

CLAUDE.md は Claude Code のパフォーマンスを大きく左右するファイルです。5分かけて書くだけで、毎回の対話品質が向上し、チーム全体の生産性アップにつながります。Auto Memory と組み合わせれば、Claude Code は使えば使うほど賢くなります。
