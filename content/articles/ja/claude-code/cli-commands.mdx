## Claude Code の CLI 体系

Claude Code は `claude` コマンドを中心に、多彩なサブコマンドとフラグを提供しています。このガイドでは、公式CLIリファレンスに基づいて全コマンドを体系的に解説します。

## 基本コマンド

### 対話モード

```bash
claude
```

引数なしで実行すると対話モードが起動します。チャット形式で Claude と会話しながらコーディングを進められます。

### クイック質問

```bash
claude "このプロジェクトの構造を説明して"
```

短いクエリを直接渡すと、対話モードで即座にそのプロンプトが送信されます。

### ワンショットモード（-p）

```bash
claude -p "README.mdを日本語に翻訳して"
```

`-p`（`--print`）フラグで非対話的に単発タスクを実行します。結果を標準出力に出力するため、パイプやリダイレクトとの組み合わせに最適です。

```bash
# パイプとの組み合わせ例
cat error.log | claude -p "このエラーログの原因を分析して"
```

## 主要フラグ一覧

### モデル選択

```bash
claude -p "コードレビューして" --model claude-opus-4-6
```

`--model` で使用するモデルを指定できます。デフォルトは Claude Sonnet ですが、複雑なタスクでは Opus を指定すると精度が上がります。

### ディレクトリ追加

```bash
claude --add-dir ../shared-lib
```

`--add-dir` で作業ディレクトリ以外のフォルダも Claude Code のコンテキストに追加できます。モノレポや共有ライブラリの参照に便利です。

### ツール制限

```bash
claude -p "テストを実行して" --allowedTools "Bash,Read"
```

`--allowedTools` で Claude Code が使用できるツールを制限できます。CI/CD パイプラインでの安全な実行に有用です。

### システムプロンプト拡張

```bash
claude -p "リファクタリングして" --append-system-prompt "コメントは必ず日本語で書くこと"
```

`--append-system-prompt` でシステムプロンプトに独自のルールを追加できます。コーディング規約の強制などに活用できます。

## 認証コマンド

```bash
# ログイン
claude auth login

# ログアウト
claude auth logout

# 認証状態の確認
claude auth status
```

<Callout type="info">
初回起動時にブラウザ経由で自動的に認証フローが始まります。API キーを使う場合は環境変数 `ANTHROPIC_API_KEY` を設定してください。
</Callout>

## 設定コマンド

```bash
# 設定の表示
claude config list

# 設定の変更
claude config set theme dark

# 設定のリセット
claude config reset
```

## MCP（Model Context Protocol）管理

```bash
# MCP サーバーの追加
claude mcp add my-server -- npx -y @example/mcp-server

# 登録済みサーバーの一覧
claude mcp list

# サーバーの削除
claude mcp remove my-server
```

MCP サーバーを通じて、Slack、GitHub、データベースなどの外部サービスと Claude Code を連携できます。

## スラッシュコマンド（対話モード内）

対話モード中に使えるスラッシュコマンドです：

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示 |
| `/clear` | 会話履歴をクリア |
| `/compact` | 会話を要約してコンテキストを圧縮 |
| `/cost` | トークン使用量を表示 |
| `/doctor` | 設定の問題を診断 |
| `/init` | CLAUDE.md プロジェクト設定を作成 |
| `/review` | コードレビューを開始 |
| `/terminal-setup` | ターミナルのキーバインドを設定 |

<Callout type="tip">
`/compact` は長時間の作業でコンテキストウィンドウが一杯になったときに特に有用です。会話内容を要約して圧縮するため、作業を中断せずに続行できます。
</Callout>

## 実践的なユースケース

### CI/CD での自動コードレビュー

```bash
claude -p "このdiffをレビューして問題があれば指摘して" \
  --allowedTools "Read" \
  --model claude-opus-4-6 < pr-diff.txt
```

### 一括ファイル処理

```bash
find src -name "*.ts" | xargs -I {} claude -p "このファイルのTypeScriptエラーを修正して: {}"
```

### セッションの再開

```bash
claude -r
```

`-r`（`--resume`）で直前のセッションを再開できます。作業の途中で中断した場合に便利です。
