## Claude Code Voice Mode とは

2026年3月3日、Anthropic は Claude Code にネイティブの **Voice Mode（音声モード）** を正式にロールアウトしました。これにより、開発者はターミナル上で音声を使って Claude Code に指示を出せるようになります。

従来のテキスト入力に加えて、音声で「この認証ミドルウェアをリファクタリングして」「失敗しているテストを説明して」といった自然言語の指示を伝えることで、タイピングを最小限に抑えながらコーディング作業を進められます。

<Callout type="info">
Voice Mode は2026年3月時点で段階的にロールアウト中です。初期段階ではユーザーの約 5% に提供されており、今後数週間で対象が拡大される予定です。Pro、Max、Team、Enterprise プランのユーザーが対象で、追加料金はかかりません。
</Callout>

## Voice Mode を有効にする方法

### /voice コマンド

Claude Code のセッション中に、以下のスラッシュコマンドを入力するだけで音声モードが有効になります。

```bash
/voice
```

有効化されると、ターミナルのプロンプトに音声入力が利用可能であることを示すインジケーターが表示されます。無効にするには、再度 `/voice` と入力してトグルします。

### Push-to-Talk（プッシュ・トゥ・トーク）方式

Voice Mode は **Push-to-Talk** 方式を採用しています。常時リスニングモードではないため、バックグラウンドノイズを拾ってしまう心配がありません。

操作手順は以下の通りです。

1. **スペースバーを長押し** して録音を開始する
2. Claude に伝えたい内容を **自然に話す**
3. **スペースバーを離す** と、音声がテキストに変換（STT）されてプロンプトに挿入される

```
[スペースバー長押し] → 話す → [スペースバーを離す] → テキストが入力欄に挿入
```

<Callout type="tip">
音声入力はカーソル位置にテキストとして挿入されます。つまり、タイプ入力と音声入力を1つのプロンプト内で自由に組み合わせることができます。たとえば、コマンドの前半をタイプし、後半を音声で補完するといった使い方が可能です。
</Callout>

### アクティベーションキーの変更

デフォルトのアクティベーションキーはスペースバーですが、`keybindings.json` でカスタマイズできます。`meta+k` のような修飾キー＋文字の組み合わせにすると、通常のタイピングとの干渉をゼロにできます。

```json
{
  "voice.activationKey": "meta+k"
}
```

## 音声入力のみ（テキスト出力）

現時点での Voice Mode は **入力のみ音声対応** です。つまり、あなたが音声で指示を出し、Claude はテキストで応答します。音声による応答（TTS）機能は現在のところ搭載されていません。

これは意図的な設計判断で、開発者がターミナルでコードを読む必要があるため、テキスト出力の方が実用的だという考えに基づいています。

## 対応言語

2026年3月のアップデートで、Voice Mode の STT（音声認識）は合計 **20言語** に対応しています。

当初の10言語に加えて、以下の10言語が新たにサポートされました。

- ロシア語
- ポーランド語
- トルコ語
- オランダ語
- ウクライナ語
- ギリシャ語
- チェコ語
- デンマーク語
- スウェーデン語
- ノルウェー語

<Callout type="warning">
日本語のサポート状況については、公式には明確な言及がありません。ただし、サードパーティの STT ツール（Aqua Voice、VoiceMode MCP など）を組み合わせることで、日本語の音声入力を実現できます。詳しくは後述の「サードパーティ STT ツール」セクションを参照してください。
</Callout>

## Voice Mode が開発ワークフローを変える理由

### 1. フローステートの維持

長いプロンプトをタイピングすると、思考の流れが途切れがちです。音声入力を使えば、頭に浮かんだアイデアをそのまま言葉にして Claude に伝えられるため、「バイブコーディング」モードを維持できます。

### 2. 複雑な指示の伝達

「このコンポーネントのステート管理を useReducer に置き換えて、アクションの型定義も追加して、テストも書いて」のような複雑な指示は、タイピングよりも話す方が速く正確に伝わります。

### 3. マルチタスク対応

画面を見ながら別のコードをレビューしつつ、音声で Claude に修正指示を出す——そんな並行作業が可能になります。

### 4. アクセシビリティの向上

手や腕に負担を抱える開発者にとって、音声入力は重要なアクセシビリティツールになります。RSI（反復性ストレス障害）のリスク軽減にも貢献します。

## 実践的な使い方

### コードレビュー

```
[スペースバー長押し]
「この PR の diff を見て、セキュリティ上の問題がないかチェックしてください。
特に SQL インジェクションと XSS の脆弱性に注目してください」
[スペースバーを離す]
```

### デバッグ

```
[スペースバー長押し]
「このテストが失敗している原因を調べてください。
エラーメッセージを読んで、根本原因を特定して修正案を出してください」
[スペースバーを離す]
```

### リファクタリング

```
[スペースバー長押し]
「この auth ミドルウェアをリファクタリングしてください。
Express のミドルウェアパターンに沿って、エラーハンドリングも追加してください」
[スペースバーを離す]
```

### プロジェクト構成の相談

```
[スペースバー長押し]
「このプロジェクトのディレクトリ構成を確認して、
Next.js の App Router のベストプラクティスに沿った改善案を提案してください」
[スペースバーを離す]
```

## サードパーティ STT ツールとの連携

Claude Code のネイティブ Voice Mode に加えて、サードパーティの音声入力ツールを使う方法もあります。特に日本語で音声入力したい場合は、これらのツールが有力な選択肢です。

### Aqua Voice

[Aqua Voice](https://withaqua.com/) は画面のコンテキストを読み取って認識精度を高める AI 音声入力ツールです。プログラミング用語の認識精度が非常に高く、Claude Code のターミナルでそのまま使えます。

```bash
# Aqua Voice は OS レベルの音声入力として動作
# カーソルがターミナルにあれば、そのまま Claude Code に入力される
```

詳しくは [Aqua Voice × Claude 活用ガイド](/ja/cowork/aqua-voice-guide) を参照してください。

### VoiceMode MCP サーバー

[VoiceMode MCP](https://getvoicemode.com/) は、MCP（Model Context Protocol）サーバーとして Claude Code に統合できる音声ツールです。OpenAI の Whisper を STT に、OpenAI TTS を音声出力に使用します。

```bash
# VoiceMode MCP のインストール
npm install -g voicemode-mcp

# Claude Code の MCP 設定に追加
claude mcp add voicemode
```

MCP サーバーとして動作するため、Claude Code が音声入力だけでなく **音声出力** も行えるようになるのが大きな特徴です。

### Claude STT

[Claude STT](https://github.com/jarrodwatts/claude-stt) は、Claude Code にストリーミング音声入力を追加するオープンソースツールです。

```bash
# Claude STT の使い方
# Ctrl+Shift+Space で録音開始
# 再度 Ctrl+Shift+Space で録音停止＆テキスト変換
```

## ネイティブ Voice Mode vs サードパーティツール比較

| 項目 | ネイティブ Voice Mode | Aqua Voice | VoiceMode MCP | Claude STT |
|------|---------------------|------------|---------------|------------|
| セットアップ | `/voice` のみ | アプリインストール | MCP 設定 | npm インストール |
| アクティベーション | スペースバー | OS のホットキー | MCP 経由 | Ctrl+Shift+Space |
| 日本語対応 | 未確認 | 対応 | Whisper 経由で対応 | Whisper 経由で対応 |
| 音声出力 | なし | なし | あり（TTS） | なし |
| 追加コスト | なし | 月額プラン | OpenAI API 料金 | OpenAI API 料金 |
| オフライン動作 | 不明 | 不可 | 不可 | ローカル Whisper で可能 |

## 利用条件と制約

### 対象プラン

Voice Mode は以下のプランで利用可能です（追加料金なし）。

- **Pro プラン**
- **Max プラン**
- **Team プラン**
- **Enterprise プラン**

Free プランのユーザーは現時点では利用できません。

### 現在の制約

- **段階的ロールアウト中**: 2026年3月時点で約 5% のユーザーに提供。今後数週間で拡大予定
- **ターミナルのみ**: VS Code 拡張機能やデスクトップアプリでの対応は未確認
- **入力のみ**: 音声での応答（TTS）は非対応
- **日本語 STT**: ネイティブ対応は未確認（サードパーティツールで代替可能）

## トラブルシューティング

### Voice Mode が表示されない

ロールアウトがまだ到達していない可能性があります。以下を確認してください。

1. Claude Code を最新バージョンに更新する
2. 有料プラン（Pro/Max/Team/Enterprise）であることを確認する
3. 数日待ってから再度確認する

```bash
# Claude Code を最新版に更新
claude update
```

### 音声が正しく認識されない

- 静かな環境で使用する
- マイクが正しく設定されているか確認する
- スペースバーを話し始める前に押し、話し終わってから離す

### スペースバーの干渉

通常のタイピング中にスペースバーが Voice Mode をトリガーしてしまう場合は、アクティベーションキーを変更します。

```json
{
  "voice.activationKey": "meta+shift+v"
}
```

## 今後の展望

Claude Code の Voice Mode はまだ初期段階ですが、今後以下のような進化が期待されます。

- **対応言語の拡大**: 日本語を含むアジア言語のネイティブサポート
- **音声出力（TTS）**: Claude の応答を音声で読み上げる機能
- **IDE 統合**: VS Code や JetBrains IDE での Voice Mode サポート
- **コンテキスト認識の強化**: コードの文脈を理解した、より正確な音声認識
- **ハンズフリーモード**: Push-to-Talk に加えて、常時リスニングモードの追加

## よくある質問（FAQ）

### Q: Voice Mode は無料プランで使えますか？

A: いいえ、現時点では Pro、Max、Team、Enterprise の有料プランのユーザーのみが対象です。追加料金はかかりません。

### Q: 日本語の音声入力は可能ですか？

A: ネイティブの Voice Mode での日本語サポートは公式に確認されていませんが、Aqua Voice や VoiceMode MCP などのサードパーティツールを使えば日本語の音声入力が可能です。

### Q: Voice Mode を有効にしたまま通常のタイピングもできますか？

A: はい。Voice Mode はスペースバー長押し時のみ音声入力として動作するため、通常のタイピングと併用できます。ただし、スペースバーの干渉が気になる場合は、アクティベーションキーを修飾キー付きに変更することをおすすめします。

### Q: Voice Mode はオフラインで使えますか？

A: Claude Code 自体がクラウドベースのサービスのため、インターネット接続が必要です。音声認識処理もサーバーサイドで行われると推測されます。

### Q: Push-to-Talk 以外の入力方式はありますか？

A: 現時点では Push-to-Talk のみです。常時リスニングモード（ハンズフリーモード）は提供されていません。バックグラウンドノイズの影響を排除し、意図的な音声入力のみを受け付けるための設計です。
