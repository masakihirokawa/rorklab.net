## はじめに

ソフトウェア開発の複雑さが増す中、単一のAIエージェントだけでは時間のかかるタスクが多くあります。Claude Code では、複数のサブエージェントを協調させることで、大規模なコード変更、並列レビュー、バッチ処理を効率的に実行できるエージェントチーム機能を提供しています。本記事では、サブエージェントの基礎から実践的なチーム構成まで、段階的に解説します。

## サブエージェントの基本概念

エージェントチームの中心は、**サブエージェント** という独立したAI単位です。メインのエージェントが `Task` ツールを使用することで、複数のサブエージェントを並列に起動し、各エージェントが異なるタスクを同時に処理します。メインエージェントはこれらのサブエージェントの結果を集約し、全体の調整役として機能します。

この仕組みにより、例えばリポジトリ全体のコードリファクタリングを複数の領域に分割し、各エージェントが並列で作業することが可能になります。結果として、単一エージェントで順序実行する場合よりも大幅に処理時間を削減できます。

## エージェントタイプの多様性

Claude Code で利用できるエージェントには、用途に応じた複数の種類があります。

**Explore エージェント** はコードベースの読み取り専用の探索に特化しており、ファイル構造やコメント分析などの軽量な検査に適しています。**Plan エージェント** は開発計画の策定や設計ドキュメントの作成に向いており、実装を伴わないタスクに活用できます。

**General-purpose エージェント** は最も汎用的で、コード読み書き、分析、ドキュメント作成など幅広いタスクに対応します。開発タスクの大部分はこのタイプで実行可能です。**Bash エージェント** はコマンドライン操作に特化しており、ビルド、テスト実行、デプロイなどのシェル操作が必要な場合に用います。

適切なエージェントタイプを選択することで、リソース効率を高め、不要なツールアクセスを制限できます。

## カスタムサブエージェントの定義

プロジェクト固有のニーズに応じて、カスタムサブエージェントを定義することができます。エージェントは `.claude/agents/` ディレクトリ内にマークダウンファイルとして配置されます。

```markdown
---
name: "TypeScript リファクター"
description: "TypeScript コードの型安全性を改善するエージェント"
tools: ["bash", "read", "write", "grep"]
model: "claude-opus-4-6"
parameters:
  max_context_size: 100000
---

このエージェントは TypeScript プロジェクトの型注釈を強化し、
any 型の使用を最小化するタスクを担当します。
```

frontmatter には `name`（エージェント名）、`description`（役割説明）、`tools`（アクセス権限）、`model`（使用モデル）などが記述されます。これにより、各エージェントの責務を明確に分離でき、セキュリティと効率性を両立できます。

## 動的エージェント定義とフラグ

実行時に `--agents` フラグを使用することで、ハードコードされたエージェント定義なしに動的にエージェントを指定できます。

```bash
claude-code --agents "code-reviewer,test-writer,doc-generator" \
  "このリポジトリの新機能をレビュー、テスト作成、ドキュメント更新してください"
```

このアプローチにより、プロジェクトの段階や要件に応じて、柔軟にエージェント構成を変更できます。

## /batch スキルによる大規模変更

コードベース全体に対する大規模な変更（例えば、ライブラリのメジャーアップグレード）を効率的に実行するには、`/batch` スキルが有効です。このスキルは自動的に複数の Git worktree を並列に作成し、各 worktree でエージェントを起動させます。

```
/batch "すべての Python ファイルで print() の形式を Python 3 に統一"
```

メインエージェントは各 worktree でのタスク実行を監視し、完了後に変更を統合します。これにより、数百から数千ファイルの一括変更も数分で完了させられます。

## /simplify スキルと並列レビュー

コードの複雑性を軽減する場合、`/simplify` スキルは 3つの並列レビューエージェントを自動起動します。各エージェントが異なる視点からコード複雑性を分析し、改善提案を提供します。

```
/simplify "lib/payment-processing.ts のシムプリフィケーション提案"
```

複数の視点から同時に分析することで、単一エージェントの見落としを補完できます。

## エージェントチームの実行モード設定

チーム実行の動作方式は、`--teammate-mode` フラグで制御できます。**auto** モードは環境に応じて最適な実行方式を自動選択し、**in-process** モードはメインプロセス内でサブエージェントを実行（軽量で高速）、**tmux** モードはターミナルマルチプレクサを使用して独立したセッションで実行（リソース隔離）します。

```bash
claude-code --teammate-mode tmux \
  "フロントエンドロジック、バックエンドAPI、テストを並列開発"
```

tmux モードは複雑なタスクや長時間実行において、安定性と可視性が高いです。

## 実践例：並列リファクタリング

実際のプロジェクトで、複雑なコンポーネントを 3つのエージェントが並列で改善する場合を考えます。

```
claude-code --agents "type-checker,performance-optimizer,security-auditor" \
  "
  以下の分担でアプリケーション全体を改善してください：
  1. type-checker: すべての TypeScript ファイルで型安全性を向上
  2. performance-optimizer: 重い処理の最適化とメモリリーク調査
  3. security-auditor: セキュリティ脆弱性の検出と修正提案
  "
```

メインエージェントはこれら 3つのサブエージェントを同時実行し、各々の結果をPRドラフトに統合します。個別のエージェントが完了するまで待つのではなく、全エージェントの並列実行により、全体の処理時間は大幅に短縮されます。

## PRレビューとテスト作成の同時実行

新しいプルリクエストの検証フローを効率化する場合、PRレビューとテストスイート作成を同時実行できます。

```
claude-code --agents "code-reviewer,test-generator" \
  "
  以下のPRを検査してください：
  - code-reviewer: コード品質とベストプラクティスをチェック
  - test-generator: 新機能に対する包括的なテストケースを作成
  "
```

従来のシーケンシャルなレビュー → テスト作成フローと異なり、両タスクが並列で進行するため、開発サイクルが高速化します。

## Agent SDK による統合と拡張

Claude Code のエージェント機能は、**Agent SDK** を通じてプログラマティックに拡張できます。SDK により、カスタムエージェントの動的構築、エージェント間の通信プロトコル定義、独自のツール実装が可能になります。

```python
from anthropic_agent_sdk import Agent, SubAgent

main_agent = Agent(
    name="Project Manager",
    tools=["task", "grep", "read"]
)

main_agent.create_subagent(
    name="Code Reviewer",
    description="Pull request 検査と品質チェック",
    model="claude-opus-4-6"
)

results = main_agent.execute(
    "新しい機能のレビュー完了"
)
```

APIレベルでの統合により、CI/CDパイプラインや自動開発ワークフローにエージェントチームを組み込むことで、完全自動化された開発プロセスを実現できます。

## チーム構築のベストプラクティス

効果的なエージェントチームを構築するには、いくつかの原則があります。各エージェントは明確な責務を持つべきであり、重複したツールアクセスを避けることでセキュリティとパフォーマンスを向上させます。タスクの粒度は、エージェントが独立して完了できる程度が最適です。

また、エージェント間の依存関係が複雑になり過ぎないよう設計することが重要です。メインエージェントが調整役となり、各サブエージェントはできるだけ自己完結した形で動作させることで、スケーラビリティが向上します。

## まとめ

Claude Code のエージェントチーム機能は、複数のAIを協調させることで、開発のあらゆる段階を並列化・自動化するツールです。サブエージェントの適切な配置、カスタム定義、実行モードの選択により、プロジェクトの規模や複雑さに応じた柔軟な構成が可能です。バッチ処理から並列レビュー、APIレベルの統合まで、エージェントチームの活用により、開発生産性を大幅に向上させることができます。
