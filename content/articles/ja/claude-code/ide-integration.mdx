# Claude Code IDE 統合ガイド — VS Code・JetBrains

Claude Code により、お気に入りの IDE 内で直接 AI を活用した支援を受けることができます。このガイドでは、VS Code と JetBrains IDE での Claude Code の統合設定と利用方法を詳しく解説します。

## 概要

Claude Code は、最新の開発環境とシームレスに統合され、IDE を離れることなく Claude の高度な推論、コード分析、生成機能にアクセスできます。VS Code を使用している場合でも JetBrains IDE を使用している場合でも、直接エディタ内から Claude の強力な機能を活用できます。

### 主な利点

- **コンテキスト認識**: Claude がコード、プロジェクト構造、ファイルをコンテキストとして理解
- **複数ファイル参照**: 複数ファイル、フォルダ、特定行範囲を会話内で参照可能
- **権限管理**: Claude がアクセスおよび変更できる内容を細かく制御
- **キーボード駆動**: 開発効率を高めるショートカット
- **MCP 統合**: Model Context Protocol サーバーで機能拡張
- **マルチセッション対応**: 複数の独立した会話を同時管理

## VS Code 拡張機能のセットアップ

### インストール要件

Claude Code の VS Code 拡張機能は VS Code Marketplace で利用でき、Anthropic により公式サポートされています。

**システム要件:**
- VS Code 1.98.0 以上
- Claude アカウントと API アクセス
- インターネット接続

**インストール手順:**

1. VS Code を開く
2. 拡張機能マーケットプレイスへ移動（Mac: Cmd+Shift+X、Windows/Linux: Ctrl+Shift+X）
3. 「Claude Code」を検索
4. Anthropic 公式の拡張機能をクリック（ID: `vscode:extension/anthropic.claude-code`）
5. 認証プロンプトに従って Claude アカウントに接続
6. 拡張機能が初期化され、Claude パネルが表示されます

### Claude Code へのアクセス

インストール後、複数の方法で Claude Code にアクセスできます：

**アクティビティバーのスパークアイコン**
左側のアクティビティバーに Claude Code アイコンが表示されます。クリックして Claude パネルを開き、現在の会話と会話履歴を確認できます。

**コマンドパレット**
- `Cmd+Shift+P`（macOS）または `Ctrl+Shift+P`（Windows/Linux）で呼び出し
- 「Claude Code」で検索して利用可能なコマンドを表示
- 「新規会話」「Claude にフォーカス」「ファイル参照」などが含まれます

**ステータスバー**
ウィンドウ下部のステータスバーに Claude Code ステータスが表示されます。クリックして機能にアクセスできます。

**コンテキストメニュー**
ファイル、フォルダ、選択コードを右クリックして、文脈に応じた参照オプションを素早く利用できます。

### 設定オプション

Claude Code 拡張機能は、ユーザー体験をカスタマイズできる複数の設定を提供します。VS Code の設定（Mac: Cmd+,）から設定にアクセスしてください。

| 設定 | 型 | デフォルト | 説明 |
|-----|-----|-----------|------|
| `claude-code.selectedModel` | 文字列 | `claude-opus-4-6` | 会話に使用する Claude モデル（例: claude-opus-4-6, claude-sonnet-4-20250514） |
| `claude-code.useTerminal` | ブール | `true` | Claude が統合ターミナルでコマンドを実行できるようにする |
| `claude-code.initialPermissionMode` | 列挙型 | `Normal` | デフォルト権限モード: Normal, Plan, または Auto-Accept |
| `claude-code.preferredLocation` | 列挙型 | `Side Panel` | Claude パネルの表示位置: Side Panel, Bottom Panel, または Fullscreen |
| `claude-code.autosave` | ブール | `true` | Claude の変更を自動保存 |
| `claude-code.respectGitIgnore` | ブール | `true` | ファイル参照時に .gitignore ルールを適用 |

**設定例（settings.json）:**

```json
{
  "claude-code.selectedModel": "claude-opus-4-6",
  "claude-code.useTerminal": true,
  "claude-code.initialPermissionMode": "Plan",
  "claude-code.preferredLocation": "Side Panel",
  "claude-code.autosave": true,
  "claude-code.respectGitIgnore": true
}
```

> **Pro ヒント**: 変更を適用する前にプレビューさせたい場合、`initialPermissionMode` を「Plan」に設定してください。学習時や重要なコードの作業に便利です。

## Claude Code でのファイル参照

Claude Code の最強機能の一つが、会話内にファイル、フォルダ、コードベースの特定セクションを直接参照できることです。

### ファイル参照の構文

Claude Code は `@` シンボルを使用してファイルとフォルダを参照します。これはコラボレーションツールのメンション機能に似ています。

**単一ファイルの参照**
```
@app.ts
@src/utils/helper.js
@package.json
```

Claude はファイル全体の内容を会話コンテキストに含めます。

**フォルダ参照**
```
@src/components/
@utils/
@tests/
```

フォルダを参照する場合、そのディレクトリ内のすべてのファイルが含まれます（有効な場合は .gitignore ルールを適用）。

**特定行範囲**
```
@app.ts#5-10
@src/api/client.ts#1-25
@middleware.js#45-60
```

ファイル内の特定行を参照して、Claude の注意を特定コードセクションに集中させます。

**PDF ドキュメントのページ**
```
@document.pdf#1-10
@api-spec.pdf#15-20
@architecture.pdf
```

Claude は PDF ドキュメントと特定ページ範囲を分析できます。API 仕様やアーキテクチャ図の参照に便利です。

**ファジー検索**
```
@auth
@component
@database
```

フルパスを入力せずにファジーマッチングでファイルをすばやく検索。Claude がマッチするファイルを候補表示します。

### ファイル参照ワークフロー

1. Claude Code 入力フィールドで `@` を入力開始
2. Claude がマッチするファイルとフォルダを表示
3. 目的のファイルまたはフォルダを選択
4. Claude が参照を挿入しハイライト表示
5. メッセージ送信—Claude が参照ファイルをコンテキストに含めます

このワークフローにより、Claude が正確で、コードを認識した支援を提供するための必要なコンテキストを持つことが保証されます。

## 権限モード

Claude Code は権限システムを実装し、Claude がファイルと環境で何をできるかを制御できます。

### 権限モードの概要

| モード | 説明 | 用途 |
|--------|------|------|
| **Normal** | Claude が提案した変更を表示し、適用前にあなたの承認を待つ | デフォルトモード、重要な変更に推奨 |
| **Plan** | Claude が詳細な計画を提示し、アクション実行前に確認を求める | 学習モード、Claude が何をするか理解したい場合 |
| **Auto-Accept** | Claude が承認を求めずに変更を自動適用 | 信頼できるシナリオ、生産性重視のワークフロー |

### 権限モードの設定

**グローバル設定:**
VS Code 設定でデフォルト権限モードを設定：
```
claude-code.initialPermissionMode: "Normal"
```

**会話ごとのオーバーライド:**
Claude パネルのドロップダウンメニューから、個別の会話の権限モードを変更できます。

> **セキュリティ注**: 「Auto-Accept」は慎重に使用してください。このモードにより Claude は承認なしでファイルを変更できます。分離されたプロジェクト内または Claude の最近の変更を検証した後にのみ有効にしてください。

## キーボードショートカット

効率的なキーボードナビゲーションは開発フローの維持に不可欠です。Claude Code は複数の最適化されたショートカットを提供します。

### 基本ショートカット

| ショートカット | macOS | Windows/Linux | アクション |
|-------------|-------|---------------|----------|
| Claude にフォーカス | Cmd+Esc | Ctrl+Esc | エディタから Claude パネルにフォーカスを移動 |
| 新規会話 | Cmd+N | Ctrl+N | 新しい Claude 会話を開始 |
| @ メンション | Option+K | Alt+K | ファイル/フォルダセレクタを開く |
| ファイル検索 | Cmd+P | Ctrl+P | クイックファイル検索（その後 @ で参照） |
| メッセージ送信 | Enter | Enter | Claude 入力フィールドでメッセージを送信 |
| コンテキストクリア | Cmd+Shift+L | Ctrl+Shift+L | 会話履歴とコンテキストをクリア |

### ナビゲーションのコツ

- エディタのどこからでも `Cmd+Esc`（または `Ctrl+Esc`）を押して Claude パネルに即座にフォーカス
- `@` を入力せずに `Option+K`（または `Alt+K`）でファイルセレクタを呼び出し
- `Cmd+P` と `@` を組み合わせて、単一操作でファイルを検索・参照

## マルチセッション対応

Claude Code は複数の独立した会話を同時にサポートし、各会話は Claude パネル内の別タブに表示されます。

### 新規セッションの作成

1. Claude パネル上部の「+」アイコンをクリック
2. または `Cmd+N`（または `Ctrl+N`）を押す
3. 新しい会話タブがフレッシュコンテキストで開きます
4. 各セッションは独立した会話履歴とファイル参照を保持

### セッション管理

- **セッション間の切り替え**: 会話タブをクリックしてセッション間を切り替え
- **セッションのクローズ**: タブの X をクリックして会話を閉じる
- **セッション永続化**: 会話履歴は自動保存され、VS Code の再起動後も保持されます
- **独立したコンテキスト**: 各セッションは個別のファイル参照と会話状態を保持

このマルチセッション機能により、コンテキストを失わずに複数のタスクを同時処理できます。

## MCP サーバー統合

`/mcp` コマンドはモデルコンテキストプロトコル（MCP）サーバーとの統合を有効にし、Claude の機能をファイル分析以上に拡張します。

### MCP サーバーの使用

```
/mcp add <サーバー名> <サーバー設定>
/mcp list
/mcp remove <サーバー名>
```

**MCP 統合例:**
```
/mcp add local-database "postgresql://localhost:5432/mydb"
/mcp list
```

設定後、Claude は会話内で MCP サーバーツールを直接使用でき、外部システム、データベース、専門的な API へのアクセスが可能になります。

> **注**: MCP サーバーセットアップは VS Code 外での設定が必要です。詳細は特定の MCP サーバードキュメントを参照してください。

## プラグイン管理

`/plugins` コマンドは Claude のプラグインエコシステムへのアクセスを提供し、特定のワークフロー向けの機能強化を可能にします。

### プラグインコマンド

```
/plugins list
/plugins enable <プラグイン名>
/plugins disable <プラグイン名>
/plugins search <検索語>
```

**利用可能なプラグイン:**
- コード分析と最適化
- ドキュメント生成
- テスト生成と検証
- セキュリティ分析
- パフォーマンスプロファイリング

プラグインは Claude Code の特定の開発タスク向け機能を拡張します。検索コマンドを使用して利用可能なプラグインを参照してください。

## 高度な機能

### Chrome 統合

Claude Code バージョン 1.0.36 以降は Claude Chrome 拡張機能との統合を含み、Web ページと Web アプリケーション分析をコンテキスト内で有効にします。

両方の拡張機能がインストールされている場合：
- Claude が会話内で開いている Web ページを参照可能
- ブラウザでレンダリングされたフロントエンドコードを分析
- ドキュメント Web サイトのコンテキストを理解
- ビジュアルフィードバック付きで Web アプリケーションをデバッグ

### 会話履歴管理

VS Code は会話履歴をローカルに自動保存します。履歴を管理するには：

1. Claude パネルを開く
2. 履歴アイコン（時計シンボル）をクリック
3. 以前の会話を参照
4. 会話をクリックして再開
5. 必要に応じて会話をアーカイブまたは削除

### パフォーマンス最適化

大規模プロジェクトの場合、Claude Code のパフォーマンスを最適化してください：

- **コンテキストサイズ制限**: 必要なファイルとフォルダのみを参照
- **行範囲使用**: 正確な行番号を指定してトークン使用量を削減
- **.gitignore 有効化**: Claude が .gitignore を尊重するようにしかなくてもいいファイルを除外
- **セッションアーカイブ**: アクティブでないセッションを閉じてメモリ使用量を削減

## JetBrains IDE 統合

Claude Code は IntelliJ IDEA、PyCharm、WebStorm、その他 JetBrains 製品を含む JetBrains IDE とシームレスに統合されます。

### インストール

1. JetBrains IDE を開く
2. Preferences → Plugins へ移動（macOS: IDE → Preferences → Plugins）
3. マーケットプレイスで「Claude Code」を検索
4. 「Install」をクリック
5. プロンプトが表示されたら IDE を再起動
6. Claude アカウントで認証

### JetBrains での Claude へのアクセス

- **ツールウィンドウ**: Claude は IDE の右側にツールウィンドウとして表示
- **コンテキストメニュー**: ファイルまたはコードセクションを右クリックして参照
- **検索**: Ctrl+Shift+F（Windows/Linux）または Cmd+Shift+F（macOS）でファイルを検索・参照

### JetBrains 固有機能

**実行構成**
Claude は実行構成を分析・理解でき、IDE 設定固有の問題デバッグに役立ちます。

**組み込みツール**
Claude が統合した JetBrains の組み込みツールとアナライザーを参照して、包括的なコード理解が可能です。

**Git 統合**
Claude は JetBrains の Git インターフェースと連動し、ブランチ構造とコミット履歴を理解します。

## トラブルシューティング

### よくある問題

**拡張機能が表示されない**
- VS Code バージョンが 1.98.0 以上か確認
- VS Code を完全に再起動
- マーケットプレイスでインストール状態を確認
- アクティビティバーで Claude Code アイコンを確認

**認証エラー**
- Claude アカウント認証情報を確認
- インターネット接続を確認
- 拡張機能設定から再度サインアウト・サインイン
- 問題が続く場合は VS Code キャッシュをクリア

**ファイル参照が機能しない**
- ファイルパスが正しいか確認
- ファイルがプロジェクトに存在するか確認
- .gitignore が必要なファイルを除外していないか確認
- ファジーマッチングの代わりにフルパスを試す

**パフォーマンス問題**
- 使用していない会話セッションを閉じる
- 参照ファイル数を制限
- 大きなファイルには行範囲を使用
- 非重要プロジェクトではオートセーブを無効化

### サポートの取得

- 公式 Claude Code ドキュメントを確認
- GitHub Issues で既知の問題を確認
- アカウントダッシュボードから Anthropic サポートに連絡
- 統合問題について IDE 固有のフォーラムを参照

## ベストプラクティス

### Claude Code の効果的な利用

1. **明確なコンテキスト**: プロジェクト全体ではなく、特定ファイルを参照
2. **焦点を絞った質問**: Claude に一度に一つの問題を解くよう依頼
3. **変更レビュー**: Claude の提案した変更を承認前に必ず確認
4. **Plan モード使用**: 新しいアプローチを試すときは「Plan」権限モードを使用
5. **セッション整理**: 異なるタスク用に別々のセッションを作成
6. **ドキュメント参照**: 利用可能な場合は API ドキュメントと仕様を参照

### セキュリティに関する考慮事項

- Claude 会話で API キーを共有しない
- 権限モードを慎重に使用—承認前にすべての変更をレビュー
- .gitignore を尊重し、機密設定ファイルを参照しない
- 共有マシンでは Auto-Accept モードに注意
- 誤った情報公開がないか定期的に会話履歴をレビュー

## まとめ

Claude Code の IDE 統合は、開発ワークフローに強力な AI 機能をもたらします。ファイル参照、権限モード、キーボードショートカット、マルチセッション管理をマスターすることで、生産性を著しく向上させることができます。

VS Code または JetBrains を使用しているかを問わず、Claude Code はお気に入りの開発環境に適応し、AI 支援開発を日常作業の一部として統合します。

Normal 権限モードから開始して Claude の機能を理解し、自信が付くにつれてより自動化されたモードを徐々に採用していってください。キーボードショートカットを活用してフローを維持し、マルチセッション対応を活用して複雑なプロジェクトを効果的に管理してください。
