## フックとは

Claude Code のフックはイベント駆動型スクリプトで、ワークフローの特定のポイントで自動的に実行されます。繰り返しのタスクを自動化し、Claude のツール実行にカスタム ロジックを注入し、Claude Code の組み込み機能を超えて機能を拡張できます。

<Callout type="info">
フックはプロジェクト ルートの `.claude/settings.json` ファイルで設定されます。それらは開発環境でローカルに実行され、カスタム自動化を完全にコントロールできます。
</Callout>

## フックの種類

Claude Code は 3 つのメイン フック タイプをサポートします:

1. **PreToolUse** — Claude がツールを実行する前に実行
2. **PostToolUse** — Claude がツールを実行した後に実行
3. **Notification** — 外部サービスにアラートと通知を送信

## 最初のフックを設定する

### .claude/settings.json を作成する

まず、プロジェクト ルートで設定ファイルを作成します:

```bash
mkdir -p .claude
touch .claude/settings.json
```

### 基本設定

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write", "file:delete"],
        "handler": "scripts/pre-write-validation.js"
      }
    ],
    "postToolUse": [
      {
        "tools": ["git:commit"],
        "handler": "scripts/post-commit-notification.js"
      }
    ]
  }
}
```

## PreToolUse フック

PreToolUse フックは Claude がツールを実行する前に実行され、パラメータを検証し、コンテキストを追加し、危険な操作を防ぐことができます。

### PreToolUse の使用例

- **検証**: ファイル操作の前にパラメータをチェック
- **セキュリティ**: クリティカル ブランチへのコミットを防止
- **ログ**: Claude が何をしようとしているかを記録
- **変更**: ツール パラメータを追加または変更

### 例: ファイル書き込み検証

`scripts/pre-write-validation.js` を作成します:

```javascript
// ファイル書き込み操作の事前検証フック
module.exports = async function preWriteValidation(context) {
  const { tool, params } = context;

  // センシティブ ディレクトリへの書き込みを防止
  const forbiddenPaths = [
    '.env',
    'node_modules',
    '.git',
    'secrets/'
  ];

  const filePath = params.path;

  for (const forbidden of forbiddenPaths) {
    if (filePath.includes(forbidden)) {
      return {
        allowed: false,
        reason: `Cannot write to ${filePath}. This path is protected.`
      };
    }
  }

  // 操作を許可
  return {
    allowed: true,
    modifiedParams: params
  };
};
```

### ファイル書き込み検証の設定

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write"],
        "handler": "scripts/pre-write-validation.js"
      }
    ]
  }
}
```

### 例: Git ブランチ保護

`scripts/pre-git-commit.js` を作成します:

```javascript
// main/master ブランチへのコミットを防止
module.exports = async function preGitCommit(context) {
  const { params } = context;

  // git から現在のブランチを取得
  const { execSync } = require('child_process');
  const currentBranch = execSync('git rev-parse --abbrev-ref HEAD')
    .toString()
    .trim();

  // メイン ブランチを保護
  const protectedBranches = ['main', 'master', 'production'];

  if (protectedBranches.includes(currentBranch)) {
    return {
      allowed: false,
      reason: `Cannot commit to protected branch: ${currentBranch}`
    };
  }

  return {
    allowed: true
  };
};
```

## PostToolUse フック

PostToolUse フックは Claude がツール操作を完了した後に実行され、結果をログ、通知をトリガー、またはクリーンアップ操作を実行できます。

### PostToolUse の使用例

- **ログ**: 成功した操作を記録
- **通知**: 変更についてチーム メンバーに警告
- **クリーンアップ**: 一時ファイルまたはキャッシュを削除
- **検証**: 操作が正しく完了したかどうかを確認

### 例: コミット通知

`scripts/post-commit-notification.js` を作成します:

```javascript
// 成功した git コミット後にチームに通知
module.exports = async function postCommitNotification(context) {
  const { params, result } = context;

  if (result.success) {
    const message = `
      Git コミット完了:
      - コミット: ${params.message}
      - ハッシュ: ${result.commitHash}
      - 作成者: ${result.author}
    `;

    // コンソールにログ
    console.log(message);

    // 通知サービスに送信 (オプション)
    // await notifySlack(message);
  }

  return { processed: true };
};
```

### 例: ビルド自動化

`scripts/post-test-automation.js` を作成します:

```javascript
// テストが成功した後、ビルドを実行
module.exports = async function postTestAutomation(context) {
  const { result, params } = context;

  if (result.testsPassed) {
    console.log('テスト成功! ビルドを実行しています...');

    const { execSync } = require('child_process');

    try {
      execSync('npm run build', { stdio: 'inherit' });
      console.log('ビルドが正常に完了しました');
      return { success: true, action: 'build' };
    } catch (error) {
      console.error('ビルド失敗:', error.message);
      return { success: false, action: 'build', error: error.message };
    }
  }

  return { processed: true };
};
```

### ビルド自動化の設定

```json
{
  "hooks": {
    "postToolUse": [
      {
        "tools": ["test"],
        "handler": "scripts/post-test-automation.js"
      }
    ]
  }
}
```

## 通知フック

通知フックは、特定のイベントが発生したときに Slack、Discord、メールなどの外部サービスにアラートを送信します。

### 例: Slack 通知

`scripts/slack-notifier.js` を作成します:

```javascript
// Slack に通知を送信
const https = require('https');

module.exports = async function slackNotifier(context) {
  const { event, data } = context;
  const webhookUrl = process.env.SLACK_WEBHOOK_URL;

  if (!webhookUrl) {
    console.warn('SLACK_WEBHOOK_URL not set');
    return;
  }

  const message = {
    text: `Claude Code Event: ${event}`,
    blocks: [
      {
        type: 'section',
        text: {
          type: 'mrkdwn',
          text: `*Event*: ${event}\n*Details*: ${JSON.stringify(data, null, 2)}`
        }
      }
    ]
  };

  return new Promise((resolve, reject) => {
    const postData = JSON.stringify(message);

    https.post(new URL(webhookUrl), {
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': postData.length
      }
    }, (res) => {
      resolve({ sent: true, status: res.statusCode });
    }).on('error', reject).end(postData);
  });
};
```

### Slack 通知の設定

```json
{
  "hooks": {
    "notifications": [
      {
        "events": ["commit", "deploy", "error"],
        "handler": "scripts/slack-notifier.js",
        "config": {
          "channel": "#dev-alerts"
        }
      }
    ]
  }
}
```

## フックのベストプラクティス

1. **フックを高速に保つ**: 長時間実行されるフックはワークフローを遅くします
2. **エラーを適切に処理する**: フックは安全に失敗する必要があります
3. **重要なアクションをログする**: フックを監査可能にしてください
4. **徹底的にテストする**: 本番環境に展開する前にフックをテストしてください
5. **フックを文書化する**: 各フックが何をするかを説明するコメントを含めてください
6. **環境変数を使う**: シークレットを `.env` ファイルに保存してください
7. **スコープを制限する**: フックをそれを必要とするツールにのみ適用してください

<Callout type="warning">
フックはあなたの環境でローカルに実行されます。信頼されていないコードを実行したり、完全に理解していない操作を実行するのにフックを使用しないでください。
</Callout>

## フックのデバッグ

### フック実行を確認する

```bash
# フック ログを表示
claude --verbose

# 特定のフックをテスト
node scripts/pre-write-validation.js
```

### 一般的な問題

| 問題 | 解決策 |
|-------|----------|
| フックが実行されない | `.claude/settings.json` の構文エラーを確認 |
| フックがエラーをスロー | `node` でフック スクリプトを独立してテスト |
| パラメータが渡されない | フック ハンドラー シグネチャが想定される形式と一致するかどうかを確認 |
| パフォーマンスの問題 | フック実行時間をプロファイルし、非同期最適化を検討 |

## 高度なフック パターン

### 条件付き実行

```javascript
module.exports = async function conditionalHook(context) {
  const isDevelopment = process.env.NODE_ENV === 'development';

  if (!isDevelopment) {
    return { allowed: true }; // 本番環境ではフックをスキップ
  }

  // 開発環境でのみ検証を実行
  return validateChanges(context);
};
```

### 複数フックのチェーン

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write"],
        "handlers": [
          "scripts/validate-syntax.js",
          "scripts/check-format.js",
          "scripts/verify-permissions.js"
        ]
      }
    ]
  }
}
```
