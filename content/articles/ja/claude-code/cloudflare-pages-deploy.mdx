# 番外編: Next.js サイトを Cloudflare Pages にデプロイする完全ガイド

Cloudflare Pages は、Next.js アプリケーションをデプロイするための強力かつ費用効率的なプラットフォームです。このガイドでは、Claude Code を活用して Next.js サイトを Cloudflare Pages に完全にデプロイするプロセスを解説します。

## Cloudflare Pages を選ぶ理由

Cloudflare Pages は、従来の Vercel と比較して多くの利点を提供します。

**無料ティアの充実度**
- 無制限の静的サイトホスティング
- 月間 50 万のリクエスト（無料プラン）
- 帯域幅無制限
- Cloudflare の CDN 全体での配信

**エッジコンピューティング**
- Cloudflare Workers による serverless functions
- グローバルな 300+ のデータセンター
- 最小レイテンシーでのコンテンツ配信

**コールドスタート対策**
Vercel では Cold Start が発生しやすいですが、Cloudflare Pages では:
- Edge Runtime による瞬時の起動
- Always-on ワーカー（有料プラン）
- リージョン別最適化

**セキュリティと DDoS 保護**
- 標準でエンタープライズグレードの DDoS 保護
- Web Application Firewall (WAF)
- 追加設定不要

## 前提条件

デプロイを始める前に、以下を準備してください。

1. **Node.js** (バージョン 18.0 以上)
2. **Cloudflare アカウント** (無料でも可)
3. **Wrangler CLI** (Cloudflare の公式ツール)
4. **GitHub リポジトリ** (デプロイ用)
5. **カスタムドメイン** (オプション、本例では claudelab.net)

Wrangler をインストールするには:

```bash
npm install -g wrangler
```

## Next.js プロジェクトのセットアップ

既存の Next.js プロジェクトがある場合は、OpenNext アダプターを追加します。

### OpenNext アダプターのインストール

```bash
npm install @opennextjs/cloudflare
```

このアダプターは、Next.js の App Router、API Routes、Image Optimization など、ほぼすべての機能を Cloudflare Pages 互換に変換します。

### 初期化コマンド

```bash
npx opennextjs-cloudflare init
```

このコマンドは以下を自動生成します:
- `open-next.config.ts` — OpenNext の設定ファイル
- `wrangler.jsonc` — Cloudflare の設定ファイル
- `.env.local` 環境変数テンプレート

## 設定ファイルの詳細解説

### open-next.config.ts

```typescript
import type { OpenNextConfig } from '@opennextjs/cloudflare';

const config: OpenNextConfig = {
  default: {
    override: {
      wrapper: '@opennextjs/cloudflare/wrappers/cloudflare-pages',
      converter: '@opennextjs/cloudflare/adapters/cloudflare-pages',
    },
  },
  middleware: {
    external: [],
    override: {
      wrapper: '@opennextjs/cloudflare/wrappers/cloudflare-pages',
      converter: '@opennextjs/cloudflare/adapters/cloudflare-pages',
    },
  },
};

export default config;
```

この設定により、Next.js のすべてのルートと API が Cloudflare Workers で動作する形に変換されます。

### wrangler.jsonc

```jsonc
{
  "name": "claudelab-pages",
  "type": "service-worker",
  "main": ".open-next/server/index.js",
  "site": {
    "bucket": ".open-next/static"
  },
  "triggers": {
    "crons": []
  },
  "env": {
    "production": {
      "name": "claudelab-pages-prod",
      "route": "claudelab.net/*"
    },
    "staging": {
      "name": "claudelab-pages-staging"
    }
  },
  "vars": {
    "SITE_NAME": "Claude Lab"
  }
}
```

<Callout type="info">
`wrangler.jsonc` はコメント付きの JSON です。Cloudflare のリリース時点では、`wrangler` は .jsonc をネイティブサポートしており、コメントも有効です。
</Callout>

## next-intl の設定（多言語対応）

claudelab.net は複数言語に対応しているため、`next-intl` ミドルウェアの設定が重要です。

### i18n ミドルウェア

```typescript
import createMiddleware from 'next-intl/middleware';
import { locales, localePrefix } from './config';

export default createMiddleware({
  locales,
  localePrefix,
  defaultLocale: 'en',
});

export const config = {
  matcher: ['/((?!_next|_vercel|.*\\..*).*)'],
};
```

<Callout type="warning">
Cloudflare Pages は Edge Runtime で実行されるため、`getRequestContext()` や `Deno` の機能は使用できません。必ず Node.js 互換の API のみを使用してください。
</Callout>

### next.config.js で i18n 設定

```javascript
const withNextIntl = require('next-intl/plugin')('./i18n.ts');

/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  experimental: {
    // Cloudflare Pages では serverComponentsExternalPackages が必要な場合がある
    serverComponentsExternalPackages: [],
  },
};

module.exports = withNextIntl(nextConfig);
```

## ビルドプロセス

### ビルドコマンド

```bash
npm run build
```

`package.json` に以下を追加してください:

```json
{
  "scripts": {
    "build": "npx opennextjs-cloudflare build",
    "deploy": "npx wrangler deploy",
    "preview": "npm run build && npx wrangler dev"
  }
}
```

### ビルド出力

成功時、`.open-next` ディレクトリが生成されます:

```
.open-next/
├── server/          # API Routes と SSR
├── static/          # 静的ファイル
├── cache/           # ISR キャッシュ
└── index.js         # エントリーポイント
```

このディレクトリ全体が Cloudflare にデプロイされます。

## ローカル開発とプレビュー

### Wrangler Dev での実行

```bash
npx wrangler dev
```

このコマンドでローカル環境が起動し、`http://localhost:8787` でアプリが実行されます。本番環境に最も近い環境でテストできます。

<Callout type="info">
Wrangler Dev は、Cloudflare Workers の Edge Runtime をエミュレートします。ローカルホストでの実行でも、実際のデプロイと同等の環境が得られます。
</Callout>

```bash
OPENAI_API_KEY=sk-xxx
```

## デプロイ方法

### 方法 1: Wrangler CLI でデプロイ

```bash
npx wrangler deploy
```

このコマンドは:
1. `.open-next` ディレクトリをビルド
2. Cloudflare API に静的ファイルをアップロード
3. Workers スクリプトをデプロイ

完了後、`https://<your-project>.pages.dev` でアクセス可能です。

### 方法 2: GitHub 統合（推奨）

GitHub にプッシュするだけで自動デプロイされます。

1. **Cloudflare ダッシュボードで設定**
   - Pages → 「新規作成」 → GitHub を選択
   - リポジトリを認可
   - ビルド設定:
     ```
     フレームワーク: Next.js
     ビルドコマンド: npm run build
     ビルド出力ディレクトリ: .open-next
     ```

2. **自動デプロイの設定**
   - すべてのプッシュで自動ビルド
   - プレビュー環境の自動生成
   - プロダクション環境（カスタムドメイン）

<Callout type="info">
GitHub 統合により、CI/CD パイプラインが自動的に設定されます。PR ごとにプレビュー環境が生成されるため、チームでのレビュープロセスが効率化されます。
</Callout>

## 環境変数の設定

### Cloudflare ダッシュボードでの設定

1. Pages プロジェクトを開く
2. 「設定」→「環境変数」
3. 各環境（本番/ステージング）ごとに変数を追加

```
STRIPE_SECRET_KEY = sk_live_xxx
DATABASE_URL = postgresql://prod-user:pass@prod-db/prod
OPENAI_API_KEY = sk-xxx
```

### Wrangler CLI での設定

```bash
wrangler secret put STRIPE_SECRET_KEY
# プロンプトで値を入力

wrangler secret put STRIPE_SECRET_KEY --env production
```

### コード内でのアクセス

API Routes や Server Components から環境変数を使用:

```typescript
// app/api/payment/route.ts
export async function POST(request: Request) {
  const stripeKey = process.env.STRIPE_SECRET_KEY;
  const stripe = new Stripe(stripeKey);

  // Stripe API を呼び出し
  return Response.json({ success: true });
}
```

## カスタムドメインの設定

### claudelab.net をバインド

1. **Cloudflare ダッシュボード**
   - Pages プロジェクト → 「カスタムドメイン」
   - claudelab.net を追加

2. **DNS 設定**
   - claudelab.net の DNS ホストで CNAME を設定:
   ```
   CNAME: claudelab.net → <your-project>.pages.dev
   ```

3. **SSL/TLS**
   - Cloudflare は自動的に Let's Encrypt 証明書を発行
   - HTTPS はデフォルトで有効

4. **リダイレクト設定**
   ```
   http://claudelab.net → https://claudelab.net （自動）
   www.claudelab.net → https://claudelab.net （wrangler で設定）
   ```

<Callout type="warning">
DNS の変更には 24 時間程度要する場合があります。ただし、Cloudflare がネームサーバーの場合は数分で反映されます。
</Callout>

## トラブルシューティング

### Edge Runtime で利用不可な Node.js API

Cloudflare Workers は Edge Runtime で実行されるため、一部の Node.js API が使用できません。

**使用不可な機能:**
- `fs` (ファイルシステム)
- `child_process` (子プロセス)
- `crypto` のいくつかのメソッド
- `process.env` の直接読み取り（環境変数は異なる方法でアクセス）

**対処方法:**
```typescript
// ❌ 不可
const fs = require('fs');
const data = fs.readFileSync('file.txt');

// ✅ 可能 (Cloudflare Workers の KV を使用)
import { kv } from '@cloudflare/workers-kv';
const data = await kv.get('file-key');
```

### Image Optimization の制限

`next/image` はデフォルトで動作しませんが、OpenNext が対応しています。

```javascript
// next.config.js
const config = {
  images: {
    remotePatterns: [
      { protocol: 'https', hostname: 'cdn.example.com' },
    ],
    // Cloudflare では external URL のみ推奨
    unoptimized: process.env.NODE_ENV === 'production',
  },
};
```

### ビルドサイズの最適化

OpenNext は複数の関数に分割できます:

```typescript
// open-next.config.ts
export default {
  split: true, // ルートごとに関数を分割
  minify: true,
  sourceMap: false,
};
```

## Claude Code ワークフロー

Claude Code を使用すると、デプロイプロセス全体が自動化されます。

### 1. 初期セットアップ

Claude Code に以下のタスクを実行させます:

```
Next.js プロジェクトに OpenNext と Wrangler を統合してください。
1. @opennextjs/cloudflare をインストール
2. open-next.config.ts を生成
3. wrangler.jsonc を設定
4. package.json のビルドスクリプトを更新
```

### 2. ビルドとローカルテスト

```
1. npm run build で Next.js をビルド
2. npx wrangler dev でローカル環境を起動
3. デプロイ前に全ページが正常に動作することを確認
```

### 3. デプロイ前チェック

Claude Code がデプロイ前に確認する項目:

- [ ] すべての環境変数が設定されているか
- [ ] `next-intl` の i18n ミドルウェアが動作しているか
- [ ] API Routes が Cloudflare Workers で実行可能か
- [ ] 静的アセットが `.open-next/static` に配置されているか

### 4. 本番デプロイ

```bash
npx wrangler deploy --env production
```

デプロイ完了後、Claude Code が以下を自動検証:

1. カスタムドメイン (claudelab.net) でアクセス可能
2. すべてのページが 200 ステータスで応答
3. 環境変数が正しく読み込まれている
4. i18n が複数言語で正常に動作
