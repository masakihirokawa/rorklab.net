## Rork Max の裏側で動く「最強コンビ」

Rork Max は、自然言語のプロンプト一つから iPhone、iPad、Apple Watch、Apple TV、Vision Pro 向けのネイティブ Swift アプリを生成する革命的なプラットフォームです。2026年2月のローンチからわずか3日で **$1.5M ARR（年間経常収益）** を達成し、AI アプリ開発の新たな基準を打ち立てました。

その裏側で動いているのが、Anthropic の **Claude Code** と **Claude Opus 4.6** です。この記事では、この最強コンビがどのようにネイティブアプリを生成しているのか、技術的に深掘りします。

## アーキテクチャの全体像

Rork Max のアプリ生成パイプラインは以下の流れで動きます。

```
ユーザーの自然言語プロンプト
    ↓
Claude Opus 4.6 がプロンプトを解析・アーキテクチャ設計
    ↓
Claude Code がSwift/SwiftUI コードを生成
    ↓
クラウド Mac でコンパイル（Xcode ビルドチェーン）
    ↓
ライブシミュレータでリアルタイムプレビュー
    ↓
1クリックで実機インストール / 2クリックでApp Store提出
```

### なぜ Claude Opus 4.6 なのか

Rork Max が Claude Opus 4.6 を採用している理由は明確です。

- **128K トークンの出力上限**: 大規模なアプリの全コードを一度に生成可能
- **コード品質**: SWE-bench Verified での高スコアが示す実践的なコーディング能力
- **Swift/SwiftUI の深い理解**: Apple のフレームワーク構造を正確に把握し、ベストプラクティスに沿ったコードを出力
- **セキュリティ意識**: Mozilla Firefox の脆弱性22件を発見した実績が示す、セキュリティを意識したコード生成

### Claude Code の役割

Claude Code は単なるコード生成ツールではなく、**エージェント的な開発環境**として機能します。Rork Max 内では以下の役割を担います。

1. **コード生成**: SwiftUI ビュー、データモデル、ビジネスロジックの作成
2. **デバッグ**: コンパイルエラーの自動検出と修正（Fix Now 機能）
3. **最適化**: メモリ使用量やレンダリングパフォーマンスの改善提案
4. **テスト**: ユニットテストの自動生成

## Apple API への完全アクセス

Rork Max × Claude Code の最大の強みは、**Apple のネイティブ API にフルアクセス**できることです。

| API / フレームワーク | 生成例 | 実用シーン |
|---------------------|--------|-----------|
| ARKit + LiDAR | 3D 空間認識アプリ | 家具配置、測量 |
| HealthKit | 健康データ連携 | フィットネストラッカー |
| Core ML + Metal | オンデバイス AI 推論 | 画像認識、物体検出 |
| Dynamic Island | ライブアクティビティ | デリバリー追跡、スポーツスコア |
| App Clips | 軽量アプリ体験 | QR コードからの即時起動 |
| Siri Intents | 音声コマンド対応 | ハンズフリー操作 |
| NFC | 非接触データ読み取り | 決済、情報タグ |

### 実例: Pokémon Go 風の AR アプリ

Rork の公式デモでは、「Pokémon Go のような AR ゲーム」というプロンプトから、ARKit と 3D レンダリングを活用した完全なアプリが生成されました。60fps で動作するネイティブパフォーマンスは、React Native ベースのクロスプラットフォームアプリでは実現困難な品質です。

## 開発ワークフロー — 0からApp Store公開まで

### Phase 1: プロンプト設計（5分）

最も重要なのは最初のプロンプトです。効果的なプロンプトには以下の要素を含めます。

- **アプリの目的**: 何をするアプリか
- **主要機能**: 3〜5個の核となる機能
- **UI の方向性**: ミニマル、リッチ、ダッシュボード型など
- **使用する Apple API**: HealthKit、ARKit など必要な場合

```
「毎日の食事を写真で記録し、AIが栄養バランスを分析する
 iPhoneアプリを作って。HealthKitと連携して体重の推移も
 表示。UIはミニマルで、ダークモード対応。」
```

### Phase 2: 生成とプレビュー（3〜10分）

Claude Opus 4.6 がアーキテクチャを設計し、Claude Code が Swift コードを生成。クラウド Mac 上でコンパイルされ、ブラウザ内のライブシミュレータでリアルタイムにプレビューできます。

### Phase 3: イテレーション

生成されたアプリに対して、自然言語で修正指示を出せます。

- 「ホーム画面にカレンダービューを追加して」
- 「写真撮影のUIをもっと大きくして」
- 「体重グラフに週平均のラインを追加して」

### Phase 4: 実機テストとApp Store 提出

- **1クリック**: Rork Companion Mac App 経由で実機にインストール
- **2クリック**: App Store Connect に直接提出

## パフォーマンス比較

| 指標 | Rork Max (Swift) | クロスプラットフォーム (React Native) |
|------|-----------------|----------------------------------|
| 起動速度 | ネイティブ（最速） | JavaScriptブリッジのオーバーヘッド |
| フレームレート | 安定60fps | 複雑なUIで低下の可能性 |
| メモリ効率 | ARC によるネイティブ管理 | JS ランタイムの追加消費 |
| Apple API | フルアクセス | ネイティブモジュール経由で制限あり |
| アプリサイズ | 最適化済み | JavaScriptバンドルの分だけ大きい |
