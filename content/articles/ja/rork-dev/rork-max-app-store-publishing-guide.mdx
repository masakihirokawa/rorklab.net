Rork Max でアプリを生成したら、次のステップは App Store での公開です。しかし、単にアップロードするだけでは不十分。Apple の審査ガイドラインは厳格で、不適切な設定や説明文があると、容赦なくリジェクトされます。

この記事では、Rork Max 生成アプリを App Store に公開する完全フロー、審査通過を確実にするためのチェックリスト、公開後の App Store Optimization（ASO）戦略まで、すべてを解説します。実装と検証を繰り返した実務的なアドバイスです。

## Rork Max とは — App Store 公開の新しい選択肢

Rork Max は、Rork の上位エディション。通常の Rork よりも高度な機能をサポートしており、特に**App Store への直接パッケージング・公開**が大きな差別化要因です。

### 通常の Rork との違い

| 項目 | 通常 Rork | Rork Max |
|---|---|---|
| **アプリ生成** | ✓ | ✓ |
| **ネイティブUI** | ✓ | ✓ |
| **App Store自動申請** | ✗ | ✓ |
| **証明書・プロビジョニング管理** | 手動 | 自動 |
| **BUID ID生成** | 手動 | 自動 |
| **プレミアム対応** | ✗ | ✓ |
| **コンシェルジュサポート** | ✗ | ✓（優先対応） |

Rork Max を使うと、従来は Apple Developer アカウント、Xcode、ビルド設定などで何日もかかる準備作業が、ほぼ自動化されます。これが「2クリック申請」の正体です。

## 事前準備チェックリスト — 10分で完了

Rork Max での公開を始める前に、以下を準備してください。

### 1. Apple Developer Program への登録

**必須**: Apple Developer Program メンバーシップ（年間$99 USD）

- [Apple Developer](https://developer.apple.com/) にアクセス
- Apple ID でサインイン（ない場合は作成）
- Developer Program に登録（クレジットカード必須）
- 登録完了まで2-3営業日

### 2. App Store Connect へのアクセス権

Apple Developer Program 登録後、[App Store Connect](https://appstoreconnect.apple.com/) にアクセスできるようになります。

**初回ログイン時のチェック:**
- 二要素認証（Two-Factor Authentication）を有効化 — **必須**
- ユーザー権限を確認（"Admin" または "App Manager" であること）

### 3. Rork Max アカウント連携

Rork Max ダッシュボードで、Apple Developer アカウントをリンク：

```
Rork Max Dashboard
 → Settings → Apple Developer
 → Connect Apple ID
 → Choose App Store Connect Org
```

Rork Max が App Store Connect に直接アクセスできるよう、OAuth 認可を許可してください。

### 4. アプリの基本情報を事前準備

Rork Max 側で以下を入力します：

- **App Name** (e.g., "MyWeatherApp")
- **Bundle Identifier** (e.g., "com.yourcompany.myweatherapp") ← 一意（ユニーク）である必要あり
- **Version** (初回は "1.0.0")
- **Privacy Policy URL** ← 公式ウェブサイトかプライバシー専用ページ
- **Support URL**（オプション）

## Rork Max での2クリック申請フロー

### Step 1: アプリ生成と検証

Rork Max でアプリを生成したら、ローカルエミュレータで動作確認します。

```
Rork Max Dashboard
 → [Your App] → Preview
 → 各機能を一通り試す
```

**チェック項目:**
- ✓ ボタン・フォームが反応するか
- ✓ データ入力・保存が動作するか
- ✓ クラッシュやエラーメッセージはないか
- ✓ UI が iPhone 各サイズで正しくレンダリングされるか（iPhone 15, 15 Plus, 15 Pro, 15 Pro Max 推奨）

### Step 2: App Store Connect メタデータ設定

Rork Max の "App Store Setup" セクションで、以下を入力：

#### 2-1. App Information
- **App Name**: App Store に表示される名前（30文字以下推奨）
- **Subtitle** (iOS 13以降): サブタイトル（30文字以下）
  - 例: "天気予報 & リアルタイム警報"
- **Primary Category**: メインカテゴリ（天気、ユーティリティ、ゲーム等）
- **Secondary Category** (オプション)

#### 2-2. App Description
App Store プレビュー画面に表示される説明文（最大4,000文字）。

**重要ポイント:**
- **最初の2文で価値を表現** — ユーザーはスクロールしない可能性が高い
- **機能箇条書き**: 最大5項目に絞る
- **行動喚起（CTA）**: 「今すぐダウンロード」は要らない（ストア表示なので暗黙的）
- **言語**: App Store 設定言語に統一（日本語版なら日本語のみ）

**例（天気アプリ）:**
```
正確なリアルタイム天気予報とアラート機能。

主な機能:
• 最大10日間の天気予報
• 天気警報（豪雨・積雪・強風）
• 複数地点の気温・湿度管理
• バックグラウンド更新（5分ごと）
• 天気に応じた着衣提案
```

#### 2-3. スクリーンショット & プレビュー

App Store では、スクリーンショットがダウンロード判定を左右します。

**スクリーンショット枚数**: 2-5枚（推奨3-4枚）

**推奨解像度:**
- iPhone 6.7" (3388×1668px) — iPhone 15 Plus 以降の大画面用
- iPhone 5.5" (1242×2208px) — 汎用サイズ

**スクリーンショット構成:**
```
1枚目: アプリの最大の価値提案（ビジュアル重視）
2枚目: 主要機能 #1
3枚目: 主要機能 #2
4枚目: 差別化要因（競合との違い）
```

**重要**: Rork Max は、アプリ内スクリーンの自動キャプチャ機能があります。それを活用しつつ、Figma や Sketch でテキストオーバーレイを追加して、ユーザーに「何ができるのか」を一目で理解させましょう。

#### 2-4. キーワード（Search Keywords）

App Store の検索で上位に表示されるキーワード。最大100文字、カンマ区切り。

**選定方法:**
```
ターゲットユーザーが検索しそうなキーワード × 競争度が低い
```

**例（天気アプリ）:**
```
天気, 天気予報, 気温, 警報, リアルタイム, 気象, 天気アプリ, 天気予報アプリ
```

**禁止事項:**
- ❌ ブランド名（Apple、WeatherCom 等）を使わない（却下される）
- ❌ 関連なきワード詰め込み（スパム判定）

### Step 3: App Privacy Policy & Support URLs

#### Privacy Policy
GDPR・CCPA 対応が必須。プライバシーポリシーを GitHub Pages や Note で公開し、URL を設定します。

**最小限の内容:**
```markdown
# プライバシーポリシー

## 収集する情報
- 位置情報（天気予報精度向上のため）
- デバイス ID（クラッシュレポート）

## データの利用
- サービス提供のみ（第三者には非提供）

## ユーザーの権利
- いつでもアカウント削除可能
```

Rork Max はこの URL を App Store Connect に自動設定できます。

#### Support URL & Email
- Support URL: アプリ内に「お問い合わせ」ページを埋め込むか、別途サポートサイトを用意
- Support Email: アプリに問題があったときの問い合わせ先

### Step 4: ビルド・署名・アップロード

Rork Max は自動処理しますが、以下の舞台裏で何が起きているか理解しておくと、トラブル対応が楽です。

#### 4-1. 自動証明書生成

Rook Max が Apple から一時的な署名証明書（Code Signing Certificate）を取得します。

```
Apple Developer Program
 → Certificates, Identifiers & Profiles
 → Identifiers
 → [Your App Bundle ID] が自動登録される
 → Capabilities (Push Notification, HealthKit 等) も自動設定
```

#### 4-2. ビルド生成（IPA ファイル作成）

Rork Max が アプリをビルドし、IPA ファイル（iPhone App Archive）を生成。このファイルが実際に App Store Connect にアップロードされます。

**ファイルサイズの確認:**
- 推奨: 100MB 未満（ダウンロード速度とクラッシュリスク）
- App Store の上限: 4GB（超過は絶対NG）

### Step 5: App Store Connect へのアップロード

Rork Max が自動で以下を実行：

1. IPA ファイルを Apple の TestFlight（ベータ配信プラットフォーム）にアップロード
2. Apple 自動スキャン（バイナリ検査、脆弱性チェック）
3. 本番ビルドとして登録

**この段階でのエラーが出た場合:**

| エラー | 原因 | 対処 |
|---|---|---|
| "ITMS-90035" | Bundle ID が重複 | Bundle ID を変更（一意にする） |
| "ITMS-90209" | プライバシーマニフェスト未設定 | Privacy Policy URL を設定 |
| "ITMS-90338" | ビルド署名エラー | Apple Developer アカウント確認 |

### Step 6: メタデータ最終確認 & 提出（いよいよ2クリック！）

Rork Max の "Ready to Submit" 画面で：

```
✓ App Name
✓ Description
✓ Keywords
✓ Screenshot x3-4枚
✓ Privacy Policy URL
✓ Support URL
✓ カテゴリ
✓ レーティング（コンテンツレーティング）
```

すべてにチェックが入ったら、**「Submit for Review」をクリック**。

Apple に自動送信されます。これが「2クリック」の正体。

**送信から数時間後:**

Apple の自動システムが初期スクリーニングを実行。
- 数分で "Processing" → "Waiting for Review" に遷移します。

## App Store 審査ガイドライン — リジェクト原因トップ10

### 1. プライバシーポリシーがない、または不十分

**リジェクト理由:** "We noticed that your app requests access to the user's personal data, but your privacy policy does not clarify how you use this data."

**対処:**
- 位置情報・連絡先・写真 → プライバシーポリシーに明記
- 外部サーバーへのデータ送信 → 「どの情報を、どこへ送るのか」を具体的に記述

### 2. キラーワード（Trademark・Brand）の使用

**リジェクト理由:** "Your app name contains a word that may infringe on Apple's intellectual property."

**対処:**
- ❌ アプリ名に "Apple", "iOS", "iPhone" を含めない
- ❌ 競合他社のブランド名（"Spotify", "Instagram" 等）を含めない
- ✓ 説明文での言及はOK（「Spotify と連携」は許可）

### 3. ゲーム内課金（IAP）の未設定

**リジェクト理由:** "Your app contains in-app purchases, but you did not configure the necessary entitlements."

**対処:**
- Rork Max で "In-App Purchases" を有効化
- 各アイテムの価格・説明文を App Store Connect で設定
- テスト用 Sandbox Account を作成・動作確認

### 4. クラッシュバグ

**リジェクト理由:** "We were unable to review your app as it crashed on launch."

**対処:**
- TestFlight でベータテスターに配布、フィードバック収集
- iPhone 複数機種（15, 15 Plus, 14, SE 等）でテスト
- Rork Max の "Debug Mode" でログ確認

### 5. 不適切なコンテンツ（ヌード画像・暴力表現等）

**リジェクト理由:** "Your app contains content that is not appropriate for the App Store."

**対処:**
- ❌ 18歳未満向けアプリでの過激なコンテンツは禁止
- ✓ 年齢制限（17+）を設定すれば、大人向けアプリとして認可される可能性あり

### 6. パーミッション（権限）の過度な要求

**リジェクト理由:** "Your app requests permission to access the user's health data, but this feature is not presented to the user."

**対処:**
- 位置情報・カメラ・マイク → ユーザーに「なぜ必要か」を説明
- iOS 14+ では "Privacy Nutrition Label" でユーザーに透明性を提供

### 7. 外部リンク・決済の誘導

**リジェクト理由:** "Your app directs users to a web link to subscribe or unlock features."

**対処:**
- App Store In-App Purchase（IAP）を使うか、
- Web 決済の場合は Apple に明記（課金は IAP 経由が原則）
- 30% の手数料を覚悟

### 8. スクリーンショット・説明文の不一致

**リジェクト理由:** "The app functionality does not match your description."

**対処:**
- スクリーンショットに表示されている機能が、実際のアプリに実装されているか確認
- 説明文を実装内容に合わせて正確に修正

### 9. ローカライゼーション不足

**リジェクト理由:** "Your app is submitted to the Japanese App Store, but the description is in English."

**対処:**
- 日本語版ストアでの提出なら、説明文・スクリーンショット・キーワード **すべて日本語** で統一
- 複数言語対応する場合は、言語ごとに別々のメタデータを設定

### 10. ビルド署名エラー

**リジェクト理由:** "Your build could not be signed."

**対処:**
- Apple Developer Program の有効期限確認（年$99で自動更新）
- Bundle ID が重複していないか確認
- Rork Max の "Reconnect Apple Developer Account" で再認可

## 審査期間 — いつ結果が来るのか？

Apple の審査期間は通常 **24〜48時間**。ただし：

- **祝日・メンテナンス時**: 3-5日に延長の可能性
- **複数回リジェクト**: 2-3回目の再提出では1回目より長くなる傾向

**進捗確認:**
```
App Store Connect
 → Your App → App Version
 → Status: Waiting for Review / In Review / Ready for Sale
```

リジェクトされた場合、Apple から詳細メールが届きます。メール内の理由コード（例：4.3）を [App Store Review Guidelines](https://developer.apple.com/app-store/review/guidelines/) で検索すると、詳しい対処法が見つかります。

## 審査通過後のリリース管理

### Phase 1: Staged Rollout（段階的公開）

審査通過後、全ユーザーへの一括公開ではなく、段階的な公開をお勧めします。

```
App Store Connect
 → [Your App] → Version
 → Phased Release
 → Start: Day 1 (5%)
 →       Day 2 (10%)
 →       Day 3 (25%)
 →       Day 4 (50%)
 →       Day 5 (100%)
```

この方法なら、初日のバグを 5% のユーザーに限定でき、クリティカルなエラーが検出されたら即座に回収・修正できます。

### Phase 2: バージョン管理

Rork Max で新しいバージョン（1.1.0, 1.2.0 等）をリリースする際：

```
Version Number: X.Y.Z
 X = Major (大機能追加)
 Y = Minor (小機能・改善)
 Z = Patch (バグ修正)
```

**例:**
- 1.0.0 → 1.0.1 (バグ修正)
- 1.0.1 → 1.1.0 (新機能追加)
- 1.1.0 → 2.0.0 (大幅リデザイン)

### Phase 3: What's New の記載

ユーザー向けの更新履歴。App Store の "Whats New" セクションに表示されます。

```
Version 1.1.0

新機能:
• オフラインモード対応
• ダークモード追加

バグ修正:
• クラッシュ問題を解決
• 通知遅延を改善
```

## ASO（App Store最適化）戦略 — ダウンロード数を伸ばす

App Store での検索順位を上げ、より多くのユーザーにアプリを見つけてもらうための施策が ASO です。

### ASO の3本柱

#### 1. キーワード最適化

**タイトル**: 最強のキーワードを入れるが、不自然でない範囲で

```
❌ 天気予報 天気 天気アプリ リアルタイム 天気警報 気温
✓ 天気予報 - リアルタイムアラート
```

**説明文**: ユーザーが検索しそうなキーワードを自然に含める

```
「正確な天気予報」「気温管理」「天気警報」「天気アプリ」
を説明文の最初の100文字に含める
```

#### 2. スクリーンショット最適化

**ユーザーの行動:**
1. アプリ名・アイコンを見て、クリック
2. スクリーンショット（最初の1-2枚）を5秒以内でスキャン
3. 「これ、欲しい」と判断したらダウンロード

**重要:**
- 1枚目は **最大の価値提案** を視覚的に（ビジネスロジックではなく）
- テキストオーバーレイは大きく、読みやすく
- カラースキームを統一（プロフェッショナル感）

#### 3. レーティング・レビュー管理

ユーザーレビューの数と質が、App Store の検索ランキングを左右します。

**高レーティングを獲得するコツ:**

```
初回起動時の "Rate This App" プロンプト
（iOS 14.3+ の SKStoreReviewController を使用）

ただし、ユーザーが操作を完了した直後が効果的。
アプリ起動直後はNG。
```

**ネガティブレビューへの対応:**

```
1星レビュー: 丁寧に返信 & アップデートで修正
「ご指摘ありがとうございます。〇〇の問題は、
バージョン1.2.0で修正予定です。
アップデート時に改めてお試しください。」

2-3星レビュー: 具体的な改善提案を求める
「〇〇機能が使いにくいとのこと、
具体的にどうなれば使いやすくなるか、
ご意見をお聞かせください。」
```

Apple のレビュー表示アルゴリズムは、「最近の高レーティング」を優先するので、継続的な改善で新しい星5を増やすことが重要です。

## よくある質問・トラブルシューティング

### Q1. 審査で「メタデータが不正確」と言われた。修正して再提出できるか？

**A:** はい。ただし、アプリのバイナリ（IPA ファイル）には変更がない場合、新しいビルドをアップロードせず、メタデータだけ修正して再提出できます。

```
App Store Connect
 → [Your App] → App Information
 → 説明文・スクリーンショットを編集
 → Save → Submit for Review
```

### Q2. AppTrackingTransparency（ATT）とは？

**A:** iOS 14.5 以降、ユーザーデータを第三者（Facebook Ads 等）に送信する際、明示的な許可が必須になりました。

```
// Swift
import AppTrackingTransparency

ATTrackingManager.requestTrackingAuthorization { status in
    // status = .authorized / .denied / .restricted
}
```

Rork Max で広告 SDK を統合する場合、ATT 対応は **必須**です。対応しないとリジェクトされます。

### Q3. Sandbox Account でテスト購入ができない

**A:** App Store Connect で Sandbox Tester を作成してください。

```
App Store Connect
 → Users and Access
 → Sandbox Testers
 → Create
 → Email: test+1@example.com
 → Password: (任意)
```

TestFlight デバイスで、Sandbox Tester アカウントでログインしてテスト購入を試みます。

### Q4. アプリがクラッシュして、審査通過できない

**A:** 以下をチェック：

1. **メモリリーク**: Xcode の Instruments で計測
2. **バックグラウンド処理**: iOS 13+ では制限あり（Background Tasks Framework 使用）
3. **ネットワークタイムアウト**: API 呼び出しにタイムアウトを設定

Rork Max は自動的にこれらの対策を施しますが、カスタムコードがあれば手作業で修正が必要です。

### Q5. 何度も リジェクトされています。別の Bundle ID で再チャレンジできるか？

**A:** 技術的には可能ですが、Apple は何度もリジェクトするアプリを「問題あり」と見做します。新しい Bundle ID でも同じ理由でリジェクトされる可能性が高いです。

**推奨:**
1. リジェクト理由を Apple メールから詳細に読む
2. App Store Review Guidelines（4.3, 4.4 等のセクション）を確認
3. アプリのロジックを根本的に修正
4. 元の Bundle ID で再チャレンジ

## まとめ

Rork Max の "2クリック App Store 申請" は、従来の複雑なビルド・署名プロセスを大幅に簡素化しました。しかし、申請後の「審査通過」「ユーザー獲得」までを確実にするには、以下の3つが不可欠です。
