## アプリクラッシュと白画面の違い

Rork で開発するとき、アプリが**クラッシュ**することと**白画面になる**ことは別の問題です。

- **クラッシュ** — アプリが完全に落ちてホーム画面に戻される
- **白画面** — アプリは起動しているが、何も表示されない状態が続く

この記事では、両方のパターンの原因と対処法を解説します。

---

## アプリクラッシュの主な原因 TOP 5

### 1. メモリ不足（Out of Memory）

**症状**: 大きな画像をたくさん読み込んだり、リストをスクロールしたりするとクラッシュする。特に古いデバイスで発生しやすい。

**原因**: 高解像度画像をそのままメモリに読み込んでいるケースが大半。

```typescript
// ❌ 間違い: 高解像度画像をそのまま表示
import { View, Image, FlatList } from "react-native";

export default function PhotoGallery({ photos }: { photos: string[] }) {
  return (
    <FlatList
      data={photos}
      renderItem={({ item }) => (
        <View>
          {/* 元の高解像度画像をそのまま表示 → メモリ不足でクラッシュ */}
          <Image
            source={{ uri: item }}
            style={{ width: "100%", height: 400 }}
          />
        </View>
      )}
      keyExtractor={(item, idx) => idx.toString()}
    />
  );
}

// ✅ 正しい: 画像をリサイズして表示
export default function PhotoGallery({ photos }: { photos: string[] }) {
  return (
    <FlatList
      data={photos}
      renderItem={({ item }) => (
        <View>
          {/* キャッシュを有効にして、リサイズのサイズを指定 */}
          <Image
            source={{ uri: item }}
            style={{ width: 300, height: 300 }}
            defaultSource={require("./placeholder.png")}
          />
        </View>
      )}
      keyExtractor={(item, idx) => idx.toString()}
      maxToRenderPerBatch={10}  {/* 一度に描画する数を制限 */}
      removeClippedSubviews={true}  {/* 画面外のビューを削除 */}
    />
  );
}
```

**解決方法**:
- 画像をリサイズして表示（`width` / `height` を明示的に指定）
- `FlatList` の `maxToRenderPerBatch` と `removeClippedSubviews` で描画数を制限
- キャッシュライブラリ（`react-native-image-cache-hoc`など）を導入

### 2. 無限ループ・無限再レンダリング

**症状**: アプリが起動後、CPU使用率が 100% に上がり、数秒でクラッシュする。

**原因**: `useEffect` の依存配列（dependency array）が空、または state の更新トリガーが永遠に続く。

```typescript
// ❌ 間違い: useEffect が無限に実行される
import { useState, useEffect } from "react";
import { View, Text } from "react-native";

export default function App() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // 依存配列がないので、毎レンダリングで実行される
    // setCount で再レンダリングが起こり、また useEffect が実行される → 無限ループ
    setCount(count + 1);
  }); // ← 依存配列がない!

  return <Text>{count}</Text>;
}

// ✅ 正しい: 依存配列を指定して制御
export default function App() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // 初回マウント時のみ実行
    console.log("Component mounted");
    return () => console.log("Component unmounted");
  }, []); // ← 空の依存配列 = 初回マウント時のみ

  return <Text>{count}</Text>;
}
```

**解決方法**:
- `useEffect` には必ず依存配列を指定する
- 依存配列を省略すると、毎レンダリングで実行される
- state 更新で無限ループが起こっていないか確認

### 3. ネイティブモジュールの互換性問題

**症状**: カメラ、GPS、ファイルシステムなどのネイティブ機能を使うとクラッシュする。

**原因**: Expo のバージョンとネイティブモジュールのバージョンがミスマッチしている。

```typescript
// ❌ インストールしたカメラライブラリが Expo バージョンと合わない
import { Camera } from "react-native-camera";

// ✅ 解決: Expo の公式ドキュメントで推奨バージョンを確認してインストール
// $ npx expo install expo-camera
// $ npm install @react-native-camera-roll/camera-roll
```

**解決方法**:
- `npx expo install` で Expo 互換版を自動インストール
- `package.json` のバージョンを確認
- `expo prebuild` でネイティブコードを再ビルド
- Expo の公式マトリックスを参照

### 4. 非同期処理のエラーハンドリング不足

**症状**: API呼び出しやファイル読み込み時にネットワークエラーが発生するとクラッシュする。

**原因**: `try-catch` や `.catch()` を忘れている。

```typescript
// ❌ 間違い: エラーハンドリングがない
import { useEffect, useState } from "react";
import { View, Text } from "react-native";

export default function UserProfile() {
  const [user, setUser] = useState(null);

  useEffect(() => {
    // ネットワークエラーが発生すると、エラーが処理されずアプリがクラッシュ
    fetch("https://api.example.com/user/123")
      .then((res) => res.json())
      .then((data) => setUser(data));
  }, []);

  return <Text>{user?.name}</Text>;
}

// ✅ 正しい: エラーハンドリングを追加
export default function UserProfile() {
  const [user, setUser] = useState(null);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch("https://api.example.com/user/123")
      .then((res) => res.json())
      .then((data) => setUser(data))
      .catch((err) => {
        console.error("Failed to fetch user:", err);
        setError("ユーザー情報の取得に失敗しました");
      });
  }, []);

  if (error) return <Text style={{ color: "red" }}>{error}</Text>;
  if (!user) return <Text>読み込み中...</Text>;

  return <Text>{user?.name}</Text>;
}
```

**解決方法**:
- すべての非同期処理に `.catch()` または `try-catch` を追加
- エラー時の UI（エラーメッセージ表示など）を用意

### 5. ナビゲーション設定のミス

**症状**: 画面遷移時にクラッシュする。特に複数タブを切り替える時に発生。

**原因**: Navigator の設定間違いや、存在しないスクリーン名の参照。

```typescript
// ❌ 間違い: スクリーン定義がない名前に遷移しようとする
import { NavigationContainer } from "@react-navigation/native";
import { createNativeStackNavigator } from "@react-navigation/native-stack";

const Stack = createNativeStackNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Home" component={HomeScreen} />
        {/* "Profile" スクリーンの定義がない */}
      </Stack.Navigator>
    </NavigationContainer>
  );
}

function HomeScreen({ navigation }: any) {
  return (
    <Button
      title="Go to Profile"
      onPress={() => navigation.navigate("Profile")} {/* クラッシュ！ */}
    />
  );
}

// ✅ 正しい: 必ずスクリーン定義を先に追加
export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen name="Profile" component={ProfileScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}
```

**解決方法**:
- `Stack.Screen` でスクリーン名を定義してから、`navigate()` で遷移
- スクリーン名は大文字小文字区別あり（`"Home"` と `"home"` は別）

## 白画面（White Screen）パターン別対処

### パターン1: 起動直後に白画面

**原因**: スプラッシュスクリーンの設定が無い、またはスプラッシュスクリーンが消えていない。

```typescript
// ✅ Expo.SplashScreen を使用して、初期化完了時に消す
import * as SplashScreen from "expo-splash-screen";
import { useCallback, useEffect, useState } from "react";
import { View, Text } from "react-native";

SplashScreen.preventAutoHideAsync(); // 自動消去を停止

export default function App() {
  const [fontsLoaded, setFontsLoaded] = useState(false);

  useEffect(() => {
    // 初期化処理（フォント読み込みなど）
    setTimeout(() => setFontsLoaded(true), 2000);
  }, []);

  const onLayoutRootView = useCallback(async () => {
    if (fontsLoaded) {
      await SplashScreen.hideAsync(); // 初期化完了後にスプラッシュスクリーンを消す
    }
  }, [fontsLoaded]);

  if (!fontsLoaded) return null;

  return (
    <View onLayout={onLayoutRootView} style={{ flex: 1 }}>
      <Text>アプリが読み込まれました</Text>
    </View>
  );
}
```

### パターン2: 特定の画面で白画面

**原因**: コンポーネント内でエラーが発生しているが、エラーバウンダリで処理されていない。

**対処法**: [「エラーバウンダリの実装」](/articles/rork-dev/rork-react-native-expo-runtime-errors-fix)を参照して、エラーバウンダリを追加。

### パターン3: アップデート後に白画面

**原因**: 古いキャッシュが残っている。

```bash
# キャッシュをクリア
npx expo start --clear

# Expo Go ではキャッシュクリア後に再度起動
```

## Rork のプレビュー画面 vs 実機の違い

Rork のプレビュー画面（ブラウザのシミュレータ）と実機（iPhone / Android デバイス）では、動作が異なることがあります。

- **プレビュー画面**: CPU スピードが速く、メモリが十分なため、実機では発生しないバグが隠れる
- **実機**: より正確なパフォーマンス情報が得られる

**推奨**: Rork で開発を始める際は、なるべく早い段階で実機デバッグに移行しましょう。

## クラッシュレポートの取得方法

Rork Companion（Mac アプリ）を使うと、クラッシュレポートが自動的に保存されます。

1. **Rork Companion を起動**
2. **アプリをテスト中にクラッシュ**
3. **Rork Companion の「Crash Logs」タブを確認**

クラッシュログには以下の情報が含まれます：

- **エラーメッセージ** — 何が起きたのか
- **スタックトレース** — どのファイルの何行目で起きたのか
- **デバイス情報** — iPhone vs Android、OS バージョン など

このログをもとに原因を特定できます。

## テスト・品質チェックのベストプラクティス

クラッシュを事前に防ぐために、開発段階で以下をチェックしましょう：

- **複数デバイスでテスト** — iPhone と Android、新旧デバイス複数台で試す
- **低メモリ環境でテスト** — デバッグ設定で「Low Memory シミュレーション」を有効化
- **ネットワークエラーシミュレーション** — オフライン状態や遅延ネットワークでテスト
- **画面遷移の全パターン確認** — すべての遷移経路を手動で確認
- **長時間アプリを起動** — メモリリークが発生していないか監視

## FAQ

**Q1: Rork のプレビューでは動くのに、実機ではクラッシュします。**

A: プレビュー環境はシミュレータなので、実機より高速・メモリが多い可能性があります。実機で発生するのであれば、メモリ最適化やパフォーマンスチューニングが必要です。

**Q2: アプリがクラッシュすると、ユーザーに何と説明すればいい?**

A: 「申し訳ありません。アプリでエラーが発生しました。最新バージョンをご利用いただくか、キャッシュをクリアして再度お試しください」と案内するのが標準的です。

**Q3: クラッシュレポートを取得できません。**

A: Rork Companion が起動しており、同じ Wi-Fi に接続されているか確認してください。また、アプリを再起動して再度テストしてください。

## デバッグの流れ（ステップバイステップ）

1. **エラーログを読む** — コンソール出力から何が起きたかを特定
2. **エラー箇所を特定** — ログのスタックトレースからファイル名・行番号を確認
3. **該当箇所のコードを確認** — 初期化漏れ、無限ループ、エラーハンドリング不足など
4. **修正コードを書く** — CLAUDE.md のベストプラクティスに従う
5. **複数デバイスでテスト** — 修正が有効か、別のデバイスでも再現するか確認
6. **本番にデプロイ** — テスト完了後、アプリストアにリリース

## まとめ

Rork アプリのクラッシュ・白画面は、以下の 5 つのパターンのいずれかに該当することがほとんどです：

1. **メモリ不足** — 画像リサイズとキャッシュ管理
2. **無限ループ** — useEffect の依存配列確認
3. **ネイティブモジュール互換性** — npx expo install と prebuild
4. **エラーハンドリング不足** — try-catch と .catch() を追加
5. **ナビゲーション設定ミス** — スクリーン定義を確認

デバッグツール（Rork Companion、クラッシュログ、エラーバウンダリ）を使いこなすことで、開発効率が飛躍的に向上します。
