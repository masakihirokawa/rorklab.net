Rork Maxを使い始めた方が最初に驚くのは、「こんなにさっとアプリの骨格ができるのか」という体験です。そして数日後に直面するのが、「プレビューでは完璧に動いていたのに、実機テストでいろんな問題が出てくる」という壁です。

これはRork Maxの弱点というよりも、ExpoやReact Nativeが持つ「プレビュー環境と本番環境の差」がRorkを通すことで表面化しているだけです。問題のパターンは決まっているので、先に知っておけば大幅に時間を節約できます。

本記事では、実際にRork Maxでアプリを開発した経験から、特によく詰まるポイント10個を取り上げます。

## 詰まりポイント1: SwiftUIコードが期待通りに生成されない

Rork MaxはExpoを主軸としつつ、iOS向けにSwiftUIコードを出力することができます。ただし「SwiftUIで書いて」と伝えるだけでは、Rorkが判断に迷って中途半端なコードになることがあります。

**プロンプト改善のポイント:**

```
❌ 「SwiftUIでタスク管理アプリを作って」

✅ 「以下の仕様でSwiftUIアプリを作ってください：
- 対象: iOS 17以降
- UI: SwiftUI + NavigationStack
- データ: @Observable マクロを使ったViewModelパターン
- 永続化: SwiftData
- タスクのCRUD機能、完了/未完了の切り替え
- 画面構成: タスク一覧画面 → タスク詳細/編集画面」
```

Rorkへのプロンプトは「何を作るか」だけでなく「どの技術スタックで作るか」まで指定することで、品質が上がります。

```swift
// Rorkが生成するViewModelの典型的な問題パターン
// ❌ 古いObservableObject + @Published パターン（iOS 17以降なら@Observableが推奨）
class TaskViewModel: ObservableObject {
    @Published var tasks: [Task] = []
}

// ✅ iOS 17以降向けの@Observableパターンを明示的に指定する
@Observable
class TaskViewModel {
    var tasks: [Task] = []
    
    func addTask(_ title: String) {
        tasks.append(Task(id: UUID(), title: title, isCompleted: false))
    }
    
    func toggleTask(_ task: Task) {
        if let index = tasks.firstIndex(where: { $0.id == task.id }) {
            tasks[index].isCompleted.toggle()
        }
    }
}
```

## 詰まりポイント2: Expo Routerのファイルベースルーティングでエラーになる

Rork Maxが生成するアプリはExpo RouterのApp Routerパターンを使います。ファイル構成が正しくないと、ルートが認識されなかったりエラーになったりします。

```
app/
├── _layout.tsx      ← ルートレイアウト（必須）
├── index.tsx        ← ホーム画面
├── (tabs)/          ← タブグループ
│   ├── _layout.tsx  ← タブレイアウト
│   ├── home.tsx
│   └── profile.tsx
└── modal.tsx        ← モーダル画面
```

よくある問題は、`_layout.tsx` の `<Stack>` や `<Tabs>` の設定が不完全なことです。

```typescript
// app/_layout.tsx の正しい基本構造
import { Stack } from 'expo-router';
import { useColorScheme } from 'react-native';

export default function RootLayout() {
  const colorScheme = useColorScheme();
  
  return (
    <Stack
      screenOptions={{
        headerStyle: { backgroundColor: colorScheme === 'dark' ? '#1a1a1a' : '#ffffff' },
        headerTintColor: colorScheme === 'dark' ? '#ffffff' : '#000000',
      }}
    >
      <Stack.Screen name="index" options={{ title: 'ホーム' }} />
      <Stack.Screen name="modal" options={{ presentation: 'modal', title: '' }} />
    </Stack>
  );
}
```

Rorkに「Expo Router App Routerパターンで作って」と明示的に伝え、生成後にこのファイル構成になっているか確認してください。

## 詰まりポイント3: Supabase認証の実装が不完全

Rork MaxとSupabaseの組み合わせはよく使われますが、認証フローの実装が不完全なケースが多いです。特に「ログイン状態の永続化」と「未認証ユーザーのリダイレクト」が漏れやすいです。

```typescript
// app/_layout.tsx での認証状態管理
import { useEffect, useState } from 'react';
import { useRouter, useSegments } from 'expo-router';
import { supabase } from '../lib/supabase';
import type { Session } from '@supabase/supabase-js';

export default function RootLayout() {
  const router = useRouter();
  const segments = useSegments();
  const [session, setSession] = useState<Session | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    // 現在のセッションを取得
    supabase.auth.getSession().then(({ data: { session } }) => {
      setSession(session);
      setIsLoading(false);
    });

    // セッション変更を監視
    const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, session) => {
      setSession(session);
    });

    return () => subscription.unsubscribe();
  }, []);

  useEffect(() => {
    if (isLoading) return;

    const inAuthGroup = segments[0] === '(auth)';

    if (\!session && \!inAuthGroup) {
      // 未認証ユーザーをログイン画面へ
      router.replace('/(auth)/login');
    } else if (session && inAuthGroup) {
      // 認証済みユーザーをメイン画面へ
      router.replace('/(tabs)/home');
    }
  }, [session, segments, isLoading]);

  if (isLoading) {
    return null; // またはスプラッシュ画面
  }

  return <Slot />;
}
```

Supabase クライアントの設定も重要です。

```typescript
// lib/supabase.ts
import { createClient } from '@supabase/supabase-js';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { AppState } from 'react-native';

const supabaseUrl = process.env.EXPO_PUBLIC_SUPABASE_URL\!;
const supabaseAnonKey = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY\!;

export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  auth: {
    storage: AsyncStorage,  // モバイルではAsyncStorageが必要
    autoRefreshToken: true,
    persistSession: true,   // セッションを永続化
    detectSessionInUrl: false, // モバイルアプリではfalse
  },
});

// バックグラウンドでのトークン自動更新
AppState.addEventListener('change', (state) => {
  if (state === 'active') {
    supabase.auth.startAutoRefresh();
  } else {
    supabase.auth.stopAutoRefresh();
  }
});
```

## 詰まりポイント4: プッシュ通知の設定

Expo Notificationsは設定が複雑で、Rorkが生成するコードが不完全なことがよくあります。

```typescript
// hooks/useNotifications.ts
import { useEffect, useRef, useState } from 'react';
import * as Notifications from 'expo-notifications';
import * as Device from 'expo-device';
import Constants from 'expo-constants';
import { Platform } from 'react-native';

// 通知の表示方法を設定（アプリがフォアグラウンドの場合も表示）
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: false,
    shouldSetBadge: false,
  }),
});

async function registerForPushNotificationsAsync(): Promise<string | undefined> {
  // 実機以外では Push Token が取得できない
  if (\!Device.isDevice) {
    console.warn('Push notifications require a physical device');
    return undefined;
  }

  // 現在の権限状態を確認
  const { status: existingStatus } = await Notifications.getPermissionsAsync();
  let finalStatus = existingStatus;

  if (existingStatus \!== 'granted') {
    const { status } = await Notifications.requestPermissionsAsync();
    finalStatus = status;
  }

  if (finalStatus \!== 'granted') {
    console.warn('Push notification permission not granted');
    return undefined;
  }

  // Expo Push Token を取得
  const projectId = Constants.expoConfig?.extra?.eas?.projectId;
  if (\!projectId) {
    throw new Error('EAS Project ID not found in app config');
  }

  const token = await Notifications.getExpoPushTokenAsync({ projectId });
  
  // Android チャンネル設定（Android のみ必要）
  if (Platform.OS === 'android') {
    await Notifications.setNotificationChannelAsync('default', {
      name: 'default',
      importance: Notifications.AndroidImportance.MAX,
      vibrationPattern: [0, 250, 250, 250],
    });
  }

  return token.data;
}

export function useNotifications() {
  const [expoPushToken, setExpoPushToken] = useState<string>();
  const notificationListener = useRef<Notifications.Subscription>();
  const responseListener = useRef<Notifications.Subscription>();

  useEffect(() => {
    registerForPushNotificationsAsync().then(setExpoPushToken);

    // 通知受信時のハンドラ
    notificationListener.current = Notifications.addNotificationReceivedListener(notification => {
      console.log('Notification received:', notification);
    });

    // 通知タップ時のハンドラ
    responseListener.current = Notifications.addNotificationResponseReceivedListener(response => {
      console.log('Notification tapped:', response);
      // ここで画面遷移などの処理を行う
    });

    return () => {
      notificationListener.current?.remove();
      responseListener.current?.remove();
    };
  }, []);

  return { expoPushToken };
}
```

`app.json` への設定も忘れずに。

```json
        }
      ]
    ]
  }
}
```

## 詰まりポイント5〜10（要点まとめ）

残りの詰まりポイントを要点で整理します。

**5. カメラ・画像選択が動かない**
`expo-image-picker` のバージョンをExpo SDKに合わせた上で、`app.json` の `plugins` に設定を追加する。iOS は `NSPhotoLibraryUsageDescription` も必要。

**6. React Queryのキャッシュが実機でクリアされない**
`QueryClient` の `staleTime` と `gcTime` を明示的に設定する。デフォルト値が環境によって異なる動作をすることがある。

**7. Android のキーボードがコンテンツを隠す**
`app.json` の `android.softwareKeyboardLayoutMode` を `"pan"` に設定するか、`KeyboardAvoidingView` を適切に配置する。

**8. EASビルドでnpm installが失敗する**
`package.json` のpeer dependencyの競合が原因のことが多い。`npx expo install --fix` の後で `npm install --legacy-peer-deps` を試す。

**9. App Store提出でメタデータ不足エラー**
`app.json` の `ios.infoPlist` にアプリで使う機能の説明文を全て記入する。Apple Reviewチームが要求するより多くの説明が必要な場合がある。

**10. Rorkが同じコードを繰り返し生成してしまう**
プロンプトに「前回の生成とは異なるアプローチで」「既存のコードベースを考慮して」と明示する。Rorkはコンテキストが長くなると同じパターンに収束しやすくなることがある。

## 本番品質のアプリを作るためのプロンプト設計

Rork Maxから最高品質のコードを引き出すには、プロンプトの設計が重要です。

```
# 効果的なRork Maxプロンプトのテンプレート

アプリ名: [名前]
技術スタック:
- フレームワーク: Expo SDK 52 + TypeScript + Expo Router v3
- バックエンド: Supabase
- 状態管理: Zustand
- スタイリング: NativeWind (Tailwind CSS for React Native)

機能要件:
1. [具体的な機能1]
2. [具体的な機能2]

非機能要件:
- エラーハンドリングを各API呼び出しに含めること
- ローディング状態のUIを実装すること
- 空データ時のEmptyStateコンポーネントを含めること

既に実装済みの内容（Rorkに追加改修させる場合）:
- [既存実装の概要]

今回の変更内容:
- [変更したい具体的な内容]
```

具体的であればあるほど、Rorkが生成するコードの品質は上がります。「なんとなく動くもの」ではなく「本番に出せるもの」を最初から目指すプロンプト設計を心がけてみてください。その積み重ねが、開発のスピードに直結します。
