NVIDIA NeMo フレームワークは、大規模言語モデル（LLM）の学習とファインチューニングを簡潔に実装するための統合プラットフォームです。本記事では、NeMo で独自モデルを構築し、NIM マイクロサービスでエッジ推論を実装し、Rork アプリケーションに統合する完全なワークフローを解説します。

## NeMo フレームワークの基本構造

NVIDIA NeMo は、エンタープライズグレードの LLM 開発を想定した包括的なフレームワークです。モデルの学習からデプロイまでの全段階を PyTorch ベースで統一的に管理できます。

### NeMo の 3 層アーキテクチャ

**レイヤー 1: Core Framework（基盤レイヤー）**
- `NeuralModule` による宣言的な計算グラフ定義
- 自動混合精度（AMP）と分散学習対応
- チェックポイント管理とハイパーパラメータ追跡

**レイヤー 2: Collections（ドメイン別実装）**
- NLP Collection（言語理解・生成）
- ASR Collection（音声認識）
- TTS Collection（音声合成）
- Vision Collection（視覚タスク）

**レイヤー 3: Pretrained Models & Tools（運用ツール）**
- NGC にホストされた事前学習モデル（Nemotron、Llama など）
- LoRA・QLoRA によるパラメータ効率的ファインチューニング
- エクスポート機能（ONNX、TensorRT、NVIDIA NIM）

### モバイルアプリに適した軽量モデル

Nemotron-ASR-Streaming は、モバイルデバイス上での推論を想定した 600M パラメータの軽量音声認識モデルです。これを基盤として、ドメイン特化なファインチューニングを加えることで、帯域幅制約下でも高精度な推論が可能になります。

### LoRA による効率的なファインチューニング

完全なファインチューニングではなく、Low-Rank Adaptation（LoRA）を用いることで、メモリ使用量を大幅に削減できます。

from nemo.collections.nlp.models import MegatronGPTModel
from nemo.core.config import OmegaConf
import torch

# LoRA コンフィグ
lora_config = OmegaConf.create({
    "target_modules": ["query_proj", "key_proj", "value_proj", "dense_h_to_4h", "dense_4h_to_h"],
    "r": 8,  # LoRA ランク
    "lora_alpha": 16,
    "lora_dropout": 0.1,
    "bias": "none",
})

# モデルロードと LoRA 初期化
model = MegatronGPTModel.from_pretrained("nvidia/megatron-gpt-345m")
model.add_lora(lora_config)

# 学習設定
trainer_config = {
    "max_epochs": 3,
    "devices": 1,
    "precision": "16-mixed",  # 混合精度
    "accumulate_grad_batches": 4,

from pytorch_lightning import Trainer

## ステップ 3: エッジ推論用へのエクスポートと最適化

### TensorRT による推論最適化

モバイルデバイスでの高速推論には、NVIDIA TensorRT による量子化とグラフ最適化が効果的です。

```python
from nemo.utils import export_to_onnx

# ONNX 形式へのエクスポート（中間フォーマット）
model.export(
    output_file="model_export/model.onnx",
    check_trace=True,
    input_example=torch.randn(1, 512)
)

# TensorRT エンジンへの変換
import tensorrt as trt

def build_tensorrt_engine(onnx_path: str, output_path: str, precision: str = "fp16"):
    """ONNX から TensorRT エンジンをビルド"""
    logger = trt.Logger(trt.Logger.WARNING)
    builder = trt.Builder(logger)
    network = builder.create_network(
        1 << int(trt.NetworkDefinitionCreationFlag.EXPLICIT_BATCH)
    )
    parser = trt.OnnxParser(network, logger)

    with open(onnx_path, 'rb') as f:
        if not parser.parse(f.read()):
            raise RuntimeError("Failed to parse ONNX model")

    config = builder.create_builder_config()
    config.set_flag(trt.BuilderFlag.GPU_FALLBACK)

    if precision == "fp16":
        config.set_flag(trt.BuilderFlag.FP16)
    elif precision == "int8":
        config.set_flag(trt.BuilderFlag.INT8)

    engine = builder.build_engine(network, config)

    with open(output_path, "wb") as f:
        f.write(engine.serialize())

    return engine

build_tensorrt_engine("model_export/model.onnx", "model_export/model.trt", "fp16")
```

## ステップ 4: NVIDIA NIM マイクロサービスの構築

### NIM による統一的な推論インターフェース

NVIDIA NIM（NVIDIA Inference Microservices）は、クラウド・データセンター・エッジにおいて一貫した推論エンドポイントを提供します。

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import torch
import asyncio
from typing import List, Optional

app = FastAPI(title="NeMo Edge Inference Service")

class InferenceRequest(BaseModel):
    text: str
    max_tokens: Optional[int] = 256
    temperature: Optional[float] = 0.7

class InferenceResponse(BaseModel):
    result: str
    tokens_used: int
    latency_ms: float

# モデルのロード（アプリケーション起動時）
device = "cuda" if torch.cuda.is_available() else "cpu"
model = None
tokenizer = None

@app.on_event("startup")
async def load_model():
    """NeMo モデルをメモリにロード"""
    global model, tokenizer
    from nemo.collections.nlp.models import MegatronGPTModel
    from transformers import AutoTokenizer

    model = MegatronGPTModel.from_pretrained("nvidia/megatron-gpt-345m")
    model = model.to(device)
    model.eval()
    tokenizer = AutoTokenizer.from_pretrained("nvidia/megatron-gpt-345m")

@app.post("/v1/completions", response_model=InferenceResponse)
async def inference(request: InferenceRequest):
    """テキスト生成推論エンドポイント"""
    import time

    start_time = time.time()

    try:
        # トークン化
        input_ids = tokenizer.encode(request.text, return_tensors="pt").to(device)

        # 推論実行
        with torch.no_grad():
            output_ids = model.generate(
                input_ids,
                max_length=request.max_tokens + input_ids.shape[1],
                temperature=request.temperature,
                top_p=0.9,
                do_sample=True,
                pad_token_id=tokenizer.eos_token_id,
            )

        # 結果のデコード
        output_text = tokenizer.decode(output_ids[0], skip_special_tokens=True)

        latency_ms = (time.time() - start_time) * 1000

        return InferenceResponse(
            result=output_text,
            tokens_used=output_ids.shape[1] - input_ids.shape[1],
            latency_ms=latency_ms,
        )

    except RuntimeError as e:
        raise HTTPException(status_code=500, detail=f"Inference failed: {str(e)}")

@app.post("/v1/health")
async def health_check():
    """ヘルスチェックエンドポイント"""
    return {
        "status": "healthy",
        "device": device,
        "model_loaded": model is not None,
    }
```

### Expo SDK 55 環境での NIM クライアント実装

[Expo SDK 55 での パフォーマンス最適化](/articles/rork-dev/expo-sdk-55-performance-optimization) を参考に、以下のようにエッジ推論クライアントを実装します。

```typescript
// libs/nemo-inference-client.ts
import axios, { AxiosInstance } from 'axios';
import { Platform } from 'react-native';

interface InferenceRequest {
  text: string;
  maxTokens?: number;
  temperature?: number;
}

interface InferenceResponse {
  result: string;
  tokensUsed: number;
  latencyMs: number;
}

class NeMoInferenceClient {
  private client: AxiosInstance;
  private baseURL: string;

  constructor(nimServiceUrl: string) {
    this.baseURL = nimServiceUrl;
    this.client = axios.create({
      baseURL: this.baseURL,
      timeout: 30000,
      headers: {
        'Content-Type': 'application/json',
      },
    });
  }

  /**
   * ローカルまたはエッジサーバーに対してテキスト生成推論を実行
   */
  async generateText(request: InferenceRequest): Promise<InferenceResponse> {
    try {
      const response = await this.client.post<InferenceResponse>(
        '/v1/completions',
        {
          text: request.text,
          max_tokens: request.maxTokens || 256,
          temperature: request.temperature || 0.7,
        }
      );

      return response.data;
    } catch (error) {
      if (axios.isAxiosError(error)) {
        throw new Error(`NeMo inference failed: ${error.message}`);
      }
      throw error;
    }
  }

  /**
   * サービスのヘルスチェック
   */
  async checkHealth(): Promise<boolean> {
    try {
      await this.client.post('/v1/health');
      return true;
    } catch {
      return false;
    }
  }
}

export default NeMoInferenceClient;
```

## ステップ 6: パフォーマンス最適化とモバイルオンデバイス推論

### オンデバイス推論（ONNX Runtime）

モバイルネットワークの遅延を避けるため、軽量モデルをデバイス上で直接実行する方法もあります。

```typescript
// libs/onnx-inference.ts
import { NativeModules, Platform } from 'react-native';

const { ONNXModule } = NativeModules;

interface ONNXInferenceConfig {
  modelPath: string;
  executionProviders?: string[];
}

class ONNXInferenceEngine {
  private initialized: boolean = false;

  async initialize(config: ONNXInferenceConfig): Promise<void> {
    if (Platform.OS !== 'android' && Platform.OS !== 'ios') {
      throw new Error('ONNX Runtime is only supported on Android and iOS');
    }

    try {
      await ONNXModule.initializeModel(config.modelPath);
      this.initialized = true;
    } catch (error) {
      throw new Error(`Failed to initialize ONNX model: ${error}`);
    }
  }

  async infer(input: number[][]): Promise<number[][]> {
    if (!this.initialized) {
      throw new Error('ONNX engine not initialized');
    }

    return ONNXModule.runInference(input);
  }

  async dispose(): Promise<void> {
    if (this.initialized) {
      await ONNXModule.dispose();
      this.initialized = false;
    }
  }
}

export default ONNXInferenceEngine;
```

### メモリ効率とバッテリー効率の考慮

```typescript
// hooks/useEdgeInferenceOptimization.ts
import { useEffect, useRef, useCallback } from 'react';
import { AppState, AppStateStatus } from 'react-native';
import NeMoInferenceClient from '../libs/nemo-inference-client';

interface OptimizationConfig {
  maxConcurrentRequests: number;
  batchSize: number;
  cacheResponses: boolean;
}

export const useEdgeInferenceOptimization = (
  nimUrl: string,
  config: OptimizationConfig
) => {
  const clientRef = useRef(new NeMoInferenceClient(nimUrl));
  const requestQueueRef = useRef<Array<() => Promise<any>>>([]);
  const cacheRef = useRef<Map<string, any>>(new Map());
  const appStateRef = useRef(AppState.currentState);

  useEffect(() => {
    const subscription = AppState.addEventListener('change', handleAppStateChange);
    return () => subscription.remove();
  }, []);

  const handleAppStateChange = useCallback((nextAppState: AppStateStatus) => {
    if (appStateRef.current.match(/inactive|background/) && nextAppState === 'active') {
      // アプリがフォアグラウンドに戻った時、推論キューを再開
      processQueue();
    }
    appStateRef.current = nextAppState;
  }, []);

  const processQueue = useCallback(async () => {
    const batch = requestQueueRef.current.splice(0, config.maxConcurrentRequests);
    await Promise.all(batch.map((fn) => fn()));
  }, [config.maxConcurrentRequests]);

  const queuedInference = useCallback(
    async (request: Parameters<typeof clientRef.current.generateText>[0]) => {
      const cacheKey = JSON.stringify(request);

      // キャッシュチェック
      if (config.cacheResponses && cacheRef.current.has(cacheKey)) {
        return cacheRef.current.get(cacheKey);
      }

      return new Promise((resolve, reject) => {
        requestQueueRef.current.push(async () => {
          try {
            const result = await clientRef.current.generateText(request);
            if (config.cacheResponses) {
              cacheRef.current.set(cacheKey, result);
            }
            resolve(result);
          } catch (error) {
            reject(error);
          }
        });

        if (requestQueueRef.current.length <= config.maxConcurrentRequests) {
          processQueue();
        }
      });
    },
    [config]
  );

  return { queuedInference };
};
```

## よくある質問（FAQ）

### NeMo でファインチューニングするのに必要なデータ量は？

NeMo では LoRA を使用することで、数百～数千サンプルからの効果的なファインチューニングが可能です。一般的には、3K～10K サンプルあれば顕著な精度向上を見込めます。ドメイン特化のコンテキストでは、1K サンプル程度でも有効です。

### エッジデバイスでの推論レイテンシをどう削減できる？

**（1）モデルサイズの最適化**: 軽量アーキテクチャ（Nemotron-ASR など）の選定
**（2）量子化**: TensorRT による INT8 量子化で 4 倍の推論高速化
**（3）キャッシング戦略**: 頻出クエリの結果をローカルに保存
**（4）バッチ処理**: 複数リクエストを一度に処理

### モバイルアプリでのプライバシー保護は？

エッジ推論により、テキストデータをサーバーに送信せず、デバイス上で処理できます。さらに、オンデバイス ONNX 推論により、ネットワーク通信を完全に排除できます。ただし、コンプライアンス要件に応じて、ローカルキャッシュの暗号化や定期的なデータ削除ポリシーを実装してください。
