## はじめに

Rork で自動生成されたアプリのプロトタイプは、すぐに公開できるレベルです。しかし **「もっとブランド色を出したい」「フォントやアイコンを統一したい」「レスポンシブ対応を微調整したい」**——こうした「洗練」のニーズは自然に生まれます。

本記事では、Rork 生成 UI を Figma に移行し、**ブランドアイデンティティを反映させた洗練された UI に仕上げるワークフロー**をご紹介します。デザインスキルが限定的でも実施できる、実践的な方法を中心に解説します。

## Rork と Figma の役割分担

Rork と Figma は、決して競合するツールではなく、**補完関係にあるツール**です。

### Rork の得意分野

- プロトタイプの高速生成（数時間で完成）
- ベースのレイアウト・機能配置の自動提案
- 複数ページの一貫性保証
- 反復的な AI 再生成（複数案の比較）

### Figma の得意分野

- ブランドカラーの統一的な適用
- フォント・アイコンセットの管理と一貫性確保
- 細かなスペーシング・パディングの調整
- チーム内でのデザインレビュー
- 開発者向けハンドオフ（デザイン仕様の記載）

## ステップ 1: Rork から Figma へのデザイン移行

### 方法 1: Rork から UI を PNG エクスポート

最もシンプルな方法は、Rork 内で生成した UI をそのまま PNG として出力し、Figma にインポートすることです。

1. **Rork で「デザインをエクスポート」を選択**
2. **PNG 形式で各ページをダウンロード**
3. **Figma で新規ファイルを作成、ファイルメニューから「インポート」**
4. **PNG をドラッグ&ドロップで挿入**

この方法の利点は、すぐに開始できることです。デメリットとしては、画像レイヤーになるため、**直接的なテキスト編集ができない**ことが挙げられます。

### 方法 2: Figma デザインファイルとしてエクスポート

Rork の一部バージョンでは、**デザインを Figma 互換形式（FDF）でエクスポート**できます。この場合：

1. **テキストレイヤー、シェイプレイヤーが編集可能**になる
2. **UI 要素を個別に選択・変更**できる
3. **デザインシステムと連携可能**

より高度なカスタマイズを目指す場合は、このオプションをお勧めします。

## ステップ 2: ブランドカラーの適用

Rork の AI は、デフォルトカラーパレットを使用します。これを**あなたのブランドカラーに統一**することが、洗練の第一歩です。

### カラーアセットの作成と体系化

Figma の左側パネルで「アセット」→「色」を選択し、体系的に色を登録します。単なる色の追加ではなく、**ブランドの色体系を数値化・管理化**することが重要です。

**推奨される色体系構成：**
- **プライマリカラー**（CTA ボタン、ハイライト用）：3段階（Dark / Default / Light）
- **セカンダリカラー**（サブアクション用）：3段階
- **グレースケール**（テキスト、背景）：6段階（#000 → #FFF）
- **ステータスカラー**（Success / Warning / Error）：各3段階
- **ニュートラルカラー**（中立的なアクセント）：2段階

**具体例：**
```
Primary:
- Primary / 900 (#1A2E4E)
- Primary / 500 (#2563EB)
- Primary / 100 (#E0E7FF)

Neutral:
- Neutral / 900 (#1F2937)
- Neutral / 500 (#6B7280)
- Neutral / 100 (#F3F4F6)

Status / Error:
- Error / 600 (#DC2626)
- Error / 400 (#F87171)
```

### カラーの一括置換と検証

Figma のプラグイン「Color Assistant」や「Replace Color」を使うと、既存の色をブランドカラーに一括置換できます：

1. **プラグインを起動**
2. **「古い色」と「新しい色」を指定**
3. **「置換」を実行**
4. **置換結果の確認**（視認性、アクセシビリティ基準への準拠など）

数十個の UI 要素でも、数秒で新しいカラーパレットに統一できます。

### アクセシビリティの検証

カラー置換後、必ず **WCAG 色コントラスト基準** をチェックします：
- **AAA標準**（推奨）：7:1以上のコントラスト比
- **AA標準**（最小限）：4.5:1以上のコントラスト比

Figmaのプラグイン「A11y – Color Contrast Checker」を使用すれば、全ての色の組み合わせを自動検証できます。

## ステップ 3: フォント・タイポグラフィの統一と微調整

Rork が使用するデフォルトフォントは、多くの場合 San Serif（特に Roboto や Inter）です。これを**あなたのブランドフォント**に変更し、視覚的一貫性を実現します。

### フォント体系の設計と実装

Figma の「アセット」セクションに、段階的なタイポグラフィスタイルを追加します。単なるフォント変更ではなく、**ブランドの印象を形成する重要な要素**として体系的に設計します：

**推奨されるフォント階層：**
- **Display（最大）**：48px, Weight 700, Line Height 120%（ページタイトル用）
- **見出し 1（H1）**：36px, Weight 700, Line Height 140%（セクション見出し）
- **見出し 2（H2）**：28px, Weight 600, Line Height 140%（サブセクション）
- **見出し 3（H3）**：24px, Weight 600, Line Height 140%（小見出し）
- **本文（Body）**：16px, Weight 400, Line Height 150%（メインテキスト）
- **Small Body**：14px, Weight 400, Line Height 150%（補足情報）
- **キャプション（Small）**：12px, Weight 500, Line Height 140%（ラベル、キャプション）
- **Micro**：11px, Weight 500, Line Height 140%（最小サイズ）

**ブランド別フォント選択ガイド：**
- **モダン・テック系**：Inter, Outfit, SF Pro Display
- **親しみやすい・カジュアル系**：Poppins, Nunito, Raleway
- **高級・プレミアム系**：Playfair Display（見出し）+ Lato（本文）
- **ミニマル系**：IBM Plex Sans, Source Sans Pro

### フォント置換と行間調整の実施

Figma のテキスト検索機能を使い、段階的にフォントを置換します：

1. **テキストレイヤーを「フォントファミリー」で絞り込み表示**
2. **複数レイヤーを同時選択**（Ctrl+click または Shift+click）
3. **右側パネルで新しいフォントを指定**
4. **Weight（太さ）と Line Height を順番に調整**

**詳細な微調整チェックポイント：**

| 要素 | 調整項目 | Rork デフォルト | 推奨値 |
|------|--------|---------|--------|
| H1見出し | Line Height | 120% | 140% |
| 本文 | Letter Spacing | 0px | +0.5px（長文向け） |
| ボタンテキスト | Line Height | 140% | 120%（コンパクト） |
| フォームラベル | Weight | 400 | 500（視認性向上） |

**重要な調整項目：**
- **行間（Line Height）の調整**：AI が自動生成したテキストは、多くの場合デフォルト値（110～120%）であり、読みやすさが損なわれることがあります。本文は150%、見出しは140%が目安です
- **文字間隔（Letter Spacing）の調整**：長文が多い場合、0.5px～1px の追加スペーシングで読みやすさが向上します
- **Weight のブレ修正**：AIが生成したテキストは、見出しと本文で Weight の差別化が曖昧なことがあります。見出しは600以上、本文は400 と明確に区別します

### フォントのロード確認

カスタムフォント（Google Fonts など）を使用する場合、以下の確認が必要です：
- **ウェイト（Weight）の全て**がロードされているか（Bold 700、Medium 500 など必要なウェイトを事前に指定）
- **日本語対応**しているか（特に Asian fonts の場合）
- **ロード時間**（重いフォントの場合、読み込み時間が増加）

## ステップ 4: アイコンセットの統一とコンポーネント化

Rork が使用するアイコンは、複数のセットが混在していることがあります。これを**統一されたアイコンセット**に置き換え、**Figmaコンポーネントとして管理**することが重要です。

### 推奨アイコンセット別の特性比較

各アイコンセットには異なる特徴があるため、ブランドや用途に応じた選択が必要です：

| セット | ストロール幅 | スタイル | 言語対応 | 推奨用途 |
|--------|-----------|--------|--------|---------|
| **Material Icons** | 24px基準 | フィル & アウトライン | 英数字中心 | エンタープライズ、汎用 |
| **Feather Icons** | 24px基準 | アウトラインのみ | 限定 | モダン、ミニマル系 |
| **SF Symbols** | 32px基準 | Apple標準 | 日本語対応 | iOS/macOS アプリ |
| **Heroicons** | 24px基準 | ミニマル | 限定 | Webアプリケーション |
| **Tabler Icons** | 24px基準 | カスタム度高 | 限定 | SaaS プロダクト |

### アイコン置換のワークフロー（段階的アプローチ）

**Step 1：アイコンセットの選定と準備**
1. **ブランドイメージに合うアイコンセットを選定**（上記の比較表を参照）
2. **Figma Community から選んだアイコンセットをコンポーネントライブラリとしてインポート**
   ```
   Figma メニュー → Assets → リンクされたライブラリから検索
   ```
3. **ライブラリの「ファイルを編集」を選択し、全アイコンを確認**

**Step 2：既存アイコンの分類と調査**
1. **Rork が使用している全アイコンをスクリーンショット化**
2. **使用されているアイコンを一覧にまとめ、新セットでの対応アイコンを確認**
3. **代替アイコンがない場合の対応案を決定**（カスタム作成 or 機能の削減）

**Step 3：段階的な置換実施**
1. **単一の画面から開始し、置換テスト**
2. **色・サイズが正しく反映されるか確認**
3. **問題がなければ他の画面に展開**

**Step 4：Figma コンポーネント化**

Figma では「コンポーネント置換」機能により、複数のアイコンを一度に新しいセットに置き換えられます：

```
対象アイコンを複数選択
→ 右クリック
→ 「コンポーネントに置き換える」
→ 新しいセットのコンポーネントを選択
```

### カスタムアイコンの作成ガイド

新セットに完全に対応していないアイコンが存在する場合、カスタム作成が必要です：

**カスタムアイコン作成のチェックリスト：**
- [ ] ストローク幅を統一（推奨：2px 前後）
- [ ] 推奨サイズを設定（24px, 32px など）
- [ ] 複数サイズでの視認性テスト（16px, 24px, 32px）
- [ ] ダークモード・ライトモード対応を確認
- [ ] カラーバリエーション（複数カラー対応版）を作成
- [ ] コンポーネント化して再利用可能にする

### アイコンのサイズ・カラー統一

すべてのアイコンが揃ったら、以下を確認します：

**統一すべき要素：**
- **ストローク幅**：2px で統一
- **基本サイズ**：24px または 32px
- **カラー**：ブランドカラーの使用色に限定
- **パディング**：アイコンの周囲の余白を一定に

Figma のコンポーネント機能を活用すれば、複数サイズ・複数カラーを一元管理できます：

```
メインコンポーネント「Icon / Home / 24 / Outline」
→ バリアント追加「Size: 24, 32, 48」
→ バリアント追加「Color: Primary, Secondary, Gray」
```

## ステップ 5: レスポンシブ対応の調整

Rork が生成した UI は、基本的にはレスポンシブ対応ですが、細かな最適化が必要なケースがあります。

### 複数デバイスサイズでの確認

Figma でフレームを複製し、異なるサイズを作成します：

- **iPhone SE（375 × 667 px）**
- **iPhone 13（390 × 844 px）**
- **iPad（768 × 1024 px）**

各サイズで、以下をチェック：

- **テキストが読みやすいか**（特に小さいデバイス）
- **ボタンが十分な大きさ（推奨 44px × 44px）**
- **画像の縦横比が保たれているか**

### Figma のコンストレイント機能

Figma の「コンストレイント」を活用すると、親フレームのサイズ変更に自動追従するレイアウトが実現できます：

1. **レイヤーを選択**
2. **右側パネルの「コンストレイント」セクションで設定**
3. **「水平：左に固定」「垂直：伸長」など、振る舞いを定義**

この設定により、異なるデバイスサイズに自動的に対応します。

## ステップ 6: コンポーネント化による一貫性確保と再利用性向上

Figma の最大の強みは、**コンポーネント機能**です。Rork で自動生成された UI 要素をコンポーネント化することで、保守性と拡張性が劇的に向上します。

### コンポーネント化の利点と戦略

**直接的な利点：**
- **一度定義すれば、全インスタンスに自動反映**
- **デザイン変更時の作業量が劇的に削減**（10画面の修正が1回で完了）
- **デザイナーと開発者間での認識ズレを防止**（仕様書となる）

**長期的な利点：**
- **新機能追加時の設計スピード向上**（既存コンポーネント再利用）
- **チームメンバーのオンボーディング加速**（「このボタンを使う」で統一）
- **デザインシステムへの発展**（複数プロジェクト間での共有）

### 実装例：ボタンコンポーネントの詳細構築

**段階 1：メインコンポーネントの定義**

```
ボタン / Primary / Large（メインコンポーネント）
├─ 背景：Primary Color
├─ テキスト：White, 16px, Weight 600
├─ パディング：12px 24px
├─ コーナー半径：8px
└─ シャドウ：Drop Shadow (0, 4px, 8px, 20% opacity)
```

**段階 2：バリアント（状態別）の追加**

```
メインコンポーネント「ボタン」
├─ Variant 1：Size（Large, Medium, Small）
├─ Variant 2：Type（Primary, Secondary, Ghost）
├─ Variant 3：State（Default, Hover, Pressed, Disabled）
└─ Variant 4：Icon（With Icon / Without Icon）
```

**段階 3：全ページへの適用**

1. **既存のボタン要素（テキスト or 画像）を全て選択**
2. **右クリック → 「このコンポーネントに置き換える」**
3. **必要に応じてバリアント（Size、Type）を選択**

メインコンポーネントを編集すれば、全インスタンスが自動更新されます。

### コンポーネント階層の設計

大規模 UI には、段階的なコンポーネント構成が効果的です：

**基本コンポーネント層：**
- Button, Input, Checkbox, Radio, Toggle
- Icon, Avatar, Badge
- Typography (H1, H2, Body など)

**複合コンポーネント層（基本の組み合わせ）：**
- Form Group (Label + Input + Error Message)
- Card (Header + Content + Footer)
- Modal (Title + Content + Actions)
- Navigation Bar

**ページレベルのパターン：**
- ログイン画面テンプレート
- 商品詳細ページテンプレート
- ダッシュボードレイアウト

### Rork 生成 UI のコンポーネント化チェックリスト

```
□ 単一の「ボタン」パターンを特定
□ 色・サイズのバリエーションを列挙
  - Primary / Secondary / Ghost
  - Large / Medium / Small
  - Default / Hover / Disabled
□ メインコンポーネントを作成
□ 全バリアントを実装
□ 全ページのボタンを置き換え
□ インスタンスの整合性を確認（色ズレ、サイズ違い）
□ 他の UI 要素（フォーム、カード等）も同様に実施
□ コンポーネントライブラリを整理・命名統一
```

## ステップ 7: Rork との反復ワークフロー

Figma での洗練が完了しても、さらに機能追加や大幅な UI 再設計が必要な場合があります。その際、Rork に戻すプロセスを想定しましょう。

### 推奨ワークフロー

```
Rork で初期案を生成
   ↓
Figma に移行・洗練（色・フォント・アイコン）
   ↓
実装チームにハンドオフ
   ↓
ユーザーフィードバック収集
   ↓
[大幅改善が必要な場合]
Rork で新案を再生成
   ↓
新案 + Figma での洗練結果を比較
   ↓
ベストなバージョンを統合
```

このサイクルを回すことで、**AI の生成スピードと人間のデザイン洗練力**が最大限に活かされます。

## デザインシステムの構築と段階的なスケーリング

複数プロジェクトで再利用できるレベルに達したら、**デザインシステム**として Figma で管理することで、組織全体の設計生産性が飛躍的に向上します。

### デザインシステムの体系的な構成

Rork で生成した UI を基盤に、以下の要素を段階的に体系化していきます：

**Phase 1：基盤層（最初の1-2プロジェクト後に構築）**
- カラーパレット（Primary / Secondary / Neutral / Status）
- タイポグラフィスタイル（Display / H1 / H2 / Body / Caption など）
- 基本コンポーネント（Button / Input / Checkbox など）

**Phase 2：発展層（3-4プロジェクト後に拡張）**
- アイコンセット（コンポーネント化、複数サイズ・色対応）
- 複合コンポーネント（Form Group / Card / Modal など）
- レイアウトパターン（Grid / Sidebar Layout など）

**Phase 3：成熟層（継続使用の中で継続的に更新）**
- ページテンプレート（ダッシュボード / リスト / 詳細表示 など）
- 画面パターン（ログイン / エラー / ローディング など）
- アニメーション・マイクロインタラクション仕様

### Figma ライブラリの構築と運用

Figma の「ライブラリ」機能により、複数ファイル間での自動同期が実現できます：

**セットアップ手順：**
1. **メインのシステムファイルを作成**
   - 名前：「Design System v1.0」など
   - 構成：Components タブ、Tokens セクション
2. **コンポーネントを全て「メインコンポーネント」として定義**
3. **「Assets」パネル → 右上のメニュー → 「ライブラリとして保存」**
4. **他のプロジェクトファイルで「リンクされたライブラリ」として接続**

**運用のポイント：**
- **バージョン管理**：破壊的な変更（コンポーネント削除など）は新バージョン（v2.0）で実施
- **段階的な更新**：既存プロジェクトに影響を与えないよう、新機能は別セクションで追加
- **ドキュメント化**：各コンポーネントに「使用ガイド」を記載（description フィールド活用）

### デザイントークンの導入

色・サイズ・スペーシング などを**トークン化**し、実装チームへの引き継ぎを円滑化します：

**トークン定義例：**
```json
{
  "colors": {
    "primary": {
      "900": "#1A2E4E",
      "600": "#2563EB",
      "100": "#E0E7FF"
    }
  },
  "spacing": {
    "xs": "4px",
    "sm": "8px",
    "md": "16px",
    "lg": "24px"
  },
  "typography": {
    "body": {
      "fontSize": "16px",
      "fontWeight": "400",
      "lineHeight": "1.5"
    }
  }
}
```

Figma の「Variables」機能（2024年以降のプランで利用可能）を使用すれば、これらのトークンを Figma 内で管理でき、JSON 出力も可能です。

## 実装チームとの連携プロセス

Figma での洗練が完了したら、開発チームへの効果的なハンドオフが重要です。

### Dev Mode による仕様書の自動生成

Figma の「Dev Mode」を活用すれば、開発に必要な全ての情報を自動抽出できます：

**Dev Mode で提供される情報：**
- コンポーネントのサイズ・距離（自動測定）
- 色コード（RGB / HEX / HSL）
- フォント情報（ファミリー / サイズ / ウェイト / 行間）
- ベクター効果（シャドウ / グラデーション）
- CSS クラス名の提案

**実施方法：**
1. Figma ファイルを開き、右上の「Dev」タブを選択
2. 対象コンポーネントをクリック
3. 右側パネルに自動抽出された仕様が表示

### 実装チーム向けドキュメントの準備

```markdown
# UI/UX 実装ガイド

## カラーパレット
- Primary: #2563EB
- Secondary: #10B981
- Neutral-700: #374151

## タイポグラフィ
- H1: Inter 36px / 700 / Line-height 1.4
- Body: Inter 16px / 400 / Line-height 1.5

## コンポーネント使用ルール
- Button は必ず "ボタン / Primary / Large" コンポーネントを使用
- Input は "Form / Input" コンポーネントを使用
- アイコンは Material Icons 24px で統一

## スペーシング規則
- マージン / パディング：8px 単位で統一
- グリッド：24px ベース
```

## まとめと次のステップ

Rork × Figma の組み合わせにより、**AI の効率性と人間のクリエイティブさ**を融合させたアプリ開発が実現します。

このワークフローを実施することで、以下が達成できます：

**チーム運用面での成果：**
- AI 生成のスピード：初期プロトタイプ完成が 数時間で実現
- 品質・一貫性：ブランドアイデンティティが全画面に統一
- チーム連携：デザイナーと実装者の意思疎通が円滑化

**ビジネス面での成果：**
- 開発期間の短縮（仕様検討の時間削減）
- デザイン修正の効率化（コンポーネント一元管理）
- 拡張性の確保（新機能追加時の実装速度向上）

次のステップとしては：

1. **実装チーム向けに、Figma での仕様書を充実させる**（色番号、フォントサイズ、コンポーネント名など）
2. **定期的なデザインレビュー会を開催し、改善案を集める**（2週間ごと、15分程度）
3. **ユーザーテストを実施し、UI/UX の効果を検証する**（簡易なアンケート、ユーザービデオインタビュー）
4. **デザインシステムを段階的に拡張**（プロジェクト完了後、学習を反映）

Rork で生成したプロトタイプは、あくまで「始まり」です。本記事で解説した 色・フォント・アイコン・コンポーネント化 といった段階的な洗練を通じて、初めて**本当にあなたのブランドを体現したアプリ**が完成します。

ぜひこのワークフローをチームに導入し、素晴らしいアプリの実現にお役立てください。何度も繰り返すことで、プロセスがさらに洗練され、次のプロジェクトはより迅速に高品質なデザインが実現できるようになるでしょう。
