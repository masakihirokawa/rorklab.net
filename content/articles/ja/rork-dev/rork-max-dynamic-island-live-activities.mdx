## Dynamic Island・Live Activities とは？Rork Maxで実現できる最先端iOS機能

iPhone 14 Pro以降に搭載された**Dynamic Island**は、フロントカメラとセンサー周辺の切り欠き部分を活用した独自UIです。アプリの状態や進行中の情報をリアルタイムでインタラクティブに表示でき、ユーザーエクスペリエンスを大幅に向上させます。

**Live Activities**はこのDynamic Islandおよびロック画面に常に最新情報を表示し続ける仕組みです。スポーツのスコア、デリバリーの配送状況、フライトの遅延情報など、リアルタイムで変化するデータを表示するのに最適です。

従来、これらの機能を実装するためにはSwift/SwiftUIの深い知識とAppleのActivityKit APIの理解が必要でした。しかし**Rork Max**を使えば、プロンプトを書くだけでこれらの高度なiOS機能を自分のアプリに組み込むことができます。

この記事では、Rork Maxを使ってDynamic IslandとLive Activitiesを実装する方法を、実際のコード例とともにステップバイステップで解説します。

## 前提知識・環境準備

実装を始める前に、以下の点を確認してください。

**必要な環境:**
- **Rork Maxプラン**（Dynamic IslandはネイティブiOS機能のためRork Maxが必要）
- iPhone 14 Pro以降のデバイス（Dynamic Island対応機種）またはXcode Simulator（iOS 16.1以降）
- Appleデベロッパーアカウント（Live Activities / Push Token取得のため）

**対応iOSバージョン:**
- Dynamic Island: iOS 16.1以降
- Live Activities: iOS 16.1以降（ロック画面のみ）、iOS 16.2以降（Dynamic Island対応）

Rork Maxは通常のRorkとは異なり、React NativeではなくネイティブのSwift/SwiftUIコードを生成します。そのためDynamic IslandやLive ActivitiesといったApple専用のAPIにフルアクセスできます。詳しくはこちらの記事も参照してください：[Rork Max Apple マルチプラットフォーム対応ガイド](/articles/rork-dev/rork-max-apple-multiplatform)

## Live Activities の仕組みを理解する

実装に入る前に、Live Activitiesのアーキテクチャを理解しておきましょう。

Live Activitiesは**ActivityKit**というフレームワークを通じて動作します。主要なコンポーネントは以下の通りです。

```swift
// ActivityKit の基本構造
import ActivityKit
import SwiftUI

// 1. アクティビティの状態を定義するデータ構造
struct DeliveryAttributes: ActivityAttributes {
    // 静的データ（アクティビティ開始時に設定・変更不可）
    public struct ContentState: Codable, Hashable {
        var currentStep: String     // 例: "配達中"
        var estimatedMinutes: Int   // 例: 15
        var driverName: String      // 例: "田中さん"
    }

    // 動的データ（更新可能）
    var orderId: String
    var restaurantName: String
}

// 2. Live Activity を開始する
func startDeliveryActivity(orderId: String, restaurant: String) {
    let attributes = DeliveryAttributes(
        orderId: orderId,
        restaurantName: restaurant
    )

    let initialState = DeliveryAttributes.ContentState(
        currentStep: "準備中",
        estimatedMinutes: 30,
        driverName: "未割当"
    )

    // ActivityKit でアクティビティを作成
    do {
        let activity = try Activity.request(
            attributes: attributes,
            contentState: initialState,
            pushType: .token  // Push通知で更新する場合
        )
        print("Live Activity 開始: \(activity.id)")
        // 出力例: Live Activity 開始: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
    } catch {
        print("エラー: \(error.localizedDescription)")
    }
}
```

このコードをRork Maxに生成させることで、デリバリーアプリのLive Activityを実装できます。

## Rork Max でのプロンプト作成：Dynamic Island UIの設計

Rork Maxでは、自然言語でUIを指定するだけでSwiftUIコードが生成されます。以下はDynamic Islandに対応したLive Activityウィジェットを作成するためのプロンプト例です。

**プロンプト例（デリバリーアプリ）:**

> 「フードデリバリーアプリに Live Activity を追加してください。Dynamic Islandのコンパクト表示では左に残り時間（数字＋分）、右に配達員のアイコンを表示。展開表示では注文ステータス（準備中→ピックアップ→配達中→完了）のプログレスバーと推定到着時刻を表示してください。ロック画面にも同じ情報を大きく表示してください。」

Rork Maxがこのプロンプトから生成するUIコードの概要：

```swift
// Rork Max が生成するLive Activity ウィジェット（概略）
import ActivityKit
import SwiftUI
import WidgetKit

struct DeliveryLiveActivityView: Widget {
    var body: some WidgetConfiguration {
        ActivityConfiguration(for: DeliveryAttributes.self) { context in
            // ロック画面 / 通知バナー の表示
            LockScreenLiveActivityView(context: context)
        } dynamicIsland: { context in
            DynamicIsland {
                // 展開時の表示（ユーザーがタップして展開）
                DynamicIslandExpandedRegion(.leading) {
                    Label(
                        "\(context.state.estimatedMinutes)分",
                        systemImage: "clock.fill"
                    )
                    .font(.title2.bold())
                    .foregroundStyle(.orange)
                }
                DynamicIslandExpandedRegion(.trailing) {
                    Image(systemName: "person.crop.circle.fill")
                        .font(.title2)
                        .foregroundStyle(.blue)
                }
                DynamicIslandExpandedRegion(.bottom) {
                    DeliveryProgressBar(
                        step: context.state.currentStep
                    )
                }
            } compactLeading: {
                // コンパクト表示・左側
                Label("\(context.state.estimatedMinutes)m", systemImage: "timer")
                    .font(.caption.bold())
            } compactTrailing: {
                // コンパクト表示・右側
                Image(systemName: "bicycle")
                    .foregroundStyle(.orange)
            } minimal: {
                // 最小表示（他のアプリのアクティビティと競合時）
                Image(systemName: "timer")
                    .foregroundStyle(.orange)
            }
        }
    }
}
```

## ステップバイステップ：フードデリバリーアプリへの実装

それでは、実際にRork Maxを使って実装する手順を見ていきましょう。

### ステップ 1: プロジェクトの作成とターゲット追加

Rork MaxでiOSアプリプロジェクトを作成後、Live Activitiesのためにウィジェット拡張ターゲットを追加します。Rork Maxに以下のようにプロンプトを入力してください。

> 「このプロジェクトに Live Activity 対応のWidget Extension を追加して、Info.plistにNSSupportsLiveActivitiesを追加してください。」

Rork Maxが自動的に:
- `DeliveryWidgetExtension` ターゲットの追加
- `Info.plist` への `NSSupportsLiveActivities: true` 設定
- ActivityKit のインポートと基本構造の生成

を行います。

### ステップ 2: データモデルの定義

Rork Maxへのプロンプト：

> 「デリバリーの配送ステータスを管理するActivityAttributesを作成してください。静的データは注文IDとレストラン名。動的データ（ContentState）は配送ステータス（String）、推定到着時間（Int、分単位）、配達員名（String）にしてください。」

### ステップ 3: Live Activity の開始・更新・終了

```swift
// Rork Max が生成する LiveActivityManager
class LiveActivityManager: ObservableObject {
    private var currentActivity: Activity<DeliveryAttributes>?

    // アクティビティ開始
    func start(orderId: String, restaurant: String) async {
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            print("Live Activities が無効です")
            return
        }

        let attributes = DeliveryAttributes(
            orderId: orderId,
            restaurantName: restaurant
        )
        let state = DeliveryAttributes.ContentState(
            currentStep: "準備中",
            estimatedMinutes: 35,
            driverName: "未割当"
        )

        currentActivity = try? Activity.request(
            attributes: attributes,
            contentState: state,
            pushType: .token
        )
    }

    // ステータス更新（Push通知またはアプリから）
    func update(step: String, minutes: Int, driver: String) async {
        let updatedState = DeliveryAttributes.ContentState(
            currentStep: step,
            estimatedMinutes: minutes,
            driverName: driver
        )

        await currentActivity?.update(using: updatedState)
        // 期待する動作: Dynamic Island と ロック画面が即時更新される
    }

    // アクティビティ終了（配達完了時）
    func end() async {
        let finalState = DeliveryAttributes.ContentState(
            currentStep: "配達完了！",
            estimatedMinutes: 0,
            driverName: currentActivity?.contentState.driverName ?? ""
        )

        await currentActivity?.end(using: finalState, dismissalPolicy: .after(.now + 30))
        // 期待する動作: 30秒後にDynamic IslandとLock Screenから消える
    }
}
```

## よくあるエラーと対処法

**エラー 1: Live Activityが表示されない**

原因: `Info.plist` に `NSSupportsLiveActivities` が設定されていない、またはユーザーが設定でLive Activitiesをオフにしている。

```swift
// 事前にLive Activitiesが有効か確認する
if ActivityAuthorizationInfo().areActivitiesEnabled {
    // Live Activityを開始
} else {
    // ユーザーに設定を案内するUIを表示
    print("設定 > [アプリ名] > Live Activities を有効にしてください")
}
```

**エラー 2: Dynamic Islandが展開しない**

原因: Dynamic IslandはiPhone 14 Pro以降専用。iPhone 14やそれ以前の機種ではロック画面とバナーのみに表示される。シミュレーターで確認する場合はiPhone 14 Pro以降を選択すること。

**エラー 3: `Activity.request` が失敗する**

原因: 同時に起動できるLive Activitiesは最大5件まで。既存のアクティビティを終了してから新しいものを開始する。

## Apple Watch への連携（応用編）

Rork MaxはApple Watchアプリも生成できます。Live Activitiesの情報をWatchComplicationに同期させることで、手首でも配達状況を確認できます。

詳細は：[Rork Max Apple Watch アプリ開発ガイド](/articles/rork-dev/rork-max-apple-watch-app)

```swift
// Watch Connectivity でLive Activityデータを共有
import WatchConnectivity

class WatchSyncManager: NSObject, WCSessionDelegate {
    func sendDeliveryUpdate(step: String, minutes: Int) {
        guard WCSession.default.isPaired && WCSession.default.isReachable else { return }

        WCSession.default.sendMessage(
            ["step": step, "minutes": minutes],
            replyHandler: nil
        )
        // Apple Watch のComplicationが自動更新される
    }
}
```

## FAQ

**Q: Rork（無料プラン）でもDynamic Islandは実装できますか？**

A: いいえ、Dynamic IslandはネイティブiOS APIであり、Rork MaxのSwift/SwiftUIコード生成が必要です。無料のRorkはReact Nativeベースのため、WidgetExtensionやActivityKitにアクセスできません。Rork Maxプランへのアップグレードが必要です。

**Q: Live Activitiesはバックグラウンドでも動作しますか？**

A: はい、Live Activitiesはアプリがバックグラウンドまたは終了していても動作します。更新にはPush通知（APNs）を使用し、サーバーから直接ActivityKitのプッシュトークンにリクエストを送ります。1時間に最大最大15回程度の更新が推奨されます（Appleのレート制限あり）。

**Q: Android版でも同様のリアルタイム表示機能を実装できますか？**

A: Androidには「Dynamic Island」相当の機能はありませんが、**Notification で進行中の通知**（Ongoing Notification）を使って類似の体験を提供できます。Rork Maxで生成したiOSアプリと並行して、RorkのReact Native版でAndroidのForeground Serviceを実装する方法があります。

**Q: Rork MaxのAIはActivityKitの最新APIに対応していますか？**

A: はい、Rork MaxはiOS 18以降のActivityKit APIにも対応しています。ただし特定の新機能については、プロンプトに「iOS 18のActivityKit APIを使用して」と明記することで、より正確なコードが生成されます。

**Q: Live Activitiesをテストするのに実機が必要ですか？**

A: iOS 16.2以降のSimulatorでもDynamic IslandをテストできますがiPhone 14 Pro以降のモデルを選択する必要があります。Push通知によるリモート更新は実機でのみテスト可能です。

Rork MaxのDynamic Island・Live Activitiesサポートを活用することで、従来は熟練のSwift開発者にしか実装できなかった高度なiOS機能が、誰でも簡単に実装できるようになりました。

今回解説した内容をまとめると：
- ActivityAttributesでデータモデルを定義し、静的データと動的データを分ける

- Push通知を使ったリアルタイム更新でサーバーからの状態変更を反映できる
- Apple Watchとの連携でマルチデバイス体験をさらに高められる

Dynamic IslandとLive Activitiesは、デリバリー・フィットネス・交通・決済など多くのジャンルのアプリに応用できます。Rork Maxを使って、ユーザーに喜ばれる差別化機能を今すぐ実装してみましょう。
