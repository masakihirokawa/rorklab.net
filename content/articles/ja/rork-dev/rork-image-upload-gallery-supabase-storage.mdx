## はじめに — モバイルアプリに画像機能が欠かせない理由

モバイルアプリを開発するとき、画像のアップロードと表示は最も頻繁に必要になる機能の一つです。SNS、ECサイト、タスク管理アプリ、ポートフォリオアプリなど、ほぼすべてのジャンルで画像を扱う場面が生まれます。

Rork を使えば、AI の力でアプリの基本構造をすばやく構築できますが、画像ストレージの設計は自分で理解しておくことが重要です。適切な設計を行わないと、アプリの表示速度低下やストレージコストの膨張、プライバシーの問題につながります。

この記事では、Rork で構築するモバイルアプリに **Supabase Storage** を組み合わせて、画像のアップロード・管理・ギャラリー表示までを一気通貫で実装する方法を解説します。Expo の `expo-image-picker` で端末の写真を取得し、Supabase Storage にアップロードし、最適化されたサムネイルをギャラリーで表示する、実用的なパターンを身につけましょう。

**この記事で学べること:**

- Supabase Storage のバケット設定とセキュリティポリシー
- `expo-image-picker` を使った画像選択とカメラ撮影
- Supabase Storage への画像アップロード実装
- FlatList を使ったパフォーマンスの高いギャラリー表示
- 画像のリサイズとキャッシュ戦略

**前提知識:** Rork の基本操作（プロジェクト作成・プレビュー）、React Native の基礎（コンポーネント・useState）。Supabase のアカウントが必要です。

## Supabase Storage の基本とバケット設定

Supabase Storage は、S3 互換のオブジェクトストレージで、画像・動画・ドキュメントなどのファイルを管理できます。無料プランでも 1GB のストレージが使えるため、個人開発のスタートには十分です。

### バケットの作成

Supabase ダッシュボードにログインし、左メニューから「Storage」を選択します。「New bucket」をクリックして、以下のように設定します。

- **Bucket name**: `app-images`
- **Public bucket**: OFF（RLS で制御するため）
- **File size limit**: 5MB（モバイル画像に適切なサイズ）
- **Allowed MIME types**: `image/jpeg, image/png, image/webp`

### Row Level Security（RLS）ポリシー

バケットを作成したら、セキュリティポリシーを設定します。以下の SQL を Supabase の SQL Editor で実行してください。

```sql
-- 認証済みユーザーが自分のフォルダにアップロードできるポリシー
CREATE POLICY "Users can upload to own folder"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- 認証済みユーザーが自分の画像を読み取れるポリシー
CREATE POLICY "Users can view own images"
ON storage.objects FOR SELECT
TO authenticated
USING (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- 認証済みユーザーが自分の画像を削除できるポリシー
CREATE POLICY "Users can delete own images"
ON storage.objects FOR DELETE
TO authenticated
USING (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);
```

このポリシーでは、各ユーザーが `{user_id}/` というフォルダ配下にのみファイルを保存・閲覧・削除できるようにしています。他のユーザーの画像にはアクセスできないため、プライバシーが保護されます。

## expo-image-picker で画像を選択する

Rork プロジェクトで `expo-image-picker` を使うと、端末のフォトライブラリやカメラから画像を取得できます。

### インストールと権限設定

Rork のプロジェクトで依存パッケージを追加します。

```bash
# Rork のターミナルまたはプロンプトで指定
npx expo install expo-image-picker expo-file-system
```

`app.json` にカメラとフォトライブラリの権限を追加します。

```json
{
  "expo": {
    "plugins": [
      [
        "expo-image-picker",
        {
          "photosPermission": "このアプリは写真を選択するためにフォトライブラリへのアクセスを必要とします。",
          "cameraPermission": "このアプリは写真を撮影するためにカメラへのアクセスを必要とします。"
        }
      ]
    ]
  }
}
```

### 画像選択コンポーネントの実装

以下のコードで、フォトライブラリからの選択とカメラ撮影の両方に対応するコンポーネントを作成します。

```typescript
// components/ImagePickerButton.tsx
import { useState } from 'react';
import { View, TouchableOpacity, Text, Alert, StyleSheet } from 'react-native';
import * as ImagePicker from 'expo-image-picker';

interface ImagePickerButtonProps {
  onImageSelected: (uri: string) => void;
}

export default function ImagePickerButton({ onImageSelected }: ImagePickerButtonProps) {
  const [loading, setLoading] = useState(false);

  // フォトライブラリから画像を選択
  const pickFromLibrary = async () => {
    const permission = await ImagePicker.requestMediaLibraryPermissionsAsync();
    if (!permission.granted) {
      Alert.alert('権限エラー', 'フォトライブラリへのアクセスが許可されていません。');
      return;
    }

    setLoading(true);
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,        // トリミングUIを表示
      aspect: [1, 1],             // 正方形にクロップ
      quality: 0.8,               // 画質80%で容量を抑制
    });

    setLoading(false);
    if (!result.canceled && result.assets[0]) {
      onImageSelected(result.assets[0].uri);
    }
  };

  // カメラで撮影
  const takePhoto = async () => {
    const permission = await ImagePicker.requestCameraPermissionsAsync();
    if (!permission.granted) {
      Alert.alert('権限エラー', 'カメラへのアクセスが許可されていません。');
      return;
    }

    setLoading(true);
    const result = await ImagePicker.launchCameraAsync({
      allowsEditing: true,
      aspect: [1, 1],
      quality: 0.8,
    });

    setLoading(false);
    if (!result.canceled && result.assets[0]) {
      onImageSelected(result.assets[0].uri);
    }
  };

  return (
    <View style={styles.container}>
      <TouchableOpacity style={styles.button} onPress={pickFromLibrary} disabled={loading}>
        <Text style={styles.buttonText}>ライブラリから選択</Text>
      </TouchableOpacity>
      <TouchableOpacity style={styles.button} onPress={takePhoto} disabled={loading}>
        <Text style={styles.buttonText}>カメラで撮影</Text>
      </TouchableOpacity>
    </View>
  );
}

// 期待される動作:
// - 「ライブラリから選択」をタップ → フォトライブラリが開き、画像を選択・トリミング可能
// - 「カメラで撮影」をタップ → カメラが起動し、撮影後にトリミング可能
// - 選択完了後、onImageSelected コールバックに画像のローカルURIが渡される

const styles = StyleSheet.create({
  container: { flexDirection: 'row', gap: 12, marginVertical: 16 },
  button: {
    flex: 1,
    backgroundColor: '#4F46E5',
    paddingVertical: 14,
    borderRadius: 10,
    alignItems: 'center',
  },
  buttonText: { color: '#fff', fontWeight: '600', fontSize: 15 },
});
```

`quality: 0.8` と `aspect: [1, 1]` を指定することで、画像サイズを抑えつつ統一感のあるギャラリーを実現できます。

## Supabase Storage への画像アップロード

選択した画像を Supabase Storage にアップロードするユーティリティ関数を作成します。

```typescript
// utils/uploadImage.ts
import * as FileSystem from 'expo-file-system';
import { supabase } from '@/lib/supabase';
import { decode } from 'base64-arraybuffer';

/**
 * 画像をSupabase Storageにアップロードする
 * @param uri - 画像のローカルURI
 * @param userId - ユーザーID（フォルダ分離用）
 * @returns アップロードされた画像のパス
 */
export async function uploadImage(
  uri: string,
  userId: string
): Promise<string> {
  // 1. ローカルファイルをBase64に変換
  const base64 = await FileSystem.readAsStringAsync(uri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // 2. ファイル名を生成（タイムスタンプ + ランダム文字列で一意性を確保）
  const timestamp = Date.now();
  const random = Math.random().toString(36).substring(2, 8);
  const fileName = `${userId}/${timestamp}-${random}.jpg`;

  // 3. Supabase Storage にアップロード
  const { data, error } = await supabase.storage
    .from('app-images')
    .upload(fileName, decode(base64), {
      contentType: 'image/jpeg',
      cacheControl: '3600',      // 1時間のキャッシュ
      upsert: false,             // 同名ファイルの上書きを禁止
    });

  if (error) {
    throw new Error(`アップロード失敗: ${error.message}`);
  }

  return data.path;
}

/**
 * 画像の公開URLを取得する
 * @param path - Storage内のファイルパス
 * @returns 署名付きURL（1時間有効）
 */
export async function getImageUrl(path: string): Promise<string> {
  const { data, error } = await supabase.storage
    .from('app-images')
    .createSignedUrl(path, 3600); // 1時間有効の署名付きURL

  if (error) {
    throw new Error(`URL生成失敗: ${error.message}`);
  }

  return data.signedUrl;
}

// 使用例:
// const path = await uploadImage(selectedUri, user.id);
// const url = await getImageUrl(path);
// → url を <Image source={{ uri: url }} /> に渡して表示
```

ここでのポイントは3つあります。第一に、`base64-arraybuffer` ライブラリの `decode` 関数を使い、Base64 文字列を ArrayBuffer に変換している点です。React Native 環境では `Blob` の直接作成に制限があるため、この変換が必要になります。第二に、ファイル名にタイムスタンプとランダム文字列を組み合わせることで、同じユーザーが短時間に複数の画像をアップロードしても名前が衝突しません。第三に、`createSignedUrl` で有効期限付きの URL を生成することで、バケットを非公開に保ちながら画像を表示できます。

## FlatList を使った高パフォーマンスなギャラリー表示

アップロードした画像をグリッド形式で表示するギャラリーコンポーネントを実装します。React Native の `FlatList` を使うことで、大量の画像でもスムーズなスクロールが可能です。

```typescript
// components/ImageGallery.tsx
import { useState, useEffect, useCallback } from 'react';
import {
  View, Image, FlatList, Dimensions,
  TouchableOpacity, ActivityIndicator, StyleSheet,
} from 'react-native';
import { supabase } from '@/lib/supabase';
import { getImageUrl } from '@/utils/uploadImage';

interface GalleryImage {
  id: string;
  path: string;
  url: string;
  createdAt: string;
}

interface ImageGalleryProps {
  userId: string;
  refreshTrigger?: number;    // 新しい画像追加時にインクリメント
  onImagePress?: (image: GalleryImage) => void;
}

const SCREEN_WIDTH = Dimensions.get('window').width;
const NUM_COLUMNS = 3;
const ITEM_SPACING = 2;
const ITEM_SIZE = (SCREEN_WIDTH - ITEM_SPACING * (NUM_COLUMNS + 1)) / NUM_COLUMNS;

export default function ImageGallery({
  userId,
  refreshTrigger,
  onImagePress,
}: ImageGalleryProps) {
  const [images, setImages] = useState<GalleryImage[]>([]);
  const [loading, setLoading] = useState(true);

  const fetchImages = useCallback(async () => {
    setLoading(true);

    // ユーザーのフォルダ内のファイル一覧を取得
    const { data: files, error } = await supabase.storage
      .from('app-images')
      .list(userId, {
        limit: 100,
        offset: 0,
        sortBy: { column: 'created_at', order: 'desc' },
      });

    if (error || !files) {
      setLoading(false);
      return;
    }

    // 各ファイルの署名付きURLを取得
    const imagePromises = files
      .filter((file) => file.name !== '.emptyFolderPlaceholder')
      .map(async (file) => {
        const path = `${userId}/${file.name}`;
        const url = await getImageUrl(path);
        return {
          id: file.id,
          path,
          url,
          createdAt: file.created_at,
        };
      });

    const resolved = await Promise.all(imagePromises);
    setImages(resolved);
    setLoading(false);
  }, [userId]);

  useEffect(() => {
    fetchImages();
  }, [fetchImages, refreshTrigger]);

  const renderItem = useCallback(
    ({ item }: { item: GalleryImage }) => (
      <TouchableOpacity
        style={styles.imageContainer}
        onPress={() => onImagePress?.(item)}
        activeOpacity={0.8}
      >
        <Image
          source={{ uri: item.url }}
          style={styles.image}
          resizeMode="cover"
        />
      </TouchableOpacity>
    ),
    [onImagePress]
  );

  if (loading) {
    return (
      <View style={styles.loadingContainer}>
        <ActivityIndicator size="large" color="#4F46E5" />
      </View>
    );
  }

  return (
    <FlatList
      data={images}
      renderItem={renderItem}
      keyExtractor={(item) => item.id}
      numColumns={NUM_COLUMNS}
      contentContainerStyle={styles.grid}
      showsVerticalScrollIndicator={false}
      // パフォーマンス最適化
      removeClippedSubviews={true}
      maxToRenderPerBatch={12}
      windowSize={5}
      getItemLayout={(_, index) => ({
        length: ITEM_SIZE + ITEM_SPACING,
        offset: (ITEM_SIZE + ITEM_SPACING) * Math.floor(index / NUM_COLUMNS),
        index,
      })}
    />
  );
}

// 期待される表示:
// - 3列のグリッドレイアウトで画像がタイル状に並ぶ
// - スクロール時にスムーズなパフォーマンス
// - 画像タップで onImagePress コールバックが発火

const styles = StyleSheet.create({
  grid: { padding: ITEM_SPACING },
  imageContainer: {
    width: ITEM_SIZE,
    height: ITEM_SIZE,
    margin: ITEM_SPACING / 2,
    borderRadius: 4,
    overflow: 'hidden',
  },
  image: { width: '100%', height: '100%' },
  loadingContainer: { flex: 1, justifyContent: 'center', alignItems: 'center' },
});
```

`FlatList` の `removeClippedSubviews`、`maxToRenderPerBatch`、`windowSize` プロパティを適切に設定することで、画面外の画像コンポーネントをメモリから解放し、スクロール時のフレームレートを維持しています。また、`getItemLayout` を指定することで、FlatList がスクロール位置を計算するためにすべての要素を事前にレンダリングする必要がなくなります。

## 画像のリサイズとキャッシュ戦略

画像をそのままアップロードすると、ストレージ容量と通信コストが増大します。アップロード前のリサイズと、表示時のキャッシュ戦略を組み合わせることで、快適なユーザー体験を維持しましょう。

### アップロード前のリサイズ

`expo-image-manipulator` を使って、アップロード前に画像をリサイズします。

```typescript
// utils/resizeImage.ts
import * as ImageManipulator from 'expo-image-manipulator';

/**
 * 画像をリサイズしてJPEGに変換する
 * @param uri - 元画像のURI
 * @param maxSize - 最大幅/高さ（px）
 * @returns リサイズ後のURI
 */
export async function resizeImage(
  uri: string,
  maxSize: number = 1024
): Promise<string> {
  const result = await ImageManipulator.manipulateAsync(
    uri,
    [{ resize: { width: maxSize } }],  // 幅基準でリサイズ（アスペクト比維持）
    {
      compress: 0.8,                    // JPEG品質80%
      format: ImageManipulator.SaveFormat.JPEG,
    }
  );

  return result.uri;
}

// 使用例:
// const resizedUri = await resizeImage(originalUri, 1024);
// const path = await uploadImage(resizedUri, user.id);
// → 元画像が4032×3024pxでも、1024×768px程度に圧縮されてアップロードされる
```

1024px にリサイズし JPEG 80% で圧縮すると、一般的なスマートフォン写真（4〜8MB）が 150〜300KB 程度に削減されます。ギャラリー表示にはこのサイズで十分な画質を保てます。

### Expo Image によるキャッシュ最適化

画像の表示側では、`expo-image` ライブラリを使うことで自動キャッシュが適用されます。標準の `Image` コンポーネントよりもキャッシュ制御が細かく、同じ画像の再ダウンロードを防ぎます。

```typescript
// expo-image を使った表示（FlatList の renderItem 内で使用）
import { Image } from 'expo-image';

<Image
  source={{ uri: item.url }}
  style={styles.image}
  contentFit="cover"
  placeholder={{ blurhash: 'LKO2?U%2Tw=w]~RBVZRi};RPxuwH' }}
  transition={200}                    // 200msのフェードイン
  cachePolicy="memory-disk"           // メモリ + ディスクキャッシュ
/>

// 期待される動作:
// - 初回読み込み時: プレースホルダー（ブラーハッシュ）→ フェードインで画像表示
// - 2回目以降: キャッシュから即座に表示（ネットワークリクエストなし）
```

`cachePolicy: "memory-disk"` を設定することで、メモリキャッシュ（アプリ起動中の高速アクセス）とディスクキャッシュ（アプリ再起動後も有効）の2段階キャッシュが適用されます。`placeholder` の `blurhash` はアップロード時にサーバー側で生成しておくと、ギャラリーの体感速度がさらに向上します。

## よくある質問（FAQ）

### Q1: Supabase Storage の無料プランで何枚くらいの画像を保存できますか？

Supabase の無料プランでは 1GB のストレージが利用できます。1枚あたり 200KB（リサイズ済み）として計算すると、約 5,000 枚の画像を保存できます。実際にはファイルサイズにばらつきがありますが、個人開発のプロトタイプやテストには十分な容量です。本格的なサービスとして運用する場合は、Pro プラン（月額 $25 / 100GB）へのアップグレードを検討してください。

### Q2: 画像のアップロード速度を改善するにはどうすれば良いですか？

3つのアプローチがあります。第一に、アップロード前に `expo-image-manipulator` で画像をリサイズすること。1024px × JPEG 80% で十分な画質を維持しつつ、ファイルサイズを大幅に削減できます。第二に、複数画像を並行アップロードする場合は `Promise.all` ではなく `Promise.allSettled` を使い、一部が失敗しても他の画像のアップロードを継続できるようにします。第三に、`expo-file-system` の `uploadAsync` を使うと、Base64 変換を経由せずに直接アップロードでき、メモリ消費が改善します。

### Q3: 他のユーザーに画像を公開共有するにはどうすれば良いですか？

RLS ポリシーを追加して、特定の条件で他ユーザーも閲覧可能にします。たとえば「公開フラグが true の画像は全ユーザーが閲覧可能」というポリシーを追加するには、画像メタデータを管理する別テーブル（`image_metadata`）を作成し、`is_public` カラムで制御するのが一般的なパターンです。バケット自体を Public にするのではなく、テーブルベースのアクセス制御と組み合わせることで、柔軟かつ安全な共有機能を実現できます。

### Q4: HEIC 形式の画像（iPhone のデフォルト形式）にも対応できますか？

`expo-image-picker` は `launchImageLibraryAsync` の呼び出し時に、iOS が自動的に HEIC を JPEG に変換してくれます。そのため、特別な対応は不要です。ただし、`expo-document-picker` など他の方法でファイルを取得した場合は HEIC のまま取得される可能性があるため、`expo-image-manipulator` で JPEG に変換するステップを追加してください。

## まとめ

この記事では、Rork と Supabase Storage を組み合わせた画像アップロード・ギャラリー表示の実装方法を解説しましました。ポイントを振り返ると、Supabase Storage のバケットと RLS ポリシーで安全なストレージ基盤を構築し、`expo-image-picker` で直感的な画像選択UIを実現し、アップロード前のリサイズでストレージコストとパフォーマンスを最適化し、FlatList のチューニングでスムーズなギャラリー表示を実現する、という一連のフローになります。

この基本パターンをマスターすれば、SNSアプリの投稿機能、ECアプリの商品画像管理、ポートフォリオアプリの作品ギャラリーなど、あらゆるアプリに画像機能を組み込めるようになります。まずは Rork で小さなプロトタイプを作って、画像アップロードの感覚をつかんでみてください。

さらに実践的なバックエンド連携の詳細については、[Rork × Supabase 認証・リアルタイム通信完全ガイド](/articles/rork-dev/rork-supabase-auth-realtime-guide) で認証やリアルタイム機能との組み合わせ方を解説しています。画像表示のパフォーマンスをさらに追求したい方は、[Rork アプリの画像最適化とキャッシュ戦略](/articles/rork-dev/rork-image-optimization-cache-strategy) もぜひご覧ください。
