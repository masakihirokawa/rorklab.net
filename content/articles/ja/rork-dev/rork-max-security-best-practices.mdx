# Rork Max セキュリティベストプラクティス

セキュリティはアプリの信頼の基盤です。このガイドでは、Rork Max アプリの脆弱性を排除し、ユーザーデータを守るための具体的な実装パターンを紹介します。

## セキュリティの三本柱

1. **データ転送**: HTTPS 通信と証明書ピンニング
2. **認証・認可**: トークンベース認証と安全な保管
3. **データ保護**: 暗号化と権限管理

## 第 1 部: API キー管理 — 絶対にしてはいけないこと

### ❌ していけないこと

```typescript
// ❌ 大間違い 1: ハードコードされたキー
const API_KEY = 'sk-abc123xyz'; // 絶対にしてはいけない

// ❌ 大間違い 2: .env ファイル
// .env ファイルをバンドルに含める
export const API_KEY = process.env.REACT_APP_OPENAI_KEY;

// ❌ 大間違い 3: AsyncStorage に平文で保存
await AsyncStorage.setItem('api_key', apiKey);
```

### ✓ 正しいパターン: バックエンド プロキシ

クライアントは決して API キーを保持しません。バックエンドを経由して API を呼び出します。

**Architecture**
```
[Rork Max App] → [Your Backend] → [OpenAI API]
  (Auth Token)    (API Key)
```

**src/services/apiClient.ts**
```typescript
import * as SecureStore from 'expo-secure-store';

interface APIResponse<T> {
  success: boolean;
  data?: T;
  error?: string;
}

export class APIClient {
  private baseURL = 'https://your-backend.com/api';
  private tokenKey = 'auth_token';

  async request<T>(
    endpoint: string,
    options: RequestInit = {}
  ): Promise<T> {
    try {
      const token = await SecureStore.getItemAsync(this.tokenKey);

      if (!token) {
        throw new Error('Authentication required');
      }

      const response = await fetch(`${this.baseURL}${endpoint}`, {
        ...options,
        headers: {
          'Content-Type': 'application/json',
          Authorization: `Bearer ${token}`,
          ...options.headers,
        },
      });

      if (!response.ok) {
        if (response.status === 401) {
          // トークン期限切れ
          await this.refreshToken();
          return this.request<T>(endpoint, options);
        }
        throw new Error(`HTTP ${response.status}`);
      }

      return response.json();
    } catch (error) {
      console.error('API request failed:', error);
      throw error;
    }
  }

  async post<T>(
    endpoint: string,
    body: any
  ): Promise<T> {
    return this.request<T>(endpoint, {
      method: 'POST',
      body: JSON.stringify(body),
    });
  }

  async refreshToken(): Promise<void> {
    try {
      const refreshToken = await SecureStore.getItemAsync('refresh_token');
      const response = await fetch(`${this.baseURL}/auth/refresh`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refreshToken }),
      });

      const data = await response.json();
      await SecureStore.setItemAsync(this.tokenKey, data.accessToken);
    } catch (error) {
      console.error('Token refresh failed:', error);
      // ログアウト処理
      await this.logout();
    }
  }

  async logout(): Promise<void> {
    await SecureStore.deleteItemAsync(this.tokenKey);
    await SecureStore.deleteItemAsync('refresh_token');
  }
}

export const apiClient = new APIClient();
```

**バックエンドの実装例** (Node.js/Express)
```typescript
// backend/routes/ai.ts
import express from 'express';
import { OpenAI } from 'openai';

const router = express.Router();
const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY, // サーバー側にのみ存在
});

// クライアントは API キーを見ることができない
router.post('/chat', async (req, res) => {
  try {
    const { messages } = req.body;

    const completion = await openai.chat.completions.create({
      model: 'gpt-4o',
      messages,
      max_tokens: 500,
    });

    res.json({
      success: true,
      data: completion.choices[0].message.content,
    });
  } catch (error) {
    res.status(500).json({
      success: false,
      error: 'Internal server error',
    });
  }
});

export default router;
```

<Callout type="warning">
**最重要**: クライアント側には決して API キー、データベースパスワード、秘密鍵を含めてはいけません。バックエンド経由でのみアクセス。
</Callout>

### 認証マネージャー

**src/services/authManager.ts**
```typescript
import * as SecureStore from 'expo-secure-store';

interface AuthTokens {
  accessToken: string;
  refreshToken: string;
  expiresAt: number;
}

export class AuthManager {
  private static readonly ACCESS_TOKEN_KEY = 'access_token';
  private static readonly REFRESH_TOKEN_KEY = 'refresh_token';
  private static readonly USER_KEY = 'user_data';

  static async saveTokens(tokens: AuthTokens): Promise<void> {
    try {
      await SecureStore.setItemAsync(
        this.ACCESS_TOKEN_KEY,
        tokens.accessToken
      );
      await SecureStore.setItemAsync(
        this.REFRESH_TOKEN_KEY,
        tokens.refreshToken
      );

      // expiresAt をメモリに保持（SecureStore には暗号化しても保存可能だが、メモリが速い）
      sessionStorage.setItem('token_expires_at', String(tokens.expiresAt));
    } catch (error) {
      console.error('Failed to save tokens:', error);
      throw error;
    }
  }

  static async getAccessToken(): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(this.ACCESS_TOKEN_KEY);
    } catch (error) {
      console.error('Failed to get access token:', error);
      return null;
    }
  }

  static async getRefreshToken(): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(this.REFRESH_TOKEN_KEY);
    } catch (error) {
      console.error('Failed to get refresh token:', error);
      return null;
    }
  }

  static async clearAll(): Promise<void> {
    try {
      await SecureStore.deleteItemAsync(this.ACCESS_TOKEN_KEY);
      await SecureStore.deleteItemAsync(this.REFRESH_TOKEN_KEY);
      await SecureStore.deleteItemAsync(this.USER_KEY);
    } catch (error) {
      console.error('Failed to clear auth data:', error);
    }
  }

  static async isTokenExpired(): Promise<boolean> {
    const expiresAt = parseInt(
      sessionStorage.getItem('token_expires_at') || '0'
    );
    return Date.now() > expiresAt;
  }
}
```

### ログイン画面での使用

**src/screens/LoginScreen.tsx**
```typescript
import React, { useState } from 'react';
import {
  View,
  TextInput,
  TouchableOpacity,
  Text,
  ActivityIndicator,
  StyleSheet,
} from 'react-native';
import { apiClient } from '../services/apiClient';
import { AuthManager } from '../services/authManager';

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, justifyContent: 'center' },
  input: {
    borderWidth: 1,
    borderColor: '#ccc',
    padding: 12,
    marginBottom: 12,
    borderRadius: 8,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 12,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
  errorText: { color: '#FF3B30', marginTop: 12 },
});

export default function LoginScreen() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  const handleLogin = async () => {
    if (!email || !password) {
      setError('メールアドレスとパスワードを入力してください');
      return;
    }

    setIsLoading(true);
    setError('');

    try {
      const response = await apiClient.post('/auth/login', {
        email,
        password,
      });

      // トークンを安全に保存
      await AuthManager.saveTokens({
        accessToken: response.accessToken,
        refreshToken: response.refreshToken,
        expiresAt: response.expiresAt,
      });

      // ナビゲーション: ホーム画面へ
      // navigation.replace('Home');
    } catch (err) {
      setError('ログインに失敗しました。再度試してください。');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="メールアドレス"
        value={email}
        onChangeText={setEmail}
        keyboardType="email-address"
        editable={!isLoading}
      />

      <TextInput
        style={styles.input}
        placeholder="パスワード"
        value={password}
        onChangeText={setPassword}
        secureTextEntry
        editable={!isLoading}
      />

      <TouchableOpacity
        style={styles.button}
        onPress={handleLogin}
        disabled={isLoading}
      >
        {isLoading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>ログイン</Text>
        )}
      </TouchableOpacity>

      {error && <Text style={styles.errorText}>{error}</Text>}
    </View>
  );
}
```

## 第 3 部: バイオメトリック認証

パスワードレス認証で UX と セキュリティを両立。

### インストール

```bash
expo install expo-local-authentication
```

**src/services/biometricAuth.ts**
```typescript
import * as LocalAuthentication from 'expo-local-authentication';
import * as SecureStore from 'expo-secure-store';

export class BiometricAuth {
  static async isAvailable(): Promise<boolean> {
    try {
      const compatible = await LocalAuthentication.hasHardwareAsync();
      const enrolled = await LocalAuthentication.isEnrolledAsync();
      return compatible && enrolled;
    } catch {
      return false;
    }
  }

  static async authenticate(): Promise<boolean> {
    try {
      const result = await LocalAuthentication.authenticateAsync({
        reason: 'アプリにアクセスするために認証してください',
        disableDeviceFallback: false,
        fallbackLabel: 'パスコードを使用',
      });

      return result.success;
    } catch (error) {
      console.error('Biometric auth failed:', error);
      return false;
    }
  }

  static async enableBiometric(
    accessToken: string
  ): Promise<void> {
    try {
      await SecureStore.setItemAsync('biometric_enabled', 'true');
      // トークンはすでに SecureStore に保存されている
    } catch (error) {
      console.error('Failed to enable biometric:', error);
    }
  }

  static async isBiometricEnabled(): Promise<boolean> {
    try {
      const enabled = await SecureStore.getItemAsync('biometric_enabled');
      return enabled === 'true';
    } catch {
      return false;
    }
  }
}
```

**ログイン画面での実装**

## 第 4 部: 通信の暗号化と証明書ピンニング

すべての通信は HTTPS で、証明書ピンニングで中間者攻撃を防止。

**src/services/certificatePinning.ts**
```typescript
import * as SecureStore from 'expo-secure-store';

interface CertificatePin {
  domain: string;
  publicKeyHash: string;
}

export class CertificatePinning {
  private static pinningRules: CertificatePin[] = [
    {
      domain: 'your-backend.com',
      publicKeyHash:
        'sha256/AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA=', // 実際の値に置き換え
    },
    {
      domain: 'api.openai.com',
      publicKeyHash:
        'sha256/BBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBBB=',
    },
  ];

  static async validateCertificate(
    domain: string
  ): Promise<boolean> {
    try {
      const pin = this.pinningRules.find((p) => p.domain === domain);
      if (!pin) {
        console.warn(`No pin found for domain: ${domain}`);
        return true; // ピンがない場合は通す
      }

      // 実装: サーバー証明書とピンを検証
      // (React Native では TLS ハンドシェイク時にカスタムロジックを挿入)
      return true;
    } catch (error) {
      console.error('Certificate validation failed:', error);
      return false;
    }
  }
}
```

## 第 5 部: データ保護 — 暗号化と削除

### センシティブデータの保護

```typescript
import * as SecureStore from 'expo-secure-store';

interface SensitiveData {
  value: string;
  encrypted: boolean;
}

export class DataProtection {
  // 機密データは SecureStore に保管
  static async saveSecret(key: string, value: string): Promise<void> {
    try {
      await SecureStore.setItemAsync(key, value);
    } catch (error) {
      console.error('Failed to save secret:', error);
    }
  }

  static async getSecret(key: string): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(key);
    } catch (error) {
      console.error('Failed to retrieve secret:', error);
      return null;
    }
  }

  // データ削除時は復旧不能に
  static async deleteSecret(key: string): Promise<void> {
    try {
      await SecureStore.deleteItemAsync(key);
    } catch (error) {
      console.error('Failed to delete secret:', error);
    }
  }

  // ログアウト時に全ての機密データを削除
  static async clearAllSecrets(keys: string[]): Promise<void> {
    try {
      await Promise.all(
        keys.map((key) => SecureStore.deleteItemAsync(key))
      );
    } catch (error) {
      console.error('Failed to clear secrets:', error);
    }
  }
}
```

### AsyncStorage vs SecureStore

| 用途 | ストレージ | 理由 |
|------|-----------|------|
| API キー | SecureStore | 暗号化必須 |
| 認証トークン | SecureStore | 暗号化必須 |
| ユーザーID | AsyncStorage | 平文でも OK |
| UI 設定 | AsyncStorage | 暗号化不要 |
| PII（個人識別情報） | SecureStore | 暗号化必須 |

## 第 6 部: OWASP Mobile Top 10 対策

### 1. 不正な Platform Usage

```typescript
// ❌ 危険
import { Platform } from 'react-native';
const secret = Platform.OS === 'ios' ? 'secret123' : 'secret456';

// ✓ 正しい
// プラットフォーム固有の処理は環境で管理
const secret = process.env.APP_SECRET; // サーバーから取得
```

### 2. Insecure Data Storage

```typescript
// ❌ 危険
AsyncStorage.setItem('user_token', token);

// ✓ 正しい
SecureStore.setItemAsync('user_token', token);
```

### 3. Insecure Communication

```typescript
// ❌ 危険
fetch('http://api.example.com/data'); // HTTP

// ✓ 正しい
fetch('https://api.example.com/data'); // HTTPS
```

### 4. Insecure Authentication

```typescript
// ❌ 危険
const password = 'hardcoded-password';

// ✓ 正しい
// バイオメトリック認証 + トークンベース認証
await BiometricAuth.authenticate();
const token = await AuthManager.getAccessToken();
```

## セキュリティ チェックリスト

### コード レベル
- [ ] API キーはハードコードされていない
- [ ] トークンは SecureStore に保管
- [ ] すべての通信は HTTPS
- [ ] 秘密情報は環境変数で管理
- [ ] エラーメッセージに詳細情報を含めない

### ネットワーク レベル
- [ ] 証明書ピンニングを実装
- [ ] キープアライブ設定で接続時間を制限
- [ ] タイムアウトを設定（通常 30 秒以下）
- [ ] TLS 1.2 以上を使用

### 認証・認可
- [ ] 多要素認証（MFA）を検討
- [ ] トークンの有効期限を短く（15～30 分）
- [ ] リフレッシュトークンで延長
- [ ] ログアウト時にトークンを無効化

### データ保護
- [ ] 不要なデータを保存しない
- [ ] ログアウト時にすべての機密データを削除
- [ ] バックアップから除外
- [ ] デバイス紛失時のリモートワイプ検討

### App Store 提出時
- [ ] プライバシーポリシーを作成
- [ ] データ使用目的を明示
- [ ] GDPR / CCPA コンプライアンス確認
- [ ] テスト用キーを使用

## まとめ

Rork Max でセキュアなアプリを構築するための鍵：

1. **API キー**: バックエンド経由でのみアクセス
2. **トークン**: SecureStore で暗号化保管
3. **通信**: すべて HTTPS + 証明書検証
4. **認証**: トークンベース + バイオメトリック
5. **データ**: 機密情報のみ暗号化、ログアウト時削除
6. **監査**: OWASP モバイル Top 10 を確認

セキュリティは一度実装したら終わりではなく、継続的な監視・更新が必要です。
