## はじめに — アプリの認証をどう選ぶか

アプリ開発において「認証」は避けて通れない重要な機能です。ユーザーがアプリを使い始める最初の関門であり、ログイン体験が悪ければ離脱率が急上昇します。

Rorkでアプリを開発していると、「どの認証サービスを選ぶべきか？」という疑問に直面する方も多いでしょう。Firebase Authentication、Supabase Auth、Auth0など選択肢はさまざまですが、2026年現在、Expo/React Nativeアプリとの相性が最も優れているのが **Clerk（クラーク）** です。

この記事では、RorkアプリにClerkを導入してGoogle・Apple・メール/パスワード認証を実装する手順を、初めての方でも迷わず進められるよう丁寧に解説します。

---

## Clerkとは？なぜRorkアプリに最適なのか

**Clerk**は、アプリケーションに認証・ユーザー管理機能を提供するSaaSサービスです。Expo/React Native向けの公式SDKが充実しており、わずか数十行のコードでプロダクションレベルの認証機能を実装できます。

### Clerkが選ばれる主な理由

- **Expo専用SDK**: `@clerk/clerk-expo` が公式提供されており、セットアップが非常にシンプル
- **豊富なソーシャルログイン**: Google、Apple、GitHub、Twitter/X、LINE など20以上のOAuthプロバイダーに対応
- **美しいUIコンポーネント**: `<SignIn />`、`<SignUp />` などのコンポーネントをそのまま使用可能
- **Webhookとバックエンド連携**: ユーザー作成・削除イベントをバックエンドに通知できる
- **無料枠が手厚い**: 月間10,000 MAUまで無料

### Firebase Auth・Supabase Authとの比較

- **Firebase Auth**: Google系サービスとの連携は強いが、iOS向けの設定が煩雑。Expoとの相性はClerkより劣る
- **Supabase Auth**: バックエンドをSupabaseで統一したい場合に最適。ただしUIは自作が必要
- **Clerk**: Expo/React Nativeアプリに特化した設計で、最も少ないコード量で実装可能

## 環境準備 — Clerkアカウントの作成とアプリ設定

### Step 1: Clerkダッシュボードでアプリを作成

[Clerk公式サイト](https://clerk.com/)にアクセスし、無料アカウントを作成します。

1. ダッシュボードから「Create application」をクリック
2. アプリ名を入力（例: `MyRorkApp`）
3. 認証方法を選択（Google、Apple、Email をチェック）
4. 「Create application」ボタンで作成

作成後、**Publishable Key**（`pk_test_...` 形式）と **Secret Key**（`sk_test_...` 形式）が発行されます。Publishable Keyはクライアントサイドで使用するため、フロントエンドコードに含めても問題ありません。

> ⚠️ Secret Keyは絶対にクライアントコードに含めないでください。バックエンドAPIのみで使用します。

### Step 2: Google OAuth設定

Clerkダッシュボードの「Configure → SSO connections → Google」から設定します。

開発環境では、Clerkが提供するデフォルトのGoogle OAuthクレデンシャルを使用できます。本番環境では、Google Cloud ConsoleでOAuth 2.0クライアントIDを発行して登録する必要があります。

### Step 3: Apple Sign In設定

App Store公開にはApple Sign Inの実装が**必須要件**です（他のソーシャルログインを提供するアプリへの義務）。

1. Apple Developer Console でService IDを作成
2. Clerk ダッシュボードの「Configure → SSO connections → Apple」に登録
3. Rork MaxのXcodeプロジェクトで「Sign In with Apple」Capabilityを有効化

## RorkプロジェクトへのClerk導入手順

### Step 1: SDKのインストール

Rorkのプロジェクトコンソールで以下のパッケージを追加します。

```bash
# Rorkのターミナルまたはパッケージ管理機能から実行
npx expo install @clerk/clerk-expo expo-secure-store
```

`expo-secure-store` はClerkがトークンをセキュアに保存するために使用します。Rorkアプリでは `expo-secure-store` が組み込まれているケースが多いため、既に存在する場合は追加不要です。

### Step 2: 環境変数の設定

プロジェクトルートの `.env` ファイルに以下を追加します。

```bash
# .env（Rorkプロジェクトのルートディレクトリ）
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=YOUR_CLERK_PUBLISHABLE_KEY
```

Rorkでは環境変数を `app.config.js` や `.env` で管理できます。`EXPO_PUBLIC_` プレフィックスを付けることでExpoクライアントコードから安全にアクセスできます。

### Step 3: ClerkProviderをアプリのルートに設定

`app/_layout.tsx`（Expo RouterのルートLayout）にClerkProviderを追加します。

```tsx
// app/_layout.tsx
import { ClerkProvider, ClerkLoaded } from '@clerk/clerk-expo';
import { tokenCache } from '@/utils/tokenCache';
import { Stack } from 'expo-router';

const publishableKey = process.env.EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY!;

if (!publishableKey) {
  throw new Error('EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY が設定されていません');
}

export default function RootLayout() {
  return (
    <ClerkProvider publishableKey={publishableKey} tokenCache={tokenCache}>
      <ClerkLoaded>
        <Stack />
      </ClerkLoaded>
    </ClerkProvider>
  );
}
```

`tokenCache` はトークンをSecureStoreに保存するユーティリティです。

```tsx
// utils/tokenCache.ts
import * as SecureStore from 'expo-secure-store';
import { TokenCache } from '@clerk/clerk-expo/dist/cache';

const createTokenCache = (): TokenCache => {
  return {
    getToken: async (key: string) => {
      try {
        return SecureStore.getItemAsync(key);
      } catch {
        return null;
      }
    },
    saveToken: (key: string, token: string) => {
      return SecureStore.setItemAsync(key, token);
    },
    clearToken: (key: string) => {
      return SecureStore.deleteItemAsync(key);
    },
  };
};

export const tokenCache = createTokenCache();
```

## Google・Apple ソーシャルログインの実装

### ログイン画面の作成

Clerkの `useOAuth` フックを使うと、数行でソーシャルログインを実装できます。

```tsx
// app/(auth)/sign-in.tsx
import { useOAuth } from '@clerk/clerk-expo';
import * as WebBrowser from 'expo-web-browser';
import { View, TouchableOpacity, Text, StyleSheet } from 'react-native';
import { useRouter } from 'expo-router';

// WebBrowserセッションをウォームアップ（iOSでの高速化）
WebBrowser.maybeCompleteAuthSession();

export default function SignInScreen() {
  const router = useRouter();

  // Googleログイン
  const { startOAuthFlow: startGoogleFlow } = useOAuth({
    strategy: 'oauth_google',
  });

  // Appleログイン
  const { startOAuthFlow: startAppleFlow } = useOAuth({
    strategy: 'oauth_apple',
  });

  const handleGoogleSignIn = async () => {
    try {
      const { createdSessionId, setActive } = await startGoogleFlow();
      if (createdSessionId) {
        // セッションをアクティブに設定
        await setActive!({ session: createdSessionId });
        router.replace('/(tabs)/home');
      }
    } catch (error) {
      console.error('Google ログインエラー:', error);
    }
  };

  const handleAppleSignIn = async () => {
    try {
      const { createdSessionId, setActive } = await startAppleFlow();
      if (createdSessionId) {
        await setActive!({ session: createdSessionId });
        router.replace('/(tabs)/home');
      }
    } catch (error) {
      console.error('Apple ログインエラー:', error);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>ログイン</Text>

      {/* Googleログインボタン */}
      <TouchableOpacity style={styles.googleButton} onPress={handleGoogleSignIn}>
        <Text style={styles.googleButtonText}>Googleでログイン</Text>
      </TouchableOpacity>

      {/* Apple Sign In（iOSでは必須） */}
      <TouchableOpacity style={styles.appleButton} onPress={handleAppleSignIn}>
        <Text style={styles.appleButtonText}>Appleでサインイン</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', padding: 24 },
  title: { fontSize: 28, fontWeight: 'bold', marginBottom: 32, textAlign: 'center' },
  googleButton: {
    backgroundColor: '#fff',
    borderWidth: 1,
    borderColor: '#dadce0',
    borderRadius: 8,
    padding: 16,
    marginBottom: 12,
    alignItems: 'center',
  },
  googleButtonText: { fontSize: 16, color: '#3c4043' },
  appleButton: {
    backgroundColor: '#000',
    borderRadius: 8,
    padding: 16,
    alignItems: 'center',
  },
  appleButtonText: { fontSize: 16, color: '#fff' },
});
```

## 認証済みルートの保護とユーザー情報の取得

### `useAuth` と `useUser` フックの活用

```tsx
// app/(tabs)/profile.tsx
import { useAuth, useUser } from '@clerk/clerk-expo';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useRouter } from 'expo-router';

export default function ProfileScreen() {
  const { isSignedIn, signOut } = useAuth();
  const { user } = useUser();
  const router = useRouter();

  // 未ログインの場合はログイン画面にリダイレクト
  if (!isSignedIn) {
    router.replace('/(auth)/sign-in');
    return null;
  }

  const handleSignOut = async () => {
    await signOut();
    router.replace('/(auth)/sign-in');
  };

  return (
    <View style={styles.container}>
      <Text style={styles.greeting}>
        こんにちは、{user?.firstName ?? 'ゲスト'} さん！
      </Text>
      <Text style={styles.email}>{user?.primaryEmailAddress?.emailAddress}</Text>
      <TouchableOpacity style={styles.signOutButton} onPress={handleSignOut}>
        <Text style={styles.signOutText}>ログアウト</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center', padding: 24 },
  greeting: { fontSize: 22, fontWeight: 'bold', marginBottom: 8 },
  email: { fontSize: 14, color: '#666', marginBottom: 32 },
  signOutButton: {
    backgroundColor: '#ff4757',
    borderRadius: 8,
    padding: 16,
    minWidth: 160,
    alignItems: 'center',
  },
  signOutText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});
```

### Expo Routerによるルート保護

`app/_layout.tsx` でサインイン状態を監視し、未認証ユーザーを自動リダイレクトできます。

```tsx
// app/_layout.tsx（認証ガード追加版）
import { useAuth } from '@clerk/clerk-expo';
import { useSegments, useRouter } from 'expo-router';
import { useEffect } from 'react';

function AuthGuard({ children }: { children: React.ReactNode }) {
  const { isSignedIn, isLoaded } = useAuth();
  const segments = useSegments();
  const router = useRouter();

  useEffect(() => {
    if (!isLoaded) return;
    const inAuthGroup = segments[0] === '(auth)';

    if (!isSignedIn && !inAuthGroup) {
      // 未認証 → ログイン画面へ
      router.replace('/(auth)/sign-in');
    } else if (isSignedIn && inAuthGroup) {
      // 認証済み → ホーム画面へ
      router.replace('/(tabs)/home');
    }
  }, [isSignedIn, isLoaded, segments]);

  return <>{children}</>;
}
```

## よくある質問（FAQ）

### Q1: Firebase AuthやSupabase Authからの移行は大変ですか？

既存の認証システムからの移行は計画が必要ですが、Clerkはユーザーデータのインポート機能を提供しています。ただし、既存のユーザーは再ログインが必要になるケースが多いため、移行タイミングを慎重に検討してください。

### Q2: 無料プランで何MAU（月間アクティブユーザー）まで使えますか？

Clerkの無料プランは **月間10,000 MAU** まで対応しています。個人開発アプリやMVPフェーズでは十分な範囲であることがほとんどです。超過した場合は有料プランへのアップグレードが必要です（料金はClerk公式サイトでご確認ください）。

### Q3: Rork Max（ネイティブSwift）ではClerkを使えますか？

Rork MaxはReact Native/Expo上で動作するため、`@clerk/clerk-expo` をそのまま利用できます。ただし、Rork MaxでSwiftコードを直接記述する場合（ネイティブモジュール等）は、ClerkのSwift SDKを別途検討する必要があります。

### Q4: App Store審査でApple Sign Inは必須ですか？

はい。**Apple Sign Inは、サードパーティのソーシャルログイン（Googleなど）を提供するすべてのアプリに必須**です。これを実装しないとApp Store審査で却下されます。Clerkを使うと、Apple Sign Inも他のOAuthプロバイダーと同様にシンプルに実装できます。

## まとめ

この記事では、RorkアプリにClerk認証を導入する方法を解説しましました。

- **Clerkはシンプルかつ強力**: Expo向け公式SDKで、最小限のコードで本番レベルの認証を実現
- **Google・Appleログインが数十行で実装可能**: `useOAuth` フック1つで複数プロバイダーに対応
- **無料枠が手厚い**: 月間10,000 MAUまで無料で使用可能
- **ルート保護も簡単**: `useAuth` と Expo Router の組み合わせで認証ガードを実装
