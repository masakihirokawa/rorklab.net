## はじめに

iPhone 14 Pro 以降に搭載された **Dynamic Island** は、画面上部のパンチホールを活用してアプリのリアルタイム情報を表示する革新的な機能です。配車アプリの到着時刻、スポーツのスコア、フードデリバリーの進捗など、ユーザーがアプリを開かなくても最新情報を確認できます。

しかし、Dynamic Island と **Live Activities** を実装するには Apple の **ActivityKit** フレームワークと WidgetKit を Swift で記述する必要があり、React Native や Expo ではネイティブの実装が困難でした。

**Rork Max** はネイティブ SwiftUI コードを生成するため、Dynamic Island と Live Activities を完全サポートしています。本記事では、Rork Max を使ってこれらの機能をゼロから実装する方法を、具体的なプロンプト例とコード解説を交えながら紹介します。

この記事で学べること：
- Dynamic Island と Live Activities の仕組み
- Rork Max での実装ステップ
- 実践的な ActivityKit コード例
- よくあるエラーと対処法
- 収益化への活用アイデア

対象読者：Rork Max の基本操作を理解しているユーザー（[Rork Max 入門ガイド](/articles/rork-basics/getting-started) を先に読むことを推奨）。

## Dynamic Island と Live Activities の仕組み

### Dynamic Island とは

Dynamic Island は iOS 16.1 で導入された UI システムで、iPhone の前面カメラ周りの「アイランド（島）」領域に、アプリが動的コンテンツを表示できます。ユーザーが長押しすると展開し、より詳細な情報を表示することも可能です。

| 表示モード | 説明 |
|------------|------|
| コンパクト（左） | 小さなアイコン（例：音楽の再生アイコン） |
| コンパクト（右） | 小さなテキストや数値（例：タイマー残り時間） |
| ミニマル | 最小サイズ（複数アプリが競合する場合） |
| 展開表示 | 長押しで表示される詳細ビュー |

### Live Activities とは

Live Activities は ActivityKit フレームワークを使ってアプリがリアルタイムのデータをロック画面や Dynamic Island に表示する仕組みです。プッシュ通知（APNs）を通じてサーバーからリアルタイム更新が可能で、最大 8 時間（または 12 時間の延長）継続できます。

```swift
// Live Activity の基本的なデータ構造
import ActivityKit

// ActivityAttributes はイベントの静的・動的データを定義する
struct DeliveryAttributes: ActivityAttributes {
    // 静的データ（Activity 開始時に固定）
    public struct ContentState: Codable, Hashable {
        var status: String        // "準備中" | "配達中" | "到着"
        var eta: Date             // 到着予定時刻
        var driverName: String    // 配達員名
        var progress: Double      // 0.0〜1.0（進捗率）
    }

    // 動的データ（リアルタイムで更新可能）
    var orderId: String
    var restaurantName: String
}

// 期待する出力: ActivityAttributes の型定義が確認できる
```

### ステップ 2：ActivityAttributes の定義

Rork Max が自動生成するコードを確認・調整します。

```swift
import ActivityKit
import SwiftUI

// DeliveryAttributes: Live Activity のデータ構造
struct DeliveryAttributes: ActivityAttributes {
    public struct ContentState: Codable, Hashable {
        var status: DeliveryStatus
        var eta: Date
        var driverName: String
        var progress: Double
        var currentLocation: String
    }

    var orderId: String
    var restaurantName: String
    var totalAmount: Double
}

// 配達状況の列挙型
enum DeliveryStatus: String, Codable {
    case preparing = "準備中"
    case pickedUp = "ピックアップ済み"
    case onTheWay = "配達中"
    case nearbyYou = "まもなく到着"
    case delivered = "配達完了"

    var icon: String {
        switch self {
        case .preparing: return "🍳"
        case .pickedUp: return "🛵"
        case .onTheWay: return "🛵"
        case .nearbyYou: return "📍"
        case .delivered: return "✅"
        }
    }
}

// 期待する出力: コンパイルエラーなし、ActivityKit のデータモデルが定義される
```

### ステップ 4：Live Activity の開始・更新・終了

```swift
import ActivityKit

class DeliveryActivityManager {
    var currentActivity: Activity<DeliveryAttributes>?

    // Live Activity を開始する
    func startActivity(orderId: String, restaurantName: String, total: Double) {
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            print("⚠️ Live Activities が無効になっています")
            return
        }

        let attributes = DeliveryAttributes(
            orderId: orderId,
            restaurantName: restaurantName,
            totalAmount: total
        )

        let initialState = DeliveryAttributes.ContentState(
            status: .preparing,
            eta: Date().addingTimeInterval(30 * 60), // 30分後
            driverName: "田中さん",
            progress: 0.1,
            currentLocation: "レストラン"
        )

        do {
            currentActivity = try Activity.request(
                attributes: attributes,
                content: .init(state: initialState, staleDate: nil),
                pushType: .token  // APNs でリアルタイム更新を有効化
            )
            print("✅ Live Activity 開始: \(currentActivity?.id ?? "")")
        } catch {
            print("❌ 開始エラー: \(error.localizedDescription)")
        }
    }

    // Live Activity を更新する
    func updateActivity(status: DeliveryStatus, progress: Double, eta: Date) async {
        guard let activity = currentActivity else { return }

        let updatedState = DeliveryAttributes.ContentState(
            status: status,
            eta: eta,
            driverName: "田中さん",
            progress: progress,
            currentLocation: status == .onTheWay ? "配達中" : "目的地付近"
        )

        await activity.update(
            ActivityContent(state: updatedState, staleDate: Date().addingTimeInterval(60))
        )
        print("🔄 Live Activity 更新: \(status.rawValue)")
    }

    // Live Activity を終了する
    func endActivity() async {
        guard let activity = currentActivity else { return }

        let finalState = DeliveryAttributes.ContentState(
            status: .delivered,
            eta: Date(),
            driverName: "田中さん",
            progress: 1.0,
            currentLocation: "配達完了"
        )

        await activity.end(
            ActivityContent(state: finalState, staleDate: Date()),
            dismissalPolicy: .after(Date().addingTimeInterval(5 * 60)) // 5分後に自動消去
        )
        currentActivity = nil
        print("🏁 Live Activity 終了")
    }
}

// 期待する出力:
// ✅ Live Activity 開始: [activity-id]
// 🔄 Live Activity 更新: 配達中
// 🏁 Live Activity 終了
```

## APNs を使ったサーバーサイド更新

本番アプリでは、サーバーから APNs 経由で Live Activities をリアルタイム更新します。

```swift
// Push Token の取得（アプリ側）
func startActivityWithPushToken() async {
    // currentActivity の pushToken を監視
    guard let activity = currentActivity else { return }

    for await pushToken in activity.pushTokenUpdates {
        let tokenString = pushToken.map { String(format: "%02x", $0) }.joined()
        print("📱 Push Token: \(tokenString)")

        // サーバーに Push Token を送信する
        await sendTokenToServer(token: tokenString, activityId: activity.id)
    }
}

// サーバーサイド（例：Node.js）から Live Activity を更新する HTTP/2 リクエスト
/*
POST https://api.sandbox.push.apple.com/3/device/{pushToken}

Headers:
  apns-push-type: liveactivity
  apns-topic: {bundleId}.push-type.liveactivity
  apns-priority: 10

Body (JSON):
{
  "aps": {
    "timestamp": 1710835200,
    "event": "update",
    "content-state": {
      "status": "onTheWay",

      "driverName": "田中さん",
      "progress": 0.7,
```

## よくあるエラーと対処法

### エラー 1：`areActivitiesEnabled` が `false`

**原因**: デバイスの設定で Live Activities が無効になっている。

**対処法**:
- ユーザーに「設定 → Face ID とパスコード → ライブアクティビティ」を有効化するよう案内する
- アプリ内で `ActivityAuthorizationInfo().areActivitiesEnabled` を確認してフォールバックUIを表示する

```swift
if !ActivityAuthorizationInfo().areActivitiesEnabled {
    // ライブアクティビティが無効の場合の代替UI（プッシュ通知など）
    scheduleNotification(eta: eta)
}
```

### エラー 2：`ActivityKit` が見つからない

**原因**: ターゲットの最小 iOS バージョンが 16.1 未満。

**Rork Max へのプロンプト例**:
```
iOS Deployment Target を 16.1 以上に設定してください。
Live Activities は iOS 16.1 以上でのみ動作します。
```

### エラー 3：Dynamic Island がシミュレーターで動作しない

**原因**: Dynamic Island は iPhone 14 Pro 以上の実機でのみ動作。シミュレーターでは一部のモデルのみ対応。

**対処法**: シミュレーターで「iPhone 15 Pro」または「iPhone 16 Pro」を選択する。

### エラー 4：Live Activity が 8 時間で終了する

**原因**: ActivityKit の制限（デフォルト最大 8 時間、最大 12 時間まで延長可能）。

**対処法**: `staleDate` を適切に設定し、必要に応じて Activity を再作成するロジックを実装する。

## FAQ

**Q. Dynamic Island は React Native（通常の Rork）で実装できますか？**

A. 標準の React Native では Dynamic Island の実装が非常に困難です。React Native はネイティブコードとのブリッジが必要で、ActivityKit / WidgetKit の完全なサポートには多くのネイティブモジュールの記述が必要になります。**Rork Max** はネイティブ SwiftUI を生成するため、Dynamic Island を AI チャットのみで実装できます。

**Q. Live Activities はどのデバイスで動作しますか？**

A. Live Activities は iOS 16.1 以上のすべての iPhone で動作しますが、Dynamic Island の「アイランド型 UI」は iPhone 14 Pro / 14 Pro Max 以降（ノッチではなくパンチホールカメラ搭載モデル）でのみ表示されます。それ以前のモデルではロック画面のみに表示されます。

**Q. Rork Max での Live Activities 実装に追加費用はかかりますか？**

A. Rork Max のプラン（$200/月）内で Live Activities の実装は可能です。ただし、サーバーサイドから APNs 経由でリアルタイム更新を行う場合は、バックエンドサーバー（例：Node.js on Cloudflare Workers）のコストが別途発生します。Rork と Cloudflare の連携については [Rork Cloudflare 連携ガイド](/articles/rork-dev/rork-algolia-search) が参考になります。

**Q. Live Activities のデータ更新頻度に制限はありますか？**

A. Apple は明確な頻度制限を公式に定めていませんが、過度な更新はシステムによってスロットリングされます。実用的には **1〜2 分に 1 回**の更新が推奨されます。緊急性の高いイベント（スコア変更など）は即座に更新しても問題ありません。
