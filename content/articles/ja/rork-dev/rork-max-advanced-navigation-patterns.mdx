## はじめに — なぜナビゲーション設計が重要なのか

モバイルアプリの品質は、画面遷移の設計で大きく左右されます。どれほど優れたUIコンポーネントを用意しても、ナビゲーションが破綻していればユーザー体験は台無しになります。

Rork Maxでアプリを構築する場合、ベースとなるのはExpo RouterとReact Navigationです。小規模なアプリであればデフォルトの設定で十分ですが、画面数が20を超えるような本格的なアプリでは、ナビゲーションの設計パターンを理解していないと保守性が急激に悪化します。

この記事では、Rork Maxで大規模アプリに耐えうるナビゲーション設計を実装するための上級テクニックを体系的に解説します。ネスト構造の設計原則から、カスタム画面遷移アニメーション、認証フローの分岐制御、共有要素トランジション、そしてパフォーマンス最適化まで、プロダクション品質のアプリに必要な知識を網羅します。

前提として、Rork Maxの基本操作とExpo Routerの基礎知識があることを想定しています。Expo Routerの入門については[Expo RouterとRorkで実装するナビゲーション設計ガイド](/articles/rork-dev/expo-router-navigation-guide)を参照してください。

## Expo Router のファイルベースルーティングを極める

Expo Routerはファイルシステムベースのルーティングを採用しており、`app/` ディレクトリ内のファイル構造がそのままルート構造になります。この仕組みを深く理解することが、高度なナビゲーション設計の第一歩です。

### ルートグループとレイアウトの階層設計

大規模アプリでは、画面を論理的なグループに分割することが不可欠です。Expo Routerの「ルートグループ」（括弧付きディレクトリ）を使うことで、URLに影響を与えずに画面を整理できます。

```typescript
// app/ ディレクトリ構造の設計例
// app/
// ├── (auth)/           ← 認証前の画面グループ
// │   ├── _layout.tsx   ← Stack Navigator（ヘッダーなし）
// │   ├── login.tsx
// │   ├── register.tsx
// │   └── forgot-password.tsx
// ├── (tabs)/           ← メインタブグループ
// │   ├── _layout.tsx   ← Tab Navigator
// │   ├── home/
// │   │   ├── _layout.tsx  ← Stack Navigator（タブ内ネスト）
// │   │   ├── index.tsx
// │   │   └── [id].tsx     ← 動的ルート
// │   ├── search.tsx
// │   ├── notifications.tsx
// │   └── profile/
// │       ├── _layout.tsx
// │       ├── index.tsx
// │       └── settings.tsx
// ├── (modals)/         ← モーダル画面グループ
// │   ├── _layout.tsx   ← Stack Navigator（presentation: modal）
// │   ├── create-post.tsx
// │   └── image-viewer.tsx
// └── _layout.tsx       ← ルートレイアウト（認証チェック）

// app/_layout.tsx — ルートレイアウトの実装
import { Stack } from 'expo-router';
import { useAuth } from '@/hooks/useAuth';

export default function RootLayout() {
  return (
    <Stack screenOptions={{ headerShown: false }}>
      <Stack.Screen name="(auth)" />
      <Stack.Screen name="(tabs)" />
      <Stack.Screen
        name="(modals)"
        options={{
          presentation: 'modal',
          animation: 'slide_from_bottom',
        }}
      />
    </Stack>
  );
}
```

このパターンのポイントは3つあります。第一に、認証画面・メイン画面・モーダルを独立したグループとして管理できる点。第二に、各グループが独自のレイアウト（Navigator）を持つことでナビゲーションの責務が分離される点。第三に、URLパスに括弧部分が含まれないため、ディープリンクが自然なパスになる点です。

## 認証フロー分岐のアーキテクチャ

本番アプリで最も複雑になりやすいのが、認証状態に応じたナビゲーション分岐です。Rork Maxでは、Expo Routerの `redirect` と `useSegments` を組み合わせることで、宣言的な認証ガードを実装できます。

### 認証コンテキストとルートガード

```typescript
// contexts/AuthContext.tsx — 認証状態管理
import React, { createContext, useContext, useEffect, useState } from 'react';
import { useRouter, useSegments, useRootNavigationState } from 'expo-router';
import * as SecureStore from 'expo-secure-store';

type AuthState = {
  user: User | null;
  isLoading: boolean;
  signIn: (email: string, password: string) => Promise<void>;
  signOut: () => Promise<void>;
};

const AuthContext = createContext<AuthState | null>(null);

export function AuthProvider({ children }: { children: React.ReactNode }) {
  const [user, setUser] = useState<User | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const router = useRouter();
  const segments = useSegments();
  const navigationState = useRootNavigationState();

  // 認証状態の初期化
  useEffect(() => {
    const restoreSession = async () => {
      try {
        const token = await SecureStore.getItemAsync('auth_token');
        if (token) {
          const userData = await verifyToken(token);
          setUser(userData);
        }
      } catch (error) {
        console.warn('セッション復元失敗:', error);
        await SecureStore.deleteItemAsync('auth_token');
      } finally {
        setIsLoading(false);
      }
    };
    restoreSession();
  }, []);

  // ナビゲーションガード — 認証状態変更時にリダイレクト
  useEffect(() => {
    if (isLoading || !navigationState?.key) return;

    const inAuthGroup = segments[0] === '(auth)';

    if (!user && !inAuthGroup) {
      // 未認証 → ログイン画面へ
      router.replace('/(auth)/login');
    } else if (user && inAuthGroup) {
      // 認証済み → メイン画面へ
      router.replace('/(tabs)/home');
    }
  }, [user, segments, isLoading, navigationState?.key]);

  const signIn = async (email: string, password: string) => {
    const { token, user: userData } = await authenticateUser(email, password);
    await SecureStore.setItemAsync('auth_token', token);
    setUser(userData);
    // useEffect内のガードが自動的にリダイレクトを処理
  };

  const signOut = async () => {
    await SecureStore.deleteItemAsync('auth_token');
    setUser(null);
    // useEffect内のガードが自動的にリダイレクトを処理
  };

  return (
    <AuthContext.Provider value={{ user, isLoading, signIn, signOut }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => {
  const context = useContext(AuthContext);
  if (!context) throw new Error('useAuth must be used within AuthProvider');
  return context;
};
```

### オンボーディングフローとの統合

新規ユーザーに対してオンボーディング画面を表示する場合、認証フローにもう一段階の分岐が必要です。

```typescript
// 拡張版ナビゲーションガード
useEffect(() => {
  if (isLoading || !navigationState?.key) return;

  const inAuthGroup = segments[0] === '(auth)';
  const inOnboarding = segments[0] === '(onboarding)';

  if (!user && !inAuthGroup) {
    router.replace('/(auth)/login');
  } else if (user && !user.onboardingCompleted && !inOnboarding) {
    // オンボーディング未完了
    router.replace('/(onboarding)/welcome');
  } else if (user && user.onboardingCompleted && (inAuthGroup || inOnboarding)) {
    router.replace('/(tabs)/home');
  }
}, [user, segments, isLoading, navigationState?.key]);
```

この設計の利点は、認証ロジックが一箇所に集約されることです。個々の画面コンポーネントが認証状態を気にする必要がなく、新しい画面を追加する際もルートグループに配置するだけでガードが自動的に適用されます。

## カスタム画面遷移アニメーション

デフォルトの画面遷移もスムーズですが、アプリのブランドに合わせたカスタムアニメーションを実装することで、プロフェッショナルな印象を与えることができます。

### Stack Navigatorのカスタムトランジション

```typescript
// app/(tabs)/home/_layout.tsx — カスタム遷移アニメーション
import { Stack } from 'expo-router';
import {
  type StackCardInterpolationProps,
  type StackCardStyleInterpolator,
} from '@react-navigation/stack';
import { Easing } from 'react-native';

// フェード + スケール遷移（iOS風）
const fadeScaleTransition: StackCardStyleInterpolator = ({
  current,
  next,
  inverted,
  layouts: { screen },
}: StackCardInterpolationProps) => {
  const progress = current.progress;
  const nextProgress = next?.progress;

  const opacity = progress.interpolate({
    inputRange: [0, 1],
    outputRange: [0, 1],
  });

  const scale = progress.interpolate({
    inputRange: [0, 1],
    outputRange: [0.92, 1],
  });

  // 次の画面が来たときに現在の画面を縮小
  const containerOpacity = nextProgress
    ? nextProgress.interpolate({
        inputRange: [0, 1],
        outputRange: [1, 0.85],
      })
    : 1;

  return {
    cardStyle: {
      opacity: containerOpacity,
      transform: [{ scale }],
    },
    overlayStyle: { opacity },
  };
};

export default function HomeLayout() {
  return (
    <Stack
      screenOptions={{
        cardStyleInterpolator: fadeScaleTransition,
        transitionSpec: {
          open: {
            animation: 'timing',
            config: {
              duration: 350,
              easing: Easing.bezier(0.2, 0.9, 0.3, 1.0),
            },
          },
          close: {
            animation: 'timing',
            config: {
              duration: 280,
              easing: Easing.bezier(0.4, 0.0, 0.2, 1.0),
            },
          },
        },
        gestureEnabled: true,
        gestureDirection: 'horizontal',
      }}
    >
      <Stack.Screen name="index" options={{ title: 'ホーム' }} />
      <Stack.Screen name="[id]" />
    </Stack>
  );
}
```

### ボトムシート風モーダル遷移

```typescript
// app/(modals)/_layout.tsx — ボトムシートモーダル
import { Stack } from 'expo-router';
import { Platform } from 'react-native';

export default function ModalLayout() {
  return (
    <Stack
      screenOptions={{
        presentation: 'transparentModal',
        headerShown: false,
        cardOverlayEnabled: true,
        cardStyle: { backgroundColor: 'transparent' },
        cardStyleInterpolator: ({ current: { progress } }) => ({
          cardStyle: {
            opacity: progress,
          },
          overlayStyle: {
            opacity: progress.interpolate({
              inputRange: [0, 1],
              outputRange: [0, 0.5],
            }),
          },
        }),
        // iOS: スワイプダウンで閉じる
        ...(Platform.OS === 'ios' && {
          gestureEnabled: true,
          gestureDirection: 'vertical',
        }),
      }}
    />
  );
}
```

アニメーションの設計で重要なのは、`Easing.bezier()` のカーブ選択です。Material Design 3のモーションガイドラインでは、表示時は `(0.2, 0.9, 0.3, 1.0)`（デセレレーション）、非表示時は `(0.4, 0.0, 0.2, 1.0)`（アクセラレーション）が推奨されています。この組み合わせにより、自然で心地よい遷移が実現できます。

## 共有要素トランジション（Shared Element Transition）

リスト画面から詳細画面への遷移で、要素が滑らかに移動するアニメーション（共有要素トランジション）は、ユーザー体験を大幅に向上させます。React Native Reanimatedを使った実装方法を解説します。

### 基本的な共有要素トランジション

```typescript
// components/SharedImage.tsx — 共有要素コンポーネント
import Animated, {
  SharedTransition,
  withSpring,
  withTiming,
} from 'react-native-reanimated';

// カスタムスプリングトランジション定義
const customTransition = SharedTransition.custom((values) => {
  'worklet';
  return {
    width: withSpring(values.targetWidth, {
      damping: 18,
      stiffness: 120,
    }),
    height: withSpring(values.targetHeight, {
      damping: 18,
      stiffness: 120,
    }),
    originX: withSpring(values.targetOriginX, {
      damping: 18,
      stiffness: 120,
    }),
    originY: withSpring(values.targetOriginY, {
      damping: 18,
      stiffness: 120,
    }),
    borderRadius: withTiming(values.targetBorderRadius, {
      duration: 300,
    }),
  };
});

type SharedImageProps = {
  id: string;
  source: { uri: string };
  style?: any;
};

export function SharedImage({ id, source, style }: SharedImageProps) {
  return (
    <Animated.Image
      sharedTransitionTag={`image-${id}`}
      sharedTransitionStyle={customTransition}
      source={source}
      style={style}
    />
  );
}

// screens/ListScreen.tsx — リスト画面での使用
import { SharedImage } from '@/components/SharedImage';
import { Link } from 'expo-router';

function PostCard({ post }: { post: Post }) {
  return (
    <Link href={`/(tabs)/home/${post.id}`} asChild>
      <Pressable style={styles.card}>
        <SharedImage
          id={post.id}
          source={{ uri: post.thumbnail }}
          style={styles.cardImage}
        />
        <Animated.Text
          sharedTransitionTag={`title-${post.id}`}
          style={styles.cardTitle}
        >
          {post.title}
        </Animated.Text>
      </Pressable>
    </Link>
  );
}

// screens/DetailScreen.tsx — 詳細画面での使用
function PostDetail() {
  const { id } = useLocalSearchParams<{ id: string }>();
  const post = usePostData(id);

  return (
    <ScrollView>
      <SharedImage
        id={id}
        source={{ uri: post.image }}
        style={styles.heroImage}
      />
      <Animated.Text
        sharedTransitionTag={`title-${id}`}
        style={styles.detailTitle}
      >
        {post.title}
      </Animated.Text>
      {/* 記事本文 */}
    </ScrollView>
  );
}
```

### 共有要素トランジションの注意点

共有要素トランジションを実装する際に注意すべきポイントがいくつかあります。

第一に、`sharedTransitionTag` はアプリ全体で一意でなければなりません。リスト内の要素には必ずIDを含めてください。同じタグが複数の画面に存在すると予期しない動作になります。

第二に、パフォーマンスの観点から、FlatListで大量のアイテムをレンダリングする場合は `initialNumToRender` と `windowSize` を適切に設定し、画面外のアイテムのトランジションタグが不要に登録されないようにします。

第三に、Android環境では共有要素トランジションのサポートが部分的なケースがあるため、プラットフォーム判定でフォールバックアニメーションを用意しておくのが安全です。

## タブとドロワーの高度な組み合わせパターン

### 条件付きタブバー表示

特定の画面ではタブバーを隠し、他の画面では表示するパターンは頻出します。

```typescript
// app/(tabs)/_layout.tsx — 条件付きタブバー
import { Tabs, usePathname } from 'expo-router';
import { Platform } from 'react-native';
import Animated, {
  useAnimatedStyle,
  withTiming,
  useSharedValue,
} from 'react-native-reanimated';
import { useEffect } from 'react';

// タブバーを隠すべきルート
const HIDDEN_TAB_ROUTES = [
  '/home/create',
  '/home/camera',
  '/profile/settings/account',
];

export default function TabLayout() {
  const pathname = usePathname();
  const tabBarVisible = useSharedValue(1);

  useEffect(() => {
    const shouldHide = HIDDEN_TAB_ROUTES.some((route) =>
      pathname.startsWith(route)
    );
    tabBarVisible.value = withTiming(shouldHide ? 0 : 1, { duration: 200 });
  }, [pathname]);

  return (
    <Tabs
      screenOptions={{
        headerShown: false,
        tabBarStyle: {
          position: 'absolute',
          borderTopWidth: 0,
          elevation: 0,
          ...Platform.select({
            ios: {
              shadowColor: '#000',
              shadowOffset: { width: 0, height: -2 },
              shadowOpacity: 0.08,
              shadowRadius: 8,
            },
          }),
        },
      }}
      tabBar={(props) => <AnimatedTabBar {...props} visible={tabBarVisible} />}
    >
      <Tabs.Screen
        name="home"
        options={{
          title: 'ホーム',
          tabBarIcon: ({ color }) => <HomeIcon color={color} />,
        }}
      />
      <Tabs.Screen
        name="search"
        options={{
          title: '検索',
          tabBarIcon: ({ color }) => <SearchIcon color={color} />,
        }}
      />
      <Tabs.Screen
        name="notifications"
        options={{
          title: '通知',
          tabBarIcon: ({ color }) => <BellIcon color={color} />,
          tabBarBadge: 3, // 未読通知数
        }}
      />
      <Tabs.Screen
        name="profile"
        options={{
          title: 'プロフィール',
          tabBarIcon: ({ color }) => <UserIcon color={color} />,
        }}
      />
    </Tabs>
  );
}

// AnimatedTabBar.tsx — アニメーション付きタブバー
function AnimatedTabBar({ visible, ...props }) {
  const animatedStyle = useAnimatedStyle(() => ({
    transform: [
      {
        translateY: (1 - visible.value) * 100, // 下にスライドして隠す
      },
    ],
    opacity: visible.value,
  }));

  return (
    <Animated.View style={[styles.tabBarContainer, animatedStyle]}>
      {/* タブバーの描画ロジック */}
    </Animated.View>
  );
}
```

## ナビゲーション状態の永続化とリストア

ユーザーがアプリをバックグラウンドに送った後、再開したときに前回の画面状態を復元する機能は、UXの向上に大きく寄与します。

```typescript
// hooks/useNavigationPersistence.ts
import { useEffect, useRef, useCallback } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';
import {
  useNavigationContainerRef,
  useRootNavigation,
} from 'expo-router';

const NAVIGATION_STATE_KEY = 'NAVIGATION_STATE_V1';

export function useNavigationPersistence() {
  const navigationRef = useNavigationContainerRef();
  const isRestored = useRef(false);

  // 状態の保存
  const persistState = useCallback(async () => {
    try {
      const state = navigationRef.current?.getRootState();
      if (state) {
        await AsyncStorage.setItem(
          NAVIGATION_STATE_KEY,
          JSON.stringify(state)
        );
      }
    } catch (error) {
      console.warn('ナビゲーション状態保存失敗:', error);
    }
  }, [navigationRef]);

  // 状態変更のリスナー
  useEffect(() => {
    const unsubscribe = navigationRef.current?.addListener(
      'state',
      persistState
    );
    return () => unsubscribe?.();
  }, [navigationRef, persistState]);

  // 状態の復元
  const restoreState = useCallback(async () => {
    if (isRestored.current) return undefined;
    isRestored.current = true;

    try {
      const savedState = await AsyncStorage.getItem(NAVIGATION_STATE_KEY);
      return savedState ? JSON.parse(savedState) : undefined;
    } catch {
      return undefined;
    }
  }, []);

  return { restoreState, persistState };
}
```

状態の永続化を実装する際は、セキュリティにも配慮が必要です。認証が必要な画面のルート情報を保存する場合は、`expo-secure-store` の使用を検討してください。また、アプリのバージョンアップでルート構造が変わった場合にクラッシュしないよう、バージョンキーを含めておくのがベストプラクティスです。

## ナビゲーションのパフォーマンス最適化

画面数が増えると、ナビゲーションのパフォーマンスが問題になるケースがあります。以下のテクニックで最適化できます。

### 遅延ローディングとコード分割

```typescript
// app/(tabs)/home/heavy-screen.tsx — 遅延ローディング
import { lazy, Suspense } from 'react';
import { ActivityIndicator, View } from 'react-native';

// 重いコンポーネントを遅延読み込み
const HeavyChart = lazy(() => import('@/components/HeavyChart'));
const DataTable = lazy(() => import('@/components/DataTable'));

function LoadingFallback() {
  return (
    <View style={styles.loadingContainer}>
      <ActivityIndicator size="large" color="#6C63FF" />
    </View>
  );
}

export default function HeavyScreen() {
  return (
    <ScrollView>
      <Text style={styles.title}>分析ダッシュボード</Text>

      <Suspense fallback={<LoadingFallback />}>
        <HeavyChart data={chartData} />
      </Suspense>

      <Suspense fallback={<LoadingFallback />}>
        <DataTable columns={columns} rows={rows} />
      </Suspense>
    </ScrollView>
  );
}
```

### 画面のプリフェッチ

```typescript
// hooks/usePrefetchRoute.ts — 画面データのプリフェッチ
import { useQueryClient } from '@tanstack/react-query';
import { useCallback } from 'react';

export function usePrefetchRoute() {
  const queryClient = useQueryClient();

  const prefetchPost = useCallback(
    (postId: string) => {
      // ユーザーが画面に遷移する前にデータを先読み
      queryClient.prefetchQuery({
        queryKey: ['post', postId],
        queryFn: () => fetchPost(postId),
        staleTime: 5 * 60 * 1000, // 5分間有効
      });
    },
    [queryClient]
  );

  return { prefetchPost };
}

// 使用例: リストアイテムのonPressInでプリフェッチ
function PostListItem({ post }: { post: Post }) {
  const { prefetchPost } = usePrefetchRoute();
  const router = useRouter();

  return (
    <Pressable
      onPressIn={() => prefetchPost(post.id)} // タッチ開始時にプリフェッチ
      onPress={() => router.push(`/(tabs)/home/${post.id}`)}
      style={styles.listItem}
    >
      <Text>{post.title}</Text>
    </Pressable>
  );
}
```

`onPressIn`（指が触れた瞬間）でプリフェッチを開始し、`onPress`（指を離した瞬間）で画面遷移する。この数十ミリ秒の差が、体感速度を大きく向上させます。

### メモリ管理とNavigator最適化

```typescript
// app/(tabs)/_layout.tsx — メモリ効率の良い設定
<Tabs
```

## ディープリンクとの統合設計

高度なナビゲーション設計では、ディープリンクとの整合性も重要です。Expo Routerのファイルベースルーティングにより、ディープリンクの設定はシンプルですが、認証ガードとの組み合わせには工夫が必要です。

```typescript
// ディープリンク + 認証ガードの統合パターン
// contexts/AuthContext.tsx に追加

const [pendingDeepLink, setPendingDeepLink] = useState<string | null>(null);

useEffect(() => {
  if (isLoading || !navigationState?.key) return;

  const inAuthGroup = segments[0] === '(auth)';

  if (!user && !inAuthGroup) {
    // 認証前のディープリンクを保存
    const currentPath = '/' + segments.join('/');
    if (currentPath !== '/(auth)/login') {
      setPendingDeepLink(currentPath);
    }
    router.replace('/(auth)/login');
  } else if (user && inAuthGroup) {
    // 認証完了後、保存していたディープリンクに遷移
    if (pendingDeepLink) {
      router.replace(pendingDeepLink);
      setPendingDeepLink(null);
    } else {
      router.replace('/(tabs)/home');
    }
  }
}, [user, segments, isLoading, navigationState?.key]);
```

ディープリンクの詳細な実装パターンについては[Rork Maxでのディープリンク・ユニバーサルリンク実装](/articles/rork-dev/rork-max-deep-linking-universal-links)で体系的に解説しています。

## よくある質問（FAQ）

### Q: Expo RouterとReact Navigationはどちらを使うべきですか？

Rork Maxでは、Expo Routerを第一選択として推奨します。Expo RouterはReact Navigationの上に構築されており、ファイルベースルーティングによる生産性向上、TypeSafeなルーティング、ディープリンクの自動設定、静的解析によるルート検証など、多くの利点があります。React Navigationの直接利用が必要になるのは、Expo Routerがサポートしていない極めて特殊なケース（たとえば動的にNavigatorの種類を切り替えるなど）に限られます。

### Q: 画面数が50を超える大規模アプリでパフォーマンスを維持するコツは？

3つの施策が効果的です。第一に、`lazy: true` と `freezeOnBlur: true` の設定で不要な画面のレンダリングを防止する。第二に、React.lazyとSuspenseで重いコンポーネントを遅延ローディングする。第三に、ルートグループを適切に分割してNavigatorのツリーを浅くする。深いネストは避け、2〜3階層以内に抑えるのが理想です。

### Q: タブの中にStack Navigatorをネストする場合、戻るボタンの挙動はどうなりますか？

タブ内のStack Navigatorでは、戻るボタンはそのスタック内で遷移を戻します。スタックのルート画面まで戻ると、Androidでは戻るボタンでアプリが終了する動作になります。タブ間の移動履歴を管理したい場合は、`backBehavior: 'history'` をTabsのオプションに設定することで、前のタブに戻る動作を実装できます。

### Q: 認証ガードを実装すると、ログイン画面が一瞬チラつくことがあります。対策は？

`isLoading` 状態の間はスプラッシュスクリーンを表示し続けることで解決します。`expo-splash-screen` の `SplashScreen.preventAutoHideAsync()` を使い、認証状態の復元が完了してから `SplashScreen.hideAsync()` を呼び出します。これにより、認証チェック中のチラつきが完全に防止されます。
