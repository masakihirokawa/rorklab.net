## アプリ設計のベストプラクティス

### プロンプト設計のコツ（効果的な指示の出し方）

Rork で高品質なアプリを生成するためには、**明確で構造化されたプロンプト** が不可欠です。

**Step 1：アプリの全体像を定義する**

```
以下の仕様のアプリを Rork で構築します。

【アプリ名】
TaskFlow Pro

【概要】
チーム向けのタスク管理アプリ。複数ユーザーでリアルタイムにタスクを共有・更新でき、
プロジェクト管理が容易。

【対応プラットフォーム】
iOS / Android（React Native）

【ターゲットユーザー】
中小企業のプロジェクトマネージャー・チームメンバー
```

**Step 2：主要な画面構成を列挙する**

```
【画面一覧】
1. 認証画面
   - ログイン（メール + パスワード）
   - サインアップ（名前 + メール + パスワード + 企業名）

2. ホーム画面
   - 本日のタスク一覧（優先度順）
   - タスクの完了・未完了をトグル
   - プロジェクトフィルタ

3. タスク詳細画面
   - タイトル・説明・期限・優先度
   - 担当者アサイン
   - コメント機能

4. プロジェクト管理画面
   - プロジェクト一覧
   - メンバー管理
   - 新規プロジェクト作成
```

**Step 3：データモデルを指定する**

```
【データベーススキーマ（Supabase PostgreSQL）】

users テーブル
- id (UUID, primary key)
- email (varchar, unique)
- full_name (varchar)
- company_name (varchar)
- created_at (timestamp)

projects テーブル
- id (UUID, primary key)
- user_id (UUID, foreign key)
- name (varchar)
- description (text)
- status (enum: 'active', 'archived')
- created_at (timestamp)

tasks テーブル
- id (UUID, primary key)
- project_id (UUID, foreign key)
- assigned_to (UUID, foreign key users)
- title (varchar)
- description (text)
- priority (enum: 'low', 'medium', 'high')
- status (enum: 'todo', 'in_progress', 'done')
- due_date (date)
- created_at (timestamp)
- updated_at (timestamp)

comments テーブル
- id (UUID, primary key)
- task_id (UUID, foreign key)
- user_id (UUID, foreign key)
- content (text)
- created_at (timestamp)
```

**Step 4：認証とセキュリティを指定する**

```
【認証・セキュリティ】
- 認証：Supabase Auth（メール + パスワード）
- RLS（Row Level Security）：有効化
  * users テーブル：自分のレコードのみ読み書き可能
  * projects テーブル：プロジェクトメンバーのみアクセス可能
  * tasks テーブル：割り当てられたユーザーと作成者がアクセス可能
- Session 管理：React Native AsyncStorage で JWT 管理
```

効果的なプロンプトを書くことで、Rork AI は自動的に上記のすべての要件を満たすコードを生成します。

**Custom Hook の例（useAuth）：**

```javascript
import { useState, useEffect, useCallback } from 'react';
import { supabase } from '../utils/supabaseClient';

export const useAuth = () => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const session = supabase.auth.session();
    setUser(session?.user ?? null);

    const { subscription } = supabase.auth.onAuthStateChange(
      (event, session) => {
        setUser(session?.user ?? null);
        setLoading(false);
      }
    );

    return () => {
      subscription?.unsubscribe();
    };
  }, []);

  const login = useCallback(async (email, password) => {
    setLoading(true);
    setError(null);
    try {
      const { data, error } = await supabase.auth.signInWithPassword({
        email,
        password,
      });
      if (error) throw error;
      setUser(data.user);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }, []);

  const logout = useCallback(async () => {
    setLoading(true);
    try {
      const { error } = await supabase.auth.signOut();
      if (error) throw error;
      setUser(null);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }, []);

  return { user, loading, error, login, logout };
};
```

### FlatList の効率的な使い方（React Native code）

大規模なリストを扱う場合、FlatList の最適化が必須です。

**基本パターン（仮想化・メモ化）：**

```javascript
import React, { useMemo, useCallback } from 'react';
import { FlatList, View, Text, StyleSheet, TouchableOpacity } from 'react-native';

const TaskListScreen = ({ tasks, onTaskPress, onCompleteTask }) => {
  // タスクを優先度でソート（useMemo で再計算を防止）
  const sortedTasks = useMemo(() => {
    const priorityMap = { high: 1, medium: 2, low: 3 };
    return [...tasks].sort(
      (a, b) => priorityMap[a.priority] - priorityMap[b.priority]
    );
  }, [tasks]);

  // キー生成関数
  const keyExtractor = useCallback((item) => item.id, []);

  // リスト項目コンポーネント（React.memo で再レンダー防止）
  const TaskItem = React.memo(({ item }) => (
    <TouchableOpacity
      style={styles.taskItem}

      activeOpacity={0.7}
    >
      <View style={styles.taskContent}>
        <Text style={styles.taskTitle} numberOfLines={1}>
          {item.title}
        </Text>
        <Text style={styles.taskDescription} numberOfLines={2}>
          {item.description}
```

**認証サービスの実装：**

```typescript
import { supabase } from './supabaseClient';

interface SignUpPayload {
  email: string;
  password: string;
  fullName: string;
  companyName: string;
}

interface TaskInsertPayload {
  projectId: string;
  title: string;
  description: string;
  priority: 'low' | 'medium' | 'high';
  dueDate: string;
  assignedTo?: string;
}

export const authService = {
  async signUp(payload: SignUpPayload) {
    // Supabase Auth でユーザー作成
    const { data, error } = await supabase.auth.signUp({
      email: payload.email,
      password: payload.password,
    });

    if (error) throw error;

    // profiles テーブルにプロフィール情報を挿入
    const { error: profileError } = await supabase
      .from('profiles')
      .insert({
        id: data.user.id,
        full_name: payload.fullName,
        company_name: payload.companyName,
      });

    if (profileError) throw profileError;

    return data;
  },

  async signIn(email: string, password: string) {
    const { data, error } = await supabase.auth.signInWithPassword({
      email,
      password,
    });

    if (error) throw error;
    return data;
  },

  async signOut() {
    const { error } = await supabase.auth.signOut();
    if (error) throw error;
  },

  async getCurrentUser() {
    const { data, error } = await supabase.auth.getUser();
    if (error) throw error;
    return data.user;
  },
};

export const taskService = {
  async getTasks(projectId: string) {
    const { data, error } = await supabase
      .from('tasks')
      .select('*')
      .eq('project_id', projectId)
      .order('created_at', { ascending: false });

    if (error) throw error;
    return data;
  },

  async createTask(payload: TaskInsertPayload) {
    const { data, error } = await supabase
      .from('tasks')
      .insert({
        project_id: payload.projectId,
        title: payload.title,
        description: payload.description,
        priority: payload.priority,
        due_date: payload.dueDate,
        assigned_to: payload.assignedTo,
        status: 'todo',
      })
      .select()
      .single();

    if (error) throw error;
    return data;
  },

  async updateTask(
    taskId: string,
    updates: Partial<{
      title: string;
      status: 'todo' | 'in_progress' | 'done';
      priority: 'low' | 'medium' | 'high';
      due_date: string;
    }>
  ) {
    const { data, error } = await supabase
      .from('tasks')
      .update(updates)
      .eq('id', taskId)
      .select()
      .single();

    if (error) throw error;
    return data;
  },

  async deleteTask(taskId: string) {
    const { error } = await supabase
      .from('tasks')
      .delete()
      .eq('id', taskId);

    if (error) throw error;
  },
};
```

### プッシュ通知の実装基礎

**Firebase Cloud Messaging（FCM）の設定：**

```typescript
import { initializeApp } from 'firebase/app';
import {
  getMessaging,
  getToken,
  onMessage,
  Messaging,
} from 'firebase/messaging';

const firebaseConfig = {
  apiKey: process.env.FIREBASE_API_KEY,
  projectId: process.env.FIREBASE_PROJECT_ID,
  messagingSenderId: process.env.FIREBASE_MESSAGING_SENDER_ID,
};

const app = initializeApp(firebaseConfig);
const messaging = getMessaging(app);

export const notificationService = {
  async initializeNotifications() {
    try {
      // FCM トークンを取得
      const token = await getToken(messaging, {
        vapidKey: process.env.FIREBASE_VAPID_KEY,
      });

      if (token) {
        console.log('FCM Token:', token);
        // トークンをサーバーに保存して、ユーザーにプッシュ通知を送信可能にする
        await this.saveFcmToken(token);
      }
    } catch (error) {
      console.error('FCM token error:', error);
    }
  },

  onMessageListener() {
    return new Promise((resolve) => {
      onMessage(messaging, (payload) => {
        console.log('Message received:', payload);
        // フォアグラウンド時の通知処理
        resolve(payload);
      });
    });
  },

  async saveFcmToken(token: string) {
    const user = await authService.getCurrentUser();
    if (!user) return;

    await supabase
      .from('user_fcm_tokens')
      .insert({
        user_id: user.id,
        token,
        device_type: 'mobile',
      });
  },
};

// 起動時に初期化
notificationService.initializeNotifications();

// フォアグラウンド時の通知受信
notificationService.onMessageListener().then((payload) => {
  // UI 表示処理など
});
```

```typescript
// ディープリンクを処理するカスタムフック
export const useDeepLinkHandler = () => {
  const route = useRoute();

  useEffect(() => {
    const handleUrl = ({ url }: { url: string }) => {
      const route = url.replace(/.*?:\/\//g, '');
      const routeName = route.split('/')[0];
      const params = route.split('/').slice(1);

      if (routeName === 'task' && params[0]) {
        // タスク詳細画面へ遷移
        navigation.navigate('TaskDetails', { taskId: params[0] });
      } else if (routeName === 'project' && params[0]) {
        // プロジェクト詳細画面へ遷移
        navigation.navigate('ProjectDetails', { projectId: params[0] });
      }
    };

    const subscription = Linking.addEventListener('url', handleUrl);

    return () => {
      subscription.remove();
    };
  }, []);
};
```
