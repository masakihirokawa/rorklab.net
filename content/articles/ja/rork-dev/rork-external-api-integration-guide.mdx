## はじめに — なぜ API 連携が重要なのか

アプリ開発において、外部サービスのデータを取得・送信する **API 連携** は欠かせないスキルです。天気予報、為替レート、ニュースフィード、AI チャットなど、現代のアプリの多くは外部 API の力を借りて動いています。

Rork を使えば、React Native / Expo ベースのアプリを AI の力で素早く構築できますが、外部 API との連携部分は開発者自身が設計する必要があります。この記事では、Rork アプリで REST API を呼び出す基本的な方法から、認証、エラーハンドリング、ローディング状態の管理まで、初心者にもわかりやすく解説します。

この記事で学べること:

- `fetch` を使った REST API リクエストの基本
- API キーの安全な管理方法
- Bearer トークンやヘッダー認証の実装
- エラーハンドリングとユーザーへのフィードバック
- ローディング UI の実装パターン

## REST API の基礎知識

REST API（Representational State Transfer）は、HTTP プロトコルを通じてデータをやり取りする仕組みです。Rork アプリでは主に以下の 4 つの HTTP メソッドを使います。

- **GET**: データの取得（例: ニュース一覧を取得）
- **POST**: データの送信・作成（例: ユーザー登録）
- **PUT / PATCH**: データの更新（例: プロフィール編集）
- **DELETE**: データの削除（例: 投稿の削除）

API からのレスポンスは通常 JSON 形式で返ってきます。Rork アプリ（React Native）では JavaScript 標準の `fetch` 関数を使ってこれらのリクエストを送信します。

## fetch でデータを取得する基本パターン

まずは最もシンプルな GET リクエストの例を見てみましょう。以下のコードは、公開 API からデータを取得してコンソールに出力します。

```javascript
// 基本的な GET リクエスト
const fetchUsers = async () => {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users');
    
    // レスポンスのステータスコードを確認
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    // JSON としてパース
    const data = await response.json();
    console.log('取得したユーザー数:', data.length);
    // 期待する出力: 取得したユーザー数: 10
    
    return data;
  } catch (error) {
    console.error('データの取得に失敗しました:', error.message);
    return [];
  }
};
```

ポイントは 3 つあります。まず `async/await` で非同期処理を簡潔に書けること、次に `response.ok` でステータスコードをチェックすること、そして `try/catch` でネットワークエラーを捕捉することです。

## Rork コンポーネントでの API 呼び出し

実際の Rork アプリ画面では、`useState` と `useEffect` を組み合わせてデータを取得・表示します。以下は天気情報を取得して表示するコンポーネントの例です。

```javascript
import React, { useState, useEffect } from 'react';
import { View, Text, ActivityIndicator, StyleSheet } from 'react-native';

// 天気情報を表示するコンポーネント
export default function WeatherScreen() {
  const [weather, setWeather] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchWeather = async () => {
      try {
        setLoading(true);
        const response = await fetch(
          'https://api.open-meteo.com/v1/forecast?latitude=35.6762&longitude=139.6503&current_weather=true'
        );

        if (!response.ok) {
          throw new Error('天気データの取得に失敗しました');
        }

        const data = await response.json();
        setWeather(data.current_weather);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchWeather();
  }, []);

  if (loading) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#4A90D9" />
        <Text>天気情報を取得中...</Text>
      </View>
    );
  }

  if (error) {
    return (
      <View style={styles.center}>
        <Text style={styles.errorText}>{error}</Text>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <Text style={styles.title}>東京の天気</Text>
      <Text style={styles.temp}>{weather.temperature}°C</Text>
      <Text>風速: {weather.windspeed} km/h</Text>
    </View>
  );
}
// 期待する出力: 東京の現在の気温と風速が画面に表示される

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  center: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 10 },
  temp: { fontSize: 48, fontWeight: 'bold', color: '#4A90D9' },
  errorText: { color: 'red', fontSize: 16 },
});
```

このパターンでは、`loading`・`error`・`data` の 3 つの状態を管理することで、ユーザーに適切なフィードバックを返しています。

## API キーの安全な管理

多くの外部 API は認証のために API キーを必要とします。Rork アプリでの API キー管理にはいくつかの方法があります。

### 環境変数を使う方法

Expo の環境変数機能を使えば、コードに直接 API キーを書かずに済みます。

```javascript
// .env ファイル（バージョン管理には含めない）
// API_KEY=YOUR_API_KEY

// アプリ内での使用例
const API_KEY = process.env.EXPO_PUBLIC_API_KEY;

const fetchData = async () => {
  const response = await fetch(
    `https://api.example.com/data?key=${API_KEY}`
  );
  const data = await response.json();
  return data;
};
// 期待する出力: API キーを安全に環境変数から読み込んでリクエスト送信
```

### Bearer トークン認証

OAuth 2.0 などのトークンベース認証では、`Authorization` ヘッダーにトークンを含めます。

```javascript
// Bearer トークンを使った認証付きリクエスト
const fetchProtectedData = async (accessToken) => {
  try {
    const response = await fetch('https://api.example.com/protected/resource', {
      method: 'GET',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
    });

    if (response.status === 401) {
      // トークン期限切れの場合はリフレッシュ処理へ
      console.log('認証エラー: トークンを更新してください');
      return null;
    }

    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }

    return await response.json();
  } catch (error) {
    console.error('認証付きリクエスト失敗:', error.message);
    return null;
  }
};
// 期待する出力: 認証が必要なエンドポイントからデータを安全に取得
```

重要なのは、API キーをソースコードにハードコードしないことです。`.env` ファイルを `.gitignore` に追加し、リポジトリにはキーが含まれないようにしましょう。

## POST リクエストでデータを送信する

フォーム送信やユーザー登録など、サーバーにデータを送る場合は POST リクエストを使います。

```javascript
// POST リクエストでデータを送信する例
const createPost = async (title, body) => {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        title: title,
        body: body,
        userId: 1,
      }),
    });

    if (!response.ok) {
      throw new Error(`送信失敗: ${response.status}`);
    }

    const result = await response.json();
    console.log('作成された投稿 ID:', result.id);
    // 期待する出力: 作成された投稿 ID: 101
    
    return result;
  } catch (error) {
    console.error('投稿の作成に失敗:', error.message);
    throw error;
  }
};
```

`Content-Type: application/json` ヘッダーを忘れずに設定し、`body` には `JSON.stringify()` でシリアライズしたデータを渡します。

## エラーハンドリングのベストプラクティス

実運用のアプリでは、ネットワークエラーや API のレート制限など、さまざまなエラーに対応する必要があります。以下は堅牢なエラーハンドリングの実装例です。

```javascript
// 堅牢なエラーハンドリング付き API クライアント
class ApiClient {
  constructor(baseUrl) {
    this.baseUrl = baseUrl;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseUrl}${endpoint}`;
    const config = {
      headers: {
        'Content-Type': 'application/json',
        ...options.headers,
      },
      ...options,
    };

    try {
      const response = await fetch(url, config);

      // ステータスコード別のエラー処理
      if (response.status === 429) {
        // レート制限に達した場合
        const retryAfter = response.headers.get('Retry-After') || 60;
        throw new Error(`レート制限: ${retryAfter}秒後に再試行してください`);
      }

      if (response.status >= 500) {
        throw new Error('サーバーエラーが発生しました。しばらく後に再試行してください');
      }

      if (response.status === 404) {
        throw new Error('リクエストしたデータが見つかりません');
      }

      if (!response.ok) {
        const errorBody = await response.text();
        throw new Error(`API エラー (${response.status}): ${errorBody}`);
      }

      return await response.json();
    } catch (error) {
      if (error.name === 'TypeError' && error.message === 'Network request failed') {
        throw new Error('ネットワークに接続できません。接続状況を確認してください');
      }
      throw error;
    }
  }

  get(endpoint) {
    return this.request(endpoint, { method: 'GET' });
  }

  post(endpoint, data) {
    return this.request(endpoint, {
      method: 'POST',
      body: JSON.stringify(data),
    });
  }
}

// 使い方
const api = new ApiClient('https://api.example.com');
// 期待する出力: ステータスコードに応じた適切なエラーメッセージがスローされる
```

このクラスベースのアプローチにより、アプリ全体で統一されたエラーハンドリングを実現できます。Rork のプロンプトで「API クライアントクラスを作成して」と指示すれば、このようなパターンを自動生成することも可能です。

さらに高度な API 設計パターンについては、[Rork Max × Supabase Edge Functions & Row Level Security セキュア API 設計完全ガイド](/articles/rork-dev/rork-max-supabase-edge-functions-rls-api-design) で詳しく解説しています。

## よくある質問（FAQ）

### Q: Rork で使える外部 API に制限はありますか？

A: Rork は React Native / Expo ベースなので、CORS の制約を受けないネイティブ HTTP リクエストが可能です。Web ブラウザと異なり、ほぼすべての REST API に直接アクセスできます。ただし、HTTPS を使用する API を推奨します。

### Q: API キーをアプリ内に埋め込んでも安全ですか？

A: クライアントサイドに埋め込まれた API キーは、リバースエンジニアリングで抽出される可能性があります。重要な API キーはバックエンド（Supabase Edge Functions など）経由でリクエストを中継し、クライアントに直接露出させないのがベストプラクティスです。

### Q: API のレスポンスが遅い場合、どう対処すればよいですか？

A: まずタイムアウトを設定しましょう。`AbortController` を使えば一定時間後にリクエストをキャンセルできます。また、レスポンスをキャッシュして 2 回目以降のアクセスを高速化したり、ローディングスケルトンを表示してユーザー体験を改善する方法も有効です。

### Q: Rork のプロンプトで API 連携を指示するコツはありますか？

A: 「〇〇 API からデータを取得して一覧表示するコンポーネントを作成して。ローディング状態とエラー表示も含めて」のように、取得するデータ、表示方法、状態管理まで具体的に指示すると、より完成度の高いコードが生成されます。
