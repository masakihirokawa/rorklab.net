Rorkアプリをシミュレータや実機で実行したとき、アプリが起動しない、クラッシュする、画面が正しく表示されないといった問題が発生することがあります。こうした実行時エラーはビルドエラーとは異なり、アプリケーション層で発生するため、デバッグアプローチも異なります。本記事では、iOS/Androidそれぞれの問題パターンと解決方法を体系的に解説します。

## 実行時エラーの診断フロー

アプリが動作しないとき、まず以下の順序で診断します。

1. **ビルドは成功したか？** → コンソールを確認
2. **シミュレータ/デバイスは起動しているか？** → デバイスリストを確認
3. **アプリは起動するか？** → ログ出力を確認
4. **クラッシュしているか？** → スタックトレースを読む
5. **表示は正しいか？** → レイアウトデバッグツールを使う

## iOS特有の問題と解決方法

### 1. iOSアプリが起動しない「App Not Installed」

シミュレータにアプリがインストールされない、または起動直後に落ちる問題です。

**原因**：
- バンドルIDの不一致
- Signing & Capabilities の設定ミス
- 古いシミュレータキャッシュ

**解決方法**：

```bash
# Step 1: Xcodeで確認
# Xcode → [プロジェクト] → Targets → General
# Bundle Identifier が com.yourcompany.appname 形式か確認

# Step 2: シミュレータをリセット
xcrun simctl erase all  # 全シミュレータをリセット
# または特定シミュレータをリセット
xcrun simctl erase "iPhone 15"

# Step 3: DerivedData をクリア
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# Step 4: 再度ビルド＆実行
npm start

# Step 5: Xcodeで実行する場合
open ios/YourApp.xcworkspace
# Cmd + R でビルド＆実行
```

**Signing & Capabilitiesの確認**：

```
Xcode → Targets → Signing & Capabilities
- Team: 開発チーム名が選択されているか
- Bundle Identifier: ユニークな識別子
- Automatic signing: ON に設定
```

### 2. iOSシミュレータでクラッシュ「EXC_BAD_ACCESS」

メモリアクセスエラーでクラッシュします。

**原因**：
- メモリリーク
- null参照
- ネイティブコード境界での型エラー

**解決方法**：

```bash
# Step 1: デバッグシンボルを有効化
# Xcode → Scheme → Edit Scheme → Run
# → Build Configuration: Debug (必須)

# Step 2: Xcodeコンソールでクラッシュログを確認
# Cmd + Shift + Y で コンソールペイン表示
# スタックトレースから問題箇所を特定

# Step 3: Address Sanitizer を有効化
# Xcode → Scheme → Edit Scheme → Run → Diagnostics
# → Address Sanitizer: ON
# より詳細なエラー情報が得られる

# Step 4: TypeScriptの型チェックを厳密に
# tsconfig.json に strict: true を設定
```

コンソール出力の読み方：

```
Thread 0 Crashed:
0   libsystem_kernel.dylib     0x00007fff2xxx xxxx
1   MyApp                      0x00010xxxx+xxx  [file.js:50]
```

行番号の1行目（MyApp）が実際のエラー箇所です。ファイル名と行番号を確認します。

### 3. iOSで「Cannot read property of undefined」

JavaScriptランタイムエラーです。

**原因**：
- 初期化前のオブジェクト参照
- APIレスポンス構造の変更
- 非同期処理のタイミング問題

**解決方法**：

```typescript
// ❌ 悪い例：チェックなし
const userName = user.profile.name;

// ✅ 良い例：安全なアクセス
const userName = user?.profile?.name ?? 'Anonymous';

// または型定義で保護
interface User {
  profile?: {
    name: string;
  };
}

const User: User = {};
const userName = user.profile?.name ?? 'Anonymous';
```

APIレスポンスの検証：

```typescript
// APIコール時は常に型チェック
const fetchUser = async () => {
  const response = await fetch('/api/user');
  if (!response.ok) {
    console.error('API Error:', response.status);
    return;
  }

  const data = await response.json();

  // 必須フィールドが存在するか確認
  if (!data?.id || !data?.name) {
    console.error('Invalid response structure');
    return;
  }

  return data;
};
```

## Android特有の問題と解決方法

### 1. Androidアプリが起動しない「App crashes on launch」

Androidエミュレータでアプリが起動直後に落ちる問題です。

**原因**：
- メモリ不足（エミュレータの割り当てRAM不足）
- マニフェスト設定エラー
- ネイティブコードの互換性問題

**解決方法**：

```bash
# Step 1: エミュレータの再起動
emulator -avd Pixel_6a -wipe-data

# Step 2: Androidのログを確認
adb logcat | grep "FATAL EXCEPTION"
# または Logcat で詳細ログを確認
adb logcat | grep -i "error\|crash"

# Step 3: エミュレータのメモリ割り当てを増加
# Android Studio → Virtual Device Manager
# → 編集 → Memory/RAM: 4GB 以上 を設定

# Step 4: AndroidManifest.xml を確認
# android/app/src/main/AndroidManifest.xml
# <uses-permission> が正しく定義されているか確認
```

AndroidManifest.xml の確認例：

```xml
<manifest>
  <!-- インターネット接続が必要な場合 -->
  <uses-permission android:name="android.permission.INTERNET" />

  <!-- カメラを使う場合 -->
  <uses-permission android:name="android.permission.CAMERA" />

  <!-- アクティビティ定義 -->
  <application>
    <activity
      android:name=".MainActivity"
      android:exported="true">
      <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
      </intent-filter>
    </activity>
  </application>
</manifest>
```

### 2. Androidでメモリ不足「OutOfMemoryError」

特に画像処理やデータ取得でメモリが枯渇します。

**原因**：
- 大きな画像を圧縮なしで読み込み
- メモリリーク（リスナー購読の未削除）
- キャッシュの無限増殖

**解決方法**：

```typescript
// 画像は圧縮して読み込み
import { Image } from 'react-native';

// ❌ 悪い例：大きな画像をそのまま読み込み
<Image
  source={{ uri: 'https://example.com/huge-image.jpg' }}
  style={{ width: 300, height: 300 }}
/>

// ✅ 良い例：キャッシュとメモリ最適化
import FastImage from 'react-native-fast-image';

<FastImage
  source={{ uri: 'https://example.com/huge-image.jpg' }}
  style={{ width: 300, height: 300 }}
  cache={FastImage.cacheControl.moderate}
/>

// useEffect でクリーンアップを忘れない
useEffect(() => {
  const subscription = someEmitter.on('event', handler);

  return () => {
    subscription.remove(); // 重要：リスナーを削除
  };
}, []);
```

メモリ監視：

```bash
# Androidメモリ使用状況をリアルタイム監視
adb shell dumpsys meminfo com.example.myapp

# ヒープメモリの詳細を確認
adb shell am dumpheap com.example.myapp /data/local/tmp/heap.bin
```

### 3. Androidで「Permission denied」エラー

ランタイムパーミッション関連のエラーです。

**原因**：
- AndroidManifest.xml にパーミッション記述なし
- ランタイムパーミッション未要求
- パーミッション名のスペルミス

**解決方法**：

```typescript
import { PermissionsAndroid } from 'react-native';

// ランタイムパーミッション要求（Android 6.0+）
const requestCameraPermission = async () => {
  try {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.CAMERA,
      {
        title: "カメラアクセス許可",
        message: "このアプリはカメラへのアクセスが必要です",
        buttonNeutral: "後で聞く",
        buttonNegative: "キャンセル",
        buttonPositive: "許可",
      },
    );

    if (granted === PermissionsAndroid.RESULTS.GRANTED) {
      console.log("Permission granted");
    } else {
      console.log("Permission denied");
    }
  } catch (err) {
    console.warn(err);
  }
};
```

AndroidManifest.xml（Android 6.0以前のため併記必須）：

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

## 共通の表示・レイアウト問題

### 1. 画面が正しく表示されない（レイアウト崩れ）

**原因**：
- デバイス画面サイズの考慮不足
- SafeAreaInsets 未対応
- Flexbox レイアウトバグ

**解決方法**：

```typescript
import { SafeAreaView, useWindowDimensions } from 'react-native';

// SafeAreaView でノッチ・ホームインジケータ対応
<SafeAreaView style={{ flex: 1 }}>
  <View style={{ flex: 1 }}>
    {/* コンテンツ */}
  </View>
</SafeAreaView>

// レスポンシブレイアウト
const { width, height } = useWindowDimensions();

const styles = StyleSheet.create({
  container: {
    flex: 1,
    width: '100%',      // 相対幅
    padding: width > 600 ? 20 : 10,  // 画面幅に応じた padding
  },
});
```

### 2. 白い画面が表示される（White Screen of Death）

**原因**：
- エントリーポイントのエラー
- RootNavi gator の初期化失敗
- AsyncStorage の初期化遅延

**解決方法**：

```typescript
// ❌ 悪い例：同期的にレンダリング
export default function App() {
  return <NavigationContainer>{/* ... */}</NavigationContainer>;
}

// ✅ 良い例：非同期処理を待つ
import { useState, useEffect } from 'react';

export default function App() {
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    const bootstrapAsync = async () => {
      try {
        // 初期化処理（AsyncStorage, AuthContext など）
        await new Promise(resolve => setTimeout(resolve, 1000));
      } catch (e) {
        console.log(e);
      } finally {
        setIsReady(true);
      }
    };

    bootstrapAsync();
  }, []);

  if (!isReady) {
    return <SplashScreen />;  // ローディング表示
  }

  return <NavigationContainer>{/* ... */}</NavigationContainer>;
}
```

## よくある問題Q&A

**Q1: iOSシミュレータで「Simulator is slower」と出ます**

A: シミュレータを再起動してください：
```bash
xcrun simctl shutdown all
open -a Simulator
```

**Q2: Androidで USB経由のデバッグが繋がりません**

A: デバイスの開発者設定を確認：
```bash
# USB デバッグを有効化（デバイス設定 → 開発者向けオプション）
adb devices  # 接続確認
adb reverse tcp:8081 tcp:8081  # ポート転送
```

**Q3: アプリは起動するが、ボタンが反応しません**

A: Pressability を確認：
```typescript
<Pressable onPress={() => console.log('pressed')}>
  <Text>ボタン</Text>
</Pressable>

// または TouchableOpacity
<TouchableOpacity onPress={handlePress}>
  <Text>ボタン</Text>
</TouchableOpacity>
```

## まとめ

実行時エラーはビルドエラーと異なり、アプリケーション層とデバイス層の複合的な問題が絡みます。診断の重要なポイント：

1. **ログを読む** — コンソール・Logcat で詳細メッセージを確認
2. **プラットフォーム固有の設定を確認** — Manifest（Android）、Signing（iOS）
3. **デバイス・シミュレータをリセット** — キャッシュが邪魔することは多い
4. **型安全性を確保** — TypeScript strict mode で実行時エラーを削減
5. **メモリ管理に注意** — 特に画像処理とリスナー購読の削除
