## Vision Pro と visionOS の基礎知識

空間コンピューティングアプリの開発に入る前に、visionOS特有の概念を理解しておきましょう。

### visionOS のUI階層

```
visionOS UI 階層
──────────────────────────────
Shared Space（共有空間）
  ├── Window（2Dウィンドウ）
  ├── Volume（3Dボリューム）
  └── Full Space（フルイマーシブ空間）
       └── Reality（現実世界との統合）
```

visionOSでは通常のiOSアプリと同じSwiftUIコードが動く場合が多いですが、空間コンピューティングならではの3D要素（Volume・Immersive Scene）を追加することで、没入型の体験が作れます。

### iOSとvisionOSの主な違い

```
項目           iOS          visionOS
───────────────────────────────────────
タッチ操作      タップ        視線+ピンチジェスチャー
ウィンドウ      画面内固定    空間に浮く
3D要素          限定的        RealityKit標準対応
深度センサー    FaceID程度    LiDARスキャン（Pro）
```

struct InteractiveVolumeView: View {
    @State private var isHighlighted = false
    
    var body: some View {
        RealityView { content in
            let box = ModelEntity(
                mesh: .generateBox(size: 0.1),

## 空間オーディオの実装

visionOSの大きな特徴の一つが「空間オーディオ」です。音が3D空間の特定の位置から聞こえる体験を作れます。

```swift
// SpatialAudioManager.swift
import RealityKit
import AVFoundation

class SpatialAudioManager {
    
    /**
     * 3D空間の指定位置から音を再生する
     * - position: 音の発生位置（現実空間の座標）
     * - soundURL: 再生する音声ファイル
     */
    func playAt(position: SIMD3<Float>, soundURL: URL) {
        // RealityKitで空間オーディオエンティティを作成
        let audioEntity = Entity()
        audioEntity.position = position
        
        // 空間オーディオ設定
        let audioResource = try? AudioFileResource.load(
            contentsOf: soundURL,
            configuration: .init()
        )
        
        if let resource = audioResource {
            audioEntity.playAudio(resource)
        }
    }
    
    /**
     * ユーザーの前方1mの位置から環境音を再生
     */
    func playAmbientSound(soundURL: URL) {
        playAt(
            position: SIMD3<Float>(0, 0, -1.0),  // 前方1m
            soundURL: soundURL
        )
    }
}
```

## FAQ

**Q: Vision Proを持っていなくてもvisionOSアプリを開発できますか？**

はい。XcodeのシミュレーターでvisionOSをシミュレートできます。実機デバッグにはVision Proが必要ですが、基本的な開発はシミュレーターで十分です。

**Q: Rork MaxはvisionOS向けコードを正確に生成できますか？**

基本的なWindowベースのUIは正確に生成できます。Immersive Space（フル没入型）は複雑なため、生成後の調整が必要な場合があります。

**Q: visionOSアプリの審査は厳しいですか？**

基本的なガイドラインはiOSと共通ですが、プライバシー（顔認識・空間データ）に関する追加要件があります。HealthKitなどのセンサーデータへのアクセスは特に慎重な扱いが必要です。

**Q: iOSとvisionOSを同時に提出できますか？**

はい。Universalバイナリとして両方のプラットフォームに対応したアプリを一つのApp Store掲載ページで提供できます。

**Q: 開発コストはiOSと比べてどれくらいかかりますか？**

visionOS固有のUI（Immersive Space・空間オーディオ）の実装に追加の工数がかかります。Rork Maxを使えば基本構成は自動生成されるため、iOSの1.3〜1.5倍程度に抑えられます。
