## はじめに

アプリ開発において、フォーム入力はユーザーとの最も重要な接点のひとつです。会員登録、ログイン、プロフィール編集、お問い合わせ、決済情報の入力など、あらゆるシーンでフォームが使われます。

しかし、適切なバリデーション（入力検証）を設計しないと、ユーザーは何が間違っているのかわからずに離脱してしまいます。Rork でアプリを作る際も、フォームの品質はアプリ全体の完成度に直結します。

この記事では、React Native エコシステムで最も人気のあるフォームライブラリ **react-hook-form** と、型安全なスキーマバリデーションライブラリ **zod** を組み合わせて、Rork アプリに堅牢なフォームバリデーションを実装する方法を解説します。

**この記事で学べること:**

- react-hook-form と zod の基本的なセットアップ方法
- リアルタイムバリデーションの実装パターン
- マルチステップフォームの設計と実装
- ユーザーに優しいエラーメッセージの表示方法

## react-hook-form と zod を選ぶ理由

React Native のフォームライブラリにはいくつかの選択肢がありますが、**react-hook-form** が特に優れている点は、再レンダリングの最小化とパフォーマンスの高さです。Formik などの代替ライブラリと比較して、入力フィールドが多いフォームでも軽快に動作します。

**zod** は TypeScript ファーストのスキーマ定義ライブラリで、バリデーションルールを型定義と同時に宣言できます。これにより、フォームの型とバリデーションルールが常に同期した状態を維持できます。

この2つを組み合わせることで、以下のメリットが得られます。

- **型安全**: フォームの値が TypeScript の型で保護される
- **宣言的**: バリデーションルールをスキーマとして一箇所に集約できる
- **高パフォーマンス**: 不要な再レンダリングを抑制し、スムーズな操作感を実現
- **エラーメッセージの一元管理**: zod スキーマにメッセージを含めることで保守性が向上

```bash

```

`@hookform/resolvers` は react-hook-form と zod を接続するアダプターパッケージです。これにより、zod スキーマをそのままフォームのバリデーションルールとして使えるようになります。

## 基本的なフォームバリデーションの実装

### zod スキーマの定義

まず、ユーザー登録フォームのバリデーションスキーマを定義します。

```typescript
// schemas/userRegistration.ts
import { z } from "zod";

// ユーザー登録フォームのバリデーションスキーマ
export const userRegistrationSchema = z.object({
  name: z
    .string()
    .min(1, "名前を入力してください")
    .max(50, "名前は50文字以内で入力してください"),
  email: z
    .string()
    .min(1, "メールアドレスを入力してください")
    .email("正しいメールアドレスの形式で入力してください"),
  password: z
    .string()
    .min(8, "パスワードは8文字以上で入力してください")
    .regex(
      /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/,
      "英大文字・英小文字・数字をそれぞれ1文字以上含めてください"
    ),
  confirmPassword: z.string().min(1, "パスワードを再入力してください"),
}).refine((data) => data.password === data.confirmPassword, {
  message: "パスワードが一致しません",
  path: ["confirmPassword"],
});

// スキーマから型を自動生成
export type UserRegistrationForm = z.infer<typeof userRegistrationSchema>;
```

ポイントは `z.infer` を使って**スキーマから TypeScript の型を自動生成**している点です。スキーマを変更すると型も自動的に追従するため、定義の二重管理を避けられます。

### フォームコンポーネントの作成

```tsx
// components/RegistrationForm.tsx
import React from "react";
import { View, Text, TextInput, TouchableOpacity, StyleSheet } from "react-native";
import { useForm, Controller } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import {
  userRegistrationSchema,
  type UserRegistrationForm,
} from "../schemas/userRegistration";

export default function RegistrationForm() {
  const {
    control,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm<UserRegistrationForm>({
    resolver: zodResolver(userRegistrationSchema),
    defaultValues: {
      name: "",
      email: "",
      password: "",
      confirmPassword: "",
    },
  });

  const onSubmit = async (data: UserRegistrationForm) => {
    // バリデーション通過後の処理
    console.log("送信データ:", data);
    // 期待する出力:
    // { name: "田中太郎", email: "tanaka@example.com",
    //   password: "MyPass123", confirmPassword: "MyPass123" }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>アカウント登録</Text>

      {/* 名前フィールド */}
      <Controller
        control={control}
        name="name"
        render={({ field: { onChange, onBlur, value } }) => (
          <View style={styles.fieldContainer}>
            <Text style={styles.label}>名前</Text>
            <TextInput
              style={[styles.input, errors.name && styles.inputError]}
              onBlur={onBlur}
              onChangeText={onChange}
              value={value}
              placeholder="山田太郎"
            />
            {errors.name && (
              <Text style={styles.errorText}>{errors.name.message}</Text>
            )}
          </View>
        )}
      />

      {/* メールアドレスフィールド */}
      <Controller
        control={control}
        name="email"
        render={({ field: { onChange, onBlur, value } }) => (
          <View style={styles.fieldContainer}>
            <Text style={styles.label}>メールアドレス</Text>
            <TextInput
              style={[styles.input, errors.email && styles.inputError]}
              onBlur={onBlur}
              onChangeText={onChange}
              value={value}
              placeholder="example@email.com"
              keyboardType="email-address"
              autoCapitalize="none"
            />
            {errors.email && (
              <Text style={styles.errorText}>{errors.email.message}</Text>
            )}
          </View>
        )}
      />

      {/* 送信ボタン */}
      <TouchableOpacity
        style={[styles.button, isSubmitting && styles.buttonDisabled]}
        onPress={handleSubmit(onSubmit)}
        disabled={isSubmitting}
      >
        <Text style={styles.buttonText}>
          {isSubmitting ? "送信中..." : "登録する"}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 24 },
  title: { fontSize: 24, fontWeight: "bold", marginBottom: 24 },
  fieldContainer: { marginBottom: 16 },
  label: { fontSize: 14, fontWeight: "600", marginBottom: 4, color: "#374151" },
  input: {
    borderWidth: 1,
    borderColor: "#D1D5DB",
    borderRadius: 8,
    padding: 12,
    fontSize: 16,
  },
  inputError: { borderColor: "#EF4444" },
  errorText: { color: "#EF4444", fontSize: 12, marginTop: 4 },
  button: {
    backgroundColor: "#6366F1",
    borderRadius: 8,
    padding: 16,
    alignItems: "center",
    marginTop: 8,
  },
  buttonDisabled: { opacity: 0.6 },
  buttonText: { color: "#FFFFFF", fontSize: 16, fontWeight: "bold" },
});
```

このコードでは `Controller` コンポーネントを使って react-hook-form と React Native の `TextInput` を接続しています。`zodResolver` がスキーマに基づくバリデーションを自動的に実行し、エラーがあれば `errors` オブジェクトに格納します。

## リアルタイムバリデーションの実装

デフォルトでは `onSubmit` 時にバリデーションが走りますが、ユーザー体験を向上させるにはリアルタイムバリデーションが効果的です。

```tsx
const {
  control,
  handleSubmit,
  formState: { errors },
} = useForm<UserRegistrationForm>({
  resolver: zodResolver(userRegistrationSchema),
  mode: "onBlur",      // フォーカスが外れた時にバリデーション
  reValidateMode: "onChange", // エラー後は入力のたびに再検証
});
```

`mode` オプションの選択肢と推奨パターンは以下のとおりです。

- **`onSubmit`**（デフォルト）: 送信時のみ検証。シンプルなフォーム向き
- **`onBlur`**: フィールドからフォーカスが外れた時に検証。推奨パターン
- **`onChange`**: 入力のたびに検証。パスワード強度メーターなど特定のフィールドに適している
- **`onTouched`**: 最初のフォーカスアウト後、以降は入力のたびに検証

多くの場合 `mode: "onBlur"` と `reValidateMode: "onChange"` の組み合わせが最適です。ユーザーが入力中にエラーが表示されることなく、フォーカスを移動した後にフィードバックが得られるためストレスが少なくなります。

## マルチステップフォームの設計

会員登録やプロフィール設定など、入力項目が多い場合はマルチステップフォームが有効です。

```tsx
// hooks/useMultiStepForm.ts
import { useState, useCallback } from "react";

export function useMultiStepForm(totalSteps: number) {
  const [currentStep, setCurrentStep] = useState(0);

  const next = useCallback(() => {
    setCurrentStep((prev) => Math.min(prev + 1, totalSteps - 1));
  }, [totalSteps]);

  const back = useCallback(() => {
    setCurrentStep((prev) => Math.max(prev - 1, 0));
  }, []);

  const goTo = useCallback((step: number) => {
    setCurrentStep(Math.max(0, Math.min(step, totalSteps - 1)));
  }, [totalSteps]);

  return {
    currentStep,
    totalSteps,
    isFirstStep: currentStep === 0,
    isLastStep: currentStep === totalSteps - 1,
    progress: (currentStep + 1) / totalSteps,
    next,
    back,
    goTo,
  };
}
```

各ステップのスキーマを個別に定義し、ステップごとにバリデーションを実行する設計にすることで、ユーザーが入力を段階的に進められるようになります。

```typescript
// schemas/multiStepProfile.ts
import { z } from "zod";

// Step 1: 基本情報
export const basicInfoSchema = z.object({
  displayName: z.string().min(1, "表示名を入力してください"),
  bio: z.string().max(200, "自己紹介は200文字以内で入力してください").optional(),
});

// Step 2: 連絡先
export const contactSchema = z.object({
  phone: z
    .string()
    .regex(/^0\d{9,10}$/, "正しい電話番号の形式で入力してください")
    .optional()
    .or(z.literal("")),
  website: z.string().url("正しいURLを入力してください").optional().or(z.literal("")),
});

// Step 3: 設定
export const preferencesSchema = z.object({
  language: z.enum(["ja", "en"], { required_error: "言語を選択してください" }),
  notifications: z.boolean().default(true),
});
```

このようにステップごとにスキーマを分離することで、各ステップで必要な入力だけをバリデーションし、不要なエラー表示を防ぐことができます。

## よくあるエラーと対処法

### TextInput の制御コンポーネント警告

`Controller` の `value` に `undefined` が渡されると、React Native が非制御コンポーネントとして扱い警告が出ることがあります。`defaultValues` で空文字列 `""` を必ず指定しましょう。

### キーボードがフォームを隠す問題

React Native では `KeyboardAvoidingView` でフォーム全体をラップし、iOS では `behavior="padding"`、Android では `behavior="height"` を設定するのが基本です。

```tsx
import { KeyboardAvoidingView, Platform } from "react-native";

<KeyboardAvoidingView
  behavior={Platform.OS === "ios" ? "padding" : "height"}
  style={{ flex: 1 }}
>
  {/* フォームコンポーネント */}
</KeyboardAvoidingView>
```

### 非同期バリデーション（メールアドレスの重複チェック等）

サーバーとの通信が必要なバリデーションは、zod の `.refine()` に非同期関数を渡すことで実現できます。

```typescript
const emailSchema = z.object({
  email: z
    .string()
    .email("正しいメールアドレスを入力してください")
    .refine(async (email) => {
      // サーバーに重複チェックを問い合わせ
      const response = await fetch(`/api/check-email?email=${email}`);
      const { available } = await response.json();
      return available;
    }, "このメールアドレスは既に使用されています"),
});
```

## FAQ

### Q: react-hook-form と Formik はどちらを使うべきですか？

パフォーマンスを重視するなら react-hook-form をおすすめします。react-hook-form は非制御コンポーネントベースで再レンダリングを最小化するため、入力フィールドが多いフォームでもスムーズに動作します。Formik は制御コンポーネントベースのため、入力のたびにフォーム全体が再レンダリングされる場合があります。

### Q: zod 以外のバリデーションライブラリ（Yup など）も使えますか？

はい、`@hookform/resolvers` は Yup、Joi、Superstruct など複数のライブラリに対応しています。ただし、TypeScript との親和性が最も高いのは zod です。`z.infer` による型推論が非常に強力で、スキーマと型の二重管理を完全に排除できます。

### Q: マルチステップフォームで前のステップの入力値を保持するにはどうすればよいですか？

react-hook-form の `useForm` はコンポーネントがアンマウントされても値を保持する `shouldUnregister: false`（デフォルト）の設定があります。そのため、ステップを切り替えても入力値は自動的に保持されます。ステップ間で共有したい場合は、親コンポーネントで `useForm` を呼び出し、各ステップに `control` を props として渡す設計が効果的です。

## まとめ

Rork アプリにおけるフォームバリデーションは、react-hook-form と zod の組み合わせによって、型安全かつ高パフォーマンスな実装が可能です。

重要なポイントを振り返ります。zod スキーマで型とバリデーションルールを一元管理すること、`mode: "onBlur"` でユーザーに優しいタイミングでフィードバックを返すこと、マルチステップフォームではステップごとにスキーマを分離して段階的にバリデーションすること、そして `KeyboardAvoidingView` やエラーメッセージのスタイリングで入力体験を向上させることが、堅牢なフォームを作るための鍵です。

フォームはアプリの「玄関口」です。ここでの体験がアプリ全体の印象を決めると言っても過言ではありません。Rork の AI コード生成と合わせて活用し、品質の高い入力体験を実現してみてください。
