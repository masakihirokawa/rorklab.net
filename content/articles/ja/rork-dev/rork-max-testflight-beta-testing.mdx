## TestFlight とは

TestFlight は Apple が提供する公式のベータテスト配布プラットフォームです。App Store に正式公開する前に、限られたテスターにアプリを配布してフィードバックを収集できます。

Rork Max で開発したネイティブ Swift アプリは、クラウドコンパイル機能を通じて IPA ファイルを生成し、そのまま TestFlight にアップロードできます。Xcode を手元にインストールする必要はありません。

## TestFlight を使うメリット

ベータテストを行うことで、正式リリース前に多くの問題を発見できます。実際のユーザーが様々なデバイスや OS バージョンで使うことで、開発中には気づかなかったクラッシュや UI の崩れが見つかることは珍しくありません。

TestFlight には内部テストと外部テストの2種類があります。内部テストは App Store Connect のチームメンバー（最大100名）に即座に配布でき、Apple の審査は不要です。外部テストは最大10,000名のテスターに配布できますが、初回ビルドは Apple によるベータアプリレビューを通過する必要があります。

## 前提条件

TestFlight でベータ配布を行うには、以下が必要です。

**Apple Developer Program** への登録が必須です。年間 $99（USD）の費用がかかります。個人開発者でも法人でも登録できます。

**Rork Max サブスクリプション** が必要です。Rork Max のクラウドコンパイル機能を使ってネイティブ Swift アプリをビルドします。

**App Store Connect** でアプリの登録が完了していること。Bundle ID の設定とアプリ情報の基本入力を済ませておきます。

## Rork Max からビルドを生成する

Rork Max でアプリの開発がある程度進んだら、ベータテスト用のビルドを生成します。

まず Rork Max のエディタでアプリのバージョン番号とビルド番号を確認します。TestFlight にアップロードするたびにビルド番号を上げる必要があるため、`1.0.0 (1)` のように管理しましょう。

Rork Max の「Build」ボタンをクリックすると、クラウド Mac 上で Swift コードがコンパイルされます。ビルドが成功すると IPA ファイルがダウンロード可能になります。

```
Rork Max エディタ
  → Build ボタン
  → クラウドコンパイル開始
  → IPA ファイル生成（数分）
  → ダウンロード
```

ビルドエラーが発生した場合は、Rork Max の「Fix Now」機能を使って自動修正を試みることができます。多くの場合、依存関係の解決やコード上の小さなミスが原因です。

## App Store Connect でアプリを設定する

[App Store Connect](https://appstoreconnect.apple.com) にサインインし、「マイ App」からアプリを選択します。まだアプリを登録していない場合は「新規 App」から作成してください。

アプリの基本情報として、アプリ名、プライマリ言語、Bundle ID、SKU を入力します。Bundle ID は Rork Max のプロジェクト設定で指定したものと一致させる必要があります。

## ビルドを TestFlight にアップロードする

IPA ファイルを TestFlight にアップロードするには、いくつかの方法があります。

**Transporter アプリ**（macOS 用）を使う方法が最も簡単です。Mac App Store から無料でダウンロードでき、IPA ファイルをドラッグ＆ドロップするだけでアップロードできます。

**altool コマンドライン** を使う方法もあります。CI/CD パイプラインに組み込む場合はこちらが便利です。

```bash
xcrun altool --upload-app \
  --type ios \
  --file YourApp.ipa \
  --apiKey YOUR_API_KEY \
  --apiIssuer YOUR_ISSUER_ID
```

アップロードが完了すると、Apple による自動処理（バイナリの検証、暗号化チェック）が行われます。通常5〜15分で処理が完了し、App Store Connect の TestFlight タブにビルドが表示されます。

## 内部テストの設定

内部テストは最速でテスターにアプリを届ける方法です。

App Store Connect の「TestFlight」タブを開き、「内部テスト」セクションで新しいグループを作成します。グループ名は「Core Team」や「開発チーム」など分かりやすい名前をつけましょう。

テスターを追加するには、App Store Connect のユーザーとして登録されている必要があります。チームメンバーの Apple ID メールアドレスを入力して招待します。

テスターはメールで招待を受け取り、iPhone や iPad の TestFlight アプリからベータ版をインストールできます。内部テストのビルドは Apple の審査を受けずに配布できるため、アップロード後すぐにテスターが利用可能です。

## 外部テストの設定

より広い範囲のテスターにアプリを配布したい場合は、外部テストを利用します。

「外部テスト」セクションで新しいグループを作成し、テスターのメールアドレスを追加します。外部テスターは App Store Connect のユーザーである必要はなく、メールアドレスだけで招待できます。

外部テストでは、テスト情報として以下を入力します。

**テストの説明**: テスターに何をテストしてほしいかを明確に記載します。「ログイン画面のUIを確認してください」「新しいカート機能を試してください」など具体的な指示が効果的です。

**フィードバック用メールアドレス**: テスターからのフィードバックを受け取るメールアドレスを設定します。

**プライバシーポリシー URL**: 外部テストでは必須です。アプリがどのようなデータを収集するかを説明するページの URL を設定します。

初回の外部テストビルドは Apple によるベータアプリレビューを通過する必要があります。レビューは通常24〜48時間で完了します。

## 公開リンクでテスターを募集する

外部テストではパブリックリンクを生成できます。このリンクを SNS やウェブサイトで共有すれば、メールアドレスを個別に登録しなくてもテスターを募集できます。

パブリックリンクには参加上限を設定できるため、テスターの数をコントロールできます。例えば最初は100名に制限し、フィードバックの管理が追いつく範囲で徐々に増やすといった運用が可能です。

## フィードバックの収集と活用

TestFlight の大きな利点は、テスターからのフィードバックを構造化された形で収集できることです。

テスターは TestFlight アプリ内から直接フィードバックを送信できます。スクリーンショットを添付したり、テキストで問題点を報告したりできます。送信されたフィードバックは App Store Connect の TestFlight セクションで一覧表示されます。

クラッシュレポートも自動的に収集されます。テスターのデバイスでクラッシュが発生すると、そのログが App Store Connect に送信されます。クラッシュログを分析することで、特定のデバイスや OS バージョンで発生する問題を特定できます。

フィードバックを受け取ったら、Rork Max で修正を行い、新しいビルドを生成して再度 TestFlight にアップロードします。このサイクルを繰り返すことで、アプリの品質を段階的に向上させていきます。

## ベータテストのベストプラクティス

効果的なベータテストを行うために、いくつかのポイントを押さえておきましょう。

**テスト計画を立てる**: 何をテストするのか、どのような環境でテストするのかを事前に決めておきます。全機能を一度にテストするのではなく、リリースごとにフォーカスするエリアを絞ると効率的です。

**多様なデバイスでテストする**: iPhone SE のような小画面デバイスから iPad Pro のような大画面デバイスまで、様々なスクリーンサイズでテストすることが重要です。Rork Max の React Native ベースのアプリも、Swift ネイティブアプリも、デバイスによってレイアウトが異なる場合があります。

**テスターとのコミュニケーション**: テストの目的や期限を明確に伝え、定期的にアップデートを配信します。テスターのモチベーションを維持するために、報告された問題が修正されたことをフィードバックしましょう。

**段階的なロールアウト**: 最初は内部テスト（5〜10名）で大きなバグを潰し、その後外部テスト（50〜100名）で幅広いフィードバックを収集する2段階のアプローチが効果的です。

## TestFlight から App Store 公開へ

ベータテストで十分な品質が確認できたら、同じビルドをそのまま App Store に提出できます。TestFlight にアップロードしたビルドは App Store Connect のリリース画面でも選択可能です。

Rork Max の2クリック App Store 公開機能と組み合わせれば、TestFlight でのテスト → App Store への提出までをスムーズに進められます。

## まとめ

Rork Max と TestFlight を組み合わせることで、Xcode を持っていなくても本格的なベータテストワークフローを構築できます。クラウドコンパイルでビルドを生成し、TestFlight で配布、フィードバックを収集して修正 — このサイクルを回すことで、App Store 公開前にアプリの品質を大幅に向上させることができます。

個人開発者であっても、友人や家族に内部テストで配布するだけで、自分では気づかなかった問題が見つかることは多々あります。正式リリース前のこの一手間が、ユーザー体験の質を決定的に左右します。
