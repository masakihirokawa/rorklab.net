## Rork MaxとHealthKitで実現するネイティブヘルスケアアプリ

iOSアプリ開発においてヘルスケアデータの活用はますます重要になっています。Apple Watchの普及とともに、歩数・心拍数・睡眠品質・消費カロリーといったデータをアプリに組み込みたいという需要は急速に高まっています。

従来、HealthKitを使ったアプリ開発には**XcodeとSwiftの深い知識**が必要でした。しかしRork Maxを使えば、AIへの自然言語プロンプトだけでネイティブSwift + HealthKit統合アプリを生成できます。本記事では、具体的なプロンプト設計からHealthKitデータ取得のコード解説、UIの仕上げ方まで詳しく解説します。

## Rork MaxがHealthKitと相性がいい理由

Rork Maxは**ネイティブSwiftUI**コードを生成します。これはReact Nativeベースの従来のRorkとは根本的に異なります。HealthKitはApple純正フレームワークであり、サードパーティブリッジを必要としないネイティブ環境での動作が前提です。

Rork Maxが内部で使用するAI（Claude Opus）は、HealthKitの`HKHealthStore`、`HKSampleType`、`HKQuery`といったAPIを正確に理解しています。そのため、以下のような複雑なヘルスデータ処理を自動的に生成できます。

- 歩数データのリアルタイム取得（`HKQuantityTypeIdentifier.stepCount`）
- 心拍数の履歴クエリ（`HKSampleQuery`）
- 睡眠分析データの取得（`HKCategoryTypeIdentifier.sleepAnalysis`）
- ヘルスデータへのアクセス権限リクエスト

## HealthKit権限の設定

HealthKitアプリで最初に実装すべきは**権限リクエスト**です。Rork Maxが生成するコードの典型的なパターンを見てみましょう。

### Info.plistの設定

```xml
<key>NSHealthShareUsageDescription</key>
<string>フィットネスデータを表示するためにHealthKitへのアクセスが必要です</string>
<key>NSHealthUpdateUsageDescription</key>
<string>ワークアウトデータを記録するためにHealthKitへの書き込みアクセスが必要です</string>
```

### SwiftUIでの権限リクエスト実装

```swift
import HealthKit
import SwiftUI

class HealthKitManager: ObservableObject {
    let healthStore = HKHealthStore()

    @Published var isAuthorized = false
    @Published var stepCount: Double = 0
    @Published var heartRate: Double = 0
    @Published var sleepHours: Double = 0

    // 読み取り権限を要求するデータタイプ
    let typesToRead: Set<HKObjectType> = [
        HKQuantityType.quantityType(forIdentifier: .stepCount)!,
        HKQuantityType.quantityType(forIdentifier: .heartRate)!,
        HKCategoryType.categoryType(forIdentifier: .sleepAnalysis)!
    ]

    func requestAuthorization() async {
        guard HKHealthStore.isHealthDataAvailable() else { return }

        do {
            try await healthStore.requestAuthorization(
                toShare: [],
                read: typesToRead
            )
            await MainActor.run {
                self.isAuthorized = true
            }
            await fetchAllHealthData()
        } catch {
            print("HealthKit authorization failed: \(error)")
        }
    }
}
```

## 歩数データのリアルタイム取得

歩数データは`HKStatisticsQuery`を使って今日のデータを取得するのが一般的です。

```swift
func fetchTodayStepCount() async {
    let stepType = HKQuantityType.quantityType(forIdentifier: .stepCount)!

    // 今日の開始時刻から現在までのデータを取得
    let startOfDay = Calendar.current.startOfDay(for: Date())
    let predicate = HKQuery.predicateForSamples(
        withStart: startOfDay,
        end: Date(),
        options: .strictStartDate
    )

    let query = HKStatisticsQuery(
        quantityType: stepType,
        quantitySamplePredicate: predicate,
        options: .cumulativeSum
    ) { _, result, error in
        guard let result = result, let sum = result.sumQuantity() else { return }

        let steps = sum.doubleValue(for: .count())
        DispatchQueue.main.async {
            self.stepCount = steps
        }
    }

    healthStore.execute(query)
}
```

このコードはRork Maxが自動生成するコードそのものです。Rork Maxは`HKStatisticsQuery`の非同期パターンを正確に実装し、`DispatchQueue.main.async`でUIスレッドへの安全な更新も行います。

## 過去7日間のデータグラフを表示する

Apple純正の`Charts`フレームワークを使えば、美しいグラフを簡単に実装できます。

### データモデルの定義

```swift
struct DailyStepData: Identifiable {
    let id = UUID()
    let date: Date
    let steps: Double

    var dayLabel: String {
        let formatter = DateFormatter()
        formatter.dateFormat = "E" // "月", "火" など
        formatter.locale = Locale(identifier: "ja_JP")
        return formatter.string(from: date)
    }
}
```

### SwiftUI ChartsでのグラフUI

```swift
import Charts

struct StepHistoryChart: View {
    @ObservedObject var healthManager: HealthKitManager

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text("過去7日間の歩数")
                .font(.headline)
                .foregroundColor(.secondary)

            Chart(healthManager.weeklySteps) { data in
                BarMark(
                    x: .value("曜日", data.dayLabel),
                    y: .value("歩数", data.steps)
                )
                .foregroundStyle(
                    LinearGradient(
                        colors: [.blue, .cyan],
                        startPoint: .bottom,
                        endPoint: .top
                    )
                )
                .cornerRadius(6)
            }
            .frame(height: 160)
            .chartYAxis {
                AxisMarks(position: .leading) { value in
                    AxisGridLine()
                    AxisValueLabel {
                        if let steps = value.as(Double.self) {
                            Text("\(Int(steps / 1000))k")
                                .font(.caption2)
                        }
                    }
                }
            }
        }
        .padding()
        .background(Color(.systemBackground))
        .cornerRadius(16)
        .shadow(color: .black.opacity(0.08), radius: 8, x: 0, y: 2)
    }
}
```

## メインダッシュボードのUI設計

Rork Maxへのプロンプトで「リング状プログレス」を指定すると、Apple Watchのアクティビティリングに似たUIを生成してくれます。

```swift
struct HealthDashboardView: View {
    @StateObject private var healthManager = HealthKitManager()

    // 目標歩数（1日10,000歩）
    let stepGoal: Double = 10000

    var stepProgress: Double {
        min(healthManager.stepCount / stepGoal, 1.0)
    }

    var body: some View {
        ScrollView {
            VStack(spacing: 20) {
                // ヘッダー
                HStack {
                    VStack(alignment: .leading) {
                        Text("Today's Activity")
                            .font(.title2.bold())
                        Text(Date(), style: .date)
                            .foregroundColor(.secondary)
                    }
                    Spacer()
                }
                .padding(.horizontal)

                // リング型プログレス
                ZStack {
                    Circle()
                        .stroke(Color.blue.opacity(0.2), lineWidth: 20)
                    Circle()
                        .trim(from: 0, to: stepProgress)
                        .stroke(
                            LinearGradient(
                                colors: [.blue, .cyan],
                                startPoint: .topLeading,
                                endPoint: .bottomTrailing
                            ),
                            style: StrokeStyle(lineWidth: 20, lineCap: .round)
                        )
                        .rotationEffect(.degrees(-90))
                        .animation(.easeInOut(duration: 1.0), value: stepProgress)

                    VStack {
                        Text("\(Int(healthManager.stepCount))")
                            .font(.system(size: 36, weight: .bold, design: .rounded))
                        Text("歩")
                            .foregroundColor(.secondary)
                    }
                }
                .frame(width: 200, height: 200)

                // メトリクスカード
                LazyVGrid(columns: [GridItem(.flexible()), GridItem(.flexible())], spacing: 16) {
                    MetricCard(
                        title: "心拍数",
                        value: "\(Int(healthManager.heartRate))",
                        unit: "BPM",
                        icon: "heart.fill",
                        color: .red
                    )
                    MetricCard(
                        title: "睡眠",
                        value: String(format: "%.1f", healthManager.sleepHours),
                        unit: "時間",
                        icon: "bed.double.fill",
                        color: .purple
                    )
                }
                .padding(.horizontal)

                // 週間グラフ
                StepHistoryChart(healthManager: healthManager)
                    .padding(.horizontal)
            }
            .padding(.top)
        }
        .task {
            await healthManager.requestAuthorization()
        }
    }
}
```

## プッシュ通知との組み合わせ

HealthKitデータとプッシュ通知を組み合わせると、ユーザーの活動状況に応じたパーソナライズド通知が実現できます。

```swift
import UserNotifications

func scheduleStepGoalNotification(currentSteps: Double) {
    let remaining = 10000 - currentSteps
    guard remaining > 0 && remaining <= 2000 else { return }

    let content = UNMutableNotificationContent()
    content.title = "目標達成まであと少し！"
    content.body = "今日の残り \(Int(remaining)) 歩で目標達成です 🏃"
    content.sound = .default

    // 1時間後に通知
    let trigger = UNTimeIntervalNotificationTrigger(timeInterval: 3600, repeats: false)
    let request = UNNotificationRequest(
        identifier: "step_goal_reminder",
        content: content,
        trigger: trigger
    )

    UNUserNotificationCenter.current().add(request)
}
```

## Rork Maxでの生成時の注意点

HealthKitアプリをRork Maxで構築する際に気をつけるべきポイントをまとめます。

**1. 権限説明文を詳細に記述する**
Info.plistの`NSHealthShareUsageDescription`は審査リジェクトの原因になりやすい項目です。プロンプトに「ユーザーの歩数と心拍数データをフィットネストラッキングのために読み取る」など目的を明記すると、適切な文言を生成してくれます。

**2. シミュレータでの制限を理解する**
HealthKitのデータはiOSシミュレータでは限定的にしかテストできません。Rork Maxが生成するコードには`HKHealthStore.isHealthDataAvailable()`チェックが含まれますが、実機テストは必須です。

**3. App Storeの審査要件**
HealthKitを使うアプリはAppleの審査が厳格です。「ヘルスデータを第三者に販売しない」という旨の宣言が必要なため、プロンプトに「ヘルスデータは端末内のみで使用し外部送信しない」と明示しましょう。
