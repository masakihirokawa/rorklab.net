## Rork で Bluetooth LE アプリを作る

近年、スマートウォッチ・血圧計・フィットネスバンド・スマートロックなど、Bluetooth LE（BLE）で通信するデバイスが急速に普及しています。こうした IoT デバイスと連携するアプリを Rork で作るには、**react-native-ble-plx** を活用するのが最短ルートです。

この記事では、Bluetooth LE の基礎知識から、Rork プロジェクトへのライブラリ導入、実際のデバイススキャン・接続・データ読み書きまでを一通り解説します。BLE 連携アプリを初めて作る方でも、手順通りに進めることで動くコードを手に入れられる内容にしています。

## Bluetooth LE の基礎知識

BLE アプリを開発する前に、以下の概念を押さえておきましょう。

**GATT（Generic Attribute Profile）** とは BLE のデータ通信規格で、デバイス間でデータを交換するための仕組みを定義します。GATT では「サービス」と「キャラクタリスティック」という2層構造でデータを管理します。

- **サービス（Service）**: 機能のグループ。例えば「心拍数サービス（UUID: 0x180D）」のように、特定の機能をまとめたもの。
- **キャラクタリスティック（Characteristic）**: 実際のデータ。サービスの中に複数存在し、値の読み取り（Read）・書き込み（Write）・通知（Notify）などの操作が可能。

大半の BLE デバイスは、Bluetooth SIG が標準化した UUID を使っています。独自デバイスの場合はメーカー固有の UUID を使います。

## 環境準備

Rork プロジェクトで BLE 機能を追加するには、以下が必要です。

- Rork または Rork Max の有効なプロジェクト
- iOS 実機（BLE はシミュレーターでは動作しません）
- Android 実機（BLE テスト用）
- `react-native-ble-plx` ライブラリ

まず、Rork の AI プロンプトで次のように依頼してセットアップを始めましょう。

```
react-native-ble-plx をインストールして、
Bluetooth LE デバイスのスキャンと接続ができるように設定してください。
iOS と Android の両方に対応させてください。
```

Rork が自動で依存関係の追加と iOS/Android の権限設定を行ってくれます。

## パーミッション設定

BLE を使うには、iOS・Android それぞれで権限の宣言が必要です。Rork が自動設定しますが、内容を理解しておくと後のデバッグに役立ちます。

**iOS（Info.plist）**

```xml
<!-- Bluetooth の使用目的を説明するメッセージ -->
<key>NSBluetoothAlwaysUsageDescription</key>
<string>スマートデバイスと連携するために Bluetooth を使用します</string>
<key>NSBluetoothPeripheralUsageDescription</key>
<string>Bluetooth デバイスと通信するために必要です</string>
```

**Android（AndroidManifest.xml）**

```xml
<!-- Android 12 以降 -->
<uses-permission android:name="android.permission.BLUETOOTH_SCAN" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />
<!-- Android 11 以前 -->
<uses-permission android:name="android.permission.BLUETOOTH" />
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
```

## BLE マネージャーの初期化

プロジェクトのルートで BleManager を初期化します。アプリ全体で1つのインスタンスを使い回すため、シングルトンパターンが推奨されます。

```typescript
import { BleManager, Device } from 'react-native-ble-plx';

// アプリ起動時に一度だけ作成する
const bleManager = new BleManager();

// Bluetooth の状態変化を監視する
const subscription = bleManager.onStateChange((state) => {
  if (state === 'PoweredOn') {
    console.log('Bluetooth が有効になりました');
    subscription.remove(); // 一度検知したら解除
  }
}, true);
```

## デバイスのスキャン（検出）

近くの BLE デバイスを検索するには `startDeviceScan` を使います。スキャンはバッテリーを消費するため、目的のデバイスが見つかり次第停止するのがベストプラクティスです。

```typescript
import { useState, useEffect } from 'react';
import { BleManager, Device } from 'react-native-ble-plx';

const bleManager = new BleManager();

export function useBLEScanner() {
  const [devices, setDevices] = useState<Device[]>([]);
  const [isScanning, setIsScanning] = useState(false);

  const startScan = () => {
    setDevices([]);
    setIsScanning(true);

    // 第1引数: スキャン対象のサービス UUID（null = すべてのデバイスを検出）
    // 第2引数: オプション（allowDuplicates: false で同一デバイスの重複を防ぐ）
    bleManager.startDeviceScan(null, { allowDuplicates: false }, (error, device) => {
      if (error) {
        console.error('スキャンエラー:', error);
        setIsScanning(false);
        return;
      }
      if (device && device.name) {
        setDevices((prev) => {
          const exists = prev.find((d) => d.id === device.id);
          return exists ? prev : [...prev, device];
        });
      }
    });

    // 10秒後に自動停止
    setTimeout(() => {
      bleManager.stopDeviceScan();
      setIsScanning(false);
    }, 10000);
  };

  const stopScan = () => {
    bleManager.stopDeviceScan();
    setIsScanning(false);
  };

  return { devices, isScanning, startScan, stopScan };
}
```

## デバイスへの接続とデータ読み取り

スキャンで見つけたデバイスに接続し、データを読み取る実装例です。ここでは汎用的な心拍数センサーを例に解説します。

```typescript
// 心拍数サービスの標準 UUID（Bluetooth SIG 定義）
const HEART_RATE_SERVICE = '0000180d-0000-1000-8000-00805f9b34fb';
const HEART_RATE_CHARACTERISTIC = '00002a37-0000-1000-8000-00805f9b34fb';

async function connectAndReadHeartRate(device: Device): Promise<number> {
  try {
    // デバイスに接続（タイムアウト: 5秒）
    const connectedDevice = await device.connect({ timeout: 5000 });
    console.log('接続成功:', connectedDevice.name);

    // サービスとキャラクタリスティックを探索する
    await connectedDevice.discoverAllServicesAndCharacteristics();

    // 心拍数の値を読み取る
    const characteristic = await connectedDevice.readCharacteristicForService(
      HEART_RATE_SERVICE,
      HEART_RATE_CHARACTERISTIC
    );

    // Base64 エンコードされたバイト列をデコード
    const rawData = Buffer.from(characteristic.value ?? '', 'base64');
    // 心拍数フォーマット: 最初のバイトがフラグ、2バイト目が心拍数
    const heartRate = rawData[1];
    console.log('心拍数:', heartRate, 'bpm');

    return heartRate;
  } catch (error) {
    console.error('接続またはデータ読み取りエラー:', error);
    throw error;
  }
}
```

## リアルタイム通知の受信（Notify）

血圧計や体温計など多くのデバイスは、値が変化したときにリアルタイムで通知を送ってきます。`monitorCharacteristicForService` を使って購読します。

```typescript
async function subscribeToHeartRate(device: Device, onUpdate: (bpm: number) => void) {
  await device.discoverAllServicesAndCharacteristics();

  // キャラクタリスティックの変化を監視する
  const subscription = device.monitorCharacteristicForService(
    HEART_RATE_SERVICE,
    HEART_RATE_CHARACTERISTIC,
    (error, characteristic) => {
      if (error) {
        console.error('通知エラー:', error);
        return;
      }
      if (characteristic?.value) {
        const rawData = Buffer.from(characteristic.value, 'base64');
        const heartRate = rawData[1];
        onUpdate(heartRate); // 値の変化をコールバックで通知
      }
    }
  );

  // 購読を解除するための関数を返す
  return () => subscription.remove();
}
```

フィットネス系アプリでは、このデータを HealthKit と連携させるとより本格的なアプリになります。詳しくは [Rork Max で HealthKit を使ったヘルスケアアプリを作る完全ガイド](/articles/rork-dev/rork-max-healthkit-guide) をご覧ください。

また、BLE に近い近距離通信技術として NFC もあります。タッチで瞬時に接続できる NFC の実装方法は [Rork Max で NFC アプリを作る：Core NFC 完全実装ガイド](/articles/rork-dev/rork-max-nfc-guide) で解説しています。

## よくあるエラーと対処法

BLE 開発では特有のエラーに遭遇しやすいです。よくあるケースと対処法をまとめます。

**エラー 1: "Cannot use BLE in current state: PoweredOff"**

Bluetooth がオフになっています。ユーザーに Bluetooth の有効化を促すダイアログを表示しましょう。

```typescript
bleManager.onStateChange((state) => {
  if (state === 'PoweredOff') {
    Alert.alert(
      'Bluetooth が無効です',
      'デバイスと連携するために Bluetooth をオンにしてください。',
      [{ text: '設定を開く', onPress: () => Linking.openSettings() }]
    );
  }
}, true);
```

**エラー 2: "Device disconnected" が頻発する**

接続が不安定な場合は、自動再接続ロジックを実装します。

```typescript
async function connectWithRetry(device: Device, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await device.connect({ timeout: 5000 });
    } catch (error) {
      if (i < maxRetries - 1) {
        console.log(`再接続試行 ${i + 1}/${maxRetries}...`);
        await new Promise((resolve) => setTimeout(resolve, 1000));
      } else {
        throw error;
      }
    }
  }
}
```

**エラー 3: Android で「位置情報の権限が必要」とエラーになる**

Android では BLE スキャンに位置情報権限が必要です（OS の仕様）。スキャン前に `PermissionsAndroid.request` で権限を要求してください。Rork のプロンプトで「Android の BLE 権限取得を実装してください」と依頼すると自動で対応してくれます。

- **フィットネストラッカー連携アプリ**: 心拍数・歩数・カロリーをリアルタイムで表示し、HealthKit に自動記録

## FAQ

**Q: BLE は iOS シミュレーターでテストできますか？**

A: できません。BLE 機能は実際の Bluetooth ハードウェアが必要なため、実機でのテストが必須です。iOS シミュレーターおよび Android エミュレーターでは BLE は動作しません。Rork Max をお使いの方は TestFlight 経由で実機テストを行ってください。

**Q: react-native-ble-plx と @react-native-community/ble-manager の違いは何ですか？**

A: `react-native-ble-plx` は Promise ベースの API と TypeScript のサポートが充実しており、メンテナンスも活発です。`ble-manager` は古くからあるライブラリで機能的に同等ですが、API の書き心地は `ble-plx` の方がモダンです。Rork での新規開発には `react-native-ble-plx` を推奨します。

**Q: BLE で大きなデータ（画像など）を転送できますか？**

A: BLE の1パケットは最大 512 バイト程度に制限されています（MTU による）。画像のような大きなデータを転送するには、データをチャンクに分割して送受信するカスタムプロトコルを実装するか、Wi-Fi や LTE 経由でのデータ転送を検討してください。小さなセンサーデータ（温度・心拍・加速度など）には BLE は非常に適しています。

**Q: アプリがバックグラウンド状態でも BLE 通知を受け取れますか？**

A: iOS では「バックグラウンドモード」の設定が必要です。Info.plist に `UIBackgroundModes` で `bluetooth-central` を追加し、Xcode の Capabilities から Background Modes を有効にすれば、バックグラウンドでも通知を受信できます。Android は標準でバックグラウンド動作が可能ですが、Doze モードの考慮が必要です。

## まとめ

Rork × react-native-ble-plx を使えば、心拍センサー・スマートロック・医療機器など多様な BLE デバイスと連携するアプリを、ネイティブの複雑な実装なしに素早く開発できます。

- BLE の基本（GATT・サービス・キャラクタリスティック）を理解する
- `react-native-ble-plx` で デバイスのスキャン・接続・データ読み取りを実装する
- Notify（通知）でリアルタイムデータを受信する
- エラーハンドリングと自動再接続を実装して安定性を高める

BLE 連携アプリは差別化しやすく、ヘルスケア・スマートホーム・業務向けなどニッチでも確実なニーズがあります。ぜひ自分だけのスマートデバイス連携アプリに挑戦してみてください。
