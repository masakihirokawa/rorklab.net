## はじめに — フードデリバリーアプリを Rork で作る理由

フードデリバリーや飲食店の注文管理アプリは、個人開発者にとって挑戦しがいのある題材のひとつです。リアルタイムの注文状態管理、商品カタログの表示、決済フロー、プッシュ通知など、モバイルアプリ開発に必要な要素がほぼすべて詰まっています。

Rork を使えば、こうした本格的なアプリをコード量を最小限に抑えながら構築できます。この記事では、フードデリバリーアプリの基本構成を Rork + Supabase + Stripe で実装する流れを、初心者〜中級者向けにわかりやすく解説します。

この記事で学べること:

- フードデリバリーアプリの画面設計と Rork でのスキャフォールディング
- Supabase を使ったメニューデータの管理とリアルタイム注文追跡
- Stripe を使った決済フローの組み込み
- プッシュ通知で注文ステータスをユーザーに伝える方法
- よくあるエラーと解決策

対象読者は、Rork の基本操作に慣れていてアプリを実際に公開してみたい方、または個人開発でサービスアプリを作りたい方です。

## アプリの全体像と必要な技術スタック

今回作るのは「飲食店向けテイクアウト注文アプリ」のプロトタイプです。機能を整理すると以下のようになります:

- **メニュー一覧画面**: 料理カテゴリ別に商品を表示
- **カート画面**: 選択した商品の確認と数量変更
- **決済画面**: Stripe によるカード・Apple Pay 決済
- **注文履歴画面**: 過去の注文と現在のステータス確認
- **プッシュ通知**: 「調理中」「準備完了」などのステータス変化を通知

技術スタックは次の通りです:

- **Rork（React Native / Expo）**: UIとナビゲーション
- **Supabase**: データベース（メニュー・注文）、認証、リアルタイムサブスクリプション
- **Stripe**: 決済処理（Stripe Payment Sheet）
- **Expo Notifications**: プッシュ通知

なお、個人開発でアプリを収益化する方法については [Rork アプリの収益化戦略 完全ガイド](/articles/rork-dev/rork-monetization-complete-guide) も参考にしてください。

## Step 1: Supabase でデータベースを設計する

まず Supabase でプロジェクトを作成し、以下のテーブルを定義します。

### テーブル構成

```sql
-- メニュー商品テーブル
create table menu_items (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  description text,
  price integer not null,   -- 円単位（例: 880）
  category text not null,   -- 例: "ハンバーガー", "ドリンク"
  image_url text,
  available boolean default true,
  created_at timestamptz default now()
);

-- 注文テーブル
create table orders (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users(id),
  status text default 'pending',  -- pending / preparing / ready / delivered
  total_amount integer not null,
  stripe_payment_intent_id text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- 注文アイテム（注文と商品の中間テーブル）
create table order_items (
  id uuid default gen_random_uuid() primary key,
  order_id uuid references orders(id) on delete cascade,
  menu_item_id uuid references menu_items(id),
  quantity integer not null default 1,
  unit_price integer not null
);
```

Row Level Security（RLS）を有効にして、ユーザーが自分の注文のみ参照・更新できるようにポリシーを設定してください。

## Step 2: Rork でメニュー画面を構築する

Rork のプロジェクトを新規作成したら、まずメニュー一覧画面を作ります。Rork のプロンプト欄に以下のように入力してみましょう。

```
Supabase の menu_items テーブルからデータを取得して、
カテゴリ別にセクション分けしたFlatListで商品一覧を表示する画面を作ってください。
各商品カードには画像・名前・値段・「カートに追加」ボタンを表示してください。
```

Rork が生成するコードのベースは以下のようになります:

```tsx
// app/(tabs)/menu.tsx（Rork が生成するコード例）
import { useEffect, useState } from "react";
import { FlatList, SectionList, Text, TouchableOpacity, View } from "react-native";
import { supabase } from "@/lib/supabase";
import { useCartStore } from "@/store/cartStore";

type MenuItem = {
  id: string;
  name: string;
  price: number;
  category: string;
  image_url: string;
};

export default function MenuScreen() {
  const [sections, setSections] = useState<{ title: string; data: MenuItem[] }[]>([]);
  const addToCart = useCartStore((s) => s.addItem);

  useEffect(() => {
    const fetchMenu = async () => {
      const { data } = await supabase
        .from("menu_items")
        .select("*")
        .eq("available", true)
        .order("category");

      if (data) {
        // カテゴリ別にグループ化
        const grouped = data.reduce((acc: Record<string, MenuItem[]>, item) => {
          (acc[item.category] ||= []).push(item);
          return acc;
        }, {});
        setSections(Object.entries(grouped).map(([title, data]) => ({ title, data })));
      }
    };
    fetchMenu();
  }, []);

  return (
    <SectionList
      sections={sections}
      keyExtractor={(item) => item.id}
      renderSectionHeader={({ section }) => (
        <Text style={{ fontWeight: "bold", padding: 12, backgroundColor: "#f5f5f5" }}>
          {section.title}
        </Text>
      )}
      renderItem={({ item }) => (
        <View style={{ flexDirection: "row", padding: 12, borderBottomWidth: 1, borderColor: "#eee" }}>
          <Text style={{ flex: 1 }}>{item.name} — ¥{item.price}</Text>
          <TouchableOpacity onPress={() => addToCart(item)}>
            <Text style={{ color: "#e63946" }}>追加</Text>
          </TouchableOpacity>
        </View>
      )}
    />
  );
}
```

Zustand などの状態管理ライブラリを使ってカート情報をグローバルに管理するのがポイントです。

## Step 3: Stripe で決済フローを実装する

注文確定画面では Stripe Payment Sheet を使います。Supabase Edge Functions でサーバーサイドに PaymentIntent を作成し、クライアントに返す設計にします。

### Supabase Edge Function（支払い意図の作成）

```typescript
// supabase/functions/create-payment-intent/index.ts
import Stripe from "https://esm.sh/stripe@14";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!);

Deno.serve(async (req) => {
  const { amount, currency = "jpy" } = await req.json();

  const paymentIntent = await stripe.paymentIntents.create({
    amount,       // 例: 1980（¥1,980）
    currency,
    automatic_payment_methods: { enabled: true },
  });

  return new Response(
    JSON.stringify({ clientSecret: paymentIntent.client_secret }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### Rork 側の実装（Payment Sheet 呼び出し）

```tsx
import { initPaymentSheet, presentPaymentSheet } from "@stripe/stripe-react-native";

const handleCheckout = async () => {
  const total = cartItems.reduce((sum, item) => sum + item.price * item.quantity, 0);

  // Edge Function で PaymentIntent を取得
  const { data } = await supabase.functions.invoke("create-payment-intent", {
    body: { amount: total },
  });

  // Payment Sheet の初期化
  await initPaymentSheet({
    paymentIntentClientSecret: data.clientSecret,
    merchantDisplayName: "My Food App",
    defaultBillingDetails: { name: user?.email },
  });

  // シートを表示
  const { error } = await presentPaymentSheet();

  if (!error) {
    // 注文を Supabase に保存
    await supabase.from("orders").insert({
      user_id: user.id,
      total_amount: total,
      status: "pending",
    });
    router.push("/order-confirmed");
  }
};
```

Stripe 決済のより詳しい実装方法は [Rork × Stripe 決済完全ガイド](/articles/rork-dev/rork-stripe-payment-integration) をご覧ください。

## Step 4: Supabase Realtime で注文ステータスをリアルタイム追跡する

ユーザーが注文した後、「調理中 → 準備完了」のステータス変化をリアルタイムで反映させます。Supabase の Realtime Subscriptions を使います。

```tsx
// 注文ステータスをリアルタイム購読
useEffect(() => {
  const channel = supabase
    .channel(`order-${orderId}`)
    .on(
      "postgres_changes",
      {
        event: "UPDATE",
        schema: "public",
        table: "orders",
        filter: `id=eq.${orderId}`,
      },
      (payload) => {
        setOrderStatus(payload.new.status);
        // ステータス変化をローカル通知
        if (payload.new.status === "ready") {
          scheduleLocalNotification("ご注文の準備ができました！");
        }
      }
    )
    .subscribe();

  return () => supabase.removeChannel(channel);
}, [orderId]);
```

プッシュ通知の詳細な設定方法については [Rork アプリへのプッシュ通知実装ガイド](/articles/rork-dev/rork-push-notifications) をご参照ください。

## よくあるエラーと対処法

### Supabase RLS でデータが取得できない

RLS ポリシーを有効にした直後は、ポリシーが設定されるまで全データが非表示になります。Supabase ダッシュボードで「Authentication > Policies」を確認し、`select` ポリシーを正しく追加してください。

### Stripe の PaymentIntent が二重作成される

`handleCheckout` をボタンのタップごとに呼ぶと、連打時に PaymentIntent が複数作られてしまいます。`isLoading` フラグを使って処理中は再実行を防いでください。

```tsx
const [isLoading, setIsLoading] = useState(false);

const handleCheckout = async () => {
  if (isLoading) return;  // ← 二重呼び出し防止
  setIsLoading(true);
  try {
    // ...決済処理
  } finally {
    setIsLoading(false);
  }
};
```

### リアルタイムサブスクリプションが切れる

アプリをバックグラウンドに移動するとチャンネルが切断されることがあります。`AppState` を監視してフォアグラウンド復帰時に再サブスクライブするロジックを追加しましょう。

## よくある質問（FAQ）

### Q1. Rork Free プランでもフードデリバリーアプリは作れますか？

はい、開発・テスト段階であれば Rork Free プラン（週5回の生成）で十分対応できます。ただし TestFlight 配布や App Store 公開には Rork Max が必要です。

### Q2. Supabase の無料枠で本番運用は可能ですか？

Supabase の無料枠（500MB DB、50,000 MAU）は個人運営の小規模アプリであれば十分使えます。ユーザーが増えてきたら Pro プランへの移行を検討してください。

### Q3. Apple Pay には対応できますか？

`@stripe/stripe-react-native` の Payment Sheet は Apple Pay / Google Pay を自動的にサポートします。Stripe ダッシュボードで Apple Pay のドメイン設定を完了させるだけで利用可能になります。

### Q4. 決済成功後のメール通知はどう実装しますか？

Stripe の Webhook イベント `payment_intent.succeeded` を Supabase Edge Function で受信し、Resend や SendGrid などのメール送信 API を呼び出す構成が一般的です。
