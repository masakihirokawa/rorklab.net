## 導入

SaaS アプリケーションやサブスクリプション型サービスを構築する際、決済処理とサブスクリプション管理は最も重要な要素の一つです。しかし、複雑なバックエンド処理や API 統合は開発時間を大幅に増加させてしまいます。

Rork Max は、このような課題を解決するローコード開発プラットフォームです。Stripe との統合により、開発者はコンプレックスな決済ロジックを実装することなく、数時間でプロダクションレディなサブスクリプション機能を構築できます。

このガイドでは、Rork Max と Stripe を組み合わせて、実用的で堅牢なサブスクリプションシステムを構築する方法を段階的に説明します。

## 前提条件

このガイドに取り組む前に、以下の準備を完了してください。

- **Rork Max アカウント**: Rork Max の基本的な使用経験があることが望ましいです。初めての方は、[Stripe 決済連携ガイド](/articles/rork-dev/rork-stripe-payment-integration)を参考にしてください。
- **Stripe アカウント**: Stripe の管理画面にアクセスでき、API キー（公開可能キーとシークレットキー）を取得できる必要があります。
- **基本的な API の知識**: REST API と JSON データ形式の基本的な理解があるとよいでしょう。
- **データベース**: ユーザーとサブスクリプション情報を保存するために、簡単なデータベーススキーマが必要です。Supabase や Firebase を使用できます（詳細は[Supabase/Firebase バックエンドガイド](/articles/rork-dev/supabase-firebase-backend)参照）。

## Stripe での準備

Rork Max で Stripe サブスクリプションを実装する前に、Stripe 側の設定を完了させることが重要です。

### 1. 商品とプランの作成

まず、Stripe の管理画面で販売する商品とプランを定義します。

1. Stripe ダッシュボードにログイン
2. **カタログ** → **商品** を選択
3. **新しい商品を追加** をクリック
4. 商品名（例：「Pro Plan」）と説明を入力
5. **価格を追加** をクリックし、以下を設定します：
   - **価格**: 月額 ¥5,000 など
   - **請求期間**: 月次（Monthly）
   - **請求サイクル**: 毎月繰り返し（Recurring）

複数のプラン（Basic、Pro、Enterprise など）を作成する場合は、この手順を繰り返します。

### 2. API キーの確認

Rork Max との連携に必要な API キーを確認します。

1. **開発者** → **API キー** を選択
2. **公開可能キー** と **シークレットキー** をメモしておきます（本番環境の場合）
3. Webhooks の設定も後で必要になるため、**Webhooks** セクションを確認しておきます

## Rork Max での設定

### 1. Stripe の接続設定

Rork Max で Stripe を接続します。

1. Rork Max ダッシュボードを開く
2. **統合** → **決済サービス** を選択
3. **Stripe** を選択し、**接続** をクリック
4. Stripe の API シークレットキーを貼り付け
5. **確認** をクリックして接続を完了

### 2. サブスクリプションテーブルの作成

データベースにサブスクリプション情報を保存するテーブルを作成します。

```sql
-- 実行例（Supabase の場合）
CREATE TABLE subscriptions (

  stripe_subscription_id TEXT UNIQUE NOT NULL,
  stripe_customer_id TEXT NOT NULL,
  plan_id TEXT NOT NULL,
  status TEXT NOT NULL, -- 'active', 'past_due', 'canceled', 'incomplete' など

-- CREATE TABLE へのメッセージ（エラーなし）
```

### 3. Webhook エンドポイントの構築

Stripe からの イベント（支払い成功、サブスクリプション更新、キャンセルなど）をリアルタイムで受信するために、Webhook エンドポイントを設定します。

```javascript
// Rork Max Webhook Handler (例)
// エンドポイント: /webhooks/stripe

exports.handler = async (event) => {
  const sig = event.headers['stripe-signature'];
  const body = event.body;

  // Stripe イベントを検証
  const stripeEvent = stripe.webhooks.constructEvent(
    body,
    sig,
    process.env.STRIPE_WEBHOOK_SECRET
  );

  // イベント種別に応じて処理
  switch (stripeEvent.type) {
    case 'customer.subscription.updated':
      // サブスクリプション更新時
      const updatedSub = stripeEvent.data.object;
      await updateSubscriptionInDB(updatedSub);
      break;

    case 'customer.subscription.deleted':
      // サブスクリプション削除時
      const deletedSub = stripeEvent.data.object;
      await cancelSubscriptionInDB(deletedSub);
      break;

    case 'invoice.payment_succeeded':
      // 支払い成功時
      const invoice = stripeEvent.data.object;
      await logPaymentSuccess(invoice);
      break;

    default:
      console.log(`未処理のイベント: ${stripeEvent.type}`);
  }

  return { statusCode: 200, body: JSON.stringify({ received: true }) };
};

// 予期される出力:
// 正常な Webhook 処理時は { statusCode: 200, body: '{"received":true}' }
```

Stripe ダッシュボードで Webhook エンドポイントを登録します。

1. **開発者** → **Webhooks** を選択
2. **エンドポイントを追加** をクリック
3. Rork Max のエンドポイント URL（例：`https://yourapp.rorklab.app/webhooks/stripe`）を入力
4. 受信するイベントを選択：
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.payment_succeeded`
   - `invoice.payment_failed`
5. **エンドポイントを追加** をクリック

## サブスクリプション機能の実装

### 1. サブスクリプション作成フロー

ユーザーがサブスクリプションを開始するプロセスを実装します。

```javascript
// Rork Max コンポーネント (例)
// サブスクリプション作成時の流れ

async function createSubscription(userId, planId, paymentMethodId) {
  try {
    // 1. Stripe の Customer を作成（既存チェック）
    let customerId = await getCustomerIdFromDB(userId);

    if (!customerId) {
      const customer = await stripe.customers.create({
        email: user.email,
        metadata: { userId: userId }
      });
      customerId = customer.id;
      await saveCustomerIdToDB(userId, customerId);
    }

    // 2. Subscription を作成
    const subscription = await stripe.subscriptions.create({
      customer: customerId,
      items: [{ price: planId }],
      payment_method: paymentMethodId,
      default_payment_method: paymentMethodId,
      expand: ['latest_invoice.payment_intent']
    });

    // 3. DB に保存
    await saveSubscriptionToDB({
      user_id: userId,
      stripe_subscription_id: subscription.id,
      stripe_customer_id: customerId,
      plan_id: planId,
      status: subscription.status,
      current_period_start: new Date(subscription.current_period_start * 1000),
      current_period_end: new Date(subscription.current_period_end * 1000)
    });

    // 4. UI に返す
    return {
      success: true,
      subscription_id: subscription.id,
      status: subscription.status
    };

  } catch (error) {
    console.error('サブスクリプション作成エラー:', error.message);
    return {
      success: false,
      error: error.message
    };
  }
}

// 予期される出力:
// 成功時: { success: true, subscription_id: 'sub_xxxxx', status: 'active' }
// エラー時: { success: false, error: 'エラーメッセージ' }
```

### 2. サブスクリプション更新・キャンセル

ユーザーがプランを変更したり、サブスクリプションをキャンセルしたりする機能を実装します。

```javascript
// プラン変更
async function updateSubscriptionPlan(subscriptionId, newPriceId) {
  const subscription = await stripe.subscriptions.retrieve(subscriptionId);

  const updated = await stripe.subscriptions.update(subscriptionId, {
    items: [{
      id: subscription.items.data[0].id,
      price: newPriceId
    }],
    proration_behavior: 'create_prorations' // 日割り計算を有効に
  });

  return { success: true, subscription: updated };
}

// サブスクリプションキャンセル（即座）
async function cancelSubscriptionImmediate(subscriptionId) {
  const cancelled = await stripe.subscriptions.del(subscriptionId);
  return { success: true, subscription: cancelled };
}

// サブスクリプションキャンセル（期末まで継続）
async function cancelSubscriptionAtPeriodEnd(subscriptionId) {
  const cancelled = await stripe.subscriptions.update(subscriptionId, {
    cancel_at_period_end: true
  });
  return { success: true, subscription: cancelled };
}
```

## トラブルシューティング

### よくある問題と解決方法

**問題 1: Webhook イベントが受信されない**

- Webhook URL が正しく登録されているか確認してください
- Stripe ダッシュボードの **Webhooks** → **イベント** から最近のイベントログを確認し、エラーがないか確認してください
- Webhook Secret が正しく環境変数に設定されているか確認してください

**問題 2: サブスクリプション作成時に「Payment failed」エラー**

- 支払い方法が有効か確認してください（テスト環境では Stripe のテストカードを使用）
- 顧客の請求先住所が正しく設定されているか確認してください
- Stripe の詐欺検出設定により拒否されていないか確認してください

**問題 3: プラン変更時に日割り計算が反映されない**

- `proration_behavior` が `create_prorations` に設定されているか確認してください
- Stripe ダッシュボードの請求設定で日割り計算が有効になっているか確認してください

## FAQ

**Q1: Rork Max でのサブスクリプション実装に、何か追加のライセンスや費用は必要ですか？**

A: Rork Max 自体は追加ライセンスなく使用できます。ただし、Stripe の取引手数料（国内取引で 3.6% + 手数料）がかかります。これはサブスクリプション型ビジネスの標準的なコストです。

**Q2: 複数の通貨（JPY、USD、EUR など）でサブスクリプションを提供したい場合はどうすればよいですか？**

A: Stripe では複数の通貨に対応しており、商品ごとに複数の通貨での価格を設定できます。ユーザーの位置情報やブラウザの言語設定に基づいて、適切な通貨のプランを表示するロジックを Rork Max で構築してください。

**Q3: サブスクリプション顧客の退職や返金リクエストに対応する場合、どのような手順を踏むべきですか？**

A: Stripe では「Refund」機能により、支払い済みの請求を返金できます。Rork Max のダッシュボードから Stripe の API を呼び出し、返金処理を実行してください。返金ポリシーは[個人開発アプリビジネスガイド](/articles/rork-business/solo-dev-app-business)を参考にして、事前に明確に定義することをお勧めします。
