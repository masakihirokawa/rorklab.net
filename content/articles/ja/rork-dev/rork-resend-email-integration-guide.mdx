アプリを開発していると、いつかは「メールを送りたい」という場面に出会います。ユーザー登録時のウェルカムメール、パスワードリセット、購入完了通知、週次レポート——これらはユーザーとの関係を深める重要なタッチポイントです。

Rork で作ったアプリにメール機能を加えるなら、今一番おすすめできるのが **Resend**（[resend.com](https://resend.com/)）です。2023年に登場した開発者向けメール送信 API で、シンプルな設計・充実した無料枠・React Email との親和性の高さが人気を呼んでいます。

本記事では、Rork アプリ（Supabase バックエンド構成）に Resend を組み込み、実用的なメール送信フローを構築する方法をステップごとに解説します。

---

## Resend とは — 従来のメール送信と何が違うか

Resend は「開発者のために作られたメール API」です。Amazon SES や SendGrid など既存サービスとの主な違いを整理します。

- **セットアップの速さ**: アカウント作成から最初のメール送信まで10分以内が目安です。DNS 設定さえ済めば、あとはAPIキーとHTTPリクエストだけで動き始めます。
- **React Email 統合**: Resend は [React Email](https://react.email/) という UI ライブラリと連携しており、JSX でメールテンプレートを書けます。HTMLメールの辛さを大幅に軽減できます。
- **充実した無料枠**: 月 3,000 通・1日100通まで無料。個人開発の初期フェーズなら十分すぎるほどです。
- **配信率の高さ**: ドメイン認証（SPF / DKIM / DMARC）を簡単に設定でき、スパムフィルターに弾かれにくい設計になっています。

## 前提条件と環境準備

本記事では以下の構成を前提とします。

- Rork で作成したアプリ（React Native / Expo ベース）
- Supabase をバックエンドとして使用（Auth + Edge Functions）
- Node.js 18 以上（ローカル開発用）

Supabase を使っていない場合でも、後述する Resend の REST API 呼び出しの考え方は Cloudflare Workers や他のサーバーレス環境でも同様に応用できます。

## Step 1: Resend アカウントのセットアップ

まず [resend.com](https://resend.com/) にアクセスし、GitHub アカウントでサインアップします（Googleも可）。

ダッシュボードに入ったら、以下の2つの作業を行います。

**① ドメインの追加と DNS 設定**

「Domains」メニューから送信元ドメインを追加します。独自ドメインがない場合は、開発中に限り `onboarding@resend.dev` を使える無料の Resend テスト用アドレスが利用できます（本番前に独自ドメインへの切り替えを推奨）。

DNS レコードを設定するとドメイン認証が完了します。大抵のDNSサービス（Cloudflare, お名前.com等）なら10〜30分で認証が通ります。

**② API キーの取得**

「API Keys」メニューから新しいAPIキーを作成します。スコープは「Sending Access」で十分です。このキーは一度しか表示されないので、必ず安全な場所に保存してください。

```bash
# .env.local に保存（gitignore 対象）
RESEND_API_KEY=YOUR_RESEND_API_KEY
```

## Step 2: Supabase Edge Function でメールを送信する

Rork アプリの場合、メール送信はクライアント（スマートフォン）から直接行うのではなく、**サーバーサイド（Edge Function）経由**で行うのがセキュリティ上の正しい設計です。APIキーはサーバー側に隠蔽します。

**Edge Function の作成**

Supabase ダッシュボードの「Edge Functions」から `send-email` 関数を新規作成します。

```typescript
// supabase/functions/send-email/index.ts
import { serve } from "https://deno.land/std@0.208.0/http/server.ts";

const RESEND_API_KEY = Deno.env.get("RESEND_API_KEY")!;

interface EmailRequest {
  to: string;
  subject: string;
  html: string;
  from?: string;
}

serve(async (req: Request) => {
  // CORS ヘッダー（Rork アプリからのリクエスト許可）
  if (req.method === "OPTIONS") {
    return new Response(null, {
      headers: {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "authorization, x-client-info, apikey, content-type",
      },
    });
  }

  try {
    const { to, subject, html, from }: EmailRequest = await req.json();

    const response = await fetch("https://api.resend.com/emails", {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${RESEND_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        from: from ?? "Your App <noreply@yourdomain.com>",
        to: [to],
        subject,
        html,
      }),
    });

    const data = await response.json();

    if (!response.ok) {
      return new Response(JSON.stringify({ error: data }), {
        status: response.status,
        headers: { "Content-Type": "application/json" },
      });
    }

    return new Response(JSON.stringify({ id: data.id, success: true }), {
      headers: { "Content-Type": "application/json" },
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: "Internal server error" }), {
      status: 500,
      headers: { "Content-Type": "application/json" },
    });
  }
});
```

**環境変数の設定**

Supabase ダッシュボードの「Settings → Edge Functions → Secrets」で `RESEND_API_KEY` を追加します。ローカル開発では `.env.local` に設定し、`supabase functions serve` 時に自動的に読み込まれます。

## Step 3: ウェルカムメールを実装する

ユーザーが新規登録したタイミングで自動的にウェルカムメールを送る実装例です。

**Supabase Database Webhook を使う方法**

Supabase の「Database → Webhooks」から `auth.users` テーブルの INSERT イベントで先ほど作成した Edge Function を呼び出すよう設定します。これにより、ユーザー登録のたびに自動でメールが送信されます。

```typescript
// send-welcome-email/index.ts（別の Edge Function として作成）
import { serve } from "https://deno.land/std@0.208.0/http/server.ts";

const RESEND_API_KEY = Deno.env.get("RESEND_API_KEY")!;

serve(async (req: Request) => {
  const payload = await req.json();
  const userEmail = payload.record?.email;
  const userName = payload.record?.raw_user_meta_data?.name ?? "新しいユーザー";

  if (!userEmail) {
    return new Response("No email", { status: 400 });
  }

  // ウェルカムメールのHTMLテンプレート
  const html = `
    <!DOCTYPE html>
    <html>
      <body style="font-family: sans-serif; max-width: 600px; margin: 0 auto; padding: 20px;">
        <h1 style="color: #333;">ようこそ、${userName} さん！</h1>
        <p>アカウントの登録が完了しました。さっそくアプリを使い始めましょう。</p>
        <a href="https://yourdomain.com/onboarding"
           style="display: inline-block; background: #6366f1; color: white;
                  padding: 12px 24px; border-radius: 8px; text-decoration: none;">
          今すぐ始める
        </a>
        <p style="color: #888; font-size: 12px; margin-top: 40px;">
          このメールに心当たりがない場合はお知らせください。
        </p>
      </body>
    </html>
  `;

  await fetch("https://api.resend.com/emails", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${RESEND_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      from: "Your App <welcome@yourdomain.com>",
      to: [userEmail],
      subject: "ようこそ！アカウント登録が完了しました",
      html,
    }),
  });

  return new Response("OK");
});
```

## Step 4: Rork アプリ側からメールを送信する

パスワードリセットや問い合わせフォーム送信など、ユーザーのアクションに応じてアプリ側からメール送信をトリガーする例です。

```typescript
// services/emailService.ts
import { supabase } from "./supabaseClient";

interface SendEmailParams {
  to: string;
  subject: string;
  html: string;
}

export async function sendEmail({ to, subject, html }: SendEmailParams) {
  const { data, error } = await supabase.functions.invoke("send-email", {
    body: { to, subject, html },
  });

  if (error) {
    console.error("メール送信エラー:", error);
    throw new Error("メールの送信に失敗しました");
  }

  return data;
}

// 使用例：問い合わせフォーム送信後に確認メールを送る
export async function sendContactConfirmation(userEmail: string, userName: string) {
  const html = `
    <h2>お問い合わせを受け付けました</h2>
    <p>${userName} 様、ご連絡ありがとうございます。</p>
    <p>担当者よりご返信いたしますので、しばらくお待ちください（通常1〜2営業日）。</p>
  `;

  return sendEmail({
    to: userEmail,
    subject: "お問い合わせ内容を確認しました",
    html,
  });
}
```

Rork のコンポーネント内では以下のように呼び出します。

```typescript
// ContactForm.tsx
import { sendContactConfirmation } from "@/services/emailService";

const handleSubmit = async () => {
  try {
    setLoading(true);
    // ... 問い合わせ内容を DB に保存する処理
    await sendContactConfirmation(userEmail, userName);
    Alert.alert("送信完了", "確認メールをお送りしました。");
  } catch (error) {
    Alert.alert("エラー", "送信に失敗しました。もう一度お試しください。");
  } finally {
    setLoading(false);
  }
};
```

## Step 5: React Email でHTMLテンプレートを作る（応用）

より洗練されたメールデザインが必要になったら、**React Email** を使うと大幅に作業が楽になります。JSX でメールテンプレートを書き、プレビューしながら開発できます。

```bash
# React Email のインストール
npm install @react-email/components react-email --save-dev
```

```typescript
// emails/WelcomeEmail.tsx
import {
  Html, Head, Body, Container, Heading, Text, Button, Hr
} from "@react-email/components";

interface WelcomeEmailProps {
  userName: string;
}

export default function WelcomeEmail({ userName }: WelcomeEmailProps) {
  return (
    <Html>
      <Head />
      <Body style={{ backgroundColor: "#f9fafb", fontFamily: "sans-serif" }}>
        <Container style={{ maxWidth: 560, margin: "0 auto", padding: 24 }}>
          <Heading>ようこそ、{userName} さん！</Heading>
          <Text>アカウント登録が完了しました。</Text>
          <Button
            href="https://yourdomain.com/onboarding"
            style={{
              backgroundColor: "#6366f1",
              color: "#fff",
              padding: "12px 24px",
              borderRadius: 8,
            }}
          >
            今すぐ始める
          </Button>
          <Hr />
          <Text style={{ fontSize: 12, color: "#888" }}>
            このメールは自動送信されています。
          </Text>
        </Container>
      </Body>
    </Html>
  );
}
```

テンプレートを HTML 文字列に変換して Resend に渡します。

```typescript
import { render } from "@react-email/render";
import WelcomeEmail from "./emails/WelcomeEmail";

const html = render(<WelcomeEmail userName="太郎" />);
// この html を Resend の body.html に渡す
```

React Email の開発サーバーを起動するとブラウザで実際のメール見た目をプレビューできます（`npx email dev`）。HTML メール開発特有の煩雑さを大幅に軽減できるため、テンプレートが増えてきたら積極的に導入する価値があります。

より本格的なバックエンド設計を学びたい方は、[Rork × Hono.js × Cloudflare Workers で本番レベル REST API を構築する完全ガイド](/articles/rork-dev/rork-hono-cloudflare-workers-rest-api-guide) もあわせてご覧ください。

## よくあるエラーと対処法

**「550 Domain not verified」エラー**

送信元ドメインの DNS 認証が完了していない状態で送信しようとしたときに発生します。Resend ダッシュボードの「Domains」でドメインのステータスが「Verified」になっているか確認してください。認証前は `onboarding@resend.dev` を代替の from アドレスとして使えます。

**「403 Forbidden」エラー**

APIキーのスコープが `Sending Access` になっているか確認します。APIキーを Supabase の Secrets に正しく設定しているかも再確認してください。環境変数名のタイポ（`RESEND_API_KEY` → `RESEND_API_KEYS` 等）は意外と多いミスです。

**メールが届かない（スパムフォルダに入る）**

DNS 認証（SPF / DKIM / DMARC）の設定を再確認してください。特に DMARC レコードが `p=none` になっているとスパム判定されやすいです。また、件名や本文に「無料」「当選」などのスパムトリガーワードが含まれていないかも確認します。

**Edge Function のタイムアウト**

Supabase Edge Function の無料枠はリクエストタイムアウトが比較的短めです（数秒程度）。Resend API のレスポンスは通常1秒以内ですが、ネットワーク遅延が発生する場合はリトライ処理を入れることを検討してください。

## FAQ

**Q. Supabase Auth の組み込みメール機能と Resend はどう使い分けますか？**

Supabase Auth にはメール認証・パスワードリセット用のメール送信機能が内蔵されており、Resend を SMTP プロバイダーとして設定することも可能です。認証系メール（サインアップ確認・パスワードリセット）は Supabase Auth に任せ、ウェルカムメールや通知など独自デザインが必要なメールは本記事の Edge Function 経由で送る、という使い分けが実用的です。

**Q. 月3,000通の無料枠を超えそうな場合の料金は？**

Resend の有料プランは月 $20 から（10万通/月）。個人開発でユーザー数が増えてきたタイミングでのアップグレードでも十分間に合うコスト感です。大量配信（ニュースレター等）が必要な場合はプランを比較検討してください。

**Q. Cloudflare Workers を使っている場合も同じコードで動きますか？**

Fetch API を使っている部分は Cloudflare Workers でもそのまま動作します。Deno 固有の API（`Deno.env.get`）部分だけ `env.RESEND_API_KEY` 形式に書き換えれば問題ありません。[Rork × Cloudflare D1 バックエンドガイド](/articles/rork-dev/rork-cloudflare-d1-sqlite-backend-guide) も参考にどうぞ。

## まとめ

Rork アプリへの Resend 統合をまとめると次のようになります。

- Resend は設定が最小限でメール送信を始められる開発者向け API
- APIキーはサーバー側（Supabase Edge Function / Cloudflare Workers）に隠蔽して使う
- Database Webhook と組み合わせれば、ユーザー登録のたびに自動でウェルカムメールを送れる
- React Email を使えばJSXでメールテンプレートをプレビューしながら開発できる
- 無料枠（月3,000通）で個人開発の初期フェーズは十分カバーできる

メール機能はアプリのユーザーリテンションを改善する地味ながら強力な手段です。プッシュ通知が届かないユーザーへのフォローアップ、購入後のフロー改善など、応用の幅は広いです。ぜひ本記事を参考に、Rork アプリのコミュニケーション設計をアップグレードしてみてください。
