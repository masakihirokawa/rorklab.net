# Rork Max アドバンス — ネイティブ API、Core ML、App Store 公開まで

Rork Max は基本的なアプリから複雑なエンタープライズアプリまで対応できます。本ガイドでは、機械学習、拡張現実、デバイス固有機能の実装方法から、App Store 公開、本番環境での運用管理まで、開発者向けの高度なトピックを実装コード付きで解説します。

## Part 1: Core ML による機械学習の統合

### Core ML の基礎

Core ML は Apple の機械学習フレームワークで、デバイス上で高速に推論を実行します。Rork Max では、事前学習済みモデルを簡単に統合できます。

#### モデルの準備

```
1. Create ML で学習済みモデルを作成（または既存の .mlmodel を用意）
2. Rork Max プロジェクトの assets フォルダに配置
3. プロンプトで「このモデルを使用する」と指定
```

#### プロンプト例

```
「植物の写真から病気を判定するアプリを作ってください。
- カメラ、または写真ライブラリから画像選択
- PlantDisease.mlmodel で推論実行
- 診断結果（病名、確度）を表示
- 該当する対策方法を提案」
```

### 生成されたコード例

Rork Max が自動生成する Core ML 統合コード：

```swift
import CoreML
import Vision

class PlantDiseaseClassifier {
    func classifyImage(_ cgImage: CGImage) async throws -> PlantDiseaseResult {
        guard let model = try? PlantDisease(configuration: MLModelConfiguration()) else {
            throw ClassifierError.modelLoadFailed
        }

        let handler = VNImageRequestHandler(cgImage: cgImage)
        var classification: PlantDiseaseResult?

        let request = VNCoreMLRequest(model: try model.model()) { request, error in
            guard let results = request.results as? [VNClassificationObservation],
                  let topResult = results.first else { return }

            classification = PlantDiseaseResult(
                disease: topResult.identifier,
                confidence: Float(topResult.confidence)
            )
        }

        try handler.perform([request])
        return classification ?? PlantDiseaseResult(disease: "Unknown", confidence: 0)
    }
}

// 実際の使用方法
@main
struct PlantApp: App {
    @State private var selectedImage: UIImage?
    @State private var diagnosis: PlantDiseaseResult?

    var body: some Scene {
        WindowGroup {
            VStack {
                if let image = selectedImage {
                    Image(uiImage: image)
                        .resizable()
                        .scaledToFit()
                }

                Button("カメラで撮影") {
                    // Camera picker
                }

                if let diagnosis = diagnosis {
                    VStack(alignment: .leading) {
                        Text("診断結果: \(diagnosis.disease)")
                            .font(.headline)
                        Text("確度: \(String(format: "%.1f%%", diagnosis.confidence * 100))")
                    }
                }
            }
            .onChange(of: selectedImage) { newImage in
                Task {
                    if let cgImage = newImage?.cgImage {
                        let classifier = PlantDiseaseClassifier()
                        diagnosis = try await classifier.classifyImage(cgImage)
                    }
                }
            }
        }
    }
}
```

## Part 2: AR と LiDAR スキャナの実装

### ARKit による拡張現実

Rork Max は ARKit フレームワークを自動統合。スマートフォンのカメラと LiDAR センサーを活用した AR 体験が実装できます。

#### AR アプリの典型的なユースケース

1. **家具配置シミュレーション**: ソファをリビングに配置して見た目を確認
2. **3D モデル可視化**: 建築設計案を空間上に表示
3. **測定ツール**: 空間内の距離や面積を計測

#### プロンプト例

```
「室内に家具を配置できる AR アプリを作ってください。
- ARViewContainer でリアルタイムカメラフィード表示
- プリセット家具（テーブル、椅子、本棚）をタップで配置
- ピンチでサイズ調整、スワイプで回転
- 配置結果を 3D モデルとして保存
- iPhone 12 Pro 以上の LiDAR スキャナ対応」
```

### 生成されるコード概要

```swift
import ARKit
import RealityKit

struct FurnitureARView: UIViewRepresentable {
    @State private var arView: ARView?

    func makeUIView(context: Context) -> ARView {
        let arView = ARView(frame: .zero)

        // AR セッション初期化
        var configuration = ARWorldTrackingConfiguration()
        if ARWorldTrackingConfiguration.supportsFrameSemantics(.personSegmentationWithDepth) {
            configuration.frameSemantics.insert(.personSegmentationWithDepth)
        }

        arView.session.run(configuration)
        return arView
    }

    func updateUIView(_ uiView: ARView, context: Context) {}
}

// 家具モデルの配置関数
func placeFurniture(_ modelName: String, at position: SIMD3<Float>) {
    // Reality Composer で設計したモデルをロード・配置
    var model = try? ModelEntity.loadModel(named: modelName)
    model?.move(to: Transform(translation: position))
    arView.scene.addAnchor(AnchorEntity(world: Transform(translation: position)))
}
```

## Part 3: Dynamic Island と Live Activities

Dynamic Island は iPhone 14 Pro 以降で利用可能な新しい UI 要素。継続的な活動（ライブスコア、配達追跡など）をコンパクト表示できます。

### プロンプト例

```
「スポーツライブスコアアプリを作ってください。
- 現在の試合スコア（チーム A vs チーム B）
- Dynamic Island にスコアをリアルタイム表示
- Live Activities で試合時間、得点推移を見せる
- プッシュ通知で重要なシーン（ゴール）を通知」
```

### 実装コード（簡略版）

```swift
import ActivityKit

// Live Activities の属性定義
struct SportsScoreActivityAttributes: ActivityAttributes {
    public typealias SportsScoreStatus = ContentState

    public struct ContentState: Codable, Hashable {
        var teamA: String
        var scoreA: Int
        var teamB: String
        var scoreB: Int
        var time: String
    }

    var matchId: String
}

// アクティビティ開始
func startLiveActivity() {
    let attributes = SportsScoreActivityAttributes(
        matchId: "match-001"
    )
    let content = ActivityContent(
        state: SportsScoreActivityAttributes.SportsScoreStatus(
            teamA: "Team Red",
            scoreA: 2,
            teamB: "Team Blue",
            scoreB: 1,
            time: "45:30"
        ),
        staleDate: Date(timeIntervalSinceNow: 60)
    )

    do {
        let activity = try Activity<SportsScoreActivityAttributes>.request(
            attributes: attributes,
            content: content
        )
    } catch {
        print("Failed to start activity: \(error)")
    }
}
```

## Part 4: HealthKit 統合による健康データ活用

HealthKit を使用すると、ユーザーの健康データ（歩数、心拍数、睡眠時間など）をアプリに統合できます。

### プロンプト例

```
「健康管理ダッシュボードを作ってください。
- HealthKit から本日の歩数、心拍数、睡眠時間をリアルタイム取得
- 過去 30 日間のトレンド表示（グラフ）
- 目標達成状況をゲージ表示
- 運動目標に対する進捗をバッジで表彰
- Health アプリとのシームレス同期」
```

### Info.plist の権限設定（自動生成）

```xml
<key>NSHealthShareUsageDescription</key>
<string>健康データへのアクセスでダッシュボード表示を実現します</string>
<key>NSHealthUpdateUsageDescription</key>
<string>運動記録をヘルスケアアプリに送信します</string>
```

### コード実装例

```swift
import HealthKit

class HealthKitManager: ObservableObject {
    @Published var todaySteps: Int = 0
    @Published var heartRate: Int = 0
    @Published var sleepHours: Double = 0

    let healthStore = HKHealthStore()

    func requestAuthorization() async throws {
        let typesToRead: Set = [
            HKObjectType.quantityType(forIdentifier: .stepCount)!,
            HKObjectType.quantityType(forIdentifier: .heartRate)!,
            HKObjectType.categoryType(forIdentifier: .sleepAnalysis)!
        ]

        try await healthStore.requestAuthorization(
            toShare: Set(),
            read: typesToRead
        )
    }

    func fetchTodaySteps() async {
        let stepsType = HKQuantityType.quantityType(forIdentifier: .stepCount)!
        let today = Calendar.current.startOfDay(for: Date())
        let predicate = HKQuery.predicateForSamples(withStart: today, end: Date())

        let query = HKStatisticsQuery(
            quantityType: stepsType,
            quantitySamplePredicate: predicate,
            options: .cumulativeSum
        ) { _, result, _ in
            guard let result = result, let sum = result.sumQuantity() else { return }
            DispatchQueue.main.async {
                self.todaySteps = Int(sum.doubleValue(for: HKUnit.count()))
            }
        }

        healthStore.execute(query)
    }
}
```

## Part 5: HomeKit による IoT デバイス制御

HomeKit は Apple のスマートホームフレームワーク。照明、温度設定、スマートロック、カメラなどを統一インターフェースで制御できます。

### プロンプト例

```
「スマートホーム管理アプリを作ってください。
- HomeKit 対応デバイス（ライト、温度計、スマートロック）を一覧表示
- タップでデバイス ON/OFF 切り替え
- スライダーで照度や温度を調整
- シーン機能（帰宅モード、就寝モード）実装
- Siri 統合で音声制御対応」
```

### 実装概要

```swift
import HomeKit

class HomeKitManager: NSObject, HMHomeManagerDelegate {
    @Published var homes: [HMHome] = []
    let homeManager = HMHomeManager()

    override init() {
        super.init()
        homeManager.delegate = self
    }

    func homeManagerDidUpdateHomes(_ manager: HMHomeManager) {
        DispatchQueue.main.async {
            self.homes = manager.homes
        }
    }

    func toggleAccessory(_ accessory: HMAccessory) {
        guard let lightbulb = accessory.services.first(
            where: { $0.serviceType == HMServiceTypeLightbulb }
        )?.characteristics.first(
            where: { $0.characteristicType == HMCharacteristicTypePowerState }
        ) else { return }

        lightbulb.writeValue(!lightbulb.value as! Bool) { error in
            if let error = error {
                print("Failed to toggle: \(error)")
            }
        }
    }
}
```

### Cloud Mac フリートの活用

継続的なテスト・ビルドが必要な場合、Rork Max の Cloud Mac フリート機能を活用します。

```
利用手順：
1. Rork Max ダッシュボード > プロジェクト設定 > Cloud Build
2. 「Enable Cloud Mac Fleet」をオン
3. ビルド構成を選択
   - Development Build（開発・テスト用）
   - Release Build（App Store 公開用）
4. 自動ビルドスケジュール設定
```

### GitHub Actions による自動化ワークフロー

```yaml
name: Auto-build & Test
on:
  push:
    branches: [ main, develop ]

jobs:
  test:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build with Xcode
        run: |
          xcodebuild -scheme RorkApp \
          -configuration Debug \
          -destination 'platform=iOS Simulator,name=iPhone 15' \
          test
      - name: Upload to Testflight
        run: |
          # Testflight アップロード処理
```

### App Store Connect での承認戦略

1. **ベータテスト**: TestFlight で External Testing（最大 10,000 ユーザー）
2. **フェーズ公開**: 初日 10% → 1 週間 50% → 段階的に 100% リリース
3. **インシデント対応**: クラッシュ検出時は即座に修正・再提出
