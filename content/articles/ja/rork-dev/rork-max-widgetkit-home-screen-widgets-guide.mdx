## WidgetKit とは — ホーム画面にアプリの価値を届ける仕組み

WidgetKit は Apple が iOS 14 で導入したフレームワークで、アプリの重要な情報をホーム画面やロック画面に直接表示できます。ユーザーがアプリを開かなくても一目で最新情報を確認できるため、アプリのエンゲージメントを大幅に向上させる強力な機能です。

Rork Max を使えば、Xcode を直接操作することなく、SwiftUI ベースの WidgetKit ウィジェットをプロンプト駆動で開発できます。この記事では、Rork Max でウィジェットを設計・実装するための基礎から実践テクニックまでを解説します。

ウィジェット開発は一見複雑に感じるかもしれませんが、SwiftUI を基盤にしているため、Rork Max の AI 支援との相性は抜群です。基本的な構造を理解すれば、さまざまな種類のウィジェットを効率的に作成できるようになります。

## WidgetKit の基本アーキテクチャ

WidgetKit のウィジェットは、3つの主要コンポーネントで構成されています。それぞれの役割を理解することが、開発をスムーズに進める鍵になります。

**Timeline Provider（タイムラインプロバイダー）** は、ウィジェットに表示するデータを供給する役割を担います。いつ、どのデータを表示するかをシステムに伝えるコンポーネントです。

**Timeline Entry（タイムラインエントリ）** は、ある時点で表示するデータを格納する構造体です。日付とともに表示内容を保持します。

**Widget View（ウィジェットビュー）** は、SwiftUI で構築する見た目の部分です。ウィジェットのサイズに応じたレイアウトを定義します。

以下は、Rork Max で天気情報ウィジェットを作成する際のプロンプト例です。

```swift
// Rork Max プロンプト例:
// 「天気情報を表示するホーム画面ウィジェットを作成してください。
//   小・中・大の3サイズに対応し、現在の気温と天気アイコンを表示します」

// Timeline Provider の基本構造
struct WeatherProvider: TimelineProvider {
    // プレビュー用のプレースホルダー
    func placeholder(in context: Context) -> WeatherEntry {
        WeatherEntry(date: Date(), temperature: 22, condition: "sunny")
    }

    // ウィジェットギャラリー用のスナップショット
    func getSnapshot(in context: Context, completion: @escaping (WeatherEntry) -> Void) {
        let entry = WeatherEntry(date: Date(), temperature: 22, condition: "sunny")
        completion(entry)
    }

    // 実際のデータ取得とタイムライン構築
    func getTimeline(in context: Context, completion: @escaping (Timeline<WeatherEntry>) -> Void) {
        Task {
            let weather = await fetchCurrentWeather()
            let entry = WeatherEntry(
                date: Date(),
                temperature: weather.temp,
                condition: weather.condition
            )
            // 30分後に次の更新をリクエスト
            let nextUpdate = Calendar.current.date(byAdding: .minute, value: 30, to: Date())!
            let timeline = Timeline(entries: [entry], policy: .after(nextUpdate))
            completion(timeline)
        }
    }
}

// Timeline Entry
struct WeatherEntry: TimelineEntry {
    let date: Date
    let temperature: Int
    let condition: String // "sunny", "cloudy", "rainy" など
}
```

このように、Provider がデータを取得し、Entry に格納し、View が描画するという流れが WidgetKit の基本です。

## ウィジェットのサイズ別レイアウト設計

WidgetKit では `systemSmall`、`systemMedium`、`systemLarge` の3つの基本サイズに加え、iOS 16 以降ではロック画面用のサイズも利用できます。各サイズの特性を理解して、最適なレイアウトを設計しましょう。

```swift
// ウィジェットビューの実装
struct WeatherWidgetView: View {
    var entry: WeatherEntry
    // ウィジェットファミリーを環境変数から取得
    @Environment(\.widgetFamily) var family

    var body: some View {
        switch family {
        case .systemSmall:
            // 小サイズ: アイコンと気温のみ
            VStack(spacing: 8) {
                Image(systemName: weatherIcon(entry.condition))
                    .font(.system(size: 36))
                    .foregroundColor(.blue)
                Text("\(entry.temperature)°")
                    .font(.system(size: 28, weight: .bold))
            }
            .containerBackground(.fill.tertiary, for: .widget)

        case .systemMedium:
            // 中サイズ: 横並びで詳細情報を追加
            HStack(spacing: 16) {
                VStack(alignment: .leading) {
                    Image(systemName: weatherIcon(entry.condition))
                        .font(.system(size: 40))
                    Text("\(entry.temperature)°")
                        .font(.title.bold())
                }
                VStack(alignment: .leading, spacing: 4) {
                    Text("現在の天気")
                        .font(.caption)
                        .foregroundColor(.secondary)
                    Text(conditionLabel(entry.condition))
                        .font(.headline)
                    Text("更新: \(entry.date, style: .time)")
                        .font(.caption2)
                        .foregroundColor(.secondary)
                }
                Spacer()
            }
            .padding()
            .containerBackground(.fill.tertiary, for: .widget)

        default:
            // 大サイズ: 時間別予報などの詳細表示
            Text("詳細天気情報")
                .containerBackground(.fill.tertiary, for: .widget)
        }
    }

    // 天気状態に応じた SF Symbols アイコンを返す
    func weatherIcon(_ condition: String) -> String {
        switch condition {
        case "sunny": return "sun.max.fill"
        case "cloudy": return "cloud.fill"
        case "rainy": return "cloud.rain.fill"
        default: return "questionmark.circle"
        }
    }
}
```

小サイズでは情報を絞り込み、中・大サイズで段階的に詳細を追加するのがベストプラクティスです。Rork Max のプロンプトでサイズごとの要件を明確に指定すると、AI が適切なレイアウトを生成してくれます。

## Interactive Widget — iOS 17 のタップ操作対応

iOS 17 から導入された Interactive Widget は、ウィジェット上で直接ボタンをタップして操作できる革新的な機能です。従来はウィジェットをタップするとアプリが起動するだけでしたが、Interactive Widget ではアプリを開かずにアクションを実行できます。

```swift
// Interactive Widget の実装例: タスク管理ウィジェット
struct TaskWidgetView: View {
    var entry: TaskEntry

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text("今日のタスク")
                .font(.headline)

            ForEach(entry.tasks.prefix(3)) { task in
                HStack {
                    // ✅ Button + AppIntent でウィジェット上から直接完了操作
                    Button(intent: ToggleTaskIntent(taskId: task.id)) {
                        Image(systemName: task.isCompleted
                            ? "checkmark.circle.fill"
                            : "circle")
                        .foregroundColor(task.isCompleted ? .green : .gray)
                    }
                    .buttonStyle(.plain)

                    Text(task.title)
                        .strikethrough(task.isCompleted)
                        .font(.subheadline)

                    Spacer()
                }
            }
        }
        .padding()
        .containerBackground(.fill.tertiary, for: .widget)
    }
}

// AppIntent でウィジェットのアクションを定義
struct ToggleTaskIntent: AppIntent {
    static var title: LocalizedStringResource = "タスクの完了切替"

    @Parameter(title: "Task ID")
    var taskId: String

    func perform() async throws -> some IntentResult {
        // データストアのタスク状態を更新
        await TaskStore.shared.toggleTask(taskId)
        return .result()
    }
}
```

Interactive Widget を活用すると、ToDo の完了チェック、タイマーの開始・停止、ショートカットの実行など、さまざまな操作をホーム画面から直接行えます。Rork Max では「インタラクティブなウィジェットを作って」とプロンプトに含めるだけで、AppIntent を使った実装を生成してくれます。

さらに高度なインタラクションとして、Dynamic Island や Live Activities との連携も可能です。ウィジェットで開始した操作の進捗を Live Activities でリアルタイム表示するといった使い方は、[Rork Max × Dynamic Island・Live Activities 実装ガイド](/articles/rork-dev/rork-max-dynamic-island-live-activities-guide) で詳しく解説しています。

## App Group でアプリとウィジェット間のデータ共有

ウィジェットはアプリ本体とは別のプロセスで動作するため、データを共有するには App Group を設定する必要があります。Rork Max では、この設定も AI に指示するだけで自動的に構成されます。

```swift
// App Group を使ったデータ共有
// アプリ本体側: データの保存
class SharedDataManager {
    static let shared = SharedDataManager()
    // App Group の ID（Rork Max が自動設定）
    private let defaults = UserDefaults(
        suiteName: "group.com.yourapp.shared"
    )

    func saveTaskList(_ tasks: [TaskItem]) {
        let encoder = JSONEncoder()
        if let data = try? encoder.encode(tasks) {
            defaults?.set(data, forKey: "taskList")
        }
        // ウィジェットのタイムラインを強制リロード
        WidgetCenter.shared.reloadTimelines(ofKind: "TaskWidget")
    }
}

// ウィジェット側: データの読み込み
struct TaskProvider: TimelineProvider {
    func getTimeline(in context: Context, completion: @escaping (Timeline<TaskEntry>) -> Void) {
        let defaults = UserDefaults(suiteName: "group.com.yourapp.shared")
        var tasks: [TaskItem] = []

        if let data = defaults?.data(forKey: "taskList"),
           let decoded = try? JSONDecoder().decode([TaskItem].self, from: data) {
            tasks = decoded
        }

        let entry = TaskEntry(date: Date(), tasks: tasks)
        let timeline = Timeline(entries: [entry], policy: .atEnd)
        completion(timeline)
    }
}
```

`WidgetCenter.shared.reloadTimelines(ofKind:)` を呼ぶことで、アプリ側でデータを更新した瞬間にウィジェットの表示も更新できます。ユーザーにとってシームレスな体験を提供するために重要なポイントです。

データの永続化についてさらに深く学びたい方は、[Rork Max × SwiftData パーシステンス完全ガイド](/articles/rork-dev/rork-max-swiftdata-persistence-guide) も併せてご覧ください。

## ディープリンクとウィジェット URL の設定

ウィジェットをタップしたときにアプリ内の特定の画面を開くには、ディープリンクを設定します。小サイズのウィジェットは全体が1つのリンク、中・大サイズのウィジェットでは `Link` ビューを使って要素ごとに異なるリンクを設定できます。

```swift
// ディープリンクの設定
struct TaskWidgetView: View {
    var entry: TaskEntry
    @Environment(\.widgetFamily) var family

    var body: some View {
        if family == .systemSmall {
            // 小サイズ: ウィジェット全体に1つの URL
            SmallTaskView(entry: entry)
                .widgetURL(URL(string: "myapp://tasks")!)
        } else {
            // 中・大サイズ: 各タスクに個別リンク
            VStack(alignment: .leading) {
                ForEach(entry.tasks.prefix(3)) { task in
                    Link(destination: URL(string: "myapp://tasks/\(task.id)")!) {
                        TaskRowView(task: task)
                    }
                }
            }
            .padding()
            .containerBackground(.fill.tertiary, for: .widget)
        }
    }
}

// アプリ側の URL ハンドリング
@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .onOpenURL { url in
                    // "myapp://tasks/abc123" を解析してナビゲーション
                    if url.host == "tasks",
                       let taskId = url.pathComponents.last {
                        NavigationManager.shared.navigateTo(.taskDetail(taskId))
                    }
                }
        }
    }
}
```

ディープリンクを適切に設定することで、ウィジェットからアプリへの導線がスムーズになり、ユーザーの利便性が大きく向上します。

## Rork Max でウィジェットを作るときの実践的なプロンプトテクニック

Rork Max の AI に効率よくウィジェットを生成してもらうためには、プロンプトの書き方にいくつかのコツがあります。

まず、**ウィジェットの目的と表示する情報を明確にする** ことが重要です。「ウィジェットを作って」という曖昧な指示ではなく、「今日の歩数と消費カロリーを表示する小サイズのヘルスケアウィジェットを作って」のように具体的に伝えましょう。

次に、**サイズごとの要件を分けて記述する** と精度が上がります。

```
プロンプト例:
「家計簿アプリのウィジェットを作成してください。

小サイズ: 今月の合計支出と予算残高を表示
中サイズ: 小サイズの内容に加え、カテゴリ別の上位3つの支出を横棒グラフで表示
大サイズ: 中サイズの内容に加え、過去7日間の日別支出を折れ線グラフで表示

配色はアプリのテーマカラー（青系）に合わせてください。
タイムラインは1時間ごとに更新します。
中・大サイズではカテゴリをタップすると、アプリのそのカテゴリの詳細画面に遷移してください。」
```

このように段階的に情報量を増やす設計を伝えると、Rork Max は各サイズに最適なレイアウトを生成します。

[Rork × Figma 連携開発ガイド](/articles/rork-dev/rork-figma-mobile-ui-design-workflow) で紹介しているデザインワークフローと組み合わせると、ウィジェットの見た目も洗練されたものに仕上がります。

## よくある質問（FAQ）

### Q: ウィジェットの更新頻度はどのくらいまで短くできますか？

WidgetKit のタイムライン更新はシステムが管理しており、開発者が「毎秒更新」のような高頻度の更新を強制することはできません。Apple の公式ガイドラインでは、1日あたり40〜70回程度のリロード予算が目安とされています。リアルタイム性が必要な場合は、WidgetKit よりも Live Activities の利用を検討してください。

### Q: ウィジェットからネットワークリクエストは送れますか？

はい、Timeline Provider の `getTimeline` メソッド内で `URLSession` を使った非同期通信が可能です。ただし、バックグラウンドでの実行時間には制限があるため、レスポンスが速い API を使用し、タイムアウトを短めに設定することをおすすめします。オフライン時のフォールバックとして、App Group 経由でキャッシュデータを表示する設計にすると堅牢です。

### Q: Rork Max で作ったウィジェットは App Store の審査に通りますか？

Rork Max はネイティブ Swift / SwiftUI コードを生成するため、Apple のガイドラインに準拠したウィジェットが作成されます。審査でよく指摘されるのは「ウィジェットに広告を表示している」「ウィジェットの内容がアプリと無関係」といったケースです。ウィジェットはアプリの補助的な機能として、ユーザーに価値ある情報を提供する設計を心がけましょう。

### Q: ロック画面ウィジェット（iOS 16+）にも対応できますか？

はい、Rork Max は iOS 16 以降のロック画面ウィジェット（`accessoryCircular`、`accessoryRectangular`、`accessoryInline`）にも対応しています。ロック画面ウィジェットは表示領域が限られるため、テキストとシンプルなアイコンに絞ったデザインが推奨されます。プロンプトで「ロック画面ウィジェットも作って」と指定すれば、対応するレイアウトが追加されます。

## まとめ

WidgetKit を使ったホーム画面ウィジェットは、アプリの日常的な利用頻度とエンゲージメントを高める非常に効果的な手段です。Rork Max なら、SwiftUI ベースのウィジェット開発をプロンプト駆動で効率的に進められます。

まずは小サイズのシンプルなウィジェットから始めて、段階的に Interactive Widget やロック画面対応へと拡張していくアプローチがおすすめです。ウィジェットの体験を磨き上げることが、アプリ全体の価値向上につながります。
