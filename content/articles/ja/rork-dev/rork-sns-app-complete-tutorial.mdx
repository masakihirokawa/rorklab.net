## はじめに — この記事で作るもの

Twitter / Instagram ライクな SNS アプリを **Rork × Supabase** でゼロから構築します。完成するアプリには以下の機能が含まれます。

- **ユーザー認証**（メール・パスワード / Google ログイン）
- **投稿機能**（テキスト + 画像アップロード）
- **タイムライン表示**（フォロー中のユーザーの投稿のみ表示）
- **フォロー／フォロワー機能**
- **いいね（Likes）機能**
- **リアルタイム更新**（Supabase Realtime）
- **プロフィール画面**（アバター、自己紹介、投稿一覧）

### 対象読者

- Rork でアプリを作ったことがある方（[Rork 入門記事](/articles/rork-basics/getting-started) 読了レベル）
- Supabase の基本概念を知っている方（[バックエンド連携ガイド](/articles/rork-dev/supabase-firebase-backend) 参照）
- 本格的なソーシャル機能を持つアプリを個人開発・受託開発したい方

### 完成イメージ

```
ホームタイムライン → 投稿詳細 → ユーザープロフィール
      ↕                ↕                  ↕
  いいね/コメント    投稿作成           フォロー/解除
      ↕
  リアルタイム更新（Supabase Realtime）
```

#### テーブル設計（SQL）

Supabase の SQL Editor で以下を実行してテーブルを作成します。

```sql
-- プロフィールテーブル（auth.users と 1:1 対応）
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  username TEXT UNIQUE NOT NULL,
  display_name TEXT,
  bio TEXT DEFAULT '',
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 投稿テーブル
CREATE TABLE posts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  image_url TEXT,
  likes_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- フォロー関係テーブル
CREATE TABLE follows (
  follower_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  following_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (follower_id, following_id)
);

-- いいねテーブル
CREATE TABLE likes (
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (user_id, post_id)
);

-- いいね数の自動更新トリガー
CREATE OR REPLACE FUNCTION update_likes_count()
RETURNS TRIGGER AS $$
BEGIN
  IF TG_OP = 'INSERT' THEN
    UPDATE posts SET likes_count = likes_count + 1 WHERE id = NEW.post_id;
  ELSIF TG_OP = 'DELETE' THEN
    UPDATE posts SET likes_count = likes_count - 1 WHERE id = OLD.post_id;
  END IF;
  RETURN NULL;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER likes_count_trigger
AFTER INSERT OR DELETE ON likes
FOR EACH ROW EXECUTE FUNCTION update_likes_count();

-- RLS（Row Level Security）有効化
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
ALTER TABLE follows ENABLE ROW LEVEL SECURITY;
ALTER TABLE likes ENABLE ROW LEVEL SECURITY;

-- RLS ポリシー（全員がプロフィール・投稿を閲覧可能、自分のデータのみ変更可）
CREATE POLICY "profiles_read_all" ON profiles FOR SELECT USING (true);
CREATE POLICY "profiles_update_own" ON profiles FOR UPDATE USING (auth.uid() = id);
CREATE POLICY "posts_read_all" ON posts FOR SELECT USING (true);
CREATE POLICY "posts_insert_own" ON posts FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "posts_delete_own" ON posts FOR DELETE USING (auth.uid() = user_id);
CREATE POLICY "follows_read_all" ON follows FOR SELECT USING (true);
CREATE POLICY "follows_manage_own" ON follows FOR ALL USING (auth.uid() = follower_id);
CREATE POLICY "likes_read_all" ON likes FOR SELECT USING (true);
CREATE POLICY "likes_manage_own" ON likes FOR ALL USING (auth.uid() = user_id);
```

> **ポイント**: `likes_count_trigger` を使うことで、いいね数の集計クエリが不要になりパフォーマンスが向上します。

```typescript
// lib/supabase.ts — Supabase クライアントの初期化
import { createClient } from '@supabase/supabase-js';
import AsyncStorage from '@react-native-async-storage/async-storage';
import { AppState } from 'react-native';

const SUPABASE_URL = process.env.EXPO_PUBLIC_SUPABASE_URL!;
const SUPABASE_ANON_KEY = process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!;

export const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY, {
  auth: {
    storage: AsyncStorage,
    autoRefreshToken: true,
    persistSession: true,
    detectSessionInUrl: false,
  },
});

// アプリがフォアグラウンドに戻ったときにセッションを自動更新
AppState.addEventListener('change', (state) => {
  if (state === 'active') {
    supabase.auth.startAutoRefresh();
  } else {
    supabase.auth.stopAutoRefresh();
  }
});
```

### Step 3: タイムライン機能（フォロー中のユーザーの投稿）

SNS の核心部分です。「フォロー中のユーザーの投稿のみ表示する」タイムラインを実装します。

**Rork プロンプト（タイムライン）:**
```
ホームタイムライン画面を実装してください。

- Supabase の posts テーブルから、自分がフォローしているユーザーの投稿のみ取得
- 投稿にはアバター・ユーザー名・本文・画像（任意）・いいね数・投稿日時を表示
- 下にスクロールで追加ロード（無限スクロール、1回 20件）
- 投稿をタップで詳細画面に遷移
- いいねボタン（タップで即時反映、オプティミスティック UI）
- 画面上部に「投稿する」ボタン
```

タイムライン取得のカスタムフック例:

```typescript
// hooks/useTimeline.ts — タイムライン取得フック
import { useState, useEffect, useCallback } from 'react';
import { supabase } from '@/lib/supabase';

const PAGE_SIZE = 20;

export function useTimeline(userId: string) {
  const [posts, setPosts] = useState<Post[]>([]);
  const [loading, setLoading] = useState(true);
  const [hasMore, setHasMore] = useState(true);
  const [page, setPage] = useState(0);

  const fetchPosts = useCallback(async (pageNum: number) => {
    // フォロー中のユーザー ID 一覧を取得
    const { data: followData } = await supabase
      .from('follows')
      .select('following_id')
      .eq('follower_id', userId);

    const followingIds = followData?.map(f => f.following_id) ?? [];
    // 自分自身の投稿も含める
    followingIds.push(userId);

    const { data, error } = await supabase
      .from('posts')
      .select(`
        id, content, image_url, likes_count, created_at,
        profiles!user_id (id, username, display_name, avatar_url),
        likes!left (user_id)
      `)
      .in('user_id', followingIds)
      .order('created_at', { ascending: false })
      .range(pageNum * PAGE_SIZE, (pageNum + 1) * PAGE_SIZE - 1);

    if (error) throw error;

    if (data.length < PAGE_SIZE) setHasMore(false);

    return data.map(post => ({
      ...post,
      isLiked: post.likes?.some(l => l.user_id === userId) ?? false,
    }));
  }, [userId]);

  useEffect(() => {
    fetchPosts(0).then(data => {
      setPosts(data);
      setLoading(false);
    });
  }, [fetchPosts]);

  const loadMore = async () => {
    if (!hasMore) return;
    const nextPage = page + 1;
    const newPosts = await fetchPosts(nextPage);
    setPosts(prev => [...prev, ...newPosts]);
    setPage(nextPage);
  };

  return { posts, loading, hasMore, loadMore, setPosts };
}
```

> **解説**: Supabase の `select` で関連テーブルを JOIN し、`likes!left` で「自分がいいねしているか」を1クエリで取得しています。`!left` は LEFT JOIN を意味し、いいねがない投稿も漏れなく取得できます。
>
> **期待される出力**: `posts` に `isLiked: boolean` フィールドが追加された配列が返される。

### Step 4: いいね機能（オプティミスティック UI）

いいねは「タップした瞬間に UI を更新し、API 処理はバックグラウンドで行う」オプティミスティック UI で実装すると体験が良くなります。

**Rork プロンプト（いいね機能）:**
```
いいね機能を実装してください。

- ハートアイコンをタップで即座にいいね数が増減（オプティミスティック UI）
- Supabase の likes テーブルに追加/削除
- API エラー時は元の状態に戻す（ロールバック）
- いいね済みのハートは赤く、未いいねは灰色で表示
```

```typescript
// hooks/useLike.ts — いいね機能フック
import { useState } from 'react';
import { supabase } from '@/lib/supabase';

export function useLike(
  postId: string,
  initialIsLiked: boolean,
  initialCount: number,
  userId: string,
  onUpdate: (postId: string, isLiked: boolean, count: number) => void
) {
  const [isLiked, setIsLiked] = useState(initialIsLiked);
  const [count, setCount] = useState(initialCount);

  const toggleLike = async () => {
    // オプティミスティック更新（即座に UI を変更）
    const newIsLiked = !isLiked;
    const newCount = newIsLiked ? count + 1 : count - 1;
    setIsLiked(newIsLiked);
    setCount(newCount);
    onUpdate(postId, newIsLiked, newCount);

    try {
      if (newIsLiked) {
        // いいね追加
        const { error } = await supabase
          .from('likes')
          .insert({ user_id: userId, post_id: postId });
        if (error) throw error;
      } else {
        // いいね削除
        const { error } = await supabase
          .from('likes')
          .delete()
          .eq('user_id', userId)
          .eq('post_id', postId);
        if (error) throw error;
      }
    } catch (error) {
      // エラー時はロールバック
      setIsLiked(initialIsLiked);
      setCount(initialCount);
      onUpdate(postId, initialIsLiked, initialCount);
      console.error('いいねの更新に失敗:', error);
    }
  };

  return { isLiked, count, toggleLike };
}
```

### Step 5: リアルタイム更新（Supabase Realtime）

フォロー中のユーザーが新しい投稿をすると、自動でタイムラインに表示されるようにします。

**Rork プロンプト（リアルタイム）:**
```
Supabase Realtime を使って、タイムラインをリアルタイム更新してください。

- フォロー中のユーザーが投稿するとタイムラインの先頭に表示
- 「新しい投稿があります」バナーを上部に表示し、タップで最上部にスクロール
- WebSocket 接続をアプリ終了時にクリーンアップ
```

```typescript
// タイムライン画面での Realtime サブスクリプション
useEffect(() => {
  const channel = supabase
    .channel('public:posts')
    .on(
      'postgres_changes',
      {
        event: 'INSERT',
        schema: 'public',
        table: 'posts',
        // フォロー中のユーザーの投稿のみ受信
        filter: `user_id=in.(${followingIds.join(',')})`,
      },
      (payload) => {
        // 新しい投稿をタイムラインの先頭に追加
        setPosts(prev => [payload.new as Post, ...prev]);
        setNewPostCount(n => n + 1);
      }
    )
    .subscribe();

  // クリーンアップ（アプリ終了 / 画面離脱時）
  return () => {
    supabase.removeChannel(channel);
  };
}, [followingIds]);
```

## よくあるエラーと対処法

### エラー 1: `RLS policy violation` — データの取得・書き込みができない

**原因**: Supabase の Row Level Security が有効なのに、ポリシーが正しく設定されていない。

**対処**:
```sql
-- ポリシーの確認
SELECT * FROM pg_policies WHERE tablename = 'posts';

-- 全員に読み取りを許可するポリシー（確認用）
CREATE POLICY "allow_read" ON posts FOR SELECT USING (true);
```

### エラー 2: リアルタイム更新が届かない

**原因**: Supabase Dashboard で Realtime が無効になっている、または `filter` の書き方が間違っている。

**対処**:
1. Supabase Dashboard → Database → Replication → `posts` テーブルの Realtime を ON にする
2. フィルタ構文を確認: `user_id=in.(id1,id2)` の形式が必要

### エラー 3: 画像が表示されない（Supabase Storage）

**原因**: Storage バケットの公開設定が Private になっている。

**対処**:
```sql
-- 公開バケットに変更
UPDATE storage.buckets SET public = true WHERE id = 'post-images';
```

または Dashboard → Storage → Policies で、`SELECT` ポリシーを追加:
```sql
CREATE POLICY "public_images" ON storage.objects FOR SELECT USING (bucket_id = 'post-images');
```

### エラー 4: `Cannot read property 'data' of undefined`

**原因**: 非同期処理のエラーハンドリングが不足している。Supabase クエリは `{ data, error }` を返すので、`error` を必ず確認する。

**対処**: 全クエリに対して以下のパターンを徹底する:
```typescript
const { data, error } = await supabase.from('posts').select('*');
if (error) {
  console.error('クエリエラー:', error.message);
  return; // ← これがないと data が null でもクラッシュしない
}
// data を安全に使用
```

## FAQ

**Q1. Supabase の無料プランで SNS アプリを運営できますか？**

A. 個人プロジェクトや MVP 段階であれば可能です。無料プランは MAU 50,000、DB 500MB、Storage 1GB、Realtime 接続 200 件同時まで対応しています。ユーザーが増えてきたら Pro プラン（$25/月）へのアップグレードを検討してください。

**Q2. タイムラインのクエリが重い・遅い場合はどうすればいいですか？**

A. インデックスの追加が効果的です。`posts.user_id` と `posts.created_at` に複合インデックスを作成してください。
```sql
CREATE INDEX idx_posts_user_created ON posts(user_id, created_at DESC);
CREATE INDEX idx_follows_follower ON follows(follower_id);
```

**Q3. 画像を圧縮してからアップロードする方法はありますか？**

A. `expo-image-manipulator` を使って、アップロード前にリサイズ・品質調整が可能です。Rork に「画像は最大幅 1080px、JPEG 品質 80% に圧縮してからアップロードして」とプロンプトを入力するだけで実装できます。

**Q4. App Store に申請する際、SNS アプリで注意すべき点はありますか？**

A. 主に以下の点に注意してください。① ユーザーが投稿したコンテンツのモデレーション機能（通報・ブロック）が必要、② Apple のプライバシーポリシー要件（データの収集・利用目的の明示）、③ 18 歳以上向けコンテンツを含む場合は年齢制限の設定。詳細は App Store 申請ガイド を参照してください。

**Q5. リアルタイム更新が多すぎてバッテリー消費が心配です。**

A. Supabase Realtime は WebSocket を使うため、常時接続になります。バッテリー消費を抑えるには「フォアグラウンドのときのみ購読、バックグラウンドでは切断」する実装を推奨します。前述の `AppState` を使った制御（`startAutoRefresh` / `stopAutoRefresh`）がそのまま使えます。

**Q6. フォロー推薦機能（おすすめユーザー）はどう実装しますか？**

A. 基本的なアプローチとして「まだフォローしていない、最近アクティブなユーザーを表示する」クエリを使います。

```sql
-- フォローしていないアクティブユーザーを最大 10 件取得
SELECT p.* FROM profiles p
LEFT JOIN follows f ON f.following_id = p.id AND f.follower_id = :current_user_id
WHERE f.following_id IS NULL
  AND p.id != :current_user_id
ORDER BY (
  SELECT COUNT(*) FROM posts WHERE user_id = p.id
    AND created_at > NOW() - INTERVAL '30 days'
) DESC
LIMIT 10;
```

## まとめ

この記事では、Rork × Supabase を使った本格的な SNS アプリの構築方法を解説しました。

- **Supabase のテーブル設計**（posts・follows・likes・profiles）と RLS でセキュリティを確保
- **Rork のプロンプト**で画面・コンポーネント・ロジックを自動生成
- **オプティミスティック UI** でいいねをリアルタイムに反映
- **Supabase Realtime** で新着投稿をリアルタイム配信
- **Supabase Storage** で画像アップロードを実現

Rork を使うことで、本来なら数週間かかるような SNS アプリのバックボーンを数時間で構築できます。あとはデザインのカスタマイズや独自機能を追加するだけで、本格的なプロダクトとして仕上げることができます。
