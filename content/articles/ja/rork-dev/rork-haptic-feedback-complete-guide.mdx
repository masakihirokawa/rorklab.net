## はじめに：触覚フィードバックがアプリ体験を変える

スマートフォンアプリを使っているとき、ボタンを押した瞬間にわずかな振動を感じたことはないでしょうか。あの微細な振動こそが **Haptic Feedback（触覚フィードバック）** です。

Appleは iPhone 7（2016年）から「Taptic Engine」と呼ばれる精密な触覚エンジンを搭載し始め、現在のiPhoneはきわめて細かな振動パターンを表現できるようになっています。Androidも同様にHaptic APIを整備しており、両プラットフォームで高品質な触覚体験を提供できる環境が整っています。

Rorkで開発するアプリにも、適切な触覚フィードバックを組み込むことで、ユーザーが「なんとなく使いやすい」と感じる洗練されたUXを実現できます。本記事では、**RorkアプリにおけるHaptic Feedbackの実装方法を基礎から実践まで**丁寧に解説します。

この記事で学べること：
- 触覚フィードバックの種類と適切な使い分け
- React NativeのAPIを使ったRorkでの実装方法
- iOS/Android両対応の実装パターン
- UXを損なわないHaptic設計の原則
- よくあるミスと回避方法

**対象読者**: Rorkでのアプリ開発経験があり、UX品質をワンランク上げたい方。React Nativeの基本的な理解があると読みやすいです。

## Haptic Feedbackの種類と使い分け

### iOSの触覚フィードバック分類

iOSのHaptic Feedbackは大きく3つのカテゴリに分類されます。

**① Impact Feedback（衝撃フィードバック）**
物体が衝突するような感触を表現します。ドラッグ＆ドロップでアイテムを配置したとき、セルを長押ししたときなど、「何かが確実に起きた」という感触に使います。

| 強度 | 用途の例 |
|------|----------|
| `light` | 小さなUI要素のタップ、チェックボックス |
| `medium` | 一般的なボタン操作、リストセルの選択 |
| `heavy` | 重要なアクション、削除確認 |
| `rigid` | 固い感触が必要な場面（iOS 13以降） |
| `soft` | 柔らかい感触（iOS 13以降） |

**② Notification Feedback（通知フィードバック）**
操作の結果をユーザーに知らせるために使います。

| タイプ | 用途 |
|--------|------|
| `success` | 処理完了、保存成功 |
| `warning` | 注意が必要な操作 |
| `error` | エラー発生、入力値が不正 |

**③ Selection Feedback（選択フィードバック）**
値が変化したときの微細な振動。ピッカーやスライダーで値が変わるたびに使います。

### Androidの触覚フィードバック

Android では `HapticFeedbackConstants` を使用します。Rorkで使用するReact Native の `react-native-haptic-feedback` ライブラリはこの差異を吸収してくれるため、クロスプラットフォームで統一した記述が可能です。

## Rorkでの実装方法

### Step 1：ライブラリの確認

Rorkが採用しているExpo/React Native環境では、Expoが提供する `expo-haptics` が最も手軽に使えます。

```bash
# Expo管理プロジェクトではデフォルトで利用可能
# Rorkプロジェクトではexpo-hapticsが既に含まれているケースが多い
npx expo install expo-haptics
```

Rorkのビルト環境ではすでにexpo系パッケージが多く含まれているため、多くの場合は追加インストール不要です。

### Step 2：基本的なImpact Feedbackの実装

```tsx
import * as Haptics from 'expo-haptics';
import { TouchableOpacity, Text, StyleSheet } from 'react-native';

// ボタンコンポーネントに触覚フィードバックを追加
const HapticButton = ({ onPress, children, intensity = 'medium' }) => {
  const handlePress = async () => {
    // タップ時にHapticフィードバックを発火
    await Haptics.impactAsync(
      intensity === 'light'
        ? Haptics.ImpactFeedbackStyle.Light
        : intensity === 'heavy'
        ? Haptics.ImpactFeedbackStyle.Heavy
        : Haptics.ImpactFeedbackStyle.Medium
    );
    onPress?.();
  };

  return (
    <TouchableOpacity style={styles.button} onPress={handlePress}>
      <Text style={styles.label}>{children}</Text>
    </TouchableOpacity>
  );
};

const styles = StyleSheet.create({
  button: {
    backgroundColor: '#007AFF',
    paddingHorizontal: 24,
    paddingVertical: 12,
    borderRadius: 12,
    alignItems: 'center',
  },
  label: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

// 使用例
// <HapticButton onPress={() => console.log('pressed')} intensity="medium">
//   保存する
// </HapticButton>
```

### Step 3：操作結果のNotification Feedback

```tsx
import * as Haptics from 'expo-haptics';
import { useState } from 'react';

// フォーム送信時の結果フィードバック例
const FormSubmitExample = () => {
  const [status, setStatus] = useState<'idle' | 'success' | 'error'>('idle');

  const handleSubmit = async () => {
    try {
      // APIリクエストなど非同期処理
      await submitForm();

      // 成功時: 心地よい達成感のある振動
      await Haptics.notificationAsync(
        Haptics.NotificationFeedbackType.Success
      );
      setStatus('success');
    } catch (err) {
      // エラー時: 注意を促す振動
      await Haptics.notificationAsync(
        Haptics.NotificationFeedbackType.Error
      );
      setStatus('error');
    }
  };

  return (/* ... */);
};
```

### Step 4：ピッカーやスライダーのSelection Feedback

```tsx
import * as Haptics from 'expo-haptics';
import { useState } from 'react';
import { View, Text, ScrollView } from 'react-native';

// スクロールピッカーで値が変わるたびに振動させる例
const HapticPicker = ({ items, onSelect }) => {
  const [selectedIndex, setSelectedIndex] = useState(0);

  const handleScroll = (index: number) => {
    if (index !== selectedIndex) {
      // 選択値が変わったタイミングで軽い振動
      Haptics.selectionAsync();
      setSelectedIndex(index);
      onSelect?.(items[index]);
    }
  };

  return (
    <View>
      {items.map((item, index) => (
        <Text
          key={index}
          onPress={() => handleScroll(index)}
          style={index === selectedIndex ? styles.selected : styles.item}
        >
          {item}
        </Text>
      ))}
    </View>
  );
};

// 期待する動作:
// → ピッカーで選択値が変わるたびに "ティック" のような微細な振動が発生
// → ユーザーは視覚的確認なしに操作が受け付けられたことを認識できる
```

## UXを損なわないHaptic設計の原則

### 原則1：過度な振動は禁物

ユーザーをもっとも悩ませるのは「振動しすぎるアプリ」です。すべてのタップに重い振動を設定すると、すぐに煩わしく感じられ、設定からHapticをオフにされてしまいます。

**推奨する振動頻度の目安:**
- 重要な操作の完了（保存、送信、決済）: `Notification.Success`
- エラー発生: `Notification.Error`
- 主要なボタンタップ: `Impact.Medium` または `Impact.Light`
- リスト選択・ピッカー操作: `Selection`
- 単なる画面遷移: **振動なし**

### 原則2：プラットフォームの差異を考慮する

iOSのTaptic EngineとAndroidの振動モーターは特性が異なります。iOSでは繊細な表現が可能ですが、Androidでは同じAPIでも粗い振動になることがあります。

```tsx
import { Platform } from 'react-native';
import * as Haptics from 'expo-haptics';

// プラットフォームに応じてフィードバックを調整する例
const adaptiveHaptic = async (type: 'success' | 'error' | 'tap') => {
  if (Platform.OS === 'ios') {
    // iOSは細かいコントロールが可能
    switch (type) {
      case 'success':
        await Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
        break;
      case 'error':
        await Haptics.notificationAsync(Haptics.NotificationFeedbackType.Error);
        break;
      case 'tap':
        await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
        break;
    }
  } else {
    // Androidでは軽めに統一（過剰振動を避ける）
    if (type !== 'tap') {
      await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
    }
  }
};
```

### 原則3：ユーザー設定を尊重する

iOSには「設定 > サウンドと触覚」でシステム全体のHapticをオフにできる機能があります。`expo-haptics` はこのシステム設定を自動的に尊重するため、アプリ側での追加対応は不要です。ただし、アプリ独自の「触覚フィードバック ON/OFF」設定をUI上に用意することも、UX向上に効果的です。

## よくあるエラーと対処法

### エラー1：「Haptics is not available on this device」

古い端末や一部のAndroid端末ではHaptic APIが未対応の場合があります。try-catchで適切にエラーハンドリングしましょう。

```tsx
const safeHaptic = async () => {
  try {
    await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
  } catch {
    // Haptic非対応端末では静かに無視する
    // エラーをユーザーに見せる必要はない
  }
};
```

### エラー2：Androidで振動しない

`AndroidManifest.xml` に `VIBRATE` パーミッションが必要です。Rorkの標準テンプレートには含まれていることが多いですが、確認しましょう。

```xml
<!-- android/app/src/main/AndroidManifest.xml -->
<uses-permission android:name="android.permission.VIBRATE" />
```

### エラー3：シミュレーターで動作しない

iOSシミュレーターはHapticに対応していません。実機でテストしてください。Android エミュレーターは一部対応しています。

## 応用：カスタムHapticパターンの設計

複雑な操作シーケンスに対して、複数のHapticを組み合わせてカスタムパターンを作ることができます。

```tsx
import * as Haptics from 'expo-haptics';

// ゲームのスコア達成時など、特別な演出として使う例
const celebrationHaptic = async () => {
  // 連続した振動で "達成感" を演出
  await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light);
  await new Promise(r => setTimeout(r, 100));
  await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
  await new Promise(r => setTimeout(r, 100));
  await Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
};

// 使用例: ゲームクリア時
// await celebrationHaptic();
// → 軽い → 中 → 成功通知 の3段階振動でお祝い演出
```

## FAQ

**Q. Haptic FeedbackはApp Storeの審査に影響しますか？**

A. 触覚フィードバック自体は審査上の問題になりません。ただし、Appleの [Human Interface Guidelines（HIG）](https://developer.apple.com/design/human-interface-guidelines/playing-haptics) に沿った使い方をすることが推奨されています。「操作の確認」「エラー通知」「選択変更」という3つの用途に限定して使うと無難です。

**Q. expo-hapticsとreact-native-haptic-feedbackの違いは何ですか？**

A. `expo-haptics` はExpoのマネージドワークフローで動作し、追加のネイティブ設定が不要です。`react-native-haptic-feedback` はより細かいAndroid向けの設定が可能ですが、ネイティブ設定が必要です。Rorkプロジェクトでは `expo-haptics` の使用が推奨です。

**Q. ゲームアプリで連続的な振動を実装したいのですが、バッテリーへの影響は？**

A. 頻繁なHaptic発火はバッテリーと端末温度に影響します。ゲームのヒット判定など頻度が高い場合は、`Impact.Light` を使う、または30fps以上は間引くなど制御することを推奨します。

## まとめと次のステップ

触覚フィードバックは、ユーザーが意識しないままアプリの使いやすさを支えている重要な要素です。適切に実装することで：

- **操作の確信度が上がる**（タップが受け付けられたとわかる）
- **エラーが直感的に伝わる**（エラー振動で視覚に頼らず気づける）
- **全体的な完成度が上がる**（細部へのこだわりがユーザーに伝わる）

まずは既存の主要ボタンに `Impact.Medium` を追加し、保存完了に `Notification.Success` を設定するところから始めてみてください。小さな変更でもユーザーの体験は大きく変わります。
