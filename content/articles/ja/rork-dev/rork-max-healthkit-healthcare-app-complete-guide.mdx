## HealthKit の基礎知識

### HealthKit とは

HealthKit は Apple が提供するヘルスデータプラットフォームです。iPhone のヘルスケアアプリを中心に、Apple Watch・サードパーティアプリのデータを一元管理します。

HealthKit のデータは大きく3種類に分類されます。

- **HKQuantityType**（数量型）: 歩数・心拍数・消費カロリーなど数値で表現できるデータ
- **HKCategoryType**（カテゴリ型）: 睡眠状態・月経周期など選択肢から選ぶデータ
- **HKCorrelationType**（相関型）: 血圧（収縮期+拡張期）など複数の数量が組み合わさったデータ

### データのアクセス権限モデル

HealthKit は読み取りと書き込みを別々に管理します。ユーザーは各データ種別ごとに個別に許可・拒否を選択できます。**アプリは許可されたデータ以外にアクセスできない仕様**になっており、許可されていないデータについてはそもそも存在するかどうかも確認できません。

プライバシー保護の観点から、HealthKit データを第三者サービスに転送する際は必ずユーザーへの説明と同意取得が必要です。

### react-native-health の初期化

HealthKit ストアとの接続を管理するユーティリティを作成します。

```typescript
// src/lib/healthKit.ts
import AppleHealthKit, {
  HealthKitPermissions,
  HealthValue,
  HealthInputOptions,
} from 'react-native-health';
import { Platform } from 'react-native';

// 取得・書き込みするデータ種別を定義
const PERMISSIONS: HealthKitPermissions = {
  permissions: {
    read: [
      AppleHealthKit.Constants.Permissions.Steps,
      AppleHealthKit.Constants.Permissions.HeartRate,
      AppleHealthKit.Constants.Permissions.SleepAnalysis,
      AppleHealthKit.Constants.Permissions.ActiveEnergyBurned,
      AppleHealthKit.Constants.Permissions.DistanceWalkingRunning,
      AppleHealthKit.Constants.Permissions.FlightsClimbed,
      AppleHealthKit.Constants.Permissions.Weight,
    ],
    write: [
      AppleHealthKit.Constants.Permissions.Weight,
      AppleHealthKit.Constants.Permissions.ActiveEnergyBurned,
    ],
  },
};

/**
 * HealthKit を初期化し、権限をリクエストする
 * @returns 初期化成功時は true
 */
export const initHealthKit = (): Promise<boolean> => {
  return new Promise((resolve, reject) => {
    if (Platform.OS !== 'ios') {
      // Android では HealthKit は使用不可（Google Health Connect を使用）
      reject(new Error('HealthKit は iOS 専用です'));
      return;
    }

    AppleHealthKit.initHealthKit(PERMISSIONS, (error: string) => {
      if (error) {
        reject(new Error(error));
        return;
      }
      resolve(true);
    });
  });
};
```

## 歩数データの取得と表示

### 今日の歩数を取得する

```typescript
// src/hooks/useStepCount.ts
import { useState, useEffect, useCallback } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';

interface StepData {
  today: number;
  weekly: { date: string; count: number }[];
  isLoading: boolean;
  error: string | null;
}

export const useStepCount = () => {
  const [data, setData] = useState<StepData>({
    today: 0,
    weekly: [],
    isLoading: true,
    error: null,
  });

  const fetchTodaySteps = useCallback((): Promise<number> => {
    return new Promise((resolve, reject) => {
      const options = {
        date: new Date().toISOString(),
        includeManuallyAdded: true,
      };

      AppleHealthKit.getStepCount(options, (err: string, result: HealthValue) => {
        if (err) {
          reject(new Error(err));
          return;
        }
        // result.value に今日の歩数が入る（例: 8432）
        resolve(Math.round(result.value));
      });
    });
  }, []);

  const fetchWeeklySteps = useCallback((): Promise<{ date: string; count: number }[]> => {
    return new Promise((resolve, reject) => {
      const startDate = new Date();
      startDate.setDate(startDate.getDate() - 6); // 7日前
      startDate.setHours(0, 0, 0, 0);

      const options = {
        startDate: startDate.toISOString(),
        endDate: new Date().toISOString(),
        includeManuallyAdded: true,
      };

      AppleHealthKit.getDailyStepCountSamples(
        options,
        (err: string, results: HealthValue[]) => {
          if (err) {
            reject(new Error(err));
            return;
          }

          const formatted = results.map((item) => ({
            date: new Date(item.startDate).toLocaleDateString('ja-JP', {
              month: 'numeric',
              day: 'numeric',
            }),
            count: Math.round(item.value),
          }));

          resolve(formatted);
        }
      );
    });
  }, []);

  useEffect(() => {
    const load = async () => {
      try {
        const [today, weekly] = await Promise.all([
          fetchTodaySteps(),
          fetchWeeklySteps(),
        ]);
        setData({ today, weekly, isLoading: false, error: null });
      } catch (e) {
        setData((prev) => ({
          ...prev,
          isLoading: false,
          error: e instanceof Error ? e.message : '不明なエラー',
        }));
      }
    };

    load();
  }, [fetchTodaySteps, fetchWeeklySteps]);

  return data;
};
```

## 睡眠データ・心拍数の記録

### 睡眠分析データの取得

HealthKit の睡眠データは `HKCategoryValueSleepAnalysis` として保存されています。値は以下の状態を含みます。

- `INBED`（0）: ベッドにいる状態
- `ASLEEP`（1）: 睡眠中（Apple Watch なしの場合はこれのみ）
- `AWAKE`（2）: 覚醒
- `CORE`（3）: コア睡眠
- `DEEP`（4）: 深い睡眠
- `REM`（5）: REM睡眠

```typescript
// src/hooks/useSleepData.ts
import { useState, useEffect } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';
import { startOfDay, endOfDay, subDays, differenceInMinutes } from 'date-fns';

interface SleepSession {
  date: string;
  totalMinutes: number;
  deepMinutes: number;
  remMinutes: number;
  coreMinutes: number;
}

export const useSleepData = (days: number = 7) => {
  const [sessions, setSessions] = useState<SleepSession[]>([]);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    const startDate = startOfDay(subDays(new Date(), days - 1)).toISOString();
    const endDate = endOfDay(new Date()).toISOString();

    AppleHealthKit.getSleepSamples(
      { startDate, endDate },
      (err: string, results: HealthValue[]) => {
        if (err) {
          console.error('睡眠データ取得エラー:', err);
          setIsLoading(false);
          return;
        }

        // 日付ごとに睡眠セッションを集計
        const dailyMap: Record<string, SleepSession> = {};

        results.forEach((sample) => {
          const dateKey = new Date(sample.startDate).toLocaleDateString('ja-JP');
          const durationMinutes = differenceInMinutes(
            new Date(sample.endDate),
            new Date(sample.startDate)
          );

          if (!dailyMap[dateKey]) {
            dailyMap[dateKey] = {
              date: dateKey,
              totalMinutes: 0,
              deepMinutes: 0,
              remMinutes: 0,
              coreMinutes: 0,
            };
          }

          // value で睡眠ステージを判定
          switch (sample.value) {
            case 'DEEP':
              dailyMap[dateKey].deepMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
            case 'REM':
              dailyMap[dateKey].remMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
            case 'CORE':
            case 'ASLEEP':
              dailyMap[dateKey].coreMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
          }
        });

        setSessions(Object.values(dailyMap).sort((a, b) => a.date.localeCompare(b.date)));
        setIsLoading(false);
      }
    );
  }, [days]);

  return { sessions, isLoading };
};
```

### 心拍数のリアルタイム取得

```typescript
// src/hooks/useHeartRate.ts
import { useState, useEffect, useRef } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';
import { subHours } from 'date-fns';

interface HeartRateData {
  current: number | null;
  resting: number | null;
  samples: { time: string; bpm: number }[];
}

export const useHeartRate = () => {
  const [data, setData] = useState<HeartRateData>({
    current: null,
    resting: null,
    samples: [],
  });

  useEffect(() => {
    const now = new Date();
    const sixHoursAgo = subHours(now, 6);

    // 過去6時間の心拍数サンプルを取得
    AppleHealthKit.getHeartRateSamples(
      {
        startDate: sixHoursAgo.toISOString(),
        endDate: now.toISOString(),
        limit: 100,
        ascending: true,
      },
      (err: string, results: HealthValue[]) => {
        if (err || !results.length) return;

        const samples = results.map((r) => ({
          time: new Date(r.startDate).toLocaleTimeString('ja-JP', {
            hour: '2-digit',
            minute: '2-digit',
          }),
          bpm: Math.round(r.value),
        }));

        const current = samples[samples.length - 1]?.bpm ?? null;
        setData((prev) => ({ ...prev, current, samples }));
      }
    );

    // 安静時心拍数（Resting Heart Rate）を別途取得
    AppleHealthKit.getRestingHeartRate(
      { date: now.toISOString() },
      (err: string, result: HealthValue) => {
        if (err || !result) return;
        setData((prev) => ({ ...prev, resting: Math.round(result.value) }));
      }
    );
  }, []);

  return data;
};
```

## バックグラウンド配信とウィジェット連携

### HealthKit バックグラウンド配信の設定

HealthKit は新しいデータが記録された際にアプリをバックグラウンドで起動できます。これにより、Apple Watch が歩数を記録するたびにアプリのデータが更新されます。

```typescript
// src/lib/healthKitBackground.ts
import AppleHealthKit from 'react-native-health';
import { AppRegistry } from 'react-native';

/**
 * バックグラウンド配信を有効化する
 * アプリ起動時（App.tsx の useEffect）で一度だけ呼び出す
 */
export const enableBackgroundDelivery = () => {
  // 歩数のバックグラウンド配信を有効化（1時間ごとに更新）
  AppleHealthKit.enableBackgroundDelivery(
    AppleHealthKit.Constants.Permissions.Steps,
    AppleHealthKit.Constants.QueryBundleQuery.HOUR,
    (err: string, result: boolean) => {
      if (err) {
        console.error('バックグラウンド配信有効化失敗:', err);
        return;
      }
      console.log('バックグラウンド配信有効化:', result);
    }
  );

  // 心拍数のバックグラウンド配信（即時通知）
  AppleHealthKit.enableBackgroundDelivery(
    AppleHealthKit.Constants.Permissions.HeartRate,
    AppleHealthKit.Constants.QueryBundleQuery.IMMEDIATE,
    (err: string, result: boolean) => {
      if (err) return;
      console.log('心拍バックグラウンド配信:', result);
    }
  );
};
```

### iOS ウィジェット向けデータ共有（App Groups）

ウィジェットとアプリ間でデータを共有するには **App Groups** を使用します。`react-native-shared-group-preferences` を導入し、Shared Container にデータを書き込みます。

```bash
npm install react-native-shared-group-preferences
```

```typescript
// src/lib/widgetDataSync.ts
import SharedGroupPreferences from 'react-native-shared-group-preferences';

const APP_GROUP_ID = 'group.com.yourdomain.healthapp';

/**
 * ウィジェット向けにヘルスデータをShared Containerへ書き込む
 */
export const syncDataToWidget = async (data: {
  steps: number;
  heartRate: number | null;
  sleepHours: number;
}) => {
  try {
    await SharedGroupPreferences.setItem(
      'healthWidgetData',
      JSON.stringify({
        ...data,
        updatedAt: new Date().toISOString(),
      }),
      APP_GROUP_ID
    );
  } catch (e) {
    console.error('ウィジェットデータ同期失敗:', e);
  }
};
```

## プライバシー設計と App Store 審査対策

### HealthKit 利用ポリシーの遵守

HealthKit を使用するアプリは Apple の審査が厳格です。プライバシー設計については[Rork アプリのプライバシー対策とデータ保護 完全ガイド](/articles/rork-dev/rork-app-privacy-data-protection-guide)も合わせて参照してください。以下のルールを必ず守ってください。

**絶対にやってはいけないこと**:

- HealthKit データを広告や第三者マーケティングに使用すること
- HealthKit データを Apple が承認していない方法でサーバーへ送信すること
- HealthKit データを他のユーザーのデータと組み合わせてプロファイリングすること

**やるべきこと**:

- プライバシーポリシーに HealthKit データの利用目的を明示する
- データを送信する場合は、必ずユーザーに明示的な同意を得る
- 可能な限りデータをデバイス上で処理し、サーバーへの送信を最小化する

### App Store 審査チェックリスト

```markdown
# HealthKit アプリ審査チェックリスト

## 必須対応
- [ ] NSHealthShareUsageDescription が具体的（「健康データを読み取る」だけでは却下される）
- [ ] NSHealthUpdateUsageDescription が具体的（書き込み権限を使う場合のみ）
- [ ] プライバシーポリシー URL が App Store Connect に登録済み
- [ ] プライバシーポリシーに HealthKit データ利用の明示あり
- [ ] HealthKit データを広告に使用していない
- [ ] Xcode で com.apple.developer.healthkit エンタイトルメントが有効

## 推奨対応
- [ ] HealthKit 権限リクエストの前にアプリ内説明画面を表示する
- [ ] 権限が拒否された場合のフォールバック UI を用意する
- [ ] テスト時のモックデータを本番ビルドから除外している
```

## よくあるエラーと対処法

### エラー1: シミュレーターでデータが取得できない

**症状**: `getStepCount` が常に 0 を返す。

**原因**: iOS シミュレーターは HealthKit のリアルデータを持っていません。

**対処法**: シミュレーターではダミーデータを挿入するか、実機でテストします。

```typescript
// src/lib/mockHealthData.ts（開発環境専用）
export const insertMockHealthData = () => {
  if (__DEV__ && Platform.OS === 'ios') {
    // シミュレーター向けのモックデータ挿入
    AppleHealthKit.saveSteps(
      { value: 7523, startDate: new Date().toISOString(), endDate: new Date().toISOString() },
      (err) => { if (err) console.error(err); }
    );
  }
};
```

### エラー2: `pod install` 後にビルドが失敗する

**症状**: `Missing required Modules: react-native-health`

**対処法**:

```bash
# ios/Podfile に以下を確認
pod 'RNAppleHealthKit', :path => '../node_modules/react-native-health'

# キャッシュをクリアして再実行
cd ios && pod deintegrate && pod install
```

### エラー3: 権限を拒否した後に再リクエストできない

**症状**: 一度拒否すると、`initHealthKit` を呼んでも権限ダイアログが表示されない。

**原因**: iOS のシステム仕様。一度拒否した権限はアプリからは再リクエストできません。

**対処法**: ユーザーを「設定 → ヘルスケア → {アプリ名}」へ誘導します。

```typescript
import { Linking } from 'react-native';

const openHealthSettings = () => {
  Linking.openURL('App-Prefs:HEALTH');
};
```

### エラー4: バックグラウンド配信が動作しない

**症状**: アプリをバックグラウンドに送ると歩数が更新されない。

**確認事項**:
- Xcode で `Background Modes > Background fetch` が有効になっているか
- `com.apple.developer.healthkit.background-delivery` エンタイトルメントが有効か

## FAQ

**Q: HealthKit は Android でも使えますか？**

A. HealthKit は iOS/watchOS 専用です。Android では **Google Health Connect** （`react-native-health-connect` ライブラリ）を使用します。クロスプラットフォーム対応にする場合は、Platform.OS で分岐して両方実装する必要があります。対応コスト削減のため、まず iOS のみでリリースし、需要を確認してから Android 対応を検討するのが現実的な戦略です。

**Q: Apple Watch のデータも取得できますか？**

A. はい。Apple Watch が計測したデータ（詳細な睡眠ステージ・心拍数・VO2 Maxなど）は HealthKit を経由して iPhone でも取得できます。Watch アプリを別途開発しなくても、iPhone アプリから Watch のデータにアクセスできます。ただし、Watch 独自のデータ（Crown 操作など）には watchOS アプリが必要です。

**Q: HealthKit データをサーバーに保存してもいいですか？**

A. Apple のガイドラインでは、HealthKit データをユーザーの事前同意なしにサーバーへ送信することを禁止しています。ただし、ユーザーのバックアップや同期目的でサーバー保存すること自体は許可されています。重要なのは、データを広告・マーケティング・第三者へ販売することが明示的に禁止されている点です。プライバシーポリシーで利用目的を明確にし、暗号化転送（HTTPS）を徹底してください。

**Q: 審査でリジェクトされやすいポイントは？**

A. 最も多いリジェクト理由は「Usage Description が不明瞭」です。`NSHealthShareUsageDescription` に「ユーザーの健康データを使用します」のような漠然とした記述をすると却下されます。「歩数・睡眠・心拍データを読み取り、毎日の健康管理と目標達成の可視化に使用します」のように具体的な用途を記載してください。

**Q: HealthKit データのテストはどうやって行いますか？**

A. シミュレーターでは「ヘルスケア」アプリからダミーデータを手動入力できます（シミュレーター → Features → HealthKit Data から CSV インポートも可能）。CI/CD 環境では実機が使えないため、HealthKit 依存部分をモック（Jest + jest-mock-react-native-health）でテストするアーキテクチャにすると自動テストが容易になります。
