Rork Max では、**自然言語のプロンプトだけで、本格的な SwiftUI ネイティブアプリを自動生成できます**。React Native のクロスプラットフォーム開発とは異なり、iOS の最新機能を直接活用した高性能なアプリが実現可能です。

本ガイドでは、Rork Max の SwiftUI 生成機能を使って、ゼロからネイティブアプリを開発し、App Store に公開するまでの全工程を、実際の操作ステップに沿って解説します。

## Rork Max における SwiftUI ネイティブ生成の位置づけ

Rork の提供する開発方式は、大きく2つに分かれます：

**クロスプラットフォーム（React Native / Expo）**
- iOS ＋ Android 同時開発
- 迅速な開発サイクル
- 統一コードベース

**ネイティブ（SwiftUI）**
- iOS 専用・高性能
- 最新 iOS 機能への対応が速い
- バッテリー効率が良い

Rork Max では、SwiftUI ネイティブ生成がフル機能で利用できるため、**iOS 市場を狙ったアプリ、または高性能が必須なアプリ**を開発する際に選択肢となります。

### SwiftUI ネイティブ開発のメリット

1. **最新 iOS 機能への即対応**
   - Dynamic Island（ダイナミックアイランド）
   - App Intents（ショートカットアプリ連携）
   - WidgetKit（ホーム画面ウィジェット）
   - HealthKit、ARKit などの高度な API

2. **パフォーマンスと効率性**
   - React Native よりもバッテリー効率が 20 ～ 40% 向上
   - フレームレート 60fps ～ 120fps の安定
   - メモリ消費量が少ない

3. **App Store 審査での有利性**
   - ネイティブアプリは信頼性評価が高い傾向
   - デバイス連携機能（カメラ、マイク、位置情報）が確実に動作

4. **開発スピード（Rork Max 利用時）**
   - 通常の SwiftUI 開発：3 ～ 6 ヶ月
   - Rork Max 利用：1 ～ 2 週間

## ステップ 1: プロジェクト作成と環境設定

### Rork Max へのアップグレード

まず、Rork のプランを確認します。

1. **Rork ダッシュボード** → **Settings** → **Plan**
2. 現在のプラン（Free / Standard / Pro / Max）を確認
3. **Max プランへアップグレード**

Max プランのメリット：
- SwiftUI ネイティブ生成
- Android Native（Kotlin）生成
- API インテグレーション（REST / GraphQL / WebSocket）
- チーム機能（複数人開発対応）
- 優先サポート

### 新プロジェクトの初期化

1. **Rork ダッシュボード** → **New Project**
2. **Project Name**: `MyNativeApp`（任意）
3. **Template**: **Blank** または **Native iOS Starter**
4. **Platform**: **iOS Native（SwiftUI）**
5. **Create** をクリック

重要: このステップで「Platform: iOS Native」を選択することで、Rork Max の SwiftUI 生成エンジンが有効になります。

## ステップ 2: アプリのコンセプト設計と AI プロンプト作成

Rork Max の SwiftUI 生成の品質は、**プロンプトの質で決まります**。曖昧な指示では、期待から外れたコードが生成されます。

### 効果的なプロンプトの構成要素

プロンプトは、以下の要素を順序立てて記述します：

```
【アプリの目的】
[簡潔に、このアプリで何を実現するのか]

【メイン機能】
1. [機能1の説明 — 画面, 操作フロー]
2. [機能2の説明]
3. [機能3の説明]

【デザイン要件】
- [色スキーム、UIコンポーネントのスタイル]
- [アニメーション、インタラクション]

【技術仕様】
- [API 連携（あれば）]
- [データストレージ（UserDefaults, Core Data, Supabase等）]
- [認証方式（ローカル, OAuth, Apple Sign-In等）]

【出力期待】
- [生成するファイル構成]
- [推奨される依存ライブラリ]
```

### 具体例：シンプルな TODO リストアプリのプロンプト

```
【アプリの目的】
ユーザーが日々のタスクを管理できるシンプルな TODO リストアプリ

【メイン機能】
1. タスク一覧表示
   - ホーム画面にすべてのタスクを表示
   - タスク完了状態をチェックボックスで切り替え
   - 完了タスクは下部に移動

2. タスク追加
   - 「+」ボタンで新規タスク入力モーダルを表示
   - テキスト入力後「保存」で一覧に追加

3. タスク削除
   - タスクをスワイプで削除確認ポップアップ
   - 「削除」ボタンで一覧から除外

【デザイン要件】
- カラー: ライトモード（白背景、青・紫アクセント）
- フォント: San Francisco（iOS 標準フォント）
- アニメーション: タスク追加時は下からスライドイン

【技術仕様】
- データストレージ: Core Data（デバイスローカル保存）
- 認証: なし（ローカル単一ユーザー）
- 外部 API: なし

【出力期待】
- SwiftUI コンポーネント（View ファイル分割）
- ViewModel（MVVM パターン）
- Core Data Model
```

## ステップ 3: 自動生成コードの確認と調整

Rork Max が生成したコードは、自動的に以下の形式で出力されます：

```
📁 MyNativeApp/
  ├── MyNativeAppApp.swift         ← エントリーポイント
  ├── Views/
  │   ├── ContentView.swift        ← メイン画面
  │   ├── AddTaskView.swift        ← タスク追加モーダル
  │   └── TaskRowView.swift        ← タスク1件分のコンポーネント
  ├── Models/
  │   └── Task.swift               ← Core Data モデル定義
  ├── ViewModels/
  │   └── TaskViewModel.swift      ← ビジネスロジック・状態管理
  └── Assets.xcassets/             ← 色・画像リソース
```

### コード品質チェックリスト

生成されたコードを確認する際、以下の点をチェックします：

- ビルド可能性: Xcode でエラーなくビルドできるか
- 実行時動作: シミュレータで実際に動作するか
- メモリリーク: 循環参照（retain cycle）がないか
- State 管理: `@State`, `@ObservedObject`, `@EnvironmentObject` の使い分けが正しいか
- UI 応答性: UI 操作に対する遅延やフリーズがないか
- エラーハンドリング: ネットワークエラー、ファイルI/O エラーが適切に処理されているか

## ステップ 4: Xcode での開発・テスト環境構築

生成されたコードを自分の開発環境に取り込みます。

### コードのダウンロードと Xcode 導入

1. Rork Max ダッシュボードで **Export / Download Code**
2. ZIP ファイルを解凍
3. `MyNativeApp.xcodeproj` を Xcode で開く

### 初回ビルドと依存関係の解決

1. Xcode メニュー → **Product** → **Build** または **Cmd + B**
2. エラーが出た場合：
   - **Build Settings** → **Search Paths** を確認
   - CocoaPods / SPM（Swift Package Manager）の依存関係を確認

### iOS シミュレータでのテスト

1. Xcode 左上の **Scheme** から **iPhone 15 Pro** など選択
2. **Product** → **Run** または **Cmd + R**
3. シミュレータが起動し、アプリが実行される

テスト項目：
- アプリが起動する
- 画面がスムーズにレンダリングされる
- タップ・スワイプなどの操作に応答する
- データが正常に保存・読み込みされる

## ステップ 5: API 連携と高度な機能追加

基本的なアプリが完成したら、外部サービスとの連携を追加します。

### REST API 連携の例

Rork Max で生成されたコードに API 連携を追加するプロンプト：

```
現在のアプリにサーバー連携機能を追加してください。

【API 仕様】
- エンドポイント: https://api.example.com/tasks
- 認証: Bearer Token (Authorization: Bearer {token})
- メソッド: GET（タスク取得）/ POST（タスク作成）

【実装要件】
1. タスク一覧を起動時にサーバーから取得
2. 新規タスク作成時に POST で送信
3. ネットワークエラーは AlertView で表示
4. ローディング表示を実装（タスク取得中は ProgressView）
```

## ステップ 6: App Store 公開準備

### Apple Developer Account の登録

1. [developer.apple.com](https://developer.apple.com) にアクセス
2. Apple ID でサインアップ
3. **Individual** または **Organization** として登録

### アプリメタデータの設定

App Store Connect で：

1. **New App**
2. **App Name**: `My To-Do`（ユーザーに表示される名前）
3. **Primary Language**: Japanese
4. **Bundle ID**: `com.example.MyNativeApp`（ユニークID）
5. **SKU**: `20260412-mytodo-001`（内部管理用コード）

### スクリーンショット・プレビュー動画の準備

App Store では、以下のアセットが必須です：

- **スクリーンショット**: 5.5 インチ / 6.5 インチの iPhone サイズ、JPEG/PNG、5 ～ 10 枚
- **プレビュー動画**: MP4, H.264, 30fps, 15 ～ 30 秒
- **アプリアイコン**: 1024 × 1024px, PNG/JPG
- **App Store イメージ**: 1200 × 630px（オプション）

## ステップ 7: ビルド・署名・提出

### ビルド署名の設定

Xcode で：

1. **Project** → **Signing & Capabilities**
2. **Team** を選択（Apple Developer Account）
3. **Bundle Identifier** が正しいことを確認
4. **Signing Certificate** が自動生成される

### TestFlight でのベータテスト（推奨）

公開前に、社内チームや友人にテストしてもらいます：

1. Xcode で **Product** → **Archive**
2. **Organizer** で Archive を選択
3. **Distribute App** → **TestFlight**
4. テスター用メールアドレスを追加

### App Store への提出

1. Archive で **Distribute App** → **App Store**
2. **App Store Connect** で自動的にアップロード
3. **Submit for Review**

審査期間: 通常 24 ～ 48 時間で審査完了。承認されると自動的に App Store で公開されます。

## ステップ 8: デプロイ後の運用・監視

### TestFlight でのクラッシュレポート監視

App Store Connect の **Analytics** → **Crash Reports** で、ユーザー環境でのクラッシュを監視します。

監視項目：
- クラッシュ率（理想値: < 0.1%）
- 主なクラッシュ地点（スタックトレース）
- iOS バージョン別の影響

### ユーザーレビュー・フィードバック管理

App Store のレビューを定期的に確認：

1. App Store Connect → **Customer Reviews**
2. 低評価（★★以下）には、サポートメール返信
3. 改善要望を次のバージョンに反映

## よくある質問（FAQ）

**Q1: Rork Max で生成した SwiftUI コードは本番環境で使える？**

A: はい。Rork Max の出力コードは、そのまま App Store に提出・公開できます。ただし、以下の点は確認必要：
- セキュリティ: API キー・トークンが hardcode されていないか
- パフォーマンス: メモリリーク・無限ループがないか
- 依存関係: 外部ライブラリが Podfile / Package.swift に明記されているか

**Q2: Core Data を使う時と使わない時の違いは？**

A: 
- **Core Data 使用**: ローカルデータベース。複数ユーザー・大量データ向け。オフライン対応。
- **UserDefaults 使用**: シンプルな設定値保存。小容量（数 MB 程度）向け。

Rork Max が「会員管理・メッセージ保存」など複雑なデータ構造を検出すると、自動で Core Data を推奨します。

**Q3: Rork Max で生成したアプリが App Store 審査で落ちることはある？**

A: あります。主な原因：
- [ ] プライバシーポリシーが設定されていない
- [ ] バッテリー消費が多い機能が審査段階で検出
- [ ] 外部認証（Apple Sign-In など）の実装不完全
- [ ] 広告 SDK の不適切な使用

Rork Max 生成後、Apple の App Review Guidelines を必ず確認します。

**Q4: 複数のデバイス（iPhone / iPad）で同じ UI を使い分けたい。**

A: Size Classes と `@Environment(\.horizontalSizeClass)` で実装：

Rork Max で「iPad 対応」を明記すると、自動で Size Classes を実装します。

**Q5: Rork Max で生成したコード内に Apple 非推奨 API が使われていないか心配。**

A: Rork Max は最新の iOS SDK（iOS 15 以上）に基づいて生成されます。ただし、念のため以下の非推奨 API は避けるプロンプトを明記：

```
非推奨 API の使用を避けてください：
- UIView（SwiftUI を使用）
- URLSession.shared.dataTask (async/await を使用)
- DispatchQueue（Task / async/await を使用)
```

## まとめ

Rork Max を使えば、**複雑な SwiftUI ネイティブアプリを短期間で開発・公開できます**。プロンプト設計から Xcode ビルド、App Store 提出まで、本ガイドで紹介したステップに沿うことで、初心者でも本格的なアプリ開発が可能です。

次のステップは以下の通り：

1. **プロンプト設計を磨く** — Rork Max の出力品質は、入力プロンプトで決まります
2. **既存 iOS アプリの参考実装を読む** — GitHub で similar app（クローンアプリ）を検索し、ベストプラクティスを学ぶ
3. **App Store のレビューガイドを読む** — Apple の審査基準を事前に理解しておく

プレミアム記事で、さらに深い実装技法（WidgetKit、ARKit、Core ML、CloudKit 同期など）も多数カバーしています。ぜひ参考にしてください。
