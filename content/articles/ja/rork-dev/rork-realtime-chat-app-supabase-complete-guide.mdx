## はじめに — なぜRork × Supabaseでチャットアプリなのか

チャットアプリは、モバイル開発で最も学びの多いプロジェクトの一つです。認証、リアルタイムデータ同期、プッシュ通知、画像アップロード、オフライン対応など、実務で求められる技術要素がほぼすべて詰まっています。

**Rork Max**はAIの支援を受けながらReact Nativeアプリを構築できるツールですが、バックエンド連携が本格化すると、AI任せでは対応しきれない設計判断が必要になります。本記事では、**Supabase**をバックエンドに採用し、認証からリアルタイムメッセージング、プッシュ通知まで、プロダクション品質のチャットアプリを段階的に構築します。

[Expo Routerのナビゲーション設計](/articles/rork-dev/expo-router-navigation-guide)を理解している前提で進めますので、初めての方はそちらを先にご覧ください。

## データベース設計 — Supabaseテーブル構成

チャットアプリのデータモデルを設計します。Supabaseの**Row Level Security (RLS)**を活用して、データベースレベルでアクセス制御を実装します。

### テーブル構成

```sql
-- ユーザープロフィール（auth.usersと連動）
create table profiles (
  id uuid references auth.users on delete cascade primary key,
  username text unique not null,
  display_name text not null,
  avatar_url text,
  push_token text, -- Expo Push Token
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- チャットルーム
create table rooms (
  id uuid default gen_random_uuid() primary key,
  name text,
  is_group boolean default false,
  created_at timestamptz default now()
);

-- ルーム参加者
create table room_members (
  room_id uuid references rooms on delete cascade,
  user_id uuid references profiles on delete cascade,
  joined_at timestamptz default now(),
  last_read_at timestamptz default now(),
  primary key (room_id, user_id)
);

-- メッセージ
create table messages (
  id uuid default gen_random_uuid() primary key,
  room_id uuid references rooms on delete cascade not null,
  sender_id uuid references profiles not null,
  content text not null,
  message_type text default 'text' check (message_type in ('text', 'image', 'system')),
  created_at timestamptz default now()
);

-- インデックス（クエリ高速化）
create index idx_messages_room_created on messages (room_id, created_at desc);
create index idx_room_members_user on room_members (user_id);
```

### Row Level Security ポリシー

```sql
-- RLSを有効化
alter table profiles enable row level security;
alter table rooms enable row level security;
alter table room_members enable row level security;
alter table messages enable row level security;

-- プロフィール: 認証ユーザーは全員閲覧可、自分のみ更新可
create policy "Profiles viewable by authenticated users"
  on profiles for select to authenticated using (true);

create policy "Users can update own profile"
  on profiles for update to authenticated using (auth.uid() = id);

-- メッセージ: ルームメンバーのみ閲覧・投稿可
create policy "Room members can view messages"
  on messages for select to authenticated using (
    exists (
      select 1 from room_members
      where room_members.room_id = messages.room_id
        and room_members.user_id = auth.uid()
    )
  );

create policy "Room members can insert messages"
  on messages for insert to authenticated with check (
    sender_id = auth.uid() and
    exists (
      select 1 from room_members
      where room_members.room_id = messages.room_id
        and room_members.user_id = auth.uid()
    )
  );
```

## 認証システムの実装

Supabase Authを使った認証フローをRorkアプリに統合します。

### 認証コンテキストの設計

```typescript
// src/contexts/AuthContext.tsx
import { createContext, useContext, useEffect, useState } from "react";
import { supabase } from "../lib/supabase";
import { Session, User } from "@supabase/supabase-js";

interface AuthState {
  user: User | null;
  session: Session | null;
  loading: boolean;
  signIn: (email: string, password: string) => Promise<void>;
  signUp: (email: string, password: string, username: string) => Promise<void>;
  signOut: () => Promise<void>;
}

const AuthContext = createContext<AuthState | null>(null);

export function AuthProvider({ children }: { children: React.ReactNode }) {
  const [user, setUser] = useState<User | null>(null);
  const [session, setSession] = useState<Session | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // 初期セッション取得
    supabase.auth.getSession().then(({ data: { session } }) => {
      setSession(session);
      setUser(session?.user ?? null);
      setLoading(false);
    });

    // セッション変更をリッスン
    const { data: { subscription } } = supabase.auth.onAuthStateChange(
      (_event, session) => {
        setSession(session);
        setUser(session?.user ?? null);
      }
    );

    return () => subscription.unsubscribe();
  }, []);

  const signUp = async (email: string, password: string, username: string) => {
    const { data, error } = await supabase.auth.signUp({
      email,
      password,
    });
    if (error) throw error;

    // プロフィール作成
    if (data.user) {
      await supabase.from("profiles").insert({
        id: data.user.id,
        username,
        display_name: username,
      });
    }
  };

  const signIn = async (email: string, password: string) => {
    const { error } = await supabase.auth.signInWithPassword({
      email,
      password,
    });
    if (error) throw error;
  };

  const signOut = async () => {
    await supabase.auth.signOut();
  };

  return (
    <AuthContext.Provider value={{ user, session, loading, signIn, signUp, signOut }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => {
  const context = useContext(AuthContext);
  if (!context) throw new Error("useAuth must be used within AuthProvider");
  return context;
};
```

## リアルタイムメッセージングの実装

Supabase Realtimeを使って、メッセージの送受信をリアルタイムに処理します。

### メッセージフックの実装

```typescript
// src/hooks/useMessages.ts
import { useEffect, useState, useCallback, useRef } from "react";
import { supabase } from "../lib/supabase";
import { useAuth } from "../contexts/AuthContext";

interface Message {
  id: string;
  room_id: string;
  sender_id: string;
  content: string;
  message_type: string;
  created_at: string;
  sender?: { username: string; avatar_url: string | null };
}

export function useMessages(roomId: string) {
  const { user } = useAuth();
  const [messages, setMessages] = useState<Message[]>([]);
  const [loading, setLoading] = useState(true);
  const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null);

  // 初期メッセージの取得
  useEffect(() => {
    const fetchMessages = async () => {
      const { data, error } = await supabase
        .from("messages")
        .select(`
          *,
          sender:profiles!sender_id (username, avatar_url)
        `)
        .eq("room_id", roomId)
        .order("created_at", { ascending: true })
        .limit(50);

      if (!error && data) {
        setMessages(data);
      }
      setLoading(false);
    };

    fetchMessages();
  }, [roomId]);

  // Realtimeサブスクリプション
  useEffect(() => {
    const channel = supabase
      .channel(`room:${roomId}`)
      .on(
        "postgres_changes",
        {
          event: "INSERT",
          schema: "public",
          table: "messages",
          filter: `room_id=eq.${roomId}`,
        },
        async (payload) => {
          // 送信者情報を取得して追加
          const { data: sender } = await supabase
            .from("profiles")
            .select("username, avatar_url")
            .eq("id", payload.new.sender_id)
            .single();

          const newMessage: Message = {
            ...payload.new as Message,
            sender: sender || undefined,
          };

          setMessages((prev) => [...prev, newMessage]);
        }
      )
      .subscribe();

    channelRef.current = channel;

    return () => {
      supabase.removeChannel(channel);
    };
  }, [roomId]);

  // メッセージ送信
  const sendMessage = useCallback(async (content: string) => {
    if (!user) return;

    const { error } = await supabase.from("messages").insert({
      room_id: roomId,
      sender_id: user.id,
      content,
      message_type: "text",
    });

    if (error) {
      console.error("Failed to send message:", error);
      throw error;
    }
  }, [roomId, user]);

  return { messages, loading, sendMessage };
}
```

## プッシュ通知の実装

Expo Notificationsを使って、新着メッセージのプッシュ通知を実装します。

### プッシュトークンの登録

```typescript
// src/lib/notifications.ts
import * as Notifications from "expo-notifications";
import * as Device from "expo-device";
import { Platform } from "react-native";
import { supabase } from "./supabase";

// 通知の表示設定
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});

export async function registerPushToken(userId: string) {
  if (!Device.isDevice) {
    console.log("Push notifications require a physical device");
    return;
  }

  // 権限リクエスト
  const { status: existingStatus } = await Notifications.getPermissionsAsync();
  let finalStatus = existingStatus;

  if (existingStatus !== "granted") {
    const { status } = await Notifications.requestPermissionsAsync();
    finalStatus = status;
  }

  if (finalStatus !== "granted") {
    console.log("Push notification permission denied");
    return;
  }

  // Expo Push Token取得
  const tokenData = await Notifications.getExpoPushTokenAsync({
    projectId: "YOUR_EXPO_PROJECT_ID", // expo.dev で取得
  });

  // Supabaseにトークン保存
  await supabase
    .from("profiles")
    .update({ push_token: tokenData.data })
    .eq("id", userId);

  // Android用の通知チャンネル設定
  if (Platform.OS === "android") {
    await Notifications.setNotificationChannelAsync("messages", {
      name: "Messages",
      importance: Notifications.AndroidImportance.HIGH,
      sound: "default",
    });
  }

  return tokenData.data;
}
```

### サーバーサイドのプッシュ通知送信

Supabaseの**Edge Functions**を使って、新着メッセージ時にプッシュ通知を送信します。

```typescript
// supabase/functions/send-push-notification/index.ts
import { serve } from "https://deno.land/std@0.177.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

serve(async (req) => {
  const { record } = await req.json();
  const { room_id, sender_id, content } = record;

  const supabase = createClient(
    Deno.env.get("SUPABASE_URL")!,
    Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
  );

  // 送信者情報を取得
  const { data: sender } = await supabase
    .from("profiles")
    .select("display_name")
    .eq("id", sender_id)
    .single();

  // ルームメンバーのPush Tokenを取得（送信者以外）
  const { data: members } = await supabase
    .from("room_members")
    .select("user_id, profiles!inner(push_token)")
    .eq("room_id", room_id)
    .neq("user_id", sender_id);

  const tokens = members
    ?.map((m: any) => m.profiles?.push_token)
    .filter(Boolean) || [];

  if (tokens.length === 0) {
    return new Response(JSON.stringify({ sent: 0 }));
  }

  // Expo Push API でバッチ送信
  const notifications = tokens.map((token: string) => ({
    to: token,
    title: sender?.display_name || "New Message",
    body: content.length > 100 ? content.slice(0, 97) + "..." : content,
    data: { room_id },
    sound: "default",
  }));

  const response = await fetch("https://exp.host/--/api/v2/push/send", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(notifications),
  });

  const result = await response.json();
  return new Response(JSON.stringify({ sent: tokens.length, result }));
});
```

## チャット画面のUI実装

メッセージリストとメッセージ入力をFlatListで効率的にレンダリングします。

```typescript
// src/screens/ChatScreen.tsx
import { useCallback, useRef, useState } from "react";
import {
  FlatList, TextInput, TouchableOpacity,
  View, Text, KeyboardAvoidingView, Platform,
} from "react-native";
import { useMessages } from "../hooks/useMessages";
import { useAuth } from "../contexts/AuthContext";

export default function ChatScreen({ route }: { route: any }) {
  const { roomId } = route.params;
  const { user } = useAuth();
  const { messages, loading, sendMessage } = useMessages(roomId);
  const [input, setInput] = useState("");
  const flatListRef = useRef<FlatList>(null);

  const handleSend = useCallback(async () => {
    if (!input.trim()) return;
    const text = input.trim();
    setInput("");
    try {
      await sendMessage(text);
    } catch (error) {
      // エラー時は入力を復元
      setInput(text);
    }
  }, [input, sendMessage]);

  const renderMessage = useCallback(({ item }: { item: any }) => {
    const isOwn = item.sender_id === user?.id;

    return (
      <View style={{
        alignSelf: isOwn ? "flex-end" : "flex-start",
        backgroundColor: isOwn ? "#007AFF" : "#E5E5EA",
        borderRadius: 16,
        paddingHorizontal: 14,
        paddingVertical: 8,
        marginVertical: 2,
        marginHorizontal: 12,
        maxWidth: "75%",
      }}>
        {!isOwn && (
          <Text style={{ fontSize: 12, color: "#666", marginBottom: 2 }}>
            {item.sender?.username}
          </Text>
        )}
        <Text style={{ color: isOwn ? "#FFF" : "#000", fontSize: 16 }}>
          {item.content}
        </Text>
      </View>
    );
  }, [user]);

  return (
    <KeyboardAvoidingView
      style={{ flex: 1 }}
      behavior={Platform.OS === "ios" ? "padding" : undefined}
      keyboardVerticalOffset={90}
    >
      <FlatList
        ref={flatListRef}
        data={messages}
        renderItem={renderMessage}
        keyExtractor={(item) => item.id}
        onContentSizeChange={() => flatListRef.current?.scrollToEnd()}
        contentContainerStyle={{ paddingVertical: 8 }}
      />
      <View style={{
        flexDirection: "row",
        padding: 8,
        borderTopWidth: 1,
        borderTopColor: "#E5E5EA",
        backgroundColor: "#FFF",
      }}>
        <TextInput
          value={input}
          onChangeText={setInput}
          placeholder="メッセージを入力..."
          style={{
            flex: 1,
            borderWidth: 1,
            borderColor: "#DDD",
            borderRadius: 20,
            paddingHorizontal: 16,
            paddingVertical: 8,
            fontSize: 16,
          }}
          multiline
          maxLength={1000}
        />
        <TouchableOpacity
          onPress={handleSend}
          disabled={!input.trim()}
          style={{
            marginLeft: 8,
            backgroundColor: input.trim() ? "#007AFF" : "#CCC",
            borderRadius: 20,
            width: 40,
            height: 40,
            alignItems: "center",
            justifyContent: "center",
          }}
        >
          <Text style={{ color: "#FFF", fontSize: 18 }}>↑</Text>
        </TouchableOpacity>
      </View>
    </KeyboardAvoidingView>
  );
}
```

## パフォーマンス最適化とオフライン対応

### メッセージのページネーション

メッセージ数が増えると初期ロードが遅くなります。カーソルベースのページネーションで過去メッセージを段階的に読み込みます。

```typescript
// useMessages.tsに追加
const loadMoreMessages = useCallback(async () => {
  if (messages.length === 0) return;

  const oldestMessage = messages[0];
  const { data } = await supabase
    .from("messages")
    .select("*, sender:profiles!sender_id (username, avatar_url)")
    .eq("room_id", roomId)
    .lt("created_at", oldestMessage.created_at)
    .order("created_at", { ascending: false })
    .limit(20);

  if (data && data.length > 0) {
    setMessages((prev) => [...data.reverse(), ...prev]);
  }
}, [messages, roomId]);
```

## よくある質問（FAQ）

### Q1: Supabase RealtimeとFirebase Realtime Databaseの違いは？

Supabase RealtimeはPostgreSQLのWAL（Write-Ahead Logging）を基盤としており、既存のRLSポリシーがそのまま適用されるため、セキュリティ設定を二重管理する必要がありません。Firebase Realtime Databaseは独自のセキュリティルール言語を使用するため、SQLとの設計思想の違いに注意が必要です。Rorkアプリではどちらでも動作しますが、SQLに慣れている場合はSupabaseの方が学習コストが低いでしょう。

### Q2: メッセージの既読管理はどう実装しますか？

`room_members`テーブルの`last_read_at`フィールドを活用します。メッセージを表示したタイミングで`last_read_at`を更新し、各メッセージの`created_at`と比較して未読数を算出します。パフォーマンスのため、未読数の計算はサーバーサイド（Supabase Edge Function または RPC関数）で行うことをおすすめします。

### Q3: 画像メッセージを送る場合、画像はどこに保存しますか？

Supabase Storageを使います。`expo-image-picker`で選択した画像をSupabase Storageにアップロードし、公開URLを`messages`テーブルの`content`に保存します（`message_type: "image"`）。RLSポリシーで、ルームメンバーのみがアップロード・閲覧できるよう制御してください。

### Q4: 1000人規模のグループチャットに対応できますか？

Supabase Realtimeのサブスクリプション数やメッセージスループットに制限があるため、大規模グループの場合はポーリング + Realtime のハイブリッド方式を検討してください。メッセージ一覧はページネーションで取得し、新着通知のみRealtimeで受信する設計が効率的です。
