## はじめに — なぜ多言語対応が重要なのか

アプリを世界に届けたいと考えたとき、最初に直面するのが **多言語対応（i18n: internationalization）** の壁です。App Store と Google Play のユーザーの約70%は英語圏以外に住んでおり、母国語で使えるアプリは自然とダウンロード率が向上します。

Rork を使えば、AI がコードの骨格を自動生成してくれるため、多言語対応に必要な設定作業も効率的に進められます。この記事では、Rork × React Native 環境で `react-i18next` を導入し、動的な言語切替を実装する方法をステップバイステップで解説します。

対象読者は、Rork でアプリを1本以上作った経験がある中級者です。基本的な React Native のコンポーネント構成を理解していることを前提としています。

## 前提知識と環境準備

多言語対応を始める前に、以下の環境が整っていることを確認してください。

- **Rork アカウント**（Free プランでも可）
- **Node.js 18 以上**（ローカルでの動作確認用）
- **Expo SDK 52 以上**（Rork が生成するデフォルト環境）

Rork のプロンプトで「多言語対応のアプリを作りたい」と伝えると、AI が基本的な i18n 構成を提案してくれます。ただし、細かい設定やベストプラクティスを理解しておくと、より堅牢な実装が可能です。

## react-i18next の導入と初期設定

React Native の多言語対応ライブラリとして最も広く使われているのが `react-i18next` です。Expo 環境との相性もよく、Rork が生成するプロジェクト構成にスムーズに統合できます。

### パッケージのインストール

```bash
# Rork プロジェクトのルートディレクトリで実行
npx expo install i18next react-i18next expo-localization
```

`expo-localization` はデバイスのシステム言語を取得するために使います。ユーザーが初めてアプリを開いたとき、自動的に適切な言語で表示するための重要なパッケージです。

### 翻訳ファイルの構成

プロジェクト内に翻訳リソースを配置します。以下の構成が一般的です。

```
src/
  locales/
    ja.json    // 日本語
    en.json    // 英語
    zh.json    // 中国語（必要に応じて追加）
  i18n.ts      // 初期化ファイル
```

日本語の翻訳ファイル（`ja.json`）の例を見てみましょう。

```json
{
  "common": {
    "appName": "マイアプリ",
    "loading": "読み込み中...",
    "error": "エラーが発生しました",
    "retry": "再試行"
  },
  "home": {
    "title": "ホーム",
    "welcome": "{{name}}さん、こんにちは！",
    "description": "今日も素敵な一日をお過ごしください"
  },
  "settings": {
    "title": "設定",
    "language": "言語",
    "theme": "テーマ",
    "notifications": "通知"
  }
}
```

英語版（`en.json`）は同じキー構造で記述します。

```json
{
  "common": {
    "appName": "My App",
    "loading": "Loading...",
    "error": "An error occurred",
    "retry": "Retry"
  },
  "home": {
    "title": "Home",
    "welcome": "Hello, {{name}}!",
    "description": "Have a wonderful day"
  },
  "settings": {
    "title": "Settings",
    "language": "Language",
    "theme": "Theme",
    "notifications": "Notifications"
  }
}
```

`{{name}}` のように二重波括弧で囲んだ部分は **補間（interpolation）** と呼ばれ、動的な値を埋め込むために使います。

### i18n 初期化ファイル

```typescript
// src/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as Localization from 'expo-localization';

import ja from './locales/ja.json';
import en from './locales/en.json';

// デバイスの言語設定を取得
const deviceLanguage = Localization.getLocales()[0]?.languageCode ?? 'en';

i18n.use(initReactI18next).init({
  resources: {
    ja: { translation: ja },
    en: { translation: en },
  },
  lng: deviceLanguage,       // デバイス言語をデフォルトに
  fallbackLng: 'en',         // 対応言語がない場合は英語にフォールバック
  interpolation: {
    escapeValue: false,       // React は自動的に XSS を防ぐため不要
  },
  compatibilityJSON: 'v4',   // React Native 環境での互換性設定
});

export default i18n;
// 期待される動作: デバイスが日本語設定なら日本語、それ以外は英語で表示
```

このファイルをアプリのエントリポイント（`App.tsx` や `_layout.tsx`）の先頭で `import './i18n'` として読み込めば、アプリ全体で翻訳機能が使えるようになります。

## コンポーネントでの翻訳の使い方

i18n の初期設定が完了したら、各コンポーネントで翻訳テキストを使ってみましょう。

### useTranslation フック

```tsx
// src/screens/HomeScreen.tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';

export default function HomeScreen() {
  const { t } = useTranslation();

  return (
    <View style={styles.container}>
      <Text style={styles.title}>{t('home.title')}</Text>
      {/* 補間を使った動的テキスト */}
      <Text style={styles.welcome}>
        {t('home.welcome', { name: 'ユーザー' })}
      </Text>
      <Text style={styles.description}>{t('home.description')}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 12 },
  welcome: { fontSize: 18, marginBottom: 8 },
  description: { fontSize: 14, color: '#666' },
});
// 期待される出力（日本語デバイス）:
// ホーム
// ユーザーさん、こんにちは！
// 今日も素敵な一日をお過ごしください
```

`t()` 関数にドット区切りのキーを渡すだけで、現在の言語に対応する翻訳テキストが返されます。コードの可読性を保ちながら、言語ごとの表示を切り替えられるのが `react-i18next` の大きな利点です。

## 言語切替機能の実装

ユーザーがアプリ内で言語を切り替えられるようにしましょう。`AsyncStorage` を使って選択を永続化します。

```tsx
// src/components/LanguageSwitcher.tsx
import React from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';
import AsyncStorage from '@react-native-async-storage/async-storage';

const LANGUAGES = [
  { code: 'ja', label: '日本語' },
  { code: 'en', label: 'English' },
];

export default function LanguageSwitcher() {
  const { i18n, t } = useTranslation();

  const changeLanguage = async (langCode: string) => {
    await i18n.changeLanguage(langCode);
    // 選択した言語を端末に保存
    await AsyncStorage.setItem('user-language', langCode);
  };

  return (
    <View style={styles.container}>
      <Text style={styles.label}>{t('settings.language')}</Text>
      <View style={styles.buttonRow}>
        {LANGUAGES.map((lang) => (
          <TouchableOpacity
            key={lang.code}
            style={[
              styles.button,
              i18n.language === lang.code && styles.activeButton,
            ]}
            onPress={() => changeLanguage(lang.code)}
          >
            <Text
              style={[
                styles.buttonText,
                i18n.language === lang.code && styles.activeText,
              ]}
            >
              {lang.label}
            </Text>
          </TouchableOpacity>
        ))}
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16 },
  label: { fontSize: 16, fontWeight: '600', marginBottom: 8 },
  buttonRow: { flexDirection: 'row', gap: 8 },
  button: {
    paddingHorizontal: 16,
    paddingVertical: 8,
    borderRadius: 8,
    backgroundColor: '#f0f0f0',
  },
  activeButton: { backgroundColor: '#007AFF' },
  buttonText: { fontSize: 14, color: '#333' },
  activeText: { color: '#fff' },
});
// 期待される動作: タップした言語が即座に反映され、再起動後も保持される
```

初期化時に保存済みの言語を復元するには、`i18n.ts` を以下のように拡張します。

```typescript
// i18n.ts に追加
import AsyncStorage from '@react-native-async-storage/async-storage';

async function initLanguage() {
  const savedLang = await AsyncStorage.getItem('user-language');
  if (savedLang) {
    i18n.changeLanguage(savedLang);
  }
}

initLanguage();
```

## 複数形・日付・通貨のローカライズ

翻訳テキストの切り替えだけでなく、数値や日付のフォーマットも言語に合わせる必要があります。

### 複数形の処理

英語には単数形・複数形の区別がありますが、日本語にはありません。`react-i18next` は言語ごとに異なる複数形ルールを自動的に適用します。

```json
// en.json
{
  "items": {
    "count_one": "{{count}} item",
    "count_other": "{{count}} items"
  }
}

// ja.json
{
  "items": {
    "count": "{{count}} 件"
  }
}
```

```tsx
// コンポーネントでの使用
<Text>{t('items.count', { count: 5 })}</Text>
// 英語: "5 items" / 日本語: "5 件"
```

### 日付と通貨

日付や通貨のフォーマットには、JavaScript 標準の `Intl` API を使うのがシンプルです。

```typescript
// src/utils/formatters.ts
export function formatDate(date: Date, locale: string): string {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);
}

export function formatCurrency(
  amount: number,
  locale: string,
  currency: string
): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(amount);
}

// 使用例
// formatDate(new Date(), 'ja') → "2026年3月28日"
// formatDate(new Date(), 'en') → "March 28, 2026"
// formatCurrency(1500, 'ja', 'JPY') → "￥1,500"
// formatCurrency(15, 'en', 'USD') → "$15.00"
```

## RTL（右から左）言語への対応

アラビア語やヘブライ語など、右から左（RTL）に記述する言語に対応する場合は、レイアウトの反転が必要です。React Native は `I18nManager` を通じて RTL レイアウトをサポートしています。

```typescript
// src/utils/rtl.ts
import { I18nManager } from 'react-native';
import * as Updates from 'expo-updates';

export async function setRTL(isRTL: boolean) {
  if (I18nManager.isRTL !== isRTL) {
    I18nManager.forceRTL(isRTL);
    I18nManager.allowRTL(isRTL);
    // RTL の切り替えにはアプリの再起動が必要
    await Updates.reloadAsync();
  }
}
```

RTL 対応が必要な言語を追加する場合は、`StyleSheet` で `flexDirection` や `textAlign` にハードコードした値を避け、`start` / `end` などの論理的な方向指定を使うと、RTL 環境でも自動的にレイアウトが反転します。

## App Store / Google Play のローカライゼーション

コード上の多言語対応が完了したら、ストアリスティングもローカライズしましょう。ストアのメタデータを現地語で用意することで、各地域での検索順位が大きく向上します。

- **アプリ名**: 各言語で最適化されたタイトルを設定（キーワードを含める）
- **説明文**: 翻訳ではなく、ターゲット市場のユーザーに響くコピーを作成
- **スクリーンショット**: 各言語のUI状態でキャプチャし、現地語のキャプションを追加
- **キーワードフィールド**（iOS）: 言語ごとに検索されやすいキーワードを設定

Rork Max を使っている場合は、2クリックで App Store に公開する際に、複数のローカライゼーションをまとめて設定できます。

さらに詳しい多地域 ASO 戦略については、[Rork アプリ グローバル展開 × 多地域ASO最適化 完全戦略ガイド](/articles/rork-business/rork-global-expansion-multi-region-aso-strategy-guide) で体系的に解説しています。

## Rork の AI プロンプトで i18n を効率化するコツ

Rork で多言語対応アプリを作るとき、プロンプトの書き方ひとつで生成されるコードの品質が変わります。以下のポイントを意識してみてください。

- **最初から多言語を指定する**: 「日本語と英語に対応したタスク管理アプリを作って」のように、言語を明示する
- **翻訳キーの命名規則を伝える**: 「翻訳キーは `画面名.要素名` の形式にしてください」と指定する
- **フォールバック言語を伝える**: 「英語をフォールバック言語にして、対応外の言語は英語表示にしてください」と指定する

こうした具体的な指示を与えると、Rork の AI が一貫性のある i18n 構成を生成してくれます。Rork のプロンプトエンジニアリングについて詳しく知りたい方は、[Rork プロンプトエンジニアリング — AI にアプリを上手に作らせるコツ](/articles/rork-basics/rork-prompt-engineering-app-design-tips) も参考になります。

## FAQ

### Q: 対応言語を後から追加するにはどうすればいいですか？

A: 新しい翻訳ファイル（例: `zh.json`）を `locales/` ディレクトリに追加し、`i18n.ts` の `resources` オブジェクトに登録するだけです。既存のキー構造をコピーして翻訳を埋めていけば、アプリ側のコンポーネントは一切変更なしで新言語に対応できます。

### Q: 翻訳テキストが長くなってレイアウトが崩れる場合はどうすればいいですか？

A: ドイツ語やフィンランド語など、英語より大幅に長くなる言語があります。`numberOfLines` と `adjustsFontSizeToFit` プロパティを活用するか、テキストコンテナに `flexShrink: 1` を設定してレイアウトの柔軟性を確保してください。デザイン段階で「テキストが1.5倍の長さになっても崩れない」ことをテスト基準にするのがおすすめです。

### Q: 翻訳の管理が大変になったらどうすればいいですか？

A: 翻訳ファイルが大きくなってきたら、ネームスペース機能を使ってファイルを分割できます。`i18next` の `ns`（namespace）オプションを使うと、`common.json`、`home.json`、`settings.json` のように画面ごとにファイルを分けて管理できます。また、Crowdin や Lokalise といった翻訳管理プラットフォームと連携すれば、翻訳者との協働もスムーズになります。
