## Figma Make × Rork — デザインから App Store まで 24 時間

2026年、モバイルアプリ開発のタイムラインは劇的に短縮されました。Figma Make がテキストから React プロトタイプを生成し、Rork がそれをネイティブ iOS/Android アプリに自動変換する統合ワークフローにより、24 時間以内にアプリストア申請まで到達することが可能になったのです。本記事では、実装から App Store 申請まで、完全なワークフローを段階的に解説します。

## ステップ 1: Figma Make でプロトタイプ生成

### Figma Make の基本操作

Figma Make は AI ネイティブなプロトタイピングツールで、テキスト説明から React コンポーネントを自動生成します。Supabase バックエンドも同時に生成可能です。

```bash
# Figma にログイン
# Figma ダッシュボードから "Make" を選択

# または CLI から
figma-make create --name="Weather App" --template=mobile

# 初期設定
figma-make config set \
  --backend=supabase \
  --framework=react-native \
  --typescript=true
```

### テキストプロンプトからのプロトタイプ生成

```

「天気アプリを作ってください。
上部に大きく現在の気温と天気アイコンを表示。
中央にこれからの3時間ごとの天気予報を水平スクロール可能なカード表示。
下部に日中の気温グラフ。
背景は現在の天気に応じて色が変わる（晴れ=青、雨=灰色、雪=白）。
ユーザーが都市名を検索できる検索バーを最上部に配置。」
```

- UI コンポーネント構造

### 動的コンテンツの追加

Figma Make が自動生成したコンポーネントに、リアルな気象 API を接続：

```typescript
// src/hooks/useWeather.ts - 自動生成（Figma Make）
// ↓ 手動調整

import { useEffect, useState } from 'react';
import { supabase } from '@/lib/supabase';

interface Weather {
  city: string;
  temperature: number;
  condition: string;
  icon: string;
  humidity: number;
  windSpeed: number;
}

export const useWeather = (city: string) => {
  const [weather, setWeather] = useState<Weather | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchWeather = async () => {
      try {
        setLoading(true);

        // Supabase Functions で天気 API を呼び出し
        const { data, error: supabaseError } = await supabase.functions.invoke(
          'fetch-weather',
          {
            body: { city }
          }
        );

        if (supabaseError) throw supabaseError;

        setWeather(data);
        setError(null);
      } catch (err) {
        setError(err instanceof Error ? err.message : 'Unknown error');
        setWeather(null);
      } finally {
        setLoading(false);
      }
    };

    if (city) {
      fetchWeather();
    }
  }, [city]);

  return { weather, loading, error };
};
```

## ステップ 3: Rork へのアプリ変換

### Rork への入力準備

Rork は React Web コードをネイティブモバイルアプリに自動変換します。Figma Make が生成した React コードは、ほぼそのまま Rork に入力できる最適な形式です。

```bash
# Rork CLI のインストール
npm install -g rork-cli

# Rork 設定ファイルを作成
rork init \
  --name="WeatherApp" \
  --source=./src \
  --template=react-native

# または interactive mode
rork init --interactive
```

### Rork 設定ファイル（rork.config.ts）

```typescript
import { RorkConfig } from 'rork/types';

export const config: RorkConfig = {
  projectName: 'WeatherApp',
  description: '天気予報アプリ',
  version: '1.0.0',

  // ターゲットプラットフォーム
  platforms: {
    ios: {
      minVersion: '14.0',
      bundleId: 'com.weatherapp.ios'
    },
    android: {
      minVersion: 23,
      packageName: 'com.weatherapp.android'
    }
  },

  // ソースコード設定
  source: {
    entry: './src/App.tsx',
    include: ['./src/**/*.{ts,tsx}'],
    exclude: ['./src/**/*.test.ts', './src/**/*.stories.tsx']
  },

  // 自動コンバージョン設定
  conversion: {
    // React Web → React Native 変換ルール
    mappings: [
      { from: 'div', to: 'View' },
      { from: 'span', to: 'Text' },
      { from: 'button', to: 'Pressable' },
      { from: 'input', to: 'TextInput' },
      { from: 'img', to: 'Image' },
      { from: 'scroll', to: 'ScrollView' }
    ],
    // CSS → StyleSheet 変換
    styleConversion: 'automatic',
    // 型安全性の維持
    typescript: true
  },

  // Supabase 連携
  backend: {
    type: 'supabase',
    url: process.env.NEXT_PUBLIC_SUPABASE_URL,
    key: process.env.NEXT_PUBLIC_SUPABASE_KEY
  },

  // ビルド出力
  output: {
    ios: './build/ios',
    android: './build/android'
  }
};
```

## ステップ 4: 手動調整と最適化

### ネイティブモジュール統合

Rork が自動変換できない、プラットフォーム固有の機能を追加：

```swift
// ios/WeatherApp/Modules/LocationManager.swift

import CoreLocation
import Rork

@objc class LocationManager: NSObject, CLLocationManagerDelegate {
  private let locationManager = CLLocationManager()

  @objc func requestLocation() -> Promise<[String: Any]> {
    return Promise { resolve, reject in
      locationManager.delegate = self

      if CLLocationManager.locationServicesEnabled() {
        locationManager.requestWhenInUseAuthorization()
        locationManager.requestLocation()
      } else {
        reject(NSError(
          domain: "LocationError",
          code: 1,
          userInfo: [NSLocalizedDescriptionKey: "位置情報サービスが無効です"]
        ))
      }
    }
  }

  func locationManager(
    _ manager: CLLocationManager,
    didUpdateLocations locations: [CLLocation]
  ) {
    guard let location = locations.last else { return }

    let result: [String: Any] = [
      "latitude": location.coordinate.latitude,
      "longitude": location.coordinate.longitude,
      "altitude": location.altitude
    ]

    // React Native 側に結果を返す
  }
}
```

```kotlin
// android/app/src/main/java/com/weatherapp/modules/LocationManager.kt

package com.weatherapp.modules

import android.Manifest
import android.content.pm.PackageManager
import android.location.Location
import androidx.core.app.ActivityCompat
import com.google.android.gms.location.LocationServices
import com.facebook.react.bridge.*

class LocationManager(reactContext: ReactApplicationContext) :
  ReactContextBaseJavaModule(reactContext) {

  private val fusedLocationClient = LocationServices.getFusedLocationProviderClient(reactContext)

  @ReactMethod
  fun requestLocation(promise: Promise) {
    try {
      if (ActivityCompat.checkSelfPermission(
          reactContext,
          Manifest.permission.ACCESS_FINE_LOCATION
        ) != PackageManager.PERMISSION_GRANTED
      ) {
        promise.reject("PERMISSION_ERROR", "位置情報パーミッションが必要です")
        return
      }

      fusedLocationClient.lastLocation.addOnSuccessListener { location: Location? ->
        val result = Arguments.createMap().apply {
          putDouble("latitude", location?.latitude ?: 0.0)
          putDouble("longitude", location?.longitude ?: 0.0)
          putDouble("altitude", location?.altitude ?: 0.0)
        }
        promise.resolve(result)
      }
    } catch (e: Exception) {
      promise.reject("LOCATION_ERROR", e.message)
    }
  }
}
```
