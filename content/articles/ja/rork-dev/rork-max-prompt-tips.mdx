# Rork Max プロンプトエンジニアリング — 高品質アプリを作る10のコツ

Rork Maxは自然言語からネイティブSwiftアプリを生成しますが、出力の品質はプロンプトの書き方に大きく左右されます。曖昧なプロンプトではありきたりなアプリになり、的確なプロンプトならApp Storeに出せるレベルのアプリが完成します。

この記事では、Rork Max特有のプロンプトテクニックを実例とともに解説します。

## プロンプトが重要な理由

Rork MaxはClaude CodeとOpus 4.6を搭載しており、ユーザーの指示を解釈してSwiftUIコードを生成します。AIは生成中にナビゲーション構造、データモデル、カラースキーム、アニメーション、エラーハンドリングなど無数の判断を行います。プロンプトが唯一の指示手段です。

良いプロンプトを書けば、修正回数が減り、コードがクリーンになり、最初の生成でイメージ通りのアプリが得られます。

## コツ1: ユーザー体験を一連の流れで記述する

機能のリストを並べるのではなく、ユーザーがアプリを開いてから何をするかを具体的に書きましょう。

### 弱いプロンプト

```
ワークアウトトラッカーアプリを作って。エクササイズ、セット、レップス機能付き。
```

### 強いプロンプト

```
ワークアウトトラッカーアプリを作成してください。アプリを開くと
今日のワークアウトプランがカード形式で表示されます。カードを
タップするとセットとレップスがチェックボックス付きで展開されます。
フローティングの「+」ボタンでカスタムエクササイズを追加できます。
全セット完了後にお祝いアニメーションが再生され、ワークアウト
サマリーがボトムタブからアクセスできる履歴画面に保存されます。
```

具体的なユーザーフローを記述することで、AIはより一貫性のある洗練されたアプリを生成できます。

## コツ2: ビジュアルスタイルを最初に指定する

Rork Maxはロジックとともにデザインも生成します。デザインの方向性を指定しないと、デフォルトの見た目になります。

### 記述例

```
ダークテーマを使用。プライマリ背景色はディープネイビー(#1A1B2E)。
アクセントカラーはインタラクティブ要素にエレクトリックブルー
(#4A90D9)、テキストにソフトホワイト(#F0F0F5)。見出しに
SF Pro Rounded、本文にSF Pro Textを使用。カードは角丸16ptで
グラスモーフィズム効果を適用。
```

HEXカラー、フォント、角丸の値、視覚効果を明示するほど、出力がイメージに近づきます。

## コツ3: ナビゲーション構造を定義する

画面のつながり方をAIに推測させず、ナビゲーションパターンを明示しましょう。

### 記述例

```
4つのタブバーを使用: ホーム(家アイコン)、検索(虫眼鏡)、
お気に入り(ハート)、プロフィール(人物)。ホームタブは
NavigationStackでドリルダウン。検索タブは上部に検索バー、
その下にフィルターチップを配置。
```

ナビゲーションの明示は、画面遷移の不整合という最もよくある問題を防ぎます。

## コツ4: エッジケースと空の状態を含める

実用的なアプリはデータがない場合、ネットワーク接続がない場合、想定外の入力に対処する必要があります。

### 記述例

```
レシピリストが空のとき、イラストと「まだレシピがありません
— +をタップして最初のレシピを追加しましょう」のテキストを表示。
ネットワークエラー時は「レシピを読み込めませんでした。接続を
確認して再試行してください」のリトライボタンを表示。ローディング中は
カードレイアウトに合わせたスケルトンビューを使用。
```

エッジケースの処理がアプリの完成度を左右します。プロンプトに含めることでAIが適切にハンドリングします。

## コツ5: 複雑なアプリは段階的に構築する

Rork Maxは反復的な開発ワークフローをサポートしています。複雑なアプリはコア機能から始めて、フォローアッププロンプトで改善しましょう。

### 反復戦略

1. **1回目**: コア画面とナビゲーション構造
2. **2回目**: データ永続化と状態管理
3. **3回目**: アニメーション、トランジション、仕上げ
4. **4回目**: ネイティブAPI統合（HealthKit、ARKitなど）

各イテレーションは前回の結果の上に構築されます。一度に全てを記述するより、このアプローチの方が確実です。

## コツ6: Appleのフレームワーク名を直接参照する

Rork MaxはネイティブSwiftを生成するため、Appleのフレームワークや機能を直接参照できます。AIはこれらの用語を正確に理解します。

### 効果的な参照

```
リストと詳細ビューの遷移にSwiftUIの.matchedGeometryEffectを使用。
ボタンタップ時にUIImpactFeedbackGeneratorで触覚フィードバックを
実装。Dynamic IslandのLive Activityでタイマーのカウントダウンを表示。
```

具体的なAPI名を指定することで曖昧さがなくなり、適切なフレームワークを使用したコードが生成されます。

## コツ7: データモデルと関連を明示する

アプリが構造化データを扱う場合、モデルを明示的に記述しましょう。

### 記述例

```
データモデル:
- Recipe: title(String）、ingredients（Ingredientの配列）、
  steps（Stringの配列）、prepTime（Int、分単位）、
  difficulty（列挙型: easy/medium/hard）、isFavorite（Bool）
- Ingredient: name（String）、quantity（Double）、unit（String）
- MealPlan: date（Date）、meals（breakfast/lunch/dinnerを
  Recipeにマッピングする辞書）
```

データ構造を明示することで、後々問題になるようなAIの仮定を防げます。

## コツ8: 比較参照を活用する

気に入ったアプリやUIパターンがあれば、参照として活用しましょう。AIは人気アプリのデザインについて幅広い知識を持っています。

### 記述例

```
習慣トラッカーのデザインで、GitHubのコントリビューショングラフの
ような連続記録カレンダーを使用。完了日は緑の四角、未完了は灰色、
強度を示すグラデーション付き。週間ビューはApple Healthのリング
スタイルの進捗インジケーターに似せる。
```

比較参照を使うと、複雑なデザインアイデアを素早く正確に伝えられます。

## コツ9: アニメーションとモーションデザインを指定する

アニメーションが機能的なプロトタイプと洗練されたアプリの違いを生みます。AIに任せきりにしないようにしましょう。

### 記述例

```
タスク完了時、チェックボックスがスプリングアニメーション（0.4秒、
わずかなバウンス）で塗りつぶされる。カードは0.3秒の
ease-in-outトランジションで「完了」セクションにスムーズに移動。
タスク完了時に.sensoryFeedback(.success)を使用。
```

SwiftUIは優れたアニメーションサポートを持っており、Rork Maxは明確な記述があれば高度なモーションデザインを実装できます。

## コツ10: 実機で早期かつ頻繁にテストする

Rork MaxではQRコードをスキャンしてiPhoneに直接アプリをインストールできます。この素早いフィードバックループを活用しましょう。

生成サイクルごとに実機にインストールし、タッチターゲット、スクロールパフォーマンス、アニメーションの滑らかさ、テキストの読みやすさを確認してください。シミュレーターでは問題なく見えても、実機では異なる感触になることがあります。

観察結果を正確なフォローアッププロンプトに活かしましょう。「UIをもっと良くして」よりも「ボトムタブバーのアイコンが小さすぎる。タップターゲットを最低44x44ポイントに拡大して」の方がはるかに効果的です。

## ボーナス: プロンプトテンプレート

Rork Maxプロジェクトに使えるテンプレートです:

```
[アプリの種類]アプリを[対象ユーザー]向けに作成してください。

## コア機能
[3〜5つの主要機能をユーザーフローの詳細とともに記述]

## ビジュアルデザイン
[カラーパレット、タイポグラフィ、コンポーネントスタイル]

## ナビゲーション
[タブ構成またはナビゲーションパターン]

## データ
[主要なデータモデルと関連]

## ネイティブAPI
[統合するAppleフレームワーク: HealthKit、ARKitなど]

## エッジケース
[空の状態、エラーハンドリング、ローディング状態]
```

## まとめ

Rork Maxでの効果的なプロンプトエンジニアリングは、具体的でありながら圧倒的にならないバランスが重要です。ユーザーが何を見て、何がどう動いて、何か問題が起きたときにどうなるかという体験を記述し、実装の詳細はAIに任せましょう。

上記のコツを使って最初のプロンプトをしっかり書き、その後反復的に改善していきましょう。各生成サイクルでAIが何をうまく解釈し、どこにより詳しいガイダンスが必要かがわかってきます。練習を重ねれば、一発でリリース可能なアプリを生成するプロンプトを書く直感が身についてきます。
