アプリの起動速度は、ユーザー体験に直結する重要な指標です。特にモバイルアプリケーションでは、起動までの数秒が離脱率を左右します。本記事では、Rork Max を使用したアプリ開発において、起動速度を3倍に短縮できる実践的な最適化テクニックをご紹介します。

## アプリ起動速度が重要な理由

GoogleのUser Experience Report によると、アプリの起動に3秒以上かかると、ユーザーの離脱率が大幅に増加します。特にスマートフォンユーザーは起動速度に敏感であり、競合アプリとの差別化要因となります。

**起動速度改善の効果：**
- ユーザー満足度 +35%
- リテンション率 +20%
- アプリストア評価向上
- CPA削減

## 1. バンドルサイズの最適化

アプリの起動速度を短縮する最も効果的な方法は、バンドルサイズを削減することです。Expo CLI が提供するツールを活用しましょう。

### バンドルサイズの分析

```bash
# バンドルサイズを可視化
npx expo-bundle-analyzer
```

このツールで、どのライブラリが容量を占めているかを特定できます。

### 不要な依存関係の削除

```json
// package.json の例
{
  "dependencies": {
    "react": "18.2.0",
    "react-native": "0.72.0",
    "expo": "~49.0.0",
    // ❌ 削除候補: 重いライブラリの代替を検討
    "lodash": "^4.17.0",  // lodash-es に変更、または必要な関数のみインポート
    // ✅ 軽量代替案
    "react-native-fast-image": "^8.0.0"  // Image の高速化
  }
}
```

lodash の場合、必要な関数のみをインポートすることで大幅に削減できます：

```javascript
// ❌ 非効率: 全体をインポート（約70KB）
import _ from 'lodash';
const sorted = _.sortBy(data, 'name');

// ✅ 効率的: 必要な関数のみインポート（約5KB）
import sortBy from 'lodash/sortBy';
const sorted = sortBy(data, 'name');
```

## 2. コード分割とレイジーローディング

アプリ起動時に必要でないコンポーネントは遅延ロードします。React Native では dynamic import を活用します。

### React.lazy と Suspense の活用

```javascript
import React, { Suspense, lazy } from 'react';
import { View, ActivityIndicator } from 'react-native';

// 遅延ロードするコンポーネント
const SettingsScreen = lazy(() => import('./screens/SettingsScreen'));
const AnalyticsScreen = lazy(() => import('./screens/AnalyticsScreen'));

// フォールバック UI
const LoadingSpinner = () => (
  <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
    <ActivityIndicator size="large" color="#007AFF" />
  </View>
);

export const AppNavigator = () => {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <SettingsScreen />
    </Suspense>
  );
};
```

### React Navigation での遅延ローディング

```javascript
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { lazy, Suspense } from 'react';

const Stack = createNativeStackNavigator();

// スクリーン
const HomeScreen = lazy(() => import('./screens/HomeScreen'));
const DetailScreen = lazy(() => import('./screens/DetailScreen'));

export const RootNavigator = () => {
  return (
    <NavigationContainer>
      <Stack.Navigator
        screenOptions={{
          headerShown: true,
        }}
      >
        <Stack.Screen
          name="Home"
          component={() => (
            <Suspense fallback={<LoadingSpinner />}>
              <HomeScreen />
            </Suspense>
          )}
        />
        <Stack.Screen
          name="Detail"
          component={() => (
            <Suspense fallback={<LoadingSpinner />}>
              <DetailScreen />
            </Suspense>
          )}
        />
      </Stack.Navigator>
    </NavigationContainer>
  );
};
```

## 3. メモリリークの防止とガベージコレクション

メモリリークはアプリケーションを徐々に遅くします。useEffect でのクリーンアップが重要です。

### 一般的なメモリリークパターンと対策

```javascript
import { useEffect, useState, useRef } from 'react';
import { fetchUserData } from './api';

export const UserProfile = ({ userId }) => {
  const [user, setUser] = useState(null);
  const isMountedRef = useRef(true);

  useEffect(() => {
    // コンポーネント初期化時の処理
    isMountedRef.current = true;

    // 非同期処理
    fetchUserData(userId).then((data) => {
      // アンマウント後の状態更新を防止
      if (isMountedRef.current) {
        setUser(data);
      }
    });

    // クリーンアップ関数
    return () => {
      isMountedRef.current = false;
    };
  }, [userId]);

  return user ? <Text>{user.name}</Text> : <ActivityIndicator />;
};
```

### EventEmitter リスナーの正しい管理

```javascript
import { useEffect } from 'react';
import { AppState } from 'react-native';
import * as Analytics from 'expo-analytics';

export const useAppStateListener = () => {
  useEffect(() => {
    const subscription = AppState.addEventListener('change', handleAppStateChange);

    function handleAppStateChange(state) {
      if (state === 'active') {
        Analytics.trackEvent('app_resumed');
      } else if (state === 'background') {
        Analytics.trackEvent('app_backgrounded');
      }
    }

    // ❌ リスナーを削除しないとメモリリーク
    // ✅ 必ずアンマウント時にリスナーを削除
    return () => {
      subscription.remove();
    };
  }, []);
};
```

## 4. ネイティブモジュールの活用

重い処理はネイティブコードで実装することで大幅な高速化が期待できます。

### Expo Modules を使用した高速化

```javascript
// nativeModule.ts
import { NativeModule, requireNativeModule } from 'expo';

interface NativeModuleType {
  performHeavyComputation(data: string): Promise<string>;
}

const NativeModule = requireNativeModule<NativeModuleType>(
  'RorkMaxNativeModule'
);

export const useHeavyComputation = (data: string) => {
  const [result, setResult] = useState<string>('');

  useEffect(() => {
    NativeModule.performHeavyComputation(data).then(setResult);
  }, [data]);

  return result;
};
```

重い計算はネイティブレイヤーで実行することで、JavaScript スレッドのブロッキングを防げます。

## 5. イメージ最適化とキャッシング

画像は多くの場合、バンドルサイズの大部分を占めます。

### 最適な画像フォーマットの選択

```javascript
import { Image } from 'react-native';
import FastImage from 'react-native-fast-image';

// ✅ キャッシング対応で高速
export const OptimizedImage = ({ uri, width = 100, height = 100 }) => {
  return (
    <FastImage
      source={{
        uri,
        priority: FastImage.priority.normal,
        cache: FastImage.cacheControl.immutable,
      }}
      style={{ width, height }}
      resizeMode={FastImage.resizeMode.contain}
    />
  );
};
```

### WebP フォーマットの活用

```bash
# ImageMagick を使用してWebP に変換（ファイルサイズを最大30%削減）
convert image.png -quality 80 image.webp
```

WebP は PNG より約30%ファイルサイズが小さく、起動時間短縮に有効です。

## 7. スタートアップスクリーンの最適化

ネイティブスプラッシュスクリーンを活用して、起動時間を隠蔽します。

```javascript
import * as SplashScreen from 'expo-splash-screen';
import { useEffect } from 'react';

// スプラッシュスクリーンを表示したまま
SplashScreen.preventAutoHideAsync();

export const App = () => {
  const [appIsReady, setAppIsReady] = useState(false);

  useEffect(() => {
    const prepare = async () => {
      try {
        // アプリ初期化処理（API呼び出しなど）
        await initializeApp();
      } catch (e) {
        console.warn(e);
      } finally {
        setAppIsReady(true);
      }
    };

    prepare();
  }, []);

  useEffect(() => {
    // 初期化完了後、スプラッシュスクリーンを隠す
    if (appIsReady) {
      SplashScreen.hideAsync();
    }
  }, [appIsReady]);

  return appIsReady ? <RootNavigator /> : null;
};
```

## 8. バンドルサイズ監視とパフォーマンスバジェット

継続的な監視により、パフォーマンス低下を早期に検出できます。

```javascript
// build.js - ビルド時にバンドルサイズをチェック
const fs = require('fs');
const path = require('path');

const checkBundleSize = () => {
  const bundlePath = path.join(__dirname, 'dist/main.jsbundle');
  const stats = fs.statSync(bundlePath);
  const sizeInMb = stats.size / 1024 / 1024;

  const MAX_BUNDLE_SIZE = 2.5; // MB

  if (sizeInMb > MAX_BUNDLE_SIZE) {
    throw new Error(
      `Bundle size exceeded! ${sizeInMb.toFixed(2)}MB > ${MAX_BUNDLE_SIZE}MB`
    );
  }

  console.log(`✅ Bundle size OK: ${sizeInMb.toFixed(2)}MB`);
};

checkBundleSize();
```

## パフォーマンス測定

改善効果を測定することが重要です。

```javascript

```

## まとめ

以下の施策を組み合わせることで、アプリ起動速度を3倍に短縮できます：

1. **バンドルサイズ削減** - 不要な依存関係を削除（約30-40%削減）
2. **コード分割** - 遅延ロードにより起動時の負荷を低減（約20-25%）
3. **メモリ管理** - リークを防止し安定性向上（約15%）
4. **ネイティブ処理** - 重い計算をネイティブで実行（約10-20%）
5. **Hermes エンジン** - JavaScript エンジンの最適化（約20-30%）
6. **スプラッシュスクリーン** - UX 向上（ユーザー体感）

これらの手法を段階的に導入すれば、確実なパフォーマンス改善が実現できます。
