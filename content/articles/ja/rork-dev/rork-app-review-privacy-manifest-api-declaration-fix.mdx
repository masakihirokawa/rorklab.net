頑張って作ったアプリを App Store Connect に提出したら、翌日こんなメールが届いた、という経験はありますか。

> ITMS-91053: Missing API declarations — Your app's code in the "YourApp.ipa" file uses required reason API. Add a value for the required reason to the PrivacyInfo.xcprivacy file.

初めて見たとき、私もどこを直せばいいのかまったく見当がつかずに数時間を費やしました。Rork Max でネイティブアプリを作っている方なら、このリジェクトに遭遇する可能性は決して低くありません。今回は、このエラーの仕組みと Rork Max プロジェクトへの具体的な対処方法をまとめてみました。

## なぜ突然リジェクトされるのか

Apple は 2024 年から「Required Reason API」というポリシーを導入しています。デバイス情報やファイルシステムにアクセスする特定の API を使う場合、**Privacy Manifest ファイル（PrivacyInfo.xcprivacy）にその使用理由を明記する**ことが義務になりました。

やっかいなのは、アプリ開発者が**直接その API を呼び出していなくても**、Expo や React Native のサードパーティライブラリが内部で使っていれば宣言が必要になる点です。Rork Max のプロジェクトには多数のライブラリが組み込まれているため、知らないうちに引っかかることがあります。

また、このチェックは App Store Connect のバリデーション（自動審査）段階で行われます。人間の審査員が見る前にはじかれるため、「何も悪いことをしていないのに」という感覚になりやすいのです。

## よく問題になる API と理由コード

リジェクトメールには対象の API 名とフレームワーク名が記載されています。Rork Max アプリで特に頻出するのは以下の4つです。

- **UserDefaults（NSUserDefaults）** — アプリの設定・状態の保存に使用。`CA92.1` が最も一般的な理由コード
- **File timestamp APIs（NSFileManager の日付属性）** — キャッシュ管理や同期チェックに使用。`C617.1` を指定
- **Disk space APIs（空き容量の取得）** — ストレージ残量の確認に使用。`E174.1`
- **System boot time APIs** — パフォーマンス計測などに使用。`35F9.1`

`@react-native-async-storage/async-storage`、`react-native-mmkv`、`expo-notifications` など、Rork Max が標準で組み込むライブラリがこれらを使っているケースが多いです。

## Privacy Manifest を追加する2つの方法

### 方法1: Rork AI プロンプトに任せる（おすすめ）

Rork のチャットインターフェイスで以下のように依頼すると、AI がプロジェクトに適切な設定を追加してくれます。

```text
App Store から以下のエラーでリジェクトされました。

ITMS-91053: Missing API declarations

対象: NSPrivacyAccessedAPICategoryUserDefaults,
      NSPrivacyAccessedAPICategoryFileTimestamp

ios/PrivacyInfo.xcprivacy ファイルを作成し、
- UserDefaults: CA92.1
- File timestamp: C617.1
の理由コードで宣言を追加してください。
```

プロンプトに「エラーメール本文をそのまま貼り付ける」と、Rork AI が対象 API を自動で特定してくれます。ただし、生成されたファイルの内容は必ず確認し、実際の使用目的と合っているかチェックしてください。理由コードが実態と合っていないとポリシー違反になります。

### 方法2: PrivacyInfo.xcprivacy を手動で編集する

Rork Max プロジェクトの `ios/` ディレクトリに `PrivacyInfo.xcprivacy` ファイルを作成（または編集）します。以下は UserDefaults と File timestamp の両方を宣言する例です。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<\!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
    "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>NSPrivacyAccessedAPITypes</key>
    <array>
        <\!-- UserDefaults: アプリ機能のための設定読み書き -->
        <dict>
            <key>NSPrivacyAccessedAPIType</key>
            <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
            <key>NSPrivacyAccessedAPITypeReasons</key>
            <array>
                <string>CA92.1</string>
            </array>
        </dict>
        <\!-- File timestamp: ファイル同期・キャッシュ管理 -->
        <dict>
            <key>NSPrivacyAccessedAPIType</key>
            <string>NSPrivacyAccessedAPICategoryFileTimestamp</string>
            <key>NSPrivacyAccessedAPITypeReasons</key>
            <array>
                <string>C617.1</string>
            </array>
        </dict>
    </array>
</dict>
</plist>
```

このファイルを追加したら、EAS Build で**新しいビルドを作り直す**ことを忘れないでください。既存のビルドには変更が反映されません。

## 依存ライブラリ側の Privacy Manifest も関係する

2024 年後半から、主要な npm パッケージは各ライブラリ自身の Privacy Manifest を持つようになっています。`expo-notifications`、`react-native-mmkv` の最新バージョンではすでに対応済みです。

もし古いバージョンのライブラリを使っている場合、ライブラリを最新版に更新するだけで問題が解決することがあります。

```bash
# 依存ライブラリの更新（Rork Max プロジェクトで実行）
npx expo install --fix

# 問題のある依存関係を診断
npx expo-doctor
```

`npx expo-doctor` は依存ライブラリの Privacy Manifest 対応状況を診断してくれる便利なコマンドです。ビルド前に実行する習慣をつけておくと、無駄なリジェクトを防ぎやすくなります。

## EAS Build キャッシュに注意

設定を変更したのにリジェクトが繰り返されるとしたら、EAS Build のキャッシュが原因かもしれません。キャッシュをクリアしてビルドし直すと解決することがあります。

```bash
# キャッシュクリアしてビルド
eas build --platform ios --clear-cache
```

また、App Store Connect にアップロードする前に「Validate App」を実行すると、ITMS-91053 エラーをアップロード前に検知できます。Xcode や Transporter でアーカイブを検証する手順は、[Apple の公式ドキュメント](https://developer.apple.com/documentation/bundleresources/privacy_manifest_files/describing_use_of_required_reason_api)に詳しく記載されています。

## 再提出前のチェックリスト

Privacy Manifest を追加したら、以下を確認してから再提出することをおすすめします。

- `PrivacyInfo.xcprivacy` の `NSPrivacyAccessedAPITypes` にリジェクトメール記載の全 API が含まれているか
- 各 API に対して適切な理由コード（1つ以上）が設定されているか
- `npx expo-doctor` を実行して依存ライブラリ側の問題がないか確認したか
- `eas build --clear-cache` で新しいビルドを作成したか
- App Store Connect のアーカイブ検証で追加の警告が出ていないか

## 同じリジェクトを繰り返さないために

一度対処したら、プロジェクトの `PrivacyInfo.xcprivacy` をバージョン管理（git）に含めておくことが大切です。新しいライブラリを追加するたびに、そのライブラリが Required Reason API を使っていないか確認する癖をつけておくと、将来的なリジェクトを防ぎやすくなります。

Rork Max は今後もネイティブ機能を強化していく方向性を示しています。それに伴って新しい API 要件が出てくる可能性もありますので、Apple の Privacy Manifest に関するドキュメントは定期的に確認することをおすすめします。

まずやることは一つ、手元のリジェクトメールを開いて対象 API を確認し、上のテンプレートを参考に `PrivacyInfo.xcprivacy` を追加してみてください。多くのケースはこれだけで解決します。
