ユーザーがアプリ内で「探しているものをすぐ見つけられる」体験は、継続率と満足度に直結します。テキストフィールドで一覧をフィルタリングするだけでは、データ量が増えると途端に使いにくくなります。そこで強力な味方になるのが **Algolia** です。

この記事では、Rorkで構築したReact Nativeアプリに、AlgoliaのマネージドサーチエンジンとJavaScript SDKを組み合わせて、**本格的なアプリ内全文検索**を実装する手順をステップバイステップで解説します。検索候補のリアルタイム表示（オートコンプリート）から、カテゴリフィルタリングの実装まで網羅しています。

## Algolia とは？なぜ選ばれるのか

Algolia はホスト型の検索・ディスカバリー API です。データを Algolia のインデックスにアップロードしておくと、SDK 経由でミリ秒単位の全文検索が行えます。主な特徴を整理すると次のとおりです。

- **高速レスポンス**: エッジに配置された分散インフラにより、世界中どこからでも平均 1〜2ms のレイテンシを実現
- **タイポ許容**: 英語・日本語ともに typo-tolerance（スペルミス許容）を標準装備
- **フィルタリング・ファセット**: カテゴリや価格帯など複数の軸での絞り込みが容易
- **無料枠あり**: 月 1 万件のレコード・月 1 万回の検索まで無料（Build プラン、2026年4月時点）

React Native アプリの検索実装には、Algolia の公式 JavaScript SDK「**algoliasearch**」がそのまま使えるため、Rork との相性は抜群です。

## Step 1: Algolia アカウントのセットアップとインデックス作成

まず [Algolia の公式サイト](https://www.algolia.com/) でアカウントを作成し、新しい **Application** を作ります。作成後、ダッシュボードの **Indices** から新しいインデックスを作成してください。今回は例として `products` というインデックス名を使います。

次に、**API Keys** の画面から以下の 2 つのキーを控えておきます。

- **Application ID**: アプリケーションを識別するID（例: `ABCDEFGHIJ`）
- **Search-Only API Key**: 検索専用の読み取りキー（クライアントに埋め込んで安全に使える）

> ⚠️ Admin API Key はサーバーサイドのデータ登録・更新にのみ使用し、アプリのコードには絶対に含めないでください。

インデックスにサンプルデータを登録するには、Algolia ダッシュボードの **Records** タブから JSON を直接アップロードするか、Node.js スクリプトで一括インポートします。本記事では商品カタログを例に進めます。

```json
[
  { "objectID": "1", "name": "ワイヤレスイヤホン Pro", "category": "audio", "price": 12800 },
  { "objectID": "2", "name": "スマートウォッチ Ultra", "category": "wearable", "price": 39800 },
  { "objectID": "3", "name": "ポータブルスピーカー Mini", "category": "audio", "price": 5980 }
]
```

## Step 2: algoliasearch SDK の導入

Rork のプロジェクトには `algoliasearch` パッケージを追加します。Rork の AI チャットに以下のプロンプトを入力するか、プロジェクトの `package.json` に直接追加してください。

```
Add the algoliasearch npm package to this project
```

もし自分で `package.json` を編集する場合は、`dependencies` に以下を追加します。

```json
{
  "dependencies": {
    "algoliasearch": "^5.0.0"
  }
}
```

インストール後、API キーは環境変数または定数ファイルで管理します。`.env` ファイルを使う場合は Expo の規約に従い `EXPO_PUBLIC_` プレフィックスを付けます。

```bash
# .env（gitignoreに追加すること）
EXPO_PUBLIC_ALGOLIA_APP_ID=YOUR_ALGOLIA_APP_ID
EXPO_PUBLIC_ALGOLIA_SEARCH_KEY=YOUR_SEARCH_ONLY_API_KEY
```

## Step 3: 検索クライアントの初期化と基本的な検索実装

Algolia クライアントを初期化するユーティリティファイルを作成します。

```typescript
// lib/algolia.ts
import algoliasearch from 'algoliasearch';

// 環境変数からAPIキーを取得（Search-Only Keyのみ使用）
const APP_ID = process.env.EXPO_PUBLIC_ALGOLIA_APP_ID ?? '';
const SEARCH_KEY = process.env.EXPO_PUBLIC_ALGOLIA_SEARCH_KEY ?? '';

// Algoliaクライアントのシングルトン
export const algoliaClient = algoliasearch(APP_ID, SEARCH_KEY);

// 使用するインデックス名
export const PRODUCTS_INDEX = 'products';
```

次に、検索ロジックを持つカスタムフックを作成します。`useDebounce` でキー入力のたびに API を叩かないよう最適化しています。

```typescript
// hooks/useAlgoliaSearch.ts
import { useState, useEffect, useCallback } from 'react';
import { algoliaClient, PRODUCTS_INDEX } from '../lib/algolia';

type SearchResult = {
  objectID: string;
  name: string;
  category: string;
  price: number;
};

export function useAlgoliaSearch() {
  const [query, setQuery] = useState('');
  const [results, setResults] = useState<SearchResult[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  // 300msデバウンスで検索（APIコール数を削減）
  useEffect(() => {
    if (!query.trim()) {
      setResults([]);
      return;
    }

    const timer = setTimeout(async () => {
      setIsLoading(true);
      setError(null);
      try {
        const { hits } = await algoliaClient.searchSingleIndex<SearchResult>({
          indexName: PRODUCTS_INDEX,
          searchParams: {
            query,
            hitsPerPage: 20,      // 1ページあたりの最大件数
            attributesToRetrieve: ['objectID', 'name', 'category', 'price'],
          },
        });
        setResults(hits as SearchResult[]);
      } catch (err) {
        setError('検索中にエラーが発生しました。しばらくしてから再試行してください。');
        console.error('Algolia search error:', err);
      } finally {
        setIsLoading(false);
      }
    }, 300);

    return () => clearTimeout(timer);
  }, [query]);

  const clearSearch = useCallback(() => {
    setQuery('');
    setResults([]);
  }, []);

  return { query, setQuery, results, isLoading, error, clearSearch };
}
```

外部APIとの連携パターン全般については [Rork 外部 API 連携ガイド](/articles/rork-dev/rork-external-api-integration-guide) も合わせてご覧ください。

## Step 4: 検索 UI コンポーネントの実装

フックをコンポーネントに接続して、検索バーと結果リストを組み立てます。

```tsx
// screens/SearchScreen.tsx
import React from 'react';
import {
  View,
  TextInput,
  FlatList,
  Text,
  TouchableOpacity,
  ActivityIndicator,
  StyleSheet,
} from 'react-native';
import { useAlgoliaSearch } from '../hooks/useAlgoliaSearch';

export default function SearchScreen() {
  const { query, setQuery, results, isLoading, error, clearSearch } = useAlgoliaSearch();

  return (
    <View style={styles.container}>
      {/* 検索バー */}
      <View style={styles.searchBar}>
        <TextInput
          style={styles.input}
          placeholder="商品を検索..."
          value={query}
          onChangeText={setQuery}
          autoCorrect={false}
          returnKeyType="search"
          clearButtonMode="while-editing"  // iOS専用クリアボタン
        />
      </View>

      {/* ローディング */}
      {isLoading && (
        <ActivityIndicator size="small" color="#6366f1" style={styles.loading} />
      )}

      {/* エラー表示 */}
      {error && (
        <Text style={styles.errorText}>{error}</Text>
      )}

      {/* 検索結果 */}
      <FlatList
        data={results}
        keyExtractor={(item) => item.objectID}
        renderItem={({ item }) => (
          <TouchableOpacity style={styles.resultItem}>
            <Text style={styles.itemName}>{item.name}</Text>
            <Text style={styles.itemMeta}>
              {item.category} · ¥{item.price.toLocaleString()}
            </Text>
          </TouchableOpacity>
        )}
        ListEmptyComponent={
          query.length > 0 && !isLoading ? (
            <Text style={styles.emptyText}>「{query}」に一致する商品は見つかりませんでした</Text>
          ) : null
        }
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#fff' },
  searchBar: {
    padding: 12,
    borderBottomWidth: 1,
    borderBottomColor: '#e5e7eb',
  },
  input: {
    backgroundColor: '#f3f4f6',
    borderRadius: 10,
    paddingHorizontal: 14,
    paddingVertical: 10,
    fontSize: 16,
  },
  loading: { marginTop: 16 },
  errorText: { color: '#ef4444', textAlign: 'center', margin: 16 },
  resultItem: {
    padding: 16,
    borderBottomWidth: 1,
    borderBottomColor: '#f3f4f6',
  },
  itemName: { fontSize: 15, fontWeight: '600', color: '#111827' },
  itemMeta: { fontSize: 13, color: '#6b7280', marginTop: 2 },
  emptyText: { textAlign: 'center', color: '#9ca3af', margin: 24 },
});
```

## Step 5: カテゴリフィルタリングの追加

Algolia のフィルタリング機能を活用すると、検索クエリにカテゴリの絞り込みを組み合わせることができます。まず Algolia ダッシュボードで **Configuration > Facets** にアクセスし、`category` を **Attribute for faceting** に追加してください。

フックにカテゴリフィルタを追加します。

```typescript
// hooks/useAlgoliaSearch.ts（フィルタ対応版 抜粋）
const { hits } = await algoliaClient.searchSingleIndex<SearchResult>({
  indexName: PRODUCTS_INDEX,
  searchParams: {
    query,
    hitsPerPage: 20,
    // カテゴリフィルタ（selectedCategory が null の場合はフィルタなし）
    filters: selectedCategory ? `category:${selectedCategory}` : undefined,
    facets: ['category'],           // ファセット集計（カテゴリ別の件数を取得）
  },
});
```

データフェッチのキャッシュ最適化には [Rork × TanStack Query データフェッチ＆キャッシュ完全ガイド](/articles/rork-dev/rork-tanstack-query-data-fetching-cache-guide) のパターンが参考になります。

## よくあるエラーと対処法

Algolia を Rork / Expo 環境で実装する際に頻出する問題と解決策をまとめます。

**エラー 1: `invalid credentials` — クライアントが初期化できない**

原因は環境変数の読み込み失敗がほとんどです。`.env` ファイルの変数名が `EXPO_PUBLIC_` で始まっているか、Expo のキャッシュをクリア（`npx expo start --clear`）した後に再起動しているかを確認してください。

**エラー 2: 日本語で検索しても結果が出ない**

Algolia のデフォルトは英語圏の tokenizer を使用します。日本語テキストを正しくインデックスするには、インデックス設定の **Language** を `Japanese` に変更し、データを再インポートしてください。また、日本語フィールドを `searchableAttributes` に明示的に追加することも重要です。

**エラー 3: 検索が遅い、API コール数が多い**

フックのデバウンス時間（サンプルでは 300ms）を調整するか、`useMemo` でクライアントのインスタンスを記憶化することで改善できます。

## FAQ

**Q. Algolia の無料プランはどのくらい使えますか？**

2026年4月時点の Build プランでは、月 1 万件のレコードと月 1 万回の検索（オペレーション）が無料です。個人開発やスタートアップの MVP 段階であれば十分な容量です。トラフィックが増えてきた段階で有料プランへの移行を検討してください。

**Q. Algolia の代替として Firebase Firestore の検索は使えますか？**

Firestore は `startsWith` などの前方一致検索は可能ですが、全文検索やタイポ許容には対応していません。検索体験にこだわる場合は Algolia、Typesense、または Meilisearch といった専用の検索エンジンを組み合わせるのがベストプラクティスです。

**Q. データを Algolia に同期する方法は？**

Supabase や Firebase などのデータベースと Algolia の同期には、Webhook + サーバーレス関数のパターンが一般的です。Supabase の Database Webhook と Algolia Admin SDK を組み合わせれば、データの追加・更新・削除を自動的に Algolia に反映できます。また、[Rork でリアルタイムチャットアプリを構築する — Supabase認証・リアルタイムDB・プッシュ通知の完全実装ガイド](/articles/rork-dev/rork-realtime-chat-app-supabase-complete-guide) では、Supabase との連携パターンをさらに詳しく解説しています。

**Q. インデックスの更新はリアルタイムに反映されますか？**

Algolia への書き込みは非同期で処理されますが、通常数秒以内に検索可能な状態になります。厳密な即時反映が必要な場合は、書き込み後に `waitForTask` を使って処理完了を待つことができます。

**Q. TypeScript の型安全性は保てますか？**

`algoliasearch` v5 以降は TypeScript ファーストで設計されており、`searchSingleIndex<T>` のようにジェネリクスで型を指定できます。`Hit<T>` 型を活用すれば、`objectID` や `_highlightResult` を含む Algolia 固有のフィールドも型安全に扱えます。

## まとめ

Rork × Algolia の組み合わせにより、React Native アプリに本格的なリアルタイム全文検索を短期間で実装できます。今回紹介したポイントを振り返ります。

- Algolia の Search-Only API Key をクライアントに埋め込み、Admin Key は絶対に使用しない
- `algoliasearch` v5 SDK でシンプルかつ型安全な検索クライアントを作成
- 300ms デバウンスで API コール数を最適化
- ファセット設定によりカテゴリフィルタリングも容易に追加可能
- 日本語検索には Language 設定の変更とフィールドの `searchableAttributes` 設定が必要
