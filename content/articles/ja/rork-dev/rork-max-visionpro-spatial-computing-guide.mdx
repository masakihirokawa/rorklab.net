## Rork Max が Vision Pro 開発を劇的に変える

2026年2月にローンチされた**Rork Max**は、ネイティブSwiftアプリをAIで生成するプラットフォームです。iPhone、iPad、Apple Watchに加え、**Apple Vision Pro（visionOS）**にも対応しており、空間コンピューティングアプリの開発がプロンプトひとつで可能になりました。

従来のvisionOS開発では、RealityKit、ARKit、SwiftUIの3D拡張など専門知識が必要でした。Rork Maxは、これらの技術スタックを抽象化し、自然言語の指示からvisionOS対応のアプリを自動生成します。ローンチからわずか3日で150万ドルのARRを達成した実績が、このツールの需要の高さを物語っています。

## Vision Pro 開発の基礎知識

### visionOS のアプリ形態

visionOS上のアプリには、大きく3つの表示形態があります。

| 形態 | 説明 | Rork Max 対応 |
|---|---|---|
| **Window** | 2DのSwiftUIウィンドウ（iPad的） | ✅ 完全対応 |
| **Volume** | 3Dコンテンツを含むボリューム | ✅ 対応 |
| **Full Space** | 没入型の空間体験 | ⚠️ 一部対応 |

### Rork Max での開発フロー

```
1. プロンプト入力（自然言語で指示）
   ↓
2. Rork Max が SwiftUI + RealityKit コードを生成
   ↓
3. クラウドMacでビルド＆シミュレーター実行
   ↓
4. ライブプレビューでリアルタイム確認
   ↓
5. ワンクリックでデバイスインストール or App Store提出
```

## 実践: Vision Pro アプリを作ってみる

### 基本的な 3D ビューアの作成

```
プロンプト例:
「3Dモデルビューアアプリを作って。
ユーザーがUSDZファイルを開いて、
ハンドジェスチャーで回転・拡大・縮小できるようにして。
背景は半透明のパススルーにして。」
```

Rork Maxが生成するコードの概要:

```swift
// Rork Max が自動生成する visionOS コード例
import SwiftUI
import RealityKit

struct ModelViewerApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .windowStyle(.volumetric) // ボリュームウィンドウ
    }
}

struct ContentView: View {
    @State private var selectedModel: URL?

    var body: some View {
        VStack {
            if let modelURL = selectedModel {
                Model3D(url: modelURL) { model in
                    model
                        .resizable()
                        .scaledToFit()
                        .frame(depth: 300) // 奥行き設定
                } placeholder: {
                    ProgressView()
                }
                .gesture(
                    RotateGesture3D()
                        .onChanged { value in
                            // ハンドジェスチャーで回転
                        }
                )
            } else {
                Button("モデルを選択") {
                    // ファイルピッカーを表示
                }
            }
        }
    }
}
```

### 空間UIの設計パターン

Vision Proアプリでは、2Dと3Dのハイブリッドなインターフェースが重要です。

```
プロンプト例:
「タスク管理アプリをVision Pro用に作って。
メインのタスクリストは2Dウィンドウで表示して、
タスクの詳細をタップすると3Dのカード型UIが
空間上にポップアップするようにして。」
```

### Ornament（オーナメント）の活用

visionOSの特徴的なUI要素として「Ornament」があります。ウィンドウの端に浮かぶ補助的なUIで、ツールバーやナビゲーションに使用します。

```
プロンプト例:
「ウィンドウの下部にOrnamentを追加して。
カテゴリフィルター（仕事、個人、買い物）のトグルボタンを
配置して。選択中のカテゴリはハイライト表示にして。」
```

## ハンドジェスチャーとアイトラッキング

### 対応ジェスチャー

Rork Maxは、以下のvisionOSジェスチャーを自動でコードに組み込みます。

- **タップ**: 人差し指と親指をつまむ
- **ドラッグ**: つまんだまま移動
- **回転**: 両手で回転操作
- **ズーム**: ピンチイン・ピンチアウト

```
プロンプト例:
「画像ギャラリーアプリを作って。
タップで画像を選択、ピンチで拡大、
両手の回転ジェスチャーで画像を回転できるようにして。
アイトラッキングで見ている画像をハイライトして。」
```

## SharePlay による共有体験

Vision Proの強力な機能の一つが、SharePlayを使ったマルチユーザー体験です。

```
プロンプト例:
「プレゼンテーションアプリを作って。
SharePlayで複数のVision Proユーザーが
同じ3Dモデルを一緒に見ながら議論できるようにして。
各ユーザーのアバターを空間上に表示して。」
```

Rork MaxはSharePlayのセッション管理やデータ同期のコードを自動生成します。[SharePlay × Rork Maxの詳細](/articles/rork-dev/rork-max-shareplay-group-activities)も併せてご覧ください。

### visionOS アプリのレビューガイドライン

Vision Proアプリには、Apple固有のレビュー基準があります。

- 空間UI要素がApple Human Interface Guidelinesに準拠していること
- ハンドジェスチャーの操作が直感的であること
- パススルー表示との調和がとれていること（不快な視覚体験を避ける）
- バッテリー消費が適切な範囲に収まること

## よくある質問（FAQ）

### Q1: visionOS アプリの開発にMacは必要ですか？

Rork Maxではクラウド上のMacでビルドが行われるため、手元のMacは不要です。Webブラウザからプロジェクトを操作し、シミュレーターのライブプレビューもクラウド上で実行されます。

### Q2: 実機のVision Proがなくてもテストできますか？

Rork Maxのシミュレーターで基本的な動作確認は可能です。ただし、ハンドジェスチャーの精度やパススルーとの調和など、実機でしか確認できない項目もあるため、最終テストでは実機を推奨します。

### Q3: 既存のiPhoneアプリをVision Pro対応にすることはできますか？

Rork Maxで作成したiPhoneアプリに対して「Vision Pro対応にして」と指示することで、visionOS向けのUIを追加生成できます。ただし、空間UIの設計は画面ベースのUIとは根本的に異なるため、大幅な調整が必要になる場合があります。
