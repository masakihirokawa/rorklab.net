## RorkアプリをGoogle Playに公開しよう

RorkはReact Native（Expo）ベースのクロスプラットフォーム開発ツールです。iOSだけでなく、Androidアプリも同じプロジェクトから生成できます。この記事では、Rorkで作ったアプリをGoogle Play Storeに公開するための手順を、準備から審査通過まで丁寧に解説します。

---

## 必要な準備

### Google Play Developer アカウント

Google Play Storeにアプリを公開するには、**Google Play Developer アカウント**が必要です。

- 登録費用: **$25（一回限り）**
- 登録先: [Google Play Console](https://play.google.com/console)
- Googleアカウントでサインインして手続きを完了する

> **Tip**: 企業・個人問わず同じ手順で登録できます。個人開発者でも問題ありません。

### Rorkで必要な設定

Rorkプロジェクトに以下の情報が設定されていることを確認してください。

| 項目 | 説明 |
|------|------|
| アプリ名 | ストアに表示される名前 |
| Bundle ID（パッケージ名） | `com.yourcompany.appname` 形式 |
| バージョン番号 | `1.0.0` など |
| アプリアイコン | 1024×1024px の PNG |

---

## Step 1: Rorkからビルドを生成する

Rorkのプロジェクト画面から「**Export / Deploy**」を選択し、Android向けビルドを作成します。

```
1. Rorkプロジェクトを開く
2. 右上の「Deploy」ボタンをクリック
3. プラットフォームとして「Android」を選択
4. 「Build for Play Store」オプションを有効にする
5. ビルドを開始する（数分かかります）
```

ビルドが完了すると、**AAB（Android App Bundle）**形式のファイルがダウンロードできます。Google Playへの提出には、従来のAPKではなくAABが推奨されています。

> **ポイント**: Rork Maxは現時点ではApple専用（SwiftUI）です。Androidアプリを公開したい場合は、オリジナルのRork（React Native）を使いましょう。

---

## Step 2: Google Play Consoleでアプリを作成する

1. [Google Play Console](https://play.google.com/console) にアクセス
2. 「**アプリを作成**」ボタンをクリック
3. 以下の情報を入力する:

```
- アプリ名: あなたのアプリの名前
- デフォルトの言語: 日本語（または英語）
- アプリかゲームか: アプリ
- 無料か有料か: 任意
```

4. 「アプリを作成」を確定する

---

## Step 3: ストアの掲載情報を設定する

アプリを審査に提出する前に、以下のストア掲載情報を設定します。

### 必須のメタデータ

```yaml
アプリ名: 50文字以内
簡単な説明: 80文字以内（検索結果に表示）
詳細な説明: 4000文字以内
```

### 必要な画像素材

| 素材 | サイズ | 枚数 |
|------|--------|------|
| アイコン | 512×512px PNG | 1枚 |
| フィーチャーグラフィック | 1024×500px | 1枚 |
| スクリーンショット（スマートフォン） | 320〜3840px | 2〜8枚 |
| スクリーンショット（タブレット） | 任意 | — |

### カテゴリとタグ

アプリのジャンルに合わせて**カテゴリ**（例：「ツール」「ライフスタイル」「仕事効率化」）を選択します。

---

## Step 4: アプリのコンテンツを設定する

Google Playでは、アプリが扱うコンテンツについての申告が必要です。

### コンテンツレーティング

「**ポリシー → コンテンツレーティング**」セクションから質問に答えて、対象年齢レーティングを取得します。これはリリース前に必須です。

### データの安全性

「**ポリシー → データの安全性**」では、アプリが収集するデータを申告します。

```
例:
- ユーザーの位置情報を収集するか？ → いいえ
- ユーザーのメールアドレスを収集するか？ → はい（ログイン機能のため）
- データの暗号化 → Expo/React Nativeのデフォルト設定を記入
```

正確な申告をしないとアプリが削除されるリスクがあるため、Rorkで生成されたコードが使っているライブラリ（Firebase、Supabaseなど）の通信を確認しておきましょう。

---

## Step 5: リリーストラックを選択してAABをアップロードする

Google Playには複数のリリーストラックがあります。

| トラック | 用途 |
|---------|------|
| 内部テスト | チームメンバー（最大100人）向けの非公開テスト |
| クローズドテスト（アルファ） | 招待制の限定テスト |
| オープンテスト（ベータ） | 一般ユーザーへの公開ベータ |
| 製品版 | 全ユーザーへの正式リリース |

初めて公開する場合は、**内部テスト → ベータ → 製品版**の順に段階的にリリースするのがおすすめです。

### AABのアップロード手順

```
1. 「リリース → 製品版（または内部テスト）」を選択
2. 「新しいリリースを作成」をクリック
3. 「アプリバンドルをアップロード」からAABファイルを選択
4. リリースノートを入力（例：「初回リリース」）
5. 「レビューを保存」→「リリース」を押す
```

---

## Step 6: 審査と公開

提出後、Googleの審査チームがアプリを確認します。

- **審査期間**: 通常 **1〜3営業日**（初回は1週間程度かかる場合も）
- **審査結果**: メールとPlay Consoleの通知で届く
- **否認の場合**: 指摘された問題を修正して再提出する

### よくある審査否認の理由

```
1. スクリーンショットがアプリの実際の画面と一致しない
2. 説明文に誇大表現が含まれている（「世界最高」など）
3. データの安全性セクションの申告が不正確
4. アプリ内のリンクが壊れている
5. プライバシーポリシーのURLが無効
```

> **Tip**: プライバシーポリシーはアプリに必須です。RorkプロジェクトにPrivacy Policyページがない場合は、ウェブ上に用意してURLをPlay Consoleに登録してください。

## Rork固有の注意点

### パッケージ名の変更はできない

一度Google Playに登録したアプリのパッケージ名（Bundle ID）は変更できません。Rorkプロジェクト作成時に、`com.yourname.appname` のような形式で**ユニークなパッケージ名**を設定しましょう。

### アップデート時のバージョンコード

アプリをアップデートする際は、`versionCode`（整数）を必ず前回より大きい数字にする必要があります。Rorkのエクスポート設定で確認してください。

```js
// app.json (Expo) の例
{
  "expo": {
    "android": {
      "versionCode": 2  // 前回が1なら2以上にする
    }
  }
}
```

### 64ビット対応

2019年以降、Google Playは64ビット対応を義務付けています。RorkはExpoベースのため、デフォルトで64ビット対応のAABが生成されます。特別な設定は不要です。

---

## まとめ

RorkアプリをGoogle Playに公開する手順を整理すると：

1. **Google Play Developer アカウント**を登録（$25）
2. Rorkから**AABビルド**を生成
3. **Play Console**でアプリを作成し、メタデータを入力
4. **コンテンツレーティング**と**データ安全性**を申告
5. **AABをアップロード**してリリーストラックを選択
6. **審査**を経て公開

iOSのApp Storeへの公開と比べると、Google Playは手数料が低く（$25の一回払い）、審査もやや寛容です。Rorkのクロスプラットフォーム機能を活用して、iOSとAndroidの両方にアプリをリリースしましょう。
