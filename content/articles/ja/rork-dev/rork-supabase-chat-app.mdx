## RorkとSupabaseの組み合わせが強い理由

Rorkはフロントエンドのアプリ生成に優れていますが、バックエンド（データベース、認証、リアルタイム通信）にはSupabaseとの連携が強力です。

SupabaseはPostgreSQLベースのBaaS（Backend as a Service）で、リアルタイムデータベース、認証、ストレージを無料枠から利用できます。Rorkとの組み合わせは、特にチャットアプリやSNSのようなリアルタイム性が求められるアプリに最適です。

## プロジェクトのセットアップ

### Supabase側の準備

1. [Supabase](https://supabase.com) でプロジェクトを作成
2. 以下のテーブルを作成

```sql
-- ユーザープロフィール
create table profiles (
  id uuid references auth.users primary key,
  username text unique not null,
  avatar_url text,
  created_at timestamp with time zone default now()
);

-- チャットルーム
create table rooms (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  created_by uuid references profiles(id),
  created_at timestamp with time zone default now()
);

-- メッセージ
create table messages (
  id uuid primary key default gen_random_uuid(),
  room_id uuid references rooms(id) on delete cascade,
  user_id uuid references profiles(id),
  content text not null,
  created_at timestamp with time zone default now()
);
```

3. Realtime を有効化（messages テーブル）

### Rork側の設定

Rorkでプロジェクトを作成し、チャットで以下のように指示します。

```
Supabaseと連携したリアルタイムチャットアプリを作ってください。

機能:
- メールアドレスでのサインアップ/ログイン
- チャットルームの一覧表示
- ルーム内でのリアルタイムメッセージング
- ユーザーアバターの表示

Supabase URL: [あなたのURL]
Supabase Anon Key: [あなたのキー]
```

## 主要機能の実装

### 認証（Supabase Auth）

Rorkは Supabase Auth を使った認証フローを自動生成します。サインアップ、ログイン、パスワードリセットの画面が自動で作られます。

追加の認証方法（Google, Apple等）が必要な場合はチャットで指示します。

```
Googleログインも追加してください。
Supabaseの認証設定でGoogle OAuthが有効になっています。
```

### リアルタイムメッセージング

Supabase Realtimeを使ったメッセージの即時配信が自動的に実装されます。新しいメッセージが投稿されると、同じルームにいる全ユーザーの画面にリアルタイムで反映されます。

### プッシュ通知

新着メッセージのプッシュ通知を追加する場合は、Expo Push Notificationsとの連携を指示します。

```
新しいメッセージが来た時にプッシュ通知を送ってください。
Expo Push Notificationsを使います。
ルームを開いている時は通知を抑制してください。
```

## パフォーマンス最適化

### メッセージの無限スクロール

大量のメッセージがあるルームでは、全メッセージを一度に読み込むとパフォーマンスが低下します。

```
メッセージを20件ずつ読み込む無限スクロールを実装してください。
上にスクロールすると過去のメッセージを追加で読み込みます。
```

### 画像メッセージ

画像の送受信を追加する場合は、Supabase Storageとの連携を指示します。

```
画像メッセージの送受信機能を追加してください。
画像はSupabase Storageにアップロードし、
サムネイルを自動生成してください。
```

## RLS（Row Level Security）の設定

セキュリティのため、Supabase側でRLSを必ず有効にしましょう。

```sql
-- messages テーブルのRLS
alter table messages enable row level security;

-- 認証ユーザーのみメッセージを読める
create policy "Users can read messages" on messages
  for select using (auth.uid() is not null);

-- 自分のメッセージのみ投稿できる
create policy "Users can insert own messages" on messages
  for insert with check (auth.uid() = user_id);
```

## まとめ

Rork × Supabaseの組み合わせは、リアルタイムアプリを最速で構築する方法の一つです。認証、データベース、リアルタイム通信というバックエンドの三大要素をSupabaseが担い、UIとアプリロジックはRorkが自動生成します。次のアプリプロジェクトでぜひ試してみてください。
