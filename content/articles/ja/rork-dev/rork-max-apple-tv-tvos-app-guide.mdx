## Apple TV アプリ開発と Rork Max の組み合わせが注目される理由

Apple TV は、リビングルームでのエンターテインメント体験を提供する大画面プラットフォームとして着実に進化を続けています。tvOS 向けのアプリ開発は以前から可能でしたが、Xcode の tvOS シミュレータでのテストやリモコン操作への対応など、独自のハードルがあり個人開発者にはなかなか手が出しにくい分野でしました。

Rork Max は、クラウド上の Mac フリートを活用して **Xcode 不要でネイティブ Swift アプリを構築・コンパイル・公開** できる AI アプリビルダーです。iPhone や iPad だけでなく、Apple Watch や Vision Pro を含む Apple マルチプラットフォームに対応しています。この仕組みを活かすことで、tvOS アプリの開発もぐっと身近なものになります。

本記事では、Rork Max を使って Apple TV アプリを開発する際の基本知識と実装パターンを、コード例を交えて解説していきます。

## tvOS アプリの特徴と iOS アプリとの違い

tvOS アプリを設計する前に、iOS アプリとの主な違いを理解しておくことが重要です。

**操作方法の違い**: Apple TV ではタッチ操作ではなく、Siri Remote（リモコン）を使ったフォーカスベースのナビゲーションが基本です。ユーザーはリモコンの Touch Surface をスワイプして画面内の要素間を移動し、クリックで決定します。この「Focus Engine」と呼ばれる仕組みに対応した UI 設計が必要です。

**画面サイズとレイアウト**: Apple TV は大画面（1920×1080 または 3840×2160）での表示を前提としています。テキストは離れた場所から読める大きさにし、余白を十分に確保するのが tvOS のデザインガイドラインです。

**バックグラウンド処理の制限**: tvOS にはバックグラウンド実行の厳しい制限があります。長時間のダウンロードやバックグラウンドオーディオなど、一部の機能は iOS と異なる対応が求められます。

**ストレージの制限**: tvOS アプリは永続的なローカルストレージが限られています。大容量のデータは iCloud やサーバーサイドで管理する設計が推奨されます。

## Rork Max で tvOS プロジェクトを始める

Rork Max の AI プロンプトで tvOS アプリの土台を生成するところから始めましょう。ポイントは、プロンプトで「tvOS 対応」であることを明確に伝えることです。

```swift
// Rork Max プロンプト例:
// 「Apple TV 向けの動画カタログアプリを作ってください。
//  トップ画面にカテゴリ一覧、各カテゴリをクリックすると
//  動画のグリッド表示、選択すると詳細画面に遷移する構成です。
//  Focus Engine に対応し、Siri Remote で操作できるようにしてください。」

// 生成されるコードのイメージ（ContentView.swift）
import SwiftUI

struct ContentView: View {
    let categories = ["映画", "ドキュメンタリー", "音楽", "スポーツ"]

    var body: some View {
        NavigationStack {
            List(categories, id: \.self) { category in
                NavigationLink(destination: VideoGridView(category: category)) {
                    HStack {
                        Image(systemName: "play.rectangle.fill")
                            .font(.title)
                        Text(category)
                            .font(.title2)
                    }
                    .padding(.vertical, 12)
                }
            }
            .navigationTitle("ビデオライブラリ")
        }
    }
}

// 期待する出力:
// Apple TV 画面にカテゴリ一覧がリスト表示され、
// Siri Remote のスワイプでフォーカスが移動、
// クリックでカテゴリ内のグリッド画面に遷移する
```

Rork Max はプロンプトの指示をもとに SwiftUI ベースのコードを生成し、クラウド Mac 上でビルドします。tvOS 用のターゲットを指定することで、Apple TV 向けの実行バイナリが生成されます。

## Focus Engine に対応した UI の実装パターン

tvOS 開発で最も重要な概念が Focus Engine です。SwiftUI では `focusable()` 修飾子や `@FocusState` を使ってフォーカス制御を行います。

```swift
import SwiftUI

struct VideoGridView: View {
    let category: String
    @FocusState private var focusedVideoID: String?

    let videos = [
        VideoItem(id: "v1", title: "サンセット・ドライブ", thumbnail: "sunset"),
        VideoItem(id: "v2", title: "深海の世界", thumbnail: "ocean"),
        VideoItem(id: "v3", title: "山頂からの景色", thumbnail: "mountain"),
        VideoItem(id: "v4", title: "都市の夜景", thumbnail: "city"),
    ]

    var body: some View {
        ScrollView {
            LazyVGrid(
                columns: [
                    GridItem(.fixed(400)),
                    GridItem(.fixed(400)),
                    GridItem(.fixed(400))
                ],
                spacing: 40
            ) {
                ForEach(videos) { video in
                    VideoCard(video: video)
                        .focusable()
                        .focused($focusedVideoID, equals: video.id)
                        // フォーカス時にカードを拡大するアニメーション
                        .scaleEffect(focusedVideoID == video.id ? 1.1 : 1.0)
                        .animation(.easeInOut(duration: 0.2), value: focusedVideoID)
                }
            }
            .padding(60)
        }
        .navigationTitle(category)
    }
}

struct VideoCard: View {
    let video: VideoItem

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            RoundedRectangle(cornerRadius: 12)
                .fill(Color.gray.opacity(0.3))
                .frame(height: 225)
                .overlay(
                    Image(systemName: "play.circle.fill")
                        .font(.system(size: 48))
                        .foregroundColor(.white)
                )
            Text(video.title)
                .font(.headline)
        }
        .frame(width: 400)
    }
}

struct VideoItem: Identifiable {
    let id: String
    let title: String
    let thumbnail: String
}

// 期待する出力:
// 3列グリッドでビデオカードが並び、
// Siri Remote でフォーカスを移動すると
// 選択中のカードが 1.1 倍に拡大表示される
```

このパターンでは、フォーカスが当たったカードが少し拡大することで、ユーザーが現在どの要素を選択しているかを視覚的にフィードバックしています。tvOS のデザインガイドラインでも推奨されているアプローチです。

## メディアプレイヤーの組み込み

Apple TV アプリの多くはメディア再生機能を含みます。tvOS では `AVPlayerViewController` が標準のメディアプレイヤー UI を提供しており、自動的に Siri Remote の再生・一時停止・早送り・巻き戻し操作に対応します。

```swift
import SwiftUI
import AVKit

struct VideoPlayerView: View {
    let videoURL: URL

    var body: some View {
        VideoPlayer(player: AVPlayer(url: videoURL))
            .ignoresSafeArea()
            .onAppear {
                // 再生開始時にオーディオセッションを設定
                try? AVAudioSession.sharedInstance().setCategory(.playback)
                try? AVAudioSession.sharedInstance().setActive(true)
            }
    }
}

// 使用例:
// VideoPlayerView(
//     videoURL: URL(string: "https://example.com/video.mp4")!
// )

// 期待する出力:
// フルスクリーンでビデオが再生され、
// Siri Remote のクリックで再生/一時停止、
// 左右スワイプでシーク操作が可能
```

SwiftUI の `VideoPlayer` ビューを使えば、数行のコードでリッチなメディア再生体験を実装できます。Rork Max のプロンプトで「動画再生機能を含めてください」と指示すれば、こうした実装が自動生成されます。

## Top Shelf 拡張でホーム画面をリッチに

Apple TV のホーム画面上部にある「Top Shelf」は、アプリのコンテンツを他のアプリと並んでプレビュー表示できる特別な領域です。Top Shelf 拡張を実装することで、ユーザーがアプリを開く前からコンテンツを訴求できます。

```swift
import TVServices

class ContentProvider: TVTopShelfContentProvider {
    override func loadTopShelfContent() async -> TVTopShelfContent? {
        // セクション形式のTop Shelf
        let items: [TVTopShelfSectionedItem] = [
            createItem(
                id: "1",
                title: "今週のおすすめ",
                imageURL: "https://example.com/featured1.jpg"
            ),
            createItem(
                id: "2",
                title: "新着コンテンツ",
                imageURL: "https://example.com/featured2.jpg"
            ),
        ]

        let section = TVTopShelfItemCollection(items: items)
        section.title = "注目のコンテンツ"

        return TVTopShelfSectionedContent(sections: [section])
    }

    private func createItem(
        id: String,
        title: String,
        imageURL: String
    ) -> TVTopShelfSectionedItem {
        let item = TVTopShelfSectionedItem(identifier: id)
        item.title = title
        if let url = URL(string: imageURL) {
            item.setImageURL(url, for: .screenScale1x)
        }
        return item
    }
}

// 期待する出力:
// Apple TV ホーム画面でアプリにフォーカスすると、
// 画面上部の Top Shelf にコンテンツのプレビューが表示される
```

Top Shelf はアプリの発見可能性を高めるための重要な要素です。定期的にコンテンツを更新することで、ユーザーのエンゲージメントを維持できます。

## tvOS アプリの App Store 公開手順

Rork Max の **2クリック App Store 公開** 機能は tvOS アプリにも対応しています。公開までの流れは以下のとおりです。

1. **Rork Max でビルド**: プロンプトでアプリを生成し、クラウド Mac 上でコンパイルが完了することを確認します
2. **App Store Connect の準備**: tvOS アプリ用のアプリ情報（スクリーンショット、説明文、カテゴリなど）を登録します。tvOS のスクリーンショットは 1920×1080 または 3840×2160 の解像度が必要です
3. **TestFlight でベータテスト**: Rork Max の TestFlight 連携を使って、実機の Apple TV でテストを行います。リモコン操作の快適さやフォーカス遷移の自然さを実機で確認するのが大切です
4. **審査提出**: App Store Connect からtvOS アプリとして審査に提出します

tvOS アプリの審査では、フォーカスベースのナビゲーションが適切に機能しているか、リモコンのみで全ての操作が完了できるかが重点的にチェックされます。タッチ操作を前提とした UI が残っていると審査でリジェクトされる可能性があるため注意してください。

より詳しい App Store 公開のワークフローについては、「[Rork Max の2クリック App Store 公開を完全解説](/articles/rork-business/rork-max-2-click-publish-complete-guide)」で手順を詳しく解説しています。

## よくある質問（FAQ）

### Q: Rork Max で tvOS アプリを作るのに追加料金はかかりますか？

A: Rork Max のプラン（$200/月）に tvOS を含むすべての Apple プラットフォーム向けビルドが含まれています。tvOS 用の追加料金はありません。無料プラン（Free）では週5回までのビルドが可能ですが、まずは無料枠で tvOS アプリの開発を試してみることをおすすめします。

### Q: iOS アプリを tvOS にそのまま移植できますか？

A: SwiftUI で構築された iOS アプリであれば、コードの多くは共通化できます。ただし、タッチ操作を前提としたジェスチャー（ピンチ、ドラッグなど）は tvOS では動作しないため、Focus Engine に対応した UI への変更が必要です。Rork Max では「この iOS アプリを tvOS 対応にしてください」とプロンプトで指示することで、必要な変更を AI が提案・実装してくれます。

### Q: Apple TV 実機がなくてもテストできますか？

A: Rork Max のクラウド環境にはブラウザシミュレータが搭載されており、基本的な画面遷移やレイアウトの確認は可能です。ただし、Siri Remote の操作感やフォーカス遷移の快適さを正確に評価するには、TestFlight を通じた実機テストを強くおすすめします。Apple TV（第4世代以降）があれば TestFlight アプリからベータ版をインストールできます。

Rork Max を活用することで、Apple TV（tvOS）アプリの開発ハードルは大きく下がります。Focus Engine への対応やメディアプレイヤーの実装、Top Shelf 拡張など、tvOS 固有の要素も SwiftUI と Rork Max の AI コード生成を組み合わせることで効率的に実装できます。
