## Rork Max地図・位置情報ガイド

現代のモバイルアプリケーションにおいて、地図機能と位置情報の活用は必須となっています。配車サービス、飲食店検索、フィットネストラッキング、配達管理など、様々なアプリケーションで位置情報は重要な役割を果たしています。

本ガイドでは、Rork Maxを使用して地図表示、ジオロケーション、マーカー配置、ルート検索を実装する方法を詳細に解説します。

## 地図表示の基本実装

### MapView のセットアップ

Rork Max では `react-native-maps` を使用して地図を表示できます。初期セットアップでは、API キー設定が必須です。

```jsx
import MapView, { Marker } from 'react-native-maps';
import { useState, useEffect } from 'react';

export function BasicMapScreen() {
  const [region, setRegion] = useState({
    latitude: 35.6762,      // 東京
    longitude: 139.6503,
    latitudeDelta: 0.0922,
    longitudeDelta: 0.0421,
  });

  const [markers, setMarkers] = useState([
    {
      id: 1,
      latitude: 35.6762,
      longitude: 139.6503,
      title: '現在地',
      description: '東京駅',
    },
  ]);

  return (
    <MapView
      style={{ flex: 1 }}
      region={region}
      onRegionChange={setRegion}
    >
      {markers.map((marker) => (
        <Marker
          key={marker.id}
          coordinate={{
            latitude: marker.latitude,
            longitude: marker.longitude,
          }}
          title={marker.title}
          description={marker.description}
        />
      ))}
    </MapView>
  );
}
```

このコード例では、東京駅を中心とした基本的な地図を表示しています。ユーザーの操作に応じて地図領域が動的に更新されます。

## ジオロケーション（位置情報取得）

### ユーザーの現在位置を取得

デバイスのGPS機能を活用して、ユーザーの現在位置をリアルタイムで取得します。

```jsx
import Geolocation from '@react-native-geolocation/geolocation';
import { useState, useEffect } from 'react';

export function CurrentLocationScreen() {
  const [currentLocation, setCurrentLocation] = useState(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    // 位置情報のリアルタイム追跡
    const watchId = Geolocation.watchPosition(
      (position) => {
        const { latitude, longitude, accuracy } = position.coords;
        setCurrentLocation({
          latitude,
          longitude,
          accuracy,
        });
        console.log(
          `現在位置: 緯度=${latitude.toFixed(4)}, 経度=${longitude.toFixed(4)}`
        );
        // 期待出力: 現在位置: 緯度=35.6762, 経度=139.6503
        setIsLoading(false);
      },
      (error) => {
        console.error('位置情報取得エラー:', error);
        setIsLoading(false);
      },
      {
        enableHighAccuracy: true,
        timeout: 15000,
        maximumAge: 10000,
      }
    );

    // クリーンアップ
    return () => Geolocation.clearWatch(watchId);
  }, []);

  if (isLoading) return <p>位置情報を取得中...</p>;

  return (
    <div>
      {currentLocation && (
        <p>
          現在地: {currentLocation.latitude.toFixed(4)},
          {currentLocation.longitude.toFixed(4)}
          (精度: {currentLocation.accuracy.toFixed(0)}m)
        </p>
      )}
    </div>
  );
}
```

`enableHighAccuracy` を `true` に設定することで、より正確な位置情報が得られます。ただし、バッテリー消費が増加するため、必要に応じて調整してください。

## マーカーと情報ウィンドウ

### 複数の目的地をマーカーで表示

アプリケーション内で複数の拠点や目的地を表示する際、マーカーを活用します。

```jsx
import MapView, { Marker, Callout } from 'react-native-maps';
import { useState } from 'react';

export function MultiMarkerMap() {
  const [selectedMarkerId, setSelectedMarkerId] = useState(null);

  const locations = [
    {
      id: 1,
      name: 'カフェA',
      latitude: 35.6775,
      longitude: 139.7505,
      category: 'cafe',
    },
    {
      id: 2,
      name: 'レストランB',
      latitude: 35.6812,
      longitude: 139.7671,
      category: 'restaurant',
    },
    {
      id: 3,
      name: '駅C',
      latitude: 35.6895,
      longitude: 139.7004,
      category: 'station',
    },
  ];

  const handleMarkerPress = (markerId) => {
    setSelectedMarkerId(markerId);
    console.log(`マーカー${markerId}が選択されました`);
    // 期待出力: マーカー1が選択されました
  };

  return (
    <MapView style={{ flex: 1 }}>
      {locations.map((location) => (
        <Marker
          key={location.id}
          coordinate={{
            latitude: location.latitude,
            longitude: location.longitude,
          }}
          onPress={() => handleMarkerPress(location.id)}
          pinColor={
            selectedMarkerId === location.id ? '#FF6B6B' : '#4ECDC4'
          }
        >
          <Callout>
            <div style={{ padding: '10px' }}>
              <p style={{ fontWeight: 'bold' }}>{location.name}</p>
              <p>カテゴリ: {location.category}</p>
            </div>
          </Callout>
        </Marker>
      ))}
    </MapView>
  );
}
```

マーカーの色をカスタマイズすることで、カテゴリー別の視覚的な区別が容易になります。

## ルート検索と方向ナビゲーション

### 2地点間のルート計算

Google Maps Directions API を利用することで、2地点間の最短ルートを計算できます。

```jsx
import MapView, { Polyline, Marker } from 'react-native-maps';
import { useState } from 'react';

export async function getDirections(origin, destination) {
  try {
    const GOOGLE_MAPS_API_KEY = 'YOUR_API_KEY_HERE';
    const response = await fetch(
      `https://maps.googleapis.com/maps/api/directions/json?` +
        `origin=${origin.latitude},${origin.longitude}&` +
        `destination=${destination.latitude},${destination.longitude}&` +
        `key=${GOOGLE_MAPS_API_KEY}`
    );

    const data = await response.json();
    if (data.routes.length > 0) {
      const route = data.routes[0];
      const distance = route.legs[0].distance.text;
      const duration = route.legs[0].duration.text;
      console.log(`ルート情報: ${distance}, 所要時間: ${duration}`);
      // 期待出力: ルート情報: 5.2 km, 所要時間: 12 mins
      return route;
    }
  } catch (error) {
    console.error('ルート取得エラー:', error);
  }
}

export function DirectionMapScreen() {
  const [routeCoordinates, setRouteCoordinates] = useState([]);

  const displayRoute = async () => {
    const origin = { latitude: 35.6762, longitude: 139.6503 };
    const destination = { latitude: 35.6895, longitude: 139.7004 };

    const route = await getDirections(origin, destination);
    if (route) {
      const points = route.overview_polyline.points;
      const decodedPoints = decodePolyline(points);
      setRouteCoordinates(decodedPoints);
    }
  };

  return (
    <MapView style={{ flex: 1 }}>
      {routeCoordinates.length > 0 && (
        <Polyline
          coordinates={routeCoordinates}
          strokeColor="#FF6B6B"
          strokeWidth={3}
        />
      )}
      <button onClick={displayRoute}>ルートを表示</button>
    </MapView>
  );
}

function decodePolyline(encoded) {
  // ポリラインをデコード
  let points = [];
  let index = 0, lat = 0, lng = 0;

  while (index < encoded.length) {
    let result = 0, shift = 0;
    let b;

    do {
      b = encoded.charCodeAt(index++) - 63;
      result |= (b & 0x1f) << shift;
      shift += 5;
    } while (b >= 0x20);

    const dlat = result & 1 ? ~(result >> 1) : result >> 1;
    lat += dlat;

    result = 0;
    shift = 0;
    do {
      b = encoded.charCodeAt(index++) - 63;
      result |= (b & 0x1f) << shift;
      shift += 5;
    } while (b >= 0x20);

    const dlng = result & 1 ? ~(result >> 1) : result >> 1;
    lng += dlng;

    points.push({
      latitude: lat / 1e5,
      longitude: lng / 1e5,
    });
  }

  return points;
}
```

## パーミッションとプライバシー

### 位置情報パーミッションの管理

iOS 13 以降では、「常に許可」「使用中のみ許可」「許可しない」の 3 段階で位置情報アクセスを制御できます。

```jsx
import { PermissionsAndroid } from 'react-native';

export async function requestLocationPermission() {
  try {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.ACCESS_FINE_LOCATION,
      {
        title: '位置情報アクセス',
        message: 'このアプリは位置情報へのアクセスが必要です',
        buttonNeutral: 'あとで',
        buttonNegative: 'キャンセル',
        buttonPositive: '許可',
      }
    );

    if (granted === PermissionsAndroid.RESULTS.GRANTED) {
      console.log('位置情報パーミッションが許可されました');
      // 期待出力: 位置情報パーミッションが許可されました
      return true;
    }
  } catch (error) {
    console.error('パーミッション要求エラー:', error);
  }
  return false;
}
```

ユーザーのプライバシー保護は信頼醸成の基本です。位置情報使用の明確な目的を説明しましょう。

## FAQ

### Q1: バッテリー消費を抑えながら位置情報を追跡したい

**A:** `watchPosition` の `maximumAge` オプションを調整します。キャッシュ値（10秒など）を活用することで、不要な GPS クエリを削減できます。また、ユーザーが画面を離れた際は `clearWatch` で追跡を停止するべきです。

### Q2: オフライン環境で地図を表示できますか？

**A:** オフライン地図表示には、事前にタイルをダウンロードする手法（`react-native-maps-offline`など）が必要です。オンライン接続が可能になったら、キャッシュを更新することをお勧めします。

### Q3: 複数マーカー表示時のパフォーマンス最適化は？

**A:** マーカー数が 100 を超える場合、クラスタリング（近い位置のマーカーをグループ化）を導入してください。`react-native-maps-clustering`などのライブラリが有用です。
