## はじめに — Lottie がRorkアプリを変える理由

アプリの第一印象を決めるのは、機能よりも「見た目と動き」です。流れるようなローディングアニメーション、歓迎感のあるオンボーディング画面、決済完了時の達成感あふれる演出——こうした細部のクオリティが、ユーザーの継続率やレビューに直結します。

**Lottie（ロッティ）** は Airbnb が開発したオープンソースのアニメーションライブラリです。Adobe After Effects で制作したアニメーションを JSON 形式で書き出し、iOS・Android・Web でそのまま再生できます。React Native 向けの `lottie-react-native` パッケージを使えば、Rork で構築したアプリにも簡単に組み込めます。

この記事では、Rork アプリに Lottie を導入する手順を基礎から解説します。「アニメーションにこだわりたいけど、複雑なコードは書きたくない」という方にも、ステップバイステップで進められる内容になっています。

この記事で学べること：

- `lottie-react-native` のインストールと基本設定
- JSON アニメーションファイルの取得・配置方法
- ローディング・オンボーディング・成功演出の実装
- アニメーションの速度・ループ・トリガーの制御
- よくあるエラーと対処法

## Lottie とは？ — JSON で動くベクターアニメーション

Lottie は、Adobe After Effects のアニメーションを `Bodymovin` プラグインでエクスポートした JSON ファイルを、ネイティブコードで再生するライブラリです。PNG スプライトや GIF とは異なり、ベクターベースのため**どんな画面サイズでもくっきり表示**されます。

主なメリットをまとめると以下の通りです。

- **軽量**: 同等の GIF に比べてファイルサイズが 90% 以上小さいケースも多い
- **スケーラブル**: ベクター描画のため Retina ディスプレイでも劣化しない
- **インタラクティブ**: 再生速度・ループ・逆再生・特定フレームへのシークが自由
- **豊富な素材**: [LottieFiles](https://lottiefiles.com/) に無料・有料の素材が数万点

React Native 環境の Rork アプリでは `lottie-react-native` v7（Expo SDK 50 以降に対応）を使います。

## 環境準備 — lottie-react-native のインストール

Rork のプロジェクトは Expo Managed Workflow をベースにしているため、以下の手順でインストールできます。Rork の AI チャットに次のプロンプトを送るだけで、多くの場合は自動的に設定してくれます。

```
lottie-react-native を使ってローディングアニメーションを表示するコンポーネントを追加してください。
```

手動でインストールする場合は、プロジェクトのターミナルで以下を実行します。

```bash
# lottie-react-native と依存パッケージをインストール
npx expo install lottie-react-native
```

Expo SDK 50 以降では、`lottie-react-native` v7 以降が推奨です。`package.json` に以下のように追加されます。

```json
{
  "dependencies": {
    "lottie-react-native": "^7.0.0"
  }
}
```

インストール後、開発サーバーを再起動してください（`npx expo start --clear`）。

## Lottie ファイルの取得と配置

アニメーションファイル（`.json`）を用意します。無料素材を手軽に入手できる方法として、[LottieFiles](https://lottiefiles.com/) が最もおすすめです。

アカウント登録（無料）後に好みのアニメーションを検索し、「Lottie JSON」形式でダウンロードします。例として「loading」「success」「onboarding」などのキーワードで検索すると、クオリティの高い素材が見つかります。

ダウンロードした `.json` ファイルは、プロジェクトの `assets/animations/` フォルダに配置します。

```
my-app/
├── assets/
│   └── animations/
│       ├── loading.json    ← ローディング用
│       ├── success.json    ← 成功演出用
│       └── onboarding.json ← オンボーディング用
├── components/
└── app/
```

## 基本実装 — アニメーションを画面に表示する

まず最もシンプルな実装から始めます。`LottieView` コンポーネントを使ってアニメーションを表示するには、以下のようなコードになります。

```tsx
import LottieView from "lottie-react-native";
import { View, StyleSheet } from "react-native";

// シンプルなローディングアニメーションコンポーネント
export default function LoadingScreen() {
  return (
    <View style={styles.container}>
      <LottieView
        source={require("../assets/animations/loading.json")} // ← アニメーションファイルのパス
        autoPlay  // 自動再生
        loop      // ループ再生
        style={styles.animation}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: "center",
    alignItems: "center",
    backgroundColor: "#ffffff",
  },
  animation: {
    width: 200,  // アニメーションの幅（px）
    height: 200, // アニメーションの高さ（px）
  },
});
```

`autoPlay` と `loop` を指定するだけで、アニメーションが自動的に繰り返し再生されます。API データの読み込み中に表示するスピナーとして、すぐに活用できます。

## 実践：3つのユースケース実装

### ユースケース1 — データ読み込み中のローディング表示

非同期処理の待機中にアニメーションを表示し、読み込み完了後にコンテンツへ切り替えるパターンです。

```tsx
import React, { useState, useEffect } from "react";
import LottieView from "lottie-react-native";
import { View, Text, StyleSheet } from "react-native";

export default function DataScreen() {
  const [isLoading, setIsLoading] = useState(true);
  const [data, setData] = useState(null);

  useEffect(() => {
    // データ取得のシミュレーション（実際はAPIコールに置き換え）
    setTimeout(() => {
      setData({ message: "データの読み込みが完了しました！" });
      setIsLoading(false); // ← ローディング終了でアニメーションを消す
    }, 2000);
  }, []);

  if (isLoading) {
    return (
      <View style={styles.center}>
        <LottieView
          source={require("../assets/animations/loading.json")}
          autoPlay
          loop
          style={styles.lottie}
        />
        <Text style={styles.text}>データを読み込んでいます...</Text>
      </View>
    );
  }

  return (
    <View style={styles.center}>
      <Text>{data?.message}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  center: { flex: 1, justifyContent: "center", alignItems: "center" },
  lottie: { width: 150, height: 150 },
  text: { marginTop: 12, color: "#666" },
});
```

### ユースケース2 — 成功・完了時の達成感演出

決済完了・タスク完了・ゴール達成など「おめでとう」シーンに使える演出です。一度だけ再生して、完了後にコールバックを実行できます。

```tsx
import React, { useRef } from "react";
import LottieView from "lottie-react-native";
import { View, Text, Button, StyleSheet } from "react-native";

export default function SuccessScreen({ onDone }: { onDone: () => void }) {
  const animationRef = useRef<LottieView>(null);

  return (
    <View style={styles.container}>
      <LottieView
        ref={animationRef}
        source={require("../assets/animations/success.json")}
        autoPlay
        loop={false} // ← 1回だけ再生（ループしない）
        onAnimationFinish={onDone} // ← 再生完了後に onDone を呼ぶ
        style={styles.lottie}
      />
      <Text style={styles.title}>購入完了！</Text>
      <Text style={styles.subtitle}>ありがとうございました</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: "center", alignItems: "center", padding: 24 },
  lottie: { width: 200, height: 200 },
  title: { fontSize: 24, fontWeight: "bold", marginTop: 16 },
  subtitle: { fontSize: 16, color: "#666", marginTop: 8 },
});
```

### ユースケース3 — オンボーディング画面での世界観演出

アプリの初回起動時にブランドのトーンを伝えるオンボーディング画面です。スワイプに合わせてアニメーションを切り替えることで、没入感のある体験を作れます。

```tsx
import React, { useState } from "react";
import LottieView from "lottie-react-native";
import { View, Text, TouchableOpacity, StyleSheet, FlatList } from "react-native";

// オンボーディングのデータ（スライドごとに異なるアニメーション）
const slides = [
  {
    id: "1",
    animation: require("../assets/animations/slide1.json"),
    title: "AIがアプリを自動生成",
    subtitle: "アイデアを伝えるだけで、Rork がコードを書いてくれます",
  },
  {
    id: "2",
    animation: require("../assets/animations/slide2.json"),
    title: "2クリックでApp Store へ",
    subtitle: "Xcode 不要。クラウド Mac が自動でビルド・申請を行います",
  },
];

export default function OnboardingScreen({ onFinish }: { onFinish: () => void }) {
  const [currentIndex, setCurrentIndex] = useState(0);
  const current = slides[currentIndex];

  return (
    <View style={styles.container}>
      <LottieView
        key={current.id} // ← スライド切替時に再マウントして再生
        source={current.animation}
        autoPlay
        loop
        style={styles.lottie}
      />
      <Text style={styles.title}>{current.title}</Text>
      <Text style={styles.subtitle}>{current.subtitle}</Text>
      <TouchableOpacity
        style={styles.button}
        onPress={() => {
          if (currentIndex < slides.length - 1) {
            setCurrentIndex(currentIndex + 1);
          } else {
            onFinish(); // ← 最後のスライドで完了
          }
        }}
      >
        <Text style={styles.buttonText}>
          {currentIndex < slides.length - 1 ? "次へ" : "はじめる"}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: "center", alignItems: "center", padding: 32 },
  lottie: { width: 280, height: 280 },
  title: { fontSize: 22, fontWeight: "bold", textAlign: "center", marginTop: 24 },
  subtitle: { fontSize: 15, color: "#555", textAlign: "center", marginTop: 12, lineHeight: 22 },
  button: { marginTop: 32, backgroundColor: "#6C63FF", paddingVertical: 14, paddingHorizontal: 48, borderRadius: 30 },
  buttonText: { color: "#fff", fontSize: 16, fontWeight: "bold" },
});
```

React Native でのアニメーション基礎を押さえたい方は、[Rork Max でアニメーションを実装する — React Native Animated 活用ガイド](/articles/rork-dev/rork-animations-guide)も参考になります。

## よくあるエラーと対処法

Lottie を導入する際に遭遇しやすいエラーをまとめます。

**エラー1：アニメーションが表示されない（真っ白のまま）**

原因として最も多いのは、`style` に `width` と `height` が指定されていないケースです。Lottie は自動でサイズを設定しないため、必ず明示してください。

```tsx
// ❌ NG：サイズ未指定
<LottieView source={...} autoPlay loop />

// ✅ OK：サイズ指定あり
<LottieView source={...} autoPlay loop style={{ width: 200, height: 200 }} />
```

**エラー2：`Unable to resolve module 'lottie-react-native'`**

開発サーバーのキャッシュが残っている可能性があります。以下を試してください。

```bash
npx expo start --clear
```

それでも解決しない場合は、`node_modules` を再インストールします。

**エラー3：アニメーションが途中で止まる**

JSON ファイルが壊れているか、After Effects の機能を使った複雑なエフェクトが含まれている可能性があります。LottieFiles でプレビューできることを確認してから使用しましょう。

**エラー4：iOS と Android でアニメーションの見た目が違う**

一部の After Effects エフェクト（ブラー、グロウなど）は iOS/Android で描画が異なります。シンプルなベクターアニメーションを選ぶか、各プラットフォームでテストしながら調整してください。

## 応用 — Lottie × Reanimated で高度なインタラクションを作る

Lottie 単体でも十分な表現が可能ですが、`react-native-reanimated` と組み合わせることで、スクロール連動やジェスチャーとアニメーションを同期させる「インタラクティブなアニメーション」が実現できます。

例えば、スクロール量に応じて Lottie アニメーションの進行フレームを変化させる実装は以下のようなイメージです。

```tsx
import LottieView from "lottie-react-native";
import Animated, { useSharedValue, useAnimatedStyle, interpolate } from "react-native-reanimated";

// スクロール位置（0〜300px）をアニメーションの進行（0〜1）にマッピング
const AnimatedLottie = Animated.createAnimatedComponent(LottieView);

function ScrollDrivenAnimation({ scrollY }: { scrollY: Animated.SharedValue<number> }) {
  const progress = interpolate(scrollY.value, [0, 300], [0, 1]);

  return (
    <AnimatedLottie
      source={require("../assets/animations/reveal.json")}
      progress={progress} // ← スクロールに応じてフレームが変化
      style={{ width: 200, height: 200 }}
    />
  );
}
```

より高度なジェスチャー連動アニメーションの実装については、[Rork × React Native Reanimated & Gesture Handler 実践ガイド](/articles/rork-dev/rork-reanimated-gesture-handler-advanced-animation)で詳しく解説しています。60fps を維持しながら滑らかな動きを実現するテクニックを網羅していますので、アニメーションの品質をさらに高めたい方はぜひご覧ください。

## FAQ

**Q1. 無料で使える Lottie アニメーション素材はどこで入手できますか？**

[LottieFiles](https://lottiefiles.com/) が最も充実しています。無料プランでも数千点以上の素材をダウンロードでき、プレビューして気に入ったものを JSON 形式で取得できます。商用利用可能なライセンスの素材も多数あるため、ライセンス表記を確認してから使用しましょう。

**Q2. アニメーションのファイルサイズはどのくらいにすべきですか？**

一般的には 1 ファイルあたり **50KB 以下** を目安にするとよいでしょう。複雑なエフェクトが含まれるファイルは数 MB になることもありますが、起動時間やメモリに影響するため、シンプルなアニメーションを選ぶのが基本です。LottieFiles のプレビュー画面でファイルサイズを確認できます。

**Q3. Rork の AI に「Lottie を使って〇〇を作って」と伝えるとどうなりますか？**

Rork の AI は `lottie-react-native` の使い方を理解しているため、「ローディング画面に Lottie アニメーションを追加して」「決済完了時に紙吹雪のアニメーションを表示して」といったプロンプトで実装コードを生成してくれます。アニメーション JSON ファイルの配置場所を伝えると、より精度の高いコードが得られます。

**Q4. iOS と Android 両方でテストする必要がありますか？**

はい、特にアニメーションの複雑度が高い場合はプラットフォームごとの差異が出ることがあります。Rork の Expo Go を使って実機確認することをおすすめします。また、[lottie-react-native の公式 GitHub](https://github.com/lottie-react-native/lottie-react-native) には既知の互換性情報がまとまっています。
