## なぜ Core Data と CloudKit なのか

モバイルアプリを開発するとき、ユーザーが入力したデータをどこに保存するかは設計上の重要な判断です。Rork Max はネイティブ Swift アプリを生成できるため、Apple が提供する **Core Data**（ローカル永続化）と **CloudKit**（iCloud クラウド同期）を組み合わせることで、オフラインでも動作しつつ複数デバイス間でデータを自動同期する体験を実現できます。

サードパーティの BaaS（Supabase や Firebase）と比較した場合のメリットは、追加のサーバーコストが不要で Apple エコシステムとの親和性が高い点です。すでに iCloud アカウントを持っているユーザーであれば、サインアップなしにシームレスな同期が可能になります。

## Core Data の基本構造

Core Data は Apple が提供するオブジェクトグラフ管理フレームワークです。SQLite をバックエンドとして利用しつつ、Swift のオブジェクトとしてデータを扱えるため、直接 SQL を書く必要はありません。

### NSPersistentContainer のセットアップ

Rork Max で生成したプロジェクトに Core Data を追加する際、最初に設定するのが `NSPersistentContainer` です。

```swift
import CoreData

class PersistenceController {
    static let shared = PersistenceController()

    let container: NSPersistentContainer

    init() {
        container = NSPersistentContainer(name: "AppModel")
        container.loadPersistentStores { description, error in
            if let error = error {
                fatalError("Core Data の読み込みに失敗: \(error)")
            }
        }
        container.viewContext.automaticallyMergesChangesFromParent = true
        container.viewContext.mergePolicy = NSMergeByPropertyObjectTrumpMergePolicy
    }
}
```

`automaticallyMergesChangesFromParent` を `true` にすることで、バックグラウンドコンテキストでの変更が自動的にビューコンテキストに反映されます。`mergePolicy` はコンフリクト発生時の挙動を決めるもので、CloudKit 連携時には `NSMergeByPropertyObjectTrumpMergePolicy` が推奨されます。

### データモデルの定義

Xcode の `.xcdatamodeld` エディタでエンティティを定義しますが、Rork Max ではプロンプトで指定できます。

```
タスク管理アプリを作成してください。
Core Data でデータを永続化し、以下のエンティティを持たせてください：
- Task: title(String), isCompleted(Bool), createdAt(Date), priority(Int16)
- Category: name(String), color(String)
- Task と Category は多対1のリレーション
```

このようにプロンプトで構造を明示すると、Rork Max が `.xcdatamodeld` ファイルとそれに対応する Swift コードを自動生成します。

## CloudKit との統合

### NSPersistentCloudKitContainer への切り替え

Core Data を CloudKit と統合する場合、`NSPersistentContainer` を `NSPersistentCloudKitContainer` に置き換えるだけで基本的な同期が動作します。

```swift
import CoreData

class PersistenceController {
    static let shared = PersistenceController()

    let container: NSPersistentCloudKitContainer

    init() {
        container = NSPersistentCloudKitContainer(name: "AppModel")

        guard let description = container.persistentStoreDescriptions.first else {
            fatalError("Store description が見つかりません")
        }
        description.setOption(true as NSNumber,
                              forKey: NSPersistentHistoryTrackingKey)
        description.setOption(true as NSNumber,
                              forKey: NSPersistentStoreRemoteChangeNotificationPostOptionKey)

        container.loadPersistentStores { description, error in
            if let error = error {
                fatalError("CloudKit Store の読み込みに失敗: \(error)")
            }
        }
        container.viewContext.automaticallyMergesChangesFromParent = true
        container.viewContext.mergePolicy = NSMergeByPropertyObjectTrumpMergePolicy
    }
}
```

`NSPersistentHistoryTrackingKey` と `NSPersistentStoreRemoteChangeNotificationPostOptionKey` の2つのオプションは、CloudKit との同期に必須です。これらを有効にすることで、リモートからの変更通知を受け取り、差分同期が正しく動作します。

### Rork Max での CloudKit 設定プロンプト

```
このタスク管理アプリに iCloud 同期を追加してください。
NSPersistentCloudKitContainer を使い、
iCloud コンテナ ID は "iCloud.com.example.taskapp" にしてください。
オフラインで編集した内容がオンライン復帰時に自動同期されるようにしてください。
```

Rork Max は Capabilities に iCloud（CloudKit）を追加し、コンテナ ID を設定したプロジェクトを生成します。

## SwiftUI での CRUD 操作

### @FetchRequest を使ったデータ表示

SwiftUI では `@FetchRequest` プロパティラッパーを使うことで、Core Data の変更をリアクティブに UI に反映できます。

```swift
struct TaskListView: View {
    @Environment(\.managedObjectContext) private var viewContext

    @FetchRequest(
        sortDescriptors: [NSSortDescriptor(keyPath: \Task.createdAt, ascending: false)],
        animation: .default
    )
    private var tasks: FetchedResults<Task>

    var body: some View {
        List {
            ForEach(tasks) { task in
                TaskRowView(task: task)
            }
            .onDelete(perform: deleteTasks)
        }
    }

    private func deleteTasks(offsets: IndexSet) {
        withAnimation {
            offsets.map { tasks[$0] }.forEach(viewContext.delete)
            try? viewContext.save()
        }
    }
}
```

### データの作成と保存

```swift
private func addTask(title: String, priority: Int16) {
    let newTask = Task(context: viewContext)
    newTask.title = title
    newTask.isCompleted = false
    newTask.createdAt = Date()
    newTask.priority = priority

    do {
        try viewContext.save()
    } catch {
        print("保存エラー: \(error)")
    }
}
```

`viewContext.save()` を呼び出すと、ローカルの SQLite に保存されると同時に CloudKit への同期がバックグラウンドで開始されます。ユーザーが意識する必要はありません。

## コンフリクト解決とベストプラクティス

### マージポリシーの選択

複数デバイスで同時に同じレコードを編集した場合、コンフリクトが発生します。Core Data には複数のマージポリシーが用意されています。

| ポリシー | 動作 | 用途 |
|---------|------|------|
| `NSMergeByPropertyStoreTrumpMergePolicy` | ストア側（リモート）を優先 | サーバーデータが正とする場合 |
| `NSMergeByPropertyObjectTrumpMergePolicy` | メモリ側（ローカル）を優先 | ユーザー操作を優先する場合 |
| `NSRollbackMergePolicy` | メモリ側の変更を破棄 | 読み取り専用ビューなど |

多くのアプリでは `NSMergeByPropertyObjectTrumpMergePolicy` が適切です。ユーザーが最後に編集した内容を保持するため、直感的な動作になります。

### バックグラウンドコンテキストの活用

大量データの読み書きはメインスレッドをブロックする原因になります。バックグラウンドコンテキストを使って処理を分離しましょう。

```swift
func importData(_ items: [ImportItem]) {
    container.performBackgroundTask { context in
        for item in items {
            let task = Task(context: context)
            task.title = item.title
            task.createdAt = Date()
        }
        try? context.save()
    }
}
```

`performBackgroundTask` は専用のバックグラウンドコンテキストを生成し、処理完了後にメインコンテキストへ自動でマージします。

## トラブルシューティング

### 同期が始まらない場合

CloudKit 同期がうまく動作しない場合、以下の点を確認してください。

1. **iCloud にサインイン済みか**: 設定アプリで iCloud アカウントが有効であることを確認
2. **Capabilities の設定**: iCloud と Background Modes（Remote notifications）が有効か
3. **コンテナ ID の一致**: コードと CloudKit Dashboard のコンテナ ID が一致しているか
4. **ネットワーク環境**: シミュレータではなく実機でテストする（シミュレータの CloudKit は不安定な場合がある）

### データが消える問題

`NSPersistentHistoryTrackingKey` を有効にしていない場合、同期のたびにデータが消失するケースがあります。前述の設定を必ず含めてください。

## Rork Max プロンプトのまとめ

Core Data + CloudKit を活用するアプリを Rork Max で生成する際のプロンプト例をまとめます。

```
レシピ管理アプリを作成してください。

技術要件:
- Core Data でローカルにデータを保存
- NSPersistentCloudKitContainer で iCloud 同期を有効化
- iCloud コンテナ ID: iCloud.com.example.recipes
- オフライン対応（ネットワーク未接続でも操作可能）

データモデル:
- Recipe: title(String), instructions(String), cookingTime(Int16),
  isFavorite(Bool), createdAt(Date), imageData(Binary)
- Ingredient: name(String), amount(String)
- Recipe と Ingredient は1対多のリレーション

UI:
- SwiftUI で実装
- レシピ一覧（お気に入りフィルター付き）
- レシピ詳細画面（材料リスト表示）
- 新規追加・編集フォーム
- 検索機能（タイトルと材料名で検索）

その他:
- マージポリシーは NSMergeByPropertyObjectTrumpMergePolicy
- バックグラウンドコンテキストでのデータインポート対応
- 複数デバイス間でリアルタイム同期
```
