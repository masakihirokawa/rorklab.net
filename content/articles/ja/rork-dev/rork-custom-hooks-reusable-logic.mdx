## はじめに — なぜカスタムフックが重要なのか

Rork でアプリを開発していると、画面が増えるにつれて同じようなロジックを何度も書いている自分に気づくことはありませんか。API からデータを取得する処理、フォームの入力状態を管理する処理、デバイスの位置情報を取得する処理——これらはどの画面でも繰り返し登場します。

React のカスタムフックは、こうしたロジックを1つの関数にまとめて、どの画面からでも呼び出せるようにする仕組みです。Rork が生成する React Native コードでもこのパターンはそのまま活用でき、コードの重複を減らしながらアプリ全体の保守性を大きく向上させることができます。

この記事では、Rork アプリ開発で実際に役立つカスタムフックのパターンを、具体的なコード例とともに解説します。

## カスタムフックの基本構造

カスタムフックとは、`use` で始まる名前の関数で、内部で React の組み込みフック（`useState`、`useEffect` など）を呼び出すものです。通常のコンポーネントと同じルールに従いますが、JSX を返す代わりにデータや関数を返します。

```typescript
// hooks/useCounter.ts — 最もシンプルなカスタムフック
import { useState, useCallback } from 'react';

export function useCounter(initialValue: number = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = useCallback(() => setCount(prev => prev + 1), []);
  const decrement = useCallback(() => setCount(prev => prev - 1), []);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);

  return { count, increment, decrement, reset };
}

// 使用例:
// const { count, increment, decrement } = useCounter(0);
```

この基本構造を理解すれば、あとはどんなロジックでもフックとして切り出すことができます。

## 実践パターン1: API通信フック `useFetch`

Rork アプリでもっとも頻繁に必要になるのが、外部 API からデータを取得する処理です。ローディング状態、エラーハンドリング、再取得ロジックを毎回書くのは非効率なので、汎用フックにまとめましょう。

```typescript
// hooks/useFetch.ts
import { useState, useEffect, useCallback } from 'react';

interface UseFetchResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  refetch: () => void;
}

export function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchData = useCallback(async () => {
    setLoading(true);
    setError(null);
    try {
      const response = await fetch(url);
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      const json = await response.json();
      setData(json);
    } catch (err) {
      setError(err instanceof Error ? err.message : '通信エラーが発生しました');
    } finally {
      setLoading(false);
    }
  }, [url]);

  useEffect(() => {
    fetchData();
  }, [fetchData]);

  return { data, loading, error, refetch: fetchData };
}

// 使用例:
// const { data: users, loading, error, refetch } = useFetch<User[]>('https://api.example.com/users');
// if (loading) return <ActivityIndicator />;
// if (error) return <Text>エラー: {error}</Text>;
```

このフックひとつで、アプリ内のあらゆるデータ取得画面を統一的に扱えるようになります。

## 実践パターン2: フォーム管理フック `useForm`

ユーザー入力を扱うフォームは、バリデーションやエラー表示のロジックが複雑になりがちです。シンプルなフォーム管理フックを作っておくと、ログイン画面からプロフィール編集画面まで共通のパターンで実装できます。

```typescript
// hooks/useForm.ts
import { useState, useCallback } from 'react';

type ValidationRules<T> = {
  [K in keyof T]?: (value: T[K]) => string | null;
};

interface UseFormResult<T> {
  values: T;
  errors: Partial<Record<keyof T, string>>;
  handleChange: <K extends keyof T>(field: K, value: T[K]) => void;
  validate: () => boolean;
  reset: () => void;
}

export function useForm<T extends Record<string, any>>(
  initialValues: T,
  rules?: ValidationRules<T>
): UseFormResult<T> {
  const [values, setValues] = useState<T>(initialValues);
  const [errors, setErrors] = useState<Partial<Record<keyof T, string>>>({});

  const handleChange = useCallback(<K extends keyof T>(field: K, value: T[K]) => {
    setValues(prev => ({ ...prev, [field]: value }));
    // フィールド変更時にそのフィールドのエラーをクリア
    setErrors(prev => ({ ...prev, [field]: undefined }));
  }, []);

  const validate = useCallback(() => {
    if (!rules) return true;
    const newErrors: Partial<Record<keyof T, string>> = {};
    let isValid = true;

    for (const [field, rule] of Object.entries(rules)) {
      const error = (rule as Function)(values[field]);
      if (error) {
        newErrors[field as keyof T] = error;
        isValid = false;
      }
    }
    setErrors(newErrors);
    return isValid;
  }, [values, rules]);

  const reset = useCallback(() => {
    setValues(initialValues);
    setErrors({});
  }, [initialValues]);

  return { values, errors, handleChange, validate, reset };
}

// 使用例:
// const { values, errors, handleChange, validate } = useForm(
//   { email: '', password: '' },
//   {
//     email: (v) => !v.includes('@') ? 'メールアドレスが無効です' : null,
//     password: (v) => v.length < 8 ? 'パスワードは8文字以上です' : null,
//   }
// );
```

フォームバリデーションのより高度な実装については、[Rork アプリのフォームバリデーション完全ガイド](/articles/rork-dev/rork-form-validation-react-hook-form-zod)も参考になります。

## 実践パターン3: デバイス情報フック `useDeviceInfo`

モバイルアプリでは、デバイスの画面サイズやネットワーク状態に応じた処理が求められます。これもカスタムフックにまとめておくと、レスポンシブ対応が格段に楽になります。

```typescript
// hooks/useDeviceInfo.ts
import { useState, useEffect } from 'react';
import { Dimensions, Platform } from 'react-native';

interface DeviceInfo {
  width: number;
  height: number;
  isTablet: boolean;
  platform: 'ios' | 'android' | 'web';
}

export function useDeviceInfo(): DeviceInfo {
  const [dimensions, setDimensions] = useState(() => Dimensions.get('window'));

  useEffect(() => {
    const subscription = Dimensions.addEventListener('change', ({ window }) => {
      setDimensions(window);
    });
    return () => subscription.remove();
  }, []);

  return {
    width: dimensions.width,
    height: dimensions.height,
    isTablet: dimensions.width >= 768,
    platform: Platform.OS as 'ios' | 'android' | 'web',
  };
}

// 使用例:
// const { width, isTablet, platform } = useDeviceInfo();
// const columns = isTablet ? 3 : 2;
```

## 実践パターン4: 非同期処理フック `useAsync`

API 通信以外にも、AsyncStorage への保存、画像のリサイズ、ファイルの読み書きなど、モバイルアプリには非同期処理が数多くあります。汎用的な `useAsync` フックを用意しておくと、どんな非同期処理にも統一的なインターフェースで対応できます。

```typescript
// hooks/useAsync.ts
import { useState, useCallback } from 'react';

interface UseAsyncResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  execute: (...args: any[]) => Promise<T | null>;
}

export function useAsync<T>(asyncFn: (...args: any[]) => Promise<T>): UseAsyncResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const execute = useCallback(async (...args: any[]) => {
    setLoading(true);
    setError(null);
    try {
      const result = await asyncFn(...args);
      setData(result);
      return result;
    } catch (err) {
      const message = err instanceof Error ? err.message : '処理に失敗しました';
      setError(message);
      return null;
    } finally {
      setLoading(false);
    }
  }, [asyncFn]);

  return { data, loading, error, execute };
}

// 使用例:
// const { data: savedItem, loading, execute: saveItem } = useAsync(
//   async (key: string, value: string) => {
//     await AsyncStorage.setItem(key, value);
//     return { key, value, savedAt: new Date() };
//   }
// );
// await saveItem('user_prefs', JSON.stringify(prefs));
```

## カスタムフックを設計するときのコツ

カスタムフックを上手に設計するためのポイントをいくつか紹介します。

**1つのフックには1つの責務** — `useFetch` がフォームのバリデーションまで行うようなフックは避けましょう。小さく、明確な役割を持たせることで、テストしやすく、再利用しやすくなります。

**命名は具体的に** — `useData` よりも `useFetch`、`useUserProfile`、`useGeolocation` のように、何をするフックなのかが名前から伝わるようにします。

**TypeScript の型を活用する** — ジェネリクスを使うことで、フックの汎用性と型安全性を両立できます。上の `useFetch<T>` がその好例です。

**クリーンアップを忘れない** — `useEffect` の中でイベントリスナーやタイマーを登録した場合は、必ずクリーンアップ関数で解除します。メモリリークの原因になります。

状態管理のより高度なパターンについては、[Rork アプリの状態管理パターン完全ガイド](/articles/rork-dev/rork-advanced-state-management-patterns)で詳しく解説しています。

## よくある質問（FAQ）

### Q: Rork が自動生成したコードにカスタムフックを追加しても問題ありませんか？

はい、問題ありません。Rork が生成するコードは標準的な React Native + Expo の構成ですので、通常の React プロジェクトと同じように `hooks/` ディレクトリを作成してカスタムフックを追加できます。Rork Max を使ってプロジェクトをエクスポートした後に追加するのがもっとも確実な方法です。

### Q: カスタムフックとコンテキスト（Context）はどう使い分ければよいですか？

カスタムフックは「ロジックの再利用」に使います。一方、Context は「複数のコンポーネント間でデータを共有する」ために使います。たとえば `useFetch` はどの画面でも独立して使えるカスタムフックですが、ログイン中のユーザー情報をアプリ全体で共有するには Context が適しています。多くの場合、Context のプロバイダー内でカスタムフックを使う設計がベストです。

### Q: カスタムフックのテストはどう書けばよいですか？

`@testing-library/react-hooks`（React Native Testing Library）の `renderHook` を使ってテストできます。フック単体でテストできるのがカスタムフックの大きなメリットです。

```typescript
import { renderHook, act } from '@testing-library/react-hooks';
import { useCounter } from './useCounter';

test('increment increases count by 1', () => {
  const { result } = renderHook(() => useCounter(0));
  act(() => { result.current.increment(); });
  expect(result.current.count).toBe(1);
});
```

## まとめ

カスタムフックは、Rork アプリのコード品質を一段引き上げるための強力な手法です。今回紹介した4つのパターン——`useFetch`（API 通信）、`useForm`（フォーム管理）、`useDeviceInfo`（デバイス情報）、`useAsync`（汎用非同期処理）——をベースに、自分のアプリに合ったフックを作成してみてください。

初めての方はまず [Rork 入門ガイド](/articles/rork-basics/getting-started)でアプリの基本的な作り方を押さえてから、カスタムフックによるリファクタリングに取り組むとスムーズです。小さなフックから始めて、徐々にアプリ全体の設計を改善していきましょう。
