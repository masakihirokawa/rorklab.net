## はじめに

フリーランスや個人事業主にとって、請求書の作成・送付・管理は避けて通れない業務です。しかし、既存の請求書サービスは月額料金が高かったり、自分のワークフローに合わなかったりすることも少なくありません。

この記事では、Rork を使ってオリジナルのインボイス（請求書）アプリを一から構築する方法を解説します。クライアント管理、請求書の PDF 生成、メール送信、さらに売上ダッシュボードまでを備えた本格的なアプリを作りましょう。

**この記事で学べること:**

- Rork でデータ駆動型のビジネスアプリを設計する方法
- react-native-html-to-pdf を使った PDF 生成の実装
- Supabase をバックエンドに使ったクライアント・請求書データの管理
- 売上の可視化ダッシュボードの構築

## 前提知識と環境準備

## アプリの全体設計

まずはアプリの画面構成とデータモデルを整理しましょう。Rork に指示を出す前に全体像を把握しておくと、一貫性のある設計ができます。

### 画面構成

今回作成するアプリは以下の 5 画面で構成します。

1. **ダッシュボード** — 月別売上グラフ・未入金リスト・直近の請求書
2. **クライアント一覧** — 取引先の登録・編集・検索
3. **請求書一覧** — ステータス別フィルタリング（下書き・送信済み・入金済み）
4. **請求書作成/編集** — 明細行の追加・税率設定・合計自動計算
5. **請求書プレビュー** — PDF 表示・メール送信・共有

### データモデル

```sql
-- クライアントテーブル
CREATE TABLE clients (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,           -- 会社名・氏名
  email TEXT,                   -- メールアドレス
  address TEXT,                 -- 住所
  phone TEXT,                   -- 電話番号
  notes TEXT,                   -- メモ
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 請求書テーブル
CREATE TABLE invoices (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  invoice_number TEXT NOT NULL, -- 請求書番号（例: INV-2026-001）
  status TEXT DEFAULT 'draft',  -- draft / sent / paid / overdue
  issue_date DATE NOT NULL,     -- 発行日
  due_date DATE NOT NULL,       -- 支払期限
  tax_rate DECIMAL DEFAULT 0.10,-- 税率（デフォルト10%）
  notes TEXT,                   -- 備考
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 請求書明細テーブル
CREATE TABLE invoice_items (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  invoice_id UUID REFERENCES invoices(id) ON DELETE CASCADE,
  description TEXT NOT NULL,    -- 項目名
  quantity INTEGER DEFAULT 1,   -- 数量
  unit_price DECIMAL NOT NULL,  -- 単価
  sort_order INTEGER DEFAULT 0  -- 並び順
);
```

## PDF 生成の実装

請求書アプリの核となる PDF 生成機能を実装します。`expo-print` と `expo-sharing` を組み合わせて、HTML テンプレートから PDF を生成し、共有できるようにしましょう。

```typescript
// utils/generateInvoicePDF.ts
import * as Print from 'expo-print';
import * as Sharing from 'expo-sharing';

interface InvoiceItem {
  description: string;
  quantity: number;
  unit_price: number;
}

interface InvoiceData {
  invoiceNumber: string;
  clientName: string;
  clientAddress: string;
  issueDate: string;
  dueDate: string;
  items: InvoiceItem[];
  taxRate: number;
  notes?: string;
}

// 請求書のHTMLテンプレートを生成
const generateHTML = (data: InvoiceData): string => {
  const subtotal = data.items.reduce(
    (sum, item) => sum + item.quantity * item.unit_price, 0
  );
  const tax = subtotal * data.taxRate;
  const total = subtotal + tax;

  const itemRows = data.items
    .map(
      (item) => `
      <tr>
        <td>${item.description}</td>
        <td class="right">${item.quantity}</td>
        <td class="right">¥${item.unit_price.toLocaleString()}</td>
        <td class="right">¥${(item.quantity * item.unit_price).toLocaleString()}</td>
      </tr>`
    )
    .join('');

  return `
    <!DOCTYPE html>
    <html>
    <head>
      <meta charset="utf-8" />
      <style>
        body { font-family: 'Helvetica Neue', sans-serif; padding: 40px; color: #333; }
        .header { display: flex; justify-content: space-between; margin-bottom: 40px; }
        .invoice-title { font-size: 28px; color: #1a2332; font-weight: bold; }
        .invoice-number { color: #666; margin-top: 4px; }
        table { width: 100%; border-collapse: collapse; margin: 20px 0; }
        th { background: #1a2332; color: white; padding: 10px; text-align: left; }
        td { padding: 10px; border-bottom: 1px solid #eee; }
        .right { text-align: right; }
        .total-section { margin-top: 20px; text-align: right; }
        .total-row { font-size: 14px; margin: 4px 0; }
        .grand-total { font-size: 20px; font-weight: bold; color: #1a2332; }
        .notes { margin-top: 30px; padding: 15px; background: #f9f9f9; border-radius: 8px; }
      </style>
    </head>
    <body>
      <div class="header">
        <div>
          <div class="invoice-title">請求書</div>
          <div class="invoice-number">${data.invoiceNumber}</div>
        </div>
        <div style="text-align: right;">
          <div>発行日: ${data.issueDate}</div>
          <div>支払期限: ${data.dueDate}</div>
        </div>
      </div>

      <div style="margin-bottom: 30px;">
        <strong>${data.clientName} 御中</strong><br/>
        ${data.clientAddress}
      </div>

      <table>
        <thead>
          <tr>
            <th>項目</th>
            <th class="right">数量</th>
            <th class="right">単価</th>
            <th class="right">金額</th>
          </tr>
        </thead>
        <tbody>${itemRows}</tbody>
      </table>

      <div class="total-section">
        <div class="total-row">小計: ¥${subtotal.toLocaleString()}</div>
        <div class="total-row">消費税 (${(data.taxRate * 100).toFixed(0)}%): ¥${tax.toLocaleString()}</div>
        <div class="grand-total">合計: ¥${total.toLocaleString()}</div>
      </div>

      ${data.notes ? `<div class="notes"><strong>備考:</strong><br/>${data.notes}</div>` : ''}
    </body>
    </html>
  `;
};

// PDF を生成して共有する
export const generateAndSharePDF = async (data: InvoiceData): Promise<void> => {
  const html = generateHTML(data);

  // HTMLからPDFを生成
  const { uri } = await Print.printToFileAsync({
    html,
    base64: false,
  });

  // 共有ダイアログを表示（メール送信、AirDrop等）
  if (await Sharing.isAvailableAsync()) {
    await Sharing.shareAsync(uri, {
      mimeType: 'application/pdf',
      dialogTitle: `${data.invoiceNumber}.pdf`,
      UTI: 'com.adobe.pdf',
    });
  }
};
```

## Supabase でバックエンドを構築する

クライアント情報と請求書データを Supabase に保存し、複数デバイスから同期できるようにします。Rork との組み合わせでは、Supabase が最も手軽にリアルタイム同期を実現できる選択肢です。

### Supabase クライアントの初期化

```typescript
// lib/supabase.ts
import { createClient } from '@supabase/supabase-js';
import AsyncStorage from '@react-native-async-storage/async-storage';

const supabaseUrl = 'https://your-project.supabase.co';
const supabaseAnonKey = 'your-anon-key';

export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  auth: {
    storage: AsyncStorage, // React Native用のストレージアダプタ
    autoRefreshToken: true,
    persistSession: true,
  },

```

### 請求書の CRUD 操作

```typescript
// hooks/useInvoices.ts
import { useState, useEffect, useCallback } from 'react';
import { supabase } from '../lib/supabase';

export function useInvoices() {
  const [invoices, setInvoices] = useState([]);
  const [loading, setLoading] = useState(true);

  // 請求書一覧を取得（クライアント情報も結合）
  const fetchInvoices = useCallback(async () => {
    setLoading(true);
    const { data, error } = await supabase
      .from('invoices')
      .select(`
        *,
        client:clients(name, email),
        items:invoice_items(*)
      `)
      .order('created_at', { ascending: false });

    if (!error && data) {
      setInvoices(data);
    }
    setLoading(false);
  }, []);

  // 請求書を作成
  const createInvoice = async (invoice, items) => {
    // 請求書番号を自動生成（例: INV-2026-042）
    const year = new Date().getFullYear();
    const { count } = await supabase
      .from('invoices')
      .select('*', { count: 'exact', head: true });
    const number = `INV-${year}-${String((count || 0) + 1).padStart(3, '0')}`;

    const { data, error } = await supabase
      .from('invoices')
      .insert({ ...invoice, invoice_number: number })
      .select()
      .single();

    if (data && items.length > 0) {
      // 明細行を一括挿入
      await supabase.from('invoice_items').insert(
        items.map((item, i) => ({
          ...item,
          invoice_id: data.id,
          sort_order: i,
        }))
      );
    }

    await fetchInvoices(); // リストを更新
    return data;
  };

  // ステータスを更新（例: draft → sent）
  const updateStatus = async (id: string, status: string) => {
    await supabase.from('invoices').update({ status }).eq('id', id);
    await fetchInvoices();
  };

  useEffect(() => {
    fetchInvoices();
  }, [fetchInvoices]);

  return { invoices, loading, createInvoice, updateStatus, refresh: fetchInvoices };
}
```

Supabase との連携の詳しい解説は [Rork × Supabase 認証・リアルタイム連携ガイド](/articles/rork-dev/rork-supabase-auth-realtime-guide) も参考にしてください。

## 売上ダッシュボードの構築

アプリの付加価値を高めるために、月別の売上推移を可視化するダッシュボードを実装します。`react-native-chart-kit` を使って折れ線グラフとサマリーカードを表示しましょう。

```typescript
// components/SalesDashboard.tsx
import React, { useMemo } from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { LineChart } from 'react-native-chart-kit';
import { Dimensions } from 'react-native';

const screenWidth = Dimensions.get('window').width;

export default function SalesDashboard({ invoices }) {
  // 月別売上を集計
  const monthlyData = useMemo(() => {
    const months = {};
    invoices
      .filter((inv) => inv.status === 'paid')
      .forEach((inv) => {
        const month = inv.issue_date.slice(0, 7); // "2026-03"
        const total = inv.items?.reduce(
          (sum, item) => sum + item.quantity * item.unit_price, 0
        ) || 0;
        months[month] = (months[month] || 0) + total * (1 + inv.tax_rate);
      });

    const labels = Object.keys(months).slice(-6);
    const data = labels.map((m) => months[m]);
    return { labels: labels.map((l) => l.slice(5)), data }; // "03" 形式
  }, [invoices]);

  // サマリー計算
  const summary = useMemo(() => {
    const paid = invoices.filter((i) => i.status === 'paid');
    const unpaid = invoices.filter((i) => i.status === 'sent' || i.status === 'overdue');
    const calcTotal = (list) =>
      list.reduce((sum, inv) => {
        const subtotal = inv.items?.reduce(
          (s, item) => s + item.quantity * item.unit_price, 0
        ) || 0;
        return sum + subtotal * (1 + inv.tax_rate);
      }, 0);
    return {
      totalRevenue: calcTotal(paid),
      unpaidAmount: calcTotal(unpaid),
      unpaidCount: unpaid.length,
    };
  }, [invoices]);

  return (
    <View style={styles.container}>
      {/* サマリーカード */}
      <View style={styles.cards}>
        <View style={[styles.card, { backgroundColor: '#e8f5e9' }]}>
          <Text style={styles.cardLabel}>入金済み</Text>
          <Text style={styles.cardValue}>¥{summary.totalRevenue.toLocaleString()}</Text>
        </View>
        <View style={[styles.card, { backgroundColor: '#fff3e0' }]}>
          <Text style={styles.cardLabel}>未入金</Text>
          <Text style={styles.cardValue}>¥{summary.unpaidAmount.toLocaleString()}</Text>
          <Text style={styles.cardSub}>{summary.unpaidCount}件</Text>
        </View>
      </View>

      {/* 月別売上グラフ */}
      {monthlyData.data.length > 0 && (
        <LineChart
          data={{
            labels: monthlyData.labels,
            datasets: [{ data: monthlyData.data }],
          }}
          width={screenWidth - 32}
          height={200}
          chartConfig={{
            backgroundColor: '#fff',
            backgroundGradientFrom: '#fff',
            backgroundGradientTo: '#fff',
            decimalPlaces: 0,
            color: (opacity = 1) => `rgba(26, 35, 50, ${opacity})`,
            labelColor: () => '#666',
          }}
          bezier
          style={styles.chart}
        />
      )}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16 },
  cards: { flexDirection: 'row', gap: 12, marginBottom: 20 },
  card: { flex: 1, padding: 16, borderRadius: 12 },
  cardLabel: { fontSize: 12, color: '#666' },
  cardValue: { fontSize: 20, fontWeight: 'bold', marginTop: 4 },
  cardSub: { fontSize: 12, color: '#999', marginTop: 2 },
  chart: { borderRadius: 12 },
});
```

## よくあるエラーと対処法

開発中に遭遇しやすい問題とその解決策をまとめます。

### expo-print が動作しない

Expo Go では `expo-print` が正常に動作しない場合があります。開発ビルド（Development Build）を使うことで解決できます。

```bash
# EAS Build で開発ビルドを作成
npx eas build --profile development --platform ios
```

### Supabase の RLS（Row Level Security）エラー

テーブル作成後に RLS を有効化すると、デフォルトではすべての操作がブロックされます。ポリシーの設定を忘れずに行いましょう。

```sql
-- 認証済みユーザーが自分のデータのみ操作できるポリシー
ALTER TABLE invoices ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can manage own invoices"
  ON invoices FOR ALL
  USING (auth.uid() = user_id);
```

### PDF の日本語フォントが文字化けする

`expo-print` で生成する PDF はシステムフォントを使用しますが、一部の環境で日本語が表示されないことがあります。HTML テンプレート内で Web フォントを指定することで解決できます。

```html
<style>
  @import url('https://fonts.googleapis.com/css2?family=Noto+Sans+JP&display=swap');
  body { font-family: 'Noto Sans JP', sans-serif; }
</style>
```

## FAQ

### Q: Rork の無料プランでもこのアプリは作れますか？

はい、開発自体は無料プランで可能です。ただし、App Store / Google Play への公開には Rork Max のサブスクリプションが必要になります。まずは無料プランで開発・テストを行い、公開準備ができた段階で Rork Max へのアップグレードを検討してください。

### Q: 請求書のテンプレートデザインをカスタマイズできますか？

もちろんです。PDF 生成に使う HTML テンプレートを自由に編集できます。CSS でロゴの配置、カラースキーム、フォントなどを変更したり、会社情報や振込先口座を固定ヘッダーとして追加したりできます。HTML と CSS の知識があればデザインの自由度は非常に高いです。

### Q: オフラインでも請求書を作成できますか？

AsyncStorage にローカルキャッシュを実装すれば、オフラインでも請求書の作成と PDF 生成が可能です。オンラインに復帰した際に Supabase と同期する仕組みを組み合わせると、外出先でもストレスなく使えるアプリになります。オフライン対応の詳しい実装パターンは既存の記事でも解説しています。

### Q: インボイス制度（適格請求書等保存方式）に対応できますか？

はい、HTML テンプレートに登録番号（T + 13桁の数字）、適用税率、税率ごとの消費税額を明記すれば、2023年10月から施行されたインボイス制度に対応した請求書を発行できます。テンプレートにこれらのフィールドを追加するだけなので、実装は比較的シンプルです。

## まとめ

この記事では、Rork を使ってフリーランス向けのインボイス（請求書）アプリを構築する方法を解説しました。

実装した主な機能を振り返ると、Supabase によるクライアント・請求書データの管理、expo-print を使った PDF 生成と共有、月別売上ダッシュボード、そして Stripe 連携によるオンライン決済です。

このアプリをベースに、定期請求の自動送信、経費管理機能、確定申告用レポートの出力など、自分のワークフローに合わせて拡張していくことができます。Rork の AI プロンプトを活用すれば、追加機能の実装もスピーディに進められるでしょう。

まずは基本的なクライアント登録と請求書作成から始めて、少しずつ機能を追加していくのがおすすめです。
