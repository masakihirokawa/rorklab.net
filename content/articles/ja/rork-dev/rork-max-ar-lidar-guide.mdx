## はじめに

2026年2月にリリースされた**Rork Max**は、ネイティブSwiftアプリを生成するAIアプリビルダーです。React Nativeベースのオリジナル「Rork」とは異なり、Rork MaxはAppleの全プラットフォーム（iPhone、iPad、Apple Watch、Apple TV、Vision Pro、iMessage）向けのネイティブコードを生成します。

その中でも特に注目を集めているのが、**AR（拡張現実）とLiDARセンサー**を活用したアプリ開発です。従来、ARKitやRealityKitを使ったアプリ開発には高度なSwift知識が必要でしたが、Rork Maxを使えば自然言語のプロンプトだけでAR機能を実装できます。

---

## Rork MaxのAR/LiDAR対応の概要

Rork Maxは以下のAR関連フレームワークに対応しています。

- **ARKit** — Appleの主力ARフレームワーク。平面検出、オブジェクト追跡、顔認識など
- **RealityKit** — 3Dオブジェクトのレンダリングと物理シミュレーション
- **Reality Composer Pro** — Apple製のARコンテンツ作成ツールとの連携
- **LiDARスキャナー** — Pro/Max iPhoneとiPad ProのLiDARを活用した精密な深度測定

特に話題になったのが、あるユーザーがRork Maxを使ってポケモンGO風のARゲームを短時間で構築したケースです。これはRork MaxのAR生成能力の高さを示す好例です。

## Step 1: Rork MaxでARアプリのプロジェクトを作成する

Rork Maxにアクセスし、以下のようなプロンプトを入力します。

### プロンプト例（家具配置ARアプリ）

```
Create a native Swift iOS app that uses ARKit and RealityKit to let users
place 3D furniture models in their room using augmented reality.

Features:
- Detect horizontal planes (floor/table surfaces)
- Show furniture catalog with 5 sample 3D models (chair, sofa, table, lamp, shelf)
- Tap to place selected furniture on detected plane
- Pinch to resize, rotate with gesture
- Tap placed furniture to delete it
- Save screenshot with AR scene
```

### プロンプト例（LiDARで部屋をスキャン）

```
Build a native Swift iOS app that uses the LiDAR scanner to create a 3D mesh
of the surrounding environment.

Features:
- Real-time 3D mesh visualization with ARKit's scene reconstruction
- Toggle wireframe/solid mesh display
- Measure distances between two tapped points
- Export mesh as USDZ file
- Works on iPhone 12 Pro and later
```

## Step 2: 生成されたSwiftコードの構造を理解する

Rork Maxが生成する典型的なARアプリのコード構造は以下のとおりです。

```swift
// ContentView.swift（生成例）
import SwiftUI
import RealityKit
import ARKit

struct ContentView: View {
    @StateObject private var arViewModel = ARViewModel()

    var body: some View {
        ZStack {
            ARViewContainer(viewModel: arViewModel)
                .ignoresSafeArea()

            VStack {
                Spacer()
                FurnitureCatalogView(viewModel: arViewModel)
                    .padding()
            }
        }
        .onAppear {
            arViewModel.setupARSession()
        }
    }
}
```

```swift
// ARViewModel.swift（生成例）
import ARKit
import RealityKit
import Combine

@MainActor
class ARViewModel: ObservableObject {
    var arView: ARView?
    @Published var detectedPlanes: [ARPlaneAnchor] = []
    @Published var selectedModel: String = "chair"

    func setupARSession() {
        let config = ARWorldTrackingConfiguration()
        config.planeDetection = [.horizontal, .vertical]

        // LiDAR搭載デバイスのシーン再構成を有効化
        if ARWorldTrackingConfiguration.supportsSceneReconstruction(.mesh) {
            config.sceneReconstruction = .mesh
        }

        arView?.session.run(config)
    }

    func placeModel(at position: SIMD3<Float>) {
        // RealityKitでモデルを配置
        guard let modelEntity = try? ModelEntity.loadModel(named: selectedModel) else { return }
        let anchor = AnchorEntity(world: position)
        anchor.addChild(modelEntity)
        arView?.scene.addAnchor(anchor)
    }
}
```

## Step 3: LiDARを活用したシーン再構成

LiDAR搭載デバイスでは、より精密なAR体験が可能です。

```swift
// LiDARシーン再構成の設定
func enableLiDARMeshVisualization(arView: ARView) {
    // メッシュのオクルージョン（手前にあるオブジェクトで背後のARオブジェクトが隠れる）
    arView.environment.sceneUnderstanding.options.insert(.occlusion)

    // 物理シミュレーション（ARオブジェクトが現実の床で止まる）
    arView.environment.sceneUnderstanding.options.insert(.physics)

    // デバッグ用メッシュ表示
    arView.debugOptions.insert(.showSceneUnderstanding)
}
```

### LiDARの主なユースケース

| ユースケース | 精度 | 必要フレームワーク |
|---|---|---|
| 家具の配置 | ±2cm | ARKit + RealityKit |
| 部屋の寸法測定 | ±1cm | ARKit + LiDAR Depth API |
| 3Dスキャン・エクスポート | 高 | ARKit + ModelIO |
| ゲームのオクルージョン | リアルタイム | RealityKit |
| ロボット・ナビゲーション | 高 | ARKit + CoreML |

## Step 4: Vision ProとのAR体験の統一

Rork Maxの強みの一つは、iPhoneで開発したARアプリをVision Pro向けにも対応させやすい点です。

```swift
// プラットフォーム分岐（生成例）
#if os(visionOS)
import RealityKit
import SwiftUI

struct ImmersiveView: View {
    var body: some View {
        RealityView { content in
            // Vision Pro向け空間コンテンツ
            let model = try? await ModelEntity(named: "furniture_chair")
            if let model = model {
                content.add(model)
            }
        }
    }
}
#else
// iPhone/iPad向けARKit実装
#endif
```

## Step 5: App Storeへの審査申請のポイント

ARカメラ機能を使うアプリには、Info.plistへの権限記述が必要です。Rork Maxは自動でこれを追加しますが、確認しておきましょう。

```xml
<!-- Info.plist（自動生成される） -->
<key>NSCameraUsageDescription</key>
<string>このアプリはAR機能のためにカメラを使用します。</string>

<!-- LiDARを使う場合 -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>ARコンテキストの精度向上に位置情報を使用します。</string>
```

App Store審査では、AR機能の実際の動作ビデオをスクリーンショットとして提出することが強く推奨されます。

## 実用的なTips

### Tip 1: プロンプトには具体的なデバイス要件を明記する

```
// 悪い例
"Make an AR app"

// 良い例
"Create a native Swift ARKit app for iPhone 12 Pro and later with LiDAR mesh
scanning. Target iOS 17+. Use RealityKit for rendering."
```

### Tip 2: フォールバック処理をプロンプトに含める

```
Add graceful fallback for devices without LiDAR: use standard plane detection
instead of scene reconstruction on non-LiDAR devices.
```

### Tip 3: パフォーマンスを意識したプロンプト

```
Optimize for 60fps AR rendering. Use LOD (Level of Detail) for 3D models.
Limit mesh reconstruction update frequency to every 0.5 seconds.
```

## Rork MaxのAR機能の限界と注意点

- **複雑なシェーダー**: カスタムMetalシェーダーは現時点では生成品質にばらつきがある
- **大規模シーン**: 多数のアンカーを扱う場合、手動最適化が必要なケースあり
- **ARCloudKit連携**: 複数デバイス間でのAR共有（Collaborative Sessions）は要確認
- **デバイステスト必須**: シミュレーターではARKit動作しないため、実機テストが必須
