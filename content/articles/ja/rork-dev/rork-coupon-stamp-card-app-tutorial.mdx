## はじめに — なぜクーポン＆スタンプカードアプリなのか

店舗の顧客管理をデジタル化したいというニーズは年々高まっています。紙のスタンプカードからアプリへ移行することで、顧客のリピート率向上やデータ分析が可能になります。Rork を使えば、AI によるコード生成の力を借りて、QRコード読取・ポイント管理・プッシュ通知といった本格的な機能を備えたアプリを効率よく構築できます。

この記事では、Rork でクーポン＆スタンプカードアプリをゼロから作り上げる手順を、コード例とともに丁寧に解説していきます。対象読者は Rork の基本操作を理解している方で、実用的な店舗向けアプリを開発したい方を想定しています。

Rork の基本的な使い方にまだ慣れていない方は、まず [Rork で最初のアプリを30分で作る入門ガイド](/articles/rork-basics/rork-first-app-30min) を参照してください。

## アプリの全体設計とデータ構造

まず、アプリの全体像を設計します。クーポン＆スタンプカードアプリに必要な主要画面は以下の通りです。

- **ホーム画面**: 現在のスタンプ数と特典までの残り回数を表示
- **QRスキャン画面**: 店舗に設置されたQRコードを読み取ってスタンプを加算
- **クーポン一覧画面**: 利用可能なクーポンと過去の使用履歴を表示
- **プロフィール画面**: ユーザー情報と通知設定

データ構造としては、以下のようなスキーマを Rork に伝えます。

```typescript
// ユーザーのスタンプカード情報
interface StampCard {
  id: string;
  userId: string;
  shopId: string;
  currentStamps: number;  // 現在のスタンプ数
  totalStamps: number;    // 累計スタンプ数
  createdAt: Date;
  updatedAt: Date;
}

// クーポン情報
interface Coupon {
  id: string;
  title: string;
  description: string;
  requiredStamps: number; // 必要スタンプ数
  discount: string;       // 例: "10%OFF", "ドリンク1杯無料"
  expiresAt: Date;
  isUsed: boolean;
}

// スタンプ履歴（いつ・どこで獲得したか）
interface StampHistory {
  id: string;
  cardId: string;
  stampCount: number;
  scannedAt: Date;
  shopName: string;
}
```

このデータ構造を Rork のプロンプトに含めることで、AI がデータフローを正しく理解した上で画面を生成してくれます。

## QRコードスキャン機能の実装

スタンプカードアプリの核となるのがQRコード読取機能です。React Native の `expo-camera` と `expo-barcode-scanner` を組み合わせて実装します。

Rork に以下のようなプロンプトを入力します。

```text
QRコードスキャン画面を作成してください。
- expo-camera を使ってカメラプレビューを表示
- QRコードを検出したら自動で読み取り
- 読み取り成功時にスタンプ加算のアニメーションを表示
- 同じQRコードの連続スキャンを防止（30秒のクールダウン）
- カメラ権限が未許可の場合は許可リクエスト画面を表示
```

Rork が生成するコードの主要部分は以下のような構成になります。

```typescript
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useState, useRef } from 'react';
import { View, Text, StyleSheet, Alert, Animated } from 'react-native';

export default function QRScanScreen() {
  const [permission, requestPermission] = useCameraPermissions();
  const [scanned, setScanned] = useState(false);
  const [lastScannedCode, setLastScannedCode] = useState('');
  const scaleAnim = useRef(new Animated.Value(0)).current;

  // QRコード読取時のハンドラ
  const handleBarcodeScanned = ({ type, data }: { type: string; data: string }) => {
    if (scanned || data === lastScannedCode) return;

    setScanned(true);
    setLastScannedCode(data);

    // スタンプ加算処理
    addStamp(data);

    // 成功アニメーション
    Animated.spring(scaleAnim, {
      toValue: 1,
      friction: 3,
      useNativeDriver: true,
    }).start();

    // 30秒後にリセット（連続スキャン防止）
    setTimeout(() => {
      setScanned(false);
      setLastScannedCode('');
      scaleAnim.setValue(0);
    }, 30000);
  };

  // カメラ権限チェック
  if (!permission?.granted) {
    return (
      <View style={styles.container}>
        <Text style={styles.message}>
          スタンプを獲得するにはカメラへのアクセスが必要です
        </Text>
        {/* 権限リクエストボタン */}
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <CameraView
        style={styles.camera}
        barcodeScannerSettings={{
          barcodeTypes: ['qr'],
        }}
        onBarcodeScanned={scanned ? undefined : handleBarcodeScanned}
      />
      {/* スキャンガイドのオーバーレイ */}
      <View style={styles.overlay}>
        <View style={styles.scanArea} />
        <Text style={styles.hint}>QRコードを枠内に合わせてください</Text>
      </View>
    </View>
  );
}
// 期待出力: カメラプレビュー上にQRコードスキャンエリアが表示され、
// QRコードを検出すると自動でスタンプが加算される
```

QRコードの詳しい実装パターンについては [Rork Max QRコード実装ガイド](/articles/rork-dev/rork-max-qr-code-guide) も合わせてご覧ください。

## スタンプカードUIの設計とアニメーション

スタンプカードの見た目は、ユーザーのリピート意欲を大きく左右します。視覚的に「あと何個で特典がもらえるか」が一目で分かるUIを設計しましょう。

```typescript
import { View, Text, StyleSheet } from 'react-native';
import { Ionicons } from '@expo/vector-icons';

interface StampCardProps {
  currentStamps: number;
  maxStamps: number;    // 特典獲得に必要なスタンプ数（例: 10）
  rewardTitle: string;  // 特典名（例: "ドリンク1杯無料"）
}

export function StampCardView({ currentStamps, maxStamps, rewardTitle }: StampCardProps) {
  const stamps = Array.from({ length: maxStamps }, (_, i) => i < currentStamps);

  return (
    <View style={styles.card}>
      <Text style={styles.shopName}>CAFE DOLICE</Text>
      <Text style={styles.rewardText}>
        {rewardTitle}まであと {maxStamps - currentStamps} 個
      </Text>

      <View style={styles.stampGrid}>
        {stamps.map((filled, index) => (
          <View key={index} style={styles.stampSlot}>
            {filled ? (
              <Ionicons name="checkmark-circle" size={36} color="#FF6B35" />
            ) : (
              <Ionicons name="ellipse-outline" size={36} color="#DDD" />
            )}
            <Text style={styles.stampNumber}>{index + 1}</Text>
          </View>
        ))}
      </View>

      <View style={styles.progressBar}>
        <View
          style={[
            styles.progressFill,
            { width: `${(currentStamps / maxStamps) * 100}%` },
          ]}
        />
      </View>
    </View>
  );
}
// 期待出力: 10個のスタンプスロットがグリッド表示され、
// 獲得済みスタンプはオレンジ色のチェックマーク、
// 未獲得スロットはグレーの円で表示される
```

Rork のプロンプトでは「カフェ風のおしゃれなデザインで」「スタンプ獲得時にバウンスアニメーションを追加して」といった指示を加えると、より洗練されたUIが生成されます。

## クーポン管理とローカルストレージ

スタンプが一定数に達したときにクーポンを自動発行する仕組みを実装します。AsyncStorage を使ったローカルデータ管理がポイントです。

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

const STAMP_KEY = '@stamp_card';
const COUPONS_KEY = '@coupons';

// スタンプを加算し、特典条件を満たしたらクーポンを発行
export async function addStampAndCheckReward(shopQRData: string): Promise<{
  newStampCount: number;
  couponIssued: boolean;
  coupon?: Coupon;
}> {
  try {
    // 現在のスタンプ数を取得
    const cardData = await AsyncStorage.getItem(STAMP_KEY);
    const card: StampCard = cardData
      ? JSON.parse(cardData)
      : { currentStamps: 0, totalStamps: 0 };

    // スタンプを加算
    card.currentStamps += 1;
    card.totalStamps += 1;
    card.updatedAt = new Date().toISOString();

    let couponIssued = false;
    let newCoupon: Coupon | undefined;

    // 10個たまったらクーポン発行
    if (card.currentStamps >= 10) {
      newCoupon = {
        id: `coupon_${Date.now()}`,
        title: 'ドリンク1杯無料クーポン',
        description: 'お好きなドリンクを1杯プレゼント',
        requiredStamps: 10,
        discount: 'ドリンク1杯無料',
        expiresAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000).toISOString(),
        isUsed: false,
      };

      // クーポンを保存
      const existingCoupons = await AsyncStorage.getItem(COUPONS_KEY);
      const coupons = existingCoupons ? JSON.parse(existingCoupons) : [];
      coupons.push(newCoupon);
      await AsyncStorage.setItem(COUPONS_KEY, JSON.stringify(coupons));

      // スタンプをリセット
      card.currentStamps = 0;
      couponIssued = true;
    }

    await AsyncStorage.setItem(STAMP_KEY, JSON.stringify(card));

    return { newStampCount: card.currentStamps, couponIssued, coupon: newCoupon };
  } catch (error) {
    console.error('スタンプ追加エラー:', error);
    throw error;
  }
}
// 期待出力: スタンプが10個に達すると自動でクーポンが発行され、
// スタンプカウントが0にリセットされる
```

バックエンドとの連携が必要な場合は、Supabase を使ってクラウドにデータを同期する構成も検討できます。認証とリアルタイム同期の組み合わせについては [Rork × Supabase 認証＆リアルタイム連携ガイド](/articles/rork-dev/rork-supabase-auth-realtime-guide) が参考になります。

## プッシュ通知でクーポン期限をリマインド

発行したクーポンの有効期限が近づいたらプッシュ通知を送ることで、クーポンの利用率を高められます。Expo の通知APIを使って実装します。

```typescript
import * as Notifications from 'expo-notifications';

// 通知の初期設定
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});

// クーポン有効期限のリマインド通知をスケジュール
export async function scheduleCouponReminder(coupon: Coupon) {
  const expiresAt = new Date(coupon.expiresAt);
  const reminderDate = new Date(expiresAt.getTime() - 3 * 24 * 60 * 60 * 1000);
  // 有効期限の3日前に通知

  if (reminderDate > new Date()) {
    await Notifications.scheduleNotificationAsync({
      content: {
        title: 'クーポンの有効期限が近づいています',
        body: `「${coupon.title}」の有効期限はあと3日です。お早めにご利用ください！`,
        data: { couponId: coupon.id },
      },
      trigger: {
        type: Notifications.SchedulableTriggerInputTypes.DATE,
        date: reminderDate,
      },
    });
  }
}

// 新しいスタンプ獲得を通知
export async function notifyStampEarned(currentCount: number, maxCount: number) {
  await Notifications.scheduleNotificationAsync({
    content: {
      title: 'スタンプを獲得しました！',
      body: `現在 ${currentCount}/${maxCount} 個。あと ${maxCount - currentCount} 個で特典をゲット！`,
    },
    trigger: null, // 即時通知
  });
}
// 期待出力: クーポン発行時に有効期限3日前のリマインドが自動設定され、
// スタンプ獲得時には即時で進捗通知が表示される
```

プッシュ通知の詳しい実装パターンは [Rork でプッシュ通知を実装するガイド](/articles/rork-dev/rork-push-notifications) で解説しています。

## よくある質問（FAQ）

### Q: QRコードの生成はアプリ側で行えますか？

はい、店舗側のQRコード生成には `react-native-qrcode-svg` ライブラリが使えます。店舗IDとタイムスタンプを含むQRコードを動的に生成し、画面に表示するか印刷用に出力できます。Rork のプロンプトで「店舗管理画面にQRコード生成機能を追加して」と指示すれば、ライブラリの導入からUI実装まで一括で生成してくれます。

### Q: オフライン環境でもスタンプは貯められますか？

AsyncStorage を使ったローカル保存の設計にしているため、オフライン環境でもスタンプの加算は可能です。ただし、QRコードの読み取り自体はカメラのみで完結するため通信は不要ですが、バックエンドと同期している場合はオンライン復帰後に自動同期する仕組みを追加すると安心です。

### Q: 複数店舗に対応するにはどうすればいいですか？

データ構造の `shopId` フィールドを活用して、店舗ごとに異なるスタンプカードを管理できます。ホーム画面では店舗一覧をカルーセル表示し、スワイプで切り替える UI が直感的です。Rork に「複数店舗のスタンプカードをカルーセルで切り替えられるようにして」と指示すれば対応可能です。

### Q: アプリを収益化するにはどんな方法がありますか？

店舗向けのSaaS型課金モデルが最も一般的です。店舗オーナーが月額でアプリ利用料を支払い、エンドユーザー（お客さん）は無料で使うという構成です。Stripe を使ったサブスクリプション実装については [Rork × Stripe サブスクリプション実装ガイド](/articles/rork-business/rork-stripe-subscription-monetization-guide) で詳しく解説しています。
