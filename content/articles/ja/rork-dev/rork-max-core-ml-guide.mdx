スマートフォンの処理能力が飛躍的に向上した今、AI の推論をクラウドではなく**デバイス上**で行う「オンデバイス AI」が主流になりつつあります。Apple の **Core ML** フレームワークはその中心的な技術であり、プライバシー保護・低レイテンシ・オフライン動作という三拍子が揃っています。このガイドでは、**Rork Max** を使ってCore ML統合アプリをゼロから構築する方法を、具体的なプロンプト例とSwiftコードで詳しく解説します。

## Core ML とは？Rork Max との関係

**Core ML** は Apple が提供するオンデバイス機械学習フレームワークです。画像分類・物体検出・自然言語処理・音声認識・姿勢推定など、幅広いAIタスクを iPhone や iPad 上で直接実行できます。

従来、Core ML を使いこなすには Xcode の知識とSwiftの習熟が必要でした。しかし **Rork Max** は Claude Code + Opus 4.6 を搭載した AI コンパイラが Swift コードを自動生成するため、Core ML の複雑な初期化処理やデータ変換を自分で書く必要がありません。

### Rork Max が Core ML に強い理由

Rork Max は React Native ベースの旧 Rork とは異なり、**ネイティブ Swift / SwiftUI** でアプリを生成します。これにより：

- `Vision`、`NaturalLanguage`、`CreateML` など Apple の ML フレームワーク群を直接呼び出し可能
- Neural Engine（Apple Silicon）を最大限活用した高速推論
- オフライン動作・プライバシー保護を標準で実現
- App Store の審査基準（リモートコード実行禁止等）を自動的に満たす構造

## Step 2：Core ML モデルをアプリに統合する

### 2-1. 事前学習済みモデルを使う（最速の方法）

Apple は [Core ML モデルギャラリー](https://developer.apple.com/machine-learning/models/) で多数の事前学習済みモデルを無料公開しています。Rork Max への統合は次のプロンプトで行えます。

```
MobileNetV2.mlmodel を Vision フレームワーク経由で呼び出すコードを追加してください。
VNImageRequestHandler を使い、UIImage を CVPixelBuffer に変換して推論を実行します。
結果の VNClassificationObservation を上位5件取得し、ConfidenceCardView コンポーネントで表示してください。
```

生成されるコードの主要部分はこのようになります：

```swift
import Vision
import CoreML
import SwiftUI

// モデルのロードと推論実行
class ImageClassifier: ObservableObject {
    @Published var results: [ClassificationResult] = []

    private lazy var model: VNCoreMLModel = {
        // MobileNetV2をCore MLモデルとしてロード
        let config = MLModelConfiguration()
        config.computeUnits = .all // Neural Engine を優先使用
        let mlModel = try! MobileNetV2(configuration: config).model
        return try! VNCoreMLModel(for: mlModel)
    }()

    func classify(image: UIImage) {
        guard let ciImage = CIImage(image: image) else { return }

        // Vision リクエストを作成
        let request = VNCoreMLRequest(model: model) { [weak self] request, error in
            guard let observations = request.results as? [VNClassificationObservation] else { return }

            // トップ5の結果をメインスレッドで更新
            DispatchQueue.main.async {
                self?.results = observations.prefix(5).map {
                    ClassificationResult(
                        label: $0.identifier,
                        confidence: Double($0.confidence) * 100
                    )
                }
            }
        }

        // 画像を処理（アスペクト比を維持してトリミング）
        request.imageCropAndScaleOption = .centerCrop

        let handler = VNImageRequestHandler(ciImage: ciImage, options: [:])
        try? handler.perform([request])
    }
}

// 結果表示用データモデル
struct ClassificationResult: Identifiable {
    let id = UUID()
    let label: String
    let confidence: Double
}
```

## Step 3：自然言語処理（NLP）を統合する

Core ML の `NaturalLanguage` フレームワークを使ったテキスト分析も Rork Max で簡単に実装できます。

```
テキスト感情分析アプリを作ってください。
- ユーザーがテキストを入力できるTextField
- NaturalLanguageフレームワークのNLTaggerでセンチメント分析
- ポジティブ/ネガティブ/ニュートラルを絵文字と色で表示（緑/赤/グレー）
- 日本語と英語の両方に対応
```

生成されるセンチメント分析コードの例：

```swift
import NaturalLanguage

class SentimentAnalyzer: ObservableObject {
    @Published var sentiment: SentimentResult = .neutral

    private let tagger = NLTagger(tagSchemes: [.sentimentScore])

    func analyze(text: String) {
        tagger.string = text

        let (tag, _) = tagger.tag(
            at: text.startIndex,
            unit: .paragraph,
            scheme: .sentimentScore
        )

        // スコアは -1.0（最もネガティブ）〜 +1.0（最もポジティブ）
        if let scoreString = tag?.rawValue, let score = Double(scoreString) {
            DispatchQueue.main.async {
                switch score {
                case 0.2...:
                    self.sentiment = .positive  // 😊 緑
                case ..<(-0.2):
                    self.sentiment = .negative  // 😞 赤
                default:
                    self.sentiment = .neutral   // 😐 グレー
                }
            }
        }
    }
}

enum SentimentResult {
    case positive, negative, neutral

    var emoji: String {
        switch self {
        case .positive: return "😊"
        case .negative: return "😞"
        case .neutral:  return "😐"
        }
    }

    var color: String {
        switch self {
        case .positive: return "green"
        case .negative: return "red"
        case .neutral:  return "gray"
        }
    }
}
```

## Step 5：パフォーマンス最適化

Core ML アプリのパフォーマンスを最大化するプロンプトの例：

```
Core MLモデルの推論を最適化してください：
1. MLModelConfiguration で computeUnits を .cpuAndNeuralEngine に設定
2. バックグラウンドキューで推論を実行（UIスレッドをブロックしない）
3. 連続フレーム処理時に VNSequenceRequestHandler を使ってモデルを再利用
4. モデルをシングルトンとしてキャッシュ（毎回ロードしない）
```

最適化されたモデルロードの実装例：

```swift
// シングルトンパターンでモデルキャッシュ
actor ModelCache {
    static let shared = ModelCache()

    private var cachedModel: VNCoreMLModel?

    func getModel() async throws -> VNCoreMLModel {
        if let model = cachedModel { return model }

        // バックグラウンドで一度だけロード
        let config = MLModelConfiguration()
        config.computeUnits = .cpuAndNeuralEngine // Neural Engine 優先、CPU をフォールバック

        let mlModel = try await MLModel.load(
            contentsOf: Bundle.main.url(forResource: "MyClassifier", withExtension: "mlmodelc")!,
            configuration: config
        )
        let visionModel = try VNCoreMLModel(for: mlModel)
        cachedModel = visionModel
        return visionModel
    }
}

// 使用例：非同期で推論実行（UIをブロックしない）
Task.detached(priority: .userInitiated) {
    let model = try await ModelCache.shared.getModel()
    // 推論処理...
}
```

## よくあるエラーと対処法

### エラー: `Error loading MLModel: No such file`

.mlmodel ファイルがバンドルに含まれていない場合に発生します。

```
Info.plist に MLModelFiles キーを追加し、
モデルファイルをBundle resources に含めるよう修正してください。
```

### エラー: `VNCoreMLModel failed: Image buffer size mismatch`

モデルの期待する入力サイズと実際の画像サイズが合わない場合に発生します。

```
VNImageRequestHandler の前に UIImage を 224x224 ピクセルにリサイズし、
ImageCropAndScaleOption を .scaleFit に変更してください。
```

### パフォーマンス低下: FPS が期待より低い

```
computeUnits を .all から .cpuAndNeuralEngine に変更してください。
GPU への転送オーバーヘッドがなくなり、Neural Engine のスループットが向上します。
```

## FAQ

### Q. Rork Max で Core ML モデルを自分でトレーニングできますか？

A. Rork Max 自体にトレーニング機能はありませんが、Apple の Create ML アプリ（無料）でモデルをトレーニングして `.mlmodel` ファイルを書き出し、Rork Max のプロンプトで「このモデルを統合してください」と指示することで組み込めます。「カスタム Core ML モデルをバンドルに追加して Vision フレームワークでロードするコードを生成してください」と伝えれば、モデルのロードと推論コードが自動生成されます。

### Q. Core ML はオフラインで動作しますか？

A. はい、Core ML はデバイス上で完結するため、ネットワーク接続は不要です。これはプライバシー保護の観点でも重要です（推論データがサーバーに送信されない）。ただし Supabase などのバックエンドと連携している機能（栄養情報の取得など）はオンライン接続が必要です。

### Q. Rork Max の Core ML 統合は App Store の審査を通過しますか？

A. 通過します。Core ML はApple公式フレームワークであり、App Store ガイドラインに完全準拠しています。Rork Max が生成するコードはリモートコードの実行を行わず、バンドルに含まれたモデルのみを使用するため、審査上の問題はありません。Rork Max の「2クリック App Store 提出」機能でそのまま申請可能です。
