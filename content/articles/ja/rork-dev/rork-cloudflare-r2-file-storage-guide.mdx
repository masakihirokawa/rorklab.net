## Cloudflare R2 とは？なぜ Rork アプリに適しているのか

Rorkでアプリを開発していると、ユーザーが撮影した写真やプロフィール画像、動画ファイルをどこに保存するかという課題に直面します。AWS S3 や Firebase Storage など選択肢はいくつかありますが、2026年現在で注目を集めているのが **Cloudflare R2** です。

Cloudflare R2 は、Amazon S3 と互換性のあるオブジェクトストレージサービスです。最大の特徴は **エグレス料金（データ転送料）が完全無料**である点で、S3では外部へのデータ転送に費用がかかりますが、R2ではそのコストがかかりません。

Rorkアプリとの相性が良い理由は主に3つあります。

まず、**コスト効率の高さ**です。無料枠として月10GBのストレージと100万回のClass Bオペレーションが含まれており、個人開発や初期フェーズのアプリであれば実質無料で利用できます。次に、**Cloudflare Workersとの親和性**です。Rorkのバックエンドとして Cloudflare Workers を利用している場合、R2バインディングを通じてサーバーレス環境からシームレスにアクセスできます。そして、**S3互換APIのサポート**により、既存のS3クライアントライブラリをそのまま流用できます。

この記事では、Rorkで作成したReact Native/Expoアプリから Cloudflare R2 にファイルをアップロードする実践的な実装方法を、ステップバイステップで解説します。

## 前提知識と準備するもの

実装を始める前に、以下を準備してください。

- Cloudflare アカウント（無料プランで可）
- Rork または Expo で作成したReact Nativeプロジェクト
- Node.js の基本的な知識
- Cloudflare Workers の基礎知識（バックエンドとして使用する場合）

今回のアーキテクチャは、Rorkアプリ → Cloudflare Workers（APIサーバー） → R2 という流れで構成します。アプリから直接R2にアクセスするのではなく、Workerを経由させることでセキュリティを担保します。

## Step 1: Cloudflare R2 バケットの作成

まずCloudflareダッシュボードにログインし、R2バケットを作成します。

Cloudflareダッシュボードのサイドメニューから「R2 オブジェクトストレージ」を選択し、「バケットを作成する」ボタンをクリックします。バケット名はグローバルでユニークである必要はありませんが、プロジェクト名と関連付けた名前（例: `my-app-uploads`）を推奨します。リージョンは「自動」を選択すると、Cloudflareが最適な場所に自動配置します。

次に、Workers が R2 にアクセスするための API トークンを作成します。「マイプロフィール」→「APIトークン」→「トークンを作成する」から、「Cloudflare R2:編集」権限を持つトークンを生成してください。

## Step 2: Cloudflare Workers バックエンドの実装

アプリから直接R2に書き込む代わりに、Cloudflare Workers を API サーバーとして使います。Worker が Presigned URL を生成し、アプリはその URL に対してファイルを直接アップロードする構成です。

まず Workers プロジェクトを作成します。

```bash
# Wrangler CLI のインストール（未インストールの場合）
npm install -g wrangler

# 新規 Workers プロジェクト作成
npm create cloudflare@latest my-app-api
cd my-app-api
```

`wrangler.toml` にR2バインディングを追加します。

```toml
name = "my-app-api"
main = "src/index.ts"
compatibility_date = "2026-04-01"

[[r2_buckets]]
binding = "FILE_BUCKET"
bucket_name = "my-app-uploads"
```

次に、Presigned URL を生成するエンドポイントを実装します。

```typescript
// src/index.ts
import { Hono } from 'hono'
import { cors } from 'hono/cors'

type Bindings = {
  FILE_BUCKET: R2Bucket
}

const app = new Hono<{ Bindings: Bindings }>()

// CORS設定（本番ではオリジンを絞ること）
app.use('*', cors({
  origin: '*',
  allowHeaders: ['Content-Type', 'Authorization'],
  allowMethods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
}))

// Presigned URL 生成エンドポイント
app.post('/api/upload/presign', async (c) => {
  const { fileName, contentType } = await c.req.json()

  // バリデーション
  const allowedTypes = ['image/jpeg', 'image/png', 'image/webp', 'video/mp4']
  if (!allowedTypes.includes(contentType)) {
    return c.json({ error: 'Unsupported file type' }, 400)
  }

  // ユニークなファイルキーを生成
  const timestamp = Date.now()
  const random = Math.random().toString(36).substring(2, 8)
  const ext = fileName.split('.').pop() || 'bin'
  const key = `uploads/${timestamp}-${random}.${ext}`

  // R2 の署名付きURL生成（有効期限15分）
  const presignedUrl = await c.env.FILE_BUCKET.createMultipartUpload(key)
    .catch(() => null)

  // 署名付きURL方式（Cloudflare R2 はPUT直接アップロードを推奨）
  // Workerを経由してアップロードを受け付ける方式
  return c.json({
    uploadUrl: `https://my-app-api.workers.dev/api/upload/${key}`,
    fileKey: key,
  })
})

// ファイルアップロード受付エンドポイント
app.put('/api/upload/:key{.+}', async (c) => {
  const key = c.req.param('key')
  const contentType = c.req.header('Content-Type') || 'application/octet-stream'

  // ファイルサイズ制限（10MB）
  const body = await c.req.arrayBuffer()
  if (body.byteLength > 10 * 1024 * 1024) {
    return c.json({ error: 'File too large. Max 10MB.' }, 413)
  }

  // R2 へのアップロード
  await c.env.FILE_BUCKET.put(key, body, {
    httpMetadata: { contentType },
  })

  // 公開URL（カスタムドメインまたは R2 公開URL）
  const publicUrl = `https://pub-xxxxx.r2.dev/${key}`

  return c.json({
    success: true,
    url: publicUrl,
    key,
  })
})

// ファイル削除エンドポイント
app.delete('/api/files/:key{.+}', async (c) => {
  const key = c.req.param('key')
  await c.env.FILE_BUCKET.delete(key)
  return c.json({ success: true })
})

export default app
```

Worker をデプロイします。

```bash
# デプロイ
wrangler deploy

# 期待する出力例:
# Uploaded my-app-api (3.42 sec)
# Published my-app-api (0.21 sec)
# https://my-app-api.your-account.workers.dev
```

## Step 3: Rorkアプリ側の実装

バックエンドが整ったら、Rorkアプリ（React Native/Expo）からファイルをアップロードする実装を追加します。

まず Expo Image Picker をインストールします。Rorkプロジェクトのプロンプトエリアに以下を入力するか、手動でコードを追加します。

```bash
npx expo install expo-image-picker expo-document-picker
```

次にファイルアップロードのユーティリティ関数を作成します。

```typescript
// utils/fileUpload.ts
const API_BASE = 'https://my-app-api.your-account.workers.dev'

export interface UploadResult {
  url: string
  key: string
}

/**
 * ファイルをCloudflare R2にアップロードする
 * @param localUri  ローカルファイルパス（expo-image-pickerの結果など）
 * @param fileName  保存するファイル名
 * @param contentType  MIMEタイプ（例: 'image/jpeg'）
 */
export async function uploadFileToR2(
  localUri: string,
  fileName: string,
  contentType: string
): Promise<UploadResult> {
  // Step 1: アップロードURLを取得
  const presignRes = await fetch(`${API_BASE}/api/upload/presign`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ fileName, contentType }),
  })

  if (!presignRes.ok) {
    throw new Error('アップロードURLの取得に失敗しました')
  }

  const { uploadUrl, fileKey } = await presignRes.json()

  // Step 2: ファイルをバイナリとして読み込む
  const fileRes = await fetch(localUri)
  const blob = await fileRes.blob()

  // Step 3: R2 にアップロード
  const uploadRes = await fetch(uploadUrl, {
    method: 'PUT',
    headers: { 'Content-Type': contentType },
    body: blob,
  })

  if (!uploadRes.ok) {
    throw new Error('ファイルのアップロードに失敗しました')
  }

  const result = await uploadRes.json()
  return { url: result.url, key: fileKey }
}
```

アップロードUIコンポーネントを実装します。

```typescript
// components/ImageUploader.tsx
import React, { useState } from 'react'
import { View, Image, TouchableOpacity, Text, ActivityIndicator, Alert } from 'react-native'
import * as ImagePicker from 'expo-image-picker'
import { uploadFileToR2 } from '../utils/fileUpload'

interface ImageUploaderProps {
  onUploadComplete: (url: string) => void
}

export default function ImageUploader({ onUploadComplete }: ImageUploaderProps) {
  const [uploading, setUploading] = useState(false)
  const [previewUri, setPreviewUri] = useState<string | null>(null)
  const [progress, setProgress] = useState(0)

  const pickAndUpload = async () => {
    // カメラロールへのアクセス権限をリクエスト
    const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync()
    if (status !== 'granted') {
      Alert.alert('権限エラー', 'カメラロールへのアクセスを許可してください。')
      return
    }

    // 画像ピッカーを起動
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      aspect: [4, 3],
      quality: 0.8, // 80%品質で圧縮（容量削減）
    })

    if (result.canceled || !result.assets[0]) return

    const asset = result.assets[0]
    setPreviewUri(asset.uri)
    setUploading(true)
    setProgress(10)

    try {
      const fileName = asset.fileName || `photo_${Date.now()}.jpg`
      const contentType = asset.mimeType || 'image/jpeg'

      setProgress(30)
      const { url } = await uploadFileToR2(asset.uri, fileName, contentType)
      setProgress(100)

      onUploadComplete(url)
      Alert.alert('完了', '画像のアップロードが完了しました！')
    } catch (error) {
      Alert.alert('エラー', 'アップロードに失敗しました。再度お試しください。')
      console.error('Upload error:', error)
    } finally {
      setUploading(false)
      setProgress(0)
    }
  }

  return (
    <View style={{ alignItems: 'center', gap: 12 }}>
      {previewUri && (
        <Image
          source={{ uri: previewUri }}
          style={{ width: 200, height: 150, borderRadius: 8 }}
          resizeMode="cover"
        />
      )}

      <TouchableOpacity
        onPress={pickAndUpload}
        disabled={uploading}
        style={{
          backgroundColor: uploading ? '#ccc' : '#F6821F',
          paddingHorizontal: 24,
          paddingVertical: 12,
          borderRadius: 8,
        }}
      >
        {uploading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={{ color: '#fff', fontWeight: 'bold' }}>
            画像を選択してアップロード
          </Text>
        )}
      </TouchableOpacity>

      {uploading && (
        <Text style={{ color: '#666' }}>アップロード中... {progress}%</Text>
      )}
    </View>
  )
}
```

## Step 4: 動画ファイルのアップロード対応

画像だけでなく動画ファイルも扱う場合は、マルチパートアップロードが必要になることがあります。Cloudflare R2 は最大 5GB までのオブジェクトをサポートしており、それ以上の場合はマルチパートアップロードAPIを使用します。

```typescript
// utils/videoUpload.ts
import * as DocumentPicker from 'expo-document-picker'
import { uploadFileToR2 } from './fileUpload'

export async function pickAndUploadVideo(
  onProgress?: (percent: number) => void
): Promise<string | null> {
  const result = await DocumentPicker.getDocumentAsync({
    type: 'video/*',
    copyToCacheDirectory: true,
  })

  if (result.canceled || !result.assets[0]) return null

  const asset = result.assets[0]

  // ファイルサイズチェック（10MB制限の場合）
  if (asset.size && asset.size > 10 * 1024 * 1024) {
    throw new Error('動画ファイルは10MB以下にしてください。')
  }

  onProgress?.(20)

  const { url } = await uploadFileToR2(
    asset.uri,
    asset.name,
    asset.mimeType || 'video/mp4'
  )

  onProgress?.(100)
  return url
}
```

## Step 5: セキュリティと本番環境への準備

開発段階では CORS をワイルドカード（`*`）で設定しましたが、本番環境では必ずオリジンを絞ってください。

```typescript
// 本番用CORS設定の例
app.use('*', cors({
  origin: (origin) => {
    // アプリの特定URLのみ許可（Expo Go の開発時URLも含める）
    const allowedOrigins = [
      'https://myapp.com',
      'exp://localhost:8081',
    ]
    return allowedOrigins.includes(origin) ? origin : null
  },
}))
```

また、悪意のあるファイルアップロードを防ぐため、ファイル種別・サイズのバリデーションを必ず実装してください。Worker で受け取ったファイルの Content-Type を検証し、許可されていない種別は 400 エラーを返すのが基本です。

認証が必要なアプリの場合は、アップロードエンドポイントに JWT 検証を追加します。Supabase Auth や Clerk を使っている場合は、リクエストヘッダーに `Authorization: Bearer <token>` を含めて Worker 側でトークンを検証する実装を追加しましょう。

## Step 6: R2 の公開URL設定（CDN配信）

アップロードしたファイルを公開アクセス可能にするには、R2バケットに「パブリックアクセス」を設定します。

Cloudflareダッシュボードでバケットを開き、「設定」タブから「パブリックアクセスを許可する」をオンにすると、`https://pub-xxxxx.r2.dev/` というURLでファイルにアクセスできます。カスタムドメインを設定すると、`https://files.myapp.com/` のような独自URLでの配信も可能です。

Cloudflare CDN と組み合わせることで、世界中のユーザーに対して高速なファイル配信が実現します。キャッシュルールを適切に設定し、画像や動画の配信コストを最小化しましょう。

## よくあるエラーと対処法

**CORS エラーが発生する**

Worker の `cors()` ミドルウェアが適用されていない、またはオリジンの設定が誤っている場合が多いです。開発時は `origin: '*'` に設定して動作確認し、本番環境では正しいオリジンに絞り込んでください。

**アップロードが途中で失敗する**

モバイル環境ではネットワークが不安定になりやすいため、リトライロジックを実装することを推奨します。`expo-network` でネットワーク状態を事前にチェックし、オフライン時はアップロードをキューに積んで後で再試行する設計が堅牢です。

**R2 バインディングが見つからないエラー**

`wrangler.toml` の `[[r2_buckets]]` セクションの `binding` 名と、TypeScript コード内で参照している名前が一致していることを確認してください。デプロイ後に wrangler のログを確認し、バインディングが正しく認識されているか確認します。

**ファイルサイズが大きすぎる**

Worker のリクエストボディには 100MB の上限があります（Enterprise プランでは緩和可能）。大容量ファイルの場合は、R2 の直接マルチパートアップロードAPIを使うか、動画の場合は Cloudflare Stream への移行を検討してください。

## FAQ

**Q: Supabase Storage と Cloudflare R2 はどちらを選ぶべきですか？**

A: バックエンドを Supabase で統一している場合は Supabase Storage が便利で、Row Level Security と連携したきめ細かなアクセス制御が可能です。一方、バックエンドに Cloudflare Workers を使っている場合や、大量のデータ転送が見込まれる場合は R2 の方がコスト効率が高いです。月10GBの無料枠内であればR2が最もコスパに優れます。

**Q: 画像の圧縮や変換は R2 側でできますか？**

A: R2 自体には変換機能はありませんが、Cloudflare Images（別サービス）と組み合わせることでWebP変換やリサイズ、トリミングを自動化できます。簡単な圧縮処理であれば、Rorkアプリ側で `expo-image-picker` の `quality` オプションや、Worker 内で Canvas API を使った変換が可能です。

**Q: R2 の費用はどのくらいかかりますか？**

A: 月10GBのストレージと100万回のClass Bオペレーション（読み取り）、100万回のClass Aオペレーション（書き込み）が無料です。超過分はストレージ1GBあたり$0.015/月、Class Aオペレーション100万回あたり$4.50です。エグレス（データ転送）費用は無料なので、配信量が多いアプリほどAWS S3より大幅にコストを削減できます。
