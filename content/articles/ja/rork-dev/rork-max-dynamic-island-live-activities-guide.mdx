## Dynamic Island と Live Activities が注目される理由

iPhone 14 Pro 以降に搭載された Dynamic Island は、通知やリアルタイム情報をカメラ周辺のピル型領域に表示する iOS の革新的な機能です。配車アプリのタイマー、スポーツのスコア速報、フードデリバリーの進捗状況など、ユーザーがアプリを開かなくても最新情報を確認できる体験を提供します。

これと連動する Live Activities は、ロック画面にもリアルタイム情報を表示するしくみで、ActivityKit フレームワークを通じてアプリから制御します。iOS 16.1 で導入されてから急速に普及し、2026年現在では多くの人気アプリが採用しています。

Rork Max を使えば、Xcode を直接操作せずとも Swift ネイティブコードを生成し、Dynamic Island と Live Activities に対応したアプリを構築できます。この記事では、ActivityKit の基本概念から実装手順までを順を追って解説します。

## ActivityKit の基本構造を理解する

Live Activities を実装するには、Apple の ActivityKit フレームワークを使います。まず押さえておきたいのは、3つの主要コンポーネントです。

**ActivityAttributes**: Live Activity で表示するデータの型を定義するプロトコルです。静的データ（変化しない情報）と動的データ（リアルタイムで更新される情報）を分けて宣言します。

**ActivityContent**: 現在の状態を保持するオブジェクトで、更新のたびに新しい ContentState を渡します。

**Widget**: Dynamic Island とロック画面に表示する UI を SwiftUI で構築します。

以下は、フードデリバリーの配達追跡を例にした ActivityAttributes の定義です。

```swift
// DeliveryActivity.swift
import ActivityKit
import SwiftUI

struct DeliveryAttributes: ActivityAttributes {
    // 静的データ（アクティビティ開始後に変わらない情報）
    var restaurantName: String
    var orderNumber: String

    // 動的データ（リアルタイムで更新される情報）
    struct ContentState: Codable, Hashable {
        var status: DeliveryStatus
        var estimatedMinutes: Int
        var driverName: String
    }
}

enum DeliveryStatus: String, Codable {
    case preparing  // 準備中
    case pickedUp   // ピックアップ完了
    case onTheWay   // 配達中
    case arriving   // まもなく到着
    case delivered   // 配達完了
}
// 期待される動作: ActivityAttributes に準拠した構造体が
// Live Activity のデータモデルとして機能する
```

`restaurantName` や `orderNumber` はアクティビティ開始時に固定され、`ContentState` 内の `status` や `estimatedMinutes` がリアルタイムに変化します。

## Rork Max で Widget Extension を構成する

Rork Max では、プロンプトに「Live Activities 対応の Widget Extension を追加して」と指示するだけで、必要なファイル群を自動生成できます。生成されるファイル構成は以下のとおりです。

- `DeliveryActivityWidget.swift` — Widget のエントリーポイント
- `DeliveryLiveActivityView.swift` — ロック画面用の UI
- `DeliveryDynamicIslandView.swift` — Dynamic Island 用の UI
- `Info.plist` — `NSSupportsLiveActivities` を `YES` に設定

ロック画面用の Live Activity UI は、次のように SwiftUI で構築します。

```swift
// DeliveryLiveActivityView.swift
import SwiftUI
import WidgetKit
import ActivityKit

struct DeliveryLiveActivityWidget: Widget {
    var body: some WidgetConfiguration {
        ActivityConfiguration(for: DeliveryAttributes.self) { context in
            // ロック画面に表示される UI
            VStack(alignment: .leading, spacing: 8) {
                HStack {
                    Text(context.attributes.restaurantName)
                        .font(.headline)
                        .foregroundColor(.primary)
                    Spacer()
                    Text("注文 #\(context.attributes.orderNumber)")
                        .font(.caption)
                        .foregroundColor(.secondary)
                }

                // 進捗バー
                ProgressView(value: progressValue(for: context.state.status))
                    .tint(.green)

                HStack {
                    Label(statusText(for: context.state.status),
                          systemImage: statusIcon(for: context.state.status))
                        .font(.subheadline)
                    Spacer()
                    Text("あと約\(context.state.estimatedMinutes)分")
                        .font(.subheadline)
                        .bold()
                }
            }
            .padding()
            .activityBackgroundTint(.white.opacity(0.9))
        } dynamicIsland: { context in
            // Dynamic Island の定義（次のセクションで解説）
            DynamicIsland {
                DynamicIslandExpandedRegion(.leading) {
                    Label(context.state.driverName, systemImage: "car.fill")
                        .font(.caption)
                }
                DynamicIslandExpandedRegion(.trailing) {
                    Text("あと\(context.state.estimatedMinutes)分")
                        .font(.caption)
                        .bold()
                }
                DynamicIslandExpandedRegion(.bottom) {
                    ProgressView(value: progressValue(for: context.state.status))
                        .tint(.green)
                }
            } compactLeading: {
                Image(systemName: statusIcon(for: context.state.status))
            } compactTrailing: {
                Text("\(context.state.estimatedMinutes)分")
                    .font(.caption)
            } minimal: {
                Image(systemName: "bag.fill")
            }
        }
    }

    // ステータスに応じた進捗値を返す
    private func progressValue(for status: DeliveryStatus) -> Double {
        switch status {
        case .preparing: return 0.2
        case .pickedUp: return 0.4
        case .onTheWay: return 0.6
        case .arriving: return 0.8
        case .delivered: return 1.0
        }
    }

    private func statusText(for status: DeliveryStatus) -> String {
        switch status {
        case .preparing: return "お店で準備中"
        case .pickedUp: return "ピックアップ完了"
        case .onTheWay: return "配達中"
        case .arriving: return "まもなく到着"
        case .delivered: return "配達完了"
        }
    }

    private func statusIcon(for status: DeliveryStatus) -> String {
        switch status {
        case .preparing: return "takeoutbag.and.cup.and.straw"
        case .pickedUp: return "bag.fill.badge.checkmark"
        case .onTheWay: return "car.fill"
        case .arriving: return "mappin.and.ellipse"
        case .delivered: return "checkmark.circle.fill"
        }
    }
}
// 期待される動作: ロック画面にレストラン名・ステータス・残り時間が
// プログレスバー付きで表示される
```

## Dynamic Island の3つの表示モードを設計する

Dynamic Island には、ユーザーの操作状態に応じた3つの表示モードがあります。

**Compact（コンパクト）**: アプリがバックグラウンドにあるときのデフォルト表示です。ピル型の左右に分かれた狭いスペースに、アイコンや短いテキストを配置します。上のコード例では `compactLeading` と `compactTrailing` で定義しています。

**Expanded（拡張）**: ユーザーが Dynamic Island を長押ししたときに表示される広い領域です。`leading`、`trailing`、`center`、`bottom` の4つのリージョンに情報を配置できます。

**Minimal（ミニマル）**: 他のアプリの Live Activity と同時に表示される場合の最小表示です。アイコン1つ程度のスペースしかないため、最も重要な情報だけを表示します。

デザインの指針として、Apple の Human Interface Guidelines では以下が推奨されています。

- Compact モードでは、左にブランドアイコン・右に最重要の数値を配置する
- Expanded モードでは、ユーザーが一目で状況を把握できるレイアウトにする
- アニメーションは控えめにし、情報の視認性を最優先にする
- ダークモード・ライトモード両方で見やすい配色を使う

## アプリ本体から Live Activity を開始・更新・終了する

Widget Extension で UI を定義したら、アプリ本体のコードから Live Activity のライフサイクルを制御します。

```swift
// DeliveryManager.swift
import ActivityKit

class DeliveryManager: ObservableObject {
    @Published var currentActivity: Activity<DeliveryAttributes>?

    // Live Activity を開始する
    func startTracking(restaurant: String, orderNumber: String, driver: String) {
        // デバイスが Live Activities をサポートしているか確認
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            print("Live Activities は無効です")
            return
        }

        let attributes = DeliveryAttributes(
            restaurantName: restaurant,
            orderNumber: orderNumber
        )
        let initialState = DeliveryAttributes.ContentState(
            status: .preparing,
            estimatedMinutes: 30,
            driverName: driver
        )

        do {
            let activity = try Activity.request(
                attributes: attributes,
                content: .init(state: initialState, staleDate: nil),
                pushType: .token  // プッシュ通知で更新する場合
            )
            currentActivity = activity
            print("Live Activity 開始: \(activity.id)")
        } catch {
            print("Live Activity の開始に失敗: \(error)")
        }
    }

    // Live Activity を更新する
    func updateStatus(newStatus: DeliveryStatus, minutes: Int, driver: String) async {
        guard let activity = currentActivity else { return }

        let updatedState = DeliveryAttributes.ContentState(
            status: newStatus,
            estimatedMinutes: minutes,
            driverName: driver
        )

        await activity.update(
            ActivityContent(state: updatedState, staleDate: nil)
        )
    }

    // Live Activity を終了する
    func completeDelivery() async {
        guard let activity = currentActivity else { return }

        let finalState = DeliveryAttributes.ContentState(
            status: .delivered,
            estimatedMinutes: 0,
            driverName: ""
        )

        await activity.end(
            ActivityContent(state: finalState, staleDate: nil),
            dismissalPolicy: .after(.now + 300)  // 5分後に自動消去
        )
        currentActivity = nil
    }
}
// 期待される動作:
// startTracking() → ロック画面と Dynamic Island に配達情報が表示される
// updateStatus() → 表示内容がリアルタイムに更新される
// completeDelivery() → 「配達完了」表示後、5分で自動消去される
```

`pushType: .token` を指定すると、サーバーからの APNs（Apple Push Notification service）プッシュ通知でリモート更新が可能になります。これにより、アプリがバックグラウンドにあってもサーバー側から状態を更新できます。

## プッシュ通知によるリモート更新を設定する

ユーザーがアプリを閉じていても Live Activity を更新するには、APNs を利用したプッシュ更新が必要です。サーバー側から更新リクエストを送る流れは以下のとおりです。

```swift
// プッシュトークンの取得（アプリ側）
func observePushToken() {
    guard let activity = currentActivity else { return }

    Task {
        for await tokenData in activity.pushTokenUpdates {
            let token = tokenData.map { String(format: "%02x", $0) }.joined()
            print("Push Token: \(token)")
            // このトークンをバックエンドサーバーに送信する
            await sendTokenToServer(token: token, activityId: activity.id)
        }
    }
}

private func sendTokenToServer(token: String, activityId: String) async {
    // サーバーへトークンを登録
    let payload: [String: Any] = [
        "activity_id": activityId,
        "push_token": token
    ]
    // YOUR_API_KEY を自身のサーバーエンドポイントに置き換える
    // POST https://your-api.example.com/register-token
    print("トークンをサーバーに登録: \(payload)")
}
// 期待される動作: Live Activity 開始後、APNs トークンが生成され
// サーバーに自動送信される
```

サーバー側では、受け取ったトークンを使って APNs に更新リクエストを送ります。Rork Max で Supabase Edge Functions と組み合わせれば、バックエンド処理もコード生成で対応可能です。

サーバーから送る APNs ペイロードの形式は以下のとおりです。

```json
{
  "aps": {
    "timestamp": 1711756800,
    "event": "update",
    "content-state": {
      "status": "onTheWay",
      "estimatedMinutes": 12,
      "driverName": "田中太郎"
    }
  }
}
```

ヘッダーに `apns-push-type: liveactivity` と `apns-topic: {BundleID}.push-type.liveactivity` を設定する点に注意してください。

## よくあるエラーと対処法

Live Activities の実装では、いくつかのつまずきポイントがあります。

**`NSSupportsLiveActivities` の設定忘れ**: Widget Extension の `Info.plist` に `NSSupportsLiveActivities = YES` を追加しないと、`Activity.request()` が無効と判定されます。Rork Max のプロンプトで「Info.plist に Live Activities のサポートフラグを追加して」と明示するのが確実です。

**ActivityAttributes が Widget とアプリで不一致**: ActivityAttributes の定義はアプリターゲットと Widget Extension ターゲットの両方で共有する必要があります。共通のフレームワークターゲットを作るか、同じファイルを両ターゲットに追加してください。

**シミュレータで Dynamic Island が表示されない**: Dynamic Island は iPhone 14 Pro 以降のシミュレータでのみ表示されます。Xcode のシミュレータ設定で対応機種を選択するか、Rork Companion アプリを使って実機で確認してください。

**更新頻度の制限**: Apple は Live Activity の更新を1時間あたり一定回数に制限しています。頻繁すぎる更新はシステムによってスロットリングされるため、重要な状態変化のみ更新するよう設計してください。

## FAQ

**Q: Dynamic Island に対応していない iPhone でも Live Activities は使えますか？**

はい。Live Activities はロック画面での表示が基本機能で、iPhone 14 Pro より前のモデル（iOS 16.1 以降）でもロック画面に表示されます。Dynamic Island はピル型のハードウェアを持つ機種でのみ追加表示されるため、非対応機種では自動的にロック画面表示のみになります。

**Q: Rork Max のプロンプトでどう指示すれば Live Activities を生成できますか？**

「フードデリバリー追跡の Live Activities を ActivityKit で実装して。Dynamic Island の Compact・Expanded・Minimal すべてのモードに対応させて」のように、具体的なユースケースと必要な表示モードを明記すると精度が上がります。データモデルの項目（表示したい情報の種類）も添えると、より完成度の高いコードが生成されます。

**Q: Live Activities の表示はどのくらいの時間持続しますか？**

Apple の仕様では、Live Activity は開始から最大8時間まで表示できます。8時間を超えると自動的に終了します。また、`staleDate` を設定すると、指定した時刻以降は「情報が古い可能性がある」旨が表示されます。アプリからの明示的な終了（`activity.end()`）には `dismissalPolicy` で消去タイミングを制御できます。
