## Dynamic Island と Live Activities とは？

iPhone 14 Pro 以降に搭載された **Dynamic Island** は、画面上部のカメラ周辺の「島」型UI領域です。アプリがバックグラウンドで動作していても、リアルタイムの情報を視覚的に表示できます。

**Live Activities** は iOS 16.1 から導入されたフレームワークで、ロック画面や Dynamic Island に最大8時間リアルタイム情報を表示する仕組みです。代表的なユースケースとしては次のものがあります。

- 配達・ライドシェアの追跡（現在地・到着時刻）
- スポーツ試合のスコア速報
- フードデリバリーの注文状況
- 音楽・ポッドキャストの再生コントロール
- 運動・タイマーアプリの進捗表示

Rork（React Native ベース）の標準機能では Dynamic Island や Live Activities を直接扱うことができません。しかし **Rork Max** を使えば Swift/SwiftUI のネイティブコードを生成・実行できるため、これらの機能を完全に実装できます。

[Rork と Rork Max の違い](/articles/rork-basics/rork-vs-rork-max) については別記事で詳しく解説していますが、簡単に言えば Rork Max はクラウド上の Mac を利用して本格的な Swift ネイティブコードを生成・ビルドできるプランです。

## なぜ Rork Max が必要なのか

Dynamic Island と Live Activities の実装には以下の技術スタックが必要です。

- **WidgetKit**: Widget Extension を定義するフレームワーク
- **ActivityKit**: Live Activities のライフサイクルを管理するフレームワーク
- **SwiftUI**: Dynamic Island のコンパクト・拡張表示を構築するUI

これらはすべて Apple 独自のフレームワークであり、React Native（JavaScript/TypeScript）から直接呼び出すことはできません。Rork Max の場合、Claude AI がプロンプトを解釈して SwiftUI コードと Widget Extension を自動生成し、Xcode なしでビルド・TestFlight 配布まで実行してくれます。

## ステップ 1：ActivityAttributes を定義する

Live Activities の核心となるのが `ActivityAttributes` プロトコルです。これはリアルタイムで更新される「動的データ」と、起動時のみ設定される「静的データ」を分けて定義します。

以下は配達追跡アプリの例です。

```swift
import ActivityKit
import Foundation

// 配達追跡用の ActivityAttributes
struct DeliveryAttributes: ActivityAttributes {
    // 動的データ（リアルタイムで更新される）
    public struct ContentState: Codable, Hashable {
        var driverName: String      // ドライバー名
        var estimatedArrival: Date  // 到着予定時刻
        var currentStatus: String   // 現在のステータス
    }

    var orderId: String             // 注文番号（静的）
    var restaurantName: String      // レストラン名（静的）
}
```

Rork Max のプロンプト例:

> 「配達追跡アプリの Live Activities を実装してください。ドライバー名、到着予定時刻、現在のステータス（準備中・配達中・到着）を Dynamic Island に表示します」

これだけで Rork Max が `ActivityAttributes` の定義から Widget Extension のUI、アプリ本体の起動コードまで一括生成します。

## ステップ 2：Dynamic Island の表示UIを作る

Dynamic Island には3つの表示モードがあります。

**コンパクト表示**（通知バーのような小さな表示）

```swift
.compactLeading {
    Image(systemName: "bicycle")
        .foregroundColor(.orange)
}
.compactTrailing {
    Text(context.state.estimatedArrival, style: .timer)
        .monospacedDigit()
        .foregroundColor(.white)
        .font(.caption)
}
```

**ミニマル表示**（他のアプリと Dynamic Island を共有する場合）

```swift
.minimal {
    Image(systemName: "bicycle.circle.fill")
        .foregroundColor(.orange)
}
```

**拡張表示**（ユーザーが Dynamic Island を長押しした場合）

```swift
.expanded {
    HStack {
        VStack(alignment: .leading, spacing: 4) {
            Text("配達中")
                .font(.caption)
                .foregroundColor(.secondary)
            Text(context.attributes.restaurantName)
                .font(.headline)
                .fontWeight(.bold)
        }
        Spacer()
        VStack(alignment: .trailing, spacing: 4) {
            Text("到着予定")
                .font(.caption)
                .foregroundColor(.secondary)
            Text(context.state.estimatedArrival, style: .timer)
                .font(.title2)
                .fontWeight(.bold)
                .foregroundColor(.orange)
        }
    }
    .padding()
}
```

## ステップ 3：Live Activities を開始・更新・終了する

アプリ本体から Live Activity を制御するコードです。

```swift
import ActivityKit

// Live Activity を開始する
func startDeliveryActivity(orderId: String, restaurant: String, driverName: String) {
    // ActivityKit が利用可能か確認
    guard ActivityAuthorizationInfo().areActivitiesEnabled else {
        print("Live Activities が無効です")
        return
    }

    let attributes = DeliveryAttributes(
        orderId: orderId,
        restaurantName: restaurant
    )

    let initialState = DeliveryAttributes.ContentState(
        driverName: driverName,
        estimatedArrival: Date().addingTimeInterval(1800), // 30分後
        currentStatus: "準備中"
    )

    do {
        let activity = try Activity.request(
            attributes: attributes,
            content: .init(state: initialState, staleDate: nil),
            pushType: nil // プッシュ通知での更新を使わない場合は nil
        )
        print("Live Activity 開始: \(activity.id)")
    } catch {
        print("開始エラー: \(error)")
    }
}

// リアルタイムで状態を更新する
func updateDeliveryStatus(activity: Activity<DeliveryAttributes>, newStatus: String, minutesLeft: Double) {
    let updatedState = DeliveryAttributes.ContentState(
        driverName: activity.content.state.driverName,
        estimatedArrival: Date().addingTimeInterval(minutesLeft * 60),
        currentStatus: newStatus
    )

    Task {
        await activity.update(.init(state: updatedState, staleDate: nil))
    }
}

// Live Activity を終了する
func endDeliveryActivity(activity: Activity<DeliveryAttributes>) {
    Task {
        await activity.end(nil, dismissalPolicy: .immediate)
    }
}
```

## プッシュ通知でサーバーから更新する

サーバーサイドからリアルタイムで Live Activity を更新するには、Push to Start / Push to Update を利用します。

```swift
// Activity 起動時にプッシュトークンを取得する
let activity = try Activity.request(
    attributes: attributes,
    content: .init(state: initialState, staleDate: nil),
    pushType: .token  // プッシュ通知を有効化
)

// プッシュトークンをサーバーに送信
for await pushToken in activity.pushTokenUpdates {
    let tokenString = pushToken.map { String(format: "%02x", $0) }.joined()
    // サーバーの API にトークンを POST する
    await sendTokenToServer(token: tokenString, activityId: activity.id)
}
```

サーバー側からの更新は Apple Push Notification service（APNs）を通じて行います。Rork Max のプロンプトで「サーバーサイドプッシュで Live Activities を更新する実装も含めて」と伝えると、Cloudflare Workers のバックエンドコードも一緒に生成してくれます。

## よくあるエラーと対処法

Dynamic Island・Live Activities の実装でよく遭遇するエラーをまとめました。

**エラー 1: ActivityKit.ActivityAuthorizationError**

`Info.plist` に `NSSupportsLiveActivities = true` が設定されていないことが原因です。Rork Max のプロンプトで「Info.plist に Live Activities を許可する設定を追加して」と指示してください。

**エラー 2: Live Activity がシミュレーターで動作しない**

Dynamic Island と Live Activities は実機でのみ動作します。Rork Max の TestFlight ビルドを使って実機テストを行ってください。プッシュ通知ベースの更新もシミュレーターでは確認できません。

**エラー 3: 表示が更新されない**

Live Activity の更新は Apple のガイドラインで1時間あたり最大15回に制限されています。頻繁な更新が必要な場合は Push to Update（サーバーからの APNs）を使用してください。

**エラー 4: 拡張表示でレイアウトが崩れる**

Dynamic Island の拡張表示は端末サイズによって表示領域が異なります。SwiftUI の `GeometryReader` を使って動的にサイズを取得し、レイアウトを調整してください。

## よくある質問（FAQ）

**Q. Dynamic Island は iPhone のどの機種から対応していますか？**

Dynamic Island は iPhone 14 Pro / 14 Pro Max 以降（iPhone 15 以降は全モデル）で利用できます。iPhone 14（通常モデル）以前の端末では、ロック画面のみに Live Activities が表示されます（Dynamic Island の表示なし）。

**Q. Live Activities の最大表示時間はどのくらいですか？**

Apple のガイドラインでは最大8時間です。8時間を超えると自動的に終了します。なお、ユーザーが手動でスワイプして消した場合もその時点で終了します。アプリ側で明示的に `activity.end()` を呼ぶことが推奨されます。

**Q. Rork（通常プラン）でも Dynamic Island は使えますか？**

React Native ベースの Rork では Dynamic Island・Live Activities の実装はできません。Swift ネイティブコードを生成できる **Rork Max** が必要です。詳しくは [Rork と Rork Max の違い](/articles/rork-basics/rork-vs-rork-max) をご確認ください。

**Q. Live Activities で外部URLの画像を表示できますか？**

WidgetKit の制約により、外部URLからの非同期画像読み込みはできません。アプリバンドル内の画像（`Image("asset-name")`）またはSFシンボルのみ使用できます。

**Q. Android でも Dynamic Island のような表示を実装できますか？**

Android には Dynamic Island に相当する標準UIはありません。Android の場合はカスタム通知（`Notification.Builder`）を使って類似の表示を実現します。Rork Max はiOSネイティブに特化しているため、Android 向けには React Native の通知ライブラリを使うアプローチがおすすめです。

## まとめ

Rork Max を使えば、Dynamic Island と Live Activities という本格的な iOS ネイティブ機能をプロンプトだけで実装できます。開発のポイントをまとめます。

- `ActivityAttributes` で動的データ（ContentState）と静的データを分けて定義する
- Dynamic Island の3つの表示モード（コンパクト・ミニマル・拡張）それぞれに SwiftUI の View を用意する
- アプリ本体から `Activity.request()` → `activity.update()` → `activity.end()` でライフサイクルを管理する
- サーバーからリアルタイム更新する場合は Push to Update（APNs）を活用する
- テストは必ず実機で行う（シミュレーター非対応）

Rork Max であれば、Xcode を一切インストールすることなく、上記の Swift コードを AI が自動生成し、TestFlight 経由で実機配布・App Store 申請まで完結できます。さらに詳しい App Store 公開の流れについては [Rork Max の2クリック App Store 公開完全ガイド](/articles/rork-business/rork-max-2-click-publish-complete-guide) で解説しています。

SwiftUI の実装をより深く理解したい方には、[「SwiftUI徹底入門」（藤治 仁 他 著）](https://www.amazon.co.jp/dp/4798167029/?tag=pinocchio-22) もあわせておすすめです。Dynamic Island の SwiftUI レイアウト設計に直接役立つ知識が体系的に学べます。
