## はじめに — なぜアプリにチャートが必要なのか

アプリ開発において、数値データをただテキストで表示するだけでは、ユーザーに本当の「気づき」を与えることはできません。家計簿アプリの月別支出、フィットネスアプリの運動記録、ビジネスアプリの売上推移——こうしたデータは、チャートやグラフとして可視化することで、はじめてユーザーの意思決定を助ける「情報」に変わります。

この記事では、Rork アプリで **Victory Native** を使ってインタラクティブなチャート＆グラフを実装する方法を、実際のダッシュボード画面を構築しながら解説します。折れ線グラフ、棒グラフ、円グラフの3種類を取り上げ、タッチ操作でツールチップを表示するインタラクション実装まで踏み込みます。

**この記事の対象読者:**

- Rork の基本操作に慣れている方（[Rork をはじめよう：5分で最初のアプリを作る](/articles/rork-basics/getting-started) を完了済み）
- アプリにデータ表示機能を追加したい方
- ダッシュボード画面を構築したい方

**前提環境:**

- Rork（無料プランでもOK、Rork Max 推奨）
- Node.js 18 以上
- React Native / Expo の基礎知識

## Victory Native とは — React Native 向けチャートライブラリの定番

Victory Native は、Formidable 社が開発したオープンソースのチャートライブラリです。Web 版の Victory をベースに、React Native 環境で動作するよう最適化されています。

**Victory Native を選ぶ理由:**

- React Native に最適化されたパフォーマンス（Skia レンダリング対応）
- 折れ線・棒・円・散布図・エリアなど豊富なチャート種類
- タッチイベント・アニメーション対応
- TypeScript 完全サポート
- Expo との高い互換性

まずはプロジェクトにインストールしましょう。

```bash
# Victory Native のインストール
npx expo install victory-native react-native-svg

# Skia レンダリングを使う場合（高パフォーマンス）
npx expo install @shopify/react-native-skia
```

## 折れ線グラフの実装 — 売上推移を可視化する

ダッシュボードの定番である折れ線グラフから始めます。ここでは月別の売上データを表示する例を実装します。

```tsx
// components/SalesLineChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import {
  VictoryChart,
  VictoryLine,
  VictoryAxis,
  VictoryTheme,
  VictoryVoronoiContainer,
  VictoryTooltip,
} from "victory-native";

// 月別売上データ（単位: 万円）
const salesData = [
  { month: "1月", sales: 120 },
  { month: "2月", sales: 145 },
  { month: "3月", sales: 132 },
  { month: "4月", sales: 178 },
  { month: "5月", sales: 195 },
  { month: "6月", sales: 210 },
];

export default function SalesLineChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>月別売上推移</Text>
      <VictoryChart
        theme={VictoryTheme.material}
        height={280}
        // VoronoiContainer でタッチ時にツールチップを表示
        containerComponent={
          <VictoryVoronoiContainer
            labels={({ datum }) => `${datum.sales}万円`}
            labelComponent={<VictoryTooltip />}
          />
        }
      >
        {/* X軸: 月名 */}
        <VictoryAxis
          tickValues={salesData.map((_, i) => i + 1)}
          tickFormat={salesData.map((d) => d.month)}
          style={{ tickLabels: { fontSize: 11 } }}
        />
        {/* Y軸: 売上金額 */}
        <VictoryAxis
          dependentAxis
          tickFormat={(t: number) => `${t}万`}
          style={{ tickLabels: { fontSize: 11 } }}
        />
        {/* 折れ線 */}
        <VictoryLine
          data={salesData}
          x={(d) => salesData.indexOf(d) + 1}
          y="sales"
          style={{
            data: { stroke: "#4F46E5", strokeWidth: 3 },
          }}
          animate={{ duration: 500 }}
        />
      </VictoryChart>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12 },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8 },
});

// 期待する出力:
// 1月〜6月の売上推移が折れ線グラフで表示される
// タッチするとツールチップで「○○万円」と表示される
```

`VictoryVoronoiContainer` がポイントです。これにより、ユーザーがグラフ上のどこをタッチしても、最も近いデータポイントのツールチップが自動的に表示されます。

## 棒グラフの実装 — カテゴリ別の比較データ

次に、カテゴリ別の比較に適した棒グラフを実装します。ここではアプリのカテゴリ別ダウンロード数を表示します。

```tsx
// components/DownloadsBarChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import {
  VictoryChart,
  VictoryBar,
  VictoryAxis,
  VictoryTheme,
} from "victory-native";

const downloadData = [
  { category: "ヘルスケア", downloads: 3200 },
  { category: "教育", downloads: 2800 },
  { category: "ファイナンス", downloads: 4100 },
  { category: "ゲーム", downloads: 5600 },
  { category: "SNS", downloads: 3900 },
];

export default function DownloadsBarChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>カテゴリ別ダウンロード数</Text>
      <VictoryChart
        theme={VictoryTheme.material}
        height={280}
        domainPadding={{ x: 30 }}
      >
        <VictoryAxis
          tickFormat={downloadData.map((d) => d.category)}
          style={{ tickLabels: { fontSize: 10, angle: -20 } }}
        />
        <VictoryAxis
          dependentAxis
          tickFormat={(t: number) =>
            t >= 1000 ? `${(t / 1000).toFixed(1)}K` : String(t)
          }
        />
        <VictoryBar
          data={downloadData}
          x="category"
          y="downloads"
          style={{
            data: {
              fill: ({ datum }) =>
                datum.downloads > 4000 ? "#10B981" : "#6366F1",
              borderRadius: 4,
            },
          }}
          animate={{ duration: 600, onLoad: { duration: 400 } }}
          barWidth={36}
        />
      </VictoryChart>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12 },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8 },
});

// 期待する出力:
// 5つのカテゴリが棒グラフで表示される
// 4000超のカテゴリは緑、それ以外は紫で色分けされる
```

`fill` に関数を渡すことで、データの値に応じて動的に色を変えられます。閾値を超えた項目をハイライトする、というビジュアルフィードバックは、ダッシュボードでは非常に効果的です。

## 円グラフの実装 — 構成比率の可視化

収益の内訳やユーザー属性の分布など、全体に対する構成比率を示すには円グラフが最適です。

```tsx
// components/RevenuePieChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import { VictoryPie, VictoryLabel } from "victory-native";

const revenueData = [
  { source: "サブスク", revenue: 45, color: "#6366F1" },
  { source: "広告", revenue: 25, color: "#10B981" },
  { source: "課金", revenue: 20, color: "#F59E0B" },
  { source: "その他", revenue: 10, color: "#94A3B8" },
];

export default function RevenuePieChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>収益構成比率</Text>
      <VictoryPie
        data={revenueData}
        x="source"
        y="revenue"
        colorScale={revenueData.map((d) => d.color)}
        labels={({ datum }) => `${datum.source}\n${datum.revenue}%`}
        labelComponent={
          <VictoryLabel style={{ fontSize: 11, fill: "#374151" }} />
        }
        innerRadius={60}
        padAngle={2}
        animate={{ duration: 500 }}
        height={280}
        width={320}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12, alignItems: "center" },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8, alignSelf: "flex-start" },
});

// 期待する出力:
// ドーナツ型の円グラフが表示される
// 各セグメントに「サブスク 45%」のようなラベルが付く
```

`innerRadius` を指定すると、中央が空洞のドーナツチャートになります。中央のスペースに合計値やKPIを表示するデザインパターンは、モバイルダッシュボードでよく使われます。

## ダッシュボード画面の組み立て — 3つのチャートを統合する

個々のチャートコンポーネントが完成したら、ダッシュボード画面として統合します。`ScrollView` でスクロール可能にし、KPI カードも追加しましょう。

```tsx
// screens/DashboardScreen.tsx
import React from "react";
import { ScrollView, View, Text, StyleSheet } from "react-native";
import SalesLineChart from "../components/SalesLineChart";
import DownloadsBarChart from "../components/DownloadsBarChart";
import RevenuePieChart from "../components/RevenuePieChart";

function KpiCard({ label, value, trend }: {
  label: string;
  value: string;
  trend: string;
}) {
  const isPositive = trend.startsWith("+");
  return (
    <View style={styles.kpiCard}>
      <Text style={styles.kpiLabel}>{label}</Text>
      <Text style={styles.kpiValue}>{value}</Text>
      <Text style={[styles.kpiTrend, { color: isPositive ? "#10B981" : "#EF4444" }]}>
        {trend}
      </Text>
    </View>
  );
}

export default function DashboardScreen() {
  return (
    <ScrollView style={styles.screen} contentContainerStyle={styles.content}>
      <Text style={styles.heading}>ダッシュボード</Text>

      {/* KPI カード */}
      <View style={styles.kpiRow}>
        <KpiCard label="月間売上" value="¥2,100,000" trend="+8.2%" />
        <KpiCard label="DAU" value="12,340" trend="+3.1%" />
        <KpiCard label="解約率" value="2.4%" trend="-0.5%" />
      </View>

      {/* チャートセクション */}
      <SalesLineChart />
      <View style={styles.spacer} />
      <DownloadsBarChart />
      <View style={styles.spacer} />
      <RevenuePieChart />
    </ScrollView>
  );
}

const styles = StyleSheet.create({
  screen: { flex: 1, backgroundColor: "#F3F4F6" },
  content: { padding: 16, paddingBottom: 40 },
  heading: { fontSize: 24, fontWeight: "800", marginBottom: 16 },
  kpiRow: { flexDirection: "row", justifyContent: "space-between", marginBottom: 16 },
  kpiCard: { flex: 1, backgroundColor: "#fff", borderRadius: 12, padding: 12, marginHorizontal: 4 },
  kpiLabel: { fontSize: 12, color: "#6B7280" },
  kpiValue: { fontSize: 20, fontWeight: "700", marginTop: 4 },
  kpiTrend: { fontSize: 12, marginTop: 2 },
  spacer: { height: 16 },
});

// 期待する出力:
// KPI カード3枚 + 折れ線・棒・円グラフが縦スクロールで表示される
```

## リアルタイムデータとの連携 — Supabase からデータを取得する

静的データではなく、サーバーからリアルタイムに取得したデータをチャートに反映する実装を見てみましょう。ここでは [Rork × Supabase Auth & Realtime ガイド](/articles/rork-dev/rork-supabase-auth-realtime-guide) で紹介した Supabase を使います。

```tsx
// hooks/useSalesData.ts — データ取得用カスタムフック
import { useState, useEffect } from "react";
import { supabase } from "../lib/supabase";

interface SalesRecord {
  month: string;
  sales: number;
}

export function useSalesData() {
  const [data, setData] = useState<SalesRecord[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetchSales() {
      const { data: records, error } = await supabase
        .from("monthly_sales")
        .select("month, sales")
        .order("month", { ascending: true });

      if (!error && records) {
        setData(records);
      }
      setLoading(false);
    }

    fetchSales();

    // リアルタイム更新をサブスクライブ
    const subscription = supabase
      .channel("sales-updates")
      .on(
        "postgres_changes",
        { event: "*", schema: "public", table: "monthly_sales" },
        () => fetchSales() // データ変更時に再取得
      )
      .subscribe();

    return () => {
      subscription.unsubscribe();
    };
  }, []);

  return { data, loading };
}

// 使い方:
// const { data, loading } = useSalesData();
// if (loading) return <ActivityIndicator />;
// <VictoryLine data={data} x="month" y="sales" />
```

カスタムフックでデータ取得ロジックを分離する方法については、[Rork アプリでカスタムフックを使いこなす](/articles/rork-dev/rork-custom-hooks-reusable-logic) も参考にしてください。

## よくあるエラーと対処法

### チャートが表示されない

react-native-svg がインストールされていない場合に発生します。`npx expo install react-native-svg` を実行してください。Expo Go では一部の SVG 機能に制限があるため、開発ビルド（`npx expo prebuild`）を検討しましょう。

### タッチイベントが反応しない

`ScrollView` 内にチャートを配置している場合、スクロールジェスチャーとチャートのタッチが競合することがあります。`VictoryChart` を `<View>` で囲み、`onStartShouldSetResponder={() => true}` を指定すると改善されます。

### データが多すぎてパフォーマンスが低下する

1,000 件以上のデータポイントを描画する場合は、Skia レンダリング (`@shopify/react-native-skia`) を導入するか、データを集約（日次→月次など）して表示する件数を減らしましょう。

## FAQ

### Q: Victory Native は無料で使えますか？

はい、Victory Native は MIT ライセンスのオープンソースライブラリです。商用アプリにも無料で利用できます。Rork の無料プランでも問題なく動作します。

### Q: Victory Native 以外のチャートライブラリも使えますか？

はい。react-native-chart-kit、react-native-gifted-charts、react-native-wagmi-charts なども Rork アプリで利用可能です。ただし、Victory Native はドキュメントの充実度、TypeScript サポート、カスタマイズ性のバランスが最も優れているため、多くの場合で最初の選択肢としておすすめです。

### Q: チャートのデザインをアプリのテーマに合わせるにはどうすればいいですか？

`VictoryTheme.material` の代わりにカスタムテーマオブジェクトを作成できます。フォント、色、軸のスタイルなどをすべてカスタマイズ可能です。`VictoryTheme.material` のソースコードをベースにして、アプリのデザインシステムに合わせて変更するのがおすすめです。
