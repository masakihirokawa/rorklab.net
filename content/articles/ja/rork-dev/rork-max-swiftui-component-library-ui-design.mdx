Rork Maxで生成されるアプリのUIは、デフォルトでも十分に使える。ダークモード対応、レスポンシブ対応、アクセシビリティ対応も自動で適用される。だが、App Storeのトップチャートに並ぶアプリには、一目で「このアプリは違う」と感じさせるUI品質がある。その差を生むのが、コンポーネントライブラリの理解と、意図的なカスタマイズだ。

多くの開発者は、Rork Maxが生成したUIをそのまま使い続ける。しかし、その下層にある40以上のSwiftUIコンポーネントを理解し、自分のブランドや設計意図に合わせて組み直すことで、初めて「自分のアプリ」になる。本記事では、Rork MaxのSwiftUIコンポーネントライブラリの全貌を解説し、実装の深さを次のレベルへ押し上げるための実践的手法を紹介する。

## Rork Max SwiftUIコンポーネントライブラリの全体マップ

Rork Maxは、Appleの公式設計言語「Human Interface Guidelines」に準拠したSwiftUIコンポーネント群を提供している。これらは大きく7つのカテゴリに分類される。

**1. レイアウトコンポーネント**（VStack, HStack, ZStack, LazyVGrid, LazyHGrid等）
- アプリケーションの骨格を形成するコンテナ
- Rork Maxは、サイズと子要素の数に基づいて自動的に最適なレイアウトを選択する設計

**2. テキストコンポーネント**（Text, Label, TextField, TextEditor等）
- タイポグラフィの一貫性を確保
- Dynamic Type対応により、ユーザーのアクセシビリティ設定に自動で追従

**3. ボタン・操作部品**（Button, Toggle, Picker, DatePicker, Slider等）
- Apple標準のUI/UXガイドラインに厳密に従う
- タッチ領域（最小44×44ポイント）が自動で確保される

**4. ナビゲーション**（NavigationStack, NavigationLink, TabView等）
- iOS 16以上のNavigationStack APIに対応
- 状態管理との統合が簡単

**5. コンテナ・オーバーレイ**（Sheet, Alert, popover, ConfirmationDialog等）
- モーダルとオーバーレイの表示/非表示を統一的に管理

**6. リストと表示**（List, ScrollView, LazyVStack等）
- パフォーマンス最適化が組み込まれている
- 大規模リスト（1,000以上の要素）でもフリーズしない

**7. 視覚効果**（Divider, ProgressView, GroupBox等）
- ユーザーの視覚的な流れを整える補助要素

Rork Maxがこれら40以上のコンポーネントを提供する理由は、**個別に最適化することで、組み合わせたときの総合的な品質が担保されるから**だ。素のSwiftUIでこれらを一から構築すると、ダークモード、Dynamic Type、アクセシビリティなどの複数の要件に対応する過程で、互いに矛盾する実装が生まれやすい。Rork Maxは、その統合を最初から考えた設計になっている。

## レイアウトの思考法：子要素の数と向きで選ぶ

SwiftUIのレイアウトシステムの最大の特徴は、「親要素が子要素に提案し、子要素がそれに応答する」という協調的な仕組みにある。これを理解することが、アプリケーション全体のUIバランスを決める。

**VStack（垂直積み重ね）を使うべき場面**：

```swift
VStack(alignment: .leading, spacing: 12) {
  Text("プロフィール")
    .font(.title2)
    .fontWeight(.bold)
  
  HStack(spacing: 12) {
    Image(systemName: "person.circle.fill")
      .font(.system(size: 48))
    VStack(alignment: .leading, spacing: 4) {
      Text("田中太郎")
        .fontWeight(.semibold)
      Text("@tanaka_taro")
        .font(.caption)
        .foregroundColor(.secondary)
    }
  }
  
  Divider()
  
  Text("自己紹介文がここに入ります。")
    .font(.body)
    .lineLimit(3)
}
.padding()
```

このコード例では、VStackが親として3つの子要素（タイトル、プロフィール行、自己紹介テキスト）を上から順に配置している。各要素の間隔は`spacing: 12`で統一される。このシンプルさが、デザインの一貫性を生む。

**HStack（水平配置）を使うべき場面**：

```swift
HStack(spacing: 8) {
  Image(systemName: "star.fill")
    .foregroundColor(.yellow)
  Text("4.8")
    .fontWeight(.semibold)
  Text("(2,341 reviews)")
    .font(.caption)
    .foregroundColor(.secondary)
}
```

横方向に並ぶ情報（アイコン、数値、サブテキスト）をまとめるときの定番。間隔の詰まり具合で、視覚的な親密性が大きく変わる。

重要なのは、**VStackとHStackを入れ子にするときの思考法**だ。プロフィールカードのように、画像（左）とテキスト（右）を配置したい場合、HStackの中にVStackを入れるのが自然だ。

```swift
HStack(alignment: .top, spacing: 12) {
  Image("user-avatar")
    .resizable()
    .scaledToFill()
    .frame(width: 60, height: 60)
    .clipShape(Circle())
  
  VStack(alignment: .leading, spacing: 4) {
    Text("ユーザー名")
      .fontWeight(.semibold)
    Text("フォロワー: 1,234")
      .font(.caption)
      .foregroundColor(.secondary)
  }
  
  Spacer()
}
```

このパターンでは、画像を固定サイズ（60×60）にして、テキストをその右隣に配置している。右端の`Spacer()`は、HStackの中で右方向の余白を吸収し、コンテンツを左寄せにする。このテクニックは非常に頻出だ。

## カスタムコンポーネント作成：再利用可能なUI単位を設計する

Rork Maxが提供するコンポーネントだけでは、アプリケーション固有のUIニーズを満たせないことが多い。ここで重要なのが、**カスタムコンポーネント化**だ。

例えば、多くのアプリで登場する「ステータスバッジ」を考えよう。これは「活動中」「オフライン」「通話中」などのステータスを視覚的に表現するコンポーネントだ。Rork Maxのデフォルトには存在しないので、自分で設計する必要がある。

```swift
struct StatusBadge: View {
  enum Status {
    case active
    case offline
    case busy
    case away
    
    var backgroundColor: Color {
      switch self {
      case .active:
        return Color.green
      case .offline:
        return Color.gray
      case .busy:
        return Color.red
      case .away:
        return Color.yellow
      }
    }
    
    var label: String {
      switch self {
      case .active:
        return "活動中"
      case .offline:
        return "オフライン"
      case .busy:
        return "通話中"
      case .away:
        return "一時離席"
      }
    }
  }
  
  let status: Status
  
  var body: some View {
    HStack(spacing: 6) {
      Circle()
        .fill(status.backgroundColor)
        .frame(width: 8, height: 8)
      
      Text(status.label)
        .font(.caption)
        .fontWeight(.medium)
    }
    .padding(.horizontal, 8)
    .padding(.vertical, 4)
    .background(status.backgroundColor.opacity(0.1))
    .cornerRadius(6)
  }
}
```

このコンポーネントは、ステータスを渡すだけで、背景色・テキスト・アイコンを自動で調整する。アプリ全体でStatusBadgeを使えば、ステータス表示の一貫性が保証される。

さらに重要なのは、このコンポーネントの**レスポンシブ対応**だ。後述するテーマシステムと組み合わせれば、ダークモード時に背景色の透明度や明度を自動で調整できる。

```swift
struct StatusBadge: View {
  @Environment(\.colorScheme) var colorScheme
  
  var body: some View {
    HStack(spacing: 6) {
      Circle()
        .fill(status.backgroundColor)
        .frame(width: 8, height: 8)
      
      Text(status.label)
        .font(.caption)
        .fontWeight(.medium)
    }
    .padding(.horizontal, 8)
    .padding(.vertical, 4)
    .background(
      status.backgroundColor
        .opacity(colorScheme == .dark ? 0.2 : 0.1)
    )
    .cornerRadius(6)
  }
}
```

`@Environment(\.colorScheme)` を使うことで、ユーザーのダークモード設定を自動で検出し、透明度を動的に変更する。ダークモードでは背景が濃いため、元の0.1では見えにくくなるので、0.2に上げている。

## テーマシステムの設計：カラーパレット・タイポグラフィ・スペーシングの統合管理

UI設計の品質を飛躍的に向上させるのが、**テーマシステム**だ。これは、アプリケーション全体で使う色・フォント・間隔を1ヶ所で定義し、全コンポーネントから参照する仕組みである。

```swift
struct AppTheme {
  // MARK: - Colors
  struct Colors {
    // Primary
    let primary = Color(red: 0.2, green: 0.5, blue: 0.95)
    let primaryLight = Color(red: 0.4, green: 0.65, blue: 1.0)
    let primaryDark = Color(red: 0.1, green: 0.35, blue: 0.8)
    
    // Semantic
    let success = Color(red: 0.2, green: 0.8, blue: 0.4)
    let warning = Color(red: 1.0, green: 0.6, blue: 0.2)
    let danger = Color(red: 0.9, green: 0.2, blue: 0.2)
    
    // Neutral
    let background = Color(UIColor { traitCollection in
      traitCollection.userInterfaceStyle == .dark
        ? UIColor(white: 0.12, alpha: 1)
        : UIColor(white: 0.97, alpha: 1)
    })
    let text = Color(UIColor { traitCollection in
      traitCollection.userInterfaceStyle == .dark
        ? UIColor(white: 0.95, alpha: 1)
        : UIColor(white: 0.1, alpha: 1)
    })
    let secondaryText = Color(UIColor { traitCollection in
      traitCollection.userInterfaceStyle == .dark
        ? UIColor(white: 0.7, alpha: 1)
        : UIColor(white: 0.5, alpha: 1)
    })
  }
  
  // MARK: - Typography
  struct Typography {
    let headline = Font.system(.title, design: .rounded)
      .weight(.bold)
    let subheadline = Font.system(.title3, design: .rounded)
      .weight(.semibold)
    let body = Font.system(.body)
    let caption = Font.system(.caption)
      .weight(.medium)
  }
  
  // MARK: - Spacing
  struct Spacing {
    let xs: CGFloat = 4
    let sm: CGFloat = 8
    let md: CGFloat = 12
    let lg: CGFloat = 16
    let xl: CGFloat = 20
    let xxl: CGFloat = 24
  }
  
  static let current = AppTheme()
}
```

このテーマは、`Environment`経由でアプリケーション全体に配布できる：

```swift
@main
struct MyApp: App {
  var body: some Scene {
    WindowGroup {
      ContentView()
        .environment(\.colorScheme, nil) // System設定に従う
    }
  }
}
```

そして、各ビューで参照する：

```swift
struct HomeView: View {
  let theme = AppTheme.current
  
  var body: some View {
    VStack(spacing: theme.spacing.lg) {
      Text("ホーム")
        .font(theme.typography.headline)
        .foregroundColor(theme.colors.text)
      
      HStack(spacing: theme.spacing.md) {
        Circle()
          .fill(theme.colors.primary)
          .frame(width: 12, height: 12)
        
        Text("アクティブ")
          .font(theme.typography.body)
          .foregroundColor(theme.colors.secondaryText)
      }
    }
    .padding(theme.spacing.lg)
  }
}
```

このアプローチの利点は、**カラーパレット変更が一瞬で全体に反映される**ことだ。アプリの色を変更するとき、`AppTheme.current` の `primary` を編集するだけで、全画面に波及する。

## アニメーション実装：段階的な動きで注目を引く

Rork Maxで生成されたUIは、デフォルトでは静止画的だ。一方、App Storeのトップアプリは、ボタンクリックやリスト遷移でわずかな動きを挟む。これが「プロフェッショナル」という印象を与える要因だ。

**基本的なフェードイン/アウト**：

```swift
struct FadingText: View {
  @State private var isVisible = false
  
  var body: some View {
    Text("これはフェードインします")
      .opacity(isVisible ? 1.0 : 0.0)
      .animation(.easeInOut(duration: 0.3), value: isVisible)
      .onAppear {
        isVisible = true
      }
  }
}
```

`.animation(_:value:)` は、`value` が変わるたびに、その値に関連するプロパティ（ここでは `opacity`）をアニメートする。`duration: 0.3` は、アニメーション時間を300ミリ秒に設定している。

**スプリングアニメーション**（ボタンクリック時）：

```swift
struct BouncingButton: View {
  @State private var isPressed = false
  
  var body: some View {
    Button(action: {
      isPressed = true
      DispatchQueue.main.asyncAfter(deadline: .now() + 0.2) {
        isPressed = false
      }
    }) {
      Image(systemName: "heart.fill")
        .scaleEffect(isPressed ? 1.2 : 1.0)
        .foregroundColor(.red)
    }
    .animation(.spring(response: 0.3, dampingFraction: 0.6), value: isPressed)
  }
}
```

`spring()` は、物理的な弾みを再現するアニメーション。`response` はアニメーション時間、`dampingFraction` は反発度を制御する。`dampingFraction` が小さいほど、よく弾む。

**スライドイン/スライドアウト**（モーダル表示）：

```swift
struct SlideUpView: View {
  @Binding var isPresented: Bool
  
  var body: some View {
    if isPresented {
      VStack {
        Text("スライドアップ表示")
          .font(.headline)
      }
      .frame(maxWidth: .infinity)
      .padding()
      .background(Color.white)
      .cornerRadius(12, corners: [.topLeft, .topRight])
      .transition(.move(edge: .bottom).combined(with: .opacity))
      .animation(.easeInOut(duration: 0.3), value: isPresented)
    }
  }
}
```

`.transition(.move(edge: .bottom))` で下から上へスライドアップし、同時に `.opacity` でフェードインする。複数のトランジションを `combined(with:)` で組み合わせることで、より洗練された動きが実現する。

## ダークモード対応：色とコントラストの意図的な調整

iOS 13からダークモードが標準になった。現在、ユーザーの約40〜50%はダークモードを使用している。適切な対応は、UIの品質評価に直結する。

Rork Maxは、デフォルトでダークモード対応を含める。しかし、カスタムカラーを使う場合、手動で調整が必要だ。

```swift
struct AdaptiveColor {
  static let buttonBackground = Color(UIColor { traitCollection in
    switch traitCollection.userInterfaceStyle {
    case .dark:
      return UIColor(hue: 0.55, saturation: 0.8, brightness: 0.9, alpha: 1)
    case .light:
      return UIColor(hue: 0.55, saturation: 0.7, brightness: 0.95, alpha: 1)
    @unknown default:
      return UIColor(hue: 0.55, saturation: 0.7, brightness: 0.95, alpha: 1)
    }
  })
}
```

ダークモードでは、同じ色相・彩度でも、より高い明度（brightness）を使う。これにより、ダークな背景に対しても十分なコントラストが確保される。

実装のコツは、**色を絶対値ではなく「明るさの相対値」で考える**ことだ。「ライトモードでは白背景に黒テキスト」という思考ではなく、「背景が暗いほど、テキストの明度を上げる」という相対的な思考が正確だ。

## アクセシビリティ対応：Dynamic Type と VoiceOver

20代のユーザーと60代のユーザーは、同じアプリを使う。フォントサイズの好みが大きく異なる。Dynamic Type対応は、オプション機能ではなく、**基本要件**だ。

Rork Maxは、Textコンポーネントで自動的にDynamic Type対応する。だが、カスタムコンポーネントでは手動対応が必要だ：

```swift
struct CustomLabel: View {
  @Environment(\.sizeCategory) var sizeCategory
  
  var body: some View {
    VStack(spacing: sizeCategory > .large ? 12 : 8) {
      Text("タイトル")
        .font(.headline)
      
      Text("説明文がここに入ります")
        .font(.body)
        .lineLimit(3)
    }
  }
}
```

`@Environment(\.sizeCategory)` は、ユーザーが設定したテキストサイズを取得する。サイズが大きいほど、コンポーネント間の間隔を広げるなど、レイアウトを適応させる。

VoiceOverアクセシビリティも重要だ：

```swift
struct StatusBadge: View {
  let status: Status
  
  var body: some View {
    HStack(spacing: 6) {
      Circle()
        .fill(status.backgroundColor)
        .frame(width: 8, height: 8)
      
      Text(status.label)
        .font(.caption)
        .fontWeight(.medium)
    }
    .padding(.horizontal, 8)
    .padding(.vertical, 4)
    .background(status.backgroundColor.opacity(0.1))
    .cornerRadius(6)
    .accessibilityLabel("ステータス：\(status.label)")
    .accessibilityHint("このユーザーは現在\(status.label)です")
  }
}
```

`.accessibilityLabel` と `.accessibilityHint` を設定することで、VoiceOverユーザーが画面の内容を正確に理解できるようになる。

## 実装例：カスタムオンボーディング画面の完全実装

ここまでの知識を組み合わせた、実際のオンボーディング画面の実装例を示す。

```swift
struct OnboardingScreen: View {
  @State private var currentPage = 0
  @State private var isAnimating = false
  let theme = AppTheme.current
  
  var body: some View {
    ZStack {
      // 背景
      theme.colors.background
        .ignoresSafeArea()
      
      VStack(spacing: theme.spacing.lg) {
        // ページインジケーター
        HStack(spacing: theme.spacing.md) {
          ForEach(0..<3, id: \.self) { index in
            Capsule()
              .fill(
                index == currentPage
                  ? theme.colors.primary
                  : theme.colors.primaryLight.opacity(0.3)
              )
              .frame(height: 4)
              .animation(.easeInOut(duration: 0.3), value: currentPage)
          }
        }
        .padding(.horizontal, theme.spacing.lg)
        
        // コンテンツエリア
        TabView(selection: $currentPage) {
          OnboardingPage(
            icon: "star.fill",
            title: "ようこそ",
            description: "このアプリを使ってあなたの生活を変えよう"
          )
          .tag(0)
          
          OnboardingPage(
            icon: "heart.fill",
            title: "簡単",
            description: "シンプルなUIで直感的に操作できます"
          )
          .tag(1)
          
          OnboardingPage(
            icon: "checkmark.circle.fill",
            title: "始めよう",
            description: "今すぐアプリを使い始めましょう"
          )
          .tag(2)
        }
        .tabViewStyle(.page(indexDisplayMode: .never))
        
        // 次へ/完了ボタン
        Button(action: {
          if currentPage < 2 {
            currentPage += 1
          }
        }) {
          HStack {
            Text(currentPage == 2 ? "完了" : "次へ")
              .fontWeight(.semibold)
            Image(systemName: "arrow.right")
              .font(.system(size: 14, weight: .semibold))
          }
          .frame(maxWidth: .infinity)
          .padding(theme.spacing.md)
          .background(theme.colors.primary)
          .foregroundColor(.white)
          .cornerRadius(8)
        }
        .padding(.horizontal, theme.spacing.lg)
        .scaleEffect(isAnimating ? 1.0 : 0.95)
        .animation(.spring(response: 0.3, dampingFraction: 0.6), value: isAnimating)
        .onAppear {
          isAnimating = true
        }
      }
      .padding(.vertical, theme.spacing.xl)
    }
  }
}

struct OnboardingPage: View {
  let icon: String
  let title: String
  let description: String
  let theme = AppTheme.current
  
  var body: some View {
    VStack(spacing: theme.spacing.xl) {
      Image(systemName: icon)
        .font(.system(size: 80))
        .foregroundColor(theme.colors.primary)
        .transition(.scale.combined(with: .opacity))
      
      VStack(spacing: theme.spacing.md) {
        Text(title)
          .font(theme.typography.headline)
          .foregroundColor(theme.colors.text)
        
        Text(description)
          .font(theme.typography.body)
          .foregroundColor(theme.colors.secondaryText)
          .multilineTextAlignment(.center)
          .lineLimit(3)
      }
    }
    .padding(theme.spacing.xl)
  }
}
```

このオンボーディング画面は、以下の要素を含む：

- テーマシステムを使用した色・フォント・間隔の一元管理
- ページ遷移時のスムーズなアニメーション
- ボタンのスプリングエフェクト
- Dynamic Type対応（`.lineLimit(3)` で長いテキストに対応）
- ダークモード自動対応

## 次に試してほしいこと

Rork Maxのコンポーネントライブラリを深く理解する道は、**実装を通じた学習**だ。この記事で紹介したテーマシステムをあなたのアプリに実装してみよう。最初は1つの色（プライマリカラー）だけを定義して、全体で参照させるところから始める。その後、段階的にセマンティックカラー（成功・警告・エラー）を追加し、最後にダークモード対応を完成させる。

この過程で、Rork Maxが生成したコンポーネントたちが、なぜそのような設計になっているのかが自然と理解できるようになる。
