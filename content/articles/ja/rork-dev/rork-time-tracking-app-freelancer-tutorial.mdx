## はじめに — なぜフリーランサーにタイムトラッキングが必要なのか

フリーランスで働く方にとって、作業時間の正確な記録は収益に直結する重要な課題です。クライアントへの請求書作成、自分の生産性の振り返り、プロジェクトごとの採算分析など、時間管理の精度がビジネスの成否を左右するといっても過言ではありません。

この記事では、Rork を使って実用的なタイムトラッキングアプリを一から構築する方法を解説します。AI によるコード生成を活用しながら、タイマー機能・プロジェクト管理・作業レポートまでを備えたアプリを効率よく開発していきましょう。

Rork の AI アプリ開発に初めて触れる方は、[Rork ではじめてのアプリを 30 分で作る入門ガイド](/articles/rork-basics/rork-first-app-30-minutes-guide)を先に読んでおくと、基本操作をスムーズに理解できます。

## 完成イメージとアプリの機能一覧

今回作成するタイムトラッキングアプリには、以下の機能を実装します。

- **タイマー機能**: ワンタップで作業時間の計測を開始・停止できるストップウォッチ型のタイマー
- **プロジェクト管理**: クライアントやプロジェクトごとに作業を分類
- **作業ログ一覧**: 日付順に記録された作業履歴の閲覧
- **日別・週別レポート**: 集計データをグラフで視覚化
- **ローカル保存**: AsyncStorage によるオフライン対応のデータ永続化

フリーランサーが日常的に使えるシンプルかつ実用的なアプリを目指します。

## タイマー機能の実装 — ストップウォッチ型のコア機能

タイムトラッキングアプリの心臓部であるタイマー機能を実装します。`useRef` と `setInterval` を組み合わせた精度の高いタイマーを作ります。

```typescript
// hooks/useTimer.ts — タイマーのカスタムフック
import { useState, useRef, useCallback } from 'react';

interface TimerState {
  isRunning: boolean;
  elapsedSeconds: number;
  startTime: number | null;
}

export function useTimer() {
  const [state, setState] = useState<TimerState>({
    isRunning: false,
    elapsedSeconds: 0,
    startTime: null,
  });
  const intervalRef = useRef<NodeJS.Timeout | null>(null);

  // タイマー開始
  const start = useCallback(() => {
    const now = Date.now();
    setState(prev => ({
      ...prev,
      isRunning: true,
      startTime: now - prev.elapsedSeconds * 1000,
    }));

    intervalRef.current = setInterval(() => {
      setState(prev => {
        if (!prev.startTime) return prev;
        return {
          ...prev,
          elapsedSeconds: Math.floor((Date.now() - prev.startTime) / 1000),
        };
      });
    }, 1000);
  }, []);

  // タイマー停止
  const stop = useCallback(() => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
    setState(prev => ({ ...prev, isRunning: false }));
  }, []);

  // タイマーリセット
  const reset = useCallback(() => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
    setState({ isRunning: false, elapsedSeconds: 0, startTime: null });
  }, []);

  return { ...state, start, stop, reset };
}
```

このカスタムフックを使えば、どの画面からでもタイマーを制御できます。`Date.now()` を基準にしているため、バックグラウンドに回っても時間のズレが最小限に抑えられます。

表示用のフォーマット関数も用意しておきましょう。

```typescript
// utils/formatTime.ts — 経過秒を HH:MM:SS 形式に変換
export function formatTime(totalSeconds: number): string {
  const hours = Math.floor(totalSeconds / 3600);
  const minutes = Math.floor((totalSeconds % 3600) / 60);
  const seconds = totalSeconds % 60;

  return [hours, minutes, seconds]
    .map(v => String(v).padStart(2, '0'))
    .join(':');
}

// 期待する出力:
// formatTime(0)     → "00:00:00"
// formatTime(90)    → "00:01:30"
// formatTime(3661)  → "01:01:01"
```

## プロジェクト管理機能 — クライアント別の作業分類

フリーランスの作業は複数のプロジェクトにまたがることが多いため、プロジェクトごとに作業を分類する機能は必須です。

まずデータの型定義を行います。

```typescript
// types/index.ts — アプリ全体の型定義
export interface Project {
  id: string;
  name: string;
  color: string;       // プロジェクトの識別色
  hourlyRate?: number; // 時給（任意）
  createdAt: number;
}

export interface TimeEntry {
  id: string;
  projectId: string;
  startTime: number;   // Unix timestamp
  endTime: number;     // Unix timestamp
  duration: number;    // 秒数
  note?: string;       // 作業メモ（任意）
  createdAt: number;
}
```

次に、AsyncStorage を使ったデータ永続化のストレージ層を実装します。

```typescript
// storage/projectStorage.ts — プロジェクトデータの永続化
import AsyncStorage from '@react-native-async-storage/async-storage';
import { Project } from '../types';

const PROJECTS_KEY = '@timetracker_projects';

export async function getProjects(): Promise<Project[]> {
  try {
    const json = await AsyncStorage.getItem(PROJECTS_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('プロジェクトの読み込みに失敗:', error);
    return [];
  }
}

export async function saveProject(project: Project): Promise<void> {
  const projects = await getProjects();
  const index = projects.findIndex(p => p.id === project.id);

  if (index >= 0) {
    projects[index] = project; // 既存プロジェクトを更新
  } else {
    projects.push(project);    // 新規追加
  }

  await AsyncStorage.setItem(PROJECTS_KEY, JSON.stringify(projects));
}

export async function deleteProject(id: string): Promise<void> {
  const projects = await getProjects();
  const filtered = projects.filter(p => p.id !== id);
  await AsyncStorage.setItem(PROJECTS_KEY, JSON.stringify(filtered));
}
```

## 作業ログの記録と一覧表示

タイマーを停止したタイミングで、作業ログ（TimeEntry）を自動保存する仕組みを作ります。

```typescript
// storage/timeEntryStorage.ts — 作業記録の永続化
import AsyncStorage from '@react-native-async-storage/async-storage';
import { TimeEntry } from '../types';

const ENTRIES_KEY = '@timetracker_entries';

export async function getTimeEntries(): Promise<TimeEntry[]> {
  try {
    const json = await AsyncStorage.getItem(ENTRIES_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('作業ログの読み込みに失敗:', error);
    return [];
  }
}

export async function saveTimeEntry(entry: TimeEntry): Promise<void> {
  const entries = await getTimeEntries();
  entries.unshift(entry); // 新しい記録を先頭に追加
  await AsyncStorage.setItem(ENTRIES_KEY, JSON.stringify(entries));
}

// 日付範囲で作業ログを絞り込む
export async function getEntriesByDateRange(
  startDate: Date,
  endDate: Date
): Promise<TimeEntry[]> {
  const entries = await getTimeEntries();
  const startTs = startDate.getTime();
  const endTs = endDate.getTime();

  return entries.filter(
    e => e.startTime >= startTs && e.startTime <= endTs
  );
}
```

作業ログの一覧画面では、`SectionList` を使って日付ごとにグループ化して表示すると見やすくなります。

```typescript
// 日付ごとにグループ化するヘルパー関数
function groupEntriesByDate(entries: TimeEntry[]) {
  const groups: Record<string, TimeEntry[]> = {};

  entries.forEach(entry => {
    const dateKey = new Date(entry.startTime)
      .toLocaleDateString('ja-JP', {
        year: 'numeric',
        month: 'long',
        day: 'numeric',
      });

    if (!groups[dateKey]) groups[dateKey] = [];
    groups[dateKey].push(entry);
  });

  return Object.entries(groups).map(([date, data]) => ({
    title: date,
    data,
  }));
}

// 期待する出力例:
// [
//   { title: "2026年3月28日", data: [TimeEntry, TimeEntry, ...] },
//   { title: "2026年3月27日", data: [TimeEntry, ...] },
// ]
```

## レポート画面 — 日別・週別の作業時間集計

フリーランサーにとって、作業時間のレポートは請求書作成や自己分析に欠かせません。日別と週別の2つのビューを切り替えられるレポート画面を実装します。

```typescript
// utils/reportUtils.ts — レポート用の集計ロジック
import { TimeEntry, Project } from '../types';

interface DailySummary {
  date: string;
  totalSeconds: number;
  byProject: Record<string, number>; // projectId -> seconds
}

// 過去7日間の日別サマリーを生成
export function getWeeklySummary(
  entries: TimeEntry[]
): DailySummary[] {
  const summaries: DailySummary[] = [];

  for (let i = 6; i >= 0; i--) {
    const date = new Date();
    date.setDate(date.getDate() - i);
    date.setHours(0, 0, 0, 0);

    const nextDate = new Date(date);
    nextDate.setDate(nextDate.getDate() + 1);

    const dayEntries = entries.filter(
      e => e.startTime >= date.getTime() && e.startTime < nextDate.getTime()
    );

    const byProject: Record<string, number> = {};
    let totalSeconds = 0;

    dayEntries.forEach(entry => {
      totalSeconds += entry.duration;
      byProject[entry.projectId] =
        (byProject[entry.projectId] || 0) + entry.duration;
    });

    summaries.push({
      date: date.toLocaleDateString('ja-JP', {
        month: 'short',
        day: 'numeric',
      }),
      totalSeconds,
      byProject,
    });
  }

  return summaries;
}

// 時給ベースの収益計算
export function calculateEarnings(
  entries: TimeEntry[],
  projects: Project[]
): number {
  return entries.reduce((total, entry) => {
    const project = projects.find(p => p.id === entry.projectId);
    if (project?.hourlyRate) {
      return total + (entry.duration / 3600) * project.hourlyRate;
    }
    return total;
  }, 0);
}

// 期待する出力例（calculateEarnings）:
// 2時間の作業（時給 3000 円のプロジェクト）→ 6000
```

レポート画面では、棒グラフで日別の作業時間を視覚化します。Rork のプロンプトで以下のように指示すると、AI がグラフコンポーネントを生成してくれます。

```text
レポート画面に棒グラフを追加してください。
- X軸：過去7日間の日付
- Y軸：作業時間（時間単位）
- 各棒はプロジェクトの色でスタック表示
- グラフの下に合計時間と推定収益を表示
```

## 実装のポイント — UX を高めるための工夫

### バックグラウンド復帰時のタイマー補正

アプリがバックグラウンドに入っても、タイマーの経過時間が正確であることが重要です。先ほどの `useTimer` フックでは `Date.now()` を基準にしているため、フォアグラウンドに戻った際に自動的に正しい経過時間が表示されます。

さらに、`AppState` を監視してバックグラウンドからの復帰時にUIを更新する処理を追加すると、よりスムーズな体験になります。

```typescript
import { AppState, AppStateStatus } from 'react-native';
import { useEffect, useRef } from 'react';

export function useAppStateRefresh(onForeground: () => void) {
  const appState = useRef(AppState.currentState);

  useEffect(() => {
    const subscription = AppState.addEventListener(
      'change',
      (nextState: AppStateStatus) => {
        if (
          appState.current.match(/inactive|background/) &&
          nextState === 'active'
        ) {
          onForeground(); // フォアグラウンド復帰時にコールバックを実行
        }
        appState.current = nextState;
      }
    );

    return () => subscription.remove();
  }, [onForeground]);
}
```

### 振動フィードバックの追加

タイマーの開始・停止時にハプティックフィードバック（振動）を追加すると、操作した実感が得られて使いやすさが向上します。Expo の `expo-haptics` モジュールを利用します。

```typescript
import * as Haptics from 'expo-haptics';

// タイマー開始時

  await Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium);
  timer.start();
}

// タイマー停止時
async function handleStop() {
  await Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success);
  timer.stop();
  // 作業ログを保存...
}
```

## よくある質問（FAQ）

### Q1: データが端末の容量を圧迫しませんか？

AsyncStorage に保存されるデータはテキスト（JSON）形式のため、数千件の作業ログを保存しても数 MB 程度です。ただし、長期間の運用では古いデータをエクスポートして削除する機能を追加するとよいでしょう。月単位で CSV 出力する機能を追加すれば、バックアップにもなります。

### Q2: 複数端末でデータを同期できますか？

この記事の実装では AsyncStorage（ローカル保存）のみですが、Supabase などの BaaS（Backend as a Service）と連携すれば、クラウド同期が可能です。Supabase のリアルタイム同期機能を使えば、複数端末で即座にデータが反映されます。

### Q3: 時給の異なるプロジェクトの収益を一括で集計できますか？

はい、`calculateEarnings` 関数がプロジェクトごとの時給を参照して自動計算します。レポート画面に合計収益を表示するだけでなく、プロジェクト別の内訳もテーブル形式で出力できます。請求書作成の元データとしても活用可能です。

### Q4: App Store に公開する際の審査で注意すべき点は？

タイムトラッキングアプリは App Store の審査でリジェクトされにくいカテゴリですが、プライバシーポリシーの記載は必須です。また、バックグラウンドでタイマーが動作する旨を `Info.plist` の `UIBackgroundModes` に正しく宣言しておく必要があります。
