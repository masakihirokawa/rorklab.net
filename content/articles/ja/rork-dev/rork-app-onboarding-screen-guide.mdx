## オンボーディング画面がアプリの成功を左右する

アプリをダウンロードしたユーザーの約25%が、初回起動後すぐにアンインストールするというデータがあります。その大きな原因のひとつが「使い方がわからない」という初回体験の失敗です。

オンボーディング画面は、ユーザーがアプリを初めて起動したときに表示される導入画面のことです。アプリの主要機能を紹介し、必要な権限を取得し、ユーザーを自然にメイン画面へと導く——この最初の数秒間の体験が、アプリの継続利用率を大きく左右します。

Rorkを使えば、AIにプロンプトを伝えるだけで美しいオンボーディング画面を短時間で構築できます。この記事では、効果的なオンボーディング画面の設計パターンから実装方法まで、実践的なノウハウをお伝えします。

## オンボーディング画面の基本設計パターン

オンボーディングにはいくつかの代表的なパターンがあります。アプリの性質や複雑さに応じて最適なものを選びましょう。

### スワイプ式チュートリアル

もっとも一般的なパターンです。3〜5枚のスライドでアプリの特徴を順番に紹介します。ユーザーは左右にスワイプして進み、最後のスライドで「はじめる」ボタンをタップしてメイン画面に遷移します。

**向いているアプリ**: 機能が多いアプリ、初めて触る概念を扱うアプリ

### プログレッシブオンボーディング

初回起動時にすべてを説明するのではなく、ユーザーが操作するたびに関連する機能をその場で案内するパターンです。コーチマーク（吹き出し）やツールチップで要素をハイライトして説明します。

**向いているアプリ**: シンプルなUI、操作しながら覚えるタイプのアプリ

### パーミッションリクエスト型

通知許可・位置情報・カメラアクセスなど、アプリに必要な権限をひとつずつ理由を説明しながらリクエストするパターンです。「なぜこの権限が必要か」を丁寧に伝えることで、許可率が大幅に向上します。

**向いているアプリ**: 通知・位置情報・カメラなどのデバイス機能を多用するアプリ

## Rorkでスワイプ式オンボーディングを実装する

ここからは、Rorkで実際にスワイプ式オンボーディング画面を構築する手順を見ていきましょう。

### ステップ1: プロジェクトの準備

Rorkのエディタを開き、新しいプロジェクトを作成するか、既存のプロジェクトに画面を追加します。オンボーディング画面は、アプリの最初のエントリーポイントとして設計します。

Rorkに次のようなプロンプトを入力してみましょう。

```
3画面のスワイプ式オンボーディングを作成してください。
- 画面1: アプリの主要機能を紹介（アイコン + タイトル + 説明文）
- 画面2: 使い方の簡単な説明（ステップ形式）
- 画面3: 「はじめる」ボタンでメイン画面へ遷移
- ページインジケーター（ドット）を画面下部に表示
- スキップボタンを右上に配置
- アニメーション付きのページ遷移
```

### ステップ2: コンポーネント構造を理解する

Rorkが生成するオンボーディング画面は、一般的に以下のコンポーネント構造になります。

```typescript
// OnboardingScreen.tsx — Rorkが生成する典型的な構造
import React, { useState, useRef } from 'react';
import {
  View,
  Text,
  FlatList,
  Dimensions,
  TouchableOpacity,
  Animated,
  StyleSheet,
} from 'react-native';
import AsyncStorage from '@react-native-async-storage/async-storage';

const { width } = Dimensions.get('window');

// オンボーディングのスライドデータ
const slides = [
  {
    id: '1',
    title: 'かんたん操作',
    description: 'AIがあなたの作業をサポート。直感的な操作で誰でもすぐに使えます。',
    icon: '🚀',
  },
  {
    id: '2',
    title: 'スマート通知',
    description: '必要な情報を最適なタイミングでお届け。大切なことを見逃しません。',
    icon: '🔔',
  },
  {
    id: '3',
    title: 'さあ、はじめましょう',
    description: 'あなたの毎日をもっと便利に。今すぐ始めましょう。',
    icon: '✨',
  },
];

export default function OnboardingScreen({ navigation }) {
  const [currentIndex, setCurrentIndex] = useState(0);
  const flatListRef = useRef(null);

  // オンボーディング完了をローカルストレージに保存
  const completeOnboarding = async () => {
    await AsyncStorage.setItem('onboarding_complete', 'true');
    navigation.replace('Main');
  };

  // スキップボタンの処理
  const handleSkip = () => {
    completeOnboarding();
  };

  // 次のスライドへ進む
  const handleNext = () => {
    if (currentIndex < slides.length - 1) {
      flatListRef.current?.scrollToIndex({ index: currentIndex + 1 });
    } else {
      completeOnboarding();
    }
  };

  return (
    <View style={styles.container}>
      {/* スキップボタン */}
      {currentIndex < slides.length - 1 && (
        <TouchableOpacity style={styles.skipButton} onPress={handleSkip}>
          <Text style={styles.skipText}>スキップ</Text>
        </TouchableOpacity>
      )}

      {/* スライド一覧 */}
      <FlatList
        ref={flatListRef}
        data={slides}
        horizontal
        pagingEnabled
        showsHorizontalScrollIndicator={false}
        onMomentumScrollEnd={(e) => {
          const index = Math.round(e.nativeEvent.contentOffset.x / width);
          setCurrentIndex(index);
        }}
        renderItem={({ item }) => (
          <View style={[styles.slide, { width }]}>
            <Text style={styles.icon}>{item.icon}</Text>
            <Text style={styles.title}>{item.title}</Text>
            <Text style={styles.description}>{item.description}</Text>
          </View>
        )}
        keyExtractor={(item) => item.id}
      />

      {/* ページインジケーター */}
      <View style={styles.pagination}>
        {slides.map((_, index) => (
          <View
            key={index}
            style={[
              styles.dot,
              currentIndex === index && styles.activeDot,
            ]}
          />
        ))}
      </View>

      {/* 次へ / はじめるボタン */}
      <TouchableOpacity style={styles.nextButton} onPress={handleNext}>
        <Text style={styles.nextButtonText}>
          {currentIndex === slides.length - 1 ? 'はじめる' : '次へ'}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

// 期待する動作:
// - 左右スワイプでスライドが切り替わる
// - ドットインジケーターが現在のページを示す
// - 「スキップ」でオンボーディングを飛ばしてメイン画面へ
// - 最後のスライドで「はじめる」をタップするとメイン画面へ遷移
```

### ステップ3: ナビゲーションとの統合

オンボーディング画面は、アプリ起動時に一度だけ表示し、完了後は再表示しない仕組みが必要です。Rorkでは Expo Router を使ったナビゲーション統合が一般的です。

```typescript
// app/_layout.tsx — 初回起動判定のルートレイアウト
import { useEffect, useState } from 'react';
import { Stack } from 'expo-router';
import AsyncStorage from '@react-native-async-storage/async-storage';

export default function RootLayout() {
  const [isFirstLaunch, setIsFirstLaunch] = useState<boolean | null>(null);

  useEffect(() => {
    const checkOnboarding = async () => {
      const completed = await AsyncStorage.getItem('onboarding_complete');
      setIsFirstLaunch(completed === null);
    };
    checkOnboarding();
  }, []);

  if (isFirstLaunch === null) return null; // ローディング中

  return (
    <Stack screenOptions={{ headerShown: false }}>
      {isFirstLaunch ? (
        <Stack.Screen name="onboarding" />
      ) : (
        <Stack.Screen name="(tabs)" />
      )}
    </Stack>
  );
}

// 期待する動作:
// - 初回起動時はオンボーディング画面を表示
// - 2回目以降はメインのタブ画面を直接表示
```

## パーミッション取得をオンボーディングに組み込む

通知やカメラなどの権限は、OSのダイアログを直接表示するよりも、事前に「なぜ必要か」を説明する画面を挟むことで許可率が大幅に向上します。

Rorkに以下のようなプロンプトを追加します。

```
オンボーディングの2画面目と3画面目の間に
パーミッションリクエスト画面を追加してください。
- 通知許可: 「重要なお知らせをリアルタイムでお届けします」
- 許可 / あとで を選択可能
- 「あとで」を選んでもオンボーディングは進行する
```

```typescript
// PermissionScreen.tsx — 通知許可リクエスト画面の例
import * as Notifications from 'expo-notifications';

const requestNotificationPermission = async () => {
  const { status } = await Notifications.requestPermissionsAsync();
  return status === 'granted';
};

// 「許可する」ボタンタップ時
const handleAllow = async () => {
  await requestNotificationPermission();
  goToNextSlide();
};

// 「あとで」ボタンタップ時
const handleLater = () => {
  goToNextSlide(); // 権限なしでも先に進める
};

// 期待する動作:
// - 「許可する」→ OS の通知許可ダイアログが表示される
// - 「あとで」→ ダイアログなしで次の画面へ
```

この「プリパーミッション画面」を設けることで、ユーザーに心の準備をさせてからOSダイアログを表示できるため、許可率が平均20〜30%向上すると言われています。

## デザインのベストプラクティス

### イラスト・アイコンの活用

オンボーディング画面では、テキストだけでなくビジュアル要素が重要です。Rorkでは、プロンプトに「Lottieアニメーションを使って」「アイコンを大きく表示して」と指定することで、視覚的に魅力的な画面を生成できます。

### 文字数を最小限に

各スライドのテキストは見出し＋説明文で50文字以内が理想です。長い説明は読まれません。要点をひとつに絞り、端的に伝えましょう。

### スキップの選択肢を必ず用意する

すでにアプリの使い方を知っているユーザーや、説明を読みたくないユーザーのために、スキップ機能は必須です。スキップボタンは右上に小さく配置するのが一般的です。

### ダークモード対応

Rorkで生成する場合は、プロンプトに「ダークモード対応」を含めると、`useColorScheme` を使ったテーマ切り替え対応のコードが生成されます。

```typescript
// ダークモード対応のスタイル定義例
import { useColorScheme } from 'react-native';

const colorScheme = useColorScheme();
const isDark = colorScheme === 'dark';

const dynamicStyles = {
  container: {
    backgroundColor: isDark ? '#1a1a2e' : '#ffffff',
  },
  title: {
    color: isDark ? '#e0e0e0' : '#1a1a2e',
  },
  description: {
    color: isDark ? '#a0a0b0' : '#666666',
  },
};

// 期待する動作:
// - システム設定に連動してライト/ダークテーマが自動切替
```

## よくある失敗と対策

### スライド枚数が多すぎる

オンボーディングは3〜5枚が最適です。それ以上になるとユーザーは途中でスキップするか、アプリを閉じてしまいます。どうしても伝えたい情報が多い場合は、プログレッシブオンボーディング（使いながら覚える方式）を併用しましょう。

### 「次へ」ボタンが小さい・わかりにくい

メインのCTA（Call To Action）ボタンは、画面下部に大きく配置しましょう。ボタンのラベルも「次へ」「はじめる」など、次にどうすべきかが明確な文言にします。

### オンボーディング後に再表示できない

設定画面から「チュートリアルを再表示」できるオプションを用意すると、後から機能を確認したいユーザーに親切です。

```typescript
// 設定画面からオンボーディングをリセットする例
const resetOnboarding = async () => {
  await AsyncStorage.removeItem('onboarding_complete');
  // アプリを再起動するとオンボーディングが再表示される
};
```

## FAQ

### Q: Rorkでオンボーディング画面を作るのにどのくらい時間がかかりますか？

A: 基本的なスワイプ式オンボーディングであれば、プロンプト入力からプレビュー確認まで約5〜10分で完成します。パーミッション取得やアニメーションを含むより高度な実装でも、30分程度で仕上がるケースがほとんどです。

### Q: オンボーディングをA/Bテストするにはどうすればよいですか？

A: Rorkで2パターンのオンボーディングを作成し、[Firebase Remote Config](/articles/rork-dev/rork-max-feature-flags-remote-config) を使ってユーザーごとに異なるバージョンを表示する方法が一般的です。完了率や離脱ポイントを分析して、より効果的なパターンを選びましょう。

### Q: Rork Maxを使うメリットはありますか？

A: Rork MaxではネイティブSwiftコードを生成できるため、SwiftUI の `TabView` と `PageTabViewStyle` を使ったよりスムーズなスワイプアニメーションを実現できます。また、App Clipsにオンボーディングを組み込む場合も [Rork Max の App Clips 機能](/articles/rork-dev/rork-max-app-clips-guide) が活用できます。

## まとめ

オンボーディング画面は、ユーザーがアプリに出会う最初の瞬間です。Rorkを活用すれば、スワイプ式チュートリアル、パーミッションリクエスト、プログレッシブ案内など、さまざまなパターンのオンボーディングをAIの力で素早く構築できます。

大切なのは「ユーザーが次に何をすべきかを明確にすること」です。スライドの枚数は3〜5枚に抑え、各画面で伝えるメッセージはひとつだけ。スキップ機能を忘れずに用意し、パーミッションは理由を添えてリクエストする——これらの基本を押さえれば、ユーザーの初回体験を大きく改善できるはずです。

ぜひRorkで実際にオンボーディング画面を作ってみて、あなたのアプリの第一印象を磨いてみてください。
