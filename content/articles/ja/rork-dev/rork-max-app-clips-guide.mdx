## App Clips とは？なぜ今注目されているのか

スマートフォンを取り出してアプリをダウンロードし、アカウント登録を済ませて——そんな手間がユーザーの離脱を招くことは、モバイル開発者であれば誰もが知っています。Apple が iOS 14 で発表した **App Clips** は、この問題を根本から解決する仕組みです。

App Clips は最大 **50MB** の軽量アプリ体験を、フルアプリのインストールなしに提供します。QRコード・NFC タグ・Safari のスマートバナー・マップ・Siri から呼び出せ、ユーザーは「タップ一発」で機能を試せます。決済・予約・チェックインといったシングルタスクに最適で、コンバージョン率の大幅な改善が期待できます。

そしてこの App Clips を、コードを書かずに **Rork Max** だけで構築できるようになりましました。本記事では、Rork Max を使った App Clips 開発の全ステップを解説します。

---

## 前提知識・環境準備

App Clips 開発を始める前に、以下を確認してください。

- **Rork Max プラン**（$200/月）に加入していること — App Clips は SwiftUI ネイティブ機能のため Rork Max が必要
- **Apple Developer Program** に登録済み（年間 $99）— App Store Connect でのプロビジョニングに必要
- **Xcode 16 以降**（ビルド確認用、必須ではないが推奨）
- iOS 16 以降のテスト端末

> **Rork（無料・スタンダードプラン）との違い**: 通常の Rork は React Native ベースのクロスプラットフォームアプリを生成します。App Clips は Apple プラットフォーム専用の SwiftUI 機能であるため、ネイティブ Swift コードを生成できる Rork Max が必要です。

## App Clips の基本概念を理解する

### 通常アプリと App Clips の構造

App Clips は、フルアプリのターゲットとして追加される「エクステンション」のような存在です。Rork Max がプロジェクトを生成すると、以下の構成になります。

```
MyApp (Main Target)
└── MyAppClip (App Clip Target)
    ├── ContentView.swift       // Clip の UI エントリポイント
    ├── AppClipExperience.swift // URL スキーム処理
    └── Assets.xcassets         // Clip 専用アセット（50MB 以内）
```

重要なルール：
- App Clip の UI はシンプルに保つ（画面数は 1〜3 程度）
- 初回起動後すぐにアクションできる設計
- Sign in with Apple または電話番号認証を使う（手入力フォームは最小化）

## ステップバイステップ：Rork Max で App Clips を構築する

### Step 1: プロジェクトの作成と Clip ターゲットの追加

Rork Max のプロンプトに以下のように入力します。

```
カフェの注文 App Clip を作ってください。
- コーヒーメニューを 3 種類表示（ラテ、アメリカーノ、カプチーノ）
- 各メニューには価格と画像を表示
- Apple Pay で即決済できるボタン
- 注文完了後に「ご注文ありがとうございます」画面を表示
- App Clip ターゲットとして設定
```

Rork Max は SwiftUI コードを生成し、自動的に App Clip ターゲットをプロジェクトに追加します。生成されるコードの例を見てみましょう。

```swift
// ContentView.swift — Rork Max が生成するコード例
import SwiftUI
import PassKit

struct AppClipContentView: View {
    @State private var selectedItem: MenuItem?
    @State private var showConfirmation = false

    let menuItems = [
        MenuItem(name: "ラテ", price: 550, imageName: "latte"),
        MenuItem(name: "アメリカーノ", price: 480, imageName: "americano"),
        MenuItem(name: "カプチーノ", price: 580, imageName: "cappuccino"),
    ]

    var body: some View {
        NavigationStack {
            VStack(spacing: 16) {
                Text("今すぐ注文")
                    .font(.largeTitle.bold())
                    .padding(.top)

                ForEach(menuItems) { item in
                    MenuCard(item: item, isSelected: selectedItem?.id == item.id) {
                        selectedItem = item
                    }
                }

                if let item = selectedItem {
                    // Apple Pay ボタン（PKPaymentAuthorizationView）
                    ApplePayButton(item: item) {
                        showConfirmation = true
                    }
                    .frame(height: 50)
                    .padding(.horizontal)
                }

                Spacer()

                // App Clip バナー：フルアプリへの誘導
                AppClipGetFullAppButton()
            }
            .padding()
            .navigationTitle("Clip メニュー")
        }
        .sheet(isPresented: $showConfirmation) {
            ConfirmationView()
        }
    }
}

// 期待する出力:
// - メニューカードが縦に並ぶリスト表示
// - タップで選択状態になり Apple Pay ボタンが出現
// - 決済後に確認シートが表示される
```

### Step 2: App Clip URL の設定（Associated Domains）

App Clips はユニバーサルリンクを使って起動します。Rork Max に以下を追加で依頼します。

```
appclips.example.com でこの App Clip を呼び出せるよう、
Associated Domains の設定と URL ハンドリングを追加してください。
```

生成される URL ハンドリングコードの例：

```swift
// AppClipExperience.swift
import SwiftUI

@main
struct MyAppClip: App {
    var body: some Scene {
        WindowGroup {
            AppClipContentView()
                .onContinueUserActivity(
                    NSUserActivityTypeBrowsingWeb,
                    perform: handleUserActivity
                )
        }
    }

    // URL から注文情報を受け取る例
    // 例: https://appclips.example.com/order?item=latte&table=5
    private func handleUserActivity(_ userActivity: NSUserActivity) {
        guard let url = userActivity.webpageURL,
              let components = URLComponents(url: url, resolvingAgainstBaseURL: true),
              let itemParam = components.queryItems?.first(where: { $0.name == "item" })?.value
        else { return }

        // URL パラメータを使ってメニュー選択を事前設定
        NotificationCenter.default.post(
            name: .appClipItemSelected,
            object: itemParam
        )
    }
}
```

### Step 3: サイズの最適化（50MB 制限への対応）

App Clip は 50MB 以内に収める必要があります。Rork Max にサイズ最適化を依頼できます。

```
App Clip のサイズを 50MB 以内に収めるため、
アセットを最適化し、不要なフレームワークを除外してください。
```

よくある最適化ポイント：
- 画像は WebP または HEIC 形式を使用
- フォントは必要な weight のみ埋め込む
- ネットワーク画像は Clip 内でキャッシュ管理

## よくあるエラーと対処法

### エラー 1: ITMS-90474「App Clip が 50MB を超えている」

```
解決策:
1. Xcode の Report Navigator でビルドサイズを確認
2. 大容量の画像アセットを CDN（Cloudflare）に移動
3. Rork Max に「App Clip ターゲットから [フレームワーク名] を除外」と依頼
```

### エラー 2: App Clip が起動しない（Associated Domains 不一致）

```
確認すべき事項:
- Apple Developer Portal で App Clips ドメインが承認されているか
- サーバーの /.well-known/apple-app-site-association に
  "appclips" エントリが正しく記述されているか

{
  "appclips": {
    "apps": ["TEAMID.com.example.myapp.Clip"]
  }
}
```

### エラー 3: Apple Pay が Simulator でテストできない

App Clips の Apple Pay は実機テストのみ対応しています。Rork Max でモック決済機能を追加することで開発中の動作確認が可能です。

```
テスト用のモック決済ボタンを追加してください。
DEBUG ビルド時は実際の Apple Pay を呼ばずに確認ダイアログを表示するだけにしてください。
```

## 応用・発展的な活用法

### 活用例 1: QR コードとの組み合わせ（飲食店・小売）

テーブルの QR コードに `https://appclips.example.com/order?table=5` という URL を埋め込むだけで、そのテーブル専用の注文 App Clip を起動できます。Rork Max で生成した App Clip に `/articles/rork-dev/rork-max-qr-code-guide` で解説している QR コード読み取り機能を組み合わせることも可能です。

### 活用例 2: NFC タグとの連携（ホテル・イベント）

NFC タグを部屋のドアや受付に設置し、ルームキーや受付チェックインを App Clip で実現します。Rork Max のプロンプトで「NFC タグ読み取り後にチェックイン App Clip を起動」と指定するだけで、CoreNFC を使ったネイティブコードが自動生成されます。

### 活用例 3: フルアプリへのスムーズな移行

App Clip から App Store への誘導は、App Clip バナー（`SKOverlay`）を使います。Rork Max に依頼することで、決済完了後に「フルアプリで過去の注文を管理できます」という誘導 UI を自動追加できます。

```swift
// Rork Max が生成するフルアプリ誘導コード例
import StoreKit

struct AppClipGetFullAppButton: View {
    var body: some View {
        Button("フルアプリをダウンロード") {
            // SKOverlay でフルアプリへ誘導
            guard let scene = UIApplication.shared.connectedScenes.first as? UIWindowScene else { return }
            let config = SKOverlay.AppClipConfiguration(position: .bottom)
            let overlay = SKOverlay(configuration: config)
            overlay.present(in: scene)
        }
        .font(.footnote)
        .foregroundColor(.secondary)
    }
}
```

## FAQ

**Q. App Clips は Android でも使えますか？**

A. いいえ、App Clips は Apple のエコシステム（iOS 16以降）専用の機能です。Android 向けには Google が「Instant Apps」という類似機能を提供していますが、仕様が異なります。Android もサポートしたい場合は、通常の Rork（React Native）でクロスプラットフォームアプリを開発し、App Clips と連携させる構成がおすすめです。

**Q. Rork Max の無料プランで App Clips は作れますか？**

A. 無料プランは週あたり約5プロンプトの制限があり、App Clips の開発には複数回のやり取りが必要になるため、実質的には Max プラン（$200/月）が必要です。ただし、まず無料プランで基本的な UI を試してみてから、App Clip ターゲットへの変換を Max プランで行うという使い方も可能です。

**Q. App Store 申請時に App Clips に追加審査は必要ですか？**

A. App Clips はメインアプリの一部としてレビューされます。ただし、App Clip の URL（Associated Domain）は App Store Connect で別途登録が必要です。また、App Clips は一定のデザインガイドライン（起動後即アクション・シンプルな UI）に従う必要があり、これを守らない場合はリジェクトされることがあります。

**Q. Rork Max で複数の App Clip Experience（URL パターン）を設定できますか？**

A. はい。App Store Connect では複数の「App Clip Experience」を登録でき、それぞれ異なる URL パターンに対して異なる UI を表示できます。Rork Max に「URL パラメータによって表示コンテンツを切り替える」よう依頼することで対応可能です。
