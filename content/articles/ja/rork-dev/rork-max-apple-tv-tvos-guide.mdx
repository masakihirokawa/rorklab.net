Apple TVはリビングルームのエンターテイメント中心地です。映画、ドラマ、ゲーム、スポーツ中継 — あらゆるコンテンツがApple TVで消費される時代、**tvOSアプリの需要は急速に高まっています**。

しかし、tvOSアプリ開発は、iOSアプリと比べて異なる設計原則が必要です。タッチスクリーンではなく、**Siri Remote という限定的な入力デバイス**に対応する必要があり、UI/UXの考え方も大きく異なります。

この記事では、Rork Maxを使ってtvOSアプリを効率的に開発する方法を、基礎から実装例まで、詳しく解説します。

## tvOSとは — iPhoneとの違いを理解する

tvOSは、**Apple TVで動作するOSで、iOSとは大きく異なるUI/UXパラダイムを採用しています。**

### iOSとtvOSの主な違い

- **入力デバイス**: iPhoneはタッチスクリーン（直感的）。Apple TVはSiri Remote（遠隔、制限的）。
- **視聴距離**: iPhoneは30cm程度の距離。Apple TVはソファから2～3メートル。つまり、**テキストは大きく、ボタンは離れていても押しやすい必要があります。**
- **インタラクション**: iOSは「タップ・スワイプ」が主体。tvOSは「方向キー移動・選択」が主体。
- **フォーカス概念**: tvOSでは、**フォーカス（ハイライト）がUIの中心**です。ユーザーは、方向キー（Up / Down / Left / Right）でフォーカスを移動し、センター（Select）で決定します。
- **永続接続**: iPhoneはバッテリー制約。Apple TVはAC電源で常時接続。バックグラウンド処理やストリーミングに向いています。

## フォーカスベースUIの設計原則

tvOSアプリの成功は、**フォーカスベースUIの理解**にかかっています。

### 原則1: すべてがフォーカス可能であるべき

iPhoneでは、ユーザーは任意のポイントをタップできます。一方、tvOSではユーザーは方向キーで**フォーカス可能なUI要素を順番に移動**します。

つまり、あなたのアプリに「タップできるが、フォーカスできないボタン」があってはいけません。逆に、すべてのインタラクティブ要素は、方向キーナビゲーションでアクセス可能である必要があります。

### 原則2: ビジュアルフィードバックが重要

iPhoneでは、ユーザーはボタンを「タップ」するので、現在地を常に認識しています。tvOSでは、ユーザーはリモコンの方向キーを押しており、画面上のフォーカス位置を視覚的に追跡する必要があります。

ボタンがフォーカスされたときの変化（色、サイズ、影）は、tvOSでは**必須のUX要件**です。

### 原則3: タブ移動の距離を最小化

テレビ画面から2～3メートル離れてリモコンを操作する場合、「Up → Right → Right → Right → Down」という複雑な移動は疲れます。

タブ順序（Tab Order）をシンプルに設計し、ユーザーが目的のボタンに**最小ステップでたどり着ける**UI設計を心がけましょう。

## Siri Remoteの機能と実装

Siri Remoteは、以下の入力機能を提供します：

- **タッチパッド**: 上下左右のスワイプ、クリック
- **方向キー**: Up / Down / Left / Right（タッチパッドでも実現可）
- **Selectボタン（中央）**: 現在フォーカスされたUIの決定
- **Menu ボタン**: 前の画面へ戻る（iOS の Back ボタンに相当）
- **Siri ボタン**: 音声入力
- **ホームボタン**: ホーム画面に戻る

SwiftUI でこれらを実装するには：

```swift
// フォーカスされたときのアクション
.onMoveCommand { direction in
    switch direction {
    case .up:
        currentSelection = max(0, currentSelection - 1)
    case .down:
        currentSelection = min(items.count - 1, currentSelection + 1)
    case .left, .right:
        break
    }
}

// Select ボタンが押されたときのアクション
.onExitCommand {
    // Menu ボタンが押されたとき（前の画面へ）
    dismiss()
}

// フォーカスの可否を制御
.focusable()
```

## Rork Max での tvOS アプリ作成フロー

### ステップ1: プロジェクトテンプレート選択

Rork Max のダッシュボードで「New Project」を選択します。

プラットフォームで **tvOS** を選択。Rork Max は自動的にtvOS用のSwiftUI テンプレートを用意します。

### ステップ2: tvOS固有のAIプロンプト

Rork Max のプロンプトボックスに、tvOS用の詳細な指示を入力します。例：

```
「Apple TV向けのメディアブラウザアプリを作成してください。

【仕様】
- 表示内容: 映画カテゴリ（Action, Drama, Comedy）をグリッドで表示
- UI: 3列×2行のGrid。各タイルには映画ポスター画像と題名を表示
- フォーカスUI: 選択時に映画タイル拡大表示、説明文が下部に表示
- ナビゲーション: 方向キー（Up/Down/Left/Right）で移動、Selectで詳細画面へ
- メニュー: Menuボタンで前の画面へ戻る
- 暗いテーマ: tvOS向けに暗い背景色（黒～濃いグレー）
- フォント: 最小サイズ28pt（2～3m離れた視聴に対応）

【実装詳細】
- SwiftUI の NavigationStack を使用
- 映画データはMockData（本番ではAPI呼び出し）
- focusable() / .onMoveCommand を使用したフォーカスハンドリング
」
```

Claude Opus 4.6 がこれを受け取り、tvOS対応のSwiftUIコードを生成します。

### ステップ3: 方向キー移動のテスト

Rork Max のブラウザシミュレータで、矢印キー（またはマウスキー）を使って方向キーをシミュレートできます。

フォーカスがスムーズに移動し、選択時に正しく反応することを確認します。

### ステップ4: クラウドコンパイル & ビルド

「Build」をクリック。Rork Max がtvOS用にコンパイルします。

完了すると、ブラウザシミュレータにApple TV Simulator が起動します。

### ステップ5: 実機テスト（Rork Companion経由）

Rork Companion を使ってApple TVに直接インストールすることで、**実際のSiri Remoteでのテスト**が可能です。

リビングのテレビ画面で確認し、UX上の問題（フォーカス移動が遅い、ボタンが小さすぎる等）があれば、Rork Max上で修正 → 再ビルド → 再インストール という高速なイテレーションができます。

## tvOS固有の実装パターン

### パターン1: グリッドレイアウトでの映画/ドラマ表示

```swift
struct MediaGridView: View {
    @State var selectedIndex = 0
    let items = ["映画1", "映画2", "映画3", "映画4", "映画5", "映画6"]

    var body: some View {
        VStack {
            LazyVGrid(
                columns: Array(repeating: GridItem(.flexible()), count: 3),
                spacing: 20
            ) {
                ForEach(Array(items.enumerated()), id: \.offset) { index, item in
                    MediaTile(
                        title: item,
                        isSelected: index == selectedIndex
                    )
                    .focusable(true) { focused in
                        if focused {
                            selectedIndex = index
                        }
                    }
                }
            }
            .padding()

            // 選択されたアイテムの詳細
            SelectedMediaDetail(item: items[selectedIndex])
        }
    }
}

struct MediaTile: View {
    let title: String
    let isSelected: Bool

    var body: some View {
        VStack {
            Image(systemName: "film.fill")
                .font(.system(size: 60))
                .foregroundColor(.white)
        }
        .frame(height: 300)
        .frame(maxWidth: .infinity)
        .background(isSelected ? Color.orange : Color.gray)
        .cornerRadius(10)
        .scaleEffect(isSelected ? 1.1 : 1.0)
        .shadow(radius: isSelected ? 10 : 0)
    }
}
```

このコードは、フォーカス時に色が変わり、サイズが拡大するメディアグリッドを実装しています。

### パターン2: ハーモニックスクロール

tvOS では、画面外のアイテムも部分的に見え、**スクロール時に奥行きのある立体感**を演出します。これを「ハーモニックスクロール」と呼びます。

```swift
struct HarmonicScrollView: View {
    let items = ["Item 1", "Item 2", "Item 3", "Item 4"]
    @State var scrollOffset: CGFloat = 0

    var body: some View {
        ScrollView(.horizontal, showsIndicators: false) {
            HStack(spacing: 30) {
                ForEach(Array(items.enumerated()), id: \.offset) { index, item in
                    VStack {
                        Image(systemName: "rectangle.fill")
                            .font(.system(size: 100))
                        Text(item)
                            .font(.system(size: 28, weight: .bold))
                    }
                    .frame(width: 200, height: 250)
                    .background(Color.blue.opacity(0.7))
                    .cornerRadius(15)
                    .transformEffect(
                        .init(scaleX: 0.9, y: 0.9)
                    ) // 外側のアイテムを小さく表示
                }
            }
            .padding()
        }
    }
}
```

### パターン3: TVMLKit（テンプレートベースUI）

一部のtvOSアプリは、**TVMLKit** というXMLベースのUI記述言語を使用します。特に、Apple TV Appのような複雑なメディア階層を扱うアプリに向いています。

```swift
// SwiftUI と TVMLKit の組み合わせ
struct AppDelegate: UIApplicationDelegate {
    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {
        // JavaScript から読み込むXMLテンプレート
        let tvmlUrl = URL(string: "http://localhost:9001/template.xml")!
        TVApplicationController.load(
            resource: tvmlUrl,
            with: TVApplicationControllerContext()
        )
        return true
    }
}
```

template.xml の例：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<document>
  <head>
    <style>
      <color name="blue">#0099FF</color>
    </style>
  </head>
  <body>
    <catalogTemplate>
      <banner>
        <title>映画カテゴリ</title>
      </banner>
      <list>
        <section>
          <lockup onSelect="showMovie(0)">
            <image src="poster1.png" width="200" height="300" />
            <title>映画タイトル1</title>
          </lockup>
        </section>
      </list>
    </catalogTemplate>
  </body>
</document>
```

TVMLKit は「サーバーからXMLを取得 → tvOSで動的にUIを生成」という使い方で、複数のApple TV版アプリを一元管理できます。

## よくある質問（FAQ）

### Q1: tvOS アプリ開発に最小限必要なスキルは？

**A1**: SwiftUI の基本知識があれば十分です。フォーカスUIは Rork Max の AI が提案してくれるので、「矢印キーでの移動」という概念を理解すれば、iOSアプリ開発の経験がなくても開始できます。

### Q2: 既存のiOSアプリをtvOSに移植できますか？

**A2**: 部分的には可能です。ビジネスロジック（データ取得、計算等）は共有できますが、UI層は **完全に書き直す必要があります**。iOSのUIがタッチ前提であれば、tvOSのフォーカスUI向けに再設計する必要があります。

### Q3: Apple TV App Store に申請する場合、レビュー期間はどのくらい？

**A3**: 通常、iOS App Store と同じく **1～3日**（Apple によって異なります）。ただし、tvOS のレビューガイドラインは iOS よりも厳しい傾向があります。特に、リモコンナビゲーションが「直感的に操作できるか」を重視します。

### Q4: Siri Remote でゲームも開発できますか？

**A4**: 可能ですが、限定的です。Siri Remote はゲーム向けに設計されていません（方向キー + Select + Menu しかない）。ゲーム開発には、MFi認定ゲームコントローラー（PlayStation 5 コントローラー等）をサポートすることをお勧めします。

### Q5: tvOS アプリの収益化方法は？

**A5**: App Storeでの有料配布、またはアプリ内購入（IAP）が主体です。tvOS ユーザーは、Apple TV+ や Apple Arcade 等のサブスクリプション購読層でもあるので、定期購読モデルも有効です。

## tvOS アプリ開発のベストプラクティス

1. **リモートから2～3メートルの距離を想定した大きなUIを設計**
   - 最小フォントサイズ: 28pt～32pt
   - ボタン最小サイズ: 100×100pt

2. **フォーカス移動を最小化**
   - グリッドは 3 列以下
   - 1画面のフォーカス可能要素は 10個以下

3. **ビジュアルフィードバックを充実させる**
   - フォーカス時のスケール拡大・色変更
   - 選択時のアニメーション

4. **低速ネットワークを想定**
   - Apple TV の Wi-Fi は不安定な場合がある
   - 画像は圧縮して、キャッシング戦略を立てる

5. **バックグラウンド処理を活用**
   - データ同期、ダウンロード等は、UI 操作中に進める
   - ローディング中に UI をブロックしない

## Rork Max でのtvOS App Store 公開

Rork Max でビルドが完了したら、App Store 公開も自動化できます：

1. **Publish to App Store** をクリック
2. tvOS アプリ専用のメタデータを入力（スクリーンショットは tvOS の 1920×1080 フォーマット）
3. **Submit for Review**

Rork Max は自動的に tvOS 用の署名・プロビジョニングを処理します。
