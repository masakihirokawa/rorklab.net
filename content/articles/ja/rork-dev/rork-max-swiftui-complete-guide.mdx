Rork Maxは単なるノーコードツールではありません。その背後には、GPT-4を活用した高度なSwiftUIコード生成エンジンが動作しており、適切なプロンプト技法を理解することで、本格的なアプリを瞬く間に開発・公開できます。本ガイドは、Rork Maxでプロ品質のアプリを実現するための完全なロードマップです。

## Rork MaxのSwiftUI生成アーキテクチャ

### 内部メカニズム

Rork Maxは、ユーザーのUIスケッチとテキスト説明をClaude APIに送信し、以下のフローでSwiftUIコードを生成します：

1. **UIレイアウト解析**: スケッチから画面構造を認識
2. **コンポーネント分解**: 再利用可能な構造体に分割
3. **ステート管理設計**: `@State`, `@StateObject`, `@EnvironmentObject` を自動判定
4. **プレビュー生成**: Preview で即座に確認可能

この過程で、モデルはAppleの最新SwiftUI APIベストプラクティスを参考にしており、iOS 17・18対応のコードを生成します。

### 品質を左右する要素

生成コードの品質は、入力プロンプトの具体性に完全に依存します。曖昧な指示と明確な指示では、出力の質が大きく異なります。

## 高品質なプロンプト術

### パターン1: コンポーネント分割指定

**低品質な指示例**:
```
ユーザープロフィールページを作って
```

**高品質な指示例**:
```
ユーザープロフィールページを作成してください。以下の独立したコンポーネントに分割すること：
- ProfileHeaderView: ユーザー写真, 名前, フォローボタン
- StatisticsView: フォロワー数, 投稿数, いいね数を3列レイアウト
- BioView: 自己紹介テキスト（最大150文字、改行対応）
- ActionButtonsView: メッセージ, シェア, 報告ボタン（HStack, 均等配置）

各コンポーネントは独立した View struct として定義し、親の ProfileView から呼び出すこと。
```

このように分割を明示すると、Rork Maxは責任分離の明確なコードを生成します。結果として、テストしやすく、再利用可能な実装になります。

### パターン2: ステート管理の明示

**低品質な指示例**:
```
商品リスト画面。商品を検索できるようにして
```

**高品質な指示例**:
```
@StateObject を使用した SearchViewModel を作成してください。以下の状態を管理すること：
- @Published var searchText: String
- @Published var results: [Product]
- @Published var isLoading: Bool
- @Published var error: String?

検索フィールド（TextField）の入力時に、searchText が変更されるたび、debounce（0.5秒待機）して API呼び出しを行う ViewModel メソッドを実装すること。
```

このレベルの具体性があれば、生成されるコードはプロダクション品質に近づきます。

### パターン3: アニメーション・トランジション指定

**低品質な指示例**:
```
ボタンを押すとモーダルが開く
```

**高品質な指示例**:
```
@State var showDetail: Bool = false を用いて、ボタンタップで DetailView をモーダル表示してください。
トランジション: .scale（スケールアニメーション, duration 0.3秒）
モーダル背景: 半透明黒（opacity 0.3）
閉じるボタン: 右上コーナー, × アイコン

また、モーダルが完全に表示された後、自動的に 3 秒後に自動クローズするタイマーを実装すること（DispatchQueue.main.asyncAfter）。
```

アニメーション細部が明記されると、ユーザー体験が飛躍的に向上します。

## 高度なAPIの組み込み実装パターン

Rork Maxで生成したベースアプリに、さらに高度なフレームワークを組み込むことで、真のネイティブアプリ機能を実現できます。

### HealthKit の統合

HealthKitを使用したフィットネスアプリの例：

```swift
import HealthKit

class HealthKitManager: NSObject, ObservableObject {
    let healthStore = HKHealthStore()
    @Published var stepCount: Int = 0
    @Published var heartRate: Int = 0

    func requestAuthorization() {
        let typesToShare: Set<HKSampleType> = []
        let typesToRead: Set<HKObjectType> = [
            HKObjectType.quantityType(forIdentifier: .stepCount)\!,
            HKObjectType.quantityType(forIdentifier: .heartRate)\!
        ]

        healthStore.requestAuthorization(toShare: typesToShare, read: typesToRead) { success, error in
            if success {
                self.fetchStepCount()
                self.fetchHeartRate()
            }
        }
    }

    func fetchStepCount() {
        let stepType = HKObjectType.quantityType(forIdentifier: .stepCount)\!
        let query = HKStatisticsQuery(quantityType: stepType, quantitySamplePredicate: nil, options: .cumulativeSum) { _, result, _ in
            DispatchQueue.main.async {
                self.stepCount = Int(result?.sumQuantity()?.doubleValue(for: HKUnit.count()) ?? 0)
            }
        }
        healthStore.execute(query)
    }

    func fetchHeartRate() {
        let heartRateType = HKObjectType.quantityType(forIdentifier: .heartRate)\!
        let query = HKSampleQuery(sampleType: heartRateType, predicate: nil, limit: 1, sortDescriptors: [NSSortDescriptor(key: HKSampleSortIdentifierStartDate, ascending: false)]) { _, samples, _ in
            DispatchQueue.main.async {
                self.heartRate = Int(samples?.first?.quantity(for: HKUnit(from: "count/min"))?.doubleValue() ?? 0)
            }
        }
        healthStore.execute(query)
    }
}
```

Rork MaxのUIと組み合わせることで、HealthKit統合されたアプリが短時間で完成します。

### StoreKit 2 による課金機能

アプリ内課金の実装：

```swift
import StoreKit

class PurchaseManager: NSObject, ObservableObject {
    @Published var unlockedFeatures: Set<String> = []

    func requestProducts() async {
        do {
            let productIDs = ["com.yourapp.premium", "com.yourapp.lifetimeAccess"]
            let products = try await Product.products(for: productIDs)
            
            for product in products {
                print("Product: \(product.displayName) - \(product.displayPrice)")
            }
        } catch {
            print("Failed to fetch products: \(error)")
        }
    }

    func purchase(_ product: Product) async {
        do {
            let result = try await product.purchase()
            
            switch result {
            case .success(let verification):
                if case .verified(let transaction) = verification {
                    unlockedFeatures.insert(product.id)
                    await transaction.finish()
                }
            case .userCancelled, .pending:
                break
            @unknown default:
                break
            }
        } catch {
            print("Purchase failed: \(error)")
        }
    }
}
```

@MainActor
final class ViewModel: ObservableObject {

    func fetchData() async {
        // 非同期処理でUI更新を最小化

## App Store最適化（ASO）の完全ガイド

### スクリーンショット最適化

App Storeに表示されるスクリーンショットは、ユーザーの購買判断を左右する最重要要素です。

**スクリーンショットの配置順序（推奨）**:
1. **1枚目**: 主要機能をビジュアルで表現（テキスト不要）
2. **2枚目**: 核となる利点（キャッチコピー + アイコン）
3. **3枚目**: 複数の機能を箇条書き表示
4. **4〜5枚目**: 使用シーン・実生活での活用イメージ

**テキストオーバーレイのベストプラクティス**:
- フォントサイズ: 28pt以上（表示確認済み）
- 色: 背景と十分なコントラスト（AAA準拠）
- 情報: 最大3行、1行最大20文字
- 位置: 画面下部（ナビゲーションバーに被らない）

### キーワード戦略

App Storeの検索キーワード（最大100文字）の設定：

1. **プライマリキーワード**: アプリの主要機能（最初の20文字）
   - 例: "フィットネストラッカー, ワークアウト"

2. **セカンダリキーワード**: ユーザーのニーズ・ユースケース
   - 例: "健康管理, 運動記録, スマートウォッチ連携"

3. **ロングテール**: 競争性が低いが高転換率のキーワード
   - 例: "無料フィットネスアプリ, Apple Watchアプリ"

Keyword Tool（App Annie等）を使用し、検索量・競争性・転換率のバランスを取ること。

### アプリ説明文（Subtitle + Description）

**Subtitle（最大30文字）**:
```
フィットネス追跡、目標達成のガイド
```

**説明文（最大4,000文字）**:
```
【主要機能】
✓ AI フィットネスコーチ：個人に最適化されたワークアウトプラン
✓ スマートウォッチ連携：Apple Watch, Garminなど対応
✓ 栄養トラッキング：食事管理と体重推移の一括管理
✓ コミュニティ機能：友人と目標共有、相互励まし

【ユーザーの声】
「3ヶ月で5kg減量できた」 — Sarah, 32歳
「毎日使うのが習慣になった」 — Mike, 28歳

【プライバシー】
Apple Healthデータは暗号化され、サーバーには保存されません。
```

**効果を高めるポイント**:
- 最初の2行で主要機能を明記
- 絵文字を適度に使用（3〜5個）
- 数字・証拠・ユーザー声を含める
- 各言語版で翻訳（直訳ではなく現地化）

### A/Bテスト戦略

App Store Connect の「App情報」で複数のバージョンをテスト：

1. **テスト1**: Subtitle のみ変更
   - A: "フィットネストラッカー"
   - B: "AI搭載フィットネスコーチ"
2. **テスト期間**: 2週間以上
3. **メトリクス**: インプレッション数, 詳細ページ閲覧率, ダウンロード数を比較
4. **勝者**: 詳細ページ閲覧率 + ダウンロード数が高いものを採用

## App Store審査対策 — 完全チェックリスト

以下の30項目をすべて確認してから審査提出すること。審査通過率が大幅に向上します。

### 機能・動作（15項目）
- [ ] アプリが起動時にクラッシュしない（全デバイス・iOS最新版で確認）
- [ ] すべてのボタン・リンクが動作する
- [ ] バックグラウンド動作が正常（音声再生、位置情報追跡等）
- [ ] ネットワーク接続喪失時の処理（エラーメッセージ表示、リトライ機能）
- [ ] オフラインモード対応（可能な範囲で機能継続）
- [ ] メモリリーク・バッテリー消費が過度でない
- [ ] データベース操作が正常（追加、更新、削除、フィルター）
- [ ] ファイルアップロード・ダウンロードが正常
- [ ] 多言語対応（日本語、英語等で正しく表示）
- [ ] アクセシビリティ対応（VoiceOver対応、テキストサイズ調整等）
- [ ] カメラ・マイク・位置情報のパーミッション要求が適切
- [ ] Apple Pay / In-App Purchase が正常に動作
- [ ] ウィジェット機能（有る場合）が正常に動作
- [ ] Watch アプリ / iPad での表示が正常
- [ ] デバイス回転（ポートレート・ランドスケープ）で UI が適切に調整される

### メタデータ（10項目）
- [ ] アプリアイコン: 1024×1024px, 完全背景色, 角丸なし
- [ ] スクリーンショット: 要求解像度（例 1290×2796px）で統一, 5〜10枚
- [ ] スクリーンショットに外部ツール・広告が含まれていない
- [ ] アプリ名: ブランド名 + 機能説明（最大30文字）
- [ ] Subtitle: 簡潔かつ魅力的（最大30文字）
- [ ] 説明文: 明確で誤字なし, 第一人称「私」「俺」は避ける
- [ ] キーワード: アプリと無関係なキーワードを含めない
- [ ] サポート URL: 有効なウェブサイト（404でない）
- [ ] プライバシーポリシー URL: 有効で、内容がアプリ機能と一致
- [ ] プレビュー動画（有る場合）: 最大30秒, H.264, アプリの実際の使用シーンを表示

### ガイドライン対応（5項目）
- [ ] 違法コンテンツ（著作権侵害, 不正な政治宣伝等）を含まない
- [ ] 過度な広告（全画面広告, pop-up）を含まない
- [ ] ユーザー情報を勝手に収集・共有しない
- [ ] 詐欺・フィッシング・ポンジスキーム的な機能がない
- [ ] 他のアプリ（Apple標準アプリ等）と競合する基本機能をコピーしていない

このチェックリストを完璧にクリアすることで、審査一回で通過する可能性が大幅に上がります。

## 実装例: 完全なフィットネスアプリ

上記の知見を統合した、実装可能なフィットネスアプリの簡易版を以下に示します：

```swift
// 1. ViewModel（ステート管理）
@MainActor
final class FitnessViewModel: ObservableObject {
    @Published var workouts: [Workout] = []
    @Published var isLoading = false

    func fetchWorkouts() async {
        isLoading = true
        // API呼び出し
        let newWorkouts = await WorkoutAPI.fetchAll()
        self.workouts = newWorkouts
        isLoading = false
    }
}

// 2. メインビュー
struct FitnessAppView: View {
    @StateObject private var viewModel = FitnessViewModel()

    var body: some View {
        NavigationStack {
            VStack {
                // ヘッダー
                FitnessHeaderView()

                // ワークアウトリスト
                if viewModel.isLoading {
                    ProgressView()
                } else {
                    List(viewModel.workouts, id: \.id) { workout in
                        NavigationLink(destination: WorkoutDetailView(workout: workout)) {
                            WorkoutRowView(workout: workout)
                        }
                    }
                }
            }
            .navigationTitle("My Workouts")
            .task {
                await viewModel.fetchWorkouts()
            }
        }
    }
}
```
