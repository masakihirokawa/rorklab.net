## Siri と App Intents とは？なぜ Rork Max が必要なのか

「Hey Siri、今日のタスクを追加して」「Siri、フィットネスアプリで5kmランを記録して」——ユーザーが声でアプリを操作できる**Siri統合**は、アプリの差別化要素として非常に有効です。

iOSのSiri対応は2つの世代に分かれます。

- **SiriKit（iOS 10〜）**: ドメインベースの旧API。メッセージ・電話・支払いなど限定されたカテゴリのみ対応
- **App Intents（iOS 16+）**: 現代的なSwift APIで、あらゆるアプリ機能をSiri・Spotlightショートカット・ウィジェット・フォーカスフィルターに対応可能

App IntentsはReact Nativeでは実装が非常に困難であり、**ネイティブSwiftが必須**です。そのためRork Maxを使うことで、複雑なApp Intentsをプロンプトだけで実装できるようになります。

この記事では、タスク管理アプリを例にApp IntentsとSiriショートカットを実装する手順を解説します。

## App Intents の基礎知識

### App Intents フレームワークの仕組み

App Intentsは`AppIntent`プロトコルに準拠したSwift structで定義します。各Intentは以下の要素を持ちます。

| 要素 | 役割 |
|------|------|
| `title` | Siri・ショートカットアプリに表示される名前 |
| `@Parameter` | ユーザーまたはSiriから受け取る入力値 |
| `perform()` | 実際の処理を行う非同期関数 |
| `IntentResult` | Siriに返す結果（成功/失敗/ダイアログ） |

### Siri との連携

App Intentsで定義したIntentは自動的に以下と連携します。

- **Siri（音声操作）**: 「Hey Siri、[アプリ名]でタスクを追加して」
- **ショートカットアプリ**: ユーザーが自分だけのオートメーションを作成可能
- **Spotlight（iOS 16+）**: 検索結果からIntentをワンタップで実行
- **ウィジェット**: ホーム画面からボタン一つでIntentを実行（iOS 17+）

## Rork Max で App Intents を実装する

### Step 1: プロジェクトの作成とプロンプト設計

[Rork Max](https://rork.com/max)で新規プロジェクトを作成し、以下のようなプロンプトを入力します。

```
タスク管理アプリを作成してください。
以下のApp Intentsを実装してください：

1. AddTaskIntent — タスクを追加する（パラメータ: タスク名、期限日）
2. ListTasksIntent — 今日期限のタスク一覧をSiriに読み上げさせる
3. CompleteTaskIntent — タスクを完了済みにする（パラメータ: タスク名）

各IntentはSiriとショートカットアプリから呼び出せるようにしてください。
```

### Step 2: App Intent の定義コード

Rork Maxが生成するApp Intentsの実装例です。

```swift
import AppIntents
import Foundation

// ─── 1. タスク追加 Intent ───
struct AddTaskIntent: AppIntent {
    // Siriとショートカットアプリに表示される名前
    static var title: LocalizedStringResource = "タスクを追加"
    static var description = IntentDescription("新しいタスクをアプリに追加します。")

    // Siriが音声でユーザーに尋ねるパラメータ
    @Parameter(title: "タスク名")
    var taskTitle: String

    @Parameter(title: "期限日", default: .today)
    var dueDate: Date

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        // TaskRepositoryはアプリのデータ管理クラス
        await TaskRepository.shared.add(
            title: taskTitle,
            dueDate: dueDate
        )

        // Siriが読み上げるレスポンス
        return .result(dialog: "「\(taskTitle)」を\(formattedDate(dueDate))までのタスクとして追加しました。")
    }

    private func formattedDate(_ date: Date) -> String {
        let formatter = DateFormatter()
        formatter.locale = Locale(identifier: "ja_JP")
        formatter.dateStyle = .short
        return formatter.string(from: date)
    }
}

// ─── 2. タスク一覧 Intent ───
struct ListTasksIntent: AppIntent {
    static var title: LocalizedStringResource = "今日のタスクを確認"
    static var description = IntentDescription("今日期限のタスク一覧を読み上げます。")

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        let tasks = await TaskRepository.shared.tasksDueToday()

        guard !tasks.isEmpty else {
            return .result(dialog: "今日期限のタスクはありません。")
        }

        let list = tasks.enumerated()
            .map { "\($0.offset + 1)番目: \($0.element.title)" }
            .joined(separator: "、")

        return .result(dialog: "今日期限のタスクは\(tasks.count)件です。\(list)")
    }
}

// ─── 3. タスク完了 Intent ───
struct CompleteTaskIntent: AppIntent {
    static var title: LocalizedStringResource = "タスクを完了"
    static var description = IntentDescription("指定したタスクを完了済みにします。")

    @Parameter(title: "タスク名")
    var taskTitle: String

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        let success = await TaskRepository.shared.complete(title: taskTitle)

        if success {
            return .result(dialog: "「\(taskTitle)」を完了しました。")
        } else {
            throw AppIntentError.taskNotFound(taskTitle)
        }
    }
}

// カスタムエラー定義
enum AppIntentError: Error, CustomLocalizedStringResourceConvertible {
    case taskNotFound(String)

    var localizedStringResource: LocalizedStringResource {
        switch self {
        case .taskNotFound(let name):
            return "「\(name)」というタスクが見つかりませんでした。"
        }
    }
}

// 期待する出力:
// - Siriで「Hey Siri、[アプリ名]でタスクを追加して」と言うと
//   Siriが「タスク名は何ですか？」と聞いてくる
// - タスク名を答えると「追加しました」と応答する
```

### Step 3: ShortcutsProvider でインテントをグループ化

複数のIntentをショートカットアプリで見やすくグループ化できます。

```swift
import AppIntents

// ショートカットアプリでの表示設定
struct TaskShortcuts: AppShortcutsProvider {
    static var appShortcuts: [AppShortcut] {
        AppShortcut(
            intent: AddTaskIntent(),
            phrases: [
                "タスクを追加",
                "\(.applicationName)でタスクを追加",
                "\(.applicationName)でやること追加"
            ],
            shortTitle: "タスク追加",
            systemImageName: "plus.circle.fill"
        )
        AppShortcut(
            intent: ListTasksIntent(),
            phrases: [
                "今日のタスク確認",
                "\(.applicationName)で今日やることを確認"
            ],
            shortTitle: "タスク一覧",
            systemImageName: "list.bullet"
        )
        AppShortcut(
            intent: CompleteTaskIntent(),
            phrases: [
                "\(.applicationName)でタスクを完了",
                "タスクを完了にして"
            ],
            shortTitle: "タスク完了",
            systemImageName: "checkmark.circle"
        )
    }
}
```

`phrases`に登録した言葉をSiriに話しかけるだけでIntentが起動します。

## App Shortcuts ウィジェットとの連携

iOS 17からApp Intentsはホーム画面ウィジェットのボタンとも連携できます。Rork Maxに以下のようなプロンプトを追加します。

```
ホーム画面ウィジェットにボタンを追加してください：
- 「タスク追加」ボタン（AppIntent: AddTaskIntent）
- 「今日のタスク」ボタン（AppIntent: ListTasksIntent）
ウィジェットサイズ: Small、Medium
```

生成されるウィジェットコードの核心部分：

```swift
import SwiftUI
import WidgetKit
import AppIntents

// ウィジェット内のボタン（iOS 17以上でAppIntentと連携）
struct TaskWidgetView: View {
    var entry: TaskEntry

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text("タスク")
                .font(.caption)
                .foregroundStyle(.secondary)

            // AppIntentと連携するボタン（ウィジェットから直接実行）
            Button(intent: AddTaskIntent()) {
                Label("追加", systemImage: "plus.circle.fill")
                    .font(.caption.bold())
            }
            .buttonStyle(.borderedProminent)
            .tint(.blue)

            if entry.todayTaskCount > 0 {
                Text("今日: \(entry.todayTaskCount)件")
                    .font(.caption2)
            }
        }
        .padding()
    }
}

// 期待する出力:
// ホーム画面のウィジェットに「追加」ボタンが表示され、
// タップするとタスク追加ダイアログが開く（アプリを開かずに操作可能）
```

## 実際のユースケース別プロンプト例

### フィットネスアプリ

```
フィットネスアプリにApp Intentsを追加してください：
- StartWorkoutIntent: 種目（ランニング/筋トレ/ヨガ）を指定してワークアウト開始
- LogWeightIntent: 体重を記録（パラメータ: 体重kg）
- DailyProgressIntent: 今日の歩数とカロリーをSiriに読み上げさせる
Siriフレーズ: 「ランニングを記録して」「今日の運動進捗を教えて」
```

### ECアプリ

```
ショッピングアプリにApp Intentsを実装してください：
- AddToCartIntent: 商品名で検索してカートに追加（Siriで「〇〇をカートに入れて」）
- CheckOrderStatusIntent: 最新の注文状況を確認
- ReorderIntent: 過去の購入から同じ商品を再注文
```

### タイマー・ポモドーロアプリ

```
ポモドーロアプリのApp Intentsを作成してください：
- StartPomodoroIntent: 作業時間（デフォルト25分）を指定してタイマー開始
- PauseTimerIntent: タイマーを一時停止
Siriフレーズ: 「ポモドーロを始めて」「タイマーを止めて」
```

## よくあるエラーと対処法

### エラー1: Siriが「このアプリではその操作はできません」と返す

**原因**: `AppShortcutsProvider`の`phrases`に登録した言葉とSiriが認識した言葉が一致しない
**対処**: Rork Maxに「SiriのフレーズをNLP（自然言語）パターンを使って改善して」と指示する

### エラー2: `@Parameter`の値がSiriから渡されない

**原因**: Parameterの`title`が不明確でSiriが何を聞けばいいか判断できない
**対処**: `@Parameter(title: "タスク名", description: "追加するタスクの名前を教えてください")`のように`description`を追加する

### エラー3: ウィジェットボタンを押してもIntentが実行されない

**原因**: iOS 17未満のデバイスではウィジェットのApp Intent連携が動作しない
**対処**: iOS 17以上を対象にする、またはボタンタップでDeepLinkを使う代替実装をRork Maxに依頼する

### エラー4: `perform()`内でCoreDataにアクセスするとクラッシュ

**原因**: App ExtensionとメインアプリでCoreDataのApp Groupが共有されていない
**対処**: Rork Maxに「App Intents ExtensionからCoreDataにアクセスするためのApp Group設定を追加して」と指示する

## FAQ

**Q. App IntentsはiPhone以外（iPad・Mac）でも動きますか？**

A. はい。App IntentsはiPhone、iPad、MacBook（macOS 13+）すべてで動作します。また、Apple WatchアプリとApp Intentsを連携させることも可能で「Hey Siri、Apple Watchから〇〇して」という操作も実現できます。

**Q. 既存のRork（React Native版）アプリにApp Intentsを追加できますか？**

A. React Nativeではネイティブモジュールを使って限定的に対応可能ですが、完全な対応にはRork Maxへの移行が推奨されます。特にウィジェット連携やSiriの自然言語理解をフル活用するにはネイティブSwiftが必要です。

**Q. App Intentsに課金機能（サブスクリプション確認など）を追加できますか？**

A. 可能です。`perform()`内で購読状態をチェックし、未加入ユーザーにはSiriを通じてアップグレードを促すダイアログを返せます。ただし、App Intents実行中にRevenueCatなどの購入フローを直接呼び出すことはできないため、アプリを開くDeepLinkで誘導するのが一般的な実装パターンです。

**Q. Siriで複数のパラメータを一度に指定できますか？**

A. はい。「タスクを明日までで買い物という名前で追加して」のように、一文で複数パラメータを指定できます。App IntentsのNLP処理がパラメータを自動的にマッピングします。`@Parameter`の定義順序と`title`の明確さが精度に影響します。

**Q. App Intentsのデバッグはどうやりますか？**

A. Rork Maxのシミュレータ環境でSiriショートカットアプリを使ってテストできます。また、Xcodeのログに`print()`出力が表示されます。Rork Maxに「App Intentsのデバッグログを詳細に出力するコードを追加して」と指示すると、各パラメータの受け取り値をコンソールで確認できます。

## まとめと次のステップ

App IntentsとSiriの統合は、ユーザーがアプリを「開かずに使う」体験を作り出します。Rork Maxを使えば、複雑なApp IntentsフレームワークをAIに生成させ、ビジネスロジックの設計に集中できます。

実装のポイントをまとめます。

1. **Intentの粒度**: 1つのIntentは1つの明確なアクションに絞る（追加・削除・確認を混ぜない）
2. **パラメータのtitleとdescription**: Siriがユーザーに何を聞くか明確に指定する
3. **AppShortcutsProvider**: フレーズを登録してSiriから自然に呼び出せるようにする
4. **エラーハンドリング**: `CustomLocalizedStringResourceConvertible`でSiriが読み上げられるエラーメッセージを定義する
