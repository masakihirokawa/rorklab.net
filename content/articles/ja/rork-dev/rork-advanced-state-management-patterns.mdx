## なぜ状態管理がRorkアプリ開発の鍵なのか

Rorkを使ったアプリ開発では、AIがUIコンポーネントの生成を大幅に自動化してくれます。しかし、アプリの規模が大きくなるにつれて「データの流れ」が複雑化し、適切な状態管理なしではバグの温床になります。

画面間でデータを共有したい、APIからの非同期データを管理したい、オフラインでもアプリを使えるようにしたい。こうした要件に応えるには、しっかりとした状態管理のアーキテクチャが必要です。

この記事では、Rorkアプリで使える状態管理パターンを、小規模から大規模まで段階的に解説します。すべてのパターンにコード例を付けているので、プロジェクトの規模に合わせて選んでください。

## パターン1: コンポーネントローカルステート — 小規模アプリの基本

最もシンプルなパターンです。各コンポーネントが自分のステートを管理します。

```typescript
// components/Counter.tsx — ローカルステートの基本
import React, { useState, useCallback } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export const Counter: React.FC = () => {
  // ローカルステート — このコンポーネント内でのみ有効
  const [count, setCount] = useState(0);

  const increment = useCallback(() => {
    setCount(prev => prev + 1);
  }, []);

  const decrement = useCallback(() => {
    setCount(prev => Math.max(0, prev - 1));
  }, []);

  return (
    <View style={styles.container}>
      <TouchableOpacity onPress={decrement} style={styles.button}>
        <Text style={styles.buttonText}>-</Text>
      </TouchableOpacity>
      <Text style={styles.count}>{count}</Text>
      <TouchableOpacity onPress={increment} style={styles.button}>
        <Text style={styles.buttonText}>+</Text>
      </TouchableOpacity>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flexDirection: 'row', alignItems: 'center', gap: 16 },
  button: {
    width: 44, height: 44, borderRadius: 22,
    backgroundColor: '#007AFF', justifyContent: 'center', alignItems: 'center'
  },
  buttonText: { color: '#fff', fontSize: 20, fontWeight: 'bold' },
  count: { fontSize: 24, fontWeight: '600', minWidth: 40, textAlign: 'center' }
});
```

このパターンは、状態が1つのコンポーネントに閉じている場合に最適です。フォームの入力値、トグルの開閉状態、アニメーションの進行度など、局所的なデータに使いましょう。

## パターン2: Context + useReducer — 中規模アプリのグローバルステート

複数の画面でデータを共有する必要がある場合は、Context APIと`useReducer`を組み合わせます。

```typescript
// store/AppContext.tsx — グローバルステート管理
import React, { createContext, useContext, useReducer, ReactNode } from 'react';

// 型定義
interface AppState {
  user: { id: string; name: string; email: string } | null;
  theme: 'light' | 'dark';
  notifications: Notification[];
  isLoading: boolean;
}

interface Notification {
  id: string;
  message: string;
  read: boolean;
  createdAt: string;
}

type AppAction =
  | { type: 'SET_USER'; payload: AppState['user'] }
  | { type: 'SET_THEME'; payload: AppState['theme'] }
  | { type: 'ADD_NOTIFICATION'; payload: Notification }
  | { type: 'MARK_READ'; payload: string }
  | { type: 'SET_LOADING'; payload: boolean };

// 初期状態
const initialState: AppState = {
  user: null,
  theme: 'light',
  notifications: [],
  isLoading: false,
};

// Reducer — 純粋関数で状態遷移を定義
function appReducer(state: AppState, action: AppAction): AppState {
  switch (action.type) {
    case 'SET_USER':
      return { ...state, user: action.payload };
    case 'SET_THEME':
      return { ...state, theme: action.payload };
    case 'ADD_NOTIFICATION':
      return {
        ...state,
        notifications: [action.payload, ...state.notifications]
      };
    case 'MARK_READ':
      return {
        ...state,
        notifications: state.notifications.map(n =>
          n.id === action.payload ? { ...n, read: true } : n
        ),
      };
    case 'SET_LOADING':
      return { ...state, isLoading: action.payload };
    default:
      return state;
  }
}

// Context作成
const AppContext = createContext<{
  state: AppState;
  dispatch: React.Dispatch<AppAction>;
} | null>(null);

// Provider コンポーネント
export const AppProvider: React.FC<{ children: ReactNode }> = ({ children }) => {
  const [state, dispatch] = useReducer(appReducer, initialState);

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
};

// カスタムフック — 型安全なアクセス
export function useAppState() {
  const context = useContext(AppContext);
  if (!context) {
    throw new Error('useAppState must be used within AppProvider');
  }
  return context;
}
```

使用例:

```typescript
// screens/ProfileScreen.tsx
import { useAppState } from '../store/AppContext';

export const ProfileScreen: React.FC = () => {
  const { state, dispatch } = useAppState();

  const toggleTheme = () => {
    dispatch({
      type: 'SET_THEME',
      payload: state.theme === 'light' ? 'dark' : 'light'
    });
  };

  return (
    <View>
      <Text>ようこそ、{state.user?.name ?? 'ゲスト'}さん</Text>
      <Text>テーマ: {state.theme}</Text>
      <TouchableOpacity onPress={toggleTheme}>
        <Text>テーマ切り替え</Text>
      </TouchableOpacity>
    </View>
  );
};
```

## パターン3: Zustandストア — 大規模アプリの本命

大規模なアプリでは、Zustandが最もバランスの良い選択肢です。ボイラープレートが少なく、TypeScriptとの相性も抜群です。

```typescript
// store/useStore.ts — Zustandストア
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface Product {
  id: string;
  name: string;
  price: number;
  image: string;
}

interface CartItem extends Product {
  quantity: number;
}

interface StoreState {
  // カート
  cart: CartItem[];
  addToCart: (product: Product) => void;
  removeFromCart: (productId: string) => void;
  updateQuantity: (productId: string, quantity: number) => void;
  clearCart: () => void;
  getCartTotal: () => number;

  // お気に入り
  favorites: string[];
  toggleFavorite: (productId: string) => void;
  isFavorite: (productId: string) => boolean;

  // 検索履歴
  searchHistory: string[];
  addSearchQuery: (query: string) => void;
  clearSearchHistory: () => void;
}

export const useStore = create<StoreState>()(
  persist(
    (set, get) => ({
      // カート
      cart: [],

      addToCart: (product) => set((state) => {
        const existing = state.cart.find(item => item.id === product.id);
        if (existing) {
          return {
            cart: state.cart.map(item =>
              item.id === product.id
                ? { ...item, quantity: item.quantity + 1 }
                : item
            ),
          };
        }
        return { cart: [...state.cart, { ...product, quantity: 1 }] };
      }),

      removeFromCart: (productId) => set((state) => ({
        cart: state.cart.filter(item => item.id !== productId),
      })),

      updateQuantity: (productId, quantity) => set((state) => ({
        cart: quantity <= 0
          ? state.cart.filter(item => item.id !== productId)
          : state.cart.map(item =>
              item.id === productId ? { ...item, quantity } : item
            ),
      })),

      clearCart: () => set({ cart: [] }),

      getCartTotal: () => {
        return get().cart.reduce(
          (total, item) => total + item.price * item.quantity, 0
        );
      },

      // お気に入り
      favorites: [],

      toggleFavorite: (productId) => set((state) => ({
        favorites: state.favorites.includes(productId)
          ? state.favorites.filter(id => id !== productId)
          : [...state.favorites, productId],
      })),

      isFavorite: (productId) => {
        return get().favorites.includes(productId);
      },

      // 検索履歴
      searchHistory: [],

      addSearchQuery: (query) => set((state) => ({
        searchHistory: [
          query,
          ...state.searchHistory.filter(q => q !== query)
        ].slice(0, 20),  // 最新20件を保持
      })),

      clearSearchHistory: () => set({ searchHistory: [] }),
    }),
    {
      name: 'app-storage',
      storage: createJSONStorage(() => AsyncStorage),
      partialize: (state) => ({
        cart: state.cart,
        favorites: state.favorites,
        searchHistory: state.searchHistory,
      }),
    }
  )
);
```

## パターン4: 非同期データ管理 — TanStack Queryとの統合

サーバーから取得するデータは、ローカルステートとは別に管理するのが鉄則です。

```typescript
// hooks/useProducts.ts — TanStack Queryによるデータフェッチ
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';

interface Product {
  id: string;
  name: string;
  price: number;
  description: string;
  category: string;
}

// API関数
const api = {
  getProducts: async (category?: string): Promise<Product[]> => {
    const url = category
      ? `/api/products?category=${category}`
      : '/api/products';
    const res = await fetch(url);
    if (!res.ok) throw new Error('Failed to fetch products');
    return res.json();
  },

  getProduct: async (id: string): Promise<Product> => {
    const res = await fetch(`/api/products/${id}`);
    if (!res.ok) throw new Error('Product not found');
    return res.json();
  },
};

// 商品一覧の取得
export function useProducts(category?: string) {
  return useQuery({
    queryKey: ['products', category],
    queryFn: () => api.getProducts(category),
    staleTime: 5 * 60 * 1000,   // 5分間はキャッシュを使用
    gcTime: 30 * 60 * 1000,     // 30分後にキャッシュ破棄
    retry: 2,
  });
}

// 商品詳細の取得
export function useProduct(id: string) {
  return useQuery({
    queryKey: ['product', id],
    queryFn: () => api.getProduct(id),
    enabled: !!id,  // idがある場合のみフェッチ
  });
}

// 使用例
// const { data: products, isLoading, error } = useProducts('electronics');
```

## パターン5: オフライン対応 — 楽観的更新とキュー

モバイルアプリではネットワーク接続が不安定な場面が頻繁にあります。オフラインでも使えるアプリにするパターンです。

```typescript
// store/offlineQueue.ts — オフライン操作キュー
import NetInfo from '@react-native-community/netinfo';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface QueuedAction {
  id: string;
  type: string;
  payload: any;
  timestamp: number;
  retryCount: number;
}

class OfflineQueue {
  private queue: QueuedAction[] = [];
  private isProcessing = false;
  private readonly STORAGE_KEY = 'offline_queue';
  private readonly MAX_RETRIES = 3;

  async init() {
    // 永続化されたキューを復元
    const stored = await AsyncStorage.getItem(this.STORAGE_KEY);
    if (stored) {
      this.queue = JSON.parse(stored);
    }

    // ネットワーク状態を監視
    NetInfo.addEventListener(state => {
      if (state.isConnected && this.queue.length > 0) {
        this.processQueue();
      }
    });
  }

  async enqueue(action: Omit<QueuedAction, 'id' | 'timestamp' | 'retryCount'>) {
    const queuedAction: QueuedAction = {
      ...action,
      id: `${Date.now()}-${Math.random().toString(36).slice(2)}`,
      timestamp: Date.now(),
      retryCount: 0,
    };

    this.queue.push(queuedAction);
    await this.persist();

    // オンラインなら即座に処理
    const netState = await NetInfo.fetch();
    if (netState.isConnected) {
      this.processQueue();
    }
  }

  private async processQueue() {
    if (this.isProcessing || this.queue.length === 0) return;
    this.isProcessing = true;

    while (this.queue.length > 0) {
      const action = this.queue[0];

      try {
        await this.executeAction(action);
        this.queue.shift(); // 成功したら削除
        await this.persist();
      } catch (error) {
        action.retryCount++;
        if (action.retryCount >= this.MAX_RETRIES) {
          console.error(`Action failed after ${this.MAX_RETRIES} retries:`, action);
          this.queue.shift();
          await this.persist();
        } else {
          break; // リトライのため処理を中断
        }
      }
    }

    this.isProcessing = false;
  }

  private async executeAction(action: QueuedAction): Promise<void> {
    // アクションタイプに応じてAPIを呼び出す
    const response = await fetch(`/api/${action.type}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(action.payload),
    });

    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }
  }

  private async persist() {
    await AsyncStorage.setItem(
      this.STORAGE_KEY,
      JSON.stringify(this.queue)
    );
  }
}

export const offlineQueue = new OfflineQueue();
```

## パフォーマンス最適化のポイント

状態管理のパフォーマンスを最適化するための実践的なテクニックを紹介します。

### セレクターによる不要な再レンダリングの防止

```typescript
// Zustandのセレクターパターン
import { useStore } from './store/useStore';
import { shallow } from 'zustand/shallow';

// ❌ 悪い例 — ストア全体を購読（どんな変更でも再レンダリング）
const Component = () => {
  const store = useStore();
  return <Text>{store.cart.length}</Text>;
};

// ✅ 良い例 — 必要な値だけを購読
const Component = () => {
  const cartCount = useStore(state => state.cart.length);
  return <Text>{cartCount}</Text>;
};

// ✅ 複数の値を購読する場合はshallow比較
const Component = () => {
  const { cart, favorites } = useStore(
    state => ({ cart: state.cart, favorites: state.favorites }),
    shallow
  );
  return <Text>{cart.length} / {favorites.length}</Text>;
};
```

### メモ化による計算コストの削減

```typescript
import { useMemo } from 'react';

const CartSummary = () => {
  const cart = useStore(state => state.cart);

  // 重い計算をメモ化
  const summary = useMemo(() => ({

    totalPrice: cart.reduce((sum, item) => sum + item.price * item.quantity, 0),
    uniqueItems: cart.length,

  return (
    <View>
      <Text>商品数: {summary.totalItems}</Text>
      <Text>合計: ¥{summary.totalPrice.toLocaleString()}</Text>
    </View>
  );
};
```

## よくある質問

### Q: どのパターンを選べばよいですか？

アプリの規模で判断してください。画面が5つ以下ならローカルステート＋Context、10画面以上ならZustand、サーバーデータが多いならTanStack Queryを組み合わせるのが最適です。

### Q: ReduxではなくZustandを推奨する理由は？

Zustandはボイラープレートが圧倒的に少なく、TypeScriptの型推論も自然に効きます。Rorkアプリでは開発スピードが重要なので、設定の手間が少ないZustandが適しています。

### Q: オフライン対応はどこまで必要ですか？

ユーザーの利用シーンによります。ECアプリなら「カート追加」「お気に入り登録」はオフライン対応が必須です。ニュースアプリなら記事のキャッシュ表示だけで十分な場合もあります。

## まとめ — プロジェクトに合った状態管理を選ぼう

Rorkアプリの状態管理は「小さく始めて、必要に応じて拡張する」のが鉄則です。最初からZustandやTanStack Queryを全面導入する必要はありません。アプリが成長するにつれて、適切なパターンを段階的に取り入れていきましょう。

大切なのは「データの流れが追跡可能であること」です。どの画面がどのデータを参照しているか、データの更新がどこに影響するかが明確であれば、アプリの規模が大きくなっても保守性を維持できます。
