# Rork で作るオフライン対応モバイルアプリ — AsyncStorage と NetInfo の活用術

モバイルアプリの開発において、ネットワーク接続が常に利用可能だと仮定するのは危険です。地下鉄での移動中、トンネル内、電波の悪い地域など、ユーザーは様々な環境でアプリを使用します。

本記事では、Rork を使ってオフライン対応のモバイルアプリを構築する方法を、実践的なコード例とともに解説します。`AsyncStorage` でローカルデータを永続化し、`NetInfo` でネットワーク接続を検知し、オフラインとオンラインをシームレスに切り替えるアーキテクチャをマスターしましょう。

## なぜオフライン対応が必要なのか

ユーザーのネットワーク環境は、開発者が想定するほど安定していません。

- **通勤中**: 地下鉄や移動中は接続が不安定
- **郊外地域**: 電波が弱い、接続が途切れやすい
- **国際移動**: SIM 切り替え時の一時的な接続喪失
- **バッテリー節約**: ユーザーが意図的にオフラインモードを使用

オフライン対応アプリは、これらの状況でもユーザーが作業を継続でき、接続が復帰した際に自動的にデータを同期できます。結果として、アプリの満足度が大幅に向上します。

## AsyncStorage でローカルデータを永続化

`AsyncStorage` は React Native アプリでローカルデータを保存するための標準的なソリューションです。キー・バリュー形式で、アプリを閉じた後も再起動時にデータが残ります。

### 基本的な使い方

```javascript
import AsyncStorage from '@react-native-async-storage/async-storage';

// データ保存
const saveUserData = async (userData) => {
  try {
    await AsyncStorage.setItem('user', JSON.stringify(userData));
  } catch (error) {
    console.error('Failed to save user data:', error);
  }
};

// データ取得
const getUserData = async () => {
  try {
    const userData = await AsyncStorage.getItem('user');
    return userData ? JSON.parse(userData) : null;
  } catch (error) {
    console.error('Failed to get user data:', error);
    return null;
  }
};
```

### キャッシュ戦略の実装

オフラインでも基本的な機能を提供するため、API から取得したデータをキャッシュします。

```javascript
// API データの取得と保存
const fetchAndCacheArticles = async () => {
  try {
    const response = await fetch('https://api.example.com/articles');
    const articles = await response.json();

    // 成功時はキャッシュ更新
    await AsyncStorage.setItem('articles', JSON.stringify(articles));
    await AsyncStorage.setItem('articles_timestamp', Date.now().toString());

    return articles;
  } catch (error) {
    // ネットワークエラー時はキャッシュから復帰
    const cachedArticles = await AsyncStorage.getItem('articles');
    return cachedArticles ? JSON.parse(cachedArticles) : [];
  }
};

// キャッシュの有効期限を確認
const isCacheExpired = async (maxAge = 3600000) => { // 1時間
  const timestamp = await AsyncStorage.getItem('articles_timestamp');
  if (!timestamp) return true;

  return Date.now() - parseInt(timestamp) > maxAge;
};
```

## NetInfo でネットワーク接続を検知

`NetInfo` ライブラリを使用することで、デバイスのネットワーク接続状態をリアルタイムで監視できます。

### 接続状態の監視

```javascript
import NetInfo from '@react-native-community/netinfo';

// ネットワーク状態を監視するカスタムフック
import { useEffect, useState } from 'react';

const useNetworkStatus = () => {
  const [isOnline, setIsOnline] = useState(true);

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      setIsOnline(state.isConnected && state.isInternetReachable);
    });

    return unsubscribe;
  }, []);

  return isOnline;
};
```

## Rork を使用した状態管理

Rork のリアクティブな状態管理は、オフラインアプリの構築に最適です。ローカルデータと同期状態を効率的に管理できます。

### 状態スライスの定義

```javascript
import { createSlice } from '@rorklab/state-core';

const todoSlice = createSlice({
  name: 'todos',
  initialState: {
    items: [],
    syncStatus: 'idle', // 'idle', 'syncing', 'error'
    lastSyncTime: null,
  },
  reducers: {
    addTodo: (state, action) => {
      state.items.push({
        id: Date.now(),
        text: action.payload,
        completed: false,
        synced: false,
      });
    },
    toggleTodo: (state, action) => {
      const todo = state.items.find(t => t.id === action.payload);
      if (todo) {
        todo.completed = !todo.completed;
        todo.synced = false; // オフライン変更を記録
      }
    },
    setSyncStatus: (state, action) => {
      state.syncStatus = action.payload;
    },
    markAllSynced: (state) => {
      state.items.forEach(item => {
        item.synced = true;
      });
      state.lastSyncTime = new Date().toISOString();
    },
  },
});

export default todoSlice;
```

## 実装パターン：オフライン対応 Todo アプリ

以下は、AsyncStorage + NetInfo + Rork を組み合わせた実装例です。

### ステップ 1: 初期化時にローカルデータを復元

```javascript
const initializeApp = async (dispatch) => {
  try {
    const savedTodos = await AsyncStorage.getItem('todos');
    if (savedTodos) {
      dispatch({ type: 'todos/setTodos', payload: JSON.parse(savedTodos) });
    }
  } catch (error) {
    console.error('Failed to restore todos:', error);
  }
};
```

### ステップ 2: Todo 追加時はローカル保存 + キュー追加

```javascript
const handleAddTodo = async (text, dispatch) => {
  // Redux に追加
  dispatch({ type: 'todos/addTodo', payload: text });

  // AsyncStorage に保存
  const state = store.getState();
  await AsyncStorage.setItem('todos', JSON.stringify(state.todos.items));

  // 同期待機中の場合はキューに追加
  if (!isOnline) {
    const queue = JSON.parse(await AsyncStorage.getItem('sync_queue') || '[]');
    queue.push({ action: 'addTodo', data: text });
    await AsyncStorage.setItem('sync_queue', JSON.stringify(queue));
  }
};
```

### ステップ 3: 接続復帰時に同期実行

```javascript
const setupSyncOnReconnect = (dispatch) => {
  NetInfo.addEventListener(async (state) => {
    if (state.isConnected && state.isInternetReachable) {
      console.log('Online detected - syncing pending changes...');
      await syncPendingChanges(dispatch);
    }
  });
};

const syncPendingChanges = async (dispatch) => {
  try {
    dispatch({ type: 'todos/setSyncStatus', payload: 'syncing' });

    const queue = JSON.parse(await AsyncStorage.getItem('sync_queue') || '[]');
    if (queue.length === 0) return;

    // サーバーに送信
    const response = await fetch('https://api.example.com/todos/sync', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ changes: queue }),
    });

    if (response.ok) {
      dispatch({ type: 'todos/markAllSynced' });
      await AsyncStorage.removeItem('sync_queue');
    }

    dispatch({ type: 'todos/setSyncStatus', payload: 'idle' });
  } catch (error) {
    console.error('Sync error:', error);
    dispatch({ type: 'todos/setSyncStatus', payload: 'error' });
  }
};
```

## ベストプラクティス

### 1. ユーザーに接続状態を伝える

接続状態の視覚的フィードバックは重要です：

```javascript
<View style={{
  padding: 12,
  backgroundColor: isOnline ? '#4CAF50' : '#FF9800',
}}>
  <Text style={{ color: 'white' }}>
    {isOnline ? '✓ オンライン' : '⚠ オフラインモード'}
  </Text>
</View>
```

### 2. 競合解決の実装

サーバーデータとローカルデータが相互に異なる場合、タイムスタンプベースで判定します：

```javascript
const resolveConflict = (local, server) => {
  return local.updatedAt > server.updatedAt ? local : server;
};
```

### 3. 同期キューのクリーンアップ

同期完了後は必ずキューを削除し、メモリ効率を保ちます。

### 4. ストレージサイズの監視

AsyncStorage の制限（通常 10MB）に注意し、定期的にクリーンアップします：

```javascript
const cleanupOldData = async () => {
  const allKeys = await AsyncStorage.getAllKeys();
  const now = Date.now();
  const maxAge = 7 * 24 * 60 * 60 * 1000; // 7日

  for (const key of allKeys) {
    const item = await AsyncStorage.getItem(key);
    const metadata = await AsyncStorage.getItem(`${key}_metadata`);
    
    if (metadata) {
      const created = parseInt(JSON.parse(metadata).createdAt);
      if (now - created > maxAge) {
        await AsyncStorage.removeItem(key);
      }
    }
  }
};
```

## 注意点とよくある落とし穴

- **エラーハンドリングを忘れずに**: ネットワーク操作は失敗することを想定
- **オフラインで実際にテスト**: シミュレーターではなく実機で検証
- **機密データを AsyncStorage に保存しない**: パスワードや API キーは安全なストレージへ
- **ストレージ容量を監視**: キャッシュが無限に増えないよう注意

AsyncStorage + NetInfo + Rork の組み合わせにより、真のオフライン対応アプリが実現できます。オフラインファーストの設計思想で、接続状態に左右されないユーザー体験を提供しましょう。
