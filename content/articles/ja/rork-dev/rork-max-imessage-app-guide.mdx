## iMessage アプリとは？会話の中に溶け込むネイティブ体験

友人とのチャットで使えるオリジナルステッカー、メッセージ上で完結するミニゲーム、グループでの投票や割り勘——iMessage アプリは、Apple のメッセージプラットフォーム上で動作する軽量なアプリケーションです。

iOS 10 で登場した iMessage アプリは、App Store の「iMessage」カテゴリからインストールでき、メッセージ画面のアプリドロワーから呼び出します。世界中の iPhone ユーザーが日常的に使う「メッセージ」アプリに直接統合されるため、ダウンロードのハードルが低く、バイラル性が非常に高いのが特徴です。

2026 年現在、**Rork Max** は iMessage Extension の自動生成をサポートしており、SwiftUI ベースのステッカーパックやインタラクティブなミニアプリを AI の力で構築できます。本記事では、Rork Max を使った iMessage アプリ開発の全工程をステップバイステップで解説します。

---

## 前提知識・環境準備

iMessage アプリ開発を始める前に、以下を確認してください。

- **Rork Max プラン**（$200/月）に加入していること — iMessage Extension は SwiftUI ネイティブ機能のため Rork Max が必須
- **Apple Developer Program** に登録済み（年間 $99）— App Store Connect でのプロビジョニングに必要
- iOS 17 以降のテスト端末（シミュレータでも基本動作は確認可能）
- iMessage アプリの種類を理解しておくこと:

| 種類 | 説明 | 複雑度 |
|------|------|--------|
| **ステッカーパック** | 画像ステッカーのセット。コード不要で画像を用意するだけ | ★☆☆ |
| **iMessage Extension** | メッセージ UI 内で動作するインタラクティブアプリ | ★★☆ |
| **スタンドアロン + iMessage** | 本体アプリ付きの iMessage 拡張 | ★★★ |

## Rork Max での iMessage アプリ作成ステップ

### ステップ 1: プロジェクト作成とプロンプト設計

Rork Max のダッシュボードで新規プロジェクトを作成し、以下のようなプロンプトを入力します。

```text
iMessage ステッカーパック + ミニアプリを作成してください。

【ステッカーパック】
- テーマ: かわいい猫のイラスト
- ステッカー数: 12枚（静止画 8枚 + アニメーション 4枚）
- 対応フォーマット: PNG (静止画), APNG (アニメーション)

【ミニアプリ】
- グループチャットで使える簡易投票機能
- 質問と選択肢を入力 → メッセージとして送信 → 参加者がタップで投票
- リアルタイムで結果をバブル上に表示

【デザイン】
- パステルカラー基調
- ダークモード対応
- SF Symbols を活用
```

Rork Max の AI がこのプロンプトを解析し、iMessage Extension のプロジェクト構造を自動生成します。

### ステップ 3: ステッカーパックの実装

ステッカーパックは iMessage アプリの中で最もシンプルな形態です。Rork Max では画像アセットをアップロードするだけで、`MSStickerBrowserViewController` ベースのステッカーパックが自動生成されます。

```swift
// Rork Max が生成するステッカーブラウザビュー
import Messages
import SwiftUI

struct StickerGalleryView: View {
    // ステッカーデータ（Rork Max が自動アセット管理）
    let stickers: [StickerItem] = [
        StickerItem(name: "happy_cat", isAnimated: false),
        StickerItem(name: "sleeping_cat", isAnimated: false),
        StickerItem(name: "dancing_cat", isAnimated: true),
        StickerItem(name: "waving_cat", isAnimated: true),
    ]

    var body: some View {
        ScrollView {
            LazyVGrid(columns: [
                GridItem(.flexible()),
                GridItem(.flexible()),
                GridItem(.flexible())
            ], spacing: 12) {
                ForEach(stickers, id: \.name) { sticker in
                    StickerCell(item: sticker)
                }
            }
            .padding()
        }
        .background(Color(.systemGroupedBackground))
    }
}

struct StickerItem {
    let name: String
    let isAnimated: Bool
}

// 期待する出力:
// 3列グリッドでステッカーが並び、ドラッグ＆ドロップで
// 会話のバブル上に貼り付けられる
```

**ステッカー画像の仕様:**

| 項目 | 要件 |
|------|------|
| フォーマット | PNG（静止画）/ APNG, GIF（アニメーション） |
| 推奨サイズ | 300×300 px 〜 618×618 px |
| ファイルサイズ | 静止画: 500KB 以下 / アニメーション: 500KB 以下 |
| 背景 | 透過推奨 |

### ステップ 4: インタラクティブメッセージ（投票機能）の実装

iMessage アプリの真価は、会話内で完結するインタラクティブな体験です。以下は Rork Max が生成する投票機能の例です。

```swift
// Rork Max 生成 — グループ投票メッセージ
import Messages
import SwiftUI

struct PollCreatorView: View {
    @State private var question = ""
    @State private var options: [String] = ["", ""]
    var onSend: (PollData) -> Void

    var body: some View {
        VStack(spacing: 16) {
            Text("投票を作成")
                .font(.headline)

            TextField("質問を入力", text: $question)
                .textFieldStyle(.roundedBorder)

            ForEach(options.indices, id: \.self) { index in
                HStack {
                    TextField("選択肢 \(index + 1)", text: $options[index])
                        .textFieldStyle(.roundedBorder)
                    if options.count > 2 {
                        Button(role: .destructive) {
                            options.remove(at: index)
                        } label: {
                            Image(systemName: "minus.circle.fill")
                        }
                    }
                }
            }

            if options.count < 6 {
                Button("選択肢を追加") {
                    options.append("")
                }
                .font(.subheadline)
            }

            Button("送信") {
                let poll = PollData(
                    question: question,
                    options: options.filter { !$0.isEmpty }
                )
                onSend(poll)
            }
            .buttonStyle(.borderedProminent)
            .disabled(question.isEmpty || options.filter { !$0.isEmpty }.count < 2)
        }
        .padding()
    }
}

struct PollData: Codable {
    let question: String
    let options: [String]
    var votes: [String: Int] = [:]
}

// 期待する出力:
// 質問入力欄 + 選択肢入力欄 + 送信ボタンが表示される
// 送信すると MSMessage としてグループチャットに投票バブルが送られる
```

### ステップ 5: MSMessage でデータをエンコードする

投票データをメッセージバブルとしてエンコードし、受信者がタップすると投票できるようにします。

```swift
// メッセージバブルの生成 — Rork Max が自動構築
func composePollMessage(poll: PollData, session: MSSession?) -> MSMessage {
    let message = MSMessage(session: session ?? MSSession())
    let layout = MSMessageTemplateLayout()

    // バブルに表示されるレイアウト
    layout.caption = "📊 \(poll.question)"
    layout.subcaption = "\(poll.options.count) つの選択肢"
    layout.trailingCaption = "タップして投票"
    layout.image = generatePollPreviewImage(poll: poll)

    message.layout = layout

    // URL にデータをエンコード（受信者側で復元）
    var components = URLComponents()
    components.queryItems = [
        URLQueryItem(name: "q", value: poll.question),
    ]
    for (i, option) in poll.options.enumerated() {
        components.queryItems?.append(
            URLQueryItem(name: "o\(i)", value: option)
        )
    }
    message.url = components.url

    return message

    // 期待する出力:
    // 「📊 今日のランチどこ行く？」のようなキャプション付きバブルが
    // 会話に表示され、タップすると投票画面が開く
}
```

## よくあるエラーと対処法

iMessage アプリ開発では、通常のアプリとは異なるエラーに遭遇することがあります。

### エラー 1: ステッカーが表示されない

**原因**: アセットカタログの構成ミス、または画像サイズが仕様外。

**対処法**: Rork Max のアセット管理画面で、画像が `Sticker Pack` フォルダ内に正しく配置されているか確認してください。サイズは 300×300 〜 618×618 px の範囲に収める必要があります。

### エラー 2: Extension が起動しない（`MSMessagesAppViewController` のクラッシュ）

**原因**: `Info.plist` の `NSExtension` 設定が不正。

**対処法**: Rork Max の「2クリック公開」を利用している場合、この設定は自動的に正しく構成されます。カスタムで編集した場合は、`NSExtensionPointIdentifier` が `com.apple.message-payload-provider` になっているか確認してください。

### エラー 3: メッセージ送信後にデータが消える

**原因**: `MSMessage` の URL エンコーディングが失敗している。

**対処法**: クエリパラメータに日本語を含む場合は、`addingPercentEncoding(withAllowedCharacters:)` で適切にエンコードされているか確認してください。Rork Max はデフォルトで UTF-8 エンコーディングを適用します。

## 応用・発展的な活用法

### ライブアクティビティとの連携

Rork Max の [Dynamic Island 機能](/articles/rork-dev/rork-max-dynamic-island) と組み合わせることで、iMessage で開始したアクティビティ（投票やゲーム）の進捗を Dynamic Island にリアルタイム表示できます。

### App Clips × iMessage

[App Clips](/articles/rork-dev/rork-max-app-clips-guide) を iMessage バブルに埋め込むと、受信者がアプリ未インストールでもインタラクティブな体験を提供できます。たとえば、レストランの割り勘アプリを App Clip として iMessage 内で共有するシナリオが考えられます。

### 収益化の可能性

iMessage ステッカーパックは App Store で有料販売（通常 120〜250 円）またはアプリ内課金で提供できます。[Rork Max の StoreKit 2 統合](/articles/rork-dev/rork-max-storekit-2-in-app-purchase)を使えば、追加ステッカーセットのアンロック機能も実装可能です。個人開発者にとって、制作コストが低くバイラル性の高いステッカーパックは優れた収益源になります。

## FAQ

### Q1: iMessage アプリだけ（本体アプリなし）で App Store に公開できますか？

はい、可能です。ステッカーパック単体や iMessage Extension 単体として App Store に提出できます。Rork Max の「2クリック公開」機能を使えば、クラウド Mac 上で自動ビルドされ、TestFlight 経由でテスト → App Store 提出まで完了します。詳しくは [2クリック公開ガイド](/articles/rork-basics/rork-max-2-click-publish-complete-guide) をご覧ください。

### Q2: アニメーションステッカーの作成方法は？

Rork Max では、APNG または GIF 形式のアニメーション画像をアップロードすることでアニメーションステッカーを作成できます。AI に「アニメーションステッカーを生成」と指示すると、フレーム生成からパッケージングまで自動で行われます。フレーム数は 10〜30 フレーム、ファイルサイズは 500KB 以下に収めてください。

### Q3: Rork（通常版）で iMessage アプリは作れますか？

iMessage Extension は SwiftUI ネイティブの機能であるため、React Native ベースの Rork 通常版では対応していません。Rork Max が必要です。Rork と Rork Max の違いについては [Rork vs Rork Max 比較記事](/articles/rork-basics/rork-vs-rork-max) を参照してください。

### Q4: グループチャットでのインタラクティブメッセージは全員に表示されますか？

はい、`MSMessage` として送信されたインタラクティブメッセージは、グループチャットの全参加者に表示されます。各参加者がバブルをタップすると、自分のデバイスで iMessage Extension が起動し、投票やゲームに参加できます。状態の同期には `MSSession` を使い、同じセッション内でメッセージを更新します。

## まとめ

Rork Max を使えば、iMessage アプリの開発は大幅に簡素化されます。ステッカーパックならアセットを用意してプロンプトを入力するだけ、インタラクティブなミニアプリも Messages Framework のボイラープレートを AI が自動生成するため、ロジック設計に集中できます。

iMessage アプリは、バイラル性が高く、ユーザー獲得コストの低いプロダクトです。とくにステッカーパックは制作コストも低いため、個人開発者が収益化の第一歩として取り組むのに最適なジャンルといえます。

Rork Max の 2クリック公開とクラウド Mac インフラを活用すれば、Xcode を開くことなく、アイデアから App Store 公開までをスピーディに完了できます。
