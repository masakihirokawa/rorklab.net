## はじめに

iOS のホーム画面や iPhone のロック画面に表示される「ウィジェット」は、アプリの情報を一目で確認できる強力な機能です。しかし、従来の開発では **WidgetKit**（Apple の公式フレームワーク）を Swift で記述する必要があり、React Native や Expo ベースのツールでは対応できませんでした。

**Rork Max** はネイティブ Swift コードを生成するため、WidgetKit を完全にサポートしています。本記事では、Rork Max を使ってウィジェットをゼロから作る方法を、実践的なプロンプト例と Swift コードの解説を交えながら紹介します。

---

## ウィジェットの種類と対応サイズ

Rork Max で作成できるウィジェットは大きく 2 種類あります。

### ホーム画面ウィジェット（SpringBoard Widget）

| サイズ | 説明 |
|--------|------|
| Small | 2×2 マス |
| Medium | 4×2 マス（横長） |
| Large | 4×4 マス |
| Extra Large | iPad 専用の大きなサイズ |

### ロック画面ウィジェット（iOS 16+）

| タイプ | 説明 |
|--------|------|
| Circular | 丸型（時計の横など） |
| Rectangular | 横長の帯状 |
| Inline | 時計の上に表示されるテキスト |

## Step 1: プロジェクトのセットアップ

まず Rork Max で新しい **iOS ネイティブプロジェクト** を作成します。既存プロジェクトへの追加も可能です。

### 推奨プロンプト

```
タスク管理アプリを作ってください。
ホーム画面に今日のタスク数を表示する Small ウィジェットと、
ロック画面に次のタスク名を表示する Circular ウィジェットを追加してください。
```

Rork Max は以下を自動生成します。

- メインアプリターゲット
- **Widget Extension** ターゲット（`TaskWidget` など）
- `AppIntent` による App と Widget 間のデータ共有コード
- `App Groups` の設定（データ共有のためのコンテナ）

## Step 2: WidgetKit の基本構造を理解する

Rork Max が生成するウィジェットコードは、次の 3 つのコンポーネントで構成されます。

### Provider（タイムライン提供者）

```swift
struct TaskWidgetProvider: TimelineProvider {
    func placeholder(in context: Context) -> TaskEntry {
        TaskEntry(date: Date(), taskCount: 3, nextTask: "朝のミーティング")
    }

    func getSnapshot(in context: Context, completion: @escaping (TaskEntry) -> Void) {
        let entry = TaskEntry(date: Date(), taskCount: 5, nextTask: "企画書を提出")
        completion(entry)
    }

    func getTimeline(in context: Context, completion: @escaping (Timeline<TaskEntry>) -> Void) {
        // App Groups 経由でリアルタイムデータを取得
        let sharedDefaults = UserDefaults(suiteName: "group.com.yourapp.tasks")
        let taskCount = sharedDefaults?.integer(forKey: "taskCount") ?? 0
        let nextTask = sharedDefaults?.string(forKey: "nextTask") ?? "タスクなし"

        let entry = TaskEntry(date: Date(), taskCount: taskCount, nextTask: nextTask)
        // 15 分後に再更新
        let nextUpdate = Calendar.current.date(byAdding: .minute, value: 15, to: Date())!
        let timeline = Timeline(entries: [entry], policy: .after(nextUpdate))
        completion(timeline)
    }
}
```

### Entry（データモデル）

```swift
struct TaskEntry: TimelineEntry {
    let date: Date
    let taskCount: Int
    let nextTask: String
}
```

### Widget View（表示 UI）

```swift
struct TaskWidgetEntryView: View {
    var entry: TaskWidgetProvider.Entry
    @Environment(\.widgetFamily) var family

    var body: some View {
        switch family {
        case .systemSmall:
            SmallWidgetView(entry: entry)
        case .accessoryCircular:
            CircularLockScreenView(entry: entry)
        case .accessoryRectangular:
            RectangularLockScreenView(entry: entry)
        default:
            SmallWidgetView(entry: entry)
        }
    }
}

struct SmallWidgetView: View {
    let entry: TaskEntry

    var body: some View {
        VStack(spacing: 8) {
            Image(systemName: "checkmark.circle.fill")
                .font(.title)
                .foregroundStyle(.blue)
            Text("\(entry.taskCount)")
                .font(.largeTitle.bold())
            Text("今日のタスク")
                .font(.caption)
                .foregroundStyle(.secondary)
        }
        .containerBackground(.fill.tertiary, for: .widget)
    }
}
```

## Step 3: アプリとウィジェット間のデータ共有

ウィジェットはアプリの **メモリ空間とは独立して動作**するため、データ共有には **App Groups** を使います。Rork Max はこの設定も自動化してくれます。

### アプリ側（メインターゲット）でデータを書き込む

```swift
// TaskViewModel.swift（Rork Max が自動生成）
class TaskViewModel: ObservableObject {
    private let sharedDefaults = UserDefaults(suiteName: "group.com.yourapp.tasks")

    func updateWidget() {
        let pendingCount = tasks.filter { !$0.isCompleted }.count
        let nextTask = tasks.first(where: { !$0.isCompleted })?.title ?? "タスクなし"

        sharedDefaults?.set(pendingCount, forKey: "taskCount")
        sharedDefaults?.set(nextTask, forKey: "nextTask")

        // WidgetKit に更新を通知
        WidgetCenter.shared.reloadAllTimelines()
    }
}
```

### プロンプト例（データ同期の追加指示）

```
タスクを完了にしたときに自動でウィジェットのカウントを更新するようにしてください。
```

Rork Max はタスク完了アクション内に `WidgetCenter.shared.reloadAllTimelines()` を自動で挿入します。

## Step 4: ロック画面ウィジェットのカスタマイズ

ロック画面ウィジェットは表示スペースが限られているため、シンプルな情報表示に徹することが重要です。

```swift
struct CircularLockScreenView: View {
    let entry: TaskEntry

    var body: some View {
        ZStack {
            AccessoryWidgetBackground()
            VStack(spacing: 2) {
                Image(systemName: "list.bullet")
                    .font(.caption2)
                Text("\(entry.taskCount)")
                    .font(.headline.bold())
            }
        }
    }
}

struct RectangularLockScreenView: View {
    let entry: TaskEntry

    var body: some View {
        HStack {
            Image(systemName: "arrow.right.circle.fill")
            VStack(alignment: .leading) {
                Text("次のタスク")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
                Text(entry.nextTask)
                    .font(.caption.bold())
                    .lineLimit(1)
            }
        }
    }
}
```

## Step 5: ウィジェットの設定（Intent）

iOS 17 以降では **AppIntent** を使ってウィジェットをカスタマイズできます。例えば「特定のプロジェクトだけ表示する」などの設定が可能です。

### プロンプト例

```
ウィジェットの設定で「表示するプロジェクト」を選べるようにしてください。
```

Rork Max が生成するコード：

```swift
struct ConfigurationAppIntent: WidgetConfigurationIntent {
    static var title: LocalizedStringResource = "タスクウィジェット設定"
    static var description = IntentDescription("表示するプロジェクトを選択します。")

    @Parameter(title: "プロジェクト")
    var project: ProjectEntity?
}
```

## Tips: より良いウィジェットを作るプロンプト術

Rork Max でウィジェットを改善するときは、**具体的な見た目と動作** を指定することがポイントです。

| 目的 | 効果的なプロンプト |
|------|------------------|
| デザイン調整 | 「ウィジェットの背景をグラデーション（青→紫）にして」 |
| データ更新頻度 | 「ウィジェットを 30 分ごとに更新して」 |
| ダークモード対応 | 「ダークモードでも見やすいカラーパレットにして」 |
| インタラクション | 「ウィジェットをタップしたらタスク追加画面を開いて」 |
| 複数サイズ対応 | 「Medium サイズでは 3 件のタスクをリスト表示して」 |

## よくあるトラブルとその解決策

### ウィジェットにデータが反映されない

App Groups の Bundle ID が一致しているか確認しましょう。Rork Max のプロンプト：

```
App Groups の設定が正しいか確認して、データが反映されない問題を修正してください。
```

### ウィジェットプレビューが表示されない

Xcode のウィジェットプレビューには `placeholder()` が使われます。ダミーデータを適切に設定してください。

### 更新が遅い

WidgetKit の更新頻度はシステムにより制限されます。重要な更新は `WidgetCenter.shared.reloadAllTimelines()` を明示的に呼び出してください。
