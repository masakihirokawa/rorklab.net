Figma と Rork アプリ開発を効率的に連携させることは、スケーラブルなモバイル UI の構築に不可欠です。しかし、デザイン側とエンジニア側の仕様がズレると、無駄な修正ループが生じてしまいます。

本記事では、**Figma から React Native への完全なワークフロー**を体系的に解説します。UI Stack による 5 状態管理から、AutoLayout の変換ルール、デザイントークンを活用したテーマ管理、そして実装の効率化まで、実践的なパターンをカバーしています。

## Rork アプリ開発における Figma の役割とワークフロー全体像

Figma は単なるデザインツールではなく、**デザイナーとエンジニアの共通言語**です。正しく運用することで、ハンドオフ品質が飛躍的に向上し、修正ループを最小化できます。

Rork アプリ開発の典型的なワークフローは以下の通りです：

1. デザイナーが Figma でコンポーネントライブラリを構築（Atomic Design 準拠）
2. Figma に UI Stack（5 状態）を明示的に定義
3. Figma Variables でデザイントークン（色、サイズ、間隔）を一元管理
4. エンジニアがこの情報を基に React Native コンポーネントを実装
5. Dev Mode + MCP で自動コード生成の検討
6. ダークモード、多数言語対応を Context API + Hooks で実装

このプロセスにより、デザイン変更がコードに迅速に反映され、品質が一貫性を保つようになります。

## UI Stack をモバイルアプリに適用する方法

UI Stack とは、UI コンポーネントが 5 つの状態を持つという概念です。これを Rork アプリに組み込むことで、エラーハンドリングと UX が飛躍的に改善します。

### 5 つの状態定義

- **Ideal**: 正常系。期待通りのデータが表示されている状態
- **Partial**: データが部分的に読み込まれている、またはスケルトン状態
- **Empty**: データが存在しない場合（検索結果なし、リストが空など）
- **Error**: エラーが発生した状態
- **Loading**: 読み込み中の状態

### Figma での UI Stack の設計

Figma では、1 つのコンポーネントに対して、これら 5 つの状態を **variant** として用意します。

```
ProductCard (Component)
├── Ideal
├── Partial (スケルトン表示)
├── Empty
├── Error
└── Loading
```

### React Native での実装例

```javascript
import React, { useState, useEffect } from 'react';
import { View, Text, ActivityIndicator, StyleSheet } from 'react-native';

export const ProductCard = ({
  productId,
  onRetry
}) => {
  const [state, setState] = useState('loading'); // loading | ideal | error | empty
  const [product, setProduct] = useState(null);

  useEffect(() => {
    fetchProduct(productId)
      .then((data) => {
        if (data) {
          setProduct(data);
          setState('ideal');
        } else {
          setState('empty');
        }
      })
      .catch(() => setState('error'));
  }, [productId]);

  const renderContent = () => {
    switch (state) {
      case 'loading':
        return <SkeletonLoader />;
      case 'ideal':
        return (
          <View>
            <Text style={styles.title}>{product.name}</Text>
            <Text style={styles.price}>¥{product.price}</Text>
          </View>
        );
      case 'empty':
        return <Text style={styles.emptyText}>商品が見つかりません</Text>;
      case 'error':
        return (
          <View>
            <Text style={styles.errorText}>エラーが発生しました</Text>
            <TouchableOpacity onPress={onRetry}>
              <Text style={styles.retryButton}>再試行</Text>
            </TouchableOpacity>
          </View>
        );
      default:
        return null;
    }
  };

  return <View style={styles.card}>{renderContent()}</View>;
};

const styles = StyleSheet.create({
  card: { padding: 16, borderRadius: 8, backgroundColor: '#fff' },
  title: { fontSize: 16, fontWeight: 'bold' },
  price: { fontSize: 14, color: '#666' },
  emptyText: { textAlign: 'center', color: '#999' },
  errorText: { color: '#d32f2f', marginBottom: 8 },
  retryButton: { color: '#1976d2', fontWeight: '600' },
});
```

### エラーバウンダリとリトライ UI

エラーバウンダリを使い、予期しないエラーをキャッチして、ユーザーフレンドリーなエラーメッセージを表示します。

```javascript
export class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  render() {
    if (this.state.hasError) {
      return (
        <View style={styles.errorContainer}>
          <Text style={styles.errorTitle}>問題が発生しました</Text>
          <Text style={styles.errorMessage}>{this.state.error?.message}</Text>
          <TouchableOpacity
            onPress={() => this.setState({ hasError: false })}
          >
            <Text style={styles.retryButton}>もう一度試す</Text>
          </TouchableOpacity>
        </View>
      );
    }
    return this.props.children;
  }
}
```

## Figma AutoLayout → React Native Flexbox 変換ルール

Figma の AutoLayout は、Web の Flexbox の概念に非常に似ています。正しく対応させることが、実装効率の鍵です。

### 対応関係表

| Figma AutoLayout | React Native Flexbox |
|---|---|
| Direction: Horizontal | `flexDirection: 'row'` |
| Direction: Vertical | `flexDirection: 'column'` |
| Gap | `gap` (RN 0.71+) |
| Padding | `padding` / `paddingHorizontal` / `paddingVertical` |
| Align: Left | `alignItems: 'flex-start'` |
| Align: Center | `alignItems: 'center'` |
| Align: Right | `alignItems: 'flex-end'` |
| Justify: Left | `justifyContent: 'flex-start'` |
| Justify: Center | `justifyContent: 'center'` |
| Justify: Space Between | `justifyContent: 'space-between'` |

### 具体的な変換コード例

**Figma 設定**:
- Direction: Horizontal
- Gap: 8
- Padding: 16（全方向）
- Align: Center

**React Native コード**:

```javascript
const styles = StyleSheet.create({
  container: {
    flexDirection: 'row',
    gap: 8,
    padding: 16,
    alignItems: 'center',
  },
  item: {
    flex: 1,
  },
});

export const HorizontalList = ({ items }) => (
  <View style={styles.container}>
    {items.map((item) => (
      <View key={item.id} style={styles.item}>
        <Text>{item.label}</Text>
      </View>
    ))}
  </View>
);
```

## Figma Variables → React Native テーマ定数の変換

デザインシステムの一貫性を保つには、デザイントークン（色、サイズ、間隔）を一元管理することが必須です。

### デザイントークンの管理方法

Figma Variables は 3 層から構成されます：

1. **Primitive（プリミティブ）**: 最基本の値。例：`#1976d2`
2. **Semantic（セマンティック）**: 意味付きの値。例：Primary Color
3. **Alias（エイリアス）**: 他のトークンへの参照

```
Primitive Layer:
  colors/blue-500: #1976d2
  colors/gray-50: #f5f5f5

Semantic Layer:
  colors/primary: colors/blue-500
  colors/background: colors/gray-50

Alias Layer:
  button/primary-bg: colors/primary
  button/disabled-bg: colors/gray-50
```

### ダークモード対応の実装

React Native の `useColorScheme` Hook を使い、システム設定を基にテーマを自動選択します。

```javascript
import {
  useColorScheme,
  StyleSheet,
} from 'react-native';
import { LIGHT_THEME, DARK_THEME } from './theme';

const ThemeContext = React.createContext();

export const ThemeProvider = ({ children }) => {
  const colorScheme = useColorScheme();
  const theme = colorScheme === 'dark' ? DARK_THEME : LIGHT_THEME;

  return (
    <ThemeContext.Provider value={theme}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => React.useContext(ThemeContext);

// コンポーネント内での使用
export const Button = ({ title, onPress }) => {
  const theme = useTheme();
  const styles = StyleSheet.create({
    button: {
      backgroundColor: theme.colors.primary,
      padding: theme.spacing.md,
      borderRadius: 4,
    },
    text: { color: '#fff', fontWeight: 'bold' },
  });

  return (
    <TouchableOpacity style={styles.button} onPress={onPress}>
      <Text style={styles.text}>{title}</Text>
    </TouchableOpacity>
  );
};
```

## Figma Dev Mode MCP を使った Rork コンポーネント生成

Figma の Dev Mode と MCP（Model Context Protocol）を組み合わせることで、デザイン仕様から自動的に React Native コンポーネントスケルトンを生成できます。

### MCP 設定方法

1. Figma Dev Mode を有効化
2. MCP ゲートウェイを設定（Figma Plugin）
3. デザインファイルをスキャン
4. 自動コード生成を実行

```bash
figma-mcp --file-key YOUR_FILE_KEY --export-format react-native
```

生成されたコードを基に、ビジネスロジックを追加して完成させます。

### 商品一覧画面の UI Stack 実装

```javascript
export const ProductListScreen = () => {
  const [state, setState] = useState('loading'); // loading | ideal | error | empty
  const [products, setProducts] = useState([]);
  const theme = useTheme();

  useEffect(() => {
    fetchProducts()
      .then((data) => {
        if (data.length === 0) {
          setState('empty');
        } else {
          setProducts(data);
          setState('ideal');
        }
      })
      .catch(() => setState('error'));
  }, []);

  const renderContent = () => {
    switch (state) {
      case 'loading':
        return <ProductListSkeleton />;
      case 'ideal':
        return (
          <FlatList
            data={products}
            keyExtractor={(item) => item.id}
            renderItem={({ item }) => <ProductCard product={item} />}
            contentContainerStyle={styles.listContent}
          />
        );
      case 'empty':
        return (
          <View style={styles.emptyContainer}>
            <Text style={{ color: theme.colors.text.secondary }}>
              商品が見つかりません
            </Text>
          </View>
        );
      case 'error':
        return (
          <View style={styles.errorContainer}>
            <Text style={{ color: theme.colors.status.error }}>
              読み込みエラー
            </Text>
          </View>
        );
    }
  };

  return <View style={styles.container}>{renderContent()}</View>;
};

const styles = StyleSheet.create({
  container: { flex: 1 },
  listContent: { padding: 16 },
  emptyContainer: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  errorContainer: { flex: 1, justifyContent: 'center', alignItems: 'center' },
});
```

## よくある質問（FAQ）

**Q1: UI Stack を導入すると、Figma ファイルが複雑になりませんか？**

A: 最初は複雑に感じるかもしれませんが、Figma の Component Set 機能を使えば、管理が容易になります。長期的には、品質と開発速度の向上によるメリットが大きいです。

**Q2: ダークモード対応は必須ですか？**

A: iOS 13 以降、Android 10 以降ではシステムレベルでダークモード設定が標準化されています。ユーザー体験向上のため、対応することを強く推奨します。

**Q3: Figma Variables で 100 以上のトークンがある場合、どう管理すればいいですか？**

A: トークンをカテゴリ（Colors, Spacing, Typography など）ごとにグループ化し、命名規則を統一します。定期的な監査と最適化も重要です。

**Q4: Dev Mode MCP で生成されたコードをそのまま使用できますか？**

A: 生成されたコードはスケルトン（基本構造）です。ビジネスロジック、状態管理、エラーハンドリングは手作業で追加する必要があります。

**Q5: デザイン変更の頻度が高い場合、ワークフローをどう対応させればいいですか?**

A: Figma を Source of Truth に位置付け、デザイン変更時には即座にアノテーションで実装上の影響を記述することが重要です。Sprint ごとに同期セッションを設けるのも効果的です。
