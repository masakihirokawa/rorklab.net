## はじめに：なぜApp Store Connect APIが個人開発者に必須なのか

App Storeにアプリをリリースしたあと、多くの開発者が抱える課題があります。「レビューを毎日手動で確認する作業が面倒」「売上データをスプレッドシートに手入力している」「ASO改善のためのデータ収集に時間をとられる」——こうした運用コストは、個人開発者にとって開発時間を圧迫する大きな問題です。

App Store Connect APIは、AppleがApp Store Connect上のあらゆるデータへのプログラムアクセスを提供するRESTful APIです。このAPIを使えば、アプリのレビュー一覧取得・売上トレンド分析・パフォーマンス指標の自動収集・さらにはTestFlightの管理まで、すべてをコードで自動化できます。

## App Store Connect APIの全体像

App Store Connect APIは、大きく4つの主要領域をカバーしています。

**アプリ管理API:** アプリのメタデータ（説明文・スクリーンショット・価格）、バージョン情報、レビューへの返信などを管理します。

**Sales and Trends API:** アプリの売上・ダウンロード数・サブスクリプション収益などのトレンドデータを取得します。AppフィギュアやData Boxのようなサードパーティツールも、このAPIを経由してデータを取得しています。

**Finance API:** 請求書・収益精算レポートなど、会計・税務に関連する財務データへアクセスします。

**TestFlight API:** ベータテスターの管理、ビルドの配布、フィードバック収集を自動化します。

## Step 1: JWT認証の実装

App Store Connect APIはすべてのリクエストでJWT（JSON Web Token）認証を要求します。これがこのAPIの最初の壁ですが、一度実装してしまえば全エンドポイントで共通利用できます。

### APIキーの取得

まず[App Store Connect](https://appstoreconnect.apple.com)にログインし、「ユーザーとアクセス」→「統合」→「App Store Connect API」に移動します。「キーを生成」をクリックし、適切な権限（Finance/Sales/App管理など）を付与してください。ダウンロードした `.p8` ファイルと、`Issuer ID`・`Key ID` を安全に保管します。

**⚠️ `.p8` ファイルは一度しかダウンロードできません。必ず安全な場所に保存してください。**

### JWTトークンの生成（Supabase Edge Functions）

```typescript
// supabase/functions/appstore-auth/jwt.ts
import { create, getNumericDate } from "https://deno.land/x/djwt@v3.0.2/mod.ts";
import { crypto } from "https://deno.land/std@0.224.0/crypto/mod.ts";

const ISSUER_ID = Deno.env.get("ASC_ISSUER_ID")!;
const KEY_ID = Deno.env.get("ASC_KEY_ID")!;
const PRIVATE_KEY_PEM = Deno.env.get("ASC_PRIVATE_KEY")!; // .p8の内容

/**
 * App Store Connect API用JWTトークンを生成する
 * 有効期限は最大20分（Appleの制限）
 */
export async function generateAscToken(): Promise<string> {
  // PEM形式の秘密鍵をインポート
  const pemBody = PRIVATE_KEY_PEM
    .replace("-----BEGIN PRIVATE KEY-----", "")
    .replace("-----END PRIVATE KEY-----", "")
    .replace(/\s/g, "");

  const binaryKey = Uint8Array.from(atob(pemBody), c => c.charCodeAt(0));

  const privateKey = await crypto.subtle.importKey(
    "pkcs8",
    binaryKey,
    { name: "ECDSA", namedCurve: "P-256" },
    false,
    ["sign"]
  );

  const now = Math.floor(Date.now() / 1000);

  const payload = {
    iss: ISSUER_ID,
    iat: now,
    exp: now + 1200, // 20分（最大値）
    aud: "appstoreconnect-v1",
  };

  const header = {
    alg: "ES256" as const,
    kid: KEY_ID,
    typ: "JWT",
  };

  return await create(header, payload, privateKey);
}

// 使用例
// const token = await generateAscToken();
// fetch("https://api.appstoreconnect.apple.com/v1/apps", {
//   headers: { Authorization: `Bearer ${token}` }
// });
// 期待する出力: 200 OK + アプリ一覧JSON
```

## Step 2: Reviews APIでレビューを自動収集・感情分析する

アプリのレビューを定期的に取得し、AIで感情分析を行うシステムは、App Storeマーケティングの質を大幅に向上させます。

### レビュー一覧の取得とAI分析

```typescript
// supabase/functions/fetch-reviews/index.ts
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";
import { generateAscToken } from "../appstore-auth/jwt.ts";

const APP_ID = Deno.env.get("APP_ID")!; // App Store Connect上のアプリID
const supabase = createClient(
  Deno.env.get("SUPABASE_URL")!,
  Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
);

interface Review {
  id: string;
  rating: number;
  title: string;
  body: string;
  reviewerNickname: string;
  createdDate: string;
  territory: string;
}

/**
 * 指定アプリの全レビューを取得（ページネーション対応）
 */
async function fetchAllReviews(token: string): Promise<Review[]> {
  const reviews: Review[] = [];
  let cursor: string | null = null;

  do {
    const url = new URL(
      `https://api.appstoreconnect.apple.com/v1/apps/${APP_ID}/customerReviews`
    );
    url.searchParams.set("limit", "200");
    url.searchParams.set("sort", "-createdDate"); // 新しい順
    if (cursor) url.searchParams.set("cursor", cursor);

    const response = await fetch(url.toString(), {
      headers: { Authorization: `Bearer ${token}` }
    });

    if (!response.ok) throw new Error(`Reviews API error: ${response.status}`);

    const data = await response.json();

    for (const item of data.data) {
      reviews.push({
        id: item.id,
        rating: item.attributes.rating,
        title: item.attributes.title ?? "",
        body: item.attributes.body,
        reviewerNickname: item.attributes.reviewerNickname,
        createdDate: item.attributes.createdDate,
        territory: item.attributes.territory,
      });
    }

    cursor = data.links?.next
      ? new URL(data.links.next).searchParams.get("cursor")
      : null;

  } while (cursor);

  return reviews;
}

/**
 * Claudeを使ったレビュー感情分析 + 返信ドラフト生成
 */
async function analyzeAndDraftReply(review: Review): Promise<{
  sentiment: "positive" | "neutral" | "negative";
  category: string;
  replyDraft: string;
}> {
  const ANTHROPIC_API_KEY = Deno.env.get("ANTHROPIC_API_KEY")!;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": ANTHROPIC_API_KEY,
      "anthropic-version": "2023-06-01",
    },
    body: JSON.stringify({
      model: "claude-haiku-4-5-20251001",
      max_tokens: 500,
      messages: [{
        role: "user",
        content: `以下のApp Storeレビューを分析して、JSONで回答してください。

レビュー（星: ${review.rating}/5）:
タイトル: ${review.title}
本文: ${review.body}

回答形式:
{
  "sentiment": "positive|neutral|negative",
  "category": "UI/UX|パフォーマンス|バグ報告|機能要望|価格|その他",
  "replyDraft": "100文字以内の丁寧な返信（日本語）"
}

replyDraftは開発者として誠実さと感謝を込めて書くこと。`
      }],
    }),
  });

  const data = await response.json();
  const text = data.content[0].text;
  const jsonMatch = text.match(/\{[\s\S]*\}/);
  return JSON.parse(jsonMatch![0]);
}

// メインハンドラー
Deno.serve(async (_req) => {
  const token = await generateAscToken();
  const reviews = await fetchAllReviews(token);

  // 直近24時間のレビューのみ処理
  const oneDayAgo = new Date(Date.now() - 86400000).toISOString();
  const newReviews = reviews.filter(r => r.createdDate > oneDayAgo);

  for (const review of newReviews) {
    const analysis = await analyzeAndDraftReply(review);

    await supabase.from("app_reviews").upsert({
      review_id: review.id,
      rating: review.rating,
      title: review.title,
      body: review.body,
      nickname: review.reviewerNickname,
      created_at: review.createdDate,
      territory: review.territory,
      sentiment: analysis.sentiment,
      category: analysis.category,
      reply_draft: analysis.replyDraft,
    });
  }

  return new Response(
    JSON.stringify({ processed: newReviews.length }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

## Step 3: Sales and Trends APIで売上を自動集計する

Sales and Trends APIは、App Storeの売上データを `.tsv.gz` 形式（TSVのgzip圧縮）で返します。このフォーマットに対応した解凍・パース処理が必要です。

```typescript
// supabase/functions/fetch-sales/index.ts
import { generateAscToken } from "../appstore-auth/jwt.ts";

const VENDOR_NUMBER = Deno.env.get("ASC_VENDOR_NUMBER")!;
// App Store Connectの「収益」→「支払いと財務レポート」で確認できる

async function fetchDailySales(token: string, date: string): Promise<unknown[]> {
  // dateはYYYY-MM-DD形式
  const url = new URL("https://api.appstoreconnect.apple.com/v1/salesReports");
  url.searchParams.set("filter[frequency]", "DAILY");
  url.searchParams.set("filter[reportType]", "SALES");
  url.searchParams.set("filter[reportSubType]", "SUMMARY");
  url.searchParams.set("filter[vendorNumber]", VENDOR_NUMBER);
  url.searchParams.set("filter[reportDate]", date);

  const response = await fetch(url.toString(), {
    headers: { Authorization: `Bearer ${token}` }
  });

  if (response.status === 404) {
    console.log(`No sales data for ${date} yet`);
    return [];
  }

  if (!response.ok) throw new Error(`Sales API error: ${response.status}`);

  // gzip解凍
  const compressed = await response.arrayBuffer();
  const ds = new DecompressionStream("gzip");
  const writer = ds.writable.getWriter();
  writer.write(compressed);
  writer.close();

  const reader = ds.readable.getReader();
  const chunks: Uint8Array[] = [];
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    chunks.push(value);
  }

  const allBytes = new Uint8Array(chunks.reduce((acc, c) => acc + c.length, 0));
  let offset = 0;
  for (const c of chunks) { allBytes.set(c, offset); offset += c.length; }
  const text = new TextDecoder().decode(allBytes);

  // TSVをパース（1行目はヘッダー）
  const lines = text.trim().split("\n");
  const headers = lines[0].split("\t");

  return lines.slice(1).map(line => {
    const values = line.split("\t");
    const record: Record<string, string> = {};
    headers.forEach((h, i) => { record[h.trim()] = values[i] || ""; });
    return record;
  });
}
```

## Step 5: 異常検知アラートと自動化パイプラインの完成形

```typescript
// supabase/functions/alert-negative-reviews/index.ts
const ALERT_THRESHOLD = 3; // 1時間以内に星1〜2が3件以上で緊急アラート

async function checkNegativeSpike() {
  const oneHourAgo = new Date(Date.now() - 3600000).toISOString();

  const { data: negativeReviews } = await supabase
    .from("app_reviews")
    .select("*")
    .lte("rating", 2)
    .gte("created_at", oneHourAgo);

  if ((negativeReviews?.length ?? 0) >= ALERT_THRESHOLD) {
    const SLACK_WEBHOOK = Deno.env.get("SLACK_WEBHOOK_URL")!;
    const message = [
      `🚨 緊急: 過去1時間で低評価レビューが${negativeReviews!.length}件急増！`,
      ...negativeReviews!.map(r => `⭐${r.rating} "${r.title}": ${r.body.slice(0, 50)}...`)
    ].join("\n");

    await fetch(SLACK_WEBHOOK, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ text: message }),
    });
    // 期待する出力: Slackに緊急アラートが投稿される
  }
}
```

**推奨スケジュール（Supabase Scheduled Functions）:**

- 毎日09:00 JST: 前日の売上データ取得・DB保存 → Slackに日次レポート送信
- 毎時: 新着レビュー取得・感情分析・DB保存 → 星1〜2のレビュー検知時に即時通知
- 毎週月曜09:00 JST: 週次ダウンロード数・収益・レビュートレンドをレポート

## Step 6: TestFlight管理の自動化

同じJWT認証を使い、TestFlightのベータテスターとビルドを管理する自動化も実装できます。

```typescript
// ベータテスターを自動招待するEdge Function
async function inviteBetaTester(
  token: string,
  appId: string,
  email: string,
  firstName: string,
  lastName: string
) {
  const response = await fetch(
    "https://api.appstoreconnect.apple.com/v1/betaTesters",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${token}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        data: {
          type: "betaTesters",
          attributes: { email, firstName, lastName },
          relationships: {
            apps: { data: [{ type: "apps", id: appId }] }
          }
        }
      })
    }
  );

  if (response.ok) {
    console.log(`✅ ${email} をTestFlightに招待しました`);
    // 期待する出力: ✅ user@example.com をTestFlightに招待しました
  } else {
    console.error(`❌ 招待失敗: ${response.status}`);
  }
}
```

## よくあるエラーと対処法

### 「401 Unauthorized」エラーが返る

JWTの有効期限切れが最も多い原因です。トークンの有効期限は最大20分（1200秒）ですが、リクエストのたびに新しいトークンを生成するか、キャッシュして15分ごとに更新する実装を推奨します。また、`.p8` ファイルの内容をSupabase Secretsに保存する際、改行コードが失われることがあります。`PRIVATE_KEY` を保存するときは、改行を `\n` で明示的にエスケープしてください。

### 「404 Not Found」（Sales API）が返る

指定した日付のデータがまだ生成されていない場合に404が返ります。Appleの売上レポートは通常、前日分が翌日の午前5〜8時（PST）に確定するため、JST換算で午後9時〜翌日正午頃以降にリクエストするようにしてください。

### レート制限に引っかかる

App Store Connect APIには1分あたりのリクエスト数制限があります（ドキュメント非公開ですが、実運用では分あたり100リクエスト以内に抑えることを推奨）。ページネーションを使う場合は、リクエスト間に100〜200msの待機を入れると安全です。

## FAQ

### Q. App Store Connect APIを使うのにApple Developer Program（年間¥12,800）への加入は必要ですか？

はい、必要です。APIキーはApple Developer Programのメンバーシップに紐づいており、アプリのプロビジョニングやApp Store公開と同じ前提条件になります。

### Q. 取得できるレビューの期間に上限はありますか？

Reviews APIは原則として全レビュー（リリース初日からの全履歴）を取得できます。ただし非常に大量のレビューがある場合はページネーションに時間がかかります。定期実行で差分のみ取得する設計（`sort=-createdDate` + 最終取得日時のDB管理）を強く推奨します。

### Q. レビューへの返信もAPIで自動化できますか？

はい、可能です。`POST /v1/customerReviewResponses` エンドポイントを使えばAPIから返信を投稿できます。ただし、Appleの審査ガイドラインに基づき、不適切なコンテンツ・スパム返信は検知・削除される場合があります。AIで生成した返信ドラフトを人間がレビューしてから投稿するワークフローの採用を推奨します。

### Q. Supabase以外のバックエンドでも同じ実装はできますか？

はい。JWT認証のロジックはNode.js・Python・Go・Rubyなど任意のサーバーサイド環境で実装できます。本記事のコードはDeno（Supabase Edge Functions）向けですが、概念はどのランタイムでも共通です。Node.js環境であれば `jsonwebtoken` + `node-forge` の組み合わせが定番です。
