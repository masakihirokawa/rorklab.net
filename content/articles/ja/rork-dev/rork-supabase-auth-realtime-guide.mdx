Rorkでアプリを開発するとき、バックエンドの構築は大きな課題の一つです。Supabaseを活用すれば、認証・データベース・リアルタイム通信といったバックエンド機能を、コードをほとんど書かずに実装できます。この記事では、RorkとSupabaseの連携方法を基礎から実践まで丁寧に解説します。

## なぜ Rork × Supabase なのか

RorkはAIを活用したアプリ開発プラットフォームで、UIの構築やロジックの生成を素早く行えます。一方、Supabaseは「Firebase の代替」として急成長しているオープンソースのBaaS（Backend as a Service）で、PostgreSQL データベース、認証、リアルタイムサブスクリプション、ストレージを提供します。

この2つを組み合わせることで、フロントエンドはRorkがAIで生成し、バックエンドはSupabaseが担当するという、効率的な開発体制が整います。

## Supabase プロジェクトのセットアップ

### アカウント作成とプロジェクト初期化

まず、Supabaseの公式サイトでプロジェクトを作成します。

```bash
# Supabase CLIを使ったプロジェクト初期化
npx supabase init

# ローカル開発環境の起動
npx supabase start

# 出力例:

```

### データベーステーブルの作成

ユーザープロフィールとメッセージを管理するテーブルを作成します。

```sql
-- ユーザープロフィールテーブル
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  avatar_url TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- メッセージテーブル（リアルタイムチャット用）
CREATE TABLE messages (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES profiles(id) NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- RLS（Row Level Security）の有効化
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;

-- ポリシー: 誰でもプロフィールを閲覧可能
CREATE POLICY "Profiles are viewable by everyone"
  ON profiles FOR SELECT USING (true);

-- ポリシー: 自分のプロフィールのみ更新可能
CREATE POLICY "Users can update own profile"
  ON profiles FOR UPDATE USING (auth.uid() = id);

-- ポリシー: 認証済みユーザーはメッセージを投稿可能
CREATE POLICY "Authenticated users can insert messages"
  ON messages FOR INSERT
  WITH CHECK (auth.uid() = user_id);

-- ポリシー: 誰でもメッセージを閲覧可能
CREATE POLICY "Messages are viewable by everyone"
  ON messages FOR SELECT USING (true);
```

## Rork でのSupabase連携実装

### Supabase クライアントの初期化

Rorkのプロジェクトに Supabase クライアントを設定します。

```typescript
// lib/supabase.ts
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = 'https://your-project.supabase.co';
const supabaseAnonKey = 'your-anon-key';

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

### 認証機能の実装

メールアドレスとパスワードによるユーザー認証を実装します。

```typescript
// 新規登録
async function signUp(email: string, password: string, username: string) {
  // 1. Supabase Authでユーザーを作成
  const { data: authData, error: authError } = await supabase.auth.signUp({
    email,
    password,
  });

  if (authError) throw authError;

  // 2. プロフィールテーブルにレコードを追加
  const { error: profileError } = await supabase
    .from('profiles')
    .insert({
      id: authData.user?.id,
      username,
    });

  if (profileError) throw profileError;

  return authData;
}

// ログイン
async function signIn(email: string, password: string) {
  const { data, error } = await supabase.auth.signInWithPassword({
    email,
    password,
  });

  if (error) throw error;
  return data;
}

// ログアウト
async function signOut() {
  const { error } = await supabase.auth.signOut();
  if (error) throw error;
}

// 使用例:
// await signUp('user@example.com', 'password123', 'testuser');
// → ユーザー作成 + プロフィール作成が完了
```

## リアルタイム機能の実装

Supabaseのリアルタイム機能を使って、メッセージが即座に全ユーザーに反映されるチャット機能を構築します。

```typescript
// リアルタイムメッセージ購読
function subscribeToMessages(onNewMessage: (message: any) => void) {
  const channel = supabase
    .channel('messages')
    .on(
      'postgres_changes',
      {
        event: 'INSERT',
        schema: 'public',
        table: 'messages',
      },
      (payload) => {
        onNewMessage(payload.new);
      }
    )
    .subscribe();

  // クリーンアップ関数を返す
  return () => {
    supabase.removeChannel(channel);
  };
}

// メッセージ送信
async function sendMessage(content: string) {
  const user = (await supabase.auth.getUser()).data.user;
  if (!user) throw new Error('Not authenticated');

  const { error } = await supabase
    .from('messages')
    .insert({
      user_id: user.id,
      content,
    });

  if (error) throw error;
}

// 使用例:
// const unsubscribe = subscribeToMessages((msg) => {
//   console.log('New message:', msg.content);
// });
// await sendMessage('Hello from Rork!');
// → 全購読者にリアルタイムで配信される
```

## よくある質問（FAQ）

### Q: Supabaseの無料プランでどこまで使えますか？

Supabaseの無料プランでは、500MBのデータベースストレージ、1GBのファイルストレージ、50,000の月間アクティブユーザー（認証）、リアルタイム接続200同時が利用可能です。個人開発やプロトタイプ段階では十分な容量です。ユーザー数が増えてきたら、月額25ドルのProプランへのアップグレードを検討してください。

### Q: Rorkのアプリからサーバーレスファンクションは使えますか？

はい、SupabaseのEdge Functions（Deno ベース）を利用できます。決済処理やメール送信など、クライアント側で実行すべきでないロジックをEdge Functionsに配置し、RorkアプリからHTTPリクエストで呼び出すことが可能です。

### Q: セキュリティ面で気を付けることは？

最も重要なのはRLS（Row Level Security）の設定です。Supabaseの `anon key` はクライアント側に公開されるため、RLSポリシーが適切に設定されていないと、誰でもデータを読み書きできてしまいます。必ずテーブルごとにRLSを有効化し、適切なポリシーを定義してください。

## まとめ

RorkとSupabaseを組み合わせることで、フロントエンドのUI生成からバックエンドの認証・データベース・リアルタイム通信まで、フルスタックなアプリ開発を効率的に行うことができます。特にRLS（Row Level Security）の設定を忘れずに行うことが、安全なアプリ構築の鍵です。

まずはシンプルな認証機能付きのアプリから始めて、リアルタイム機能やEdge Functionsを段階的に追加していくことをお勧めします。
