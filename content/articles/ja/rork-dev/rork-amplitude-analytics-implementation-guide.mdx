## Amplitude Analyticsとは？Rorkアプリに導入すべき理由

アプリをリリースした後、「ユーザーがどこで離脱しているのか」「どの機能が使われているのか」を把握せずに改善を続けるのは、目を閉じたまま道を歩くようなものです。**Amplitude Analytics**は、モバイルアプリ向けのプロダクトアナリティクスツールとして世界トップクラスのシェアを持ち、PostHog・Mixpanelと並ぶ3大ツールの一つです。

Rorkで作成したReact Native / Expoベースのアプリに対して、Amplitudeを導入することで以下のような分析が可能になります。

- **イベントトラッキング**：ボタンタップ・画面遷移・購入完了など、あらゆるユーザー操作を記録
- **ファネル分析**：「登録→チュートリアル→課金」といった転換率の可視化
- **リテンション分析**：初日・7日後・30日後のユーザー継続率の計測
- **コホート分析**：特定のユーザーグループの行動パターン比較
- **A/Bテスト統合**：Amplitude ExperimentsによるUI改善の効果測定

無料プラン（月間100万イベントまで）でも十分実用的な分析が可能なため、個人開発者からスタートアップまで幅広く活用されています。

## インストールと初期設定

まず、RorkプロジェクトにAmplitudeのSDKを追加します。2026年現在、React Native向けには公式パッケージ `@amplitude/analytics-react-native` が推奨されています。

```bash
# Amplitudeパッケージのインストール
npm install @amplitude/analytics-react-native

# Expoプロジェクトの場合、expo-device と expo-application も必要
npx expo install expo-device expo-application
```

次に、Amplitudeダッシュボード（[amplitude.com](https://amplitude.com)）でプロジェクトを作成し、APIキーを取得します。APIキーは環境変数として管理しましょう。

```typescript
// app/_layout.tsx（エントリーポイントでAmplitudeを初期化）
import { init } from '@amplitude/analytics-react-native';
import { useEffect } from 'react';

export default function RootLayout() {
  useEffect(() => {
    // APIキーは環境変数から取得（直接ハードコードしない）
    init(process.env.EXPO_PUBLIC_AMPLITUDE_API_KEY!, {
      // オプション設定
      trackingOptions: {
        ipAddress: false,       // IPアドレスの記録を無効化（プライバシー配慮）
        language: true,         // 言語設定を記録
        platform: true,         // プラットフォームを記録
        osVersion: true,        // OSバージョンを記録
        appVersion: true,       // アプリバージョンを記録
      },
      logLevel: __DEV__ ? 3 : 0, // 開発中はデバッグログを表示
    });
  }, []);

  // ... ナビゲーション設定など
}
```

`.env.local` に APIキーを設定します:

```bash
# .env.local
EXPO_PUBLIC_AMPLITUDE_API_KEY=YOUR_AMPLITUDE_API_KEY
```

> **セキュリティの注意**: `EXPO_PUBLIC_` プレフィックスのある変数はクライアントサイドに公開されます。Amplitudeのフロントエンド用APIキーは公開しても問題ありませんが、シークレットキーは絶対に公開しないようにしましょう。

## イベントトラッキングの実装

Amplitudeの核心は**イベントトラッキング**です。ユーザーの重要なアクションを記録することで、アプリの使われ方を深く理解できます。

### カスタムフックでイベントを統一管理

アプリ全体でAmplitudeを使いやすくするために、カスタムフックを作成することをおすすめします。

```typescript
// hooks/useAmplitude.ts
import { track, setUserId, identify, Identify } from '@amplitude/analytics-react-native';

export const useAmplitude = () => {
  // ユーザーIDを設定（ログイン後に呼び出す）
  const setUser = (userId: string, userProperties?: Record<string, unknown>) => {
    setUserId(userId);
    if (userProperties) {
      const identifyEvent = new Identify();
      Object.entries(userProperties).forEach(([key, value]) => {
        identifyEvent.set(key, value);
      });
      identify(identifyEvent);
    }
  };

  // イベントを記録するメイン関数
  const trackEvent = (
    eventName: string,
    eventProperties?: Record<string, unknown>
  ) => {
    track(eventName, eventProperties);
  };

  // よく使うイベントの型安全なラッパー
  const trackScreenView = (screenName: string) => {
    trackEvent('Screen Viewed', { screen_name: screenName });
  };

  const trackButtonTap = (buttonName: string, context?: string) => {
    trackEvent('Button Tapped', { button_name: buttonName, context });
  };

  const trackPurchase = (productId: string, price: number, currency = 'JPY') => {
    trackEvent('Purchase Completed', { product_id: productId, price, currency });
  };

  return { setUser, trackEvent, trackScreenView, trackButtonTap, trackPurchase };
};
```

### 実際の使用例

```typescript
// screens/HomeScreen.tsx
import { useEffect } from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { useAmplitude } from '@/hooks/useAmplitude';

export default function HomeScreen() {
  const { trackScreenView, trackButtonTap } = useAmplitude();

  useEffect(() => {
    // 画面表示をトラッキング
    trackScreenView('Home');
  }, []);

  const handleUpgradePress = () => {
    // ボタンタップをトラッキング（文脈情報も記録）
    trackButtonTap('Upgrade to Premium', 'home_banner');
    // ... 課金フローへ遷移
  };

  return (
    <TouchableOpacity onPress={handleUpgradePress}>
      <Text>プレミアムにアップグレード</Text>
    </TouchableOpacity>
  );
}
```

### 重要なイベント設計のベストプラクティス

イベント名の命名規則を統一することが、後のダッシュボード分析を大きく左右します。一般的に`Object Action`形式（例：`Purchase Completed`、`Screen Viewed`）が推奨されています。

## ファネル分析でコンバージョン率を把握する

ファネル分析は、「何人のユーザーが特定の一連のアクションを完了したか」を可視化します。例えば、登録完了から課金完了までのコンバージョン率を計測することで、どのステップに改善の余地があるかが明確になります。

### 登録フローのファネル実装例

```typescript
// services/analytics/funnel.ts
import { track } from '@amplitude/analytics-react-native';

// 各ステップでイベントを記録
export const funnelEvents = {
  // 登録フロー
  registrationStarted: () =>
    track('Registration Started'),

  emailEntered: (method: 'email' | 'apple' | 'google') =>
    track('Email Entered', { auth_method: method }),

  registrationCompleted: (userId: string) =>
    track('Registration Completed', { user_id: userId }),

  // オンボーディングフロー
  onboardingStepCompleted: (step: number, stepName: string) =>
    track('Onboarding Step Completed', { step_number: step, step_name: stepName }),

  onboardingCompleted: () =>
    track('Onboarding Completed'),

  // 課金フロー
  paywallViewed: (source: string) =>
    track('Paywall Viewed', { source }),

  purchaseInitiated: (planId: string, price: number) =>
    track('Purchase Initiated', { plan_id: planId, price }),

  purchaseCompleted: (planId: string, price: number) =>
    track('Purchase Completed', { plan_id: planId, price, currency: 'JPY' }),

  purchaseFailed: (reason: string) =>
    track('Purchase Failed', { failure_reason: reason }),
};
```

Amplitudeダッシュボードでは、これらのイベントを使ったファネルを視覚的に構築できます。たとえば「Registration Started → Email Entered → Registration Completed → Onboarding Completed → Purchase Completed」という5ステップのファネルを設定すると、各ステップでの離脱率が一目でわかります。

さらに詳細なアナリティクス戦略については、[Rork アプリ App Store Connect Analytics 入門ガイド](/articles/rork-business/rork-app-store-connect-analytics-beginners-guide) と組み合わせることで、ストア外部の流入データと内部の行動データを統合的に把握できます。

## リテンション分析とユーザー継続率の改善

アプリの成功指標として、リテンション率（継続率）は非常に重要です。「Day 1・Day 7・Day 30」のリテンション率を計測・改善することが、持続的な成長の鍵となります。

Amplitudeのリテンション分析機能では、特定のイベント（例：「App Opened」）を起点に、何日後に何%のユーザーが戻ってきたかを自動計算します。

```typescript
// アプリ起動イベントを毎回記録（リテンション計測の起点）
import { track } from '@amplitude/analytics-react-native';
import { AppState, AppStateStatus } from 'react-native';
import { useEffect, useRef } from 'react';

export function useAppStateTracking() {
  const appState = useRef(AppState.currentState);

  useEffect(() => {
    const subscription = AppState.addEventListener('change', (nextAppState: AppStateStatus) => {
      if (
        appState.current.match(/inactive|background/) &&
        nextAppState === 'active'
      ) {
        // バックグラウンドからフォアグラウンドに戻ったとき
        track('App Opened', { source: 'background_to_foreground' });
      }
      appState.current = nextAppState;
    });

    // 初回起動時
    track('App Opened', { source: 'cold_start' });

    return () => subscription.remove();
  }, []);
}
```

リテンション率が低い場合は、プッシュ通知の再エンゲージメント施策や、[Rork アプリのアプリ内レビュープロンプト実装ガイド](/articles/rork-dev/rork-inapp-review-prompt-ios-android-guide) を参考にしたエンゲージメント向上が効果的です。

## PostHogとの使い分け・Amplitudeが向いているケース

すでにRorkアプリのアナリティクスツールとして [PostHog を導入している方](/articles/rork-dev/rork-posthog-product-analytics-guide) もいるかと思います。Amplitudeは以下のケースで特に強みを発揮します。

- **ファネル分析・コホート分析を深掘りしたい場合**：Amplitudeのグラフ機能は直感的で分析の自由度が高い
- **プロダクトチームが複数人いる場合**：Amplitudeはチームでのデータ共有・ダッシュボード作成に優れている
- **A/Bテスト（Amplitude Experiments）を活用したい場合**：FeatureFlagsとの統合が強力
- **大規模なデータを扱う場合**：エンタープライズ向けの機能が充実している

一方でPostHogは、自己ホスティングオプションやオープンソース性、セッションリプレイ機能において優れています。アプリの規模や目的に合わせて選択しましょう。

モバイルアナリティクスの選定で迷っている方には、[Rork アプリ パフォーマンス最適化完全ガイド](/articles/rork-dev/rork-app-performance-optimization-complete-guide) の「計測ファーストの思考法」セクションも参考になります。

## よくある疑問（FAQ）

**Q. Amplitudeの無料プランはどこまで使えますか？**

A. 無料プランでは月間100万イベントまで記録でき、基本的なファネル分析・リテンション分析・コホート分析が利用できます。個人開発のアプリや初期フェーズのスタートアップであれば、無料プランで十分な場合が多いです。ユーザー数が数万人規模になってから有料プランへの移行を検討しましょう。

**Q. Expoのmanaged workflowでAmplitudeは動作しますか？**

A. はい、`@amplitude/analytics-react-native` はExpo managed workflowでも動作します。ただし、バックグラウンド処理の一部機能（セッション管理など）はEAS Buildを利用したカスタムdevクライアントでの動作確認を推奨します。`expo-device` と `expo-application` を一緒にインストールすることで、デバイス情報の取得精度も向上します。

**Q. 個人情報保護法・GDPRへの対応は必要ですか？**

A. はい、ユーザーのアクティビティを記録する場合、プライバシーポリシーへの記載が必要です。Amplitudeは標準でIPアドレスの匿名化オプションを提供しています。初期設定の `trackingOptions: { ipAddress: false }` を設定することをおすすめします。また、EUユーザーを対象とする場合はAmplitudeのGDPRコンプライアンス設定も合わせてご確認ください。

**Q. 既存のコードに大量のtrackイベントを追加するのが大変です。効率的な方法はありますか？**

A. Expo Router使用時は、`_layout.tsx` に画面遷移のリスナーを設定することで、すべての画面遷移を自動記録できます。ボタンなど個別のUIについては、カスタムコンポーネントでラップして共通のトラッキングロジックを付与するパターンが効率的です。すべてのイベントを追加しようとせず、まずコンバージョンファネルと主要機能の5〜10イベントから始めることを推奨します。

## まとめ：データドリブンなアプリ改善を始めよう

Amplitudeを導入することで、これまで「感覚」で行っていたアプリ改善を、データに基づいた意思決定へと変えることができます。まずは無料プランで以下の3ステップから始めてみてください。

1. **SDKを導入して初期化する**（所要時間：30分）
2. **主要イベント10個をトラッキングする**（登録・コア機能使用・課金フロー）
3. **1週間後にAmplitudeダッシュボードを確認し、最初の改善仮説を立てる**

データが蓄積されれば、ファネル改善・プッシュ通知の最適化・新機能の優先度決定など、あらゆる判断がより確実なものになります。Rorkで作ったアプリを、データの力で継続的に成長させていきましょう。

モバイルアナリティクスのさらなる活用について詳しく知りたい方には、[「モバイルアプリ解析完全ガイド」（小西一有・著）](https://www.amazon.co.jp/dp/4839981701/?tag=pinocchio-22) がおすすめです。アナリティクス設計からデータ活用まで体系的に解説された一冊です。
