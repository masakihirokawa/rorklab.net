アクセシビリティは単なるコンプライアンス要件ではなく、ユーザーベースを広げ、より良いUX を全員に提供するためのデザイン哲学です。Rork Maxで構築したアプリをスクリーンリーダー・大文字サイズ・色覚多様性に対応させることで、障害を持つユーザーを含むすべてのユーザーが平等にアプリを利用できるようになります。

## なぜアクセシビリティが重要か

**App Store 審査要件**: Appleは2023年からアクセシビリティ対応をアプリ審査の必須項目にしました。対応不足だと審査に落ちる可能性があります。

**ユーザーリーチ**: 世界人口の約15%（11億人以上）が何らかの障害を持っています。アクセシビリティ対応により、これらのユーザーへのアプリケーションの利用を可能にします。

**法的要件**: 米国ADA（障害を持つアメリカ人法）、EU WCAG 2.1（Web Content Accessibility Guidelines）など、地域によってはアクセシビリティが法的義務です。

**ビジネス効果**: SEOランキング向上、ユーザー満足度向上、ブランドイメージ向上。

## accessibilityLabel の実装

すべてのインタラクティブ要素にセマンティックなラベルを付与します。スクリーンリーダー（VoiceOverやTalkBack）ユーザーは視覚情報に頼れないため、テキストラベルが絶対に必要です。

```typescript
import { TouchableOpacity, Text, View } from "react-native";

export function AddButton() {
  return (
    <TouchableOpacity
      accessibilityLabel="新しいタスクを追加"
      accessibilityRole="button"
      onPress={() => console.log("Add task")}
      style={{
        width: 56,
        height: 56,
        borderRadius: 28,
        backgroundColor: "#007AFF",
        justifyContent: "center",
        alignItems: "center",
      }}
    >
      <Text style={{ fontSize: 24, color: "#fff" }}>+</Text>
    </TouchableOpacity>
  );
}
```

**accessibilityLabel のベストプラクティス:**
- 簡潔で説明的（2～5語が目安）
- 「ボタン」「リンク」などの要素タイプは含めない（accessibilityRoleで既に指定）
- アクションを説明する動詞で始める（「追加」「削除」など）
- ローカライゼーション対応（文字列をリソースファイルに外部化）

## accessibilityHint の活用

`accessibilityHint` は `accessibilityLabel` の追加説明です。ユーザーが要素を選択した後に読み上げられます。

```typescript
<TouchableOpacity
  accessibilityLabel="保存"
  accessibilityHint="現在のフォーム内容を保存します"
  accessibilityRole="button"
  onPress={handleSave}
>
  <Text>保存</Text>
</TouchableOpacity>
```

何をするかはラベルで、なぜするか / その結果は hint で説明します。

## accessibilityRole の正確な指定

各要素の役割を正確に指定することで、スクリーンリーダーが適切に解釈します：

```typescript
import { Text, View, Switch, TextInput, ScrollView } from "react-native";

// ボタン
<TouchableOpacity accessibilityRole="button">
  <Text>クリック</Text>
</TouchableOpacity>

// テキスト入力
<TextInput
  accessibilityRole="search"
  placeholder="検索"
/>

// ラジオボタン
<View accessibilityRole="radio" accessibilityState={{ selected: true }}>
  <Text>オプションA</Text>
</View>

// チェックボックス
<View accessibilityRole="checkbox" accessibilityState={{ checked: false }}>
  <Text>同意する</Text>
</View>

// ヘッダー
<View accessibilityRole="header">
  <Text style={{ fontSize: 24, fontWeight: "bold" }}>セクションタイトル</Text>
</View>

// リスト
<ScrollView accessibilityRole="list">
  {items.map((item, idx) => (
    <View key={idx} accessibilityRole="listitem">
      <Text>{item.title}</Text>
    </View>
  ))}
</ScrollView>
```

## スクリーンリーダー ナビゲーション順序

VoiceOver や TalkBack がフォーカスする順序を制御し、アプリの論理的なフローに合わせます。

```typescript
import { View, Text, Button, AccessibilityInfo } from "react-native";

export function FormScreen() {
  const firstInputRef = React.useRef<TextInput>(null);

  return (
    <View
      accessibilityViewIsModal={true} // モーダル表示時はこのView外へのフォーカスを遮断
      accessible={true}
    >
      <Text accessibilityRole="header">ユーザー登録</Text>

      {/* ナビゲーション順序: メール → パスワード → 登録ボタン */}
      <TextInput
        ref={firstInputRef}
        placeholder="メールアドレス"
        accessibilityLabel="メールアドレス入力"
        accessibilityViewIsModal={false}
      />

      <TextInput
        placeholder="パスワード"
        secureTextEntry={true}
        accessibilityLabel="パスワード入力"
      />

      <Button
        title="登録"
        accessibilityLabel="登録"
        onPress={handleSignUp}
      />

      {/* フォーカス状態の自動設定 */}
      React.useEffect(() => {
        if (firstInputRef.current) {
          AccessibilityInfo.setAccessibilityFocus(
            findNodeHandle(firstInputRef.current)
          );
        }
      }, []);
    </View>
  );
}
```

<Callout type="info">
**Tip**: `accessibilityViewIsModal={true}` を使用すると、モーダルダイアログ外のコンテンツへのフォーカスを遮断でき、ユーザーが背景要素を誤って選択することを防げます。
</Callout>

## 動的フォント サイズ（Dynamic Type）

ユーザーがシステムレベルでフォントサイズを変更できるよう対応します。これは視力が低下したユーザーにとって重要です。

```typescript
import { useWindowDimensions, Text, StyleSheet } from "react-native";

export function AccessibleText({ children, variant = "body" }) {
  const fontScale = useWindowDimensions().fontScale;

  const styles = StyleSheet.create({
    heading: {
      fontSize: 24 * fontScale,
      fontWeight: "bold",
      marginBottom: 16,
    },
    body: {
      fontSize: 16 * fontScale,
      lineHeight: 24 * fontScale,
    },
    caption: {
      fontSize: 14 * fontScale,
    },
  });

  const styleKey = variant as keyof typeof styles;
  return <Text style={styles[styleKey]}>{children}</Text>;
}
```

React Nativeの `Text` コンポーネントは `allowFontScaling` プロップをデフォルトで `true` に設定しています。あえてスケーリングを無効化する必要はありません。

```typescript
// フォントスケーリングを有効化（推奨）
<Text allowFontScaling={true}>
  このテキストはユーザーのフォントサイズに応じてスケールします
</Text>

// フォントスケーリングを無効化（非推奨）
<Text allowFontScaling={false}>
  このテキストは常に同じサイズです（避けるべき）
</Text>
```

## 色覚多様性への対応

色覚多様性（色盲・色弱）を持つユーザーは人口の約8%です。色だけに頼ったUI は避けます。

```typescript
import { View, Text, StyleSheet } from "react-native";

const styles = StyleSheet.create({
  statusContainer: {
    flexDirection: "row",
    alignItems: "center",
    padding: 12,
    borderRadius: 8,
  },
  statusSuccess: {
    backgroundColor: "#4CAF50", // 緑
  },
  statusError: {
    backgroundColor: "#f44336", // 赤
  },
  statusIcon: {
    marginRight: 12,
    fontSize: 20,
  },
});

// 良い例: 色 + アイコン + テキストで情報を伝える
export function StatusMessage({ type, message }) {
  const isSuccess = type === "success";
  return (
    <View
      style={[
        styles.statusContainer,
        isSuccess ? styles.statusSuccess : styles.statusError,
      ]}
    >
      <Text style={styles.statusIcon}>{isSuccess ? "✓" : "✕"}</Text>
      <Text style={{ fontWeight: "600" }}>{message}</Text>
    </View>
  );
}

// より良い例: パターンも使用
export function AccessibleStatusBadge({ type }) {
  const patterns = {
    success: { color: "#4CAF50", pattern: "diagonal", icon: "✓" },
    warning: { color: "#FF9800", pattern: "horizontal", icon: "!" },
    error: { color: "#f44336", pattern: "dots", icon: "✕" },
  };

  const patternStyle = patterns[type];
  return (
    <View
      style={{
        backgroundColor: patternStyle.color,
        padding: 12,
        borderRadius: 8,
        borderBottomWidth: 3,
        borderBottomColor: "rgba(0,0,0,0.2)",
      }}
    >
      <Text style={{ fontSize: 20 }}>{patternStyle.icon}</Text>
    </View>
  );
}
```

**WCAG色コントラスト要件:**
- AA レベル（推奨）: 4.5:1（通常テキスト）、3:1（大サイズテキスト）
- AAA レベル（優秀）: 7:1（通常テキスト）、4.5:1（大サイズテキスト）

オンライン色覚障害シミュレータ（https://www.color-blindness.com/）で色覚多様性ユーザーがアプリをどう見るか検証できます。

## タッチターゲット サイズ

モバイルアクセシビリティではタッチターゲットサイズが重要です。Apple・Googleの推奨値は44×44ポイント以上。

```typescript
import { TouchableOpacity, View, Text } from "react-native";

// ❌ 悪い例: ターゲットサイズが小さい
<TouchableOpacity
  style={{ width: 24, height: 24 }}
  onPress={() => {}}
>
  <Text>削除</Text>
</TouchableOpacity>

// ✅ 良い例: 44×44pt以上
<TouchableOpacity
  style={{ width: 44, height: 44, justifyContent: "center", alignItems: "center" }}
  onPress={() => {}}
>
  <Text>削除</Text>
</TouchableOpacity>

// ✅ さらに良い例: hitSlop で見た目は小さく、タッチ領域は大きく
<TouchableOpacity
  style={{ width: 24, height: 24, justifyContent: "center", alignItems: "center" }}
  hitSlop={{ top: 10, bottom: 10, left: 10, right: 10 }}
  onPress={() => {}}
>
  <Text style={{ fontSize: 12 }}>削除</Text>
</TouchableOpacity>
```

`hitSlop` はタッチ反応エリアを拡張しますが、視覚的には小さく見えます。細かいアイコンボタンに有効です。

```typescript
import { render, screen } from "@testing-library/react-native";

test("ボタンが正しいラベルを持つ", () => {
```

## App Store 審査 チェックリスト

Appleのアクセシビリティ審査に合格するために：

1. **スクリーンリーダー対応**
   - [ ] すべてのインタラクティブ要素に `accessibilityLabel` がある
   - [ ] VoiceOver でナビゲーション順序が論理的か確認
   - [ ] 装飾的なみ画像に `accessible={false}` を設定

2. **フォントサイズ**
   - [ ] `allowFontScaling={true}` で動的フォントに対応
   - [ ] 最大フォントサイズでもレイアウトが崩れない

3. **色コントラスト**
   - [ ] テキスト・背景の色コントラスト比がWCAG AA以上
   - [ ] ダークモードでもコントラスト要件を満たす

4. **タッチターゲット**
   - [ ] すべてのボタンが44×44pt以上（hitSlop含む）

5. **フォーカス管理**
   - [ ] キーボード操作でアプリのすべての機能にアクセス可能
   - [ ] フォーカスが見える状態で、どの要素がフォーカスされているか明確

6. **App Store Connect での記入**
   - アクセシビリティ機能説明セクションを記入:
     ```
     VoiceOver対応: 有
     ダイナミックタイプ対応: 有
     ホームボタンジェスチャー対応: 有
     画面上のテキストのサイズ調整: 有
     ```

## よくある落選理由

1. **スクリーンリーダー無応答**: 要素がVoiceOverで選択不可
2. **タッチターゲット小さすぎ**: アイコンボタンが24×24pt未満
3. **色コントラスト不足**: テキスト・背景の比率が3:1以下
4. **フォーカストラップ**: VoiceOver で特定要素から抜け出せない
5. **説明不足**: App Store での「アクセシビリティ機能」セクションが空白
