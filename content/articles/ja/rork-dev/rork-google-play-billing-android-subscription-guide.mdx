## はじめに：Google Play Billing とは

アプリで安定した収益を得るうえで、**Google Play Billing API** の理解は欠かせません。Google Play Billing は Android アプリ内で課金処理を行うための公式 API です。月額・年額のサブスクリプションから、コンテンツの単発購入まで幅広い課金形態をサポートしています。

Rork で作成したアプリは React Native / Expo ベースで動作するため、Google Play Billing を扱うためのライブラリを活用することで、比較的短期間で課金機能を組み込むことができます。この記事では、**react-native-iap** を使った実装方法を中心に、Google Play Console での設定から実機テスト・公開までの流れを丁寧に解説します。

## Google Play Billing の仕組みを理解する

実装に入る前に、課金の種類と処理フローを整理しておきましょう。

Google Play Billing が扱う課金には大きく2種類あります。

- **サブスクリプション（定期課金）**: 月額・年額など一定期間ごとに自動更新される課金形態。アプリの継続利用を促すのに最適です。
- **消費型アイテム（都度課金）**: コインや特定機能のアンロックなど、一回きりの購入。消費後に再購入が可能です。
- **非消費型アイテム（永久購入）**: 一度購入すると永続的に利用できるアドオンやプレミアムアクセスなど。

処理フローはシンプルです。アプリが Google Play に購入リクエストを送り、ユーザーが決済を完了すると、Google のサーバーが購入トークンを返します。このトークンをバックエンドで検証することで、課金状態を安全に確認できます。

## 環境準備：Google Play Console の設定

まずは Google Play Console でアプリと課金アイテムを準備します。

**1. アプリを Google Play Console に登録する**

Google Play に公開するアプリは事前にコンソールへ登録が必要です。基本的なアプリ情報の入力と初回 APK（または AAB）のアップロードを済ませてください。まだアプリを公開していない場合は、[Rork アプリを Google Play に公開する手順](/articles/rork-dev/google-play-publishing) も合わせてご参照ください。

**2. サブスクリプション商品を作成する**

コンソールの「収益化 → 定期購入」から新しいサブスクリプションを作成します。

- **商品ID（Product ID）**: コード内で参照するユニークな識別子です（例: `premium_monthly`）
- **価格**: 各国の通貨で設定可能です
- **課金サイクル**: 月次、年次、週次などから選択できます
- **無料トライアル**: 設定することでユーザーの初期障壁を下げられます

**3. サービスアカウントの作成（バックエンド検証用）**

購入トークンをサーバー側で検証する場合は、Google Cloud Console でサービスアカウントを作成し、JSON キーをダウンロードしておきます。フロントエンドのみで完結する簡易実装では省略可能ですが、セキュリティ上はバックエンド検証を推奨します。

## react-native-iap の導入

Rork プロジェクトに `react-native-iap` を追加します。このライブラリは iOS（StoreKit）と Android（Google Play Billing）の両方を統一的なインターフェースで扱えるため、クロスプラットフォーム開発に非常に便利です。

```bash
# プロジェクトに react-native-iap を追加
npm install react-native-iap

# EAS Build 使用時は expo-build-properties も設定
npx expo install expo-build-properties
```

`app.json` または `app.config.js` に以下の設定を追加します。

```json
{
  "expo": {
    "plugins": [
      [
        "expo-build-properties",
        {
          "android": {
            "compileSdkVersion": 35,
            "targetSdkVersion": 35
          }
        }
      ]
    ]
  }
}
```

## サブスクリプションの実装コード

以下は、サブスクリプション購入フローの基本的な実装例です。

```tsx
import React, { useEffect, useState } from 'react';
import { View, Text, TouchableOpacity, Alert, Platform } from 'react-native';
import {
  initConnection,
  getSubscriptions,
  requestSubscription,
  purchaseUpdatedListener,
  purchaseErrorListener,
  finishTransaction,
  type SubscriptionAndroid,
} from 'react-native-iap';

// Google Play Console で設定した商品ID
const SUBSCRIPTION_SKUS = Platform.select({
  android: ['premium_monthly', 'premium_yearly'],
  ios: ['premium_monthly', 'premium_yearly'],
}) ?? [];

export default function SubscriptionScreen() {
  const [subscriptions, setSubscriptions] = useState<SubscriptionAndroid[]>([]);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    // Google Play Billing との接続を初期化
    const initIAP = async () => {
      try {
        await initConnection();
        // 利用可能なサブスクリプション一覧を取得
        const subs = await getSubscriptions({ skus: SUBSCRIPTION_SKUS });
        setSubscriptions(subs as SubscriptionAndroid[]);
      } catch (error) {
        console.error('IAP初期化エラー:', error);
      }
    };

    initIAP();

    // 購入成功時のリスナー
    const purchaseListener = purchaseUpdatedListener(async (purchase) => {
      const receipt = purchase.transactionReceipt;
      if (receipt) {
        try {
          // バックエンドにトークンを送信して検証（推奨）
          await verifyPurchaseOnServer(purchase.purchaseToken ?? '');
          // 購入を完了（Acknowledge）しないと3日後に自動返金される
          await finishTransaction({ purchase, isConsumable: false });
          Alert.alert('購入完了', 'プレミアムプランが有効になりました！');
        } catch (e) {
          console.error('購入処理エラー:', e);
        }
      }
    });

    // 購入エラー時のリスナー
    const errorListener = purchaseErrorListener((error) => {
      console.error('購入エラー:', error.code, error.message);
    });

    return () => {
      purchaseListener.remove();
      errorListener.remove();
    };
  }, []);

  // サブスクリプションの購入を開始
  const handleSubscribe = async (productId: string) => {
    setLoading(true);
    try {
      await requestSubscription({
        sku: productId,
        ...(Platform.OS === 'android' && {
          subscriptionOffers: [{ sku: productId, offerToken: '' }],
        }),
      });
    } catch (error) {
      Alert.alert('エラー', '購入処理に失敗しました。再度お試しください。');
    } finally {
      setLoading(false);
    }
  };

  // バックエンドで購入を検証する関数（例）
  const verifyPurchaseOnServer = async (purchaseToken: string) => {
    const response = await fetch('https://your-api.example.com/verify-purchase', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ purchaseToken, packageName: 'com.yourapp.name' }),
    });
    if (!response.ok) throw new Error('サーバー検証失敗');
  };

  return (
    <View style={{ flex: 1, padding: 24 }}>
      <Text style={{ fontSize: 22, fontWeight: 'bold', marginBottom: 16 }}>
        プレミアムプランを選択
      </Text>
      {subscriptions.map((sub) => (
        <TouchableOpacity
          key={sub.productId}
          onPress={() => handleSubscribe(sub.productId)}
          disabled={loading}
          style={{
            backgroundColor: '#4F46E5',
            padding: 16,
            borderRadius: 12,
            marginBottom: 12,
          }}
        >
          <Text style={{ color: 'white', fontSize: 16, fontWeight: '600' }}>
            {sub.title} — {sub.localizedPrice}/月
          </Text>
        </TouchableOpacity>
      ))}
    </View>
  );
}
```

このコードのポイントは、購入後に必ず `finishTransaction` を呼び出すことです。これを忘れると Google Play が「未消化の購入」と判断し、3日後に自動的に返金処理が行われてしまいます。

## テスト方法：ライセンステスターの活用

実際の課金をせずにテストするには、**Google Play Console のライセンステスター**機能を使います。

Google Play Console の「設定 → ライセンステスト」にテスト用の Google アカウントを登録すると、そのアカウントで行った購入は実際に請求されず、サブスクリプションのサイクルも短縮されます（月次サブスクリプションが5分ごとに更新されるなど）。

テスト時のよくある落とし穴をまとめます。

- **内部テストトラックへのアップロードが必要**: ライセンステスト機能を有効化するには、内部テストトラックに APK がアップロードされている必要があります
- **実機でのテストが必須**: エミュレーターでは Google Play Billing が正常動作しない場合があります
- **商品のアクティベーション**: 商品を作成後、有効化（Active 状態）にしないと取得できません

## よくあるエラーと対処法

Google Play Billing の実装でよく遭遇するエラーと解決策を解説します。

**`ITEM_NOT_OWNED` エラー**

サブスクリプションがすでに有効な状態で再購入しようとすると発生します。購入前に現在のサブスクリプション状態を確認するロジックを追加しましょう。

**`BILLING_UNAVAILABLE` エラー**

デバイスの Google Play サービスが古い場合や、テスト環境の設定に問題がある場合に発生します。テスト端末の Google Play サービスを最新版に更新してください。

**Acknowledge されない購入**

`finishTransaction` の呼び出しを忘れると、購入が「Pending」状態のまま3日後に返金されます。購入リスナー内で確実に呼び出すようにしましょう。

## よくある質問（FAQ）

**Q. Rork で作ったアプリに Google Play Billing を実装するとき、Rork Max が必要ですか？**

A. `react-native-iap` の追加はカスタムネイティブモジュールの組み込みを伴うため、EAS Build（Expo Application Services）が使えるプランを推奨します。Rork のプランによって EAS Build の利用可否が異なりますので、[Rork の料金プラン](/articles/rork-business/rork-pricing-plans) でご確認ください。

**Q. Google Play のサブスクリプション課金はいつアプリ側で受け取れますか？**

A. ユーザーが購入を完了した後、`purchaseUpdatedListener` が即座に呼び出されます。ただし、定期更新（月次・年次）のタイミングではバックエンド側で Webhook を受け取る方式（Google Play Developer Notifications）の実装も合わせて検討することをおすすめします。

**Q. 購入の検証をバックエンドで行わないと問題がありますか？**

A. フロントエンドのみの実装は不正利用のリスクがあります。攻撃者がトークンを偽造してプレミアム機能を不正に解除できる可能性があるため、本番リリースではバックエンドによる検証を必ず実装してください。Google Play Developer API を使った検証には、先述のサービスアカウント設定が必要です。

**Q. iOS と Android の両方で課金を実装したい場合はどうすればよいですか？**

A. `react-native-iap` はクロスプラットフォームに対応しているため、`Platform.select` で SKU を分岐するだけで、基本的なコードは共通化できます。ただし、iOS は StoreKit・Android は Google Play と、ストアごとに審査や設定が異なる点に注意してください。

この記事では、Rork で作成した Android アプリに Google Play Billing API を使ったサブスクリプション課金を実装する手順を解説しました。
