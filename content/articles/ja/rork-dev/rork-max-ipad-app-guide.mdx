## はじめに

iPad は大画面ならではの生産性を提供するプラットフォームです。しかし、iPhone 向けのアプリをそのまま iPad で動かすと、画面の大部分が無駄になり、ユーザー体験が大きく損なわれます。Apple はマルチカラムレイアウト、Split View、Slide Over、ドラッグ＆ドロップなど iPad 専用の機能を提供しており、これらに対応することで App Store 審査での評価も向上します。

**Rork Max** はネイティブ SwiftUI コードを生成するため、iPad の全機能にフルアクセスできます。本記事では、Rork Max の AI チャットだけで iPad 最適化アプリをゼロから構築する方法を、プロンプト例と実際のコードを交えて解説します。

## iPad アプリ設計の基本原則

### iPhone と iPad の違い

iPad アプリを設計する際に最も重要なのは「画面サイズに応じた適応型レイアウト」です。iPhone では縦方向のシングルカラムが基本ですが、iPad では横方向の余白を活かしたマルチカラムレイアウトが求められます。

| 要素 | iPhone | iPad |
|------|--------|------|
| レイアウト | シングルカラム | 2〜3カラム |
| ナビゲーション | NavigationStack（プッシュ遷移） | NavigationSplitView（サイドバー＋詳細） |
| マルチタスク | 非対応 | Split View / Slide Over |
| 入力 | タッチ | タッチ + Apple Pencil + キーボード + トラックパッド |
| ドラッグ＆ドロップ | アプリ内のみ | アプリ間で可能 |

### Apple の iPad 設計ガイドラインのポイント

Apple の Human Interface Guidelines では、iPad アプリに以下の対応を推奨しています：

1. **全画面サイズへの対応** — ポートレートとランドスケープの両方で美しく表示
2. **マルチタスクサポート** — Split View と Slide Over でのレイアウト崩れ防止
3. **ポインタ対応** — トラックパッドやマウスのホバー効果
4. **キーボードショートカット** — 外部キーボード使用時のショートカット対応

## Rork Max で iPad 対応プロジェクトを作成する

### Step 1: プロジェクトの初期設定

Rork Max でプロジェクトを作成する際、iPad 対応を明示するプロンプトを最初に伝えます。

**おすすめプロンプト例：**

```
iPad と iPhone の両方に対応したノート管理アプリを作ってください。
iPad では NavigationSplitView を使った2カラムレイアウト、
iPhone では NavigationStack を使ったシングルカラムレイアウトにしてください。
Split View と Slide Over にも対応してください。
```

Rork Max の AI がこのプロンプトを解釈し、デバイスの画面サイズに応じて自動的にレイアウトを切り替える SwiftUI コードを生成します。

### Step 2: NavigationSplitView の実装

NavigationSplitView は iPad で最も重要な UI コンポーネントです。サイドバー、コンテンツ一覧、詳細画面を1つのビューで管理できます。

```swift
// NavigationSplitView を使った iPad 対応レイアウト
// iPad: サイドバー + リスト + 詳細の3カラム表示
// iPhone: 自動的にシングルカラムのプッシュ遷移に切り替わる

struct ContentView: View {
    @State private var selectedCategory: Category?
    @State private var selectedNote: Note?
    @State private var columnVisibility: NavigationSplitViewVisibility = .all

    var body: some View {
        NavigationSplitView(columnVisibility: $columnVisibility) {
            // サイドバー: カテゴリ一覧
            List(categories, selection: $selectedCategory) { category in
                Label(category.name, systemImage: category.icon)
            }
            .navigationTitle("カテゴリ")
        } content: {
            // コンテンツ: ノート一覧
            if let category = selectedCategory {
                NoteListView(
                    category: category,
                    selectedNote: $selectedNote
                )
            } else {
                ContentUnavailableView(
                    "カテゴリを選択",
                    systemImage: "folder",
                    description: Text("左のサイドバーからカテゴリを選んでください")
                )
            }
        } detail: {
            // 詳細: ノート編集
            if let note = selectedNote {
                NoteDetailView(note: note)
            } else {
                ContentUnavailableView(
                    "ノートを選択",
                    systemImage: "doc.text",
                    description: Text("ノートを選んで編集を始めましょう")
                )
            }
        }
    }
}

// 期待する出力:
// iPad横向き → サイドバー | ノート一覧 | ノート詳細 の3カラム表示
// iPad縦向き → サイドバーが折りたたみ可能な2カラム表示
// iPhone → 通常のプッシュ遷移型ナビゲーション
```

このコードのポイントは `columnVisibility` パラメータです。ユーザーがサイドバーの表示・非表示を制御でき、iPad の画面回転に応じて自動的にレイアウトが調整されます。

## マルチタスク対応 — Split View と Slide Over

### Split View への対応

iPad ユーザーの多くは Split View を使って2つのアプリを同時に表示します。Rork Max で生成されたアプリはデフォルトで Split View に対応していますが、レイアウトが崩れないように注意が必要です。

**Rork Max に伝えるプロンプト：**

```
Split View で画面幅が狭くなった場合にも
レイアウトが崩れないようにしてください。
horizontalSizeClass を使って
Compact と Regular で表示を切り替えてください。
```

```swift
// Split View 対応: 画面幅に応じたアダプティブレイアウト
struct AdaptiveNoteView: View {
    @Environment(\.horizontalSizeClass) private var sizeClass
    @State private var selectedNote: Note?

    var body: some View {
        if sizeClass == .regular {
            // iPad の広い画面 → 2カラム表示
            HStack(spacing: 0) {
                NoteListView(selectedNote: $selectedNote)
                    .frame(width: 320)

                Divider()

                if let note = selectedNote {
                    NoteEditorView(note: note)
                } else {
                    placeholderView
                }
            }
        } else {
            // iPhone または Split View で幅が狭い場合
            // → シングルカラム表示
            NavigationStack {
                NoteListView(selectedNote: $selectedNote)
                    .navigationDestination(item: $selectedNote) { note in
                        NoteEditorView(note: note)
                    }
            }
        }
    }

    private var placeholderView: some View {
        ContentUnavailableView(
            "ノートを選択してください",
            systemImage: "square.and.pencil"
        )
        .frame(maxWidth: .infinity, maxHeight: .infinity)
    }
}

// 期待する出力:
// Regular（iPad全画面/広いSplit View） → 左にリスト、右にエディタ
// Compact（iPhone/狭いSplit View） → リスト → タップで遷移
```

### Slide Over 対応の注意点

Slide Over では画面幅が iPhone 相当まで狭くなります。`horizontalSizeClass` が `.compact` に変わるため、上記のコードで自動的にシングルカラムに切り替わります。追加のコードは不要ですが、ツールバーやボタンが隠れないようにテストすることが重要です。

## ドラッグ＆ドロップの実装

iPad の大きな特徴の1つが、アプリ間のドラッグ＆ドロップです。ユーザーが写真アプリから画像をドラッグして自分のアプリにドロップする、といった操作を実現できます。

**Rork Max プロンプト例：**

```
ノートに画像を追加できるようにしてください。
iPad でアプリ間のドラッグ＆ドロップで画像を追加できるようにし、
dropDestination と Transferable を使って実装してください。
```

```swift
// iPad ドラッグ＆ドロップ対応: 画像の受け取り
struct NoteEditorView: View {
    @Bindable var note: Note
    @State private var isDropTargetActive = false

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 16) {
                TextField("タイトル", text: $note.title)
                    .font(.title)

                TextEditor(text: $note.body)
                    .frame(minHeight: 200)

                // 画像ドロップエリア
                imageDropArea
            }
            .padding()
        }
        .navigationTitle("ノート編集")
    }

    private var imageDropArea: some View {
        ZStack {
            RoundedRectangle(cornerRadius: 12)
                .strokeBorder(
                    style: StrokeStyle(lineWidth: 2, dash: [8])
                )
                .foregroundStyle(isDropTargetActive ? .blue : .secondary)
                .frame(height: 150)

            VStack(spacing: 8) {
                Image(systemName: "photo.on.rectangle.angled")
                    .font(.largeTitle)
                Text("画像をここにドラッグ＆ドロップ")
                    .font(.subheadline)
            }
            .foregroundStyle(.secondary)
        }
        .dropDestination(for: Data.self) { items, _ in
            // ドロップされた画像データを処理
            for item in items {
                if let image = UIImage(data: item) {
                    note.images.append(image)
                }
            }
            return !items.isEmpty
        } isTargeted: { targeted in
            isDropTargetActive = targeted
        }
    }
}

// 期待する出力:
// 写真アプリからドラッグ → 点線エリアが青色にハイライト → ドロップで画像追加
```

## Apple Pencil 対応の基本

Rork Max では Apple Pencil を活用したお絵描き機能やメモ機能も実装できます。SwiftUI の `PencilKit` フレームワークを通じて、手書きやスケッチ機能をアプリに追加できます。

**プロンプト例：**

```
Apple Pencil で手書きメモが書けるキャンバスを追加してください。
PencilKit の PKCanvasView を使い、
ペンの種類や色を選択できるツールバーも表示してください。
```

```swift
// Apple Pencil 対応: PencilKit を使った手書きキャンバス
import PencilKit

struct DrawingCanvasView: UIViewRepresentable {
    @Binding var canvasView: PKCanvasView
    @Binding var toolPicker: PKToolPicker

    func makeUIView(context: Context) -> PKCanvasView {
        canvasView.drawingPolicy = .anyInput  // Pencil以外も許可
        canvasView.backgroundColor = .systemBackground

        // ツールピッカーを表示
        toolPicker.setVisible(true, forFirstResponder: canvasView)
        toolPicker.addObserver(canvasView)
        canvasView.becomeFirstResponder()

        return canvasView
    }

    func updateUIView(_ uiView: PKCanvasView, context: Context) {}
}

// SwiftUI ビューで使用する例
struct HandwritingNoteView: View {
    @State private var canvasView = PKCanvasView()
    @State private var toolPicker = PKToolPicker()

    var body: some View {
        VStack {
            DrawingCanvasView(
                canvasView: $canvasView,
                toolPicker: $toolPicker
            )
        }
        .navigationTitle("手書きメモ")
        .toolbar {
            ToolbarItem(placement: .topBarTrailing) {
                Button("クリア") {
                    canvasView.drawing = PKDrawing()
                }
            }
        }
    }
}

// 期待する出力:
// Apple Pencil でキャンバスに自由に描画
// ツールバーからペン・マーカー・消しゴムを選択可能
```

## キーボードショートカットの追加

iPad で外部キーボードを使うユーザーは多く、キーボードショートカットを追加することでプロフェッショナルなアプリ体験を提供できます。

```swift
// キーボードショートカットの実装例
struct NoteAppView: View {
    @State private var showNewNote = false
    @State private var searchText = ""

    var body: some View {
        NavigationSplitView {
            NoteListView()
                .searchable(text: $searchText)
        } detail: {
            NoteDetailView()
        }
        .sheet(isPresented: $showNewNote) {
            NewNoteView()
        }
        // Command + N で新規ノート作成
        .keyboardShortcut("n", modifiers: .command)
        // Command + F で検索にフォーカス（searchable が自動処理）
    }
}

// DiscoverabilityGroup でショートカット一覧を表示
// iPad で Command キーを長押しするとショートカット一覧が表示される

// 期待する出力:
// ⌘N → 新規ノート作成シートが表示
// ⌘ 長押し → ショートカット一覧がオーバーレイ表示
```

## よくあるエラーと対処法

### レイアウトが iPad で崩れる

Rork Max に「固定幅を使わず、`frame(maxWidth: .infinity)` やジオメトリリーダーで柔軟に対応して」と伝えてください。固定サイズの指定は iPad の多様な画面サイズで問題を起こします。

### Split View でサイドバーが消える

`NavigationSplitView` の `columnVisibility` を `.all` に設定し、`navigationSplitViewStyle(.balanced)` を追加してバランスの取れた表示にします。

### Apple Pencil の入力が反応しない

`PKCanvasView` の `drawingPolicy` を `.anyInput` に設定していることを確認してください。`.pencilOnly` にすると指での入力が無効になります。

## FAQ

### Q: Rork Max で作った iPad アプリは iPhone でも動きますか？

はい、動きます。NavigationSplitView は iPhone では自動的に NavigationStack と同等の挙動になります。`horizontalSizeClass` を活用すれば、1つのコードベースで iPad と iPhone の両方に最適化されたレイアウトを提供できます。Rork Max のプロンプトで「iPhone と iPad の両方に対応」と指定すれば、AI が適切なアダプティブレイアウトを生成します。

### Q: iPad mini にも対応できますか？

はい、iPad mini は画面サイズが小さいものの `horizontalSizeClass` は `.regular` です。NavigationSplitView の `columnVisibility` を `.automatic` に設定しておけば、画面幅に応じてシステムが自動的にカラム数を調整します。特別な対応は不要で、Rork Max が生成するコードはそのまま iPad mini でも適切に表示されます。

### Q: Stage Manager には対応していますか？

Rork Max で生成される SwiftUI アプリは Stage Manager に自動対応しています。Stage Manager はアプリのウィンドウサイズを自由に変更できる macOS ライクな機能ですが、SwiftUI のアダプティブレイアウトがそのまま活用されるため、追加のコードは不要です。ただし、極端に小さいウィンドウサイズでのレイアウトをテストすることを推奨します。
