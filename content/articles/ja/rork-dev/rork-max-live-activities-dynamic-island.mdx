## はじめに

iOS 16.1以降、**Live Activities（ライブアクティビティ）**と**Dynamic Island（ダイナミックアイランド）**は、ユーザーにリアルタイム情報を届ける最も効果的なUIとなりましました。フードデリバリーの配送状況、スポーツのスコア、ライドシェアの到着時刻など、アプリを開かなくてもロック画面や待機中のアイランドから最新情報を確認できます。

Rork Maxなら、SwiftのActivityKitを直接書く必要なく、**プロンプトを入力するだけで**これらの高度なネイティブ機能を実装できます。この記事では、フードデリバリーアプリを例に、Live ActivitiesとDynamic Islandを実装する手順をステップごとに解説します。

この記事で学べること：

- Live Activities と Dynamic Island の基本的な仕組み
- Rork Max で ActivityKit を活用するためのプロンプト設計
- ロック画面・Dynamic Island それぞれのUI実装
- よくあるエラーとトラブルシューティング

**対象読者**: Rork Maxを使ったことがある方、iOSアプリ開発に興味がある個人開発者・小規模チーム

---

## Live Activities と Dynamic Island とは

**Live Activities**はiOS 16.1で導入されたAppleの機能で、ロック画面や通知エリアに最大8時間リアルタイム情報を表示できます。ActivityKitフレームワークを使い、SwiftUIでUIを定義します。

**Dynamic Island**はiPhone 14 Pro以降搭載のUI領域で、True Depth Cameraシステムを囲む黒い「島」がインタラクティブなUIとして機能します。3つの表示モードがあります：

| モード | 説明 | サイズ感 |
|--------|------|----------|
| コンパクト | 他のアプリがフォアグラウンド時、小さく表示 | 約120×36pt |
| ミニマル | 2つのアクティビティが同時実行中 | 非常に小さい |
| 拡張 | ユーザーが長押し時、情報を詳しく表示 | 最大160pt高さ |

Live ActivitiesとDynamic Islandは密接に連携しており、**同じActivityAttributes**を共有しながら、それぞれ異なるUIを持ちます。

## ステップバイステップ：フードデリバリーアプリで実装

今回は「注文してから届くまでの配送状況」をLive ActivitiesとDynamic Islandで表示するアプリを作ります。

### Step 1: Rork Maxでプロジェクトを作成

Rork Max（[rork.com/max](https://rork.com/max)）にアクセスし、新規プロジェクトを作成します。最初のプロンプトで全体像を伝えましょう：

```text
フードデリバリーアプリを作ってください。
注文後、配送状況（準備中→ピックアップ→配達中→完了）をLive Activitiesで
ロック画面とDynamic Islandに表示する機能を含めてください。
配達員の現在地と到着予定時刻も表示します。
```

Rork Maxは次のような構造を自動生成します：

- `DeliveryActivityAttributes.swift` — ActivityKit用のデータ定義
- `DeliveryLiveActivityView.swift` — ロック画面UI
- `DeliveryDynamicIslandView.swift` — Dynamic Island UI
- `OrderViewModel.swift` — 状態管理とアクティビティ制御

### Step 2: ActivityAttributes の確認・カスタマイズ

Rork Maxが生成した `DeliveryActivityAttributes.swift` の内容を確認します：

```swift
import ActivityKit
import Foundation

// 配送アクティビティの属性定義
struct DeliveryActivityAttributes: ActivityAttributes {
    // 更新可能なリアルタイム状態
    public struct ContentState: Codable, Hashable {
        var status: DeliveryStatus     // 配送ステータス
        var driverName: String          // 配達員の名前
        var estimatedArrival: Date      // 到着予定時刻
        var currentStep: Int            // 現在のステップ（0-3）
    }

    var restaurantName: String          // レストラン名（静的）
    var orderId: String                 // 注文ID（静的）
}

enum DeliveryStatus: String, Codable {
    case preparing  = "準備中"
    case pickedUp   = "ピックアップ完了"
    case delivering = "配達中"
    case delivered  = "配達完了"

    // SF Symbols アイコン名
    var iconName: String {
        switch self {
        case .preparing:  return "fork.knife"
        case .pickedUp:   return "bag.fill"
        case .delivering: return "bicycle"
        case .delivered:  return "checkmark.circle.fill"
        }
    }
}
```

**期待する出力**: `DeliveryActivityAttributes`がActivityKitに準拠したSwift structとして生成され、Xcodeビルドエラーなしでコンパイルされます。

### Step 3: Dynamic Island UIを追加プロンプトで改良

デフォルト生成のDynamic Island UIをより魅力的にするには、追加プロンプトを使います：

```text
Dynamic Islandのコンパクトモードを改善してください：
- リーディング側：配達員アイコン（SF Symbols "bicycle"）をアニメーション付きで
- トレーリング側：到着予定時間（分）を大きなフォントで
- 拡張モード：ステータスバー（4ステップ）と到着予定タイマーを表示
```

生成されるDynamic Island UIの例：

```swift
DynamicIsland {
    // 拡張時のUI
    DynamicIslandExpandedRegion(.leading) {
        Image(systemName: context.state.status.iconName)
            .font(.title2)
            .foregroundStyle(.orange)
    }
    DynamicIslandExpandedRegion(.trailing) {
        VStack(alignment: .trailing, spacing: 2) {
            Text(context.state.driverName)
                .font(.caption2)
                .foregroundStyle(.secondary)
            Text(context.state.estimatedArrival, style: .timer)
                .font(.caption)
                .monospacedDigit()
        }
    }
    DynamicIslandExpandedRegion(.bottom) {
        DeliveryProgressBar(currentStep: context.state.currentStep)
            .padding(.horizontal, 12)
    }
} compactLeading: {
    // コンパクト左側：配達アイコン（アニメーション付き）
    Image(systemName: context.state.status.iconName)
        .foregroundStyle(.orange)
        .symbolEffect(.variableColor.iterative) // iOS 17以上
} compactTrailing: {
    // コンパクト右側：残り時間カウントダウン
    Text(context.state.estimatedArrival, style: .timer)
        .foregroundStyle(.primary)
        .monospacedDigit()
        .frame(maxWidth: 44)
} minimal: {
    // ミニマル表示（2つのアクティビティが同時実行時）
    Image(systemName: context.state.status.iconName)
        .foregroundStyle(.orange)
}
```

### Step 4: Live Activity を開始・更新・終了する

注文確定時にLive Activityを開始するコードもRork Maxが自動生成します：

```swift
// OrderViewModel.swift — ライフサイクル管理

// 注文確定時に呼び出す関数
func startDeliveryTracking(order: Order) async {
    // Live Activitiesが許可されているか確認
    guard ActivityAuthorizationInfo().areActivitiesEnabled else {
        print("⚠️ Live Activities が許可されていません")
        return
    }

    let attributes = DeliveryActivityAttributes(
        restaurantName: order.restaurantName,
        orderId: order.id
    )
    let initialState = DeliveryActivityAttributes.ContentState(
        status: .preparing,
        driverName: "配達員を探しています",
        estimatedArrival: Date().addingTimeInterval(30 * 60), // 30分後
        currentStep: 0
    )

    do {
        let activity = try Activity<DeliveryActivityAttributes>.request(
            attributes: attributes,
            content: .init(state: initialState, staleDate: nil)
        )
        self.deliveryActivity = activity
        print("✅ Live Activity開始: \(activity.id)")
    } catch {
        print("❌ Live Activity開始エラー: \(error.localizedDescription)")
    }
}

// 配送ステータス更新時（バックエンドからの通知を受信後に呼び出す）
func updateDeliveryStatus(status: DeliveryStatus, eta: Date, driver: String) async {
    let updatedState = DeliveryActivityAttributes.ContentState(
        status: status,
        driverName: driver,
        estimatedArrival: eta,
        currentStep: status == .preparing ? 0 :
                     status == .pickedUp ? 1 :
                     status == .delivering ? 2 : 3
    )
    await deliveryActivity?.update(
        ActivityContent(state: updatedState, staleDate: nil)
    )
}

// 配達完了時
func endTracking() async {
    await deliveryActivity?.end(nil, dismissalPolicy: .after(Date().addingTimeInterval(5)))
}
```

## よくあるエラーと対処法

**エラー1: `ActivityAuthorizationInfo().areActivitiesEnabled` が `false` を返す**

Info.plistに `NSSupportsLiveActivities` キーを追加し、`YES` に設定する必要があります。Rork Maxに「Info.plistにNSSupportsLiveActivitiesをtrueで追加して」とプロンプトするだけでOKです。

**エラー2: Dynamic Island が表示されない**

シミュレータはiPhone 14 Pro以降を選択してください。実機テストの場合はiOS 16.1以上が必要です。

**エラー3: `Activity.request()` でクラッシュ**

ActivityAttributesの `ContentState` がCodableに完全準拠していない可能性があります。enumにRawValueがない場合クラッシュします。Rork Maxに「ContentStateのCodable準拠を確認・修正して」と指示してください。

**エラー4: タイマーが `0:00` のまま表示される**

`Text(date, style: .timer)` は渡したDateが未来である必要があります。過去の日時を渡すと `0:00` になります。ETAの計算で現在時刻より未来の値が設定されているか確認してください。

## 応用・発展的な活用法

Live ActivitiesはPush Notificationsと組み合わせることで、**サーバーから直接更新**できます。Rork Maxでは「APNs経由でLive Activitiesを更新する機能を追加して」とプロンプトするだけで、プッシュトークンの取得・APNsペイロードのフォーマット例が生成されます。

他の活用例：

- **スポーツアプリ**: 試合のスコアとタイマーをリアルタイム表示
- **フィットネスアプリ**: 現在のランニングペースと残り距離
- **タイマーアプリ**: ポモドーロタイマーの残り時間

ウィジェットとの違いや連携については[Rork Max でホーム画面＆ロック画面ウィジェットを作る完全ガイド](/articles/rork-dev/rork-max-widget-guide)も参照してください。また、Rork MaxでApple全プラットフォーム向けに展開する方法は[Rork Max でApple全プラットフォーム対応アプリを開発する方法](/articles/rork-dev/rork-max-apple-multiplatform)で詳しく解説しています。

## FAQ

**Q. Live ActivitiesはAndroidでも動きますか？**

A. いいえ、Live ActivitiesはApple独自の機能でiOS専用です。AndroidではRork（通常版）のReact Native製アプリでForeground Serviceを使った通知バーUIが代替手段になります。

**Q. Live Activitiesの最大表示時間はどれくらいですか？**

A. 最大8時間です（ユーザーが手動で終了しない場合）。配送完了など特定イベントでアクティビティを終了させることが重要です。コードでは `activity.end()` を呼び出します。放置すると古い情報がロック画面に残り続けてしまいます。

**Q. 無料プランのRork Maxでも実装できますか？**

A. 週5プロンプトの無料枠でも実装自体は可能ですが、Dynamic Islandの3モードすべてを反復改善するにはプロプランが実用的です。プロンプトを効率化するコツはRork Max プロンプトエンジニアリングを参照してください。
