### Gemini API をアプリに組み込む：実装コード例

Rork のバックエンド（Node.js / Python）で、Gemini API を呼び出すコード例：

```javascript
// Rork Backend (Node.js / Firebase Cloud Functions)

const axios = require('axios');

async function getAiRecommendation(userId, dailyIntake, goal) {
  // 1. ユーザーの過去データをデータベースから取得
  const userData = await db.collection('users').doc(userId).get();

  // 2. Gemini API へのリクエスト構築
  const prompt = `
ユーザーは以下の情報を持ちます:
- 1日の栄養摂取: ${dailyIntake.calories} kcal, タンパク質 ${dailyIntake.protein}g
- 目標: ${goal === 'weight_loss' ? '体重減少' : goal === 'muscle_gain' ? '筋力増加' : '維持'}
- 過去7日の平均活動量: ${userData.avgSteps} 歩

上記に基づいて、次の食事での推奨を提案してください。
出力形式:
{
  "recommendation": "提案テキスト（1-2文）",
  "suggested_foods": ["食べ物1", "食べ物2", "食べ物3"],
  "estimated_calories": 400,
  "estimated_protein": 20
}
  `;

  // 3. Google Cloud 認証（サービスアカウント）
  const authResponse = await axios.post(
    'https://oauth2.googleapis.com/token',
    {
      client_email: process.env.GCP_SERVICE_ACCOUNT_EMAIL,
      private_key: process.env.GCP_PRIVATE_KEY,
      type: 'service_account'
    }
  );

  // 4. Gemini API へのリクエスト（プロトタイプ実装）
  // 注: Gemini API の実装フローは2026年3月時点で変動の可能性あり
  const geminiResponse = await axios.post(
    'https://generativelanguage.googleapis.com/v1/models/gemini-3.1-pro:generateContent',
    {
      contents: [{
        parts: [{
          text: prompt
        }]
      }]
    },
    {
      headers: {
        'Authorization': `Bearer ${authResponse.data.access_token}`,
        'Content-Type': 'application/json'
      }
    }
  );

  // 5. レスポンスを JSON パース
  const content = geminiResponse.data.candidates[0].content.parts[0].text;
  const recommendation = JSON.parse(content);

  // 6. 推奨情報をデータベースに保存（監査ログ・ユーザー履歴）
  await db.collection('recommendations').add({
    user_id: userId,
    created_at: new Date(),
    recommendation_text: recommendation.recommendation,
    suggested_foods: recommendation.suggested_foods,
    estimated_calories: recommendation.estimated_calories
  });

  return recommendation;
}

// Rork Backend のエンドポイント
exports.getRecommendation = async (req, res) => {
  const { user_id, daily_intake, goal } = req.body;

  try {
    const recommendation = await getAiRecommendation(user_id, daily_intake, goal);
    res.json({ success: true, data: recommendation });
  } catch (error) {
    console.error('Gemini API Error:', error);
    res.status(500).json({ success: false, error: error.message });
  }
};
```

### Google Fit / Apple Health の連携実装

```javascript
// Google Fit からの健康データ取得

async function fetchGoogleFitData(userId, accessToken) {
  // Google Fit REST API
  const response = await axios.post(
    'https://www.googleapis.com/fitness/v1/users/me/dataset:aggregate',
    {
      aggregateBy: [
        {
          dataTypeName: 'com.google.step_count.delta',
          dataSourceId: 'derived:com.google.step_count.delta:com.google.android.gms:estimated_steps'
        },
        {
          dataTypeName: 'com.google.calories.expended',
          dataSourceId: 'derived:com.google.calories.expended:com.google.android.gms:from_activities'
        }
      ],
      bucketByTime: { durationMillis: 86400000 }, // 1日単位
      startTimeMillis: Date.now() - (7 * 24 * 60 * 60 * 1000), // 過去7日
      endTimeMillis: Date.now()
    },
    {
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    }
  );

  // データを整形
  const dailyData = response.data.bucket.map(bucket => ({
    date: new Date(parseInt(bucket.startTimeMillis)),
    steps: bucket.dataset[0].point[0]?.value[0]?.intVal || 0,
    caloriesBurned: bucket.dataset[1].point[0]?.value[0]?.fpVal || 0
  }));

  return dailyData;
}
```

## FAQ

### Q. Google AI Pro と Ultra、まずどちらを選ぶべきですか？

A. **初期開発なら Pro で開始**をお勧めします。月額 $19.99 で十分な機能があります。以下の場合に Ultra へアップグレード検討:
- Gemini API の推論がボトルネックになってきた（Deep Think 検討）
- 自動テスト・レビューの負荷が増加（Jules 20x higher limits）
- ASO 監視の完全自動化が必要（Project Mariner）

### Q. Gemini API の使用量に上限はありますか？

A. Google AI Pro のクレジット：月額 1,000 AI クレジット付与。Gemini API のテキスト生成は約 1クレジット = 1,000トークン。アプリの推奨生成（平均 200トークン）なら、月 1,000 クレジットで約 5,000回分。月 100,000ユーザーアプリでも対応可能です。

### Q. Rork と他のノーコード（Flutter Flow, Flutterflow など）との比較は？

A. Rork の特徴：
- **Gemini API 統合**: ネイティブに Google AI と連携
- **Veo 統合**: 動画生成がアプリ内から可能
- **コスト**: 他より安価（月 $50～）
- **パフォーマンス**: ネイティブコンパイルで高速

欠点: エコシステムが Flutter Flow より小規模。複雑なカスタマイズは手コーディング必要。

### Q. Apple App Store と Google Play Store の両方で公開する場合、コストは倍になりますか？

A. いいえ。Google AI Pro は両プラットフォーム共用です。Rork も iOS / Android 両対応アプリを同一プロジェクトで管理。追加コストなく両ストアで公開可能。

### Q. Google Fit / Apple Health と連携する場合、ユーザー承認は毎回必要ですか？

A. 初回のみです。ユーザーが一度「Google Fit へのアクセス許可」「Apple Health へのアクセス許可」を与えると、その後はアプリが自動で定期同期します。ただし、トークンの有効期限（通常 1年）後は再認可が必要。

### Q. NotebookLM で 500個のノートブック管理は実際可能ですか？

A. Google AI Pro では最大 500個作成可能ですが、実運用では 50～100個程度がベストです。500個超の管理が必要なら、Google Cloud Firestore と Notion で独立的に資料を管理し、必要時に NotebookLM ノートブックを再作成する方法が推奨されます。

TypeScriptでの堅牢な開発を体系的に学びたい方には、「[プロを目指す人のためのTypeScript入門](https://www.amazon.co.jp/dp/4297127474/?tag=pinocchio-22)」（鈴木僚太 著）が大いに役立つでしょう。
