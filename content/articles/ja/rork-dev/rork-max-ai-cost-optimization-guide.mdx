## はじめに

Rork MaxでAI機能を充実させると、月々のAPI費用が予想以上に膨らむことがあります。特に、チャットボット・AIアシスタント・コンテンツ生成機能を実装したアプリでは、ユーザー数の増加とともにコストが急増するケースが少なくありません。この記事では、品質を落とさずにAIコストを50〜70%削減するための実践的な手法を解説します。

アプリのパフォーマンス全般については[Rorkアプリ高度なパフォーマンス最適化ガイド](/articles/rork-dev/advanced-rork-app-performance-optimization)もあわせてご覧ください。

## Rork MaxのAI利用コスト構造

Rork Maxのコストは主に以下の要素で構成されています。

- **入力トークン費用**: プロンプト・コンテキストとして送信するテキストの量
- **出力トークン費用**: AIが生成するレスポンスの量（入力より高い場合が多い）
- **キャッシュ読み取り費用**: キャッシュされたプロンプトを再利用する際のコスト（通常の10分の1程度）
- **モデルコスト差**: Claude Opus 4.6はHaiku 4.5の約20倍のコスト

Rork Maxプランには月次クレジット上限があり、超過すると従量課金が発生します。詳細は[Rork Max AI機能ガイド](/articles/rork-ai/rork-max-ai-features)でご確認ください。

## 手法1: システムプロンプトの最適化

システムプロンプトは全リクエストで送信されるため、ここを圧縮するだけで大きな節約になります。

**Before（冗長な例）**:
```
あなたは親切で丁寧なAIアシスタントです。ユーザーの質問に対して、
できる限り詳しく、わかりやすく、丁寧に回答してください。
また、ユーザーが困っている場合は、追加の情報を提供して、
最善のサポートをしてください。常に礼儀正しく、プロフェッショナルに
振る舞い、ユーザーの満足度を最大化してください。
```

**After（最適化例）**:
```
親切なAIアシスタント。簡潔・正確・役立つ回答を提供。
```

この変更だけで入力トークンを80%以上削減できる場合があります。

## 手法2: コンテキストウィンドウの管理

長い会話履歴を全て送り続けると、コストが雪だるま式に増加します。

```javascript
// 会話履歴を直近N件に制限するユーティリティ
function trimConversationHistory(messages, maxMessages = 10) {
  // システムメッセージは常に保持
  const systemMessages = messages.filter(m => m.role === 'system');
  const conversationMessages = messages.filter(m => m.role !== 'system');

  // 直近maxMessages件のみ保持
  const trimmedConversation = conversationMessages.slice(-maxMessages);

  return [...systemMessages, ...trimmedConversation];
}

// 使用例
const optimizedMessages = trimConversationHistory(conversationHistory, 6);
// 期待する効果: 会話が長くなっても常に一定のコスト
```

## 手法3: プロンプトキャッシュの活用

Claude APIのプロンプトキャッシュ機能を使うと、繰り返し送信するコンテンツのコストを約90%削減できます。

```javascript
// Rork MaxでプロンプトキャッシュAPIを活用する例
const response = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': process.env.ANTHROPIC_API_KEY,
    'anthropic-version': '2023-06-01',
    'anthropic-beta': 'prompt-caching-2024-07-31'
  },
  body: JSON.stringify({
    model: 'claude-haiku-4-5-20251001',
    max_tokens: 1024,
    system: [
      {
        type: 'text',
        text: '長い製品マニュアルや知識ベースのテキストをここに配置',
        // cache_controlを付けるとこのブロックがキャッシュされる
        cache_control: { type: 'ephemeral' }
      }
    ],
    messages: [
      {
        role: 'user',
        content: 'この製品の使い方を教えてください'
      }
    ]
  })
});

// キャッシュヒット時: cache_read_input_tokensが通常の10%のコスト
// 期待するレスポンスヘッダーの一部:
// cache_creation_input_tokens: 1250  (初回のみ発生)
// cache_read_input_tokens: 1250      (2回目以降)
```

## 手法4: タスク別モデル選択

全てのリクエストに最高性能モデルを使う必要はありません。

タスク別の推奨モデル:

- **シンプルなQ&A・分類**: Claude Haiku 4.5（最安値・高速）
- **一般的な文章生成・要約**: Claude Sonnet 4.6（コストと品質のバランス）
- **複雑な推論・コード生成**: Claude Opus 4.6（最高品質、高コスト）

```javascript
// タスクの複雑度に応じてモデルを自動選択
function selectModel(taskComplexity) {
  if (taskComplexity === 'simple') {
    return 'claude-haiku-4-5-20251001';   // 最安値
  } else if (taskComplexity === 'medium') {
    return 'claude-sonnet-4-6';           // バランス型
  } else {
    return 'claude-opus-4-6';             // 最高品質
  }
}

// 使用例: 単純なカテゴリ分類にはHaikuを使用
const model = selectModel('simple');
console.log(model); // claude-haiku-4-5-20251001
```

## 手法5: 出力長の制限と構造化

不必要に長い回答を防ぐことでコストを削減できます。

```javascript
// max_tokensで出力を制限し、構造化レスポンスを要求
const prompt = `以下のレビューをポジティブ・ネガティブ・ニュートラルに分類してください。
JSONのみで回答: {"sentiment": "positive|negative|neutral", "score": 0.0-1.0}

レビュー: ${userReview}`;

const response = await callClaude({
  model: 'claude-haiku-4-5-20251001',
  max_tokens: 50,  // 短い構造化レスポンスには小さな値で十分
  messages: [{ role: 'user', content: prompt }]
});

// 期待する出力:
// {"sentiment": "positive", "score": 0.85}
```

## コスト監視とアラートの設定

定期的にコストをモニタリングする仕組みを作りましょう。Rork MaxのAPI使用量はAnthropicのダッシュボード（[console.anthropic.com](https://console.anthropic.com)）で確認できます。

目安のコスト閾値設定:
- **日次**: 想定月次コスト ÷ 30 × 1.5 でアラート
- **週次**: 想定月次コスト ÷ 4 × 1.3 でレビュー

## よくある質問

**Q: Rork Maxのクレジットが足りなくなったらどうなりますか？**

A: Rork Maxのプランクレジットを超過すると、従量課金に切り替わります。超過を防ぐには、月初に使用量の上限を設定し、Anthropicのダッシュボードでアラートを有効にしておくことをおすすめします。

**Q: プロンプトキャッシュはどのモデルで使えますか？**

A: Claudeの全モデル（Opus・Sonnet・Haiku各シリーズ）でプロンプトキャッシュが利用できます。ただし、キャッシュは5分間有効のため、頻繁に呼び出すユースケースで効果が大きくなります。

**Q: AIの品質を落とさずにコストを下げる最善策は？**

A: 最も効果的なのはプロンプトキャッシュ + タスク別モデル選択の組み合わせです。長い知識ベースをキャッシュしながら、シンプルなタスクにHaikuを使うことで、品質を維持しつつコストを50〜70%削減できます。
