## アプリのクラッシュはリリース後に初めてわかる

アプリをリリースした後、最も怖いのは「ユーザーが黙って離れていくこと」です。クラッシュや予期しないエラーが発生しても、多くのユーザーは報告せずにアンインストールしてしまいます。App Store のレビューで初めて「頻繁にフリーズする」と書かれて気づく、という経験をした個人開発者も少なくありません。

Sentry はこの問題を解決するオープンソースのエラー監視プラットフォームです。React Native（Rork が内部で使用するフレームワーク）に対応しており、クラッシュが発生した瞬間にスタックトレース・デバイス情報・再現手順を自動で収集してくれます。この記事では、Rork アプリに Sentry を導入して本番品質の監視体制を構築する方法を、ステップバイステップで解説します。

## Sentry とは何か、なぜ Rork 開発に必要か

Sentry は 2012 年にサンフランシスコで創業されたエラー監視サービスで、2026 年現在、世界 400 万以上の開発者が利用しています。個人プランは無料から利用でき、月間 5,000 件のエラーイベントまで無償で使えます。

Rork で開発したアプリに Sentry が必要な理由は主に3つあります。

**1. ローカル環境では再現できないクラッシュを検知できる**
iOS/Android の実機では OS バージョン・メモリ使用量・ネットワーク状況が開発環境と大きく異なります。Sentry は本番環境で発生したクラッシュのみを正確に捉えます。

**2. エラーの影響範囲を即座に把握できる**
「このエラーは全ユーザーの何%に発生しているか」「どのデバイスで多いか」がダッシュボードで一目瞭然です。対処の優先度を正しく判断できます。

**3. リリースごとの品質変化を追跡できる**
新バージョンをリリースした際に、エラー率が上がっていないかをリリース比較機能で確認できます。回帰バグの早期発見に役立ちます。

## 事前準備 — Sentry アカウントとプロジェクトの作成

まず [sentry.io](https://sentry.io) にアクセスしてアカウントを作成します。GitHub アカウントでのログインが便利です。

1. ダッシュボードで「新しいプロジェクト」を作成
2. プラットフォームとして「React Native」を選択
3. プロジェクト名を入力（例: `my-rork-app`）
4. 「プロジェクトを作成」をクリック

プロジェクトを作成すると、**DSN（Data Source Name）**と呼ばれる接続文字列が表示されます。この値は後の設定で使用するので控えておきましょう。形式は以下のようになります：

```
https://xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx@o123456.ingest.sentry.io/7654321
```

## Rork アプリへの Sentry SDK 導入

### パッケージのインストール

Rork の AI チャットで以下のメッセージを入力し、必要なパッケージをインストールします：

```
@sentry/react-native パッケージを追加して、アプリ全体でエラーを自動キャプチャする設定をしてください。
DSN は後で設定するので、環境変数から読み込む形にしてください。
```

Rork が自動で `package.json` と必要なファイルを更新します。手動で進める場合は以下のコマンドを実行します：

```bash
# Rork のターミナルまたはローカル環境で実行
npx expo install @sentry/react-native
```

### app.json への Expo プラグイン追加

Sentry の自動計装（ソースマップのアップロードなど）を有効にするため、`app.json` に設定を追加します：

```json
{
  "expo": {
    "plugins": [
      [
        "@sentry/react-native/expo",
        {
          "url": "https://sentry.io/",
          "organization": "your-org-slug",
          "project": "your-project-slug"
        }
      ]
    ]
  }
}
```

`organization` と `project` のスラッグは Sentry ダッシュボードの URL から確認できます（`https://sentry.io/organizations/{org}/projects/{project}/`）。

### Sentry の初期化

アプリのエントリーポイント（通常は `App.tsx` または `_layout.tsx`）に Sentry の初期化コードを追加します：

```typescript
import * as Sentry from "@sentry/react-native";

// アプリ起動時に1回だけ実行
Sentry.init({
  dsn: process.env.EXPO_PUBLIC_SENTRY_DSN, // 環境変数から DSN を読み込む

  // 本番環境のみエラーを送信
  enabled: process.env.NODE_ENV === "production",

  // パフォーマンス監視（トランザクションのサンプリングレート）
  tracesSampleRate: 0.2,

  // セッションリプレイ（オプション）
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,

  // リリースバージョンを自動で追跡
  // Expo の場合は自動設定されるためコメントアウト可能
  // release: "my-rork-app@1.0.0",
});

// アプリのルートコンポーネントを Sentry でラップ
export default Sentry.wrap(App);
```

`.env` ファイル（または Rork の環境変数設定）に DSN を追加します：

```bash
# .env.production
EXPO_PUBLIC_SENTRY_DSN=https://your-dsn@sentry.io/your-project-id
```

**⚠️ DSN をコードに直接記述しないでください。** `.env` ファイルを使い、`.gitignore` に追加することで GitHub に公開されないようにします。

### 手動でエラーをキャプチャする

自動キャプチャに加えて、特定の処理でエラーを手動でレポートすることもできます：

```typescript
import * as Sentry from "@sentry/react-native";

async function fetchUserData(userId: string) {
  try {
    const response = await fetch(`https://api.example.com/users/${userId}`);
    if (!response.ok) {
      // HTTP エラーを Sentry に送信
      Sentry.captureException(new Error(`API Error: ${response.status}`), {
        tags: {
          screen: "UserProfile",
          action: "fetchUserData",
        },
        extra: {
          userId,
          statusCode: response.status,
        },
      });
      return null;
    }
    return await response.json();
  } catch (error) {
    // ネットワークエラーなど
    Sentry.captureException(error);
    throw error;
  }
}
```

### ユーザー情報の付加

誰のデバイスでエラーが発生したかを追跡することで、再現性の高いバグ報告が可能になります：

```typescript
// ログイン後に実行
Sentry.setUser({
  id: user.id,
  email: user.email,
  // 個人情報を含む場合はプライバシーポリシーに明記すること
});

// ログアウト時にクリア
Sentry.setUser(null);
```

## アラートとダッシュボードの設定

SDK の導入が完了したら、Sentry ダッシュボードで通知ルールを設定します。

### Slack への通知設定

Sentry ダッシュボードの「Alerts」→「Create Alert」から設定します。個人開発の場合は以下のルールがおすすめです：

- **条件**: 新しいエラーが初めて発生した場合
- **アクション**: メールまたは Slack に通知
- **フィルター**: エラー数が1時間で10件以上（スパムを防ぐため）

クラッシュが発生した際に LINE や Slack に即座に通知が来るように設定しておくと、ユーザーが気づく前に修正を開始できます。

### パフォーマンスモニタリング

Sentry はエラー監視だけでなく、アプリのパフォーマンスも監視できます。特に API レスポンス時間やスクリーン遷移の速度を計測する「トランザクション」機能が便利です：

```typescript
// API 呼び出しのパフォーマンスを計測
const transaction = Sentry.startTransaction({
  name: "fetch-articles",
  op: "http.client",
});

try {
  const data = await fetchArticles();
  transaction.setStatus("ok");
} catch (error) {
  transaction.setStatus("internal_error");
  Sentry.captureException(error);
} finally {
  transaction.finish(); // 計測終了・データを送信
}
```

## よくある質問（FAQ）

**Q1. 無料プランでどこまで使えますか？**

Sentry の無料プランは月間 5,000 エラーイベントまで、データ保持期間は 30 日間です。個人開発アプリの初期段階では十分な量です。月間アクティブユーザーが1万人を超えてきたら有料プランへの移行を検討しましょう。

**Q2. ユーザーの個人情報は Sentry に送信されますか？**

デフォルトでは IPアドレスが含まれますが、`sendDefaultPii: false`（デフォルト値）で無効化できます。`Sentry.setUser()` で追加した情報は意図的に送信したものです。プライバシーポリシーにエラー監視ツールの利用を明記することを推奨します。

**Q3. Expo Go でも動作しますか？**

`@sentry/react-native` は Expo Go では一部機能が制限されます。特にネイティブクラッシュの検知には Expo Dev Client（EAS Build）が必要です。Rork Max を使用している場合はネイティブビルドが可能なため、フル機能を利用できます。

**Q4. Firebase Crashlytics との違いは何ですか？**

Firebase Crashlytics はクラッシュ専用で、Google のエコシステムとの連携が強みです。Sentry はクラッシュ以外のエラー（API エラー・カスタムイベント）も監視でき、パフォーマンス監視・リリース追跡・アラート設定の自由度が高い点が特徴です。どちらを使うかはチームの好みによりますが、Sentry はフロントエンド・バックエンドを横断して使えるため、フルスタック開発には特に便利です。

## まとめ — クラッシュ監視は「リリースの当たり前」

Sentry の導入は一度設定すれば、その後は自動でクラッシュを捕捉し続けてくれます。アプリの品質を維持するために最も費用対効果の高い投資のひとつです。

設定の流れをまとめると：

- Sentry アカウントを作成して React Native プロジェクトを作成する
- `@sentry/react-native` を Rork アプリに追加する
- `Sentry.init()` で DSN を設定し、ルートコンポーネントをラップする
- アラートルールを設定して通知を受け取れるようにする
- リリースごとにエラー率を確認して品質を維持する
