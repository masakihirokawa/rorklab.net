## はじめに — アプリ内レビュー促進がなぜ重要なのか

アプリのダウンロード数を増やすうえで、**App StoreおよびGoogle Playでの評価スコア**は非常に大きな影響を持っています。評価が4.0以上のアプリは検索結果の上位に表示されやすく、ユーザーがダウンロードを決める際の重要な判断材料にもなります。

しかし、多くのユーザーは自発的にレビューを書いてくれません。実際にレビューを投稿するのは、全体のわずか数%程度と言われています。そこで重要なのが「**アプリ内レビュー促進（In-App Review）**」の仕組みです。

アプリ内レビュー促進とは、ユーザーがアプリを使っているときに、OS標準のレビューダイアログを表示して評価や感想の入力を促す機能です。App Storeを開く手間が省けるため、レビューの投稿率が大きく向上します。

この記事では、Rorkで開発したiOS・Androidアプリにアプリ内レビュー促進を実装する方法を、コード例を交えてわかりやすく解説します。

## iOS向け実装 — SKStoreReviewController

### 仕組みを理解する

iOSでは、**SKStoreReviewController**（StoreKit）を通じてApp Storeのレビューダイアログを表示できます。ただし、以下のOSレベルの制限があります。

- 同一バージョンで表示できるのは**最大3回/年**
- OSが表示タイミングを最終判断する（呼び出しても必ず表示されるわけではない）
- シミュレーターや開発ビルドでは常に表示される（テスト向け）

Expoを使うRorkアプリでは、`expo-store-review`パッケージを利用するのが最もシンプルです。

### インストールと基本実装

```bash
# expo-store-reviewをインストール
npx expo install expo-store-review
```

```typescript
// src/hooks/useReviewPrompt.ts
import * as StoreReview from 'expo-store-review';
import AsyncStorage from '@react-native-async-storage/async-storage';

const REVIEW_PROMPTED_KEY = 'review_prompted_count';
const LAUNCH_COUNT_KEY = 'app_launch_count';
const LAST_PROMPT_DATE_KEY = 'last_review_prompt_date';

/**
 * レビュー促進の条件チェックと実行フック
 */
export function useReviewPrompt() {
  const requestReview = async () => {
    try {
      // StoreReviewが利用可能か確認
      const isAvailable = await StoreReview.isAvailableAsync();
      if (!isAvailable) return;

      // ダイアログを表示リクエスト（OSが最終判断）
      await StoreReview.requestReview();

      // 表示日時を記録
      await AsyncStorage.setItem(
        LAST_PROMPT_DATE_KEY,
        new Date().toISOString()
      );
    } catch (error) {
      console.error('レビュー促進エラー:', error);
    }
  };

  return { requestReview };
}
```

### 適切なタイミングで表示する条件ロジック

```typescript
// src/utils/reviewTrigger.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const MIN_LAUNCH_COUNT = 5;         // 最低5回起動後
const MIN_DAYS_SINCE_INSTALL = 3;   // インストール3日後
const MIN_DAYS_BETWEEN_PROMPTS = 60; // 前回表示から60日以上

/**
 * レビュー促進を表示すべきか判定する
 * @returns true = 表示してよい / false = スキップ
 */
export async function shouldShowReviewPrompt(): Promise<boolean> {
  // 1. 起動回数チェック
  const launchCountStr = await AsyncStorage.getItem('app_launch_count');
  const launchCount = parseInt(launchCountStr || '0', 10);
  if (launchCount < MIN_LAUNCH_COUNT) return false;

  // 2. インストールからの経過日数チェック
  const installDateStr = await AsyncStorage.getItem('app_install_date');
  if (installDateStr) {
    const installDate = new Date(installDateStr);
    const daysSinceInstall =
      (Date.now() - installDate.getTime()) / (1000 * 60 * 60 * 24);
    if (daysSinceInstall < MIN_DAYS_SINCE_INSTALL) return false;
  }

  // 3. 前回プロンプトからの経過日数チェック
  const lastPromptStr = await AsyncStorage.getItem('last_review_prompt_date');
  if (lastPromptStr) {
    const lastPromptDate = new Date(lastPromptStr);
    const daysSinceLastPrompt =
      (Date.now() - lastPromptDate.getTime()) / (1000 * 60 * 60 * 24);
    if (daysSinceLastPrompt < MIN_DAYS_BETWEEN_PROMPTS) return false;
  }

  return true; // すべての条件を満たした
}
```

### 画面コンポーネントへの組み込み

```tsx
// src/screens/HomeScreen.tsx（一部抜粋）
import { useEffect } from 'react';
import { useReviewPrompt } from '../hooks/useReviewPrompt';
import { shouldShowReviewPrompt } from '../utils/reviewTrigger';
import AsyncStorage from '@react-native-async-storage/async-storage';

export function HomeScreen() {
  const { requestReview } = useReviewPrompt();

  useEffect(() => {
    // 起動回数をインクリメント
    incrementLaunchCount();
    // レビュー促進の条件チェック
    checkAndRequestReview();
  }, []);

  const incrementLaunchCount = async () => {
    const current = parseInt(
      (await AsyncStorage.getItem('app_launch_count')) || '0', 10
    );
    await AsyncStorage.setItem('app_launch_count', String(current + 1));
  };

  const checkAndRequestReview = async () => {
    const should = await shouldShowReviewPrompt();
    if (should) {
      // 少し遅延させてUXを自然に
      setTimeout(() => requestReview(), 2000);
    }
  };

  return (
    // ... 通常のUI
  );
}
```

## Android向け実装 — Google Play In-App Review API

### 仕組みと制限

AndroidのIn-App Review APIはiOSと同様、Google Playのレビューダイアログをアプリ内に表示します。制限として:

- **1ユーザーあたりの表示頻度に上限**あり（Googleが判断、具体的な回数は非公開）
- 実機のGoogle Playがインストールされていない環境（エミュレーター）では動作しない

`expo-store-review`はiOS・Androidともに対応しているため、**同じAPIで両プラットフォームに対応可能**です。

```typescript
// src/utils/reviewHelper.ts
import * as StoreReview from 'expo-store-review';
import { Platform } from 'react-native';

/**
 * プラットフォーム別のIn-App Reviewを実行
 * - iOS: SKStoreReviewController
 * - Android: Google Play In-App Review API
 */
export async function triggerInAppReview(): Promise<void> {
  try {
    const isAvailable = await StoreReview.isAvailableAsync();

    if (isAvailable) {
      // どちらのプラットフォームでも同じAPIを使用
      await StoreReview.requestReview();
      console.log(`[${Platform.OS}] レビュープロンプトをリクエストしました`);
    } else {
      // フォールバック: ストアページを直接開く
      await StoreReview.openStoreReviewPage();
      // 期待出力: ストアアプリが起動し、評価ページが表示される
    }
  } catch (e) {
    console.warn('レビュー促進をスキップしました:', e);
  }
}
```

## レビュー促進のベストプラクティス

### ✅ 効果的なタイミングの選び方

レビュー促進は、ユーザーが**ポジティブな体験をした直後**に表示することが重要です。タイミングが悪いと低評価につながることもあります。

- **良いタイミング**: タスク完了後・目標達成後・良い結果が出た後
- **悪いタイミング**: アプリ起動直後・エラー発生後・課金フロー中・チュートリアル中

```typescript
// 特定のポジティブアクションに紐づける例
async function onTaskCompleted(taskCount: number) {
  // 3つタスクを完了したタイミングでレビュー促進
  if (taskCount === 3 || taskCount === 10 || taskCount === 50) {
    const should = await shouldShowReviewPrompt();
    if (should) {
      await triggerInAppReview();
    }
  }
}
```

### ✅ ユーザーセグメントで絞り込む

すべてのユーザーに表示するより、**エンゲージメントの高いユーザー**に絞ることで、高評価のレビューが集まりやすくなります。

```typescript
// エンゲージメント判定の例
async function isEngagedUser(): Promise<boolean> {
  const sessionCount = await getSessionCount();
  const totalUsageMinutes = await getTotalUsageMinutes();
  const hasCompletedOnboarding = await AsyncStorage.getItem('onboarding_done');

  return (
    sessionCount >= 5 &&
    totalUsageMinutes >= 30 &&
    hasCompletedOnboarding === 'true'
  );
}
```

### ✅ カスタムフィードバック画面を前に置く

OS標準のダイアログを表示する前に、自前の**満足度確認画面**を挟む手法も効果的です。

```tsx
// 満足度確認→肯定的なら公式レビューへ誘導
function SatisfactionGate() {
  const [step, setStep] = useState<'ask' | 'review' | 'feedback'>('ask');

  if (step === 'ask') {
    return (

        <Button

          }}
        />
        <Button
          title="改善してほしい点があります"
          onPress={() => setStep('feedback')} // 内部フィードバックフォームへ
        />
      </View>
    );
  }
  // ... review・feedbackステップのUI
}
```

## よくあるエラーと対処法

### エラー1: 「isAvailableAsync()がfalseを返す」

**原因**: 開発ビルドではなく本番ビルドを実機にインストールしていない、またはAndroidエミュレーターを使用しています。

**対処**: TestFlightやGoogle Play Internal Testingで配布したビルドで検証します。

### エラー2: 「requestReview()を呼び出しても何も表示されない」

**原因**: OSの制限（年3回/iOSなど）や、すでに表示済みでクォータが消費されています。

**対処**: これは正常な動作です。開発・テスト時は`openStoreReviewPage()`でストアページへ直接飛ぶフォールバックも用意しておきましょう。

### エラー3: 「Android実機でクラッシュする」

**原因**: Google Play開発者サービスのバージョンが古い。

**対処**: アプリの`minSdkVersion`を確認し、最新のExpo SDKを使用します。`package.json`のexpo-store-reviewバージョンも確認すること。

## 効果測定と改善サイクル

実装後は効果を定量的に測定し、継続的に改善することが大切です。

```typescript
// レビュープロンプト表示イベントをFirebaseに送信する例
import analytics from '@react-native-firebase/analytics';

async function triggerReviewWithTracking() {
  // 表示イベントを記録
  await analytics().logEvent('review_prompt_shown', {
    platform: Platform.OS,
    app_version: Constants.expoConfig?.version,
    session_count: await getSessionCount(),
  });

  await triggerInAppReview();
}
```

測定すべき指標:
- **表示率**: プロンプトを表示できたセッションの割合
- **レビュー投稿数の変化**: 実装前後でApp Store Connect / Google Play Consoleのレビュー件数を比較
- **平均評価スコアの推移**: 実装後に評価が改善されているか確認

## FAQ

**Q: iOS・Androidで同じコードを使えますか？**

A: はい、`expo-store-review`を使えば一つのコードでiOS・Android両方に対応できます。内部的にはiOSでSKStoreReviewController、AndroidでGoogle Play In-App Review APIが呼び出されます。

**Q: テスト環境でレビューダイアログが表示されないのですが？**

A: iOSシミュレーターでは常に表示されますが、Androidエミュレーターでは表示されません。Androidのテストは、Google Play Internal Testingで配布した実機ビルドで行う必要があります。

**Q: ユーザーがレビューダイアログを閉じたか確認できますか？**

A: iOS・Androidともに、ユーザーがダイアログをどのように操作したか（閉じたか、評価したか）をアプリ側で検知することはプライバシー上の理由から**できません**。代わりに、プロンプト表示から数日後にApp Store Connectのレビュー数の変化を確認する方法が一般的です。

## まとめ

Rork製アプリにアプリ内レビュー促進を実装することで、App Store・Google Playの評価スコアを自然かつ効果的に向上させることができます。重要なポイントをまとめると:

- **`expo-store-review`** を使えばiOS・Android両対応がシンプルに実現できる
- OSレベルの**表示制限**があるため、ポジティブな体験の直後など適切なタイミングで呼び出すことが重要
- エンゲージメントの高いユーザーに絞ることで、高評価レビューが集まりやすくなる
- カスタム満足度確認を前に置くことで、低評価レビューを内部フィードバックに誘導できる
- 実装後は必ず効果を測定し、継続的に改善サイクルを回すこと
