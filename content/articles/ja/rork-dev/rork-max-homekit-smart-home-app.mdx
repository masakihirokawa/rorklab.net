スマートホーム市場は年々拡大を続けており、Apple の **HomeKit** は iOS ユーザーにとってもっとも身近なスマートホームプラットフォームです。照明の調光、エアコンの温度設定、ドアロックの施錠確認など、日常のあらゆるシーンを iPhone から制御できます。

しかし、HomeKit 対応アプリを一から開発するには Swift と Xcode の深い知識が必要でしました。**Rork Max** を使えば、プロンプトによる指示だけでネイティブ Swift コードを生成し、HomeKit API を統合した本格的なスマートホームアプリを短期間で構築できます。

この記事では、HomeKit の基本概念から実際のアプリ構築までをステップバイステップで解説します。

## HomeKit の基本アーキテクチャ

HomeKit は Apple が提供するスマートホームフレームワークで、以下の階層構造でデバイスを管理します。

**HMHome（ホーム）** はもっとも上位の単位で、物理的な住居に対応します。一人のユーザーが複数のホーム（自宅、別荘など）を持つことが可能です。その下に **HMRoom（部屋）** があり、リビング、寝室、キッチンなどを定義します。

各部屋にはスマートデバイスを表す **HMAccessory（アクセサリ）** が紐づきます。照明器具やサーモスタットがこれにあたります。アクセサリは **HMService（サービス）** を持ち、1 台のデバイスが複数の機能（照明の ON/OFF と明るさ調整など）を提供します。

さらにサービスには **HMCharacteristic（特性）** が含まれ、実際の値（明るさ 80%、温度 24℃ など）を読み書きするインターフェースとなります。

## Rork Max プロジェクトの初期設定

Rork Max で HomeKit アプリを始めるには、まずプロジェクトを作成します。

### プロンプト例

Rork Max のチャット画面で次のようなプロンプトを入力します。

```
HomeKit対応のスマートホームアプリを作ってください。
- ホーム内のデバイス一覧をリスト表示
- 照明のON/OFFと明るさ調整
- サーモスタットの温度設定
- デバイスのステータスをリアルタイムで監視
- ダッシュボード画面にルームごとのデバイス数を表示
```

Rork Max は Claude Opus 4.6 を活用して、この要件をネイティブ Swift + SwiftUI コードに変換します。生成されるプロジェクトには HomeKit フレームワークのインポートと基本的な構造が含まれます。

### Entitlements の設定

HomeKit を利用するには、Apple Developer Program のアカウントで HomeKit の Capability を有効にする必要があります。Rork Max の場合、クラウドコンパイル時に以下の Entitlements が自動的に追加されます。

```xml
<key>com.apple.developer.homekit</key>
<true/>
```

また、`Info.plist` には HomeKit へのアクセス理由を記述する必要があります。

```xml
<key>NSHomeKitUsageDescription</key>
<string>スマートホームデバイスの制御に HomeKit を使用します</string>
```

Rork Max はプロンプトから HomeKit を使う意図を検出し、これらの設定を自動的に含めたプロジェクトを生成します。

## デバイス一覧画面の実装

HomeKit アプリのコア機能は、ホーム内のアクセサリを一覧表示することです。

### HMHomeManager の初期化

HomeKit API の入口は `HMHomeManager` です。このオブジェクトを通じてユーザーのホーム情報にアクセスします。

```swift
import HomeKit

class HomeKitManager: NSObject, ObservableObject {
    private let homeManager = HMHomeManager()
    @Published var homes: [HMHome] = []
    @Published var primaryHome: HMHome?

    override init() {
        super.init()
        homeManager.delegate = self
    }
}

extension HomeKitManager: HMHomeManagerDelegate {
    func homeManagerDidUpdateHomes(_ manager: HMHomeManager) {
        DispatchQueue.main.async {
            self.homes = manager.homes
            self.primaryHome = manager.primaryHome
        }
    }
}
```

Rork Max でこのコードを生成する際は、プロンプトで「HMHomeManager を使って全ホームを取得し、SwiftUI の @Published で状態を管理して」と指示するだけで適切な構造が出力されます。

### アクセサリリストの SwiftUI ビュー

取得したアクセサリをリスト表示する画面を作ります。

```swift
struct AccessoryListView: View {
    @ObservedObject var homeKit: HomeKitManager

    var body: some View {
        NavigationView {
            List {
                ForEach(homeKit.primaryHome?.rooms ?? [], id: \.uniqueIdentifier) { room in
                    Section(header: Text(room.name)) {
                        ForEach(room.accessories, id: \.uniqueIdentifier) { accessory in
                            AccessoryRow(accessory: accessory)
                        }
                    }
                }
            }
            .navigationTitle("マイデバイス")
        }
    }
}
```

各ルームごとにセクションを分け、アクセサリ名とアイコンを表示します。Rork Max はデバイスの種類（照明、サーモスタット、ロックなど）を判別して適切な SF Symbols アイコンを自動で割り当てるコードを生成します。

## 照明コントロールの実装

スマートホームアプリでもっとも頻繁に使われる機能が照明の制御です。

### ON/OFF トグル

照明の ON/OFF は `HMCharacteristic` の `HMCharacteristicTypePowerState` を操作します。

```swift
func toggleLight(_ accessory: HMAccessory) {
    guard let lightService = accessory.services.first(where: {
        $0.serviceType == HMServiceTypeLightbulb
    }) else { return }

    guard let powerChar = lightService.characteristics.first(where: {
        $0.characteristicType == HMCharacteristicTypePowerState
    }) else { return }

    let newValue = !(powerChar.value as? Bool ?? false)
    powerChar.writeValue(newValue) { error in
        if let error = error {
            print("照明の切り替えに失敗: \(error.localizedDescription)")
        }
    }
}
```

### 明るさスライダー

明るさは `HMCharacteristicTypeBrightness` で 0〜100 の整数値として制御します。SwiftUI の `Slider` と組み合わせれば直感的な UI が作れます。

```swift
struct BrightnessSlider: View {
    let characteristic: HMCharacteristic
    @State private var brightness: Double = 50

    var body: some View {
        VStack {
            Text("明るさ: \(Int(brightness))%")
            Slider(value: $brightness, in: 0...100, step: 1) { editing in
                if !editing {
                    characteristic.writeValue(Int(brightness)) { error in
                        if let error = error {
                            print("明るさ調整エラー: \(error.localizedDescription)")
                        }
                    }
                }
            }
        }
        .onAppear {
            brightness = Double(characteristic.value as? Int ?? 50)
        }
    }
}
```

Rork Max のプロンプトでは「照明の明るさをスライダーで調整できるようにして。値の変更はスライダーを離したタイミングで送信して」と伝えると、上記のようなコードが生成されます。

## サーモスタット制御

温度管理はスマートホームのもう一つの主要機能です。

### 現在の温度と目標温度

サーモスタットは `HMServiceTypeThermostat` サービスを持ち、主要な特性として現在温度（`HMCharacteristicTypeCurrentTemperature`）と目標温度（`HMCharacteristicTypeTargetTemperature`）を提供します。

```swift
struct ThermostatView: View {
    let accessory: HMAccessory
    @State private var targetTemp: Double = 22.0
    @State private var currentTemp: Double = 0.0

    var thermostatService: HMService? {
        accessory.services.first { $0.serviceType == HMServiceTypeThermostat }
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("現在の室温")
                .font(.subheadline)
                .foregroundColor(.secondary)
            Text("\(currentTemp, specifier: "%.1f")°C")
                .font(.system(size: 48, weight: .thin))

            Divider()

            Text("目標温度")
                .font(.subheadline)
            HStack {
                Button(action: { adjustTarget(-0.5) }) {
                    Image(systemName: "minus.circle.fill")
                        .font(.title)
                }
                Text("\(targetTemp, specifier: "%.1f")°C")
                    .font(.system(size: 36, weight: .medium))
                    .frame(width: 120)
                Button(action: { adjustTarget(0.5) }) {
                    Image(systemName: "plus.circle.fill")
                        .font(.title)
                }
            }
        }
        .onAppear { loadTemperatures() }
    }

    private func adjustTarget(_ delta: Double) {
        targetTemp += delta
        targetTemp = max(16, min(30, targetTemp))
        writeTargetTemperature()
    }
}
```

温度の範囲は一般的に 16〜30℃ に制限し、0.5℃ 刻みで調整できるようにします。

## ダッシュボード画面の構築

ユーザーが一目でスマートホームの状態を把握できるダッシュボードを作りましょう。

### ルーム別サマリーカード

各ルームのデバイス数とアクティブ数を表示するカード型レイアウトは、SwiftUI の `LazyVGrid` で実現できます。

```swift
struct DashboardView: View {
    @ObservedObject var homeKit: HomeKitManager
    let columns = [GridItem(.flexible()), GridItem(.flexible())]

    var body: some View {
        ScrollView {
            LazyVGrid(columns: columns, spacing: 16) {
                ForEach(homeKit.primaryHome?.rooms ?? [], id: \.uniqueIdentifier) { room in
                    RoomCard(room: room)
                }
            }
            .padding()
        }
        .navigationTitle("ダッシュボード")
    }
}

struct RoomCard: View {
    let room: HMRoom

    var body: some View {
        VStack(alignment: .leading, spacing: 8) {
            Text(room.name)
                .font(.headline)
            Text("\(room.accessories.count) デバイス")
                .font(.caption)
                .foregroundColor(.secondary)
            HStack {
                ForEach(room.accessories.prefix(4), id: \.uniqueIdentifier) { accessory in
                    Image(systemName: iconForAccessory(accessory))
                        .foregroundColor(.blue)
                }
            }
        }
        .padding()
        .background(Color(.systemGray6))
        .cornerRadius(12)
    }
}
```

Rork Max に「ダッシュボードをグリッドレイアウトで作って、各ルームをカードとして表示して」と伝えるとこのような構造が生成されます。

## リアルタイム通知とオートメーション

HomeKit はデバイスの状態変化をリアルタイムで検知できます。

### 特性値の監視

`HMCharacteristic` の `enableNotification` を有効にすると、値が変化したタイミングでデリゲートメソッドが呼ばれます。

```swift
func startMonitoring(characteristic: HMCharacteristic) {
    characteristic.enableNotification(true) { error in
        if let error = error {
            print("通知登録エラー: \(error.localizedDescription)")
        }
    }
}
```

デリゲートの `accessory(_:service:didUpdateValueFor:)` でUIを更新すれば、他のアプリやシーンで状態が変わった場合にも即座に反映されます。

### シーンとオートメーション

HomeKit の **HMActionSet（シーン）** を使えば、複数のデバイスを一括操作するプリセットを作れます。「おやすみシーン」なら照明を全消灯、エアコンを 26℃ に設定する、といった組み合わせです。

```swift
func executeScene(_ scene: HMActionSet) {
    homeKit.primaryHome?.executeActionSet(scene) { error in
        if let error = error {
            print("シーン実行エラー: \(error.localizedDescription)")
        }
    }
}
```

Rork Max ではシーン一覧画面とワンタップ実行ボタンを含む UI を自動生成できます。

## テストとデバッグのポイント

HomeKit アプリの開発で注意すべきポイントをまとめます。

### HomeKit Accessory Simulator

Apple は Xcode に付属する **HomeKit Accessory Simulator** を提供しています。実際のスマートデバイスがなくても、仮想的な照明・サーモスタット・ロックを作成してアプリのテストが可能です。Rork Max で生成したアプリをシミュレータ上で動作確認する際に活用できます。

### よくあるエラーと対処法

HomeKit の開発でよく遭遇するエラーには以下のようなものがあります。Entitlements が未設定の場合は「HomeKit is not available」エラーが出ます。この場合は Apple Developer Portal で App ID の HomeKit Capability が有効になっているか確認してください。

ユーザーがホームを未作成の状態でアクセスすると `homes` 配列が空になります。初回起動時は「ホームを作成してください」という案内画面を表示するのがベストプラクティスです。

## App Store 公開に向けた仕上げ

HomeKit アプリを App Store に公開する際の追加考慮事項があります。

Apple の審査ガイドラインでは、HomeKit を使用するアプリはスマートホーム関連の実用的な機能を提供する必要があります。単なるデモアプリは審査でリジェクトされる可能性があるため、実際のユースケースに基づいた設計が重要です。

プライバシーポリシーには HomeKit を通じて取得するデータ（デバイス名、ルーム構成、使用パターン）の取り扱いを明記しましょう。Rork Max の 2 クリック App Store 公開機能を使えば、ビルドからサブミッションまでスムーズに進められます。
