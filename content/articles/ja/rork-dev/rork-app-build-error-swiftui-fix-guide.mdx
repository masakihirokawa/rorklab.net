## SwiftUIビルドエラーはなぜ起こる？

Rorkで生成したアプリをXcodeでビルドしようとすると、SwiftUI関連のエラーが出ることがあります。これは決して珍しくなく、単に設定を調整するだけで解決することがほとんどです。

Rorkが生成するコードはSwiftUI + SwiftData + Native APIの組み合わせで、非常に高度な機能を詰め込んでいます。そのため、Xcodeのバージョンやプロジェクト設定、そしてMacのメモリ状況によって、一時的なコンパイルエラーが発生することがあるのです。

この記事では、実際のエラーメッセージとその原因、確実な解決方法を紹介します。ビルドが完了したら、次は [TestFlight配信とApp Store公開](/articles/rork-business/rork-testflight-appstore-publish-troubleshooting)に進みます。

## エラー1：SwiftUI Preview が落ちる

### 症状

Canvas に「Xcode encountered an error: Cannot preview in this file」と表示されたり、Preview が繰り返しクラッシュする場合があります。

### 原因

- Rork が生成したビューが複雑すぎて、Preview プロセスのメモリが不足している
- `@Observable` macro や `@State` の変更検知が Preview で正しく動作していない
- Xcode キャッシュが破損している

### 解決方法

**ステップ1：キャッシュをクリアする**

```bash
# Xcodeビルドキャッシュをクリア
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# Xcodeを再起動
killall Xcode
```

**ステップ2：Preview を再設定する**

Canvas でエラーが表示されている場合、Resume ボタンを押すか、Editor → Canvas を再度有効にします。

**ステップ3：プレビュー用の簡易モックを作成する**

大規模なビューの場合、Preview 専用の簡易版を作成することで高速化します：

```swift
#Preview {
  // 簡易モックで Preview
  ContentView()
    .modelContainer(for: Item.self, inMemory: true)
}
```

**ステップ4：メモリを解放する**

Mac のメモリが不足している場合、他のアプリを閉じて Xcode に割き当てるメモリを増やします。Activity Monitor を開いて、「Xcode」のメモリ使用量を確認しましょう。目安は 4GB 以上が理想です。

## エラー2：Type checking takes too long

### 症状

Xcode が「Type checking takes too long for this file」と警告を出し、コンパイルが非常に遅くなります。数分待ってもビルドが完了しません。

### 原因

Rork が複雑な SwiftUI ビューツリーや関数型プログラミングを駆使してコードを生成している場合、Swift コンパイラが全体の型推論に時間がかかります。特に以下の場合に発生しやすいです：

- `@ViewBuilder` を多重ネストしている
- `Optional` や `Result` 型を複数チェーンしている
- `@State` や `@ObservedRealmObject` の依存関係が複雑

### 解決方法

**方法A：型アノテーションを明示的に追加する**

Rork が生成したコードの戻り値を明確に型指定します：

```swift
var body: some View {
  VStack {
    // 内容
  }
}
```

`some View` ではなく、より具体的な型を指定することでコンパイラの負担が減ります。

**方法B：ビューを分割する**

大規模なビューを複数のサブビューに分割し、それぞれ独立した構造体にします：

```swift
struct ParentView: View {
  var body: some View {
    HeaderSection()
    ContentSection()
    FooterSection()
  }
}

struct HeaderSection: View {
  var body: some View {
    // Header 専用
  }
}
```

**方法C：Xcode のビルド設定を最適化する**

Build Settings で以下を変更します：

```
Optimization Level: None（-Onone）
Swift Compiler - Code Generation: Optimization for Speed: No
```

これでコンパイルは遅くなりますが、型チェック警告が減ります。

## エラー3：Missing import statement

### 症状

「error: cannot find 'someFunction' in scope」「Use of undeclared identifier」といったエラーが出ます。

### 原因

Rork が生成したコード内で必要なフレームワークやモジュールのインポートが不足しています。よくあるケース：

- `SwiftData` の `@Model` macro を使っているのに `import SwiftData` がない
- `HealthKit` や `CoreLocation` などのネイティブフレームワークの import 漏れ
- 自動生成されたコンポーネント間の import 不足

### 解決方法

**自動修復を試す**

Xcode のエラーメッセージの下に「Fix」ボタンが出ることがあります。これをクリックすると自動で import が追加されます。

**手動で追加する場合**

ファイルの最上部に以下を追加します：

```swift
import SwiftUI
import SwiftData
import Foundation
// その他必要なフレームワーク
```

**よくあるインポート一覧**

| 機能 | import文 |
|---|---|
| UI基本 | `import SwiftUI` |
| データ永続化 | `import SwiftData` |
| ヘルスケア連携 | `import HealthKit` |
| 位置情報 | `import CoreLocation` |
| 画像処理 | `import Vision` |
| ファイルアクセス | `import UniformTypeIdentifiers` |
| 通知 | `import UserNotifications` |

## エラー4：iOS バージョン互換性エラー

### 症状

「This declaration requires iOS 17.0 or newer」のような警告やエラーが出ます。あなたのプロジェクトが iOS 16 対応なのに、iOS 17 以降の API を使用しているコードが含まれています。

### 原因

Rork が最新の SwiftUI 5.0+ API（iOS 17+ のみ対応）を使ってコードを生成している一方で、プロジェクトの Deployment Target が iOS 16 に設定されているミスマッチです。

### 解決方法

**方法A：Deployment Target を上げる（推奨）**

1. Xcode で「Project」→「Build Settings」を開く
2. 「Minimum Deployments」または「iOS Deployment Target」を検索
3. iOS 17 以上に変更

```
iOS Deployment Target: 17.0
```

**方法B：互換性マークを追加する**

iOS 16 対応を維持したい場合、API にバージョン条件をつけます：

```swift
if #available(iOS 17, *) {
  // iOS 17+ のコード
} else {
  // iOS 16 用の代替実装
}
```

## エラー5：Xcode バージョンの不整合

### 症状

「Cannot load underlying module for 'some_module'」または「Module compiled with Swift 5.X cannot be imported by the Swift 5.Y compiler」のようなエラーが出ます。

### 原因

ローカル開発環境の Xcode バージョンと、Rork が前提にしている Swift バージョンが異なっています。たとえば、古い Xcode 15.0 で Swift 5.8 のコードをコンパイルしようとしている場合です。

### 解決方法

**Xcode を最新版にアップデート**

```bash
# Mac App Store から最新版をインストール
# または

# コマンドラインから最新版をセット
sudo xcode-select --install
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

バージョンを確認：

```bash
swift --version
```

Rork が推奨するバージョンは **Xcode 16.0 以上**です。

## エラー6：SwiftUI Layout の計算エラー

### 症状

ビルドは成功するものの、実行時に「Publishing changes from background threads is not allowed」や「State mutation during view update」といったランタイム警告が出ます。

### 原因

Rork が生成したレイアウトロジックで、メインスレッド以外から UI 状態を更新しています。これは非同期処理（API呼び出しやデータベースアクセス）の結果を直接 `@State` に反映させるときに起こります。

### 解決方法

**メインスレッドで実行する**

```swift
DispatchQueue.main.async {
  self.isLoading = false
  self.data = result
}
```

または、Task を使用します：

```swift
Task {
  let result = await fetchData()
  self.data = result
}
```

## よくある質問（FAQ）

### Q1：「Undefined symbol」エラーが毎回出ます

**A：** Linked Frameworks and Libraries が不足しています。Build Phases で、必要なフレームワークが全て追加されているか確認してください。SwiftData を使う場合は「SwiftData.framework」を手動で追加することが必要な場合があります。

### Q2：Preview が機能しないので、毎回シミュレータで動作確認しています

**A：** Preview がクラッシュする場合、プレビュー用に簡易データを用意してモックにすることで高速化できます。大規模なビューの場合、プレビュー専用の軽量バージョンを作成するのが定石です。

### Q3：Swift Playgrounds で Rork のコードを実行できますか？

**A：** Playgrounds は iOS シミュレータの制御に制限があるため、フルスタック Rork アプリの実行には向きません。Xcode シミュレータまたは実機テストを使用してください。

## まとめ

SwiftUI のビルドエラーは、Xcode キャッシュのクリア、型アノテーションの明示化、バージョン確認で 90% 以上解決できます。重要なのは、エラーメッセージを落ち着いて読むことと、段階的に原因を絞り込むことです。

もし上記の方法でも解決しない場合は、Xcode の Clean Build Folder（Shift + Cmd + K）を試してから、DerivedData フォルダ全体を削除して Xcode を再起動してみてください。

ビルドが成功したら、[Rorkアプリのデータベース・バックエンド接続エラー解決法](/articles/rork-dev/rork-database-backend-connection-error-fix)も参考にしながら、バックエンド設定を進めるのがお勧めです。
