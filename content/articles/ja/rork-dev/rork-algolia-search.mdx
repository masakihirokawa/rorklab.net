## はじめに：なぜRorkアプリに全文検索が必要か

Rorkで作ったECアプリ、レシピアプリ、メモアプリ——ユーザーが増えてコンテンツが膨らむほど、**「探せない」ストレス**がアプリ評価を下げます。`LIKE '%keyword%'` による SQLの部分一致検索は、データ件数が数千件を超えると途端にレスポンスが遅くなり、日本語の形態素解析にも対応しません。

この記事では、世界トップクラスの検索 SaaS である **Algolia** を Rork アプリに組み込み、**ミリ秒レベルのリアルタイム全文検索**を実装する方法をステップバイステップで解説します。Algolia は 1 ヶ月 10,000 リクエストまで無料で利用でき、個人開発・スモールビジネスのプロダクト検索に最適です。

この記事を読み終えると、以下のことができるようになります：

- Algolia のインデックス（検索データベース）を設計・作成する
- Rork / React Native アプリに `instantsearch-expo` を導入する
- SearchBox・Hits コンポーネントをカスタマイズする
- Supabase または Firebase とリアルタイム同期する
- 日本語検索精度を高める設定を行う

**対象読者**: Rork でアプリを開発済みで、検索機能の追加を検討している中級者（TypeScript・React Native の基礎知識があること）

### Algolia の基本概念

Algolia では以下の用語を理解しておく必要があります：

- **Index（インデックス）**: Algolia 上の「テーブル」。商品・記事・ユーザーなど種類ごとに作成する
- **Record（レコード）**: インデックス内の1件のデータ（JSON オブジェクト）
- **Application ID**: Algolia アプリケーションの識別子
- **Search API Key**: クライアント側（アプリ）で使用する公開キー
- **Admin API Key**: サーバー側でのインデックス操作に使用する秘密キー（アプリに埋め込み禁止）

## Algolia のセットアップ

### ステップ 1：Algolia アカウントの作成とインデックス作成

1. [Algolia ダッシュボード](https://dashboard.algolia.com/) にログイン
2. 「Indices」→「Create Index」をクリック
3. インデックス名を入力（例：`recipes`、`products`、`articles`）
4. 「API Keys」セクションから以下をメモ：
   - **Application ID** (例: `ABCD1234EF`)
   - **Search-Only API Key** (例: `abc123...`)
   - **Admin API Key**（サーバー専用・アプリに含めない）

### ステップ 2：テストデータのアップロード

Algolia ダッシュボードで「Add records」→「Upload JSON file」から、または `algoliasearch` パッケージを使ってコードからアップロードできます。

以下はレシピアプリのサンプルデータです：

```json
[
  {
    "objectID": "recipe_001",
    "name": "アボカドトースト",
    "category": "朝食",
    "ingredients": ["アボカド", "パン", "レモン", "塩"],
    "cookTime": 10,
    "tags": ["簡単", "ヘルシー", "ベジタリアン"]
  },
  {
    "objectID": "recipe_002",
    "name": "チキン南蛮",
    "category": "夕食",
    "ingredients": ["鶏もも肉", "卵", "小麦粉", "タルタルソース"],
    "cookTime": 35,
    "tags": ["ボリューム", "和食", "人気"]
  }
]
```

`objectID` は必須フィールドです。Supabase の `id` や Firebase の `docId` をそのまま使うと後の同期が楽になります。

## Rork アプリへの Algolia 統合

### ステップ 3：必要なパッケージのインストール

Rork の GitHub 連携でリポジトリをエクスポートし、以下のパッケージを追加します：

```bash
npx expo install algoliasearch @instantsearch/react-hooks react-instantsearch-core
```

> **Rork でのパッケージ追加方法**: Rork のチャット画面で「algoliasearch パッケージを追加して」とプロンプトすると、`package.json` の更新と `npm install` を自動で行ってくれます。

### ステップ 4：Algolia クライアントの初期化

`lib/algolia.ts` ファイルを作成して設定を一元管理します：

```typescript
// lib/algolia.ts
import algoliasearch from 'algoliasearch/lite';

// 環境変数から読み込む（ハードコーディングしない）
const APP_ID = process.env.EXPO_PUBLIC_ALGOLIA_APP_ID ?? '';
const SEARCH_KEY = process.env.EXPO_PUBLIC_ALGOLIA_SEARCH_KEY ?? '';

export const searchClient = algoliasearch(APP_ID, SEARCH_KEY);
export const INDEX_NAME = 'recipes'; // 自分のインデックス名に変更

// 期待する出力: searchClient オブジェクトが生成され、検索クエリを送信できる状態
```

`.env.local`（またはEAS Secrets）に以下を追加：

```bash
EXPO_PUBLIC_ALGOLIA_APP_ID=ABCD1234EF
EXPO_PUBLIC_ALGOLIA_SEARCH_KEY=abc123...
```

> ⚠️ **Admin API Key は絶対に `EXPO_PUBLIC_` 変数に入れないでください。** `EXPO_PUBLIC_` プレフィックスのある変数はアプリバンドルに含まれ、ユーザーが確認できます。Admin キーはサーバー側（Cloud Functions・Edge Functions）のみで使用してください。

### ステップ 5：InstantSearch プロバイダーのセットアップ

`App.tsx` または最上位のレイアウトコンポーネントに `InstantSearch` プロバイダーを追加します：

```typescript
// App.tsx
import React from 'react';
import { InstantSearch } from 'react-instantsearch-core';
import { searchClient, INDEX_NAME } from '@/lib/algolia';
import SearchScreen from '@/screens/SearchScreen';

export default function App() {
  return (
    <InstantSearch
      searchClient={searchClient}
      indexName={INDEX_NAME}
      future={{ preserveSharedStateOnUnmount: true }}
    >
      <SearchScreen />
    </InstantSearch>
  );
}
```

### ステップ 6：検索 UI の実装（SearchBox + Hits）

`screens/SearchScreen.tsx` に検索ボックスと結果リストを実装します：

```typescript
// screens/SearchScreen.tsx
import React from 'react';
import {
  View,
  TextInput,
  FlatList,
  Text,
  TouchableOpacity,
  StyleSheet,
} from 'react-native';
import { useSearchBox, useHits } from 'react-instantsearch-core';

// --- SearchBox コンポーネント ---
function SearchBox() {
  const { query, refine, clear } = useSearchBox();

  return (
    <View style={styles.searchContainer}>
      <TextInput
        style={styles.input}
        value={query}
        onChangeText={refine}
        placeholder="レシピを検索..."
        placeholderTextColor="#999"
        returnKeyType="search"
        clearButtonMode="while-editing"
        autoCorrect={false}
      />
      {query.length > 0 && (
        <TouchableOpacity onPress={clear} style={styles.clearButton}>
          <Text style={styles.clearText}>✕</Text>
        </TouchableOpacity>
      )}
    </View>
  );
}

// --- Hit（検索結果の1件）の型定義 ---
type RecipeHit = {
  objectID: string;
  name: string;
  category: string;
  cookTime: number;
  tags: string[];
};

// --- Hits コンポーネント ---
function RecipeHits() {
  const { hits } = useHits<RecipeHit>();

  if (hits.length === 0) {
    return (
      <View style={styles.empty}>
        <Text style={styles.emptyText}>レシピが見つかりませんでした</Text>
      </View>
    );
  }

  return (
    <FlatList
      data={hits}
      keyExtractor={(item) => item.objectID}
      renderItem={({ item }) => (
        <TouchableOpacity style={styles.hitCard}>
          <Text style={styles.hitName}>{item.name}</Text>
          <Text style={styles.hitMeta}>
            {item.category} · {item.cookTime}分
          </Text>
          <View style={styles.tagRow}>
            {item.tags.map((tag) => (
              <View key={tag} style={styles.tag}>
                <Text style={styles.tagText}>{tag}</Text>
              </View>
            ))}
          </View>
        </TouchableOpacity>
      )}
    />
  );
}

// --- メインスクリーン ---
export default function SearchScreen() {
  return (
    <View style={styles.container}>
      <SearchBox />
      <RecipeHits />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f5f5f5' },
  searchContainer: {
    flexDirection: 'row',
    alignItems: 'center',
    margin: 16,
    backgroundColor: '#fff',
    borderRadius: 12,
    paddingHorizontal: 12,
    elevation: 2,
    shadowColor: '#000',
    shadowOpacity: 0.08,
    shadowRadius: 4,
  },
  input: {
    flex: 1,
    height: 48,
    fontSize: 16,
    color: '#333',
  },
  clearButton: { padding: 8 },
  clearText: { fontSize: 14, color: '#999' },
  hitCard: {
    backgroundColor: '#fff',
    marginHorizontal: 16,
    marginBottom: 10,
    padding: 16,
    borderRadius: 12,
    elevation: 1,
  },
  hitName: { fontSize: 17, fontWeight: '600', color: '#222' },
  hitMeta: { fontSize: 13, color: '#666', marginTop: 4 },
  tagRow: { flexDirection: 'row', flexWrap: 'wrap', marginTop: 8, gap: 6 },
  tag: {
    backgroundColor: '#e8f4fd',
    borderRadius: 20,
    paddingHorizontal: 10,
    paddingVertical: 4,
  },
  tagText: { fontSize: 12, color: '#0078d4' },
  empty: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  emptyText: { fontSize: 15, color: '#999' },
});

// 期待する出力: 文字入力のたびにAlgoliaへクエリが送信され、結果が即座にリスト表示される
```

## Supabase / Firebase との同期設定

Rork アプリのデータベースが Supabase や Firebase の場合、データ変更時に Algolia のインデックスを自動同期する仕組みが必要です。

### Supabase との同期（Edge Function）

Supabase の Database Webhook と Edge Function を組み合わせて、レコードの INSERT/UPDATE/DELETE を検知し Algolia に反映します：

```typescript
// supabase/functions/sync-algolia/index.ts
import algoliasearch from 'https://esm.sh/algoliasearch@5';

const client = algoliasearch(
  Deno.env.get('ALGOLIA_APP_ID')!,
  Deno.env.get('ALGOLIA_ADMIN_API_KEY')! // サーバー側なので Admin Key を使用
);

Deno.serve(async (req) => {
  const payload = await req.json();
  const { type, record, old_record } = payload;

  const index = client.initIndex('recipes');

  if (type === 'INSERT' || type === 'UPDATE') {
    // Supabase の id を Algolia の objectID として使用
    await index.saveObject({ ...record, objectID: record.id });
  } else if (type === 'DELETE') {
    await index.deleteObject(old_record.id);
  }

  return new Response(JSON.stringify({ success: true }), {
    headers: { 'Content-Type': 'application/json' },
  });
});

// 期待する出力: Supabase のレコード変更が数秒以内に Algolia のインデックスに反映される
```

Supabase ダッシュボードで「Database Webhooks」を設定し、`INSERT`・`UPDATE`・`DELETE` イベントで上記 Edge Function を呼び出すよう設定してください。

### Firebase との同期（Cloud Functions）

```typescript
// functions/src/index.ts
import * as functions from 'firebase-functions';
import algoliasearch from 'algoliasearch';

const client = algoliasearch(
  functions.config().algolia.app_id,
  functions.config().algolia.admin_key
);
const index = client.initIndex('recipes');

// Firestore の recipes コレクションを監視
export const syncToAlgolia = functions.firestore
  .document('recipes/{docId}')
  .onWrite(async (change, context) => {
    const docId = context.params.docId;

    if (!change.after.exists) {
      // ドキュメント削除
      await index.deleteObject(docId);
    } else {
      // 追加・更新
      const data = change.after.data()!;
      await index.saveObject({ ...data, objectID: docId });
    }
  });
```

## 日本語検索精度を高める設定

Algolia はデフォルトで日本語に対応していますが、以下の設定でさらに精度が向上します。

### インデックス設定（ダッシュボード or API）

```javascript
// スクリプトでインデックス設定を行う場合
// scripts/configure-algolia.mjs
import algoliasearch from 'algoliasearch';

const client = algoliasearch(
  process.env.ALGOLIA_APP_ID,
  process.env.ALGOLIA_ADMIN_KEY
);
const index = client.initIndex('recipes');

await index.setSettings({
  // 検索対象フィールドと優先順位
  searchableAttributes: [
    'name',          // 最優先
    'category',
    'ingredients',
    'tags',
  ],
  // ランキングに使うカスタム指標
  customRanking: ['desc(cookTime)'],
  // タイポ許容（日本語は false 推奨）
  typoTolerance: false,
  // 日本語専用設定
  ignorePlurals: false,
  removeStopWords: false,
  // 検索結果に含めるフィールド（不要なフィールドを除外してレスポンスを軽量化）
  attributesToRetrieve: ['name', 'category', 'cookTime', 'tags'],
  // ハイライト対象フィールド
  attributesToHighlight: ['name', 'ingredients'],
});

console.log('✅ Algolia インデックス設定が完了しました');
// 期待する出力: ✅ Algolia インデックス設定が完了しました
```

## よくあるエラーと対処法

### エラー 1：「Invalid Application-ID or API Key」

```
Error: Invalid Application-ID or API Key
```

**原因**: 環境変数の設定ミス、またはキーの種類を間違えている（Admin Key を使用すべき場所で Search Key を使用など）。

**対処**: `.env.local` の `EXPO_PUBLIC_ALGOLIA_APP_ID` と `EXPO_PUBLIC_ALGOLIA_SEARCH_KEY` の値を Algolia ダッシュボードで再確認する。EAS Build を使っている場合は `eas secret:list` でシークレットが登録されているか確認する。

### エラー 2：検索結果が表示されない

**原因**: インデックスにデータが存在しない、または `searchableAttributes` に検索したいフィールドが含まれていない。

**対処**: Algolia ダッシュボードの「Browse」でインデックスにレコードが存在するか確認する。`searchableAttributes` 設定にフィールド名が含まれているか確認する。

### エラー 3：日本語で検索すると部分一致しない

**原因**: Algolia の標準設定では日本語の形態素解析が限定的。

**対処**: インデックス設定で `queryLanguages: ['ja']` と `indexLanguages: ['ja']` を追加する。それでも解消しない場合は、`name` フィールドに読み仮名（ふりがな）や英語表記を追加したフィールドをインデックスに含める。

## 応用：フィルタリングとファセット検索

Algolia の強力な機能の一つが**ファセット検索**（カテゴリ・価格帯などで絞り込む検索）です。Rork アプリのショッピングや不動産アプリに特に有効です。

```typescript
// screens/SearchScreen.tsx に追加
import { useRefinementList } from 'react-instantsearch-core';

function CategoryFilter() {
  const { items, refine } = useRefinementList({ attribute: 'category' });

  return (
    <View style={{ flexDirection: 'row', padding: 8, gap: 8 }}>
      {items.map((item) => (
        <TouchableOpacity
          key={item.value}
          onPress={() => refine(item.value)}
          style={{
            backgroundColor: item.isRefined ? '#0078d4' : '#e0e0e0',
            borderRadius: 20,
            paddingHorizontal: 14,
            paddingVertical: 6,
          }}
        >
          <Text style={{ color: item.isRefined ? '#fff' : '#333', fontSize: 13 }}>
            {item.label} ({item.count})
          </Text>
        </TouchableOpacity>
      ))}
    </View>
  );
}
```

ファセット検索を有効にするには、インデックス設定に `attributesForFaceting: ['category', 'tags']` を追加します。

## FAQ

**Q: Algolia の無料プランでどの程度まで使えますか？**

A: Algolia の無料プランは月 10,000 リクエスト（検索 + インデックス操作の合計）まで利用できます。アクティブユーザーが数百人規模の個人開発アプリであれば十分です。1,000 人を超えるDAUのアプリでは Starter プラン（$29/月〜）への移行を検討してください。

**Q: Rork のプロンプトだけで Algolia 統合はできますか？**

A: 基本的な `SearchBox` と `Hits` コンポーネントの生成は Rork のプロンプトで対応できます。「Algoliaを使ったリアルタイム検索UIを作って。Application ID は XXX、Search Key は YYY」と入力することで雛形が生成されます。ただし、Supabase/Firebase との同期ロジックはサーバー側の設定が必要なため、この記事のコードを参考に手動で設定することをおすすめします。

**Q: Algolia の代替サービスはありますか？**

A: はい。代表的な代替サービスとして **Typesense**（オープンソース、セルフホスト可）、**Meilisearch**（高速・日本語対応良好）、**Elasticsearch**（エンタープライズ向け）があります。Rork アプリへの統合方法は Algolia と概ね同じで、パッケージ名と API が異なるのみです。小規模アプリで完全無料にこだわる場合は Typesense Cloud の Free Tier も選択肢です。
