## RorkアプリでRevenueCatを使う理由

Rorkで素晴らしいアプリを作り上げても、継続的な収益を得る仕組みがなければビジネスとして成立しません。モバイルアプリの収益化において最も強力な手段のひとつが**アプリ内課金（In-App Purchase）**と**サブスクリプション**です。

App StoreとGoogle Playはそれぞれ独自のIAP（In-App Purchase）APIを持っており、両方に対応するのは従来非常に複雑でした。ここで登場するのが**RevenueCat**です。RevenueCatはiOSとAndroidのIAPを統一したSDKで管理でき、Rorkのプロンプト駆動開発とも非常に相性が良いツールです。

本記事では、RorkアプリにRevenueCatを統合してサブスクリプションを実装する方法を、ステップバイステップで解説します。

---

## RevenueCatとは何か

RevenueCatは、モバイルアプリの**サブスクリプション・アプリ内課金を一元管理**するためのプラットフォームです。2018年の創業以降、世界中の個人開発者・スタートアップに利用され、管理する月間収益は数千億円規模に達しています。

### RevenueCatの主なメリット

- **クロスプラットフォーム対応**: App Store（iOS）とGoogle Play（Android）を統一APIで管理
- **レシート検証の自動化**: サーバーサイドでの不正対策を自動処理
- **リアルタイムダッシュボード**: MRR・チャーン率・LTVなどをリアルタイムで可視化
- **Webhookとインテグレーション**: Slack・Amplitude・Mixpanelなど主要ツールと連携
- **無料プラン**: 月間収益が$2,500未満なら無料で利用可能

Stripeと異なり、RevenueCatはApp StoreやGoogle Playのネイティブ課金システムを通じて決済を行うため、**プラットフォームのガイドラインに完全準拠**しているのも大きな特徴です。

## 事前準備

実装を始める前に以下の準備が必要です。

### 必要なもの

1. **Rorkアカウント**（Junior プラン以上を推奨）
2. **RevenueCatアカウント**（無料で作成可能）
3. **Apple Developer Program**（$99/年）
4. **Google Play Developer アカウント**（$25 一回限り）
5. App Store ConnectおよびGoogle Play Consoleでの商品設定

### App Store Connectでの商品設定

RevenueCatを動かす前に、App Store ConnectでIAP商品を作成します。

1. App Store Connect → 「マイApp」→ 対象アプリを選択
2. 「アプリ内課金」タブ → 「+」ボタンをクリック
3. 「自動更新サブスクリプション」を選択
4. 商品ID（例: `com.yourapp.premium_monthly`）と表示名を設定
5. 価格設定（例: 月額¥600）を完了させる

Google Playも同様に、Google Play Console の「収益化 → 定期購入」から商品を作成します。

## RevenueCatのセットアップ

### Step 1: RevenueCatプロジェクトを作成

[RevenueCat](https://www.revenuecat.com/) にサインアップし、新規プロジェクトを作成します。

1. ダッシュボード → 「+ New Project」
2. プロジェクト名を入力（例: `MyApp`）
3. iOSとAndroidのアプリを追加
4. **APIキー**（Public SDK Key）を取得 — 後で使用します

### Step 2: Entitlementsとofferingsを設定

RevenueCatの「Entitlements（権限）」はユーザーがアクセスできるコンテンツの概念です。

- **Entitlement** 例: `premium`（プレミアム機能へのアクセス）
- **Product** 例: App Store / Google Playで作成した商品ID
- **Offering** 例: ユーザーに表示するプラン一覧

ダッシュボードで `premium` というEntitlementを作成し、App StoreとGoogle Playの商品IDをアタッチします。

### Step 4: RevenueCat SDKの初期化

Rorkが生成したコードに以下の初期化処理が含まれていることを確認します。

```typescript
// app/_layout.tsx（Expo Router使用時）
import Purchases, { LOG_LEVEL } from 'react-native-purchases';
import { Platform } from 'react-native';

const REVENUECAT_IOS_KEY = 'appl_xxxxxxxxxxxxxxxxxxxxxxxx';
const REVENUECAT_ANDROID_KEY = 'goog_xxxxxxxxxxxxxxxxxxxxxxxx';

export default function RootLayout() {
  useEffect(() => {
    // RevenueCatの初期化（アプリ起動時に1回だけ実行）
    if (Platform.OS === 'ios') {
      Purchases.configure({ apiKey: REVENUECAT_IOS_KEY });
    } else {
      Purchases.configure({ apiKey: REVENUECAT_ANDROID_KEY });
    }

    // 開発時のみデバッグログを有効化
    if (__DEV__) {
      Purchases.setLogLevel(LOG_LEVEL.DEBUG);
    }
  }, []);

  return <Stack />;
}
// 期待する動作: アプリ起動時にRevenueCatが初期化され、購入履歴が自動同期される
```

### Step 5: 購入状態の管理

アプリ全体でサブスクリプション状態を管理するContextを作成します。

```typescript
// contexts/PurchaseContext.tsx
import React, { createContext, useContext, useEffect, useState } from 'react';
import Purchases, { CustomerInfo } from 'react-native-purchases';

interface PurchaseContextType {
  isPremium: boolean;
  customerInfo: CustomerInfo | null;
  isLoading: boolean;
  checkPremiumStatus: () => Promise<void>;
}

const PurchaseContext = createContext<PurchaseContextType | null>(null);

export function PurchaseProvider({ children }: { children: React.ReactNode }) {
  const [isPremium, setIsPremium] = useState(false);
  const [customerInfo, setCustomerInfo] = useState<CustomerInfo | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  const checkPremiumStatus = async () => {
    try {
      const info = await Purchases.getCustomerInfo();
      setCustomerInfo(info);
      // "premium"はRevenueCatダッシュボードで設定したEntitlement ID
      setIsPremium(info.entitlements.active['premium'] !== undefined);
    } catch (error) {
      console.error('RevenueCat エラー:', error);
    } finally {
      setIsLoading(false);
    }
  };

  useEffect(() => {
    checkPremiumStatus();

    // 購入状態の変更をリアルタイムで監視
    const customerInfoUpdatedListener = Purchases.addCustomerInfoUpdateListener(
      (info) => {
        setCustomerInfo(info);
        setIsPremium(info.entitlements.active['premium'] !== undefined);
      }
    );

    return () => customerInfoUpdatedListener.remove();
  }, []);

  return (
    <PurchaseContext.Provider
      value={{ isPremium, customerInfo, isLoading, checkPremiumStatus }}
    >
      {children}
    </PurchaseContext.Provider>
  );
}

export const usePurchase = () => {
  const context = useContext(PurchaseContext);
  if (!context) throw new Error('usePurchase must be used within PurchaseProvider');
  return context;
};
// 期待する動作: isPremiumがtrueなら有料ユーザー、falseなら無料ユーザーとして扱われる
```

### Step 6: ペイウォール（購入画面）の実装

```typescript
// screens/PaywallScreen.tsx
import React, { useEffect, useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet, Alert } from 'react-native';
import Purchases, { Package, PurchasesOffering } from 'react-native-purchases';
import { usePurchase } from '../contexts/PurchaseContext';

export default function PaywallScreen({ onClose }: { onClose: () => void }) {
  const [offering, setOffering] = useState<PurchasesOffering | null>(null);
  const [isPurchasing, setIsPurchasing] = useState(false);
  const { checkPremiumStatus } = usePurchase();

  useEffect(() => {
    fetchOfferings();
  }, []);

  const fetchOfferings = async () => {
    try {
      const offerings = await Purchases.getOfferings();
      // "default"はRevenueCatダッシュボードで設定したOffering ID
      if (offerings.current) {
        setOffering(offerings.current);
      }
    } catch (error) {
      console.error('Offerings取得エラー:', error);
    }
  };

  const handlePurchase = async (pkg: Package) => {
    setIsPurchasing(true);
    try {
      const { customerInfo } = await Purchases.purchasePackage(pkg);
      if (customerInfo.entitlements.active['premium']) {
        await checkPremiumStatus();
        Alert.alert('購入完了！', 'プレミアムプランへようこそ！');
        onClose();
      }
    } catch (error: any) {
      if (!error.userCancelled) {
        Alert.alert('購入エラー', error.message);
      }
    } finally {
      setIsPurchasing(false);
    }
  };

  const handleRestore = async () => {
    try {
      const customerInfo = await Purchases.restorePurchases();
      if (customerInfo.entitlements.active['premium']) {
        await checkPremiumStatus();
        Alert.alert('復元完了', '購入履歴を復元しました！');
        onClose();
      } else {
        Alert.alert('復元', '復元できる購入履歴がありませんでした。');
      }
    } catch (error: any) {
      Alert.alert('復元エラー', error.message);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>プレミアムプランにアップグレード</Text>
      <Text style={styles.subtitle}>すべての機能を無制限で使えます</Text>

      {offering?.availablePackages.map((pkg) => (
        <TouchableOpacity
          key={pkg.identifier}
          style={styles.packageButton}
          onPress={() => handlePurchase(pkg)}
          disabled={isPurchasing}
        >
          <Text style={styles.packageTitle}>{pkg.product.title}</Text>
          <Text style={styles.packagePrice}>{pkg.product.priceString} / 月</Text>
        </TouchableOpacity>
      ))}

      <TouchableOpacity style={styles.restoreButton} onPress={handleRestore}>
        <Text style={styles.restoreText}>購入を復元する</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 24, backgroundColor: '#fff' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 8 },
  subtitle: { fontSize: 16, color: '#666', marginBottom: 32 },
  packageButton: {
    backgroundColor: '#007AFF', borderRadius: 12,
    padding: 16, marginBottom: 12, alignItems: 'center',
  },
  packageTitle: { color: '#fff', fontSize: 18, fontWeight: '600' },
  packagePrice: { color: 'rgba(255,255,255,0.8)', fontSize: 14, marginTop: 4 },
  restoreButton: { marginTop: 16, alignItems: 'center' },
  restoreText: { color: '#007AFF', fontSize: 14 },
});
// 期待する動作: App Store/Google Playの購入ダイアログが表示され、
// 購入後にentitlementsがアクティブになる
```

## よくあるエラーと対処法

### エラー1: 「This in-app purchase has already been bought」

**原因**: 既に購入済みの商品を再購入しようとしている
**対処**: `Purchases.restorePurchases()` を呼び出して購入を復元します

### エラー2: サンドボックス環境でのテスト失敗

**原因**: App Store ConnectでSandboxテスター設定が未完了
**対処**: App Store Connect → 「ユーザーとアクセス」→「テスター」でSandboxアカウントを作成し、実機でサインインしてテストします

### エラー3: Google Play商品が「利用不可」と表示される

**原因**: Google Play ConsoleでPublishされた商品が有効化されていない、またはアプリが審査前の状態
**対処**: 商品のステータスを「有効」に設定し、アプリを内部テストトラックに公開してからテストします

### エラー4: `Purchases.configure()` が複数回呼ばれる

**原因**: コンポーネントの再レンダリングで初期化処理が繰り返し実行される
**対処**: `useEffect` の依存配列を `[]`（空）にして、アプリ起動時の1回だけ実行されるようにします

## 応用：RevenueCatのWebhookとアナリティクス活用

RevenueCatダッシュボードの強力な機能のひとつが**Webhook**です。購入・解約・更新などのイベントをリアルタイムで外部サービスに通知できます。

### 活用例

- **Slack通知**: 新規購入が発生したらSlackに通知
- **Amplitude/Mixpanel連携**: 課金ユーザーの行動をトラッキング
- **バックエンド同期**: SupabaseやFirebaseのユーザーレコードを自動更新

Rorkで生成したSupabaseバックエンドと組み合わせる場合、RevenueCatのWebhookをSupabase Edge Functionsで受け取り、ユーザーのサブスクリプション状態をデータベースに同期させることが可能です。

```typescript
// Supabase Edge Function例（webhook受信）
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

serve(async (req) => {
  const payload = await req.json()
  const { event, app_user_id, product_id } = payload

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  if (event === 'INITIAL_PURCHASE' || event === 'RENEWAL') {
    // ユーザーのプレミアムフラグを更新
    await supabase
      .from('profiles')
      .update({ is_premium: true, subscription_product: product_id })
      .eq('id', app_user_id)
  } else if (event === 'EXPIRATION' || event === 'CANCELLATION') {
    await supabase
      .from('profiles')
      .update({ is_premium: false })
      .eq('id', app_user_id)
  }

  return new Response('ok', { status: 200 })
})
// 期待する動作: RevenueCatのイベントに応じてDBのサブスクリプション状態が自動更新される
```

## FAQ

**Q: RevenueCatは無料で使えますか？**

A: はい、月間追跡収益が$2,500未満であれば無料プランで全機能を利用できます。それ以上になるとトランザクション手数料が発生しますが、管理コストを大幅に削減できるため費用対効果は高いです。

**Q: Rork MaxのネイティブSwiftアプリでもRevenueCatは使えますか？**

A: はい、RevenueCatはSwiftUI向けのネイティブSDK（RevenueCat iOS SDK）を提供しており、Rork Maxで生成したSwiftアプリにも統合できます。Rork MaxのプロンプトにRevenueCatの統合を指示すれば、Claude Opus 4.6がSwiftUIのコードを生成します。

**Q: App StoreとGoogle Playの両方を同時に管理できますか？**

A: はい、RevenueCatの最大のメリットのひとつがクロスプラットフォーム対応です。ひとつのRevenueCatプロジェクトでiOSとAndroidの両方の商品・顧客・収益を一元管理できます。

**Q: サブスクリプションのトライアル（無料体験）は実装できますか？**

A: App Store Connectでサブスクリプション商品に「無料トライアル」を設定すれば、RevenueCat経由で自動的にトライアルが有効になります。コード側での追加実装は不要です。

**Q: ユーザーがアプリを再インストールした場合、購入履歴はどうなりますか？**

A: `Purchases.restorePurchases()` を呼び出すことで、App StoreまたはGoogle Playに紐づいた購入履歴を復元できます。ペイウォール画面に「購入を復元する」ボタンを設置することが推奨されます（App Storeのガイドラインでも必須）。

## まとめ

RevenueCatとRorkの組み合わせは、個人開発者がモバイルアプリを収益化するための**最短・最強の手段**のひとつです。

今回解説した実装ポイントをまとめると以下のとおりです。

- RevenueCatダッシュボードでEntitlement・Product・Offeringを正しく設定する
- アプリ起動時に `Purchases.configure()` を1回だけ実行する
- `PurchaseContext` でサブスクリプション状態をアプリ全体で共有する
- ペイウォール画面と「購入を復元する」ボタンを必ず実装する
- Webhookを活用してバックエンドとの同期を自動化する

Rorkのプロンプト開発の速度と、RevenueCatの堅牢な課金管理を組み合わせることで、リリースから収益化まで驚くほど短期間で実現できます。まずはRevenueCatの無料プランでテスト実装から始めてみましょう。
