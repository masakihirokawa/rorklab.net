サブスクリプションアプリの成長において、「獲得コスト（CAC）を下げる」と同じくらい重要なのが「既存ユーザーをつなぎとめること」です。月間解約率（Churn Rate）が1%改善するだけで、LTV（顧客生涯価値）は数十%向上するケースも珍しくありません。

本記事では、RevenueCat が提供する高度な機能——オファーコード、Win-Back Offers、Cancellation Survey、Grace Period——を Rork Max アプリに組み込み、チャーンを抑制して LTV を最大化する実践手法を詳しく解説します。

## RevenueCat の解約防止エコシステムを理解する

まず、RevenueCat が解約対策に提供するツールを整理しておきましょう。

**Offerings（オファリング）と Packages（パッケージ）**
サブスクリプションの種類（月額・年額・生涯）を管理する単位です。Win-Back Offers もここで定義します。

**Promotional Offers（プロモーショナルオファー）**
Apple StoreKit 2 / Google Play Billing に対応した割引オファーです。既存・元契約者向けの特別価格を提示できます。

**Offer Codes（オファーコード）**
SNSキャンペーン・メルマガ・インフルエンサー経由で配布する無料/割引コード。一定期間のトライアルや割引サブスクリプションを提供できます。

**Grace Period（グレースピリオド）**
決済失敗時に一定期間（iOS: 最大16日、Android: 最大30日）サービス継続を許可する機能。Involuntary Churn（意図しない解約）を削減します。

**Webhooks & Event Streams**
「試用期間終了」「更新失敗」「解約」などのライフサイクルイベントをバックエンドで受信し、適切なフォローアップアクションを自動化できます。

## オファーコードの実装

### App Store Connect でオファーコードを作成する

App Store Connect の「App内課金」>「サブスクリプション」から、対象サブスクリプションを選択し、「オファーコード」タブで作成します。

- **オファータイプ**: 無料期間、割引、固定価格（Pay Up Front）の3種類
- **コード数**: 最大1,000コード/オファー（バッチ生成可能）
- **期間制限**: 終了日を設定して期間限定キャンペーンに活用

### Rork Max でコード入力UIを実装する

## Win-Back Offers（復帰オファー）の設計と実装

Win-Back Offers は、かつてサブスクリプションを保持していたが解約したユーザーを対象にした特別割引オファーです。Apple StoreKit 2 に対応した強力な機能です。

### App Store Connect での設定

1. 対象サブスクリプション → 「サブスクリプションオファー」→「Win-Backオファー」を作成
2. 以下のパラメータを設定します:
   - **資格**: 過去に購読していたユーザーのみ
   - **割引タイプ**: 無料期間（2週間〜1ヶ月が効果的）または大幅割引（50〜70%オフ）
   - **表示タイミング**: 解約から30日後〜90日後が最も有効

### RevenueCat 経由で Win-Back Offer を提示する

```typescript
import Purchases, {
  PurchasesOfferings,
  PurchasesPackage,
  PromotionalOffer
} from 'react-native-purchases';

// Win-Back対象ユーザーかどうかを確認
const checkWinBackEligibility = async (): Promise<boolean> => {
  const customerInfo = await Purchases.getCustomerInfo();

  // 現在のサブスクリプションがない & 過去に購読歴がある
  const hasActiveSubscription = Object.keys(
    customerInfo.entitlements.active
  ).length > 0;

  const hadPreviousSubscription =
    customerInfo.allPurchasedProductIdentifiers.includes('premium_monthly') ||
    customerInfo.allPurchasedProductIdentifiers.includes('premium_yearly');

  return !hasActiveSubscription && hadPreviousSubscription;
};

// Win-Back Offer を取得して提示
const presentWinBackOffer = async () => {
  try {
    const offerings = await Purchases.getOfferings();
    const winBackOffering = offerings.all['win_back'];

    if (!winBackOffering) {
      // Win-Back Offering が設定されていない場合は通常プランを表示
      presentStandardOffering();
      return;
    }

    const monthlyPackage = winBackOffering.monthly;
    if (!monthlyPackage) return;

    // プロモーショナルオファーを取得
    const promotionalOffer = await Purchases.getPromotionalOffer(
      monthlyPackage.product,
      monthlyPackage.product.discounts?.[0]
    );

    if (promotionalOffer) {
      // Win-Back 専用の割引価格で購入フローを開始
      const { customerInfo } = await Purchases.purchaseDiscountedPackage(
        monthlyPackage,
        promotionalOffer
      );

      if (customerInfo.entitlements.active['premium']) {
        // 購読再開成功 → お帰りウェルカム画面へ
        navigation.replace('WelcomeBack');
      }
    }
  } catch (e: any) {
    if (!e.userCancelled) {
      console.error('Win-Back purchase error:', e);
    }
  }
};
```

### Win-Back メール・プッシュ通知の設計

RevenueCat の Webhook で `EXPIRATION` イベントを受信したら、バックエンドで以下のシーケンスを実行します。

```typescript
// バックエンド (Hono on Cloudflare Workers)
import { Hono } from 'hono';

const app = new Hono();

app.post('/webhooks/revenuecat', async (c) => {
  const body = await c.req.json();
  const { event } = body;

  switch (event.type) {
    case 'EXPIRATION': {
      const userId = event.app_user_id;
      const productId = event.product_id;

      // 解約後 1日: 「ご解約のお知らせ」メール（感謝トーン）
      await scheduleEmail(userId, 'expiration_day1', { productId });

      // 解約後 7日: 「フィードバックお願い」メール + 短期割引オファー
      await scheduleEmail(userId, 'expiration_day7', {
        offerCode: 'COMEBACK30',  // 30%オフ1ヶ月
        expiresAt: new Date(Date.now() + 14 * 24 * 60 * 60 * 1000)
      });

      // 解約後 30日: Win-Back Offer 案内プッシュ通知
      await schedulePushNotification(userId, 'winback_day30');

      // 解約後 90日: 最終オファー（最大割引）
      await scheduleEmail(userId, 'expiration_day90', {
        offerCode: 'WINBACK50',  // 50%オフ2ヶ月
        isLastOffer: true
      });
      break;
    }
    case 'RENEWAL': {
      // 更新成功 → スケジュールされたWin-Backメールをキャンセル
      await cancelScheduledEmails(event.app_user_id);
      break;
    }
  }

  return c.json({ received: true });
});
```

## Grace Period（グレースピリオド）で Involuntary Churn を削減する

決済失敗による意図しない解約（Involuntary Churn）は、サブスクリプション収益の8〜15%を占めるとされています。Grace Period を設定することで、ユーザーに決済情報更新の猶予を与えられます。

### 設定手順

**iOS**: App Store Connect → App情報 → 「グレースピリオド」を有効化（最大16日）

**Android**: Google Play Console → 収益化 → 定期購入設定 → グレースピリオドを有効化（最大30日）

### アプリ内でグレースピリオドを検出する

```typescript
import Purchases, { CustomerInfo } from 'react-native-purchases';

const checkSubscriptionStatus = async () => {
  const customerInfo = await Purchases.getCustomerInfo();
  const premiumEntitlement = customerInfo.entitlements.all['premium'];

  if (!premiumEntitlement) {
    // 購読なし
    return { status: 'free' };
  }

  const isActive = premiumEntitlement.isActive;
  const willRenew = premiumEntitlement.willRenew;
  const billingIssue = premiumEntitlement.billingIssueDetectedAt;
  const gracePeriodExpiry = premiumEntitlement.gracePeriodExpiresDate;

  if (isActive && billingIssue) {
    // グレースピリオド中 → 決済更新を優しく促す
    return {
      status: 'grace_period',
      expiresAt: gracePeriodExpiry,
      daysRemaining: Math.ceil(
        (new Date(gracePeriodExpiry!).getTime() - Date.now()) /
        (1000 * 60 * 60 * 24)
      )
    };
  }

  if (isActive && !willRenew) {
    // 解約予告（期間終了後に失効）
    return {
      status: 'cancelling',
      expiresAt: premiumEntitlement.expirationDate
    };
  }

  if (isActive) {
    return { status: 'active' };
  }

  return { status: 'expired' };
};

// グレースピリオドバナー表示コンポーネント
const GracePeriodBanner = ({ daysRemaining }: { daysRemaining: number }) => (
  <View style={styles.warningBanner}>
    <Ionicons name="warning-outline" size={20} color="#FF9500" />
    <Text style={styles.warningText}>
      お支払い情報の更新が必要です。あと{daysRemaining}日でサービスが一時停止されます。
    </Text>
    <TouchableOpacity
      onPress={() => Linking.openURL('https://apps.apple.com/account/subscriptions')}
    >
      <Text style={styles.updateLink}>今すぐ更新</Text>
    </TouchableOpacity>
  </View>
);
```

## Cancellation Survey（解約理由アンケート）で離脱を防ぐ

解約ボタンを押したユーザーに理由を尋ね、理由に応じた引き留めオファーを提示することで、解約率を大幅に削減できます。

### 解約フローの設計

```typescript
type CancellationReason =
  | 'too_expensive'
  | 'not_using_enough'
  | 'missing_features'
  | 'technical_issues'
  | 'temporary_break'
  | 'other';

interface CancellationOffer {
  reason: CancellationReason;
  offerType: 'pause' | 'discount' | 'downgrade' | 'feedback';
  discount?: number;  // パーセント
  pauseDuration?: number;  // 日数
}

const CANCELLATION_OFFERS: Record<CancellationReason, CancellationOffer> = {
  too_expensive: {
    reason: 'too_expensive',
    offerType: 'discount',
    discount: 50,
  },
  not_using_enough: {
    reason: 'not_using_enough',
    offerType: 'pause',
    pauseDuration: 30,
  },
  missing_features: {
    reason: 'missing_features',
    offerType: 'feedback',
  },
  technical_issues: {
    reason: 'technical_issues',
    offerType: 'feedback',
  },
  temporary_break: {
    reason: 'temporary_break',
    offerType: 'pause',
    pauseDuration: 30,
  },
  other: {
    reason: 'other',
    offerType: 'discount',
    discount: 30,
  },
};

// 解約フロースクリーン
export const CancellationFlowScreen = () => {
  const [step, setStep] = useState<'survey' | 'offer' | 'confirmed'>('survey');
  const [selectedReason, setSelectedReason] = useState<CancellationReason | null>(null);

  const handleReasonSelect = (reason: CancellationReason) => {
    setSelectedReason(reason);
    setStep('offer');
  };

  const handleAcceptOffer = async () => {
    if (!selectedReason) return;
    const offer = CANCELLATION_OFFERS[selectedReason];

    if (offer.offerType === 'discount') {
      // 割引オファーを適用して解約をキャンセル
      await applyRetentionDiscount(offer.discount!);
    } else if (offer.offerType === 'pause') {
      // サブスクリプションを一時停止（RevenueCat Pausing）
      await pauseSubscription(offer.pauseDuration!);
    }

    navigation.replace('RetentionSuccess', { offerType: offer.offerType });
  };

  const handleConfirmCancellation = async () => {
    // 解約処理: App Store / Google Play の設定画面を開く
    if (Platform.OS === 'ios') {
      await Linking.openURL('https://apps.apple.com/account/subscriptions');
    } else {
      await Linking.openURL(
        'https://play.google.com/store/account/subscriptions'
      );
    }
    setStep('confirmed');
  };

  if (step === 'survey') {
    return (
      <View style={styles.container}>
        <Text style={styles.title}>解約の理由をお聞かせください</Text>
        {(
          [
            { key: 'too_expensive', label: '料金が高い' },
            { key: 'not_using_enough', label: 'あまり使っていない' },
            { key: 'missing_features', label: '欲しい機能がない' },
            { key: 'technical_issues', label: '技術的な問題がある' },
            { key: 'temporary_break', label: 'しばらく休憩したい' },
            { key: 'other', label: 'その他' },
          ] as { key: CancellationReason; label: string }[]
        ).map(({ key, label }) => (
          <TouchableOpacity
            key={key}
            style={styles.reasonButton}
            onPress={() => handleReasonSelect(key)}
          >
            <Text style={styles.reasonText}>{label}</Text>
            <Ionicons name="chevron-forward" size={16} color="#999" />
          </TouchableOpacity>
        ))}
      </View>
    );
  }

  if (step === 'offer') {
    const offer = CANCELLATION_OFFERS[selectedReason!];
    return (
      <RetentionOfferCard
        offer={offer}
        onAccept={handleAcceptOffer}
        onDecline={handleConfirmCancellation}
      />
    );
  }

  return <CancellationConfirmedScreen />;
};
```

## LTV 分析ダッシュボードの構築

RevenueCat のデータを活用して、コホート別 LTV・チャーンレート・MRR（月次経常収益）をトラッキングします。

### RevenueCat Charts API でデータを取得する

```typescript
// バックエンド: RevenueCat REST API からデータ集計
const fetchRevenueCatMetrics = async () => {
  const headers = {
    Authorization: `Bearer ${process.env.REVENUECAT_SECRET_KEY}`,
    'Content-Type': 'application/json',
  };

  // MRR（月次経常収益）
  const mrrResponse = await fetch(
    'https://api.revenuecat.com/v1/charts/mrr',
    { headers }
  );
  const mrrData = await mrrResponse.json();

  // チャーンレート
  const churnResponse = await fetch(
    'https://api.revenuecat.com/v1/charts/churned_subscribers',
    { headers }
  );
  const churnData = await churnResponse.json();

  // コンバージョン率（Trial → Paid）
  const conversionResponse = await fetch(
    'https://api.revenuecat.com/v1/charts/trial_conversion',
    { headers }
  );
  const conversionData = await conversionResponse.json();

  return { mrr: mrrData, churn: churnData, conversion: conversionData };
};
```

### チャーン削減の目標設定

効果的なチャーン対策を実施した場合の期待値を整理しておきましょう。

Grace Period を有効化することで、Involuntary Churn（決済失敗による解約）を平均30〜50%削減できます。

Cancellation Survey + 引き留めオファーにより、Voluntary Churn（意図的な解約）の10〜20%を防止できます。

Win-Back Campaigns では、解約後30日以内のユーザーで10〜25%が復帰するケースが報告されています。

これらを組み合わせると、月間チャーンレートが3%のアプリが2%台前半まで低下し、12ヶ月後のユーザー残存率が約12%向上します。

## まとめ：チャーン対策は「収益の守り」

新規ユーザーを増やすことも重要ですが、既存ユーザーを守ることのコスパは圧倒的です。本記事で紹介した施策を段階的に導入してみてください。

まずは **Grace Period の有効化**から始めましょう。設定数分で完了し、即効性があります。次に **Cancellation Survey** を実装して解約理由のデータを集めます。データが蓄積されたら理由に合わせた **引き留めオファー** を設計し、最終的に **Win-Back キャンペーン** で離脱ユーザーの復帰を狙います。

RevenueCat はこれらを一元管理できる優れたプラットフォームです。Rork Max で構築したアプリに積極的に組み込み、持続可能な収益基盤を築いていきましょう。
