## はじめに — なぜサブスク管理アプリが求められるのか

Netflix、Spotify、Adobe、iCloud、ジムの月会費……。いつの間にか増えていくサブスクリプションサービスの支出を正確に把握できている人は、意外と少ないのではないでしょうか。調査によると、平均的なユーザーは月に12〜15件のサブスクリプションを利用しており、そのうち約3割は「使っていないのに解約を忘れている」サービスだといわれています。

この記事では、Rork を使ってサブスクリプション管理アプリをゼロから構築します。完成するアプリの主な機能は次の通りです。

- サブスクリプション一覧の登録・編集・削除
- 月額・年額の支出グラフ表示（円グラフ・棒グラフ）
- 次回更新日のリマインダー通知
- カテゴリ別のフィルタリング

対象読者は、Rork の基本操作を理解しており、中級レベルのアプリ開発に挑戦したい方です。React Native や Expo の前提知識は不要ですが、あると理解がスムーズです。

## 前提条件と環境準備

開発を始める前に、以下の環境を確認してください。

- **Rork アカウント**: 無料プランでも開発は可能ですが、実機テストには Rork Max プランが便利です
- **ブラウザ**: Chrome / Safari / Edge の最新版
- **（任意）Rork Companion アプリ**: 実機プレビュー用にスマートフォンにインストールしておくと便利です

Rork のダッシュボードにアクセスし、「New Project」から新しいプロジェクトを作成しましょう。プロジェクト名は `SubTracker` とします。

## ステップ 1 — データモデルの設計

まず、サブスクリプション情報を管理するデータ構造を設計します。Rork の AI プロンプトに以下のように入力してみましょう。

```
サブスクリプション管理アプリのデータモデルを作成してください。
各サブスクリプションには、名前、金額、通貨、請求サイクル（月額/年額）、
カテゴリ、次回更新日、アイコンカラーを含めてください。
```

Rork が生成する TypeScript の型定義は、おおよそ以下のようになります。

```typescript
// サブスクリプションのデータ型
type Subscription = {
  id: string;
  name: string;           // サービス名（例: "Netflix"）
  amount: number;         // 金額
  currency: "JPY" | "USD"; // 通貨
  cycle: "monthly" | "yearly"; // 請求サイクル
  category: string;       // カテゴリ（例: "エンタメ", "仕事"）
  nextBillingDate: string; // 次回請求日（ISO 8601）
  color: string;          // テーマカラー（例: "#E50914"）
  isActive: boolean;      // 有効/休止フラグ
  createdAt: string;
};
// 期待される利用例:
// { id: "1", name: "Netflix", amount: 1490, currency: "JPY",
//   cycle: "monthly", category: "エンタメ", nextBillingDate: "2026-04-15",
//   color: "#E50914", isActive: true, createdAt: "2026-03-29" }
```

このデータモデルがアプリ全体の基盤になります。カテゴリは自由入力でも構いませんが、あらかじめ「エンタメ」「仕事」「ライフスタイル」「学習」「その他」の5つをデフォルト値として用意しておくと、ユーザーにとって使いやすくなります。

## ステップ 2 — ホーム画面の構築

ホーム画面では、月間の合計支出と登録済みサブスクリプションの一覧を表示します。Rork に次のプロンプトを入力します。

```
ホーム画面を作成してください。
上部に月間合計支出を大きく表示し、
その下にサブスクリプションのカード一覧をスクロールリストで表示します。
各カードにはサービス名、金額、次回請求日、カテゴリバッジを含めてください。
```

ここで重要なのは、金額表示のフォーマットです。日本円は小数点なし、ドルは小数点2桁で表示する必要があります。

```typescript
// 通貨に応じたフォーマット関数
const formatCurrency = (amount: number, currency: string): string => {
  return new Intl.NumberFormat(
    currency === "JPY" ? "ja-JP" : "en-US",
    { style: "currency", currency }
  ).format(amount);
};

// 使用例
console.log(formatCurrency(1490, "JPY")); // → "￥1,490"
console.log(formatCurrency(9.99, "USD")); // → "$9.99"
```

月間合計を計算する際は、年額サブスクリプションを12で割って月額換算することを忘れないでください。

```typescript
// 月額換算で合計支出を計算
const calculateMonthlyTotal = (subs: Subscription[]): number => {
  return subs
    .filter((s) => s.isActive)
    .reduce((total, sub) => {
      const monthly = sub.cycle === "yearly"
        ? sub.amount / 12
        : sub.amount;
      return total + monthly;
    }, 0);
};
// 期待される出力例:
// Netflix ¥1,490(月額) + Adobe ¥28,776(年額→月額¥2,398) = ¥3,888
```

## ステップ 3 — 登録フォームの実装

新しいサブスクリプションを追加するためのフォーム画面を作ります。

```
サブスクリプション登録フォームを作成してください。
入力項目: サービス名、金額、通貨（JPY/USD切替）、請求サイクル（月額/年額）、
カテゴリ（ドロップダウン）、次回請求日（日付ピッカー）、テーマカラー（カラーピッカー）。
バリデーション付きで、入力不備があればエラーメッセージを表示してください。
```

フォームのバリデーションでは、以下のルールを適用しましょう。

- サービス名: 1〜50文字（必須）
- 金額: 0より大きい数値（必須）
- 次回請求日: 今日以降の日付のみ許可

Rork が生成するフォームコンポーネントは、React Hook Form に似たパターンで状態管理されます。入力値が変わるたびに即座にバリデーションが走る「リアルタイムバリデーション」を有効にすると、ユーザー体験が向上します。

## ステップ 4 — 支出グラフの可視化

データの可視化は、このアプリの核心部分です。Rork で以下のプロンプトを使ってグラフ画面を構築します。

```
支出分析画面を作成してください。
1. カテゴリ別の支出割合を円グラフで表示
2. 過去6ヶ月の支出推移を棒グラフで表示
3. 月額合計と年間見積もりを数値で表示してください。
```

React Native でグラフを描画するには、`react-native-chart-kit` や `victory-native` が定番ライブラリです。Rork は適切なライブラリを自動選択してくれますが、カスタマイズしたい場合は明示的に指定することもできます。

```typescript
// カテゴリ別データを円グラフ用に変換
const getCategoryChartData = (subs: Subscription[]) => {
  const categoryMap = new Map<string, number>();

  subs.filter((s) => s.isActive).forEach((sub) => {
    const monthly = sub.cycle === "yearly"
      ? sub.amount / 12
      : sub.amount;
    const current = categoryMap.get(sub.category) || 0;
    categoryMap.set(sub.category, current + monthly);
  });

  return Array.from(categoryMap.entries()).map(([name, amount]) => ({
    name,
    amount: Math.round(amount),
    color: CATEGORY_COLORS[name] || "#999999",
  }));
};

// 期待される出力:
// [
//   { name: "エンタメ", amount: 2980, color: "#FF6384" },
//   { name: "仕事",    amount: 4580, color: "#36A2EB" },
//   { name: "学習",    amount: 1200, color: "#FFCE56" },
// ]
```

グラフ画面では、「もっとも高額なサブスクリプション TOP 3」や「来月の予想支出」なども表示すると、ユーザーにとって実用的な画面になります。

データの可視化についてさらに詳しいテクニックを知りたい方は、[Rork × Victory Native でインタラクティブなチャートダッシュボードを作る](/articles/rork-dev/rork-victory-native-charts-dashboard) も参考になります。

## ステップ 5 — リマインダー通知の設定

サブスクリプションの更新日が近づいたときにプッシュ通知を送る機能は、このアプリの大きな付加価値です。

```
各サブスクリプションの次回更新日の3日前にローカル通知を送る機能を
実装してください。通知にはサービス名と金額を含めてください。
```

Expo の `expo-notifications` を使ったローカル通知のスケジューリングは、以下のパターンで実装できます。

```typescript
import * as Notifications from "expo-notifications";

// リマインダー通知をスケジュール
const scheduleReminder = async (sub: Subscription) => {
  const billingDate = new Date(sub.nextBillingDate);
  const reminderDate = new Date(billingDate);
  reminderDate.setDate(reminderDate.getDate() - 3); // 3日前

  // 過去の日付ならスキップ
  if (reminderDate <= new Date()) return;

  await Notifications.scheduleNotificationAsync({
    content: {
      title: "サブスク更新リマインダー",
      body: `${sub.name} の更新日が近づいています（${formatCurrency(sub.amount, sub.currency)}）`,
    },
    trigger: { date: reminderDate },
  });
};
// 期待される動作:
// Netflix（次回更新: 4/15）→ 4/12 に「Netflixの更新日が近づいています（¥1,490）」と通知
```

通知の許可ダイアログは、アプリの初回起動時ではなく、ユーザーが最初のサブスクリプションを登録した直後に表示するのがベストプラクティスです。文脈のない許可リクエストは拒否されやすいためです。

## ステップ 6 — カテゴリフィルターとソート機能

登録数が増えてくると、一覧をフィルタリング・ソートする機能が欠かせません。

```
ホーム画面のサブスクリプション一覧に、カテゴリフィルター（水平スクロールのチップUI）と
ソート機能（金額順、更新日順、名前順）を追加してください。
```

フィルターUIは、画面上部に横スクロール可能なチップ（タグ）を配置するパターンが直感的です。「すべて」を含む各カテゴリチップをタップすると、リストが即座にフィルタリングされます。

```typescript
// フィルターとソートを適用
const getFilteredSubscriptions = (
  subs: Subscription[],
  category: string | null,
  sortBy: "amount" | "date" | "name"
): Subscription[] => {
  let filtered = category
    ? subs.filter((s) => s.category === category)
    : subs;

  return filtered.sort((a, b) => {
    switch (sortBy) {
      case "amount":
        return b.amount - a.amount; // 高額順
      case "date":
        return new Date(a.nextBillingDate).getTime()
             - new Date(b.nextBillingDate).getTime(); // 更新日が近い順
      case "name":
        return a.name.localeCompare(b.name); // 名前順
      default:
        return 0;
    }
  });
};
```

## よくあるエラーと対処法

サブスク管理アプリの開発中に遭遇しやすい問題と、その解決方法をまとめます。

**日付ピッカーがタイムゾーンでずれる**: `new Date()` はデバイスのローカルタイムゾーンを使いますが、ISO文字列のパースはUTCとして扱われることがあります。日付の比較には `toLocaleDateString()` や `date-fns` ライブラリの `parseISO` を使いましょう。

**通知が届かない（iOS）**: iOS ではアプリがバックグラウンドにある場合、通知の許可が必須です。`Notifications.getPermissionsAsync()` で現在の許可状態を確認し、未許可なら `requestPermissionsAsync()` を呼び出してください。

**グラフが空で表示される**: データが0件の場合にグラフコンポーネントがエラーを起こすケースがあります。データが空のときは「まだサブスクリプションが登録されていません」というプレースホルダーを表示するよう、条件分岐を追加しましょう。

## FAQ

**Q: AsyncStorage のデータ容量に制限はありますか？**

A: AsyncStorage の推奨上限は約6MBです。サブスクリプションのデータは1件あたり数百バイト程度なので、数千件登録しても容量の心配はありません。ただし、より堅牢なデータ永続化が必要な場合は、Supabase との連携を検討してください。

**Q: 為替レートを自動で取得して統一通貨で表示できますか？**

A: はい、可能です。為替レートAPIとして [exchangerate-api.com](https://www.exchangerate-api.com/) などの無料APIを使い、日次で為替レートを取得してキャッシュする方法が一般的です。ただし、無料プランにはリクエスト制限があるため、レートのキャッシュ間隔を24時間程度に設定するのが現実的です。

**Q: 家族やパートナーとサブスク情報を共有する機能は追加できますか？**

A: Supabase の Row Level Security（RLS）を活用すれば、特定のユーザー同士でデータを共有する機能を実装できます。共有グループの概念を導入し、同じグループに属するユーザーが互いのサブスクリプション情報を閲覧できるようにする設計が効果的です。

**Q: このアプリを App Store に公開するには何が必要ですか？**

A: Rork Max プランを利用すれば、2クリックで App Store への申請が可能です。Apple Developer Program（年間 $99）への登録が前提となります。審査に通過するためのチェックポイントについては、[Rork アプリの App Store 審査完全攻略ガイド](/articles/rork-business/rork-app-store-review-complete-checklist) を参照してください。
