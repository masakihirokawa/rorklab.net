## RorkアプリでStripe決済を使う意義

AIアプリビルダーでアプリを作ることができても、収益化の仕組みを組み込まなければビジネスとして成立しません。Rorkで構築したモバイルアプリに**Stripe**を統合することで、単発購入・サブスクリプション・マーケットプレイス決済など、あらゆる課金モデルを実現できます。

本記事では、Rorkのプロンプトベース開発との親和性が高いStripe連携の方法を、実際のコード例を交えながら解説します。

---

## Stripeとは何か

Stripeは世界180カ国以上で使われている決済インフラです。クレジットカード・デビットカード・Apple Pay・Google Payなど多様な支払い手段に対応しており、React Native（Rorkが使用するフレームワーク）向けの公式SDKも提供されています。

Rorkで生成されたアプリはReact Nativeベースのため、`@stripe/stripe-react-native` パッケージを利用した統合が最適です。

---

## 事前準備

### 1. Stripeアカウントの作成

[Stripe Dashboard](https://dashboard.stripe.com) でアカウントを作成し、以下の情報を取得します。

- **Publishable Key**（公開キー）: `pk_live_XXXX` または `pk_test_XXXX`
- **Secret Key**（秘密キー）: サーバーサイドのみで使用

### 2. バックエンドの用意

Stripeの決済フローでは、**PaymentIntent**をサーバー側で作成する必要があります。Rorkアプリと組み合わせる場合、以下のいずれかを選択してください。

- **Supabase Edge Functions**（推奨）
- Firebase Cloud Functions
- Cloudflare Workers

## Rorkプロンプトで決済機能を追加する

Rorkの強みは自然言語プロンプトでコードを生成できる点です。以下のようなプロンプトを使いましょう。

```
このアプリにStripe決済を追加してください。
ユーザーが「購入する」ボタンを押すと、金額1,200円の
PaymentSheetが開き、カード情報を入力して支払いできるようにしてください。
バックエンドはSupabase Edge Functionsを使います。
```

Rorkはこのプロンプトから以下を自動生成します。

1. `@stripe/stripe-react-native` のインポートと設定
2. `StripeProvider` のラッパーコンポーネント
3. `PaymentSheet` の初期化ロジック
4. 決済完了後の処理（UIの更新など）

## 実装コード例

### アプリ側（React Native / Rork生成コード）

```tsx
import { StripeProvider, usePaymentSheet } from "@stripe/stripe-react-native";
import { useState } from "react";
import { Button, View, Text, StyleSheet } from "react-native";

const PUBLISHABLE_KEY = "pk_live_YOUR_KEY";

function CheckoutScreen() {
  const { initPaymentSheet, presentPaymentSheet } = usePaymentSheet();
  const [loading, setLoading] = useState(false);

  const initializePayment = async () => {
    setLoading(true);
    try {
      // バックエンドからPaymentIntentを取得
      const response = await fetch(
        "https://YOUR_PROJECT.supabase.co/functions/v1/create-payment-intent",
        {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ amount: 1200, currency: "jpy" }),
        }
      );
      const { paymentIntent, ephemeralKey, customer } = await response.json();

      const { error } = await initPaymentSheet({
        customerId: customer,
        customerEphemeralKeySecret: ephemeralKey,
        paymentIntentClientSecret: paymentIntent,
        merchantDisplayName: "My Rork App",
        defaultBillingDetails: { address: { country: "JP" } },
      });

      if (!error) {
        const { error: presentError } = await presentPaymentSheet();
        if (!presentError) {
          alert("決済が完了しました！");
        }
      }
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.price}>¥1,200</Text>
      <Button
        title={loading ? "処理中..." : "今すぐ購入"}
        onPress={initializePayment}
        disabled={loading}
      />
    </View>
  );
}

export default function App() {
  return (
    <StripeProvider publishableKey={PUBLISHABLE_KEY}>
      <CheckoutScreen />
    </StripeProvider>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: "center", justifyContent: "center" },
  price: { fontSize: 32, fontWeight: "bold", marginBottom: 20 },
});
```

### バックエンド側（Supabase Edge Function）

```typescript
// supabase/functions/create-payment-intent/index.ts
import Stripe from "https://esm.sh/stripe@14.0.0";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!, {
  apiVersion: "2024-06-20",
});

Deno.serve(async (req) => {
  const { amount, currency } = await req.json();

  const customer = await stripe.customers.create();
  const ephemeralKey = await stripe.ephemeralKeys.create(
    { customer: customer.id },
    { apiVersion: "2024-06-20" }
  );
  const paymentIntent = await stripe.paymentIntents.create({
    amount,
    currency,
    customer: customer.id,
    automatic_payment_methods: { enabled: true },
  });

  return new Response(
    JSON.stringify({
      paymentIntent: paymentIntent.client_secret,
      ephemeralKey: ephemeralKey.secret,
      customer: customer.id,
    }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

## サブスクリプション課金を実装する

月額・年額サブスクリプションは、**RevenueCat**と組み合わせると実装が格段に簡単になります。RevenueCatはApp StoreとGoogle Playのサブスクリプション管理を統一するSDKで、RorkのプロンプトでもRevenueCat統合を指示できます。

```
RevenueCatを使って月額500円のプレミアムプランを実装してください。
未登録ユーザーには機能を制限し、購入後にフル機能を解放してください。
```

RevenueCatを使うメリット:

- App Store / Google Playの課金を1つのAPIで管理
- レシート検証の自動化
- 収益ダッシュボードが利用可能
- Webhook経由でSupabaseとの同期が容易

## Rork Maxでの決済実装

Rork MaxはネイティブSwift/SwiftUIアプリを生成するため、決済実装のアプローチが異なります。

- **Apple Pay / StoreKit 2**: ネイティブApple決済に完全対応
- **Stripe iOS SDK**: Rork Maxが生成したSwiftコードに`Stripe`ライブラリを追加可能
- **iAP（In-App Purchase）**: App Storeの審査を経たアプリ内課金にもStoreKit 2で対応

Rork Maxプロンプト例:

```
SwiftUIアプリにApple Payを実装してください。
商品価格は¥1,500で、支払い成功後にコンテンツをアンロックする仕組みにしてください。
```

## セキュリティのベストプラクティス

Stripe統合において特に重要なのが**Secret Keyの管理**です。

| 項目 | 推奨対応 |
|------|---------|
| Secret Keyの保存 | 環境変数（`.env`）のみ。コードに直書き禁止 |
| Publishable Keyのみクライアントに公開 | `pk_live_` または `pk_test_` で始まるキー |
| Webhookの署名検証 | Stripeダッシュボードで取得したWebhook Secretを使用 |
| テスト環境と本番環境の分離 | `pk_test_` と `pk_live_` を環境ごとに切り替え |

RorkでAPIキーを管理する場合は、`.env`ファイルや環境変数設定のプロンプトを活用しましょう。

```
Stripe Secret Keyを環境変数STRIPE_SECRET_KEYとして読み込み、
ハードコードされないようにしてください。
```

## 収益化チェックリスト

Rorkアプリを収益化する前に確認すべき項目です。

- [ ] Stripeアカウントが本人確認済み（KYC完了）
- [ ] テスト環境でエンドツーエンドの決済フローを確認
- [ ] Webhook設定（支払い完了・失敗・返金の通知）
- [ ] エラーハンドリング（ネットワークエラー・カード拒否）
- [ ] App Store審査対策（In-App PurchaseはApple経由が必須）
- [ ] プライバシーポリシー・利用規約に決済条件を明記

## まとめ

RorkとStripeを組み合わせることで、AIで生成したアプリに本格的な決済機能を迅速に追加できます。ポイントは以下の3点です。

1. **バックエンド（Supabase Edge Functions等）でPaymentIntentを作成**する
2. **クライアント側はStripe React Native SDKのPaymentSheetを使う**と最小実装で決済UIが完成
3. **サブスクリプションはRevenueCatを活用**すると管理が格段に楽になる

Rork Maxを使えばネイティブSwiftのApple Pay統合も同様のプロンプトベースのアプローチで実現できます。個人開発者でも収益化の仕組みを素早く組み込み、マネタイズへの道を開きましょう。
