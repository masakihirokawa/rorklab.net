## はじめに — アプリの開発だけでなく運用もAIに任せる

Rork を使えばモバイルアプリの開発は劇的に高速化されます。しかし、アプリ公開後の**運用**は依然として大きな労力を要する領域です。コンテンツの更新、ユーザーからの問い合わせ対応、クラッシュレポートの分析、ストアレビューへの返信——個人開発者にとって、これらを手動で行い続けるのは現実的ではありません。

本記事では、Rork で構築したモバイルアプリの運用を自律AIエージェントで自動化するアーキテクチャを設計します。ローコードツール（Zapier, Make.com 等）ではなく、TypeScript ベースの耐久性タスク実行基盤を使い、信頼性の高い24時間運用を実現する方法を体系的に解説します。

## ローコードツールの限界と卒業の判断基準

### いつローコードを卒業すべきか

以下のいずれかに該当する場合、コードベースの自動化に移行する検討を始めるべきです。

- Zapier や Make.com の実行回数制限に達している
- ワークフローの条件分岐が10段階以上になっている
- エラー発生時のリカバリーを細かく制御したい
- AIモデルの切り替えやプロンプトの動的変更が必要
- 複数のAIモデルを組み合わせた処理がある

### TypeScript ベースの利点

- **完全な制御**: 条件分岐、リトライ、フォールバックを自由に記述
- **型安全性**: TypeScript の型チェックでランタイムエラーを防止
- **テスタビリティ**: ユニットテスト・統合テストが書ける
- **バージョン管理**: Gitで全ての変更履歴を追跡
- **AIモデル非依存**: Claude、Gemini、GPT を同一コードベースで切り替え可能

## 自律運用の3つの柱

### 柱1: コンテンツ自動更新エージェント

アプリ内のコンテンツ（ニュース、お知らせ、おすすめ商品など）を定期的に更新するエージェントです。

```typescript
import { schedules } from "@trigger.dev/sdk/v3";
import Anthropic from "@anthropic-ai/sdk";

const anthropic = new Anthropic();

// 毎朝8時にアプリ内コンテンツを更新
export const dailyContentUpdate = schedules.task({
  id: "app-content-updater",
  cron: "0 8 * * *",
  maxDuration: 3600,
  run: async () => {
    // Step 1: 最新トレンドを収集
    const trends = await fetchLatestTrends();

    // Step 2: AIでコンテンツを生成
    const content = await anthropic.messages.create({
      model: "claude-sonnet-4-6",
      max_tokens: 2048,
      messages: [{
        role: "user",
        content: `以下のトレンドデータに基づいて、
アプリのホーム画面に表示する「今日のおすすめ」を3件生成してください。
各項目はタイトル（20文字以内）と説明文（50文字以内）で構成します。
トレンドデータ: ${JSON.stringify(trends)}`
      }]
    });

    // Step 3: APIに投稿
    const recommendations = parseRecommendations(
      content.content[0].type === "text" ? content.content[0].text : ""
    );
    await updateAppContent(recommendations);

    // Step 4: 結果をSlackに通知
    await notifySlack(`コンテンツ更新完了: ${recommendations.length}件`);

    return { updated: recommendations.length };
  }
});
```

### 柱2: ユーザーサポート自動化エージェント

ストアレビューやアプリ内の問い合わせに自動対応するエージェントです。

```typescript
import { task } from "@trigger.dev/sdk/v3";

// ストアレビューへの自動返信
export const reviewResponder = task({
  id: "store-review-responder",
  run: async (payload: {
    platform: "ios" | "android";
    rating: number;
    reviewText: string;
    reviewerName: string;
  }) => {
    // 低評価レビュー（3以下）は人間にエスカレーション
    if (payload.rating <= 3) {
      const analysis = await anthropic.messages.create({
        model: "claude-sonnet-4-6",
        max_tokens: 1024,
        messages: [{
          role: "user",
          content: `以下のアプリレビューを分析し、問題のカテゴリ（バグ/UX/機能要望/その他）と
推奨対応を提案してください:
評価: ${payload.rating}/5
レビュー: ${payload.reviewText}`
        }]
      });

      // Slackのサポートチャンネルに転送
      await notifySlack({
        channel: "#app-support",
        text: `低評価レビュー検知\n${analysis.content[0].type === "text" ? analysis.content[0].text : ""}`
      });

      return { action: "escalated", rating: payload.rating };
    }

    // 高評価レビューへの自動お礼返信
    const reply = await anthropic.messages.create({
      model: "claude-sonnet-4-6",
      max_tokens: 512,
      messages: [{
        role: "user",
        content: `以下のアプリレビューに対する感謝の返信を生成してください。
丁寧で温かいトーンで、具体的なフィードバックに触れてください。
150文字以内で簡潔に。
レビュー: ${payload.reviewText}`
      }]
    });

    // App Store Connect / Google Play Console APIで返信
    await postStoreReply({
      platform: payload.platform,
      replyText: reply.content[0].type === "text" ? reply.content[0].text : ""
    });

    return { action: "auto_replied", rating: payload.rating };
  }
});
```

### 柱3: パフォーマンス監視エージェント

アプリのクラッシュレポートやパフォーマンスメトリクスを監視し、問題を自動検出するエージェントです。

```typescript
import { schedules } from "@trigger.dev/sdk/v3";

// 1時間ごとにパフォーマンスチェック
export const performanceMonitor = schedules.task({
  id: "app-performance-monitor",
  cron: "0 * * * *",
  run: async () => {
    // Step 1: クラッシュレポートを取得
    const crashes = await fetchCrashReports({ period: "1h" });

    // 急増検知（前の時間帯の2倍以上）
    const previousCrashes = await fetchCrashReports({
      period: "1h",
      offset: "1h"
    });

    if (crashes.length > previousCrashes.length * 2 && crashes.length > 5) {
      // Step 2: AIでクラッシュ原因を分析
      const analysis = await anthropic.messages.create({
        model: "claude-sonnet-4-6",
        max_tokens: 2048,
        messages: [{
          role: "user",
          content: `以下のクラッシュレポートを分析し、
共通の原因パターンと推奨される修正方針を提案してください:
${crashes.map(c => c.stackTrace).join("\n---\n")}`
        }]
      });

      // 緊急通知
      await notifySlack({
        channel: "#app-alerts",
        text: `クラッシュ急増検知 (${crashes.length}件/h)\n${analysis.content[0].type === "text" ? analysis.content[0].text : ""}`
      });
    }

    // Step 3: メトリクスをダッシュボードに記録
    await recordMetrics({
      crashes: crashes.length,
      timestamp: new Date().toISOString()
    });

    return { crashCount: crashes.length, alert: crashes.length > previousCrashes.length * 2 };
  }
});
```

## エスカレーション設計

自律運用で最も重要なのは「いつ人間に引き継ぐか」の設計です。

### 自動エスカレーションのトリガー

- ストアレビューが星3以下
- クラッシュ件数が直前1時間の2倍以上
- ユーザーからの問い合わせに「返金」「法的」「個人情報」を含む
- AI分析の信頼度が閾値（70%）を下回る

### エスカレーション時の情報提供

人間に引き継ぐ際、AIエージェントは以下の情報を必ず添付します。

- 問題の要約（1行）
- カテゴリ分類（バグ/UX/機能要望/セキュリティ/法的）
- 推奨対応（具体的なアクション案）
- 関連する過去の類似ケース

## よくある質問

### Q. Rork で作ったアプリの構造を変えずに自律運用を追加できますか？

はい。自律運用エージェントはアプリ本体とは独立したバックエンドプロセスとして動作します。アプリのコードを変更する必要はなく、APIエンドポイントやWebhookを介して連携します。

### Q. 複数のアプリを同じ基盤で運用できますか？

はい。Trigger.dev のタスクはアプリごとにIDを分けて定義できるため、複数アプリの運用タスクを単一のインフラで管理できます。

### Q. AIモデルのコストが心配です。節約方法はありますか？

定型的な分析（クラッシュの既知パターン検出など）はルールベースで先にフィルタリングし、AIはルールで処理できないケースのみに使用する「関心の分離」を適用してください。これにより、API呼び出し回数を50〜70%削減できます。
