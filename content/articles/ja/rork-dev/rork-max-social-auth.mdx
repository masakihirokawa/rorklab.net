## Rork Max ソーシャルログイン実装ガイド

ユーザーが複数のアプリケーションで認証情報を管理することの煩雑さは、登録放棄率の増加につながります。ソーシャルログイン機能により、ユーザーは既存の Google、Apple、GitHub アカウントで簡単にログインでき、ユーザー体験が劇的に向上します。

本ガイドでは、Rork Max でこれら主要なソーシャルプロバイダーの認証を実装する方法を詳細に解説します。

## OAuth 2.0 の基本理解

### ソーシャル認証の仕組み

OAuth 2.0 は、ユーザーが自身のパスワードを第三者に与えることなく、リソースへのアクセスを許可できるプロトコルです。

基本的なフロー：
1. ユーザーが「Google でログイン」をクリック
2. Google の認可画面にリダイレクト
3. ユーザーが認可
4. アプリが認可コードを受け取る
5. バックエンド が認可コードをトークンに交換
6. ユーザーセッションを作成

## Google ログイン実装

### Google Sign-In の統合

Google ログインは最も広く使われるソーシャルプロバイダーです。

```jsx
import { GoogleSignin, statusCodes } from '@react-native-google-signin/google-signin';
import { useState } from 'react';

// GoogleSignin の初期化
GoogleSignin.configure({
  webClientId: process.env.GOOGLE_WEB_CLIENT_ID,
  iosClientId: process.env.GOOGLE_IOS_CLIENT_ID,
  androidClientId: process.env.GOOGLE_ANDROID_CLIENT_ID,
  scopes: ['profile', 'email'],
});

export function GoogleLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);
  const [isLoading, setIsLoading] = useState(false);

  const handleGoogleLogin = async () => {
    setIsLoading(true);
    try {
      // Google ログイン実行
      await GoogleSignin.hasPlayServices();
      const userInfo = await GoogleSignin.signIn();

      console.log(`Google ログイン成功: ${userInfo.user.email}`);
      // 期待出力: Google ログイン成功: user@gmail.com

      // バックエンドに ID トークンを送信
      const idToken = userInfo.idToken;
      const backendResponse = await fetch('/api/auth/google', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ idToken }),
      });

      const { sessionToken } = await backendResponse.json();

      // セッションを保存
      localStorage.setItem('sessionToken', sessionToken);
      setUserInfo(userInfo.user);
    } catch (error) {
      if (error.code === statusCodes.SIGN_IN_CANCELLED) {
        console.log('ユーザーがログインをキャンセルしました');
      } else if (error.code === statusCodes.IN_PROGRESS) {
        console.log('ログイン処理中...');
      } else if (error.code === statusCodes.PLAY_SERVICES_NOT_AVAILABLE) {
        console.error('Google Play Services が利用不可');
      } else {
        console.error('ログインエラー:', error);
      }
    } finally {
      setIsLoading(false);
    }
  };

  const handleLogout = async () => {
    try {
      await GoogleSignin.signOut();
      localStorage.removeItem('sessionToken');
      setUserInfo(null);
      console.log('ログアウト完了');
      // 期待出力: ログアウト完了
    } catch (error) {
      console.error('ログアウトエラー:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleGoogleLogin} disabled={isLoading}>
          {isLoading ? 'ログイン中...' : 'Google でログイン'}
        </button>
      ) : (
        <div>
          <p>ログイン済み: {userInfo.email}</p>
          <img src={userInfo.photo} alt="Profile" style={{ width: '50px', borderRadius: '50%' }} />
          <button onClick={handleLogout}>ログアウト</button>
        </div>
      )}
    </div>
  );
}
```

## Apple Sign-In 実装

### iOS での Sign in with Apple

iOS 13 以降では、App Store のアプリ審査に Sign in with Apple の実装が推奨される場合があります。

```jsx
import { appleAuth, AppleAuthRequestScope, AppleAuthRequestOperation } from '@react-native-apple-authentication';
import { useState } from 'react';

export function AppleLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);

  const handleAppleLogin = async () => {
    try {
      // Apple Sign-In フローを実行
      const appleAuthRequestResponse = await appleAuth.performRequest({
        requestedOperation: AppleAuthRequestOperation.LOGIN,
        requestedScopes: [
          AppleAuthRequestScope.EMAIL,
          AppleAuthRequestScope.FULL_NAME,
        ],
      });

      // ユーザーが認可をキャンセル
      if (appleAuthRequestResponse.state === appleAuth.STATE.CANCELLED) {
        console.log('Apple ログインがキャンセルされました');
        return;
      }

      const { identityToken, user } = appleAuthRequestResponse;

      console.log(`Apple ログイン成功: ${user?.email}`);
      // 期待出力: Apple ログイン成功: user@example.com

      // バックエンドに ID トークンを送信
      const backendResponse = await fetch('/api/auth/apple', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ identityToken }),
      });

      const { sessionToken } = await backendResponse.json();
      localStorage.setItem('sessionToken', sessionToken);

      setUserInfo({
        email: user?.email,
        name: `${user?.name?.firstName || ''} ${user?.name?.lastName || ''}`,
      });
    } catch (error) {
      console.error('Apple ログインエラー:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleAppleLogin} style={{ backgroundColor: '#000', color: '#fff', padding: '10px 20px' }}>
          Apple でサインイン
        </button>
      ) : (
        <div>
          <p>ログイン済み: {userInfo.email}</p>
          <button onClick={() => {
            localStorage.removeItem('sessionToken');
            setUserInfo(null);
          }}>ログアウト</button>
        </div>
      )}
    </div>
  );
}
```

## GitHub ログイン実装

### Developer 向け GitHub 認証

開発者向けアプリケーションでは、GitHub ログインが一般的です。

```jsx
import { useState } from 'react';

export function GitHubLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);
  const [isLoading, setIsLoading] = useState(false);

  const handleGitHubLogin = async () => {
    setIsLoading(true);
    try {
      // OAuth の認可画面にリダイレクト
      const clientId = process.env.GITHUB_CLIENT_ID;
      const redirectUri = encodeURIComponent(`${process.env.APP_URL}/auth/github/callback`);
      const scope = 'user:email';

      const authUrl = `https://github.com/login/oauth/authorize?` +
        `client_id=${clientId}&` +
        `redirect_uri=${redirectUri}&` +
        `scope=${scope}`;

      // ブラウザで OAuth フローを開始
      window.location.href = authUrl;
    } catch (error) {
      console.error('GitHub ログインエラー:', error);
      setIsLoading(false);
    }
  };

  // OAuth コールバック処理
  const handleAuthCallback = async (code) => {
    try {
      // バックエンドがコードをトークンに交換
      const response = await fetch('/api/auth/github/callback', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ code }),
      });

      const { sessionToken, user } = await response.json();

      console.log(`GitHub ログイン成功: ${user.login}`);
      // 期待出力: GitHub ログイン成功: octocat

      localStorage.setItem('sessionToken', sessionToken);
      setUserInfo(user);
    } catch (error) {
      console.error('コールバック処理エラー:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleGitHubLogin} disabled={isLoading} style={{ backgroundColor: '#333' }}>
          {isLoading ? 'ログイン中...' : 'GitHub でログイン'}
        </button>
      ) : (
        <div>
          <p>ログイン済み: {userInfo.login}</p>
          <button onClick={() => {
            localStorage.removeItem('sessionToken');
            setUserInfo(null);
          }}>ログアウト</button>
        </div>
      )}
    </div>
  );
}
```

## トークン管理とセッション保持

### セキュアなトークン保管

トークンはローカルストレージではなく、より安全な場所に保管すべきです。

```jsx
import * as SecureStore from 'expo-secure-store';

export class AuthTokenManager {
  static async saveToken(token) {
    try {
      await SecureStore.setItemAsync('authToken', token);
      console.log('トークンを安全に保存しました');
      // 期待出力: トークンを安全に保存しました
    } catch (error) {
      console.error('トークン保存エラー:', error);
    }
  }

  static async getToken() {
    try {
      const token = await SecureStore.getItemAsync('authToken');
      return token;
    } catch (error) {
      console.error('トークン取得エラー:', error);
      return null;
    }
  }

  static async deleteToken() {
    try {
      await SecureStore.deleteItemAsync('authToken');
      console.log('トークンを削除しました');
      // 期待出力: トークンを削除しました
    } catch (error) {
      console.error('トークン削除エラー:', error);
    }
  }

  static async isTokenValid(token) {
    // JWT の有効期限をチェック
    try {
      const payload = JSON.parse(atob(token.split('.')[1]));
      const expiryTime = payload.exp * 1000;
      return Date.now() < expiryTime;
    } catch (error) {
      return false;
    }
  }
}
```

## FAQ

### Q1: 複数のソーシャルプロバイダーで同じメールアドレスの場合の対応は？

**A:** ユーザーがメールアドレスを一意の識別子として使用している場合、既存アカウントにリンクするか、新規アカウント作成を促すか、ユーザーに選択させることが標準的です。メールアドレス確認フローを実装し、認証後にアカウント統合を提案しましょう。

### Q2: ソーシャルログイン後、ユーザーが個人アカウントに移行したい場合は？

**A:** パスワード設定フローを実装し、ユーザーが自由にパスワードを設定できるようにしましょう。その後は、従来のメール+パスワード認証でもログイン可能にすることで、柔軟性が向上します。

### Q3: ソーシャルプロバイダーの追加スコープをリクエストする方法は？

**A:** 初期ログイン時に最小限のスコープをリクエストし、後で追加スコープが必要になったら incremental authorization を使用します。Google や Apple では、既存トークンを持つ状態で追加スコープでログインフローを再実行できます。
