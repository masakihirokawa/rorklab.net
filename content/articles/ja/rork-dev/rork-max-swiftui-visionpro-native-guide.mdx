Apple の Vision Pro が登場し、空間コンピューティングの時代が幕を開けました。その一方で、visionOS アプリ開発は iOS や Mac の開発と比べて初期学習コストが高く、多くの開発者にとって敷居の高い領域となっています。

Rork Max は、この課題を大きく解決するツールです。**AI ネイティブコード生成エンジン**を搭載し、プロンプトやスケッチ、Figma デザインからわずか数秒で **本番品質の SwiftUI コード**を生成できます。さらに、iOS と visionOS の**コード共有**メカニズムにより、2 つのプラットフォーム向けアプリを同時に開発できるのが最大の強みです。

本ガイドでは、Rork Max を使った Vision Pro ネイティブアプリ開発の全工程を、初期設定から App Store リリース、サブスクリプション収益化まで、実装レベルで解説します。このガイドを読み終わった後、あなたは自信を持って visionOS アプリ開発に取り組める状態になるでしょう。

## Rork Max の基礎概念とネイティブコード生成エンジン

### Rork Max とは何か

Rork Max は、生成 AI と LLM の力を活かし、**自然言語やビジュアルデザインからネイティブアプリコードを自動生成する開発プラットフォーム**です。特に iOS / visionOS 開発に特化しており、SwiftUI を中心とした最新の Apple フレームワークを完全サポートしています。

従来のローコード・ノーコードツールとは異なり、Rork Max は **実行可能で保守性の高い本番コード**を生成します。これは、生成コードが単なる「プロトタイプ」ではなく、App Store でそのまま公開できる品質を目指しているからです。

### AI ネイティブコード生成エンジンの仕組み

Rork Max の核となるのが **AI ネイティブコード生成エンジン**です。このエンジンは以下の 3 つの入力形式をサポートしています：

1. **自然言語プロンプト**
   ```
   「ユーザーが写真をアップロード→AIで自動タグ付け→ギャラリー表示するアプリ」
   ```
   このような日本語の要件記述から、フルスタックの iOS アプリ構造を自動生成します。

2. **スケッチ・ワイヤーフレーム**
   紙やデジタルホワイトボード上のスケッチをアップロードすると、Rork Max がそれを解析し、UI レイアウトと初期ロジックを自動構成します。

3. **Figma デザイン**
   Figma ファイルを直接インポートし、デザイントークン（色・フォント・スペーシング）と コンポーネント階層を自動的に SwiftUI コードに変換します。

生成エンジンは、入力内容を理解した上で、Apple の最新ガイドラインに沿った **セキュア・高性能・保守しやすい**コードを生成します。これにより、開発者は時間のかかるボイラープレート作成から解放され、本当に価値のある機能実装に集中できるのです。

## iOS と visionOS のコード共有アーキテクチャ

### プラットフォーム共有の設計思想

Rork Max で生成される SwiftUI コードは、**iOS と visionOS の両プラットフォームで動作する共通コード**として設計されています。これは単に「同じコードを使える」というレベルではなく、各プラットフォームの UI パラダイムの違いを吸収し、自動的に適応する仕組みになっています。

```swift
import SwiftUI

// 両プラットフォーム対応のメインビュー
struct ContentView: View {
    @Environment(\.horizontalSizeClass) var sizeClass
    @State private var selectedItem: Item?

    var body: some View {
        #if os(visionOS)
        // Vision Pro 向け: 3 次元空間レイアウト
        VStack {
            WindowGroup {
                MainWindow()
            }
            .windowStyle(.plain)

            if let item = selectedItem {
                detailVolume(for: item)
            }
        }
        #else
        // iOS 向け: 従来のナビゲーション
        NavigationStack {
            ListView()
                .navigationDestination(item: $selectedItem) { item in
                    DetailView(item: item)
                }
        }
        #endif
    }
}
```

### visionOS 固有のレイアウト手法

Vision Pro では、従来の 2D 画面上での UI ではなく、**3 次元空間内に複数のウィンドウやボリュームを配置する**アプローチになります。Rork Max は、この空間コンピューティング独特の UI パターンをサポートしています：

1. **Window** — 従来のアプリウィンドウを 3D 空間に配置
   ```swift
   WindowGroup {
       ContentView()
   }
   .windowStyle(.plain)
   .defaultSize(width: 600, height: 800)
   ```

2. **Volume** — 3D オブジェクトを含む立体的なコンテンツ領域
   ```swift
   VolumeGroup {
       ModelView()
           .frame(depth: 300)
   }
   .volumeStyle(.standard)
   ```

3. **ImmersiveSpace** — ユーザーの視界全体を覆う没入型環境
   ```swift
   ImmersiveSpace(id: "immersive") {
       RealityComposer2Content()
   }
   .immersionStyle(selection: $immersion, in: .full)
   ```

Rork Max は、アプリの要件に応じてこれらのパターンを自動的に選択し、最適なレイアウト構造を生成します。

## Vision Pro UI 設計パターン と RealityKit 統合

### 空間 UI の 5 つの設計原則

Vision Pro アプリを成功させるためには、従来の iOS UI 設計の常識が通用しない部分があります。Rork Max で生成するコードは、以下の 5 つの設計原則に基づいています：

1. **スケーラビリティ** — Window の大きさと距離が可変。ユーザーがジェスチャで自由にリサイズできることを前提とした UI 設計
2. **奥行き感** — 2D 平面ではなく、Z 軸方向の奥行きを活かし、層状の情報階層を表現
3. **視線インタラクション** — タッチの代わりに視線（gaze）とハンドジェスチャが主要な入力手段
4. **周辺認識** — ユーザーが周囲のパススルー風景を見守りながら操作できるレイアウト
5. **パフォーマンス優先** — GPU 負荷が限定的であることを前提とした効率的な描画

### 実装例：3D モデルビューアアプリ

```swift
import SwiftUI
import RealityKit

struct ModelViewerApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }

        ImmersiveSpace(id: "viewer") {
            ModelViewerSpace()
        }
        .immersionStyle(selection: .constant(.mixed), in: .mixed)
    }
}

struct ModelViewerSpace: View {
    @State private var selectedModel: String = "default"
    @State private var rotationX: Float = 0
    @State private var rotationY: Float = 0

    var body: some View {
        ZStack {
            // 背景パススルー
            Model3D(
                named: selectedModel,
                bundle: .main
            )
            .frame(depth: 200)
            .gesture(
                DragGesture()
                    .onChanged { value in
                        rotationY = Float(value.translation.width) * 0.01
                        rotationX = Float(value.translation.height) * 0.01
                    }
            )
            .rotation3D(
                .degrees(Double(rotationX)),
                axis: (1, 0, 0)
            )
            .rotation3D(
                .degrees(Double(rotationY)),
                axis: (0, 1, 0)
            )

            // コントロールパネル（左側）
            VStack(spacing: 16) {
                Text("Model Viewer")
                    .font(.system(size: 28, weight: .bold))

                Picker("Model", selection: $selectedModel) {
                    ForEach(["model1", "model2", "model3"], id: \.self) { name in
                        Text(name).tag(name)
                    }
                }
                .pickerStyle(.segmented)

                Spacer()
            }
            .padding(24)
            .frame(maxWidth: 250, alignment: .leading)
            .background(Material.thickMaterial)
            .offset(x: -300, z: 50)
        }
    }
}
```

このコードは Rork Max で「3D モデルをハンドジェスチャで回転させ、左側のパネルで複数モデルを切り替える Vision Pro アプリ」というプロンプトから自動生成されるものの例です。

### RealityKit と SwiftUI の統合パターン

Rork Max は、RealityKit（Apple の 3D・AR フレームワーク）と SwiftUI の統合を簡潔に扱えます：

```swift
import RealityKit

struct ARViewContainer: UIViewRepresentable {
    @ObservedRealmObject var arModel: ARModel

    func makeUIView(context: Context) -> ARView {
        let arView = ARView(frame: .zero)
        loadModel(into: arView)
        return arView
    }

    func updateUIView(_ uiView: ARView, context: Context) {
        // 状態変更時の更新処理
        if arModel.needsUpdate {
            loadModel(into: uiView)
        }
    }

    private func loadModel(into arView: ARView) {
        Task {
            guard let modelAnchor = try? await Experience.loadModel() else { return }
            arView.scene.addAnchor(modelAnchor)
        }
    }
}
```

## サブスクリプション収益化とストライプ決済の実装

### IAP（In-App Purchase）の設定

多くの Vision Pro アプリは、プレミアム機能をサブスクリプション（月額・年額）で提供しています。Rork Max では IAP の設定と処理が自動生成されます：

```swift
import StoreKit

@MainActor
class SubscriptionManager: ObservableObject {
    @Published var availableSubscriptions: [Product] = []
    @Published var purchasedSubscriptionId: String?
    @Published var isSubscriptionActive: Bool = false

    init() {
        Task {
            await loadSubscriptions()
            await checkSubscriptionStatus()
        }
    }

    func loadSubscriptions() async {
        do {
            let productIds = ["com.example.fitpro.monthly",
                              "com.example.fitpro.yearly"]
            availableSubscriptions = try await Product.products(for: productIds)
        } catch {
            print("Failed to load subscriptions: \(error)")
        }
    }

    func purchaseSubscription(_ product: Product) async {
        do {
            let result = try await product.purchase()
            switch result {
            case .success(let verification):
                let transaction = try checkVerified(verification)
                await updateSubscriptionStatus(transaction)
                await transaction.finish()

            case .userCancelled:
                print("User cancelled subscription")

            case .pending:
                print("Subscription pending parent approval")
            @unknown default:
                break
            }
        } catch {
            print("Purchase failed: \(error)")
        }
    }

    func checkSubscriptionStatus() async {
        for await result in Transaction.currentEntitlements {
            let transaction = try checkVerified(result)

            if transaction.revocationDate == nil {
                isSubscriptionActive = true
                purchasedSubscriptionId = transaction.productID
            }
        }
    }

    private func checkVerified<T>(_ result: VerificationResult<T>) throws -> T {
        switch result {
        case .unverified:
            throw NSError(domain: "Unverified transaction", code: -1)
        case .verified(let safe):
            return safe
        }
    }
}
```
