Rork Maxが生成するネイティブSwiftコードは、React Native系のツールでは不可能だったApple固有フレームワークにフルアクセスできます。その中でも見落とされがちなのが**PassKit（Apple Wallet）**です。ロイヤリティカード・搭乗券・イベントチケット・クーポン配布など、ビジネス活用の幅が広いWalletパス機能を、Rork MaxのAIプロンプトだけで実装できます。本記事では、PassKitの基礎から実際のアプリ組み込みまでをステップバイステップで解説します。

## PassKitとApple Walletとは

**PassKit**はAppleが提供するフレームワークで、iPhoneの「Wallet」アプリにパス（デジタルカード）を追加・管理するための機能を提供します。

### Walletパスで実現できること

PassKitを使ったWalletパスは、次のようなユースケースに対応しています。

- **ストアカード / ロイヤリティカード**：スタンプカード・ポイントカードのデジタル版
- **搭乗券 / 乗車券**：航空券・電車・バスのeチケット
- **イベントチケット**：コンサート・スポーツ観戦・映画館チケット
- **クーポン / 割引券**：プロモーション配布・時限クーポン
- **会員証**：ジム・サブスクリプションサービスの会員証

### なぜRork MaxでPassKitを使うのか

従来のReact Nativeベースのツール（無印Rorkを含む）では、PassKitへのネイティブアクセスが制限されていましました。Rork MaxはSwiftUIアプリを直接生成するため、PassKitを完全に活用できます。

- ✅ Walletへのパス追加ボタン（`PKAddPassesViewController`）を標準実装
- ✅ バーコード・QRコードをパスに埋め込み可能
- ✅ プッシュ通知でパスのリアルタイム更新が可能
- ✅ Apple Payと組み合わせた決済フローの構築

| パスタイプ | PassKit定数 | 主な用途 |
|------------|------------|---------|
| 搭乗券 | `PKPassType.boardingPass` | 航空・鉄道チケット |
| クーポン | `PKPassType.coupon` | 割引券・プロモーション |
| イベントチケット | `PKPassType.eventTicket` | コンサート・スポーツ |
| ストアカード | `PKPassType.storeCard` | ロイヤリティ・ポイントカード |
| 汎用 | `PKPassType.generic` | 会員証・その他 |

## Step 2：pass.jsonを作成する

Apple Walletパスの実態は**`.pkpass`ファイル**で、内部に`pass.json`を含むZIPアーカイブです。Rork Maxに生成させるためのpass.json構造を理解しましょう。

### カフェスタンプカードのpass.json例

```json
{
  "formatVersion": 1,
  "passTypeIdentifier": "pass.com.yourdomain.cafeloyalty",
  "serialNumber": "cafe-stamp-001",
  "teamIdentifier": "ABCDE12345",
  "webServiceURL": "https://your-api.com/passes/",
  "authenticationToken": "vxwxd7J8AlNNFPS8k0a0FfUFtq0ewzFdc",
  "organizationName": "My Café",
  "description": "カフェスタンプカード",
  "logoText": "My Café",
  "foregroundColor": "rgb(255, 255, 255)",
  "backgroundColor": "rgb(60, 65, 76)",
  "storeCard": {
    "primaryFields": [
      {
        "key": "balance",
        "label": "スタンプ",
        "value": "3/10",
        "textAlignment": "PKTextAlignmentCenter"
      }
    ],
    "secondaryFields": [
      {
        "key": "member",
        "label": "会員番号",
        "value": "00012345"
      },
      {
        "key": "expiry",
        "label": "有効期限",
        "value": "2026-12-31",
        "dateStyle": "PKDateStyleShort"
      }
    ],
    "barcode": {
      "message": "00012345",
      "format": "PKBarcodeFormatQR",
      "messageEncoding": "iso-8859-1"
    }
  }
}
```

### 重要なフィールド解説

- `passTypeIdentifier`：Developer Portalで作成したPass Type ID
- `teamIdentifier`：Apple DeveloperのTeam ID（Developer Portalで確認）
- `serialNumber`：パスの一意識別子（ユーザーごとにユニークにする）
- `authenticationToken`：更新通知の認証トークン（32文字以上）
- `webServiceURL`：パス更新APIのベースURL（HTTPS必須）

## Step 3：SwiftUIアプリへの実装

Rork Maxで生成されたコードに、PassKitの主要コンポーネントを組み込みます。

### PassKitManagerクラスの作成

Rork Maxへのプロンプト：

```
PassKitManagerクラスを作成してください。以下の機能を含めてください：
- canAddPasses() でWalletパス追加が可能か確認
- addPass(url: URL) でURLからパスを取得してWalletに追加
- PKAddPassButtonを表示するSwiftUI View
- パス追加成功・失敗のアラート表示
```

生成されるSwiftUIコードの例：

```swift
import SwiftUI
import PassKit

// MARK: - PassKit Manager
class PassKitManager: NSObject, ObservableObject {
    @Published var isAddingPass = false
    @Published var showSuccessAlert = false
    @Published var showErrorAlert = false
    @Published var errorMessage = ""

    /// Walletパスの追加が可能か確認
    /// - Returns: PassKitが利用可能な場合はtrue
    func canAddPasses() -> Bool {
        return PKAddPassesViewController.canAddPasses()
    }

    /// URLからパスを取得してWalletに追加
    /// - Parameter url: .pkpassファイルのURL
    func addPass(from url: URL) async {
        do {
            let data = try Data(contentsOf: url)
            let pass = try PKPass(data: data)

            await MainActor.run {
                self.presentAddPassController(for: pass)
            }
        } catch {
            await MainActor.run {
                self.errorMessage = "パスの読み込みに失敗しました: \(error.localizedDescription)"
                self.showErrorAlert = true
            }
        }
    }

    /// PKAddPassesViewControllerを表示
    private func presentAddPassController(for pass: PKPass) {
        guard let addVC = PKAddPassesViewController(pass: pass),
              let rootVC = UIApplication.shared.connectedScenes
                .compactMap({ $0 as? UIWindowScene })
                .first?.windows.first?.rootViewController else { return }

        isAddingPass = true
        rootVC.present(addVC, animated: true)
    }
}

// MARK: - Wallet Add Button View
struct WalletAddButton: View {
    @StateObject private var passKitManager = PassKitManager()
    let passURL: URL

    var body: some View {
        if passKitManager.canAddPasses() {
            Button(action: {
                Task {
                    await passKitManager.addPass(from: passURL)
                }
            }) {
                // Apple公式デザインのWalletボタン
                Image("PKAddToWalletButton")
                    .resizable()
                    .aspectRatio(contentMode: .fit)
                    .frame(height: 44)
            }
            .alert("追加完了", isPresented: $passKitManager.showSuccessAlert) {
                Button("OK") {}
            } message: {
                Text("Walletにカードを追加しました")
            }
            .alert("エラー", isPresented: $passKitManager.showErrorAlert) {
                Button("OK") {}
            } message: {
                Text(passKitManager.errorMessage)
            }
        } else {
            Text("このデバイスではWalletパスを使用できません")
                .foregroundColor(.secondary)
                .font(.caption)
        }
    }
}
// 期待する出力: "Walletに追加"ボタンが表示される（PKAddToWalletButtonのApple公式デザイン）
```

## Step 4：パスのサーバーサイド署名と配信

`.pkpass`ファイルは、Apple Developer証明書で**署名**する必要があります。署名なしのパスはWalletに追加できません。

### Cloudflare Workers + Supabaseでの配信フロー

Rork Maxと相性の良い構成でパスを配信するフローは以下の通りです。

```
ユーザーのアプリ
    ↓ ユーザーID送信
Cloudflare Workers（APIエンドポイント）
    ↓ ユーザー情報取得
Supabase（会員情報DB）
    ↓ パスJSONをNode.jsで生成・署名
署名済み.pkpassファイル
    ↓ Base64エンコードで返却
アプリがWalletに追加
```

### Node.jsでのパス署名コード

```javascript
// Cloudflare Worker / Node.js でのパス署名処理
const { execSync } = require('child_process');
const fs = require('fs');
const path = require('path');
const archiver = require('archiver');

async function createSignedPass(passData, userId) {
  const tempDir = `/tmp/pass_${userId}`;
  fs.mkdirSync(tempDir, { recursive: true });

  // pass.jsonを書き出し
  fs.writeFileSync(
    path.join(tempDir, 'pass.json'),
    JSON.stringify({ ...passData, serialNumber: userId })
  );

  // 画像ファイルをコピー
  ['icon.png', 'icon@2x.png', 'logo.png', 'logo@2x.png'].forEach(img => {
    fs.copyFileSync(`./assets/${img}`, path.join(tempDir, img));
  });

  // manifest.jsonを生成（SHA1ハッシュ）
  const manifest = {};
  for (const file of fs.readdirSync(tempDir)) {
    const hash = execSync(`sha1sum ${path.join(tempDir, file)}`).toString().split(' ')[0];
    manifest[file] = hash;
  }
  fs.writeFileSync(path.join(tempDir, 'manifest.json'), JSON.stringify(manifest));

  // OpenSSLで署名
  execSync(`openssl smime -binary -sign \
    -certfile ./certs/AppleWWDRCA.pem \
    -signer ./certs/passcertificate.pem \
    -inkey ./certs/passkey.pem \
    -in ${path.join(tempDir, 'manifest.json')} \
    -out ${path.join(tempDir, 'signature')} \
    -outform DER`);

  // .pkpassファイルを作成（ZIP）
  const output = fs.createWriteStream(`/tmp/${userId}.pkpass`);
  const archive = archiver('zip', { zlib: { level: 9 } });
  archive.pipe(output);
  archive.directory(tempDir, false);
  await archive.finalize();

  return fs.readFileSync(`/tmp/${userId}.pkpass`);
}
// 期待する出力: 署名済みの.pkpassファイルバッファが返される
```

## Step 5：パスのリアルタイム更新

PassKitはプッシュ通知を使ってパスの内容をリアルタイムで更新できます。スタンプ追加やポイント更新をWallet上のカードに即座に反映させることが可能です。

### 更新フロー

1. ユーザーがアプリでスタンプを押す
2. バックエンドAPIがAPNs（Apple Push Notification service）に通知を送信
3. iOSがWalletパスの更新をサーバーに問い合わせ
4. サーバーが新しい署名済みパスを返す
5. Walletのカードが自動更新される

### PassKit更新APIのエンドポイント

```
GET  /passes/{passTypeIdentifier}/{serialNumber}
     → 最新のパスデータを返す

POST /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}/{serialNumber}
     → デバイスの更新登録

DELETE /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}/{serialNumber}
     → デバイスの更新解除

GET  /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}
     → 更新が必要なシリアル番号一覧を返す
```

## よくあるエラーと対処法

### エラー1：「Walletに追加できません」が表示される

**原因**：パスの署名が無効か、証明書の有効期限切れ。

**対処法**：
1. Apple Developer Portalで証明書の有効期限を確認
2. `openssl verify`コマンドで署名を検証
3. `teamIdentifier`がDeveloper AccountのTeam IDと一致しているか確認

### エラー2：PKAddPassesViewControllerが表示されない

**原因**：シミュレーターではPassKitの動作が制限されています。

**対処法**：実機（iPhone）でテストしてください。Rork Maxの「1-Click Install」機能で実機に直接インストールできます。

### エラー3：バーコードが読み取れない

**原因**：`messageEncoding`が不正か、バーコードデータが長すぎる。

**対処法**：QRコードは2,953文字まで、Code 128は不定長ですが短い方が読み取り精度が上がります。`messageEncoding`は`iso-8859-1`を使用してください。

## FAQ

**Q1. PassKitの実装にApple Developer登録は必須ですか？**

はい、必須です。Pass Type IDの作成とパスの署名証明書の取得にApple Developer Program（年間$99）のメンバーシップが必要です。ただし、テスト段階ではTestFlightを通じて実機テストが可能です。

**Q2. Walletパスは更新ごとに新しいパスを発行する必要がありますか？**

いいえ。PassKitのアップデートAPIを実装すれば、既存のパスをサイレントに更新できます。ユーザーはウォレット内のカードが自動更新されるのを確認できます（スタンプ追加後にアニメーションで数値が変わるなど）。

**Q3. Rork MaxのどのプランでPassKitを実装できますか？**

Rork MaxはすべてのSwiftUIコードにアクセスできるため、PassKitの実装はどのプランでも可能です。ただし、生成されたコードをTestFlight配布・App Store申請するには、有料のRork Maxプランが必要です。

**Q4. Android版のGoogle Walletパスも作成できますか？**

Rork（React Native版）を使用することで、Google Wallet（Google Pay API for Passes）にも対応できます。別途Google Cloudの設定と`google-pay/passes`ライブラリが必要です。
