## モバイルアプリとオフライン対応

現代のモバイルアプリでは、**ネットワーク接続がない状態でも動作すること**が求められるケースが増えています。電車の中、飛行機内、電波の弱い場所でも使えるアプリは、ユーザー体験が格段に向上します。

Rork Max アプリは React Native ベースなので、ネイティブのストレージ機能を使ったオフライン対応が可能です。

## AsyncStorage — 軽量なキー・バリューストア

設定値・ユーザープロファイル・小さなデータには `AsyncStorage` が最適です。

```
Rork Max へのプロンプト:
「ユーザーの設定（テーマ・言語設定）を AsyncStorage で保存・読み込みする機能を追加してください。」
```

生成されるコード例:

```javascript
import AsyncStorage from '@react-native-async-storage/async-storage';

// データを保存
export async function saveSettings(settings) {
  try {
    await AsyncStorage.setItem('user_settings', JSON.stringify(settings));
  } catch (error) {
    console.error('Failed to save settings:', error);
  }
}

// データを読み込み
export async function loadSettings() {
  try {
    const stored = await AsyncStorage.getItem('user_settings');
    return stored ? JSON.parse(stored) : null;
  } catch (error) {
    console.error('Failed to load settings:', error);
    return null;
  }
}

// データを削除
export async function clearSettings() {
  await AsyncStorage.removeItem('user_settings');
}
```

## expo-sqlite — 本格的なローカルデータベース

To-Do リスト、メモ、オフラインで使う記事など、複雑なデータには `expo-sqlite` を使います。

```
Rork Max へのプロンプト:
「タスクリストを expo-sqlite でローカルに保存する機能を追加してください。
CRUD 操作（作成・読み取り・更新・削除）に対応させてください。」
```

```javascript
import * as SQLite from 'expo-sqlite';

const db = SQLite.openDatabase('myapp.db');

// テーブルの作成
export function initDatabase() {
  db.transaction((tx) => {
    tx.executeSql(
      `CREATE TABLE IF NOT EXISTS tasks (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT NOT NULL,
        completed INTEGER DEFAULT 0,
        created_at TEXT DEFAULT CURRENT_TIMESTAMP
      )`,
      [],
      () => console.log('Table created'),
      (_, error) => console.error('Table creation failed:', error)
    );
  });
}

// タスクの追加
export function addTask(title, callback) {
  db.transaction((tx) => {
    tx.executeSql(
      'INSERT INTO tasks (title) VALUES (?)',
      [title],
      (_, result) => callback(result.insertId),
      (_, error) => console.error('Insert failed:', error)
    );
  });
}

// 全タスクの取得
export function getTasks(callback) {
  db.transaction((tx) => {
    tx.executeSql(
      'SELECT * FROM tasks ORDER BY created_at DESC',
      [],
      (_, result) => callback(result.rows._array),
      (_, error) => console.error('Query failed:', error)
    );
  });
}
```

## ネットワーク状態の監視

オフライン時に適切なUIを表示するには、ネットワーク状態を監視します。

```javascript
import NetInfo from '@react-native-community/netinfo';
import { useState, useEffect } from 'react';

export function useNetworkStatus() {
  const [isOnline, setIsOnline] = useState(true);

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener((state) => {
      setIsOnline(state.isConnected && state.isInternetReachable);
    });
    return unsubscribe;
  }, []);

  return isOnline;
}

// 使用例
function MyApp() {
  const isOnline = useNetworkStatus();

  return (
    <View>
      {!isOnline && (
        <View style={styles.offlineBanner}>
          <Text>オフラインモードです</Text>
        </View>
      )}
      {/* メインコンテンツ */}
    </View>
  );
}
```

## まとめ

Rork Max でのオフライン対応には:

- **AsyncStorage**: 設定・セッションなど単純なキー・バリューデータ
- **expo-sqlite**: タスク・メモなど構造化データのローカルDB
- **NetInfo**: ネットワーク状態の監視とUIフィードバック

Rork Max へのプロンプトでこれらを組み込んでもらうと、オフラインでも使えるしっかりしたアプリが完成します。
