## はじめに

Apple の **SharePlay** は、FaceTime 通話中やメッセージアプリ上で複数のユーザーがリアルタイムに同じ体験を共有できる機能です。映画の同時視聴、ゲームの対戦プレイ、ホワイトボードの共同編集など、ユーザー同士のつながりを深めるアプリ体験を提供できます。

しかし、SharePlay を実現するには Apple の **Group Activities** フレームワークを理解し、セッション管理やデータ同期のロジックを Swift で実装する必要があります。これは従来のアプリ開発では、かなりハードルが高い作業でしました。

**Rork Max** はネイティブ SwiftUI コードを AI で自動生成するため、SharePlay と Group Activities の複雑な実装もプロンプトベースで進めることが可能です。本記事では、Rork Max を使って SharePlay 対応アプリをゼロから構築する方法を、プロンプト例とコード解説を交えてステップバイステップで紹介します。

この記事で学べること：

- SharePlay と Group Activities の仕組みと全体像
- Rork Max での SharePlay 実装ステップ
- GroupActivity プロトコルと GroupSession の使い方
- リアルタイムデータ同期の実践コード
- FaceTime・メッセージ両方での動作対応
- よくあるエラーと対処法

対象読者：Rork Max の基本操作を理解している方（[Rork Max と通常版 Rork の違い](/articles/rork-basics/rork-vs-rork-max) を先にお読みいただくと、よりスムーズです）。

## SharePlay と Group Activities の仕組み

### SharePlay とは

SharePlay は iOS 15 で導入され、iOS 17 以降で大幅に強化された Apple のフレームワークです。FaceTime 通話中はもちろん、iMessage のグループチャット上でもリアルタイムの共有体験を提供できます。

主なユースケースは以下のとおりです。

- **コンテンツの同時視聴**: 映画や動画を友達と一緒に視聴し、再生位置を同期
- **共同作業**: ホワイトボードやドキュメントをリアルタイムで共同編集
- **対戦ゲーム**: ターン制やリアルタイム対戦をシームレスに実現
- **フィットネス**: トレーニングメニューを共有しながら一緒に運動
- **ショッピング**: 商品を一緒に見ながら感想を共有

### Group Activities フレームワークの構成

SharePlay の技術的な基盤は **Group Activities** フレームワークです。主要な構成要素は次のとおりです。

1. **GroupActivity**: 共有アクティビティの定義（プロトコル）
2. **GroupSession**: アクティビティのセッション管理
3. **GroupSessionMessenger**: 参加者間のメッセージ送受信
4. **GroupStateObserver**: セッション状態の監視

```swift
// GroupActivity プロトコルの基本構造
import GroupActivities

struct DrawingActivity: GroupActivity {
    // アクティビティのメタ情報
    var metadata: GroupActivityMetadata {
        var meta = GroupActivityMetadata()
        meta.title = "一緒にお絵描き"
        meta.subtitle = "リアルタイムで一緒に絵を描こう"
        meta.type = .generic
        return meta
    }
}

// 期待する出力:
// FaceTime / メッセージから起動可能な
// 共有アクティビティが定義される
```

## Rork Max で SharePlay 対応アプリを作る手順

### 前提条件

- **Rork Max プラン**（ネイティブ Swift コード生成が必要）
- **Apple Developer Program** への登録（SharePlay テストに必要）
- **iOS 17 以降**のデバイス2台（シミュレータでもテスト可能）

### Step 1: プロジェクトの作成

まず Rork Max で新しいプロジェクトを作成し、以下のプロンプトでアプリの骨格を構築します。

```text
リアルタイム共同お絵描きアプリを作ってください。
以下の機能を含めてください：

1. キャンバス画面（指で線を描ける）
2. 色の選択パレット（8色）
3. 線の太さ調整（3段階）
4. クリアボタン
5. SharePlay 対応（FaceTime 中に共同描画可能）

SwiftUI + Canvas を使い、シンプルで直感的な UI にしてください。
```

Rork Max はこのプロンプトから SwiftUI ベースのプロジェクト構造を自動生成します。重要なのは「SharePlay 対応」というキーワードを明示的に含めることで、Group Activities フレームワークの初期設定も同時に行われる点です。

### Step 2: GroupActivity の定義

Rork Max が生成するコードには、`GroupActivity` プロトコルに準拠したアクティビティ定義が含まれます。以下はその中核部分です。

```swift
import GroupActivities
import SwiftUI

// 共同描画アクティビティの定義
struct CollaborativeDrawingActivity: GroupActivity {
    // 各参加者で共有されるアクティビティの情報
    static let activityIdentifier = "com.example.drawing.collaborative"

    var metadata: GroupActivityMetadata {
        var meta = GroupActivityMetadata()
        meta.title = NSLocalizedString(
            "共同お絵描き", comment: "SharePlay activity title"
        )
        meta.subtitle = NSLocalizedString(
            "一緒にキャンバスに絵を描こう",
            comment: "SharePlay activity subtitle"
        )
        meta.type = .generic
        // SharePlay シートに表示されるアイコン
        meta.preferredBroadcastOptions = .default
        return meta
    }
}

// 期待する出力:
// ビルド成功 — FaceTime の SharePlay メニューに
// 「共同お絵描き」が表示される
```

### Step 3: セッション管理の実装

次に、`GroupSession` を使ったセッション管理を実装します。Rork Max には以下のプロンプトで指示します。

```text
SharePlay のセッション管理を実装してください。
以下の要件：

1. FaceTime 通話中に自動的にアクティビティを提案
2. 参加者の入退室を監視
3. セッション終了時のクリーンアップ処理
4. エラーハンドリング（接続切れ等）
```

Rork Max が生成するセッション管理のコードは次のようになります。

```swift
import GroupActivities
import Combine

@MainActor
class DrawingSessionManager: ObservableObject {
    @Published var session: GroupSession<CollaborativeDrawingActivity>?
    @Published var messenger: GroupSessionMessenger?
    @Published var participants: Set<Participant> = []
    @Published var isSharePlayActive = false

    private var subscriptions = Set<AnyCancellable>()
    private var tasks = Set<Task<Void, Never>>()

    // セッションの監視を開始
    func startObserving() {
        let task = Task {
            for await session in CollaborativeDrawingActivity.sessions() {
                self.configureSession(session)
            }
        }
        tasks.insert(task)
    }

    private func configureSession(
        _ session: GroupSession<CollaborativeDrawingActivity>
    ) {
        self.session = session
        self.messenger = GroupSessionMessenger(session: session)
        self.isSharePlayActive = true

        // 参加者の変更を監視
        session.$activeParticipants
            .sink { [weak self] participants in
                self?.participants = participants
            }
            .store(in: &subscriptions)

        // セッション状態の変更を監視
        session.$state
            .sink { [weak self] state in
                if case .invalidated = state {
                    self?.cleanupSession()
                }
            }
            .store(in: &subscriptions)

        // セッションに参加
        session.join()
    }

    private func cleanupSession() {
        session = nil
        messenger = nil
        isSharePlayActive = false
        participants = []
        subscriptions.removeAll()
    }
}

// 期待する出力:
// SharePlay セッション開始時にログ出力:
// "Session joined with 2 participants"
```

### Step 4: リアルタイムデータ同期

SharePlay の核心は、参加者間のリアルタイムデータ同期です。描画データを `GroupSessionMessenger` で送受信する実装を追加します。

```text
描画データのリアルタイム同期を実装してください。
要件：

1. 描画ストロークを他の参加者にリアルタイム送信
2. 受信したストロークをキャンバスに即座に反映
3. Codable な DrawingMessage 構造体を定義
4. 遅延を最小限にする最適化
```

```swift
import Foundation
import GroupActivities

// 送受信するメッセージの定義
struct DrawingMessage: Codable, Sendable {
    let id: UUID
    let points: [CGPointCodable]
    let color: String
    let lineWidth: CGFloat
    let timestamp: Date

    struct CGPointCodable: Codable, Sendable {
        let x: CGFloat
        let y: CGFloat
    }
}

// メッセージの送受信処理
extension DrawingSessionManager {
    // ストロークを全参加者に送信
    func sendStroke(_ stroke: DrawingStroke) async {
        guard let messenger else { return }

        let message = DrawingMessage(
            id: UUID(),
            points: stroke.points.map {
                DrawingMessage.CGPointCodable(x: $0.x, y: $0.y)
            },
            color: stroke.color.hexString,
            lineWidth: stroke.lineWidth,
            timestamp: Date()
        )

        do {
            // reliable: 確実な配信を保証
            try await messenger.send(message)
        } catch {
            print("送信エラー: \(error.localizedDescription)")
        }
    }

    // 他の参加者からのストロークを受信
    func startReceiving(
        onReceive: @escaping (DrawingMessage) -> Void
    ) {
        let task = Task {
            guard let messenger else { return }
            for await (message, _) in messenger.messages(
                of: DrawingMessage.self
            ) {
                await MainActor.run {
                    onReceive(message)
                }
            }
        }
        tasks.insert(task)
    }
}

// 期待する出力:
// ユーザーAが描いた線がリアルタイムで
// ユーザーBのキャンバスに表示される
```

## FaceTime とメッセージの両方に対応する

iOS 17 以降では、SharePlay は FaceTime だけでなく **iMessage のグループチャット** からも利用できるようになりましました。Rork Max で両方に対応するには、以下のプロンプトを使います。

```text
SharePlay をメッセージアプリからも起動できるようにしてください。
SharePlay ボタンを UI に配置し、FaceTime 通話外でも
グループアクティビティを開始できるようにしてください。
```

```swift
import GroupActivities
import SwiftUI

struct SharePlayButton: View {
    @StateObject private var activityState
        = GroupStateObserver()
    @State private var isActivitySharingSheetPresented = false

    var body: some View {
        Button(action: {
            // FaceTime 通話中なら直接開始、
            // そうでなければ共有シートを表示
            if activityState.isEligibleForGroupSession {
                startActivity()
            } else {
                isActivitySharingSheetPresented = true
            }
        }) {
            Label("一緒に描く", systemImage: "shareplay")
                .font(.headline)
                .foregroundColor(.white)
                .padding(.horizontal, 20)
                .padding(.vertical, 12)
                .background(Color.blue)
                .cornerRadius(12)
        }
        .sheet(isPresented: $isActivitySharingSheetPresented) {
            // メッセージ経由で共有する場合のシート
            ActivitySharingSheet(
                activity: CollaborativeDrawingActivity()
            )
        }
    }

    private func startActivity() {
        Task {
            let activity = CollaborativeDrawingActivity()
            let result = try await activity.activate()
            // セッション開始をハンドリング
            print("アクティビティ開始: \(result)")
        }
    }
}

// 期待する出力:
// 「一緒に描く」ボタンタップで
// SharePlay セッション開始 or 共有シート表示
```

`GroupStateObserver` を使うことで、現在 FaceTime 通話中かどうかを確認し、適切な UI を出し分けることができます。

### パフォーマンス最適化

リアルタイム同期のパフォーマンスを向上させるには、次の工夫が有効です。

- **送信頻度のスロットリング**: 描画ポイントを間引いて送信量を削減
- **バッチ送信**: 複数のポイントをまとめて1回で送信
- **差分同期**: 変更があった部分のみを送信

```swift
// 送信頻度のスロットリング例
class ThrottledSender {
    private var pendingPoints: [CGPoint] = []
    private var lastSendTime: Date = .distantPast
    private let minInterval: TimeInterval = 0.05 // 50ms

    func addPoint(_ point: CGPoint) {
        pendingPoints.append(point)

        let now = Date()
        if now.timeIntervalSince(lastSendTime) >= minInterval {
            flush()
        }
    }

    private func flush() {
        guard !pendingPoints.isEmpty else { return }
        // バッチ送信
        let points = pendingPoints
        pendingPoints = []
        lastSendTime = Date()

        Task {
            await sessionManager.sendPoints(points)
        }
    }
}

// 期待する出力:
// 描画時のネットワーク使用量が約60%削減され、
// 遅延も50ms以下に抑えられる
```

## 実践的なユースケース

SharePlay と Group Activities を活用すれば、さまざまなジャンルのアプリに共有体験を組み込めます。

### クイズアプリ

友達同士でリアルタイムにクイズ対戦ができるアプリです。問題の表示、回答の送信、スコアの更新をすべて SharePlay で同期します。

### 共同プレイリスト

音楽の共有プレイリストを作成し、参加者全員がリアルタイムで曲を追加・投票できるアプリです。MusicKit と組み合わせることで、Apple Music の楽曲を SharePlay 上で同時再生できます。

### フィットネスチャレンジ

トレーニングメニューを共有しながら、友達と一緒にワークアウトするアプリです。HealthKit のデータをリアルタイムに共有し、お互いの進捗を確認できます。

### 旅行プランニング

地図上でピンを共有しながら、旅行先を一緒に計画するアプリです。MapKit と連携して、リアルタイムにルートや観光スポットを共有できます。

## よくある質問（FAQ）

### Q1: SharePlay は無料プランの Rork でも使えますか？

SharePlay を実装するにはネイティブ Swift コードの生成が必要なため、**Rork Max プラン** が必須です。無料プランの Rork は React Native / Expo ベースのコードを生成するため、Group Activities フレームワークを直接利用できません。Rork Max は月額 $200 のプランですが、SharePlay のようなネイティブ機能をフル活用できるメリットがあります。

### Q2: SharePlay は何人まで参加できますか？

Apple の公式ドキュメントによると、SharePlay セッションには **最大32人** まで参加可能です。ただし、リアルタイム描画のような高頻度なデータ同期を行うアプリでは、参加人数が増えるとパフォーマンスへの影響が出る可能性があります。4〜8人程度を推奨参加人数として設計すると、安定した体験を提供しやすくなります。

### Q3: SharePlay 対応アプリは App Store の審査で特別な対応が必要ですか？

基本的に通常のアプリ審査と同じプロセスです。ただし、Info.plist に `NSSupportsGroupActivities` を `YES` に設定する必要があります。Rork Max ではプロジェクト設定で自動的に追加されますが、念のため App Store 提出前に確認しておくと安心です。また、審査チームが SharePlay の動作を確認できるよう、テスト手順をレビューノートに記載することを推奨します。

### Q4: Android ユーザーとも SharePlay できますか？

SharePlay は Apple のエコシステム専用の機能であるため、Android ユーザーとの直接的な SharePlay は現在サポートされていません。クロスプラットフォームでリアルタイム共有体験を実現したい場合は、WebSocket や Firebase Realtime Database を使った独自の同期機能を Rork（通常版）で実装する方法があります。詳しくは [Rork × Supabase でリアルタイムチャットアプリを作る](/articles/rork-dev/rork-supabase-chat-app) をご参照ください。

## まとめ

SharePlay と Group Activities は、アプリに「一緒に使う体験」を加えることで、ユーザーのエンゲージメントと継続率を大幅に向上させるポテンシャルを持つ機能です。従来は実装のハードルが高かった SharePlay も、Rork Max のプロンプトベース開発なら効率的に構築できます。

本記事で紹介した共同描画アプリのパターンを応用すれば、クイズ、フィットネス、旅行プランニングなど、さまざまなジャンルで SharePlay 対応アプリを作ることが可能です。

まずはシンプルなデータ同期から始めて、段階的に機能を追加していくアプローチがおすすめです。SharePlay 対応は App Store での差別化ポイントにもなるので、ぜひ挑戦してみてください。

Rork Max の基本から復習したい方は [Rork Max と通常版 Rork の違い](/articles/rork-basics/rork-vs-rork-max) も合わせてご覧ください。また、アプリの収益化については Rork アプリ収益化完全ガイド も参考になります。
