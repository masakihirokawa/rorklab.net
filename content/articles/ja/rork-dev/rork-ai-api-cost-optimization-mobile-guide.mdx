## はじめに — AI APIコストがアプリの収益を圧迫する問題

モバイルアプリにAI機能を組み込むことは、もはや差別化要因ではなく標準的な要件になりつつあります。チャットボット、画像認識、テキスト要約、パーソナライズされたレコメンデーション——これらの機能は、ユーザー体験を大幅に向上させる一方で、API呼び出しのたびに発生する従量課金が、特に個人開発者にとって深刻なコスト問題を引き起こします。

Rorkで開発したアプリにClaude、Gemini、GPT-4oなどのAI APIを統合する場合、ユーザー数が増えるほどAPI費用は指数関数的に膨らみます。月間1,000人のアクティブユーザーが1日平均10回AIリクエストを送ると、トークン消費量だけで月額数万円に達するケースも珍しくありません。

この記事では、RorkアプリでAI APIコストを最小限に抑えるための実践的なテクニックを、コード例とともに具体的に解説します。キャッシュ戦略、プロンプト最適化、オンデバイスAIの活用、バッチ処理まで、すぐに使える手法を網羅しています。

## レスポンスキャッシュで同一リクエストの重複課金を防ぐ

AI APIコスト削減で最も即効性があるのが、レスポンスキャッシュの導入です。多くのアプリでは、同じような質問や同一の入力に対して繰り返しAPIを呼び出しており、これが無駄なコストの最大の原因になっています。

キャッシュ戦略は、リクエストの内容をハッシュ化してキーとし、レスポンスをローカルストレージやサーバー側のKVストアに保存するのが基本です。

```typescript
// AI APIレスポンスキャッシュの実装例
import AsyncStorage from '@react-native-async-storage/async-storage';
import * as Crypto from 'expo-crypto';

interface CacheEntry {
  response: string;
  timestamp: number;
  tokens_used: number;
}

const CACHE_TTL = 24 * 60 * 60 * 1000; // 24時間

async function getCachedAIResponse(prompt: string): Promise<string | null> {
  const hash = await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    prompt.trim().toLowerCase()
  );
  const key = `ai_cache_${hash}`;

  try {
    const cached = await AsyncStorage.getItem(key);
    if (cached) {
      const entry: CacheEntry = JSON.parse(cached);
      if (Date.now() - entry.timestamp < CACHE_TTL) {
        console.log(`Cache hit — saved ${entry.tokens_used} tokens`);
        return entry.response;
      }
    }
  } catch (error) {
    console.warn('Cache read error:', error);
  }
  return null;
}

async function callAIWithCache(prompt: string): Promise<string> {
  // キャッシュチェック
  const cached = await getCachedAIResponse(prompt);
  if (cached) return cached;

  // APIを呼び出し
  const response = await fetch('https://api.example.com/ai/chat', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` },
    body: JSON.stringify({ prompt, max_tokens: 500 }),
  });

  const data = await response.json();
  // レスポンスをキャッシュに保存
  const hash = await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    prompt.trim().toLowerCase()
  );
  await AsyncStorage.setItem(`ai_cache_${hash}`, JSON.stringify({
    response: data.text,
    timestamp: Date.now(),
    tokens_used: data.usage.total_tokens,
  }));

  return data.text;
}
```

このキャッシュ層を導入するだけで、典型的なアプリではAPI呼び出しの30〜50%を削減できます。FAQ系の機能やカテゴリ分類など、入力パターンが限られる機能では、キャッシュヒット率が80%を超えることも珍しくありません。

## プロンプト圧縮とトークン削減の具体的テクニック

AI APIの料金はトークン数に比例するため、同じ結果を得るためにより少ないトークンで済むプロンプトを設計することが、コスト削減の鍵になります。

プロンプト圧縮のポイントは以下の3つです。

**システムプロンプトの最適化**: 冗長な指示を簡潔にまとめます。「あなたは親切なアシスタントです。ユーザーの質問に丁寧に答えてください。回答は正確で、わかりやすくしてください」のような記述は、「簡潔で正確に回答」の一言で置き換えられます。

**コンテキストウィンドウの管理**: チャット履歴を全て送信するのではなく、直近の3〜5ターンだけを送信する、または要約を送信する方法に切り替えます。

**出力トークンの制限**: `max_tokens` パラメータを必要最小限に設定します。

```typescript
// プロンプト最適化ユーティリティ
function optimizeConversationHistory(
  messages: Array<{ role: string; content: string }>,
  maxTurns: number = 4,
  maxTokensEstimate: number = 1000
): Array<{ role: string; content: string }> {
  // システムプロンプトは常に保持
  const system = messages.filter(m => m.role === 'system');
  const conversation = messages.filter(m => m.role !== 'system');

  // 直近のターンのみ保持
  const recent = conversation.slice(-maxTurns * 2);

  // トークン数の概算（日本語: 1文字≒1.5トークン / 英語: 1ワード≒1.3トークン）
  let totalChars = system.reduce((sum, m) => sum + m.content.length, 0);
  const filtered = [];

  for (const msg of recent) {
    totalChars += msg.content.length;
    if (totalChars * 1.5 > maxTokensEstimate) break;
    filtered.push(msg);
  }

  return [...system, ...filtered];
}

// 使用例: トークン消費を約60%削減
const optimized = optimizeConversationHistory(chatHistory, 4, 800);
const response = await callAI(optimized);
```

さらに、レスポンスフォーマットを指定することで出力トークンも削減できます。「JSON形式で返してください」と指定すると、自然言語の説明文が省略され、出力トークン数が大幅に減少します。

## オンデバイスAIでAPI呼び出し自体を減らす

コスト最適化の究極の手段は、API呼び出し自体を減らすことです。テキスト分類、センチメント分析、簡単な質問応答など、軽量なタスクはオンデバイスのMLモデルで処理し、複雑なタスクだけをクラウドAPIに委ねる「ティアード（段階的）AI」アーキテクチャが効果的です。

```typescript
// ティアードAIアーキテクチャの実装例
type AITier = 'on-device' | 'cloud-light' | 'cloud-heavy';

interface TaskClassification {
  tier: AITier;
  estimatedCost: number; // USD
}

function classifyTask(input: string, taskType: string): TaskClassification {
  // タスクの複雑さに応じてティアを決定
  const simplePatterns = ['分類', '感情分析', 'はい/いいえ', 'カテゴリ'];
  const mediumPatterns = ['要約', '翻訳', 'キーワード抽出'];

  if (simplePatterns.some(p => taskType.includes(p))) {
    return { tier: 'on-device', estimatedCost: 0 };
  }
  if (mediumPatterns.some(p => taskType.includes(p))) {
    return { tier: 'cloud-light', estimatedCost: 0.001 };
  }
  return { tier: 'cloud-heavy', estimatedCost: 0.01 };
}

async function processWithTieredAI(
  input: string,
  taskType: string
): Promise<string> {
  const { tier } = classifyTask(input, taskType);

  switch (tier) {
    case 'on-device':
      // TensorFlow Lite / Core MLで処理（コスト: $0）
      return await runOnDeviceModel(input, taskType);

    case 'cloud-light':
      // 軽量モデル（Claude Haiku / Gemini Flash）で処理
      return await callLightweightAPI(input);

    case 'cloud-heavy':
      // 高性能モデル（Claude Opus / GPT-4o）で処理
      return await callPremiumAPI(input);
  }
}
```

このアプローチでは、リクエストの50〜70%をオンデバイスで処理することで、API費用を大幅に圧縮できます。より詳しいオンデバイスAI実装については、[Rork × Core ML カスタムモデル開発 × オンデバイスAI 完全ガイド](/articles/rork-ai/rork-core-ml-custom-model-on-device-ai-guide) で解説しています。

## バッチ処理とリクエスト集約で効率を最大化する

個別のAPIリクエストを逐一送信するのではなく、複数のリクエストをまとめてバッチ処理することで、オーバーヘッドを削減し、コスト効率を高められます。

```typescript
// バッチリクエストマネージャー
class AIBatchProcessor {
  private queue: Array<{
    prompt: string;
    resolve: (value: string) => void;
    reject: (reason: any) => void;
  }> = [];
  private timer: NodeJS.Timeout | null = null;
  private readonly batchSize = 5;
  private readonly batchDelay = 2000; // 2秒待ってからバッチ送信

  async add(prompt: string): Promise<string> {
    return new Promise((resolve, reject) => {
      this.queue.push({ prompt, resolve, reject });

      if (this.queue.length >= this.batchSize) {
        this.flush();
      } else if (!this.timer) {
        this.timer = setTimeout(() => this.flush(), this.batchDelay);
      }
    });
  }

  private async flush(): Promise<void> {
    if (this.timer) {
      clearTimeout(this.timer);
      this.timer = null;
    }

    const batch = this.queue.splice(0, this.batchSize);
    if (batch.length === 0) return;

    try {
      // 複数プロンプトを1回のAPI呼び出しで処理
      const combinedPrompt = batch.map(
        (item, i) => `[Task ${i + 1}]: ${item.prompt}`
      ).join('\n---\n');

      const response = await fetch('https://api.example.com/ai/chat', {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` },
        body: JSON.stringify({
          prompt: `以下の${batch.length}件のタスクにそれぞれ回答してください。\n${combinedPrompt}`,
          max_tokens: batch.length * 300,
        }),
      });

      const data = await response.json();
      // レスポンスを分割して各リクエストに配布
      const results = parseMultiResponse(data.text, batch.length);
      batch.forEach((item, i) => item.resolve(results[i]));
    } catch (error) {
      batch.forEach(item => item.reject(error));
    }
  }
}

// 使用例
const batchProcessor = new AIBatchProcessor();
// 複数のUI要素が個別にAIを呼び出しても、内部でバッチ化される
const [summary, tags, sentiment] = await Promise.all([
  batchProcessor.add('この記事を3行で要約して'),
  batchProcessor.add('この記事のタグを5つ抽出して'),
  batchProcessor.add('この記事のセンチメントを判定して'),
]);
```

バッチ処理によって、3回のAPI呼び出しが1回に集約されます。APIのリクエスト数が減ることで、レートリミットにも余裕が生まれ、結果的にアプリ全体のレスポンスも安定します。

## コスト監視ダッシュボードを実装して予算超過を防ぐ

コスト最適化の大前提として、現在どれだけのAPI費用が発生しているかを可視化する仕組みが必要です。Supabaseと組み合わせることで、リアルタイムのコスト監視ダッシュボードを構築できます。

```typescript
// APIコスト追跡ユーティリティ
interface UsageRecord {
  user_id: string;
  model: string;
  input_tokens: number;
  output_tokens: number;
  cost_usd: number;
  timestamp: string;
}

const MODEL_PRICING: Record<string, { input: number; output: number }> = {
  'claude-opus':   { input: 0.015,  output: 0.075  }, // per 1K tokens
  'claude-sonnet': { input: 0.003,  output: 0.015  },
  'claude-haiku':  { input: 0.00025, output: 0.00125 },
  'gemini-pro':    { input: 0.00125, output: 0.005  },
  'gpt-4o':        { input: 0.005,  output: 0.015  },
};

function calculateCost(
  model: string,
  inputTokens: number,
  outputTokens: number
): number {
  const pricing = MODEL_PRICING[model];
  if (!pricing) return 0;
  return (inputTokens / 1000) * pricing.input
       + (outputTokens / 1000) * pricing.output;
}

// Supabaseに使用量を記録
async function trackUsage(record: UsageRecord): Promise<void> {
  await supabase.from('ai_usage').insert(record);

  // 月間予算チェック
  const { data } = await supabase
    .from('ai_usage')
    .select('cost_usd')
    .gte('timestamp', getFirstDayOfMonth());

  const totalCost = data?.reduce((sum, r) => sum + r.cost_usd, 0) ?? 0;
  if (totalCost > MONTHLY_BUDGET * 0.8) {
    await sendBudgetAlert(totalCost, MONTHLY_BUDGET);
  }
}
```

このように使用量を追跡し、月間予算の80%に達した時点でアラートを発信することで、予期しないコスト超過を未然に防げます。

## FAQ

**Q: AI APIのコスト最適化で最も効果が高いテクニックはどれですか？**

A: 最も即効性が高いのはレスポンスキャッシュの導入です。一般的なアプリでは30〜50%のリクエストが類似または同一であり、キャッシュだけでその分のAPI費用をゼロにできます。次に効果的なのが、オンデバイスAIとクラウドAPIを使い分けるティアードアーキテクチャです。

**Q: キャッシュのTTL（有効期限）はどのくらいに設定すべきですか？**

A: コンテンツの性質によります。FAQ回答やカテゴリ分類のように結果が安定しているものは24〜72時間、ニュース要約のようにリアルタイム性が重要なものは1〜6時間が目安です。ユーザーに手動リフレッシュ機能を提供し、必要なときだけキャッシュを破棄できるようにするのもおすすめです。

**Q: オンデバイスAIモデルはどの程度の精度が期待できますか？**

A: テキスト分類やセンチメント分析など、タスクが明確なものでは90%以上の精度が得られます。ただし、複雑な推論や創造的なテキスト生成はクラウドAPIに劣るため、タスクの特性に応じた使い分けが重要です。Core MLやTensorFlow Liteの量子化モデルを使えば、端末のメモリ消費も抑えられます。

**Q: 個人開発のアプリで月間のAI API予算の目安はいくらですか？**

A: ユーザー数とリクエスト頻度に依存しますが、月間アクティブユーザー（MAU）1,000人規模なら、本記事の最適化テクニックを適用した上で月額$10〜$30が現実的な目安です。キャッシュとオンデバイスAIを組み合わせれば、最適化前と比べて60〜80%のコスト削減が見込めます。
