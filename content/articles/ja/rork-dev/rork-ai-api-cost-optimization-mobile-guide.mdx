## はじめに — AI機能のコストが個人開発者を悩ませる理由

Rorkを使えば、AIチャットボットやレコメンデーション、画像認識といった高度な機能をアプリに組み込むことが驚くほど簡単になります。しかし、リリース後にユーザーが増えてくると直面するのが「AI APIの利用コスト」という現実的な課題です。

GPT-4oやGemini、Claude APIの従量課金は、1リクエストあたりの費用は小さくても、月間数万リクエストになると無視できない金額に膨らみます。個人開発者にとって、月の利益がそのままAPI費用に消えてしまうのは避けたいところです。

この記事では、Rorkで開発するモバイルアプリにAI機能を実装する際、コストを最小限に抑えながらユーザー体験を維持するための具体的なテクニックを解説します。トークン数の最適化からレスポンスキャッシュ、オンデバイス推論への切り替え、そしてコスト監視の仕組みづくりまで、実践的なアプローチをお伝えします。

## AI APIのコスト構造を理解する

コスト最適化の第一歩は、APIの課金体系を正しく理解することです。主要なAI APIの料金は、入力トークンと出力トークンの合計で決まります。

- **入力トークン**: プロンプト（システムプロンプト＋ユーザー入力＋会話履歴）に含まれるトークン数
- **出力トークン**: AIが生成するレスポンスのトークン数（入力より単価が高い傾向）
- **モデルの選択**: GPT-4oとGPT-4o-miniでは10倍以上のコスト差がある

たとえば、チャットボットアプリで1回のやり取りに平均2,000トークン（入力1,500＋出力500）を消費する場合、1日1,000ユーザーが平均5回会話すると月間で約1億5,000万トークンになります。モデル選択を誤ると、月額数十万円のコストが発生しかねません。

### コスト計算の実装例

Rorkアプリ内でAPIコストを見積もるユーティリティ関数を用意しておくと、開発段階から意識的にコスト管理ができます。

```typescript
// utils/aiCostEstimator.ts
// AI APIのコスト計算ユーティリティ

interface CostRate {
  inputPer1kTokens: number;
  outputPer1kTokens: number;
}

// 主要モデルの料金表（USD）— 2026年3月時点
const MODEL_RATES: Record<string, CostRate> = {
  'gpt-4o': { inputPer1kTokens: 0.0025, outputPer1kTokens: 0.01 },
  'gpt-4o-mini': { inputPer1kTokens: 0.00015, outputPer1kTokens: 0.0006 },
  'claude-sonnet': { inputPer1kTokens: 0.003, outputPer1kTokens: 0.015 },
  'gemini-pro': { inputPer1kTokens: 0.00125, outputPer1kTokens: 0.005 },
};

export function estimateCost(
  model: string,
  inputTokens: number,
  outputTokens: number
): number {
  const rate = MODEL_RATES[model];
  if (!rate) return 0;
  return (
    (inputTokens / 1000) * rate.inputPer1kTokens +
    (outputTokens / 1000) * rate.outputPer1kTokens
  );
}

// 使用例: estimateCost('gpt-4o-mini', 1500, 500)
// 結果: 約 $0.000525（1リクエストあたり）
```

## トークン数を減らす5つの実践テクニック

コスト削減で最も効果的なのは、そもそもAPIに送信するトークン数を減らすことです。

### 1. システムプロンプトの圧縮

長文のシステムプロンプトは毎回のリクエストに含まれるため、最もコストに影響します。冗長な説明を削ぎ落とし、箇条書きで簡潔にまとめましょう。

```typescript
// ❌ 冗長なプロンプト（約200トークン）
const verbosePrompt = `
あなたはフィットネスアプリのアシスタントです。
ユーザーの健康管理をサポートする役割を担っています。
ユーザーが質問をしたら、丁寧に回答してください。
回答は分かりやすく、専門用語は避けてください。
健康に関する重要なアドバイスをする際は、
医師への相談を推奨してください。
`;

// ✅ 圧縮版（約80トークン）
const concisePrompt = `役割:フィットネスアシスタント。簡潔・平易に回答。医療助言時は受診推奨を付記。`;
```

### 2. 会話履歴のウィンドウ制限

チャット機能では会話が長くなるほどトークン数が増加します。直近のN件だけをAPIに送る「スライディングウィンドウ」方式が有効です。

```typescript
// hooks/useChatHistory.ts
// 会話履歴をスライディングウィンドウで管理

const MAX_HISTORY_TURNS = 6; // 直近6往復のみ保持

function trimHistory(messages: Message[]): Message[] {
  const systemMsg = messages.find(m => m.role === 'system');
  const conversations = messages.filter(m => m.role !== 'system');

  // 直近のやり取りのみ残す
  const trimmed = conversations.slice(-MAX_HISTORY_TURNS * 2);

  return systemMsg ? [systemMsg, ...trimmed] : trimmed;
}
```

### 3. ユーザー入力の前処理

APIに送信する前に、不要な空白や繰り返しを除去するだけでもトークン数を削減できます。

### 4. レスポンス長の制御

`max_tokens` パラメータを用途に合わせて適切に設定します。商品説明の生成なら200トークン、チャット応答なら500トークンなど、場面ごとに上限を決めておくとコストの予測が立てやすくなります。

### 5. モデルのタスク別使い分け

すべてのリクエストに高性能モデルを使う必要はありません。タスクの複雑さに応じてモデルを切り替える「ルーティング」パターンが効果的です。

```typescript
// utils/modelRouter.ts
// タスクの複雑さに応じてモデルを自動選択

type TaskComplexity = 'simple' | 'moderate' | 'complex';

function selectModel(task: TaskComplexity): string {
  switch (task) {
    case 'simple':
      // 分類・要約・定型応答 → 軽量モデル
      return 'gpt-4o-mini';
    case 'moderate':
      // 一般的なチャット応答 → 中間モデル
      return 'gemini-pro';
    case 'complex':
      // コード生成・高度な推論 → 高性能モデル
      return 'gpt-4o';
  }
}

// 使用例: 感情分析は simple、レシピ提案は moderate
```

## レスポンスキャッシュでAPI呼び出しを削減する

同じ質問や類似のリクエストに対して毎回APIを呼び出すのは無駄です。キャッシュ戦略を導入することで、API呼び出し回数を大幅に減らせます。

### 完全一致キャッシュの実装

```typescript
// services/aiCache.ts
// AsyncStorageを使ったAIレスポンスキャッシュ

import AsyncStorage from '@react-native-async-storage/async-storage';

const CACHE_PREFIX = 'ai_cache_';
const CACHE_TTL = 24 * 60 * 60 * 1000; // 24時間

interface CachedResponse {
  response: string;
  timestamp: number;
  model: string;
}

// プロンプトからキャッシュキーを生成
function getCacheKey(prompt: string): string {
  let hash = 0;
  for (let i = 0; i < prompt.length; i++) {
    const char = prompt.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash |= 0;
  }
  return `${CACHE_PREFIX}${Math.abs(hash).toString(36)}`;
}

export async function getCachedResponse(
  prompt: string
): Promise<string | null> {
  try {
    const key = getCacheKey(prompt);
    const cached = await AsyncStorage.getItem(key);
    if (!cached) return null;

    const data: CachedResponse = JSON.parse(cached);
    // TTL超過なら無効化
    if (Date.now() - data.timestamp > CACHE_TTL) {
      await AsyncStorage.removeItem(key);
      return null;
    }
    return data.response;
  } catch {
    return null;
  }
}

export async function setCachedResponse(
  prompt: string,
  response: string,
  model: string
): Promise<void> {
  const key = getCacheKey(prompt);
  const data: CachedResponse = {
    response,
    timestamp: Date.now(),
    model,
  };
  await AsyncStorage.setItem(key, JSON.stringify(data));
}
```

キャッシュのヒット率を高めるコツは、ユーザー固有の情報（名前や日付など）をプロンプトから分離し、テンプレート部分をキャッシュキーに使うことです。たとえばレシピ提案アプリなら「鶏肉 簡単 30分以内」という条件をキーにして、ユーザー名は後から差し込む設計にすると、異なるユーザーでもキャッシュを共有できます。

## オンデバイス推論でAPIコストをゼロにする

コスト削減の究極の手段は、クラウドAPIを呼ばずにデバイス上で推論を完結させることです。Rork MaxではCore MLモデルをアプリに組み込めるため、特定のタスクをオフライン・無料で処理できます。

### オンデバイス推論が向いているタスク

- **テキスト分類**（感情分析・スパム判定・カテゴリ分け）
- **画像分類**（写真のタグ付け・オブジェクト検出）
- **音声認識**（Whisperベースの文字起こし）
- **キーワード抽出**（テキストからの重要語句抽出）

逆に、長文生成やマルチターン会話など、大規模言語モデルの能力が必要なタスクにはクラウドAPIが適しています。

```typescript
// services/onDeviceClassifier.ts
// Core MLを使ったオンデバイステキスト分類

import { NativeModules } from 'react-native';

const { CoreMLBridge } = NativeModules;

export async function classifySentiment(
  text: string
): Promise<'positive' | 'negative' | 'neutral'> {
  try {
    // Core MLモデルによるオンデバイス推論（APIコスト: $0）
    const result = await CoreMLBridge.predict('SentimentClassifier', {
      text,
    });
    return result.label;
  } catch (error) {
    // フォールバック: クラウドAPIを使用
    console.warn('On-device inference failed, falling back to API');
    return classifySentimentViaAPI(text);
  }
}
```

オンデバイスAIの実装についてより詳しく知りたい方は、[Rork × Core ML カスタムモデル開発 × オンデバイスAI 完全ガイド](/articles/rork-ai/rork-core-ml-custom-model-on-device-ai-guide) で設計パターンからデプロイまでを体系的に解説しています。

## コスト監視ダッシュボードを構築する

最適化を継続するためには、コストの可視化が欠かせません。Supabaseを使ってAPIの利用状況を記録し、管理画面で確認できる仕組みを作りましょう。

```typescript
// services/costTracker.ts
// API利用ログをSupabaseに記録

import { supabase } from './supabaseClient';

interface UsageLog {
  model: string;
  input_tokens: number;
  output_tokens: number;
  cost_usd: number;
  feature: string; // どの機能から呼ばれたか
  cached: boolean;
}

export async function logApiUsage(usage: UsageLog): Promise<void> {
  await supabase.from('ai_usage_logs').insert({
    ...usage,
    created_at: new Date().toISOString(),
  });
}

// 日次コストサマリーを取得
export async function getDailyCostSummary(
  date: string
): Promise<{ totalCost: number; totalRequests: number; cacheHitRate: number }> {
  const { data } = await supabase
    .from('ai_usage_logs')
    .select('cost_usd, cached')
    .gte('created_at', `${date}T00:00:00`)
    .lt('created_at', `${date}T23:59:59`);

  if (!data) return { totalCost: 0, totalRequests: 0, cacheHitRate: 0 };

  const totalCost = data.reduce((sum, d) => sum + d.cost_usd, 0);
  const cachedCount = data.filter(d => d.cached).length;

  return {
    totalCost,
    totalRequests: data.length,
    cacheHitRate: data.length > 0 ? cachedCount / data.length : 0,
  };
}
```

月額の予算上限を設定し、閾値を超えたらアラートを送る仕組みも加えておくと安心です。予期せぬトラフィック急増によるコスト爆発を防ぐことができます。

## よくある質問（FAQ）

### Q1: AIのAPIコストはどのくらいが目安ですか？

個人開発のアプリであれば、月額$10〜$50程度に収まるのが理想的です。DAU（日間アクティブユーザー）1,000人規模のチャットアプリの場合、この記事で紹介した最適化テクニック（モデル選択＋キャッシュ＋トークン削減）を組み合わせることで、月額$20〜$30程度に抑えられるケースが多いです。

### Q2: キャッシュを導入するとユーザー体験に影響はありませんか？

適切に実装すれば、むしろユーザー体験は向上します。キャッシュヒット時はAPI呼び出しが不要なのでレスポンスが瞬時に返り、体感速度が大幅に改善されます。TTL（有効期限）を用途に合わせて設定し、古い情報が表示されないよう注意しましょう。

### Q3: オンデバイス推論のデメリットは何ですか？

主なデメリットは、モデルサイズによるアプリ容量の増加と、デバイス性能への依存です。Core MLモデルは数MB〜数百MBになるため、アプリのダウンロードサイズに影響します。また、古いデバイスでは推論速度が遅くなる可能性があります。重要なタスクにはクラウドAPIへのフォールバックを必ず用意しておきましょう。

### Q4: 複数のAIモデルを組み合わせるのは複雑になりませんか？

モデルルーティングの仕組みを最初に作っておけば、呼び出し側のコードはシンプルに保てます。タスクの種類をenumで定義し、ルーターが適切なモデルを自動選択する設計にすることで、各機能のコードは「何をしたいか」だけを指定すれば済むようになります。

## まとめ

AI APIのコスト最適化は、個人開発者がAI搭載アプリを持続的に運営するための重要なスキルです。この記事で紹介した4つの柱を組み合わせることで、コストを60〜80%削減しながらユーザー体験を維持できます。

- **トークン最適化**: プロンプト圧縮・履歴制限・モデルルーティングで無駄を排除
- **キャッシュ戦略**: 同一・類似リクエストの再利用でAPI呼び出しを削減
- **オンデバイス推論**: 分類・検出タスクをCore MLに移行してコストをゼロに
- **コスト監視**: 利用状況を可視化し、予算超過を未然に防止

Rorkなら、これらの最適化テクニックをアプリに段階的に組み込んでいくことが可能です。まずはモデルの使い分けとキャッシュから始めて、効果を確認しながらオンデバイス推論にも挑戦してみてください。
