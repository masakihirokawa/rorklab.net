## はじめに — アプリ収益化の第一歩

個人開発でアプリを作ったものの、収益化の方法に悩んでいる方は多いのではないでしょうか。広告モデルは収益が不安定になりがちですし、買い切りモデルでは継続的な収入を得にくいという課題があります。

サブスクリプション（定期課金）は、安定した収益基盤を構築するための最も効果的な手段の一つです。そして Rork を使えば、Stripe と連携したサブスクリプション機能を驚くほど簡単に実装できます。

本記事では、Rork プロジェクトの作成から Stripe の設定、実際の課金フローの実装、テスト決済の確認まで、サブスクリプション機能の導入に必要な全工程をカバーします。

## 前提知識と環境準備

### 必要なもの

実装を始める前に、以下を準備してください。

- **Rork アカウント**: [rork.com](https://rork.com) で無料アカウントを作成
- **Stripe アカウント**: [stripe.com](https://stripe.com) でアカウントを作成（テストモードで開始）
- **Node.js 18以上**: ローカル開発用
- **Expo Go アプリ**: 実機テスト用（iOS / Android）

### Rork プロジェクトの初期設定

```bash
# Rork CLI でプロジェクトを作成
npx create-rork-app my-subscription-app
cd my-subscription-app

# Stripe関連パッケージをインストール
npm install @stripe/stripe-react-native
npm install stripe  # サーバーサイド用

# 環境変数ファイルを作成
cat > .env << EOF
STRIPE_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxx
STRIPE_SECRET_KEY=sk_test_xxxxxxxxxxxxx
STRIPE_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxx
EOF

# 期待される出力:
# ✅ Project created successfully
# ✅ Dependencies installed
```

Stripe のキーはダッシュボードの「開発者」→「APIキー」から取得できます。本番デプロイ前にテストキーからライブキーに切り替えることを忘れないでください。

## Stripe 商品とプランの設定

### Stripe ダッシュボードでの設定

Stripe ダッシュボードで商品（Product）と価格（Price）を作成します。

```bash
# Stripe CLI を使ってコマンドラインから作成することも可能
stripe products create \
  --name="Pro Plan" \
  --description="全機能アクセス + 優先サポート"

# 月額プランの価格を設定
stripe prices create \
  --product=prod_xxxxx \
  --unit-amount=380 \
  --currency=jpy \
  --recurring[interval]=month

# 年額プランの価格を設定（2ヶ月分お得）
stripe prices create \
  --product=prod_xxxxx \
  --unit-amount=3800 \
  --currency=jpy \
  --recurring[interval]=year

# 期待される出力:
# {
#   "id": "price_xxxxxxxxxxxxx",
#   "object": "price",
#   "active": true,
#   "currency": "jpy",
#   "unit_amount": 380,
#   ...
# }
```

      const response = await fetch("/api/create-checkout-session", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ priceId: plan.priceId }),
      });

      const { sessionId } = await response.json();
      // Stripe Checkoutにリダイレクト（またはアプリ内決済）
      // ...
    } catch (error) {
      Alert.alert("エラー", "決済処理中にエラーが発生しました。");

      setLoading(false);
    }

    <View style={styles.container}>
      <Text style={styles.title}>プランを選択</Text>

### サーバーサイド API エンドポイント

```typescript
// api/create-checkout-session.ts
import Stripe from "stripe";

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: "2025-12-18.acacia",
});

export async function POST(request: Request) {
  try {
    const { priceId } = await request.json();

    const session = await stripe.checkout.sessions.create({
      mode: "subscription",
      payment_method_types: ["card"],
      line_items: [{ price: priceId, quantity: 1 }],
      success_url: `${process.env.APP_URL}/subscription/success?session_id={CHECKOUT_SESSION_ID}`,
      cancel_url: `${process.env.APP_URL}/subscription/cancel`,
      metadata: {
        plan_type: "pro",  // チップと区別するために必須
      },
    });

    return Response.json({ sessionId: session.id });
  } catch (error) {
    console.error("Checkout session creation failed:", error);
    return Response.json(
      { error: "Failed to create checkout session" },
      { status: 500 }
    );
  }
}
```

## Webhook によるサブスクリプション管理

Stripe からのイベント通知を受け取り、サブスクリプションの状態を管理します。

```typescript
// api/webhook.ts
import Stripe from "stripe";

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: "2025-12-18.acacia",
});

export async function POST(request: Request) {
  const body = await request.text();
  const signature = request.headers.get("stripe-signature")!;

  let event: Stripe.Event;
  try {
    event = stripe.webhooks.constructEvent(
      body,
      signature,
      process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    return Response.json({ error: "Webhook signature verification failed" }, { status: 400 });
  }

  // イベントタイプに応じた処理
  switch (event.type) {
    case "checkout.session.completed": {
      const session = event.data.object as Stripe.Checkout.Session;
      // ユーザーのサブスクリプション状態を更新
      await updateUserSubscription(session.customer as string, "active");
      break;
    }
    case "customer.subscription.updated": {
      const subscription = event.data.object as Stripe.Subscription;
      await updateUserSubscription(
        subscription.customer as string,
        subscription.status
      );
      break;
    }
    case "customer.subscription.deleted": {
      const subscription = event.data.object as Stripe.Subscription;
      await updateUserSubscription(subscription.customer as string, "canceled");
      break;
    }
  }

  return Response.json({ received: true });
}

// 期待される動作:
// 1. ユーザーが決済完了 → checkout.session.completed → ステータス: active
// 2. プラン変更 → customer.subscription.updated → ステータス更新
// 3. 解約 → customer.subscription.deleted → ステータス: canceled
```

## テスト決済の実行

実装が完了したら、Stripe のテストモードで決済フローを確認します。

```bash
# Stripe CLI でWebhookをローカルに転送
stripe listen --forward-to localhost:3000/api/webhook

# テスト用カード番号
# 成功: 4242 4242 4242 4242
# 残高不足: 4000 0000 0000 9995
# 認証必要: 4000 0025 0000 3155

# Webhook イベントの手動トリガー（テスト用）
stripe trigger checkout.session.completed
```

テストモードでは実際の課金は発生しません。本番デプロイ前に、成功・失敗・認証要求の3パターンを必ずテストしてください。

## よくある質問（FAQ）

### Q1: Apple / Google のアプリ内課金と Stripe、どちらを使うべきですか？

デジタルコンテンツやアプリ内機能のアンロックには、各ストアのアプリ内課金（IAP）を使用する必要があります（App Store / Google Play のポリシー）。一方、物理商品やアプリ外のサービス（SaaS、Web サービスへのアクセス権など）には Stripe を使用できます。Rork でサブスクリプションを実装する場合、Webベースの課金フロー（Stripe Checkout）を使うことで、ストア手数料（15〜30%）を回避できるケースがあります。

### Q2: サブスクリプションの解約処理はどう実装しますか？

Stripe の Customer Portal を利用するのが最も簡単です。`stripe.billingPortal.sessions.create()` でポータルURLを生成し、ユーザーをリダイレクトします。解約・プラン変更・支払い方法の更新を Stripe 側のUIで処理できるため、自前で管理画面を作る必要がありません。

### Q3: 無料トライアル期間は設定できますか？

はい、Stripe の Price 作成時に `trial_period_days` を指定するか、Checkout Session 作成時に `subscription_data.trial_period_days` を設定します。トライアル期間中は課金されず、期間終了後に自動的に初回課金が行われます。トライアル終了前にメール通知を送る仕組みも Stripe が標準で提供しています。

## まとめ

Rork と Stripe を組み合わせることで、個人開発のアプリにも本格的なサブスクリプション課金機能を導入できます。本記事で紹介した実装パターンは、月額プラン・年額プランの選択画面から、Webhook によるステータス管理、テスト決済まで、実際のリリースに必要な全工程をカバーしています。
