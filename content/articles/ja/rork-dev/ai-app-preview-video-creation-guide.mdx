## はじめに

アプリの成功を左右する最初の接点が、App Store や Google Play のプレビュー動画です。統計によると、プレビュー動画付きのアプリはダウンロード数が30〜50%増加すると言われています。

しかし、従来の動画制作には大きな課題がありました：

- **時間がかかる** — デバイスでの画面録画、編集、エフェクト追加に数日要する
- **コストが高い** — プロの動画制作会社に依頼すると数万円以上
- **スキルが必要** — 動画編集ソフト（After Effects等）の習得が必須
- **修正が大変** — テキスト変更や言語追加にはすべてやり直し

**それが、AI動画生成ツールで激変しています。**

テキストプロンプトやスクリーンショット数枚で、数分でプロ品質の動画が生成できるようになりました。本ガイドでは、Runway ML、Synthesia、D-ID といった最新AIツールを組み合わせて、画面録画なしでApp Store / Google Playのプレビュー動画を制作する手順を、実践的に解説します。

## App Store / Google Play のプレビュー動画仕様

プレビュー動画を制作する前に、各プラットフォームの厳密な仕様を理解する必要があります。

### App Store の要件

| 項目 | 要件 |
|---|---|
| **ファイル形式** | MP4（H.264コーデック推奨） |
| **解像度** | iPhone: 1080×1920px（縦型） |
| | iPad: 1536×2048px（縦型） |
| **フレームレート** | 30fps または 60fps |
| **ビットレート** | 5〜10 Mbps 推奨 |
| **最大ファイルサイズ** | 4GB |
| **動画長** | 15秒〜30秒 推奨（最大30秒） |
| **字幕** | オプション（多言語対応時は字幕必須） |

### Google Play の要件

| 項目 | 要件 |
|---|---|
| **ファイル形式** | MP4, WebM |
| **解像度** | 縦型: 1080×1920px 推奨 |
| | 横型: 1920×1080px 対応 |
| **フレームレート** | 24〜60fps |
| **最大ファイルサイズ** | 500MB |
| **動画長** | 最大45秒 |
| **音声** | AACオーディオ 128kbps推奨 |

**ポイント**: 両プラットフォーム対応するなら、**1080×1920px、30fps、MP4形式** で統一するのがおすすめです。

## AIツールで動画を生成する手順

### Runway ML を使った動画生成

Runway ML は、テキストプロンプトから高品質な動画を生成できるAIツールです。Gen-3 モデルを使うと、複雑なアニメーションやトランジションも自動生成できます。

**ステップ1: Runway MLで動画シーンを生成**

```
プロンプト例:
"モダンなスマートフォンアプリのインターフェース。
画面上にUIエレメントが次々と出現し、
タップアニメーション、スムーズなトランジション。
明るいグラデーション背景。
プロダクトデモ動画スタイル。"
```

生成パラメータ:
- **Duration**: 15秒
- **Resolution**: 1080×1920 (縦)
- **FPS**: 30
- **Style**: Professional / Cinematic

### Synthesia を使った音声付き動画生成

Synthesia はAIアバターと音声を自動生成し、プレゼンテーション動画を作成できます。アプリの機能説明や利点を、自然な喋り方で説明する動画が簡単に作れます。

**ステップ2: Synthesia でスクリプト作成と読み上げ**

1. **スクリプト作成** — アプリ紹介文を100〜150文字にまとめる
2. **AIアバター選択** — 20+のアバターから選択
3. **音声設定** — 日本語・英語など複数言語に対応
4. **生成** — 数分で動画が完成

## ffmpeg でのフォーマット変換と最適化

複数のAIツールから出力した動画を、App Store / Google Play の仕様に最適化するには ffmpeg が強力です。

### 基本的な変換コマンド

```bash
# 解像度・フレームレート・ビットレート を一度に変換
ffmpeg -i input.mp4 \
  -vf "scale=1080:1920:force_original_aspect_ratio=increase:force_divisible_by=2" \
  -r 30 \
  -c:v libx264 \
  -preset slow \
  -crf 18 \
  -b:v 8M \
  -c:a aac \
  -b:a 128k \
  -t 30 \
  output_appstore.mp4
```

**パラメータ解説**:
- `-vf scale=1080:1920` — 解像度を1080×1920にスケール
- `-r 30` — フレームレートを30fpsに統一
- `-preset slow` — 高品質（slow/medium/fast から選択）
- `-crf 18` — 品質指標（0=最高品質, 28=低品質）
- `-b:v 8M` — ビデオビットレート 8Mbps
- `-b:a 128k` — オーディオビットレート 128kbps

### 複数動画の一括処理

```bash
# ディレクトリ内すべての MP4 を変換
for file in *.mp4; do
  ffmpeg -i "$file" \
    -vf "scale=1080:1920:force_original_aspect_ratio=increase" \
    -r 30 \
    -c:v libx264 \
    -preset slow \
    -crf 18 \
    -b:v 8M \
    -c:a aac \
    -b:a 128k \
    "optimized_${file}"
done
```

## テキストオーバーレイとローカライゼーション

プレビュー動画には、ユーザーの目を引くテキストオーバーレイが効果的です。また、グローバル展開を想定して複数言語対応が重要です。

### FFmpeg でテキストオーバーレイを追加

```bash
ffmpeg -i input.mp4 \
  -vf "drawtext=text='30秒でマスター':fontfile=/path/to/font.ttf:fontsize=60:fontcolor=white:x=(w-text_w)/2:y=(h-text_h)/2:enable='between(t,0,5)'" \
  output_with_text.mp4
```

**ポイント**:
- `enable='between(t,0,5)'` — 0秒から5秒の間だけテキスト表示
- `fontcolor=white` — テキスト色（背景が暗い場合は白、明るい場合は黒）
- `fontsize=60` — フォントサイズ（デバイスサイズに応じて調整）

### 複数言語版の効率的な制作

1. **マスター動画を1本作成** — テキストなしのベース動画
2. **言語別にテキストレイアウトをカスタマイズ** — 日本語、英語、中文各版
3. **バッチ処理で一括生成** — ffmpeg のスクリプト化で自動化

```bash
#!/bin/bash
# 言語配列
languages=("ja" "en" "zh")
texts=("プレビュー動画" "Preview Video" "预览视频")

for i in "${!languages[@]}"; do
  lang="${languages[$i]}"
  text="${texts[$i]}"

  ffmpeg -i master.mp4 \
    -vf "drawtext=text='$text':fontfile=fonts/${lang}.ttf:fontsize=60:fontcolor=white:x=(w-text_w)/2:y=50" \
    "preview_${lang}.mp4"
done
```

## よくある質問（FAQ）

### Q1: AI生成動画は著作権に問題ありませんか？

**A**: Runway ML、Synthesia などのツールで生成した動画の著作権は、一般的に利用者（あなた）に帰属します。ただし利用規約を確認する必要があります。

Runway ML の場合、有料プラン（Pro以上）なら商用利用が可能で、著作権も安心です。無料トライアルで生成した動画は商用利用に制限があるため注意してください。

**対応策**:
- 商用利用する場合は、必ず有料プランにアップグレード
- 利用規約で「商用ライセンス」の表記を確認

### Q2: 画面録画動画とAI生成動画、どちらが効果的ですか？

**A**: **用途によって使い分けます**。

| 方法 | 向いているケース |
|---|---|
| **AI生成動画** | 短時間で多言語版が必要、スマホアプリのプロトタイプ紹介、イメージ重視 |
| **画面録画動画** | 実際の操作感を見せたい、複雑なUIフローの説明、リアリティが重要 |

**ハイブリッド戦略**:
- AI生成動画で「ビジュアルアピール」を担当
- 重要な操作フローは画面録画で「信頼性」を担当
- 両者をMONTAGEで組み合わせる

### Q3: 複数言語対応する場合、制作時間はどのくらい？

**A**: **AI生成であれば1日で完了**です。

| フェーズ | 所要時間 |
|---|---|
| プロンプト作成＆ブレスト | 30分 |
| AI動画生成（5言語 × 3シーン） | 2〜3時間 |
| ffmpegで最適化 | 30分 |
| App Store / Google Play へのアップロード | 1時間 |
| **合計** | **4〜5時間** |

従来の画面録画・編集方式なら5〜10日かかっていた作業が、AI活用で1日に短縮できます。
