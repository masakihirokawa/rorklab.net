## はじめに — ニュースアプリで学ぶ API 連携の基礎

外部 API と連携してデータを取得・表示するアプリは、モバイル開発の現場で最も頻繁に登場するパターンのひとつです。天気アプリ、株価アプリ、SNS フィードなど、その応用範囲は非常に広く、一度この設計を習得すれば幅広いアプリ開発に活かせます。

このチュートリアルでは、**Rork を使ってニュースアグリゲーターアプリを構築**します。NewsAPI という無料で利用できる記事配信 API を活用し、カテゴリ別フィルター・キーワード検索・お気に入り記事の保存・簡単なオフライン対応まで、実用的な機能を段階的に実装していきます。

このチュートリアルで学べること:

- Rork での外部 REST API の叩き方（fetch / async-await）
- フラットリスト（FlatList）を使った記事一覧の表示
- カテゴリタブによる絞り込み UI の実装
- AsyncStorage を使ったお気に入り記事のローカル保存
- ローディング・エラー・空状態のハンドリング

完成するアプリのイメージは、テクノロジー・スポーツ・ビジネスなど複数カテゴリのニュースをスクロールして読み、気になる記事を保存しておける「自分だけのニュースリーダー」です。ぜひ最後まで一緒に作り上げましょう。

## Step 2: NewsAPI からデータを取得する

### API キーの設定

Rork のプロジェクトに `src/services/newsApi.ts` ファイルを作成し、API 呼び出しのロジックを集約します。

```typescript
// src/services/newsApi.ts
// ⚠️ 本番環境では API キーを環境変数で管理し、
//    クライアントサイドに直接埋め込まないようにしましょう

const API_KEY = "YOUR_NEWSAPI_KEY"; // 取得した API キーに置き換える
const BASE_URL = "https://newsapi.org/v2";

export type NewsCategory =
  | "general"
  | "technology"
  | "sports"
  | "business"
  | "entertainment";

export interface Article {
  source: { id: string | null; name: string };
  title: string;
  description: string | null;
  url: string;
  urlToImage: string | null;
  publishedAt: string;
  content: string | null;
}

export interface NewsResponse {
  status: string;
  totalResults: number;
  articles: Article[];
}

// カテゴリ別のトップ記事を取得
export async function fetchTopHeadlines(
  category: NewsCategory = "general",
  country: string = "jp"
): Promise<Article[]> {
  const url = `${BASE_URL}/top-headlines?country=${country}&category=${category}&apiKey=${API_KEY}`;

  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`API エラー: ${response.status}`);
  }

  const data: NewsResponse = await response.json();
  // タイトルや画像が null の記事を除外する
  return data.articles.filter((a) => a.title && a.title !== "[Removed]");
}

// キーワードで記事を検索
export async function searchNews(query: string): Promise<Article[]> {
  const encoded = encodeURIComponent(query);
  const url = `${BASE_URL}/everything?q=${encoded}&language=ja&sortBy=publishedAt&apiKey=${API_KEY}`;

  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`検索エラー: ${response.status}`);
  }

  const data: NewsResponse = await response.json();
  return data.articles.filter((a) => a.title && a.title !== "[Removed]");
}
```

この `newsApi.ts` に API 呼び出し関連のロジックをまとめることで、画面コンポーネントはデータ取得の詳細を知らなくて済み、後から API を差し替えるときも1ファイルの変更で完結します。

## Step 3: ホーム画面の実装（カテゴリタブ + FlatList）

```tsx
// src/screens/HomeScreen.tsx
import React, { useState, useEffect, useCallback } from "react";
import {
  View,
  Text,
  FlatList,
  TouchableOpacity,
  StyleSheet,
  ActivityIndicator,
  RefreshControl,
} from "react-native";
import { fetchTopHeadlines, Article, NewsCategory } from "../services/newsApi";
import ArticleCard from "../components/ArticleCard";

const CATEGORIES: { label: string; value: NewsCategory }[] = [
  { label: "全て", value: "general" },
  { label: "テクノロジー", value: "technology" },
  { label: "スポーツ", value: "sports" },
  { label: "ビジネス", value: "business" },
  { label: "エンタメ", value: "entertainment" },
];

export default function HomeScreen() {
  const [selectedCategory, setSelectedCategory] =
    useState<NewsCategory>("general");
  const [articles, setArticles] = useState<Article[]>([]);
  const [loading, setLoading] = useState(true);
  const [refreshing, setRefreshing] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const loadArticles = useCallback(async (category: NewsCategory) => {
    try {
      setError(null);
      const data = await fetchTopHeadlines(category);
      setArticles(data);
    } catch (e) {
      setError("記事の取得に失敗しました。通信環境を確認してください。");
    } finally {
      setLoading(false);
      setRefreshing(false);
    }
  }, []);

  useEffect(() => {
    setLoading(true);
    loadArticles(selectedCategory);
  }, [selectedCategory]);

  const handleRefresh = () => {
    setRefreshing(true);
    loadArticles(selectedCategory);
  };

  if (loading) {
    return (
      <View style={styles.centered}>
        <ActivityIndicator size="large" color="#007AFF" />
        <Text style={styles.loadingText}>記事を読み込み中...</Text>
      </View>
    );
  }

  if (error) {
    return (
      <View style={styles.centered}>
        <Text style={styles.errorText}>{error}</Text>
        <TouchableOpacity
          onPress={() => loadArticles(selectedCategory)}
          style={styles.retryButton}
        >
          <Text style={styles.retryText}>再試行</Text>
        </TouchableOpacity>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      {/* カテゴリタブ */}
      <FlatList
        horizontal
        showsHorizontalScrollIndicator={false}
        data={CATEGORIES}
        keyExtractor={(item) => item.value}
        contentContainerStyle={styles.tabContainer}
        renderItem={({ item }) => (
          <TouchableOpacity
            style={[
              styles.tab,
              selectedCategory === item.value && styles.activeTab,
            ]}
            onPress={() => setSelectedCategory(item.value)}
          >
            <Text
              style={[
                styles.tabText,
                selectedCategory === item.value && styles.activeTabText,
              ]}
            >
              {item.label}
            </Text>
          </TouchableOpacity>
        )}
      />

      {/* 記事一覧 */}
      <FlatList
        data={articles}
        keyExtractor={(item) => item.url}
        renderItem={({ item }) => <ArticleCard article={item} />}
        refreshControl={
          <RefreshControl refreshing={refreshing} onRefresh={handleRefresh} />
        }
        ListEmptyComponent={
          <View style={styles.centered}>
            <Text style={styles.emptyText}>
              このカテゴリの記事が見つかりません
            </Text>
          </View>
        }
        contentContainerStyle={
          articles.length === 0 ? styles.emptyContainer : undefined
        }
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  centered: { flex: 1, justifyContent: "center", alignItems: "center", padding: 20 },
  tabContainer: { paddingHorizontal: 12, paddingVertical: 10, gap: 8 },
  tab: {
    paddingHorizontal: 16, paddingVertical: 8,
    borderRadius: 20, backgroundColor: "#e0e0e0",
  },
  activeTab: { backgroundColor: "#007AFF" },
  tabText: { fontSize: 14, color: "#555" },
  activeTabText: { color: "#fff", fontWeight: "600" },
  loadingText: { marginTop: 12, color: "#888" },
  errorText: { color: "#cc0000", textAlign: "center", marginBottom: 16 },
  retryButton: {
    paddingHorizontal: 24, paddingVertical: 10,
    backgroundColor: "#007AFF", borderRadius: 8,
  },
  retryText: { color: "#fff", fontWeight: "600" },
  emptyText: { color: "#888", fontSize: 15 },
  emptyContainer: { flex: 1 },
});
```

ポイントは3つです。まず `useEffect` の依存配列に `selectedCategory` を含めることで、タブ切り替え時に自動で再フェッチが走ります。次にプルリフレッシュ（`RefreshControl`）を付けることで、ユーザーが意図的に最新記事を取得できるようにしています。そしてローディング・エラー・空状態の3パターンをそれぞれハンドリングしています。

## Step 5: お気に入り機能の実装（AsyncStorage）

お気に入り記事は AsyncStorage を使ってローカルに保存します。アプリを閉じても記事が残るのがポイントです。

```typescript
// src/hooks/useFavorites.ts
import { useState, useEffect, useCallback } from "react";
import AsyncStorage from "@react-native-async-storage/async-storage";
import { Article } from "../services/newsApi";

const STORAGE_KEY = "@news_favorites";

export function useFavorites() {
  const [favorites, setFavorites] = useState<Article[]>([]);

  // 起動時に保存済みのお気に入りを読み込む
  useEffect(() => {
    (async () => {
      try {
        const stored = await AsyncStorage.getItem(STORAGE_KEY);
        if (stored) setFavorites(JSON.parse(stored));
      } catch (e) {
        console.error("お気に入りの読み込みエラー:", e);
      }
    })();
  }, []);

  // お気に入り状態を保存する内部関数
  const save = useCallback(async (items: Article[]) => {
    try {
      await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(items));
      setFavorites(items);
    } catch (e) {
      console.error("お気に入りの保存エラー:", e);
    }
  }, []);

  // お気に入りに追加 / 削除（トグル）
  const toggleFavorite = useCallback(
    (article: Article) => {
      const exists = favorites.some((f) => f.url === article.url);
      const updated = exists
        ? favorites.filter((f) => f.url !== article.url)
        : [...favorites, article];
      save(updated);
    },
    [favorites, save]
  );

  // 特定の記事がお気に入りかどうかを確認
  const isFavorite = useCallback(
    (url: string) => favorites.some((f) => f.url === url),
    [favorites]
  );

  return { favorites, toggleFavorite, isFavorite };
}
```

このカスタムフック `useFavorites` はアプリ全体で再利用できます。記事一覧画面とお気に入り画面の両方から呼び出して使いましょう。ローカルデータの永続化についてさらに深く学びたい方には、[Rork アプリのローカルデータ永続化ガイド — AsyncStorage・MMKV・SQLite の選び方と実装](/articles/rork-dev/rork-local-storage-asyncstorage-mmkv-sqlite-guide)をおすすめします。MMKV や SQLite など、より高速・高機能な選択肢も詳しく解説されています。

## Step 6: 検索画面の実装

```tsx
// src/screens/SearchScreen.tsx
import React, { useState, useCallback, useRef } from "react";
import {
  View,
  TextInput,
  FlatList,
  ActivityIndicator,
  Text,
  StyleSheet,
  TouchableOpacity,
} from "react-native";
import { searchNews, Article } from "../services/newsApi";
import ArticleCard from "../components/ArticleCard";
import { useDebouncedCallback } from "use-debounce";

export default function SearchScreen() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<Article[]>([]);
  const [loading, setLoading] = useState(false);
  const [searched, setSearched] = useState(false);

  // 入力から 500ms 待ってから検索（連続入力時のリクエスト過多を防ぐ）
  const debouncedSearch = useDebouncedCallback(async (text: string) => {
    if (text.trim().length < 2) return;
    setLoading(true);
    setSearched(true);
    try {
      const data = await searchNews(text);
      setResults(data);
    } catch {
      setResults([]);
    } finally {
      setLoading(false);
    }
  }, 500);

  const handleChangeText = (text: string) => {
    setQuery(text);
    debouncedSearch(text);
  };

  return (
    <View style={styles.container}>
      <View style={styles.searchBar}>
        <Text style={styles.searchIcon}>🔍</Text>
        <TextInput
          style={styles.input}
          placeholder="キーワードで検索..."
          value={query}
          onChangeText={handleChangeText}
          returnKeyType="search"
          clearButtonMode="while-editing"
        />
      </View>

      {loading && (
        <ActivityIndicator style={styles.spinner} color="#007AFF" />
      )}

      {!loading && searched && results.length === 0 && (
        <View style={styles.empty}>
          <Text style={styles.emptyText}>「{query}」に一致する記事がありません</Text>
        </View>
      )}

      <FlatList
        data={results}
        keyExtractor={(item) => item.url}
        renderItem={({ item }) => <ArticleCard article={item} />}
        keyboardShouldPersistTaps="handled"
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  searchBar: {
    flexDirection: "row",
    alignItems: "center",
    margin: 12,
    paddingHorizontal: 12,
    backgroundColor: "#fff",
    borderRadius: 12,
    elevation: 2,
    shadowColor: "#000",
    shadowOffset: { width: 0, height: 1 },
    shadowOpacity: 0.08,
    shadowRadius: 4,
  },
  searchIcon: { fontSize: 16, marginRight: 8 },
  input: { flex: 1, height: 44, fontSize: 16 },
  spinner: { marginTop: 30 },
  empty: { flex: 1, justifyContent: "center", alignItems: "center" },
  emptyText: { color: "#888", fontSize: 15 },
});
```

検索入力に **デバウンス処理**（`use-debounce` パッケージ）を入れているのがポイントです。ユーザーが文字を入力するたびに API リクエストを送ると無料枠をすぐ使い切ってしまうため、0.5秒の遅延を設けて入力が落ち着いたタイミングで検索します。

## Step 7: よくあるエラーと対処法

### ① ネットワークリクエストがブロックされる

`fetch` が失敗する場合、まず `app.json` の `android.usesCleartextTraffic` を確認しましょう（HTTP の場合）。HTTPS の NewsAPI は基本的に問題ありませんが、開発サーバー経由の場合は設定が必要なことがあります。

### ② 画像が表示されない

`urlToImage` が `null` の記事や、画像 URL が期限切れのケースがあります。`ArticleCard` でプレースホルダーを表示する実装にしておくと安心です（Step 4 のコードでは `imagePlaceholder` で対応しています）。

### ③ デバウンスが効かない

`useCallback` や `useRef` を使わずに `debounce` を直接呼ぶと、レンダーごとに新しい関数が生成されてデバウンスがリセットされます。`use-debounce` の `useDebouncedCallback` を使うとこの問題を回避できます。

### ④ AsyncStorage の値が消える

開発中は Expo のファストリフレッシュで状態がリセットされることがあります。本番ビルドで動作を確認し、問題なければ開発時の挙動として許容しましょう。

## FAQ

**Q. NewsAPI の無料プランで本番アプリを公開できますか？**

A. 無料プランは開発目的限定で、本番環境（ストアで公開するアプリ）には有料プランが必要です。本番公開を考えているなら、Newsdata.io や GNews など無料本番利用が許可されている代替 API の利用も検討してください。

**Q. 日本語のニュースを取得するにはどうすればいいですか？**

A. `fetchTopHeadlines` の `country` パラメーターを `jp`（デフォルト）、`searchNews` の `language` パラメーターを `ja` に設定することで日本語記事を優先できます。ただし NewsAPI の日本語ソースは英語に比べて少ないため、実際には日英混在になることがあります。

**Q. リアルタイムに記事を更新するにはどうすればいいですか？**

A. `setInterval` を使って定期的に `loadArticles` を呼び出す方法が手軽です。ただし、無料プランのレート制限に注意してください。バックグラウンドでの更新には `expo-background-fetch` と組み合わせると、アプリが開いていない状態でも定期更新が可能になります。

**Q. お気に入り記事の件数が増えてもパフォーマンスは大丈夫ですか？**

A. AsyncStorage は少量〜中量のデータには適していますが、数百件を超えるとシリアライズ・デシリアライズのコストが問題になることがあります。大量データを扱う場合は SQLite や MMKV への移行を検討しましょう。詳しくは [Rork アプリのローカルデータ永続化ガイド — AsyncStorage・MMKV・SQLite の選び方と実装](/articles/rork-dev/rork-local-storage-asyncstorage-mmkv-sqlite-guide) をご参照ください。

**Q. Rork でニュースアプリをオフライン対応にするには？**

A. 前回フェッチした記事を AsyncStorage に保存し、ネットワーク接続がない場合はキャッシュデータを表示する方法が基本です。[Rork で作るオフライン対応モバイルアプリ — AsyncStorage と NetInfo の活用術](/articles/rork-dev/rork-offline-ready-asyncstorage-netinfo)で実装パターンを詳しく解説しています。
