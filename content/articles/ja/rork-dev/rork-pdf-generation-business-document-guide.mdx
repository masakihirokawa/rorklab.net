## はじめに

ビジネス向けアプリを開発していると、「PDFを生成して送信したい」というニーズは非常によく耳にします。見積書・請求書・作業報告書・健康診断レポートなど、PDF出力が求められる場面は数多くあります。

Rorkを使ったアプリ開発でも、`expo-print` と `expo-sharing` を組み合わせることで、HTMLテンプレートからPDFを動的に生成し、共有・保存する機能を実装できます。本記事では、実際に動くコードを交えながら、PDF生成アプリの作り方をステップバイステップで解説します。

対象読者は、Rorkでの基本的なアプリ開発に慣れており、REST APIやファイル操作に関する基礎知識をお持ちの方を想定しています。もし外部APIの連携に不安がある場合は、[Rork で外部 API を連携する実践ガイド](/articles/rork-dev/rork-external-api-integration-guide)も合わせてご確認ください。

## PDF生成の仕組みとライブラリ選定

### expo-print の役割

Expo（Rorkの基盤）には `expo-print` というライブラリが用意されており、HTML文字列からPDFファイルを生成することができます。これは以下の流れで動作します。

1. JavaScriptでHTMLテンプレートを文字列として作成
2. `Print.printToFileAsync()` に渡してPDFファイルとして保存
3. `expo-sharing` の `shareAsync()` でユーザーがPDFをメール・クラウドなどへ共有

この仕組みの最大の利点は、**HTMLとCSSをそのままデザインに使える**点です。既存のWebデザインの知識を活かして、レイアウトを自由に組めます。

### 必要なパッケージ

Rorkプロジェクトで以下の依存関係が必要です。Rorkのプロンプトに「expo-printとexpo-sharingを追加して」と伝えると自動でインストールされます。

```
expo-print
expo-sharing
expo-file-system（ファイルパス確認・削除に使用）
```

## ステップバイステップ：請求書PDF生成アプリの実装

### Step 1: HTMLテンプレートの作成

まず、請求書のHTMLテンプレートを定義します。JavaScriptのテンプレートリテラルを使って、動的なデータを埋め込みます。

```typescript
// lib/invoiceTemplate.ts

interface InvoiceData {
  invoiceNumber: string;
  date: string;
  dueDate: string;
  companyName: string;
  companyAddress: string;
  clientName: string;
  clientAddress: string;
  items: { description: string; quantity: number; unitPrice: number }[];
  taxRate: number;
  note?: string;
}

export function generateInvoiceHTML(data: InvoiceData): string {
  const subtotal = data.items.reduce(
    (sum, item) => sum + item.quantity * item.unitPrice,
    0
  );
  const tax = Math.round(subtotal * data.taxRate);
  const total = subtotal + tax;

  const itemRows = data.items
    .map(
      (item) => `
      <tr>
        <td style="padding: 10px; border-bottom: 1px solid #eee;">${item.description}</td>
        <td style="padding: 10px; border-bottom: 1px solid #eee; text-align: center;">${item.quantity}</td>
        <td style="padding: 10px; border-bottom: 1px solid #eee; text-align: right;">¥${item.unitPrice.toLocaleString()}</td>
        <td style="padding: 10px; border-bottom: 1px solid #eee; text-align: right;">¥${(item.quantity * item.unitPrice).toLocaleString()}</td>
      </tr>
    `
    )
    .join("");

  return `
    <!DOCTYPE html>
    <html lang="ja">
    <head>
      <meta charset="UTF-8" />
      <style>
        body { font-family: 'Helvetica Neue', Arial, sans-serif; margin: 40px; color: #333; }
        h1 { color: #2563eb; font-size: 28px; margin-bottom: 4px; }
        .meta { display: flex; justify-content: space-between; margin-bottom: 30px; }
        .meta-block p { margin: 2px 0; font-size: 13px; }
        table { width: 100%; border-collapse: collapse; margin-top: 20px; }
        th { background: #2563eb; color: white; padding: 10px; text-align: left; font-size: 13px; }
        td { font-size: 13px; }
        .total-area { margin-top: 20px; text-align: right; }
        .total-area p { margin: 4px 0; font-size: 14px; }
        .total-area .grand-total { font-size: 18px; font-weight: bold; color: #2563eb; }
        .note { margin-top: 30px; font-size: 12px; color: #666; border-top: 1px solid #ddd; padding-top: 16px; }
      </style>
    </head>
    <body>
      <h1>請求書</h1>
      <p style="color:#888; font-size:13px;">Invoice No. ${data.invoiceNumber}</p>
      <div class="meta">
        <div class="meta-block">
          <p><strong>${data.companyName}</strong></p>
          <p>${data.companyAddress}</p>
        </div>
        <div class="meta-block" style="text-align:right;">
          <p>請求先: <strong>${data.clientName}</strong></p>
          <p>${data.clientAddress}</p>
          <p>発行日: ${data.date}</p>
          <p>支払期限: ${data.dueDate}</p>
        </div>
      </div>

      <table>
        <thead>
          <tr>
            <th>品目・サービス</th>
            <th style="text-align:center; width:80px;">数量</th>
            <th style="text-align:right; width:120px;">単価</th>
            <th style="text-align:right; width:120px;">小計</th>
          </tr>
        </thead>
        <tbody>${itemRows}</tbody>
      </table>

      <div class="total-area">
        <p>小計: ¥${subtotal.toLocaleString()}</p>
        <p>消費税 (${data.taxRate * 100}%): ¥${tax.toLocaleString()}</p>
        <p class="grand-total">合計: ¥${total.toLocaleString()}</p>
      </div>

      ${data.note ? `<div class="note"><strong>備考:</strong> ${data.note}</div>` : ""}
    </body>
    </html>
  `;
}
```

このHTMLテンプレートでは、企業名・品目・合計金額などを動的に埋め込んでいます。CSSはインラインまたは `<style>` タグで記述するとPDF変換後も正しく反映されます。

### Step 2: PDF生成とシェア機能の実装

次に、実際にPDFを生成してユーザーに共有するロジックを実装します。

```typescript
// lib/pdfService.ts
import * as Print from "expo-print";
import * as Sharing from "expo-sharing";
import { Platform } from "react-native";

export async function generateAndSharePDF(html: string, filename: string) {
  try {
    // HTMLからPDFファイルを生成
    const { uri } = await Print.printToFileAsync({
      html,
      base64: false,
    });

    // 期待する出力: uri = "file:///var/mobile/.../tmp/xxx.pdf"
    console.log("PDF generated at:", uri);

    // シェア可能かチェック（実機のみ）
    const isAvailable = await Sharing.isAvailableAsync();
    if (!isAvailable) {
      console.warn("Sharing is not available on this device");
      return;
    }

    // PDF共有ダイアログを表示
    await Sharing.shareAsync(uri, {
      mimeType: "application/pdf",
      dialogTitle: `${filename}を共有`,
      UTI: "com.adobe.pdf", // iOS用
    });
  } catch (error) {
    console.error("PDF generation error:", error);
    throw error;
  }
}
```

`printToFileAsync()` が生成するファイルはデバイスの一時ディレクトリに保存されます。`shareAsync()` を呼び出すと、iOS・Androidそれぞれのネイティブ共有シートが表示され、メール・AirDrop・Dropboxなどへの送信が可能です。

## よくあるエラーと対処法

### エラー1: 日本語が豆腐（□□□）になる

**原因**: PDFレンダリングエンジンがデバイスのフォントに依存しており、日本語フォントが埋め込まれないケース。

**対処法**: CSSで `font-family` をシステムフォントに指定するか、Google FontsなどのWebフォントをBase64エンコードしてHTMLに埋め込みます。

```html
<style>
  @import url("https://fonts.googleapis.com/css2?family=Noto+Sans+JP&display=swap");
  body {
    font-family: "Noto Sans JP", sans-serif;
  }
</style>
```

ただし、オフライン環境ではWebフォントが読み込めないため注意が必要です。

---

### エラー2: `printToFileAsync` がクラッシュする（Expo Go）

**原因**: `expo-print` は Expo Go では動作が制限されている場合があります。

**対処法**: 開発環境では EAS Build や Expo Dev Client を使ってテストします。Rorkの「プレビュー」機能ではなく、実機ビルドで動作確認してください。

---

### エラー3: PDFの改ページが崩れる

**原因**: 長いリストやテーブルが自動改ページに対応していません。

**対処法**: CSSの `page-break` プロパティを活用します。

```css
.no-break {
  page-break-inside: avoid;
}
tr {
  page-break-inside: avoid;
}
```

## FAQ

**Q1: expo-print はiOSとAndroidの両方で使えますか？**

A: はい、`expo-print` はiOS・Android両プラットフォームに対応しています。ただし、プレビュー表示（`Print.printAsync()`）はiOSのみです。PDFファイルを生成して共有する `printToFileAsync()` は両OS で問題なく動作します。

**Q2: 生成されたPDFのサイズは調整できますか？**

A: `printToFileAsync()` のオプションに `width` と `height` を指定することでページサイズを変更できます。A4サイズにしたい場合は `{ width: 595, height: 842 }` （ポイント単位）を指定してください。

**Q3: PDFにロゴ画像を入れたい場合は？**

A: HTMLの `<img>` タグで画像を埋め込む方法が最も簡単です。ローカル画像はBase64エンコードしてデータURLとして指定します。例えば `<img src="data:image/png;base64,..." />` という形式です。`expo-file-system` でアプリ内の画像ファイルをBase64に変換できます。

**Q4: ページ番号を自動で入れることはできますか？**

A: CSSの `@page` ルールと `counter` を組み合わせることで対応できます。ただし、`expo-print` のレンダリングエンジンによっては `@page` ルールのサポートが限定的なため、フッターにJavaScriptで計算したページ番号をHTMLとして直接挿入する方法が確実です。

**Q5: Webビューで事前プレビューを表示してからPDF化したい場合は？**

A: `expo-web-view` を使ってHTMLプレビューを表示し、確認ボタン押下後に `expo-print` でPDF化するフローが一般的です。ユーザー体験の向上に効果的な設計です。

## まとめ

本記事では、RorkアプリでPDF生成・共有機能を実装する手順を解説しましました。主なポイントをまとめます。

- `expo-print` の `printToFileAsync()` でHTMLからPDFを生成できる
- `expo-sharing` の `shareAsync()` でiOS・Android双方のネイティブ共有シートを呼び出せる
- HTMLテンプレートはJavaScriptのテンプレートリテラルで動的に作成し、デザインの自由度が高い
- 日本語フォントの文字化けにはWebフォントのBase64埋め込みで対応する
- クラウドストレージへの自動保存やメール送信など、バックエンドと組み合わせると活用の幅がさらに広がる

PDF生成はフリーランスや中小企業向けのビジネスアプリに特に相性が良く、請求書・見積書・報告書といった業務文書の自動化ニーズに応えられます。ぜひ自分のアプリに組み込んでみてください。
