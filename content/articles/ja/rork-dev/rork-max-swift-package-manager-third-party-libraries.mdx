Rork MaxでSwiftUIのネイティブアプリが作れるようになって、最初に壁にぶつかるのが外部ライブラリの追加だ。Rork Maxが生成するコードは標準のSwiftUIフレームワークをベースにしているが、Firebase SDKやAlamofire、Kingfisherなどのサードパーティライブラリを使いたい場面は必ずやってくる。

Swift Package Manager（SPM）は、Xcode 12以降はApple純正のパッケージ管理ツールとして標準搭載されており、Rork MaxのプロジェクトでもSPMを通じて外部ライブラリを統合できる。ただし、Rork MaxのAI生成コードとSPMの統合にはいくつか注意点がある。この記事でまとめて説明する。

## Rork MaxプロジェクトのSPM対応状況

Rork Maxが生成するSwiftUIプロジェクトは、Xcodeの標準プロジェクト構成に準拠している。つまり、通常のSwiftUIプロジェクトと同様にSPMでパッケージを追加できる。

ただし1つ知っておくべきことがある。Rork Maxが生成したコードを直接Xcodeで開いて作業する場合と、Rork Companionアプリ経由でシミュレーター/実機確認する場合で、ワークフローが異なる。**外部ライブラリの追加はXcode上での作業が必要**で、Rork Companion内のプレビューだけでは確認できない。

## Firebase SDKを追加する手順

最も多い需要がFirebaseの統合なので、これを例に解説する。

### ステップ1: Xcodeでプロジェクトを開く

RorkからエクスポートしたプロジェクトをXcodeで開く。

```
File → Open → [プロジェクトフォルダ] → [プロジェクト名].xcodeproj
```

### ステップ2: SPMでFirebaseを追加する

```
File → Add Package Dependencies...
```

検索フィールドに以下のURLを入力する。

```
https://github.com/firebase/firebase-ios-sdk
```

バージョンは`Up to Next Major Version`から最新安定版（`11.x.x`）を選択する。

追加するライブラリを選択する画面では、使いたいモジュールだけを選ぶ。全部選ぶと不必要にバイナリサイズが膨らむ。

- FirebaseCore（必須）
- FirebaseFirestore（Firestoreを使う場合）
- FirebaseAuth（認証を使う場合）
- FirebaseAnalytics（アナリティクスを使う場合）
- FirebaseCrashlytics（クラッシュ解析を使う場合）

### ステップ3: AppDelegateでFirebaseを初期化する

Rork MaxのアプリはSwiftUIのApp構造体を使っているため、`AppDelegate`がない場合がある。その場合は`@UIApplicationDelegateAdaptor`を使う。

```swift
// RorkアプリのメインApp構造体に追加
import SwiftUI
import FirebaseCore

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]? = nil
    ) -> Bool {
        FirebaseApp.configure()
        return true
    }
}

@main
struct MyRorkApp: App {
    // FirebaseをSwiftUIアプリに統合するためのアダプター
    @UIApplicationDelegateAdaptor(AppDelegate.self) var delegate
    
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

`GoogleService-Info.plist`はFirebaseコンソールからダウンロードして、**プロジェクトルートに追加**する。Xcodeのプロジェクトナビゲーターにドラッグ＆ドロップし、「Copy items if needed」にチェックを入れる。

## Firestoreをリアルタイムで使うサンプル

統合が完了したら、実際にFirestoreからデータを取得するコードをRorkで書かせてみよう。Rorkに以下のようなプロンプトを渡すと適切なSwiftUIコードが生成される。

```swift
import SwiftUI
import FirebaseFirestore

// Firestoreからメッセージを取得してリアルタイム表示するビュー
struct MessageListView: View {
    @State private var messages: [String] = []
    private let db = Firestore.firestore()
    
    var body: some View {
        List(messages, id: \.self) { message in
            Text(message)
        }
        .navigationTitle("メッセージ")
        .onAppear {
            // リアルタイムリスナーを設定
            db.collection("messages")
                .order(by: "timestamp", descending: true)
                .limit(to: 20)
                .addSnapshotListener { snapshot, error in
                    guard let documents = snapshot?.documents else {
                        print("エラー: \(error?.localizedDescription ?? "不明")")
                        return
                    }
                    messages = documents.compactMap { doc in
                        doc.data()["text"] as? String
                    }
                }
        }
    }
}
```

## よくあるビルドエラーと対処法

**`No such module 'FirebaseFirestore'`**

SPMのパッケージが正しく解決されていない。`File → Packages → Reset Package Caches`を試してから、再ビルドする。それでも解決しない場合は、`DerivedData`フォルダを削除してXcodeを再起動する。

```bash
# DerivedDataの場所
~/Library/Developer/Xcode/DerivedData/
```

**ビルド時間が極端に長い**

Firebaseは初回ビルドで数分かかることがある。2回目以降はキャッシュが効いて速くなる。特にFirebaseAnalyticsはバイナリが大きいため、不要なら追加しないことをすすめる。

**`Multiple commands produce...` エラー**

同じファイルが複数のターゲットに含まれているときに発生する。Xcodeのプロジェクト設定でターゲットメンバーシップを確認する。

## Alamofireとその他ライブラリの追加

同じSPM手順で他のライブラリも追加できる。よく使われるもののURLを記しておく。

- Alamofire（ネットワーク通信）: `https://github.com/Alamofire/Alamofire`
- Kingfisher（画像キャッシュ）: `https://github.com/onevcat/Kingfisher`
- SwiftUI Navigation（画面遷移管理）: `https://github.com/pointfreeco/swiftui-navigation`
- Lottie（アニメーション）: `https://github.com/airbnb/lottie-ios`

## Rork CompanionでSPM統合後のコードを確認する

SPMでライブラリを追加した後のコードをRork Companion経由でプレビューする場合、**ビルドにはXcodeが必要**だ。Rork Companion単体では外部ライブラリのリゾルブができないため、一度Xcodeでビルドが通ることを確認してから、Rork Companionにインストールする流れになる。

実際の開発フローとしては、Rorkでロジックのプロトタイプを作り → Xcodeでライブラリを追加して本番化 → 微調整はRorkに戻って聞く、というサイクルが効率的だ。

SPMの統合は最初だけ少し手間がかかるが、一度設定してしまえばあとはRorkの強みであるSwiftUIのコード生成をフルに活かしながら開発を進められる。
