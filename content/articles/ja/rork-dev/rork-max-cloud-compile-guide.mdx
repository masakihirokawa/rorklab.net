Rork Maxの最大の特徴は、**ローカルMacを必要としないクラウドコンパイル機能**です。WindowsやLinuxからでも、ブラウザさえあれば、iOS・iPadOS・watchOSなどのネイティブアプリをビルド・テストできます。この記事では、Rork Maxのクラウドコンパイルの仕組みから実際の使い方、よくある質問まで、実践的に解説します。

## クラウドコンパイルとは — なぜこれまでMacが必要だったのか

従来、iOSアプリを開発するには、XcodeがインストールされたMacが**必須**でした。なぜなら、AppleのツールチェーンがmacOSでのみ動作するためです。

しかし**Rork Maxは、クラウド上に複数のMacを所有**しており、あなたのコードをそこでコンパイル・ビルドしてくれます。つまり、ローカルMacは不要。Windows PCやLinuxマシンから、Rork Maxのブラウザインターフェースにアクセスするだけで、プロフェッショナルなネイティブアプリを開発できるようになりました。

この革新は、開発環境のハードルを大きく下げます：

- **初期投資削減**: 高額なMacを購入する必要がない
- **マルチプラットフォーム対応**: Windowsユーザーも、Linuxユーザーも、等しくiOSアプリを作れる
- **チーム開発の効率化**: 全員が同じクラウド環境を使うため、環境差分がない

## Rork Maxのクラウドコンパイル環境の全体像

Rork Maxが提供する環境は以下の通りです：

- **クラウドMacフリート**: 複数のMacインスタンスが常に待機。あなたのビルドリクエストを受け取ると、自動的に割り当てられます
- **Swift/Swiftui**: 最新のAppleツールチェーン（Swift 6対応）を搭載
- **签名・プロビジョニング**: Apple Developer Account との連携により、自動的にコード署名を処理
- **マルチプラットフォーム**: iPhone、iPad、Apple Watch、Apple TV、Vision Pro、iMessageなど、Appleのあらゆるプラットフォームに対応

背景にあるのは、**Claude Code + Opus 4.6** というAIの力です。Rork Maxはあなたの自然言語の指示（「ボタンをタップしたら画面が遷移するアプリ作って」）を理解し、自動的にSwiftUIコードを生成し、クラウド上でビルド・テストしてくれます。

## ブラウザシミュレータ — リアルタイムプレビューの驚異

Rork Maxのもう一つの強力な機能が、**ブラウザベースのシミュレータ**です。

従来のXcodeシミュレータは、ローカルマシンのリソースを大量に消費し、起動が遅く、パフォーマンスも不安定でした。Rork Maxはこれを完全にクラウド化しました。

クラウド上で実行されているiOS Simulatorの画面は、**低遅延プロトコルでリアルタイムにブラウザにストリーミング**されます。まるでローカルで実行しているかのような体験が得られます：

- **タップ・スワイプ** → Simulatorにイベント送信
- **シェイク操作** → 対応
- **ボタン操作** → カメラ、マイク、GPS等の仮装センサー制御

体感レイテンシは数十ミリ秒で、実用的な開発速度が実現されています。

## ステップ1: Rork Max アカウントのセットアップ

まず、Rork Maxの公式サイト（[Rork Max公式](https://www.rork.ai)）にアクセスし、アカウントを作成します。

- **無料プラン**: 週約5回のプロンプト実行（クラウドコンパイル・シミュレータ実行）に制限されます
- **Max プラン**: 月額$200で、クラウドコンパイル・シミュレータの無制限使用が可能
- **Companion Mac**: 最初の3つのiOSアプリをローカル（Companion Macで）テストできます（Apple Developer Accountなしで）

アカウント作成後、Apple Developer Accountをリンクします（有料プランを利用する場合）。これにより、App Storeへの公開まで自動化できます。

## ステップ2: Rork MaxでのiOSアプリ作成フロー

### 2-1. プロジェクト作成

Rork Maxのダッシュボードで「New Project」を選択します。対象プラットフォーム（iOS、iPadOS、watchOS等）を指定し、「Create」をクリック。

### 2-2. AIによるコード生成

Rork Maxの自然言語プロンプトボックスに、あなたのアプリのコンセプトを入力します。例：

```
「ユーザーが入力したテキストを保存し、リスト表示するTODOアプリを作成してください。
- UIはSwiftUI
- データはUserDefaults
- iPhone/iPad両対応
- ダークモード対応
」
```

Claude Opus 4.6がこれを受け取り、完全なSwiftUIコードを生成します。

### 2-3. クラウドコンパイル

「Build」ボタンをクリックすると、コードはクラウド上のMacに送信され、自動的にコンパイル・署名されます。通常、数秒～数十秒でビルドが完了します。

### 2-4. ブラウザシミュレータでテスト

ビルドが完了すると、ブラウザ内にシミュレータが起動します。あなたのアプリがリアルタイムで動作しているのを見ることができます。

## Rork Companion Mac — ローカルテスト・実機配布

**Rork Companion Mac** は、あなたのMacにインストールする軽量アプリです。以下が可能になります：

- **ローカルテスト**: ビルドしたアプリを、実際のiPhoneやiPadにワイヤレスでインストール
- **Apple Developer Account不要**: Companion Macの「信頼済みアプリ」機能を使い、最初の3つのiOSアプリを、Developer Accountなしでインストール可能（Apple IDだけで十分）
- **高速なイテレーション**: Rork Max上で修正 → Companion Macから実機にインストール、という流れで高速にデバッグできます

セットアップ方法：

1. [Rork公式](https://www.rork.ai/companion)からCompanion Macをダウンロード
2. Macにインストール
3. Rork Maxアカウントでログイン
4. iPhoneをMacに接続し、Companion Macから「Install」をクリック

## App Store公開の自動化

Rork Maxの最も時間短縮できる機能が、**App Store公開の2クリック化**です。

従来、App Store公開には以下の煩雑な手順が必要でした：

- アプリ署名・プロビジョニング
- ビルド番号・バージョンの管理
- App Storeメタデータ（説明文、スクリーンショット等）の入力
- TestFlight配布
- 審査リクエスト

Rork Maxでは、これが以下のようにシンプルになります：

1. 「Publish to App Store」をクリック
2. メタデータを編集（アプリ説明、価格カテゴリ等）
3. 「Submit for Review」をクリック

あとはAppleの審査を待つだけ。署名・プロビジョニング・ビルド管理は全て自動化されています。

## よくある質問（FAQ）

### Q1: クラウドコンパイルって、実際のところセキュアですか？

**A1**: はい。Rork Maxは以下のセキュリティ対策を実施しています：

- **エンドツーエンド暗号化**: コードはSSL/TLSで暗号化され、Rork Maxサーバーに送信されます
- **Apple Developer Credential の非保存**: API Keyやプロビジョニングプロファイルは、ビルド実行時にのみ一時的にロードされ、保存されません
- **SOC 2 Type II認証**: 業界標準のセキュリティ監査をパス

ただし、機密情報（APIキーなど）をコード内に埋め込む場合は、環境変数や`.env`ファイルで管理し、Rork Maxのシークレット管理機能を利用してください。

### Q2: オフラインでも開発できますか？

**A2**: いいえ。Rork Maxはクラウドサービスなため、インターネット接続が必須です。ただし、ローカルエディタ（VS Code等）でコードを編集してから、Rork Maxにアップロードする、という流れは可能です。

### Q3: 複数のプラットフォーム（iPhone + Apple Watch）を同時に開発できますか？

**A3**: はい。1つのプロジェクト内で、複数のターゲットプラットフォームを指定できます。ビルド時に、各プラットフォーム用のバイナリが自動生成されます。

### Q4: 既存のXcodeプロジェクトをRork Maxに移行できますか？

**A4**: Rork Maxは、プロジェクト構造を自動的に検出し、既存のXcodeプロジェクトを自動変換できます。ただし、複雑なCocoa Podsやビルドスクリプトがある場合は、若干の手動調整が必要な場合があります。

### Q5: コンパイルエラーが出たとき、ログはどこで見られますか？

**A5**: Rork MaxのDashboardの「Build Logs」セクションで、詳細なコンパイルログ・エラーメッセージが確認できます。また、Claude AIに「このエラーを修正して」と入力すると、自動的にコード修正を提案してくれます。
