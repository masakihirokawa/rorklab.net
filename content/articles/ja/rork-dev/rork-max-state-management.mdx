# Rork Max 状態管理完全ガイド

アプリが成長するにつれて、状態管理は重要な課題になります。このガイドでは、Zustand、React Context API、Redux Toolkit の三つの主要な状態管理ライブラリを比較し、各々の実装パターンと最適な使い分けを解説します。

## 状態管理ライブラリの選択マトリクス

| 特性 | Context API | Zustand | Redux Toolkit |
|------|-------------|---------|---------------|
| 学習曲線 | 低 | 低 | 中～高 |
| バンドルサイズ | 0 KB | 3 KB | 15 KB |
| パフォーマンス | ⚠️ リレンダリング注意 | ✓ 最適化済み | ✓ 最適化済み |
| 複雑な状態 | △ 小～中規模向け | ✓ 推奨 | ✓ 推奨 |
| ボイラープレート | 少ない | 最小 | 多い |
| DevTools | ✗ | ✓ | ✓ |
| タイムトラベルデバッグ | ✗ | ✓ (拡張機能) | ✓ |
| 非同期処理 | 手動 | 手動 | Redux Thunk |
| TypeScript 対応 | ✓ | ✓ (優良) | ✓ |

<Callout type="info">
**選択基準**:
- **小規模アプリ（<10画面）**: Context API
- **中～大規模アプリ**: Zustand
- **大規模で複雑な状態**: Redux Toolkit
</Callout>

## Zustand — 最小主義の状態管理

Zustand は最小のボイラープレートで強力な状態管理を実現します。

### インストール

```bash
npm install zustand
npm install --save-dev @react-native-async-storage/async-storage
```

### 基本的なストア作成

**src/store/userStore.ts**
```typescript
import { create } from 'zustand';
import { devtools, persist } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface User {
  id: string;
  name: string;
  email: string;
  avatar?: string;
}

interface UserState {
  user: User | null;
  isLoading: boolean;
  error: string | null;
  setUser: (user: User | null) => void;
  updateUser: (updates: Partial<User>) => void;
  logout: () => void;
  fetchUser: (userId: string) => Promise<void>;
}

export const useUserStore = create<UserState>()(
  devtools(
    persist(
      (set, get) => ({
        user: null,
        isLoading: false,
        error: null,

        setUser: (user) => set({ user, error: null }),

        updateUser: (updates) =>
          set((state) => ({
            user: state.user ? { ...state.user, ...updates } : null,
          })),

        logout: () => set({ user: null, error: null }),

        fetchUser: async (userId) => {
          set({ isLoading: true });
          try {
            const response = await fetch(`/api/users/${userId}`);
            const data = await response.json();
            set({ user: data, error: null, isLoading: false });
          } catch (error) {
            set({
              error: (error as Error).message,
              isLoading: false,
            });
          }
        },
      }),
      {
        name: 'user-store',
        storage: {
          getItem: async (name) => {
            const data = await AsyncStorage.getItem(name);
            return data ? JSON.parse(data) : null;
          },
          setItem: async (name, value) => {
            await AsyncStorage.setItem(name, JSON.stringify(value));
          },
          removeItem: async (name) => {
            await AsyncStorage.removeItem(name);
          },
        },
      }
    )
  )
);
```

### パフォーマンス最適化 — セレクタの使い分け

<Callout type="warning">
**重要**: 不要なリレンダリングを避けるため、セレクタ関数は必ず使用してください。さもなければ、ストア内のどの値が変更されたか関係なく、全て再レンダリングされます。
</Callout>

```typescript
// ❌ 良くない例：毎回全ストアをサブスクライブ
const user = useUserStore();

// ✓ 良い例：必要な状態だけをセレクト
const user = useUserStore((state) => state.user);
const isLoading = useUserStore((state) => state.isLoading);

// ✓ オブジェクトはメモ化して比較
const userData = useUserStore(
  (state) => ({
    name: state.user?.name,
    email: state.user?.email,
  }),
  (a, b) => a.name === b.name && a.email === b.email
);
```

## React Context API — ビルトイン状態管理

Context API はバンドルサイズが小さく、小～中規模アプリに適しています。

**src/contexts/ThemeContext.tsx**
```typescript
import React, { createContext, useContext, useState, useCallback } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

type Theme = 'light' | 'dark' | 'auto';

interface ThemeContextType {
  theme: Theme;
  setTheme: (theme: Theme) => Promise<void>;
  isDark: boolean;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setThemeState] = useState<Theme>('auto');
  const [isDark, setIsDark] = useState(false);

  const setTheme = useCallback(async (newTheme: Theme) => {
    setThemeState(newTheme);
    await AsyncStorage.setItem('app_theme', newTheme);

    if (newTheme === 'dark') {
      setIsDark(true);
    } else if (newTheme === 'light') {
      setIsDark(false);
    } else {
      // 'auto': デバイス設定に従う
      // (実装は Appearance API を使用)
    }
  }, []);

  return (
    <ThemeContext.Provider value={{ theme, setTheme, isDark }}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within ThemeProvider');
  }
  return context;
}
```

### Context API のパフォーマンス問題と対策

```typescript
// ❌ 問題：値が変わるたびに全コンシューマーが再レンダリング
const value = { user, setUser, isLoading };
return <UserContext.Provider value={value}>{children}</UserContext.Provider>;

// ✓ 解決：値をメモ化
const value = useMemo(
  () => ({ user, setUser, isLoading }),
  [user, isLoading]
);
return <UserContext.Provider value={value}>{children}</UserContext.Provider>;
```

## Redux Toolkit — エンタープライズレベルの状態管理

大規模で複雑な状態が必要な場合、Redux Toolkit がお勧めです。

### インストール

```bash
npm install @reduxjs/toolkit react-redux
```

### ストア設定

**src/store/slices/authSlice.ts**
```typescript
import { createSlice, createAsyncThunk, PayloadAction } from '@reduxjs/toolkit';

interface AuthState {
  token: string | null;
  user: { id: string; name: string } | null;
  status: 'idle' | 'loading' | 'succeeded' | 'failed';
  error: string | null;
}

const initialState: AuthState = {
  token: null,
  user: null,
  status: 'idle',
  error: null,
};

export const loginUser = createAsyncThunk(
  'auth/loginUser',
  async (
    { email, password }: { email: string; password: string },
    { rejectWithValue }
  ) => {
    try {
      const response = await fetch('/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email, password }),
      });

      if (!response.ok) {
        return rejectWithValue('ログインに失敗しました');
      }

      const data = await response.json();
      return { token: data.token, user: data.user };
    } catch (error) {
      return rejectWithValue((error as Error).message);
    }
  }
);

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    logout: (state) => {
      state.token = null;
      state.user = null;
    },
  },
  extraReducers: (builder) => {
    builder
      .addCase(loginUser.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(loginUser.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.token = action.payload.token;
        state.user = action.payload.user;
      })
      .addCase(loginUser.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.payload as string;
      });
  },
});

export const { logout } = authSlice.actions;
export default authSlice.reducer;
```

### コンポーネントでの使用

import React, { useState } from 'react';
import { View, TextInput, TouchableOpacity, Text, ActivityIndicator } from 'react-native';
import { useDispatch, useSelector } from 'react-redux';
import { loginUser } from '../store/slices/authSlice';

export default function Login() {
  const dispatch = useDispatch<AppDispatch>();
  const { status, error } = useSelector((state: RootState) => state.auth);

  const [password, setPassword] = useState('');

          borderColor: '#ccc',

          marginBottom: 12,
          borderRadius: 8,

      <TextInput
        placeholder="パスワード"
        value={password}
        onChangeText={setPassword}
        secureTextEntry
        style={{
          borderWidth: 1,
          borderColor: '#ccc',
          padding: 12,
          marginBottom: 20,
          borderRadius: 8,
        }}
      />

      <TouchableOpacity
        onPress={handleLogin}
        disabled={status === 'loading'}
        style={{
          backgroundColor: status === 'loading' ? '#ccc' : '#007AFF',
          padding: 12,
          borderRadius: 8,
          alignItems: 'center',

          </Text>
        )}
      </TouchableOpacity>

      {error && (
        <Text style={{ color: '#FF3B30', marginTop: 16, textAlign: 'center' }}>
          {error}
        </Text>
      )}
    </View>
  );
}

## 永続化 — AsyncStorage と MMKV

状態を永続化することで、アプリ再起動時のセッション復元が可能になります。

### Zustand + AsyncStorage

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

export const useAuthStore = create(
  persist(
    (set) => ({
      token: null,
      setToken: (token) => set({ token }),
    }),
    {
      name: 'auth-storage',
      storage: {
        getItem: async (name) => {
          const item = await AsyncStorage.getItem(name);
          return item ? JSON.parse(item) : null;
        },
        setItem: async (name, value) => {
          await AsyncStorage.setItem(name, JSON.stringify(value));
        },
        removeItem: async (name) => {
          await AsyncStorage.removeItem(name);
        },
      },
    }
  )
);
```

### MMKV を使った高速化

```bash
npm install react-native-mmkv
```

```typescript
import { MMKV } from 'react-native-mmkv';

const storage = new MMKV();

export const useAuthStore = create(
  persist(
    (set) => ({
      token: null,
      setToken: (token) => set({ token }),
    }),
    {
      name: 'auth-storage',
      storage: {
        getItem: (name) => {
          const value = storage.getString(name);
          return value ? JSON.parse(value) : null;
        },
        setItem: (name, value) => {
          storage.set(name, JSON.stringify(value));
        },
        removeItem: (name) => {
          storage.delete(name);
        },
      },
    }
  )
);
```

<Callout type="info">
**MMKV の利点**: AsyncStorage より 100 倍高速です。大量のデータを頻繁に保存・読み込みする場合に活躍します。
</Callout>

## 比較表：Zustand vs Context vs Redux

| タスク | Zustand | Context API | Redux |
|------|---------|-------------|-------|
| グローバル状態 | ✓ 推奨 | △ 小規模向け | ✓ 推奨 |
| 非同期処理 | ✓ (手動) | △ (手動) | ✓ (Thunk 統合) |
| DevTools | ✓ | ✗ | ✓ |
| ボイラープレート | 最小 | 少ない | 多い |
| パフォーマンス | ✓ | ⚠️ | ✓ |
| TypeScript | ✓ | ✓ | ✓ |

## まとめ

- **小規模アプリ**: Context API で十分
- **中～大規模**: Zustand の最小ボイラープレートと柔軟性
- **エンタープライズ**: Redux Toolkit のデバッグ機能と構造化

各々に利点があります。プロジェクトの規模と複雑さに応じて、最適なツールを選択してください。
