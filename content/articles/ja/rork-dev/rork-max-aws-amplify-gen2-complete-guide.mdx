個人開発の領域では Supabase や Firebase が事実上の標準になっていますが、クライアントワークや業務アプリの世界では話が違います。「既存の AWS インフラに乗せてほしい」「HIPAA/SOC2 準拠が必要」「Cognito のユーザープールがすでにある」——そういった要件を持つ案件に出会ったことがある方も多いのではないでしょうか。

私自身、個人開発の傍らエンタープライズ向けの受託案件も手がけていますが、AWS Amplify を Rork Max と組み合わせる際の情報が極端に少なくて困ることがありました。公式ドキュメントは JavaScript SDK 前提の記述が多く、React Native / Expo ベースの Rork Max に適用する際には細かい読み替えが必要です。

この記事では、Rork Max プロジェクトに AWS Amplify Gen 2 を導入し、Cognito 認証・AppSync GraphQL API・S3 ストレージを一気通貫で動かすまでの全手順を、実際に動作するコードと一緒にお伝えします。

## なぜ Firebase/Supabase ではなく AWS Amplify なのか

まず正直にお伝えしておくと、**個人開発の新規プロジェクトなら Supabase を選ぶことが多いです**。Supabase はセットアップが速く、Row Level Security も直感的で、Rork との相性も良好です。

ではなぜ Amplify を学ぶ必要があるのか、というと主に次の3つのシナリオがあります。

**① 既存の AWS インフラとの統合が必要な場合**
クライアントがすでに AWS 上にシステムを構築していて、Lambda・RDS・SQS などを使っているケースです。バックエンドチームが Cognito でユーザー管理をしている場合、フロントエンドの認証層も Cognito で統一するのが自然な設計です。

**② コンプライアンス要件がある場合**
医療・金融・行政系のアプリでは、データが AWS リージョン内に存在することや、特定の準拠認証（HIPAA、FedRAMP）が求められることがあります。こういった要件は Amplify + AWS サービスの組み合わせで対応しやすいです。

**③ エンタープライズ規模のスケーラビリティが必要な場合**
AppSync は大規模なリアルタイムアプリに向いており、数十万接続でも安定して動作します。DynamoDB の auto-scaling と組み合わせると、個人開発ではなかなか到達しないスケールにも対応できます。

## Amplify Gen 2 とは——Gen 1 との違い

AWS Amplify には Gen 1（旧来の CLI ベース）と Gen 2（2024年 GA）があります。**必ず Gen 2 を使ってください**。Gen 1 は `amplify push` で CloudFormation を直接操作する設計で、デバッグが困難でした。

Gen 2 では TypeScript ファーストの CDK-like な定義になっています：

```typescript
// amplify/backend.ts（Gen 2 の設定ファイル）
import { defineBackend } from "@aws-amplify/backend";
import { auth } from "./auth/resource";
import { data } from "./data/resource";
import { storage } from "./storage/resource";

// 認証・データ・ストレージを TypeScript で定義
export const backend = defineBackend({
  auth,
  data,
  storage,
});
```

このように Infrastructure as Code として宣言的に書けるため、Git 管理がしやすく、チーム開発でも変更の意図が伝わりやすくなっています。

## Step 1：プロジェクトのセットアップ

### Amplify CLI のインストールと設定

まず開発マシンに Amplify CLI をセットアップします。Node.js 18以上が前提です：

```bash
# Amplify CLI のインストール

# AWS credentials の設定（既に設定済みの場合はスキップ）
aws configure
# → AWS Access Key ID、Secret Access Key、リージョン（ap-northeast-1 推奨）を入力
```

Rork Max で生成した Expo プロジェクトのルートで実行します：

### 認証フックの実装

Rork Max の画面コンポーネントで使うカスタムフックを作成します：

```typescript
// hooks/useAuth.ts
import { useState, useEffect } from "react";
import {
  signIn,
  signOut,
  signUp,
  confirmSignUp,
  getCurrentUser,
  fetchUserAttributes,
  resetPassword,
  confirmResetPassword,
  AuthError,
} from "aws-amplify/auth";

type AuthUser = {
  userId: string;
  username: string;
  email: string;
  plan?: string;
};

export function useAuth() {
  const [user, setUser] = useState<AuthUser | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  // アプリ起動時にセッションを復元
  useEffect(() => {
    checkCurrentUser();
  }, []);

  async function checkCurrentUser() {
    try {
      const { userId, username } = await getCurrentUser();
      const attributes = await fetchUserAttributes();
      setUser({
        userId,
        username,
        email: attributes.email ?? "",
        plan: attributes["custom:plan"],
      });
    } catch {
      // ログインしていない場合は null のまま（エラーではない）
      setUser(null);
    } finally {
      setIsLoading(false);
    }
  }

  async function handleSignIn(email: string, password: string) {
    setError(null);
    try {
      const result = await signIn({ username: email, password });

      if (result.isSignedIn) {
        await checkCurrentUser();
        return { success: true };
      }

      // MFA が必要な場合の処理
      if (result.nextStep.signInStep === "CONFIRM_SIGN_IN_WITH_TOTP_CODE") {
        return { success: false, needsMFA: true };
      }

      return { success: false };
    } catch (err) {
      const authError = err as AuthError;

      // エラーコードを日本語に変換
      const messages: Record<string, string> = {
        NotAuthorizedException: "メールアドレスまたはパスワードが正しくありません",
        UserNotFoundException: "このメールアドレスは登録されていません",
        UserNotConfirmedException: "メールアドレスの確認が完了していません",
        TooManyRequestsException: "試行回数が多すぎます。しばらく待ってから再試行してください",
      };

      const message = messages[authError.name] ?? authError.message;
      setError(message);
      return { success: false, error: message };
    }
  }

  async function handleSignUp(email: string, password: string) {
    setError(null);
    try {
      const { nextStep } = await signUp({
        username: email,
        password,
        options: {
          userAttributes: { email },
        },
      });
      return {
        success: true,
        needsConfirmation: nextStep.signUpStep === "CONFIRM_SIGN_UP",
      };
    } catch (err) {
      const authError = err as AuthError;
      const message =
        authError.name === "UsernameExistsException"
          ? "このメールアドレスはすでに登録されています"
          : authError.message;
      setError(message);
      return { success: false, error: message };
    }
  }

  async function handleConfirmSignUp(email: string, code: string) {
    try {
      await confirmSignUp({ username: email, confirmationCode: code });
      return { success: true };
    } catch (err) {
      const authError = err as AuthError;
      setError(
        authError.name === "CodeMismatchException"
          ? "確認コードが正しくありません"
          : authError.message
      );
      return { success: false };
    }
  }

  async function handleSignOut() {
    await signOut();
    setUser(null);
  }

  return {
    user,
    isLoading,
    error,
    isAuthenticated: \!\!user,
    handleSignIn,
    handleSignUp,
    handleConfirmSignUp,
    handleSignOut,
  };
}
```

### ログイン画面への組み込み

Rork Max が生成したログイン画面に上記フックを組み込む例です：

```typescript
// app/(auth)/sign-in.tsx
import { useState } from "react";
import { View, TextInput, TouchableOpacity, Text, Alert } from "react-native";
import { router } from "expo-router";
import { useAuth } from "@/hooks/useAuth";

export default function SignInScreen() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [isSubmitting, setIsSubmitting] = useState(false);
  const { handleSignIn, error } = useAuth();

  async function onSubmit() {
    if (\!email || \!password) return;
    setIsSubmitting(true);

    const result = await handleSignIn(email, password);

    if (result.success) {
      router.replace("/(tabs)/home");
    } else if (result.needsMFA) {
      router.push("/(auth)/mfa");
    }
    // エラーは error state から表示
    setIsSubmitting(false);
  }

  return (
    <View style={{ flex: 1, padding: 24, justifyContent: "center" }}>
      <TextInput
        value={email}
        onChangeText={setEmail}
        placeholder="メールアドレス"
        keyboardType="email-address"
        autoCapitalize="none"
        autoComplete="email"
      />
      <TextInput
        value={password}
        onChangeText={setPassword}
        placeholder="パスワード"
        secureTextEntry
        autoComplete="password"
      />
      {error && (
        <Text style={{ color: "red", marginBottom: 8 }}>{error}</Text>
      )}
      <TouchableOpacity onPress={onSubmit} disabled={isSubmitting}>
        <Text>{isSubmitting ? "ログイン中..." : "ログイン"}</Text>
      </TouchableOpacity>
    </View>
  );
}
```

## Step 3：AppSync GraphQL API でデータを管理する

AppSync は AWS のマネージド GraphQL サービスです。DynamoDB をバックエンドに持ち、リアルタイムサブスクリプションをネイティブでサポートしています。

### データモデルの定義

```typescript
// amplify/data/resource.ts
import { type ClientSchema, a, defineData } from "@aws-amplify/backend";

const schema = a.schema({
  // タスク管理アプリの例
  Todo: a
    .model({
      title: a.string().required(),
      description: a.string(),
      isCompleted: a.boolean().default(false),
      priority: a.enum(["LOW", "MEDIUM", "HIGH"]),
      dueDate: a.datetime(),
      // 所有者（Cognito ユーザー ID）を自動的に設定
      owner: a.string(),
    })
    .authorization((allow) => [
      // ログインユーザーは自分のデータのみ CRUD 可能
      allow.owner(),
      // 管理者は全データ読み取り可能
      allow.groups(["Admin"]).to(["read"]),
    ]),

  // プロフィール情報
  UserProfile: a
    .model({
      displayName: a.string().required(),
      avatarUrl: a.string(),
      bio: a.string(),
      todoCount: a.integer().default(0),
    })
    .authorization((allow) => [
      allow.owner(),
      allow.authenticated().to(["read"]), // 他のユーザーの基本情報は閲覧可能
    ]),
});

export type Schema = ClientSchema<typeof schema>;

export const data = defineData({
  schema,
  authorizationModes: {
    defaultAuthorizationMode: "userPool", // Cognito 認証がデフォルト
    apiKeyAuthorizationMode: {
      // 公開 API 用（未認証ユーザーがアクセスできるデータに使用）
      expiresInDays: 30,
    },
  },
});
```

### データ操作フックの実装

```typescript
// hooks/useTodos.ts
import { useState, useEffect } from "react";
import { generateClient } from "aws-amplify/data";
import type { Schema } from "@/amplify/data/resource";

// 型安全なクライアントを生成
const client = generateClient<Schema>();

export function useTodos() {
  const [todos, setTodos] = useState<Schema["Todo"]["type"][]>([]);
  const [isLoading, setIsLoading] = useState(true);

  // TODO の一覧取得
  useEffect(() => {
    const subscription = client.models.Todo.observeQuery({
      filter: { isCompleted: { eq: false } },
      sort: [{ field: "priority", direction: "DESC" }],
    }).subscribe({
      next: ({ items, isSynced }) => {
        setTodos(items);
        if (isSynced) setIsLoading(false);
      },
      error: (err) => {
        console.error("Todo の取得に失敗しました:", err);
        setIsLoading(false);
      },
    });

    // クリーンアップ：コンポーネントのアンマウント時にサブスクリプションを解除
    return () => subscription.unsubscribe();
  }, []);

  // TODO の作成
  async function createTodo(input: {
    title: string;
    description?: string;
    priority?: "LOW" | "MEDIUM" | "HIGH";
    dueDate?: Date;
  }) {
    try {
      const { data, errors } = await client.models.Todo.create({
        title: input.title,
        description: input.description,
        priority: input.priority ?? "MEDIUM",
        dueDate: input.dueDate?.toISOString(),
        isCompleted: false,
      });

      if (errors) {
        throw new Error(errors.map((e) => e.message).join(", "));
      }

      return { success: true, todo: data };
    } catch (err) {
      console.error("Todo の作成に失敗しました:", err);
      return { success: false, error: String(err) };
    }
  }

  // TODO の完了状態を更新
  async function toggleTodo(id: string, currentState: boolean) {
    const { data, errors } = await client.models.Todo.update({
      id,
      isCompleted: \!currentState,
    });

    if (errors) {
      console.error("更新エラー:", errors);
    }
    return data;
  }

  // TODO の削除
  async function deleteTodo(id: string) {
    await client.models.Todo.delete({ id });
  }

  return { todos, isLoading, createTodo, toggleTodo, deleteTodo };
}
```

### リアルタイムサブスクリプション

AppSync の強力な機能の一つがリアルタイムサブスクリプションです。他のユーザーの操作を即座に反映したい場合（チャット・コラボレーションツールなど）に威力を発揮します：

```typescript
// リアルタイム更新を受信する例
useEffect(() => {
  // 新しい TODO が作成されたときのサブスクリプション
  const createSub = client.models.Todo.onCreate().subscribe({
    next: (data) => {
      console.log("新しい TODO が作成されました:", data);
      // UI を更新
    },
    error: (error) => console.error("サブスクリプションエラー:", error),
  });

  // TODO が更新されたときのサブスクリプション
  const updateSub = client.models.Todo.onUpdate().subscribe({
    next: (data) => {
      setTodos((prev) =>
        prev.map((t) => (t.id === data.id ? data : t))
      );
    },
    error: (error) => console.error("サブスクリプションエラー:", error),
  });

  return () => {
    createSub.unsubscribe();
    updateSub.unsubscribe();
  };
}, []);
```

## Step 4：S3 ストレージでファイルをアップロード・管理する

プロフィール画像やドキュメントのアップロード機能を実装します。

### ストレージリソースの定義

```typescript
// amplify/storage/resource.ts
import { defineStorage } from "@aws-amplify/backend";

export const storage = defineStorage({
  name: "myAppStorage",
  access: (allow) => ({
    // 公開読み取り可能なパス（プロフィール画像など）
    "public/*": [
      allow.guest.to(["read"]),
      allow.authenticated.to(["read", "write", "delete"]),
    ],
    // 所有者のみアクセス可能なプライベートデータ
    "private/{entity_id}/*": [
      allow.entity("identity").to(["read", "write", "delete"]),
    ],
    // 認証済みユーザー全員が読めるが、書けるのは所有者のみ
    "protected/{entity_id}/*": [
      allow.authenticated.to(["read"]),
      allow.entity("identity").to(["write", "delete"]),
    ],
  }),
});
```

### ファイルアップロードフックの実装

```typescript
// hooks/useStorage.ts
import { useState } from "react";
import { uploadData, getUrl, remove, downloadData } from "aws-amplify/storage";
import * as ImagePicker from "expo-image-picker";
import * as FileSystem from "expo-file-system";

export function useStorage() {
  const [uploadProgress, setUploadProgress] = useState(0);
  const [isUploading, setIsUploading] = useState(false);

  // プロフィール画像のアップロード
  async function uploadProfileImage(userId: string): Promise<string | null> {
    // カメラロールの権限確認
    const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();
    if (status \!== "granted") {
      return null;
    }

    // 画像を選択
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      aspect: [1, 1],
      quality: 0.8, // 圧縮率 80%
    });

    if (result.canceled) return null;

    const imageUri = result.assets[0].uri;
    const fileName = `profile_${userId}_${Date.now()}.jpg`;
    const s3Key = `protected/${userId}/${fileName}`;

    setIsUploading(true);
    setUploadProgress(0);

    try {
      // URI を Blob に変換（React Native では fetch を使用）
      const response = await fetch(imageUri);
      const blob = await response.blob();

      // S3 へアップロード（進捗追跡付き）
      const uploadTask = uploadData({
        path: s3Key,
        data: blob,
        options: {
          contentType: "image/jpeg",
          onProgress: ({ transferredBytes, totalBytes }) => {
            if (totalBytes) {
              const progress = Math.round((transferredBytes / totalBytes) * 100);
              setUploadProgress(progress);
            }
          },
          metadata: {
            userId,
            uploadedAt: new Date().toISOString(),
          },
        },
      });

      const { path } = await uploadTask.result;

      // 署名付き URL を取得（期限: 24時間）
      const { url } = await getUrl({
        path,
        options: { expiresIn: 86400, validateObjectExistence: true },
      });

      return url.toString();
    } catch (err) {
      console.error("アップロードエラー:", err);
      return null;
    } finally {
      setIsUploading(false);
      setUploadProgress(0);
    }
  }

  // ファイルの削除
  async function deleteFile(path: string) {
    try {
      await remove({ path });
      return true;
    } catch (err) {
      console.error("削除エラー:", err);
      return false;
    }
  }

  // ファイルのダウンロード（ローカルキャッシュ付き）
  async function downloadFile(path: string): Promise<string | null> {
    const localPath = `${FileSystem.cacheDirectory}${path.split("/").pop()}`;

    // ローカルキャッシュが存在する場合はそれを返す
    const fileInfo = await FileSystem.getInfoAsync(localPath);
    if (fileInfo.exists) return localPath;

    try {
      const result = await downloadData({ path }).result;
      const arrayBuffer = await result.body.arrayBuffer();
      const base64 = Buffer.from(arrayBuffer).toString("base64");
      await FileSystem.writeAsStringAsync(localPath, base64, {
        encoding: FileSystem.EncodingType.Base64,
      });
      return localPath;
    } catch (err) {
      console.error("ダウンロードエラー:", err);
      return null;
    }
  }

  return {
    uploadProfileImage,
    deleteFile,
    downloadFile,
    uploadProgress,
    isUploading,
  };
}
```

## Step 6：よくあるエラーと落とし穴

実際に Rork Max × Amplify を組み合わせる際にハマりやすいポイントを整理します。

**落とし穴① Hermes エンジンとの非互換**
Rork Max は Expo SDK 50以降で Hermes をデフォルトとして使用しています。Amplify の一部のパッケージが `TextDecoder` や `crypto` API を使っており、Hermes では動作しないことがあります。

```bash
# 必要なポリフィルをインストール
npm install --save react-native-get-random-values core-js

# app/_layout.tsx の先頭に追加（他のインポートより前）
import "react-native-get-random-values";
import "core-js/stable/atob";
import "core-js/stable/btoa";
```

**落とし穴② amplify_outputs.json が eas build で見つからない**
EAS Build は `.gitignore` に入っているファイルをビルド環境に持ち込みません。次の方法で対応します：

```json
// eas.json に secretFiles を追加
{
  "build": {
    "production": {
      "secretFiles": ["amplify_outputs.json"]
    }
  }
}
```

EAS の管理画面で `amplify_outputs.json` の内容を Secret として登録しておきます。

**落とし穴③ AppSync のリアルタイムサブスクリプションが iOS シミュレータで動かない**
iOS シミュレータでは WebSocket の接続が不安定になることがあります。実機または Expo Go でテストしてください。また、`@aws-amplify/pubsub` が明示的にインポートされていることを確認します：

```typescript
// app/_layout.tsx に追加
import "@aws-amplify/pubsub"; // サブスクリプション機能を有効化
```

**落とし穴④ S3 アップロードで CORS エラーが発生する**
Amplify サンドボックスでは CORS は自動設定されますが、既存の S3 バケットを使用する場合は手動設定が必要です。S3 コンソール → バケット → アクセス許可 → CORS 設定：

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": ["ETag", "x-amz-server-side-encryption"]
  }
]
```

**落とし穴⑤ Cognito のトークン有効期限とリフレッシュ**
Cognito のアクセストークンはデフォルトで1時間で期限切れになります。`aws-amplify` は自動的にリフレッシュしますが、ネットワークがない状態から復帰した際にリフレッシュが失敗するケースがあります：

```typescript
// App.tsx でネットワーク復帰時にセッションを再確認
import NetInfo from "@react-native-community/netinfo";
import { fetchAuthSession } from "aws-amplify/auth";

useEffect(() => {
  const unsubscribe = NetInfo.addEventListener(async (state) => {
    if (state.isConnected) {
      try {
        // ネットワーク復帰時にトークンを強制リフレッシュ
        await fetchAuthSession({ forceRefresh: true });
      } catch {
        // セッション切れの場合はログイン画面へ
        router.replace("/(auth)/sign-in");
      }
    }
  });
  return unsubscribe;
}, []);
```

**落とし穴⑥ AppSync の N+1 クエリ問題**
AppSync + DynamoDB の組み合わせでリレーションデータを取得する際、素朴な実装では N+1 問題が発生します。Amplify Gen 2 では `belongsTo` / `hasMany` を使ってリレーションを定義し、一括取得を使うのが正解です：

```typescript
// 正しいリレーション定義
const schema = a.schema({
  Post: a
    .model({
      title: a.string().required(),
      content: a.string(),
      // 著者との 1対1 リレーション
      author: a.belongsTo("UserProfile", "authorId"),
      authorId: a.id(),
      // コメントとの 1対多 リレーション
      comments: a.hasMany("Comment", "postId"),
    })
    .authorization((allow) => [allow.authenticated()]),
});

// 取得時にリレーションを include する
const { data } = await client.models.Post.list({
  selectionSet: [
    "id",
    "title",
    "author.displayName",
    "author.avatarUrl",
    "comments.id",
    "comments.content",
  ],
});
```

## Step 7：本番環境での IAM セキュリティ設計

本番環境では最小権限の原則に従ったIAMポリシーが重要です。

Amplify が自動生成する IAM ロールに加えて、Lambda 関数などの追加リソースには手動でポリシーを付与する必要があります：

```typescript
// amplify/backend.ts での Lambda 権限設定
import { defineBackend } from "@aws-amplify/backend";
import { auth } from "./auth/resource";
import { data } from "./data/resource";
import { storage } from "./storage/resource";
import { PolicyStatement, Effect, ServicePrincipal } from "aws-cdk-lib/aws-iam";

const backend = defineBackend({ auth, data, storage });

// 認証済みユーザーに特定の S3 操作のみ許可
const { cfnResources } = backend.auth;
const { identityPool } = cfnResources;

// S3 へのアクセスポリシーを明示的に制限
// 例: private/ パス以外への書き込みを禁止
backend.storage.resources.bucket.addToResourcePolicy(
  new PolicyStatement({
    effect: Effect.DENY,
    principals: [new ServicePrincipal("cognito-identity.amazonaws.com")],
    actions: ["s3:PutObject"],
    notResources: [
      `${backend.storage.resources.bucket.bucketArn}/private/\${cognito-identity.amazonaws.com:sub}/*`,
    ],
  })
);
```

## まとめ：次の一歩

AWS Amplify Gen 2 × Rork Max の組み合わせは、セットアップの初期コストはあるものの、エンタープライズ要件を満たすモバイルアプリを個人でも構築できる強力な選択肢です。

Supabase や Firebase との決定的な違いは、**既存の AWS エコシステムとの接続性**です。Lambda、SQS、SNS、Bedrock など、AWS の豊富なマネージドサービスと組み合わせることで、実現できるユースケースの幅が大きく広がります。

まず手始めに、自分の Rork Max プロジェクトで `npx create-amplify` を実行して、サンドボックス環境を立ち上げてみてください。Cognito 認証が動くところまで確認できれば、AppSync と S3 の追加も自然な流れで進められます。

AWS の世界は広大ですが、Amplify Gen 2 はその入口を大幅に下げてくれています。ぜひ試してみてください。
