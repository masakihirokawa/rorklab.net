## RorkとGitHubを連携させる理由

Rorkは自然言語プロンプトだけでモバイルアプリを生成できる強力なAI開発ツールですが、プロジェクトが本番レベルに成長するにつれて、**コードのバージョン管理**と**チーム開発の効率化**が不可欠になります。GitHubとの連携は、その両方を解決する最善の方法です。

Rorkで生成されたコードはReact Native（Expo）ベースであり、完全にオープンなソースコードです。これをGitHubリポジトリで管理することで、以下のメリットが得られます。

- **変更履歴の追跡**：AIが生成したコードの差分を明確に把握できる
- **ロールバック**：バグが混入しても以前の動作バージョンに即座に戻せる
- **チーム共有**：デザイナーやエンジニアがコードレビューに参加できる
- **CI/CD自動化**：GitHub Actionsでビルド・テストを自動化できる
- **外部ツール連携**：Vercel、Cloudflare Pages、EASなどとシームレスに統合できる

このガイドでは、RorkプロジェクトをGitHubで管理するための完全なワークフローを、実際のコマンドとコード例を交えて解説します。

## 前提条件

作業を始める前に以下を確認してください。

- **Rorkアカウント**（無料プランでもコードエクスポートは可能）
- **GitHubアカウント**（無料）
- **Node.js 18以上**（ローカルでのビルド確認用）
- **Expo CLI**（`npm install -g @expo/cli`）
- **git**コマンドラインツール

なお、Rork Maxを使用している場合はSwift/Xcodeのエクスポートが別フローになりますが、このガイドでは主に**Rork（React Native/Expo版）**を対象としています。

## Step 1：RorkからコードをエクスポートしてGitHubに連携する

### 1-1. Rorkダッシュボードからコードをエクスポート

Rorkのプロジェクトダッシュボードを開き、右上の「**Export Code**」または「**GitHub**」ボタンをクリックします。

Rorkには2つのエクスポート方式があります。

**方式A：ZIPダウンロード**
プロジェクト全体がZIPファイルでダウンロードされます。ローカルで解凍後、自前のGitHubリポジトリに手動でプッシュします。

**方式B：GitHub連携（推奨）**
RorkをGitHub OAuthで認証すると、指定したリポジトリに直接プッシュできます。以降のプロンプト実行時も、差分が自動でコミットされます。

### 1-2. 新しいGitHubリポジトリを作成する

```bash
# GitHub CLIを使用する場合
gh repo create my-rork-app --private --description "Rorkで生成したモバイルアプリ"

# または github.com でリポジトリを作成後、ローカルにクローン
git clone https://github.com/あなたのユーザー名/my-rork-app.git
cd my-rork-app
```

### 1-3. エクスポートしたコードを初期コミット

ZIPダウンロード方式を選んだ場合は、以下の手順で初期コミットを行います。

```bash
# ZIPを解凍してリポジトリディレクトリにコピー
unzip rork-export.zip -d my-rork-app/
cd my-rork-app

# .gitignoreを設定（Rorkエクスポートに含まれている場合はスキップ）
cat << 'EOF' > .gitignore
node_modules/
.expo/
dist/
.env
.env.local
*.eas.json
ios/
android/
EOF

# 初期コミット
git init
git add .
git commit -m "Initial: Rork generated app scaffold"
git remote add origin https://github.com/あなたのユーザー名/my-rork-app.git
git push -u origin main
```

コミット後、GitHubリポジトリにReact Native/Expoのプロジェクト構造が正しくアップロードされていることを確認します。

## Step 2：ブランチ戦略とRorkとの組み合わせ

RorkのAI生成コードをGitHubで管理する際の**推奨ブランチ戦略**を紹介します。

### 2-1. Rork専用ブランチを活用する

Rorkのプロンプト実行ごとに差分が発生します。これを`main`に直接コミットすると、意図しない変更が本番コードに混入するリスクがあります。以下のブランチ構成がおすすめです。

```
main        ← 本番リリース用（安定版）
├── develop ← 開発統合ブランチ
│   ├── rork/feature-login     ← Rorkで生成したログイン機能
│   ├── rork/feature-profile   ← Rorkで生成したプロフィール画面
│   └── fix/bugfix-xxx         ← 手動修正のバグフィックス
```

```bash
# 新機能をRorkで作成する前にブランチを切る
git checkout develop
git pull origin develop
git checkout -b rork/feature-onboarding

# Rorkでコードを生成・エクスポート後、差分をコミット
git add .
git commit -m "rork: Add onboarding screen flow (AI generated)"

# developへPull Requestを作成してレビュー後マージ
git push origin rork/feature-onboarding
gh pr create --base develop --title "Add onboarding screen" --body "Rorkで生成したオンボーディング画面"
```

### 2-2. コミットメッセージの規約

Rork生成コードと手動修正を区別するために、以下の接頭辞を使用します。

| 接頭辞 | 意味 |
|--------|------|
| `rork:` | Rorkのプロンプトで生成したコード |
| `fix:` | バグ修正 |
| `feat:` | 手動で追加した新機能 |
| `style:` | UIのデザイン調整 |
| `chore:` | 設定変更・依存関係更新 |

## Step 3：GitHub ActionsでCI/CDを自動化する

GitHub ActionsとExpo EASを組み合わせることで、`main`ブランチへのプッシュ時に自動ビルドを走らせることができます。

### 3-1. Expo EASの設定

```bash
# EAS CLIをインストール
npm install -g eas-cli

# Expoアカウントにログイン
eas login

# プロジェクトを初期化
eas build:configure
```

`eas.json`が生成されるので、ビルドプロファイルを確認します。

```json
// eas.json
{
  "cli": {
    "version": ">= 5.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "ios": {
        "simulator": true
      }
    },
    "production": {
      "autoIncrement": true
    }
  },
  "submit": {
    "production": {}
  }
}
```

### 3-2. GitHub Actionsワークフローを作成する

```yaml
# .github/workflows/eas-build.yml
name: EAS Build

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      # リポジトリをチェックアウト
      - name: Checkout repository
        uses: actions/checkout@v4

      # Node.jsのセットアップ
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"

      # 依存関係をインストール
      - name: Install dependencies
        run: npm ci

      # Expoにログイン（GitHubシークレットから認証情報を取得）
      - name: Setup EAS
        uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      # プレビュービルドを実行（iOS シミュレーター向け）
      - name: Build preview (iOS)
        run: eas build --platform ios --profile preview --non-interactive
        # 期待出力: Build #XXXX submitted. Build URL: https://expo.dev/...
```

**GitHubシークレットの設定方法**：

1. ExpoダッシュボードでAPIトークンを生成（Settings > Access Tokens）
2. GitHubリポジトリの `Settings > Secrets and variables > Actions` に `EXPO_TOKEN` として登録

これで`main`へのプッシュごとに自動ビルドが走り、Expoダッシュボードでビルド状況を確認できます。

## よくあるエラーと対処法

RorkコードをGitHubに連携する際に発生しがちな問題とその解決策をまとめます。

**エラー1: `error: failed to push some refs to 'origin'`**

原因：リモートとローカルの履歴が一致しない場合に発生します。

```bash
# リベースして再プッシュ
git pull --rebase origin main
git push origin main
```

**エラー2: `npm ci` でビルドが失敗する**

原因：Rorkがエクスポートした`package-lock.json`が古い場合があります。

```bash
# package-lock.jsonを再生成してコミット
rm package-lock.json
npm install
git add package-lock.json
git commit -m "chore: regenerate package-lock.json"
```

**エラー3: `EXPO_TOKEN` の認証エラー**

原因：トークンの有効期限切れまたは権限不足。Expoダッシュボードで新しいトークンを生成し、GitHubシークレットを更新してください。

**エラー4: Rorkで再生成後に大量の差分が出る**

Rorkはプロンプト実行のたびにコード全体を再生成する場合があります。差分が大きすぎる場合は `git diff --stat` でファイル単位の変更量を確認し、意図しない変更がないかレビューしましょう。

```bash
# 変更の概要を確認
git diff --stat HEAD

# 特定ファイルの差分を確認
git diff HEAD -- src/screens/HomeScreen.tsx
```

## 応用：チーム開発での活用

RorkとGitHubを使ったチーム開発では、以下の工夫が効果的です。

### Pull Requestテンプレートの活用

`.github/pull_request_template.md`を作成してRork生成コードのレビューを標準化します。

```markdown
## 概要
<!-- Rorkで生成した機能の説明 -->

## 使用したプロンプト
<!-- Rorkに入力したプロンプトを記載 -->

## スクリーンショット
<!-- 実機またはシミュレーターのスクリーンショット -->

## チェックリスト
- [ ] 手動でUIを確認した
- [ ] 既存機能に影響がないことを確認した
- [ ] コード内に機密情報（APIキーなど）が含まれていないことを確認した
```

### Codeownersの設定

```bash
# .github/CODEOWNERS
# 重要ファイルの変更には特定メンバーのレビューを必須にする
/src/config/          @チームリード
/src/api/             @バックエンド担当
*.env*                @チームリード
```

## FAQ

**Q: Rorkの無料プランでもGitHubエクスポートは使えますか？**

A: はい、Rorkの無料プランでもコードのZIPダウンロードは利用可能です。GitHub連携（OAuth経由の直接プッシュ）は有料プランで利用可能です。無料プランの場合はZIPダウンロード後に手動でGitHubリポジトリにプッシュする手順（Step 1-3）をご利用ください。

**Q: RorkとGitHubを連携した後、コードを手動で編集してもRorkで再生成できますか？**

A: Rorkのプロンプトを実行すると、AIがコード全体を再生成するため、手動編集の一部が上書きされる場合があります。重要なカスタマイズは専用のブランチ（`custom/`プレフィックスなど）で管理し、Rork生成コードとは分けて管理することを推奨します。

**Q: プライベートリポジトリでも連携できますか？**

A: はい、GitHub OAuthでRorkにリポジトリへのアクセス権を付与する際に、プライベートリポジトリを選択できます。機密性の高いプロジェクトには必ずプライベートリポジトリを使用してください。

**Q: GitHub ActionsでExpo EASのビルド時間はどれくらいかかりますか？**

A: iOSのpreviewビルドで通常5〜15分、本番ビルドで15〜30分程度です。EASのビルドキューの状況によって変動します。並列ビルドが必要な場合はEASのプレミアムプランをご検討ください。

## まとめ

RorkとGitHubを連携することで、AIによる高速なアプリ生成と、プロフェッショナルな開発ワークフローを両立させることができます。今回解説した主なポイントは以下のとおりです。

GitHub連携によってRorkのAI生成コードはバージョン管理の恩恵を受け、チームでの開発・レビューが可能になります。ブランチ戦略（`rork/feature-xxx`）でAI生成コードと手動修正を分離することで、コードの品質管理が格段に向上します。GitHub ActionsとExpo EASを組み合わせることで、プッシュのたびにビルドが自動実行され、App Storeへの提出まで自動化できます。
