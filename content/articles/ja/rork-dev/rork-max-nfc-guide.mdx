iPhone には NFC（Near Field Communication）チップが搭載されており、交通系 IC カードの読み取りや、スマートロック、電子名刺など幅広いユースケースで活用されています。しかし、従来は Xcode を使ったネイティブ開発が必要でした。**Rork Max** を使えば、ブラウザ上のプロンプト操作だけで Core NFC を使った本格的な iOS アプリを短時間で構築できます。

この記事では、NFC タグの読み取り・書き込みから NDEF メッセージの解析まで、実際のコードを交えながら丁寧に解説します。

## NFC と Core NFC の基礎知識

NFC は 13.56 MHz の短距離無線通信規格で、スマートフォンと専用タグ（ステッカー型チップ）の間で数センチの距離内でデータをやり取りします。iPhone 7 以降（iOS 11+）で利用可能になり、iOS 13 からはバックグラウンドでのタグ読み取りにも対応しています。

Apple が提供する **Core NFC** フレームワークを通じて、アプリから NFC タグの読み取り・書き込みが可能です。対応フォーマットは NDEF（NFC Data Exchange Format）が主流で、URL・テキスト・vCard など様々なデータ型を格納できます。

Rork Max はネイティブ Swift を生成するため、Core NFC との統合がシームレスです。React Native 版の Rork ではネイティブモジュールのブリッジが必要になる場面でも、Rork Max なら純粋な SwiftUI + Swift コードとして出力されます。

## 開発環境の準備と Entitlements 設定

Rork Max で NFC アプリを作る前に、いくつかの設定が必要です。

### Apple Developer Program への登録

NFC の書き込み機能（NFCNDEFReaderSession での write）には、Apple Developer Program（年間 99 ドル）への登録と、**Near Field Communication Tag Reading** Entitlement の申請が必要です。Rork Max のプロジェクト設定画面から Capability を追加するか、以下のプロンプトで指定します。

```
「NFC タグの読み取りと書き込みができる iOS アプリを作ってください。
 Core NFC の NFCNDEFReaderSession を使用し、
 Info.plist に NFCReaderUsageDescription を追加してください。」
```

### Info.plist への記述

Rork Max は以下のエントリを自動で `Info.plist` に追加します。手動で追加する場合は下記を参考にしてください。

```xml
<!-- Info.plist への追加項目 -->
<key>NFCReaderUsageDescription</key>
<string>NFC タグを読み取るためにアクセスが必要です</string>
```

この説明文はユーザーへの許可ダイアログに表示されるため、具体的でわかりやすい文言にしましょう。

## NFC タグ読み取り機能の実装

いよいよ実装です。Rork Max に以下のようなプロンプトを入力します。

```
「SwiftUI で NFC タグリーダーアプリを作ってください。
 - タップしたボタンで NFC スキャンを開始する
 - NDEF メッセージを読み取り、URL やテキストを画面に表示する
 - スキャン結果の履歴をリストで保持する
 - エラー時はアラートで通知する」
```

Rork Max が生成するコードの骨格は以下のようになります。

```swift
import SwiftUI
import CoreNFC

// NFCNDEFReaderSession のデリゲートを実装するビューモデル
class NFCReaderViewModel: NSObject, ObservableObject, NFCNDEFReaderSessionDelegate {

    @Published var scannedMessages: [String] = []  // スキャン結果の履歴
    @Published var alertMessage: String = ""
    @Published var showAlert: Bool = false

    private var session: NFCNDEFReaderSession?

    // スキャンを開始する
    func startScanning() {
        guard NFCNDEFReaderSession.readingAvailable else {
            alertMessage = "このデバイスは NFC に対応していません"
            showAlert = true
            return
        }
        session = NFCNDEFReaderSession(delegate: self, queue: nil, invalidateAfterFirstRead: false)
        session?.alertMessage = "iPhone を NFC タグに近づけてください"
        session?.begin()
    }

    // NDEF メッセージを受信したときに呼ばれる
    func readerSession(_ session: NFCNDEFReaderSession, didDetectNDEFs messages: [NFCNDEFMessage]) {
        for message in messages {
            for record in message.records {
                // ペイロードを UTF-8 テキストとしてデコード
                if let text = String(data: record.payload, encoding: .utf8) {
                    DispatchQueue.main.async {
                        self.scannedMessages.append(text)
                        // 期待する出力例: "https://example.com" や "Hello, NFC!"
                    }
                }
            }
        }
    }

    // セッション無効化時のエラーハンドリング
    func readerSession(_ session: NFCNDEFReaderSession, didInvalidateWithError error: Error) {
        if let readerError = error as? NFCReaderError,
           readerError.code == .readerSessionInvalidationErrorFirstNDEFTagRead ||
           readerError.code == .readerSessionInvalidationErrorUserCanceled {
            return  // 正常終了はエラーとして扱わない
        }
        DispatchQueue.main.async {
            self.alertMessage = error.localizedDescription
            self.showAlert = true
        }
    }
}
```

このビューモデルを SwiftUI の `View` から呼び出すだけで、ボタンタップで NFC スキャンが起動します。

## NFC タグへの書き込み実装

読み取りに続き、NFC タグへのデータ書き込みを実装します。書き込みには `NFCNDEFTag` プロトコルを使用します。

```swift
// URL を NFC タグに書き込む関数
func writeURL(urlString: String) {
    session = NFCNDEFReaderSession(delegate: self, queue: nil, invalidateAfterFirstRead: false)
    session?.alertMessage = "書き込み先の NFC タグに iPhone を近づけてください"
    writePayload = urlString  // 書き込む URL を保持
    isWriteMode = true
    session?.begin()
}

// タグが検出されたときの処理（書き込みモード）
func readerSession(_ session: NFCNDEFReaderSession, didDetect tags: [NFCNDEFTag]) {
    guard let tag = tags.first else { return }

    session.connect(to: tag) { error in
        guard error == nil else { return }

        // NDEF URL レコードを作成
        guard let url = URL(string: self.writePayload),
              let record = NFCNDEFPayload.wellKnownTypeURIPayload(url: url) else {
            session.invalidate(errorMessage: "URL の形式が正しくありません")
            return
        }

        let message = NFCNDEFMessage(records: [record])

        // タグに書き込む
        tag.writeNDEF(message) { error in
            if let error = error {
                // エラー例: タグが書き込み保護されている場合
                session.invalidate(errorMessage: "書き込みに失敗しました: \(error.localizedDescription)")
            } else {
                // 成功: "URL を書き込みました" とタグに保存される
                session.alertMessage = "✅ NFC タグへの書き込みが完了しました"
                session.invalidate()
            }
        }
    }
}
```

書き込み後は iPhone を NFC タグに近づけるだけで、別のスマートフォンからも URL を読み取れるようになります。

## よくあるエラーと対処法

### `readingAvailable` が `false` を返す

iPhone 5s 以前の端末や、シミュレーターでは NFC が利用できません。実機でテストしてください。また、iOS のバージョンが 11 未満の場合も NFC は使えないため、最小ターゲットバージョンを `iOS 13.0` 以上に設定することを推奨します。

### 書き込みができず `NFCReaderErrorUnsupportedFeature` が出る

NFC タグの書き込みには Entitlement が必要です。Xcode プロジェクトの `Signing & Capabilities` タブで **Near Field Communication Tag Reading** を追加し、Apple Developer Portal で Provisioning Profile を再生成してください。Rork Max では「NFC 書き込み機能を追加してください」と追記するだけで対応コードが生成されます。

### NDEF メッセージのペイロード解析がうまくいかない

`NFCNDEFPayload` の `typeNameFormat` によってデコード方法が異なります。URL レコードは `wellKnownTypeURIPayload`、テキストレコードは `wellKnownTypeTextPayload(locale:)` を使用します。Rork Max に「テキストと URL の両方を解析してください」と指示すると、適切な分岐コードが生成されます。

## 応用：電子名刺アプリの構築

NFC の代表的なユースケースが電子名刺です。Rork Max に以下のプロンプトを入力すると、名前・肩書き・SNS リンクをまとめた vCard を NFC タグに書き込み、相手の iPhone で読み取ると連絡先に追加される仕組みを持つアプリが生成されます。

```
「NFC 電子名刺アプリを作ってください。
 - 自分の名前・会社・メール・Twitter を入力するフォーム
 - 入力した情報を vCard 形式で NFC タグに書き込むボタン
 - 他の人の NFC タグを読み取り、連絡先に保存するボタン
 - 受け取った名刺の一覧を表示する履歴画面」
```

このようにプロンプトだけで電子名刺の読み書き両対応アプリが完成します。Stripe 課金を組み合わせてプレミアム名刺テンプレートを販売するビジネスモデルも検討できます。収益化については [Rork Max + Stripe サブスクリプション実装](/articles/rork-dev/rork-max-stripe-subscription) もあわせてご参照ください。

## FAQ

**Q: NFC タグはどこで購入できますか？**

A: Amazon や AliExpress で「NFC タグ ステッカー NTAG215」などで検索すると、1 枚あたり数十円から入手できます。NTAG213・NTAG215・NTAG216 が iOS に広く対応しており、Rork Max で生成したアプリとの互換性も良好です。

**Q: Android でも同じ NFC タグを読み取れますか？**

A: NDEF 形式で書き込んだタグは Android でも読み取れます。ただし、Rork Max で生成するのは iOS（Swift）アプリのため、Android 対応が必要な場合はクロスプラットフォーム版の Rork（React Native）を使うか、Android 向け別アプリを開発することになります。

**Q: NFC タグに保存できるデータ量はどのくらいですか？**

A: タグの種類によりますが、NTAG213 は 144 バイト、NTAG215 は 504 バイト、NTAG216 は 888 バイトです。URL や短いテキストであれば NTAG213 で十分です。大量のデータを扱う場合は NFC をトリガーとして使い、実データはクラウドに保存してリンクだけをタグに書き込む設計が一般的です。

**Q: バックグラウンドでも NFC タグを読み取れますか？**

A: iOS 13 以降の対応機種（iPhone XS 以降）では、アプリを起動していない状態でも特定のドメインの URL タグを読み取れる「バックグラウンド Tag Reading」が利用可能です。ただし NDEF URL レコードのみ対応で、アプリのカスタム処理は走りません。アプリを起動させてカスタム処理を実行するには、ユニバーサルリンクと組み合わせる必要があります。詳しくは [Rork Max ディープリンク・ユニバーサルリンク実装ガイド](/articles/rork-dev/rork-max-deep-linking-universal-links) をご覧ください。

Rork Max を使えば、NFC タグの読み取り・書き込みを実装した本格的な iOS アプリをプロンプトだけで構築できます。Core NFC フレームワークのネイティブ統合により、Xcode なしでも高品質なコードが生成されます。電子名刺・スマートロック連携・商品認証など、NFC の活用シーンは多岐にわたります。
