## はじめに

モバイルアプリにとって、プッシュ通知はユーザーエンゲージメントを高める最も効果的な機能のひとつです。Rork Maxは、ネイティブSwiftコードを自動生成するAIアプリビルダーであり、Apple Push Notification service（APNs）との連携も自然言語のプロンプトで実装できます。この記事では、Rork Maxを使ってiOSアプリにプッシュ通知を実装する方法をステップバイステップで解説します。

## プッシュ通知の仕組み

iOSのプッシュ通知は、以下の流れで動作します：

1. アプリがユーザーに通知許可を要求する
2. ユーザーが許可すると、APNs（Apple Push Notification service）からデバイストークンを取得
3. デバイストークンをサーバー（バックエンド）に送信・保存
4. バックエンドがAPNsにリクエストを送信
5. APNsがデバイスにプッシュ通知を配信

Rork Maxでは、ステップ1〜2のネイティブコード（SwiftUI）を自動生成し、ステップ3〜5のバックエンド連携もCloudflare WorkersやSupabaseと組み合わせて実装できます。

## Step 1: Apple Developer Programの設定

プッシュ通知を使うには、Apple Developer Program（年間99ドル）のアカウントが必要です。

### APNsキーの取得

1. [Apple Developer Portal](https://developer.apple.com/) にログイン
2. **Certificates, Identifiers & Profiles** → **Keys** に移動
3. **+** ボタンをクリックして新しいキーを作成
4. **Apple Push Notifications service (APNs)** にチェックを入れる
5. キーをダウンロードし、安全な場所に保管

<Callout type="warning">
APNsキー（`.p8`ファイル）は一度しかダウンロードできません。必ず安全に保管してください。
</Callout>

メモしておく情報：
- **Key ID**: ダウンロードしたキーのID（10桁）
- **Team ID**: Apple Developer アカウントのチームID
- **Bundle ID**: アプリのバンドルID（例: `com.yourname.appname`）

## Step 2: Rork MaxでSwiftUIコードを生成

Rork Maxのエディタに以下のプロンプトを入力します：

```
iOSアプリにプッシュ通知機能を追加してください。
要件：
- 起動時にユーザーに通知許可を求める
- APNsデバイストークンを取得してコンソールに表示する
- UserNotificationsフレームワークを使用する
- デバイストークンをHTTP POSTリクエストでhttps://api.myapp.com/register-deviceに送信する
```

Rork Maxは以下のようなSwiftUIコードを生成します：

```swift
import SwiftUI
import UserNotifications

@main
struct MyApp: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var appDelegate

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}

class AppDelegate: NSObject, UIApplicationDelegate, UNUserNotificationCenterDelegate {
    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        requestNotificationPermission()
        return true
    }

    func requestNotificationPermission() {
        UNUserNotificationCenter.current().delegate = self
        UNUserNotificationCenter.current().requestAuthorization(
            options: [.alert, .badge, .sound]
        ) { granted, error in
            if granted {
                DispatchQueue.main.async {
                    UIApplication.shared.registerForRemoteNotifications()
                }
            }
        }
    }

    func application(_ application: UIApplication,
                     didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
        let token = deviceToken.map { String(format: "%02.2hhx", $0) }.joined()
        print("APNs Token: \(token)")
        registerDeviceToken(token)
    }

    func application(_ application: UIApplication,
                     didFailToRegisterForRemoteNotificationsWithError error: Error) {
        print("Failed to register: \(error)")
    }

    func registerDeviceToken(_ token: String) {
        guard let url = URL(string: "https://api.myapp.com/register-device") else { return }
        var request = URLRequest(url: url)
        request.httpMethod = "POST"
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")
        let body = ["device_token": token, "platform": "ios"]
        request.httpBody = try? JSONSerialization.data(withJSONObject: body)
        URLSession.shared.dataTask(with: request).resume()
    }
}
```

## Step 3: Cloudflare Workersでバックエンドを構築

プッシュ通知を送信するバックエンドを、Cloudflare Workers上に実装します。

```javascript
// Cloudflare Workers: push-notification-api
export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    if (url.pathname === '/register-device' && request.method === 'POST') {
      return handleRegister(request, env);
    }

    if (url.pathname === '/send-push' && request.method === 'POST') {
      return handleSendPush(request, env);
    }

    return new Response('Not Found', { status: 404 });
  }
};

async function handleRegister(request, env) {
  const { device_token, platform } = await request.json();

  await env.TOKENS.put(`device:${device_token}`, JSON.stringify({
    token: device_token,
    platform,
    registered_at: new Date().toISOString()
  }));

  return Response.json({ success: true });
}

async function handleSendPush(request, env) {
  const { device_token, title, body } = await request.json();

  const jwt = await generateAPNsJWT(env);

  const response = await fetch(
    `https://api.push.apple.com/3/device/${device_token}`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${jwt}`,
        'apns-topic': env.BUNDLE_ID,
        'apns-push-type': 'alert',
        'content-type': 'application/json'
      },
      body: JSON.stringify({
        aps: {
          alert: { title, body },
          sound: 'default',
          badge: 1
        }
      })
    }
  );

  if (response.status === 200) {
    return Response.json({ success: true });
  } else {
    const error = await response.text();
    return Response.json({ success: false, error }, { status: 500 });
  }
}
```

### Cloudflare WorkersのEnvironment Variables設定

| 変数名 | 値 |
|--------|----|
| `APNS_KEY` | `.p8`ファイルの内容 |
| `APNS_KEY_ID` | APNsキーのID（10桁） |
| `TEAM_ID` | Apple Developer チームID |
| `BUNDLE_ID` | アプリのバンドルID |

## Step 4: テストとデバッグ

<Callout type="info">
プッシュ通知は実機でのみ動作します。シミュレーターでは通知許可のダイアログは表示されますが、実際のAPNsトークンは取得できません。
</Callout>

### 実機でのテスト手順

1. Rork Maxで生成したアプリをQRコードで実機にインストール
2. アプリを起動し、通知許可を許可
3. コンソールに表示されるAPNsトークンをコピー
4. curl でプッシュ通知を送信してテスト：

```bash
curl -X POST https://your-worker.workers.dev/send-push \
  -H "Content-Type: application/json" \
  -d '{
    "device_token": "YOUR_DEVICE_TOKEN",
    "title": "テスト通知",
    "body": "Rork Maxからこんにちは！"
  }'
```

### よくあるエラーと対処法

| エラー | 原因 | 対処法 |
|--------|------|--------|
| `BadDeviceToken` | トークンが無効 | 実機で再取得 |
| `InvalidBundleId` | Bundle IDが一致しない | Developer Portalで確認 |
| `ExpiredProviderToken` | JWTが期限切れ（1時間） | JWT再生成ロジックを実装 |
| `TooManyRequests` | レート制限 | 送信間隔を調整 |
