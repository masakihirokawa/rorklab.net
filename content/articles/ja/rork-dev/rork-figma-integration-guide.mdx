## Rork と Figma の連携でできること

**Figma** でデザインしたUIをそのまま Rork のアプリコードに変換できます。デザイナーとエンジニアの間にある「デザインを見てコードを書き直す」という無駄な工程を削減し、**デザインの意図を正確にアプリに反映** することが可能になります。

Rork × Figma 連携の主なメリットは次の通りです。

- デザインカンプからの手動実装作業が不要になる
- デザイナーが作ったコンポーネントをそのまま活用できる
- スペーシング・カラー・タイポグラフィのデザイントークンを自動で取得できる
- デザインの更新をアプリに反映するサイクルが大幅に短縮される

---

## 連携の前提条件

Rork × Figma 連携を行うには、以下が必要です。

- Rork アカウント（無料プランでも基本連携は可能）
- Figma アカウント（フリープランでも可）
- Figma の **アクセストークン**（Personal Access Token）

---

## Step 1：Figma のアクセストークンを取得する

1. Figma にログインし、右上のプロフィールアイコンをクリック
2. **「Settings」** を選択
3. **「Personal access tokens」** セクションに移動
4. **「Generate new token」** をクリックしてトークンを生成
5. 生成されたトークンをコピーして安全な場所に保存

> ⚠️ アクセストークンは一度しか表示されません。必ずすぐにコピーしておいてください。

---

## Step 2：Rork に Figma を連携する

1. Rork のプロジェクト画面を開く
2. 設定アイコン または **「Integrations」** メニューをクリック
3. **「Figma」** を選択
4. Figma のアクセストークンを入力して **「Connect」** をクリック
5. インポートしたいFigmaファイルのURLを入力

---

## Step 3：デザインをRorkにインポートする

Figmaファイルが連携されると、デザインの構造をRorkが解析します。

```
インポートされるもの：
- フレーム（画面単位のデザイン）
- コンポーネント（ボタン・カード・ナビゲーション等）
- カラースタイル（ブランドカラー・テキストカラー等）
- テキストスタイル（フォントファミリー・サイズ・ウェイト）
- スペーシング（余白の定義）
```

### インポート時の注意点

**コンポーネント名はわかりやすく設定する**
Figmaのレイヤー名がそのままコンポーネント名になります。「Frame 3」ではなく「LoginButton」「ProductCard」のように、役割がわかる名前にしておきましょう。

**Auto Layout を積極的に使う**
Figma の Auto Layout を使って設計されたコンポーネントは、Rork が React Native の Flexbox レイアウトにより正確に変換できます。

**バリアントを定義しておく**
ボタンの「default/hover/disabled」や、カードの「expanded/collapsed」などのバリアントを Figma で定義しておくと、Rork が状態管理も含めたコンポーネントを生成しやすくなります。

---

## Step 4：デザイントークンの活用

デザイントークン（カラー・スペーシング・タイポグラフィの定義）を Figma から Rork に取り込むことで、アプリ全体で一貫したスタイルを維持できます。

```javascript
// Figmaから取り込まれたデザイントークンの例
export const theme = {
  colors: {
    primary: '#0066FF',      // Figmaのブランドカラーがそのまま反映
    secondary: '#FF6600',
    background: '#F5F5F5',
    text: {
      primary: '#1A1A1A',
      secondary: '#666666',
    },
  },
  spacing: {
    xs: 4,
    sm: 8,
    md: 16,
    lg: 24,
    xl: 32,
  },
  typography: {
    heading1: {
      fontFamily: 'Inter',
      fontSize: 28,
      fontWeight: '700',
    },
    body: {
      fontFamily: 'Inter',
      fontSize: 16,
      fontWeight: '400',
    },
  },
};
```

このテーマファイルを使うことで、Figmaでカラーを変更した際にアプリ全体のカラーを1箇所更新するだけで反映できます。

---

## Step 5：生成されたコードをカスタマイズする

Figmaからインポートされたコードは完璧ではありません。Rork Companion を使って、インタラクションやビジネスロジックを追加する必要があります。

```
Figmaからインポートされたもの（静的なUI）：
- レイアウト・見た目・スタイリング

Rork Companionで追加するもの（動的なふるまい）：
- ボタンのタップイベント
- フォームのバリデーション
- APIとのデータ連携
- 画面遷移のナビゲーション
- アニメーション・トランジション
```

## Figmaデザインを良質なRorkコードに変換するベストプラクティス

### デザイン前の準備

- Figmaでコンポーネントをきちんと定義しておく（シンボル/コンポーネント化）
- カラースタイル・テキストスタイルをFigmaの「Styles」で管理しておく
- 8pxグリッドシステムなど、一貫したスペーシングルールを使う

### インポート後の確認項目

- フォントが正しく読み込まれているか（カスタムフォントは別途設定が必要な場合あり）
- ピクセルパーフェクトでなくてもよいが、デザインの意図が伝わっているか確認
- 画像・アイコンが正しく表示されているか

## よくある質問（FAQ）

**Q. Figmaの有料プランが必要ですか？**

基本的なインポートはFigmaのフリープランでも可能です。ただし、チームでのコラボレーションやブランチ機能を使う場合はFigmaの有料プランが必要になります。

**Q. デザインを更新した後、Rorkに再同期できますか？**

はい。Figmaのデザインを更新した後、Rorkから再インポートを行うことで差分を反映できます。ただし、Rorkで加えたコードの変更が上書きされる場合があるため、同期のタイミングには注意が必要です。

**Q. Figmaのプロトタイプ（インタラクション定義）も読み込まれますか？**

現時点では、FigmaのプロトタイプはRorkには直接インポートされません。画面遷移やインタラクションはRork Companionに指示を出して実装する形になります。

---

## まとめ

Rork × Figma 連携を使うことで、デザインとアプリ開発のギャップを大幅に縮めることができます。

1. **Figma のアクセストークン**を取得して Rork に設定
2. **デザインをインポート**してコンポーネント・デザイントークンを取得
3. **Rork Companion** でインタラクションとビジネスロジックを追加
4. **デザイントークン**を活用して全体のスタイル一貫性を保つ

デザイナーと開発者が同じFigmaファイルを起点に作業できるため、コミュニケーションコストも大幅に削減されます。ぜひ試してみてください。

Figma AI の最新動向については、[Figma AIエージェントによるアプリUI自動化ガイド](/articles/rork-ai/figma-ai-agent-app-ui-automation)もあわせてご覧ください。
