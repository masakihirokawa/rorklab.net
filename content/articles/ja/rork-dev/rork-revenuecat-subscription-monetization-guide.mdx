# Rork × RevenueCAT課金完全実装ガイド：月3万円を稼ぐサブスクアプリの作り方

この記事では、**RorkアプリにRevenueCATを組み込んで月3万円の収益を稼ぐまでの全手順** を解説します。コピペで動くコード、価格設計のノウハウ、App Store/Google Play課金審査の通すコツまで、実践的な情報を全て公開します。

## RevenueCATとは？ なぜRorkアプリに必要か

### RevenueCATの3つのメリット

**1. コード削減** — Apple、Google、Stripe の課金API を統一インターフェースで操作
- App Store IAP（In-App Purchase）
- Google Play Billing
- Stripe の API
すべて同じコードで対応できます。

**2. サンドボックステスト簡略化** — 本番環境に影響なくテスト
- RevenueCAT Dashboard で テスト購入を追跡
- Sandbox 環境の トランザクション を確認
- クラッシュレポート（Crash Analytics）が統合

**3. ビジネスロジック集約** — サーバーレスで課金ロジックを運用
- クライアント側は「購入成功」「失敗」を受け取るだけ
- バックエンド実装不要（RevenueCAT がサーバー代わり）
- 複数プラットフォーム間でのユーザー認識を自動化

## 収益目標の設計：月3万円を稼ぐための計算式

月3万円（≒ $200）を稼ぐために、どの料金設定を選ぶべきか？

### パターン別シミュレーション

**パターン① 月額 ¥980($7)、コンバージョン率 5%**
```
必要ユーザー数 = ¥30,000 / ¥980 = 31ユーザー
DAU(日次アクティブユーザー) = 31 / 0.05 = 620 DAU
→ 月間 DAU 約 2,000 必要（30日想定）
```

**パターン② 月額 ¥480($3.5)、コンバージョン率 8%**
```
必要ユーザー数 = ¥30,000 / ¥480 = 63ユーザー
DAU = 63 / 0.08 = 788 DAU
→ 月間 DAU 約 2,600 必要
```

**パターン③ 永続版 ¥2,980($20)、コンバージョン率 2%**
```
必要ユーザー数 = ¥30,000 / ¥2,980 = 11ユーザー/月
DAU = 11 / 0.02 = 550 DAU
→ 月間 DAU 約 1,800 必要
```

### 最適な価格選択

- **初心者アプリ** — 月額¥480（$3.5）が最適。気軽に試すユーザーが多い
- **ニッチなアプリ** — 月額¥980（$7）。専門性が高いほどコンバージョン率が上がる
- **1回限りの価値** — 永続版 ¥2,980（$20）。永続版は初期導入が大変だが、一度売れると安定

### ステップ2: Rork コードにサービス層を追加

```typescript
// src/services/revenue-cat.ts
import Purchases from '@react-native-purchases/purchases';

const REVENUECAT_API_KEY = process.env.EXPO_PUBLIC_REVENUECAT_KEY || '';
const ENTITLEMENT_ID = 'premium_access'; // RevenueCAT Dashboard で定義

interface UserSubscriptionStatus {
  isPremium: boolean;
  expirationDate?: string;
  autoRenewEnabled?: boolean;
}

export class RevenueCATService {
  static async initialize() {
    try {
      await Purchases.configure({
        apiKey: REVENUECAT_API_KEY,
        appUserID: 'optional-user-id', // または null（自動生成）
      });
    } catch (error) {
      console.error('RevenueCAT init failed:', error);
    }
  }

  static async getOfferings() {
    try {
      const offerings = await Purchases.getOfferings();
      return offerings.current?.availablePackages || [];
    } catch (error) {
      console.error('Failed to fetch offerings:', error);
      return [];
    }
  }

  static async purchasePackage(pkg: any) {
    try {
      const { customerInfo } = await Purchases.purchasePackage(pkg);
      return {
        success: true,
        customerInfo,
      };
    } catch (error: any) {
      if (error.code === 'PurchaseCancelledError') {
        return { success: false, cancelled: true };
      }
      console.error('Purchase failed:', error);
      return { success: false, error };
    }
  }

  static async restorePurchases() {
    try {
      const customerInfo = await Purchases.restorePurchases();
      return customerInfo;
    } catch (error) {
      console.error('Restore failed:', error);
      return null;
    }
  }

  static async getSubscriptionStatus(): Promise<UserSubscriptionStatus> {
    try {
      const customerInfo = await Purchases.getCustomerInfo();
      const entitlement = customerInfo.entitlements.active[ENTITLEMENT_ID];

      if (entitlement) {
        return {
          isPremium: true,
          expirationDate: entitlement.expirationDate,
          autoRenewEnabled: entitlement.renewalInfo?.willRenew,
        };
      }

      return { isPremium: false };
    } catch (error) {
      console.error('Failed to get subscription status:', error);
      return { isPremium: false };
    }
  }
}
```

### ステップ3: グローバル状態管理（Context）

```typescript
// src/context/SubscriptionContext.tsx
import React, { createContext, useEffect, useState } from 'react';
import { RevenueCATService } from '../services/revenue-cat';

interface SubscriptionContextType {
  isPremium: boolean;
  offerings: any[];
  isLoading: boolean;
  error?: string;
  purchase: (pkg: any) => Promise<void>;
  restore: () => Promise<void>;
}

export const SubscriptionContext = createContext<SubscriptionContextType | null>(null);

export function SubscriptionProvider({ children }: { children: React.ReactNode }) {
  const [isPremium, setIsPremium] = useState(false);
  const [offerings, setOfferings] = useState([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | undefined>();

  useEffect(() => {
    initializeRevenueCAT();
  }, []);

  const initializeRevenueCAT = async () => {
    try {
      setIsLoading(true);
      await RevenueCATService.initialize();

      // 初期状態取得
      const status = await RevenueCATService.getSubscriptionStatus();
      setIsPremium(status.isPremium);

      const offerings = await RevenueCATService.getOfferings();
      setOfferings(offerings);
    } catch (err: any) {
      setError(err.message || 'Failed to initialize RevenueCAT');
    } finally {
      setIsLoading(false);
    }
  };

  const purchase = async (pkg: any) => {
    try {
      setIsLoading(true);
      const result = await RevenueCATService.purchasePackage(pkg);

      if (result.success) {
        const status = await RevenueCATService.getSubscriptionStatus();
        setIsPremium(status.isPremium);
      } else if (!result.cancelled) {
        setError('Purchase failed');
      }
    } finally {
      setIsLoading(false);
    }
  };

  const restore = async () => {
    try {
      setIsLoading(true);
      await RevenueCATService.restorePurchases();
      const status = await RevenueCATService.getSubscriptionStatus();
      setIsPremium(status.isPremium);
    } catch (err: any) {
      setError('Restore failed');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <SubscriptionContext.Provider value={{ isPremium, offerings, isLoading, error, purchase, restore }}>
      {children}
    </SubscriptionContext.Provider>
  );
}

export function useSubscription() {
  const context = React.useContext(SubscriptionContext);
  if (!context) {
    throw new Error('useSubscription must be used within SubscriptionProvider');
  }
  return context;
}
```

## App Store/Google Play課金審査のポイント

### App Store での審査

Appleは以下を厳しく確認します：

- **購入の透明性** — 購入後に何が得られるか明確か
- **購入復元** — ユーザーが別デバイスで復元できるか
- **自動更新の説明** — サブスクはいつ更新されるか明記か
- **キャンセル方法** — ユーザーが簡単に解約できるか

**対応方法**:
```
App Store Connect → アプリ情報 → 価格と利用可能性
→ 「自動更新」セクションで、更新期間・価格・キャンセル方法を明確に記載
```

### Google Play での審査

Google Play はApple より審査が緩いですが、以下は確認します：

- **BillingClient の適切な使用** — API を正しく呼び出しているか
- **テスト購入アカウント** — ホワイトリストに登録されているか

**対応方法**:
```
Google Play Console → アプリ → テスター → テストメール登録
```

### よくあるリジェクト理由

- 「購入画面で利用規約が表示されていない」 → RevenueCAT ペイウォールに法務文を追加
- 「領域によって価格が異なるのに説明がない」 → App Store での地域別価格設定を確認
- 「トライアル後の自動更新が明確でない」 → Reviewer Information（審査用コメント）に明記

## FAQ

### Q1: レシート検証は必要？

**A**: RevenueCATが自動で行うので、Rork コード側で実装不要です。ただし、独自のバックエンド（Node.js/Python等）を持ち、購入を確認したい場合は、RevenueCATのサーバー側API（`/customers/{app_user_id}`）を呼び出してください。

### Q2: 複数のサブスクプランを提供する場合？

**A**: RevenueCAT の「Package」概念を使用します。同じ エンタイトルメント ID に複数のプランを紐付けることで、ユーザーは自由にプランを切り替えられます。

```javascript
const monthlyPackage = offerings.find(o => o.identifier === 'premium_monthly');
const annualPackage = offerings.find(o => o.identifier === 'premium_annual');
// どちらも同じ 'premium_access' エンタイトルメント
```

### Q3: チップ（ワンタイム購入）と定期購入の併用？

**A**: App Store IAP には 2 つのタイプがあります：
- **Consumable** — ワンタイム購入（チップ）
- **Non-Consumable / Auto-Renewable Subscription** — 定期購入

RevenueCATで両方を管理できます。

### Q4: 国ごとに価格を変えたい

**A**: App Store Connect と Google Play Console で 地域別価格設定が可能です。RevenueCATは自動で現在の言語に応じた価格を表示します。

### Q5: 家族共有対応は？

**A**: Apple の「Family Sharing」はApp Store が自動的に処理します。Rork コード側では追加実装不要。ただし、Google Play の「ファミリーライブラリ」は設定が別途必要です。

## まとめ

RevenueCAT を使うことで、複雑な課金ロジックを驚くほど簡潔に実装できます。

月3万円の収益を目指すなら：
- **正しい価格設定** — ¥480〜¥980/月の範囲から開始
- **無料トライアル 7日** — 最初は標準期間で運用
- **ペイウォール最適化** — A/B テスト で コンバージョン率を高める
- **App Store/Google Play 審査対応** — 早期から審査チェックリストを確認

これらを実装すれば、Rork Max で作ったアプリが立派なマネタイズエンジンになります。
