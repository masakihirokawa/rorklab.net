モバイルアプリをリリースしてユーザーに継続的に使ってもらうには、**プッシュ通知**が欠かせません。「新しいメッセージが届いた」「セールが始まった」「タスクの期限が近づいている」——こうした通知がユーザーをアプリへ呼び戻します。このガイドでは、Rork で生成したアプリに Expo Notifications を使ってプッシュ通知を組み込む方法を、コード例を交えて徹底解説します。

## プッシュ通知の仕組みを理解する

Rork が生成するアプリは **Expo（React Native）** をベースにしています。Expo には `expo-notifications` というライブラリが用意されており、iOS（APNs）と Android（FCM）の両プラットフォームに対応したプッシュ通知を統一 API で扱えます。

通知の流れは次のとおりです。

```
アプリ起動
  ↓
expo-notifications でデバイストークンを取得
  ↓
トークンをバックエンド（Supabase 等）に保存
  ↓
サーバーサイド / 管理画面から Expo Push API へリクエスト
  ↓
Expo Push Service が APNs / FCM に中継
  ↓
ユーザーのデバイスに通知が届く
```

Expo Push Service を使うと、APNs・FCM の証明書管理を Expo 側に任せられるため、個人開発者でもシンプルに通知を送れます。

## Step 1：expo-notifications のセットアップ

Rork に以下のようなプロンプトを送ると、必要なパッケージとコードが自動生成されます。

```
expo-notifications を使ってプッシュ通知を実装して。
デバイストークンを取得してコンソールに出力し、
通知を受信したときにアラートを表示するようにして。
```

Rork は `app.json` の設定変更・パッケージのインストール・通知ハンドラの実装まで一括で行います。生成されるコードのベースは次のようになります。

```typescript
// hooks/usePushNotifications.ts
// プッシュ通知のセットアップとトークン取得を管理するカスタムフック

import { useEffect, useRef, useState } from 'react';
import * as Notifications from 'expo-notifications';
import * as Device from 'expo-device';
import Constants from 'expo-constants';
import { Platform } from 'react-native';

// 通知を受信したときにバナー＋音で表示する設定
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,   // バナー通知を表示
    shouldPlaySound: true,   // 通知音を鳴らす
    shouldSetBadge: false,   // バッジ数は変更しない
  }),
});

export function usePushNotifications() {
  const [expoPushToken, setExpoPushToken] = useState<string | null>(null);
  const [notification, setNotification] = useState<Notifications.Notification | null>(null);
  const notificationListener = useRef<Notifications.EventSubscription>();
  const responseListener = useRef<Notifications.EventSubscription>();

  useEffect(() => {
    // デバイストークンを取得する非同期関数
    async function registerForPushNotifications() {
      // 実機以外（シミュレーター等）では通知が取得できない
      if (!Device.isDevice) {
        console.warn('プッシュ通知は実機のみサポートされています');
        return;
      }

      // 通知の許可をリクエスト
      const { status: existingStatus } = await Notifications.getPermissionsAsync();
      let finalStatus = existingStatus;

      if (existingStatus !== 'granted') {
        const { status } = await Notifications.requestPermissionsAsync();
        finalStatus = status;
      }

      if (finalStatus !== 'granted') {
        console.warn('プッシュ通知の許可が得られませんでした');
        return;
      }

      // Expo プロジェクトIDを使ってトークンを取得
      const projectId = Constants.expoConfig?.extra?.eas?.projectId;
      const tokenData = await Notifications.getExpoPushTokenAsync({ projectId });
      const token = tokenData.data;

      console.log('Expo Push Token:', token);
      // 期待する出力例: ExponentPushToken[xxxxxxxxxxxxxxxxxxxxxx]
      setExpoPushToken(token);

      // Android のチャンネル設定（Android 8.0 以上で必須）
      if (Platform.OS === 'android') {
        await Notifications.setNotificationChannelAsync('default', {
          name: 'Default',
          importance: Notifications.AndroidImportance.MAX,
          vibrationPattern: [0, 250, 250, 250],
          lightColor: '#6c63ff',
        });
      }
    }

    registerForPushNotifications();

    // 通知を受信したときのリスナー
    notificationListener.current = Notifications.addNotificationReceivedListener(
      (notification) => {
        setNotification(notification);
        console.log('通知受信:', notification.request.content.title);
      }
    );

    // 通知をタップしたときのリスナー
    responseListener.current = Notifications.addNotificationResponseReceivedListener(
      (response) => {
        const data = response.notification.request.content.data;
        console.log('通知タップ時のデータ:', data);
        // ここで画面遷移などのアクションを追加できる
      }
    );

    return () => {
      // コンポーネントアンマウント時にリスナーを解除
      notificationListener.current?.remove();
      responseListener.current?.remove();
    };
  }, []);

  return { expoPushToken, notification };
}
```

**期待する動作：**
- アプリ起動時に通知許可ダイアログが表示される
- 許可後、`ExponentPushToken[...]` 形式のトークンがコンソールに出力される
- アプリ起動中に通知を受信するとバナーと音が発生する

## Step 2：トークンをバックエンドに保存する

取得したトークンは、サーバーから通知を送るために**バックエンドに保存**する必要があります。Supabase を使う場合は次のように実装します。

```typescript
// lib/savePushToken.ts
// 取得したトークンを Supabase の user_push_tokens テーブルに保存する

import { supabase } from './supabase';

export async function savePushTokenToSupabase(
  userId: string,
  token: string
): Promise<void> {
  const { error } = await supabase
    .from('user_push_tokens')
    .upsert(
      { user_id: userId, expo_push_token: token, updated_at: new Date().toISOString() },
      { onConflict: 'user_id' }  // 同じユーザーのレコードは上書き
    );

  if (error) {
    console.error('トークン保存エラー:', error.message);
    throw error;
  }

  console.log('トークンを保存しました:', token);
  // 期待する出力: トークンを保存しました: ExponentPushToken[xxx...]
}
```

Supabase の `user_push_tokens` テーブルには、`user_id（text）`・`expo_push_token（text）`・`updated_at（timestamptz）` の3カラムを用意してください。Rork へのプロンプトで「Supabase に user_push_tokens テーブルを作って、ユーザーのトークンを保存するようにして」と指示すれば、テーブル定義とコードを自動生成してくれます。

バックエンド連携の詳細は「[Rork × Supabase チャットアプリ実装ガイド](/articles/rork-dev/rork-supabase-chat-app)」も参考にしてください。

## Step 3：バックエンドから通知を送信する

トークンを取得・保存したら、**サーバーサイドから通知を送信**します。Expo Push API は HTTP リクエストを受け付けるので、Node.js や Supabase Edge Functions から簡単に呼び出せます。

```typescript
// functions/send-push-notification/index.ts
// Supabase Edge Function でプッシュ通知を送信する例

import { serve } from 'https://deno.land/std@0.168.0/http/server.ts';

interface PushNotificationPayload {
  to: string | string[];      // Expo Push Token（複数可）
  title: string;              // 通知タイトル
  body: string;               // 通知本文
  data?: Record<string, unknown>; // アプリ側で受け取る追加データ
  sound?: 'default' | null;   // 通知音
  badge?: number;             // バッジ数
}

serve(async (req) => {
  const { userId, title, body, data } = await req.json();

  // Supabase からトークンを取得
  const supabaseUrl = Deno.env.get('SUPABASE_URL')!;
  const supabaseKey = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!;

  const response = await fetch(
    `${supabaseUrl}/rest/v1/user_push_tokens?user_id=eq.${userId}`,
    { headers: { apikey: supabaseKey, Authorization: `Bearer ${supabaseKey}` } }
  );
  const tokens = await response.json();

  if (!tokens.length) {
    return new Response(JSON.stringify({ error: 'トークンが見つかりません' }), { status: 404 });
  }

  // Expo Push API にリクエスト
  const payload: PushNotificationPayload = {
    to: tokens.map((t: { expo_push_token: string }) => t.expo_push_token),
    title,
    body,
    data: data ?? {},
    sound: 'default',
  };

  const pushResponse = await fetch('https://exp.host/--/api/v2/push/send', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Accept: 'application/json',
    },
    body: JSON.stringify(payload),
  });

  const result = await pushResponse.json();
  console.log('送信結果:', JSON.stringify(result));
  // 期待する出力: { "data": [{ "status": "ok", "id": "xxxxx" }] }

  return new Response(JSON.stringify(result), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

このエッジ関数をデプロイすれば、任意のタイミング（ユーザーのアクション・定期実行・Webhook など）で通知を送れます。

## Step 4：通知タップ時の画面遷移を実装する

通知をタップしたときに特定の画面へ遷移させるには、`data` フィールドを活用します。

```typescript
// app/_layout.tsx（抜粋）
// 通知タップ時に data.screen の画面へ自動遷移

import { useEffect } from 'react';
import { useRouter } from 'expo-router';
import * as Notifications from 'expo-notifications';

export default function RootLayout() {
  const router = useRouter();

  useEffect(() => {
    const subscription = Notifications.addNotificationResponseReceivedListener(
      (response) => {
        const screen = response.notification.request.content.data?.screen as string;
        if (screen) {
          // data.screen に指定した画面パスへ遷移
          router.push(screen as never);
          // 例: { screen: '/messages/123' } → チャット画面を開く
        }
      }
    );
    return () => subscription.remove();
  }, [router]);

  // ... 以下、通常のレイアウトコード
}
```

送信時の `data` に `{ screen: '/messages/123' }` のように設定しておくと、通知タップで直接そのチャットルームへ飛ばすことができます。より高度なデバッグ手法は「[Fix Now 自動デバッグ機能の使い方](/articles/rork-basics/fix-now-auto-debug)」も参照してください。

## Rork Max でのプッシュ通知実装

[Rork Max](/articles/rork-dev/rork-max-native-api-guide) を使う場合、より高度なネイティブ通知機能も実装できます。

- **リッチ通知**: 通知に画像・動画・カスタムアクションボタンを追加（iOS 15+ / Android 12+）
- **サイレント通知**: バックグラウンドでデータ同期をトリガーする通知（`content-available: 1`）
- **スケジュール通知**: 指定日時・繰り返しパターンでローカル通知を配信

Rork Max では Claude Opus が複雑なネイティブ API の実装を担当するため、「iOS のリッチ通知にアクションボタン（既読／返信）を追加して」といった高度な指示でも精度高く実装されます。

## よくある質問（FAQ）

**Q. シミュレーターで通知のテストができません**

A. Expo Push Token の取得には**実機が必要**です。シミュレーター（iOS Simulator / Android Emulator）では `Device.isDevice` が `false` になるため、トークン取得がスキップされます。実機をお持ちでない場合は、[Expo Go](https://expo.dev/go) アプリをインストールした実機か、EAS Build でビルドした開発用アプリを使ってください。

**Q. iOS でプッシュ通知が届きません**

A. 以下を順番に確認してください。①`app.json` の `ios.bundleIdentifier` が Apple Developer Portal の App ID と一致しているか、②`eas.json` の `submit.production.ios` に正しい Apple アカウント情報が設定されているか、③Expo Dev Client ではなく EAS Build でビルドした本番／開発ビルドを使っているか。Expo Go 経由では iOS のプッシュ通知は機能しません（Expo の共有証明書でのみ動作）。

**Q. 大量のユーザーに一斉通知を送るには？**

A. Expo Push API は1回のリクエストで最大**100トークン**まで配列で指定できます。それ以上のユーザーへの一斉通知（ブロードキャスト）は、100件ずつに分割してリクエストを送るか、Supabase の pg_cron で定期的にバッチ送信するアーキテクチャが有効です。また、通知の開封率向上には、セグメント配信（特定のユーザー属性に絞った通知）とパーソナライズドメッセージが効果的です。

**Q. Expo Push Token はいつ失効しますか？**

A. ユーザーがアプリを再インストールしたとき・OS のアップデートによってデバイス識別子が変わったとき・通知許可を取り消したときに古いトークンが無効になります。Expo Push API からのレスポンスで `DeviceNotRegistered` エラーが返ってきた場合は、そのトークンをデータベースから削除する処理を実装しておくと、無効トークンへの無駄な送信を防げます。

## まとめ：Rork × Expo Notifications で手早く通知を実装する

プッシュ通知の実装は、以下の4ステップで完結します。

1. `expo-notifications` でデバイストークンを取得・許可リクエスト
2. トークンをバックエンド（Supabase 等）に保存
3. サーバーサイドから Expo Push API を呼び出して通知を送信
4. 通知タップ時のディープリンクで画面遷移を実装

Rork に「プッシュ通知を実装して、トークンを Supabase に保存して、バックエンドから送信できるようにして」と伝えるだけで、今回紹介したコードの大部分が自動生成されます。あとはプロジェクト固有の ID・テーブル名・画面パスを調整するだけです。

プッシュ通知を活用してユーザーエンゲージメントを高め、アプリのリテンション改善につなげましょう。
