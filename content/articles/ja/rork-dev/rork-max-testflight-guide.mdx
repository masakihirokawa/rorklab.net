## はじめに：なぜTestFlightが必要なのか

Rork Maxで作ったアプリが「シミュレーターで動く」状態になったら、次のステップは**実機でのベータテスト**です。TestFlightはAppleが提供する公式ベータ配布プラットフォームで、App Store本審査の前に最大10,000人のテスターに実機でアプリを配布できます。

この記事で学べること：

- TestFlightの仕組みと内部・外部テスターの違い
- App Store Connectでのアプリ登録手順
- Rork MaxからIPAをアップロードする方法
- 内部テスター・外部テスターへの招待フロー
- よくあるエラーとその対処法
- リリースサイクルを短縮する実践テクニック

**対象読者**: Rork Maxでアプリを作り、初めてTestFlight配布に挑戦する個人開発者

---

## 前提知識・環境準備

### 必要なもの

| 項目 | 内容 |
|------|------|
| Apple Developer Program | 年間99ドル（個人／法人）の有料メンバーシップが必須 |
| App Store Connectアカウント | Apple IDでログイン可能（Developer Program会員は自動取得） |
| Rork Maxサブスクリプション | 月額200ドル（ネイティブSwiftビルドに必要） |
| iOSデバイス（任意） | テスター側で実機確認する場合。シミュレーターのみでも登録は可能 |

> **注意**: Apple Developer Programへの加入は[developer.apple.com](https://developer.apple.com)から行います。審査に数時間〜1営業日かかることがあるため、リリース前日に申請するのは避けましょう。

## TestFlightとは：基礎知識

TestFlightはAppleが提供するベータテストサービスで、App Storeの正式審査を通さずにiOSアプリを配布できます。

### 内部テスターと外部テスターの違い

| 区分 | 上限人数 | 招待方法 | App Store審査 |
|------|----------|----------|---------------|
| 内部テスター | 100名 | Apple IDメール招待（Developer Program会員のみ） | 不要 |
| 外部テスター | 10,000名 | メールまたは公開リンク | 初回ビルドのみ簡易審査あり（Beta App Review） |

**個人開発者の典型的なフロー**: まず社内（内部テスター）でクラッシュを潰してから、外部テスターに公開リンクを配布してフィードバックを集める、という2段階が王道です。

## ステップ1：App Store Connectにアプリを登録する

### 1-1. 新規Appを作成

1. [App Store Connect](https://appstoreconnect.apple.com)にアクセスしてログイン
2. 「マイApp」→「＋」ボタン→「新規App」をクリック
3. 以下の情報を入力する

```
プラットフォーム: iOS
名前: アプリの正式名称（後から変更可能）
プライマリ言語: 日本語
バンドルID: Rork Maxが生成したBundle ID（例: net.rorklab.myapp）
SKU: 任意のユニークな文字列（例: MYAPP001）
ユーザーアクセス: フルアクセス
```

> **Bundle IDの確認方法**: Rork Maxのダッシュボードで対象プロジェクトを開き、「Settings」→「App ID」から確認できます。

### 1-2. Bundle IDをApple Developerポータルに登録

Rork MaxはApp IDを自動生成しますが、初回は手動登録が必要な場合があります。

1. [developer.apple.com/account](https://developer.apple.com/account)にアクセス
2. 「Certificates, IDs & Profiles」→「Identifiers」→「+」をクリック
3. 「App IDs」を選択→「App」を選択
4. Bundle ID（Explicit）にRork Maxと同じIDを入力
5. 必要なCapability（Push Notifications, HealthKit等）を有効化して登録

## ステップ2：Rork MaxからIPAをエクスポートする

Rork Maxではシミュレーターで動作確認後、**2クリックでApp Storeアップロード用のIPAファイル**を生成できます。

```
// Rork Maxのエクスポート手順（概念コード）
// 実際の操作はダッシュボードのGUI上で行います

1. プロジェクトダッシュボードを開く
2. 右上「Deploy」→「TestFlight / App Store」を選択
3. ビルド設定を確認
   - バージョン番号: 1.0.0（例）
   - ビルド番号: 1（毎回インクリメントが必要）
4. 「Build & Upload to App Store Connect」ボタンをクリック
5. Apple IDとApp-Specific Passwordを入力して認証

// 期待する出力:
// ✅ Build succeeded
// ✅ IPA uploaded to App Store Connect
// ⏳ Processing... (通常5〜15分)
```

> **App-Specific Passwordの取得**: [appleid.apple.com](https://appleid.apple.com)にログイン→「サインインとセキュリティ」→「App用パスワード」で生成します。通常のApple IDパスワードではなく、このApp-Specific Passwordを使用してください。

### アップロード後の確認

Rork Maxがアップロードに成功したら、App Store Connectの「TestFlight」タブに移動し、ビルドが「処理中」→「テスト準備完了」になるまで待ちます（通常5〜30分）。

## ステップ3：内部テスターに配布する

### 3-1. テスターグループの作成

1. App Store Connect→対象App→「TestFlight」タブ
2. 「内部テスト」→「グループを追加」（例: "開発チーム"）
3. Apple Developer Program会員のApple IDを追加

### 3-2. ビルドをグループに紐付け

1. ビルドが「テスト準備完了」になったことを確認
2. テスターグループの「ビルドを追加」→対象ビルドを選択
3. テスターにTestFlightアプリ（App Store無料）経由で招待メールが届く

> 内部テスターはビルドごとの詳細な審査なしに即時テスト可能です。招待メールから**TestFlightアプリ→「承諾」→インストール**の3ステップでテスト開始できます。

---

## ステップ4：外部テスターに公開リンクを配布する

外部テスター（最大10,000名）への配布は、初回ビルドのみ**Beta App Review**（通常24〜48時間）が必要です。審査通過後は同グループ内では追加審査なしに更新ビルドを配布できます。

### 4-1. 外部テストグループの作成

1. 「外部テスト」→「グループを追加」（例: "ベータユーザー"）
2. 「公開リンク」を有効化すると、URLを知っている人なら誰でも参加可能になる
3. テスト情報（ベータAppの説明、フィードバックメールアドレス）を入力

### 4-2. Beta App Reviewに提出

```
// 外部テスト審査提出チェックリスト
✅ アプリのスクリーンショットまたはデモ動画（必須ではないが推奨）
✅ ベータApp説明（何をテストしてほしいか明記）
✅ フィードバック連絡先メールアドレス
✅ コンテンツの権利（サードパーティコンテンツを含む場合）
✅ クラッシュが著しく多いビルドは提出前に内部テストで修正すること
```

審査通過後、グループに招待したテスターにTestFlight経由でアプリが届きます。公開リンクをSNSやメルマガで共有すれば、大規模なベータテストを無料で実施できます。

## よくあるエラーと対処法

### エラー1: 「Missing Compliance」警告

**原因**: アプリが暗号化（HTTPS通信等）を使用している場合に表示される輸出規制申告。

**対処法**:
1. App Store Connect→ビルド→「輸出コンプライアンス情報」
2. 「アルゴリズム暗号化を使用していますか？」→「はい（標準暗号化のみ）」を選択
3. 回答後にビルドが有効化される

### エラー2: 「This build has expired」

**原因**: TestFlightビルドは90日間で自動期限切れになります。

**対処法**: Rork Maxから新しいビルド番号でビルドを再アップロードし、テスターに再インストールを依頼する。

### エラー3: ビルド番号の重複エラー

**原因**: 同一バージョン番号でビルド番号を変えずに再アップロードしようとした。

**対処法**:
```
// ビルド番号は毎回インクリメントする必要がある
// 間違い例
version: "1.0.0", build: "1" → 再アップロード時も build: "1"（❌）

// 正しい例
version: "1.0.0", build: "1" → 修正後は build: "2"（✅）
version: "1.0.0", build: "2" → さらに修正は build: "3"（✅）
```

### エラー4: 「Invalid Bundle ID」

**原因**: Rork MaxのBundle IDとApp Store ConnectのBundle IDが一致していない。

**対処法**: App Store Connect→App情報でBundle IDを確認し、Rork Maxの設定と完全一致させる（大文字小文字も含む）。

## 応用：TestFlightを活用したリリースサイクルの最適化

### 週次ベータサイクルの設計

プロとして個人開発をスケールさせるには、定期的なベータリリースサイクルが重要です。

```
月曜: 新機能実装・Rork Maxでビルド
火曜: 内部テスターにビルド配信→フィードバック収集
水曜: クラッシュ修正・UX改善
木曜: 修正版ビルドを外部テスターに配信
金曜: フィードバック分析→翌週の優先事項決定
```

### フィードバック収集のコツ

- TestFlightアプリにはスクリーンショットにコメントを付けてフィードバックできる機能がある
- クラッシュレポートはApp Store Connect→「フィードバック」で確認可能
- Rork LabのDiscordやSlackチャンネルで外部テスターコミュニティを作ると継続率が上がる

### RevenueCatと組み合わせたサブスクリプションテスト

有料機能のテストには、TestFlight環境でのSandbox課金を活用します。

```swift
// TestFlight環境ではSandboxアカウントで購入テストが可能
// App Store Connectでサンドボックステスターアカウントを作成し、
// デバイスのApp Store設定でサンドボックスアカウントにサインイン

// RevenueCatのデバッグログを有効化（開発時のみ）
// Rork MaxのAIに以下のプロンプトを投げる:
// "RevenueCatのdebugLogsEnabledをtrueに設定して"
// → 期待する出力: Purchases.logLevel = .debug が追加される
```

詳細は[RorkアプリにRevenueCatでアプリ内課金を実装する完全ガイド](/articles/rork-dev/rork-revenuecat-inapp-purchase)を参照してください。

## FAQ

### Q1: TestFlightの審査（Beta App Review）はApp Store審査より厳しいですか？

A: いいえ、Beta App Reviewは本審査より大幅に簡略化されており、通常24〜48時間で完了します。ただし、クラッシュが多発するビルドや、明らかにガイドライン違反のコンテンツは拒否されます。外部テスト前に必ず内部テストでクラッシュを潰しておきましょう。

### Q2: TestFlightは無料で利用できますか？

A: TestFlight自体は無料ですが、利用にはApple Developer Program（年間99ドル）への加入が必須です。テスター側はTestFlightアプリ（App Store無料）をインストールするだけで参加できます。

### Q3: Rork Maxで作ったアプリは、TestFlightを使わずに実機テストできますか？

A: Rork Maxのクラウドシミュレーターで基本動作は確認できます。実機インストールには原則としてTestFlight（またはXcode経由のAd Hoc配布）が必要ですが、TestFlightが最も手軽で推奨されるワークフローです。

### Q4: 外部テスターが90日の期限内にアップデートが来ない場合はどうなりますか？

A: テスターのビルドが期限切れになり、アプリが起動できなくなります。新しいビルドをアップロードするか、App Store本審査を通過してリリースする必要があります。ユーザーには期限切れ前にTestFlightアプリから通知が届きます。

## まとめ

TestFlightを使ったベータ配布は、Rork Maxアプリを本番リリース前に磨き上げる最も効果的な方法です。今回のガイドをまとめると：

1. **App Store Connectでアプリ登録** → Bundle IDの一致を確認
2. **Rork MaxからIPAをアップロード** → ビルド番号を毎回インクリメント
3. **内部テスターで品質チェック** → クラッシュを先に潰す
4. **外部テスターへの公開** → Beta App Reviewを通過したら最大10,000名に配布可能
5. **週次サイクルでフィードバックを回す** → ユーザーの声をRork Maxのプロンプトに活かす

次のステップとして、TestFlightで十分なフィードバックが集まったら[Rorkで作ったアプリをApp StoreとGoogle Playに公開する方法](/articles/rork-dev/app-store-publishing)でいよいよ本番リリースに挑戦しましょう。
