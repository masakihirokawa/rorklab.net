## はじめに：インディーデベロッパーの収益化の現実

「Rork でアプリを作れるようになった。次は収益化したい」——この記事を読んでいるあなたは、まさにそのステージに来ているのではないでしょうか。

個人でアプリを開発し、それを収益の柱にすることは、今この瞬間も現実的に可能です。ただし、アプリの収益化は「作って公開すれば自然と稼げる」ほど甘くはありません。適切な収益モデルを選び、ユーザー体験を損なわない範囲で最大化する設計が必要です。

筆者は 2014 年から個人でアプリ開発を続け、AdMob による広告収益が月間 150 万円を超えた経験があります。その経験をもとに、Rork を使ったアプリ収益化の実践的な戦略を体系的に解説します。

## 収益モデルの選び方：3 つの主要パターン

アプリの収益化には主に 3 つのモデルがあります。それぞれの特徴と適したアプリの種類を理解したうえで選択してください。

### モデル 1：広告収益（AdMob）

**特徴**: 無料アプリで広告を表示し、インプレッション・クリックで収益を得るモデル

**向いているアプリ**:
- 日常的に繰り返し使われるアプリ（習慣・記録・メモ系）
- ゲームアプリ（特にハイパーカジュアルゲーム）
- ツール系アプリ（計算機・天気・変換ツール等）

**収益の目安（参考値）**:
- バナー広告 eCPM: ¥100〜¥400（日本・iOS）
- インタースティシャル広告 eCPM: ¥500〜¥2,000
- リワード広告 eCPM: ¥1,000〜¥5,000

**設計のポイント**: 広告はユーザーが「自然に目にするタイミング」に配置します。コンテンツの途中に割り込む広告はユーザー離脱の原因になります。

### モデル 2：サブスクリプション（定期課金）

**特徴**: 月額・年額の定期課金で継続収益を得るモデル

**向いているアプリ**:
- コンテンツが継続的に更新されるアプリ（ニュース・学習・フィットネス）
- クラウド同期・バックアップ機能を持つアプリ
- プレミアム機能を段階的に提供するアプリ

**価格帯の目安（日本市場）**:
- 月額: ¥150〜¥600
- 年額: ¥1,200〜¥3,000（月額換算で割安設定）

**設計のポイント**: 無料トライアル（7〜14 日間）と年額プランがコンバージョン率向上に効果的です。

### モデル 3：アプリ内課金（1 回払い）

**特徴**: 特定の機能・コンテンツを一度だけ購入するモデル

**向いているアプリ**:
- プレミアム機能のアンロック（広告除去・追加テーマ等）
- デジタルコンテンツ販売（壁紙・スタンプ・フォント等）
- ゲームのアイテム・レベル購入

**設計のポイント**: 「広告除去 ¥250」のような明確な価値提案が最も購入されやすいです。

### 収益化対応のアプリ構造

Rork に以下のプロンプトを与えると、収益化対応のアプリ骨格を生成できます。

```
壁紙アプリを作ってください。以下の収益化モデルを組み込んでください：

【機能設計】
- 無料ユーザー：壁紙は広告表示後にダウンロード可能
- プレミアムユーザー（月額 ¥200）：広告なし・全カテゴリ解放・高解像度ダウンロード

【AdMob 設定】

- リワード広告：「広告を見て無料ダウンロード」ボタン
```

### RevenueCat の統合

サブスクリプション管理には **RevenueCat** が最もおすすめです。App Store・Google Play 両方に対応しており、Rork で生成したコードに組み込みやすいです。

```typescript
// RevenueCat の初期化（App.tsx）
import Purchases from "react-native-purchases";

export default function App() {
  useEffect(() => {
    // APIキーはプラットフォームで分岐
    const apiKey = Platform.OS === "ios"
      ? "YOUR_REVENUECAT_IOS_KEY"
      : "YOUR_REVENUECAT_ANDROID_KEY";

    Purchases.configure({ apiKey });
  }, []);
}
```

```typescript
// サブスクリプション購入フロー
async function purchaseSubscription() {
  try {
    const offerings = await Purchases.getOfferings();
    const monthly = offerings.current?.monthly;

    if (!monthly) throw new Error("プランが見つかりません");

    const { customerInfo } = await Purchases.purchasePackage(monthly);

    // 購入完了 → プレミアム機能を解放
    if (customerInfo.activeSubscriptions.length > 0) {
      setPremiumUser(true);
    }
  } catch (error) {
    if (!error.userCancelled) {
      Alert.alert("購入エラー", error.message);
    }
  }
}
```

```typescript
// リワード広告（見て無料ダウンロード）
import { RewardedAd, RewardedAdEventType } from "react-native-google-mobile-ads";

async function watchAdAndDownload(wallpaperId: string) {
  const rewarded = RewardedAd.createForAdRequest(
    "ca-app-pub-XXXXXXXXXXXXXXXX/XXXXXXXXXX"
  );

  return new Promise<void>((resolve, reject) => {
    rewarded.addAdEventListener(RewardedAdEventType.LOADED, () => {
      rewarded.show();
    });

      // 広告を閉じた（報酬なし）

    });

  });
```

## AdMob 収益を最大化する設計戦略

### 広告配置のゴールデンルール

効果的な AdMob 広告配置のために、以下の原則を守ってください。

**原則 1：ユーザーの自然な「区切り」に配置する**

広告を表示するベストなタイミングは、ユーザーがタスクを完了した直後です。

- アプリ起動時のスプラッシュ画面後（スタートアップインタースティシャル）
- コンテンツをダウンロード・保存した後
- ゲームのステージクリア後
- 設定変更を適用した後

**原則 2：「5 の法則」でインタースティシャルを制御する**

インタースティシャル（全画面広告）は、5 回に 1 回程度が最適です。頻度が高すぎるとユーザーがアプリを削除します。

```typescript
// インタースティシャルの表示頻度制御
const AD_FREQUENCY = 5; // 5 回に 1 回
let actionCount = 0;

function onUserAction() {
  actionCount++;

  if (actionCount % AD_FREQUENCY === 0) {
    showInterstitialAd();
  }
}
```

**原則 3：リワード広告は「選択肢」として提示する**

「広告を見て無料で使う」vs「課金して広告なしで使う」の選択肢を常に用意します。ユーザーに選択権を与えることで、広告への抵抗感を下げられます。

### eCPM を上げるための施策

eCPM（1,000 インプレッションあたりの収益）を上げるための具体的施策：

- **メディエーション（Mediation）の設定**: Google AdMob のメディエーションで複数の広告ネットワークを競わせると eCPM が 20〜40% 改善することがあります
- **広告ユニットのテスト**: A/B テストで広告フォーマット・配置を最適化する
- **ユーザーセグメント別の広告最適化**: インタクティブな広告が反応しやすいユーザー層を特定する

## サブスクリプションのコンバージョン率を上げる戦略

### ペイウォール設計の 3 原則

**原則 1：無料で十分な価値を提供してから有料を提案する**

「使ってみたら良かったので課金した」という体験設計が最も効果的です。無料で使えることへの満足感が十分に高まったタイミングで、有料プランを提案します。

**原則 2：年額プランを主役にする**

年額プランは月額換算で 30〜50% 割安に設定し、UIで目立たせます。月額に比べてチャーン（解約）率が大幅に低く、LTV（顧客生涯価値）が高くなります。

**原則 3：具体的なメリットを伝える**

「プレミアムにアップグレード」という抽象的な言葉より、「広告が消えて、高解像度ダウンロードができ、全カテゴリの 500 枚の壁紙が使い放題になります」という具体的な説明の方がコンバージョン率が高くなります。

```typescript
// ペイウォール画面のサンプル設計
function PaywallScreen({ onDismiss, onPurchase }) {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>プレミアムプランで体験を向上</Text>

      {/* 具体的なメリットリスト */}
      <View style={styles.benefits}>
        {[
          { icon: "🚫", text: "広告完全除去" },
          { icon: "📱", text: "高解像度（4K）ダウンロード" },

      {/* 年額プランを目立たせる */}
      <PlanCard
        title="年額プラン ¥1,200"

        badge="おすすめ"
        onPress={() => onPurchase("annual")}
        highlighted
      />
      <PlanCard
        title="月額プラン ¥200"
        onPress={() => onPurchase("monthly")}
      />

      {/* 安心感を与えるコピー */}
      <Text style={styles.footer}>
        7日間無料トライアル付き。いつでもキャンセル可能。
      </Text>

      <TouchableOpacity onPress={onDismiss}>
        <Text style={styles.dismissText}>後で試す</Text>
      </TouchableOpacity>
    </View>
  );
}
```

## アナリティクスで収益を継続的に改善する

### 計測すべき KPI

収益を継続的に改善するために追跡すべき指標：

- **DAU（Daily Active Users）**: 日次アクティブユーザー数
- **ARPDAU（Average Revenue Per Daily Active User）**: ユーザー 1 人あたりの日次収益
- **広告 eCPM**: 1,000 インプレッションあたりの収益
- **サブスクリプション転換率**: トライアル開始数 / 有料転換数
- **チャーン率（月次）**: 毎月解約するユーザーの割合
- **LTV（顧客生涯価値）**: 1 ユーザーの生涯収益見込み

### Firebase Analytics との連携

Rork で生成したアプリに Firebase Analytics を組み込む例：

```typescript
import analytics from "@react-native-firebase/analytics";

// 広告視聴イベント
async function logAdView(adType: "banner" | "interstitial" | "rewarded") {
  await analytics().logEvent("ad_impression", {
    ad_type: adType,
    screen: currentScreen,
  });
}

// 課金完了イベント
async function logPurchase(plan: "monthly" | "annual", price: number) {
  await analytics().logPurchase({
    currency: "JPY",
    value: price,
    items: [{ item_id: `subscription_${plan}`, item_name: `${plan} subscription` }],
  });
}
```

## よくあるエラーと対処法

### エラー 1：AdMob の広告が表示されない

**症状**: 本番アプリで広告が一切表示されない

**対処法**:
- テスト広告ユニット ID を本番 ID に変更したか確認する
- `app.json` の `googleMobileAdsAppId` に正しい AdMob アプリ ID を設定する
- App Store / Google Play の審査が完了するまで広告が制限される場合がある（通常 24〜72 時間）

### エラー 2：RevenueCat の購入が反映されない

**症状**: 課金は完了しているが、アプリ内でプレミアム機能が解放されない

**対処法**:
- `Purchases.restorePurchases()` を実装しているか確認する
- App Store / Google Play の課金テスト環境（Sandbox）と本番環境でキー設定が異なる場合がある
- `customerInfo.entitlements.active` の Entitlement ID が RevenueCat ダッシュボードの設定と一致しているか確認する

### エラー 3：サブスクリプションのレビュー却下

**症状**: アプリ審査でサブスクリプション機能が却下される

**対処法**:
- App Store：「特典の説明」「キャンセル方法の説明」をアプリ内・メタデータ双方に記載する
- Apple のサブスクリプションガイドラインに沿って「購入の回復」機能を実装する
- 自動更新サブスクリプションの「利用規約」と「プライバシーポリシー」へのリンクを設ける

## FAQ

**Q1. どの収益モデルが最初に試すべきですか？**

A. 最初は AdMob 広告がもっとも始めやすいです。理由は審査が不要（バナー広告はアプリ公開翌日から表示可能）、初期投資ゼロ、収益が積み上がって仕組みが理解しやすいからです。ある程度の DAU（目安：日本市場なら 1,000 DAU 以上）が得られたら、サブスクリプションを追加することを検討してください。

**Q2. 日本市場で何 DAU あれば月収 10 万円を狙えますか？**

A. 収益は eCPM と広告の視聴頻度によって大きく異なりますが、一般的な目安として、インタースティシャル・リワード広告を適切に設計したアプリで日本市場なら 3,000〜5,000 DAU で月収 10 万円前後が狙えます。ただし、アプリのカテゴリ・広告配置・ユーザー層によって大きく変わります。

**Q3. Rork で作ったアプリにサードパーティ SDK を追加できますか？**

A. はい、可能です。Rork が生成するコードは React Native / Expo ベースなので、標準的な npm パッケージ・Expo プラグインを追加できます。ただし、ネイティブモジュール（AdMob・RevenueCat 等）は EAS Build が必要です。Rork 上で「AdMob と RevenueCat を使った〇〇アプリを作って」と指示すると、EAS Build に対応したプロジェクト構成を生成してくれます。

## まとめ

Rork でアプリを収益化するための核心をまとめます。

- **収益モデルを早期に決める**: 設計段階から収益化の仕組みをアーキテクチャに組み込む
- **AdMob は「区切り」に配置する**: ユーザーの自然なタスク完了タイミングに広告を表示する
- **サブスクリプションは年額を主役にする**: 月額より年額の方が LTV が高く、チャーンが少ない
- **RevenueCat で課金を統合管理する**: iOS/Android 両対応・分析機能付きで最もおすすめ
- **KPI を計測して継続的に改善する**: DAU・ARPDAU・転換率を追跡し、データで意思決定する

個人開発のアプリで収益を上げることは、適切な設計と継続的な改善があれば十分に達成可能です。Rork の力を借りて開発時間を短縮しながら、収益化の仕組みを磨いていきましょう。

アプリ収益化のさらに深い戦略を学びたい方には、[「ゼロから始めるアプリ収益化大全」（Repro 著）](https://www.amazon.co.jp/dp/4815606781/?tag=pinocchio-22)がおすすめです。実際の収益データを交えた実践的な内容で、インディーデベロッパーに役立つ知見が詰まっています。
