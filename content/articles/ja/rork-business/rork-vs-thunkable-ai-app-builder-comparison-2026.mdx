## はじめに — なぜ今、RorkとThunkableを比較するのか

2026年のモバイルアプリ開発は、AIアプリビルダーの台頭によって大きく変わりつつあります。Gartnerの予測では、2026年末までに新規アプリ開発の75%がローコード/ノーコードプラットフォームで構築されるとされています。

そのなかでも注目を集めるのが **Rork** と **Thunkable** の2つのプラットフォームです。どちらもノーコードでモバイルアプリを開発できますが、アーキテクチャやAI活用のアプローチは大きく異なります。この記事では、両プラットフォームの特徴を多角的に比較し、あなたのプロジェクトに最適な選択肢を見つけるお手伝いをします。

## Rorkの概要 — AI駆動のネイティブアプリ開発

Rorkは、自然言語のプロンプトからモバイルアプリを自動生成するAIアプリビルダーです。a16zから280万ドルの資金調達を受け、2026年には月間74万3,000人のユーザーが訪問する急成長プラットフォームとなっています。

Rorkの最大の特徴は、上位プラン **Rork Max** によるネイティブSwiftUIアプリの生成です。Xcodeなしで、クラウド上のMacフリートを活用してSwiftUIアプリをビルドし、わずか2クリックでApp Storeに申請できます。

主な特徴をまとめると以下のとおりです。

- **AIモデル**: Claude Opus 4.6を搭載し、高精度なコード生成を実現
- **対応プラットフォーム**: iOS（SwiftUI ネイティブ）、Android、React Native / Expo
- **Rork Max ネイティブ機能**: Dynamic Island、Live Activities、HealthKit、HomeKit、ARKit、Core ML、NFC、App Clips
- **公開フロー**: 2クリックでApp Store / Google Play に申請可能
- **実機テスト**: Rork Companionアプリで即座にプレビュー
- **料金**: 無料プラン（週5回生成）と Rork Max（$200/月）

## Thunkableの概要 — ビジュアルブロックベースの開発環境

Thunkableは2016年に登場した老舗のノーコードアプリ開発プラットフォームです。MIT App Inventorの影響を受けたブロックベースのビジュアルプログラミング環境が特徴で、プログラミング経験がない方でも直感的にアプリのロジックを構築できます。

主な特徴は次のとおりです。

- **開発方式**: ドラッグ＆ドロップのUIビルダー + ブロック型ロジックエディタ
- **対応プラットフォーム**: iOS、Android（クロスプラットフォーム）
- **AI機能**: 2025年後半からAIアシスタント機能を追加（テキスト生成・画像認識など外部API連携）
- **テンプレート**: 豊富なスターターテンプレートを提供
- **料金**: 無料プラン、Starter（$13/月）、Pro（$38/月）、Business（$99/月）
- **教育向け**: 学校やワークショップでの利用実績が豊富

## 開発体験の比較 — AIプロンプト vs ブロックプログラミング

Rorkの開発体験は**対話型**です。自然言語でアプリの要件を伝えると、AIがUI・ロジック・バックエンド接続までを一括生成します。たとえば「タスク管理アプリを作りたい。カテゴリ分け、期限設定、通知機能が欲しい」と伝えるだけで、動作するアプリのプロトタイプが数分で完成します。

```bash
# Rork でのアプリ生成例（CLIからの操作イメージ）
rork create "家計簿アプリ。月ごとの支出グラフ、カテゴリ別集計、CSV出力機能付き"
# → 数分でReact Native / Expoプロジェクトが自動生成される
# → Rork Companionで即座にスマートフォン上でプレビュー可能
```

一方、Thunkableでは**ビジュアルブロック**を組み合わせてロジックを構築します。条件分岐やループをパズルのように組み立てていく方式は、プログラミングの基礎概念を学びながら開発したい方に適しています。ただし、複雑なロジックになるとブロックの数が膨大になり、管理が難しくなることがあります。

開発スピードの面では、Rorkが圧倒的に速いです。AIが要件を解釈してコードを自動生成するため、プロトタイプの完成まで数分〜数十分で済みます。Thunkableの場合、UIの配置とロジックのブロック構築を手作業で行うため、同程度のアプリで数時間〜数日かかることが一般的です。

## 対応プラットフォームとネイティブ機能

プラットフォーム対応の幅と深さは、両者の大きな差別化ポイントです。

**Rork（特にRork Max）のネイティブ対応:**

- iOS: SwiftUIネイティブ（Dynamic Island、Live Activities、Widgets、Siri Intents対応）
- Apple Watch / Apple TV / Vision Pro にも対応
- Android: React Native / Expo ベース
- ハードウェアAPI: ARKit、LiDAR、Metal 3D、NFC、Core ML、HealthKit、HomeKit

**Thunkableのクロスプラットフォーム対応:**

- iOS / Android: 単一コードベースからのクロスビルド
- Webアプリとしての公開にも一部対応
- ハードウェアAPI: カメラ、GPS、加速度センサーなど基本的なAPIに対応
- ネイティブ機能（Dynamic IslandやWidgetなど）は非対応

Rork Max ならではのネイティブ機能統合は、特に iOS アプリで差が顕著です。Dynamic Islandでリアルタイム情報を表示したり、HealthKitと連携してヘルスケアデータを扱うアプリを作りたい場合、Rork Maxが唯一の選択肢になります。

一方、「iOSとAndroidで同じ見た目のシンプルなアプリを素早く作りたい」というニーズには、Thunkableのクロスプラットフォームアプローチも有効です。

## 料金プランの比較

コスト面は、プロジェクトの規模と予算によって最適な選択が変わります。

**Rork の料金体系:**

- **Free**: 週5回のアプリ生成、React Native / Expo ベース
- **Rork Max**: $200/月 — ネイティブSwiftUI生成、クラウドMacビルド、2クリックApp Store公開、全ネイティブAPI対応

**Thunkable の料金体系:**

- **Free**: 1プロジェクト、Thunkableブランディング付き
- **Starter**: $13/月 — 無制限プロジェクト、ブランディング削除
- **Pro**: $38/月 — カスタムフォント、AdMob連携、プライベートプロジェクト
- **Business**: $99/月 — チーム機能、優先サポート、API統合

月額費用だけを見ると Thunkable のほうが安価ですが、「何ができるか」を考慮すると判断が変わります。Rork Max はネイティブアプリのビルド・申請・公開までワンストップで完結するため、別途 Apple Developer Program（$99/年）と Mac（Xcode用）を用意する必要がありません。クラウドMacフリートによるビルドが含まれているため、Macを持っていない開発者にとっては実質的なコスト差はさらに縮まります。

## AI機能の深さと精度

AI活用のアプローチは両者で根本的に異なります。

Rorkは**AI-first**の設計思想で、アプリ開発プロセス全体にAIが統合されています。Claude Opus 4.6を搭載し、要件定義からコード生成、デバッグ、最適化まで一貫してAIが支援します。生成されるコードは実際のSwiftUIやReact Nativeのコードであり、エクスポートして手動で編集することも可能です。

```typescript
// Rork が生成するコードの例（React Native / Expo）
import { useState, useEffect } from 'react';
import { View, Text, FlatList } from 'react-native';
import AsyncStorage from '@react-native-async-storage/async-storage';

// Rork AIが生成した家計簿のカテゴリ別集計コンポーネント
export default function ExpenseSummary() {
  const [expenses, setExpenses] = useState([]);
  const [summary, setSummary] = useState({});

  useEffect(() => {
    loadExpenses();
  }, []);

  const loadExpenses = async () => {
    const data = await AsyncStorage.getItem('expenses');
    if (data) {
      const parsed = JSON.parse(data);
      setExpenses(parsed);
      // カテゴリ別に自動集計
      const grouped = parsed.reduce((acc, item) => {
        acc[item.category] = (acc[item.category] || 0) + item.amount;
        return acc;
      }, {});
      setSummary(grouped);
    }
  };

  return (
    <View style={{ padding: 16 }}>
      <Text style={{ fontSize: 20, fontWeight: 'bold' }}>
        カテゴリ別支出
      </Text>
      {Object.entries(summary).map(([category, total]) => (
        <View key={category} style={{ flexDirection: 'row', justifyContent: 'space-between', paddingVertical: 8 }}>
          <Text>{category}</Text>
          <Text>¥{total.toLocaleString()}</Text>
        </View>
      ))}
    </View>
  );
}
```

Thunkableは従来のビジュアル開発にAI機能を**後付け**で追加するアプローチです。OpenAIやGoogle Cloud Vision などの外部APIと接続するブロックが提供されており、テキスト生成や画像認識をアプリに組み込めます。ただし、AI がアプリのロジックや UI を自動生成するわけではなく、AI はアプリの「機能の一部」として利用する形です。

## どちらを選ぶべきか — ユースケース別の推奨

**Rork がおすすめのケース:**

- 本格的なiOSネイティブアプリを開発したい方
- プログラミング経験は浅いが、プロ品質のアプリを素早く形にしたい方
- Dynamic Island、HealthKit、ARKit などのiOS固有機能を活用したい方
- アプリを App Store / Google Play に実際に公開して収益化を目指す方
- AIを活用して開発スピードを最大化したい個人開発者やスタートアップ

**Thunkable がおすすめのケース:**

- プログラミングの基礎概念を学びながらアプリを作りたい学生や初心者
- 教育現場でのプログラミング教材として活用したい教育者
- シンプルなクロスプラットフォームアプリを低コストで作りたい方
- ビジュアルブロックベースの開発環境が好みの方

アプリ開発のビジネス活用を考えている方には、Rorkの方が将来的な拡張性と収益化の可能性で優れています。Rorkでのアプリ開発と収益化の全体像については、[Rorkアプリ収益化の完全ガイド](/articles/rork-business/rork-app-monetization-complete-guide) で体系的に解説しています。

## よくある質問（FAQ）

### Q1: プログラミング未経験でも Rork でアプリを作れますか？

はい、Rorkは自然言語でアプリの要件を伝えるだけでコードを自動生成するため、プログラミング未経験の方でもアプリを作成できます。Rork Companionアプリを使えば、生成したアプリを即座にスマートフォンで確認できます。初めてのアプリ開発には [Rork 入門ガイド](/articles/rork-basics/getting-started) が参考になります。

### Q2: Thunkable で作ったアプリを Rork に移行できますか？

直接的なインポート機能はありませんが、Thunkableで培ったアプリの設計思想やUI構成を、Rorkのプロンプトとして再入力することで再構築できます。Rorkの方がネイティブ機能への対応が広いため、移行によって機能面でのグレードアップが期待できます。

### Q3: 2026年の時点で、AIアプリビルダーとしてはどちらが先進的ですか？

AI統合の深さという観点では、Rorkが先進的です。Rorkは開発プロセス全体にAI（Claude Opus 4.6）が組み込まれており、要件定義からコード生成・デバッグ・最適化まで一貫してAI支援を受けられます。Thunkableは従来のビジュアル開発にAI APIを連携する形であり、開発プロセス自体のAI化はRorkほど進んでいません。

### Q4: チーム開発にはどちらが向いていますか？

Thunkable Businessプランにはチーム機能が含まれており、複数人での共同開発に対応しています。Rorkは現時点では個人開発者向けの機能が中心ですが、GitHubとの連携により、生成されたコードをチームで管理・編集することは可能です。

## まとめ

Rork と Thunkable は、どちらもモバイルアプリ開発を民主化するプラットフォームですが、そのアプローチは大きく異なります。

Rorkは **AI-first** のアプローチで、自然言語からネイティブアプリを直接生成し、2クリックでストア公開まで完結します。特にRork Maxのネイティブ SwiftUI 対応は、従来のノーコードツールでは実現できなかった品質のiOSアプリを可能にしています。

Thunkableは**ビジュアルブロック**ベースの教育的アプローチで、プログラミングの概念を学びながら開発できる点に強みがあります。コスト面でも手頃で、教育やプロトタイピングに適しています。

2026年のAIアプリ開発の潮流を考えると、実用的なアプリを素早く形にし、ストアに公開して収益化を目指すなら Rork が有力な選択肢です。一方、プログラミング教育やビジュアル開発を重視するなら Thunkable も検討に値します。本記事のテーマをさらに深めたい方には「[プロダクトマネジメントのすべて（及川卓也 著）](https://www.amazon.co.jp/dp/4798166391/?tag=pinocchio-22)」がおすすめです。
