## はじめに

Unity でモバイルゲームを開発しました。さて、公開します。でも、開発にかかった時間と労力を考えると、完全に無料で公開するのは心が痛みます。でも、どうやってお金を稼ぐのか？多くの個人開発者がこの問題に直面しています。

モバイルゲームの収益化方法は複数あり、それぞれメリット・デメリットがあります。本記事では、最も一般的な3つの方法を解説します：

1. **AdMob 広告**：ゲーム内に広告を表示し、表示・クリック時に報酬を得る
2. **アプリ内課金（IAP）**：プレイヤーがゲーム内でアイテムやコンテンツを購入
3. **サブスクリプション**：月額または年額で定期的な収益を得る

それぞれの特徴、導入難易度、期待収益を理解することで、自分のゲームに最適な収益化戦略を選択できます。

## 収益化の3つの方法の比較表

| 方法 | 導入難易度 | 初期収益 | スケーリング | ユーザー満足度 | 推奨対象 |
|------|---------|---------|------------|-----------|---------|
| **AdMob 広告** | 低い | 低い | 高い | 中程度 | 大量ユーザー獲得型 |
| **アプリ内課金（IAP）** | 中程度 | 中程度 | 高い | 低い | コア層強化型 |
| **サブスクリプション** | 高い | 低い（初期） | 中程度 | 低い | 継続プレイ向け |

理想的には、これら3つを組み合わせることで、複数の収入源から安定した収益を得られます。

## 方法 1: AdMob 広告による収益化

AdMob は Google が提供するモバイル広告プラットフォームです。ゲーム内に広告スペースを配置し、ユーザーが広告を閲覧・クリックするたびに報酬を得られます。

### AdMob の基本仕組み

```
ユーザーがゲームをプレイ
          ↓
広告スペース（バナー・インタースティシャル）を表示
          ↓
ユーザーが広告を閲覧またはクリック
          ↓
Google が広告主に課金
          ↓
報酬がパブリッシャー（あなた）に支払われる
```

### AdMob セットアップの手順

**ステップ 1: Google AdMob アカウント作成**

1. [AdMob](https://admob.google.com/) にアクセス
2. Google アカウントでサインイン（またはアカウント作成）
3.「新しいアプリを追加」からプラットフォーム「Android」または「iOS」を選択

**ステップ 2: Unity での実装**

Google Mobile Ads SDK をインポート：

```
Window → TextMesh Pro → Import TMP Essential Resources
```

次に、Google Mobile Ads SDK for Unity を [GitHub](https://github.com/googleads/google-mobile-ads-sdk-unity) からダウンロード、またはパッケージマネージャーから導入：

```
Window → Package Manager → Add package from git URL
https://github.com/googleads/google-mobile-ads-sdk-unity.git
```

**ステップ 3: C# スクリプトでの実装**

```csharp
using UnityEngine;
using GoogleMobileAds.Client;

public class AdManager : MonoBehaviour
{
    public static AdManager Instance { get; private set; }

    private BannerView bannerView;
    private InterstitialAd interstitialAd;

    private const string BANNER_AD_ID = "ca-app-pub-xxxxxxxxxxxxxxxx/yyyyyyyyyyyyy";  // Android
    private const string INTERSTITIAL_AD_ID = "ca-app-pub-xxxxxxxxxxxxxxxx/yyyyyyyyyyyyy";

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }

        Instance = this;
        DontDestroyOnLoad(gameObject);
    }

    private void Start()
    {
        // Google Mobile Ads SDK を初期化
        MobileAds.Initialize(initStatus => { });

        CreateBannerView();
        LoadInterstitialAd();
    }

    private void CreateBannerView()
    {
        bannerView = new BannerView(BANNER_AD_ID, AdSize.Banner, AdPosition.Bottom);
        bannerView.OnAdLoaded += OnBannerLoaded;
        bannerView.OnAdFailedToLoad += OnBannerFailedToLoad;

        var adRequest = new AdRequest();
        bannerView.LoadAd(adRequest);
    }

    private void OnBannerLoaded()
    {
        Debug.Log("Banner ad loaded successfully");
    }

    private void OnBannerFailedToLoad(LoadAdError error)
    {
        Debug.LogError($"Banner failed to load: {error}");
    }

    public void LoadInterstitialAd()
    {
        var adRequest = new AdRequest();

        InterstitialAd.Load(INTERSTITIAL_AD_ID, adRequest, OnInterstitialLoaded);
    }

    private void OnInterstitialLoaded(InterstitialAd ad, LoadAdError error)
    {
        if (error != null || ad == null)
        {
            Debug.LogError("Interstitial failed to load");
            return;
        }

        interstitialAd = ad;
        interstitialAd.OnAdFullScreenContentClosed += OnInterstitialClosed;
    }

    public void ShowInterstitialAd()
    {
        if (interstitialAd != null && interstitialAd.CanShowAd())
        {
            interstitialAd.Show();
        }
        else
        {
            Debug.LogWarning("Interstitial ad is not ready");
        }
    }

    private void OnInterstitialClosed()
    {
        Debug.Log("Interstitial ad closed");
        LoadInterstitialAd();  // 次の広告をプリロード
    }
}
```

### AdMob 広告の適切な配置

AdMob による収益を最大化するには、広告配置が重要です：

- **バナー広告**：画面下部に常時表示。クリックスルーレートは 0.1～0.5% 程度
- **インタースティシャル広告**：ゲームステージ完了時、ゲームオーバー時など、自然な中断点で表示。クリックスルーレートは 1～3%
- **リワード広告**：「広告を見たらコイン 100 枚獲得」など、ユーザーに見返りを提供。エンゲージメントが高い

**例：ゲームオーバー画面での広告表示**

```csharp
public class GameOverPanel : MonoBehaviour
{
    public void OnGameOver()
    {
        // ゲームオーバー 2 秒後に広告を表示
        Invoke(nameof(ShowAd), 2f);
    }

    private void ShowAd()
    {
        AdManager.Instance.ShowInterstitialAd();
    }
}
```

### AdMob 推定収益

- **CPM（1000 インプレッション当たりの報酬）**：$2～$10（ゲーム、地域により変動）
- **CPC（クリック当たりの報酬）**：$0.01～$0.50

例：1 日 10,000 ユーザーがアクティブで、各ユーザーが 5 本の広告を見る場合

```
50,000 インプレッション × $5 / 1000 = $250 / 日
= 約 $7,500 / 月
```

ただし、この数字は理想的なケースです。実際の収益は ER（ユーザー保持率）、広告のクリック率、地域などに大きく左右されます。

## 方法 2: アプリ内課金（IAP）による収益化

アプリ内課金は、ゲーム内でプレイヤーがお金を支払い、アイテムやコンテンツを購入する仕組みです。広告よりも単価が高く、より多くの収益が期待できます。

### IAP の基本仕組み

```
ユーザーが購入ボタンをクリック
          ↓
IAP システムが Google Play Store / App Store と連携
          ↓
ユーザーが決済情報を入力
          ↓
プラットフォームが課金処理
          ↓
パブリッシャー（あなた）が 70% の手数料を受け取る
```

### IAP 対応パッケージの導入

Unity IAP は Purchasing パッケージで提供されています：

```
Window → TextMesh Pro → Import TMP Essential Resources
Window → Package Manager → Unity Purchasing
```

### C# スクリプトでの実装

```csharp
using UnityEngine;
using UnityEngine.Purchasing;

public class IAPManager : MonoBehaviour, IStoreListener
{
    public static IAPManager Instance { get; private set; }

    private IStoreController storeController;
    private IExtensionProvider extensionProvider;

    // 製品 ID（Google Play Store / App Store に登録したもの）
    private const string GOLD_100_PRODUCT_ID = "com.example.game.gold100";
    private const string GOLD_500_PRODUCT_ID = "com.example.game.gold500";
    private const string REMOVE_ADS_PRODUCT_ID = "com.example.game.noads";
    private const string SUBSCRIPTION_PRODUCT_ID = "com.example.game.premium_monthly";

    private void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }

        Instance = this;
        DontDestroyOnLoad(gameObject);

        InitializeIAP();
    }

    private void InitializeIAP()
    {
        var builder = ConfigurationBuilder.Instance(StandardPricingStrategy.Instance());

        // 消費可能な商品（購入するたびに消える）
        builder.AddProduct(GOLD_100_PRODUCT_ID, ProductType.Consumable);
        builder.AddProduct(GOLD_500_PRODUCT_ID, ProductType.Consumable);

        // 非消費可能な商品（1 回のみ購入）
        builder.AddProduct(REMOVE_ADS_PRODUCT_ID, ProductType.NonConsumable);

        // サブスクリプション
        builder.AddProduct(SUBSCRIPTION_PRODUCT_ID, ProductType.Subscription);

        UnityPurchasing.Initialize(this, builder);
    }

    public void OnInitialized(IStoreController controller, IExtensionProvider extensions)
    {
        storeController = controller;
        extensionProvider = extensions;
        Debug.Log("IAP Initialized successfully");
    }

    public void OnInitializationFailed(InitializationFailureReason error)
    {
        Debug.LogError($"IAP Initialization failed: {error}");
    }

    public PurchaseProcessingResult ProcessPurchase(PurchaseEventArgs purchaseEvent)
    {
        string productID = purchaseEvent.purchasedProduct.definition.id;

        switch (productID)
        {
            case GOLD_100_PRODUCT_ID:
                AddGold(100);
                break;

            case GOLD_500_PRODUCT_ID:
                AddGold(500);
                break;

            case REMOVE_ADS_PRODUCT_ID:
                RemoveAds();
                PlayerPrefs.SetInt("AdsRemoved", 1);
                break;

            case SUBSCRIPTION_PRODUCT_ID:
                EnablePremiumFeatures();
                break;
        }

        return PurchaseProcessingResult.Complete;
    }

    public void OnPurchaseFailed(Product product, PurchaseFailureReason failureReason)
    {
        Debug.LogError($"Purchase failed: {product.definition.id} - {failureReason}");
    }

    public void BuyGold(string productID)
    {
        storeController.InitiatePurchase(productID);
    }

    private void AddGold(int amount)
    {
        int currentGold = PlayerPrefs.GetInt("Gold", 0);
        PlayerPrefs.SetInt("Gold", currentGold + amount);
        Debug.Log($"Gold added: {amount}. Total: {currentGold + amount}");
    }

    private void RemoveAds()
    {
        Debug.Log("Ads removed");
    }

    private void EnablePremiumFeatures()
    {
        Debug.Log("Premium features enabled");
    }
}
```

### IAP の価格設定戦略

効果的な IAP 価格設定は、ゲームの種類とターゲットプレイヤーに依存します：

**カジュアルゲーム向け**
- $0.99：小額アイテム（スタミナ回復など）
- $4.99：中量アイテム（デイリーボーナス）
- $9.99：大量アイテム（シーズンパス）

**ハイアルゲーム向け**
- $19.99～$99.99：プレミアムコンテンツ

**推奨ピラミッド構造**
- 90%：$0.99～$4.99 の軽い購入
- 8%：$9.99～$19.99 の中程度購入
- 2%：$99.99 以上の重課金

## 方法 3: サブスクリプション（定期課金）

サブスクリプションは、毎月または毎年定期的に課金される仕組みです。初期段階では収益が少なくても、継続率が高ければ安定した収益源になります。

### サブスクリプション実装例

```csharp
public class SubscriptionManager : MonoBehaviour
{
    public bool IsPremiumActive
    {
        get
        {
            if (!PlayerPrefs.HasKey("SubscriptionExpireDate"))
                return false;

            long expireTimestamp = long.Parse(PlayerPrefs.GetString("SubscriptionExpireDate"));
            return System.DateTime.Now.Ticks < expireTimestamp;
        }
    }

    public void OnSubscriptionPurchased()
    {
        // 購入日から 30 日後を有効期限として設定
        var expireDate = System.DateTime.Now.AddDays(30);
        PlayerPrefs.SetString("SubscriptionExpireDate", expireDate.Ticks.ToString());

        EnablePremiumFeatures();
    }

    private void EnablePremiumFeatures()
    {
        // プレミアム機能を有効化
        // 広告削除、プレミアムコンテンツアンロック など
    }
}
```

### 有効期限切れ時の処理

```csharp
private void Start()
{
    if (IsPremiumActive)
    {
        EnablePremiumFeatures();
    }
    else if (PlayerPrefs.HasKey("SubscriptionExpireDate"))
    {
        // 有効期限が切れた場合、再購入を促す
        ShowResubscribePrompt();
    }
}

private void ShowResubscribePrompt()
{
    var panel = Instantiate(resubscribePromptPrefab);
    var button = panel.GetComponentInChildren<Button>();
    button.onClick.AddListener(() => IAPManager.Instance.BuyGold("com.example.game.premium_monthly"));
}
```

## 複数の収益化方法の組み合わせ戦略

理想的には、3 つの方法を組み合わせることで、複数の収入源から安定した収益を得られます：

### シナリオ 1: カジュアルゲーム（ハイユーザー数）

```
60% AdMob 広告
30% アプリ内課金（軽い消費アイテム）
10% サブスクリプション（広告なしオプション）
```

### シナリオ 2: コアゲーマー向けゲーム（低ユーザー数、高 ARPPU）

```
20% AdMob 広告（オプション）
70% アプリ内課金（重課金）
10% サブスクリプション（プレミアムパス）
```

### 実装上の注意：ユーザー体験の損傷を避ける

複数の収益化方法を組み合わせる際、過度な課金推奨や広告の多さはユーザーを不快にさせ、アンインストール増加につながります。

**推奨事項**：
- 広告表示の頻度は 10～15 分ごとに 1 回程度
- 強制広告ではなく、リワード広告を中心に
- IAP 推奨は 1 日 1～2 回まで
- サブスクリプションのメリットを明確に表示

## Rork Max との組み合わせ方

Rork Max は、ゲーム配信プラットフォームです。Unity ゲーム開発者は、Rork Max を通じてゲームを配信し、追加の収益化チャネルを確保できます。

### Rork Max の利点

1. **複数プラットフォームへの一括配信**：iOS、Android、Web など複数プラットフォームに対応
2. **分析ツール内蔵**：プレイヤーのDAU、ARPPU、リテンション率などを可視化
3. **分散収益**：AdMob、IAP 以外にも、Rork Max 経由での追加報酬が可能
4. **プロモーション機能**：Rork 内でゲームをフィーチャーしてもらい、発見性向上

### Rork Max 配信の流れ

```
Unity ゲーム開発完了
          ↓
Rork Max にゲームを申請
          ↓
審査（1～3 週間）
          ↓
承認・配信開始
          ↓
Rork Max 内でプロモーション
          ↓
追加ユーザー獲得
          ↓
AdMob・IAP・サブスク での収益増加
```

## 月間収益予測シミュレーション

実際の収益がどの程度になるか、シミュレーションしてみましょう。

### 仮定

- **DAU（日次アクティブユーザー）**: 10,000
- **セッション長**：15 分
- **広告表示回数**：セッションあたり 2～3 回
- **IAP 購入率**：DAU の 2%（月間）
- **平均購入額**：$2
- **サブスク購読率**：DAU の 0.5%

### 計算

**AdMob 収益**
```
10,000 DAU × 2.5 広告 / セッション × 30 日 = 750,000 インプレッション
750,000 × $5 CPM / 1000 = $3,750 / 月
```

**IAP 収益**
```
10,000 DAU × 2% × $2 平均 × 30 日 = $12,000 / 月
```

**サブスク収益**
```
10,000 DAU × 0.5% × $4.99 / 月 = $2,495 / 月
```

**合計月間収益**：$18,245

ただし、この数字はあくまで理想的なケースです。実際の収益は ゲームの質、ジャンル、マーケティング施策により大きく変動します。

## 収益化導入時のよくある質問

### Q1: 広告の表示が多すぎるとアンインストール増加しないか？

A: はい、過度な広告はユーザーの不満につながります。推奨は 10～15 分ごと。ただし、リワード広告（見ると報酬）なら許容度が高いです。

### Q2: IAP をすぐに実装すべき？

A: 最初は AdMob のみ導入し、ある程度ユーザーが集まってから IAP を追加するのがおすすめです。IAP は購買意欲のあるコアプレイヤーに対して機能するため。

### Q3: サブスクは初期段階で必要？

A: 初期段階は不要です。ゲームが安定し、継続プレイヤーが多くなってから導入するのが効果的です。

### Q4: Google Play Store・App Store の手数料はどうなる？

A: Google Play Store、App Store ともに 30% の手数料を取られます。つまり、$100 の売上は $70 の実収益になります。

## 収益化の倫理的配慮

ゲームの収益化は重要ですが、ユーザー体験を損傷させるべきではありません。以下を心がけてください：

- **透明性**：何にお金がかかるのか、明確に表示する
- **選択肢**：課金なしでもゲームが楽しめるようにする
- **フェアネス**：課金ユーザーが圧倒的に有利になるバランスは避ける

## まとめ

Unity モバイルゲームの収益化には、3 つの主要な方法があります：

1. **AdMob 広告**：低難度で実装でき、高ユーザー数で安定収益が期待できる
2. **アプリ内課金（IAP）**：単価が高く、コアプレイヤーからの収益源として有効
3. **サブスクリプション**：継続プレイヤーから安定した月間収益を得られる

これら 3 つを自分のゲーム特性に応じて組み合わせることで、複数の収入源から安定した収益を実現できます。

加えて、Rork Max などの配信プラットフォームを活用することで、さらなるユーザー獲得と収益化の機会を広げることが可能です。

重要なのは、ユーザー体験を損なわない範囲内で、バランスの取れた収益化戦略を構築することです。
