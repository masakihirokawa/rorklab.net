## TestFlight配信からApp Store公開までの道のり

Rorkで生成したアプリを世界中のユーザーに届けるには、Apple の厳密なレビュープロセスを通す必要があります。TestFlight でのベータテストから App Store 公開まで、多くの開発者がトラブルに遭遇します。

この記事では、その過程で起きやすい問題と、確実な対処方法を初心者向けに解説します。もし Xcode でビルドエラーが発生している場合は、[Rorkアプリのビルドエラー完全解決ガイド](/articles/rork-dev/rork-app-build-error-swiftui-fix-guide)も参考にしてください。

## トラブル1：プロビジョニングプロファイルエラー

### 症状

「Signing for Identifier failed」「Provisioning Profile not found」といったエラーが App Store Connect でアップロード時に出ます。

### 原因

プロビジョニングプロファイルが以下の理由で無効になっています：

- App ID が存在しない、または削除されている
- 証明書（Signing Certificate）が失効している（年1回更新が必要）
- デバイス登録が古い、または UDID が削除されている

### 解決方法

**ステップ1：プロビジョニングプロファイルを再生成する**

1. Apple Developer Center（developer.apple.com）にログイン
2. 「Certificates, IDs & Profiles」に移動
3. 「Profiles」セクションで、アプリのプロフィールを選択し削除
4. 「+ ボタン」で新規作成
5. 「iOS App Distribution」を選択（App Store 配信用）
6. アプリの Bundle ID を指定
7. ダウンロード後、Xcode にドラッグ＆ドロップ

**ステップ2：Xcode でプロビジョニングプロファイルを確認**

Xcode で「Project」→「Build Settings」を開き、以下を確認：

```
Signing & Capabilities:
  Team: あなたのAppleチーム
  Provisioning Profile: 自動選択（または上記で生成したプロフィール）
```

「Automatically manage signing」がオンになっていることが重要です。

## トラブル2：署名証明書（Signing Certificate）の期限切れ

### 症状

「Certificate has expired」といったエラーが Xcode に表示されます。

### 原因

Apple の署名証明書の有効期限は1年です。期限を過ぎると、ビルド時に署名できなくなります。

### 解決方法

**新しい証明書を生成する**

1. Developer Center で「Certificates」セクションに移動
2. 期限切れの証明書を削除
3. 「+ ボタン」で新規作成
4. 「Apple Development」（開発用）または「Apple Distribution」（配信用）を選択
5. CSR（Certificate Signing Request）をアップロード
6. ダウンロード後、Keychain Access にドラッグ＆ドロップ

**Keychain に証明書を追加する**

```bash
# ダウンロードした .cer ファイルをダブルクリック
# または
open ~/Downloads/certificate.cer
```

この後、Xcode は自動的に新しい証明書を認識します。

## トラブル3：App Store Connect 申請が拒否される

### 症状

TestFlight でテスト配信はできたが、App Store 申請が拒否（Rejection）された。理由メールが届く。

### 一般的な拒否理由と対処法

| 拒否理由 | 対処方法 |
|---|---|
| ビルドが古い、または不完全 | 最新バージョンで再ビルド＆再アップロード |
| プライバシーポリシーがない、または要件不足 | プライバシーポリシーを記載し、App Store Connect で URL を設定 |
| ハードウェアやフレームワークの依存性が不明確 | 「App Privacy」でデータ収集項目を正確に入力 |
| テスト用アカウント情報がない、または不正確 | テスト用の有効なメール＆パスワードを提供（App Store Connect の「Test Information」に記入） |
| アプリ内課金の実装が不正 | StoreKit 2 を正しく実装、価格設定を「Pricing and Availability」で確認 |

**プライバシーポリシーの作成（推奨テンプレート）**

```
プライバシーポリシー

このアプリは以下の情報を収集しません：
- 個人情報（名前、メールアドレス等）
- 位置情報
- 健康データ
- 連絡先

このアプリが収集する情報：
- 匿名の使用統計（Amplitude、Google Analytics等）
```

## トラブル4：App Store スクリーンショット要件

### 症状

「Screenshots do not show the app in use」と審査で指摘される。スクリーンショットの要件が不明確。

### スクリーンショットの要件

Apple は以下の基準でスクリーンショットを評価します：

- **言語を統一する**：日本語アプリなら全スクリーンショットを日本語で表示
- **実機またはシミュレータで撮影**：テンプレートやモックアップではなく、実際に動作するアプリのスクショ
- **ユーザーに有益な機能を表示**：メイン画面、チュートリアル、コア機能を順に見せる
- **テキストは簡潔に**：1スクショあたり 3〜5 行程度のキャプション

### スクリーンショット撮影の手順（Xcode）

```bash
# Xcode シミュレータで自動スクショ撮影
# 1. Product → Scheme → Edit Scheme で「Run」を選択
# 2. 「Pre-actions」で以下スクリプトを追加：

osascript -e '
tell application "System Events"
  key code 19 using {command down, shift down}  -- Cmd+Shift+5
end tell
'

# 3. アプリを Run すると自動でスクショが撮影される
```

または、シミュレータの Screenshot ボタン（Cmd+S）で手動撮影。

## トラブル5：ビルドが old / invalid

### 症状

「This build is old or has become invalid」と App Store Connect に表示される。

### 原因

- ビルドをアップロード後、90 日以上 App Store Connect に提出していない
- ビルドで使用された証明書が失効している

### 解決方法

新しいビルドを再生成してアップロードします：

```bash
# Xcode で Product → Archive
# または
xcodebuild archive ...
```

その後 App Store Connect にアップロード（Organizer から「Upload to App Store Connect」）。

## トラブル6：TestFlight 内部テストと外部テスト

### 症状

「Why can't I invite external testers?」テスター招待ができない。

### 原因

- ビルドが TestFlight の処理中（24 時間程度かかる場合がある）
- 外部テスト用ビルドが承認されていない
- テスター用メールアドレスが既に他の Apple ID と紐付いている

### 解決方法

**内部テストの場合**

1. TestFlight → Internal Testing タブ
2. テスター（チームメンバー）を追加（既に有効な Apple ID が必要）

**外部テストの場合**

1. TestFlight → External Testing タブ
2. ビルドが「Ready to Test」になるまで待つ
3. テスター数は最大 10,000 人
4. テスター用リンク（公開 URL）を生成、共有

ビルド承認が遅い場合は、Apple に連絡します：

```
App Store Connect → My Apps → TestFlight → External Testing → Help
```

## よくある質問（FAQ）

### Q1：App Store 申請に何日かかりますか？

**A：** 通常 1〜3 営業日です。ただし、拒否を受けると対応時間が追加されます。初回申請は事前に十分なテスト期間を確保しておくことをお勧めします。

### Q2：リジェクト後、どのくらい早く再申請できますか？

**A：** 拒否メールへの対応後、すぐに同じビルドを再提出するか、新ビルドをアップロードして再申請できます。ただし、最初の審査結果から 30 日以内に対応しないと、申請は自動的に無効になります。

### Q3：中国（App Store China）に公開する場合、特別な手続きが必要ですか？

**A：** はい。中国向けアプリは ICP 備案番号（Internet Content Provider registration）が必須です。また、プライバシーポリシーやデータセンター所在地に関する厳しい規制があります。専門家のサポートが推奨されます。

## まとめ

TestFlight から App Store 公開までのプロセスは、手順が多く一見複雑に見えます。ですが、要点は証明書と プロビジョニングプロファイルの有効性、プライバシー情報の正確性、スクリーンショットの品質です。

これらを事前にチェックすれば、申請拒否を大幅に減らせます。万が一拒否されても、Apple の指摘事項は具体的で対応しやすいものがほとんどです。
