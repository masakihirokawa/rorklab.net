## はじめに：Rork で SaaS ができる時代

「SaaS（Software as a Service）」といえば、以前は大規模なエンジニアチームが数ヶ月かけて構築するものでしました。しかし今、**Rork と AI の組み合わせにより、個人開発者でも月額課金型の SaaS アプリを短期間で立ち上げられる**時代になっています。

この記事では、SaaS アプリの基本構成から、Rork を使った実装手順、App Store への公開と収益化まで、初心者でもステップバイステップで理解できるよう丁寧に解説します。「副業でアプリを作りたい」「ノーコードで起業したい」という方にとって、実践的なロードマップになるはずです。

この記事で学べること：

- SaaS アプリの基本的なアーキテクチャ
- Rork で認証・データベース・サブスク決済を組み合わせる方法

## SaaS アプリとは何か？モバイル SaaS の可能性

SaaS とは、ユーザーがアプリを「買い切り」ではなく**月額・年額のサブスクリプション形式で利用する**ソフトウェアサービスです。代表例としては Notion、Figma、Spotify などが挙げられます。

モバイルアプリにおける SaaS は、以下のような特徴があります：

- **継続収益**：一度ユーザーを獲得すれば、毎月安定した収益が入る
- **スケーラビリティ**：追加コストなしにユーザー数を増やせる
- **アップデート価値**：定期的な新機能追加でチャーン（解約）を防げる

Rork で作れるモバイル SaaS の例：

- タスク管理・プロジェクト管理アプリ（月額 ¥300〜¥1,000）
- 語学学習・資格勉強サポートアプリ
- ヘルスケア・フィットネストラッキングアプリ
- 家計管理・支出記録アプリ
- フリーランス向け請求書・見積もりアプリ

**なぜ Rork が SaaS 開発に向いているのか：** Rork は Supabase や Stripe とのネイティブ連携が強く、認証・データベース・決済の三点セットをプロンプトベースで素早く実装できます。従来なら数週間かかっていたバックエンド構築が、数時間に短縮されることも珍しくありません。

## SaaS アプリの基本アーキテクチャ

Rork で構築するモバイル SaaS の一般的な構成は以下の通りです：

```
[フロントエンド: Rork（React Native / Swift）]
        ↓
[認証: Supabase Auth / Firebase Auth]
        ↓
[データベース: Supabase（PostgreSQL）]
        ↓
[課金: Stripe / RevenueCat / StoreKit 2]
        ↓
[バックエンド API: Supabase Edge Functions]
```

### 認証（Authentication）

ユーザーアカウントの管理はすべての SaaS の基盤です。Rork では Supabase Auth を使ったメール認証・Google ログイン・Apple ログインを簡単に実装できます。

```javascript
// Supabase Auth でメール＆パスワードサインアップ
import { supabase } from './supabaseClient';

const signUp = async (email, password) => {
  const { data, error } = await supabase.auth.signUp({
    email,
    password,
  });
  if (error) {
    console.error('サインアップエラー:', error.message);
    return null;
  }
  // data.user に新規ユーザー情報が入る
  console.log('ユーザー作成完了:', data.user.id);
  return data.user;
};
```

### データベース設計

Supabase（PostgreSQL）を使う場合、シンプルな SaaS で必要な最低限のテーブルは次のようになります：

```sql
-- ユーザーテーブル（Supabase Auth と連携）
CREATE TABLE profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  display_name TEXT,
  plan_type TEXT DEFAULT 'free', -- 'free' | 'pro' | 'premium'
  subscription_end_date TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- コンテンツテーブル（例: タスク管理アプリ）
CREATE TABLE tasks (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  is_completed BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Row Level Security を有効化（自分のデータだけアクセス可能）
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users can access own tasks" ON tasks
  FOR ALL USING (auth.uid() = user_id);
```

### サブスクリプション決済

モバイル SaaS の収益化には大きく2つの選択肢があります：

- **Stripe（Web 決済）**: カード決済で柔軟な価格設定が可能。App Store 手数料（30%）が不要
- **StoreKit 2 / RevenueCat（In-App Purchase）**: App Store ネイティブ課金。ユーザーの信頼度が高い

個人開発の初期段階では、Stripe でシンプルに始めて、成長後に In-App Purchase を追加するアプローチが現実的です。

## Rork で SaaS MVP を作る：5つのステップ

### Step 1：アイデアの検証とターゲット設定

SaaS で失敗する最大の理由は「作ってから売ろうとすること」です。まず以下を明確にしましょう：

- **誰のどんな課題を解決するか**（例：フリーランスが毎月手動で作っている請求書作業を自動化）
- **なぜ毎月お金を払う価値があるか**（例：毎月5時間の作業が10分になる）
- **競合アプリとの差別化ポイント**（例：日本語対応・税込み請求書・青色申告対応）

Rork に次のようなプロンプトを送ると、アイデアの骨格を素早く整理できます：

```
フリーランス向けの請求書管理 SaaS モバイルアプリを作りたい。
主な機能:
1. 顧客管理（名前・住所・メール）
2. 請求書作成（品目・金額・消費税・合計）
3. PDF 出力
4. 支払いステータス管理（未払い・支払済み）
5. 月額 ¥500 のサブスクリプション（無料プランは月3件まで）

React Native + Supabase + Stripe で実装してください。
```

### Step 2：認証画面の実装

Rork に「Supabase Auth を使ったログイン・サインアップ画面」を作成するよう指示します。Apple ログインは App Store 審査で必須（サードパーティ認証を提供する場合）なので、最初から対応しておくと安心です。

### Step 3：コアコンテンツ機能の実装

SaaS の「価値の核心」となる機能を実装します。タスク管理アプリであれば CRUD（作成・読み取り・更新・削除）操作、フィットネスアプリであれば記録・グラフ表示などです。

最初は**機能を絞る**ことが成功の鍵です。「あれもこれも」と詰め込むと開発が長期化し、ユーザーも混乱します。

### Step 4：プレミアムプランとペイウォールの実装

無料プランと有料プランの境界を決め、プレミアム機能をロックします：

```javascript
// プランチェック関数
const checkPlanAccess = (userPlan, requiredPlan) => {
  const planLevels = { free: 0, pro: 1, premium: 2 };
  return planLevels[userPlan] >= planLevels[requiredPlan];
};

// 使用例：プレミアム機能を開く前にチェック
const handleExportPDF = () => {
  if (!checkPlanAccess(user.plan_type, 'pro')) {
    // アップグレード促進モーダルを表示
    setShowUpgradeModal(true);
    return;
  }
  // PDF エクスポート処理
  exportToPDF(invoice);
};
```

### Step 5：TestFlight でテストして App Store 公開

MVP が完成したら、TestFlight で実機テストを行い、App Store に申請します。Rork Max ユーザーであれば「2クリック公開」機能で、Xcode なしでそのままストアに申請できます。

## 収益化戦略：個人 SaaS で月収を安定させるために

### フリーミアムモデルの設計

個人開発 SaaS では、**フリーミアム（無料 + 有料）モデル**が最もユーザー獲得に効果的です。設計の原則は：

- **無料プランは価値を感じるが、物足りなさを感じるレベル**に調整する
- **有料プランへのアップグレードトリガー**を自然に作る（容量上限・機能制限など）
- **有料プランの価格は迷わず決める**：個人向けなら月額 ¥300〜¥1,000 が一般的

価格設定の例（タスク管理アプリの場合）：

- **無料プラン**: タスク最大20件、プロジェクト1件
- **Pro プラン（¥480/月）**: タスク無制限、プロジェクト10件、リマインダー機能
- **Premium プラン（¥980/月）**: すべての機能 + チームシェア + カレンダー連携

### チャーン（解約）を防ぐための工夫

SaaS ビジネスの健全性を示す指標として「チャーンレート（月次解約率）」があります。個人開発 SaaS では以下を意識しましょう：

- **オンボーディングを丁寧に**: 初回ログイン時のウォークスルーで「価値」を素早く体験させる
- **週次・月次のサマリー通知**: 「先週は5タスク完了しました！」のような進捗リマインダー
- **年払いオプション**: 月額の2ヶ月分程度の割引で年払いに誘導するとチャーンが大幅に減る

さらに詳しい収益化の戦略については、[Rork × Stripe サブスクリプション実装 — アプリ収益化の完全ガイド](/articles/rork-business/rork-stripe-subscription-monetization-guide) で実装レベルの詳細を解説しています。

## よくあるエラーと対処法

Rork で SaaS を構築する際に直面しやすい問題と、その解決策をまとめます。

### Supabase RLS でデータが取得できない

Row Level Security（RLS）を有効化した後、データが空になる問題が頻発します。

```javascript
// ❌ よくある間違い：RLS のポリシーが未設定でデータが取れない
const { data } = await supabase.from('tasks').select('*');
// data が [] になってしまう

// ✅ 修正：ポリシーを正しく設定する
// Supabase ダッシュボード > Table Editor > tasks > RLS Policies
// 「Enable RLS」+ 「New Policy」から「Allow all to authenticated users」を追加
```

### Stripe Webhook が本番環境で動かない

開発環境ではローカルの Stripe CLI を使うため、本番にデプロイすると Webhook が機能しないケースがあります。Supabase Edge Functions を Webhook エンドポイントとして設定し、Stripe ダッシュボードから本番 URL を登録することで解決します。

### App Store 審査でサブスクリプションがリジェクトされる

Apple の審査ガイドライン（3.1.2）により、iOS アプリ内のデジタルコンテンツ購読には原則 In-App Purchase が必要です。Web サービスへのサブスクリプションは Stripe 直接決済が認められていますが、「App 内で完結するサービス」は StoreKit 2 経由にする必要があります。

## FAQ

**Q1. Rork で本当に SaaS は作れますか？バックエンドが複雑ではないですか？**

はい、Rork と Supabase の組み合わせであれば、認証・データベース・API すべてをプロンプトで実装できます。Supabase はバックエンドをほぼ自動で管理してくれるため、個人開発者でも本格的な SaaS が構築可能です。特に CRUD 操作の多いビジネス系アプリは相性が抜群です。

**Q2. 月額収益を得るには App Store の審査は厳しいですか？**

サブスクリプション機能自体は審査で特別厳しくなるわけではありませんが、「無料トライアル期間の明示」「解約方法の案内」「価格表示の正確性」など、Apple のガイドラインに沿った実装が必要です。Rork でアプリを作る際に、プロンプトに「App Store ガイドラインに準拠した」と明示すると、適切な実装をサポートしてもらいやすくなります。

**Q3. SaaS の収益が出るまでにどのくらいかかりますか？**

ニッチに特化したアプリであれば、ローンチ後3〜6ヶ月で月収1〜5万円に達するケースがあります。ただし、成功の鍵はアプリの質よりも「マーケティング（App Store SEO、SNS 発信）」にあります。良いアプリを作るだけでなく、ターゲットユーザーにリーチする発信活動も並行して行いましょう。

**Q4. 個人開発 SaaS で法人化は必要ですか？**

月収が数万円程度であれば、個人事業主（フリーランス）のままで十分です。売上が年間100万円を超えるようになったら、税理士に相談して最適な形態を選ぶのが賢明です。Rork ラボには個人開発の事業化を詳しく解説した記事もあるので参考にしてください。
