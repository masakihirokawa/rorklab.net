## はじめに — なぜ Rork アプリに広告収益を組み込むべきか

アプリ収益化にはサブスクリプション・買い切り課金・広告の3つの柱があります。なかでも **広告モデル** は、ユーザーに無料でアプリを提供しながら開発者が収入を得られる手法として、個人開発者に特に人気があります。

Google AdMob は世界最大のモバイル広告プラットフォームであり、React Native（Expo）ベースの Rork アプリとも組み合わせやすいのが特長です。本記事では、Rork で構築したアプリに AdMob 広告を導入する手順を、広告フォーマットごとに丁寧に解説します。

この記事の対象読者は以下のとおりです。

- Rork でアプリを作ったが、まだ収益化していない方
- サブスクリプション課金と広告モデルの併用を検討している方
- AdMob の導入手順を一から知りたい初心者〜中級者

## AdMob の基本 — 3つの広告フォーマットを理解する

AdMob が提供する主要な広告フォーマットは次の3つです。

**バナー広告（Banner Ads）**は、画面の上端または下端に常時表示される帯状の広告です。ユーザー体験への影響が少なく、安定したインプレッション収益を得られます。eCPM（1,000回表示あたりの収益）は控えめですが、表示回数の多さでカバーできます。

**インタースティシャル広告（Interstitial Ads）**は、画面遷移のタイミングで全画面表示される広告です。バナー広告よりも eCPM が高いものの、表示頻度が多すぎるとユーザー離脱の原因になります。「3画面遷移に1回」などのルールを設けるのがベストプラクティスです。

**リワード広告（Rewarded Ads）**は、ユーザーが任意で視聴し、報酬（アプリ内通貨・追加機能など）を受け取る広告です。eCPM が最も高く、ユーザー満足度も維持しやすいため、ゲームアプリやツール系アプリで特に効果的です。

## AdMob アカウントの作成と広告ユニットの設定

まず [AdMob コンソール](https://admob.google.com/) でアカウントを作成し、アプリを登録します。

**ステップ 1：アプリの登録**

AdMob コンソールの「アプリ」→「アプリを追加」を選択します。App Store / Google Play に公開済みの場合はストア URL で検索できます。未公開の場合は「アプリはまだ公開されていません」を選んで手動登録します。

**ステップ 2：広告ユニットの作成**

登録したアプリに対して、バナー・インタースティシャル・リワードの各広告ユニットを作成します。作成するとそれぞれに **広告ユニット ID**（`ca-app-pub-XXXXX/YYYYY` 形式）が発行されるので控えておきます。

**ステップ 3：app-ads.txt の設置**

AdMob の不正防止のために、ウェブサイトのルートディレクトリに `app-ads.txt` を配置します。AdMob コンソールの案内に従ってパブリッシャー ID を記載してください。

## Rork プロジェクトへの AdMob SDK 導入

Rork は React Native（Expo）ベースなので、`react-native-google-mobile-ads` ライブラリを使用します。

```bash
# Rork プロジェクトのターミナルで実行
npx expo install react-native-google-mobile-ads
```

次に、`app.json`（または `app.config.js`）に AdMob のアプリ ID を追加します。

```json
{
  "expo": {
    "plugins": [
      [
        "react-native-google-mobile-ads",
        {
          "androidAppId": "ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY",
          "iosAppId": "ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY"
        }
      ]
    ]
  }
}
```

> **注意**: 上記のアプリ ID は AdMob コンソールで発行されたものに置き換えてください。テスト段階では Google が提供するテスト用 ID を使用することを推奨します。

初期化処理はアプリのエントリーポイントで行います。

```typescript
// App.tsx または app/_layout.tsx
import mobileAds from 'react-native-google-mobile-ads';

// アプリ起動時に AdMob SDK を初期化
mobileAds()
  .initialize()
  .then(adapterStatuses => {
    console.log('AdMob initialized:', adapterStatuses);
    // 初期化完了後に広告のリクエストが可能になる
  });
```

## バナー広告の実装 — 画面下部に常時表示する

バナー広告は最もシンプルに実装できるフォーマットです。

```tsx
import React from 'react';
import { View, StyleSheet } from 'react-native';
import { BannerAd, BannerAdSize, TestIds } from 'react-native-google-mobile-ads';

// テスト用の広告ユニットID（本番リリース時は実際のIDに差し替え）
const BANNER_AD_UNIT_ID = __DEV__
  ? TestIds.BANNER
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      {/* メインコンテンツ */}
      <View style={styles.content}>
        {/* アプリのUI */}
      </View>

      {/* 画面下部のバナー広告 */}
      <BannerAd
        unitId={BANNER_AD_UNIT_ID}
        size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
        requestOptions={{
          requestNonPersonalizedAdsOnly: true,
        }}
        onAdLoaded={() => console.log('Banner ad loaded')}
        onAdFailedToLoad={(error) => console.error('Banner ad failed:', error)}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
  content: { flex: 1 },
});
// 期待する出力: 画面下部にアダプティブバナー広告が表示される
```

`ANCHORED_ADAPTIVE_BANNER` は端末の画面幅に自動調整されるバナーサイズで、現在の推奨設定です。

## インタースティシャル広告の実装 — 画面遷移時に表示する

インタースティシャル広告は事前にロードしておき、適切なタイミングで表示します。

```tsx
import React, { useEffect, useState } from 'react';
import { Button, View } from 'react-native';
import {
  InterstitialAd,
  AdEventType,
  TestIds,
} from 'react-native-google-mobile-ads';

const INTERSTITIAL_AD_UNIT_ID = __DEV__
  ? TestIds.INTERSTITIAL
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

// インスタンスを作成（コンポーネント外で1回だけ）
const interstitial = InterstitialAd.createForAdRequest(
  INTERSTITIAL_AD_UNIT_ID,
  { requestNonPersonalizedAdsOnly: true }
);

export default function ResultScreen() {
  const [adLoaded, setAdLoaded] = useState(false);

  useEffect(() => {
    // 広告ロード完了イベントを監視
    const unsubscribeLoaded = interstitial.addAdEventListener(
      AdEventType.LOADED,
      () => setAdLoaded(true)
    );
    // 広告クローズ後に次の広告をプリロード
    const unsubscribeClosed = interstitial.addAdEventListener(
      AdEventType.CLOSED,
      () => {
        setAdLoaded(false);
        interstitial.load(); // 次回用にプリロード
      }
    );

    interstitial.load(); // 初回ロード

    return () => {
      unsubscribeLoaded();
      unsubscribeClosed();
    };
  }, []);

  const showAd = () => {
    if (adLoaded) {
      interstitial.show();
    }
  };

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Button title="次の画面へ" onPress={showAd} />
    </View>
  );
}
// 期待する出力: ボタン押下時に全画面広告が表示され、閉じた後に次の広告がプリロードされる
```

**表示頻度の制御**がインタースティシャル広告では非常に重要です。画面遷移のたびに表示するとユーザー体験が著しく低下します。「3回に1回」「5分おきに最大1回」などのロジックを自前で管理しましょう。

## リワード広告の実装 — ユーザーに報酬を提供する

リワード広告は eCPM が最も高く、ユーザーにとっても「報酬を得られる」というメリットがあるため、Win-Win の広告フォーマットです。

```tsx
import React, { useEffect, useState } from 'react';
import { Button, View, Alert } from 'react-native';
import {
  RewardedAd,
  RewardedAdEventType,
  TestIds,
} from 'react-native-google-mobile-ads';

const REWARDED_AD_UNIT_ID = __DEV__
  ? TestIds.REWARDED
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

const rewarded = RewardedAd.createForAdRequest(
  REWARDED_AD_UNIT_ID,
  { requestNonPersonalizedAdsOnly: true }
);

export default function RewardScreen() {
  const [adLoaded, setAdLoaded] = useState(false);

  useEffect(() => {
    const unsubscribeLoaded = rewarded.addAdEventListener(
      RewardedAdEventType.LOADED,
      () => setAdLoaded(true)
    );
    // ユーザーが動画を最後まで視聴した時に報酬を付与
    const unsubscribeEarned = rewarded.addAdEventListener(
      RewardedAdEventType.EARNED_REWARD,
      (reward) => {
        Alert.alert(
          '報酬獲得！',
          `${reward.amount} ${reward.type} を獲得しました`
        );
        // ここでアプリ内通貨やプレミアム機能のアンロック処理を実行
      }
    );

    rewarded.load();

    return () => {
      unsubscribeLoaded();
      unsubscribeEarned();
    };
  }, []);

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Button
        title="動画を見てコインを獲得"
        onPress={() => {
          if (adLoaded) {
            rewarded.show();
            setAdLoaded(false);
            rewarded.load(); // 次回用にプリロード
          }
        }}
        disabled={!adLoaded}
      />
    </View>
  );
}
// 期待する出力: 動画視聴後にAlertで報酬内容が表示される
```

## eCPM 最適化と広告配置のベストプラクティス

AdMob で収益を最大化するには、広告の「配置」と「頻度」が鍵です。

**配置のコツ**として、バナー広告は画面下部の固定位置が最も効果的です。スクロール可能なコンテンツの途中に挟む「インフィード広告」も高い視認性が得られます。インタースティシャル広告は、タスク完了後や自然なブレークポイント（ステージクリア後、検索結果表示前など）に配置すると離脱を防げます。

**頻度制限（Frequency Capping）**として、インタースティシャルは1セッションあたり最大3回、最低60秒の間隔を空けることを推奨します。Google 自体も過度な表示を行うアプリにはペナルティを課す可能性があります。

**広告メディエーション**の導入も検討しましょう。AdMob メディエーションを使うと、Meta Audience Network・Unity Ads・AppLovin などの複数の広告ネットワークを統合し、最も高い eCPM を自動選択してくれます。設定は AdMob コンソールのメディエーションタブから行えます。

**A/B テストの活用**として、AdMob には広告のフォーマットや配置をテストする機能があります。バナーの表示位置やインタースティシャルの表示頻度を変えて、収益とユーザーリテンションのバランスを検証しましょう。

## 広告とサブスクリプション課金の併用戦略

個人開発者にとって最も効果的な収益モデルは、**無料（広告あり）+ 有料（広告なし）** のフリーミアム構造です。

具体的には、無料ユーザーにはバナー広告とインタースティシャル広告を表示し、有料プラン（月額課金や買い切り）を購入したユーザーには広告を非表示にします。この設計により、広告収益とサブスクリプション収益の両方を得られます。

```typescript
// 広告表示の制御例
const shouldShowAds = !userSubscription.isPremium;

// コンポーネント内で条件分岐
{shouldShowAds && (
  <BannerAd
    unitId={BANNER_AD_UNIT_ID}
    size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
  />
)}
```

Rork で Stripe を使ったサブスクリプション課金を実装する方法については、[Rork × Stripe サブスクリプション実装 完全ガイド](/articles/rork-business/rork-stripe-subscription-monetization-guide) で詳しく解説しています。

## よくある質問（FAQ）

### Q: AdMob のテスト広告と本番広告の切り替えはどうするのですか？

開発中は必ず `TestIds`（テスト ID）を使用してください。テスト ID を使わずに開発中の端末で実際の広告をクリックすると、AdMob アカウントが停止される可能性があります。本番リリース時に環境変数や `__DEV__` フラグで切り替える実装がベストです。

### Q: 広告の eCPM を上げるにはどうすればよいですか？

まずリワード広告を優先的に導入してください。次にメディエーションを有効にして複数のネットワークを競合させます。さらに、ユーザーのエンゲージメントが高いアプリほど広告単価が上がるため、リテンション率の改善にも取り組みましょう。

### Q: AdMob は Expo Go で動作しますか？

いいえ、`react-native-google-mobile-ads` はネイティブモジュールを含むため Expo Go では動作しません。`expo prebuild` でネイティブプロジェクトを生成するか、EAS Build を使った Development Build で実機テストしてください。Rork Max をお使いの場合は、クラウド Mac 上でネイティブビルドが自動実行されます。

### Q: GDPR や CCPA など個人情報保護への対応は必要ですか？

はい。EU の GDPR や米カリフォルニア州の CCPA に対応するため、初回起動時に広告パーソナライゼーションへの同意を取得する UMP（User Messaging Platform）SDK の導入が推奨されます。AdMob コンソールの「プライバシーとメッセージ」からコンプライアンスメッセージを設定できます。

## まとめ

Rork で開発したアプリに AdMob 広告を導入することで、ダウンロード数がそのまま収益に直結するビジネスモデルを構築できます。バナー広告で安定した基盤収益を確保し、インタースティシャル広告で収益を上乗せし、リワード広告でユーザー体験と収益を両立させる — この3段構えが広告収益化の王道パターンです。

さらに収益を最大化したい方には、広告とサブスクリプション課金を組み合わせた戦略も非常に有効です。より実践的な収益化テクニックについては [Rork アプリ自動収益化パイプライン完全設計](/articles/rork-business/rork-automated-revenue-pipeline-complete-design) もあわせてご覧ください。
