## アプリ収益化で失敗する理由と成功の設計思想

Rork を使って素晴らしいアプリを作ることができても、収益化で躓いてしまうケースは少なくありません。多くの開発者が「とりあえず広告を入れる」という方法を選びますが、それだけでは月収は伸びません。

本記事では、収益を継続的に最大化するための**収益モデル設計**の考え方から、具体的な実装方法まで体系的に解説します。

### 収益モデルの3つの柱

成功しているアプリの多くは、以下の3つを組み合わせています。

**① 広告収益（AdMob）**: ユーザーベースが大きい場合に効果的。DAU × 平均ARPDAU でスケールする。

**② アプリ内課金（IAP）**: 一時的な価値提供。追加コンテンツ・機能解放・消耗品等。

**③ サブスクリプション**: 継続的な価値提供。プレミアム機能へのアクセス権。

この3つをどう組み合わせるかは、アプリのジャンルとターゲットユーザーによって大きく異なります。

### ジャンル別の推奨収益モデル

| ジャンル | 最適なモデル | 理由 |
|---------|-----------|------|
| ゲーム | 広告 + 消耗型IAP | DAU が高く、消耗品の繰り返し購入が期待できる |
| ツール・生産性 | サブスクリプション | 継続利用価値が高く、解約率が低い |
| コンテンツ | フリーミアム + サブスク | コンテンツの深みを価値として提供できる |
| 健康・フィットネス | サブスクリプション | 習慣化による長期継続が期待できる |

## 広告収益の最大化：AdMob × Rork の実装

Rork で AdMob を実装する際の、収益を最大化するための設定方法を紹介します。

### 広告フォーマットの選択と配置

```javascript
// Rork での AdMob 実装例
import { useState, useEffect } from 'react';
import { View, Text, TouchableOpacity } from 'react-native';

// インタースティシャル広告（最高収益・最大15-25%のeCPM）
const InterstitialAdManager = {
  // 広告を事前にロードしておく（UX を損なわないため）
  preload: async () => {
    await AdMob.requestInterstitial({
      adUnitId: Platform.OS === 'ios'
        ? 'ca-app-pub-XXXXXXXXXXXXXXXX/XXXXXXXXXX'
        : 'ca-app-pub-XXXXXXXXXXXXXXXX/XXXXXXXXXX',
    });
  },

  // 自然なタイミングで表示（レベルクリア・タスク完了後等）
  show: async (context: string) => {
    const isLoaded = await AdMob.isInterstitialLoaded();
    if (isLoaded) {
      await AdMob.showInterstitial();
      analytics.track('ad_shown', { type: 'interstitial', context });
    }
  },
};

// リワード広告（最高eCPM・ユーザー満足度も高い）
const RewardedAdManager = {
  show: async (reward: { type: string; amount: number }) => {
    return new Promise((resolve, reject) => {
      AdMob.showRewardedAd({
        onRewarded: (rewardData) => {
          // ユーザーへの報酬付与
          giveReward(reward);
          resolve(rewardData);
        },
        onDismissed: () => resolve(null),
        onError: reject,
      });
    });
  },
};
```

### 広告収益を2倍にする「メディエーション」

AdMob のメディエーション機能を使うと、複数の広告ネットワークを競わせてeCPMを向上できます。

```javascript
// AdMob Mediation の設定（rork.config.js）
export const adConfig = {
  mediation: {
    networks: ['admob', 'meta', 'applovin', 'unity'],
    waterfall: [
      { network: 'applovin', floorPrice: 2.0 },  // $2.00 以上のみ
      { network: 'meta', floorPrice: 1.5 },
      { network: 'unity', floorPrice: 1.0 },
      { network: 'admob', floorPrice: 0 },         // フォールバック
    ],
  },
  // 広告表示の適切な頻度制御
  frequency: {
    interstitial: {
      minIntervalSeconds: 60,    // 最低60秒間隔
      maxPerSession: 8,          // セッションあたり最大8回
    },
  },
};
```

実測値として、メディエーション導入前後でeCPMが**平均40〜60%向上**することが多いです。

## サブスクリプション設計：解約されにくいプランを作る

サブスクリプションは正しく設計しないと解約率（チャーンレート）が高くなり、収益が安定しません。

### 効果的な価格設定の法則

**アンカリング効果を使った3段階プライシング:**

```javascript
// src/screens/PaywallScreen.tsx
const subscriptionPlans = [
  {
    id: 'monthly',
    title: '月額プラン',
    price: '¥480',
    period: '月',
    highlight: false,
    features: ['プレミアム機能すべて', '広告非表示'],
    badge: null,
  },
  {
    id: 'yearly',
    title: '年額プラン',
    price: '¥2,880',
    period: '年',
    highlight: true,  // このプランを目立たせる
    features: ['プレミアム機能すべて', '広告非表示', '優先サポート', '新機能先行アクセス'],
    badge: '50% OFF',  // ¥480×12 = ¥5,760 との比較
    originalPrice: '¥5,760',
  },
  {
    id: 'lifetime',
    title: '永久ライセンス',
    price: '¥9,800',
    period: '一度きり',
    highlight: false,
    features: ['プレミアム機能すべて', '広告非表示', '生涯アップデート'],
    badge: '人気',
  },
];
```

**年額プランへの誘導が重要な理由:** 月額 ¥480 で解約されると LTV（顧客生涯価値）が低くなります。年額 ¥2,880 なら、たとえ1年後に解約しても LTV は6倍になります。

### 解約を防ぐエンゲージメント設計

```javascript
// src/hooks/useSubscriptionEngagement.ts
export function useSubscriptionEngagement() {
  const checkEngagement = async (userId: string) => {
    const usage = await getUsageStats(userId);

    // 7日間未使用のユーザーへの再エンゲージメント
    if (usage.daysSinceLastOpen >= 7) {
      await sendPushNotification({
        title: '久しぶり！',
        body: 'プレミアム機能を活用していますか？使い方のコツをお送りします。',
        data: { action: 'show_tips' },
      });
    }

    // 解約前の「ポーズ」機能提案
    if (usage.unsubscribeIntent) {
      await showPauseOffer({
        discountPercent: 50,
        pauseDays: 30,
        message: '1ヶ月お休みしますか？50%割引で再開できます。',
      });
    }
  };
}
```

## アプリ内課金の設計：心理的に買いたくなる仕組み

### 消耗品 IAP の最適化

ゲームやサービス系アプリでは、消耗品（コイン・ポイント等）の設計が収益に直結します。

```javascript
// コイン（消耗品）の価格設定
const coinPackages = [
  { id: 'coins_small',  coins: 100,  price: 120,  highlight: false },
  { id: 'coins_medium', coins: 600,  price: 500,  highlight: true,  bonus: '20%お得' },
  { id: 'coins_large',  coins: 2000, price: 1200, highlight: false, bonus: '67%お得' },
  { id: 'coins_mega',   coins: 6000, price: 3000, highlight: false, bonus: '150%お得' },
];

// 最も収益貢献が高いのは「medium」パッケージ（心理的な中間点）
// Rork での IAP 実装
async function purchaseCoins(packageId: string) {
  try {
    const result = await InAppPurchases.purchaseItemAsync(packageId);
    if (result.responseCode === IAPResponseCode.OK) {
      const pkg = coinPackages.find(p => p.id === packageId);
      await addCoinsToUser(pkg!.coins);
      analytics.track('iap_purchase', {
        packageId,
        revenue: pkg!.price / 100,
        currency: 'JPY',
      });
    }
  } catch (error) {
    console.error('IAP エラー:', error);
  }
}
```
