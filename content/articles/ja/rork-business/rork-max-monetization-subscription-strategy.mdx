## Rork Max アプリの収益化 — 「作って終わり」にしない

Rork Maxの登場により、ネイティブSwiftアプリの開発は驚くほど簡単になりましました。しかし、アプリを作ることとアプリで収益を上げることは、まったく別のスキルです。

本記事では、Rork Maxで生成したアプリに収益化機能を組み込み、持続的な収入源に育てるための戦略を体系的に解説します。StoreKit 2を使ったサブスクリプション実装から、価格設定の心理学、App Store最適化（ASO）まで、実践的なテクニックを網羅します。

## 収益モデルの選択

### 主要な収益モデル比較

| モデル | メリット | デメリット | 適合するアプリ |
|---|---|---|---|
| サブスクリプション | 安定した月次収入 | 解約リスク | ユーティリティ、生産性 |
| 買い切り | シンプル、解約なし | 収入が一過性 | ツール、ゲーム |
| フリーミアム | 広いユーザーベース | 低い課金率（2-5%） | SNS、コンテンツ |
| 広告収入 | 無料ユーザーからも収入 | UX低下リスク | ゲーム、メディア |
| 消耗型課金 | 繰り返し購入 | 収入の予測が難しい | ゲーム内通貨 |

### 2026年のトレンド

App Storeの売上データによると、2026年のモバイルアプリ収益の**72%がサブスクリプション**から生まれています。特にAIを活用したアプリでは、ユーザーが継続的な価値に対価を払う傾向が強く、サブスクリプションモデルが最適です。

### 生成されるコードの構造

```swift
// Rork Max が生成する StoreKit 2 コード例
import StoreKit

class SubscriptionManager: ObservableObject {
    @Published var products: [Product] = []
    @Published var purchasedProductIDs: Set<String> = []

    // 商品ID定義
    let productIDs = [
        "com.app.pro.monthly",    // Pro月額
        "com.app.premium.yearly"  // Premium年額
    ]

    // 商品情報の取得
    func loadProducts() async {
        do {
            products = try await Product.products(for: productIDs)
        } catch {
            print("Failed to load products: \(error)")
        }
    }

    // 購入処理
    func purchase(_ product: Product) async throws -> Transaction? {
        let result = try await product.purchase()

        switch result {
        case .success(let verification):
            let transaction = try checkVerified(verification)
            await transaction.finish()
            purchasedProductIDs.insert(product.id)
            return transaction
        case .userCancelled, .pending:
            return nil
        @unknown default:
            return nil
        }
    }

    // トランザクション検証
    private func checkVerified<T>(_ result: VerificationResult<T>) throws -> T {
        switch result {
        case .unverified:
            throw StoreError.failedVerification
        case .verified(let safe):
            return safe
        }
    }
}
```

## 価格設定の戦略

### 地域別価格設定

Rork Maxでは、App Store Connectの「価格同等化」機能を活用して、地域ごとに適切な価格を設定できます。

```
プロンプト例:
「以下の価格体系で設定して:
- 日本: Pro ¥480/月、Premium ¥4,800/年
- 米国: Pro $3.99/月、Premium $39.99/年
- 東南アジア: Pro $1.99/月、Premium $19.99/年
App Store Connectの価格帯に合わせて端数調整して。」
```

### 心理的価格設定のポイント

1. **アンカリング効果**: 最高額プランを最初に表示し、中間プランがお得に見えるようにする
2. **年額割引**: 月額の10ヶ月分 = 年額として「2ヶ月無料」を訴求
3. **無料トライアル**: 7日間の体験期間で機能を実感させてから課金に移行
4. **端数価格**: ¥500ではなく¥480、$4ではなく$3.99

### スクリーンショット戦略

App Storeのスクリーンショットは、ダウンロード率に直結する最重要要素です。

```
スクリーンショット構成の推奨パターン:
1枚目: メイン機能のデモ（最もインパクトのある画面）
2枚目: AIによる自動処理の結果画面
3枚目: サブスクリプションで解放される機能
4枚目: 設定・カスタマイズ画面
5枚目: ユーザーレビュー or 実績データ
```

## 広告統合

フリーミアムモデルの場合、無料ユーザーからの広告収入も重要な収益源です。

### AdMob 統合

```
プロンプト例:
「Google AdMobを統合して。
配置:
- メイン画面の下部にバナー広告
- 機能使用後にインタースティシャル広告（5回に1回）
- リワード広告（見ると1日プレミアム機能が使える）
Pro以上のサブスクリプション購入者には全広告非表示。」
```

### 広告とサブスクリプションの共存設計

```
無料ユーザー:
  └── 基本機能 + 広告表示 → 広告収入

Pro ユーザー:
  └── 全機能 + 広告非表示 → サブスク収入

Premium ユーザー:
  └── 全機能 + 広告非表示 + 優先サポート → サブスク収入
```

## 収益分析とKPI管理

### 追跡すべき主要KPI

```
- MRR（Monthly Recurring Revenue）: 月次経常収益
- ARPU（Average Revenue Per User）: ユーザーあたり平均収益
- LTV（Lifetime Value）: 顧客生涯価値
- Churn Rate: 月次解約率
- Trial-to-Paid Conversion: トライアルからの課金転換率
- ROAS（Return On Ad Spend）: 広告費用対効果
```

## よくある質問（FAQ）

### Q1: Rork Max で生成したアプリにサブスクリプションを後から追加できますか？

はい、既存のRork Maxプロジェクトに「サブスクリプション機能を追加して」と指示するだけで、StoreKit 2の実装が追加されます。ペイウォールUI、購入フロー、復元機能までが自動生成されます。

### Q2: Apple のアプリ内課金の手数料はどのくらいですか？

Appleはアプリ内課金に対して、年間売上100万ドル以下の場合は15%、100万ドル超の場合は30%の手数料を徴収します。サブスクリプションの場合、同一ユーザーが1年以上継続すると手数料が15%に下がります。

### Q3: 日本向けの価格設定で気をつけることはありますか？

日本市場では、月額380〜480円帯がもっとも受け入れられやすい価格帯です。年額プランは月額の10ヶ月分に設定して「2ヶ月分お得」を訴求するのが効果的です。また、日本のユーザーは無料トライアルの有無を重視する傾向があるため、7日間の無料体験を必ず含めましょう。
