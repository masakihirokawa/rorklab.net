## ASO Basics for Rork Users

Apps developed with Rork require the same App Store and Google Play optimization as native apps. However, Rork users typically have less technical development experience, so ASO approaches must be simple and efficient.

### What is ASO?

ASO (App Store Optimization) optimizes apps for app store search algorithms. Just as SEO targets websites, ASO targets app stores. Key optimization elements:

- **Title & Subtitle**: Keyword inclusion
- **Keyword Field**: App Store exclusive (max 100 chars)
- **Description**: User-focused text + keyword inclusion
- **Screenshot Captions**: Multilingual support
- **Reviews & Ratings**: 4.0+ rating critical

---

## App Store / Google Play Launch with Rork

### Build, Sign, and Submit Workflow

Rork-developed apps require the following for store submission:

**App Store (iOS)**:
1. Register Apple Developer Program ($99/year)
2. Create App ID
3. Build (.ipa file generation in Xcode)
4. Submit via Apple Transporter
5. Wait for review (1-2 days)

**Google Play (Android)**:
1. Register Google Play Developer Program ($25 one-time)
2. Generate App Signing Key
3. Build (.apk/.aab file generation in Android Studio)
4. Submit via Google Play Console
5. Auto-review (hours)

Rork natively supports both iOS and Android. Once build settings are complete, the necessary files (.ipa, .aab) auto-generate for store submission.

### App Store (iOS)

**Title (30 characters maximum)**:
```
❌ Weak: MyApp

Includes keyword "task manager"
```

**Subtitle (30 characters maximum)**:
```
❌ Weak: Best app
✅ Strong: Organize & Plan Your Day

Includes secondary keyword "organize"
```

### Google Play (Android)

**Title (50 characters optimal)**:
```
❌ Weak: MyApp
✅ Strong: TaskNow - To Do List & Task Manager

Multiple keywords in title (Google Play prioritizes title heavily)
```

**Description Opening (first 3 lines)**:
```
Perfect task management app to organize your daily activities.
Track your to-do list and boost productivity with TaskNow.
Simple, fast, and reliable task organizer for everyone.
```

On Google Play, the description opening is most critical. Include 3-4 keywords naturally in this section.

## Multilingual Localization Strategy

### Multilingual Support with Rork

Rork makes supporting multiple languages relatively straightforward. These genres have high multilingual demand:

**Categories Requiring Multilingual Support**:
- Business & productivity apps (global demand)
- Language learning apps (multilingual is core function)
- Utility apps (diverse local needs)

**Lower-Priority Multilingual Categories**:
- Japan-specific games
- Japanese content-dependent apps

### Minimal Multilingual Launch (English + Japanese)

Most Rork users realistically start with English + Japanese:

```
【English Version】
Title: TaskNow - To Do List Manager
Subtitle: Organize & Plan Your Day
```

1. **Phase 1** (Launch): English + Japanese

Phased rollout ensures localization quality per language.

## Review Acquisition Strategy

### In-App Review Promotion

Implement Rork with iOS `SKStoreReviewController` and Android `ReviewManager API`:

**Implementation Pattern**:
1. After user has used app 5-10 times
2. Immediately following positive action (task completion, etc.)
3. Natural review prompt popup

Both `SKStoreReviewController` and `ReviewManager API` display native rating screens, resulting in 3-5x higher ratings than custom dialogs due to lower user friction.

### Optimal Timing for Review Requests

- ❌ First app launch (users unfamiliar with features)
- ✅ After positive first experience (5-10 uses)
- ✅ Following specific feature completion (e.g., "Task complete!")

### Responding to Low Ratings

Google Play allows direct replies to low-rating reviewers. Rork users should check regularly:

```
Low Rating Review: "App is slow"
↓
Reply: "Thank you for reporting. We've improved performance in
version X. Please try again."
```

Thoughtful responses to low ratings often result in rating improvements.

## A/B Testing & Metadata Update Cycle

### Regular Metadata Updates

ASO requires ongoing optimization. Monthly 1-2 metadata updates recommended:

```
Week 1: Keyword Research
  → Find new keywords via App Annie / App Figures

Week 2: Metadata Update
  → Update title, description, keywords
  → Submit to App Store / Google Play

Week 3-4: Measure Results
  → Track download and CVR changes via App Analytics
  → Continue successful keywords into next month
```

### Key Metrics to Track

- **Downloads**: Compare before/after metadata changes
- **Keyword Ranking**: Record weekly in App Figures
- **CVR (Conversion Rate)**: Product page views ÷ Downloads
- **Rating**: Maintain 4.0+

## Common Rork User Pitfalls

### Pitfall 1: Keyword Overloading in Title

```
❌ Bad Example:
"TaskNow - Task Manager To Do List Productivity App Organizer"
(57 characters → exceeds Google Play 50-char limit)

✅ Correct:
"TaskNow - Task Manager To Do List Productivity App"
(50 characters, 4 keywords naturally included)
```

### Pitfall 2: Unrealistic Expectations for Multilingual Support

```
❌ Bad Approach: Target 10 languages at launch
  → Poor translation quality, complex management

✅ Correct: Start with English + Japanese, expand gradually
  → High-quality few languages outperform low-quality many languages
```

### Pitfall 3: Wrong Review Promotion Timing

```
❌ Bad Timing: Request review at app first launch
  → Users don't understand features → low ratings

✅ Correct: Promote after 5-10 uses, following positive action
  → Users satisfied with experience → high ratings
```
