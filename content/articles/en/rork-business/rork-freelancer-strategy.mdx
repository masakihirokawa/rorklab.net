## Step 2: Finding Clients

### Freelance Marketplaces

Platforms like **Upwork, Toptal, and Contra** are excellent starting points. Search the mobile app development category and make sure your profile clearly showcases your Rork-powered work.

Key elements for a strong profile:
- Screenshots or demo videos of Rork-built apps
- Delivery timeframes ("Most apps delivered in 1–2 weeks")
- Your tech stack (React Native, Expo, Supabase, etc.)
- Client testimonials or reviews

### Content-Driven Outreach on Social Media

Post development case studies and time-lapse build videos on LinkedIn, X (Twitter), and TikTok regularly. A 30-to-60-second video showing an app being built from prompt to prototype is highly shareable and positions you as someone who delivers results fast.

### Targeting Local Businesses

Restaurants, gyms, salons, and retailers often want loyalty apps, reservation systems, or simple customer-facing tools. Cold outreach to local businesses can be highly effective because the competition is lower and decision-making is faster.

## Step 3: Writing a Winning Proposal

A great proposal focuses on outcomes, not tools.

### What to Include

1. **Problem statement**: The specific pain the client is experiencing
2. **Proposed solution**: How the app solves that problem
3. **Feature list**: With mockups or screenshots if available
4. **Timeline**: Milestone-based schedule
5. **Pricing and payment terms**: 50% upfront is standard
6. **Portfolio**: Relevant past work

## Step 4: Development Workflow

### Phase 1: Requirements Discovery (1–2 days)

Clarify the following with your client before writing a single line of code:

- Target users (age range, tech literacy)
- Must-have features vs. nice-to-have features
- Design preferences (reference apps or competitors)
- Backend data structure needs
- Post-launch update frequency

### Phase 3: Customization and Refinement (2–5 days)

After Rork generates the base, add client-specific requirements and any custom business logic.

```javascript
// Example: Supabase client setup with Expo SecureStore
import { createClient } from '@supabase/supabase-js';
import * as SecureStore from 'expo-secure-store';

// Custom storage adapter using Expo SecureStore for secure token persistence
const ExpoSecureStoreAdapter = {
  getItem: (key: string) => SecureStore.getItemAsync(key),
  setItem: (key: string, value: string) => SecureStore.setItemAsync(key, value),
  removeItem: (key: string) => SecureStore.deleteItemAsync(key),
};

export const supabase = createClient(
  process.env.EXPO_PUBLIC_SUPABASE_URL!,
  process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!,
  {
    auth: {
      storage: ExpoSecureStoreAdapter,
      autoRefreshToken: true,
      persistSession: true,
      detectSessionInUrl: false,
    },
  }
);

// Fetch paginated workout records for a user
export async function getWorkoutRecords(userId: string, limit = 30) {
  const { data, error } = await supabase
    .from('workout_records')
    .select('id, exercise, sets, reps, weight, created_at')
    .eq('user_id', userId)
    .order('created_at', { ascending: false })
    .limit(limit);

  if (error) throw new Error(`Failed to fetch records: ${error.message}`);

  // Expected output:
  // [
  //   { id: '1', exercise: 'Squat', sets: 3, reps: 10, weight: 176, created_at: '2026-03-15T09:00:00Z' },
  //   { id: '2', exercise: 'Bench Press', sets: 4, reps: 8, weight: 154, created_at: '2026-03-14T10:30:00Z' },
  //   ...
  // ]
  return data;
}
```

## Client Communication Best Practices

### Should You Tell Clients You Use AI Tools?

Our recommendation: **yes, but frame it correctly**.

Weak framing: "AI writes the code so it's cheaper."
→ Triggers quality concerns.

Strong framing: "I use cutting-edge AI-assisted development tools that allow me to deliver production-ready React Native apps significantly faster than traditional methods, without compromising on code quality or App Store readiness."

Clients care about outcomes, not methods. Lead with the benefits—faster delivery, proven tech stack, App Store compliant—and Rork becomes a selling point, not a liability.

### Managing Scope Creep

Define a change management policy upfront and include it in your contract:

- Adjustments to agreed features: included (up to revision limit)
- New feature additions: separate quote required
- Major design overhauls: separate quote required
- Removal of agreed features: no refund

Clarity here prevents the vast majority of disputes and protects both parties.

## Frequently Asked Questions

**Q1: Will apps built with Rork pass App Store and Google Play review?**

Yes. Rork Max generates production-ready React Native + Expo code that meets Apple and Google's technical requirements. Approval depends on your app's content and compliance with platform policies, but there's no technical barrier to submission. Follow the [App Store Publishing Guide](/articles/rork-dev/app-store-publishing) for a step-by-step walkthrough.

**Q2: What if a client requests the source code?**

You can provide source code generated by Rork Max. We recommend pricing source code delivery as an add-on (typically +$1,000–$2,000) and clearly defining what post-delivery support is included when the client takes ownership of the code.

**Q3: Can someone with no coding experience start freelancing with Rork?**

You can handle many basic projects, but we recommend learning JavaScript and React Native fundamentals before taking paid work. A one-to-two week online course will give you enough knowledge to troubleshoot issues, handle environment configurations, and respond confidently when clients ask technical questions.

**Q4: How do I handle multiple client projects simultaneously?**

Rork Max supports multiple projects. Keep each client's work in a separate project to prevent context mixing. Use a project management tool (Notion, Linear, or Trello) to track milestones across clients and avoid dropped balls.

**Q5: How should I price my services compared to traditional developers?**

Price based on value delivered, not hours spent. A client who needs an app in two weeks instead of three months will happily pay a premium. Don't race to the bottom—your speed and quality are genuine differentiators that command higher rates than traditional developers who need twice the time.
