## The Challenge: Building Games People Love and Pay For

You've built a mobile game in Unity. Users are downloading it. Engagement looks promising. Now comes the question every indie developer faces: How do you actually earn money?

The answer isn't "make it pay-to-win" or "fill it with ads." The most successful mobile games use a balanced monetization strategy combining multiple revenue streams.

This guide covers three proven approaches, their implementation complexity, revenue potential, and strategic combinations for maximum profitability.

## Three Monetization Methods Compared

| Strategy | Implementation | Initial Revenue | Scaling Potential | User Experience |
|----------|--------------|-----------------|-------------------|-----------------|
| **AdMob Ads** | Easy | Low | High | Medium friction |
| **In-App Purchases** | Medium | Medium | High | Low friction |
| **Subscriptions** | Complex | Low initially | Medium | Highest friction |

The reality: Most successful games use all three in combination.

## Method 1: AdMob Banner & Interstitial Ads

### How AdMob Works

```
Player launches game
        ↓
Banner ad displays at bottom of screen
        ↓
Player views ad or taps it
        ↓
Google charges advertiser
        ↓
You earn revenue share (typically 60-70%)
```

### Getting Started: 5-Minute Setup

**Step 1: Create AdMob Account**
1. Visit [admob.google.com](https://admob.google.com)
2. Sign in with Google account
3. Click "Add your first app"
4. Platform: Select "Android" or "iOS"

**Step 2: Create Ad Units**

For each ad type you want to display:
- Banner ads (small, constant)
- Interstitials (full-screen, between gameplay)
- Rewarded ads (full-screen, give player reward)

AdMob generates unit IDs you'll use in your game code.

**Step 3: Integrate Google Mobile Ads SDK**

In Unity:

```bash
Window → TextMesh Pro → Import TMP Essential Resources
(This is usually a one-time setup)

Then:
Window → Package Manager → Add package from git URL
https://github.com/googleads/google-mobile-ads-sdk-unity.git
```

### Implementing Banner Ads

```csharp
using UnityEngine;
using GoogleMobileAds.Client;

public class AdManager : MonoBehaviour
{
    private BannerView bannerView;

    // Your AdMob IDs
    private const string BANNER_AD_ID = "ca-app-pub-xxxxxxxxxxxxxxxx/yyyyyyyyyyyyy";

    private void Start()
    {
        // Initialize Google Mobile Ads SDK
        MobileAds.Initialize(initStatus => {});

        // Create and load banner ad
        CreateBannerView();
    }

    private void CreateBannerView()
    {
        // Create new banner at bottom of screen
        bannerView = new BannerView(BANNER_AD_ID, AdSize.Banner, AdPosition.Bottom);

        // Subscribe to events
        bannerView.OnAdLoaded += OnBannerLoaded;
        bannerView.OnAdFailedToLoad += OnBannerFailedToLoad;

        // Load ad
        var request = new AdRequest();
        bannerView.LoadAd(request);
    }

    private void OnBannerLoaded()
    {
        Debug.Log("Banner loaded successfully");
    }

    private void OnBannerFailedToLoad(LoadAdError error)
    {
        Debug.LogError($"Banner failed: {error}");
    }

    public void DestroyBannerView()
    {
        if (bannerView != null)
        {
            bannerView.Destroy();
        }
    }
}
```

### Implementing Interstitial Ads

Interstitials are full-screen ads shown at natural breakpoints (level completion, menu transitions).

```csharp
public class InterstitialAdManager : MonoBehaviour
{
    private InterstitialAd interstitialAd;
    private const string INTERSTITIAL_AD_ID = "ca-app-pub-xxxxxxxxxxxxxxxx/yyyyyyyyyyyyy";

    private void Start()
    {
        LoadInterstitialAd();
    }

    public void LoadInterstitialAd()
    {
        var request = new AdRequest();
        InterstitialAd.Load(INTERSTITIAL_AD_ID, request, OnInterstitialLoaded);
    }

    private void OnInterstitialLoaded(InterstitialAd ad, LoadAdError error)
    {
        if (error != null || ad == null)
        {
            Debug.LogError("Interstitial failed to load");
            return;
        }

        interstitialAd = ad;
        interstitialAd.OnAdFullScreenContentClosed += OnInterstitialClosed;
    }

    public void ShowInterstitial()
    {
        if (interstitialAd != null)
        {
            interstitialAd.Show();
        }
        else
        {
            Debug.Log("Interstitial not ready");
        }
    }

    private void OnInterstitialClosed()
    {
        // Reload for next time
        LoadInterstitialAd();
    }
}
```

**Best placement for interstitials**:
- After level completion
- When player enters a menu
- Before in-game store
- Before pause screen

**Critical rule**: Never show interstitials mid-gameplay. Players hate it and will abandon your game.

### Revenue Expectations

```
Scenario: Puzzle game, 10,000 daily active users

Banner ads (always visible):
- Impressions/day: 10,000 × 5 sessions × 5 min average = 50,000
- eCPM: $1-3
- Daily revenue: 50,000 × $2.50 / 1,000 = $125
- Monthly: ~$3,750

Interstitials (3 per session average):
- Impressions/day: 10,000 × 3 = 30,000
- eCPM: $3-8
- Daily revenue: 30,000 × $5.50 / 1,000 = $165
- Monthly: ~$4,950

Combined AdMob Revenue: ~$8,700/month from 10K DAU
```

## Method 2: In-App Purchases (IAP)

### What Boosts IAP Revenue

Players purchase virtual items, power-ups, or convenience features. This works best when:
- Items are genuinely useful (not pay-to-win)
- Purchase is optional (players can progress free)
- Items offer clear value

### Common IAP Types

```
【Consumables】(one-time use)
- Extra lives: ¥100-500
- Coins/currency packages: ¥500-3,000
- Temporary boosts: ¥150-300

Expected conversion: 2-5% of DAU

【Non-Consumables】(permanent)
- Remove ads: ¥980-1,980
- Unlock character: ¥500-1,000
- Permanent boosts: ¥1,980-4,980

Expected conversion: 1-3% of DAU

【Subscriptions】(recurring)
- Premium pass: ¥1,980/month
- Battle pass: ¥980/season

Expected conversion: 0.5-2% of DAU
```

### Basic IAP Implementation (StoreKit 2 Framework)

```csharp
using UnityEngine;
using UnityEngine.Purchasing;
using UnityEngine.Purchasing.Extension;

public class IAPManager : MonoBehaviour, IDetailedStoreListener
{
    private IStoreController storeController;
    private IExtensionProvider extensionProvider;

    private const string REMOVE_ADS = "remove_ads";
    private const string COINS_100 = "coins_100";

    private void Start()
    {
        InitializePurchasing();
    }

    private void InitializePurchasing()
    {
        var builder = ConfigurationBuilder.Instance(StandardPurchasingModule.Instance());

        // Non-consumable
        builder.AddProduct(REMOVE_ADS, ProductType.NonConsumable);

        // Consumable
        builder.AddProduct(COINS_100, ProductType.Consumable);

        UnityPurchasing.Initialize(this, builder.Build());
    }

    public void PurchaseRemoveAds()
    {
        if (storeController != null)
        {
            storeController.InitiatePurchase(REMOVE_ADS);
        }
    }

    public void PurchaseCoins()
    {
        if (storeController != null)
        {
            storeController.InitiatePurchase(COINS_100);
        }
    }

    // Implement IDetailedStoreListener methods
    public void OnInitialized(IStoreController controller, IExtensionProvider extensions)
    {
        storeController = controller;
        extensionProvider = extensions;
        Debug.Log("IAP Initialized");
    }

    public void OnPurchaseComplete(Product product)
    {
        Debug.Log($"Purchase successful: {product.definition.id}");

        if (product.definition.id == REMOVE_ADS)
        {
            RemoveAdsFromGame();
        }
        else if (product.definition.id == COINS_100)
        {
            AddCoinsToPlayer(100);
        }
    }

    public void OnPurchaseFailed(Product product, PurchaseFailureReason failureReason)
    {
        Debug.LogError($"Purchase failed: {failureReason}");
    }

    private void RemoveAdsFromGame()
    {
        // Disable AdManager or stop showing ads
        PlayerPrefs.SetInt("AdsRemoved", 1);
        PlayerPrefs.Save();
    }

    private void AddCoinsToPlayer(int amount)
    {
        int currentCoins = PlayerPrefs.GetInt("PlayerCoins", 0);
        PlayerPrefs.SetInt("PlayerCoins", currentCoins + amount);
    }
}
```

### IAP Best Practices

```
✓ DO:
- Offer "remove ads" as first purchase
- Make consumables cheap (¥150-500)
- Show clear value before purchase dialog
- Never require IAP to progress normally
- Track purchase events for analytics

✗ DON'T:
- Use dark patterns to trick purchases
- Make game unplayable without IAP
- Show purchase prompts constantly
- Charge for progress gates
- Hide refund/restore options
```

## Method 3: Subscription Model

Subscriptions provide recurring revenue but require more thoughtful implementation.

### Battle Pass Model (Season-Based)

Most common in games: Players pay per season for exclusive rewards.

```
Season 1: "Summer Adventure" (¥980/season)
- 30 exclusive cosmetics
- 50 premium currency
- XP boost (2×)
- Season lasts 6 weeks

Expected: 3-8% of DAU purchases
10K DAU × 5% × ¥980 = ¥490K per season
```

### Subscription Tier (Premium Membership)

Monthly recurring revenue from optional membership:

```
Example: Game+

¥1,980/month includes:
- Daily bonus coins (500)
- Double XP weekends
- Exclusive cosmetics
- Early access to new levels

Expected: 0.5-2% recurring subscribers
10K DAU × 1% × ¥1,980 = ¥198K monthly
```

## Combining All Three: The Optimal Strategy

Most successful games use an intentional combination:

### Free Players (70% of users)
- See banner ads (generate ¥X)
- Occasionally see interstitials (generate ¥Y)
- Can play indefinitely without spending

### IAP Buyers (20% of users)
- See fewer ads if they purchase "Remove Ads"
- Purchase power-ups, cosmetics, currency
- One-time or occasional spenders

### Subscribers (10% of users)
- Premium membership for recurring revenue
- Battle pass for seasonal engagement
- Highest-value players

### Expected Revenue Breakdown (10K DAU)

```
AdMob (70% users seeing ads):
- Banner: ¥2,000/month
- Interstitials: ¥2,500/month
- Subtotal: ¥4,500/month

IAP (20% users):
- Avg spend per IAP buyer: ¥2,000/month
- 10K DAU × 20% × ¥2,000 = ¥40,000/month

Subscriptions (10% users):
- Avg subscriber: ¥1,980/month
- 10K DAU × 10% × 1 sub/user × ¥1,980 = ¥19,800/month

Total Monthly Revenue: ~¥64,300 (from 10K DAU)
```

## Monetization Timing

**DO NOT:**
- Show paywall immediately
- Force ads in first 5 minutes
- Require purchase to start playing

**DO:**
- Let players enjoy 20-30 minutes ad-free
- Show first IAP prompt after 30+ minutes
- Only show subscriptions after clear value demonstration

## Analytics: Measuring What Works

Track these metrics to optimize monetization:

```
【Ad Metrics】
- Impressions per user
- CTR (click-through rate)
- eCPM (revenue per 1K impressions)

【IAP Metrics】
- ARPU (average revenue per user)
- Conversion rate (% of users who purchase)
- Average purchase value
- LTV (lifetime value)

【Subscription Metrics】
- Churn rate (monthly cancellation %)
- Average subscriber lifetime
- Conversion rate from free to paid

Tools:
- Firebase Analytics (free, built-in)
- GameAnalytics (specialized for games)
- Unity Analytics
```

## Conclusion

Monetization success comes from balance: providing a great free experience while offering optional paid features that genuinely enhance gameplay.

The best approach combines:
1. **AdMob** for steady baseline revenue from all players
2. **IAP** for engaged players willing to spend
3. **Subscriptions** for committed long-term players

Start with ads, add IAP gradually, introduce subscriptions once you understand your player base.

Monitor analytics, iterate, and never sacrifice game quality for monetization. Players are sophisticated—they know when you're being greedy. Respect their experience, and they'll support your game financially.
