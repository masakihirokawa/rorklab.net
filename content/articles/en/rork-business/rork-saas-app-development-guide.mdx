## What Is SaaS? The Case for Mobile SaaS

SaaS refers to software that users pay for on an ongoing subscription basis — monthly or annually — rather than a one-time purchase. Think Notion, Figma, or Spotify.

For mobile apps, SaaS has some compelling advantages:

- **Recurring revenue**: Once you acquire a user, you earn every month they stay subscribed
- **Scalability**: You can grow your user base without proportional increases in cost
- **Update value**: Regular new features give users reasons to stay

Examples of mobile SaaS you could build with Rork:

- Task management or project tracking apps ($3–$10/month)
- Language learning or study support apps
- Health and fitness tracking apps
- Personal finance and expense management apps
- Freelancer invoice and estimate tools

**Why Rork is well-suited for SaaS**: Rork has strong native integrations with Supabase and Stripe, meaning you can implement the three pillars of SaaS — authentication, database, and payments — through prompt-driven development. What used to take weeks can now be built in hours.

## Core Architecture of a SaaS App

Here's a typical architecture for a mobile SaaS built with Rork:

```
[Frontend: Rork (React Native / SwiftUI)]
        ↓
[Auth: Supabase Auth / Firebase Auth]
        ↓
[Database: Supabase (PostgreSQL)]
        ↓

        ↓
[Backend API: Supabase Edge Functions]
```

### Authentication

User account management is the foundation of any SaaS product. Rork makes it straightforward to implement email/password auth, Google Sign-In, and Apple Sign-In via Supabase Auth.

```javascript
// Sign up a new user with Supabase Auth
import { supabase } from './supabaseClient';

const signUp = async (email, password) => {
  const { data, error } = await supabase.auth.signUp({
    email,
    password,
  });
  if (error) {
    console.error('Sign-up error:', error.message);
    return null;
  }
  // data.user contains the new user's info
  console.log('User created:', data.user.id);
  return data.user;
};
```

### Database Design

Using Supabase (PostgreSQL), here's a minimal schema to get a SaaS off the ground:

```sql
-- User profiles (linked to Supabase Auth)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users PRIMARY KEY,
  display_name TEXT,
  plan_type TEXT DEFAULT 'free', -- 'free' | 'pro' | 'premium'
  subscription_end_date TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Content table (example: task management app)
CREATE TABLE tasks (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  title TEXT NOT NULL,
  is_completed BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Row Level Security (users can only access their own data)
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users can access own tasks" ON tasks
  FOR ALL USING (auth.uid() = user_id);
```

### Subscription Billing

For monetizing a mobile SaaS, you have two main options:

- **Stripe (web billing)**: Flexible pricing with no App Store fee (30%), card payments
- **StoreKit 2 / RevenueCat (In-App Purchase)**: Native App Store billing, high user trust

For early-stage solo development, starting with Stripe and adding In-App Purchase as you grow is a practical approach.

## Building a SaaS MVP with Rork: 5 Steps

### Step 1: Validate Your Idea and Define Your Target

The number one reason SaaS products fail is building before validating. Before writing a single line of code, clarify:

- **Who has the problem and what is it?** (e.g., freelancers spend hours every month creating invoices manually)
- **Why is it worth paying monthly?** (e.g., 5 hours of work becomes 10 minutes)
- **How is it different from existing apps?** (e.g., localized for Japanese tax law, PDF export, blue-form accounting support)

You can send a prompt like this to Rork to quickly scaffold your MVP:

```
Build a freelance invoice management SaaS app for mobile.
Core features:
1. Client management (name, address, email)
2. Invoice creation (line items, amounts, tax, totals)
3. PDF export
4. Payment status tracking (unpaid / paid)
5. Monthly subscription at $3/month (free plan: 3 invoices/month)

Implement with React Native + Supabase + Stripe.
```

### Step 2: Build the Auth Screens

Ask Rork to create login and sign-up screens using Supabase Auth. Apple Sign-In is required by App Store guidelines when you offer any third-party authentication, so it's worth including from the start.

### Step 3: Implement the Core Feature Set

This is the "value engine" of your SaaS. For a task manager, that means CRUD operations. For a fitness app, it means logging and data visualization. Keep it focused — feature creep is the enemy of a fast MVP.

### Step 4: Add Premium Plans and Paywalls

Define the boundary between free and paid, then lock premium features behind it:

```javascript
// Plan access checker
const checkPlanAccess = (userPlan, requiredPlan) => {
  const planLevels = { free: 0, pro: 1, premium: 2 };
  return planLevels[userPlan] >= planLevels[requiredPlan];
};

// Example: gate a premium feature
const handleExportPDF = () => {
  if (!checkPlanAccess(user.plan_type, 'pro')) {
    // Show upgrade prompt
    setShowUpgradeModal(true);
    return;
  }
  // Proceed with PDF export
  exportToPDF(invoice);
};
```

### Step 5: Test on TestFlight and Submit to the App Store

Once your MVP is ready, test it via TestFlight and submit to the App Store. If you're on Rork Max, the 2-click publish feature lets you submit without Xcode.

## Monetization Strategy: Building Stable Monthly Revenue

### Designing a Freemium Model

For a solo SaaS, a **freemium model** (free tier + paid plans) is usually the most effective for user acquisition. The key principles:

- The **free plan should deliver real value, but leave users wanting more** (storage limits, feature caps, etc.)
- Create **natural upgrade triggers** — users should feel the limitation organically, not artificially
- **Don't overthink pricing** — for consumer apps, $3–$10/month is a good starting range

Sample pricing for a task management app:

- **Free**: Up to 20 tasks, 1 project
- **Pro ($4/month)**: Unlimited tasks, 10 projects, reminders
- **Premium ($8/month)**: All features + team sharing + calendar sync

### Reducing Churn

Churn rate — the percentage of users who cancel each month — is one of the most important health metrics for SaaS. To keep it low:

- **Invest in onboarding**: A first-run walkthrough that delivers the "aha moment" fast dramatically improves retention
- **Send progress notifications**: "You completed 5 tasks last week!" gives users a reason to stay engaged
- **Offer annual billing**: A 2-month discount for annual payment can cut churn significantly

## Common Errors and How to Fix Them

### Supabase RLS Returning Empty Data

After enabling Row Level Security, data queries return empty arrays. This is almost always a missing or misconfigured policy.

```javascript
// ❌ Problem: RLS enabled but no policy defined — returns []
const { data } = await supabase.from('tasks').select('*');

// ✅ Fix: Add a policy in the Supabase dashboard
// Table Editor > tasks > RLS Policies
// Enable RLS + Add "Allow all to authenticated users" policy
```

### Stripe Webhooks Not Firing in Production

In development you use the Stripe CLI locally. In production, you need to register your actual endpoint URL in the Stripe Dashboard. Use a Supabase Edge Function as the webhook handler and register its URL in Stripe.

### App Store Rejection for Subscription Features

Apple's guideline 3.1.2 requires In-App Purchase for digital content subscriptions that are consumed within the app. Stripe direct billing is permitted for services delivered outside the app (e.g., web services). Make sure you understand which category your SaaS falls into before submitting.

## FAQ

**Q1. Can you actually build a real SaaS with Rork, or is the backend too complex?**

Rork combined with Supabase handles authentication, database, and API logic entirely through prompts. Supabase manages most of the backend infrastructure automatically. For CRUD-heavy business apps — the most common type of SaaS — Rork and Supabase are an excellent fit.

**Q2. Is the App Store review process strict for subscription apps?**

Subscription functionality itself isn't harder to get approved. What matters is compliance: clearly state the free trial period (if any), include cancellation instructions, and display pricing accurately. Mentioning "App Store guideline compliant" in your Rork prompts helps ensure proper implementation.

**Q3. How long does it take to generate revenue from a solo SaaS?**

Apps targeting a specific niche can reach $100–$500/month in recurring revenue within 3–6 months of launch. The biggest factor is usually **discoverability** — App Store SEO, social media presence, and community engagement — not just the quality of the app.

**Q4. Do I need to form a company to run a SaaS business?**

At small revenue levels, operating as a sole proprietor is perfectly fine. Once annual revenue starts to grow meaningfully, consulting an accountant to find the optimal business structure is a smart move.
