## Pre-Launch Checklist: What You Need Before Publishing

Success requires thorough preparation before publishing. Complete these steps before attempting to upload.

### 1. Create an Apple Developer Account

App Store publication requires **Apple Developer Program** membership.

**Registration Steps**:

- Visit [Apple Developer](https://developer.apple.com)
- Click "Account" in the top right
- Sign in with your Apple ID (create one if needed)
- Proceed to Developer Program registration
- Pay the **$99 annual fee** with a credit card
- Receive confirmation email with access code
- Gain access to App Store Connect

**Requirements**:
- Apple ID (iCloud account)
- Credit card for $99 annual fee
- Personal information (name, address)

### 2. Configure App Store Connect Account

After Apple Developer Program registration, log into [App Store Connect](https://appstoreconnect.apple.com).

**Key Setup Tasks**:

- Navigate to "Apps" section
- Create new app entry
- Configure **Bundle ID** (example: `com.yourcompany.appname`)
- Set **App Privacy Policy** URL
- Create **SKU** identifier (internal tracking code)
- Enter basic app information

### 3. Create Google Play Developer Account

Similarly, Google Play Store registration is required.

**Registration Steps**:

- Visit [Google Play Console](https://play.google.com/console)
- Sign in with Google Account
- Complete Developer Program registration (**$25 one-time fee**)
- Receive confirmation email
- Gain full access to publish

**Requirements**:
- Google Account
- Credit card
- Personal information

### 4. Prepare Core App Information

Required for both platforms:

- **App Name**: Official title displayed on stores
- **Description**: 160 characters max, conveying app value
- **Screenshots**: iOS needs 5, Android needs up to 8 (create with Figma)
- **Preview Video** (optional): 30 seconds maximum, showing app usage
- **Age Rating**: 17+, 12+, 4+, or appropriate category
- **Category**: Business, Education, Games, Utilities, etc.
- **Keywords**: Up to 15 comma-separated search terms

Incomplete information delays store approval, so prepare everything before beginning upload.

## Apple Developer Setup and Configuration

### Detailed Developer Program Registration

1. **Verify Apple ID**: Required for registration. Create an iCloud account if you don't have one
2. **Pay Annual Fee**: $99 credit card payment (approximately $100 USD)
3. **Select Account Type**: Choose "Individual" (personal) or "Company" (business)
4. **Verify Email**: Apple sends confirmation email. Click verification link
5. **Wait for Activation**: Apple Developer Program activation typically occurs within 3 business days

### Complete App Store Connect Setup

After logging into App Store Connect:

**Create New App**:
- "My Apps" → "+" button → "New App"
- Platform: Select iOS
- App Name: Official title (displayed on App Store)
- Primary Language: Japanese or English
- Bundle ID: Reverse domain format (example: `jp.example.myapp`)
- SKU: Internal identifier (unchangeable after creation)

**Configure App Provider**:
- Navigate to "App Information" → "Provider"
- Enter your name (personal) or company name (business)

**Address Encryption Requirements**:
- Go to "App Information" → "Encryption"
- Select "Yes" if your app includes VPN, messaging, email, or financial features
- Select "No" for content delivery apps

**Set Privacy Policy**:
- Navigate to "App Privacy" section
- Provide Privacy Policy URL (hosted on Google Sites, Notion, etc.)
- This is mandatory for App Store approval

## Building and Uploading with Rork Max

This is where Rork Max delivers extraordinary value. What traditionally required Xcode and manual signing now happens automatically.

### Step 1: Generate Your App in Rork Max

1. Log into Rork Dashboard
2. Click "Create New App"
3. Describe your app concept (Companion AI provides guidance)
4. Finalize design and features
5. Click "Generate App"
6. Upon completion, access the generated app from Dashboard

### Step 2: Access Publishing Settings

1. Open your completed app details
2. Click "Publishing" in the left sidebar
3. Locate "Publish to App Store" section

### Step 3: Enter Apple Developer Credentials

1. Gather Apple Developer Program registration details
2. On Rork Max Publishing page, click "Connect Apple Account"
3. Enter Apple ID and password (or 2-factor authentication code)
4. "Team ID" auto-populates

**Security Note**: Rork Max uses official Apple APIs. Your credentials are processed securely and never stored on servers.

### Step 4: Sync App Store Connect Settings

1. Enter "Bundle ID" (example: `com.yourcompany.appname`)
2. Confirm "App Name" (as displayed on App Store)
3. Verify "Version Number" (defaults to 1.0.0 for first release)
4. Enter "SKU" (internal identifier)

### Step 5: Build & Automatic Signing

1. Click "Build & Sign for App Store"
2. Rork Max automatically executes in background:
   - React Native code compilation
   - iOS source code generation
   - Apple developer certificate signing
   - IPA file (iOS binary) generation
3. Progress indicator displays; typically completes in 2-5 minutes

### Step 6: Upload to App Store Connect

1. After build completion, "Upload to App Store Connect" button activates
2. Click to automatically upload IPA to App Store Connect
3. Upload completes within seconds
4. "View in App Store Connect" link opens the App Store Connect dashboard

### Step 7: Final Verification in App Store Connect

In App Store Connect, ensure the following are complete:

- **App Icon**: 1024×1024px PNG, no transparency
- **Screenshots**: 5 per language (iPhone), 5-8 per language (iPad)
- **Description**: 160 characters maximum
- **Age Rating**: Select 17+, 12+, 4+, or appropriate category
- **Category**: Choose from available categories
- **Keywords**: Up to 15 comma-separated terms
- **Preview Video** (optional): Maximum 30 seconds

All fields are mandatory for App Store review approval.

## Publishing to Google Play Store

Google Play publication is similarly automated, and Google's review process is typically less stringent than Apple's.

### Step 1: Create App in Google Play Console

1. Log into [Google Play Console](https://play.google.com/console)
2. Click "Create app"
3. App name: Enter your app's title
4. Default language: Select Japanese or English
5. App type: Choose "App" or "Game"
6. Click "Create app"

### Step 2: Open Google Play Settings in Rork Max

1. From Rork Max Dashboard, select your app
2. Click "Publishing" → "Publish to Google Play"
3. Click "Connect Google Account"

### Step 3: Google Play API Authentication

1. Sign in with your Google Account (same one used for Google Play Developer Program)
2. Click "Allow" to grant Rork Max Google Play API access
3. OAuth token automatically generates and transmits to Rork Max

### Step 4: Generate Android Bundle & Sign

1. Enter "Package Name" (example: `com.yourcompany.appname`)
2. Confirm "App Version" (defaults to 1.0)
3. Click "Build & Sign for Google Play"
4. Rork Max executes in background:
   - React Native + Android code compilation
   - Android App Bundle (AAB) generation
   - Release Key signing for Google Play
5. Typically completes in 3-8 minutes

### Step 5: Upload to Google Play Console

1. Upon build completion, click "Upload to Google Play"
2. Rork Max automatically uploads AAB to Google Play Console
3. App appears in Internal Testing track (for initial testing)

### Step 6: Complete Google Play Console Configuration

In Google Play Console, complete the following:

- **App Icon**: 512×512px PNG format
- **Feature Graphic**: 1024×500px (displayed on store listing)
- **Screenshots**: Maximum 8, with separate sets per language
- **Short Description**: Maximum 80 characters
- **Full Description**: Maximum 4,000 characters
- **Content Rating**: Select 3+, 6+, 12+, 16+, or 18+
- **Category**: Select from 30+ available categories
- **Privacy Policy**: URL is mandatory (required even for apps without personal data collection)

### Step 7: Submit for Publication

1. In Google Play Console, navigate to "Testing" → "Internal testing"
2. Verify app functions correctly in testing track
3. Proceed to "Release management" → "Production"
4. Click "Create new release"
5. Click "Review" for final verification
6. Click "Release" to submit for approval

Google Play typically approves within 2-24 hours.

## Common Issues and Solutions

### 1. Bundle ID Already in Use (App Store)

**Error Message**: "Bundle ID is already in use"

**Cause**: Another app uses the same Bundle ID

**Solution**:
- Modify Bundle ID (example: `com.yourcompany.appname2`)
- Or delete previously published app from App Store Connect

### 2. Apple Developer Program Confirmation Email Missing

**Cause**: Incorrect email configuration or spam folder filtering

**Solution**:
- Verify Apple ID email settings
- Check spam/junk email folder
- Contact Apple Developer Support (1 business day response)

### 3. Rork Max Cannot Build App

**Error Message**: "Build failed: Xcode compilation error"

**Cause**: Code issues (infinite loops, undefined variables, etc.)

**Solution**:
- Ask Rork Companion: "Build failed. What's the issue?"
- AI automatically diagnoses and repairs code
- Click "Build & Sign" again

### 4. TestFlight Upload Fails

**Cause**: Missing app icon, screenshots, or description

**Solution**:
- Verify all required fields in App Store Connect are complete
- Confirm app icon is 1024×1024px PNG without transparency
- Verify screenshots are configured per language

### 5. Google Play Shows "Manifest Errors"

**Error Message**: "AndroidManifest.xml syntax error"

**Cause**: Android manifest generation failed

**Solution**:
- Click "Re-generate Android Build" in Rork Max Dashboard
- Or contact Rork Support (typical response time: few hours)

### 6. App Store Review Rejects with "Guideline 4.3"

**Error Message**: "This app contains embedded executable code"

**Cause**: App dynamically downloads and executes code (Apple prohibits this)

**Solution**:
- Tell Rork Companion: "Apple rejected with Guideline 4.3"
- AI automatically modifies problematic code
- Resubmit to App Store

### 7. Google Play Rates App as "Low Quality"

**Cause**: App crashes frequently or becomes unresponsive

**Solution**:
- Check Sentry crash logs for error details
- Request Rork Companion to fix issues
- Test thoroughly in TestFlight
- Republish updated version
