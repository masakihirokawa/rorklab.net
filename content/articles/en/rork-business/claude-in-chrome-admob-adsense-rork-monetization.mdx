## The Revenue Generation Pipeline

### Understanding Ad Formats and Revenue Potential

Your Rork app can generate revenue through multiple ad formats, each with distinct characteristics:

| Ad Format | Revenue Range | User Experience | Best Placement |
|-----------|--------------|-----------------|----------------|
| **Banner Ads** | $1-5 eCPM | Low (constant display) | Home screen, lists |
| **Interstitials** | $3-10 eCPM | Medium (interrupts) | Level completion, tab switches |
| **Rewarded Ads** | $5-15 eCPM | High (opt-in) | Bonus rewards, gacha |
| **Custom Networks** | Variable | Medium-High | Strategic placement |

Understanding these differences is the first step toward revenue optimization.

### Mediation Network Optimization

Rather than relying solely on Google AdMob, set up mediation to source ads from multiple networks and automatically select highest-paying options.

```
"In AdMob Mediation, create mediation group:

Group name: 'all_formats_mediation'
Formats: Banner, Interstitial, Rewarded

【Network Waterfall】
1. Google AdMob Network (priority)
   - Algorithm: Auto-select highest bid

2. Meta Audience Network
   - CPM floor: $2

3. Pangle (ByteDance)
   - CPM floor: $1.50

4. Unity Ads
   - CPM floor: $1.00

After setup, report:
- Each network's performance
- Fill rate by network
- Average eCPM per network
- Recommendations for floor adjustment"
```

Proper mediation configuration typically increases eCPM by 20-40% versus AdMob alone.

## Firebase Analytics Integration

Connect AdMob data with Firebase Analytics to understand the relationship between user behavior and monetization.

### Monitoring ARPU (Average Revenue Per User)

```
"In Firebase Analytics, confirm:

【Basic User Metrics】
- DAU (daily active users)
- MAU (monthly active users)
- New user count

【Revenue Metrics with AdMob】
- ARPU: (Total estimated revenue) / DAU
- ARPPU: Revenue / Paying users (if applicable)

- Revenue: $300
- ARPU: $0.03

【Trend Analysis】
- 7-day ARPU trend
- ARPU growth/decline pattern
```

Higher ARPU indicates better monetization. Continuous ARPU improvement is the north star for app profitability.

## Common Questions & Troubleshooting

### Q: Banner vs. Interstitial—which drives more revenue?

**Answer**: It depends on your app, but generally:

| Metric | Banner | Interstitial |
|--------|--------|--------------|
| eCPM | $1-5 | $3-10 |
| User Experience | Low friction | Medium friction |
| CTR | 0.5-1% | 2-5% |

**Recommendation**: Combine both. Use banners for continuous revenue, interstitials at natural breakpoints (level completion, screen changes). Test 60:40 or 70:30 ratios, then optimize based on results.

### Q: eCPM is declining. How do I fix it?

**Priority-ordered solutions**:

1. **Review mediation setup**
   - Add Meta, Pangle, and other networks
   - Adjust CPM floors appropriately
   - Expected result: 20-40% eCPM increase

2. **Improve traffic quality**
   - Enable fraud filters in AdMob
   - Block low-quality traffic sources

3. **Geographic targeting**
   - Attract traffic from high-eCPM countries (US, Canada, Nordic)
   - Naturally prioritizes quality markets

4. **Ad format optimization**
   - Increase rewarded ads (highest eCPM)
   - Test different interstitial placements
   - A/B test placement changes

### Q: I have multiple Rork apps. How do I monitor all of them?

**Two approaches**:

**Method 1: Separate tasks per app**
```
- Task 1: App A (8:00 AM)
- Task 2: App B (8:15 AM)
- Task 3: App C (8:30 AM)
```

**Method 2: Consolidated dashboard**
```
- Single task aggregates all apps
- Trade-off: Loses detail for each app
- Better for high-level overview
```

### Q: My AdMob account was restricted. Can Claude in Chrome prevent this?

Unfortunately no, Claude in Chrome cannot prevent policy violations. However, it can help you detect issues early:

```
"Weekly check: Access AdMob Policy Center.
Report any warnings or restrictions immediately."
```

**Prevention strategies**:
- Understand your user demographic (age restrictions for kids' apps)
- Implement click fraud detection
- Regularly review AdMob policy updates
- Never allow invalid traffic

### Q: Users are viewing too many rewarded ads. How do I detect ad fatigue?

```
"In Firebase Analytics, create user segments:
- Daily rewarded ad views per user
- Compare retention rates for:
  - Low viewers (<3/day)
  - Medium viewers (3-5/day)
  - High viewers (>5/day)

If high viewers show lower retention,
implement viewing limits."
```

Implement cooldown periods between rewarded ads to maintain user satisfaction while preserving revenue.
