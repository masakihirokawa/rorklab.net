## Freemium Tier Strategy: Designing Your Paywall

Your free tier is your acquisition funnel. Carefully design it to:
- Give users genuine value (they keep using it)
- Create clear friction at premium features (they understand why they should pay)

### Example: Scheduler AI (Productivity App)

**Free Tier (Forever Free)**:
- 1 AI scheduling suggestion per month
- 1 calendar sync
- Email and web support access via FAQ
- Standard notification capabilities
- Basic reporting

**Pro Tier (¥1,980/month or ¥19,800/year)**:
- Unlimited AI suggestions
- 5 synchronized calendars
- Priority email support (24-hour response)

- 1-click Google Calendar sync

**Premium Tier (¥4,980/month or ¥49,800/year)**:
- Everything in Pro
- API access for developers
- Custom AI prompt engineering

- Advanced integrations

### Revenue Expectations by Tier


## Paywall Optimization: Psychology and Timing

The difference between a struggling subscription app and a successful one often comes down to paywall design and timing.

### When to Show the Paywall

**Premature**: Day 1-2
- Users still exploring
- Too early to appreciate value
- Conversion: <1%

**Optimal**: Day 3-5
- Users have tried features
- Value proposition is clear
- Friction point is obvious
- Conversion: 5-15%

**Too Late**: Week 2+
- Users have already found free workarounds
- Expectation of free service entrenched
- Conversion: <3%

**Implementation in Rork Max**:
```
Trigger paywall after:
1. User completes 3 feature interactions, OR
2. User attempts to access Pro feature, OR
3. 5 days have passed since installation

Whichever comes first.
```

### Paywall Message Design

The copy matters immensely. Test multiple approaches:

**Approach 1: FOMO (Fear of Missing Out)**
```
"Unlock Unlimited AI Suggestions
See what Pro users are discovering daily

Join 50,000+ productivity experts using Pro"
```

**Approach 2: Value Comparison**
```
Free: 1 suggestion/month
Pro: Unlimited suggestions

Time saved per week:
Free: 15 minutes
Pro: 3+ hours

That's 12+ hours monthly"
```

**Approach 3: Social Proof**
```
"Join 50,000+ users upgrading to Pro

Average rating: ★★★★★ (8,247 reviews)

'This saved my life.' - Sarah M.
'Best productivity app.' - James L."
```

**Approach 4: Simple Value Promise**
```

Unlimited AI Scheduling
+ Priority Support
+ Advanced Analytics
+ Team Collaboration

¥1,980/month
(or ¥19,800/year, save 2 months)

[Start 7-day free trial]

```

Most successful apps use Approach 4 (simple) combined with Approach 3 (social proof).

## Retention: Keeping Subscribers from Churning

Acquiring a subscriber at ¥1,980/month is only the beginning. Keeping them is what separates sustainable businesses from flash-in-the-pan successes.

### Churn Prevention Strategies

**Strategy 1: Regular Value Communication**
```
Monday: Weekly productivity metrics email
"You saved 4.5 hours this week using AI scheduling"

Thursday: Feature spotlight
"Did you know? Our new team collaboration feature..."

Saturday: Community digest
"Here's what top productivity users are doing..."
```

**Strategy 2: Feature Releases for Paid Tiers**
```
March: New feature for Pro users only
April: New feature for Premium users
May: Roll all features to Pro + new Premium feature

Pattern: Paid tiers always have something new to explore
```

**Strategy 3: Retention Milestones**
```
Day 30: "You've saved 15+ hours in your first month!"
Day 90: "You've rescheduled 100+ events. Keep going!"
Day 365: "Annual reward: 30-day extension free"
```

**Strategy 4: Exit Survey for Canceling Users**
```
When subscriber cancels:
"Help us improve. Why are you canceling?
- Missing features
- Too expensive
- Found alternative
- No longer needed
- Other [text field]"

Result: Understand why people leave,
fix top issues for existing users
```

### Target Churn Rates by Month

```
Month 1: 10-15% (normal trial abandonment)
Month 2-3: 5-8% (feature explorers leaving)
Month 4-6: 2-3% (reached equilibrium)
Month 7+: 1-2% (true sticky users)

Healthy annual churn: 25-30%
Poor retention: >40% annual churn
Excellent retention: <20% annual churn
```

```
Cohort 1: Launch at ¥1,980

- If <5%: Price might be too high

Cohort 2: A/B test
- 50% see ¥1,980

Cohort 3: Implement optimal price
```

**Typical result**: ¥1,980-2,480 is the sweet spot for consumer productivity apps.

## App Store Compliance: Apple's Subscription Requirements

### What Apple Requires

**1. Clear pricing disclosure**
- Annual cost must be prominently displayed
- Free trial duration must be shown
- Billing frequency must be explicit

**2. Easy cancellation**
- Users must cancel within the app
- Instructions must be clearly available
- Cancellation process: maximum 2 taps

**3. Privacy policy**
- Must explain data collection
- Explain how payment information is handled
- Link must be functional from App Store listing

**4. No misleading claims**
- Don't claim "unlimited" if limited
- Trial period must be genuine (you must deliver value)
- Refund policy must be accurate

### Common Rejection Reasons

- Paywall appears before user can try anything
- "Subscribe now" button is manipulative (dark patterns)
- Cancellation is harder than signup
- Privacy policy is missing or inaccessible
- Trial period is too short or doesn't deliver value

### Approval Rate

Apps with well-designed subscription systems:
- First submission approval: 90%+
- If rejected: Simple fixes (usually messaging) get approval on resubmission
