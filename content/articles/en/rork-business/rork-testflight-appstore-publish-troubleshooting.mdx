## The Path from TestFlight to App Store

Getting a Rork-generated app into users' hands requires navigating Apple's rigorous review process. From beta testing on TestFlight to public App Store release, many developers encounter obstacles. This guide covers common issues and practical fixes for beginners. If you encounter Xcode build errors before publishing, the [Complete Guide to Fixing Rork App Build Errors](/articles/rork-dev/rork-app-build-error-swiftui-fix-guide) is a helpful reference.

## Trouble 1: Provisioning Profile Errors

### Symptoms

"Signing for Identifier failed" or "Provisioning Profile not found" appears during upload to App Store Connect.

### Root Causes

Your provisioning profile may be invalid due to:

- Missing or deleted App ID
- Expired signing certificate (renewed annually)
- Outdated device registration or deleted UDIDs

### Solutions

**Step 1: Regenerate Provisioning Profile**

1. Log into Apple Developer Center (developer.apple.com)
2. Navigate to "Certificates, IDs & Profiles"
3. Select "Profiles," find your app's profile, and delete it
4. Click "+" to create new profile
5. Choose "iOS App Distribution" (for App Store release)
6. Specify your app's Bundle ID
7. Download and drag-drop into Xcode

**Step 2: Verify in Xcode**

Open Project → Build Settings and confirm:

```
Signing & Capabilities:
  Team: Your Apple Team
  Provisioning Profile: Automatic selection (or your new profile)
```

Ensure "Automatically manage signing" is enabled.

## Trouble 2: Signing Certificate Expiration

### Symptoms

"Certificate has expired" appears in Xcode.

### Root Causes

Apple signing certificates expire after one year. Expired certificates prevent successful code signing.

### Solutions

**Generate a New Certificate**

1. Go to Developer Center → Certificates
2. Delete expired certificate
3. Click "+" to create new one
4. Select "Apple Development" (development) or "Apple Distribution" (release)
5. Upload a CSR (Certificate Signing Request)
6. Download and drag into Keychain Access

**Add to Keychain**

```bash
# Double-click downloaded .cer file
open ~/Downloads/certificate.cer
```

Xcode automatically recognizes the new certificate afterward.

## Trouble 3: App Store Review Rejection

### Symptoms

TestFlight works, but App Store submission gets rejected with a detailed explanation email.

### Common Rejection Reasons and Fixes

| Reason | Fix |
|---|---|
| Old or incomplete build | Rebuild with latest version and re-upload |
| Missing or incomplete privacy policy | Add policy URL to "Privacy Policy" field in App Store Connect |
| Unclear hardware/framework dependencies | Complete "App Privacy" section accurately |
| Missing or invalid test account info | Provide valid test email/password in "Test Information" |
| Incorrect in-app purchase implementation | Properly configure StoreKit 2 and pricing in "Pricing and Availability" |

**Privacy Policy Template**

```
Privacy Policy

This app does NOT collect:
- Personal information (name, email, etc.)
- Location data
- Health data
- Contacts

This app collects:
- Anonymous usage statistics (Amplitude, Google Analytics)
```

## Trouble 4: App Store Screenshot Requirements

### Symptoms

Reviewer feedback: "Screenshots do not show the app in use." Unclear screenshot standards.

### Apple's Screenshot Standards

- **Consistent language**: All screenshots in Japanese for Japanese apps
- **Real device or simulator**: Actual app screenshots, not mockups
- **Show core features**: Main screen, onboarding, key functionality in order
- **Brief captions**: 3-5 lines max per screenshot

### Taking Screenshots with Xcode

```bash
# Option 1: Simulator screenshot (manual)
# Press Cmd+S in Xcode simulator, screenshot saves to Desktop

# Option 2: Automated screenshot capture
osascript -e '
tell application "System Events"
  key code 19 using {command down, shift down}  -- Cmd+Shift+5
end tell
'
```

## Trouble 5: Build Becomes Old or Invalid

### Symptoms

"This build is old or has become invalid" appears in App Store Connect.

### Root Causes

- Build uploaded but not submitted to review within 90 days
- Signing certificate used for build has expired

### Solutions

Create and upload a fresh build:

```bash
# In Xcode: Product → Archive
# Or from command line:
xcodebuild archive ...
```

Upload to App Store Connect via Organizer → "Upload to App Store Connect."

## Trouble 6: TestFlight Internal vs. External Testing

### Symptoms

"Why can't I invite external testers?" Tester invitations fail.

### Root Causes

- Build still processing on TestFlight (can take 24 hours)
- External test build not yet approved
- Tester email already linked to different Apple ID

### Solutions

**For Internal Testing:**

1. TestFlight → Internal Testing tab
2. Add testers (team members with Apple IDs)

**For External Testing:**

1. TestFlight → External Testing tab
2. Wait for build to show "Ready to Test"
3. Up to 10,000 external testers supported
4. Generate and share public test link

If approval is slow, contact Apple:

```
App Store Connect → My Apps → TestFlight → External Testing → Help
```

## Frequently Asked Questions

### Q1: How long does App Store review take?

**A:** Typically 1-3 business days. Rejections add processing time. Budget extra time for first submission and allow adequate TestFlight testing beforehand.

### Q2: How soon can I resubmit after rejection?

**A:** Immediately after addressing reviewer feedback—either with the same build or a new one. Submissions expire 30 days after rejection if you don't respond.

### Q3: Do I need special procedures for China's App Store?

**A:** Yes. China requires ICP registration, strict privacy policies, and data center location disclosure. Professional guidance is recommended for China distribution.
