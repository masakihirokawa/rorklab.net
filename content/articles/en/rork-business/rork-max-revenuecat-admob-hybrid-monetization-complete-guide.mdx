## Introduction

When my Rork Max app hit 100k monthly downloads, I faced a choice: ads only, subscriptions only, or both?

Initially, ads were fine. One million impressions daily. Six figures monthly revenue.

Then I noticed: **the most active users, the ones who open the app daily, were the most frustrated by ads.**

I tried hybrid monetization.

- Free users see reasonable ads
- Ad-averse users can subscribe
- Premium users get ad removal plus exclusive features

Result: **2.3x revenue increase.** Ad ARPU dropped, but subscription revenue more than compensated. High-value users migrated to subscriptions; remaining free users' average ARPU actually rose.

This article covers the complete implementation.

## Why Hybrid? — The Case for Both

Most developers think in binary: ads or subscriptions. Together, they're exponentially more powerful.

### Ads Only: Simple, But Limited Upside

**Pros**:
- Simple implementation (just Google Mobile Ads SDK)
- Minimal user friction
- Helps early adoption

**Cons**:
- Low ARPU (¥2-5 per user)
- ARPU decreases as you scale (geographic mix)
- Reaching ¥1M+ monthly revenue is nearly impossible

Real example: My app's ad ARPU fell from ¥3.50 in January to ¥2.80 in March, despite 40% user growth.

### Subscriptions Only: High Value, Low Adoption

**Pros**:
- High per-user revenue (¥480/month = ¥5,760 annual LTV)
- Loyal, engaged user base

**Cons**:
- Initial conversion is brutal (0.3-0.5% of free users)
- No free trial means fewer downloads
- Hard to compete on subscriptions alone if product isn't unique

### Hybrid: Multiplicative Effect

**Pros**:
- Free tier drives scale (ad monetization)
- Quality users convert to premium (high LTV)
- Users feel they have a choice (fairness perception)
- Total revenue > either approach alone

**Cons**:
- Complex implementation (RevenueCat + Billing API + AdMob)
- Stricter App Store review (guidelines 4.3, 3.1.1)
- Poor UX can feel exploitative

**Golden rule**: Separate "ad removal" subscription from "extra features" subscription.

## System Architecture — Rook Max + RevenueCat + AdMob + Firebase

End-to-end user flow:

```
User Opens App
  ↓
Firebase Authentication
  ↓
├─ RevenueCat API: Check subscription status
│   ├─ if Premium → Skip ad networks
│   └─ if Free → Load ads
├─ Google Mobile Ads SDK
│   ├─ Interstitial (full-screen)
│   ├─ Rewarded (watch video)
│   └─ Native (feed-integrated)
└─ Firestore: Persist user state & purchase history
```

### Role of Each Component

- **Firebase**: User authentication + Firestore persistence
- **RevenueCat**: Unified subscription management (App Store + Google Play)
- **Google Mobile Ads SDK**: Ad serving and CPM optimization
- **Rork Max**: App generation and code customization

## RevenueCat Setup — Entitlements and Products

### Step 1: Create RevenueCat Project

1. Sign up at [RevenueCat.com](https://revenuecat.com)
2. Create new app project
3. Connect App Store Connect API key and Google Play Service Account

```
Project Settings
├─ iOS: App Store Connect API Key (JWT format)
└─ Android: Google Play Service Account JSON
```

### Step 2: Define Entitlements

Entitlements represent user permissions. Create:

- `premium`: Ad removal + extra features
- `supporter`: Optional one-time donation

### Step 3: Create Products

Define subscription products for each platform.

**iOS**:
```
Product ID: com.yourcompany.appname.premium_monthly
Entitlements: premium
```

**Android**:
```
Product ID: com.yourcompany.appname.premium_monthly
Entitlements: premium
```

**Critical**: Use identical product IDs across platforms. RevenueCat auto-maps them.

### Step 4: Integration into Rork Max

When generating your Rork Max app, explicitly request RevenueCat integration:

```swift
import RevenueCat

// Initialize RevenueCat
Purchases.configure(withAPIKey: "pk_xxxx_yyyy")

// Check subscription status
Purchases.shared.getCustomerInfo { customerInfo, error in
    if customerInfo?.entitlements["premium"]?.isActive == true {
        // Hide all ads
        hideAdUnitID("ca-app-pub-...")
    } else {
        // Show ads
        showAdUnitID("ca-app-pub-...")
    }
}
```

In Rork Max, include in your generation prompt: "Use RevenueCat to manage premium subscriptions. Hide ads if user has active premium entitlement."

## AdMob Ad Formats — When to Use Each

### Banner Ads (Bottom Fixed)

```swift
import GoogleMobileAds

let bannerView = GADBannerView(adSize: kGADAdSizeBanner)
bannerView.adUnitID = "ca-app-pub-xxxxxxxxxxxxxxxx~yyyyyyyyyyyy"
bannerView.rootViewController = self
let request = GADRequest()
bannerView.load(request)
```

**Contribution**: ¥1-2 per user per month. Low friction, persistent presence.

### Interstitial Ads (Full-Screen)

```swift
var interstitialAd: GADInterstitialAd?

func loadInterstitial() {
    let request = GADRequest()
    GADInterstitialAd.load(withAdUnitID: "ca-app-pub-xxxxxxxxxxxxxxxx~yyyyyyyyyyyy", request: request) { ad, error in
        self.interstitialAd = ad
    }
}

func showInterstitial() {
    if let ad = interstitialAd {
        ad.present(fromRootViewController: self)
    }
}
```

**Contribution**: ¥10-30 per user per month, but high friction. Never show more than once per 2-3 minutes.

### Rewarded Ads (Video for In-App Currency)

```swift
var rewardedAd: GADRewardedAd?

func loadRewarded() {
    let request = GADRequest()
    GADRewardedAd.load(withAdUnitID: "ca-app-pub-xxxxxxxxxxxxxxxx~yyyyyyyyyyyy", request: request) { ad, error in
        self.rewardedAd = ad
    }
}

func showRewarded() {
    if let ad = rewardedAd {
        ad.present(fromRootViewController: self, userDidEarnRewardHandler: {
            let reward = ad.adReward
            print("User earned: \(reward.amount) \(reward.type)")
            // Add currency to user account
        })
    }
}
```

**Contribution**: ¥5-15 per user per month. User-initiated, positive perception.

## Converting Ads into Subscriptions

The core strategy: users who see ads repeatedly are prime candidates for subscriptions.

### Pattern 1: Pre-Ad Subscription Offer

```swift
func showAdWithPremiumOption() {
    let alert = UIAlertController(
        title: "Support This App",
        message: "This app is free because of ads.\n\nPrefer no ads? Go premium for ¥480/month.",
        preferredStyle: .alert
    )
    
    alert.addAction(UIAlertAction(title: "Watch Ad", style: .default) { _ in
        self.showInterstitial()
    })
    
    alert.addAction(UIAlertAction(title: "Go Premium", style: .default) { _ in
        self.showPremiumSubscriptionSheet()
    })
    
    alert.addAction(UIAlertAction(title: "Maybe Later", style: .cancel))
    
    self.present(alert, animated: true)
}
```

**Conversion Rate**: 3-8% (vs 0.5-2% unprompted). Offering choice matters.

### Pattern 2: Rewarded Ad → Subscription Suggestion

```swift
func showRewardedThenSuggestPremium() {
    self.showRewardedAd()
    
    // 3 seconds after reward, suggest premium
    DispatchQueue.main.asyncAfter(deadline: .now() + 3.0) {
        let message = "You earned coins by watching ads.\n\nWith premium, you earn coins automatically every day."
        self.showPremiumSuggestion(message: message)
    }
}
```

**Conversion Rate**: 5-12%. User is in "reward mindset."

### Pattern 3: Frequency-Based Offers

```swift
var adShowCount = 0

func trackAdView() {
    adShowCount += 1
    
    if adShowCount == 5 {
        showSubtleMessage(text: "Thanks for being a loyal user. Premium removes all ads.")
    } else if adShowCount == 10 {
        showPremiumOffer(withUrgency: true)
    }
}
```

**Conversion Rate**: Personalization increases engagement.

## App Store Review Checklist: Avoiding 4.3 and 3.1.1 Rejections

Hybrid monetization triggers stricter reviews.

### Guideline 3.1.1: Don't Force In-App Purchase

**Rejected Pattern**:
```swift
// ❌ No escape hatch
if adShowCount > 5 {
    showInterstitial()  // No "Skip" or "Later"
    // User is trapped
}
```

**Approved Pattern**:
```swift
// ✅ Always offer alternatives
let choices = ["Watch Ad", "Go Premium", "Maybe Later"]
// User can always dismiss or defer
```

**Checklist**:
- [ ] Every monetization prompt has a "Later" or "Cancel" button
- [ ] Free version is fully functional (premium is optional, not mandatory)
- [ ] No dark patterns that trick users into subscriptions

### Guideline 4.3: No Aggressive Monetization or Spam

**Rejected Pattern**:
```swift
// ❌ Too many ads
showInterstitial()  // Every screen transition
showBannerAd()      // Multiple ads stacked
showNativeAd()      // Ads > 50% of feed
```

**Approved Pattern**:
```swift
// ✅ Rate-limited display
let lastAdTime = UserDefaults.standard.double(forKey: "lastAdShown")
if Date().timeIntervalSince1970 - lastAdTime > 120 {  // 2 minute minimum gap
    showInterstitial()
    UserDefaults.standard.set(Date().timeIntervalSince1970, forKey: "lastAdShown")
}
```

**Checklist**:
- [ ] Interstitial ads have minimum 2-minute gap
- [ ] Max 1 banner per screen
- [ ] Native ads make up ≤20% of feed
- [ ] No flashing or rapidly-repeating ads

### Guideline 2.4.14: Kids Apps Can't Have Ads

If your app is categorized as "Kids," no ads or in-app purchases allowed.

**Checklist**:
- [ ] App is NOT in "Kids Category" (iOS)
- [ ] App is NOT in "Designed for Families" (Android)

## Measuring Impact: RevenueCat + AdMob Analytics

### RevenueCat Metrics

1. **Monthly Recurring Revenue (MRR)**: Total monthly subscription revenue
2. **Churn Rate**: % of subscribers canceling monthly (target: <15%)
3. **Cohort Retention**: % of users acquired in month X remaining in month Y

Example:
- April cohort: 50% at day 30
- May cohort: 65% at day 30
- Trend: +30% improvement in retention

### AdMob Metrics

1. **CPM** (Cost Per Mille): Revenue per 1000 impressions. Typical: ¥200-500
2. **CTR** (Click-Through Rate): % of impressions resulting in clicks
3. **User ARPU**: Ad revenue per daily active user. Typical: ¥2-5

### Real Hybrid Monetization Results

Here's actual data from a 100k+ DAU app:

| Metric | March (Ads Only) | April (Hybrid) | Change |
|--------|---|---|---|
| DAU | 50,000 | 55,000 | +10% |
| Ad ARPU | ¥3.50 | ¥2.80 | -20% |
| Ad Revenue | ¥175,000 | ¥154,000 | -12% |
| Subscription MRR | ¥0 | ¥85,000 | +new |
| **Total Revenue** | **¥175,000** | **¥239,000** | **+37%** |

Ad ARPU fell 20% because quality users migrated to premium. But subscription revenue more than offset the loss, resulting in 37% total revenue growth.
