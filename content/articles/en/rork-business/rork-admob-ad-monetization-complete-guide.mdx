## Why You Should Add Ad Revenue to Your Rork App

When it comes to app monetization, there are three main pillars: subscriptions, one-time purchases, and advertising. Among these, the **ad-supported model** stands out for indie developers because it lets you offer your app for free while still generating income.

Google AdMob is the world's largest mobile advertising platform, and it integrates smoothly with React Native (Expo)-based Rork apps. In this guide, we'll walk through the entire process of adding AdMob ads to a Rork app, covering each ad format with practical code examples.

This article is designed for developers who have built an app with Rork but haven't monetized it yet, those considering a hybrid model combining subscriptions with ads, and beginners to intermediate developers looking for a step-by-step AdMob integration guide.

## Understanding AdMob's Three Main Ad Formats

AdMob offers three primary ad formats, each with distinct characteristics.

**Banner Ads** are rectangular ads displayed at the top or bottom of the screen. They have minimal impact on user experience and provide steady impression-based revenue. While the eCPM (earnings per thousand impressions) is relatively modest, the high volume of impressions compensates for this.

**Interstitial Ads** are full-screen ads that appear during natural transition points, such as between screens or after completing a task. They offer higher eCPM than banners, but showing them too frequently can drive users away. A good rule of thumb is to display them once every three screen transitions.

**Rewarded Ads** are opt-in video ads that users choose to watch in exchange for in-app rewards like virtual currency or premium features. They deliver the highest eCPM of all formats, and because the user initiates the experience, satisfaction remains high. These are especially effective in gaming and utility apps.

## Setting Up Your AdMob Account and Ad Units

Start by creating an account on the [AdMob Console](https://admob.google.com/) and registering your app.

**Step 1: Register Your App**

Navigate to "Apps" → "Add App" in the AdMob console. If your app is already published on the App Store or Google Play, you can search for it by store URL. For unpublished apps, select "App isn't published yet" and register it manually.

**Step 2: Create Ad Units**

For your registered app, create ad units for each format — banner, interstitial, and rewarded. Each unit generates an **Ad Unit ID** in the format `ca-app-pub-XXXXX/YYYYY`. Keep these IDs handy.

**Step 3: Set Up app-ads.txt**

To prevent ad fraud, place an `app-ads.txt` file at the root of your website domain with your AdMob publisher ID. Follow the instructions in the AdMob console for the exact content.

## Installing the AdMob SDK in Your Rork Project

Since Rork is built on React Native (Expo), we'll use the `react-native-google-mobile-ads` library.

```bash
# Run in your Rork project terminal
npx expo install react-native-google-mobile-ads
```

Next, add your AdMob App ID to `app.json` (or `app.config.js`):

```json
{
  "expo": {
    "plugins": [
      [
        "react-native-google-mobile-ads",
        {
          "androidAppId": "ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY",
          "iosAppId": "ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY"
        }
      ]
    ]
  }
}
```

> **Note**: Replace the app IDs above with the ones from your AdMob console. During development, use Google's test IDs to avoid account violations.

Initialize the SDK at your app's entry point:

```typescript
// App.tsx or app/_layout.tsx
import mobileAds from 'react-native-google-mobile-ads';

// Initialize AdMob SDK on app startup
mobileAds()
  .initialize()
  .then(adapterStatuses => {
    console.log('AdMob initialized:', adapterStatuses);
    // Ad requests can be made after initialization completes
  });
```

## Implementing Banner Ads — Always-On Revenue at the Bottom

Banner ads are the simplest format to implement and provide a consistent revenue baseline.

```tsx
import React from 'react';
import { View, StyleSheet } from 'react-native';
import { BannerAd, BannerAdSize, TestIds } from 'react-native-google-mobile-ads';

// Use test IDs during development; switch to real IDs for production
const BANNER_AD_UNIT_ID = __DEV__
  ? TestIds.BANNER
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      {/* Main content area */}
      <View style={styles.content}>
        {/* Your app UI goes here */}
      </View>

      {/* Fixed banner ad at the bottom */}
      <BannerAd
        unitId={BANNER_AD_UNIT_ID}
        size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
        requestOptions={{
          requestNonPersonalizedAdsOnly: true,
        }}
        onAdLoaded={() => console.log('Banner ad loaded')}
        onAdFailedToLoad={(error) => console.error('Banner ad failed:', error)}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
  content: { flex: 1 },
});
// Expected output: An adaptive banner ad appears fixed at the bottom of the screen
```

The `ANCHORED_ADAPTIVE_BANNER` size automatically adjusts to the device's screen width and is the currently recommended setting from Google.

## Implementing Interstitial Ads — Full-Screen Ads at Transition Points

Interstitial ads should be preloaded and shown at natural break points in the user flow.

```tsx
import React, { useEffect, useState } from 'react';
import { Button, View } from 'react-native';
import {
  InterstitialAd,
  AdEventType,
  TestIds,
} from 'react-native-google-mobile-ads';

const INTERSTITIAL_AD_UNIT_ID = __DEV__
  ? TestIds.INTERSTITIAL
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

// Create the ad instance outside the component (once only)
const interstitial = InterstitialAd.createForAdRequest(
  INTERSTITIAL_AD_UNIT_ID,
  { requestNonPersonalizedAdsOnly: true }
);

export default function ResultScreen() {
  const [adLoaded, setAdLoaded] = useState(false);

  useEffect(() => {
    // Listen for the ad loaded event
    const unsubscribeLoaded = interstitial.addAdEventListener(
      AdEventType.LOADED,
      () => setAdLoaded(true)
    );
    // Preload the next ad after the current one is closed
    const unsubscribeClosed = interstitial.addAdEventListener(
      AdEventType.CLOSED,
      () => {
        setAdLoaded(false);
        interstitial.load(); // Preload for next time
      }
    );

    interstitial.load(); // Initial load

    return () => {
      unsubscribeLoaded();
      unsubscribeClosed();
    };
  }, []);

  const showAd = () => {
    if (adLoaded) {
      interstitial.show();
    }
  };

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Button title="Continue" onPress={showAd} />
    </View>
  );
}
// Expected output: A full-screen ad appears on button press, then preloads the next ad after dismissal
```

**Frequency control** is critical for interstitial ads. Showing one on every screen transition will tank your user retention. Implement logic like "once every 3 transitions" or "at most once per 5 minutes" to keep the experience smooth.

## Implementing Rewarded Ads — Give Users a Reason to Watch

Rewarded ads deliver the highest eCPM and create a win-win scenario: users get in-app rewards, and you earn premium ad revenue.

```tsx
import React, { useEffect, useState } from 'react';
import { Button, View, Alert } from 'react-native';
import {
  RewardedAd,
  RewardedAdEventType,
  TestIds,
} from 'react-native-google-mobile-ads';

const REWARDED_AD_UNIT_ID = __DEV__
  ? TestIds.REWARDED
  : 'ca-app-pub-XXXXXXXXXXXXXXXX/YYYYYYYYYY';

const rewarded = RewardedAd.createForAdRequest(
  REWARDED_AD_UNIT_ID,
  { requestNonPersonalizedAdsOnly: true }
);

export default function RewardScreen() {
  const [adLoaded, setAdLoaded] = useState(false);

  useEffect(() => {
    const unsubscribeLoaded = rewarded.addAdEventListener(
      RewardedAdEventType.LOADED,
      () => setAdLoaded(true)
    );
    // Grant reward when the user finishes watching the video
    const unsubscribeEarned = rewarded.addAdEventListener(
      RewardedAdEventType.EARNED_REWARD,
      (reward) => {
        Alert.alert(
          'Reward Earned!',
          `You earned ${reward.amount} ${reward.type}`
        );
        // Execute in-app currency or feature unlock logic here
      }
    );

    rewarded.load();

    return () => {
      unsubscribeLoaded();
      unsubscribeEarned();
    };
  }, []);

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <Button
        title="Watch Video to Earn Coins"
        onPress={() => {
          if (adLoaded) {
            rewarded.show();
            setAdLoaded(false);
            rewarded.load(); // Preload for next time
          }
        }}
        disabled={!adLoaded}
      />
    </View>
  );
}
// Expected output: After watching the full video, an Alert shows the reward details
```

## Optimizing eCPM and Ad Placement Best Practices

Maximizing your AdMob revenue comes down to smart **placement** and **frequency**.

**Placement tips**: For banner ads, a fixed position at the bottom of the screen delivers the best results. In-feed banners placed within scrollable content also achieve high visibility. For interstitials, place them after task completion or at natural break points — after a game level, after saving a document, or before showing search results.

**Frequency capping**: Limit interstitials to a maximum of three per session with at least 60 seconds between each. Google may penalize apps that show ads too aggressively, and excessive frequency will hurt your retention metrics.

**Ad mediation** is worth setting up once your app has consistent traffic. AdMob Mediation aggregates multiple ad networks — Meta Audience Network, Unity Ads, AppLovin, and more — and automatically selects the highest-paying ad for each impression. Configure this from the Mediation tab in the AdMob console.

**A/B testing** your ad configuration can significantly boost revenue. Experiment with banner positions, interstitial frequency, and rewarded ad triggers to find the sweet spot between revenue and user retention.

## Combining Ads with Subscription Revenue

For indie developers, the most effective revenue model is a **freemium structure: free with ads + paid without ads**.

In practice, this means showing banner and interstitial ads to free users while removing all ads for subscribers. This dual approach captures revenue from both user segments.

```typescript
// Control ad visibility based on subscription status
const shouldShowAds = !userSubscription.isPremium;

// Conditionally render ads in your components
{shouldShowAds && (
  <BannerAd
    unitId={BANNER_AD_UNIT_ID}
    size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
  />
)}
```

## Frequently Asked Questions

### How do I switch between test ads and production ads?

Always use `TestIds` during development. Clicking real ads on your own device during development can result in your AdMob account being suspended. Use the `__DEV__` flag or environment variables to switch between test and production ad unit IDs at release time.

### How can I increase my eCPM?

Start by prioritizing rewarded ads, which consistently deliver the highest eCPM. Then enable mediation to let multiple ad networks compete for your inventory. Beyond ad configuration, focus on improving user engagement and retention — apps with higher session times and repeat usage command higher ad rates.

### Does AdMob work with Expo Go?

No. The `react-native-google-mobile-ads` library includes native modules that aren't compatible with Expo Go. You'll need to generate native projects with `expo prebuild` or use EAS Build to create a Development Build for testing on real devices. If you're using Rork Max, native builds are automatically handled on cloud Mac infrastructure.

### Do I need to handle GDPR and CCPA compliance?

Yes. To comply with the EU's GDPR and California's CCPA, you should implement the UMP (User Messaging Platform) SDK to collect user consent for ad personalization on first launch. You can configure compliance messages from the "Privacy & messaging" section in the AdMob console.
