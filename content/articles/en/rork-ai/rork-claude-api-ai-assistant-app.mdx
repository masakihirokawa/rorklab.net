# Build an AI Assistant App with Rork and Claude API

Adding AI assistant capabilities to mobile apps has become one of the most requested features in 2026. Anthropic's Claude API stands out for its natural conversational quality, strong safety guardrails, and excellent support for long-context interactions.

In this guide, you'll learn how to integrate Claude API into a Rork app to build a fully functional AI assistant. We'll cover everything from secure API key management and streaming responses to conversation context handling and production-grade error handling.

## Prerequisites and Setup

Before diving in, make sure you have the following ready.

You should be comfortable with basic Rork development. If you're new to the platform, start with the [Getting Started with Rork](/en/articles/rork-basics/getting-started) guide first.

What you'll need:

- A Rork account (the free plan works for development)
- An Anthropic API key (sign up at [console.anthropic.com](https://console.anthropic.com))
- Node.js 18 or later installed locally
- Basic familiarity with TypeScript and React Native

Regarding costs, Claude Sonnet 4.6 is priced at $3 per million input tokens and $15 per million output tokens as of March 2026. For a personal project, expect to spend around $5–10 per month, making it very accessible for indie developers.

## Setting Up the Anthropic SDK

Install the official Anthropic TypeScript SDK in your Rork project.

```bash
# Run from your Rork project root
npm install @anthropic-ai/sdk
```

Store your API key as an environment variable. Add the following to your `.env` file:

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-api03-xxxxxxxxxxxxxxxxxxxxx
```

Next, create a utility module to initialize the API client.

```typescript
// src/lib/claude.ts
import Anthropic from "@anthropic-ai/sdk";

// Singleton API client instance
const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

export default anthropic;
```

**Important**: Never embed your API key directly in your React Native client code. Always route requests through a backend API to keep your credentials secure. Rork's server-side API routes make this straightforward.

## Building the Backend API Route

All Claude API requests should be handled server-side. Here's a basic API route implementation:

```typescript
// src/app/api/chat/route.ts
import anthropic from "@/lib/claude";
import { NextRequest, NextResponse } from "next/server";

// Message type definition
interface ChatMessage {
  role: "user" | "assistant";
  content: string;
}

export async function POST(request: NextRequest) {
  try {
    const { messages }: { messages: ChatMessage[] } = await request.json();

    // Send request to Claude API
    const response = await anthropic.messages.create({
      model: "claude-sonnet-4-6",  // Latest Sonnet model
      max_tokens: 1024,
      system: "You are a helpful and knowledgeable AI assistant. Respond clearly and concisely.",
      messages: messages,
    });

    // Extract text response
    const assistantMessage =
      response.content[0].type === "text"
        ? response.content[0].text
        : "";

    return NextResponse.json({
      message: assistantMessage,
      usage: response.usage,  // Return token usage for monitoring
    });
  } catch (error) {
    console.error("Claude API Error:", error);
    return NextResponse.json(
      { error: "Failed to get a response from the AI assistant" },
      { status: 500 }
    );
  }
}
```

This code passes the full conversation history from the client to the Claude API. The `system` parameter defines the assistant's behavior and personality — customize this to match your app's purpose.

## Implementing Streaming Responses

Streaming dramatically improves the user experience by displaying Claude's response in real time, character by character, instead of making users wait for the entire response to generate.

```typescript
// src/app/api/chat/stream/route.ts
import anthropic from "@/lib/claude";
import { NextRequest } from "next/server";

export async function POST(request: NextRequest) {
  const { messages } = await request.json();

  // Create a streaming response
  const stream = await anthropic.messages.stream({
    model: "claude-sonnet-4-6",
    max_tokens: 1024,
    system: "You are a helpful and knowledgeable AI assistant. Respond clearly and concisely.",
    messages: messages,
  });

  // Return as a ReadableStream to the client
  const encoder = new TextEncoder();
  const readableStream = new ReadableStream({
    async start(controller) {
      for await (const event of stream) {
        if (
          event.type === "content_block_delta" &&
          event.delta.type === "text_delta"
        ) {
          // Send text deltas in SSE format
          const data = JSON.stringify({ text: event.delta.text });
          controller.enqueue(
            encoder.encode(`data: ${data}\n\n`)
          );
        }
      }
      controller.enqueue(encoder.encode("data: [DONE]\n\n"));
      controller.close();
    },
  });

  return new Response(readableStream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      Connection: "keep-alive",
    },
  });
}
```

## Building the Chat UI

Here's the React Native chat screen implementation with full streaming support:

```typescript
// src/components/ChatScreen.tsx
import React, { useState, useRef, useCallback } from "react";
import {
  View,
  Text,
  TextInput,
  FlatList,
  TouchableOpacity,
  StyleSheet,
  ActivityIndicator,
  KeyboardAvoidingView,
  Platform,
} from "react-native";

interface Message {
  id: string;
  role: "user" | "assistant";
  content: string;
}

export default function ChatScreen() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState("");
  const [isLoading, setIsLoading] = useState(false);
  const flatListRef = useRef<FlatList>(null);

  const sendMessage = useCallback(async () => {
    if (!input.trim() || isLoading) return;

    const userMessage: Message = {
      id: Date.now().toString(),
      role: "user",
      content: input.trim(),
    };

    // Add user message to the list
    const updatedMessages = [...messages, userMessage];
    setMessages(updatedMessages);
    setInput("");
    setIsLoading(true);

    // Add a placeholder for the assistant response
    const assistantId = (Date.now() + 1).toString();
    setMessages((prev) => [
      ...prev,
      { id: assistantId, role: "assistant", content: "" },
    ]);

    try {
      // Call the streaming API endpoint
      const response = await fetch("/api/chat/stream", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          messages: updatedMessages.map(({ role, content }) => ({
            role,
            content,
          })),
        }),
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();

      if (!reader) throw new Error("Failed to read stream");

      // Read streaming data incrementally
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const chunk = decoder.decode(value);
        const lines = chunk.split("\n");

        for (const line of lines) {
          if (line.startsWith("data: ") && line !== "data: [DONE]") {
            const { text } = JSON.parse(line.slice(6));
            // Update the assistant message incrementally
            setMessages((prev) =>
              prev.map((msg) =>
                msg.id === assistantId
                  ? { ...msg, content: msg.content + text }
                  : msg
              )
            );
          }
        }
      }
    } catch (error) {
      console.error("Communication error:", error);
      setMessages((prev) =>
        prev.map((msg) =>
          msg.id === assistantId
            ? { ...msg, content: "Sorry, an error occurred. Please try again." }
            : msg
        )
      );
    } finally {
      setIsLoading(false);
    }
  }, [input, messages, isLoading]);

  return (
    <KeyboardAvoidingView
      style={styles.container}
      behavior={Platform.OS === "ios" ? "padding" : "height"}
    >
      <FlatList
        ref={flatListRef}
        data={messages}
        keyExtractor={(item) => item.id}
        onContentSizeChange={() =>
          flatListRef.current?.scrollToEnd({ animated: true })
        }
        renderItem={({ item }) => (
          <View
            style={[
              styles.bubble,
              item.role === "user" ? styles.userBubble : styles.assistantBubble,
            ]}
          >
            <Text style={styles.messageText}>{item.content}</Text>
          </View>
        )}
      />
      <View style={styles.inputRow}>
        <TextInput
          style={styles.input}
          value={input}
          onChangeText={setInput}
          placeholder="Type a message..."
          multiline
        />
        <TouchableOpacity
          style={styles.sendButton}
          onPress={sendMessage}
          disabled={isLoading}
        >
          {isLoading ? (
            <ActivityIndicator color="#fff" />
          ) : (
            <Text style={styles.sendText}>Send</Text>
          )}
        </TouchableOpacity>
      </View>
    </KeyboardAvoidingView>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  bubble: { padding: 12, borderRadius: 16, marginVertical: 4, marginHorizontal: 12, maxWidth: "80%" },
  userBubble: { backgroundColor: "#007AFF", alignSelf: "flex-end" },
  assistantBubble: { backgroundColor: "#fff", alignSelf: "flex-start" },
  messageText: { fontSize: 16, lineHeight: 22 },
  inputRow: { flexDirection: "row", padding: 8, backgroundColor: "#fff" },
  input: { flex: 1, borderWidth: 1, borderColor: "#ddd", borderRadius: 20, paddingHorizontal: 16, paddingVertical: 8, fontSize: 16 },
  sendButton: { marginLeft: 8, backgroundColor: "#007AFF", borderRadius: 20, paddingHorizontal: 20, justifyContent: "center" },
  sendText: { color: "#fff", fontWeight: "bold" },
});
```

## Managing Conversation Context and Token Optimization

Claude API requires sending the full conversation history with each request. As conversations grow longer, token costs increase, so smart context management is essential.

```typescript
// src/lib/conversation.ts

interface ConversationMessage {
  role: "user" | "assistant";
  content: string;
  timestamp: number;
}

// Sliding window: send only the most recent N messages
export function getRecentMessages(
  messages: ConversationMessage[],
  maxMessages: number = 20
): ConversationMessage[] {
  if (messages.length <= maxMessages) return messages;

  // Keep the first message (context setter) and recent messages
  const first = messages[0];
  const recent = messages.slice(-maxMessages + 1);
  return [first, ...recent];
}

// Rough token estimation (Japanese ≈ 1.5 tokens/char, English ≈ 0.25 tokens/char)
export function estimateTokens(text: string): number {
  const japaneseChars = (text.match(/[\u3000-\u9fff]/g) || []).length;
  const otherChars = text.length - japaneseChars;
  return Math.ceil(japaneseChars * 1.5 + otherChars * 0.25);
}

// Estimate total tokens for the entire conversation
export function estimateConversationTokens(
  messages: ConversationMessage[]
): number {
  return messages.reduce(
    (total, msg) => total + estimateTokens(msg.content),
    0
  );
}
```

The key trick for keeping costs down in long conversations is the sliding window approach — sending only the most recent messages while including a summary in the `system` prompt to maintain context even after older messages are trimmed.

## Error Handling and Rate Limit Protection

Robust error handling is critical for production apps. Here's a retry mechanism with exponential backoff:

```typescript
// src/lib/claude-with-retry.ts
import anthropic from "./claude";

const MAX_RETRIES = 3;
const BASE_DELAY = 1000; // 1 second

export async function sendWithRetry(
  messages: Array<{ role: "user" | "assistant"; content: string }>,
  systemPrompt: string,
  retries = MAX_RETRIES
): Promise<string> {
  for (let attempt = 0; attempt <= retries; attempt++) {
    try {
      const response = await anthropic.messages.create({
        model: "claude-sonnet-4-6",
        max_tokens: 1024,
        system: systemPrompt,
        messages,
      });

      return response.content[0].type === "text"
        ? response.content[0].text
        : "";
    } catch (error: any) {
      // Rate limit (429) — retry with backoff
      if (error.status === 429 && attempt < retries) {
        const delay = BASE_DELAY * Math.pow(2, attempt);
        console.warn(
          `Rate limited. Retrying in ${delay}ms (${attempt + 1}/${retries})`
        );
        await new Promise((resolve) => setTimeout(resolve, delay));
        continue;
      }

      // Auth error (401) — no point retrying
      if (error.status === 401) {
        throw new Error("Invalid API key. Please check your configuration.");
      }

      // Server error (5xx) — retry with backoff
      if (error.status >= 500 && attempt < retries) {
        const delay = BASE_DELAY * Math.pow(2, attempt);
        await new Promise((resolve) => setTimeout(resolve, delay));
        continue;
      }

      throw error;
    }
  }

  throw new Error("Maximum retry attempts reached");
}
```

Exponential backoff ensures that when you hit a rate limit, your app automatically waits and retries with increasing delays rather than bombarding the API and showing errors to users.

## Frequently Asked Questions

### Q1: How much does Claude API cost for a chat app?

With Claude Sonnet 4.6, a typical conversation of about 10 exchanges costs roughly $0.01–0.03. Even at 1,000 conversations per month, you're looking at $10–30 — well within reach for solo developers. Anthropic also lets you set monthly spending limits in the console for extra peace of mind.

### Q2: Should I choose Claude API or OpenAI's GPT-4o?

It depends on your use case. Claude excels at long-context processing (up to 200K tokens) and comes with built-in safety guardrails. GPT-4o has a head start in multimodal capabilities like image and audio processing. For conversational quality, especially in nuanced or safety-sensitive contexts, Claude is widely praised. Rork also has a [GPT-4o integration guide](/en/articles/rork-ai/gpt-4o-rork-openai-app-development) if you'd like to compare both approaches side by side.

### Q3: What happens without streaming responses?

Without streaming, users stare at a blank screen while Claude generates the full response — typically 5–15 seconds for longer replies. This is a major cause of app abandonment. With streaming, text appears in real time, making the perceived wait time essentially zero and keeping users engaged.

### Q4: What should I do if my API key is compromised?

If you suspect your key has been leaked, immediately revoke it in the Anthropic Console and issue a new one. Unauthorized usage can lead to unexpected charges. As a preventive measure, always set a monthly spending cap and monitor your usage dashboard regularly.
