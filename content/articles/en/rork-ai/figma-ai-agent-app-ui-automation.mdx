## Introduction

On March 24, 2026, Figma announced a game-changing capability: **AI agents can now directly manipulate the Figma canvas**. No more text descriptions or CSS proposals—agents can now create frames, place components, and apply design tokens natively.

This shifts the design workflow from sequential (request → design → implement) to parallel (AI designs while developers code). For Rork developers, this means UI creation time collapses from weeks to hours.

---

## What's New: Figma AI Agent Features

### The Three Components

1. **use_figma Tool**: A Model Context Protocol (MCP) server exposing native Figma operations
2. **Skills Markdown**: Instruction sets that guide agent behavior through design tasks
3. **Enhanced Code Connect**: Tighter designer ↔ developer collaboration through automated code mapping

Together, these enable a unified workflow where design and code coexist in version-controlled synchrony.

### Why It Matters

**Before**:
Request → Designer (manual) → File → Developer (manual) → Code

**Now**:
Request → AI Agent (automated) → Design + Code → Ready to integrate

The bottleneck shifts from "waiting for design assets" to "integrating pre-generated components."

## The use_figma Tool

### Available Operations

| Operation | Purpose | Use Case |
|---|---|---|
| `createFrame()` | Create a canvas frame | Screen layouts |
| `createComponent()` | Define reusable component | Buttons, cards, inputs |
| `createVariant()` | Add component state | Hover, active, disabled |
| `placeAsset()` | Insert images or icons | Branding, illustrations |
| `setVariable()` | Define design token | Color, size, typography |
| `applyToken()` | Apply design token | Enforce brand consistency |
| `createLayout()` | Set Auto Layout | Responsive design |

### Code Example

```typescript
import { useFigma } from "@figma/ai-sdk";

async function generateLoginScreen() {
  const figma = useFigma({
    token: process.env.FIGMA_TOKEN,
    fileKey: "YOUR_FILE_KEY",
  });

  // Create main frame
  const frame = await figma.createFrame({
    name: "Login Screen",
    width: 375,
    height: 667,
  });

  // Add brand logo
  const logo = await figma.placeAsset({
    parentId: frame.id,
    assetUrl: "https://brand.com/logo.png",
    width: 120,
    height: 120,
  });

  // Email input component
  const emailInput = await figma.createComponent({
    parentId: frame.id,
    name: "Email Input",
    type: "text_input",
    width: 335,
    height: 48,
    placeholder: "Enter email",
  });

  // Password input
  const passwordInput = await figma.createComponent({
    parentId: frame.id,
    name: "Password Input",
    type: "text_input",
    width: 335,
    height: 48,
    placeholder: "Enter password",
    isSecure: true,
  });

  // Login button
  const button = await figma.createComponent({
    parentId: frame.id,
    name: "Login Button",
    type: "button",
    width: 335,
    height: 48,
    label: "Log In",
    backgroundColor: "#007AFF",
  });

  // Enable responsive layout
  await figma.createLayout({
    targetId: frame.id,
    mode: "vertical",
    spacing: 20,
    padding: 20,
  });

  return frame;
}
```

## Skills: Markdown-Driven Design Automation

### What Are Skills?

Skills are markdown files that instruct AI agents how to accomplish design tasks. They function like "design recipes."

### Button System Example

```markdown
# Button Component System

## Objective
Create a complete button component library with variants.

## Variants Required
- Styles: Primary, Secondary, Danger
- Sizes: Small (32px), Medium (48px), Large (56px)
- States: Default, Hover, Active, Disabled

## Step 1: Create Main Component
1. Frame named "Button/Primary/Medium"
2. Dimensions: 200×48px
3. Fill: #007AFF (Primary Blue)
4. Border radius: 8px
5. Text label: size 16, weight Bold

## Step 2: Generate Variants
For each style × size combination:
- Auto-generate corresponding frame
- Maintain consistent spacing and typography

## Step 3: Add Interactive States
For each variant, create:
- Default: opacity 100%
- Hover: background −10% brightness
- Active: background −20% brightness
- Disabled: opacity 50%

## Step 4: Create Design Tokens
Define variables:
- color-primary: #007AFF
- color-secondary: #E5E5EA
- spacing: 8px, 16px, 24px

## Safety Constraints
- Never delete existing components
- Maintain 8px grid alignment
- Ensure WCAG AA contrast minimum
```

The AI agent reads this and executes step-by-step without human intervention.

## Code Connect Integration

### Automated Design ↔ Code Synchronization

```typescript
// figma.config.ts
import { figma } from "@figma/code-connect";

figma.connect(
  "Button/Primary/Medium",
  {
    component: "./components/Button.tsx",
    props: {
      variant: figma.enum("Variant"),
      size: figma.enum("Size"),
      disabled: figma.boolean("Disabled"),
      label: figma.string("Label"),
    },
  },
);
```

When the AI creates a button component in Figma, Code Connect automatically generates TypeScript props, ensuring design and code stay in sync.

## Integration with Rork Development

### Proposed Workflow

1. **Define Requirements**: "Build a secure login flow"
2. **Run AI Agent**: Invokes Figma MCP to generate screens, components, states
3. **Apply Code Connect**: Extracts component definitions into TypeScript
4. **Integrate with Rork**: Drops components into app scaffolding
5. **Auto-Test**: AI validates responsiveness, accessibility, functionality

### Command Example

```bash
# Initialize Rork project with Figma MCP
rork init --with-figma-mcp

# Run AI to design AND code login screen
rork agent run --task "Generate login screen from requirements"

# Inspect generated design
rork figma review

# Integrate components
rork integrate figma:LoginScreen --target src/screens/
```

## FAQ

### Q1: Is Figma AI agent capability available now?

**A:** Yes. It's available to Figma Pro and above. As of March 2026, use_figma tool access is unrestricted.

### Q2: Can I use this with existing Figma files?

**A:** Yes. Add a new page to your file and let the AI work within that scope. Existing designs remain untouched.

### Q3: How good is the auto-generated component code?

**A:** Structure and props are solid. Business logic (API calls, validation) still requires human implementation. The AI handles the tedious structural work; developers focus on behavior.
