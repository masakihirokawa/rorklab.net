## What Is Rork Companion? How It Differs from Rork's Main Builder

Rork Companion is Rork's newest feature: an **AI pair programmer** designed to work alongside your existing code. While it might seem similar to standard Rork at first glance, its purpose is fundamentally different.

Standard Rork is a **code generation engine**. Feed it a prompt, and within minutes, you get a fully functional mobile or web application built from scratch. It's the 0→1 solution.

Rork Companion operates in a completely different space. Instead of generating apps from prompts, **it engages in dialogue with your existing codebase**, helping you debug, improve, extend, and refactor code you've already written. It's the 1→100 accelerator.

Think of it this way:
- **Rork (standard)** = Zero-to-one app generation engine
- **Rork Companion** = Interactive AI assistant that transforms and enhances existing code

The result? Your development workflow becomes more fluid, faster, and smarter. Companion becomes that experienced senior engineer who sits next to you, always ready with advice.

## What You Can Do with Rork Companion

Rork Companion's capabilities are remarkably broad, covering nearly every challenge you'll face in real-world development.

**Code Explanation and Understanding**
Inheriting a large codebase? Wrestling with complex logic? Paste the code into Companion, and it breaks down what's happening, why it matters, and the design intent behind it—all in clear, accessible language.

**Debugging and Error Resolution**
Your app crashes. The error logs are cryptic. Companion analyzes the error message and your code snippet, pinpoints the root cause, and suggests fixes. That infamous `undefined is not a function` in React Native? Companion has a solution in seconds.

**Feature Implementation Guidance**
Want to add user authentication? Integrate payments? Implement push notifications? Companion walks you through each step, ensuring your additions integrate smoothly with existing code and follow best practices.

**Refactoring Support**
Your codebase has grown messy. Logic is duplicated across multiple files. Maintainability is suffering. Show Companion your code, and it proposes a refactoring strategy with concrete improvements.

**Test Generation**
Companion can generate unit tests, integration tests, and test scaffolding for your code. Test-driven development (TDD) becomes easier and faster to implement.

## Getting Started with Rork Companion

To use Rork Companion, you need **Rork Max**, the premium tier. It's not available on the free plan.

Once you upgrade to Rork Max, a new **Companion tab** appears in your Rork dashboard. Click it, and you're instantly in an interactive chat interface with Companion.

The workflow is straightforward:

1. Paste your code into the code pane (or upload a file)
2. Type your question or request in the chat box
3. Press Enter, and Companion responds immediately
4. Ask follow-ups or request modifications

Example prompts:
- "Why is this useEffect creating an infinite loop?"
- "How do I make this React Native styling responsive?"
- "Can you refactor this function so it's more testable?"

Companion delivers accurate, production-ready answers every time.

## Real-World Development Workflow with Rork Companion

Here's how Companion fits into actual development scenarios.

**Scenario 1: Rapid Bug Resolution**

Your app has a critical issue: "Users can't see the screen after logging in." You'd normally block 2–3 hours for debugging.

Instead, you paste your AsyncStorage retrieval code, navigation logic, and auth state management into Companion and ask: "Why aren't users seeing the screen after login?"

Companion identifies:
- A timing issue in your authentication promise chain
- The exact line causing the problem
- Three possible fixes, with code examples

What would have taken hours takes 20 minutes.

**Scenario 2: Structured Feature Implementation**

You need to add Stripe payments to your app.

Show Companion your existing user management code and database schema, then ask: "What's the best way to integrate Stripe payments?"

You get back:
- An implementation pattern that fits your codebase
- Required libraries and setup steps
- Error handling examples
- A test code skeleton

Feature architecture decisions that normally take hours are resolved in minutes.

## Tips for Getting the Most Out of Rork Companion

**Ask specific questions**
"The code doesn't work" is vague. "The state from AsyncStorage is undefined after navigation" is actionable. Companion gives better answers to specific questions.

**Provide sufficient context**
Don't show Companion just the broken function. Include the parent component calling it, utility functions it depends on, and related state management. Context leads to smarter answers.

**Dialogue iteratively**
Instead of asking everything at once, break your problem into stages. "First, let's confirm this timing issue. Then we'll refactor the structure." Incremental dialogue often leads to more elegant solutions.

**Customize, don't copy-paste**
Companion's code is an excellent starting point, not a finished solution. Adapt it to your project's unique requirements and constraints. Verify your modifications with follow-up questions.

**Request periodic code reviews**
After implementing a new feature or major refactoring, ask Companion: "Are there security concerns in this code?" and "Where can we optimize performance?" Catching issues before production is invaluable.
