## Replicate vs. fal.ai — How to Choose

The two leading AI generation API platforms are Replicate and fal.ai. The right choice depends on your use case.

### Replicate

Replicate's primary strength is model variety. Thousands of models are available — Flux Dev, Flux Schnell, SDXL, Stable Video Diffusion, and many niche options. Billing is pure pay-per-use based on GPU runtime, with inexpensive models running as low as $0.001–$0.005 per image.

- **Strengths**: Enormous model catalog, custom model deployment, excellent documentation
- **Weaknesses**: Higher latency (cold starts can take tens of seconds), no WebSocket support

### fal.ai

fal.ai is purpose-built for real-time generation. Its WebSocket and Server-Sent Events (SSE) support lets you stream generation progress directly to your UI. With Flux Schnell on a warm server, results return in 1–2 seconds.

- **Strengths**: Low latency, real-time streaming, official React Native SDK
- **Weaknesses**: Smaller model selection compared to Replicate; some niche models unavailable

### Decision Framework

- **Real-time feel matters** (show a live preview while the user types) → fal.ai
- **You need diverse or custom models** → Replicate
- **Best of both** → Hybrid: use fal.ai for standard image generation, Replicate for specialized models

This guide implements a **hybrid architecture**: fal.ai as the primary backend, Replicate as a fallback for models not available on fal.ai.

## Image Generation with Flux Schnell

### Supabase Edge Function

Create a backend function that accepts generation requests and forwards them to fal.ai or Replicate.

```typescript
// supabase/functions/generate-image/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const FAL_API_KEY = Deno.env.get("FAL_API_KEY")!;
const SUPABASE_URL = Deno.env.get("SUPABASE_URL")!;
const SUPABASE_SERVICE_KEY = Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!;

serve(async (req) => {
  const { prompt, negativePrompt, style, aspectRatio, userId } = await req.json();

  // Check credits (see billing section below)
  const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_KEY);
  const { data: user } = await supabase
    .from("users")
    .select("credits")
    .eq("id", userId)
    .single();

  if (!user || user.credits < 1) {
    return new Response(
      JSON.stringify({ error: "Insufficient credits" }),
      { status: 402 }
    );
  }

  // Merge style preset into prompt
  const styleMap: Record<string, string> = {
    anime: "anime style, highly detailed, vibrant colors, Studio Ghibli inspired",
    photo: "photorealistic, professional photography, 8k, sharp focus",
    oil: "oil painting, impressionist style, textured brush strokes",
    pixel: "pixel art, 16-bit style, retro game aesthetic",
  };

  const enhancedPrompt = `${prompt}, ${styleMap[style] || ""}`.trim();

  // Call fal.ai (Flux Schnell model)
  const falResponse = await fetch(
    "https://fal.run/fal-ai/flux/schnell",
    {
      method: "POST",
      headers: {
        "Authorization": `Key ${FAL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        prompt: enhancedPrompt,
        negative_prompt: negativePrompt || "blur, low quality, watermark",
        image_size: aspectRatioToSize(aspectRatio),
        num_inference_steps: 4,
        seed: Math.floor(Math.random() * 1000000),
        enable_safety_checker: true,
        safety_checker_version: "v2",
      }),
    }
  );

  const result = await falResponse.json();

  if (!result.images?.[0]?.url) {
    return new Response(
      JSON.stringify({ error: "Generation failed" }),
      { status: 500 }
    );
  }

  // Deduct 1 credit
  await supabase
    .from("users")
    .update({ credits: user.credits - 1 })
    .eq("id", userId);

  // Save generation history
  const imageUrl = result.images[0].url;
  await supabase.from("generations").insert({
    user_id: userId,
    prompt: enhancedPrompt,
    image_url: imageUrl,
    model: "flux-schnell",
    aspect_ratio: aspectRatio,
    credits_used: 1,
  });

  return new Response(
    JSON.stringify({ imageUrl, creditsRemaining: user.credits - 1 }),
    { headers: { "Content-Type": "application/json" } }
  );
});

function aspectRatioToSize(ratio: string): string {
  const sizeMap: Record<string, string> = {
    "1:1": "square",
    "16:9": "landscape_16_9",
    "9:16": "portrait_16_9",
  };
  return sizeMap[ratio] || "square";
}
```

### React Native Client Hook

```typescript
// src/hooks/useImageGeneration.ts
import { useState, useRef } from "react";
import { supabase } from "../lib/supabase";
import { useAuthStore } from "../stores/authStore";
import { useCreditsStore } from "../stores/creditsStore";

export interface GenerationParams {
  prompt: string;
  negativePrompt?: string;
  style: "anime" | "photo" | "oil" | "pixel";
  aspectRatio: "1:1" | "16:9" | "9:16";
}

export function useImageGeneration() {
  const [isGenerating, setIsGenerating] = useState(false);
  const [progress, setProgress] = useState(0);
  const [generatedImageUrl, setGeneratedImageUrl] = useState<string | null>(null);
  const [error, setError] = useState<string | null>(null);
  const abortRef = useRef(false);

  const { user } = useAuthStore();
  const { credits, updateCredits } = useCreditsStore();

  const generate = async (params: GenerationParams) => {
    if (!user) return;
    if (credits < 1) {
      setError("You're out of credits. Please top up to continue.");
      return;
    }

    setIsGenerating(true);
    setProgress(10);
    setError(null);
    abortRef.current = false;

    // Animate progress while waiting
    const progressInterval = setInterval(() => {
      setProgress((prev) => {
        if (abortRef.current || prev >= 90) {
          clearInterval(progressInterval);
          return prev;
        }
        return prev + Math.random() * 15;
      });
    }, 400);

    try {
      const { data, error: fnError } = await supabase.functions.invoke(
        "generate-image",
        {
          body: {
            prompt: params.prompt,
            negativePrompt: params.negativePrompt,
            style: params.style,
            aspectRatio: params.aspectRatio,
            userId: user.id,
          },
        }
      );

      if (abortRef.current) return;
      if (fnError || data?.error) throw new Error(data?.error || "Generation failed");

      setProgress(100);
      setGeneratedImageUrl(data.imageUrl);
      updateCredits(data.creditsRemaining);
    } catch (err) {
      if (!abortRef.current) {
        setError(err instanceof Error ? err.message : "An unexpected error occurred");
      }
    } finally {
      clearInterval(progressInterval);
      setIsGenerating(false);
    }
  };

  const cancel = () => {
    abortRef.current = true;
    setIsGenerating(false);
    setProgress(0);
  };

  return { generate, cancel, isGenerating, progress, generatedImageUrl, error };
}
```

## Video Generation with Wan 2.1

Video generation takes significantly longer (30–120 seconds), so it needs an async job pattern. fal.ai's queue API lets you submit a job and poll for — or receive a webhook notification about — the result.

### Video Generation Edge Function

```typescript
// supabase/functions/generate-video/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const FAL_API_KEY = Deno.env.get("FAL_API_KEY")!;

serve(async (req) => {
  const { prompt, userId } = await req.json();
  const supabase = createClient(
    Deno.env.get("SUPABASE_URL")!,
    Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
  );

  // Video generation costs 5 credits
  const { data: user } = await supabase
    .from("users")
    .select("credits")
    .eq("id", userId)
    .single();

  if (!user || user.credits < 5) {
    return new Response(
      JSON.stringify({ error: "Video generation requires 5 credits" }),
      { status: 402 }
    );
  }

  // Submit to fal.ai queue (Wan 2.1 T2V model)
  const queueResponse = await fetch(
    "https://queue.fal.run/fal-ai/wan-t2v",
    {
      method: "POST",
      headers: {
        "Authorization": `Key ${FAL_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        prompt,
        num_frames: 81,       // ~3 seconds of video at 24fps
        frame_rate: 24,
        resolution: "480p",
        num_inference_steps: 30,
      }),
    }
  );

  const { request_id } = await queueResponse.json();

  // Save job ID for the client to poll
  await supabase.from("video_jobs").insert({
    user_id: userId,
    request_id,
    prompt,
    status: "pending",
    credits_reserved: 5,
  });

  // Deduct credits upfront (refund logic handled separately on failure)
  await supabase
    .from("users")
    .update({ credits: user.credits - 5 })
    .eq("id", userId);

  return new Response(
    JSON.stringify({ requestId: request_id }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### Polling for Status

```typescript
// src/hooks/useVideoGeneration.ts
export function useVideoGeneration() {
  const [jobStatus, setJobStatus] = useState<
    "idle" | "pending" | "processing" | "completed" | "failed"
  >("idle");
  const [videoUrl, setVideoUrl] = useState<string | null>(null);
  const [progressPercent, setProgressPercent] = useState(0);
  const pollingRef = useRef<ReturnType<typeof setInterval> | null>(null);

  const pollJobStatus = async (requestId: string) => {
    pollingRef.current = setInterval(async () => {
      const { data } = await supabase.functions.invoke("check-video-job", {
        body: { requestId },
      });

      if (data?.status === "completed") {
        clearInterval(pollingRef.current!);
        setVideoUrl(data.videoUrl);
        setJobStatus("completed");
        setProgressPercent(100);
      } else if (data?.status === "failed") {
        clearInterval(pollingRef.current!);
        setJobStatus("failed");
      } else {
        // Estimate progress from elapsed time (0–90%)
        setProgressPercent((prev) => Math.min(prev + 8, 90));
      }
    }, 5000); // Poll every 5 seconds
  };

  return { generateVideo, jobStatus, videoUrl, progressPercent, pollJobStatus };
}
```

## Managing, Saving, and Sharing Generated Content

### Persisting to Supabase Storage

The image URLs returned by fal.ai are temporary and may expire within hours. In production, transfer them to your own Supabase Storage bucket immediately after generation to make them permanent.

```typescript
// src/lib/saveGeneratedImage.ts
import * as FileSystem from "expo-file-system";
import { decode } from "base64-arraybuffer";
import { supabase } from "./supabase";

export async function saveToSupabaseStorage(
  temporaryUrl: string,
  userId: string
): Promise<string> {
  const downloadResult = await FileSystem.downloadAsync(
    temporaryUrl,
    FileSystem.cacheDirectory + "temp_image.jpg"
  );

  const base64 = await FileSystem.readAsStringAsync(
    downloadResult.uri,
    { encoding: FileSystem.EncodingType.Base64 }
  );

  const fileName = `${userId}/${Date.now()}.jpg`;
  const { error } = await supabase.storage
    .from("generations")
    .upload(fileName, decode(base64), {
      contentType: "image/jpeg",
      upsert: false,
    });

  if (error) throw error;

  const { data: { publicUrl } } = supabase.storage
    .from("generations")
    .getPublicUrl(fileName);

  return publicUrl;
}
```

### Saving to the Device Camera Roll

```typescript
import * as MediaLibrary from "expo-media-library";
```

## Credit-Based Billing: Design and Implementation

### Structuring Your Credit Packs

For AI generation apps, credit-based billing (pay-as-you-go) typically converts better than subscriptions, because users can try the product with a low upfront commitment.

Recommended credit pack structure:

- **Starter (free)**: 10 credits — new user signup bonus
- **Light pack**: 50 credits — $1.20 (~$0.024/credit)
- **Standard pack**: 200 credits — $3.80 (~$0.019/credit)
- **Pro pack**: 500 credits — $7.80 (~$0.0156/credit)

Credit consumption rates:
- Image (Flux Schnell): 1 credit
- Image (SDXL high-res): 2 credits
- Video (Wan 2.1 short): 5 credits
- Video (Wan 2.1 extended): 10 credits

### Stripe Checkout Integration

```typescript
// supabase/functions/create-credit-checkout/index.ts
import Stripe from "https://esm.sh/stripe@14";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!, {
  apiVersion: "2023-10-16",
});

const CREDIT_PACKS = {
  lite:     { credits: 50,  price_usd: 120 },   // in cents
  standard: { credits: 200, price_usd: 380 },
  pro:      { credits: 500, price_usd: 780 },
};

serve(async (req) => {
  const { packId, userId, successUrl, cancelUrl } = await req.json();
  const pack = CREDIT_PACKS[packId as keyof typeof CREDIT_PACKS];
  if (!pack) return new Response(JSON.stringify({ error: "Invalid pack ID" }), { status: 400 });

  const session = await stripe.checkout.sessions.create({
    payment_method_types: ["card"],
    line_items: [
      {
        price_data: {
          currency: "usd",
          unit_amount: pack.price_usd,
          product_data: {
            name: `${pack.credits} AI Generation Credits`,
            description: `Use these credits to generate AI images and videos`,
            images: ["https://your-app.com/credit-icon.png"],
          },
        },
        quantity: 1,
      },
    ],
    mode: "payment",
    success_url: successUrl,
    cancel_url: cancelUrl,
    metadata: {
      user_id: userId,
      pack_id: packId,
      credits: pack.credits.toString(),
    },
  });

  return new Response(
    JSON.stringify({ checkoutUrl: session.url }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### Granting Credits via Webhook

```typescript
// supabase/functions/stripe-webhook/index.ts
serve(async (req) => {
  const signature = req.headers.get("stripe-signature")!;
  const body = await req.text();

  const event = stripe.webhooks.constructEvent(
    body, signature, Deno.env.get("STRIPE_WEBHOOK_SECRET")!
  );

  if (event.type === "checkout.session.completed") {
    const session = event.data.object as Stripe.Checkout.Session;
    const { user_id, credits } = session.metadata!;

    const supabase = createClient(
      Deno.env.get("SUPABASE_URL")!,
      Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
    );

    const { data: user } = await supabase
      .from("users").select("credits").eq("id", user_id).single();

    await supabase
      .from("users")
      .update({ credits: (user?.credits ?? 0) + parseInt(credits) })
      .eq("id", user_id);

    await supabase.from("credit_purchases").insert({
      user_id,
      credits_added: parseInt(credits),
      stripe_session_id: session.id,
      amount_cents: session.amount_total,
    });
  }

  return new Response(JSON.stringify({ received: true }));
});
```

### Handling Credit Refunds and Edge Cases

Real-world billing always involves edge cases. Users lose their internet connection mid-generation, the AI API returns an error after the credit was already deducted, or the safety checker blocks a legitimate prompt. Having a refund policy — and implementing it automatically — builds trust and reduces support requests.

The recommended approach is a **two-phase commit** pattern:

1. **Reserve** the credit (mark it as "pending") before sending the generation request
2. **Confirm** (finalize the deduction) only after the generation URL is returned
3. **Release** the reservation automatically if the generation fails or is blocked

This pattern requires two columns in your `users` table: `credits` (confirmed balance) and `credits_reserved` (in-flight deductions). The Edge Function updates `credits_reserved` on request start, moves the deduction to `credits` on success, and releases the reservation on failure. The implementation adds complexity but pays off at scale — angry users who feel charged for nothing churn fast.

### Free Trial Credits and Growth Mechanics

New user activation is the critical metric for AI generation apps. A generous signup bonus (10–20 free generations) lets users experience the full value proposition before committing money. To maximize conversion from free to paid, show the remaining credit balance prominently in the UI, trigger an "almost out of credits" nudge at 2–3 credits remaining, and make the purchase flow accessible with a single tap from anywhere in the app.

A/B test the signup bonus size. In many apps, giving 20 credits instead of 10 increases Day-7 retention by 15–20%, which more than offsets the slightly higher API cost per new user.

## App Store Review Strategy and Content Moderation

### Apple's Requirements for AI Generation Apps

Apple's guidelines for AI-generated content have tightened in recent years. As of 2026, the key requirements are:

- **Age rating (4+–17+)**: If generated content could include sensitive material, set the App Store Connect Content Rating accordingly and implement guardrails
- **AI disclosure**: Apps must clearly indicate that content is AI-generated — a badge or disclaimer on each generated image is sufficient
- **No realistic likenesses of real people**: Generating images that closely replicate real individuals is a common rejection trigger

### Building a Moderation Layer

```typescript
// src/lib/contentModeration.ts
const BANNED_KEYWORDS = [
  // Use a more comprehensive list in production
  "nsfw", "explicit", "nude", "violence-extreme",
];

export function sanitizePrompt(prompt: string): {
  sanitized: string;
  blocked: boolean;
  reason?: string;
} {
  const lower = prompt.toLowerCase();

  for (const keyword of BANNED_KEYWORDS) {
    if (lower.includes(keyword)) {
      return {
        sanitized: "",
        blocked: true,
        reason: `Prompt contains a restricted keyword: "${keyword}"`,
      };
    }
  }

  return { sanitized: prompt, blocked: false };
}
```

### Using fal.ai's Built-In Safety Checker

## Performance Optimization and Cost Management

### Caching Strategy

For identical prompt + style combinations, fix the seed and cache the result. This eliminates redundant API calls and reduces costs significantly.

```typescript
// src/lib/generationCache.ts
import AsyncStorage from "@react-native-async-storage/async-storage";

const CACHE_PREFIX = "gen_cache_";
const CACHE_TTL_MS = 24 * 60 * 60 * 1000; // 24 hours

export async function getCachedGeneration(
  prompt: string,
  style: string,
  aspectRatio: string
): Promise<string | null> {
  const key = `${CACHE_PREFIX}${btoa(prompt + style + aspectRatio)}`;
  const cached = await AsyncStorage.getItem(key);
  if (!cached) return null;

  const { url, timestamp } = JSON.parse(cached);
  if (Date.now() - timestamp > CACHE_TTL_MS) {
    await AsyncStorage.removeItem(key);
    return null;
  }

  return url;
}

export async function setCachedGeneration(
  prompt: string, style: string, aspectRatio: string, url: string
): Promise<void> {
  const key = `${CACHE_PREFIX}${btoa(prompt + style + aspectRatio)}`;
  await AsyncStorage.setItem(key, JSON.stringify({ url, timestamp: Date.now() }));
}
```

### Model Selection for Cost Efficiency

Use a two-stage generation flow: a cheap, fast "preview" generation followed by a high-quality "final" generation. This preserves quality while keeping per-session costs low.

- **Preview / drafts**: Flux Schnell (~$0.001/image)
- **Final output**: Flux Dev (~$0.025/image)
- **Short video (3s)**: Wan 2.1 T2V (~$0.05/video)

For more on monitoring app-wide performance and costs, see the [App Performance Optimization Complete Guide](/articles/rork-dev/rork-app-performance-optimization-complete-guide).

### Monitoring Your API Spend

As traffic grows, a surprise bill from fal.ai or Replicate can wipe out your margins. Set up spending alerts on both platforms (both support email notifications when monthly spend exceeds a threshold). On the application side, track `credits_used` per day in your `generations` table and build a simple admin dashboard that shows:

- **Daily API cost** (credits consumed × average cost per credit)
- **Revenue** (Stripe webhook data)
- **Margin** ((revenue − API cost) / revenue)

When margins compress below your target (typically 60–70% for consumer apps), it's a signal to either raise credit prices, reduce per-generation quality on lower credit tiers, or introduce a higher-priced "premium quality" credit type.

### Batch Generation and Queue Management

Some users will want to generate many images at once — for wallpaper apps, this might be 20–50 images in a single session. Implement a local generation queue rather than firing all requests simultaneously, which can trigger rate limiting on the API side and degrade everyone's experience.

```typescript
// src/lib/generationQueue.ts
import PQueue from "p-queue"; // npm install p-queue

// Limit to 3 concurrent generation requests per session
const queue = new PQueue({ concurrency: 3 });

export async function enqueueGeneration(
  params: GenerationParams,
  onComplete: (url: string) => void
) {
  return queue.add(async () => {
    const result = await generateImage(params);
    onComplete(result.imageUrl);
    return result;
  });
}
```

PQueue is lightweight and integrates cleanly with React state. Display the queue position to users ("3 images remaining in queue") so they know their request is being processed, not lost.

## FAQ

**Q: Is there a monthly flat fee for fal.ai?**

A: No — fal.ai is fully pay-per-use. You're only charged for completed generation requests. As of April 2026, new accounts receive approximately $5 in free credits, making development and testing essentially free. For production workloads, you simply top up as needed.

**Q: How do I add Supabase Edge Functions to a Rork project?**

A: Tell Rork: "Please create a Supabase Edge Function that accepts image generation requests and proxies them to fal.ai." Rork Max will generate the complete function code along with Supabase CLI setup instructions. If you want to use the code from this article directly, paste it in and say "Create an Edge Function with this code." Rork will wire up the deployment configuration automatically.

**Q: What are the most common reasons AI generation apps get rejected from the App Store?**

A: The three most common rejection triggers are: (1) **Insufficient moderation** — the app can produce inappropriate content without any guardrails (fix: Safety Checker + keyword filters + content policy); (2) **Missing AI disclosure** — no clear indication that images are AI-generated (fix: add an "AI Generated" badge to each image); (3) **Weak privacy policy** — it's unclear whether user prompts are stored server-side or used for model training (fix: explicitly document your data handling practices). Address all three before submission.

**Q: How do I prevent credit abuse from users calling the generation API directly?**

A: Never expose fal.ai or Replicate API keys to the client. Always route through your own Edge Function, which validates the user's JWT (`supabase.auth.getUser(token)`) before processing any request. Layering on a rate limit — for example, no more than 3 requests per minute per user — provides additional protection against bulk scraping of credits.
