## Introduction — Why Build an AI Business Card Scanner?

After receiving a business card at a networking event, manually typing contact details into your phone is tedious and error-prone. An AI-powered business card scanner app solves this problem by letting you snap a photo and automatically extract names, company names, email addresses, phone numbers, and more — saving them directly to your contacts.

In this tutorial, we'll build a complete AI business card scanner app from scratch using Rork. We'll cover camera capture, OCR (Optical Character Recognition) text extraction, AI-powered structured data conversion, and automatic contact import on the device. Every step includes working code examples you can adapt for your own projects.

**What you'll learn:**

- Implementing camera capture with Expo Camera
- OCR text recognition using Google Cloud Vision API
- Structuring extracted text with Claude API / Gemini API
- Auto-saving contacts with Expo Contacts
- Managing scan history and UI design best practices

**Target audience:** Intermediate developers who understand Rork basics and want to build a practical app combining AI APIs with native device features.

## Implementing Camera Capture

The camera is the entry point of our business card scanner. Using `expo-camera`, we can access the native camera directly within a Rork project.

```typescript
// CameraScreen.tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useState, useRef } from 'react';
import { View, TouchableOpacity, Text, StyleSheet } from 'react-native';

export default function CameraScreen({ onCapture }: { onCapture: (uri: string) => void }) {
  const [permission, requestPermission] = useCameraPermissions();
  const cameraRef = useRef<CameraView>(null);

  // Check and request camera permission
  if (\!permission?.granted) {
    return (
      <View style={styles.container}>
        <Text style={styles.message}>Camera access is required</Text>
        <TouchableOpacity style={styles.button} onPress={requestPermission}>
          <Text style={styles.buttonText}>Grant Access</Text>
        </TouchableOpacity>
      </View>
    );
  }

  const handleCapture = async () => {
    if (cameraRef.current) {
      // Capture at high quality for better OCR accuracy
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.9,
        base64: true,
      });
      if (photo?.uri) {
        onCapture(photo.uri);
      }
    }
  };

  return (
    <View style={styles.container}>
      <CameraView ref={cameraRef} style={styles.camera} facing="back">
        {/* Card alignment guide */}
        <View style={styles.overlay}>
          <View style={styles.cardGuide} />
          <Text style={styles.guideText}>Align the card within the frame</Text>
        </View>
      </CameraView>
      <TouchableOpacity style={styles.captureButton} onPress={handleCapture}>
        <Text style={styles.captureText}>Capture</Text>
      </TouchableOpacity>
    </View>
  );
}
// Expected behavior: Camera preview is displayed, and pressing
// the capture button returns the image URI via callback
```

The key detail here is setting `quality: 0.9` to capture at high resolution. OCR accuracy depends heavily on image quality, so avoid excessive compression. Adding a guide frame overlay also helps users position the card correctly within the viewfinder.

You can also add gallery selection using `expo-image-picker` for previously photographed cards:

```typescript
import * as ImagePicker from 'expo-image-picker';

const pickFromGallery = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ['images'],
    quality: 0.9,
    base64: true,
  });
  if (\!result.canceled && result.assets[0]) {
    onCapture(result.assets[0].uri);
  }
};
// Expected behavior: Gallery opens, and the selected image URI is returned
```

## OCR Text Recognition

To extract text from the captured business card image, we'll use Google Cloud Vision API's OCR capabilities. Cloud Vision handles mixed-language text well, making it ideal for business cards that may contain both English and local language text.

```typescript
// services/ocrService.ts
import * as FileSystem from 'expo-file-system';

const VISION_API_ENDPOINT = 'https://vision.googleapis.com/v1/images:annotate';

export async function extractTextFromImage(imageUri: string): Promise<string> {
  // Convert image to Base64
  const base64Image = await FileSystem.readAsStringAsync(imageUri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // Send request to Cloud Vision API
  const response = await fetch(
    `${VISION_API_ENDPOINT}?key=YOUR_CLOUD_VISION_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        requests: [
          {
            image: { content: base64Image },
            features: [
              { type: 'TEXT_DETECTION', maxResults: 1 },
            ],
            imageContext: {
              languageHints: ['en', 'ja'],
            },
          },
        ],
      }),
    }
  );

  const data = await response.json();
  const fullText = data.responses?.[0]?.fullTextAnnotation?.text || '';
  return fullText;
}
// Expected output example:
// "Acme Corporation\nJohn Smith\nProduct Manager\n
//  john@acme.com\n+1-555-0123\nwww.acme.com"
```

**Tips for improving OCR accuracy:**

- Ensure image resolution is at least 1000px
- Set `languageHints` to match expected languages on the card
- Encourage users to capture in even lighting without flash
- Consider adding image deskew preprocessing for tilted captures

As an alternative to Cloud Vision, ML Kit Text Recognition can run OCR on-device without network connectivity. However, for business use cases requiring high accuracy across multiple languages, the Cloud API approach generally performs better.

## AI-Powered Structured Data Extraction

Raw OCR text is just a block of strings — it doesn't tell us which line is a name, which is an email, or which is a phone number. This is where AI shines: it can understand context and accurately map unstructured text to structured fields.

Here's an implementation using the Claude API:

```typescript
// services/aiParserService.ts

interface BusinessCardData {
  name: string;
  company: string;
  department?: string;
  title?: string;
  email?: string;
  phone?: string;
  mobile?: string;
  fax?: string;
  website?: string;
  address?: string;
  postalCode?: string;
}

export async function parseBusinessCard(ocrText: string): Promise<BusinessCardData> {
  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY',
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify({
      model: 'claude-sonnet-4-5-20250514',
      max_tokens: 1024,
      messages: [
        {
          role: 'user',
          content: `The following text was extracted from a business card via OCR.
Extract the contact information and return it as JSON.

OCR Text:
${ocrText}

Return the following JSON format (use null for missing fields):
{
  "name": "Full name",
  "company": "Company name",
  "department": "Department",
  "title": "Job title",
  "email": "Email address",
  "phone": "Office phone",
  "mobile": "Mobile phone",
  "fax": "Fax number",
  "website": "Website URL",
  "address": "Street address",
  "postalCode": "Postal/ZIP code"
}

Return only the JSON. No explanations needed.`,
        },
      ],
    }),
  });

  const data = await response.json();
  const jsonText = data.content[0].text;
  return JSON.parse(jsonText) as BusinessCardData;
}
// Expected output:
// {
//   "name": "John Smith",
//   "company": "Acme Corporation",
//   "title": "Product Manager",
//   "email": "john@acme.com",
//   "phone": "+1-555-0123",
//   "website": "www.acme.com",
//   ...
// }
```

The biggest advantage of using AI here is its ability to handle varying layouts and formats. Every business card is different — text positions, font sizes, and formatting all vary. While regex-based parsing would break on edge cases, AI understands the semantic meaning and correctly assigns fields regardless of layout.

**Cost optimization tip:** Since business card text is relatively short, you can use `claude-haiku-4-5-20251001` to significantly reduce costs while maintaining good accuracy. For more on managing AI costs in your app, see [Rork AI API Cost Optimization Guide](/en/articles/rork-dev/rork-ai-api-cost-optimization-mobile-guide).

## Saving to Device Contacts

Once we have structured data, we can save it directly to the device's contact list using `expo-contacts`.

```typescript
// services/contactService.ts
import * as Contacts from 'expo-contacts';

export async function saveToContacts(card: BusinessCardData): Promise<string> {
  // Request contacts permission
  const { status } = await Contacts.requestPermissionsAsync();
  if (status \!== 'granted') {
    throw new Error('Contacts access was not granted');
  }

  // Build the contact object
  const contact: Contacts.Contact = {
    contactType: Contacts.ContactTypes.Person,
    name: card.name,
    firstName: card.name.split(' ')[0] || card.name,
    lastName: card.name.split(' ').slice(1).join(' ') || '',
    company: card.company,
    jobTitle: card.title || '',
    department: card.department || '',
    emails: card.email
      ? [{ email: card.email, label: 'work' }]
      : [],
    phoneNumbers: [
      ...(card.phone
        ? [{ number: card.phone, label: 'work' }]
        : []),
      ...(card.mobile
        ? [{ number: card.mobile, label: 'mobile' }]
        : []),
    ],
    urlAddresses: card.website
      ? [{ url: card.website, label: 'work' }]
      : [],
    addresses: card.address
      ? [{
          street: card.address,
          postalCode: card.postalCode || '',
          label: 'work',
        }]
      : [],
    note: `Scanned: ${new Date().toLocaleDateString('en-US')}`,
  };

  // Save the contact
  const contactId = await Contacts.addContactAsync(contact);
  return contactId;
}
// Expected behavior: A new entry is added to the device's contacts
```

It's also a good idea to check for duplicate contacts before saving:

```typescript
// Duplicate check
export async function checkDuplicate(name: string): Promise<boolean> {
  const { data } = await Contacts.getContactsAsync({
    fields: [Contacts.Fields.Name],
    name: name,
  });
  return data.length > 0;
}
```

## Scan Result Screen and History Management

The scan result screen lets users review and edit the extracted information before saving. Since AI parsing isn't always perfect, allowing manual corrections is essential for a good user experience.

```typescript
// ScanResultScreen.tsx
import { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, ScrollView } from 'react-native';

export default function ScanResultScreen({
  cardData,
  onSave,
}: {
  cardData: BusinessCardData;
  onSave: (data: BusinessCardData) => void;
}) {
  const [editData, setEditData] = useState(cardData);

  const fields = [
    { key: 'name', label: 'Name', icon: '👤' },
    { key: 'company', label: 'Company', icon: '🏢' },
    { key: 'title', label: 'Job Title', icon: '💼' },
    { key: 'email', label: 'Email', icon: '✉️' },
    { key: 'phone', label: 'Phone', icon: '📞' },
    { key: 'mobile', label: 'Mobile', icon: '📱' },
    { key: 'website', label: 'Website', icon: '🌐' },
    { key: 'address', label: 'Address', icon: '📍' },
  ];

  return (
    <ScrollView style={{ flex: 1, padding: 16 }}>
      <Text style={{ fontSize: 20, fontWeight: 'bold', marginBottom: 16 }}>
        Scan Results
      </Text>
      {fields.map(({ key, label, icon }) => (
        <View key={key} style={{ marginBottom: 12 }}>
          <Text style={{ fontSize: 14, color: '#666', marginBottom: 4 }}>
            {icon} {label}
          </Text>
          <TextInput
            value={editData[key as keyof BusinessCardData] || ''}
            onChangeText={(text) =>
              setEditData((prev) => ({ ...prev, [key]: text }))
            }
            style={{
              borderWidth: 1,
              borderColor: '#ddd',
              borderRadius: 8,
              padding: 12,
              fontSize: 16,
            }}
            placeholder={`Enter ${label.toLowerCase()}`}

      <TouchableOpacity
        style={{
          backgroundColor: '#007AFF',
          padding: 16,

          alignItems: 'center',
          marginTop: 16,
        }}
        onPress={() => onSave(editData)}
      >
        <Text style={{ color: '#fff', fontSize: 16, fontWeight: '600' }}>
          Save to Contacts
        </Text>
      </TouchableOpacity>
    </ScrollView>
  );
}
```

For scan history, use `AsyncStorage` or `expo-sqlite` to persist data locally so users can search and revisit previously scanned cards:

```typescript
// services/historyService.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const HISTORY_KEY = 'scan_history';

export async function addToHistory(
  card: BusinessCardData,
  imageUri: string
): Promise<void> {
  const history = await getHistory();
  history.unshift({
    id: Date.now().toString(),
    card,
    imageUri,
    scannedAt: new Date().toISOString(),
  });
  // Keep the latest 100 entries
  await AsyncStorage.setItem(
    HISTORY_KEY,
    JSON.stringify(history.slice(0, 100))
  );
}

export async function getHistory() {
  const json = await AsyncStorage.getItem(HISTORY_KEY);
  return json ? JSON.parse(json) : [];
}
// Expected behavior: Scan history is persisted and retrievable locally
```

## Security and Privacy Considerations

Business cards contain personal information, so security in your app is particularly important.

**API key management:** Never hardcode API keys in your app's source code. Instead, route requests through a server-side proxy using Supabase Edge Functions or Cloudflare Workers.

```typescript
// Server-side proxy request example
const response = await fetch('https://your-api-proxy.workers.dev/ocr', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${userToken}`,
  },
  body: JSON.stringify({ image: base64Image }),
});
```

**Image data handling:** When sending business card images to the cloud for OCR processing, clearly communicate that images are deleted immediately after processing. Consider offering an on-device-only mode that avoids cloud transmission entirely.

**Contacts access:** When requesting `expo-contacts` permissions, display a clear explanation of why the app needs access. Apple's guidelines require a specific purpose string, and transparent communication builds user trust.

## Frequently Asked Questions

### Q1: How accurate is the OCR recognition?

With Google Cloud Vision API, you can expect 90-95% accuracy on typical business cards. However, highly designed cards with decorative fonts or text overlaid on background images may see reduced accuracy. The AI structuring step helps compensate by understanding context even when individual characters are misread.

### Q2: Can this work offline?

Full offline functionality requires ML Kit Text Recognition for on-device OCR. ML Kit supports multiple languages, though accuracy is slightly lower than the Cloud API. The AI structuring step requires an internet connection. A hybrid approach is possible: run OCR offline and batch-process AI structuring when connectivity returns.

### Q3: How much do the APIs cost?

Google Cloud Vision API offers 1,000 free requests per month. For personal use, this is usually more than enough. Using Claude's Haiku model for text parsing costs approximately $0.0001 per request. Even scanning 500 cards per month would cost well under $1 in API fees.

### Q4: Are there App Store review considerations?

You'll need to provide clear usage descriptions for Camera and Contacts permissions in `Info.plist`. A privacy policy is mandatory since the app handles personal data. Apple scrutinizes camera and contacts access closely, so be specific about why each permission is needed.

### Q5: Can it handle double-sided cards?

Yes — capture each side separately and pass both OCR results to the AI with a prompt like "The following are OCR results from the front and back of a business card. Merge both sides into a single contact record." The AI will intelligently combine the information from both sides.
