## Introduction — Snap a Photo, Get Your Calories

One of the biggest friction points in any diet or health tracking routine is manually logging every meal. Searching for each ingredient, estimating portion sizes, and entering numbers takes time — and most people give up within a week.

In this tutorial, we'll build an AI-powered diet tracking app using Rork and Gemini Vision API. Users simply take a photo of their meal, and the AI automatically identifies the dishes, estimates calories, and breaks down macronutrients (protein, fat, and carbohydrates). No manual entry required.

What you'll learn:

- How to call Gemini Vision API from a Rork app to analyze food images
- Implementing the full flow from camera capture to AI analysis to data persistence
- Building a daily and weekly nutrition dashboard UI
- Persisting meal data locally with AsyncStorage

This guide is aimed at intermediate Rork developers. If you're just getting started, check out [The Complete Guide to Building Your First Rork App](/articles/rork-basics/rork-zero-to-app-complete-guide) first.

## App Architecture Overview

Let's start by mapping out the screens and data flow for our diet tracker.

**Screen Structure (4 screens):**

- **Home Screen**: Today's meal log with a calorie and nutrient summary
- **Camera Screen**: Capture a food photo or select from the library
- **Analysis Results Screen**: Review and optionally edit AI-estimated nutrition data
- **Weekly Report Screen**: A 7-day calorie trend chart

**Data Flow:**

1. User captures a food photo with the camera
2. The image is Base64-encoded and sent to Gemini Vision API
3. The AI returns dish names, estimated calories, and macros as JSON
4. User reviews the results and makes manual corrections if needed
5. Data is saved to AsyncStorage
6. The home screen and weekly report update accordingly

## Analyzing Food Images with Gemini Vision API

The core of this app is Gemini's image understanding capability. With a well-crafted prompt, we can extract dish names and nutritional information from a single photo.

Here's the service module that handles the API call:

```typescript
// services/foodAnalyzer.ts
// Service to analyze food images using Gemini Vision API

const GEMINI_API_URL = "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent";

interface NutritionResult {
  dishName: string;
  calories: number;
  protein: number;   // grams
  fat: number;       // grams
  carbs: number;     // grams
  confidence: "high" | "medium" | "low";
}

export async function analyzeFoodImage(
  base64Image: string,
  apiKey: string
): Promise<NutritionResult[]> {
  const prompt = `
    Analyze this food photo.
    For each dish in the image, return a JSON array with:
    - dishName: name of the dish (in English)
    - calories: estimated calories (kcal, integer)
    - protein: protein content (g, 1 decimal place)
    - fat: fat content (g, 1 decimal place)
    - carbs: carbohydrate content (g, 1 decimal place)
    - confidence: estimation confidence ("high", "medium", "low")
    
    Base estimates on a standard single serving.
    Return ONLY the JSON array, no other text.
  `;

  const response = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      contents: [{
        parts: [
          { text: prompt },
          {
            inlineData: {
              mimeType: "image/jpeg",
              data: base64Image,
            },
          },
        ],
      }],
    }),
  });

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;

  // Extract and parse the JSON portion
  const jsonMatch = text.match(/\[[\s\S]*\]/);
  if (!jsonMatch) throw new Error("Failed to parse AI response");

  return JSON.parse(jsonMatch[0]) as NutritionResult[];
}
```

The key design decision here is instructing the AI to return "ONLY the JSON array" — this makes response parsing straightforward and reliable.

## Camera Capture and Image Handling

Rork leverages `expo-image-picker` for camera access and image selection. Here's a custom hook that handles the full capture-and-analyze flow:

```typescript
// hooks/useFoodCamera.ts
// Custom hook for camera capture and Base64 conversion

import * as ImagePicker from "expo-image-picker";
import { useState } from "react";
import { analyzeFoodImage } from "../services/foodAnalyzer";

export function useFoodCamera(apiKey: string) {
  const [isAnalyzing, setIsAnalyzing] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const captureAndAnalyze = async () => {
    setError(null);
    
    // Request camera permission
    const permission = await ImagePicker.requestCameraPermissionsAsync();
    if (!permission.granted) {
      setError("Camera access is required to analyze meals");
      return null;
    }

    // Launch the camera
    const result = await ImagePicker.launchCameraAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      quality: 0.7,       // Reduce quality to minimize upload size
      base64: true,        // Enable Base64 encoding
      allowsEditing: true, // Allow cropping
      aspect: [4, 3],
    });

    if (result.canceled || !result.assets[0].base64) return null;

    // Analyze the food with AI
    setIsAnalyzing(true);
    try {
      const nutrition = await analyzeFoodImage(
        result.assets[0].base64,
        apiKey
      );
      return {
        imageUri: result.assets[0].uri,
        nutrition,
        timestamp: new Date().toISOString(),
      };
    } catch (err) {
      setError("Failed to analyze the meal. Please try again.");
      return null;
    } finally {
      setIsAnalyzing(false);
    }
  };

  return { captureAndAnalyze, isAnalyzing, error };
}

// Usage:
// const { captureAndAnalyze, isAnalyzing } = useFoodCamera("YOUR_GEMINI_API_KEY");
// const meal = await captureAndAnalyze();
// → { imageUri: "file://...", nutrition: [{dishName: "Chicken Rice", calories: 580, ...}], timestamp: "..." }
```

Notice that we set `quality: 0.7` to reduce file size. Gemini API has request size limits, so sending full-resolution images can cause failures. A quality setting between 0.5 and 0.7 provides a good balance between recognition accuracy and transfer speed.

## Data Persistence and Calorie Aggregation

We need a storage layer to save meal records and compute daily totals. Here's the implementation using AsyncStorage:

```typescript
// services/mealStorage.ts
// Meal record persistence with AsyncStorage

import AsyncStorage from "@react-native-async-storage/async-storage";

interface MealRecord {
  id: string;
  imageUri: string;
  dishes: {
    dishName: string;
    calories: number;
    protein: number;
    fat: number;
    carbs: number;
  }[];
  totalCalories: number;
  timestamp: string;
}

const STORAGE_KEY = "meal_records";

// Save a new meal record
export async function saveMealRecord(meal: MealRecord): Promise<void> {
  const existing = await getMealRecords();
  existing.push(meal);
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(existing));
}

// Retrieve all meal records
export async function getMealRecords(): Promise<MealRecord[]> {
  const data = await AsyncStorage.getItem(STORAGE_KEY);
  return data ? JSON.parse(data) : [];
}

// Calculate daily nutrition totals for a given date
export async function getDailySummary(date: string) {
  const records = await getMealRecords();
  const dayRecords = records.filter(
    (r) => r.timestamp.startsWith(date)
  );

  return dayRecords.reduce(
    (sum, record) => ({
      totalCalories: sum.totalCalories + record.totalCalories,
      totalProtein: sum.totalProtein +
        record.dishes.reduce((s, d) => s + d.protein, 0),
      totalFat: sum.totalFat +
        record.dishes.reduce((s, d) => s + d.fat, 0),
      totalCarbs: sum.totalCarbs +
        record.dishes.reduce((s, d) => s + d.carbs, 0),
      mealCount: sum.mealCount + 1,
    }),
    { totalCalories: 0, totalProtein: 0, totalFat: 0, totalCarbs: 0, mealCount: 0 }
  );
}

// Get daily calorie data for the past 7 days (for chart display)
export async function getWeeklyCalories(): Promise<{ date: string; calories: number }[]> {
  const result: { date: string; calories: number }[] = [];
  
  for (let i = 6; i >= 0; i--) {
    const d = new Date();
    d.setDate(d.getDate() - i);
    const dateStr = d.toISOString().split("T")[0];
    const summary = await getDailySummary(dateStr);
    result.push({ date: dateStr, calories: summary.totalCalories });
  }
  
  return result;
}
```

The `getDailySummary` function filters records by matching the date prefix of the ISO timestamp. It's a simple approach, but perfectly adequate for a personal diet tracking app.

Here's the circular progress component:

```tsx
// components/CalorieRing.tsx
// Circular progress indicator for calorie goal tracking

import React from "react";
import { View, Text, StyleSheet } from "react-native";
import Svg, { Circle } from "react-native-svg";

interface Props {
  current: number;
  target: number;
}

export function CalorieRing({ current, target }: Props) {
  const progress = Math.min(current / target, 1);
  const radius = 80;
  const strokeWidth = 12;
  const circumference = 2 * Math.PI * radius;
  const strokeDashoffset = circumference * (1 - progress);

  return (
    <View style={styles.container}>
      <Svg width={200} height={200}>
        {/* Background circle */}
        <Circle
          cx={100} cy={100} r={radius}
          stroke="#E8F5E9"
          strokeWidth={strokeWidth}
          fill="none"
        />
        {/* Progress circle */}
        <Circle
          cx={100} cy={100} r={radius}
          stroke="#4CAF50"
          strokeWidth={strokeWidth}
          fill="none"
          strokeDasharray={circumference}
          strokeDashoffset={strokeDashoffset}
          strokeLinecap="round"
          rotation={-90}
          origin="100,100"
        />
      </Svg>
      <View style={styles.textContainer}>
        <Text style={styles.calories}>{current}</Text>
        <Text style={styles.unit}>/ {target} kcal</Text>
      </View>
    </View>
  );
}

// Usage: <CalorieRing current={1250} target={2000} />
// → Displays a green circular progress bar filled to 62.5%
```

## Improving AI Analysis Accuracy with Prompt Engineering

The quality of your food analysis hinges on prompt design. Here are techniques that significantly improve accuracy:

**1. Multi-dish handling**: Meals often contain multiple dishes on a single plate or tray. Our prompt explicitly asks for "each dish," which ensures the AI identifies items individually rather than treating the entire photo as one meal.

**2. Portion size estimation**: Adding context like "use the plate size and surrounding objects as scale references to estimate portion sizes" helps the AI make more accurate quantity estimates.

**3. Cuisine-specific optimization**: If your target audience primarily eats a specific cuisine, guide the AI accordingly. For Japanese food, reference the Standard Tables of Food Composition. For Western meals, USDA food databases work well.

```typescript
  [{ "dishName": "name", "calories": number, "protein": number, 
     "fat": number, "carbs": number, "confidence": "high|medium|low" }]
`;
```

## Common Errors and Troubleshooting

Here are the most common issues you'll encounter during development and how to fix them:

**Camera permissions not granted**: On iOS, you need to add `NSCameraUsageDescription` to `app.json` under `ios.infoPlist`. Include a clear description like "This app uses the camera to analyze your meals and estimate nutritional content."

**Slow Gemini API responses**: Large images take longer to process. Setting `quality` to 0.5–0.7 in the image picker provides adequate recognition accuracy while keeping transfer times reasonable.

**JSON parsing errors**: Occasionally, the AI may include explanatory text alongside the JSON. The regex extraction (`text.match(/\[[\s\S]*\]/)`) in our code handles this gracefully, but you should also implement a retry mechanism for edge cases.

**Inaccurate calorie estimates**: AI estimates are approximations, not medical-grade measurements. Always provide an edit screen where users can correct values, and clearly label results as "AI-estimated" in your UI.

## FAQ

### How accurate is the AI calorie estimation?

Gemini Vision API typically estimates calories within ±20–30% of actual values. Single-dish meals like a bowl of ramen or a plate of pasta tend to be more accurate, while mixed dishes or meals with hidden ingredients (sauces, oils) have larger margins of error. The app should clearly communicate that these are estimates and provide manual editing capabilities for users who need more precision.

### Can the app work offline?

The photo capture and AI analysis require an internet connection since we're calling the Gemini API. However, browsing past records and viewing charts works entirely offline thanks to AsyncStorage. For a more robust experience, you could queue photos taken offline and batch-analyze them when connectivity returns. Check out the [Rork Offline Support Guide](/articles/rork-dev/rork-offline-support) for implementation patterns.

### Can I run this within Gemini API's free tier?

Yes, Gemini API's free tier is sufficient for personal use and prototyping. Each image analysis costs roughly $0.001–$0.003. As your user base grows, you'll want to implement rate limiting and consider caching repeated dishes. For detailed API setup instructions, see the [Rork × Gemini Integration Guide](/articles/rork-ai/rork-gemini-integration).

### Can I add input methods beyond photos?

Absolutely. You can add text-based search where users type a dish name and the AI estimates its nutritional content via Gemini's text API. Barcode scanning for packaged foods is another popular addition — libraries like `expo-barcode-scanner` make this straightforward to implement in Rork.
