## Why Text-to-Speech Matters for Mobile Apps

Voice experiences in mobile apps have become a genuine competitive advantage. Whether you're building a navigation app that reads out directions when users' hands are occupied, adding accessibility support for visually impaired users, or creating a language learning app where pronunciation matters — text-to-speech (TTS) opens up a whole new layer of interaction.

Adding TTS to a Rork-powered React Native app is easier than you might think. This guide covers two approaches:

- **expo-speech**: Expo's built-in TTS library that calls iOS and Android's native speech engines with no extra cost or external API required.
- **ElevenLabs API**: One of the most capable AI voice synthesis services available today, delivering near-human quality audio in 28+ languages.

Knowing when to use each is the key — let's start with the simpler one.

---

## Instant TTS with expo-speech

`expo-speech` ships as part of the Expo SDK, so no additional installation is needed. It uses `AVSpeechSynthesizer` on iOS and the Android TTS engine on Android, giving you a consistent cross-platform experience out of the box.

### Basic Implementation

```tsx
import * as Speech from 'expo-speech';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export default function TTSBasicScreen() {
  const text = 'Hello! This is a text-to-speech demo for your Rork app.';

  const speak = () => {
    // Start speaking
    Speech.speak(text, {
      language: 'en-US',  // Language code
      pitch: 1.0,         // Voice pitch (0.5–2.0)
      rate: 0.9,          // Speaking rate (0.1–2.0)
      onStart: () => console.log('Speech started'),
      onDone: () => console.log('Speech finished'),
      onError: (err) => console.error('Speech error:', err),
    });
  };

  const stop = () => {
    // Stop speech mid-sentence
    Speech.stop();
  };

  return (
    <View style={styles.container}>
      <Text style={styles.label}>{text}</Text>
      <TouchableOpacity style={styles.button} onPress={speak}>
        <Text style={styles.buttonText}>▶ Speak</Text>
      </TouchableOpacity>
      <TouchableOpacity style={styles.stopButton} onPress={stop}>
        <Text style={styles.buttonText}>■ Stop</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 24 },
  label: { fontSize: 16, marginBottom: 24, textAlign: 'center', lineHeight: 26 },
  button: { backgroundColor: '#4F46E5', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8, marginBottom: 12 },
  stopButton: { backgroundColor: '#6B7280', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});

// Expected behavior:
// Tapping Speak reads the text aloud using the device's native voice
// Tapping Stop immediately interrupts playback
```

### Listing Available Voices

Devices ship with multiple voice options. You can let users choose their preferred voice by fetching the available list:

```tsx
import * as Speech from 'expo-speech';
import { useEffect, useState } from 'react';

export function useAvailableVoices(language: string) {
  const [voices, setVoices] = useState<Speech.Voice[]>([]);

  useEffect(() => {
    Speech.getAvailableVoicesAsync().then((all) => {
      // Filter to voices matching the requested language
      const filtered = all.filter((v) => v.language.startsWith(language));
      setVoices(filtered);
    });
  }, [language]);

  return voices;
}
```

Plug this hook into a picker component and you have a voice selector that feels native and polished.

## High-Quality AI Voice with ElevenLabs API

The system voices in `expo-speech` are functional, but they have a distinctly robotic quality. For language learning apps, storytelling, or any experience where voice quality is central to the product, **ElevenLabs API** is worth the upgrade.

### Setup

1. Create an account at [ElevenLabs](https://elevenlabs.io/) (free tier covers 10,000 characters/month)
2. Grab your API key from the dashboard
3. Install `expo-av` for audio playback:

```bash
npx expo install expo-av
```

### Calling the ElevenLabs API

```tsx
import { Audio } from 'expo-av';
import { useState } from 'react';
import { View, Text, TouchableOpacity, ActivityIndicator, StyleSheet } from 'react-native';

// ⚠️ Always use environment variables — never hardcode API keys
const ELEVENLABS_API_KEY = process.env.EXPO_PUBLIC_ELEVENLABS_API_KEY ?? '';
// Find your Voice ID in the ElevenLabs dashboard
const VOICE_ID = 'YOUR_VOICE_ID';

export default function ElevenLabsTTSScreen() {
  const [isLoading, setIsLoading] = useState(false);
  const [sound, setSound] = useState<Audio.Sound | null>(null);

  const speak = async (text: string) => {
    setIsLoading(true);
    try {
      // Request MP3 audio from ElevenLabs
      const response = await fetch(
        `https://api.elevenlabs.io/v1/text-to-speech/${VOICE_ID}`,
        {
          method: 'POST',
          headers: {
            'xi-api-key': ELEVENLABS_API_KEY,
            'Content-Type': 'application/json',
          },
          body: JSON.stringify({
            text,
            model_id: 'eleven_multilingual_v2', // Supports 28+ languages
            voice_settings: {
              stability: 0.5,        // Voice consistency (0–1)
              similarity_boost: 0.8, // Voice accuracy (0–1)
            },
          }),
        }
      );

      if (!response.ok) throw new Error(`API Error: ${response.status}`);

      // Convert the audio blob to a Base64 data URI for expo-av
      const audioBlob = await response.blob();
      const reader = new FileReader();
      reader.onload = async () => {
        const dataUri = reader.result as string;
        const { sound: newSound } = await Audio.Sound.createAsync({ uri: dataUri });
        setSound(newSound);
        await newSound.playAsync();
      };
      reader.readAsDataURL(audioBlob);
    } catch (error) {
      console.error('TTS error:', error);
    } finally {
      setIsLoading(false);
    }
  };

  const stopSound = async () => {
    if (sound) {
      await sound.stopAsync();
      await sound.unloadAsync();
      setSound(null);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>AI Voice Demo</Text>
      {isLoading ? (
        <ActivityIndicator size="large" color="#4F46E5" />
      ) : (
        <TouchableOpacity
          style={styles.button}
          onPress={() => speak('Hello! This is ElevenLabs AI voice — sounds pretty natural, right?')}
        >
          <Text style={styles.buttonText}>▶ Speak with AI Voice</Text>
        </TouchableOpacity>
      )}
      <TouchableOpacity style={styles.stopButton} onPress={stopSound}>
        <Text style={styles.buttonText}>■ Stop</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center', padding: 24 },
  title: { fontSize: 20, fontWeight: '700', marginBottom: 32 },
  button: { backgroundColor: '#4F46E5', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8, marginBottom: 12 },
  stopButton: { backgroundColor: '#6B7280', paddingVertical: 14, paddingHorizontal: 32, borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});

// Expected behavior:
// Tapping the button triggers an API call; a loading spinner shows during the request
// The AI-generated audio plays automatically once the response is received
// Tapping Stop halts playback immediately
```

> **Cost tip**: ElevenLabs charges by character count. Cache audio for static or repeated phrases, and consider trimming long texts before sending to reduce API usage.

## Common Errors and How to Fix Them

### expo-speech Plays Silently on iOS Simulator

The iOS simulator's audio output can be unreliable. **Always test on a real device** for TTS features. If you must use the simulator, enable audio under Simulator → Device → Toggle Audio.

### ElevenLabs Returns a 401 Error

This means your API key isn't being read correctly. Make sure your environment variable is prefixed with `EXPO_PUBLIC_` so Expo exposes it to the client bundle. Store it in `.env.local` and add that file to `.gitignore` to keep it out of version control.

### Audio Feels Laggy or Choppy

ElevenLabs requires a network round-trip, so there will always be some latency. Two strategies help:

- **Split text into sentences** and start playing the first segment while fetching the rest.
- **Pre-generate and cache audio** for known static content — record the MP3 once and store it locally, then just play it on demand.

## FAQ

**Q1. Is expo-speech free to use?**

Yes, `expo-speech` is part of the Expo SDK and carries no additional cost. It runs entirely on-device using the system TTS engine, which also means it works offline. The trade-off is that voice quality depends on what the user's device has installed.

**Q2. Can I use languages other than English?**

Absolutely. Pass any IETF language tag to `expo-speech` — `ja-JP` for Japanese, `es-ES` for Spanish, `fr-FR` for French, and so on. ElevenLabs' `eleven_multilingual_v2` model auto-detects the language of your input text and responds accordingly.

**Q3. Will TTS keep playing when the app goes to the background?**

Basic `expo-speech` is tied to the foreground. For background audio, use `expo-av` with `Audio.setAudioModeAsync({ staysActiveInBackground: true })`. When playing ElevenLabs-generated MP3s through `expo-av`, background playback works well with this setting.

**Q4. Can I combine TTS with speech-to-text (STT)?**

Yes — and it's a powerful combination. Use Whisper to convert user speech to text, process it with an AI model, then read the response back with TTS. [Building a Voice Recognition App with Rork and OpenAI Whisper API](/articles/rork-ai/rork-whisper-voice-recognition) walks through the STT side, and this article covers the TTS side — together they give you a full voice interface.
