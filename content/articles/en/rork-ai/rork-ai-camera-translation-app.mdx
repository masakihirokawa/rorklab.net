## Introduction — Breaking Language Barriers with Your Camera

We've all been there: standing in front of a foreign-language menu at a restaurant, squinting at a sign in an unfamiliar country, or trying to decipher a product label during an overseas trip. Camera translation apps solve this everyday problem elegantly — point your phone at any text, tap a button, and get an instant translation.

In this tutorial, you'll build a fully functional camera translation app using **Rork**, **Google Cloud Vision API** (for OCR), and **Cloud Translation API**. By the end, you'll have a cross-platform iOS/Android app that captures images, extracts text, and translates it into the language of your choice.

What you'll learn in this article:

- How to integrate camera capture in Rork using `expo-camera`
- Sending images to Google Cloud Vision API for optical character recognition (OCR)
- Translating recognized text with Cloud Translation API
- Designing a clean and intuitive translation UI

**Who this is for**: Rork users who are comfortable with the basics and ready to tackle API integration.

## Prerequisites and Setup

### What You'll Need

Before diving in, make sure you have the following ready:

- A **Rork account** (Free or Max plan)
- A **Google Cloud Platform (GCP) account** (free tier available)
- A physical smartphone for camera testing

### Enabling Google Cloud APIs

Log in to the [GCP Console](https://console.cloud.google.com) and follow these steps:

1. Create a new project (e.g., `camera-translate-app`)
2. Go to **APIs & Services** → **Library**
3. Search for **Cloud Vision API** and enable it
4. Search for **Cloud Translation API** and enable it
5. Navigate to **Credentials** → **Create API Key** and save your key

You'll use this API key in Rork. In the code examples below, we use `YOUR_GOOGLE_API_KEY` as a placeholder — replace it with your actual key before testing.

## Camera Integration and Image Capture

### Using expo-camera

Rork is built on React Native/Expo, which means `expo-camera` handles camera access out of the box. Use this prompt to generate the camera component:

```
Use expo-camera to implement:
- Camera permission request on first launch
- Front/rear camera toggle button
- Capture button that takes a still photo
- Return the captured image as a Base64 string for API use
```

Here's the core of what Rork will generate:

```tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useRef, useState } from 'react';

export default function CameraScreen() {
  const [permission, requestPermission] = useCameraPermissions();
  const [facing, setFacing] = useState<'back' | 'front'>('back');
  const cameraRef = useRef<CameraView>(null);

  // Capture a photo and return it as Base64
  const takePicture = async () => {
    if (!cameraRef.current) return;
    const photo = await cameraRef.current.takePictureAsync({
      base64: true,   // Required for Vision API
      quality: 0.8,   // Balance quality vs. file size
    });
    return photo.base64;
  };

  return (
    <CameraView ref={cameraRef} facing={facing} style={{ flex: 1 }}>
      {/* Camera UI goes here */}
    </CameraView>
  );
}
```

**Why `base64: true`?** The Google Cloud Vision API accepts images as Base64-encoded strings, so requesting this format at capture time saves an extra conversion step.

## Implementing OCR with Google Cloud Vision API

### Extracting Text from Images

Once you have the Base64 image, send it to the Vision API to extract any text it contains.

```typescript
// utils/visionApi.ts

const VISION_API_KEY = 'YOUR_GOOGLE_API_KEY';
const VISION_API_URL = `https://vision.googleapis.com/v1/images:annotate?key=${VISION_API_KEY}`;

export async function detectTextFromImage(base64Image: string): Promise<string> {
  const requestBody = {
    requests: [
      {
        image: {
          content: base64Image,
        },
        features: [
          {
            type: 'TEXT_DETECTION',
            maxResults: 1,
          },
        ],
      },
    ],
  };

  const response = await fetch(VISION_API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestBody),
  });

  const data = await response.json();
  const fullText = data.responses?.[0]?.fullTextAnnotation?.text;

  if (!fullText) {
    throw new Error('No text detected in the image.');
  }

  // Expected output example: "Hello World\nThis is a sample text."
  return fullText;
}
```

**`TEXT_DETECTION` vs. `DOCUMENT_TEXT_DETECTION`**: Use `TEXT_DETECTION` for short, sparse text like signs and labels. Switch to `DOCUMENT_TEXT_DETECTION` for structured documents like menus, brochures, or forms — it performs better with multi-column layouts.

## Implementing Translation with Cloud Translation API

### Translating the Recognized Text

With the extracted text in hand, pass it to the Translation API:

```typescript
// utils/translationApi.ts

const TRANSLATION_API_KEY = 'YOUR_GOOGLE_API_KEY';
const TRANSLATION_API_URL = `https://translation.googleapis.com/language/translate/v2?key=${TRANSLATION_API_KEY}`;

export const LANGUAGES: Record<string, string> = {
  'ja': 'Japanese',
  'en': 'English',
  'zh': 'Chinese (Simplified)',
  'ko': 'Korean',
  'fr': 'French',
  'es': 'Spanish',
  'de': 'German',
};

export async function translateText(
  text: string,
  targetLanguage: string
): Promise<{ translatedText: string; detectedSourceLanguage: string }> {
  const requestBody = {
    q: text,
    target: targetLanguage,
    format: 'text',
    // Omitting 'source' lets the API auto-detect the source language
  };

  const response = await fetch(TRANSLATION_API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(requestBody),
  });

  const data = await response.json();
  const translation = data.data?.translations?.[0];

  if (!translation) {
    throw new Error('Translation failed.');
  }

  return {
    translatedText: translation.translatedText,
    detectedSourceLanguage: translation.detectedSourceLanguage ?? 'unknown',
    // Expected: { translatedText: "こんにちは世界", detectedSourceLanguage: "en" }
  };
}
```

**Automatic language detection**: By omitting the `source` parameter, the API will automatically identify the source language. This makes the app feel natural — users simply point and translate without worrying about settings.

## Putting It All Together

### The Main Processing Flow

Now combine the OCR and translation steps into a single handler:

```tsx
// screens/ResultScreen.tsx

import { useState } from 'react';
import { detectTextFromImage } from '../utils/visionApi';
import { translateText } from '../utils/translationApi';

export default function ResultScreen({ capturedBase64 }: { capturedBase64: string }) {
  const [detectedText, setDetectedText] = useState('');
  const [translatedText, setTranslatedText] = useState('');
  const [targetLang, setTargetLang] = useState('ja');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  const handleTranslate = async () => {
    setIsLoading(true);
    setError('');
    try {
      // Step 1: Extract text via OCR
      const text = await detectTextFromImage(capturedBase64);
      setDetectedText(text);

      // Step 2: Translate the extracted text
      const result = await translateText(text, targetLang);
      setTranslatedText(result.translatedText);
    } catch (err) {
      setError('Something went wrong. Please try again.');
    } finally {
      setIsLoading(false);
    }
  };

  // Render the UI...
}
```

## Common Errors and How to Fix Them

### 403 Forbidden or Invalid API Key

**Symptom**: The API returns a `403` error or "API key not valid" message.

**Fix**: Check the API key restrictions in the GCP Console. During development, you can leave restrictions off. For production, set Android/iOS app restrictions. Also confirm that both Vision API and Translation API are enabled for your project.

### No Text Detected

**Symptom**: The app returns "No text detected in the image."

**Fix**: Check the following:

- Is the image in focus and well-lit?
- Is the text large enough relative to the frame?
- Try switching to `DOCUMENT_TEXT_DETECTION` for dense text
- Increase `quality` in `takePictureAsync` to `1.0` for critical use cases

### Awkward or Garbled Translations

**Symptom**: The translated text is grammatically odd or doesn't make sense.

**Fix**: OCR output often includes extra newlines and symbols. Pre-process the text before translation:

```typescript
const cleanedText = rawText.replace(/\n+/g, ' ').trim();
```

This simple cleanup can noticeably improve translation quality.

## FAQ

**Q1. Is Google Cloud Vision API really free?**

A. Yes — up to 1,000 units per month at no charge (as of April 2026). Beyond that, TEXT_DETECTION is priced at $1.50 per 1,000 units. For a personal or small-scale app, you'll likely stay within the free tier. Set up a budget alert in GCP to avoid surprises.

**Q2. Can I build this on the Rork Free plan?**

A. You can develop and preview the app on Free, but testing camera functionality on a real device and publishing to the App Store or Google Play requires Rork Max. A good approach is to build and iterate in Free, then upgrade when you're ready for device testing.

**Q3. How do I improve translation accuracy?**

A. Three things help the most: ① Better photos (good lighting, stable hands, appropriate distance), ② Pre-processing the OCR output to remove noise, and ③ Switching to a specialized API like DeepL for domain-specific text (medical, legal, technical). For most general use cases, Google's Translation API performs very well.

**Q4. Does this work on both iOS and Android?**

A. Yes. Because Rork is built on React Native/Expo, your code runs on both platforms from a single codebase. Remember to configure camera permission descriptions separately: `expo.ios.infoPlist.NSCameraUsageDescription` for iOS and `expo.android.permissions` for Android in your `app.json`.

**Q5. Is it safe to include the API key directly in the code?**

A. For development, it's acceptable but not ideal. For production, move the API calls to a backend service (Supabase Edge Functions or Cloudflare Workers) so the key never reaches client devices. As a minimum precaution during development, restrict the key to specific IP addresses in the GCP Console.
