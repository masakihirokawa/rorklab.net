### Implementing the Gemini API Call

The core of our API integration is a service function. The key insight is getting Gemini to respond with **structured JSON** rather than free-form text.

```javascript
// services/geminiService.js
const GEMINI_API_KEY = 'YOUR_API_KEY'; // Use environment variables in production
const API_URL = `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=${GEMINI_API_KEY}`;

export async function generateTravelPlan({ destination, days, budget, style }) {
  const budgetMap = { low: 'Budget-friendly (under $50/day)', medium: 'Moderate ($100-200/day)', high: 'Luxury (no budget limit)' };
  const styleMap = { sightseeing: 'Tourist attractions & landmarks', food: 'Food & culinary experiences', activity: 'Adventure & outdoor activities', relaxing: 'Relaxation & leisure' };

  const prompt = `
You are a travel planner. Create a travel itinerary in JSON format based on these conditions:

Conditions:
- Destination: ${destination}
- Duration: ${days} days
- Budget: ${budgetMap[budget]}
- Style: ${styleMap[style]}

Respond with ONLY the following JSON format (no explanatory text):
{
  "title": "Plan title",
  "summary": "Brief overview of the plan (2 sentences)",
  "days": [
    {
      "day": 1,
      "theme": "Theme for the day",
      "spots": [
        {
          "name": "Spot name",
          "description": "Description (about 50 words)",
          "duration": "Estimated time (e.g., 2 hours)",
          "cost": "Estimated cost (e.g., $15)",
          "tip": "Quick tip or advice"
        }
      ]
    }
  ],
  "totalBudget": "Total estimated cost",
  "packingTips": ["Packing tip 1", "Packing tip 2"]
}`;

  const response = await fetch(API_URL, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: {
        temperature: 0.8,       // Balance between creativity and accuracy
        maxOutputTokens: 4096,  // Accommodate longer itineraries
        responseMimeType: 'application/json', // Force JSON output
      },
    }),
  });

  const data = await response.json();
  // Extract the text content from Gemini's response
  const text = data.candidates[0].content.parts[0].text;
  return JSON.parse(text);
}
// Expected output: A structured travel plan JSON object
```

### Prompt Design Best Practices

To get high-quality travel plans from AI, here are the key prompt engineering principles at work.

**1. Be explicit about output format** — Including the JSON schema directly in the prompt ensures you get parseable, structured data. Setting `responseMimeType: 'application/json'` prevents Gemini from returning anything other than JSON.

**2. Provide concrete constraints** — Instead of passing abstract labels like "low/medium/high" for budget, we translate them into specific dollar ranges. This gives the AI much more useful context to work with.

**3. Tune the temperature** — A value of `0.8` strikes a nice balance. Your plans will have variety without being wildly impractical. Lower it to `0.6` if you prefer more predictable, conventional suggestions.

## Building the UI — Input Form and Results Display

### The Input Screen Component

Here's the form where users enter their trip preferences.

```javascript
// screens/HomeScreen.jsx
import React, { useState } from 'react';
import { View, Text, TextInput, TouchableOpacity, ScrollView, ActivityIndicator } from 'react-native';
import { Picker } from '@react-native-picker/picker';
import { generateTravelPlan } from '../services/geminiService';

export default function HomeScreen({ navigation }) {
  const [destination, setDestination] = useState('');
  const [days, setDays] = useState(3);
  const [budget, setBudget] = useState('medium');
  const [style, setStyle] = useState('sightseeing');
  const [loading, setLoading] = useState(false);

  const handleGenerate = async () => {
    if (!destination.trim()) {
      alert('Please enter a destination');
      return;
    }
    setLoading(true);
    try {
      const plan = await generateTravelPlan({ destination, days, budget, style });
      navigation.navigate('Plan', { plan });
    } catch (error) {
      alert('Failed to generate the plan. Please try again.');
      console.error('Gemini API error:', error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <ScrollView style={{ flex: 1, padding: 20, backgroundColor: '#F8FAFB' }}>
      <Text style={{ fontSize: 28, fontWeight: 'bold', marginBottom: 8 }}>
        AI Travel Planner
      </Text>
      <Text style={{ fontSize: 16, color: '#666', marginBottom: 24 }}>
        Enter your preferences and let AI craft the perfect itinerary
      </Text>

      <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 6 }}>Destination</Text>
      <TextInput
        value={destination}
        onChangeText={setDestination}
        placeholder="e.g., Kyoto, Bangkok, Barcelona"
        style={{ borderWidth: 1, borderColor: '#DDD', borderRadius: 12, padding: 14, marginBottom: 16, fontSize: 16, backgroundColor: '#FFF' }}
      />

      <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 6 }}>Trip Duration</Text>
      <Picker selectedValue={days} onValueChange={setDays} style={{ marginBottom: 16 }}>
        {[...Array(14)].map((_, i) => (
          <Picker.Item key={i + 1} label={`${i + 1} day${i > 0 ? 's' : ''}`} value={i + 1} />
        ))}
      </Picker>

      <TouchableOpacity
        onPress={handleGenerate}
        disabled={loading}
        style={{ backgroundColor: loading ? '#AAA' : '#4A90D9', borderRadius: 12, padding: 16, alignItems: 'center', marginTop: 16 }}
      >
        {loading ? (
          <ActivityIndicator color="#FFF" />
        ) : (
          <Text style={{ color: '#FFF', fontSize: 18, fontWeight: 'bold' }}>Generate Plan</Text>
        )}
      </TouchableOpacity>
    </ScrollView>
  );
}
// Expected behavior: After filling in the form and tapping "Generate Plan",
// the Gemini API is called and the user is navigated to the results screen
```

### The Results Screen — Day-by-Day Cards

This screen takes the JSON from Gemini and renders it as a clean, card-based itinerary.

```javascript
// screens/PlanScreen.jsx
import React from 'react';
import { View, Text, ScrollView, TouchableOpacity } from 'react-native';

export default function PlanScreen({ route, navigation }) {
  const { plan } = route.params;

  return (
    <ScrollView style={{ flex: 1, backgroundColor: '#F8FAFB', padding: 16 }}>
      <Text style={{ fontSize: 24, fontWeight: 'bold', marginBottom: 4 }}>{plan.title}</Text>
      <Text style={{ fontSize: 14, color: '#666', marginBottom: 20 }}>{plan.summary}</Text>

      {plan.days.map((day) => (
        <View key={day.day} style={{ backgroundColor: '#FFF', borderRadius: 16, padding: 16, marginBottom: 12, shadowColor: '#000', shadowOpacity: 0.05, shadowRadius: 8 }}>
          <Text style={{ fontSize: 18, fontWeight: '700', color: '#4A90D9', marginBottom: 8 }}>
            Day {day.day}: {day.theme}
          </Text>
          {day.spots.map((spot, idx) => (
            <TouchableOpacity
              key={idx}
              onPress={() => navigation.navigate('Detail', { spot })}
              style={{ paddingVertical: 8, borderBottomWidth: idx < day.spots.length - 1 ? 1 : 0, borderBottomColor: '#EEE' }}
            >
              <Text style={{ fontSize: 16, fontWeight: '600' }}>{spot.name}</Text>
              <Text style={{ fontSize: 13, color: '#888', marginTop: 2 }}>
                {spot.duration} · {spot.cost}
              </Text>
            </TouchableOpacity>
          ))}
        </View>
      ))}

      <View style={{ backgroundColor: '#E8F4FD', borderRadius: 12, padding: 16, marginTop: 8, marginBottom: 32 }}>
        <Text style={{ fontSize: 16, fontWeight: '700', marginBottom: 8 }}>Estimated Total: {plan.totalBudget}</Text>
        <Text style={{ fontSize: 14, fontWeight: '600', marginBottom: 4 }}>Packing Tips:</Text>
        {plan.packingTips.map((tip, i) => (
          <Text key={i} style={{ fontSize: 14, color: '#555' }}>• {tip}</Text>
        ))}
      </View>
    </ScrollView>
  );
}
// Expected behavior: Day cards stack vertically, tapping any spot navigates to the detail screen
```

## Error Handling and UX Improvements

In a real-world app, you need to gracefully handle network failures and API rate limits.

### Implementing Retry Logic

```javascript
// utils/retry.js
export async function withRetry(fn, maxRetries = 3, delay = 1000) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxRetries) throw error;
      // Exponential backoff: 1s → 2s → 4s
      await new Promise(resolve => setTimeout(resolve, delay * Math.pow(2, attempt - 1)));
    }
  }
}

// Usage
const plan = await withRetry(() =>
  generateTravelPlan({ destination, days, budget, style })
);
// Expected behavior: Retries up to 3 times on API error; throws only after all retries fail
```

### Keeping Your API Key Safe

Hard-coding the key is fine during development, but for production you should always manage API keys server-side — using Supabase Edge Functions, Cloudflare Workers, or a similar approach. Embedding keys in client-side code makes them vulnerable to reverse engineering.

```javascript
// Example: Hiding the API key behind a Supabase Edge Function
// supabase/functions/generate-plan/index.ts
const GEMINI_KEY = Deno.env.get('GEMINI_API_KEY');

Deno.serve(async (req) => {
  const { destination, days, budget, style } = await req.json();
  // Call Gemini API here and return the result
  // The client only needs to call this Supabase Function URL
});
// Expected behavior: API key stays server-side only and is never exposed to the client
```

## Frequently Asked Questions (FAQ)

### Q1: How much can I use the Gemini API for free?

As of March 2026, Gemini 2.0 Flash's free tier allows 15 requests per minute and 1,500 requests per day. That's more than enough for personal projects and early-stage apps. If your user base grows significantly, you can upgrade to a pay-as-you-go plan.

### Q2: Can I use a different AI model instead of Gemini?

Absolutely. Just swap out the API endpoint and request format in `geminiService.js`, and you can use OpenAI's GPT, Anthropic's Claude, or any other provider. The prompt structure transfers almost directly. For a deeper dive into AI integrations, check out the [Rork × Gemini Integration Guide](/en/articles/rork-ai/rork-gemini-integration).

### Q3: How can I improve the quality of generated plans?

Add more specific constraints to your prompt — things like "include hidden gems that locals recommend" or "account for travel time between locations." You can also lower the `temperature` to around `0.6` for more conventional, tried-and-true recommendations.

### Q4: Can this work offline?

Plan generation requires an API call, so you need an internet connection for that step. However, you can save generated plans to AsyncStorage so users can view their itineraries offline anytime.

### Q5: How do I add map integration?

You can instruct Gemini to include coordinates for each spot in its response, then display them using `react-native-maps`. For a detailed walkthrough on map integration, see the [MapView and Spot Sharing App Tutorial](/en/articles/rork-dev/rork-mapview-supabase-spot-sharing-app).
