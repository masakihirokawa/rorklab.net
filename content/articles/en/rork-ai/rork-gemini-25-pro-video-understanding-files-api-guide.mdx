Imagine handing your app a video and having it come back with a thoughtful, detailed response about what happened in it. That's what Gemini 2.5 Pro's Video Understanding capability makes possible — and it's now within reach for indie developers building with Rork.

The hardest part isn't the AI itself. It's figuring out *which upload method to use* and *how to handle the async processing lifecycle* without introducing subtle bugs that only show up in production. Most documentation glosses over this, which is why so many implementations end up with intermittent failures that are hard to debug.

This guide walks through everything from that foundational decision all the way to shipping a monetizable app.

## What Gemini 2.5 Pro Can Do with Video

Gemini 2.5 Pro doesn't just look at individual frames — it understands video as a sequence of events with context, motion, and intent. In practice, this means it can:

Explain what's happening across the duration of a clip. Identify specific moments or timestamps where something occurs. Analyze movement patterns that span multiple frames. Incorporate audio context when a video includes sound.

Supported formats include MP4, MOV, AVI, and MKV. With the Files API, you can send videos up to 1 hour long. For inline base64, shorter clips under about 20MB work best.

The real capability difference from image analysis is temporal understanding. A single frame of someone throwing a baseball tells you one thing. A 15-second clip tells you about their wind-up, release mechanics, follow-through, and balance. That's what makes video analysis genuinely worth building on.

## Files API vs Inline Base64: Making the Right Call

There are two ways to get video into Gemini: upload it through the Files API first, then reference it by URI, or encode it as base64 and send it directly in the request body.

The decision tree is straightforward:

Use the **Files API** when the video is over 20MB, when you want to reuse the same file across multiple analysis requests, or when you want to defer the upload from the analysis timing for UX reasons.

Use **inline base64** when the video is under 20MB and you only need to analyze it once. It's simpler and skips the upload latency — which matters for short clips where users expect fast feedback.

For most Rork apps where users pick videos from their camera roll, you'll default to the Files API. Smartphone video, even a 30-second clip in 4K, often exceeds 20MB.

## Backend: Cloudflare Workers Video Analysis API

Calling Gemini directly from the Rork app would expose your API key. A Cloudflare Workers backend solves that while giving you a clean place to handle the async upload lifecycle.

```typescript
// workers/src/video-analysis.ts
// Cloudflare Workers endpoint for Gemini Video Understanding

import { GoogleGenAI } from '@google/genai';

interface Env {
  GEMINI_API_KEY: string;
}

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    if (request.method \!== 'POST') {
      return new Response('Method Not Allowed', { status: 405 });
    }

    const url = new URL(request.url);

    if (url.pathname === '/api/upload-video') {
      return handleVideoUpload(request, env);
    }

    if (url.pathname === '/api/analyze-video') {
      return handleVideoAnalysis(request, env);
    }

    return new Response('Not Found', { status: 404 });
  }
};

async function handleVideoUpload(request: Request, env: Env): Promise<Response> {
  try {
    const formData = await request.formData();
    const videoFile = formData.get('video') as File;

    if (\!videoFile) {
      return Response.json({ error: 'No video file provided' }, { status: 400 });
    }

    const uploadUrl = 'https://generativelanguage.googleapis.com/upload/v1beta/files';
    const arrayBuffer = await videoFile.arrayBuffer();

    // Initiate resumable upload
    const initResponse = await fetch(`${uploadUrl}?key=${env.GEMINI_API_KEY}`, {
      method: 'POST',
      headers: {
        'X-Goog-Upload-Protocol': 'resumable',
        'X-Goog-Upload-Command': 'start',
        'X-Goog-Upload-Header-Content-Length': arrayBuffer.byteLength.toString(),
        'X-Goog-Upload-Header-Content-Type': videoFile.type,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        file: { display_name: videoFile.name }
      })
    });

    if (\!initResponse.ok) {
      throw new Error(`Upload init failed: ${initResponse.status}`);
    }

    const uploadUri = initResponse.headers.get('X-Goog-Upload-URL');
    if (\!uploadUri) throw new Error('No upload URI received');

    // Upload the file body
    const uploadResponse = await fetch(uploadUri, {
      method: 'POST',
      headers: {
        'Content-Length': arrayBuffer.byteLength.toString(),
        'X-Goog-Upload-Offset': '0',
        'X-Goog-Upload-Command': 'upload, finalize',
      },
      body: arrayBuffer
    });

    if (\!uploadResponse.ok) {
      const errorText = await uploadResponse.text();
      throw new Error(`Upload failed: ${errorText}`);
    }

    const fileData = await uploadResponse.json() as {
      file: { uri: string; name: string; state: string }
    };

    return Response.json({
      fileUri: fileData.file.uri,
      fileName: fileData.file.name,
      state: fileData.file.state
    });

  } catch (error) {
    console.error('Upload error:', error);
    return Response.json(
      { error: error instanceof Error ? error.message : 'Upload failed' },
      { status: 500 }
    );
  }
}

async function handleVideoAnalysis(request: Request, env: Env): Promise<Response> {
  try {
    const { fileUri, prompt, mimeType } = await request.json() as {
      fileUri: string;
      prompt: string;
      mimeType: string;
    };

    if (\!fileUri || \!prompt) {
      return Response.json(
        { error: 'fileUri and prompt are required' },
        { status: 400 }
      );
    }

    const genAI = new GoogleGenAI({ apiKey: env.GEMINI_API_KEY });

    // Poll until file is ACTIVE — skipping this causes intermittent failures
    let fileState = 'PROCESSING';
    let attempts = 0;
    const maxAttempts = 10;

    while (fileState === 'PROCESSING' && attempts < maxAttempts) {
      await new Promise(resolve => setTimeout(resolve, 2000));

      const fileNameMatch = fileUri.match(/files\/([^/]+)/);
      if (\!fileNameMatch) break;

      const statusResponse = await fetch(
        `https://generativelanguage.googleapis.com/v1beta/files/${fileNameMatch[1]}?key=${env.GEMINI_API_KEY}`
      );
      const statusData = await statusResponse.json() as { state: string };
      fileState = statusData.state;
      attempts++;
    }

    if (fileState \!== 'ACTIVE') {
      return Response.json(
        { error: 'File is not ready for analysis' },
        { status: 422 }
      );
    }

    // Run the analysis with Gemini 2.5 Pro
    const result = await genAI.models.generateContent({
      model: 'gemini-2.5-pro-preview-0325',
      contents: [
        {
          parts: [
            {
              fileData: {
                fileUri: fileUri,
                mimeType: mimeType || 'video/mp4'
              }
            },
            { text: prompt }
          ]
        }
      ],
      config: {
        temperature: 0.2,
        maxOutputTokens: 2048,
      }
    });

    return Response.json({ analysis: result.text });

  } catch (error) {
    console.error('Analysis error:', error);
    return Response.json(
      { error: error instanceof Error ? error.message : 'Analysis failed' },
      { status: 500 }
    );
  }
}
```

The polling loop between upload and analysis is the single most important thing to get right. Send an analysis request before the file reaches `ACTIVE` state and you'll get an error — but because it's timing-dependent, it succeeds in testing and fails in production at unpredictable rates.

## Cost Optimization: Model Selection and Frame Sampling

Not every video needs Gemini 2.5 Pro. Knowing when to step down to Flash saves significant cost at scale.

```typescript
// workers/src/model-selector.ts
// Automatically select the right model based on analysis complexity

type AnalysisType = 'simple_summary' | 'detailed_analysis' | 'expert_coaching';

interface ModelConfig {
  model: string;
  maxOutputTokens: number;
  temperature: number;
}

function selectModel(analysisType: AnalysisType): ModelConfig {
  const configs: Record<AnalysisType, ModelConfig> = {
    // Quick summaries — Flash handles these well
    simple_summary: {
      model: 'gemini-2.5-flash-preview-04-17',
      maxOutputTokens: 512,
      temperature: 0.3,
    },
    // Multi-angle analysis — Pro is worth the cost here
    detailed_analysis: {
      model: 'gemini-2.5-pro-preview-0325',
      maxOutputTokens: 1024,
      temperature: 0.2,
    },
    // High-precision coaching — Pro with tight temperature
    expert_coaching: {
      model: 'gemini-2.5-pro-preview-0325',
      maxOutputTokens: 2048,
      temperature: 0.1,
    }
  };
  return configs[analysisType];
}

function getVideoStrategy(durationSeconds: number): {
  model: AnalysisType;
  shouldSample: boolean;
  samplingIntervalSec: number;
} {
  if (durationSeconds <= 30) {
    // Short clips: send whole thing, use Pro for best quality
    return { model: 'detailed_analysis', shouldSample: false, samplingIntervalSec: 0 };
  } else if (durationSeconds <= 180) {
    // Medium clips: Flash handles summaries fine
    return { model: 'simple_summary', shouldSample: false, samplingIntervalSec: 0 };
  } else {
    // Long clips: sample key frames, use Flash
    return { model: 'simple_summary', shouldSample: true, samplingIntervalSec: 10 };
  }
}

export { selectModel, getVideoStrategy };
```

For small videos that need quick turnaround, skip the Files API entirely and go inline:

```typescript
// workers/src/inline-video-analysis.ts
// Direct base64 path for small, one-time analysis

import { GoogleGenAI } from '@google/genai';

interface Env {
  GEMINI_API_KEY: string;
}

export async function analyzeVideoInline(
  videoBuffer: ArrayBuffer,
  mimeType: string,
  prompt: string,
  env: Env
): Promise<string> {
  const sizeInMB = videoBuffer.byteLength / (1024 * 1024);
  if (sizeInMB > 20) {
    throw new Error('Video too large for inline analysis. Use Files API instead.');
  }

  const genAI = new GoogleGenAI({ apiKey: env.GEMINI_API_KEY });

  // Convert ArrayBuffer to base64
  const uint8Array = new Uint8Array(videoBuffer);
  const base64Video = btoa(
    Array.from(uint8Array).map(b => String.fromCharCode(b)).join('')
  );

  const result = await genAI.models.generateContent({
    model: 'gemini-2.5-flash-preview-04-17',
    contents: [
      {
        parts: [
          { inlineData: { mimeType, data: base64Video } },
          { text: prompt }
        ]
      }
    ],
    config: { temperature: 0.2, maxOutputTokens: 1024 }
  });

  return result.text ?? '';
}
```

## Common Pitfalls That Will Burn You in Production

These are the issues that tend not to show up in happy-path testing but cause real failures with real users.

**Analyzing a file before it's ACTIVE**

This is the most common one. The upload returns immediately, but Gemini still needs time to process the file on its end. Send the analysis request too early and you'll get a "file not ready" error. The polling loop in the backend example is non-optional.

**Expecting uploaded files to persist**

Files uploaded through the Files API are deleted after 48 hours. If your app caches a `fileUri` and tries to reuse it later, it will work for a day or two and then start failing. Either re-upload when needed or cache the analysis result rather than the file URI.

**Video orientation issues on mobile**

Smartphones embed rotation metadata in EXIF, and Gemini usually interprets it correctly. When it doesn't, explicitly mentioning the orientation in your prompt — "This is a portrait-orientation video filmed on a smartphone" — resolves most cases.

**Token exhaustion on long videos**

By default, Gemini samples about one frame per second from a video, and each frame costs 258 tokens. A 5-minute video at 24fps, naively sent, would be enormous. Always sample or trim before sending anything longer than a couple of minutes.

## Advanced Pattern: Building an Analysis History System

Once the core analysis pipeline works, the next step for most apps is persisting results so users can track progress over time. Here's a Supabase-backed pattern that works well alongside the Cloudflare Workers backend.

First, define the schema. A simple table with `user_id`, `video_filename`, `analysis_type`, `result`, and `created_at` is enough to get started. For a sports coaching app, you'd also want `sport_type` and a numeric `score` field if you're building any kind of progress visualization.

```typescript
// supabase/schema.sql (run in Supabase SQL editor)
CREATE TABLE video_analyses (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE,
  video_filename text NOT NULL,
  analysis_type text NOT NULL CHECK (analysis_type IN ('sports_form', 'recipe_recognition', 'lecture_summary')),
  gemini_file_uri text,
  result text NOT NULL,
  processing_time_ms integer,
  model_used text NOT NULL,
  created_at timestamptz DEFAULT now()
);

-- Enable RLS so users only see their own analyses
ALTER TABLE video_analyses ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users can read own analyses"
  ON video_analyses FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "users can insert own analyses"
  ON video_analyses FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

On the Rork app side, saving and loading history looks like this:

```typescript
// lib/analysis-history.ts
// Supabase integration for persisting analysis results

import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.EXPO_PUBLIC_SUPABASE_URL\!,
  process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY\!
);

interface AnalysisRecord {
  id: string;
  video_filename: string;
  analysis_type: string;
  result: string;
  processing_time_ms: number;
  model_used: string;
  created_at: string;
}

export async function saveAnalysis(params: {
  videoFilename: string;
  analysisType: string;
  geminiFileUri: string;
  result: string;
  processingTimeMs: number;
  modelUsed: string;
}): Promise<{ id: string } | null> {
  const { data: { user } } = await supabase.auth.getUser();
  if (\!user) return null;

  const { data, error } = await supabase
    .from('video_analyses')
    .insert({
      user_id: user.id,
      video_filename: params.videoFilename,
      analysis_type: params.analysisType,
      gemini_file_uri: params.geminiFileUri,
      result: params.result,
      processing_time_ms: params.processingTimeMs,
      model_used: params.modelUsed,
    })
    .select('id')
    .single();

  if (error) {
    console.error('Failed to save analysis:', error.message);
    return null;
  }

  return data;
}

export async function loadAnalysisHistory(limit = 20): Promise<AnalysisRecord[]> {
  const { data, error } = await supabase
    .from('video_analyses')
    .select('id, video_filename, analysis_type, result, processing_time_ms, model_used, created_at')
    .order('created_at', { ascending: false })
    .limit(limit);

  if (error) {
    console.error('Failed to load history:', error.message);
    return [];
  }

  return data ?? [];
}

// Usage rate limiting: check how many analyses the user has done this month
export async function getMonthlyAnalysisCount(): Promise<number> {
  const startOfMonth = new Date();
  startOfMonth.setDate(1);
  startOfMonth.setHours(0, 0, 0, 0);

  const { count, error } = await supabase
    .from('video_analyses')
    .select('*', { count: 'exact', head: true })
    .gte('created_at', startOfMonth.toISOString());

  if (error) return 0;
  return count ?? 0;
}
```

The `getMonthlyAnalysisCount` function is the key piece for enforcing a free tier limit. Before running an analysis, check the count and gate accordingly. This is much simpler to implement correctly than trying to use Stripe metered billing for something this lightweight.

## Prompt Engineering for Video Analysis

The quality of Gemini's output depends heavily on how you structure the prompt. A few principles that consistently improve results:

**Be explicit about what you want Gemini to ignore**

When analyzing a running form video, you might not care about the background, other runners, or the weather. Telling Gemini "focus only on the runner in the foreground, ignore other people and background elements" gives you tighter, more actionable feedback.

**Specify the output format**

Gemini will give better-structured output when you tell it exactly how to format the response. "Format your response as a numbered list with headers" or "Use markdown with ## for each section" saves you from having to parse freeform text in your app.

**Include context about the user**

For a sports coaching app, prompts become significantly more useful when they include the user's skill level and specific goals. "This person is an intermediate-level tennis player working on their serve consistency" gives Gemini context to calibrate its feedback appropriately.

Here's a more complete prompt template that incorporates these principles:

```typescript
// lib/prompt-templates.ts
// Context-aware prompt generation for video analysis

interface UserContext {
  skillLevel: 'beginner' | 'intermediate' | 'advanced';
  sport: string;
  focusArea?: string;  // e.g., "serve mechanics", "footwork"
  injuryHistory?: string;
}

function buildSportsCoachingPrompt(context: UserContext): string {
  return `You are analyzing a ${context.sport} video for a ${context.skillLevel}-level athlete.

${context.focusArea ? `Their specific focus today is: ${context.focusArea}.` : ''}
${context.injuryHistory ? `Note: They have a history of ${context.injuryHistory}. Flag anything that could aggravate this.` : ''}

Focus ONLY on the primary athlete in frame. Ignore background, spectators, and environmental factors.

Provide your analysis in exactly this format:

## What You're Doing Well
(2-3 specific observations about effective technique)

## Primary Area to Improve
(The single most impactful change they can make, with clear explanation of why)

## Secondary Observations
(1-2 additional notes, lower priority)

## This Session's Focus Point
(One concrete thing to practice at the next session — specific enough to be actionable)

Keep the tone encouraging but honest. Be specific about body position, timing, and movement patterns.`;
}

export { buildSportsCoachingPrompt };
```

The difference between a generic prompt and a context-aware one like this is significant. In practice, users at the beginner level need encouraging, jargon-free feedback, while advanced athletes want precise technical language and comparative references. Building user profiles and routing to appropriate prompt templates is worth the effort.

## Rate Limiting and Abuse Prevention

One thing that's easy to overlook when building AI-powered apps: a motivated user (or a bot) can drain your API budget quickly if you don't add rate limiting on the server side.

The client-side monthly count is a UX convenience, not a security control. Users who know what they're doing can bypass it. Always enforce limits server-side too.

```typescript
// workers/src/rate-limiter.ts
// Cloudflare Workers KV-based rate limiting

interface Env {
  RATE_LIMIT_KV: KVNamespace;
  GEMINI_API_KEY: string;
}

interface RateLimitConfig {
  freeLimit: number;
  paidLimit: number;
  windowSeconds: number;  // 2592000 = 30 days
}

const DEFAULT_CONFIG: RateLimitConfig = {
  freeLimit: 3,
  paidLimit: 999,
  windowSeconds: 2592000,
};

export async function checkRateLimit(
  userId: string,
  isPaidUser: boolean,
  env: Env
): Promise<{ allowed: boolean; remaining: number; resetAt: number }> {
  const key = `ratelimit:${userId}:${Math.floor(Date.now() / (DEFAULT_CONFIG.windowSeconds * 1000))}`;
  const limit = isPaidUser ? DEFAULT_CONFIG.paidLimit : DEFAULT_CONFIG.freeLimit;

  const current = await env.RATE_LIMIT_KV.get(key);
  const count = current ? parseInt(current, 10) : 0;

  if (count >= limit) {
    const resetAt = (Math.floor(Date.now() / (DEFAULT_CONFIG.windowSeconds * 1000)) + 1)
      * DEFAULT_CONFIG.windowSeconds * 1000;
    return { allowed: false, remaining: 0, resetAt };
  }

  // Increment counter
  await env.RATE_LIMIT_KV.put(key, String(count + 1), {
    expirationTtl: DEFAULT_CONFIG.windowSeconds,
  });

  return {
    allowed: true,
    remaining: limit - count - 1,
    resetAt: 0,
  };
}
```

Using Cloudflare Workers KV for rate limiting fits naturally with the rest of this architecture and requires no additional infrastructure. The window key is based on a 30-day period, which aligns with monthly subscription billing.

## Thinking About What to Build

The three use cases in the code examples — sports coaching, recipe recognition, and lecture summarization — are starting points, not a complete list. Some other directions worth considering:

Physical therapy exercises (is the patient doing this correctly?), interview practice (analyzing body language and presentation), instrument practice (checking guitar finger positioning or piano technique), construction site inspection (flagging safety issues or progress documentation), and sign language translation are all areas where video-based AI feedback adds genuine value.

The common thread: there's a domain where humans with expertise currently charge high rates for short consultations, and video makes it possible to deliver a fraction of that value at scale and low cost. That's the business model worth building toward.

The technical foundation you now have — upload, process, analyze, persist results — scales to any of these directions. The product differentiation comes from the domain expertise you encode into your prompts and the UX you build around the core analysis loop.
