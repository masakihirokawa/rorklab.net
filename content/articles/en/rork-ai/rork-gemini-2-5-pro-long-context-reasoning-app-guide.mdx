## Introduction: Why Gemini 2.5 Pro?

Gemini 2.5 Pro, released in 2025, fundamentally changed the landscape of AI app development. With a context window of up to 1 million tokens (roughly 750,000 words) and a "Thinking Mode" that makes reasoning processes visible, it enables complex tasks that were impossible with previous models.

By integrating Gemini 2.5 Pro into your Rork app, you can build applications like:

- Legaltech tools that review entire contracts (100+ pages) in real time
- Developer tools that scan thousands of lines of code for refactoring opportunities
- Business intelligence apps that deeply analyze financial reports and research papers
- Compliance checkers that cross-reference multiple documents to surface inconsistencies

This guide covers everything you need to implement Gemini 2.5 Pro in Rork Max — from basic API setup through cost optimization design patterns. It's aimed at intermediate-to-advanced developers who already know Rork and TypeScript basics and are ready to tackle serious AI features.

## Gemini 2.5 Pro Key Features and Use Cases

### The Power of 1 Million Tokens

While GPT-4o supports 128,000 tokens and Claude 3.7 supports 200,000 tokens, Gemini 2.5 Pro handles up to **1,000,000 tokens** (about 750,000 words) in a single context. That's equivalent to ten average novels or around 15,000 lines of source code.

With that context size, the following tasks can be completed in a single API call:

- **Legal documents**: Feed an entire contract plus relevant case law plus internal guidelines to check for consistency
- **Code review**: Analyze a large repository's files in one batch to identify technical debt
- **Financial analysis**: Input multiple years of financial statements to extract anomalies and trends
- **Academic research**: Load dozens of papers simultaneously to generate a meta-analysis report

### Thinking Mode: Transparent Reasoning

Thinking Mode (enabled via `thinkingConfig`) outputs the model's "thinking process" before it reaches an answer. It excels at tasks requiring **multi-step logical reasoning** rather than simple information retrieval — mathematical proofs, legal reasoning, and business strategy formulation, for example.

For users, seeing "why the AI reached this conclusion" builds trust. For enterprise applications, this transparency is often the deciding factor in adoption.

### Flash vs. Pro: Choosing the Right Model

Gemini 2.5 Pro is powerful, but you don't need it for every query. For cost efficiency, the ideal architecture uses Gemini 2.5 Flash for simple tasks and routes only complex tasks to Pro.

Use Gemini 2.5 Flash for short Q&A, single-document summaries under 50 pages, and everyday chatbot conversations. Use Gemini 2.5 Pro for cross-document analysis, tasks requiring legal or medical judgment, and code reviews covering more than 10,000 lines.

### Implementing the Model Router

Here's a router that automatically switches between Flash and Pro based on task complexity:

```typescript
// src/lib/gemini-router.ts
import { geminiFlash, geminiPro } from "./gemini";

interface AnalysisTask {
  content: string;
  taskType: "simple" | "complex" | "legal" | "financial" | "code";
  requiresThinking?: boolean;
}

interface AnalysisResult {
  response: string;
  modelUsed: "flash" | "pro";
  tokenCount?: number;
}

/**
 * Auto-selects Flash or Pro based on task complexity.
 * Expected output: { response: "analysis text", modelUsed: "pro", tokenCount: 15000 }
 */
export async function analyzeWithRouter(task: AnalysisTask): Promise<AnalysisResult> {
  const shouldUsePro =
    task.taskType === "legal" ||
    task.taskType === "financial" ||
    task.taskType === "code" ||
    task.content.length > 50000 || // Use Pro for content over 50,000 characters
    task.requiresThinking === true;

  const model = shouldUsePro ? geminiPro : geminiFlash;
  const modelUsed = shouldUsePro ? "pro" : "flash";

  const result = await model.generateContent(task.content);
  const response = result.response.text();

  return {
    response,
    modelUsed,
    tokenCount: result.response.usageMetadata?.totalTokenCount,
  };
}
```

## Building a Long-Document Analysis App

### Document Upload and Processing Pipeline

Here's an implementation that accepts PDFs or long text and generates detailed analysis reports:

```typescript
// src/services/document-analyzer.ts
import { geminiPro } from "../lib/gemini";

interface DocumentAnalysisRequest {
  documentText: string;
  analysisType: "summary" | "legal_review" | "financial_audit" | "risk_assessment";
  userQuestion?: string;
}

interface DocumentAnalysisResult {
  summary: string;
  keyFindings: string[];
  risks: string[];
  recommendations: string[];
  rawResponse: string;
}

const ANALYSIS_PROMPTS = {
  summary: `Analyze the following document thoroughly and output in JSON format:
    - summary: 3-5 sentence overview
    - keyFindings: Important discoveries (array)
    - risks: Risks or issues (array)
    - recommendations: Recommended actions (array)`,

  legal_review: `As a legal expert, carefully review the following contract or legal document and output in JSON format:
    - summary: Document overview and purpose
    - keyFindings: Key clauses, rights, and obligations (array)
    - risks: Legal risks or unfavorable terms (array)
    - recommendations: Points to negotiate or revise (array)`,

  financial_audit: `As a financial analyst, analyze the following financial data and output in JSON format:
    - summary: Overall financial health assessment
    - keyFindings: Key financial metrics and trends (array)
    - risks: Financial risks and concerns (array)
    - recommendations: Improvement actions and investment considerations (array)`,

  risk_assessment: `As a risk management specialist, identify risks in the following document and output in JSON format:
    - summary: Risk assessment overview
    - keyFindings: Identified risk factors (array)
    - risks: Risks categorized as high/medium/low (array)
    - recommendations: Risk mitigation strategies (array)`,
};

/**
 * Analyzes a long document using Gemini 2.5 Pro.
 * Expected output: { summary: "...", keyFindings: [...], risks: [...], recommendations: [...] }
 */
export async function analyzeDocument(
  request: DocumentAnalysisRequest
): Promise<DocumentAnalysisResult> {
  const systemPrompt = ANALYSIS_PROMPTS[request.analysisType];

  const userPrompt = request.userQuestion
    ? `${systemPrompt}\n\nPay special attention to: ${request.userQuestion}\n\n---\n\n${request.documentText}`
    : `${systemPrompt}\n\n---\n\n${request.documentText}`;

  const result = await geminiPro.generateContent({
    contents: [{ role: "user", parts: [{ text: userPrompt }] }],
    generationConfig: {
      responseMimeType: "application/json",
    },
  });

  const rawResponse = result.response.text();

  let parsed: DocumentAnalysisResult;
  try {
    parsed = JSON.parse(rawResponse);
  } catch {
    parsed = {
      summary: rawResponse,
      keyFindings: [],
      risks: [],
      recommendations: [],
      rawResponse,
    };
  }

  return { ...parsed, rawResponse };
}
```

### Streaming Implementation for Better UX

Document analysis can take seconds to dozens of seconds. Using the streaming API lets you display text to users as it's generated:

```typescript
// src/services/streaming-analyzer.ts
import { geminiPro } from "../lib/gemini";

interface StreamingCallbacks {
  onChunk: (text: string) => void;
  onComplete: (fullText: string) => void;
  onError: (error: Error) => void;
}

export async function analyzeDocumentStream(
  prompt: string,
  callbacks: StreamingCallbacks
): Promise<void> {
  try {
    const result = await geminiPro.generateContentStream(prompt);

    let fullText = "";

    for await (const chunk of result.stream) {
      const chunkText = chunk.text();
      fullText += chunkText;
      callbacks.onChunk(chunkText);
    }

    callbacks.onComplete(fullText);
  } catch (error) {
    callbacks.onError(error as Error);
  }
}
```

## Implementing Thinking Mode: Making Reasoning Visible

### Calling the API with Thinking Mode Enabled

Thinking Mode outputs the model's internal reasoning process before it delivers an answer. It's especially useful for legal judgments or complex analyses where showing "why the AI decided this" is essential.

```typescript
// src/services/thinking-analyzer.ts
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GOOGLE_AI_API_KEY!);

interface ThinkingAnalysisResult {
  thinkingProcess: string;
  finalAnswer: string;
}

/**
 * Analyzes a complex question using Thinking Mode.
 * Expected output: { thinkingProcess: "First, verifying assumptions...", finalAnswer: "In conclusion..." }
 */
export async function analyzeWithThinking(
  question: string,
  context?: string
): Promise<ThinkingAnalysisResult> {
  const model = genAI.getGenerativeModel({
    model: "gemini-2.5-pro",
  });

  const fullPrompt = context
    ? `Background information:\n${context}\n\nQuestion:\n${question}`
    : question;

  const result = await model.generateContent({
    contents: [{ role: "user", parts: [{ text: fullPrompt }] }],
    generationConfig: {
      temperature: 1, // Recommended setting for Thinking Mode
    },
  });

  const text = result.response.text();

  // Split thinking process from final answer
  const thinkingMatch = text.match(/<think>([\s\S]*?)<\/think>/);
  const thinkingProcess = thinkingMatch ? thinkingMatch[1].trim() : "";
  const finalAnswer = thinkingProcess
    ? text.replace(/<think>[\s\S]*?<\/think>/, "").trim()
    : text;

  return { thinkingProcess, finalAnswer };
}
```

## Cost Optimization and Production Best Practices

### Monitoring Token Usage and Calculating Costs

Gemini 2.5 Pro is powerful, but costs add up when processing large volumes of tokens. Production environments must always monitor usage.

```typescript
// src/lib/cost-tracker.ts

// Reference prices as of April 2026 (USD) — verify current pricing in Google AI Studio
const COST_PER_TOKEN = {
  "gemini-2.5-pro": {
    input: 0.000003,   // ~$3 per million tokens (reference value)
    output: 0.000015,  // ~$15 per million tokens (reference value)
  },
  "gemini-2.5-flash": {
    input: 0.0000003,  // ~$0.30 per million tokens (reference value)
    output: 0.0000015, // ~$1.50 per million tokens (reference value)
  },
};

interface UsageMetrics {
  model: "gemini-2.5-pro" | "gemini-2.5-flash";
  inputTokens: number;
  outputTokens: number;
}

export function calculateCost(usage: UsageMetrics): {
  inputCost: number;
  outputCost: number;
  totalCost: number;
} {
  const rates = COST_PER_TOKEN[usage.model];
  const inputCost = usage.inputTokens * rates.input;
  const outputCost = usage.outputTokens * rates.output;
  return { inputCost, outputCost, totalCost: inputCost + outputCost };
}

export function estimateTokenCount(text: string): number {
  // Rough estimate: ~4 chars/token for English, ~1.5 chars/token for Japanese
  const japaneseCharCount = (text.match(/[\u3000-\u9FFF]/g) || []).length;
  const otherCharCount = text.length - japaneseCharCount;
  return Math.ceil(japaneseCharCount / 1.5 + otherCharCount / 4);
}
```

### Rate Limiting and Retry Strategy

```typescript
// src/lib/retry-handler.ts

interface RetryConfig {
  maxRetries: number;
  initialDelayMs: number;
  backoffMultiplier: number;
}

const DEFAULT_RETRY_CONFIG: RetryConfig = {
  maxRetries: 3,
  initialDelayMs: 1000,
  backoffMultiplier: 2,
};

/**
 * Retries an API call with exponential backoff.
 * Expected output: normal API response, or throws error after max retries
 */
export async function withRetry<T>(
  apiCall: () => Promise<T>,
  config: RetryConfig = DEFAULT_RETRY_CONFIG
): Promise<T> {
  let lastError: Error;
  let delay = config.initialDelayMs;

  for (let attempt = 0; attempt <= config.maxRetries; attempt++) {
    try {
      return await apiCall();
    } catch (error) {
      lastError = error as Error;

      if (error instanceof Error && error.message.includes("429")) {
        console.log(`Rate limited: retrying in ${delay}ms (${attempt + 1}/${config.maxRetries})`);
        await new Promise((resolve) => setTimeout(resolve, delay));
        delay *= config.backoffMultiplier;
      } else {
        throw error;
      }
    }
  }

  throw lastError!;
}
```

### Caching Strategy to Reduce Costs

Caching responses for repeated accesses to the same document can drastically cut API costs:

```typescript
// src/lib/response-cache.ts
import AsyncStorage from "@react-native-async-storage/async-storage";

const CACHE_TTL_MS = 24 * 60 * 60 * 1000; // 24 hours

function generateCacheKey(prompt: string): string {
  // Use a simple hash as the cache key
  let hash = 0;
  for (let i = 0; i < Math.min(prompt.length, 200); i++) {
    hash = ((hash << 5) - hash) + prompt.charCodeAt(i);
    hash |= 0;
  }
  return `gemini_cache_${Math.abs(hash).toString(16)}`;
}

export async function getCachedResponse(prompt: string): Promise<string | null> {
  const key = generateCacheKey(prompt);
  try {
    const cached = await AsyncStorage.getItem(key);
    if (!cached) return null;
    const { response, timestamp } = JSON.parse(cached);
    if (Date.now() - timestamp > CACHE_TTL_MS) {
      await AsyncStorage.removeItem(key);
      return null;
    }
    return response;
  } catch {
    return null;
  }
}

export async function setCachedResponse(prompt: string, response: string): Promise<void> {
  const key = generateCacheKey(prompt);
  await AsyncStorage.setItem(
    key,
    JSON.stringify({ response, timestamp: Date.now() })
  );
}
```

## Common Errors and How to Fix Them

**Context length exceeded error**: If you exceed the 1M token limit, you'll receive a `CONTEXT_LENGTH_EXCEEDED` error. Split the document into chunks and make multiple API calls, or adopt a two-stage approach: generate a summary first, then feed the summary into a detailed analysis.

**Rate limiting error (429)**: This occurs when you exceed the API call frequency limit. Implement exponential backoff as shown in the `withRetry` implementation above, and display an appropriate "analyzing" state to the user.

**JSON parse error**: Even with `responseMimeType: "application/json"` specified, complex structures can occasionally fail to parse. Always include `try/catch` handling and fall back to displaying the raw string response as-is.

**Timeout**: Processing large documents can take 30 seconds or more. Set your client-side timeout to 60–120 seconds, and use the streaming API to display progress during processing to improve UX.

## FAQ

**Q. How should I choose between Gemini 2.5 Pro and Flash?**

A. Use Flash for simple Q&A, short document summaries (under 50 pages), and everyday chatbot conversations. Use Pro for professional judgment requiring legal, medical, or financial expertise; documents exceeding 10,000 characters; and complex reasoning tasks that benefit from Thinking Mode. The `analyzeWithRouter` implementation introduced in this article lets you cut costs significantly by applying this split automatically.

**Q. Would I ever realistically use the full 1 million token context?**

A. In typical app development, a standard contract (20–30 pages) runs about 15,000–25,000 tokens, and a codebase of ~10,000 lines runs about 40,000–60,000 tokens. Using the full 1M tokens is limited to specialized cases like academic research or large-scale legaltech, but the 50,000–200,000 token range is where you'll see a meaningful advantage in everyday use.

**Q. Does Thinking Mode need to be enabled all the time?**

A. No. Thinking Mode slows down response generation and increases token consumption. Reserve it for tasks where transparency matters most — legal judgments, medical advice, complex mathematical problems. For general Q&A and information retrieval, plain Flash is faster and cheaper.
