## Introduction

Tracking personal expenses shouldn't feel like a chore. What if you could simply snap a photo of your receipt and have an app automatically extract the amount, identify the store, and categorize the expense — all without typing a single number?

In this tutorial, we'll build an AI-powered receipt scanner budget app using Rork. You'll learn how to implement camera capture, extract text from receipt images using OCR (Optical Character Recognition), leverage the Gemini API for automatic expense categorization, and visualize monthly spending reports with charts.

**What you'll learn:**

- How to implement camera capture and gallery selection in a Rork app
- Using Cloud Vision API for OCR text extraction from receipts
- Leveraging Gemini API for automatic expense categorization
- Building monthly spending reports with chart visualizations

**Who this is for:** Intermediate Rork developers who understand the basics and want to build an app with API integrations.

## App Architecture Overview

Let's start by mapping out the app's structure. The app consists of four main screens:

```
ReceiptBudgetApp/
├── HomeScreen          # Monthly summary + recent entries
├── CaptureScreen       # Camera capture / gallery picker
├── ResultScreen        # OCR results review + category editing
└── ReportScreen        # Monthly report (category charts)
```

Here's the data flow:

1. User captures a receipt photo or selects one from the gallery
2. The image is sent to Cloud Vision API for text extraction
3. Extracted text is sent to Gemini API for structured data parsing (amount, store, category)
4. Results are saved to local storage
5. Monthly reports visualize spending by category

## Implementing Camera Capture and Gallery Selection

Rork leverages `expo-camera` and `expo-image-picker` for camera functionality. Here's the core implementation for the CaptureScreen:

```typescript
// CaptureScreen.tsx — Receipt capture screen
import * as ImagePicker from 'expo-image-picker';
import { CameraView, useCameraPermissions } from 'expo-camera';

export default function CaptureScreen({ navigation }) {
  const [permission, requestPermission] = useCameraPermissions();
  const cameraRef = useRef(null);

  // Capture with camera
  const handleCapture = async () => {
    if (cameraRef.current) {
      const photo = await cameraRef.current.takePictureAsync({
        base64: true,  // Get base64 for OCR API submission
        quality: 0.8,  // Balance between file size and quality
      });
      navigation.navigate('Result', { imageBase64: photo.base64 });
    }
  };

  // Pick from gallery
  const handlePickImage = async () => {
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      base64: true,
      quality: 0.8,
    });

    if (!result.canceled && result.assets[0].base64) {
      navigation.navigate('Result', {
        imageBase64: result.assets[0].base64,
      });
    }
  };

  return (
    // Camera preview + capture button + gallery button UI
    // ...
  );
}
```

The key detail here is the `base64: true` option. Since we need to send the image data directly to the OCR API in the next step, having it pre-encoded as Base64 saves an extra conversion step.

## OCR Text Extraction with Cloud Vision API

To read text from the captured receipt image, we'll use Google Cloud Vision API's `TEXT_DETECTION` feature:

```typescript
// services/ocrService.ts — OCR text extraction service
const VISION_API_URL =
  'https://vision.googleapis.com/v1/images:annotate';

export async function extractTextFromReceipt(
  imageBase64: string
): Promise<string> {
  const requestBody = {
    requests: [
      {
        image: { content: imageBase64 },
        features: [{ type: 'TEXT_DETECTION', maxResults: 1 }],
      },
    ],
  };

  const response = await fetch(
    `${VISION_API_URL}?key=YOUR_VISION_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(requestBody),
    }
  );

  const data = await response.json();
  // fullTextAnnotation contains the concatenated text
  const fullText =
    data.responses?.[0]?.fullTextAnnotation?.text ?? '';

  return fullText;
}
// Expected output example:
// "Whole Foods Market\n03/27/2026\nOrganic Milk $4.99\nBread $3.49\nTotal $8.48"
```

Cloud Vision API handles multiple languages well, including mixed-language receipts. The `fullTextAnnotation.text` field contains the complete receipt text with line breaks, which we can pass directly to the AI in the next step.

## Automatic Categorization with Gemini API

Now we'll send the OCR text to Gemini API to extract structured data — store name, date, amount, and spending category. Prompt engineering is the key to making this work reliably.

```typescript
// services/categorizationService.ts — AI categorization
const GEMINI_API_URL =
  'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';

interface ReceiptData {
  storeName: string;
  date: string;
  totalAmount: number;
  category: string;
  items: { name: string; price: number }[];
}

export async function categorizeReceipt(
  ocrText: string
): Promise<ReceiptData> {
  const prompt = `
Analyze the following OCR text from a receipt and extract the information in JSON format.

【Fields to extract】
- storeName: Store or merchant name
- date: Date in YYYY-MM-DD format
- totalAmount: Total amount as a number (no currency symbol)
- category: Expense category (choose exactly one):
  Groceries, Household, Transportation, Dining Out, Healthcare, Clothing, Entertainment, Education, Utilities, Other
- items: List of items with name and price

【OCR Text】
${ocrText}

【Output format】
Return only valid JSON. No explanations or markdown.
`;

  const response = await fetch(
    `${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        contents: [{ parts: [{ text: prompt }] }],
        generationConfig: {
          responseMimeType: 'application/json',
        },
      }),
    }
  );

  const data = await response.json();
  const jsonText =
    data.candidates?.[0]?.content?.parts?.[0]?.text ?? '{}';

  return JSON.parse(jsonText) as ReceiptData;
}
// Expected output:
// {
//   "storeName": "Whole Foods Market",
//   "date": "2026-03-27",
//   "totalAmount": 8.48,
//   "category": "Groceries",
//   "items": [
//     { "name": "Organic Milk", "price": 4.99 },
//     { "name": "Bread", "price": 3.49 }
//   ]
// }
```

Setting `responseMimeType: 'application/json'` tells Gemini to return only JSON, which makes response parsing much more reliable.

### Tips to Improve Categorization Accuracy

Here are some strategies to boost the AI's classification performance:

- **Be specific with category definitions**: Instead of just "Groceries," add context like "Food items purchased at supermarkets and convenience stores"
- **Include few-shot examples**: Pass previous classification results for the same store as examples in the prompt
- **Learn from user corrections**: When users manually change a category, store that feedback to refine future classifications

## Result Screen with Manual Editing

After OCR and AI classification, we display the results and let users make corrections if needed:

```typescript
// ResultScreen.tsx — Review and edit OCR results
export default function ResultScreen({ route, navigation }) {
  const { imageBase64 } = route.params;
  const [receiptData, setReceiptData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    processReceipt();
  }, []);

  const processReceipt = async () => {
    try {
      // Step 1: Extract text via OCR
      const ocrText = await extractTextFromReceipt(imageBase64);
      // Step 2: Categorize with AI
      const categorized = await categorizeReceipt(ocrText);
      setReceiptData(categorized);
    } catch (error) {
      Alert.alert('Error', 'Failed to process the receipt');
    } finally {
      setLoading(false);
    }
  };

  const handleSave = async () => {
    // Save receipt data to AsyncStorage
    const existing = await AsyncStorage.getItem('receipts');
    const receipts = existing ? JSON.parse(existing) : [];
    receipts.push({
      ...receiptData,
      id: Date.now().toString(),
      createdAt: new Date().toISOString(),
    });
    await AsyncStorage.setItem('receipts', JSON.stringify(receipts));
    navigation.navigate('Home');
  };

  // Category dropdown for manual changes
  // Manual amount correction support
  // ...
}
```

When users correct the AI's categorization, storing those corrections creates a valuable feedback loop that you can use to improve prompts over time.

## Monthly Report Visualization

With accumulated receipt data, we can generate monthly spending reports. Let's display category breakdowns as a pie chart:

```typescript
// ReportScreen.tsx — Monthly spending report
import { PieChart } from 'react-native-chart-kit';

export default function ReportScreen() {
  const [monthlyData, setMonthlyData] = useState([]);

  useEffect(() => {
    loadMonthlyReport();
  }, []);

  const loadMonthlyReport = async () => {
    const stored = await AsyncStorage.getItem('receipts');
    const receipts = stored ? JSON.parse(stored) : [];

    // Filter and aggregate current month's data by category
    const now = new Date();
    const thisMonth = receipts.filter((r) => {
      const d = new Date(r.date);
      return (
        d.getMonth() === now.getMonth() &&
        d.getFullYear() === now.getFullYear()
      );
    });

    const categoryTotals = {};
    thisMonth.forEach((r) => {
      const cat = r.category || 'Other';
      categoryTotals[cat] =
        (categoryTotals[cat] || 0) + r.totalAmount;
    });

    // Transform for PieChart format
    const COLORS = [
      '#FF6384', '#36A2EB', '#FFCE56', '#4BC0C0',
      '#9966FF', '#FF9F40', '#C9CBCF', '#7BC67E',
    ];

    const chartData = Object.entries(categoryTotals).map(
      ([name, amount], index) => ({
        name,
        amount,
        color: COLORS[index % COLORS.length],
        legendFontColor: '#333',
        legendFontSize: 14,
      })
    );

    setMonthlyData(chartData);
  };

  return (
    <View style={{ padding: 16 }}>
      <Text style={{ fontSize: 20, fontWeight: 'bold' }}>
        Monthly Spending Report
      </Text>
      {monthlyData.length > 0 && (
        <PieChart
          data={monthlyData}
          width={350}
          height={220}
          chartConfig={{
            color: (opacity = 1) =>
              `rgba(0, 0, 0, ${opacity})`,
          }}
          accessor="amount"
          backgroundColor="transparent"
          paddingLeft="15"
        />
      )}
    </View>
  );
}
```

## Common Errors and Fixes

### Camera Permission Denied

When the user denies camera permission through `expo-camera`'s `useCameraPermissions`, add a fallback that guides them to the device settings. You can open the app's settings page directly with `Linking.openSettings()`.

### Low OCR Accuracy

Receipt scanning accuracy depends heavily on lighting and image quality. Consider adding a capture guide overlay that encourages users to photograph receipts on flat surfaces in well-lit areas without shadows. For preprocessing, applying contrast adjustment and grayscale conversion before sending to the API can significantly improve results.

### Gemini API Timeouts

Long receipts or high API load can cause timeouts. Implement timeout handling with `AbortController` and add retry logic with exponential backoff. Setting a reasonable timeout of 15–20 seconds covers most cases.

## FAQ

**Q: How much does Cloud Vision API cost?**

A: Google Cloud Vision API offers 1,000 free requests per month. For personal budget tracking, this is more than enough unless you're scanning over 30 receipts per day. Beyond the free tier, pricing is $1.50 per 1,000 requests.

**Q: Does the app work offline?**

A: Photo capture and viewing saved data work offline. However, OCR text extraction and AI categorization require an internet connection. For a better offline experience, implement a queue that saves images locally and processes them in batch once connectivity is restored.

**Q: Can this handle receipts in different languages?**

A: Yes. Cloud Vision API supports over 100 languages, and Gemini API can process multilingual text. Just adjust the category names and prompt language to match your target audience.

**Q: Where is the data stored?**

A: In this tutorial, all data is saved in AsyncStorage (device-local storage). If you need cloud sync or multi-device access, consider integrating a backend like Supabase.
