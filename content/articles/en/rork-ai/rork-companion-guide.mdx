## What Is Rork Companion?

**Rork Companion** is the AI pair programmer built directly into the Rork platform. It goes beyond generating code on demand — it lets you have a genuine **back-and-forth conversation** about your app's features, bugs, and improvements, and implements changes in real time as you discuss them.

Traditional AI coding tools work in one direction: you write a prompt, you wait for output. Rork Companion turns that into a dialogue — more like pairing with a knowledgeable developer than submitting a ticket to an automated system.

---

## What Rork Companion Can Do

### 1. Conversational App Improvement

Describe what you want in plain English: "Make this button stand out more" or "Redirect the user to the dashboard after login." Companion locates the relevant code and implements the change.

### 2. Bug Investigation and Debugging

Paste an error message and ask "Why is this happening?" Companion reads the error, traces the likely cause through your code, and proposes a fix. Stack traces that feel opaque to newer developers get translated into plain explanations.

### 3. Feature Addition and Extension

Tell Companion what you want to add — push notifications, dark mode support, offline data storage with SQLite — and it walks you through the implementation options, then builds whichever approach you choose.

### 4. Code Explanation

Ask "What does this code do?" and Companion explains the component or logic in plain language. Especially useful when inheriting a project or trying to understand auto-generated code.

## How to Give Effective Instructions

The quality of Companion's output correlates directly with the quality of your instructions.

### Strong instruction examples

```
"I want to show an onboarding screen on first launch only.
Save a 'seen' flag in AsyncStorage so subsequent launches skip it."

"The card list on the home screen won't scroll.
I'm using FlatList but the parent View might be missing flex:1 — check and fix."

"Show a loading spinner while fetching from the API.
On success, display the data. On error, show a retry button."
```

### Weak instructions and how to fix them

- "Make the app better" → Too vague. Try: "Improve the loading time of the home screen list"
- "There's an error" → Missing context. Try: paste the full error message and say "what's causing this and how do I fix it?"
- "Make it look cooler" → No standard to work from. Try: "Align the design with Material Design 3 and add a ripple effect to buttons"

### Three principles for better instructions

**State the goal**: Begin with what you want to accomplish — "I want users to be able to do X more easily" is a useful frame.

**Describe the current state**: "Right now it does X → I want it to do Y instead" helps Companion understand the delta rather than guessing at intent.

**Declare constraints**: "Without adding new libraries," "without breaking the existing styles," or "keep it to React Native only" prevent Companion from introducing changes you don't want.

## Common Usage Patterns

### Pattern 1: Adding a New Screen

```
"Add a settings screen with the following user-configurable options:
- Notification toggle (on/off)
- Theme (light/dark)
- Font size (small/medium/large)

Persist all settings in AsyncStorage so they survive app restarts."
```

### Pattern 2: API Integration

```
"Fetch a list of posts from JSONPlaceholder's /posts endpoint and display them.

Requirements:
- Use fetch (no external HTTP library)
- Show an ActivityIndicator while loading
- Show an alert on error
- Display results in a FlatList with title and body preview per item

Add this inside the existing home screen, not as a new screen."
```

### Pattern 3: Performance Optimization

```
"The list scrolls sluggishly. Please investigate and fix:
1. Verify keyExtractor is set correctly on the FlatList
2. Apply memo and useCallback to prevent unnecessary re-renders
3. Add getItemLayout if items have fixed height

After making changes, explain what each fix does and why it helps."
```

## Important Notes When Using Rork Companion

### Preview before applying

Companion shows you its proposed changes before applying them. Review multi-file changes especially carefully — it's easy to miss second-order effects.

### Work incrementally

Break large feature requests into smaller steps. Smaller changes are easier to review, easier to roll back, and more likely to match your intent on the first try.

### Take snapshots before big changes

Before adding a major feature, create a manual snapshot. It gives you a clean rollback point if something goes sideways.

## Frequently Asked Questions

**Q. Is Rork Companion free to use?**

Rork Companion is included in Rork's subscription plans, with usage limits varying by tier. Check Rork's current pricing page for specifics. Rork Max provides the most Companion usage with the highest limits.

**Q. Can I give instructions in Japanese?**

Yes — Rork Companion handles Japanese instructions. That said, technical terms (component names, library names, method signatures) are often more reliably recognized in English. Mixing natural language context in Japanese with English technical terms tends to work well.

**Q. What frameworks and languages does Companion support?**

Rork is built around React Native development, so Companion's expertise is strongest with React Native, JavaScript, and TypeScript. It also handles common ecosystem libraries well — React Navigation, Expo, Reanimated, and similar.

**Q. What if Companion generates incorrect code?**

Before applying any change, ask "Why is this implementation correct?" to check Companion's reasoning. If you spot an issue, point it out specifically — "shouldn't this use useEffect instead?" gets a better response than "this is wrong." To revert, just tell Companion "undo that change" or "go back to the previous version."
