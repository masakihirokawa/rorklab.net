While Rork uses Gemini 2.0 Flash as its default AI engine for generating apps, you can also **embed external AI services directly into the apps you build**. OpenAI's **GPT-4o** stands out as a particularly compelling choice — it handles text generation, image understanding, and audio processing within a single multimodal model, making it a natural fit for mobile applications.

This guide walks you through integrating GPT-4o into a Rork-built app, from obtaining your API key to implementing real-world features.

## What Is GPT-4o?

GPT-4o (the "o" stands for "omni") is OpenAI's flagship model. Its defining characteristic is the ability to process text, images, and audio through a single unified architecture, with significantly faster response times compared to its predecessor, GPT-4 Turbo.

By integrating GPT-4o into a Rork app, you can build features like these:

**Text-based AI features**: Chatbots, text summarization, translation, and content generation. Users ask questions in natural language, and GPT-4o responds instantly.

**Image recognition**: Users snap a photo with their camera, send it to GPT-4o, and receive an intelligent analysis. Think recipe suggestions from food photos, or product information extraction from packaging.

**Structured data output**: With JSON mode, GPT-4o returns responses as structured data that you can bind directly to your app's UI, dramatically simplifying your parsing logic.

## Getting and Managing Your OpenAI API Key

To use GPT-4o, you'll need an OpenAI API key.

Head to OpenAI's platform (platform.openai.com), create an account, and generate a new key from the "API Keys" section of the dashboard.

Key management is critical. **Never hardcode your API key into the app's source code.** In a React Native app, the JavaScript bundle is unpacked on the device, making any embedded keys relatively easy to extract.

The recommended approach is to route API calls through a **backend proxy**. Set up a Supabase Edge Function or Cloudflare Worker as an intermediary, store your API key in server-side environment variables, and have your app send requests to the proxy. The proxy calls OpenAI and forwards the response back. This keeps your key secure and gives you a single point of control for rate limiting and usage monitoring.

## The Core API Call Structure

The heart of any GPT-4o integration is the API call. Here's the typical structure Rork generates:

```typescript
const callGPT4o = async (messages: Message[]) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: messages.map(m => ({
        role: m.role,
        content: m.content,
      })),
      temperature: 0.7,
      max_tokens: 1024,
    }),
  });

  const data = await response.json();
  return data.choices[0].message.content;
};
```

The `temperature` parameter controls output randomness. Use around 0.7 for conversational chatbots, or 0.2–0.3 when you need factual, deterministic answers. `max_tokens` caps the response length — on mobile, overly long responses can hurt the user experience, so tune this to match your use case.

## Implementing Image Recognition

GPT-4o's multimodal capabilities let you send images captured by the device camera for AI-powered analysis.

Prompt Rork like this:

```
Build a food photo analyzer app:
- Show camera preview using expo-camera
- Add a capture button to take photos
- Base64-encode the captured image and send it to GPT-4o
- Use GPT-4o's vision capability to analyze the image
- Display results as cards (dish name, estimated calories, key nutrients)
```

When sending images to GPT-4o, you include the image data in the `content` field using a multipart message format:

```typescript
const analyzeImage = async (base64Image: string) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: [
        {
          role: 'user',
          content: [
            {
              type: 'text',
              text: 'Identify this dish and provide: dish name, estimated calories, and key nutrients. Respond in JSON format.',
            },
            {
              type: 'image_url',
              image_url: {
                url: `data:image/jpeg;base64,${base64Image}`,
                detail: 'low',
              },
            },
          ],
        },
      ],
      response_format: { type: 'json_object' },
    }),
  });

  const data = await response.json();
  return JSON.parse(data.choices[0].message.content);
};
```

Setting `detail` to `low` reduces image resolution and keeps token consumption down. On mobile, where bandwidth can be limited, switching between `low` and `high` depending on the use case is a smart optimization.

## Implementing Streaming Responses

Leaving users staring at a blank screen while waiting for a response is a poor experience. GPT-4o supports streaming, allowing you to display generated text in real time as it arrives.

```typescript
const streamGPT4o = async (
  messages: Message[],
  onChunk: (text: string) => void
) => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o',
      messages: messages.map(m => ({ role: m.role, content: m.content })),
      stream: true,
    }),
  });

  const reader = response.body?.getReader();
  const decoder = new TextDecoder();

  while (reader) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value);
    const lines = chunk.split('\n').filter(line => line.startsWith('data: '));

    for (const line of lines) {
      const json = line.replace('data: ', '');
      if (json === '[DONE]') return;

      const parsed = JSON.parse(json);
      const content = parsed.choices[0]?.delta?.content;
      if (content) onChunk(content);
    }
  }
};
```

When prompting Rork for streaming support, phrases like "display the response in real time, character by character" or "implement a typing animation like ChatGPT" will guide it toward generating the right code.

## Cost Management Best Practices

GPT-4o is billed on a per-token basis. With a large user base, costs can escalate quickly if left unchecked.

**Request limits**: Cap the number of requests per user per day. Track counts using AsyncStorage or Supabase, and show a friendly message when the limit is reached.

**Conversation history compression**: Sending the entire chat history with every request inflates token usage. Limit context to the most recent 10 messages, or use a summarization step to condense older conversations.

**Model routing**: Not every request needs GPT-4o. Use GPT-4o-mini for simple queries and reserve GPT-4o for complex analysis tasks. Implementing dynamic model selection can cut costs dramatically.

**Response caching**: Cache responses for identical queries locally to avoid redundant API calls.

## GPT-4o vs. Gemini: Understanding the Roles

Rork's default AI engine, Gemini 2.0 Flash, and the GPT-4o you embed in your app serve fundamentally different purposes.

Gemini 2.0 Flash powers **the app generation stage** — it's the AI that turns your prompts into code and designs the app's architecture.

GPT-4o, on the other hand, is embedded as **a user-facing AI feature** within the finished app. It provides chatbot interactions, image analysis, content generation, and other intelligent capabilities for your end users.

Think of it as a two-layer architecture: Rork + Gemini **builds** the app, and GPT-4o **makes it intelligent**. Of course, you could also use the Gemini API or Claude API for in-app AI features — choose the model that best fits your project's requirements.
