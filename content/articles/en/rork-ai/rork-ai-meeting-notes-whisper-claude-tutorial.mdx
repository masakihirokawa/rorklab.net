## Introduction — Why Build an AI Meeting Notes App?

Taking notes during meetings, writing summaries afterward, and extracting action items — these tasks can easily consume 30 minutes or more every single time. An AI meeting notes app transforms this entire workflow into a simple "press record and go" experience.

In this tutorial, you'll build a fully functional AI meeting notes app with Rork that includes the following features:

- **Audio Recording**: Capture meeting audio using the device microphone
- **Transcription**: Convert speech to text with OpenAI's Whisper API
- **AI Summarization**: Generate automatic summaries and extract action items using Claude API
- **History Management**: Browse and search past meeting notes

By combining Rork's code generation capabilities with Whisper and Claude — two powerful AI APIs — you can build a production-quality meeting notes app efficiently and with minimal boilerplate.

This tutorial is aimed at intermediate developers, but anyone with basic Rork experience and foundational React Native knowledge should be able to follow along.

## Implementing Audio Recording

To record audio in React Native, you'll use the `expo-av` library. In a Rork project, you can simply describe what you need in a prompt and the dependency installation is handled automatically.

Here's a custom hook that implements the core recording functionality:

```typescript
// hooks/useAudioRecorder.ts
import { Audio } from 'expo-av';
import { useState, useRef } from 'react';

export function useAudioRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const [recordingUri, setRecordingUri] = useState<string | null>(null);
  const recordingRef = useRef<Audio.Recording | null>(null);

  const startRecording = async () => {
    // Request microphone permission
    const permission = await Audio.requestPermissionsAsync();
    if (!permission.granted) {
      throw new Error('Microphone permission is required');
    }

    // Configure audio mode (optimized for Whisper API)
    await Audio.setAudioModeAsync({
      allowsRecordingIOS: true,
      playsInSilentModeIOS: true,
    });

    const recording = new Audio.Recording();
    await recording.prepareToRecordAsync({
      android: {
        extension: '.m4a',
        outputFormat: Audio.AndroidOutputFormat.MPEG_4,
        audioEncoder: Audio.AndroidAudioEncoder.AAC,
        sampleRate: 16000,  // Recommended for Whisper
        numberOfChannels: 1,
        bitRate: 128000,
      },
      ios: {
        extension: '.m4a',
        audioQuality: Audio.IOSAudioQuality.HIGH,
        sampleRate: 16000,
        numberOfChannels: 1,
        bitRate: 128000,
        outputFormat: Audio.IOSOutputFormat.MPEG4AAC,
      },
      web: { mimeType: 'audio/webm', bitsPerSecond: 128000 },
    });

    await recording.startAsync();
    recordingRef.current = recording;
    setIsRecording(true);
  };

  const stopRecording = async () => {
    if (!recordingRef.current) return null;

    await recordingRef.current.stopAndUnloadAsync();
    const uri = recordingRef.current.getURI();
    recordingRef.current = null;
    setIsRecording(false);
    setRecordingUri(uri);
    return uri;
  };

  return { isRecording, recordingUri, startRecording, stopRecording };
}
```

**Key Insight**: Whisper API achieves the best accuracy with a 16kHz sample rate. Recording in mono (single channel) also keeps file sizes small, which speeds up the upload to the API.

As for file size limits, the Whisper API accepts files up to 25MB. A typical one-hour meeting recorded with the settings above will produce roughly 10MB — well within the limit.

## Transcribing Audio with the Whisper API

Once you have a recorded audio file, send it to OpenAI's Whisper API for transcription.

```typescript
// services/transcription.ts
export async function transcribeAudio(audioUri: string): Promise<string> {
  const formData = new FormData();

  // Attach the audio file to FormData
  formData.append('file', {
    uri: audioUri,
    type: 'audio/m4a',
    name: 'meeting-audio.m4a',
  } as any);

  formData.append('model', 'whisper-1');
  formData.append('language', 'ja');        // Specify Japanese
  formData.append('response_format', 'verbose_json');
  formData.append('timestamp_granularities[]', 'segment');

  const response = await fetch(
    'https://api.openai.com/v1/audio/transcriptions',
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${YOUR_OPENAI_API_KEY}`,
      },
      body: formData,
    }
  );

  if (!response.ok) {
    const error = await response.json();
    throw new Error(`Transcription failed: ${error.error?.message}`);
  }

  const result = await response.json();

  // Generate timestamped transcript from segments
  const transcriptWithTimestamps = result.segments
    .map((seg: any) => {
      const minutes = Math.floor(seg.start / 60);
      const seconds = Math.floor(seg.start % 60);
      const timestamp = `[${minutes}:${seconds.toString().padStart(2, '0')}]`;
      return `${timestamp} ${seg.text}`;
    })
    .join('\n');

  return transcriptWithTimestamps;
}

// Expected output:
// [0:00] Let's begin today's weekly meeting.
// [0:05] First, let's review last week's task progress.
// [0:12] The design revisions are complete. They're pending review.
```

**Why `verbose_json`?**: Instead of plain text, specifying `verbose_json` gives you per-segment timestamps. This enables a "jump to source" feature where users can tap a summary point and see the exact moment it was said.

**Cost Estimate**: Whisper API pricing is $0.006 per minute. A one-hour meeting costs roughly $0.36 — remarkably affordable for production use.

## Auto-Summarizing with the Claude API

With the transcript in hand, use Claude API to generate an intelligent summary with structured data extraction.

```typescript
// services/summarize.ts
interface MeetingSummary {
  title: string;
  overview: string;
  keyPoints: string[];
  actionItems: { task: string; assignee: string; deadline: string }[];
  decisions: string[];
  nextSteps: string;
}

export async function summarizeMeeting(
  transcript: string
): Promise<MeetingSummary> {
  const systemPrompt = `You are a skilled business assistant.
Given a meeting transcript, generate a JSON summary in this format:

{
  "title": "Meeting title (inferred from content)",
  "overview": "Overall summary of the meeting (3-5 sentences)",
  "keyPoints": ["Key point 1", "Key point 2", ...],
  "actionItems": [
    {"task": "Task description", "assignee": "Person name", "deadline": "Due date"}
  ],
  "decisions": ["Decision 1", "Decision 2", ...],
  "nextSteps": "Summary of what needs to happen before the next meeting"
}

If an assignee is unknown, use "TBD". If a deadline is unknown, use "To be confirmed".
Output only valid JSON with no additional text.`;

  const response = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': YOUR_ANTHROPIC_API_KEY,
      'anthropic-version': '2023-06-01',
    },
    body: JSON.stringify({
      model: 'claude-sonnet-4-6-20250329',
      max_tokens: 2048,
      system: systemPrompt,
      messages: [
        {
          role: 'user',
          content: `Please summarize the following meeting transcript:\n\n${transcript}`,
        },
      ],
    }),
  });

  if (!response.ok) {
    throw new Error('Failed to generate summary');
  }

  const data = await response.json();
  const content = data.content[0].text;
  return JSON.parse(content) as MeetingSummary;
}
```

**Prompt Design Tip**: By strictly specifying the output format as JSON, you get consistently parseable structured data. Including fallback instructions (like "use TBD if unknown") ensures the process runs smoothly even when information is incomplete.

## Building the End-to-End Pipeline

Now let's combine all the individual features into a seamless pipeline that handles recording, transcription, and summarization with a single tap.

```typescript
// hooks/useMeetingPipeline.ts
import { useState } from 'react';
import { useAudioRecorder } from './useAudioRecorder';
import { transcribeAudio } from '../services/transcription';
import { summarizeMeeting } from '../services/summarize';

type PipelineStatus =
  | 'idle'
  | 'recording'
  | 'transcribing'
  | 'summarizing'
  | 'complete'
  | 'error';

export function useMeetingPipeline() {
  const { isRecording, startRecording, stopRecording } = useAudioRecorder();
  const [status, setStatus] = useState<PipelineStatus>('idle');
  const [transcript, setTranscript] = useState('');
  const [summary, setSummary] = useState<MeetingSummary | null>(null);
  const [error, setError] = useState<string | null>(null);

  const start = async () => {
    setError(null);
    setStatus('recording');
    await startRecording();
  };

  const stop = async () => {
    try {
      // Step 1: Stop recording
      const uri = await stopRecording();
      if (!uri) throw new Error('Recording file not found');

      // Step 2: Transcribe
      setStatus('transcribing');
      const text = await transcribeAudio(uri);
      setTranscript(text);

      // Step 3: AI summarization
      setStatus('summarizing');
      const result = await summarizeMeeting(text);
      setSummary(result);

      setStatus('complete');
    } catch (err) {
      setError(err instanceof Error ? err.message : 'An error occurred during processing');
      setStatus('error');
    }
  };

  return { status, transcript, summary, error, start, stop, isRecording };
}
```

**UX Consideration**: By tracking each step's status, you can show clear progress indicators like "Transcribing..." and "Generating summary..." to keep users informed. For a one-hour meeting, expect approximately 30 seconds for transcription and 10 seconds for summarization.

## Meeting History and Search

To manage past meeting notes, implement local storage with AsyncStorage along with a search capability.

```typescript
// services/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const MEETINGS_KEY = 'meeting_notes';

export interface SavedMeeting {
  id: string;
  date: string;
  title: string;
  overview: string;
  transcript: string;
  summary: MeetingSummary;
}

export async function saveMeeting(meeting: SavedMeeting) {
  const existing = await getMeetings();
  existing.unshift(meeting); // Most recent first
  await AsyncStorage.setItem(MEETINGS_KEY, JSON.stringify(existing));
}

export async function getMeetings(): Promise<SavedMeeting[]> {
  const data = await AsyncStorage.getItem(MEETINGS_KEY);
  return data ? JSON.parse(data) : [];
}

// Full-text search across titles, overviews, and transcripts
export async function searchMeetings(query: string): Promise<SavedMeeting[]> {
  const meetings = await getMeetings();
  const lowerQuery = query.toLowerCase();
  return meetings.filter(
    (m) =>
      m.title.toLowerCase().includes(lowerQuery) ||
      m.overview.toLowerCase().includes(lowerQuery) ||
      m.transcript.toLowerCase().includes(lowerQuery)
  );
}
```

## Security and Cost Management Best Practices

When deploying an AI-powered app to production, security and cost management become critical concerns.

**Securing Your API Keys:**

Never embed API keys directly in client-side code. In production, always route API calls through your own backend server.

```typescript
// ❌ Dangerous: API key exposed in client code
const response = await fetch('https://api.openai.com/v1/audio/transcriptions', {
  headers: { Authorization: `Bearer ${YOUR_API_KEY}` },
});

// ✅ Safe: Route through your own backend
const response = await fetch('https://your-backend.com/api/transcribe', {
  method: 'POST',
  headers: { Authorization: `Bearer ${userToken}` },
  body: formData,
});
```

You can set up a serverless API proxy using Supabase Edge Functions or Cloudflare Workers. In Rork, just prompt "Create a Supabase Edge Function that proxies requests to the Whisper API" and it will generate the corresponding code.

**Cost Control Strategies:**

- Set monthly spending limits on both OpenAI and Anthropic dashboards
- Implement per-user usage caps (e.g., 5 free transcriptions per month)
- Split long recordings into chunks for predictable per-session costs

**Cost Breakdown:**

- 30-minute meeting: Whisper $0.18 + Claude ~$0.02 = **~$0.20**
- 1-hour meeting: Whisper $0.36 + Claude ~$0.04 = **~$0.40**
- 20 one-hour meetings per month: **~$8/month**

## Frequently Asked Questions (FAQ)

### Q1: How accurate is Whisper for Japanese meeting transcription?

Whisper is a multilingual model with strong Japanese language support. Specifying `language: 'ja'` explicitly optimizes recognition for Japanese audio. That said, specialized jargon or internal company terminology may occasionally be misrecognized, so we recommend reviewing transcripts for important meetings.

### Q2: What happens if the app goes to the background while recording?

On iOS, you need to enable background audio recording by adding the `audio` mode to `UIBackgroundModes` in `Info.plist`. In Rork, you can prompt "Enable background audio recording" to configure this automatically. On Android, running the recording as a foreground service is the recommended approach.

### Q3: Can the app work offline?

The recording feature itself works offline, but transcription (Whisper API) and summarization (Claude API) require an internet connection. A good pattern is to record offline and implement a queue that automatically processes recordings when connectivity is restored.

### Q4: Can it distinguish between different speakers?

The Whisper API alone does not support speaker diarization. If you need speaker identification, you can either integrate with APIs that support it natively (such as AssemblyAI or Deepgram), or add a post-processing step where Claude attempts to identify speakers from contextual clues in the transcript.
