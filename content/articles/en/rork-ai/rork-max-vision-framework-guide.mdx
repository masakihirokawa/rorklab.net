# Rork Max × Vision Framework Guide

Apple's Vision framework provides high-accuracy image analysis capabilities built directly into iOS. With Rork Max, you can integrate these powerful features into your apps using nothing more than AI-driven prompts.

This guide covers three core Vision capabilities — **text recognition (OCR)**, **object detection**, and **barcode scanning** — with practical Rork Max prompts and Swift code examples.

## What Is the Vision Framework?

Vision is Apple's native image analysis framework, available since iOS 11. It runs machine learning models directly on the device, so it works without a network connection and keeps user data private.

Here's what makes it stand out:

- **On-device processing**: No data leaves the device, preserving user privacy while delivering fast results
- **Neural Engine acceleration**: Takes advantage of the iPhone's dedicated AI chip for faster inference
- **Broad request types**: Over 20 analysis capabilities including text recognition, face detection, object tracking, and barcode reading
- **Core ML integration**: You can plug custom models into the Vision pipeline for specialized tasks

## Implementing Text Recognition (OCR)

Text recognition powers a wide range of apps — business card readers, receipt scanners, document digitizers, and more.

### Rork Max Prompt Example

To add text recognition to your Rork Max app, try a prompt like this:

```
Build an app that reads text from camera-captured images.
- Show a full-screen camera view
- Add a shutter button to capture photos
- Use VNRecognizeTextRequest from Vision to extract text from the captured image
- Display recognized text in a scrollable list
- Tapping a text item copies it to the clipboard
- Support both English and Japanese text recognition
```

### Key Implementation Details

The core of the generated code revolves around `VNRecognizeTextRequest`. Starting with iOS 16, the `automaticallyDetectsLanguage` property enables multi-language recognition without explicitly specifying target languages.

```swift
let request = VNRecognizeTextRequest { request, error in
    guard let observations = request.results as? [VNRecognizedTextObservation] else { return }
    let recognizedText = observations.compactMap { observation in
        observation.topCandidates(1).first?.string
    }
    DispatchQueue.main.async {
        self.detectedTexts = recognizedText
    }
}
request.recognitionLevel = .accurate
request.automaticallyDetectsLanguage = true
```

There are two recognition levels to choose from: `.fast` and `.accurate`. A common pattern is to use `.fast` for real-time camera previews and `.accurate` for processing after the user taps the shutter button.

### Tips for Improving Accuracy

Setting `recognitionLanguages` to prioritize specific languages improves recognition accuracy for those languages. The `minimumTextHeight` property helps filter out noise by ignoring text below a certain size threshold.

Image preprocessing also matters. Low-contrast or blurry images produce worse results, so applying `CIFilter` adjustments after capture can significantly improve recognition quality.

## Implementing Object Detection

Vision's object detection features let you identify and locate objects within images — useful for document scanning, augmented reality overlays, and inventory management.

### Rork Max Prompt Example

```
Build an app that overlays object detection results on a live camera preview.
- Use AVCaptureSession to capture the camera feed
- Detect rectangles using VNDetectRectanglesRequest
- Draw bounding boxes around detected rectangles on the camera preview
- Add a document scan mode that crops and perspective-corrects the detected rectangle
```

### Rectangle Detection in Practice

Rectangle detection is the foundation of document scanning. `VNDetectRectanglesRequest` identifies document and card edges, then applies perspective correction to produce a front-facing image.

```swift
let rectangleRequest = VNDetectRectanglesRequest { request, error in
    guard let results = request.results as? [VNRectangleObservation],
          let rectangle = results.first else { return }

    let topLeft = rectangle.topLeft
    let topRight = rectangle.topRight
    let bottomLeft = rectangle.bottomLeft
    let bottomRight = rectangle.bottomRight

    // Apply perspective correction with CIFilter
    self.applyPerspectiveCorrection(
        topLeft: topLeft, topRight: topRight,
        bottomLeft: bottomLeft, bottomRight: bottomRight
    )
}
rectangleRequest.minimumConfidence = 0.8
rectangleRequest.maximumObservations = 1
```

Setting `minimumConfidence` higher reduces false positives. For document scanning, 0.8 or above is a good threshold.

### Integrating Custom Object Detection Models

For more advanced object detection, you can integrate custom models trained with Create ML into the Vision pipeline. In Rork Max, use a prompt like:

```
Add a Core ML object detection model (.mlmodel) to the project
and perform real-time inference on the camera feed.
Use VNCoreMLRequest to integrate it into the Vision pipeline.
```

## Barcode & QR Code Scanning

Vision handles barcode and QR code reading out of the box — ideal for product management, ticket validation, and URL sharing.

### Rork Max Prompt Example

```
Build a scanner app that reads QR codes and barcodes.
- Show a camera preview with a scan frame in the center
- Use VNDetectBarcodesRequest to support QR, EAN-13, and Code 128 formats
- Display scan results in card-style UI
- Add an "Open in Safari" button when the result contains a URL
- Trigger haptic feedback on successful scan
```

### Key Implementation Details

`VNDetectBarcodesRequest` lets you specify which symbologies to detect through its `symbologies` property.

```swift
let barcodeRequest = VNDetectBarcodesRequest { request, error in
    guard let results = request.results as? [VNBarcodeObservation] else { return }
    for barcode in results {
        if let payload = barcode.payloadStringValue {
            DispatchQueue.main.async {
                self.scannedCodes.append(ScannedCode(
                    value: payload,
                    symbology: barcode.symbology,
                    timestamp: Date()
                ))
            }
        }
    }
}
barcodeRequest.symbologies = [.qr, .ean13, .code128, .pdf417]
```

When implementing continuous scanning, you'll need duplicate detection logic. A common approach is caching recent results and skipping identical values within a time window — say, 3 seconds.

## Performance Optimization

Real-time Vision processing requires thoughtful performance tuning.

**Reuse request objects**: While `VNImageRequestHandler` is created fresh for each frame, `VNRequest` objects can and should be reused. Creating new ones every frame increases memory overhead.

**Separate processing queues**: Keep camera frame capture on the main thread and Vision inference on a background queue to avoid dropping UI frames.

```swift
private let visionQueue = DispatchQueue(
    label: "com.app.vision",
    qos: .userInitiated
)
```

**Skip frames**: You rarely need to process every frame from a 30fps camera feed. Analyzing every second or third frame is usually sufficient for a smooth user experience.

**Set a Region of Interest (ROI)**: Processing only the scan frame area instead of the entire image improves both speed and accuracy.

```swift
request.regionOfInterest = CGRect(x: 0.25, y: 0.25, width: 0.5, height: 0.5)
```

## Best Practices for Rork Max Development

Here are some tips for building Vision-powered apps efficiently with Rork Max.

**Build incrementally**: Start with a prompt that just displays the camera preview. Once that works, add Vision processing in a follow-up prompt. This step-by-step approach produces more reliable results.

**Specify error handling explicitly**: Include instructions in your prompt for handling camera permission denials and Vision request failures. Rork Max generates better fallback UI when you describe the expected behavior.

**Add a test mode**: Ask Rork Max to include a test mode that processes static images. This lets you test Vision features in the simulator before moving to a physical device, speeding up your development cycle.

**Define camera permission strings**: Specify the `NSCameraUsageDescription` text in your prompt. A clear, user-friendly permission message reduces the risk of App Store review rejections.

The Vision framework brings text recognition, object detection, and barcode scanning to iOS apps without relying on cloud services. Combined with Rork Max's prompt-driven development, you can integrate these features in a fraction of the time traditional coding requires.

With privacy-conscious users on the rise, on-device AI capabilities are a meaningful competitive advantage. Start with text recognition to get a feel for the Vision framework, then explore object detection and barcode scanning to build truly powerful apps.
