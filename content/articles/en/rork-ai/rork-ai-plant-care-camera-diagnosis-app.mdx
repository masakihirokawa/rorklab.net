## Introduction — Using AI to Monitor Plant Health

"Are those yellow spots on my leaves something to worry about?" "How often should I actually water this plant?" If you've ever kept houseplants, you know these questions all too well. Thanks to advances in AI image recognition, it's now possible to build a mobile app that diagnoses plant health simply by pointing your phone's camera at a leaf.

In this tutorial, we'll walk through building an **AI-powered plant care app** with Rork — from prompt design to UI implementation. The finished app will let users photograph their plants, then have an AI analyze leaf color, shape, spots, and more, returning actionable care recommendations.

**What you'll learn:**

- How to implement camera functionality in Rork and send captured images to an AI
- Prompt engineering techniques for multimodal AI (image + text) plant diagnosis
- Building intuitive UI components for displaying diagnosis results
- Persisting diagnosis history so users can track plant health over time

**Prerequisites:** Familiarity with Rork basics and an interest in building AI-powered apps.

## Implementing Camera Functionality — expo-camera and expo-image-picker

The foundation of any plant diagnosis app is camera access. With Rork's React Native / Expo ecosystem, you can implement camera functionality with surprisingly little code.

Here's the kind of prompt you'd give Rork to generate the camera screen:

```
Create a camera screen for photographing plants.
Include the following features:
- Real-time camera preview
- Circular capture button centered at the bottom of the screen
- Gallery picker button to select existing photos
- Post-capture preview screen with "Diagnose" and "Retake" buttons
```

The generated code uses `expo-camera` for the live preview and `expo-image-picker` for gallery access. The key detail is encoding the captured image as base64 so it can be sent to the AI API.

```typescript
// Convert captured image to base64 for AI API
import * as FileSystem from 'expo-file-system';

const captureAndEncode = async (cameraRef: CameraView) => {
  // Take a photo and get the image URI
  const photo = await cameraRef.takePictureAsync({
    quality: 0.8,  // Image quality (0-1) — 0.8 balances quality and transfer speed
    base64: true,   // Get base64 encoding alongside the file
  });

  // photo.base64 contains the encoded image data
  return {
    uri: photo.uri,
    base64: photo.base64,
  };
};
// Expected output: { uri: "file:///...", base64: "/9j/4AAQSkZ..." }
```

The `quality` parameter is important here. Setting it to `1.0` produces large files that slow down API calls. For plant diagnosis purposes, `0.7–0.8` provides more than enough detail for accurate analysis.

## AI Prompt Design — The Key to Accurate Diagnoses

Simply sending an image to an AI won't produce useful results. **Prompt design** is what makes the difference between a vague response and an actionable diagnosis.

Here's a system prompt optimized for plant health analysis:

```typescript
// System prompt for plant diagnosis
const PLANT_DIAGNOSIS_PROMPT = `
You are an AI assistant specializing in plant health diagnosis.
Analyze the plant image provided by the user and respond in the following format:

## Response Format
1. **Plant Species**: Estimated plant name based on the image (use "Unable to identify" if unclear)
2. **Health Status**: Rate as "Healthy", "Needs Attention", or "Action Required"
3. **Observed Symptoms**: Specific findings about leaf color, shape, spots, wilting, etc.
4. **Likely Causes**: Underwatering, insufficient light, disease, pests, etc.
5. **Recommended Actions**: Up to 3 specific care suggestions
6. **Urgency Level**: "Low", "Medium", or "High"

## Guidelines
- Use beginner-friendly language and avoid overly technical terminology
- Use hedging language such as "this may indicate..." rather than definitive claims
- If the image is unclear, suggest retaking the photo
`;
// Expected output: Structured JSON diagnosis result
```

The crucial element here is **specifying a clear output format**. When the AI returns structured data, your app can parse it reliably and map each field to the appropriate UI element.

To implement this in Rork, you'd provide a prompt like this:

```
Implement the AI diagnosis feature:
- Send the captured image as base64 to the AI API
- Parse the response as JSON into structured data
- Display results on screen (plant name, health status, symptoms, recommendations)
- Show a skeleton UI while loading
- Display a retry button on error
```

## Building the Results UI — Communicating Clearly with Users

The diagnosis results need to be presented in a way that's immediately understandable. Simply dumping text on the screen won't convey the value of the AI diagnosis.

Here's a UI specification to give Rork:

```
Create the diagnosis results screen:

1. Header section:
   - Captured plant image (rounded corners with shadow)
   - Health status badge (Healthy=green, Attention=yellow, Action Required=red)

2. Diagnosis card:
   - Plant name with species icon
   - Symptom details (collapsible accordion sections)
   - Causes list with icons

3. Recommendations section:
   - Each recommendation displayed as a card
   - Ordered by priority
   - "Learn more" tap to expand additional details

4. Action buttons:
   - "Save to History" button
   - "Take Another Photo" button
   - "Share" button

Use a green-themed color palette.
```

Color-coding health status gives users an instant visual understanding of their plant's condition.

## Saving Diagnosis History — Persisting Data with AsyncStorage

A one-time diagnosis has limited value. By saving diagnosis history locally, users can track their plants' health over time and spot trends.

```typescript
// Save and retrieve diagnosis history using AsyncStorage
import AsyncStorage from '@react-native-async-storage/async-storage';

interface DiagnosisRecord {
  id: string;
  plantName: string;
  imageUri: string;
  status: HealthStatus;
  symptoms: string[];
  recommendations: string[];
  diagnosedAt: string;  // ISO 8601 format
}

// Save a diagnosis result
const saveDiagnosis = async (record: DiagnosisRecord) => {
  const existing = await AsyncStorage.getItem('diagnosis_history');
  const history: DiagnosisRecord[] = existing ? JSON.parse(existing) : [];

  history.unshift(record);  // Add new record to the front

  // Keep a maximum of 100 records to conserve storage
  const trimmed = history.slice(0, 100);
  await AsyncStorage.setItem('diagnosis_history', JSON.stringify(trimmed));
};

// Retrieve diagnosis history
const getDiagnosisHistory = async (): Promise<DiagnosisRecord[]> => {
  const data = await AsyncStorage.getItem('diagnosis_history');
  return data ? JSON.parse(data) : [];
};
// Expected output: [{ id: "abc123", plantName: "Monstera", status: "good", ... }]
```

For the history screen, prompt Rork with something like:

```
Create a diagnosis history screen:
- Display history as cards in a FlatList
- Each card shows plant photo thumbnail, plant name, health badge, and diagnosis date
- Tap a card to navigate to the detail screen
- Search bar and health status filter at the top
- Empty state: "No diagnosis history yet" with a button to start scanning
- Swipe to delete
```

By regularly photographing the same plant, users can monitor health trends over time — an incredibly valuable feature for gardening beginners.

## Advanced Features — Watering Reminders and Plant Encyclopedia

Once the core diagnosis feature is working, add engagement-driving features to keep users coming back.

**Watering reminders** use the AI's diagnosis to suggest optimal watering frequency, then notify users via local push notifications.

```typescript
// Schedule watering reminders
import * as Notifications from 'expo-notifications';

const scheduleWateringReminder = async (
  plantName: string,
  intervalDays: number  // AI-recommended watering interval in days
) => {
  await Notifications.scheduleNotificationAsync({
    content: {
      title: 'Time to water your plant 💧',
      body: `${plantName} needs watering. This schedule is based on your last diagnosis.`,
      sound: 'default',
    },
    trigger: {
      type: Notifications.SchedulableTriggerInputTypes.TIME_INTERVAL,
      seconds: intervalDays * 24 * 60 * 60,
      repeats: true,
    },
  });
};
// Expected output: Recurring push notification at the specified interval
```

A **plant encyclopedia** compiles information from past diagnoses into a personal reference. As the AI identifies plants and provides details like scientific names, native habitats, and optimal light conditions, a knowledge base naturally builds up over time.

## Frequently Asked Questions

### Q1: How accurate is AI plant diagnosis?

AI-powered plant diagnosis performs well for common symptoms like underwatering, insufficient light, and widespread diseases. However, it shouldn't replace professional pathological diagnosis. For serious symptoms, it's good practice to include a prompt in your app encouraging users to consult a local garden center or plant clinic.

### Q2: How much does the API cost per diagnosis?

Multimodal API calls (image + text) cost more than text-only requests. Expect roughly a few cents per diagnosis. To optimize costs, compress image resolution appropriately (using the `quality: 0.8` setting described above), implement caching to avoid re-diagnosing the same image, and consider batching requests where possible.

### Q3: Can the app work offline?

Full offline diagnosis would require on-device AI models like TensorFlow Lite or Core ML. With Rork's standard cloud-based approach, the diagnosis itself requires an internet connection. However, cached diagnosis results and watering reminders work offline. A good UX pattern is to show a "Check your connection" message when offline and automatically run the diagnosis once connectivity is restored.

### Q4: How can I monetize this app?

A **freemium model** works well: offer basic diagnosis for free and charge for premium features. Consider limiting free users to 5 diagnoses per month (unlimited for paid), offering detailed growth analysis reports, personalized AI care plans, and an ad-free experience as paid tiers. Rork integrates smoothly with Stripe for subscription billing — explore existing guides on the topic for implementation details.
