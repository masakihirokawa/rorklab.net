## Introduction — An App That Turns Your Fridge Into a Menu

We've all been there: standing in front of an open fridge, staring at a random collection of ingredients, wondering what to cook. What if your phone could figure that out for you?

In this tutorial, you'll build an AI-powered app with Rork that uses Gemini Vision API to recognize ingredients from a photo and automatically suggest recipes you can make right now. No coding background is required — Rork's AI generation handles the heavy lifting while you focus on the creative decisions.

Here's what you'll learn:

- How to implement camera capture and gallery selection with Expo Image Picker
- Connecting to Gemini Vision API for ingredient recognition
- Designing effective prompts for recipe generation from detected ingredients
- Building a favorites system with AsyncStorage
- Screen navigation patterns and UI design tips for food apps

This tutorial is aimed at intermediate Rork users who have built one or two apps before. Even if camera features or API integration are new to you, the step-by-step approach will guide you through every part of the process.

## App Architecture — Three Screens for a Clean UX

Let's start by mapping out the app structure. To keep things intuitive, we'll use a straightforward three-screen layout.

**Home Screen (Camera)**: The first thing users see is a prominent camera button. They can either take a photo of their fridge contents or select an existing image from their gallery. Once captured, the AI recognition process kicks in.

**Recipe List Screen**: After processing, the app displays recognized ingredient tags alongside 3–5 recipe suggestions. Each recipe card shows estimated cooking time, difficulty level, and whether any additional ingredients are needed.

**Recipe Detail Screen**: Tapping a recipe reveals the full ingredient list, step-by-step instructions, and cooking tips. A heart button lets users save recipes to their favorites for later.

Here's an example prompt to generate this structure in Rork:

```
Build an AI ingredient recognition recipe app.

Screens:
1. Home - Camera capture button, gallery picker, recent scan history
2. Recipe list - Recognized ingredient tags, recommended recipe cards
3. Recipe detail - Ingredients, steps, cooking time, favorite button

Design: Warm orange and cream color scheme,
rounded cards with shadows, ingredient tags as chips
```

Feed this prompt into Rork and the basic screen layout and navigation will be generated automatically. From here, we'll add the camera functionality and AI integration.

## Implementing Camera Capture with Expo Image Picker

The first step in ingredient recognition is letting users take photos. Rork comes with Expo Image Picker pre-installed, so you can use camera features without any additional setup.

```typescript
// Camera capture and gallery selection implementation
import * as ImagePicker from 'expo-image-picker';

// Function to take a photo
const takePhoto = async () => {
  // Request camera permission
  const { status } = await ImagePicker.requestCameraPermissionsAsync();
  if (status !== 'granted') {
    alert('Please allow camera access to use this feature');
    return;
  }

  // Launch camera and capture
  const result = await ImagePicker.launchCameraAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    quality: 0.8,        // Quality (0-1): 0.8 balances quality and file size
    base64: true,         // Enable Base64 encoding for API submission
    allowsEditing: true,  // Allow cropping after capture
    aspect: [4, 3],       // Crop aspect ratio
  });

  if (!result.canceled && result.assets[0]) {
    // Capture successful — send to recognition API
    const base64Image = result.assets[0].base64;
    await recognizeIngredients(base64Image);
  }
};

// Function to select from gallery
const pickFromGallery = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    quality: 0.8,
    base64: true,
    allowsEditing: true,
  });

  if (!result.canceled && result.assets[0]) {
    const base64Image = result.assets[0].base64;
    await recognizeIngredients(base64Image);
  }
};
```

The key setting here is `base64: true`. This returns the captured image as a Base64 string, which can be sent directly to the Gemini Vision API. Setting `quality: 0.8` strikes the best balance between image quality and file size — important since it affects API response times.

## Recognizing Ingredients with Gemini Vision API

To identify ingredients from photos, we'll use Google's Gemini Vision API. Gemini is a multimodal AI that can understand image content and output structured text descriptions.

Let's build a utility function for API communication:

```typescript
// Recognize ingredients using Gemini Vision API
const GEMINI_API_URL = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent';

interface RecognizedIngredient {
  name: string;       // Ingredient name
  confidence: number; // Recognition confidence (0-1)
  category: string;   // Category (vegetable, meat, fish, dairy, etc.)
}

const recognizeIngredients = async (base64Image: string): Promise<RecognizedIngredient[]> => {
  try {
    const response = await fetch(`${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        contents: [{
          parts: [
            {
              text: `Identify all food ingredients visible in this image.
              Respond in this exact JSON format:
              [
                {
                  "name": "ingredient name",
                  "confidence": 0.95,
                  "category": "vegetable|meat|fish|dairy|grain|condiment|fruit|other"
                }
              ]
              Rules:
              - Do not include non-food items (plates, shelves, plastic wrap, etc.)
              - Exclude ingredients with confidence below 0.5
              - Return only the JSON array with no additional text`
            },
            {
              inlineData: {
                mimeType: 'image/jpeg',
                data: base64Image,
              },
            },
          ],
        }],
        generationConfig: {
          temperature: 0.1,  // Low temperature for stable recognition results
          maxOutputTokens: 1024,
        },
      }),
    });

    const data = await response.json();
    const text = data.candidates[0].content.parts[0].text;

    // Parse and return the JSON
    const jsonMatch = text.match(/\[[\s\S]*\]/);
    if (jsonMatch) {
      return JSON.parse(jsonMatch[0]);
    }
    return [];
  } catch (error) {
    console.error('Ingredient recognition error:', error);
    return [];
  }
};
// Expected output example:
// [
//   { "name": "tomatoes", "confidence": 0.97, "category": "vegetable" },
//   { "name": "chicken breast", "confidence": 0.92, "category": "meat" },
//   { "name": "onion", "confidence": 0.89, "category": "vegetable" },
//   { "name": "garlic", "confidence": 0.85, "category": "vegetable" }
// ]
```

A few important design decisions here: `temperature: 0.1` prioritizes accuracy over creativity, which is exactly what you want for ingredient recognition. By explicitly instructing the model to "return only the JSON array," we ensure the output is easy to parse programmatically.

## Generating Recipes from Recognized Ingredients

Once ingredients are identified, the next step is generating recipe suggestions. We'll use Gemini again, this time as a text generation model.

```typescript
// Generate recipes from an ingredient list
interface Recipe {
  id: string;
  title: string;
  cookingTime: number;   // minutes
  difficulty: 'easy' | 'medium' | 'hard';
  servings: number;
  ingredients: { name: string; amount: string; isAvailable: boolean }[];
  steps: string[];
  tips: string;
}

const generateRecipes = async (
  ingredients: RecognizedIngredient[]
): Promise<Recipe[]> => {
  const ingredientNames = ingredients.map(i => i.name).join(', ');

  const response = await fetch(`${GEMINI_API_URL}?key=YOUR_GEMINI_API_KEY`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{
        parts: [{
          text: `Suggest 3-5 recipes using the following ingredients.

Available ingredients: ${ingredientNames}

Respond in this exact JSON format:
[
  {
    "id": "unique-id",
    "title": "Recipe name",
    "cookingTime": 20,
    "difficulty": "easy",
    "servings": 2,
    "ingredients": [
      { "name": "ingredient", "amount": "quantity", "isAvailable": true }
    ],
    "steps": ["Step 1", "Step 2", "Step 3"],
    "tips": "Cooking tips or variations"
  }
]

Rules:
- Focus recipes on the available ingredients
- Mark additional ingredients as isAvailable: false
- Limit additional ingredients to common pantry staples (salt, soy sauce, oil, etc.)
- Difficulty: easy (under 15 min), medium (under 30 min), hard (over 30 min)
- Prioritize practical, everyday home cooking
- Return only the JSON array`
        }],
      }],
      generationConfig: {
        temperature: 0.7,  // Moderate diversity for recipe suggestions
        maxOutputTokens: 4096,
      },
    }),
  });

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;
  const jsonMatch = text.match(/\[[\s\S]*\]/);

  if (jsonMatch) {
    return JSON.parse(jsonMatch[0]);
  }
  return [];
};
// Expected output (for tomatoes, chicken breast, onion, garlic):
// [
//   {
//     "id": "chicken-tomato-1",
//     "title": "Chicken Tomato Stew",
//     "cookingTime": 25,
//     "difficulty": "easy",
//     "servings": 2,
//     "ingredients": [
//       { "name": "chicken breast", "amount": "200g", "isAvailable": true },
//       { "name": "tomatoes", "amount": "2 medium", "isAvailable": true },
//       ...
//     ],
//     "steps": ["Cut chicken into bite-sized pieces", "Dice onion and mince garlic", ...],
//     "tips": "Coat chicken in cornstarch before cooking for extra juiciness"
//   },
//   ...
// ]
```

Notice how the `temperature` for recipe generation is set to `0.7` compared to `0.1` for ingredient recognition. Recipe suggestions benefit from variety — users don't want the same three recipes every time they scan the same ingredients.

## Building a Favorites System with AsyncStorage

Let users save recipes they love. AsyncStorage comes ready to use in Rork apps, making local data persistence straightforward.

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

const FAVORITES_KEY = '@recipe_favorites';

// Save a recipe to favorites
const saveFavorite = async (recipe: Recipe) => {
  try {
    const existing = await AsyncStorage.getItem(FAVORITES_KEY);
    const favorites: Recipe[] = existing ? JSON.parse(existing) : [];

    // Check for duplicates
    if (favorites.some(f => f.id === recipe.id)) {
      return; // Already saved
    }

    favorites.unshift(recipe); // Add to the beginning
    await AsyncStorage.setItem(FAVORITES_KEY, JSON.stringify(favorites));
  } catch (error) {
    console.error('Save error:', error);
  }
};

// Get all favorite recipes
const getFavorites = async (): Promise<Recipe[]> => {
  try {
    const data = await AsyncStorage.getItem(FAVORITES_KEY);
    return data ? JSON.parse(data) : [];
  } catch (error) {
    console.error('Fetch error:', error);
    return [];
  }
};

// Remove a recipe from favorites
const removeFavorite = async (recipeId: string) => {
  try {
    const existing = await AsyncStorage.getItem(FAVORITES_KEY);
    const favorites: Recipe[] = existing ? JSON.parse(existing) : [];
    const updated = favorites.filter(f => f.id !== recipeId);
    await AsyncStorage.setItem(FAVORITES_KEY, JSON.stringify(updated));
  } catch (error) {
    console.error('Remove error:', error);
  }
};
```

This implementation works well for most use cases. If you expect users to save more than 100 recipes, consider migrating to MMKV or SQLite for better performance. AsyncStorage is optimized for small to medium datasets.

// Calculate ingredient coverage percentage
const calculateCoverage = (recipe: Recipe): number => {
  const total = recipe.ingredients.length;
  const available = recipe.ingredients.filter(i => i.isAvailable).length;

## Error Handling and Offline Considerations

Error handling is often overlooked in AI-powered apps. When network connectivity is spotty or the API times out, your app should handle it gracefully.

```typescript
import NetInfo from '@react-native-community/netinfo';

// Check network before calling the API
const safeRecognize = async (base64Image: string) => {
  // Verify network connection
  const netState = await NetInfo.fetch();
  if (!netState.isConnected) {
    alert('Please check your internet connection. Ingredient recognition requires network access.');
    return null;
  }

  // API call with timeout
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), 15000); // 15-second timeout

  try {
    const result = await recognizeIngredients(base64Image);
    clearTimeout(timeoutId);

    if (result.length === 0) {
      // No ingredients detected
      alert('Could not recognize any ingredients. Try taking the photo in better lighting with ingredients clearly visible.');
      return null;
    }

    return result;
  } catch (error) {
    clearTimeout(timeoutId);
    if (error instanceof Error && error.name === 'AbortError') {
      alert('Request timed out. Please try again.');
    } else {
      alert('Something went wrong. Please wait a moment and try again.');
    }
    return null;
  }
};
```

You can also cache previous recognition results so the app skips the API call when users scan the same combination of ingredients. This reduces API costs (Gemini uses pay-per-use pricing) and improves the user experience.

## Frequently Asked Questions

### Q1: How much does the Gemini API cost?

Gemini 2.5 Flash includes a generous free tier with a daily request allowance. For personal projects with moderate usage (a few dozen requests per day), you'll likely stay within the free limits. Vision requests cost slightly more than text-only requests, but typically amount to just a few cents each. For production apps serving many users, monitor your usage in Google AI Studio and set up budget alerts.

### Q2: How accurate is the ingredient recognition?

Accuracy depends heavily on photo conditions. With good lighting and clearly visible ingredients, you can expect 90%+ accuracy. Dim environments or overlapping items reduce precision. To improve results, add "prioritize common grocery store ingredients" to your prompt, and provide a UI where users can manually correct the recognition results — this makes the app far more practical.

### Q3: Does the camera work on both iOS and Android?

Yes, Expo Image Picker supports both platforms. Since Rork generates cross-platform apps, camera features work on iOS and Android without additional configuration. The permission request behavior differs slightly between platforms — iOS requires a usage description in `Info.plist` — but Expo handles this automatically.

### Q4: Should users be able to edit the recognition results?

Absolutely. Since AI recognition isn't perfect, providing a UI where users can add, remove, or correct ingredient tags makes the app significantly more useful. Display recognized ingredients as tappable chips that can be deleted, with a text input for manual additions.
