When using Rork AI to generate apps, prompts don't always produce the expected results. While you may have a clear vision of what app to build, the quality of your prompt dramatically affects the generated output. This FAQ guide covers effective prompt techniques and answers common questions about generating better apps with Rork AI.

## Core Principles of Effective Prompting

To generate high-quality apps with Rork AI, focus on three key elements:

1. **Specificity**: Replace vague language with concrete feature and design requirements
2. **Sequencing**: Break complex requests into manageable steps
3. **Validation**: Verify generated apps work as intended and iterate for improvement

## Frequently Asked Questions

### Q1: My prompt generated a completely different app than expected. What went wrong?

**A1: Your prompt was too vague. Generic terms like "AI," "search," or "social media" don't give the AI enough direction on priorities.**

Compare these prompts:

```
❌ Vague:
"Create a task management app"

✅ Specific:
"Create a task management app with:
- Task categories (work, personal, shopping)
- Deadline and time settings
- Checkmark display for completed tasks
- Progress gauge per category
- Dark mode support
- Home screen prioritizes today's tasks"
```

Key points:

- **List features**: Use bullet points for each capability
- **Show priorities**: Indicate which features are primary vs. secondary
- **Specify design**: Mention layouts ("card-based") and color preferences
- **Describe usage**: Include how users will actually interact ("users check daily progress each morning")

### Q2: The generated app's design looks cheap. How can I improve it?

**A2: You need to explicitly specify design requirements. Without guidance, AI defaults to minimalist designs.**

Design specification example:

```
"Generate with these design requirements:
- Color scheme: navy (#1a2847) and coral pink (#ff6b6b)
- Fonts: headers in SF Bold, body in SF Regular
- Spacing: generous padding (16px minimum)
- Icons: use Heroicons
- Shadows: subtle iOS-style shadows
- Buttons: 12px rounded corners, primary color background, white text
- Gradients: subtle gradient on home background (navy to deep purple)"
```

Reference design systems:

- "Follow Material Design 3 guidelines"
- "Follow Apple iOS 18 Human Interface Guidelines"
- "Use TailwindCSS color system (slate, blue, indigo, etc.)"

### Q3: The app generated but the internal calculation logic seems wrong.

**A3: You need to specify business logic in detail. Include formulas and algorithms explicitly.**

Logic specification example:

```
"Create a budget app with:

【Monthly category totals】
- Sum expenses by category (food, transport, entertainment)

【Budget comparison】
- Display (actual / budget × 100) as percentage
- Show red warning when exceeding 100%

【Savings calculation】
- Monthly savings = income - expenses
- Annual savings rate = annual savings / annual income × 100"
```

### Q4: Buttons don't respond to taps. How should I specify interaction?

**A4: Clearly specify what happens when users interact with elements.**

Interaction specification:

```
"When user taps 'Add' button on task form:
1. Validate task name is not empty
2. If empty: show toast 'Enter task name' (disappears after 1 second)
3. If valid:
   - Save task to local storage
   - Return to main screen
   - Auto-refresh task list
4. During save: disable 'Add' button and show loading spinner"
```

For each action, specify:

- **Trigger**: button tap, swipe, input focus, etc.
- **Validation**: required field checks, value ranges, etc.
- **Success behavior**: navigate screen, save data, show notification
- **Error behavior**: error message, retry option, etc.

### Q5: How should I specify API integration and API key handling?

**A5: Write detailed API integration requirements with security considerations.**

API integration example:

```
"Integrate with OpenWeather API:

【API specification】
- Endpoint: https://api.openweathermap.org/data/2.5/weather
- Method: GET
- Timeout: 10 seconds

【API key management】
- Read API key from environment variable OPENWEATHER_API_KEY
- Store in .env file (never hardcode)
- Never log or display API key

【Response handling】
- Success (200): Extract and display main, temp, humidity
- Not found (404): Show 'City not found'
- Network error: Show 'Connection failed. Check internet.'

【Caching】
- Cache same-city weather for 5 minutes to reduce requests"
```

### Q6: Can I add debug mode or test features?

**A6: Yes. Specify development features separately or use conditional logic.**

Development environment specification:

```
"Add these development features (hidden in production):

【Debug panel】
- Hidden button at screen bottom-right (tap 5 times to show)
- Show: app version, local storage data (JSON), last API call info

【Logging】
- Log all API calls to console
- Log user actions (button taps, screen transitions)
- Log stack traces on errors

【Environment detection】
- Read REACT_APP_ENV variable ('development' or 'production')
- Only show debug features in development mode"
```

### Q7: The app runs but feels really slow. Why?

**A7: Without explicit performance requirements, AI only optimizes to the minimum level needed to function.**

Performance specification:

```
"Meet these performance requirements:

【Screen loading】
- Home screen initial load: under 1 second

【List rendering】
- Smoothly scroll through 100 items
- Use virtual scrolling to render only visible items

【Image handling】
- Compress images on download (max width 800px)
- Cache locally to prevent duplicate downloads

【API calls】
- Use pagination (20 items per page)
- Cache data for 5 minutes, don't refetch identical requests"
```

### Q8: Can I specify app icon and splash screen design?

**A8: Yes. You can request logo/icon generation, though design tools like Figma work better for fine-tuning visuals.**

Visual element specification:

```
"Generate these visual elements:

【App icon】
- Shape: square with rounded corners
- Design: simple calculator symbol (+-÷×)
- Colors: gradient navy to light blue

【Splash screen】
- Background: gradient navy to deep purple
- Center: app logo
- Bottom: app name with 'Loading...'
- Animation: logo fades in over 2 seconds"
```

### Q9: Same prompt generates different apps each time I run it.

**A9: AI generation has inherent randomness. Increase specificity and add constraints for better reproducibility.**

Reproducibility tips:

```
❌ Vague (changes each time):
"Simple shopping list app"

✅ Specific (consistent):
"Shopping list app (follow exactly):
【Features】
1. Home: list with item name, quantity, checkbox
2. Add: '+' button → form → add item
3. Delete: left swipe → confirm → delete
4. Persistence: save to AsyncStorage
【UI】
- Home: list + 'Shopping List' header + '+' button
- Colors: white background, black text
- Font: sans-serif"
```

### Q10: How can I iterate more efficiently on prompts?

**A10: Use iterative prompting. Build features step-by-step rather than trying to perfect everything at once.**

Iterative workflow:

```
1. 【Initial prompt】
   Clearly describe core features (first generation)

2. 【Evaluation】
   Check: Does it launch? Do expected features work?

3. 【Targeted fixes】
   Be specific about problems:
   ❌ "Make it look better"
   ✅ "Make home button larger (50x50px), dark blue background (#1a3a52)"

4. 【Staged expansion】
   After core features work, add one layer at a time:
   - Generation 1: basic CRUD
   - Generation 2: form validation
   - Generation 3: design refinement
   - Generation 4: performance optimization

5. 【Document your best prompt】
   Save final working prompt as reference for future similar apps"
```

## Common Failure Patterns and Solutions

- **Failure**: App won't launch → **Solution**: Specify "Expo SDK 50+" and minimum OS versions
- **Failure**: Wrong features generated → **Solution**: Bullet-point feature list with priorities
- **Failure**: UI broken on some devices → **Solution**: Test on "iPhone 14, 15, and iPad Pro"
- **Failure**: Poor performance → **Solution**: Explicitly request caching, pagination, virtual scrolling
- **Failure**: Security issues → **Solution**: Specify API key handling and data privacy requirements

## Summary

Three principles unlock better Rork AI generations: **specificity, sequencing, and validation**.

1. **Start with core features** → Get a working baseline
2. **Add design requirements** → Polish the appearance
3. **Detail interactions and logic** → Refine behavior and algorithms
4. **Optimize performance and security** → Production-ready

Cycling through these steps repeatedly creates high-quality, professional apps. The difference between a vague prompt and a detailed one is often night and day in terms of the quality of generated app.
