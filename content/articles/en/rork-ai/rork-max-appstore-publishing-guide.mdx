## App Store Publishing in 2 Clicks

Traditional app publishing is complex: certificate setup (30 mins), build config (20 mins), TestFlight registration (10 mins), App Store submission (1 hour) = 2+ hours total.

With Rork Max's auto-deployment, **all of this completes in just minutes**.

## The 2-Step App Store Deployment Process

### Step 1: Project Setup (First Time Only)

In Rork Max editor:

```
[Settings] → [App Store Settings]
  ↓
[Sign In with Apple ID]
  ↓
[Enable Auto Certificate Generation]
```

On first sign-in, Rork automatically generates your developer certificate and saves it to your keychain.

### Step 2: Click Deploy

```
[Build Complete Notification]
  ↓
Click [Deploy] Button
  ↓
Choose Distribution:
  • TestFlight (Beta Testing)
  • App Store (Production)
```

This single step automatically:

1. Performs final code checks
2. Generates build artifacts
3. Uploads to App Store Connect
4. Distributes to TestFlight (if selected)

## TestFlight Beta Testing

### Inviting Testers

In App Store Connect, specify tester email addresses. Rork automatically sends invitations:

```
Tester Email → [Invite] → Auto Email Sent
                  ↓
             TestFlight Acceptance
```

```
├ Testers: 25
```

Review these metrics before proceeding to production.

## App Store Production Release

### Pre-Release Checklist

Rork Max automatically verifies before release:

```
□ Privacy Policy URL configured
□ Screenshots (all 5 languages)
□ App Icon (1024x1024)
□ Age Rating (4+, 12+, 17+)
□ Pricing Tier
```

Missing items trigger automatic deployment pause with clear warnings.

### Release Workflow

**1. Update Version Number**

```
Rork Max → [Settings] → [Version Management]
  Current: 1.0.0
  Next: 1.0.1 (Patch)
    or
  Next: 1.1.0 (Feature)
    or
  Next: 2.0.0 (Major)
```

**2. Write Release Notes**

```

- Bug Fix: Login screen crash
- Performance: List rendering 25% faster
- Feature: Dark mode support
```

**3. Execute Production Deployment**

```
[Deploy] → [App Store Production Release]
  ↓
Auto Build & Upload
  ↓
Auto Submit to App Store Connect
  ↓
Apple Review Begins
```

## Common Questions

### Q: Is Apple Developer Account registration required?

A: Yes. Apple Developer Program membership ($99/year) is required. Rork eliminates certificate management hassle, not Apple's fees.

### Q: How long after TestFlight upload until release?

A: Apple's auto-review averages 1-2 hours. Even accounting for weekends/holidays, usually completes within 24 hours.

### Q: Can I auto-manage version numbers?

A: Yes. Via Rork → [Settings] → [Auto Version Management]:

- Increment only for patch releases
- Increment every deployment
- Manual specification

### Q: Can you auto-generate multi-language screenshots?

A: Partially. Rork auto-generates Japanese and English screenshots. Other languages require manual upload.

### Q: Can I rollback after release?

A: Yes. From App Store Connect, "Cancel release" rolls back to the previous version automatically. (Existing users won't be affected.)

## Auto Failsafe Features

Deployment automatically pauses in these cases:

1. **Crash Rate ≥5%**: 1 in 5 TestFlight testers experiencing crashes
2. **Unresolved Bug Reports**: Critical issues in App Store Connect
3. **Missing Screenshots**: Required languages or screen sizes absent
4. **Guideline Violations**: Configuration doesn't meet Apple requirements

These are safety nets preventing bad releases.

### Rollout Abort Feature

If production crash rate jumps ≥10% within 5 minutes, Rork automatically stops distribution:

```
⚠️ Alert: Crash Rate Spike (10.5%)
   Emergency Rollback to v1.0.0 Executed

   Investigation:
   • Check LogRocket for crash stack traces
   • Run Crash Analyzer for memory leaks
```
