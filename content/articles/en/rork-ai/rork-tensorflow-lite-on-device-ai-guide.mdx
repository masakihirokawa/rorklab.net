## What Is TensorFlow Lite — The Foundation of Mobile ML

TensorFlow Lite (TFLite) is a lightweight inference runtime from TensorFlow, built specifically for mobile and edge devices. By converting a standard TensorFlow model into the `.tflite` format, you can run predictions directly on a smartphone without any server round-trip.

### Cloud AI vs. Edge AI

| Aspect | Cloud AI | Edge AI (TFLite) |
|--------|----------|------------------|
| Network | Required | Not needed (works offline) |
| Latency | 100–500ms | 10–50ms |
| Privacy | Data sent to external servers | All processing stays on device |
| Cost | API usage fees | Free (model bundled with app) |
| Model Size | Unlimited | Optimized to a few MB |

Edge AI is especially valuable for domains where privacy matters — healthcare, finance, education — since user data never leaves the device.

### Initializing TensorFlow.js

Initialize TensorFlow.js at your app's entry point. This only needs to run once at startup.

```typescript
// app/_layout.tsx or App.tsx
import * as tf from '@tensorflow/tfjs';
import { bundleResourceIO } from '@tensorflow/tfjs-react-native';
import { useEffect, useState } from 'react';

export default function RootLayout() {
  const [isTfReady, setIsTfReady] = useState(false);

  useEffect(() => {
    const initTf = async () => {
      // Initialize the TensorFlow.js backend
      await tf.ready();
      console.log('TensorFlow.js ready, backend:', tf.getBackend());
      setIsTfReady(true);
    };
    initTf();
  }, []);

  if (!isTfReady) {
    return <LoadingScreen message="Preparing AI engine..." />;
  }

  return <MainApp />;
}
// Expected output: "TensorFlow.js ready, backend: rn-webgl"
```

## Integrating an Image Classification Model — Adding MobileNet to Your App

### Step 1: Prepare the Pre-trained Model

Download MobileNet V2 from TensorFlow Hub — a lightweight image classification model — and convert it to TFLite format. Pre-converted models are also available.

```typescript
// hooks/useImageClassifier.ts
import * as tf from '@tensorflow/tfjs';
import { bundleResourceIO } from '@tensorflow/tfjs-react-native';

// Import model and label assets
const modelJSON = require('../assets/model/mobilenet_v2/model.json');
const modelWeights = require('../assets/model/mobilenet_v2/weights.bin');

export function useImageClassifier() {
  const [model, setModel] = useState<tf.GraphModel | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    const loadModel = async () => {
      try {
        // Load the bundled model
        const loadedModel = await tf.loadGraphModel(
          bundleResourceIO(modelJSON, modelWeights)
        );
        setModel(loadedModel);
        console.log('MobileNet V2 model loaded successfully');
      } catch (error) {
        console.error('Model loading error:', error);
      } finally {
        setIsLoading(false);
      }
    };
    loadModel();
  }, []);

  return { model, isLoading };
}
// Expected output: "MobileNet V2 model loaded successfully"
```

### Step 2: Preprocess Images and Run Inference

Transform images from the camera or gallery into the input tensor format the model expects, then run inference.

```typescript
// utils/imageClassification.ts
import * as tf from '@tensorflow/tfjs';
import { decodeJpeg } from '@tensorflow/tfjs-react-native';
import * as FileSystem from 'expo-file-system';
import { IMAGENET_LABELS } from '../constants/imagenetLabels';

interface ClassificationResult {
  label: string;
  confidence: number;
}

export async function classifyImage(
  model: tf.GraphModel,
  imageUri: string
): Promise<ClassificationResult[]> {
  // Read the image as binary data
  const imageData = await FileSystem.readAsStringAsync(imageUri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // Convert Base64 → Uint8Array → Tensor
  const rawBytes = tf.util.encodeString(imageData, 'base64');
  const imageTensor = decodeJpeg(new Uint8Array(rawBytes.buffer));

  // Resize to MobileNet V2 input size (224x224)
  const resized = tf.image.resizeBilinear(imageTensor, [224, 224]);

  // Normalize pixel values (0–255 → -1–1)
  const normalized = resized.div(127.5).sub(1.0);

  // Add batch dimension [224,224,3] → [1,224,224,3]
  const batched = normalized.expandDims(0);

  // Run inference
  const predictions = model.predict(batched) as tf.Tensor;
  const probabilities = await predictions.data();

  // Get top 5 results
  const topK = Array.from(probabilities)
    .map((prob, index) => ({ label: IMAGENET_LABELS[index], confidence: prob }))
    .sort((a, b) => b.confidence - a.confidence)
    .slice(0, 5);

  // Free memory (important: prevents memory leaks)
  tf.dispose([imageTensor, resized, normalized, batched, predictions]);

  return topK;
}
// Expected output example:
// [
//   { label: "golden retriever", confidence: 0.89 },
//   { label: "Labrador retriever", confidence: 0.05 },
//   { label: "cocker spaniel", confidence: 0.02 },
//   ...
// ]
```

## Implementing Real-Time Camera Inference

Beyond classifying images from the gallery, let's also implement real-time inference from the camera's live preview.

```typescript
// screens/RealtimeClassifier.tsx
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useRef, useState, useCallback } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useImageClassifier } from '../hooks/useImageClassifier';
import { classifyImage } from '../utils/imageClassification';
import { ClassificationResultView } from '../components/ClassificationResult';

export default function RealtimeClassifier() {
  const cameraRef = useRef<CameraView>(null);
  const [permission, requestPermission] = useCameraPermissions();
  const { model, isLoading } = useImageClassifier();
  const [results, setResults] = useState<{ label: string; confidence: number }[]>([]);
  const [isProcessing, setIsProcessing] = useState(false);

  const handleCapture = useCallback(async () => {
    if (!cameraRef.current || !model || isProcessing) return;

    setIsProcessing(true);
    try {
      // Capture a frame from the camera
      const photo = await cameraRef.current.takePictureAsync({
        quality: 0.5,  // Low resolution is sufficient for inference
        skipProcessing: true,
      });

      if (photo?.uri) {
        const classificationResults = await classifyImage(model, photo.uri);
        setResults(classificationResults);
      }
    } catch (error) {
      console.error('Capture error:', error);
    } finally {
      setIsProcessing(false);
    }
  }, [model, isProcessing]);

  if (!permission?.granted) {
    return (
      <View style={styles.center}>
        <Text>Camera access permission is required</Text>
        <TouchableOpacity style={styles.button} onPress={requestPermission}>
          <Text style={styles.buttonText}>Grant Permission</Text>
        </TouchableOpacity>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <CameraView ref={cameraRef} style={styles.camera} facing="back" />

      <TouchableOpacity
        style={[styles.captureButton, isProcessing && styles.disabled]}
        onPress={handleCapture}
        disabled={isLoading || isProcessing}
      >
        <Text style={styles.captureText}>
          {isProcessing ? 'Recognizing...' : 'Capture & Classify'}
        </Text>
      </TouchableOpacity>

      {results.length > 0 && <ClassificationResultView results={results} />}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1 },
  camera: { flex: 1 },
  center: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  button: { marginTop: 16, paddingHorizontal: 24, paddingVertical: 12, backgroundColor: '#007AFF', borderRadius: 8 },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
  captureButton: { position: 'absolute', bottom: 180, alignSelf: 'center', paddingHorizontal: 32, paddingVertical: 16, backgroundColor: '#007AFF', borderRadius: 24 },
  disabled: { opacity: 0.5 },
  captureText: { color: '#fff', fontSize: 16, fontWeight: '700' },
});
```

## Model Optimization — Improving App Size and Inference Speed

### Quantization for Model Compression

TensorFlow Lite models can be significantly reduced in size through quantization.

```python
# scripts/quantize_model.py
# Run in a Python environment (for model conversion)
import tensorflow as tf

# Load the SavedModel
converter = tf.lite.TFLiteConverter.from_saved_model('saved_model_dir')

# Apply dynamic range quantization (the simplest approach)
converter.optimizations = [tf.lite.Optimize.DEFAULT]

# Run the conversion
tflite_model = converter.convert()

# Save the quantized model
with open('mobilenet_v2_quantized.tflite', 'wb') as f:
    f.write(tflite_model)

print(f'Original model: ~14MB → Quantized: ~{len(tflite_model) / 1024 / 1024:.1f}MB')
# Expected output: "Original model: ~14MB → Quantized: ~3.5MB"
```

### Optimization Best Practices

| Technique | Effect | Trade-off |
|-----------|--------|-----------|
| Dynamic Range Quantization | 75% size reduction | 1–2% accuracy drop |
| Float16 Quantization | 50% size reduction | Best with GPU inference |
| Integer Quantization (INT8) | 75% size reduction + faster inference | Requires calibration data |
| Model Distillation | 2–5x faster inference | Needs a teacher model |

The right choice depends on your use case. For real-time camera inference, INT8 quantization is highly effective. For medical or precision-critical applications, float16 tends to be the safer bet.

## Designing for Offline Operation

The biggest advantage of edge AI is that it works without a network connection. Combining these patterns lets you build robust offline AI apps.

```typescript
// hooks/useOfflineAI.ts
import NetInfo from '@react-native-community/netinfo';
import { useState, useEffect } from 'react';

type AIMode = 'edge' | 'cloud' | 'hybrid';

export function useOfflineAI() {
  const [isConnected, setIsConnected] = useState(true);
  const [aiMode, setAiMode] = useState<AIMode>('hybrid');

  useEffect(() => {
    const unsubscribe = NetInfo.addEventListener(state => {
      setIsConnected(state.isConnected ?? false);

      // Automatically switch AI mode based on connectivity
      if (!state.isConnected) {
        setAiMode('edge');     // Offline → edge AI only
      } else if (state.type === 'wifi') {
        setAiMode('cloud');    // Wi-Fi → cloud AI (higher accuracy)
      } else {
        setAiMode('hybrid');   // Mobile data → hybrid approach
      }
    });

    return () => unsubscribe();
  }, []);

  return { isConnected, aiMode };
}
// Usage:
// const { aiMode } = useOfflineAI();
// if (aiMode === 'edge') → run inference with TFLite
// if (aiMode === 'cloud') → call a cloud API
// if (aiMode === 'hybrid') → fast edge inference + cloud refinement
```

By combining a high-accuracy server model with a lightweight on-device model, you can maximize both user experience and prediction quality. ## Frequently Asked Questions (FAQ)

### Q1: How much does a TensorFlow Lite model add to app size?

The quantized MobileNet V2 model is roughly 3.5 MB — a modest increase relative to overall app size. If you're bundling multiple models, consider using On-Demand Resources (iOS) or Play Feature Delivery (Android) to download models only when needed, keeping the initial download small.

### Q2: Should I use Core ML or TensorFlow Lite?

If you're targeting iOS only, Core ML provides superior performance by leveraging Apple's Neural Engine hardware. For cross-platform apps targeting both iOS and Android, TensorFlow Lite is the better fit. Since Rork generates React Native-based apps, TensorFlow Lite tends to be easier to integrate. For a detailed look at Core ML, see [Rork Max × Core ML: The Complete Guide to Building On-Device AI Apps](/articles/rork-dev/rork-max-core-ml-guide).

### Q3: What frame rate can I expect for real-time inference?

With a quantized MobileNet V2 model, you can achieve approximately 30 FPS on iPhone 14 and later, and 15–20 FPS on mid-range Android devices. Enabling the GPU delegate can push speeds even higher. Note that React Native's bridge overhead introduces about a 20–30% latency penalty compared to pure native implementations.

### Q4: Can I use custom models (my own classifiers)?

Absolutely. Any model trained with TensorFlow or Keras can be converted to TFLite format. Using transfer learning, you can fine-tune MobileNet V2 with as few as a few hundred images to create custom classifiers for food recognition, plant identification, product cataloging, and more.

### Q5: Does on-device inference drain the battery?

Battery consumption per inference is negligible — roughly 0.001% per image classification. However, continuous real-time camera inference over extended periods does add up. Best practices include throttling inference intervals (e.g., every 500ms instead of every frame) and pausing inference when the app moves to the background.
