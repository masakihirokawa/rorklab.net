A designer perfects a UI in Figma. A developer rewrites the code from scratch. Information is lost. Implementation costs explode. The original vision fades.

**Rork Max × Figma integration** solves this fundamental problem. It automatically generates **production-quality app code** directly from Figma files in seconds, completely eliminating the gap between design and implementation.

This guide walks you through the complete design-to-code workflow using Rork Max and Figma—from setup and component structure, through code generation and verification, to team operations and ongoing maintenance.

## Understanding Rork Max × Figma Integration

### What Rork Max Extracts From Figma

When Rork Max reads a Figma file, it automatically detects and converts four critical design elements:

1. **Design Tokens**
   - Color palettes (global color variables)
   - Typography (font name, size, weight, line height)
   - Spacing and layout scales
   - Shadows, corner radius, borders

2. **Component Structure**
   - Figma components → Structured SwiftUI/React Views
   - Component variants → Conditional logic and state management
   - Instances → Automatically inferred props

3. **Layout Information**
   - Auto Layout settings → HStack/VStack/Spacer
   - Grid constraints → frame, padding, offset
   - Responsive behavior → Width-based branching

4. **Visual Properties**
   - Background colors, borders, gradients
   - Opacity, blend modes
   - Graphics (icons, images) → Asset references

### The Data Flow: Figma to Production Code

```
┌─────────────────┐
│  Figma File     │
│  (Complete UI)  │
└────────┬────────┘
         │
         ↓ Export
┌─────────────────────────────────┐
│ Figma API / Plugin              │
│ - Extract design tokens         │
│ - Analyze component structure   │
│ - Generate metadata             │
└────────┬────────────────────────┘
         │
         ↓ JSON/XML Metadata
┌─────────────────────────────────┐
│ Rork Max AI Engine              │
│ - Parse structured data         │
│ - Generate SwiftUI/React Native │
│ - Apply design tokens           │
└────────┬────────────────────────┘
         │
         ↓
┌─────────────────────────────────┐
│ Production-Ready App Code       │
│ (iOS / Android / Web)           │
└─────────────────────────────────┘
```

### Using Variants Effectively

Figma **Variants** dramatically improve Rork Max conversion accuracy:

```
Button Component:
┌─ Variants
│  ├─ Style: Primary | Secondary | Tertiary
│  ├─ Size: Small | Medium | Large
│  ├─ State: Default | Hover | Active | Disabled
│  └─ Icon: Yes | No
```

Auto-generated SwiftUI code:

```swift
struct Button: View {
    enum Style {
        case primary
        case secondary
        case tertiary
    }

    enum Size {
        case small
        case medium
        case large

        var fontSize: CGFloat {
            switch self {
            case .small: return 12
            case .medium: return 14
            case .large: return 16
            }
        }

        var padding: EdgeInsets {
            switch self {
            case .small:
                return EdgeInsets(top: 6, leading: 12, bottom: 6, trailing: 12)
            case .medium:
                return EdgeInsets(top: 10, leading: 16, bottom: 10, trailing: 16)
            case .large:
                return EdgeInsets(top: 14, leading: 20, bottom: 14, trailing: 20)
            }
        }
    }

    let title: String
    let style: Style
    let size: Size
    let isDisabled: Bool
    let icon: String?
    let action: () -> Void

    @State private var isPressed = false

    var body: some View {
        Button(action: action) {
            HStack(spacing: 8) {
                if let icon = icon {
                    Image(systemName: icon)
                        .font(.system(size: size.fontSize))
                }

                Text(title)
                    .font(.system(size: size.fontSize, weight: .semibold))
            }
            .padding(size.padding)
            .foregroundColor(foregroundColor)
            .background(backgroundColor)
            .cornerRadius(8)
            .opacity(isPressed || isDisabled ? 0.8 : 1.0)
        }
        .disabled(isDisabled)
    }

    private var backgroundColor: Color {
        if isDisabled {
            return ColorPalette.Gray.c100
        }

        switch style {
        case .primary:
            return ColorPalette.Primary.c600
        case .secondary:
            return ColorPalette.Secondary.c600
        case .tertiary:
            return Color.clear
        }
    }

    private var foregroundColor: Color {
        if isDisabled {
            return ColorPalette.Gray.c400
        }

        switch style {
        case .primary, .secondary:
            return .white
        case .tertiary:
            return ColorPalette.Primary.c600
        }
    }
}
```

## Step 5: Responsive Design and Animations

### Multi-Device Responsive Generation

Rork Max recognizes Figma **Responsive Design** settings and auto-generates SwiftUI responsive code:

```swift
struct ResponsiveLayout: View {
    @Environment(\.horizontalSizeClass) var sizeClass
    @Environment(\.verticalSizeClass) var vSize

    var body: some View {
        Group {
            if sizeClass == .compact && vSize == .regular {
                // Mobile portrait
                mobilePortraitLayout
            } else if sizeClass == .compact && vSize == .compact {
                // Mobile landscape
                mobileLandscapeLayout
            } else if sizeClass == .regular {
                // Tablet / iPad
                tabletLayout
            }
        }
    }

    var mobilePortraitLayout: some View {
        VStack(spacing: 16) {
            HeaderView()
            ContentView()
            FooterView()
        }
    }

    var tabletLayout: some View {
        HStack(spacing: 24) {
            SidebarView()
                .frame(width: 280)

            ContentView()
        }
    }
}
```

### Figma Animations → SwiftUI Animations

Figma-specified transitions automatically convert to SwiftUI animations:

```swift
struct AnimatedProductCard: View {
    @State private var isExpanded = false

    var body: some View {
        VStack {
            // Figma: "Ease-in-out, 300ms"

            Text("Product Details")
                .font(.headline)
                .frame(maxWidth: .infinity)
                .padding(16)
                .background(ColorPalette.Primary.c600)

                    withAnimation(.easeInOut(duration: 0.3)) {

                    Text("Description")
                    Text("Price: $50")
                    Button("Add to Cart") {}
                }
                .padding(16)
                .transition(.opacity.combined(with: .move(edge: .bottom)))
            }
        }
        .background(Color.white)
        .cornerRadius(12)
    }
}

## Step 6: Optimizing Team Collaboration

### Efficient Designer-Developer Workflow Patterns

#### Pattern A: Sprint Start (New Feature)

```
1. Designer: Complete Figma design
   (All components, variants, screens)

2. Designer: Final Design System review
   - Color tokens exist?
   - Typography tokens defined?
   - Spacing scale configured?
   - Component naming follows convention?

3. Developer: Run "Rork Max → Export to Code"

4. Developer: Review & merge generated code
   (Usually < 30 minutes)

5. Developer: Add API connections & logic
   (Generated Views already production-quality)

6. QA: Verify design-to-implementation match
```

#### Pattern B: Design Updates (Existing Feature)

```
Designer: Changes Button color in Figma
  ↓
Rork Max plugin: Detects change
  ↓
Developer: Reviews diff (typically 5-line change)
  ↓
PR → Merge

Time to completion: < 5 minutes
```

struct GeneratedButton: View {
    let title: String

// Developer extensions
// → Button+Extensions.swift
extension GeneratedButton {
    @State private var isLoading = false

    var bodyWithLoading: some View {
        Button(action: {
            isLoading = true
            action()
            DispatchQueue.main.asyncAfter(deadline: .now() + 2.0) {
                isLoading = false
            }
        }) {
            HStack {
                if isLoading {
                    ProgressView()
                        .progressViewStyle(.circular)
                } else {
                    Text(title)
                }
            }
            .frame(maxWidth: .infinity)
            .padding(12)
            .background(isLoading ? ColorPalette.Gray.c300 : ColorPalette.Primary.c600)
            .foregroundColor(.white)
            .cornerRadius(8)
        }
        .disabled(isLoading)
    }
}
```
