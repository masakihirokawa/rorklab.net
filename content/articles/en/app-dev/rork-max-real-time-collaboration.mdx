## How Real-Time Features Transform Apps

Notion, Figma, Google Docs — today's leading productivity apps all share one thing: real-time collaboration. Multiple users can edit the same document simultaneously, seeing each other's cursors and changes as they happen. This experience dramatically boosts user engagement and retention.

With Rork Max, you can implement these real-time features in mobile apps with surprisingly little code. This article walks through real-time collaboration using Supabase Realtime as the backend, covering three concrete use cases.

---

## Architecture Overview

Here's the system we'll build:

```
[User A's Rork App]
      ↕ WebSocket
[Supabase Realtime Server]
      ↕ WebSocket
[User B's Rork App]
```

Supabase Realtime delivers PostgreSQL changes to clients in real time through three mechanisms: Broadcast, Presence, and PostgreSQL Changes. By prompting Rork Max with the right instructions, you can avoid writing complex WebSocket implementations from scratch.

## Setup: Preparing Your Supabase Project

Create a Supabase project and set up the following table:

```sql
-- Documents table
CREATE TABLE documents (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  title text NOT NULL,
  content text DEFAULT '',
  updated_at timestamptz DEFAULT now(),
  updated_by text
);

-- Enable realtime
ALTER PUBLICATION supabase_realtime ADD TABLE documents;

-- Row Level Security
ALTER TABLE documents ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public access" ON documents
  FOR ALL USING (true) WITH CHECK (true);
```

Then create a new Rork Max project and configure:

```
Supabase URL: [your-project].supabase.co
Supabase Anon Key: [your-anon-key]
```

## Use Case 1: Real-Time Chat

Start with the simplest real-time feature. Prompt for Rork Max:

```
Build a real-time chat screen using Supabase Realtime Broadcast channels,
with room IDs as the channel identifier.

Requirements:
- Messages saved to a Supabase messages table
- New messages instantly broadcast to all room members
- My messages on the right, others on the left
- Display username and timestamp
- Show typing indicators
```

The core logic Rork Max generates:

```typescript
import { createClient } from '@supabase/supabase-js';
import { useEffect, useState, useRef } from 'react';

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

interface Message {
  id: string;
  userId: string;
  userName: string;
  text: string;
  timestamp: string;
}

export function useRealTimeChat(roomId: string, userId: string) {
  const [messages, setMessages] = useState<Message[]>([]);
  const [typingUsers, setTypingUsers] = useState<string[]>([]);
  const channelRef = useRef<ReturnType<typeof supabase.channel> | null>(null);

  useEffect(() => {
    const channel = supabase.channel(`room:${roomId}`, {
      config: { broadcast: { self: false } },
    });

    channelRef.current = channel;

    // Receive messages
    channel.on('broadcast', { event: 'message' }, ({ payload }) => {
      setMessages(prev => [...prev, payload as Message]);
    });

    // Typing indicators
    channel.on('broadcast', { event: 'typing' }, ({ payload }) => {
      const { userName, isTyping } = payload;
      setTypingUsers(prev =>
        isTyping
          ? [...prev.filter(u => u !== userName), userName]
          : prev.filter(u => u !== userName)
      );
    });

    channel.subscribe();

    // Load existing messages
    supabase
      .from('messages')
      .select('*')
      .eq('room_id', roomId)
      .order('timestamp', { ascending: true })
      .then(({ data }) => {
        if (data) setMessages(data);
      });

    return () => {
      channel.unsubscribe();
    };
  }, [roomId]);

  const sendMessage = async (text: string, userName: string) => {
    const message: Message = {
      id: crypto.randomUUID(),
      userId,
      userName,
      text,
      timestamp: new Date().toISOString(),
    };

    // Save to DB + broadcast
    await supabase.from('messages').insert({ ...message, room_id: roomId });
    await channelRef.current?.send({
      type: 'broadcast',
      event: 'message',
      payload: message,
    });
  };

  const sendTyping = async (userName: string, isTyping: boolean) => {
    await channelRef.current?.send({
      type: 'broadcast',
      event: 'typing',
      payload: { userName, isTyping },
    });
  };

  return { messages, typingUsers, sendMessage, sendTyping };
}
```

## Use Case 2: Presence (Who's Online)

Show which users are currently viewing the same screen in real time:

```
Use Supabase Realtime Presence to display avatar icons of all users
currently viewing the same document in the header.
Users should automatically disappear from the list when they leave the app.
```

```typescript
export function usePresence(documentId: string, currentUser: User) {
  const [onlineUsers, setOnlineUsers] = useState<User[]>([]);

  useEffect(() => {
    const channel = supabase.channel(`presence:${documentId}`);

    channel
      .on('presence', { event: 'sync' }, () => {
        const state = channel.presenceState<User>();
        const users = Object.values(state).flat();
        setOnlineUsers(users);
      })
      .subscribe(async (status) => {
        if (status === 'SUBSCRIBED') {
          await channel.track({
            id: currentUser.id,
            name: currentUser.name,
            avatar: currentUser.avatar,
            joinedAt: new Date().toISOString(),
          });
        }
      });

    return () => {
      channel.unsubscribe();
    };
  }, [documentId, currentUser.id]);

  return { onlineUsers };
}
```

## Use Case 3: Real-Time Document Sync

Use PostgreSQL Changes to broadcast database updates to all connected clients:

```
Build a document editor that broadcasts changes from the documents table
to all clients in real time using Supabase Realtime PostgreSQL Changes.
Use debouncing to prevent excessive database writes while typing.
```

```typescript
export function useRealtimeDocument(documentId: string) {
  const [content, setContent] = useState('');
  const [lastEditor, setLastEditor] = useState<string | null>(null);
  const debounceRef = useRef<NodeJS.Timeout>();

  useEffect(() => {
    // Fetch initial data
    supabase
      .from('documents')
      .select('content, updated_by')
      .eq('id', documentId)
      .single()
      .then(({ data }) => {
        if (data) {
          setContent(data.content);
          setLastEditor(data.updated_by);
        }
      });

    // Subscribe to real-time changes
    const channel = supabase
      .channel(`document:${documentId}`)
      .on(
        'postgres_changes',
        {
          event: 'UPDATE',
          schema: 'public',
          table: 'documents',
          filter: `id=eq.${documentId}`,
        },
        (payload) => {
          setContent(payload.new.content);
          setLastEditor(payload.new.updated_by);
        }
      )
      .subscribe();

    return () => {
      channel.unsubscribe();
    };
  }, [documentId]);

  const updateContent = useCallback((newContent: string, userId: string) => {
    setContent(newContent); // Optimistic local update

    // Debounce: write to DB after 500ms
    clearTimeout(debounceRef.current);
    debounceRef.current = setTimeout(async () => {
      await supabase
        .from('documents')
        .update({
          content: newContent,
          updated_by: userId,
          updated_at: new Date().toISOString(),
        })
        .eq('id', documentId);
    }, 500);
  }, [documentId]);

  return { content, lastEditor, updateContent };
}
```

## Conflict Resolution: Optimistic Updates and LWW

When multiple users edit simultaneously, conflicts can occur. Prompt Rork Max to handle this:

```
If two users edit the document at the same time, use Last Write Wins (LWW)
conflict resolution. Show a lock indicator on fields being edited by another
user so editors know when someone else is working there.
```

Rork Max will auto-generate the lock management logic and visual feedback. For straightforward use cases that don't require operational transformation (OT) or CRDTs, LWW is sufficient and much simpler to implement.

## Performance Optimization Tips

Real-time features can put significant load on your network. Ask Rork Max to include these optimizations:

**1. Batch message processing**
Debounce frequent broadcasts by 100ms to reduce network calls

**2. Proper channel lifecycle management**
Always call `unsubscribe()` when navigating away from a screen

**3. Use filtering**
Use the `filter` parameter in `postgres_changes` to avoid subscribing to irrelevant rows

**4. Manage connection counts**
Supabase's free plan has concurrent connection limits — keep channel counts minimal
