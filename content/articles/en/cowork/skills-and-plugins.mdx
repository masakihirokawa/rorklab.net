## What Are Skills?

Claude Code and Cowork come packed with many features out of the box, but by adding "skills," you can extend them to handle even more specialized tasks. A skill is a set of instructions written in a markdown file called SKILL.md that defines how Claude should behave in specific situations.

For example, you can create skills like "prioritize security and readability during code review" or "run tests before pushing to production during deployment" — custom instructions tailored to your team or personal workflow, saved and reusable across sessions.

Skills follow the [Agent Skills](https://agentskills.io) open standard, making them compatible with AI tools beyond Claude Code.

## Skill Structure

A skill is simply a directory containing a SKILL.md file.

```
my-skill/
├── SKILL.md           # Main instruction file (required)
├── reference.md       # Detailed reference (optional)
├── examples/
│   └── sample.md      # Output samples (optional)
└── scripts/
    └── helper.sh      # Executable scripts (optional)
```

The SKILL.md file consists of YAML frontmatter and markdown instructions.

```yaml
---
name: code-review
description: Review code for quality and security. Use during PR reviews.
---

When reviewing code, check the following:

1. **Code structure**: Are function and module responsibilities clear?
2. **Error handling**: Is exception handling appropriate?
3. **Security**: Are there SQL injection or XSS vulnerabilities?
4. **Test coverage**: Are tests written for the changes?
```

## Key Frontmatter Fields

The YAML frontmatter at the top of SKILL.md lets you fine-tune skill behavior.

`name` is the display name and becomes the `/name` slash command. `description` helps Claude decide whether to automatically invoke the skill based on context. Setting `disable-model-invocation` to `true` means the skill only runs when explicitly called with `/name` — ideal for workflows with side effects like deployments or commits.

`allowed-tools` specifies which tools can be used without permission during skill execution, and setting `context` to `fork` runs the skill as an isolated subagent.

## Where to Save Skills

Where you place a skill determines its availability.

Personal skills go in `~/.claude/skills/<skill-name>/SKILL.md` and are available across all projects. Project skills go in `.claude/skills/<skill-name>/SKILL.md` and work only within that project. Plugin skills live in `<plugin>/skills/<skill-name>/SKILL.md` and are available wherever the plugin is enabled.

Enterprise-wide distribution is possible through managed settings.

## How to Invoke Skills

There are two ways to invoke a skill.

For manual invocation, use a slash command:

```
/code-review src/auth/login.ts
```

For automatic invocation, Claude evaluates the conversation context and loads the appropriate skill automatically. For example, if you ask "please review this code," a skill whose `description` mentions "code review" or "PR review" activates automatically.

You can pass arguments using the `$ARGUMENTS` placeholder:

```yaml
---
name: fix-issue
description: Fix a GitHub issue
disable-model-invocation: true
---

Fix GitHub Issue $ARGUMENTS by following these steps:

1. Read the issue description
2. Implement the fix
3. Write tests
4. Create a commit
```

Running `/fix-issue 123` replaces `$ARGUMENTS` with `123`.

## Practical Examples

### Example 1: Enforcing Commit Message Conventions

```yaml
---
name: commit
description: Create commit messages in Conventional Commits format
disable-model-invocation: true
---

Analyze the changes and create a commit using Conventional Commits format.

Format: <type>(<scope>): <description>

Choose type from:
- feat: New feature
- fix: Bug fix
- docs: Documentation changes
- refactor: Refactoring
- test: Adding or modifying tests
- chore: Build or tool changes
```

### Example 2: PR Review with Dynamic Context

```yaml
---
name: pr-summary
description: Summarize pull request changes
context: fork
agent: Explore
---

## PR Information
- Diff: !`gh pr diff`
- Comments: !`gh pr view --comments`
- Changed files: !`gh pr diff --name-only`

## Task
Provide a concise summary of this pull request.
```

The `!`command`` syntax executes shell commands before the skill content is sent to Claude, injecting the results directly into the prompt.

## What Are Plugins?

Plugins package skills, agents, hooks, and MCP servers into a single distributable extension. If you only need skills for personal use, placing them in `.claude/skills/` is sufficient, but plugins are the way to go when sharing with teams or distributing to the community.

The basic plugin structure looks like this:

```
my-plugin/
├── .claude-plugin/
│   └── plugin.json       # Plugin metadata
├── skills/
│   └── review/
│       └── SKILL.md
├── agents/
│   └── reviewer.md
├── hooks/
│   └── hooks.json
├── .mcp.json             # MCP server config
└── .lsp.json             # LSP server config
```

The `plugin.json` contains metadata like name, description, and version.

```json
{
  "name": "my-team-tools",
  "description": "A set of skills and hooks for team development",
  "version": "1.0.0",
  "author": {
    "name": "Your Team"
  }
}
```

## Installing and Managing Plugins

Install plugins from the CLI:

```bash
claude plugin install <plugin-name>@<marketplace>
```

Control installation scope with `--scope user` (available across all projects, default), `--scope project` (shared with team via `.claude/settings.json`), or `--scope local` (project-local, gitignored).

Disabling, enabling, and uninstalling are also straightforward:

```bash
claude plugin disable <plugin-name>
claude plugin enable <plugin-name>
claude plugin uninstall <plugin-name>
```

## Plugin Marketplaces and skills.sh

Plugins are distributed through "marketplaces" — Git repositories or URL endpoints that catalog plugins.

The official marketplace is maintained by Anthropic and can be browsed from the Discover tab in Claude Code's `/plugin` menu. You can submit your own plugin at [claude.ai/settings/plugins/submit](https://claude.ai/settings/plugins/submit).

### skills.sh — The Community Skills Hub

[skills.sh](https://skills.sh) is a community hub where Claude Code skills and plugins are published. You can browse and search user-created skills, and install them easily through GitHub integration.

Over 100 production-ready skills covering engineering, marketing, compliance, and more are available. Simply register a marketplace and install what you need.

### Team Marketplaces

For sharing proprietary plugins within your team, you can create a marketplace as a GitHub repository. Write a `marketplace.json` catalog, and team members can register the repository URL to install plugins from the `/plugin` menu.

## MCP (Model Context Protocol) Integration

MCP is an open protocol for connecting AI tools to external data sources and services. Claude Code uses MCP to integrate with hundreds of services including Google Drive, Slack, Jira, GitHub, and databases.

### Configuring MCP Servers

Add MCP servers using the `claude mcp add` command:

```bash
# Add GitHub MCP server
claude mcp add github -- npx -y @modelcontextprotocol/server-github

# Add Figma MCP server
claude mcp add figma -- npx -y figma-developer-mcp --figma-api-key=YOUR_KEY

# Connect to PostgreSQL database
claude mcp add postgres -- npx -y @modelcontextprotocol/server-postgres postgresql://localhost/mydb
```

You can also use a `.mcp.json` configuration file for project-shared settings:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}
```

Environment variables like `${GITHUB_TOKEN}` keep API keys out of the file, making it safe to share configurations across teams.

### MCP Use Cases

With GitHub MCP configured, you can simply say "Fix Issue #42 and create a PR," and Claude will read the issue, implement the fix, and create a pull request automatically.

Figma MCP lets you pull component and style information directly from design files, enabling code generation based on actual designs. This dramatically streamlines the design-to-code workflow.

Google's [Stitch](https://stitch.ai) MCP enables generating UI designs from text prompts and using them directly in Claude Code. You can browse projects, view screen designs, and extract design tokens — all from within your chat interface.

For mobile app development, [Rork](https://rork.com) integration is also noteworthy. Rork Max uses Claude Code and Opus 4.6 as its backend to auto-generate native Swift apps from just an app description. Everything from UI to navigation, data persistence, and Apple API integration is handled by AI.

Slack MCP allows you to analyze messages in specific channels or automatically notify about Claude's work results.

### Combining Plugins and MCP

Plugins can include `.mcp.json` files, allowing you to distribute skills and MCP server configurations as a package. For example, you could create a "Figma Design Review" plugin that bundles a Figma MCP server config with review skills, and distribute it to your entire team.

## Skills in Cowork

Skills play an important role in the Cowork environment too. Cowork includes built-in skills for PDF processing, Excel creation, PowerPoint generation, and more — automatically loaded by Claude at the appropriate time.

For example, when you ask "create an Excel file," the xlsx skill activates automatically, handling library installation and formatting to produce high-quality spreadsheets.

You can add custom skills to Cowork as well. Create a `.claude/skills/` directory in your working folder with SKILL.md files, and they'll be available in your Cowork sessions.

## Bundled Skills

Claude Code comes with several pre-installed bundled skills.

`/simplify` reviews recently changed files for code reuse, quality, and efficiency, then applies improvements. It runs three review agents in parallel and aggregates results, making it far more efficient than manual review.

`/batch` orchestrates large-scale changes across the entire codebase in parallel. Provide a description of the change, and it analyzes the codebase, breaks it into tasks, and runs each in an independent git worktree. For example: `/batch migrate src/ from Solid to React`.

`/debug` analyzes Claude Code session debug logs to help troubleshoot issues.

## Summary

Claude Code's skill and plugin system is a powerful toolkit for extending AI capabilities to match your needs. Writing a single SKILL.md file is all it takes to customize Claude's behavior.

Start with personal skills, share them with your team when they prove useful, and publish as plugins when they could benefit the wider community. If you find yourself giving Claude the same instructions repeatedly, that's a great candidate for turning into a skill.
