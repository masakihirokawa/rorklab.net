## What Is Prompt Caching?

Claude API's **prompt caching** reuses the same context across API requests to significantly reduce input token costs. When you repeatedly send requests with identical system prompts or documents, subsequent requests receive substantial discounts.

## When to Use It

Prompt caching is especially effective in these patterns:

- Including long system prompts in every request
- Asking multiple questions while referencing a large document
- Reusing the same few-shot examples
- Sending the same context frequently in RAG applications

## Basic Usage

Specify `cache_control` on the parts you want to cache.

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "You are a legal assistant. Answer questions based on this contract.\n\n[30,000 character contract text]",
            "cache_control": {"type": "ephemeral"}
        }
    ],
    messages=[
        {"role": "user", "content": "Summarize the termination conditions"}
    ]
)
```

### cache_control Types

- **ephemeral**: Cache persists for a minimum of 5 minutes (TTL resets on each use)

## Cost Structure

Three pricing tiers apply based on cache state:

1. **Cache write**: 25% surcharge on normal input token price (first time only)
2. **Cache read**: **90% discount** on normal input token price
3. **Normal input**: Standard pricing for non-cached portions

This means the same context costs 1/10th from the second request onward.

## Checking Cache in Responses

Monitor cache effectiveness via the `usage` field in responses.

```json
{
  "usage": {
    "input_tokens": 50,
    "output_tokens": 200,
    "cache_creation_input_tokens": 15000,
    "cache_read_input_tokens": 0
  }
}
```

Second request:

```json
{
  "usage": {
    "input_tokens": 50,
    "output_tokens": 180,
    "cache_creation_input_tokens": 0,
    "cache_read_input_tokens": 15000
  }
}
```

## Design Best Practices

### Place Cacheable Content First

Put static content at the beginning of your message array and variable content at the end.

### Ensure Sufficient Length

Cached content must be at least 1,024 tokens (2,048 for Claude Haiku). Shorter content won't create a cache.

### Design Cache Breakpoints

You can specify `cache_control` in up to 4 places. Set them at reusable boundaries: system prompts, documents, few-shot examples.

## Practical Example: Chatbot

```python
# Cache long system prompt + documentation
system_message = {
    "type": "text",
    "text": system_prompt + "\n\n" + product_documentation,
    "cache_control": {"type": "ephemeral"}
}

# User conversation (variable part)
messages = conversation_history + [
    {"role": "user", "content": user_question}
]
```

With this design, tens of thousands of system prompt tokens are processed at 1/10th cost from the second request onward.

## Summary

Prompt caching delivers significant cost savings, especially for applications with long contexts. Since it only requires thoughtful system prompt design, every API developer should implement this feature.
