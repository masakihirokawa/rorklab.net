## What Is Batch Processing?

The Claude API's **Message Batches** feature lets you submit large numbers of requests in bulk for asynchronous processing. Compared to standard API calls, it offers up to 50% cost savings with results delivered within 24 hours.

## Use Cases

### Data Analysis

Sentiment analysis, text classification, and summarization across thousands of customer reviews.

### Content Generation

Bulk generation of product descriptions, metadata, and translated content.

### Data Transformation

Batch conversion of unstructured text into JSON or spreadsheet formats.

## Basic Usage

### Creating a Batch

Example using the Python SDK:

```python
import anthropic

client = anthropic.Anthropic()

# Create a batch request
batch = client.messages.batches.create(
    requests=[
        {
            "custom_id": "review-001",
            "params": {
                "model": "claude-sonnet-4-6",
                "max_tokens": 1024,
                "messages": [
                    {"role": "user", "content": "Analyze the sentiment of this review: Great product!"}
                ]
            }
        },
        {
            "custom_id": "review-002",
            "params": {
                "model": "claude-sonnet-4-6",
                "max_tokens": 1024,
                "messages": [
                    {"role": "user", "content": "Analyze the sentiment of this review: Very disappointing."}
                ]
            }
        }
    ]
)

print(f"Batch ID: {batch.id}")
```

### Checking Batch Status

```python
# Check processing status
batch_status = client.messages.batches.retrieve(batch.id)
print(f"Status: {batch_status.processing_status}")
print(f"Succeeded: {batch_status.request_counts.succeeded}")
print(f"Errored: {batch_status.request_counts.errored}")
```

### Retrieving Results

```python
# Get results after processing completes
for result in client.messages.batches.results(batch.id):
    print(f"ID: {result.custom_id}")
    if result.result.type == "succeeded":
        print(f"Response: {result.result.message.content[0].text}")
    else:
        print(f"Error: {result.result.error}")
```

## Design Considerations

### Using custom_id Effectively

Assign a unique `custom_id` to each request to easily map results back to your original data. Use file names, database IDs, or row numbers.

### Error Handling

Individual requests within a batch may fail. Always check the success or failure status of each request when processing results.

### Optimizing Batch Size

Each batch can contain up to 10,000 requests. Split larger datasets across multiple batches.

## Cost Optimization Tips

1. **Choose the right model**: Use Haiku for simple classification tasks
2. **Set max_tokens appropriately**: Keep it as low as needed to reduce costs
3. **Always use batches**: When real-time responses are not required
4. **Optimize prompts**: Write concise prompts to minimize input token usage

## Summary

Message Batches is essential for large-scale data processing. With better cost efficiency and relaxed rate limits, it is ideal for periodic data analysis and content generation pipelines.
