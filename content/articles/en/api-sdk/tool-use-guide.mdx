## What is Tool Use?

Tool use, also known as function calling, allows Claude to request the execution of external tools and functions. Rather than generating text about what it would do, Claude can directly invoke APIs, call functions, and interact with external systems.

<Callout type="info">
Tool use enables Claude to:
- Query databases and APIs
- Read and write files
- Perform calculations
- Execute custom business logic
- Integrate with external services
</Callout>

## Basic Tool Definition

Tools are defined in your API request using the `tools` parameter. Each tool has a name, description, and input schema.

### Tool Schema Structure

```json
{
  "name": "get_weather",
  "description": "Get the current weather for a location",
  "input_schema": {
    "type": "object",
    "properties": {
      "location": {
        "type": "string",
        "description": "The city and state/country"
      },
      "unit": {
        "type": "string",
        "enum": ["celsius", "fahrenheit"],
        "description": "Temperature unit"
      }
    },
    "required": ["location"]
  }
}
```

### Tool Definition Best Practices

- **Clear names**: Use descriptive tool names (e.g., `query_customer_db` not `func1`)
- **Detailed descriptions**: Explain what the tool does and when to use it
- **Complete schema**: Define all parameters and their constraints
- **Specify required fields**: Mark parameters that must be provided
- **Add examples**: Include enum values or constraints in descriptions

### Make a Request with Tools

```python
messages = [
    {
        "role": "user",
        "content": "Get the user info for user123 and send them a welcome email"
    }
]

    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages

```

## Handling Tool Results

When Claude decides to use a tool, it returns a `tool_use` content block with the tool name and input parameters. You must then execute the tool and return the result.

### Processing Tool Use Responses

```python
# Execute the request
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=messages
)

# Check if Claude wants to use a tool
if response.stop_reason == "tool_use":
    # Find the tool use block
    tool_use_block = next(
        block for block in response.content
        if block.type == "tool_use"
    )

    tool_name = tool_use_block.name
    tool_input = tool_use_block.input

    print(f"Claude wants to use: {tool_name}")
    print(f"With input: {tool_input}")
```

### Implementing Tool Handlers

```python
def execute_tool(tool_name: str, tool_input: dict):
    """Execute a tool and return the result"""

    if tool_name == "get_user_info":
        return get_user_info(tool_input["user_id"])
    elif tool_name == "send_email":
        return send_email(
            tool_input["to"],
            tool_input["subject"],
            tool_input["body"]
        )
    else:
        return {"error": f"Unknown tool: {tool_name}"}

def get_user_info(user_id: str):
    """Retrieve user information from database"""
    # Simulate database call
    return {
        "user_id": user_id,
        "name": "John Doe",
        "email": "john@example.com"
    }

def send_email(to: str, subject: str, body: str):
    """Send an email"""
    # Simulate email sending
    return {
        "success": True,
        "message": f"Email sent to {to}"
    }
```

## Multi-Turn Conversations with Tool Use

Tool use typically requires multiple turns: Claude requests a tool, you execute it, and Claude continues based on the result.

### Complete Multi-Turn Example

```python
def process_tool_use(user_message: str):
    messages = [
        {"role": "user", "content": user_message}
    ]

    while True:
        # Get response from Claude
        response = client.messages.create(
            model="claude-opus-4-6",
            max_tokens=1024,
            tools=tools,
            messages=messages
        )

        # If Claude generated text, return it
        if response.stop_reason == "end_turn":
            text_blocks = [
                block.text for block in response.content
                if hasattr(block, "text")
            ]
            return "\n".join(text_blocks)

        # If Claude wants to use a tool
        if response.stop_reason == "tool_use":
            # Find tool use block
            tool_use_block = next(
                block for block in response.content
                if block.type == "tool_use"
            )

            # Execute the tool
            tool_result = execute_tool(
                tool_use_block.name,
                tool_use_block.input
            )

            # Add Claude's response and tool result to messages
            messages.append({"role": "assistant", "content": response.content})
            messages.append({
                "role": "user",
                "content": [
                    {
                        "type": "tool_result",
                        "tool_use_id": tool_use_block.id,
                        "content": json.dumps(tool_result)
                    }
                ]
            })

            # Continue the loop
            continue

# Use it
result = process_tool_use("Who is user123 and send them a welcome email?")
print(result)
```

## The tool_choice Parameter

The `tool_choice` parameter controls when Claude can use tools.

### Tool Choice Options

```python
# Claude can choose to use tools or not
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice="auto",  # Default: Claude decides
    messages=messages
)

# Claude must use a tool for this request
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice="required",  # Must use a tool
    messages=messages
)

# Claude must use a specific tool
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    tool_choice={"type": "tool", "name": "get_user_info"},
    messages=messages
)

# Claude cannot use tools (text only)
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    # No tools parameter
    messages=messages
)
```

## TypeScript Implementation

### Define and Use Tools

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

const tools: Anthropic.Tool[] = [
  {
    name: "get_user_info",
    description: "Retrieve user information from the database",
    input_schema: {
      type: "object",
      properties: {
        user_id: {
          type: "string",
          description: "The unique user identifier"
        }
      },
      required: ["user_id"]
    }
  }
];

async function processToolUse(userMessage: string): Promise<string> {
  const messages: Anthropic.MessageParam[] = [
    { role: "user", content: userMessage }
  ];

  while (true) {
    const response = await client.messages.create({
      model: "claude-opus-4-6",
      max_tokens: 1024,
      tools: tools,
      messages: messages
    });

    if (response.stop_reason === "end_turn") {
      return response.content
        .filter((block): block is Anthropic.TextBlock => block.type === "text")
        .map(block => block.text)
        .join("\n");
    }

    if (response.stop_reason === "tool_use") {
      const toolUseBlock = response.content.find(
        (block): block is Anthropic.ToolUseBlock => block.type === "tool_use"
      );

      if (!toolUseBlock) break;

      const toolResult = executeToolFunction(
        toolUseBlock.name,
        toolUseBlock.input as Record<string, unknown>
      );

      messages.push({ role: "assistant", content: response.content });
      messages.push({
        role: "user",
        content: [
          {
            type: "tool_result",
            tool_use_id: toolUseBlock.id,
            content: JSON.stringify(toolResult)
          }
        ]
      });
    }
  }

  return "Error: Unexpected response";
}

function executeToolFunction(
  name: string,
  input: Record<string, unknown>
): unknown {
  switch (name) {
    case "get_user_info":
      return { user_id: input.user_id, name: "John Doe" };
    default:
      return { error: "Unknown tool" };
  }
}
```

## Error Handling

Always handle tool execution errors gracefully:

```python
def execute_tool_safely(tool_name: str, tool_input: dict):
    """Execute a tool with error handling"""
    try:
        if tool_name == "get_user_info":
            user_id = tool_input.get("user_id")
            if not user_id:
                return {"error": "user_id is required"}
            return get_user_info(user_id)
        else:
            return {"error": f"Unknown tool: {tool_name}"}
    except Exception as e:
        return {"error": f"Tool execution failed: {str(e)}"}
```

When returning errors to Claude, always include them in the tool result:

```python
messages.append({
    "role": "user",
    "content": [
        {
            "type": "tool_result",
            "tool_use_id": tool_use_block.id,
            "content": json.dumps(tool_result),
            "is_error": "error" in tool_result  # Mark as error if needed
        }
    ]
})
```

## Best Practices

1. **Design clear tool boundaries**: Each tool should do one thing well
2. **Validate inputs**: Check parameters before executing
3. **Return structured results**: Return JSON with clear success/error fields
4. **Provide context**: Give Claude clear error messages to work with
5. **Limit tool count**: 3-5 well-designed tools work better than 20 mediocre ones
6. **Test thoroughly**: Test all tool paths before production
7. **Document parameters**: Make tool descriptions comprehensive
8. **Handle timeouts**: Set reasonable execution timeouts for tools
