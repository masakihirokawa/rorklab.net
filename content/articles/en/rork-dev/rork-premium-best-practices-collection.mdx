## App Monetization Best Practices

### StoreKit 2 IAP Verification — Serverless Verification Pattern

**Best Practice: Support StoreKit 2's Transaction.listen with AppSync or Serverless JWT, and manage subscription renewal deadlines centrally on the server side using Product.SubscriptionInfo.**

In StoreKit 2, transactions are signed as JWS tokens, allowing servers to verify directly from Apple. The standard pattern is to separate local Transaction.listen for UI purposes from server verification for revenue-related operations.

```typescript
// Transaction verification pattern with React Native + TypeScript
import { useEffect, useState } from 'react';
import StoreKit from 'react-native-storekit2';

export function IAPManager() {
  const [transactions, setTransactions] = useState<string[]>([]);

  useEffect(() => {
    // Transaction listener for UI sync
    const unlistener = StoreKit.onTransaction(async (tx) => {
      // Send JWS token to server
      const response = await fetch('/api/verify-transaction', {
        method: 'POST',
        body: JSON.stringify({ transactionJWS: tx.jws }),
      });
      const verified = await response.json();
      if (verified.isValid) {
        setTransactions([...transactions, tx.id]);
      }
    });
    return () => unlistener();
  }, []);

  return <View>{/* UI */}</View>;
}
```

### RevenueCat Offering / Package Design

**Best Practice: Determine the Offering/Package hierarchy (Trial → Monthly → Annual) upfront, and use Remote Config alongside A/B testing.**

RevenueCat allows you to manage pricing across multiple platforms (Apple, Google, Stripe) centrally. The key is that your app only calls the RevenueCat API, while complex logic lives on the server side.

```typescript
// Offering / Entitlement check with RevenueCat
import Purchases from 'react-native-purchases';

async function checkSubscriptionStatus() {
  const offerings = await Purchases.getOfferings();
  const currentOffering = offerings?.current;

  if (!currentOffering) {
    console.log('No offerings available');
    return;
  }

  // Fetch premium package (change via Remote Config for A/B testing)
  const packages = currentOffering.availablePackages;
  const monthlyPkg = packages.find((p) => p.identifier === 'premium_monthly');

  // Check entitlements
  const customerInfo = await Purchases.getCustomerInfo();
  const isPremium = customerInfo.entitlements.active['premium'] !== undefined;

  return { monthlyPkg, isPremium };
}
```

### Stripe Subscription Management — Webhook Synchronization

**Best Practice: Record all state transitions (Payment Intent → Subscription → Invoice) via webhooks, and ensure idempotency.**

In Stripe, invoice.payment_succeeded and invoice.payment_failed are critical webhooks. Update your database based on these events while keeping the client side loosely coupled.

```typescript
// Webhook handler (Node.js Express)
app.post('/webhooks/stripe', express.raw({ type: 'application/json' }), async (req, res) => {
  const sig = req.headers['stripe-signature'];
  let event;

  try {
    event = stripe.webhooks.constructEvent(req.body, sig, STRIPE_WEBHOOK_SECRET);
  } catch (err) {
    res.status(400).send(`Webhook Error: ${err}`);
    return;
  }

  // Use idempotency key to prevent duplicate execution
  const idempotencyKey = event.id;
  const existing = await db.webhookEvent.findUnique({ where: { idempotencyKey } });
  if (existing) {
    res.json({ received: true });
    return;
  }

  switch (event.type) {
    case 'invoice.payment_succeeded': {
      const invoice = event.data.object;
      await db.subscription.update({
        where: { stripeSubId: invoice.subscription },
        data: { status: 'active', nextBillingDate: new Date(invoice.next_payment_attempt * 1000) },
      });
      break;
    }
  }

  res.json({ received: true });
});
```

### AdMob Ad Layout Optimization

**Best Practice: Use a three-tier structure of banner + interstitial + rewarded ads, and monitor eCPM daily.**

Balancing user experience with revenue requires separating ad types by flow. Banners are neutral, interstitials mark session boundaries, and rewarded ads are for optional actions.

```typescript
// AdMob ad display manager
import { InterstitialAd, BannerAd, RewardedAd, TestIds } from 'react-native-google-mobile-ads';

class AdManager {
  static showInterstitial = async () => {
    const interstitial = InterstitialAd.createForAdRequest(AD_UNIT_ID.INTERSTITIAL, {
      keywords: ['game', 'fun'],
    });

    await interstitial.load();
    interstitial.show();
  };

  static showRewarded = async (onRewardEarned: () => void) => {
    const rewarded = RewardedAd.createForAdRequest(AD_UNIT_ID.REWARDED, {
      keywords: ['game', 'puzzle'],
    });

    rewarded.addAdEventListener('onAdClosed', () => console.log('Rewarded Ad closed'));
    rewarded.addAdEventListener('onReward', ({ amount, type }) => {
      onRewardEarned();
    });

    await rewarded.load();
    rewarded.show();
  };
}
```

### Revenue Model Design — Freemium Funnel

**Best Practice: Use a three-tier model (7-day Trial → $4.99/month → $39.99/year) with "restrictions" to guide users gradually toward paid tiers.**

Successful freemium apps let users experience core features freely while keeping advanced features behind a paywall. Standard practice is to show "unlock this feature with Premium" on the 2nd or 3rd interaction, not on first tap.

```typescript
// Freemium logic
interface FeatureGate {
  name: string;
  requiredLevel: 'free' | 'premium';
  trialDaysLeft: number;
}

export function canUseFeature(feature: FeatureGate, userStatus: UserStatus): boolean {
  if (feature.requiredLevel === 'free') return true;

  // Premium check
  if (userStatus.subscription?.isActive) return true;

  // During trial?
  if (userStatus.subscription?.trialEndsAt && userStatus.subscription.trialEndsAt > new Date()) {
    return true;
  }

  return false;
}
```

## Production Quality Assurance — Testing, Monitoring, CI/CD

### Jest Component Unit Testing

**Best Practice: Structure component tests in three parts (render + user interaction + snapshot) with a target coverage of 80%.**

React Native Jest tests are JavaScript-based rather than DOM-based, making native module mocking crucial.

```typescript
// Button component test example
import { render, screen, fireEvent } from '@testing-library/react-native';
import Button from '@/components/Button';

describe('Button Component', () => {
  it('renders with correct label', () => {
    const { getByText } = render(<Button label="Press me" />);
    expect(getByText('Press me')).toBeTruthy();
  });

  it('calls onPress when tapped', () => {
    const mockPress = jest.fn();
    const { getByRole } = render(<Button label="Tap" onPress={mockPress} />);
    fireEvent.press(getByRole('button'));
    expect(mockPress).toHaveBeenCalledTimes(1);
  });

  it('matches snapshot', () => {
    const { toJSON } = render(<Button label="Snapshot" variant="primary" />);
    expect(toJSON()).toMatchSnapshot();
  });
});
```

## Performance Optimization Best Practices

### React Memo for Rendering Optimization

**Best Practice: Apply React.memo to child elements in large lists (100+ items), and memoize props objects with useMemo.**

In React Native, unnecessary re-renders directly impact performance. Always use memo in FlatList renderItem.

```typescript
// Optimized list item
const ListItem = React.memo(({ item, onPress }: ListItemProps) => (
  <TouchableOpacity onPress={() => onPress(item.id)}>
    <Text>{item.name}</Text>
  </TouchableOpacity>
), (prevProps, nextProps) => {
  // Custom comparison: only check item.id and onPress
  return prevProps.item.id === nextProps.item.id &&
         prevProps.onPress === nextProps.onPress;
});

// Parent component stabilizes props with useMemo
export function ArticleList({ articles }: ArticleListProps) {
  const handlePress = useCallback((id: string) => {
    navigation.push('Detail', { articleId: id });
  }, [navigation]);

  const memoizedItems = useMemo(
    () => articles.map((a) => ({ ...a })),
    [articles],
  );

  return (
    <FlatList
      data={memoizedItems}
      renderItem={({ item }) => (
        <ListItem item={item} onPress={handlePress} />
      )}
      keyExtractor={(item) => item.id}
    />
  );
}
```

### Offline First — SQLite + CRDT Synchronization

**Best Practice: Keep local database in SQLite, auto-resolve conflicts with CRDT algorithms, and batch-sync on network recovery.**

In offline-first design, the local database is the source of truth, with the server confirming the next state.

```typescript
// SQLite + CRDT sync pattern
import SQLite from 'react-native-sqlite-storage';
import { Automerge } from '@automerge/automerge';

class OfflineSync {
  private db: SQLiteDatabase;
  private automerge = Automerge.new();

  async syncWithServer() {
    // Send local changes to server
    const localChanges = await this.db.executeSql(
      'SELECT * FROM changes WHERE synced = 0',
    );

    const response = await fetch('/api/sync', {
      method: 'POST',
      body: JSON.stringify({
        changes: localChanges[0].rows._array,
        clientId: this.clientId,
      }),
    });

    const serverChanges = await response.json();

    // Resolve conflicts with CRDT
    const merged = Automerge.merge(this.automerge, serverChanges.doc);

    // Update database
    await this.db.executeSql('UPDATE changes SET synced = 1');
  }
}
```

### Gemini API Streaming Chatbot

**Best Practice: Display in real-time with Server-Sent Events (SSE) and flush buffers every 50ms to avoid chuntering effects.**

Streaming AI responses significantly improves UX.

```typescript
// Gemini API streaming implementation
async function streamChatResponse(message: string) {
  const response = await fetch('https://generativelanguage.googleapis.com/v1/models/gemini-pro:streamGenerateContent', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: message }] }],
    }),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value);
    const lines = buffer.split('\n');

    // Process only complete lines
    for (let i = 0; i < lines.length - 1; i++) {
      const line = lines[i];
      if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));
        if (data.candidates?.[0]?.content?.parts?.[0]?.text) {
          updateChatUI(data.candidates[0].content.parts[0].text);
        }
      }
    }
    buffer = lines[lines.length - 1];
  }
}
```

### Cloudflare Workers AI with Rate Limiting + Caching

// Using AI Gateway in Cloudflare Worker
export default {
  async fetch(request: Request, env: Env) {
    const cache = caches.default;
    const cacheKey = new Request(request.url, { method: 'GET' });
    let response = await cache.match(cacheKey);

    if (!response) {
      // Request to AI Gateway (within rate limit)
      response = await fetch('https://api.cloudflare.com/client/v4/accounts/xxx/ai/run/text_generation', {

    return response;
  },
};

### React Native + Expo AI Development — Typed Interface

**Best Practice: Validate AI responses with Zod schema and auto-retry on incomplete responses.**

AI output is unpredictable, making type safety critical.

```typescript
import { z } from 'zod';

// Type definition for AI response
const AIResponseSchema = z.object({
  suggestion: z.string(),
  confidence: z.number().min(0).max(1),
  relatedTopics: z.array(z.string()),
});

type AIResponse = z.infer<typeof AIResponseSchema>;

async function generateAIInsight(userInput: string): Promise<AIResponse> {
  let retries = 0;
  while (retries < 3) {
    try {
      const raw = await fetch('/api/ai/generate', {
        method: 'POST',
        body: JSON.stringify({ prompt: userInput }),
      }).then((r) => r.json());

      return AIResponseSchema.parse(raw);
    } catch (error) {
      retries++;
      if (retries === 3) throw error;
      await new Promise((r) => setTimeout(r, 1000 * retries));
    }
  }
  throw new Error('Failed to generate AI response');
}
```
