## Introduction

Forms are one of the most critical touchpoints between your app and its users. Sign-up flows, login screens, profile editors, contact forms, payment inputs — they all rely on forms. And when validation is poorly implemented, users get frustrated and abandon the process entirely.

When building apps with Rork, the quality of your form handling directly impacts the overall polish of your product. A beautifully designed app with clunky, confusing form validation will leave a poor impression.

In this guide, you'll learn how to combine **react-hook-form** — the most popular form library in the React Native ecosystem — with **zod**, a type-safe schema validation library, to build robust, user-friendly forms in your Rork apps.

**What you'll learn:**

- How to set up react-hook-form with zod in a Rork project
- Implementing real-time validation patterns
- Designing and building multi-step forms
- Displaying user-friendly error messages

## Why react-hook-form and zod?

There are several form libraries available for React Native, but **react-hook-form** stands out for its performance. It minimizes re-renders by using uncontrolled components under the hood, which means even forms with many fields stay responsive and smooth. Compared to alternatives like Formik, it handles complex forms with noticeably less overhead.

**zod** is a TypeScript-first schema declaration library that lets you define validation rules and TypeScript types in a single place. When the schema changes, the types update automatically — no more keeping separate type definitions in sync with your validation logic.

Together, they offer a powerful combination:

- **Type safety**: Form values are fully typed through TypeScript
- **Declarative rules**: Validation logic lives in one centralized schema
- **High performance**: Minimal re-renders for a smooth user experience
- **Centralized error messages**: Messages defined in the zod schema make maintenance straightforward

## Setup and Installation

Install the required packages in your Rork project:

```bash
# Run in your Rork project root
npx expo install react-hook-form @hookform/resolvers zod
```

The `@hookform/resolvers` package acts as a bridge between react-hook-form and zod, allowing you to use zod schemas directly as form validation rules.

## Implementing Basic Form Validation

### Defining a zod Schema

Start by defining a validation schema for a user registration form:

```typescript
// schemas/userRegistration.ts
import { z } from "zod";

// Validation schema for user registration
export const userRegistrationSchema = z.object({
  name: z
    .string()
    .min(1, "Name is required")
    .max(50, "Name must be 50 characters or fewer"),
  email: z
    .string()
    .min(1, "Email is required")
    .email("Please enter a valid email address"),
  password: z
    .string()
    .min(8, "Password must be at least 8 characters")
    .regex(
      /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/,
      "Must include at least one uppercase letter, one lowercase letter, and one number"
    ),
  confirmPassword: z.string().min(1, "Please confirm your password"),
}).refine((data) => data.password === data.confirmPassword, {
  message: "Passwords do not match",
  path: ["confirmPassword"],
});

// Auto-generate TypeScript type from the schema
export type UserRegistrationForm = z.infer<typeof userRegistrationSchema>;
```

The key here is using `z.infer` to **automatically derive the TypeScript type from the schema**. When you update the schema, the type updates with it — eliminating the risk of type definitions drifting out of sync.

### Building the Form Component

```tsx
// components/RegistrationForm.tsx
import React from "react";
import { View, Text, TextInput, TouchableOpacity, StyleSheet } from "react-native";
import { useForm, Controller } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import {
  userRegistrationSchema,
  type UserRegistrationForm,
} from "../schemas/userRegistration";

export default function RegistrationForm() {
  const {
    control,
    handleSubmit,
    formState: { errors, isSubmitting },
  } = useForm<UserRegistrationForm>({
    resolver: zodResolver(userRegistrationSchema),
    defaultValues: {
      name: "",
      email: "",
      password: "",
      confirmPassword: "",
    },
  });

  const onSubmit = async (data: UserRegistrationForm) => {
    // Runs only after all validation passes
    console.log("Submitted data:", data);
    // Expected output:
    // { name: "John Doe", email: "john@example.com",
    //   password: "MyPass123", confirmPassword: "MyPass123" }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Create Account</Text>

      {/* Name field */}
      <Controller
        control={control}
        name="name"
        render={({ field: { onChange, onBlur, value } }) => (
          <View style={styles.fieldContainer}>
            <Text style={styles.label}>Name</Text>
            <TextInput
              style={[styles.input, errors.name && styles.inputError]}
              onBlur={onBlur}
              onChangeText={onChange}
              value={value}
              placeholder="John Doe"
            />
            {errors.name && (
              <Text style={styles.errorText}>{errors.name.message}</Text>
            )}
          </View>
        )}
      />

      {/* Email field */}
      <Controller
        control={control}
        name="email"
        render={({ field: { onChange, onBlur, value } }) => (
          <View style={styles.fieldContainer}>
            <Text style={styles.label}>Email</Text>
            <TextInput
              style={[styles.input, errors.email && styles.inputError]}
              onBlur={onBlur}
              onChangeText={onChange}
              value={value}
              placeholder="you@example.com"
              keyboardType="email-address"
              autoCapitalize="none"
            />
            {errors.email && (
              <Text style={styles.errorText}>{errors.email.message}</Text>
            )}
          </View>
        )}
      />

      {/* Submit button */}
      <TouchableOpacity
        style={[styles.button, isSubmitting && styles.buttonDisabled]}
        onPress={handleSubmit(onSubmit)}
        disabled={isSubmitting}
      >
        <Text style={styles.buttonText}>
          {isSubmitting ? "Submitting..." : "Sign Up"}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 24 },
  title: { fontSize: 24, fontWeight: "bold", marginBottom: 24 },
  fieldContainer: { marginBottom: 16 },
  label: { fontSize: 14, fontWeight: "600", marginBottom: 4, color: "#374151" },
  input: {
    borderWidth: 1,
    borderColor: "#D1D5DB",
    borderRadius: 8,
    padding: 12,
    fontSize: 16,
  },
  inputError: { borderColor: "#EF4444" },
  errorText: { color: "#EF4444", fontSize: 12, marginTop: 4 },
  button: {
    backgroundColor: "#6366F1",
    borderRadius: 8,
    padding: 16,
    alignItems: "center",
    marginTop: 8,
  },
  buttonDisabled: { opacity: 0.6 },
  buttonText: { color: "#FFFFFF", fontSize: 16, fontWeight: "bold" },
});
```

This component uses `Controller` to connect react-hook-form with React Native's `TextInput`. The `zodResolver` automatically runs validation against your schema and populates the `errors` object with any issues it finds.

## Real-Time Validation

By default, validation runs on form submission. But for a better user experience, you can trigger validation as users interact with each field:

```tsx
const {
  control,
  handleSubmit,
  formState: { errors },
} = useForm<UserRegistrationForm>({
  resolver: zodResolver(userRegistrationSchema),
  mode: "onBlur",           // Validate when a field loses focus
  reValidateMode: "onChange", // Re-validate on every keystroke after an error
});
```

Here's how each `mode` option behaves:

- **`onSubmit`** (default): Validates only when the form is submitted. Best for simple forms.
- **`onBlur`**: Validates when a field loses focus. This is the recommended pattern for most use cases.
- **`onChange`**: Validates on every keystroke. Useful for specific fields like password strength meters.
- **`onTouched`**: Validates on first blur, then on every change afterward.

For most forms, `mode: "onBlur"` combined with `reValidateMode: "onChange"` hits the sweet spot. Users won't see errors while they're still typing, but they'll get immediate feedback once they move to the next field.

## Designing Multi-Step Forms

When you have many input fields — think user onboarding or detailed profile setup — splitting the form into steps reduces cognitive load.

```tsx
// hooks/useMultiStepForm.ts
import { useState, useCallback } from "react";

export function useMultiStepForm(totalSteps: number) {
  const [currentStep, setCurrentStep] = useState(0);

  const next = useCallback(() => {
    setCurrentStep((prev) => Math.min(prev + 1, totalSteps - 1));
  }, [totalSteps]);

  const back = useCallback(() => {
    setCurrentStep((prev) => Math.max(prev - 1, 0));
  }, []);

  const goTo = useCallback((step: number) => {
    setCurrentStep(Math.max(0, Math.min(step, totalSteps - 1)));
  }, [totalSteps]);

  return {
    currentStep,
    totalSteps,
    isFirstStep: currentStep === 0,
    isLastStep: currentStep === totalSteps - 1,
    progress: (currentStep + 1) / totalSteps,
    next,
    back,
    goTo,
  };
}
```

Define separate schemas for each step, then validate only the current step's fields before advancing:

```typescript
// schemas/multiStepProfile.ts
import { z } from "zod";

// Step 1: Basic info
export const basicInfoSchema = z.object({
  displayName: z.string().min(1, "Display name is required"),
  bio: z.string().max(200, "Bio must be 200 characters or fewer").optional(),
});

// Step 2: Contact details
export const contactSchema = z.object({
  phone: z
    .string()
    .regex(/^\+?[\d\s-]{7,15}$/, "Please enter a valid phone number")
    .optional()
    .or(z.literal("")),
  website: z.string().url("Please enter a valid URL").optional().or(z.literal("")),
});

// Step 3: Preferences
export const preferencesSchema = z.object({
  language: z.enum(["ja", "en"], { required_error: "Please select a language" }),
  notifications: z.boolean().default(true),
});
```

By separating schemas per step, you validate only what's relevant at each stage, avoiding confusing error messages about fields the user hasn't even seen yet.

## Common Issues and Solutions

### Controlled Component Warnings with TextInput

If `undefined` gets passed as the `value` to a `Controller`, React Native treats it as an uncontrolled component and logs a warning. Always set `defaultValues` with empty strings (`""`) for every field.

### Keyboard Covering the Form

Wrap your form in a `KeyboardAvoidingView` to prevent the keyboard from hiding input fields. Use `behavior="padding"` on iOS and `behavior="height"` on Android:

```tsx
import { KeyboardAvoidingView, Platform } from "react-native";

<KeyboardAvoidingView
  behavior={Platform.OS === "ios" ? "padding" : "height"}
  style={{ flex: 1 }}
>
  {/* Your form components */}
</KeyboardAvoidingView>
```

### Async Validation (e.g., Checking Email Availability)

For validation that requires a server call, pass an async function to zod's `.refine()`:

```typescript
const emailSchema = z.object({
  email: z
    .string()
    .email("Please enter a valid email address")
    .refine(async (email) => {
      // Check availability against your API
      const response = await fetch(`/api/check-email?email=${email}`);
      const { available } = await response.json();
      return available;
    }, "This email is already taken"),
});
```

## FAQ

### Q: Should I use react-hook-form or Formik?

For performance, react-hook-form is the stronger choice. It uses uncontrolled components internally, which means it minimizes re-renders across the entire form. Formik uses controlled components, so every keystroke can trigger a re-render of the whole form tree. For forms with more than a handful of fields, the difference in responsiveness is noticeable.

### Q: Can I use other validation libraries like Yup instead of zod?

Absolutely. The `@hookform/resolvers` package supports Yup, Joi, Superstruct, and several others. That said, zod offers the best TypeScript integration thanks to `z.infer`, which eliminates the need to maintain separate type definitions alongside your validation schemas.

### Q: How do I preserve input values when navigating between steps in a multi-step form?

By default, `useForm` keeps field values even when components unmount (`shouldUnregister: false`). So switching between steps won't lose any data. For the cleanest architecture, call `useForm` in the parent component and pass the `control` prop down to each step component.

Form validation in Rork apps becomes both type-safe and performant when you pair react-hook-form with zod. The key takeaways are: centralize your type definitions and validation rules in zod schemas, use `mode: "onBlur"` for user-friendly validation timing, split complex forms into steps with per-step schemas, and always wrap forms in `KeyboardAvoidingView` for a smooth mobile experience.
