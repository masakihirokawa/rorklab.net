## What Is Amplitude Analytics — and Why Should Your Rork App Use It?

Shipping an app without understanding how users actually behave inside it is like navigating with your eyes closed. **Amplitude Analytics** is one of the world's leading product analytics platforms for mobile apps — alongside PostHog and Mixpanel — and it gives you the visibility you need to make confident, data-driven decisions.

For Rork-generated React Native / Expo apps, integrating Amplitude unlocks a powerful suite of insights:

- **Event tracking** — Record any user interaction: button taps, screen views, purchases, and more
- **Funnel analysis** — Visualize conversion rates across multi-step flows (e.g., sign-up → onboarding → payment)
- **Retention analysis** — Measure how many users come back on Day 1, Day 7, and Day 30
- **Cohort analysis** — Compare behavior across distinct user segments
- **A/B test integration** — Measure the impact of UI changes with Amplitude Experiments

Amplitude's generous free tier (up to 1 million events/month) makes it a practical choice for indie developers and early-stage startups alike.

## Installation and Initial Setup

Start by adding the official Amplitude SDK to your Rork project. As of 2026, `@amplitude/analytics-react-native` is the recommended package for React Native and Expo apps.

```bash
# Install Amplitude
npm install @amplitude/analytics-react-native

# For Expo projects, these peer dependencies are also required
npx expo install expo-device expo-application
```

Create a project in the [Amplitude dashboard](https://amplitude.com) to get your API key, then store it as an environment variable.

```typescript
// app/_layout.tsx — initialize Amplitude at the app entry point
import { init } from '@amplitude/analytics-react-native';
import { useEffect } from 'react';

export default function RootLayout() {
  useEffect(() => {
    // Load your API key from environment variables — never hardcode it
    init(process.env.EXPO_PUBLIC_AMPLITUDE_API_KEY!, {
      trackingOptions: {
        ipAddress: false,       // Disable IP logging for privacy
        language: true,         // Track device language
        platform: true,         // Track platform (iOS/Android)
        osVersion: true,        // Track OS version
        appVersion: true,       // Track app version
      },
      logLevel: __DEV__ ? 3 : 0, // Show debug logs in development only
    });
  }, []);

  // ... navigation setup
}
```

Add your key to `.env.local`:

```bash
# .env.local
EXPO_PUBLIC_AMPLITUDE_API_KEY=YOUR_AMPLITUDE_API_KEY
```

> **Security note**: Variables prefixed with `EXPO_PUBLIC_` are exposed to the client bundle. Your front-end Amplitude API key is safe to include this way — but never expose any secret or server-side keys.

## Implementing Event Tracking

Event tracking is the foundation of everything in Amplitude. Every meaningful user action you record becomes a building block for deeper analysis.

### A Custom Hook for Consistent Tracking

Centralizing your tracking logic in a custom hook keeps your codebase clean and makes it easy to add or modify events in one place.

```typescript
// hooks/useAmplitude.ts
import { track, setUserId, identify, Identify } from '@amplitude/analytics-react-native';

export const useAmplitude = () => {
  // Call this after a user logs in
  const setUser = (userId: string, userProperties?: Record<string, unknown>) => {
    setUserId(userId);
    if (userProperties) {
      const identifyEvent = new Identify();
      Object.entries(userProperties).forEach(([key, value]) => {
        identifyEvent.set(key, value);
      });
      identify(identifyEvent);
    }
  };

  // Core tracking function
  const trackEvent = (
    eventName: string,
    eventProperties?: Record<string, unknown>
  ) => {
    track(eventName, eventProperties);
  };

  // Typed convenience wrappers for common events
  const trackScreenView = (screenName: string) => {
    trackEvent('Screen Viewed', { screen_name: screenName });
  };

  const trackButtonTap = (buttonName: string, context?: string) => {
    trackEvent('Button Tapped', { button_name: buttonName, context });
  };

  const trackPurchase = (productId: string, price: number, currency = 'USD') => {
    trackEvent('Purchase Completed', { product_id: productId, price, currency });
  };

  return { setUser, trackEvent, trackScreenView, trackButtonTap, trackPurchase };
};
```

### Using the Hook in a Screen

```typescript
// screens/HomeScreen.tsx
import { useEffect } from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { useAmplitude } from '@/hooks/useAmplitude';

export default function HomeScreen() {
  const { trackScreenView, trackButtonTap } = useAmplitude();

  useEffect(() => {
    trackScreenView('Home');
  }, []);

  const handleUpgradePress = () => {
    trackButtonTap('Upgrade to Premium', 'home_banner');
    // Navigate to paywall...
  };

  return (
    <TouchableOpacity onPress={handleUpgradePress}>
      <Text>Upgrade to Premium</Text>
    </TouchableOpacity>
  );
}
```

### Event Naming Best Practices

Consistent naming conventions matter — inconsistent event names make dashboards noisy and hard to interpret. The widely recommended pattern is `Object Action` (e.g., `Purchase Completed`, `Screen Viewed`, `Button Tapped`).

## Funnel Analysis: Visualizing Your Conversion Path

Funnel analysis answers the question: "Of the users who started this flow, how many completed it?" Whether it's registration, onboarding, or a payment flow, knowing where users drop off is the first step to fixing it.

### Implementing a Registration and Payment Funnel

```typescript
// services/analytics/funnel.ts
import { track } from '@amplitude/analytics-react-native';

export const funnelEvents = {
  // Registration flow
  registrationStarted: () =>
    track('Registration Started'),

  emailEntered: (method: 'email' | 'apple' | 'google') =>
    track('Email Entered', { auth_method: method }),

  registrationCompleted: (userId: string) =>
    track('Registration Completed', { user_id: userId }),

  // Onboarding flow
  onboardingStepCompleted: (step: number, stepName: string) =>
    track('Onboarding Step Completed', { step_number: step, step_name: stepName }),

  onboardingCompleted: () =>
    track('Onboarding Completed'),

  // Purchase flow
  paywallViewed: (source: string) =>
    track('Paywall Viewed', { source }),

  purchaseInitiated: (planId: string, price: number) =>
    track('Purchase Initiated', { plan_id: planId, price }),

  purchaseCompleted: (planId: string, price: number) =>
    track('Purchase Completed', { plan_id: planId, price, currency: 'USD' }),

  purchaseFailed: (reason: string) =>
    track('Purchase Failed', { failure_reason: reason }),
};
```

In the Amplitude dashboard, you can build a funnel using these events — for example: `Registration Started → Email Entered → Registration Completed → Onboarding Completed → Purchase Completed`. Amplitude will show you exactly how many users completed each step and where the biggest drop-offs occur.

Pairing Amplitude funnel data with store-level insights from [Rork App Store Connect Analytics: A Beginner's Guide](/articles/rork-business/rork-app-store-connect-analytics-beginners-guide) gives you a complete picture of user behavior from discovery to conversion.

## Retention Analysis: Measuring Long-Term Engagement

Retention is one of the most important metrics for any app. A high Day 1 retention (users who return the day after they first open the app) indicates strong initial engagement, while Day 30 retention reflects your app's long-term value.

Amplitude automatically calculates retention from a starting event — most commonly an `App Opened` event.

```typescript
// Track app opens for retention measurement
import { track } from '@amplitude/analytics-react-native';
import { AppState, AppStateStatus } from 'react-native';
import { useEffect, useRef } from 'react';

export function useAppStateTracking() {
  const appState = useRef(AppState.currentState);

  useEffect(() => {
    const subscription = AppState.addEventListener('change', (nextAppState: AppStateStatus) => {
      if (
        appState.current.match(/inactive|background/) &&
        nextAppState === 'active'
      ) {
        // Returning from background
        track('App Opened', { source: 'background_to_foreground' });
      }
      appState.current = nextAppState;
    });

    // First cold start
    track('App Opened', { source: 'cold_start' });

    return () => subscription.remove();
  }, []);
}
```

If your retention numbers are lower than expected, push notifications and in-app prompts can help. The [Rork In-App Review Prompt Guide](/articles/rork-dev/rork-inapp-review-prompt-ios-android-guide) covers practical engagement techniques you can layer on top of your analytics setup.

## Amplitude vs. PostHog: Choosing the Right Tool

If you've already explored [PostHog for your Rork app](/articles/rork-dev/rork-posthog-product-analytics-guide), you might be wondering when to choose Amplitude instead. Here's a practical breakdown:

Amplitude tends to excel when:

- **Funnel and cohort analysis** is your primary focus — Amplitude's UI is purpose-built for this
- **Your team is growing** — Amplitude's sharing and collaboration features shine at scale
- **A/B testing (Amplitude Experiments)** is part of your optimization workflow
- **Enterprise needs** — Amplitude has strong governance and data pipeline integrations

PostHog has the edge when you want open-source or self-hosted options, built-in session replay, or a tighter integration with your engineering workflow.

For a broader perspective on performance measurement, the [Rork App Performance Optimization Guide](/articles/rork-dev/rork-app-performance-optimization-complete-guide) covers the mindset of measurement-first development that pairs well with any analytics stack.

## Frequently Asked Questions

**Q. How much does Amplitude cost, and is the free plan enough for early-stage apps?**

A. Amplitude's free plan supports up to 1 million events per month and includes core features like funnel analysis, retention charts, and cohort analysis. For most indie apps and early-stage startups, this is more than enough. Consider upgrading to a paid plan once your user base grows to tens of thousands of active users.

**Q. Does Amplitude work with Expo managed workflow?**

A. Yes, `@amplitude/analytics-react-native` is compatible with Expo managed workflow. For the most accurate session tracking behavior, testing with an EAS Build (custom dev client) is recommended. Make sure to also install `expo-device` and `expo-application` to enable full device metadata collection.

**Q. How should I handle GDPR and privacy compliance?**

A. You'll need to mention Amplitude in your privacy policy if you're collecting user activity data. Set `trackingOptions: { ipAddress: false }` in your init config to anonymize IP addresses by default. If you have EU users, review Amplitude's GDPR compliance documentation and consider adding a consent flow before initializing the SDK.

**Q. Adding tracking events throughout my codebase feels overwhelming. Is there an efficient approach?**

A. Start small — focus on 5 to 10 events that cover your most important conversion funnel and core features. If you're using Expo Router, you can add a navigation listener in `_layout.tsx` to automatically capture all screen views without modifying individual screens. For UI interactions, wrapping shared components (like a `TrackedButton`) with tracking logic keeps the implementation centralized and maintainable.

Integrating Amplitude transforms your development process from reactive to proactive. Instead of guessing what's wrong when growth stalls, you'll have the data to pinpoint exactly where users are struggling — and what to fix first.
