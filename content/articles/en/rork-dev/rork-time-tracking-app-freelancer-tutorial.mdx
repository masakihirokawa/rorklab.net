For anyone working freelance, accurate time tracking is directly tied to revenue. Whether you're preparing invoices for clients, reviewing your own productivity, or analyzing profitability across projects, the precision of your time management can make or break your business.

In this tutorial, we'll walk through building a practical time tracking app from scratch using Rork. By leveraging Rork's AI-powered code generation, we'll efficiently create an app that includes a timer, project management, work logging, and reporting features.

## Building the Timer — The Core Feature

The timer is the heart of any time tracking app. We'll build a custom hook that combines `useRef` and `setInterval` for an accurate stopwatch.

```typescript
// hooks/useTimer.ts — Custom timer hook
import { useState, useRef, useCallback } from 'react';

interface TimerState {
  isRunning: boolean;
  elapsedSeconds: number;
  startTime: number | null;
}

export function useTimer() {
  const [state, setState] = useState<TimerState>({
    isRunning: false,
    elapsedSeconds: 0,
    startTime: null,
  });
  const intervalRef = useRef<NodeJS.Timeout | null>(null);

  // Start the timer
  const start = useCallback(() => {
    const now = Date.now();
    setState(prev => ({
      ...prev,
      isRunning: true,
      startTime: now - prev.elapsedSeconds * 1000,
    }));

    intervalRef.current = setInterval(() => {
      setState(prev => {
        if (!prev.startTime) return prev;
        return {
          ...prev,
          elapsedSeconds: Math.floor((Date.now() - prev.startTime) / 1000),
        };
      });
    }, 1000);
  }, []);

  // Stop the timer
  const stop = useCallback(() => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
    setState(prev => ({ ...prev, isRunning: false }));
  }, []);

  // Reset the timer
  const reset = useCallback(() => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
    setState({ isRunning: false, elapsedSeconds: 0, startTime: null });
  }, []);

  return { ...state, start, stop, reset };
}
```

This custom hook can be used from any screen. Because it's anchored to `Date.now()`, the elapsed time stays accurate even when the app goes to the background.

We'll also need a formatting utility:

```typescript
// utils/formatTime.ts — Convert elapsed seconds to HH:MM:SS
export function formatTime(totalSeconds: number): string {
  const hours = Math.floor(totalSeconds / 3600);
  const minutes = Math.floor((totalSeconds % 3600) / 60);
  const seconds = totalSeconds % 60;

  return [hours, minutes, seconds]
    .map(v => String(v).padStart(2, '0'))
    .join(':');
}

// Expected output:

// formatTime(90)    → "00:01:30"
```

Freelancers typically juggle multiple projects at once, so the ability to categorize work by project is essential.

Next, we'll implement the storage layer using AsyncStorage:

```typescript
// storage/projectStorage.ts — Project data persistence
import AsyncStorage from '@react-native-async-storage/async-storage';
import { Project } from '../types';

const PROJECTS_KEY = '@timetracker_projects';

export async function getProjects(): Promise<Project[]> {
  try {
    const json = await AsyncStorage.getItem(PROJECTS_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('Failed to load projects:', error);
    return [];
  }
}

export async function saveProject(project: Project): Promise<void> {
  const projects = await getProjects();
  const index = projects.findIndex(p => p.id === project.id);

  if (index >= 0) {
    projects[index] = project; // Update existing
  } else {
    projects.push(project);    // Add new
  }

  await AsyncStorage.setItem(PROJECTS_KEY, JSON.stringify(projects));
}

export async function deleteProject(id: string): Promise<void> {
  const projects = await getProjects();
  const filtered = projects.filter(p => p.id !== id);
  await AsyncStorage.setItem(PROJECTS_KEY, JSON.stringify(filtered));
}
```

## Recording and Displaying Work Logs

When the timer stops, we automatically save the session as a TimeEntry:

```typescript
// storage/timeEntryStorage.ts — Time entry persistence
import AsyncStorage from '@react-native-async-storage/async-storage';
import { TimeEntry } from '../types';

const ENTRIES_KEY = '@timetracker_entries';

export async function getTimeEntries(): Promise<TimeEntry[]> {
  try {
    const json = await AsyncStorage.getItem(ENTRIES_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('Failed to load time entries:', error);
    return [];
  }
}

export async function saveTimeEntry(entry: TimeEntry): Promise<void> {
  const entries = await getTimeEntries();
  entries.unshift(entry); // Add newest entry to the top
  await AsyncStorage.setItem(ENTRIES_KEY, JSON.stringify(entries));
}

// Filter entries by date range
export async function getEntriesByDateRange(
  startDate: Date,
  endDate: Date
): Promise<TimeEntry[]> {
  const entries = await getTimeEntries();
  const startTs = startDate.getTime();
  const endTs = endDate.getTime();

  return entries.filter(
    e => e.startTime >= startTs && e.startTime <= endTs
  );
}
```

```typescript
// Helper to group entries by date
function groupEntriesByDate(entries: TimeEntry[]) {
  const groups: Record<string, TimeEntry[]> = {};

  entries.forEach(entry => {
    const dateKey = new Date(entry.startTime)
      .toLocaleDateString('en-US', {
        year: 'numeric',
        month: 'long',
        day: 'numeric',
      });

    if (!groups[dateKey]) groups[dateKey] = [];
    groups[dateKey].push(entry);
  });

  return Object.entries(groups).map(([date, data]) => ({
    title: date,
    data,
  }));
}
```

## Reports Screen — Daily and Weekly Summaries

For freelancers, time reports are indispensable for invoicing and self-analysis. We'll build a reports screen with toggleable daily and weekly views.

```typescript
// utils/reportUtils.ts — Report aggregation logic
import { TimeEntry, Project } from '../types';

interface DailySummary {
  date: string;
  totalSeconds: number;
  byProject: Record<string, number>; // projectId -> seconds
}

// Generate a 7-day summary
export function getWeeklySummary(
  entries: TimeEntry[]
): DailySummary[] {
  const summaries: DailySummary[] = [];

  for (let i = 6; i >= 0; i--) {
    const date = new Date();
    date.setDate(date.getDate() - i);
    date.setHours(0, 0, 0, 0);

    const nextDate = new Date(date);
    nextDate.setDate(nextDate.getDate() + 1);

    const dayEntries = entries.filter(
      e => e.startTime >= date.getTime() && e.startTime < nextDate.getTime()
    );

    const byProject: Record<string, number> = {};
    let totalSeconds = 0;

    dayEntries.forEach(entry => {
      totalSeconds += entry.duration;
      byProject[entry.projectId] =
        (byProject[entry.projectId] || 0) + entry.duration;
    });

    summaries.push({
      date: date.toLocaleDateString('en-US', {
        month: 'short',
        day: 'numeric',
      }),
      totalSeconds,
      byProject,
    });
  }

  return summaries;
}

// Calculate earnings based on hourly rates
export function calculateEarnings(
  entries: TimeEntry[],
  projects: Project[]
): number {
  return entries.reduce((total, entry) => {
    const project = projects.find(p => p.id === entry.projectId);
    if (project?.hourlyRate) {
      return total + (entry.duration / 3600) * project.hourlyRate;
    }
    return total;
  }, 0);
}

// Expected output (calculateEarnings):
// 2 hours of work (project hourly rate: $50) → 100
```

## Implementation Tips — Polishing the UX

### Timer Accuracy After Background Resume

It's crucial that the timer stays accurate when the app returns from the background. Our `useTimer` hook uses `Date.now()` as its anchor, so the elapsed time is automatically correct upon resume.

Adding an `AppState` listener ensures the UI refreshes immediately:

```typescript
import { AppState, AppStateStatus } from 'react-native';
import { useEffect, useRef } from 'react';

export function useAppStateRefresh(onForeground: () => void) {
  const appState = useRef(AppState.currentState);

  useEffect(() => {
    const subscription = AppState.addEventListener(
      'change',
      (nextState: AppStateStatus) => {
        if (
          appState.current.match(/inactive|background/) &&
          nextState === 'active'
        ) {
          onForeground(); // Trigger callback when returning to foreground
        }
        appState.current = nextState;
      }
    );

    return () => subscription.remove();
  }, [onForeground]);
}
```

## Frequently Asked Questions

### Q1: Will the data take up too much storage on the device?

AsyncStorage stores data as text (JSON), so even thousands of time entries only take up a few megabytes. For long-term use, consider adding a CSV export feature so users can archive older data and free up space.

### Q2: Can I sync data across multiple devices?

This tutorial uses local-only storage with AsyncStorage, but you can integrate a BaaS like Supabase for cloud synchronization. Supabase's real-time sync feature allows instant data reflection across all your devices.

### Q3: Can the app calculate earnings for projects with different hourly rates?

Absolutely. The `calculateEarnings` function references each project's hourly rate and automatically computes the totals. You can display both a grand total and a per-project breakdown on the reports screen — perfect for generating invoices.

### Q4: Are there any App Store review concerns for this type of app?

Time tracking apps are generally well-received in App Store reviews. Just make sure to include a privacy policy and properly declare background timer usage in `Info.plist` under `UIBackgroundModes` if your timer runs in the background.
