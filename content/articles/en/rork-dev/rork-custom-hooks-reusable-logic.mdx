## Why Custom Hooks Matter in Rork Development

If you've been building apps with Rork, you've probably noticed a pattern: the same logic keeps showing up across different screens. Fetching data from an API, managing form inputs, reading device dimensions — these operations appear everywhere, and copying the same code into every component quickly becomes a maintenance headache.

Custom hooks solve this problem elegantly. They let you extract stateful logic into standalone functions that any component can reuse. Since Rork generates standard React Native code, custom hooks work seamlessly within the Rork ecosystem. The result is cleaner components, fewer bugs, and an architecture that scales gracefully as your app grows.

In this guide, we'll walk through four practical custom hook patterns that cover the most common needs in mobile app development.

## The Anatomy of a Custom Hook

A custom hook is simply a function whose name starts with `use` and that calls other React hooks internally. Unlike components, hooks don't return JSX — they return data and functions that components can consume.

```typescript
// hooks/useCounter.ts — The simplest possible custom hook
import { useState, useCallback } from 'react';

export function useCounter(initialValue: number = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = useCallback(() => setCount(prev => prev + 1), []);
  const decrement = useCallback(() => setCount(prev => prev - 1), []);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);

  return { count, increment, decrement, reset };
}

// Usage:
// const { count, increment, decrement } = useCounter(0);
```

Once you grasp this structure, you can extract virtually any piece of logic into a reusable hook.

## Pattern 1: Data Fetching with `useFetch`

The most common operation in any Rork app is fetching data from an external API. Rather than writing loading states, error handling, and retry logic in every screen, wrap it all in a single hook.

```typescript
// hooks/useFetch.ts
import { useState, useEffect, useCallback } from 'react';

interface UseFetchResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  refetch: () => void;
}

export function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  const fetchData = useCallback(async () => {
    setLoading(true);
    setError(null);
    try {
      const response = await fetch(url);
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      const json = await response.json();
      setData(json);
    } catch (err) {
      setError(err instanceof Error ? err.message : 'A network error occurred');
    } finally {
      setLoading(false);
    }
  }, [url]);

  useEffect(() => {
    fetchData();
  }, [fetchData]);

  return { data, loading, error, refetch: fetchData };
}

// Usage:
// const { data: users, loading, error, refetch } = useFetch<User[]>('https://api.example.com/users');
// if (loading) return <ActivityIndicator />;
// if (error) return <Text>Error: {error}</Text>;
```

With this single hook, every data-fetching screen in your app follows the same consistent pattern.

## Pattern 2: Form Management with `useForm`

Forms are notorious for accumulating boilerplate — input state, validation logic, error messages. A lightweight form hook keeps things manageable from login screens to profile editors.

```typescript
// hooks/useForm.ts
import { useState, useCallback } from 'react';

type ValidationRules<T> = {
  [K in keyof T]?: (value: T[K]) => string | null;
};

interface UseFormResult<T> {
  values: T;
  errors: Partial<Record<keyof T, string>>;
  handleChange: <K extends keyof T>(field: K, value: T[K]) => void;
  validate: () => boolean;
  reset: () => void;
}

export function useForm<T extends Record<string, any>>(
  initialValues: T,
  rules?: ValidationRules<T>
): UseFormResult<T> {
  const [values, setValues] = useState<T>(initialValues);
  const [errors, setErrors] = useState<Partial<Record<keyof T, string>>>({});

  const handleChange = useCallback(<K extends keyof T>(field: K, value: T[K]) => {
    setValues(prev => ({ ...prev, [field]: value }));
    // Clear the error for this field when the user starts typing
    setErrors(prev => ({ ...prev, [field]: undefined }));
  }, []);

  const validate = useCallback(() => {
    if (!rules) return true;
    const newErrors: Partial<Record<keyof T, string>> = {};
    let isValid = true;

    for (const [field, rule] of Object.entries(rules)) {
      const error = (rule as Function)(values[field]);
      if (error) {
        newErrors[field as keyof T] = error;
        isValid = false;
      }
    }
    setErrors(newErrors);
    return isValid;
  }, [values, rules]);

  const reset = useCallback(() => {
    setValues(initialValues);
    setErrors({});
  }, [initialValues]);

  return { values, errors, handleChange, validate, reset };
}

// Usage:
// const { values, errors, handleChange, validate } = useForm(
//   { email: '', password: '' },
//   {
//     email: (v) => !v.includes('@') ? 'Invalid email address' : null,
//     password: (v) => v.length < 8 ? 'Password must be at least 8 characters' : null,
//   }
// );
```

## Pattern 3: Device Information with `useDeviceInfo`

Mobile apps frequently need to adapt their layout based on screen size or platform. Wrapping this logic in a hook makes responsive design straightforward.

```typescript
// hooks/useDeviceInfo.ts
import { useState, useEffect } from 'react';
import { Dimensions, Platform } from 'react-native';

interface DeviceInfo {
  width: number;
  height: number;
  isTablet: boolean;
  platform: 'ios' | 'android' | 'web';
}

export function useDeviceInfo(): DeviceInfo {
  const [dimensions, setDimensions] = useState(() => Dimensions.get('window'));

  useEffect(() => {
    const subscription = Dimensions.addEventListener('change', ({ window }) => {
      setDimensions(window);
    });
    return () => subscription.remove();
  }, []);

  return {
    width: dimensions.width,
    height: dimensions.height,
    isTablet: dimensions.width >= 768,
    platform: Platform.OS as 'ios' | 'android' | 'web',
  };
}

// Usage:
// const { width, isTablet, platform } = useDeviceInfo();
// const columns = isTablet ? 3 : 2;
```

## Pattern 4: Generic Async Operations with `useAsync`

Beyond API calls, mobile apps are full of async operations — saving to AsyncStorage, resizing images, reading files. A generic `useAsync` hook gives you a unified interface for any asynchronous task.

```typescript
// hooks/useAsync.ts
import { useState, useCallback } from 'react';

interface UseAsyncResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  execute: (...args: any[]) => Promise<T | null>;
}

export function useAsync<T>(asyncFn: (...args: any[]) => Promise<T>): UseAsyncResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const execute = useCallback(async (...args: any[]) => {
    setLoading(true);
    setError(null);
    try {
      const result = await asyncFn(...args);
      setData(result);
      return result;
    } catch (err) {
      const message = err instanceof Error ? err.message : 'Operation failed';
      setError(message);
      return null;
    } finally {
      setLoading(false);
    }
  }, [asyncFn]);

  return { data, loading, error, execute };
}

// Usage:
// const { data: savedItem, loading, execute: saveItem } = useAsync(
//   async (key: string, value: string) => {
//     await AsyncStorage.setItem(key, value);
//     return { key, value, savedAt: new Date() };
//   }
// );
// await saveItem('user_prefs', JSON.stringify(prefs));
```

## Tips for Designing Better Custom Hooks

Here are a few principles that will help you write hooks that stand the test of time.

**One hook, one job.** A `useFetch` hook shouldn't also handle form validation. Keep each hook focused on a single responsibility — it makes testing easier and composition more natural.

**Be specific with names.** `useData` tells you nothing. `useFetch`, `useUserProfile`, `useGeolocation` — names like these communicate intent instantly, both to your future self and to anyone reading your code.

**Lean into TypeScript generics.** Generics let you build hooks that are both type-safe and flexible. The `useFetch<T>` pattern above is a perfect example: one hook that works with any data shape while still giving you full autocomplete.

**Always clean up.** If your `useEffect` registers event listeners, timers, or subscriptions, return a cleanup function. Failing to do so is one of the most common sources of memory leaks in React Native apps.

## Frequently Asked Questions

### Can I add custom hooks to code generated by Rork?

Absolutely. Rork generates standard React Native + Expo projects, so you can create a `hooks/` directory and add custom hooks just like you would in any React project. The most reliable workflow is to export your project using Rork Max and then add your hooks to the exported codebase.

### How do custom hooks differ from React Context?

Custom hooks are about **reusing logic** — the same hook used in two components maintains independent state in each. Context is about **sharing data** across components without prop drilling. In practice, you'll often use both together: a Context provider that uses custom hooks internally, then exposes shared state to the rest of the app.

### How do I test custom hooks?

Use the `renderHook` utility from `@testing-library/react-hooks` (or React Native Testing Library). This lets you test hooks in isolation without mounting a full component tree — one of the biggest advantages of extracting logic into hooks.

```typescript
import { renderHook, act } from '@testing-library/react-hooks';
import { useCounter } from './useCounter';

test('increment increases count by 1', () => {
  const { result } = renderHook(() => useCounter(0));
  act(() => { result.current.increment(); });
  expect(result.current.count).toBe(1);
});
```
