## Introduction — Why We All Need a Subscription Manager

Netflix, Spotify, Adobe, iCloud, gym memberships… Subscription services have a way of quietly multiplying until you're spending far more than you realize. Studies suggest the average person maintains 12–15 active subscriptions, and roughly 30% of those are services they've forgotten about but never cancelled.

In this tutorial, we'll build a subscription management app from scratch using Rork. Here's what the finished app will include:

- Add, edit, and delete subscription entries
- Visualize monthly and yearly spending with pie and bar charts
- Receive renewal reminder notifications
- Filter subscriptions by category

This guide is aimed at developers who are comfortable with Rork basics and ready to tackle an intermediate-level project. No prior React Native or Expo experience is required, though familiarity with these tools will help.

## Step 1 — Designing the Data Model

The first step is defining the data structure for our subscriptions. Enter the following prompt in Rork's AI editor:

```
Create a data model for a subscription management app.
Each subscription should include: name, amount, currency,
billing cycle (monthly/yearly), category, next billing date,
and a theme color.
```

Rork will generate a TypeScript type definition similar to this:

```typescript
// Subscription data type
type Subscription = {
  id: string;
  name: string;           // Service name (e.g., "Netflix")
  amount: number;         // Price
  currency: "JPY" | "USD"; // Currency
  cycle: "monthly" | "yearly"; // Billing cycle
  category: string;       // Category (e.g., "Entertainment", "Work")
  nextBillingDate: string; // Next billing date (ISO 8601)
  color: string;          // Theme color (e.g., "#E50914")
  isActive: boolean;      // Active/paused flag
  createdAt: string;
};
// Expected usage:
// { id: "1", name: "Netflix", amount: 15.49, currency: "USD",
//   cycle: "monthly", category: "Entertainment", nextBillingDate: "2026-04-15",
//   color: "#E50914", isActive: true, createdAt: "2026-03-29" }
```

This data model forms the foundation of the entire app. While categories can accept free-form input, providing defaults like "Entertainment," "Work," "Lifestyle," "Learning," and "Other" improves the user experience.

## Step 2 — Building the Home Screen

The home screen displays the total monthly spending and a scrollable list of subscriptions. Use this prompt:

```
Create a home screen that shows the total monthly spending
prominently at the top, with a scrollable card list of
subscriptions below. Each card should display the service name,
amount, next billing date, and a category badge.
```

Pay close attention to currency formatting — Japanese yen uses no decimals, while US dollars need two decimal places.

```typescript
// Currency formatting helper
const formatCurrency = (amount: number, currency: string): string => {
  return new Intl.NumberFormat(
    currency === "JPY" ? "ja-JP" : "en-US",
    { style: "currency", currency }
  ).format(amount);
};

// Usage
console.log(formatCurrency(1490, "JPY")); // → "¥1,490"
console.log(formatCurrency(9.99, "USD")); // → "$9.99"
```

When calculating the monthly total, remember to divide yearly subscriptions by 12 for a proper monthly estimate.

```typescript
// Calculate total monthly spending
const calculateMonthlyTotal = (subs: Subscription[]): number => {
  return subs
    .filter((s) => s.isActive)
    .reduce((total, sub) => {
      const monthly = sub.cycle === "yearly"
        ? sub.amount / 12
        : sub.amount;
      return total + monthly;
    }, 0);
};
// Expected output:
// Netflix $15.49 (monthly) + Adobe $239.88 (yearly → $19.99/mo) = $35.48
```

## Step 4 — Visualizing Spending Data

Data visualization is the heart of this app. Build the analytics screen with:

```
Create a spending analysis screen with:
1. A pie chart showing spending breakdown by category
2. A bar chart showing spending trends over the last 6 months
3. Numeric displays for monthly total and annual estimate.
```

For charting in React Native, popular libraries include `react-native-chart-kit` and `victory-native`. Rork automatically selects an appropriate library, but you can specify one explicitly if you prefer.

```typescript
// Transform category data for pie chart
const getCategoryChartData = (subs: Subscription[]) => {
  const categoryMap = new Map<string, number>();

  subs.filter((s) => s.isActive).forEach((sub) => {
    const monthly = sub.cycle === "yearly"
      ? sub.amount / 12
      : sub.amount;
    const current = categoryMap.get(sub.category) || 0;
    categoryMap.set(sub.category, current + monthly);
  });

  return Array.from(categoryMap.entries()).map(([name, amount]) => ({
    name,
    amount: Math.round(amount * 100) / 100,
    color: CATEGORY_COLORS[name] || "#999999",
  }));
};

// Expected output:
// [
//   { name: "Entertainment", amount: 25.48, color: "#FF6384" },
//   { name: "Work",          amount: 39.98, color: "#36A2EB" },
//   { name: "Learning",      amount: 12.99, color: "#FFCE56" },
// ]
```

Consider adding a "Top 3 Most Expensive Subscriptions" section and a "Next Month's Projected Spending" estimate for extra utility.

## Step 5 — Setting Up Renewal Reminders

Push notifications that alert users before a subscription renews add significant value to the app.

```
Implement local notifications that fire 3 days before each
subscription's next billing date. Include the service name
and amount in the notification.
```

Using Expo's `expo-notifications`, you can schedule local reminders like this:

```typescript
import * as Notifications from "expo-notifications";

// Schedule a renewal reminder
const scheduleReminder = async (sub: Subscription) => {
  const billingDate = new Date(sub.nextBillingDate);
  const reminderDate = new Date(billingDate);
  reminderDate.setDate(reminderDate.getDate() - 3); // 3 days before

  // Skip if the reminder date has already passed
  if (reminderDate <= new Date()) return;

  await Notifications.scheduleNotificationAsync({
    content: {
      title: "Subscription Renewal Reminder",
      body: `${sub.name} renews soon (${formatCurrency(sub.amount, sub.currency)})`,
    },
    trigger: { date: reminderDate },
  });
};
// Expected behavior:
// Netflix (renews Apr 15) → Notification on Apr 12:
// "Netflix renews soon ($15.49)"
```

A best practice is to request notification permissions right after the user adds their first subscription, rather than on initial launch. Users are more likely to grant permission when they understand the context.

## Step 6 — Category Filters and Sorting

As the subscription list grows, filtering and sorting become essential.

```
Add a category filter (horizontal scrolling chip UI) and sort
options (by amount, by renewal date, by name) to the home screen
subscription list.
```

A horizontal chip bar at the top of the screen is the most intuitive pattern. Tapping "All" or any specific category chip instantly filters the list.

```typescript
// Apply filter and sort
const getFilteredSubscriptions = (
  subs: Subscription[],
  category: string | null,
  sortBy: "amount" | "date" | "name"
): Subscription[] => {
  let filtered = category
    ? subs.filter((s) => s.category === category)
    : subs;

  return filtered.sort((a, b) => {
    switch (sortBy) {
      case "amount":
        return b.amount - a.amount; // Highest first
      case "date":
        return new Date(a.nextBillingDate).getTime()
             - new Date(b.nextBillingDate).getTime(); // Soonest first
      case "name":
        return a.name.localeCompare(b.name); // Alphabetical
      default:
        return 0;
    }
  });
};
```

## Common Errors and How to Fix Them

Here are some issues you might encounter during development:

**Date picker shows wrong dates due to timezone**: `new Date()` uses the device's local timezone, but parsing ISO strings may default to UTC. Use `toLocaleDateString()` or `date-fns`'s `parseISO` for consistent date handling.

**Notifications not appearing (iOS)**: iOS requires explicit notification permission. Check the current status with `Notifications.getPermissionsAsync()` and call `requestPermissionsAsync()` if it hasn't been granted.

**Charts render blank**: Chart components may throw errors when the data array is empty. Always add a conditional check to display a placeholder message like "No subscriptions added yet" when there's no data.

## FAQ

**Q: Is there a storage limit for AsyncStorage?**

A: The recommended limit for AsyncStorage is around 6 MB. Since each subscription entry is only a few hundred bytes, you could store thousands of entries without any issues. For more robust data persistence, consider integrating with Supabase.

**Q: Can I fetch exchange rates to display everything in one currency?**

A: Absolutely. You can use a free API like [exchangerate-api.com](https://www.exchangerate-api.com/) to fetch daily rates and cache them locally. Free plans have request limits, so caching rates for 24 hours is a practical approach.

**Q: Can I add a feature to share subscriptions with family or a partner?**

A: Yes — by leveraging Supabase's Row Level Security (RLS), you can implement shared access between specific users. Create a "shared group" concept where members of the same group can view each other's subscription data.

**Q: What do I need to publish this app on the App Store?**

A: With Rork Max, you can submit to the App Store in just two clicks. You'll need an Apple Developer Program membership ($99/year). For a detailed review checklist, see [The Complete App Store Review Guide for Rork Apps](/en/articles/rork-business/rork-app-store-review-complete-checklist).
