For solo app development, Supabase and Firebase have become the default — and for good reason. But once you step into client work or business apps, the landscape shifts. "We need this on our existing AWS infrastructure." "We're required to use Cognito for user management." "HIPAA compliance is non-negotiable." If any of those sound familiar, this article is for you.

I've spent time connecting Rork Max projects to AWS Amplify for enterprise clients, and the biggest friction is always the same: official docs are written for JavaScript SDK users, and translating them to React Native / Expo takes more trial and error than it should. This guide skips that friction and gives you the direct path.

By the end, you'll have a working Rork Max app with Cognito authentication, AppSync GraphQL data management, and S3 file storage — along with the practical knowledge to adapt it for your specific requirements.

## Why AWS Amplify Instead of Firebase or Supabase?

Honest answer: if you're starting a greenfield project for yourself, Supabase is probably the better choice. It's faster to set up, the Row Level Security model is intuitive, and it integrates smoothly with Rork.

That said, there are three scenarios where Amplify makes sense:

**① You're connecting to existing AWS infrastructure**
When a client's backend team already manages users through Cognito User Pools, running a separate auth system alongside it creates more problems than it solves. Unifying on Cognito keeps everything in one place.

**② Compliance requirements specify AWS**
Healthcare (HIPAA), financial services, and government work often mandate that data stays within specific AWS regions and services. Amplify is the natural fit here.

**③ You need enterprise-scale real-time features**
AppSync handles tens of thousands of simultaneous WebSocket connections gracefully. Combined with DynamoDB auto-scaling, you get infrastructure that doesn't break a sweat under serious load.

## Gen 1 vs Gen 2 — Use Gen 2

AWS Amplify Gen 2 reached general availability in 2024 and changed the development model significantly. Gen 1 used a CLI-driven CloudFormation approach that was frustrating to debug. Gen 2 takes a TypeScript-first, CDK-like approach:

```typescript
// amplify/backend.ts (Gen 2 entry point)
import { defineBackend } from "@aws-amplify/backend";
import { auth } from "./auth/resource";
import { data } from "./data/resource";
import { storage } from "./storage/resource";

// Declare all backend resources in TypeScript
export const backend = defineBackend({
  auth,
  data,
  storage,
});
```

Everything is version-controlled, changes are auditable, and local sandbox environments deploy in under two minutes. There's no good reason to start with Gen 1 for a new project.

### Initialize Amplify in Your App Layout

Add this to the top of `app/_layout.tsx`:

```typescript
// app/_layout.tsx
import { Amplify } from "aws-amplify";
import outputs from "../amplify_outputs.json"; // Generated after first deploy
import "core-js/stable"; // Polyfill for Hermes engine

// Initialize once at app startup
Amplify.configure(outputs, {
  Storage: {
    S3: {
      prefixResolver: async ({ accessLevel, targetIdentityId }) => {
        if (accessLevel === "guest") return "public/";
        if (accessLevel === "protected") return `protected/${targetIdentityId}/`;
        return `private/${targetIdentityId}/`;
      },
    },
  },
});

  // Rork Max generated layout code continues here
}
```

### Auth Hook

```typescript
// hooks/useAuth.ts
import { useState, useEffect } from "react";
import {
  signIn,
  signOut,
  signUp,
  confirmSignUp,
  getCurrentUser,
  fetchUserAttributes,
  AuthError,
} from "aws-amplify/auth";

type AuthUser = {
  userId: string;
  username: string;
  email: string;
  plan?: string;
};

export function useAuth() {
  const [user, setUser] = useState<AuthUser | null>(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    checkCurrentUser();
  }, []);

  async function checkCurrentUser() {
    try {
      const { userId, username } = await getCurrentUser();
      const attributes = await fetchUserAttributes();
      setUser({
        userId,
        username,
        email: attributes.email ?? "",
        plan: attributes["custom:plan"],
      });
    } catch {
      setUser(null);
    } finally {
      setIsLoading(false);
    }
  }

  async function handleSignIn(email: string, password: string) {
    setError(null);
    try {
      const result = await signIn({ username: email, password });

      if (result.isSignedIn) {
        await checkCurrentUser();
        return { success: true };
      }

      if (result.nextStep.signInStep === "CONFIRM_SIGN_IN_WITH_TOTP_CODE") {
        return { success: false, needsMFA: true };
      }

      return { success: false };
    } catch (err) {
      const authError = err as AuthError;

      // Map Cognito error codes to user-friendly messages
      const messages: Record<string, string> = {
        NotAuthorizedException: "Incorrect email or password",
        UserNotFoundException: "No account found with this email",
        UserNotConfirmedException: "Please verify your email address first",
        TooManyRequestsException: "Too many attempts. Please try again later",
      };

      const message = messages[authError.name] ?? authError.message;
      setError(message);
      return { success: false, error: message };
    }
  }

  async function handleSignUp(email: string, password: string) {
    setError(null);
    try {
      const { nextStep } = await signUp({
        username: email,
        password,
        options: { userAttributes: { email } },
      });
      return {
        success: true,
        needsConfirmation: nextStep.signUpStep === "CONFIRM_SIGN_UP",
      };
    } catch (err) {
      const authError = err as AuthError;
      const message =
        authError.name === "UsernameExistsException"
          ? "An account with this email already exists"
          : authError.message;
      setError(message);
      return { success: false, error: message };
    }
  }

  async function handleSignOut() {
    await signOut();
    setUser(null);
  }

  return {
    user,
    isLoading,
    error,
    isAuthenticated: \!\!user,
    handleSignIn,
    handleSignUp,
    handleSignOut,
  };
}
```

## Step 3: AppSync GraphQL API

AppSync is AWS's managed GraphQL service backed by DynamoDB, with native real-time subscription support.

### Define Your Data Schema

```typescript
// amplify/data/resource.ts
import { type ClientSchema, a, defineData } from "@aws-amplify/backend";

const schema = a.schema({
  Todo: a
    .model({
      title: a.string().required(),
      description: a.string(),
      isCompleted: a.boolean().default(false),
      priority: a.enum(["LOW", "MEDIUM", "HIGH"]),
      dueDate: a.datetime(),
      owner: a.string(),
    })
    .authorization((allow) => [
      allow.owner(), // Users can only access their own todos
      allow.groups(["Admin"]).to(["read"]),
    ]),

  UserProfile: a
    .model({
      displayName: a.string().required(),
      avatarUrl: a.string(),
      bio: a.string(),
      todoCount: a.integer().default(0),
    })
    .authorization((allow) => [
      allow.owner(),
      allow.authenticated().to(["read"]),
    ]),
});

export type Schema = ClientSchema<typeof schema>;

export const data = defineData({
  schema,
  authorizationModes: {
    defaultAuthorizationMode: "userPool",
    apiKeyAuthorizationMode: {
      expiresInDays: 30,
    },
  },
});
```

### Type-Safe Data Hook with Real-Time Updates

```typescript
// hooks/useTodos.ts
import { useState, useEffect } from "react";
import { generateClient } from "aws-amplify/data";
import type { Schema } from "@/amplify/data/resource";

const client = generateClient<Schema>();

export function useTodos() {
  const [todos, setTodos] = useState<Schema["Todo"]["type"][]>([]);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    // observeQuery gives you both initial data and real-time updates
    const subscription = client.models.Todo.observeQuery({
      filter: { isCompleted: { eq: false } },
      sort: [{ field: "priority", direction: "DESC" }],
    }).subscribe({
      next: ({ items, isSynced }) => {
        setTodos(items);
        if (isSynced) setIsLoading(false);
      },
      error: (err) => {
        console.error("Failed to fetch todos:", err);
        setIsLoading(false);
      },
    });

    return () => subscription.unsubscribe();
  }, []);

  async function createTodo(input: {
    title: string;
    description?: string;
    priority?: "LOW" | "MEDIUM" | "HIGH";
    dueDate?: Date;
  }) {
    try {
      const { data, errors } = await client.models.Todo.create({
        title: input.title,
        description: input.description,
        priority: input.priority ?? "MEDIUM",
        dueDate: input.dueDate?.toISOString(),
        isCompleted: false,
      });

      if (errors) {
        throw new Error(errors.map((e) => e.message).join(", "));
      }

      return { success: true, todo: data };
    } catch (err) {
      console.error("Failed to create todo:", err);
      return { success: false, error: String(err) };
    }
  }

  async function toggleTodo(id: string, currentState: boolean) {
    const { data } = await client.models.Todo.update({
      id,
      isCompleted: \!currentState,
    });
    return data;
  }

  async function deleteTodo(id: string) {
    await client.models.Todo.delete({ id });
  }

  return { todos, isLoading, createTodo, toggleTodo, deleteTodo };
}
```

### Real-Time Subscriptions for Collaborative Features

When you need live updates across users (chat, collaborative editing, shared dashboards):

```typescript
useEffect(() => {
  // Subscribe to todos created by anyone (useful in shared workspaces)
  const createSub = client.models.Todo.onCreate().subscribe({
    next: (data) => {
      setTodos((prev) => [...prev, data]);
    },
    error: (error) => console.error("Subscription error:", error),
  });

  const updateSub = client.models.Todo.onUpdate().subscribe({
    next: (data) => {
      setTodos((prev) =>
        prev.map((t) => (t.id === data.id ? data : t))
      );
    },
    error: (error) => console.error("Subscription error:", error),
  });

  return () => {
    createSub.unsubscribe();
    updateSub.unsubscribe();
  };
}, []);
```

## Step 4: S3 File Storage

Profile images, documents, exports — here's a complete upload implementation with progress tracking:

### Storage Resource

```typescript
// amplify/storage/resource.ts
import { defineStorage } from "@aws-amplify/backend";

export const storage = defineStorage({
  name: "myAppStorage",
  access: (allow) => ({
    "public/*": [
      allow.guest.to(["read"]),
      allow.authenticated.to(["read", "write", "delete"]),
    ],
    "private/{entity_id}/*": [
      allow.entity("identity").to(["read", "write", "delete"]),
    ],
    "protected/{entity_id}/*": [
      allow.authenticated.to(["read"]),
      allow.entity("identity").to(["write", "delete"]),
    ],
  }),
});
```

### Upload Hook with Progress Tracking

```typescript
// hooks/useStorage.ts
import { useState } from "react";
import { uploadData, getUrl, remove } from "aws-amplify/storage";
import * as ImagePicker from "expo-image-picker";

export function useStorage() {
  const [uploadProgress, setUploadProgress] = useState(0);
  const [isUploading, setIsUploading] = useState(false);

  async function uploadProfileImage(userId: string): Promise<string | null> {
    const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();
    if (status \!== "granted") return null;

    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,
      aspect: [1, 1],
      quality: 0.8,
    });

    if (result.canceled) return null;

    const imageUri = result.assets[0].uri;
    const fileName = `profile_${userId}_${Date.now()}.jpg`;
    const s3Key = `protected/${userId}/${fileName}`;

    setIsUploading(true);
    setUploadProgress(0);

    try {
      const response = await fetch(imageUri);
      const blob = await response.blob();

      const uploadTask = uploadData({
        path: s3Key,
        data: blob,
        options: {
          contentType: "image/jpeg",
          onProgress: ({ transferredBytes, totalBytes }) => {
            if (totalBytes) {
              setUploadProgress(Math.round((transferredBytes / totalBytes) * 100));
            }
          },
          metadata: {
            userId,
            uploadedAt: new Date().toISOString(),
          },
        },
      });

      const { path } = await uploadTask.result;

      // Get a time-limited signed URL (24 hours)
      const { url } = await getUrl({
        path,
        options: { expiresIn: 86400, validateObjectExistence: true },
      });

      return url.toString();
    } catch (err) {
      console.error("Upload failed:", err);
      return null;
    } finally {
      setIsUploading(false);
      setUploadProgress(0);
    }
  }

  async function deleteFile(path: string) {
    try {
      await remove({ path });
      return true;
    } catch (err) {
      console.error("Delete failed:", err);
      return false;
    }
  }

  return { uploadProfileImage, deleteFile, uploadProgress, isUploading };
}
```

## Step 6: Common Pitfalls and How to Avoid Them

These are the issues I've hit — often multiple times — when working with Amplify on React Native/Expo projects.

**Pitfall #1: Hermes Engine Incompatibility**
Rork Max projects use Hermes by default since Expo SDK 50. Amplify packages that rely on `TextDecoder`, `crypto`, or `atob/btoa` will fail silently or throw at runtime.

```bash
npm install --save react-native-get-random-values core-js
```

```typescript
// Add to the very top of app/_layout.tsx, before all other imports
import "react-native-get-random-values";
import "core-js/stable/atob";
import "core-js/stable/btoa";
import "@aws-amplify/pubsub"; // Required for real-time subscriptions
```

**Pitfall #2: amplify_outputs.json Missing from EAS Builds**
EAS Build doesn't include `.gitignore`d files. Fix this in `eas.json`:

```json
{
  "build": {
    "production": {
      "secretFiles": ["amplify_outputs.json"]
    }
  }
}
```

Upload the file content as a secret in the EAS dashboard.

**Pitfall #3: AppSync Subscriptions Unstable on iOS Simulator**
WebSocket connections on iOS Simulator can be flaky. Test subscriptions on a real device or Expo Go. Also verify that `@aws-amplify/pubsub` is imported (see Pitfall #1).

**Pitfall #4: S3 CORS Errors with Existing Buckets**
Amplify auto-configures CORS for sandbox-created buckets, but not for existing ones. In S3 Console → Bucket → Permissions → CORS:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST", "DELETE"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": ["ETag", "x-amz-server-side-encryption"]
  }
]
```

**Pitfall #5: Token Expiry After Network Recovery**
Cognito access tokens expire after 1 hour. Amplify auto-refreshes them, but the refresh can fail when coming back from a no-network state:

```typescript
import NetInfo from "@react-native-community/netinfo";
import { fetchAuthSession } from "aws-amplify/auth";

useEffect(() => {
  const unsubscribe = NetInfo.addEventListener(async (state) => {
    if (state.isConnected) {
      try {
        await fetchAuthSession({ forceRefresh: true });
      } catch {
        // Session expired — redirect to login
        router.replace("/(auth)/sign-in");
      }
    }
  });
  return unsubscribe;
}, []);
```

**Pitfall #6: N+1 Queries with AppSync Relations**
Naive relational data fetching creates N+1 queries. Use `belongsTo` / `hasMany` in your schema and `selectionSet` when querying:

```typescript
// Define relations properly in schema first
const schema = a.schema({
  Post: a.model({
    title: a.string().required(),
    author: a.belongsTo("UserProfile", "authorId"),
    authorId: a.id(),
    comments: a.hasMany("Comment", "postId"),
  })
  .authorization((allow) => [allow.authenticated()]),
});

// Then fetch with a single request using selectionSet
const { data } = await client.models.Post.list({
  selectionSet: [
    "id",
    "title",
    "author.displayName",
    "author.avatarUrl",
    "comments.id",
    "comments.content",
  ],
});
```

## Step 7: Production IAM Security Design

Amplify auto-generates reasonable IAM roles, but for production you'll want to tighten them. The principle of least privilege applies:

```typescript
// amplify/backend.ts — lock down S3 write access
import { defineBackend } from "@aws-amplify/backend";
import { auth } from "./auth/resource";
import { data } from "./data/resource";
import { storage } from "./storage/resource";
import { PolicyStatement, Effect, ServicePrincipal } from "aws-cdk-lib/aws-iam";

const backend = defineBackend({ auth, data, storage });

// Deny writes to any path outside the user's own private prefix
backend.storage.resources.bucket.addToResourcePolicy(
  new PolicyStatement({
    effect: Effect.DENY,
    principals: [new ServicePrincipal("cognito-identity.amazonaws.com")],
    actions: ["s3:PutObject"],
    notResources: [
      `${backend.storage.resources.bucket.bucketArn}/private/\${cognito-identity.amazonaws.com:sub}/*`,
    ],
  })
);
```

For production Cognito pools, also consider:
- Enabling advanced security features (compromised credential detection)
- Setting up WAF rules on the Cognito endpoint
- Configuring CloudWatch alarms on failed authentication attempts

## What's Next

Getting Cognito auth running in Rork Max is the hardest part — once that's working, AppSync and S3 fall into place quickly. The sandbox environment is your best friend during development: `npx ampx sandbox` keeps everything in sync as you iterate.

If you're integrating with an existing AWS environment rather than starting fresh, the key is getting the `amplify_outputs.json` right. You can generate this file from an existing Cognito User Pool and AppSync API instead of creating new ones — the Amplify docs cover the "bring your own backend" path if you need it.

The AWS ecosystem is broad, but Amplify Gen 2 gives you a well-lit path through it. Start small, get auth working, and add the pieces you need from there.
