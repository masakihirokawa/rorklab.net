## visionOS Fundamentals

Before diving into development, it's worth understanding concepts unique to visionOS.

### visionOS UI Hierarchy

```
visionOS UI Hierarchy
──────────────────────────────
Shared Space
  ├── Window (2D window)
  ├── Volume (3D volume)
  └── Full Space (fully immersive)
       └── Reality (real-world integration)
```

Many standard iOS SwiftUI apps work on visionOS with minimal changes. Adding 3D elements (Volumes and Immersive Scenes) unlocks the truly immersive experience that makes Vision Pro special.

### Key Differences: iOS vs visionOS

```
Feature         iOS              visionOS
──────────────────────────────────────────────
Touch input     Tap              Eye tracking + pinch
Windows         Screen-fixed     Float in space
3D elements     Limited          RealityKit native
Depth sensor    FaceID           LiDAR (Pro)
```

## Implementing Spatial Audio

Spatial audio is one of visionOS's standout features — sounds appear to come from specific locations in 3D space.

```swift
// SpatialAudioManager.swift
import RealityKit
import AVFoundation

class SpatialAudioManager {
    
    /**
     * Play audio from a specific position in 3D space
     * - position: Sound source position (spatial coordinates)
     * - soundURL: Audio file to play
     */
    func playAt(position: SIMD3<Float>, soundURL: URL) {
        let audioEntity = Entity()
        audioEntity.position = position
        
        let audioResource = try? AudioFileResource.load(
            contentsOf: soundURL,
            configuration: .init()
        )
        
        if let resource = audioResource {
            audioEntity.playAudio(resource)
        }
    }
    
    /**
     * Play ambient sound from 1 meter in front of user
     */
    func playAmbientSound(soundURL: URL) {
        playAt(
            position: SIMD3<Float>(0, 0, -1.0),  // 1m in front
            soundURL: soundURL
        )
    }
}
```

## FAQ

**Q: Can I develop visionOS apps without owning a Vision Pro?**

Yes. Xcode's visionOS simulator handles most development. A physical device is needed for final testing, but you can build and prototype with the simulator.

**Q: How accurately does Rork Max generate visionOS code?**

Window-based UIs are generated accurately. Immersive Space (fully immersive) is more complex and may require manual adjustments post-generation.

**Q: Is visionOS App Review stricter than iOS?**

Core guidelines are shared with iOS, but additional requirements apply around privacy (facial recognition, spatial data). Be especially careful with HealthKit or sensor data access.

**Q: Can I submit iOS and visionOS together?**

Yes. A Universal binary serves both platforms from a single App Store listing.

**Q: How much more development time does visionOS need vs iOS?**

visionOS-specific UI (Immersive Space, spatial audio) adds development time. With Rork Max auto-generating the base structure, expect roughly 1.3–1.5x the iOS development effort.
