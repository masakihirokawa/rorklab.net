## Understanding WebRTC Architecture

### From Browser Technology to Mobile

WebRTC (Web Real-Time Communication) is an open-source project originally developed to enable real-time communication in browsers without plugins. Since Google released it in 2011, it has become the standard technology for video calls, voice calls, and data transfer.

In React Native, the `react-native-webrtc` library provides access to native WebRTC functionality. Combined with Rork Max's AI code generation, you can efficiently build production-quality video calling apps.

### The Three Pillars of P2P Connections

WebRTC's P2P (Peer-to-Peer) connections rely on three core protocols working together.

**SDP (Session Description Protocol):** Think of this as the "contract" for a call. It communicates media capabilities like codecs, resolution, and bandwidth to the other party. The caller creates an "Offer," the receiver sends back an "Answer," and the negotiation is complete.

**ICE (Interactive Connectivity Establishment):** This is the framework for finding the optimal communication path through NATs and firewalls. It gathers multiple candidates (ICE Candidates), runs connectivity checks, and selects the most efficient route.

**STUN/TURN Servers:** STUN (Session Traversal Utilities for NAT) discovers your public IP address, while TURN (Traversal Using Relays around NAT) acts as a relay server when direct connection isn't possible.

```typescript
// Basic WebRTC connection flow
// 1. Create RTCPeerConnection
const configuration: RTCConfiguration = {
  iceServers: [
    // STUN: For discovering public IP (free)
    { urls: 'stun:stun.l.google.com:19302' },
    { urls: 'stun:stun1.l.google.com:19302' },
    // TURN: For NAT traversal relay (use your own or a service in production)
    {
      urls: 'turn:your-turn-server.example.com:3478',
      username: 'YOUR_TURN_USERNAME',
      credential: 'YOUR_TURN_CREDENTIAL',
    },
  ],
};

const peerConnection = new RTCPeerConnection(configuration);

// 2. Get local media stream and add tracks
const localStream = await mediaDevices.getUserMedia({
  audio: true,
  video: {
    facingMode: 'user', // Front camera
    width: { ideal: 1280 },
    height: { ideal: 720 },
  },
});

localStream.getTracks().forEach((track) => {
  peerConnection.addTrack(track, localStream);
});

// 3. Send ICE candidates to the remote peer via signaling
peerConnection.onicecandidate = (event) => {
  if (event.candidate) {
    sendToSignalingServer({
      type: 'ice-candidate',
      candidate: event.candidate,
    });
  }
};

// 4. Receive remote stream
peerConnection.ontrack = (event) => {
  // Set event.streams[0] as the remote video source
  setRemoteStream(event.streams[0]);
};
```

This code forms the backbone of a WebRTC connection. The key insight here is that while `RTCPeerConnection` handles the negotiation of communication paths, **the mechanism for delivering SDPs and ICE candidates to the other party (signaling) is not part of the WebRTC specification**. In the next section, we'll dive deep into signaling design patterns.

## Building a Signaling Server with Supabase Realtime

### Why Supabase Realtime Is the Right Choice

The signaling server's role is to relay SDPs and ICE candidates in real time. While there are many options — WebSocket, Firebase Realtime Database, Socket.io — **Supabase Realtime** offers the best fit within the Rork ecosystem.

Here's why. First, Supabase integrates authentication, database, and storage, giving you a consistent development experience from user management to call history storage. Second, the Realtime Broadcast feature enables instant message exchange between clients in a specific channel, delivering the low-latency communication that signaling demands. Third, combined with Row Level Security (RLS), you can protect call privacy at the database level.

### Database Schema Design

```sql
-- Table for managing call sessions
CREATE TABLE call_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  room_id TEXT NOT NULL UNIQUE,
  caller_id UUID REFERENCES auth.users(id) NOT NULL,
  callee_id UUID REFERENCES auth.users(id),
  status TEXT NOT NULL DEFAULT 'waiting'
    CHECK (status IN ('waiting', 'ringing', 'active', 'ended', 'missed')),
  started_at TIMESTAMPTZ,
  ended_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  -- Metadata (call duration, quality score, etc.)
  metadata JSONB DEFAULT '{}'::jsonb
);

-- RLS: Only participants can access
ALTER TABLE call_sessions ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Only participants can view" ON call_sessions
  FOR SELECT USING (
    auth.uid() = caller_id OR auth.uid() = callee_id
  );

CREATE POLICY "Authenticated users can initiate calls" ON call_sessions
  FOR INSERT WITH CHECK (auth.uid() = caller_id);

CREATE POLICY "Only participants can update" ON call_sessions
  FOR UPDATE USING (
    auth.uid() = caller_id OR auth.uid() = callee_id
  );
```

### Implementing the Signaling Flow

```typescript
import { createClient } from '@supabase/supabase-js';

// Signaling Manager
class SignalingManager {
  private supabase;
  private channel;
  private roomId: string;

  constructor(supabaseUrl: string, supabaseKey: string, roomId: string) {
    this.supabase = createClient(supabaseUrl, supabaseKey);
    this.roomId = roomId;
  }

  // Join channel and listen for signaling messages
  async join(userId: string, handlers: SignalingHandlers) {
    this.channel = this.supabase.channel(`call:${this.roomId}`, {
      config: {
        broadcast: { self: false }, // Don't broadcast to self
      },
    });

    this.channel
      .on('broadcast', { event: 'offer' }, ({ payload }) => {
        handlers.onOffer(payload);
      })
      .on('broadcast', { event: 'answer' }, ({ payload }) => {
        handlers.onAnswer(payload);
      })
      .on('broadcast', { event: 'ice-candidate' }, ({ payload }) => {
        handlers.onIceCandidate(payload);
      })
      .on('broadcast', { event: 'hangup' }, ({ payload }) => {
        handlers.onHangup(payload);
      })
      .subscribe();
  }

  // Send SDP Offer
  async sendOffer(offer: RTCSessionDescriptionInit) {
    await this.channel.send({
      type: 'broadcast',
      event: 'offer',
      payload: { sdp: offer },
    });
  }

  // Send SDP Answer
  async sendAnswer(answer: RTCSessionDescriptionInit) {
    await this.channel.send({
      type: 'broadcast',
      event: 'answer',
      payload: { sdp: answer },
    });
  }

  // Send ICE candidate
  async sendIceCandidate(candidate: RTCIceCandidate) {
    await this.channel.send({
      type: 'broadcast',
      event: 'ice-candidate',
      payload: { candidate },
    });
  }

  // End call
  async hangup(userId: string) {
    await this.channel.send({
      type: 'broadcast',
      event: 'hangup',
      payload: { userId },
    });
    this.channel.unsubscribe();
  }
}

// Handler type definitions
interface SignalingHandlers {
  onOffer: (payload: { sdp: RTCSessionDescriptionInit }) => void;
  onAnswer: (payload: { sdp: RTCSessionDescriptionInit }) => void;
  onIceCandidate: (payload: { candidate: RTCIceCandidate }) => void;
  onHangup: (payload: { userId: string }) => void;
}
```

This signaling manager uses Supabase's Broadcast feature to create a channel per call room, relaying SDPs and ICE candidates in real time. The `self: false` setting prevents the loop where your own messages are received by yourself.

## Building the Video Call UI with Rork Max

### Component Architecture

The video call screen consists of multiple layers. Leveraging Rork Max's AI code generation for the UI while manually optimizing the parts that directly affect call quality is the most effective approach.

```typescript
import React, { useState, useEffect, useRef, useCallback } from 'react';
import {
  View,
  StyleSheet,
  TouchableOpacity,
  Text,
  Dimensions,
  Animated,
} from 'react-native';
import {
  RTCView,
  mediaDevices,
  RTCPeerConnection,
  RTCSessionDescription,
  RTCIceCandidate,
} from 'react-native-webrtc';

const { width: SCREEN_WIDTH, height: SCREEN_HEIGHT } = Dimensions.get('window');

interface VideoCallScreenProps {
  roomId: string;
  userId: string;
  isCaller: boolean;
  onCallEnd: () => void;
}

export const VideoCallScreen: React.FC<VideoCallScreenProps> = ({
  roomId,
  userId,
  isCaller,
  onCallEnd,
}) => {
  const [localStream, setLocalStream] = useState<MediaStream | null>(null);
  const [remoteStream, setRemoteStream] = useState<MediaStream | null>(null);
  const [isMuted, setIsMuted] = useState(false);
  const [isCameraOff, setIsCameraOff] = useState(false);
  const [isFrontCamera, setIsFrontCamera] = useState(true);
  const [callDuration, setCallDuration] = useState(0);
  const [connectionState, setConnectionState] = useState<string>('new');

  const peerConnection = useRef<RTCPeerConnection | null>(null);
  const signalingManager = useRef<SignalingManager | null>(null);
  const durationTimer = useRef<NodeJS.Timeout | null>(null);

  // Format call duration
  const formatDuration = useCallback((seconds: number): string => {
    const mins = Math.floor(seconds / 60);
    const secs = seconds % 60;
    return `${mins.toString().padStart(2, '0')}:${secs
      .toString()
      .padStart(2, '0')}`;
  }, []);

  useEffect(() => {
    initializeCall();
    return () => {
      cleanup();
    };
  }, []);

  const initializeCall = async () => {
    try {
      // Get local media
      const stream = await mediaDevices.getUserMedia({
        audio: {
          echoCancellation: true,
          noiseSuppression: true,
          autoGainControl: true,
        },
        video: {
          facingMode: 'user',
          width: { ideal: 1280 },
          height: { ideal: 720 },
          frameRate: { ideal: 30, max: 30 },
        },
      });
      setLocalStream(stream);

      // Initialize PeerConnection
      const pc = new RTCPeerConnection({
        iceServers: [
          { urls: 'stun:stun.l.google.com:19302' },
          { urls: 'stun:stun1.l.google.com:19302' },
        ],
      });

      // Add tracks
      stream.getTracks().forEach((track) => {
        pc.addTrack(track, stream);
      });

      // Receive remote stream
      pc.ontrack = (event) => {
        if (event.streams && event.streams[0]) {
          setRemoteStream(event.streams[0]);
          startDurationTimer();
        }
      };

      // Monitor connection state
      pc.onconnectionstatechange = () => {
        setConnectionState(pc.connectionState);
        if (
          pc.connectionState === 'disconnected' ||
          pc.connectionState === 'failed'
        ) {
          handleCallEnd();
        }
      };

      peerConnection.current = pc;

      // Initialize signaling
      const sm = new SignalingManager(
        'YOUR_SUPABASE_URL',
        'YOUR_SUPABASE_ANON_KEY',
        roomId
      );

      // Send ICE candidates
      pc.onicecandidate = (event) => {
        if (event.candidate) {
          sm.sendIceCandidate(event.candidate);
        }
      };

      await sm.join(userId, {
        onOffer: async ({ sdp }) => {
          await pc.setRemoteDescription(new RTCSessionDescription(sdp));
          const answer = await pc.createAnswer();
          await pc.setLocalDescription(answer);
          sm.sendAnswer(answer);
        },
        onAnswer: async ({ sdp }) => {
          await pc.setRemoteDescription(new RTCSessionDescription(sdp));
        },
        onIceCandidate: async ({ candidate }) => {
          await pc.addIceCandidate(new RTCIceCandidate(candidate));
        },
        onHangup: () => {
          handleCallEnd();
        },
      });

      signalingManager.current = sm;

      // Create Offer if caller
      if (isCaller) {
        const offer = await pc.createOffer({
          offerToReceiveAudio: true,
          offerToReceiveVideo: true,
        });
        await pc.setLocalDescription(offer);
        sm.sendOffer(offer);
      }
    } catch (error) {
      console.error('Failed to initialize call:', error);
    }
  };

  const startDurationTimer = () => {
    durationTimer.current = setInterval(() => {
      setCallDuration((prev) => prev + 1);
    }, 1000);
  };

  const toggleMute = () => {
    if (localStream) {
      localStream.getAudioTracks().forEach((track) => {
        track.enabled = !track.enabled;
      });
      setIsMuted(!isMuted);
    }
  };

  const toggleCamera = () => {
    if (localStream) {
      localStream.getVideoTracks().forEach((track) => {
        track.enabled = !track.enabled;
      });
      setIsCameraOff(!isCameraOff);
    }
  };

  const switchCamera = () => {
    if (localStream) {
      localStream.getVideoTracks().forEach((track) => {
        // @ts-ignore — React Native WebRTC specific method
        track._switchCamera();
      });
      setIsFrontCamera(!isFrontCamera);
    }
  };

  const handleCallEnd = () => {
    signalingManager.current?.hangup(userId);
    cleanup();
    onCallEnd();
  };

  const cleanup = () => {
    if (durationTimer.current) {
      clearInterval(durationTimer.current);
    }
    localStream?.getTracks().forEach((track) => track.stop());
    peerConnection.current?.close();
  };

  return (
    <View style={styles.container}>
      {/* Remote video (full screen) */}
      {remoteStream ? (
        <RTCView
          streamURL={remoteStream.toURL()}
          style={styles.remoteVideo}
          objectFit="cover"
          zOrder={0}
        />
      ) : (
        <View style={styles.waitingContainer}>
          <Text style={styles.waitingText}>Connecting...</Text>
          <Text style={styles.connectionState}>{connectionState}</Text>
        </View>
      )}

      {/* Local video (picture-in-picture) */}
      {localStream && (
        <RTCView
          streamURL={localStream.toURL()}
          style={styles.localVideo}
          objectFit="cover"
          mirror={isFrontCamera}
          zOrder={1}
        />
      )}

      {/* Call duration */}
      {remoteStream && (
        <View style={styles.durationContainer}>
          <Text style={styles.durationText}>
            {formatDuration(callDuration)}
          </Text>
        </View>
      )}

      {/* Control bar */}
      <View style={styles.controlBar}>
        <TouchableOpacity
          style={[styles.controlButton, isMuted && styles.activeButton]}
          onPress={toggleMute}
        >
          <Text style={styles.controlIcon}>{isMuted ? '🔇' : '🔊'}</Text>
          <Text style={styles.controlLabel}>
            {isMuted ? 'Unmute' : 'Mute'}
          </Text>
        </TouchableOpacity>

        <TouchableOpacity
          style={[styles.controlButton, styles.hangupButton]}
          onPress={handleCallEnd}
        >
          <Text style={styles.hangupIcon}>📞</Text>
          <Text style={styles.controlLabel}>End</Text>
        </TouchableOpacity>

        <TouchableOpacity
          style={[styles.controlButton, isCameraOff && styles.activeButton]}
          onPress={toggleCamera}
        >
          <Text style={styles.controlIcon}>
            {isCameraOff ? '📷' : '📹'}
          </Text>
          <Text style={styles.controlLabel}>
            {isCameraOff ? 'Camera Off' : 'Camera'}
          </Text>
        </TouchableOpacity>

        <TouchableOpacity style={styles.controlButton} onPress={switchCamera}>
          <Text style={styles.controlIcon}>🔄</Text>
          <Text style={styles.controlLabel}>Flip</Text>
        </TouchableOpacity>
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#000',
  },
```

## Optimizing Call Quality

### Adaptive Bitrate Control

In mobile environments, network conditions are constantly changing. Switching from Wi-Fi to cellular, moving through areas with weak signals — bandwidth is always in flux. While WebRTC has built-in adaptive bitrate capabilities, additional optimization can dramatically improve the experience.

```typescript
// Dynamic video parameter adjustment based on network quality
class QualityManager {
  private peerConnection: RTCPeerConnection;
  private currentQuality: 'high' | 'medium' | 'low' = 'high';

  // Video quality presets
  private readonly qualityPresets = {
    high: {
      maxBitrate: 2500000, // 2.5 Mbps
      maxFramerate: 30,
      scaleResolutionDownBy: 1,
    },
    medium: {
      maxBitrate: 1000000, // 1 Mbps
      maxFramerate: 24,
      scaleResolutionDownBy: 1.5,
    },
    low: {
      maxBitrate: 400000, // 400 Kbps
      maxFramerate: 15,
      scaleResolutionDownBy: 2,
    },
  };

  constructor(pc: RTCPeerConnection) {
    this.peerConnection = pc;
  }

  // Monitor network stats and auto-adjust quality
  async startMonitoring(intervalMs: number = 3000) {
    setInterval(async () => {
      const stats = await this.peerConnection.getStats();
      let packetsLost = 0;
      let packetsReceived = 0;
      let roundTripTime = 0;

      stats.forEach((report) => {
        if (report.type === 'inbound-rtp' && report.kind === 'video') {
          packetsLost = report.packetsLost || 0;
          packetsReceived = report.packetsReceived || 0;
        }
        if (report.type === 'candidate-pair' && report.state === 'succeeded') {
          roundTripTime = report.currentRoundTripTime || 0;
        }
      });

      // Determine quality based on packet loss rate and RTT
      const lossRate =
        packetsReceived > 0
          ? packetsLost / (packetsLost + packetsReceived)
          : 0;

      let targetQuality: 'high' | 'medium' | 'low';
      if (lossRate < 0.02 && roundTripTime < 0.15) {
        targetQuality = 'high';
      } else if (lossRate < 0.05 && roundTripTime < 0.3) {
        targetQuality = 'medium';
      } else {
        targetQuality = 'low';
      }

      if (targetQuality !== this.currentQuality) {
        await this.applyQuality(targetQuality);
        this.currentQuality = targetQuality;
      }
    }, intervalMs);
  }

  private async applyQuality(quality: 'high' | 'medium' | 'low') {
    const preset = this.qualityPresets[quality];
    const sender = this.peerConnection
      .getSenders()
      .find((s) => s.track?.kind === 'video');

    if (sender) {
      const params = sender.getParameters();
      if (!params.encodings) {
        params.encodings = [{}];
      }
      params.encodings[0].maxBitrate = preset.maxBitrate;
      params.encodings[0].maxFramerate = preset.maxFramerate;
      params.encodings[0].scaleResolutionDownBy =
        preset.scaleResolutionDownBy;
      await sender.setParameters(params);
      console.log(`Video quality changed to ${quality}`);
    }
  }
}
```

### Automatic Recovery from Connection Drops

In mobile apps, temporary disconnections are common — entering tunnels, switching between Wi-Fi networks, and so on. Rather than showing "Disconnected" and ending the call, implementing automatic reconnection dramatically improves user experience.

```typescript
// Connection recovery via ICE restart
class ConnectionRecovery {
  private peerConnection: RTCPeerConnection;
  private signalingManager: SignalingManager;
  private maxRetries = 3;
  private retryCount = 0;
  private retryTimeoutId: NodeJS.Timeout | null = null;

  constructor(pc: RTCPeerConnection, sm: SignalingManager) {
    this.peerConnection = pc;
    this.signalingManager = sm;
    this.setupMonitoring();
  }

  private setupMonitoring() {
    this.peerConnection.oniceconnectionstatechange = () => {
      const state = this.peerConnection.iceConnectionState;

      switch (state) {
        case 'disconnected':
          // Wait for natural recovery on brief disconnections (5s)
          this.retryTimeoutId = setTimeout(() => {
            if (
              this.peerConnection.iceConnectionState === 'disconnected'
            ) {
              this.attemptRecovery();
            }
          }, 5000);
          break;
        case 'failed':
          // Attempt ICE restart on complete failure
          this.attemptRecovery();
          break;
        case 'connected':
          // Reset retry counter on successful reconnection
          this.retryCount = 0;
          if (this.retryTimeoutId) {
            clearTimeout(this.retryTimeoutId);
          }
          break;
      }
    };
  }

  private async attemptRecovery() {
    if (this.retryCount >= this.maxRetries) {
      console.log('Max reconnection attempts reached. Ending call.');
      return;
    }

    this.retryCount++;
    console.log(
      `Attempting reconnection... (${this.retryCount}/${this.maxRetries})`
    );

    try {
      // ICE restart: Gather new ICE candidates and re-establish connection
      const offer = await this.peerConnection.createOffer({
        iceRestart: true,
      });
      await this.peerConnection.setLocalDescription(offer);
      await this.signalingManager.sendOffer(offer);
    } catch (error) {
      console.error('Reconnection failed:', error);
      // Retry with exponential backoff
      setTimeout(
        () => this.attemptRecovery(),
        Math.pow(2, this.retryCount) * 1000
      );
    }
  }
}
```

## Implementing Screen Sharing

Screen sharing significantly increases the value proposition of a video calling app. It's essential for tech support, presentations, pair programming, and many other use cases.

### Screen Capture on iOS

On iOS, `RPScreenRecorder` captures the screen content, which is then sent as a WebRTC media track. In React Native, `react-native-webrtc` supports this functionality.

```typescript
import { mediaDevices } from 'react-native-webrtc';

class ScreenShareManager {
  private peerConnection: RTCPeerConnection;
  private screenStream: MediaStream | null = null;
  private originalVideoTrack: MediaStreamTrack | null = null;
  private originalVideoSender: RTCRtpSender | null = null;

  constructor(pc: RTCPeerConnection) {
    this.peerConnection = pc;
  }

  async startScreenShare(): Promise<boolean> {
    try {
      // Get screen capture stream
      // iOS: Uses RPScreenRecorder, Android: Uses MediaProjection API internally
      this.screenStream = await mediaDevices.getDisplayMedia({
        video: true,
        audio: false, // Set to true to share system audio
      });

      const screenTrack = this.screenStream.getVideoTracks()[0];

      // Replace existing video track with screen share track
      const videoSender = this.peerConnection
        .getSenders()
        .find((s) => s.track?.kind === 'video');

      if (videoSender) {
        this.originalVideoTrack = videoSender.track;
        this.originalVideoSender = videoSender;
        await videoSender.replaceTrack(screenTrack);
      }

      // Handle when user stops screen sharing via system UI
      screenTrack.onended = () => {
        this.stopScreenShare();
      };

      return true;
    } catch (error) {
      console.error('Failed to start screen sharing:', error);
      return false;
    }
  }

  async stopScreenShare(): Promise<void> {
    // Restore original camera feed
    if (this.originalVideoSender && this.originalVideoTrack) {
      await this.originalVideoSender.replaceTrack(this.originalVideoTrack);
    }

    // Stop screen share stream
    this.screenStream?.getTracks().forEach((track) => track.stop());
    this.screenStream = null;
    this.originalVideoTrack = null;
    this.originalVideoSender = null;
  }
}
```

The key here is using the `replaceTrack()` method. By replacing the existing video track rather than adding a new one, you can switch seamlessly without SDP renegotiation.

## Scaling to Group Calls — SFU Architecture

### The Limits of P2P and the Need for SFU

For 1-on-1 calls, P2P connections work perfectly. But with 3 or more participants, each device needs to maintain separate P2P connections with every other participant, causing device load and bandwidth consumption to spike rapidly. Having each device maintain 3 outgoing streams for a 4-person call simply isn't practical on mobile devices.

The solution is a **SFU (Selective Forwarding Unit)**. An SFU receives a single stream from each participant and selectively forwards it to others. Each device only needs to send 1 stream and receive N-1 streams, dramatically reducing client-side load.

### SFU Client Implementation

```typescript
// SFU-based group call manager
class GroupCallManager {
  private localStream: MediaStream | null = null;
  private publisherConnection: RTCPeerConnection | null = null;
  // Per-participant subscriber PeerConnections
  private subscriberConnections: Map<string, RTCPeerConnection> = new Map();
  private participants: Map<string, MediaStream> = new Map();
  private sfuEndpoint: string;
  private roomId: string;

  constructor(sfuEndpoint: string, roomId: string) {
    this.sfuEndpoint = sfuEndpoint;
    this.roomId = roomId;
  }

  // Publish local stream to SFU
  async publishLocalStream(stream: MediaStream): Promise<void> {
    this.localStream = stream;

    const pc = new RTCPeerConnection({
      iceServers: [{ urls: 'stun:stun.l.google.com:19302' }],
    });

    stream.getTracks().forEach((track) => {
      pc.addTrack(track, stream);
    });

    // Negotiate with SFU
    const offer = await pc.createOffer();
    await pc.setLocalDescription(offer);

    const response = await fetch(`${this.sfuEndpoint}/publish`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        roomId: this.roomId,
        sdp: offer.sdp,
      }),
    });

    const { sdp: answerSdp } = await response.json();
    await pc.setRemoteDescription(
      new RTCSessionDescription({ type: 'answer', sdp: answerSdp })
    );

    this.publisherConnection = pc;
  }

  // Subscribe to a specific participant's stream from SFU
  async subscribeToParticipant(participantId: string): Promise<MediaStream> {
    const pc = new RTCPeerConnection({
      iceServers: [{ urls: 'stun:stun.l.google.com:19302' }],
    });

    // Add receive-only transceivers
    pc.addTransceiver('video', { direction: 'recvonly' });
    pc.addTransceiver('audio', { direction: 'recvonly' });

    const remoteStream = new MediaStream();
    pc.ontrack = (event) => {
      remoteStream.addTrack(event.track);
    };

    const offer = await pc.createOffer();
    await pc.setLocalDescription(offer);

    const response = await fetch(`${this.sfuEndpoint}/subscribe`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        roomId: this.roomId,
        participantId,
        sdp: offer.sdp,
      }),
    });

    const { sdp: answerSdp } = await response.json();
    await pc.setRemoteDescription(
      new RTCSessionDescription({ type: 'answer', sdp: answerSdp })
    );

    this.subscriberConnections.set(participantId, pc);
    this.participants.set(participantId, remoteStream);

    return remoteStream;
  }

  // Handle participant departure
  removeParticipant(participantId: string): void {
    const pc = this.subscriberConnections.get(participantId);
    if (pc) {
      pc.close();
      this.subscriberConnections.delete(participantId);
    }
    this.participants.delete(participantId);
  }

  // Clean up all connections
  async disconnect(): Promise<void> {
    this.publisherConnection?.close();
    this.subscriberConnections.forEach((pc) => pc.close());
    this.subscriberConnections.clear();
    this.participants.clear();
    this.localStream?.getTracks().forEach((track) => track.stop());
  }
}
```

For the server side, you can use Cloudflare's **Calls API**, **LiveKit**, or open-source SFU servers like **Janus**. Choose between managed services and self-hosting based on the scale of the app you're building with Rork Max.

## Implementing Call Recording

The ability to record calls for later review is particularly valuable in education, consulting, and customer support scenarios.

```typescript
// Client-side call recording using MediaRecorder API
class CallRecorder {
  private mediaRecorder: MediaRecorder | null = null;
  private recordedChunks: Blob[] = [];
  private isRecording = false;

  // Mix local and remote streams for recording
  async startRecording(
    localStream: MediaStream,
    remoteStream: MediaStream
  ): Promise<void> {
    // Mix streams using AudioContext
    const audioContext = new AudioContext();
    const destination = audioContext.createMediaStreamDestination();

    // Local audio
    if (localStream.getAudioTracks().length > 0) {
      const localSource = audioContext.createMediaStreamSource(
        new MediaStream(localStream.getAudioTracks())
      );
      localSource.connect(destination);
    }

    // Remote audio
    if (remoteStream.getAudioTracks().length > 0) {
      const remoteSource = audioContext.createMediaStreamSource(
        new MediaStream(remoteStream.getAudioTracks())
      );
      remoteSource.connect(destination);
    }

    // Record remote video with mixed audio
    const mixedStream = new MediaStream([
      ...remoteStream.getVideoTracks(),
      ...destination.stream.getAudioTracks(),
    ]);

    this.mediaRecorder = new MediaRecorder(mixedStream, {
      mimeType: 'video/webm;codecs=vp8,opus',
      videoBitsPerSecond: 2500000,
    });

    this.mediaRecorder.ondataavailable = (event) => {
      if (event.data.size > 0) {
        this.recordedChunks.push(event.data);
      }
    };

    this.mediaRecorder.start(1000); // Get chunks every second
    this.isRecording = true;
  }

  async stopRecording(): Promise<Blob | null> {
    return new Promise((resolve) => {
      if (!this.mediaRecorder || !this.isRecording) {
        resolve(null);
        return;
      }

      this.mediaRecorder.onstop = () => {
        const blob = new Blob(this.recordedChunks, {
          type: 'video/webm',
        });
        this.recordedChunks = [];
        this.isRecording = false;
        resolve(blob);
      };

      this.mediaRecorder.stop();
    });
  }
}
```

For storing recorded videos, Supabase Storage is the most efficient option. Use multipart uploads for large video files and configure bucket policies to restrict access to call participants only.

## Production Deployment Considerations

### Choosing a TURN Server

In production, a reliable TURN server is essential. When connecting from behind corporate firewalls or when the mobile carrier uses Symmetric NAT, STUN alone fails to establish connections in roughly 10-15% of cases.

Your main options include **Cloudflare TURN** (low latency, globally distributed), **Twilio Network Traversal Service** (pay-as-you-go, proven track record), and **coturn** (open source, self-hosted). While self-hosting coturn is more cost-effective, the operational overhead makes managed services more practical for most teams.

### Permission Management Best Practices

Camera and microphone permissions directly impact user experience. Rather than showing the system permission dialog immediately, presenting a "pre-permission" screen that explains why access is needed can significantly increase approval rates.

```typescript
import { Platform, PermissionsAndroid } from 'react-native';
import { check, request, PERMISSIONS, RESULTS } from 'react-native-permissions';

async function requestMediaPermissions(): Promise<boolean> {
  if (Platform.OS === 'ios') {
    const cameraStatus = await request(PERMISSIONS.IOS.CAMERA);
    const micStatus = await request(PERMISSIONS.IOS.MICROPHONE);
    return (
      cameraStatus === RESULTS.GRANTED && micStatus === RESULTS.GRANTED
    );
  }

  if (Platform.OS === 'android') {
    const grants = await PermissionsAndroid.requestMultiple([
      PermissionsAndroid.PERMISSIONS.CAMERA,
      PermissionsAndroid.PERMISSIONS.RECORD_AUDIO,
    ]);
    return (
      grants[PermissionsAndroid.PERMISSIONS.CAMERA] ===
        PermissionsAndroid.RESULTS.GRANTED &&
      grants[PermissionsAndroid.PERMISSIONS.RECORD_AUDIO] ===
        PermissionsAndroid.RESULTS.GRANTED
    );
  }

  return false;
}
```

## Frequently Asked Questions (FAQ)

### Q1: How much data does a WebRTC video call consume?

It depends on video quality, but a 720p, 30fps 1-on-1 video call typically uses about 1.5-2.5 GB of data per hour. By implementing adaptive bitrate control, video quality automatically adjusts to network conditions, ensuring stable calls even on cellular connections. Audio-only calls consume roughly 30-50 MB per hour.

### Q2: Can P2P connections work without STUN/TURN servers?

On the same local network (same Wi-Fi, for example), connections may work without STUN/TURN servers. However, for internet-based communication, NAT (Network Address Translation) intervenes, requiring STUN servers for public IP discovery. In Symmetric NAT environments, TURN server relaying is mandatory. Always provision TURN servers for production apps.

### Q3: What's the difference between WebRTC and paid services like Twilio or Agora?

WebRTC is a low-level communication protocol that requires you to build your own signaling and server infrastructure. Twilio Video and Agora provide these as fully managed services. Self-implementation offers advantages in customization and cost, but requires development and operational investment. If your monthly call volume is low or you need to launch quickly, managed services are the more pragmatic choice.

### Q4: What's the maximum number of participants for group calls?

With P2P architecture, 3-4 participants is the practical limit. With an SFU, you can scale to 20-50 participants depending on client device capabilities. For mobile apps specifically, consider battery drain and memory constraints — display video for 4-9 participants in a grid layout, and for larger groups, show only the active speaker prominently.
