## What Is Core ML, and Why Does It Matter for Rork Max?

**Core ML** is Apple's on-device machine learning framework. It lets you run AI models — image classification, object detection, natural language processing, speech recognition, pose estimation, and more — directly on iPhone and iPad hardware, with no internet connection required.

Historically, integrating Core ML required Xcode expertise and solid Swift knowledge. **Rork Max** changes this entirely. Its AI compiler (powered by Claude Code + Opus 4.6) auto-generates all the boilerplate Swift code, including model loading, data type conversions, and Vision request handling.

## Step 2: Integrate a Core ML Model

### 2-1. Using Pre-Trained Models (Fastest Approach)

Apple publishes dozens of pre-trained models on their [Core ML model gallery](https://developer.apple.com/machine-learning/models/), all free to download. To integrate one via Rork Max:

```
Add code to load MobileNetV2.mlmodel via the Vision framework.
Use VNImageRequestHandler to convert UIImage to CVPixelBuffer and run inference.
Retrieve the top 5 VNClassificationObservation results and display them in a ConfidenceCardView component.
```

Here's what the generated classifier code looks like:

```swift
import Vision
import CoreML
import SwiftUI

class ImageClassifier: ObservableObject {
    @Published var results: [ClassificationResult] = []

    private lazy var model: VNCoreMLModel = {
        // Load MobileNetV2 as a Core ML model
        let config = MLModelConfiguration()
        config.computeUnits = .all // Prioritize Neural Engine
        let mlModel = try! MobileNetV2(configuration: config).model
        return try! VNCoreMLModel(for: mlModel)
    }()

    func classify(image: UIImage) {
        guard let ciImage = CIImage(image: image) else { return }

        let request = VNCoreMLRequest(model: model) { [weak self] request, error in
            guard let observations = request.results as? [VNClassificationObservation] else { return }

            // Update results on the main thread
            DispatchQueue.main.async {
                self?.results = observations.prefix(5).map {
                    ClassificationResult(
                        label: $0.identifier,
                        confidence: Double($0.confidence) * 100
                    )
                }
            }
        }

        // Center-crop to preserve aspect ratio during classification
        request.imageCropAndScaleOption = .centerCrop

        let handler = VNImageRequestHandler(ciImage: ciImage, options: [:])
        try? handler.perform([request])
    }
}

struct ClassificationResult: Identifiable {
    let id = UUID()
    let label: String
    let confidence: Double
}
```

## Step 3: Add Natural Language Processing

Core ML's `NaturalLanguage` framework is just as easy to integrate. Here's a prompt for a sentiment analysis feature:

```
Build a text sentiment analysis screen:
- A TextField where users can type or paste text
- Use NLTagger with sentimentScore to detect sentiment
- Display Positive / Negative / Neutral with emoji and color-coding (green/red/gray)
- Support both English and Japanese automatically
```

The generated Swift sentiment analysis code:

```swift
import NaturalLanguage

class SentimentAnalyzer: ObservableObject {
    @Published var sentiment: SentimentResult = .neutral

    private let tagger = NLTagger(tagSchemes: [.sentimentScore])

    func analyze(text: String) {
        tagger.string = text

        let (tag, _) = tagger.tag(
            at: text.startIndex,
            unit: .paragraph,
            scheme: .sentimentScore
        )

        // Scores range from -1.0 (most negative) to +1.0 (most positive)
        if let scoreString = tag?.rawValue, let score = Double(scoreString) {
            DispatchQueue.main.async {
                switch score {
                case 0.2...:
                    self.sentiment = .positive  // 😊 Green
                case ..<(-0.2):
                    self.sentiment = .negative  // 😞 Red
                default:
                    self.sentiment = .neutral   // 😐 Gray
                }
            }
        }
    }
}

enum SentimentResult {
    case positive, negative, neutral

    var emoji: String {
        switch self {
        case .positive: return "😊"
        case .negative: return "😞"
        case .neutral:  return "😐"
        }
    }
}
```

## Step 5: Optimize for Performance

Use these prompts to squeeze maximum performance from Core ML:

```
Optimize Core ML inference performance:
1. Set MLModelConfiguration computeUnits to .cpuAndNeuralEngine
2. Run inference on a background queue (never block the UI thread)
3. Use VNSequenceRequestHandler for continuous frame processing to reuse the model
4. Cache the model as a singleton (avoid re-loading on every inference call)
```

Optimized model caching with Swift actors:

```swift
// Thread-safe singleton model cache using Swift actors
actor ModelCache {
    static let shared = ModelCache()

    private var cachedModel: VNCoreMLModel?

    func getModel() async throws -> VNCoreMLModel {
        if let model = cachedModel { return model }

        let config = MLModelConfiguration()
        config.computeUnits = .cpuAndNeuralEngine // Neural Engine first, CPU fallback

        let mlModel = try await MLModel.load(
            contentsOf: Bundle.main.url(forResource: "MyClassifier", withExtension: "mlmodelc")!,
            configuration: config
        )
        let visionModel = try VNCoreMLModel(for: mlModel)
        cachedModel = visionModel
        return visionModel
    }
}

// Usage: run inference asynchronously without blocking UI
Task.detached(priority: .userInitiated) {
    let model = try await ModelCache.shared.getModel()
    // Perform inference...
}
```

## Common Errors and Fixes

### Error: `Error loading MLModel: No such file`

The `.mlmodel` file isn't included in the app bundle.

```
Add the MLModelFiles key to Info.plist and ensure the model file is listed
under Bundle Resources in the build settings. Regenerate the app with these changes.
```

### Error: `VNCoreMLModel failed: Image buffer size mismatch`

The model expects a different input image size than what's being provided.

```
Resize UIImage to 224x224 pixels before creating VNImageRequestHandler,
and change imageCropAndScaleOption to .scaleFit.
```

### Low FPS performance

```
Change computeUnits from .all to .cpuAndNeuralEngine.
This removes GPU transfer overhead and improves Neural Engine throughput
for most vision models on iPhone 12 and later.
```

## Frequently Asked Questions

### Q. Can I train a Core ML model inside Rork Max?

A. Rork Max doesn't include a model training environment, but you can train models using Apple's free **Create ML** app on your Mac, export the `.mlmodel` file, and instruct Rork Max: "Integrate this custom Core ML model — add it to the bundle and generate inference code using the Vision framework." Rork Max handles all the Swift boilerplate from there.

### Q. Does Core ML work offline?

A. Yes — Core ML runs entirely on-device with no network connection required. Your users' images, text, and sensor data never leave the device, which is a strong privacy guarantee. Features backed by external APIs (like fetching nutrition data from Supabase) will still need connectivity, but the Core ML inference itself is always offline.

### Q. Will Core ML apps built with Rork Max pass App Store review?

A. Yes. Core ML is a first-party Apple framework and is fully compliant with all App Store guidelines. Rork Max-generated apps use only bundled models (no remote code execution), which satisfies Apple's security requirements. You can submit directly using Rork Max's two-click App Store submission feature.
