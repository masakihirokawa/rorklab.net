## Social Authentication in Rork Max

Users increasingly expect frictionless authentication experiences. Asking them to create yet another password-protected account creates barriers to adoption. Social login eliminates this friction by letting users sign in with existing Google, Apple, or GitHub accounts—dramatically improving both registration completion rates and user satisfaction.

This guide walks you through implementing OAuth 2.0-based authentication with major social providers in Rork Max applications.

## Understanding OAuth 2.0

### How Social Authentication Works

OAuth 2.0 enables users to grant your app access to their resources without sharing passwords with you. This standard protocol underpins modern authentication.

The basic flow:
1. User clicks "Sign in with Google"
2. User is redirected to Google's authorization screen
3. User grants permission
4. Your app receives an authorization code
5. Your backend exchanges the code for a token
6. User session is created

This approach protects user credentials while enabling seamless authentication.

## Implementing Google Sign-In

### Google Sign-In Integration

Google login is the most widely adopted social login method worldwide.

```jsx
import { GoogleSignin, statusCodes } from '@react-native-google-signin/google-signin';
import { useState } from 'react';

// Initialize GoogleSignin
GoogleSignin.configure({
  webClientId: process.env.GOOGLE_WEB_CLIENT_ID,
  iosClientId: process.env.GOOGLE_IOS_CLIENT_ID,
  androidClientId: process.env.GOOGLE_ANDROID_CLIENT_ID,
  scopes: ['profile', 'email'],
});

export function GoogleLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);
  const [isLoading, setIsLoading] = useState(false);

  const handleGoogleLogin = async () => {
    setIsLoading(true);
    try {
      // Execute Google Sign-In
      await GoogleSignin.hasPlayServices();
      const userInfo = await GoogleSignin.signIn();

      console.log(`Google login successful: ${userInfo.user.email}`);
      // Expected output: Google login successful: user@gmail.com

      // Send ID token to backend
      const idToken = userInfo.idToken;
      const backendResponse = await fetch('/api/auth/google', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ idToken }),
      });

      const { sessionToken } = await backendResponse.json();

      // Save session
      localStorage.setItem('sessionToken', sessionToken);
      setUserInfo(userInfo.user);
    } catch (error) {
      if (error.code === statusCodes.SIGN_IN_CANCELLED) {
        console.log('User cancelled login');
      } else if (error.code === statusCodes.IN_PROGRESS) {
        console.log('Sign in in progress...');
      } else if (error.code === statusCodes.PLAY_SERVICES_NOT_AVAILABLE) {
        console.error('Google Play Services not available');
      } else {
        console.error('Login error:', error);
      }
    } finally {
      setIsLoading(false);
    }
  };

  const handleLogout = async () => {
    try {
      await GoogleSignin.signOut();
      localStorage.removeItem('sessionToken');
      setUserInfo(null);
      console.log('Logout complete');
      // Expected output: Logout complete
    } catch (error) {
      console.error('Logout error:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleGoogleLogin} disabled={isLoading}>
          {isLoading ? 'Signing in...' : 'Sign in with Google'}
        </button>
      ) : (
        <div>
          <p>Signed in as: {userInfo.email}</p>
          <img src={userInfo.photo} alt="Profile" style={{ width: '50px', borderRadius: '50%' }} />
          <button onClick={handleLogout}>Sign Out</button>
        </div>
      )}
    </div>
  );
}
```

## Implementing Apple Sign-In

### Native iOS Sign in with Apple

iOS 13+ made Sign in with Apple available, and Apple now encourages its adoption in App Store apps.

```jsx
import { appleAuth, AppleAuthRequestScope, AppleAuthRequestOperation } from '@react-native-apple-authentication';
import { useState } from 'react';

export function AppleLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);

  const handleAppleLogin = async () => {
    try {
      // Execute Apple Sign-In flow
      const appleAuthRequestResponse = await appleAuth.performRequest({
        requestedOperation: AppleAuthRequestOperation.LOGIN,
        requestedScopes: [
          AppleAuthRequestScope.EMAIL,
          AppleAuthRequestScope.FULL_NAME,
        ],
      });

      // User cancelled authorization
      if (appleAuthRequestResponse.state === appleAuth.STATE.CANCELLED) {
        console.log('Apple login was cancelled');
        return;
      }

      const { identityToken, user } = appleAuthRequestResponse;

      console.log(`Apple login successful: ${user?.email}`);
      // Expected output: Apple login successful: user@example.com

      // Send ID token to backend
      const backendResponse = await fetch('/api/auth/apple', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ identityToken }),
      });

      const { sessionToken } = await backendResponse.json();
      localStorage.setItem('sessionToken', sessionToken);

      setUserInfo({
        email: user?.email,
        name: `${user?.name?.firstName || ''} ${user?.name?.lastName || ''}`,
      });
    } catch (error) {
      console.error('Apple login error:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleAppleLogin} style={{ backgroundColor: '#000', color: '#fff', padding: '10px 20px' }}>
          Sign in with Apple
        </button>
      ) : (
        <div>
          <p>Signed in as: {userInfo.email}</p>
          <button onClick={() => {
            localStorage.removeItem('sessionToken');
            setUserInfo(null);
          }}>Sign Out</button>
        </div>
      )}
    </div>
  );
}
```

## Implementing GitHub Login

### GitHub Authentication for Developer Apps

For developer-oriented applications, GitHub authentication is a natural choice.

```jsx
import { useState } from 'react';

export function GitHubLoginScreen() {
  const [userInfo, setUserInfo] = useState(null);
  const [isLoading, setIsLoading] = useState(false);

  const handleGitHubLogin = async () => {
    setIsLoading(true);
    try {
      // Redirect to GitHub OAuth authorization screen
      const clientId = process.env.GITHUB_CLIENT_ID;
      const redirectUri = encodeURIComponent(`${process.env.APP_URL}/auth/github/callback`);
      const scope = 'user:email';

      const authUrl = `https://github.com/login/oauth/authorize?` +
        `client_id=${clientId}&` +
        `redirect_uri=${redirectUri}&` +
        `scope=${scope}`;

      // Start OAuth flow in browser
      window.location.href = authUrl;
    } catch (error) {
      console.error('GitHub login error:', error);
      setIsLoading(false);
    }
  };

  // Handle OAuth callback
  const handleAuthCallback = async (code) => {
    try {
      // Backend exchanges code for token
      const response = await fetch('/api/auth/github/callback', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ code }),
      });

      const { sessionToken, user } = await response.json();

      console.log(`GitHub login successful: ${user.login}`);
      // Expected output: GitHub login successful: octocat

      localStorage.setItem('sessionToken', sessionToken);
      setUserInfo(user);
    } catch (error) {
      console.error('Callback processing error:', error);
    }
  };

  return (
    <div style={{ padding: '20px' }}>
      {!userInfo ? (
        <button onClick={handleGitHubLogin} disabled={isLoading} style={{ backgroundColor: '#333' }}>
          {isLoading ? 'Signing in...' : 'Sign in with GitHub'}
        </button>
      ) : (
        <div>
          <p>Signed in as: {userInfo.login}</p>
          <button onClick={() => {
            localStorage.removeItem('sessionToken');
            setUserInfo(null);
          }}>Sign Out</button>
        </div>
      )}
    </div>
  );
}
```

## Token Management and Session Persistence

### Secure Token Storage

Tokens should never be stored in localStorage. Use secure storage mechanisms instead.

```jsx
import * as SecureStore from 'expo-secure-store';

export class AuthTokenManager {
  static async saveToken(token) {
    try {
      await SecureStore.setItemAsync('authToken', token);
      console.log('Token securely saved');
      // Expected output: Token securely saved
    } catch (error) {
      console.error('Token save error:', error);
    }
  }

  static async getToken() {
    try {
      const token = await SecureStore.getItemAsync('authToken');
      return token;
    } catch (error) {
      console.error('Token retrieval error:', error);
      return null;
    }
  }

  static async deleteToken() {
    try {
      await SecureStore.deleteItemAsync('authToken');
      console.log('Token deleted');
      // Expected output: Token deleted
    } catch (error) {
      console.error('Token deletion error:', error);
    }
  }

  static async isTokenValid(token) {
    // Check JWT expiration
    try {
      const payload = JSON.parse(atob(token.split('.')[1]));
      const expiryTime = payload.exp * 1000;
      return Date.now() < expiryTime;
    } catch (error) {
      return false;
    }
  }
}
```

## Frequently Asked Questions

### Q1: How should I handle the same email address across multiple social providers?

**A:** When a user signs in with a different provider using the same email, offer to link the accounts to their existing profile, create a new profile, or let them choose. Implementing email verification ensures you're connecting the right accounts.

### Q2: Can users transition from social login to password-based authentication?

**A:** Yes—implement a password setup flow allowing users to set their own credentials. Afterward, support both traditional email+password login and social login for maximum flexibility.

### Q3: How do I request additional scopes after initial login?

**A:** Use incremental authorization. Request minimal scopes initially, then prompt for additional permissions later when needed. Google and Apple both support re-triggering auth flows to add new scopes to existing tokens.
