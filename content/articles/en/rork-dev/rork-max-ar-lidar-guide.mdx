## Introduction

**Rork Max**, launched in February 2026, is a game-changer for iOS developers who want to build augmented reality experiences without writing Swift from scratch. Unlike the original Rork (which targets React Native), Rork Max generates native Swift code for the full Apple ecosystem — iPhone, iPad, Apple Watch, Apple TV, Vision Pro, and iMessage.

One capability that's generating serious excitement is **AR and LiDAR app development**. Traditionally, ARKit and RealityKit required deep Swift expertise. With Rork Max, you can describe your AR experience in plain English and get production-quality Swift code in minutes.

---

## What AR/LiDAR Capabilities Does Rork Max Support?

Rork Max can generate code targeting all major Apple AR frameworks:

- **ARKit** — Apple's flagship AR framework: plane detection, object tracking, face tracking, image anchors
- **RealityKit** — 3D rendering engine with physics simulation and material support
- **Reality Composer Pro** — integration hooks for Apple's AR content creation tool
- **LiDAR Scanner API** — precise depth data from the LiDAR sensor on Pro/Max iPhones and iPad Pro

A particularly viral demo: a developer built a Pokémon GO-style AR game using Rork Max in a single afternoon. This highlights just how capable the platform has become for spatial computing.

## Step 1: Crafting Your AR Prompt in Rork Max

The quality of your output depends heavily on how specifically you describe the AR experience. Here are two real-world prompt examples.

### Furniture Placement App

```
Create a native Swift iOS app using ARKit and RealityKit that lets users
place 3D furniture models in their room using augmented reality.

Features:
- Detect horizontal planes (floors and table surfaces)
- Furniture catalog with 5 sample 3D models (chair, sofa, table, lamp, shelf)
- Tap detected plane to place selected furniture
- Pinch to resize, rotation gesture to rotate
- Tap placed furniture to delete it
- Screenshot button to save the AR scene
- Target iOS 17+, iPhone 11 and later
```

### LiDAR Room Scanner

```
Build a native Swift iOS app that uses the LiDAR scanner to create a
real-time 3D mesh of the surrounding environment.

Features:
- Live 3D mesh visualization using ARKit scene reconstruction
- Toggle between wireframe and solid mesh display
- Measure distances between two tapped points on the mesh
- Export captured mesh as a USDZ file to Files app
- Graceful fallback for non-LiDAR devices using standard plane detection
- Target iPhone 12 Pro and later, iOS 17+
```

## Step 2: Understanding the Generated Swift Code

Here's a typical structure Rork Max generates for an AR app:

```swift
// ContentView.swift
import SwiftUI
import RealityKit
import ARKit

struct ContentView: View {
    @StateObject private var arViewModel = ARViewModel()

    var body: some View {
        ZStack {
            ARViewContainer(viewModel: arViewModel)
                .ignoresSafeArea()

            VStack {
                Spacer()
                FurnitureCatalogView(viewModel: arViewModel)
                    .padding(.bottom, 24)
            }
        }
        .onAppear {
            arViewModel.setupARSession()
        }
    }
}
```

```swift
// ARViewModel.swift
import ARKit
import RealityKit
import Combine

@MainActor
class ARViewModel: ObservableObject {
    var arView: ARView?
    @Published var detectedPlanes: [ARPlaneAnchor] = []
    @Published var selectedModel: String = "chair"

    func setupARSession() {
        let config = ARWorldTrackingConfiguration()
        config.planeDetection = [.horizontal, .vertical]

        // Enable LiDAR scene reconstruction on supported devices
        if ARWorldTrackingConfiguration.supportsSceneReconstruction(.mesh) {
            config.sceneReconstruction = .mesh
        }

        arView?.session.run(config)
    }

    func placeModel(at position: SIMD3<Float>) {
        guard let modelEntity = try? ModelEntity.loadModel(named: selectedModel) else { return }
        let anchor = AnchorEntity(world: position)
        anchor.addChild(modelEntity)
        arView?.scene.addAnchor(anchor)
    }
}
```

## Step 3: Leveraging LiDAR for Scene Reconstruction

On LiDAR-equipped devices, you can enable significantly richer AR experiences:

```swift
func enableLiDARMeshVisualization(arView: ARView) {
    // Occlusion: real-world objects hide virtual objects behind them
    arView.environment.sceneUnderstanding.options.insert(.occlusion)

    // Physics: AR objects interact with real-world surfaces
    arView.environment.sceneUnderstanding.options.insert(.physics)

    // Debug: visualize the LiDAR mesh for development
    arView.debugOptions.insert(.showSceneUnderstanding)
}
```

## Step 4: Cross-Platform AR — iPhone to Vision Pro

One of Rork Max's most powerful features is generating code that scales across Apple platforms. Here's how the generated code handles Vision Pro:

```swift
#if os(visionOS)
import RealityKit
import SwiftUI

struct ImmersiveView: View {
    var body: some View {
        RealityView { content in
            // Vision Pro spatial content
            if let model = try? await ModelEntity(named: "furniture_chair") {
                content.add(model)
            }
        }
    }
}
#else
// iPhone/iPad ARKit implementation
#endif
```

This platform-aware code generation means your furniture placement app can run natively on both an iPhone (with ARKit) and Vision Pro (with immersive spaces) from a single Rork Max project.

## Step 5: App Store Submission — AR-Specific Requirements

AR camera apps require proper permission strings in `Info.plist`. Rork Max generates these automatically, but verify they're present:

```xml
<key>NSCameraUsageDescription</key>
<string>This app uses the camera to enable augmented reality features.</string>

<!-- If your app uses location for AR precision -->
<key>NSLocationWhenInUseUsageDescription</key>
<string>Location is used to improve AR context accuracy.</string>
```

**App Store review tips for AR apps:**
- Include a real-device demo video in your screenshots (simulator won't run ARKit)
- Clearly describe LiDAR requirements in your App Store description
- Test on the oldest supported device — AR performance varies significantly across hardware

## Prompting Tips for Better AR Output

### Tip 1: Be specific about device and OS targets

```
// Too vague
"Make an AR app"

// Much better
"Create a native Swift ARKit app for iPhone 12 Pro and later with LiDAR
mesh scanning. Target iOS 17+. Use RealityKit 4 for rendering."
```

### Tip 2: Request fallback behavior explicitly

```
Add graceful fallback for non-LiDAR devices: use standard horizontal
plane detection instead of scene reconstruction on iPhone 11 and earlier.
```

### Tip 3: Include performance constraints

```
Optimize for 60fps AR rendering. Use LOD (Level of Detail) for 3D
models beyond 2 meters. Limit mesh reconstruction updates to 2Hz.
```

## Current Limitations

Rork Max is powerful, but there are real-world constraints to keep in mind:

- **Custom Metal shaders**: Complex shader code quality varies — iterative refinement may be needed
- **Large-scale scenes**: Apps with many simultaneous anchors may require manual optimization
- **Multi-user AR**: ARKit Collaborative Sessions need careful verification post-generation
- **Physical device required**: ARKit doesn't run in the simulator — always test on hardware
