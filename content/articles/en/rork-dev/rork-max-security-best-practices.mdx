# Rork Max Security Best Practices

Security breaches destroy user trust instantly. This guide walks through proven patterns to eliminate vulnerabilities, from API key management to data encryption, with production-ready code examples.

## The Three Pillars of Mobile App Security

1. **Transport Security**: HTTPS and certificate pinning
2. **Authentication & Authorization**: Token-based auth with secure storage
3. **Data Protection**: Encryption at rest and in transit

## Part 1: API Key Management — The Golden Rule

### ❌ Never Do This

```typescript
// ❌ Hardcoded secrets
const API_KEY = 'sk-abc123xyz'; // Exposed in source code

// ❌ .env in bundle
export const OPENAI_KEY = process.env.REACT_APP_OPENAI_KEY; // Visible in APK

// ❌ Plaintext storage
await AsyncStorage.setItem('api_key', apiKey); // Readable on device
```

### ✓ The Correct Pattern: Backend Proxy

The mobile client never holds API keys. All requests route through your backend.

```
[Rork Max App] → [Your Backend] → [OpenAI/Gemini API]
  (Auth Token)    (API Key)      (Secured)
```

**src/services/apiClient.ts**
```typescript
import * as SecureStore from 'expo-secure-store';

export class APIClient {
  private baseURL = 'https://your-backend.com/api';
  private tokenKey = 'auth_token';

  async request<T>(
    endpoint: string,
    options: RequestInit = {}
  ): Promise<T> {
    try {
      // Get auth token from secure storage (NOT API key)
      const token = await SecureStore.getItemAsync(this.tokenKey);

      if (!token) {
        throw new Error('Not authenticated');
      }

      const response = await fetch(`${this.baseURL}${endpoint}`, {
        ...options,
        headers: {
          'Content-Type': 'application/json',
          Authorization: `Bearer ${token}`, // Token, never API key
          ...options.headers,
        },
      });

      if (!response.ok) {
        if (response.status === 401) {
          // Token expired, refresh
          await this.refreshToken();
          return this.request<T>(endpoint, options);
        }
        throw new Error(`HTTP ${response.status}`);
      }

      return response.json();
    } catch (error) {
      console.error('API error:', error);
      throw error;
    }
  }

  async post<T>(endpoint: string, body: any): Promise<T> {
    return this.request<T>(endpoint, {
      method: 'POST',
      body: JSON.stringify(body),
    });
  }

  async refreshToken(): Promise<void> {
    try {
      const refreshToken = await SecureStore.getItemAsync('refresh_token');
      const response = await fetch(`${this.baseURL}/auth/refresh`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ refreshToken }),
      });

      const data = await response.json();
      await SecureStore.setItemAsync(this.tokenKey, data.accessToken);
    } catch (error) {
      console.error('Refresh failed:', error);
      await this.logout();
    }
  }

  async logout(): Promise<void> {
    await SecureStore.deleteItemAsync(this.tokenKey);
    await SecureStore.deleteItemAsync('refresh_token');
  }
}

export const apiClient = new APIClient();
```

**Backend Implementation** (Node.js/Express)
```typescript
// backend/routes/ai.ts
import express from 'express';
import { OpenAI } from 'openai';
import { verifyToken } from '../middleware/auth';

const router = express.Router();
const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY, // Server-side only
});

router.post('/chat', verifyToken, async (req, res) => {
  try {
    const { messages } = req.body;

    // Client never sees API key
    const completion = await openai.chat.completions.create({
      model: 'gpt-4o',
      messages,
      max_tokens: 500,
    });

    res.json({
      success: true,
      data: completion.choices[0].message.content,
    });
  } catch (error) {
    res.status(500).json({
      success: false,
      error: 'Internal error',
    });
  }
});

export default router;
```

<Callout type="warning">
**Critical Rule**: Never include API keys, database credentials, or secrets in client code or environment files that get bundled. Use a backend proxy exclusively.
</Callout>

## Part 2: Secure Token Storage

Use `expo-secure-store` to encrypt tokens at rest.

### Installation

```bash
expo install expo-secure-store
```

### Auth Manager

**src/services/authManager.ts**
```typescript
import * as SecureStore from 'expo-secure-store';

export interface AuthTokens {
  accessToken: string;
  refreshToken: string;
  expiresAt: number;
}

export class AuthManager {
  private static readonly ACCESS_TOKEN_KEY = 'access_token';
  private static readonly REFRESH_TOKEN_KEY = 'refresh_token';
  private static readonly USER_DATA_KEY = 'user_data';

  // Save tokens securely
  static async saveTokens(tokens: AuthTokens): Promise<void> {
    try {
      await SecureStore.setItemAsync(
        this.ACCESS_TOKEN_KEY,
        tokens.accessToken
      );
      await SecureStore.setItemAsync(
        this.REFRESH_TOKEN_KEY,
        tokens.refreshToken
      );
      // Note: expiresAt can be stored in memory or SecureStore
    } catch (error) {
      console.error('Failed to save tokens:', error);
      throw error;
    }
  }

  static async getAccessToken(): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(this.ACCESS_TOKEN_KEY);
    } catch {
      return null;
    }
  }

  static async getRefreshToken(): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(this.REFRESH_TOKEN_KEY);
    } catch {
      return null;
    }
  }

  static async clearAll(): Promise<void> {
    try {
      await SecureStore.deleteItemAsync(this.ACCESS_TOKEN_KEY);
      await SecureStore.deleteItemAsync(this.REFRESH_TOKEN_KEY);
      await SecureStore.deleteItemAsync(this.USER_DATA_KEY);
    } catch (error) {
      console.error('Logout failed:', error);
    }
  }
}
```

### Login Implementation

**src/screens/LoginScreen.tsx**
```typescript
import React, { useState } from 'react';
import {
  View,
  TextInput,
  TouchableOpacity,
  Text,
  ActivityIndicator,
  StyleSheet,
} from 'react-native';
import { apiClient } from '../services/apiClient';
import { AuthManager } from '../services/authManager';

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, justifyContent: 'center' },
  input: {
    borderWidth: 1,
    borderColor: '#ccc',
    padding: 12,
    marginBottom: 12,
    borderRadius: 8,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 12,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: '600' },
  errorText: { color: '#FF3B30', marginTop: 12 },
});

export default function LoginScreen({ navigation }) {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState('');

  const handleLogin = async () => {
    if (!email || !password) {
      setError('Email and password required');
      return;
    }

    setIsLoading(true);
    setError('');

    try {
      const response = await apiClient.post('/auth/login', {
        email,
        password,
      });

      // Tokens are encrypted and stored securely
      await AuthManager.saveTokens({
        accessToken: response.accessToken,
        refreshToken: response.refreshToken,
        expiresAt: response.expiresAt,
      });

      navigation.replace('Home');
    } catch (err) {
      setError('Login failed. Check credentials and try again.');
    } finally {
      setIsLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <TextInput
        style={styles.input}
        placeholder="Email"
        value={email}
        onChangeText={setEmail}
        keyboardType="email-address"
        autoCapitalize="none"
        editable={!isLoading}
      />

      <TextInput
        style={styles.input}
        placeholder="Password"
        value={password}
        onChangeText={setPassword}
        secureTextEntry
        editable={!isLoading}
      />

      <TouchableOpacity
        style={styles.button}
        onPress={handleLogin}
        disabled={isLoading}
      >
        {isLoading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>Sign In</Text>
        )}
      </TouchableOpacity>

      {error && <Text style={styles.errorText}>{error}</Text>}
    </View>
  );
}
```

## Part 3: Biometric Authentication

Passphrase-less auth improves both UX and security.

### Installation

```bash
expo install expo-local-authentication
```

**src/services/biometricAuth.ts**
```typescript
import * as LocalAuthentication from 'expo-local-authentication';

export class BiometricAuth {
  static async isAvailable(): Promise<boolean> {
    try {
      const compatible = await LocalAuthentication.hasHardwareAsync();
      const enrolled = await LocalAuthentication.isEnrolledAsync();
      return compatible && enrolled;
    } catch {
      return false;
    }
  }

  static async authenticate(): Promise<boolean> {
    try {
      const result = await LocalAuthentication.authenticateAsync({
        reason: 'Authenticate to access your app',
        disableDeviceFallback: false,
        fallbackLabel: 'Use passcode',
      });

      return result.success;
    } catch (error) {
      console.error('Biometric auth failed:', error);
      return false;
    }
  }
}
```

**Integrate with Login**
```typescript
useEffect(() => {
  checkBiometric();
}, []);

const checkBiometric = async () => {
  const available = await BiometricAuth.isAvailable();

  if (available) {
    const success = await BiometricAuth.authenticate();
    if (success) {
      // User authenticated, proceed to home
      navigation.replace('Home');
    }
  }
};
```

## Part 4: HTTPS & Certificate Pinning

Every request must use HTTPS. Consider certificate pinning for sensitive backends.

**src/services/httpClient.ts**
```typescript
export class HTTPClient {
  static async request<T>(
    url: string,
    options: RequestInit = {}
  ): Promise<T> {
    // Enforce HTTPS
    if (!url.startsWith('https://')) {
      throw new Error('Only HTTPS connections allowed');
    }

    const response = await fetch(url, {
      ...options,
      headers: {
        'User-Agent': 'RorkMax/1.0',
        ...options.headers,
      },
    });

    if (!response.ok) {
      const error = await response.text();
      throw new Error(`HTTP ${response.status}: ${error}`);
    }

    return response.json();
  }
}
```

**Timeout Configuration**
```typescript
const controller = new AbortController();

  });
  // ...

  clearTimeout(timeout);
}
```

## Part 5: Data Protection at Rest

Separate storage strategies by sensitivity.

**src/services/storage.ts**
```typescript
import * as SecureStore from 'expo-secure-store';
import AsyncStorage from '@react-native-async-storage/async-storage';

export class SecureStorage {
  // Sensitive data: encrypted
  static async saveSensitive(key: string, value: string): Promise<void> {
    await SecureStore.setItemAsync(key, value);
  }

  static async getSensitive(key: string): Promise<string | null> {
    try {
      return await SecureStore.getItemAsync(key);
    } catch {
      return null;
    }
  }

  // Non-sensitive data: fast storage
  static async saveNonSensitive(key: string, value: any): Promise<void> {
    await AsyncStorage.setItem(key, JSON.stringify(value));
  }

  static async getNonSensitive(key: string): Promise<any | null> {
    try {
      const item = await AsyncStorage.getItem(key);
      return item ? JSON.parse(item) : null;
    } catch {
      return null;
    }
  }

  // Complete logout: clear all sensitive data
  static async clearAll(): Promise<void> {
    const sensitiveKeys = ['access_token', 'refresh_token', 'user_secrets'];
    await Promise.all(
      sensitiveKeys.map((key) => SecureStore.deleteItemAsync(key))
    );
  }
}
```

### Storage Decision Matrix

| Data | Storage | Reason |
|------|---------|--------|
| API keys | ✗ Never | Backend only |
| Access tokens | SecureStore | Encrypted |
| User ID | AsyncStorage | Non-sensitive |
| Theme preference | AsyncStorage | Non-sensitive |
| Payment info | ✗ Never | PCI compliance |
| Health data | SecureStore | Sensitive |

## Part 6: OWASP Mobile Security Top 10

### 1. Insecure Data Storage

```typescript
// ❌ Bad: Plain text
await AsyncStorage.setItem('token', token);

// ✓ Good: Encrypted
await SecureStore.setItemAsync('token', token);
```

### 2. Insecure Communication

```typescript
// ❌ Bad: HTTP
fetch('http://api.example.com/data');

// ✓ Good: HTTPS only
fetch('https://api.example.com/data');
```

### 3. Insecure Authentication

```typescript
// ❌ Bad: Hardcoded credentials
const password = 'admin123';

// ✓ Good: OAuth2 + biometric
const success = await BiometricAuth.authenticate();
const token = await AuthManager.getAccessToken();
```

### 4. Insufficient Cryptography

```typescript
// ✓ Use native crypto when available
// React Native SecureStore uses platform crypto (Keychain/KeyStore)
// For custom encryption, use: expo-crypto or TweetNaCl.js
```

### 5. Reverse Engineering

```typescript
// ❌ Avoid logging secrets
console.log('Token:', token); // Never in production

// ✓ Use conditional logging
if (__DEV__) {
  console.log('Debug info...');
}
```

## Security Checklist

### Code
- [ ] No hardcoded secrets
- [ ] Tokens in SecureStore only
- [ ] All network requests use HTTPS
- [ ] Sensitive logs stripped in production
- [ ] Error messages don't reveal internals

### Authentication
- [ ] Token refresh before expiry
- [ ] Tokens cleared on logout
- [ ] Access token lifetime < 1 hour
- [ ] Refresh token rotation considered
- [ ] Biometric auth for convenience

### Network
- [ ] HTTPS enforced
- [ ] Certificate validation enabled
- [ ] Request timeouts set (30s max)
- [ ] No plaintext credentials in requests

### Data
- [ ] Only necessary data collected
- [ ] PII encrypted at rest
- [ ] Old data deleted on logout
- [ ] Backups excluded from encryption
- [ ] GDPR/CCPA compliant

### App Store Submission
- [ ] Privacy policy present
- [ ] Data usage declared
- [ ] No malware/jailbreak detection bypass
- [ ] Test credentials used (not real accounts)

## Example: Complete Login Flow

```typescript
import { authManager } from '../services/authManager';
import { BiometricAuth } from '../services/biometricAuth';
import { apiClient } from '../services/apiClient';

export async function initializeAuth() {
  // 1. Check if user was previously logged in
  const accessToken = await authManager.getAccessToken();
  if (accessToken) {
    // 2. Optionally require biometric re-auth
    const biometricAvailable = await BiometricAuth.isAvailable();
    if (biometricAvailable) {
      const authenticated = await BiometricAuth.authenticate();
      if (!authenticated) {
        return false;
      }
    }
    // 3. Use existing token
    return true;
  }

  // 4. First time: show login screen
  return false;
}
```

## Summary

Secure Rork Max apps require:

1. **No Secrets in Code**: API keys go to backend only
2. **Encrypted Storage**: SecureStore for tokens and PII
3. **HTTPS Always**: No HTTP connections
4. **Token Rotation**: Short-lived access tokens
5. **Biometric Auth**: Convenience + security
6. **Clear on Logout**: Delete all sensitive data
7. **Monitor & Update**: Security patches matter

Follow these patterns and your app will pass security audits and earn user trust.
