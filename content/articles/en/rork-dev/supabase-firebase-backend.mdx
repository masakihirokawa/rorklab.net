# Backend Integration with Rork: Supabase and Firebase Setup Guide

Building a modern application requires robust backend infrastructure to handle authentication, data storage, and real-time synchronization. Rork makes it simple to integrate multiple backend services into your application. This guide covers the two most popular options: Supabase and Firebase.

## Understanding Rork's Backend Flexibility

Rork supports multiple backend platforms out of the box:
- **Supabase**: PostgreSQL-based with Row Level Security
- **Firebase**: Google's NoSQL approach with Firestore
- **Airtable**: Spreadsheet-like database for rapid prototyping
- **Custom APIs**: Integrate any REST or GraphQL endpoint

This flexibility allows you to choose the backend that best fits your project requirements, team expertise, and scaling strategy.

## Supabase Integration

Supabase is an open-source Firebase alternative built on PostgreSQL. It's ideal for applications that need structured data, complex queries, and fine-grained access control.

### Why Choose Supabase?

**PostgreSQL Foundation**: You get a full-featured relational database with ACID compliance. Define your schema upfront with tables, relationships, and constraints.

**Row Level Security (RLS)**: Implement database-level security policies that automatically enforce data access rules. Users can only see or modify their own data without additional application logic.

**Real-time Subscriptions**: Supabase provides built-in real-time capabilities. Subscribe to database changes and updates propagate instantly to connected clients.

```
// Example: Real-time subscription in Rork
const subscription = supabase
  .from('tasks')
  .on('*', payload => {
    console.log('Change received:', payload)
  })
  .subscribe()
```

### Setting Up Supabase with Rork

1. **Create a Supabase project** at supabase.com with your PostgreSQL database
2. **In Rork, use the prompt interface**: "Connect to Supabase database" or "Add Supabase backend"
3. **Provide credentials**:
   - Project URL
   - Public anon key
   - Service role key (for server-side operations)

4. **Define your schema** using Supabase's SQL editor or migrations
5. **Enable RLS policies** for each table to enforce data access rules

### Authentication with Supabase

Supabase handles multiple authentication methods:

```
// Email/password signup
const { user, session, error } = await supabase.auth.signUp({
  email: 'user@example.com',
  password: 'password123'
})

// Social login (OAuth)
const { user, session, error } = await supabase.auth.signInWithOAuth({
  provider: 'github'
})
```

Supabase supports email/password, magic links, OAuth (GitHub, Google, Discord), and SAML for enterprise applications.

## Firebase Integration

Firebase is Google's platform for building scalable applications. It takes a different approach with Firestore—a NoSQL document database that doesn't require upfront schema definition.

### Why Choose Firebase?

**NoSQL Flexibility**: Firestore stores data as collections of documents. You don't need to create tables or migrations beforehand. Add fields dynamically as your data requirements evolve.

**Just-in-Time Collections**: Collections are created when you first write data to them. This rapid prototyping approach is perfect for startups and fast-moving projects.

**Firebase Authentication**: Built-in support for email/password, phone authentication, OAuth, and anonymous login.

**Cloud Functions**: Write serverless functions triggered by database events, HTTP requests, or scheduled tasks.

```
// Example: Firestore query in Rork
const tasksRef = collection(db, 'users', userId, 'tasks')
const q = query(tasksRef, where('completed', '==', false))
const snapshot = await getDocs(q)
```

### Setting Up Firebase with Rork

1. **Create a Firebase project** at firebase.google.com
2. **Enable Firestore** in your Firebase console
3. **In Rork, use the prompt interface**: "Connect to Firebase" or "Add Firebase backend"
4. **Provide your Firebase config**:
   - Project ID
   - API Key
   - Auth Domain
   - Database URL

5. **Set up Firestore security rules** to restrict data access
6. **Enable authentication methods** in Firebase Console

### Authentication with Firebase

Firebase provides multiple authentication options:

```
// Email/password login
const userCredential = await signInWithEmailAndPassword(
  auth,
  email,
  password
)

// Google sign-in
const provider = new GoogleAuthProvider()
const result = await signInWithPopup(auth, provider)

// Anonymous login for trial users
const result = await signInAnonymously(auth)
```

## Comparing Supabase vs Firebase

| Feature | Supabase | Firebase |
|---------|----------|----------|
| Database Model | Relational (PostgreSQL) | NoSQL (Firestore) |
| Schema Definition | Define upfront | Dynamic/flexible |
| Complex Queries | Native SQL support | Limited query capabilities |
| Real-time | Built-in subscriptions | Real-time listeners |
| Authentication | Multiple methods | Multiple methods |
| Cost Model | Usage-based | Per operation |
| Self-hosting | Available | Google Cloud only |

**Choose Supabase if**:
- Your data has complex relationships
- You need powerful query capabilities
- You prefer structured schemas
- You want to self-host

**Choose Firebase if**:
- You need rapid prototyping
- Your data structure evolves frequently
- You want Google Cloud integration
- You prefer a fully managed service

## Connecting Backends in Rork via Prompts

Rork's AI understands natural language instructions for backend setup:

- "Add Supabase database for storing user notes"
- "Connect Firebase for real-time messaging"
- "Create authentication with email and Google login"
- "Set up Firestore collection for product inventory"

The AI generates the necessary configuration code, environment variables, and integration boilerplate automatically.

## Best Practices

**1. Environment Variables**: Never hardcode API keys. Use `.env.local` for sensitive credentials in development.

```
SUPABASE_URL=your_url
SUPABASE_ANON_KEY=your_key
FIREBASE_PROJECT_ID=your_project
```

**2. Type Safety**: Both Supabase and Firebase support TypeScript. Use types to catch errors at development time.

**3. Data Validation**: Validate data on both client and server. Don't rely solely on frontend validation.

**4. Security Rules**: Invest time in writing proper security rules or RLS policies. Test them thoroughly before production.

**5. Monitoring**: Set up logging and monitoring to track authentication events, database errors, and performance issues.

**6. Backup Strategy**: Regularly backup your data. Supabase and Firebase both provide backup solutions.

## Conclusion

Rork simplifies backend integration by supporting both Supabase and Firebase natively. Supabase excels when you need relational data structures and complex queries, while Firebase shines with rapid prototyping and NoSQL flexibility. Start with the platform that matches your current needs, and Rork makes it straightforward to migrate between backends as your application evolves.
