Apple's **SharePlay** lets multiple users share real-time experiences during FaceTime calls or iMessage group chats. Whether it's watching a movie together, playing a multiplayer game, co-editing a whiteboard, or working out side by side, SharePlay turns your app into a social experience.

The catch? Building SharePlay features requires working with Apple's **Group Activities** framework, managing sessions, and implementing real-time data synchronization in Swift. For many developers, that's been a significant technical barrier.

**Rork Max** removes that barrier. ## How SharePlay and Group Activities Work

### What Is SharePlay?

SharePlay was introduced in iOS 15 and received major upgrades in iOS 17. It enables shared experiences not just during FaceTime calls, but also through iMessage group conversations.

Here are some of the most compelling use cases:

- **Synchronized viewing**: Watch movies or videos together with playback in perfect sync
- **Collaborative editing**: Co-edit whiteboards, documents, or designs in real time
- **Multiplayer gaming**: Turn-based or real-time multiplayer without a custom backend
- **Group fitness**: Share workout routines and track progress together
- **Shopping together**: Browse products as a group and share reactions

### The Group Activities Framework

Under the hood, SharePlay is powered by the **Group Activities** framework. Here are the key components you'll work with:

1. **GroupActivity**: A protocol that defines a shareable activity
2. **GroupSession**: Manages the lifecycle of an active session
3. **GroupSessionMessenger**: Handles message passing between participants
4. **GroupStateObserver**: Monitors whether the device is eligible for group sessions

```swift
// Basic GroupActivity protocol structure
import GroupActivities

struct DrawingActivity: GroupActivity {
    // Metadata displayed in the SharePlay sheet
    var metadata: GroupActivityMetadata {
        var meta = GroupActivityMetadata()
        meta.title = "Draw Together"
        meta.subtitle = "Collaborate on a canvas in real time"
        meta.type = .generic
        return meta
    }
}

// Expected output:
// A sharable activity that appears in the
// FaceTime / Messages SharePlay menu
```

### Step 3: Implement Session Management

Next, you'll need a session manager to handle the lifecycle of SharePlay sessions. Use this prompt with Rork Max:

```text
Implement SharePlay session management with these requirements:

1. Automatically propose the activity during FaceTime calls
2. Monitor participant join/leave events
3. Clean up resources when the session ends
4. Handle errors gracefully (dropped connections, etc.)
```

Here's the session manager that Rork Max generates:

```swift
import GroupActivities
import Combine

@MainActor
class DrawingSessionManager: ObservableObject {
    @Published var session: GroupSession<CollaborativeDrawingActivity>?
    @Published var messenger: GroupSessionMessenger?
    @Published var participants: Set<Participant> = []
    @Published var isSharePlayActive = false

    private var subscriptions = Set<AnyCancellable>()
    private var tasks = Set<Task<Void, Never>>()

    // Start observing for incoming sessions
    func startObserving() {
        let task = Task {
            for await session in CollaborativeDrawingActivity.sessions() {
                self.configureSession(session)
            }
        }
        tasks.insert(task)
    }

    private func configureSession(
        _ session: GroupSession<CollaborativeDrawingActivity>
    ) {
        self.session = session
        self.messenger = GroupSessionMessenger(session: session)
        self.isSharePlayActive = true

        // Observe participant changes
        session.$activeParticipants
            .sink { [weak self] participants in
                self?.participants = participants
            }
            .store(in: &subscriptions)

        // Observe session state changes
        session.$state
            .sink { [weak self] state in
                if case .invalidated = state {
                    self?.cleanupSession()
                }
            }
            .store(in: &subscriptions)

        // Join the session
        session.join()
    }

    private func cleanupSession() {
        session = nil
        messenger = nil
        isSharePlayActive = false
        participants = []
        subscriptions.removeAll()
    }
}

// Expected output:
// When a SharePlay session starts, logs:
// "Session joined with 2 participants"
```

### Step 4: Real-Time Data Synchronization

The heart of any SharePlay app is real-time data sync. Here's how to send and receive drawing data using `GroupSessionMessenger`:

```text
Implement real-time drawing data synchronization:

1. Send drawing strokes to all participants in real time
2. Immediately render received strokes on the canvas
3. Define a Codable DrawingMessage struct
4. Optimize for minimal latency
```

```swift
import Foundation
import GroupActivities

// Define the message structure for syncing
struct DrawingMessage: Codable, Sendable {
    let id: UUID
    let points: [CGPointCodable]
    let color: String
    let lineWidth: CGFloat
    let timestamp: Date

    struct CGPointCodable: Codable, Sendable {
        let x: CGFloat
        let y: CGFloat
    }
}

// Send and receive drawing data
extension DrawingSessionManager {
    // Broadcast a stroke to all participants
    func sendStroke(_ stroke: DrawingStroke) async {
        guard let messenger else { return }

        let message = DrawingMessage(
            id: UUID(),
            points: stroke.points.map {
                DrawingMessage.CGPointCodable(x: $0.x, y: $0.y)
            },
            color: stroke.color.hexString,
            lineWidth: stroke.lineWidth,
            timestamp: Date()
        )

        do {
            // reliable: guarantees delivery
            try await messenger.send(message)
        } catch {
            print("Send error: \(error.localizedDescription)")
        }
    }

    // Listen for strokes from other participants
    func startReceiving(
        onReceive: @escaping (DrawingMessage) -> Void
    ) {
        let task = Task {
            guard let messenger else { return }
            for await (message, _) in messenger.messages(
                of: DrawingMessage.self
            ) {
                await MainActor.run {
                    onReceive(message)
                }
            }
        }
        tasks.insert(task)
    }
}

// Expected output:
// Lines drawn by User A appear on
// User B's canvas in real time
```

## Supporting Both FaceTime and Messages

Starting with iOS 17, SharePlay works through **iMessage group chats** as well as FaceTime. To support both channels in your Rork Max app, use this prompt:

```text
Make SharePlay accessible from the Messages app too.
Add a SharePlay button to the UI that starts a group activity
even outside of FaceTime calls.
```

```swift
import GroupActivities
import SwiftUI

struct SharePlayButton: View {
    @StateObject private var activityState
        = GroupStateObserver()
    @State private var isActivitySharingSheetPresented = false

    var body: some View {
        Button(action: {
            // If in a FaceTime call, start directly;
            // otherwise show the sharing sheet
            if activityState.isEligibleForGroupSession {
                startActivity()
            } else {
                isActivitySharingSheetPresented = true
            }
        }) {
            Label("Draw Together", systemImage: "shareplay")
                .font(.headline)
                .foregroundColor(.white)
                .padding(.horizontal, 20)
                .padding(.vertical, 12)
                .background(Color.blue)
                .cornerRadius(12)
        }
        .sheet(isPresented: $isActivitySharingSheetPresented) {
            // Show the sharing sheet for Messages
            ActivitySharingSheet(
                activity: CollaborativeDrawingActivity()
            )
        }
    }

    private func startActivity() {
        Task {
            let activity = CollaborativeDrawingActivity()
            let result = try await activity.activate()
            print("Activity started: \(result)")
        }
    }
}

// Expected output:
// Tapping "Draw Together" either starts a SharePlay
// session or shows the sharing sheet
```

The `GroupStateObserver` checks whether the device is currently in a FaceTime call, so your UI can adapt accordingly.

Testing SharePlay is a bit different from testing a standard app. Here's how to make the process smoother.

Xcode lets you run two simulators side by side to test SharePlay. Open your Rork Max project in Xcode and follow these steps:


### Performance Optimization

For smooth real-time sync, consider these strategies:

- **Throttle send frequency**: Downsample drawing points to reduce data volume
- **Batch messages**: Group multiple points into a single message
- **Delta sync**: Only send what has changed

```swift
// Throttling example for drawing data
class ThrottledSender {
    private var pendingPoints: [CGPoint] = []
    private var lastSendTime: Date = .distantPast
    private let minInterval: TimeInterval = 0.05 // 50ms

    func addPoint(_ point: CGPoint) {
        pendingPoints.append(point)

        let now = Date()
        if now.timeIntervalSince(lastSendTime) >= minInterval {
            flush()
        }
    }

    private func flush() {
        guard !pendingPoints.isEmpty else { return }
        let points = pendingPoints
        pendingPoints = []
        lastSendTime = Date()

        Task {
            await sessionManager.sendPoints(points)
        }
    }
}

// Expected output:
// Network usage drops ~60% while keeping
// latency under 50ms
```

## Frequently Asked Questions

### Q1: Can I use SharePlay with the free Rork plan?

No. SharePlay requires native Swift code, which is only available on the **Rork Max plan**. The standard Rork plan generates React Native / Expo code, which doesn't have direct access to Apple's Group Activities framework. Rork Max costs $200/month but gives you full access to native iOS features like SharePlay, Dynamic Island, and more.

### Q2: How many people can join a SharePlay session?

According to Apple's documentation, SharePlay sessions support **up to 32 participants**. However, apps with high-frequency data sync (like real-time drawing) may experience performance issues with large groups. We recommend designing for 4–8 participants as the sweet spot for a smooth experience.

### Q3: Does App Store review require anything special for SharePlay apps?

The review process is largely the same as any other app. You do need to make sure `NSSupportsGroupActivities` is set to `YES` in your Info.plist. Rork Max handles this automatically in the project configuration, but it's worth double-checking before submission. We also recommend including clear testing instructions in your review notes so the App Store team can verify SharePlay functionality.

### Q4: Can Android users join a SharePlay session?

SharePlay is exclusive to the Apple ecosystem, so Android users can't participate directly. If you need cross-platform real-time sharing, you'll want to build a custom sync solution using WebSockets or Firebase Realtime Database in standard Rork. Check out [Building a Real-Time Chat App with Rork and Supabase](/en/articles/rork-dev/rork-supabase-chat-app) for a cross-platform approach.
