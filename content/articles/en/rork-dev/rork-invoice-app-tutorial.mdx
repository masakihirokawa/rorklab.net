## Introduction

For freelancers and independent contractors, creating, sending, and managing invoices is an unavoidable part of the business. Yet many existing invoicing services come with steep monthly fees or don't quite fit your workflow.

In this tutorial, we'll build a custom invoice app from scratch using Rork. By the end, you'll have a fully functional app with client management, PDF invoice generation, email sharing, and a sales dashboard to track your revenue.

**What you'll learn:**

- How to design data-driven business apps with Rork
- Implementing PDF generation using expo-print
- Managing client and invoice data with Supabase as a backend
- Building a revenue visualization dashboard

## App Architecture

Let's map out the screen structure and data models before writing any code. Having a clear blueprint upfront helps you give Rork more precise prompts and ensures a consistent design.

### Screen Structure

Our app will consist of five main screens:

1. **Dashboard** — Monthly revenue chart, unpaid invoice list, and recent activity
2. **Client List** — Add, edit, and search your clients
3. **Invoice List** — Filter by status (draft, sent, paid, overdue)
4. **Invoice Editor** — Dynamically add line items, set tax rates, auto-calculate totals
5. **Invoice Preview** — View the rendered PDF, share via email or other channels

### Data Model

```sql
-- Clients table
CREATE TABLE clients (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,           -- Company or individual name
  email TEXT,                   -- Email address
  address TEXT,                 -- Mailing address
  phone TEXT,                   -- Phone number
  notes TEXT,                   -- Additional notes
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Invoices table
CREATE TABLE invoices (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  client_id UUID REFERENCES clients(id),
  invoice_number TEXT NOT NULL, -- e.g., INV-2026-001
  status TEXT DEFAULT 'draft',  -- draft / sent / paid / overdue
  issue_date DATE NOT NULL,     -- Issue date
  due_date DATE NOT NULL,       -- Payment due date
  tax_rate DECIMAL DEFAULT 0.10,-- Tax rate (default 10%)
  notes TEXT,                   -- Remarks
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Invoice line items table
CREATE TABLE invoice_items (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  invoice_id UUID REFERENCES invoices(id) ON DELETE CASCADE,
  description TEXT NOT NULL,    -- Item description
  quantity INTEGER DEFAULT 1,   -- Quantity
  unit_price DECIMAL NOT NULL,  -- Unit price
  sort_order INTEGER DEFAULT 0  -- Display order
);
```

## Implementing PDF Generation

The heart of any invoice app is its ability to generate professional-looking PDFs. We'll combine `expo-print` and `expo-sharing` to create PDFs from an HTML template and share them with clients.

```typescript
// utils/generateInvoicePDF.ts
import * as Print from 'expo-print';
import * as Sharing from 'expo-sharing';

interface InvoiceItem {
  description: string;
  quantity: number;
  unit_price: number;
}

interface InvoiceData {
  invoiceNumber: string;
  clientName: string;
  clientAddress: string;
  issueDate: string;
  dueDate: string;
  items: InvoiceItem[];
  taxRate: number;
  notes?: string;
}

// Generate HTML template for the invoice
const generateHTML = (data: InvoiceData): string => {
  const subtotal = data.items.reduce(
    (sum, item) => sum + item.quantity * item.unit_price, 0
  );
  const tax = subtotal * data.taxRate;
  const total = subtotal + tax;

  const itemRows = data.items
    .map(
      (item) => `
      <tr>
        <td>${item.description}</td>
        <td class="right">${item.quantity}</td>
        <td class="right">$${item.unit_price.toLocaleString()}</td>
        <td class="right">$${(item.quantity * item.unit_price).toLocaleString()}</td>
      </tr>`
    )
    .join('');

  return `
    <!DOCTYPE html>
    <html>
    <head>
      <meta charset="utf-8" />
      <style>
        body { font-family: 'Helvetica Neue', sans-serif; padding: 40px; color: #333; }
        .header { display: flex; justify-content: space-between; margin-bottom: 40px; }
        .invoice-title { font-size: 28px; color: #1a2332; font-weight: bold; }
        .invoice-number { color: #666; margin-top: 4px; }
        table { width: 100%; border-collapse: collapse; margin: 20px 0; }
        th { background: #1a2332; color: white; padding: 10px; text-align: left; }
        td { padding: 10px; border-bottom: 1px solid #eee; }
        .right { text-align: right; }
        .total-section { margin-top: 20px; text-align: right; }
        .total-row { font-size: 14px; margin: 4px 0; }
        .grand-total { font-size: 20px; font-weight: bold; color: #1a2332; }
        .notes { margin-top: 30px; padding: 15px; background: #f9f9f9; border-radius: 8px; }
      </style>
    </head>
    <body>
      <div class="header">
        <div>
          <div class="invoice-title">INVOICE</div>
          <div class="invoice-number">${data.invoiceNumber}</div>
        </div>
        <div style="text-align: right;">
          <div>Date: ${data.issueDate}</div>
          <div>Due: ${data.dueDate}</div>
        </div>
      </div>

      <div style="margin-bottom: 30px;">
        <strong>Bill To:</strong><br/>
        ${data.clientName}<br/>
        ${data.clientAddress}
      </div>

      <table>
        <thead>
          <tr>
            <th>Description</th>
            <th class="right">Qty</th>
            <th class="right">Rate</th>
            <th class="right">Amount</th>
          </tr>
        </thead>
        <tbody>${itemRows}</tbody>
      </table>

      <div class="total-section">
        <div class="total-row">Subtotal: $${subtotal.toLocaleString()}</div>
        <div class="total-row">Tax (${(data.taxRate * 100).toFixed(0)}%): $${tax.toLocaleString()}</div>
        <div class="grand-total">Total: $${total.toLocaleString()}</div>
      </div>

      ${data.notes ? `<div class="notes"><strong>Notes:</strong><br/>${data.notes}</div>` : ''}
    </body>
    </html>
  `;
};

// Generate PDF and open the share dialog
export const generateAndSharePDF = async (data: InvoiceData): Promise<void> => {
  const html = generateHTML(data);

  // Generate PDF from HTML
  const { uri } = await Print.printToFileAsync({
    html,
    base64: false,
  });

  // Open share dialog (email, AirDrop, etc.)
  if (await Sharing.isAvailableAsync()) {
    await Sharing.shareAsync(uri, {
      mimeType: 'application/pdf',
      dialogTitle: `${data.invoiceNumber}.pdf`,
      UTI: 'com.adobe.pdf',
    });
  }
};
```

### Building the Preview Screen

Before exporting, users should be able to preview their invoice. Here's how to render the HTML template in a WebView:

```typescript
// screens/InvoicePreview.tsx
import React, { useMemo } from 'react';
import { View, ScrollView, TouchableOpacity, Text, StyleSheet } from 'react-native';
import { WebView } from 'react-native-webview';
import { generateAndSharePDF } from '../utils/generateInvoicePDF';

export default function InvoicePreview({ route }) {
  const { invoice, client, items } = route.params;

  const invoiceData = useMemo(() => ({
    invoiceNumber: invoice.invoice_number,
    clientName: client.name,
    clientAddress: client.address || '',
    issueDate: invoice.issue_date,
    dueDate: invoice.due_date,
    items: items,
    taxRate: invoice.tax_rate,
    notes: invoice.notes,
  }), [invoice, client, items]);

  const handleExportPDF = async () => {
    try {
      await generateAndSharePDF(invoiceData);

      console.error('PDF generation failed:', error);

    <View style={styles.container}>
      <WebView

        source={{ html: generateHTML(invoiceData) }}

      />
      <View style={styles.actions}>
        <TouchableOpacity style={styles.button} onPress={handleExportPDF}>
          <Text style={styles.buttonText}>Share as PDF</Text>
        </TouchableOpacity>
      </View>
    </View>
  );
}
```

## Setting Up the Supabase Backend

We'll store client and invoice data in Supabase so everything syncs across devices. Among the backend options available for Rork apps, Supabase offers the most straightforward path to real-time data synchronization.

### Initializing the Supabase Client

```typescript
// lib/supabase.ts
import { createClient } from '@supabase/supabase-js';
import AsyncStorage from '@react-native-async-storage/async-storage';

const supabaseUrl = 'https://your-project.supabase.co';
const supabaseAnonKey = 'your-anon-key';

export const supabase = createClient(supabaseUrl, supabaseAnonKey, {
  auth: {
    storage: AsyncStorage, // Storage adapter for React Native
    autoRefreshToken: true,
    persistSession: true,
  },
});
```

### Invoice CRUD Operations

```typescript
// hooks/useInvoices.ts
import { useState, useEffect, useCallback } from 'react';
import { supabase } from '../lib/supabase';

export function useInvoices() {
  const [invoices, setInvoices] = useState([]);
  const [loading, setLoading] = useState(true);

  // Fetch all invoices with related client info
  const fetchInvoices = useCallback(async () => {
    setLoading(true);
    const { data, error } = await supabase
      .from('invoices')
      .select(`
        *,
        client:clients(name, email),
        items:invoice_items(*)
      `)
      .order('created_at', { ascending: false });

    if (!error && data) {
      setInvoices(data);
    }
    setLoading(false);
  }, []);

  // Create a new invoice
  const createInvoice = async (invoice, items) => {
    // Auto-generate invoice number (e.g., INV-2026-042)
    const year = new Date().getFullYear();
    const { count } = await supabase
      .from('invoices')
      .select('*', { count: 'exact', head: true });
    const number = `INV-${year}-${String((count || 0) + 1).padStart(3, '0')}`;

    const { data, error } = await supabase
      .from('invoices')
      .insert({ ...invoice, invoice_number: number })
      .select()
      .single();

    if (data && items.length > 0) {
      // Batch insert line items
      await supabase.from('invoice_items').insert(
        items.map((item, i) => ({
          ...item,
          invoice_id: data.id,
          sort_order: i,
        }))
      );
    }

    await fetchInvoices(); // Refresh the list
    return data;
  };

  // Update invoice status (e.g., draft → sent)
  const updateStatus = async (id: string, status: string) => {
    await supabase.from('invoices').update({ status }).eq('id', id);
    await fetchInvoices();
  };

  useEffect(() => {
    fetchInvoices();
  }, [fetchInvoices]);

  return { invoices, loading, createInvoice, updateStatus, refresh: fetchInvoices };
}
```

## Building the Sales Dashboard

To add real value to the app, let's build a dashboard that visualizes monthly revenue trends. We'll use `react-native-chart-kit` to render a line chart alongside summary cards.

```typescript
// components/SalesDashboard.tsx
import React, { useMemo } from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { LineChart } from 'react-native-chart-kit';
import { Dimensions } from 'react-native';

const screenWidth = Dimensions.get('window').width;

export default function SalesDashboard({ invoices }) {
  // Aggregate monthly revenue
  const monthlyData = useMemo(() => {
    const months = {};
    invoices
      .filter((inv) => inv.status === 'paid')
      .forEach((inv) => {
        const month = inv.issue_date.slice(0, 7); // "2026-03"
        const total = inv.items?.reduce(
          (sum, item) => sum + item.quantity * item.unit_price, 0
        ) || 0;
        months[month] = (months[month] || 0) + total * (1 + inv.tax_rate);
      });

    const labels = Object.keys(months).slice(-6);
    const data = labels.map((m) => months[m]);
    return { labels: labels.map((l) => l.slice(5)), data }; // "03" format
  }, [invoices]);

  // Calculate summary figures
  const summary = useMemo(() => {
    const paid = invoices.filter((i) => i.status === 'paid');
    const unpaid = invoices.filter((i) => i.status === 'sent' || i.status === 'overdue');
    const calcTotal = (list) =>
      list.reduce((sum, inv) => {
        const subtotal = inv.items?.reduce(
          (s, item) => s + item.quantity * item.unit_price, 0
        ) || 0;
        return sum + subtotal * (1 + inv.tax_rate);
      }, 0);
    return {
      totalRevenue: calcTotal(paid),
      unpaidAmount: calcTotal(unpaid),
      unpaidCount: unpaid.length,
    };
  }, [invoices]);

  return (
    <View style={styles.container}>
      {/* Summary Cards */}
      <View style={styles.cards}>
        <View style={[styles.card, { backgroundColor: '#e8f5e9' }]}>
          <Text style={styles.cardLabel}>Paid</Text>
          <Text style={styles.cardValue}>${summary.totalRevenue.toLocaleString()}</Text>
        </View>
        <View style={[styles.card, { backgroundColor: '#fff3e0' }]}>
          <Text style={styles.cardLabel}>Unpaid</Text>
          <Text style={styles.cardValue}>${summary.unpaidAmount.toLocaleString()}</Text>
          <Text style={styles.cardSub}>{summary.unpaidCount} invoices</Text>
        </View>
      </View>

      {/* Monthly Revenue Chart */}
      {monthlyData.data.length > 0 && (
        <LineChart
          data={{
            labels: monthlyData.labels,
            datasets: [{ data: monthlyData.data }],
          }}
          width={screenWidth - 32}
          height={200}
          chartConfig={{
            backgroundColor: '#fff',
            backgroundGradientFrom: '#fff',
            backgroundGradientTo: '#fff',
            decimalPlaces: 0,
            color: (opacity = 1) => `rgba(26, 35, 50, ${opacity})`,
            labelColor: () => '#666',
          }}
          bezier
          style={styles.chart}
        />
      )}
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16 },
  cards: { flexDirection: 'row', gap: 12, marginBottom: 20 },
  card: { flex: 1, padding: 16, borderRadius: 12 },
  cardLabel: { fontSize: 12, color: '#666' },
  cardValue: { fontSize: 20, fontWeight: 'bold', marginTop: 4 },
  cardSub: { fontSize: 12, color: '#999', marginTop: 2 },
  chart: { borderRadius: 12 },
});
```

## Common Errors and Fixes

Here are some issues you're likely to run into during development, along with their solutions.

### expo-print Not Working

`expo-print` may not function correctly in Expo Go. Creating a Development Build resolves this issue:

```bash
# Create a development build with EAS
npx eas build --profile development --platform ios
```

### Supabase RLS (Row Level Security) Errors

When you enable RLS on a table, all operations are blocked by default until you add policies. Don't forget to configure them:

```sql
-- Policy allowing authenticated users to manage their own data
ALTER TABLE invoices ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can manage own invoices"
  ON invoices FOR ALL
  USING (auth.uid() = user_id);
```

### PDF Fonts Not Rendering Properly

In some environments, system fonts may not render correctly in generated PDFs. You can fix this by specifying a web font in your HTML template:

```html
<style>
  @import url('https://fonts.googleapis.com/css2?family=Inter&display=swap');
  body { font-family: 'Inter', sans-serif; }
</style>
```

## FAQ

### Q: Can I build this app on Rork's free plan?

Yes, you can develop and test the app entirely on the free plan. However, publishing to the App Store or Google Play requires a Rork Max subscription. We recommend building and testing with the free plan first, then upgrading to Rork Max when you're ready to ship.

### Q: Can I customize the invoice template design?

Absolutely. Since the PDF is generated from an HTML template, you have full control over the design. You can adjust the logo placement, color scheme, fonts, and add fixed headers with your company information and bank details. If you're comfortable with HTML and CSS, the customization possibilities are virtually limitless.

### Q: Does the app work offline?

With a local cache layer using AsyncStorage, you can create invoices and generate PDFs even without an internet connection. When connectivity is restored, the app syncs with Supabase automatically. This pattern is covered in more detail in our existing articles on offline-first architecture.

### Q: Can I adapt this for different tax systems?

Yes. The tax rate field in the data model is fully configurable per invoice. Whether you need to handle Japan's qualified invoice system, EU VAT, or US sales tax, you can customize the HTML template to display the required tax information, registration numbers, and breakdowns for your jurisdiction.
