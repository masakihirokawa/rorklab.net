## Why App Store Connect API Is Essential for Indie Developers

After launching an app on the App Store, most developers fall into the same operational trap: manually checking reviews every morning, copying sales numbers into a spreadsheet, and spending hours on data gathering that could be automated. For solo developers, this admin overhead directly eats into build time.

The App Store Connect API is Apple's RESTful interface for programmatic access to everything in App Store Connect — reviews, sales trends, analytics, TestFlight management, and more. Once implemented, this API lets you automate your entire operational loop and surface real-time data inside your own Rork-built dashboard.

This guide walks through JWT authentication setup, then covers the Reviews API, Sales API, and Analytics API in depth. You'll end up with a complete automation pipeline built on Rork and Supabase Edge Functions that runs itself.

**Who this guide is for:**
- Developers with live apps (or apps in development) on the App Store
- Anyone who wants to automate App Store Connect management tasks
- Rork Max users building native iOS apps with a backend
- Intermediate-to-advanced developers comfortable with backend integrations

## App Store Connect API Overview

The API covers four major areas:

**App Management API:** Manage app metadata (descriptions, screenshots, pricing), version info, and customer review responses.

**Sales and Trends API:** Retrieve download counts, revenue, and subscription data. Third-party tools like AppFigures and Data Box all pull their data through this same API.

**Finance API:** Access financial reports and settlement data for accounting and tax purposes.

**TestFlight API:** Automate beta tester management, build distribution, and feedback collection.

## Step 1: Implementing JWT Authentication

Every App Store Connect API request requires JWT (JSON Web Token) authentication. This is the first hurdle, but once implemented it works across all endpoints.

### Getting Your API Key

Log into [App Store Connect](https://appstoreconnect.apple.com) and navigate to Users and Access → Integrations → App Store Connect API. Click Generate Key, assign the appropriate permissions (Finance, Sales, App Management, etc.), and securely store the downloaded `.p8` file along with your `Issuer ID` and `Key ID`.

**⚠️ The `.p8` file can only be downloaded once. Store it somewhere safe immediately.**

### Generating JWT Tokens (Supabase Edge Functions)

```typescript
// supabase/functions/appstore-auth/jwt.ts
import { create } from "https://deno.land/x/djwt@v3.0.2/mod.ts";
import { crypto } from "https://deno.land/std@0.224.0/crypto/mod.ts";

const ISSUER_ID = Deno.env.get("ASC_ISSUER_ID")!;
const KEY_ID = Deno.env.get("ASC_KEY_ID")!;
const PRIVATE_KEY_PEM = Deno.env.get("ASC_PRIVATE_KEY")!; // Contents of .p8 file

/**
 * Generate a JWT token for App Store Connect API
 * Max validity: 20 minutes (Apple's limit)
 */
export async function generateAscToken(): Promise<string> {
  // Import PEM-encoded private key
  const pemBody = PRIVATE_KEY_PEM
    .replace("-----BEGIN PRIVATE KEY-----", "")
    .replace("-----END PRIVATE KEY-----", "")
    .replace(/\s/g, "");

  const binaryKey = Uint8Array.from(atob(pemBody), c => c.charCodeAt(0));

  const privateKey = await crypto.subtle.importKey(
    "pkcs8",
    binaryKey,
    { name: "ECDSA", namedCurve: "P-256" },
    false,
    ["sign"]
  );

  const now = Math.floor(Date.now() / 1000);

  const payload = {
    iss: ISSUER_ID,
    iat: now,
    exp: now + 1200, // 20 minutes (maximum)
    aud: "appstoreconnect-v1",
  };

  const header = {
    alg: "ES256" as const,
    kid: KEY_ID,
    typ: "JWT",
  };

  return await create(header, payload, privateKey);
}

// Usage example:
// const token = await generateAscToken();
// const res = await fetch("https://api.appstoreconnect.apple.com/v1/apps", {
//   headers: { Authorization: `Bearer ${token}` }
// });
// Expected output: 200 OK + JSON list of your apps
```

## Step 2: Reviews API — Auto-Fetch and Sentiment Analysis

Automatically pulling your reviews and running AI sentiment analysis dramatically improves how you respond to user feedback and spot product issues early.

### Fetching All Reviews with Pagination

```typescript
// supabase/functions/fetch-reviews/index.ts
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";
import { generateAscToken } from "../appstore-auth/jwt.ts";

const APP_ID = Deno.env.get("APP_ID")!;
const supabase = createClient(
  Deno.env.get("SUPABASE_URL")!,
  Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
);

interface Review {
  id: string;
  rating: number;
  title: string;
  body: string;
  reviewerNickname: string;
  createdDate: string;
  territory: string;
}

async function fetchAllReviews(token: string): Promise<Review[]> {
  const reviews: Review[] = [];
  let cursor: string | null = null;

  do {
    const url = new URL(
      `https://api.appstoreconnect.apple.com/v1/apps/${APP_ID}/customerReviews`
    );
    url.searchParams.set("limit", "200");
    url.searchParams.set("sort", "-createdDate"); // newest first
    if (cursor) url.searchParams.set("cursor", cursor);

    const response = await fetch(url.toString(), {
      headers: { Authorization: `Bearer ${token}` }
    });

    if (!response.ok) throw new Error(`Reviews API error: ${response.status}`);

    const data = await response.json();

    for (const item of data.data) {
      reviews.push({
        id: item.id,
        rating: item.attributes.rating,
        title: item.attributes.title ?? "",
        body: item.attributes.body,
        reviewerNickname: item.attributes.reviewerNickname,
        createdDate: item.attributes.createdDate,
        territory: item.attributes.territory,
      });
    }

    cursor = data.links?.next
      ? new URL(data.links.next).searchParams.get("cursor")
      : null;

  } while (cursor);

  return reviews;
}

/**
 * Use Claude to analyze review sentiment and draft a reply
 */
async function analyzeAndDraftReply(review: Review): Promise<{
  sentiment: "positive" | "neutral" | "negative";
  category: string;
  replyDraft: string;
}> {
  const ANTHROPIC_API_KEY = Deno.env.get("ANTHROPIC_API_KEY")!;

  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": ANTHROPIC_API_KEY,
      "anthropic-version": "2023-06-01",
    },
    body: JSON.stringify({
      model: "claude-haiku-4-5-20251001",
      max_tokens: 500,
      messages: [{
        role: "user",
        content: `Analyze this App Store review and respond with JSON.

Review (${review.rating}/5 stars):
Title: ${review.title}
Body: ${review.body}

Response format:
{
  "sentiment": "positive|neutral|negative",
  "category": "UI/UX|Performance|Bug Report|Feature Request|Pricing|Other",
  "replyDraft": "A concise, sincere developer reply under 100 words"
}

The reply should be grateful and authentic.`
      }],
    }),
  });

  const data = await response.json();
  const text = data.content[0].text;
  const jsonMatch = text.match(/\{[\s\S]*\}/);
  return JSON.parse(jsonMatch![0]);
}

// Main handler
Deno.serve(async (_req) => {
  const token = await generateAscToken();
  const reviews = await fetchAllReviews(token);

  // Only process reviews from the last 24 hours
  const oneDayAgo = new Date(Date.now() - 86400000).toISOString();
  const newReviews = reviews.filter(r => r.createdDate > oneDayAgo);

  for (const review of newReviews) {
    const analysis = await analyzeAndDraftReply(review);

    await supabase.from("app_reviews").upsert({
      review_id: review.id,
      rating: review.rating,
      title: review.title,
      body: review.body,
      nickname: review.reviewerNickname,
      created_at: review.createdDate,
      territory: review.territory,
      sentiment: analysis.sentiment,
      category: analysis.category,
      reply_draft: analysis.replyDraft,
    });
  }

  return new Response(
    JSON.stringify({ processed: newReviews.length }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

## Step 3: Sales and Trends API — Automated Revenue Tracking

The Sales API returns data in `.tsv.gz` format (gzip-compressed TSV). You need to handle decompression and parsing.

```typescript
// supabase/functions/fetch-sales/index.ts
import { generateAscToken } from "../appstore-auth/jwt.ts";

const VENDOR_NUMBER = Deno.env.get("ASC_VENDOR_NUMBER")!;
// Find this in App Store Connect → Revenue → Payments and Financial Reports

async function fetchDailySales(token: string, date: string): Promise<unknown[]> {
  // date format: YYYY-MM-DD
  const url = new URL("https://api.appstoreconnect.apple.com/v1/salesReports");
  url.searchParams.set("filter[frequency]", "DAILY");
  url.searchParams.set("filter[reportType]", "SALES");
  url.searchParams.set("filter[reportSubType]", "SUMMARY");
  url.searchParams.set("filter[vendorNumber]", VENDOR_NUMBER);
  url.searchParams.set("filter[reportDate]", date);

  const response = await fetch(url.toString(), {
    headers: { Authorization: `Bearer ${token}` }
  });

  if (response.status === 404) {
    console.log(`No sales data available for ${date} yet`);
    return [];
  }

  if (!response.ok) throw new Error(`Sales API error: ${response.status}`);

  // Decompress gzip
  const compressed = await response.arrayBuffer();
  const ds = new DecompressionStream("gzip");
  const writer = ds.writable.getWriter();
  writer.write(compressed);
  writer.close();

  const reader = ds.readable.getReader();
  const chunks: Uint8Array[] = [];
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    chunks.push(value);
  }

  const allBytes = new Uint8Array(chunks.reduce((acc, c) => acc + c.length, 0));
  let offset = 0;
  for (const c of chunks) { allBytes.set(c, offset); offset += c.length; }
  const text = new TextDecoder().decode(allBytes);

  // Parse TSV (first line = headers)
  const lines = text.trim().split("\n");
  const headers = lines[0].split("\t");

  return lines.slice(1).map(line => {
    const values = line.split("\t");
    const record: Record<string, string> = {};
    headers.forEach((h, i) => { record[h.trim()] = values[i] || ""; });
    return record;
  });
}
```

## Step 5: Negative Review Spike Alerts and Full Pipeline

```typescript
// supabase/functions/alert-negative-reviews/index.ts
const ALERT_THRESHOLD = 3; // Fire alert if 3+ low-rating reviews arrive within 1 hour

async function checkNegativeSpike() {
  const oneHourAgo = new Date(Date.now() - 3600000).toISOString();

  const { data: negativeReviews } = await supabase
    .from("app_reviews")
    .select("*")
    .lte("rating", 2)
    .gte("created_at", oneHourAgo);

  if ((negativeReviews?.length ?? 0) >= ALERT_THRESHOLD) {
    const SLACK_WEBHOOK = Deno.env.get("SLACK_WEBHOOK_URL")!;
    const message = [
      `🚨 Alert: ${negativeReviews!.length} low-rating reviews in the last hour!`,
      ...negativeReviews!.map(r => `⭐${r.rating} "${r.title}": ${r.body.slice(0, 60)}...`)
    ].join("\n");

    await fetch(SLACK_WEBHOOK, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ text: message }),
    });
    // Expected output: Urgent Slack alert with review snippets
  }
}
```

## Step 6: Automating TestFlight Management

The same JWT auth works for TestFlight too. Here's how to programmatically invite beta testers:

```typescript
async function inviteBetaTester(
  token: string,
  appId: string,
  email: string,
  firstName: string,
  lastName: string
) {
  const response = await fetch(
    "https://api.appstoreconnect.apple.com/v1/betaTesters",
    {
      method: "POST",
      headers: {
        Authorization: `Bearer ${token}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        data: {
          type: "betaTesters",
          attributes: { email, firstName, lastName },
          relationships: {
            apps: { data: [{ type: "apps", id: appId }] }
          }
        }
      })
    }
  );

  if (response.ok) {
    console.log(`✅ Invited ${email} to TestFlight`);
    // Expected output: ✅ Invited user@example.com to TestFlight
  } else {
    console.error(`❌ Invite failed: ${response.status}`);
  }
}
```

## Common Errors and How to Fix Them

### "401 Unauthorized" Responses

Expired JWT is the most common cause. Tokens are valid for a maximum of 20 minutes, so either generate a fresh token for every request, or cache the token and refresh it every 15 minutes. If you're storing the `.p8` content in Supabase Secrets, make sure line breaks are escaped as `\n` — they often get stripped during storage.

### "404 Not Found" from the Sales API

This just means the report for that date hasn't been generated yet. Apple typically finalizes the previous day's sales report around 5–8 AM PST. In JST, that's roughly 9 PM–12 PM the following day. Schedule your sales fetch accordingly.

### Hitting Rate Limits

The API's exact rate limits aren't publicly documented, but in practice stay under 100 requests per minute to be safe. When paginating through large review sets, add a 100–200ms delay between requests.

## FAQ

### Q. Do I need to pay for Apple Developer Program membership to use the API?

Yes. API keys are tied to Apple Developer Program membership — the same requirement as app provisioning and App Store distribution. At ¥12,800/year (or $99/year), it's a prerequisite for any serious iOS development.

### Q. Can I retrieve reviews going back to my app's launch date?

Yes. The Reviews API returns your full review history by default. For apps with large review volumes, use the `sort=-createdDate` parameter and track your last-fetched timestamp in the database to process only new reviews on each run.

### Q. Can I post review responses through the API too?

Yes. The `POST /v1/customerReviewResponses` endpoint lets you submit replies programmatically. That said, Apple's review guidelines apply — spammy or inappropriate responses may be removed. We strongly recommend using AI to draft replies and having a human approve them before posting.

### Q. Does this work with backends other than Supabase?

Absolutely. The JWT authentication logic works in any server-side environment: Node.js, Python, Go, Ruby, etc. The code in this guide is written for Deno (Supabase Edge Functions), but the concepts translate directly. In Node.js, `jsonwebtoken` + `node-forge` is a common combination.
