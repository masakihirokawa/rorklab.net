## What Is Cloudflare D1 — and Why Does It Pair So Well with Rork?

Cloudflare D1 is a **serverless SQLite database** that runs on Cloudflare Workers. Unlike traditional databases pinned to a single region, D1 executes queries close to your users across Cloudflare's global edge network, giving you consistently low latency without managing any servers.

For Rork developers, D1 brings several compelling advantages:

- **Generous free tier**: 50,000 queries per day and 5 GB of storage at no cost
- **Global low latency**: queries run at the edge closest to each user
- **Familiar SQL**: standard SQL syntax, similar to PostgreSQL or MySQL, keeps the learning curve gentle
- **Tight Workers integration**: combine with Hono for a slim REST API in minutes
- **Cloudflare ecosystem synergy**: works seamlessly alongside R2 (file storage) and KV (caching)

If you've already experimented with [building a REST API with Rork + Hono on Cloudflare Workers](/articles/rork-dev/rork-hono-cloudflare-workers-rest-api-guide), adding D1 as your persistent data layer is a natural next step.

## Architecture Overview

Here's the stack we're building in this guide:

- **Frontend**: Rork (React Native / Expo) mobile app
- **API layer**: Cloudflare Workers + Hono (REST API)
- **Database**: Cloudflare D1 (SQLite)
- **Auth**: Bearer token (simplified demo approach)

The request flow looks like this:

Rork app → HTTPS request → Cloudflare Workers (Hono) → D1 query → JSON response

This setup costs almost nothing to run and scales automatically as your user base grows.

## Step 1: Create a Cloudflare D1 Database

Use the Wrangler CLI to spin up a new D1 database.

```bash
# Log in to Cloudflare
npx wrangler login

# Create the database
npx wrangler d1 create rork-app-db

# Sample output:
# ✅ Successfully created DB 'rork-app-db'
# database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

Note the `database_id` — you'll need it in the next step.

```toml
[[d1_databases]]
binding = "DB"
database_name = "rork-app-db"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"  # ← your ID from Step 1
```

## Step 3: Create the Schema via Migrations

Managing D1 through migration files is the recommended approach. Create a `migrations/` folder and define the initial schema:

```sql
-- migrations/0001_init.sql
CREATE TABLE IF NOT EXISTS items (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id    TEXT    NOT NULL,
  title      TEXT    NOT NULL,
  body       TEXT,
  created_at TEXT    DEFAULT (datetime('now')),
  updated_at TEXT    DEFAULT (datetime('now'))
);

-- Speed up lookups by user_id
CREATE INDEX IF NOT EXISTS idx_items_user_id ON items(user_id);
```

Apply the migration to your local dev environment and to production:

```bash
# Apply locally (development)
npx wrangler d1 migrations apply rork-app-db --local

# Apply to production
npx wrangler d1 migrations apply rork-app-db
```

## Step 4: Implement the CRUD API with Hono

Open `src/index.ts` and build out the four core endpoints:

```typescript
// src/index.ts
import { Hono } from 'hono'
import { cors } from 'hono/cors'

type Bindings = {
  DB: D1Database
}

const app = new Hono<{ Bindings: Bindings }>()

// Allow requests from the Rork mobile app
app.use('*', cors({
  origin: '*',
  allowMethods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowHeaders: ['Content-Type', 'Authorization'],
}))

// ── List items ──
app.get('/items', async (c) => {
  const userId = c.req.header('X-User-Id') ?? 'anonymous'
  const { results } = await c.env.DB.prepare(
    'SELECT * FROM items WHERE user_id = ? ORDER BY created_at DESC'
  ).bind(userId).all()
  return c.json({ items: results })
})

// ── Create an item ──
app.post('/items', async (c) => {
  const userId = c.req.header('X-User-Id') ?? 'anonymous'
  const { title, body } = await c.req.json<{ title: string; body?: string }>()

  if (!title) {
    return c.json({ error: 'title is required' }, 400)
  }

  const result = await c.env.DB.prepare(
    'INSERT INTO items (user_id, title, body) VALUES (?, ?, ?) RETURNING *'
  ).bind(userId, title, body ?? '').first()

  return c.json({ item: result }, 201)
})

// ── Update an item ──
app.put('/items/:id', async (c) => {
  const userId = c.req.header('X-User-Id') ?? 'anonymous'
  const id = c.req.param('id')
  const { title, body } = await c.req.json<{ title?: string; body?: string }>()

  const result = await c.env.DB.prepare(
    `UPDATE items SET
       title = COALESCE(?, title),
       body  = COALESCE(?, body),
       updated_at = datetime('now')
     WHERE id = ? AND user_id = ?
     RETURNING *`
  ).bind(title ?? null, body ?? null, id, userId).first()

  if (!result) return c.json({ error: 'Not found' }, 404)
  return c.json({ item: result })
})

// ── Delete an item ──
app.delete('/items/:id', async (c) => {
  const userId = c.req.header('X-User-Id') ?? 'anonymous'
  const id = c.req.param('id')

  const { meta } = await c.env.DB.prepare(
    'DELETE FROM items WHERE id = ? AND user_id = ?'
  ).bind(id, userId).run()

  if (meta.changes === 0) return c.json({ error: 'Not found' }, 404)
  return c.json({ success: true })
})

export default app
```

Verify the API works locally:

```bash

# → API running at http://localhost:8787

# Quick smoke test
```

## Step 5: Connect Your Rork App to the D1 API

Inside your Rork project, create a custom hook that talks to the Workers API:

```typescript
// hooks/useItems.ts (inside your Rork project)
import { useState, useEffect, useCallback } from 'react'

const API_BASE = 'https://rork-d1-api.YOUR_SUBDOMAIN.workers.dev'
const USER_ID  = 'user_001' // replace with real auth in production

type Item = {
  id: number
  user_id: string
  title: string
  body: string
  created_at: string
}

export function useItems() {
  const [items, setItems]   = useState<Item[]>([])
  const [loading, setLoading] = useState(false)
  const [error, setError]   = useState<string | null>(null)

  const headers = {
    'Content-Type': 'application/json',
    'X-User-Id': USER_ID,
  }

  const fetchItems = useCallback(async () => {
    setLoading(true)
    try {
      const res  = await fetch(`${API_BASE}/items`, { headers })
      const data = await res.json<{ items: Item[] }>()
      setItems(data.items)
    } catch {
      setError('Failed to load items')
    } finally {
      setLoading(false)
    }
  }, [])

  const createItem = async (title: string, body?: string) => {
    const res = await fetch(`${API_BASE}/items`, {
      method: 'POST',
      headers,
      body: JSON.stringify({ title, body }),
    })
    if (!res.ok) throw new Error('Failed to create item')
    await fetchItems()
  }

  const deleteItem = async (id: number) => {
    await fetch(`${API_BASE}/items/${id}`, { method: 'DELETE', headers })
    setItems((prev) => prev.filter((item) => item.id !== id))
  }

  useEffect(() => { fetchItems() }, [fetchItems])

  return { items, loading, error, createItem, deleteItem, refetch: fetchItems }
}
```

Wire it up in a screen component:

```tsx
// screens/ItemListScreen.tsx (inside your Rork project)
import { View, Text, FlatList, TouchableOpacity, TextInput } from 'react-native'
import { useItems } from '../hooks/useItems'
import { useState } from 'react'

export default function ItemListScreen() {
  const { items, loading, createItem, deleteItem } = useItems()
  const [newTitle, setNewTitle] = useState('')

  const handleAdd = async () => {
    if (!newTitle.trim()) return
    await createItem(newTitle.trim())
    setNewTitle('')
  }

  return (
    <View style={{ flex: 1, padding: 16 }}>
      <View style={{ flexDirection: 'row', marginBottom: 12 }}>
        <TextInput
          value={newTitle}
          onChangeText={setNewTitle}
          placeholder="Add a new item"
          style={{
            flex: 1, borderWidth: 1, borderColor: '#ccc',
            padding: 8, borderRadius: 6, marginRight: 8,
          }}
        />
        <TouchableOpacity
          onPress={handleAdd}
          style={{ backgroundColor: '#F48024', padding: 10, borderRadius: 6 }}
        >
          <Text style={{ color: '#fff' }}>Add</Text>
        </TouchableOpacity>
      </View>

      {loading ? (
        <Text>Loading…</Text>
      ) : (
        <FlatList
          data={items}
          keyExtractor={(item) => String(item.id)}
          renderItem={({ item }) => (
            <View style={{
              flexDirection: 'row', alignItems: 'center',
              padding: 12, borderBottomWidth: 1, borderColor: '#eee',
            }}>
              <Text style={{ flex: 1 }}>{item.title}</Text>
              <TouchableOpacity onPress={() => deleteItem(item.id)}>
                <Text style={{ color: 'red' }}>Delete</Text>
              </TouchableOpacity>
            </View>
          )}
        />
      )}
    </View>
  )
}
```

## Common Errors and Fixes

**Error: `D1_ERROR: no such table: items`**

The migration hasn't been applied yet. Run:

```bash
npx wrangler d1 migrations apply rork-app-db
```

**Error: CORS error when calling from mobile**

Check the CORS middleware in your Hono handler. Setting `origin: '*'` resolves most cases during development. In production, restrict it to your specific domain for better security.

**Error: `TypeError: c.env.DB is undefined`**

The D1 binding in `wrangler.toml` is misconfigured. Double-check that `database_id` matches the ID returned when you created the database.

## Going Further: Pagination with D1

For apps that manage large datasets, add pagination to keep responses fast:

```typescript
// Paginated list endpoint
app.get('/items', async (c) => {
  const userId = c.req.header('X-User-Id') ?? 'anonymous'
  const page   = parseInt(c.req.query('page')  ?? '1')
  const limit  = parseInt(c.req.query('limit') ?? '20')
  const offset = (page - 1) * limit

  const { results } = await c.env.DB.prepare(
    'SELECT * FROM items WHERE user_id = ? ORDER BY created_at DESC LIMIT ? OFFSET ?'
  ).bind(userId, limit, offset).all()

  const { results: countResult } = await c.env.DB.prepare(
    'SELECT COUNT(*) as total FROM items WHERE user_id = ?'
  ).bind(userId).all()

  const total = (countResult[0] as { total: number }).total

  return c.json({
    items: results,
    pagination: { page, limit, total, totalPages: Math.ceil(total / limit) },
  })
})
```

Pair this with infinite scroll or numbered pages on the Rork side. For performance best practices in Rork apps, see [the complete Rork performance optimization guide](/articles/rork-dev/rork-app-performance-optimization-complete-guide).

## FAQ

**Q. Is Cloudflare D1 actually free to use?**

Yes — the free tier covers 50,000 read/write requests per day and 5 GB of storage. That's more than enough for indie projects and early-stage apps. If your app grows beyond those limits, the Workers Paid plan ($5/month) scales up capacity substantially.

**Q. Should I choose D1 over Supabase or Firebase?**

It depends on your needs. D1 shines when you want **simple CRUD + low cost + tight Cloudflare integration**. Supabase is a better choice when you need real-time subscriptions or a built-in auth system out of the box. Firebase is worth considering for apps that require deep Google ecosystem integration. For a straightforward backend without real-time sync, D1 is hard to beat on cost and simplicity.

**Q. Can I browse my D1 data from a GUI?**

Yes. Head to the Cloudflare dashboard (`dash.cloudflare.com`) → Workers & Pages → D1 to run SQL queries in your browser. You can also query directly from the terminal with `npx wrangler d1 execute rork-app-db --command "SELECT * FROM items LIMIT 10"`.

**Q. Does D1 support transactions?**

Yes. D1 supports standard SQL transactions (`BEGIN` / `COMMIT` / `ROLLBACK`), so atomic multi-table writes are possible. Keep in mind that D1 is SQLite under the hood, so some advanced PostgreSQL features (JSON operators, full-text search, etc.) are unavailable.
