# Building Offline-Ready Mobile Apps with Rork — Mastering AsyncStorage and NetInfo

One of the most frustrating user experiences in mobile applications is encountering network errors. Users expect apps to work seamlessly whether they're on a fast 5G connection, a slow cafe WiFi, or completely offline.

In this guide, we'll explore how to build genuinely offline-capable mobile apps using Rork, combining two essential React Native libraries: **AsyncStorage** for local data persistence and **NetInfo** for connectivity monitoring.

## Why Offline Support Matters

Modern mobile users are increasingly nomadic. They work on trains, in elevators, in basements, and other areas with poor or no connectivity. Apps that handle these situations gracefully gain user trust and loyalty.

With Rork's reactive state management and these powerful libraries, creating an offline-first architecture becomes straightforward.

## AsyncStorage: Local Data Persistence

AsyncStorage is a simple, asynchronous, persistent key-value store in React Native. Think of it as the browser's `localStorage` for mobile apps—perfect for caching data when the connection is unavailable.

```javascript
import AsyncStorage from '@react-native-async-storage/async-storage';

// Save data
await AsyncStorage.setItem('user_data', JSON.stringify(userData));

// Retrieve data
const stored = await AsyncStorage.getItem('user_data');
const userData = stored ? JSON.parse(stored) : null;
```

### Implementing Cache Strategy

A smart offline strategy caches API responses locally, so when the network fails, users can still access recently fetched data.

```javascript
const fetchAndCacheArticles = async () => {
  try {
    const response = await fetch('https://api.example.com/articles');
    const articles = await response.json();
    
    await AsyncStorage.setItem('articles', JSON.stringify(articles));
    await AsyncStorage.setItem('cache_time', Date.now().toString());
    return articles;
  } catch (error) {
    // On network error, load from cache
    const cached = await AsyncStorage.getItem('articles');
    return cached ? JSON.parse(cached) : [];
  }
};
```

## NetInfo: Detecting Network Changes

NetInfo allows your app to detect network status changes in real-time and respond appropriately.

```javascript
import NetInfo from '@react-native-community/netinfo';

// Check current connection
const state = await NetInfo.fetch();
console.log(state.isConnected); // true/false
console.log(state.type); // 'wifi', 'cellular', 'none'

// Listen for changes
const unsubscribe = NetInfo.addEventListener(state => {
  if (state.isConnected) {
    console.log('Back online!');
    syncPendingChanges();
  } else {
    console.log('Offline mode activated');
  }
});
```

## Building an Offline-Ready Todo App

Let's implement a complete example: a todo app that syncs changes when the connection returns.

### State Management with Rork

```javascript
import { createSlice, configureStore } from '@rorklab/state-core';

const todoSlice = createSlice({
  name: 'todos',
  initialState: {
    items: [],
    pendingSync: false,
  },
  reducers: {
    addTodo: (state, action) => {
      state.items.push({
        id: Date.now(),
        text: action.payload,
        completed: false,
        synced: false,
      });
    },
    toggleTodo: (state, action) => {
      const todo = state.items.find(t => t.id === action.payload);
      if (todo) {
        todo.completed = !todo.completed;
        todo.synced = false;
      }
    },
    markSynced: (state, action) => {
      const todo = state.items.find(t => t.id === action.payload);
      if (todo) todo.synced = true;
    },
  },
});
```

### Persistence Layer

```javascript
class TodoPersistence {
  async saveTodos(todos) {
    await AsyncStorage.setItem('todos', JSON.stringify(todos));
  }

  async loadTodos() {
    const stored = await AsyncStorage.getItem('todos');
    return stored ? JSON.parse(stored) : [];
  }

  async queueForSync(todo) {
    const queue = await AsyncStorage.getItem('sync_queue') || '[]';
    const items = JSON.parse(queue);
    items.push(todo);
    await AsyncStorage.setItem('sync_queue', JSON.stringify(items));
  }
}
```

### Connectivity Handler

```javascript
class ConnectivityHandler {
  async initialize() {
    NetInfo.addEventListener(state => {
      if (state.isConnected && state.isInternetReachable) {
        this.syncPendingTodos();
      }
    });
  }

  async syncPendingTodos() {
    const queue = JSON.parse(
      await AsyncStorage.getItem('sync_queue') || '[]'
    );
    
    if (queue.length === 0) return;

    try {
      await fetch('https://api.example.com/todos/sync', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ todos: queue }),
      });
      
      await AsyncStorage.removeItem('sync_queue');
    } catch (error) {
      console.error('Sync failed:', error);
    }
  }
}
```

## Best Practices

### 1. Always Show Network Status

Users need to know whether their app is offline. Show a clear indicator:

```javascript
<View style={[
  styles.statusBar,
  { backgroundColor: isOnline ? '#4CAF50' : '#FF9800' }
]}>
  <Text>{isOnline ? '✓ Online' : '⚠ Offline'}</Text>
</View>
```

### 2. Disable Non-Functional Features Gracefully

Don't let users attempt actions that require connectivity:

```javascript
<TouchableOpacity disabled={!isOnline}>
  <Text>{isOnline ? 'Upload Photo' : 'Upload (Offline)'}</Text>
</TouchableOpacity>
```

### 3. Handle Conflict Resolution

When syncing, newer local changes should take precedence:

```javascript
const shouldUseLocal = local.updatedAt > server.updatedAt;
```

### 4. Batch Sync Operations

Send multiple changes in a single request to reduce server load:

```javascript
const batches = chunkArray(queue, 50);
for (const batch of batches) {
  await sendBatch(batch);
}
```

### 5. Monitor Storage Limits

AsyncStorage has size limits. Clean up old data:

```javascript
async function cleanupOldData(maxAgeDays = 7) {
  const allKeys = await AsyncStorage.getAllKeys();
  const cutoffTime = Date.now() - (maxAgeDays * 24 * 60 * 60 * 1000);
  
  for (const key of allKeys) {
    const item = await AsyncStorage.getItem(key);
    if (item && !item.isEssential) {
      await AsyncStorage.removeItem(key);
    }
  }
}
```

## Testing Offline Scenarios

Always test in realistic network conditions:

1. Toggle Airplane Mode to test complete offline
2. Use Network Throttling in dev tools for slow connections
3. Simulate Network Drops mid-operation to test error handling
4. Test Sync Recovery by going offline, making changes, then coming back online

## Common Mistakes to Avoid

- Not Testing Offline: Build it, test it offline before shipping
- Ignoring Sync Failures: Always handle failed syncs gracefully
- Storing Sensitive Data: Use secure storage for passwords and tokens
- Assuming Fast Networks: Always assume networks are slow or unavailable

## Conclusion

Building offline-ready apps with Rork, AsyncStorage, and NetInfo gives your users a genuinely resilient experience. By thinking offline-first and layering sync on top, you create apps that are not only more robust but also faster and more responsive.

The pattern is simple: save locally first, sync when possible, gracefully handle failures. Your users will notice the difference.
