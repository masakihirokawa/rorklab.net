# Rork Max i18n & Localization Guide

Building apps for a global audience requires robust internationalization (i18n) support. This guide walks you through implementing professional-grade multilingual features in Rork Max using react-i18next and expo-localization, with practical examples at every step.

## Installation & Setup

```bash
npm install i18next react-i18next expo-localization

# For TypeScript projects
npm install --save-dev @types/i18next
```

<Callout type="info">
**Why both libraries?** react-i18next handles translation management and React integration, while expo-localization detects the device's language settings. Combined, they create an app that adapts automatically to the user's environment.
</Callout>

## Step 1: Organize Translation Files

Structure your translations hierarchically for maintainability and scalability.

```
src/
  locales/
    ja/
      common.json       # Shared UI labels
      home.json        # Home screen
      settings.json    # Settings screen
    en/
      common.json
      home.json
      settings.json
    es/
      common.json
      home.json
      settings.json
```

## Step 2: Initialize i18next

Create a centralized configuration file that loads all translations and detects the device language.

**src/i18n/config.ts**
```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as Localization from 'expo-localization';

// Import all translation files
import jaCommon from '../locales/ja/common.json';
import jaHome from '../locales/ja/home.json';
import enCommon from '../locales/en/common.json';
import enHome from '../locales/en/home.json';
import esCommon from '../locales/es/common.json';
import esHome from '../locales/es/home.json';

const resources = {
  ja: {
    common: jaCommon,
    home: jaHome,
  },
  en: {
    common: enCommon,
    home: enHome,
  },
  es: {
    common: esCommon,
    home: esHome,
  },
};

// Auto-detect device language
const deviceLanguage = Localization.getLocales()[0]?.languageCode || 'en';

i18n
  .use(initReactI18next)
  .init({
    resources,
    lng: deviceLanguage,
    fallbackLng: 'en',
    defaultNS: 'common',
    ns: ['common', 'home', 'settings'],
    interpolation: {
      escapeValue: false, // React handles escaping
    },
    react: {
      useSuspense: false,
    },
  });

export default i18n;
```

**src/App.tsx**
```typescript
import './i18n/config'; // Must import at the top
import { I18nextProvider } from 'react-i18next';
import i18n from './i18n/config';
import { NavigationContainer } from '@react-navigation/native';
import HomeStack from './navigation/HomeStack';

export default function App() {
  return (
    <I18nextProvider i18n={i18n}>
      <NavigationContainer>
        <HomeStack />
      </NavigationContainer>
    </I18nextProvider>
  );
}
```

<Callout type="warning">
**Critical**: Always import the i18n config at the top of App.tsx before using any components. Failure to do so will result in missing translations.
</Callout>

## Step 3: Use Translations in Components

The `useTranslation` hook gives you access to the translation function (`t`) and i18n instance.

**src/screens/Home.tsx**
```typescript
import React from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
    justifyContent: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 12,
  },
  greeting: {
    fontSize: 18,
    marginTop: 16,
    lineHeight: 24,
  },
  button: {
    marginTop: 24,
    padding: 12,
    backgroundColor: '#007AFF',
    borderRadius: 8,
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    textAlign: 'center',
    fontWeight: '600',
  },
});

export default function Home() {
  const { t, i18n } = useTranslation(['home', 'common']);

  // Variable interpolation
  const greeting = t('home:greeting', { name: 'Alice' });

  return (
    <View style={styles.container}>
      <Text style={styles.title}>{t('common:app_name')}</Text>
      <Text style={styles.greeting}>{greeting}</Text>

      {/* Language Switcher */}
      <TouchableOpacity
        style={styles.button}
        onPress={() => {
          const nextLang = i18n.language === 'en' ? 'ja' : 'en';
          i18n.changeLanguage(nextLang);
        }}
      >
        <Text style={styles.buttonText}>
          {t('common:next')} ({i18n.language === 'en' ? '日本語' : 'English'})
        </Text>
      </TouchableOpacity>
    </View>
  );
}
```

## Step 4: Handle Pluralization, Dates & Numbers

i18next supports plural forms. Use the Intl API for formatting dates and numbers based on locale.

**locales/en/home.json**
```json
{
  "tasks_count": {
    "zero": "No tasks",
    "one": "1 task",
    "other": "{{count}} tasks"
  }
}
```

**src/screens/TaskList.tsx**
```typescript
import React, { useState } from 'react';
import { View, Text, FlatList, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';

const styles = StyleSheet.create({
  container: { flex: 1, padding: 16 },
  header: { fontSize: 20, fontWeight: 'bold', marginBottom: 16 },
  text: { fontSize: 16, marginBottom: 8 },
});

export default function TaskList() {
  const { t, i18n } = useTranslation('home');
  const [tasks] = useState([
    { id: '1', title: 'Buy groceries' },
    { id: '2', title: 'Finish presentation' },
    { id: '3', title: 'Call client' },
  ]);

  const taskCount = tasks.length;

  return (
    <View style={styles.container}>
      {/* Plural handling */}
      <Text style={styles.header}>
        {t('tasks_count', { count: taskCount })}
      </Text>

      {/* Date formatting with Intl */}
      <Text style={styles.text}>
        {new Intl.DateTimeFormat(i18n.language, {
          year: 'numeric',
          month: 'long',
          day: 'numeric',
        }).format(new Date())}
      </Text>

      {/* List of tasks */}
      <FlatList
        data={tasks}
        keyExtractor={(item) => item.id}
        renderItem={({ item }) => <Text style={styles.text}>• {item.title}</Text>}
        scrollEnabled={false}
      />
    </View>
  );
}
```

## Step 5: Dynamic Language Switching

Allow users to change language from the Settings screen and persist their preference.

**src/screens/Settings.tsx**
```typescript
import React, { useState, useEffect } from 'react';
import { View, Text, ScrollView, TouchableOpacity, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';
import AsyncStorage from '@react-native-async-storage/async-storage';

const LANGUAGE_KEY = 'app_language_preference';

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  heading: { fontSize: 24, fontWeight: 'bold', marginBottom: 20 },
  optionButton: {
    padding: 16,
    marginBottom: 12,
    backgroundColor: '#F5F5F5',
    borderRadius: 8,
    borderWidth: 2,
    borderColor: 'transparent',
  },
  optionButtonActive: {
    backgroundColor: '#E3F2FD',
    borderColor: '#007AFF',
  },
  optionText: { fontSize: 16 },
});

const LANGUAGES = [
  { code: 'en', label: 'English' },
  { code: 'ja', label: '日本語' },
  { code: 'es', label: 'Español' },
  { code: 'fr', label: 'Français' },
];

export default function Settings() {
  const { t, i18n } = useTranslation();
  const [selectedLang, setSelectedLang] = useState(i18n.language);

  useEffect(() => {
    // Load saved preference on mount
    loadSavedLanguage();
  }, []);

  const loadSavedLanguage = async () => {
    try {
      const saved = await AsyncStorage.getItem(LANGUAGE_KEY);
      if (saved && saved !== i18n.language) {
        setSelectedLang(saved);
        await i18n.changeLanguage(saved);
      }
    } catch (error) {
      console.error('Failed to load language preference:', error);
    }
  };

  const handleLanguageChange = async (langCode: string) => {
    try {
      setSelectedLang(langCode);
      await i18n.changeLanguage(langCode);
      await AsyncStorage.setItem(LANGUAGE_KEY, langCode);
    } catch (error) {
      console.error('Failed to save language preference:', error);
    }
  };

  return (
    <ScrollView style={styles.container}>
      <Text style={styles.heading}>{t('common:language')}</Text>

      {LANGUAGES.map((lang) => (
        <TouchableOpacity
          key={lang.code}
          style={[
            styles.optionButton,
            selectedLang === lang.code && styles.optionButtonActive,
          ]}
          onPress={() => handleLanguageChange(lang.code)}
        >
          <Text style={styles.optionText}>{lang.label}</Text>
        </TouchableOpacity>
      ))}
    </ScrollView>
  );
}
```

## Step 6: RTL (Right-to-Left) Language Support

Support RTL languages like Arabic, Hebrew, and Urdu by configuring the I18nManager.

**src/utils/rtl.ts**
```typescript
import { I18n } from 'i18next';
import { I18nManager } from 'react-native';

const RTL_LANGUAGES = ['ar', 'he', 'ur', 'fa', 'yi'];

export function initializeRTL(i18n: I18n) {
  // Set initial RTL status
  const isRTL = RTL_LANGUAGES.includes(i18n.language);
  I18nManager.forceRTL(isRTL);

  // Listen for language changes
  i18n.on('languageChanged', (lng) => {
    const shouldBeRTL = RTL_LANGUAGES.includes(lng);
    I18nManager.forceRTL(shouldBeRTL);
  });
}
```

**src/App.tsx**
```typescript
import { initializeRTL } from './utils/rtl';
import i18n from './i18n/config';

useEffect(() => {
  initializeRTL(i18n);
}, []);
```

<Callout type="info">
**RTL Best Practices**: When I18nManager.forceRTL(true) is set, flexDirection automatically reverses. Avoid direction-dependent styles like marginLeft; use marginStart and marginEnd instead, or rely on flex layout.
</Callout>

## Best Practices

### 1. Namespace Organization
```typescript
// Load only the namespaces you need
const { t } = useTranslation(['home', 'profile']);

// Access with namespace prefix
t('home:welcome')  // ✓ Good
t('profile:name')  // ✓ Good
t('unknown:key')   // ✗ Will not work
```

### 2. Type-Safe Translation Keys
```typescript
// src/types/i18n.ts
export const TRANSLATION_KEYS = {
  common: ['welcome', 'back', 'next', 'save'] as const,
  home: ['title', 'greeting', 'tasks'] as const,
  settings: ['language', 'theme', 'notifications'] as const,
} as const;

type TranslationKey =
  | typeof TRANSLATION_KEYS.common[number]
  | typeof TRANSLATION_KEYS.home[number];
```

### 3. Validate Translation Completeness
```typescript
// scripts/validate-translations.js
const fs = require('fs');
const path = require('path');

const locales = ['en', 'ja', 'es'];
const namespaces = ['common', 'home', 'settings'];

for (const ns of namespaces) {
  const keys = {};

  for (const locale of locales) {
    const filePath = path.join(
      __dirname,
      `../src/locales/${locale}/${ns}.json`
    );
    const data = JSON.parse(fs.readFileSync(filePath, 'utf-8'));
    const fileKeys = Object.keys(data).sort();

    if (!keys[locale]) keys[locale] = fileKeys;

    if (fileKeys.toString() !== keys[locales[0]].toString()) {
      console.error(
        `Missing translations in ${locale}/${ns}.json`
      );
      process.exit(1);
    }
  }
}
console.log('✓ All translations complete');
```

### 4. Lazy-Load Large Translation Files
```typescript
import i18next from 'i18next';
import HttpBackend from 'i18next-http-backend';

i18next
  .use(HttpBackend)
  .init({
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
  });
```

## Summary

Implementing i18n in Rork Max requires careful planning but pays off with global reach:

- **File Organization**: Structure translations by namespace and locale
- **Auto-Detection**: Leverage expo-localization for device language
- **Dynamic Switching**: Let users choose their language with persistence
- **RTL Support**: Automatically adapt layout for right-to-left languages
- **Type Safety**: Validate keys and prevent missing translations
- **App Store Ready**: Localize metadata for each market

With these foundations, you can confidently build apps for a truly global audience.
