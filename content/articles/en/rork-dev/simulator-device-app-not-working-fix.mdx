Running a Rork app on a simulator or physical device can expose runtime issues that don't appear during the build phase. Apps may fail to launch, crash immediately, or display incorrectly. These runtime errors require a different debugging approach than build errors. This guide systematically covers iOS and Android-specific problems and their solutions.

## Diagnostic Flow for Runtime Errors

When an app doesn't work, diagnose in this order:

1. **Did build succeed?** → Check console output
2. **Is simulator/device running?** → Verify device list
3. **Does app launch?** → Check logs
4. **Is app crashing?** → Read stack trace
5. **Is display correct?** → Use layout debugging

## iOS-Specific Issues and Solutions

### 1. iOS App Won't Install: "App Not Installed"

The simulator fails to install or the app crashes immediately upon launch.

**Root Causes**:
- Bundle ID mismatch
- Signing & Capabilities misconfiguration
- Stale simulator cache

**Solution Steps**:

```bash
# Step 1: Verify Bundle ID in Xcode
# Xcode → [Project] → Targets → General
# Check Bundle Identifier is in format: com.yourcompany.appname

# Step 2: Reset simulator
xcrun simctl erase all        # Reset all simulators
# or reset specific simulator
xcrun simctl erase "iPhone 15"

# Step 3: Clear DerivedData
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# Step 4: Rebuild and run
npm start

# Step 5: Running from Xcode directly
open ios/YourApp.xcworkspace
# Cmd + R to build and run
```

**Signing & Capabilities Checklist**:

```
Xcode → Targets → Signing & Capabilities
- Team: Development team selected
- Bundle Identifier: Unique identifier set
- Automatic signing: Enabled
```

### 2. iOS Simulator Crash: "EXC_BAD_ACCESS"

Memory access error causing immediate crash.

**Root Causes**:
- Memory leak
- Null reference
- Type error at native code boundary

**Solution**:

```bash
# Step 1: Enable debug symbols
# Xcode → Scheme → Edit Scheme → Run
# → Build Configuration: Debug (required)

# Step 2: Read crash log in Xcode console
# Cmd + Shift + Y to show console
# Identify the crashing thread and function

# Step 3: Enable Address Sanitizer
# Xcode → Scheme → Edit Scheme → Run → Diagnostics
# → Address Sanitizer: ON
# Provides detailed error information

# Step 4: Enable strict TypeScript checking
# tsconfig.json: "strict": true
```

Reading console output:

```
Thread 0 Crashed:
0   libsystem_kernel.dylib     0x00007fff2xxx xxxx
1   MyApp                      0x00010xxxx+xxx  [file.js:50]
```

Line 1 (MyApp) is your actual error location. Check the filename and line number.

### 3. iOS Runtime Error: "Cannot read property of undefined"

JavaScript runtime error.

**Root Causes**:
- Accessing properties before initialization
- API response structure changed
- Async timing issues

**Solution**:

```typescript
// ❌ Wrong: No null checking
const userName = user.profile.name;

// ✅ Correct: Safe property access
const userName = user?.profile?.name ?? 'Anonymous';

// Or use strict typing
interface User {
  profile?: {
    name: string;
  };
}

const user: User = {};
const userName = user.profile?.name ?? 'Anonymous';
```

API Response Validation:

```typescript
// Always validate API responses
const fetchUser = async () => {
  const response = await fetch('/api/user');
  if (!response.ok) {
    console.error('API Error:', response.status);
    return;
  }

  const data = await response.json();

  // Verify required fields exist
  if (!data?.id || !data?.name) {
    console.error('Invalid response structure');
    return;
  }

  return data;
};
```

## Android-Specific Issues and Solutions

### 1. Android App Won't Launch: "App crashes on launch"

App terminates immediately after launch on emulator.

**Root Causes**:
- Insufficient emulator memory
- AndroidManifest misconfiguration
- Native code compatibility issue

**Solution Steps**:

```bash
# Step 1: Restart emulator with fresh data
emulator -avd Pixel_6a -wipe-data

# Step 2: Check Android logs for crash
adb logcat | grep "FATAL EXCEPTION"
# or view detailed logs
adb logcat | grep -i "error\|crash"

# Step 3: Increase emulator memory allocation
# Android Studio → Virtual Device Manager
# → Edit → Memory/RAM: Set to 4GB or higher

# Step 4: Verify AndroidManifest.xml
# android/app/src/main/AndroidManifest.xml
# Check <uses-permission> declarations
```

AndroidManifest.xml Example:

```xml
<manifest>
  <!-- Internet permission for network calls -->
  <uses-permission android:name="android.permission.INTERNET" />

  <!-- Camera permission -->
  <uses-permission android:name="android.permission.CAMERA" />

  <!-- Activity configuration -->
  <application>
    <activity
      android:name=".MainActivity"
      android:exported="true">
      <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
      </intent-filter>
    </activity>
  </application>
</manifest>
```

### 2. Android Memory Error: "OutOfMemoryError"

Heap memory exhaustion, especially during image or data processing.

**Root Causes**:
- Loading large images without compression
- Memory leak from unremoved listeners
- Unbounded cache growth

**Solution**:

```typescript
// Load images efficiently
import { Image } from 'react-native';

// ❌ Wrong: Load large image without optimization
<Image
  source={{ uri: 'https://example.com/huge-image.jpg' }}
  style={{ width: 300, height: 300 }}
/>

// ✅ Correct: Use caching and compression library
import FastImage from 'react-native-fast-image';

<FastImage
  source={{ uri: 'https://example.com/huge-image.jpg' }}
  style={{ width: 300, height: 300 }}
  cache={FastImage.cacheControl.moderate}
/>

// Clean up listeners in useEffect
useEffect(() => {
  const subscription = someEmitter.on('event', handler);

  return () => {
    subscription.remove(); // Critical: remove listener
  };
}, []);
```

Monitor Memory:

```bash
# Real-time memory monitoring
adb shell dumpsys meminfo com.example.myapp

# Detailed heap information
adb shell am dumpheap com.example.myapp /data/local/tmp/heap.bin
```

### 3. Android Permission Error: "Permission denied"

Runtime permission handling failure.

**Root Causes**:
- Missing AndroidManifest.xml declaration
- Runtime permission not requested
- Permission name misspelled

**Solution**:

```typescript
import { PermissionsAndroid } from 'react-native';

// Request runtime permission (Android 6.0+)
const requestCameraPermission = async () => {
  try {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.CAMERA,
      {
        title: "Camera Permission",
        message: "This app needs camera access",
        buttonNeutral: "Ask Me Later",
        buttonNegative: "Cancel",
        buttonPositive: "OK",
      },
    );

    if (granted === PermissionsAndroid.RESULTS.GRANTED) {
      console.log("Permission granted");
    } else {
      console.log("Permission denied");
    }
  } catch (err) {
    console.warn(err);
  }
};
```

AndroidManifest.xml (required for Android <6.0):

```xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
```

## Common Layout and Display Issues

### 1. Screen Not Displaying Correctly (Layout Broken)

**Root Causes**:
- Screen size variations not handled
- SafeAreaInsets not implemented
- Flexbox layout bug

**Solution**:

```typescript
import { SafeAreaView, useWindowDimensions } from 'react-native';

// SafeAreaView handles notches and home indicators
<SafeAreaView style={{ flex: 1 }}>
  <View style={{ flex: 1 }}>
    {/* Your content */}
  </View>
</SafeAreaView>

// Responsive layout
const { width, height } = useWindowDimensions();

const styles = StyleSheet.create({
  container: {
    flex: 1,
    width: '100%',                    // Relative width
    padding: width > 600 ? 20 : 10,   // Responsive padding
  },
});
```

### 2. White Screen of Death

**Root Causes**:
- Error in entry point
- Navigation initialization failure
- AsyncStorage initialization delay

**Solution**:

```typescript
// ❌ Wrong: Synchronous render
export default function App() {
  return <NavigationContainer>{/* ... */}</NavigationContainer>;
}

// ✅ Correct: Wait for async initialization
import { useState, useEffect } from 'react';

export default function App() {
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    const bootstrapAsync = async () => {
      try {
        // Initialize async operations (AsyncStorage, auth, etc.)
        await new Promise(resolve => setTimeout(resolve, 1000));
      } catch (e) {
        console.log(e);
      } finally {
        setIsReady(true);
      }
    };

    bootstrapAsync();
  }, []);

  if (!isReady) {
    return <SplashScreen />;  // Show loading
  }

  return <NavigationContainer>{/* ... */}</NavigationContainer>;
}
```

## Frequently Asked Questions

**Q1: iOS Simulator shows "Simulator is slower"**

A: Restart the simulator:
```bash
xcrun simctl shutdown all
open -a Simulator
```

**Q2: Android USB debugging won't connect**

A: Verify device developer settings:
```bash
# Enable USB debugging in device settings
adb devices              # Verify connection
adb reverse tcp:8081 tcp:8081  # Port forwarding
```

**Q3: App launches but buttons don't respond**

A: Check Pressability:
```typescript
<Pressable onPress={() => console.log('pressed')}>
  <Text>Button</Text>
</Pressable>

// Or use TouchableOpacity
<TouchableOpacity onPress={handlePress}>
  <Text>Button</Text>
</TouchableOpacity>
```

## Summary

Runtime errors combine application and device-level issues. Key diagnostic principles:

1. **Read logs** — Check console and Logcat for detailed messages
2. **Verify platform configs** — AndroidManifest (Android) and Signing (iOS)
3. **Reset device/simulator** — Stale caches cause most issues
4. **Enforce type safety** — TypeScript strict mode prevents many runtime errors
5. **Manage memory carefully** — Image processing and listener cleanup are critical

Following this systematic approach and understanding platform-specific requirements will resolve most runtime issues in Rork development.
