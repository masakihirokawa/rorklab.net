Seamless integration between Figma and Rork app development is essential for building scalable mobile UIs. However, misalignment between design and engineering specifications leads to wasteful revision cycles and delayed releases.

This comprehensive guide covers the **complete Figma-to-React Native workflow**. From UI Stack five-state management through AutoLayout conversion rules, design token theming, and dark mode support, you'll find production-ready implementation patterns for each stage.

## Figma's Role in Rork Development and the Complete Workflow

Figma is not merely a design tool—it's the **shared language between designers and engineers**. Proper governance dramatically improves handoff quality and minimizes revision cycles.

The typical Rork app development workflow looks like this:

1. Designers build a component library in Figma (Atomic Design compliant)
2. UI Stack (5 states) is explicitly defined in Figma
3. Figma Variables centralize design tokens (colors, sizes, spacing)
4. Engineers implement React Native components based on this specification
5. Dev Mode + MCP enables automated code generation workflows
6. Context API + Hooks handle dark mode and internationalization

This process ensures design changes propagate quickly to code while maintaining design consistency.

## Implementing UI Stack in Mobile Apps

UI Stack is the concept that every UI component exists in five distinct states. Integrating this into Rork apps dramatically improves error handling and user experience.

### Five State Definitions

- **Ideal**: The happy path. Data displays as expected
- **Partial**: Data is partially loaded, skeleton state
- **Empty**: No data exists (empty search results, blank list)
- **Error**: An error occurred during data fetch or processing
- **Loading**: Data is being fetched

### Designing UI Stack in Figma

In Figma, each component includes these five states as **variants**.

```
ProductCard (Component)
├── Ideal
├── Partial (skeleton display)
├── Empty
├── Error
└── Loading
```

### React Native Implementation

```javascript
import React, { useState, useEffect } from 'react';
import { View, Text, ActivityIndicator, StyleSheet, TouchableOpacity } from 'react-native';

export const ProductCard = ({
  productId,
  onRetry
}) => {
  const [state, setState] = useState('loading'); // loading | ideal | error | empty
  const [product, setProduct] = useState(null);

  useEffect(() => {
    fetchProduct(productId)
      .then((data) => {
        if (data) {
          setProduct(data);
          setState('ideal');
        } else {
          setState('empty');
        }
      })
      .catch(() => setState('error'));
  }, [productId]);

  const renderContent = () => {
    switch (state) {
      case 'loading':
        return <SkeletonLoader />;
      case 'ideal':
        return (
          <View>
            <Text style={styles.title}>{product.name}</Text>
            <Text style={styles.price}>${product.price}</Text>
          </View>
        );
      case 'empty':
        return <Text style={styles.emptyText}>No product found</Text>;
      case 'error':
        return (
          <View>
            <Text style={styles.errorText}>An error occurred</Text>
            <TouchableOpacity onPress={onRetry}>
              <Text style={styles.retryButton}>Retry</Text>
            </TouchableOpacity>
          </View>
        );
      default:
        return null;
    }
  };

  return <View style={styles.card}>{renderContent()}</View>;
};

const styles = StyleSheet.create({
  card: { padding: 16, borderRadius: 8, backgroundColor: '#fff' },
  title: { fontSize: 16, fontWeight: 'bold' },
  price: { fontSize: 14, color: '#666' },
  emptyText: { textAlign: 'center', color: '#999' },
  errorText: { color: '#d32f2f', marginBottom: 8 },
  retryButton: { color: '#1976d2', fontWeight: '600' },
});
```

### Error Boundary and Retry UI

Error boundaries gracefully catch unexpected errors and display user-friendly messages.

```javascript
export class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  render() {
    if (this.state.hasError) {
      return (
        <View style={styles.errorContainer}>
          <Text style={styles.errorTitle}>Something went wrong</Text>
          <Text style={styles.errorMessage}>{this.state.error?.message}</Text>
          <TouchableOpacity
            onPress={() => this.setState({ hasError: false })}
          >
            <Text style={styles.retryButton}>Try again</Text>
          </TouchableOpacity>
        </View>
      );
    }
    return this.props.children;
  }
}
```

## Figma AutoLayout to React Native Flexbox Conversion Rules

Figma's AutoLayout closely mirrors web Flexbox concepts. Mastering the conversion is key to implementation efficiency.

### Conversion Mapping

| Figma AutoLayout | React Native Flexbox |
|---|---|
| Direction: Horizontal | `flexDirection: 'row'` |
| Direction: Vertical | `flexDirection: 'column'` |
| Gap | `gap` (RN 0.71+) |
| Padding | `padding` / `paddingHorizontal` / `paddingVertical` |
| Align: Left | `alignItems: 'flex-start'` |
| Align: Center | `alignItems: 'center'` |
| Align: Right | `alignItems: 'flex-end'` |
| Justify: Left | `justifyContent: 'flex-start'` |
| Justify: Center | `justifyContent: 'center'` |
| Justify: Space Between | `justifyContent: 'space-between'` |

### Concrete Conversion Examples

**Figma Settings**:
- Direction: Horizontal
- Gap: 8
- Padding: 16 (all sides)
- Align: Center

**React Native Code**:

```javascript
const styles = StyleSheet.create({
  container: {
    flexDirection: 'row',
    gap: 8,
    padding: 16,
    alignItems: 'center',
  },
  item: {
    flex: 1,
  },
});

export const HorizontalList = ({ items }) => (
  <View style={styles.container}>
    {items.map((item) => (
      <View key={item.id} style={styles.item}>
        <Text>{item.label}</Text>
      </View>
    ))}
  </View>
);
```

## Figma Variables to React Native Theme Conversion

Maintaining design system consistency requires centralizing design tokens (colors, sizes, spacing).

### Design Token Management Strategy

Figma Variables organize into three layers:

1. **Primitive**: Base values like `#1976d2`
2. **Semantic**: Meaningful values like "Primary Color"
3. **Alias**: References to other tokens

```
Primitive Layer:
  colors/blue-500: #1976d2
  colors/gray-50: #f5f5f5

Semantic Layer:
  colors/primary: colors/blue-500
  colors/background: colors/gray-50

Alias Layer:
  button/primary-bg: colors/primary
  button/disabled-bg: colors/gray-50
```

### Dark Mode Implementation with useColorScheme

React Native's `useColorScheme` Hook automatically selects the theme based on system settings.

```javascript
import {
  useColorScheme,
  StyleSheet,
} from 'react-native';
import { LIGHT_THEME, DARK_THEME } from './theme';

const ThemeContext = React.createContext();

export const ThemeProvider = ({ children }) => {
  const colorScheme = useColorScheme();
  const theme = colorScheme === 'dark' ? DARK_THEME : LIGHT_THEME;

  return (
    <ThemeContext.Provider value={theme}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => React.useContext(ThemeContext);

// Usage in components
export const Button = ({ title, onPress }) => {
  const theme = useTheme();
  const styles = StyleSheet.create({
    button: {
      backgroundColor: theme.colors.primary,
      padding: theme.spacing.md,
      borderRadius: 4,
    },
    text: { color: '#fff', fontWeight: 'bold' },
  });

  return (
    <TouchableOpacity style={styles.button} onPress={onPress}>
      <Text style={styles.text}>{title}</Text>
    </TouchableOpacity>
  );
};
```

## Automated Component Generation with Figma Dev Mode MCP

Combining Figma's Dev Mode with MCP (Model Context Protocol) enables automatic React Native component scaffold generation from design specifications.

### MCP Setup

1. Enable Figma Dev Mode
2. Configure MCP gateway (Figma Plugin)
3. Scan design file for components
4. Execute automated code generation


Add business logic to the generated scaffolds to complete implementation.

```javascript
export const ProductListScreen = () => {

  const [products, setProducts] = useState([]);

          setProducts(data);

  const renderContent = () => {
```

## Frequently Asked Questions

**Q1: Doesn't UI Stack make Figma files overly complex?**

A: Initial complexity is mitigated by Figma's Component Set feature. The long-term benefits in quality consistency and development velocity far outweigh setup costs.

**Q2: Is dark mode support mandatory?**

A: iOS 13+ and Android 10+ standardized dark mode at the OS level. Supporting it is essential for modern user experience.

**Q3: How do I manage 100+ design tokens?**

A: Group tokens by category (Colors, Spacing, Typography) and enforce naming conventions. Conduct quarterly audits and optimization reviews.

**Q4: Can I use code generated by Dev Mode MCP directly?**

A: Generated code provides structural scaffolds only. Add business logic, state management, and error handling manually.

**Q5: When design changes frequently, how should I adapt the workflow?**

A: Position Figma as the single source of truth. Document impacts immediately via annotations. Schedule weekly sync sessions.
