## Why State Management Is the Key to Rork App Development

When building apps with Rork, AI handles much of the UI component generation for you. But as your app grows, data flows become complex, and without proper state management, bugs creep in fast.

Sharing data across screens, managing async API data, keeping the app functional offline — these requirements all demand a solid state management architecture.

This guide walks through state management patterns for Rork apps, from small-scale to production-grade, with code examples for every pattern so you can pick what fits your project.

## Pattern 1: Component Local State — The Small App Foundation

The simplest pattern. Each component manages its own state.

```typescript
// components/Counter.tsx — Local state basics
import React, { useState, useCallback } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export const Counter: React.FC = () => {
  // Local state — only exists within this component
  const [count, setCount] = useState(0);

  const increment = useCallback(() => {
    setCount(prev => prev + 1);
  }, []);

  const decrement = useCallback(() => {
    setCount(prev => Math.max(0, prev - 1));
  }, []);

  return (
    <View style={styles.container}>
      <TouchableOpacity onPress={decrement} style={styles.button}>
        <Text style={styles.buttonText}>-</Text>
      </TouchableOpacity>
      <Text style={styles.count}>{count}</Text>
      <TouchableOpacity onPress={increment} style={styles.button}>
        <Text style={styles.buttonText}>+</Text>
      </TouchableOpacity>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flexDirection: 'row', alignItems: 'center', gap: 16 },
  button: {
    width: 44, height: 44, borderRadius: 22,
    backgroundColor: '#007AFF', justifyContent: 'center', alignItems: 'center'
  },
  buttonText: { color: '#fff', fontSize: 20, fontWeight: 'bold' },
  count: { fontSize: 24, fontWeight: '600', minWidth: 40, textAlign: 'center' }
});
```

This pattern is ideal when state is confined to a single component — form input values, toggle states, animation progress, and similar localized data.

## Pattern 2: Context + useReducer — Global State for Mid-Size Apps

When multiple screens need to share data, combine the Context API with `useReducer`.

```typescript
// store/AppContext.tsx — Global state management
import React, { createContext, useContext, useReducer, ReactNode } from 'react';

// Type definitions
interface AppState {
  user: { id: string; name: string; email: string } | null;
  theme: 'light' | 'dark';
  notifications: Notification[];
  isLoading: boolean;
}

interface Notification {
  id: string;
  message: string;
  read: boolean;
  createdAt: string;
}

type AppAction =
  | { type: 'SET_USER'; payload: AppState['user'] }
  | { type: 'SET_THEME'; payload: AppState['theme'] }
  | { type: 'ADD_NOTIFICATION'; payload: Notification }
  | { type: 'MARK_READ'; payload: string }
  | { type: 'SET_LOADING'; payload: boolean };

// Initial state
const initialState: AppState = {
  user: null,
  theme: 'light',
  notifications: [],
  isLoading: false,
};

// Reducer — pure function defining state transitions
function appReducer(state: AppState, action: AppAction): AppState {
  switch (action.type) {
    case 'SET_USER':
      return { ...state, user: action.payload };
    case 'SET_THEME':
      return { ...state, theme: action.payload };
    case 'ADD_NOTIFICATION':
      return {
        ...state,
        notifications: [action.payload, ...state.notifications]
      };
    case 'MARK_READ':
      return {
        ...state,
        notifications: state.notifications.map(n =>
          n.id === action.payload ? { ...n, read: true } : n
        ),
      };
    case 'SET_LOADING':
      return { ...state, isLoading: action.payload };
    default:
      return state;
  }
}

// Create context
const AppContext = createContext<{
  state: AppState;
  dispatch: React.Dispatch<AppAction>;
} | null>(null);

// Provider component
export const AppProvider: React.FC<{ children: ReactNode }> = ({ children }) => {
  const [state, dispatch] = useReducer(appReducer, initialState);

  return (
    <AppContext.Provider value={{ state, dispatch }}>
      {children}
    </AppContext.Provider>
  );
};

// Custom hook — type-safe access
export function useAppState() {
  const context = useContext(AppContext);
  if (!context) {
    throw new Error('useAppState must be used within AppProvider');
  }
  return context;
}
```

Usage:

```typescript
// screens/ProfileScreen.tsx
import { useAppState } from '../store/AppContext';

export const ProfileScreen: React.FC = () => {
  const { state, dispatch } = useAppState();

  const toggleTheme = () => {
    dispatch({
      type: 'SET_THEME',
      payload: state.theme === 'light' ? 'dark' : 'light'
    });
  };

  return (
    <View>
      <Text>Welcome, {state.user?.name ?? 'Guest'}</Text>
      <Text>Theme: {state.theme}</Text>
      <TouchableOpacity onPress={toggleTheme}>
        <Text>Toggle Theme</Text>
      </TouchableOpacity>
    </View>
  );
};
```

## Pattern 3: Zustand Store — The Go-To for Large Apps

For large-scale apps, Zustand offers the best balance of simplicity and power. Minimal boilerplate with excellent TypeScript support.

```typescript
// store/useStore.ts — Zustand store
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface Product {
  id: string;
  name: string;
  price: number;
  image: string;
}

interface CartItem extends Product {
  quantity: number;
}

interface StoreState {
  // Cart
  cart: CartItem[];
  addToCart: (product: Product) => void;
  removeFromCart: (productId: string) => void;
  updateQuantity: (productId: string, quantity: number) => void;
  clearCart: () => void;
  getCartTotal: () => number;

  // Favorites
  favorites: string[];
  toggleFavorite: (productId: string) => void;
  isFavorite: (productId: string) => boolean;

  // Search history
  searchHistory: string[];
  addSearchQuery: (query: string) => void;
  clearSearchHistory: () => void;
}

export const useStore = create<StoreState>()(
  persist(
    (set, get) => ({
      // Cart
      cart: [],

      addToCart: (product) => set((state) => {
        const existing = state.cart.find(item => item.id === product.id);
        if (existing) {
          return {
            cart: state.cart.map(item =>
              item.id === product.id
                ? { ...item, quantity: item.quantity + 1 }
                : item
            ),
          };
        }
        return { cart: [...state.cart, { ...product, quantity: 1 }] };
      }),

      removeFromCart: (productId) => set((state) => ({
        cart: state.cart.filter(item => item.id !== productId),
      })),

      updateQuantity: (productId, quantity) => set((state) => ({
        cart: quantity <= 0
          ? state.cart.filter(item => item.id !== productId)
          : state.cart.map(item =>
              item.id === productId ? { ...item, quantity } : item
            ),
      })),

      clearCart: () => set({ cart: [] }),

      getCartTotal: () => {
        return get().cart.reduce(
          (total, item) => total + item.price * item.quantity, 0
        );
      },

      // Favorites
      favorites: [],

      toggleFavorite: (productId) => set((state) => ({
        favorites: state.favorites.includes(productId)
          ? state.favorites.filter(id => id !== productId)
          : [...state.favorites, productId],
      })),

      isFavorite: (productId) => {
        return get().favorites.includes(productId);
      },

      // Search history
      searchHistory: [],

      addSearchQuery: (query) => set((state) => ({
        searchHistory: [
          query,
          ...state.searchHistory.filter(q => q !== query)
        ].slice(0, 20),  // Keep latest 20
      })),

      clearSearchHistory: () => set({ searchHistory: [] }),
    }),
    {
      name: 'app-storage',
      storage: createJSONStorage(() => AsyncStorage),
      partialize: (state) => ({
        cart: state.cart,
        favorites: state.favorites,
        searchHistory: state.searchHistory,
      }),
    }
  )
);
```

## Pattern 4: Async Data Management — TanStack Query Integration

Server-fetched data should be managed separately from local state. That's a fundamental rule.

```typescript
// hooks/useProducts.ts — Data fetching with TanStack Query
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';

interface Product {
  id: string;
  name: string;
  price: number;
  description: string;
  category: string;
}

// API functions
const api = {
  getProducts: async (category?: string): Promise<Product[]> => {
    const url = category
      ? `/api/products?category=${category}`
      : '/api/products';
    const res = await fetch(url);
    if (!res.ok) throw new Error('Failed to fetch products');
    return res.json();
  },

  getProduct: async (id: string): Promise<Product> => {
    const res = await fetch(`/api/products/${id}`);
    if (!res.ok) throw new Error('Product not found');
    return res.json();
  },
};

// Fetch product list
export function useProducts(category?: string) {
  return useQuery({
    queryKey: ['products', category],
    queryFn: () => api.getProducts(category),
    staleTime: 5 * 60 * 1000,   // Use cache for 5 minutes
    gcTime: 30 * 60 * 1000,     // Garbage collect after 30 minutes
    retry: 2,
  });
}

// Fetch product details
export function useProduct(id: string) {
  return useQuery({
    queryKey: ['product', id],
    queryFn: () => api.getProduct(id),
    enabled: !!id,  // Only fetch when id is available
  });
}

// Usage
// const { data: products, isLoading, error } = useProducts('electronics');
```

## Pattern 5: Offline Support — Optimistic Updates and Queuing

Mobile apps frequently encounter unstable network connections. Here's a pattern that keeps your app functional offline:

```typescript
// store/offlineQueue.ts — Offline action queue
import NetInfo from '@react-native-community/netinfo';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface QueuedAction {
  id: string;
  type: string;
  payload: any;
  timestamp: number;
  retryCount: number;
}

class OfflineQueue {
  private queue: QueuedAction[] = [];
  private isProcessing = false;
  private readonly STORAGE_KEY = 'offline_queue';
  private readonly MAX_RETRIES = 3;

  async init() {
    // Restore persisted queue
    const stored = await AsyncStorage.getItem(this.STORAGE_KEY);
    if (stored) {
      this.queue = JSON.parse(stored);
    }

    // Monitor network state
    NetInfo.addEventListener(state => {
      if (state.isConnected && this.queue.length > 0) {
        this.processQueue();
      }
    });
  }

  async enqueue(action: Omit<QueuedAction, 'id' | 'timestamp' | 'retryCount'>) {
    const queuedAction: QueuedAction = {
      ...action,
      id: `${Date.now()}-${Math.random().toString(36).slice(2)}`,
      timestamp: Date.now(),
      retryCount: 0,
    };

    this.queue.push(queuedAction);
    await this.persist();

    // Process immediately if online
    const netState = await NetInfo.fetch();
    if (netState.isConnected) {
      this.processQueue();
    }
  }

  private async processQueue() {
    if (this.isProcessing || this.queue.length === 0) return;
    this.isProcessing = true;

    while (this.queue.length > 0) {
      const action = this.queue[0];

      try {
        await this.executeAction(action);
        this.queue.shift(); // Remove on success
        await this.persist();
      } catch (error) {
        action.retryCount++;
        if (action.retryCount >= this.MAX_RETRIES) {
          console.error(`Action failed after ${this.MAX_RETRIES} retries:`, action);
          this.queue.shift();
          await this.persist();
        } else {
          break; // Pause for retry
        }
      }
    }

    this.isProcessing = false;
  }

  private async executeAction(action: QueuedAction): Promise<void> {
    const response = await fetch(`/api/${action.type}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(action.payload),
    });

    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }
  }

  private async persist() {
    await AsyncStorage.setItem(
      this.STORAGE_KEY,
      JSON.stringify(this.queue)
    );
  }
}

export const offlineQueue = new OfflineQueue();
```

## Performance Optimization Tips

Here are practical techniques to keep your state management performant.

### Prevent Unnecessary Re-renders with Selectors

```typescript
// Zustand selector pattern
import { useStore } from './store/useStore';
import { shallow } from 'zustand/shallow';

// ❌ Bad — subscribes to entire store (re-renders on any change)
const Component = () => {
  const store = useStore();
  return <Text>{store.cart.length}</Text>;
};

// ✅ Good — subscribes only to what you need
const Component = () => {
  const cartCount = useStore(state => state.cart.length);
  return <Text>{cartCount}</Text>;
};

// ✅ Multiple values with shallow comparison
const Component = () => {
  const { cart, favorites } = useStore(
    state => ({ cart: state.cart, favorites: state.favorites }),
    shallow
  );
  return <Text>{cart.length} / {favorites.length}</Text>;
};
```

### Reduce Computation Cost with Memoization

```typescript
import { useMemo } from 'react';

const CartSummary = () => {
  const cart = useStore(state => state.cart);

  // Memoize expensive calculations
  const summary = useMemo(() => ({
    totalItems: cart.reduce((sum, item) => sum + item.quantity, 0),
    totalPrice: cart.reduce((sum, item) => sum + item.price * item.quantity, 0),
    uniqueItems: cart.length,
  }), [cart]);

  return (
    <View>
      <Text>Items: {summary.totalItems}</Text>
      <Text>Total: ${summary.totalPrice.toLocaleString()}</Text>
    </View>
  );
};
```

## FAQ

### Q: Which pattern should I choose?

Let your app's scale guide you. Under 5 screens, local state plus Context is fine. Over 10 screens, reach for Zustand. Heavy server data? Add TanStack Query to the mix.

### Q: Why Zustand over Redux?

Zustand has dramatically less boilerplate with naturally flowing TypeScript inference. Since development speed matters in Rork apps, the lower setup overhead makes Zustand the better fit.

### Q: How much offline support do I really need?

It depends on your users' context. For e-commerce, "add to cart" and "favorite" actions should work offline. For a news app, caching articles for reading might be enough.

## Wrapping Up — Choose the Right State Management for Your Project

The golden rule for Rork app state management: start small and expand as needed. You don't need to adopt Zustand or TanStack Query from day one. Layer in the right patterns as your app grows.

What matters most is keeping your data flow traceable. When you can clearly see which screens reference which data and how updates propagate, your app stays maintainable no matter how large it gets.
