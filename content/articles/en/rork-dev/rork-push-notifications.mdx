Push notifications are one of the most effective tools for keeping users engaged with your app. Whether you're alerting users about new messages, announcing a sale, or reminding them of upcoming deadlines, notifications bring people back. This guide walks you through implementing push notifications in a Rork-generated app using Expo Notifications—from device token registration to backend delivery.

## How Push Notifications Work with Rork

All apps generated by Rork are built on **Expo (React Native)**. Expo provides the `expo-notifications` library, which gives you a unified API for both iOS (APNs) and Android (FCM) push notifications.

Here's the end-to-end flow:

```
App launches
  ↓
expo-notifications requests permission & retrieves device token
  ↓
Token saved to backend (Supabase, etc.)
  ↓
Server calls Expo Push API with token + message
  ↓
Expo Push Service forwards to APNs / FCM
  ↓
Notification delivered to user's device
```

Using the Expo Push Service means you don't have to manage APNs certificates or FCM credentials yourself—Expo handles the relay, making this approach ideal for solo developers and small teams.

## Step 1: Set Up expo-notifications

Send Rork a prompt like this to scaffold the entire setup automatically:

```
Implement push notifications using expo-notifications.
Retrieve the device token and log it to the console.
Show an alert when a notification is received.
```

Rork will update `app.json`, install the necessary packages, and generate the notification handler. The resulting hook looks like this:

```typescript
// hooks/usePushNotifications.ts
// Custom hook to manage push notification setup and token retrieval

import { useEffect, useRef, useState } from 'react';
import * as Notifications from 'expo-notifications';
import * as Device from 'expo-device';
import Constants from 'expo-constants';
import { Platform } from 'react-native';

// Show alert banner + play sound when a notification is received in the foreground
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,   // display banner
    shouldPlaySound: true,   // play notification sound
    shouldSetBadge: false,   // leave badge count unchanged
  }),
});

export function usePushNotifications() {
  const [expoPushToken, setExpoPushToken] = useState<string | null>(null);
  const [notification, setNotification] = useState<Notifications.Notification | null>(null);
  const notificationListener = useRef<Notifications.EventSubscription>();
  const responseListener = useRef<Notifications.EventSubscription>();

  useEffect(() => {
    async function registerForPushNotifications() {
      // Push notifications require a physical device
      if (!Device.isDevice) {
        console.warn('Push notifications are only supported on physical devices.');
        return;
      }

      // Request notification permission
      const { status: existingStatus } = await Notifications.getPermissionsAsync();
      let finalStatus = existingStatus;

      if (existingStatus !== 'granted') {
        const { status } = await Notifications.requestPermissionsAsync();
        finalStatus = status;
      }

      if (finalStatus !== 'granted') {
        console.warn('Push notification permission was denied.');
        return;
      }

      // Retrieve the Expo push token using the EAS project ID
      const projectId = Constants.expoConfig?.extra?.eas?.projectId;
      const tokenData = await Notifications.getExpoPushTokenAsync({ projectId });
      const token = tokenData.data;

      console.log('Expo Push Token:', token);
      // Expected output: ExponentPushToken[xxxxxxxxxxxxxxxxxxxxxx]
      setExpoPushToken(token);

      // Android requires a notification channel (Android 8.0+)
      if (Platform.OS === 'android') {
        await Notifications.setNotificationChannelAsync('default', {
          name: 'Default',
          importance: Notifications.AndroidImportance.MAX,
          vibrationPattern: [0, 250, 250, 250],
          lightColor: '#6c63ff',
        });
      }
    }

    registerForPushNotifications();

    // Listener for notifications received while the app is in the foreground
    notificationListener.current = Notifications.addNotificationReceivedListener(
      (notification) => {
        setNotification(notification);
        console.log('Notification received:', notification.request.content.title);
      }
    );

    // Listener for when the user taps a notification
    responseListener.current = Notifications.addNotificationResponseReceivedListener(
      (response) => {
        const data = response.notification.request.content.data;
        console.log('Notification tapped, data:', data);
        // Add navigation logic here based on data.screen
      }
    );

    return () => {
      notificationListener.current?.remove();
      responseListener.current?.remove();
    };
  }, []);

  return { expoPushToken, notification };
}
```

**Expected behavior:**
- A permission dialog appears when the app first launches
- After granting permission, an `ExponentPushToken[...]` is logged to the console
- Notifications received while the app is open trigger a banner and sound

## Step 2: Save the Token to Your Backend

The token must be stored on your backend so you can send notifications later. Here's how to do it with Supabase:

```typescript
// lib/savePushToken.ts
// Saves the retrieved token to the user_push_tokens table in Supabase

import { supabase } from './supabase';

export async function savePushTokenToSupabase(
  userId: string,
  token: string
): Promise<void> {
  const { error } = await supabase
    .from('user_push_tokens')
    .upsert(
      { user_id: userId, expo_push_token: token, updated_at: new Date().toISOString() },
      { onConflict: 'user_id' }  // overwrite existing record for the same user
    );

  if (error) {
    console.error('Failed to save token:', error.message);
    throw error;
  }

  console.log('Token saved successfully:', token);
  // Expected output: Token saved successfully: ExponentPushToken[xxx...]
}
```

Set up a `user_push_tokens` table in Supabase with columns: `user_id (text)`, `expo_push_token (text)`, and `updated_at (timestamptz)`. You can prompt Rork with "Create a user_push_tokens table in Supabase and save the user's Expo push token when they log in" to have it generate both the schema and the saving logic automatically.

For more on Supabase integration patterns, see the [Rork × Supabase Chat App Guide](/articles/rork-dev/rork-supabase-chat-app).

## Step 3: Send Notifications from Your Backend

With tokens stored, you're ready to send notifications. The Expo Push API accepts standard HTTP POST requests, making it easy to call from a Supabase Edge Function, Node.js server, or any HTTP-capable environment:

```typescript
// functions/send-push-notification/index.ts
// Supabase Edge Function that sends a push notification to a specific user

import { serve } from 'https://deno.land/std@0.168.0/http/server.ts';

interface PushNotificationPayload {
  to: string | string[];           // one or more Expo Push Tokens
  title: string;                   // notification title
  body: string;                    // notification body text
  data?: Record<string, unknown>;  // custom data passed to the app
  sound?: 'default' | null;        // notification sound
  badge?: number;                  // badge count update
}

serve(async (req) => {
  const { userId, title, body, data } = await req.json();

  // Fetch the token(s) for this user from Supabase
  const supabaseUrl = Deno.env.get('SUPABASE_URL')!;
  const supabaseKey = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!;

  const response = await fetch(
    `${supabaseUrl}/rest/v1/user_push_tokens?user_id=eq.${userId}`,
    { headers: { apikey: supabaseKey, Authorization: `Bearer ${supabaseKey}` } }
  );
  const tokens = await response.json();

  if (!tokens.length) {
    return new Response(JSON.stringify({ error: 'No token found for this user.' }), { status: 404 });
  }

  // Send to Expo Push API
  const payload: PushNotificationPayload = {
    to: tokens.map((t: { expo_push_token: string }) => t.expo_push_token),
    title,
    body,
    data: data ?? {},
    sound: 'default',
  };

  const pushResponse = await fetch('https://exp.host/--/api/v2/push/send', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Accept: 'application/json',
    },
    body: JSON.stringify(payload),
  });

  const result = await pushResponse.json();
  console.log('Push result:', JSON.stringify(result));
  // Expected output: { "data": [{ "status": "ok", "id": "xxxxx" }] }

  return new Response(JSON.stringify(result), {
    headers: { 'Content-Type': 'application/json' },
  });
});
```

Once deployed, this Edge Function can be triggered by user actions, scheduled jobs, or webhooks to send targeted notifications at exactly the right moment.

## Step 4: Deep Link to a Screen on Notification Tap

When a user taps a notification, you can navigate them directly to a relevant screen using the `data` field:

```typescript
// app/_layout.tsx (excerpt)
// Automatically navigate to data.screen when a notification is tapped

import { useEffect } from 'react';
import { useRouter } from 'expo-router';
import * as Notifications from 'expo-notifications';

export default function RootLayout() {
  const router = useRouter();

  useEffect(() => {
    const subscription = Notifications.addNotificationResponseReceivedListener(
      (response) => {
        const screen = response.notification.request.content.data?.screen as string;
        if (screen) {
          router.push(screen as never);
          // Example: { screen: '/messages/123' } opens that chat thread directly
        }
      }
    );
    return () => subscription.remove();
  }, [router]);

  // ... rest of layout
}
```

Set `data: { screen: '/messages/123' }` when sending the notification, and tapping it will route the user straight to that conversation. For tips on debugging navigation issues, check the [Fix Now Auto-Debug Guide](/articles/rork-basics/fix-now-auto-debug).

## FAQ

**Q. I can't get a push token in the simulator. What's wrong?**

A. Expo Push Token retrieval requires a **physical device**. On simulators and emulators, `Device.isDevice` returns `false`, so token registration is skipped by design. Test on a real device running Expo Go or an EAS Development Build.

**Q. Push notifications aren't being delivered on iOS.**

A. Check the following in order: (1) the `ios.bundleIdentifier` in `app.json` matches the App ID registered in your Apple Developer account; (2) you're using an EAS Build—not Expo Go—for production push delivery; (3) your EAS project is linked to the correct Apple team. Expo Go uses a shared certificate and does not support custom APNs delivery for your app's bundle ID.

**Q. How do I send a broadcast notification to all users?**

A. The Expo Push API accepts up to **100 tokens per request** as an array. For large-scale broadcasts, split your token list into chunks of 100 and send batched requests. A Supabase pg_cron job or a queue-based worker is a clean way to handle this at scale. For higher engagement, consider segmenting by user attributes and personalizing the message body.

**Q. How do I handle expired or invalid tokens?**

A. When a user uninstalls and reinstalls your app, or revokes notification permissions, their old token becomes invalid. The Expo Push API returns a `DeviceNotRegistered` error for stale tokens. Add a cleanup step in your backend that deletes any token returning this error, preventing unnecessary API calls and keeping your token table clean.

## Summary

Implementing push notifications in a Rork app comes down to four steps:

1. Use `expo-notifications` to request permission and retrieve the device token
2. Save the token to your backend (e.g., Supabase)
3. Call the Expo Push API from a server to send notifications
4. Use `data.screen` to deep-link users to the right screen on tap

You can prompt Rork with "Add push notifications, save the token to Supabase, and send from a backend Edge Function" to generate the majority of this code automatically. From there, it's just a matter of plugging in your project IDs, table names, and screen paths.

Push notifications, done right, are one of the highest-ROI features you can add to any mobile app. Build them in, and watch your retention numbers climb.
