## Understanding the HomeKit Architecture

HomeKit organizes devices in a clear hierarchy that mirrors the physical layout of a home.

**HMHome** sits at the top and represents a physical residence. A single user can have multiple homes — a primary residence, a vacation house, and so on. Each home contains one or more **HMRoom** objects (living room, bedroom, kitchen).

Rooms hold **HMAccessory** objects, which represent individual smart devices like bulbs and thermostats. Each accessory exposes one or more **HMService** instances — a single device might offer both power-toggle and brightness-adjustment services.

Finally, services contain **HMCharacteristic** objects. These are the actual read/write endpoints for values like brightness percentage or target temperature.

## Building the Device List

The heart of any smart home app is a clear, organized view of every connected device.

### Initializing HMHomeManager

`HMHomeManager` is your gateway to the user's HomeKit data. It asynchronously loads all configured homes and notifies your app through a delegate.

```swift
import HomeKit

class HomeKitManager: NSObject, ObservableObject {
    private let homeManager = HMHomeManager()
    @Published var homes: [HMHome] = []
    @Published var primaryHome: HMHome?

    override init() {
        super.init()
        homeManager.delegate = self
    }
}

extension HomeKitManager: HMHomeManagerDelegate {
    func homeManagerDidUpdateHomes(_ manager: HMHomeManager) {
        DispatchQueue.main.async {
            self.homes = manager.homes
            self.primaryHome = manager.primaryHome
        }
    }
}
```

Tell Rork Max to "use HMHomeManager with a delegate pattern and expose homes as @Published properties" and it will produce a clean, idiomatic implementation.

## Controlling Lights

Lighting is the most commonly used smart home feature, and HomeKit makes it refreshingly simple to implement.

### Power Toggle

Toggling a light on or off involves writing to the `HMCharacteristicTypePowerState` characteristic:

```swift
func toggleLight(_ accessory: HMAccessory) {
    guard let lightService = accessory.services.first(where: {
        $0.serviceType == HMServiceTypeLightbulb
    }) else { return }

    guard let powerChar = lightService.characteristics.first(where: {
        $0.characteristicType == HMCharacteristicTypePowerState
    }) else { return }

    let newValue = !(powerChar.value as? Bool ?? false)
    powerChar.writeValue(newValue) { error in
        if let error = error {
            print("Failed to toggle light: \(error.localizedDescription)")
        }
    }
}
```

### Brightness Slider

Brightness is controlled through `HMCharacteristicTypeBrightness`, which accepts an integer between 0 and 100. Pairing it with a SwiftUI `Slider` creates an intuitive interface:

```swift
struct BrightnessSlider: View {
    let characteristic: HMCharacteristic
    @State private var brightness: Double = 50

    var body: some View {
        VStack {
            Text("Brightness: \(Int(brightness))%")
            Slider(value: $brightness, in: 0...100, step: 1) { editing in
                if !editing {
                    characteristic.writeValue(Int(brightness)) { error in
                        if let error = error {
                            print("Brightness error: \(error.localizedDescription)")
                        }
                    }
                }
            }
        }
        .onAppear {
            brightness = Double(characteristic.value as? Int ?? 50)
        }
    }
}
```

Prompt Rork Max with "add a brightness slider that only sends the value when the user lifts their finger" to get this behavior out of the box.

## Thermostat Control

Temperature management is another key pillar of smart home apps.

### Reading and Setting Temperature

A thermostat accessory exposes `HMServiceTypeThermostat` with characteristics for current temperature and target temperature.

```swift
struct ThermostatView: View {
    let accessory: HMAccessory
    @State private var targetTemp: Double = 22.0
    @State private var currentTemp: Double = 0.0

    var thermostatService: HMService? {
        accessory.services.first { $0.serviceType == HMServiceTypeThermostat }
    }

    var body: some View {
        VStack(spacing: 20) {
            Text("Current Temperature")
                .font(.subheadline)
                .foregroundColor(.secondary)
            Text("\(currentTemp, specifier: "%.1f")°C")
                .font(.system(size: 48, weight: .thin))

            Divider()

            Text("Target Temperature")
                .font(.subheadline)
            HStack {
                Button(action: { adjustTarget(-0.5) }) {
                    Image(systemName: "minus.circle.fill")
                        .font(.title)
                }
                Text("\(targetTemp, specifier: "%.1f")°C")
                    .font(.system(size: 36, weight: .medium))
                    .frame(width: 120)
                Button(action: { adjustTarget(0.5) }) {
                    Image(systemName: "plus.circle.fill")
                        .font(.title)
                }
            }
        }
        .onAppear { loadTemperatures() }
    }

    private func adjustTarget(_ delta: Double) {
        targetTemp += delta
        targetTemp = max(16, min(30, targetTemp))
        writeTargetTemperature()
    }
}
```

The temperature range is typically clamped to 16–30°C in 0.5° increments, providing a safe and practical adjustment range for most climates.

## Real-Time Notifications and Scenes

HomeKit supports push-style notifications when device states change, keeping your UI always in sync.

### Monitoring Characteristic Values

Enable notifications on any characteristic to receive callbacks whenever its value changes:

```swift
func startMonitoring(characteristic: HMCharacteristic) {
    characteristic.enableNotification(true) { error in
        if let error = error {
            print("Notification registration error: \(error.localizedDescription)")
        }
    }
}
```

Handle updates in the `accessory(_:service:didUpdateValueFor:)` delegate method to refresh your UI instantly — even when changes originate from another app or a physical button press.

### Scenes and Automation

HomeKit **HMActionSet (scenes)** let you trigger multiple device actions at once. A "Good Night" scene might turn off all lights and set the thermostat to 26°C:

```swift
func executeScene(_ scene: HMActionSet) {
    homeKit.primaryHome?.executeActionSet(scene) { error in
        if let error = error {
            print("Scene execution error: \(error.localizedDescription)")
        }
    }
}
```

Rork Max can generate a scene list with one-tap execution buttons, making it easy for users to activate their favorite presets.

## Testing and Debugging Tips

A few practical considerations will save you hours of debugging.

### HomeKit Accessory Simulator

Apple provides the **HomeKit Accessory Simulator** as part of Xcode's additional tools. It lets you create virtual lights, thermostats, and locks without needing any physical hardware. This is invaluable for testing your Rork Max-generated app in the iOS Simulator.

### Common Pitfalls

If you see a "HomeKit is not available" error, double-check that the HomeKit capability is enabled in your Apple Developer Portal for the relevant App ID.

When a user hasn't created any homes yet, the `homes` array will be empty. A thoughtful onboarding screen that guides users to create their first home in the Apple Home app goes a long way toward a smooth first-run experience.

## Preparing for the App Store

There are a few additional considerations when publishing a HomeKit app.

Apple's review guidelines require HomeKit apps to offer genuine smart-home functionality — demo-only apps risk rejection. Make sure your app provides real utility for managing connected devices.

Your privacy policy should clearly describe what HomeKit data your app accesses (device names, room layouts, usage patterns) and how it's handled.
