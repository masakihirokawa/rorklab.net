One of the most powerful—and most complex—features in Rork Max is the ability to run code in the background. When implemented correctly, background processing enables your app to sync data, download resources, and notify users even when they're not actively using your app.

The challenge? iOS's background execution model is tightly constrained by Apple's security and battery-life requirements. Misstep here and your app either crashes, gets rejected during review, or violates Apple's guidelines. This guide walks you through the four main background processing mechanisms and shows you how Rork Max-generated code fits into a production-ready pattern.

## Four Ways iOS Runs Code in the Background

Each mechanism serves a different use case. Understanding the distinctions will save you hours of debugging.

### 1. BGTaskScheduler — Scheduled Processing Tasks

**BGTaskScheduler** (iOS 13+) is the modern, recommended way to run tasks periodically in the background. It's ideal for database cleanup, data synchronization, or report generation—work that needs to happen regularly but not at a precise moment.

```swift
import BackgroundTasks

// Step 1: Register the task handler (call once in AppDelegate or SceneDelegate)
func registerBackgroundTasks() {
    let success = BGTaskScheduler.shared.register(
        forTaskWithIdentifier: "com.example.app.cleanup",
        using: nil
    ) { task in
        self.handleCleanupTask(task: task as! BGProcessingTask)
    }

    guard success else {
        print("Failed to register background task")
        return
    }
}

// Step 2: Implement the task handler
func handleCleanupTask(task: BGProcessingTask) {
    // If the system decides your task has run long enough,
    // this block fires—you must clean up gracefully
    task.expirationHandler = {
        print("Task expired, cleaning up...")
    }

    DispatchQueue.global().async {
        // Perform database optimization or similar work
        self.optimizeDatabase()

        // CRITICAL: Always call this when done
        task.setTaskCompleted(success: true)
    }
}

// Step 3: Schedule the task
func scheduleCleanupTask() {
    let request = BGProcessingTaskRequest(
        identifier: "com.example.app.cleanup"
    )
    request.requiresNetworkConnectivity = false
    request.requiresExternalPower = false

    do {
        try BGTaskScheduler.shared.submit(request)
        print("Cleanup task scheduled")
    } catch {
        print("Failed to schedule cleanup: \(error)")
    }
}
```

**Key Constraints:**
- The OS decides *when* your task runs (roughly "every few hours," not precise)
- Tasks must complete within 30 seconds
- Tasks run preferentially when the device is plugged in
- You must declare task identifiers in `Info.plist` under `BGTaskSchedulerPermittedIdentifiers`

### 2. Background App Refresh — Low-Frequency Periodic Updates

**Background App Refresh** runs even less frequently than BGTaskScheduler (once or twice daily) and is useful when you want to refresh major app content—like a news app fetching the latest articles.

```swift
import BackgroundTasks

func scheduleBackgroundRefresh() {
    let request = BGAppRefreshTaskRequest(identifier: "com.example.app.refresh")
    request.earliestBeginDate = Date(timeIntervalSinceNow: 15 * 60)

    do {
        try BGTaskScheduler.shared.submit(request)
    } catch {
        print("Failed to schedule background refresh: \(error)")
    }
}

// Handle the refresh task
func handleBackgroundRefresh(task: BGAppRefreshTask) {
    task.expirationHandler = {
        task.setTaskCompleted(success: false)
    }

    // Async work (API calls, data fetching)
    fetchLatestData { success in
        task.setTaskCompleted(success: success)
    }
}
```

### 3. Background Download/Upload — Large File Transfers

**URLSessionConfiguration.background** lets you download or upload large files even if your app terminates. The OS manages the transfer on your behalf.

```swift
import Foundation

func startBackgroundDownload(url: URL) {
    var config = URLSessionConfiguration.background(
        withIdentifier: "com.example.app.download"
    )
    config.isDiscretionary = true  // Defer if battery-saver mode is on
    config.sessionSendsLaunchEvents = true  // Relaunch app when done

    let session = URLSession(
        configuration: config,
        delegate: DownloadDelegate(),
        delegateQueue: nil
    )

    let downloadTask = session.downloadTask(with: url)
    downloadTask.resume()
}

// URLSessionDownloadDelegate callback
class DownloadDelegate: NSObject, URLSessionDownloadDelegate {
    func urlSession(
        _ session: URLSession,
        downloadTask: URLSessionDownloadTask,
        didFinishDownloadingTo location: URL
    ) {
        let fileManager = FileManager.default
        let documentsURL = fileManager.urls(
            for: .documentDirectory,
            in: .userDomainMask
        )[0]

        let destinationURL = documentsURL.appendingPathComponent("downloaded.zip")

        do {
            try fileManager.moveItem(at: location, to: destinationURL)
            print("Download complete: \(destinationURL)")
        } catch {
            print("Move error: \(error)")
        }
    }

    func urlSession(
        _ session: URLSession,
        downloadTask: URLSessionDownloadTask,
        didFailWithError error: Error
    ) {
        print("Download failed: \(error)")
    }
}
```

### 4. Local Notifications — Notifying Users from Background Work

Once your background task completes, use **UNUserNotificationCenter** to notify the user.

```swift
import UserNotifications

func scheduleLocalNotification(
    title: String,
    body: String,
    delaySeconds: TimeInterval = 5
) {
    let content = UNMutableNotificationContent()
    content.title = title
    content.body = body
    content.sound = .default
    content.badge = NSNumber(
        value: UIApplication.shared.applicationIconBadgeNumber + 1
    )

    // Custom data for deep-linking when user taps notification
    content.userInfo = ["articleID": "12345"]

    let trigger = UNTimeIntervalNotificationTrigger(
        timeInterval: delaySeconds,
        repeats: false
    )

    let request = UNNotificationRequest(
        identifier: UUID().uuidString,
        content: content,
        trigger: trigger
    )

    UNUserNotificationCenter.current().add(request) { error in
        if let error = error {
            print("Notification error: \(error)")
        }
    }
}

// Handle user tapping the notification
func userNotificationCenter(
    _ center: UNUserNotificationCenter,
    didReceive response: UNNotificationResponse,
    withCompletionHandler completionHandler: @escaping () -> Void
) {
    let userInfo = response.notification.request.content.userInfo

    if let articleID = userInfo["articleID"] as? String {
        // Navigate to the article in your app
        navigateToArticle(id: articleID)
    }

    completionHandler()
}
```

## Three Real-World Patterns for Rork Max

### Pattern 1: Daily Data Synchronization

Sync your local database with the server once per day.

```swift
class SyncService {
    static func setupDailySync() {
        let identifier = "com.rorkapp.daily-sync"

        BGTaskScheduler.shared.register(
            forTaskWithIdentifier: identifier,
            using: .main
        ) { task in
            Self.performSync(task: task as! BGProcessingTask)
        }
    }

    static func scheduleNextSync() {
        let request = BGProcessingTaskRequest(
            identifier: "com.rorkapp.daily-sync"
        )
        request.requiresNetworkConnectivity = true

        do {
            try BGTaskScheduler.shared.submit(request)
        } catch {
            print("Sync scheduling failed: \(error)")
        }
    }

    static func performSync(task: BGProcessingTask) {
        task.expirationHandler = {
            task.setTaskCompleted(success: false)
        }

        // API call
        URLSession.shared.dataTask(
            with: URL(string: "https://api.example.com/sync")!
        ) { data, response, error in

            if error == nil, let data = data {
                self.saveToDatabase(data)
                task.setTaskCompleted(success: true)
            } else {
                task.setTaskCompleted(success: false)
            }

            // Schedule next sync
            self.scheduleNextSync()
        }.resume()
    }

    static func saveToDatabase(_ data: Data) {
        // Use Realm, CoreData, or your Rork Max backend
    }
}
```

### Pattern 2: Background File Downloads

Download maps, resources, or other large files while the app is closed.

```swift
class BackgroundDownloadManager: NSObject, URLSessionDownloadDelegate {
    static let shared = BackgroundDownloadManager()

    var session: URLSession!

    override init() {
        super.init()

        let config = URLSessionConfiguration.background(
            withIdentifier: "com.rorkapp.bgdownload"
        )
        config.isDiscretionary = true
        self.session = URLSession(
            configuration: config,
            delegate: self,
            delegateQueue: nil
        )
    }

    func downloadLargeFile(url: URL) {
        let task = session.downloadTask(with: url)
        task.resume()
    }

    func urlSession(
        _ session: URLSession,
        downloadTask: URLSessionDownloadTask,
        didFinishDownloadingTo location: URL
    ) {
        let fileManager = FileManager.default
        let docsURL = fileManager.urls(for: .documentDirectory, in: .userDomainMask)[0]
        let fileName = downloadTask.originalRequest?.url?.lastPathComponent ?? "download"
        let destination = docsURL.appendingPathComponent(fileName)

        do {
            try fileManager.moveItem(at: location, to: destination)

            // Notify the user
            notifyDownloadComplete(fileName: fileName)
        } catch {
            print("Error: \(error)")
        }
    }

    func notifyDownloadComplete(fileName: String) {
        let content = UNMutableNotificationContent()
        content.title = "Download Complete"
        content.body = "\(fileName) is ready"
        content.sound = .default

        let trigger = UNTimeIntervalNotificationTrigger(timeInterval: 1, repeats: false)
        let request = UNNotificationRequest(
            identifier: UUID().uuidString,
            content: content,
            trigger: trigger
        )

        UNUserNotificationCenter.current().add(request)
    }
}
```

### Pattern 3: Local Data Processing (No Network)

Compress files, clean up old caches, or process images entirely offline.

```swift
class DataProcessingService {
    static func scheduleOfflineProcessing() {
        let identifier = "com.rorkapp.offline-process"

        BGTaskScheduler.shared.register(
            forTaskWithIdentifier: identifier,
            using: nil
        ) { task in
            self.processLocalData(task: task as! BGProcessingTask)
        }
    }

    static func requestProcessing() {
        let request = BGProcessingTaskRequest(
            identifier: "com.rorkapp.offline-process"
        )
        request.requiresNetworkConnectivity = false
        request.requiresExternalPower = false

        do {
            try BGTaskScheduler.shared.submit(request)
        } catch {
            print("Processing request failed: \(error)")
        }
    }

    static func processLocalData(task: BGProcessingTask) {
        task.expirationHandler = {
            task.setTaskCompleted(success: false)
        }

        DispatchQueue.global(qos: .background).async {
            // Example: Delete cache files older than 30 days
            let docsURL = FileManager.default.urls(
                for: .documentDirectory,
                in: .userDomainMask
            )[0]

            let cacheURL = docsURL.appendingPathComponent("Cache")

            do {
                let files = try FileManager.default.contentsOfDirectory(
                    at: cacheURL,
                    includingPropertiesForKeys: [.contentModificationDateKey]
                )

                let thirtyDaysAgo = Date(timeIntervalSinceNow: -30 * 24 * 3600)

                for file in files {
                    let attrs = try FileManager.default.attributesOfItem(atPath: file.path)
                    if let modDate = attrs[.modificationDate] as? Date,
                       modDate < thirtyDaysAgo {
                        try FileManager.default.removeItem(at: file)
                    }
                }

                task.setTaskCompleted(success: true)
            } catch {
                task.setTaskCompleted(success: false)
            }
        }
    }
}
```

## Essential Info.plist Configuration

To enable background processing, add the following to your project's `Info.plist`:

```xml
<key>BGTaskSchedulerPermittedIdentifiers</key>
<array>
    <string>com.rorkapp.daily-sync</string>
    <string>com.rorkapp.offline-process</string>
</array>
```

Also in Xcode: **Signing & Capabilities** → **+ Capability** → **Background Modes**. Enable:
- ✅ Background App Refresh
- ✅ Background Processing

## Common Questions (FAQ)

**Q: Can I predict *when* a BGTaskScheduler task will run?**

No. The OS decides based on device state (charging?), network connectivity, battery mode, and user patterns. Tasks might run within minutes or hours. Never rely on precise timing.

If you need exact timing, either run the task in the foreground or use a backend scheduler with push notifications.

**Q: What if the network drops during a background task?**

URLSession automatically retries. If you've set `requiresNetworkConnectivity = true`, the task is deferred until connectivity returns. If it drops mid-task, your `expirationHandler` fires and you must clean up gracefully.

**Q: Can I make multiple API calls in a background task?**

Yes, but keep it to 2–3 lightweight calls *max* (30-second limit). For complex operations, aggregate on your backend and have it return a single response.

**Q: Can I send a local notification from a background task?**

Absolutely. You have 30 seconds to call `UNUserNotificationCenter.current().add(request)` and it will be delivered when the task completes.

**Q: How do I test background tasks in the Xcode simulator?**

In **Scheme** > **Run** > **Pre-actions**, add a build phase that triggers your task. For real devices, use the simulator's debug menu or manually submit the task.

## Best Practices

1. **Register tasks once at app launch** — in `application(_:didFinishLaunchingWithOptions:)`
2. **Always implement `expirationHandler`** — graceful shutdown is critical
3. **Respect the 30-second limit** — design work to fit
4. **Declare network requirements explicitly** — set `requiresNetworkConnectivity` appropriately
5. **Pair with local notifications** — let users know when work completes
6. **Monitor with Instruments** — use System Trace to see actual execution times
7. **Test on a real device** — simulator behavior differs from production
