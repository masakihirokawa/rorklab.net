## Section 1: App Store Preview Video Specifications and Impact

### 1.1 Official App Store Requirements

```
【Apple Official Requirements】

【Video Dimensions】
- iPhone (Portrait): 9:16 aspect ratio
  * Minimum: 1080 × 1920 pixels
  * Recommended: 1440 × 2560 pixels
- iPad (Landscape): 16:9 aspect ratio
  * 1440 × 810 pixels

【Duration】
- 15-30 seconds: Optimal for ASO ranking boost
- Maximum: 30 seconds
- Upload up to 3 preview videos (Preview 1, 2, 3)

【Format】
- File format: MP4 (H.264 codec)
- Frame rate: 25 fps or 30 fps
- File size: Maximum 100 MB

【Audio】
- Audio supported (stereo recommended)
- Silent videos acceptable
- Music: Must be royalty-free or properly licensed

【Text & Graphics】
- Text: Minimum 16pt (larger for readability)
- Logos and characters: Copyrighted-cleared assets only
- Screenshots: Current version app UI
```

## Section 5: Integration with Marketing and Analytics

### 5.1 Tracking Preview Video Performance

```csharp
// Firebase Analytics integration for video tracking

public class AppPreviewAnalytics : MonoBehaviour
{
    // Log when app preview appears in App Store
    public void LogPreviewVideoImpressions(string videoId, string previewNumber)
    {
        FirebaseAnalytics.LogEvent(
            "app_preview_viewed",
            new Parameter("video_id", videoId),
            new Parameter("preview_number", previewNumber)
        );
    }

    // Track downloads that came from preview video views
    public void LogConversionFromPreview(string videoId, string downloadSource)
    {
        FirebaseAnalytics.LogEvent(
            "converted_from_preview",
            new Parameter("video_id", videoId),
            new Parameter("source", downloadSource)
        );
    }

    // Monitor video skip-through rates (critical for ASO)
    public void LogPreviewSkipRate(string videoId, float skipPercentage)
    {
        FirebaseAnalytics.LogEvent(
            "preview_skip_rate",
            new Parameter("video_id", videoId),
            new Parameter("skip_percentage", skipPercentage.ToString())
        );
    }

    // A/B test different preview videos
    public void LogABTestVariant(string testId, string variant)
    {
        FirebaseAnalytics.SetUserProperty("preview_variant", variant);
        FirebaseAnalytics.LogEvent(
            "ab_test_preview",
            new Parameter("test_id", testId),
            new Parameter("variant", variant)
        );
    }
}
```

## Section 7: Troubleshooting and Best Practices

### 7.1 Common Issues

```
Q1: Pollo AI export is too large (>100MB)
A: Lower video resolution in export settings.
   Use Preset: "App Store Optimized"
   Frame rate: 25 fps (instead of 30)

Q2: Suno AI music doesn't sync with video pacing
A: Adjust video clips to match music tempo.
   Or regenerate Suno with different BPM.
   Use Suno prompt: "Exactly 20 seconds"

Q3: App Store upload fails with "invalid format"
A: Verify MP4 codec is H.264 (not H.265)
   Check file size <100MB
   Use macOS QuickTime to export—it ensures compatibility

Q4: Video has poor skip-through rate (<50%)
A: First 3 seconds are critical—show immediate value
   Add text overlay in first second
   Increase visual contrast and color saturation
   Consider shorter, punchier transitions
```

### 7.2 Best Practices

```
【Production Best Practices】

1. Always show actual app UI in first 3 seconds
   └─ Don't waste time on intro animations

2. Use high-contrast text for readability
   └─ White on dark background preferred
   └─ Minimum 16pt, ideally 24pt+

3. Match Suno AI tempo to app interactions
   └─ Fast cuts = faster tempo BGM
   └─ Slow reveals = slower tempo BGM

4. Include a clear CTA in final 5 seconds
   └─ "Download Now" button or text
   └─ App Store rating display

5. A/B test ruthlessly
   └─ Different opening scenes
   └─ Various music styles
   └─ Multiple feature sequences

6. Update videos monthly
   └─ Refresh UI screenshots when app updates
   └─ Adapt to seasonal trends
   └─ Respond to competitor videos
```
