## Why Use RevenueCat with Rork

Building a great app with Rork is only half the journey — generating sustainable revenue is what turns a side project into a real business. In-app purchases and subscriptions are the most powerful monetization tools available to mobile developers today, and **RevenueCat** is the industry-standard SDK for handling them.

The App Store and Google Play each have their own In-App Purchase (IAP) APIs, and managing both independently is notoriously complex. RevenueCat solves this by providing a unified layer over both platforms, making it an ideal companion for Rork's prompt-driven development workflow.

This guide walks you through integrating RevenueCat into a Rork app from scratch, complete with working code examples and real-world best practices.

---

## What Is RevenueCat?

RevenueCat is a subscription management platform for mobile apps that abstracts away the complexity of native store billing APIs. Since its founding in 2018, it has become the go-to solution for indie developers and startups — tracking billions in annual revenue across its platform.

### Key Benefits

- **Cross-platform unified API**: Manage App Store and Google Play subscriptions with a single SDK
- **Automatic receipt validation**: Server-side fraud protection handled for you
- **Real-time analytics dashboard**: Track MRR, churn rate, LTV, and more
- **Webhooks and integrations**: Connect to Slack, Amplitude, Mixpanel, and other tools
- **Free tier**: No cost until you exceed $2,500/month in tracked revenue

Unlike Stripe — which handles web-based payments — RevenueCat works through Apple and Google's native billing systems, ensuring full compliance with platform guidelines out of the box.

## Prerequisites

Before diving in, make sure you have the following ready:

1. A **Rork account** (Junior plan or above recommended)
2. A **RevenueCat account** (free to create at revenuecat.com)
3. **Apple Developer Program** membership ($99/year)
4. **Google Play Developer** account ($25 one-time fee)
5. In-App Purchase products configured in App Store Connect and Google Play Console

### Setting Up Products in App Store Connect

1. Go to App Store Connect → "My Apps" → select your app
2. Click "In-App Purchases" → tap "+"
3. Select "Auto-Renewable Subscription"
4. Enter a product ID (e.g., `com.yourapp.premium_monthly`) and display name
5. Set your pricing tier (e.g., $4.99/month)

Repeat the same process in Google Play Console under "Monetize → Subscriptions."

## RevenueCat Setup

### Step 1: Create a RevenueCat Project

Sign up at [RevenueCat](https://www.revenuecat.com/) and create a new project.

1. Dashboard → "+ New Project"
2. Name your project (e.g., `MyApp`)
3. Add your iOS and Android apps
4. Copy your **Public SDK Key** — you'll need it shortly

### Step 2: Configure Entitlements and Offerings

RevenueCat's **Entitlements** define what content or features a paying user can access.

- **Entitlement** example: `premium` (access to all premium features)
- **Product**: attach your App Store / Google Play product IDs here
- **Offering**: the set of plans you present to the user at the paywall

Create a `premium` Entitlement in the dashboard and attach both your iOS and Android product IDs to it.

### Step 4: Initialize the RevenueCat SDK

Verify that the generated code includes proper SDK initialization:

```typescript
// app/_layout.tsx (Expo Router)
import Purchases, { LOG_LEVEL } from 'react-native-purchases';
import { Platform } from 'react-native';

const REVENUECAT_IOS_KEY = 'appl_xxxxxxxxxxxxxxxxxxxxxxxx';
const REVENUECAT_ANDROID_KEY = 'goog_xxxxxxxxxxxxxxxxxxxxxxxx';

export default function RootLayout() {
  useEffect(() => {
    // Initialize RevenueCat once at app launch
    if (Platform.OS === 'ios') {
      Purchases.configure({ apiKey: REVENUECAT_IOS_KEY });
    } else {
      Purchases.configure({ apiKey: REVENUECAT_ANDROID_KEY });
    }

    // Enable verbose logging in development only
    if (__DEV__) {
      Purchases.setLogLevel(LOG_LEVEL.DEBUG);
    }
  }, []);

  return <Stack />;
}
// Expected behavior: RevenueCat initializes on app launch and
// automatically syncs the user's purchase history
```

### Step 5: Manage Subscription State Globally

Create a Context to share subscription status across your entire app:

```typescript
// contexts/PurchaseContext.tsx
import React, { createContext, useContext, useEffect, useState } from 'react';
import Purchases, { CustomerInfo } from 'react-native-purchases';

interface PurchaseContextType {
  isPremium: boolean;
  customerInfo: CustomerInfo | null;
  isLoading: boolean;
  checkPremiumStatus: () => Promise<void>;
}

const PurchaseContext = createContext<PurchaseContextType | null>(null);

export function PurchaseProvider({ children }: { children: React.ReactNode }) {
  const [isPremium, setIsPremium] = useState(false);
  const [customerInfo, setCustomerInfo] = useState<CustomerInfo | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  const checkPremiumStatus = async () => {
    try {
      const info = await Purchases.getCustomerInfo();
      setCustomerInfo(info);
      // "premium" must match the Entitlement ID in your RevenueCat dashboard
      setIsPremium(info.entitlements.active['premium'] !== undefined);
    } catch (error) {
      console.error('RevenueCat error:', error);
    } finally {
      setIsLoading(false);
    }
  };

  useEffect(() => {
    checkPremiumStatus();

    // Listen for real-time subscription changes
    const listener = Purchases.addCustomerInfoUpdateListener((info) => {
      setCustomerInfo(info);
      setIsPremium(info.entitlements.active['premium'] !== undefined);
    });

    return () => listener.remove();
  }, []);

  return (
    <PurchaseContext.Provider
      value={{ isPremium, customerInfo, isLoading, checkPremiumStatus }}
    >
      {children}
    </PurchaseContext.Provider>
  );
}

export const usePurchase = () => {
  const context = useContext(PurchaseContext);
  if (!context) throw new Error('usePurchase must be used within PurchaseProvider');
  return context;
};
// Expected behavior: isPremium is true for paying subscribers,
// false for free users, and updates instantly on purchase or cancellation
```

### Step 6: Build the Paywall Screen

```typescript
// screens/PaywallScreen.tsx
import React, { useEffect, useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet, Alert } from 'react-native';
import Purchases, { Package, PurchasesOffering } from 'react-native-purchases';
import { usePurchase } from '../contexts/PurchaseContext';

export default function PaywallScreen({ onClose }: { onClose: () => void }) {
  const [offering, setOffering] = useState<PurchasesOffering | null>(null);
  const [isPurchasing, setIsPurchasing] = useState(false);
  const { checkPremiumStatus } = usePurchase();

  useEffect(() => {
    fetchOfferings();
  }, []);

  const fetchOfferings = async () => {
    try {
      const offerings = await Purchases.getOfferings();
      if (offerings.current) {
        setOffering(offerings.current);
      }
    } catch (error) {
      console.error('Failed to fetch offerings:', error);
    }
  };

  const handlePurchase = async (pkg: Package) => {
    setIsPurchasing(true);
    try {
      const { customerInfo } = await Purchases.purchasePackage(pkg);
      if (customerInfo.entitlements.active['premium']) {
        await checkPremiumStatus();
        Alert.alert('Welcome to Premium! 🎉', 'Your subscription is now active.');
        onClose();
      }
    } catch (error: any) {
      if (!error.userCancelled) {
        Alert.alert('Purchase Error', error.message);
      }
    } finally {
      setIsPurchasing(false);
    }
  };

  const handleRestore = async () => {
    try {
      const customerInfo = await Purchases.restorePurchases();
      if (customerInfo.entitlements.active['premium']) {
        await checkPremiumStatus();
        Alert.alert('Restored!', 'Your previous purchase has been restored.');
        onClose();
      } else {
        Alert.alert('No Purchases Found', 'We could not find any previous purchases.');
      }
    } catch (error: any) {
      Alert.alert('Restore Error', error.message);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Upgrade to Premium</Text>
      <Text style={styles.subtitle}>Unlock all features with no limits</Text>

      {offering?.availablePackages.map((pkg) => (
        <TouchableOpacity
          key={pkg.identifier}
          style={styles.packageButton}
          onPress={() => handlePurchase(pkg)}
          disabled={isPurchasing}
        >
          <Text style={styles.packageTitle}>{pkg.product.title}</Text>
          <Text style={styles.packagePrice}>{pkg.product.priceString} / month</Text>
        </TouchableOpacity>
      ))}

      <TouchableOpacity style={styles.restoreButton} onPress={handleRestore}>
        <Text style={styles.restoreText}>Restore Purchases</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 24, backgroundColor: '#fff' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 8 },
  subtitle: { fontSize: 16, color: '#666', marginBottom: 32 },
  packageButton: {
    backgroundColor: '#007AFF', borderRadius: 12,
    padding: 16, marginBottom: 12, alignItems: 'center',
  },
  packageTitle: { color: '#fff', fontSize: 18, fontWeight: '600' },
  packagePrice: { color: 'rgba(255,255,255,0.8)', fontSize: 14, marginTop: 4 },
  restoreButton: { marginTop: 16, alignItems: 'center' },
  restoreText: { color: '#007AFF', fontSize: 14 },
});
// Expected behavior: The native App Store / Google Play purchase sheet
// appears, and on success the user's entitlement becomes active
```

## Common Errors and Fixes

### Error: "This in-app purchase has already been bought"

**Cause**: The user is trying to purchase something they already own.
**Fix**: Call `Purchases.restorePurchases()` to sync their existing subscription.

### Error: Sandbox testing fails on device

**Cause**: No Sandbox tester account is configured in App Store Connect.
**Fix**: Go to App Store Connect → "Users and Access" → "Sandbox Testers", create a test account, and sign in on your physical device before testing.

### Error: Google Play product shows "not available"

**Cause**: The product hasn't been activated, or the app hasn't been published to at least the internal testing track.
**Fix**: Set the product status to "Active" in Google Play Console and publish your app to the internal test track before attempting purchases.

### Error: `Purchases.configure()` called multiple times

**Cause**: The initialization runs on every component re-render.
**Fix**: Ensure `useEffect` has an empty dependency array `[]` so it runs only once at app startup.

## Advanced: Webhooks and Backend Sync

RevenueCat's **Webhook** feature lets you receive real-time events — new purchase, renewal, cancellation, trial start — and react to them in your backend.

### Practical Use Cases

- **Slack alerts**: Notify your team channel every time a new subscription starts
- **Analytics pipelines**: Push revenue events to Amplitude or Mixpanel
- **Database sync**: Keep your Supabase or Firebase user records in sync with subscription status

Here's an example Supabase Edge Function that receives RevenueCat webhooks and updates your database:

```typescript
// Supabase Edge Function: handle-revenuecat-webhook
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

serve(async (req) => {
  const payload = await req.json()
  const { event, app_user_id, product_id } = payload

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  if (event === 'INITIAL_PURCHASE' || event === 'RENEWAL') {
    await supabase
      .from('profiles')
      .update({ is_premium: true, subscription_product: product_id })
      .eq('id', app_user_id)
  } else if (event === 'EXPIRATION' || event === 'CANCELLATION') {
    await supabase
      .from('profiles')
      .update({ is_premium: false })
      .eq('id', app_user_id)
  }

  return new Response('ok', { status: 200 })
})
// Expected behavior: Database subscription status updates automatically
// in response to every RevenueCat lifecycle event
```

## FAQ

**Q: Is RevenueCat free to use?**

A: Yes — RevenueCat is free as long as your app's monthly tracked revenue stays under $2,500. Beyond that, a percentage-based fee applies, but the time savings on cross-platform billing management far outweigh the cost.

**Q: Does RevenueCat work with Rork Max native Swift apps?**

A: Absolutely. RevenueCat provides a native Swift SDK for iOS. When building with Rork Max, prompt Claude Opus 4.6 to integrate the RevenueCat iOS SDK and it will generate the appropriate SwiftUI code, including `StoreKit` configuration and paywall screens.

**Q: Can I manage iOS and Android subscriptions in one place?**

A: That's exactly what RevenueCat was built for. One project, one dashboard — you can see all subscribers, revenue metrics, and entitlement status across both platforms in a unified view.

**Q: How do I implement a free trial period?**

A: Configure a free trial (e.g., 7 days) directly in App Store Connect or Google Play Console on the subscription product. RevenueCat automatically handles the trial state — no extra code needed. You can detect trial status via `customerInfo.entitlements.active['premium']?.periodType === 'trial'`.

**Q: What happens if a user reinstalls the app?**

A: Call `Purchases.restorePurchases()` to restore their previous purchases tied to their App Store or Google Play account. Always include a "Restore Purchases" button in your paywall — it's required by the App Store Review Guidelines.
