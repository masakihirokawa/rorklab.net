## Why App Intents — and Why Rork Max?

"Hey Siri, add a task to my app." "Siri, what's on my list today?" Voice control through Siri is one of the most underused differentiators in the App Store. Users who can interact with your app without picking up their phone are measurably more engaged — yet most indie apps skip the integration because the implementation overhead has traditionally been high.

That overhead disappeared with **App Intents** (iOS 16+), Apple's modern Swift-native framework that connects your app to Siri, the Shortcuts app, Spotlight, Focus Filters, and home screen widget buttons. The catch: App Intents require native Swift. React Native wrappers exist, but they're brittle and lag behind the framework.

**Rork Max** generates native Swift, which means App Intents are first-class. Describe the voice commands you want, and Rork Max scaffolds the entire conformance — `AppIntent` structs, `AppShortcutsProvider`, parameter definitions, localized dialog strings, and error handling — in one focused prompt session.

This guide walks through a complete implementation using a task management app as the example. The same patterns apply to fitness apps, e-commerce, timers, or any app that benefits from voice-driven actions.

## App Intents Architecture

### The Building Blocks

Every App Intent is a Swift struct conforming to the `AppIntent` protocol with three parts:

| Component | Purpose |
|-----------|---------|
| `title` | Displayed in Siri, Shortcuts app, and Spotlight |
| `@Parameter` | Inputs Siri collects from the user via voice |
| `perform()` | Async function that runs your app logic and returns a Siri dialog |

When a user says "Hey Siri, add a task to [Your App]," the system maps the phrase to your `AppShortcutsProvider`, determines the matching intent, collects any missing parameters through follow-up questions, then calls `perform()`.

### What App Intents Connect To

Unlike the older SiriKit framework — which was limited to a fixed set of domains like messaging and payments — App Intents work for any app feature you choose to expose:

- **Siri** — voice-triggered on device or via AirPods
- **Shortcuts app** — users build personal automations around your intents
- **Spotlight** — tap an intent directly from iOS search results
- **Widget buttons** (iOS 17+) — home screen buttons that run an intent without opening the app
- **Apple Watch** — intents work across the Apple ecosystem automatically

## Building App Intents with Rork Max

### Step 1: Prompt for the Full Intent Set

Create a new Rork Max project and provide a detailed opening prompt:

```text
Build a task management iOS app with App Intents for Siri and the Shortcuts app.

Implement three intents:
1. AddTaskIntent — add a task with a title and optional due date
2. ListTasksIntent — have Siri read out today's tasks
3. CompleteTaskIntent — mark a task complete by title

Register all three in an AppShortcutsProvider so users can trigger them by voice.
```

Rork Max will generate:

- `TaskAppIntents.swift` — all three `AppIntent` structs
- `TaskShortcuts.swift` — `AppShortcutsProvider` with Siri phrases
- Updated `TaskRepository` — the data layer wired to the intents

### Step 2: Review the Generated Intent Code

The core of what Rork Max produces:

```swift
import AppIntents
import Foundation

// ─── Add Task Intent ───
struct AddTaskIntent: AppIntent {
    static var title: LocalizedStringResource = "Add Task"
    static var description = IntentDescription("Add a new task to your list.")

    // Siri will ask "What's the task?" if the user doesn't provide it upfront
    @Parameter(title: "Task Title")
    var taskTitle: String

    @Parameter(title: "Due Date", default: .today)
    var dueDate: Date

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        await TaskRepository.shared.add(title: taskTitle, dueDate: dueDate)
        return .result(dialog: "Added "\(taskTitle)" to your tasks.")
    }
}

// ─── List Tasks Intent ───
struct ListTasksIntent: AppIntent {
    static var title: LocalizedStringResource = "List Today's Tasks"
    static var description = IntentDescription("Read out the tasks due today.")

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        let tasks = await TaskRepository.shared.tasksDueToday()

        guard !tasks.isEmpty else {
            return .result(dialog: "You have no tasks due today.")
        }

        let list = tasks.map { "• \($0.title)" }.joined(separator: ", ")
        return .result(dialog: "You have \(tasks.count) tasks today: \(list).")
    }
}

// ─── Complete Task Intent ───
struct CompleteTaskIntent: AppIntent {
    static var title: LocalizedStringResource = "Complete Task"
    static var description = IntentDescription("Mark a task as done.")

    @Parameter(title: "Task Title")
    var taskTitle: String

    @MainActor
    func perform() async throws -> some IntentResult & ProvidesDialog {
        let found = await TaskRepository.shared.complete(title: taskTitle)
        guard found else {
            throw IntentError.taskNotFound(taskTitle)
        }
        return .result(dialog: "Marked "\(taskTitle)" as complete.")
    }
}

// Custom error with a Siri-readable message
enum IntentError: Error, CustomLocalizedStringResourceConvertible {
    case taskNotFound(String)

    var localizedStringResource: LocalizedStringResource {
        switch self {
        case .taskNotFound(let name):
            return "I couldn't find a task named "\(name)"."
        }
    }
}

// Expected output:
// Saying "Hey Siri, add a task to [App Name]" triggers a follow-up:
// Siri: "What's the task?" → User: "Buy groceries" → Siri: "Added 'Buy groceries' to your tasks."
```

### Step 3: Register Siri Phrases

Provide the `AppShortcutsProvider` to define the specific phrases Siri recognizes:

```swift
import AppIntents

struct TaskShortcuts: AppShortcutsProvider {
    static var appShortcuts: [AppShortcut] {
        AppShortcut(
            intent: AddTaskIntent(),
            phrases: [
                "Add a task",
                "Add a task to \(.applicationName)",
                "New task in \(.applicationName)"
            ],
            shortTitle: "Add Task",
            systemImageName: "plus.circle.fill"
        )
        AppShortcut(
            intent: ListTasksIntent(),
            phrases: [
                "What's on my list today",
                "Show my tasks in \(.applicationName)",
                "Read my \(.applicationName) tasks"
            ],
            shortTitle: "Today's Tasks",
            systemImageName: "list.bullet"
        )
        AppShortcut(
            intent: CompleteTaskIntent(),
            phrases: [
                "Complete a task in \(.applicationName)",
                "Mark a task done",
                "Finish a task"
            ],
            shortTitle: "Complete Task",
            systemImageName: "checkmark.circle"
        )
    }
}
```

`\(.applicationName)` is a special placeholder that expands to your app's name at runtime, ensuring Siri anchors phrases to your specific app.

## Adding Widget Buttons (iOS 17+)

App Intents become even more powerful when paired with home screen widget buttons. Add this to your Rork Max session:

```text
Add a Small and Medium home screen widget with two buttons:
- "Add Task" button that triggers AddTaskIntent
- "My List" button that triggers ListTasksIntent
Use a clean, minimal design.
```

The generated widget view:

```swift
import SwiftUI
import WidgetKit
import AppIntents

struct TaskWidgetView: View {
    var entry: TaskEntry

    var body: some View {
        VStack(alignment: .leading, spacing: 10) {
            Text("\(entry.pendingCount) tasks today")
                .font(.caption)
                .foregroundStyle(.secondary)

            // Runs AddTaskIntent without opening the app (iOS 17+)
            Button(intent: AddTaskIntent()) {
                Label("Add Task", systemImage: "plus.circle.fill")
                    .font(.caption.bold())
            }
            .buttonStyle(.borderedProminent)
            .tint(.blue)
        }
        .padding()
    }
}

// Expected output:
// Tapping the widget button triggers AddTaskIntent directly on the Lock Screen
// or Home Screen — no app launch needed.
```

## Common Errors and Fixes

**Siri says "I can't do that in this app"**

Your phrase isn't registered in `AppShortcutsProvider`, or the registered phrase doesn't closely match what you said. Prompt Rork Max: *"Add more natural language variations to the Siri phrases in AppShortcutsProvider."*

**`perform()` runs but Siri shows a generic error**

The error thrown doesn't conform to `CustomLocalizedStringResourceConvertible`. Add the conformance so Siri can vocalize a helpful message instead of a raw error string.

**Widget button does nothing when tapped**

Widget App Intent buttons require iOS 17+. On iOS 16, prompt Rork Max: *"Fall back to a deep link for the widget button on iOS 16."*

**Intent crashes when accessing Core Data**

App Intent extensions run in a separate process. Shared Core Data requires App Groups. Prompt: *"Configure App Groups so the App Intent extension can access the shared Core Data store."*

## FAQ

**Do App Intents work on iPad and Mac?**

Yes. App Intents run natively on iPhone, iPad, and Mac (via Mac Catalyst or native macOS, iOS 16+ / macOS 13+). Apple Watch is also supported, so a single intent definition reaches every device in the user's ecosystem.

**Can I use App Intents with the standard Rork (React Native) version?**

React Native community packages exist for limited SiriKit support, but full App Intents — especially widget button integration, Focus Filters, and custom parameter types — require native Swift. Rork Max is the right choice for a complete implementation.

**How do I test App Intents in Rork Max?**

Use the Shortcuts app on a simulator or real device: tap the "+" button, search for your app's name, and you'll see your registered intents. Trigger them directly in the Shortcuts app before testing via Siri voice. Rork Max's streaming simulator also supports Shortcuts app testing.

**Can App Intents gate features behind a subscription?**

Yes. Inside `perform()`, check the user's subscription state. If they're not subscribed, throw an error with a message like "Upgrade to Premium to use voice commands." Avoid launching a purchase sheet from within the intent — instead, return a deep link that opens the paywall in the app.

**How many Siri phrases can I register per intent?**

Apple recommends 3–5 phrases per intent. Each phrase should cover a different natural way to phrase the request. The `\(.applicationName)` placeholder is required in at least one phrase per intent per Apple's guidelines.

App Intents unlock a layer of engagement that most apps leave on the table. Users who can say "Hey Siri, add a task" or tap a widget button without opening your app return more often and rate the experience higher.
