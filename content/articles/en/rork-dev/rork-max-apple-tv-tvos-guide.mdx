## tvOS: The Forgotten Platform

tvOS is Apple's operating system for Apple TV. While iOS dominates phones, tvOS powers the living room. Yet many developers overlook it:

- **Smaller app ecosystem**: Fewer competitors means more visibility for quality apps.
- **Engaged users**: Apple TV users are TV watchers — a captive, engaged audience.
- **Higher monetization**: Subscription and in-app purchase rates tend to be higher on tvOS than iOS.
- **Unique UX**: tvOS forces better design discipline because the constraints (focus-based navigation, remote control, viewing distance) demand intentionality.

## Key Differences: tvOS vs. iOS

Understanding these differences is essential:

- **Input Device**: iOS uses a touchscreen (direct), tvOS uses Siri Remote (remote, indirect)
- **Viewing Distance**: iOS at 30cm (close), tvOS at 2–3 meters (far)
- **Navigation**: iOS allows tapping anywhere, tvOS uses arrow keys + Select
- **Focus**: iOS has implicit focus (wherever you tap), tvOS uses explicit focus (highlighted UI element)
- **Power**: iOS runs on battery, tvOS is always connected to AC power
- **Screen Size**: iOS is 4–7 inches, tvOS is 55–65 inches or larger

Because users sit **2–3 meters away**, text must be large (minimum 28pt), and buttons must be far apart. Touch is impossible; instead, users navigate via **directional arrows and a central Select button** on the Siri Remote.

This is called **focus-based navigation**. It's not unique to tvOS — it's the same model used by most TV platforms (Android TV, Fire TV, Samsung Smart TV) and game consoles. But it's radically different from iOS.

## Focus-Based UI: Design Principles

### Principle 1: Every Interactive Element Must Be Focusable

On iOS, users can tap any point. On tvOS, users navigate via arrow keys, so only "focusable" UI elements can be interacted with.

Your app must ensure:
- All buttons, inputs, and interactive elements are reachable via arrow keys.
- No "hidden" buttons that exist but can't be focused.
- Tab order (the sequence of focus movement) flows logically left-to-right, top-to-bottom.

### Principle 2: Visual Feedback is Non-Negotiable

On iOS, a button tap is immediate visual feedback. On tvOS, the user presses an arrow key and must **see** which UI element is now focused. Without strong visual cues, users get lost.

When a UI element gains focus:
- **Color change**: Gray → Bright (e.g., orange, blue)
- **Scale increase**: Grow 5–10% larger
- **Shadow/glow**: Add depth with a shadow or glow effect
- **Text update**: Show more details about the focused item below or beside it

### Principle 3: Minimize Tab Stops

Imagine pressing Right → Right → Right → Down → Left five times to reach a button. Exhausting. Design your layout so users reach any interactive element in **3–4 button presses max**.

Good tvOS grids are **3 columns wide, not 5**. Good tvOS forms have **fewer fields, not more**.

## The Siri Remote: Input Model

The Siri Remote has these controls:

- **Touch Pad** (top): Swipe up/down/left/right, or click
- **Direction Keys** (arrows): Up, Down, Left, Right
- **Select** (center): Choose the focused element
- **Menu** (bottom-left): Go back to the previous screen
- **Siri** (microphone): Voice input
- **Home** (top-left): Return to home screen

In SwiftUI, you handle these with:

```swift
// Respond to directional input
.onMoveCommand { direction in
    switch direction {
    case .up:
        focusedIndex = max(0, focusedIndex - 1)
    case .down:
        focusedIndex = min(itemCount - 1, focusedIndex + 1)
    case .left, .right:
        // Horizontal navigation
        break
    }
}

// Respond to Menu button
.onExitCommand {
    dismiss()
}

// Mark an element as focusable
.focusable()
```

## Code Patterns for tvOS

### Pattern 1: Focus-Based Grid

```swift
struct MovieGridView: View {
    @State var selectedIndex: Int = 0
    let movies = ["Inception", "Interstellar", "Tenet", "Oppenheimer", "Dunkirk", "The Prestige"]

    var body: some View {
        VStack {
            LazyVGrid(
                columns: Array(repeating: GridItem(.flexible()), count: 3),
                spacing: 30
            ) {
                ForEach(Array(movies.enumerated()), id: \.offset) { index, movie in
                    MovieTile(
                        title: movie,
                        isSelected: index == selectedIndex
                    )
                    .focusable(true) { focused in
                        if focused {
                            selectedIndex = index
                        }
                    }
                }
            }
            .padding(40)

            SelectedMovieDetail(title: movies[selectedIndex])
                .padding(.top, 40)
        }
        .onMoveCommand { direction in
            let columns = 3
            let rows = (movies.count + columns - 1) / columns

            switch direction {
            case .up:
                selectedIndex = max(0, selectedIndex - columns)
            case .down:
                selectedIndex = min(movies.count - 1, selectedIndex + columns)
            case .left:
                if selectedIndex % columns > 0 {
                    selectedIndex -= 1
                }
            case .right:
                if selectedIndex % columns < columns - 1 && selectedIndex < movies.count - 1 {
                    selectedIndex += 1
                }
            }
        }
    }
}

struct MovieTile: View {
    let title: String
    let isSelected: Bool

    var body: some View {
        VStack {
            Image(systemName: "film.fill")
                .font(.system(size: 80))
                .foregroundColor(.white)

            Text(title)
                .font(.system(size: 20, weight: .semibold))
                .lineLimit(2)
        }
        .frame(height: 280)
        .frame(maxWidth: .infinity)
        .background(isSelected ? Color.orange : Color.gray.opacity(0.5))
        .cornerRadius(15)
        .scaleEffect(isSelected ? 1.12 : 1.0)
        .shadow(
            color: isSelected ? Color.orange.opacity(0.6) : .clear,
            radius: isSelected ? 20 : 0
        )
        .animation(.easeInOut(duration: 0.2), value: isSelected)
    }
}

struct SelectedMovieDetail: View {
    let title: String

    var body: some View {
        VStack(alignment: .leading) {
            Text(title)
                .font(.system(size: 32, weight: .bold))
            Text("Based on a classic sci-fi story, this visionary film explores time, memory, and alternate realities.")
                .font(.system(size: 18))
                .foregroundColor(.gray)
                .lineLimit(3)
        }
        .padding(20)
        .background(Color.black.opacity(0.7))
        .cornerRadius(10)
    }
}
```

This pattern:
- Uses `@State` to track selected index
- `.onMoveCommand` maps arrow keys to grid movement (up, down, left, right)
- `.focusable()` marks tiles as focusable
- `.scaleEffect()` and `.shadow()` provide visual feedback when selected

                        NavigationLink(value: category) {
                            HStack {
                                Image(systemName: "film")
                                    .font(.system(size: 32))

                                    .font(.system(size: 28, weight: .semibold))

                            }

                            .background(Color.blue.opacity(0.7))

## FAQ

### Q: Can I reuse iOS code in tvOS?

**A**: Logic (data models, networking, business rules) — yes. UI — almost never. tvOS UI is fundamentally different (focus-based, large text, remote control). Plan for a complete UI redesign.

### Q: What's the minimum font size for tvOS?

**A**: 28pt. Users sit 2–3 meters away. Smaller text becomes illegible.

### Q: How long is the tvOS App Store review?

**A**: Typically 1–3 days, same as iOS. tvOS review guidelines emphasize **remote interaction quality** — make sure your focus UI is intuitive.

### Q: Do I need a paid Apple Developer account to publish tvOS apps?

**A**: Yes, the same account used for iOS. It costs $99/year.

### Q: Can tvOS apps play video and audio?

**A**: Yes. Use `AVPlayer` for video, `AVAudioPlayer` for audio. tvOS users expect media apps; build with streaming and playback in mind.

### Q: What about games on tvOS?

**A**: Games are less common on tvOS. The Siri Remote has limited controls (directional + select + menu). For games, support MFi controllers (PlayStation 5, Xbox, etc.). Rork Max can help generate game logic; focus on responsive controller handling.

## tvOS Best Practices

1. **Typography**: Minimum 28pt. Headings 40pt+. Subtext 20pt.
2. **Color**: Use high contrast. Avoid thin strokes (they disappear on large TVs).
3. **Focus**: Always highlight the focused element. Use scale, color, and shadow.
4. **Tab Order**: Limit focusable elements to 10 per screen. Make navigation intuitive.
5. **Network**: Assume slower Wi-Fi. Cache aggressively. Pre-buffer video.
6. **Gestures**: Avoid long swipe chains. Arrow keys + select is the primary UX.
7. **Parallax**: Optional, but subtle parallax animations delight users.
