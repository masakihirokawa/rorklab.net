## Why Authentication Matters More Than You Think

Authentication is the front door of your app. A clunky login experience sends users straight to the uninstall button — and an insecure one can destroy months of hard work overnight.

As a Rork developer, you've probably asked yourself: "Which auth service should I use?" Firebase Authentication, Supabase Auth, Auth0 — the options can feel overwhelming. But in 2026, one service has pulled clearly ahead for Expo and React Native apps: **Clerk**.

In this guide, you'll learn how to integrate Clerk into a Rork app and set up Google Sign-In, Apple Sign In, and email/password authentication from scratch.

---

## What Is Clerk — and Why Is It Perfect for Rork?

**Clerk** is an authentication and user management platform designed with developer experience at its core. It ships an official `@clerk/clerk-expo` SDK, pre-built UI components, and a generous free tier that covers most indie apps.

### Why developers choose Clerk over the alternatives

- **Purpose-built Expo SDK**: The `@clerk/clerk-expo` package is maintained by the Clerk team and integrates naturally with Expo Router
- **20+ OAuth providers**: Google, Apple, GitHub, Discord, and more — all configured through one dashboard
- **Polished UI components**: Drop `<SignIn />` and `<SignUp />` into your app and you're done
- **Secure token storage**: Clerk handles SecureStore automatically, so you don't need to roll your own token cache
- **Free up to 10,000 MAU**: Generous enough for most indie and early-stage apps

### Clerk vs Firebase Auth vs Supabase Auth

- **Firebase Auth**: Great if you're already using Firebase services. Expo setup is messier than Clerk's
- **Supabase Auth**: Best choice if your entire backend lives in Supabase — but you'll write the UI yourself
- **Clerk**: Wins on developer experience and out-of-the-box Expo/React Native support. If fast implementation matters, Clerk is the clear choice

## Setup: Creating Your Clerk App and Configuring OAuth

### Step 1: Create a Clerk application

Visit [clerk.com](https://clerk.com), create a free account, and hit "Create application."

1. Enter your app name (e.g., `MyRorkApp`)
2. Select your sign-in methods — check **Google**, **Apple**, and **Email**
3. Click "Create application"

You'll receive a **Publishable Key** (`pk_test_...`) and a **Secret Key** (`sk_test_...`). The Publishable Key is safe for client-side code. Never put the Secret Key anywhere near your frontend.

### Step 2: Configure Google OAuth

In your Clerk dashboard, go to **Configure → SSO connections → Google**.

For development, you can use Clerk's shared credentials — no Google Cloud Console setup needed. For production, create an OAuth 2.0 Client ID in Google Cloud Console and paste it into Clerk.

### Step 3: Set Up Apple Sign In

Apple Sign In is **required by App Store guidelines** for any app that offers third-party social login. Skip it and Apple will reject your submission.

1. Create a Service ID in Apple Developer Console
2. Add it under **Configure → SSO connections → Apple** in Clerk
3. Enable the "Sign In with Apple" capability in your Xcode project (Rork Max handles this automatically in most cases)

## Installing and Configuring Clerk in Your Rork Project

### Step 1: Install the SDK

Open your Rork project's terminal panel and run:

```bash
npx expo install @clerk/clerk-expo expo-secure-store
```

`expo-secure-store` is used by Clerk to persist tokens safely on the device. If your Rork project already includes it, you can skip this.

### Step 2: Add the environment variable

Create or update your `.env` file at the project root:

```bash
# .env
EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=YOUR_CLERK_PUBLISHABLE_KEY
```

The `EXPO_PUBLIC_` prefix makes the variable accessible in Expo client code without exposing it to the wrong environments.

### Step 3: Wrap your app with ClerkProvider

Update `app/_layout.tsx` (the root layout in Expo Router):

```tsx
// app/_layout.tsx
import { ClerkProvider, ClerkLoaded } from '@clerk/clerk-expo';
import { tokenCache } from '@/utils/tokenCache';
import { Stack } from 'expo-router';

const publishableKey = process.env.EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY!;

if (!publishableKey) {
  throw new Error('Missing EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY in .env');
}

export default function RootLayout() {
  return (
    <ClerkProvider publishableKey={publishableKey} tokenCache={tokenCache}>
      <ClerkLoaded>
        <Stack />
      </ClerkLoaded>
    </ClerkProvider>
  );
}
```

Set up the token cache utility to store Clerk's tokens in SecureStore:

```tsx
// utils/tokenCache.ts
import * as SecureStore from 'expo-secure-store';
import { TokenCache } from '@clerk/clerk-expo/dist/cache';

const createTokenCache = (): TokenCache => ({
  getToken: async (key: string) => {
    try {
      return await SecureStore.getItemAsync(key);
    } catch {
      return null;
    }
  },
  saveToken: (key: string, token: string) =>
    SecureStore.setItemAsync(key, token),
  clearToken: (key: string) =>
    SecureStore.deleteItemAsync(key),
});

export const tokenCache = createTokenCache();
```

## Implementing Google and Apple Social Login

### Building the sign-in screen

The `useOAuth` hook is Clerk's clean API for social authentication:

```tsx
// app/(auth)/sign-in.tsx
import { useOAuth } from '@clerk/clerk-expo';
import * as WebBrowser from 'expo-web-browser';
import { View, TouchableOpacity, Text, StyleSheet } from 'react-native';
import { useRouter } from 'expo-router';

// Required on iOS to properly dismiss the browser session
WebBrowser.maybeCompleteAuthSession();

export default function SignInScreen() {
  const router = useRouter();

  const { startOAuthFlow: startGoogleFlow } = useOAuth({
    strategy: 'oauth_google',
  });
  const { startOAuthFlow: startAppleFlow } = useOAuth({
    strategy: 'oauth_apple',
  });

  const handleGoogleSignIn = async () => {
    try {
      const { createdSessionId, setActive } = await startGoogleFlow();
      if (createdSessionId) {
        await setActive!({ session: createdSessionId });
        router.replace('/(tabs)/home');
      }
    } catch (error) {
      console.error('Google sign-in failed:', error);
    }
  };

  const handleAppleSignIn = async () => {
    try {
      const { createdSessionId, setActive } = await startAppleFlow();
      if (createdSessionId) {
        await setActive!({ session: createdSessionId });
        router.replace('/(tabs)/home');
      }
    } catch (error) {
      console.error('Apple sign-in failed:', error);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Sign In</Text>
      <TouchableOpacity style={styles.googleButton} onPress={handleGoogleSignIn}>
        <Text style={styles.googleButtonText}>Continue with Google</Text>
      </TouchableOpacity>
      <TouchableOpacity style={styles.appleButton} onPress={handleAppleSignIn}>
        <Text style={styles.appleButtonText}>Sign in with Apple</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', padding: 24 },
  title: { fontSize: 28, fontWeight: 'bold', marginBottom: 32, textAlign: 'center' },
  googleButton: {
    backgroundColor: '#fff',
    borderWidth: 1,
    borderColor: '#dadce0',
    borderRadius: 8,
    padding: 16,
    marginBottom: 12,
    alignItems: 'center',
  },
  googleButtonText: { fontSize: 16, color: '#3c4043' },
  appleButton: {
    backgroundColor: '#000',
    borderRadius: 8,
    padding: 16,
    alignItems: 'center',
  },
  appleButtonText: { fontSize: 16, color: '#fff' },
});
```

## Protecting Routes and Accessing User Data

### Using `useAuth` and `useUser`

```tsx
// app/(tabs)/profile.tsx
import { useAuth, useUser } from '@clerk/clerk-expo';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useRouter } from 'expo-router';

export default function ProfileScreen() {
  const { isSignedIn, signOut } = useAuth();
  const { user } = useUser();
  const router = useRouter();

  // Redirect unauthenticated users
  if (!isSignedIn) {
    router.replace('/(auth)/sign-in');
    return null;
  }

  const handleSignOut = async () => {
    await signOut();
    router.replace('/(auth)/sign-in');
  };

  return (
    <View style={styles.container}>
      <Text style={styles.greeting}>
        Hello, {user?.firstName ?? 'there'}!
      </Text>
      <Text style={styles.email}>
        {user?.primaryEmailAddress?.emailAddress}
      </Text>
      <TouchableOpacity style={styles.signOutButton} onPress={handleSignOut}>
        <Text style={styles.signOutText}>Sign Out</Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center', padding: 24 },
  greeting: { fontSize: 22, fontWeight: 'bold', marginBottom: 8 },
  email: { fontSize: 14, color: '#666', marginBottom: 32 },
  signOutButton: {
    backgroundColor: '#ff4757',
    borderRadius: 8,
    padding: 16,
    minWidth: 160,
    alignItems: 'center',
  },
  signOutText: { color: '#fff', fontSize: 16, fontWeight: '600' },
});
```

### Auth guard with Expo Router

Add this component to your root `_layout.tsx` to automatically redirect users based on their auth state:

```tsx
// Auth guard component
import { useAuth } from '@clerk/clerk-expo';
import { useSegments, useRouter } from 'expo-router';
import { useEffect } from 'react';

function AuthGuard({ children }: { children: React.ReactNode }) {
  const { isSignedIn, isLoaded } = useAuth();
  const segments = useSegments();
  const router = useRouter();

  useEffect(() => {
    if (!isLoaded) return;
    const inAuthGroup = segments[0] === '(auth)';

    if (!isSignedIn && !inAuthGroup) {
      router.replace('/(auth)/sign-in');
    } else if (isSignedIn && inAuthGroup) {
      router.replace('/(tabs)/home');
    }
  }, [isSignedIn, isLoaded, segments]);

  return <>{children}</>;
}
```

## Frequently Asked Questions

### Q1: Is it hard to migrate from Firebase Auth or Supabase Auth to Clerk?

Migration is doable but requires planning. Clerk supports importing users from other systems, but existing users will typically need to re-authenticate after the switch. If you're early in development, switching to Clerk now is much easier than doing it later.

### Q2: How many monthly active users does the free tier cover?

Clerk's free plan supports up to **10,000 MAU per month**. For indie apps and early-stage products, this is usually more than enough. Paid plans kick in when you scale past that threshold.

### Q3: Does Clerk work with Rork Max (native Swift)?

Yes — Rork Max runs on React Native/Expo under the hood, so `@clerk/clerk-expo` works the same way. If you're writing native Swift modules, you'd use Clerk's Swift SDK separately, but for most Rork Max use cases the Expo SDK is sufficient.

### Q4: Is Apple Sign In really required for App Store approval?

Absolutely. Apple's guidelines require that any app offering a third-party social login (Google, Facebook, etc.) must also provide Apple Sign In as an option. Skipping it is one of the most common causes of App Store rejection.
