## Spatial Computing, Simplified

Apple Vision Pro introduced a fundamentally new paradigm: spatial computing. Eye tracking, hand gestures, immersive 3D environments — building apps that leverage these capabilities traditionally required deep expertise in SwiftUI, RealityKit, ARKit, and Xcode.

**Rork Max removes that barrier entirely.** You can now create and publish native visionOS apps using nothing but natural language prompts. This guide walks you through the complete process.

## Understanding visionOS UI Concepts

Before diving in, familiarize yourself with visionOS-specific UI primitives that Rork Max will work with:

**Window**: A 2D content panel that floats in the user's space — similar to a regular app window but positioned in mixed reality.

**Volume**: A bounded 3D container for displaying three-dimensional content anchored in the real world.

**Immersive Space**: A full-environment experience that replaces the user's surroundings. Ideal for games, virtual tours, and deep-focus experiences.

**Ornament**: An accessory UI attached to the edge of a window — great for toolbars and supplemental controls.

Rork Max understands all of these constructs and selects the appropriate structure based on your natural language description.

## Step 1: Create a Vision Pro Project

Open Rork Max and create a new project. Specify Vision Pro as your platform in the prompt or settings panel.

```
Project name: SolarSystemVR
Platform: Apple Vision Pro (visionOS)
Description: An educational app where users explore the solar system in 3D.
Display all planets in an immersive space.
When a planet is pinch-tapped, show a detail window with facts and stats.
Planets orbit the sun at realistic relative speeds.
```

From this single prompt, Rork Max generates:

- An ImmersiveSpace with a RealityKit 3D scene
- Orbiting planet objects with animation
- Pinch gesture interaction handlers
- Planet detail Windows with facts content
- SwiftUI main menu interface

## Step 2: Design the Spatial UI

Depth and dimensionality are critical to a great Vision Pro experience. Refine the UI through follow-up prompts:

```

2. Show a planet list as horizontally scrollable glass-morphism cards
3. Each card should show a rotating 3D mini planet
4. Animate a zoom-in effect when a card is selected
```

Rork Max is trained on Apple's visionOS Human Interface Guidelines and automatically handles appropriate depth, sizing, and contrast for spatial interfaces.

## Step 7: Publish to the App Store

Vision Pro app publishing follows the same flow as iPhone — made even simpler by Rork Max.

1. **Click "Publish"** in Rork Max — automatic build and archive
2. **Connect Apple Developer Account** (first time only)
3. **Fill in App Store Connect metadata**
   - App name and description (bilingual recommended)
   - Screenshots (Vision Pro requires special capture dimensions)
   - Age rating and category
4. **Optional: TestFlight beta** — distribute to testers for feedback
5. **Submit for review** — Apple typically responds within 1–3 days

**Important for Vision Pro apps**: Apple pays close attention to how eye-tracking and hand-tracking data is used. Your privacy policy must clearly describe any data collection. Rork Max auto-generates a privacy policy template that covers these requirements.

## Best Practices for visionOS Development

### Design for Depth

The z-axis is your new dimension. Establish a clear depth hierarchy:

```
Optimize depth placement:
- Main content: 70cm in front of user
- Secondary menus: 1m in front
- Environmental background: 3m+
- All text labels: Billboard-facing (always facing the user)
```

### Performance Optimization

Vision Pro renders at high refresh rates across a wide field of view. Optimize aggressively:

```
Optimize rendering performance:
- Skip rendering planets outside the view frustum (frustum culling)
- Use low-poly LOD models for distant planets
- Limit shadows to nearby objects only
- Maintain 60fps+ across all interactions
```

### Accessibility

Not all users have the same physical capabilities:

```
Add accessibility support:
- VoiceOver labels for all planets (with descriptive text)
- Dynamic Type support for all text elements
- Fallback to hand-tap interactions for users with eye-tracking limitations
```

## Frequently Asked Questions

### Do I need a physical Vision Pro to develop?

No. The visionOS Simulator on Mac lets you build and test without hardware. Mouse input substitutes for hand gestures, and the simulator renders the full spatial environment. For complete gesture and eye-tracking testing, a physical device is ideal — but you can ship a polished app without one.

### Can I port my existing iPhone app to Vision Pro?

Yes. Tell Rork Max: "Adapt this iPhone app for Vision Pro. Use a Window layout and replace tap interactions with pinch gestures." Rork Max identifies incompatible APIs and suggests visionOS-compatible alternatives. Full one-shot porting isn't guaranteed for complex apps, but the AI dramatically reduces migration time.

### Is App Store review stricter for Vision Pro apps?

The core review process is the same, but Vision Pro apps face additional scrutiny around privacy — specifically eye-tracking and hand-tracking data. You must clearly disclose how this data is handled. Also avoid keeping ImmersiveSpace active unnecessarily, as Apple may flag this as a poor user experience. Rork Max follows Apple's best practices by default.

### Do I need to supply my own 3D models?

No. Rork Max includes an asset library covering common objects — just describe what you need. For custom models, upload USDZ files created with Reality Composer, Blender, or any compatible tool. The AI integrates them into your scene automatically.

### Can I integrate Siri with my Vision Pro app?

Yes — using App Intents. Just tell Rork Max: "Add a Siri shortcut that opens the immersive space when the user says 'Explore planets'." The AI handles the App Intents implementation. See the [Rork Max Siri & App Intents guide](/articles/rork-dev/rork-max-siri-app-intents) for more.
