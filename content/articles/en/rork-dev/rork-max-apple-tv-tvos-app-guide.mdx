## How tvOS Apps Differ from iOS Apps

Before diving into code, it's important to understand the key differences between tvOS and iOS.

**Input model**: Apple TV uses the Siri Remote with a touch-sensitive surface for navigation. Instead of direct touch, users swipe to move focus between on-screen elements and click to select. This is managed by the **Focus Engine**, which is central to every tvOS app.

**Display characteristics**: Apple TV outputs to large screens at 1920×1080 or 3840×2160 resolution. Text needs to be legible from across the room, and generous spacing is essential per Apple's tvOS Human Interface Guidelines.

**Background execution limits**: tvOS imposes strict limitations on background processing. Long-running downloads and background audio require special handling compared to iOS.

**Storage constraints**: tvOS apps have limited persistent local storage. Heavy data should be managed through iCloud or server-side storage.

## Implementing Focus Engine-Aware UI Patterns

The Focus Engine is the most critical concept in tvOS development. In SwiftUI, you manage focus with the `focusable()` modifier and `@FocusState` property wrapper.

```swift
import SwiftUI

struct VideoGridView: View {
    let category: String
    @FocusState private var focusedVideoID: String?

    let videos = [
        VideoItem(id: "v1", title: "Sunset Drive", thumbnail: "sunset"),
        VideoItem(id: "v2", title: "Deep Ocean World", thumbnail: "ocean"),
        VideoItem(id: "v3", title: "Mountain Summit", thumbnail: "mountain"),
        VideoItem(id: "v4", title: "City Lights", thumbnail: "city"),
    ]

    var body: some View {
        ScrollView {
            LazyVGrid(
                columns: [
                    GridItem(.fixed(400)),
                    GridItem(.fixed(400)),
                    GridItem(.fixed(400))
                ],
                spacing: 40
            ) {
                ForEach(videos) { video in
                    VideoCard(video: video)
                        .focusable()
                        .focused($focusedVideoID, equals: video.id)
                        // Scale up the focused card for visual feedback
                        .scaleEffect(focusedVideoID == video.id ? 1.1 : 1.0)
                        .animation(.easeInOut(duration: 0.2), value: focusedVideoID)
                }
            }
            .padding(60)
        }
        .navigationTitle(category)
    }
}

struct VideoCard: View {
    let video: VideoItem

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            RoundedRectangle(cornerRadius: 12)
                .fill(Color.gray.opacity(0.3))
                .frame(height: 225)
                .overlay(
                    Image(systemName: "play.circle.fill")
                        .font(.system(size: 48))
                        .foregroundColor(.white)
                )
            Text(video.title)
                .font(.headline)
        }
        .frame(width: 400)
    }
}

struct VideoItem: Identifiable {
    let id: String
    let title: String
    let thumbnail: String
}

// Expected output:
// A 3-column grid of video cards. As the user swipes
// the Siri Remote, the focused card scales up to 1.1x,
// providing clear visual feedback on the current selection.
```

This pattern — scaling focused elements — is a tvOS design guideline best practice. It provides the visual clarity users need when navigating with a remote control instead of direct touch.

## Integrating the Media Player

Most Apple TV apps involve some form of media playback. tvOS ships with `AVPlayerViewController`, which provides a full-featured player UI that automatically responds to Siri Remote controls — play, pause, seek, and scrub are all handled out of the box.

```swift
import SwiftUI
import AVKit

struct VideoPlayerView: View {
    let videoURL: URL

    var body: some View {
        VideoPlayer(player: AVPlayer(url: videoURL))
            .ignoresSafeArea()
            .onAppear {
                // Configure audio session for playback
                try? AVAudioSession.sharedInstance().setCategory(.playback)
                try? AVAudioSession.sharedInstance().setActive(true)
            }
    }
}

// Usage:
// VideoPlayerView(
//     videoURL: URL(string: "https://example.com/video.mp4")!
// )

// Expected output:
// Full-screen video playback with native tvOS controls.
// Click to play/pause, swipe left/right to seek.
```

With SwiftUI's `VideoPlayer` view, you get a rich playback experience in just a few lines of code. Tell Rork Max to "include video playback" in your prompt, and it will generate this kind of implementation automatically.

## Building a Top Shelf Extension for the Home Screen

The Top Shelf is a special area at the top of the Apple TV home screen that previews your app's content alongside other apps. Implementing a Top Shelf extension is an excellent way to drive engagement by showcasing featured content before users even open your app.

```swift
import TVServices

class ContentProvider: TVTopShelfContentProvider {
    override func loadTopShelfContent() async -> TVTopShelfContent? {
        // Create sectioned Top Shelf items
        let items: [TVTopShelfSectionedItem] = [
            createItem(
                id: "1",
                title: "Featured This Week",
                imageURL: "https://example.com/featured1.jpg"
            ),
            createItem(
                id: "2",
                title: "New Releases",
                imageURL: "https://example.com/featured2.jpg"
            ),
        ]

        let section = TVTopShelfItemCollection(items: items)
        section.title = "Spotlight"

        return TVTopShelfSectionedContent(sections: [section])
    }

    private func createItem(
        id: String,
        title: String,
        imageURL: String
    ) -> TVTopShelfSectionedItem {
        let item = TVTopShelfSectionedItem(identifier: id)
        item.title = title
        if let url = URL(string: imageURL) {
            item.setImageURL(url, for: .screenScale1x)
        }
        return item
    }
}

// Expected output:
// When the user focuses on your app on the Apple TV home screen,
// the Top Shelf area displays featured content previews.
```

Regularly updating your Top Shelf content keeps your app visible and encourages repeat visits — a key engagement strategy for tvOS apps.
