## Introduction

iOS widgets — those glanceable displays on your Home Screen and Lock Screen — are one of the most powerful ways to keep users engaged with your app. But traditionally, building them required deep knowledge of **WidgetKit**, Apple's native Swift framework, which puts them out of reach for most cross-platform tools like React Native or Expo.

**Rork Max** changes everything. Because it generates true native Swift code, it has full access to WidgetKit. In this guide, we'll walk through building widgets from scratch using only conversational prompts — no Xcode expertise needed.

---

## Widget Types and Sizes

Rork Max supports all major widget types:

### Home Screen Widgets

| Size | Description |
|------|-------------|
| Small | 2×2 grid |
| Medium | 4×2 grid (landscape) |
| Large | 4×4 grid |
| Extra Large | iPad only |

### Lock Screen Widgets (iOS 16+)

| Type | Description |
|------|-------------|
| Circular | Round badge near the clock |
| Rectangular | Horizontal banner below the clock |
| Inline | Text displayed above the clock face |

## Step 1: Setting Up Your Project

Start by creating a new **native iOS project** in Rork Max, or add widgets to an existing one.

### Recommended Prompt

```
Build a task management app.
Add a Small Home Screen widget showing today's task count,
and a Circular Lock Screen widget showing the next task name.
```

Rork Max will automatically generate:

- The main app target
- A **Widget Extension** target (e.g., `TaskWidget`)
- `AppIntent` code for App ↔ Widget data sharing
- **App Groups** entitlement configuration for the shared data container

## Step 2: Understanding the WidgetKit Structure

Every widget Rork Max generates is built around three components:

### Provider (Timeline Data Source)

```swift
struct TaskWidgetProvider: TimelineProvider {
    func placeholder(in context: Context) -> TaskEntry {
        TaskEntry(date: Date(), taskCount: 3, nextTask: "Morning standup")
    }

    func getSnapshot(in context: Context, completion: @escaping (TaskEntry) -> Void) {
        let entry = TaskEntry(date: Date(), taskCount: 5, nextTask: "Submit proposal")
        completion(entry)
    }

    func getTimeline(in context: Context, completion: @escaping (Timeline<TaskEntry>) -> Void) {
        // Fetch live data via App Groups shared container
        let sharedDefaults = UserDefaults(suiteName: "group.com.yourapp.tasks")
        let taskCount = sharedDefaults?.integer(forKey: "taskCount") ?? 0
        let nextTask = sharedDefaults?.string(forKey: "nextTask") ?? "No tasks"

        let entry = TaskEntry(date: Date(), taskCount: taskCount, nextTask: nextTask)
        // Refresh every 15 minutes
        let nextUpdate = Calendar.current.date(byAdding: .minute, value: 15, to: Date())!
        let timeline = Timeline(entries: [entry], policy: .after(nextUpdate))
        completion(timeline)
    }
}
```

### Entry (Data Model)

```swift
struct TaskEntry: TimelineEntry {
    let date: Date
    let taskCount: Int
    let nextTask: String
}
```

### Widget View (UI Layer)

```swift
struct TaskWidgetEntryView: View {
    var entry: TaskWidgetProvider.Entry
    @Environment(\.widgetFamily) var family

    var body: some View {
        switch family {
        case .systemSmall:
            SmallWidgetView(entry: entry)
        case .accessoryCircular:
            CircularLockScreenView(entry: entry)
        case .accessoryRectangular:
            RectangularLockScreenView(entry: entry)
        default:
            SmallWidgetView(entry: entry)
        }
    }
}

struct SmallWidgetView: View {
    let entry: TaskEntry

    var body: some View {
        VStack(spacing: 8) {
            Image(systemName: "checkmark.circle.fill")
                .font(.title)
                .foregroundStyle(.blue)
            Text("\(entry.taskCount)")
                .font(.largeTitle.bold())
            Text("Tasks Today")
                .font(.caption)
                .foregroundStyle(.secondary)
        }
        .containerBackground(.fill.tertiary, for: .widget)
    }
}
```

## Step 3: Sharing Data Between App and Widget

Widgets run in their own **separate process**, isolated from your main app. The bridge between them is **App Groups** — a shared `UserDefaults` container that both targets can read and write.

Rork Max handles the App Groups configuration automatically. Here's what the generated code looks like:

```swift
// TaskViewModel.swift (auto-generated by Rork Max)
class TaskViewModel: ObservableObject {
    private let sharedDefaults = UserDefaults(suiteName: "group.com.yourapp.tasks")

    func updateWidget() {
        let pendingCount = tasks.filter { !$0.isCompleted }.count
        let nextTask = tasks.first(where: { !$0.isCompleted })?.title ?? "No tasks"

        sharedDefaults?.set(pendingCount, forKey: "taskCount")
        sharedDefaults?.set(nextTask, forKey: "nextTask")

        // Tell WidgetKit to refresh immediately
        WidgetCenter.shared.reloadAllTimelines()
    }
}
```

### Prompt to trigger this behavior

```
When a task is marked complete, automatically refresh the widget data.
```

Rork Max will inject `WidgetCenter.shared.reloadAllTimelines()` into every task completion action.

## Step 4: Designing the Lock Screen Widget

Lock Screen widgets have tight space constraints. Keep them minimal and scannable:

```swift
struct CircularLockScreenView: View {
    let entry: TaskEntry

    var body: some View {
        ZStack {
            AccessoryWidgetBackground()
            VStack(spacing: 2) {
                Image(systemName: "list.bullet")
                    .font(.caption2)
                Text("\(entry.taskCount)")
                    .font(.headline.bold())
            }
        }
    }
}

struct RectangularLockScreenView: View {
    let entry: TaskEntry

    var body: some View {
        HStack {
            Image(systemName: "arrow.right.circle.fill")
            VStack(alignment: .leading) {
                Text("Next Up")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
                Text(entry.nextTask)
                    .font(.caption.bold())
                    .lineLimit(1)
            }
        }
    }
}
```

## Step 5: User-Configurable Widgets with AppIntent

Starting with iOS 17, you can let users **configure widgets** using `AppIntent`. For example, choosing which project's tasks to display.

### Prompt

```
Let users pick which project to display in the widget settings.
```

Rork Max will generate:

```swift
struct ConfigurationAppIntent: WidgetConfigurationIntent {
    static var title: LocalizedStringResource = "Task Widget Settings"
    static var description = IntentDescription("Choose which project to display.")

    @Parameter(title: "Project")
    var project: ProjectEntity?
}
```

## Troubleshooting Common Issues

### Widget data isn't updating

Check that the App Groups Bundle ID is consistent across both targets. Use this prompt:

```
Verify that App Groups are configured correctly and fix the data not refreshing issue.
```

### Widget preview shows placeholder data

The `placeholder()` method is used in previews and should show representative dummy data. Rork Max generates this automatically, but you can fine-tune it by specifying what sample data to use.

### Updates feel delayed

WidgetKit throttles refresh requests. For time-sensitive updates, always call `WidgetCenter.shared.reloadAllTimelines()` explicitly from your app's code when the underlying data changes.
