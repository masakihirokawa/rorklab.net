## Why Multilingual Support Matters

If you want your app to reach a global audience, internationalization (i18n) is one of the highest-impact investments you can make. Around 70% of App Store and Google Play users live outside English-speaking countries, and apps available in a user's native language consistently see higher download rates and engagement.

With Rork handling the heavy lifting of code generation, adding multilingual support becomes a streamlined process rather than a daunting task. In this guide, we'll walk through implementing `react-i18next` in a Rork-powered React Native app, from initial setup to dynamic language switching and store localization.

This guide is aimed at intermediate developers who have built at least one app with Rork and are comfortable with basic React Native component patterns.

## Prerequisites and Setup

Before getting started, make sure you have the following in place:

- **Rork account** (Free plan works fine for this tutorial)
- **Node.js 18+** (for local development and testing)
- **Expo SDK 52+** (the default environment Rork generates)

When you tell Rork's AI something like "build a multilingual task manager app," it will suggest a basic i18n structure. Understanding the underlying patterns, however, will help you build a more robust implementation.

## Installing and Configuring react-i18next

The most widely used internationalization library for React Native is `react-i18next`. It integrates smoothly with Expo and the project structure that Rork generates.

### Package Installation

```bash
# Run in your Rork project's root directory
npx expo install i18next react-i18next expo-localization
```

The `expo-localization` package detects the device's system language, so your app can automatically display the right language when a user opens it for the first time.

### Translation File Structure

Organize your translation resources within the project. Here's a standard approach:

```
src/
  locales/
    ja.json    // Japanese
    en.json    // English
    zh.json    // Chinese (add as needed)
  i18n.ts      // Initialization file
```

Here's what a Japanese translation file (`ja.json`) looks like:

```json
{
  "common": {
    "appName": "マイアプリ",
    "loading": "読み込み中...",
    "error": "エラーが発生しました",
    "retry": "再試行"
  },
  "home": {
    "title": "ホーム",
    "welcome": "{{name}}さん、こんにちは！",
    "description": "今日も素敵な一日をお過ごしください"
  },
  "settings": {
    "title": "設定",
    "language": "言語",
    "theme": "テーマ",
    "notifications": "通知"
  }
}
```

The English version (`en.json`) follows the same key structure:

```json
{
  "common": {
    "appName": "My App",
    "loading": "Loading...",
    "error": "An error occurred",
    "retry": "Retry"
  },
  "home": {
    "title": "Home",
    "welcome": "Hello, {{name}}!",
    "description": "Have a wonderful day"
  },
  "settings": {
    "title": "Settings",
    "language": "Language",
    "theme": "Theme",
    "notifications": "Notifications"
  }
}
```

The `{{name}}` syntax is called **interpolation** — it lets you inject dynamic values into translated strings.

### The i18n Initialization File

```typescript
// src/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import * as Localization from 'expo-localization';

import ja from './locales/ja.json';
import en from './locales/en.json';

// Get the device's language setting
const deviceLanguage = Localization.getLocales()[0]?.languageCode ?? 'en';

i18n.use(initReactI18next).init({
  resources: {
    ja: { translation: ja },
    en: { translation: en },
  },
  lng: deviceLanguage,       // Default to the device language
  fallbackLng: 'en',         // Fall back to English if language isn't supported
  interpolation: {
    escapeValue: false,       // React handles XSS protection automatically
  },
  compatibilityJSON: 'v4',   // Required for React Native compatibility
});

export default i18n;
// Expected behavior: shows Japanese on Japanese devices, English otherwise
```

Import this file at the top of your app's entry point (`App.tsx` or `_layout.tsx`) with `import './i18n'`, and translation will be available throughout your entire app.

## Using Translations in Components

With i18n configured, you can start using translated text in your components.

### The useTranslation Hook

```tsx
// src/screens/HomeScreen.tsx
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';

export default function HomeScreen() {
  const { t } = useTranslation();

  return (
    <View style={styles.container}>
      <Text style={styles.title}>{t('home.title')}</Text>
      {/* Dynamic text with interpolation */}
      <Text style={styles.welcome}>
        {t('home.welcome', { name: 'User' })}
      </Text>
      <Text style={styles.description}>{t('home.description')}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: 'center', alignItems: 'center' },
  title: { fontSize: 24, fontWeight: 'bold', marginBottom: 12 },
  welcome: { fontSize: 18, marginBottom: 8 },
  description: { fontSize: 14, color: '#666' },
});
// Expected output (English device):
// Home
// Hello, User!
// Have a wonderful day
```

Simply pass a dot-separated key to the `t()` function, and it returns the translated string for the current language. This keeps your code readable while supporting multiple languages seamlessly.

## Implementing a Language Switcher

Let users change the app's language on the fly. We'll use `AsyncStorage` to persist their preference across sessions.

```tsx
// src/components/LanguageSwitcher.tsx
import React from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useTranslation } from 'react-i18next';
import AsyncStorage from '@react-native-async-storage/async-storage';

const LANGUAGES = [
  { code: 'ja', label: '日本語' },
  { code: 'en', label: 'English' },
];

export default function LanguageSwitcher() {
  const { i18n, t } = useTranslation();

  const changeLanguage = async (langCode: string) => {
    await i18n.changeLanguage(langCode);
    // Save the selected language to device storage
    await AsyncStorage.setItem('user-language', langCode);
  };

  return (
    <View style={styles.container}>
      <Text style={styles.label}>{t('settings.language')}</Text>
      <View style={styles.buttonRow}>
        {LANGUAGES.map((lang) => (
          <TouchableOpacity
            key={lang.code}
            style={[
              styles.button,
              i18n.language === lang.code && styles.activeButton,
            ]}
            onPress={() => changeLanguage(lang.code)}
          >
            <Text
              style={[
                styles.buttonText,
                i18n.language === lang.code && styles.activeText,
              ]}
            >
              {lang.label}
            </Text>
          </TouchableOpacity>
        ))}
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16 },
  label: { fontSize: 16, fontWeight: '600', marginBottom: 8 },
  buttonRow: { flexDirection: 'row', gap: 8 },
  button: {
    paddingHorizontal: 16,
    paddingVertical: 8,
    borderRadius: 8,
    backgroundColor: '#f0f0f0',
  },
  activeButton: { backgroundColor: '#007AFF' },
  buttonText: { fontSize: 14, color: '#333' },
  activeText: { color: '#fff' },
});
// Expected behavior: tapping a language instantly switches the UI and persists the choice
```

To restore the saved language on app launch, extend your `i18n.ts`:

```typescript
// Add to i18n.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

async function initLanguage() {
  const savedLang = await AsyncStorage.getItem('user-language');
  if (savedLang) {
    i18n.changeLanguage(savedLang);
  }
}

initLanguage();
```

## Localizing Plurals, Dates, and Currency

Beyond text translation, you'll need to format numbers, dates, and currencies according to each locale.

### Handling Plurals

English distinguishes between singular and plural forms, while Japanese doesn't. `react-i18next` automatically applies the correct pluralization rules for each language.

```json
// en.json
{
  "items": {
    "count_one": "{{count}} item",
    "count_other": "{{count}} items"
  }
}

// ja.json
{
  "items": {
    "count": "{{count}} 件"
  }
}
```

```tsx
// Usage in a component
<Text>{t('items.count', { count: 5 })}</Text>
// English: "5 items" / Japanese: "5 件"
```

### Dates and Currency

For date and currency formatting, the built-in JavaScript `Intl` API is the cleanest approach.

```typescript
// src/utils/formatters.ts
export function formatDate(date: Date, locale: string): string {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);
}

export function formatCurrency(
  amount: number,
  locale: string,
  currency: string
): string {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
  }).format(amount);
}

// Examples
// formatDate(new Date(), 'ja') → "2026年3月28日"
// formatDate(new Date(), 'en') → "March 28, 2026"
// formatCurrency(1500, 'ja', 'JPY') → "￥1,500"
// formatCurrency(15, 'en', 'USD') → "$15.00"
```

## Supporting RTL Languages

If you plan to support Arabic, Hebrew, or other right-to-left (RTL) languages, your layout needs to mirror accordingly. React Native supports RTL through `I18nManager`.

```typescript
// src/utils/rtl.ts
import { I18nManager } from 'react-native';
import * as Updates from 'expo-updates';

export async function setRTL(isRTL: boolean) {
  if (I18nManager.isRTL !== isRTL) {
    I18nManager.forceRTL(isRTL);
    I18nManager.allowRTL(isRTL);
    // Switching RTL requires an app reload
    await Updates.reloadAsync();
  }
}
```

When building for RTL support, avoid hardcoding `left` and `right` in your `StyleSheet`. Use logical properties like `start` and `end` instead — React Native will automatically flip them in RTL mode.

## Localizing Your App Store Listing

Once your app's code supports multiple languages, don't forget to localize your store presence. Store metadata in the local language can dramatically improve search visibility in each region.

- **App name**: Craft an optimized title for each language, including relevant keywords
- **Description**: Write compelling copy tailored to the target market rather than just translating
- **Screenshots**: Capture the UI in each language state and add localized captions
- **Keywords field** (iOS): Set region-specific search keywords for each language

If you're using Rork Max, you can configure multiple localizations when publishing to the App Store with the 2-click publish feature.

## Tips for Using Rork's AI Prompts with i18n

The way you write prompts in Rork can significantly affect the quality of the generated i18n code. Here are some tips to get better results:

- **Specify languages upfront**: Say "Build a task management app that supports Japanese and English" rather than adding localization later
- **Define naming conventions**: Tell the AI "Use `screenName.elementName` as the translation key format"
- **Set fallback behavior**: Instruct "Use English as the fallback language for any unsupported locales"

These specific instructions help Rork's AI generate consistent, well-structured i18n code from the start. For more on getting the best results from Rork's AI, see [Rork Prompt Engineering — Tips for Getting AI to Build Better Apps](/articles/rork-basics/rork-prompt-engineering-app-design-tips).

## FAQ

### Q: How do I add a new language after the initial setup?

A: Create a new translation file (e.g., `zh.json`) in the `locales/` directory and register it in the `resources` object in `i18n.ts`. Copy the existing key structure and fill in the translations. Your components don't need any changes — they'll automatically pick up the new language option.

### Q: What if translated text is too long and breaks the layout?

A: Some languages like German or Finnish produce significantly longer text than English. Use the `numberOfLines` and `adjustsFontSizeToFit` props, or set `flexShrink: 1` on text containers to keep layouts flexible. A good design rule of thumb is to test with text that's 1.5x the length of your English strings.

### Q: How do I manage translations as the app grows?

A: When your translation files get large, use i18next's namespace feature to split them into separate files like `common.json`, `home.json`, and `settings.json`. For team workflows, consider integrating with translation management platforms like Crowdin or Lokalise — they make collaborating with translators much more efficient.
