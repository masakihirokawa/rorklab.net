## What Are Dynamic Island and Live Activities — and Why Rork Max Makes Them Accessible

The **Dynamic Island**, introduced with iPhone 14 Pro, transforms the front-camera cutout into a living, interactive UI surface. Combined with **Live Activities**, it lets your app display real-time, persistent information — delivery status, sports scores, flight countdowns — directly on the lock screen and in the pill-shaped island at the top of the display.

Until Rork Max, building these features required deep expertise in Swift, SwiftUI, and Apple's ActivityKit framework. Rork Max generates native Swift/SwiftUI code from plain English prompts, giving any developer access to even the most advanced iOS APIs.

In this guide you'll learn how to implement Dynamic Island and Live Activities in a food delivery app using Rork Max — from setting up the Widget Extension target to pushing real-time updates from your server.

## How Live Activities Work Under the Hood

Live Activities are powered by **ActivityKit** — a framework that manages the lifecycle of ongoing real-time widgets. Here's the conceptual model:

```swift
// Core ActivityKit structure
import ActivityKit
import SwiftUI

// 1. Define your data model
struct DeliveryAttributes: ActivityAttributes {
    // Static data — set at creation, cannot change
    var orderId: String
    var restaurantName: String

    // Dynamic data — updated throughout the activity's lifetime
    public struct ContentState: Codable, Hashable {
        var currentStep: String      // e.g. "Out for Delivery"
        var estimatedMinutes: Int    // e.g. 12
        var driverName: String       // e.g. "Alex"
    }
}

// 2. Start a Live Activity
func startDeliveryActivity(orderId: String, restaurant: String) {
    let attributes = DeliveryAttributes(
        orderId: orderId,
        restaurantName: restaurant
    )
    let initialState = DeliveryAttributes.ContentState(
        currentStep: "Preparing",
        estimatedMinutes: 30,
        driverName: "Unassigned"
    )

    do {
        let activity = try Activity.request(
            attributes: attributes,
            contentState: initialState,
            pushType: .token   // enables server-side push updates
        )
        print("Live Activity started: \(activity.id)")
        // Expected output: Live Activity started: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
    } catch {
        print("Failed to start Live Activity: \(error.localizedDescription)")
    }
}
```

## Writing Effective Rork Max Prompts for Dynamic Island

The key to getting great Dynamic Island UI from Rork Max is being specific about each display region. Dynamic Island has four distinct regions you can populate:

| Region | Description |
|---|---|
| `compactLeading` | Left side of the compact pill |
| `compactTrailing` | Right side of the compact pill |
| `minimal` | Tiny icon shown when competing with another app's activity |
| Expanded | Full interactive view when the user long-presses the island |

**Example prompt for a delivery app:**

> "Add a Live Activity to this food delivery app. In the compact Dynamic Island view, show the remaining time (number + 'min') on the left and a bicycle icon on the right. In the expanded view, show an order progress bar with four steps (Preparing → Picked Up → On the Way → Delivered) and the estimated arrival time. Mirror the same information on the lock screen in a larger layout."

## Step-by-Step Implementation

### Step 1: Add the Widget Extension Target

Prompt Rork Max to set up the required Xcode target and entitlements:

> "Add a Widget Extension target named 'DeliveryWidget' to this project. Enable Live Activities by setting NSSupportsLiveActivities to true in the app's Info.plist."

Rork Max will configure:
- A new Widget Extension target with WidgetKit + ActivityKit
- `NSSupportsLiveActivities: YES` in `Info.plist`
- Shared App Group for data passing between the app and widget

### Step 2: Build the Activity Manager

```swift
// Rork Max–generated LiveActivityManager
import ActivityKit
import Foundation

@MainActor
class LiveActivityManager: ObservableObject {
    private var currentActivity: Activity<DeliveryAttributes>?

    // Start tracking a delivery
    func startTracking(orderId: String, restaurant: String) async {
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            // Prompt user to enable Live Activities in Settings
            return
        }

        let attributes = DeliveryAttributes(
            orderId: orderId,
            restaurantName: restaurant
        )
        let state = DeliveryAttributes.ContentState(
            currentStep: "Preparing",
            estimatedMinutes: 35,
            driverName: "Unassigned"
        )

        currentActivity = try? Activity.request(
            attributes: attributes,
            contentState: state,
            pushType: .token
        )

        // Send push token to your server
        if let token = currentActivity?.pushToken {
            await sendPushTokenToServer(token)
        }
    }

    // Update the delivery status
    func updateStatus(step: String, minutes: Int, driver: String) async {
        let newState = DeliveryAttributes.ContentState(
            currentStep: step,
            estimatedMinutes: minutes,
            driverName: driver
        )
        await currentActivity?.update(using: newState)
        // Expected result: Dynamic Island and lock screen update instantly
    }

    // End the activity when delivery is complete
    func completeDelivery() async {
        let finalState = DeliveryAttributes.ContentState(
            currentStep: "Delivered!",
            estimatedMinutes: 0,
            driverName: currentActivity?.contentState.driverName ?? ""
        )
        await currentActivity?.end(
            using: finalState,
            dismissalPolicy: .after(.now + 30)
        )
        // Expected result: Activity disappears from Dynamic Island after 30 seconds
    }
}
```

### Step 3: Enable Server-Side Push Updates

For production apps, your server pushes Live Activity updates directly via APNs without requiring the app to be open. For the full push notification setup, see [Push Notifications with Rork Max](/articles/rork-dev/push-notifications-rork-max).

The Live Activity push payload format differs from standard notifications:

```json
{
  "aps": {
    "timestamp": 1741910400,
    "event": "update",
    "content-state": {
      "currentStep": "Out for Delivery",
      "estimatedMinutes": 8,
      "driverName": "Alex"
    },
    "alert": {
      "title": "Almost there!",
      "body": "Alex is 8 minutes away"
    },
    "relevance-score": 100,
    "stale-date": 1741914000
  }
}
```

Key fields:
- `event`: `"update"` to refresh state, `"end"` to dismiss the activity
- `content-state`: Must match your `ContentState` Codable structure exactly
- `stale-date`: Unix timestamp after which the activity is marked as stale if not updated

## Common Errors and How to Fix Them

**Problem: Live Activity doesn't appear**

Check that `NSSupportsLiveActivities` is set to `YES` in your app's `Info.plist` (not the widget target's plist). Also verify the user hasn't disabled Live Activities in **Settings → [Your App] → Live Activities**.

```swift
// Always check authorization before requesting
let info = ActivityAuthorizationInfo()
guard info.areActivitiesEnabled else {
    // Show a Settings deep-link to the user
    return
}
```

**Problem: Dynamic Island doesn't expand on tap**

Dynamic Island expansion requires the device to run iOS 16.2+ on iPhone 14 Pro or later. On older devices, the activity appears as a banner and on the lock screen — the Dynamic Island UI is silently ignored. Always test on the correct simulator model.

**Problem: `Activity.request` throws an error**

iOS limits concurrent Live Activities to **5 per app**. End stale activities before starting new ones:

```swift
// End all existing activities before starting a new one
for activity in Activity<DeliveryAttributes>.activities {
    await activity.end(dismissalPolicy: .immediate)
}
```

**Problem: Push updates aren't reflected in the Dynamic Island**

Ensure the APNs topic uses the `.liveactivity` suffix: `com.yourcompany.yourapp.push-type.liveactivity`. Also confirm the push token is refreshed — Live Activity push tokens rotate and must be re-sent to your server when they change.

## FAQ

**Do I need Rork Max to implement Dynamic Island, or can I use the free Rork plan?**

You need Rork Max. The free Rork plan generates React Native (Expo) code which cannot access native Swift APIs like ActivityKit or WidgetKit. Dynamic Island and Live Activities are exclusively available through Rork Max's Swift/SwiftUI code generation.

**How many Live Activities can run at the same time?**

iOS 16 allows up to 5 concurrent Live Activities per app. From iOS 17 onward, Apple increased this limit. However, it's best practice to end completed activities promptly to conserve battery and avoid confusing the user.

**Can I update a Live Activity when the app is in the background or closed?**

Yes — that's the primary advantage of Live Activities. Once you have the push token, your server can send updates directly to Apple's servers (APNs), which relay them to the device regardless of whether the app is running. The activity continues displaying until you explicitly end it or it reaches its maximum duration (8 hours by default, extendable to 12 hours if the user keeps the app in the foreground).

**Does Rork Max support the new ActivityKit API introduced in iOS 18?**

Yes. Rork Max stays current with the latest iOS SDK. If you want to target a specific iOS version, include it in your prompt — for example: "Use the iOS 18 ActivityKit API with the updated ContentMargins and tinted icons support."

**How do I test Live Activities without a physical iPhone?**

Use Xcode Simulator with an iPhone 14 Pro or later model selected (Simulator → Device). You can simulate Live Activity state changes directly from Xcode's Debug menu → Simulate Live Activity. Remote push testing requires a physical device and a valid APNs token.
