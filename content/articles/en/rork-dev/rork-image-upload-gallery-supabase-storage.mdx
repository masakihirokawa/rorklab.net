## Introduction — Why Image Features Are Essential for Mobile Apps

When building a mobile app, image upload and display functionality is one of the most commonly needed features. Whether you're creating a social media platform, an e-commerce app, a task manager, or a portfolio showcase, nearly every genre of app involves handling images at some point.

Rork lets you quickly scaffold your app's basic structure with AI, but understanding how to architect your image storage layer is critical. Without proper planning, you risk degraded load times, ballooning storage costs, and potential privacy issues.

In this guide, we'll walk through implementing end-to-end image handling in a Rork-built mobile app using **Supabase Storage**. You'll learn how to pick images from the device using Expo's `expo-image-picker`, upload them to Supabase Storage, and display optimized thumbnails in a performant gallery — a pattern you can apply to virtually any app.

**What you'll learn:**

- Setting up Supabase Storage buckets and security policies
- Picking images and taking photos with `expo-image-picker`
- Uploading images to Supabase Storage
- Building a high-performance gallery with FlatList
- Image resizing and caching strategies

**Prerequisites:** Familiarity with Rork basics (project creation and preview), React Native fundamentals (components, useState). A Supabase account is required.

## Supabase Storage Fundamentals and Bucket Setup

Supabase Storage is an S3-compatible object storage service that handles images, videos, documents, and other files. The free tier includes 1GB of storage, which is more than enough to get started with a personal project.

### Creating a Bucket

Log into the Supabase dashboard and select "Storage" from the left menu. Click "New bucket" and configure it as follows:

- **Bucket name**: `app-images`
- **Public bucket**: OFF (we'll control access via RLS)
- **File size limit**: 5MB (appropriate for mobile photos)
- **Allowed MIME types**: `image/jpeg, image/png, image/webp`

### Row Level Security (RLS) Policies

After creating the bucket, set up security policies. Run the following SQL in the Supabase SQL Editor:

```sql
-- Allow authenticated users to upload to their own folder
CREATE POLICY "Users can upload to own folder"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- Allow authenticated users to view their own images
CREATE POLICY "Users can view own images"
ON storage.objects FOR SELECT
TO authenticated
USING (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);

-- Allow authenticated users to delete their own images
CREATE POLICY "Users can delete own images"
ON storage.objects FOR DELETE
TO authenticated
USING (
  bucket_id = 'app-images'
  AND (storage.foldername(name))[1] = auth.uid()::text
);
```

These policies ensure each user can only store, view, and delete files within their own `{user_id}/` folder. Other users' images remain completely inaccessible, protecting privacy by default.

## Picking Images with expo-image-picker

In your Rork project, `expo-image-picker` provides access to the device's photo library and camera for selecting images.

### Installation and Permission Setup

Add the required dependencies to your Rork project:

```bash
# Run in Rork's terminal or specify via prompt
npx expo install expo-image-picker expo-file-system
```

Add camera and photo library permissions in `app.json`:

```json
{
  "expo": {
    "plugins": [
      [
        "expo-image-picker",
        {
          "photosPermission": "This app needs access to your photo library to select images.",
          "cameraPermission": "This app needs camera access to take photos."
        }
      ]
    ]
  }
}
```

### Building the Image Picker Component

The following component handles both photo library selection and camera capture:

```typescript
// components/ImagePickerButton.tsx
import { useState } from 'react';
import { View, TouchableOpacity, Text, Alert, StyleSheet } from 'react-native';
import * as ImagePicker from 'expo-image-picker';

interface ImagePickerButtonProps {
  onImageSelected: (uri: string) => void;
}

export default function ImagePickerButton({ onImageSelected }: ImagePickerButtonProps) {
  const [loading, setLoading] = useState(false);

  // Pick image from photo library
  const pickFromLibrary = async () => {
    const permission = await ImagePicker.requestMediaLibraryPermissionsAsync();
    if (!permission.granted) {
      Alert.alert('Permission Required', 'Please allow access to your photo library.');
      return;
    }

    setLoading(true);
    const result = await ImagePicker.launchImageLibraryAsync({
      mediaTypes: ImagePicker.MediaTypeOptions.Images,
      allowsEditing: true,        // Show crop UI
      aspect: [1, 1],             // Square crop
      quality: 0.8,               // 80% quality to reduce file size
    });

    setLoading(false);
    if (!result.canceled && result.assets[0]) {
      onImageSelected(result.assets[0].uri);
    }
  };

  // Take a photo with camera
  const takePhoto = async () => {
    const permission = await ImagePicker.requestCameraPermissionsAsync();
    if (!permission.granted) {
      Alert.alert('Permission Required', 'Please allow camera access to take photos.');
      return;
    }

    setLoading(true);
    const result = await ImagePicker.launchCameraAsync({
      allowsEditing: true,
      aspect: [1, 1],
      quality: 0.8,
    });

    setLoading(false);
    if (!result.canceled && result.assets[0]) {
      onImageSelected(result.assets[0].uri);
    }
  };

  return (
    <View style={styles.container}>
      <TouchableOpacity style={styles.button} onPress={pickFromLibrary} disabled={loading}>
        <Text style={styles.buttonText}>Choose from Library</Text>
      </TouchableOpacity>
      <TouchableOpacity style={styles.button} onPress={takePhoto} disabled={loading}>
        <Text style={styles.buttonText}>Take a Photo</Text>
      </TouchableOpacity>
    </View>
  );
}

// Expected behavior:
// - Tap "Choose from Library" → Photo library opens, user can select and crop
// - Tap "Take a Photo" → Camera launches, user captures and crops
// - After selection, the local image URI is passed to onImageSelected callback

const styles = StyleSheet.create({
  container: { flexDirection: 'row', gap: 12, marginVertical: 16 },
  button: {
    flex: 1,
    backgroundColor: '#4F46E5',
    paddingVertical: 14,
    borderRadius: 10,
    alignItems: 'center',
  },
  buttonText: { color: '#fff', fontWeight: '600', fontSize: 15 },
});
```

Setting `quality: 0.8` and `aspect: [1, 1]` keeps file sizes manageable while producing a consistent, uniform gallery layout.

## Uploading Images to Supabase Storage

Now let's create a utility function to upload selected images to Supabase Storage:

```typescript
// utils/uploadImage.ts
import * as FileSystem from 'expo-file-system';
import { supabase } from '@/lib/supabase';
import { decode } from 'base64-arraybuffer';

/**
 * Upload an image to Supabase Storage
 * @param uri - Local image URI
 * @param userId - User ID (for folder isolation)
 * @returns The uploaded image path in storage
 */
export async function uploadImage(
  uri: string,
  userId: string
): Promise<string> {
  // 1. Convert the local file to Base64
  const base64 = await FileSystem.readAsStringAsync(uri, {
    encoding: FileSystem.EncodingType.Base64,
  });

  // 2. Generate a unique filename (timestamp + random string)
  const timestamp = Date.now();
  const random = Math.random().toString(36).substring(2, 8);
  const fileName = `${userId}/${timestamp}-${random}.jpg`;

  // 3. Upload to Supabase Storage
  const { data, error } = await supabase.storage
    .from('app-images')
    .upload(fileName, decode(base64), {
      contentType: 'image/jpeg',
      cacheControl: '3600',      // 1-hour cache
      upsert: false,             // Prevent overwriting existing files
    });

  if (error) {
    throw new Error(`Upload failed: ${error.message}`);
  }

  return data.path;
}

/**
 * Get a public URL for an image
 * @param path - File path within Storage
 * @returns Signed URL (valid for 1 hour)
 */
export async function getImageUrl(path: string): Promise<string> {
  const { data, error } = await supabase.storage
    .from('app-images')
    .createSignedUrl(path, 3600); // 1-hour signed URL

  if (error) {
    throw new Error(`URL generation failed: ${error.message}`);
  }

  return data.signedUrl;
}

// Usage example:
// const path = await uploadImage(selectedUri, user.id);
// const url = await getImageUrl(path);
// → Pass url to <Image source={{ uri: url }} /> for display
```

Three key design decisions deserve attention here. First, we use the `decode` function from `base64-arraybuffer` to convert Base64 strings to ArrayBuffer, since React Native environments have limitations around direct `Blob` creation. Second, combining timestamps with random strings in filenames prevents collisions even when users upload multiple images in quick succession. Third, using `createSignedUrl` with an expiration generates time-limited URLs, allowing us to keep the bucket private while still displaying images in the app.

## High-Performance Gallery Display with FlatList

Let's build a gallery component that displays uploaded images in a grid layout. Using React Native's `FlatList` ensures smooth scrolling even with hundreds of images:

```typescript
// components/ImageGallery.tsx
import { useState, useEffect, useCallback } from 'react';
import {
  View, Image, FlatList, Dimensions,
  TouchableOpacity, ActivityIndicator, StyleSheet,
} from 'react-native';
import { supabase } from '@/lib/supabase';
import { getImageUrl } from '@/utils/uploadImage';

interface GalleryImage {
  id: string;
  path: string;
  url: string;
  createdAt: string;
}

interface ImageGalleryProps {
  userId: string;
  refreshTrigger?: number;    // Increment to trigger refresh after upload
  onImagePress?: (image: GalleryImage) => void;
}

const SCREEN_WIDTH = Dimensions.get('window').width;
const NUM_COLUMNS = 3;
const ITEM_SPACING = 2;
const ITEM_SIZE = (SCREEN_WIDTH - ITEM_SPACING * (NUM_COLUMNS + 1)) / NUM_COLUMNS;

export default function ImageGallery({
  userId,
  refreshTrigger,
  onImagePress,
}: ImageGalleryProps) {
  const [images, setImages] = useState<GalleryImage[]>([]);
  const [loading, setLoading] = useState(true);

  const fetchImages = useCallback(async () => {
    setLoading(true);

    // List files in the user's folder
    const { data: files, error } = await supabase.storage
      .from('app-images')
      .list(userId, {
        limit: 100,
        offset: 0,
        sortBy: { column: 'created_at', order: 'desc' },
      });

    if (error || !files) {
      setLoading(false);
      return;
    }

    // Get signed URLs for each file
    const imagePromises = files
      .filter((file) => file.name !== '.emptyFolderPlaceholder')
      .map(async (file) => {
        const path = `${userId}/${file.name}`;
        const url = await getImageUrl(path);
        return {
          id: file.id,
          path,
          url,
          createdAt: file.created_at,
        };
      });

    const resolved = await Promise.all(imagePromises);
    setImages(resolved);
    setLoading(false);
  }, [userId]);

  useEffect(() => {
    fetchImages();
  }, [fetchImages, refreshTrigger]);

  const renderItem = useCallback(
    ({ item }: { item: GalleryImage }) => (
      <TouchableOpacity
        style={styles.imageContainer}
        onPress={() => onImagePress?.(item)}
        activeOpacity={0.8}
      >
        <Image
          source={{ uri: item.url }}
          style={styles.image}
          resizeMode="cover"
        />
      </TouchableOpacity>
    ),
    [onImagePress]
  );

  if (loading) {
    return (
      <View style={styles.loadingContainer}>
        <ActivityIndicator size="large" color="#4F46E5" />
      </View>
    );
  }

  return (
    <FlatList
      data={images}
      renderItem={renderItem}
      keyExtractor={(item) => item.id}
      numColumns={NUM_COLUMNS}
      contentContainerStyle={styles.grid}
      showsVerticalScrollIndicator={false}
      // Performance optimizations
      removeClippedSubviews={true}
      maxToRenderPerBatch={12}
      windowSize={5}
      getItemLayout={(_, index) => ({
        length: ITEM_SIZE + ITEM_SPACING,
        offset: (ITEM_SIZE + ITEM_SPACING) * Math.floor(index / NUM_COLUMNS),
        index,
      })}
    />
  );
}

// Expected output:
// - Images displayed in a 3-column grid layout
// - Smooth scrolling performance even with many images
// - Tapping an image triggers the onImagePress callback

const styles = StyleSheet.create({
  grid: { padding: ITEM_SPACING },
  imageContainer: {
    width: ITEM_SIZE,
    height: ITEM_SIZE,
    margin: ITEM_SPACING / 2,
    borderRadius: 4,
    overflow: 'hidden',
  },
  image: { width: '100%', height: '100%' },
  loadingContainer: { flex: 1, justifyContent: 'center', alignItems: 'center' },
});
```

The combination of `removeClippedSubviews`, `maxToRenderPerBatch`, and `windowSize` props on FlatList frees up memory for off-screen image components and maintains a smooth frame rate during scrolling. Providing `getItemLayout` eliminates the need for FlatList to pre-render all items to calculate scroll positions.

## Image Resizing and Caching Strategies

Uploading full-resolution images straight from the camera leads to bloated storage usage and slow load times. Combining pre-upload resizing with smart display-side caching keeps your app responsive while minimizing costs.

### Pre-Upload Resizing

Use `expo-image-manipulator` to resize images before upload:

```typescript
// utils/resizeImage.ts
import * as ImageManipulator from 'expo-image-manipulator';

/**
 * Resize an image and convert it to JPEG format
 * @param uri - Original image URI
 * @param maxSize - Maximum width/height in pixels
 * @returns URI of the resized image
 */
export async function resizeImage(
  uri: string,
  maxSize: number = 1024
): Promise<string> {
  const result = await ImageManipulator.manipulateAsync(
    uri,
    [{ resize: { width: maxSize } }],  // Resize by width (maintains aspect ratio)
    {
      compress: 0.8,                    // 80% JPEG quality
      format: ImageManipulator.SaveFormat.JPEG,
    }
  );

  return result.uri;
}

// Usage example:
// const resizedUri = await resizeImage(originalUri, 1024);
// const path = await uploadImage(resizedUri, user.id);
// → Even a 4032×3024px photo gets compressed to ~1024×768px before upload
```

Resizing to 1024px and compressing at JPEG 80% typically reduces a smartphone photo from 4–8MB down to 150–300KB. This size delivers perfectly adequate quality for gallery display.

### Cache Optimization with Expo Image

On the display side, the `expo-image` library provides automatic caching that's significantly more sophisticated than the built-in `Image` component:

```typescript
// Using expo-image for display (inside FlatList renderItem)
import { Image } from 'expo-image';

<Image
  source={{ uri: item.url }}
  style={styles.image}
  contentFit="cover"
  placeholder={{ blurhash: 'LKO2?U%2Tw=w]~RBVZRi};RPxuwH' }}
  transition={200}                    // 200ms fade-in
  cachePolicy="memory-disk"           // Memory + disk caching
/>

// Expected behavior:
// - First load: Placeholder (blurhash) → Fade-in to actual image
// - Subsequent loads: Instant display from cache (no network request)
```

Setting `cachePolicy: "memory-disk"` enables two-tier caching: memory cache (fast access while the app is running) and disk cache (persists across app restarts). For an even better perceived loading speed, generate the `blurhash` value server-side during the upload step.

## Frequently Asked Questions (FAQ)

### Q1: How many images can I store on Supabase's free tier?

Supabase's free tier provides 1GB of storage. Assuming approximately 200KB per resized image, that works out to roughly 5,000 images. Actual numbers will vary based on image content and compression, but it's plenty for prototyping and personal projects. For production use at scale, consider upgrading to the Pro plan ($25/month for 100GB).

### Q2: How can I improve image upload speed?

Three approaches will make the biggest difference. First, resize images before upload using `expo-image-manipulator` — 1024px at JPEG 80% maintains good quality while dramatically reducing file size. Second, when uploading multiple images in parallel, use `Promise.allSettled` instead of `Promise.all` so that a single failure doesn't abort all remaining uploads. Third, consider using `expo-file-system`'s `uploadAsync` method, which can upload files directly without the Base64 conversion step, reducing memory consumption.

### Q3: How do I share images publicly with other users?

Add an RLS policy that allows public access under specific conditions. The recommended pattern is to create a separate `image_metadata` table with an `is_public` column, then write a policy like "images where `is_public = true` are viewable by all authenticated users." This table-driven approach gives you far more flexibility than simply making the bucket public — you can implement features like shared albums, temporary links, and granular permissions.

### Q4: Does this approach handle HEIC images from iPhones?

Yes. When using `expo-image-picker`'s `launchImageLibraryAsync`, iOS automatically converts HEIC images to JPEG before returning them. No special handling is needed. However, if you acquire images through other means like `expo-document-picker`, they may arrive in HEIC format — in that case, add a conversion step using `expo-image-manipulator` to ensure JPEG output.
