## What Rork × Figma Integration Gives You

Connecting Figma to Rork lets you import your UI designs directly into your app project — eliminating the time-consuming step of manually translating design files into code. The design's intent arrives intact, not filtered through a developer's interpretation.

Key benefits of the integration:

- Eliminate manual reimplementation of designs from scratch
- Reuse designer-created components directly in your app
- Pull color, spacing, and typography tokens from Figma automatically
- Tighten the design → review → update cycle significantly

---

## Prerequisites

Before connecting Rork and Figma, you'll need:

- A Rork account (basic integration works on free plans)
- A Figma account (free plan is sufficient)
- A Figma **Personal Access Token**

---

## Step 1: Get Your Figma Personal Access Token

1. Log into Figma and click your profile icon in the top right
2. Select **"Settings"**
3. Scroll to the **"Personal access tokens"** section
4. Click **"Generate new token"** and give it a descriptive name
5. Copy the token immediately and store it somewhere secure

> ⚠️ Figma only shows this token once. If you miss it, you'll need to generate a new one.

---

## Step 2: Connect Figma to Rork

1. Open your Rork project
2. Click the Settings icon or navigate to **"Integrations"**
3. Select **"Figma"**
4. Paste your Personal Access Token and click **"Connect"**
5. Enter the URL of the Figma file you want to import

---

## Step 3: Import Your Design

Once Figma is connected, Rork analyzes your design file's structure.

```
What gets imported:
- Frames (screen-level designs)
- Components (buttons, cards, navigation bars, etc.)
- Color styles (brand colors, text colors, states)
- Text styles (font families, sizes, weights)
- Spacing definitions
```

### Design tips that improve import quality

**Name layers meaningfully in Figma**
Layer names become component names in Rork. "Frame 3" helps no one; "LoginButton" or "ProductCard" does. Rename before you import.

**Use Auto Layout extensively**
Components built with Figma's Auto Layout translate more accurately to React Native's Flexbox model. Non-Auto-Layout designs may require more manual correction after import.

**Define variants before importing**
Button states (default/disabled/loading), card modes (expanded/collapsed) — defining variants in Figma gives Rork the context to generate components with proper state handling.

## Step 4: Use Design Tokens

Design tokens — your Figma color, spacing, and typography definitions — can be imported into Rork and used as a shared theme file across your entire app.

```javascript
// Example theme generated from Figma design tokens
export const theme = {
  colors: {
    primary: '#0066FF',       // From Figma brand color style
    secondary: '#FF6600',
    background: '#F5F5F5',
    text: {
      primary: '#1A1A1A',
      secondary: '#666666',
    },
  },
  spacing: {
    xs: 4,
    sm: 8,
    md: 16,
    lg: 24,
    xl: 32,
  },
  typography: {
    heading1: {
      fontFamily: 'Inter',
      fontSize: 28,
      fontWeight: '700',
    },
    body: {
      fontFamily: 'Inter',
      fontSize: 16,
      fontWeight: '400',
    },
  },
};
```

With this theme file in place, a brand color change in Figma becomes a single-line update in your codebase rather than a hunt-and-replace across dozens of files.

## Step 5: Add Interactivity with Rork Companion

What Figma imports is static UI — the visual structure and styles. Rork Companion handles everything dynamic.

```
Figma import provides (static):
- Layout and visual structure
- Colors, typography, spacing
- Component shapes and variants

Rork Companion adds (dynamic):
- Button tap handlers and callbacks
- Form validation logic
- API data fetching and display
- Screen navigation transitions
- Animations and gesture responses
```

After importing, open a conversation with Companion and describe what each screen needs to do. Companion reads the imported component structure and implements the behavior on top of it.

## Best Practices for High-Quality Figma → Rork Conversion

### Before designing in Figma

- Define and organize components properly (use Figma's Components panel, not loose groups)
- Store all colors and text styles in Figma's Styles panel
- Use a consistent spacing system (8px grid is a common standard)

### After importing into Rork

- Verify fonts loaded correctly (custom fonts may need manual installation)
- Check that images and icons are displaying — file references sometimes break across tools
- The import doesn't need to be pixel-perfect; confirm the design *intent* is preserved

## Frequently Asked Questions

**Q. Do I need a Figma paid plan?**

No. Basic import works with a Figma free account. You'll need a paid Figma plan if you use team libraries, branching, or other collaboration features.

**Q. Can I re-sync after updating the design in Figma?**

Yes. Trigger a re-import from Rork after updating your Figma file to pull in the latest changes. Be aware that re-importing may overwrite code you've added in Rork — coordinate syncs carefully if you have existing customizations.

**Q. Does Rork import Figma prototype interactions?**

Not directly. Figma prototype connections (click targets, overlays, transitions) don't translate to Rork. Use Rork Companion to implement those interactions in code after the visual import.

## Summary

Rork × Figma integration narrows the gap between "design" and "working app" substantially:

1. **Get a Figma Personal Access Token** and configure it in Rork's integration settings
2. **Import your design** — components, color styles, text styles, and spacing all come across
3. **Use design tokens** as a shared theme file to keep styles consistent app-wide
4. **Add interactivity with Rork Companion** — tap handlers, navigation, API calls, animations

Designers and developers sharing the same Figma file as the source of truth reduces miscommunication and speeds up iteration. If you haven't tried the integration yet, a small component import is a low-risk way to see what the workflow feels like.
