Building a mobile app with Rork Max gives you powerful UI creation capabilities, but adding intelligent AI features requires a scalable, low-latency backend. This guide walks through constructing a production-ready AI backend using Cloudflare Workers, AI Gateway, Workers AI, and D1 SQLite—all without managing servers.

## Architecture Overview

Understanding the request flow from Rork Max apps to Cloudflare infrastructure is essential.

```
Rork Max App (React Native)
        ↓
    HTTPS Request
        ↓
Cloudflare Workers
        ↓
    ┌─────────────────────────┐
    │ AI Gateway              │ ← Cost reduction, caching, observability
    │ ↓                       │
    │ OpenAI / Anthropic API  │
    │ or                      │
    │ Workers AI (Llama etc)  │
    └─────────────────────────┘
        ↓
    D1 SQLite
   (User data, chat history)
```

Why Cloudflare Workers for AI backends:

- **Edge Computing**: Global network delivers responses with minimal latency
- **Serverless Scale**: Auto-scales from zero to millions of requests
- **AI Gateway**: Caches LLM API calls, reduces costs, provides observability
- **Workers AI**: Run inference at the edge (Llama, Mistral), sub-100ms responses
- **D1 Native**: SQLite database built-in for user data and conversation history

## Cloudflare Workers Setup

Initialize a new Workers project with TypeScript:

```bash
npm create cloudflare@latest rork-ai-backend -- --typescript
cd rork-ai-backend
```

Edit `wrangler.toml` to configure AI Gateway and D1 bindings:

```toml
name = "rork-ai-backend"
main = "src/index.ts"
compatibility_date = "2026-03-10"

[env.production]
name = "rork-ai-backend-prod"

[[d1_databases]]
binding = "DB"
database_name = "rork_data"
database_id = "xxxxxxx"

[[vars]]
AI_GATEWAY_URL = "https://gateway.ai.cloudflare.com/v1/YOUR-ACCOUNT-ID/YOUR-GATEWAY-ID"
JWT_SECRET = "your-jwt-secret-key"
OPENAI_API_KEY = ""
ALLOWED_ORIGINS = "https://yourdomain.com"
```

Create the D1 database:

```bash
wrangler d1 create rork_data
```

Copy the `database_id` from the output into `wrangler.toml`.

## AI Gateway Configuration

Log into the Cloudflare dashboard and navigate to **AI → AI Gateway**. Create a new gateway:

```
Gateway Name: rork-ai-gateway
Provider: OpenAI (or Anthropic)
```

AI Gateway provides:

- **Request Caching**: Cache identical prompts, eliminate duplicate API calls
- **Multi-Provider Routing**: Route requests to different LLM providers based on cost/latency
- **Observability**: Log all requests, track token usage, identify bottlenecks
- **Rate Limiting**: Protect against abuse, enforce per-user/per-app quotas

## Workers AI Integration

Run Llama 2, Mistral 7B, and other models directly at the edge—no external API calls:

```typescript
// src/ai.ts
import { Ai } from "@cloudflare/ai";

export async function generateWithWorkersAI(
  ai: Ai,
  prompt: string
): Promise<string> {
  const response = await ai.run("@cf/meta/llama-2-7b-chat-int8", {
    prompt: prompt,
    max_tokens: 512,
  });

  return response.response;
}

// Streaming support for real-time responses
export async function generateWithWorkersAIStream(
  ai: Ai,
  prompt: string
): Promise<ReadableStream<string>> {
  const response = await ai.run("@cf/meta/llama-2-7b-chat-int8", {
    prompt: prompt,
    max_tokens: 512,
    stream: true,
  });

  return response as ReadableStream<string>;
}
```

Available models: `@cf/meta/llama-2-7b-chat-int8`, `@cf/mistral/mistral-7b-instruct-v0.1`, and many others.

## JWT Authentication Middleware

Secure all requests from Rork Max with JWT tokens:

```typescript
// src/auth.ts
import { sign, verify } from "jsonwebtoken";

const JWT_SECRET = "your-secret-key";

export function createToken(userId: string): string {
  return sign(
    {
      userId,
      iat: Math.floor(Date.now() / 1000),
      exp: Math.floor(Date.now() / 1000) + 86400 * 7, // 7 days
    },
    JWT_SECRET
  );
}

export function verifyToken(token: string): { userId: string } | null {
  try {
    const decoded = verify(token, JWT_SECRET) as any;
    return { userId: decoded.userId };
  } catch (error) {
    return null;
  }
}

export async function authenticateRequest(request: Request): Promise<string | null> {
  const authHeader = request.headers.get("Authorization");
  if (!authHeader?.startsWith("Bearer ")) {
    return null;
  }

  const token = authHeader.slice(7);
  const payload = verifyToken(token);
  return payload?.userId ?? null;
}
```

## RESTful API Endpoints

Design clean API endpoints that Rork Max can call via standard fetch:

```typescript
// src/index.ts
import { Router } from "itty-router";
import { json } from "itty-router-extra";
import { authenticateRequest } from "./auth";
import { generateWithWorkersAI } from "./ai";

const router = Router();

const corsHeaders = {
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
  "Access-Control-Allow-Headers": "Content-Type, Authorization",
};

// Main chat endpoint
router.post("/api/chat", async (request, env) => {
  const userId = await authenticateRequest(request);
  if (!userId) {
    return json({ error: "Unauthorized" }, { status: 401 });
  }

  const { message, conversationId, model = "gpt-3.5-turbo" } = await request.json();

  // Retrieve conversation history
  const history = await env.DB.prepare(
    `SELECT role, content FROM chat_messages
     WHERE conversation_id = ?
     ORDER BY created_at DESC LIMIT 20`
  )
    .bind(conversationId)
    .all();

  // Call LLM through AI Gateway
  const aiGatewayUrl = env.AI_GATEWAY_URL;
  const aiResponse = await fetch(`${aiGatewayUrl}/chat/completions`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${env.OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model,
      messages: [
        ...history.map((h: any) => ({
          role: h.role,
          content: h.content,
        })),
        { role: "user", content: message },
      ],
      temperature: 0.7,
      max_tokens: 1024,
    }),
  });

  const aiData = await aiResponse.json();
  const assistantMessage = aiData.choices[0].message.content;
  const tokensUsed = aiData.usage.total_tokens;

  // Persist messages to D1
  const messageId = crypto.randomUUID();
  await env.DB.prepare(
    `INSERT INTO chat_messages
     (id, conversation_id, role, content, tokens_used, created_at)
     VALUES (?, ?, ?, ?, ?, ?)`
  )
    .bind(messageId, conversationId, "user", message, 0, Date.now())
    .run();

  const assistantId = crypto.randomUUID();
  await env.DB.prepare(
    `INSERT INTO chat_messages
     (id, conversation_id, role, content, tokens_used, created_at)
     VALUES (?, ?, ?, ?, ?, ?)`
  )
    .bind(assistantId, conversationId, "assistant", assistantMessage, tokensUsed, Date.now())
    .run();

  return json({
    id: assistantId,
    conversationId,
    content: assistantMessage,
    tokensUsed,
    timestamp: Date.now(),
  });
});

router.options("*", () => {
  return new Response(null, {
    headers: corsHeaders,
  });
});

export default {
  fetch: router.handle,
};
```

## Connecting from Rork Max

Call the Workers API from your Rork-generated React Native app:

```typescript
import { useState } from "react";
import { View, Text, TextInput, Button, ScrollView } from "react-native";

export default function ChatScreen() {
  const [messages, setMessages] = useState<any[]>([]);
  const [input, setInput] = useState("");
  const [loading, setLoading] = useState(false);
  const [token, setToken] = useState("");

  const sendMessage = async () => {
    if (!input.trim()) return;

    setLoading(true);
    try {
      const response = await fetch(
        "https://rork-ai-backend.workers.dev/api/chat",
        {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
            Authorization: `Bearer ${token}`,
          },
          body: JSON.stringify({
            message: input,
            conversationId: "conv_123",
            model: "gpt-3.5-turbo",
          }),
        }
      );

      if (!response.ok) {
        throw new Error(`API Error: ${response.status}`);
      }

      const result = await response.json();
      setMessages((prev) => [
        ...prev,
        { role: "user", content: input },
        { role: "assistant", content: result.content },
      ]);
      setInput("");
    } catch (error) {
      console.error("Chat error:", error);
      alert("Failed to send message");
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={{ flex: 1, padding: 16 }}>
      <ScrollView style={{ flex: 1, marginBottom: 16 }}>
        {messages.map((msg, idx) => (
          <View
            key={idx}
            style={{
              padding: 8,
              marginVertical: 4,
              backgroundColor: msg.role === "user" ? "#e3f2fd" : "#f5f5f5",
              borderRadius: 8,
            }}
          >
            <Text style={{ fontWeight: "600" }}>{msg.role}</Text>
            <Text>{msg.content}</Text>
          </View>
        ))}
      </ScrollView>
      <TextInput
        value={input}
        onChangeText={setInput}
        placeholder="Type a message..."
        style={{ borderWidth: 1, padding: 8, marginBottom: 8 }}
      />
      <Button title={loading ? "Sending..." : "Send"} onPress={sendMessage} disabled={loading} />
    </View>
  );
}
```

<Callout type="info">
**Token Management**: On app startup, fetch a JWT token from your backend auth endpoint and store it in AsyncStorage. Include this token in the Authorization header for all subsequent API calls.
</Callout>

## Streaming Responses

Enable real-time text streaming for a better user experience:

```typescript
// Streaming endpoint
router.post("/api/chat/stream", async (request, env) => {
  const userId = await authenticateRequest(request);
  if (!userId) {
    return json({ error: "Unauthorized" }, { status: 401 });
  }

  const { message, conversationId } = await request.json();
  const ai = new Ai(env.AI);

  const stream = await generateWithWorkersAIStream(ai, message);

  return new Response(stream, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      Connection: "keep-alive",
      ...corsHeaders,
    },
  });
});
```

Consume streaming responses in Rork with Server-Sent Events:

```typescript
const eventSource = new EventSource(
  "https://rork-ai-backend.workers.dev/api/chat/stream"
);
eventSource.onmessage = (event) => {
  const chunk = event.data;
  setMessages((prev) => {
    const last = prev[prev.length - 1];
    return [
      ...prev.slice(0, -1),
      { ...last, content: last.content + chunk },
    ];
  });
};
```

## Cost Analysis

**Workers AI vs OpenAI API:**

- Workers AI: $0.05 per 1M inference tokens
- OpenAI GPT-3.5: $0.5 per 1M tokens (10× more expensive)

However, Workers AI has massive latency advantages and caches well, offsetting raw per-token costs.

**AI Gateway Caching**: If 50% of your requests hit the cache, you cut API costs in half. For typical chat apps, this saves thousands per month.

**D1 Cost**: First 5GB free, then $0.75/GB. 500M monthly messages consume ~10MB—well within free tier.

<Callout type="warning">
**API Key Management**: Never hardcode API keys. Always store them in Cloudflare Secrets and reference via `env.OPENAI_API_KEY`, etc.
</Callout>
