## Introduction

Freelancer platforms, skill-sharing services, peer-to-peer marketplaces — these all share a common business model: connect sellers and buyers, and take a cut of every transaction. For indie developers, this is one of the most compelling revenue models available.

Historically, building a marketplace app required a team of backend engineers and months of development. With **Rork Max + Supabase + Stripe Connect**, you can build this solo — and ship it in a fraction of the time.

In this tutorial, you'll implement:

- **Dual-role user management** (sellers and buyers via Supabase Auth)
- **Service listing, discovery, and booking flow**
- **Stripe Connect split payments** (with automatic platform fee collection)
- **Booking management and real-time messaging**
- **App Store submission with Rork Max's 2-click publishing**

This tutorial is aimed at developers who are already comfortable with Rork Max basics and have a working knowledge of Supabase. If you're new to Rork, start with [Getting Started with Rork](/en/articles/rork-basics/getting-started) first.

### Setting Up Stripe Connect

Stripe Connect allows your platform to accept payments on behalf of sellers (Connected Accounts) and automatically route funds, minus your platform fee. For marketplaces, the **Express account** flow is the right choice — it keeps onboarding simple for sellers while giving you full control over the payment flow.

In your Stripe Dashboard:

1. Go to **Connect → Settings → Account types** and enable **Express**
2. Register webhooks for `account.updated`, `payment_intent.succeeded`, and `transfer.created`
3. Use the Stripe CLI to test locally: `stripe listen --forward-to localhost:3000/api/webhook`

### Supabase Schema Design

You can ask Rork Max to help design your database schema — just type "design a Supabase schema for a service marketplace app" in the chat. Here's the schema we'll use in this tutorial:

```sql
-- User profiles (shared between sellers and buyers)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  display_name TEXT NOT NULL,
  bio TEXT,
  avatar_url TEXT,
  stripe_account_id TEXT,       -- Seller's Stripe Express account ID
  is_seller BOOLEAN DEFAULT FALSE,
  rating NUMERIC(3,2) DEFAULT 0,
  review_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Service listings
CREATE TABLE services (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  seller_id UUID REFERENCES profiles(id) NOT NULL,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  price INTEGER NOT NULL,        -- Price in cents (or smallest currency unit)
  category TEXT NOT NULL,
  delivery_days INTEGER DEFAULT 3,
  thumbnail_url TEXT,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Orders
CREATE TABLE orders (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  service_id UUID REFERENCES services(id) NOT NULL,
  buyer_id UUID REFERENCES profiles(id) NOT NULL,
  seller_id UUID REFERENCES profiles(id) NOT NULL,
  amount INTEGER NOT NULL,
  platform_fee INTEGER NOT NULL,
  stripe_payment_intent_id TEXT,
  stripe_transfer_id TEXT,
  status TEXT DEFAULT 'pending',  -- pending / paid / in_progress / completed / cancelled
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

-- Messages
CREATE TABLE messages (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  order_id UUID REFERENCES orders(id) NOT NULL,
  sender_id UUID REFERENCES profiles(id) NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Reviews
CREATE TABLE reviews (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  order_id UUID REFERENCES orders(id) NOT NULL,
  reviewer_id UUID REFERENCES profiles(id) NOT NULL,
  reviewee_id UUID REFERENCES profiles(id) NOT NULL,
  rating INTEGER CHECK (rating BETWEEN 1 AND 5),
  comment TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

Don't forget to enable Row Level Security (RLS). Ask Rork Max: "Generate RLS policies for the tables above."

## Architecture & Core Concepts

### How Marketplace Payments Work with Stripe Connect

The payment flow works like this:

```
Buyer → Your Platform (Stripe Account) → Seller (Connected Account)
              ↑ Platform fee retained automatically
```

In practice:
1. Buyer creates a `PaymentIntent` with `transfer_data.destination` set to the seller's Connected Account ID
2. After payment succeeds, funds are automatically transferred to the seller minus your platform fee
3. Your platform retains the fee with zero additional code

**Example with a 15% platform fee:**
- Service price: $100
- Seller receives: $85
- Platform fee: $15 (+ Stripe processing fee)

### Server-Side vs. Client-Side

Stripe Connect operations must always happen server-side. Never expose your Stripe secret key in client code. In this tutorial, we use **Supabase Edge Functions** (Deno runtime) as our serverless backend.

### Step 2: Seller Onboarding with Stripe Connect

When a user taps "Start Selling" for the first time, trigger the Stripe Express onboarding flow via a Supabase Edge Function:

```typescript
// supabase/functions/create-stripe-account/index.ts
import Stripe from "https://esm.sh/stripe@14.0.0";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!);
const supabase = createClient(
  Deno.env.get("SUPABASE_URL")!,
  Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
);

Deno.serve(async (req) => {
  const { userId } = await req.json();

  // Create a Stripe Express account
  const account = await stripe.accounts.create({
    type: "express",
    country: "US",
    capabilities: {
      transfers: { requested: true },
    },
  });

  // Save the Stripe account ID to the user's profile
  await supabase
    .from("profiles")
    .update({ stripe_account_id: account.id, is_seller: true })
    .eq("id", userId);

  // Generate an onboarding link (Stripe-hosted KYC form)
  const accountLink = await stripe.accountLinks.create({
    account: account.id,
    refresh_url: "https://yourapp.com/seller/onboarding/refresh",
    return_url: "https://yourapp.com/seller/onboarding/complete",
    type: "account_onboarding",
  });

  return new Response(JSON.stringify({ url: accountLink.url }), {
    headers: { "Content-Type": "application/json" },
  });
});
```

Tell Rork Max: "Open the onboarding URL in a SafariViewController when the user taps 'Start Selling'." The AI will wire everything up in SwiftUI.

### Step 3: Split Payment Implementation

When a buyer places an order, create a PaymentIntent with automatic fund splitting:

```typescript
// supabase/functions/create-payment-intent/index.ts
import Stripe from "https://esm.sh/stripe@14.0.0";
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!);
const supabase = createClient(
  Deno.env.get("SUPABASE_URL")!,
  Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!
);

const PLATFORM_FEE_RATE = 0.15; // 15% platform fee

Deno.serve(async (req) => {
  const { serviceId, buyerId } = await req.json();

  // Fetch service and seller's Connected Account ID
  const { data: service } = await supabase
    .from("services")
    .select("*, profiles!seller_id(stripe_account_id)")
    .eq("id", serviceId)
    .single();

  const sellerStripeAccountId = service.profiles.stripe_account_id;
  const amount = service.price;
  const platformFee = Math.round(amount * PLATFORM_FEE_RATE);

  // Create PaymentIntent with automatic transfer to seller
  const paymentIntent = await stripe.paymentIntents.create({
    amount,
    currency: "usd",
    application_fee_amount: platformFee,
    transfer_data: {
      destination: sellerStripeAccountId,
    },
    metadata: { serviceId, buyerId, sellerId: service.seller_id },
  });

  // Create order record with 'pending' status
  await supabase.from("orders").insert({
    service_id: serviceId,
    buyer_id: buyerId,
    seller_id: service.seller_id,
    amount,
    platform_fee: platformFee,
    stripe_payment_intent_id: paymentIntent.id,
  });

  return new Response(
    JSON.stringify({ clientSecret: paymentIntent.client_secret }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### Step 4: Real-Time Messaging with Supabase Realtime

After an order is placed, buyers and sellers need to communicate. Ask Rork Max:

> "Build a real-time chat screen using Supabase Realtime. Subscribe to the `messages` table filtered by order ID. Show messages in a scrollable list with timestamps. Place a text input and send button at the bottom."

The key pattern Rork Max will generate looks like this:

```swift
// Rork Max generates the complete SwiftUI view — here's the core subscription logic
.onAppear {
    channel = supabase.realtimeV2.channel("order-\(orderId)")
    channel?.on("INSERT", filter: .init(
      event: "INSERT", schema: "public", table: "messages",
      filter: "order_id=eq.\(orderId)"
    )) { payload in
        if let msg = try? JSONDecoder().decode(Message.self, from: payload.rawData) {
            DispatchQueue.main.async { messages.append(msg) }
        }
    }
    channel?.subscribe()
}
.onDisappear { channel?.unsubscribe() }
```

### Step 5: Reviews & Rating System

After a transaction is complete, enable both parties to leave reviews. Ask Rork Max to build the review screen, then add this Postgres trigger to keep ratings updated automatically:

```sql
-- Automatically update seller rating after each review
CREATE OR REPLACE FUNCTION update_seller_rating()
RETURNS TRIGGER AS $$
BEGIN
  UPDATE profiles
  SET
    rating = (
      SELECT AVG(rating)::NUMERIC(3,2)
      FROM reviews WHERE reviewee_id = NEW.reviewee_id
    ),
    review_count = (
      SELECT COUNT(*) FROM reviews WHERE reviewee_id = NEW.reviewee_id
    )
  WHERE id = NEW.reviewee_id;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER after_review_insert
AFTER INSERT ON reviews
FOR EACH ROW EXECUTE FUNCTION update_seller_rating();
```

## Advanced Patterns

### Full-Text Search for Service Discovery

As your listings grow, search becomes critical. Supabase supports PostgreSQL full-text search out of the box:

```sql
-- Add a search vector column
ALTER TABLE services ADD COLUMN search_vector TSVECTOR;
UPDATE services SET search_vector = to_tsvector('english', title || ' ' || description);
CREATE INDEX services_search_idx ON services USING GIN(search_vector);

-- Full-text search query
SELECT * FROM services
WHERE search_vector @@ plainto_tsquery('english', 'logo design branding')
ORDER BY ts_rank(search_vector, plainto_tsquery('english', 'logo design branding')) DESC;
```

### Webhook-Driven Order Status Updates

Use a Stripe webhook to automatically update order status when payment succeeds:

```typescript
// supabase/functions/stripe-webhook/index.ts
Deno.serve(async (req) => {
  const signature = req.headers.get("stripe-signature")!;
  const body = await req.text();

  const event = stripe.webhooks.constructEvent(
    body, signature, Deno.env.get("STRIPE_WEBHOOK_SECRET")!
  );

  if (event.type === "payment_intent.succeeded") {
    const pi = event.data.object;
    await supabase
      .from("orders")
      .update({ status: "paid" })
      .eq("stripe_payment_intent_id", pi.id);
  }

  return new Response(JSON.stringify({ received: true }));
});
```

## Troubleshooting

**Issue: Stripe Connect onboarding link returns an error**
- Cause: In test mode, some countries require specific test data
- Fix: Use Stripe's test bypass mode in the Dashboard, or ensure you're providing valid test account information

**Issue: "No such account" error when creating PaymentIntent**
- Cause: `stripe_account_id` is NULL — seller hasn't completed onboarding
- Fix: Check that `profiles.stripe_account_id` is populated and the seller's KYC is complete before allowing orders

**Issue: Supabase Realtime not receiving messages**
- Cause: RLS policies blocking the Realtime subscription
- Fix: Ensure your RLS policy grants `SELECT` access on `messages` for authenticated users who are party to the order

**Issue: Rork Max SwiftUI preview crashes**
- Fix: Paste the Xcode error message directly into Rork Max chat. The AI debugs and fixes SwiftUI preview issues automatically

## Publishing & Monetization

### App Store Review Tips for Marketplaces

Marketplace apps require extra care for App Store approval:

- **In-App Purchase requirement**: If you're selling digital goods or services (not physical products or human labor), Apple requires IAP. Review the App Store Review Guidelines for your specific use case
- **User safety features**: Implement a reporting/flagging system for listings. Review teams look favorably on platforms that actively prevent fraud
- **Clear refund policy**: Document your refund and dispute resolution process prominently

Rork Max's 2-click App Store publishing handles the build and submission automatically. See [App Store Publishing Guide](/en/articles/rork-basics/app-store-publishing) for details.

### Platform Fee Strategy

Your fee structure is a key competitive lever. Market benchmarks:

| Platform | Fee Structure |
|----------|--------------|
| Fiverr | 20% (seller) + 5.5% (buyer) |
| Upwork | 5–20% (sliding scale, seller only) |
| Airbnb | 3–5% (host) + 14–16% (guest) |
| Etsy | 6.5% transaction + listing fee |

A good starting point for new marketplaces is **10–15% seller-side only**. This keeps the buyer experience frictionless while still generating sustainable revenue. With Stripe Connect, you can change the fee rate with a single variable update.

For a deeper dive into revenue models, see [Complete Monetization Guide for Rork Apps](/en/articles/rork-business/rork-monetization-complete-guide).

## FAQ

**Q1. How long does Stripe Connect KYC verification take?**

For US-based individuals and businesses, verification typically completes within 1–2 business days. In test mode, you can bypass KYC entirely using Stripe's test account data. Always run thorough testing before going live.

**Q2. Can international sellers use Stripe Connect?**

Yes. Stripe Express accounts are available in 47+ countries. Supported currencies and payout schedules vary by country. Check Stripe's availability page before expanding to new markets.

**Q3. Can I edit the Swift code Rork Max generates?**

Absolutely. Rork Max outputs a complete, standard Xcode project. You can open it in Xcode and make any customizations you need. Think of the AI-generated code as a strong starting point, not a black box.

**Q4. Do I need a legal entity to run a marketplace?**

This depends on your jurisdiction and transaction volume. In many regions, running a marketplace that handles money between third parties may trigger regulatory requirements (e.g., money transmitter licenses in the US). Consult a lawyer familiar with fintech regulations before launching publicly.

**Q5. What's the difference between Stripe Connect Express and Custom?**

Express accounts use Stripe-hosted onboarding and a Stripe dashboard for sellers — simple but less customizable. Custom accounts give you full control over the UI and seller experience, but require significantly more compliance work. For most indie marketplace products, Express is the right choice.

**Q6. How do I handle disputes and refunds?**

Stripe Connect supports refunds and dispute management through the Dashboard or API. For your marketplace, define a clear dispute resolution policy: who can initiate a refund, within what timeframe, and who absorbs the Stripe chargeback fee. Build a basic dispute UI in Rork Max to let both parties communicate before escalating.

**Q7. Should I use Supabase or Firebase for a marketplace backend?**

Supabase (PostgreSQL) is a better fit for marketplace-style applications because of its relational data model, native support for complex joins, Row Level Security, and full-text search. For a detailed comparison, see [Supabase vs Firebase for Rork Apps](/en/articles/rork-dev/supabase-firebase-backend).
