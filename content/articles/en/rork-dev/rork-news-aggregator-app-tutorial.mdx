## Introduction — Learning API Integration Through a News App

Fetching data from external APIs and displaying it in a mobile UI is one of the most common patterns in app development. Weather apps, stock tickers, social media feeds — once you understand this pattern, you can apply it almost anywhere.

In this tutorial, we'll build a **news aggregator app using Rork and NewsAPI**, a free article delivery service. Step by step, we'll cover category-based filtering, keyword search, saving favorite articles with AsyncStorage, and a simple offline fallback.

By the end, you'll have a fully functional personal news reader — and more importantly, a solid grasp of how to build any API-driven app in Rork.

What you'll learn:

- How to call external REST APIs in Rork using `fetch` and `async/await`
- Rendering article lists efficiently with `FlatList`
- Building a horizontal category tab UI
- Persisting favorite articles locally with AsyncStorage
- Handling loading, error, and empty states gracefully

If you're new to API integration in Rork, the [Building a Weather App with Rork — Weather API Integration and Real-Time UI](/articles/rork-basics/rork-weather-app-api-tutorial) tutorial is a great starting point too.

## Step 2: Connect to NewsAPI

### Create a Service Layer

Create `src/services/newsApi.ts` to centralize all API logic. Keeping fetch calls out of your UI components makes the code easier to test and maintain.

```typescript
// src/services/newsApi.ts
// ⚠️ In production, manage API keys via environment variables — 
//    never hard-code them in client-side code

const API_KEY = "YOUR_NEWSAPI_KEY"; // Replace with your key
const BASE_URL = "https://newsapi.org/v2";

export type NewsCategory =
  | "general"
  | "technology"
  | "sports"
  | "business"
  | "entertainment";

export interface Article {
  source: { id: string | null; name: string };
  title: string;
  description: string | null;
  url: string;
  urlToImage: string | null;
  publishedAt: string;
  content: string | null;
}

export interface NewsResponse {
  status: string;
  totalResults: number;
  articles: Article[];
}

// Fetch top headlines by category
export async function fetchTopHeadlines(
  category: NewsCategory = "general",
  country: string = "us"
): Promise<Article[]> {
  const url = `${BASE_URL}/top-headlines?country=${country}&category=${category}&apiKey=${API_KEY}`;

  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`API error: ${response.status}`);
  }

  const data: NewsResponse = await response.json();
  // Filter out articles with null titles or removed content
  return data.articles.filter((a) => a.title && a.title !== "[Removed]");
}

// Search articles by keyword
export async function searchNews(query: string): Promise<Article[]> {
  const encoded = encodeURIComponent(query);
  const url = `${BASE_URL}/everything?q=${encoded}&language=en&sortBy=publishedAt&apiKey=${API_KEY}`;

  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`Search error: ${response.status}`);
  }

  const data: NewsResponse = await response.json();
  return data.articles.filter((a) => a.title && a.title !== "[Removed]");
}
```

This service layer abstracts API details away from your screens. If you ever swap NewsAPI for a different provider, you'll only need to update this one file.

## Step 3: Build the Home Screen (Category Tabs + FlatList)

```tsx
// src/screens/HomeScreen.tsx
import React, { useState, useEffect, useCallback } from "react";
import {
  View,
  Text,
  FlatList,
  TouchableOpacity,
  StyleSheet,
  ActivityIndicator,
  RefreshControl,
} from "react-native";
import { fetchTopHeadlines, Article, NewsCategory } from "../services/newsApi";
import ArticleCard from "../components/ArticleCard";

const CATEGORIES: { label: string; value: NewsCategory }[] = [
  { label: "All", value: "general" },
  { label: "Tech", value: "technology" },
  { label: "Sports", value: "sports" },
  { label: "Business", value: "business" },
  { label: "Entertainment", value: "entertainment" },
];

export default function HomeScreen() {
  const [selectedCategory, setSelectedCategory] =
    useState<NewsCategory>("general");
  const [articles, setArticles] = useState<Article[]>([]);
  const [loading, setLoading] = useState(true);
  const [refreshing, setRefreshing] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const loadArticles = useCallback(async (category: NewsCategory) => {
    try {
      setError(null);
      const data = await fetchTopHeadlines(category);
      setArticles(data);
    } catch (e) {
      setError("Failed to load articles. Please check your connection.");
    } finally {
      setLoading(false);
      setRefreshing(false);
    }
  }, []);

  useEffect(() => {
    setLoading(true);
    loadArticles(selectedCategory);
  }, [selectedCategory]);

  const handleRefresh = () => {
    setRefreshing(true);
    loadArticles(selectedCategory);
  };

  if (loading) {
    return (
      <View style={styles.centered}>
        <ActivityIndicator size="large" color="#007AFF" />
        <Text style={styles.loadingText}>Loading articles...</Text>
      </View>
    );
  }

  if (error) {
    return (
      <View style={styles.centered}>
        <Text style={styles.errorText}>{error}</Text>
        <TouchableOpacity
          onPress={() => loadArticles(selectedCategory)}
          style={styles.retryButton}
        >
          <Text style={styles.retryText}>Try Again</Text>
        </TouchableOpacity>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      {/* Category tabs */}
      <FlatList
        horizontal
        showsHorizontalScrollIndicator={false}
        data={CATEGORIES}
        keyExtractor={(item) => item.value}
        contentContainerStyle={styles.tabContainer}
        renderItem={({ item }) => (
          <TouchableOpacity
            style={[
              styles.tab,
              selectedCategory === item.value && styles.activeTab,
            ]}
            onPress={() => setSelectedCategory(item.value)}
          >
            <Text
              style={[
                styles.tabText,
                selectedCategory === item.value && styles.activeTabText,
              ]}
            >
              {item.label}
            </Text>
          </TouchableOpacity>
        )}
      />

      {/* Article list */}
      <FlatList
        data={articles}
        keyExtractor={(item) => item.url}
        renderItem={({ item }) => <ArticleCard article={item} />}
        refreshControl={
          <RefreshControl refreshing={refreshing} onRefresh={handleRefresh} />
        }
        ListEmptyComponent={
          <View style={styles.centered}>
            <Text style={styles.emptyText}>No articles found in this category</Text>
          </View>
        }
        contentContainerStyle={
          articles.length === 0 ? styles.emptyContainer : undefined
        }
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  centered: { flex: 1, justifyContent: "center", alignItems: "center", padding: 20 },
  tabContainer: { paddingHorizontal: 12, paddingVertical: 10, gap: 8 },
  tab: {
    paddingHorizontal: 16, paddingVertical: 8,
    borderRadius: 20, backgroundColor: "#e0e0e0",
  },
  activeTab: { backgroundColor: "#007AFF" },
  tabText: { fontSize: 14, color: "#555" },
  activeTabText: { color: "#fff", fontWeight: "600" },
  loadingText: { marginTop: 12, color: "#888" },
  errorText: { color: "#cc0000", textAlign: "center", marginBottom: 16 },
  retryButton: {
    paddingHorizontal: 24, paddingVertical: 10,
    backgroundColor: "#007AFF", borderRadius: 8,
  },
  retryText: { color: "#fff", fontWeight: "600" },
  emptyText: { color: "#888", fontSize: 15 },
  emptyContainer: { flex: 1 },
});
```

Three things worth noting here. First, including `selectedCategory` in `useEffect`'s dependency array means the list automatically re-fetches when you switch tabs. Second, pull-to-refresh (`RefreshControl`) gives users a manual way to grab the latest stories. Third, all three UI states — loading, error, and empty — have their own dedicated views, which makes for a much more polished experience.

## Step 4: Build the Article Card Component

```tsx
// src/components/ArticleCard.tsx
import React from "react";
import {
  View,
  Text,
  Image,
  TouchableOpacity,
  StyleSheet,

import { Article } from "../services/newsApi";
import { formatDistanceToNow } from "date-fns";

interface Props {
  article: Article;
}

export default function ArticleCard({ article }: Props) {
  const navigation = useNavigation<any>();

  const timeAgo = formatDistanceToNow(new Date(article.publishedAt), {
    addSuffix: true,

    <TouchableOpacity
      style={styles.card}
      onPress={() => navigation.navigate("ArticleDetail", { article })}
```

## Step 5: Implement Favorites with AsyncStorage

The favorites feature persists saved articles locally so they survive app restarts.

```typescript
// src/hooks/useFavorites.ts
import { useState, useEffect, useCallback } from "react";
import AsyncStorage from "@react-native-async-storage/async-storage";
import { Article } from "../services/newsApi";

const STORAGE_KEY = "@news_favorites";

export function useFavorites() {
  const [favorites, setFavorites] = useState<Article[]>([]);

  // Load saved favorites on mount
  useEffect(() => {
    (async () => {
      try {
        const stored = await AsyncStorage.getItem(STORAGE_KEY);
        if (stored) setFavorites(JSON.parse(stored));
      } catch (e) {
        console.error("Error loading favorites:", e);
      }
    })();
  }, []);

  const save = useCallback(async (items: Article[]) => {
    try {
      await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(items));
      setFavorites(items);
    } catch (e) {
      console.error("Error saving favorites:", e);
    }
  }, []);

  // Toggle an article's favorite status
  const toggleFavorite = useCallback(
    (article: Article) => {
      const exists = favorites.some((f) => f.url === article.url);
      const updated = exists
        ? favorites.filter((f) => f.url !== article.url)
        : [...favorites, article];
      save(updated);
    },
    [favorites, save]
  );

  const isFavorite = useCallback(
    (url: string) => favorites.some((f) => f.url === url),
    [favorites]
  );

  return { favorites, toggleFavorite, isFavorite };
}
```

This `useFavorites` hook is reusable across both the Home and Favorites screens. For a deeper look at local data persistence options — including MMKV and SQLite for high-volume use cases — check out the [Rork Local Data Persistence Guide — AsyncStorage, MMKV & SQLite](/articles/rork-dev/rork-local-storage-asyncstorage-mmkv-sqlite-guide).

## Step 6: Build the Search Screen

```tsx
// src/screens/SearchScreen.tsx
import React, { useState } from "react";
import {
  View,
  TextInput,
  FlatList,
  ActivityIndicator,
  Text,
  StyleSheet,
} from "react-native";
import { searchNews, Article } from "../services/newsApi";
import ArticleCard from "../components/ArticleCard";
import { useDebouncedCallback } from "use-debounce";

export default function SearchScreen() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState<Article[]>([]);
  const [loading, setLoading] = useState(false);
  const [searched, setSearched] = useState(false);

  // Wait 500ms after the user stops typing before making an API call
  const debouncedSearch = useDebouncedCallback(async (text: string) => {
    if (text.trim().length < 2) return;
    setLoading(true);
    setSearched(true);
    try {
      const data = await searchNews(text);
      setResults(data);
    } catch {
      setResults([]);
    } finally {
      setLoading(false);
    }
  }, 500);

  const handleChangeText = (text: string) => {
    setQuery(text);
    debouncedSearch(text);
  };

  return (
    <View style={styles.container}>
      <View style={styles.searchBar}>
        <Text style={styles.searchIcon}>🔍</Text>
        <TextInput
          style={styles.input}
          placeholder="Search articles..."
          value={query}
          onChangeText={handleChangeText}
          returnKeyType="search"
          clearButtonMode="while-editing"
        />
      </View>

      {loading && <ActivityIndicator style={styles.spinner} color="#007AFF" />}

      {!loading && searched && results.length === 0 && (
        <View style={styles.empty}>
          <Text style={styles.emptyText}>No results for "{query}"</Text>
        </View>
      )}

      <FlatList
        data={results}
        keyExtractor={(item) => item.url}
        renderItem={({ item }) => <ArticleCard article={item} />}
        keyboardShouldPersistTaps="handled"
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: "#f5f5f5" },
  searchBar: {
    flexDirection: "row",
    alignItems: "center",
    margin: 12,
    paddingHorizontal: 12,
    backgroundColor: "#fff",
    borderRadius: 12,
    elevation: 2,
    shadowColor: "#000",
    shadowOffset: { width: 0, height: 1 },
    shadowOpacity: 0.08,
    shadowRadius: 4,
  },
  searchIcon: { fontSize: 16, marginRight: 8 },
  input: { flex: 1, height: 44, fontSize: 16 },
  spinner: { marginTop: 30 },
  empty: { flex: 1, justifyContent: "center", alignItems: "center" },
  emptyText: { color: "#888", fontSize: 15 },
});
```

The debounce pattern here is critical: without it, every keystroke would fire an API request and quickly exhaust your free tier quota. The `use-debounce` package's `useDebouncedCallback` is the cleanest way to handle this in React.

## Step 7: Common Errors and Fixes

### ① Network requests blocked

If `fetch` fails consistently, double-check `app.json` for Android's `usesCleartextTraffic` setting. HTTPS endpoints like NewsAPI are generally fine, but development proxies can sometimes interfere.

### ② Images not showing

`urlToImage` can be `null` or point to an expired CDN URL. Always implement a placeholder fallback in your `ArticleCard` component, as shown in Step 4.

### ③ Debounce not working as expected

If you define a debounced function without `useCallback` or `useDebouncedCallback`, it gets recreated on every render, effectively disabling the debounce. Always use `useDebouncedCallback` from the `use-debounce` package.

### ④ AsyncStorage values disappearing in development

Expo's fast refresh can reset in-memory state. This is expected in development — verify behavior in a production build before treating it as a bug.

## FAQ

**Q. Can I use NewsAPI in a production app on the free plan?**

A. No — the free plan is for development only. For published apps, you'll need a paid subscription. Alternatives like Newsdata.io and GNews offer more generous free-tier policies for production use.

**Q. How do I show more sources or add international news?**

A. Adjust the `country` parameter in `fetchTopHeadlines` (e.g., `"gb"` for UK, `"de"` for Germany), or use the `everything` endpoint with a `language` parameter for broader coverage.

**Q. How can I auto-refresh articles in the background?**

A. Use `setInterval` to call `loadArticles` periodically within the app, being mindful of rate limits. For true background refresh, combine `expo-background-fetch` with your API call — this lets articles update even when the app isn't open.

**Q. My favorites list is getting large — will performance suffer?**

A. AsyncStorage handles modest amounts of data well, but serializing hundreds of articles on every read/write adds overhead. For large datasets, consider migrating to MMKV or SQLite. The [Rork Local Data Persistence Guide — AsyncStorage, MMKV & SQLite](/articles/rork-dev/rork-local-storage-asyncstorage-mmkv-sqlite-guide) covers this migration path in detail.

**Q. How do I add offline support so cached articles load without a connection?**

A. Save the last-fetched articles to AsyncStorage after each successful API call. On load, check network availability with `NetInfo` and fall back to cached data if offline. The [Building Offline-Ready Mobile Apps with Rork — AsyncStorage and NetInfo](/articles/rork-dev/rork-offline-ready-asyncstorage-netinfo) tutorial walks through this exact pattern.
