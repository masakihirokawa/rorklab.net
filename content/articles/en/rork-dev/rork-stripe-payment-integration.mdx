## Why Add Stripe Payments to Your Rork App?

Building a mobile app with Rork's AI-powered platform is fast — but shipping a revenue-generating product requires a solid payment layer. Integrating **Stripe** into your Rork-built app unlocks one-time purchases, subscriptions, and marketplace billing, all without leaving the prompt-driven workflow you already love.

This guide walks you through a complete Stripe integration for both the original Rork (React Native) and Rork Max (native Swift), with working code examples you can drop directly into your project.

---

## Why Stripe?

Stripe operates in 180+ countries and supports credit cards, debit cards, Apple Pay, Google Pay, and regional payment methods. For React Native apps — the runtime Rork generates — Stripe provides a first-party SDK (`@stripe/stripe-react-native`) that offers a pre-built **PaymentSheet UI**, reducing payment screen implementation to just a few lines of code.

---

## Prerequisites

### 1. Create a Stripe Account

Sign up at [Stripe Dashboard](https://dashboard.stripe.com) and collect:

- **Publishable Key** (safe to expose in client code): `pk_live_XXXX` or `pk_test_XXXX`
- **Secret Key** (server-only): never embed this in client code

### 2. Choose a Backend

Stripe requires a server to create **PaymentIntents** securely. Pair your Rork app with one of the following:

- **Supabase Edge Functions** (recommended — works seamlessly with Rork)
- Firebase Cloud Functions
- Cloudflare Workers

## Prompting Rork for Payment Integration

Rork understands natural language, so use a descriptive prompt to scaffold the integration:

```
Add Stripe payments to this app.
When the user taps "Buy Now", open a PaymentSheet for $12.00.
Use a Supabase Edge Function as the backend to create the PaymentIntent.
After a successful payment, unlock the premium content screen.
```

Rork will generate:

1. Installation of `@stripe/stripe-react-native`
2. A `StripeProvider` wrapper at the app root
3. PaymentSheet initialization with your Supabase function endpoint
4. Post-payment state updates to reveal gated content

## Code Walkthrough

### Client — React Native (Rork-generated)

```tsx
import { StripeProvider, usePaymentSheet } from "@stripe/stripe-react-native";
import { useState } from "react";
import { Button, View, Text, StyleSheet } from "react-native";

const PUBLISHABLE_KEY = "pk_live_YOUR_KEY";

function CheckoutScreen() {
  const { initPaymentSheet, presentPaymentSheet } = usePaymentSheet();
  const [loading, setLoading] = useState(false);
  const [purchased, setPurchased] = useState(false);

  const handlePurchase = async () => {
    setLoading(true);
    try {
      // Fetch PaymentIntent from your Supabase function
      const res = await fetch(
        "https://YOUR_PROJECT.supabase.co/functions/v1/create-payment-intent",
        {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ amount: 1200, currency: "usd" }),
        }
      );
      const { paymentIntent, ephemeralKey, customer } = await res.json();

      const { error: initError } = await initPaymentSheet({
        customerId: customer,
        customerEphemeralKeySecret: ephemeralKey,
        paymentIntentClientSecret: paymentIntent,
        merchantDisplayName: "My Rork App",
      });

      if (!initError) {
        const { error: presentError } = await presentPaymentSheet();
        if (!presentError) {
          setPurchased(true);
        }
      }
    } finally {
      setLoading(false);
    }
  };

  if (purchased) {
    return (
      <View style={styles.container}>
        <Text style={styles.success}>🎉 Premium unlocked!</Text>
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <Text style={styles.price}>$12.00</Text>
      <Button
        title={loading ? "Processing…" : "Buy Now"}
        onPress={handlePurchase}
        disabled={loading}
      />
    </View>
  );
}

export default function App() {
  return (
    <StripeProvider publishableKey={PUBLISHABLE_KEY}>
      <CheckoutScreen />
    </StripeProvider>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: "center", justifyContent: "center" },
  price: { fontSize: 32, fontWeight: "bold", marginBottom: 20 },
  success: { fontSize: 24, color: "#22c55e" },
});
```

### Server — Supabase Edge Function

```typescript
// supabase/functions/create-payment-intent/index.ts
import Stripe from "https://esm.sh/stripe@14.0.0";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!, {
  apiVersion: "2024-06-20",
});

Deno.serve(async (req) => {
  const { amount, currency } = await req.json();

  const customer = await stripe.customers.create();
  const ephemeralKey = await stripe.ephemeralKeys.create(
    { customer: customer.id },
    { apiVersion: "2024-06-20" }
  );
  const paymentIntent = await stripe.paymentIntents.create({
    amount,
    currency,
    customer: customer.id,
    automatic_payment_methods: { enabled: true },
  });

  return new Response(
    JSON.stringify({
      paymentIntent: paymentIntent.client_secret,
      ephemeralKey: ephemeralKey.secret,
      customer: customer.id,
    }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

Deploy with `supabase functions deploy create-payment-intent` and add your `STRIPE_SECRET_KEY` to the Supabase dashboard secrets.

## Subscriptions with RevenueCat

For recurring billing, **RevenueCat** is the easiest path when building with Rork. It wraps both the App Store and Google Play subscription APIs behind a single interface.

Prompt Rork to scaffold RevenueCat:

```
Add RevenueCat subscriptions.
Users on the free tier see ads.
Subscribers on the $4.99/month plan get an ad-free experience and unlimited saves.
```

Rork will wire up:

- RevenueCat SDK initialization
- Paywall screen with plan selection
- Purchase and restore-purchase flows
- Entitlement checks to gate features

RevenueCat advantages over raw StoreKit or Google Play Billing:

| Feature | RevenueCat | Manual Implementation |
|---------|-----------|----------------------|
| Cross-platform billing | ✅ Unified API | ❌ Two separate SDKs |
| Receipt validation | ✅ Automatic | ❌ Custom server logic |
| Revenue analytics | ✅ Built-in dashboard | ❌ DIY |
| Webhooks (Supabase sync) | ✅ Easy | ❌ Complex |

```

```

Rork Max will generate native Swift classes for the `STPPaymentContext` or the newer `PaymentSheet` Swift API.

## Security Checklist

Stripe integration introduces sensitive keys — handle them carefully:

| Rule | Implementation |
|------|---------------|
| Never embed Secret Key in client | Use environment variables on the server |
| Use test keys during development | `pk_test_` / `sk_test_` in dev, `pk_live_` / `sk_live_` in production |
| Validate Webhook signatures | Use `STRIPE_WEBHOOK_SECRET` to verify event authenticity |
| Enable HTTPS everywhere | Supabase Edge Functions are HTTPS by default |

Tell Rork to load secrets from environment variables:

```
Load STRIPE_SECRET_KEY from environment variables.
Never hardcode it in any file.
```

## Summary

Adding Stripe payments to a Rork app requires three components working together:

1. **A backend function** (Supabase Edge Function) that creates PaymentIntents server-side with your Stripe Secret Key.
2. **The Stripe React Native SDK's PaymentSheet** on the client, initialized with the PaymentIntent secret from your backend.
3. **RevenueCat** for subscriptions, which abstracts the App Store and Google Play billing complexity into a single API.

Rork Max users can take the same prompt-driven approach but leverage native StoreKit 2 and the Stripe iOS SDK for a fully native payment experience. Whichever path you choose, Rork's AI scaffolding means you can go from idea to a working payment screen in under an hour.
