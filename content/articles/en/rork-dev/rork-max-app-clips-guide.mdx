## What Are App Clips — and Why Do They Matter?

Ask any mobile developer about their biggest conversion challenge, and they'll tell you the same thing: the install barrier. Users who might happily spend 30 seconds trying out a feature will abandon the experience the moment they're asked to download a 200MB app, create an account, and log in.

Apple's **App Clips**, introduced in iOS 14, solve this problem elegantly. An App Clip is a lightweight slice of your app — capped at 50MB — that launches instantly from a QR code, NFC tag, Safari Smart App Banner, Maps, or Siri. No download required. No sign-up friction. Just your core experience, delivered in seconds.

The use cases are compelling: restaurant ordering, hotel check-in, retail coupons, event ticketing, and parking payments. And now, with **Rork Max**, you can build them without writing a single line of Swift yourself.

## Understanding the App Clip Architecture

When Rork Max generates your project with an App Clip target, the structure looks like this:

```
MyApp (Main Target)
└── MyAppClip (App Clip Target)
    ├── ContentView.swift       // Clip UI entry point
    ├── AppClipExperience.swift // URL scheme handling
    └── Assets.xcassets         // Clip-specific assets (under 50MB)
```

Key design principles for App Clips:
- Keep the UI focused — 1 to 3 screens maximum
- Enable action immediately after launch
- Use Sign in with Apple or phone number authentication (minimize form fields)
- Guide users toward the full app after completing the task

## Step-by-Step: Building an App Clip with Rork Max

### Step 1: Create the Project and App Clip Target

Enter the following prompt in Rork Max:

```
Create a café ordering App Clip with:
- 3 coffee menu items (Latte, Americano, Cappuccino) with prices and images
- Tap to select an item
- Apple Pay checkout button
- Order confirmation screen after payment
- Set this up as an App Clip target
```

Rork Max will generate SwiftUI code and automatically configure the App Clip target. Here's an example of what it produces:

```swift
// ContentView.swift — Generated by Rork Max
import SwiftUI
import PassKit

struct AppClipContentView: View {
    @State private var selectedItem: MenuItem?
    @State private var showConfirmation = false

    let menuItems = [
        MenuItem(name: "Latte", price: 5.50, imageName: "latte"),
        MenuItem(name: "Americano", price: 4.80, imageName: "americano"),
        MenuItem(name: "Cappuccino", price: 5.80, imageName: "cappuccino"),
    ]

    var body: some View {
        NavigationStack {
            VStack(spacing: 16) {
                Text("Order Now")
                    .font(.largeTitle.bold())
                    .padding(.top)

                ForEach(menuItems) { item in
                    MenuCard(item: item, isSelected: selectedItem?.id == item.id) {
                        selectedItem = item
                    }
                }

                if let item = selectedItem {
                    // Apple Pay button using PassKit
                    ApplePayButton(item: item) {
                        showConfirmation = true
                    }
                    .frame(height: 50)
                    .padding(.horizontal)
                }

                Spacer()

                // Banner to guide users to the full app
                AppClipGetFullAppButton()
            }
            .padding()
            .navigationTitle("Menu")
        }
        .sheet(isPresented: $showConfirmation) {
            ConfirmationView()
        }
    }
}

// Expected output:
// - Vertical list of menu cards
// - Tapping a card selects it and reveals the Apple Pay button
// - Payment triggers the confirmation sheet
```

### Step 2: Configure App Clip URLs (Associated Domains)

App Clips launch via universal links. Ask Rork Max to handle URL routing:

```
Add Associated Domains configuration and URL handling so this
App Clip can be invoked from appclips.example.com
```

Generated URL handling code:

```swift
// AppClipExperience.swift
import SwiftUI

@main
struct MyAppClip: App {
    var body: some Scene {
        WindowGroup {
            AppClipContentView()
                .onContinueUserActivity(
                    NSUserActivityTypeBrowsingWeb,
                    perform: handleUserActivity
                )
        }
    }

    // Extract order data from the invocation URL
    // Example: https://appclips.example.com/order?item=latte&table=5
    private func handleUserActivity(_ userActivity: NSUserActivity) {
        guard let url = userActivity.webpageURL,
              let components = URLComponents(url: url, resolvingAgainstBaseURL: true),
              let itemParam = components.queryItems?.first(where: { $0.name == "item" })?.value
        else { return }

        NotificationCenter.default.post(
            name: .appClipItemSelected,
            object: itemParam
        )
    }
}
```

### Step 3: Optimize for the 50MB Size Limit

App Clips must stay under 50MB. Ask Rork Max to optimize your binary:

```
Optimize the App Clip to stay under 50MB — compress assets,
strip unused frameworks, and use WebP images where possible.
```

Common optimizations:
- Use WebP or HEIC for images instead of PNG
- Embed only the font weights you actually use
- Offload large media to a CDN (Cloudflare works well here)

## Troubleshooting Common Errors

### Error 1: ITMS-90474 — App Clip Exceeds 50MB

```
Fix:
1. Check build size in Xcode's Report Navigator
2. Move large image assets to Cloudflare CDN
3. Tell Rork Max: "Remove [framework name] from the App Clip target"
```

### Error 2: App Clip Won't Launch (Associated Domains Mismatch)

```
Check:
- Is the App Clips domain approved in the Apple Developer Portal?
- Does your server's /.well-known/apple-app-site-association file
  contain the correct "appclips" entry?

{
  "appclips": {
    "apps": ["TEAMID.com.example.myapp.Clip"]
  }
}
```

### Error 3: Apple Pay Doesn't Work in Simulator

Apple Pay in App Clips requires a real device. Add a mock payment mode for development:

```
Add a mock payment button for DEBUG builds.
Show a confirmation dialog instead of triggering Apple Pay
when running in the Simulator.
```

## Advanced Use Cases

### Use Case 1: QR Code Ordering (Restaurants and Retail)

Embed a URL like `https://appclips.example.com/order?table=5` in a QR code on each table. When customers scan it, they get a table-specific ordering experience — no app required. You can combine this with Rork Max's native QR scanning capabilities covered in [Rork Max QR Code Guide](/en/articles/rork-dev/rork-max-qr-code-guide).

### Use Case 2: NFC-Powered Check-In (Hotels and Events)

Place NFC tags at hotel room doors or event entrances. Guests tap their phone and the App Clip launches a check-in flow powered by CoreNFC — which Rork Max generates automatically when you specify it in your prompt.

### Use Case 3: Smooth Transition to the Full App

Use `SKOverlay` to suggest the full app after a successful interaction:

```swift
// Generated by Rork Max: guide users to the full app after checkout
import StoreKit

struct AppClipGetFullAppButton: View {
    var body: some View {
        Button("Get the full app") {
            guard let scene = UIApplication.shared.connectedScenes.first as? UIWindowScene else { return }
            let config = SKOverlay.AppClipConfiguration(position: .bottom)
            let overlay = SKOverlay(configuration: config)
            overlay.present(in: scene)
        }
        .font(.footnote)
        .foregroundColor(.secondary)
    }
}
```

## FAQ

**Q. Can App Clips work on Android too?**

A. No — App Clips are Apple-only (iOS 16+). Google offers a similar feature called Instant Apps for Android, but the two are built on completely different architectures. If you need to support both platforms, build a standard Rork (React Native) app alongside your Rork Max App Clip.

**Q. Can I build App Clips on Rork Max's free tier?**

A. The free tier limits you to about 5 prompts per week, which isn't enough for a full App Clip build cycle. In practice, you'll need the Max plan ($200/month) to develop App Clips. That said, you can prototype the UI on the free tier and then switch to the Max plan to finalize the App Clip target configuration.

**Q. Does an App Clip require a separate App Store review?**

A. App Clips are reviewed alongside your main app — there's no separate submission process. However, you do need to register each App Clip Experience URL in App Store Connect separately. Also, Apple's App Clip design guidelines (single-purpose UI, instant action) are enforced during review, so keep your Clip focused to avoid rejection.

**Q. Can Rork Max generate multiple App Clip Experiences for different URL patterns?**

A. Yes. App Store Connect supports multiple App Clip Experiences, each mapped to a different URL pattern. Tell Rork Max to "switch the displayed content based on URL query parameters" and it will handle the routing logic automatically.
