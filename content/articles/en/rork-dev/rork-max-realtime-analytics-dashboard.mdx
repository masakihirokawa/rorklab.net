- Live active user count that updates automatically

### Setting Up the Supabase Schema

First, create the event storage table in your Supabase project. Open the SQL Editor in your Supabase Dashboard and run the following:

```sql
-- Analytics events table
create table analytics_events (
  id uuid default gen_random_uuid() primary key,
  event_name text not null,
  user_id uuid,
  properties jsonb default '{}',
  created_at timestamptz default now()
);

-- Enable real-time broadcasting
alter publication supabase_realtime add table analytics_events;

-- Hourly aggregation view for charts
create or replace view hourly_event_counts as
select
  date_trunc('hour', created_at) as hour,
  event_name,
  count(*) as event_count
from analytics_events
where created_at > now() - interval '24 hours'
group by date_trunc('hour', created_at), event_name
order by hour desc;

-- RLS policy (admin-only read access)
alter table analytics_events enable row level security;
create policy "Admin read access"
  on analytics_events for select
  using (auth.jwt() ->> 'role' = 'admin');
```

The key design choice here is using a JSONB column for `properties`. This gives you the flexibility to store any metadata — screen names, button IDs, version numbers — without needing schema migrations every time you add a new event property.

## Implementing the WebSocket Connection

The core of the dashboard is a `RealtimeManager` class that handles the WebSocket subscription to Supabase. Here's the full implementation:

```swift
import Foundation
import Supabase

// Manages real-time event streaming from Supabase
@MainActor
class RealtimeManager: ObservableObject {
    @Published var activeUsers: Int = 0
    @Published var recentEvents: [AnalyticsEvent] = []
    @Published var isConnected: Bool = false

    private let client: SupabaseClient
    private var channel: RealtimeChannelV2?

    init() {
        // Initialize the Supabase client
        self.client = SupabaseClient(
            supabaseURL: URL(string: "https://your-project.supabase.co")!,
            supabaseKey: "your-anon-key"
        )
    }

    // Start listening for new events via WebSocket
    func subscribe() async {
        let channel = client.realtimeV2.channel("analytics")

        // Watch for INSERT events on the analytics table
        let insertions = channel.postgresChange(
            InsertAction.self,
            table: "analytics_events"
        )

        await channel.subscribe()
        self.channel = channel
        self.isConnected = true

        // Process the async event stream
        for await insertion in insertions {
            let event = AnalyticsEvent(
                id: insertion.record["id"]?.stringValue ?? "",
                eventName: insertion.record["event_name"]?.stringValue ?? "",
                properties: insertion.record["properties"]?.objectValue ?? [:],
                createdAt: Date()
            )
            // Keep the 50 most recent events
            self.recentEvents.insert(event, at: 0)
            if self.recentEvents.count > 50 {
                self.recentEvents.removeLast()
            }
            // Recalculate active user count
            await self.refreshActiveUsers()
        }
    }

    // Count unique users from the last 5 minutes
    func refreshActiveUsers() async {
        do {
            let response = try await client
                .from("analytics_events")
                .select("user_id")
                .gte("created_at", value: ISO8601DateFormatter()
                    .string(from: Date().addingTimeInterval(-300)))
                .execute()
            // Count distinct user_ids
            let decoded = try JSONDecoder()
                .decode([[String: String]].self, from: response.data)
            let uniqueUsers = Set(decoded.compactMap { $0["user_id"] })
            self.activeUsers = uniqueUsers.count
        } catch {
            print("Active users fetch error: \(error)")
        }
    }

    // Clean up the subscription
    func unsubscribe() async {
        if let channel = self.channel {
            await client.realtimeV2.removeChannel(channel)
        }
        self.isConnected = false
    }
}
```

Three things worth highlighting in this implementation.

First, the `@MainActor` annotation ensures all UI-bound property updates happen on the main thread — a requirement for SwiftUI's reactive rendering.

Second, the `for await` loop leverages Supabase's Realtime V2 SDK, which conforms to `AsyncSequence`. This integrates naturally with Swift Concurrency and makes the streaming code remarkably clean.

Third, the 50-event rolling buffer prevents memory growth during long-running sessions. Even if the dashboard stays open for hours, memory usage remains constant.

## Batch Processing and Performance Optimization

Real-time streaming is powerful, but it can become a bottleneck under heavy traffic. Here are two key optimization patterns.

### Event Batching

Instead of sending each event individually, buffer them and flush in batches:

```swift
actor EventBatcher {
    private var buffer: [[String: AnyJSON]] = []
    private let client: SupabaseClient
    private let flushInterval: TimeInterval = 5.0
    private let maxBatchSize: Int = 50

    init(client: SupabaseClient) {
        self.client = client
        Task { await self.startPeriodicFlush() }
    }

    func add(event: [String: AnyJSON]) {
        buffer.append(event)
        if buffer.count >= maxBatchSize {
            Task { await flush() }
        }
    }

    private func flush() async {
        guard !buffer.isEmpty else { return }
        let batch = buffer
        buffer = []
        do {
            try await client
                .from("analytics_events")
                .insert(batch)
                .execute()
        } catch {
            // Put failed events back in the buffer
            buffer.insert(contentsOf: batch, at: 0)
            print("Batch insert failed: \(error)")
        }
    }

    private func startPeriodicFlush() async {
        while !Task.isCancelled {
            try? await Task.sleep(for: .seconds(flushInterval))
            await flush()
        }
    }
}
// Expected output: Batch INSERT executes every 5 seconds or every 50 events
```

### Dashboard-Side Throttling

When events are streaming in rapidly, you'll want to limit how often the UI refreshes:

```swift
extension RealtimeManager {
    // Limit UI refreshes to once per second
    func throttledRefresh() {
        let now = Date()
        if now.timeIntervalSince(lastRefresh) < 1.0 {
            return
        }
        lastRefresh = now
        Task { await refreshActiveUsers() }
    }
}
```

## Security and Access Control

An analytics dashboard contains sensitive data, so proper access control is essential.

### Strengthening Row Level Security

```sql
-- Only users with the admin role can access dashboard data
create policy "Dashboard admin only"
  on analytics_events for select
  using (
    exists (
      select 1 from user_roles
      where user_roles.user_id = auth.uid()
      and user_roles.role = 'admin'
    )
  );

-- Only authenticated users can write events
create policy "Authenticated insert"
  on analytics_events for insert
  with check (auth.uid() is not null);
```

### Secure API Key Management

In your Rork Max project, avoid hardcoding API keys. Load them from the app's configuration instead:

```swift
enum Config {
    static let supabaseURL: URL = {
        guard let urlString = Bundle.main
            .object(forInfoDictionaryKey: "SUPABASE_URL") as? String,
              let url = URL(string: urlString) else {
            fatalError("SUPABASE_URL not configured")
        }
        return url
    }()

    static let supabaseKey: String = {
        guard let key = Bundle.main
            .object(forInfoDictionaryKey: "SUPABASE_KEY") as? String else {
            fatalError("SUPABASE_KEY not configured")
        }
        return key
    }()
}
// Expected output: Keys are loaded safely from Info.plist
```

## FAQ

### Q1: Is the Supabase free tier enough for this?

Supabase's free tier supports up to 200 concurrent Realtime connections and 2 million Realtime messages per month. That's plenty for personal projects and prototypes. For production apps with significant traffic, consider upgrading to the Pro plan ($25/month). You can monitor connection counts through the Realtime Inspector in your Supabase dashboard.

### Q2: How do you handle WebSocket disconnections?

The Supabase Swift SDK includes automatic reconnection out of the box. However, you may lose events during extended disconnections. The recommended approach is to add reconnection detection to your `RealtimeManager` and implement a "gap fill" pattern — fetching missed events via a standard HTTP query when the connection recovers.

### Q3: Can I build this with the React Native version of Rork instead?

The overall architecture is the same, but you'd swap in React Native libraries for the UI components (such as react-native-charts-wrapper). You'll also need to account for React Native's limitations around background WebSocket connections. For reliability and performance, Rork Max's native Swift approach has a clear advantage here.

### Q4: How do I adapt the dashboard for iPad and Mac?

Since Rork Max generates SwiftUI code, iPad support is relatively straightforward. Switch to a `NavigationSplitView` for a sidebar layout and use the `horizontalSizeClass` environment variable to adapt based on screen size. Mac Catalyst support also requires minimal changes when your codebase is SwiftUI-native.
