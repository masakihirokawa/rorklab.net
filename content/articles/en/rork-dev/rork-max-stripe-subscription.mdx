## Introduction

Building a SaaS application or subscription-based service requires robust payment processing and subscription management. However, the complexity of implementing these features often consumes significant development time, pulling attention away from core product work.

Rork Max solves this problem. By integrating with Stripe, developers can build production-ready subscription systems in hours rather than weeks—without wrestling with complex backend logic or API integrations.

This guide walks you through implementing a practical, scalable subscription system using Rork Max and Stripe. Whether you're launching a premium tier, tiered pricing model, or entirely subscription-based product, you'll learn the exact steps to get from zero to production.

## Prerequisites

Before diving into this guide, make sure you have:

- **A Rork Max account** with basic platform familiarity. If you're new, start with our [Stripe payment integration guide](/articles/rork-dev/rork-stripe-payment-integration).
- **An active Stripe account** with API access. You'll need your publishable and secret keys.
- **Comfort with REST APIs** and JSON data structures.
- **A database** to store customer and subscription data. We'll cover Supabase and Firebase options in the [backend guide](/articles/rork-dev/supabase-firebase-backend).

## Setting Up Stripe

Before building anything in Rork Max, you need to configure your products and pricing in Stripe.

### 1. Create Products and Plans

Start by defining what you're selling.

1. Log into your Stripe dashboard
2. Navigate to **Products** under **Catalog**
3. Click **Add product**
4. Enter a product name (e.g., "Pro Plan") and description
5. Click **Add price** and configure:
   - **Price**: e.g., $99/month
   - **Billing period**: Monthly
   - **Recurring**: Yes

Create multiple plans (Starter, Pro, Enterprise) by repeating this process.

### 2. Get Your API Keys

Rork Max needs your Stripe API credentials.

1. Go to **Developers** → **API Keys**
2. Copy your **Publishable key** and **Secret key**
3. Note the **Webhook signing secret** location—you'll need it later

## Configuring Rork Max

### 1. Connect Stripe

Link your Stripe account to Rork Max.

1. Open your Rork Max dashboard
2. Navigate to **Integrations** → **Payment Services**
3. Select **Stripe** and click **Connect**
4. Paste your Stripe secret key
5. Click **Confirm**

### 3. Build Webhook Handler

Stripe sends real-time events (payment success, subscription updates, cancellations). Your webhook endpoint listens for these.

```javascript
// Rork Max Webhook Handler
// Endpoint: /webhooks/stripe

exports.handler = async (event) => {
  const sig = event.headers['stripe-signature'];
  const body = event.body;

  // Verify the event came from Stripe
  const stripeEvent = stripe.webhooks.constructEvent(
    body,
    sig,
    process.env.STRIPE_WEBHOOK_SECRET
  );

  // Handle different event types
  switch (stripeEvent.type) {
    case 'customer.subscription.updated':
      const updatedSub = stripeEvent.data.object;
      await updateSubscriptionInDB(updatedSub);
      break;

    case 'customer.subscription.deleted':
      const deletedSub = stripeEvent.data.object;
      await cancelSubscriptionInDB(deletedSub);
      break;

    case 'invoice.payment_succeeded':
      const invoice = stripeEvent.data.object;
      await logPaymentSuccess(invoice);
      break;

    default:
      console.log(`Unhandled event type: ${stripeEvent.type}`);
  }

  return { statusCode: 200, body: JSON.stringify({ received: true }) };
};

// Expected output:
// On successful webhook: { statusCode: 200, body: '{"received":true}' }
```

Register the webhook endpoint in Stripe:

1. Go to **Developers** → **Webhooks**
2. Click **Add an endpoint**
3. Enter your Rork Max endpoint URL (e.g., `https://yourapp.rorklab.app/webhooks/stripe`)
4. Select events to listen for:
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.payment_succeeded`
   - `invoice.payment_failed`
5. Click **Add endpoint**

## Implementing Subscription Features

### 1. Creating Subscriptions

Implement the flow for users to start a subscription.

```javascript
// Rork Max Component
// Subscribe User to Plan

async function createSubscription(userId, planId, paymentMethodId) {
  try {
    // 1. Get or create Stripe Customer
    let customerId = await getCustomerIdFromDB(userId);

    if (!customerId) {
      const customer = await stripe.customers.create({
        email: user.email,
        metadata: { userId: userId }
      });
      customerId = customer.id;
      await saveCustomerIdToDB(userId, customerId);
    }

    // 2. Create subscription
    const subscription = await stripe.subscriptions.create({
      customer: customerId,
      items: [{ price: planId }],
      payment_method: paymentMethodId,
      default_payment_method: paymentMethodId,
      expand: ['latest_invoice.payment_intent']
    });

    // 3. Store in database
    await saveSubscriptionToDB({
      user_id: userId,
      stripe_subscription_id: subscription.id,
      stripe_customer_id: customerId,
      plan_id: planId,
      status: subscription.status,
      current_period_start: new Date(subscription.current_period_start * 1000),
      current_period_end: new Date(subscription.current_period_end * 1000)
    });

    // 4. Return success response
    return {
      success: true,
      subscription_id: subscription.id,
      status: subscription.status
    };

  } catch (error) {
    console.error('Subscription creation failed:', error.message);
    return {
      success: false,
      error: error.message
    };
  }
}

// Expected output:
// Success: { success: true, subscription_id: 'sub_xxxxx', status: 'active' }
// Error: { success: false, error: 'Error message' }
```

### 2. Changing and Canceling Plans

Let users upgrade, downgrade, or cancel their subscriptions.

```javascript
// Change plan
async function updateSubscriptionPlan(subscriptionId, newPriceId) {
  const subscription = await stripe.subscriptions.retrieve(subscriptionId);

  const updated = await stripe.subscriptions.update(subscriptionId, {
    items: [{
      id: subscription.items.data[0].id,
      price: newPriceId
    }],
    proration_behavior: 'create_prorations' // Enable prorating
  });

  return { success: true, subscription: updated };
}

// Cancel immediately
async function cancelSubscriptionImmediate(subscriptionId) {
  const cancelled = await stripe.subscriptions.del(subscriptionId);
  return { success: true, subscription: cancelled };
}

// Cancel at end of billing period
async function cancelSubscriptionAtPeriodEnd(subscriptionId) {
  const cancelled = await stripe.subscriptions.update(subscriptionId, {
    cancel_at_period_end: true
  });
  return { success: true, subscription: cancelled };
}
```

## Troubleshooting

### Common Issues

**Webhooks aren't being received**

- Verify the webhook URL is correctly registered in Stripe
- Check the **Events** log in Stripe's Webhooks section for errors
- Confirm your webhook secret is properly set in your environment variables

**Payment declined when creating subscription**

- Ensure the payment method is valid (use Stripe test cards in development)
- Check that customer billing details are complete
- Review Stripe's fraud detection settings—legitimate payments may be blocked

**Proration not calculating when changing plans**

- Verify `proration_behavior` is set to `create_prorations`
- Check that proration is enabled in your Stripe billing settings

## FAQ

**Q1: Are there additional licensing costs to implement subscriptions in Rork Max?**

A: Rork Max itself has no additional fees. You pay Stripe's standard processing fee (typically 2.9% + $0.30 per transaction for card payments). This is the standard cost of processing online payments.

**Q2: How do I support multiple currencies (USD, EUR, JPY, etc.)?**

A: Stripe supports multi-currency pricing. Create price variants for each currency in your products. In Rork Max, detect the user's location or currency preference and display the appropriate pricing option.

**Q3: What's the process for refunding customers or handling subscription disputes?**

A: Stripe provides refund functionality through the dashboard or API. Create a Rork Max function that calls the Stripe refund endpoint. Define a clear refund policy first—see our [solo dev guide](/articles/rork-business/solo-dev-app-business) for business best practices.

**Q4: Can I test subscriptions before going live?**

A: Yes. Stripe provides test mode with test card numbers. Use `pk_test_...` and `sk_test_...` keys for development, then switch to live keys (`pk_live_...` and `sk_live_...`) for production. Always test the full flow: subscription creation, renewal, cancellation, and refunds.
