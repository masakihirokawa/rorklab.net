# Rork Max State Management Guide

As Rork Max apps grow in complexity, choosing the right state management solution becomes critical. This guide compares three leading approaches—Zustand, Context API, and Redux Toolkit—with practical examples, performance tips, and decision-making frameworks.

## Comparison Matrix

| Feature | Context API | Zustand | Redux Toolkit |
|---------|-------------|---------|---------------|
| Learning Curve | Low | Low | Medium–High |
| Bundle Size | 0 KB | 3 KB | 15 KB |
| Performance | ⚠️ Careful | ✓ Optimized | ✓ Optimized |
| Complex State | △ Small–Medium | ✓ Recommended | ✓ Recommended |
| Boilerplate | Minimal | Minimal | Significant |
| DevTools | ✗ | ✓ | ✓ |
| Time-Travel Debugging | ✗ | ✓ | ✓ |
| Async Handling | Manual | Manual | Redux Thunk |
| TypeScript Support | ✓ | ✓ Excellent | ✓ |

<Callout type="info">
**Quick Decision Guide**:
- **Small apps (<10 screens)**: Context API
- **Medium to large apps**: Zustand
- **Large apps with complex async logic**: Redux Toolkit
</Callout>

## Zustand — Minimal & Powerful

Zustand provides powerful state management with minimal boilerplate and excellent TypeScript support.

### Installation

```bash
npm install zustand
npm install --save-dev @react-native-async-storage/async-storage
```

### Basic Store

**src/store/userStore.ts**
```typescript
import { create } from 'zustand';
import { devtools, persist } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface User {
  id: string;
  name: string;
  email: string;
  avatar?: string;
}

interface UserState {
  user: User | null;
  isLoading: boolean;
  error: string | null;
  setUser: (user: User | null) => void;
  updateUser: (updates: Partial<User>) => void;
  logout: () => void;
  fetchUser: (userId: string) => Promise<void>;
}

export const useUserStore = create<UserState>()(
  devtools(
    persist(
      (set, get) => ({
        user: null,
        isLoading: false,
        error: null,

        setUser: (user) => {
          set({ user, error: null });
        },

        updateUser: (updates) => {
          set((state) => ({
            user: state.user ? { ...state.user, ...updates } : null,
          }));
        },

        logout: () => {
          set({ user: null, error: null });
        },

        fetchUser: async (userId) => {
          set({ isLoading: true });
          try {
            const response = await fetch(`/api/users/${userId}`);
            const data = await response.json();
            set({ user: data, error: null, isLoading: false });
          } catch (error) {
            set({
              error: (error as Error).message,
              isLoading: false,
            });
          }
        },
      }),
      {
        name: 'user-store',
        storage: {
          getItem: async (name) => {
            const data = await AsyncStorage.getItem(name);
            return data ? JSON.parse(data) : null;
          },
          setItem: async (name, value) => {
            await AsyncStorage.setItem(name, JSON.stringify(value));
          },
          removeItem: async (name) => {
            await AsyncStorage.removeItem(name);
          },
        },
      }
    )
  )
);
```

```typescript
        <Text style={styles.logoutText}>Logout</Text>
      </TouchableOpacity>
    </View>
  );
}
```

### Performance Optimization — Selectors Matter

<Callout type="warning">
**Critical**: Always use selector functions. Without them, the component re-renders whenever any state changes, regardless of relevance.
</Callout>

```typescript
// ❌ Bad: Subscribes to entire store
const fullState = useUserStore();

// ✓ Good: Select only what you need
const user = useUserStore((state) => state.user);
const isLoading = useUserStore((state) => state.isLoading);

// ✓ Memoize object selectors to prevent re-renders
const userData = useUserStore(
  (state) => ({
    name: state.user?.name,
    email: state.user?.email,
  }),
  (a, b) => a?.name === b?.name && a?.email === b?.email
);
```

## React Context API — Built-in Solution

For small to medium apps, Context API is sufficient and requires no external dependencies.

**src/contexts/ThemeContext.tsx**
```typescript
import React, {
  createContext,
  useContext,
  useState,
  useCallback,
  useMemo,
} from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

type Theme = 'light' | 'dark' | 'auto';

interface ThemeContextType {
  theme: Theme;
  setTheme: (theme: Theme) => Promise<void>;
  isDark: boolean;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setThemeState] = useState<Theme>('auto');
  const [isDark, setIsDark] = useState(false);

  const setTheme = useCallback(async (newTheme: Theme) => {
    setThemeState(newTheme);
    await AsyncStorage.setItem('app_theme', newTheme);

    if (newTheme === 'dark') {
      setIsDark(true);
    } else if (newTheme === 'light') {
      setIsDark(false);
    }
    // 'auto': Handle device settings (use Appearance API)
  }, []);

  // ✓ Memoize to prevent unnecessary provider re-renders
  const value = useMemo(
    () => ({ theme, setTheme, isDark }),
    [theme, setTheme, isDark]
  );

  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  );
}

export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within ThemeProvider');
  }
  return context;
}
```

**src/App.tsx**

### Solving Context API Performance Issues

```typescript
// ❌ Problem: All consumers re-render when any value changes
const value = { user, setUser, isLoading };
return (
  <UserContext.Provider value={value}>
    {children}
  </UserContext.Provider>
);

// ✓ Solution: Memoize the value
const value = useMemo(
  () => ({ user, setUser, isLoading }),
  [user, isLoading]
);
return (
  <UserContext.Provider value={value}>
    {children}
  </UserContext.Provider>
);

// ✓ Alternative: Split into multiple contexts
return (
  <UserStateContext.Provider value={user}>
    <UserActionsContext.Provider value={{ setUser }}>
      {children}
    </UserActionsContext.Provider>
  </UserStateContext.Provider>
);
```

## Redux Toolkit — Enterprise-Grade

For large, complex apps with sophisticated async workflows, Redux Toolkit provides structure and powerful debugging tools.

### Create a Slice

**src/store/slices/authSlice.ts**
```typescript
import { createSlice, createAsyncThunk, PayloadAction } from '@reduxjs/toolkit';

interface AuthState {
  token: string | null;
  user: { id: string; name: string } | null;
  status: 'idle' | 'loading' | 'succeeded' | 'failed';
  error: string | null;
}

const initialState: AuthState = {
  token: null,
  user: null,
  status: 'idle',
  error: null,
};

export const loginUser = createAsyncThunk(
  'auth/loginUser',
  async (
    { email, password }: { email: string; password: string },
    { rejectWithValue }
  ) => {
    try {
      const response = await fetch('/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email, password }),
      });

      if (!response.ok) {
        return rejectWithValue('Login failed');
      }

      const data = await response.json();
      return { token: data.token, user: data.user };
    } catch (error) {
      return rejectWithValue((error as Error).message);
    }
  }
);

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    logout: (state) => {
      state.token = null;
      state.user = null;
    },
  },
  extraReducers: (builder) => {
    builder
      .addCase(loginUser.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(loginUser.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.token = action.payload.token;
        state.user = action.payload.user;
      })
      .addCase(loginUser.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.payload as string;
      });
  },
});

export const { logout } = authSlice.actions;
export default authSlice.reducer;
```

### Use in Components

```typescript
import React, { useState } from 'react';
import {
  View,
  TextInput,
  TouchableOpacity,
  Text,
  ActivityIndicator,
  StyleSheet,

import { RootState, AppDispatch } from '../store';

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20, justifyContent: 'center' },
  input: {
    borderWidth: 1,
    borderColor: '#ccc',
    padding: 12,
    marginBottom: 12,
    borderRadius: 8,
  },
  button: {
    backgroundColor: '#007AFF',
    padding: 12,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonDisabled: {
    backgroundColor: '#ccc',
  },
  buttonText: { color: '#fff', fontSize: 16, fontWeight: 'bold' },
  errorText: { color: '#FF3B30', marginTop: 16, textAlign: 'center' },

  const dispatch = useDispatch<AppDispatch>();
  const { status, error } = useSelector((state: RootState) => state.auth);

    await dispatch(loginUser({ email, password }));
  };

      <TextInput
        placeholder="Email"
        value={email}
        onChangeText={setEmail}

        placeholder="Password"
        value={password}
        onChangeText={setPassword}

        editable={!isLoading}
        style={styles.input}
      />

      <TouchableOpacity
        onPress={handleLogin}
        disabled={isLoading}
        style={[styles.button, isLoading && styles.buttonDisabled]}
      >
        {isLoading ? (
          <ActivityIndicator color="#fff" />
        ) : (
          <Text style={styles.buttonText}>Login</Text>

  );
}
```

## Persistence — AsyncStorage vs MMKV

Persist state across app launches for seamless session recovery.

### Zustand + AsyncStorage

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

export const useAuthStore = create(
  persist(
    (set) => ({
      token: null,
      setToken: (token) => set({ token }),
    }),
    {
      name: 'auth-storage',
      storage: {
        getItem: async (name) => {
          const item = await AsyncStorage.getItem(name);
          return item ? JSON.parse(item) : null;
        },
        setItem: async (name, value) => {
          await AsyncStorage.setItem(name, JSON.stringify(value));
        },
        removeItem: async (name) => {
          await AsyncStorage.removeItem(name);
        },
      },
    }
  )
);
```

### Use MMKV for Performance

```bash
npm install react-native-mmkv
```

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import { MMKV } from 'react-native-mmkv';

const storage = new MMKV();

export const useAuthStore = create(
  persist(
    (set) => ({
      token: null,
      setToken: (token) => set({ token }),
    }),
    {
      name: 'auth-storage',
      storage: {
        getItem: (name) => {
          const value = storage.getString(name);
          return value ? JSON.parse(value) : null;
        },
        setItem: (name, value) => {
          storage.set(name, JSON.stringify(value));
        },
        removeItem: (name) => {
          storage.delete(name);
        },
      },
    }
  )
);
```

<Callout type="info">
**MMKV Benefits**: 100x faster than AsyncStorage. Ideal for high-frequency writes and large datasets. Use it for frequently updated state like user preferences and cache data.
</Callout>

## Decision Matrix: When to Use What

| Scenario | Best Choice | Reason |
|----------|------------|--------|
| Simple theme toggle | Context API | Zero dependencies |
| Global user state | Zustand | Minimal setup |
| Shopping cart | Zustand or Redux | Persists across sessions |
| Complex multi-step form | Zustand | Flexible actions |
| Real-time notifications | Redux | Thunk async handling |
| Enterprise app | Redux | DevTools, structure |

## Best Practices

### 1. Separate State Logic
```typescript
// ✓ Split stores by domain
const useAuthStore = create(...);  // Authentication
const useCartStore = create(...);  // Shopping
const useUIStore = create(...);    // UI state
```

### 2. Avoid Over-Nesting
```typescript
// ❌ Too deep
const state = useStore((s) => s.auth.user.profile.settings.theme);

// ✓ Flatten or use separate selectors
const theme = useUIStore((s) => s.theme);
```

### 3. Normalize Data
```typescript
// ✓ Normalize, don't nest
{
  users: { '1': { id: '1', name: 'Alice' } },
  posts: { 'p1': { id: 'p1', userId: '1' } },
}

// Instead of:
{
  users: [{ id: '1', name: 'Alice', posts: [...] }]
}
```

## Summary

- **Context API**: Perfect for small apps, no setup overhead
- **Zustand**: Sweet spot for most apps—minimal boilerplate, great TypeScript
- **Redux**: Enterprise solutions with complex workflows and debugging needs

Choose based on app complexity, team expertise, and debugging requirements.
