One of the most underrated advantages of Rork Max is its ability to tap into native Apple frameworks that React Native-based tools simply cannot access. **PassKit** — the framework powering Apple Wallet — is a prime example. Whether you're building a coffee shop loyalty card, an event ticketing app, or a digital boarding pass system, Rork Max can generate the SwiftUI code to make it happen. ## What Is PassKit and Apple Wallet?

**PassKit** is Apple's framework for adding and managing passes (digital cards) in the native Wallet app on iPhone. It enables a wide range of real-world use cases for iOS app developers.

### What You Can Build with PassKit

- **Store cards / Loyalty cards** — Digital stamp cards and points programs
- **Boarding passes** — Airline, train, and bus e-tickets
- **Event tickets** — Concert, sports, and cinema tickets
- **Coupons** — Promotional codes and time-limited discounts
- **Generic passes** — Gym memberships, subscription IDs, and more

### Why Use Rork Max for PassKit?

Standard Rork (React Native / Expo) has limited PassKit support. Rork Max generates native SwiftUI code, giving you complete access to the PassKit framework:

- ✅ Native `PKAddPassesViewController` for adding passes to Wallet
- ✅ Embedded barcodes and QR codes in passes
- ✅ Real-time pass updates via push notifications
- ✅ Combine with Apple Pay for full checkout-to-wallet flows

## Step 3: Implement PassKit in Your SwiftUI App

Rork Max generates the SwiftUI scaffolding. Here's how to prompt it for the key PassKit components.

### Prompt Rork Max for PassKitManager

```
Create a PassKitManager class with:
- canAddPasses() to check if Wallet passes are supported
- addPass(url: URL) to fetch and add a pass from a URL
- A SwiftUI View that shows PKAddPassButton
- Alerts for success and failure states
```

### Generated SwiftUI Code Example

```swift
import SwiftUI
import PassKit

// MARK: - PassKit Manager
class PassKitManager: NSObject, ObservableObject {
    @Published var isAddingPass = false
    @Published var showSuccessAlert = false
    @Published var showErrorAlert = false
    @Published var errorMessage = ""

    /// Check whether this device supports adding Wallet passes
    /// - Returns: true if PassKit is available
    func canAddPasses() -> Bool {
        return PKAddPassesViewController.canAddPasses()
    }

    /// Fetch a pass from a URL and add it to Wallet
    /// - Parameter url: URL pointing to a .pkpass file
    func addPass(from url: URL) async {
        do {
            let data = try Data(contentsOf: url)
            let pass = try PKPass(data: data)

            await MainActor.run {
                self.presentAddPassController(for: pass)
            }
        } catch {
            await MainActor.run {
                self.errorMessage = "Failed to load pass: \(error.localizedDescription)"
                self.showErrorAlert = true
            }
        }
    }

    /// Present the native Add to Wallet sheet
    private func presentAddPassController(for pass: PKPass) {
        guard let addVC = PKAddPassesViewController(pass: pass),
              let rootVC = UIApplication.shared.connectedScenes
                .compactMap({ $0 as? UIWindowScene })
                .first?.windows.first?.rootViewController else { return }

        isAddingPass = true
        rootVC.present(addVC, animated: true)
    }
}

// MARK: - Wallet Add Button View
struct WalletAddButton: View {
    @StateObject private var passKitManager = PassKitManager()
    let passURL: URL

    var body: some View {
        if passKitManager.canAddPasses() {
            Button(action: {
                Task {
                    await passKitManager.addPass(from: passURL)
                }
            }) {
                // Official Apple "Add to Wallet" button design
                Image("PKAddToWalletButton")
                    .resizable()
                    .aspectRatio(contentMode: .fit)
                    .frame(height: 44)
            }
            .alert("Added!", isPresented: $passKitManager.showSuccessAlert) {
                Button("OK") {}
            } message: {
                Text("Your card has been added to Wallet.")
            }
            .alert("Error", isPresented: $passKitManager.showErrorAlert) {
                Button("OK") {}
            } message: {
                Text(passKitManager.errorMessage)
            }
        } else {
            Text("Wallet passes are not supported on this device.")
                .foregroundColor(.secondary)
                .font(.caption)
        }
    }
}
// Expected output: Apple-style "Add to Wallet" button rendered in your SwiftUI view
```

## Step 4: Server-Side Pass Signing and Delivery

`.pkpass` files must be **cryptographically signed** with your Apple Developer certificate before they can be added to Wallet.

### Delivery Architecture: Cloudflare Workers + Supabase

A Rork-friendly backend stack for pass delivery:

```
iOS App (Rork Max)
    ↓  Send user ID
Cloudflare Workers (API endpoint)
    ↓  Fetch user data
Supabase (membership database)
    ↓  Generate + sign pass.json in Node.js
Signed .pkpass file
    ↓  Return Base64-encoded
App adds pass to Wallet
```

### Node.js Pass Signing Code

```javascript
// Cloudflare Worker / Node.js — pass signing pipeline
const { execSync } = require('child_process');
const fs = require('fs');
const path = require('path');
const archiver = require('archiver');

async function createSignedPass(passData, userId) {
  const tempDir = `/tmp/pass_${userId}`;
  fs.mkdirSync(tempDir, { recursive: true });

  // Write pass.json
  fs.writeFileSync(
    path.join(tempDir, 'pass.json'),
    JSON.stringify({ ...passData, serialNumber: userId })
  );

  // Copy image assets
  ['icon.png', 'icon@2x.png', 'logo.png', 'logo@2x.png'].forEach(img => {
    fs.copyFileSync(`./assets/${img}`, path.join(tempDir, img));
  });

  // Generate manifest.json with SHA1 hashes
  const manifest = {};
  for (const file of fs.readdirSync(tempDir)) {
    const hash = execSync(`sha1sum ${path.join(tempDir, file)}`).toString().split(' ')[0];
    manifest[file] = hash;
  }
  fs.writeFileSync(path.join(tempDir, 'manifest.json'), JSON.stringify(manifest));

  // Sign with OpenSSL
  execSync(`openssl smime -binary -sign \
    -certfile ./certs/AppleWWDRCA.pem \
    -signer ./certs/passcertificate.pem \
    -inkey ./certs/passkey.pem \
    -in ${path.join(tempDir, 'manifest.json')} \
    -out ${path.join(tempDir, 'signature')} \
    -outform DER`);

  // Package as .pkpass ZIP
  const output = fs.createWriteStream(`/tmp/${userId}.pkpass`);
  const archive = archiver('zip', { zlib: { level: 9 } });
  archive.pipe(output);
  archive.directory(tempDir, false);
  await archive.finalize();

  return fs.readFileSync(`/tmp/${userId}.pkpass`);
}
// Expected output: A signed .pkpass file buffer ready to serve to the iOS app
```

## Step 5: Real-Time Pass Updates

PassKit supports push-based pass updates, so you can refresh a loyalty card's stamp count or a ticket's gate number without the user opening the app.

### Update Flow

1. User adds a stamp in the app
2. Backend sends an APNs push notification to the device
3. iOS queries your pass update API for new data
4. Server returns a freshly signed pass
5. Wallet card updates automatically with animation

### Required PassKit Update API Endpoints

```
GET  /passes/{passTypeIdentifier}/{serialNumber}
     → Return latest pass data

POST /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}/{serialNumber}
     → Register device for updates

DELETE /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}/{serialNumber}
     → Unregister device

GET  /devices/{deviceLibraryIdentifier}/registrations/{passTypeIdentifier}
     → Return list of serial numbers needing updates
```

## Common Errors and Fixes

### Error 1: "Cannot Add Passes to Wallet"

**Cause:** Invalid pass signature or expired certificate.

**Fix:**
1. Check certificate validity in Apple Developer Portal
2. Run `openssl verify` to validate the signature
3. Confirm that `teamIdentifier` matches your actual Team ID

### Error 2: PKAddPassesViewController Does Not Appear

**Cause:** PassKit has limited functionality in the iOS Simulator.

**Fix:** Test on a physical iPhone. Use Rork Max's 1-Click Install feature to sideload directly to your device.

### Error 3: Barcode Is Not Scannable

**Cause:** Incorrect `messageEncoding` or barcode data is too long.

**Fix:** Use `iso-8859-1` for `messageEncoding`. For QR codes, keep the payload under 2,953 characters for reliable scanning.

## FAQ

**Q1. Is an Apple Developer account required to implement PassKit?**

Yes. You need an Apple Developer Program membership ($99/year) to create a Pass Type ID and obtain a signing certificate. You can still test on real devices via TestFlight without a full App Store submission.

**Q2. Do I need to reissue a new pass every time data changes?**

No. If you implement the PassKit update API endpoints on your server, iOS will silently update existing passes when you push an APNs notification. Users see the card update automatically in their Wallet — no action required on their end.

**Q3. Which Rork Max plan supports PassKit implementation?**

Rork Max generates PassKit-compatible SwiftUI code on all plans. However, to distribute the app via TestFlight or submit to the App Store, you'll need a paid Rork Max subscription.

**Q4. Can I also create Google Wallet passes for Android?**

Yes, using standard Rork (React Native) with the Google Pay API for Passes. This requires a separate Google Cloud project and the `@google-pay/button-react-native` library. The server-side signing flow is similar but uses JWTs instead of OpenSSL.
