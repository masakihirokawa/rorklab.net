## HealthKit Fundamentals

### What Is HealthKit?

HealthKit is Apple's centralized health data platform. It connects the iPhone Health app with Apple Watch and third-party apps, acting as a single source of truth for personal health data.

HealthKit data falls into three main types.

- **HKQuantityType** (quantitative): Steps, heart rate, calories — anything expressible as a number
- **HKCategoryType** (categorical): Sleep analysis, menstrual cycle — data with discrete states
- **HKCorrelationType** (correlated): Blood pressure, which combines multiple quantities (systolic + diastolic)

### The Permission Model

HealthKit manages read and write permissions separately. Users can grant or deny access for each individual data type. Critically, **your app cannot detect whether unauthorized data exists** — HealthKit returns no indication that the data is there at all.

When transmitting HealthKit data to any third-party service, you must always obtain explicit informed consent from the user.

## Setting Up HealthKit Permissions in Rork Max

### Info.plist Configuration

Add the following to your `app.json` (or `app.config.js`):

```json
{
  "expo": {
    "ios": {
      "infoPlist": {
        "NSHealthShareUsageDescription": "We read your steps, sleep, and heart rate to help you track and visualize your daily health.",
        "NSHealthUpdateUsageDescription": "We write workout and body weight data back to Apple Health on your behalf."
      },
      "entitlements": {
        "com.apple.developer.healthkit": true,
        "com.apple.developer.healthkit.access": []
      }
    }
  }
}
```

### Initializing react-native-health

Create a utility module to manage the HealthKit store connection:

```typescript
// src/lib/healthKit.ts
import AppleHealthKit, {
  HealthKitPermissions,
  HealthValue,
} from 'react-native-health';
import { Platform } from 'react-native';

const PERMISSIONS: HealthKitPermissions = {
  permissions: {
    read: [
      AppleHealthKit.Constants.Permissions.Steps,
      AppleHealthKit.Constants.Permissions.HeartRate,
      AppleHealthKit.Constants.Permissions.SleepAnalysis,
      AppleHealthKit.Constants.Permissions.ActiveEnergyBurned,
      AppleHealthKit.Constants.Permissions.DistanceWalkingRunning,
      AppleHealthKit.Constants.Permissions.FlightsClimbed,
      AppleHealthKit.Constants.Permissions.Weight,
    ],
    write: [
      AppleHealthKit.Constants.Permissions.Weight,
      AppleHealthKit.Constants.Permissions.ActiveEnergyBurned,
    ],
  },
};

/**
 * Initialize HealthKit and request permissions.
 * @returns true on success
 */
export const initHealthKit = (): Promise<boolean> => {
  return new Promise((resolve, reject) => {
    if (Platform.OS !== 'ios') {
      // HealthKit is iOS-only (use Google Health Connect for Android)
      reject(new Error('HealthKit is only available on iOS'));
      return;
    }

    AppleHealthKit.initHealthKit(PERMISSIONS, (error: string) => {
      if (error) {
        reject(new Error(error));
        return;
      }
      resolve(true);
    });
  });
};
```

## Fetching and Displaying Step Count Data

### Today's Step Count

```typescript
// src/hooks/useStepCount.ts
import { useState, useEffect, useCallback } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';

interface StepData {
  today: number;
  weekly: { date: string; count: number }[];
  isLoading: boolean;
  error: string | null;
}

export const useStepCount = () => {
  const [data, setData] = useState<StepData>({
    today: 0,
    weekly: [],
    isLoading: true,
    error: null,
  });

  const fetchTodaySteps = useCallback((): Promise<number> => {
    return new Promise((resolve, reject) => {
      const options = {
        date: new Date().toISOString(),
        includeManuallyAdded: true,
      };

      AppleHealthKit.getStepCount(options, (err: string, result: HealthValue) => {
        if (err) {
          reject(new Error(err));
          return;
        }
        // result.value contains today's step count (e.g., 8432)
        resolve(Math.round(result.value));
      });
    });
  }, []);

  const fetchWeeklySteps = useCallback((): Promise<{ date: string; count: number }[]> => {
    return new Promise((resolve, reject) => {
      const startDate = new Date();
      startDate.setDate(startDate.getDate() - 6); // 7 days ago
      startDate.setHours(0, 0, 0, 0);

      const options = {
        startDate: startDate.toISOString(),
        endDate: new Date().toISOString(),
        includeManuallyAdded: true,
      };

      AppleHealthKit.getDailyStepCountSamples(
        options,
        (err: string, results: HealthValue[]) => {
          if (err) {
            reject(new Error(err));
            return;
          }

          const formatted = results.map((item) => ({
            date: new Date(item.startDate).toLocaleDateString('en-US', {
              month: 'numeric',
              day: 'numeric',
            }),
            count: Math.round(item.value),
          }));

          resolve(formatted);
        }
      );
    });
  }, []);

  useEffect(() => {
    const load = async () => {
      try {
        const [today, weekly] = await Promise.all([
          fetchTodaySteps(),
          fetchWeeklySteps(),
        ]);
        setData({ today, weekly, isLoading: false, error: null });
      } catch (e) {
        setData((prev) => ({
          ...prev,
          isLoading: false,
          error: e instanceof Error ? e.message : 'Unknown error',
        }));
      }
    };

    load();
  }, [fetchTodaySteps, fetchWeeklySteps]);

  return data;
};
```

## Fetching Sleep and Heart Rate Data

### Sleep Analysis

HealthKit sleep data uses `HKCategoryValueSleepAnalysis` with the following states:

- `INBED` (0): In bed but not necessarily sleeping
- `ASLEEP` (1): Asleep (generic, without Apple Watch)
- `AWAKE` (2): Awake
- `CORE` (3): Core sleep
- `DEEP` (4): Deep sleep
- `REM` (5): REM sleep

```typescript
// src/hooks/useSleepData.ts
import { useState, useEffect } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';
import { startOfDay, endOfDay, subDays, differenceInMinutes } from 'date-fns';

interface SleepSession {
  date: string;
  totalMinutes: number;
  deepMinutes: number;
  remMinutes: number;
  coreMinutes: number;
}

export const useSleepData = (days: number = 7) => {
  const [sessions, setSessions] = useState<SleepSession[]>([]);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    const startDate = startOfDay(subDays(new Date(), days - 1)).toISOString();
    const endDate = endOfDay(new Date()).toISOString();

    AppleHealthKit.getSleepSamples(
      { startDate, endDate },
      (err: string, results: HealthValue[]) => {
        if (err) {
          console.error('Sleep data error:', err);
          setIsLoading(false);
          return;
        }

        const dailyMap: Record<string, SleepSession> = {};

        results.forEach((sample) => {
          const dateKey = new Date(sample.startDate).toLocaleDateString('en-US');
          const durationMinutes = differenceInMinutes(
            new Date(sample.endDate),
            new Date(sample.startDate)
          );

          if (!dailyMap[dateKey]) {
            dailyMap[dateKey] = {
              date: dateKey,
              totalMinutes: 0,
              deepMinutes: 0,
              remMinutes: 0,
              coreMinutes: 0,
            };
          }

          switch (sample.value) {
            case 'DEEP':
              dailyMap[dateKey].deepMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
            case 'REM':
              dailyMap[dateKey].remMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
            case 'CORE':
            case 'ASLEEP':
              dailyMap[dateKey].coreMinutes += durationMinutes;
              dailyMap[dateKey].totalMinutes += durationMinutes;
              break;
          }
        });

        setSessions(Object.values(dailyMap).sort((a, b) => a.date.localeCompare(b.date)));
        setIsLoading(false);
      }
    );
  }, [days]);

  return { sessions, isLoading };
};
```

### Real-Time Heart Rate

```typescript
// src/hooks/useHeartRate.ts
import { useState, useEffect } from 'react';
import AppleHealthKit, { HealthValue } from 'react-native-health';
import { subHours } from 'date-fns';

interface HeartRateData {
  current: number | null;
  resting: number | null;
  samples: { time: string; bpm: number }[];
}

export const useHeartRate = () => {
  const [data, setData] = useState<HeartRateData>({
    current: null,
    resting: null,
    samples: [],
  });

  useEffect(() => {
    const now = new Date();
    const sixHoursAgo = subHours(now, 6);

    AppleHealthKit.getHeartRateSamples(
      {
        startDate: sixHoursAgo.toISOString(),
        endDate: now.toISOString(),
        limit: 100,
        ascending: true,
      },
      (err: string, results: HealthValue[]) => {
        if (err || !results.length) return;

        const samples = results.map((r) => ({
          time: new Date(r.startDate).toLocaleTimeString('en-US', {
            hour: '2-digit',
            minute: '2-digit',
          }),
          bpm: Math.round(r.value),
        }));

        const current = samples[samples.length - 1]?.bpm ?? null;
        setData((prev) => ({ ...prev, current, samples }));
      }
    );

    AppleHealthKit.getRestingHeartRate(
      { date: now.toISOString() },
      (err: string, result: HealthValue) => {
        if (err || !result) return;
        setData((prev) => ({ ...prev, resting: Math.round(result.value) }));
      }
    );
  }, []);

  return data;
};
```

## Background Delivery and Widget Integration

### Enabling HealthKit Background Delivery

HealthKit can wake your app in the background whenever new data is recorded, keeping your stats up-to-date without the user opening the app.

```typescript
// src/lib/healthKitBackground.ts
import AppleHealthKit from 'react-native-health';

/**
 * Enable background delivery — call once on app launch (in App.tsx useEffect)
 */
export const enableBackgroundDelivery = () => {
  // Step count updates hourly
  AppleHealthKit.enableBackgroundDelivery(
    AppleHealthKit.Constants.Permissions.Steps,
    AppleHealthKit.Constants.QueryBundleQuery.HOUR,
    (err: string, result: boolean) => {
      if (err) {
        console.error('Background delivery setup failed:', err);
        return;
      }
      console.log('Step background delivery enabled:', result);
    }
  );

  // Heart rate updates immediately
  AppleHealthKit.enableBackgroundDelivery(
    AppleHealthKit.Constants.Permissions.HeartRate,
    AppleHealthKit.Constants.QueryBundleQuery.IMMEDIATE,
    (err: string, result: boolean) => {
      if (err) return;
      console.log('Heart rate background delivery enabled:', result);
    }
  );
};
```

### Sharing Data with iOS Widgets (App Groups)

To share data between your app and its widget extension, use **App Groups** with the `react-native-shared-group-preferences` library.

```bash
npm install react-native-shared-group-preferences
```

```typescript
// src/lib/widgetDataSync.ts
import SharedGroupPreferences from 'react-native-shared-group-preferences';

const APP_GROUP_ID = 'group.com.yourdomain.healthapp';

/**
 * Write health metrics to Shared Container for widget display
 */
export const syncDataToWidget = async (data: {
  steps: number;
  heartRate: number | null;
  sleepHours: number;
}) => {
  try {
    await SharedGroupPreferences.setItem(
      'healthWidgetData',
      JSON.stringify({ ...data, updatedAt: new Date().toISOString() }),
      APP_GROUP_ID
    );
  } catch (e) {
    console.error('Widget sync failed:', e);
  }
};
```

## Privacy Design and App Store Review Strategy

### HealthKit Usage Policies You Must Follow

Apple enforces strict HealthKit guidelines. For a broader look at privacy architecture in Rork apps, see [The Complete Guide to Privacy and Data Protection in Rork Apps](/articles/rork-dev/rork-app-privacy-data-protection-guide). These rules are non-negotiable.

**Absolutely prohibited:**
- Using HealthKit data for advertising or third-party marketing
- Selling or sharing HealthKit data with data brokers
- Combining HealthKit data with other identifiers to build user profiles

**Required:**
- Clearly state HealthKit data usage in your privacy policy
- Obtain explicit user consent before transmitting data to any server
- Process data on-device whenever possible; minimize server-side storage

### App Store Review Checklist

```markdown
## HealthKit App Review Checklist

### Must-Have
- [ ] NSHealthShareUsageDescription is specific (vague descriptions = rejection)
- [ ] NSHealthUpdateUsageDescription is specific (only if writing data)
- [ ] Privacy policy URL is registered in App Store Connect
- [ ] Privacy policy explicitly covers HealthKit data usage
- [ ] No HealthKit data is used for advertising
- [ ] com.apple.developer.healthkit entitlement is enabled in Xcode

### Recommended
- [ ] Show an in-app explanation screen before requesting HealthKit permissions
- [ ] Provide a fallback UI when permission is denied
- [ ] Mock data is excluded from production builds
```

## Common Errors and Fixes

### Error 1: Step count always returns 0 on simulator

**Cause**: The iOS simulator has no real HealthKit data.

**Fix**: Insert mock data in the simulator via Xcode (Features → HealthKit Data), or test exclusively on a physical device. In dev mode, you can insert dummy data programmatically:

```typescript
// Dev-only mock data insertion
if (__DEV__ && Platform.OS === 'ios') {
  AppleHealthKit.saveSteps(
    { value: 7523, startDate: new Date().toISOString(), endDate: new Date().toISOString() },
    (err) => { if (err) console.error(err); }
  );
}
```

### Error 2: Build fails after pod install

**Symptom**: `Missing required Modules: react-native-health`

**Fix**:

```bash
# Verify your ios/Podfile includes:
pod 'RNAppleHealthKit', :path => '../node_modules/react-native-health'

# Clear and reinstall
cd ios && pod deintegrate && pod install
```

### Error 3: Permission dialog doesn't appear after denial

**Cause**: iOS doesn't allow re-requesting permissions once the user has denied them through the system dialog — this is by design.

**Fix**: Guide the user to Settings manually.

```typescript
import { Linking } from 'react-native';

const openHealthSettings = () => {
  Linking.openURL('App-Prefs:HEALTH');
};
```

### Error 4: Background delivery isn't firing

**Check**:
- Is **Background Modes → Background fetch** enabled in your Xcode target capabilities?
- Is the `com.apple.developer.healthkit.background-delivery` entitlement enabled?

## FAQ

**Q: Does HealthKit work on Android?**

HealthKit is iOS/watchOS only. For Android, use **Google Health Connect** via the `react-native-health-connect` library. For a cross-platform app, branch on `Platform.OS` and implement both. A practical approach is to launch iOS first, validate demand, then add Android support based on user feedback.

**Q: Can I access Apple Watch data from my iPhone app?**

Yes. Data recorded by Apple Watch — detailed sleep stages, heart rate samples, VO2 Max, and more — flows into HealthKit and is accessible from your iPhone app. You don't need a separate watchOS app to read Watch data. A Watch companion app is only needed if you want to control Watch-specific interactions (like the Digital Crown).

**Q: Is it okay to store HealthKit data on my server?**

Apple permits server-side storage for purposes like user backup and sync, provided you have the user's explicit consent. What's strictly prohibited is using HealthKit data for advertising, selling it to third parties, or sharing it without consent. Encrypt all data in transit (HTTPS) and be explicit about server storage in your privacy policy.

**Q: What's the most common reason HealthKit apps get rejected?**

The leading cause of rejection is a vague `NSHealthShareUsageDescription`. Strings like "We use your health data" get rejected consistently. Be specific: "We read your step count, sleep analysis, and heart rate to display daily wellness summaries and track progress toward your fitness goals." Specificity is what passes review.

**Q: How do I test HealthKit in CI/CD environments without a physical device?**

For unit tests, mock the HealthKit layer entirely using Jest (e.g., `jest-mock-react-native-health`). Isolate your hooks and business logic from direct HealthKit calls behind an abstraction layer, so the core logic is testable without hardware. Reserve physical device tests for integration and E2E suites in your manual QA phase.
