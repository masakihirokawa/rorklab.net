At some point in every app's lifecycle, you need to send emails — a welcome message when someone signs up, a password reset link, a purchase confirmation, a weekly digest. These touchpoints build trust and bring users back.

If you're building with Rork and looking for the cleanest way to add email to your stack, **Resend** is the answer. Launched in 2023 and beloved by developers, it offers a minimal setup, a generous free tier, and first-class support for React Email templates.

This guide walks you through integrating Resend into a Rork app (using Supabase as the backend), so you can send production-quality emails without wrestling with legacy email infrastructure.

---

## What Is Resend — and Why Choose It?

Resend is an email API built for developers. Here's how it compares to older options like Amazon SES or SendGrid:

- **Fast setup**: From account creation to your first sent email in under 10 minutes. Configure DNS once, then it's just API keys and HTTP requests.
- **React Email integration**: Resend pairs natively with [React Email](https://react.email/), letting you write email templates in JSX — no more wrestling with HTML email quirks.
- **Generous free tier**: 3,000 emails/month and 100/day at no cost. Plenty for indie apps in early stages.
- **High deliverability**: SPF, DKIM, and DMARC configuration is guided and straightforward, reducing the chance of landing in spam folders.

## Step 1: Set Up Your Resend Account

Head to [resend.com](https://resend.com/) and sign up with GitHub or Google.

Once you're in the dashboard, you need to do two things:

**① Add and verify your domain**

Under "Domains," add the domain you want to send from. If you don't have a custom domain yet, you can use `onboarding@resend.dev` during development — just make sure to switch to a real domain before going to production.

Adding the DNS records (provided by Resend) usually takes 10–30 minutes to propagate depending on your DNS provider (Cloudflare, GoDaddy, etc.).

**② Get an API key**

Under "API Keys," create a new key with "Sending Access" scope. Copy it immediately — you won't see it again.

```bash
# Save to .env.local (and add to .gitignore)
RESEND_API_KEY=YOUR_RESEND_API_KEY
```

## Step 2: Create a Supabase Edge Function to Send Emails

In Rork apps, email sending should happen server-side, not from the mobile client. Keeping your API key on the server prevents it from being exposed in the app bundle.

**Create the Edge Function**

In the Supabase dashboard, go to "Edge Functions" and create a new function called `send-email`.

```typescript
// supabase/functions/send-email/index.ts
import { serve } from "https://deno.land/std@0.208.0/http/server.ts";

const RESEND_API_KEY = Deno.env.get("RESEND_API_KEY")!;

interface EmailRequest {
  to: string;
  subject: string;
  html: string;
  from?: string;
}

serve(async (req: Request) => {
  // Handle CORS preflight for requests from Rork app
  if (req.method === "OPTIONS") {
    return new Response(null, {
      headers: {
        "Access-Control-Allow-Origin": "*",
        "Access-Control-Allow-Headers": "authorization, x-client-info, apikey, content-type",
      },
    });
  }

  try {
    const { to, subject, html, from }: EmailRequest = await req.json();

    const response = await fetch("https://api.resend.com/emails", {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${RESEND_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        from: from ?? "Your App <noreply@yourdomain.com>",
        to: [to],
        subject,
        html,
      }),
    });

    const data = await response.json();

    if (!response.ok) {
      return new Response(JSON.stringify({ error: data }), {
        status: response.status,
        headers: { "Content-Type": "application/json" },
      });
    }

    return new Response(JSON.stringify({ id: data.id, success: true }), {
      headers: { "Content-Type": "application/json" },
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: "Internal server error" }), {
      status: 500,
      headers: { "Content-Type": "application/json" },
    });
  }
});
```

**Set the environment variable**

In the Supabase dashboard under "Settings → Edge Functions → Secrets," add `RESEND_API_KEY`. For local development, add it to `.env.local` — Supabase CLI picks it up automatically when you run `supabase functions serve`.

## Step 3: Automate Welcome Emails with Database Webhooks

Rather than calling the email function manually every time someone signs up, you can automate it using Supabase Database Webhooks.

In the Supabase dashboard, go to "Database → Webhooks" and create a webhook that fires on `INSERT` into the `auth.users` table, pointing to your Edge Function.

Here's a dedicated welcome email function:

```typescript
// supabase/functions/send-welcome-email/index.ts
import { serve } from "https://deno.land/std@0.208.0/http/server.ts";

const RESEND_API_KEY = Deno.env.get("RESEND_API_KEY")!;

serve(async (req: Request) => {
  const payload = await req.json();
  const userEmail = payload.record?.email;
  const userName = payload.record?.raw_user_meta_data?.name ?? "there";

  if (!userEmail) {
    return new Response("No email", { status: 400 });
  }

  const html = `
    <!DOCTYPE html>
    <html>
      <body style="font-family: sans-serif; max-width: 600px; margin: 0 auto; padding: 20px;">
        <h1 style="color: #333;">Welcome, ${userName}!</h1>
        <p>Your account is all set. Let's get started.</p>
        <a href="https://yourdomain.com/onboarding"
           style="display: inline-block; background: #6366f1; color: white;
                  padding: 12px 24px; border-radius: 8px; text-decoration: none;">
          Get Started
        </a>
        <p style="color: #888; font-size: 12px; margin-top: 40px;">
          If you didn't create an account, you can safely ignore this email.
        </p>
      </body>
    </html>
  `;

  await fetch("https://api.resend.com/emails", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${RESEND_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      from: "Your App <welcome@yourdomain.com>",
      to: [userEmail],
      subject: "Welcome! Your account is ready",
      html,
    }),
  });

  return new Response("OK");
});
```

Now every new signup automatically gets a welcome email — no extra code in your Rork app required.

## Step 4: Trigger Emails from the Rork App

For actions like contact form submissions or in-app notification preferences, you'll want to call your Edge Function directly from the Rork app.

```typescript
// services/emailService.ts
import { supabase } from "./supabaseClient";

interface SendEmailParams {
  to: string;
  subject: string;
  html: string;
}

export async function sendEmail({ to, subject, html }: SendEmailParams) {
  const { data, error } = await supabase.functions.invoke("send-email", {
    body: { to, subject, html },
  });

  if (error) {
    console.error("Email send error:", error);
    throw new Error("Failed to send email");
  }

  return data;
}

// Example: send a confirmation after a contact form submission
export async function sendContactConfirmation(userEmail: string, userName: string) {
  const html = `
    <h2>We received your message</h2>
    <p>Hi ${userName}, thanks for reaching out!</p>
    <p>We'll get back to you within 1–2 business days.</p>
  `;

  return sendEmail({
    to: userEmail,
    subject: "We got your message",
    html,
  });
}
```

In your Rork component:

```typescript
// ContactForm.tsx
import { sendContactConfirmation } from "@/services/emailService";

const handleSubmit = async () => {
  try {
    setLoading(true);
    // ... save inquiry to database
    await sendContactConfirmation(userEmail, userName);
    Alert.alert("Sent!", "Check your inbox for a confirmation.");
  } catch (error) {
    Alert.alert("Error", "Something went wrong. Please try again.");
  } finally {
    setLoading(false);
  }
};
```

## Step 5: Level Up with React Email Templates

When your emails need more polish — custom branding, responsive layouts, dynamic content — **React Email** eliminates the pain of writing raw HTML email.

```bash
npm install @react-email/components react-email --save-dev
```

```typescript
// emails/WelcomeEmail.tsx
import {
  Html, Head, Body, Container, Heading, Text, Button, Hr
} from "@react-email/components";

interface WelcomeEmailProps {
  userName: string;
}

export default function WelcomeEmail({ userName }: WelcomeEmailProps) {
  return (
    <Html>
      <Head />
      <Body style={{ backgroundColor: "#f9fafb", fontFamily: "sans-serif" }}>
        <Container style={{ maxWidth: 560, margin: "0 auto", padding: 24 }}>
          <Heading>Welcome, {userName}!</Heading>
          <Text>Your account is ready. Here's what to do next.</Text>
          <Button
            href="https://yourdomain.com/onboarding"
            style={{
              backgroundColor: "#6366f1",
              color: "#fff",
              padding: "12px 24px",
              borderRadius: 8,
            }}
          >
            Get Started
          </Button>
          <Hr />
          <Text style={{ fontSize: 12, color: "#888" }}>
            This is an automated message. Please don't reply directly.
          </Text>
        </Container>
      </Body>
    </Html>
  );
}
```

Render the template to an HTML string and pass it to Resend:

```typescript
import { render } from "@react-email/render";
import WelcomeEmail from "./emails/WelcomeEmail";

// Renders to an HTML string — pass this as `html` to Resend
const html = render(<WelcomeEmail userName="Alex" />);
```

Run `npx email dev` to preview templates in your browser as you build them. It's a game-changer compared to the old "send → check inbox → tweak → repeat" cycle.

## Troubleshooting Common Errors

**"550 Domain not verified" error**

Your sending domain isn't verified yet. Check the "Domains" page in your Resend dashboard and confirm the status shows "Verified." In the meantime, use `onboarding@resend.dev` as a temporary from address.

**"403 Forbidden" error**

Double-check that your API key has "Sending Access" scope. Also verify that the key is correctly saved as a Supabase Secret (watch out for typos like `RESEND_API_KEY` vs. `RESEND_APIKEY`).

**Emails landing in spam**

Ensure all three DNS records are set: SPF, DKIM, and DMARC. A DMARC policy of `p=none` provides the weakest protection — consider upgrading to `p=quarantine` once you've confirmed your setup is working correctly.

**Edge Function timeout**

Resend API responses are typically under a second, but if you experience timeouts on Supabase's free tier, consider adding a retry with exponential backoff or migrating to a paid Supabase plan for longer function execution limits.

## FAQ

**Q. How does this relate to Supabase Auth's built-in email feature?**

Supabase Auth handles signup confirmation and password reset emails natively, and you can point it at Resend as a custom SMTP provider. A clean separation: let Supabase Auth handle authentication emails, and use a dedicated Edge Function (as shown here) for app-specific emails like welcome flows, notifications, and receipts.

**Q. What happens when I exceed the free tier's 3,000 emails/month?**

Resend's paid plans start at $20/month for 100,000 emails. For most indie apps in early growth, the free tier will cover you for quite a while. For newsletter or bulk sending workloads, compare the per-email cost across providers before committing.

**Q. Does this work with Cloudflare Workers instead of Supabase?**

Yes — anywhere you can make a `fetch` call, you can use Resend. Swap `Deno.env.get("RESEND_API_KEY")` for `env.RESEND_API_KEY` in a Cloudflare Worker context. See also: [Rork × Cloudflare D1 backend guide](/articles/rork-dev/rork-cloudflare-d1-sqlite-backend-guide).

## Summary

Adding email to your Rork app with Resend comes down to a few clear steps:

- Sign up for Resend, verify your domain, and grab an API key
- Create a Supabase Edge Function that calls the Resend API (never from the client side)
- Automate welcome emails with Database Webhooks on the `auth.users` table
- Invoke the Edge Function from your Rork app for action-triggered emails
- Use React Email for reusable, previewable HTML email templates

Email is one of the highest-ROI retention tools in your toolbox — especially for reaching users who've disabled push notifications. A well-timed welcome email or re-engagement nudge can meaningfully improve activation and retention rates.
