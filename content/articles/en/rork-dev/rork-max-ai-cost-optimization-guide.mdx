## Introduction

Building AI-powered features in Rork Max apps is transformative, but it can quickly become an expensive experiment. Whether you're adding a chatbot, an AI writing assistant, or personalized content generation, the API costs can spiral out of control as your user base grows.

The good news: you don't have to choose between feature richness and affordability. This guide shares five proven techniques that can slash your AI costs by 50–70% without compromising quality.

For broader app performance optimization strategies, see [Advanced Rork App Performance Optimization](/articles/rork-dev/advanced-rork-app-performance-optimization).

## Understanding Rork Max AI Cost Structure

Before optimizing, let's understand how costs accumulate. Rork Max billing breaks down into four components.

- **Input token costs**: Every character in your prompt and system instructions counts
- **Output token costs**: Model responses typically cost more per token than inputs
- **Prompt cache read costs**: Reusing cached prompts costs ~90% less than fresh requests
- **Model pricing tiers**: Claude Opus 4.6 is roughly 20x more expensive than Haiku 4.5

Rork Max gives you monthly credits. Exceeding them triggers pay-as-you-go billing. Learn more in [Rork Max AI Features Guide](/articles/rork-ai/rork-max-ai-features).

## Technique 1: Compress Your System Prompt

Your system prompt ships with every API call. Trim it, and you immediately cut costs.

**Before (verbose)**:
```
You are a helpful and kind AI assistant. When users ask
questions, provide detailed, clear, and thoughtful responses.
Always be respectful and professional. Go the extra mile to
help users find what they need.
```

**After (optimized)**:
```
Helpful AI assistant. Clear, accurate, useful responses.
```

This alone can cut input tokens by 80%+ with no quality loss.

## Technique 2: Manage Conversation Context

Sending the entire chat history with every request is a silent cost killer.

```javascript
// Trim conversation history to prevent unbounded token growth
function trimConversationHistory(messages, maxMessages = 10) {
  // Always preserve system context
  const systemMessages = messages.filter(m => m.role === 'system');
  const conversationMessages = messages.filter(m => m.role !== 'system');

  // Keep only the most recent exchanges
  const trimmedConversation = conversationMessages.slice(-maxMessages);

  return [...systemMessages, ...trimmedConversation];
}

// Usage: keep multi-turn conversations cost-predictable
const optimizedMessages = trimConversationHistory(conversationHistory, 6);
// Expected effect: conversation length doesn't inflate costs indefinitely
```

## Technique 3: Leverage Prompt Caching

Claude's prompt caching feature reduces the cost of repeated context by ~90%.

```javascript
// Example: using prompt caching with product manuals or knowledge bases
const response = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': process.env.ANTHROPIC_API_KEY,
    'anthropic-version': '2023-06-01',
    'anthropic-beta': 'prompt-caching-2024-07-31'
  },
  body: JSON.stringify({
    model: 'claude-haiku-4-5-20251001',
    max_tokens: 1024,
    system: [
      {
        type: 'text',
        text: 'Long product documentation or knowledge base goes here...',
        cache_control: { type: 'ephemeral' }  // Enable caching
      }
    ],
    messages: [
      {
        role: 'user',
        content: 'How do I use feature X?'
      }
    ]
  })
});

// First request: cache_creation_input_tokens billed at standard rate
// Subsequent requests: cache_read_input_tokens billed at ~10% of standard rate
// Response headers show cache hit:
// cache_creation_input_tokens: 1250  (first call only)
// cache_read_input_tokens: 1250      (subsequent calls, same 5-min window)
```

## Technique 4: Route Tasks to the Right Model

Not every request deserves your most powerful (and expensive) model.

Task-to-model mapping:

- **Simple classification, Q&A**: Claude Haiku 4.5 (fastest, cheapest)
- **Standard text generation, summarization**: Claude Sonnet 4.6 (balanced cost/quality)
- **Complex reasoning, code synthesis**: Claude Opus 4.6 (best quality, highest cost)

```javascript
// Automatically select the right model based on task difficulty
function selectModel(taskComplexity) {
  if (taskComplexity === 'simple') {
    return 'claude-haiku-4-5-20251001';   // optimal for binary/ternary choices
  } else if (taskComplexity === 'medium') {
    return 'claude-sonnet-4-6';           // balanced for most tasks
  } else {
    return 'claude-opus-4-6';             // save for multi-step reasoning
  }
}

// Example: sentiment classification uses Haiku
const model = selectModel('simple');
console.log(model); // claude-haiku-4-5-20251001
```

## Technique 5: Cap Output Length with Structured Responses

Longer outputs = higher bills. Enforce structured, concise responses.

```javascript
// Limit output and request JSON-only responses
const prompt = `Classify the following review as positive, negative, or neutral.
Reply with JSON only: {"sentiment": "positive|negative|neutral", "confidence": 0.0-1.0}

Review: ${userReview}`;

const response = await callClaude({
  model: 'claude-haiku-4-5-20251001',
  max_tokens: 50,  // structured responses don't need more
  messages: [{ role: 'user', content: prompt }]
});

// Expected output:
// {"sentiment": "positive", "confidence": 0.87}
```

## Frequently Asked Questions

**Q: What happens if I exceed my Rork Max monthly credits?**

A: You'll switch to pay-per-use billing. Prevent overages by setting spending limits early in the month and enabling dashboard alerts in Anthropic Console.

**Q: Which Claude models support prompt caching?**

A: All Claude models (Opus, Sonnet, Haiku families) support caching. Caches are valid for 5 minutes, so caching is most effective for high-frequency repeated requests.

**Q: What's the best way to cut costs without sacrificing quality?**

A: Combine prompt caching with task-based model routing. Cache large knowledge bases while routing simple tasks to Haiku. This combination typically cuts costs 50–70% with no perceptible quality loss.

**Q: Can I use caching for real-time user interactions?**

A: Yes, caching works well for real-time apps—especially if your system prompt or knowledge base is large and reused across many users. The 5-minute cache window is sufficient for most conversational flows.

**Q: How do I measure my cost savings?**

A: Compare your monthly spend before and after optimization using Anthropic Console's usage breakdown. Track cache hit rates in your monitoring dashboard to ensure caching is actually working.

## Summary

Optimize in this order for maximum impact: (1) compress your system prompt, (2) trim context windows, (3) implement prompt caching, (4) route tasks to appropriate models, (5) cap output lengths.

Start today by refactoring your system prompt. It's the quickest win.
