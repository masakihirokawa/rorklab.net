## Most Crashes Go Unnoticed Until It's Too Late

After shipping an app, one of the scariest realities is that users often leave silently. When a crash or unexpected error happens, most people don't file a report — they just uninstall. Many indie developers only find out about critical bugs through App Store reviews, long after the damage is done.

Sentry is an open-source error monitoring platform that solves this problem. It integrates seamlessly with React Native (the framework powering Rork apps) and automatically captures stack traces, device information, and reproduction context the moment a crash occurs. In this guide, we'll walk through setting up Sentry in your Rork app from scratch — so you can build a production-grade monitoring system with confidence.

## What Is Sentry and Why Does Your Rork App Need It?

Founded in San Francisco in 2012, Sentry is used by over four million developers worldwide as of 2026. Its free tier supports up to 5,000 error events per month, making it an accessible starting point for indie developers.

Here are three reasons Sentry is valuable for Rork app development:

**1. Catch crashes you can't reproduce locally**
Real devices differ from your development machine in OS version, memory pressure, and network conditions. Sentry captures exactly what happens in production — not just in your simulator.

**2. Understand the blast radius immediately**
Sentry's dashboard shows you at a glance what percentage of users are affected, which device models are impacted most, and how recently the issue started. This helps you prioritize fixes rationally rather than reactively.

**3. Track quality across releases**
When you ship a new version, Sentry's release comparison feature lets you check whether your error rate has gone up — helping you catch regressions before they compound.

## Step 1 — Create a Sentry Account and Project

Head to [sentry.io](https://sentry.io) and sign up for a free account. Logging in with your GitHub account is the quickest path.

1. In the dashboard, create a new project
2. Select **React Native** as the platform
3. Enter a project name (e.g., `my-rork-app`)
4. Click **Create Project**

After creation, you'll see your **DSN (Data Source Name)** — a connection string that looks like this:

```
https://xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx@o123456.ingest.sentry.io/7654321
```

Save this value; you'll need it shortly.

## Step 2 — Add the Sentry SDK to Your Rork App

### Installing the Package

In Rork's AI chat, send this message to add the required package automatically:

```
Add the @sentry/react-native package and configure it to automatically capture errors app-wide.
Load the DSN from an environment variable so it's not hardcoded.
```

Rork will update `package.json` and the necessary files for you. If you prefer to do it manually, run:

```bash
npx expo install @sentry/react-native
```

### Adding the Expo Plugin to app.json

To enable automatic instrumentation (including source map uploads), add the Sentry plugin to `app.json`:

```json
{
  "expo": {
    "plugins": [
      [
        "@sentry/react-native/expo",
        {
          "url": "https://sentry.io/",
          "organization": "your-org-slug",
          "project": "your-project-slug"
        }
      ]
    ]
  }
}
```

Find your `organization` and `project` slugs in the Sentry dashboard URL: `https://sentry.io/organizations/{org}/projects/{project}/`.

### Initializing Sentry

Add the initialization code to your app's entry point (`App.tsx` or `_layout.tsx`):

```typescript
import * as Sentry from "@sentry/react-native";

// Called once when the app starts
Sentry.init({
  dsn: process.env.EXPO_PUBLIC_SENTRY_DSN, // Load DSN from environment variable

  // Only send errors in production
  enabled: process.env.NODE_ENV === "production",

  // Performance monitoring sample rate (20% of transactions)
  tracesSampleRate: 0.2,

  // Session replay (optional)
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
});

// Wrap your root component to capture React render errors
export default Sentry.wrap(App);
```

Add your DSN to your environment file:

```bash
# .env.production
EXPO_PUBLIC_SENTRY_DSN=https://your-dsn@sentry.io/your-project-id
```

**⚠️ Never hardcode your DSN directly in source code.** Use `.env` files and add them to `.gitignore` to keep credentials out of version control.

### Capturing Errors Manually

Beyond automatic crash detection, you can manually report errors from specific code paths:

```typescript
import * as Sentry from "@sentry/react-native";

async function fetchUserData(userId: string) {
  try {
    const response = await fetch(`https://api.example.com/users/${userId}`);
    if (!response.ok) {
      // Report HTTP errors to Sentry with context
      Sentry.captureException(new Error(`API Error: ${response.status}`), {
        tags: {
          screen: "UserProfile",
          action: "fetchUserData",
        },
        extra: {
          userId,
          statusCode: response.status,
        },
      });
      return null;
    }
    return await response.json();
  } catch (error) {
    // Network errors and other exceptions
    Sentry.captureException(error);
    throw error;
  }
}
```

### Attaching User Context

Associating errors with specific users makes it far easier to reproduce and prioritize issues:

```typescript
// Call after successful login
Sentry.setUser({
  id: user.id,
  email: user.email,
  // Disclose data collection in your privacy policy
});

// Clear on logout
Sentry.setUser(null);
```

## Step 3 — Configure Alerts and Dashboard

Once the SDK is in place, set up notification rules in the Sentry dashboard.

### Setting Up Notifications

Navigate to **Alerts** → **Create Alert** in your Sentry project. For solo developers, these rules work well:

- **Condition**: A new issue is first seen
- **Action**: Send an email or Slack notification
- **Filter**: Error count exceeds 10 in one hour (to reduce noise)

Getting notified the moment a crash occurs — before users even notice — is one of the most powerful aspects of real-time monitoring.

### Performance Monitoring

Sentry can also track app performance, not just crashes. Use transactions to measure API response times and screen load speeds:

```typescript
// Measure how long an API call takes
const transaction = Sentry.startTransaction({
  name: "fetch-articles",
  op: "http.client",
});

try {
  const data = await fetchArticles();
  transaction.setStatus("ok");
} catch (error) {
  transaction.setStatus("internal_error");
  Sentry.captureException(error);
} finally {
  transaction.finish(); // Send timing data to Sentry
}
```

## Frequently Asked Questions

**Q1. How much can I do on the free plan?**

Sentry's free tier includes 5,000 error events per month with a 30-day data retention window. This is more than enough to get started with an indie app. Once your monthly active users exceed roughly 10,000, consider upgrading to a paid plan.

**Q2. Will Sentry send my users' personal data?**

By default, IP addresses are included unless you set `sendDefaultPii: false` (the default value). Any user data added via `Sentry.setUser()` is intentionally sent. Make sure to mention error monitoring tools in your privacy policy.

**Q3. Does Sentry work with Expo Go?**

`@sentry/react-native` has limited functionality in Expo Go. Native crash detection in particular requires Expo Dev Client (EAS Build). If you're using Rork Max, which supports native builds, you can take advantage of the full feature set.

**Q4. How does Sentry compare to Firebase Crashlytics?**

Firebase Crashlytics specializes in crash detection and integrates tightly with Google's ecosystem. Sentry covers a broader scope — non-crash errors, API failures, custom events, performance monitoring, and release tracking — with more flexible alerting. Sentry is particularly useful if you want unified visibility across mobile and backend services.
