This advanced guide walks through the complete workflow of building custom models with NeMo, implementing edge inference via NIM microservices, and seamlessly integrating them into Rork applications.

## Understanding the NeMo Framework Architecture

NVIDIA NeMo provides a unified, PyTorch-based platform for the entire lifecycle of LLM development—from model design and training through to production deployment at scale. Its modular design supports both centralized cloud environments and distributed edge inference scenarios.

### Three-Layer Architecture

**Layer 1: Core Framework (Foundation)**
The foundation layer provides declarative computational graph definition through `NeuralModule`, enabling automatic differentiation, mixed-precision training (AMP), and distributed training across multiple GPUs or TPUs. Checkpoint management and experiment tracking are built in, allowing seamless resumption of interrupted training jobs.

**Layer 2: Collections (Domain-Specific Implementations)**
Each collection is a curated set of models and architectures optimized for specific tasks:
- **NLP Collection**: Language understanding, text generation, question answering
- **ASR Collection**: Automatic speech recognition with streaming support
- **TTS Collection**: Text-to-speech with natural prosody
- **Vision Collection**: Visual understanding and multimodal tasks

**Layer 3: Pretrained Models & Tools (Production Ready)**
Hosted on NVIDIA's NGC (GPU Cloud), this layer provides pre-trained checkpoints, export utilities, and optimization tools. Parameter-efficient fine-tuning methods like LoRA and QLoRA reduce memory footprint and training time, making advanced AI accessible on resource-constrained devices.

### Mobile-Optimized Models: Nemotron-ASR-Streaming

For edge deployment on mobile devices, Nemotron-ASR-Streaming—a 600M parameter lightweight speech recognition model—provides state-of-the-art accuracy while remaining responsive on smartphones and tablets. This model serves as an excellent foundation for domain-specific fine-tuning without the computational burden of larger models.

## Step 2: Domain-Specific Fine-Tuning with Your Data

### Preparing and Validating Training Data

Effective fine-tuning begins with high-quality, domain-specific data. For a mobile application context, curate text that represents real user inputs and expected model outputs:

```python
import json
from pathlib import Path
import re

def prepare_training_data(raw_texts: list[str], output_file: str):
    """Prepare and validate training data in JSONL format"""
    processed_data = []

    for text in raw_texts:
        # Normalize whitespace
        normalized = re.sub(r'\s+', ' ', text).strip()

        # Remove control characters and invalid sequences
        normalized = re.sub(
            r'[^\w\s\-\.\:\,\!\?]',
            '',
            normalized,
            flags=re.UNICODE
        )

        if len(normalized) > 10:  # Filter out very short examples
            processed_data.append({
                "text": normalized,
                "domain": "mobile_ui",
                "length": len(normalized.split()),
            })

    # Save as JSONL
    with open(output_file, 'w', encoding='utf-8') as f:
        for item in processed_data:
            f.write(json.dumps(item, ensure_ascii=False) + '\n')

    print(f"Prepared {len(processed_data)} training examples → {output_file}")
    return processed_data

# Usage
raw_texts = [
    "User voice input transcription for mobile UI",
    "Real-time text processing needed for responsive apps",
]
prepare_training_data(raw_texts, "data/mobile_train.jsonl")
```

### Efficient Fine-Tuning with LoRA

Instead of updating all model parameters (which would require enormous GPU memory), Low-Rank Adaptation (LoRA) introduces a small number of trainable parameters that modulate the original weights. This approach reduces memory usage by 90%+ while maintaining competitive accuracy:

```python
from nemo.collections.nlp.models import MegatronGPTModel
from nemo.core.config import OmegaConf
import torch
from pytorch_lightning import Trainer, seed_everything

seed_everything(42)

# Configure LoRA
lora_config = OmegaConf.create({
    "target_modules": [
        "query_proj", "key_proj", "value_proj",
        "dense_h_to_4h", "dense_4h_to_h"
    ],
    "r": 8,  # LoRA rank
    "lora_alpha": 16,  # Scaling factor
    "lora_dropout": 0.1,
    "bias": "none",
})

# Load pre-trained model and apply LoRA
model = MegatronGPTModel.from_pretrained("nvidia/megatron-gpt-345m")
model.add_lora(lora_config)

# Freeze non-LoRA parameters to save memory
for name, param in model.named_parameters():
    if "lora" not in name:
        param.requires_grad = False

# Training configuration optimized for single GPU
trainer_config = {
    "max_epochs": 3,
    "devices": 1,
    "accelerator": "gpu",
    "precision": "16-mixed",  # Mixed precision for efficiency
    "accumulate_grad_batches": 4,
    "val_check_interval": 100,
    "log_every_n_steps": 10,
    "enable_progress_bar": True,
}

trainer = Trainer(**trainer_config)

# Assuming train_loader and val_loader are prepared
trainer.fit(model, train_dataloaders=train_loader, val_dataloaders=val_loader)

# Save only LoRA weights (much smaller than full model)
model.save_lora_weights("checkpoints/mobile_lora.pt")
print("LoRA weights saved for edge deployment")
```

## Step 3: Exporting and Optimizing for Edge Inference

### Converting to ONNX and TensorRT

Edge devices benefit from optimized inference formats. ONNX Runtime on mobile and TensorRT on edge servers provide significantly faster inference:

```python
import tensorrt as trt
from nemo.utils import export_to_onnx
import torch

# Export NeMo model to ONNX (intermediate format)
dummy_input = torch.randn(1, 512, dtype=torch.long)
model.export(
    output_file="model_export/model.onnx",
    check_trace=True,
    input_example=dummy_input
)

def create_tensorrt_engine(
    onnx_path: str,
    output_path: str,
    precision: str = "fp16"
):
    """
    Build a TensorRT engine from ONNX model for optimized inference.

    Args:
        onnx_path: Path to ONNX model file
        output_path: Path to save TensorRT engine
        precision: "fp32", "fp16", or "int8"
    """
    logger = trt.Logger(trt.Logger.WARNING)
    builder = trt.Builder(logger)

    network = builder.create_network(
        1 << int(trt.NetworkDefinitionCreationFlag.EXPLICIT_BATCH)
    )
    parser = trt.OnnxParser(network, logger)

    # Parse ONNX model
    with open(onnx_path, 'rb') as model_file:
        if not parser.parse(model_file.read()):
            raise RuntimeError("Failed to parse ONNX model")

    # Configure builder for optimization
    config = builder.create_builder_config()
    config.set_flag(trt.BuilderFlag.GPU_FALLBACK)

    # Set precision
    if precision == "fp16":
        config.set_flag(trt.BuilderFlag.FP16)
    elif precision == "int8":
        config.set_flag(trt.BuilderFlag.INT8)
        # INT8 requires calibration data (implementation omitted)

    # Build engine
    engine = builder.build_engine(network, config)

    # Serialize and save
    with open(output_path, 'wb') as engine_file:
        engine_file.write(engine.serialize())

    print(f"TensorRT engine saved: {output_path}")
    return engine

# Create optimized inference engine
create_tensorrt_engine(
    "model_export/model.onnx",
    "model_export/model.trt",
    precision="fp16"
)
```

## Step 4: Building NVIDIA NIM Microservices

### Unified Inference Endpoint with FastAPI

NVIDIA NIM provides a standardized microservice interface for inference across cloud, data center, and edge environments. Here's how to build a production-ready inference service:

```python
# inference_server.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import torch
import time
from typing import Optional
import logging

# Configure logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

app = FastAPI(
    title="NeMo Edge Inference Service",
    version="1.0.0",
    description="High-performance LLM inference for mobile edge deployment"
)

# Request/Response models
class InferenceRequest(BaseModel):
    text: str
    max_tokens: Optional[int] = 256
    temperature: Optional[float] = 0.7
    top_p: Optional[float] = 0.9

class InferenceResponse(BaseModel):
    result: str
    tokens_used: int
    latency_ms: float
    model: str

# Global state
model = None
tokenizer = None
device = None

@app.on_event("startup")
async def startup_event():
    """Load model on application startup"""
    global model, tokenizer, device

    device = "cuda" if torch.cuda.is_available() else "cpu"
    logger.info(f"Using device: {device}")

    try:
        from nemo.collections.nlp.models import MegatronGPTModel
        from transformers import AutoTokenizer

        # Load pre-trained model
        model = MegatronGPTModel.from_pretrained("nvidia/megatron-gpt-345m")
        model = model.to(device)
        model.eval()

        # Load tokenizer
        tokenizer = AutoTokenizer.from_pretrained("nvidia/megatron-gpt-345m")

        logger.info("Model loaded successfully")
    except Exception as e:
        logger.error(f"Failed to load model: {e}")
        raise

@app.post("/v1/completions", response_model=InferenceResponse)
async def completions(request: InferenceRequest):
    """Text generation inference endpoint"""
    if model is None:
        raise HTTPException(status_code=503, detail="Model not loaded")

    start_time = time.time()

    try:
        # Tokenize input
        input_ids = tokenizer.encode(request.text, return_tensors="pt").to(device)
        batch_size = input_ids.shape[0]

        # Validate input
        if input_ids.shape[1] > 2048:
            raise HTTPException(
                status_code=400,
                detail="Input sequence exceeds maximum length (2048)"
            )

        # Run inference
        with torch.no_grad():
            output_ids = model.generate(
                input_ids,
                max_length=min(request.max_tokens + input_ids.shape[1], 4096),
                temperature=request.temperature,
                top_p=request.top_p,
                do_sample=True,
                pad_token_id=tokenizer.eos_token_id,
                eos_token_id=tokenizer.eos_token_id,
            )

        # Decode output
        output_text = tokenizer.decode(output_ids[0], skip_special_tokens=True)
        tokens_generated = output_ids.shape[1] - input_ids.shape[1]

        latency_ms = (time.time() - start_time) * 1000

        logger.info(
            f"Inference completed: {tokens_generated} tokens in {latency_ms:.2f}ms"
        )

        return InferenceResponse(
            result=output_text,
            tokens_used=tokens_generated,
            latency_ms=latency_ms,
            model="megatron-gpt-345m"
        )

    except torch.cuda.OutOfMemoryError:
        raise HTTPException(
            status_code=503,
            detail="GPU out of memory"
        )
    except Exception as e:
        logger.error(f"Inference error: {e}")
        raise HTTPException(status_code=500, detail=str(e))

@app.post("/v1/health")
async def health():
    """Health check endpoint"""
    return {
        "status": "healthy",
        "device": str(device),
        "model_loaded": model is not None,
        "timestamp": time.time(),
    }

@app.get("/v1/models")
async def list_models():
    """List available models"""
    return {
        "object": "list",
        "data": [
            {
                "id": "megatron-gpt-345m",
                "object": "model",
                "owned_by": "nvidia",
                "permissions": []
            }
        ]
    }
```

## Step 5: Integrating with React Native Applications

### NIM Client Implementation with Expo SDK 55

Reference [Expo SDK 55 Performance Optimization](/articles/rork-dev/expo-sdk-55-performance-optimization) for context-aware implementation patterns. Here's a robust NIM client:

```typescript
// libs/nemo-inference-client.ts
import axios, { AxiosInstance, AxiosError } from 'axios';
import { Platform } from 'react-native';

interface InferenceRequest {
  text: string;
  maxTokens?: number;
  temperature?: number;
  topP?: number;
}

interface InferenceResponse {
  result: string;
  tokensUsed: number;
  latencyMs: number;
  model: string;
}

interface HealthCheckResponse {
  status: string;
  device: string;
  modelLoaded: boolean;
  timestamp: number;
}

export class NeMoInferenceClient {
  private client: AxiosInstance;
  private baseURL: string;
  private retryConfig: { maxRetries: number; retryDelay: number };

  constructor(nimServiceUrl: string, maxRetries: number = 3) {
    this.baseURL = nimServiceUrl;
    this.retryConfig = {
      maxRetries,
      retryDelay: 1000, // ms
    };

    this.client = axios.create({
      baseURL: this.baseURL,
      timeout: 30000,
      headers: {
        'Content-Type': 'application/json',
        'User-Agent': `NeMo-Client/${Platform.OS}`,
      },
    });

    // Add retry interceptor
    this.client.interceptors.response.use(
      (response) => response,
      async (error: AxiosError) => {
        const config = error.config as any;
        if (!config || !config.retry) {
          config.retry = 0;
        }

        if (config.retry < this.retryConfig.maxRetries) {
          config.retry += 1;
          await this._delay(this.retryConfig.retryDelay);
          return this.client(config);
        }

        return Promise.reject(error);
      }
    );
  }

  private _delay(ms: number): Promise<void> {
    return new Promise((resolve) => setTimeout(resolve, ms));
  }

  async generateText(request: InferenceRequest): Promise<InferenceResponse> {
    try {
      const response = await this.client.post<InferenceResponse>(
        '/v1/completions',
        {
          text: request.text,
          max_tokens: request.maxTokens || 256,
          temperature: request.temperature || 0.7,
          top_p: request.topP || 0.9,
        }
      );

      return response.data;
    } catch (error) {
      if (axios.isAxiosError(error)) {
        throw new Error(
          `NeMo inference failed: ${error.response?.status} - ${error.message}`
        );
      }
      throw error;
    }
  }

  async listModels(): Promise<string[]> {
    try {
      const response = await this.client.get('/v1/models');
      return response.data.data.map((m: any) => m.id);
    } catch (error) {
      throw new Error(`Failed to list models: ${error}`);
    }
  }

  async checkHealth(): Promise<HealthCheckResponse> {
    try {
      const response = await this.client.post<HealthCheckResponse>('/v1/health');
      return response.data;
    } catch {
      throw new Error('Health check failed');
    }
  }
}

export default NeMoInferenceClient;
```

## Step 6: Performance Optimization and On-Device Inference

### Memory and Battery Efficiency

Optimize resource consumption on mobile devices by implementing intelligent request batching and caching:

```typescript
// hooks/useEdgeInferenceCache.ts
import { useEffect, useRef, useCallback } from 'react';
import NeMoInferenceClient from '../libs/nemo-inference-client';

interface CacheEntry {
  result: string;
  timestamp: number;
  ttl: number; // milliseconds
}

export const useEdgeInferenceCache = (nimUrl: string, cacheTTL: number = 3600000) => {
  const clientRef = useRef(new NeMoInferenceClient(nimUrl));
  const cacheRef = useRef<Map<string, CacheEntry>>(new Map());

  const getCacheKey = useCallback((text: string): string => {
    return Buffer.from(text).toString('base64');
  }, []);

  const isCacheValid = useCallback((entry: CacheEntry): boolean => {
    return Date.now() - entry.timestamp < entry.ttl;
  }, []);

  const getCached = useCallback(
    (text: string) => {
      const key = getCacheKey(text);
      const entry = cacheRef.current.get(key);

      if (entry && isCacheValid(entry)) {
        return entry.result;
      }

      if (entry) {
        cacheRef.current.delete(key);
      }

      return null;
    },
    [getCacheKey, isCacheValid]
  );

  const inferenceWithCache = useCallback(
    async (text: string) => {
      // Check cache first
      const cached = getCached(text);
      if (cached) {
        return cached;
      }

      // Perform inference
      const response = await clientRef.current.generateText({
        text,
        maxTokens: 256,
        temperature: 0.7,
      });

      // Cache result
      const key = getCacheKey(text);
      cacheRef.current.set(key, {
        result: response.result,
        timestamp: Date.now(),
        ttl: cacheTTL,
      });

      return response.result;
    },
    [getCached, getCacheKey, cacheTTL]
  );

  const clearCache = useCallback(() => {
    cacheRef.current.clear();
  }, []);

  const getCacheStats = useCallback(() => {
    return {
      entries: cacheRef.current.size,
      memoryEstimate: cacheRef.current.size * 1024, // rough estimate
    };
  }, []);

  return {
    inferenceWithCache,
    clearCache,
    getCacheStats,
  };
};
```

### How much training data does NeMo require for effective fine-tuning?

NeMo with LoRA can achieve meaningful improvements with relatively modest datasets. For domain-specific fine-tuning, 1,000–10,000 examples typically yield significant accuracy gains. With QLoRA (quantized LoRA), even smaller datasets (100–500 examples) can produce useful domain adaptation.

### What are the key strategies for reducing edge inference latency?

**Model optimization**: Choose lightweight architectures (Nemotron-ASR, Phi-2) suited to deployment constraints.

**Quantization**: INT8 quantization via TensorRT delivers 3–4× speedup with minimal accuracy loss.

**Caching**: Store frequently-accessed results locally to eliminate redundant computation.

**Batching**: Process multiple requests together when possible to amortize computational overhead.

### How can we ensure privacy with edge inference?

By running inference on-device or within your infrastructure, you avoid transmitting sensitive user data to third-party servers. Additionally, implement local cache encryption and automatic data purging policies to comply with privacy regulations.

### What's the difference between cloud NIM and edge NIM?

Cloud NIM runs on data centers with powerful GPUs, supporting larger models and higher throughput. Edge NIM uses optimized, smaller models and runs on resource-constrained devices (mobile, IoT, embedded systems). Both expose the same API, making it simple to scale or relocate workloads.
