## Introduction

Push notifications are one of the most powerful tools for driving user engagement in mobile apps. Rork Max — the AI-powered native Swift app builder — makes it straightforward to implement Apple Push Notification service (APNs) integration through natural language prompts. This guide walks you through the entire process, from configuring your Apple Developer account to sending your first notification from a Cloudflare Workers backend.

## How iOS Push Notifications Work

Before diving in, here's the end-to-end flow:

1. Your app requests notification permission from the user
2. If granted, iOS registers with APNs and returns a unique device token
3. Your app sends the device token to your backend server
4. Your backend uses the token to send push notifications via APNs
5. APNs delivers the notification to the user's device

Rork Max handles steps 1–2 automatically with SwiftUI code generation. For steps 3–5, we'll use Cloudflare Workers as a lightweight, scalable backend.

## Step 1: Configure Apple Developer Settings

You'll need an active Apple Developer Program membership ($99/year) to use APNs.

### Generate an APNs Authentication Key

1. Sign in to the [Apple Developer Portal](https://developer.apple.com/)
2. Navigate to **Certificates, Identifiers & Profiles** → **Keys**
3. Click **+** to create a new key
4. Check **Apple Push Notifications service (APNs)**
5. Download the `.p8` key file and store it securely

<Callout type="warning">
You can only download the APNs key once. Store the `.p8` file in a secure location immediately after downloading.
</Callout>

Note these values — you'll need them later:
- **Key ID**: 10-character identifier for your key
- **Team ID**: Your Apple Developer team identifier
- **Bundle ID**: Your app's bundle identifier (e.g., `com.yourname.appname`)

## Step 2: Generate SwiftUI Code with Rork Max

Open Rork Max and enter a prompt like this:

```
Add push notification support to my iOS app.
Requirements:
- Request notification permission on launch
- Retrieve the APNs device token and log it to the console
- Use the UserNotifications framework
- Send the device token via HTTP POST to https://api.myapp.com/register-device
```

Rork Max will generate the following SwiftUI and AppDelegate code:

```swift
import SwiftUI
import UserNotifications

@main
struct MyApp: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var appDelegate

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}

class AppDelegate: NSObject, UIApplicationDelegate, UNUserNotificationCenterDelegate {
    func application(_ application: UIApplication,
                     didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
        requestNotificationPermission()
        return true
    }

    func requestNotificationPermission() {
        UNUserNotificationCenter.current().delegate = self
        UNUserNotificationCenter.current().requestAuthorization(
            options: [.alert, .badge, .sound]
        ) { granted, error in
            if granted {
                DispatchQueue.main.async {
                    UIApplication.shared.registerForRemoteNotifications()
                }
            }
        }
    }

    func application(_ application: UIApplication,
                     didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
        let token = deviceToken.map { String(format: "%02.2hhx", $0) }.joined()
        print("APNs Token: \(token)")
        registerDeviceToken(token)
    }

    func application(_ application: UIApplication,
                     didFailToRegisterForRemoteNotificationsWithError error: Error) {
        print("Failed to register: \(error)")
    }

    func registerDeviceToken(_ token: String) {
        guard let url = URL(string: "https://api.myapp.com/register-device") else { return }
        var request = URLRequest(url: url)
        request.httpMethod = "POST"
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")
        let body = ["device_token": token, "platform": "ios"]
        request.httpBody = try? JSONSerialization.data(withJSONObject: body)
        URLSession.shared.dataTask(with: request).resume()
    }
}
```

## Step 3: Build the Backend with Cloudflare Workers

We'll use Cloudflare Workers to handle device token registration and push notification delivery — no server management required.

```javascript
// Cloudflare Workers: push-notification-api
export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    if (url.pathname === '/register-device' && request.method === 'POST') {
      return handleRegister(request, env);
    }

    if (url.pathname === '/send-push' && request.method === 'POST') {
      return handleSendPush(request, env);
    }

    return new Response('Not Found', { status: 404 });
  }
};

async function handleRegister(request, env) {
  const { device_token, platform } = await request.json();

  // Store token in Cloudflare KV
  await env.TOKENS.put(`device:${device_token}`, JSON.stringify({
    token: device_token,
    platform,
    registered_at: new Date().toISOString()
  }));

  return Response.json({ success: true });
}

async function handleSendPush(request, env) {
  const { device_token, title, body } = await request.json();

  // Generate APNs JWT auth token
  const jwt = await generateAPNsJWT(env);

  const response = await fetch(
    `https://api.push.apple.com/3/device/${device_token}`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${jwt}`,
        'apns-topic': env.BUNDLE_ID,
        'apns-push-type': 'alert',
        'content-type': 'application/json'
      },
      body: JSON.stringify({
        aps: {
          alert: { title, body },
          sound: 'default',
          badge: 1
        }
      })
    }
  );

  if (response.status === 200) {
    return Response.json({ success: true });
  } else {
    const error = await response.text();
    return Response.json({ success: false, error }, { status: 500 });
  }
}
```

### Cloudflare Workers Environment Variables

Add these secrets to your Worker's settings:

| Variable | Value |
|----------|-------|
| `APNS_KEY` | Contents of your `.p8` file |
| `APNS_KEY_ID` | Your 10-character key ID |
| `TEAM_ID` | Your Apple Developer team ID |
| `BUNDLE_ID` | Your app's bundle identifier |

## Step 4: Testing on a Real Device

<Callout type="info">
Push notifications require a physical device. The iOS Simulator will show the permission dialog but cannot receive APNs notifications.
</Callout>

### Testing Workflow

1. Install your Rork Max app on a real iPhone via QR code
2. Launch the app and grant notification permission
3. Copy the APNs token logged to the console
4. Test by sending a push with curl:

```bash
curl -X POST https://your-worker.workers.dev/send-push \
  -H "Content-Type: application/json" \
  -d '{
    "device_token": "YOUR_DEVICE_TOKEN",
    "title": "Test Notification",
    "body": "Hello from Rork Max!"
  }'
```

### Common Errors and Fixes

| Error | Cause | Fix |
|-------|-------|-----|
| `BadDeviceToken` | Invalid or expired token | Re-register on a real device |
| `InvalidBundleId` | Bundle ID mismatch | Check your APNs key settings |
| `ExpiredProviderToken` | JWT older than 1 hour | Implement JWT refresh logic |
| `TooManyRequests` | APNs rate limit hit | Add throttling or batch sends |
