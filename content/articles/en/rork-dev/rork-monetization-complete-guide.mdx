## Introduction: Three Ways to Monetize a Rork App

Apps built with Rork can be monetized through three primary strategies:

1. **Subscriptions** — Monthly/annual recurring billing (highest LTV)
2. **One-Time Purchase (Lifetime)** — Pay once, access forever
3. **Ads (AdMob)** — Revenue from free users

This guide focuses on **RevenueCat-powered subscriptions + lifetime purchases** — the combination that consistently generates the most sustainable revenue for indie app developers.

<Callout type="info">
Rork is built on React Native, so RevenueCat's React Native SDK works out of the box. You'll need an Apple Developer Program account and/or a Google Play Console account to distribute paid apps.
</Callout>

## 1. RevenueCat Setup

### Installation

Tell Rork to install RevenueCat via a prompt:

```
Add RevenueCat in-app purchases to this app.
Install react-native-purchases.
Configure with iOS and Android API keys.
```

Or add directly:

```bash
npm install react-native-purchases
```

### App Initialization (`App.tsx`)

```typescript
// App.tsx
import { useEffect } from 'react';
import Purchases, { LOG_LEVEL } from 'react-native-purchases';
import { Platform } from 'react-native';

const RC_IOS_KEY = 'appl_xxxxxxxxxxxxxxxxxxxxxxxx';
const RC_ANDROID_KEY = 'goog_xxxxxxxxxxxxxxxxxxxxxxxx';

export default function App() {
  useEffect(() => {
    const init = async () => {
      if (__DEV__) {
        Purchases.setLogLevel(LOG_LEVEL.DEBUG);
      }
      await Purchases.configure({
        apiKey: Platform.OS === 'ios' ? RC_IOS_KEY : RC_ANDROID_KEY,
      });
    };
    init();
  }, []);

  return <RootNavigator />;
}
```

## 2. Purchase State Management

```typescript
// contexts/PurchaseContext.tsx
import React, {
  createContext, useContext, useState, useEffect, useCallback,
} from 'react';
import Purchases, {
  CustomerInfo,
  PurchasesOffering,
  PurchasesPackage,
} from 'react-native-purchases';

interface PurchaseContextType {
  customerInfo: CustomerInfo | null;
  currentOffering: PurchasesOffering | null;
  isPro: boolean;
  isPremium: boolean;
  isLoading: boolean;
  purchasePackage: (pkg: PurchasesPackage) => Promise<boolean>;
  restorePurchases: () => Promise<void>;
  refreshCustomerInfo: () => Promise<void>;
}

const PurchaseContext = createContext<PurchaseContextType | null>(null);

export function PurchaseProvider({ children }: { children: React.ReactNode }) {
  const [customerInfo, setCustomerInfo] = useState<CustomerInfo | null>(null);
  const [currentOffering, setCurrentOffering] = useState<PurchasesOffering | null>(null);
  const [isLoading, setIsLoading] = useState(true);

  // Check entitlements — these are configured in RevenueCat dashboard
  const isPro = Boolean(customerInfo?.entitlements.active['pro']);
  const isPremium = Boolean(customerInfo?.entitlements.active['premium']);

  useEffect(() => {
    const init = async () => {
      try {
        const [info, offerings] = await Promise.all([
          Purchases.getCustomerInfo(),
          Purchases.getOfferings(),
        ]);
        setCustomerInfo(info);
        setCurrentOffering(offerings.current);
      } catch (err) {
        console.error('RevenueCat init error:', err);
      } finally {
        setIsLoading(false);
      }
    };
    init();

    // Listen for purchase state changes (e.g. subscription renewed)
    const listener = Purchases.addCustomerInfoUpdateListener(setCustomerInfo);
    return () => listener.remove();
  }, []);

  const purchasePackage = useCallback(async (pkg: PurchasesPackage): Promise<boolean> => {
    try {
      const { customerInfo: newInfo } = await Purchases.purchasePackage(pkg);
      setCustomerInfo(newInfo);
      return true;
    } catch (err: any) {
      if (err.userCancelled) return false;
      throw err;
    }
  }, []);

  const restorePurchases = useCallback(async () => {
    const info = await Purchases.restorePurchases();
    setCustomerInfo(info);
  }, []);

  const refreshCustomerInfo = useCallback(async () => {
    const info = await Purchases.getCustomerInfo();
    setCustomerInfo(info);
  }, []);

  return (
    <PurchaseContext.Provider value={{
      customerInfo, currentOffering, isPro, isPremium,
      isLoading, purchasePackage, restorePurchases, refreshCustomerInfo,
    }}>
      {children}
    </PurchaseContext.Provider>
  );
}

export function usePurchase() {
  const ctx = useContext(PurchaseContext);
  if (!ctx) throw new Error('usePurchase must be used within PurchaseProvider');
  return ctx;
}
```

## 3. Paywall UI

```typescript
// components/Paywall.tsx
import React, { useState } from 'react';
import {
  View, Text, TouchableOpacity, StyleSheet,
  ActivityIndicator, Alert, ScrollView,
} from 'react-native';
import { PurchasesPackage } from 'react-native-purchases';
import { usePurchase } from '../contexts/PurchaseContext';

interface PaywallProps {
  onDismiss: () => void;
  onSuccess: () => void;
}

const BENEFITS = [
  'Unlimited access to all features',
  'No ads',
  'Cloud sync across all devices',
  'Priority customer support',
];

export function Paywall({ onDismiss, onSuccess }: PaywallProps) {
  const { currentOffering, purchasePackage, restorePurchases, isLoading } = usePurchase();
  const [purchasing, setPurchasing] = useState<string | null>(null);
  const [restoring, setRestoring] = useState(false);

  const packages = currentOffering?.availablePackages ?? [];
  const monthlyPkg = packages.find(p => p.packageType === 'MONTHLY');
  const annualPkg = packages.find(p => p.packageType === 'ANNUAL');
  const lifetimePkg = packages.find(p => p.packageType === 'LIFETIME');

  const handlePurchase = async (pkg: PurchasesPackage) => {
    setPurchasing(pkg.identifier);
    try {
      const success = await purchasePackage(pkg);
      if (success) onSuccess();
    } catch (err: any) {
      Alert.alert('Purchase Error', err.message ?? 'Something went wrong. Please try again.');
    } finally {
      setPurchasing(null);
    }
  };

  const handleRestore = async () => {
    setRestoring(true);
    try {
      await restorePurchases();
      Alert.alert('Restored', 'Your purchases have been restored.');
      onDismiss();
    } catch {
      Alert.alert('Restore Failed', 'Could not restore purchases. Please try again.');
    } finally {
      setRestoring(false);
    }
  };

  if (isLoading) {
    return (
      <View style={styles.loadingContainer}>
        <ActivityIndicator size="large" color="#4CE8A0" />
      </View>
    );
  }

  return (
    <ScrollView contentContainerStyle={styles.container}>
      <Text style={styles.title}>Upgrade to Premium</Text>
      <Text style={styles.subtitle}>Unlock everything — no limits</Text>

      <View style={styles.benefitsContainer}>
        {BENEFITS.map((b, i) => (
          <View key={i} style={styles.benefitRow}>
            <Text style={styles.checkmark}>✓</Text>
            <Text style={styles.benefitText}>{b}</Text>
          </View>
        ))}
      </View>

      {annualPkg && (
        <PurchaseButton
          pkg={annualPkg}
          label={`Annual — ${annualPkg.product.priceString}/yr`}
          badge="Best Value"
          isPurchasing={purchasing === annualPkg.identifier}
          onPress={() => handlePurchase(annualPkg)}
        />
      )}
      {monthlyPkg && (
        <PurchaseButton
          pkg={monthlyPkg}
          label={`Monthly — ${monthlyPkg.product.priceString}/mo`}
          isPurchasing={purchasing === monthlyPkg.identifier}
          onPress={() => handlePurchase(monthlyPkg)}
        />
      )}
      {lifetimePkg && (
        <PurchaseButton
          pkg={lifetimePkg}
          label={`Lifetime — ${lifetimePkg.product.priceString} once`}
          isPurchasing={purchasing === lifetimePkg.identifier}
          onPress={() => handlePurchase(lifetimePkg)}
        />
      )}

      <TouchableOpacity onPress={handleRestore} disabled={restoring}>
        <Text style={styles.restoreText}>
          {restoring ? 'Restoring...' : 'Restore Purchases'}
        </Text>
      </TouchableOpacity>

      <Text style={styles.legalText}>
        Charged to your iTunes/Google Play account.{'\n'}
        Subscriptions auto-renew unless cancelled 24hrs before renewal.
      </Text>
    </ScrollView>
  );
}

function PurchaseButton({ pkg, label, badge, isPurchasing, onPress }: {
  pkg: PurchasesPackage;
  label: string;
  badge?: string;
  isPurchasing: boolean;
  onPress: () => void;
}) {
  return (
    <TouchableOpacity
      style={[styles.btn, badge ? styles.featuredBtn : null]}
      onPress={onPress}
      disabled={isPurchasing}
    >
      {badge && <Text style={styles.badge}>{badge}</Text>}
      {isPurchasing
        ? <ActivityIndicator color="#fff" />
        : <Text style={styles.btnText}>{label}</Text>
      }
    </TouchableOpacity>
  );
}

const styles = StyleSheet.create({
  container: { padding: 24, paddingBottom: 48 },
  loadingContainer: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  title: { fontSize: 24, fontWeight: '700', color: '#fff', textAlign: 'center', marginBottom: 8 },
  subtitle: { fontSize: 15, color: '#aaa', textAlign: 'center', marginBottom: 24 },
  benefitsContainer: { marginBottom: 28 },
  benefitRow: { flexDirection: 'row', alignItems: 'center', marginBottom: 10 },
  checkmark: { color: '#4CE8A0', fontSize: 16, marginRight: 10, fontWeight: '700' },
  benefitText: { color: '#ddd', fontSize: 14 },
  btn: {
    backgroundColor: '#1e1e1e',
    borderWidth: 1,
    borderColor: '#ffffff15',
    borderRadius: 12,
    padding: 16,
    marginBottom: 12,
    alignItems: 'center',
  },
  featuredBtn: { borderColor: '#4CE8A0', backgroundColor: '#4CE8A010' },
  btnText: { color: '#fff', fontSize: 15, fontWeight: '600' },
  badge: {
    backgroundColor: '#4CE8A0',
```

## 4. App Store Review Compliance

Common rejection reasons and how to avoid them:

```typescript
/**
 * Apple App Store Requirements (mandatory):
 *
 * ✅ Restore Purchases button
 *    Every paywall screen MUST have "Restore Purchases"
 *    Rejections without it are automatic
 *
 * ✅ Subscription disclosure
 *    Show: trial period, price, renewal frequency, cancellation method
 *    Text: "Subscription auto-renews unless cancelled..."
 *
 * ✅ Free trial clarity
 *    If offering a trial: clearly state duration and post-trial price
 *
 * ✅ Immediate entitlement
 *    Features MUST unlock immediately after successful purchase
 *    Do not require app restart or server sync delay
 *
 * Google Play requirements:
 *
 * ✅ Billing disclosure
 *    Price and billing interval must be shown before purchase
 *
 * ✅ Cancellation instructions
 *    Users must be able to cancel easily (link to Play subscription settings)
 */

// App Store Connect configuration guide
const ASC_SETUP = {
  subscriptionGroup: "Premium Access",
  products: [
    "com.yourapp.pro.monthly",   // $4.99/month
    "com.yourapp.pro.annual",    // $29.99/year (~50% off)
    "com.yourapp.premium.lifetime", // $49.99 one-time
  ],
  // Always provide test credentials in review notes:
  reviewNote: "Sandbox test account: sandbox@example.com / Pass: Test1234!",
};
```

## Summary

RevenueCat abstracts the complexity of Apple and Google billing APIs into a clean, unified SDK. The key principles for a successful monetization implementation:

1. **Entitlements** control feature access — never check `productId` directly
2. **Restore Purchases** is not optional — Apple will reject without it
3. **Pricing strategy** matters more than implementation — annual plans with ~40% discount vs monthly consistently convert better than monthly-only

Track MRR, churn rate, and LTV in the RevenueCat dashboard to iterate on your pricing and paywall copy continuously.
