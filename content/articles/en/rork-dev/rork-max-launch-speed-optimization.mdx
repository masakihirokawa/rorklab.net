## 1. Bundle Size Optimization

The most effective way to reduce startup time is to minimize bundle size. Expo CLI provides powerful tools for this analysis.

### Analyzing Bundle Size

```bash
# Visualize which modules consume the most space
npx expo-bundle-analyzer
```

This tool helps identify which libraries are bloating your bundle and where optimization efforts should focus.

### Eliminating Unnecessary Dependencies

```json
{
  "dependencies": {
    "react": "18.2.0",
    "react-native": "0.72.0",
    "expo": "~49.0.0",
    // ❌ Optimization opportunity: lodash is ~70KB
    // ✅ Solution: Use lodash-es or import only needed functions
    "lodash": "^4.17.0",
    "react-native-fast-image": "^8.0.0"
  }
}
```

Smart imports dramatically reduce bundle size:

```javascript
// ❌ Inefficient: imports entire lodash library (~70KB)
import _ from 'lodash';
const sorted = _.sortBy(data, 'name');

// ✅ Efficient: import only needed function (~5KB)
import sortBy from 'lodash/sortBy';
const sorted = sortBy(data, 'name');
```

By importing only the functions you use, you can reduce bundle size by over 90% for that library.

## 2. Code Splitting and Lazy Loading

Don't load components during app startup if they're not immediately needed. React Native supports dynamic imports for this purpose.

### Using React.lazy with Suspense

```javascript
import React, { Suspense, lazy } from 'react';
import { View, ActivityIndicator } from 'react-native';

// Components are loaded on demand
const SettingsScreen = lazy(() => import('./screens/SettingsScreen'));
const AnalyticsScreen = lazy(() => import('./screens/AnalyticsScreen'));

// Fallback UI while loading
const LoadingSpinner = () => (
  <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
    <ActivityIndicator size="large" color="#007AFF" />
  </View>
);

export const AppNavigator = () => {
  return (
    <Suspense fallback={<LoadingSpinner />}>
      <SettingsScreen />
    </Suspense>
  );
};
```

### Lazy Loading with React Navigation

```javascript
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { lazy, Suspense } from 'react';

const Stack = createNativeStackNavigator();

// Define screens with lazy loading
const HomeScreen = lazy(() => import('./screens/HomeScreen'));
const DetailScreen = lazy(() => import('./screens/DetailScreen'));

export const RootNavigator = () => {
  return (
    <NavigationContainer>
      <Stack.Navigator
        screenOptions={{
          headerShown: true,
        }}
      >
        <Stack.Screen
          name="Home"
          component={() => (
            <Suspense fallback={<LoadingSpinner />}>
              <HomeScreen />
            </Suspense>
          )}
        />
        <Stack.Screen
          name="Detail"
          component={() => (
            <Suspense fallback={<LoadingSpinner />}>
              <DetailScreen />
            </Suspense>
          )}
        />
      </Stack.Navigator>
    </NavigationContainer>
  );
};
```

This approach defers loading of non-critical screens until they're actually needed, reducing initial startup time.

## 3. Memory Leak Prevention and Garbage Collection

Memory leaks gradually degrade app performance. Proper cleanup in useEffect is essential for long-term performance.

### Common Memory Leak Patterns and Solutions

```javascript
import { useEffect, useState, useRef } from 'react';
import { fetchUserData } from './api';

export const UserProfile = ({ userId }) => {
  const [user, setUser] = useState(null);
  const isMountedRef = useRef(true);

  useEffect(() => {
    isMountedRef.current = true;

    // Async operation
    fetchUserData(userId).then((data) => {
      // Prevent state updates if component unmounted
      if (isMountedRef.current) {
        setUser(data);
      }
    });

    // Cleanup function prevents memory leaks
    return () => {
      isMountedRef.current = false;
    };
  }, [userId]);

  return user ? <Text>{user.name}</Text> : <ActivityIndicator />;
};
```

### Proper EventEmitter Listener Management

```javascript
import { useEffect } from 'react';
import { AppState } from 'react-native';
import * as Analytics from 'expo-analytics';

export const useAppStateListener = () => {
  useEffect(() => {
    const subscription = AppState.addEventListener(
      'change',
      handleAppStateChange
    );

    function handleAppStateChange(state) {
      if (state === 'active') {
        Analytics.trackEvent('app_resumed');
      } else if (state === 'background') {
        Analytics.trackEvent('app_backgrounded');
      }
    }

    // ❌ Without cleanup: memory leak
    // ✅ Always remove listeners on unmount
    return () => {
      subscription.remove();
    };
  }, []);
};
```

## 4. Leveraging Native Modules

Offloading heavy computations to native code can dramatically improve performance.

### Using Expo Modules for Performance-Critical Operations

```javascript
// nativeModule.ts
import { NativeModule, requireNativeModule } from 'expo';

interface NativeModuleType {
  performHeavyComputation(data: string): Promise<string>;
}

const NativeModule = requireNativeModule<NativeModuleType>(
  'RorkMaxNativeModule'
);

export const useHeavyComputation = (data: string) => {
  const [result, setResult] = useState<string>('');

  useEffect(() => {
    NativeModule.performHeavyComputation(data).then(setResult);
  }, [data]);

  return result;
};
```

Executing heavy computations in native code prevents JavaScript thread blocking, keeping the UI responsive.

## 5. Image Optimization and Caching

Images often comprise a large portion of app bundle size. Optimization here yields significant returns.

### Choosing Optimal Image Formats

```javascript
import { Image } from 'react-native';
import FastImage from 'react-native-fast-image';

// ✅ FastImage includes intelligent caching
export const OptimizedImage = ({ uri, width = 100, height = 100 }) => {
  return (
    <FastImage
      source={{
        uri,
        priority: FastImage.priority.normal,
        cache: FastImage.cacheControl.immutable,
      }}
      style={{ width, height }}
      resizeMode={FastImage.resizeMode.contain}
    />
  );
};
```

### Embracing WebP Format

```bash
# Convert images to WebP format (30% smaller file sizes)
convert image.png -quality 80 image.webp
```

WebP provides significantly smaller file sizes than PNG—often 30% reduction—making it ideal for reducing launch time impact.

## 7. Optimizing Startup Screens

Utilize native splash screens to mask startup time and improve perceived performance.

```javascript
import * as SplashScreen from 'expo-splash-screen';
import { useEffect } from 'react';

// Keep splash screen visible while app initializes
SplashScreen.preventAutoHideAsync();

export const App = () => {
  const [appIsReady, setAppIsReady] = useState(false);

  useEffect(() => {
    const prepare = async () => {
      try {
        // Perform app initialization (API calls, data loading, etc.)
        await initializeApp();
      } catch (e) {
        console.warn(e);
      } finally {
        setAppIsReady(true);
      }
    };

    prepare();
  }, []);

  useEffect(() => {
    // Hide splash screen after initialization completes
    if (appIsReady) {
      SplashScreen.hideAsync();
    }
  }, [appIsReady]);

  return appIsReady ? <RootNavigator /> : null;
};
```

This approach keeps the splash screen visible during initialization, providing visual feedback to users and masking the actual startup time.

## 8. Bundle Size Monitoring and Performance Budgets

Continuous monitoring helps detect performance regressions early before they impact users.

```javascript
// build.js - Enforce bundle size limits during build
const fs = require('fs');
const path = require('path');

const checkBundleSize = () => {
  const bundlePath = path.join(__dirname, 'dist/main.jsbundle');
  const stats = fs.statSync(bundlePath);
  const sizeInMb = stats.size / 1024 / 1024;

  const MAX_BUNDLE_SIZE = 2.5; // MB

  if (sizeInMb > MAX_BUNDLE_SIZE) {
    throw new Error(
      `Bundle size exceeded! ${sizeInMb.toFixed(2)}MB > ${MAX_BUNDLE_SIZE}MB`
    );
  }

  console.log(`✅ Bundle size OK: ${sizeInMb.toFixed(2)}MB`);
};

checkBundleSize();
```
