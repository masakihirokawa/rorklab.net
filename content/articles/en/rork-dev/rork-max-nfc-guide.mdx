## Understanding NFC and Core NFC

NFC (Near Field Communication) operates at 13.56 MHz and enables data exchange between devices within a few centimeters. Apple has supported NFC reading since iPhone 7 (iOS 11) and added background tag reading on iPhone XS and later (iOS 13+).

**Core NFC** is Apple's framework for interacting with NFC tags from within your app. The primary data format is NDEF (NFC Data Exchange Format), which lets you store URLs, plain text, vCards, custom binary data, and more in standardized records.

Because Rork Max generates native Swift rather than a React Native wrapper, it can fully leverage Core NFC without any bridging layer or third-party library. The code you get is clean, idiomatic Swift — the same quality you'd write by hand in Xcode.

## Setting Up: Entitlements and Info.plist

Before writing any code, you need a couple of configuration items.

### Apple Developer Program

NFC write access requires the **Near Field Communication Tag Reading** entitlement, which Apple grants to registered developers. Sign up for the Apple Developer Program ($99/year) if you haven't already.

### Prompt Rork Max to Handle the Setup

The easiest way is to tell Rork Max upfront:

```
"Create an iOS NFC app using Core NFC.
 Add the Near Field Communication Tag Reading capability,
 and include the NFCReaderUsageDescription key in Info.plist
 with the message: 'Required to read NFC tags.'"
```

Rork Max will add the following to `Info.plist` automatically:

```xml
<!-- Info.plist entry required for NFC access -->
<key>NFCReaderUsageDescription</key>
<string>Required to read NFC tags.</string>
```

Without this key, the app will crash at runtime when it tries to start an NFC session. Keep the description clear and user-friendly — it appears in the system permission dialog.

## Implementing NFC Tag Reading

Here's the prompt to get a working NFC reader in Rork Max:

```
"Build a SwiftUI NFC tag reader app that:
 - Starts an NFC scan when a button is tapped
 - Reads NDEF messages and displays URLs and text on screen
 - Maintains a scrollable history of all scanned results
 - Shows an alert on error"
```

The ViewModel Rork Max generates follows this pattern:

```swift
import SwiftUI
import CoreNFC

// ViewModel that acts as the NFCNDEFReaderSession delegate
class NFCReaderViewModel: NSObject, ObservableObject, NFCNDEFReaderSessionDelegate {

    @Published var scannedMessages: [String] = []  // Scan history
    @Published var alertMessage: String = ""
    @Published var showAlert: Bool = false

    private var session: NFCNDEFReaderSession?

    // Call this from your SwiftUI button action
    func startScanning() {
        guard NFCNDEFReaderSession.readingAvailable else {
            alertMessage = "NFC is not available on this device."
            showAlert = true
            return
        }
        session = NFCNDEFReaderSession(
            delegate: self,
            queue: nil,
            invalidateAfterFirstRead: false
        )
        session?.alertMessage = "Hold your iPhone near an NFC tag."
        session?.begin()
    }

    // Called when one or more NDEF messages are detected
    func readerSession(_ session: NFCNDEFReaderSession, didDetectNDEFs messages: [NFCNDEFMessage]) {
        for message in messages {
            for record in message.records {
                // Decode payload as UTF-8 text
                // Expected output: e.g., "https://example.com" or "Hello, NFC!"
                if let text = String(data: record.payload, encoding: .utf8) {
                    DispatchQueue.main.async {
                        self.scannedMessages.append(text)
                    }
                }
            }
        }
    }

    // Handle session errors
    func readerSession(_ session: NFCNDEFReaderSession, didInvalidateWithError error: Error) {
        // Ignore expected invalidation codes (user canceled or tag already read)
        if let readerError = error as? NFCReaderError {
            let code = readerError.code
            if code == .readerSessionInvalidationErrorFirstNDEFTagRead ||
               code == .readerSessionInvalidationErrorUserCanceled {
                return
            }
        }
        DispatchQueue.main.async {
            self.alertMessage = error.localizedDescription
            self.showAlert = true
        }
    }
}
```

Attach this ViewModel to a SwiftUI `View` and call `startScanning()` from a button. Rork Max generates the full View scaffold automatically when you describe the UI in your prompt.

## Writing Data to NFC Tags

Reading is just the beginning. Here's how to write a URL to a blank NFC tag:

```swift
// Write a URL to an NFC tag using NFCNDEFTag
func writeURL(urlString: String) {
    writePayload = urlString  // Store the URL to write
    isWriteMode = true
    session = NFCNDEFReaderSession(delegate: self, queue: nil, invalidateAfterFirstRead: false)
    session?.alertMessage = "Hold your iPhone near the NFC tag to write."
    session?.begin()
}

// Called when a writable NFC tag is detected
func readerSession(_ session: NFCNDEFReaderSession, didDetect tags: [NFCNDEFTag]) {
    guard let tag = tags.first else { return }

    // Connect to the tag
    session.connect(to: tag) { error in
        guard error == nil else {
            session.invalidate(errorMessage: "Could not connect to tag.")
            return
        }

        // Build a well-known URI NDEF record
        guard let url = URL(string: self.writePayload),
              let record = NFCNDEFPayload.wellKnownTypeURIPayload(url: url) else {
            session.invalidate(errorMessage: "Invalid URL format.")
            return
        }

        let message = NFCNDEFMessage(records: [record])

        // Write the NDEF message to the tag
        tag.writeNDEF(message) { error in
            if let error = error {
                // Common failure: tag is read-only or locked
                session.invalidate(errorMessage: "Write failed: \(error.localizedDescription)")
            } else {
                // Success — the URL is now stored on the tag
                session.alertMessage = "✅ Tag written successfully!"
                session.invalidate()
            }
        }
    }
}
```

Once written, any NFC-capable phone (iOS or Android) can read the tag and open the URL without needing your app installed.

## Troubleshooting Common Errors

### `readingAvailable` returns `false`

NFC is not available on iPhone 6s or earlier, or in the Simulator. Always test on a physical device. Set your deployment target to iOS 13.0 or later to access the full Core NFC API.

### `NFCReaderErrorUnsupportedFeature` when writing

Tag writing requires the NFC entitlement in your Provisioning Profile. In Rork Max, add this line to your prompt: *"Enable the NFC Tag Reading entitlement for write access."* Rork Max will add the capability to the project, and you'll need to regenerate your Provisioning Profile in the Apple Developer Portal.

### Payload decoding returns garbled text

Different `typeNameFormat` values require different decoding strategies. URLs use `wellKnownTypeURIPayload`; plain text uses `wellKnownTypeTextPayload(locale:)`. Tell Rork Max: *"Parse both URL and plain text NDEF records with proper type detection."*

## FAQ

**Q: Where can I buy NFC tags for testing?**

A: Search for "NTAG215 NFC stickers" on Amazon or AliExpress. NTAG213, NTAG215, and NTAG216 are all widely compatible with iOS. Prices start at a few cents per tag in bulk. For most apps, NTAG213 (144 bytes) is sufficient for URLs and short text.

**Q: Can Android phones read tags written by a Rork Max app?**

A: Yes. NDEF-formatted tags are a cross-platform standard. A URL or text written by your Rork Max app will open correctly on any NFC-capable Android device. The tag stores the data — not the app — so no special software is needed to read it.

**Q: How much data can an NFC tag hold?**

A: Storage depends on the tag type: NTAG213 holds 144 bytes, NTAG215 holds 504 bytes, and NTAG216 holds 888 bytes. For large data (images, long JSON), store the content in the cloud and write only a short URL or ID to the tag — a common and effective pattern.

**Q: How does background NFC tag reading work?**

A: On iPhone XS and later (iOS 13+), the system can read NDEF URL records in the background without your app being open. It uses Universal Links to route the user to your app. To set this up in Rork Max, include this in your prompt: *"Support background NFC tag reading using Universal Links."* For a full walkthrough, see [Rork Max Deep Linking & Universal Links Guide](/articles/rork-dev/rork-max-deep-linking-universal-links).
