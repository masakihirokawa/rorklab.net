## Why Your App Needs Data Visualization

Displaying raw numbers in a text list doesn't cut it for modern mobile apps. Whether you're building a personal finance tracker, a fitness app, or a business analytics tool, your users need to *see* their data — not just read it. A well-designed chart turns a wall of numbers into an actionable insight in a single glance.

In this guide, you'll learn how to add interactive charts and graphs to your Rork app using **Victory Native**, one of the most popular charting libraries in the React Native ecosystem. We'll build three chart types — line, bar, and pie — and combine them into a fully functional dashboard screen with touch-based tooltips.

**Who this guide is for:**

- Developers comfortable with Rork basics (you've completed [Getting Started with Rork](/articles/rork-basics/getting-started))
- Anyone building an app that displays data — analytics dashboards, trackers, reports
- Developers who want reusable chart components they can drop into any screen

**What you'll need:**

- Rork (free plan works; Rork Max recommended)
- Node.js 18+
- Basic familiarity with React Native and Expo

## What Is Victory Native?

Victory Native is an open-source charting library built by Formidable. It's the React Native counterpart of the popular Victory library for the web, optimized for mobile performance.

**Why Victory Native stands out:**

- Performance-optimized for React Native (with optional Skia rendering)
- Rich chart types: line, bar, pie, scatter, area, and more
- Built-in touch interactions and animations
- Full TypeScript support
- Excellent Expo compatibility

Let's start by adding it to your project.

```bash
# Install Victory Native and its peer dependency
npx expo install victory-native react-native-svg

# Optional: enable Skia rendering for better performance
npx expo install @shopify/react-native-skia
```

## Building a Line Chart — Visualizing Trends Over Time

Line charts are the bread and butter of any dashboard. Let's build one that displays monthly revenue data with interactive tooltips.

```tsx
// components/SalesLineChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import {
  VictoryChart,
  VictoryLine,
  VictoryAxis,
  VictoryTheme,
  VictoryVoronoiContainer,
  VictoryTooltip,
} from "victory-native";

// Monthly revenue data (in thousands of dollars)
const salesData = [
  { month: "Jan", sales: 12.0 },
  { month: "Feb", sales: 14.5 },
  { month: "Mar", sales: 13.2 },
  { month: "Apr", sales: 17.8 },
  { month: "May", sales: 19.5 },
  { month: "Jun", sales: 21.0 },
];

export default function SalesLineChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Monthly Revenue</Text>
      <VictoryChart
        theme={VictoryTheme.material}
        height={280}
        // VoronoiContainer enables touch-based tooltips
        containerComponent={
          <VictoryVoronoiContainer
            labels={({ datum }) => `$${datum.sales}K`}
            labelComponent={<VictoryTooltip />}
          />
        }
      >
        {/* X-axis: month labels */}
        <VictoryAxis
          tickValues={salesData.map((_, i) => i + 1)}
          tickFormat={salesData.map((d) => d.month)}
          style={{ tickLabels: { fontSize: 11 } }}
        />
        {/* Y-axis: revenue */}
        <VictoryAxis
          dependentAxis
          tickFormat={(t: number) => `$${t}K`}
          style={{ tickLabels: { fontSize: 11 } }}
        />
        {/* The line itself */}
        <VictoryLine
          data={salesData}
          x={(d) => salesData.indexOf(d) + 1}
          y="sales"
          style={{
            data: { stroke: "#4F46E5", strokeWidth: 3 },
          }}
          animate={{ duration: 500 }}
        />
      </VictoryChart>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12 },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8 },
});

// Expected output:
// A line chart showing Jan–Jun revenue trend
// Tapping anywhere shows a tooltip with the nearest data point value
```

The key here is `VictoryVoronoiContainer`. It automatically detects the nearest data point to the user's touch and displays a tooltip — no manual hit-testing needed.

## Building a Bar Chart — Comparing Categories

Bar charts are perfect for side-by-side comparisons. Here's one that shows app downloads broken down by category.

```tsx
// components/DownloadsBarChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import {
  VictoryChart,
  VictoryBar,
  VictoryAxis,
  VictoryTheme,
} from "victory-native";

const downloadData = [
  { category: "Health", downloads: 3200 },
  { category: "Education", downloads: 2800 },
  { category: "Finance", downloads: 4100 },
  { category: "Gaming", downloads: 5600 },
  { category: "Social", downloads: 3900 },
];

export default function DownloadsBarChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Downloads by Category</Text>
      <VictoryChart
        theme={VictoryTheme.material}
        height={280}
        domainPadding={{ x: 30 }}
      >
        <VictoryAxis
          tickFormat={downloadData.map((d) => d.category)}
          style={{ tickLabels: { fontSize: 10, angle: -20 } }}
        />
        <VictoryAxis
          dependentAxis
          tickFormat={(t: number) =>
            t >= 1000 ? `${(t / 1000).toFixed(1)}K` : String(t)
          }
        />
        <VictoryBar
          data={downloadData}
          x="category"
          y="downloads"
          style={{
            data: {
              fill: ({ datum }) =>
                datum.downloads > 4000 ? "#10B981" : "#6366F1",
              borderRadius: 4,
            },
          }}
          animate={{ duration: 600, onLoad: { duration: 400 } }}
          barWidth={36}
        />
      </VictoryChart>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12 },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8 },
});

// Expected output:
// 5 bars showing download counts per category
// Bars exceeding 4000 downloads are green; others are purple
```

By passing a function to `fill`, you can conditionally color bars based on their values. This kind of visual threshold feedback is a powerful UX pattern for dashboards — users can instantly spot which categories are outperforming.

## Building a Pie Chart — Showing Proportions

Pie charts (and their donut variant) are ideal for visualizing how parts contribute to a whole — revenue breakdown, user demographics, traffic sources.

```tsx
// components/RevenuePieChart.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";
import { VictoryPie, VictoryLabel } from "victory-native";

const revenueData = [
  { source: "Subscriptions", revenue: 45, color: "#6366F1" },
  { source: "Ads", revenue: 25, color: "#10B981" },
  { source: "In-App", revenue: 20, color: "#F59E0B" },
  { source: "Other", revenue: 10, color: "#94A3B8" },
];

export default function RevenuePieChart() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Revenue Breakdown</Text>
      <VictoryPie
        data={revenueData}
        x="source"
        y="revenue"
        colorScale={revenueData.map((d) => d.color)}
        labels={({ datum }) => `${datum.source}\n${datum.revenue}%`}
        labelComponent={
          <VictoryLabel style={{ fontSize: 11, fill: "#374151" }} />
        }
        innerRadius={60}
        padAngle={2}
        animate={{ duration: 500 }}
        height={280}
        width={320}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { padding: 16, backgroundColor: "#fff", borderRadius: 12, alignItems: "center" },
  title: { fontSize: 16, fontWeight: "700", marginBottom: 8, alignSelf: "flex-start" },
});

// Expected output:
// A donut chart showing four revenue segments
// Each segment is labeled with its source and percentage
```

Setting `innerRadius` creates the donut hole in the center. A common design pattern is to place a summary KPI (like total revenue) inside that empty space for maximum information density.

## Assembling the Dashboard Screen

With the individual chart components ready, let's bring them together into a scrollable dashboard screen, complete with KPI summary cards at the top.

```tsx
// screens/DashboardScreen.tsx
import React from "react";
import { ScrollView, View, Text, StyleSheet } from "react-native";
import SalesLineChart from "../components/SalesLineChart";
import DownloadsBarChart from "../components/DownloadsBarChart";
import RevenuePieChart from "../components/RevenuePieChart";

function KpiCard({ label, value, trend }: {
  label: string;
  value: string;
  trend: string;
}) {
  const isPositive = trend.startsWith("+");
  return (
    <View style={styles.kpiCard}>
      <Text style={styles.kpiLabel}>{label}</Text>
      <Text style={styles.kpiValue}>{value}</Text>
      <Text style={[styles.kpiTrend, { color: isPositive ? "#10B981" : "#EF4444" }]}>
        {trend}
      </Text>
    </View>
  );
}

export default function DashboardScreen() {
  return (
    <ScrollView style={styles.screen} contentContainerStyle={styles.content}>
      <Text style={styles.heading}>Dashboard</Text>

      {/* KPI summary cards */}
      <View style={styles.kpiRow}>
        <KpiCard label="Revenue" value="$21.0K" trend="+8.2%" />
        <KpiCard label="DAU" value="12,340" trend="+3.1%" />
        <KpiCard label="Churn" value="2.4%" trend="-0.5%" />
      </View>

      {/* Chart sections */}
      <SalesLineChart />
      <View style={styles.spacer} />
      <DownloadsBarChart />
      <View style={styles.spacer} />
      <RevenuePieChart />
    </ScrollView>
  );
}

const styles = StyleSheet.create({
  screen: { flex: 1, backgroundColor: "#F3F4F6" },
  content: { padding: 16, paddingBottom: 40 },
  heading: { fontSize: 24, fontWeight: "800", marginBottom: 16 },
  kpiRow: { flexDirection: "row", justifyContent: "space-between", marginBottom: 16 },
  kpiCard: { flex: 1, backgroundColor: "#fff", borderRadius: 12, padding: 12, marginHorizontal: 4 },
  kpiLabel: { fontSize: 12, color: "#6B7280" },
  kpiValue: { fontSize: 20, fontWeight: "700", marginTop: 4 },
  kpiTrend: { fontSize: 12, marginTop: 2 },
  spacer: { height: 16 },
});

// Expected output:
// A scrollable dashboard with 3 KPI cards and 3 chart sections
```

## Connecting to Real-Time Data with Supabase

Static data is great for prototyping, but production apps need live data. Here's how to fetch chart data from Supabase and keep it in sync using real-time subscriptions. For a deeper dive into Supabase integration, see [Rork × Supabase Auth & Realtime Guide](/articles/rork-dev/rork-supabase-auth-realtime-guide).

```tsx
// hooks/useSalesData.ts — custom hook for fetching chart data
import { useState, useEffect } from "react";
import { supabase } from "../lib/supabase";

interface SalesRecord {
  month: string;
  sales: number;
}

export function useSalesData() {
  const [data, setData] = useState<SalesRecord[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetchSales() {
      const { data: records, error } = await supabase
        .from("monthly_sales")
        .select("month, sales")
        .order("month", { ascending: true });

      if (!error && records) {
        setData(records);
      }
      setLoading(false);
    }

    fetchSales();

    // Subscribe to real-time changes
    const subscription = supabase
      .channel("sales-updates")
      .on(
        "postgres_changes",
        { event: "*", schema: "public", table: "monthly_sales" },
        () => fetchSales() // re-fetch on any change
      )
      .subscribe();

    return () => {
      subscription.unsubscribe();
    };
  }, []);

  return { data, loading };
}

// Usage:
// const { data, loading } = useSalesData();
// if (loading) return <ActivityIndicator />;
// <VictoryLine data={data} x="month" y="sales" />
```

For more on structuring reusable data-fetching hooks like this, check out [Mastering Custom Hooks in Rork Apps](/articles/rork-dev/rork-custom-hooks-reusable-logic).

## Common Errors and How to Fix Them

### Charts aren't rendering at all

This usually means `react-native-svg` isn't installed. Run `npx expo install react-native-svg` and restart your dev server. If you're using Expo Go, note that some SVG features are limited — consider switching to a development build (`npx expo prebuild`).

### Touch events aren't working inside ScrollView

When charts are nested inside a `ScrollView`, scroll gestures can intercept chart touch events. Wrap your `VictoryChart` in a `<View onStartShouldSetResponder={() => true}>` to give the chart priority for touch handling.

### Performance drops with large datasets

If you're rendering more than 1,000 data points, either switch to Skia rendering (`@shopify/react-native-skia`) or aggregate your data (e.g., daily → monthly) before passing it to the chart component. Victory Native handles hundreds of points smoothly, but thousands will cause noticeable jank on mid-range devices.

## FAQ

### Is Victory Native free to use?

Yes. Victory Native is open-source under the MIT license, which means you can use it in commercial apps at no cost. It works with Rork's free plan as well.

### Can I use other charting libraries with Rork?

Absolutely. Alternatives include react-native-chart-kit, react-native-gifted-charts, and react-native-wagmi-charts. That said, Victory Native offers the best balance of documentation quality, TypeScript support, and customization depth, making it the go-to choice for most use cases.

### How do I match chart colors to my app's theme?

Instead of using `VictoryTheme.material`, create a custom theme object. You can control fonts, colors, axis styles, and more. A good starting point is to copy the `VictoryTheme.material` source and adjust it to match your design system.
