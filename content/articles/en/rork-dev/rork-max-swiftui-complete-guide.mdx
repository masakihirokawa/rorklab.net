## Understanding Rork Max's SwiftUI Generation Architecture

### How It Works

Rork Max follows a proven pipeline:

1. **UI Recognition**: Analyzes your sketch and text description
2. **Component Decomposition**: Breaks UI into reusable SwiftUI structures
3. **State Management**: Automatically chooses between `@State`, `@StateObject`, and `@EnvironmentObject`
4. **Code Generation**: Outputs iOS 17/18–compliant SwiftUI matching Apple's best practices
5. **Live Preview**: Renders immediately in Xcode or the Rork preview pane

The quality of generated code depends almost entirely on prompt clarity. A vague request produces mediocre code. A detailed specification produces production-ready code.

### What Affects Code Quality

- Prompt specificity (the primary factor)
- UI complexity (more complex UIs need clearer breakdown)
- Use of technical terminology (mentioning SwiftUI patterns directly)
- Explicit state management requirements

## Advanced Prompt Techniques

### Technique 1: Explicit Component Decomposition

**Weak prompt**:
```
Create a user profile screen
```

**Strong prompt**:
```
Create a user profile screen with these independent components:

- ProfileHeaderView: displays user photo (circular, 120×120), name, bio, and follow button
- StatisticsView: shows followers, posts, and likes in a 3-column layout with SF Symbols icons
- BioView: renders multi-line bio text (max 150 chars) with line breaks preserved
- ActionButtonsView: message, share, and report buttons in equal-width HStack with 10pt spacing

Each component must be a separate View struct. ProfileView should compose them using VStack with 16pt spacing.
```

Explicit decomposition yields code that's testable, reusable, and maintainable.

### Technique 2: State Management Specification

**Weak prompt**:
```
Add search functionality to the product list
```

**Strong prompt**:
```
Create a SearchViewModel using @StateObject with these published properties:
- searchText: String (empty initially)
- results: [Product] (initially empty)
- isLoading: Bool
- error: String?

When searchText changes, wait 0.5 seconds (debounce) then call searchAPI(). While loading, show a ProgressView. Display error message if API fails.
```

This specificity ensures the generated code follows MVVM pattern and handles edge cases.

### Technique 3: Animation and Transition Details

**Weak prompt**:
```
Show modal when button is tapped
```

**Strong prompt**:
```
When the button is tapped, present DetailView as a modal with:
- Transition: .scale + .opacity (0.3s duration)
- Overlay: semi-transparent black background (0.3 opacity)
- Close button: top-right corner, ✕ icon, white color
- Auto-dismiss: 3 seconds after appearing (DispatchQueue.main.asyncAfter)

Use @State var showDetail to control presentation.
```

This level of detail produces polished, production-grade UX.

## Integrating Advanced APIs

### HealthKit Integration

Build a fitness app that reads health data:

```swift
import HealthKit

class HealthKitManager: NSObject, ObservableObject {
    let healthStore = HKHealthStore()
    @Published var stepCount: Int = 0
    @Published var heartRate: Int = 0

    func requestAuthorization() {
        let typesToShare: Set<HKSampleType> = []
        let typesToRead: Set<HKObjectType> = [
            HKObjectType.quantityType(forIdentifier: .stepCount)\!,
            HKObjectType.quantityType(forIdentifier: .heartRate)\!
        ]

        healthStore.requestAuthorization(toShare: typesToShare, read: typesToRead) { success, error in
            if success {
                self.fetchStepCount()
                self.fetchHeartRate()
            }
        }
    }

    func fetchStepCount() {
        let stepType = HKObjectType.quantityType(forIdentifier: .stepCount)\!
        let query = HKStatisticsQuery(
            quantityType: stepType,
            quantitySamplePredicate: nil,
            options: .cumulativeSum
        ) { _, result, _ in
            DispatchQueue.main.async {
                self.stepCount = Int(result?.sumQuantity()?.doubleValue(for: HKUnit.count()) ?? 0)
            }
        }
        healthStore.execute(query)
    }
}
```

### Core ML Model Integration

Embed a machine learning model for image classification:

```swift
import Vision
import CoreML

class MLModelManager {
    let model = try\! YourCoreMLModel(configuration: .init())

    func classifyImage(_ image: UIImage) -> String? {
        guard let ciImage = CIImage(image: image) else { return nil }

        let request = VNCoreMLRequest(
            model: try\! VNCoreMLModel(for: model.model)
        ) { request, error in
            if let results = request.results as? [VNClassificationObservation],
               let topResult = results.first {
                print("Classification: \(topResult.identifier) (\(topResult.confidence * 100)%)")
            }
        }

        let handler = VNImageRequestHandler(ciImage: ciImage)
        try? handler.perform([request])
        return nil
    }
}
```

### StoreKit 2 for In-App Purchases

Implement modern in-app purchasing:

```swift
import StoreKit

class PurchaseManager: NSObject, ObservableObject {
    @Published var unlockedFeatures: Set<String> = []

    func purchasePremium() async {
        do {
            let products = try await Product.products(for: ["com.yourapp.premium"])
            guard let product = products.first else { return }

            let result = try await product.purchase()

            switch result {
            case .success(let verification):
                if case .verified(let transaction) = verification {
                    unlockedFeatures.insert(product.id)
                    await transaction.finish()
                }
            case .userCancelled, .pending:
                break
            @unknown default:
                break
            }
        } catch {
            print("Purchase failed: \(error)")
        }
    }
}
```

## Performance Optimization

### Smooth Scrolling (60fps)

For large lists, optimize rendering:

```swift
struct OptimizedListView: View {
    @State var items: [Item] = (0..<1000).map { Item(id: $0) }

    var body: some View {
        List {
            ForEach(items, id: \.id) { item in
                ItemRow(item: item)
                    .id(item.id)  // Critical for scroll performance
            }
        }
        .listStyle(.plain)
    }
}

struct ItemRow: View {
    let item: Item

    var body: some View {
        HStack {
            Image(systemName: "star.fill")
            VStack(alignment: .leading) {
                Text(item.title).font(.headline)
                Text(item.description)
                    .font(.caption)
                    .lineLimit(1)
            }
        }
        .frame(height: 60)
    }
}
```

## App Store Optimization (ASO) Playbook

### Screenshot Strategy

Screenshots are your first impression. Use this order:

1. **Screenshot 1**: Hero shot of main feature (no text)
2. **Screenshot 2**: One key benefit with icon + text (max 20 chars)
3. **Screenshot 3**: Multiple features as bullet list
4. **Screenshots 4–5**: Real-world usage scenarios

**Text guidelines**:
- Font size: 28pt minimum
- Color: High contrast (WCAG AAA)
- Content: Max 3 lines, 20 characters per line
- Position: Bottom half (don't obscure navigation)

**Exact dimensions**:
- iPhone 15 Pro Max: 1290×2796 px
- iPhone 15: 1179×2556 px
- iPad Pro 12.9": 2048×2732 px

### Keyword Strategy

Use App Store keyword field (100 characters) strategically:

```
Primary: Fitness Tracker, Workouts
Secondary: Health, Exercise, Weight Loss
Long-tail: Free Fitness App, Apple Watch
```

**Tool**: Use Keyword Tool or App Annie to balance search volume vs. competition.

### Subtitle and Description

**Subtitle (30 chars max)**:
```
AI fitness coach for your goals
```

**Description (4,000 chars max)**:
```
Key Features
✓ AI-Powered Workouts: Personalized plans
✓ Watch Integration: Apple Watch, Garmin
✓ Food Logging: Track nutrition daily
✓ Community: Motivate friends

Why Users Love It
"Lost 5kg in 3 months" — Sarah, 32
"My daily habit now" — Mike, 28

Privacy First
Your Apple Health data is encrypted. We never store it on servers.
```

**Optimization tips**:
- Open with core feature in first 2 lines
- Use 3–5 strategic emojis
- Include numbers, testimonials, evidence
- Localize per market (don't just translate)

### A/B Testing Your Metadata

Use App Store Connect experiments:

1. **Test 1**: Subtitle A vs. B
   - A: "Fitness Tracker"
   - B: "AI Fitness Coach"
2. **Duration**: 2+ weeks
3. **Metric**: Conversion rate (detailed page views → downloads)
4. **Winner**: Adopt the subtitle with higher conversion

## 30-Point App Store Approval Checklist

Complete all 30 items before submission. This ensures first-try approval.

### Functionality (15 items)
- [ ] App doesn't crash on launch
- [ ] All buttons and links work
- [ ] Background features function (music, location)
- [ ] Network errors handled gracefully
- [ ] Works offline where applicable
- [ ] No memory leaks or battery drain
- [ ] Database operations work (create, read, update, delete)
- [ ] File upload/download functions properly
- [ ] Multi-language support correct
- [ ] VoiceOver and accessibility features work
- [ ] Permissions requests are appropriate
- [ ] In-app purchases and payments work
- [ ] Widgets function correctly (if present)
- [ ] iPad and Apple Watch display correctly
- [ ] Rotation (portrait/landscape) works

### Metadata (10 items)
- [ ] App icon: 1024×1024px, solid background, no rounded corners
- [ ] Screenshots: correct resolution (e.g., 1290×2796px), 5–10 images
- [ ] Screenshots: no external tools or ads visible
- [ ] App name: brand + feature descriptor (max 30 chars)
- [ ] Subtitle: compelling and concise (max 30 chars)
- [ ] Description: clear, error-free, professional tone
- [ ] Keywords: relevant and not misleading
- [ ] Support URL: valid, no 404 errors
- [ ] Privacy policy URL: valid and aligns with app features
- [ ] Preview video (if included): max 30s, H.264, shows real usage

### Compliance (5 items)
- [ ] No illegal content (copyright, etc.)
- [ ] No excessive ads or pop-ups
- [ ] Doesn't collect/share user data without permission
- [ ] No fraud or phishing
- [ ] Doesn't duplicate Apple's basic functionality

This checklist is your path to a first-submission approval.

## Complete Example: Production Fitness App

Here's a minimal, production-ready fitness app combining all concepts:

```swift
// ViewModel
@MainActor
final class FitnessViewModel: ObservableObject {
    @Published var workouts: [Workout] = []
    @Published var isLoading = false

    func fetchWorkouts() async {
        isLoading = true
        let newWorkouts = await WorkoutAPI.fetchAll()
        self.workouts = newWorkouts
        isLoading = false
    }
}

// Main View
struct FitnessAppView: View {
    @StateObject private var viewModel = FitnessViewModel()

    var body: some View {
        NavigationStack {
            VStack {
                if viewModel.isLoading {
                    ProgressView()
                } else {
                    List(viewModel.workouts, id: \.id) { workout in
                        NavigationLink(destination: WorkoutDetailView(workout: workout)) {
                            WorkoutRowView(workout: workout)
                        }
                    }
                }
            }
            .navigationTitle("My Workouts")
            .task {
                await viewModel.fetchWorkouts()
            }
        }
    }
}
```
