## What Is an AI Stylist App?

We've all stood in front of our closet wondering "What should I wear today?" An AI stylist app solves this by letting users photograph and catalog their wardrobe, then getting AI-powered outfit recommendations based on weather, occasion, and personal style.

With Rork, you can build an app that combines image recognition and AI recommendation without writing much code from scratch. In this article, we'll walk through creating a fully functional AI stylist app step by step using Rork's prompt-driven development.

## Capturing Clothing and Image Recognition

Implement a system where the camera captures clothing and AI automatically categorizes items into tops, bottoms, outerwear, shoes, and accessories.

```typescript
// types/wardrobe.ts
export type ClothingCategory =
  | "tops"
  | "bottoms"
  | "outerwear"
  | "shoes"
  | "accessories";

export type ClothingColor =
  | "black" | "white" | "gray" | "navy"
  | "red" | "blue" | "green" | "brown"
  | "beige" | "pink" | "other";

export interface ClothingItem {
  id: string;
  imageUri: string;
  category: ClothingCategory;
  color: ClothingColor;
  tags: string[];        // casual, formal, sporty, etc.
  addedAt: string;
  wearCount: number;
  lastWornAt?: string;
}
```

```typescript
// utils/clothing-classifier.ts
// Auto-classify clothing using Gemini API
export async function classifyClothing(
  imageBase64: string
): Promise<{
  category: ClothingCategory;
  color: ClothingColor;
  tags: string[];
}> {
  const response = await fetch(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_GEMINI_API_KEY",
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{
          parts: [
            {
              inlineData: {
                mimeType: "image/jpeg",
                data: imageBase64,
              },
            },
            {
              text: `Analyze this clothing image and respond in the following JSON format only.
{
  "category": "tops" | "bottoms" | "outerwear" | "shoes" | "accessories",
  "color": "black" | "white" | "gray" | "navy" | "red" | "blue" | "green" | "brown" | "beige" | "pink" | "other",
  "tags": ["casual", "formal", "sporty" — up to 3 style tags]
}
Do not include any text outside the JSON.`,
            },
          ],
        }],
      }),
    }
  );

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;
  return JSON.parse(text);
}
```

The key here is leveraging Gemini's multimodal API to automatically extract category, color, and style tags from clothing images. Users simply take a photo — no tedious manual input needed.

## Wardrobe Data Management

Store registered clothing data locally with AsyncStorage. You can prompt Rork to generate custom hooks like this.

```typescript
// hooks/useWardrobe.ts
import { useState, useEffect, useCallback } from "react";
import AsyncStorage from "@react-native-async-storage/async-storage";
import { ClothingItem, ClothingCategory } from "@/types/wardrobe";

const STORAGE_KEY = "@wardrobe_items";

export function useWardrobe() {
  const [items, setItems] = useState<ClothingItem[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    loadItems();
  }, []);

  const loadItems = async () => {
    try {
      const stored = await AsyncStorage.getItem(STORAGE_KEY);
      if (stored) setItems(JSON.parse(stored));
    } catch (error) {
      console.error("Failed to load wardrobe:", error);
    } finally {
      setLoading(false);
    }
  };

  const addItem = useCallback(async (item: ClothingItem) => {
    const updated = [...items, item];
    setItems(updated);
    await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(updated));
  }, [items]);

  const getByCategory = useCallback(
    (category: ClothingCategory) =>
      items.filter((item) => item.category === category),
    [items]
  );

  const recordWear = useCallback(async (id: string) => {
    const updated = items.map((item) =>
      item.id === id
        ? {
            ...item,
            wearCount: item.wearCount + 1,
            lastWornAt: new Date().toISOString(),
          }
        : item
    );
    setItems(updated);
    await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(updated));
  }, [items]);

  return { items, loading, addItem, getByCategory, recordWear };
}
```

## AI Outfit Recommendation Engine

Build the core feature: combining wardrobe items into outfits that match specified conditions.

```typescript
// utils/outfit-recommender.ts
import { ClothingItem } from "@/types/wardrobe";

interface OutfitRequest {
  occasion: "casual" | "business" | "date" | "sports";
  weather: "hot" | "warm" | "cool" | "cold";
  preference?: string; // Additional user requests
}

export interface OutfitSuggestion {
  items: ClothingItem[];
  reasoning: string;
  styleScore: number; // 1-10
}

export async function generateOutfit(
  wardrobe: ClothingItem[],
  request: OutfitRequest
): Promise<OutfitSuggestion[]> {
  const wardrobeDesc = wardrobe.map((item) =>
    `[ID:${item.id}] ${item.category} - ${item.color} (${item.tags.join(", ")})`
  ).join("\n");

  const response = await fetch(
    "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=YOUR_GEMINI_API_KEY",
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        contents: [{
          parts: [{
            text: `You are a professional stylist.
From the following wardrobe, suggest 3 outfits matching the conditions.

Wardrobe:
${wardrobeDesc}

Conditions:
- Occasion: ${request.occasion}
- Weather: ${request.weather}
${request.preference ? `- Extra request: ${request.preference}` : ""}

Respond as a JSON array:
[{
  "item_ids": ["array of item IDs to use"],
  "reasoning": "Why this outfit works (in English)",
  "style_score": 8
}]`,
          }],
        }],
      }),
    }
  );

  const data = await response.json();
  const text = data.candidates[0].content.parts[0].text;
  const suggestions = JSON.parse(text);

  return suggestions.map((s: { item_ids: string[]; reasoning: string; style_score: number }) => ({
    items: s.item_ids
      .map((id: string) => wardrobe.find((w) => w.id === id))
      .filter(Boolean) as ClothingItem[],
    reasoning: s.reasoning,
    styleScore: s.style_score,
  }));
}
```

## FAQ

**Q: How accurate is the image recognition?**

Using Gemini 2.0 Flash's multimodal capabilities, you can expect over 90% accuracy for standard clothing category classification. However, items with similar color tones or complex patterns may occasionally be misclassified, so it's best to include a UI that lets users manually correct the AI's classification.

**Q: Does it work offline?**

Closet data and past outfit results are stored locally in AsyncStorage, so browsing is possible offline. However, auto-classifying new items and generating outfit recommendations requires an API connection.

**Q: How much does the API cost?**

Gemini 2.0 Flash has very low per-image costs, and personal use typically fits within the free tier. If you're generating around 10 outfit recommendations per day, the monthly API cost is essentially zero.

**Q: How long does it take to build this app with Rork?**

For core functionality (item registration, grid view, AI outfit suggestions), you can have a working prototype in 2-3 hours using Rork's prompt-driven development. Including UI polish and edge case handling, you can finish within a single day.
