## Part 1: Native Integration Architecture Fundamentals

### Expo Modules and the Bridge Layer

Rork Max internally uses the Expo Modules framework to bridge JavaScript calls to native code (Swift/Kotlin). Understanding this layer is critical:

```
┌─────────────────────────────────────────┐
│  Rork Max UI Builder (JavaScript Code)   │
└────────────────┬────────────────────────┘
                 │ useNativeModule() hook
┌────────────────▼────────────────────────┐
│  Expo Modules Bridge (Auto-generated)    │
└────────────────┬────────────────────────┘
    ┌───────────▼───────────┐
    │  iOS Native (Swift)    │  Android Native (Kotlin)
    │  ・HealthKit          │  ・Health Connect API
    │  ・AVFoundation        │  ・CameraX / Camera2
    │  ・CoreML              │  ・TensorFlow Lite
    │  ・ARKit               │  ・ARCore
    └───────────────────────┘
```

**Critical Point**: iOS and Android concepts differ fundamentally. It's not just "API calls"—each platform has distinct `PermissionScope` and `Callback` patterns that must be mastered separately.

### Type-Safe Bridge Definition in TypeScript

Expo Modules requires TypeScript type definitions that align with native implementations:

```typescript
// Corresponds to ios/ExampleNativeModule.swift + android/ExampleNativeModule.kt
export interface NativeModuleConfig {
  getDeviceHealth: {
    params: {
      dataType: "steps" | "heart_rate" | "calories";
      fromDate: number; // Unix timestamp (ms)
      toDate: number;
    };
    returns: {
      value: number;
      unit: string;
      timestamp: number;
    }[];
  };
}

// ←→ Swift definition
// func getDeviceHealth(dataType: String, fromDate: Double, toDate: Double) -> [[String: Any]]
```

**Implementation Tips**:
- Map TypeScript types 1:1 with Swift/Kotlin types
- Make null/undefined handling explicit
- Segment large data structures (iOS/Android have messaging size limits)

## Part 2: HealthKit Integration (iOS Healthcare Core)

### HealthKit Permission Request Flow

The most common failure point in iOS health apps is incorrect permission request timing. This flow must be **exactly** implemented:

```
User launches app
  ↓
Check if HealthKit available at app launch
  ├─ Not Determined (never asked)
  │  └→ Call requestAuthorization()
  │     ↓
  │     OS shows permission dialog (multi-select data types)
  │     ↓
  │     Authorized / Denied / Restricted
  │
  ├─ Authorized (already granted)
  │  └→ Immediately call HKHealthStore.execute(query)
  │
  └─ Denied (already rejected)
     └→ Redirect to Settings (URL Scheme: `prefs://Health`)
```

**Complete Working Implementation (Swift)**:

```swift
// ios/RorkHealthModule.swift
import HealthKit

class RorkHealthModule: Module {
  let healthStore = HKHealthStore()
  
  func requestHealthAuthorization() throws -> [String: Any] {
    guard HKHealthStore.isHealthDataAvailable() else {
      throw NSError(domain: "HealthKit", code: 1, userInfo: [NSLocalizedDescriptionKey: "HealthKit not available"])
    }
    
    let allTypes: Set<HKSampleType> = [
      HKObjectType.quantityType(forIdentifier: .stepCount)\!,
      HKObjectType.quantityType(forIdentifier: .heartRate)\!,
      HKObjectType.quantityType(forIdentifier: .activeEnergyBurned)\!,
    ]
    
    var authStatus: [String: String] = [:]
    
    healthStore.requestAuthorization(toShare: nil, read: allTypes) { success, error in
      if success {
        for sampleType in allTypes {
          let status = self.healthStore.authorizationStatus(for: sampleType)
          authStatus[sampleType.identifier] = self.statusString(status)
        }
      }
    }
    
    return ["authorized": \!authStatus.values.contains("Denied")]
  }
  
  func getStepCount(fromDate: Double, toDate: Double) throws -> [[String: Any]] {
    let predicate = HKQuery.predicateForSamples(withStart: Date(timeIntervalSince1970: fromDate / 1000),
                                                end: Date(timeIntervalSince1970: toDate / 1000),
                                                options: .strictStartDate)
    
    let query = HKSampleQuery(
      sampleType: HKObjectType.quantityType(forIdentifier: .stepCount)\!,
      predicate: predicate,
      limit: HKObjectQueryNoLimit,
      sortDescriptors: [NSSortDescriptor(key: HKSampleSortIdentifierStartDate, ascending: true)]
    ) { _, samples, error in
      // Implementation continues...
    }
    
    healthStore.execute(query)
    return [] // Implementation example
  }
  
  private func statusString(_ status: HKAuthorizationStatus) -> String {
    switch status {
    case .notDetermined: return "NotDetermined"
    case .denied: return "Denied"
    case .authorized: return "Authorized"
    @unknown default: return "Unknown"
    }
  }
}
```

### Common Pitfalls: Version Compatibility and Memory Leaks

**Pitfall 1: iOS 16+ HKSampleQuery Deprecation**

iOS 16 deprecated `HKSampleQuery` with no official replacement until iOS 16.4. Always use version guards:

```swift
// ❌ Old approach (iOS 15 and earlier)
let query = HKSampleQuery(sampleType: ..., predicate: ..., limit: ...)

// ✅ Modern approach (iOS 16+)
if #available(iOS 16.0, *) {
  let predicate = HKSampleQueryDescriptor(
    sampleType: HKObjectType.quantityType(forIdentifier: .stepCount)\!,
    predicate: HKQuery.predicateForSamples(withStart: startDate, end: endDate, options: [])
  )
  let results = try await healthStore.samples(for: predicate)
} else {
  // Fallback to HKSampleQuery
}
```

**Pitfall 2: Memory Leak After Query Execution**

HealthKit queries that don't complete cleanup leak memory, especially in background execution:

```swift
// ❌ Bad: Memory leaks
func runHealthQuery() {
  let query = HKSampleQuery(...)
  healthStore.execute(query) // Query starts
  // ... function ends without cleanup
  // Memory leaks, background execution continues indefinitely
}

// ✅ Good: Proper cleanup
func runHealthQuery() async {
  let query = HKSampleQuery(...)
  var observations: [HKSample] = []
  
  let semaphore = DispatchSemaphore(value: 0)
  healthStore.execute(query) { _, samples, error in
    observations = samples ?? []
    semaphore.signal() // Signal completion
  }
  semaphore.wait()
  // ← Function exits only after query completion
}
```

## Part 3: CameraX Integration (Android Camera)

### CameraX: Escaping the Callback Pyramid

Android camera processing is complex. Direct `Camera2 API` usage leads to callback hell, but **CameraX** simplifies this dramatically:

```kotlin
// android/RorkCameraModule.kt
import androidx.camera.core.Camera
import androidx.camera.core.CameraSelector
import androidx.camera.core.ImageCapture
import androidx.camera.core.Preview
import androidx.camera.lifecycle.ProcessCameraProvider
import androidx.camera.video.VideoCapture
import androidx.camera.video.Recorder
import androidx.camera.video.QualitySelector

class RorkCameraModule : Module() {
  private lateinit var cameraProvider: ProcessCameraProvider
  private var imageCapture: ImageCapture? = null
  
  suspend fun startCameraPreview(): Map<String, Any> {
    val cameraProvider = ProcessCameraProvider.getInstance().await()
    
    val preview = Preview.Builder().build().also {
      it.setSurfaceProvider(surfaceProvider) // Rork's surface
    }
    
    val imageCapture = ImageCapture.Builder()
      .setTargetResolution(android.util.Size(1920, 1080))
      .setFlashMode(ImageCapture.FLASH_MODE_AUTO)
      .build()
    
    val cameraSelector = CameraSelector.DEFAULT_BACK_CAMERA
    
    try {
      cameraProvider.unbindAll()
      cameraProvider.bindToLifecycle(lifecycleOwner, cameraSelector, preview, imageCapture)
      this.imageCapture = imageCapture
      return mapOf("status" to "preview_started")
    } catch (e: Exception) {
      return mapOf("error" to e.message)
    }
  }
  
  fun captureImage(savePath: String): Map<String, Any> {
    val imageCapture = imageCapture ?: return mapOf("error" to "Camera not initialized")
    
    val outputOptions = ImageCapture.OutputFileOptions
      .Builder(java.io.File(savePath))
      .build()
    
    imageCapture.takePicture(
      outputOptions,
      ContextCompat.getMainExecutor(context),
      object : ImageCapture.OnImageSavedCallback {
        override fun onImageSaved(outputFileResults: ImageCapture.OutputFileResults) {
          // Success → return file path to JavaScript
        }
        
        override fun onError(exception: ImageCaptureException) {
          // Error handling
        }
      }
    )
    
    return mapOf("status" to "capture_started")
  }
}
```

### Permission Handling (Android 14+ Partial Access)

Android 14 introduced granular photo/video access—a new permission pattern:

```kotlin
// AndroidManifest.xml
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" /> <\!-- API 33+ -->
<uses-permission android:name="android.permission.READ_MEDIA_VIDEO" />

// Runtime permission request
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
  // API 33+: Request READ_MEDIA_IMAGES and READ_MEDIA_VIDEO separately
  requestPermissions(
    arrayOf(
      android.Manifest.permission.CAMERA,
      android.Manifest.permission.READ_MEDIA_IMAGES
    ),
    CAMERA_PERMISSION_CODE
  )
} else {
  // API 32 and below: Use READ_EXTERNAL_STORAGE
  requestPermissions(
    arrayOf(
      android.Manifest.permission.CAMERA,
      android.Manifest.permission.READ_EXTERNAL_STORAGE
    ),
    CAMERA_PERMISSION_CODE
  )
}
```

## Part 4: AR/ML Integration Patterns

### iOS: ARKit + CoreML (Visual AI)

Combining ARKit and CoreML enables real-time object recognition with AR overlays—powerful for furniture placement simulators, medical diagnostics, and spatial computing apps.

```swift
// ios/RorkARMLModule.swift
import ARKit
import Vision
import CoreML

class RorkARMLModule: Module {
  var arSession: ARSession = ARSession()
  var mlModel: VNCoreMLModel?
  
  func startARSession() throws -> [String: Any] {
    guard ARWorldTrackingConfiguration.isSupported else {
      throw NSError(domain: "ARKit", code: 1, userInfo: [NSLocalizedDescriptionKey: "AR not supported"])
    }
    
    let configuration = ARWorldTrackingConfiguration()
    configuration.planeDetection = [.horizontal, .vertical]
    configuration.frameSemantics.insert(.personSegmentationWithDepth)
    arSession.run(configuration)
    
    // Load CoreML model (example: YOLO v8 object detection)
    let modelUrl = Bundle.main.url(forResource: "YOLOv8", withExtension: "mlmodelc")\!
    mlModel = try VNCoreMLModel(for: MLModel(contentsOf: modelUrl))
    
    return ["status": "ar_session_started"]
  }
  
  func detectObjectsInFrame(imageBuffer: CVImageBuffer) -> [[String: Any]] {
    guard let mlModel = mlModel else { return [] }
    
    let request = VNCoreMLRequest(model: mlModel) { request, error in
      if let error = error {
        print("Detection error:", error)
        return
      }
      
      let results = request.results as? [VNRecognizedObjectObservation] ?? []
      // Overlay detection results in AR space (add Node)
    }
    
    request.imageCropAndScaleOption = .scaleFill
    
    let handler = VNImageRequestHandler(cvPixelBuffer: imageBuffer, options: [:])
    try? handler.perform([request])
    
    return [] // Detection results (AR Node positions)
  }
}
```

**Critical**: CoreML inference on GPU is performance-critical. To maintain 60 FPS, keep inference time under 16ms. For slower models, run inference every second (30 FPS) and cache results between frames.

### Android: ML Kit + ARCore

Google ML Kit + ARCore is the Android standard for vision AI:

```kotlin
// android/RorkARMLModule.kt
import com.google.mlkit.vision.common.InputImage
import com.google.mlkit.vision.objects.ObjectDetection
import com.google.ar.core.Session
import com.google.ar.core.Anchor

class RorkARMLModule : Module() {
  private val objectDetector = ObjectDetection.getClient(ObjectDetectorOptions.DEFAULT)
  private lateinit var arSession: Session
  
  suspend fun detectObjectsAndPlaceAnchors(imageBytes: ByteArray): List<Map<String, Any>> {
    val inputImage = InputImage.fromByteArray(
      imageBytes,
      1920, // width
      1080, // height
      InputImage.IMAGE_FORMAT_NV21
    )
    
    val detectionTask = objectDetector.process(inputImage)
    val detectionResults = Tasks.await(detectionTask)
    
    val anchors = mutableListOf<Map<String, Any>>()
    for (detection in detectionResults) {
      val boundingBox = detection.boundingBox
      // Calculate 3D position in AR space corresponding to bounding box
      val hitTest = arSession.hitTest(
        (boundingBox.centerX()),
        (boundingBox.centerY())
      )
      
      if (hitTest.isNotEmpty()) {
        val anchor = hitTest[0].createAnchor()
        anchors.add(mapOf(
          "label" to (detection.labels.firstOrNull()?.text ?: "unknown"),
          "confidence" to (detection.labels.firstOrNull()?.confidence ?: 0f),
          "anchorId" to anchor.nativeHandle
        ))
      }
    }
    
    return anchors
  }
}
```

## Part 5: NFC and Payment Integration

### iOS: Core NFC (Apple Pay Integration)

NFC (Near Field Communication) enables reading payment cards and digital passes:

```swift
// ios/RorkNFCModule.swift
import CoreNFC

class RorkNFCModule: Module, NFCNDEFReaderSessionDelegate {
  var nfcSession: NFCNDEFReaderSession?
  
  func startNFCReading() throws -> [String: Any] {
    guard NFCNDEFReaderSession.readingAvailable else {
      throw NSError(domain: "NFC", code: 1, userInfo: [NSLocalizedDescriptionKey: "NFC not available"])
    }
    
    nfcSession = NFCNDEFReaderSession(delegate: self, queue: .main, invalidateAfterFirstRead: false)
    nfcSession?.alertMessage = "Hold iPhone near tag"
    nfcSession?.begin()
    
    return ["status": "nfc_reading_started"]
  }
  
  func readerSession(
    _ session: NFCNDEFReaderSession,
    didDetectNDEFs messages: [NFCNDEFMessage]
  ) {
    var recordsData: [[String: Any]] = []
    
    for message in messages {
      for record in message.records {
        recordsData.append([
          "type": String(data: record.type, encoding: .utf8) ?? "",
          "payload": record.payload.base64EncodedString(),
          "id": record.identifier.base64EncodedString()
        ])
      }
    }
    
    // Return data to JavaScript (sendEvent)
  }
  
  func readerSession(_ session: NFCNDEFReaderSession, didInvalidateWithError error: Error) {
    print("NFC session error:", error)
  }
}
```

## Part 7: Production Considerations

### Memory and Battery Optimization

HealthKit, CameraX, and AR modules consume significant memory and battery. In production:

1. **Limit background execution**: Use `PermittedBackgroundTaskScheduler` for specific time windows only
2. **Leverage caching**: Hourly data sync suffices; avoid 1-second intervals
3. **Monitor battery level**: Reduce update frequency in low-power mode

```typescript
import { useBattery } from "expo-device";

export function OptimizedHealthModule() {
  const { batteryLevel, lowPowerMode } = useBattery();
  
  React.useEffect(() => {
    const interval = lowPowerMode ? 300000 : 60000; // 5min vs 1min
    const timer = setInterval(() => {
      RorkHealthModule.syncHealthData();
    }, interval);
    
    return () => clearInterval(timer);
  }, [lowPowerMode]);
}
```

## Frequently Asked Questions (FAQ)

**Q1: Must I learn Swift/Kotlin to create native modules?**

A: Basic syntax understanding (variables, types, methods) is sufficient. ChatGPT/Claude handle complex questions. However, understand memory management concepts (Swift ARC, Kotlin GC) thoroughly.

**Q2: Can I share code between iOS and Android?**

A: Cross-platform frameworks (C++) exist theoretically, but are not recommended in Rork Max. Platform-optimized implementations are more maintainable.

**Q3: Will Apple/Google reject my app for using native modules?**

A: Legitimate use cases (HealthKit=health, Camera=photos, NFC=payments) pass review. Clearly explain necessity in the review description.

**Q4: How do I switch code between test and production?**

A: Branch on `__DEV__`:

```typescript
if (__DEV__) {
  // Test mock
  setSteps(10000);
} else {
  // Production
  const steps = await RorkHealthModule.getStepCount(fromDate, toDate);
  setSteps(steps);
}
```

**Q5: Which tools detect memory leaks?**

A: Xcode Memory Debugger (iOS) and Android Studio Memory Profiler. Check memory graphs regularly, especially after background execution.

```typescript
export function FitnessGate() {
  const [authorized, setAuthorized] = React.useState(false);
  
  React.useEffect(() => {
    (async () => {
      if (Platform.OS === "ios") {
        const result = await RorkFitnessModule.requestHealthAuthorization();

        const result = await RorkFitnessModule.requestGoogleFitAuth();
        setAuthorized(result.authorized);
      }

// src/useFitnessData.ts
export function useFitnessData() {
  const [data, setData] = React.useState({ steps: 0, heart_rate: 0, calories: 0 });
  
  React.useEffect(() => {
    const interval = setInterval(async () => {
      const fromDate = new Date();
      fromDate.setHours(0, 0, 0, 0);
      
      const result = Platform.OS === "ios"
        ? await RorkFitnessModule.getTodayStats(fromDate.getTime())
        : await RorkFitnessModule.getGoogleFitStats(fromDate.getTime());
      
      setData(result);
      
      // Detect goal achievement and notify
      if (result.steps >= 10000 && data.steps < 10000) {
        await notificationService.sendNotification({
          title: "Goal Reached\!",
          body: "You've achieved your daily step goal"
        });
      }
    }, 60000); // Sync every 1 minute
    
    return () => clearInterval(interval);
  }, [data.steps]);
  
  return data;
}
```
