## Introduction — Why Lottie Transforms Your Rork App

First impressions in mobile apps aren't just about features — they're about *feel*. A silky loading animation, a welcoming onboarding experience, the satisfying burst of a payment confirmation — these small moments define whether users stick around or walk away.

**Lottie** is an open-source animation library originally developed by Airbnb. It renders Adobe After Effects animations exported as JSON files natively on iOS, Android, and the web. Thanks to the `lottie-react-native` package, you can bring this same quality into any Rork-built app.

In this guide, we'll walk through integrating Lottie into a Rork project from scratch — no prior animation experience required.

What you'll learn:

- Installing and configuring `lottie-react-native`
- Finding and placing Lottie JSON animation files
- Building loading screens, onboarding flows, and success animations
- Controlling playback speed, looping, and event callbacks
- Troubleshooting common errors

## What Is Lottie? — Vector Animations Powered by JSON

Lottie renders After Effects animations exported via the `Bodymovin` plugin as native vector graphics. Unlike GIF or PNG sprite sheets, Lottie animations are resolution-independent — they look crisp on every screen, from a compact iPhone SE to the largest iPad Pro.

Key advantages at a glance:

- **Lightweight**: Lottie files are often 90%+ smaller than equivalent GIFs
- **Scalable**: Vector-based rendering eliminates pixelation at any size
- **Interactive**: Control speed, direction, looping, and frame position programmatically
- **Vast library**: [LottieFiles](https://lottiefiles.com/) hosts tens of thousands of free and premium animations

For Rork apps built on Expo Managed Workflow, we'll use `lottie-react-native` v7 (compatible with Expo SDK 50 and later).

## Setup — Installing lottie-react-native

Since Rork projects are based on Expo Managed Workflow, installation is straightforward. You can even let Rork's AI handle it — just send a prompt like:

```
Add a loading animation using lottie-react-native to the home screen.
```

For manual installation, run the following in your project terminal:

```bash
# Install lottie-react-native via Expo
npx expo install lottie-react-native
```

After installation, your `package.json` will include:

```json
{
  "dependencies": {
    "lottie-react-native": "^7.0.0"
  }
}
```

Restart the development server after installation (`npx expo start --clear`) to ensure the new package is picked up.

## Getting Animation Files

You'll need `.json` animation files to display. The easiest place to find high-quality, free animations is [LottieFiles](https://lottiefiles.com/). Create a free account, search for animations using keywords like "loading," "success," or "onboarding," and download them in **Lottie JSON** format.

Place the downloaded files in an `assets/animations/` folder in your project:

```
my-app/
├── assets/
│   └── animations/
│       ├── loading.json      ← for loading states
│       ├── success.json      ← for success/completion screens
│       └── onboarding.json   ← for onboarding slides
├── components/
└── app/
```

## Basic Implementation — Displaying Your First Animation

Here's the simplest possible implementation using the `LottieView` component:

```tsx
import LottieView from "lottie-react-native";
import { View, StyleSheet } from "react-native";

// A simple looping loading animation component
export default function LoadingScreen() {
  return (
    <View style={styles.container}>
      <LottieView
        source={require("../assets/animations/loading.json")} // path to your animation file
        autoPlay  // start playing immediately
        loop      // loop indefinitely
        style={styles.animation}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: "center",
    alignItems: "center",
    backgroundColor: "#ffffff",
  },
  animation: {
    width: 200,  // animation width in pixels
    height: 200, // animation height in pixels
  },
});
```

With just `autoPlay` and `loop`, the animation plays automatically and repeats. This is perfect as a loading spinner while waiting for API data.

## Three Real-World Use Cases

### Use Case 1 — Loading State While Fetching Data

Show an animation during async operations and switch to content once data arrives.

```tsx
import React, { useState, useEffect } from "react";
import LottieView from "lottie-react-native";
import { View, Text, StyleSheet } from "react-native";

export default function DataScreen() {
  const [isLoading, setIsLoading] = useState(true);
  const [data, setData] = useState(null);

  useEffect(() => {
    // Replace this with your actual API call
    setTimeout(() => {
      setData({ message: "Data loaded successfully!" });
      setIsLoading(false); // hide the animation when done
    }, 2000);
  }, []);

  if (isLoading) {
    return (
      <View style={styles.center}>
        <LottieView
          source={require("../assets/animations/loading.json")}
          autoPlay
          loop
          style={styles.lottie}
        />
        <Text style={styles.text}>Loading your data...</Text>
      </View>
    );
  }

  return (
    <View style={styles.center}>
      <Text>{data?.message}</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  center: { flex: 1, justifyContent: "center", alignItems: "center" },
  lottie: { width: 150, height: 150 },
  text: { marginTop: 12, color: "#666" },
});
```

### Use Case 2 — Success/Completion Animation

Celebrate moments like payment confirmation, task completion, or goal achievement. Play the animation once, then trigger a callback.

```tsx
import React, { useRef } from "react";
import LottieView from "lottie-react-native";
import { View, Text, StyleSheet } from "react-native";

export default function SuccessScreen({ onDone }: { onDone: () => void }) {
  const animationRef = useRef<LottieView>(null);

  return (
    <View style={styles.container}>
      <LottieView
        ref={animationRef}
        source={require("../assets/animations/success.json")}
        autoPlay
        loop={false}         // play once and stop
        onAnimationFinish={onDone}  // callback when animation ends
        style={styles.lottie}
      />
      <Text style={styles.title}>Purchase Complete!</Text>
      <Text style={styles.subtitle}>Thank you for your order</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: "center", alignItems: "center", padding: 24 },
  lottie: { width: 200, height: 200 },
  title: { fontSize: 24, fontWeight: "bold", marginTop: 16 },
  subtitle: { fontSize: 16, color: "#666", marginTop: 8 },
});
```

### Use Case 3 — Animated Onboarding Slides

Use Lottie to give each onboarding slide a distinct personality. Switching animations per slide creates a polished, branded experience.

```tsx
import React, { useState } from "react";
import LottieView from "lottie-react-native";
import { View, Text, TouchableOpacity, StyleSheet } from "react-native";

const slides = [
  {
    id: "1",
    animation: require("../assets/animations/slide1.json"),
    title: "AI Builds Your App",
    subtitle: "Just describe your idea — Rork writes the code for you",
  },
  {
    id: "2",
    animation: require("../assets/animations/slide2.json"),
    title: "Publish in 2 Clicks",
    subtitle: "No Xcode needed. Cloud Mac handles building and submission",
  },
];

export default function OnboardingScreen({ onFinish }: { onFinish: () => void }) {
  const [currentIndex, setCurrentIndex] = useState(0);
  const current = slides[currentIndex];

  return (
    <View style={styles.container}>
      <LottieView
        key={current.id}  // remount on slide change to replay animation
        source={current.animation}
        autoPlay
        loop
        style={styles.lottie}
      />
      <Text style={styles.title}>{current.title}</Text>
      <Text style={styles.subtitle}>{current.subtitle}</Text>
      <TouchableOpacity
        style={styles.button}
        onPress={() => {
          if (currentIndex < slides.length - 1) {
            setCurrentIndex(currentIndex + 1);
          } else {
            onFinish();
          }
        }}
      >
        <Text style={styles.buttonText}>
          {currentIndex < slides.length - 1 ? "Next" : "Get Started"}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, justifyContent: "center", alignItems: "center", padding: 32 },
  lottie: { width: 280, height: 280 },
  title: { fontSize: 22, fontWeight: "bold", textAlign: "center", marginTop: 24 },
  subtitle: { fontSize: 15, color: "#555", textAlign: "center", marginTop: 12, lineHeight: 22 },
  button: { marginTop: 32, backgroundColor: "#6C63FF", paddingVertical: 14, paddingHorizontal: 48, borderRadius: 30 },
  buttonText: { color: "#fff", fontSize: 16, fontWeight: "bold" },
});
```

## Common Errors and Fixes

**Error 1: Animation isn't visible (blank screen)**

The most common cause is a missing `width` and `height` in the `style` prop. Lottie doesn't infer dimensions automatically.

```tsx
// ❌ Wrong — no size specified
<LottieView source={...} autoPlay loop />

// ✅ Correct — explicit size
<LottieView source={...} autoPlay loop style={{ width: 200, height: 200 }} />
```

**Error 2: `Unable to resolve module 'lottie-react-native'`**

Clear the Expo cache and restart:

```bash
npx expo start --clear
```

If that doesn't work, delete `node_modules` and reinstall.

**Error 3: Animation stops partway through**

The JSON file may be corrupted, or it may contain After Effects features not supported by the Lottie renderer. Always preview animations on LottieFiles before downloading.

**Error 4: Animation looks different on iOS vs Android**

Some After Effects effects (blur, glow, etc.) render differently across platforms. Stick to simple vector animations and test on both platforms during development.

## Advanced — Lottie + Reanimated for Interactive Animations

Lottie alone covers most use cases, but pairing it with `react-native-reanimated` unlocks scroll-driven and gesture-driven animations — where the animation's progress is tied directly to user input.

Here's a conceptual example of driving a Lottie animation with scroll position:

```tsx
import LottieView from "lottie-react-native";
import Animated, { useSharedValue, interpolate } from "react-native-reanimated";

// Map scroll position (0–300px) to animation progress (0–1)
const AnimatedLottie = Animated.createAnimatedComponent(LottieView);

function ScrollDrivenAnimation({ scrollY }: { scrollY: Animated.SharedValue<number> }) {
  const progress = interpolate(scrollY.value, [0, 300], [0, 1]);

  return (
    <AnimatedLottie
      source={require("../assets/animations/reveal.json")}
      progress={progress} // animation frame updates as user scrolls
      style={{ width: 200, height: 200 }}
    />
  );
}
```

## FAQ

**Q1. Where can I find free Lottie animation files?**

[LottieFiles](https://lottiefiles.com/) is the go-to resource. A free account gives access to thousands of animations. Always check the license before using in a commercial app — many are free for commercial use, but some require attribution.

**Q2. What's a good file size limit for Lottie animations?**

Aim for **under 50KB** per animation file. Complex After Effects compositions can balloon to several megabytes and will noticeably impact memory usage and startup time. When in doubt, choose simpler animations.

**Q3. Can Rork's AI generate Lottie integration code automatically?**

Yes. Rork understands `lottie-react-native` and can scaffold loading screens, success animations, and onboarding flows based on a prompt. Provide the path to your JSON file and describe what you want — the AI handles the rest.

**Q4. Do I need to test on both iOS and Android?**

Yes, especially for complex animations. Some After Effects features render differently per platform. Use Expo Go for quick real-device testing on both iOS and Android before submitting to the stores.
