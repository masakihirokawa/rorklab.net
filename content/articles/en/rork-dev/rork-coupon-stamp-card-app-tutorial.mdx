## Introduction — Why Build a Coupon & Stamp Card App?

The demand for digitizing customer loyalty programs has been steadily growing. Transitioning from paper stamp cards to a mobile app enables businesses to boost repeat customer rates and leverage data analytics. With Rork's AI-powered code generation, you can efficiently build an app with production-ready features like QR code scanning, point management, and push notifications.

In this tutorial, we'll walk through building a complete coupon and stamp card app from scratch using Rork, complete with code examples and practical guidance. This article is designed for developers who are familiar with Rork's basics and want to build a real-world retail application.

If you're new to Rork, start with [Build Your First App in 30 Minutes with Rork](/articles/rork-basics/rork-first-app-30min) to get up to speed.

## App Architecture and Data Structure

Let's start by mapping out the overall architecture. A coupon and stamp card app needs these core screens:

- **Home Screen**: Displays current stamp count and progress toward the next reward
- **QR Scan Screen**: Reads QR codes placed at stores to add stamps
- **Coupon List Screen**: Shows available coupons and usage history
- **Profile Screen**: User information and notification settings

For the data layer, we'll define these schemas to guide Rork's code generation:

```typescript
// User's stamp card information
interface StampCard {
  id: string;
  userId: string;
  shopId: string;
  currentStamps: number;  // Current stamp count
  totalStamps: number;    // Lifetime total stamps
  createdAt: Date;
  updatedAt: Date;
}

// Coupon information
interface Coupon {
  id: string;
  title: string;
  description: string;
  requiredStamps: number; // Stamps needed to unlock
  discount: string;       // e.g., "10% OFF", "Free drink"
  expiresAt: Date;
  isUsed: boolean;
}

// Stamp history (when and where each stamp was earned)
interface StampHistory {
  id: string;
  cardId: string;
  stampCount: number;
  scannedAt: Date;
  shopName: string;
}
```

Including this data structure in your Rork prompt helps the AI understand the complete data flow and generate screens that work together coherently.

## Implementing QR Code Scanning

The heart of any stamp card app is the QR code scanner. We'll combine `expo-camera` with barcode detection to create a smooth scanning experience.

Here's how to prompt Rork:

```text
Create a QR code scanning screen.
- Use expo-camera to display a camera preview
- Auto-detect and read QR codes
- Show a stamp-earned animation on successful scan
- Prevent duplicate scans of the same QR code (30-second cooldown)
- Display a permission request screen if camera access is not granted
```

The core component Rork generates will look something like this:

```typescript
import { CameraView, useCameraPermissions } from 'expo-camera';
import { useState, useRef } from 'react';
import { View, Text, StyleSheet, Alert, Animated } from 'react-native';

export default function QRScanScreen() {
  const [permission, requestPermission] = useCameraPermissions();
  const [scanned, setScanned] = useState(false);
  const [lastScannedCode, setLastScannedCode] = useState('');
  const scaleAnim = useRef(new Animated.Value(0)).current;

  // Handler for when a barcode is scanned
  const handleBarcodeScanned = ({ type, data }: { type: string; data: string }) => {
    if (scanned || data === lastScannedCode) return;

    setScanned(true);
    setLastScannedCode(data);

    // Add stamp
    addStamp(data);

    // Success animation
    Animated.spring(scaleAnim, {
      toValue: 1,
      friction: 3,
      useNativeDriver: true,
    }).start();

    // Reset after 30 seconds (prevent duplicate scans)
    setTimeout(() => {
      setScanned(false);
      setLastScannedCode('');
      scaleAnim.setValue(0);
    }, 30000);
  };

  // Camera permission check
  if (!permission?.granted) {
    return (
      <View style={styles.container}>
        <Text style={styles.message}>
          Camera access is required to earn stamps
        </Text>
        {/* Permission request button */}
      </View>
    );
  }

  return (
    <View style={styles.container}>
      <CameraView
        style={styles.camera}
        barcodeScannerSettings={{
          barcodeTypes: ['qr'],
        }}
        onBarcodeScanned={scanned ? undefined : handleBarcodeScanned}
      />
      {/* Scan guide overlay */}
      <View style={styles.overlay}>
        <View style={styles.scanArea} />
        <Text style={styles.hint}>Align the QR code within the frame</Text>
      </View>
    </View>
  );
}
// Expected output: A camera preview with a QR code scan area overlay.
// When a QR code is detected, stamps are automatically added.
```

## Designing the Stamp Card UI with Animations

The visual design of the stamp card directly impacts how motivated users feel to keep coming back. The key is making it immediately obvious how many stamps remain until the next reward.

```typescript
import { View, Text, StyleSheet } from 'react-native';
import { Ionicons } from '@expo/vector-icons';

interface StampCardProps {
  currentStamps: number;
  maxStamps: number;    // Stamps required for reward (e.g., 10)
  rewardTitle: string;  // Reward name (e.g., "Free drink")
}

export function StampCardView({ currentStamps, maxStamps, rewardTitle }: StampCardProps) {
  const stamps = Array.from({ length: maxStamps }, (_, i) => i < currentStamps);

  return (
    <View style={styles.card}>
      <Text style={styles.shopName}>CAFE DOLICE</Text>
      <Text style={styles.rewardText}>
        {maxStamps - currentStamps} more until {rewardTitle}
      </Text>

      <View style={styles.stampGrid}>
        {stamps.map((filled, index) => (
          <View key={index} style={styles.stampSlot}>
            {filled ? (
              <Ionicons name="checkmark-circle" size={36} color="#FF6B35" />
            ) : (
              <Ionicons name="ellipse-outline" size={36} color="#DDD" />
            )}
            <Text style={styles.stampNumber}>{index + 1}</Text>
          </View>
        ))}
      </View>

      <View style={styles.progressBar}>
        <View
          style={[
            styles.progressFill,
            { width: `${(currentStamps / maxStamps) * 100}%` },
          ]}
        />
      </View>
    </View>
  );
}
// Expected output: A grid of 10 stamp slots where earned stamps
// display as orange checkmarks and empty slots as gray circles,
// with a progress bar below showing completion percentage.
```

When prompting Rork, adding instructions like "use a warm cafe-style design" or "add a bounce animation when a stamp is earned" will result in more polished and engaging UI components.

## Coupon Management with Local Storage

Next, let's implement automatic coupon issuance when stamps reach a threshold. AsyncStorage handles the local data persistence layer.

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

const STAMP_KEY = '@stamp_card';
const COUPONS_KEY = '@coupons';

// Add a stamp and issue a coupon if the reward threshold is met
export async function addStampAndCheckReward(shopQRData: string): Promise<{
  newStampCount: number;
  couponIssued: boolean;
  coupon?: Coupon;
}> {
  try {
    // Get current stamp count
    const cardData = await AsyncStorage.getItem(STAMP_KEY);
    const card: StampCard = cardData
      ? JSON.parse(cardData)
      : { currentStamps: 0, totalStamps: 0 };

    // Increment stamps
    card.currentStamps += 1;
    card.totalStamps += 1;
    card.updatedAt = new Date().toISOString();

    let couponIssued = false;
    let newCoupon: Coupon | undefined;

    // Issue a coupon when 10 stamps are collected
    if (card.currentStamps >= 10) {
      newCoupon = {
        id: `coupon_${Date.now()}`,
        title: 'Free Drink Coupon',
        description: 'Enjoy any drink on the house',
        requiredStamps: 10,
        discount: 'One free drink',
        expiresAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000).toISOString(),
        isUsed: false,
      };

      // Save coupon
      const existingCoupons = await AsyncStorage.getItem(COUPONS_KEY);
      const coupons = existingCoupons ? JSON.parse(existingCoupons) : [];
      coupons.push(newCoupon);
      await AsyncStorage.setItem(COUPONS_KEY, JSON.stringify(coupons));

      // Reset stamp count
      card.currentStamps = 0;
      couponIssued = true;
    }

    await AsyncStorage.setItem(STAMP_KEY, JSON.stringify(card));

    return { newStampCount: card.currentStamps, couponIssued, coupon: newCoupon };
  } catch (error) {
    console.error('Error adding stamp:', error);
    throw error;
  }
}
// Expected output: When stamps reach 10, a coupon is automatically
// issued and the stamp count resets to 0.
```

## Push Notifications for Coupon Expiry Reminders

Sending push notifications when coupons are about to expire significantly improves redemption rates. Here's how to implement scheduled reminders using Expo's notification API:

```typescript
import * as Notifications from 'expo-notifications';

// Configure notification behavior
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: true,
  }),
});

// Schedule a reminder notification before coupon expiry
export async function scheduleCouponReminder(coupon: Coupon) {
  const expiresAt = new Date(coupon.expiresAt);
  const reminderDate = new Date(expiresAt.getTime() - 3 * 24 * 60 * 60 * 1000);
  // Notify 3 days before expiry

  if (reminderDate > new Date()) {
    await Notifications.scheduleNotificationAsync({
      content: {
        title: 'Your coupon expires soon!',
        body: `"${coupon.title}" expires in 3 days. Use it before it's gone!`,
        data: { couponId: coupon.id },
      },
      trigger: {
        type: Notifications.SchedulableTriggerInputTypes.DATE,
        date: reminderDate,
      },
    });
  }
}

// Send an immediate notification when a stamp is earned
export async function notifyStampEarned(currentCount: number, maxCount: number) {
  await Notifications.scheduleNotificationAsync({
    content: {
      title: 'Stamp earned!',
      body: `You now have ${currentCount}/${maxCount} stamps. ${maxCount - currentCount} more to go for your reward!`,
    },
    trigger: null, // Immediate notification
  });
}
// Expected output: When a coupon is issued, a reminder is automatically
// scheduled for 3 days before expiry. Stamp earnings trigger immediate
// progress notifications.
```

## Frequently Asked Questions

### Can I generate QR codes on the app side for stores?

Absolutely. For generating QR codes at the store level, you can use the `react-native-qrcode-svg` library. It dynamically creates QR codes containing the shop ID and timestamp, which can be displayed on screen or exported for printing. Simply prompt Rork with "add a QR code generation feature to the store admin screen" and it will handle the library setup and UI implementation.

### Will stamps work in offline environments?

Since we're using AsyncStorage for local data persistence, stamps can be added even without an internet connection. The QR code scanning itself relies solely on the camera and doesn't require connectivity. If you're syncing with a backend, add an automatic sync mechanism that pushes pending changes when the device comes back online.

### How do I support multiple stores?

Use the `shopId` field in the data structure to manage separate stamp cards for each store. On the home screen, a horizontally scrollable carousel is an intuitive way to let users switch between stores. Prompt Rork with "make stamp cards switchable via a carousel for multiple stores" and it will generate the appropriate UI.

### What are the monetization options for this type of app?

The most common model is a B2B SaaS subscription — store owners pay a monthly fee to use the platform while end users (customers) use it for free. For implementing subscription billing with Stripe, see [Rork x Stripe Subscription Implementation Guide](/articles/rork-business/rork-stripe-subscription-monetization-guide) for a comprehensive walkthrough.
