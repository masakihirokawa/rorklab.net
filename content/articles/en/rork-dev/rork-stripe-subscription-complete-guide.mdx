## Introduction — The First Step to App Monetization

You've built an app, but figuring out how to monetize it is a challenge many indie developers face. Ad-based models produce unpredictable revenue, and one-time purchases make it difficult to build a sustainable income stream.

Subscription billing is one of the most effective ways to build a stable revenue foundation. And with Rork, integrating Stripe-powered subscriptions into your app is surprisingly straightforward.

This article covers the entire process — from creating a Rork project and configuring Stripe, to implementing the billing flow and verifying test payments.

## Prerequisites and Setup

### What You'll Need

Before diving in, make sure you have the following ready.

- **Rork account**: Create a free account at [rork.com](https://rork.com)
- **Stripe account**: Sign up at [stripe.com](https://stripe.com) (start in test mode)
- **Node.js 18+**: For local development
- **Expo Go app**: For testing on physical devices (iOS / Android)

### Rork Project Setup

```bash
# Create a new project with Rork CLI
npx create-rork-app my-subscription-app
cd my-subscription-app

# Install Stripe packages
npm install @stripe/stripe-react-native
npm install stripe  # Server-side

# Create environment variables file
cat > .env << EOF
STRIPE_PUBLISHABLE_KEY=pk_test_xxxxxxxxxxxxx
STRIPE_SECRET_KEY=sk_test_xxxxxxxxxxxxx
STRIPE_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxx
EOF

# Expected output:
# ✅ Project created successfully
# ✅ Dependencies installed
```

Get your Stripe keys from the Dashboard under "Developers" → "API keys." Remember to switch from test keys to live keys before deploying to production.

        body: JSON.stringify({ priceId: plan.priceId }),
      });

      const { sessionId } = await response.json();
      // Redirect to Stripe Checkout (or in-app payment)
      // ...
    } catch (error) {
      Alert.alert("Error", "Something went wrong during payment processing.");
    } finally {
      setLoading(false);
    }
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Choose Your Plan</Text>
      {PLANS.map((plan) => (
        <TouchableOpacity
          key={plan.id}
          style={[
            styles.planCard,
            selectedPlan === plan.id && styles.selectedCard,
          ]}

        disabled={loading}
      >

          {loading ? "Processing..." : "Subscribe"}

    </View>

### Server-Side API Endpoint

```typescript
// api/create-checkout-session.ts
import Stripe from "stripe";

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: "2025-12-18.acacia",
});

export async function POST(request: Request) {
  try {
    const { priceId } = await request.json();

    const session = await stripe.checkout.sessions.create({
      mode: "subscription",
      payment_method_types: ["card"],
      line_items: [{ price: priceId, quantity: 1 }],
      success_url: `${process.env.APP_URL}/subscription/success?session_id={CHECKOUT_SESSION_ID}`,
      cancel_url: `${process.env.APP_URL}/subscription/cancel`,
      metadata: {
        plan_type: "pro",  // Required to distinguish from tips
      },
    });

    return Response.json({ sessionId: session.id });
  } catch (error) {
    console.error("Checkout session creation failed:", error);
    return Response.json(
      { error: "Failed to create checkout session" },
      { status: 500 }
    );
  }
}
```

## Webhook-Based Subscription Management

Handle Stripe event notifications to manage subscription states.

```typescript
// api/webhook.ts
import Stripe from "stripe";

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!, {
  apiVersion: "2025-12-18.acacia",
});

export async function POST(request: Request) {
  const body = await request.text();
  const signature = request.headers.get("stripe-signature")!;

  let event: Stripe.Event;
  try {
    event = stripe.webhooks.constructEvent(
      body,
      signature,
      process.env.STRIPE_WEBHOOK_SECRET!
    );
  } catch (err) {
    return Response.json({ error: "Webhook signature verification failed" }, { status: 400 });
  }

  switch (event.type) {
    case "checkout.session.completed": {
      const session = event.data.object as Stripe.Checkout.Session;
      await updateUserSubscription(session.customer as string, "active");
      break;
    }
    case "customer.subscription.updated": {
      const subscription = event.data.object as Stripe.Subscription;
      await updateUserSubscription(
        subscription.customer as string,
        subscription.status
      );
      break;
    }
    case "customer.subscription.deleted": {
      const subscription = event.data.object as Stripe.Subscription;
      await updateUserSubscription(subscription.customer as string, "canceled");
      break;
    }
  }

  return Response.json({ received: true });
}

// Expected behavior:
// 1. User completes payment → checkout.session.completed → status: active
// 2. Plan change → customer.subscription.updated → status updated
// 3. Cancellation → customer.subscription.deleted → status: canceled
```

## Running Test Payments

Once implementation is complete, verify the payment flow using Stripe's test mode.

```bash
# Forward webhooks to your local server
stripe listen --forward-to localhost:3000/api/webhook

# Test card numbers
# Success: 4242 4242 4242 4242
# Insufficient funds: 4000 0000 0000 9995
# Authentication required: 4000 0025 0000 3155

# Manually trigger webhook events for testing
stripe trigger checkout.session.completed
```

No real charges are made in test mode. Before going live, always test all three scenarios: success, failure, and authentication required.

## Frequently Asked Questions (FAQ)

### Q1: Should I use Apple/Google in-app purchases or Stripe?

For digital content and in-app feature unlocks, you're required to use each store's in-app purchase system (per App Store / Google Play policies). However, for physical goods or services outside the app (SaaS, web service access), Stripe is a valid option. When implementing subscriptions with Rork, using a web-based billing flow (Stripe Checkout) can help you avoid store commission fees (15-30%) in eligible cases.

### Q2: How do I implement subscription cancellation?

The easiest approach is using Stripe's Customer Portal. Generate a portal URL with `stripe.billingPortal.sessions.create()` and redirect the user. The portal handles cancellations, plan changes, and payment method updates through Stripe's own UI, so you don't need to build a management screen from scratch.

### Q3: Can I offer a free trial period?

Yes. Specify `trial_period_days` when creating the Price, or set `subscription_data.trial_period_days` when creating the Checkout Session. No charges occur during the trial, and the first billing happens automatically when the trial ends. Stripe also provides built-in email notifications before the trial expires.
