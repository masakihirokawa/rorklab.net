## Introduction: What Is Google Play Billing?

If you want your Android app to generate sustainable revenue, understanding the **Google Play Billing API** is essential. It is Google's official payment infrastructure that handles in-app purchases and subscriptions — from monthly recurring plans to one-time content unlocks.

Apps built with Rork run on a React Native/Expo foundation, which means you can integrate Google Play Billing using battle-tested open-source libraries without writing any native code yourself. This guide walks you through the entire process: setting up products in Google Play Console, integrating **react-native-iap**, writing the purchase flow, and testing before you ship.

## Understanding the Google Play Billing Model

Before diving into code, it helps to know the three purchase types available:

- **Subscriptions (recurring billing)**: Monthly or annual plans that auto-renew. Ideal for apps offering ongoing value like content, tools, or premium features.
- **Consumable products**: Items like in-app currency or power-ups that can be re-purchased after being "consumed."
- **Non-consumable products**: Permanent unlocks like a one-time premium upgrade that the user keeps forever.

The billing flow is straightforward. Your app requests a purchase from Google Play, the user completes payment, and Google returns a **purchase token**. You then verify that token — ideally on your backend — to grant the appropriate access.

## Step 1: Set Up Google Play Console

Before writing a single line of code, you need to configure your app and products in **Google Play Console**.

**Register your app**

Your app must be registered in Google Play Console with at least one APK or Android App Bundle uploaded to any test track. If you haven't published your app yet, check out [Publishing Your Rork App to Google Play](/articles/rork-dev/google-play-publishing) first.

**Create a subscription product**

Navigate to **Monetize → Subscriptions** and create a new subscription.

- **Product ID**: A unique identifier used in code (e.g., `premium_monthly`). Cannot be changed after creation.
- **Price**: Set per country and currency as needed.
- **Billing period**: Choose monthly, annual, weekly, or custom cycles.
- **Free trial**: Offering a trial period significantly lowers the conversion barrier for new users.

After creating the product, make sure to **activate it** — inactive products cannot be fetched by the SDK.

**Create a service account (for backend verification)**

If you plan to verify purchases server-side (strongly recommended for production), create a service account in Google Cloud Console with the Google Play Developer API enabled, download the JSON key, and link it to your Play Console account.

## Step 2: Add react-native-iap to Your Rork Project

`react-native-iap` provides a unified API for both iOS StoreKit and Android Google Play Billing, making it the go-to choice for cross-platform in-app purchases.

```bash
# Install react-native-iap
npm install react-native-iap

# Install expo-build-properties for native config
npx expo install expo-build-properties
```

Add the following to your `app.json` to ensure the correct Android SDK versions:

```json
{
  "expo": {
    "plugins": [
      [
        "expo-build-properties",
        {
          "android": {
            "compileSdkVersion": 35,
            "targetSdkVersion": 35
          }
        }
      ]
    ]
  }
}
```

## Step 3: Implement the Subscription Purchase Flow

Here is a clean, production-ready implementation of a subscription screen:

```tsx
import React, { useEffect, useState } from 'react';
import { View, Text, TouchableOpacity, Alert, Platform } from 'react-native';
import {
  initConnection,
  getSubscriptions,
  requestSubscription,
  purchaseUpdatedListener,
  purchaseErrorListener,
  finishTransaction,
  type SubscriptionAndroid,
} from 'react-native-iap';

// Product IDs defined in Google Play Console
const SUBSCRIPTION_SKUS = Platform.select({
  android: ['premium_monthly', 'premium_yearly'],
  ios: ['premium_monthly', 'premium_yearly'],
}) ?? [];

export default function SubscriptionScreen() {
  const [subscriptions, setSubscriptions] = useState<SubscriptionAndroid[]>([]);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    const initIAP = async () => {
      try {
        // Establish connection to Google Play Billing
        await initConnection();
        // Fetch available subscription products
        const subs = await getSubscriptions({ skus: SUBSCRIPTION_SKUS });
        setSubscriptions(subs as SubscriptionAndroid[]);
      } catch (error) {
        console.error('IAP initialization error:', error);
      }
    };

    initIAP();

    // Listen for successful purchases
    const purchaseListener = purchaseUpdatedListener(async (purchase) => {
      if (purchase.transactionReceipt) {
        try {
          // Send token to your backend for verification (recommended)
          await verifyPurchaseOnServer(purchase.purchaseToken ?? '');
          // CRITICAL: Always call finishTransaction — Google refunds unacknowledged
          // purchases after 3 days
          await finishTransaction({ purchase, isConsumable: false });
          Alert.alert('Success', 'Your premium plan is now active!');
        } catch (e) {
          console.error('Purchase processing error:', e);
        }
      }
    });

    // Listen for purchase errors
    const errorListener = purchaseErrorListener((error) => {
      console.error('Purchase error:', error.code, error.message);
    });

    return () => {
      purchaseListener.remove();
      errorListener.remove();
    };
  }, []);

  const handleSubscribe = async (productId: string) => {
    setLoading(true);
    try {
      await requestSubscription({
        sku: productId,
        ...(Platform.OS === 'android' && {
          subscriptionOffers: [{ sku: productId, offerToken: '' }],
        }),
      });
    } catch (error) {
      Alert.alert('Error', 'Purchase failed. Please try again.');
    } finally {
      setLoading(false);
    }
  };

  // Example backend verification call
  const verifyPurchaseOnServer = async (purchaseToken: string) => {
    const response = await fetch('https://your-api.example.com/verify-purchase', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ purchaseToken, packageName: 'com.yourapp.name' }),
    });
    if (!response.ok) throw new Error('Server verification failed');
  };

  return (
    <View style={{ flex: 1, padding: 24 }}>
      <Text style={{ fontSize: 22, fontWeight: 'bold', marginBottom: 16 }}>
        Choose Your Plan
      </Text>
      {subscriptions.map((sub) => (
        <TouchableOpacity
          key={sub.productId}
          onPress={() => handleSubscribe(sub.productId)}
          disabled={loading}
          style={{
            backgroundColor: '#4F46E5',
            padding: 16,
            borderRadius: 12,
            marginBottom: 12,
          }}
        >
          <Text style={{ color: 'white', fontSize: 16, fontWeight: '600' }}>
            {sub.title} — {sub.localizedPrice}/month
          </Text>
        </TouchableOpacity>
      ))}
    </View>
  );
}
```

The most important thing to remember: **always call `finishTransaction` after a successful purchase**. Google Play automatically reverses any purchase that isn't acknowledged within three days.

## Step 4: Testing with License Testers

Google Play provides a **License Tester** feature that lets you test purchases without real charges.

Go to **Google Play Console → Setup → License testing** and add your test Google account email. Once added, purchases made by that account will be free, and subscription renewal periods are dramatically shortened (e.g., a monthly subscription renews every five minutes), making it easy to test the full renewal cycle.

A few things to keep in mind during testing:

- You need at least one APK uploaded to an internal test track before license testing works.
- Always test on a real Android device — the Google Play Billing API does not work reliably on emulators.
- Confirm that your subscription product is in **Active** state in the console.

## Common Errors and How to Fix Them

**`ITEM_NOT_OWNED` error**

This occurs when a user tries to purchase a subscription they already own. Add a check for the current subscription status before initiating a new purchase request.

**`BILLING_UNAVAILABLE` error**

Usually caused by outdated Google Play Services on the test device, or the app not being linked to the correct Google account. Update Google Play Services and verify the device is signed in with the correct account.

**Purchase stuck in Pending state**

If `finishTransaction` is never called, the purchase remains "pending" and will be refunded by Google after three days. Make sure your `purchaseUpdatedListener` reliably calls `finishTransaction` on every successful purchase.

## FAQ

**Q. Do I need Rork Max to implement Google Play Billing?**

A. Since `react-native-iap` involves native module integration, you'll need EAS Build to compile the app with the billing library included. Whether EAS Build is available depends on your Rork plan — check the [Rork pricing plans](/articles/rork-business/rork-pricing-plans) for details.

**Q. When does my app receive the purchase update after a user subscribes?**

A. The `purchaseUpdatedListener` fires immediately after a successful transaction. For recurring renewals (monthly, annual), it's best practice to set up **Google Play Developer Notifications** via Pub/Sub so your backend is notified at each renewal event, rather than relying solely on client-side detection.

**Q. Is backend verification really necessary?**

A. Yes, for any production app. Without server-side verification, a motivated attacker could replay a purchase token or fabricate a successful purchase response to unlock premium features. The Google Play Developer API — accessed with your service account credentials — lets you verify a token's authenticity and check whether the subscription is still active.

**Q. How do I handle subscription restoration for returning users?**

A. Call `getAvailablePurchases()` from `react-native-iap` when the app launches or when a user taps a "Restore Purchases" button. This retrieves all non-consumed and active subscription purchases associated with the user's Google account.
