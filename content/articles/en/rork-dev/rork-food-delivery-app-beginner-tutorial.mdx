## Introduction — Why Build a Food Delivery App with Rork?

A food delivery or restaurant ordering app is one of the most rewarding projects for indie developers. It touches nearly every fundamental mobile development skill: real-time data, product catalogs, payment flows, and push notifications. And with Rork, you can pull all of this together in a fraction of the usual time.

In this tutorial, we'll walk through building a takeout ordering app prototype using **Rork + Supabase + Stripe**. By the end, you'll have a working app with a menu screen, a cart, a checkout flow, and real-time order tracking.

What you'll learn:

- How to scaffold a food delivery app's screens in Rork
- Managing menu data and tracking orders in real time with Supabase
- Integrating Stripe for seamless payment processing
- Sending push notifications when order status changes
- How to handle common errors along the way

This guide is aimed at developers who are comfortable with Rork basics and want to build something they can actually ship.

## Step 1: Design the Database in Supabase

Start by creating a Supabase project and setting up three tables.

### Table Schema

```sql
-- Menu items
create table menu_items (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  description text,
  price integer not null,   -- in smallest currency unit (cents/yen)
  category text not null,   -- e.g. "Burgers", "Drinks"
  image_url text,
  available boolean default true,
  created_at timestamptz default now()
);

-- Orders
create table orders (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users(id),
  status text default 'pending',  -- pending / preparing / ready / delivered
  total_amount integer not null,
  stripe_payment_intent_id text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Order line items
create table order_items (
  id uuid default gen_random_uuid() primary key,
  order_id uuid references orders(id) on delete cascade,
  menu_item_id uuid references menu_items(id),
  quantity integer not null default 1,
  unit_price integer not null
);
```

Enable **Row Level Security (RLS)** on both `orders` and `order_items` so users can only access their own data.

## Step 2: Build the Menu Screen in Rork

Create a new Rork project and describe the menu screen in the prompt:

```
Fetch data from the Supabase menu_items table and display it in a
SectionList grouped by category. Each item card should show the image,
name, price, and an "Add to Cart" button.
```

Rork will generate something like this:

```tsx
// app/(tabs)/menu.tsx
import { useEffect, useState } from "react";
import { SectionList, Text, TouchableOpacity, View } from "react-native";
import { supabase } from "@/lib/supabase";
import { useCartStore } from "@/store/cartStore";

type MenuItem = {
  id: string;
  name: string;
  price: number;
  category: string;
  image_url: string;
};

export default function MenuScreen() {
  const [sections, setSections] = useState<{ title: string; data: MenuItem[] }[]>([]);
  const addToCart = useCartStore((s) => s.addItem);

  useEffect(() => {
    const fetchMenu = async () => {
      const { data } = await supabase
        .from("menu_items")
        .select("*")
        .eq("available", true)
        .order("category");

      if (data) {
        const grouped = data.reduce((acc: Record<string, MenuItem[]>, item) => {
          (acc[item.category] ||= []).push(item);
          return acc;
        }, {});
        setSections(
          Object.entries(grouped).map(([title, data]) => ({ title, data }))
        );
      }
    };
    fetchMenu();
  }, []);

  return (
    <SectionList
      sections={sections}
      keyExtractor={(item) => item.id}
      renderSectionHeader={({ section }) => (
        <Text style={{ fontWeight: "bold", padding: 12, backgroundColor: "#f5f5f5" }}>
          {section.title}
        </Text>
      )}
      renderItem={({ item }) => (
        <View style={{ flexDirection: "row", padding: 12, borderBottomWidth: 1, borderColor: "#eee" }}>
          <Text style={{ flex: 1 }}>{item.name} — ${(item.price / 100).toFixed(2)}</Text>
          <TouchableOpacity onPress={() => addToCart(item)}>
            <Text style={{ color: "#e63946" }}>Add</Text>
          </TouchableOpacity>
        </View>
      )}
    />
  );
}
```

Use Zustand or another lightweight store to manage cart state globally across screens.

## Step 3: Integrate Stripe for Checkout

For payments, use **Stripe Payment Sheet** — it handles card entry, Apple Pay, and Google Pay with minimal code. Create a Supabase Edge Function to generate a PaymentIntent server-side.

### Supabase Edge Function

```typescript
// supabase/functions/create-payment-intent/index.ts
import Stripe from "https://esm.sh/stripe@14";

const stripe = new Stripe(Deno.env.get("STRIPE_SECRET_KEY")!);

Deno.serve(async (req) => {
  const { amount, currency = "usd" } = await req.json();

  const paymentIntent = await stripe.paymentIntents.create({
    amount,   // e.g. 1980 for $19.80
    currency,
    automatic_payment_methods: { enabled: true },
  });

  return new Response(
    JSON.stringify({ clientSecret: paymentIntent.client_secret }),
    { headers: { "Content-Type": "application/json" } }
  );
});
```

### Client-Side Checkout (Rork)

```tsx
import { initPaymentSheet, presentPaymentSheet } from "@stripe/stripe-react-native";

const handleCheckout = async () => {
  if (isLoading) return; // prevent double-tap
  setIsLoading(true);

  try {
    const total = cartItems.reduce((sum, item) => sum + item.price * item.quantity, 0);

    // Get PaymentIntent from Edge Function
    const { data } = await supabase.functions.invoke("create-payment-intent", {
      body: { amount: total },
    });

    // Initialize Payment Sheet
    await initPaymentSheet({
      paymentIntentClientSecret: data.clientSecret,
      merchantDisplayName: "My Food App",
    });

    // Present Payment Sheet
    const { error } = await presentPaymentSheet();

    if (!error) {
      // Save order to Supabase
      await supabase.from("orders").insert({
        user_id: user.id,
        total_amount: total,
        status: "pending",
      });
      router.push("/order-confirmed");
    }
  } finally {
    setIsLoading(false);
  }
};
```

## Step 4: Real-Time Order Tracking with Supabase

Once an order is placed, use Supabase Realtime to watch for status updates and reflect them instantly on the user's screen.

```tsx
useEffect(() => {
  const channel = supabase
    .channel(`order-${orderId}`)
    .on(
      "postgres_changes",
      {
        event: "UPDATE",
        schema: "public",
        table: "orders",
        filter: `id=eq.${orderId}`,
      },
      (payload) => {
        setOrderStatus(payload.new.status);
        if (payload.new.status === "ready") {
          scheduleLocalNotification("Your order is ready for pickup!");
        }
      }
    )
    .subscribe();

  return () => supabase.removeChannel(channel);
}, [orderId]);
```

## Common Errors and Fixes

### Supabase data not loading with RLS enabled

When you enable RLS, all data is hidden by default until you add policies. Go to **Authentication → Policies** in the Supabase dashboard and add a `SELECT` policy for authenticated users. Example:

```sql
create policy "Users can view available menu items"
on menu_items for select
using (available = true);
```

### PaymentIntent being created multiple times on button spam

Always guard your checkout handler with a loading state to prevent duplicate PaymentIntents. The `isLoading` flag pattern in Step 3 above handles this.

### Realtime subscription dropping when app goes to background

Subscribe to `AppState` changes and re-subscribe to the Supabase channel when the app returns to the foreground:

```tsx
AppState.addEventListener("change", (nextState) => {
  if (nextState === "active") {
    // Re-subscribe to the order channel
  }
});
```

## FAQ

### Can I build this on the Rork Free plan?

Yes — the Free plan (5 generations per week) is perfectly fine for development and testing. You'll need Rork Max to build to TestFlight or publish to the App Store.

### Is Supabase's free tier enough for production?

For a small-scale indie app, Supabase's free tier (500 MB database, 50,000 monthly active users) should cover you well. Upgrade to Pro as your user base grows.

### Does Stripe Payment Sheet support Apple Pay?

Yes. The `@stripe/stripe-react-native` Payment Sheet automatically includes Apple Pay and Google Pay when enabled in your Stripe dashboard and properly configured in your app's entitlements.

### How do I send email confirmations after a successful payment?

Subscribe to the `payment_intent.succeeded` Stripe webhook in a Supabase Edge Function and call a transactional email service like Resend or SendGrid from there.
