## Introduction

Rork's AI-generated app prototypes are immediately useful, but **"I want stronger brand presence", "I need unified fonts and icons", "responsive scaling needs tweaking"**—these refinement desires are natural and valuable.

This guide walks you through migrating a Rork-generated UI into Figma and applying **brand-driven refinement to create truly polished, recognizable interfaces**. You don't need advanced design skills—just follow the structured steps to elevate AI-generated output into production-quality design.

## Rork and Figma: Complementary, Not Competing

These tools work together beautifully:

### Rork Excels At:
- High-speed prototype generation (hours, not days)
- Auto-generated layouts and feature placement
- Consistency across multiple screens
- Iterative AI regeneration for exploring options

### Figma Excels At:
- Unified brand color application
- Font and icon system management
- Precise spacing and padding adjustments
- Design review and team collaboration
- Developer handoff and specification

## Step 1: Migrating Rork Design to Figma

### Method 1: PNG Export and Import (Simplest)

Export your Rork-generated UI as PNG and bring it into Figma:

1. **In Rork: Select "Export Design"**
2. **Download each page as PNG**
3. **In Figma: Create new file → File menu → Import**
4. **Drag and drop PNG into canvas**

**Advantage**: Instant start.
**Limitation**: Results in image layers—direct text editing isn't possible.

### Method 2: Figma-Native Export (Advanced)

Some Rork versions export directly to Figma format (FDF):

1. **Text and shape layers remain fully editable**
2. **Individual UI elements are selectable**
3. **Design system integration possible**

For deeper customization, this method is recommended.

## Step 2: Apply Brand Colors

Rork uses default colors. **Replace them with your brand palette** to create immediate visual cohesion.

### Building Your Color System

Open Figma's Assets panel → Colors. Structure colors systematically:

**Recommended Structure:**
- **Primary Colors** (CTAs, highlights): 3 variants (Dark / Default / Light)
- **Secondary Colors** (supporting actions): 3 variants
- **Grayscale** (text, backgrounds): 6 steps (#000 to #FFF)
- **Status Colors** (success/warning/error): 3 variants each
- **Neutral Accents**: 2 variants

**Example Setup:**
```
Primary:
  - Primary/900 (#1A2E4E)
  - Primary/500 (#2563EB)
  - Primary/100 (#E0E7FF)

Neutral:
  - Neutral/900 (#1F2937)
  - Neutral/500 (#6B7280)
  - Neutral/100 (#F3F4F6)

Status/Error:
  - Error/600 (#DC2626)
  - Error/400 (#F87171)
```

### Bulk Color Replacement

Use Figma plugins like "Color Assistant" or "Replace Color":

1. **Open plugin**
2. **Specify old color → new color**
3. **Execute replacement**
4. **Verify visual results**

Dozens of UI elements update in seconds.

### Accessibility Verification

After color replacement, verify **WCAG contrast standards**:
- **AAA (recommended)**: 7:1 contrast ratio minimum
- **AA (minimum)**: 4.5:1 contrast ratio minimum

Use Figma's "A11y – Color Contrast Checker" plugin to auto-validate all color combinations.

## Step 3: Unify Typography

Rork typically uses default sans-serif (Roboto, Inter). **Replace with your brand font** for visual consistency.

### Designing Your Typography System

In Figma's Assets panel, create hierarchical text styles:

**Recommended Type Hierarchy:**
- **Display** (48px, weight 700) — Page titles
- **Heading 1** (36px, weight 700) — Section titles
- **Heading 2** (28px, weight 600) — Subsection titles
- **Heading 3** (24px, weight 600) — Small headings
- **Body** (16px, weight 400) — Main content
- **Small Body** (14px, weight 400) — Supporting text
- **Caption** (12px, weight 500) — Labels, captions
- **Micro** (11px, weight 500) — Smallest size

**Font Selection by Brand:**
- **Modern/Tech**: Inter, Outfit, SF Pro Display
- **Friendly/Casual**: Poppins, Nunito, Raleway
- **Premium/Luxury**: Playfair Display (headings) + Lato (body)
- **Minimal**: IBM Plex Sans, Source Sans Pro

### Refining Typography Details

Rork's default settings often need adjustment:

| Element | Adjustment | Rork Default | Recommended |
|---------|------------|--------------|-------------|
| H1 Heading | Line Height | 120% | 140% |
| Body Text | Letter Spacing | 0px | +0.5px (for longer text) |
| Button Text | Line Height | 140% | 120% (compact) |
| Form Labels | Weight | 400 | 500 (better visibility) |

**Critical Tweaks:**
- **Line Height**: AI often defaults to tight 110-120%. Increase to 150% for body, 140% for headings
- **Letter Spacing**: Add 0.5-1px for long-form content
- **Weight Differentiation**: Clearly separate headings (600+) from body (400)

## Step 4: Unify Icons

Rork may mix icon sources. **Create a consistent icon system** for professional polish.

### Comparing Icon Sets

| Set | Stroke Width | Style | Language Support | Best For |
|-----|-------------|-------|-----------------|----------|
| **Material Icons** | 24px | Fill & outline | Limited | Enterprise, universal |
| **Feather Icons** | 24px | Outline only | Limited | Modern, minimal |
| **SF Symbols** | 32px | Apple standard | Japanese-supported | iOS/macOS apps |
| **Heroicons** | 24px | Minimal | Limited | Web applications |
| **Tabler Icons** | 24px | Customizable | Limited | SaaS products |

### Systematic Icon Replacement

**Step 1: Select Icon Set**
1. Match to brand identity
2. Import from Figma Community as component library

**Step 2: Audit Existing Icons**
1. Screenshot all current icons
2. List which ones need replacing
3. Check new set for equivalents

**Step 3: Replace Gradually**
1. Start with one screen
2. Test color and size compatibility
3. Scale to other screens

**Step 4: Component Replacement in Figma**
```
Select icons → Right-click → Replace with component → Choose new icon set
```

### Icon System Best Practices

Ensure consistency:

- **Stroke Width**: 2px standard
- **Base Size**: 24px or 32px
- **Color**: Limit to brand colors
- **Padding**: Consistent whitespace around icons

Use Figma's component variants for size and color flexibility:
```
Icon / Home / 24 / Outline
  → Variants: Size (24, 32, 48)
  → Variants: Color (Primary, Secondary, Gray)
```

## Step 5: Responsive Adjustment

Rork UIs are responsive-ready, but fine-tuning improves consistency.

### Multi-Device Testing

Duplicate your frame at different sizes:
- **iPhone SE**: 375 × 667 px
- **iPhone 13**: 390 × 844 px
- **iPad**: 768 × 1024 px

Check each size for:
- **Text readability** (especially small text)
- **Button sizes** (minimum 44 × 44 px)
- **Image aspect ratios** (preserved correctly?)

### Using Figma's Constraints

Figma's constraint feature auto-adapts to parent frame resizing:

1. **Select element**
2. **Right panel → Constraints section**
3. **Define behavior** (left-fixed, stretch, etc.)

This creates auto-responsive layouts.

## Step 6: Component-Based Architecture

**Figma's component system is your strongest asset.** Componentizing Rork UI elements dramatically improves maintainability.

### Benefits of Componentization

- **Single-source-of-truth**: Edit once, update everywhere
- **Reduced revisions**: Color/style changes propagate instantly
- **Scalable team workflows**: Shared component library
- **Clear design system**: Components become the spec

### Building Button Components (Example)

**Phase 1: Define Main Component**
```
Button / Primary / Large
  ├─ Background: Primary Color
  ├─ Text: White, 16px, weight 600
  ├─ Padding: 12px 24px
  ├─ Corner Radius: 8px
  └─ Shadow: Drop shadow (0, 4px, 8px, 20% opacity)
```

**Phase 2: Add Variants (States)**
```
Button component
  ├─ Variant: Size (Large, Medium, Small)
  ├─ Variant: Type (Primary, Secondary, Ghost)
  ├─ Variant: State (Default, Hover, Pressed, Disabled)
  └─ Variant: Icon (With Icon / Without Icon)
```

**Phase 3: Apply Across App**
1. Select all button instances
2. Right-click → Replace with component
3. Assign appropriate variants

Main component edits now auto-update all instances.

### Component Hierarchy

Scale thoughtfully:

**Foundation Layer:**
- Button, Input, Checkbox, Radio, Toggle
- Icon, Avatar, Badge
- Typography styles (H1, H2, Body, etc.)

**Composite Layer:**
- Form Group (Label + Input + Error Message)
- Card (Header + Content + Footer)
- Modal (Title + Content + Actions)
- Navigation Bar

**Page Patterns:**
- Login screen template
- Product detail page
- Dashboard layout

### Component Audit Checklist

```
□ Identify single button pattern
□ List all variants (color, size, state)
□ Create main component
□ Build all variants
□ Replace all app buttons with components
□ Verify consistency (no color/size outliers)
□ Repeat for forms, cards, other elements
□ Organize component library alphabetically
```

## Step 7: Iterative Rork-Figma Workflow

Design refinement in Figma isn't the end—stay ready to loop back to Rork for significant changes:

### Recommended Cycle

```
Rork generates initial UI
  ↓
Migrate to Figma
  ↓
Refine (colors, fonts, icons, components)
  ↓
Hand off to dev team
  ↓
Collect user feedback
  ↓
[Major redesign needed?]
Return to Rork for regeneration
  ↓
Compare new output + Figma refinements
  ↓
Integrate best version
```

This cycle maximizes AI generation speed while leveraging human refinement skill.

## Building a Design System

Once you've componentized Rork UI across multiple projects, mature into a **formal Design System**:

### Progressive Development

**Phase 1 (Projects 1-2):**
- Color palette (Primary / Secondary / Neutral / Status)
- Typography system (Display through Micro)
- Basic components (Button, Input, Checkbox)

**Phase 2 (Projects 3-4):**
- Icon system (multiple sizes, colors)
- Composite components (Form, Card, Modal)
- Layout patterns (Grid, Sidebar)

**Phase 3 (Ongoing):**
- Page templates (Dashboard, Lists, Details)
- Screen patterns (Login, Error, Loading)
- Animation / micro-interaction specs

### Figma Library Management

Create a central "Design System" file and link it to project files:

1. **Main file**: "Design System v1.0"
   - All components as main components
   - Color tokens, typography tokens
2. **Mark as library**: Assets → Library → Save as library
3. **Link in projects**: Other files → Assets → Linked libraries → Connect

**Governance:**
- Version control: v2.0 for breaking changes
- Gradual updates: New features in separate sections
- Documentation: Component descriptions and usage rules

### Design Tokens

Tokenize values for dev handoff:

```json
{
  "colors": {
    "primary": {
      "900": "#1A2E4E",
      "600": "#2563EB",
      "100": "#E0E7FF"
    }
  },
  "spacing": {
    "xs": "4px",
    "sm": "8px",
    "md": "16px",
    "lg": "24px"
  },
  "typography": {
    "body": {
      "fontSize": "16px",
      "fontWeight": "400",
      "lineHeight": "1.5"
    }
  }
}
```

Figma's Variables feature (2024+) enables in-tool token management and JSON export.

## Developer Handoff

Once Figma refinement is complete, share specifications with your dev team:

### Figma Dev Mode

Enable Dev Mode to auto-extract specifications:
1. Open Figma file → Dev tab (top right)
2. Click component → View specs in right panel
3. Auto-extracted info: dimensions, colors, fonts, shadows, CSS suggestions

### Developer Documentation Template

```markdown
# UI Implementation Guide

## Color Palette
- Primary: #2563EB
- Secondary: #10B981
- Neutral-700: #374151

## Typography
- H1: Inter 36px / 700 / line-height 1.4
- Body: Inter 16px / 400 / line-height 1.5

## Component Rules
- Buttons: Use "Button / Primary / Large" component
- Inputs: Use "Form / Input" component
- Icons: Material Icons 24px, unified

## Spacing Rules
- Margins/Padding: 8px units
- Grid: 24px base
```
