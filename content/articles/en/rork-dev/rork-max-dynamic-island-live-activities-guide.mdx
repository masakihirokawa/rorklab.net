## What Are Dynamic Island and Live Activities?

**Dynamic Island** — introduced with the iPhone 14 Pro — is the pill-shaped cutout at the top of the screen that Apple has turned into an interactive UI surface. Even when your app is in the background, it can push real-time information right into that island for users to see at a glance.

**Live Activities**, available since iOS 16.1, is the framework that powers this. It lets your app display live-updating content on the Lock Screen and inside Dynamic Island for up to eight hours. Common use cases include:

- Delivery and rideshare tracking (live location, ETA)
- Sports score tickers
- Food order status updates
- Music and podcast playback controls
- Workout timers and fitness progress

Standard Rork (React Native-based) cannot access Dynamic Island or Live Activities natively. But with **Rork Max**, Claude AI generates Swift/SwiftUI native code for you, making full Dynamic Island support possible — no Xcode required.

## Step 1: Define Your ActivityAttributes

`ActivityAttributes` is the data contract for your Live Activity. It separates **static data** (set once at launch) from **dynamic data** (updated in real time via `ContentState`).

Here's an example for a delivery tracking app:

```swift
import ActivityKit
import Foundation

struct DeliveryAttributes: ActivityAttributes {
    // Dynamic data — updated in real time
    public struct ContentState: Codable, Hashable {
        var driverName: String       // Driver's name
        var estimatedArrival: Date   // Estimated arrival time
        var currentStatus: String    // "Preparing" / "On the way" / "Arrived"
    }

    // Static data — set once when the activity starts
    var orderId: String
    var restaurantName: String
}
```

## Step 3: Start, Update, and End a Live Activity

Here is the complete lifecycle management code for your main app target:

```swift
import ActivityKit

// Start a Live Activity
func startDeliveryActivity(orderId: String, restaurant: String, driverName: String) {
    guard ActivityAuthorizationInfo().areActivitiesEnabled else {
        print("Live Activities are disabled on this device.")
        return
    }

    let attributes = DeliveryAttributes(
        orderId: orderId,
        restaurantName: restaurant
    )

    let initialState = DeliveryAttributes.ContentState(
        driverName: driverName,
        estimatedArrival: Date().addingTimeInterval(1800), // ETA: 30 minutes from now
        currentStatus: "Preparing"
    )

    do {
        let activity = try Activity.request(
            attributes: attributes,
            content: .init(state: initialState, staleDate: nil),
            pushType: nil // Set to .token if you want server-side push updates
        )
        print("Live Activity started: \(activity.id)")
    } catch {
        print("Failed to start Live Activity: \(error)")
    }
}

// Update the Live Activity in real time
func updateDeliveryStatus(
    activity: Activity<DeliveryAttributes>,
    newStatus: String,
    minutesLeft: Double
) {
    let updatedState = DeliveryAttributes.ContentState(
        driverName: activity.content.state.driverName,
        estimatedArrival: Date().addingTimeInterval(minutesLeft * 60),
        currentStatus: newStatus
    )

    Task {
        await activity.update(.init(state: updatedState, staleDate: nil))
    }
}

// End the Live Activity when the order is delivered
func endDeliveryActivity(activity: Activity<DeliveryAttributes>) {
    Task {
        await activity.end(nil, dismissalPolicy: .immediate)
    }
}
```

## Updating via Push Notifications (Server-Side)

For apps where status changes are driven by your backend (e.g., when a driver's GPS updates), use **Push to Update** to send changes from your server via APNs.

```swift
// Request an activity with push token support
let activity = try Activity.request(
    attributes: attributes,
    content: .init(state: initialState, staleDate: nil),
    pushType: .token  // Enable push updates
)

// Listen for push token changes and relay to your server
for await pushToken in activity.pushTokenUpdates {
    let tokenString = pushToken.map { String(format: "%02x", $0) }.joined()
    await sendTokenToServer(token: tokenString, activityId: activity.id)
}
```

Your server then sends an APNs payload with the updated `ContentState` JSON. When you prompt Rork Max to include "server-side push updates for Live Activities," it will also generate the corresponding Cloudflare Workers backend code.

## Common Errors and Fixes

Here are the most frequent issues developers run into with Dynamic Island and Live Activities.

**Error: ActivityKit.ActivityAuthorizationError**

The `NSSupportsLiveActivities` key is missing from `Info.plist`. Prompt Rork Max: "Add NSSupportsLiveActivities = true to Info.plist" to fix this.

**Live Activity doesn't appear in the Simulator**

Dynamic Island and Live Activities only work on physical devices. Use Rork Max's TestFlight distribution to test on a real iPhone 14 Pro or later.

**Updates stop showing after a while**

Apple limits Live Activity updates to roughly 15 per hour from the app side. For higher-frequency updates, switch to server-side push updates via APNs.

**Expanded view layout breaks on different device sizes**

Dynamic Island's expanded region varies by device. Use SwiftUI's `GeometryReader` to measure available space and adapt your layout accordingly.

## Frequently Asked Questions

**Q: Which iPhone models support Dynamic Island?**

Dynamic Island is available on iPhone 14 Pro, 14 Pro Max, and all iPhone 15 / 16 models. On older iPhones (14 standard and below), Live Activities still appear on the Lock Screen, but there is no Dynamic Island UI.

**Q: How long can a Live Activity stay visible?**

Up to eight hours per Apple's guidelines. After that the system automatically ends it. Users can also dismiss it manually by swiping it away. Always call `activity.end()` on your side when the real-world event is over.

**Q: Can standard Rork (not Max) implement Dynamic Island?**

No. Standard Rork runs on React Native and cannot access Apple's Swift-only frameworks. **Rork Max** is required. See [Rork vs Rork Max](/articles/rork-basics/rork-vs-rork-max) for a full plan comparison.

**Q: Can I load images from a remote URL inside Dynamic Island?**

WidgetKit does not support async remote image loading. Use SF Symbols or images bundled inside your app target (`Image("asset-name")`).

**Q: Does Android have something similar to Dynamic Island?**

Android doesn't have a native Dynamic Island equivalent. For Android users, you can achieve similar real-time notifications using custom `Notification.Builder` layouts. Rork Max focuses on iOS-native features; for Android real-time UI, use React Native's notification libraries instead.
