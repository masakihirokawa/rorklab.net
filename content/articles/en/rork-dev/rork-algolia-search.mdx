## Why Your Rork App Needs Full-Text Search

Whether you're building a recipe app, an e-commerce store, or a notes app with Rork, one pain point always surfaces as your content grows: **users can't find what they're looking for**. A basic SQL `LIKE '%keyword%'` query degrades badly at a few thousand records, and it has no concept of relevance, typo tolerance, or language-aware tokenization.

**Algolia** solves this at the infrastructure level. It's a hosted search-as-a-service platform that delivers millisecond search responses, intelligent ranking, and faceted filtering — all accessible via a React Native-friendly SDK. The free tier covers up to 10,000 requests per month, making it a natural fit for indie developers and small products built on Rork.

By the end of this guide you'll be able to:

- Design and populate an Algolia index from your existing data
- Add `InstantSearch`, `SearchBox`, and `Hits` components to a Rork app
- Sync records automatically from Supabase or Firebase
- Tune the index for better search quality

**Target audience**: Rork developers who have a working app and want to add production-grade search. Familiarity with TypeScript and React Native basics is assumed.

### Core Algolia Concepts

- **Index** — Algolia's equivalent of a database table. Create one per data type (products, articles, users).
- **Record** — A single JSON object stored in an index.
- **Application ID** — Identifier for your Algolia app.
- **Search API Key** — Public key used in the mobile client (safe to expose).
- **Admin API Key** — Server-only key for write operations. Never embed this in the app.

## Setting Up Algolia

### Step 1: Create Your Index

1. Log in to the [Algolia Dashboard](https://dashboard.algolia.com/)
2. Navigate to **Indices** → **Create Index**
3. Name your index (e.g., `recipes`, `products`, `articles`)
4. From **API Keys**, copy your **Application ID** and **Search-Only API Key**

### Step 2: Upload Sample Records

You can upload records from the dashboard or programmatically. Here's what a recipe index might look like:

```json
[
  {
    "objectID": "recipe_001",
    "name": "Avocado Toast",
    "category": "Breakfast",
    "ingredients": ["avocado", "bread", "lemon", "salt"],
    "cookTime": 10,
    "tags": ["quick", "healthy", "vegetarian"]
  },
  {
    "objectID": "recipe_002",
    "name": "Chicken Katsu",
    "category": "Dinner",
    "ingredients": ["chicken thigh", "egg", "flour", "tartar sauce"],
    "cookTime": 35,
    "tags": ["hearty", "japanese", "popular"]
  }
]
```

`objectID` is required — using your database's primary key here makes syncing straightforward later.

## Integrating Algolia into Your Rork App

### Step 3: Install Dependencies

Export your Rork project to GitHub, then install the necessary packages:

```bash
npx expo install algoliasearch @instantsearch/react-hooks react-instantsearch-core
```

> **Rork tip**: You can also type "add the algoliasearch package" directly in the Rork chat, and it will update `package.json` and run `npm install` automatically.

### Step 4: Initialize the Algolia Client

Create `lib/algolia.ts` to keep your configuration in one place:

```typescript
// lib/algolia.ts
import algoliasearch from 'algoliasearch/lite';

const APP_ID = process.env.EXPO_PUBLIC_ALGOLIA_APP_ID ?? '';
const SEARCH_KEY = process.env.EXPO_PUBLIC_ALGOLIA_SEARCH_KEY ?? '';

export const searchClient = algoliasearch(APP_ID, SEARCH_KEY);
export const INDEX_NAME = 'recipes'; // replace with your index name

// Expected output: searchClient object ready to send queries to Algolia
```

Add these to `.env.local` (or EAS Secrets for CI builds):

```bash
EXPO_PUBLIC_ALGOLIA_APP_ID=YOUR_APP_ID
EXPO_PUBLIC_ALGOLIA_SEARCH_KEY=YOUR_SEARCH_KEY
```

> ⚠️ **Never put the Admin API Key into a `EXPO_PUBLIC_` variable.** These get bundled into your app binary and are visible to end users. Admin keys belong exclusively in server-side environments (Cloud Functions, Edge Functions, etc.).

### Step 5: Wrap Your App with InstantSearch

Add the `InstantSearch` provider at the top of your component tree:

```typescript
// App.tsx
import React from 'react';
import { InstantSearch } from 'react-instantsearch-core';
import { searchClient, INDEX_NAME } from '@/lib/algolia';
import SearchScreen from '@/screens/SearchScreen';

export default function App() {
  return (
    <InstantSearch
      searchClient={searchClient}
      indexName={INDEX_NAME}
      future={{ preserveSharedStateOnUnmount: true }}
    >
      <SearchScreen />
    </InstantSearch>
  );
}
```

### Step 6: Build the Search UI (SearchBox + Hits)

```typescript
// screens/SearchScreen.tsx
import React from 'react';
import {
  View,
  TextInput,
  FlatList,
  Text,
  TouchableOpacity,
  StyleSheet,
} from 'react-native';
import { useSearchBox, useHits } from 'react-instantsearch-core';

// --- SearchBox ---
function SearchBox() {
  const { query, refine, clear } = useSearchBox();

  return (
    <View style={styles.searchContainer}>
      <TextInput
        style={styles.input}
        value={query}
        onChangeText={refine}
        placeholder="Search recipes..."
        placeholderTextColor="#999"
        returnKeyType="search"
        clearButtonMode="while-editing"
        autoCorrect={false}
      />
      {query.length > 0 && (
        <TouchableOpacity onPress={clear} style={styles.clearButton}>
          <Text style={styles.clearText}>✕</Text>
        </TouchableOpacity>
      )}
    </View>
  );
}

// --- Hit type ---
type RecipeHit = {
  objectID: string;
  name: string;
  category: string;
  cookTime: number;
  tags: string[];
};

// --- Hits list ---
function RecipeHits() {
  const { hits } = useHits<RecipeHit>();

  if (hits.length === 0) {
    return (
      <View style={styles.empty}>
        <Text style={styles.emptyText}>No recipes found</Text>
      </View>
    );
  }

  return (
    <FlatList
      data={hits}
      keyExtractor={(item) => item.objectID}
      renderItem={({ item }) => (
        <TouchableOpacity style={styles.hitCard}>
          <Text style={styles.hitName}>{item.name}</Text>
          <Text style={styles.hitMeta}>
            {item.category} · {item.cookTime} min
          </Text>
          <View style={styles.tagRow}>
            {item.tags.map((tag) => (
              <View key={tag} style={styles.tag}>
                <Text style={styles.tagText}>{tag}</Text>
              </View>
            ))}
          </View>
        </TouchableOpacity>
      )}
    />
  );
}

export default function SearchScreen() {
  return (
    <View style={styles.container}>
      <SearchBox />
      <RecipeHits />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, backgroundColor: '#f5f5f5' },
  searchContainer: {
    flexDirection: 'row',
    alignItems: 'center',
    margin: 16,
    backgroundColor: '#fff',
    borderRadius: 12,
    paddingHorizontal: 12,
    elevation: 2,
    shadowColor: '#000',
    shadowOpacity: 0.08,
    shadowRadius: 4,
  },
  input: { flex: 1, height: 48, fontSize: 16, color: '#333' },
  clearButton: { padding: 8 },
  clearText: { fontSize: 14, color: '#999' },
  hitCard: {
    backgroundColor: '#fff',
    marginHorizontal: 16,
    marginBottom: 10,
    padding: 16,
    borderRadius: 12,
    elevation: 1,
  },
  hitName: { fontSize: 17, fontWeight: '600', color: '#222' },
  hitMeta: { fontSize: 13, color: '#666', marginTop: 4 },
  tagRow: { flexDirection: 'row', flexWrap: 'wrap', marginTop: 8, gap: 6 },
  tag: {
    backgroundColor: '#e8f4fd',
    borderRadius: 20,
    paddingHorizontal: 10,
    paddingVertical: 4,
  },
  tagText: { fontSize: 12, color: '#0078d4' },
  empty: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  emptyText: { fontSize: 15, color: '#999' },
});

// Expected output: As the user types, Algolia queries fire and results render in real time
```

## Syncing Records with Supabase or Firebase

Any time a record is created, updated, or deleted in your database, you need to mirror that change to Algolia.

### Supabase (Edge Function + Database Webhook)

```typescript
// supabase/functions/sync-algolia/index.ts
import algoliasearch from 'https://esm.sh/algoliasearch@5';

const client = algoliasearch(
  Deno.env.get('ALGOLIA_APP_ID')!,
  Deno.env.get('ALGOLIA_ADMIN_API_KEY')! // server-side: Admin key is fine here
);

Deno.serve(async (req) => {
  const { type, record, old_record } = await req.json();
  const index = client.initIndex('recipes');

  if (type === 'INSERT' || type === 'UPDATE') {
    await index.saveObject({ ...record, objectID: record.id });
  } else if (type === 'DELETE') {
    await index.deleteObject(old_record.id);
  }

  return new Response(JSON.stringify({ ok: true }), {
    headers: { 'Content-Type': 'application/json' },
  });
});

// Expected output: Supabase record changes appear in Algolia within a few seconds
```

Go to Supabase Dashboard → **Database Webhooks** and point `INSERT`, `UPDATE`, and `DELETE` events at this function.

### Firebase (Cloud Functions)

```typescript
// functions/src/index.ts
import * as functions from 'firebase-functions';
import algoliasearch from 'algoliasearch';

const client = algoliasearch(
  functions.config().algolia.app_id,
  functions.config().algolia.admin_key
);
const index = client.initIndex('recipes');

export const syncToAlgolia = functions.firestore
  .document('recipes/{docId}')
  .onWrite(async (change, context) => {
    const docId = context.params.docId;
    if (!change.after.exists) {
      await index.deleteObject(docId);
    } else {
      await index.saveObject({ ...change.after.data()!, objectID: docId });
    }
  });
```

## Index Configuration for Better Search Quality

Fine-tune your index settings to improve relevance and performance:

```javascript
// scripts/configure-algolia.mjs
import algoliasearch from 'algoliasearch';

const client = algoliasearch(
  process.env.ALGOLIA_APP_ID,
  process.env.ALGOLIA_ADMIN_KEY
);
const index = client.initIndex('recipes');

await index.setSettings({
  // Fields to search and their priority
  searchableAttributes: [
    'name',         // highest priority
    'category',
    'ingredients',
    'tags',
  ],
  // Custom ranking metric
  customRanking: ['asc(cookTime)'],
  // Typo tolerance (disable for CJK languages)
  typoTolerance: false,
  // Only return fields the UI actually needs
  attributesToRetrieve: ['name', 'category', 'cookTime', 'tags'],
  // Enable highlighting on these fields
  attributesToHighlight: ['name', 'ingredients'],
  // Enable faceting for filters
  attributesForFaceting: ['category', 'tags'],
});

console.log('✅ Algolia index settings saved');
// Expected output: ✅ Algolia index settings saved
```

## Common Errors and Fixes

### "Invalid Application-ID or API Key"

Double-check the values in your `.env.local` file against the Algolia Dashboard. If you're using EAS Build, run `eas secret:list` to confirm the secrets are registered. Make sure you're using the **Search-Only API Key** on the client, not the Admin Key.

### No results returned

Verify that the index actually contains records by browsing it in the Algolia Dashboard. Then check that your `searchableAttributes` setting includes the fields you expect to match. A common mistake is uploading records but forgetting to configure which fields are searchable.

### Stale results after a database update

If you're using Supabase Webhooks, check the Edge Function logs in the Supabase Dashboard for errors. For Firebase, check Cloud Functions logs in the Firebase Console. A failed sync will leave Algolia out of sync until the next successful event.

## Advanced: Faceted Filtering

Algolia's faceting lets users filter results by category, price range, rating, or any other attribute — a pattern common in e-commerce and listing apps:

```typescript
import { useRefinementList } from 'react-instantsearch-core';

function CategoryFilter() {
  const { items, refine } = useRefinementList({ attribute: 'category' });

  return (
    <View style={{ flexDirection: 'row', flexWrap: 'wrap', padding: 8, gap: 8 }}>
      {items.map((item) => (
        <TouchableOpacity
          key={item.value}
          onPress={() => refine(item.value)}
          style={{
            backgroundColor: item.isRefined ? '#0078d4' : '#e0e0e0',
            borderRadius: 20,
            paddingHorizontal: 14,
            paddingVertical: 6,
          }}
        >
          <Text style={{ color: item.isRefined ? '#fff' : '#333', fontSize: 13 }}>
            {item.label} ({item.count})
          </Text>
        </TouchableOpacity>
      ))}
    </View>
  );
}
```

This requires `attributesForFaceting: ['category', 'tags']` in your index settings (shown in the configuration script above).

## FAQ

**Q: How far does Algolia's free plan go?**

A: The free tier includes 10,000 requests per month across search and indexing operations combined. For a solo indie app with a few hundred daily active users, this is typically sufficient. Once you exceed that, the Starter plan begins at $29/month.

**Q: Can Rork's AI generate the Algolia integration automatically?**

A: Rork can scaffold the basic `SearchBox` and `Hits` components if you prompt it with "build a search screen using Algolia, Application ID is X, Search Key is Y". The server-side sync logic for Supabase or Firebase requires manual setup, so use the code in this guide as a starting point.

**Q: What are good alternatives to Algolia?**

A: **Typesense** is an open-source alternative with a generous free cloud tier and excellent support for CJK (Chinese/Japanese/Korean) languages. **Meilisearch** is another fast, developer-friendly option. Both use a similar SDK pattern to Algolia, so migrating requires only swapping the client initialization and minor API differences.
