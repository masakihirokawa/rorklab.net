## Figma Make × Rork — From Design to App Store in 24 Hours

In 2026, the timeline for mobile app development has become dramatically shorter. With the integrated workflow where Figma Make generates React prototypes from text and Rork automatically converts them to native iOS/Android apps, it's now possible to reach App Store submission within 24 hours. This article provides a complete step-by-step walkthrough from implementation to App Store submission.

## Step 1: Prototype Generation with Figma Make

### Basic Operation of Figma Make

Figma Make is an AI-native prototyping tool that automatically generates React components from text descriptions. It can also generate Supabase backends simultaneously.

```bash
# Log in to Figma
# Select "Make" from the Figma dashboard

# Or via CLI
figma-make create --name="Weather App" --template=mobile

# Initial configuration
figma-make config set \
  --backend=supabase \
  --framework=react-native \
  --typescript=true
```

### Prototype Generation from Text Prompts

```
Example Figma Make Prompt:

"Create a weather app.
Display current temperature and weather icon prominently at the top.
In the center, show hourly weather forecasts for the next 3 hours in a horizontally scrollable card layout.
At the bottom, display a temperature graph for the day.
Background color changes based on current weather (blue for clear, gray for rainy, white for snowy).
Place a search bar at the very top where users can search for cities."
```

The Figma Make AI automatically generates from this prompt:
- UI component structure

### Adding Dynamic Content

Connect real-world weather APIs to the auto-generated components from Figma Make:

```typescript
// src/hooks/useWeather.ts - Auto-generated by Figma Make
// ↓ Manual adjustment

import { useEffect, useState } from 'react';
import { supabase } from '@/lib/supabase';

interface Weather {
  city: string;
  temperature: number;
  condition: string;
  icon: string;
  humidity: number;
  windSpeed: number;
}

export const useWeather = (city: string) => {
  const [weather, setWeather] = useState<Weather | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchWeather = async () => {
      try {
        setLoading(true);

        // Call weather API via Supabase Functions
        const { data, error: supabaseError } = await supabase.functions.invoke(
          'fetch-weather',
          {
            body: { city }
          }
        );

        if (supabaseError) throw supabaseError;

        setWeather(data);
        setError(null);
      } catch (err) {
        setError(err instanceof Error ? err.message : 'Unknown error');
        setWeather(null);
      } finally {
        setLoading(false);
      }
    };

    if (city) {
      fetchWeather();
    }
  }, [city]);

  return { weather, loading, error };
};
```

## Step 3: App Conversion with Rork

### Preparing Input for Rork

Rork automatically converts React Web code to native mobile apps. The React code generated by Figma Make is in an optimal format for Rork input.

```bash
# Install Rork CLI
npm install -g rork-cli

# Create Rork configuration
rork init \
  --name="WeatherApp" \
  --source=./src \
  --template=react-native

# Or use interactive mode
rork init --interactive
```

### Rork Configuration File (rork.config.ts)

```typescript
import { RorkConfig } from 'rork/types';

export const config: RorkConfig = {
  projectName: 'WeatherApp',
  description: 'Weather forecast app',
  version: '1.0.0',

  // Target platforms
  platforms: {
    ios: {
      minVersion: '14.0',
      bundleId: 'com.weatherapp.ios'
    },
    android: {
      minVersion: 23,
      packageName: 'com.weatherapp.android'
    }
  },

  // Source code configuration
  source: {
    entry: './src/App.tsx',
    include: ['./src/**/*.{ts,tsx}'],
    exclude: ['./src/**/*.test.ts', './src/**/*.stories.tsx']
  },

  // Automatic conversion configuration
  conversion: {
    // React Web → React Native conversion rules
    mappings: [
      { from: 'div', to: 'View' },
      { from: 'span', to: 'Text' },
      { from: 'button', to: 'Pressable' },
      { from: 'input', to: 'TextInput' },
      { from: 'img', to: 'Image' },
      { from: 'scroll', to: 'ScrollView' }
    ],
    // CSS → StyleSheet conversion
    styleConversion: 'automatic',
    // Maintain type safety
    typescript: true
  },

  // Supabase integration
  backend: {
    type: 'supabase',
    url: process.env.NEXT_PUBLIC_SUPABASE_URL,
    key: process.env.NEXT_PUBLIC_SUPABASE_KEY
  },

  // Build output
  output: {
    ios: './build/ios',
    android: './build/android'
  }
};
```

## Step 4: Manual Adjustments and Optimization

### Native Module Integration

Add platform-specific features that Rork cannot automatically convert:

```swift
// ios/WeatherApp/Modules/LocationManager.swift

import CoreLocation
import Rork

@objc class LocationManager: NSObject, CLLocationManagerDelegate {
  private let locationManager = CLLocationManager()

  @objc func requestLocation() -> Promise<[String: Any]> {
    return Promise { resolve, reject in
      locationManager.delegate = self

      if CLLocationManager.locationServicesEnabled() {
        locationManager.requestWhenInUseAuthorization()
        locationManager.requestLocation()
      } else {
        reject(NSError(
          domain: "LocationError",
          code: 1,
          userInfo: [NSLocalizedDescriptionKey: "Location services are disabled"]
        ))
      }
    }
  }

  func locationManager(
    _ manager: CLLocationManager,
    didUpdateLocations locations: [CLLocation]
  ) {
    guard let location = locations.last else { return }

    let result: [String: Any] = [
      "latitude": location.coordinate.latitude,
      "longitude": location.coordinate.longitude,
      "altitude": location.altitude
    ]

    // Return result to React Native side
  }
}
```

```kotlin
// android/app/src/main/java/com/weatherapp/modules/LocationManager.kt

package com.weatherapp.modules

import android.Manifest
import android.content.pm.PackageManager
import android.location.Location
import androidx.core.app.ActivityCompat
import com.google.android.gms.location.LocationServices
import com.facebook.react.bridge.*

class LocationManager(reactContext: ReactApplicationContext) :
  ReactContextBaseJavaModule(reactContext) {

  private val fusedLocationClient = LocationServices.getFusedLocationProviderClient(reactContext)

  @ReactMethod
  fun requestLocation(promise: Promise) {
    try {
      if (ActivityCompat.checkSelfPermission(
          reactContext,
          Manifest.permission.ACCESS_FINE_LOCATION
        ) != PackageManager.PERMISSION_GRANTED
      ) {
        promise.reject("PERMISSION_ERROR", "Location permission required")
        return
      }

      fusedLocationClient.lastLocation.addOnSuccessListener { location: Location? ->
        val result = Arguments.createMap().apply {
          putDouble("latitude", location?.latitude ?: 0.0)
          putDouble("longitude", location?.longitude ?: 0.0)
          putDouble("altitude", location?.altitude ?: 0.0)
        }
        promise.resolve(result)
      }
    } catch (e: Exception) {
      promise.reject("LOCATION_ERROR", e.message)
    }
  }
}
```
