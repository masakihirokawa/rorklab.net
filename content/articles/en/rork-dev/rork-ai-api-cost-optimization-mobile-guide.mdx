## Introduction — When AI API Costs Threaten Your App's Profitability

Integrating AI features into mobile apps has quickly shifted from a competitive advantage to a baseline expectation. Chatbots, image recognition, text summarization, and personalized recommendations all deliver tremendous user value — but they also come with pay-per-use API costs that can quickly spiral out of control, especially for indie developers.

When you integrate AI APIs like Claude, Gemini, or GPT-4o into a Rork app, costs scale with your user base. If 1,000 monthly active users each make 10 AI requests per day, your token consumption alone can easily reach hundreds of dollars per month.

This guide walks through practical, code-level techniques for minimizing AI API costs in Rork apps. You'll learn how to implement response caching, optimize prompts for fewer tokens, leverage on-device AI for simple tasks, and batch requests for maximum efficiency.

## Response Caching — Eliminate Duplicate API Calls

The single most impactful cost reduction technique is response caching. Many apps repeatedly call the API with identical or near-identical inputs, and each duplicate call is money wasted.

The core strategy is simple: hash the request content, use it as a cache key, and store responses in local storage or a server-side KV store.

```typescript
// AI API response cache implementation
import AsyncStorage from '@react-native-async-storage/async-storage';
import * as Crypto from 'expo-crypto';

interface CacheEntry {
  response: string;
  timestamp: number;
  tokens_used: number;
}

const CACHE_TTL = 24 * 60 * 60 * 1000; // 24 hours

async function getCachedAIResponse(prompt: string): Promise<string | null> {
  const hash = await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    prompt.trim().toLowerCase()
  );
  const key = `ai_cache_${hash}`;

  try {
    const cached = await AsyncStorage.getItem(key);
    if (cached) {
      const entry: CacheEntry = JSON.parse(cached);
      if (Date.now() - entry.timestamp < CACHE_TTL) {
        console.log(`Cache hit — saved ${entry.tokens_used} tokens`);
        return entry.response;
      }
    }
  } catch (error) {
    console.warn('Cache read error:', error);
  }
  return null;
}

async function callAIWithCache(prompt: string): Promise<string> {
  // Check cache first
  const cached = await getCachedAIResponse(prompt);
  if (cached) return cached;

  // Call the API
  const response = await fetch('https://api.example.com/ai/chat', {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` },
    body: JSON.stringify({ prompt, max_tokens: 500 }),
  });

  const data = await response.json();
  // Store response in cache
  const hash = await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    prompt.trim().toLowerCase()
  );
  await AsyncStorage.setItem(`ai_cache_${hash}`, JSON.stringify({
    response: data.text,
    timestamp: Date.now(),
    tokens_used: data.usage.total_tokens,
  }));

  return data.text;
}
```

Adding this caching layer alone typically reduces API calls by 30–50% in a typical app. For features like FAQ responses or category classification where input patterns are limited, cache hit rates above 80% are common.

## Prompt Compression and Token Reduction Techniques

Since AI API pricing is directly proportional to token count, designing prompts that achieve the same result with fewer tokens is a fundamental cost lever.

There are three key areas for prompt optimization.

**System prompt optimization**: Trim verbose instructions down to essentials. A system prompt like "You are a helpful assistant. Please answer the user's questions politely and accurately with clear explanations" can be replaced with "Answer concisely and accurately."

**Context window management**: Instead of sending the entire chat history, send only the last 3–5 turns or a summary of the conversation.

**Output token limits**: Set the `max_tokens` parameter to the minimum necessary for each task.

```typescript
// Conversation history optimization utility
function optimizeConversationHistory(
  messages: Array<{ role: string; content: string }>,
  maxTurns: number = 4,
  maxTokensEstimate: number = 1000
): Array<{ role: string; content: string }> {
  // Always keep system prompt
  const system = messages.filter(m => m.role === 'system');
  const conversation = messages.filter(m => m.role !== 'system');

  // Keep only the most recent turns
  const recent = conversation.slice(-maxTurns * 2);

  // Rough token estimate (English: ~1.3 tokens per word)
  let totalChars = system.reduce((sum, m) => sum + m.content.length, 0);
  const filtered = [];

  for (const msg of recent) {
    totalChars += msg.content.length;
    if (totalChars * 0.3 > maxTokensEstimate) break;
    filtered.push(msg);
  }

  return [...system, ...filtered];
}

// Usage: reduces token consumption by approximately 60%
const optimized = optimizeConversationHistory(chatHistory, 4, 800);
const response = await callAI(optimized);
```

You can further reduce output tokens by specifying the response format. Adding "Return as JSON" to your prompt eliminates natural language padding and can cut output tokens significantly.

## On-Device AI — Reduce API Calls at the Source

The ultimate cost optimization strategy is to eliminate API calls altogether for tasks that don't need cloud-scale AI. Text classification, sentiment analysis, and simple Q&A can run locally using on-device ML models, while only complex tasks are routed to cloud APIs. This "tiered AI" architecture is highly effective.

```typescript
// Tiered AI architecture implementation
type AITier = 'on-device' | 'cloud-light' | 'cloud-heavy';

interface TaskClassification {
  tier: AITier;
  estimatedCost: number; // USD
}

function classifyTask(input: string, taskType: string): TaskClassification {
  // Determine tier based on task complexity
  const simplePatterns = ['classify', 'sentiment', 'yes/no', 'category'];
  const mediumPatterns = ['summarize', 'translate', 'extract keywords'];

  if (simplePatterns.some(p => taskType.includes(p))) {
    return { tier: 'on-device', estimatedCost: 0 };
  }
  if (mediumPatterns.some(p => taskType.includes(p))) {
    return { tier: 'cloud-light', estimatedCost: 0.001 };
  }
  return { tier: 'cloud-heavy', estimatedCost: 0.01 };
}

async function processWithTieredAI(
  input: string,
  taskType: string
): Promise<string> {
  const { tier } = classifyTask(input, taskType);

  switch (tier) {
    case 'on-device':
      // Process with TensorFlow Lite / Core ML (cost: $0)
      return await runOnDeviceModel(input, taskType);

    case 'cloud-light':
      // Use lightweight model (Claude Haiku / Gemini Flash)
      return await callLightweightAPI(input);

    case 'cloud-heavy':
      // Use premium model (Claude Opus / GPT-4o)
      return await callPremiumAPI(input);
  }
}
```

With this approach, 50–70% of requests can be handled on-device, dramatically cutting your API bill. ## Batch Processing and Request Aggregation for Maximum Efficiency

Instead of firing individual API requests as they come in, grouping multiple requests into batches reduces overhead and improves cost efficiency.

```typescript
// Batch request manager
class AIBatchProcessor {
  private queue: Array<{
    prompt: string;
    resolve: (value: string) => void;
    reject: (reason: any) => void;
  }> = [];
  private timer: NodeJS.Timeout | null = null;
  private readonly batchSize = 5;
  private readonly batchDelay = 2000; // Wait 2 seconds before sending batch

  async add(prompt: string): Promise<string> {
    return new Promise((resolve, reject) => {
      this.queue.push({ prompt, resolve, reject });

      if (this.queue.length >= this.batchSize) {
        this.flush();
      } else if (!this.timer) {
        this.timer = setTimeout(() => this.flush(), this.batchDelay);
      }
    });
  }

  private async flush(): Promise<void> {
    if (this.timer) {
      clearTimeout(this.timer);
      this.timer = null;
    }

    const batch = this.queue.splice(0, this.batchSize);
    if (batch.length === 0) return;

    try {
      // Process multiple prompts in a single API call
      const combinedPrompt = batch.map(
        (item, i) => `[Task ${i + 1}]: ${item.prompt}`
      ).join('\n---\n');

      const response = await fetch('https://api.example.com/ai/chat', {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` },
        body: JSON.stringify({
          prompt: `Answer each of the following ${batch.length} tasks separately.\n${combinedPrompt}`,
          max_tokens: batch.length * 300,
        }),
      });

      const data = await response.json();
      // Split response and distribute to each request
      const results = parseMultiResponse(data.text, batch.length);
      batch.forEach((item, i) => item.resolve(results[i]));
    } catch (error) {
      batch.forEach(item => item.reject(error));
    }
  }
}

// Usage: Multiple UI elements calling AI individually get batched internally
const batchProcessor = new AIBatchProcessor();
const [summary, tags, sentiment] = await Promise.all([
  batchProcessor.add('Summarize this article in 3 lines'),
  batchProcessor.add('Extract 5 tags from this article'),
  batchProcessor.add('Determine the sentiment of this article'),
]);
```

Batch processing consolidates three API calls into one. Fewer total requests also means more headroom under rate limits, resulting in more stable app performance overall.

## Implement a Cost Monitoring Dashboard to Prevent Budget Overruns

The foundation of any cost optimization strategy is visibility. You need to know exactly how much you're spending. Combining usage tracking with Supabase gives you a real-time cost monitoring dashboard.

```typescript
// API cost tracking utility
interface UsageRecord {
  user_id: string;
  model: string;
  input_tokens: number;
  output_tokens: number;
  cost_usd: number;
  timestamp: string;
}

const MODEL_PRICING: Record<string, { input: number; output: number }> = {
  'claude-opus':   { input: 0.015,  output: 0.075  }, // per 1K tokens
  'claude-sonnet': { input: 0.003,  output: 0.015  },
  'claude-haiku':  { input: 0.00025, output: 0.00125 },
  'gemini-pro':    { input: 0.00125, output: 0.005  },
  'gpt-4o':        { input: 0.005,  output: 0.015  },
};

function calculateCost(
  model: string,
  inputTokens: number,
  outputTokens: number
): number {
  const pricing = MODEL_PRICING[model];
  if (!pricing) return 0;
  return (inputTokens / 1000) * pricing.input
       + (outputTokens / 1000) * pricing.output;
}

// Record usage to Supabase
async function trackUsage(record: UsageRecord): Promise<void> {
  await supabase.from('ai_usage').insert(record);

  // Monthly budget check
  const { data } = await supabase
    .from('ai_usage')
    .select('cost_usd')
    .gte('timestamp', getFirstDayOfMonth());

  const totalCost = data?.reduce((sum, r) => sum + r.cost_usd, 0) ?? 0;
  if (totalCost > MONTHLY_BUDGET * 0.8) {
    await sendBudgetAlert(totalCost, MONTHLY_BUDGET);
  }
}
```

By tracking usage and triggering an alert at 80% of your monthly budget, you can catch unexpected cost spikes before they become a problem.

## FAQ

**Q: Which AI API cost optimization technique has the highest impact?**

A: Response caching delivers the most immediate results. In a typical app, 30–50% of requests are similar or identical, and caching eliminates their API cost entirely. The next most effective technique is the tiered AI architecture that routes simple tasks to on-device models.

**Q: What cache TTL (time-to-live) should I use?**

A: It depends on your content. FAQ responses and category classifications that rarely change work well with 24–72 hour TTLs. Time-sensitive content like news summaries should use 1–6 hours. Consider giving users a manual refresh option so they can bust the cache when they need fresh results.

**Q: How accurate are on-device AI models compared to cloud APIs?**

A: For well-defined tasks like text classification and sentiment analysis, on-device models can achieve over 90% accuracy. However, they fall short on complex reasoning and creative text generation, which is why the tiered approach is essential — route each task to the right tier based on its complexity. Quantized Core ML and TensorFlow Lite models keep memory usage low on mobile devices.

**Q: What's a realistic monthly AI API budget for an indie app?**

A: It depends on your user base and request frequency, but for an app with around 1,000 MAU, a budget of $10–$30 per month is realistic after applying the optimization techniques in this article. Combining caching with on-device AI can reduce costs by 60–80% compared to an unoptimized baseline.
