## Introduction — Why AI Costs Keep Indie Developers Up at Night

Rork makes it incredibly easy to add powerful AI features to your mobile apps — chatbots, recommendations, image recognition, and more. But once your app gains traction, you'll face a reality check: **AI API costs can eat your entire profit margin**.

Pay-per-token pricing from OpenAI, Google, and Anthropic might seem trivial per request, but at tens of thousands of requests per month, costs add up fast. For indie developers, watching your revenue disappear into API bills isn't sustainable.

This guide walks you through practical, battle-tested techniques for keeping AI costs under control in Rork apps. From token optimization and response caching to on-device inference and cost monitoring, you'll learn how to cut costs by 60–80% without compromising user experience.

## Understanding AI API Cost Structure

The first step to optimization is understanding how you're being charged. Most AI APIs bill based on the total number of input and output tokens.

- **Input tokens**: Everything in your prompt — system instructions, user input, and conversation history
- **Output tokens**: The AI's response (typically more expensive per token than input)
- **Model selection**: GPT-4o vs GPT-4o-mini can mean a 10x cost difference

For example, a chatbot app averaging 2,000 tokens per exchange (1,500 input + 500 output), with 1,000 daily users making 5 queries each, consumes roughly 150 million tokens per month. Choose the wrong model and you could be looking at thousands of dollars in monthly costs.

### Implementing a Cost Estimator

Building a cost estimation utility early in development helps you stay cost-aware from day one.

```typescript
// utils/aiCostEstimator.ts
// AI API cost calculation utility

interface CostRate {
  inputPer1kTokens: number;
  outputPer1kTokens: number;
}

// Major model pricing (USD) — as of March 2026
const MODEL_RATES: Record<string, CostRate> = {
  'gpt-4o': { inputPer1kTokens: 0.0025, outputPer1kTokens: 0.01 },
  'gpt-4o-mini': { inputPer1kTokens: 0.00015, outputPer1kTokens: 0.0006 },
  'claude-sonnet': { inputPer1kTokens: 0.003, outputPer1kTokens: 0.015 },
  'gemini-pro': { inputPer1kTokens: 0.00125, outputPer1kTokens: 0.005 },
};

export function estimateCost(
  model: string,
  inputTokens: number,
  outputTokens: number
): number {
  const rate = MODEL_RATES[model];
  if (!rate) return 0;
  return (
    (inputTokens / 1000) * rate.inputPer1kTokens +
    (outputTokens / 1000) * rate.outputPer1kTokens
  );
}

// Example: estimateCost('gpt-4o-mini', 1500, 500)
// Result: ~$0.000525 per request
```

## Five Practical Techniques to Reduce Token Usage

The most impactful cost reduction comes from sending fewer tokens to the API in the first place.

### 1. Compress Your System Prompts

System prompts are included in every single request, making them the biggest ongoing cost driver. Strip out redundant instructions and keep them concise.

```typescript
// ❌ Verbose prompt (~200 tokens)
const verbosePrompt = `
You are a fitness app assistant designed to help users
manage their health and wellness journey. When users ask
questions, please respond in a friendly and helpful manner.
Avoid using technical jargon and keep your responses
easy to understand. When giving important health advice,
always recommend consulting a medical professional.
`;

// ✅ Compressed version (~60 tokens)
const concisePrompt = `Role: fitness assistant. Reply concisely in plain language. Add medical disclaimer for health advice.`;
```

### 2. Limit Conversation History with a Sliding Window

In chat features, token counts grow with every message. A sliding window approach keeps only the most recent exchanges.

```typescript
// hooks/useChatHistory.ts
// Manage conversation history with a sliding window

const MAX_HISTORY_TURNS = 6; // Keep only the last 6 exchanges

function trimHistory(messages: Message[]): Message[] {
  const systemMsg = messages.find(m => m.role === 'system');
  const conversations = messages.filter(m => m.role !== 'system');

  // Keep only recent exchanges
  const trimmed = conversations.slice(-MAX_HISTORY_TURNS * 2);

  return systemMsg ? [systemMsg, ...trimmed] : trimmed;
}
```

### 3. Pre-process User Input

Before sending to the API, strip unnecessary whitespace, repeated characters, and normalize the text. Even small reductions compound over thousands of requests.

### 4. Control Response Length

Set appropriate `max_tokens` limits for each use case. Product descriptions might need 200 tokens, while chat responses could use 500. Setting explicit limits makes costs predictable.

### 5. Route Tasks to the Right Model

Not every request needs your most powerful (and expensive) model. A routing pattern that matches task complexity to model capability can dramatically reduce costs.

```typescript
// utils/modelRouter.ts
// Automatically select model based on task complexity

type TaskComplexity = 'simple' | 'moderate' | 'complex';

function selectModel(task: TaskComplexity): string {
  switch (task) {
    case 'simple':
      // Classification, summaries, templated responses
      return 'gpt-4o-mini';
    case 'moderate':
      // General chat responses
      return 'gemini-pro';
    case 'complex':
      // Code generation, advanced reasoning
      return 'gpt-4o';
  }
}

// Example: sentiment analysis → simple, recipe suggestions → moderate
```

## Response Caching to Slash API Calls

Calling the API for identical or near-identical requests is wasteful. A caching layer can significantly reduce the number of actual API calls.

### Implementing an Exact-Match Cache

```typescript
// services/aiCache.ts
// AI response cache using AsyncStorage

import AsyncStorage from '@react-native-async-storage/async-storage';

const CACHE_PREFIX = 'ai_cache_';
const CACHE_TTL = 24 * 60 * 60 * 1000; // 24 hours

interface CachedResponse {
  response: string;
  timestamp: number;
  model: string;
}

// Generate a cache key from the prompt
function getCacheKey(prompt: string): string {
  let hash = 0;
  for (let i = 0; i < prompt.length; i++) {
    const char = prompt.charCodeAt(i);
    hash = ((hash << 5) - hash) + char;
    hash |= 0;
  }
  return `${CACHE_PREFIX}${Math.abs(hash).toString(36)}`;
}

export async function getCachedResponse(
  prompt: string
): Promise<string | null> {
  try {
    const key = getCacheKey(prompt);
    const cached = await AsyncStorage.getItem(key);
    if (!cached) return null;

    const data: CachedResponse = JSON.parse(cached);
    // Invalidate if past TTL
    if (Date.now() - data.timestamp > CACHE_TTL) {
      await AsyncStorage.removeItem(key);
      return null;
    }
    return data.response;
  } catch {
    return null;
  }
}

export async function setCachedResponse(
  prompt: string,
  response: string,
  model: string
): Promise<void> {
  const key = getCacheKey(prompt);
  const data: CachedResponse = {
    response,
    timestamp: Date.now(),
    model,
  };
  await AsyncStorage.setItem(key, JSON.stringify(data));
}
```

To maximize cache hit rates, separate user-specific data (names, dates) from the prompt template and use only the template as the cache key. For a recipe app, cache on the query "chicken easy 30 minutes" rather than including the user's name — different users with the same query will share cached results.

## On-Device Inference: Reducing API Costs to Zero

The ultimate cost reduction strategy is running inference directly on the device, eliminating cloud API calls entirely. With Rork Max, you can embed Core ML models to handle specific tasks offline and for free.

### Tasks Well-Suited for On-Device Inference

- **Text classification** (sentiment analysis, spam detection, categorization)
- **Image classification** (photo tagging, object detection)
- **Speech recognition** (Whisper-based transcription)
- **Keyword extraction** (pulling key phrases from text)

Tasks that require large language model capabilities — like long-form generation or multi-turn conversations — are better handled by cloud APIs.

```typescript
// services/onDeviceClassifier.ts
// On-device text classification using Core ML

import { NativeModules } from 'react-native';

const { CoreMLBridge } = NativeModules;

export async function classifySentiment(
  text: string
): Promise<'positive' | 'negative' | 'neutral'> {
  try {
    // On-device inference via Core ML (API cost: $0)
    const result = await CoreMLBridge.predict('SentimentClassifier', {
      text,
    });
    return result.label;
  } catch (error) {
    // Fallback: use cloud API
    console.warn('On-device inference failed, falling back to API');
    return classifySentimentViaAPI(text);
  }
}
```

## Building a Cost Monitoring Dashboard

Sustained optimization requires visibility. Using Supabase, you can log API usage and build a dashboard to track costs in real time.

```typescript
// services/costTracker.ts
// Log API usage to Supabase

import { supabase } from './supabaseClient';

interface UsageLog {
  model: string;
  input_tokens: number;
  output_tokens: number;
  cost_usd: number;
  feature: string; // Which feature triggered the call
  cached: boolean;
}

export async function logApiUsage(usage: UsageLog): Promise<void> {
  await supabase.from('ai_usage_logs').insert({
    ...usage,
    created_at: new Date().toISOString(),
  });
}

// Get daily cost summary
export async function getDailyCostSummary(
  date: string
): Promise<{ totalCost: number; totalRequests: number; cacheHitRate: number }> {
  const { data } = await supabase
    .from('ai_usage_logs')
    .select('cost_usd, cached')
    .gte('created_at', `${date}T00:00:00`)
    .lt('created_at', `${date}T23:59:59`);

  if (!data) return { totalCost: 0, totalRequests: 0, cacheHitRate: 0 };

  const totalCost = data.reduce((sum, d) => sum + d.cost_usd, 0);
  const cachedCount = data.filter(d => d.cached).length;

  return {
    totalCost,
    totalRequests: data.length,
    cacheHitRate: data.length > 0 ? cachedCount / data.length : 0,
  };
}
```

Set a monthly budget cap and configure alerts when spending approaches the threshold. This protects against unexpected cost spikes from traffic surges.

## Frequently Asked Questions (FAQ)

### Q1: What's a reasonable AI API budget for an indie app?

For indie apps, aim to keep monthly costs between $10–$50. A chat app with around 1,000 DAU can typically stay within $20–$30/month by combining the optimization techniques in this article — model routing, caching, and token reduction.

### Q2: Does caching affect the user experience?

When implemented well, caching actually *improves* the experience. Cache hits return instantly without waiting for an API call, making the app feel much snappier. Just set appropriate TTLs for different content types so users don't see stale responses.

### Q3: What are the downsides of on-device inference?

The main trade-offs are increased app size (Core ML models range from a few MB to hundreds of MB) and device performance dependency. Older devices may run inference slowly. Always implement a cloud API fallback for critical tasks to ensure reliability.

### Q4: Isn't managing multiple AI models complicated?

Not if you build a routing layer upfront. Define task types as an enum, and let the router automatically select the appropriate model. Each feature only needs to specify *what* it wants done — the routing layer handles *how*.

## Summary

AI API cost optimization is an essential skill for indie developers building AI-powered apps that need to be financially sustainable. By combining the four pillars covered in this guide, you can reduce costs by 60–80% while maintaining a great user experience.

- **Token optimization**: Compress prompts, limit history, and route to appropriate models
- **Caching strategy**: Reuse responses for identical and similar requests
- **On-device inference**: Move classification and detection tasks to Core ML at zero cost
- **Cost monitoring**: Visualize usage and prevent budget overruns

With Rork, you can incrementally adopt these optimization techniques. Start with model routing and caching, measure the impact, and then explore on-device inference as your next step.
