## Maps & Location Features in Rork Max

Location-based functionality has become central to modern mobile applications. From ride-sharing platforms and restaurant discovery to fitness tracking and delivery management, geolocation capabilities enable rich user experiences that were unimaginable just a decade ago.

This comprehensive guide covers implementing maps, geolocation tracking, markers, and route navigation in Rork Max applications.

## Setting Up Basic Map Display

### MapView Fundamentals

Rork Max leverages `react-native-maps` to provide cross-platform map functionality. Proper initialization with API keys is essential for full functionality.

```jsx
import MapView, { Marker } from 'react-native-maps';
import { useState, useEffect } from 'react';

export function BasicMapScreen() {
  const [region, setRegion] = useState({
    latitude: 35.6762,      // Tokyo
    longitude: 139.6503,
    latitudeDelta: 0.0922,
    longitudeDelta: 0.0421,
  });

  const [markers, setMarkers] = useState([
    {
      id: 1,
      latitude: 35.6762,
      longitude: 139.6503,
      title: 'Current Location',
      description: 'Tokyo Station',
    },
  ]);

  return (
    <MapView
      style={{ flex: 1 }}
      region={region}
      onRegionChange={setRegion}
    >
      {markers.map((marker) => (
        <Marker
          key={marker.id}
          coordinate={{
            latitude: marker.latitude,
            longitude: marker.longitude,
          }}
          title={marker.title}
          description={marker.description}
        />
      ))}
    </MapView>
  );
}
```

This creates an interactive map centered on Tokyo with a single marker. The `onRegionChange` callback allows you to respond to user pan and zoom gestures.

## Implementing Geolocation

### Real-Time Location Tracking

The Geolocation API provides access to the device's GPS for continuous position updates.

```jsx
import Geolocation from '@react-native-geolocation/geolocation';
import { useState, useEffect } from 'react';

export function CurrentLocationScreen() {
  const [currentLocation, setCurrentLocation] = useState(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    // Watch position for real-time updates
    const watchId = Geolocation.watchPosition(
      (position) => {
        const { latitude, longitude, accuracy } = position.coords;
        setCurrentLocation({
          latitude,
          longitude,
          accuracy,
        });
        console.log(
          `Current location: lat=${latitude.toFixed(4)}, lon=${longitude.toFixed(4)}`
        );
        // Expected output: Current location: lat=35.6762, lon=139.6503
        setIsLoading(false);
      },
      (error) => {
        console.error('Geolocation error:', error);
        setIsLoading(false);
      },
      {
        enableHighAccuracy: true,
        timeout: 15000,
        maximumAge: 10000,
      }
    );

    // Cleanup
    return () => Geolocation.clearWatch(watchId);
  }, []);

  if (isLoading) return <p>Fetching location...</p>;

  return (
    <div>
      {currentLocation && (
        <p>
          Location: {currentLocation.latitude.toFixed(4)},
          {currentLocation.longitude.toFixed(4)}
          (Accuracy: {currentLocation.accuracy.toFixed(0)}m)
        </p>
      )}
    </div>
  );
}
```

Setting `enableHighAccuracy` to true provides better precision at the cost of increased battery drain. Adjust based on your app's requirements.

## Working with Markers

### Displaying Multiple Points of Interest

Markers effectively visualize multiple locations on your map. Customizing their appearance helps users distinguish between different location types.

```jsx
import MapView, { Marker, Callout } from 'react-native-maps';
import { useState } from 'react';

export function MultiMarkerMap() {
  const [selectedMarkerId, setSelectedMarkerId] = useState(null);

  const locations = [
    {
      id: 1,
      name: 'Cafe A',
      latitude: 35.6775,
      longitude: 139.7505,
      category: 'cafe',
    },
    {
      id: 2,
      name: 'Restaurant B',
      latitude: 35.6812,
      longitude: 139.7671,
      category: 'restaurant',
    },
    {
      id: 3,
      name: 'Station C',
      latitude: 35.6895,
      longitude: 139.7004,
      category: 'station',
    },
  ];

  const handleMarkerPress = (markerId) => {
    setSelectedMarkerId(markerId);
    console.log(`Marker ${markerId} selected`);
    // Expected output: Marker 1 selected
  };

  return (
    <MapView style={{ flex: 1 }}>
      {locations.map((location) => (
        <Marker
          key={location.id}
          coordinate={{
            latitude: location.latitude,
            longitude: location.longitude,
          }}
          onPress={() => handleMarkerPress(location.id)}
          pinColor={
            selectedMarkerId === location.id ? '#FF6B6B' : '#4ECDC4'
          }
        >
          <Callout>
            <div style={{ padding: '10px' }}>
              <p style={{ fontWeight: 'bold' }}>{location.name}</p>
              <p>Type: {location.category}</p>
            </div>
          </Callout>
        </Marker>
      ))}
    </MapView>
  );
}
```

The color-coded markers provide instant visual feedback about location categories, improving usability.

## Route Navigation and Directions

### Computing Routes Between Points

Integrating the Google Maps Directions API enables turn-by-turn navigation and distance/duration calculations.

```jsx
import MapView, { Polyline, Marker } from 'react-native-maps';
import { useState } from 'react';

export async function getDirections(origin, destination) {
  try {
    const GOOGLE_MAPS_API_KEY = 'YOUR_API_KEY_HERE';
    const response = await fetch(
      `https://maps.googleapis.com/maps/api/directions/json?` +
        `origin=${origin.latitude},${origin.longitude}&` +
        `destination=${destination.latitude},${destination.longitude}&` +
        `key=${GOOGLE_MAPS_API_KEY}`
    );

    const data = await response.json();
    if (data.routes.length > 0) {
      const route = data.routes[0];
      const distance = route.legs[0].distance.text;
      const duration = route.legs[0].duration.text;
      console.log(`Route info: ${distance}, Duration: ${duration}`);
      // Expected output: Route info: 5.2 km, Duration: 12 mins
      return route;
    }
  } catch (error) {
    console.error('Directions API error:', error);
  }
}

export function DirectionMapScreen() {
  const [routeCoordinates, setRouteCoordinates] = useState([]);

  const displayRoute = async () => {
    const origin = { latitude: 35.6762, longitude: 139.6503 };
    const destination = { latitude: 35.6895, longitude: 139.7004 };

    const route = await getDirections(origin, destination);
    if (route) {
      const points = route.overview_polyline.points;
      const decodedPoints = decodePolyline(points);
      setRouteCoordinates(decodedPoints);
    }
  };

  return (
    <MapView style={{ flex: 1 }}>
      {routeCoordinates.length > 0 && (
        <Polyline
          coordinates={routeCoordinates}
          strokeColor="#FF6B6B"
          strokeWidth={3}
        />
      )}
      <button onClick={displayRoute}>Show Route</button>
    </MapView>
  );
}

function decodePolyline(encoded) {
  // Decode polyline algorithm
  let points = [];
  let index = 0, lat = 0, lng = 0;

  while (index < encoded.length) {
    let result = 0, shift = 0;
    let b;

    do {
      b = encoded.charCodeAt(index++) - 63;
      result |= (b & 0x1f) << shift;
      shift += 5;
    } while (b >= 0x20);

    const dlat = result & 1 ? ~(result >> 1) : result >> 1;
    lat += dlat;

    result = 0;
    shift = 0;
    do {
      b = encoded.charCodeAt(index++) - 63;
      result |= (b & 0x1f) << shift;
      shift += 5;
    } while (b >= 0x20);

    const dlng = result & 1 ? ~(result >> 1) : result >> 1;
    lng += dlng;

    points.push({
      latitude: lat / 1e5,
      longitude: lng / 1e5,
    });
  }

  return points;
}
```

## Managing Location Permissions

### Handling Privacy-First Permissions

Modern platforms enforce granular permission controls. iOS 13+ offers three levels: "Always," "While Using," and "Never."

```jsx
import { PermissionsAndroid } from 'react-native';

export async function requestLocationPermission() {
  try {
    const granted = await PermissionsAndroid.request(
      PermissionsAndroid.PERMISSIONS.ACCESS_FINE_LOCATION,
      {
        title: 'Location Access',
        message: 'This app needs access to your location',
        buttonNeutral: 'Ask Me Later',
        buttonNegative: 'Cancel',
        buttonPositive: 'OK',
      }
    );

    if (granted === PermissionsAndroid.RESULTS.GRANTED) {
      console.log('Location permission granted');
      // Expected output: Location permission granted
      return true;
    }
  } catch (error) {
    console.error('Permission request error:', error);
  }
  return false;
}
```

Transparency about location usage builds user trust. Always explain why your app needs this sensitive permission.

## Frequently Asked Questions

### Q1: How can I reduce battery drain while tracking location?

**A:** Adjust the `maximumAge` option to cache recent positions (e.g., 10 seconds). This reduces unnecessary GPS queries. Also call `clearWatch` when users navigate away from the tracking screen.

### Q2: Can maps work offline in Rork Max?

**A:** Offline map display requires pre-downloading tiles using libraries like `react-native-maps-offline`. Sync cached tiles when connectivity returns for current data.

### Q3: How do I optimize performance with many markers?

**A:** Implement marker clustering for 100+ markers. Libraries like `react-native-maps-clustering` group nearby markers automatically, dramatically improving performance and user experience.
