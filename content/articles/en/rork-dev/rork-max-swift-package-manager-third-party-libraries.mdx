Rork Max generates solid SwiftUI code, but the first real friction point for most developers is adding third-party libraries. The generated code sticks to Apple's standard frameworks, but Firebase, Alamofire, Kingfisher, and others come up quickly in any serious app. Swift Package Manager (SPM) â€” Apple's built-in package manager since Xcode 12 â€” handles this, but there are a few things to know before diving in.

## How SPM Works with Rork Max Projects

Rork Max exports standard Xcode project files, which means SPM works the same way it does in any SwiftUI project. The key distinction to understand upfront: **adding packages requires working in Xcode directly**. The Rork Companion app handles SwiftUI previews, but resolving external package dependencies happens at the Xcode level.

## Adding Firebase SDK Step by Step

Firebase is the most common integration request, so it's the best example to walk through.

### Step 1: Open the project in Xcode

Open the exported project folder from Rork:

```
File â†’ Open â†’ [project folder] â†’ [ProjectName].xcodeproj
```

### Step 2: Add Firebase via SPM

```
File â†’ Add Package Dependencies...
```

Enter this URL in the search field:

```
https://github.com/firebase/firebase-ios-sdk
```

Select `Up to Next Major Version` starting from the latest stable release (`11.x.x`).

In the package selection screen, only check what you actually need â€” adding all Firebase packages significantly bloats the binary:

- FirebaseCore (required for any Firebase product)
- FirebaseFirestore (if using Cloud Firestore)
- FirebaseAuth (if using authentication)
- FirebaseAnalytics (if using analytics)
- FirebaseCrashlytics (if using crash reporting)

### Step 3: Initialize Firebase in the App entry point

Rork Max generates apps using SwiftUI's `App` struct rather than a traditional `UIApplicationDelegate`. Use `@UIApplicationDelegateAdaptor` to bridge Firebase initialization:

```swift
import SwiftUI
import FirebaseCore

class AppDelegate: NSObject, UIApplicationDelegate {
    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]? = nil
    ) -> Bool {
        FirebaseApp.configure()
        return true
    }
}

@main
struct MyRorkApp: App {
    @UIApplicationDelegateAdaptor(AppDelegate.self) var delegate
    
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

Add `GoogleService-Info.plist` (downloaded from the Firebase Console) to the project root. Drag it into Xcode's project navigator and check "Copy items if needed."

## Real-Time Firestore in a SwiftUI View

Once integrated, Rork handles the SwiftUI side naturally. Here's a real-time data view:

```swift
import SwiftUI
import FirebaseFirestore

struct MessageListView: View {
    @State private var messages: [String] = []
    private let db = Firestore.firestore()
    
    var body: some View {
        List(messages, id: \.self) { message in
            Text(message)
        }
        .navigationTitle("Messages")
        .onAppear {
            db.collection("messages")
                .order(by: "timestamp", descending: true)
                .limit(to: 20)
                .addSnapshotListener { snapshot, error in
                    guard let documents = snapshot?.documents else {
                        print("Error: \(error?.localizedDescription ?? "unknown")")
                        return
                    }
                    messages = documents.compactMap { $0.data()["text"] as? String }
                }
        }
    }
}
```

## Common Build Errors

**`No such module 'FirebaseFirestore'`**

The package didn't resolve correctly. Try:
`File â†’ Packages â†’ Reset Package Caches`, then rebuild.

If that doesn't work, delete DerivedData and restart Xcode:

```bash
# DerivedData location
~/Library/Developer/Xcode/DerivedData/
```

**Build takes 10+ minutes after adding Firebase**

Expected on first build â€” Firebase's binary frameworks are large. Subsequent builds use the cache and are much faster. If you don't need FirebaseAnalytics, skip it; it's the heaviest package.

**`Multiple commands produce...` error**

A file is included in multiple targets. Check the target membership in Xcode's file inspector (right panel) and remove duplicates.

## After SPM Integration: Using Rork Companion

After adding packages in Xcode, build and run on simulator or device through Xcode first to confirm everything compiles. Then install the app on device via Rork Companion for ongoing development.

A practical workflow: use Rork to prototype UI and logic, integrate packages in Xcode when the prototype needs external capabilities, then return to Rork for iteration. The two environments complement each other well â€” Rork is fast for UI work, Xcode handles the build toolchain that Rork's previews abstract away.

The SPM setup takes a bit of time once, but after that you have the full Swift ecosystem available to combine with Rork Max's generation capabilities.
