## Why Edge Inference — Breaking Free from Cloud AI Limitations

After building and shipping a Rork app to the App Store or Google Play, many developers hit the same wall: **API costs**. As your user base grows, cloud AI call costs grow with it. When server expenses overtake app revenue — the dreaded "AI cost spiral" — the business model collapses.

Gemma 4's edge models (E2B / E4B) solve this problem at its root. The model runs directly on the user's device, so **API call costs are zero**. But cost savings are just the beginning:

- **Full offline operation**: AI features work without any network connection — in subways, on airplanes, in remote areas
- **Privacy by design**: User data never leaves the device, simplifying GDPR, CCPA, and other privacy regulation compliance
- **Ultra-low latency**: No network round-trip means response times of 10–50ms — effectively zero perceived delay
- **Infinite scalability**: Even with a million users, your server load stays flat

This guide covers the advanced techniques needed to implement Gemma 4 edge inference in Rork apps at production quality. We'll walk through model quantization, MediaPipe integration, offline-first architecture, custom fine-tuning, and battery consumption optimization.

## Understanding Gemma 4's Edge Model Architecture

Correct edge implementation starts with understanding how Gemma 4's edge models are designed.

### E2B and E4B Internals

Gemma 4 E2B (~1.2B parameters) and E4B (~4B parameters) are built using Knowledge Distillation from Gemini 3. By compressing a large teacher model's knowledge into a compact student model, they achieve remarkably high accuracy relative to their parameter count.

- **Input modalities**: Text + Image + Audio (both E2B and E4B support all three)
- **Context length**: Up to 8,192 tokens (edge models; distinct from the cloud models' 256K)
- **Inference engine**: Google AI Edge SDK / MediaPipe LLM Inference API
- **Supported platforms**: iOS (Core ML / Metal), Android (GPU delegate / NNAPI)

### Memory Requirements

```
E2B FP16:  ~2.4 GB RAM (iPhone 15+ recommended)
E2B INT8:  ~1.2 GB RAM (iPhone 13+ recommended)
E2B INT4:  ~0.6 GB RAM (runs on virtually all current smartphones)

E4B FP16:  ~8.0 GB RAM (iPad Pro / high-end Android only)
E4B INT8:  ~4.0 GB RAM (iPhone 15 Pro+)
E4B INT4:  ~2.0 GB RAM (iPhone 14+ recommended)
```

Choosing the right **model size × quantization level** combination for your Rork app's target devices is a critical architectural decision.

## Optimizing Models with INT4 / INT8 Quantization

Quantization converts model weights to lower-precision numerical representations, dramatically reducing file size and improving inference speed. Gemma 4 officially supports INT4 and INT8 quantized variants.

### Downloading and Converting Quantized Models

```python
# quantize_gemma4.py — Prepare quantized Gemma 4 models for edge deployment
# Requirements: Python 3.10+, pip install ai-edge-torch mediapipe

import ai_edge_torch
from ai_edge_torch.generative.quantize import quant_recipes

# Convert Gemma 4 E2B with INT4 quantization
# Downloads pre-trained model from Hugging Face
model_path = "google/gemma-4-e2b"

# Define INT4 quantization recipe
quant_config = quant_recipes.full_int4_weight_only()

# Convert to TFLite format (for mobile deployment)
edge_model = ai_edge_torch.generative.convert(
    model_path=model_path,
    quant_config=quant_config,
    target="tflite",
)

# Save the converted model
edge_model.export("gemma4_e2b_int4.tflite")
print(f"Model size: {edge_model.size_mb:.1f} MB")
# Expected output: Model size: 620.0 MB

# INT8 quantization (when accuracy is the priority)
quant_config_int8 = quant_recipes.full_int8_weight_only()
edge_model_int8 = ai_edge_torch.generative.convert(
    model_path=model_path,
    quant_config=quant_config_int8,
    target="tflite",
)
edge_model_int8.export("gemma4_e2b_int8.tflite")
print(f"INT8 Model size: {edge_model_int8.size_mb:.1f} MB")
# Expected output: INT8 Model size: 1200.0 MB
```

### Accuracy Impact of Quantization

Quantization trades a small amount of accuracy for significant size and speed improvements:

- **INT8**: 1–2% accuracy drop from FP16. Virtually indistinguishable for most use cases
- **INT4**: 3–5% accuracy drop from FP16. Sufficient for classification, summarization, and chat, though complex reasoning tasks may show noticeable degradation

Choose **INT4 for lightweight/speed-first** deployments or **INT8 for accuracy-first** scenarios based on your Rork app's requirements.

## Integrating with MediaPipe LLM Inference API

Google's MediaPipe is the official inference engine for running Gemma 4 on mobile devices. In a Rork app (React Native / Expo), you bridge to it through native modules.

### Native Module Implementation (iOS)

```swift
// ios/GemmaModule.swift — iOS native module
import MediaPipeTasksGenAI
import React

@objc(GemmaModule)
class GemmaModule: NSObject {
  private var llmInference: LlmInference?
  
  @objc
  func initialize(_ modelPath: String,
                  resolver resolve: @escaping RCTPromiseResolveBlock,
                  rejecter reject: @escaping RCTPromiseRejectBlock) {
    do {
      // Configure MediaPipe LLM Inference
      let options = LlmInference.Options()
      options.modelPath = modelPath
      options.maxTokens = 2048
      options.temperature = 0.7
      
      // Enable GPU acceleration (Metal)
      options.preferredBackend = .gpu
      
      self.llmInference = try LlmInference(options: options)
      resolve(["status": "initialized", "backend": "gpu"])
    } catch {
      reject("INIT_ERROR", "Failed to initialize Gemma 4: \(error)", error)
    }
  }
  
  @objc
  func generateResponse(_ prompt: String,
                        resolver resolve: @escaping RCTPromiseResolveBlock,
                        rejecter reject: @escaping RCTPromiseRejectBlock) {
    guard let inference = self.llmInference else {
      reject("NOT_INIT", "Model not initialized", nil)
      return
    }
    
    DispatchQueue.global(qos: .userInitiated).async {
      do {
        let startTime = CFAbsoluteTimeGetCurrent()
        let response = try inference.generateResponse(inputText: prompt)
        let elapsed = CFAbsoluteTimeGetCurrent() - startTime
        
        resolve([
          "text": response,
          "latencyMs": Int(elapsed * 1000),
        ])
      } catch {
        reject("GEN_ERROR", "Generation failed: \(error)", error)
      }
    }
  }
}
```

### Native Module Implementation (Android)

```kotlin
// android/GemmaModule.kt — Android native module
package com.rorkapp.gemma

import com.google.mediapipe.tasks.genai.llminference.LlmInference
import com.facebook.react.bridge.*

class GemmaModule(reactContext: ReactApplicationContext) :
    ReactContextBaseJavaModule(reactContext) {

    private var llmInference: LlmInference? = null

    override fun getName() = "GemmaModule"

    fun initialize(modelPath: String, promise: Promise) {
        try {
            val options = LlmInference.LlmInferenceOptions.builder()
                .setModelPath(modelPath)
                .setMaxTokens(2048)
                .setTemperature(0.7f)
                .setPreferredBackend(LlmInference.Backend.GPU) // GPU delegate
                .build()

            llmInference = LlmInference.createFromOptions(
                reactApplicationContext, options
            )
            
            val result = Arguments.createMap().apply {
                putString("status", "initialized")
                putString("backend", "gpu")
            }
            promise.resolve(result)
        } catch (e: Exception) {
            promise.reject("INIT_ERROR", "Failed to initialize Gemma 4", e)
        }
    }

    @ReactMethod
    fun generateResponse(prompt: String, promise: Promise) {
        val inference = llmInference ?: run {
            promise.reject("NOT_INIT", "Model not initialized")
            return
        }

        Thread {
            try {
                val startTime = System.currentTimeMillis()
                val response = inference.generateResponse(prompt)
                val elapsed = System.currentTimeMillis() - startTime

                val result = Arguments.createMap().apply {
                    putString("text", response)
                    putInt("latencyMs", elapsed.toInt())
                }
                promise.resolve(result)
            } catch (e: Exception) {
                promise.reject("GEN_ERROR", "Generation failed", e)
            }
        }.start()
    }
}
```

### React Native Bridge (TypeScript)

```typescript
// services/gemmaEdge.ts — Gemma 4 edge inference bridge
import { NativeModules, Platform } from 'react-native';
import * as FileSystem from 'expo-file-system';

const { GemmaModule } = NativeModules;

// Model file storage location
const MODEL_DIR = `${FileSystem.documentDirectory}models/`;
const MODEL_FILENAME = 'gemma4_e2b_int4.tflite';

export interface EdgeResponse {
  text: string;
  latencyMs: number;
}

/**
 * Initialize Gemma 4 edge model
 * - Downloads model on first launch (~600MB)
 * - Uses local cache on subsequent launches
 */
export async function initializeGemmaEdge(
  onProgress?: (progress: number) => void
): Promise<void> {
  const modelPath = `${MODEL_DIR}${MODEL_FILENAME}`;
  
  // Check if model file exists locally
  const fileInfo = await FileSystem.getInfoAsync(modelPath);
  
  if (!fileInfo.exists) {
    // First run: download the model
    await FileSystem.makeDirectoryAsync(MODEL_DIR, { intermediates: true });
    
    const downloadUrl = 'https://your-cdn.example.com/models/gemma4_e2b_int4.tflite';
    const downloadResumable = FileSystem.createDownloadResumable(
      downloadUrl,
      modelPath,
      {},
      (downloadProgress) => {
        const progress = downloadProgress.totalBytesWritten /
                         downloadProgress.totalBytesExpectedToWrite;
        onProgress?.(progress);
      }
    );
    
    await downloadResumable.downloadAsync();
  }

  // Initialize native module with model
  await GemmaModule.initialize(modelPath);
}

/**
 * Generate text (edge inference)
 */
export async function generateEdgeResponse(prompt: string): Promise<EdgeResponse> {
  return await GemmaModule.generateResponse(prompt);
}
```

## Offline-First Design Patterns

The biggest advantage of edge inference is **complete offline operation**. But simply running a model locally isn't enough — you need to design your entire app with an offline-first mindset.

### Hybrid AI Architecture

The most practical approach dynamically switches between edge inference and cloud API based on network state.

```typescript
// services/hybridAI.ts — Hybrid AI manager
import NetInfo from '@react-native-community/netinfo';
import { generateEdgeResponse, EdgeResponse } from './gemmaEdge';
import { callGemma4 } from './gemma4';

type AIMode = 'edge' | 'cloud' | 'auto';

interface HybridConfig {
  mode: AIMode;
  edgeMaxTokens: number;    // Edge model output limit
  cloudFallback: boolean;   // Fallback to cloud on edge failure
  qualityThreshold: number; // Quality score threshold (0-1)
}

const DEFAULT_CONFIG: HybridConfig = {
  mode: 'auto',
  edgeMaxTokens: 512,
  cloudFallback: true,
  qualityThreshold: 0.7,
};

export async function hybridGenerate(
  prompt: string,
  config: Partial<HybridConfig> = {}
): Promise<{ text: string; source: 'edge' | 'cloud'; latencyMs: number }> {
  const cfg = { ...DEFAULT_CONFIG, ...config };
  
  // Check network state
  const netState = await NetInfo.fetch();
  const isOnline = netState.isConnected && netState.isInternetReachable;
  
  // Mode selection logic
  const useEdge = cfg.mode === 'edge' ||
                  cfg.mode === 'auto' && !isOnline ||
                  cfg.mode === 'auto' && prompt.length < 500;
  
  if (useEdge) {
    try {
      const startTime = Date.now();
      const edgeResult = await generateEdgeResponse(prompt);
      return {
        text: edgeResult.text,
        source: 'edge',
        latencyMs: Date.now() - startTime,
      };
    } catch (error) {
      // Fallback to cloud on edge failure
      if (cfg.cloudFallback && isOnline) {
        const cloudResult = await callGemma4({ text: prompt });
        return { text: cloudResult.text, source: 'cloud', latencyMs: 0 };
      }
      throw error;
    }
  }
  
  // Cloud inference
  const startTime = Date.now();
  const cloudResult = await callGemma4({ text: prompt });
  return {
    text: cloudResult.text,
    source: 'cloud',
    latencyMs: Date.now() - startTime,
  };
}
```

### Offline Request Queuing

A queuing system that automatically processes cloud requests when the network comes back is also essential.

```typescript
// services/offlineQueue.ts — Offline request queue
import AsyncStorage from '@react-native-async-storage/async-storage';
import NetInfo from '@react-native-community/netinfo';

interface QueuedRequest {
  id: string;
  prompt: string;
  timestamp: number;
  priority: 'high' | 'normal' | 'low';
}

const QUEUE_KEY = 'gemma4_offline_queue';

export async function enqueueRequest(
  prompt: string,
  priority: 'high' | 'normal' | 'low' = 'normal'
): Promise<string> {
  const id = `req_${Date.now()}_${Math.random().toString(36).slice(2)}`;
  const queue = await getQueue();
  queue.push({ id, prompt, timestamp: Date.now(), priority });
  
  // Sort by priority (high → normal → low)
  queue.sort((a, b) => {
    const order = { high: 0, normal: 1, low: 2 };
    return order[a.priority] - order[b.priority];
  });
  
  await AsyncStorage.setItem(QUEUE_KEY, JSON.stringify(queue));
  return id;
}

// Auto-process when network returns
NetInfo.addEventListener(async (state) => {
  if (state.isConnected && state.isInternetReachable) {
    await processQueue();
  }
});

async function processQueue(): Promise<void> {
  const queue = await getQueue();
  if (queue.length === 0) return;
  
  for (const request of queue) {
    try {
      const result = await callGemma4({ text: request.prompt });
      // Save result to local storage
      await AsyncStorage.setItem(`result_${request.id}`, JSON.stringify(result));
    } catch {
      break; // Stop on network error
    }
  }
  
  // Remove processed requests from queue
  const remaining = await getQueue();
  const processed = remaining.filter(r =>
    !queue.some(q => q.id === r.id)
  );
  await AsyncStorage.setItem(QUEUE_KEY, JSON.stringify(processed));
}

async function getQueue(): Promise<QueuedRequest[]> {
  const data = await AsyncStorage.getItem(QUEUE_KEY);
  return data ? JSON.parse(data) : [];
}
```

## Building Custom Models with Fine-Tuning

Gemma 4's Apache 2.0 license means you can freely fine-tune it for commercial purposes. By embedding domain-specific knowledge, you achieve accuracy levels that generic models simply cannot match.

### Parameter-Efficient Fine-Tuning with LoRA

```python
# fine_tune_gemma4.py — LoRA fine-tuning script
# Requirements: Python 3.10+, GPU recommended (A100 / L4)
# pip install transformers peft datasets torch accelerate

from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments
from peft import LoraConfig, get_peft_model, TaskType
from datasets import load_dataset
import torch

# Load Gemma 4 E4B as the base model
model_name = "google/gemma-4-e4b"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

# LoRA configuration (memory-efficient fine-tuning)
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=16,                  # LoRA rank (8-64; higher = more expressive but more memory)
    lora_alpha=32,         # Scaling factor
    lora_dropout=0.05,     # Prevent overfitting
    target_modules=[       # Only update attention layers
        "q_proj", "k_proj", "v_proj", "o_proj",
    ],
)

# Apply LoRA adapter
model = get_peft_model(model, lora_config)
print(f"Trainable parameters: {model.print_trainable_parameters()}")
# Expected output: trainable params: 6,553,600 || all params: 4,000,000,000 || trainable%: 0.1638

# Prepare custom dataset (example: recipe app)
dataset = load_dataset("json", data_files="recipe_training_data.jsonl")

# Training configuration
training_args = TrainingArguments(
    output_dir="./gemma4-recipe-lora",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    bf16=True,
    logging_steps=10,
    save_strategy="epoch",
)

# Run fine-tuning
from transformers import Trainer
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    tokenizer=tokenizer,
)
trainer.train()

# Save LoRA adapter (stored separately from base model — ~25MB)
model.save_pretrained("./gemma4-recipe-lora-adapter")
```

### Converting Fine-Tuned Models for Edge Deployment

## Battery Consumption Optimization

Edge inference heavily uses the device's GPU/NPU, so battery impact demands careful attention.

### Inference Scheduling

```typescript
// services/batteryAware.ts — Battery-aware inference manager
import * as Battery from 'expo-battery';
import { AppState, AppStateStatus } from 'react-native';

interface InferencePolicy {
  maxConcurrent: number;
  maxTokensPerRequest: number;
  cooldownMs: number;       // Minimum interval between consecutive inferences
  backgroundAllowed: boolean;
}

// Policy based on battery level
function getPolicy(batteryLevel: number, isCharging: boolean): InferencePolicy {
  if (isCharging) {
    return { maxConcurrent: 3, maxTokensPerRequest: 2048, cooldownMs: 0, backgroundAllowed: true };
  }
  if (batteryLevel > 0.5) {
    return { maxConcurrent: 2, maxTokensPerRequest: 1024, cooldownMs: 500, backgroundAllowed: true };
  }
  if (batteryLevel > 0.2) {
    return { maxConcurrent: 1, maxTokensPerRequest: 512, cooldownMs: 2000, backgroundAllowed: false };
  }
  // Below 20%: minimize inference
  return { maxConcurrent: 1, maxTokensPerRequest: 256, cooldownMs: 5000, backgroundAllowed: false };
}

export class BatteryAwareInference {
  private lastInferenceTime = 0;
  private activeCount = 0;

  async canInfer(): Promise<{ allowed: boolean; reason?: string }> {
    const batteryLevel = await Battery.getBatteryLevelAsync();
    const batteryState = await Battery.getBatteryStateAsync();
    const isCharging = batteryState === Battery.BatteryState.CHARGING;
    const policy = getPolicy(batteryLevel, isCharging);

    // Background state check
    if (AppState.currentState !== 'active' && !policy.backgroundAllowed) {
      return { allowed: false, reason: 'Background inference paused to save battery' };
    }

    // Concurrency check
    if (this.activeCount >= policy.maxConcurrent) {
      return { allowed: false, reason: 'A request is already being processed' };
    }

    // Cooldown check
    const elapsed = Date.now() - this.lastInferenceTime;
    if (elapsed < policy.cooldownMs) {
      return { allowed: false, reason: `Please wait ${Math.ceil((policy.cooldownMs - elapsed) / 1000)} seconds` };
    }

    return { allowed: true };
  }

  async inferWithPolicy(prompt: string): Promise<EdgeResponse> {
    const check = await this.canInfer();
    if (!check.allowed) {
      throw new Error(check.reason);
    }

    this.activeCount++;
    this.lastInferenceTime = Date.now();

    try {
      return await generateEdgeResponse(prompt);
    } finally {
      this.activeCount--;
    }
  }
}
```

### Thermal Throttling Mitigation

Extended inference sessions can cause device overheating, triggering OS-level performance throttling. Detecting thermal state proactively lets you adjust inference frequency before performance degrades.

```typescript
// utils/thermalMonitor.ts — Thermal state monitoring
// iOS: Monitors ProcessInfo.thermalState
// Android: Monitors BatteryManager.EXTRA_TEMPERATURE

export type ThermalState = 'nominal' | 'fair' | 'serious' | 'critical';

export function adjustInferenceForThermal(state: ThermalState): {
  shouldDefer: boolean;
  reducedTokens: number;
} {
  switch (state) {
    case 'nominal':
      return { shouldDefer: false, reducedTokens: 2048 };
    case 'fair':
      return { shouldDefer: false, reducedTokens: 1024 };
    case 'serious':
      return { shouldDefer: true, reducedTokens: 512 };
    case 'critical':
      return { shouldDefer: true, reducedTokens: 0 }; // Halt inference entirely
  }
}
```

## FAQ

**Q: What are the minimum device specs for edge inference?**

A: For Gemma 4 E2B INT4, which uses about 600MB of RAM, devices with 4GB+ RAM are recommended. Specifically: iPhone 12 or later on iOS, and mid-range or better Android phones from 2022 onward. E4B requires iPhone 14+ or Android devices with 8GB+ RAM.

**Q: Won't the model download make my app bundle too large?**

A: Don't include the model in your app bundle. Instead, download it from a CDN on first launch. Show a progress bar during download to maintain good UX. The `initializeGemmaEdge` function in this guide implements exactly this pattern.

**Q: How much training data do I need for fine-tuning?**

A: LoRA fine-tuning is effective with relatively small datasets. Noticeable improvements often appear with 500–1,000 domain-specific samples. However, data quality matters far more than quantity. Focus on curating high-quality prompt-response pairs rather than amassing large volumes of mediocre data.

**Q: What should I do if the app crashes during edge inference?**

A: The most common cause is out-of-memory (OOM). Three solutions: (1) Use a smaller model (E2B) or more aggressive quantization (INT4), (2) Handle `didReceiveMemoryWarning` (iOS) / low memory callbacks (Android) to release non-critical resources, (3) Check available device memory before inference and fall back to cloud if insufficient.

**Q: Are there App Store / Google Play restrictions on edge AI models?**

A: Currently, there are no review guidelines prohibiting on-device AI models. However, for large model downloads, Apple restricts cellular downloads over 200MB, so ensure you require Wi-Fi for the initial model download. Also consider including AI feature descriptions in your privacy policy.
