## Understanding Dynamic Island & Live Activities

### What Is Dynamic Island?

Dynamic Island is a UI system introduced in iOS 16.1 that lets apps display dynamic content in the area surrounding the front camera cutout. When tapped and held, it expands to show richer information.

| Display Mode | Description |
|--------------|-------------|
| Compact Leading | Small icon on the left (e.g., music playback icon) |
| Compact Trailing | Short text or number on the right (e.g., timer countdown) |
| Minimal | Smallest size when multiple apps compete |
| Expanded | Detailed view shown when user long-presses |

### What Are Live Activities?

Live Activities use Apple's ActivityKit framework to display real-time app data on the Lock Screen and in Dynamic Island. They can receive real-time updates via push notifications (APNs) and stay active for up to 8 hours (extendable to 12).

```swift
// Basic data structure for a Live Activity
import ActivityKit

// ActivityAttributes defines your activity's static and dynamic data
struct DeliveryAttributes: ActivityAttributes {
    // Dynamic data — can be updated in real time
    public struct ContentState: Codable, Hashable {
        var status: String        // "Preparing" | "On the Way" | "Delivered"
        var eta: Date             // Estimated arrival time
        var driverName: String    // Delivery driver's name
        var progress: Double      // 0.0 to 1.0
    }

    // Static data — fixed when Activity starts
    var orderId: String
    var restaurantName: String
}

// Expected output: ActivityAttributes type definition compiles without errors
```

### Step 4: Start, Update, and End a Live Activity

```swift
import ActivityKit

class DeliveryActivityManager {
    var currentActivity: Activity<DeliveryAttributes>?

    // Start a Live Activity
    func startActivity(orderId: String, restaurantName: String, total: Double) {
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            print("⚠️ Live Activities are disabled on this device")
            return
        }

        let attributes = DeliveryAttributes(
            orderId: orderId,
            restaurantName: restaurantName,
            totalAmount: total
        )

        let initialState = DeliveryAttributes.ContentState(
            status: .preparing,
            eta: Date().addingTimeInterval(30 * 60), // 30 minutes from now
            driverName: "Alex",
            progress: 0.1,
            currentLocation: "Restaurant"
        )

        do {
            currentActivity = try Activity.request(
                attributes: attributes,
                content: .init(state: initialState, staleDate: nil),
                pushType: .token  // Enable real-time updates via APNs
            )
            print("✅ Live Activity started: \(currentActivity?.id ?? "")")
        } catch {
            print("❌ Failed to start: \(error.localizedDescription)")
        }
    }

    // Update the Live Activity
    func updateActivity(status: DeliveryStatus, progress: Double, eta: Date) async {
        guard let activity = currentActivity else { return }

        let updatedState = DeliveryAttributes.ContentState(
            status: status,
            eta: eta,
            driverName: "Alex",
            progress: progress,
            currentLocation: status == .onTheWay ? "En route" : "Near you"
        )

        await activity.update(
            ActivityContent(state: updatedState, staleDate: Date().addingTimeInterval(60))
        )
        print("🔄 Updated: \(status.rawValue)")
    }

    // End the Live Activity
    func endActivity() async {
        guard let activity = currentActivity else { return }

        let finalState = DeliveryAttributes.ContentState(
            status: .delivered,
            eta: Date(),
            driverName: "Alex",
            progress: 1.0,
            currentLocation: "Delivered"
        )

        await activity.end(
            ActivityContent(state: finalState, staleDate: Date()),
            dismissalPolicy: .after(Date().addingTimeInterval(5 * 60)) // Auto-dismiss after 5 min
        )
        currentActivity = nil
        print("🏁 Live Activity ended")
    }
}

// Expected output:
// ✅ Live Activity started: [activity-id]
// 🔄 Updated: On the Way
// 🏁 Live Activity ended
```

## Real-Time Updates via APNs

In production apps, your server sends push notifications to update Live Activities in real time.

```swift
// Capture the push token from the device
func startActivityWithPushToken() async {
    guard let activity = currentActivity else { return }

    for await pushToken in activity.pushTokenUpdates {
        let tokenString = pushToken.map { String(format: "%02x", $0) }.joined()
        print("📱 Push Token: \(tokenString)")

        // Send this token to your backend
        await sendTokenToServer(token: tokenString, activityId: activity.id)
    }
}
```

## Common Errors and Fixes

### Error 1: `areActivitiesEnabled` returns `false`

**Cause**: The user has disabled Live Activities in device settings.

**Fix**: Guide users to **Settings → Face ID & Passcode → Live Activities** and toggle it on. In your app, gracefully fall back to push notifications:

```swift
if !ActivityAuthorizationInfo().areActivitiesEnabled {
    // Fallback: use a regular push notification instead
    scheduleNotification(eta: eta)
}
```

### Error 2: `ActivityKit` module not found

**Cause**: The deployment target is below iOS 16.1.

**Rork Max prompt to fix**:
```
Set the iOS Deployment Target to 16.1 or later.
Live Activities require iOS 16.1+.
```

### Error 3: Dynamic Island doesn't appear in the simulator

**Cause**: Dynamic Island only works on iPhone 14 Pro and later physical hardware, and select simulator models.

**Fix**: In Xcode Simulator, select **iPhone 15 Pro** or **iPhone 16 Pro** as the device.

### Error 4: Live Activity stops after 8 hours

**Cause**: ActivityKit's built-in time limit (default 8 hours, extendable to 12 hours).

**Fix**: Set `staleDate` appropriately and implement logic to restart the Activity if the event continues beyond the limit.

## FAQ

**Q. Can I implement Dynamic Island in regular Rork (React Native)?**

A. It's technically possible but very complex. React Native requires custom native modules to bridge into ActivityKit and WidgetKit. **Rork Max** generates native Swift, so Dynamic Island works out of the box with a simple AI prompt — no bridging required.

**Q. Which devices support Dynamic Island vs. Live Activities?**

A. Live Activities (Lock Screen cards) work on **all iPhones running iOS 16.1+**. The pill-shaped Dynamic Island UI only appears on **iPhone 14 Pro, 14 Pro Max, and all iPhone 15 / 16 models** — devices with the punch-hole camera cutout rather than a notch.

**Q. Does Rork Max cost extra for Live Activities support?**

A. No — Live Activities implementation is included in the Rork Max plan ($200/month). However, if you want server-side APNs updates (real-time pushes), you'll need a backend (e.g., Cloudflare Workers), which has its own cost. For backend setup guidance, see the [Rork Supabase Chat App guide](/en/articles/rork-dev/rork-supabase-chat-app).

**Q. How often can I update a Live Activity?**

A. Apple doesn't publish a hard rate limit, but excessive updates are throttled by the system. In practice, **once every 1–2 minutes** is safe for most use cases. High-urgency events (score changes, gate updates) can be pushed immediately without issue.
