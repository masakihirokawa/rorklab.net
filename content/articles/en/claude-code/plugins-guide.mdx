## Plugin System Overview

Claude Code plugins are packages that extend Claude Code's capabilities. A plugin can include up to five component types:

- **Skills** — Prompt templates and workflow definitions
- **Agents** — Autonomous agents specialized for specific tasks
- **MCP servers** — External service integrations (Model Context Protocol)
- **Hooks** — Event-driven automation (e.g., auto-processing on file changes)
- **LSP servers** — Language Server Protocol for editor integration

## Plugin Structure

Plugins follow this directory layout:

```
my-plugin/
├── plugin.json        # Manifest file (required)
├── skills/
│   └── my-skill/
│       └── SKILL.md   # Skill definition
├── agents/
│   └── my-agent.md    # Agent definition
├── mcp/
│   └── server.js      # MCP server implementation
└── hooks/
    └── on-save.sh     # Hook handler
```

## The plugin.json Manifest

The core of every plugin is `plugin.json`:

```json
{
  "name": "my-awesome-plugin",
  "version": "1.0.0",
  "description": "Automatically checks code quality for your project",
  "author": "your-name",
  "components": {
    "skills": ["skills/code-review"],
    "mcpServers": {
      "quality-check": {
        "command": "node",
        "args": ["mcp/server.js"]
      }
    }
  }
}
```

<Callout type="info">
For the complete manifest field specification, see the official Plugins Reference documentation.
</Callout>

## Installation Scopes

Plugins can be installed at four scopes:

| Scope | Description | Use Case |
|-------|-------------|----------|
| `user` | User-global | Personal shared tools |
| `project` | Project-specific | Team-shared config |
| `local` | Local only | Experimental plugins |
| `managed` | Organization-managed | Enterprise deployments |

## CLI Plugin Management

```bash
# Install
claude plugin install @example/my-plugin

# Uninstall
claude plugin uninstall @example/my-plugin

# Enable / Disable
claude plugin enable my-plugin
claude plugin disable my-plugin

# Update
claude plugin update my-plugin

# List all
claude plugin list
```

## Tutorial: Build a Simple Skill Plugin

Let's create a code review skill plugin as the simplest example.

### 1. Create the Directory

```bash
mkdir -p my-review-plugin/skills/review
```

### 2. Create plugin.json

```json
{
  "name": "code-review-en",
  "version": "0.1.0",
  "description": "A skill for structured code reviews",
  "components": {
    "skills": ["skills/review"]
  }
}
```

### 3. Create the Skill File

`skills/review/SKILL.md`:

```markdown
---
description: "Reviews code quality and provides structured improvement suggestions"
---

# Code Review Skill

Review the code for the following aspects:

1. **Potential bugs** — Edge cases and hidden issues
2. **Performance** — Inefficient operations, N+1 queries
3. **Security** — Vulnerabilities, injection risks
4. **Readability** — Naming conventions, comments, clarity
5. **Testing** — Missing test coverage

For each finding, include a severity level (High/Medium/Low) and a fix example.
```

### 4. Install Locally

```bash
claude plugin install ./my-review-plugin --scope local
```

<Callout type="tip">
Use `--scope local` during development, then publish to npm to share with other users.
</Callout>

## Integrating MCP Servers

Including an MCP server in your plugin lets Claude Code access external APIs and databases.

```json
{
  "components": {
    "mcpServers": {
      "my-db": {
        "command": "npx",
        "args": ["-y", "@example/db-mcp-server"],
        "env": {
          "DATABASE_URL": "${DATABASE_URL}"
        }
      }
    }
  }
}
```

<Callout type="warning">
MCP servers run in your local environment. Only install from trusted sources and manage secrets via environment variables.
</Callout>
