## What Is CLAUDE.md?

**CLAUDE.md** is a markdown file that Claude Code automatically reads at the start of every session. Place it in your project root to communicate coding standards, architecture decisions, preferred libraries, and review checklists to Claude.

## Why You Need It

Claude Code starts fresh each session without context from previous interactions. With CLAUDE.md, you never need to repeat the same explanations, and project-specific rules are applied consistently.

## Writing CLAUDE.md

Create a `CLAUDE.md` file in your project root.

```markdown
# Project Overview
E-commerce backend API (Node.js + Express + TypeScript)

# Coding Standards
- Always add TypeScript type annotations to functions
- Use Result type pattern for error handling
- Use Jest for tests; write at least one test per function

# Architecture
- src/routes/ — API route definitions
- src/services/ — Business logic
- src/models/ — Data models (Prisma)
- src/middleware/ — Auth and validation middleware

# Build & Test
- Build: npm run build
- Test: npm test
- Dev server: npm run dev

# Notes
- Database: PostgreSQL
- Env vars defined in .env.local (not committed)
- API responses must use { success, data, error } format
```

## File Locations and Priority

CLAUDE.md files can be placed in multiple locations with different scopes:

1. **Project root** (`./CLAUDE.md`): Applies to the entire project
2. **Subdirectories** (`./src/CLAUDE.md`): Applies to specific directories
3. **Home directory** (`~/.claude/CLAUDE.md`): Applies to all projects

When multiple files exist, all are loaded, with more specific scopes taking priority.

## Auto Memory

Claude Code includes an **Auto Memory** feature that automatically saves learnings during work.

### What Gets Remembered

- Project build commands and their results
- Problems discovered during debugging and their solutions
- Frequently used file paths and patterns
- User preferences and workflows

### Benefits

With Auto Memory, Claude Code deepens its project understanding the more you use it—without manually updating CLAUDE.md. Learnings from previous sessions carry forward.

## Team Usage

Commit CLAUDE.md to your repository so all team members share the same instructions with Claude Code.

```
# Review Checklist
Before creating a PR, verify:
- [ ] No TypeScript type errors (tsc --noEmit)
- [ ] All tests pass (npm test)
- [ ] No ESLint warnings (npm run lint)
- [ ] OpenAPI spec updated if API changes were made
```

## Tips for Writing Effective CLAUDE.md

1. **Keep it concise**: Claude reads it every session, so remove unnecessary info
2. **Use structure**: Organize with headers and lists
3. **Keep it updated**: Evolve CLAUDE.md as your project changes
4. **Be specific**: Instead of "write good code," say "keep functions under 30 lines"

## Summary

CLAUDE.md significantly impacts Claude Code's performance. Spending five minutes writing one improves every interaction and boosts team productivity. Combined with Auto Memory, Claude Code gets smarter the more you use it.
