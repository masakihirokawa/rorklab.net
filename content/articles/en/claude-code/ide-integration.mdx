# Claude Code IDE Integration Guide — VS Code & JetBrains

Claude Code transforms your development workflow by integrating AI-powered assistance directly into your favorite IDEs. This comprehensive guide walks you through setting up and mastering Claude Code integration in both VS Code and JetBrains IDEs.

## Overview

Claude Code provides seamless integration with modern development environments, allowing you to leverage AI capabilities without leaving your IDE. Whether you're using VS Code or a JetBrains IDE, you can access Claude's advanced reasoning, code analysis, and generation features directly within your editor.

### Key Benefits

- **Contextual Understanding**: Claude understands your code, project structure, and files in context
- **Multi-File References**: Reference multiple files, folders, and specific line ranges in your conversations
- **Permission Control**: Fine-grained control over what Claude can access and modify
- **Keyboard-Driven Workflow**: Optimized shortcuts for efficient development
- **MCP Integration**: Extend Claude's capabilities with Model Context Protocol servers
- **Multi-Session Support**: Manage multiple independent conversations simultaneously

## VS Code Extension Setup

### Installation and Requirements

The Claude Code extension for VS Code is available on the VS Code Marketplace and officially maintained by Anthropic.

**Requirements:**
- VS Code 1.98.0 or later
- Active Claude account with API access
- Internet connection for API communication

**Installation Steps:**

1. Open VS Code
2. Navigate to the Extensions marketplace (Cmd+Shift+X on macOS, Ctrl+Shift+X on Windows/Linux)
3. Search for "Claude Code" by Anthropic
4. Click "Install" on the official extension (ID: `vscode:extension/anthropic.claude-code`)
5. Follow the authentication prompts to connect your Claude account
6. The extension will initialize and display the Claude panel

### Accessing Claude Code

Once installed, you can access Claude Code through multiple entry points:

**Spark Icon in Activity Bar**
The Claude Code icon appears in the left-hand Activity Bar. Click it to open the Claude panel with your current conversations and conversation history.

**Command Palette**
- Press `Cmd+Shift+P` (macOS) or `Ctrl+Shift+P` (Windows/Linux)
- Search for "Claude Code" to see available commands
- Common commands include "New Conversation", "Focus Claude", and "Reference File"

**Status Bar**
The status bar at the bottom of your window displays Claude Code status. Click the Claude indicator to quickly access the extension features.

**Context Menu**
Right-click on files, folders, or selected code to quickly reference them in Claude Code with contextual menu options.

### Configuration Settings

Claude Code extension provides several configurable settings to customize your experience. Access settings via VS Code's Settings interface (Cmd+, on macOS).

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `claude-code.selectedModel` | string | `claude-opus-4-6` | The Claude model to use for conversations (e.g., claude-opus-4-6, claude-sonnet-4-20250514) |
| `claude-code.useTerminal` | boolean | `true` | Enable integrated terminal access for Claude to execute commands |
| `claude-code.initialPermissionMode` | enum | `Normal` | Default permission mode: Normal, Plan, or Auto-Accept |
| `claude-code.preferredLocation` | enum | `Side Panel` | Where Claude panel appears: Side Panel, Bottom Panel, or Fullscreen |
| `claude-code.autosave` | boolean | `true` | Automatically save changes made by Claude |
| `claude-code.respectGitIgnore` | boolean | `true` | Respect .gitignore rules when referencing files |

**Example Configuration (settings.json):**

```json
{
  "claude-code.selectedModel": "claude-opus-4-6",
  "claude-code.useTerminal": true,
  "claude-code.initialPermissionMode": "Plan",
  "claude-code.preferredLocation": "Side Panel",
  "claude-code.autosave": true,
  "claude-code.respectGitIgnore": true
}
```

> **Pro Tip**: Set `initialPermissionMode` to "Plan" if you want Claude to preview changes before applying them. This is useful when learning or working on critical code.

## File Referencing in Claude Code

One of Claude Code's most powerful features is the ability to reference files, folders, and specific sections of your codebase directly in conversations.

### File Reference Syntax

Claude Code uses the `@` symbol to reference files and folders. This works similar to mentions in collaborative tools.

**Single File Reference**
```
@app.ts
@src/utils/helper.js
@package.json
```

Claude will include the entire file contents in the conversation context.

**Folder References**
```
@src/components/
@utils/
@tests/
```

When referencing a folder, Claude includes all files in that directory, respecting your `.gitignore` rules if enabled.

**Specific Line Ranges**
```
@app.ts#5-10
@src/api/client.ts#1-25
@middleware.js#45-60
```

Reference specific lines within a file to focus Claude's attention on particular code sections.

**PDF Document Pages**
```
@document.pdf#1-10
@api-spec.pdf#15-20
@architecture.pdf
```

Claude can analyze PDF documents and specific page ranges, useful for referencing API specifications or architectural diagrams.

**Fuzzy File Search**
```
@auth
@component
@database
```

Use fuzzy matching to quickly find files without typing the full path. Claude will suggest matching files as you type.

### File Reference Workflow

1. Start typing `@` in the Claude Code input field
2. Claude displays matching files and folders
3. Select the desired file or folder
4. Claude inserts the reference and highlights it
5. Send your message—Claude includes the referenced files in the context

This workflow ensures Claude has the necessary context to provide accurate, code-aware assistance.

## Permission Modes

Claude Code implements a permission system to give you control over what Claude can do with your files and environment.

### Permission Mode Overview

| Mode | Description | Use Case |
|------|-------------|----------|
| **Normal** | Claude shows proposed changes and waits for your approval before applying them | Default mode; recommended for critical changes |
| **Plan** | Claude presents a detailed plan and asks for confirmation before executing actions | Learning mode; understand what Claude will do |
| **Auto-Accept** | Claude automatically applies changes without asking for approval | Trusted scenarios; productivity-focused workflow |

### Setting Permission Modes

**Global Setting:**
Configure the default permission mode in VS Code settings:
```
claude-code.initialPermissionMode: "Normal"
```

**Per-Conversation Override:**
You can change the permission mode for individual conversations within the Claude panel by selecting the desired mode from the dropdown menu.

> **Security Note**: Use "Auto-Accept" cautiously. This mode allows Claude to modify files without approval. Only enable it when working in isolated projects or after verifying Claude's recent changes.

## Keyboard Shortcuts

Efficient keyboard navigation is essential for maintaining flow during development. Claude Code provides several optimized shortcuts.

### Essential Shortcuts

| Shortcut | macOS | Windows/Linux | Action |
|----------|-------|---------------|--------|
| Focus Claude | Cmd+Esc | Ctrl+Esc | Move focus to Claude panel from editor |
| New Conversation | Cmd+N | Ctrl+N | Start a new Claude conversation |
| @ Mention | Option+K | Alt+K | Open file/folder selector for quick references |
| Search Files | Cmd+P | Ctrl+P | Quick file search (then @ to reference) |
| Send Message | Enter | Enter | Send message in Claude input field |
| Clear Context | Cmd+Shift+L | Ctrl+Shift+L | Clear conversation history and context |

### Navigation Tips

- Press `Cmd+Esc` (or `Ctrl+Esc`) from any editor position to immediately focus the Claude panel
- Use `Option+K` (or `Alt+K`) to bring up the file selector without typing `@`
- Combine `Cmd+P` with `@` to search and reference files in a single operation

## Multi-Session Support

Claude Code supports multiple independent conversations simultaneously, each in a separate tab within the Claude panel.

### Creating New Sessions

1. Click the "+" icon at the top of the Claude panel
2. Or press `Cmd+N` (or `Ctrl+N`)
3. A new conversation tab opens with fresh context
4. Each session maintains its own conversation history and file references

### Session Management

- **Switch Between Sessions**: Click conversation tabs to switch between open sessions
- **Close Sessions**: Click the X on a tab to close that conversation
- **Session Persistence**: Conversation history is automatically saved and persists across VS Code restarts
- **Independent Context**: Each session maintains separate file references and conversation state

This multi-session capability allows you to work on different tasks simultaneously without losing context.

## MCP Server Integration

The `/mcp` command enables integration with Model Context Protocol (MCP) servers, extending Claude's capabilities beyond file analysis.

### Using MCP Servers

```
/mcp add <server-name> <server-config>
/mcp list
/mcp remove <server-name>
```

**Example MCP Integration:**
```
/mcp add local-database "postgresql://localhost:5432/mydb"
/mcp list
```

Once configured, Claude can use MCP server tools directly in conversations, providing access to external systems, databases, and specialized APIs.

> **Note**: MCP server setup requires configuration outside of VS Code. Consult the specific MCP server documentation for initialization steps.

## Plugin Management

The `/plugins` command provides access to Claude's plugin ecosystem, enabling enhanced functionality for specific workflows.

### Plugin Commands

```
/plugins list
/plugins enable <plugin-name>
/plugins disable <plugin-name>
/plugins search <query>
```

**Available Plugins:**
- Code analysis and optimization
- Documentation generation
- Test generation and validation
- Security analysis
- Performance profiling

Plugins extend Claude Code's capabilities for specialized development tasks. Browse available plugins using the search command.

## Advanced Features

### Chrome Integration

Claude Code version 1.0.36 and later includes integration with the Claude Chrome extension, enabling Claude to analyze web pages and web applications in context.

When both extensions are installed:
- Claude can reference open web pages in conversations
- Analyze frontend code rendered in the browser
- Understand context from documentation websites
- Debug web applications with visual feedback

### Conversation History Management

VS Code automatically saves conversation history locally. To manage your history:

1. Open the Claude panel
2. Click the history icon (clock symbol)
3. Browse previous conversations
4. Click a conversation to resume it
5. Archive or delete conversations as needed

### Performance Optimization

For large projects, optimize Claude Code's performance:

- **Limit Context Size**: Reference only necessary files and folders
- **Use Line Ranges**: Specify exact line numbers to reduce token usage
- **Enable .gitignore**: Let Claude respect your .gitignore to avoid unnecessary files
- **Archive Sessions**: Close inactive sessions to reduce memory usage

## JetBrains IDE Integration

Claude Code integrates seamlessly with JetBrains IDEs including IntelliJ IDEA, PyCharm, WebStorm, and other JetBrains products.

### Installation

1. Open JetBrains IDE
2. Navigate to Preferences → Plugins (or IDE → Preferences → Plugins on macOS)
3. Search for "Claude Code" in the Marketplace
4. Click "Install"
5. Restart the IDE when prompted
6. Authenticate with your Claude account

### Accessing Claude in JetBrains

- **Tool Window**: Claude appears as a tool window on the right side of the IDE
- **Context Menu**: Right-click files or code sections to reference them
- **Search**: Use Ctrl+Shift+F (Windows/Linux) or Cmd+Shift+F (macOS) to search and reference files

### JetBrains-Specific Features

**Run Configurations**
Claude can analyze and understand your run configurations, helping debug issues specific to your IDE setup.

**Built-in Tools**
Reference JetBrains built-in tools and analyzers that Claude integrates with for comprehensive code understanding.

**Git Integration**
Claude works with JetBrains' Git interface, understanding your branch structure and commit history.

## Troubleshooting

### Common Issues

**Extension Not Showing**
- Verify VS Code version is 1.98.0 or later
- Restart VS Code completely
- Check Extension marketplace for installation status
- Review Activity Bar for the Claude Code icon

**Authentication Failures**
- Verify your Claude account credentials
- Check internet connection
- Sign out and sign in again via the extension settings
- Clear VS Code cache if issues persist

**File References Not Working**
- Ensure file paths are correct
- Verify files exist in your project
- Check that .gitignore isn't excluding necessary files
- Try using the full file path instead of fuzzy matching

**Performance Issues**
- Close unused conversation sessions
- Limit the number of referenced files
- Use line ranges for large files
- Disable autosave for non-critical projects

### Getting Help

- Check the official Claude Code documentation
- Review GitHub issues for known problems
- Contact Anthropic support through your account dashboard
- Consult IDE-specific forums for integration questions

## Best Practices

### Effective Claude Code Usage

1. **Clear Context**: Provide specific file references rather than entire projects
2. **Focused Requests**: Ask Claude to solve one problem at a time
3. **Review Changes**: Always review Claude's proposed changes before accepting
4. **Plan Mode**: Use "Plan" permission mode when trying new approaches
5. **Session Organization**: Create separate sessions for distinct tasks
6. **Documentation**: Reference API docs and specs when available

### Security Considerations

- Never share API keys in Claude conversations
- Use permission modes carefully—review all changes before auto-accepting
- Respect .gitignore and don't reference sensitive configuration files
- Be cautious with Auto-Accept mode on shared machines
- Regularly review conversation history for accidental information exposure
