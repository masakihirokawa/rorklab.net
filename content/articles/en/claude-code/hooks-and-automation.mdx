## What Are Hooks?

Hooks in Claude Code are event-driven scripts that execute automatically at specific points in your workflow. They let you automate repetitive tasks, inject custom logic into Claude's tool execution, and extend Claude Code's functionality beyond built-in capabilities.

<Callout type="info">
Hooks are configured in the `.claude/settings.json` file at your project root. They run locally in your development environment, giving you full control over custom automation.
</Callout>

## Types of Hooks

Claude Code supports three main hook types:

1. **PreToolUse** — Runs before Claude executes a tool
2. **PostToolUse** — Runs after Claude executes a tool
3. **Notification** — Sends alerts and notifications to external services

## Setting Up Your First Hook

### Creating .claude/settings.json

First, create the configuration file at your project root:

```bash
mkdir -p .claude
touch .claude/settings.json
```

### Basic Configuration

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write", "file:delete"],
        "handler": "scripts/pre-write-validation.js"
      }
    ],
    "postToolUse": [
      {
        "tools": ["git:commit"],
        "handler": "scripts/post-commit-notification.js"
      }
    ]
  }
}
```

## PreToolUse Hooks

PreToolUse hooks run before Claude executes a tool, allowing you to validate parameters, add context, or prevent dangerous operations.

### Use Cases for PreToolUse

- **Validation**: Check parameters before file operations
- **Security**: Prevent commits to critical branches
- **Logging**: Record what Claude is about to do
- **Modification**: Add or modify tool parameters

### Example: File Write Validation

Create `scripts/pre-write-validation.js`:

```javascript
// Pre-validation hook for file write operations
module.exports = async function preWriteValidation(context) {
  const { tool, params } = context;

  // Prevent writing to sensitive directories
  const forbiddenPaths = [
    '.env',
    'node_modules',
    '.git',
    'secrets/'
  ];

  const filePath = params.path;

  for (const forbidden of forbiddenPaths) {
    if (filePath.includes(forbidden)) {
      return {
        allowed: false,
        reason: `Cannot write to ${filePath}. This path is protected.`
      };
    }
  }

  // Allow the operation
  return {
    allowed: true,
    modifiedParams: params
  };
};
```

### Configuration for File Write Validation

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write"],
        "handler": "scripts/pre-write-validation.js"
      }
    ]
  }
}
```

### Example: Git Branch Protection

Create `scripts/pre-git-commit.js`:

```javascript
// Prevent commits to main/master branches
module.exports = async function preGitCommit(context) {
  const { params } = context;

  // Get current branch from git
  const { execSync } = require('child_process');
  const currentBranch = execSync('git rev-parse --abbrev-ref HEAD')
    .toString()
    .trim();

  // Protect main branches
  const protectedBranches = ['main', 'master', 'production'];

  if (protectedBranches.includes(currentBranch)) {
    return {
      allowed: false,
      reason: `Cannot commit to protected branch: ${currentBranch}`
    };
  }

  return {
    allowed: true
  };
};
```

## PostToolUse Hooks

PostToolUse hooks execute after Claude completes a tool operation, allowing you to log results, trigger notifications, or perform cleanup operations.

### Use Cases for PostToolUse

- **Logging**: Record successful operations
- **Notifications**: Alert team members of changes
- **Cleanup**: Remove temporary files or cache
- **Validation**: Verify operations completed correctly

### Example: Commit Notification

Create `scripts/post-commit-notification.js`:

```javascript
// Notify team after successful git commits
module.exports = async function postCommitNotification(context) {
  const { params, result } = context;

  if (result.success) {
    const message = `
      Git Commit Completed:
      - Commit: ${params.message}
      - Hash: ${result.commitHash}
      - Author: ${result.author}
    `;

    // Log to console
    console.log(message);

    // Send to notification service (optional)
    // await notifySlack(message);
  }

  return { processed: true };
};
```

### Example: Build Automation

Create `scripts/post-test-automation.js`:

```javascript
// Run build after test succeeds
module.exports = async function postTestAutomation(context) {
  const { result, params } = context;

  if (result.testsPassed) {
    console.log('Tests passed! Running build...');

    const { execSync } = require('child_process');

    try {
      execSync('npm run build', { stdio: 'inherit' });
      console.log('Build completed successfully');
      return { success: true, action: 'build' };
    } catch (error) {
      console.error('Build failed:', error.message);
      return { success: false, action: 'build', error: error.message };
    }
  }

  return { processed: true };
};
```

### Configuration for Build Automation

```json
{
  "hooks": {
    "postToolUse": [
      {
        "tools": ["test"],
        "handler": "scripts/post-test-automation.js"
      }
    ]
  }
}
```

## Notification Hooks

Notification hooks send alerts to external services like Slack, Discord, or email when specific events occur.

### Example: Slack Notifications

Create `scripts/slack-notifier.js`:

```javascript
// Send notifications to Slack
const https = require('https');

module.exports = async function slackNotifier(context) {
  const { event, data } = context;
  const webhookUrl = process.env.SLACK_WEBHOOK_URL;

  if (!webhookUrl) {
    console.warn('SLACK_WEBHOOK_URL not set');
    return;
  }

  const message = {
    text: `Claude Code Event: ${event}`,
    blocks: [
      {
        type: 'section',
        text: {
          type: 'mrkdwn',
          text: `*Event*: ${event}\n*Details*: ${JSON.stringify(data, null, 2)}`
        }
      }
    ]
  };

  return new Promise((resolve, reject) => {
    const postData = JSON.stringify(message);

    https.post(new URL(webhookUrl), {
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': postData.length
      }
    }, (res) => {
      resolve({ sent: true, status: res.statusCode });
    }).on('error', reject).end(postData);
  });
};
```

### Configuration for Slack Notifications

```json
{
  "hooks": {
    "notifications": [
      {
        "events": ["commit", "deploy", "error"],
        "handler": "scripts/slack-notifier.js",
        "config": {
          "channel": "#dev-alerts"
        }
      }
    ]
  }
}
```

## Best Practices for Hooks

1. **Keep hooks fast**: Long-running hooks slow down your workflow
2. **Handle errors gracefully**: Hooks should fail safely
3. **Log important actions**: Make hooks auditable
4. **Test thoroughly**: Test hooks before deploying to production
5. **Document your hooks**: Include comments explaining what each does
6. **Use environment variables**: Store secrets in `.env` files
7. **Limit scope**: Apply hooks only to the tools that need them

<Callout type="warning">
Hooks run locally in your environment. Never use hooks to execute untrusted code or operations you don't fully understand.
</Callout>

## Debugging Hooks

### Check Hook Execution

```bash
# View hook logs
claude --verbose

# Test a specific hook
node scripts/pre-write-validation.js
```

### Common Issues

| Issue | Solution |
|-------|----------|
| Hook not running | Check `.claude/settings.json` for syntax errors |
| Hook throws error | Test hook script independently with `node` |
| Parameters not passed | Verify hook handler signature matches expected format |
| Performance issues | Profile hook execution time, consider async optimization |

## Advanced Hook Patterns

### Conditional Execution

```javascript
module.exports = async function conditionalHook(context) {
  const isDevelopment = process.env.NODE_ENV === 'development';

  if (!isDevelopment) {
    return { allowed: true }; // Skip hook in production
  }

  // Execute validation only in development
  return validateChanges(context);
};
```

### Chaining Multiple Hooks

```json
{
  "hooks": {
    "preToolUse": [
      {
        "tools": ["file:write"],
        "handlers": [
          "scripts/validate-syntax.js",
          "scripts/check-format.js",
          "scripts/verify-permissions.js"
        ]
      }
    ]
  }
}
```
