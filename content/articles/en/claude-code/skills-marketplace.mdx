## What is skills.sh?

skills.sh is an open-source agent skills marketplace that extends the capabilities of Claude Code, Cowork, and other AI agents. Built on the open standard defined by [agentskills.io](https://agentskills.io/specification), skills.sh provides a unified ecosystem where developers can discover, install, and publish skills that add new functionality to their AI-powered workflows.

As of March 2026, the skills marketplace hosts thousands of verified skills spanning creative applications (art, music, design), technical tasks (testing, code review, git automation), and enterprise workflows (communications, branding, and data processing).

<Callout type="info">
Skills are fundamentally different from slash commands. Skills are model-invoked—Claude automatically decides when to use them based on context. Slash commands, by contrast, are user-invoked—you explicitly type the command to trigger them. This means skills enable more intelligent, context-aware automation in your workflows without requiring explicit user action.
</Callout>

## Understanding Skills Architecture

A skill is a modular capability that contains:

- **SKILL.md file**: A Markdown file with YAML frontmatter containing metadata and instructions
- **Metadata**: Information about the skill's purpose, version, author, and triggers
- **Instructions**: Detailed guidance for how the agent should use this skill
- **Optional resources**: Supporting files like scripts, references, and assets

Skills are stored in two possible locations:
- **Global installation**: `~/.claude/skills/` for skills available across all projects
- **Project-specific installation**: `./.claude/skills/` within a project directory

The modular design allows agents to automatically load relevant skills when needed, creating a seamless and context-aware experience.

## Installing the Skills CLI

The Skills CLI is the command-line tool for managing skills. Install it using npm:

```bash
npm install -g skills
```

Or use npx to run commands without installing:

```bash
npx skills --help
```

<Callout type="tip">
If you're using Node.js, you likely have npm already installed. Verify your setup with `npm --version` and `node --version` before proceeding.
</Callout>

## Discovering and Installing Skills

### Finding Skills

Search the marketplace for skills matching your needs:

```bash
npx skills find [query]
```

**Common search examples:**

```bash
# Find React and Next.js development skills
npx skills find react

# Find deployment-related skills
npx skills find deploy

# Find testing frameworks and tools
npx skills find testing

# Find Git automation skills
npx skills find git

# Find code review and linting skills
npx skills find review
```

Each search result displays the skill name, description, version, and author information. This helps you evaluate whether a skill matches your specific requirements.

### Installing Skills

Once you've identified a skill, install it with:

```bash
npx skills add owner/repo@skill-name
```

**Installation examples:**

```bash
# Install a specific skill to your project
npx skills add vercel/skills@react-best-practices

# Install globally (available in all projects)
npx skills add vercel/skills@react-best-practices -g

# Install silently without confirmation prompts
npx skills add vercel/skills@react-best-practices -y

# Install globally and silently
npx skills add vercel/skills@react-best-practices -g -y
```

Global installation (`-g` flag) makes the skill available in all your projects, while project-level installation stores skills only in the current project's `.claude/skills/` directory.

<Callout type="warning">
Always review a skill's source and description before installation. While the marketplace contains community-verified skills, ensure you trust the author and understand what the skill does in your workflows.
</Callout>

## Managing Your Skills

### Checking for Updates

View all installed skills and check for available updates:

```bash
npx skills check
```

This command displays which skills have newer versions available and provides guidance on updating them.

### Updating Skills

Keep your skills current with the latest improvements and bug fixes:

```bash
npx skills update
```

This updates all installed skills to their latest versions. You can also update individual skills by specifying the skill name.

### Listing Installed Skills

View all skills currently installed in your project:

```bash
npx skills list
```

### Removing Skills

Remove skills you no longer use:

```bash
npx skills remove [skill-name]
```

Or use the interactive removal tool:

```bash
npx skills remove
```

This provides a menu to select which skills to uninstall.

## Understanding the SKILL.md Format

Every skill is defined by a `SKILL.md` file containing YAML frontmatter and Markdown content.

### Frontmatter Structure

```yaml
---
name: "React Best Practices"
description: "Guide Claude through React component patterns, hooks best practices, and performance optimization"
version: "1.0.0"
author: "Vercel Labs"
license: "MIT"
triggers: ["react", "component", "hooks"]
metadata:
  category: "development"
  difficulty: "intermediate"
  keywords: ["react", "javascript", "frontend"]
---
```

**Frontmatter fields:**

| Field | Description | Required |
|-------|-------------|----------|
| `name` | Skill display name | Yes |
| `description` | Brief explanation of skill purpose | Yes |
| `version` | Semantic version number | Yes |
| `author` | Skill creator/maintainer | Yes |
| `license` | Open source license type | Yes |
| `triggers` | Keywords that activate this skill | No |
| `metadata` | Additional categorization and tagging | No |

### Markdown Content

After the frontmatter, write clear instructions for how the agent should use this skill:

```markdown
# React Best Practices

When helping users with React development, apply these patterns:

## Component Structure
- Use functional components with hooks
- Extract complex logic into custom hooks
- Keep components focused and single-responsibility

## Performance Optimization
- Use React.memo() for expensive components
- Implement useMemo() for expensive calculations
- Profile with React DevTools Profiler

## Testing Best Practices
- Write tests focusing on user behavior
- Use React Testing Library
- Avoid testing implementation details
```

<Callout type="tip">
Keep SKILL.md instructions concise and under 5,000 tokens. Focus on the essential guidance that will help Claude make better decisions in your workflows.
</Callout>

## Creating Custom Skills

### Initializing a New Skill

Create a new skill project structure:

```bash
npx skills init my-custom-skill
```

This generates:
- `SKILL.md` with template frontmatter and content
- Optional directories for scripts, references, and assets
- A basic project structure

### Skill Development Workflow

1. **Define the Purpose**: Clearly state what task or domain your skill addresses
2. **Write Frontmatter**: Complete all required metadata fields
3. **Document Instructions**: Provide clear, actionable guidance for the AI agent
4. **Add Resources** (optional): Include scripts or reference files in supporting directories
5. **Test Locally**: Verify the skill works with Claude Code before publishing
6. **Publish to GitHub**: Push your skill repository for sharing

### Example Custom Skill

Create a skill for your specific workflow, like automated testing:

```yaml
---
name: "Automated Test Generation"
description: "Generate comprehensive test suites for JavaScript functions using Jest"
version: "1.0.0"
author: "Your Name"
license: "MIT"
triggers: ["test", "jest", "testing"]
---

# Automated Test Generation

When asked to write tests for JavaScript functions:

## Test Structure
- Create test files named `[function].test.js`
- Organize tests into logical describe() blocks
- Use clear, descriptive test names

## Coverage Requirements
- Test happy path scenarios
- Test edge cases and error conditions
- Achieve minimum 80% code coverage
- Include integration tests for complex functions

## Best Practices
- Use beforeEach() for test setup
- Mock external dependencies
- Keep assertions focused and specific
```

### Testing Your Skill

Before publishing, test your skill with Claude Code:

```bash
# Copy your skill to the local skills directory
cp -r my-custom-skill ~/.claude/skills/

# Test with Claude Code
npx claude code "test my new skill"
```

## Skills in Cowork

Cowork, the desktop application for AI-assisted document and spreadsheet editing, automatically loads compatible skills to enhance productivity.

## Publishing Your Skill

### Preparing Your Repository

1. Create a GitHub repository for your skill
2. Structure your repository:
   ```
   my-skill/
   ├── SKILL.md
   ├── scripts/
   ├── references/
   ├── assets/
   └── README.md
   ```

3. Write a comprehensive README explaining the skill's purpose and usage
4. Tag your repository with `agent-skill` and `skills` labels

### Registering with skills.sh

1. Push your skill repository to GitHub
2. Visit the skills.sh registry and submit your skill
3. Provide:
   - Repository URL
   - Skill name and description
   - Category and keywords
   - License type

4. The community reviews and verifies your skill
5. Once approved, it appears in marketplace searches

### Documentation Best Practices

- Include usage examples in your README
- Provide clear descriptions of what the skill does
- Document any external dependencies
- Add troubleshooting sections for common issues
- Maintain a CHANGELOG documenting updates

## Troubleshooting Common Issues

### Skill Not Found in Search

**Problem**: `npx skills find [query]` returns no results

**Solutions**:
- Try broader search terms (e.g., "react" instead of "react-hooks")
- Check the skill name on GitHub and search by author
- Verify the marketplace is accessible: `npx skills check-connection`

### Installation Fails with Permission Error

**Problem**: Installation fails with permission denied

**Solutions**:
```bash
# Use sudo for global installation
sudo npx skills add owner/repo@skill-name -g

# Or fix npm permissions
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
export PATH=~/.npm-global/bin:$PATH
```

### Skill Not Being Used by Claude

**Problem**: Installed skill isn't helping with relevant tasks

**Solutions**:
- Verify installation: `npx skills list`
- Check skill triggers: `npx skills info owner/repo@skill-name`
- Review the SKILL.md instructions for clarity
- Test with explicit mentions: "Use my custom-skill for this task"

### Conflicting Skills

**Problem**: Multiple skills causing unexpected behavior

**Solutions**:
- Remove conflicting skills: `npx skills remove conflicting-skill`
- Use project-specific installation to isolate skills
- Document skill compatibility in your project README

### Update Failures

**Problem**: `npx skills update` fails or rolls back

**Solutions**:
```bash
# Force update a specific skill
npx skills add owner/repo@skill-name --force

# Check skill version compatibility
npx skills check

# Revert to previous version if needed
npx skills add owner/repo@skill-name@0.9.0
```

## Best Practices for Skill Management

### Organization

- **Use global installation** for skills you use across many projects
- **Use project installation** for domain-specific skills
- **Document your skill setup** in project README files
- **Version control your .claude/skills directory** when using project-specific skills

### Performance

- **Remove unused skills** to reduce context overhead
- **Regularly update** to get bug fixes and improvements
- **Monitor skill performance** and check usage frequency
- **Consolidate overlapping skills** when possible

### Security

- **Review skill sources** before installation
- **Check license compatibility** with your projects
- **Audit installed skills** periodically for vulnerabilities
- **Keep skills updated** for the latest security patches

## Advanced: Skill Configuration

### Project-Level Configuration

Create a `.claude/skills.config.json` to customize skill behavior:

```json
{
  "autoLoad": true,
  "maxTokens": 5000,
  "enabledSkills": [
    "vercel/skills@react-best-practices",
    "composio/skills@git-automation"
  ],
  "disabledSkills": [
    "old-skill@outdated"
  ]
}
```

### Environment Variables

Set skill configuration via environment variables:

```bash
# Disable specific skills
export DISABLED_SKILLS="old-skill,experimental-skill"

# Set maximum token usage
export SKILL_MAX_TOKENS=8000

# Use custom skill registry
export SKILLS_REGISTRY="https://custom-registry.example.com"
```
