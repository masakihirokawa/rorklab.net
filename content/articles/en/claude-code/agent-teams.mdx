## Understanding Agent Teams in Claude Code

Modern development workflows often require juggling multiple tasks simultaneously: refactoring code while reviewing pull requests, writing tests while documenting API changes, or analyzing performance across different modules. Claude Code's agent team system lets you distribute these tasks across specialized AI agents that work in parallel, coordinating through a main orchestrating agent.

At the core of this system is the subagent concept. When your main Claude Code session encounters a complex task, it can spawn specialized subagents to handle specific responsibilities. Rather than the main agent trying to do everything sequentially, it breaks work into logical chunks and coordinates multiple agents working in parallel. This not only speeds up development but also allows each agent to specialize in what it does best.

## Agent Types and Specialization

Claude Code provides several predefined agent types, each designed for specific use cases in your workflow:

**Explore agents** operate in read-only mode, making them perfect for investigation and analysis tasks. They can examine your codebase, search for patterns, review documentation, and provide insights without risking unintended modifications. Use these when you need comprehensive code analysis or when you want to verify assumptions before making changes.

**Plan agents** focus on strategic thinking and decision-making. These agents excel at architecture review, refactoring strategy, migration planning, and creating implementation roadmaps. They combine analysis with forward-looking recommendations.

**General-purpose agents** handle the full spectrum of development tasks, from writing code to refactoring to documentation updates. They're your go-to choice when you need flexibility and don't need specialization.

**Bash agents** specialize in command-line operations and scripting. They're essential for deployment tasks, environment setup, performance testing, and any work that requires direct shell access. Their specialization makes them faster and more reliable for command-heavy workflows.

## Creating Custom Subagents

Beyond the built-in types, you can define custom subagents tailored to your project's specific needs. Store agent definitions in a `.claude/agents/` directory within your workspace. Each agent is defined as a Markdown file with YAML frontmatter:

```markdown
---
name: "Security Auditor"
description: "Specializes in identifying security vulnerabilities and recommending fixes"
tools: ["read", "grep", "bash"]
model: "claude-opus-4-6"
---

You are a security specialist focused on identifying and fixing vulnerabilities in code.
When reviewing code, prioritize OWASP Top 10 issues, input validation, authentication
checks, and data protection. Provide clear remediation steps for any issues found.
```

This custom agent inherits the tools you specify and can be invoked alongside your other agents. You define what tools it has access to, what model powers it, and its specialized instructions through the system prompt.

## Dynamic Agent Definition with --agents Flag

For workflows where agent composition changes based on the task at hand, use the `--agents` flag to define agents dynamically:

```bash
claude code --agents "security-reviewer,performance-auditor,api-designer" \
  "Refactor the authentication module with security, performance, and API design review"
```

This approach lets you compose agent teams on-the-fly without predefined configuration. The specified agents spin up, each review the codebase from their specialty angle, and coordinate findings back through the main agent.

## Parallel Processing at Scale: The /batch Skill

When you need to apply consistent changes across a large codebase, the `/batch` skill orchestrates parallel operations using temporary git worktrees. Rather than applying changes sequentially (which would be slow), `/batch` creates isolated working trees for each task, allowing multiple transformations to happen simultaneously:

```bash
/batch apply-linting-rules
```

This might create five parallel worktrees, each handling different modules. As each worktree completes its task, changes merge back to your main branch. The parallelization dramatically reduces execution time for large-scale refactoring initiatives.

## Multi-Agent Review with /simplify

Code simplification often benefits from multiple perspectives. The `/simplify` skill automatically spawns three parallel review agents, each analyzing code from a different angle—one focusing on readability, another on performance, and a third on maintainability. They work simultaneously, then coordinate their feedback for a comprehensive simplification strategy.

## Agent Team Configuration and Coordination Modes

Control how your agent team communicates and coordinates using the `--teammate-mode` flag:

- **auto mode** lets Claude Code choose the best coordination method based on your system and task complexity
- **in-process mode** runs all agents within your current session, useful for lightweight tasks or constrained environments
- **tmux mode** creates separate tmux sessions for each agent, providing isolation and independent execution

```bash
claude code --teammate-mode tmux \
  --agents "ui-specialist,api-specialist,database-specialist" \
  "Redesign the user service endpoint"
```

In tmux mode, each agent operates in its own session with independent terminal state, making it ideal for tasks requiring clean separation or different environment variables.

## Practical Applications: Real-World Examples

Consider a scenario where you're refactoring a large microservice. Rather than doing this sequentially, spawn specialized agents: one focusing on API contract changes, another on database migrations, and a third on testing updates. While one agent modifies the API layer, another can simultaneously work on the data model, and the third crafts comprehensive tests. The main agent coordinates progress, resolves conflicts, and ensures consistency.

Another common workflow involves simultaneous PR review and test expansion. When a colleague submits a pull request, launch a review agent to analyze code quality while spawning another agent to identify untested code paths and draft additional test cases. Both work in parallel, and within minutes you have comprehensive review feedback plus expanded test coverage—something that would take much longer sequentially.

Large-scale code migration projects particularly benefit from agent teams. You might spawn agents to migrate different modules to a new framework simultaneously. Each agent operates independently, encounters module-specific challenges, and learns from attempts. The main agent coordinates shared concerns like shared utilities and ensures the final result cohesively fits together.

## Integration with the Agent SDK

Claude Code's agent team system builds on top of the Claude Agent SDK, providing a high-level interface for multi-agent coordination. If you need deeper customization or want to build agent teams programmatically, the SDK offers API-level integration:

```python
from anthropic import Anthropic

client = Anthropic()

# Create a team of specialized agents
agents = [
    {"name": "code_reviewer", "instructions": "Review code for quality and best practices"},
    {"name": "test_writer", "instructions": "Write comprehensive test cases"},
    {"name": "documenter", "instructions": "Create clear documentation"}
]

# Orchestrate parallel execution and coordinate results
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=4096,
    instructions="Coordinate these specialized agents to improve our codebase"
)
```

Through the SDK, you can build custom orchestration logic, integrate agent teams into automated pipelines, and create sophisticated multi-agent workflows tailored to your specific development process.

## Getting Started with Agent Teams

Start by identifying workflow bottlenecks where sequential processing slows you down. Could code review happen in parallel with test expansion? Could architectural planning happen while implementation begins on separate modules? These are prime candidates for agent teams.

Begin with predefined agent types, then experiment with custom agents as you discover specialty needs in your workflow. The `--agents` flag provides flexibility for exploration without requiring permanent configuration. As your team scales, invest time in defining reusable custom agents that encode best practices for your organization.

Agent teams transform development from a sequential process into a coordinated, parallel effort. When used strategically, they accelerate iteration cycles, improve code quality through multiple perspectives, and let you accomplish in hours what once took days.
