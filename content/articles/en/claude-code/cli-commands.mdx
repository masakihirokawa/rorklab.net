## Claude Code CLI Overview

Claude Code centers around the `claude` command, offering a rich set of subcommands and flags. This guide systematically covers all commands based on the official CLI reference.

## Core Commands

### Interactive Mode

```bash
claude
```

Running without arguments launches interactive mode. You can have a back-and-forth conversation with Claude while coding.

### Quick Queries

```bash
claude "Explain the structure of this project"
```

Pass a short query directly to start interactive mode with that prompt sent immediately.

### One-Shot Mode (-p)

```bash
claude -p "Translate README.md to Japanese"
```

The `-p` (`--print`) flag executes a single task non-interactively. Output goes to stdout, making it perfect for piping and redirection.

```bash
# Piping example
cat error.log | claude -p "Analyze the root cause of these errors"
```

## Key Flags

### Model Selection

```bash
claude -p "Review this code" --model claude-opus-4-6
```

Use `--model` to specify which model to use. Default is Claude Sonnet, but Opus delivers higher accuracy for complex tasks.

### Directory Addition

```bash
claude --add-dir ../shared-lib
```

`--add-dir` adds folders outside the working directory to Claude Code's context. Useful for monorepos and shared libraries.

### Tool Restrictions

```bash
claude -p "Run the tests" --allowedTools "Bash,Read"
```

`--allowedTools` restricts which tools Claude Code can use. Essential for safe execution in CI/CD pipelines.

### System Prompt Extension

```bash
claude -p "Refactor this" --append-system-prompt "Always write comments in English"
```

`--append-system-prompt` appends custom rules to the system prompt. Great for enforcing coding conventions.

## Authentication Commands

```bash
# Login
claude auth login

# Logout
claude auth logout

# Check status
claude auth status
```

<Callout type="info">
The first run automatically starts the browser-based auth flow. For API key usage, set the `ANTHROPIC_API_KEY` environment variable.
</Callout>

## Configuration Commands

```bash
# View settings
claude config list

# Change a setting
claude config set theme dark

# Reset settings
claude config reset
```

## MCP (Model Context Protocol) Management

```bash
# Add an MCP server
claude mcp add my-server -- npx -y @example/mcp-server

# List registered servers
claude mcp list

# Remove a server
claude mcp remove my-server
```

MCP servers let you connect Claude Code to external services like Slack, GitHub, and databases.

## Slash Commands (Inside Interactive Mode)

| Command | Description |
|---------|-------------|
| `/help` | Show help |
| `/clear` | Clear conversation history |
| `/compact` | Summarize and compress context |
| `/cost` | Show token usage |
| `/doctor` | Diagnose configuration issues |
| `/init` | Create CLAUDE.md project config |
| `/review` | Start a code review |
| `/terminal-setup` | Configure terminal key bindings |

<Callout type="tip">
`/compact` is especially useful during long sessions when the context window fills up. It summarizes the conversation so you can keep working without interruption.
</Callout>

## Practical Use Cases

### Automated Code Review in CI/CD

```bash
claude -p "Review this diff and flag any issues" \
  --allowedTools "Read" \
  --model claude-opus-4-6 < pr-diff.txt
```

### Batch File Processing

```bash
find src -name "*.ts" | xargs -I {} claude -p "Fix TypeScript errors in: {}"
```

### Resume Previous Session

```bash
claude -r
```

`-r` (`--resume`) resumes your last session. Handy when you need to pick up where you left off.
