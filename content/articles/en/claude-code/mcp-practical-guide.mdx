## How MCP Transforms the Claude Experience

MCP (Model Context Protocol) is an open standard protocol announced by Anthropic in November 2024. Designed as a unified interface connecting AI models to external tools and data sources, it has grown in just one year to an ecosystem with over 97 million monthly SDK downloads and 10,000+ active servers.

Claude Code, Claude Desktop, and many other clients including Cursor and VS Code natively support MCP—simply adding an MCP server infinitely extends Claude's capabilities.

<Callout type="info">
**Core MCP Concepts:**
- **Tools** — Actions Claude can execute (API calls, DB queries, etc.)
- **Resources** — Data sources Claude can read (files, DB records, etc.)
- **Prompts** — Pre-defined templates and workflows
</Callout>

## Top 7 Recommended MCP Servers

### 1. GitHub MCP Server

GitHub's official MCP server is the first server adopted by over 90% of Claude Code users. It provides Claude access to nearly all GitHub features: repository operations, PR creation and review, issue management, code search, and CI/CD status monitoring.

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    }
  }
}
```

**Use Case:** Instructions like "Check the review comments on this PR and fix them" or "Create a branch based on Issue #42 and write the code" work naturally.

### 2. Filesystem MCP Server

Provides access to the local filesystem. It includes explicit permission gates for safety, allowing Claude to directly read, write, search, and organize files.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
```

**Use Case:** Hand your entire project folder to Claude with requests like "Organize the file structure in this directory" or "Find and clean up old log files."

### 3. Figma MCP Server (Dev Mode)

Figma's official Dev Mode MCP server lets Claude generate code directly from design data. Instead of working from screenshots, it understands the actual design structure—layers, components, Auto Layout, and design tokens—enabling high-accuracy code conversion.

**Use Case:** "Convert this Figma frame to a React component" or "Update styles to match the design tokens."

### 4. Sentry MCP Server

This server integrates with real-time error tracking. Claude can retrieve full error context—stack traces, user impact, frequency—from Sentry without opening the UI and suggest fixes.

**Use Case:** "Show me today's error list" or "Identify the cause of this TypeError and write a fix."

### 5. Brave Search MCP Server

Adds web search capabilities to Claude. When you need to check the latest documentation or library information during development, you can get search results directly within the conversation.

### 6. PostgreSQL / SQLite MCP Server

Enables direct database querying. Useful for verifying data during development, inserting test data, and analyzing schemas through Claude.

### 7. Cloudflare MCP Server

Manage Cloudflare resources—Workers, KV, D1, R2—through Claude. Convenient for checking deployment status and modifying worker configurations.

## Remote MCP Servers and Authentication

MCP initially centered on local execution (stdio), but now supports remote servers as well.

### Streamable HTTP Transport

The current MCP standard transport is **Streamable HTTP**. The legacy SSE (Server-Sent Events) method is being gradually deprecated.

### OAuth 2.1 Authentication

Remote MCP servers support authentication via the OAuth 2.1 protocol. Claude's OAuth callback URL is `https://claude.ai/api/mcp/auth_callback`, with support for token expiry and refresh.

<Callout type="warning">
**Security Best Practices:**
- Treat each MCP server as a microservice with its own blast radius
- Require TLS and client-server authentication
- Scope API keys to minimum privileges; start high-impact servers in read-only mode
- Manage credentials like `GITHUB_PERSONAL_ACCESS_TOKEN` via environment variables
</Callout>

## Practical Scenarios: Automating Development with MCP

### Scenario 1: Issue → Code → PR Full Automation

Combining GitHub MCP + Filesystem MCP lets Claude read issue requirements, create a branch, write code, and open a PR.

```
You: Read the requirements from Issue #15 and implement them, then create a PR
Claude: Let me check Issue #15... It's about "Dark mode support."
        I'll create a feature/dark-mode branch and start implementing.
        ...(coding)...
        PR #23 has been created. Please review.
```

### Scenario 2: Error Detection → Root Cause → Fix

By chaining Sentry MCP + GitHub MCP, you can handle everything from production error detection to creating a fix PR in one flow.

### Scenario 3: Design → Code → Deploy

Linking Figma MCP → Filesystem MCP → GitHub MCP → Cloudflare MCP lets Claude assist the entire pipeline from design to deployment.

## Configuring MCP Servers

### In Claude Code

```bash
# Project scope (.mcp.json)
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxx -- npx -y @modelcontextprotocol/server-github

# Global scope
claude mcp add --scope user filesystem -- npx -y @modelcontextprotocol/server-filesystem ~/projects
```

### In Claude Desktop

Add MCP server configurations to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/you/projects"]
    }
  }
}
```

MCP has rapidly gained adoption as the standard protocol for connecting Claude to external tools. By combining key servers like GitHub, Figma, and Sentry, Claude can assist your entire development workflow—from issue management and design conversion to error handling and deployment.

Start with the GitHub MCP server and add more servers as your project needs grow. That's the most practical approach.
