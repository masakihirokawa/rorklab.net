## What Is Claude Code Voice Mode?

On March 3, 2026, Anthropic officially began rolling out native **Voice Mode** for Claude Code. This feature lets developers issue spoken commands directly from the terminal, enabling a hands-free coding workflow.

Instead of typing out every instruction, you can now speak naturally — saying things like "refactor the authentication middleware" or "explain this failing test" — while Claude Code transcribes and processes your commands.

<Callout type="info">
Voice Mode is currently in a gradual rollout as of March 2026. It's available to approximately 5% of users initially, with broader access expected over the coming weeks. Available on Pro, Max, Team, and Enterprise plans at no additional cost.
</Callout>

## How to Enable Voice Mode

### The /voice Command

Simply type the following slash command during any Claude Code session to toggle voice mode on:

```bash
/voice
```

Once enabled, you'll see an indicator in your terminal prompt confirming that voice input is available. To disable it, type `/voice` again.

### Push-to-Talk Mechanism

Voice Mode uses a **Push-to-Talk** approach — there's no always-listening mode, so you don't have to worry about background noise triggering unintended input.

Here's how it works:

1. **Hold the spacebar** to start recording
2. **Speak naturally** about what you want Claude to do
3. **Release the spacebar** — your speech is transcribed (STT) and inserted into the prompt

```
[Hold spacebar] → Speak → [Release spacebar] → Text appears in input field
```

<Callout type="tip">
Voice input is inserted as text at your cursor position. This means you can freely mix typed and spoken input within a single prompt. For example, type the beginning of a command and use voice to complete the rest.
</Callout>

### Customizing the Activation Key

The default activation key is the spacebar, but you can customize it in `keybindings.json`. Using a modifier+letter combination like `meta+k` eliminates any interference with normal typing.

```json
{
  "voice.activationKey": "meta+k"
}
```

## Voice Input Only (Text Output)

The current Voice Mode is **input-only**. You speak your instructions, and Claude responds with text. There is no text-to-speech (TTS) output at this time.

This is an intentional design choice — developers need to read code and terminal output, making text responses more practical for coding workflows.

## Why Voice Mode Transforms Development Workflows

### 1. Maintaining Flow State

Typing long prompts breaks your train of thought. With voice input, you can articulate ideas as they come to mind, staying in "vibe coding" mode where you describe what you want conversationally while the AI handles the details.

### 2. Communicating Complex Instructions

Instructions like "Replace the state management in this component with useReducer, add type definitions for the actions, and write tests too" are faster and more natural to speak than to type.

### 3. Multitasking

You can review code on screen while simultaneously giving Claude voice instructions for modifications — true parallel workflows become possible.

### 4. Accessibility

For developers with hand or arm strain, voice input serves as a critical accessibility tool. It can significantly reduce the risk of RSI (Repetitive Strain Injury).

## Third-Party STT Tool Integration

In addition to native Voice Mode, third-party speech-to-text tools offer alternative approaches — particularly useful for languages not yet natively supported.

### Aqua Voice

[Aqua Voice](https://withaqua.com/) is an AI-powered dictation tool that reads screen context to improve recognition accuracy. It has exceptionally high accuracy for programming terminology and works directly in the Claude Code terminal.

```bash
# Aqua Voice operates as an OS-level dictation tool
# When your cursor is in the terminal, text goes directly to Claude Code
```

For details, see our [Aqua Voice × Claude Guide](/en/cowork/aqua-voice-guide).

### VoiceMode MCP Server

[VoiceMode MCP](https://getvoicemode.com/) integrates with Claude Code as an MCP (Model Context Protocol) server. It uses OpenAI Whisper for STT and OpenAI TTS for speech output.

```bash
# Install VoiceMode MCP
npm install -g voicemode-mcp

# Add to Claude Code's MCP configuration
claude mcp add voicemode
```

The key advantage of VoiceMode MCP is that it enables **voice output** (TTS) from Claude, not just voice input.

### Claude STT

[Claude STT](https://github.com/jarrodwatts/claude-stt) is an open-source tool that adds streaming speech-to-text input to Claude Code.

```bash
# Using Claude STT
# Ctrl+Shift+Space to start recording
# Ctrl+Shift+Space again to stop and transcribe
```

## Requirements and Limitations

### Supported Plans

Voice Mode is available on the following plans at no additional cost:

- **Pro Plan**
- **Max Plan**
- **Team Plan**
- **Enterprise Plan**

Free plan users do not currently have access.

### Current Limitations

- **Gradual rollout**: Available to approximately 5% of users as of March 2026, expanding over coming weeks
- **Terminal only**: VS Code extension and desktop app support is unconfirmed
- **Input only**: No text-to-speech (TTS) responses
- **Language coverage**: Not all languages are natively supported yet

## Troubleshooting

### Voice Mode Not Appearing

The rollout may not have reached your account yet. Check the following:

1. Update Claude Code to the latest version
2. Confirm you're on a paid plan (Pro/Max/Team/Enterprise)
3. Wait a few days and check again

```bash
# Update Claude Code to the latest version
claude update
```

### Speech Not Recognized Correctly

- Use voice mode in a quiet environment
- Verify your microphone is properly configured
- Press the spacebar before you start speaking, and release after you finish

### Spacebar Interference

If the spacebar triggers Voice Mode during normal typing, change the activation key:

```json
{
  "voice.activationKey": "meta+shift+v"
}
```

## FAQ

### Q: Can I use Voice Mode on the free plan?

A: No, Voice Mode is currently available only on paid plans (Pro, Max, Team, Enterprise). There is no additional charge for these plan holders.

### Q: Can I still type normally with Voice Mode enabled?

A: Yes. Voice Mode only activates when you hold the spacebar, so regular typing works as expected. If spacebar interference is a concern, change the activation key to a modifier combination.

### Q: Does Voice Mode work offline?

A: Claude Code is a cloud-based service that requires an internet connection. Speech recognition processing is also presumed to be server-side.

### Q: Is there any mode other than Push-to-Talk?

A: Currently, Push-to-Talk is the only input method. There is no always-listening (hands-free) mode. This design prevents background noise from triggering unintended input.

### Q: Can I combine Voice Mode with other Claude Code features?

A: Yes. Voice Mode works alongside all existing Claude Code features including MCP servers, agent teams, hooks, and plugins. It's simply an alternative input method — everything else functions exactly as before.
