# Bonus: Complete Guide to Deploying Next.js on Cloudflare Pages

Cloudflare Pages is a powerful and cost-effective platform for deploying Next.js applications. In this guide, we'll walk through the complete process of deploying a Next.js site to Cloudflare Pages using Claude Code, covering everything from setup to production.

## Why Choose Cloudflare Pages?

Cloudflare Pages offers significant advantages over traditional platforms like Vercel.

**Generous Free Tier**
- Unlimited static site hosting
- 500,000 requests per month (free plan)
- Unlimited bandwidth
- Global delivery via Cloudflare's CDN

**Edge Computing at Scale**
- Serverless functions via Cloudflare Workers
- 300+ data centers worldwide
- Minimal latency for content delivery
- No geographic restrictions

**Zero Cold Start Issues**
Unlike Vercel, where cold starts are common, Cloudflare Pages provides:
- Instant activation via Edge Runtime
- Always-on workers (paid plans)
- Region-specific optimization

**Enterprise-Grade Security**
- DDoS protection included by default
- Web Application Firewall (WAF)
- No additional configuration required

## Prerequisites

Before you begin, ensure you have:

1. **Node.js** (version 18.0 or higher)
2. **Cloudflare Account** (free tier available)
3. **Wrangler CLI** (Cloudflare's official tool)
4. **GitHub Repository** (for deployment)
5. **Custom Domain** (optional, claudelab.net in this example)

Install Wrangler globally:

```bash
npm install -g wrangler
```

## Setting Up Your Next.js Project

If you have an existing Next.js project, you'll need to add the OpenNext adapter.

### Installing the OpenNext Adapter

```bash
npm install @opennextjs/cloudflare
```

This adapter converts nearly all Next.js features—including App Router, API Routes, and Image Optimization—to be compatible with Cloudflare Pages.

### Initialization Command

```bash
npx opennextjs-cloudflare init
```

This command automatically generates:
- `open-next.config.ts` — OpenNext configuration
- `wrangler.jsonc` — Cloudflare configuration
- `.env.local` — Environment variable template

## Configuration Files Explained

### open-next.config.ts

```typescript
import type { OpenNextConfig } from '@opennextjs/cloudflare';

const config: OpenNextConfig = {
  default: {
    override: {
      wrapper: '@opennextjs/cloudflare/wrappers/cloudflare-pages',
      converter: '@opennextjs/cloudflare/adapters/cloudflare-pages',
    },
  },
  middleware: {
    external: [],
    override: {
      wrapper: '@opennextjs/cloudflare/wrappers/cloudflare-pages',
      converter: '@opennextjs/cloudflare/adapters/cloudflare-pages',
    },
  },
};

export default config;
```

This configuration ensures all Next.js routes and APIs are converted to run on Cloudflare Workers.

### wrangler.jsonc

```jsonc
{
  "name": "claudelab-pages",
  "type": "service-worker",
  "main": ".open-next/server/index.js",
  "site": {
    "bucket": ".open-next/static"
  },
  "triggers": {
    "crons": []
  },
  "env": {
    "production": {
      "name": "claudelab-pages-prod",
      "route": "claudelab.net/*"
    },
    "staging": {
      "name": "claudelab-pages-staging"
    }
  },
  "vars": {
    "SITE_NAME": "Claude Lab"
  }
}
```

<Callout type="info">
`wrangler.jsonc` is JSON with comments. Wrangler natively supports .jsonc format and comments are valid.
</Callout>

## Configuring next-intl for Multi-Language Support

Since claudelab.net supports multiple languages, configuring the `next-intl` middleware is essential.

### i18n Middleware

```typescript
import createMiddleware from 'next-intl/middleware';
import { locales, localePrefix } from './config';

export default createMiddleware({
  locales,
  localePrefix,
  defaultLocale: 'en',
});

export const config = {
  matcher: ['/((?!_next|_vercel|.*\\..*).*)'],
};
```

<Callout type="warning">
Cloudflare Pages runs on Edge Runtime, so certain APIs like `getRequestContext()` or Deno features are unavailable. Always use Node.js-compatible APIs only.
</Callout>

### Setting i18n in next.config.js

```javascript
const withNextIntl = require('next-intl/plugin')('./i18n.ts');

/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  experimental: {
    // May be needed for Cloudflare Pages
    serverComponentsExternalPackages: [],
  },
};

module.exports = withNextIntl(nextConfig);
```

## The Build Process

### Build Command

```bash
npm run build
```

Add this to your `package.json`:

```json
{
  "scripts": {
    "build": "npx opennextjs-cloudflare build",
    "deploy": "npx wrangler deploy",
    "preview": "npm run build && npx wrangler dev"
  }
}
```

### Build Output

On success, a `.open-next` directory is generated:

```
.open-next/
├── server/          # API Routes and SSR
├── static/          # Static files
├── cache/           # ISR cache
└── index.js         # Entry point
```

This entire directory is deployed to Cloudflare.

## Deployment Methods

### Method 1: Deploy with Wrangler CLI

```bash
npx wrangler deploy
```

This command:
1. Builds the `.open-next` directory
2. Uploads static files to Cloudflare
3. Deploys the Workers script

After completion, your site is accessible at `https://<your-project>.pages.dev`.

### Method 2: GitHub Integration (Recommended)

Deploy automatically by pushing to GitHub.

1. **Configure in Cloudflare Dashboard**
   - Pages → Create → Select GitHub
   - Authorize your repository
   - Build settings:
     ```
     Framework: Next.js
     Build command: npm run build
     Build output directory: .open-next
     ```

2. **Enable Auto-Deployment**
   - Automatic builds on every push
   - Preview environments per PR
   - Production environment (custom domain)

<Callout type="info">
GitHub integration automatically sets up your CI/CD pipeline. Preview environments are generated for each PR, making team review processes more efficient.
</Callout>

## Setting Environment Variables

### Via Cloudflare Dashboard

1. Open your Pages project
2. Navigate to Settings → Environment Variables
3. Add variables for each environment (production/staging)

```
STRIPE_SECRET_KEY = sk_live_xxx
DATABASE_URL = postgresql://prod-user:pass@prod-db/prod
OPENAI_API_KEY = sk-xxx
```

### Via Wrangler CLI

```bash
wrangler secret put STRIPE_SECRET_KEY
# Enter the value at the prompt

wrangler secret put STRIPE_SECRET_KEY --env production
```

### Accessing in Code

Use environment variables in API Routes and Server Components:

```typescript
// app/api/payment/route.ts
export async function POST(request: Request) {
  const stripeKey = process.env.STRIPE_SECRET_KEY;
  const stripe = new Stripe(stripeKey);

  // Call Stripe API
  return Response.json({ success: true });
}
```

## Setting Up a Custom Domain

### Binding claudelab.net

1. **In Cloudflare Dashboard**
   - Pages project → Custom Domains
   - Add claudelab.net

2. **Configure DNS**
   - At your DNS provider for claudelab.net, set CNAME:
   ```
   CNAME: claudelab.net → <your-project>.pages.dev
   ```

3. **SSL/TLS Setup**
   - Cloudflare automatically issues Let's Encrypt certificates
   - HTTPS is enabled by default

4. **Redirect Configuration**
   ```
   http://claudelab.net → https://claudelab.net (automatic)
   www.claudelab.net → https://claudelab.net (via wrangler)
   ```

<Callout type="warning">
DNS changes may take up to 24 hours to propagate. However, if Cloudflare is your nameserver, propagation typically occurs within minutes.
</Callout>

## Troubleshooting

### Node.js APIs Unavailable in Edge Runtime

Cloudflare Workers runs on Edge Runtime, so some Node.js APIs are unavailable.

**Unavailable Features:**
- `fs` (filesystem)
- `child_process` (subprocess)
- Some `crypto` methods
- Direct `process.env` access (use different methods)

**Workaround:**
```typescript
// ❌ Not available
const fs = require('fs');
const data = fs.readFileSync('file.txt');

// ✅ Use Cloudflare Workers KV instead
import { kv } from '@cloudflare/workers-kv';
const data = await kv.get('file-key');
```

### Image Optimization Limitations

`next/image` doesn't work by default, but OpenNext handles this:

```javascript
// next.config.js
const config = {
  images: {
    remotePatterns: [
      { protocol: 'https', hostname: 'cdn.example.com' },
    ],
    // Cloudflare recommends external URLs only
    unoptimized: process.env.NODE_ENV === 'production',
  },
};
```

### Optimizing Build Size

OpenNext can split functions by route:

```typescript
// open-next.config.ts
export default {
  split: true, // Split functions by route
  minify: true,
  sourceMap: false,
};
```
