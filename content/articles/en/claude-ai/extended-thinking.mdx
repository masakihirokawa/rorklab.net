## What Is Extended Thinking?

**Extended Thinking** is a feature that allows Claude to show its step-by-step reasoning process before delivering a final answer. By explicitly unfolding the reasoning that would normally happen internally, Claude produces more accurate responses for complex problems.

## Why It Matters

When humans solve a difficult math problem, they don't jump straight to the answer—they work through it on paper. Extended Thinking gives Claude the same "think on paper" process.

### High-Impact Tasks

- Multi-step mathematical proofs
- Complex code debugging and architecture
- Legal document and contract analysis
- Scientific hypothesis verification
- Logical structuring of long-form content

### Limited Impact Tasks

- Simple Q&A ("What is the capital of Japan?")
- Routine document creation
- Short translations

## Using It on Claude.ai

On Claude.ai, no special configuration is needed. Claude automatically adjusts thinking depth based on question complexity. You can also explicitly request deeper thinking.

```
"Please think through this problem carefully, step by step, before answering"
```

## Using It via API

Enable Extended Thinking by setting the `thinking` parameter in your API request.

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000
    },
    messages=[{
        "role": "user",
        "content": "Prove that there are infinitely many prime numbers"
    }]
)

for block in response.content:
    if block.type == "thinking":
        print(f"Thinking: {block.thinking}")
    elif block.type == "text":
        print(f"Response: {block.text}")
```

### Parameters

- **type**: `"enabled"` for manual activation. Claude Opus 4.6 also supports `"adaptive"`
- **budget_tokens**: Maximum tokens for thinking. Must be less than `max_tokens`

## Adaptive Thinking

Claude Opus 4.6 introduces **Adaptive Thinking**, a new mode that automatically adjusts thinking depth.

```python
thinking={"type": "adaptive"}
```

No `budget_tokens` needed—Claude automatically calibrates how deeply to reason based on problem complexity.

## Cost Considerations

Extended Thinking tokens are billable. Thinking tokens count as output tokens, so your `budget_tokens` setting directly affects cost.

Setting a large thinking budget for simple questions wastes money. Match the budget to task complexity for optimal cost-quality balance.

## Practical Examples

### Code Review

```
thinking={"type": "enabled", "budget_tokens": 8000}

"Check this code for security vulnerabilities, testing against each common attack pattern"
```

### Data Analysis

```
thinking={"type": "enabled", "budget_tokens": 12000}

"Analyze the trends in this sales data and provide a forecast with supporting evidence"
```

## Summary

Extended Thinking transforms Claude from an "instant answer machine" into a "deep thinking partner." Use it actively for complex reasoning tasks and stick with standard mode for simple ones to optimize the balance between quality and cost.
