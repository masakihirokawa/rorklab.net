## Why Build a Portfolio App?

If you're a freelance designer or indie developer, your portfolio is your calling card. While websites are the standard, having a **mobile app** for your portfolio unlocks unique advantages.

Imagine pulling up your app during a client meeting to showcase your work instantly. Or sending push notifications when you publish new projects. Having your own app on the App Store or Google Play is itself a powerful demonstration of your technical skills.

In this guide, we'll walk you through building a professional portfolio app using [Rork](https://rork.com/) — with no coding required.

## What You'll Learn

By the end of this guide, you'll know how to:

- Create a new portfolio app project in Rork
- Build a work gallery, detail view, and profile screen
- Implement image galleries with category filtering
- Add animations and dark mode for a polished look
- Test on real devices and publish to the App Store / Google Play

**Target audience**: Rork beginners to intermediate users, designers, freelance developers

## Prerequisites

Before you start, make sure you have:

- **A Rork account**: Sign up at [rork.com](https://rork.com/) (Rork Max plan recommended)
- **Project assets**: Images and descriptions for 5–10 of your best projects
- **Profile details**: Your name, bio, social media links, and contact info

If you're new to Rork, check out "[What is Rork? Getting Started with AI App Development](/en/articles/rork-basics/what-is-rork)" first.

## Step 1: Create Your Project with a Well-Crafted Prompt

The most important part of building with Rork is your **initial prompt**. A specific, structured prompt dramatically improves the quality of the generated app.

Enter something like this:

```text
Create a portfolio app with the following screens:

【Screens】
1. Home: Grid layout (2 columns) showing projects with category filter
2. Project Detail: Large image, project title, description, technology tags
3. Profile: Avatar, bio, skill bars, social media links
4. Contact: Name, email, and message form

【Design Requirements】
- Minimal, modern design
- Dark mode support
- Smooth page transition animations
- Tap-to-zoom image viewer

【Categories】
- UI/UX Design
- Mobile Apps
- Websites
- Illustration
```

The key here is providing **clearly numbered screen descriptions**. Rork's AI generates significantly better results when instructions are well-structured.

## Step 2: Refine the Generated App

Once Rork generates the initial app, review the preview and make adjustments.

### Optimizing the Grid Layout

The gallery layout makes the biggest visual impact. If adjustments are needed, send a follow-up prompt:

```text
Update the project grid with these changes:
- Set card spacing to 12px
- Add subtle shadows to each card
- Lock image aspect ratio to 4:3
- Add a slight lift animation on tap
```

### Structuring Your Portfolio Data

Organize your portfolio items with a clear data structure:

```typescript
// Portfolio item type definition
interface PortfolioItem {
  id: string;
  title: string;           // Project name
  category: string;        // Category (UI Design, App, etc.)
  thumbnail: string;       // Thumbnail image URL
  images: string[];        // Detail image URLs
  description: string;     // Project summary
  technologies: string[];  // Tech stack used
  url?: string;            // External link (optional)
  date: string;            // Creation date
}

// Example portfolio data
const portfolioItems: PortfolioItem[] = [
  {
    id: "1",
    title: "Fitness App UI",
    category: "UI/UX Design",
    thumbnail: "https://example.com/thumb1.jpg",
    images: [
      "https://example.com/detail1-1.jpg",
      "https://example.com/detail1-2.jpg",
    ],
    description: "UI design for a healthcare company's fitness application",
    technologies: ["Figma", "React Native", "Rork"],
    url: "https://example.com/project1",
    date: "2026-02",
  },
];
```

## Step 3: Implement Category Filtering

Category filtering is essential for a portfolio app — it helps visitors quickly find the type of work they're looking for.

Add the filter with this prompt:

```text
Add a category filter to the top of the home screen:
- Horizontal scrollable tab bar
- Tabs: "All", "UI/UX Design", "Mobile Apps", "Websites", "Illustration"
- Active tab highlighted with accent color
- Fade animation when switching categories
```

The best approach is to make changes incrementally, **verifying each step** before moving on. This prevents Rork from breaking existing functionality. For more tips, see "[Rork Prompt Engineering Tips](/en/articles/rork-basics/rork-prompt-engineering-app-design-tips)."

## Step 4: Build the Profile Screen

Your profile screen is where you sell yourself. Make it count with skill bars, a compelling bio, and easy-to-access social links.

```text
Update the profile screen:
- Circular avatar image at the top (120px)
- Name and title (e.g., "Freelance Designer")
- Short bio (3–4 lines)
- Skills section with animated progress bars:
  - Figma: 95%
  - React Native: 85%
  - UI Design: 90%
  - Illustration: 80%
- Social media icon buttons in a row (Instagram, X, GitHub, LinkedIn)
- "Hire Me" button that navigates to the Contact screen
```

## Step 5: Add Dark Mode and Animations

Polish your app's visual appeal with dark mode and smooth animations.

```text
Please make these design improvements:
1. Auto dark mode toggle (follows device system settings)
2. Fade-in + slide-up animation for project cards on load
3. Smooth transitions between screens
4. Animated skill bars on the profile screen (grow from zero on appear)
```

Dark mode is especially important for design portfolios — it makes your work pop against the dark background. With Rork Max, [dark mode support](/en/articles/rork-dev/rork-max-dark-mode-theming) is just one prompt away.

## Step 6: Set Up the Contact Form

Create a contact form so potential clients can reach you directly. Rork integrates well with [Supabase](/en/articles/rork-dev/rork-auth-firebase-supabase) for storing form submissions.

```text
Create the contact screen with:
- Input fields: Name (required), Email (required), Message (required, multiline)
- Validation (empty check, email format check)
- Loading spinner on the submit button
- Thank you modal on successful submission
- Save data to Supabase
```

## Testing and Publishing

Once your app is complete, test it on real devices and prepare for launch.

With Rork Max, you can use EAS Build to generate both iOS and Android binaries. Share a TestFlight build with beta testers, gather feedback, and refine before submitting to the App Store and Google Play.

For detailed publishing steps, see the "[App Store Publishing Guide](/en/articles/rork-basics/app-store-publishing)" and "[Google Play Publishing Guide](/en/articles/rork-basics/google-play-publishing)."

## Frequently Asked Questions (FAQ)

### Q1: Can I build a portfolio app on Rork's free plan?

Yes, you can build the basic features on the free plan. However, automatic dark mode switching, advanced animations, and EAS Build for real device testing require the Rork Max plan. We recommend starting with the free plan and upgrading as needed.

### Q2: Where should I host my portfolio images?

You have several options. Supabase Storage lets you upload images directly from within the app. Cloudflare R2 and AWS S3 are also solid choices. For performance, optimize your thumbnails to around 400×300px and detail images to about 1200px wide.

### Q3: How much does it cost to publish a portfolio app?

The Apple Developer Program costs $99/year, and a Google Play Developer registration is a one-time $25 fee. Rork itself is free on the basic plan, with Rork Max available on a monthly subscription. For details, see "[Rork Pricing Plans](/en/articles/rork-business/rork-pricing-plans)."

### Q4: How do I add new projects after publishing?

The simplest approach is to update your project data in Rork and redeploy. For more frequent updates, consider using Supabase as a backend CMS — you can build an admin screen within the same app to add and edit projects without redeploying.
