## Creating Your Project and Designing Prompts

### Start a New Rork Project

Head over to [rork.com](https://rork.com/) and create a new project. Give it a clear name like "EventCountdown" so it's easy to find later.

Rork generates your app's structure based on AI prompts. Try entering something like this:

```text
Build an event countdown app.

Features:
- Users can add events with a name and target date
- Each event shows the remaining days in real time
- Events are displayed in a card-based list view
- Users can add, edit, and delete events
- Events can be color-coded by category (birthday, travel, work, other)

Design:
- Modern card-based UI
- Dark mode support
- Large, prominent countdown numbers
- Different icons and colors for each category
```

Once you submit this prompt, Rork will automatically generate the screen layout and core components for your app.

### Tips for Writing Better Prompts

Getting great results from Rork comes down to how well you communicate your vision.

**List specific features** — "Make a countdown app" is too vague. Spell out exactly what your app should do so the AI can deliver accurate results.

**Describe the design direction** — Keywords like "modern," "minimal," or "colorful" help Rork understand the aesthetic you're going for.

**Explain the user flow** — Phrases like "tapping the add button opens a form" give Rork enough context to structure the navigation correctly.

## Implementing the Date Calculation Logic

The heart of any countdown app is the date math. Here's the core logic that Rork generates under the hood:

```typescript
// Calculate remaining days until a target date
const calculateDaysRemaining = (targetDate: string): number => {
  const now = new Date();
  const target = new Date(targetDate);

  // Reset time portions to compare dates only
  now.setHours(0, 0, 0, 0);
  target.setHours(0, 0, 0, 0);

  const diffTime = target.getTime() - now.getTime();
  const diffDays = Math.ceil(diffTime / (1000 * 60 * 60 * 24));

  return diffDays;
};

// Usage example
const daysLeft = calculateDaysRemaining("2026-12-25");
console.log(`${daysLeft} days until Christmas`);
// Expected output: 276 days until Christmas
```

The key detail here is calling `setHours(0, 0, 0, 0)` to zero out the time portion. Without this, checking at 11 PM might show one fewer day than expected, which would confuse users.

### Showing a Detailed Countdown

If you want to display hours, minutes, and seconds alongside the day count, you'll need a real-time update mechanism:

```typescript
// Calculate detailed time remaining
const calculateTimeRemaining = (targetDate: string) => {
  const now = new Date();
  const target = new Date(targetDate);
  const diff = target.getTime() - now.getTime();

  if (diff <= 0) {
    return { days: 0, hours: 0, minutes: 0, seconds: 0, isPast: true };
  }

  return {
    days: Math.floor(diff / (1000 * 60 * 60 * 24)),
    hours: Math.floor((diff % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60)),
    minutes: Math.floor((diff % (1000 * 60 * 60)) / (1000 * 60)),
    seconds: Math.floor((diff % (1000 * 60)) / 1000),
    isPast: false,
  };
};

// Expected output:
// { days: 276, hours: 14, minutes: 32, seconds: 8, isPast: false }
```

To add this feature in Rork, simply tell the AI: "Show the remaining time in days, hours, minutes, and seconds with a live ticker."

### Displaying the Event List

Registered events should appear as styled cards in a scrollable list. Sorting them by days remaining — closest deadline first — makes the app feel intuitive and useful.

```typescript
// Sort events by remaining days (ascending)
const sortedEvents = events
  .filter((event) => calculateDaysRemaining(event.date) >= 0)
  .sort((a, b) => {
    const daysA = calculateDaysRemaining(a.date);
    const daysB = calculateDaysRemaining(b.date);
    return daysA - daysB;
  });

// Expected output (as of March 24):

// ]
```

## Data Persistence and State Management

To keep event data safe between app launches, you'll need to persist it to local storage. In a React Native + Expo environment, `AsyncStorage` is the standard approach.

```typescript
import AsyncStorage from "@react-native-async-storage/async-storage";

// Type definition for countdown events
interface CountdownEvent {
  id: string;
  name: string;
  date: string;
  category: "birthday" | "travel" | "work" | "school" | "other";
  memo?: string;
  createdAt: string;
}

// Save events to local storage
const saveEvents = async (events: CountdownEvent[]) => {
  try {
    await AsyncStorage.setItem("countdown_events", JSON.stringify(events));
  } catch (error) {
    console.error("Failed to save events:", error);
  }
};

// Load events from local storage
const loadEvents = async (): Promise<CountdownEvent[]> => {
  try {
    const data = await AsyncStorage.getItem("countdown_events");
    return data ? JSON.parse(data) : [];
  } catch (error) {
    console.error("Failed to load events:", error);
    return [];
  }
};
```

Rork's generated code often includes state management out of the box, but persistence may not always be included. If your data disappears when you restart the app, just tell Rork: "Persist event data using AsyncStorage so it survives app restarts." For a deeper dive into offline data handling, see [Building Offline-Ready Rork Apps with AsyncStorage and NetInfo](/en/articles/rork-dev/rork-offline-ready-asyncstorage-netinfo).

## Common Issues and Fixes

### Date Is Off by One Day

This usually happens due to timezone handling. When you create a Date object from a string like `new Date("2026-12-25")`, it may be interpreted as UTC. Calling `setHours(0, 0, 0, 0)` to reset to local midnight is the most reliable fix.

### Past Events Still Showing Up

Add a filter to hide events whose date has already passed. You can either tell Rork "Automatically hide events that have already passed" or add a `.filter()` that only keeps events where `daysRemaining >= 0`.

### Data Disappears After Restarting the App

This means AsyncStorage persistence hasn't been implemented. Tell Rork: "Save all event data locally so it persists between app launches."

## FAQ

### Can I build a countdown app with no programming experience?

Absolutely. Rork's AI generates all the code for you based on natural language prompts. You describe what you want in plain English, and Rork builds the screens, logic, and styling automatically. No coding skills required.

### How accurate is the countdown?

For day-level countdowns, JavaScript's Date object provides more than enough precision. If you add a seconds-level live ticker using `setInterval`, the countdown updates every second and stays accurate throughout.

### Is there a limit on how many events I can track?

AsyncStorage typically supports up to around 6MB of data, which is enough for hundreds of events. For normal usage, you won't hit any practical limits.

### Can I publish my countdown app to the App Store?

Yes. Rork includes built-in publishing tools for submitting apps to both the App Store and Google Play. With Rork Max, you can submit to the App Store in just two clicks. Refer to the store submission guide for step-by-step instructions.
