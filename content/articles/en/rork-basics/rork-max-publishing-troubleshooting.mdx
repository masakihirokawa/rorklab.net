Rork Max's one-click publishing is powerful, but things don't always go smoothly. This guide walks through the most common issues you'll encounter when publishing apps with Rork Max and how to fix them.

## Apple Developer Account Setup Issues

Most publishing failures trace back to Apple Developer Account configuration.

### Provisioning Profile Not Found

**Error**: "Provisioning Profile not found"

**Causes**:
- Not registered for Apple Developer Program
- Team ID not correctly configured
- Certificate cache corrupted in Xcode

**Solution**:
1. Reconnect your Apple Developer Account in Rork Max settings
2. If you use Two-Factor Authentication, generate an App Password and use it instead of your account password
3. Explicitly select your team in Rork Max and save

### Code Signing Failed

**Error**: "Failed to sign the build" or "Code signing is required"

**Causes**: Development certificate is invalid or expired

**Solution**:
1. Log into [Apple Developer Website](https://developer.apple.com)
2. Navigate to "Certificates, Identifiers & Profiles" → "Certificates"
3. Verify you have valid certificates; create new ones if needed
4. Clear Xcode cache: `rm -rf ~/Library/MobileDevice/Provisioning\ Profiles/*`
5. Restart Xcode

### Bundle ID Already in Use

**Error**: "Bundle ID is already in use by another team"

**Causes**: 
- Attempting to use the same Bundle ID for multiple apps
- Using an old Bundle ID from a previous app

**Solution**:
1. Use a different Bundle ID in Rork Max (e.g., `com.mycompany.myapp2`)
2. Check Apple Developer Website to see if the Bundle ID is already registered
3. Follow the naming convention: `com.[company].[appname]`

## App Store Review Rejections

Your build succeeds, but then App Store review rejects it. Here's how to handle the most common rejection reasons.

### Guideline 2.1 — App Crashes

**Error**: "The app crashed on launch"

**Cause**: Unhandled exception on startup

**Solution**:
- Review the code generated by Rork Max
- Use TestFlight to reproduce the crash
- Check crash logs in Xcode console
- Fix the issue and resubmit with a new build number

### Guideline 2.1 — Incomplete Implementation

**Error**: "The feature is not fully implemented or behaves unexpectedly"

**Cause**: Gap between app description and actual functionality

**Solution**:
- Review your App Store Connect metadata
- Screenshots should only show implemented features
- Keywords should match actual app functionality
- Don't include "Coming Soon" features in marketing materials

### Guideline 5.1.1 — Missing Privacy Policy

**Error**: "This app requests user-sensitive data but does not have a privacy policy"

**Cause**: Accessing sensitive data (location, camera, contacts) without a privacy policy

**Solution**:
1. Add a Privacy Policy URL in App Store Connect
2. Document:
   - What data you collect
   - Why you collect it
   - How you share it (if at all)
   - How users can delete their data

### Guideline 4.3 — Looks Like Spam

**Error**: "Your app looks like spam or is functionally similar to an existing app"

**Cause**: Low-quality appearance or too similar to existing apps

**Solution**:
- Add unique features
- Use high-quality screenshots (actual device screenshots, not mockups)
- Include a preview video
- Write detailed, clear app descriptions

## Metadata and Screenshots

### Incorrect Screenshot Dimensions

**Error**: "The screenshot dimensions are incorrect"

**Solution** — Use exact dimensions per device:
- iPhone 15 Pro Max: 1290×2796 px
- iPhone 15: 1179×2556 px
- iPad Pro (12.9): 2048×2732 px

### App Icon Issues

**Error**: "The app icon does not meet our requirements"

**Requirements**:
- Dimensions: 1024×1024 px
- Format: PNG, JPEG, or GIF
- No rounded corners (App Store applies them automatically)
- Solid background (no transparency)

### Poor App Description

Even if your app is approved, a weak description hurts downloads and ratings.

**Best practices**:
- Lead with your main feature in the first two lines
- Use bullet points to highlight benefits
- Write naturally—avoid keyword stuffing
- Use region-specific language for each market

## Build Error Diagnosis

### Build Failed — No Details

**How to debug**:
1. Check Rork Max build history
2. Open the build log for error messages
3. Common errors:
   - `Module not found`: Missing dependency
   - `Syntax error`: Bug in generated code (report to Rork support)
   - `Unable to boot simulator`: Restart the simulator

### Out of Memory During Build

**Symptom**: Build process terminates mid-way

**Solution**:
- Close unnecessary applications
- Clear Xcode cache: `rm -rf ~/Library/Developer/Xcode/DerivedData/*`
- Restart your Mac
- As a last resort: Uninstall and reinstall Rork Max

## TestFlight Beta Testing

Always test with TestFlight before submitting to the App Store.

### Uploading to TestFlight

1. After Rork Max build completes, tap "Send to TestFlight"
2. In App Store Connect, open the TestFlight tab
3. Add internal testers (yourself or team)
4. Wait for build processing (typically 1–3 hours)
5. Install TestFlight on your iPhone and install the beta build

### Beta Testing Checklist

- [ ] App launches without crashing
- [ ] Core features work as expected
- [ ] No crashes during normal use
- [ ] UI renders correctly on iPhone and iPad
- [ ] Permission requests (location, camera) appear correctly
- [ ] Background processes work as designed

If you find issues during testing, fix them, create a new build, and resubmit to TestFlight.

## Common "Not Found" and Blank Screen Issues

### App Not Found in App Store

**Cause**: Incorrect build selected in App Store Connect

**Solution**:
1. In App Store Connect, open the Builds tab
2. Verify the correct build is in "Submitted" status
3. If needed, select a different build

### App Launches but Shows Blank Screen

**Cause**: UI rendering failed or assets didn't load

**Solution**:
- Use Rork Max preview to test locally
- Verify assets (images) are included in the build
- If the app needs internet, ensure network connectivity in TestFlight
- Open the project in Xcode and check the simulator console for errors

## Post-Launch Updates

### Publishing a New Version

1. Update your app in Rork Max
2. In App Store Connect, increment the build number (e.g., 1.0.1)
3. Write detailed release notes in "What's New"
4. Submit for review

### Common Update Rejections

- New features not described in release notes
- Removed features not explained
- Privacy policy outdated

Address these by thoroughly updating your release notes and app settings.

## Summary

Publishing with Rork Max is straightforward, but success requires understanding App Store guidelines and thorough TestFlight testing. Use this guide's checklist as you prepare your app for the world.
