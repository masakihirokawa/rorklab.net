Finishing a Rork Max app is one challenge. Getting it into the App Store is another. Apple's review process looks intimidating the first time, but the common failure points are predictable — and avoidable.

## What You'll Need

Before starting, have these ready:

- **Apple Developer Program membership** ($99/year, individual or organization) — sign up at [developer.apple.com](https://developer.apple.com). Approval can take 24–48 hours.
- **A Mac with Xcode** — required for the final upload step even if you export from Rork Max
- **App Store Connect access** — comes automatically with your Developer membership

## Step 1: Export from Rork Max

In Rork Max, go to File → Export → iOS App.

Key settings to configure:

- **Bundle Identifier**: `com.yourname.appname` — choose carefully, this is hard to change later
- **Display Name**: what users see on their home screen
- **Version**: `1.0.0` for a first release
- **Build Number**: `1` — increment this with each upload

## Step 2: Configure in Xcode

Open the exported project in Xcode.

Under Project Settings → Signing & Capabilities:
- Set **Team** to your Apple Developer team
- Confirm the **Bundle Identifier** matches what you set in Rork Max
- Enable **Automatically manage signing**

**Icon check**: App Store review rejects icons that contain alpha (transparency) channels. Inspect your exported icon and ensure it's fully opaque.

## Step 3: Create the App in App Store Connect

Log in to [App Store Connect](https://appstoreconnect.apple.com) → My Apps → + → New App.

You'll need:
- App name (30 character max)
- Bundle ID (from Step 1)
- SKU (internal identifier — any alphanumeric string works)
- Primary language

## Step 4: Complete the App Metadata

This is where many developers rush and later get rejected.

**Description**: The first three lines appear before "More" — make them count. State the core value clearly without requiring users to expand.

**Screenshots required**:
- iPhone 6.5" (required)
- iPhone 5.5" (required)
- iPad only if you're submitting for iPad
- Simulator screenshots are accepted for review purposes

**Privacy policy**: Required for all App Store submissions, even if your app collects no data. Host a simple page on GitHub Pages, Notion, or any public URL.

## Step 5: Upload the Build

In Xcode: Product → Archive → Distribute App → App Store Connect → Upload

The build appears in App Store Connect under TestFlight & Builds within about 30 minutes.

## Common Rejection Reasons

**1. Crashes during review**
Apple's reviewers actually run the app. Test thoroughly on a real device via TestFlight before submitting.

**2. Misleading descriptions**
Don't describe features the app doesn't have. Reviewers will catch discrepancies between the description and the actual app.

**3. Requiring sign-in for core features**
If your app forces account creation before showing any meaningful content, Apple may reject it. Offer a guest mode, or make the reason for sign-in obvious.

**4. Icon with transparency**
Already mentioned, but worth repeating — alpha channels in icons cause instant rejection.

## Review Timeline

First-time submissions typically receive a decision within 24–48 hours. Rejections come with a specific reason; addressing it and resubmitting usually resolves the issue. Include a note in the "App Review Information" field explaining what you changed.

## Summary

Publishing through Rork Max follows the same path as any Xcode project. The Rork-specific part is the export step — after that, you're in standard Apple territory. Your first submission takes the most time. By your second or third app, the process becomes routine.
