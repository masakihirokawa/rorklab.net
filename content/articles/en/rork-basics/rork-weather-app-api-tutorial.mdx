## Introduction — Learn API Integration by Building a Weather App

A weather app is one of the best projects for learning the fundamentals of mobile app development. The core workflow — fetching data from an external API and displaying it in real time — is a pattern you'll use in virtually every app you build.

With **Rork**, you can build an API-powered weather app in a fraction of the time it would take with traditional development. This tutorial walks you through the entire process: connecting to the OpenWeatherMap API, displaying current conditions and a 5-day forecast, and creating a dynamic UI that responds to weather changes.

Here's what you'll learn:

- How to build an app with external API calls in Rork
- What the OpenWeatherMap free tier offers
- How to use device location to show local weather
- How to implement dynamic UI theming based on conditions

This guide is aimed at beginners who are new to Rork or have never built an API-connected app before. If you've gone through the [Getting Started guide](/en/articles/rork-basics/getting-started), you'll have everything you need.

## Prerequisites and Setup

### What You'll Need

Before we start building, make sure you have the following ready:

1. **A Rork account** — the free plan works for this project
2. **An OpenWeatherMap API key** — sign up at [openweathermap.org](https://openweathermap.org/api) (free)
3. **Basic familiarity with app development concepts** — knowing how to write prompts is enough

### Getting Your OpenWeatherMap API Key

OpenWeatherMap is one of the most widely used weather data APIs. The free tier allows 60 requests per minute, which is more than sufficient for a personal app.

1. Go to [openweathermap.org](https://openweathermap.org/) and create an account
2. Navigate to the "API keys" tab in your dashboard
3. Copy the default key or generate a new one

Note that new API keys can take up to 2 hours to activate, so it's a good idea to do this step first.

## Creating Your Project in Rork

### Writing an Effective Prompt

The quality of your Rork prompt directly affects the quality of the generated app. For a weather app, a detailed and specific prompt works best:

```text
Create a weather forecast app with the following features:

[Core Features]
- Display real-time weather for the user's current location
- Show temperature (current, high, low), humidity, and wind speed
- Display a 5-day weather forecast
- Include a city search feature

[Design]
- Background color changes based on weather conditions (sunny, cloudy, rainy, snowy)
- Large weather icons for easy readability
- Card-based layout for organized information display
- Bottom tab navigation (Current Weather / Forecast / Settings)

[Technical Requirements]
- Use OpenWeatherMap API
- Use expo-location for device location
- Save favorite cities with AsyncStorage
- Support Celsius/Fahrenheit toggle
```

Being specific like this gives Rork the context it needs to generate a well-structured, functional app.

### What to Check After Generation

Once Rork generates your app, review the following:

- **File structure**: Are screens, components, and services properly separated?
- **API calls**: Is `fetch` or `axios` being used correctly for API requests?
- **Error handling**: Does the app handle network failures gracefully?

## Implementing API Integration with OpenWeatherMap

### The Core API Request Structure

The weather data fetching functions form the backbone of your app. Here's what Rork typically generates:

```typescript
// services/weatherApi.ts
const API_KEY = 'YOUR_API_KEY';
const BASE_URL = 'https://api.openweathermap.org/data/2.5';

// Fetch current weather by city name
export const fetchWeatherByCity = async (city: string) => {
  const response = await fetch(
    `${BASE_URL}/weather?q=${city}&appid=${API_KEY}&units=metric`
  );
  if (!response.ok) throw new Error('Failed to fetch weather data');
  return response.json();
};

// Fetch 5-day forecast by coordinates
export const fetchForecast = async (lat: number, lon: number) => {
  const response = await fetch(
    `${BASE_URL}/forecast?lat=${lat}&lon=${lon}&appid=${API_KEY}&units=metric`
  );
  if (!response.ok) throw new Error('Failed to fetch forecast data');
  return response.json();
};
```

The `units=metric` parameter ensures temperatures are returned in Celsius. You can switch to `units=imperial` for Fahrenheit.

### Understanding the API Response

Here's what a typical response looks like:

```json
{
  "name": "Tokyo",
  "main": {
    "temp": 18.5,
    "feels_like": 17.2,
    "humidity": 62,

      "main": "Clouds",

      "icon": "04d"

```

## Adding Location Services with expo-location

### Auto-Detecting the User's Location

Showing weather for the user's current location is a must-have feature. Since Rork generates Expo-based apps, you can leverage `expo-location` for this.

```typescript
// hooks/useLocation.ts
import * as Location from 'expo-location';
import { useState, useEffect } from 'react';

export const useLocation = () => {
  const [location, setLocation] = useState<Location.LocationObject | null>(null);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    (async () => {
      // Request location permission
      const { status } = await Location.requestForegroundPermissionsAsync();
      if (status !== 'granted') {
        setError('Location permission was denied');
        return;
      }
      // Get current position
      const currentLocation = await Location.getCurrentPositionAsync({});
      setLocation(currentLocation);
    })();
  }, []);

  return { location, error };
};
```

### Handling Permission Denials Gracefully

Not every user will grant location access, and your app should work either way. Providing a manual city search as a fallback ensures a smooth experience regardless of permission status.

## Building a Dynamic Weather UI

### Changing the Background Based on Conditions

A weather app feels more immersive when the UI visually reflects current conditions. Here's how to implement dynamic theming:

```typescript
// utils/weatherTheme.ts
export const getWeatherTheme = (weatherMain: string) => {
  const themes: Record<string, { background: string[]; textColor: string }> = {
    Clear: {
      background: ['#4DA0B0', '#D39D38'], // Sunny gradient
      textColor: '#FFFFFF',
    },
    Clouds: {
      background: ['#757F9A', '#D7DDE8'], // Gray gradient
      textColor: '#333333',
    },
    Rain: {
      background: ['#373B44', '#4286f4'], // Dark blue
      textColor: '#FFFFFF',
    },
    Snow: {
      background: ['#E6DADA', '#274046'], // Snowy tones
      textColor: '#333333',
    },
  };
  return themes[weatherMain] || themes['Clear'];
};
```

Including a prompt like "change the background color based on weather conditions" when generating your app in Rork will produce similar code automatically.

## Common Errors and How to Fix Them

### API Key Not Working

OpenWeatherMap API keys can take up to 2 hours to become active after creation. If you're seeing a "401 Unauthorized" error, wait a while and try again.

### Location Not Available

In simulator or emulator environments, the device location defaults to a predefined coordinate (usually near San Francisco). Test on a physical device or update the simulator's location settings to verify your implementation.

### Type Errors from API Data

The shape of the API response may not always match your expectations. Use TypeScript interfaces to define the expected structure, and leverage optional chaining (`?.`) to prevent runtime crashes from missing fields.

## FAQ

### Can I use the OpenWeatherMap API for free?

Yes. The free tier supports up to 60 API calls per minute, which is plenty for a personal weather app. If your app grows and you need higher limits, paid plans are available with additional features like minute-by-minute forecasts and historical data.

### Can I build this app on Rork's free plan?

Absolutely. Rork Free allows up to 5 project generations per week, which is enough to build a basic weather app. If you plan to iterate heavily on the design or add advanced features, the Rork Max plan will give you unlimited generations.

### How do I display weather data in the user's language?

Add the `lang` parameter to your API request (e.g., `lang=ja` for Japanese, `lang=es` for Spanish). This localizes the weather description text returned by OpenWeatherMap. For city names and other UI text, you'll want to manage translations separately.

### Can I publish this weather app to the App Store?

Definitely. If you're on Rork Max, you can go from development to TestFlight beta testing to App Store submission entirely within Rork's dashboard. Weather apps are a beginner-friendly category for your first store listing since they tend to pass review without issues.
