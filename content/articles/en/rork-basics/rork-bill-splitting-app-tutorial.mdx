## Introduction — Why Build a Bill Splitting App?

Splitting dinner with friends, sharing hotel costs on a trip, collecting dues for a group outing — the need to divide expenses comes up more often than you'd think. While there are plenty of apps out there for this, building your own gives you the freedom to customize it exactly the way you want.

With Rork, you can create a fully functional bill splitting app without writing a single line of code. In this tutorial, we'll build an app that includes the following features:

- **Group creation**: Organize expenses by event — dinner, trip, household, you name it
- **Member management**: Add and remove participants with ease
- **Expense tracking**: Record who paid what and for what
- **Automatic calculations**: Instantly see who owes whom and how much
- **History**: Browse past groups and settlements at a glance

Rork's AI app builder handles the heavy lifting, so you can focus on designing a tool that actually fits your life.

## Step 1 — Creating Your Project and Designing the Prompt

Start by creating a new project in Rork. Give it a clear name like "SplitEasy" or "FairShare."

The key to getting great results from Rork is writing a specific, detailed prompt. Here's a solid starting point:

```text
Build a bill splitting and group expense management app.

Core features:
1. Create groups (with a name and members)
2. Record expenses (who paid, amount, description)
3. Automatic bill splitting (even split among members)
4. Show settlement details (who pays whom and how much)
5. Group list and expense history

Design requirements:
- Clean, modern UI
- Tab navigation (Groups, New Group, History)
- Currency display in USD ($)
- Dark mode support
```

When you submit this prompt, Rork's AI will design the screen layout, navigation structure, and data models automatically.

### Tips for Writing Better Prompts

A few things to keep in mind when crafting your prompt:

- **Use bullet points for features**: This helps the AI understand the structure clearly
- **Describe the design direction**: Keywords like "clean" and "modern" go a long way
- **Specify data formats**: Currency symbols, sort order, and display preferences matter

## Step 2 — Understanding the Data Structure

To understand how the app works under the hood, let's look at the data models Rork will generate. This gives you a mental map of how everything connects.

```typescript
// Group type definition
type Group = {
  id: string;
  name: string;           // Group name (e.g., "Okinawa Trip")
  members: string[];      // Array of member names
  createdAt: Date;        // Creation timestamp
};

// Expense type definition
type Expense = {
  id: string;
  groupId: string;        // ID of the parent group
  paidBy: string;         // Name of the person who paid
  amount: number;         // Amount paid
  description: string;    // What the expense was for (e.g., "Lunch")
  createdAt: Date;        // Record timestamp
};

// Settlement type definition
type Settlement = {
  from: string;           // Person who owes money
  to: string;             // Person who is owed
  amount: number;         // Amount to transfer
};
```

The AI creates these models and wires them up to the UI components automatically.

## Step 3 — The Bill Splitting Logic

The heart of any bill splitting app is the calculation engine. Here's what the core logic looks like in the generated code:

```typescript
// Bill splitting calculation function
// Determines the minimum number of transfers needed to settle up
function calculateSettlements(
  expenses: Expense[],
  members: string[]
): Settlement[] {
  // Calculate total paid by each member
  const paid: Record<string, number> = {};
  members.forEach(m => { paid[m] = 0; });
  expenses.forEach(e => { paid[e.paidBy] += e.amount; });

  // Calculate per-person share
  const total = expenses.reduce((sum, e) => sum + e.amount, 0);
  const perPerson = total / members.length;

  // Calculate balance (positive = owed money, negative = owes money)
  const balances = members.map(m => ({
    name: m,
    balance: paid[m] - perPerson,
  }));

  // Generate settlement pairs
  const settlements: Settlement[] = [];
  const debtors = balances.filter(b => b.balance < 0)
    .sort((a, b) => a.balance - b.balance);
  const creditors = balances.filter(b => b.balance > 0)
    .sort((a, b) => b.balance - a.balance);

  let i = 0, j = 0;
  while (i < debtors.length && j < creditors.length) {
    const amount = Math.min(-debtors[i].balance, creditors[j].balance);
    if (amount > 0) {
      settlements.push({
        from: debtors[i].name,
        to: creditors[j].name,
        amount: Math.round(amount * 100) / 100, // Round to cents
      });
    }
    debtors[i].balance += amount;
    creditors[j].balance -= amount;
    if (debtors[i].balance === 0) i++;
    if (creditors[j].balance === 0) j++;
  }

  return settlements;
}

// Example usage:
// 3 friends on a trip. Alice paid $150, Bob paid $90, Charlie paid $60
// Per person share: $100
// → Charlie pays Alice $40, Bob pays Alice $10
```

This algorithm minimizes the number of transfers needed, making settlements as simple as possible.

## Step 4 — Customizing the UI

Once Rork generates the initial screens, you can refine the design with follow-up prompts. Try something like this:

```text
Please make these improvements:
1. Show member count and total amount summary on the group list screen
2. Display a numeric keypad when entering amounts
3. Show settlement results as cards with arrow icons (from → to)
4. Color-code groups by type (travel = blue, dining = green, other = gray)
```

### Design Considerations

For a bill splitting app, the most important thing is making amounts easy to read at a glance. Use large font sizes for dollar amounts and make sure the "who pays whom" flow is immediately obvious.

Rork also lets you tap on individual elements in the preview to fine-tune colors, font sizes, and spacing directly.

## Step 5 — Persisting Data and Managing History

To make sure your data survives app restarts, you'll want to use local storage. Send this follow-up prompt to Rork:

```text
Use AsyncStorage to persist group and expense data locally.
Automatically load data on app startup and save whenever new data is added.
Also add the ability to delete a group by long-pressing on it.
```

This ensures that all your groups and expense records are saved on the device and restored when the app is reopened.

For more advanced needs — like syncing data across multiple devices — you can integrate with backend services like Supabase or Firebase.

## Common Issues and How to Fix Them

Here are some problems you might run into while building your bill splitting app, along with solutions.

**Rounding errors in calculations**: When expenses don't split evenly, you may see small discrepancies. Use `Math.round()` for whole-number currencies or round to two decimal places for cents. You can also build in a rule where the group organizer covers any leftover cents.

**Duplicate member names**: If two members share the same name, things can get confusing. Ask Rork to manage members by unique IDs internally while displaying their chosen names on screen.

**Can't edit or delete expenses**: The initial generation might not include these features. Just send a follow-up prompt like "Make expenses editable on tap and deletable with a left swipe."

## Going Further — Advanced Features to Add

Once your basic app is working, consider adding these features to make it even more useful.

**Uneven splits**: Not everyone should always pay the same amount — think adults vs. children or different consumption levels. You can prompt Rork to add per-member weight ratios (like 1.0, 0.5) for flexible splitting.

**Receipt photo attachments**: Let users attach a photo of the receipt to each expense for easy reference later. This uses Expo's ImagePicker under the hood.

**Payment reminders**: Push notifications that remind members who haven't settled up yet can be a great quality-of-life addition.

**CSV export**: Allow users to export a group's expense data as a CSV file for use in Excel or Google Sheets — perfect for detailed record-keeping.

## FAQ

### Can I really build this without any coding experience?

Absolutely. Rork's AI handles all the code generation, so you don't need to know JavaScript, React Native, or any other programming language. Start by copying the prompt examples in this tutorial, and as you get comfortable, experiment with your own customizations.

### Can I share the app with friends?

Yes! You can share a preview link through Expo Go so anyone can try your app on their phone. If you want to distribute it more formally, you can publish it to the App Store or Google Play. The [Getting Started guide](/en/articles/rork-basics/getting-started) has more details on the publishing process.

### Does it work offline?

With AsyncStorage-based local persistence, the core bill splitting and recording features work without an internet connection. However, if you add cloud sync through a service like Supabase, that part will require connectivity.

### How do I add support for multiple currencies?

Send Rork a follow-up prompt like "Add a currency selector that supports USD, EUR, and JPY, and let each group choose its own currency." For extra utility, you could even integrate a live exchange rate API to handle conversions for international trips.
