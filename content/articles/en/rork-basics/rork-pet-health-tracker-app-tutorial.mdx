## Introduction — Build a Pet Health Tracker with Rork

If you're a pet owner, you've probably wished for a simple way to keep track of your pet's weight, feeding schedule, and vet appointments all in one place. With Rork, you can turn that idea into a working mobile app — no coding experience required.

In this tutorial, we'll walk through building a pet health tracker app from scratch using Rork AI. Along the way, you'll learn essential app development patterns including form handling, local data storage with AsyncStorage, and displaying records in a scrollable list.

**What you'll learn:**

- How to prompt Rork to generate screens for a pet management app
- Form input basics and validation techniques
- Saving data locally with AsyncStorage
- Displaying records with FlatList
- Working with date pickers and category selectors

**Who this is for:** Complete beginners to Rork or those who've just started exploring the platform.

## Prerequisites and Setup

Before we dive in, here's what you'll need:

- **A Rork account**: The free plan works fine for this project. You get up to 5 app generations per week
- **A modern browser**: Latest Chrome or Safari recommended
- **Basic Rork familiarity**: Knowing how to create a project and enter prompts will help, though we'll cover the essentials

If you haven't used Rork before, check out the [Getting Started with Rork guide](/en/articles/rork-basics/getting-started) first to get up to speed quickly.

## Step 1: Create Your Project

Log into Rork and hit the "New Project" button on your dashboard. Give it a clear name like "PetCare Tracker," then enter the following prompt to generate the app's foundation:

```text
Build a pet health management app with the following screens:
1. Home screen — display registered pets as cards in a grid
2. Add Pet screen — form with name, species (dog/cat/other), birthday, and weight
3. Health Log screen — add weight measurements, meal entries, and vet visit records with dates
4. Records screen — show all health records in chronological order

Use soft pastel colors and friendly pet-themed icons for the UI.
```

Once you submit this prompt, Rork AI will generate the initial structure with all four screens, complete with navigation between them.

## Step 2: Design Your Data Model

After reviewing the generated code, let's make sure our data structure is clean and well-organized. A pet health tracker deals with two main types of data:

```typescript
// Type definition for pet information
interface Pet {
  id: string;
  name: string;           // Pet's name
  species: 'dog' | 'cat' | 'other';  // Species
  birthday: string;       // Birthday (ISO format)
  weight: number;         // Weight in kg
  imageUri?: string;      // Profile photo (optional)
}

// Type definition for health records
interface HealthRecord {
  id: string;
  petId: string;          // Associated pet ID
  type: 'weight' | 'meal' | 'vet';   // Record type
  date: string;           // Record date
  value: string;          // Record content
  notes?: string;         // Additional notes (optional)
}
```

Tell Rork to "restructure the code to match this data model" and it will update the components accordingly.

## Step 3: Build the Input Form

The Add Pet screen needs proper validation to make sure users enter valid information. Send Rork this prompt to generate a production-ready form:

```text
Add validation to the pet registration form:
- Name: required, 1-20 characters
- Species: required, dropdown selection (dog/cat/other)
- Birthday: required, date picker (no future dates allowed)
- Weight: required, range 0.1-200kg, one decimal place

Show validation errors in red text below each input field.
```

By specifying exact rules in your prompt, Rork AI generates form components with meaningful validation logic rather than placeholder code.

## Step 4: Persist Data with AsyncStorage

To make sure your data survives app restarts, we'll use AsyncStorage for local persistence. Think of it like localStorage for mobile apps — a simple key-value store that keeps data on the device.

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

// Function to save pet list
const savePets = async (pets: Pet[]): Promise<void> => {
  try {
    const jsonValue = JSON.stringify(pets);
    await AsyncStorage.setItem('@petcare_pets', jsonValue);
  } catch (e) {
    console.error('Failed to save pets:', e);
  }
};

// Function to load pet list
const loadPets = async (): Promise<Pet[]> => {
  try {
    const jsonValue = await AsyncStorage.getItem('@petcare_pets');
    return jsonValue != null ? JSON.parse(jsonValue) : [];
  } catch (e) {
    console.error('Failed to load pets:', e);
    return [];
  }
};

// Usage: Load data when the app launches
const [pets, setPets] = useState<Pet[]>([]);

useEffect(() => {
  loadPets().then(setPets);
}, []);
```

**Expected behavior**: When the app launches, `loadPets` runs and loads any previously saved pet data into memory. Whenever a new pet is registered, `savePets` writes the updated list to storage so it persists between sessions.

## Step 5: Display Records in a List

To show health records in a readable, chronological format, we'll use React Native's FlatList component. Prompt Rork with these specifications:

```text
Improve the health records list screen:
- Use FlatList to display records sorted newest first
- Color-code each record type with a distinct icon:
  - Weight: blue, scale icon
  - Meals: green, fork icon
  - Vet visits: orange, hospital icon
- Each card shows the date and record details
- Support pull-to-refresh
```

FlatList is the go-to choice for scrollable lists in React Native because it only renders items that are currently visible on screen, keeping performance smooth even with hundreds of records.

## Common Issues and Fixes

### AsyncStorage data disappears

During development, clearing Expo's cache also wipes AsyncStorage data. A helpful workaround is to create a developer menu with a "Load Sample Data" button so you can quickly repopulate test data after a cache clear.

### Date picker not showing up

Date pickers behave differently on iOS and Android. When using `@react-native-community/datetimepicker`, tell Rork explicitly: "Use a cross-platform date picker that works on both iOS and Android" to avoid platform-specific rendering issues.

### Images not displaying

For pet profile photos, you'll need `expo-image-picker`. Prompt Rork with "Add the ability to select photos from the camera roll" and it will handle the permission configuration along with the image selection logic.

## FAQ

### Can I build this app on Rork's free plan?

Absolutely. The free plan gives you 5 generations per week, which is enough to build the core features. The key is writing detailed prompts that cover multiple requirements at once, so each generation produces more complete results.

### Can I store data in the cloud instead of locally?

This tutorial uses on-device storage (AsyncStorage) for simplicity, but Rork has built-in Supabase integration that makes cloud storage straightforward. You can migrate from local to cloud storage without rebuilding the app — just swap the storage functions with Supabase API calls.

### Does the app work on both iOS and Android?

Yes. Rork builds on React Native and Expo, so a single codebase produces apps for both platforms. Just keep in mind that some features like camera access and notifications may behave slightly differently across platforms, so test on both when possible.

### How do I publish the finished app?

You can use Expo's EAS Build to create production binaries and submit them to the App Store or Google Play. For a step-by-step publishing guide, check out [Build and publish your first Rork app in 30 minutes](/en/articles/rork-basics/rork-first-app-30-minutes-guide).
