## Building the Timer — Accurately Tracking Study Time

The timer is the heart of the app. When a user picks a subject and taps start, the clock begins counting.

```typescript
// hooks/useStudyTimer.ts
import { useState, useEffect, useRef, useCallback } from 'react';

export function useStudyTimer() {
  const [isRunning, setIsRunning] = useState(false);
  const [elapsed, setElapsed] = useState(0); // Seconds elapsed

  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);

  // Start the timer
  const start = useCallback(() => {
    setIsRunning(true);
  }, []);

  // Stop and return elapsed time
  const stop = useCallback(() => {
    setIsRunning(false);
    const finalElapsed = elapsed;
    setElapsed(0);
    return finalElapsed;
  }, [elapsed]);

  // Tick every second
  useEffect(() => {
    if (isRunning) {
      intervalRef.current = setInterval(() => {
        setElapsed((prev) => prev + 1);
      }, 1000);
    }
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
    };
  }, [isRunning]);

  // Format as "01:23:45"
  const formatTime = (seconds: number): string => {
    const h = Math.floor(seconds / 3600);
    const m = Math.floor((seconds % 3600) / 60);
    const s = seconds % 60;
    return `${String(h).padStart(2, '0')}:${String(m).padStart(2, '0')}:${String(s).padStart(2, '0')}`;
  };

  return {
    isRunning,
    elapsed,
    displayTime: formatTime(elapsed),
    start,
    stop,
  };
}
// Expected output:
// isRunning: true/false — whether the timer is active
// elapsed: 0, 1, 2, ... — increments every second
// displayTime: "00:00:00" → "00:01:30" → "01:00:00"
```

By encapsulating the logic in a custom hook called `useStudyTimer`, you can reuse it from any screen. The `useRef` pattern for storing the interval ID ensures proper cleanup when the component unmounts — a standard React Native best practice.

## Persisting Data — Saving Sessions with AsyncStorage

Study sessions measured by the timer need to survive app restarts. AsyncStorage provides a simple key-value store for local persistence.

```typescript
// utils/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import { StudySession } from '../types/study';

const SESSIONS_KEY = '@study_sessions';

// Save a new session
export async function saveSession(session: StudySession): Promise<void> {
  const existing = await getSessions();
  existing.push(session);
  await AsyncStorage.setItem(SESSIONS_KEY, JSON.stringify(existing));
}

// Retrieve all sessions
export async function getSessions(): Promise<StudySession[]> {
  const data = await AsyncStorage.getItem(SESSIONS_KEY);
  return data ? JSON.parse(data) : [];
}

// Retrieve sessions within a date range
export async function getSessionsByDateRange(
  startDate: Date,
  endDate: Date
): Promise<StudySession[]> {
  const sessions = await getSessions();
  return sessions.filter((s) => {
    const sessionDate = new Date(s.startTime);
    return sessionDate >= startDate && sessionDate <= endDate;
  });
}
// Expected output:
// getSessions() → [{ id: "abc123", subjectId: "math", startTime: "2026-03-26T10:00:00", duration: 1800, memo: "Calculus practice" }]
```

The `getSessionsByDateRange` function is key — you'll use it in the statistics screen to calculate totals for "this week" or "this month" without loading every record.

## Building the Timer Screen UI

Now let's wire everything together into a real screen. Use Rork's AI to generate the base layout, then refine it manually as needed.

```tsx
// screens/TimerScreen.tsx (simplified)
import React, { useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import { useStudyTimer } from '../hooks/useStudyTimer';
import { saveSession } from '../utils/storage';
import { DEFAULT_SUBJECTS, Subject } from '../types/study';

export default function TimerScreen() {
  const [selectedSubject, setSelectedSubject] = useState<Subject | null>(null);
  const { isRunning, displayTime, start, stop } = useStudyTimer();

  const handleStop = async () => {
    if (!selectedSubject) return;
    const duration = stop();
    // Ignore sessions shorter than 30 seconds
    if (duration < 30) return;

    await saveSession({
      id: Date.now().toString(),
      subjectId: selectedSubject.id,
      startTime: new Date(Date.now() - duration * 1000).toISOString(),
      duration,
      memo: '',
    });
  };

  return (
    <View style={styles.container}>
      {/* Subject picker */}
      <View style={styles.subjects}>
        {DEFAULT_SUBJECTS.map((subject) => (
          <TouchableOpacity
            key={subject.id}
            style={[
              styles.subjectButton,
              selectedSubject?.id === subject.id && { backgroundColor: subject.color },
            ]}
            onPress={() => setSelectedSubject(subject)}
          >
            <Text style={styles.subjectIcon}>{subject.icon}</Text>
            <Text style={styles.subjectName}>{subject.name}</Text>
          </TouchableOpacity>
        ))}
      </View>

      {/* Timer display */}
      <Text style={styles.timer}>{displayTime}</Text>

      {/* Start / Stop button */}
      <TouchableOpacity
        style={[styles.actionButton, isRunning ? styles.stopButton : styles.startButton]}
        onPress={isRunning ? handleStop : start}
        disabled={!selectedSubject}
      >
        <Text style={styles.actionText}>
          {isRunning ? 'Stop' : 'Start'}
        </Text>
      </TouchableOpacity>
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center', padding: 20 },
  subjects: { flexDirection: 'row', flexWrap: 'wrap', justifyContent: 'center', gap: 12, marginBottom: 40 },
  subjectButton: { alignItems: 'center', padding: 12, borderRadius: 12, backgroundColor: '#F0F0F0', width: 80 },
  subjectIcon: { fontSize: 28 },
  subjectName: { fontSize: 12, marginTop: 4, color: '#333' },
  timer: { fontSize: 64, fontWeight: '200', fontVariant: ['tabular-nums'], marginBottom: 40 },
  actionButton: { paddingHorizontal: 48, paddingVertical: 16, borderRadius: 32 },
  startButton: { backgroundColor: '#4A90D9' },
  stopButton: { backgroundColor: '#E74C3C' },
  actionText: { color: '#FFF', fontSize: 20, fontWeight: '600' },
});
```

Requiring subject selection before the timer starts prevents accidental untagged recordings. The 30-second minimum guard keeps your data clean by filtering out false starts.

## The Statistics Screen — Charting Progress with Victory Native

This is where the magic happens. By visualizing accumulated data, you turn raw numbers into motivation. Use [Victory Native](https://github.com/FormidableLabs/victory-native) to render pie charts for subject breakdown and bar charts for daily trends.

```typescript
// utils/statistics.ts
import { StudySession, DEFAULT_SUBJECTS } from '../types/study';

// Calculate total study time per subject
export function calcSubjectTotals(sessions: StudySession[]) {
  const totals: Record<string, number> = {};

  sessions.forEach((s) => {
    totals[s.subjectId] = (totals[s.subjectId] || 0) + s.duration;
  });

  return DEFAULT_SUBJECTS
    .filter((subj) => totals[subj.id] > 0)
    .map((subj) => ({
      x: subj.name,
      y: Math.round(totals[subj.id] / 60), // Convert to minutes
      color: subj.color,
    }));
}

// Calculate daily totals for the past 7 days
export function calcDailyTotals(sessions: StudySession[]) {
  const days: Record<string, number> = {};
  const now = new Date();

  for (let i = 6; i >= 0; i--) {
    const d = new Date(now);
    d.setDate(d.getDate() - i);
    const key = d.toLocaleDateString('en-US', { month: 'short', day: 'numeric' });
    days[key] = 0;
  }

  sessions.forEach((s) => {
    const d = new Date(s.startTime);
    const key = d.toLocaleDateString('en-US', { month: 'short', day: 'numeric' });
    if (key in days) {
      days[key] += s.duration;
    }
  });

  return Object.entries(days).map(([label, seconds]) => ({
    x: label,
    y: Math.round(seconds / 60), // Convert to minutes
  }));
}
// Expected output:
// calcSubjectTotals → [{ x: "Math", y: 90, color: "#4A90D9" }, { x: "English", y: 45, color: "#E8734A" }]
// calcDailyTotals → [{ x: "Mar 20", y: 120 }, { x: "Mar 21", y: 45 }, ...]
```

Feed this data into Victory Native's `VictoryPie` and `VictoryBar` components, and you've got an insightful dashboard. The bar chart shows daily trends — "I studied more yesterday than the day before" — while the pie chart reveals subject balance, helping you plan what to focus on next.

## Weekly Report — Reinforcing Study Habits

Adding a weekly summary that displays when the user opens the app helps reinforce consistent study habits.

```typescript
// utils/weeklyReport.ts
import { getSessionsByDateRange } from './storage';
import { DEFAULT_SUBJECTS } from '../types/study';

export async function generateWeeklyReport() {
  const now = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 24 * 60 * 60 * 1000);
  const sessions = await getSessionsByDateRange(weekAgo, now);

  const totalMinutes = Math.round(
    sessions.reduce((sum, s) => sum + s.duration, 0) / 60
  );

  const activeDays = new Set(
    sessions.map((s) => new Date(s.startTime).toDateString())
  ).size;

  const topSubject = sessions.reduce((acc, s) => {
    acc[s.subjectId] = (acc[s.subjectId] || 0) + s.duration;
    return acc;
  }, {} as Record<string, number>);

  const topId = Object.entries(topSubject).sort(([, a], [, b]) => b - a)[0]?.[0];
  const topName = DEFAULT_SUBJECTS.find((s) => s.id === topId)?.name || '—';

  return {
    totalMinutes,          // Total weekly minutes
    activeDays,            // Days with at least one session
    sessionCount: sessions.length,
    topSubject: topName,   // Most-studied subject
  };
}
// Expected output:
// { totalMinutes: 420, activeDays: 5, sessionCount: 12, topSubject: "Math" }
```

Display this report data in a card widget on the home screen, and users get an instant sense of accomplishment every time they open the app.

## Frequently Asked Questions

### Q1: Can I sync my study data to the cloud?

AsyncStorage is local-only, so switching devices means losing your data. To add cloud sync, check out [Build User Authentication in Rork — Firebase and Supabase Integration Guide](/articles/rork-dev/rork-auth-firebase-supabase) for connecting to Supabase or Firebase. The `StudySession` type can be directly mapped to a database table.

### Q2: Does this work on both iOS and Android?

Absolutely. Since Rork is built on React Native with Expo, all the code in this tutorial runs on both platforms. Timer accuracy and background behavior may vary slightly between operating systems, but foreground tracking works reliably on both iOS and Android.

### Q3: How do I let users add their own custom subjects?

Replace the hardcoded `DEFAULT_SUBJECTS` array with a dynamic list stored in AsyncStorage. Then create a settings screen where users can add, edit, or delete subjects — including custom names, colors, and icons.

### Q4: What makes Rork a good choice for building this kind of app?

Rork generates UI from prompts while giving you full access to the React Native source code. This means you can build custom hooks, write your own data aggregation logic, and fine-tune details that pure no-code tools can't handle. If you're just getting started, the [Getting Started with Rork guide](/articles/rork-basics/getting-started) covers the essentials.
