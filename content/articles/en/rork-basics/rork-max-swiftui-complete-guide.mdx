## Cloud Mac Builds Without Xcode: The 2-Click Publishing Process

Traditionally, you'd need to install Xcode (about 12GB) and use your local machine as a compile machine. Rork Max changes this entirely.

### Cloud Compilation Flow

1. **Prompt Input** → SwiftUI code generation (AI in browser)
2. **Cloud Mac Reservation** → Rork Max backend automatically provisions a virtual Mac
3. **Remote Build** → Xcode runs automatically on cloud Mac, IPA generated
4. **QR Code Download** → Scan QR code on your device to install the test app

### Cloud Mac in Detail

When you click the "Build" button in the Rork Max dashboard:

1. Rork Max backend spins up a macOS virtual machine (takes a few seconds on first run)
2. Latest Xcode automatically installs (a few minutes on first run)
3. Your SwiftUI project deploys to that virtual Mac
4. `xcodebuild` automatically executes
5. IPA (iOS App Package) file is generated
6. Dashboard displays download link and QR code

Scan the QR code with a registered developer iPhone, and your app installs in seconds.

## Common Errors and Fixes

Sometimes you'll encounter build errors with Rork Max. Here are the most common issues and solutions.

### Error 1: "Provisioning Profile not found"

**Cause**: Incomplete Apple Developer registration or expired certificates

**Solution**:
1. Log into Apple Developer account
2. Check "iOS Development" certificate in Certificates, Identifiers & Profiles
3. If valid for another year, OK. If expired, create a new one
4. In Rork Max settings, run "Re-sync Apple credentials"

### Error 2: "SwiftUI compilation failed"

**Cause**: Using outdated SwiftUI syntax or SDK version mismatch

**Solution**:
1. Rork Max dashboard → Settings → iOS SDK Version: Select iOS 17.0 or higher
2. In your prompt, explicitly add "Use latest SwiftUI 6 syntax"
3. Use the "Fix SwiftUI compiler errors" button for AI-assisted fixes

### Error 3: "Build timeout"

**Cause**: Temporary cloud Mac congestion or overly long compilation

**Solution**:
1. Wait a few minutes and click "Retry Build"
2. Remove unnecessary features and regenerate
3. Check you're not building Metal + Widget + ARKit simultaneously (not currently supported in combination)

### Error 4: "App Store rejection: Missing privacy manifest"

**Cause**: iOS 17.4+ requires privacy declaration (NSPrivacyTracking, etc.)

**Solution**:
1. Go to "Privacy Configuration" panel in Rork Max
2. Check required APIs: HealthKit usage → set NSHealthShareUsageDescription
3. Location usage → set NSLocationWhenInUseUsageDescription
4. Rork Max auto-generates Privacy Manifest (PrivacyInfo.xcprivacy) and includes it
