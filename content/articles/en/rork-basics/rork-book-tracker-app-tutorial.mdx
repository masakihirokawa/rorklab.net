## Build Your Own Personal Book Tracker with Rork

Ever wanted to keep track of the books you've read, but found existing apps too cluttered or missing the features you actually need? With Rork, you can build a fully customized book tracker app using nothing but natural language prompts — no programming experience required.

In this tutorial, we'll walk through creating a book tracker app from scratch. You'll be able to add books, track your reading status, rate what you've read, jot down notes, and even view reading statistics. The whole process takes about 30 to 45 minutes.

## What You'll Need Before Starting

Before diving in, make sure you have the following ready:

- **A Rork account**: Sign up for free at [rork.app](https://rork.app) — the Free plan is enough to get started
- **Basic familiarity with Rork**: You should know how to create a new project (if you're brand new, check out the "[Getting Started with Rork](/en/articles/rork-basics/getting-started)" guide first)

That's it. No coding skills, no command line tools, no complicated setup.

## Planning Your App's Features

Before writing any prompts, it helps to have a clear picture of what you're building. Here's the feature set for our book tracker:

**Core features:**
- Add books with title, author, and genre
- Track reading status: "Want to Read," "Reading," and "Finished"
- Rate books on a 5-star scale and add personal notes
- Filter your library by genre or reading status

**Screen layout:**
- Home screen (bookshelf list with status tabs)
- Add Book screen (input form)
- Book Detail screen (view and edit ratings, notes)
- Stats screen (reading count charts and genre breakdown)

## Step 1: Create a New Rork Project

Log in to Rork and tap "New Project" from your dashboard. Name it something like "BookTracker" and choose the "Blank" template. Starting from scratch helps you understand how each piece fits together.

## Step 2: Build the Home Screen with an AI Prompt

With your project open, type the following prompt into Rork's AI chat:

```text
Create a home screen for a book tracker app.

Requirements:
- A clean header at the top with the app name "My Book Shelf"
- Three tabs to filter books: "Want to Read," "Reading," and "Finished"
- Each tab displays book cards in a list (showing title, author, genre tag, and star rating)
- A floating action button (+) in the bottom-right corner to add a new book
- Tapping a card navigates to the book detail screen
- Use a warm, muted beige color palette
- Show an empty state illustration with "No books yet" when the list is empty
```

Rork will generate React Native components with tab navigation, card layouts, and proper styling — all from this single prompt.

### Tips for Better Prompts

The more specific you are, the better the results. Mention colors, layout preferences, and interaction details. Think of it like giving instructions to a designer: the clearer the brief, the closer the output matches your vision.

## Step 3: Create the Add Book Form

Next, let's build the screen where users can add new books to their library:

```text
Create an "Add Book" screen with a form.

Form fields:
- Book title (required, text input)
- Author name (required, text input)
- Genre (dropdown select: Fiction, Business, Self-Help, Tech, Essay, Other)
- Reading status (radio buttons: Want to Read / Reading / Finished)
- Star rating (tap 1–5 stars to select, defaults to unrated)
- Reading notes (multiline text, optional)
- Submit button

Validation:
- Title and author are required — show an error if empty
- On successful submission, navigate back to the home screen and add the book to the list

Match the design style from the home screen.
```

This prompt generates a complete form with input validation and navigation logic in one go.

## Step 4: Add the Book Detail and Edit Screen

Now let's create the screen that appears when a user taps on a book card:

```text
Create a book detail screen.

Display:
- Book title in a large font
- Author name
- Genre tag
- Current reading status (tappable to change)
- Star rating (tappable to update)
- Reading notes (tap to enter edit mode)
- Date added

Actions:
- An "Edit" button to switch all fields to edit mode
- A "Delete" button that shows a confirmation dialog before removing the book
- When the status changes to "Finished," automatically record today as the completion date
```

The automatic completion date is a small but powerful detail — it feeds directly into the stats screen we'll build next.

## Step 5: Build a Reading Stats Dashboard

A stats screen turns raw data into motivation. Here's the prompt:

```text
Create a reading statistics screen.

Stats to display:
- Books finished this month vs. a monthly goal (default: 5 books, configurable in settings)
- Total books finished (all time)
- Genre breakdown as a pie chart
- Monthly reading trend as a bar chart (last 6 months)
- Average star rating across all rated books

Design:
- Use colorful but muted chart colors
- Make numbers large and easy to read at a glance
```

Rork will leverage charting libraries like `react-native-chart-kit` to render these visualizations automatically.

## Step 6: Set Up Local Data Persistence

Without data persistence, everything disappears when you close the app. Let's fix that:

```text
Use AsyncStorage to persist book data locally.

Requirements:
- Automatically save whenever a book is added, edited, or deleted
- Automatically load all saved data when the app launches
- Store data as a JSON array
```

This ensures your reading log survives app restarts. If you want cloud sync later, you can connect Supabase or Firebase — see our "[Firebase and Supabase Backend Comparison](/en/articles/rork-dev/supabase-firebase-backend)" for details.

## Common Issues and How to Fix Them

### The AI-generated UI doesn't match your expectations

Don't restart from scratch. Instead, give targeted feedback like "Move the tabs to the top of the screen and make them rounded." Rork's AI works best with incremental refinement — small, specific corrections get you to the finish line faster than rewriting the entire prompt.

### Data isn't saving properly

Check that AsyncStorage is initialized correctly. You can ask the AI: "Help me debug why AsyncStorage isn't persisting data" — it'll generate diagnostic code and walk you through the fix.

### Navigation between screens isn't working

Rork uses Expo Router for navigation. Make sure the screen filenames and route names match up. If something seems off, ask the AI to "show the current navigation structure" so you can spot mismatches.

## Ideas for Extending Your App

Once the basics are working, consider adding these features:

- **ISBN barcode scanning**: Use the device camera to scan a book's barcode and auto-fill title and author via an external API
- **Reading timer**: Track how long you spend reading each session
- **Cover images**: Snap a photo of the book cover or paste an image URL
- **Social sharing**: Share your favorite reads on social media
- **Search and sort**: Find books by title or author, sort by rating or date added

Each of these can be added with a single prompt in Rork. Start with the basics and layer on new features as you go.

## FAQ

### Do I really need zero coding experience?

Yes. Every feature in this tutorial is built by typing prompts in plain English. If something needs tweaking, you can describe the change you want, and Rork's AI handles the code. It's like having a conversation with a developer who builds exactly what you describe.

### Can I build this on Rork's Free plan?

The Free plan allows up to 5 project generations or edits per week. You can complete the core features of this tutorial within that limit. If you plan to iterate frequently or add many enhancements, the Rork Max plan gives you unlimited access.

### Can I publish this app to the App Store or Google Play?

Absolutely. With Rork Max, you can deploy to TestFlight or submit to the App Store in just two clicks. For a complete walkthrough, check out the "[App Store Publishing Guide](/en/articles/rork-dev/app-store-publishing)."

### Can I sync my data across multiple devices?

This tutorial uses local storage (AsyncStorage), which keeps data on a single device. For cloud sync across devices, you can integrate Supabase or Firebase as a backend. Our "[Firebase and Supabase Backend Comparison](/en/articles/rork-dev/supabase-firebase-backend)" guide covers the setup process.
