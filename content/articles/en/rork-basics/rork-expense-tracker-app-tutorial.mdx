# Build an Expense Tracker App with Rork: Complete Development Tutorial

Want to track your spending but find most budgeting apps too bloated with features you'll never use? With **Rork**, you can build a custom expense tracker tailored to your exact needs using nothing but AI prompts. No coding experience required — you'll have a fully functional app with category sorting, visual charts, and persistent storage in about 30 minutes.

This tutorial walks you through building an expense tracker from scratch, complete with real prompt examples and code samples at each stage.

## What You'll Build

By the end of this tutorial, you'll have a working expense tracker app with:

- Income and expense recording with editing
- Automatic category classification (food, transport, entertainment, etc.)
- Monthly spending charts (pie charts and bar graphs)
- Monthly summary reports
- Local data persistence via AsyncStorage

This guide assumes you've used Rork at least once before. If you're brand new, start with the [Getting Started with Rork](/en/articles/rork-basics/getting-started) guide first.

## Step 1: Create the Project and Base UI

Log into Rork and create a new project. Your first prompt should define the entire app structure.

### Initial Prompt

```text
Build an expense tracker app with these screens:

1. Home screen: Show this month's total spending and a list of recent transactions
2. Add screen: Form with amount, category, notes, and date inputs
3. Reports screen: Monthly spending as pie chart and bar graph
4. Settings screen: Add and edit expense categories

Use a minimal design with green (#2E7D32) as the primary color.
Add tab navigation at the bottom.
```

This single prompt generates a four-screen app with tab navigation, layouts, and basic components. Rork handles the scaffolding so you start with a clean, structured UI from the beginning.

## Step 2: Design the Data Model and Store

The backbone of any expense tracker is how it manages data. Give Rork specific instructions about your data structure to generate a robust state management solution.

### Data Model Prompt

```text
Manage expense data with this structure:

- id: Unique ID (UUID)
- amount: Number
- category: One of "Food", "Transport", "Household", "Entertainment", "Medical", "Education", "Other"
- note: Optional memo
- date: ISO date string
- type: "income" or "expense"

Create a Zustand store with AsyncStorage persistence.
Include helper functions for monthly filtering and category-based aggregation.
```

### Generated Store Code

Here's what Rork generates for the store. Comments explain what each function does.

```typescript
// stores/expenseStore.ts
import { create } from 'zustand';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface Transaction {
  id: string;
  amount: number;
  category: string;
  note: string;
  date: string;
  type: 'income' | 'expense';
}

interface ExpenseStore {
  transactions: Transaction[];
  addTransaction: (tx: Omit<Transaction, 'id'>) => void;
  getMonthlyTotal: (year: number, month: number) => number;
  getCategoryBreakdown: (year: number, month: number) => Record<string, number>;
}

// Create the Zustand store
export const useExpenseStore = create<ExpenseStore>((set, get) => ({
  transactions: [],

  // Add a new transaction and persist to AsyncStorage
  addTransaction: (tx) => {
    const newTx = { ...tx, id: crypto.randomUUID() };
    set((state) => {
      const updated = [...state.transactions, newTx];
      AsyncStorage.setItem('transactions', JSON.stringify(updated));
      return { transactions: updated };
    });
  },

  // Calculate total expenses for a given month
  getMonthlyTotal: (year, month) => {
    return get()
      .transactions.filter(
        (tx) =>
          tx.type === 'expense' &&
          new Date(tx.date).getFullYear() === year &&
          new Date(tx.date).getMonth() === month
      )
      .reduce((sum, tx) => sum + tx.amount, 0);
  },

  // Get expense breakdown by category for a given month
  getCategoryBreakdown: (year, month) => {
    const filtered = get().transactions.filter(
      (tx) =>
        tx.type === 'expense' &&
        new Date(tx.date).getFullYear() === year &&
        new Date(tx.date).getMonth() === month
    );
    return filtered.reduce(
      (acc, tx) => {
        acc[tx.category] = (acc[tx.category] || 0) + tx.amount;
        return acc;
      },
      {} as Record<string, number>
    );
  },
}));

// Expected output:
// getMonthlyTotal(2026, 2) → 85000 (total March expenses)
// getCategoryBreakdown(2026, 2) → { "Food": 35000, "Transport": 12000, "Entertainment": 18000, ... }
```

The auto-generated form covers the basics, but a few targeted prompts will significantly improve the user experience.

### UI Refinement Prompt


## Step 4: Build the Chart and Report Screen

The most rewarding part of an expense tracker is seeing your data visualized. Add these prompts for the report screen.

### Report Screen Prompt

```text
Build the Reports screen with these sections:

Top: Month selector (previous/next buttons with a "March 2026" label)

Section 1 - Summary cards:
- Total expenses this month
- Total income this month
- Net balance (green if positive, red if negative)

Section 2 - Pie chart:
- Category breakdown with fixed colors (Food: orange, Transport: blue, Entertainment: purple, etc.)
- Tap a segment to show category name and amount tooltip

Section 3 - Bar chart:
- Monthly expense trend for the last 6 months
- Auto-scaling Y axis

Use react-native-chart-kit for all charts.
```

### Generated Chart Component

```tsx
// components/MonthlyPieChart.tsx
import { PieChart } from 'react-native-chart-kit';
import { Dimensions } from 'react-native';

const screenWidth = Dimensions.get('window').width;

// Fixed color mapping per category
const CATEGORY_COLORS: Record<string, string> = {
  Food: '#FF9800',
  Transport: '#2196F3',
  Household: '#4CAF50',
  Entertainment: '#9C27B0',
  Medical: '#F44336',
  Education: '#00BCD4',
  Other: '#607D8B',
};

interface Props {
  breakdown: Record<string, number>;
}

export function MonthlyPieChart({ breakdown }: Props) {
  const data = Object.entries(breakdown).map(([name, amount]) => ({
    name,
    amount,
    color: CATEGORY_COLORS[name] || '#999',
    legendFontColor: '#333',
    legendFontSize: 12,
  }));

  return (
    <PieChart
      data={data}
      width={screenWidth - 32}
      height={220}
      chartConfig={{
        color: (opacity = 1) => `rgba(0, 0, 0, ${opacity})`,
      }}
      accessor="amount"
      backgroundColor="transparent"
      paddingLeft="15"
    />
  );
}

// Expected output: A color-coded pie chart where the largest
// expense category (e.g., Food in orange) takes the biggest segment
```

## Common Errors and Fixes

Here are the most frequent issues you'll encounter when building an expense tracker and how to resolve them.

**Charts not rendering**: The `react-native-chart-kit` package may not be installed. Add a follow-up prompt: "Install react-native-chart-kit and display the chart."

**Currency formatting issues**: `toLocaleString('en-US')` can behave inconsistently across environments. Ask Rork to generate a custom formatting function instead.

**AsyncStorage data disappearing**: In Expo development mode, clearing the cache also wipes AsyncStorage. This won't happen in production builds. For long-term reliability, consider migrating to a cloud database like Supabase. See the Supabase Chat App tutorial for implementation details.

## FAQ

### Can I build an expense tracker on Rork's free plan?

Yes, the free plan supports building a basic expense tracker. However, the free tier limits the number of prompts per week, so adding advanced features may require upgrading to a paid plan.

### Can I use a different charting library?

Absolutely. Rork supports `victory-native`, `react-native-svg-charts`, and others in addition to `react-native-chart-kit`. Just specify the library name in your prompt and Rork will generate compatible code.

### How do I add user authentication to my expense tracker?

Prompt Rork with something like "Add email authentication using Firebase Auth" and it will generate the complete login flow. Social login with Google and Apple is also supported. For a detailed walkthrough, see the [Authentication Guide: Firebase & Supabase](/en/articles/rork-dev/rork-auth-firebase-supabase).

### Does this work on both iPhone and Android?

Yes. Rork generates React Native (Expo) apps that run on both iOS and Android. If you want to target iOS specifically with native Swift, Rork Max offers that option as well.
