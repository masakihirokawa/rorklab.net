## Code Snippet of the Week

From the monetization guide, here's the minimum AdMob banner setup for a Rork app via Expo:

```typescript
import { BannerAd, BannerAdSize, TestIds } from "react-native-google-mobile-ads";

const adUnitId = __DEV__
  ? TestIds.ADAPTIVE_BANNER
  : "ca-app-pub-YOUR_ADMOB_ID/YOUR_AD_UNIT_ID";

export function AdBanner() {
  return (
    <BannerAd
      unitId={adUnitId}
      size={BannerAdSize.ANCHORED_ADAPTIVE_BANNER}
      requestOptions={{
        requestNonPersonalizedAdsOnly: true,
      }}
    />
  );
}
```

Using `TestIds.ADAPTIVE_BANNER` during development keeps test impressions out of your live AdMob account. `requestNonPersonalizedAdsOnly: true` is the safe default for users who haven't granted ATT (App Tracking Transparency) permission.

## FAQ

**Q: Is it complicated to add AdMob to a Rork app?**

A: AdMob doesn't work in Expo Go, so you'll need a development build (expo-dev-client) set up first. Once that's done, the actual implementation is placing a component — it's straightforward. The setup is the only friction point.

**Q: What's the difference between Rork and Rork Max?**

A: Rork Max adds capabilities needed for production-quality commercial apps: complex UI patterns, native module integration, and advanced debugging tools. If you're actively working toward monetization, it's worth evaluating the upgrade.
