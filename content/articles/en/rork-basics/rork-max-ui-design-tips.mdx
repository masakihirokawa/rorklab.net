# Rork Max UI Design Tips — 10 Rules for Beautiful Apps

Great UI design isn't magic—it's following proven principles consistently. This guide shares 10 actionable rules used by professional designers, with code examples you can apply immediately.

## Rule 1: The 60-30-10 Color Principle

Professional UIs follow a proven color ratio that creates harmony and visual hierarchy.

- **60%**: Primary color (background, main brand)
- **30%**: Secondary color (UI elements, cards, text)
- **10%**: Accent color (buttons, highlights, CTAs)

**src/styles/colors.ts**
```typescript
export const lightTheme = {
  // 60% — Primary (Background)
  primary: '#FFFFFF',
  primaryLight: '#F9F9F9',
  primaryDark: '#F5F5F5',

  // 30% — Secondary (Text, Cards)
  secondary: '#666666',
  secondaryLight: '#999999',
  secondaryLighter: '#CCCCCC',

  // 10% — Accent (CTAs, Highlights)
  accent: '#007AFF',
  accentDark: '#0051D5',
  accentLight: '#40B0FF',

  // Semantic
  success: '#34C759',
  warning: '#FF9500',
  error: '#FF3B30',
  info: '#00C7E8',
};

export const darkTheme = {
  primary: '#1C1C1E',
  primaryLight: '#2C2C2E',
  secondary: '#8E8E93',
  accent: '#0A84FF',
  success: '#30B0C0',
  warning: '#FF9500',
  error: '#FF453A',
};
```

**src/screens/Home.tsx**
```typescript
import { lightTheme } from '../styles/colors';
import { View, Text, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  container: {
    flex: 1,

    padding: 16,
  },

    backgroundColor: lightTheme.primaryDark,

    borderRadius: 12,
    marginBottom: 12,
  },
  button: {
    backgroundColor: lightTheme.accent,
    paddingVertical: 12,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonText: {
    color: '#fff',
    fontWeight: '600',
  },
});

export default function Home() {
  return (
    <View style={styles.container}>
      <View style={styles.card}>
        <Text style={{ color: lightTheme.secondary }}>
          Secondary text
        </Text>
      </View>
    </View>
  );
}
```

<Callout type="info">
**Pro Tip**: Test your color palette with colorblind-friendly tools. Websites like ColorOracle simulate different types of color blindness to ensure your design is accessible.
</Callout>

## Rule 2: Typography Hierarchy

Clear size progression makes content scannable and establishes visual hierarchy.

**src/styles/typography.ts**
```typescript
import { StyleSheet } from 'react-native';

  // Large heading — section titles

    fontSize: 32,

    lineHeight: 40,

  // Medium heading — card titles

  // Small heading

  // Body text — standard content
  body: {
    fontSize: 16,
    fontWeight: '400',
```

## Rule 3: Spacing & Padding Consistency

Use an 8px grid system. Consistency creates rhythm and balance.

**src/styles/spacing.ts**
```typescript
export const spacing = {
  xs: 4,      // Micro adjustments
  sm: 8,      // Tightly spaced elements
  md: 12,     // Default padding
  lg: 16,     // Standard card padding
  xl: 24,     // Section spacing
  xxl: 32,    // Large containers
  xxxl: 48,   // Screen-level padding
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: spacing.lg,      // 16px
  },
  section: {
    marginBottom: spacing.xl,  // 24px
  },
  card: {
    padding: spacing.lg,       // 16px
    marginBottom: spacing.md,  // 12px
  },
  buttonGroup: {
    gap: spacing.sm,           // 8px between buttons
  },
});
```

| Component | Padding | Margin Bottom |
|-----------|---------|---------------|
| Card | 16px | 12px |
| Button | 12px V, 16px H | 8px |
| List Item | 16px | 8px |
| Section | 24px | 24px |

## Rule 4: Minimum Touch Target — 44×44px

Apple and Google both recommend minimum 44×44pt touch targets for accessibility.

**src/components/Button.tsx**
```typescript
import { TouchableOpacity, Text, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  button: {
    minHeight: 44,
    minWidth: 44,
    paddingVertical: 12,
    paddingHorizontal: 16,
    borderRadius: 8,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#007AFF',
  },
  buttonText: {
    color: '#fff',
    fontSize: 16,
    fontWeight: '600',
  },
});

export default function Button({ title, onPress }) {
  return (
    <TouchableOpacity
      style={styles.button}
      onPress={onPress}
      activeOpacity={0.8}
      // hitSlop expands touch area without affecting layout
      hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}
    >
      <Text style={styles.buttonText}>{title}</Text>
    </TouchableOpacity>
  );
}
```

## Rule 5: Purposeful Animation

Animation should inform, not distract. Every animation communicates state change.

**src/components/FadeInView.tsx**
```typescript
import { useEffect, useRef } from 'react';
import { Animated, View } from 'react-native';

export default function FadeInView({ children, duration = 400 }) {
  const fadeAnim = useRef(new Animated.Value(0)).current;

  useEffect(() => {
    Animated.timing(fadeAnim, {
      toValue: 1,
      duration,
      useNativeDriver: true,
    }).start();
  }, [fadeAnim, duration]);

  return (
    <Animated.View style={{ opacity: fadeAnim }}>
      {children}
    </Animated.View>
  );
}
```

**Scale Animation on Press**
```typescript
import { Animated, TouchableOpacity } from 'react-native';
import { useRef } from 'react';

export default function ScalableButton({ onPress, children }) {
  const scaleAnim = useRef(new Animated.Value(1)).current;

  const handlePressIn = () => {
    Animated.spring(scaleAnim, {
      toValue: 0.95,
      useNativeDriver: true,
    }).start();
  };

  const handlePressOut = () => {
    Animated.spring(scaleAnim, {
      toValue: 1,
      useNativeDriver: true,
    }).start();
  };

  return (
    <TouchableOpacity
      onPressIn={handlePressIn}
      onPressOut={handlePressOut}
      onPress={onPress}
    >
      <Animated.View style={{ transform: [{ scale: scaleAnim }] }}>
        {children}
      </Animated.View>
    </TouchableOpacity>
  );
}
```

<Callout type="info">
**Animation Timing**:
- Page transitions: 200–300ms
- Icon interactions: 200ms
- Button feedback: 100–150ms
- Always use `useNativeDriver={true}` for performance
</Callout>

## Rule 6: Dark Mode Support

Dark mode is standard on iOS and Android. Detect and adapt automatically.

**src/hooks/useTheme.ts**
```typescript
import { useColorScheme } from 'react-native';
import { lightTheme, darkTheme } from '../styles/colors';

export function useTheme() {
  const colorScheme = useColorScheme();
  return colorScheme === 'dark' ? darkTheme : lightTheme;
}
```

**src/screens/Profile.tsx**
```typescript
import { useTheme } from '../hooks/useTheme';
import { View, Text } from 'react-native';

export default function Profile() {
  const theme = useTheme();

  return (
    <View style={{ backgroundColor: theme.primary, flex: 1, padding: 16 }}>
      <Text style={{ color: theme.secondary }}>User Profile</Text>
    </View>
  );
}
```

**Navigation Theme**
```typescript
import { useColorScheme } from 'react-native';
import {
  NavigationContainer,
  DarkTheme,
  DefaultTheme,
} from '@react-navigation/native';

export default function App() {

  return (
    <NavigationContainer
      theme={colorScheme === 'dark' ? DarkTheme : DefaultTheme}

```

## Rule 7: Icon Consistency

Pick one icon library and stick with it. Material Icons or SF Symbols work well.

**src/components/Icon.tsx**
```typescript
import { Platform } from 'react-native';
import MaterialIcons from '@expo/vector-icons/MaterialIcons';
import Ionicons from '@expo/vector-icons/Ionicons';

interface IconProps {
  name: string;
  size?: number;
  color?: string;
}

export function Icon({ name, size = 24, color = '#000' }: IconProps) {
  // Use Ionicons on iOS, Material on Android
  if (Platform.OS === 'ios') {
    return <Ionicons name={name} size={size} color={color} />;
  }

  return <MaterialIcons name={name} size={size} color={color} />;
}
```

## Rule 8: State Representations

Every screen needs three states: Loading, Empty, and Error.

**src/components/StateViews.tsx**
```typescript
import { View, Text, TouchableOpacity, ActivityIndicator } from 'react-native';
import { useTheme } from '../hooks/useTheme';

export function LoadingView() {
  const theme = useTheme();
  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      <ActivityIndicator size="large" color={theme.accent} />
      <Text style={{ marginTop: 12, color: theme.secondary }}>
        Loading...
      </Text>
    </View>
  );
}

export function EmptyView({ title, message }: { title: string; message: string }) {
  const theme = useTheme();
  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center', padding: 32 }}>
      <Text style={{ fontSize: 18, fontWeight: '600', color: theme.secondary }}>
        {title}
      </Text>
      <Text
        style={{
          fontSize: 14,
          color: theme.secondaryLight,
          marginTop: 8,
          textAlign: 'center',
        }}
      >
        {message}
      </Text>
    </View>
  );
}

export function ErrorView({
  title,
  message,
  onRetry,
}: {
  title: string;
  message: string;
  onRetry: () => void;
}) {
  const theme = useTheme();
  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center', padding: 32 }}>
      <Text style={{ fontSize: 18, fontWeight: '600', color: theme.error }}>
        {title}
      </Text>
      <Text style={{ color: theme.secondary, marginTop: 8, textAlign: 'center' }}>
        {message}
      </Text>
      <TouchableOpacity
        style={{
          marginTop: 24,
          paddingVertical: 10,
          paddingHorizontal: 20,
          backgroundColor: theme.accent,
          borderRadius: 8,
        }}
        onPress={onRetry}
      >
        <Text style={{ color: '#fff', fontWeight: '600' }}>Retry</Text>
      </TouchableOpacity>
    </View>
  );
}
```

## Rule 9: List Performance with FlatList

Never render 1000 items at once. Use FlatList with optimizations.

```typescript
import { FlatList, View, Text } from 'react-native';

const data = Array.from({ length: 10000 }, (_, i) => ({
  id: String(i),
  title: `Item ${i + 1}`,
}));

export default function ItemList() {
  return (
    <FlatList
      data={data}
      renderItem={({ item }) => (
        <View style={{ padding: 12, borderBottomWidth: 1, borderColor: '#eee' }}>
          <Text>{item.title}</Text>
        </View>
      )}
      keyExtractor={(item) => item.id}
      // Performance optimizations
      maxToRenderPerBatch={10}
      updateCellsBatchingPeriod={50}
      removeClippedSubviews
      initialNumToRender={20}
    />
  );
}
```

## Rule 10: Accessibility

Design for everyone. Screen readers, color blindness, and motor difficulties matter.

**src/components/AccessibleButton.tsx**
```typescript
import { TouchableOpacity, Text } from 'react-native';

export default function AccessibleButton({
  title,
  onPress,
}: {
  title: string;
  onPress: () => void;
}) {
  return (
    <TouchableOpacity
      onPress={onPress}
      accessible
      accessibilityLabel={title}
      accessibilityHint="Double-tap to activate"
      accessibilityRole="button"
      accessibilityState={{ enabled: true }}
    >
      <Text
        style={{
          fontSize: 16,
          fontWeight: '600',
          color: '#007AFF',
        }}
      >
        {title}
      </Text>
    </TouchableOpacity>
  );
}
```
