## Introduction: Build an AI Study App with Rork

Language learning, exam prep, professional certifications — flashcard apps have been a staple of mobile learning for years. Apps like Anki and Quizlet serve millions of users worldwide, and there's still plenty of room for indie developers to carve out a niche with a focused, well-designed study app.

In this guide, you'll learn how to build an AI-powered flashcard app from scratch using **Rork** — no prior coding knowledge required. By chatting with Rork's AI, you'll generate a fully functional React Native app that works on both iOS and Android.

By the end of this tutorial, you'll be able to:

- Create the core structure of a flashcard app in Rork
- Implement card add, edit, and delete functionality
- Build a quiz mode with correct/incorrect feedback
- Save study progress locally on-device
- Use AI prompts to auto-generate flashcard decks

## Step 3: Implement the Card Flip Animation

The card flip animation is the centerpiece of any flashcard app. Prompt Rork like this:

```
Implement a card flip animation:
- Tapping the card rotates it 180 degrees horizontally
- Front face: question text on blue background
- Back face: answer text on green background
- Use the Animated API and optimize for performance with useNativeDriver
```

Here's an example of what Rork generates:

```typescript
// FlashCard.tsx — Card flip component
import React, { useState, useRef } from 'react';
import { View, Text, TouchableOpacity, Animated, StyleSheet } from 'react-native';

interface FlashCardProps {
  front: string;  // Question text
  back: string;   // Answer text
}

export const FlashCard: React.FC<FlashCardProps> = ({ front, back }) => {
  const [isFlipped, setIsFlipped] = useState(false);
  const flipAnim = useRef(new Animated.Value(0)).current;

  const flipCard = () => {
    Animated.spring(flipAnim, {
      toValue: isFlipped ? 0 : 1,
      friction: 8,
      tension: 10,
      useNativeDriver: true,
    }).start();
    setIsFlipped(!isFlipped);
  };

  const frontInterpolate = flipAnim.interpolate({
    inputRange: [0, 1],
    outputRange: ['0deg', '180deg'],
  });
  const backInterpolate = flipAnim.interpolate({
    inputRange: [0, 1],
    outputRange: ['180deg', '360deg'],
  });

  return (
    <TouchableOpacity onPress={flipCard} activeOpacity={1}>
      {/* Front face */}
      <Animated.View style={[styles.card, styles.front, { transform: [{ rotateY: frontInterpolate }] }]}>
        <Text style={styles.cardText}>{front}</Text>
        <Text style={styles.hint}>Tap to flip</Text>
      </Animated.View>
      {/* Back face */}
      <Animated.View style={[styles.card, styles.back, { transform: [{ rotateY: backInterpolate }] }]}>
        <Text style={styles.cardText}>{back}</Text>
      </Animated.View>
    </TouchableOpacity>
  );
};

const styles = StyleSheet.create({
  card: {
    width: 320,
    height: 200,
    borderRadius: 16,
    justifyContent: 'center',
    alignItems: 'center',
    padding: 24,
    backfaceVisibility: 'hidden',
    position: 'absolute',
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 4 },
    shadowOpacity: 0.15,
    shadowRadius: 8,
    elevation: 5,
  },
  front: { backgroundColor: '#3B82F6' },  // Blue: question side
  back:  { backgroundColor: '#10B981' },  // Green: answer side
  cardText: { fontSize: 22, fontWeight: '700', color: '#fff', textAlign: 'center' },
  hint:     { fontSize: 12, color: 'rgba(255,255,255,0.7)', marginTop: 12 },
});

// Expected output:
// Front (blue): displays "apple"
// Tap → 180° flip animation
// Back (green): displays "りんご"
```

## Step 4: Save Study Progress Locally

Use AsyncStorage to persist study records (correct count, incorrect count, last studied date) on-device. Ask Rork:

```
"Add AsyncStorage to save and load each card's learning record:
correct count, incorrect count, and last studied date."
```

```typescript
// storage.ts — Save and load study progress
import AsyncStorage from '@react-native-async-storage/async-storage';

interface CardProgress {
  cardId: string;
  correct: number;
  incorrect: number;
  lastStudied: string; // ISO 8601
}

export const saveCardProgress = async (progress: CardProgress): Promise<void> => {
  const key = `progress_${progress.cardId}`;
  await AsyncStorage.setItem(key, JSON.stringify(progress));
};

export const loadAllProgress = async (): Promise<Record<string, CardProgress>> => {
  const keys = await AsyncStorage.getAllKeys();
  const progressKeys = keys.filter(k => k.startsWith('progress_'));
  const pairs = await AsyncStorage.multiGet(progressKeys);

  return pairs.reduce((acc, [key, value]) => {
    if (value) {
      const cardId = key.replace('progress_', '');
      acc[cardId] = JSON.parse(value);
    }
    return acc;
  }, {} as Record<string, CardProgress>);
};

export const calculateAccuracy = (correct: number, incorrect: number): number => {
  const total = correct + incorrect;
  if (total === 0) return 0;
  return Math.round((correct / total) * 100); // e.g. 8/10 → 80
};
```

## Step 5: AI-Powered Card Generation

One of Rork's greatest strengths is AI integration. You can add a feature that auto-generates flashcard decks from a topic the user types in:

```
"Add an AI card generation feature using the OpenAI API.
When a user enters a topic (e.g., 'English prepositions'), generate 10 flashcards
automatically. The response should be a JSON array: [{front: '...', back: '...'}]"
```

```typescript
// autoGenerateCards.ts — Generate flashcards with OpenAI
const OPENAI_API_KEY = process.env.EXPO_PUBLIC_OPENAI_API_KEY || '';

interface GeneratedCard {
  front: string;
  back: string;
}

export const generateFlashcards = async (topic: string): Promise<GeneratedCard[]> => {
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${OPENAI_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [
        {
          role: 'system',
          content: 'You are a helpful tutor. Generate flashcards as a JSON array: [{front: "question", back: "answer"}]',
        },
        {
          role: 'user',
          content: `Generate 10 flashcards about: "${topic}". Return only valid JSON.`,
        },
      ],
      temperature: 0.7,
    }),
  });

  const data = await response.json();
  const content = data.choices[0].message.content;

  const jsonMatch = content.match(/\[[\s\S]*\]/);
  return jsonMatch ? JSON.parse(jsonMatch[0]) : [];
};

// Expected output (topic: "English prepositions"):
// [
//   { front: "at", back: "indicates a specific point in time or place" },
//   { front: "in", back: "inside a space or during a period" },
//   ...
// ]
```

## Step 6: Common Errors and How to Fix Them

### Error 1: Cards won't flip correctly

**Cause:** `backfaceVisibility: 'hidden'` not rendering as expected on Android.
**Fix:** Tell Rork to "add `overflow: 'hidden'` to the card container and test on both iOS and Android simulators."

### Error 2: AsyncStorage data disappears

**Cause:** Data is wiped when Expo Go cache is cleared or the app is reinstalled.
**Fix:** For production, sync data to Supabase or Firebase so it persists across installs.

### Error 3: OpenAI API calls are blocked

**Cause:** API key not set or CORS issue.
**Fix:** Verify `EXPO_PUBLIC_OPENAI_API_KEY` is defined in your `.env` file. For production, route API calls through a backend (e.g., Cloudflare Workers) to keep the key secure.
