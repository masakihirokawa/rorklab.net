## Getting Started: From Zero to First App in 30 Minutes

### Step 1: Create Your Rork Account

Visit rork.ai and hit "Get Started." You'll see options to sign up via email, Google, or GitHub. Pick whatever you use most frequently. Verify your email, and you're in.

You'll land on your dashboard—this is your control center for all things app development.

### Step 2: Create Your First Project

Click "+ New Project." You'll be asked for:

- **Project name** (e.g., "WeatherApp", "TaskMaster", "DeliveryTracker")
- **Description** (optional but helpful for you to remember what it does)
- **Template choice**
  - Blank (recommended for beginners)
  - React Native template
  - Navigation template

Go with Blank. It's the most flexible.

### Step 3: Write a Killer Prompt

This is where the magic happens. **The quality of your prompt directly determines the quality of your app.**

A weak prompt:
```
Create a weather app.
```

A strong prompt:
```
I want to build a weather forecast application with these features:

Core Features:
- Auto-detect user's current location and display weather
- Search for weather by city name
- Show current temperature, "feels like" temperature, humidity, wind speed
- Display 5-day forecast with daily highs and lows
- Toggle between Celsius and Fahrenheit

Design Requirements:
- Background color changes based on weather (blue for sunny, gray for cloudy, dark blue for rain)
- Large, prominent weather icon in the center
- Cards to organize detailed information
- Bottom navigation: Current Weather | Forecast | Settings
- Clean, modern aesthetic with good readability

Technical:
- Use OpenWeatherMap API for weather data
- Save favorite cities using local storage
- Handle network errors gracefully
- Responsive design that works on all phone sizes
```

The second prompt gives Rork everything it needs. The AI understands context, intention, and design philosophy. It will generate not just code, but thoughtfully-designed code.

## Refining Your App: The Iterative Loop

Rork uses a **conversational interface**. You don't need to understand the code; you just describe what you want changed.

"Make the weather icon 50% larger."
"Change the primary color to a deeper blue."
"Add the current air quality index below the temperature."
"Let users save multiple favorite cities and switch between them."

Rork reads your request, modifies the relevant files, and updates the preview. You can iterate like this dozens of times. Most developers refine their app through 3-5 rounds of feedback before they're happy.

### When You Need to Dive into Code

For 95% of use cases, Rork's visual interface is enough. But if you want to tweak something specific—add a custom animation, adjust an API request, fine-tune logic—you can. Click the "Code" tab and you're looking at TypeScript and React Native code:

```typescript
// services/weatherApi.ts
const API_KEY = 'YOUR_KEY';
const BASE_URL = 'https://api.openweathermap.org/data/2.5';

export const getWeatherByCity = async (city: string) => {
  const url = `${BASE_URL}/weather?q=${city}&appid=${API_KEY}&units=metric`;
  const response = await fetch(url);
  if (!response.ok) throw new Error('Failed to fetch weather');
  return response.json();
};
```

The code is clean, well-structured, and follows React Native best practices. You can edit directly if you want.

## Testing Your App: Making Sure It Actually Works

### Simulator Testing

Rork includes built-in iOS (Safari-based) and Android emulators. Tap buttons. Try different cities. Trigger error states. See how your app behaves. The preview updates in real-time as you make code changes.

This is how you catch issues before users see them.

### Common Issues and Quick Fixes

| Problem | Cause | Solution |
|---------|-------|----------|
| API returns 401 error | Invalid API key or not yet activated | Check your API key format. Wait 2 hours after creation. |
| Location won't turn on | Simulator doesn't have location set | Go to simulator settings, manually set location. |
| Image not showing | URL is broken or domain is blocked | Verify URL works in browser. Upload image directly if needed. |
| Blank white screen | React component error | Check the console tab for error messages. |

### Real Device Testing (Don't Skip This)

Simulators lie sometimes. Processing power is different on real devices. Touch responsiveness feels different. Network connectivity is real.

Rork uses Expo, which means you have two options:

**Option 1: Expo Go (Easiest)**
- Install "Expo Go" app on your iPhone or Android phone
- Rork generates a QR code
- Scan it. Your app installs and runs on your real phone.
- Any code changes in Rork instantly refresh on your phone.

**Option 2: TestFlight/Beta**
- Rork builds a production version
- You distribute via TestFlight (iOS) or Google Play Beta
- Get feedback from real testers before full launch

## Publishing to App Stores: The Final Mile

### What You Need

| Platform | Cost | Valid For | Requirements |
|----------|------|-----------|--------------|
| Apple Developer | $99/year | 1 year | Apple ID, credit card, identity verification |
| Google Play | $25 | Forever | Google account, credit card |

Apple's process is more involved (identity verification, business registration). Start that at least 3 weeks before you want to launch.

### The Publishing Process (Rork Pro & Max)

If you're on Rork Pro or Max, here's your flow:

**1. Prepare Store Listing**
- App name (max 30 characters)
- Description (max 4,000 characters)
- 2-8 screenshots (1170 × 2532 px)
- Privacy policy URL
- Support email

**2. Rork Builds Your App**
- Automatically generates iOS (.ipa) and Android (.aab) files
- Takes 5-10 minutes
- Downloads ready to submit

**3. Submit to Stores**
- Rork Max: One-click auto-submit
- Pro: Upload to Apple and Google consoles yourself (takes 10 minutes)

Apple typically approves in 1-3 days. Google usually responds within 4 hours.

### What Gets Apps Rejected (and How to Avoid It)

- **Missing privacy policy** — Required if your app asks for location, contacts, camera, etc.
- **Copied apps** — Don't try to submit a direct clone of a famous app
- **Misleading claims** — If you say "free," but it's got paid features, Apple will reject it
- **Low quality** — Apps with obvious bugs, confusing UI, or poor design get rejected
- **Copyright issues** — Using copyrighted images, music, or brand names without permission

Most rejections are avoidable. Follow the guidelines, test thoroughly, and be honest in your app description.

## Frequently Asked Questions

### Can apps built with Rork really get approved by Apple and Google?

Yes. Rork generates code that meets all official guidelines. Many developers have already shipped Rork-built apps to real users. What matters is your app's content and functionality—the platform doesn't matter.

### What if I want to switch to native development later?

You can export your entire codebase from Rork. It's real React Native code. You can take it anywhere and maintain it yourself. But you'd lose Rork's super-fast iteration and AI assistance. Not recommended unless you have a specific reason.

### Can I build multiple apps with Rork Pro?

Absolutely. Pro is designed for prolific developers. You get unlimited generation, so you could ship 5 or 10 apps if you wanted. Each one builds on what you've learned.

### How fast is a Rork app compared to a native app?

Rork apps are React Native, which means they're slightly slower than Swift or Kotlin apps written by hand. In practice, unless you're building a demanding 3D game or real-time graphics app, users won't notice. Most Rork apps feel snappy and responsive.

### If Rork shuts down, what happens to my app?

Your app keeps working. You can export the source code anytime. Rork is a tool, not a service you depend on. Your app is yours.

### Do I own my app and its revenue?

100% yes. Rork isn't taking any cut. If you publish on the App Store and earn money, that's all you. The intellectual property is entirely yours.
