The restaurant industry is undergoing rapid digital transformation. From QR code menus to online reservations, demand for apps that enhance the guest experience has never been higher. With Rork Max, you can build a professional restaurant app without writing a single line of code.

## Defining Your Restaurant App Requirements

Before you start building, it helps to clarify what features you actually need. A well-rounded restaurant app typically covers:

- **Menu display**: Categorized menu listing with detail pages
- **Table reservations**: Date, time, party size, and seating type selection
- **Order management**: Mobile ordering from the table with kitchen notifications
- **Payments**: Card processing via Stripe
- **Staff dashboard**: Centralized view of reservations and active orders

This guide walks through building all of these features step by step.

## Supabase Schema Design

Supabase serves as the backend. Here's the table structure you'll need:

**menu_items table**

```
id            uuid        (primary key)
name          text        (item name)
description   text        (description)
price         integer     (price in cents)
category      text        (appetizer/main/drink/dessert)
image_url     text        (image URL)
is_available  boolean     (availability flag)
created_at    timestamp
```

**reservations table**

```
id            uuid        (primary key)
customer_name text        (guest name)
phone         text        (phone number)
party_size    integer     (number of guests)
reserved_at   timestamp   (reservation date/time)
table_type    text        (counter/table/private)
status        text        (pending/confirmed/cancelled)
notes         text        (special requests)
created_at    timestamp
```

**orders table**

```
id            uuid        (primary key)
table_number  text        (table identifier)
items         jsonb       (array of ordered items)
total_amount  integer     (total in cents)
status        text        (pending/cooking/ready/served/paid)
stripe_session_id text   (payment session ID)
created_at    timestamp
```

Once you've created these tables in the Supabase dashboard, configure Row Level Security (RLS) policies to require authentication for staff-side operations.

## Setting Up Your Rork Max Project

Open Rork Max and start a new project. The more specific your initial prompt, the better your starting point will be.

**Sample prompt:**

```
Build a mobile app for a restaurant with the following features:
1. Menu list screen with category filtering
2. Menu item detail screen (image, price, description)
3. Table reservation form (date, time, party size, seating type)
4. Ordering screen (add to cart, adjust quantities)
5. Order confirmation and payment screen
6. Staff dashboard (reservation and order management with status updates)

Backend: Supabase (I'll add credentials after generation)
Payments: Stripe
Design: Warm, inviting style with earthy brown tones
```

After Rork generates the initial app, add your Supabase project URL and anon key in the settings panel.

## Fetching Menu Data from Supabase

Once connected, tell Rork how to pull data from your tables:

```
Fetch menu items from the Supabase menu_items table.
Only show items where is_available is true.
Allow filtering by category using tab buttons at the top of the screen.
```

Rork will update the menu screen to display live data from your database.

## Building the Reservation Form

Reservations are a key feature. Use this prompt to set them up:

```
Implement the reservation form with:
- Date picker (calendar UI, past dates disabled)
- Time selector (5:00 PM to 10:00 PM, 30-minute intervals)
- Party size selector (1 to 20 guests)
- Seating type selection (counter / table / private room)
- Fields for guest name and phone number
Save submissions to the Supabase reservations table and show a confirmation screen after success.
```

Also add duplicate reservation checking so guests can't double-book the same time slot and seating type.

## Integrating Stripe Payments

Add your Stripe publishable key in Rork's settings, then prompt:

```
Add Stripe payment to the order confirmation screen.
Calculate the total, launch Stripe Checkout, and update the order status to 'paid' in Supabase after a successful payment.
Store the Stripe session ID in the stripe_session_id column.
```

Use the test card `4242 4242 4242 4242` with any future expiry date and any three-digit CVC to verify the integration before going live.

## Adding the Staff Dashboard

```
Add a staff dashboard screen with:
- Today's reservations sorted by time
- Active orders with table number, status, and elapsed time
- Buttons to update order status (cooking / ready / served)
- Login with email and password using Supabase Auth
Redirect unauthenticated users away from the dashboard.
```

Pair this with RLS policies in Supabase so that staff data is only accessible to authenticated users.

## Testing Your App

Once everything is connected, test each flow end to end. Seed a few menu items directly in Supabase and confirm they appear correctly in the app. Submit a test reservation and verify the data lands in the reservations table. Run through an order to payment using Stripe test mode, and confirm the order status updates correctly.

Pay special attention to payment failure handling — errors should be displayed clearly, and order status should not change until payment is confirmed.
