## Implementing the Flashcard System

The core of any flashcard app is the spaced repetition algorithm. Cards you answer correctly appear less frequently over time, while missed cards come back sooner. This approach is scientifically proven to improve long-term retention.

Use Rork's follow-up prompt feature to implement this logic:

```typescript
// Spaced repetition algorithm (simplified SM-2)
const calculateNextReview = (
  card: Card,
  isCorrect: boolean
): Card => {
  const now = new Date();
  let { difficulty, reviewCount, correctCount } = card;

  if (isCorrect) {
    correctCount += 1;
    // Lower difficulty and increase interval on correct answer
    difficulty = Math.max(1, difficulty - 1);
    const intervalDays = Math.pow(2, correctCount - 1); // 1, 2, 4, 8, 16 days...
    const nextDate = new Date(now.getTime() + intervalDays * 86400000);

    return {
      ...card,
      difficulty,
      reviewCount: reviewCount + 1,
      correctCount,
      nextReviewDate: nextDate.toISOString(),
    };
  } else {
    // Increase difficulty and schedule review for tomorrow
    difficulty = Math.min(5, difficulty + 1);
    const nextDate = new Date(now.getTime() + 86400000);

    return {
      ...card,
      difficulty,
      reviewCount: reviewCount + 1,
      correctCount: 0, // Reset consecutive correct streak
      nextReviewDate: nextDate.toISOString(),
    };
  }
};

// Expected output:
// Correct → difficulty: 2→1, nextReviewDate: 2 days later
// Incorrect → difficulty: 2→3, nextReviewDate: tomorrow, correctCount: reset to 0
```

## Building the Quiz Mode

Quiz mode uses your flashcard data to auto-generate multiple-choice questions, giving you an objective measure of how well you've retained the material.

```typescript
// Auto-generate quiz questions from deck data
const generateQuizQuestions = (
  deck: Deck,
  questionCount: number = 10
): QuizQuestion[] => {
  const cards = [...deck.cards];
  // Prioritize cards with higher difficulty
  cards.sort((a, b) => b.difficulty - a.difficulty);
  const selectedCards = cards.slice(0, questionCount);

  return selectedCards.map((card) => {
    // Pick 3 random wrong answers from other cards
    const otherCards = deck.cards.filter((c) => c.id !== card.id);
    const shuffled = otherCards.sort(() => Math.random() - 0.5);
    const wrongAnswers = shuffled.slice(0, 3).map((c) => c.back);

    // Shuffle all 4 options including the correct answer
    const options = [...wrongAnswers, card.back].sort(
      () => Math.random() - 0.5
    );

    return {
      question: card.front,
      correctAnswer: card.back,
      options,
      cardId: card.id,
    };
  });
};

// Expected output:
// [
//   {
//     question: "sustainable",
//     correctAnswer: "持続可能な",
//     options: ["持続可能な", "temporary", "economic", "innovative"],
//     cardId: "card-001"
//   },
//   ...
// ]
```

## Creating the Progress Dashboard

Seeing your own growth is one of the strongest motivators for sticking with a study habit. The progress dashboard visualizes daily study volume and accuracy trends over time.

```typescript
// Study log type and weekly summary calculation
interface StudyLog {
  date: string;        // YYYY-MM-DD
  cardsStudied: number;
  correctAnswers: number;
  totalAnswers: number;
  studyTimeMinutes: number;
}

// Calculate a 7-day summary from study logs
const getWeeklySummary = (logs: StudyLog[]): WeeklySummary => {
  const now = new Date();
  const weekAgo = new Date(now.getTime() - 7 * 86400000);

  const weekLogs = logs.filter(
    (log) => new Date(log.date) >= weekAgo
  );

  const totalCards = weekLogs.reduce(
    (sum, log) => sum + log.cardsStudied, 0
  );
  const totalCorrect = weekLogs.reduce(
    (sum, log) => sum + log.correctAnswers, 0
  );
  const totalAnswers = weekLogs.reduce(
    (sum, log) => sum + log.totalAnswers, 0
  );
  const totalMinutes = weekLogs.reduce(
    (sum, log) => sum + log.studyTimeMinutes, 0
  );

  return {
    totalCards,
    accuracy: totalAnswers > 0
      ? Math.round((totalCorrect / totalAnswers) * 100)
      : 0,
    totalMinutes,
    streak: calculateStreak(logs), // Consecutive study days
  };
};

// Expected output:
// { totalCards: 84, accuracy: 78, totalMinutes: 45, streak: 5 }
```

## Persisting Data Locally

Explicitly instructing Rork to handle data persistence with AsyncStorage ensures your users' progress is saved between sessions.

```typescript
import AsyncStorage from '@react-native-async-storage/async-storage';

// Save and load deck data
const STORAGE_KEY = '@language_app_decks';

const saveDecks = async (decks: Deck[]): Promise<void> => {
  try {
    const jsonValue = JSON.stringify(decks);
    await AsyncStorage.setItem(STORAGE_KEY, jsonValue);
  } catch (error) {
    console.error('Failed to save decks:', error);
  }
};

const loadDecks = async (): Promise<Deck[]> => {
  try {
    const jsonValue = await AsyncStorage.getItem(STORAGE_KEY);
    return jsonValue != null ? JSON.parse(jsonValue) : [];
  } catch (error) {
    console.error('Failed to load decks:', error);
    return [];
  }
};

// Expected output: Deck data persists across app restarts
```

## Frequently Asked Questions

### Q1: Is there a limit to how many cards I can add to a deck?

The practical limit comes from AsyncStorage capacity — around 6MB on Android, with no hard limit on iOS. Since each card takes just a few hundred bytes, you can comfortably store thousands of cards. For larger datasets, consider integrating an external database like Supabase.

### Q2: Can I add text-to-speech functionality?

Absolutely. Expo's `expo-speech` library makes it straightforward. Tell Rork to "add a text-to-speech feature that reads the front and back of each card when tapped, using expo-speech." It will generate the pronunciation in the target language automatically.

### Q3: Can I share decks with other users?

Yes, with some additional setup. Start by implementing a JSON export/import feature for decks. Then, if you want real-time sharing, connect Supabase as your backend and let users publish and subscribe to shared decks.

### Q4: How can I monetize this app?

Language learning apps lend themselves well to several revenue models: selling premium deck packs via Stripe, displaying ads in the free tier, or offering a subscription that delivers new decks monthly. With Rork Max, you can publish to the App Store and implement in-app purchases seamlessly.
