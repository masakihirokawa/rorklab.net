## Step 2: Understanding the Recording Mechanism

At the heart of the generated code is the `expo-av` library, which handles all audio recording. Understanding how it works will help you customize the app later.

### The Basic Recording Flow

```typescript
import { Audio } from 'expo-av';

// Request microphone permission
const { granted } = await Audio.requestPermissionsAsync();

// Configure audio mode
await Audio.setAudioModeAsync({
  allowsRecordingIOS: true,
  playsInSilentModeIOS: true,
});

// Start recording
const { recording } = await Audio.Recording.createAsync(
  Audio.RecordingOptionsPresets.HIGH_QUALITY // High quality audio
);

// Stop recording and get the file URI
await recording.stopAndUnloadAsync();
const uri = recording.getURI(); // Local file path
```

The key here is `Audio.RecordingOptionsPresets.HIGH_QUALITY`, which captures clear, detailed audio. You can switch to `LOW_QUALITY` if you want to save storage space.

### A Custom Recording Hook

Here's an enhanced version of the recording hook that also tracks recording duration:

```typescript
// hooks/useAudioRecorder.ts
import { useState, useRef, useCallback } from 'react';
import { Audio } from 'expo-av';

export function useAudioRecorder() {
  const [isRecording, setIsRecording] = useState(false);
  const [duration, setDuration] = useState(0);
  const recordingRef = useRef<Audio.Recording | null>(null);
  const timerRef = useRef<NodeJS.Timeout | null>(null);

  // Start recording
  const startRecording = useCallback(async () => {
    const { granted } = await Audio.requestPermissionsAsync();
    if (!granted) return;

    await Audio.setAudioModeAsync({
      allowsRecordingIOS: true,
      playsInSilentModeIOS: true,
    });

    const { recording } = await Audio.Recording.createAsync(
      Audio.RecordingOptionsPresets.HIGH_QUALITY
    );

    recordingRef.current = recording;
    setIsRecording(true);
    setDuration(0);

    // Track recording duration
    timerRef.current = setInterval(() => {
      setDuration(prev => prev + 1);
    }, 1000);
  }, []);

  // Stop recording
  const stopRecording = useCallback(async () => {
    if (!recordingRef.current) return null;

    if (timerRef.current) clearInterval(timerRef.current);

    await recordingRef.current.stopAndUnloadAsync();
    const uri = recordingRef.current.getURI();

    recordingRef.current = null;
    setIsRecording(false);

    return { uri, duration };
  }, [duration]);

  return { isRecording, duration, startRecording, stopRecording };
}
```

## Step 3: Implementing Playback

Playing back recorded memos uses the `Sound` object from `expo-av`. Here's how it works:

```typescript
// hooks/useAudioPlayer.ts
import { useState, useRef, useCallback } from 'react';
import { Audio } from 'expo-av';

export function useAudioPlayer() {
  const [isPlaying, setIsPlaying] = useState(false);
  const [position, setPosition] = useState(0);
  const [totalDuration, setTotalDuration] = useState(0);
  const soundRef = useRef<Audio.Sound | null>(null);

  // Load and play audio
  const play = useCallback(async (uri: string) => {
    if (soundRef.current) {
      await soundRef.current.unloadAsync();
    }

    const { sound } = await Audio.Sound.createAsync(
      { uri },
      { shouldPlay: true },
      (status) => {
        if (status.isLoaded) {
          setPosition(status.positionMillis);
          setTotalDuration(status.durationMillis ?? 0);
          setIsPlaying(status.isPlaying);

          if (status.didJustFinish) {
            setIsPlaying(false);
            setPosition(0);
          }
        }
      }
    );

    soundRef.current = sound;
  }, []);

  // Toggle pause / resume
  const togglePause = useCallback(async () => {
    if (!soundRef.current) return;
    if (isPlaying) {
      await soundRef.current.pauseAsync();
    } else {
      await soundRef.current.playAsync();
    }
  }, [isPlaying]);

  return { isPlaying, position, totalDuration, play, togglePause };
}
```

## Step 4: Adding Transcription with the Whisper API

This is where the app really stands out. By integrating OpenAI's Whisper API, every recording is automatically converted to searchable text.

### How Whisper Works

Whisper is OpenAI's speech recognition model that supports multiple languages with high accuracy. You send an audio file and receive the transcribed text in return.

```typescript
// utils/transcribe.ts

const OPENAI_API_KEY = 'your-api-key-here'; // Use environment variables in production

export async function transcribeAudio(uri: string): Promise<string> {
  const formData = new FormData();
  formData.append('file', {
    uri,
    type: 'audio/m4a',
    name: 'recording.m4a',
  } as any);
  formData.append('model', 'whisper-1');
  formData.append('language', 'ja'); // Set to 'en' for English

  const response = await fetch(
    'https://api.openai.com/v1/audio/transcriptions',
    {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${OPENAI_API_KEY}`,
      },
      body: formData,
    }
  );

  const data = await response.json();
  return data.text;
}

// Usage:
// const text = await transcribeAudio(recordingUri);
// Expected output: "In today's meeting we discussed progress on the new project"
```

### Auto-Transcribe on Recording Stop

Wire the transcription into the recording flow so it happens automatically:

```typescript
const handleStopRecording = async () => {
  const result = await stopRecording();
  if (!result) return;

  // Run transcription in the background
  const transcription = await transcribeAudio(result.uri);

  // Save as a new memo
  const newMemo = {
    id: Date.now().toString(),
    title: transcription.slice(0, 30) + '...', // Use first 30 chars as title
    uri: result.uri,
    duration: result.duration,
    transcription,
    createdAt: new Date().toISOString(),
  };

  await saveMemo(newMemo);
};
```

## Step 5: Persisting Data with AsyncStorage

Memo metadata is stored locally using `AsyncStorage`. The actual audio files live on the device's file system, and we store their paths as part of each memo's metadata.

```typescript
// utils/storage.ts
import AsyncStorage from '@react-native-async-storage/async-storage';

const STORAGE_KEY = 'voice_memos';

interface VoiceMemo {
  id: string;
  title: string;
  uri: string;
  duration: number;
  transcription: string;
  createdAt: string;
}

// Retrieve all memos
export async function getMemos(): Promise<VoiceMemo[]> {
  const data = await AsyncStorage.getItem(STORAGE_KEY);
  return data ? JSON.parse(data) : [];
}

// Save a new memo
export async function saveMemo(memo: VoiceMemo): Promise<void> {
  const memos = await getMemos();
  memos.unshift(memo); // Newest first
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(memos));
}

// Delete a memo
export async function deleteMemo(id: string): Promise<void> {
  const memos = await getMemos();
  const filtered = memos.filter(m => m.id !== id);
  await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(filtered));
}
```

### The Record Button Animation

A pulsing animation is a simple but effective way to communicate that recording is in progress:

```typescript
import { useEffect, useRef } from 'react';
import { Animated, Pressable, StyleSheet } from 'react-native';

function RecordButton({ isRecording, onPress }: {
  isRecording: boolean;
  onPress: () => void;
}) {
  const scale = useRef(new Animated.Value(1)).current;

  useEffect(() => {
    if (isRecording) {
      Animated.loop(
        Animated.sequence([
          Animated.timing(scale, {
            toValue: 1.2,
            duration: 500,
            useNativeDriver: true,
          }),
          Animated.timing(scale, {
            toValue: 1.0,
            duration: 500,
            useNativeDriver: true,
          }),
        ])
      ).start();
    } else {
      scale.setValue(1);
    }
  }, [isRecording]);

  return (
    <Pressable onPress={onPress}>
      <Animated.View
        style={[
          styles.button,
          isRecording && styles.recording,
          { transform: [{ scale }] },
        ]}
      />
    </Pressable>
  );
}

const styles = StyleSheet.create({
  button: {
    width: 80,
    height: 80,
    borderRadius: 40,
    backgroundColor: '#FF3B30',
  },
  recording: {
    backgroundColor: '#FF6B6B',
  },
});
```

## Common Issues & Fixes

### Microphone Permission Denied

Both iOS and Android require explicit permission to use the microphone. Make sure to handle the case where the user denies the request:

```typescript
const { granted } = await Audio.requestPermissionsAsync();
if (!granted) {
  Alert.alert(
    'Microphone Access Required',
    'Please enable microphone access in your device settings.',
    [{ text: 'OK' }]
  );
  return;
}
```

### Recording Doesn't Work on Simulator

The iOS Simulator may not support microphone input. Always test audio features on a physical device. If you're using Rork Max, the [Rork Companion Mac App](/en/articles/rork-dev/rork-companion-mac-app) makes on-device testing quick and easy.

### Audio Files Are Too Large

Long recordings can consume significant storage. Consider using `Audio.RecordingOptionsPresets.LOW_QUALITY` for less critical recordings, or set a maximum recording length (e.g., 5 minutes) to keep file sizes manageable.

## FAQ

### Q1: Can I do transcription without the Whisper API?

Yes. You can use the `@react-native-voice/voice` native module for on-device speech recognition. However, Whisper offers superior accuracy and multilingual support. For the best user experience, the API-based approach is recommended.

### Q2: Can I store recordings in the cloud?

Absolutely. You can integrate with Supabase Storage or Firebase Cloud Storage to sync recordings across devices. For a detailed walkthrough, see the [Supabase and Firebase Integration Guide](/en/articles/rork-dev/supabase-firebase-backend).

### Q3: Does it work offline?

Recording and playback work fully offline. However, Whisper API transcription requires an internet connection. If you need offline transcription, consider using an on-device speech recognition model as a fallback.

### Q4: Can I build this on Rork's free plan?

The core recording and playback features can be built on the free plan. However, custom code editing and native API access may require Rork Max. For full pricing details, check the [Rork Pricing Plans Guide](/en/articles/rork-business/rork-pricing-plans).
