## Why iPad Support Matters

When you run an iPhone app on an iPad without proper optimization, the result is either a tiny interface floating in the center of a large screen or a stretched-out version that looks awkward. App Store reviewers pay close attention to iPad compatibility, and poor iPad support consistently drags down user ratings.

iPads are everywhere — from classrooms to corporate boardrooms. The 2026 iPad lineup (iPad mini, iPad Air, iPad Pro 11" and 13") spans a wide range of screen sizes, and users expect your app to handle multitasking features like Split View and Slide Over gracefully.

With Rork, you can describe your iPad layout requirements in a prompt, and the AI handles most of the responsive layout generation automatically.

---

## Specifying iPad Layouts in Rork

### Writing Effective Prompts

When asking Rork to build an iPad-compatible app, include specific layout instructions for different screen sizes.

**Basic prompt example:**

```
Build a notes app. On iPad, use a two-column layout with a list of notes
on the left and note details on the right. On iPhone, show the list and
details as separate screens.
```

When you specify "on iPad... on iPhone..." explicitly, Rork generates responsive layouts using `useWindowDimensions` and Flexbox-based responsive patterns.

### Screen Size Detection

The code Rork generates typically uses React Native's `useWindowDimensions` hook to determine the current screen width and switch layouts conditionally.

```typescript
const { width } = useWindowDimensions();
const isTablet = width >= 768;
```

This simple check enables two-column layouts on tablets while keeping a single-column stack on phones.

## Implementing Multi-Column Navigation

One of the most effective iPad patterns is multi-column navigation — the master/detail layout familiar from Apple's own Mail, Notes, and Settings apps.

### Prompt Examples

```
Build a task management app.
On iPad in landscape, use a three-column layout:
- Left sidebar (200px): project list
- Middle column (300px): task list
- Right main area: task details and editing
On iPhone, use standard stack navigation.
```

Rork generates responsive layout components using the `Dimensions` API. You can fine-tune column widths and collapse behavior through follow-up prompts.

### Switching Between Tab Bar and Sidebar

On iPhone, a tab bar sits at the bottom of the screen. On iPad, converting that tab bar into a left sidebar is a common and effective pattern.

```
On iPad, replace the tab bar with a left sidebar navigation.
Include icons and labels, and highlight the selected tab.
```

## Handling Split View and Slide Over

Many iPad users work with two apps side by side using Split View. If your app doesn't handle width changes properly, the layout breaks or becomes unusable.

### How Rork Handles This

Since Rork generates React Native (Expo) apps, iOS multitasking features are supported at the framework level. However, layouts with hard-coded widths will break when Split View resizes the available width.

**Prompt tips:**

```
Use percentage-based widths instead of fixed pixel values for all layouts.
Make sure the layout looks correct when the screen width changes in Split View.
The layout should remain usable down to a minimum width of 320px.
```

Rork responds to these instructions by generating `flex`-based styles and percentage widths throughout the app.

## Native iPad Features with Rork Max

Rork Max gives you access to Swift-native code, unlocking iPad-specific capabilities that go beyond what React Native offers.

### Apple Pencil Support

With Rork Max, you can use the PencilKit framework to implement handwriting input and sketching features powered by Apple Pencil.

```
Build a drawing app. Display a full-screen canvas that supports Apple Pencil.
Line thickness should respond to pressure. Include an eraser tool and color picker.
```

Rork Max generates a native drawing view using `PKCanvasView` with pressure sensitivity and palm rejection enabled by default.

### Drag and Drop

iPad supports drag and drop between apps. Users can drag a photo from the Photos app and drop it into yours, for example.

```
Build a photo organizer app. Allow adding images via drag and drop from other apps.
Also support rearranging photos within the app using drag gestures.
```

### Keyboard Shortcuts

More and more iPad users connect external keyboards. Rork Max makes it straightforward to add keyboard shortcuts using `UIKeyCommand`.

```
In the text editor app, add keyboard shortcuts:
Cmd+S to save, Cmd+B for bold, Cmd+Z for undo.
```

## Responsive Images and Font Sizes

### Image Optimization

Images designed for iPhone can look blurry on the iPad's high-resolution display.

**Prompt example:**

```
Resize images based on the device's screen width.
Use high-resolution images on iPad and smaller images on iPhone to save bandwidth.
```

### Dynamic Type Support

Your app should respect the system font size that users set in their accessibility settings.

```
Follow the system's Dynamic Type settings for all text sizes.
On iPad's larger screen, increase the base font size slightly for readability.
```

## App Store Review Tips for iPad Apps

When submitting an iPad-compatible app to the App Store, you need to provide screenshots at specific sizes.

**Required screenshot dimensions:**

- iPad Pro 13-inch (2048 × 2732 px)
- iPad Pro 11-inch (1668 × 2388 px)

You can capture these using the Rork Max Companion App on a physical iPad or through the iOS Simulator.

### Common Rejection Reasons

- UI that simply stretches the iPhone layout without adaptation
- Broken layouts in landscape orientation
- Crashes when using Split View
- Touch targets that are too small (minimum 44×44pt required)

Address each of these explicitly in your prompts and Rork will generate the appropriate handling code.

## Hands-On Tutorial: Building an iPad-Ready Notes App

Let's walk through building an iPad-optimized notes app in Rork from start to finish.

### Step 1: Initial Prompt

```
Build a notes app with these requirements:

[Shared Features]
- Create, edit, and delete notes
- Markdown support
- Search functionality

[iPad Layout]
- Two columns: note list on the left, editor on the right
- Sidebar width is 30% of the screen
- Support both landscape and portrait orientations

[iPhone Layout]
- Separate list and editor screens
- Navigation with a back button
```

### Step 2: Preview and Adjust

Check both iPhone and iPad previews in Rork. If the layout needs tweaking, send follow-up prompts.

```
Make the iPad sidebar collapsible.
When collapsed, show a hamburger menu to reopen it.
```

### Step 3: Polish

```
Add iPad keyboard shortcuts:
- Cmd+N for new note
- Cmd+F to search
- Cmd+Delete to delete note (with confirmation dialog)
```

## Frequently Asked Questions

### Can I add iPad support to an existing iPhone app?

Yes. In Rork, you can open your existing project and tell it to "make this app work well on iPad too." Rork will suggest and generate responsive layout changes to adapt your existing screens for larger displays.

### Can I build iPad apps with the free version of Rork?

Basic iPad support through React Native responsive layouts works with the free tier. However, native features like Apple Pencil input and drag-and-drop require Rork Max.

### Can I design completely different interfaces for iPad and iPhone?

Absolutely. Specify separate layouts for iPad and iPhone in your prompt, and Rork generates conditional UI code that renders entirely different interfaces based on the screen size.

### Should I support landscape orientation on iPad?

It's strongly recommended. A large portion of iPad users prefer landscape orientation, and App Store reviewers check landscape display. Include "support both landscape and portrait" in your prompts.

### Is Split View support mandatory?

It's not strictly required, but apps without Split View support tend to receive lower ratings. When you use percentage-based layouts in Rork, Split View compatibility comes almost for free.
