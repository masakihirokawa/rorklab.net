## Introduction — Learn App Development Fundamentals by Building a Shopping List

A shopping list is one of the most universally useful apps you can build. Behind its simple appearance lies every fundamental concept of app development: creating, reading, updating, and deleting data — commonly known as CRUD operations.

In this tutorial, you'll build a fully functional shopping list app from scratch using Rork. No programming experience required. With Rork, you describe what you want in plain language, and AI generates a working app for you.

Here's what you'll learn:

- How to build a list-based app with Rork
- Implementing add, edit, delete, and check-off functionality
- Organizing items by category with filtering
- Persisting data locally with AsyncStorage
- Polishing your UI for a store-ready look

This tutorial is designed for anyone getting started with Rork or building their first app. If you're already comfortable with the basics, you might also enjoy [Your First App in 30 Minutes](/en/articles/rork-basics/rork-first-app-30min) for a quick-start refresher.

## Prerequisites and Setup

### What You'll Need

Before you start building, make sure you have the following ready:

- **A Rork account**: Sign up for free at [rork.com](https://rork.com/). The free plan lets you generate up to 5 apps per week
- **A modern browser**: Chrome, Safari, or Edge will all work
- **A smartphone (optional)**: Install the Rork Companion app to preview your creation on a real device

### How Rork Works

Building apps with Rork is a conversation. You describe what you want in natural language, and the AI generates React Native code automatically. Think of it as pair-programming with an AI that handles all the heavy lifting.

If you're brand new to Rork, [Zero IT Knowledge to App Developer](/en/articles/rork-basics/rork-zero-to-app-complete-guide) is a great place to start before diving into this tutorial.

## Step 1: Create Your Project

Log into Rork and create a new project. Type the following prompt into the input field:

```text
Build a shopping list app.

Features:
- Add items with a name, quantity, and category
- Tap items to mark them as checked
- Swipe to delete items
- Filter by category (Produce, Meat & Fish, Dairy, Beverages, Household, Other)
- Save data locally on the device

Design:
- Clean white background
- Color-coded category badges
- Strikethrough text for checked items
- Floating add button at the bottom of the screen
```

From this single prompt, Rork will generate a complete app with screen layouts, data management, and interactive features.

### How Autocomplete Works

The autocomplete feature Rork generates saves your input history to AsyncStorage and filters it as you type:

```typescript
// Input history storage (example of Rork-generated code)
import AsyncStorage from '@react-native-async-storage/async-storage';

const HISTORY_KEY = '@shopping_history';

// Add a new item name to history
const addToHistory = async (itemName: string) => {
  const history = await getHistory();
  // Remove duplicates and keep the 50 most recent entries
  const updated = [itemName, ...history.filter(h => h !== itemName)].slice(0, 50);
  await AsyncStorage.setItem(HISTORY_KEY, JSON.stringify(updated));
};

// Search history for matching suggestions
const searchHistory = async (query: string): Promise<string[]> => {
  const history = await getHistory();
  return history.filter(item =>
    item.toLowerCase().includes(query.toLowerCase())
  );
};
```

## Step 4: Ensure Data Persistence

The most critical feature of a shopping list app is that your data survives an app restart. Rork uses AsyncStorage for local persistence by default, but let's make sure everything works correctly:

```text
Please verify and improve data persistence:

- Items should persist when the app is closed and reopened
- Save automatically after every change (no manual save button needed)
- Show a toast message if saving fails
```

### The AsyncStorage Pattern

Here's what the data persistence code Rork generates typically looks like:

```typescript
// Data persistence (example of Rork-generated code)
const STORAGE_KEY = '@shopping_list';

const saveItems = async (items: ShoppingItem[]) => {
  try {
    await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(items));
  } catch (error) {
    console.error('Failed to save items:', error);
    // Show a toast notification
    Alert.alert('Error', 'Failed to save your list');
  }
};

// Load data when the app starts
const loadItems = async (): Promise<ShoppingItem[]> => {
  try {
    const json = await AsyncStorage.getItem(STORAGE_KEY);
    return json ? JSON.parse(json) : [];
  } catch (error) {
    console.error('Failed to load items:', error);
    return [];
  }
};
```

With all the features in place, let's give the app a professional finish:

Improve the design with these changes:

### Design Tips for Rork

The key to getting polished results from Rork is specificity. Instead of vague instructions like "make it pretty," describe concrete details: "white background with green accent color," "cards with 8px border radius." The more specific your prompt, the closer the output will match your vision.

## Frequently Asked Questions

### Q: Can I sync my shopping list across multiple devices?

The initial version uses AsyncStorage, which stores data locally on a single device. To enable cross-device syncing, you can ask Rork to integrate a backend like Supabase. Try prompting: "Connect to Supabase so logged-in users can sync their lists across devices." Rork will handle the database setup and authentication flow.

### Q: Can I publish this app to the App Store or Google Play?

Absolutely. With a Rork Max plan, you can submit your app directly to the App Store and Google Play. Even on the free plan, you can test on your own iPhone using the Rork Companion app. Check Rork's official documentation for detailed submission instructions.

### Q: How do I let users create their own custom categories?

Simply tell Rork: "Add a settings screen where users can create, edit, and delete custom categories with custom colors and icons." Rork will generate a complete settings screen with category management built in.
